### Title
Missing Unconditional `nOpt /= 0` Sanity Check in `ppuWellFormed` Allows Malformed Governance Proposal Submission in Conway Era — (`eras/conway/impl/src/Cardano/Ledger/Conway/PParams.hs`)

---

### Summary

The `ppuWellFormed` function in Conway era only validates `nOpt /= 0` for protocol version ≥ 11. For protocol versions 9–10 (the initial Conway deployment window), a `ParameterChange` governance proposal setting `nOpt = 0` passes the well-formedness check and is accepted into the governance system without being rejected as `MalformedProposal`. If such a proposal is subsequently ratified, it causes a runtime `error "nOpt is zero"` inside `mkPoolRewardInfo` at every epoch boundary, crashing all honest nodes.

---

### Finding Description

`ppuWellFormed` in `ConwayEra` guards the `nOpt` field conditionally:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/PParams.hs, lines 951-952
, pvMajor pv < natVersion @11
    || isValid (/= 0) ppuNOptL
```

The boolean short-circuits to `True` whenever `pvMajor pv < 11`, meaning the `nOpt /= 0` predicate is **never evaluated** for Conway protocol versions 9 or 10. A `ParameterChange` governance action carrying `ppuNOptL = SJust 0` therefore passes `actionWellFormed` and is inserted into the proposal tree without triggering `MalformedProposal`. [1](#0-0) 

The `actionWellFormed` function in the GOV rule delegates entirely to `ppuWellFormed` for `ParameterChange` actions:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Gov.hs, lines 393-399
actionWellFormed pv ga = failureUnless isWellFormed $ MalformedProposal ga
  where
    isWellFormed = case ga of
      ParameterChange _ ppd _ -> ppuWellFormed pv ppd
      _ -> True
``` [2](#0-1) 

The well-formedness check is invoked at proposal submission time: [3](#0-2) 

Once `nOpt = 0` is enacted, the epoch-boundary reward calculation calls `mkPoolRewardInfo`, which contains:

```haskell
-- eras/shelley/impl/src/Cardano/Ledger/Shelley/Rewards.hs, line 384
pp_nOpt = (pp ^. ppNOptL) `nonZeroOr` error "nOpt is zero"
``` [4](#0-3) 

`nonZeroOr` returns its second argument when the first is zero. Because the second argument is `error "nOpt is zero"`, evaluating `pp_nOpt` throws a Haskell exception, crashing every node that attempts to compute rewards at the epoch boundary.

The contrast with the Dijkstra era is instructive: its `ppuWellFormed` drops the protocol-version guard entirely, making the `nOpt /= 0` check unconditional:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/PParams.hs, line 556
, isValid (/= 0) ppuNOptL
``` [5](#0-4) 

This confirms the Conway guard is an oversight, not intentional design.

---

### Impact Explanation

**Medium** — An unprivileged actor (any user who can pay the governance action deposit) can submit a `ParameterChange` proposal with `nOpt = 0` in Conway era (protocol version 9 or 10). This proposal bypasses the intended `ppuWellFormed` validation gate and is accepted into the on-chain proposal tree. The proposal itself constitutes an attacker-controlled input that exceeds intended validation limits. If the proposal is subsequently ratified, the impact escalates to **High**: every honest node crashes at the next epoch boundary when `mkPoolRewardInfo` evaluates `error "nOpt is zero"`, permanently halting the chain until a hard fork is applied.

---

### Likelihood Explanation

Submission is trivially reachable by any user willing to pay the governance action deposit — no special role or key is required. Ratification requires governance majority (DReps + SPOs + Committee), which is a higher bar, but the submission-time bypass of `ppuWellFormed` is unconditionally reachable by an unprivileged actor and represents the direct root cause.

---

### Recommendation

Remove the protocol-version guard from the `nOpt` check in `ppuWellFormed` for `ConwayEra`, making it unconditional, consistent with the Dijkstra implementation:

```haskell
-- Replace:
, pvMajor pv < natVersion @11
    || isValid (/= 0) ppuNOptL
-- With:
, isValid (/= 0) ppuNOptL
``` [6](#0-5) 

---

### Proof of Concept

1. In Conway era with `pvMajor pv = 9` or `10`, construct a `ParameterChange` governance action:
   ```
   ParameterChange SNothing (emptyPParamsUpdate & ppuNOptL .~ SJust 0) policy
   ```
2. Submit the proposal. The GOV rule calls `actionWellFormed pv (ParameterChange ...)`, which calls `ppuWellFormed pv ppu`. Because `pvMajor pv < 11`, the `nOpt` predicate is skipped and `ppuWellFormed` returns `True`.
3. The proposal is accepted into the governance system (no `MalformedProposal` failure).
4. If ratified and enacted via `ENACT`, `ensCurPParams` is updated with `nOpt = 0`.
5. At the next epoch boundary, `mkPoolRewardInfo` evaluates `(pp ^. ppNOptL) `nonZeroOr` error "nOpt is zero"` with `ppNOptL = 0`, triggering the `error` and crashing every node. [7](#0-6) [8](#0-7)
