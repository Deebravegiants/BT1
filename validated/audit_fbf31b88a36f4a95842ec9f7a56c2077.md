Audit Report

## Title
Rewards Permanently Locked When `totalKernelStaked` Drops to Zero During Active Reward Period - (File: `contracts/KERNEL/KernelDepositPool.sol`)

## Summary
In `KernelDepositPool.sol`, the `updateReward` modifier unconditionally advances `updatedAt` to `lastTimeRewardApplicable()` even when `totalKernelStaked == 0`. When all stakers exit during an active reward window, the elapsed time is silently skipped on the next stake, and the corresponding `rewardRate × elapsed` tokens are permanently locked in the contract with no recovery path. The contract's own NatSpec at L18–22 explicitly acknowledges this behavior.

## Finding Description
The `rewardPerToken()` function freezes at `rewardPerTokenStored` whenever `totalKernelStaked == 0`:

```solidity
// L408-414
function rewardPerToken() public view returns (uint256) {
    if (totalKernelStaked == 0) {
        return rewardPerTokenStored;
    }
    return rewardPerTokenStored + (rewardRate * (lastTimeRewardApplicable() - updatedAt) * DECIMAL_PRECISION)
        / totalKernelStaked;
}
```

The `updateReward` modifier always advances `updatedAt` regardless:

```solidity
// L232-242
modifier updateReward(address _account) {
    rewardPerTokenStored = rewardPerToken();
    updatedAt = lastTimeRewardApplicable();  // always advances
    ...
    _;
}
```

Exploit sequence:

1. Alice is the sole staker. She calls `initiateWithdrawal(totalStake)`. The modifier runs first (while `totalKernelStaked > 0`), correctly checkpointing `rewardPerTokenStored` and setting `updatedAt = T1`. Then the function body sets `totalKernelStaked = 0`.
2. Time advances from T1 to T2 (e.g., 15 days). `rewardRate` is still active (`finishAt > T2`). `rewardPerToken()` returns the frozen `rewardPerTokenStored` for the entire interval.
3. Bob calls `stake`. The modifier runs: `rewardPerToken()` still returns frozen `rewardPerTokenStored` (because `totalKernelStaked` is still 0 at modifier execution time), but `updatedAt` is advanced to T2. Then the function body sets `totalKernelStaked = Bob's amount`.
4. All future `rewardPerToken()` calls compute from T2 onward. The `rewardRate × (T2 - T1)` tokens are permanently stranded — no function in the contract can recover or redistribute them.

The `notifyRewardAmount` guard at L570 (`if (totalKernelStaked == 0) revert NoStakedTokens()`) only prevents starting a new period with zero stakers; it does not prevent stakers from all exiting mid-period, and it provides no recovery mechanism for already-locked rewards.

## Impact Explanation
**Medium — Permanent freezing of unclaimed yield.** Reward tokens (`rewardsToken`) become permanently irrecoverable in the contract. The locked amount equals `rewardRate × (duration of zero-staked interval)`. For a 30-day reward window, a zero-staked period of even a few days can represent a material fraction of the total reward allocation. There is no admin rescue function, no rollover mechanism, and no way to reclaim these tokens.

## Likelihood Explanation
No privileged access is required. Any combination of stakers calling the public `initiateWithdrawal` function is sufficient to drain `totalKernelStaked` to zero. This can occur organically (mass loss of confidence, coordinated exit) or accidentally. The `withdrawalDelay` is admin-configurable down to 1 second (only the upper bound of 30 days is enforced), making rapid exits feasible. The scenario is repeatable across every reward period.

## Recommendation
Advance `updatedAt` only when there are stakers to receive rewards, so the zero-staked interval is not silently consumed:

```solidity
modifier updateReward(address _account) {
    rewardPerTokenStored = rewardPerToken();
    if (totalKernelStaked > 0) {
        updatedAt = lastTimeRewardApplicable();
    }
    if (_account != address(0)) {
        rewards[_account] = earned(_account);
        userRewardPerTokenPaid[_account] = rewardPerTokenStored;
    }
    _;
}
```

This preserves the un-advanced `updatedAt` so that when the next staker arrives, the full elapsed interval (including the zero-staked gap) is correctly accounted for and distributed going forward.

## Proof of Concept

Foundry test sequence (local fork or unit test):

1. Deploy `KernelDepositPool`, set `duration = 30 days`, call `notifyRewardAmount(1_000e18)` with Alice staked for `100e18`. Record `rewardRate = 1_000e18 / 30 days`.
2. Warp 15 days. Alice calls `initiateWithdrawal(100e18)`. Assert `totalKernelStaked == 0`, `updatedAt == block.timestamp` (T1 = start + 15 days).
3. Warp another 15 days (T2 = start + 30 days, i.e., `finishAt`). Assert `rewardPerToken() == rewardPerTokenStored` (frozen).
4. Bob calls `stake(1e18)`. Assert `updatedAt == finishAt` (T2), `rewardPerTokenStored` unchanged.
5. Bob calls `getReward()`. Assert Bob earned only rewards from T2 onward (≈ 0, since `finishAt` was reached).
6. Assert `rewardsToken.balanceOf(address(pool)) ≈ 500e18` (the 15-day zero-staked portion), with no mechanism to withdraw it. Confirm the contract holds excess reward tokens that can never be claimed. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
