Audit Report

## Title
Zero wrsETH Minted on Dust Deposits Due to Missing Zero-Amount Output Guard - (File: contracts/pools/RSETHPoolV3.sol)

## Summary
All four L2 pool deposit functions (`RSETHPoolV3`, `RSETHPoolNoWrapper`, `RSETHPoolV3ExternalBridge`, `RSETHPoolV3WithNativeChainBridge`) compute the wrsETH/rsETH mint amount via integer division that truncates to zero for small inputs, yet no guard reverts when the result is zero. A user who sends a non-zero dust amount of ETH or tokens receives zero wrsETH in return, with their deposited assets permanently absorbed into the pool.

## Finding Description
In `RSETHPoolV3.viewSwapRsETHAmountAndFee` (line 307), the ETH-path share calculation is:

```solidity
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
```

`rsETHToETHrate` is the rsETH/ETH exchange rate in 1e18 precision, starting at 1e18 and growing monotonically as yield accrues. Once `rsETHToETHrate > 1e18` (the normal post-launch state), any deposit where `amountAfterFee * 1e18 < rsETHToETHrate` produces `rsETHAmount = 0` due to Solidity integer division truncation.

The deposit function then proceeds unconditionally:

```solidity
// RSETHPoolV3.sol lines 256-262
if (amount == 0) revert InvalidAmount();
(uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
feeEarnedInETH += fee;
wrsETH.mint(msg.sender, rsETHAmount);  // mints 0 — no revert
```

The only input guard is `if (amount == 0) revert InvalidAmount()`, which does not protect against zero output. The same pattern is confirmed in all four pool variants:
- `RSETHPoolV3.sol` lines 256–262, 307
- `RSETHPoolNoWrapper.sol` lines 235–241, 285
- `RSETHPoolV3ExternalBridge.sol` lines 375–381, 426
- `RSETHPoolV3WithNativeChainBridge.sol` lines 292–298, 343

## Impact Explanation
This matches the allowed Low impact: **Contract fails to deliver promised returns, but doesn't lose value** (from the protocol's perspective). A user depositing a dust amount (e.g., 1 wei) when `rsETHToETHrate > 1e18` has their ETH accepted, `wrsETH.mint(msg.sender, 0)` is called silently, and the deposited ETH is permanently absorbed into the pool balance, accruing to existing wrsETH holders. The per-transaction loss is bounded to dust amounts (sub-wei threshold), making this Low severity.

## Likelihood Explanation
The condition is reachable by any unprivileged caller via the public `deposit` functions on any deployed L2 pool. No special role, front-running, or external dependency compromise is required. The condition activates as soon as `rsETHToETHrate` exceeds 1e18, which is the normal operating state of the protocol after any yield has accrued. Any user who accidentally sends 1 wei (scripting error, test transaction, wallet rounding) will silently lose it with no revert and no on-chain indication of failure.

## Recommendation
Add a zero-output guard immediately after computing `rsETHAmount` in each deposit function across all four pool variants:

```solidity
(uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
if (rsETHAmount == 0) revert InvalidAmount();
feeEarnedInETH += fee;
wrsETH.mint(msg.sender, rsETHAmount);
```

Apply the same guard to the token deposit path (`deposit(address, uint256, string)`) in all four contracts.

## Proof of Concept
Assume `rsETHToETHrate = 1.05e18` (5% yield accrued, realistic near-term value):

1. Alice calls `RSETHPoolV3.deposit{value: 1}("")` (1 wei ETH).
2. `fee = 1 * feeBps / 10_000 = 0` (rounds down for any reasonable `feeBps`).
3. `amountAfterFee = 1 - 0 = 1`.
4. `rsETHAmount = 1 * 1e18 / 1.05e18 = 0` (Solidity truncation).
5. `feeEarnedInETH += 0`.
6. `wrsETH.mint(Alice, 0)` — Alice receives 0 wrsETH.
7. Alice's 1 wei is permanently held in the pool.

**Foundry test plan:**
```solidity
function test_dustDepositMintsZero() public {
    // Fork with rsETHToETHrate = 1.05e18
    vm.mockCall(oracle, abi.encodeWithSelector(IOracle.getRate.selector), abi.encode(1.05e18));
    uint256 balanceBefore = wrsETH.balanceOf(alice);
    vm.deal(alice, 1);
    vm.prank(alice);
    pool.deposit{value: 1}("");
    assertEq(wrsETH.balanceOf(alice), balanceBefore); // alice received 0 wrsETH
    assertEq(address(pool).balance, 1);               // pool absorbed the 1 wei
}
```
