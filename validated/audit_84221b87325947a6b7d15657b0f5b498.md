### Title
Unbounded Gas Consumption in `updateRSETHPrice` via O(M×N) External Calls — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a permissionless `public` function. Its internal call chain executes a nested loop of external calls that scales as O(M × 3N), where M is the number of supported assets and N is the number of NodeDelegators. As the protocol grows, this function will eventually exceed the block gas limit, permanently freezing rsETH price updates.

---

### Finding Description

**Entry point — no role restriction:**

`updateRSETHPrice()` is callable by any address: [1](#0-0) 

**Outer loop — iterates over M supported assets:**

`_getTotalEthInProtocol()` fetches the full `supportedAssetList` and loops over every asset: [2](#0-1) 

For each asset it makes:
1. One external call to `getAssetPrice(asset)` (price oracle).
2. One external call to `ILRTDepositPool.getTotalAssetDeposits(asset)`.

**Inner loop — iterates over N NDCs per asset:**

`getTotalAssetDeposits` delegates to `getAssetDistributionData`, which loops over every NDC in `nodeDelegatorQueue`: [3](#0-2) 

Per NDC per asset, three external calls are made:
- `IERC20(asset).balanceOf(nodeDelegatorQueue[i])`
- `INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset)`
- `INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset)`

**Additional unbounded dimension — queued EigenLayer withdrawals:**

`getAssetUnstaking` itself contains a nested loop over all queued EigenLayer withdrawals for each NDC: [4](#0-3) 

This adds a third dimension to gas consumption.

**Caps are soft and admin-controlled:**

- `maxNodeDelegatorLimit` defaults to 10 but `updateMaxNodeDelegatorLimit` can raise it without bound: [5](#0-4) 

- There is no hard cap on the number of supported assets; `addNewSupportedAsset` is gated only by `TIME_LOCK_ROLE`: [6](#0-5) 

---

### Impact Explanation

When M × N grows large enough, `updateRSETHPrice()` will revert out-of-gas on every call. Because the function is the sole mechanism for updating `rsETHPrice`, a frozen price means:

- Deposits via `depositAsset`/`depositETH` use a stale price (incorrect rsETH minting).
- The protocol's fee accrual and downside-protection logic stop functioning.
- `updateRSETHPriceAsManager()` (manager-only) calls the same `_updateRsETHPrice()` and is equally affected.

This maps directly to **Medium — Unbounded gas consumption** leading to permanent freezing of the price-update mechanism.

---

### Likelihood Explanation

The protocol is designed to scale: more LST assets can be added via governance, and more NDCs can be added by the admin to handle larger TVL. Neither action requires malicious intent. A legitimate protocol expansion to, e.g., 10 assets × 20 NDCs produces at minimum 10 × (1 + 60) = 610 external calls, each costing thousands of gas in cold-storage scenarios, plus the recursive `getAssetUnstaking` loops. This is a realistic operational trajectory, not a theoretical edge case.

---

### Recommendation

1. **Cache NDC balances off-chain / use a push model**: Have NDCs report their balances to a registry on state changes rather than pulling them all on every price update.
2. **Hard-cap `maxNodeDelegatorLimit`** at a value proven safe under worst-case gas analysis.
3. **Hard-cap `supportedAssetList` length** in `addNewSupportedAsset`.
4. **Split `_getTotalEthInProtocol` into paginated calls** so the price update can be computed incrementally across multiple transactions.

        // 3. uint256 gasBefore = gasleft();

Run on a local Hardhat/Foundry fork with `M=10, N=20` (both within plausible admin-set limits). Measure gas; at cold storage prices the call will approach or exceed 30M gas, proving the invariant break without any mainnet interaction.

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
