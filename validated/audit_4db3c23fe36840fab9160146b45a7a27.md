### Title
Missing lower-bound validation on `fee_proposal_fri` during initiation window allows wrong `ProposalCommitment` and fee market corruption - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` enforces a margin check on `fee_proposal_fri` only when `fee_actual` is `Some`. During the first `fee_proposal_window_size` blocks (`fee_actual = None`), any `Some(GasPrice(0))` value passes unchecked. A malicious proposer can inject `fee_proposal_fri = Some(GasPrice(0))`, causing every validator to compute `ProposalCommitment = Poseidon(partial_block_hash, 0)` instead of the honest `Poseidon(partial_block_hash, min_gas_price)`. Consensus finalizes the wrong commitment, the `fee_proposals_window` is seeded with zero values, and the `parent_proposal_commitment` carried in the cende blob is permanently wrong for those heights.

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` validates `fee_proposal_fri` in two steps:

**Step 1 – presence check** (lines 371–394): rejects `None` when version ≥ V0_14_3, rejects `Some` when version < V0_14_3.

**Step 2 – margin check** (lines 398–416): only executes when `fee_actual` is `Some`:

```rust
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    // bounds check
}
```

`fee_actual` is `None` for the first `fee_proposal_window_size` blocks because `compute_fee_actual` returns `None` when `height < window_size`:

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ...");
    return None;
};
```

During this initiation window, a malicious proposer sends `fee_proposal_fri = Some(GasPrice(0))`. The validator passes both checks (presence: `Some(_)` ✓; margin: skipped because `fee_actual = None`). The validator then computes:

```rust
let batcher_block_commitment = proposal_commitment_from(
    finished_info.proposal_commitment.partial_block_hash,
    fee_proposal,   // ← Some(GasPrice(0)) from the malicious init
);
```

`proposal_commitment_from` computes:

```rust
ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
// = Poseidon(partial_block_hash, 0)
```

The malicious proposer also sets `received_fin.proposal_commitment = Poseidon(partial_block_hash, 0)`. The `ProposalFinMismatch` check at line 244 passes:

```rust
if built_block != received_fin.proposal_commitment {
    return Err(ValidateProposalError::ProposalFinMismatch);
}
```

Consensus votes on and finalizes `Poseidon(partial_block_hash, 0)`.

After finalization, `finalize_decision` calls `record_fee_proposal(height, init.fee_proposal_fri)`, storing `GasPrice(0)` in `fee_proposals_window`. Once the window fills, `compute_fee_actual` returns a median that includes these zero values, pulling the fee market down. The `parent_proposal_commitment` in the cende blob is also computed with the wrong parent fee:

```rust
central_objects.parent_proposal_commitment
    .map(|c| proposal_commitment_from(c.partial_block_hash, parent_fee_proposal))
```

where `parent_fee_proposal` is read from the corrupted `fee_proposals_window`.

The honest proposer always emits `fee_proposal_fri >= min_gas_price` (falls back to `self.l2_gas_price` which is ≥ `min_gas_price` = 8 gwei). No existing guard enforces this minimum on the validator side during the initiation window.

### Impact Explanation

**Wrong `ProposalCommitment` accepted by consensus**: The finalized block carries `Poseidon(partial_block_hash, 0)` instead of `Poseidon(partial_block_hash, min_gas_price)`. This is the value consensus signs over and that the cende pipeline uses to verify the chain of block commitments. The `parent_proposal_commitment` field in every subsequent cende blob is wrong for the affected heights.

**Corrupted fee market**: `fee_proposals_window` is seeded with `GasPrice(0)` entries. Once the window fills, `compute_fee_actual` returns a median that includes zeros, causing `fee_proposal_bounds` to accept abnormally low `fee_proposal_fri` values in future blocks. This allows a sustained downward manipulation of the L2 gas price, with direct economic impact on fee revenue and transaction admission thresholds.

### Likelihood Explanation

The attack requires the attacker to be the block proposer during the initiation window (first `fee_proposal_window_size` blocks). This window occurs at genesis and after any chain revert. In a permissioned validator set the attacker must control a proposer slot; in a permissionless or rotating set the window is reachable by any validator. The manipulation is silent (no error is logged for a zero `fee_proposal_fri` during the initiation window) and the wrong commitment is indistinguishable from a legitimate one to downstream consumers.

### Recommendation

Add a minimum-value check on `fee_proposal_fri` unconditionally when it is `Some`, regardless of whether `fee_actual` is available. The minimum should be `VersionedConstants::latest_constants().min_gas_price`:

```rust
// After the presence check, before the margin check:
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    let min_gas_price = VersionedConstants::latest_constants().min_gas_price;
    if fee_proposal < min_gas_price {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "fee_proposal_fri {} is below minimum gas price {}",
                fee_proposal.0, min_gas_price.0
            ),
        ));
    }
}
```

This mirrors the invariant already enforced on the proposer side in `compute_proposer_fee_proposal`, which always returns at least `self.l2_gas_price` (≥ `min_gas_price`).

### Proof of Concept

1. Network starts at height 0 with `starknet_version >= V0_14_3` and `fee_proposal_window_size = W`.
2. For heights 0 through W-1, `compute_fee_actual` returns `None` (initiation window).
3. Malicious proposer at height H < W constructs `ProposalInit` with `fee_proposal_fri = Some(GasPrice(0))`.
4. Malicious proposer computes `commitment = Poseidon(partial_block_hash, 0)` and sends `ProposalFin { proposal_commitment: commitment, ... }`.
5. Validator calls `is_proposal_init_valid`: presence check passes (`Some(0)` is `Some`); margin check is skipped (`fee_actual = None`).
6. Validator calls `handle_proposal_part` → `proposal_commitment_from(partial_block_hash, Some(GasPrice(0)))` = `Poseidon(partial_block_hash, 0)`.
7. `built_block == received_fin.proposal_commitment` → no `ProposalFinMismatch`.
8. Validator returns `Ok(Poseidon(partial_block_hash, 0))` to consensus.
9. Consensus finalizes the block with commitment `Poseidon(partial_block_hash, 0)`.
10. `record_fee_proposal(H, Some(GasPrice(0)))` stores 0 in `fee_proposals_window`.
11. After W blocks, `compute_fee_actual` returns a median that includes the injected zeros, corrupting the fee market for all subsequent blocks.

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
