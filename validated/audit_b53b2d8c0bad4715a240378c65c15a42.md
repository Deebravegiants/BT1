### Title
Ephemeral Coin Phantom Balance Injection via Missing Removals Check in `validate_received_state_from_peer` — (`chia/wallet/wallet_node.py`)

---

### Summary

`validate_received_state_from_peer` in `chia/wallet/wallet_node.py` never checks the removals Merkle set when a coin state arrives with `spent_height=None` and the coin is not yet known locally (`current=None`). An unprivileged peer can send a crafted `CoinState` for a real ephemeral coin (created and spent in the same block) claiming it is unspent. The additions Merkle proof is cryptographically valid (the coin really was added), so all checks pass, and the wallet registers a phantom unspent coin record, inflating its spendable balance.

---

### Finding Description

The removals validation in `validate_received_state_from_peer` is gated behind two conditions, neither of which fires for the ephemeral-coin attack case:

**Branch 1** (lines 1588–1612): only executes when `current is not None` — i.e., the wallet already has a local record for the coin.

**Branch 2** (lines 1614–1641): only executes when `spent_height is not None` — i.e., the peer itself reports the coin as spent. [1](#0-0) 

When `current=None` and `coin_state.spent_height=None`, the function:

1. Fetches the header block for `created_height=N`.
2. Calls `request_and_validate_additions` — verifies the coin's puzzle hash is in the additions Merkle set for block N. For an ephemeral coin this proof is **cryptographically valid** because the coin really was added.
3. Calls `validate_block_inclusion` — verifies block N is in the longest chain. Also valid.
4. Skips both removals branches entirely.
5. Returns `True`, causing the coin to be accepted as unspent. [2](#0-1) 

The additions Merkle set in a real block includes ephemeral coins (created and spent in the same block), as confirmed by `validate_block_merkle_roots` which iterates all `tx_additions` without filtering out ephemeral ones: [3](#0-2) 

The removals Merkle set for the same block also contains the ephemeral coin's name. The wallet never requests or validates this set for the creation block when `spent_height=None` and `current=None`. [4](#0-3) 

---

### Impact Explanation

A malicious untrusted peer can cause the victim wallet to register a phantom unspent `WalletCoinRecord` for any coin whose puzzle hash the wallet is subscribed to. This corrupts the wallet's coin accounting: the displayed balance is inflated, and the wallet may attempt to include the phantom coin in spend bundles (which will be rejected by honest full nodes). For CAT or offer wallets, the phantom coin could cause failed offer settlements or incorrect asset accounting. This maps to **High: Corruption of coin records / wallet sync state with direct security impact**, and potentially **Critical: unauthorized accounting change affecting XCH/CATs**.

---

### Likelihood Explanation

The attacker only needs to:
1. Know one of the victim's subscribed puzzle hashes (trivially obtained from any public address).
2. Craft a transaction creating and spending an ephemeral coin at that puzzle hash in the same block (standard spend bundle, no special privileges).
3. Connect to the victim wallet as an untrusted peer (open to any network participant).
4. Send `CoinState(ephemeral_coin, spent_height=None, created_height=N)`.

No cryptographic forgery is required. The additions Merkle proof is genuine. The only deception is omitting `spent_height`.

---

### Recommendation

After the additions check passes and `coin_state.spent_height is None`, immediately validate that the coin's name is **excluded** from the removals Merkle set of the same creation block. Concretely, after line 1585, add a call to `request_and_validate_removals` against `state_block.foliage_transaction_block.removals_root` for `coin_state.coin.name()`, and require the result to be `True` (exclusion proven) before proceeding. This closes the gap for the `current=None, spent_height=None` case.

---

### Proof of Concept

```python
# 1. Attacker crafts a spend bundle that creates and immediately spends
#    a coin at victim_puzzle_hash (ephemeral coin) in block N.
#    This is a normal spend bundle; any node can submit it.

# 2. Attacker connects to victim wallet as an untrusted peer.

# 3. Attacker sends:
coin = Coin(parent_coin_info, victim_puzzle_hash, amount)
# coin.name() is in block N's additions_root AND removals_root
malicious_state = CoinState(coin, spent_height=None, created_height=N)

# 4. validate_received_state_from_peer flow:
#    - current = None  (wallet has never seen this coin)
#    - spent_height = None
#    - request_and_validate_additions(..., victim_puzzle_hash, additions_root)
#      → True  (coin really IS in additions Merkle set; proof is valid)
#    - validate_block_inclusion(state_block, ...)
#      → True  (block N is real and in the chain)
#    - Branch (current is not None): SKIPPED
#    - Branch (spent_height is not None): SKIPPED
#    - Returns True

# 5. wallet_state_manager.add_coin_states([malicious_state], ...)
#    → phantom unspent WalletCoinRecord inserted for victim_puzzle_hash
#    → wallet balance inflated by `amount`
``` [5](#0-4)

    async def validate_received_state_from_peer(
        self,
        coin_state: CoinState,
        peer: WSChiaConnection,
        peer_request_cache: PeerRequestCache,
        fork_height: uint32 | None,
    ) -> bool:
        """
        Returns True if the coin_state is valid and included in the blockchain proved by the weight proof.
        """
        if peer.closed:
            return False
        # Only use the cache if we are talking about states before the fork point. If we are evaluating something
        # in a reorg, we cannot use the cache, since we don't know if it's actually in the new chain after the reorg.
        if can_use_peer_request_cache(coin_state, peer_request_cache, fork_height):
            return True

        spent_height: uint32 | None = None if coin_state.spent_height is None else uint32(coin_state.spent_height)
        confirmed_height: uint32 | None = (
            None if coin_state.created_height is None else uint32(coin_state.created_height)
        )
        current = await self.wallet_state_manager.coin_store.get_coin_record(coin_state.coin.name())
        # if remote state is same as current local state we skip validation

        # CoinRecord unspent = height 0, coin state = None. We adjust for comparison below
        current_spent_height = None
        if current is not None and current.spent_block_height != 0:
            current_spent_height = current.spent_block_height

        # Same as current state, nothing to do
        if (
            current is not None
            and current_spent_height == spent_height
            and current.confirmed_block_height == confirmed_height
        ):
            peer_request_cache.add_to_states_validated(coin_state)
            return True

        reorg_mode = False

        # If coin was removed from the blockchain
        if confirmed_height is None:
            if current is None:
                # Coin does not exist in local DB, so no need to do anything
                return False
            # This coin got reorged
            reorg_mode = True
            confirmed_height = current.confirmed_block_height

        # request header block for created height
        state_block: HeaderBlock | None = peer_request_cache.get_block(confirmed_height)
        if state_block is None or reorg_mode:
            state_block = await request_and_validate_header_block(peer, confirmed_height, self.log)
            if state_block is None:
                return False
            peer_request_cache.add_to_blocks(state_block)
        if state_block.foliage_transaction_block is None:
            return False
        # get proof of inclusion
        validate_additions_result = await request_and_validate_additions(
            peer,
            peer_request_cache,
            state_block.height,
            state_block.header_hash,
            coin_state.coin.puzzle_hash,
            state_block.foliage_transaction_block.additions_root,
        )

        if validate_additions_result is None:
            return False
        if validate_additions_result is False:
            self.log.warning("Validate false 1")
            await peer.close(9999)
            return False

        # If spent_height is None, we need to validate that the creation block is actually in the longest blockchain.
        # Otherwise, we don't have to, since we will validate the spent block later.
        if coin_state.spent_height is None:
            validated = await self.validate_block_inclusion(state_block, peer, peer_request_cache)
            if not validated:
                return False

        # TODO: make sure all cases are covered
        if current is not None:
            if spent_height is None and current.spent_block_height != 0:
                # Peer is telling us that coin that was previously known to be spent is not spent anymore
                # Check old state
                spent_state_block = await request_and_validate_header_block(peer, current.spent_block_height, self.log)
                if spent_state_block is None:
                    return False
                peer_request_cache.add_to_blocks(spent_state_block)
                assert spent_state_block.foliage_transaction_block is not None
                validate_removals_result = await request_and_validate_removals(
                    peer,
                    current.spent_block_height,
                    spent_state_block.header_hash,
                    coin_state.coin.name(),
                    spent_state_block.foliage_transaction_block.removals_root,
                )
                if validate_removals_result is None:
                    return False
                if validate_removals_result is False:
                    self.log.warning("Validate false 2")
                    await peer.close(9999)
                    return False
                validated = await self.validate_block_inclusion(spent_state_block, peer, peer_request_cache)
                if not validated:
                    return False

        if spent_height is not None:
            # request header block for created height
            cached_spent_state_block = peer_request_cache.get_block(spent_height)
            if cached_spent_state_block is None:
                spent_state_block = await request_and_validate_header_block(peer, spent_height, self.log)
                if spent_state_block is None:
                    return False
                peer_request_cache.add_to_blocks(spent_state_block)
            else:
                spent_state_block = cached_spent_state_block
            if spent_state_block.foliage_transaction_block is None:
                return False
            validate_removals_result = await request_and_validate_removals(
                peer,
                spent_state_block.height,
                spent_state_block.header_hash,
                coin_state.coin.name(),
                spent_state_block.foliage_transaction_block.removals_root,
            )
            if validate_removals_result is None:
                return False
            if validate_removals_result is False:
                self.log.warning("Validate false 3")
                await peer.close(9999)
                return False
            validated = await self.validate_block_inclusion(spent_state_block, peer, peer_request_cache)
            if not validated:
                return False
        peer_request_cache.add_to_states_validated(coin_state)

        return True

def validate_removals(
    coins: list[tuple[bytes32, Coin | None]], proofs: list[tuple[bytes32, bytes]] | None, root: bytes32
) -> bool:
    if proofs is None:
        # If there are no proofs, it means all removals were returned in the response.
        # we must find the ones relevant to our wallets.

        # Verify removals root
        removals_items = [name for name, coin in coins if coin is not None]
        removals_root = bytes32(compute_merkle_set_root(removals_items))
        if root != removals_root:
            return False
    else:
        # This means the full node has responded only with the relevant removals
        # for our wallet. Each merkle proof must be verified.
        if len(coins) != len(proofs):
            return False
        for i in range(len(coins)):
            # Coins are in the same order as proofs
            if coins[i][0] != proofs[i][0]:
                return False
            coin = coins[i][1]
            if coin is None:
                # Verifies merkle proof of exclusion
                not_included = confirm_not_included_already_hashed(
                    root,
                    coins[i][0],
                    proofs[i][1],
                )
                if not_included is False:
                    return False
            else:
                # Verifies merkle proof of inclusion of coin name
                if coins[i][0] != coin.name():
                    return False
                included = confirm_included_already_hashed(
                    root,
                    coin.name(),
                    proofs[i][1],
                )
                if included is False:
                    return False
    return True
