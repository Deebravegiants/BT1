Audit Report

## Title
Unrestricted `receive()` on `LRTDepositPool` Allows Donation to Inflate `totalETHInProtocol`, Triggering Illegitimate Fee Minting — (`contracts/LRTDepositPool.sol`, `contracts/LRTOracle.sol`)

## Summary
`LRTDepositPool` exposes a bare, permissionless `receive()` fallback that accepts ETH from any caller. Because `getETHDistributionData()` returns `address(this).balance` as `ethLyingInDepositPool`, any donated ETH is immediately counted in `totalETHInProtocol`. When `updateRSETHPrice()` is subsequently called, the oracle treats the TVL increase as staking yield, computes a `protocolFeeInETH` on it, and mints rsETH to the treasury — redirecting a portion of the donation away from rsETH holders to the treasury without legitimate staking activity.

## Finding Description
**Root cause — unrestricted ETH entry point:**
`LRTDepositPool` at [1](#0-0)  accepts ETH from any address with no access control, unlike the named entry points (`receiveFromRewardReceiver`, `receiveFromNodeDelegator`, `receiveFromLRTConverter`) at [2](#0-1)  which are also payable but semantically scoped.

**Raw balance included in TVL:**
`getETHDistributionData()` sets `ethLyingInDepositPool = address(this).balance` at [3](#0-2)  — the full contract balance including any donated ETH. `getAssetDistributionData()` routes ETH through this path at [4](#0-3)  and `getTotalAssetDeposits()` sums all components at [5](#0-4) .

**Oracle counts donation as yield:**
`_getTotalEthInProtocol()` calls `getTotalAssetDeposits` for every supported asset at [6](#0-5) . In `_updateRsETHPrice()`, the fee is computed as `rewardAmount = totalETHInProtocol - previousTVL` (which includes donated ETH) at [7](#0-6) .

**Fee rsETH minted to treasury:**
The computed fee is minted as rsETH to the treasury at [8](#0-7) , increasing rsETH supply without a corresponding increase in backing for existing holders.

**Existing guards are insufficient:**
- `pricePercentageLimit` only blocks calls where the price increase exceeds the configured threshold; small donations or a zero limit pass freely ( [9](#0-8) ).
- `maxFeeMintAmountPerDay` caps per-day damage but does not prevent the attack ( [10](#0-9) ).

## Impact Explanation
This is **High — Theft of unclaimed yield**. The donated ETH constitutes yield that should accrue entirely to rsETH holders as increased backing. Instead, `donation × feeBPS/10000` is siphoned to the treasury as newly minted rsETH. Every existing rsETH holder is diluted by exactly the fee portion of the donation. The treasury receives rsETH it did not earn through legitimate staking activity. The impact is concrete and directly measurable: `rsETH holders lose yield = donation × protocolFeeInBPS / 10_000`.

## Likelihood Explanation
The `receive()` fallback is completely open — no role, no minimum, no guard. `updateRSETHPrice()` is also public and callable by anyone when not paused. The attacker loses the donated ETH, so direct financial motivation requires either treasury control or a short position on rsETH. However, the path is fully permissionless, locally reproducible on unmodified code, and repeatable up to the `maxFeeMintAmountPerDay` cap per day.

## Recommendation
1. **Restrict the `receive()` fallback** to only accept ETH from known, trusted senders (e.g., `receiveFromRewardReceiver`, `receiveFromNodeDelegator`, `receiveFromLRTConverter`). Revert on arbitrary ETH transfers.
2. Alternatively, **replace `address(this).balance` with an explicit storage variable** in `getETHDistributionData()` that is only incremented through the controlled entry points (`depositETH`, `receiveFromRewardReceiver`, etc.), so donated ETH is never counted in TVL.

## Proof of Concept
```solidity
// Foundry fork test (local fork, no mainnet testing)
function testDonationInflatesFee() public {
    // Precondition: rsETH supply > 0, protocolFeeInBPS > 0
    uint256 treasuryBalanceBefore = rsETH.balanceOf(treasury);

    // Step 1: attacker donates 1 ETH directly to LRTDepositPool via receive()
    vm.deal(attacker, 1 ether);
    vm.prank(attacker);
    (bool ok,) = address(lrtDepositPool).call{value: 1 ether}("");
    require(ok);

    // Step 2: anyone calls updateRSETHPrice (public, permissionless)
    lrtOracle.updateRSETHPrice();

    // Step 3: treasury received fee rsETH minted on the donated ETH
    assertGt(rsETH.balanceOf(treasury), treasuryBalanceBefore);
    // rsETH holders received only (1 - feeBPS/10000) of the donation value
}
```
The assertion `treasuryBalanceAfter > treasuryBalanceBefore` confirms fee rsETH was minted on donated ETH, diluting all existing rsETH holders by the fee portion of the donation.
