### Title
Unvalidated `builder` Field in `ProposalInit` Allows Malicious Proposer to Redirect Sequencer Fees and Corrupt Block Header — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` in `validate_proposal.rs` validates most fields of `ProposalInit` but silently accepts any value for `builder`. In `convert_to_sn_api_block_info`, `init.builder` is mapped directly to `sequencer_address` in the `BlockInfo` passed to the batcher. Because `sequencer_address` governs where transaction fees are transferred and what the `get_sequencer_address` syscall returns, a malicious proposer can set `builder` to an arbitrary address, redirect all block fees to that address, and permanently store a wrong sequencer address in the block header.

---

### Finding Description

`ProposalInit` carries a `builder` field (the address of the block builder/sequencer):

```rust
// crates/apollo_protobuf/src/consensus.rs:107
pub builder: ContractAddress,
```

`is_proposal_init_valid` validates `height`, `timestamp`, `starknet_version`, `version_constant_commitment`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, and `fee_proposal_fri`. It does **not** validate `builder`:

```rust
// crates/apollo_consensus_orchestrator/src/validate_proposal.rs:252-418
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    ...
) -> ValidateProposalResult<()> {
    // timestamp, starknet_version, version_constant_commitment,
    // height, l1_da_mode, l2_gas_price_fri, L1 prices, fee_proposal_fri
    // ... builder is never checked ...
    Ok(())
}
```

`ProposalInitValidation` does not even carry a reference value for `builder`:

```rust
// crates/apollo_consensus_orchestrator/src/validate_proposal.rs:75-85
pub(crate) struct ProposalInitValidation {
    pub height: BlockNumber,
    pub block_timestamp_window_seconds: u64,
    pub previous_proposal_init: Option<PreviousProposalInitInfo>,
    pub l1_da_mode: L1DataAvailabilityMode,
    pub l2_gas_price_fri: GasPrice,
    pub starknet_version: StarknetVersion,
    pub fee_actual: Option<GasPrice>,
    // no builder field
}
```

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`:

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs:329-332
Ok(starknet_api::block::BlockInfo {
    block_number: init.height,
    block_timestamp: BlockTimestamp(init.timestamp),
    sequencer_address: init.builder,   // <-- unvalidated wire value
    ...
})
```

This `BlockInfo` is forwarded to the batcher as `ValidateBlockInput.block_info`, which the blockifier uses for the entire block's execution context. The `sequencer_address` from this context is used in two critical places:

1. **Fee transfers** — `add_fee_to_sequencer_balance` in `fee_utils.rs` credits fees to `block_context.block_info.sequencer_address`.
2. **`is_sequencer_the_sender` shortcut** — if `sequencer_address == sender_address`, the fee transfer is skipped entirely (zero fee charged).
3. **`get_sequencer_address` syscall** — returns `sequencer_address` to executing contracts.

After consensus, `finalize_decision` calls `convert_to_sn_api_block_info(init)` again and passes the result to `update_state_sync_with_new_block`, permanently storing the attacker-supplied `builder` as the block's sequencer address in the block header.

---

### Impact Explanation

A malicious proposer who wins a consensus round can set `init.builder` to any address. Validators accept the proposal because `builder` is never checked. The consequences are:

- **Fee theft**: All transaction fees from the block are credited to the attacker-controlled address instead of the legitimate sequencer.
- **Fee bypass**: If the attacker sets `builder` equal to a transaction sender's address, `is_sequencer_the_sender()` returns `true` and the fee transfer is skipped, allowing zero-fee execution.
- **Wrong `get_sequencer_address` syscall result**: Contracts that call `get_sequencer_address` during execution receive the attacker's address, producing wrong execution outputs and potentially wrong receipts/events.
- **Corrupted block header**: The wrong sequencer address is permanently committed to the block header in state sync storage.

This satisfies:
- *Critical: Wrong state, receipt, event, storage value, or revert result from blockifier/syscall/execution logic for accepted input.*
- *Critical: Incorrect fee, gas, resource accounting, refund, balance with economic impact.*

---

### Likelihood Explanation

The attacker must be a consensus participant who wins the proposer role for a round. In Tendermint-style consensus the proposer is selected algorithmically from the validator set — no special admin privilege is required beyond being a registered validator. Any validator who is scheduled as proposer for a round can exploit this. The attack requires no interaction with other participants and leaves no on-chain trace distinguishing it from a legitimate proposal.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and check it in `is_proposal_init_valid`:

```rust
pub(crate) struct ProposalInitValidation {
    ...
    pub builder: ContractAddress,  // add this
}
```

```rust
// in is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder
        ),
    ));
}
```

The expected `builder` value should be derived from the local node's configuration (`static_config.builder_address`) or from the committee assignment for the height, mirroring how the proposer sets it in `initiate_build`.

---

### Proof of Concept

1. Attacker is a validator and wins the proposer slot for block N.
2. In `initiate_build`, the attacker sets `init.builder = attacker_controlled_address` instead of `self.config.static_config.builder_address`.
3. The proposal is streamed to peer validators.
4. Each validator calls `validate_proposal` → `is_proposal_init_valid`. The function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, L1 prices, `starknet_version`, `version_constant_commitment`, `fee_proposal_fri` — but never `builder`. The proposal passes validation.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing a `BlockInfo` with `sequencer_address = attacker_controlled_address`.
6. The batcher executes all transactions in the block with this `BlockInfo`. Every fee transfer credits `attacker_controlled_address`.
7. Consensus reaches decision. `finalize_decision` calls `convert_to_sn_api_block_info(init)` again and passes the result to `update_state_sync_with_new_block`, permanently storing `attacker_controlled_address` as the block's sequencer in the block header.
8. All fees from block N are in the attacker's account; the block header records the wrong sequencer.

async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)

) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
