### Title
`verify_foreign_transaction` Ignores Caller Identity in Request Key and Signed Payload, Enabling Cross-Chain Replay — (File: `crates/contract/src/lib.rs`)

### Summary

The `verify_foreign_transaction` function does not bind its response to the caller's identity (`predecessor_id`). Unlike `sign()` and `request_app_private_key()`, which both derive a caller-specific tweak/app_id from `predecessor_id`, `verify_foreign_transaction` silently discards the `predecessor_id` returned by `check_request_preconditions()`. The MPC network signs the foreign-chain payload under the **root key** (no tweak), producing a caller-agnostic `(payload_hash, signature)` that any on-chain observer can replay against consuming bridge contracts.

### Finding Description

**`sign()` correctly binds the response to the caller:**

```rust
let (domain_config, predecessor) = self.check_request_preconditions(...);
let request = SignatureRequest::new(
    request.domain_id, request.payload,
    &predecessor,       // ← predecessor bound into tweak
    &request.path,
);
``` [1](#0-0) 

**`request_app_private_key()` correctly binds the response to the caller:**

```rust
let (_, predecessor) = self.check_request_preconditions(...);
let request = CKDRequest::new(
    request.app_public_key, domain_id,
    &predecessor,           // ← predecessor bound into app_id
    &request.derivation_path,
);
``` [2](#0-1) 

**`verify_foreign_transaction()` silently drops the return value:**

```rust
self.check_request_preconditions(   // ← return value IGNORED entirely
    request.domain_id,
    DomainPurpose::ForeignTx,
    ...
);
// ...
let request = args_into_verify_foreign_tx_request(request); // no predecessor
```
