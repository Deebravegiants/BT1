### Title
TEE Image Grace-Period Boundary Bypass via Off-by-One in `valid_entries` - (File: `crates/contract/src/tee/proposal.rs`)

---

### Summary

`AllowedDockerImageHashes::valid_entries` uses `rposition` to locate the cutoff index for expired TEE image hashes, then returns `entries[cutoff_index..]`. Because `rposition` returns the index of the **last** expired entry and the slice is inclusive of that index, the most recently expired image is always included in the "valid" set. A node running an image whose upgrade grace period has already elapsed can therefore continue to submit passing attestations and remain a recognized participant indefinitely — bypassing the enforcement window the grace period was designed to provide.

---

### Finding Description

`AllowedDockerImageHashes::valid_entries` in `crates/contract/src/tee/proposal.rs` (lines 170–193) computes which TEE image hashes are still within their upgrade grace period:

```rust
fn valid_entries(&self, tee_upgrade_deadline_duration: Duration) -> Vec<AllowedMpcDockerImage> {
    let current_time = Timestamp::now();
    let cutoff_index = self
        .allowed_tee_proposals
        .iter()
        .rposition(|allowed_docker_image| {
            let Some(grace_period_deadline) = allowed_docker_image
                .added
                .checked_add(tee_upgrade_deadline_duration)
            else { return true; };
            grace_period_deadline < current_time   // true  ⟺  entry is expired
        })
        .unwrap_or(0);

    self.allowed_tee_proposals
        .get(cutoff_index..)          // ← includes the expired entry at cutoff_index
        .unwrap_or(&[])
        .to_vec()
}
``` [1](#0-0) 

`rposition` returns the index of the **rightmost** element for which the predicate is `true` — i.e., the most recently expired entry. The subsequent `get(cutoff_index..)` slice is **inclusive** of that index, so the expired entry is always returned as valid.

Concrete example with entries `[A(expired), B(valid)]`:
- `rposition` → `Some(0)` (A is the last expired entry)
- `get(0..)` → `[A(expired), B(valid)]`
- Image A, whose grace period has elapsed, is still treated as valid.

The correct slice would be `get(cutoff_index + 1..)` (with a guard to preserve the "always keep at least one" invariant for the all-expired case).

`valid_entries` feeds directly into:

1. `get_allowed_mpc_docker_image_hashes` → called inside `add_participant` to verify a freshly submitted attestation. [2](#0-1) 

2. `reverify_participants` → called inside `reverify_and_cleanup_participants` to decide which participants retain a valid TEE status. [3](#0-2) 

Both paths accept the expired image hash as legitimate, so a node running the expired image passes both initial attestation submission and periodic re-verification.

---

### Impact Explanation

The TEE image upgrade grace period (`tee_upgrade_deadline_duration`, configured in `crates/contract/src/config.rs`) is the contract's primary mechanism for enforcing that all MPC nodes migrate to a new, vetted image after a security-relevant update. Once the deadline passes, nodes still running the old image should be rejected and eventually removed via `reverify_and_cleanup_participants`.

Due to the off-by-one, the most recently expired image is never actually rejected: it remains in the allowed set until a **second** newer image is added and its own grace period also expires. In practice this means:

- A node running a deprecated (potentially vulnerable) image continues to submit valid attestations.
- `reverify_and_cleanup_participants` returns `TeeValidationResult::Full` for that node, suppressing any resharing that would otherwise evict it.
- The node participates in threshold signing rounds, potentially exploiting a known vulnerability in the old image to leak key-share material or produce unauthorized signatures.

This matches the **Medium** allowed impact: *participant-state manipulation that breaks production safety/accounting invariants without relying on network-level DoS or operator misconfiguration*.

---

### Likelihood Explanation

No special attacker action is required at the boundary. Once governance votes in a new image and the old image's grace period expires, **any** node that has not upgraded automatically benefits from the bypass — it continues to pass attestation checks without any additional steps. The only prerequisite is that at least one participant is still running the old image when the deadline passes, which is a realistic operational scenario (slow upgrades, deliberate non-compliance by a Byzantine participant below the signing threshold).

---

### Recommendation

Change the slice to exclude the expired entry while preserving the "always keep at least one" invariant:

```rust
let cutoff_index = self
    .allowed_tee_proposals
    .iter()
    .rposition(|allowed_docker_image| {
        ...
        grace_period_deadline < current_time
    })
    .map(|i| i + 1)                          // exclude the expired entry
    .unwrap_or(0);

// Always keep at least the last entry even if everything is expired.
let cutoff_index = cutoff_index
    .min(self.allowed_tee_proposals.len().saturating_sub(1));

self.allowed_tee_proposals
    .get(cutoff_index..)
    .unwrap_or(&[])
    .to_vec()
```

This ensures that an image whose grace period has elapsed is excluded from attestation verification as soon as a newer image exists, while still guaranteeing the list is never empty.

---

### Proof of Concept

1. Governance votes in image **A** at block time `T₀`. Grace period = `G`.
2. Governance votes in image **B** at block time `T₀ + G/2` (before A expires).
3. At block time `T₀ + G + 1` image A's deadline has passed (`A.added + G < now`).
4. A Byzantine node still running image **A** calls `submit_participant_info` with an attestation bound to image A.
5. Inside `add_participant`, `get_allowed_mpc_docker_image_hashes(G)` calls `valid_entries`:
   - entries = `[A(added=T₀, expired), B(added=T₀+G/2, valid)]`
   - `rposition` → `Some(0)` (A is the last expired entry)
   - `get(0..)` → `[A, B]` — A is returned as valid
6. The attestation passes; the node is stored as a valid participant.
7. `reverify_and_cleanup_participants` later re-runs the same check and again includes A, so the node is never evicted.
8. The Byzantine node participates in signing rounds with a potentially exploitable old image, bypassing the upgrade enforcement window. [1](#0-0) [4](#0-3)

        let current_time = Timestamp::now();

        let cutoff_index = self
            .allowed_tee_proposals
            .iter()
            .rposition(|allowed_docker_image| {

                    .added
                    .checked_add(tee_upgrade_deadline_duration)
                else {
                    log!("Error: timestamp overflowed when calculating grace_period_deadline.");
                    return true;
                };
                // if the grace period for this docker hash is in the past, then older hashes are no longer accepted

            })
            .unwrap_or(0);

            .get(cutoff_index..)

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub fn reverify_and_cleanup_participants(
        &mut self,
        participants: &Participants,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeValidationResult {
        self.allowed_docker_image_hashes
            .cleanup_expired_hashes(tee_upgrade_deadline_duration);

        let participants_with_valid_attestation: Vec<_> = participants
            .participants()
            .iter()
            .filter(|(_, _, participant_info)| {
                // Use the stored NodeId (keyed by TLS public key) so the real
                // `account_public_key` participates in re-verification. If
                // there is no stored attestation for this TLS key, the
                // participant is invalid.
                let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
                else {
                    return false;
                };

                let tee_status =
                    self.reverify_participants(&node_id, tee_upgrade_deadline_duration);

                matches!(tee_status, TeeQuoteStatus::Valid)
            })
            .cloned()
            .collect();

        if participants_with_valid_attestation.len() != participants.len() {
            let participants_with_valid_attestation =
                Participants::init(participants.next_id(), participants_with_valid_attestation);

            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            }
        } else {
            TeeValidationResult::Full
        }
    }
