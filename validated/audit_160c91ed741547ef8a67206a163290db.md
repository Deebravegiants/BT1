Audit Report

## Title
rsETH Holders Can Avoid Slashing Losses by Withdrawing at Stale `rsETHPrice` Before Oracle Update - (File: contracts/LRTWithdrawalManager.sol)

## Summary
`LRTOracle` stores `rsETHPrice` as a state variable updated only when `updateRSETHPrice()` is explicitly called by an off-chain bot. `instantWithdrawal` in `LRTWithdrawalManager` computes asset payouts using this stored price without triggering an update, creating a window after a slashing event where users can withdraw at the pre-slash price and receive more assets than their proportional share. The resulting loss is transferred entirely to remaining rsETH holders.

## Finding Description
`rsETHPrice` is a plain storage variable in `LRTOracle`: [1](#0-0) 

`updateRSETHPrice()` is public but never called atomically within any withdrawal path: [2](#0-1) 

`getExpectedAssetAmount` reads the stored price directly: [3](#0-2) 

`instantWithdrawal` calls `getExpectedAssetAmount` and immediately disburses assets without first refreshing the price: [4](#0-3) 

The only existing guard is the downside-protection mechanism in `_updateRsETHPrice`, which pauses the protocol only when the price drop exceeds `pricePercentageLimit`: [5](#0-4) 

For slashing events below that threshold, the protocol remains live and all withdrawals execute at the stale, inflated price. An attacker monitoring EigenLayer slashing events on-chain can call `instantWithdrawal` in the window between the slashing being reflected in TVL and the oracle bot calling `updateRSETHPrice()`, burning rsETH at the pre-slash rate and receiving more underlying assets than entitled.

## Impact Explanation
Exiting users receive excess assets drawn from `LRTUnstakingVault` at the stale price. When `updateRSETHPrice()` is eventually called, the lower price is reflected and all remaining rsETH holders bear the full slashing loss, including the portion that should have been borne by the exiting users. This is a direct, quantifiable transfer of value — proportional loss shifting — from remaining holders to early exiters, matching **High: Theft of unclaimed yield**.

## Likelihood Explanation
EigenLayer slashing is an expected, documented risk for restaking protocols. `updateRSETHPrice()` is driven by off-chain bots on a periodic schedule, not per-block, making the stale-price window non-trivial. `instantWithdrawal` is an unprivileged, publicly callable function requiring only rsETH balance. Any rsETH holder monitoring EigenLayer events can exploit this without any special access. The attack is repeatable across every slashing event that falls below `pricePercentageLimit`.

## Recommendation
Before computing the payout in `instantWithdrawal` (and `initiateWithdrawal`), call `updateRSETHPrice()` to ensure the price reflects current TVL. Alternatively, compute the payout using a live TVL calculation rather than the stored `rsETHPrice`, or revert if the stored price is older than a configurable staleness threshold.

## Proof of Concept
1. Protocol TVL = 1000 ETH, rsETH supply = 1000, `rsETHPrice` = 1.0 ETH/rsETH.
2. EigenLayer slashing event reduces actual TVL to 990 ETH. `rsETHPrice` is still 1.0 (not yet updated by bot). Price drop is 1%, assumed below `pricePercentageLimit`, so no pause is triggered.
3. Alice holds 100 rsETH. She calls `instantWithdrawal(ETH, 100e18, "")`.
4. `getExpectedAssetAmount` returns `100 * 1.0 / 1.0 = 100 ETH` (stale price).
5. Alice receives 100 ETH and burns 100 rsETH. Vault pays out 100 ETH.
6. Remaining TVL = 890 ETH, remaining rsETH supply = 900.
7. Bot calls `updateRSETHPrice()`: new price = 890/900 ≈ 0.9889 ETH/rsETH.
8. Alice avoided her ~1 ETH proportional share of the 10 ETH loss. The 900 remaining holders bear the full 10 ETH loss instead of 9 ETH.

**Foundry fork test plan**: Fork mainnet, set `rsETHPrice` to a known value, simulate a TVL reduction (e.g., by manipulating the underlying asset balance in the unstaking vault), call `instantWithdrawal` before calling `updateRSETHPrice()`, then call `updateRSETHPrice()` and assert that the new price reflects a larger-than-proportional loss for remaining holders.
