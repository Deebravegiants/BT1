### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Corrupt Block Hash and Redirect Sequencer Fees — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates most fields of `ProposalInit` (height, l1_da_mode, l2_gas_price_fri, all L1 gas prices, starknet_version, version_constant_commitment, fee_proposal_fri) but **never checks `ProposalInit.builder`**. The `builder` field ("Address of the one who builds/sequences the block") is forwarded verbatim through `convert_to_sn_api_block_info` as `BlockInfo.sequencer_address`, which is then baked into `PartialBlockHashComponents.sequencer` and hashed into the block commitment. A malicious-but-legitimate proposer can set `builder` to any arbitrary address; every validator will accept the proposal, execute transactions with the wrong sequencer address (redirecting fee payments), and commit a block whose partial block hash encodes the attacker-chosen address.

---

### Finding Description

**`ProposalInit` structure** — `builder` is a distinct, proposer-supplied field: [1](#0-0) 

**`is_proposal_init_valid` — exhaustive field-by-field check that never touches `builder`:** [2](#0-1) 

The function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`. `builder` is absent from every branch.

**`initiate_validation` passes the proposer-supplied `builder` directly to the batcher as `block_info`:** [3](#0-2) 

**`PartialBlockHashComponents` includes `sequencer` (= `block_info.sequencer_address` = `init.builder`) in the block hash:** [4](#0-3) [5](#0-4) 

**The consensus manager validates `proposer` but has no corresponding check for `builder`:** [6](#0-5) 

**The final `ProposalFinMismatch` guard cannot catch this** because both the proposer and the validator compute the partial block hash using the same (attacker-chosen) `builder` address, so they agree: [7](#0-6) 

**The proposer sets `builder` from its own local config; no network-wide constraint exists:** [8](#0-7) 

---

### Impact Explanation

| Corrupted value | Effect |
|---|---|
| `PartialBlockHashComponents.sequencer` | Block hash encodes attacker-chosen address; every node that re-derives the hash from stored components will produce the same wrong hash, making the corruption permanent and consensus-consistent |
| `BlockInfo.sequencer_address` during execution | The blockifier transfers transaction fees to the attacker-controlled address instead of the legitimate sequencer, constituting a direct economic loss |

Both effects match the allowed impact scope: *"Wrong state … storage value"* and *"Incorrect fee … balance … with economic impact"*.

---

### Likelihood Explanation

The trigger requires a validator-set member to act as the malicious proposer for a given height/round. This is semi-privileged (not fully unprivileged), but it is a realistic threat in a permissioned or partially-trusted validator set, and the broken invariant — that every field of `ProposalInit` that enters the block hash must be independently validated — is clear and exploitable without any additional preconditions once the attacker holds the proposer role.

---

### Recommendation

Add `builder` to the fields checked inside `is_proposal_init_valid`. The simplest correct invariant is that `builder` must equal the `proposer` address already validated by the consensus manager, or alternatively be compared against a locally-configured `builder_address` stored in `ContextStaticConfig`. Either approach closes the gap:

```rust
// inside is_proposal_init_valid, after the proposer is known:
if init_proposed.builder != expected_builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(...));
}
```

---

### Proof of Concept

1. Attacker controls validator `V` which wins the proposer slot for block `N`.
2. `V` constructs `ProposalInit` with all legitimate fields **except** `builder = attacker_wallet`.
3. `V` streams the proposal to all validators.
4. Each validator calls `is_proposal_init_valid` → passes (no `builder` check).
5. Each validator calls `initiate_validation` → batcher receives `BlockInfo { sequencer_address: attacker_wallet, … }`.
6. Blockifier executes all transactions; fee-transfer syscalls credit `attacker_wallet`.
7. `PartialBlockHashComponents { sequencer: attacker_wallet, … }` is hashed → `partial_block_hash` matches the proposer's `ProposalFin.proposal_commitment` (both sides used the same `builder`).
8. `validate_proposal` returns `Ok(built_block)` — proposal accepted.
9. `decision_reached` commits the block; `sequencer_address = attacker_wallet` is persisted in the block header and the block hash.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
