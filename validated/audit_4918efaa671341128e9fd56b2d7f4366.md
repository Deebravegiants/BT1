### Title
Permanent Freezing of Funds via `removeAllowedToken` With Outstanding Collateral — (`contracts/agETH/AGETHTokenWrapper.sol`)

---

### Summary

`removeAllowedToken` sets `allowedTokens[_asset] = false` with no check on the contract's current balance of that asset. Because `_withdraw` gates every redemption on `allowedTokens[_asset]`, any altAgETH already held by the wrapper becomes permanently unrecoverable the moment the admin removes the token. The MINTER_ROLE + BRIDGER_ROLE bridge flow is the canonical path that deposits collateral without a corresponding user-initiated deposit, making the frozen amount entirely opaque to the admin at removal time.

---

### Finding Description

**Relevant code paths:**

`_withdraw` unconditionally reverts if the asset is not allowed: [1](#0-0) 

`removeAllowedToken` flips the flag with no balance guard: [2](#0-1) 

`mint` inflates `totalSupply` without depositing any altAgETH (the intended L2 bridge-mint path): [3](#0-2) 

`depositBridgerAssets` uses `maxAmountToDepositBridgerAsset` as its only guard: [4](#0-3) 

`maxAmountToDepositBridgerAsset` = `totalSupply() - balanceOf(asset)`, so a prior `mint(K)` opens a deposit window of exactly K: [5](#0-4) 

**Exploit sequence (concrete, locally reproducible):**

| Step | Action | totalSupply | altAgETH in contract |
|------|--------|-------------|----------------------|
| 0 | S users deposit S altAgETH | S | S |
| 1 | MINTER_ROLE: `mint(bridge, K)` | S+K | S |
| 2 | BRIDGER_ROLE: `depositBridgerAssets(altAgETH, K)` | S+K | S+K |
| 3 | S original holders: `withdraw(altAgETH, S)` | K | K |
| 4 | DEFAULT_ADMIN_ROLE: `removeAllowedToken(altAgETH)` | K | K |
| 5 | K bridge-mint holders: `withdraw(altAgETH, K)` → **reverts TokenNotAllowed** | K | K (frozen) |

Steps 1–3 are the **intended** cross-chain bridge flow. Step 4 is a routine admin deprecation. There is no on-chain signal to warn the admin that K altAgETH is still owed to K wrapper holders.

The simpler degenerate case (no bridge involvement) also works: any user deposits → admin removes token → user cannot withdraw. The bridge path merely makes the frozen collateral invisible to the admin.

---

### Impact Explanation

The altAgETH tokens locked in the contract after step 4 are **permanently irrecoverable**: `_withdraw` reverts, there is no emergency-sweep function, and the wrapper is upgradeable only by the admin who already removed the token. The frozen amount is principal + accrued yield — this is **Critical: permanent freezing of funds**, not merely unclaimed yield. The question's Medium framing understates the severity.

---

### Likelihood Explanation

The admin has a legitimate reason to call `removeAllowedToken` (e.g., deprecating a bridged altAgETH variant). The contract provides no balance check, no warning event, and no migration path. The bridge-mint flow means collateral can be present in the contract without any corresponding user-deposit record, making it easy for the admin to overlook. Likelihood is **Medium** (requires an admin action, but one that is operationally expected over the contract's lifetime).

---

### Recommendation

Add a balance guard in `removeAllowedToken`:

```solidity
function removeAllowedToken(address _asset) external onlyRole(DEFAULT_ADMIN_ROLE) {
    require(
        ERC20Upgradeable(_asset).balanceOf(address(this)) == 0,
        "Outstanding collateral: drain balance first"
    );
    allowedTokens[_asset] = false;
    emit TokenRemoved(_asset);
}
```

Alternatively, allow withdrawal of a removed token via a separate `emergencyWithdraw` path that bypasses the `allowedTokens` check but still burns wrapper tokens 1:1.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

// Assumes a local/fork test environment with mock ERC20 for altAgETH.
function test_permanentFreeze() public {
    // Setup: S = 100e18 deposited by users
    altAgETH.mint(user, 100e18);
    vm.prank(user);
    altAgETH.approve(address(wrapper), 100e18);
    vm.prank(user);
    wrapper.deposit(address(altAgETH), 100e18);
    // totalSupply = 100e18, altAgETH balance = 100e18

    // Step 1: bridge mints K = 50e18 wrapper tokens
    vm.prank(minterRole);
    wrapper.mint(bridgeRecipient, 50e18);
    // totalSupply = 150e18, altAgETH balance = 100e18

    // Step 2: bridger deposits 50e18 altAgETH to back minted tokens
    altAgETH.mint(bridger, 50e18);
    vm.prank(bridger);
    altAgETH.approve(address(wrapper), 50e18);
    vm.prank(bridger);
    wrapper.depositBridgerAssets(address(altAgETH), 50e18);
    // totalSupply = 150e18, altAgETH balance = 150e18

    // Step 3: original user withdraws
    vm.prank(user);
    wrapper.withdraw(address(altAgETH), 100e18);
    // totalSupply = 50e18, altAgETH balance = 50e18

    // Step 4: admin removes token (routine deprecation)
    vm.prank(admin);
    wrapper.removeAllowedToken(address(altAgETH));

    // Step 5: bridge recipient cannot withdraw — funds frozen
    vm.prank(bridgeRecipient);
    vm.expectRevert(AGETHTokenWrapper.TokenNotAllowed.selector);
    wrapper.withdraw(address(altAgETH), 50e18);

    // 50e18 altAgETH permanently locked, no recovery path
    assertEq(altAgETH.balanceOf(address(wrapper)), 50e18);
}
```
