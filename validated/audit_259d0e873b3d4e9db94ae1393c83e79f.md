### Title
Missing Predicate Check Allows Multiple Sub-Transactions to Claim the Same Deposit Refund - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubLedgers.hs`)

---

### Summary

In the Dijkstra era, a top-level transaction may embed multiple sub-transactions. The `SUBLEDGERS` rule folds over them sequentially, threading the ledger state. No predicate check prevents two sub-transactions from each including `UnRegDepositTxCert` for the same staking credential. The codebase explicitly acknowledges this gap with a disabled test (`xit`) and the comment `"TODO: predicate failure not yet implemented"`. This is the direct Cardano analog of the TokenFlow reentrancy: just as `moveOut()` recalculates a balance difference after a callback manipulates intermediate state, the sub-transaction mechanism allows intermediate cert-state manipulation to be exploited across sequential sub-transactions, enabling a deposit to be claimed more than once.

---

### Finding Description

**Root cause — `SUBLEDGERS` folds without cross-sub-tx deduplication**

`dijkstraSubLedgersTransition` in `SubLedgers.hs` processes every sub-transaction with a plain `foldM`:

```haskell
dijkstraSubLedgersTransition = do
  TRC (env, ledgerState, subTxs) <- judgmentContext
  foldM
    ( \ls subTx ->
        trans @(EraRule "SUBLEDGER" era) $ TRC (env, ls, subTx)
    )
    ledgerState
    subTxs
``` [1](#0-0) 

Each iteration calls `SUBLEDGER`, which calls `SUBENTITIES` to process certificates (including `UnRegDepositTxCert`) and then `SUBUTXOW`/`SUBUTXO` to check the UTxO balance.

**`SUBENTITIES` processes `UnRegDepositTxCert` per sub-transaction**

`dijkstraSubEntitiesTransition` in `SubEntities.hs` calls `SUBCERTS` with the current cert state and then applies `applyDirectDeposits`:

```haskell
certStateAfterSubCerts <-
    trans @(EraRule "SUBCERTS" era) $ TRC (subCertsEnv, certStateBeforeSubCerts, certificates)
...
pure $ certStateAfterSubCerts & certDStateL . accountsL %~ applyDirectDeposits directDeposits
``` [2](#0-1) 

When `UnRegDepositTxCert stakingCred deposit` is processed, the credential is removed from the cert state and the deposit is returned to the UTxO balance via `updateUTxOStateNoFees` in `SUBUTXO`.

**No cross-sub-transaction guard exists**

The `SUBLEDGER` rule passes the updated `certState` to the next iteration, but there is no check that prevents a second sub-transaction from re-presenting `UnRegDepositTxCert` for the same credential. The `SUBUTXO` balance check uses the deposit amount declared in the transaction body:

```haskell
newState <-
    Shelley.updateUTxOStateNoFees
      pp
      utxoState
      txBody
      certState          -- original certState for this sub-tx, not the post-SUBENTITIES one
      (utxosGovState utxoState)
      ...
``` [3](#0-2) 

The `certState` passed to `SUBUTXOW` is the one captured at the start of the current `SUBLEDGER` invocation — before `SUBENTITIES` ran — so the balance check for sub-tx N uses the deposit refund amount from the body without verifying it against the post-sub-tx-N-1 deposit pot.

**Codebase explicitly acknowledges the missing check**

The test suite contains a disabled test that directly describes the attack scenario:

```haskell
xit "Multiple subtransactions cannot get the same refund" $ do
    stakingCred <- KeyHashObj <$> freshKeyHash
    _ <- registerStakeCredential stakingCred
    keyDeposit <- getsPParams ppKeyDepositL
    ...
    let
      subTx1 = mkBasicTx mkBasicTxBody
        & bodyTxL . inputsTxBodyL  .~ Set.singleton input1
        & bodyTxL . certsTxBodyL   .~ SSeq.singleton (UnRegDepositTxCert stakingCred keyDeposit)
      subTx2 = mkBasicTx mkBasicTxBody
        & bodyTxL . inputsTxBodyL  .~ Set.singleton input2
        & bodyTxL . certsTxBodyL   .~ SSeq.singleton (UnRegDepositTxCert stakingCred keyDeposit)
      tx = mkBasicTx mkBasicTxBody
        & bodyTxL . subTransactionsTxBodyL .~ OMap.fromFoldable [subTx1, subTx2]
    submitFailingTx tx . NE.singleton $ error "TODO: predicate failure not yet implemented"
``` [4](#0-3) 

The `xit` marker disables the test because the predicate failure that should reject this transaction has not been implemented. The `error "TODO: predicate failure not yet implemented"` placeholder confirms the check is absent: if the rejection were already in place, the test would be enabled with the concrete `PredicateFailure` constructor.

**Structural parallel to the TokenFlow reentrancy**

| TokenFlow (Solidity) | Dijkstra (Cardano) |
|---|---|
| `moveOut()` measures received tokens as `balanceAfter − balanceBefore` | `SUBUTXO` measures consumed value using the deposit amount declared in the sub-tx body |
| ERC-777 callback re-enters `moveOut()` before the outer call reads the final balance | A second sub-transaction re-presents `UnRegDepositTxCert` before the deposit pot is reconciled |
| Netflow inflated beyond tokens actually transferred | Deposit pot decremented beyond the single deposit actually paid |

### Impact Explanation

Each sub-transaction that successfully processes `UnRegDepositTxCert stakingCred D` causes the deposit pot (`utxosDeposited`) to decrease by `D` and adds `D` to the sub-transaction's consumed value, allowing the sub-transaction's outputs to be `D` larger than its inputs. With N sub-transactions each claiming the same credential:

- Deposit pot decreases by `N × D`
- Only `1 × D` was ever deposited
- The attacker's UTxO outputs are `(N − 1) × D` larger than they should be

This is **direct creation of ADA** through an invalid ledger state transition — the total ADA in circulation (UTxO + deposit pot + fees + rewards + reserves + treasury) is no longer conserved. This matches the **Critical** impact tier: *Direct loss, creation, or destruction of ADA or native assets through an invalid ledger state transition.*

---

### Likelihood Explanation

The attacker needs only to:
1. Register one staking credential (paying `keyDeposit`, currently 2 ADA on mainnet).
2. Construct a top-level Dijkstra transaction embedding N sub-transactions, each body containing `UnRegDepositTxCert stakingCred keyDeposit` and distinct UTxO inputs to satisfy the per-sub-tx balance check.
3. Submit the transaction.

No privileged access, governance majority, or external dependency is required. The attack is fully attacker-controlled and deterministic. The only prerequisite is the Dijkstra era being active on-chain.

---

### Recommendation

Add a cross-sub-transaction deduplication check in `SUBLEDGERS` or `SUBLEDGER`. Concretely:

1. Before folding over sub-transactions, collect the set of credentials targeted by `UnRegDepositTxCert` across **all** sub-transactions in the batch.
2. Reject the top-level transaction if any credential appears in more than one sub-transaction's `UnRegDepositTxCert` set (new `DijkstraLedgerPredFailure` constructor, resolving the `TODO`).
3. Alternatively, enforce the check inside `SUBCERTS`/`SUBDELEG` by verifying that the credential is still registered in the **current** (post-previous-sub-tx) cert state before accepting `UnRegDepositTxCert`.

---

### Proof of Concept

The attack path, using the existing (disabled) test scaffold:

```
1. Alice registers stakingCred, paying keyDeposit D into the deposit pot.

2. Alice builds:
     subTx1: inputs=[utxo1], certs=[UnRegDepositTxCert stakingCred D], outputs=[addr1: value1 + D]
     subTx2: inputs=[utxo2], certs=[UnRegDepositTxCert stakingCred D], outputs=[addr2: value2 + D]
     topTx:  subTransactions=[subTx1, subTx2]

3. SUBLEDGERS folds:
     Iteration 1 (subTx1):
       SUBENTITIES: unregisters stakingCred, deposit pot -= D
       SUBUTXO:     consumed = utxo1 + D (refund), produced = addr1 output → balances
     Iteration 2 (subTx2):
       SUBENTITIES: [missing check] processes UnRegDepositTxCert again, deposit pot -= D
       SUBUTXO:     consumed = utxo2 + D (refund), produced = addr2 output → balances

4. Net effect: deposit pot decreased by 2D, but only D was deposited.
   Alice's outputs contain D of ADA created from nothing.
``` [4](#0-3) [1](#0-0) [5](#0-4) [6](#0-5)

dijkstraSubEntitiesTransition = do
  TRC (subCertsEnv, certState, certificates) <- judgmentContext
  let tx = certsTx subCertsEnv
      pp = certsPParams subCertsEnv
      curEpoch = certsCurrentEpoch subCertsEnv
      withdrawals = tx ^. bodyTxL . withdrawalsTxBodyL
      accounts = certState ^. certDStateL . accountsL

  runTest $ Conway.validateWithdrawalsDelegated accounts tx

  network <- liftSTS $ asks networkId
  let (missingWithdrawals, exceededWithdrawals) =
        case withdrawalsThatExceedAccountBalance withdrawals network accounts of
          Nothing -> (Map.empty, Map.empty)
          Just (missing, exceeded) -> (unWithdrawals missing, exceeded)
  failOnNonEmptyMap missingWithdrawals $
    injectFailure . SubWithdrawalsMissingAccounts . Withdrawals . NEM.toMap
  failOnNonEmptyMap exceededWithdrawals $ injectFailure . SubWithdrawalAmountsExceedAccountBalances

  let certStateBeforeSubCerts =
        certState
          & Conway.updateDormantDRepExpiries tx curEpoch
          & Conway.updateVotingDRepExpiries tx curEpoch (pp ^. ppDRepActivityL)
          & certDStateL . accountsL %~ applyWithdrawals withdrawals
  certStateAfterSubCerts <-
    trans @(EraRule "SUBCERTS" era) $ TRC (subCertsEnv, certStateBeforeSubCerts, certificates)

  let directDeposits = tx ^. bodyTxL . directDepositsTxBodyL
      accountsAfterSubCerts = certStateAfterSubCerts ^. certDStateL . accountsL
  failOnJust (directDepositsMissingAccounts directDeposits accountsAfterSubCerts) $
    injectFailure . SubDirectDepositsToMissingAccounts

  pure $ certStateAfterSubCerts & certDStateL . accountsL %~ applyDirectDeposits directDeposits

dijkstraSubUtxoTransition = do
  TRC (SubUtxoEnv slot pp certState originalUtxo (IsValid isValid), utxoState, stAnnTx) <-
    judgmentContext
  let tx = stAnnTx ^. txStAnnTxG

  let txBody = tx ^. bodyTxL

  runTest $ Allegra.validateOutsideValidityIntervalUTxO slot txBody

  sysSt <- liftSTS $ asks systemStart
  ei <- liftSTS $ asks epochInfo
  runTest $ Alonzo.validateOutsideForecast ei slot sysSt tx

  let allSizedOutputs = txBody ^. allSizedOutputsTxBodyF
  let allOutputs = fmap sizedValue allSizedOutputs
  runTest $ Alonzo.validateOutputTooBigUTxO pp allOutputs

  runTest $ Shelley.validateInputSetEmptyUTxO txBody

  let inputs = txBody ^. inputsTxBodyL
  let refInputs = txBody ^. referenceInputsTxBodyL
  runTest $ Shelley.validateBadInputsUTxO originalUtxo (inputs `Set.union` refInputs)
  runTest $ Shelley.validateBadInputsUTxO (utxosUtxo utxoState) inputs

  runTestOnSignal $ Shelley.validateOutputBootAddrAttrsTooBig allOutputs

  runTestOnSignal $ Babbage.validateOutputTooSmallUTxO pp allSizedOutputs

  netId <- liftSTS $ asks networkId
  runTestOnSignal $ Shelley.validateWrongNetwork netId allOutputs
  runTestOnSignal $ Shelley.validateWrongNetworkWithdrawal netId txBody
  runTestOnSignal $ validateWrongNetworkInDirectDeposit netId txBody
  runTestOnSignal $ Alonzo.validateWrongNetworkInTxBody netId txBody

  if isValid
    then do
      newState <-
        Shelley.updateUTxOStateNoFees
          pp
          utxoState
          txBody
          certState
          (utxosGovState utxoState)
          (tellEvent . TotalDeposits (hashAnnotated txBody))
          (\a b -> tellEvent $ TxUTxODiff a b)
      pure $ newState & utxosDonationL <>~ txBody ^. treasuryDonationTxBodyL
    else
      pure utxoState
