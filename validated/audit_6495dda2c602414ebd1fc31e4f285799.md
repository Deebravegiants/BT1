### Title
Subtransaction Reference Script Costs Excluded from Minimum Fee Calculation in DijkstraEra - (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

In the Dijkstra era, the minimum fee calculation for a top-level transaction does not account for reference scripts used by its embedded subtransactions. The `getMinFeeTxUtxo` instance for `DijkstraEra` delegates to `getConwayMinFeeTxUtxo`, which only measures reference script bytes from the top-level transaction's inputs. A purpose-built function `batchNonDistinctRefScriptsSize` exists and correctly aggregates reference script sizes across all subtransactions, but it is used only for the per-transaction size cap check — not for the fee floor check. An attacker can therefore submit a batch transaction whose subtransactions reference large scripts, imposing real deserialization costs on every validating node while paying a fee that does not cover those costs.

---

### Finding Description

The Dijkstra era introduces subtransactions (`subTransactionsTxBodyL`), which are embedded `SubTx`-level transactions inside a top-level `TopTx`. Each subtransaction can carry its own spending inputs and reference inputs, and the UTxO entries pointed to by those inputs may contain reference scripts that must be deserialized during validation.

The Conway-era fix (ADR-009) established that reference script deserialization overhead must be priced into the transaction fee via the tiered `tierRefScriptFee` formula. The total reference script size is computed by `txNonDistinctRefScriptsSize` and fed into `getConwayMinFeeTx` as `refScriptsSize`.

For `DijkstraEra`, the `EraUTxO` instance sets:

```haskell
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
``` [1](#0-0) 

`getConwayMinFeeTxUtxo` is:

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
``` [2](#0-1) 

`txNonDistinctRefScriptsSize` only unions the top-level transaction's `referenceInputsTxBodyL` and `inputsTxBodyL`:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [3](#0-2) 

Subtransactions are stored in `subTransactionsTxBodyL` and are **not** iterated here. The correct aggregation function already exists:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
``` [4](#0-3) 

`batchNonDistinctRefScriptsSize` is used for the size-cap check in `validateAllRefScriptSize`:

```haskell
validateAllRefScriptSize pp utxo tx =
  let totalRefScriptSize = batchNonDistinctRefScriptsSize utxo tx
      maxRefScriptSizePerTx = fromIntegral @Word32 @Int $ pp ^. ppMaxRefScriptSizePerTxG
   in failureUnless (totalRefScriptSize <= maxRefScriptSizePerTx) $ ...
``` [5](#0-4) 

But the fee floor check in `dijkstraUtxoTransition` calls `Shelley.validateFeeTooSmallUTxO pp tx originalUtxo`, which internally calls `getMinFeeTxUtxo` — the Conway version that ignores subtransaction reference scripts:

```haskell
{- minfee pp txTop utxo₀ ≤ txfee txb -}
runTest $ Shelley.validateFeeTooSmallUTxO pp tx originalUtxo
``` [6](#0-5) 

Scripts from subtransactions are collected and deserialized during validation via `getDijkstraScriptsProvided`, which explicitly iterates all subtransactions:

```haskell
getDijkstraScriptsProvided utxo tx =
  withBothTxLevels tx
    ( \topTx -> ScriptsProvided $ Map.unions $
        unScriptsProvided (getBabbageScriptsProvided utxo topTx)
          : [ unScriptsProvided (getBabbageScriptsProvided utxo subTx)
            | subTx <- OMap.elems (topTx ^. bodyTxL . subTransactionsTxBodyL) ] )
    (getBabbageScriptsProvided utxo)
``` [7](#0-6) 

The deserialization cost is therefore real and imposed on every validating node, but the fee floor does not price it.

---

### Impact Explanation

**Medium. Attacker-controlled transactions modify fees outside design parameters.**

The Conway-era ADR-009 explicitly established that reference script deserialization overhead must be covered by the transaction fee to prevent DDoS. In Dijkstra, a transaction author can embed subtransactions that collectively reference up to `maxRefScriptSizePerTx` bytes of scripts (enforced by `validateAllRefScriptSize`), while the fee floor is computed as if those subtransaction reference scripts do not exist. The attacker pays only the base `alonzoMinFeeTx` component for the top-level transaction, with zero reference-script surcharge for subtransaction scripts. This allows sustained submission of transactions that are cheap to create but expensive to validate, undermining the economic deterrent that ADR-009 was designed to provide.

---

### Likelihood Explanation

Any unprivileged transaction submitter on the Dijkstra mainnet can exploit this. The attack requires only:
1. Deploying UTxO entries containing large reference scripts (a normal on-chain operation).
2. Constructing a top-level Dijkstra transaction with subtransactions whose reference inputs point to those UTxOs.
3. Setting the top-level fee to the Conway-style minimum (which ignores subtransaction reference scripts).

The ledger will accept the transaction because `validateFeeTooSmallUTxO` uses `getConwayMinFeeTxUtxo` (top-level only), and `validateAllRefScriptSize` only enforces a size cap, not a fee floor, for subtransaction scripts.

---

### Recommendation

Override `getMinFeeTxUtxo` in the `EraUTxO DijkstraEra` instance to use `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
-- In eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs
getDijkstraMinFeeTxUtxo :: (EraTx era, BabbageEraTxBody era, DijkstraEraTxBody era)
  => PParams era -> Tx l era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx

instance EraUTxO DijkstraEra where
  ...
  getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo
```

This mirrors the pattern already used for `validateAllRefScriptSize` and ensures the fee floor covers the full deserialization cost imposed by both the top-level transaction and all its subtransactions.

---

### Proof of Concept

1. Deploy a UTxO containing a large Plutus V3 reference script (e.g., 50 KiB, well within the per-tx limit).
2. Construct a Dijkstra `TopTx` with zero reference inputs at the top level, but with one subtransaction whose `referenceInputsTxBodyL` points to the large-script UTxO.
3. Compute the minimum fee using `getConwayMinFeeTxUtxo` (what the ledger enforces): it returns `alonzoMinFeeTx pp tx` with `refScriptsSize = 0` because `txNonDistinctRefScriptsSize` sees no reference inputs on the top-level body.
4. Compute the correct fee using `batchNonDistinctRefScriptsSize`: it returns `alonzoMinFeeTx pp tx + tierRefScriptFee ... 51200` (50 KiB surcharge).
5. Submit the transaction with the lower fee. The ledger accepts it (`validateFeeTooSmallUTxO` passes), even though `validateAllRefScriptSize` would reject it only if the total exceeds `maxRefScriptSizePerTx` — not because the fee is too low.

The gap between the two fee values equals `tierRefScriptFee multiplier stride costPerByte 51200`, which at current Conway parameters (15 lovelace/byte, stride 25600, multiplier 1.2) is approximately `25600 * 15 + 25600 * 18 = 384000 + 460800 = 844800` lovelace (~0.84 ADA) per 50 KiB of subtransaction reference scripts — uncharged per transaction. getConwayMinFeeTx pp tx refScriptsSize =
  alonzoMinFeeTx pp tx <+> refScriptsFee

      tierRefScriptFee

        (fromIntegral @Word32 @Int . unNonZero $ pp ^. ppRefScriptCostStrideG)
        refScriptCostPerByte
