### Title
Vote Signature Verification Not Wired Into Consensus Handler — (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary
The `apollo_signature_manager` crate provides fully-implemented `sign_precommit_vote` and `verify_precommit_vote_signature` functions, and the `SignatureManager` component is registered with a server. However, neither function is called in the production consensus vote creation or vote handling path. Votes are emitted with `RawSignature::default()` and accepted without any signature check, meaning any network peer can forge votes from any committee member address.

### Finding Description

**Vote creation — signature is always the zero default:**

In `crates/apollo_consensus/src/state_machine.rs`, `make_self_vote` constructs every outgoing vote with an explicit TODO and a zero signature:

```rust
// TODO(Asmaa): sign the vote
signature: RawSignature::default(),
``` [1](#0-0) 

**Vote reception — signature is never checked:**

In `crates/apollo_consensus/src/single_height_consensus.rs`, `handle_vote` performs only a height check and a committee-membership check, with an explicit TODO for the missing verification:

```rust
pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
    // TODO(Asmaa): verify the signature
``` [2](#0-1) 

The only guard present is:
```rust
if !self.committee.members().iter().any(|s| s.address == vote.voter) {
``` [3](#0-2) 

This checks that the claimed `voter` address is a known committee member, but committee membership is public information. There is no cryptographic binding between the claimed `voter` address and the peer that sent the message.

**The verification infrastructure exists but is not wired in:**

`verify_precommit_vote_signature` is fully implemented in `crates/apollo_signature_manager/src/signature_manager.rs`: [4](#0-3) 

The `SignatureManager` component has a registered server handler for `SignPrecommitVote`: [5](#0-4) 

However, `SequencerConsensusContextDeps` — the struct that wires external dependencies into the consensus context — contains no `signature_manager_client` field: [6](#0-5) 

The `SignatureManager` is therefore unreachable from the consensus vote path.

### Impact Explanation

Committee member addresses are public (they are distributed as part of the committee configuration). Any network peer can:

1. Enumerate all committee member addresses.
2. Craft `Vote` messages claiming `voter = <any_committee_member>` with `proposal_commitment = <attacker_chosen_value>` and `signature = RawSignature::default()`.
3. Deliver these forged votes before the real validators' votes arrive. Because the first vote for a `(round, voter)` pair is accepted and a later conflicting vote is silently dropped (`VoteStatus::Conflict` → `return VecDeque::new()`), the forged vote displaces the real one.
4. Accumulate enough forged votes to satisfy the quorum threshold, causing consensus to decide on an attacker-chosen `ProposalCommitment` / block.

A wrong consensus decision propagates to a wrong `partial_block_hash`, wrong state root, wrong transaction receipts, and wrong L1 anchoring data — matching the "High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload" impact category.

### Likelihood Explanation

The trigger requires only network access: any peer that can deliver messages to a consensus node before the legitimate validators can exploit this. No privileged access, no special account, and no on-chain state is required. The attack window is every consensus round.

### Recommendation

1. Wire `SignatureManagerClient` into `SequencerConsensusContextDeps` and call `sign_precommit_vote` inside `make_self_vote` to replace `RawSignature::default()`.
2. Call `verify_precommit_vote_signature` (or the equivalent client call) inside `handle_vote` before the vote is accepted into the state machine, and drop votes whose signature does not verify against the claimed `voter`'s public key.
3. Remove the two `// TODO(Asmaa):` placeholders once the above is done.

### Proof of Concept

```
Attacker (any network peer):
  for each validator_addr in committee.members():
      send Vote {
          vote_type: Precommit,
          height: current_height,
          round: current_round,
          proposal_commitment: Some(attacker_chosen_commitment),
          voter: validator_addr,          // public info
          signature: RawSignature::default(),  // zero bytes, never checked
      }

handle_vote() in single_height_consensus.rs:
  // height check passes
  // committee membership check passes (validator_addr is in committee)
  // TODO(Asmaa): verify the signature  ← never executed
  // vote is accepted and counted toward quorum
```

After sending forged votes for a quorum-weight subset of the committee, `upon_decision` fires with the attacker-chosen `ProposalCommitment`, and `SMRequest::DecisionReached` is emitted with a block the attacker controls. [7](#0-6)
