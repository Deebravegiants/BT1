### Title
Anyone Can Force-Claim Tokens on Behalf of Any User, Deducting Fees Without Consent - (`contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

### Summary

`MerkleDistributor.claim()` accepts an arbitrary `account` address parameter but performs no check that `msg.sender == account`. Any unprivileged caller can trigger a claim for any eligible user, causing the protocol fee to be deducted from that user's allocation and the claim state to be permanently consumed — without the user's knowledge or consent.

### Finding Description

`MerkleDistributor.claim()` takes `account` as a caller-supplied parameter and uses it as both the proof subject and the token recipient:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
    // No check: require(msg.sender == account)
    ...
    uint256 fee = (claimableAmount * feeInBPS) / 10_000;
    uint256 amountToSend = claimableAmount - fee;

    IERC20(token).safeTransfer(account, amountToSend);
    IERC20(token).safeTransfer(protocolTreasury, fee);
    ...
}
```

There is no `msg.sender == account` guard. The Merkle proof only proves that `account` is entitled to `cumulativeAmount`; it does not prove that the caller is `account`. Any third party who knows the Merkle proof data (which is public off-chain) can submit it and force the claim.

Contrast this with `KernelMerkleDistributor._processClaim()`, which explicitly enforces:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

`MerkleDistributor` has no equivalent guard.

### Impact Explanation

When a forced claim is executed:

1. **Fee is deducted from the user's allocation** — `fee = claimableAmount * feeInBPS / 10_000` is sent to `protocolTreasury`, not to the user. If the owner later reduces `feeInBPS`, a user who intended to wait would have received more tokens. By forcing the claim at the current (higher) fee rate, the attacker permanently reduces the user's yield.
2. **Claim state is permanently consumed** — `userClaims[account].lastClaimedIndex` and `userClaims[account].cumulativeAmount` are updated, preventing the user from ever reclaiming at a more favorable time or fee rate.
3. **Tokens are sent to the correct address** — the user does receive `amountToSend`, so there is no direct theft of principal, but the fee portion of their unclaimed yield is extracted without consent.

Impact: **High — Theft of unclaimed yield** (fee is taken from the user's allocation without authorization).

### Likelihood Explanation

All Merkle proof data (index, account, cumulativeAmount, proof) is public. Any observer of the off-chain distribution data or on-chain events can reconstruct valid proofs for any eligible user. No special privilege is required. The attack is trivially executable by any EOA.

### Recommendation

Add a `msg.sender == account` check inside `claim()`, mirroring the pattern already used in `KernelMerkleDistributor`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Alternatively, replace the `account` parameter with `msg.sender` directly, as is done in `KernelTop100MerkleDistributor.claim()`:

```solidity
function claim(uint256 amount, bytes32[] calldata merkleProof) external {
    address user = msg.sender;
    ...
}
```

### Proof of Concept

1. Protocol deploys `MerkleDistributor` with `feeInBPS = 500` (5%).
2. Alice is eligible for 1000 tokens. She intends to wait until the owner reduces the fee.
3. Bob (attacker) observes Alice's Merkle proof from the off-chain distribution data.
4. Bob calls `claim(index, alice, 1000e18, proof)`.
5. Alice receives 950 tokens; 50 tokens go to `protocolTreasury`.
6. Alice's claim state is marked consumed. She cannot reclaim at a lower fee rate.
7. Bob spent only gas; Alice permanently lost 50 tokens of yield she would have retained by waiting. [1](#0-0) [2](#0-1) [3](#0-2)

```text
    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
