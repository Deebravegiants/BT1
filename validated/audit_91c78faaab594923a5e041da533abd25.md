### Title
Silent Duplicate-Key Overwrite in P2P `ContractDiff` Decoding Allows Malicious Peer to Corrupt Node Storage — (`crates/apollo_protobuf/src/converters/state_diff.rs`)

---

### Summary

A malicious unauthenticated p2p peer can send a `ContractDiff` protobuf message containing duplicate `ContractStoredValue` entries for the same storage key. The `TryFrom<protobuf::ContractDiff> for ContractDiff` implementation silently discards the first value and retains the last via `IndexMap::collect`. Because the p2p sync client validates only `state_diff_length` (not `state_diff_commitment`) before writing to storage, the attacker can craft a message whose post-deduplication length exactly matches the expected length while substituting an arbitrary value for any storage slot.

---

### Finding Description

**Deduplication at decode time (no error raised):** [1](#0-0) 

`IndexMap::collect` on duplicate keys silently keeps the last entry. No error is returned; the caller receives a structurally valid `ContractDiff` with the wrong value.

**`len()` is computed on the already-deduplicated map:** [2](#0-1) 

`StateDiffChunk::len()` counts `contract_diff.storage_diffs.len()` — the size of the post-dedup `IndexMap`. Sending N+1 protobuf entries where two share the same key yields a decoded chunk of length N, identical to the honest block's `state_diff_length`.

**Only length is validated; commitment is never checked:** [3](#0-2) 

`parse_data_for_block` reads `target_state_diff_len` from the stored block header and loops until `current_state_diff_len == target_state_diff_len`. There is no call to `calculate_state_diff_hash` and no comparison against the `state_diff_commitment` field that is present in the header. The `state_diff_commitment` validation that exists in `apollo_committer` is not exercised on this path. [4](#0-3) 

The corrupted `ThinStateDiff` is written directly to storage via `append_state_diff` with no further integrity check.

**Intra-chunk conflict detection is absent:** [5](#0-4) 

`unite_state_diffs` detects duplicate keys only when they appear across *separate* `ContractDiff` chunks. Duplicates within a single chunk are invisible to it because deduplication already occurred inside `TryFrom`.

---

### Impact Explanation

A syncing node accepts and persists a `ThinStateDiff` containing attacker-chosen storage values. The node's storage trie diverges from the honest chain, producing a different global root. Any subsequent RPC queries, state proofs, or execution traces served by this node reflect the corrupted state.

---

### Likelihood Explanation

Any peer that the syncing node connects to for state diff data can mount this attack. No authentication, stake, or special privilege is required. The attacker only needs to be the responding peer for a single state diff query. The length check is trivially satisfied by padding with one extra duplicate entry per substituted key.

---

### Recommendation

1. After assembling the full `ThinStateDiff` in `parse_data_for_block`, compute `calculate_state_diff_hash(&result)` and compare it against the `state_diff_commitment` stored in the block header. Reject the peer if they differ.
2. Inside `TryFrom<protobuf::ContractDiff> for ContractDiff`, detect duplicate keys explicitly and return a `ProtobufConversionError` rather than silently overwriting.

---

### Proof of Concept

```rust
// Construct a ContractDiff protobuf with two entries for the same key
let key_felt = Felt::from(1u64);
let honest_value = Felt::from(100u64);
let evil_value   = Felt::from(999u64);

let proto = protobuf::ContractDiff {
    address: Some(some_address()),
    values: vec![
        protobuf::ContractStoredValue {
            key:   Some(key_felt.into()),
            value: Some(honest_value.into()),
        },
        protobuf::ContractStoredValue {   // duplicate key, different value
            key:   Some(key_felt.into()),
            value: Some(evil_value.into()),
        },
    ],
    ..Default::default()
};

let decoded = ContractDiff::try_from(proto).unwrap();
let storage_key = StorageKey(key_felt.try_into().unwrap());

// Only the last (evil) value is retained; honest_value is silently dropped.
assert_eq!(*decoded.storage_diffs.get(&storage_key).unwrap(), evil_value);
// decoded.len() == 1, matching a block whose state_diff_length == 1.
// No state_diff_commitment check prevents this from being written to storage.
```
