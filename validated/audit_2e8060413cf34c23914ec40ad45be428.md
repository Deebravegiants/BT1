### Title
`getMinFeeTxUtxo` Omits Sub-Transaction Reference Scripts in Fee Validation, Enabling Below-Minimum Fee Submission - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

In the Dijkstra era, the `EraUTxO DijkstraEra` instance delegates `getMinFeeTxUtxo` to `getConwayMinFeeTxUtxo`, which computes the reference-script fee component using only the top-level transaction's inputs and reference inputs. However, Dijkstra top-level transactions can embed sub-transactions (`dtbrSubTransactions`), each of which may reference UTxO entries containing reference scripts. A helper function `batchNonDistinctRefScriptsSize` exists that correctly aggregates reference-script sizes across the top-level transaction and all sub-transactions, but it is never wired into the fee-validation path. As a result, the minimum fee enforced by the ledger is systematically lower than the fee the protocol intends to charge, allowing an attacker to submit transactions that pay less than the correct minimum fee.

---

### Finding Description

The Dijkstra era introduces nested ("sub") transactions. A top-level `DijkstraTx` body (`DijkstraTxBodyRaw TopTx`) carries a field `dtbrSubTransactions :: OMap TxId (Tx SubTx era)`, and each sub-transaction body (`DijkstraSubTxBodyRaw`) has its own `dstbrSpendInputs` and `dstbrReferenceInputs` sets that may resolve to UTxO entries containing reference scripts.

The fee-validation path is:

1. `feesOK` (Alonzo UTXO rule) calls `getMinFeeTxUtxo pp tx utxo`.
2. For Dijkstra, `getMinFeeTxUtxo = getConwayMinFeeTxUtxo` (line 141 of `UTxO.hs`).
3. `getConwayMinFeeTxUtxo` calls `getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx`.
4. `txNonDistinctRefScriptsSize` only unions `referenceInputsTxBodyL` and `inputsTxBodyL` of the **top-level** transaction body. [1](#0-0) [2](#0-1) [3](#0-2) 

Meanwhile, `batchNonDistinctRefScriptsSize` — defined in the same file — correctly adds the sub-transaction reference-script sizes:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
``` [4](#0-3) 

This function is exported but is **not** used in `getMinFeeTxUtxo`. The fee-validation path therefore ignores all reference scripts reachable only through sub-transaction inputs.

---

### Impact Explanation

**Medium. Attacker-controlled transactions modify fees outside design parameters.**

The `minFeeRefScriptCostPerByte` / tiered `tierRefScriptFee` mechanism was introduced specifically to price the deserialization overhead of reference scripts and deter DDoS attacks (as documented in ADR-009). By embedding large reference scripts exclusively in sub-transaction inputs, an attacker can force the node to deserialize those scripts during validation while paying a fee computed as if those scripts did not exist. The fee paid is lower than the protocol-intended minimum, violating the fee-floor invariant enforced by `feesOK`. [5](#0-4) 

---

### Likelihood Explanation

**High.** Any Dijkstra transaction that includes sub-transactions with reference inputs pointing to UTxO entries containing scripts will trigger the undercount. No special privileges are required; any transaction author can construct such a transaction. The discrepancy grows linearly (and super-linearly under the tiered pricing) with the total size of reference scripts used exclusively by sub-transactions.

---

### Recommendation

Replace `getConwayMinFeeTxUtxo` with a Dijkstra-specific override that uses `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
getDijkstraMinFeeTxUtxo :: (EraTx era, BabbageEraTxBody era, DijkstraEraTxBody era)
  => PParams era -> Tx l era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx
```

And wire it into the `EraUTxO DijkstraEra` instance:

```haskell
getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo
``` [6](#0-5) 

---

### Proof of Concept

1. Publish a UTxO entry containing a large Plutus reference script (e.g., 50 KiB).
2. Construct a Dijkstra top-level transaction whose **top-level** body has no reference inputs pointing to that UTxO entry.
3. Embed a sub-transaction (`DijkstraSubTx`) whose `dstbrReferenceInputs` references that UTxO entry.
4. Compute the fee using `getConwayMinFeeTxUtxo` (as the ledger does): `txNonDistinctRefScriptsSize` returns 0 for the top-level tx, so `tierRefScriptFee` contributes 0 extra lovelace.
5. Submit the transaction with a fee equal to the base `alonzoMinFeeTx` (no ref-script surcharge).
6. The ledger's `feesOK` check passes because `minFee <= theFee` is satisfied with the underestimated minimum.
7. The node nonetheless deserializes the 50 KiB reference script for sub-transaction validation, incurring the full computational cost without the corresponding fee. [4](#0-3) [7](#0-6)

txNonDistinctRefScriptsSize :: (EraTx era, BabbageEraTxBody era) => UTxO era -> Tx l era -> Int
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts

getConwayMinFeeTx pp tx refScriptsSize =
  alonzoMinFeeTx pp tx <+> refScriptsFee
  where

    refScriptsFee =
      tierRefScriptFee
