Audit Report

## Title
Missing Chainlink Staleness Threshold Enables Stale-Rate wrsETH Over-Issuance — (`contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`)

## Summary

`ChainlinkOracleForRSETHPoolCollateral.getRate()` performs three guards on `latestRoundData()` output but omits a `block.timestamp - updatedAt` bound. Because rsETH is a yield-bearing token whose ETH value increases monotonically, a stale oracle returns an old, lower rate; `RSETHPoolV2NBA.deposit()` divides by that lower rate and mints more wrsETH than the deposited ETH can redeem. With no daily mint cap in `RSETHPoolV2NBA`, the aggregate over-issuance is unbounded across the stale window, constituting protocol insolvency.

## Finding Description

`ChainlinkOracleForRSETHPoolCollateral.getRate()` (L26–37) calls `latestRoundData()` and applies exactly three guards:

```solidity
if (answeredInRound < roundID) revert StalePrice();   // L30
if (timestamp == 0)            revert IncompleteRound(); // L31
if (ethPrice <= 0)             revert InvalidPrice();    // L32
``` [1](#0-0) 

For Chainlink OCR feeds, when neither the price-deviation threshold nor the heartbeat has fired, no new round is initiated: `roundID == answeredInRound` for the last completed round, so the first check passes unconditionally. The `timestamp == 0` check only catches an uninitialised round slot. Neither check bounds `block.timestamp - updatedAt`. The missing guard is:

```solidity
if (block.timestamp - timestamp > MAX_STALENESS) revert StalePrice();
```

`RSETHPoolV2NBA.deposit()` calls `viewSwapRsETHAmountAndFee`, which calls `getRate()` and computes:

```solidity
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
``` [2](#0-1) 

rsETH is a yield-bearing token: its ETH value increases over time (e.g., from 1.03 ETH/rsETH to 1.05 ETH/rsETH). A stale oracle returns the old, lower rate. Dividing by a lower rate produces a larger `rsETHAmount`, so every depositor receives more wrsETH than the deposited ETH can redeem. `deposit()` has no daily mint cap: [3](#0-2) 

## Impact Explanation

**Critical — Protocol insolvency.** Every deposit during a stale-oracle window mints excess wrsETH. The over-issuance per deposit is proportional to the rsETH yield accrued since the last oracle update. With no mint cap in `RSETHPoolV2NBA`, the aggregate shortfall accumulates unboundedly across all depositors during the stale window. The pool has issued more wrsETH claims than the ETH it holds can back at redemption time.

## Likelihood Explanation

rsETH/ETH is a low-volatility staking derivative; price-deviation triggers rarely fire. The heartbeat interval (typically 24 h) is the primary update trigger. Network congestion or oracle-node issues can delay updates beyond the heartbeat. No privileged access is required — any user calling `deposit()` during the stale window passively exploits it. The condition is realistic and repeatable.

## Recommendation

Add an immutable `MAX_STALENESS` constant and enforce it in `getRate()`:

```solidity
uint256 public constant MAX_STALENESS = 24 hours;

function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID)                       revert StalePrice();
    if (timestamp == 0)                                  revert IncompleteRound();
    if (block.timestamp - timestamp > MAX_STALENESS)     revert StalePrice();
    if (ethPrice <= 0)                                   revert InvalidPrice();

    return uint256(ethPrice) * 1e18 / 10 ** uint256(AggregatorV3Interface(oracle).decimals());
}
```

Additionally, consider adding a daily mint cap to `RSETHPoolV2NBA` to bound worst-case exposure from any oracle failure.

## Proof of Concept

1. Fork mainnet at a recent block.
2. Record wrsETH balance; call `RSETHPoolV2NBA.deposit{value: 100 ether}("")` — record `mintedFresh`.
3. `vm.warp(block.timestamp + 48 hours)` without touching the Chainlink feed.
4. Call `deposit{value: 100 ether}("")` again — record `mintedStale`.
5. Assert `mintedStale > mintedFresh`.

Because `ChainlinkOracleForRSETHPoolCollateral.getRate()` has no `block.timestamp - updatedAt` check, it returns the same old (lower) rate after the warp. `viewSwapRsETHAmountAndFee` divides by the lower rate and mints more wrsETH for the same ETH input, confirming the insolvency gap.
