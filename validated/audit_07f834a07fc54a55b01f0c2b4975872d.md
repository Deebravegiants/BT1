### Title
Harvester Can Unconditionally Redirect Farmer Reward Address Without Authorization Enforcement - (File: `chia/farmer/farmer_api.py`)

### Summary
A connected harvester can redirect the farmer's 1/8 block reward (the `farmer_reward_address`) to any arbitrary address by setting `farmer_reward_address_override` in a `RespondSignatures` message. The farmer applies this override unconditionally. The CHIP-22 fee quality check exists only as a log warning and is triggered by a separate message (`NewProofOfSpace`), not by `RespondSignatures` — meaning a harvester can bypass even the warning by omitting the override from `NewProofOfSpace` and only injecting it in `RespondSignatures`.

### Finding Description

In `FarmerAPI._process_respond_signatures`, the farmer unconditionally replaces its configured `farmer_target` with whatever address the harvester supplies in `RespondSignatures.farmer_reward_address_override`:

```python
farmer_reward_address = self.farmer.farmer_target
if response.farmer_reward_address_override is not None:
    farmer_reward_address = response.farmer_reward_address_override
    include_source_signature_data = True
```

This overridden address is then embedded directly into `DeclareProofOfSpace` and submitted to the full node, which creates the block with the farmer reward going to the harvester-supplied address. [1](#0-0) 

The CHIP-22 fee quality enforcement in `notify_farmer_reward_taken_by_harvester_as_fee` is only triggered when `farmer_reward_address_override` is present in the **`NewProofOfSpace`** message (a separate, earlier message): [2](#0-1) 

Even when triggered, `notify_farmer_reward_taken_by_harvester_as_fee` only **logs** a warning — it does not block the override or abort the block submission: [3](#0-2) 

There is no cross-check between the `farmer_reward_address_override` in `NewProofOfSpace` and the one in `RespondSignatures`. A harvester can send `NewProofOfSpace` with `farmer_reward_address_override = None` (avoiding the log entirely), then inject the override only in `RespondSignatures`, where no fee quality check exists at all. [4](#0-3) 

### Impact Explanation

The farmer's 1/8 block reward (currently ~0.25 XCH per block) is permanently redirected to an attacker-controlled address. The farmer receives nothing for that block. This is an unauthorized payout redirection of XCH, matching the Critical/High impact category: "Bypass of authorization that enables payout redirection." [5](#0-4) 

### Likelihood Explanation

CHIP-22 explicitly supports third-party harvesters connecting to a farmer. Any third-party harvester service (which a farmer intentionally connects to) can exploit this. The attacker does not need to compromise keys or the host — only a valid harvester connection is required. The attack is silent: no error is raised, and the farmer may not notice the reward was redirected. [6](#0-5) 

### Recommendation

Enforce the fee quality check as a hard gate in `_process_respond_signatures`: if `farmer_reward_address_override` is set in `RespondSignatures` but the corresponding `NewProofOfSpace` did not carry a matching override, or if the fee quality is insufficient, reject the override and use `self.farmer.farmer_target`. Do not rely solely on logging. [7](#0-6) 

### Proof of Concept

1. Attacker operates a harvester connected to the victim farmer (e.g., as a third-party harvester service per CHIP-22).
2. Harvester finds a valid proof of space and sends `NewProofOfSpace` with `farmer_reward_address_override = None` — no fee quality log is triggered.
3. Farmer sends `RequestSignatures` to the harvester.
4. Harvester responds with `RespondSignatures` where `farmer_reward_address_override = attacker_address`.
5. `_process_respond_signatures` applies the override unconditionally (no check, no log).
6. Farmer submits `DeclareProofOfSpace` with `farmer_reward_address = attacker_address` to the full node.
7. The full node creates the block; the farmer's 1/8 reward coin is created at the attacker's puzzle hash. The farmer receives nothing. [8](#0-7)

    def notify_farmer_reward_taken_by_harvester_as_fee(
        self, sp: farmer_protocol.NewSignagePoint, proof_of_space: harvester_protocol.NewProofOfSpace
    ) -> None:
        """
        Apply a fee quality convention (see CHIP-22: https://github.com/Chia-Network/chips/pull/88)
        given the proof and signage point. This will be tested against the fee threshold reported
        by the harvester (if any), and logged.
        """
        assert proof_of_space.farmer_reward_address_override is not None

        challenge_str = str(sp.challenge_hash)

        ph_prefix = self.config["network_overrides"]["config"][self.config["selected_network"]]["address_prefix"]
        farmer_reward_puzzle_hash = encode_puzzle_hash(proof_of_space.farmer_reward_address_override, ph_prefix)

        self.log.info(
            f"Farmer reward for challenge '{challenge_str}' "
            + f"taken by harvester for reward address '{farmer_reward_puzzle_hash}'"
        )

        fee_quality = calculate_harvester_fee_quality(proof_of_space.proof.proof, sp.challenge_hash)
        fee_quality_rate = float(fee_quality) / float(0xFFFFFFFF) * 100.0

        if proof_of_space.fee_info is not None:
            fee_threshold = proof_of_space.fee_info.applied_fee_threshold
            fee_threshold_rate = float(fee_threshold) / float(0xFFFFFFFF) * 100.0

            if fee_quality <= fee_threshold:
                self.log.info(
                    f"Fee threshold passed for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
            else:
                self.log.warning(
                    f"Invalid fee threshold for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
                self.log.warning(
                    "Harvester illegitimately took a fee reward that "
                    + "did not belong to it or it incorrectly applied the fee convention."
                )
        else:
            self.log.warning(
                "Harvester illegitimately took reward by failing to provide its fee rate "
                + f"for challenge '{challenge_str}'. "
                + f"Fee quality was {fee_quality_rate:.3f}% ({fee_quality} or 0x{fee_quality:08x})"
            )
