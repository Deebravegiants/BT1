Audit Report

## Title
Missing Staleness Metadata for `rsETHPrice` Component in `RSETHPriceFeed.latestRoundData()` - (File: contracts/oracles/RSETHPriceFeed.sol)

## Summary
`RSETHPriceFeed.latestRoundData()` computes a composite rsETH/USD price by multiplying the live Chainlink ETH/USD answer by `RS_ETH_ORACLE.rsETHPrice()`, but returns `updatedAt` sourced exclusively from the ETH/USD feed. Because `LRTOracle.rsETHPrice` is a plain state variable with no associated timestamp, and `updateRSETHPrice()` is blocked by `whenNotPaused`, a protocol auto-pause (triggered by price-deviation logic in `_updateRsETHPrice`) freezes `rsETHPrice` while the ETH/USD feed continues updating normally. Any external consumer performing a standard Chainlink staleness check against the returned `updatedAt` will silently accept a stale rsETH rate.

## Finding Description
`RSETHPriceFeed.latestRoundData()` reads both price components but propagates only the ETH/USD round metadata:

```solidity
// contracts/oracles/RSETHPriceFeed.sol L63-70
function latestRoundData() external view
    returns (uint80 roundId, int256 answer, uint256 startedAt, uint256 updatedAt, uint80 answeredInRound)
{
    (roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
    answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
}
``` [1](#0-0) 

`RS_ETH_ORACLE.rsETHPrice()` resolves to the bare state variable `uint256 public override rsETHPrice` in `LRTOracle`, which carries no last-updated timestamp: [2](#0-1) 

`rsETHPrice` is written only inside `_updateRsETHPrice()`, which is reachable only through `updateRSETHPrice()` (public, but `whenNotPaused`) or `updateRSETHPriceAsManager()` (manager-only): [3](#0-2) 

The auto-pause path in `_updateRsETHPrice()` fires without any privileged action — any caller of the public `updateRSETHPrice()` can trigger it when the price has dropped beyond `pricePercentageLimit`:

```solidity
// contracts/LRTOracle.sol L270-281
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;          // rsETHPrice is NOT updated; frozen at previous value
}
``` [4](#0-3) 

After the auto-pause, `rsETHPrice` is frozen. The Chainlink ETH/USD feed continues updating, so `updatedAt` returned by `RSETHPriceFeed` remains fresh, while the rsETH/ETH component embedded in `answer` is stale. The identical structural gap exists in `getRoundData()`: [5](#0-4) 

`ChainlinkOracleForRSETHPoolCollateral.getRate()` checks `answeredInRound < roundID` and `timestamp == 0` — both sourced from the ETH/USD round — so if `RSETHPriceFeed` is its underlying oracle, the rsETH staleness is invisible to these guards: [6](#0-5) 

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

`RSETHPriceFeed` implements `AggregatorV3Interface`, whose `updatedAt` field is the standard signal for price freshness. By returning only the ETH/USD timestamp, the contract misrepresents the composite price as fresh when the rsETH/ETH component may be arbitrarily stale. External lending markets or collateral managers that integrate this feed and rely on `updatedAt` for staleness gating will price rsETH collateral incorrectly without any on-chain signal of the problem. No direct in-protocol fund loss occurs within this repository, but the contract fails to deliver the freshness guarantee it advertises.

## Likelihood Explanation
The auto-pause is reachable by any unprivileged caller via the public `updateRSETHPrice()` function whenever the rsETH price has declined beyond `pricePercentageLimit` relative to `highestRsethPrice`. This is a designed safety mechanism, not an edge case. Once triggered, the freeze persists until an admin calls `unpause()`, during which the ETH/USD feed continues updating normally, widening the staleness gap. No attacker capability beyond calling a public function is required.

## Recommendation
Record a `rsETHPriceUpdatedAt` timestamp in `LRTOracle` each time `rsETHPrice` is written (at `LRTOracle.sol` line 313), then expose it via the `IRSETHOracle` interface. In `RSETHPriceFeed.latestRoundData()`, return `min(ethToUsdUpdatedAt, rsETHPriceUpdatedAt)` as the composite `updatedAt`. This ensures any consumer's staleness check covers both price components. Alternatively, add an explicit heartbeat revert inside `latestRoundData()` that reverts if `block.timestamp - rsETHPriceUpdatedAt` exceeds an acceptable bound.

## Proof of Concept
1. Deploy or fork with `pricePercentageLimit` set to a non-zero value.
2. Call `updateRSETHPrice()` from any EOA when the computed `newRsETHPrice` is below `highestRsethPrice` by more than `pricePercentageLimit`. `_updateRsETHPrice()` auto-pauses and returns without updating `rsETHPrice`.
3. Advance time (e.g., `vm.warp(block.timestamp + 2 hours)`). The Chainlink ETH/USD feed mock continues returning a fresh `updatedAt`.
4. Call `RSETHPriceFeed.latestRoundData()`. Observe that `updatedAt` is current (from ETH/USD) while `answer` still embeds the pre-pause `rsETHPrice`.
5. A consumer checking `block.timestamp - updatedAt < heartbeat` passes the staleness guard and accepts the stale composite price.
