### Title
Missing Zero-Value Guard in `sendETHToL1ViaBridge` Allows Spurious Zero-ETH Bridge Messages - (`contracts/bridges/OptimismMessenger.sol`)

---

### Summary

`OptimismMessenger.sendETHToL1ViaBridge` lacks a `value == 0` revert, unlike its sibling `LineaMessenger`. A caller can invoke it with `value=0` and `msg.value=0`, bypassing the only guard and forwarding a zero-value call to the Optimism native bridge.

---

### Finding Description

`OptimismMessenger.sendETHToL1ViaBridge` contains exactly one input guard:

```solidity
if (msg.value != value) revert MismatchedMsgValue();
``` [1](#0-0) 

When both `msg.value` and `value` are `0`, the condition `0 != 0` evaluates to `false`, so no revert occurs. The function then calls:

```solidity
IOptimismMessenger(l2bridge).bridgeETHTo{ value: 0 }(target, DEFAULT_GAS_LIMIT, bytes(""));
```

The Optimism standard bridge's `bridgeETHTo` does not revert on zero ETH, so the call succeeds, emitting a bridge initiation event for 0 ETH and consuming L2 gas without transferring any value.

By contrast, `LineaMessenger.sendETHToL1ViaBridge` explicitly guards against this:

```solidity
if (value == 0) revert ZeroAmount();
``` [2](#0-1) 

`OptimismMessenger` has **no access control** on `sendETHToL1ViaBridge` — it is a plain `external payable` function. Any EOA or contract can call it directly, independently of the `BRIDGER_ROLE`-gated pool contracts.

The pool-side caller (`RSETHPool.bridgeAssetsViaNativeBridge`) also has a latent path: if `getETHBalanceMinusFees()` returns `0` (pool balance equals accumulated fees), it calls the messenger with `ethBalanceMinusFees = 0` and no upstream guard stops it. [3](#0-2) 

---

### Impact Explanation

A zero-value bridge call:
- Submits a spurious cross-chain message to the Optimism bridge, consuming L2 gas.
- Emits a `BridgedETHToL1ViaNativeBridge` event with `ethBalanceMinusFees = 0`, which can mislead L1 accounting systems or off-chain monitors that track bridged ETH.
- Delivers no ETH to L1, violating the invariant that a bridge operation must move a non-zero amount.

No funds are lost; the impact is classified as **Low — contract fails to deliver promised returns, but doesn't lose value**.

---

### Likelihood Explanation

- The function is permissionless (no role check), so any external caller can trigger it at zero cost.
- The pool path is gated by `BRIDGER_ROLE`, but the messenger itself is not, making direct exploitation trivially reachable.
- The Optimism bridge does not revert on zero-value calls, so the full call sequence completes without error.

---

### Recommendation

Add a zero-value guard at the top of `sendETHToL1ViaBridge`, mirroring `LineaMessenger`:

```solidity
function sendETHToL1ViaBridge(address l2bridge, address target, uint256 value) external payable nonReentrant {
+   if (value == 0) revert ZeroAmount(); // add ZeroAmount to IL2Messenger or a local error
    if (msg.value != value) revert MismatchedMsgValue();
    IOptimismMessenger(l2bridge).bridgeETHTo{ value: value }(target, DEFAULT_GAS_LIMIT, bytes(""));
}
```

Optionally, promote `ZeroAmount` into the shared `IL2Messenger` interface so all messenger implementations enforce it consistently.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";
import "contracts/bridges/OptimismMessenger.sol";

contract MockOptimismBridge {
    event BridgeETHTo(address to, uint32 gasLimit, bytes extraData, uint256 value);
    function bridgeETHTo(address _to, uint32 _minGasLimit, bytes memory _extraData) external payable {
        emit BridgeETHTo(_to, _minGasLimit, _extraData, msg.value);
    }
}

contract OptimismMessengerZeroValueTest is Test {
    OptimismMessenger messenger;
    MockOptimismBridge bridge;

    function setUp() public {
        messenger = new OptimismMessenger();
        bridge = new MockOptimismBridge();
    }

    function test_zeroValueBridgeSucceeds() public {
        // Should revert but does NOT — zero-value bridge message is accepted
        messenger.sendETHToL1ViaBridge{value: 0}(address(bridge), address(0xBEEF), 0);
        // If we reach here, the spurious zero-value call succeeded
    }
}
```

The test passes (no revert), confirming the zero-value bridge message is submitted successfully.
