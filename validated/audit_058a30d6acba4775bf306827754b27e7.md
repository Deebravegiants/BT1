### Title
`EthXPriceOracle` uses deprecated `ethXStakePoolsManagerProxyAddress` for live price fetching — (File: `contracts/oracles/EthXPriceOracle.sol`)

---

### Summary
`EthXPriceOracle.sol` contains a storage variable `ethXStakePoolsManagerProxyAddress` explicitly annotated `/// @dev deprecated`, yet `getAssetPrice()` still calls `getExchangeRate()` on that deprecated address. This is the direct analog of the reported pattern: a deprecated contract address is still wired into a live pricing path, meaning the protocol may silently consume stale or diverged exchange-rate data for ETHX.

---

### Finding Description
In `EthXPriceOracle.sol`, line 21 marks the field as deprecated:

```solidity
/// @dev deprecated
address public ethXStakePoolsManagerProxyAddress;
```

A `reinitializer(2)` (`initialize2`) was added to derive the canonical `ethxAddress` from the current Stader config:

```solidity
function initialize2() external reinitializer(2) {
    address staderConfigProxyAddress = IETHXStakePoolsManager(ethXStakePoolsManagerProxyAddress).staderConfig();
    ethxAddress = IStaderConfig(staderConfigProxyAddress).getETHxToken();
}
```

However, the price-fetching function was never updated and still reads from the deprecated proxy:

```solidity
function getAssetPrice(address asset) external view returns (uint256) {
    if (asset != ethxAddress) { revert InvalidAsset(); }
    return IETHXStakePoolsManager(ethXStakePoolsManagerProxyAddress).getExchangeRate();
}
```

`ethxAddress` is now sourced from the current Stader config, but the exchange rate is still fetched from the old, deprecated `ethXStakePoolsManagerProxyAddress`. If Stader has migrated exchange-rate accounting to a newer contract (as the deprecation notice implies), the two can diverge, causing `getAssetPrice` to return a rate that no longer reflects the true ETHX/ETH ratio.

This price feeds directly into `LRTOracle._getTotalEthInProtocol()` → `_updateRsETHPrice()`, which sets the global `rsETHPrice` used for all rsETH minting and redemption.

---

### Impact Explanation
A wrong ETHX/ETH rate propagates into the rsETH price calculation. If the deprecated contract under-reports the ETHX rate, depositors receive more rsETH per ETHX than warranted, diluting existing holders. If it over-reports, depositors receive less rsETH, failing to deliver the promised return. Either direction constitutes the protocol not delivering its promised exchange rate.

**Impact**: Low — Contract fails to deliver promised returns, but doesn't lose value (rate divergence direction determines who bears the shortfall).

---

### Likelihood Explanation
The deprecation annotation is present in the production source, confirming the developers are aware the address is superseded. Any Stader upgrade that changes `getExchangeRate()` semantics on the old proxy (or stops updating it) immediately activates the divergence. The path is reachable by any caller of `LRTOracle.updateRSETHPrice()`, which is a public, permissionless function.

---

### Recommendation
Update `EthXPriceOracle.getAssetPrice()` to fetch the exchange rate from the current, non-deprecated Stader StakePoolsManager contract. Since `initialize2()` already retrieves the live `staderConfig` address, the same pattern can be used to resolve the current manager and call `getExchangeRate()` on it, or store the current manager address during `initialize2()` and use that for price fetching.
