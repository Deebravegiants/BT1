Audit Report

## Title
Unrestricted `FeeReceiver.sendFunds()` Enables Unprivileged Fee Extraction from rsETH Holders — (`contracts/FeeReceiver.sol`, `contracts/LRTOracle.sol`)

## Summary
`FeeReceiver.sendFunds()` carries no access-control modifier, allowing any caller to push arbitrary ETH into `LRTDepositPool`. Because `LRTOracle._updateRsETHPrice()` treats every increase in `address(LRTDepositPool).balance` as genuine staking yield, a subsequent public call to `updateRSETHPrice()` mints protocol-fee rsETH to `PROTOCOL_TREASURY` on the donated amount, permanently diverting yield that should have accrued to rsETH holders.

## Finding Description

**Root cause — missing access control on `sendFunds()`**

`FeeReceiver.sendFunds()` is declared `external` with no role modifier:

```solidity
// contracts/FeeReceiver.sol L53-58
function sendFunds() external {
    uint256 balance = address(this).balance;
    ILRTDepositPool(depositPool).receiveFromRewardReceiver{ value: balance }();
    emit MevRewardsAddedToTVL(balance);
}
``` [1](#0-0) 

The contract does use `AccessControlUpgradeable` and grants `MANAGER` to a trusted address, but `sendFunds()` is never decorated with `onlyRole`. [2](#0-1) 

**Step 2 — `receiveFromRewardReceiver()` is equally unrestricted**

```solidity
// contracts/LRTDepositPool.sol L61
function receiveFromRewardReceiver() external payable { }
``` [3](#0-2) 

Any caller can push ETH into the deposit pool through this path.

**Step 3 — ETH TVL is the raw `address(this).balance`**

`getETHDistributionData()` returns the live balance with no distinction between depositor funds and donated ETH:

```solidity
// contracts/LRTDepositPool.sol L480
ethLyingInDepositPool = address(this).balance;
``` [4](#0-3) 

**Step 4 — `updateRSETHPrice()` is callable by anyone**

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [5](#0-4) 

**Step 5 — Any TVL increase triggers fee minting**

`_updateRsETHPrice()` computes `previousTVL = rsethSupply × rsETHPrice` and treats any excess as yield:

```solidity
// contracts/LRTOracle.sol L244-247
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
``` [6](#0-5) 

The fee is then minted as rsETH to `PROTOCOL_TREASURY`:

```solidity
// contracts/LRTOracle.sol L299-307
if (protocolFeeInETH > 0) {
    uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);
    _checkAndUpdateDailyFeeMintLimit(rsethAmountToMintAsProtocolFee);
    if (rsethAmountToMintAsProtocolFee > 0) {
        address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
    }
}
``` [7](#0-6) 

**Exploit path:**
1. Attacker sends `D` ETH to `FeeReceiver` (accepted by `receive()`).
2. Attacker calls `FeeReceiver.sendFunds()` — no guard, succeeds.
3. `D` ETH lands in `LRTDepositPool` via `receiveFromRewardReceiver()`.
4. Attacker calls `LRTOracle.updateRSETHPrice()` — only `whenNotPaused`.
5. Oracle sees `totalETHInProtocol > previousTVL` by `D`, computes `protocolFeeInETH = D × protocolFeeInBPS / 10_000`, mints fee rsETH to treasury.
6. rsETH holders' share is diluted; the treasury captures yield that should have been theirs.

**Why existing guards are insufficient:**

- `pricePercentageLimit`: bypassed by donating amounts small enough to keep the price increase within the configured threshold, or entirely absent when `pricePercentageLimit == 0`.
- `maxFeeMintAmountPerDay`: limits per-day damage but does not prevent the attack; it can be repeated across days. The attack requires `maxFeeMintAmountPerDay > 0` (i.e., fee minting is enabled), which is the normal operating state.
- `_checkAndUpdateDailyFeeMintLimit`: only caps the total daily fee, not the source of the yield.

## Impact Explanation

**High — Theft of unclaimed yield.**

rsETH holders are entitled to the full price appreciation from staking rewards. When an unprivileged attacker donates `D` ETH and triggers the fee mechanism, the treasury mints rsETH worth `D × protocolFeeInBPS / 10_000` ETH, permanently diluting the rsETH supply. The yield that should have accrued to existing rsETH holders is instead captured by the treasury. The attacker loses `D` ETH (it remains in the protocol), but rsETH holders suffer a concrete, irreversible loss of yield proportional to the protocol fee rate.

## Likelihood Explanation

- No privileged role is required; any EOA or contract can execute the full attack sequence.
- The only precondition is that the oracle is unpaused (normal operating state) and `maxFeeMintAmountPerDay > 0` (fee minting is enabled).
- The `pricePercentageLimit` guard can be circumvented by splitting donations into small increments.
- The attack is repeatable across days, limited only by the attacker's willingness to donate ETH.

## Recommendation

1. Add `onlyRole(LRTConstants.MANAGER)` to `FeeReceiver.sendFunds()`.
2. Restrict `LRTDepositPool.receiveFromRewardReceiver()` to the registered `FeeReceiver` address (e.g., `require(msg.sender == lrtConfig.getContract(LRTConstants.LRT_FEE_RECEIVER))`).
3. Consider computing yield only against a snapshotted balance taken at the last price update, rather than the live raw `address(this).balance`, to prevent any unsanctioned ETH injection from being treated as yield.

## Proof of Concept

```solidity
// Foundry fork/unit test
function testDonationFeeExploit() public {
    // Preconditions: protocol has 100 ETH TVL, rsETH supply = 100e18,
    // rsETHPrice = 1e18, protocolFeeInBPS = 1000 (10%),
    // maxFeeMintAmountPerDay > 0, pricePercentageLimit = 0

    address attacker = makeAddr("attacker");
    vm.deal(attacker, 1 ether);

    uint256 treasuryBefore = rsETH.balanceOf(treasury);

    vm.startPrank(attacker);
    // Step 1: donate 1 ETH to FeeReceiver — accepted by receive()
    (bool ok,) = address(feeReceiver).call{value: 1 ether}("");
    require(ok);

    // Step 2: push ETH into deposit pool — no access control on sendFunds()
    feeReceiver.sendFunds();

    // Step 3: trigger fee minting — no access control on updateRSETHPrice()
    lrtOracle.updateRSETHPrice();
    vm.stopPrank();

    uint256 treasuryAfter = rsETH.balanceOf(treasury);

    // Treasury received ~0.1 ETH worth of rsETH minted on the 1 ETH donation
    // rsETH holders lost that yield to the treasury
    assertGt(treasuryAfter, treasuryBefore, "fee minted on donated ETH");
}
```
