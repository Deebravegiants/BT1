Audit Report

## Title
Protocol fee charged on yield accrued during pause window due to stale `rsETHPrice` baseline — (File: contracts/LRTOracle.sol)

## Summary

`_updateRsETHPrice()` computes the fee baseline using the stored `rsETHPrice`, which is never updated while the protocol is paused because `updateRSETHPrice()` is gated by `whenNotPaused`. The `protocolPaused` guard only checks the *current* pause state at call time, not whether a pause occurred in the past. On the first `updateRSETHPrice()` call after unpause, the entire TVL increase — including yield that accrued while users were locked out — is treated as a reward subject to the protocol fee, constituting theft of unclaimed yield from rsETH holders.

## Finding Description

`updateRSETHPrice()` is blocked during a pause:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, the fee baseline is derived from the stored `rsETHPrice`:

```solidity
// contracts/LRTOracle.sol L234
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);
```

The guard that suppresses fee minting checks only the *current* pause state:

```solidity
// contracts/LRTOracle.sol L240-247
bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

uint256 protocolFeeInETH = 0;
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
```

The auto-pause path pauses without updating `rsETHPrice`:

```solidity
// contracts/LRTOracle.sol L277-281
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;   // rsETHPrice is NOT updated
}
```

**Exploit sequence:**
1. Protocol is running normally; `rsETHPrice = P₀`, `previousTVL = TVL₀`.
2. Price drops beyond `pricePercentageLimit` → auto-pause fires at L277–281; `rsETHPrice` stays at `P₀`.
3. During the pause window, underlying LSTs (stETH, ETHx) continue accruing staking rewards; TVL grows to `TVL₁`.
4. Admin unpauses. Anyone calls `updateRSETHPrice()`.
5. `protocolPaused = false`; `previousTVL = rsethSupply × P₀ = TVL₀`; `rewardAmount = TVL₁ − TVL₀` (includes all paused-period yield); fee is minted on the full amount.

`updateRSETHPriceAsManager()` (L94–96) can bypass the pause and update `rsETHPrice` without taking a fee, but there is no on-chain enforcement requiring a manager to call it before unpausing.

## Impact Explanation

rsETH holders lose the protocol fee percentage applied to all staking rewards that accrued during the pause window — a period when they were unable to withdraw or exit their position. This is a concrete **theft of unclaimed yield** (High severity). The `protocolPaused` guard was designed to prevent exactly this, but the implementation leaves a gap: the fee is still effectively charged on paused-period yield, just deferred to the first post-unpause price update.

## Likelihood Explanation

The auto-pause trigger at L273–281 fires automatically whenever `newRsETHPrice` drops beyond `pricePercentageLimit` relative to `highestRsethPrice` — no privileged action is required to initiate the pause. After an auto-pause, the protocol must be investigated and manually unpaused by an admin; during that window (potentially hours to days) staking rewards continue to accrue. There is no on-chain enforcement requiring a manager to call `updateRSETHPriceAsManager()` before unpausing. This scenario is realistic, repeatable, and requires no attacker — it is a structural accounting flaw that manifests on every auto-pause/unpause cycle.

## Recommendation

On unpause (or as part of the unpause transaction in `LRTOracle.unpause()`), call `_updateRsETHPrice()` internally with fee minting suppressed (e.g., by temporarily setting a flag or by recording the TVL at pause time and using it as the fee baseline on the first post-unpause update). This ensures `rsETHPrice` is brought up to the current TVL before the public `updateRSETHPrice()` is re-enabled, so no fee is charged on yield that accrued during the pause.

## Proof of Concept

```
State before pause:
  rsETHPrice        = 1.05 ETH/rsETH
  rsethSupply       = 1,000,000 rsETH
  previousTVL       = 1,050,000 ETH

Auto-pause fires (price drop beyond pricePercentageLimit).
  updateRSETHPrice() blocked — rsETHPrice stays at 1.05.
  Staking rewards accrue over 7 days: +700 ETH (~4% APY / 52 weeks).

Admin unpauses. Anyone calls updateRSETHPrice():
  totalETHInProtocol = 1,050,700 ETH
  previousTVL        = 1,050,000 ETH  (stale)
  protocolPaused     = false
  rewardAmount       = 700 ETH  ← includes 7 days of paused-period yield
  protocolFeeInETH   = 700 × 1000 / 10000 = 70 ETH  (10% fee)

rsETH holders receive 630 ETH of yield instead of 700 ETH.
70 ETH is minted as rsETH to the treasury, diluting all holders
for yield they earned while locked out of the protocol.
```

**Foundry test plan:** Deploy `LRTOracle` with a mock deposit pool and withdrawal manager. Set `protocolFeeInBPS = 1000` (10%). Call `updateRSETHPrice()` to establish baseline. Simulate price drop beyond `pricePercentageLimit` to trigger auto-pause. Advance block timestamp by 7 days (simulating LST reward accrual by increasing mock TVL). Call `unpause()` as admin. Call `updateRSETHPrice()`. Assert that `FeeMinted` event amount equals `(TVL_after − TVL_before) × 10%`, confirming fee was charged on paused-period yield.
