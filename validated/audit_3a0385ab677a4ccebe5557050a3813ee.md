### Title
`l2_gas_price_wei` Derived from Independently-Validated `l1_gas_price_fri`/`l1_gas_price_wei` Ratio Allows Proposer to Skew ETH-Denomination Fee Prices — (File: `crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`convert_to_sn_api_block_info` derives `l2_gas_price_wei` by converting `l2_gas_price_fri` through an implied ETH→FRI rate computed as `l1_gas_price_fri * WEI_PER_ETH / l1_gas_price_wei` (both taken from the proposer's `ProposalInit`). The validator checks `l1_gas_price_fri` and `l1_gas_price_wei` independently against its own oracle reference, each within a ±10% margin, but never validates their **ratio**. A proposer can legally set `l1_gas_price_fri` at the lower margin and `l1_gas_price_wei` at the upper margin (or vice versa), skewing the implied rate by up to `(1+m)/(1-m)` ≈ 1.22× and causing `l2_gas_price_wei` to be proportionally wrong. The corrupted value is then used by the blockifier for all ETH-fee (V1/V2) transaction fee validation and charging.

---

### Finding Description

**Step 1 — Proposer builds `ProposalInit`.**

In `initiate_build` (`crates/apollo_consensus_orchestrator/src/build_proposal.rs`, line 162–188), the proposer calls `get_l1_prices_in_fri_and_wei` to obtain `l1_gas_price_fri` and `l1_gas_price_wei` from the oracle, then embeds them directly into `ProposalInit`.

**Step 2 — Validator checks each price independently.**

In `is_proposal_init_valid` (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, lines 322–368), the validator calls `get_l1_prices_in_fri_and_wei` with its own oracle read and then applies four independent `within_margin` checks:

```rust

    && within_margin(l1_data_gas_price_fri_proposed, l1_data_gas_price_fri, l1_gas_price_margin_percent)
    && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
    && within_margin(l1_data_gas_price_wei_proposed, l1_data_gas_price_wei, l1_gas_price_margin_percent))
``` [1](#0-0) 

No check is performed on the **ratio** `l1_gas_price_fri / l1_gas_price_wei`.

**Step 3 — `l2_gas_price_wei` is derived from the proposer's ratio.**

After validation passes, both proposer and validator call `convert_to_sn_api_block_info`, which calls `calculate_eth_to_fri_rate`:

```rust

**Step 4 — Corrupted `l2_gas_price_wei` enters block execution.**

The `BlockInfo` with the wrong `eth_gas_prices.l2_gas_price` is passed to the batcher for both proposal building and validation: [5](#0-4) 

The blockifier uses `eth_gas_prices.l2_gas_price` for all ETH-fee (V1/V2) transaction fee validation and charging via `GasVector::cost` and `get_fee_by_gas_vector`. [6](#0-5) 

---

### Impact Explanation

A malicious proposer sets `l1_gas_price_fri = reference_fri × 0.9` (lower margin) and `l1_gas_price_wei = reference_wei × 1.1` (upper margin). Both pass the independent 10% margin checks. The implied rate becomes `0.9/1.1 ≈ 0.818×` the actual ETH→FRI rate. Consequently:

```
l2_gas_price_wei = l2_gas_price_fri × WEI_PER_ETH / (0.818 × actual_rate)
                 ≈ 1.22 × correct_l2_gas_price_wei
```

ETH-fee transactions are charged ~22% more than the correct amount, or valid transactions whose `max_fee` is between the correct and inflated price are incorrectly rejected. The inverse skew (fri high, wei low) deflates `l2_gas_price_wei` by ~18%, causing ETH-fee transactions to be undercharged, representing economic loss to the protocol.

This matches the "Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact" impact class.

---

### Likelihood Explanation

In a decentralized consensus setup, any validator can become a proposer. A proposer is not a privileged operator — it is a rotating role. The attack requires only that the proposer craft a `ProposalInit` with `l1_gas_price_fri` and `l1_gas_price_wei` at opposite ends of their respective margins, which is trivially achievable since both values are set by the proposer itself. No external precondition is needed beyond being selected as proposer.

---

### Recommendation

Replace the independent margin checks with a joint constraint that also validates the ratio `l1_gas_price_fri / l1_gas_price_wei` against the validator's own oracle-derived ETH→FRI rate. Alternatively, do not derive `l2_gas_price_wei` from the proposer-supplied `l1_gas_price_fri`/`l1_gas_price_wei` ratio; instead derive it directly from the validator's own oracle ETH→FRI rate (which is already available in `get_l1_prices_in_fri_and_wei_and_conversion_rate`'s return value) applied to `l2_gas_price_fri`.

---

### Proof of Concept

```
Assume:
  actual ETH→FRI rate R = 10^21 (1 ETH = 1000 STRK)
  reference l1_gas_price_wei  = 10^9 wei  (1 Gwei)
  reference l1_gas_price_fri  = 10^9 * R / WEI_PER_ETH = 10^9 * 10^21 / 10^18 = 10^12 fri
  l2_gas_price_fri (exact, validated) = 8 * 10^9 fri

Proposer sets (both within 10% margin):
  l1_gas_price_fri_proposed  = 0.9 * 10^12 = 9 * 10^11 fri   ← lower margin
  l1_gas_price_wei_proposed  = 1.1 * 10^9  = 1.1 * 10^9 wei  ← upper margin

Implied rate = 9*10^11 * 10^18 / (1.1*10^9) ≈ 8.18 * 10^20  (vs actual 10^21)

l2_gas_price_wei = 8*10^9 * 10^18 / (8.18*10^20) ≈ 9.78 * 10^6 wei
correct value    = 8*10^9 * 10^18 / 10^21         = 8 * 10^6 wei

Error: +22.3% inflation of l2_gas_price_wei
```

An ETH-fee transaction with `max_fee` between `8 * 10^6` and `9.78 * 10^6` wei per L2 gas unit would be incorrectly rejected despite being valid at the true price.

```rust
fn calculate_eth_to_fri_rate(
    proposal_init_info: &PreviousProposalInitInfo,
) -> Result<u128, StarknetApiError> {
    let eth_to_fri_rate = proposal_init_info
        .l1_prices_fri
        .l1_gas_price
        .0
        .checked_mul(WEI_PER_ETH)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Fri should be small enough to multiply by WEI_PER_ETH. Previous \
                 proposal init info: {:?}",
                proposal_init_info
            ))
        })?
        .checked_div(proposal_init_info.l1_prices_wei.l1_gas_price.0)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Wei should be non-zero. Previous proposal init info: {:?}",
                proposal_init_info
            ))
        })?;
    if eth_to_fri_rate == 0 {
        return Err(StarknetApiError::GasPriceConversionError(format!(
            "Eth to fri rate is zero. Previous proposal init info: {:?}",
            proposal_init_info
        )));
    }
    Ok(eth_to_fri_rate)
}
