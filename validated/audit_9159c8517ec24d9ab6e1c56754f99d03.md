### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Inject Arbitrary `sequencer_address` into Committed Block Header and Execution Context — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries a `builder` field ("Address of the one who builds/sequences the block"). This field is consumed verbatim as the block's `sequencer_address` in both the execution `BlockInfo` and the committed `BlockHeaderWithoutHash`. However, `is_proposal_init_valid` — the sole proposal-init gate — never checks `init_proposed.builder` against any locally-trusted reference. Any scheduled proposer can therefore set `builder` to an arbitrary address, causing every validator to execute and commit a block whose `sequencer_address` is attacker-controlled.

---

### Finding Description

`ProposalInit` is defined with a `builder` field: [1](#0-0) 

During `finalize_decision`, `init.builder` is directly assigned as the block's `sequencer` in `BlockHeaderWithoutHash`: [2](#0-1) 

This `sequencer` field is then forwarded to state-sync storage and is also the value used by `PartialBlockHashComponents::new` to compute the partial block hash that becomes the `ProposalCommitment`: [3](#0-2) 

The full block hash chains over `sequencer_address`: [4](#0-3) 

The `is_proposal_init_valid` function validates `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri`. It never inspects `init_proposed.builder`: [5](#0-4) 

`ProposalInitValidation` — the struct that carries the locally-trusted reference values — has no `builder` field at all: [6](#0-5) 

Because both proposer and validator derive `sequencer_address` from the same `init.builder` wire value, the batcher's `partial_block_hash` on both sides will agree, the `ProposalFinMismatch` check passes, and the tampered value is silently committed.

---

### Impact Explanation

1. **Wrong block header committed to storage.** The `sequencer_address` stored in `BlockHeaderWithoutHash.sequencer` and returned by every RPC call (`starknet_getBlockWithTxs`, etc.) will be the attacker-supplied address, not the legitimate builder address.

2. **Wrong block hash.** `sequencer_address` is a direct input to `calculate_block_hash`. A forged `builder` produces a different block hash than an honest node would compute independently, corrupting the on-chain commitment.

3. **Fee redirection.** The `sequencer_address` in `BlockInfo` is the address to which transaction fees are credited during execution. A malicious proposer can redirect all fees for the block to an address they control.

---

### Likelihood Explanation

The attacker must be the legitimately scheduled proposer for the target height/round. Within a BFT committee this is a rotating role, so any committee member can exploit this during their proposer turn. No external capability is required beyond being a validator node.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and check it inside `is_proposal_init_valid`:

```rust
// In ProposalInitValidation:
pub builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder
        ),
    ));
}
```

The reference value should be populated from the node's own configured `builder_address` when constructing `ProposalInitValidation` in `validate_proposal` (mirroring how `l1_da_mode` and `l2_gas_price_fri` are sourced from local config).

---

### Proof of Concept

1. A malicious node is the scheduled proposer for height H.
2. It calls `build_proposal` normally but overrides `init.builder` with an attacker-controlled address `ATTACKER_ADDR` before streaming `ProposalPart::Init`.
3. Validator nodes receive the `ProposalInit` with `builder = ATTACKER_ADDR`.
4. `is_proposal_init_valid` checks timestamp, gas prices, starknet version, etc. — it never checks `builder`. Validation passes.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` → `block_info.sequencer_address = ATTACKER_ADDR`. The batcher executes the block with `sequencer_address = ATTACKER_ADDR`, crediting all fees to `ATTACKER_ADDR`.
6. `finish_proposal` returns a `partial_block_hash` computed with `sequencer = ATTACKER_ADDR`. The proposer's `ProposalFin.proposal_commitment` was also computed with `ATTACKER_ADDR`, so `built_block == received_fin.proposal_commitment` — no mismatch.
7. `decision_reached` calls `update_state_sync_with_new_block`, which writes `BlockHeaderWithoutHash { sequencer: ATTACKER_ADDR, … }` to storage.
8. All subsequent RPC queries for block H return `sequencer_address = ATTACKER_ADDR`, and the block hash is computed over the forged address.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
