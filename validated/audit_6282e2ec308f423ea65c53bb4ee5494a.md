Audit Report

## Title
Public `updateRSETHPrice()` Enables Deposit-at-Stale-Price / Oracle-Update / Instant-Withdrawal Sandwich to Steal Staking Yield ‚Äî (`contracts/LRTOracle.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is callable by any unprivileged address. An attacker deposits ETH at a stale (lower) stored rsETH price, immediately triggers the oracle update to push accrued staking rewards into the stored price, then calls `instantWithdrawal()` in the same or next block to redeem at the higher price ‚Äî extracting accrued yield that belongs to existing rsETH holders. The attack requires no privileged access and is repeatable on every reward accrual cycle.

## Finding Description

**Root cause ‚Äî `updateRSETHPrice()` is unrestricted:**

`LRTOracle.sol` line 87‚Äì89 exposes the price update to any caller:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

**Price staleness window:** `_updateRsETHPrice()` computes `newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply)` (line 250). `totalETHInProtocol` grows continuously as EigenLayer strategies earn staking rewards. Between calls, the stored `rsETHPrice` is stale (lower than the true value).

**Deposit uses stale price:** `LRTDepositPool.getRsETHAmountToMint()` (line 520) computes:
```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```
A lower `rsETHPrice` mints more rsETH per unit of ETH deposited than the true value warrants.

**Instant withdrawal uses the updated price:** `instantWithdrawal()` (lines 212‚Äì253) calls `getExpectedAssetAmount()` (line 228), which reads `lrtOracle.rsETHPrice()` at execution time (line 593):
```solidity
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
```
After the attacker triggers `updateRSETHPrice()`, this returns a higher ETH amount for the same rsETH.

**`pricePercentageLimit` defaults to 0:** The variable is declared at line 29 with no initializer. When zero, the guard at lines 256‚Äì257:
```solidity
bool isPriceIncreaseOffLimit =
    pricePercentageLimit > 0 && priceDifference > pricePercentageLimit.mulWad(highestRsethPrice);
```
is always `false`, so non-manager callers face no ceiling on the price jump they can push through.

**Instant withdrawal has no delay:** `instantWithdrawal()` (line 212) carries `onlyInstantWithdrawalAllowed(asset)` but no block-delay check. When `isInstantWithdrawalEnabled[asset]` is `true`, the full sequence ‚Äî deposit ‚Üí oracle update ‚Üí withdrawal ‚Äî executes in consecutive blocks.

**Exploit arithmetic:**

Let `T_old` = TVL at last oracle update, `T` = current TVL (rewards accrued), `S` = rsETH supply, `P1 = T_old/S` (stored stale price).

- Attacker deposits `X` ETH ‚Üí receives `X/P1` rsETH (excess over fair `X/P2`).
- After `updateRSETHPrice()`, new stored price `P_new = (T+X)/(S+X/P1)`.
- Attacker withdraws `X/P1` rsETH ‚Üí receives `(X/P1) * P_new` ETH.
- Profit = `X * (T - T_old) / (T_old + X)` ‚Äî a positive quantity whenever rewards have accrued (`T > T_old`).

The profit is a direct extraction of accrued staking yield from existing holders.

## Impact Explanation

**High ‚Äî Theft of unclaimed yield.** The attacker captures accrued staking rewards that should be distributed pro-rata to all existing rsETH holders. Each attack cycle extracts yield proportional to the attacker's share of TVL after deposit. When `isInstantWithdrawalEnabled` is `true`, the attack completes in consecutive blocks with no lock-up, making it continuously repeatable and scalable with deposit size.

## Likelihood Explanation

- `updateRSETHPrice()` is `public` ‚Äî no role, no key, no governance action required.
- Staking rewards accrue every block; the price is always slightly stale between updates.
- `pricePercentageLimit` is `0` by default, imposing no cap on non-manager callers.
- `isInstantWithdrawalEnabled` is a manager-controlled flag; when enabled (a documented feature), the attack is near-atomic.
- The attack is repeatable on every reward accrual cycle and scales linearly with deposit size.
- No front-running of a privileged transaction is required; the attacker directly controls the oracle update.

## Recommendation

1. **Restrict `updateRSETHPrice()`** to an authorized keeper or `onlyLRTManager`, removing the public entry point entirely.
2. **Enforce a minimum lock-up between deposit and withdrawal initiation** (e.g., 24 hours) so that an attacker cannot deposit at a stale price and immediately redeem at the updated price.
3. **Set `pricePercentageLimit` to a non-zero value before deployment** so that even if the function remains public, the per-call price jump is bounded and non-manager callers revert on large jumps.
4. **For instant withdrawal**, consider reading the oracle price at deposit time and using `min(depositPrice, currentPrice)` for the withdrawal calculation, or disabling instant withdrawal until a deposit lock-up is enforced.

## Proof of Concept

**Preconditions:** Staking rewards have accrued; stored `rsETHPrice = P1`; true price `P2 > P1`. `isInstantWithdrawalEnabled[ETH] = true`. `pricePercentageLimit = 0`.

**Step 1 ‚Äî Deposit at stale price (Block N):**
```
LRTDepositPool.depositETH{value: X}(minRSETH, "")
‚Üí rsethMinted = X / P1   (more than fair X / P2)
```

**Step 2 ‚Äî Trigger oracle update (Block N or N+1):**
```
LRTOracle.updateRSETHPrice()
‚Üí rsETHPrice updated to P_new = (T + X) / (S + X/P1)
```

**Step 3 ‚Äî Instant withdrawal at updated price (Block N+1):**
```
LRTWithdrawalManager.instantWithdrawal(ETH, X/P1 rsETH, "")
‚Üí assetAmountUnlocked = (X/P1) * P_new
‚Üí userAmount = assetAmountUnlocked - fee
```

**Attacker profit (before fee):**
```
profit = (X/P1) * P_new - X = X * (T - T_old) / (T_old + X)
```

For 7 days of accrued rewards at 4% APY (‚âà 0.077% increase) with a 1,000 ETH deposit: profit ‚âà 0.77 ETH before gas and instant-withdrawal fee ‚Äî repeatable every reward cycle.

**Foundry fork test outline:**
1. Fork mainnet at a block where rewards have accrued since the last `updateRSETHPrice()` call.
2. Impersonate an EOA with no roles; call `depositETH{value: 1000 ether}`.
3. Call `LRTOracle.updateRSETHPrice()` from the same EOA.
4. Call `LRTWithdrawalManager.instantWithdrawal(ETH_TOKEN, rsETHBalance, "")`.
5. Assert `address(attacker).balance > 1000 ether` (net of fee).
