### Title
Missing `payload_hash` Binding Validation in `respond_verify_foreign_tx` Enables Cross-Request Signature Replay — (File: crates/contract/src/lib.rs)

---

### Summary

The `respond_verify_foreign_tx` contract method accepts a caller-supplied `response.payload_hash` without verifying that it is the correct hash for the stored `VerifyForeignTransactionRequest`. A Byzantine attested participant (strictly below the signing threshold) who has observed a valid MPC signature from a prior legitimate signing round can replay that signature against a different pending request, causing the user to receive a forged foreign-chain attestation.

---

### Finding Description

**Vulnerability class:** Missing account/address validation — a field that should be constrained to a specific derived value (the hash of the foreign-chain payload for the stored request) is not validated, allowing a Byzantine node to substitute an arbitrary previously-seen hash.
