### Title
Missing `Q == sum(Qi)` Consistency Check in `get_public_key_compressed` Allows Attacker-Controlled Public Key Substitution — (`src/cbmpc/api/ecdsa_mp.cpp`)

---

### Summary

`get_public_key_compressed` accepts any syntactically valid key blob and returns whatever `Q_compressed` is stored in it, without verifying that `Q` equals the sum of the individual Qi shares or that `x_share * G == Qi_self`. An attacker who can supply a crafted blob causes the honest party to learn a wrong public key, enabling downstream signature-verification bypass.

---

### Finding Description

**Factual correction on call sequence:** The question states `get_public_key_compressed → parse_key_blob_any_version`. That is wrong. `get_public_key_compressed` does its own inline parsing (lines 463–475); it never calls `parse_key_blob_any_version`. The functions that do call `parse_key_blob_any_version` are `get_public_share_compressed`, `detach_private_scalar`, and `attach_private_scalar`. The call-sequence premise is incorrect, but the underlying gap is real and independently verifiable.

**The actual gap — `get_public_key_compressed` (lines 459–479):**

```cpp
error_t get_public_key_compressed(mem_t key_blob, buf_t& pub_key) {
  key_blob_v1_t blob;
  error_t rv = coinbase::convert(blob, key_blob);   // deserialize
  if (rv) return rv;
  // version check (accepts 1 or 2)
  // curve validity check
  coinbase::crypto::ecc_point_t Q(curve);
  if (rv = Q.from_bin(curve, blob.Q_compressed)) return coinbase::error(rv, "invalid key blob");
  pub_key = Q.to_compressed_bin();   // ← returned verbatim
  return SUCCESS;
}
```

The function validates only that `Q_compressed` decodes to a valid curve point. It performs **no** check that:
- `Q == sum(Qi)` (the additive-share consistency invariant), and
- `x_share * G == Qi_self` (the private-share binding). [1](#0-0) 

**Contrast with `deserialize_key_blob` (lines 172–179)**, which enforces both invariants before any key material is used:

```cpp
coinbase::crypto::ecc_point_t Q_sum = curve.infinity();
for (const auto& kv : Qis) Q_sum += kv.second;
if (Q != Q_sum) return coinbase::error(E_FORMAT, "invalid key blob");   // ← guard 1

if (blob.x_share * G != it_self->second) return coinbase::error(E_FORMAT, "invalid key blob");  // ← guard 2
``` [2](#0-1) 

`get_public_key_compressed` has neither guard. A blob with version=2 (`ac_key_blob_version_v1 = 2`) passes the version check just as version=1 does: [3](#0-2) 

---

### Impact Explanation

An attacker who controls the blob bytes passed to `get_public_key_compressed` can embed any valid curve point as `Q_compressed`. The function returns `SUCCESS` and the caller receives a public key that does not correspond to the actual MPC key shares. If the caller uses that public key for ECDSA signature verification (the documented purpose of the function), the attacker — who knows the private scalar `x_a` such that `Q_compressed = x_a * G` — can produce valid ECDSA signatures that the honest party accepts as legitimate. This is a concrete signature-verification bypass.

---

### Likelihood Explanation

The attack requires the attacker to supply a crafted blob to the honest party. Realistic vectors include: a backup/restore flow where the blob transits an untrusted channel, a multi-party key-management system where one party's stored blob is attacker-writable, or any application that passes externally-sourced blobs to `get_public_key_compressed` before verifying their provenance. The C API wrapper `cbmpc_ecdsa_mp_get_public_key_compressed` exposes the same gap at the C boundary. [4](#0-3) 

---

### Recommendation

Add the same two consistency checks that `deserialize_key_blob` performs inside `get_public_key_compressed`:

1. Parse all `Qis_compressed` entries into curve points.
2. Assert `Q == sum(Qi)`.
3. Assert `x_share * G == Qi_self` (or, if `x_share` may be zeroed for public-only blobs, skip step 3 but still enforce step 2).

Alternatively, factor the validation into a shared helper so that `get_public_key_compressed`, `get_public_share_compressed`, `detach_private_scalar`, and `attach_private_scalar` all call the same validated-parse path rather than each doing ad-hoc subsets of the checks.

### Proof of Concept

// Construct a blob where Q_compressed is an attacker-chosen point
// that does NOT equal sum(Qi).
key_blob_v1_t crafted;
crafted.version = 2;  // ac_key_blob_version_v1
crafted.curve    = static_cast<uint32_t>(curve_id::secp256k1);
crafted.party_name = "alice";

// Legitimate share: x_alice * G = Qi_alice
bn_t x_alice = bn_t::rand(curve.order());
ecc_point_t Qi_alice = x_alice * curve.generator();
crafted.x_share = x_alice;
crafted.Qis_compressed["alice"] = Qi_alice.to_compressed_bin();

// Attacker's own key: x_attacker * G = Q_attacker ≠ Qi_alice
bn_t x_attacker = bn_t::rand(curve.order());
ecc_point_t Q_attacker = x_attacker * curve.generator();
crafted.Q_compressed = Q_attacker.to_compressed_bin();  // ← wrong Q

buf_t serialized = coinbase::convert(crafted);

// get_public_key_compressed returns SUCCESS with Q_attacker
buf_t pub_key;
EXPECT_EQ(get_public_key_compressed(serialized, pub_key), SUCCESS);
EXPECT_EQ(pub_key, Q_attacker.to_compressed_bin());  // wrong key accepted

// deserialize_key_blob (with a matching job) would reject it at the Q==sum(Qi) check
coinbase::mpc::ecdsampc::key_t key;
job_mp_t job = make_single_party_job("alice");
EXPECT_NE(deserialize_key_blob(job, serialized, key), SUCCESS);
```

The attacker, knowing `x_attacker`, can now produce ECDSA signatures that verify against `pub_key`, which the honest party believes is the legitimate MPC public key.
