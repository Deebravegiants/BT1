### Title
Block Stuffing Enables Selective Chainlink Feed Staleness, Causing Mixed-TVL rsETH Mispricing — (`contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice` reads `latestRoundData()` but silently discards `updatedAt`, providing no staleness guard. An attacker can use block stuffing to prevent Chainlink keepers from updating one specific asset feed while other feeds update normally. `LRTOracle._getTotalEthInProtocol` then computes a mixed TVL — fresh prices for some assets, stale price for the targeted asset — producing a systematically biased `rsETHPrice` that can slip under the `pricePercentageLimit` guard.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice` destructures `latestRoundData()` but ignores `updatedAt`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

No maximum age is enforced. By contrast, the pool-side oracle wrapper in the same repo explicitly checks `answeredInRound < roundID` and `timestamp == 0`: [2](#0-1) 

`_getTotalEthInProtocol` iterates every supported asset and calls `getAssetPrice` for each, accumulating their ETH values:

```solidity
uint256 assetER = getAssetPrice(asset);
uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
``` [3](#0-2) 

`_updateRsETHPrice` then derives `newRsETHPrice` from this total: [4](#0-3) 

The `pricePercentageLimit` guard compares `newRsETHPrice` only against `highestRsethPrice` (the all-time high), not against the true current market price: [5](#0-4) 

**Attack path:**

1. Attacker identifies an asset (e.g., stETH) whose Chainlink feed is about to update upward (price rising).
2. Attacker stuffs blocks — filling block gas with high-fee transactions — for the duration of the Chainlink heartbeat for that one feed, preventing keeper transactions from landing.
3. The stETH feed remains stale at the pre-rise price; the ETHx feed updates normally.
4. `updateRSETHPrice()` is called (permissionless): [6](#0-5) 
5. `_getTotalEthInProtocol` uses the stale (lower) stETH price and the fresh ETHx price → TVL is understated.
6. `newRsETHPrice` is set below the true value. The deviation is small enough (bounded by the stale period) to stay under `pricePercentageLimit`.
7. Attacker deposits stETH at the artificially low rsETH price, receiving more rsETH tokens than the true TVL warrants.
8. When the stale feed eventually updates, `rsETHPrice` jumps to reflect true TVL, and the attacker's excess rsETH is now worth more ETH.

---

### Impact Explanation

`rsETHPrice` is set to a value below the true TVL/supply ratio. Depositors of the stale-priced asset receive more rsETH than they are entitled to, diluting existing holders. The protocol fails to deliver the promised invariant that `rsETHPrice == totalETH / rsethSupply` at current market prices. This matches **Low — Contract fails to deliver promised returns, but doesn't lose value**, enabled via **Low — Block stuffing**.

---

### Likelihood Explanation

Block stuffing on Ethereum mainnet is expensive but has been executed in practice (e.g., against on-chain auctions). Chainlink heartbeats for LST/ETH feeds are typically 1 hour; stuffing for a fraction of that window is sufficient. The profit opportunity scales with the total stETH deposits in the protocol, which can be large enough to justify the gas cost. `updateRSETHPrice` is permissionless, so the attacker controls the timing of the price commit.

---

### Recommendation

Add a configurable maximum staleness age to `ChainlinkPriceOracle.getAssetPrice` and revert if `block.timestamp - updatedAt > maxStaleness`:

```solidity
(, int256 price,, uint256 updatedAt,) = priceFeed.latestRoundData();
if (block.timestamp - updatedAt > maxStaleness[asset]) revert StalePrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

This mirrors the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`. [7](#0-6)

### Proof of Concept

Invariant fuzz test (local fork, unmodified contracts):

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";

contract MockFeed {
    int256 public price;
    uint256 public updatedAt;

    function latestRoundData() external view returns (uint80,int256,uint256,uint256,uint80) {
        return (1, price, updatedAt, updatedAt, 1);

contract BlockStuffingTest is Test {
    // Deploy LRTOracle with two assets:

    // After updateRSETHPrice():
    //   assert rsETHPrice != true_tvl / rseth_supply
    function testMixedStaleness() public {
        // ... setup LRTOracle, LRTConfig, two mock feeds, mint rsETH supply ...
        // Pin feed A at 1e18, advance time past heartbeat, update feed B to 1.05e18
        // Call updateRSETHPrice()
        // Compute true_tvl = depositA * 1.05e18 + depositB * 1.05e18
        // assertNotEq(oracle.rsETHPrice(), true_tvl / rsethSupply);
    }

```

The assertion will fail (prices diverge), confirming the invariant break without any admin compromise.
