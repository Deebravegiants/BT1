Audit Report

## Title
Zero wrsETH Minted on Sub-Threshold Deposits Due to Floor-Division Truncation — (File: contracts/pools/RSETHPoolV3.sol)

## Summary
In all L2 pool variants, `viewSwapRsETHAmountAndFee` computes the rsETH amount via floor (truncating) integer division. When the deposited amount is smaller than `rsETHToETHrate / 1e18`, the result truncates to zero. The `deposit` function performs no post-computation zero check before calling `wrsETH.mint(msg.sender, 0)`, so the depositor's ETH is retained by the pool while they receive zero wrsETH.

## Finding Description
In `RSETHPoolV3.viewSwapRsETHAmountAndFee`:

```solidity
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
```

When `amountAfterFee * 1e18 < rsETHToETHrate`, Solidity floor division yields `rsETHAmount = 0`. [1](#0-0) 

The `deposit` function guards only against `amount == 0`, not against a zero computed output:

```solidity
if (amount == 0) revert InvalidAmount();
(uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
feeEarnedInETH += fee;
wrsETH.mint(msg.sender, rsETHAmount);   // rsETHAmount may be 0
``` [2](#0-1) 

The `wrsETH.mint` implementation delegates directly to OpenZeppelin's `_mint`, which does not revert on a zero amount:

```solidity
function mint(address _to, uint256 _amount) external onlyRole(MINTER_ROLE) {
    _mint(_to, _amount);
}
``` [3](#0-2) 

The same pattern is confirmed in `RSETHPoolV2`, `RSETHPoolV2ExternalBridge`, `RSETHPoolV2NBA`, `RSETHPoolV3ExternalBridge`, and `RSETHPoolV3WithNativeChainBridge`: [4](#0-3) [5](#0-4) [6](#0-5) 

For the token-deposit overload, the same truncation applies:

```solidity
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
``` [7](#0-6) 

In `LRTDepositPool`, the slippage guard `rsethAmountToMint < minRSETHAmountExpected` does not protect against a zero mint when the caller passes `minRSETHAmountExpected = 0` (the default), since `0 < 0` is false: [8](#0-7) 

## Impact Explanation
A depositor who sends a non-zero ETH (or token) amount smaller than `rsETHToETHrate / 1e18` wei receives zero wrsETH. Their ETH is retained by the pool with no corresponding share minted and no refund mechanism. This constitutes the contract failing to deliver its promised return (wrsETH in exchange for ETH), mapping to the allowed **Low** impact: *Contract fails to deliver promised returns*.

## Likelihood Explanation
The rsETH/ETH rate starts at `1e18` and grows monotonically. Once it exceeds `1e18` (which occurs naturally as restaking yield accrues), any deposit of exactly 1 wei produces `rsETHAmount = 0`. The condition is reachable by any unprivileged external caller at any time after protocol launch. The affected deposit amount per call is at most 1–2 wei, making the per-transaction loss trivially small, but there is no on-chain protection preventing repeated calls. **Likelihood: Low.**

## Recommendation
Add a zero-check on the computed rsETH amount before minting in all `deposit` functions across all pool variants:

```solidity
if (rsETHAmount == 0) revert InvalidAmount();
```

Apply the same fix to the token-deposit overload. For `LRTDepositPool._beforeDeposit`, enforce `minRSETHAmountExpected > 0` or add an explicit `if (rsethAmountToMint == 0) revert InvalidAmount()` guard regardless of the caller-supplied minimum.

## Proof of Concept
1. Deploy `RSETHPoolV3` with `feeBps = 0`. Set the oracle rate to `1.05e18` (realistic post-yield value).
2. Call `RSETHPoolV3.deposit{value: 1}("")` (1 wei ETH).
3. `viewSwapRsETHAmountAndFee(1)` computes: `fee = 0`, `amountAfterFee = 1`, `rsETHAmount = 1 * 1e18 / 1.05e18 = 0`.
4. `wrsETH.mint(msg.sender, 0)` executes successfully — zero tokens minted, no revert.
5. The caller's 1 wei ETH is retained by the pool; the caller holds no wrsETH and has no claim to the deposited ETH.

**Foundry fuzz test plan:**
```solidity
function testFuzz_zeroMintOnSmallDeposit(uint256 amount) public {
    vm.assume(amount > 0 && amount < rsETHToETHrate / 1e18 + 1);
    uint256 balBefore = wrsETH.balanceOf(user);
    vm.prank(user);
    pool.deposit{value: amount}("");
    assertEq(wrsETH.balanceOf(user), balBefore); // zero minted
}
```
