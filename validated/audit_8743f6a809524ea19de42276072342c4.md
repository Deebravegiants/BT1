Audit Report

## Title
Unbounded Nested-Loop Gas Consumption in `updateRSETHPrice()` Can Permanently Stale the rsETH Price â€” (File: `contracts/LRTOracle.sol`)

## Summary
`LRTOracle.updateRSETHPrice()` is a public, permissionless function whose gas cost scales multiplicatively across supported assets, node delegators, EigenLayer queued withdrawals per NDC, and strategies per withdrawal. The protocol's only mitigation â€” a global `maxUncompletedWithdrawalCount` cap of 80 â€” is insufficient because EigenLayer operator-forced undelegations bypass the cap and add withdrawals directly to EigenLayer's queue, while the cap also fails to account for the multiplicative effect of supported assets. If the function exceeds the block gas limit, the rsETH price becomes permanently stale, disabling minting accuracy, withdrawal accuracy, and the downside-protection pause mechanism.

## Finding Description
The full call chain is confirmed in the codebase:

1. `updateRSETHPrice()` at `LRTOracle.sol:87` is `public whenNotPaused` with no role restriction â€” any external caller can invoke it.
2. It calls `_updateRsETHPrice()` at `LRTOracle.sol:214`, which calls `_getTotalEthInProtocol()` at `LRTOracle.sol:231`.
3. `_getTotalEthInProtocol()` (`LRTOracle.sol:336`) iterates over every `supportedAssets` entry and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each.
4. `getTotalAssetDeposits()` (`LRTDepositPool.sol:385`) calls `getAssetDistributionData(asset)`.
5. `getAssetDistributionData()` (`LRTDepositPool.sol:447â€“455`) loops over every NDC in `nodeDelegatorQueue` and calls `INodeDelegator.getAssetUnstaking(asset)` for each.
6. `getAssetUnstaking()` (`NodeDelegator.sol:405â€“427`) calls `_getDelegationManager().getQueuedWithdrawals(address(this))` â€” a live EigenLayer external call â€” and then runs a **nested loop** over every withdrawal and every strategy within each withdrawal, making additional external calls to `strategy.underlyingToken()` and `strategy.sharesToUnderlyingView()` in the inner body.

The total gas cost scales as:
```
supportedAssets.length
  Ă— nodeDelegatorQueue.length
    Ă— queuedWithdrawals.length (per NDC, fetched live from EigenLayer)
      Ă— withdrawal.strategies.length
```

The protocol's only guard is `maxUncompletedWithdrawalCount` capped at 80 in `LRTUnstakingVault.setMaxUncompletedWithdrawalCount()` (`LRTUnstakingVault.sol:153`). This cap is **insufficient** for two reasons:

- **Forced undelegations bypass the cap.** The comment at `LRTUnstakingVault.sol:151â€“152` explicitly acknowledges: *"Need buffer for theoretical operator forced undelegations (ndc count * asset count = 15)"*. EigenLayer operators can force-undelegate an NDC at any time, adding queued withdrawals directly to EigenLayer's state without going through the protocol's `increaseUncompletedWithdrawalCount()` path. The `getQueuedWithdrawals` call in `getAssetUnstaking` reads directly from EigenLayer, so these extra withdrawals are fully iterated.
- **The cap is global, not per-NDC, and does not account for the asset multiplier.** `getAssetUnstaking` is called once per asset per NDC. With `A` supported assets and `N` NDCs, `getAssetUnstaking` is invoked `A Ă— N` times, and each invocation fetches and iterates **all** queued withdrawals for that NDC. The effective iteration count is `A Ă— ÎŁ(withdrawals_per_NDC Ă— strategies_per_withdrawal)`, not just `maxUncompletedWithdrawalCount`.

`updateRSETHPriceAsManager()` (`LRTOracle.sol:94`) also calls `_updateRsETHPrice()`, so even the privileged manager path is equally affected.

## Impact Explanation
If `updateRSETHPrice()` exceeds the block gas limit, the `rsETHPrice` state variable becomes permanently stale. This concretely causes:
- **Incorrect minting**: `getRsETHAmountToMint()` in `LRTDepositPool.sol:520` reads `lrtOracle.rsETHPrice()` directly, so depositors receive rsETH at a stale rate.
- **Incorrect withdrawals**: `LRTWithdrawalManager` reads `lrtOracle.rsETHPrice()` for withdrawal amount calculations.
- **Downside protection disabled**: The price-drop pause logic at `LRTOracle.sol:270â€“281` is inside `_updateRsETHPrice()` and never executes if the function is uncallable.

**Impact: Medium â€” Unbounded gas consumption** (matching the allowed scope), with downstream effects on minting/withdrawal accuracy and safety mechanisms.

## Likelihood Explanation
No malicious actor is required. The gas cost grows through entirely normal protocol operation: adding supported LSTs, deploying more NDC contracts, and EigenLayer operators queuing or force-undelegating withdrawals. The team's own comment at `LRTUnstakingVault.sol:151` acknowledges the gas ceiling is near current operational parameters (80 tracked + up to 15 forced = 95). As the protocol scales in assets and NDCs, the multiplicative factor pushes the function toward and beyond the 30M block gas limit.

## Recommendation
1. **Cache `getAssetUnstaking` results off-chain** and push them on-chain via a privileged setter, rather than recomputing live inside `updateRSETHPrice()`.
2. **Decouple TVL accounting from price updates**: store per-NDC asset balances in a mapping updated lazily by operators; have `_getTotalEthInProtocol()` read from that mapping instead of making live external calls.
3. **Enforce per-NDC withdrawal caps** at the EigenLayer level (not just a global protocol counter) to bound the inner loop independently of forced undelegations.
4. **Benchmark gas cost** at maximum expected `supportedAssets Ă— NDC Ă— withdrawal Ă— strategy` cardinality and enforce hard limits that keep the function safely below the block gas limit.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
