### Title
Unvalidated Negative Chainlink `int256` Price Cast Triggers Solidity 0.8 Overflow Revert, Freezing All Deposits and rsETH Price Updates — (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` casts the raw `int256` Chainlink answer directly to `uint256` without first validating it is positive. In Solidity 0.8, if Chainlink returns a zero or negative value, the explicit cast silently wraps to a value near `type(uint256).max`, and the immediately following checked multiplication by `1e18` overflows and **reverts**. This makes `getAssetPrice()` permanently uncallable for the affected asset until an admin intervenes, halting all L1 deposits and rsETH price updates. The sibling contract `ChainlinkOracleForRSETHPoolCollateral.sol` already applies the correct guard, proving the protocol is aware of the pattern but failed to apply it to the core L1 oracle.

---

### Finding Description

`ChainlinkPriceOracle.sol` line 54:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`latestRoundData()` returns `int256 answer`. There is no guard against `price <= 0`. In Solidity 0.8:

- **Explicit type conversions** (e.g., `uint256(int256(-1))`) do **not** revert — they silently wrap. `uint256(-1)` = `type(uint256).max` ≈ 1.16 × 10^77.
- **Arithmetic operations** (`+`, `-`, `*`) **do** revert on overflow.

Therefore, if `price` is negative, `uint256(price)` wraps to a near-maximal value without reverting, and the subsequent `uint256(price) * 1e18` immediately overflows and **reverts** under Solidity 0.8's checked arithmetic. This is the direct analog to the TickMath pattern: a value that would have silently wrapped in Solidity <0.8 now causes a hard revert in 0.8.

The contrast with `ChainlinkOracleForRSETHPoolCollateral.sol` is explicit:

```solidity
// ChainlinkOracleForRSETHPoolCollateral.sol — line 32 (correct guard present)
if (ethPrice <= 0) revert InvalidPrice();
uint256 normalizedPrice = uint256(ethPrice) * 1e18 / 10 ** uint256(...decimals());
```

`ChainlinkPriceOracle.sol` has no equivalent guard.

---

### Impact Explanation

`ChainlinkPriceOracle.getAssetPrice()` is the price source for the entire L1 protocol:

1. `LRTOracle.getAssetPrice(asset)` → `LRTOracle._getTotalEthInProtocol()` → `LRTOracle._updateRsETHPrice()` / `updateRSETHPrice()` — rsETH price updates revert for all callers.
2. `LRTDepositPool.getRsETHAmountToMint()` calls `lrtOracle.getAssetPrice(asset)` → `LRTDepositPool.depositAsset()` and `depositETH()` revert for every user.

All deposits are frozen and the rsETH exchange rate cannot be updated until an admin replaces the price oracle. **Impact: Medium — Temporary freezing of funds.**

---

### Likelihood Explanation

Chainlink price feeds return `int256` and can return zero or negative values in documented edge cases: when the underlying price hits the feed's configured `minAnswer` circuit breaker (which can be set to 1 or even 0), during aggregator misconfiguration, or during extreme market events. The probability per block is low, but the missing guard is an internal code defect — not solely an external dependency issue — as demonstrated by the fact that `ChainlinkOracleForRSETHPoolCollateral.sol` in the same repository correctly validates the price before the identical cast-and-multiply pattern.

---

### Recommendation

Add a non-positive price check in `ChainlinkPriceOracle.getAssetPrice()` before the cast, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral.sol`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();

    if (price <= 0) revert InvalidPrice();  // add this guard

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

### Proof of Concept

1. Chainlink `latestRoundData()` returns `price = -1` for a supported LST asset (e.g., stETH/ETH feed hits its `minAnswer`).
2. `ChainlinkPriceOracle.getAssetPrice(stETH)` executes:
   - `uint256(-1)` = `type(uint256).max` = 115792089237316195423570985008687907853269984665640564039457584007913129639935 (no revert — explicit cast).
   - `type(uint256).max * 1e18` → integer overflow → **Solidity 0.8 reverts**.
3. `LRTOracle.getAssetPrice(stETH)` reverts (propagated).
4. `LRTOracle._getTotalEthInProtocol()` reverts → `LRTOracle.updateRSETHPrice()` reverts.
5. `LRTDepositPool.getRsETHAmountToMint(stETH, amount)` reverts → `LRTDepositPool.depositAsset(stETH, ...)` reverts for every depositor.
6. All L1 deposits are frozen until an admin calls `updatePriceFeedFor` to replace the oracle.
