### Title
rsETH Supply Snapshot Taken Before Fee Mint Causes Inflated Stored Price, Shortchanging Depositors - (File: contracts/LRTOracle.sol)

### Summary

In `LRTOracle._updateRsETHPrice()`, the `rsethSupply` snapshot is captured at line 216, before the protocol fee rsETH is minted to the treasury at line 306. The `newRsETHPrice` is computed using this pre-mint supply and then stored at line 313. Because the fee mint increases the actual total supply after the price is computed, the stored `rsETHPrice` is persistently higher than the true post-mint exchange rate, causing every subsequent depositor to receive fewer rsETH tokens than they are entitled to.

### Finding Description

`_updateRsETHPrice()` executes in this order:

1. **Line 216** — `rsethSupply = IRSETH(rsETHTokenAddress).totalSupply()` — supply snapshot taken.
2. **Line 250** — `newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply)` — price computed using the pre-mint supply.
3. **Line 306** — `IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee)` — fee rsETH minted, **increasing `totalSupply()`** but not `rsethSupply`.
4. **Line 313** — `rsETHPrice = newRsETHPrice` — the inflated price (computed with the smaller pre-mint denominator) is stored. [1](#0-0) [2](#0-1) [3](#0-2) 

Let:
- `S` = `rsethSupply` (pre-mint)
- `E` = `totalETHInProtocol`
- `F` = `protocolFeeInETH`
- `M` = fee rsETH minted = `F * S / (E − F)`

Stored price: `P_stored = (E − F) / S`

True post-mint price: `P_true = (E − F) / (S + M)`

Since `S + M > S`, we have `P_stored > P_true`. The ratio is `P_stored / P_true = (S + M) / S = 1 + F/(E − F)`.

### Impact Explanation

`getRsETHAmountToMint()` divides by `lrtOracle.rsETHPrice()` to determine how many rsETH tokens a depositor receives. [4](#0-3) 

Because `rsETHPrice` is inflated by the factor `(S + M)/S`, every depositor between two price updates receives `d * M / (E − F)` fewer rsETH tokens than they should for a deposit of `d` ETH. This value is silently redistributed to existing rsETH holders (including the treasury). The loss per depositor is proportional to `F / (E − F)` — i.e., the fee-to-net-TVL ratio. At a 10% protocol fee on 5% APY rewards on a $1 B TVL, this is roughly 0.001% per price update, but it compounds with every update and every depositor.

Additionally, `highestRsethPrice` may be set to the inflated `P_stored` rather than the true price, slightly raising the downside-protection pause threshold. [5](#0-4) 

**Impact classification:** Low — contract fails to deliver promised returns (depositors receive fewer rsETH than entitled), but no direct fund theft occurs.

### Likelihood Explanation

This triggers on every call to `updateRSETHPrice()` / `updateRSETHPriceAsManager()` whenever `protocolFeeInETH > 0` (i.e., whenever the protocol TVL has grown since the last update). Both functions are publicly callable (`updateRSETHPrice` has no access restriction beyond `whenNotPaused`), so any depositor or keeper can trigger the price update. The condition is met in normal protocol operation whenever staking rewards accrue. [6](#0-5) 

### Recommendation

Move the `rsethSupply` read to **after** the fee mint, so the stored price reflects the actual post-mint total supply:

```solidity
// After minting fee rsETH (line 306), re-read the supply:
uint256 rsethSupplyAfterMint = IRSETH(rsETHTokenAddress).totalSupply();
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupplyAfterMint);
rsETHPrice = newRsETHPrice;
```

Alternatively, compute `newRsETHPrice` using `rsethSupply + rsethAmountToMintAsProtocolFee` as the denominator before the mint occurs.

### Proof of Concept

Concrete numbers:
- `S` = 1,000,000 rsETH, `E` = 1,050,000 ETH, `F` = 5,000 ETH (10% fee on 50,000 ETH reward)
- `M` = 5,000 × 1,000,000 / 1,045,000 ≈ 4,785 rsETH minted to treasury
- `P_stored` = 1,045,000 / 1,000,000 = **1.045 ETH/rsETH**
- `P_true` = 1,045,000 / 1,004,785 ≈ **1.04001 ETH/rsETH**

A user depositing 1,000 ETH after the update:
- Receives `1,000 / 1.045` ≈ **957.85 rsETH** (using stored price)
- Should receive `1,000 / 1.04001` ≈ **961.53 rsETH** (using true price)
- **Loss: ~3.68 rsETH** (~0.46% of the deposit) per price update cycle in this scenario. [7](#0-6)

    function _updateRsETHPrice() internal {
        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;
        }

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

        // only take fee if TVL increased and protocol is not paused
        uint256 protocolFeeInETH = 0;
        if (!protocolPaused && totalETHInProtocol > previousTVL) {
            uint256 rewardAmount = totalETHInProtocol - previousTVL;
            protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
        }

        // compute new rsETH price based on total ETH minus fee
        uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

        if (newRsETHPrice > highestRsethPrice) {
            // check if the price is above the threshold
            uint256 priceDifference = newRsETHPrice - highestRsethPrice;
            // pricePercentageLimit is in 1e18 precision (100% = 1e18, 1% = 1e16)
            bool isPriceIncreaseOffLimit =
                pricePercentageLimit > 0 && priceDifference > pricePercentageLimit.mulWad(highestRsethPrice);

            // check if the price difference is above the threshold
            if (isPriceIncreaseOffLimit) {
                // if sender has a manager role, this doesn't revert.
                // if not, it reverts as price went above the threshold
                if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) {
                    revert PriceAboveDailyThreshold();
                }
            }
        }

        // downside protection — pause if price drops too far
        if (newRsETHPrice < highestRsethPrice) {
            uint256 diff = highestRsethPrice - newRsETHPrice;
            // normalizing to 1e18
            bool isPriceDecreaseOffLimit =
                pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);

            // if price decrease is off limit, pause the protocol (unless it's already paused)
            if (isPriceDecreaseOffLimit) {
                if (!lrtDepositPool.paused()) lrtDepositPool.pause();
                if (!withdrawalManager.paused()) withdrawalManager.pause();
                _pause();
                return;
            }

            // if price has decreased compared to the previous price, emit an event to reflect that
            if (previousPrice > newRsETHPrice) {
                emit RsETHPriceDecrease(newRsETHPrice, previousPrice);
            }

            // emit an event to notify that the price is currently below the peak (all time high) price
            emit RsETHPriceBelowPeak(highestRsethPrice, newRsETHPrice);
        }

        // update highest price if new price exceeds it
        if (newRsETHPrice > highestRsethPrice) {
            highestRsethPrice = newRsETHPrice;
        }

        // mint protocol fee as rsETH if there's a fee to take
        if (protocolFeeInETH > 0) {
            // Calculate rsETH amount to mint as protocol fee
            uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);

            _checkAndUpdateDailyFeeMintLimit(rsethAmountToMintAsProtocolFee);
            if (rsethAmountToMintAsProtocolFee > 0) {
                address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
                IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
                emit FeeMinted(treasury, rsethAmountToMintAsProtocolFee);
            }
        } else {
            _checkAndUpdateDailyFeeMintLimit(0);
        }

        rsETHPrice = newRsETHPrice;

        emit RsETHPriceUpdate(rsETHPrice, previousPrice);
    }
