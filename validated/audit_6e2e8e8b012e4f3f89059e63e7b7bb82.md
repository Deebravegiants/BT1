### Title
Unvalidated `ProposalInit.builder` Allows Malicious Proposer to Inject Arbitrary Sequencer Address into Block Execution — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`ProposalInit` carries a `builder` field (the sequencer/block-builder address). This field is passed verbatim into `convert_to_sn_api_block_info` as `sequencer_address` and is used for fee collection and the `get_sequencer_address` syscall during block execution. The proposal-validation path (`is_proposal_init_valid`) never checks `builder` against any locally-trusted value. A malicious-but-legitimate proposer can set `builder` to any address; all validators will accept the proposal and execute the block with that injected sequencer address.

### Finding Description

`ProposalInit` has two identity fields:

- `proposer` — the consensus participant who proposed the block
- `builder` — the address that builds/sequences the block (used as `sequencer_address` in execution) [1](#0-0) 

The `proposer` field is checked at the consensus layer: [2](#0-1) 

The `builder` field is **never checked**. `is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, L1 gas prices, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`, but contains no check on `builder`: [3](#0-2) 

The `ProposalInitValidation` struct — the sole source of locally-trusted reference values — has no `builder` field at all: [4](#0-3) 

After passing `is_proposal_init_valid`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`: [5](#0-4) 

This `BlockInfo` is forwarded to the batcher for block execution. The batcher uses `sequencer_address` for:

1. **Fee collection** — transaction fees are transferred to `sequencer_address` during execution.
2. **`get_sequencer_address` syscall** — contracts that call this syscall receive the injected address.
3. **Block header / state commitment** — `sequencer_address` is part of `BlockHeaderWithoutHash` and thus the block hash: [6](#0-5) 

The `ProposalFin` commitment comparison does **not** protect against this: the batcher computes the commitment using the injected `builder` address, so the commitment still matches the proposer's claimed value.

### Impact Explanation

A malicious proposer sets `builder` to an attacker-controlled address. All validators accept the proposal (no check exists), execute the block with that address as `sequencer_address`, and commit the resulting state. Consequences:

- **Fee theft**: All transaction fees for the block are transferred to the attacker's address instead of the legitimate sequencer.
- **Wrong block header**: The committed block header contains the injected sequencer address, producing a wrong state root and block hash that is nonetheless accepted by the entire network.
- **Syscall poisoning**: Any contract calling `get_sequencer_address` during that block receives the attacker's address, potentially breaking sequencer-gated logic.

This matches: *Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.*

### Likelihood Explanation

Requires the attacker to be a legitimate consensus validator who is selected as proposer for a round. In a decentralized validator set this is a routine event. No external capability beyond normal proposer rights is needed; the attack is a single-field modification to the outbound `ProposalInit` message.

### Recommendation

Add `builder` to `ProposalInitValidation` and check it in `is_proposal_init_valid`:

```rust
// In ProposalInitValidation:
pub expected_builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder),
    ));
}
```

The `expected_builder` value should be derived from the locally-configured builder address (the same source used in `initiate_build`), not from the network message.

### Proof of Concept

1. Attacker controls a validator node that is selected as proposer for block N.
2. In `initiate_build`, the attacker patches `builder: attacker_address` instead of `args.builder_address`.
3. The modified `ProposalInit` is streamed to all validators.
4. Each validator calls `is_proposal_init_valid` — passes (no `builder` check).
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)` → `sequencer_address = attacker_address`.
6. The batcher executes all transactions with `sequencer_address = attacker_address`; fees are transferred to the attacker.
7. `finish_proposal` returns a `partial_block_hash` computed with `attacker_address` as sequencer; the `ProposalFin` commitment matches.
8. Consensus reaches decision; the block is committed with the wrong sequencer address and all fees redirected. [7](#0-6) [8](#0-7) [9](#0-8)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
