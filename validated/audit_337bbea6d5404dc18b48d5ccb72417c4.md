Audit Report

## Title
Missing Chainlink Oracle Staleness and Validity Checks Allow Stale Price to Corrupt rsETH Exchange Rate - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice` calls `latestRoundData()` but discards every return value except `price`, performing no staleness check, no positive-price check, and no round-completeness check. Because `LRTOracle.updateRSETHPrice()` is a public, permissionless function, any caller can push a stale (artificially low) asset price into the stored `rsETHPrice`, allowing a subsequent deposit to mint excess rsETH and extract unclaimed yield from existing holders.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice` (lines 49–55) silently discards `updatedAt`, `roundId`, and `answeredInRound`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Three invariants are never enforced:
1. **Staleness** — `updatedAt` is never compared to `block.timestamp`. A feed that has not been updated for hours is accepted as current.
2. **Positive price** — `int256 price` is cast directly to `uint256`. A zero price returns `0`; a negative price wraps to a near-`type(uint256).max` value, causing an arithmetic overflow revert in Solidity 0.8.x downstream.
3. **Round completeness** — `answeredInRound >= roundId` is never verified.

The stale price flows through `LRTOracle.getAssetPrice` → `_getTotalEthInProtocol` (lines 331–349) → `_updateRsETHPrice` (lines 214–316), which writes the corrupted value to `rsETHPrice`. `updateRSETHPrice()` (line 87) is `public` with only a `whenNotPaused` guard, so any unprivileged caller can trigger this at will.

The `pricePercentageLimit` downside-protection check (lines 270–281) is a partial mitigation: it auto-pauses the protocol if the price drop exceeds the configured threshold. However, it provides no protection when `pricePercentageLimit == 0` (unset), and it does not fire when the stale price drop falls within the configured limit (e.g., a 3 % stale drop against a 5 % limit passes through silently).

## Impact Explanation
**High — Theft of unclaimed yield.**

When a Chainlink LST/ETH feed is stale and reports a price below the true market price, `_getTotalEthInProtocol` underestimates total ETH backing rsETH. `newRsETHPrice` is set below its true value and written to `rsETHPrice`. Any deposit made immediately after — via `LRTDepositPool.depositETH` or `depositAsset` on L1, or via `RSETHPoolV3.deposit` on L2 — uses the depressed rate to compute `rsethAmountToMint`, minting more rsETH than the deposited value warrants. This dilutes the accrued yield of all existing rsETH holders, constituting theft of unclaimed yield. Repeated exploitation trends toward protocol insolvency.

## Likelihood Explanation
Chainlink LST/ETH feeds (stETH/ETH, rETH/ETH, etc.) have heartbeat intervals of up to 24 hours. During network congestion or sequencer downtime the feed can remain stale for an extended window. Because `updateRSETHPrice()` is public, an attacker can monitor the feed's `updatedAt` timestamp off-chain and call the function precisely when the feed is stale but has not yet triggered the `pricePercentageLimit` auto-pause. The attack is repeatable every time the feed enters a stale window. Likelihood: **Medium**.

## Recommendation
Add staleness, positive-price, and round-completeness guards in `getAssetPrice`. Per-feed thresholds should be stored in a mapping since heartbeat intervals differ across feeds:

```solidity
mapping(address asset => uint256 maxStaleness) public maxStalenessPerFeed;

function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price, , uint256 updatedAt, uint80 answeredInRound)
        = priceFeed.latestRoundData();

    require(price > 0, "ChainlinkPriceOracle: non-positive price");
    uint256 staleness = maxStalenessPerFeed[asset];
    require(staleness > 0 && block.timestamp - updatedAt <= staleness,
            "ChainlinkPriceOracle: stale price");
    require(answeredInRound >= roundId, "ChainlinkPriceOracle: incomplete round");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept
1. The stETH/ETH Chainlink feed has a 24 h heartbeat; its last update was 23 h 50 min ago. The stale price is 0.97 ETH (true market: 1.00 ETH).
2. Attacker calls `LRTOracle.updateRSETHPrice()` (public, no role required). [1](#0-0) 
3. `getAssetPrice(stETH)` calls `ChainlinkPriceOracle.getAssetPrice`, which returns `0.97e18` with no staleness check. [2](#0-1) 
4. `_getTotalEthInProtocol()` underestimates total ETH by ~3 % of the stETH TVL. [3](#0-2) 
5. `newRsETHPrice` is computed ~3 % below its true value and written to `rsETHPrice`. The `pricePercentageLimit` check does not fire if the limit is ≥ 3 % or is unset (0). [4](#0-3) 
6. Attacker immediately calls `LRTDepositPool.depositETH{value: 10 ether}(0, "")`. The deposit pool reads the depressed `rsETHPrice` and mints ~3 % excess rsETH to the attacker.
7. The excess rsETH represents yield diluted from all existing rsETH holders.
8. **Foundry fork test**: fork mainnet, mock the stETH/ETH Chainlink feed to return a price last updated 24 h ago, call `updateRSETHPrice()`, record `rsETHPrice`, then call `depositETH` and assert that `rsethAmountMinted > depositValue / trueRsETHPrice`.
