### Title
Proposer Can Set Arbitrary `fee_proposal_fri` During Startup Window, Permanently Corrupting SNIP-35 Fee Market — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` explicitly skips bounds enforcement on `fee_proposal_fri` whenever `proposal_init_validation.fee_actual` is `None` — which is the case for the first `fee_proposal_window_size` blocks (startup / near-genesis). A malicious proposer can set `fee_proposal_fri` to any value (e.g., `0` or `u128::MAX`) during this unchecked window. Once the window fills, `fee_actual` is computed as the median of those extreme values, permanently locking the SNIP-35 fee market at an extreme price and bounding all future `fee_proposal_fri` values to that extreme range.

---

### Finding Description

In `is_proposal_init_valid`, the fee-proposal bounds check is guarded by a pattern match that silently skips enforcement when `fee_actual` is `None`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    ...
}
``` [1](#0-0) 

`fee_actual` is `None` whenever `compute_fee_actual` returns `None`, which happens for the first `window_size` blocks because the sliding window has not yet accumulated enough entries:

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ({window_size})");
    return None;
};
``` [2](#0-1) 

An honest proposer freezes `fee_proposal_fri` at `l2_gas_price` during this period:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [3](#0-2) 

But the validator does **not** enforce this fallback value. Any `fee_proposal_fri` value — including `0` or `u128::MAX` — passes `is_proposal_init_valid` during the startup window.

The accepted `fee_proposal_fri` is then recorded into `fee_proposals_window` via `record_fee_proposal` at decision time:

```rust
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [4](#0-3) 

Once the window fills, `fee_actual` is the median of those stored values. If a malicious proposer set `fee_proposal_fri = 0` for all `window_size` blocks, then `fee_actual = 0`. The bounds function then produces:

```rust
pub(crate) fn fee_proposal_bounds(fee_actual: GasPrice, margin_ppt: u128) -> (u128, u128) {
    let fee_actual_u256 = U256::from(fee_actual.0); // 0
    let upper = u128::try_from(fee_actual_u256 * scaled / denom).unwrap_or(u128::MAX); // 0
    let lower = u128::try_from(fee_actual_u256 * denom / scaled).unwrap_or(0);         // 0
    (0, 0)
}
``` [5](#0-4) 

All future proposers are then forced to set `fee_proposal_fri = 0`, permanently locking the SNIP-35 fee market at zero. The corrupted value is also forwarded to the centralized cende pipeline:

```rust
fee_proposal_info: FeeProposalInfo { fee_proposal_fri: init.fee_proposal_fri },
``` [6](#0-5) 

And stored in `BlockHeaderWithoutHash.fee_proposal_fri`, which is persisted to storage and propagated via state sync:

```rust
fee_proposal_fri: init.fee_proposal_fri,
``` [7](#0-6) 

---

### Impact Explanation

**Impact: High — Wrong authoritative fee-market value with economic effect.**

The `fee_proposal_fri` field is the SNIP-35 oracle-derived L2 gas fee recommendation. It is:
1. Stored in `BlockHeaderWithoutHash` and persisted to storage.
2. Propagated to all peers via state sync.
3. Used to compute `fee_actual` (the median sliding window), which bounds all future `fee_proposal_fri` values.
4. Forwarded to the centralized cende pipeline as authoritative fee-market data.

Setting `fee_proposal_fri = 0` during the startup window permanently locks `fee_actual = 0`, forcing all future proposers to also emit `fee_proposal_fri = 0` (since `fee_proposal_bounds(0, margin) = (0, 0)`). This permanently disables the SNIP-35 fee adjustment mechanism and corrupts the fee-market data stored on-chain and in the centralized pipeline. Setting `fee_proposal_fri = u128::MAX` has the symmetric effect of locking fees at the maximum.

The `fee_proposal_fri` is also bound into the `ProposalCommitment` via Poseidon hash, so the corrupted value is signed over by all validators:

```rust
ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
``` [8](#0-7) 

---

### Likelihood Explanation

**Likelihood: Low-Medium.**

The attack window is limited to the first `fee_proposal_window_size` blocks of a new chain or after a restart/revert. The attacker must be selected as proposer during this window, which requires being a consensus participant. However, in a Tendermint-based system, any validator can be selected as proposer in their round, so any malicious validator can exploit this during the startup window. The window is also predictable (genesis or restart), making it a known attack surface.

---

### Recommendation

When `fee_actual` is `None` (startup window), the validator should enforce that `fee_proposal_fri` equals the expected fallback value (`l2_gas_price`), mirroring what honest proposers emit:

```rust
// When fee_actual is None, enforce the fallback: fee_proposal must equal l2_gas_price.
if let (None, Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    if fee_proposal != proposal_init_validation.l2_gas_price_fri {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "fee_proposal during startup must equal l2_gas_price: expected={}, got={}",
                proposal_init_validation.l2_gas_price_fri.0, fee_proposal.0
            ),
        ));
    }
}
```

This ensures the startup window cannot be used to seed the `fee_proposals_window` with arbitrary values.

---

### Proof of Concept

1. Chain starts at height 0 (or restarts). `fee_proposals_window` is empty; `fee_actual = None` for heights `0..window_size`.
2. A malicious validator is selected as proposer for any block in `[0, window_size)`.
3. The proposer constructs a `ProposalInit` with `fee_proposal_fri = GasPrice(0)` (or `u128::MAX`).
4. `is_proposal_init_valid` is called. The check at line 398 evaluates `(None, Some(0))` — the `if let` does not match, so the bounds check is skipped entirely. The proposal is accepted.
5. `decision_reached` calls `record_fee_proposal(height, Some(GasPrice(0)))`, inserting `0` into `fee_proposals_window`.
6. Steps 2–5 repeat for all `window_size` blocks.
7. At height `window_size`, `compute_fee_actual` returns `Some(GasPrice(0))` (median of all zeros).
8. `fee_proposal_bounds(GasPrice(0), margin_ppt)` returns `(0, 0)`.
9. All future proposers must set `fee_proposal_fri = 0`; any other value is rejected by validators.
10. The SNIP-35 fee market is permanently locked at zero. The corrupted value is stored in every block header and forwarded to the centralized pipeline.
