### Title
Unguarded `assert` in `__get_cc_sub_slot` Crashes Weight-Proof Validation via Crafted Peer Proof — (File: `chia/full_node/weight_proof.py`)

### Summary
A malicious peer can send a crafted `WeightProof` containing a non-first segment whose challenge block sits at position 0 in `sub_slots`. When that segment is sampled for validation, `__get_cc_sub_slot` is called with `idx=0`, causing `range(0)` to produce an empty iterator, leaving `sub_slot = None`, and triggering `assert sub_slot is not None` — an `AssertionError` that crashes weight-proof validation.

### Finding Description

`__get_cc_sub_slot` searches backwards through `sub_slots[0:idx]` for a slot-end entry:

```python
def __get_cc_sub_slot(sub_slots, idx, ses):
    sub_slot: SubSlotData | None = None
    for i in reversed(range(idx)):   # range(0) is empty when idx == 0
        sub_slot = sub_slots[i]
        if sub_slot.cc_slot_end_info is not None:
            break

    assert sub_slot is not None          # line 1539 — AssertionError when idx == 0
    assert sub_slot.cc_slot_end_info is not None
``` [1](#0-0) 

`__get_cc_sub_slot` is called from `__validate_pospace` whenever the genesis-challenge shortcut does not apply:

```python
if first_in_sub_epoch and segment.sub_epoch_n == 0 and idx == 0:
    cc_sub_slot_hash = constants.GENESIS_CHALLENGE
else:
    cc_sub_slot_hash = __get_cc_sub_slot(segment.sub_slots, idx, ses).get_hash()
``` [2](#0-1) 

For any non-first segment (`first_in_sub_epoch = False`) or any segment in `sub_epoch_n > 0`, the `else` branch is taken unconditionally, even when `idx == 0`.

**Why the existing `__get_rc_sub_slot` fix does not protect this path.**

`_validate_sub_epoch_segments` calls `__get_rc_sub_slot` only for `segments[0]` of each sub-epoch:

```python
if sub_epoch_n > 0:
    rc_sub_slot = __get_rc_sub_slot(constants, segments[0], summaries, curr_ssi)
    if rc_sub_slot is None:
        return None
``` [3](#0-2) 

Subsequent segments (`segments[1]`, `segments[2]`, …) are passed directly to `_validate_segment` without any equivalent guard. A malicious peer can craft `segments[1]` with its challenge block at position 0 in `sub_slots`; `__get_rc_sub_slot` is never called for it, so the crash path is fully open.

The test suite explicitly documents the analogous crash in `__get_rc_sub_slot` and confirms it was a real, exploitable `AssertionError`:

> "Pre-fix: AssertionError in `__get_rc_sub_slot` crashes validation. Post-fix: the malformed segment causes a hash mismatch, returning `(False, 0)` cleanly." [4](#0-3) 

`__get_cc_sub_slot` was never given the same treatment.

### Impact Explanation

An `AssertionError` raised inside `_validate_sub_epoch_segments` propagates through `validate_weight_proof_inner` and `validate_weight_proof`. Depending on where the exception is caught in the full-node sync path, this causes either a crash of the syncing process or an unhandled task failure. Either outcome prevents the targeted node from completing sync for as long as the attacker can re-deliver the malicious proof. This maps to the **High** impact class: "Permanent or long-lived inability for honest nodes … to process … sync updates … under normal network assumptions."

### Likelihood Explanation

Any peer that can respond to a `request_proof_of_weight` message can deliver a crafted `WeightProof`. The attacker controls `sub_epoch_data`, which seeds the RNG (`summaries[-2].get_hash()`), so they can deterministically choose the `sampled_seg_index` that selects their malformed segment. No keys, admin access, or cryptographic break are required. The maximum allowed segment count per sub-epoch is ~25, so the attacker has ample room to include a valid `segments[0]` (to pass `__get_rc_sub_slot`) alongside a malformed `segments[1]`. [5](#0-4) 

### Recommendation

Replace the bare `assert` statements in `__get_cc_sub_slot` with explicit guards that return an error sentinel (e.g., `None`) and log the malformed input, mirroring the fix already applied to `__get_rc_sub_slot`:

```python
def __get_cc_sub_slot(sub_slots, idx, ses):
    sub_slot: SubSlotData | None = None
    for i in reversed(range(idx)):
        sub_slot = sub_slots[i]
        if sub_slot.cc_slot_end_info is not None:
            break

    if sub_slot is None or sub_slot.cc_slot_end_info is None:
        log.error("segment missing slot-end entry before challenge block (cc)")
        return None   # caller must handle None

    ...
```

Callers of `__get_cc_sub_slot` (currently `__validate_pospace`) must then propagate `None` upward rather than calling `.get_hash()` on it.

### Proof of Concept

1. Craft a `WeightProof` for `sub_epoch_n = 1` with two segments:
   - `segments[0]`: structurally valid (passes `__get_rc_sub_slot`).
   - `segments[1]`: `sub_slots = [challenge_sub_slot_data]` — a single entry with `cc_slot_end = None` (challenge block) at index 0.
2. Adjust `sub_epoch_data` so that `summaries[-2].get_hash()` seeds the RNG to produce `sampled_seg_index = 1`.
3. Deliver this proof in response to a `request_proof_of_weight` message from a syncing full node.
4. The node calls `validate_weight_proof` → `_validate_sub_epoch_segments` → `_validate_segment(segments[1], sampled=True)` → `__validate_pospace(segment, idx=0, first_in_sub_epoch=False)` → `__get_cc_sub_slot(sub_slots, 0, ses)` → `assert sub_slot is not None` → `AssertionError`.     constants: ConsensusConstants,
    segment: SubEpochChallengeSegment,
    summaries: list[SubEpochSummary],
    curr_ssi: uint64,
) -> RewardChainSubSlot | None:
    ses = summaries[uint32(segment.sub_epoch_n - 1)]
    # find first challenge in sub epoch
    first_idx = None
    first = None
    for idx, curr in enumerate(segment.sub_slots):
        if curr.cc_slot_end is None:
            first_idx = idx
            first = curr
            break

    if first_idx is None or first is None or first.signage_point_index is None:
        log.error("segment missing challenge block or signage_point_index")
        return None
    idx = first_idx
    slots = segment.sub_slots

    # number of slots to look for
    slots_n = 1
    if is_overflow_block(constants, first.signage_point_index):
        if idx >= 2 and slots[idx - 2].cc_slot_end is None:
            slots_n = 2

    new_diff = None if ses is None else ses.new_difficulty
    new_ssi = None if ses is None else ses.new_sub_slot_iters
    ses_hash: bytes32 | None = None if ses is None else ses.get_hash()
    overflow = is_overflow_block(constants, first.signage_point_index)
    if overflow:
        if idx >= 2 and slots[idx - 2].cc_slot_end is not None and slots[idx - 1].cc_slot_end is not None:
            ses_hash = None
            new_ssi = None
            new_diff = None

    sub_slot = slots[idx]
    while True:
        if sub_slot.cc_slot_end:
            slots_n -= 1
            if slots_n == 0:
                break
        idx -= 1
        if idx < 0:
            log.error("malformed segment: no slot-end entry before challenge block")
            return None
        sub_slot = slots[idx]
