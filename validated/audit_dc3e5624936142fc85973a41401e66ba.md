### Title
Dijkstra Era Subtransactions Allow Double-Claiming of Stake Credential Deposit Refunds, Creating ADA From Nothing - (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/TxCert.hs`)

---

### Summary

In the Dijkstra era, the `dijkstraTotalRefundsTxCerts` function reads deposit refund amounts directly from certificate fields rather than from the ledger state. A disabled test explicitly documents that the predicate preventing multiple subtransactions from claiming the same deposit refund **has not been implemented**. An unprivileged user can craft a top-level transaction containing N subtransactions each bearing an `UnRegDepositTxCert` for the same staking credential, causing the deposit pot to be decremented N times while only one deposit was ever paid, directly creating ADA from nothing.

---

### Finding Description

**Root cause — `dijkstraTotalRefundsTxCerts` ignores ledger state**

In Conway and earlier eras, `getTotalRefundsTxCerts` receives a lookup function that resolves the actual deposit stored in the ledger for each credential:

```haskell
-- Conway
conwayTotalRefundsTxCerts pp lookupStakingDeposit lookupDRepDeposit certs =
  shelleyTotalRefundsTxCerts pp lookupStakingDeposit certs
    <+> conwayDRepRefundsTxCerts lookupDRepDeposit certs
```

In Dijkstra, the lookup arguments are discarded entirely and the refund is read straight from the certificate body:

```haskell
-- Dijkstra (eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/TxCert.hs:228-238)
dijkstraTotalRefundsTxCerts = foldMap' $ \case
  UnRegDepositTxCert _ deposit -> deposit
  UnRegDRepTxCert  _ deposit -> deposit
  _ -> zero

-- line 285
getTotalRefundsTxCerts _ _ _ = dijkstraTotalRefundsTxCerts
```

This value is consumed by `Shelley.updateUTxOStateNoFees` (called from `dijkstraSubUtxoTransition`) to decrement `utxosDeposited`. Because the amount comes from the certificate and not from a single authoritative ledger lookup, every subtransaction that carries an `UnRegDepositTxCert` for the same credential will independently contribute its stated deposit to the total refund calculation.

**Acknowledged missing guard — disabled test**

The test suite for the Dijkstra era explicitly records that the cross-subtransaction guard is absent:

```haskell
-- eras/dijkstra/impl/testlib/Test/Cardano/Ledger/Dijkstra/Imp/CertSpec.hs:53-75
xit "Multiple subtransactions cannot get the same refund" $ do
  stakingCred <- KeyHashObj <$> freshKeyHash
  _ <- registerStakeCredential stakingCred
  keyDeposit <- getsPParams ppKeyDepositL
  ...
  let
    subTx1 = ... & bodyTxL . certsTxBodyL .~ SSeq.singleton
                     (UnRegDepositTxCert stakingCred keyDeposit)
    subTx2 = ... & bodyTxL . certsTxBodyL .~ SSeq.singleton
                     (UnRegDepositTxCert stakingCred keyDeposit)
    tx = mkBasicTx mkBasicTxBody
           & bodyTxL . subTransactionsTxBodyL .~ OMap.fromFoldable [subTx1, subTx2]
  submitFailingTx tx . NE.singleton $
    error "TODO: predicate failure not yet implemented"
```

`xit` disables the test; the body contains `error "TODO: predicate failure not yet implemented"`, confirming that no predicate failure type exists yet to reject this transaction. The transaction is therefore accepted by the ledger.

**How the accounting breaks**

`updateUTxOStateNoFees` adjusts `utxosDeposited` by `deposits - refunds` for each subtransaction. With N subtransactions each carrying `UnRegDepositTxCert cred D`:

```
utxosDeposited after subTx1 = utxosDeposited₀ − D
utxosDeposited after subTx2 = utxosDeposited₀ − 2D
...
utxosDeposited after subTxN = utxosDeposited₀ − N·D
```

Only one deposit of D was ever paid. The deposit pot is driven below its true obligation by `(N−1)·D`, and the corresponding UTxO outputs created by the subtransactions collectively receive `N·D` in refund value. The difference `(N−1)·D` is ADA created from nothing.

---

### Impact Explanation

**Critical — Direct creation of ADA through an invalid ledger state transition.**

The `utxosDeposited` invariant (`utxosDeposited = sumObligation (obligationCertState certState)`) is violated: the deposit pot is decremented more than the actual obligations warrant. The excess lovelace appears in UTxO outputs, permanently inflating the total ADA supply without any corresponding reserve or treasury debit. Recovery would require a hard fork to correct the ledger state.

---

### Likelihood Explanation

The Dijkstra era is under active development and the nested-transaction feature is new. The missing guard is explicitly acknowledged in the test suite. Any user who can submit a valid Dijkstra-era transaction (no special role required) can trigger this by constructing a top-level transaction with two or more subtransactions that each carry an `UnRegDepositTxCert` for the same credential. The attack requires only a single on-chain transaction and produces an immediate, irreversible accounting error.

---

### Recommendation

1. **Aggregate deregistrations across all subtransactions before processing any of them.** Before the `SUBLEDGERS` rule applies individual subtransactions, collect the full set of `(credential, deposit)` pairs from every subtransaction's certificate list and verify that no credential appears more than once.

2. **Implement the missing predicate failure.** Define a `DijkstraSubLedgerPredFailure` (or equivalent) variant such as `DuplicateUnRegDepositAcrossSubTxs` and wire it into the `SUBLEDGERS` / `SUBUTXO` transition so the disabled test can be re-enabled with a concrete expected failure.

3. **Restore ledger-state lookup for refunds in subtransactions.** Even though Dijkstra certificates carry the deposit amount for convenience, the `SUBLEDGER` rule should cross-check the stated amount against the actual deposit recorded in `DState` (via `lookupDepositDState`) to prevent both over-claiming and incorrect amounts.

---

### Proof of Concept

```
1. Alice registers staking credential C, paying keyDeposit D.
   → utxosDeposited increases by D.

2. Alice constructs:
     subTx1 = { certs: [UnRegDepositTxCert C D], inputs: [utxo1] }
     subTx2 = { certs: [UnRegDepositTxCert C D], inputs: [utxo2] }
     topTx  = { subTransactions: [subTx1, subTx2] }

3. topTx is submitted. Because no cross-subtransaction guard exists
   (predicate failure not yet implemented), both subtransactions are
   accepted.

4. dijkstraTotalRefundsTxCerts reads D from each certificate:
     subTx1 refund = D  →  utxosDeposited decremented by D
     subTx2 refund = D  →  utxosDeposited decremented by D again

5. Alice's UTxO outputs from subTx1 and subTx2 together contain 2·D.
   Only D was ever deposited.
   Net ADA created: D (= keyDeposit, currently 2 ADA on mainnet).
   Repeating with N subtransactions yields (N−1)·D ADA per transaction.
