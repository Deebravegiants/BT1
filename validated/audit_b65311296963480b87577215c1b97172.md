### Title
Unprotected `L1GasPriceProvider::initialize()` Allows Re-initialization, Wiping Accumulated Gas Price History and Enabling Gas Price Manipulation - (File: crates/apollo_l1_gas_price/src/l1_gas_price_provider.rs)

### Summary

`L1GasPriceProvider::initialize()` contains no idempotency guard. Any caller who can reach the `RemoteL1GasPriceServer` endpoint can send `L1GasPriceRequest::Initialize` to silently replace the live `RingBuffer<GasPriceData>` with a fresh empty one, erasing all accumulated L1 gas price history. Because the same unauthenticated channel also accepts `L1GasPriceRequest::AddGasPrice`, an attacker can follow the wipe with injected fake price samples, causing the validator's L1 gas price margin check inside `is_proposal_init_valid` to accept proposals whose `l1_gas_price_fri` / `l1_data_gas_price_fri` fields are anchored to attacker-controlled values.

### Finding Description

`L1GasPriceProvider::initialize()` unconditionally overwrites `price_samples_by_block`:

```rust
pub fn initialize(&mut self) -> L1GasPriceProviderResult<()> {
    info!("Initializing L1GasPriceProvider with config: {:?}", self.config);
    self.price_samples_by_block = Some(RingBuffer::new(self.config.storage_limit));
    Ok(())
}
``` [1](#0-0) 

There is no check whether `price_samples_by_block` is already `Some(...)`. Compare this with the analogous `L1EventsProvider::initialize()`, which explicitly panics if called outside the `Uninitialized` state:

```rust
if !self.state.is_uninitialized() {
    panic!("Called initialize while not in Uninitialized state...");
}
``` [2](#0-1) 

The `initialize` operation is exposed as a first-class, unauthenticated message variant in the component protocol:

```rust
pub enum L1GasPriceRequest {
    Initialize,
    GetGasPrice(BlockTimestamp),
    AddGasPrice(GasPriceData),
    ...
}
``` [3](#0-2) 

The `ComponentRequestHandler` dispatches it with no caller verification:

```rust
L1GasPriceRequest::Initialize => L1GasPriceResponse::Initialize(self.initialize()),
``` [4](#0-3) 

The `L1GasPriceProviderClient` trait exposes `initialize()` as a public async method callable by any holder of the client handle, including the `RemoteComponentClient` path that communicates over a network socket: [5](#0-4) 

### Impact Explanation

**Phase 1 – History wipe (DoS on gas price validation):**  
After re-initialization, `price_samples_by_block` is an empty `RingBuffer`. Every call to `get_price_info()` returns `MissingDataError` until the scraper repopulates the buffer. During this window, `get_l1_prices_in_fri_and_wei` (called from `is_proposal_init_valid`) fails, causing all incoming proposals to be rejected by the validator. This stalls consensus. [6](#0-5) 

**Phase 2 – Gas price poisoning (wrong fee/gas accounting):**  
Because `AddGasPrice` is equally unauthenticated, an attacker can immediately follow the wipe with `add_price_info()` calls carrying attacker-chosen `base_fee_per_gas` and `blob_fee` values. The validator's margin check in `is_proposal_init_valid` then compares the proposer's `l1_gas_price_fri` against a reference derived from the poisoned buffer. A proposer colluding with the attacker can embed artificially low L1 gas prices in `ProposalInit`, and the validator will accept them, resulting in blocks committed with incorrect L1 gas price fields — a direct economic impact on fee accounting. [7](#0-6) 

### Likelihood Explanation

In the distributed microservices deployment, `RemoteL1GasPriceServer` listens on a configured TCP port with no application-layer authentication. Any process that can reach that port — a compromised co-located service, a misconfigured Kubernetes network policy, or an attacker on the same internal network segment — can send `L1GasPriceRequest::Initialize`. The scraper itself calls `initialize()` exactly once at startup: [8](#0-7) 

There is no token, nonce, or identity check preventing a second call. The `L1EventsProvider` demonstrates the correct pattern (state guard + panic) exists in the same codebase, confirming the omission in `L1GasPriceProvider` is unintentional.

### Recommendation

Add an already-initialized guard to `L1GasPriceProvider::initialize()`, mirroring the pattern used in `L1EventsProvider`:

```rust
pub fn initialize(&mut self) -> L1GasPriceProviderResult<()> {
    if self.price_samples_by_block.is_some() {
        return Err(L1GasPriceProviderError::AlreadyInitializedError);
    }
    info!("Initializing L1GasPriceProvider with config: {:?}", self.config);
    self.price_samples_by_block = Some(RingBuffer::new(self.config.storage_limit));
    Ok(())
}
```

Add a test asserting that a second call to `initialize()` returns an error rather than silently resetting state.

### Proof of Concept

```
1. Provider starts normally; scraper calls initialize() → RingBuffer populated with N blocks.
2. Attacker sends L1GasPriceRequest::Initialize to RemoteL1GasPriceServer.
   → price_samples_by_block replaced with empty RingBuffer (all history lost).
3. Attacker sends L1GasPriceRequest::AddGasPrice(GasPriceData { block_number: 0,
       timestamp: T, price_info: PriceInfo { base_fee_per_gas: 1, blob_fee: 1 } })
   → Buffer now contains one entry with gas price = 1 wei.
4. Proposer (colluding) builds a ProposalInit with l1_gas_price_fri derived from 1 wei.
5. Validator calls is_proposal_init_valid → get_l1_prices_in_fri_and_wei reads the
   poisoned buffer → reference price = 1 wei → margin check passes.
6. Validator accepts the proposal; block is committed with l1_gas_price_fri = 1 wei,
   causing all transactions in that block to pay near-zero L1 gas fees.
