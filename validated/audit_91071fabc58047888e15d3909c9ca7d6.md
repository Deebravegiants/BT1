### Title
Hardcoded Default Keyring Passphrase Allows Decryption of All Private Keys Without User Authorization - (File: chia/util/keyring_wrapper.py)

### Summary
Chia's `FileKeyring` encrypts private keys using a symmetric key derived from a master passphrase. When no user-supplied master passphrase is set (the default state for most users), the keyring is encrypted using a hardcoded, publicly known passphrase: `"$ chia passphrase set # all the cool kids are doing it!"`. Any party that can read the `keyring.yaml` file — including malicious co-resident processes, cloud backup services, or any process running as the same OS user — can decrypt all stored private keys and gain full, unauthorized signing authority over every wallet managed by that node.

### Finding Description
In `chia/util/keyring_wrapper.py` line 26, a static fallback passphrase is defined:

```python
DEFAULT_PASSPHRASE_IF_NO_MASTER_PASSPHRASE = "$ chia passphrase set # all the cool kids are doing it!"
``` [1](#0-0) 

This constant is used in two critical paths:

1. **Initial passphrase bootstrap** — `_get_initial_cached_passphrase` falls back to this constant when no OS credential store entry exists, meaning every fresh install or passphrase-less user starts with this known value as the active encryption key. [2](#0-1) 

2. **Explicit passphrase removal** — `remove_master_passphrase` re-encrypts the entire keyring with this constant, so users who deliberately remove their passphrase are left with a publicly known encryption key. [3](#0-2) 

The keyring file (`keyring.yaml`) is encrypted using ChaCha20Poly1305 with a key derived via PBKDF2 from this passphrase and a stored salt. Both the salt and the ciphertext are stored in plaintext in the file. [4](#0-3) [5](#0-4) 

The decrypted keyring contains raw BLS12-381 private key bytes concatenated with 32-byte entropy (the mnemonic seed), stored per key slot. [6](#0-5) 

### Impact Explanation
An attacker who obtains a copy of `keyring.yaml` — via cloud sync (Dropbox, iCloud, OneDrive), backup software, a co-resident malicious process running as the same OS user, or any other file-read path — can immediately decrypt all private keys using the publicly known passphrase. With the recovered BLS private keys, the attacker can sign arbitrary spend bundles, redirect all XCH/CAT/NFT/DID balances, and take full control of pool wallets and singleton-controlled assets. This satisfies the **High** impact criterion: bypass of keychain authorization enabling unauthorized signing and coin control.

### Likelihood Explanation
The default state for the majority of Chia users is no master passphrase. The constant is in a public open-source repository. Cloud backup of the `~/.chia` directory is common. A malicious application running as the same OS user (e.g., a compromised npm package in a developer environment) can read the file without any privilege escalation.

### Recommendation
- Replace the hardcoded default passphrase with a per-installation randomly generated secret stored in the OS credential store (macOS Keychain / Windows Credential Manager / libsecret on Linux), generated at first run.
- If no OS credential store is available, require the user to set a passphrase before any keys are written to disk.
- At minimum, document prominently that without a user-set passphrase, the keyring file is encrypted with a publicly known key and provides no confidentiality if the file is exfiltrated.

### Proof of Concept

```python
import base64, hashlib, yaml
from cryptography.hazmat.primitives.ciphers.aead import ChaCha20Poly1305

CHECKBYTES_VALUE = b"5f365b8292ee505b"
DEFAULT_PASSPHRASE = "$ chia passphrase set # all the cool kids are doing it!"

# Load the victim's keyring.yaml (readable by any same-user process)
with open("/home/victim/.chia/mainnet/config/ssl/../../../keyring.yaml") as f:
    kr = yaml.safe_load(f)

salt  = bytes.fromhex(kr["salt"])
nonce = bytes.fromhex(kr["nonce"])
data  = base64.b64decode(kr["data"])

# Derive the symmetric key exactly as Chia does
key = hashlib.pbkdf2_hmac("sha256", DEFAULT_PASSPHRASE.encode(), salt, 100000)

# Decrypt
plaintext = ChaCha20Poly1305(key).decrypt(nonce, data, None)
assert plaintext[:16] == CHECKBYTES_VALUE

import yaml as y
decrypted = y.safe_load(plaintext[16:])
# decrypted now contains all private key bytes (pubkey || entropy per slot)
print(decrypted)
```

The recovered entropy bytes can be converted directly to the 24-word mnemonic seed, giving the attacker complete, irrevocable control of all wallets.

    def _get_initial_cached_passphrase(self) -> str:
        """
        Grab the saved passphrase from the OS credential store (if available), otherwise

        """
        from chia.util.keychain import supports_os_passphrase_storage

        passphrase: str | None = None

        if supports_os_passphrase_storage():
