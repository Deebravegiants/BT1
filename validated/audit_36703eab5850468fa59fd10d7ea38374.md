### Title
Unbounded `fee_proposal_fri` Accepted During Startup Window Enables Persistent L2 Gas Price Inflation — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

In `is_proposal_init_valid()`, the validator explicitly skips bounds-checking on the proposer-supplied `fee_proposal_fri` whenever `fee_actual` is `None`. `fee_actual` is `None` for the first `fee_proposal_window_size` (10) blocks after V0_14_3 activation — including the entire V0_14_2→V0_14_3 upgrade transition window. A malicious proposer who controls the majority of proposer slots during this window can inject an arbitrarily large `fee_proposal_fri` value. Once those values dominate the sliding-window median, `fee_actual` is permanently inflated, which raises the `effective_min` floor in `calculate_next_l2_gas_price_for_fin()`, causing all subsequent users to pay inflated L2 gas fees.

---

### Finding Description

**Broken invariant — unchecked proposer-controlled value enters the fee-price floor.**

In `is_proposal_init_valid()`, after all other checks pass, the code reaches:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(...);
    }
}
``` [1](#0-0) 

When `proposal_init_validation.fee_actual` is `None`, the entire `if let` body is skipped. No bound is placed on `init_proposed.fee_proposal_fri`. The validator accepts any value — including `u128::MAX`.

`fee_actual` is `None` whenever `compute_fee_actual()` cannot fill a complete window of `window_size` (10) entries with `Some(GasPrice)` values:

```rust
Some(None) | None => {
    warn!(...);
    return None;
}
``` [2](#0-1) 

This condition holds for the first 10 blocks after V0_14_3 activation (genesis) **and** for the entire transition period after the V0_14_2→V0_14_3 upgrade, because pre-V0_14_3 blocks carry `fee_proposal_fri = None`, and `compute_fee_actual()` returns `None` if any window entry is `None`.

The accepted `fee_proposal_fri` is recorded into `fee_proposals_window`:

```rust
fn record_fee_proposal(&mut self, height: BlockNumber, fee_proposal_fri: Option<GasPrice>) {
    self.fee_proposals_window.insert(height, fee_proposal_fri);
}
``` [3](#0-2) 

Once the window fills with 10 V0_14_3 blocks, `compute_fee_actual()` returns the median. If the malicious proposer controlled ≥6 of those 10 blocks and set `fee_proposal_fri = u128::MAX`, the median is `u128::MAX`.

This inflated `fee_actual` is then used as the `effective_min` floor in `calculate_next_l2_gas_price_for_fin()`:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [4](#0-3) 

`calculate_next_base_gas_price()` enforces `GasPrice(max(adjusted_price, min_gas_price.0))`, so the L2 gas price can never fall below `effective_min`. With `effective_min = u128::MAX`, the price ratchets upward at 0.3% per block (governed by `MIN_GAS_PRICE_INCREASE_DENOMINATOR = 333`) until it reaches `u128::MAX`. [5](#0-4) 

Once `fee_actual` is inflated, recovery is extremely slow. Honest proposers can only reduce `fee_proposal_fri` by ±0.2% per block (`fee_proposal_margin_ppt = 2`), so the window median decreases by roughly 2% per 10 blocks. Recovering from `u128::MAX` to 8 gwei takes on the order of 22,000 blocks (~16 hours at 2.6 s/block). [6](#0-5) 

---

### Impact Explanation

**Impact: Critical — Incorrect fee/gas price with economic impact.**

The inflated `next_l2_gas_price` is written into `BlockHeaderWithoutHash.next_l2_gas_price` and propagated to all subsequent blocks:

```rust
let block_header_without_hash = BlockHeaderWithoutHash {
    ...
    next_l2_gas_price: self.l2_gas_price,
    fee_proposal_fri: init.fee_proposal_fri,
    ...
};
``` [7](#0-6) 

Every transaction executed against those blocks is charged at the inflated L2 gas price. Users who set a high `max_price_per_unit` (e.g., for urgent transactions) pay the inflated fee, draining their STRK balance. Users with lower bounds have their transactions rejected. The effect persists for thousands of blocks.

---

### Likelihood Explanation

**Likelihood: Low-Medium.**

The attack requires the malicious proposer to control ≥6 of the 10 startup-window proposer slots. In a BFT network with deterministic proposer selection (stake-weighted), a validator holding ≥50% of stake achieves this with certainty. A validator holding less stake has a non-negligible probability (e.g., ~37% for a 1-of-10 equal-stake validator). The attack window is predictable and occurs at every V0_14_3 upgrade boundary and at genesis.

The validator does not reject the inflated value because the `if let (Some(fee_actual), Some(fee_proposal))` guard silently passes when `fee_actual` is `None` — there is no explicit rejection path for the `(None, Some(_))` case at V0_14_3+.

---

### Recommendation

1. **Enforce an absolute cap on `fee_proposal_fri` when `fee_actual` is `None`.** During the startup window, bound `fee_proposal_fri` to a configurable maximum (e.g., `max_l2_gas_price_fri` from `ContextDynamicConfig`, or a multiple of `min_gas_price`). Reject proposals that exceed it.

2. **Add an explicit validation arm for the `(None, Some(fee_proposal))` case** at V0_14_3+, rather than silently skipping bounds:
   ```rust
   (None, true) => {
       // fee_actual unavailable; enforce absolute cap instead of geometric bounds
       if fee_proposal.0 > MAX_STARTUP_FEE_PROPOSAL {
           return Err(InvalidProposalInit(..., "fee_proposal exceeds startup cap"));
       }
   }
   ```

3. **Consider seeding the window from the last known L2 gas price** at upgrade time so `fee_actual` is never `None` after V0_14_3 activation, eliminating the bypass window entirely.

---

### Proof of Concept

**Setup**: Network upgrades to V0_14_3 at height H. Heights H−10 to H−1 are pre-V0_14_3 (fee_proposal = None). Malicious validator M controls proposer slots at heights H, H+2, H+4, H+6, H+8, H+10 (6 of the first 11 V0_14_3 blocks).

**Step 1**: At heights H through H+9, `compute_fee_actual()` returns `None` (window contains pre-V0_14_3 `None` entries). Validator calls `is_proposal_init_valid()` — the `if let (Some(fee_actual), ...)` guard is skipped. M sets `fee_proposal_fri = u128::MAX` for each of its 6 slots. Validators accept all proposals. [1](#0-0) 

**Step 2**: At height H+10, the window covers heights H through H+9. All entries are `Some(...)`. `compute_fee_actual()` returns `Some(median([u128::MAX × 6, 8gwei × 4])) = u128::MAX`. [8](#0-7) 

**Step 3**: `calculate_next_l2_gas_price_for_fin()` sets `effective_min = u128::MAX`. `calculate_next_base_gas_price()` returns `8gwei + 8gwei/333 ≈ 8.024gwei` (first step), but `effective_min = u128::MAX` forces the result to `u128::MAX` after ~22,000 blocks of 0.3% increases. [4](#0-3) 

**Step 4**: All subsequent blocks carry `next_l2_gas_price` ratcheting toward `u128::MAX`. Users paying L2 gas fees are charged at the inflated rate. Recovery requires ~22,000 blocks of honest proposals each reducing `fee_proposal_fri` by 0.2%. [9](#0-8)

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
