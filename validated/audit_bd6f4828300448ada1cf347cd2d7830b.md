### Title
Setting `min_gas_price_percentage` to zero silently disables the L2 gas-price floor check, allowing zero-fee transactions through gateway admission — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidatorConfig::min_gas_price_percentage` is a `u8` that controls the minimum L2 gas price a transaction must offer relative to the previous block's L2 gas price. When set to `0`, the computed threshold becomes `0`, and the comparison `tx_l2_gas_price.0 < threshold` is always `false` — meaning **every transaction passes regardless of its offered gas price, including zero-fee transactions**. There is no validation preventing `min_gas_price_percentage` from being set to zero, and the codebase explicitly tests and documents this as an intentional escape hatch.

---

### Finding Description

In `validate_tx_l2_gas_price_within_threshold`, the threshold is computed as:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold { ... }
``` [1](#0-0) 

When `min_gas_price_percentage = 0`, `gas_price_threshold_multiplier = 0/100 = 0`, so `threshold = 0`. The condition `tx_l2_gas_price.0 < 0` is always false for a `u128`, so the check is completely bypassed. A transaction with `l2_gas.max_price_per_unit = 0` passes this gate.

The config field has type `u8` with no `#[validate(range(min = 1))]` or similar constraint:

```rust
pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
``` [2](#0-1) 

The test suite explicitly documents this behavior as a passing case:

```rust
#[case::gas_price_check_disabled_when_percentage_zero_pass(
    100_u128.try_into().unwrap(),
    0,
    0_u128.into(),
    Ok(()),
)]
``` [3](#0-2) 

The stateless validator has its own `min_gas_price` floor (default 8 Gwei), but that check is also gated behind `validate_resource_bounds: bool`, which is a separate shared pointer that can be set to `false` during bootstrap:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
``` [4](#0-3) 

However, the stateless `min_gas_price` check only applies when `validate_resource_bounds = true`. When `validate_resource_bounds = false` (bootstrap mode), **both** the stateless and stateful gas-price checks are disabled simultaneously via the shared config pointer:

```rust
set_pointing_param_paths(&[
    "gateway_config.static_config.stateful_tx_validator_config.validate_resource_bounds",
    "gateway_config.static_config.stateless_tx_validator_config.validate_resource_bounds",
    "mempool_config.static_config.validate_resource_bounds",
]),
``` [5](#0-4) 

The analog to the external bug is: `min_gas_price_percentage = 0` is the sequencer's equivalent of "setting the deposit limit to zero" — it silently disables the gas-price admission floor without any error or warning, allowing zero-fee transactions to pass the stateful gateway check even when `validate_resource_bounds = true`.

---

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions.**

A transaction with `l2_gas.max_price_per_unit = 0` submitted when `min_gas_price_percentage = 0` (and `validate_resource_bounds = true`) will:

1. Pass `StatelessTransactionValidator::validate_resource_bounds` — because the stateless check only requires that at least one resource bound is nonzero (e.g., `l1_gas` or `l1_data_gas` can be nonzero), and the `min_gas_price` check only applies to `l2_gas.max_price_per_unit`.
2. Pass `StatefulTransactionValidator::validate_resource_bounds` — because `threshold = 0` and `0 < 0` is false.
3. Enter the mempool and be sequenced.

Once sequenced, the blockifier's `check_fee_bounds` will compare the transaction's `max_price_per_unit` against the actual block gas price. If the block gas price is nonzero (which it always is in production), the transaction will fail `InsufficientResourceBounds` at execution time — but it has already consumed mempool capacity and gateway resources, and the sequencer has wasted block-building effort on it.

More critically: if `validate_resource_bounds = false` (bootstrap mode), the stateless `min_gas_price` floor is also disabled, so zero-fee transactions pass all gateway checks and reach the mempool with no rejection at any layer.

---

### Likelihood Explanation

**Likelihood: Low-to-Medium.**

- An operator misconfiguring `min_gas_price_percentage = 0` by mistake (intending to "disable" the percentage check or set it to a very low value) is plausible.
- The config is a plain `u8` with no validation constraint, and the documentation says "E.g., 80 to require 80% of threshold" — an operator might set `0` thinking it means "no minimum" or "accept all."
- The bootstrap flag `validate_resource_bounds = false` is a documented operational mode that disables all gas-price checks simultaneously, which is a broader but intentional exposure window.

---

### Recommendation

1. Add a `#[validate(range(min = 1))]` attribute or a startup assertion on `min_gas_price_percentage` to reject zero at configuration load time.
2. Alternatively, treat `min_gas_price_percentage = 0` as "use the stateless `min_gas_price` floor" rather than "no floor."
3. Document clearly that `min_gas_price_percentage = 0` disables the dynamic gas-price threshold check entirely, so operators understand the security implication.

---

### Proof of Concept

**Configuration:**
```json
{
  "gateway_config.static_config.stateful_tx_validator_config.min_gas_price_percentage": 0,
  "gateway_config.static_config.stateful_tx_validator_config.validate_resource_bounds": true
}
```

**Transaction:**
```rust
ValidResourceBounds::AllResources(AllResourceBounds {
    l2_gas: ResourceBounds {
        max_price_per_unit: GasPrice(0),  // zero fee
        max_amount: GasAmount(1_000_000),
    },
    l1_gas: ResourceBounds::default(),
    l1_data_gas: ResourceBounds::default(),
})
```

**Trace:**
1. `StatelessTransactionValidator::validate_resource_bounds`: `max_possible_fee(Tip::ZERO)` may be nonzero if `l2_gas.max_amount > 0` even with price 0 — actually `max_possible_fee = max_amount * max_price_per_unit = 0`. This triggers `ZeroResourceBounds` **only if all three resources are zero**. If `l1_gas` has nonzero amount/price, the stateless check passes.
2. `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold`: `threshold = (0/100) * prev_price = 0`; `0 < 0` is false → **passes**.
3. Transaction enters mempool with zero L2 gas price.

The existing test `gas_price_check_disabled_when_percentage_zero_pass` confirms this behavior is reachable and produces `Ok(())`: [3](#0-2)
