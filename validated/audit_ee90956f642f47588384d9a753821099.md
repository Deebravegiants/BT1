### Title
Missing `DirectDeposits` in Value Conservation Check Allows Arbitrary ADA Creation - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

The Dijkstra era introduces a `DirectDeposits` field in the transaction body that directly credits ADA into registered stake accounts. This field is applied to account balances via `applyDirectDeposits` but is **absent from both sides of the value conservation equation** (`consumed == produced`). An unprivileged transaction sender can therefore specify arbitrary `DirectDeposits` amounts, pass the preservation-of-value check without providing the corresponding ADA from UTxO inputs, and withdraw the inflated account balances — creating ADA out of thin air.

---

### Finding Description

**New field: `DirectDeposits`**

The Dijkstra era adds `dtbrDirectDeposits` / `dstbrDirectDeposits` (a `Map AccountAddress Coin`) to both top-level and sub-transaction bodies. [1](#0-0) 

**State mutation: `applyDirectDeposits`**

In `dijkstraSubEntitiesTransition`, after certificate processing, the ledger unconditionally applies the `DirectDeposits` map to account balances:

```haskell
let directDeposits = tx ^. bodyTxL . directDepositsTxBodyL
    accountsAfterSubCerts = certStateAfterSubCerts ^. certDStateL . accountsL
failOnJust (directDepositsMissingAccounts directDeposits accountsAfterSubCerts) $
  injectFailure . SubDirectDepositsToMissingAccounts
pure $ certStateAfterSubCerts & certDStateL . accountsL %~ applyDirectDeposits directDeposits
``` [2](#0-1) 

The only guard is `directDepositsMissingAccounts` — it rejects deposits to *unregistered* accounts, but performs **no check that the deposited ADA is funded by the transaction's UTxO inputs**. [3](#0-2) 

**Value conservation check omits `DirectDeposits`**

The Dijkstra UTXO transition rule enforces value conservation at line 381:

```haskell
{- consumed pp utxo₀ txb = produced pp certState txb -}
runTest $ Shelley.validateValueNotConservedUTxO pp originalUtxo certState txBody
``` [4](#0-3) 

`validateValueNotConservedUTxO` dispatches to the era's `consumed` and `produced` methods. For Dijkstra:

- **`consumed = conwayConsumed`** → calls `getConsumedValue = getConsumedDijkstraValue`
- **`getProducedValue = getProducedDijkstraValue`** [5](#0-4) 

`getConsumedDijkstraValue` aggregates UTxO inputs, withdrawals, and refunds from the top-level body and all sub-transactions — **no `DirectDeposits`**: [6](#0-5) 

`dijkstraProducedValue` (top-level) and `dijkstraSubTxProducedValue` (sub-tx) aggregate UTxO outputs, fees, certificate deposits, and treasury donations — **no `DirectDeposits`**: [7](#0-6) [8](#0-7) 

Because `DirectDeposits` appears on neither side of the equation, the check `consumed == produced` is satisfied regardless of the `DirectDeposits` amounts. The ledger then credits those amounts to account balances with no corresponding debit from UTxO.

---

### Impact Explanation

**Critical — Direct creation of ADA through an invalid ledger state transition.**

An attacker can inflate stake-account balances by an arbitrary amount without consuming any ADA from UTxO inputs. Those balances are subsequently withdrawable via the normal `Withdrawals` mechanism (which is validated against the account balance). The net effect is unbounded ADA minting, violating the fixed-supply invariant of the Cardano protocol.

---

### Likelihood Explanation

The Dijkstra era is not yet deployed on mainnet, but it is present in the production repository and is the target of active development. Any unprivileged transaction sender can exploit this once the era activates: the attack requires only a registered stake account and a single crafted transaction. No special keys, governance majority, or privileged access is needed.

---

### Recommendation

Include the sum of all `DirectDeposits` coin amounts in the **produced** side of the value conservation equation, mirroring how `treasuryDonationTxBodyL` is already handled in `conwayProducedValue`:

```haskell
-- In dijkstraProducedValue / dijkstraSubTxProducedValue, add:
<> inject (fold . unDirectDeposits $ txBody ^. directDepositsTxBodyL)
```

This ensures that every lovelace credited to an account via `DirectDeposits` must be funded by UTxO inputs, preserving the preservation-of-value invariant.

---

### Proof of Concept

1. **Setup**: Attacker registers stake credential `cred`, yielding a registered `AccountAddress acct`.
2. **Craft transaction**: Build a Dijkstra-era transaction (or sub-transaction) with:
   - UTxO inputs totalling `X` lovelace
   - UTxO outputs totalling `X − fee` lovelace
   - `DirectDeposits = { acct → Y }` for an arbitrary large `Y`
3. **Value conservation check passes**:
   - `consumed = X` (UTxO inputs only; `DirectDeposits` absent)
   - `produced = (X − fee) + fee = X` (outputs + fee; `DirectDeposits` absent)
   - `consumed == produced` ✓
4. **`applyDirectDeposits` executes**: `acct`'s balance increases by `Y` lovelace with no corresponding UTxO debit.
5. **Withdraw**: In a subsequent transaction, include `Withdrawals = { acct → Y }`. The withdrawal validation confirms the balance equals `Y` and drains it to a UTxO output, netting the attacker `Y` lovelace created from nothing.

getConsumedDijkstraValue ::
  forall era l.
  ( DijkstraEraTxBody era
  , EraUTxO era
  , Value era ~ MaryValue
  , STxLevel l era ~ STxBothLevels l era
  ) =>
  PParams era ->
  (Credential Staking -> Maybe Coin) ->
  (Credential DRepRole -> Maybe Coin) ->
  UTxO era ->
  TxBody l era ->
  Value era
getConsumedDijkstraValue pp lookupStakingDeposit lookupDRepDeposit utxo txBody =
  withBothTxLevels
    txBody
    ( \topTxBody ->
        txBodyConsumedValue topTxBody <> subTransactionsConsumedValue topTxBody
    )
    txBodyConsumedValue
  where
    txBodyConsumedValue :: forall m. TxBody m era -> Value era
    txBodyConsumedValue = getConsumedMaryValue pp lookupStakingDeposit lookupDRepDeposit utxo
    subTransactionsConsumedValue topTxBody =
      foldMap'
        (getConsumedValue pp lookupStakingDeposit lookupDRepDeposit utxo . view bodyTxL)
        (topTxBody ^. subTransactionsTxBodyL)
