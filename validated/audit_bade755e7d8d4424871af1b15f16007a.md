Audit Report

## Title
Missing Chainlink Oracle Validity and Staleness Checks Allow Invalid Prices to Corrupt rsETH Exchange Rate - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice` calls `latestRoundData()` but discards all validity indicators, casting the raw `int256 answer` directly to `uint256` with no staleness, incomplete-round, or non-positive-price checks. This corrupted price flows through `LRTOracle._getTotalEthInProtocol()` into `_updateRsETHPrice()`, setting the on-chain `rsETHPrice` storage variable that governs every deposit and withdrawal. The public `updateRSETHPrice()` entry point allows any unprivileged caller to trigger this at the moment a feed is stale.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice` (lines 52–54) reads:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values except `answer` are silently discarded. Three validity conditions are never checked:

1. **Staleness**: `answeredInRound < roundId` — never checked.
2. **Incomplete round**: `updatedAt == 0` — never checked.
3. **Non-positive price**: `price <= 0` — never checked; a negative `int256` wraps to near-`type(uint256).max` on cast; zero produces zero.

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate` (lines 30–32) performs all three checks:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The unguarded oracle is the one wired into `LRTOracle.getAssetPrice` (line 157), which is called inside `_getTotalEthInProtocol` (line 339) to compute `totalETHInProtocol`. That value feeds directly into `_updateRsETHPrice` (line 250):

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
```

The `pricePercentageLimit` guard (lines 256–266, 273–281) only activates when `pricePercentageLimit > 0`; it defaults to zero and is not enforced by default, providing no protection in the default configuration.

The full call chain reachable by any unprivileged caller:
`updateRSETHPrice()` (public, `whenNotPaused`) → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(asset)` → `ChainlinkPriceOracle.getAssetPrice(asset)` → stale/invalid price accepted → corrupted `rsETHPrice` written to storage.

## Impact Explanation
**Theft of unclaimed yield (High).** When a feed goes stale at a price *lower* than the true market price, `rsETHPrice` is deflated. New depositors receive more rsETH tokens than the protocol's actual backing warrants, directly diluting the yield accrued by existing rsETH holders. This is a concrete, quantifiable transfer of value from existing holders to new depositors — matching the "Theft of unclaimed yield" impact class. The inverse (stale price higher than actual) causes depositors to receive fewer rsETH than owed, constituting a loss to depositors. Both directions are reachable through the same unguarded code path.

## Likelihood Explanation
Chainlink feeds go stale during network congestion, sequencer outages (L2), or extreme volatility. `updateRSETHPrice()` is public and requires no role. Any unprivileged external actor can call it at the exact moment a feed is stale, with no preconditions beyond the contract not being paused. The attack is repeatable every time a feed lags. No admin collusion, governance capture, or oracle operator compromise is required.

## Recommendation
Apply the same three-check pattern already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    require(answeredInRound >= roundId, "Stale price");
    require(updatedAt != 0, "Incomplete round");
    require(price > 0, "Invalid price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, add a heartbeat check: `require(block.timestamp - updatedAt <= MAX_STALENESS, "Price too old")` tuned per feed. Also enforce a non-zero default for `pricePercentageLimit` as a secondary defense.

## Proof of Concept
1. Deploy a mock Chainlink aggregator that returns `answeredInRound < roundId` (stale) with a price 5% below the true market price for a supported LST asset.
2. Wire it as the `assetPriceFeed` for that asset in `ChainlinkPriceOracle`.
3. Call `LRTOracle.updateRSETHPrice()` from an unprivileged EOA.
4. Observe `rsETHPrice` is set ~5% below the correct value (no revert occurs).
5. Call `LRTDepositPool.depositAsset` with 1 ETH worth of the LST; observe the caller receives ~5% more rsETH than the protocol's backing supports.
6. Existing rsETH holders now hold tokens backed by proportionally less ETH — their unclaimed yield has been transferred to the new depositor.

Foundry fork test: fork mainnet, manipulate the Chainlink aggregator's `latestRoundData` return via `vm.mockCall` to return a stale round with a deflated price, call `updateRSETHPrice()`, assert `rsETHPrice` is set to the stale value, then assert a subsequent deposit mints excess rsETH relative to the correct price.
