### Title
Unprotected `initialize` Allows Front-Run Takeover of Bridge Signer Authority ŌĆö (`File: evm/src/omni-bridge/contracts/OmniBridge.sol`)

### Summary
`OmniBridge.initialize` is `public` with no caller restriction and no zero-address guard on `nearBridgeDerivedAddress_`. If the proxy is deployed without an atomic init call, an unprivileged attacker can front-run initialization, install their own ECDSA key as `nearBridgeDerivedAddress`, and become `DEFAULT_ADMIN_ROLE`, enabling unlimited unauthorized minting via `finTransfer`.

### Finding Description
`OmniBridge.initialize` is declared `public initializer` with no `msg.sender` check and no zero-address validation on its critical parameters:

```solidity
function initialize(
    address tokenImplementationAddress_,
    address nearBridgeDerivedAddress_,   // ŌåÉ no require != address(0)
    uint8 omniBridgeChainId_
) public initializer {                   // ŌåÉ no caller restriction
    tokenImplementationAddress = tokenImplementationAddress_;
    nearBridgeDerivedAddress = nearBridgeDerivedAddress_;
    ...
    _grantRole(DEFAULT_ADMIN_ROLE, _msgSender());   // ŌåÉ caller becomes admin
    _grantRole(PAUSABLE_ADMIN_ROLE, _msgSender());
}
``` [1](#0-0) 

The `_disableInitializers()` in the constructor protects only the bare implementation contract. [2](#0-1) 

If the proxy is deployed in a two-step pattern (deploy proxy with empty calldata, then call `initialize` in a separate transaction), the window between those two transactions is exploitable. An attacker who calls `initialize` first:

1. Becomes `DEFAULT_ADMIN_ROLE` and `PAUSABLE_ADMIN_ROLE`.
2. Stores an attacker-controlled address as `nearBridgeDerivedAddress`.
3. Blocks the legitimate deployer from ever calling `initialize` again (the `initializer` modifier enforces one-time execution).
4. Can call `setNearBridgeDerivedAddress` to rotate the key at will, since they hold `DEFAULT_ADMIN_ROLE`. [3](#0-2) 

The same pattern applies to `OmniBridgeWormhole.initializeWormhole`, which is also `external initializer` with no caller guard. [4](#0-3) 

### Impact Explanation
`nearBridgeDerivedAddress` is the sole trust anchor for all inbound settlement. Both `finTransfer` and `deployToken` gate execution on `ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress`. [5](#0-4) 

An attacker who controls `nearBridgeDerivedAddress` can:
- Forge arbitrary `finTransfer` payloads signed with their own key, minting unbacked bridge tokens to any recipient (wrapped ERC-20 via `IBridgeToken.mint`, native ETH via `call{value}`, or ERC-20 vault release via `safeTransfer`). [6](#0-5) 
- Forge `deployToken` payloads to register malicious token contracts. [7](#0-6) 

This is **Critical** ŌĆö unauthorized creation and release of bridge assets through verification failure.

### Likelihood Explanation
Likelihood is **Medium**. The attack requires the proxy to be deployed without an atomic init call. This is not the default ERC1967Proxy constructor pattern (which bundles deployment and init in one transaction), but it is a common deployment mistake and is not prevented anywhere in the contract. A mempool-watching bot can detect an uninitialized proxy and front-run the `initialize` call in the same block. No privileged access, leaked keys, or external dependency compromise is required.

### Recommendation
1. **Add a deployer check** inside `initialize` (e.g., `require(msg.sender == _expectedDeployer, "unauthorized")`), or use OpenZeppelin's `Ownable` initializer pattern that restricts the first call.
2. **Add zero-address guards** for both `nearBridgeDerivedAddress_` and `tokenImplementationAddress_`:
   ```solidity
   require(_nearBridgeDerivedAddress != address(0), "zero signer");
   require(_tokenImplementationAddress != address(0), "zero impl");
   ```
3. **Always deploy the proxy with atomic initialization** ŌĆö pass the encoded `initialize` calldata directly to the `ERC1967Proxy` constructor so deployment and initialization are a single transaction.

### Proof of Concept
```
Block N:   Deployer broadcasts: new ERC1967Proxy(impl, "")
           (proxy deployed, initialize not yet called)

Block N:   Attacker sees uninitialized proxy in mempool,
           broadcasts with higher gas:
           OmniBridge(proxy).initialize(
               impl,
               attacker_controlled_address,  // nearBridgeDerivedAddress
               chainId
           )
           ŌåÆ attacker is now DEFAULT_ADMIN_ROLE
           ŌåÆ nearBridgeDerivedAddress = attacker key

Block N+1: Attacker signs a TransferMessagePayload with their key:
           {destinationNonce: 1, tokenAddress: wETH, amount: 1000e18, recipient: attacker}
           Calls finTransfer(attackerSignature, payload)
           ŌåÆ ECDSA.recover matches nearBridgeDerivedAddress Ō£ō
           ŌåÆ isBridgeToken[wETH] check passes if wETH is a bridge token
           ŌåÆ IBridgeToken(wETH).mint(attacker, 1000e18) executes
           ŌåÆ Unbacked bridge tokens minted to attacker
``` [8](#0-7)
