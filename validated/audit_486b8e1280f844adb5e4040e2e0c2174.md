Audit Report

## Title
Missing Stale Price Check in `ChainlinkPriceOracle.getAssetPrice()` Enables Forced Protocol Pause - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `updatedAt`, `answeredInRound`, and `roundId`, and performs no zero-price guard. The unchecked price feeds directly into `LRTOracle._updateRsETHPrice()`, which is callable by any unprivileged user via the public `updateRSETHPrice()`. A stale low price can cause the downside-protection branch to auto-pause the entire protocol, temporarily freezing all user deposits and withdrawals.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` at [1](#0-0)  discards four of the five return values of `latestRoundData()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No check is made on `updatedAt` (heartbeat staleness), `answeredInRound >= roundId` (incomplete round), or `price > 0` (zero answer). By contrast, the sibling contract `ChainlinkOracleForRSETHPoolCollateral.getRate()` performs all three checks correctly: [2](#0-1) 

The stale price propagates through the following public call chain:

1. Any EOA calls `LRTOracle.updateRSETHPrice()` — public, only `whenNotPaused` guard: [3](#0-2) 
2. `_updateRsETHPrice()` calls `_getTotalEthInProtocol()`: [4](#0-3) 
3. `_getTotalEthInProtocol()` calls `getAssetPrice(asset)` for every supported asset, multiplying by total deposits: [5](#0-4) 
4. A stale/low price deflates `totalETHInProtocol`, causing `newRsETHPrice` to fall below `highestRsethPrice` by more than `pricePercentageLimit`.
5. The downside-protection branch executes: [6](#0-5) 

```solidity
if (!lrtDepositPool.paused()) lrtDepositPool.pause();
if (!withdrawalManager.paused()) withdrawalManager.pause();
_pause();
return;
```

This pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. The `LRTOracle` can only be unpaused by an admin via `unpause()`, which requires `onlyLRTAdmin`. [7](#0-6) 

A secondary impact exists if `price == 0`: `uint256(0) * 1e18 / 10**decimals = 0`, so the affected asset contributes 0 ETH to `totalETHInProtocol`. This artificially deflates `newRsETHPrice`, allowing new depositors to receive more rsETH than entitled, diluting existing holders' unclaimed yield.

## Impact Explanation
**Primary — Medium: Temporary freezing of funds.** Any unprivileged caller can trigger a full protocol pause (deposits and withdrawals halted) by calling `updateRSETHPrice()` during a Chainlink feed staleness window, if `pricePercentageLimit` is set. Funds remain frozen until an admin manually calls `unpause()` on each contract.

**Secondary — High: Theft of unclaimed yield.** If `price == 0` (Chainlink returns zero when no answer is reached), `newRsETHPrice` is artificially deflated. New depositors receive excess rsETH at the expense of existing holders, constituting theft of unclaimed yield.

## Likelihood Explanation
`updateRSETHPrice()` is public with no access control beyond `whenNotPaused`. Chainlink LST/ETH feeds (stETH/ETH, cbETH/ETH) have heartbeats of 1–24 hours; missed updates during network congestion are documented behavior. No special attacker capability is required — the attacker waits for a stale round and calls the function. The `pricePercentageLimit` is a configurable value set by admin; if non-zero (the expected production configuration given the downside-protection logic exists), the pause path is reachable.

## Recommendation
Apply the same validation pattern already used in `ChainlinkOracleForRSETHPoolCollateral`: [8](#0-7) 

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    if (block.timestamp - updatedAt > MAX_ORACLE_DELAY) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

A per-feed configurable `maxDelay` is preferable since different Chainlink feeds have different heartbeat intervals.

## Proof of Concept
**Temporary freeze path (Foundry fork test):**
1. Fork mainnet with a supported LST asset (e.g., stETH) whose Chainlink feed has a heartbeat of 24 hours.
2. `vm.warp(block.timestamp + 25 hours)` to simulate a stale feed (no `updatedAt` advancement).
3. Confirm `pricePercentageLimit > 0` is set on `LRTOracle`.
4. Call `LRTOracle.updateRSETHPrice()` from an unprivileged EOA (`vm.prank(attacker)`).
5. `ChainlinkPriceOracle.getAssetPrice(stETH)` returns the last stale (artificially low) price without reverting.
6. `newRsETHPrice` is computed significantly below `highestRsethPrice`.
7. Assert `lrtDepositPool.paused() == true`, `withdrawalManager.paused() == true`, `lrtOracle.paused == true`.
8. Confirm no unprivileged caller can unpause — only `onlyLRTAdmin` can call `unpause()`.
