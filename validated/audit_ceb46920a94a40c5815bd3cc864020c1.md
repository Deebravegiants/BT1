Audit Report

## Title
Undistributed Reward Tokens Permanently Stranded When All Stakers Exit Before Reward Period Ends - (File: contracts/KERNEL/KernelDepositPool.sol)

## Summary

`KernelDepositPool.notifyRewardAmount()` uses a transfer-in delta (`balanceAfter - balanceBefore`) to compute `receivedAmount`, and when the previous reward period has expired (`block.timestamp >= finishAt`), sets `rewardRate = receivedAmount / duration`, ignoring any pre-existing token balance in the contract. If all stakers exit during an active reward period, the `NoStakedTokens` guard prevents the admin from rolling over the stranded balance before `finishAt`, and once the period expires those tokens are silently excluded from all future reward calculations with no recovery path.

## Finding Description

The `notifyRewardAmount` function at lines 566–592 computes the reward amount exclusively from the transfer delta:

```solidity
uint256 balanceBefore = rewardsToken.balanceOf(address(this));
rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
uint256 balanceAfter = rewardsToken.balanceOf(address(this));
uint256 receivedAmount = balanceAfter - balanceBefore;
```

When the prior period has ended, the new rate is set as:

```solidity
if (block.timestamp >= finishAt) {
    rewardRate = receivedAmount / duration;
}
```

Any tokens already sitting in the contract (stranded from the prior period) are part of `balanceBefore` and are therefore invisible to `receivedAmount`. They are never included in any future `rewardRate`.

The `NoStakedTokens` guard at line 570 prevents the admin from calling `notifyRewardAmount` while the pool is empty, closing the only window in which the admin could roll over the stranded balance into a new period before `finishAt`. The contract's own NatSpec at lines 17–22 explicitly acknowledges this behavior and states the mitigation is purely operational ("ensuring there are always some tokens staked"), which is not enforced at the code level and can fail under normal market conditions.

There is no sweep, rescue, or recovery function anywhere in the contract.

**Exploit path:**
1. Admin calls `notifyRewardAmount(1000e18)` with `duration = 7 days`. `finishAt = T + 7d`.
2. At `T + 3d`, all stakers call `initiateWithdrawal` / `claimWithdrawal`. `totalKernelStaked = 0`. ~571e18 tokens remain unallocated.
3. Admin attempts `notifyRewardAmount(...)` to rescue — reverts with `NoStakedTokens()`.
4. At `T + 7d`, a new staker deposits. Admin calls `notifyRewardAmount(1000e18)`.
5. `block.timestamp >= finishAt` → `rewardRate = 1000e18 / duration`. The 571e18 stranded tokens are in `balanceBefore`, excluded from `receivedAmount`, and permanently locked.

## Impact Explanation

Reward tokens legitimately deposited by the admin for distribution to stakers become permanently locked in the contract with no path to distribution or recovery. This is a concrete instance of **Medium — Permanent freezing of unclaimed yield**. Stakers who would have earned the tail-end rewards of the period are permanently denied that yield.

## Likelihood Explanation

The trigger is entirely normal user behavior: stakers calling `initiateWithdrawal` and `claimWithdrawal`. No attacker coordination, privilege, or special condition is required beyond a pool that empties during an active reward window — a realistic scenario during market volatility or a competing yield opportunity. The `NoStakedTokens` guard, intended to prevent a different class of issue, actively prevents the admin from mitigating the situation once it occurs.

## Recommendation

When starting a new reward period after the previous one has expired, include any excess contract balance in the new rate calculation:

```solidity
if (block.timestamp >= finishAt) {
    uint256 leftover = rewardsToken.balanceOf(address(this)) - receivedAmount;
    rewardRate = (receivedAmount + leftover) / duration;
} else {
    uint256 remaining = (finishAt - block.timestamp) * rewardRate;
    rewardRate = (receivedAmount + remaining) / duration;
}
```

Alternatively, add an admin-callable function to re-queue stranded tokens into the next reward period, or remove the `NoStakedTokens` guard from `notifyRewardAmount` (accepting the known zero-staker edge case) so the admin can roll over the balance before `finishAt`.

## Proof of Concept

Foundry test outline:

```solidity
function test_strandedRewards() public {
    // 1. Admin notifies 1000e18 rewards, duration = 7 days
    vm.prank(admin);
    pool.notifyRewardAmount(1000e18);

    // 2. At T+3d, all stakers withdraw
    vm.warp(block.timestamp + 3 days);
    vm.prank(staker);
    pool.initiateWithdrawal(stakerBalance);
    // wait withdrawalDelay, then claimWithdrawal
    vm.warp(block.timestamp + withdrawalDelay);
    vm.prank(staker);
    pool.claimWithdrawal(1);
    assertEq(pool.totalKernelStaked(), 0);

    // 3. Admin cannot call notifyRewardAmount — NoStakedTokens
    vm.prank(admin);
    vm.expectRevert(KernelDepositPool.NoStakedTokens.selector);
    pool.notifyRewardAmount(1000e18);

    // 4. After finishAt, new staker deposits
    vm.warp(pool.finishAt() + 1);
    vm.prank(newStaker);
    pool.stake(1e18);

    // 5. Admin notifies 1000e18 new rewards
    uint256 contractBalanceBefore = rewardsToken.balanceOf(address(pool));
    vm.prank(admin);
    pool.notifyRewardAmount(1000e18);

    // 6. Stranded tokens (~571e18) are still in contract but excluded from rewardRate
    uint256 contractBalanceAfter = rewardsToken.balanceOf(address(pool));
    // contractBalanceAfter > 1000e18 (includes stranded tokens)
    // but rewardRate == 1000e18 / duration (stranded tokens ignored)
    assertGt(contractBalanceAfter, 1000e18);
    assertEq(pool.rewardRate(), 1000e18 / pool.duration());
    // Stranded tokens are permanently locked — no staker can ever earn them
}

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
