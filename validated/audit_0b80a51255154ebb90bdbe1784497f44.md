## Code Trace

**Entry point**: `process_tx_internal` in `chain/client/src/rpc_handler.rs`

**Step 1 — Congestion info lookup (SPICE path)**: [1](#0-0) 

When SPICE is enabled, `spice_shard_congestion_info` is called for the receiver shard. It returns `Option<ExtendedCongestionInfo>`.

**Step 2 — `spice_shard_congestion_info` can return `None`**: [2](#0-1) 

The function tries two sources:
1. `get_chunk_extra` — only succeeds if the node **tracks** the shard
2. `get_execution_result_from_store` — reads from `DBCol::execution_results`, written **asynchronously** by `SpiceCoreWriterActor`

If both fail, it returns `None`. The docstring explicitly states: *"Returns `None` when the shard is neither tracked nor yet certified."*

**Step 3 — `validate_tx` skips the congestion check when `None`**: [3](#0-2) 

The `if let Some(congestion_info) = receiver_congestion_info` guard means the entire `shard_accepts_transactions()` check is silently skipped when `receiver_congestion_info` is `None`. The transaction proceeds to `validate_transaction` with no congestion enforcement.

**Step 4 — Shard tracking check is on the SIGNER's shard, not the receiver's**: [4](#0-3) 

The node checks whether it tracks the **signer's** shard. If it does, it performs full state validation and then forwards the transaction. The receiver shard's congestion is never re-checked.

**Step 5 — Chunk producer has the same `None`-bypass**: [5](#0-4) 

`congestion_control_accepts_transaction` also returns `Ok(true)` when congestion info is absent for the receiving shard.

---

## Root Cause

`DBCol::execution_results` is written by `SpiceCoreWriterActor` **asynchronously** when it collects endorsements. A non-tracking RPC node may never receive endorsements for shards it doesn't track, meaning `DBCol::execution_results` is never populated for those shards on that node. The `SpiceCoreReader::last_certified_state_root` has a slow-path fallback that reads execution results from block bodies for non-tracked shards, but `spice_shard_congestion_info` does **not** implement this fallback — it only reads from `DBCol::execution_results`. [6](#0-5) 

The slow-path recovery from block ancestry exists in `SpiceCoreReader` but is absent from `spice_shard_congestion_info`, creating a persistent (not merely transient) gap for non-tracking nodes.

### Title
SPICE: `spice_shard_congestion_info` returns `None` for non-tracked shards, bypassing RPC-layer congestion rejection in `validate_tx` — (`chain/chain/src/spice/chunk_application.rs`, `chain/client/src/rpc_handler.rs`)

### Summary
Under the SPICE protocol feature, an unprivileged RPC client can submit a transaction whose receiver account maps to a shard the local node does not track. `spice_shard_congestion_info` returns `None` for that shard (because neither `get_chunk_extra` nor `get_execution_result_from_store` succeeds), causing `validate_tx` to skip `CongestionControl::shard_accepts_transactions` entirely. The transaction passes basic validation, is forwarded to chunk producers, and may be included in a chunk even when the receiver shard is fully congested.

### Finding Description
In `process_tx_internal`, the SPICE path calls `spice_shard_congestion_info` to obtain the receiver shard's congestion level. This function returns `None` when the node does not track the shard and `DBCol::execution_results` has no entry for it. `validate_tx` wraps the congestion check in `if let Some(congestion_info) = receiver_congestion_info`, so a `None` result silently skips the check. The subsequent shard-tracking guard checks the **signer's** shard, not the receiver's, so a node tracking the signer's shard but not the receiver's shard will proceed to full state validation and forward the transaction. The chunk producer's `congestion_control_accepts_transaction` has the same `None → Ok(true)` bypass, so the transaction may also pass the chunk-inclusion filter.

### Impact Explanation
Transactions to a fully congested receiver shard are admitted at the RPC layer and forwarded to chunk producers, violating the invariant that congested-shard transactions are rejected early. The resulting receipts enter the congested shard's delayed-receipt queue, amplifying congestion and wasting network and chunk-space resources. No consensus-layer state corruption or nonce/balance inconsistency results, because the transaction itself is valid and applied normally; only the receipt is delayed.

### Likelihood Explanation
The condition is persistent (not merely a transient race) for any RPC node that does not track the receiver shard and does not receive endorsements for it. In a multi-shard SPICE deployment, RPC nodes commonly track only a subset of shards. An attacker who knows a shard is congested can consistently bypass the RPC check by submitting to a non-tracking node.

### Recommendation
In `spice_shard_congestion_info`, add the same block-ancestry slow-path fallback used by `SpiceCoreReader::last_certified_state_root`: when `get_execution_result_from_store` returns `None`, walk the canonical ancestry to recover the execution result from block core statements. Alternatively, treat a `None` result as "unknown congestion" and apply a conservative default (e.g., `ExtendedCongestionInfo::default()` with maximum congestion) rather than skipping the check.

### Proof of Concept
In a SPICE test-loop test with two shards, congest shard B by dropping its chunks until `missed_chunks_count >= max_congestion_missed_chunks`. Start a second node that tracks only shard A. Submit a transaction from a shard-A account to a shard-B account via the non-tracking node's RPC. Observe that `process_tx_internal` returns `ProcessTxResponse::ValidTx` or `RequestRouted` rather than `InvalidTx(ShardCongested/ShardStuck)`, confirming the bypass. The existing non-SPICE test `slow_test_one_shard_congested` (marked `#[cfg_attr(feature = "protocol_feature_spice", ignore)]`) demonstrates the expected rejection behavior and can be adapted. [7](#0-6)

pub fn spice_shard_congestion_info(
    chain_store: &ChainStoreAdapter,
    shard_layout: &ShardLayout,
    block_header: &BlockHeader,
    shard_id: ShardId,
) -> Option<ExtendedCongestionInfo> {
    let shard_uid = ShardUId::from_shard_id_and_layout(shard_id, shard_layout);
    let chunk_store = chain_store.chunk_store();
    let congestion_info =
        if let Ok(chunk_extra) = chunk_store.get_chunk_extra(block_header.hash(), &shard_uid) {
            chunk_extra.congestion_info()
        } else {
            get_execution_result_from_store(chain_store, block_header.hash(), shard_id)?
                .chunk_extra
                .congestion_info()
        };
    Some(ExtendedCongestionInfo::new(congestion_info, 0))
}

#[cfg_attr(feature = "protocol_feature_spice", ignore)]
fn slow_test_one_shard_congested() {
    init_test_logger();

    let mut accounts = make_accounts(NUM_PRODUCERS + NUM_VALIDATORS);
    let shard1_acc1: AccountId = "0000".parse().unwrap();
    let shard1_acc2: AccountId = "1111".parse().unwrap();
    let shard2_acc1: AccountId = "xxxx".parse().unwrap();
    accounts.push(shard1_acc1.clone());
    accounts.push(shard1_acc2.clone());
    accounts.push(shard2_acc1.clone());

    let runtime_config = RuntimeConfig::test();
    let max_missed_chunks = runtime_config.congestion_control_config.max_congestion_missed_chunks;
    let epoch_length = max_missed_chunks * 100;
    let (env, rpc_id) = setup(&accounts, epoch_length, &["near"]);
    assert_eq!(rpc_id, shard1_acc1);

    // Assert correct shard layout
    let shard_layout = &env.shared_state.genesis.config.shard_layout;
    let shard1 = shard_layout.account_id_to_shard_id(&shard1_acc1);
    let shard2 = shard_layout.account_id_to_shard_id(&shard2_acc1);
    assert_eq!(shard1, shard_layout.account_id_to_shard_id(&shard1_acc2));
    assert_ne!(shard1, shard2);

    // Configure test env to drop chunks for shard 2
    let dropped_chunks = [
        (shard2, (0..epoch_length).map(|_| false).collect_vec()), // All chunks in the epoch
    ]
    .into_iter()
    .collect();
    let mut env = env.drop(DropCondition::ChunksProducedByHeight(dropped_chunks));

    // Run for `max_missed_chunks * 2` blocks to make sure shard 2 is congested
    run_for_number_of_blocks(&mut env, &rpc_id, max_missed_chunks as usize * 2);

    // Check if shard 2 is congested
    let client_handle = env.node_datas[0].client_sender.actor_handle();
    let client_actor = env.test_loop.data.get(&client_handle);
    let head = client_actor.client.chain.get_head_block().unwrap();
    let missed_chunks = head.block_congestion_info().get(&shard2).unwrap().missed_chunks_count;
    assert!(missed_chunks >= max_missed_chunks);

    // Send transfer from shard 1 to shard 1 – should succeed
    let block_time = client_actor.client.config.max_block_production_delay.get();
    let tx = env.node_for_account(&rpc_id).tx_send_money(
        &shard1_acc1,
        &shard1_acc2,
        Balance::from_near(1),
    );
    let tx_outcome = execute_tx(
        &mut env.test_loop,
        &rpc_id,
        TransactionRunner::new(tx, false),
        &env.node_datas,
        block_time * 3,
    )
    .unwrap();
    assert_matches!(tx_outcome.status, FinalExecutionStatus::SuccessValue(_));

    // Send transfer from shard 1 to shard 2 – should fail, because shard 2 is congested
    let tx = env.node_for_account(&rpc_id).tx_send_money(
        &shard1_acc1,
        &shard2_acc1,
        Balance::from_near(1),
    );
    let tx_error = execute_tx(
        &mut env.test_loop,
        &rpc_id,
        TransactionRunner::new(tx, false),
        &env.node_datas,
        block_time * 3,
    )
    .unwrap_err();
    assert_matches!(tx_error, InvalidTxError::ShardStuck { shard_id, .. } if shard_id == Into::<u32>::into(shard2));
}
