### Title
Missing Q-to-Qi Binding Check in `attach_private_scalar` Allows Attacker-Controlled Combined Public Key - (`src/cbmpc/api/eddsa2pc.cpp`)

### Summary

`attach_private_scalar` verifies that the caller's private scalar `x` satisfies `x*G == Qi_self`, but performs **no check** that the combined public key `Q` stored in the `public_key_blob` is consistent with `Qi_self`. An attacker who controls the `public_key_blob` input can supply any valid Ed25519 curve point as `Q`, pass all validation, and obtain a key blob whose combined public key is entirely attacker-chosen. Signing with that blob causes the honest peer to co-sign under the attacker-substituted `Q`.

---

### Finding Description

The relevant code is in `attach_private_scalar` at `src/cbmpc/api/eddsa2pc.cpp` lines 176–222.

The function performs these checks:

1. Parses `public_key_blob` → extracts `pub.Q_compressed`, `pub.role`, `pub.curve`
2. Validates `pub.role <= 1`, `pub.curve == ed25519`, `pub.Q_compressed` not empty
3. Parses `public_share_compressed` → `Qi_self`; validates it is on-curve and in-subgroup
4. Parses `private_scalar_fixed` → `x`; validates `x` is in range
5. **Checks `x * G == Qi_self`** — the only binding check
6. Parses `pub.Q_compressed` → `Q`; validates it is on-curve (no subgroup check, no relation to `Qi_self`)
7. Writes `pub.x_share = x`, `pub.Q_compressed = Q.to_compressed_bin()`, returns blob [1](#0-0) 

Step 6 is the gap. The function validates `Q` as a syntactically valid Ed25519 point, but never checks:

- `Q - Qi_self` is a valid non-infinity subgroup point (i.e., the peer's implied share is well-formed)
- `Q != Qi_self` (peer share is non-zero)
- Any relationship between `Q` and `Qi_self` at all

The API header documents the guarantee as only `x*G == public_share_compressed`: [2](#0-1) 

This is exactly the missing invariant: the `public_key_blob` is attacker-supplied, and `Q` inside it is never bound to `Qi_self`.

---

### Impact Explanation

The attacker is a legitimate API caller who is one of the two parties (e.g., P1). The attack flow:

1. Attacker runs DKG normally with the honest peer, establishing `Q_real = x1*G + x2*G`.
2. Attacker calls `detach_private_scalar` on their own blob to obtain `public_blob_real` (containing `Q_real`) and `x1_fixed`.
3. Attacker crafts a malicious `public_blob_evil` by serializing a `key_blob_v1_t` with `Q_compressed = Q_evil` (any valid Ed25519 subgroup point, e.g. `2*G`), keeping `role` and `curve` valid.
4. Attacker calls `attach_private_scalar(public_blob_evil, x1_fixed, x1*G, &out)`. The function checks `x1*G == Qi_self` ✓, checks `Q_evil` is on-curve ✓, and returns a blob with `Q = Q_evil`, `x_share = x1`.
5. Attacker initiates `sign` with the honest peer. The honest peer uses their own blob (with `Q_real`); the attacker uses the tampered blob (with `Q_evil`). The signing protocol uses `Q` from each party's local blob independently — there is no cross-party Q reconciliation during signing.
6. P1 (attacker) receives the final signature. The signature is verifiable under `Q_evil`, not `Q_real`. [3](#0-2) 

The honest peer co-signs without any indication that the combined key has been substituted. The attacker obtains a valid EdDSA signature under an attacker-chosen public key, with the honest peer's cryptographic contribution.

---

### Likelihood Explanation

- The attacker is a legitimate party in the 2-party protocol — no threshold collusion required.
- The malicious `public_key_blob` is a straightforward serialization of a `key_blob_v1_t` struct with an arbitrary `Q_compressed` field; no cryptographic forgery is needed.
- The honest peer has no mechanism to detect the substitution: `Q` is never exchanged or compared during signing.
- The `detach`/`attach` API is explicitly designed for backup/restore workflows, making this a natural attacker entry point. [4](#0-3) 

---

### Recommendation

In `attach_private_scalar`, after computing `Qi_self` and validating `Q`, add:

```cpp
// Derive the peer's implied share and validate it.
coinbase::crypto::ecc_point_t Qi_peer = Q - Qi_self;
if (Qi_peer.is_infinity()) return coinbase::error(E_FORMAT, "Q equals Qi_self: peer share is zero");
if (!Qi_peer.is_in_subgroup()) return coinbase::error(E_FORMAT, "Q - Qi_self is not in subgroup");
```

This ensures that `Q` is cryptographically bound to `Qi_self`: the peer's implied share `Q - Qi_self` must be a valid, non-trivial subgroup point. An attacker who does not know a valid `Qi_peer` consistent with `Q` cannot pass this check. [5](#0-4)

// x*G as Qi, but Q set to 2*G, and asserts the call returns E_FORMAT.

TEST(ApiEdDSA2pc, AttachPrivateScalar_QNotConsistentWithQi) {
  using namespace coinbase;
  using namespace coinbase::crypto;

  const auto curve = curve_ed25519;
  const auto& G = curve.generator();
  const mod_t& q = curve.order();

  // x = 1, Qi_self = 1*G = G
  bn_t x = bn_t::from_bin(buf_t{1});
  ecc_point_t Qi_self = x * G;

  // Q_evil = 2*G  (unrelated to any valid combined key with Qi_self = G
  //                unless peer share = G, but we don't check that here)
  bn_t two = bn_t::from_bin(buf_t{2});
  ecc_point_t Q_evil = two * G;

  // Serialize a public_key_blob with Q = Q_evil
  // (mimic key_blob_v1_t layout: version=1, role=0, curve=ed25519, Q_compressed, x_share=q)
  // ... (serialization omitted for brevity, follows key_blob_v1_t::convert layout)

  buf_t public_share = Qi_self.to_compressed_bin();
  buf_t x_fixed = x.to_bin(32);

  buf_t out;
  error_t rv = coinbase::api::eddsa_2p::attach_private_scalar(
      public_blob_evil, x_fixed, public_share, out);

  // Without the fix, rv == SUCCESS (vulnerability confirmed).
  // With the fix, rv == E_FORMAT.
  EXPECT_EQ(rv, E_FORMAT);
}

error_t attach_private_scalar(mem_t public_key_blob, mem_t private_scalar_fixed, mem_t public_share_compressed,
                              buf_t& out_key_blob) {
  if (const error_t rv = coinbase::api::detail::validate_mem_arg_max_size(public_key_blob, "public_key_blob",
                                                                          coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))
    return rv;
  key_blob_v1_t pub;
  error_t rv = coinbase::convert(pub, public_key_blob);
  if (rv) return rv;
  if (pub.version != key_blob_version_v1) return coinbase::error(E_FORMAT, "unsupported key blob version");
  if (pub.role > 1) return coinbase::error(E_FORMAT, "invalid key blob role");
  if (static_cast<curve_id>(pub.curve) != curve_id::ed25519) return coinbase::error(E_FORMAT, "invalid key blob curve");
  if (pub.Q_compressed.empty()) return coinbase::error(E_FORMAT, "invalid key blob");

  const auto curve = coinbase::crypto::curve_ed25519;
  const coinbase::crypto::mod_t& q = curve.order();
  const int order_size = q.get_bin_size();
  if (order_size <= 0) return coinbase::error(E_GENERAL, "invalid curve order size");

  if (const error_t rvm = coinbase::api::detail::validate_mem_arg(private_scalar_fixed, "private_scalar_fixed"))
    return rvm;
  if (private_scalar_fixed.size != order_size) return coinbase::error(E_BADARG, "private_scalar_fixed wrong size");

  if (const error_t rvp = coinbase::api::detail::validate_mem_arg(public_share_compressed, "public_share_compressed"))
    return rvp;

  coinbase::crypto::ecc_point_t Qi_self(curve);
  if (rv = Qi_self.from_bin(curve, public_share_compressed))
    return coinbase::error(rv, "invalid public_share_compressed");
  if (rv = curve.check(Qi_self)) return coinbase::error(rv, "invalid public_share_compressed");
  if (!Qi_self.is_in_subgroup()) return coinbase::error(E_FORMAT, "invalid public_share_compressed");

  const coinbase::crypto::bn_t x = coinbase::crypto::bn_t::from_bin(private_scalar_fixed) % q;
  if (!q.is_in_range(x)) return coinbase::error(E_FORMAT, "invalid private_scalar_fixed");

  const auto& G = curve.generator();
  if (x * G != Qi_self) return coinbase::error(E_FORMAT, "x_share mismatch key blob");

  // Validate and normalize global public key encoding.
  coinbase::crypto::ecc_point_t Q(curve);
  if (rv = Q.from_bin(curve, pub.Q_compressed)) return coinbase::error(rv, "invalid key blob");
  if (rv = curve.check(Q)) return coinbase::error(rv, "invalid key blob");

  pub.x_share = x;
  pub.Q_compressed = Q.to_compressed_bin();
  out_key_blob = coinbase::convert(pub);
  return SUCCESS;
}
