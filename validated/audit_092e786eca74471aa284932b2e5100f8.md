Audit Report

## Title
Unguarded `claim()` in `MerkleDistributor` Allows Anyone to Force Fee-Deducting Claims on Behalf of Any User - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary
`MerkleDistributor.claim()` accepts a caller-controlled `account` parameter with no `msg.sender == account` enforcement. Any unprivileged caller can supply a victim's publicly available merkle proof and force a claim on their behalf, permanently deducting up to 10% of the victim's unclaimed yield as a protocol fee sent to `protocolTreasury`. The newer `KernelMerkleDistributor` contract contains an explicit fix for this exact pattern that was never backported.

## Finding Description
`MerkleDistributor.claim()` (L97–147) accepts `account` as an external parameter and performs no check that `msg.sender == account`:

```solidity
// contracts/utils/MerkleDistributor/MerkleDistributor.sol L97-106
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
```

The merkle proof is constructed from `(index, account, cumulativeAmount)` (L120), not from `msg.sender`, so any caller who obtains the victim's tuple — which is published off-chain for self-claiming — can supply a valid proof for any `account`. The function then deducts a fee of up to 10% (`MAX_FEE_IN_BPS = 1000`, L47) and routes it to `protocolTreasury` (L138–144):

```solidity
uint256 fee = (claimableAmount * feeInBPS) / 10_000;
uint256 amountToSend = claimableAmount - fee;
IERC20(token).safeTransfer(account, amountToSend);
IERC20(token).safeTransfer(protocolTreasury, fee);
```

After execution, `userClaims[account].lastClaimedIndex` is updated (L134), permanently consuming the claim index. The victim cannot re-claim, cannot wait for a fee reduction, and has no recourse.

The developers recognized and fixed this exact pattern in `KernelMerkleDistributor._processClaim()` (L311–313):

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

This guard was never backported to `MerkleDistributor`.

## Impact Explanation
**High. Theft of unclaimed yield.** When `feeInBPS > 0`, a forced claim permanently reduces the victim's yield by up to 10% (up to `MAX_FEE_IN_BPS = 1000` BPS). The fee is irrecoverably transferred to `protocolTreasury` and the claim index is consumed. This matches the allowed impact "High. Theft of unclaimed yield" exactly.

## Likelihood Explanation
**High.** The attack requires no privilege. Merkle distribution contracts publish `(index, account, cumulativeAmount, merkleProof)` tuples off-chain so users can self-claim. Any observer can collect a victim's tuple and replay it. The only precondition is that the victim has not yet claimed. The attack is repeatable across all unclaimed accounts in every distribution round.

## Recommendation
Add `require(account == msg.sender, "Unauthorized")` (or the equivalent custom error revert) at the top of `MerkleDistributor.claim()`, consistent with the guard already present in `KernelMerkleDistributor._processClaim()` at L311–313 of `contracts/KERNEL/KernelMerkleDistributor.sol`.

## Proof of Concept
```
Setup:
  MerkleDistributor deployed with feeInBPS = 500 (5%)
  Alice is entitled to 100e18 tokens at index 1
  Alice's (index=1, account=alice, cumulativeAmount=100e18, proof=[...]) is published off-chain

Attack (Foundry test):
  vm.prank(attacker);  // attacker != alice
  merkleDistributor.claim(1, alice, 100e18, aliceProof);

Assertions:
  assertEq(token.balanceOf(alice), 95e18);           // alice receives only 95%
  assertEq(token.balanceOf(protocolTreasury), 5e18); // 5% permanently taken
  assertTrue(merkleDistributor.isClaimed(1, alice)); // claim index consumed
  // alice cannot re-claim; she permanently lost 5e18 without consent
```
