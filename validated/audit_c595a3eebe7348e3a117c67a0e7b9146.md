### Title
Unvalidated `ProposalInit.builder` Field Accepted as `sequencer_address` in Block Execution, Redirecting All Transaction Fees — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates every security-sensitive field of `ProposalInit` except `builder`. That field is passed verbatim to `convert_to_sn_api_block_info` as `sequencer_address` in the `BlockInfo` handed to the blockifier. A malicious proposer can set `builder` to any address; the validator accepts the proposal, executes all transactions with the attacker-controlled sequencer address, and commits a block in which all transaction fees are credited to the attacker.

---

### Finding Description

`ProposalInit` carries a `builder` field — the address of the entity that sequences the block — distinct from `proposer` (the consensus identity). [1](#0-0) 

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in the `BlockInfo` that drives execution: [2](#0-1) 

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `timestamp`, `starknet_version`, `version_constant_commitment`, all four L1 gas price fields, and `fee_proposal_fri`. It never checks `builder`: [3](#0-2) 

`ProposalInitValidation` — the struct that carries the validator's reference values — contains no `builder_address` field, so there is no reference value to compare against even if a check were added ad-hoc: [4](#0-3) 

`proposer` is validated separately in `handle_proposal` (checked against the committee-derived expected proposer), but `builder` is never checked anywhere in the validation path: [5](#0-4) 

The unvalidated `builder` value flows into `initiate_validation` → `convert_to_sn_api_block_info` → `ValidateBlockInput.block_info` → batcher execution: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

`sequencer_address` in `BlockInfo` is the address that receives all transaction fee payments. The blockifier's `complete_fee_transfer_flow` reads and updates the sequencer balance at this address: [8](#0-7) 

By setting `builder` to an attacker-controlled address, a malicious proposer causes:

1. **All transaction fees in the block to be credited to the attacker's address** instead of the legitimate sequencer — a direct economic theft matching the "Incorrect fee … balance … with economic impact" critical impact category.
2. **Every `get_sequencer_address` syscall in the block returns the wrong address**, corrupting execution results for any contract that branches on the sequencer identity.
3. **The committed block header contains the wrong `sequencer_address`**, producing a wrong block hash and wrong state root that downstream provers and verifiers will accept because the batcher computed the `ProposalCommitment` from this corrupted execution.

---

### Likelihood Explanation

The attack requires only that the malicious node be the elected proposer for a round — a normal, unprivileged role in Tendermint rotation. No special permissions, no key compromise, and no off-chain coordination are needed. The proposer simply sends a `ProposalInit` with `builder` set to an attacker-controlled address. Every validator node will accept it because `is_proposal_init_valid` has no check for this field and `ProposalInitValidation` carries no reference value for it.

---

### Recommendation

1. Add `builder_address: ContractAddress` to `ProposalInitValidation`, populated from the node's own configured builder address (the same source used in `initiate_build`).
2. In `is_proposal_init_valid`, assert `init_proposed.builder == proposal_init_validation.builder_address` and return `InvalidProposalInit` on mismatch, consistent with how `height`, `l1_da_mode`, and `l2_gas_price_fri` are checked.
3. Add a unit test analogous to `invalid_proposal_init` that mutates `init.builder` and asserts rejection.

---

### Proof of Concept

**Attacker-controlled proposer node** modifies `initiate_build` to set:

```rust
builder: attacker_address,  // instead of args.builder_address
```

**Validator node** receives the `ProposalInit`, calls `is_proposal_init_valid`:

- `height` ✓, `l1_da_mode` ✓, `l2_gas_price_fri` ✓, `timestamp` ✓, `starknet_version` ✓, `version_constant_commitment` ✓, all L1 gas prices ✓, `fee_proposal_fri` ✓
- `builder` — **not checked, passes silently**

`initiate_validation` calls `convert_to_sn_api_block_info(&init)`:

```rust
sequencer_address: init.builder,  // = attacker_address
```

The batcher executes all transactions with `sequencer_address = attacker_address`. Every fee transfer credits `attacker_address`. The batcher returns a `ProposalCommitment` derived from this execution. The validator sends it to consensus. If 2/3+ validators are honest but none of them are the proposer for this round, they all accept the proposal and the block is committed with the attacker's address as sequencer, permanently recording the wrong fee distribution in the state. #[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
