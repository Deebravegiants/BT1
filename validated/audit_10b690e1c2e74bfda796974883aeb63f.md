### Title
`bits_to_bytes_floor` Used Instead of `bits_to_bytes` in EdDSA HD Key Refresh Reduces Statistical Security of Refresh Masks — (File: `src/cbmpc/protocol/hd_keyset_eddsa_2p.cpp`)

---

### Summary

In `key_share_eddsa_hdmpc_2p_t::refresh`, the byte-size of each random refresh mask is computed with `bits_to_bytes_floor` (floor division, `bits >> 3`) instead of `bits_to_bytes` (ceiling division, `(bits + 7) >> 3`). For Ed25519 this silently discards 7 bits from each mask, reducing the statistical security of the key-refresh step from the intended `SEC_P_STAT = 40` bits to 33 bits. The function is reachable through the public API `coinbase::api::hd_keyset_eddsa_2p::refresh`.

---

### Finding Description

In `src/cbmpc/protocol/hd_keyset_eddsa_2p.cpp` lines 66–71:

```cpp
int rand_bitlen = q.get_bits_count() + SEC_P_STAT;   // 255 + 40 = 295 for Ed25519
int rand_size = bits_to_bytes_floor(rand_bitlen);     // ← floor: 295 >> 3 = 36 bytes = 288 bits
if (rv = agree_random(job, 2 * rand_bitlen, rand)) return rv;

bn_t r_x = bn_t::from_bin(rand.take(rand_size)) % q;
bn_t r_k = bn_t::from_bin(rand.skip(rand_size).take(rand_size)) % q;
```

The design intent is that each mask `r_x`, `r_k` is drawn from `rand_bitlen` bits of shared randomness so that, after reduction mod `q`, the statistical distance from uniform is `2^{-SEC_P_STAT}`. This requires **at least** `rand_bitlen` bits, i.e. `bits_to_bytes(rand_bitlen)` bytes (ceiling).

`bits_to_bytes_floor` is defined as `bits >> 3` (floor), while `bits_to_bytes` is `(bits + 7) >> 3` (ceiling):

```cpp
// include-internal/cbmpc/internal/core/utils.h
inline int bits_to_bytes_floor(int bits) { return bits >> 3; }   // floor
inline int bits_to_bytes(int bits)       { return (bits + 7) >> 3; }  // ceiling
```

For Ed25519 (`q.get_bits_count() = 255`, `SEC_P_STAT = 40`):

| | Value |
|---|---|
| `rand_bitlen` | 295 bits |
| `bits_to_bytes_floor(295)` | 36 bytes = **288 bits** |
| `bits_to_bytes(295)` | 37 bytes = 296 bits |
| Shortfall | **7 bits** |

`agree_random(job, 2 * rand_bitlen, rand)` generates `bits_to_bytes(590) = 74` bytes, but the code only consumes `36 + 36 = 72` bytes. The last 2 bytes are silently discarded, and each mask carries only 288 bits of entropy instead of 295.

After `% q` (255-bit reduction), the statistical distance from uniform is:
- **Actual:** `2^{-(288 − 255)} = 2^{-33}`
- **Intended:** `2^{-(295 − 255)} = 2^{-40}`

The same `bits_to_bytes_floor` pattern does **not** appear in the analogous ECDSA HD refresh path; every other random-sampling site in the library uses `bits_to_bytes` (ceiling).

---

### Impact Explanation

The key-refresh protocol is supposed to produce new shares `(x_share_new, k_share_new)` that are statistically indistinguishable from fresh uniform shares, with security `2^{-40}`. Because the masks are 7 bits short, the actual statistical distance is `2^{-33}`. An adversary who can observe or influence `≈ 2^{33}` refresh transcripts gains a non-negligible distinguishing advantage over the intended security level, potentially leaking partial information about the underlying EdDSA private key shares `x_share` and `k_share` that are used in subsequent signing operations.

---

### Likelihood Explanation

The bug is triggered on every call to the public API `coinbase::api::hd_keyset_eddsa_2p::refresh`. It is unconditional and deterministic: every refresh on Ed25519 produces masks that are 7 bits shorter than intended. No special attacker capability is required to reach this code path.

---

### Recommendation

Replace `bits_to_bytes_floor` with `bits_to_bytes` on line 67 of `src/cbmpc/protocol/hd_keyset_eddsa_2p.cpp`:

```cpp
// Before (wrong – floor division loses 7 bits for Ed25519):
int rand_size = bits_to_bytes_floor(rand_bitlen);

// After (correct – ceiling division preserves all rand_bitlen bits):
int rand_size = bits_to_bytes(rand_bitlen);
```

This matches the pattern used everywhere else in the library (e.g. `drbg_aes_ctr_t::gen_bn`, `hash_string_t::bitlen`, `hash_numbers_t::mod`) and restores the intended `2^{-40}` statistical security.
