### Title
Unauthenticated `farmer_reward_address_override` in `RespondSignatures` Enables Harvester-Controlled Block Reward Redirection — (`chia/farmer/farmer_api.py`)

---

### Summary

A malicious harvester peer can send a `RespondSignatures` message with `farmer_reward_address_override` set to an arbitrary attacker-controlled `bytes32`. The farmer's `_process_respond_signatures` blindly substitutes this value as `farmer_reward_address` in `DeclareProofOfSpace` and broadcasts it to all full nodes — with no cryptographic proof that the farmer authorized the override.

---

### Finding Description

In `_process_respond_signatures`, the farmer resolves the final reward address as follows:

```python
farmer_reward_address = self.farmer.farmer_target
if response.farmer_reward_address_override is not None:
    farmer_reward_address = response.farmer_reward_address_override   # ← no auth check
    include_source_signature_data = True
``` [1](#0-0) 

This `farmer_reward_address` is then placed directly into `DeclareProofOfSpace`:

```python
return farmer_protocol.DeclareProofOfSpace(
    ...
    farmer_reward_address,   # ← attacker-supplied value
    ...
)
``` [2](#0-1) 

The resulting message is broadcast to all full nodes:

```python
await self.farmer.server.send_to_all([message], NodeType.FULL_NODE)
``` [3](#0-2) 

The `respond_signatures` handler that feeds into this path accepts messages from any connected harvester peer: [4](#0-3) 

The only side-effect of the override being present is that `include_source_signature_data` is flipped to `True` — this does not constitute authorization. The BLS signatures assembled in the same function (`agg_sig_cc_sp`, `agg_sig_rc_sp`) cover the signage-point hashes, **not** the `farmer_reward_address`, so the full node cannot detect the substitution from the signature alone.

---

### Impact Explanation

Any harvester that is legitimately connected to a farmer (or that can establish a harvester connection) can redirect 100% of the farmer's block reward to an attacker-controlled puzzle hash. The farmer's private key is never consulted for this decision. The impact is permanent per-block reward diversion for every winning block whose `RespondSignatures` the attacker intercepts or originates.

---

### Likelihood Explanation

The preconditions are low-friction:
1. The attacker must be a connected harvester (normal operational relationship, or a compromised/malicious harvester operator).
2. A valid `sp_hash` and matching proof-of-space entry must exist in farmer state — both are populated during normal farming operation before `RespondSignatures` is sent.
3. No farmer key material, no MITM, no broken crypto required.

---

### Recommendation

The `farmer_reward_address_override` must be cryptographically authorized by the farmer before it is accepted. Options:

- **Reject the override entirely** if the design intent is that only the farmer controls the reward address. Remove the substitution at lines 917–918 and always use `self.farmer.farmer_target`.
- **Require a farmer-signed authorization**: the harvester must include a BLS signature over `(farmer_reward_address_override || sp_hash)` using the farmer's key (`response.farmer_pk`), and `_process_respond_signatures` must verify this signature before accepting the override.

---

### Proof of Concept

```python
# Minimal unit-test sketch (no network required)
import asyncio
from unittest.mock import MagicMock, AsyncMock
from chia_rs.sized_bytes import bytes32
from chia.farmer.farmer_api import FarmerAPI
from chia.protocols import harvester_protocol

ATTACKER_PH = bytes32(b"\xde\xad" * 16)

async def test_reward_redirect():
    farmer = MagicMock()
    farmer.farmer_target = bytes32(b"\x00" * 32)   # legitimate address
    farmer.sps = {SP_HASH: [sp_mock]}
    farmer.proofs_of_space = {SP_HASH: [(PLOT_ID, pospace_mock)]}
    farmer.get_private_keys.return_value = [sk_mock]
    farmer.constants = ...

    api = FarmerAPI(farmer)

    malicious_response = harvester_protocol.RespondSignatures(
        plot_identifier=PLOT_ID,
        challenge_hash=CHALLENGE_HASH,
        sp_hash=SP_HASH,
        local_pk=local_pk,
        farmer_pk=farmer_pk,
        message_signatures=[(SP_HASH, harv_sig_cc), (RC_SP, harv_sig_rc)],
        include_source_signature_data=False,
        farmer_reward_address_override=ATTACKER_PH,   # ← injected
    )

    result = api._process_respond_signatures(malicious_response)
    assert result is not None
    # Invariant violated: reward address is attacker-controlled, not farmer_target
    assert result.farmer_reward_address == ATTACKER_PH
    assert result.farmer_reward_address != farmer.farmer_target
```

The `DeclareProofOfSpace.farmer_reward_address` equals `ATTACKER_PH`, not `farmer.farmer_target`, confirming the invariant is violated with no key authorization. [5](#0-4)
