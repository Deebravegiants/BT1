### Title
Missing subgroup check on `root.Q` / `root.K` in `blob_to_keyset` allows torsion-contaminated Ed25519 public key to be accepted ŌĆö (`src/cbmpc/api/hd_keyset_eddsa_2p.cpp`)

### Summary
`blob_to_keyset` deserializes `root_Q_compressed` and `root_K_compressed` using `from_bin`, which only verifies that the point satisfies the curve equation. It never calls `curve.check`, which is the only guard that enforces prime-order subgroup membership. An attacker who supplies a crafted `keyset_blob` containing a torsion point for `root_Q_compressed` causes `extract_root_public_key_compressed` (and `refresh`/`derive_eddsa_2p_keys`) to return `SUCCESS` with a non-subgroup Ed25519 public key.

### Finding Description
`blob_to_keyset` in `src/cbmpc/api/hd_keyset_eddsa_2p.cpp` (lines 78ŌĆō80):

```cpp
error_t rv = keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed);
if (rv) return rv;
return keyset.root.K.from_bin(keyset.curve, blob.root_K_compressed);
// ŌåÉ no curve.check(keyset.root.Q) or curve.check(keyset.root.K)
```

Compare with `blob_to_key` in `src/cbmpc/api/eddsa2pc.cpp` (line 42):

```cpp
if (key.curve.check(key.Q)) return coinbase::error(E_FORMAT, "invalid key blob");
```

`ecurve_ed_t::from_bin` delegates to `ec25519_core::from_bin`, which solves `x┬▓ = (y┬▓ŌłÆ1)/(dy┬▓+1)` and returns `SUCCESS` for any valid curve point ŌĆö including the 8 torsion points of Ed25519's cofactor-8 group. `ecurve_t::check` is the only function that calls `is_in_subgroup()`, which performs the cofactor-multiplication test `P*(qŌłÆ1) == ŌłÆP`. Without it, torsion points pass silently.

The attack path for `extract_root_public_key_compressed`:
1. Craft a `keyset_blob_v1_t` with `root_Q_compressed` = compressed encoding of the order-2 torsion point (x=0, y=pŌłÆ1, i.e., bytes `ec ff ff ŌĆ” ff 7f`).
2. Call `extract_root_public_key_compressed(crafted_blob, out_Q_compressed)`.
3. `deserialize_keyset_blob` ŌåÆ `blob_to_keyset` ŌåÆ `from_bin` succeeds (torsion point is on the curve).
4. No `curve.check` is called; `keyset.root.Q` holds the torsion point.
5. `out_Q_compressed = keyset.root.Q.to_compressed_bin()` returns the torsion point bytes.
6. Return value is `SUCCESS`.

The same missing check affects `refresh` and `derive_eddsa_2p_keys`, which also call `deserialize_keyset_blob` and then use `keyset.root.Q` in the HD derivation protocol.

### Impact Explanation
`extract_root_public_key_compressed` is a public API whose documented contract is to return a valid 32-byte Ed25519 compressed public key. When it returns `SUCCESS` with a torsion-contaminated point, any caller that trusts the return code and uses the bytes as an Ed25519 public key receives an invalid key without any indication of the problem. This is a public-API-reachable validation bypass that produces accepted invalid cryptographic output (High impact per scope). For `refresh` and `derive_eddsa_2p_keys`, the torsion-contaminated `root.Q` propagates into the 2-party HD derivation protocol, potentially corrupting derived key material.

### Likelihood Explanation
The attacker only needs to supply a crafted blob to the public API ŌĆö no threshold collusion, no privileged access, no network position. The torsion point encoding is fixed and publicly known. The missing check is a straightforward omission relative to the parallel `blob_to_key` function in the same codebase.

### Recommendation
Add `curve.check` calls for both `root.Q` and `root.K` immediately after their respective `from_bin` calls in `blob_to_keyset`:

```cpp
error_t rv = keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed);
if (rv) return rv;
if (keyset.curve.check(keyset.root.Q)) return coinbase::error(E_FORMAT, "invalid keyset blob");

rv = keyset.root.K.from_bin(keyset.curve, blob.root_K_compressed);
if (rv) return rv;
if (keyset.curve.check(keyset.root.K)) return coinbase::error(E_FORMAT, "invalid keyset blob");
return SUCCESS;
```

This mirrors the pattern already used in `blob_to_key` (`eddsa2pc.cpp` line 42) and in `deserialize_ac_key_blob` (`eddsa_mp.cpp` line 242).
