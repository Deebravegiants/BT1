### Title
Startup/Transition Window Bypasses `fee_proposal_fri` Bounds Enforcement, Allowing Arbitrary Gas-Price Seeding — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

During the first `fee_proposal_window_size` (10) blocks after V0_14_3 activation — or near genesis — `fee_actual` is `None` because the sliding-window median cannot yet be computed. The validator's `is_proposal_init_valid` explicitly skips the `fee_proposal_fri` bounds check in this case. A malicious proposer can therefore set `fee_proposal_fri` to any value (e.g., `u128::MAX`) and every honest validator will accept the proposal. Those extreme values are committed to the chain, seed the `fee_proposals_window`, and after `window_size` blocks cause `fee_actual` — and therefore `next_l2_gas_price` — to become extreme for all subsequent blocks.

---

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` enforces that a proposer's `fee_proposal_fri` lies within a tight geometric band (`fee_proposal_margin_ppt = 2 ppt`) around `fee_actual`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

The guard is a Rust `if let` that only fires when **both** `fee_actual` and `fee_proposal` are `Some`. When `fee_actual` is `None` — which happens for any height below `window_size` or during the V0_14_2 → V0_14_3 transition when the window still contains pre-V0_14_3 `None` entries — the entire bounds check is silently skipped.

`fee_actual` is computed as the median of the `fee_proposals_window`:

```rust

    Some(None) | None => {
        warn!("Cannot compute fee_actual for height {height}: fee_proposals_window has no recorded fee_proposal for height {source_height}");
        return None;

``` [2](#0-1) 

The honest proposer is supposed to freeze at `l2_gas_price` when `fee_actual` is `None`:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [3](#0-2) 

But the validator **never enforces** this freeze. The comment "the validator derives the same fallback so both sides agree" describes only what an honest proposer does; it is not a validation rule.

The accepted `fee_proposal_fri` is then:
1. Bound into the `ProposalCommitment` via `Poseidon(partial_block_hash, fee_proposal_fri)`, so consensus signs over the extreme value:

```rust
ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
``` [4](#0-3) 

2. Stored in `BlockHeaderWithoutHash.fee_proposal_fri` and recorded into `fee_proposals_window`:

```rust
fee_proposal_fri: init.fee_proposal_fri,
``` [5](#0-4) 

3. After `window_size` blocks, `compute_fee_actual` returns the median of the seeded extreme values, which becomes the floor for `next_l2_gas_price` via `calculate_next_l2_gas_price_for_fin`:

```rust

        &self.fee_proposals_window,
        height,
        VersionedConstants::latest_constants().fee_proposal_window_size,
    );
    calculate_next_l2_gas_price_for_fin(
        self.l2_gas_price, height, l2_gas_used,
        self.config.dynamic_config.override_l2_gas_price_fri,
        &self.config.dynamic_config.min_l2_gas_price_per_height,
        fee_actual,
    )
}
``` [6](#0-5) 

The `next_l2_gas_price` is stored in the block header and becomes the `l2_gas_price` (part of `PartialBlockHashComponents`) for the next block, directly affecting all transaction fee calculations.

---

### Impact Explanation

A malicious proposer who wins any of the first 10 rounds after V0_14_3 activation (or during the V0_14_2 → V0_14_3 transition window) can set `fee_proposal_fri = u128::MAX`. Every honest validator accepts the proposal because `fee_actual` is `None` and the bounds check is skipped. After 10 such blocks, `compute_fee_actual` returns `u128::MAX` (median of extreme values). `calculate_next_l2_gas_price_for_fin` uses this as a floor, making `next_l2_gas_price = u128::MAX` for all subsequent blocks. Every transaction in those blocks is charged at `l2_gas_price = u128::MAX`, causing all transactions to fail with fee-overflow or to drain user balances. This is a permanent, chain-wide economic disruption that cannot be corrected without a protocol-level intervention.

The corrupted `next_l2_gas_price` is also part of `PartialBlockHashComponents` (as `l2_gas_price` of the next block), so the wrong value is committed into the block hash and the state root, making it an incorrect state commitment.

---

### Likelihood Explanation

The attack window is exactly `fee_proposal_window_size` (10) blocks, which occurs at every V0_14_3 protocol activation and at genesis. In a multi-validator BFT consensus setup (the design target of this sequencer), any validator that wins a proposer slot during those 10 blocks can trigger the attack. The attack requires no special privileges beyond being a scheduled proposer, which is a normal role in the consensus rotation.

---

### Recommendation

When `fee_actual` is `None` (startup/transition), the validator should enforce that `fee_proposal_fri` equals the locally-computed fallback (`l2_gas_price`), not accept any arbitrary value. Concretely, add a check in `is_proposal_init_valid`:

```rust
if fee_actual.is_none() {
    // Startup: proposer must freeze at l2_gas_price.
    if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
        if fee_proposal != proposal_init_validation.l2_gas_price_fri {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must equal l2_gas_price during startup \
                     (fee_actual unavailable): expected={}, got={}",
                    proposal_init_validation.l2_gas_price_fri.0, fee_proposal.0
                ),
            ));
        }
    }
}
```

This mirrors the honest proposer's own fallback logic and closes the invariant gap.

### Proof of Concept

1. Network activates V0_14_3 at block height H. `fee_proposals_window` is empty or contains only `None` entries (pre-V0_14_3 blocks).
2. For heights H through H+9, `compute_fee_actual` returns `None` because `height < window_size` or the window contains `None` entries.
3. A malicious proposer wins the proposer slot for height H. It constructs a `ProposalInit` with `fee_proposal_fri = Some(GasPrice(u128::MAX))`.
4. Every honest validator calls `is_proposal_init_valid`. The check at line 398 evaluates `if let (Some(fee_actual), Some(fee_proposal)) = (None, Some(u128::MAX))` — the pattern does not match, so the bounds check is skipped entirely. The proposal is accepted.
5. `decision_reached` calls `record_fee_proposal(H, Some(GasPrice(u128::MAX)))`, seeding the window.
6. The malicious proposer repeats for heights H+1 through H+9 (or bribes/controls enough proposers to do so).
7. At height H+10, `compute_fee_actual` returns `Some(GasPrice(u128::MAX))` (median of 10 extreme values).
8. `calculate_next_l2_gas_price_for_fin(..., fee_actual = Some(u128::MAX))` returns `GasPrice(u128::MAX)`.
9. Block H+10's header stores `next_l2_gas_price = u128::MAX`. Block H+11 uses `l2_gas_price = u128::MAX`.
10. All transactions in block H+11 and beyond are charged at `l2_gas_price = u128::MAX`, causing universal fee failure or balance drain. [1](#0-0) [8](#0-7) [9](#0-8)

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
