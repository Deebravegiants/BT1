### Title
Malicious P2P Peer Permanently Corrupts Deprecated Class Storage via Missing Hash Verification and Write-Once Idempotency — (`crates/apollo_p2p_sync/src/client/class.rs`, `crates/apollo_class_manager/src/class_storage.rs`)

---

### Summary

The p2p sync `write_to_storage` path for deprecated (Cairo 0) classes stores peer-supplied bytecode under a class hash **without verifying that the bytecode actually hashes to that class hash**. A `write-once` idempotency guard in `FsClassStorage::set_deprecated_class` then permanently blocks any subsequent honest write for the same hash. The result is that a single malicious p2p peer can permanently install attacker-controlled bytecode under any deprecated class hash the node has not yet synced, and the blockifier will silently execute that bytecode for every future invocation of that class.

---

### Finding Description

**Step 1 — No content-hash verification in `write_to_storage`.**

`write_to_storage` in `class.rs` iterates over the peer-supplied `(class_hash, deprecated_class)` pairs and calls `add_deprecated_class` for each one: [1](#0-0) 

The only validation performed by `parse_data_for_block` is that the `class_hash` appears in the already-committed state diff and that no duplicate hashes are sent in the same response: [2](#0-1) 

There is **no check** that the supplied class body actually hashes to `class_hash`. The code itself acknowledges this with an explicit TODO: [3](#0-2) 

**Step 2 — `FsClassStorage::set_deprecated_class` is write-once.**

`FsClassStorage::set_deprecated_class` checks whether the class already exists on disk and, if so, returns `Ok(())` without writing: [4](#0-3) 

The same early-return guard exists in the caching layer (`CachedClassStorage`): [5](#0-4) 

The comment in `rename_to_persistent_dir` explicitly relies on a **content-addressing invariant** that is never enforced: [6](#0-5) 

**Step 3 — `add_deprecated_class` passes through without verification.**

`ClassManager::add_deprecated_class` delegates directly to `set_deprecated_class` with no interposed hash check: [7](#0-6) 

**Combined effect:** A malicious peer sends `(class_hash_X, attacker_bytecode)` where `class_hash_X` is a real hash from the state diff but `attacker_bytecode` does not hash to it. The node stores `attacker_bytecode` under `class_hash_X`. Every subsequent call to `set_deprecated_class` for `class_hash_X` — from any honest peer — is silently dropped by the idempotency guard. The class manager is permanently poisoned.

---

### Impact Explanation

The blockifier retrieves deprecated class bytecode from the class manager and executes it directly. There is no re-verification of the class hash against the bytecode at execution time in the blockifier path (the OS-level hash check in `load_deprecated_class` only runs during proving, not during sequencer block production). Any transaction that invokes a contract whose class was poisoned will execute the attacker's bytecode, producing wrong storage writes, wrong events, wrong return values, and wrong state roots — all of which are committed to the chain by the sequencer.

Impact: **Critical — Wrong compiled class / contract code selected for execution.**

---

### Likelihood Explanation

The p2p sync client connects to unauthenticated peers. Any peer that the node selects for class sync can supply this payload. The attack requires only that the attacker's peer is contacted before an honest peer delivers the class for a given hash. Because the class sync marker advances block-by-block and classes are fetched on demand, the window is open for every deprecated class that has not yet been synced.

---

### Recommendation

1. **Verify the class hash before storage.** In `write_to_storage` (or inside `add_deprecated_class`), compute `compute_deprecated_class_hash(&deprecated_class)` and assert it equals `class_hash`. Reject and disconnect the peer if it does not match. The function already exists in the codebase: [8](#0-7) 

2. **Remove or condition the idempotency guard.** The write-once guard is only safe if the content-addressing invariant is guaranteed. Until hash verification is in place, the guard should at minimum log a mismatch and surface an error rather than silently succeed.

3. **Resolve the TODO.** The comment at line 39 of `class.rs` explicitly flags this missing check; it should be treated as a security-critical gap, not a deferred nicety.

---

### Proof of Concept

```
1. Attacker runs a p2p peer that the syncing sequencer node connects to.
2. The node requests classes for block N, which declares deprecated class_hash_X.
3. Attacker responds with (class_hash_X, attacker_bytecode) where
   compute_deprecated_class_hash(attacker_bytecode) != class_hash_X.
4. parse_data_for_block accepts the response: class_hash_X is in the state diff,
   no duplicate, no content check.
5. write_to_storage calls add_deprecated_class(class_hash_X, attacker_bytecode).
6. FsClassStorage::set_deprecated_class: contains_deprecated_class(class_hash_X)
   == false → writes attacker_bytecode to disk.
7. Node later connects to an honest peer for the same block.
8. Honest peer sends (class_hash_X, correct_bytecode).
9. FsClassStorage::set_deprecated_class: contains_deprecated_class(class_hash_X)
   == true → returns Ok(()) without writing. Poison is permanent.
10. Any transaction invoking a contract of class class_hash_X now executes
    attacker_bytecode via the blockifier.
```

pub fn compute_deprecated_class_hash(
    contract_class: &ContractClass,
) -> Result<Felt, HintedClassHashError> {
    let hinted_class_hash = compute_cairo_hinted_class_hash(contract_class)?;
    let contract_definition_vec = serde_json::to_vec(contract_class)?;
    let contract_definition: CairoContractDefinition<'_> =
        serde_json::from_slice(&contract_definition_vec)?;

    let FlatEntryPointFelts { external, l1_handler, constructor } =
        get_flat_entry_point_felts(&contract_definition.entry_points_by_type);
    let builtins = ascii_strs_as_felts(&contract_definition.program.builtins);
    let bytecode = hex_strs_as_felts(&contract_definition.program.data);

    let mut hash_state = HashState::<Pedersen>::new();
    hash_state.update_single(&DEPRECATED_COMPILED_CLASS_VERSION);
    hash_state.update_with_hashchain(&external);
    hash_state.update_with_hashchain(&l1_handler);
    hash_state.update_with_hashchain(&constructor);
    hash_state.update_with_hashchain(&builtins);
    hash_state.update_single(&hinted_class_hash);
    hash_state.update_with_hashchain(&bytecode);
    Ok(hash_state.finalize())
