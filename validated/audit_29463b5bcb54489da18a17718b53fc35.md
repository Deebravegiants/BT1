### Title
Missing RSA Modulus Primality Validation in `base_pke_rsa_ek_from_modulus` Allows PVE Ciphertext Under Attacker-Chosen Weak Key - (`src/cbmpc/api/pve_base_pke.cpp`)

### Summary
`base_pke_rsa_ek_from_modulus` accepts any 256-byte non-zero value as an RSA modulus with no primality or RSA-structure check. A caller supplying a weak composite modulus (e.g., n = 3 × large_prime, padded to 256 bytes) receives `SUCCESS` and a valid ek blob. `cbmpc_pve_encrypt` then produces a PVE ciphertext under that weak key, returning `CBMPC_SUCCESS`. The attacker trivially factors n and recovers the encrypted scalar.

### Finding Description
`base_pke_rsa_ek_from_modulus` performs only two checks before constructing the ek blob: [1](#0-0) 

It then calls `pk.set(n_bn, e_bn)`, which invokes `EVP_PKEY_fromdata` with `EVP_PKEY_PUBLIC_KEY` — OpenSSL does not perform primality checks on n during public key import: [2](#0-1) 

The ECIES counterpart explicitly validates the imported point against the curve: [3](#0-2) 

No equivalent guard exists for RSA. The resulting ek blob is accepted by `encrypt`, which routes through `kem_policy_rsa_oaep_t::encapsulate`: [4](#0-3) 

This calls `encrypt_oaep_with_seed`, which pads then calls `encrypt_raw` → `EVP_PKEY_encrypt` with `RSA_NO_PADDING`, computing `padded^e mod n` for any non-zero n: [5](#0-4) [6](#0-5) 

The OAEP padding size check (`emlen < 2*mdlen+1`) passes trivially for a 256-byte modulus with SHA-256 (255 ≥ 65). No layer in the chain rejects a structurally invalid modulus.

### Impact Explanation
The PVE ciphertext is produced under an attacker-chosen composite modulus. The attacker factors n (trivially if a small prime factor was chosen, e.g., p=3), computes `d = e⁻¹ mod φ(n)`, and decrypts the RSA-OAEP KEM ciphertext to recover the KEM shared secret, then decrypts the AES-GCM DEM layer to recover the encrypted scalar x. The backup is fully recoverable without the HSM. This is the primary use case for `base_pke_rsa_ek_from_modulus` — HSM-backed key recovery — so the impact directly undermines the security guarantee of the PVE scheme.

### Likelihood Explanation
The function is explicitly designed for HSM scenarios where the modulus is exported from an external device (e.g., YubiHSM 2). A malicious or compromised HSM vendor, or an attacker who can intercept/substitute the exported modulus before it reaches `base_pke_rsa_ek_from_modulus`, can supply a weak modulus. The library provides no defense. The asymmetry with the ECIES counterpart (which validates the point) makes this an exploitable oversight rather than a documented caller responsibility.

### Recommendation
Add RSA modulus structural validation in `base_pke_rsa_ek_from_modulus` before constructing the ek blob:
- Check that n is odd (`BN_is_odd(n_bn)`).
- Check that n has the expected bit length (2048 bits, i.e., the high bit is set).
- Optionally run a probabilistic primality check on n to confirm it is composite (RSA moduli must be composite; a prime n would be trivially invertible). OpenSSL provides `BN_check_prime` for this.
- Mirror the pattern of `base_pke_ecies_p256_ek_from_oct`, which calls `curve_p256.check(pk)` before accepting the key. [7](#0-6)

error_t base_pke_rsa_ek_from_modulus(mem_t modulus, buf_t& out_ek) {
  if (const error_t rv = coinbase::api::detail::validate_mem_arg_max_size(modulus, "modulus",
                                                                          coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))
    return rv;

  constexpr int kExpectedModulusBytes = coinbase::crypto::RSA_KEY_LENGTH / 8;
  if (modulus.size != kExpectedModulusBytes)
    return coinbase::error(E_BADARG, "modulus must be exactly 256 bytes (RSA-2048)");

  constexpr unsigned long kDefaultExponent = 65537;

  BIGNUM* n_bn = BN_bin2bn(modulus.data, modulus.size, nullptr);
  if (!n_bn) return coinbase::error(E_GENERAL, "BN_bin2bn(modulus) failed");

  if (BN_is_zero(n_bn)) {
    BN_free(n_bn);
    return coinbase::error(E_BADARG, "modulus must not be zero");
  }

  BIGNUM* e_bn = BN_new();
  if (!e_bn) {
    BN_free(n_bn);
    return coinbase::error(E_GENERAL, "BN_new(e) failed");
  }
  BN_set_word(e_bn, kDefaultExponent);

  coinbase::crypto::rsa_pub_key_t pk;
  pk.set(n_bn, e_bn);
  BN_free(n_bn);
  BN_free(e_bn);

  base_pke_ek_blob_v1_t ek_blob;
  ek_blob.key_type = static_cast<uint32_t>(base_pke_key_type_v1::rsa_oaep_2048);
  ek_blob.rsa_ek = std::move(pk);

  out_ek = coinbase::convert(ek_blob);
  return SUCCESS;
}

error_t rsa_pub_key_t::encrypt_raw(mem_t in, buf_t& out) const {
  int n_size = size();
  if (n_size != in.size) return coinbase::error(E_CRYPTO);

  scoped_ptr_t<EVP_PKEY_CTX> ctx = EVP_PKEY_CTX_new(ptr, NULL);
  if (EVP_PKEY_encrypt_init(ctx) <= 0) return openssl_error("RSA encrypt RAW error");
  if (EVP_PKEY_CTX_set_rsa_padding(ctx, RSA_NO_PADDING) <= 0) return openssl_error("RSA encrypt RAW error");
  size_t outlen = n_size;
  if (EVP_PKEY_encrypt(ctx, out.alloc(n_size), &outlen, in.data, in.size) <= 0)
    return openssl_error("RSA encrypt RAW error");
  return SUCCESS;
