### Title
Missing Stale-Vote Cleanup in `vote_cancel_resharing` Allows Threshold Bypass for Governance Decisions - (File: `crates/contract/src/lib.rs`)

---

### Summary

`vote_reshared()` spawns six cleanup promises to remove stale governance votes from participants who are no longer in the active set. `vote_cancel_resharing()`, which also transitions the contract back to a Running state, performs no equivalent cleanup. Stale votes cast by participants who were in the proposed-but-cancelled resharing set remain in every governance vote map and count toward thresholds in subsequent calls, lowering the effective quorum required to whitelist a malicious Docker image hash or launcher hash.

---

### Finding Description

When resharing completes successfully via `vote_reshared()`, the contract spawns six detached cleanup promises: [1](#0-0) 

These remove stale votes from `proposed_updates`, `tee_state` (code-hash, launcher, measurement votes), `foreign_chains`, `node_migrations`, and `tee_verifier_votes` for accounts that are no longer participants.

`vote_cancel_resharing()` also transitions the contract back to a Running state (the previous one), but spawns **no cleanup at all**: [2](#0-1) 

During resharing, `vote_code_hash`, `vote_add_launcher_hash`, `vote_add_os_measurement`, and `vote_tee_verifier_change` authenticate callers against the **new proposed participant set** (the active set during resharing, per `assert_caller_is_attested_participant_and_protocol_active`): [3](#0-2) 

Participants who exist only in the proposed new set — not in the old Running set — can therefore cast governance votes during resharing. If resharing is subsequently cancelled, those participants are no longer in the active set, but their votes remain in:

- `tee_state.votes` (Docker image hash votes)
- `tee_state.launcher_votes`
- `tee_state.measurement_votes`
- `proposed_updates.vote_by_participant`
- `tee_verifier_votes`

The existing test suite explicitly documents that without `clean_non_participant_votes`, stale votes from removed participants **do** count toward quorum: [4](#0-3) 

The `vote_code_hash` threshold check reads the raw vote count returned by `tee_state.vote()`, which includes stale entries: [5](#0-4) 

`clean_non_participant_votes` is the function that would remove these entries, but it is never called after `vote_cancel_resharing`: [6](#0-5) 

---

### Impact Explanation

An adversary who is included in a resharing proposal (requiring threshold votes from existing participants, but achievable via social engineering or a legitimate operator rotation) can vote for a malicious Docker image hash during resharing. If the resharing is then cancelled — a realistic operational event — those votes persist. Subsequent votes from a reduced number of legitimate old participants can push the tally past the threshold, whitelisting a malicious image. Nodes running that image can then submit valid TEE attestations, join the participant set, and participate in threshold signing, enabling unauthorized signature issuance or key-share exposure.

This breaks the production safety invariant that governance thresholds must be met by **current** participants only, as explicitly stated in the `vote_update` comment: [7](#0-6) 

---

### Likelihood Explanation

A resharing cancellation is a realistic operational event (network instability, participant disagreement, or a failed attempt). The attacker must be included in a resharing proposal, which requires threshold votes from existing participants — a meaningful but not insurmountable bar (operator rotation, social engineering). Once included, casting a vote for a malicious hash during resharing is a single on-chain call. The stale vote then persists indefinitely until a manual cleanup is triggered.

---

### Recommendation

Add the same set of cleanup promises to `vote_cancel_resharing()` that `vote_reshared()` already spawns. At minimum, spawn `CLEAN_TEE_STATUS`, `REMOVE_NON_PARTICIPANT_UPDATE_VOTES`, `REMOVE_NON_PARTICIPANT_TEE_VERIFIER_VOTES`, and `CLEAN_FOREIGN_CHAIN_DATA` as detached promises after the state transition succeeds, mirroring the pattern at lines 1175–1235 of `lib.rs`. [1](#0-0) 

---

### Proof of Concept

1. Contract is in Running state with old participants `{P1, P2, P3}`, threshold = 2.
2. `vote_new_parameters` is called to propose resharing with new set `{P1, P2, P3, P4}`.
3. During resharing, `P4` (attacker) calls `vote_code_hash(malicious_hash)`. Vote count for `malicious_hash` = 1. Threshold not yet met.
4. `vote_cancel_resharing` is called by old participants; resharing is cancelled. Contract reverts to `{P1, P2, P3}`. `P4`'s vote for `malicious_hash` remains in `tee_state.votes`.
5. `P1` calls `vote_code_hash(malicious_hash)`. `tee_state.vote()` returns count = 2 (P4's stale vote + P1's fresh vote). Threshold = 2 is met.
6. `whitelist_tee_proposal(malicious_hash)` executes — the malicious image hash is now whitelisted with only **one** legitimate participant vote instead of the required two.
7. `P4` submits a TEE attestation referencing `malicious_hash`, passes `add_participant`, and joins the active set, gaining signing capability. [2](#0-1) [8](#0-7)

    #[handle_result]
    pub fn vote_code_hash(&mut self, code_hash: NodeImageHash) -> Result<(), Error> {
        log!(
            "vote_code_hash: signer={}, code_hash={:?}",
            env::signer_account_id(),
            code_hash,
        );
        self.voter_or_panic();

        let threshold_parameters = self.protocol_state.threshold_parameters_or_panic();

        let participant = AuthenticatedParticipantId::new(threshold_parameters.participants())?;
        let votes = self.tee_state.vote(code_hash, &participant);

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        // If the vote threshold is met and the new Docker hash is allowed by the TEE's RTMR3,
        // update the state
        if votes >= self.threshold()?.value() {
            self.tee_state
                .whitelist_tee_proposal(code_hash, tee_upgrade_deadline_duration);
        }

        Ok(())
    }

    /// Stale CodeHashesVotes entries from removed participants must not count toward
    /// quorum after resharing.
    ///
    /// Scenario (N=5, T=3):
    /// 1. P1 and P2 vote for malicious hash before resharing.
    /// 2. Resharing removes P1 and P2. New set: {P3, P4, P5}.
    /// 3. clean_non_participant_votes removes stale votes.
    /// 4. P3 votes for the same hash — only 1 vote, not 3.
    #[test]
    fn test_clean_non_participant_votes_removes_stale_votes() {
        // Build 5 participants
        let mut all_participants = Participants::new();
        let mut account_ids = Vec::new();
        for i in 0..5 {
            let (account_id, info) = gen_participant(i);
            account_ids.push(account_id.clone());
            all_participants.insert(account_id, info).unwrap();
        }

        let mut tee_state = TeeState::default();

        // P0 and P1 vote for a malicious hash before resharing
        let malicious_hash = NodeImageHash::from([0xAA; 32]);
        for account_id in &account_ids[0..2] {
            let mut ctx = VMContextBuilder::new();
            ctx.signer_account_id(account_id.clone());
            testing_env!(ctx.build());
            let auth_id = AuthenticatedParticipantId::new(&all_participants).unwrap();
            tee_state.votes.vote(malicious_hash, &auth_id);
        }
        assert_eq!(tee_state.votes.proposal_by_account.len(), 2);

        // Resharing removes P0 and P1. New participant set: {P2, P3, P4}.
        let new_participants = all_participants.subset(2..5);

        // Clean non-participants (as done by CLEAN_TEE_STATUS after resharing)
        tee_state.clean_non_participant_votes(&new_participants);

        // Stale votes must be removed
        assert_eq!(tee_state.votes.proposal_by_account.len(), 0);

        // P2 votes for the same malicious hash — should be only 1 vote, not 3
        let p2_account = &account_ids[2];
        let mut ctx = VMContextBuilder::new();
        ctx.signer_account_id(p2_account.clone());
        testing_env!(ctx.build());
        let auth_id = AuthenticatedParticipantId::new(&new_participants).unwrap();
        let vote_count = tee_state.votes.vote(malicious_hash, &auth_id);
        assert_eq!(vote_count, 1, "Only the fresh vote from P2 should count");
    }
