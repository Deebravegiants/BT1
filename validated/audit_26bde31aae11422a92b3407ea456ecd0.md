Audit Report

## Title
`stakedButUnverifiedNativeETH` Never Decremented on Pre-Verification Beacon-Chain Slash, Permanently Overstating TVL and Enabling Redemption at Inflated rsETH Price — (`contracts/NodeDelegator.sol`)

## Summary
`NodeDelegator.stake32Eth` unconditionally increments `stakedButUnverifiedNativeETH` by 32 ETH, and the only decrement path is inside `verifyWithdrawalCredentials`, which subtracts a flat `validatorFields.length * 32 ether` regardless of the validator's actual post-slash effective balance. When a beacon-chain slash reduces a validator's balance below 32 ETH before credentials are verified, the protocol's reported TVL permanently overstates the real value by the slashing delta. Any rsETH holder can redeem via the permissionless `instantWithdrawal` path at this inflated price, extracting more ETH than their proportional share; the deficit is socialised to remaining holders when `verifyWithdrawalCredentials` is eventually called and EigenLayer awards only the reduced effective balance.

## Finding Description

**Root cause — single increment, single flat decrement, no slash-aware correction:**

`stake32Eth` increments `stakedButUnverifiedNativeETH` by exactly 32 ETH: [1](#0-0) 

The only decrement is in `verifyWithdrawalCredentials`, which blindly subtracts `validatorFields.length * 32 ether`: [2](#0-1) 

There is no code path that reads the actual beacon-chain effective balance or adjusts `stakedButUnverifiedNativeETH` when a slash occurs before verification. The variable is a storage slot on `NodeDelegator` with no external write path other than these two locations. [3](#0-2) 

**TVL propagation — overstated value flows into price:**

`getEffectivePodShares()` sums `stakedButUnverifiedNativeETH` with EigenLayer's `withdrawableShare` unconditionally: [4](#0-3) 

`getETHDistributionData` aggregates this across all NDCs: [5](#0-4) 

`_getTotalEthInProtocol` sums all asset deposits including ETH: [6](#0-5) 

**Permissionless price update — no access control:**

`updateRSETHPrice()` is `public` with only a `whenNotPaused` guard; any EOA can call it: [7](#0-6) 

The new price is computed directly from the overstated TVL: [8](#0-7) 

**Why the `pricePercentageLimit` guard does not block the attack:**

The guard at L252–266 only triggers when `newRsETHPrice > highestRsethPrice`. A beacon-chain slash does not cause the reported price to *increase* — `stakedButUnverifiedNativeETH` is unchanged by the slash, so the TVL figure and therefore the price remain at the same level they were immediately after staking. The price is already overstated relative to reality; calling `updateRSETHPrice()` after the slash does not push the price above `highestRsethPrice`, so the guard never fires. [9](#0-8) 

**Redemption at inflated price:**

`instantWithdrawal` is a permissionless, non-operator path: [10](#0-9) 

`getExpectedAssetAmount` multiplies the rsETH amount by the current (overstated) `rsETHPrice`: [11](#0-10) 

**Exploit sequence:**
1. Operator calls `stake32Eth`; `stakedButUnverifiedNativeETH += 32 ETH`. Price is accurate.
2. Validator is slashed on the beacon chain; effective balance drops to, e.g., 30 ETH. `stakedButUnverifiedNativeETH` is not updated — still 32 ETH.
3. Attacker observes the slash via public beacon-chain data.
4. Attacker calls `updateRSETHPrice()` (no role required). Price is unchanged from the protocol's view but is now overstated by `2 ETH / rsETHSupply` relative to real value.
5. Attacker calls `instantWithdrawal(ETH, attackerRsETH, "")`. Receives `attackerRsETH * inflatedPrice / 1e18` ETH — more than their proportional share of real TVL.
6. Operator eventually calls `verifyWithdrawalCredentials`; EigenLayer awards 30 ETH shares; `stakedButUnverifiedNativeETH -= 32 ETH`. Net TVL drops by 2 ETH. Remaining holders bear the loss.

## Impact Explanation

**High — Theft of unclaimed yield.**

The attacker redeems rsETH at a price that overstates the protocol's real ETH holdings by the slashing delta. The shortfall is socialised to remaining rsETH holders when the price corrects after `verifyWithdrawalCredentials`. The magnitude per event is bounded by the initial slashing penalty (typically 1–2 ETH per validator), but the attack is repeatable across multiple slashed validators and multiple NDCs. The loss is concrete and measurable, not hypothetical.

## Likelihood Explanation

- Beacon-chain slashings are infrequent but are real production events on Ethereum mainnet.
- The vulnerable window (between `stake32Eth` and `verifyWithdrawalCredentials`) is days to weeks in normal operation.
- `updateRSETHPrice()` requires no role; any EOA can call it.
- `instantWithdrawal` requires no operator involvement and is a supported production path.
- The `pricePercentageLimit` upside guard is not triggered because the price does not increase — it was already overstated from the moment of staking.
- No admin compromise, governance attack, or oracle manipulation is required.

## Recommendation

In `verifyWithdrawalCredentials`, replace the flat `validatorFields.length * 32 ether` decrement with a delta-based approach:

1. Read `podOwnerDepositShares` from `IEigenPodManager` **before** calling `eigenPod.verifyWithdrawalCredentials`.
2. Call `eigenPod.verifyWithdrawalCredentials`.
3. Read `podOwnerDepositShares` again **after** the call.
4. Decrement `stakedButUnverifiedNativeETH` by `validatorFields.length * 32 ether` as today, but also record the actual awarded shares delta so `getEffectivePodShares()` reflects the real post-slash balance immediately.

Alternatively, add an operator-callable function that accepts a beacon proof of a slash and writes down `stakedButUnverifiedNativeETH` by the proven loss amount, allowing TVL correction before `verifyWithdrawalCredentials` is called.

## Proof of Concept

```solidity
// Foundry fork test (Holesky or mainnet fork)
function test_slashBeforeVerification_inflatesPrice() external {
    // 1. Stake one validator via NodeDelegator
    vm.prank(operator);
    nodeDelegator.stake32Eth{value: 32 ether}(pubkey, sig, ddRoot);
    assertEq(nodeDelegator.stakedButUnverifiedNativeETH(), 32 ether);

    // 2. Simulate beacon-chain slash: validator effective balance = 30 ETH.
    //    No on-chain state changes in NodeDelegator — stakedButUnverifiedNativeETH unchanged.
    //    EigenPodManager.getWithdrawableShares returns 0 (credentials not yet verified).

    // 3. Record rsETH price before and after slash
    uint256 priceBefore = lrtOracle.rsETHPrice();

    // 4. Attacker calls updateRSETHPrice (no role required)
    vm.prank(attacker);
    lrtOracle.updateRSETHPrice();
    uint256 priceAfter = lrtOracle.rsETHPrice();

    // Price is overstated by 2 ETH / rsETHSupply relative to real value
    // (priceBefore == priceAfter from protocol's view, but real value is 2 ETH less)

    // 5. Attacker redeems rsETH at inflated price
    uint256 ethBefore = attacker.balance;
    vm.prank(attacker);
    withdrawalManager.instantWithdrawal(ETH, attackerRsETH, "");
    uint256 ethReceived = attacker.balance - ethBefore;

    // 6. verifyWithdrawalCredentials: EigenLayer awards 30 ETH shares
    vm.prank(operator);
    nodeDelegator.verifyWithdrawalCredentials(...); // stakedButUnverifiedNativeETH -= 32 ETH

    // 7. Price drops; remaining holders bear 2 ETH loss
    lrtOracle.updateRSETHPrice();
    uint256 priceFinal = lrtOracle.rsETHPrice();
    assertLt(priceFinal, priceAfter); // price corrected downward

    // 8. Assert attacker received more than proportional share
    assertGt(ethReceived, attackerRsETH * realPrice / 1e18);
}
```
