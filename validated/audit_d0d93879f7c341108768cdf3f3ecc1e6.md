### Title
Production Consensus Signing Uses Publicly-Known Hardcoded Private Key — (`File: crates/apollo_signature_manager/src/lib.rs`)

---

### Summary

The production `create_signature_manager()` factory function unconditionally instantiates `LocalKeyStore` with a hardcoded, publicly-visible private key (`0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133`). This key is used to sign consensus **precommit votes** and **peer identity challenges**. Because the key is embedded in the open source repository, any observer can derive it and forge valid signatures for arbitrary block hashes, impersonating the sequencer node in consensus.

---

### Finding Description

`crates/apollo_signature_manager/src/lib.rs` exports the production factory:

```rust
pub fn create_signature_manager() -> SignatureManager {
    SignatureManager::new()   // → LocalKeyStoreSignatureManager::new()
                              //   → LocalKeyStore::new_for_testing()
}
``` [1](#0-0) 

`LocalKeyStoreSignatureManager::new()` calls `LocalKeyStore::new_for_testing()`, which hard-codes both the private and public key as compile-time constants:

```rust
pub(crate) const fn new_for_testing() -> Self {
    const PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
        "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
    ));
    const PUBLIC_KEY: PublicKey = PublicKey(Felt::from_hex_unchecked(
        "0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a",
    ));
    Self { private_key: PRIVATE_KEY, public_key: PUBLIC_KEY }
}
``` [2](#0-1) 

This function is **not** gated by `#[cfg(test)]`. The production node component builder calls `create_signature_manager()` directly:

```rust
let signature_manager = match config.components.signature_manager.execution_mode {
    ReactiveComponentExecutionMode::LocalExecutionWithRemoteDisabled
    | ReactiveComponentExecutionMode::LocalExecutionWithRemoteEnabled => {
        Some(create_signature_manager())
    }
    ...
};
``` [3](#0-2) 

The `SignatureManager` uses this key to produce two categories of signatures:

- `sign_precommit_vote(block_hash)` — ECDSA signature over a consensus precommit vote
- `sign_identification(peer_id, challenge)` — ECDSA signature over a peer identity challenge [4](#0-3) 

The TODO comment in `lib.rs` explicitly acknowledges the key management is unresolved for production:

> `// TODO(Elin): understand how key store would look in production and better define the way the signature manager is created.` [1](#0-0) 

---

### Impact Explanation

The private key `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` is publicly readable from the repository. Any party can:

1. Compute `ecdsa_sign(PRIVATE_KEY, build_precommit_vote_message_digest(any_block_hash))` to produce a valid precommit vote signature for **any** block hash.
2. Impersonate the sequencer's consensus identity, forging precommit votes that pass `verify_precommit_vote_signature` checks against the known public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a`.
3. Forge peer identity signatures, passing `verify_identity` checks.

This maps to the allowed impact: **"Transaction conversion or signature/hash logic binds the wrong signer"** and **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value"** — specifically, consensus precommit votes signed by a universally-known key carry no authenticity guarantee.

---

### Likelihood Explanation

- The private key is in a public repository, readable by anyone.
- No runtime configuration, environment variable, or secret injection overrides it — `create_signature_manager()` takes no arguments and ignores node config.
- The `testing_secrets.json` file shows that network `secret_key` fields are injected via config, but no equivalent injection path exists for the signature manager's private key. [5](#0-4) 

---

### Recommendation

**Short term:** Gate `LocalKeyStore::new_for_testing()` with `#[cfg(test)]` or `#[cfg(feature = "testing")]` so it cannot be compiled into production binaries. Add a compile-time or startup-time assertion that panics if no real key is configured.

**Long term:** Implement a production `KeyStore` that loads the private key from a secrets manager, HSM, or at minimum from a `Sensitive<PrivateKey>` config field (analogous to how `consensus_manager_config.network_config.secret_key` is handled as `"privacy": "Private"` in the config schema). The `_new(private_key: PrivateKey)` constructor already exists on `LocalKeyStore` and is ready to be wired to a config-supplied key. [6](#0-5)

### Proof of Concept

```rust
use starknet_core::crypto::ecdsa_sign;
use starknet_types_core::felt::Felt;

// Publicly known key from source code
let private_key = Felt::from_hex_unchecked(
    "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133"
);

// Forge a precommit vote for any block hash
let target_block_hash = Felt::from_hex_unchecked("0xdeadbeef");
let mut message = b"PRECOMMIT_VOTE".to_vec();
message.extend_from_slice(&target_block_hash.to_bytes_be());
let digest = blake2s_to_felt(&message);

// Produces a signature that passes verify_precommit_vote_signature()
let forged_sig = ecdsa_sign(&private_key, &digest).unwrap();
```

The forged signature will pass `verify_precommit_vote_signature(target_block_hash, forged_sig.into(), PUBLIC_KEY)` because the public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a` is also hardcoded and publicly known, and the ECDSA relationship holds. [7](#0-6) [8](#0-7)

fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
