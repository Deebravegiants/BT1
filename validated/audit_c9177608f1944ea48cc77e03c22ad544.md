### Title
Missing Cairo0 Class Existence Check in `ExecutionStateReader::get_compiled_class` When Class Manager Is Active - (File: crates/apollo_rpc_execution/src/state_reader.rs)

### Summary

`ExecutionStateReader::get_compiled_class` skips the state-number existence check for Cairo0 (deprecated) contract classes when the class manager client is configured. For Cairo1 classes the same code path calls `is_contract_class_declared` against `self.state_number` before returning the class; for Cairo0 classes it returns the class unconditionally. A caller that queries `starknet_call`, `starknet_estimateFee`, or `starknet_simulateTransactions` at a historical block number before a Cairo0 class was declared will receive an authoritative-looking successful result instead of the correct `UndeclaredClassHash` error.

### Finding Description

In `crates/apollo_rpc_execution/src/state_reader.rs`, `ExecutionStateReader::get_compiled_class` has three code paths:

1. **Pending data** – returns whatever is in the pending block (no state-number check needed).
2. **Class manager path** (lines 115–141) – queries `class_manager_client.get_executable(class_hash)`. For `ContractClass::V1` it calls `is_contract_class_declared(..., self.state_number)` and returns `UndeclaredClassHash` if the class was not yet declared at that state. For `ContractClass::V0` it returns the class **without any existence check**, with a TODO comment: `// TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.`
3. **Storage fallback** (lines 144–160) – calls `get_contract_class(..., self.state_number)` which correctly enforces the state-number boundary.

The storage fallback is only reached when `class_manager_handle` is `None`. In the production RPC execution configuration the class manager is wired in, so the class manager path is always taken and the Cairo0 existence check is permanently bypassed. [1](#0-0) 

The asymmetry is explicit: Cairo1 classes are guarded, Cairo0 classes are not. [2](#0-1) 

The class manager stores classes globally without block-number metadata. Once a Cairo0 class is added (e.g., during sync of block N), `get_executable` will return it for any query regardless of the `state_number` of the `ExecutionStateReader`. [3](#0-2) 

### Impact Explanation

Any RPC method that creates an `ExecutionStateReader` with a class manager handle and a historical `state_number` is affected:

- `starknet_call` at block B < N returns a successful execution result for a transaction that invokes (via `library_call` syscall) a Cairo0 class declared in block N. The correct result is a revert with `UndeclaredClassHash`.
- `starknet_estimateFee` returns a fee estimate for a transaction that would actually fail on-chain.
- `starknet_simulateTransactions` shows a successful simulation trace.

These are authoritative-looking wrong values served by the RPC layer, matching the **High** impact criterion: *"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."*

### Likelihood Explanation

- The class manager is the production configuration; `class_manager_handle` is `None` only in tests.
- Cairo0 classes are legacy but still present on mainnet and testnet.
- The trigger is unprivileged: any user can call `starknet_call` or `starknet_estimateFee` with an arbitrary `block_id`.
- The scenario requires querying at a block before a Cairo0 class was declared while a contract in scope makes a `library_call` to that class — a realistic pattern for contracts that use shared Cairo0 libraries.
- The TODO comment confirms the developers are aware the check is missing and have not yet fixed it.

### Recommendation

Add the same `is_contract_class_declared` (or equivalent `get_deprecated_class_definition_block_number`) check for `ContractClass::V0` in the class manager branch, mirroring the Cairo1 guard:

```rust
ContractClass::V0(deprecated_contract_class) => {
    // Check that the class was declared at or before self.state_number.
    let declared_block = self.storage_reader
        .begin_ro_txn().map_err(storage_err_to_state_err)?
        .get_state_reader().map_err(storage_err_to_state_err)?
        .get_deprecated_class_definition_block_number(&class_hash)
        .map_err(storage_err_to_state_err)?;
    match declared_block {
        Some(bn) if !self.state_number.is_before(bn) => {
            Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
        }
        _ => Err(StateError::UndeclaredClassHash(class_hash)),
    }
}
```

The `SyncStateReader` used in the gateway/batcher path already implements this correctly via `is_class_declared_at`, which checks both Cairo1 and Cairo0 block numbers. [4](#0-3) 

### Proof of Concept

1. Node is running with the class manager enabled (`class_manager_handle` is `Some`).
2. Block 0 is committed. Block 1 declares Cairo0 class `C` (hash `0xC1a55`). Block 2 deploys contract `A` with class `C`.
3. Contract `B` (deployed in block 0 with a Cairo1 class) contains an entry point that issues a `library_call` to class `C`.
4. Call `starknet_call` targeting contract `B`'s entry point at `block_id = 0` (before `C` is declared).
5. `ExecutionStateReader` is created with `state_number = right_after_block(0)` and `class_manager_handle = Some(...)`.
6. During execution, the `library_call` triggers `get_compiled_class(0xC1a55)`.
7. `class_manager_client.get_executable(0xC1a55)` returns `ContractClass::V0(...)` (class was added to the manager when block 1 was synced).
8. **No existence check is performed.** The class is returned successfully.
9. The `library_call` executes and `starknet_call` returns a successful result.
10. **Expected**: `starknet_call` should return an execution error with `UndeclaredClassHash(0xC1a55)` because the class was not declared at block 0. [5](#0-4)

    async fn is_class_declared_at(
        &self,
        block_number: BlockNumber,
        class_hash: ClassHash,
    ) -> StateSyncResult<bool> {
        if self.is_cairo_1_class_declared_at(block_number, class_hash).await? {
            return Ok(true);
        }

        let storage_reader = self.storage_reader.clone();
        // TODO(noamsp): Add unit testing for cairo0
        let deprecated_class_definition_block_number_opt = storage_reader
            .begin_ro_txn()?
            .get_state_reader()?
            .get_deprecated_class_definition_block_number(&class_hash)?;

        Ok(deprecated_class_definition_block_number_opt.is_some_and(
            |deprecated_class_definition_block_number| {
                deprecated_class_definition_block_number <= block_number
            },
        ))
    }
