### Title
Non-Deterministic Proposal Validation via Divergent L1 Gas Price Reference Values in `is_proposal_init_valid` — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid`, called on every validator node during consensus proposal validation, queries the local L1 gas price provider via `get_l1_prices_in_fri_and_wei`. When the provider or oracle fails, the function silently falls back to either the previous block's prices or configured minimum defaults — **without returning an error**. Because different validator nodes may be at different states of their local L1 gas price provider (fresh data vs. stale vs. defaults), they compute different reference values for the margin check. The same `ProposalInit` can therefore be accepted by some validators and rejected by others, breaking the determinism invariant of proposal validation.

---

**Call path:**
`validate_proposal` → `is_proposal_init_valid` → `get_l1_prices_in_fri_and_wei` → `get_l1_prices_in_fri_and_wei_and_conversion_rate`

Inside `get_l1_prices_in_fri_and_wei_and_conversion_rate` (`crates/apollo_consensus_orchestrator/src/utils.rs`, lines 136–223), two async calls are made concurrently:

```rust

```

If either call fails (e.g., `StaleL1GasPricesError`, `MissingDataError`, `NotInitializedError`, or any `ExchangeRateOracleClientError`), the function **does not propagate the error**. Instead it silently falls through a three-tier fallback:

1. **Live data** — both calls succeed → use transformed live prices.
2. **Previous block** — either call fails → reuse `previous_proposal_init` L1 prices.
3. **Configured minimums** — no previous block available → use `min_l1_gas_price_wei` / `min_l1_data_gas_price_wei` with `DEFAULT_ETH_TO_FRI_RATE` (10²¹).

The caller `is_proposal_init_valid` receives whatever reference value was computed and immediately uses it for the margin check across all four L1 gas price fields (`l1_gas_price_fri`, `l1_data_gas_price_fri`, `l1_gas_price_wei`, `l1_data_gas_price_wei`):

```rust
    init_proposed.timestamp,
    proposal_init_validation.previous_proposal_init.as_ref(),

.await;

    && within_margin(l1_data_gas_price_fri_proposed, l1_data_gas_price_fri, ...)
    && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, ...)
    && within_margin(l1_data_gas_price_wei_proposed, l1_data_gas_price_wei, ...))

    return Err(ValidateProposalError::InvalidProposalInit(...));
}
```

Because `get_l1_prices_in_fri_and_wei` never returns an error, `is_proposal_init_valid` always completes — but with a reference value that is **node-local and non-deterministic**. Two validators running the same proposal at the same time can compute entirely different reference prices depending on whether their L1 gas price provider returned fresh data, stale data, or fell all the way to the configured minimum floor.

---

### Impact Explanation

**Broken invariant:** All validators must reach the same accept/reject decision for the same `ProposalInit`. The L1 gas price margin check violates this because the reference value is computed locally from a fallible, node-specific oracle.

**Liveness (primary):** A validator whose oracle is stale falls back to the previous block's prices. If the actual L1 gas price has moved significantly since the previous block (a common occurrence during Ethereum congestion), the proposer's prices — which are within margin of the live reference — will be outside margin of the stale reference. Those validators emit `InvalidProposalInit` and withhold their vote. If enough validators are in this state, the proposal fails to reach the 2/3+ threshold, the round times out, and the chain stalls.

**Fee integrity (secondary):** If a majority of validators fall back to the configured minimum floor (e.g., at startup or after a prolonged oracle outage), a proposer can commit a block with L1 gas prices that are within margin of the floor but far below the actual L1 cost. The block is committed with those artificially low prices, causing all fee calculations for that block to use incorrect L1 gas prices — a direct economic impact on fee accounting.

---

### Likelihood Explanation

`StaleL1GasPricesError` fires whenever the L1 gas price provider has not received new data for `max_time_gap_seconds`. This can happen during:
- Periods of high Ethereum activity where the L1 scraper falls behind.
- Transient network partitions between the sequencer node and its Ethereum RPC endpoint.
- Node restarts before the provider re-initializes.

The divergence between nodes is worst when L1 gas prices change rapidly (e.g., a 2× spike during an Ethereum congestion event). With a margin of ~10% (as implied by the test `rejects_proposal_init_l1_gas_price_out_of_margin` which uses a 10× multiplier to exceed it), a 2× price change is sufficient to push a proposer's valid price outside the stale-fallback reference on affected validators.

---

### Recommendation

The reference value used in `is_proposal_init_valid` should be deterministic across all validators for the same proposal. The safest fix is to anchor the reference to the **previous block's committed L1 gas prices** (already available in `proposal_init_validation.previous_proposal_init`) rather than querying the local oracle at validation time. The oracle query is appropriate for the **proposer** (who sets the prices), but the **validator** should check the proposed prices against a consensus-agreed anchor, not a locally-computed one. Alternatively, if the oracle is unavailable, the validator should skip the L1 gas price check rather than silently substituting a potentially divergent fallback value.

---

### Proof of Concept

1. L1 gas price spikes: previous block committed `l1_gas_price_wei = 100 gwei`; live price is now `200 gwei`.
2. **Node A** (fresh oracle data): `get_l1_prices_in_fri_and_wei` returns `reference = 200 gwei`.
3. **Node B** (oracle returns `StaleL1GasPricesError`): falls back to previous block → `reference = 100 gwei`.
4. Proposer (with fresh data) sets `l1_gas_price_wei = 190 gwei` (within 10% of 200 gwei).
5. **Node A**: `within_margin(190, 200, 10)` → `|190−200| = 10 ≤ 20` → **ACCEPT**.
6. **Node B**: `within_margin(190, 100, 10)` → `|190−100| = 90 > 10` → **REJECT** (`InvalidProposalInit`).
7. If a sufficient fraction of validators are in Node B's state, the proposal fails to reach 2/3+ votes, the round times out, and the chain stalls — triggered by a natural oracle lag, no malicious actor required. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

```rust
/// Returns whether `proposed` is within `margin_percent` of the locally-trusted `reference`,
/// i.e. within the symmetric band `[reference*(1-m), reference*(1+m)]`.
///
/// The band is anchored to `reference` (the node's own L1 oracle read), not to the
/// proposer-supplied `proposed`: anchoring to `proposed` would let a malicious proposer scale the
/// band width with its own input and widen it in its favor.
fn within_margin(proposed: GasPrice, reference: GasPrice, margin_percent: u128) -> bool {
    // For small numbers (e.g., less than 10 wei, if margin is 10%), even an off-by-one
    // error might be bigger than the margin, even if it is just a rounding error.
    // We make an exception for such mismatch, and don't bother checking percentages
    // if the difference in price is only one wei.
    if proposed.0.abs_diff(reference.0) <= GAS_PRICE_ABS_DIFF_MARGIN {
        return true;
    }
    // Saturate: `reference.0 * margin_percent` can overflow u128 on large WEI prices.
    let margin = reference.0.saturating_mul(margin_percent) / 100;
    proposed.0.abs_diff(reference.0) <= margin
}
```

```rust
pub(crate) async fn get_l1_prices_in_fri_and_wei_and_conversion_rate(
    l1_gas_price_provider_client: Arc<dyn L1GasPriceProviderClient>,
    timestamp: u64,
    previous_proposal_init: Option<&PreviousProposalInitInfo>,
    gas_price_params: &GasPriceParams,
) -> (L1PricesInFri, L1PricesInWei, u128) {
    // One of these paths should fill the return values:
    // 1. Both L1 gas price and eth/strk rate are Ok, use those.
    // 2. Otherwise, use previous block info.
    // 3. If that isn't available either, use min gas prices and default eth/strk rate.

    // Get the eth to fri rate from the oracle, and the L1 gas price (in wei) from the provider.
    let (eth_to_fri_rate, price_info) = tokio::join!(
        l1_gas_price_provider_client.get_rate(timestamp),
        l1_gas_price_provider_client.get_price_info(BlockTimestamp(timestamp))
    );
    if price_info.is_err() {
        warn!("Failed to get l1 gas price from provider: {:?}", price_info);
        CONSENSUS_L1_GAS_PRICE_PROVIDER_ERROR.increment(1);
    }
    if eth_to_fri_rate.is_err() {
        warn!("Failed to get eth to fri rate from oracle: {:?}", eth_to_fri_rate);
    }
    if let (Ok(eth_to_fri_rate), Ok(mut price_info)) = (eth_to_fri_rate, price_info) {
        // Both L1 prices and rate are Ok, so we can use them.
        info!(
            "raw eth_to_fri_rate (from oracle): {eth_to_fri_rate}, raw l1 gas price wei (from \
             provider): {price_info:?}"
        );
        apply_fee_transformations(&mut price_info, gas_price_params);
        let prices_in_wei = L1PricesInWei {
            l1_gas_price: price_info.base_fee_per_gas,
            l1_data_gas_price: price_info.blob_fee,
        };
        // Apply the eth/strk rate to get prices in fri.
        let l1_gas_prices_fri_result =
            L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate);
        // If conversion fails, leave return_value=None to try backup methods.
        if let Ok(prices_in_fri) = l1_gas_prices_fri_result {
            return (prices_in_fri, prices_in_wei, eth_to_fri_rate);
        } else {
            warn!(
                "Failed to convert L1 gas prices to FRI: {:?}",
                l1_gas_prices_fri_result.clone().err()
            );
        }
    }

    // One or both (oracle/provider) have failed to fetch, or failure in conversion, so we need to
    // try to use the previous block info.
    if let Some(prev_info) = previous_proposal_init {
        let prev_l1_gas_price_wei = prev_info.l1_prices_wei.clone();
        let prev_l1_gas_price = prev_info.l1_prices_fri.clone();
        // This calculation can fail if gas price is too high, or zero, or if the prices cause the
        // rate to be zero.
        let eth_to_fri_rate = calculate_eth_to_fri_rate(prev_info);
        match eth_to_fri_rate {
            Ok(eth_to_fri_rate) => {
                info!(
                    "Using previous block info: wei prices: {:?}, fri prices: {:?}, eth to fri \
                     rate: {:?}",
                    prev_l1_gas_price_wei, prev_l1_gas_price, eth_to_fri_rate
                );
                return (prev_l1_gas_price, prev_l1_gas_price_wei, eth_to_fri_rate);
            }
            Err(error) => {
                warn!(
                    "Error calculating eth to fri rate from previous block info: {:?}: {:?}",
                    prev_info, error
                );
                // Do not use previous block info. Prefer the default values instead.
            }
        }
    }

    let default_l1_gas_prices_wei = L1PricesInWei {
        l1_gas_price: gas_price_params.min_l1_gas_price_wei,
        l1_data_gas_price: gas_price_params.min_l1_data_gas_price_wei,
    };
    let default_l1_gas_prices_fri =
        L1PricesInFri::convert_from_wei(&default_l1_gas_prices_wei, DEFAULT_ETH_TO_FRI_RATE)
            .expect("Default values should be convertible between wei and fri.");
    info!(
        "Using default values: fri prices: {:?}, wei prices: {:?}, eth to fri rate: {:?}",
        default_l1_gas_prices_fri, default_l1_gas_prices_wei, DEFAULT_ETH_TO_FRI_RATE
    );
```
