Audit Report

## Title
Missing Chainlink Price Feed Staleness Validation Enables Stale-Price rsETH Minting — (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` discards all validity fields from `latestRoundData()`, silently accepting stale or frozen feed prices. Because `LRTOracle.updateRSETHPrice()` is a public, permissionless function, any caller can commit a stale-low LST price to storage as `rsETHPrice`. A depositor who then deposits a non-stale asset (e.g., ETH) receives more rsETH than their deposit warrants, extracting value from existing holders.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` reads only the `price` field and discards `updatedAt`, `answeredInRound`, and `roundId`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol  line 52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No check is made that `answeredInRound >= roundId`, `updatedAt != 0`, `price > 0`, or that `block.timestamp - updatedAt` is within the feed's heartbeat window. By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` performs all three validity checks before returning a price:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol  lines 30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The unguarded oracle is the one used by `LRTOracle` for all L1 LST assets. `LRTOracle.getAssetPrice()` delegates directly to it:

```solidity
// contracts/LRTOracle.sol  line 157
return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
```

That price feeds `_getTotalEthInProtocol()` → `_updateRsETHPrice()`, which writes the new `rsETHPrice` to storage. `updateRSETHPrice()` is public with no access control beyond the pause flag:

```solidity
// contracts/LRTOracle.sol  lines 87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

The minting calculation in `LRTDepositPool.getRsETHAmountToMint()` divides by the stored `rsETHPrice`:

```solidity
// contracts/LRTDepositPool.sol  line 520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

**Exploit arithmetic:** Suppose the protocol holds N stETH and M ETH, rsETH supply is S, true stETH/ETH price is `P_true`, and the stale price is `P_stale < P_true`.

- Stale `rsETHPrice` = `(N·P_stale + M) / S` — lower than true value.
- Attacker deposits X ETH: `rsethAmountToMint = X·1e18 / stale_rsETHPrice`.
- True fair amount = `X·1e18 / true_rsETHPrice`.
- Ratio = `true_rsETHPrice / stale_rsETHPrice = (N·P_true + M) / (N·P_stale + M) > 1`.

The attacker receives more rsETH than their ETH deposit warrants. When the feed recovers and `rsETHPrice` is corrected upward, the attacker's excess rsETH is backed by value transferred from existing holders.

The `pricePercentageLimit` downside-protection guard (lines 270–282 of `LRTOracle.sol`) only triggers a pause when the price drop exceeds the configured threshold. If `pricePercentageLimit == 0` (unconfigured) or the staleness deviation is small (e.g., 0.1–0.5%), the guard does not fire and deposits remain open.

## Impact Explanation
**High — Theft of unclaimed yield / share dilution.** The attacker mints rsETH at a below-true price, then holds excess rsETH after the price corrects. The excess is directly backed by the yield and principal of existing rsETH holders. The magnitude scales linearly with the price deviation and deposit size; a 1% stale deviation on a large deposit is a quantifiable, direct extraction of value.

## Likelihood Explanation
**Medium.** Chainlink LST/ETH feeds have 1-hour heartbeat intervals and deviation thresholds. Staleness windows occur during Ethereum network congestion, Chainlink node outages, and feed migration periods. The attack requires no special role, no governance capture, and no private key. Any depositor can execute it permissionlessly by monitoring feed staleness and calling `updateRSETHPrice()` at the right moment. The attack is repeatable across every staleness event.

## Recommendation
Add staleness and validity guards to `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    if (block.timestamp - updatedAt > MAX_STALENESS) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

`MAX_STALENESS` should be set per-asset based on the Chainlink feed's documented heartbeat interval (e.g., 3600 seconds for a 1-hour heartbeat feed, with a small buffer).

## Proof of Concept
**Foundry fork test outline:**

1. Fork mainnet at a block where a stETH/ETH Chainlink feed round is stale (or mock `latestRoundData()` to return an old `updatedAt` and a price 1% below current).
2. Call `LRTOracle.updateRSETHPrice()` as an unprivileged EOA — succeeds because the function is public.
3. Assert `rsETHPrice` is now set below its true value (compare against a fresh oracle call).
4. Deposit a large amount of ETH via `LRTDepositPool.depositETH()` as the attacker.
5. Assert `rsethAmountToMint > depositAmount * 1e18 / true_rsETHPrice` — attacker received excess rsETH.
6. Advance time / mock feed recovery; call `updateRSETHPrice()` again.
7. Assert `rsETHPrice` is now higher, and the attacker's rsETH balance represents more ETH value than was deposited — confirming dilution of existing holders.

**Minimal call sequence (no fork):**
1. Deploy mock `AggregatorV3Interface` returning `updatedAt = block.timestamp - 2 hours` and `price = 0.99e18`.
2. Set it as the stETH price feed in `ChainlinkPriceOracle`.
3. Call `LRTOracle.updateRSETHPrice()` — no revert.
4. Verify `rsETHPrice` is depressed relative to a fresh-price baseline.
5. Deposit ETH and verify excess rsETH minted.
