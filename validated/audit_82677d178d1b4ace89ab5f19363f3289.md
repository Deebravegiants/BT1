### Title
Unvalidated `ProposalInit.builder` Address Allows Malicious Proposer to Redirect Sequencer Fees and Corrupt Block Header Commitments - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates many `ProposalInit` fields (height, timestamp, gas prices, starknet version, l1_da_mode, fee_proposal) but never checks the `builder` field. That field is passed verbatim as `sequencer_address` into block execution, controlling where transaction fees are sent and what value is committed into the block header. A malicious proposer can set `builder` to any address, redirecting all fees from that block and committing a block with a wrong sequencer address.

### Finding Description

`ProposalInit.builder` is documented as "Address of the one who builds/sequences the block." [1](#0-0) 

The validator's `ProposalInitValidation` struct has no `builder` field: [2](#0-1) 

`is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`, but never touches `builder`: [3](#0-2) 

After passing validation, `initiate_validation` calls `convert_to_sn_api_block_info`, which maps `init.builder` directly to `sequencer_address`: [4](#0-3) 

The same field is used as `SequencerContractAddress` when the block is committed to state sync: [5](#0-4) 

The proposer sets `builder` from its own local config (`args.builder_address`): [6](#0-5) 

The validator has the correct value in `ContextStaticConfig.builder_address` (e.g., `0x1176a1bd84444c89232ec27754698e5d2e7e1a7f1539f12027f28b23ec9f3d8` in production): [7](#0-6) 

But that value is never compared against the incoming `init.builder`.

### Impact Explanation

`sequencer_address` in `BlockInfo` is the recipient of all transaction fees in the block (the OS-level fee transfer calls `block_context.block_info_for_execute.sequencer_address` as the transfer recipient): [8](#0-7) 

A malicious proposer can:
1. **Steal fees**: Set `builder` to an attacker-controlled address; all fees from that block are transferred there.
2. **Corrupt block header**: The committed `BlockHeaderWithoutHash.sequencer` field is wrong, producing an incorrect block hash that propagates into the state commitment and all downstream proofs.
3. **Cause fee-token balance divergence**: The concurrency fee-transfer path asserts `sender_address != sequencer_address`; a crafted `builder` equal to a sender address would panic the node. [9](#0-8) 

This matches the "Incorrect fee/balance with economic impact" (Critical) and "Wrong state/receipt/block commitment" (Critical) impact categories.

### Likelihood Explanation

Any validator in the committee can be elected proposer in a given round. No additional privilege beyond committee membership is required. The attack is a single-field substitution in the `ProposalInit` protobuf message, trivially achievable by any node that controls its own sequencer binary.

### Recommendation

Add `builder_address: ContractAddress` to `ProposalInitValidation` and add an equality check in `is_proposal_init_valid`:

```rust
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

Populate `builder_address` from `self.config.static_config.builder_address` at the call site in `sequencer_consensus_context.rs`, mirroring how `l1_da_mode` and `l2_gas_price_fri` are already propagated into `ProposalInitValidation`. [10](#0-9) 

### Proof of Concept

1. Malicious validator is elected proposer for height H.
2. It constructs `ProposalInit { builder: attacker_address, height: H, ... }` with all other fields correct.
3. Honest validators call `is_proposal_init_valid` — all checks pass because `builder` is never examined.
4. `initiate_validation` → `convert_to_sn_api_block_info` produces `BlockInfo { sequencer_address: attacker_address, ... }`.
5. The batcher executes all transactions in the block with `sequencer_address = attacker_address`; the OS-level fee transfer sends every fee to `attacker_address`.
6. `decision_reached` commits the block; `BlockHeaderWithoutHash.sequencer = SequencerContractAddress(attacker_address)` is stored and hashed into the block commitment.
7. The legitimate sequencer receives zero fees; the attacker receives all fees; the committed block hash is wrong. [11](#0-10) [12](#0-11)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
