### Title
Missing Chainlink Circuit-Breaker (`minAnswer`/`maxAnswer`) Validation Allows Inflated rsETH Minting on LST Depeg - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` and returns the raw answer with no validation against Chainlink's built-in `minAnswer`/`maxAnswer` circuit-breaker bounds. If a supported LST asset depegs below Chainlink's `minAnswer` floor, the feed will silently return `minAnswer` instead of the real (lower) price. The protocol will treat this as a valid price, allowing an attacker to deposit the devalued LST and receive rsETH minted at the inflated circuit-breaker price.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` is the sole price source for all LST assets deposited into the protocol:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
``` [1](#0-0) 

The function performs no check against the aggregator's `minAnswer` or `maxAnswer` circuit-breaker values. Every Chainlink price feed has an on-chain `minAnswer` (accessible via `AggregatorV2V3Interface.minAnswer()`). When the real market price of an asset falls below this floor, Chainlink's aggregator stops updating and instead returns `minAnswer` indefinitely. The protocol has no way to detect this condition.

`LRTOracle.getAssetPrice()` delegates directly to this oracle with no additional validation:

```solidity
function getAssetPrice(address asset) public view onlySupportedOracle(asset) returns (uint256) {
    return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
}
``` [2](#0-1) 

This price is then used in `LRTDepositPool.depositAsset()` (via `_beforeDeposit`) to compute how many rsETH tokens to mint for a depositor: [3](#0-2) 

The `rsETHPrice` used in `_updateRsETHPrice()` is also derived from `_getTotalEthInProtocol()`, which aggregates asset values using the same oracle path, meaning the inflated price propagates into the rsETH/ETH exchange rate as well. [4](#0-3) 

---

### Impact Explanation

**Critical — Direct theft of user funds / Protocol insolvency.**

An attacker who deposits a devalued LST (real price < `minAnswer`) receives rsETH minted at the inflated `minAnswer` price. They can immediately request withdrawal in a different, correctly-priced asset (e.g., ETH), extracting more value than they deposited. This drains the protocol's ETH reserves and socializes the loss across all rsETH holders, constituting direct theft of at-rest user funds and potential protocol insolvency.

---

### Likelihood Explanation

**Low-to-Medium.** This exact scenario occurred with LUNA (UST depeg, May 2022) and has been documented in multiple Chainlink-integrated protocols. The LRT-rsETH protocol supports several LSTs (stETH, rETH, ETHx, sfrxETH, swETH, etc.), each of which carries smart-contract and slashing risk. A severe slashing event or LST protocol exploit could cause a rapid depeg below `minAnswer`. The condition is externally triggered (no admin action required) and the exploit path is permissionless.

---

### Recommendation

Query the aggregator's `minAnswer` and `maxAnswer` at call time and revert if the returned price is at or near either bound:

```solidity
IAccessControlledOffchainAggregator aggregator =
    IAccessControlledOffchainAggregator(priceFeed.aggregator());
int192 minAnswer = aggregator.minAnswer();
int192 maxAnswer = aggregator.maxAnswer();

// Use a gap to catch prices pinned at the circuit-breaker boundary
if (price <= minAnswer || price >= maxAnswer) {
    revert CircuitBreakerTripped();
}
```

Alternatively, integrate a secondary on-chain price source (e.g., a Uniswap TWAP or Curve pool rate) and revert if the two prices diverge beyond a configurable threshold.

---

### Proof of Concept

1. Assume `stETH/ETH` Chainlink feed has `minAnswer = 0.9e18` (90% of ETH).
2. A slashing event causes stETH's real market price to drop to `0.5e18`.
3. Chainlink's aggregator stops updating and returns `0.9e18` for all subsequent calls.
4. Attacker calls `LRTDepositPool.depositAsset(stETH, 1000e18, 0, "")`.
5. `ChainlinkPriceOracle.getAssetPrice(stETH)` returns `0.9e18` (circuit-breaker floor).
6. Protocol mints rsETH worth `900 ETH` for 1000 stETH that is only worth `500 ETH` on the market.
7. Attacker requests withdrawal in ETH, receiving `900 ETH` from the protocol's reserves.
8. Net profit: `400 ETH` stolen from existing rsETH holders. [1](#0-0) [2](#0-1) [3](#0-2)
