Audit Report

## Title
Nested Loop Over `supportedAssets × nodeDelegatorQueue × getQueuedWithdrawals` in `updateRSETHPrice()` Causes Unbounded Gas Consumption — (`contracts/LRTOracle.sol`, `contracts/LRTDepositPool.sol`, `contracts/NodeDelegator.sol`)

## Summary
`updateRSETHPrice()` is a public, permissionless function whose internal call chain traverses three nested loops — supported assets (N), NDCs (M), and EigenLayer queued withdrawals per NDC (K) — with no gas budget check. As the protocol scales through normal admin and operator operations, the function's gas cost grows as O(N × M × K), and `getQueuedWithdrawals` is called redundantly N times per NDC instead of once. When the product N × M × K is large enough, every call to `updateRSETHPrice()` reverts out-of-gas, permanently preventing rsETH price updates.

## Finding Description
The full call chain is confirmed in the codebase:

**Loop 1** — `_getTotalEthInProtocol()` in `LRTOracle.sol` iterates over every supported asset and calls `getTotalAssetDeposits(asset)` for each: [1](#0-0) 

**Loop 2** — `getAssetDistributionData(asset)` in `LRTDepositPool.sol` iterates over every NDC in `nodeDelegatorQueue` and calls `getAssetUnstaking(asset)` on each NDC: [2](#0-1) 

**Loop 3** — `getAssetUnstaking(asset)` in `NodeDelegator.sol` makes an external call to `delegationManager.getQueuedWithdrawals(address(this))` and iterates over every returned withdrawal: [3](#0-2) 

The critical redundancy: because `getAssetDistributionData` is called once per asset (Loop 1), and each call re-enters Loop 2 and calls `getAssetUnstaking` per NDC, `getQueuedWithdrawals(NDC_j)` is invoked **N × M** times total — N times per NDC — even though its result is asset-independent. This multiplies the external call cost by N.

`updateRSETHPrice()` carries only a `whenNotPaused` guard and no role restriction, so any unprivileged caller can trigger it: [4](#0-3) 

`updateRSETHPriceAsManager()` calls the same `_updateRsETHPrice()` and is equally affected: [5](#0-4) 

No gas guard, loop cap, or caching mechanism exists anywhere in the call chain.

## Impact Explanation
**Medium — Unbounded gas consumption / permanent freezing of price updates.**

When N × M × K grows large enough to exceed the block gas limit (~30 M gas on mainnet), every call to `updateRSETHPrice()` reverts out-of-gas. Because this is the only public path to refresh `rsETHPrice`, the stored price becomes permanently stale. Downstream effects include `getRsETHAmountToMint()` mispricing all deposits (uses stale `rsETHPrice`) and protocol fee minting being blocked. This matches the allowed impact "Medium. Unbounded gas consumption."

## Likelihood Explanation
All preconditions are reachable through normal, non-malicious protocol operation:

1. Admin adds supported assets via `addNewSupportedAsset()` — legitimate scaling.
2. Admin adds NDCs via `addNodeDelegatorContractToQueue()` — `maxNodeDelegatorLimit` defaults to 10 but is freely adjustable upward by admin: [6](#0-5) [7](#0-6) 
3. Operator calls `initiateUnstaking()` repeatedly — each call creates one queued withdrawal, bounded by `maxUncompletedWithdrawalCount` in `LRTUnstakingVault`, which is also admin-settable with no hardcoded ceiling: [8](#0-7) [9](#0-8) 

An unprivileged caller then simply calls `updateRSETHPrice()` and the function OOGs. No malicious intent is required from any party.

## Recommendation
1. **Cache `getQueuedWithdrawals` per NDC**: Restructure `getAssetDistributionData` (or introduce a new aggregation function) so that `getQueuedWithdrawals(NDC_j)` is called exactly once per NDC, and the result is filtered across all assets in a single pass — eliminating the N-fold redundancy.
2. **Aggregate all assets in a single NDC loop**: Instead of calling `getAssetDistributionData` once per asset (each re-entering the NDC loop), compute all asset totals in a single pass over NDCs.
3. **Add a gas guard**: Revert with a descriptive error if `gasleft()` falls below a safe threshold before entering the NDC loop.
4. **Document and enforce safe caps**: Set `maxNodeDelegatorLimit` and `maxUncompletedWithdrawalCount` at values that keep worst-case gas within a safe fraction of the block limit, and document those bounds.

## Proof of Concept
Foundry fork test plan:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract PoC_UnboundedGas is Test {
    function test_updateRSETHPrice_OOG() public {
        // 1. Fork mainnet, impersonate admin
        // 2. Add 5 supported assets via addNewSupportedAsset()
        // 3. Increase maxNodeDelegatorLimit to 20, deploy and add 20 NDCs
        // 4. Set maxUncompletedWithdrawalCount to 50 in LRTUnstakingVault
        // 5. Impersonate operator, call initiateUnstaking() 50 times per NDC
        //    (each call queues one withdrawal in EigenLayer's DelegationManager)
        // 6. Call updateRSETHPrice() as unprivileged caller and measure gas
        uint256 gasBefore = gasleft();
        try lrtOracle.updateRSETHPrice() { }
        catch { }
        uint256 gasUsed = gasBefore - gasleft();
        // With N=5, M=20, K=50: 100 external getQueuedWithdrawals calls + 5000 inner iterations
        assertLt(gasUsed, 30_000_000, "Exceeded block gas limit");
    }
}
```

The fuzz target is `(N, M, K)` — assert that `updateRSETHPrice()` does not revert with OOG for any combination reachable under the current admin-configurable caps.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
