### Title
Valid-Paillier Interactive ZK Proof Accepts Prime Modulus, Enabling Full Secret-Share Recovery — (`src/cbmpc/zk/zk_paillier.cpp`, `src/cbmpc/crypto/base_paillier.cpp`)

### Summary

A Byzantine P1 in the ECDSA-2PC DKG can supply a 2048-bit prime N as its Paillier modulus. Every guard in `step4_p2_output` passes, including the `valid_paillier_interactive_t` ZK proof, because for prime N the N-th-root relation `σ^N ≡ ρ (mod N)` collapses to `σ ≡ ρ (mod N)` by Fermat's little theorem, which P1 trivially satisfies by setting `σ_i = ρ_i`. P2 accepts the DKG output. Since `φ(N) = N−1` is publicly computable for prime N, any party can decrypt `c_key` and recover P1's secret share `x1`, yielding the full ECDSA private key.

### Finding Description

**Guard 1 — `is_valid_modulus`** [1](#0-0) 

Only checks `m > 1`, bit-length ≤ 8192, and `m.is_odd()`. A 2048-bit prime satisfies all three.

**Guard 2 — `create_pub`** [2](#0-1) 

Delegates entirely to `is_valid_modulus` plus a bit-size bound. No compositeness check. Returns `SUCCESS` for prime N.

**Guard 3 — `verify_cipher`** [3](#0-2) 

Checks `0 < c_key < N²` and `gcd(c_key, N) = 1`. For prime N every non-zero element is coprime, so P1 can produce a valid-looking `c_key`.

**Guard 4 — `check_integer_with_small_primes`** [4](#0-3) 

Checks divisibility only by primes up to `alpha = 2^13 = 8192`. A 2048-bit prime has no factors ≤ 8192, so this check passes.

**Guard 5 — the N-th-root relation in `valid_paillier_interactive_t::verify`** [5](#0-4) 

The verifier checks `σ_i^N ≡ ρ_i (mod N)` for each `ρ_i` derived deterministically from the verifier's challenge `kV`, `N`, and `prover_pid`. For prime N, Fermat's little theorem gives `a^N ≡ a (mod N)` for all `a`. The check therefore reduces to `σ_i ≡ ρ_i (mod N)`. P1 knows `kV` (received in `step2_p2_to_p1`), `N`, and `prover_pid`, so it can compute every `ρ_i` and set `σ_i = ρ_i`. The check passes unconditionally.

**Guard 6 — `coprime(rho_prod, N)`** [6](#0-5) 

For prime N every non-zero element is coprime, so this passes.

**Full DKG acceptance path** [7](#0-6) 

All six guards pass. `step4_p2_output` returns `SUCCESS` with a prime Paillier modulus installed in P2's key state.

### Impact Explanation

For prime N, `φ(N) = N−1` is publicly computable. Standard Paillier decryption requires only `φ(N)`. Any party (including a passive eavesdropper who observed `c_key` on the wire) can compute `φ(N) = N−1`, derive the decryption exponent, and recover `x1` — P1's secret ECDSA share — from `c_key`. Combined with P2's public share, this yields the full ECDSA private key. Impact is **Critical**: a single Byzantine peer below threshold recovers the joint private key.

### Likelihood Explanation

The attack requires P1 to generate a 2048-bit prime (feasible with standard primality generation) and craft `σ_i = ρ_i` (trivial given knowledge of `kV`, `N`, `prover_pid`). No threshold collusion, no side-channel, no host compromise is needed. The attack is fully deterministic and requires only control of P1's outgoing DKG messages.

### Recommendation

The `valid_paillier_interactive_t` (and `valid_paillier_t`) proof must explicitly reject prime N. The standard fix is to add a Miller-Rabin or Baillie-PSW primality test on N inside the verifier and return an error if N is prime. Alternatively, require a proof of knowledge of the factorization (e.g., a Blum-integer or two-prime-factor proof). The small-factors trial-division check is not a substitute: it only rules out factors ≤ 8192, not primality of a 2048-bit number.

### Proof of Concept

```

  N = a random 2048-bit prime (generated by P1 using bn_t::generate_prime(2048, false))
  x1 = P1's secret share in Z_q

  Send N, c_key, ZK commitments

Step 2 (P2 → P1):
  P2 sends challenge kV (random 128-bit value)

Step 3 (P1 → P2):
  For each i in [0, t):
    rho_i = drbg(hash(kV, N, prover_pid)).gen_bn(N)   [P1 computes this deterministically]
    sigma_i = rho_i                                    [trivially satisfies sigma_i^N ≡ rho_i (mod N)]
  Send sigma array

Step 4 (P2 verifies):
  create_pub(N)                    → SUCCESS (N is odd, 2048 bits)
  verify_cipher(c_key)             → SUCCESS (c_key ∈ (0,N²), coprime with prime N)
  check_integer_with_small_primes  → SUCCESS (prime N has no factors ≤ 8192)
  sigma_i^N mod N == rho_i         → SUCCESS (Fermat: sigma_i^N ≡ sigma_i ≡ rho_i mod N)
  coprime(rho_prod, N)             → SUCCESS (all rho_i coprime with prime N)
  → step4_p2_output returns SUCCESS

Decryption by any party:
  phi_N = N - 1                    [public for prime N]
  Decrypt c_key using phi_N        → recovers x1
  Full ECDSA private key = x1 + x2 (mod q)
```
