Audit Report

## Title
Stale `rsETHPrice` Silently Propagated as Fresh by `RSETHPriceFeed` When Protocol Is Paused — (File: `contracts/oracles/RSETHPriceFeed.sol`)

## Summary

`RSETHPriceFeed.latestRoundData()` computes rsETH/USD by multiplying the live Chainlink ETH/USD answer by `LRTOracle.rsETHPrice()`, but returns only the ETH/USD `updatedAt` timestamp. When `LRTOracle._updateRsETHPrice()` detects a price drop exceeding `pricePercentageLimit`, it pauses the protocol and returns early — leaving `rsETHPrice` frozen at its pre-drop value. Because `updateRSETHPrice()` is gated by `whenNotPaused`, and even `updateRSETHPriceAsManager()` re-triggers the same early-return path while the price remains below the threshold, `rsETHPrice` cannot be refreshed. External lending protocols consuming `RSETHPriceFeed` receive an inflated rsETH/USD price paired with a fresh `updatedAt`, enabling over-borrowing against rsETH collateral.

## Finding Description

**Root cause 1 — `RSETHPriceFeed` surfaces only the ETH/USD staleness timestamp:**

`contracts/oracles/RSETHPriceFeed.sol` lines 63–70:
```solidity
function latestRoundData() external view returns (...) {
    (roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
    answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
}
```
`updatedAt` is taken from the ETH/USD Chainlink feed (refreshed every few minutes). The rsETH/ETH component — `RS_ETH_ORACLE.rsETHPrice()` — is a stored state variable whose own update timestamp is never surfaced or checked.

**Root cause 2 — `updateRSETHPrice()` is blocked when paused:**

`contracts/LRTOracle.sol` lines 87–89:
```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

**Root cause 3 — downside protection pauses and returns without writing the new (lower) price:**

`contracts/LRTOracle.sol` lines 270–282:
```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;   // rsETHPrice is NOT updated — stays at pre-drop value
}
```
`rsETHPrice` is only written at line 313, which is never reached when the early return fires.

**Root cause 4 — `updateRSETHPriceAsManager()` also cannot escape the freeze:**

`contracts/LRTOracle.sol` lines 94–96:
```solidity
function updateRSETHPriceAsManager() external onlyLRTManager {
    _updateRsETHPrice();
}
```
This function bypasses `whenNotPaused`, but `_updateRsETHPrice()` re-evaluates the downside protection check on every call. While the price remains below the threshold, the same `isPriceDecreaseOffLimit` branch fires, calls `_pause()` (a no-op since already paused, per line 320: `if (paused) return;`), and returns early again. `rsETHPrice` remains frozen regardless of who calls the update.

**Exploit sequence:**
1. A significant price drop causes `newRsETHPrice` to fall more than `pricePercentageLimit` below `highestRsethPrice`.
2. `_updateRsETHPrice()` calls `_pause()` and returns. `rsETHPrice` stays at, e.g., `1.05e18` (pre-drop); true price is `0.95e18`.
3. All callers of `updateRSETHPrice()` revert (`whenNotPaused`). `updateRSETHPriceAsManager()` also re-triggers the early return.
4. `RSETHPriceFeed.latestRoundData()` returns:
   - `answer` = `1.05e18 * ETH_USD / 1e18` → inflated by ~10%.
   - `updatedAt` = ETH/USD Chainlink `updatedAt` → appears fresh (e.g., 2 minutes ago).
5. Lending protocol passes its staleness check on `updatedAt`, values rsETH at the inflated price.
6. Attacker deposits 1000 rsETH (true value: 950 ETH), borrows 1050 ETH worth of assets.
7. Attacker defaults; lending protocol holds 950 ETH of collateral against 1050 ETH of debt.

## Impact Explanation

**Critical — Direct theft of user funds.** Lenders in any protocol integrated with `RSETHPriceFeed` (e.g., Aave, Morpho, Compound) can be drained by borrowers who exploit the inflated collateral valuation. The loss is proportional to the price discrepancy and the total rsETH collateral posted during the pause window. This matches the allowed Critical impact: *direct theft of any user funds*.

## Likelihood Explanation

**Medium-High.** The trigger — a price drop exceeding `pricePercentageLimit` — is an explicitly designed circuit-breaker expected to fire during adverse market events (EigenLayer slashing, LST depeg). The `pricePercentageLimit` is configurable; conservative settings (1–2%) can be triggered by routine volatility. The exploitation window lasts until an admin unpauses and the price recovers above the threshold (or `pricePercentageLimit` is adjusted), which may take hours to days. `RSETHPriceFeed` is purpose-built as a Chainlink-compatible feed for external integrations, making consumption by lending protocols highly probable. No special attacker capability is required beyond holding rsETH.

## Recommendation

1. **Write the new (lower) price before pausing.** In `_updateRsETHPrice()`, assign `rsETHPrice = newRsETHPrice` before calling `_pause()` and returning, so the stored value reflects reality even while the protocol is paused.

2. **Surface the rsETH price update timestamp.** Store a `rsETHPriceUpdatedAt` timestamp in `LRTOracle` whenever `rsETHPrice` is written. In `RSETHPriceFeed.latestRoundData()`, return `min(ethUpdatedAt, rsETHPriceUpdatedAt)` as `updatedAt`.

3. **Add a staleness guard in `RSETHPriceFeed`.** Revert if `rsETHPriceUpdatedAt` is older than a configurable threshold (e.g., 24 hours).
