### Title
Pool Singleton `relative_lock_height` Timer Reset via Absorb Spend Traps Farmer in `LEAVING_POOL` State Indefinitely - (File: `chia/pools/pool_puzzles.py`)

### Summary

The pool protocol's `ASSERT_HEIGHT_RELATIVE` exit condition in the waiting-room inner puzzle is measured from the **current singleton coin's `confirmed_block_index`**, not from the block at which the farmer first entered `LEAVING_POOL`. Because an absorb spend (reward claim) while in the waiting room spends the current singleton coin and creates a fresh one, each absorb resets the relative-height clock. A pool operator can therefore keep a farmer trapped in `LEAVING_POOL` indefinitely—and keep collecting their farming rewards—by performing absorb spends just before the lock height expires.

### Finding Description

When a farmer calls `pw_self_pool` / `self_pool`, the singleton transitions to `LEAVING_POOL` (the "waiting room"). The waiting-room inner puzzle (`POOL_WAITINGROOM_INNERPUZ`) enforces exit via `ASSERT_HEIGHT_RELATIVE relative_lock_height`. Under Chia's UTXO model, `ASSERT_HEIGHT_RELATIVE` is evaluated against the `confirmed_block_index` of the **coin being spent** — i.e., the singleton coin that currently exists on-chain.

`create_absorb_spend` in `pool_puzzles.py` supports the waiting-room state explicitly:

```python
elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
    # inner sol is (spend_type, destination_puzhash, pool_reward_amount, pool_reward_height, extra_data)
    inner_sol = Program.to([0, reward_amount, height])
``` [1](#0-0) 

This absorb spend **requires no owner signature** — it is permissionless by design so the pool can claim rewards without farmer cooperation. When it executes, the old singleton coin is destroyed and a new singleton coin (identical puzzle hash, still in `LEAVING_POOL`) is created at the current block height. The next exit attempt must satisfy `ASSERT_HEIGHT_RELATIVE` against this **new** coin's creation height, effectively restarting the countdown.

The wallet-side check in `self_pool` and `new_peak` only guards against the wallet submitting the second travel transaction too early; it does not prevent the on-chain timer from being reset by absorb spends:

```python
if (
    await self.wallet_state_manager.blockchain.get_finished_sync_up_to()
    <= last_height + current_state.current.relative_lock_height
):
    raise ValueError(...)
``` [2](#0-1) 

The lifecycle test confirms absorb spends are valid while in the waiting room:

```python
# ABSORB WHILE IN WAITING ROOM
coin_sols = create_absorb_spend(travel_coinsol, target_pool_state, launcher_coin, 3, ...)
coin_db.update_coin_store_for_spend_bundle(SpendBundle(coin_sols, G2Element()), ...)
``` [3](#0-2) 

And the exit path is confirmed to use `ASSERT_HEIGHT_RELATIVE`:

```python
with pytest.raises(BadSpendBundleError,
    match=re.escape(f"condition validation failure {Err.ASSERT_HEIGHT_RELATIVE_FAILED!s}")):
    coin_db.update_coin_store_for_spend_bundle(SpendBundle([return_coinsol], sig), ...)
``` [4](#0-3) 

### Impact Explanation

While the farmer is trapped in `LEAVING_POOL`, their farming rewards continue flowing to the pool's `target_puzzle_hash` (the pool's address). The farmer cannot transition to `SELF_POOLING` or a different pool. This constitutes:

- **Permanent or long-lived inability** for an honest farmer to execute a valid pool action (`pw_self_pool` / exit from waiting room).
- **Pool membership state corruption** with direct financial impact: the pool continues receiving the farmer's block rewards indefinitely against the farmer's declared intent. [5](#0-4) 

### Likelihood Explanation

The pool operator has a direct financial incentive to trap a farmer who is leaving, especially if the farmer recently won a block. The pool naturally generates coinbase rewards (p2_singleton coins) during normal operation, so triggering absorb spends costs nothing extra. The pool only needs to submit one absorb spend per `relative_lock_height - 1` blocks. With the maximum `relative_lock_height` of 1000 blocks (~6.5 hours), the pool must act roughly once per 6 hours to maintain the trap indefinitely. [6](#0-5) 

### Recommendation

Replace the `ASSERT_HEIGHT_RELATIVE` exit condition with an `ASSERT_HEIGHT_ABSOLUTE` condition anchored to the block at which the singleton **first entered** `LEAVING_POOL`. This height should be embedded in the waiting-room coin's solution or memo at the time of the travel spend and verified by the puzzle, so that absorb spends cannot advance the reference point. Alternatively, prevent absorb spends entirely while in `LEAVING_POOL` state (spend_type=0 disabled in the waiting-room puzzle), forcing the pool to wait until after the farmer exits before claiming any pending rewards.

### Proof of Concept

1. Farmer joins pool with `relative_lock_height = 1000`. Pool begins farming.
2. At block **H**, farmer calls `pw_self_pool`; singleton enters `LEAVING_POOL`. Exit becomes valid at block H + 1000.
3. At block **H + 999**, pool submits an absorb spend (claiming a legitimate coinbase reward). A new singleton coin is created at block H + 999. Exit now requires block H + 999 + 1000 = H + 1999.
4. At block **H + 1998**, pool submits another absorb spend. Exit now requires block H + 2998.
5. Repeat indefinitely. The farmer's singleton remains in `LEAVING_POOL` forever; all farming rewards continue going to the pool.

def create_waiting_room_inner_puzzle(
    target_puzzle_hash: bytes32,
    relative_lock_height: uint32,
    owner_pubkey: G1Element,
    launcher_id: bytes32,
    genesis_challenge: bytes32,
    delay_time: uint64,
    delay_ph: bytes32,
) -> Program:
    pool_reward_prefix = bytes32(genesis_challenge[:16] + b"\x00" * 16)
    p2_singleton_puzzle_hash: bytes32 = launcher_id_to_p2_puzzle_hash(launcher_id, delay_time, delay_ph)
    return POOL_WAITING_ROOM_MOD.curry(
        target_puzzle_hash, p2_singleton_puzzle_hash, bytes(owner_pubkey), pool_reward_prefix, relative_lock_height
    )

def create_absorb_spend(
    last_coin_spend: CoinSpend,
    current_state: PoolState,
    launcher_coin: Coin,
    height: uint32,
    genesis_challenge: bytes32,
    delay_time: uint64,
    delay_ph: bytes32,
) -> list[CoinSpend]:
    inner_puzzle: Program = pool_state_to_inner_puzzle(
        current_state, launcher_coin.name(), genesis_challenge, delay_time, delay_ph
    )
    reward_amount: uint64 = calculate_pool_reward(height)
    if is_pool_member_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol: Program = Program.to([reward_amount, height])
    elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, destination_puzhash, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol = Program.to([0, reward_amount, height])
    else:
        raise ValueError
    # full sol = (parent_info, my_amount, inner_solution)
    coin: Coin | None = get_most_recent_singleton_coin_from_coin_spend(last_coin_spend)
    assert coin is not None

    if coin.parent_coin_info == launcher_coin.name():
        parent_info: Program = Program.to([launcher_coin.parent_coin_info, launcher_coin.amount])
    else:
        p = Program.from_bytes(bytes(last_coin_spend.puzzle_reveal))
        last_coin_spend_inner_puzzle: Program | None = get_inner_puzzle_from_puzzle(p)
        assert last_coin_spend_inner_puzzle is not None
        parent_info = Program.to(
            [
                last_coin_spend.coin.parent_coin_info,
                last_coin_spend_inner_puzzle.get_tree_hash(),
                last_coin_spend.coin.amount,
            ]
        )
    full_solution: SerializedProgram = SerializedProgram.to([parent_info, last_coin_spend.coin.amount, inner_sol])
    full_puzzle: SerializedProgram = create_full_puzzle(inner_puzzle, launcher_coin.name()).to_serialized()
    assert coin.puzzle_hash == full_puzzle.get_tree_hash()

    reward_parent: bytes32 = pool_parent_id(height, genesis_challenge)
    p2_singleton_puzzle = create_p2_singleton_puzzle(
        SINGLETON_MOD_HASH, launcher_coin.name(), delay_time, delay_ph
    ).to_serialized()
    reward_coin: Coin = Coin(reward_parent, p2_singleton_puzzle.get_tree_hash(), reward_amount)
    p2_singleton_solution = SerializedProgram.to([inner_puzzle.get_tree_hash(), reward_coin.name()])
    assert p2_singleton_puzzle.get_tree_hash() == reward_coin.puzzle_hash
    assert full_puzzle.get_tree_hash() == coin.puzzle_hash
    assert get_inner_puzzle_from_puzzle(Program.from_bytes(bytes(full_puzzle))) is not None

    coin_spends = [
        CoinSpend(coin, full_puzzle, full_solution),
        CoinSpend(reward_coin, p2_singleton_puzzle, p2_singleton_solution),
    ]
    return coin_spends
```
