Looking at the exact code paths involved:

### Title
Negative `m` in `RequireMOf` bypasses signature threshold — (`eras/shelley/impl/src/Cardano/Ledger/Shelley/Scripts.hs`)

---

### Summary

The `DecCBOR` instance for `MultiSigRaw` decodes the `m` field of `MultiSigMOf` as a plain `Int` with no non-negativity guard. Because `isValidMOf` returns `True` whenever `n <= 0`, a CBOR-crafted `RequireMOf (-1) []` script validates unconditionally with zero witnesses, violating the formal spec invariant that `m ∈ ℕ`.

---

### Finding Description

**Root cause 1 — unchecked CBOR decode of `m`:**

In the `DecCBOR (Annotator (MultiSigRaw era))` instance, tag `3` decodes `m` via `decCBOR :: Decoder s Int`, which maps directly to `decodeInt`. CBOR natively encodes negative integers (major type 1), and `decodeInt` accepts them without restriction. No `when (m < 0) $ fail ...` guard exists. [1](#0-0) 

**Root cause 2 — `isValidMOf` base case:**

```haskell
isValidMOf n StrictSeq.Empty = n <= 0
```

With `n = -1` and an empty child sequence, `-1 <= 0` is `True`, so the script immediately validates. [2](#0-1) 

**Root cause 3 — `MultiSigMOf` field type is `!Int`, not `!Natural`:** [3](#0-2) 

**Formal spec deviation:**

The Shelley formal spec defines `RequireMOf m ts` with `m ∈ ℕ` (natural numbers). The same constraint appears in the Allegra/Timelock decoder, which has the identical `isValidMOf` pattern. [4](#0-3) 

**UTXOW validation path — why the transaction is accepted:**

`validateFailedNativeScripts` calls `validateNativeScript tx` on every provided native script. `validateNativeScript` calls `evalMultiSig` → `isValidMOf (-1) StrictSeq.Empty = True`. The script is not added to `failedScripts`, so `ScriptWitnessNotValidatingUTXOW` is never raised. [5](#0-4) 

No key witnesses are required for a pure script-locked UTxO, so `validateNeededWitnesses` also passes with an empty `addrTxWitsL`.

The same bug exists in the Allegra `Timelock` decoder and `evalTimelock`: [6](#0-5) 

---

### Impact Explanation

An attacker who is a script author can craft a `RequireMOf (-1) []` script, compute its hash, and present it to a counterparty as a legitimate `m`-of-`n` multisig. Any UTxO locked at that script address can be spent by the attacker alone with zero key witnesses. This fits the **Medium** impact scope: attacker-controlled serialized scripts exceed intended validation limits (the intended invariant is that `m` valid witnesses are required; with `m = -1`, zero are required).

---

### Likelihood Explanation

Exploitation requires the attacker to be a script author who convinces a victim to lock funds at the malformed script address (e.g., in a multisig setup where the attacker proposes the script). The victim must fail to independently decode and verify the script. This is a realistic but non-trivial social-engineering step. The attacker cannot retroactively affect existing UTxOs locked by legitimately constructed scripts.

---

### Recommendation

1. Change `MultiSigMOf !Int` to `MultiSigMOf !Natural` in `MultiSigRaw`, or add an explicit non-negativity check in the `DecCBOR` instance:
   ```haskell
   3 -> do
     m <- decCBOR
     when (m < 0) $ fail "RequireMOf: negative threshold"
     ...
   ```
2. Apply the same fix to the `Timelock` / `TimelockRaw` decoder in `eras/allegra`.
3. Add a property test asserting that any deserialized `MultiSigMOf n _` has `n >= 0`.

---

### Proof of Concept

```
-- CBOR for: array(3) [ 3, -1, array(0)[] ]
-- hex: 83 03 20 80
-- tag=3 (RequireMOf), m=0x20 (CBOR negative: -(0+1)=-1), children=[]

evalMultiSig Set.empty (RequireMOf (-1) StrictSeq.Empty)
-- => isValidMOf (-1) StrictSeq.Empty
-- => (-1) <= 0
-- => True   ← script validates with zero witnesses
```

Submit a transaction spending a UTxO locked by this script hash with `addrTxWitsL = Set.empty` and `scriptTxWitsL = Map.singleton scriptHash malformedScript`. Assert the transaction is accepted (it will be, incorrectly). [7](#0-6) [8](#0-7)

evalMultiSig vhks = go
  where
    -- The important part of this validator is that it will stop as soon as it reaches the
    -- required number of valid scripts
    isValidMOf n StrictSeq.Empty = n <= 0
    isValidMOf n (msig StrictSeq.:<| msigs) =
      n <= 0 || if go msig then isValidMOf (n - 1) msigs else isValidMOf n msigs
    go = \case
      RequireSignature hk -> Set.member hk vhks
      RequireAllOf msigs -> all go msigs
      RequireAnyOf msigs -> any go msigs
      RequireMOf m msigs -> isValidMOf m msigs
      _ -> error "Impossible: All NativeScripts should have been accounted for"
