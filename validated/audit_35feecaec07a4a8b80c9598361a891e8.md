### Title
Sub-Transaction Reference Script Fees Excluded from Minimum Fee Calculation - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

In the Dijkstra era, the `getMinFeeTxUtxo` instance for `DijkstraEra` delegates to `getConwayMinFeeTxUtxo`, which only counts reference scripts belonging to the **top-level** transaction when computing the tiered reference-script fee surcharge. Reference scripts used by **sub-transactions** (`SubTx`) embedded in the same batch are silently excluded. A dedicated helper `batchNonDistinctRefScriptsSize` that correctly aggregates reference-script sizes across the entire batch exists in the same file but is never wired into the fee-validation path.

---

### Finding Description

The Dijkstra era introduces nested ("sub") transactions. A top-level `DijkstraTxBodyRaw TopTx` carries an `OMap` of sub-transactions (`dtbrSubTransactions`). Sub-transactions have their own `referenceInputsTxBodyL` and `inputsTxBodyL` fields, and those inputs may resolve to UTxO entries that carry reference scripts.

The minimum-fee check in `dijkstraUtxoTransition` is:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Utxo.hs:373
runTest $ Shelley.validateFeeTooSmallUTxO pp tx originalUtxo
```

`validateFeeTooSmallUTxO` calls `getMinFeeTxUtxo pp tx utxo`. For `DijkstraEra` this resolves to:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs:141
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
```

`getConwayMinFeeTxUtxo` is:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs:174-175
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
```

`txNonDistinctRefScriptsSize` only unions the **top-level** body's `inputsTxBodyL` and `referenceInputsTxBodyL`:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs:183-187
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
```

Sub-transactions are never visited. The correct batch-aware helper is defined in the same file but unused in the fee path:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs:264-277
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
```

The `SUBUTXO` rule that processes each sub-transaction explicitly calls `updateUTxOStateNoFees` and performs no fee check of its own:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxo.hs:267-268
newState <-
  Shelley.updateUTxOStateNoFees ...
```

The sub-transaction body (`DijkstraSubTxBodyRaw`) has no `fee` field at all:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/TxBody.hs:189-209
DijkstraSubTxBodyRaw ::
  { dstbrSpendInputs    :: !(Set TxIn)
  , dstbrReferenceInputs :: !(Set TxIn)
  ...
  -- no fee field
  } -> DijkstraTxBodyRaw SubTx era
```

The design intent is that the single top-level fee covers the whole batch. However, the minimum-fee formula only charges for reference scripts in the top-level inputs, so the reference-script surcharge for sub-transaction inputs is zero regardless of how many or how large those scripts are.

---

### Impact Explanation

The tiered reference-script fee (`tierRefScriptFee`) was introduced specifically to deter DDoS attacks via large, cheap-to-execute but expensive-to-deserialize scripts (see `docs/adr/2024-08-14_009-refscripts-fee-change.md`). By embedding sub-transactions that reference large scripts, an attacker can force validators to deserialize arbitrarily large reference scripts while paying only the top-level transaction's (potentially zero) reference-script surcharge. This modifies the effective fee paid outside the design parameters of the protocol, matching the **Medium** impact class: *attacker-controlled transactions modify fees outside design parameters*.

---

### Likelihood Explanation

Any unprivileged transaction submitter can craft a Dijkstra top-level transaction with one or more sub-transactions whose `referenceInputsTxBodyL` point to UTxO entries carrying large Plutus scripts. No special role, key, or governance threshold is required. The Dijkstra era is the current tip of the era chain, so the attack surface is live.

---

### Recommendation

Replace the `getMinFeeTxUtxo` instance for `DijkstraEra` with a version that uses `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs
getDijkstraMinFeeTxUtxo :: (EraTx era, BabbageEraTxBody era, DijkstraEraTxBody era)
  => PParams era -> Tx TopTx era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx

instance EraUTxO DijkstraEra where
  ...
  getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo
```

Additionally, verify whether the per-transaction and per-block reference-script size caps (`ConwayTxRefScriptsSizeTooBig`) should also be applied to the aggregate batch size, since the `SUBLEDGER` rule currently treats that failure as `error "Impossible"`.

---

### Proof of Concept

1. Publish several large Plutus V3 scripts as reference scripts in UTxO entries (e.g., 50 KiB each).
2. Construct a Dijkstra top-level transaction whose body contains N sub-transactions, each with `referenceInputsTxBodyL` pointing to those large-script UTxO entries.
3. Set the top-level `feeTxBodyL` to the minimum fee computed by `getConwayMinFeeTxUtxo` (which ignores sub-transaction reference scripts).
4. Submit the transaction. `validateFeeTooSmallUTxO` passes because it calls `getConwayMinFeeTxUtxo`, which returns a fee that does not include the sub-transaction reference-script surcharge.
5. Validators must deserialize all referenced scripts across all sub-transactions, incurring O(N × script_size) deserialization work, while the submitter paid only the top-level reference-script fee (potentially zero if the top-level body references no scripts itself). -- | Calculate the total size of reference scripts used by the transactions. Duplicate
-- scripts will be counted as many times as they occur, since there is never a reason to
-- include an input with the same reference script.

-- only used once in this computation.
