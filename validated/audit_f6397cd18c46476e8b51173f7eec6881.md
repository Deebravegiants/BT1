### Title
Consensus Vote Signature Never Verified — Forged Votes from Any Validator Identity Accepted — (`File: crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

The Apollo consensus layer accepts prevote and precommit votes from peers without ever verifying the cryptographic signature attached to each `Vote`. The only admission check is that `vote.voter` names a known committee member. Because votes are also emitted with a blank `RawSignature::default()`, any connected peer can forge votes attributed to any validator, manufacture a quorum, and drive the node to a `DecisionReached` outcome for an arbitrary `ProposalCommitment`.

### Finding Description

**Signing side — votes are emitted unsigned.**

`StateMachine::make_self_vote` constructs every outgoing prevote and precommit with an explicit placeholder:

```rust
// TODO(Asmaa): sign the vote
signature: RawSignature::default(),
``` [1](#0-0) 

**Verification side — the signature field is never checked.**

`SingleHeightConsensus::handle_vote` is the sole entry point for inbound votes. It checks height and committee membership, then immediately feeds the vote into the state machine. The only comment about the signature is another deferred TODO:

```rust
// TODO(Asmaa): verify the signature
``` [2](#0-1) 

The manager-level `handle_vote` performs no signature check either; it only dispatches to `shc.handle_vote`: [3](#0-2) 

**Admission gate is purely identity-based.**

The only guard against a forged vote is:

```rust
if !self.committee.members().iter().any(|s| s.address == vote.voter) {
    debug!("Ignoring vote from non validator: vote={:?}", vote);
    return VecDeque::new();
}
``` [4](#0-3) 

`vote.voter` is a plain `ContractAddress` field in the protobuf-decoded `Vote` struct — it is attacker-controlled. Any peer that knows the committee composition (public information) can set `vote.voter` to any validator address and have the vote accepted.

**Quorum and decision path.**

Once accepted, the vote is inserted into `self.prevotes` or `self.precommits` with the full stake weight of the impersonated validator. `upon_decision` fires when the precommit map accumulates enough weight: [5](#0-4) 

The resulting `Decision` is returned to the manager and propagated to `context.decision_reached`, committing the attacker-chosen `ProposalCommitment` as the decided block.

### Impact Explanation

A single malicious peer connected to the gossip network can:

1. Enumerate the current committee (public on-chain data).
2. Craft `Vote` messages with `vote.voter` set to each high-weight validator, `vote.proposal_commitment` set to any target hash, and `vote.signature` left as the default empty value.
3. Broadcast these forged votes. Each passes the committee-membership check and is counted at full stake weight.
4. Drive the local node to `DecisionReached` for the attacker-chosen commitment, causing the node to commit a wrong block hash, wrong state root, wrong receipt/event commitment, and wrong transaction set.

This matches the **Critical** impact category: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input*, and also *Invalid or unauthorized Starknet transaction accepted through account validation, signature, nonce, chain id, fee/resource bound, paymaster, or account-deployment logic* (the decided block may contain transactions the honest network never agreed to).

### Likelihood Explanation

The attack requires only network connectivity to a target node and knowledge of the current validator committee (both trivially available). No privileged access, no special transaction, no resource exhaustion. The forged votes are structurally valid protobuf messages and pass every existing filter. Likelihood is **High**.

### Recommendation

1. **Sign every outgoing vote.** Replace `RawSignature::default()` in `make_self_vote` with a real ECDSA signature over a canonical digest of `(vote_type, height, round, proposal_commitment)` using the node's consensus private key via `SignatureManager::sign_precommit_vote` (or a new `sign_vote` variant).

2. **Verify every incoming vote.** In `SingleHeightConsensus::handle_vote`, after confirming committee membership, look up the validator's public key and call `verify_precommit_vote_signature` (or the equivalent vote-specific verifier). Reject votes that fail verification and call `report_peer`.

3. **Bind the digest to height, round, and vote type** to prevent cross-round or cross-type replay of a legitimately signed vote.

### Proof of Concept

```
// Attacker knows committee = [ValidatorA (weight=4000), ValidatorB (weight=3000), ...]
// Target node needs 2/3 of 10000 = 6667 weight to decide.

// Forge a precommit from ValidatorA:
let forged_precommit_a = Vote {
    vote_type: VoteType::Precommit,
    height: current_height,
    round: 0,
    proposal_commitment: Some(attacker_chosen_commitment),
    voter: ValidatorA_address,   // attacker-controlled field
    signature: RawSignature::default(),  // empty — never checked
};

// Forge a precommit from ValidatorB:
let forged_precommit_b = Vote {
    vote_type: VoteType::Precommit,
    height: current_height,
    round: 0,
    proposal_commitment: Some(attacker_chosen_commitment),
    voter: ValidatorB_address,
    signature: RawSignature::default(),
};

// Broadcast both over the gossip channel.
// handle_vote passes committee check (voter is a known address).
// Signature is never verified.
// Combined weight = 7000 > 6667 → upon_decision fires → DecisionReached(attacker_chosen_commitment).
```

The node commits `attacker_chosen_commitment` as the decided block, diverging from the honest network's state.
