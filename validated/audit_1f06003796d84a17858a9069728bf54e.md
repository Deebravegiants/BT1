Audit Report

## Title
`SfrxETHPriceOracle.getAssetPrice()` Returns sfrxETH/frxETH Rate Instead of sfrxETH/ETH Rate, Mispricing the Asset - (File: contracts/oracles/SfrxETHPriceOracle.sol)

## Summary

`SfrxETHPriceOracle.getAssetPrice()` calls `ISfrxETH.pricePerShare()`, which returns the sfrxETH→frxETH exchange rate, not the sfrxETH→ETH rate. The oracle's own interface comment confirms this: `"How much frxETH is 1E18 sfrxETH worth."` Because frxETH persistently trades at a discount to ETH on secondary markets, the oracle structurally overstates the ETH value of sfrxETH, causing sfrxETH depositors to receive excess rsETH and diluting all existing rsETH holders on every deposit.

## Finding Description

`SfrxETHPriceOracle` is the registered price oracle for sfrxETH in `LRTOracle`. Its `getAssetPrice()` implementation at line 40 of `contracts/oracles/SfrxETHPriceOracle.sol` returns `ISfrxETH(sfrxETHContractAddress).pricePerShare()` directly. The interface definition at lines 8–11 of the same file states: `"How much frxETH is 1E18 sfrxETH worth."` This confirms the return value is denominated in frxETH, not ETH.

sfrxETH (0xac3E018457B222d93114458476f3E3416Abbe38F) is an ERC4626 vault whose underlying asset is frxETH, Frax's ETH-pegged liquid staking token. `pricePerShare()` is the standard ERC4626 share-to-asset conversion, yielding sfrxETH→frxETH. frxETH is not ETH; it trades at a persistent discount (historically 0.05%–2%) on secondary markets.

The exploit path is:
1. A user calls `LRTDepositPool.depositAsset(sfrxETH, amount, 0, "")`.
2. `_beforeDeposit` calls `getRsETHAmountToMint(sfrxETH, amount)` (line 665 of `contracts/LRTDepositPool.sol`).
3. `getRsETHAmountToMint` computes `(amount * lrtOracle.getAssetPrice(sfrxETH)) / lrtOracle.rsETHPrice()` (line 520 of `contracts/LRTDepositPool.sol`).
4. `lrtOracle.getAssetPrice(sfrxETH)` delegates to `SfrxETHPriceOracle.getAssetPrice(sfrxETH)` (line 157 of `contracts/LRTOracle.sol`), which returns `pricePerShare()` — the sfrxETH/frxETH rate, not sfrxETH/ETH.
5. The numerator is overstated by the frxETH/ETH discount, so the depositor receives more rsETH than the ETH value they contributed warrants.

The same inflated price flows into `_getTotalEthInProtocol()` (lines 331–349 of `contracts/LRTOracle.sol`), which sums `totalAssetAmt * getAssetPrice(asset)` for all supported assets, overstating `totalETHInProtocol` and therefore `rsETHPrice`. No existing guard checks whether the returned price is denominated in ETH vs. frxETH.

## Impact Explanation

Every sfrxETH deposit mints slightly more rsETH than the depositor's true ETH contribution warrants, proportionally diluting all existing rsETH holders. The magnitude equals the frxETH/ETH discount (persistent, non-zero, historically 0.05%–2%) applied to the entire sfrxETH TVL. This matches the allowed impact: **Low — Contract fails to deliver promised returns to existing rsETH holders, but does not result in direct fund loss.**

## Likelihood Explanation

Likelihood is **High**. sfrxETH is a listed supported asset with `SfrxETHPriceOracle` deployed and registered. Every call to `depositAsset(sfrxETH, ...)` and every call to `updateRSETHPrice()` exercises the vulnerable code path. No attacker action, special condition, or privileged access is required — the mispricing is structural and continuous.

## Recommendation

`getAssetPrice()` must return the sfrxETH/ETH rate. The correct two-step conversion is:

```
sfrxETH/ETH = (sfrxETH/frxETH) * (frxETH/ETH)
```

The frxETH/ETH rate should be sourced from a Chainlink frxETH/ETH feed or the Curve frxETH/ETH pool's `price_oracle`. The oracle should compose both rates:

```solidity
function getAssetPrice(address asset) external view returns (uint256) {
    if (asset != sfrxETHContractAddress) revert InvalidAsset();
    uint256 sfrxEthToFrxEth = ISfrxETH(sfrxETHContractAddress).pricePerShare();
    uint256 frxEthToEth = IFrxEthOracle(frxEthOracle).getRate(); // frxETH/ETH in 1e18
    return sfrxEthToFrxEth * frxEthToEth / 1e18;
}
```

## Proof of Concept

1. Confirm `SfrxETHPriceOracle` is the registered oracle for sfrxETH in `LRTOracle.assetPriceOracle`.
2. Call `SfrxETHPriceOracle.getAssetPrice(sfrxETH)` — returns `pricePerShare()`, e.g. `1.08e18`.
3. Query the Curve frxETH/ETH pool or Chainlink for the frxETH/ETH rate — returns e.g. `0.998e18`.
4. True sfrxETH/ETH = `1.08e18 * 0.998e18 / 1e18 = 1.07784e18`.
5. Oracle returns `1.08e18`; overstatement = `~0.2%`.
6. A user depositing `1e18` sfrxETH receives `1.08e18 / rsETHPrice` rsETH instead of the correct `1.07784e18 / rsETHPrice`, minting ~0.2% excess rsETH at the expense of all existing holders.
7. Foundry fork test: fork mainnet, call `depositAsset(sfrxETH, 1e18, 0, "")`, compare rsETH minted against the value computed using the two-step conversion; the difference equals the frxETH/ETH discount. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
