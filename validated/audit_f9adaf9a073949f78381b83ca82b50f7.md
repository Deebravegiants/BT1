### Title
Unvalidated HTTP-redirect URL silently overwrites pool config, bypassing HTTPS enforcement and enabling pool reward diversion — (File: `chia/farmer/farmer.py`)

### Summary
`Farmer._pool_get_pool_info()` follows HTTP 301/308 redirects and extracts the final destination URL as `new_pool_url`. `update_pool_state()` then writes this URL directly to the persistent `PoolingShareState` config without any validation — no HTTPS enforcement, no domain check, no user confirmation. The same code path that enforces HTTPS for the original pool URL (lines 600–603) is never applied to the redirect-derived URL. A pool operator can silently redirect a farmer's entire pool communication channel to an arbitrary attacker-controlled endpoint.

### Finding Description

In `_pool_get_pool_info()`, when the pool server responds with a 301 or 308 redirect, the code captures the final URL:

```python
new_pool_url: str | None = None
response_url_str = f"{resp.url}"
if (
    response_url_str != url
    and len(resp.history) > 0
    and all(r.status in {301, 308} for r in resp.history)
):
    new_pool_url = response_url_str.replace("/pool_info", "")
``` [1](#0-0) 

This `new_pool_url` is returned inside `GetPoolInfoResult` and then unconditionally written to the persistent pool config in `update_pool_state()`:

```python
if pool_info_result is not None and pool_info_result.new_pool_url is not None:
    with PoolingShareState.acquire(
        root_path=self._root_path, p2_singleton_puzzle_hash=p2_singleton_puzzle_hash
    ) as editable_pool_config:
        editable_pool_config.pool_url = pool_info_result.new_pool_url
``` [2](#0-1) 

The HTTPS enforcement that guards the original `pool_config.pool_url` on mainnet is applied only before the pool info fetch, not to the redirect destination:

```python
enforce_https = config["full_node"]["selected_network"] == "mainnet"
if enforce_https and not pool_config.pool_url.startswith("https://"):
    self.log.error(f"Pool URLs must be HTTPS on mainnet {pool_config.pool_url}")
    continue
``` [3](#0-2) 

After `pool_url` is overwritten, every subsequent pool operation — `GET /farmer`, `POST /farmer`, `PUT /farmer`, `POST /partial` — is directed to the new unvalidated URL: [4](#0-3) [5](#0-4) 

The `PoolingShareState` config is persisted to disk, so the redirect survives farmer restarts. [6](#0-5) 

### Impact Explanation

**High — Corruption of pool membership state and pool reward diversion.**

1. **Silent pool URL hijack**: A pool operator issues a 301/308 redirect from their HTTPS endpoint to any URL (including `http://` on mainnet, bypassing the HTTPS requirement). The farmer's config is permanently overwritten without notification.
2. **Pool reward diversion**: All partial submissions (`POST /partial`) go to the attacker-controlled server. The attacker does not credit the farmer's points. The farmer loses all pool rewards for as long as the redirect persists.
3. **HTTPS downgrade on mainnet**: The redirect can point to an HTTP endpoint, enabling network-level MITM attacks on pool protocol messages (authentication tokens, partial payloads).
4. **Persistent state corruption**: `PoolingShareState` is written to disk; the corrupted `pool_url` persists across restarts until the farmer manually intervenes.

### Likelihood Explanation

Any pool operator (including one who initially appeared legitimate) can trigger this by configuring their web server to issue a 301/308 redirect. The farmer has no visibility into the URL change — there is no log warning, no user prompt, and no HTTPS check on the redirect destination. The `_periodically_update_pool_state_task` runs every 60 seconds, so the redirect takes effect within one polling cycle. [7](#0-6) 

### Recommendation

Apply the same HTTPS enforcement to `new_pool_url` before writing it to the config:

```python
if pool_info_result is not None and pool_info_result.new_pool_url is not None:
    new_url = pool_info_result.new_pool_url
    if enforce_https and not new_url.startswith("https://"):
        self.log.error(f"Redirect pool URL must be HTTPS on mainnet: {new_url}")
    else:
        with PoolingShareState.acquire(...) as editable_pool_config:
            editable_pool_config.pool_url = new_url
```

Additionally, consider logging a warning whenever the pool URL changes automatically, so farmers are aware of the redirect.

### Proof of Concept

1. Farmer joins `https://pool.example.com` (HTTPS, passes mainnet check).
2. Pool operator configures nginx: `return 308 https://attacker.com/pool_info;` on the `/pool_info` endpoint.
3. On the next `update_pool_state()` cycle (~60 s), `_pool_get_pool_info()` follows the redirect. `resp.url` becomes `https://attacker.com/pool_info`; `new_pool_url` is set to `https://attacker.com`.
4. `editable_pool_config.pool_url = "https://attacker.com"` is written to `pooling_share_state.yaml` with no validation or warning.
5. All subsequent `POST /partial` calls go to `https://attacker.com/partial`. The attacker's server returns `{"new_difficulty": 1}` (valid JSON) but never credits points to the farmer's account.
6. The farmer's `points_acknowledged` stops growing; pool rewards are diverted. The farmer's singleton on-chain is unaffected, but they receive no pool payouts.
