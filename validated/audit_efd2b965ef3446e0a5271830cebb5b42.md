### Title
P2P Peer Can Inject Arbitrary Transaction Hash via Unvalidated `TransactionInBlock.transaction_hash`, Corrupting the `transaction_hash → transaction_index` Storage Mapping — (`crates/apollo_p2p_sync/src/client/transaction.rs`)

### Summary
`TransactionStreamFactory::parse_data_for_block` accepts `transaction_hash` from an untrusted p2p peer's protobuf message verbatim, without recomputing it from the transaction body. The injected hash is written directly into the node's persistent `transaction_hash_to_idx` table, permanently corrupting the hash-to-index mapping for that block.

### Finding Description
In `parse_data_for_block` [1](#0-0) , the deserialized `transaction_hash` from the peer is pushed into `block_body.transaction_hashes` with only a developer TODO noting the missing validation. The protobuf conversion in `TryFrom<protobuf::TransactionInBlock> for (Transaction, TransactionHash)` [2](#0-1)  converts the raw wire `Felt` directly into a `TransactionHash` with no cross-check against the transaction body fields. The resulting `BlockBody` is then passed to `write_to_storage` → `append_body` → `write_transactions`, which at line 622 inserts the peer-supplied hash as the key in `transaction_hash_to_idx_table`. [3](#0-2) 

### Impact Explanation
After a malicious peer sends a `TransactionWithReceipt` with a valid InvokeV3 body but `transaction_hash = Felt::ZERO` (or any arbitrary value):

- `starknet_getTransactionByHash(correct_hash)` returns `None` — the correct hash was never stored.
- `starknet_getTransactionByHash(Felt::ZERO)` returns the transaction body — the injected hash is the stored key.
- The `transaction_hash → transaction_index` mapping is permanently corrupted for that block, affecting all downstream RPC calls that rely on it (`get_transaction_idx_by_hash`, `get_transaction_hash_by_idx`). [4](#0-3) 

This matches the allowed impact: **"High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."**

### Likelihood Explanation
Any unauthenticated p2p peer can trigger this. The p2p sync client sends queries to network peers and processes their responses through `parse_data_for_block`. No authentication, rate-limiting, or hash recomputation guards this path. The developer TODO at line 88 confirms the validation gap is known and unaddressed. [5](#0-4) 

### Recommendation
After deserializing the transaction body from the protobuf, recompute the transaction hash using the chain-specific hash function and compare it against the peer-supplied `transaction_hash` field. Reject (report peer + retry) if they differ. This recomputation should happen inside `parse_data_for_block` before pushing to `block_body.transaction_hashes`, resolving the existing TODO.

### Proof of Concept
1. Craft a `TransactionsResponse` protobuf where `TransactionWithReceipt.transaction` is a valid `InvokeV3` body but `TransactionInBlock.transaction_hash` is set to `Felt::ZERO`.
2. Serve this response from a peer node to a syncing client.
3. `parse_data_for_block` collects `n_transactions` items (matching the stored header's `n_transactions` count) and returns `Ok(Some((block_body, block_number)))` with `block_body.transaction_hashes = [Felt::ZERO]`.
4. `write_to_storage` commits this to storage.
5. Assert: `get_transaction_idx_by_hash(&TransactionHash(Felt::ZERO))` returns `Some(index)`, while `get_transaction_idx_by_hash(&correct_hash)` returns `None`. [6](#0-5)

    fn parse_data_for_block<'a>(
        transactions_response_manager: &'a mut ClientResponsesManager<DataOrFin<FullTransaction>>,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut block_body = BlockBody::default();
            let mut current_transaction_len = 0;
            let target_transaction_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .n_transactions;
            while current_transaction_len < target_transaction_len {
                let maybe_transaction = transactions_response_manager.next().await.ok_or(
                    ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }),
                )?;
                let Some(FullTransaction { transaction, transaction_output, transaction_hash }) =
                    maybe_transaction?.0
                else {
                    if current_transaction_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::NotEnoughTransactions {
                            expected: target_transaction_len,
                            actual: current_transaction_len,
                            block_number: block_number.0,
                        }));
                    }
                };
                block_body.transactions.push(transaction);
                block_body.transaction_outputs.push(transaction_output);
                // TODO(eitan): Validate transaction hash from untrusted sources
                block_body.transaction_hashes.push(transaction_hash);
                current_transaction_len += 1;
            }
            Ok(Some((block_body, block_number)))
        }
        .boxed()
    }

            .try_into()

                DeclareTransaction::V0(DeclareTransactionV0V1::try_from(declare_v0)?),
            ),
