### Title
`l2_gas_price` Mutated Before Block Validation in `try_sync()` Is Not Reverted on Failure - (File: crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs)

---

### Summary

In `SequencerConsensusContext::try_sync()`, `self.l2_gas_price` is permanently overwritten from the sync block's `next_l2_gas_price` **before** the block is validated. If the subsequent validation check fails (block number mismatch or timestamp out of range), the function returns `false` but the mutation to `self.l2_gas_price` is never reverted. This is the direct sequencer analog of the external report's pattern: a state variable is initialized at the start of an operation and never cleared/reset when the operation fails.

---

### Finding Description

In `try_sync()`:

```rust

The mutation at line 1056–1059 happens unconditionally on any successfully fetched block, before the block-number and timestamp checks at lines 1067–1080. If those checks fail, `self.l2_gas_price` is left permanently set to the value from the rejected sync block.

The corrupted `self.l2_gas_price` then flows into two critical downstream paths:

**1. Proposal validation** — `validate_proposal()` and `set_height_and_round()` both construct `ProposalInitValidation` using `self.l2_gas_price` as the expected `l2_gas_price_fri`:

---

### Impact Explanation

**As a validator node:** After a failed `try_sync`, the node's `l2_gas_price` is set to the value from the rejected block. The next incoming proposal from an honest proposer (who computed the correct gas price) will be rejected by `is_proposal_init_valid` because the validator's expected `l2_gas_price_fri` no longer matches. This causes the validator to silently drop valid proposals, stalling consensus for that height.

**As a proposer node:** After a failed `try_sync`, the node builds its next proposal using the corrupted `l2_gas_price` as `l2_gas_price_fri`. All other validators (with the correct gas price) will reject this proposal. The proposer's block is never accepted, and the incorrect gas price is embedded in the outgoing `ProposalInit` stream, which constitutes an authoritative-looking wrong value for any observer.

**Fee/gas accounting:** `self.l2_gas_price` is also written into `BlockHeaderWithoutHash.next_l2_gas_price` via `update_state_sync_with_new_block` at decision time. If a block is eventually committed while `l2_gas_price` is corrupted, the `next_l2_gas_price` stored in the block header — which governs fee calculations for the following block — will be wrong, producing an incorrect fee/gas accounting effect with economic impact. [5](#0-4) 

---

### Likelihood Explanation

`try_sync` is called by the consensus manager whenever consensus fails for a height or the node falls behind. The validation check that can fail has two branches:

1. **`block_number != height`**: The state_sync_client returns a block whose stored `block_number` field does not match the requested height. This can occur due to a bug in the sync layer or a malicious P2P peer.
2. **Timestamp out of range**: The block's timestamp is either before the previous block's timestamp or beyond `now + block_timestamp_window_seconds`. The latter can be triggered by clock skew between nodes — a common operational condition — where the synced block was produced by a node whose clock is ahead of the local clock by more than `block_timestamp_window_seconds`.

Both conditions are reachable in production without privileged access.

---

### Recommendation

Move the `self.l2_gas_price` mutation to **after** the validation check passes, or save the previous value and restore it on the early-return path:

```rust
// Save old value before any mutation
let old_l2_gas_price = self.l2_gas_price;

// ... validation checks ...
if !(block_number == height && ...) {
    self.l2_gas_price = old_l2_gas_price; // revert
    return false;
}

// Only mutate after validation succeeds
self.l2_gas_price = max(
    sync_block.block_header_without_hash.next_l2_gas_price,
    VersionedConstants::latest_constants().min_gas_price,
);
```

The same pattern should be applied to the second early-return path (when `add_sync_block` fails), where `self.l2_gas_price`, `self.fee_proposals_window`, and `self.previous_proposal_init` have all been mutated but the batcher has not accepted the block. [6](#0-5) 

---

### Proof of Concept

1. Node N is a validator at height H. `self.l2_gas_price` is correctly set to `P_correct`.
2. The consensus manager calls `try_sync(H)`.
3. `state_sync_client.get_block(H)` returns a block with `next_l2_gas_price = P_wrong` and a timestamp `T_future > now + block_timestamp_window_seconds` (e.g., due to clock skew from the syncing peer).
4. Line 1056: `self.l2_gas_price` is set to `max(P_wrong, min_gas_price)` = `P_wrong`.
5. Line 1067–1080: timestamp check fails (`T_future > now + window`). Function returns `false`.
6. `self.l2_gas_price` is now permanently `P_wrong`.
7. The honest proposer for height H builds a proposal with `l2_gas_price_fri = P_correct`.
8. Node N calls `validate_proposal`. `ProposalInitValidation.l2_gas_price_fri = P_wrong`.
9. `is_proposal_init_valid` rejects the proposal: `P_correct != P_wrong`.
10. Node N never votes for the valid proposal; consensus for height H stalls on this validator.

    async fn try_sync(&mut self, height: BlockNumber) -> bool {
        let sync_block = match self.deps.state_sync_client.get_block(height).await {
            Err(StateSyncClientError::StateSyncError(StateSyncError::BlockNotFound(_))) => {
                return false;
            }
            Err(e) => {
                error!("Sync returned an error: {e:?}");
                return false;
            }
            Ok(block) => block,
        };
        // May be default for blocks older than 0.14.0, ensure min gas price is met.
        self.l2_gas_price = max(
            sync_block.block_header_without_hash.next_l2_gas_price,
            VersionedConstants::latest_constants().min_gas_price,
        );

        // TODO(Asmaa): validate starknet_version and parent_hash when they are stored.
        let block_number = sync_block.block_header_without_hash.block_number;
        let timestamp = sync_block.block_header_without_hash.timestamp;
        let last_block_timestamp =
            self.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
        let now: u64 = self.deps.clock.unix_now();
        if !(block_number == height
            && timestamp.0 >= last_block_timestamp
            && timestamp.0 <= now + self.config.static_config.block_timestamp_window_seconds)
        {
            warn!(
                "Invalid block info: expected block number {}, got {}, expected timestamp range \
                 [{}, {}], got {}",
                height,
                block_number,
                last_block_timestamp,
                now + self.config.static_config.block_timestamp_window_seconds,
                timestamp.0,
            );
            return false;
        }
        self.record_fee_proposal(height, sync_block.block_header_without_hash.fee_proposal_fri);
        self.previous_proposal_init =
            Some(previous_proposal_init_from_block_header(&sync_block.block_header_without_hash));
        self.interrupt_active_proposal().await;

        info!(
            "Adding sync block to Batcher for height {}",
            sync_block.block_header_without_hash.block_number,
        );
        if let Err(e) = self.deps.batcher.add_sync_block(sync_block).await {
            error!("Failed to add sync block to Batcher: {e:?}");
            return false;
        }

        true
    }
