### Title
Unguarded `receive()` in `LRTUnstakingVault` Allows Artificial TVL Inflation to Trigger Excess Protocol-Fee rsETH Minting, Diluting Existing Holder Yield â€” (File: `contracts/LRTUnstakingVault.sol`)

---

### Summary

Anyone can send ETH directly to `LRTUnstakingVault` via its unguarded `receive()` function. Because `LRTDepositPool.getETHDistributionData()` reads `lrtUnstakingVault.balance` as the raw native balance, this inflates `totalETHInProtocol` in `LRTOracle._updateRsETHPrice()`. The oracle then treats the entire balance increase as protocol yield, computes a fee on it, and mints excess rsETH to the treasury â€” permanently diluting the yield owed to existing rsETH holders.

---

### Finding Description

**Step 1 â€” Unguarded entry point.**

`LRTUnstakingVault.receive()` accepts ETH from any caller with no role check: [1](#0-0) 

**Step 2 â€” Raw balance used as TVL component.**

`LRTDepositPool.getETHDistributionData()` reads the vault's native balance directly: [2](#0-1) 

This value flows into `getTotalAssetDeposits(ETH)` as `assetLyingUnstakingVault`: [3](#0-2) 

**Step 3 â€” Oracle treats balance increase as yield.**

`LRTOracle._updateRsETHPrice()` computes `totalETHInProtocol` via `_getTotalEthInProtocol()`, which calls `getTotalAssetDeposits` for every supported asset: [4](#0-3) 

Any increase over `previousTVL` is classified as a reward, and a protocol fee is extracted from it: [5](#0-4) 

**Step 4 â€” Excess rsETH minted to treasury.**

The fee is converted to rsETH at the new (inflated) price and minted to the treasury: [6](#0-5) 

This increases total rsETH supply without a corresponding increase in per-holder backing, permanently reducing the rsETH price relative to what it would have been without the fee extraction.

---

### Impact Explanation

Existing rsETH holders lose a portion of their entitled yield. Concretely: if the protocol has TVL `T`, rsETH supply `S`, price `T/S`, and an attacker donates `X` ETH with fee rate `f`:

- Without fee: new price = `(T+X)/S` â€” holders capture full `X`.
- With fee: `fee = XÂ·f`, new price â‰ˆ `(T+Xâˆ’fee)/S` â€” holders lose `fee` worth of ETH yield to the treasury.

The treasury receives rsETH worth `fee` ETH, diluting all existing holders. The attack is repeatable up to the `maxFeeMintAmountPerDay` cap per day.

**Impact: High â€” Theft of unclaimed yield.**

---

### Likelihood Explanation

- No role, signature, or approval required â€” any EOA or contract can call `LRTUnstakingVault.receive()`.
- `updateRSETHPrice()` is public and callable by anyone when not paused.
- The only partial mitigant is `pricePercentageLimit` (reverts if price increase exceeds threshold for non-managers) and `maxFeeMintAmountPerDay` (caps daily fee minting). Both limit magnitude but do not prevent the attack.
- The attacker loses `X` ETH to cause `XÂ·f` ETH in yield theft. At a 10% fee rate, the attacker spends 10 ETH to steal 1 ETH of yield â€” economically irrational for profit, but viable as a griefing/competitive attack or if the attacker is short rsETH.

---

### Recommendation

1. **Restrict `receive()` in `LRTUnstakingVault`** to only accept ETH from known protocol contracts (e.g., `LRTDepositPool`, `NodeDelegator`), similar to how `receiveFromLRTDepositPool()` and `receiveFromNodeDelegator()` are named-entry-point functions.
2. **Track deposited ETH via an internal accounting variable** (e.g., `uint256 public ethDeposited`) rather than relying on `address(this).balance` in `getETHDistributionData()`. Increment it only in authorized receive paths. Use this variable instead of raw `.balance` in `getETHDistributionData()`.

// Fork test (Foundry)
function testInflateUnstakingVaultTriggersFee() public {
    // Setup: protocol has 1000 ETH TVL, 1000 rsETH supply, price = 1.0
    uint256 previousPrice = lrtOracle.rsETHPrice(); // 1e18

    // Attacker sends 100 ETH directly to LRTUnstakingVault
    address attacker = makeAddr("attacker");
    vm.deal(attacker, 100 ether);
    vm.prank(attacker);
    (bool ok,) = address(lrtUnstakingVault).call{value: 100 ether}("");
    require(ok);

    // Anyone calls updateRSETHPrice
    lrtOracle.updateRSETHPrice();

    uint256 newPrice = lrtOracle.rsETHPrice();
    uint256 treasuryBalance = rsETH.balanceOf(treasury);

    // Assert treasury received excess rsETH (fee on donated ETH)
    assertGt(treasuryBalance, 0, "treasury should have received fee rsETH");

    // Assert price is lower than it would be without fee extraction
    // Without fee: price = 1100/1000 = 1.1e18
    // With fee (10%): price < 1.1e18
    assertLt(newPrice, 1.1 ether, "price should be diluted by fee");
    assertGt(newPrice, previousPrice, "price still increased");
}
