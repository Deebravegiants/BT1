### Title
Missing Context Binding in TDH2 Partial-Decryption Proof Enables Cross-Ciphertext Replay — (`File: src/cbmpc/crypto/tdh2.cpp`)

### Summary

The Schnorr-like DLEQ proof produced by `private_share_t::decrypt()` hashes only `(Xi, Yi, Zi)` without including the ciphertext base-point `R1` or the public share `Qi`. Because `R1` is not in the challenge hash, a partial-decryption proof generated for one ciphertext is cryptographically valid for any other ciphertext that carries the same `R1`. A malicious combiner who controls the encryptor role can create two ciphertexts sharing the same `R1`, collect honest parties' partial decryptions for the first, and replay them to decrypt the second — obtaining a TDH2 plaintext that the honest share-holders never consented to reveal.

### Finding Description

In `private_share_t::decrypt()` the partial-decryption proof is constructed as:

```cpp
Xi = x * R1;
bn_t si = curve.get_random_value();
ecc_point_t Yi = si * R1;
ecc_point_t Zi = si * G;
ei = ro::hash_number(Xi, Yi, Zi).mod(q);   // ← no R1, no Qi, no label
MODULO(q) fi = si + x * ei;
``` [1](#0-0) 

The verifier in `check_partial_decryption_helper()` reconstructs `Yi` and `Zi` from the supplied `ciphertext.R1` and the caller-supplied `Qi`, then recomputes the same minimal hash:

```cpp
ecc_point_t Yi = fi * R1 - ei * Xi;
ecc_point_t Zi = fi * G  - ei * Qi;
bn_t ei_test = ro::hash_number(Xi, Yi, Zi).mod(q);
``` [2](#0-1) 

Because `R1` is absent from the hash, the tuple `(Xi, ei, fi)` is a valid proof for **every** ciphertext whose `R1` field equals the one used during proof generation. The combiner path in `combine_additive()` first verifies the target ciphertext (so `CT2` must be a well-formed ciphertext), then calls `check_partial_decryption_helper` with `CT2.R1`:

```cpp
if (rv = ciphertext.verify(pub_key, label)) return rv;
...
if (rv = partial_decryption.check_partial_decryption_helper(Qi[rid-1], ciphertext, curve)) return rv;
V += partial_decryption.Xi;
``` [3](#0-2) 

If `CT2.R1 == CT1.R1`, the proof from `CT1` passes verbatim for `CT2`, and `V = Σ Xi = x·R1 = r·Q` correctly decrypts `CT2`.

The public C API entry points are:

- `cbmpc_tdh2_partial_decrypt` → `coinbase::api::tdh2::partial_decrypt` → `private_share_t::decrypt()`
- `cbmpc_tdh2_combine_additive` → `coinbase::api::tdh2::combine_additive` → `combine_additive()` [4](#0-3) [5](#0-4) 

### Impact Explanation

A malicious combiner (who also controls the encryptor role, or who can observe `r` for a ciphertext) can:

1. Encrypt two messages under the same randomness `r`, producing `CT1` and `CT2` with identical `R1 = r·G`.
2. Present `CT1` to the honest share-holders and collect their partial decryptions `{(Xi, ei, fi)}`.
3. Call `combine_additive(pk, Qi, label2, partials_from_CT1, CT2, plaintext)`.
4. The ciphertext verification passes for `CT2` (it is a valid ciphertext). The proof check passes because `R1` is the same. `V = x·R1` is correct for `CT2`, so AES-GCM decryption succeeds.

The honest share-holders never consented to decrypt `CT2`. The attacker obtains the TDH2-protected plaintext of `CT2` without a threshold of honest parties authorizing that specific decryption. This violates the core TDH2 security property and falls squarely in the "single malicious peer obtains a valid TDH2 decryption result without the required honest participants" impact class.

### Likelihood Explanation

The attack requires the adversary to control the encryptor (or to reuse `r` across two ciphertexts). In practice the encryptor and combiner are often the same application-layer entity (the party that wants to decrypt is also the one that encrypted). The public API imposes no restriction on who may call `cbmpc_tdh2_partial_decrypt` with a crafted ciphertext, so the precondition is reachable without any threshold collusion.

### Recommendation

Include `R1` and `Qi` (and optionally the label) in the challenge hash of the partial-decryption proof, matching the pattern used by every other ZK proof in the library (which always hashes all public statement inputs):

```cpp
// In private_share_t::decrypt():
ei = ro::hash_number(Xi, Yi, Zi, R1, pub_key.Q /* or Qi */, label).mod(q);

// In check_partial_decryption_helper():
bn_t ei_test = ro::hash_number(Xi, Yi, Zi, R1, Qi, label).mod(q);
```

This binds each partial-decryption proof to the specific ciphertext and public share for which it was generated, preventing cross-ciphertext replay.

### Proof of Concept

```
Setup:
  - n=2 additive TDH2 shares (x0, x1) with public key Q = (x0+x1)·G
  - Honest parties hold private_share_t{x=x0, rid=1} and {x=x1, rid=2}

Step 1 – Craft two ciphertexts with the same R1:
  r = random scalar
  CT1 = encrypt(plain1, label1, r, s1, iv1)   // R1 = r·G
  CT2 = encrypt(plain2, label2, r, s2, iv2)   // R1 = r·G  (same r)

Step 2 – Collect honest partial decryptions for CT1:
  partial0 = share0.decrypt(CT1, label1)  →  (Xi0, ei0, fi0)
  partial1 = share1.decrypt(CT1, label1)  →  (Xi1, ei1, fi1)

Step 3 – Replay against CT2:
  combine_additive(pk, [Q0,Q1], label2, [partial0, partial1], CT2, out)

Step 4 – Verify:
  check_partial_decryption_helper uses CT2.R1 == CT1.R1
  Yi = fi0·R1 - ei0·Xi0  (same as for CT1, since R1 identical)
  Zi = fi0·G  - ei0·Q0   (same)
  H(Xi0, Yi, Zi) == ei0  ✓  (hash unchanged)
  V = Xi0 + Xi1 = x0·R1 + x1·R1 = x·R1 = r·Q
  AES-GCM decrypts CT2 successfully → out == plain2
```

The honest share-holders only ever saw `CT1`; `CT2` was decrypted without their participation.

cbmpc_error_t cbmpc_tdh2_combine_additive(cmem_t public_key, cmems_t public_shares, cmem_t label,
                                          cmems_t partial_decryptions, cmem_t ciphertext, cmem_t* out_plaintext) {
  try {
    if (!out_plaintext) return E_BADARG;
    *out_plaintext = cmem_t{nullptr, 0};
    const auto vpk = validate_cmem(public_key);
    if (vpk) return vpk;
    const auto vlb = validate_cmem(label);
    if (vlb) return vlb;
    const auto vct = validate_cmem(ciphertext);
    if (vct) return vct;

    std::vector<coinbase::mem_t> pub_shares;
    std::vector<coinbase::mem_t> partials;
    auto rv = view_cmems(public_shares, pub_shares);
    if (rv) return rv;
    rv = view_cmems(partial_decryptions, partials);
    if (rv) return rv;

    coinbase::buf_t plain;
    const coinbase::error_t crv = coinbase::api::tdh2::combine_additive(
        view_cmem(public_key), pub_shares, view_cmem(label), partials, view_cmem(ciphertext), plain);
    if (crv) return crv;

    return alloc_cmem_from_buf(plain, out_plaintext);
  } catch (const std::bad_alloc&) {
    if (out_plaintext) *out_plaintext = cmem_t{nullptr, 0};
    return E_INSUFFICIENT;
  } catch (...) {
    if (out_plaintext) *out_plaintext = cmem_t{nullptr, 0};
    return E_GENERAL;
  }
}
