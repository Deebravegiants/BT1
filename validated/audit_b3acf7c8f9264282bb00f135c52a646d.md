### Title
Unverified Vote Signatures Allow Any Network Peer to Forge Consensus Votes and Bypass Quorum — (`crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`SingleHeightConsensus::handle_vote` accepts incoming votes from the network and checks only that the claimed `vote.voter` address is a member of the committee. It never verifies the cryptographic signature on the vote. A single malicious peer can therefore forge prevotes and precommits claiming to be any committee member, accumulate the full 2/3 weighted quorum from forged messages alone, and force a consensus decision (or a nil-quorum liveness stall) without any legitimate validator having voted.

---

1. Height match.
2. Committee membership: `self.committee.members().iter().any(|s| s.address == vote.voter)`.

### Impact Explanation

A single malicious peer that is connected to the gossip network can:

**Attack A — Force premature decision (safety):**
1. Wait for a legitimate proposal to complete validation (so `self.proposals` has an entry for the round).
2. Craft forged precommits with `voter` set to committee members whose combined weight exceeds the 2/3 Byzantine quorum, including the virtual proposer address.
3. Broadcast them. `handle_vote` accepts each one; the state machine accumulates their weights.
4. `upon_decision` fires, committing the block before any real validator has voted.
5. Other validators that have not yet received the real precommits will not reach the same decision at the same round, causing a consensus split and a wrong committed `ProposalCommitment` / block hash on the targeted node.

**Attack B — Force nil-quorum liveness stall:**
1. Craft forged nil prevotes (`proposal_commitment = None`) from committee members totalling >2/3 weight.
2. The targeted validator's state machine reaches nil-prevote quorum, broadcasts a nil precommit, and advances to the next round — even though the legitimate proposer's block was valid.
3. Repeated across rounds, this permanently stalls the targeted node.

Both attacks require only network connectivity to the target node; no privileged role, no stake, and no knowledge of any private key is needed. The committee member addresses are public (derived from the staking contract).

---

### Likelihood Explanation

The committee is public. The `Vote` struct is a plain protobuf message with a `voter: ContractAddress` field and a `signature: RawSignature` field that is never read on the receiving side. Any peer that can send gossip messages to a validator node can execute this attack. The network is currently a small trusted set, but the code is production-path and the guard is explicitly deferred with a TODO.

---

### Recommendation

Implement the deferred signature verification in `handle_vote` before the committee-membership check:

1. Maintain a `validator_address → PublicKey` map alongside the committee (populated when the committee is fetched from the staking contract).
2. In `handle_vote`, after the height and membership checks, call `verify_precommit_vote_signature` (already implemented in `apollo_signature_manager`) with the vote's `proposal_commitment` hash, the vote's `signature`, and the public key for `vote.voter`. Reject the vote if verification fails or if no public key is registered for that address.
3. Apply the same check to future-height votes before they are replayed into the live-height SHC.

The signing counterpart (`// TODO(Asmaa): sign the vote` in `make_self_vote`) must be completed simultaneously so that honest validators produce verifiable votes.

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }

    fn upon_decision(&mut self, round: u32) -> VecDeque<SMRequest> {
        let Some((Some(proposal_id), _)) = self.proposals.get(&round) else {
            return VecDeque::new();
        };
        if !self.value_has_enough_votes(&self.precommits, round, &Some(*proposal_id), &self.quorum)
        {
            return VecDeque::new();
        }
        if !self.virtual_proposer_in_favor(&self.precommits, round, &Some(*proposal_id)) {
            return VecDeque::new();
        }
        // Collect all supporting precommits for this proposal and round.
        let supporting_precommits: Vec<Vote> = self
            .precommits
            .iter()
            .filter(|(&(r, _voter), (v, _w))| {
                r == round && v.proposal_commitment == Some(*proposal_id)
            })
            .map(|(_vote_key, (v, _w))| v.clone())
            .collect();

        let decision = Decision { precommits: supporting_precommits, block: *proposal_id, round };
        VecDeque::from([SMRequest::DecisionReached(decision)])
    }
