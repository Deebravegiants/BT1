### Title
Arithmetic Underflow in `getTokenBalanceMinusFees` via Rebasing Token Negative Rebase Permanently Freezes All Token Collateral — (`contracts/pools/RSETHPool.sol`)

### Summary
`getTokenBalanceMinusFees` computes `balanceOf(pool) - feeEarnedInToken[token]` with no underflow guard. If a rebasing token is added as a supported token and subsequently undergoes a negative rebase that reduces the pool's balance below the accrued `feeEarnedInToken[token]`, every function that calls `getTokenBalanceMinusFees` — namely `bridgeTokens` and `moveAssetsForBridging` — reverts permanently. No recovery path exists, freezing all token collateral in the pool forever.

### Finding Description
`addSupportedToken` (TIMELOCK_ROLE) places no restriction on token type, so a rebasing token (e.g., stETH or any elastic-supply ERC20) can be legitimately added. [1](#0-0) 

When users deposit the rebasing token, `feeEarnedInToken[token]` accumulates: [2](#0-1) 

A negative rebase reduces `IERC20(token).balanceOf(address(this))` below `feeEarnedInToken[token]`. The subtraction in `getTokenBalanceMinusFees` then reverts with arithmetic underflow (Solidity 0.8 checked arithmetic): [3](#0-2) 

Both bridging exit paths call this function unconditionally: [4](#0-3) [5](#0-4) 

All recovery paths are also blocked:
- `withdrawFees(receiver, token)` attempts `safeTransfer(receiver, feeEarnedInToken[token])` which reverts because the actual balance is insufficient. [6](#0-5) 
- `removeSupportedToken` requires `balanceOf(pool) == 0`, which fails while any balance remains. [7](#0-6) 
- No emergency sweep or admin rescue function exists in the contract. [8](#0-7) 

### Impact Explanation
All token collateral held by the pool — both the principal portion and the fee portion — is permanently frozen with no on-chain recovery mechanism. This matches **Critical: Permanent freezing of funds**.

### Likelihood Explanation
Requires two conditions: (1) a TIMELOCK_ROLE admin adds a rebasing token as a supported token (a legitimate but mistaken admin action, not a key compromise), and (2) the rebasing token undergoes a negative rebase event. Both conditions are plausible in production. The contract provides no documentation or enforcement preventing rebasing tokens from being added.

### Recommendation
1. In `addSupportedToken`, validate that the token is not a rebasing token (e.g., by checking that `totalSupply` and `balanceOf` are stable, or by maintaining an explicit allowlist of non-rebasing tokens).
2. Add a `saturatingSub` or `min(balanceOf, feeEarned)` guard in `getTokenBalanceMinusFees`:
   ```solidity
   function getTokenBalanceMinusFees(address token) public view returns (uint256) {
       uint256 bal = IERC20(token).balanceOf(address(this));
       uint256 fee = feeEarnedInToken[token];
       return bal > fee ? bal - fee : 0;
   }
   ```
3. Add an emergency admin sweep function to recover tokens in degenerate states.

    /// @dev Adds a supported token
    /// @param token The token address
    /// @param oracle The oracle address for the token
    /// @param bridge The bridge address for the token
    function addSupportedToken(address token, address oracle, address bridge) external onlyRole(TIMELOCK_ROLE) {
        UtilLib.checkNonZeroAddress(token);
        UtilLib.checkNonZeroAddress(oracle);
        UtilLib.checkNonZeroAddress(bridge);

        if (supportedTokenOracle[token] != address(0)) {
            revert AlreadySupportedToken();
        }
        if (tokenBridge[token] != address(0)) {
            revert AlreadySupportedToken();
        }
        if (IOracle(oracle).getRate() == 0) {
            revert UnsupportedOracle();
        }
        supportedTokenList.push(token);
        supportedTokenOracle[token] = oracle;
        tokenBridge[token] = bridge;

        emit AddSupportedToken(token, oracle, bridge);
    }

    /// @dev Removes a supported token
    /// @param token The token address
    function removeSupportedToken(address token, uint256 tokenIndex) external onlyRole(TIMELOCK_ROLE) {
        UtilLib.checkNonZeroAddress(token);
        if (supportedTokenList[tokenIndex] != token) revert TokenNotFoundError();
        if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();

        delete supportedTokenOracle[token];
        delete tokenBridge[token];
        supportedTokenList[tokenIndex] = supportedTokenList[supportedTokenList.length - 1];
        supportedTokenList.pop();
        emit RemovedSupportedToken(token);
    }
