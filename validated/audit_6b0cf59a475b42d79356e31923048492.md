### Title
P2P Peer Can Inject Arbitrary `revert_reason` into Stored Transaction Outputs, Corrupting `starknet_getTransactionReceipt` — (`crates/apollo_p2p_sync/src/client/transaction.rs`)

### Summary
`TransactionStreamFactory::parse_data_for_block` accepts `transaction_output` (including `execution_status` and `revert_reason`) from unauthenticated p2p peers and writes it directly to storage with no validation against the `receipt_commitment` already stored in the block header. The RPC handler `get_non_pending_receipt` then reads and serves this peer-supplied value verbatim, so any p2p peer can make `starknet_getTransactionReceipt` return an attacker-controlled `revert_reason` string — or flip a succeeded transaction to appear reverted.

### Finding Description

**Step 1 — Peer supplies fabricated `transaction_output`.**

`parse_data_for_block` receives `FullTransaction { transaction, transaction_output, transaction_hash }` from the network. The only check performed is that the count of received transactions equals `n_transactions` from the stored header. The `transaction_output` field — which carries `execution_status` and, when reverted, an arbitrary `revert_reason: String` — is pushed into `block_body.transaction_outputs` without any content validation. [1](#0-0) 

A TODO in the same function acknowledges that even the transaction hash is not validated from untrusted sources: [2](#0-1) 

**Step 2 — Fabricated output is committed to storage.**

`write_to_storage` calls `append_body` with the peer-supplied `BlockBody`, persisting the fabricated `transaction_output` (including `revert_reason`) to the node's database. [3](#0-2) 

**Step 3 — No cross-check against `receipt_commitment`.**

The block header (synced separately) contains a `receipt_commitment` — a Poseidon Merkle root over all receipt hashes, where each receipt hash commits to the `revert_reason` via `starknet_keccak_hash(revert_reason.as_bytes())`. [4](#0-3) 

The grep search for any validation of `transaction_output` against `receipt_commitment` inside `crates/apollo_p2p_sync` returns **zero matches**. The stored outputs are never verified against this commitment.

**Step 4 — RPC serves the fabricated value.**

`get_non_pending_receipt` reads the stored `transaction_output` directly from the database and returns it in the `starknet_getTransactionReceipt` response, including the peer-supplied `revert_reason`. [5](#0-4) 

The `RevertedTransactionExecutionStatus` struct itself carries a TODO confirming no validation of the string content exists: [6](#0-5) 

### Impact Explanation
Any p2p peer the victim node connects to can:
1. Send `execution_status = Reverted { revert_reason: "<arbitrary string>" }` for a transaction that actually succeeded, making `starknet_getTransactionReceipt` report it as failed with an attacker-chosen message.
2. Replace the true revert reason of a genuinely reverted transaction with arbitrary content.

The corrupted receipt is served authoritatively by the node's RPC to all downstream clients (wallets, explorers, dApps) that query `starknet_getTransactionReceipt` or `starknet_getBlockWithReceipts`.

### Likelihood Explanation
The attacker only needs to be a p2p peer that the victim node queries for transaction data — a low-trust, unauthenticated role. No validator, operator, or privileged access is required. The node actively queries peers for block bodies, so any peer that responds to a transaction query can exploit this.

### Recommendation
After collecting all `transaction_output` values for a block, recompute the `receipt_commitment` from the stored outputs and compare it against the `receipt_commitment` already stored in the block header. Reject and report the peer if the commitments do not match. This mirrors the existing pattern used for `state_diff_length` validation in `StateDiffStreamBuilder`. [7](#0-6)

```rust
if let Some(expected_receipt_commitment) = stored_header.receipt_commitment {
    let receipt_elements: Vec<ReceiptElement> = block_body.transaction_hashes
        .iter()
        .zip(&block_body.transaction_outputs)
        .map(|(hash, output)| ReceiptElement { transaction_hash: *hash, transaction_output: output.into() })
        .collect();
    let actual = calculate_receipt_commitment::<Poseidon>(&receipt_elements);
    if actual != expected_receipt_commitment {
        return Err(ParseDataError::BadPeer(BadPeerError::InvalidReceiptCommitment));
    }
}
```
