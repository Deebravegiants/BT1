### Title
Unbounded `fee_proposal_fri` During Startup Window Bypasses Fee-Market Validation and Corrupts Future `fee_actual` — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

In `is_proposal_init_valid`, the bounds check on `ProposalInit.fee_proposal_fri` is gated on `proposal_init_validation.fee_actual` being `Some`. During the startup window — the first `fee_proposal_window_size` blocks, when the sliding window has not yet accumulated enough entries — `fee_actual` is `None` and the bounds check is silently skipped. A Byzantine proposer can therefore set `fee_proposal_fri` to any value (e.g., `u128::MAX`) during this window. The value passes validation, is committed to the block header, is stored in `fee_proposals_window`, and poisons the median used to compute `fee_actual` for all subsequent blocks, permanently corrupting the L2 gas-price feed.

---

### Finding Description

**Root cause — `is_proposal_init_valid`:**

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

When `fee_actual = None` the entire `if let` body is skipped. No alternative bound (e.g., against `l2_gas_price`) is applied.

**Proposer fallback (not mirrored in validator):**

The honest proposer falls back to `l2_gas_price` when `fee_actual` is unavailable:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [2](#0-1) 

The validator never checks that the received `fee_proposal_fri` equals or is near `l2_gas_price` when `fee_actual = None`.

**Validation path that accepts the proposal:**

`validate_proposal` calls `is_proposal_init_valid` first; if it returns `Ok(())`, `initiate_validation` is called and the batcher begins executing the block. The malicious `fee_proposal_fri` is already accepted at this point. [3](#0-2) 

**Commitment binding — the value is locked in:**

After the batcher finishes, `proposal_commitment_from` hashes `fee_proposal_fri` into the `ProposalCommitment`:

```rust
ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
``` [4](#0-3) 

The commitment is what consensus votes on; once 2f+1 validators sign it, the block — including the malicious `fee_proposal_fri` — is irreversibly committed.

**Downstream corruption — `fee_proposals_window` and `fee_actual`:**

On `decision_reached`, `record_fee_proposal` stores the committed `fee_proposal_fri` in the sliding window: [5](#0-4) 

`compute_fee_actual` then computes the median of that window for every subsequent height: [6](#0-5) 

If `window_size` startup blocks all carry `fee_proposal_fri = u128::MAX`, the median becomes `u128::MAX`. Once `fee_actual = u128::MAX`, `fee_proposal_bounds` returns `[u128::MAX / (1+margin), u128::MAX]`, forcing every future proposer to publish a `fee_proposal_fri` near `u128::MAX`. This value is then fed into `calculate_next_l2_gas_price_for_fin` (via `fee_actual` in `ProposalBuildArguments`), permanently inflating `l2_gas_price` for all subsequent blocks. [7](#0-6) 

---

### Impact Explanation

A Byzantine proposer active during the startup window can:

1. Set `fee_proposal_fri = u128::MAX` in every `ProposalInit` it sends.
2. All validators accept the proposal (bounds check is skipped).
3. The value is committed to the block header and stored in `fee_proposals_window`.
4. After `window_size` blocks, `fee_actual` becomes `u128::MAX`.
5. All future `fee_proposal_fri` values are forced near `u128::MAX`.
6. `l2_gas_price` is permanently inflated to `u128::MAX`, making every Starknet transaction prohibitively expensive.

This is a **Critical** impact: incorrect fee/gas accounting with irreversible economic impact on the entire network.

---

### Likelihood Explanation

The startup window is a fixed, predictable period at chain genesis or after a restart. Any Byzantine validator that is scheduled as proposer during those first `fee_proposal_window_size` blocks can trigger this. In a BFT committee of `n` validators with up to `f < n/3` Byzantine, the probability of at least one Byzantine proposer during the window is non-negligible, especially at genesis when the committee is small or when a coordinated attacker controls multiple early validators.

---

### Recommendation

When `fee_actual` is `None`, the validator should enforce that `fee_proposal_fri` is within a symmetric margin of `l2_gas_price_fri` (the same fallback the honest proposer uses). Concretely, in `is_proposal_init_valid`, add:

```rust
let reference = match proposal_init_validation.fee_actual {
    Some(fa) => fa,
    None => proposal_init_validation.l2_gas_price_fri, // honest proposer's fallback
};
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    let (lower, upper) = fee_proposal_bounds(
        reference,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower || fee_proposal.0 > upper {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
```

`ProposalInitValidation` already carries `l2_gas_price_fri`, so no new fields are needed. [8](#0-7)

### Proof of Concept

```
Precondition: chain at genesis; fee_proposal_window_size = W (e.g., 10).

For blocks 0 .. W-1 (startup window, fee_actual = None):
  Attacker is proposer for at least one block.
  Attacker sets ProposalInit { fee_proposal_fri: Some(GasPrice(u128::MAX)), ... }.

Validator path:
  is_proposal_init_valid:
    fee_actual = None  →  bounds check skipped  →  Ok(())
  initiate_validation: batcher accepts block.
  handle_proposal_part / finish_proposal: batcher returns partial_block_hash.
  proposal_commitment_from(partial, Some(u128::MAX)):
    = Poseidon(partial, u128::MAX)   ← committed value

decision_reached:
  record_fee_proposal(height, Some(u128::MAX))
  fee_proposals_window[height] = u128::MAX

After W blocks:
  compute_fee_actual(window, height W, W) = median([..., u128::MAX, ...])
  If majority of window entries are u128::MAX → fee_actual = u128::MAX

Height W+1 onward:
  fee_proposal_bounds(u128::MAX, margin) = [u128::MAX/(1+m), u128::MAX]
  Every proposer must publish fee_proposal_fri ≈ u128::MAX.
  calculate_next_l2_gas_price_for_fin uses fee_actual ≈ u128::MAX.
  l2_gas_price → u128::MAX for all future blocks.
  All user transactions fail or cost u128::MAX FRI.
``` [9](#0-8) [10](#0-9) [6](#0-5)

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
