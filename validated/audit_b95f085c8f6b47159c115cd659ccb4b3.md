### Title
Unauthenticated `init` Function Allows Any Caller to Hijack MPC Contract Participant Set Before Legitimate Initialization - (File: `crates/contract/src/lib.rs`)

---

### Summary

The `init` function of the MPC contract carries no caller authentication (`#[private]` is absent), and all deployment scripts deploy the contract with `without-init-call`. This creates a front-runnable window between contract deployment and the legitimate `init` call during which any NEAR account can call `init` with attacker-controlled `ThresholdParameters`, installing themselves as the sole participant with threshold 1 and permanently seizing control of the MPC signing network.

---

### Finding Description

The contract is deployed across every deployment path using `without-init-call`: [1](#0-0) [2](#0-1) 

This means the contract code is live on-chain but its state is uninitialized. The legitimate `init` call is issued in a separate, subsequent transaction: [3](#0-2) 

The `init` function itself has **no `#[private]` attribute and no predecessor/signer check**: [4](#0-3) 

Compare this to `init_running`, which is correctly gated: [5](#0-4) 

NEAR SDK's `#[init]` only prevents double-initialization — it does **not** restrict which account may call the function. Any NEAR account can call `init` before the legitimate operator does, supplying arbitrary `ThresholdParameters`. The `parameters.validate()` call only checks internal consistency (threshold ≤ participant count); it does not verify the identity of the caller or the participants. [6](#0-5) 

Once the attacker's `init` call lands, the NEAR runtime prevents any subsequent `init` call (state already exists), so the legitimate operator's transaction fails permanently.

The test suite confirms `init_running` is correctly protected but reveals no analogous protection for `init`: [7](#0-6) 

---

### Impact Explanation

After a successful front-run of `init`, the attacker controls the entire `ThresholdParameters` stored in `RunningContractState`: [8](#0-7) 

With themselves as the sole participant (threshold = 1), the attacker can:

1. Call `vote_add_domains` to register signing domains.
2. Call `respond` / `respond_ckd` / `respond_verify_foreign_tx` — all of which gate on `assert_caller_is_attested_participant_and_protocol_active`, which checks membership in the attacker-controlled participant set. [9](#0-8) 

3. Respond to any user `sign` request with an attacker-generated signature, delivering a fraudulent signature to the caller's yield callback.
4. Permanently freeze the legitimate MPC network: the real operators cannot call `init` again, and `vote_new_parameters` requires being a registered participant.

**Impact class:** Critical — unauthorized threshold signature issuance and permanent seizure of the MPC signing contract, matching "Unauthorized transaction execution, threshold signature issuance … without the required participant authorization."

---

### Likelihood Explanation

NEAR is a public blockchain. Every transaction — including the `deploy` transaction — is immediately visible in the mempool and on-chain. An attacker monitoring for contract deployments to the known MPC contract account can detect the deployment block and submit their `init` call in the very next block. No privileged access, no key material, and no collusion is required. The attack is a single transaction costing negligible gas.

---

### Recommendation

Add `#[private]` to `init` so that only the contract account itself (i.e., `predecessor_account_id == current_account_id`) can call it, consistent with how `init_running` is protected:

```rust
#[handle_result]
#[private]          // ← add this
#[init]
pub fn init(
    parameters: dtos::ThresholdParameters,
    init_config: Option<dtos::InitConfig>,
) -> Result<Self, Error> { … }
```

Alternatively, deploy the contract **with** an atomic init call (`with-init-call`) so there is never an uninitialized window. Both mitigations should be applied together.

---

### Proof of Concept

1. Operator broadcasts: `near contract deploy mpc-contract.near use-file mpc_contract.wasm without-init-call …`
2. Attacker observes the deployment in block N.
3. In block N+1, attacker broadcasts:
   ```
   near contract call-function as-transaction mpc-contract.near init \
     json-args '{"parameters": {"threshold": 1, "participants": {"next_id": 1, "participants": [["attacker.near", 0, {"tls_public_key": "<attacker_key>", "url": "https://attacker.example"}]]}}}' \
     prepaid-gas '300.0 Tgas' attached-deposit '0 NEAR' \
     sign-as attacker.near …
   ```
4. Operator's legitimate `init` call (block N+2) panics: `"Cannot initialize contract: already initialized"`.
5. Attacker calls `vote_add_domains` to add a Secp256k1 signing domain.
6. Any user calling `sign(…)` receives a signature produced by the attacker's key, not the legitimate MPC threshold key. [10](#0-9) [11](#0-10)

    #[handle_result]
    #[init]
    pub fn init(
        parameters: dtos::ThresholdParameters,
        init_config: Option<dtos::InitConfig>,
    ) -> Result<Self, Error> {
        let parameters: ThresholdParameters = parameters.try_into_contract_type()?;
        // Log participant count and hash - full parameters exceed NEAR's 16KB log limit at ~100 participants
        let params_hash = env::sha256_array(borsh::to_vec(&parameters).unwrap());
        log!(
            "init: signer={}, num_participants={}, parameters_hash={:?}, init_config={:?}",
            env::signer_account_id(),
            parameters.participants().len(),
            params_hash,
            init_config,
        );
        parameters.validate()?;

        // TODO(#1087): Every participant must have a valid attestation, otherwise we risk
        // participants being immediately kicked out once contract transitions into running.
        let initial_participants = parameters.participants();
        let tee_state = TeeState::with_mocked_participant_attestations(initial_participants);

        Ok(Self {
            protocol_state: ProtocolContractState::Running(RunningContractState::new(
                DomainRegistry::default(),
                Keyset::new(EpochId::new(0), Vec::new()),
                parameters,
                AddDomainsVotes::default(),
            )),
            pending_signature_requests: LookupMap::new(StorageKey::PendingSignatureRequestsV4),
            pending_ckd_requests: LookupMap::new(StorageKey::PendingCKDRequestsV3),
            pending_verify_foreign_tx_requests: LookupMap::new(
                StorageKey::PendingVerifyForeignTxRequestsV2,
            ),
            proposed_updates: ProposedUpdates::default(),
            config: init_config.map(Into::into).unwrap_or_default(),
            tee_state,
            accept_requests: true,
            node_migrations: NodeMigrations::default(),
            metrics: Default::default(),
            node_foreign_chain_support: Default::default(),
            foreign_chains: Lazy::new(
                StorageKey::ForeignChainMetadata,
                ForeignChainsMetadata::default(),

            tee_verifier_account_id: None,
            tee_verifier_votes: TeeVerifierVotes::default(),

async fn init_running_rejects_external_callers_pre_initialization() {
    let (worker, contract) = init().await;
    let number_of_participants = 2;
    let (accounts, participants) = gen_accounts(&worker, number_of_participants).await;

    let threshold_parameters = ThresholdParameters::new(
        participants.clone(),
        Threshold::new(number_of_participants as u64),
    )
    .unwrap();

    let init_running_args = serde_json::json!({
            "domains": [],
            "next_domain_id": 0,
            "keyset": Keyset::new(EpochId::new(2), vec![]),
            "parameters": threshold_parameters,
    });

    let execution_error = accounts[0]
        .call(contract.id(), method_names::INIT_RUNNING)
        .max_gas()
        .args_json(init_running_args)
        .transact()
        .await
        .unwrap()
        .into_result()
        .expect_err("method is private and not callable from participant account.");

    let error_message = format!("{:?}", execution_error);

    let expected_error_message = "Smart contract panicked: Method init_running is private";

    assert!(
        error_message.contains(expected_error_message),
        "init_running call was accepted by external caller. expected method to be private. {:?}",
        error_message
    )
