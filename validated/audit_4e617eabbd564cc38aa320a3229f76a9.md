Audit Report

## Title
Missing Chainlink Response Validation in `ChainlinkPriceOracle.getAssetPrice()` Enables Corrupted rsETH Price - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards every safety field, returning an unvalidated price directly into `LRTOracle._getTotalEthInProtocol()`. A zero or stale Chainlink answer silently zeroes out an asset's entire TVL contribution, corrupting `rsETHPrice`. Depending on whether `pricePercentageLimit` is configured, this results in either a temporary protocol-wide freeze (Medium) or direct theft of funds via inflated rsETH minting (Critical).

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` reads only the `price` field and discards `roundId`, `updatedAt`, and `answeredInRound`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52-54
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No check is made that `price > 0`, `answeredInRound >= roundId`, or `updatedAt != 0`. The protocol's own `ChainlinkOracleForRSETHPoolCollateral`, used for L2 pool collateral, performs all three checks and reverts on failure:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The unvalidated price feeds directly into `LRTOracle._getTotalEthInProtocol()`:

```solidity
// contracts/LRTOracle.sol L339-343
uint256 assetER = getAssetPrice(asset);
uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
```

If `assetER == 0`, `mulWad(0) == 0`, silently erasing that asset's entire TVL. The corrupted `totalETHInProtocol` propagates to:

```solidity
// contracts/LRTOracle.sol L250
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
```

`updateRSETHPrice()` is public and permissionless:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

**Path A — `pricePercentageLimit > 0` (expected production config):**
The downside protection logic at `LRTOracle.sol L277-281` pauses `LRTDepositPool` and `LRTWithdrawalManager` when `newRsETHPrice` drops beyond the configured threshold. This is a temporary freeze of all user funds triggered by a corrupted oracle value that passed no validation gate.

**Path B — `pricePercentageLimit == 0` (limit not configured, or drop within limit):**
The corrupted price is written to `rsETHPrice` without revert. Any depositor calling `depositAsset()` before correction receives inflated rsETH:

```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

If asset X's price feed returns 0 (corrupting `rsETHPrice` downward), a depositor of asset Y (whose price feed is valid) receives `(Y_amount * Y_price) / corrupted_low_rsETHPrice` — more rsETH than their deposit is worth.

## Impact Explanation

**Path A (Medium):** Temporary freezing of all user funds. The protocol-wide pause triggered by a corrupted oracle value locks deposits and withdrawals until an admin manually unpauses. This matches the allowed impact "Temporary freezing of funds."

**Path B (Critical):** Direct theft of protocol funds. An attacker deposits a valid asset after `rsETHPrice` is corrupted downward, mints excess rsETH, and redeems it after oracle recovery, extracting value from existing holders. This matches the allowed impact "Direct theft of any user funds."

## Likelihood Explanation

Chainlink feeds can return `price = 0` or stale data during L2 sequencer downtime (feed not updated, `updatedAt` not checked), feed deprecation or misconfiguration, or circuit-breaker events where the answer is clamped to `minAnswer`. `updateRSETHPrice()` requires no privileged role; any external caller can trigger it the moment a bad Chainlink round is observed. The attacker needs only to monitor the Chainlink feed and call the public function — no special access, no victim interaction required. The inconsistency with `ChainlinkOracleForRSETHPoolCollateral` confirms the protocol is aware of these failure modes and has already implemented the fix in one context.

## Recommendation

Apply the same three guards already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

if (answeredInRound < roundId) revert StalePrice();
if (updatedAt == 0)            revert IncompleteRound();
if (price <= 0)                revert InvalidPrice();
// Optionally add a heartbeat staleness window:
// if (block.timestamp - updatedAt > HEARTBEAT) revert StalePrice();
```

## Proof of Concept

1. Chainlink's stETH/ETH feed enters a bad round and returns `price = 0` (or a stale round where `answeredInRound < roundId`).
2. Attacker calls `LRTOracle.updateRSETHPrice()` (no access control, only `whenNotPaused`).
3. `_getTotalEthInProtocol()` calls `getAssetPrice(stETH)` → `ChainlinkPriceOracle` returns `0`.
4. `totalAssetAmt.mulWad(0) == 0` — stETH's entire balance (e.g., 50% of TVL) is zeroed.
5. `newRsETHPrice = (0.5 × correctTVL) / rsethSupply` — approximately half the correct price.
6. **Path A:** If `pricePercentageLimit > 0` and the drop exceeds the threshold, `LRTDepositPool` and `LRTWithdrawalManager` are paused — all user funds temporarily frozen.
7. **Path B:** If `pricePercentageLimit == 0`, `rsETHPrice` is written as `~0.5 × 1e18`. Attacker calls `depositAsset(cbETH, X)` → `rsethAmountToMint = X * cbETHPrice / (0.5 × 1e18) ≈ 2X` worth of rsETH minted. After oracle recovers, attacker redeems excess rsETH, extracting value from existing holders.

**Foundry fork test outline:**
- Fork mainnet; deploy mock Chainlink feed returning `price = 0` for stETH.
- Call `updateRSETHPrice()` as an unprivileged address.
- Assert `rsETHPrice` is corrupted (Path B) or protocol is paused (Path A).
- For Path B: deposit cbETH, assert `rsethAmountToMint > fair_value`, redeem after oracle fix, assert profit.
