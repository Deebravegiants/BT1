### Title
`stakedButUnverifiedNativeETH` Not Decrementable After Full Slash Permanently Traps NodeDelegator in Queue - (`contracts/NodeDelegator.sol`, `contracts/LRTDepositPool.sol`)

---

### Summary

When ETH is staked via `stake32Eth()` but withdrawal credentials have not yet been verified with EigenLayer, `stakedButUnverifiedNativeETH` is incremented. If the delegated operator is subsequently slashed to `maxMagnitude = 0` by an AVS, `verifyWithdrawalCredentials()` becomes permanently uncallable (EigenLayer blocks `increaseDelegatedShares()` when `maxMagnitude = 0`), making it impossible to decrement `stakedButUnverifiedNativeETH`. Because `getEffectivePodShares()` returns `stakedButUnverifiedNativeETH + withdrawableShare`, it permanently returns a nonzero value, causing `_checkResidueEthBalance()` to always revert with `NodeDelegatorHasETH`, permanently trapping the NodeDelegator in the queue with no admin escape hatch.

---

### Finding Description

**Root cause — `getEffectivePodShares()` sums two independent quantities:**

```solidity
// contracts/NodeDelegator.sol:556-562
function getEffectivePodShares() external view override returns (uint256 ethStaked) {
    uint256 withdrawableShare =
        NodeDelegatorHelper.getWithdrawableShare(lrtConfig, IStrategy(lrtConfig.beaconChainETHStrategy()));
    return stakedButUnverifiedNativeETH + withdrawableShare;
}
```

`withdrawableShare` is derived from EigenLayer's `DelegationManager.getWithdrawableShares()` and is correctly reduced to 0 after a full slash. However, `stakedButUnverifiedNativeETH` is a local storage variable that EigenLayer's slashing mechanism never touches. [1](#0-0) 

**`stakedButUnverifiedNativeETH` can only be decremented via `verifyWithdrawalCredentials()`:**

```solidity
// contracts/NodeDelegator.sol:235-244
stakedButUnverifiedNativeETH -= (validatorFields.length * (32 ether));
eigenPod.verifyWithdrawalCredentials(...);
```

Internally, `eigenPod.verifyWithdrawalCredentials()` calls `EigenPodManager.recordBeaconChainETHBalanceUpdate()` → `DelegationManager.increaseDelegatedShares()`. The `IDelegationManager` interface explicitly documents: *"If the operator was slashed 100% for the strategy (the operator's maxMagnitude = 0), then increasing delegated shares is blocked and will revert."* After a full slash, this call reverts, making `verifyWithdrawalCredentials()` permanently uncallable, and `stakedButUnverifiedNativeETH` permanently non-zero. [2](#0-1) [3](#0-2) 

**`_checkResidueEthBalance()` gates NDC removal with a strict `!= 0` check:**

```solidity
// contracts/LRTDepositPool.sol:616-624
function _checkResidueEthBalance(address nodeDelegatorAddress) internal view {
    if (
        INodeDelegator(nodeDelegatorAddress).getEffectivePodShares() != 0
            || address(nodeDelegatorAddress).balance > maxNegligibleAmount
            || INodeDelegator(nodeDelegatorAddress).getAssetUnstaking(LRTConstants.ETH_TOKEN) > 0
    ) {
        revert NodeDelegatorHasETH();
    }
}
```

Since `getEffectivePodShares()` permanently returns `stakedButUnverifiedNativeETH > 0`, this check always reverts, and `_removeNodeDelegatorContractFromQueue()` (called by both `removeNodeDelegatorContractFromQueue()` and `removeManyNodeDelegatorContractsFromQueue()`) is permanently blocked. [4](#0-3) [5](#0-4) 

**No admin escape hatch exists.** Searching the entire `NodeDelegator.sol`, `stakedButUnverifiedNativeETH` is only ever written in `stake32Eth()` (increment) and `verifyWithdrawalCredentials()` (decrement). There is no setter, no admin override, and no upgrade path that bypasses the check without a full proxy upgrade. [6](#0-5) 

---

### Impact Explanation

**Critical — Permanent freezing of funds and permanent freezing of the NodeDelegator.**

1. **ETH frozen**: The 32 ETH per unverified validator is in the beacon chain. After a full slash (`maxMagnitude = 0`), the validator can exit and ETH lands in the EigenPod, but `startCheckpoint()` → `completeCheckpoint()` would attempt to award shares via `increaseDelegatedShares()`, which reverts at `maxMagnitude = 0`. The ETH is irrecoverable through any EigenLayer mechanism.

2. **NDC permanently stuck**: `removeNodeDelegatorContractFromQueue()` always reverts with `NodeDelegatorHasETH`. The NDC cannot be dequeued.

3. **Protocol migration blocked**: If the NDC queue is at `maxNodeDelegatorLimit`, new NDCs cannot be added (`MaximumNodeDelegatorLimitReached`), preventing the protocol from migrating to a new operator. [7](#0-6) 

---

### Likelihood Explanation

**Low-Medium.** The preconditions are:
1. ETH staked via `stake32Eth()` but `verifyWithdrawalCredentials()` not yet called — this is a normal operational window that can last hours to days.
2. The operator is registered with an AVS that slashes to `maxMagnitude = 0` — this is a legitimate EigenLayer operation callable by any registered AVS against any operator that has allocated magnitude to it.

The operator choosing to register with a malicious or misbehaving AVS is a realistic risk in the EigenLayer ecosystem. This is not "external protocol compromise" — it is EigenLayer's slashing mechanism functioning as designed, exposing a gap in LRT-rsETH's accounting.

---

### Recommendation

1. **Add an admin function to reset `stakedButUnverifiedNativeETH`** (e.g., `onlyLRTAdmin`) for use in post-slash recovery scenarios.
2. **Alternatively, modify `_checkResidueEthBalance()`** to allow removal when `withdrawableShare == 0` and the operator's `maxMagnitude == 0` (i.e., a full slash has occurred), treating the `stakedButUnverifiedNativeETH` as irrecoverably burned.
3. **Consider a `forceRemoveNodeDelegator()` function** callable by admin that bypasses the ETH balance check when the operator has been fully slashed, to allow protocol migration.

---

### Proof of Concept

```solidity
// Fork test (local/private testnet)
function test_slashTrapsNDC() public {
    // 1. Stake 32 ETH, do NOT verify credentials
    nodeDelegator.stake32Eth(pubkey, sig, root);
    assertEq(nodeDelegator.stakedButUnverifiedNativeETH(), 32 ether);

    // 2. AVS slashes operator to maxMagnitude = 0
    IAllocationManager.SlashingParams memory params = IAllocationManager.SlashingParams({
        operator: operator,
        operatorSetId: 1,
        strategies: beaconStrategies,
        wadsToSlash: [1e18], // 100%
        description: "slash"
    });
    vm.prank(avs);
    allocationManager.slashOperator(avs, params);

    // 3. getEffectivePodShares() returns stakedButUnverifiedNativeETH (withdrawableShare = 0)
    assertEq(nodeDelegator.getEffectivePodShares(), 32 ether);

    // 4. removeNodeDelegatorContractFromQueue reverts permanently
    vm.prank(admin);
    vm.expectRevert(ILRTDepositPool.NodeDelegatorHasETH.selector);
    lrtDepositPool.removeNodeDelegatorContractFromQueue(address(nodeDelegator));

    // 5. verifyWithdrawalCredentials also reverts (maxMagnitude = 0 blocks increaseDelegatedShares)
    vm.prank(operator);
    vm.expectRevert();
    nodeDelegator.verifyWithdrawalCredentials(ts, stateRootProof, indices, proofs, fields);
}
```
