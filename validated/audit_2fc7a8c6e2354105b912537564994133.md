Audit Report

## Title
Pause Bypass in `RSETHMultiChainRateProvider` Allows Stale Pre-Drop Rate Broadcast to All L2 Receivers — (`contracts/cross-chain/RSETHMultiChainRateProvider.sol`)

## Summary

`RSETHMultiChainRateProvider.getLatestRate()` reads `rsETHPrice` directly from `LRTOracle` storage with no pause check. When `LRTOracle` auto-pauses due to a price drop exceeding `pricePercentageLimit`, `rsETHPrice` is frozen at the pre-drop inflated value. Because `MultiChainRateProvider.updateRate()` is permissionless and performs no pause check, any caller can broadcast this stale, inflated rate to every L2 receiver, enabling redemption of rsETH at above-true-value prices on L2 protocols.

## Finding Description

`RSETHMultiChainRateProvider.getLatestRate()` is a raw storage read with no pause guard:

```solidity
// contracts/cross-chain/RSETHMultiChainRateProvider.sol L26-28
function getLatestRate() public view override returns (uint256) {
    return ILRTOracle(rsETHPriceOracle).rsETHPrice();
}
```

`LRTOracle._updateRsETHPrice()` contains a downside-protection path that auto-pauses the oracle and returns early **before** writing the new lower price to `rsETHPrice`:

```solidity
// contracts/LRTOracle.sol L277-282
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;   // <-- exits before rsETHPrice = newRsETHPrice at L313
}
```

The write that would update the price to the new lower value only occurs at line 313, which is never reached when the early return fires:

```solidity
// contracts/LRTOracle.sol L313
rsETHPrice = newRsETHPrice;
```

After auto-pause, `updateRSETHPrice()` is blocked by `whenNotPaused`:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

`MultiChainRateProvider.updateRate()` has no access control and no pause check — it is callable by any EOA or contract:

```solidity
// contracts/cross-chain/MultiChainRateProvider.sol L108-130
function updateRate() external payable nonReentrant {
    uint256 latestRate = getLatestRate();
    rate = latestRate;
    ...
    ILayerZeroEndpoint(layerZeroEndpoint).send{value: estimatedFee}(...);
```

`CrossChainRateReceiver.lzReceive()` stores whatever rate arrives unconditionally:

```solidity
// contracts/cross-chain/CrossChainRateReceiver.sol L93-95
uint256 _rate = abi.decode(_payload, (uint256));
rate = _rate;
```

**Exploit path:**
1. A price drop event (e.g., validator slashing) causes `_updateRsETHPrice()` to compute `newRsETHPrice` below the `pricePercentageLimit` threshold.
2. The auto-pause fires: `_pause()` is called and the function returns early. `rsETHPrice` remains at the pre-drop inflated value.
3. An attacker calls `RSETHMultiChainRateProvider.updateRate{value: fee}()`. No revert occurs — `getLatestRate()` returns the stale inflated `rsETHPrice`.
4. The inflated rate is sent via LayerZero to every registered L2 receiver, which stores it unconditionally.
5. L2 protocols pricing rsETH redemptions against this rate pay out more ETH per rsETH than the asset is worth, draining ETH from L2 liquidity pools.

## Impact Explanation

The auto-pause mechanism's explicit purpose is to halt rate propagation when a significant price anomaly is detected. This bypass defeats that guarantee entirely. L2 protocols that use the broadcast rate for rsETH/ETH pricing will accept redemptions at the pre-drop inflated rate, constituting direct theft of ETH from L2 liquidity providers and pool participants. This matches **Critical — direct theft of user funds**.

## Likelihood Explanation

- The auto-pause requires no admin action; it is triggered automatically by any call to `updateRSETHPrice()` when the computed price drop exceeds `pricePercentageLimit`.
- `updateRate()` is permissionless; the only cost is gas and LayerZero messaging fees, trivial relative to the potential profit from redeeming rsETH at an inflated rate.
- The window between auto-pause and manual unpause (requiring `onlyLRTAdmin`) can span hours to days, providing ample time for exploitation.
- The condition is repeatable: any sufficiently large price drop event re-triggers the same path.

## Recommendation

Add a pause check in `RSETHMultiChainRateProvider.getLatestRate()`:

```solidity
function getLatestRate() public view override returns (uint256) {
    require(!ILRTOracle(rsETHPriceOracle).paused(), "Oracle paused");
    return ILRTOracle(rsETHPriceOracle).rsETHPrice();
}
```

Alternatively, add the check in `MultiChainRateProvider.updateRate()` itself, or expose a pause-aware getter on `LRTOracle` that reverts when paused and call that instead of reading `rsETHPrice` directly.

## Proof of Concept

```solidity
// Foundry fork test (Ethereum mainnet fork)
function test_pauseBypassBroadcastsStaleRate() public {
    // 1. Reference deployed LRTOracle and RSETHMultiChainRateProvider
    // 2. Record current rsETHPrice (pre-drop)
    uint256 stalePre = lrtOracle.rsETHPrice();

    // 3. Mock asset price oracle to return a value that causes a drop
    //    exceeding pricePercentageLimit (e.g., >1% drop)
    mockAssetOracle.setPrice(assetAddr, stalePre * 98 / 100);

    // 4. Call updateRSETHPrice() — triggers auto-pause, returns early
    lrtOracle.updateRSETHPrice();

    // 5. Assert oracle is paused and rsETHPrice unchanged
    assertTrue(lrtOracle.paused());
    assertEq(lrtOracle.rsETHPrice(), stalePre);

    // 6. Attacker calls updateRate — succeeds despite oracle being paused
    uint256 fee = rateProvider.estimateTotalFee();
    vm.deal(attacker, fee);
    vm.prank(attacker);
    rateProvider.updateRate{value: fee}();

    // 7. Assert provider rate equals stale pre-drop value
    assertEq(rateProvider.rate(), stalePre);

    // 8. Assert L2 receiver (or mock) also stores the stale inflated rate
    assertEq(l2Receiver.rate(), stalePre);
    // True protocol rate is ~2% lower; L2 redemptions at stalePre drain ETH
}
