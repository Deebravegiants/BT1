### Title
P2P Sync Class Hash Mismatch Allows Malicious Peer to Permanently Corrupt Class Manager State — (`crates/apollo_p2p_sync/src/client/class.rs`)

---

### Summary

`write_to_storage` in the P2P sync class pipeline calls `add_class(class_body)` but discards the returned `ClassHashes.class_hash` (the hash the class manager actually computed and stored under). Because `parse_data_for_block` only verifies that the peer-supplied `class_hash` label exists in the state diff — never that `calculate_class_hash(class_body) == class_hash` — a malicious peer can supply a valid-looking Sierra body whose real hash differs from the state-diff key. The class manager stores the body under the wrong key, the block marker advances, and the claimed hash is permanently absent from the class manager.

---

### Finding Description

**Step 1 — `parse_data_for_block` accepts the peer's label without verifying the body.**

The peer sends `(class_body C, H_claimed)`. The only check performed is:

```rust
// class.rs line 133
declared_classes.get(&class_hash).is_some()  // H_claimed ∈ state diff?
```

If `H_claimed` is a valid entry in the already-stored state diff, the pair is accepted and inserted into `declared_classes_result` as `(H_claimed → C)`. [1](#0-0) 

There is no call to `C.calculate_class_hash()` and no comparison against `H_claimed`.

**Step 2 — `write_to_storage` ignores the class manager's authoritative hash.**

```rust
// class.rs lines 41-48
while let Err(err) = class_manager_client.add_class(class.clone()).await {
    warn!(...);
}
// Ok(ClassHashes { class_hash: H_actual, .. }) is silently dropped
```

The `while let Err` idiom consumes and discards the `Ok` value. The TODO at line 39 explicitly documents the missing check:

> `// TODO(shahak): Verify class hash matches class manager response. report if not.` [2](#0-1) 

**Step 3 — The class manager stores under the body's real hash, not the claimed one.**

```rust
// class_manager.rs lines 72-109
let sierra_class = SierraContractClass::try_from(&class)?;
let class_hash = sierra_class.calculate_class_hash();   // H_actual
...
self.classes.set_class(class_hash, class, ...);          // stored under H_actual
Ok(ClassHashes { class_hash, ... })                      // H_actual returned, ignored
``` [3](#0-2) 

**Step 4 — The block marker advances unconditionally.**

```rust
// class.rs lines 67-70
storage_writer
    .begin_rw_txn()?
    .update_class_manager_block_marker(&self.2.unchecked_next())?
    .commit()?;
``` [4](#0-3) 

After this commit, the sync loop will never revisit the block. `H_claimed` is permanently absent from the class manager; `H_actual` (the body's real hash) is present instead.

---

### Impact Explanation

Any node syncing via P2P that connects to a malicious peer will have a permanently corrupted class manager for every block the peer serves with mismatched class bodies. Concretely:

- `get_sierra(H_claimed)` → `None` (wrong; should return the class)
- `get_executable(H_claimed)` → `None` (wrong)
- Any contract deployed with class `H_claimed` cannot be executed on this node
- RPC endpoints (`starknet_getClass`, fee estimation, simulation, tracing) return `ClassNotFound` for a hash that is legitimately declared on-chain

This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

---

### Likelihood Explanation

The attacker only needs to be a reachable P2P peer. No validator, operator, or privileged role is required. The state diff (containing `H_claimed`) is public. Constructing a Sierra class body whose `calculate_class_hash` differs from a target hash is straightforward (e.g., modify any field — ABI string, sierra program — while keeping the class structurally valid so it passes version and length checks). The missing check is self-documented as a TODO, confirming it has never been enforced.

---

### Recommendation

In `write_to_storage`, after `add_class` succeeds, compare the returned hash against the state-diff key and treat a mismatch as a `BadPeerError`:

```rust
match class_manager_client.add_class(class.clone()).await {
    Ok(ClassHashes { class_hash: actual, .. }) if actual != class_hash => {
        // report bad peer, abort or skip block
    }
    Err(err) => { warn!(...); /* retry */ }
    Ok(_) => {}
}
```

Alternatively, validate `class_body.calculate_class_hash() == class_hash` inside `parse_data_for_block` before inserting into `declared_classes_result`, so the peer is penalized immediately.

---

### Proof of Concept

1. Obtain `H_claimed` from any block's state diff (public data).
2. Construct a `SierraContractClass` body `C` such that `C.calculate_class_hash() = H_actual ≠ H_claimed` (e.g., append a byte to the ABI string).
3. As a P2P peer, respond to the victim's class-sync request for that block with `(C, H_claimed)`.
4. `parse_data_for_block` accepts the pair (H_claimed ∈ state diff ✓, no body-hash check).
5. `write_to_storage` calls `add_class(C)` → class manager stores `C` under `H_actual`; `Ok(ClassHashes { class_hash: H_actual })` is discarded.
6. Marker advances past the block.
7. Assert: `get_sierra(H_claimed)` → `None`; `get_sierra(H_actual)` → `Some(C)`.
