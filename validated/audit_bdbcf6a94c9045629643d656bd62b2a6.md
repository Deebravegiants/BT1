### Title
Inbound WALLET/FARMER Connection Slots Permanently Exhaustible by Silent Peers — (`chia/server/server.py`)

---

### Summary

`garbage_collect_connections_task` only evicts inactive `FULL_NODE` connections. `WALLET` and `FARMER` inbound connections are never garbage-collected for inactivity. Because the public Chia CA key is shipped with the software and the peer's declared `node_type` is taken verbatim from the handshake with no cross-check against the TLS cert, any attacker can generate unlimited unique peer identities, fill all `max_inbound_wallet` (default 20) or `max_inbound_farmer` (default 10) slots with silent connections, and permanently prevent honest wallets or farmers from connecting to that full node.

---

### Finding Description

**Root cause 1 — GC only covers `FULL_NODE` connections** [1](#0-0) 

The `elif` branch that marks a connection for removal requires **both** conditions:
- `self._local_type in {NodeType.FULL_NODE, NodeType.WALLET}` (the local node is a full node or wallet)
- `connection.connection_type == NodeType.FULL_NODE` (the *remote* peer is a full node)

Connections where `connection.connection_type` is `WALLET`, `FARMER`, or `HARVESTER` are never added to `to_remove` for inactivity, regardless of how long they have been silent.

**Root cause 2 — Inbound slot limits exist for WALLET and FARMER** [2](#0-1) 

Once `inbound_count >= max_inbound_wallet` (default 20) or `>= max_inbound_farmer` (default 10), `should_accept_inbound` returns `False` and the new connection is immediately closed. [3](#0-2) 

**Root cause 3 — `node_type` is self-declared, not validated against the TLS cert** [4](#0-3) 

`remote_node_type` is taken directly from `inbound_handshake.node_type`. There is no check that the declared type matches the TLS certificate type or any other constraint.

**Root cause 4 — Public Chia CA key is distributed with the software** [5](#0-4) [6](#0-5) 

`get_chia_ca_crt_key()` reads `chia_ca.key` from the package. `create_all_ssl` writes it to every installation's disk. Any attacker can call `generate_ca_signed_cert` with this key to produce arbitrarily many unique TLS certs, each yielding a distinct `peer_node_id` (SHA-256 of DER fingerprint). The full node's public SSL server context accepts any cert signed by this CA. [7](#0-6) 

**Root cause 5 — `last_message_time` initialises to 0, but the check is gated on `FULL_NODE`** [8](#0-7) 

A silent WALLET connection has `last_message_time = 0`, so `time.time() - 0 > 1800` is immediately true — but the GC condition never reaches that branch for non-`FULL_NODE` connection types.

---

### Impact Explanation

An attacker with 20 unique Chia-CA-signed certs (trivially generated from the bundled `chia_ca.key`) can permanently occupy all `max_inbound_wallet` slots on a target full node. New honest wallets are rejected with "Inbound limit reached" for as long as the attacker's TCP connections remain open. The same attack applies to `max_inbound_farmer` (10 slots). This is a **long-lived, targeted denial of service** against a specific full node's ability to serve wallets and farmers, matching the High impact criterion: *"Permanent or long-lived inability for honest wallets, farmers… to process… sync updates… under normal network assumptions."*

---

### Likelihood Explanation

- **Attacker capability**: Any party who has installed Chia (or downloaded the package) possesses `chia_ca.key` and can call `generate_ca_signed_cert` in a loop.
- **Effort**: Generating 20 certs and opening 20 WebSocket connections is trivial scripting.
- **Persistence**: The server-side `web.WebSocketResponse` has no heartbeat configured, so the attacker only needs to keep the TCP sockets open. The GC task runs every 600 s but will never evict these connections.
- **No special privileges required**: The attack is fully unprivileged and remote.

---

### Recommendation

Add inactivity-based GC for `WALLET` and `FARMER` inbound connections, mirroring the existing `FULL_NODE` logic:

```python
elif self._local_type == NodeType.FULL_NODE and connection.connection_type in {
    NodeType.WALLET, NodeType.FARMER
}:
    if time.time() - connection.last_message_time > 1800:
        to_remove.append(connection)
```

Alternatively, apply a single inactivity threshold to **all** inbound connection types, not just `FULL_NODE`.

    def accept_inbound_connections(self, node_type: NodeType) -> bool:
        """Check if there are available inbound slots for this node type per config.yaml limits."""
        if not self._local_type == NodeType.FULL_NODE:
            return True
        inbound_count = len(self.get_connections(node_type, outbound=False))
        if node_type == NodeType.FULL_NODE:

        if node_type == NodeType.FARMER:
            return inbound_count < cast(int, self.config.get("max_inbound_farmer", 10))
        return False

    one_day = datetime.timedelta(1, 0, 0)
    root_cert = x509.load_pem_x509_certificate(ca_crt, default_backend())
    root_key = load_pem_private_key(ca_key, None, default_backend())

    cert_key = rsa.generate_private_key(public_exponent=65537, key_size=2048, backend=default_backend())
    new_subject = x509.Name(
        [
            x509.NameAttribute(NameOID.COMMON_NAME, "Chia"),
            x509.NameAttribute(NameOID.ORGANIZATION_NAME, "Chia"),
            x509.NameAttribute(NameOID.ORGANIZATIONAL_UNIT_NAME, "Organic Farming Division"),
        ]
    )

    cert = (
        x509.CertificateBuilder()
        .subject_name(new_subject)
        .issuer_name(root_cert.issuer)
        .public_key(cert_key.public_key())
        .serial_number(x509.random_serial_number())
        .not_valid_before(datetime.datetime.today() - one_day)
        .not_valid_after(datetime.datetime(2100, 8, 2))
        .add_extension(
            x509.SubjectAlternativeName([x509.DNSName("chia.net")]),
            critical=False,
        )
        .sign(root_key, hashes.SHA256(), default_backend())  # type: ignore[arg-type]
    )

    cert_pem = cert.public_bytes(encoding=serialization.Encoding.PEM)
    key_pem = cert_key.private_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PrivateFormat.PKCS8,
        encryption_algorithm=serialization.NoEncryption(),
    )

    write_ssl_cert_and_key(cert_out, cert_pem, key_out, key_pem)
