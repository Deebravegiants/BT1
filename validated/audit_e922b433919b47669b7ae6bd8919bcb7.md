Audit Report

## Title
Missing Staleness Validation in `ChainlinkPriceOracle.getAssetPrice()` Enables Over-Minting of rsETH via Stale Feed — (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards every staleness indicator (`roundId`, `updatedAt`, `answeredInRound`), while the sibling contract `ChainlinkOracleForRSETHPoolCollateral.getRate()` performs all three staleness checks. Because `LRTOracle.updateRSETHPrice()` is an unrestricted public function, any caller can commit a stale-price-derived `rsETHPrice` to storage, causing `LRTDepositPool.getRsETHAmountToMint()` to over-mint rsETH and dilute existing holders' unclaimed yield.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` reads from Chainlink but discards all staleness signals:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52-54
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`roundId`, `updatedAt`, and `answeredInRound` are all ignored. No check for `answeredInRound < roundId` (stale round), `updatedAt == 0` (incomplete round), or `price <= 0` (invalid price) is performed. [1](#0-0) 

In contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` performs all three checks:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L27-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

The exploit path is fully reachable by an unprivileged caller:

1. `updateRSETHPrice()` is `public whenNotPaused` — no role check. [3](#0-2) 

2. It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which iterates over all supported assets and calls `getAssetPrice(asset)` for each. [4](#0-3) 

3. A stale, artificially-low price for any LST (e.g., stETH/ETH) underestimates `totalETHInProtocol`, producing a lower `newRsETHPrice` that is written to `rsETHPrice`. [5](#0-4) 

4. `getRsETHAmountToMint` divides by `lrtOracle.rsETHPrice()` — a lower denominator yields more rsETH per unit deposited. [6](#0-5) 

**Why the `pricePercentageLimit` guard is insufficient:**
The downside-protection logic only auto-pauses when `diff > pricePercentageLimit.mulWad(highestRsethPrice)`. If `pricePercentageLimit` is zero (unset), the condition `pricePercentageLimit > 0` is false and no pause occurs regardless of the price drop. Even when set, a stale price within the configured band (e.g., 0.2% deviation) passes silently and still produces measurable over-minting. [7](#0-6) 

## Impact Explanation

**High — Theft of unclaimed yield.**

When a Chainlink feed for a supported LST goes stale with a price below the true market price, an attacker can call the public `updateRSETHPrice()` to commit the underestimated `rsETHPrice`, then immediately deposit ETH (whose oracle is unaffected) to receive more rsETH than the true exchange rate warrants. Once the feed recovers and `rsETHPrice` corrects upward, the attacker redeems the over-minted rsETH for more underlying value than was deposited, extracting value from existing rsETH holders through share dilution. This is a concrete, quantifiable theft of unclaimed yield, not a theoretical or best-practice concern.

## Likelihood Explanation

Chainlink feeds go stale during periods of network congestion or oracle operator issues. The attack requires no special permissions — `updateRSETHPrice()` is callable by any EOA. The attacker only needs to observe a stale feed and act within the staleness window before the feed self-corrects. The `pricePercentageLimit` guard provides no protection when unset and only partial protection when set to a small value. This is a realistic, externally-reachable condition with documented historical precedent on Ethereum mainnet.

## Recommendation

Add staleness validation to `ChainlinkPriceOracle.getAssetPrice()` consistent with the checks already present in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, add a per-feed configurable heartbeat check: `if (block.timestamp - updatedAt > maxStaleness[asset]) revert StalePrice();`.

## Proof of Concept

1. Assume stETH/ETH Chainlink feed heartbeat is 24 hours. The feed goes stale (last update 23h 59m ago) with a price of `0.999e18` while the true price is `1.001e18`.
2. Attacker calls `LRTOracle.updateRSETHPrice()` (public, no access control).
3. `_getTotalEthInProtocol()` calls `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns stale `0.999e18` with no revert.
4. `totalETHInProtocol` is underestimated → `newRsETHPrice` is set lower than true value (e.g., `1.049e18` instead of `1.051e18`). The `pricePercentageLimit` guard does not trigger because the deviation (~0.19%) is within any reasonable configured band, or because `pricePercentageLimit` is zero.
5. Attacker calls `LRTDepositPool.depositETH{value: 100 ether}(0, "")`.
6. `getRsETHAmountToMint` computes: `100e18 * 1e18 / 1.049e18 ≈ 95.33 rsETH` instead of the correct `100e18 * 1e18 / 1.051e18 ≈ 95.15 rsETH` — attacker receives ~0.18 rsETH excess.
7. Chainlink feed updates; `rsETHPrice` corrects to `1.051e18`. Attacker redeems 95.33 rsETH for more ETH than deposited, extracting value from existing holders.

**Foundry fork test plan:** Fork mainnet, mock a Chainlink aggregator returning a stale `answeredInRound < roundId` response for the stETH/ETH feed, call `updateRSETHPrice()` as an unprivileged EOA, assert `rsETHPrice` is set to the stale-derived value, then call `depositETH` and assert the minted rsETH exceeds the amount that would be minted at the true price.
