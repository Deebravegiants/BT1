### Title
Missing Chainlink Staleness Check Enables Over-Minting of rsETH via Stale High Asset Price — (`contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all freshness fields (`updatedAt`, `roundId`, `answeredInRound`). When a Chainlink feed goes stale at a price higher than the true market price, `LRTDepositPool.getRsETHAmountToMint()` mints rsETH proportional to the inflated stale price while `rsETHPrice` still reflects the last correctly-computed value. The attacker receives more rsETH than their collateral is worth in ETH, extracting value from existing depositors and reducing the protocol's collateral ratio below 1.0.

---

### Finding Description

**Root cause — `ChainlinkPriceOracle.getAssetPrice()`:**

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol  line 52
(, int256 price,,,) = priceFeed.latestRoundData();
```

All five return values of `latestRoundData()` are available (`roundId`, `answer`, `startedAt`, `updatedAt`, `answeredInRound`), but only `answer` is used. There is no check on `updatedAt` (heartbeat staleness) or `answeredInRound < roundId` (incomplete round). [1](#0-0) 

The same codebase already implements both checks in a sibling oracle contract:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
``` [2](#0-1) 

**Minting formula — `LRTDepositPool.getRsETHAmountToMint()`:**

```solidity
// contracts/LRTDepositPool.sol  line 520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [3](#0-2) 

`lrtOracle.getAssetPrice(asset)` is a **live Chainlink call** (no cache). `lrtOracle.rsETHPrice()` is a **stored value** updated only when `updateRSETHPrice()` is called. This temporal mismatch is the exploit surface.

**`rsETHPrice` update path — `LRTOracle._getTotalEthInProtocol()`:**

```solidity
// contracts/LRTOracle.sol  line 339
uint256 assetER = getAssetPrice(asset);   // also calls ChainlinkPriceOracle
``` [4](#0-3) 

Both the numerator (`getAssetPrice`) and the denominator (`rsETHPrice`) ultimately depend on the same Chainlink feed, but they are evaluated at **different times**.

---

### Impact Explanation

**Attack window (no admin action required):**

| Time | Event |
|------|-------|
| T0 | `updateRSETHPrice()` called; stETH/ETH = 1.00, `rsETHPrice` stored = 1.00e18 |
| T1 | Chainlink feed goes stale; last reported value = 1.05e18; true market price drops to 0.99e18 |
| T2 | Attacker calls `depositAsset(stETH, D)`: `rsethAmountToMint = D × 1.05e18 / 1.00e18 = 1.05D` |
| T3 | `updateRSETHPrice()` called with stale 1.05 → `rsETHPrice` inflates, `highestRsethPrice` rises |
| T4 | Chainlink feed resumes; reports 0.99e18 |
| T5 | `updateRSETHPrice()` called; TVL computed at 0.99 → `rsETHPrice` drops |
| T6 | Attacker redeems 1.05D rsETH; receives `1.05D × rsETHPrice_corrected` ETH > D ETH deposited |

The attacker's rsETH represents a claim on ~6% more ETH than deposited. The shortfall is borne by all other rsETH holders, reducing the collateral ratio below 1.0 — **protocol insolvency**.

The `pricePercentageLimit` downside-pause guard (lines 270–282) only triggers on `updateRSETHPrice()` calls after the price corrects; it does not prevent the initial over-minting and may not be configured (default value is 0). [5](#0-4) 

---

### Likelihood Explanation

Chainlink feeds go stale during Ethereum network congestion (high gas prices prevent keeper updates) and during oracle infrastructure incidents — both are documented historical events. The stETH/ETH feed has a 24-hour heartbeat, meaning a stale price can persist for up to 24 hours before any on-chain mechanism detects it. The deposit path (`depositAsset`) is permissionless and open to any user. No privileged role, front-running, or brute-force is required.

---

### Recommendation

Add staleness and validity checks to `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optional: if (block.timestamp - updatedAt > MAX_HEARTBEAT) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Fuzz: stalePriceHigh in [1.01e18, 1.10e18], truePrice = 0.99e18
// Invariant: minted rsETH value at corrected price <= deposit value

contract MockStaleFeed {
    int256 public staleAnswer;
    uint8 public decimals_ = 18;
    constructor(int256 _answer) { staleAnswer = _answer; }
    function decimals() external view returns (uint8) { return decimals_; }
    function latestRoundData() external view returns (
        uint80, int256, uint256, uint256, uint80
    ) {
        // answeredInRound < roundId  AND  updatedAt is old — both staleness signals
        return (2, staleAnswer, 0, block.timestamp - 2 days, 1);
    }
}

// Setup:
//   1. Deploy ChainlinkPriceOracle, set stETH feed = MockStaleFeed(stalePriceHigh)
//   2. Set rsETHPrice = 1e18 (last correct update)
//   3. Call depositAsset(stETH, 1e18)
//      → rsethAmountToMint = 1e18 * stalePriceHigh / 1e18 = stalePriceHigh
//   4. Simulate updateRSETHPrice() with corrected feed (truePrice = 0.99e18)
//      → newRsETHPrice = totalETH_at_truePrice / rsethSupply
//      → totalETH = 1e18 * 0.99e18 / 1e18 = 0.99e18
//      → rsethSupply = stalePriceHigh (e.g. 1.05e18)
//      → newRsETHPrice = 0.99e18 / 1.05e18 ≈ 0.9428e18
//   5. Attacker redeems stalePriceHigh rsETH at newRsETHPrice
//      → payout = stalePriceHigh * newRsETHPrice / 1e18
//             = 1.05e18 * 0.9428e18 / 1e18 ≈ 0.99e18
//      → deposited 1e18 stETH worth 0.99e18 ETH, receives 0.99e18 ETH — break even here
//
// The real profit comes from the window where rsETHPrice has NOT yet been updated
// after the stale price inflated it at T3:
//   rsETHPrice_inflated = 1.05e18 (updated with stale feed)
//   Attacker redeems before T5: payout = 1.05e18 * 1.05e18 / 1e18 = 1.1025e18 ETH
//   Deposited: 1e18 stETH worth 0.99e18 ETH → profit ≈ 0.1125e18 ETH per unit
//
// assert(payoutAtCorrectedPrice > depositValueInETH);  // FAILS without staleness check
// assert(collateralRatio < 1e18);                      // FAILS without staleness check
```

The `MockStaleFeed` returns `answeredInRound = 1 < roundId = 2` and `updatedAt = block.timestamp - 2 days` — both are staleness signals that `ChainlinkPriceOracle.getAssetPrice()` silently ignores, while `ChainlinkOracleForRSETHPoolCollateral.getRate()` would correctly revert. [1](#0-0) [6](#0-5)
