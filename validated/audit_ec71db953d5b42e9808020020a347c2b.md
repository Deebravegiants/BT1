### Title
Stale Stored Attestation Used in Participant Authorization Bypasses Expiry and Revocation Checks - (File: `crates/contract/src/tee/tee_state.rs`)

### Summary

`is_caller_an_attested_participant()` checks only that a stored `NodeAttestation` entry exists for the caller's TLS key. It never calls `re_verify()` to confirm the stored attestation is still valid at the current block timestamp or that the node's image hash is still in the allowed list. This is the direct analog of ConicFinance's `_cachedTotalUnderlying` bug: a stale cached value is used where the live, expiry-aware check (`reverify_participants()`) should be used instead.

### Finding Description

`is_caller_an_attested_participant()` in `crates/contract/src/tee/tee_state.rs` performs only an existence check on `stored_attestations`: [1](#0-0) 

It retrieves the stored `NodeAttestation` and verifies identity fields (`account_id`, `account_public_key`), but never calls `re_verify()` on `verified_attestation`. In contrast, `reverify_participants()` explicitly calls `re_verify()` with the current block timestamp and the current allowed-hash lists: [2](#0-1) 

`re_verify()` enforces two critical checks that `is_caller_an_attested_participant()` skips:

1. **Expiry**: `expiry_timestamp_seconds < timestamp_seconds` → `VerificationError`
2. **Allowlist**: the stored `mpc_image_hash` must still be in `allowed_mpc_docker_image_hashes` [3](#0-2) 

`assert_caller_is_attested_participant_and_protocol_active()` delegates entirely to `is_caller_an_attested_participant()`: [4](#0-3) 

This guard is the sole TEE-attestation check on every critical node-facing method:

- `respond()` — submits threshold signature responses
- `respond_ckd()` — submits confidential key derivation responses
- `respond_verify_foreign_tx()` — submits foreign-chain verification responses
- `vote_pk()` — votes on the DKG public key
- `vote_reshared()` — votes to complete resharing
- `start_keygen_instance()` / `start_reshare_instance()` — initiates key events [5](#0-4) [6](#0-5) 

The `clean_invalid_attestations` sweep that would eventually evict expired entries is only triggered as a detached promise after `vote_reshared()` completes, is bounded by `max_scan`, and is not guaranteed to run before the next signing round: [7](#0-6) 

### Impact Explanation

**Medium.** The production safety invariant is: *only nodes running currently-approved TEE software may participate in protocol operations*. This invariant is broken in two concrete scenarios:

1. **Attestation expiry**: A node's `VerifiedAttestation` carries an `expiry_timestamp_seconds` stamped at submission time (default 7 days). After expiry, the node should be excluded. Because `is_caller_an_attested_participant()` never checks the timestamp, the node continues to call `respond()`, `vote_pk()`, and `vote_reshared()` indefinitely until a resharing cycle happens to trigger `clean_invalid_attestations`.

2. **Image-hash revocation**: When operators remove a compromised MPC image hash from the allowed list (the primary response to a discovered vulnerability), nodes running that image should be immediately excluded. Because `is_caller_an_attested_participant()` never checks `allowed_mpc_docker_image_hashes`, nodes running the revoked image continue to participate in signing and key-event voting. A node running a known-vulnerable image that can still call `respond()` may participate in threshold signing while its key share is at risk of extraction.

This breaks the contract execution-flow and participant-state invariants without requiring network-level DoS or operator misconfiguration.

### Likelihood Explanation

**Medium.** Attestation expiry is a routine event (every 7 days by default). Any node that fails to renew — or any node whose image hash is revoked — immediately enters the vulnerable window. The window persists until the next resharing cycle completes and `clean_invalid_attestations` runs. No special attacker capability is required: the node simply continues calling the same contract methods it always called.

### Recommendation

Replace the existence-only check in `is_caller_an_attested_participant()` with a call to `reverify_participants()`, which already encapsulates the correct expiry and allowlist logic:

```rust
pub(crate) fn is_caller_an_attested_participant(
    &self,
    participants: &Participants,
    tee_upgrade_deadline_duration: Duration,
) -> Result<(), AttestationCheckError> {
    let signer_id = env::signer_account_id();
    let signer_account_pk = env::signer_account_pk();

    let info = participants
        .info(&signer_id)
        .ok_or(AttestationCheckError::CallerNotParticipant)?;

    let attestation = self
        .stored_attestations
        .get(&info.tls_public_key)
        .ok_or(AttestationCheckError::AttestationNotFound)?;

    if attestation.node_id.account_id != signer_id {
        return Err(AttestationCheckError::AttestationOwnerMismatch);
    }

    let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
        .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
    if attestation.node_id.account_public_key != signer_ed25519 {
        return Err(AttestationCheckError::AttestationKeyMismatch);
    }

    // NEW: re-verify expiry and allowlist
    match self.reverify_participants(&attestation.node_id, tee_upgrade_deadline_duration) {
        TeeQuoteStatus::Valid => Ok(()),
        TeeQuoteStatus::Invalid(reason) => Err(AttestationCheckError::AttestationExpiredOrRevoked(reason)),
    }
}
```

Propagate `tee_upgrade_deadline_duration` from `self.config` through `assert_caller_is_attested_participant_and_protocol_active()`.

### Proof of Concept

1. Node A submits a valid attestation with `expiry_timestamp_seconds = T`. The attestation is stored in `stored_attestations`.
2. Block time advances past `T`. Node A's attestation is now expired.
3. Operators also remove Node A's image hash from `allowed_docker_image_hashes` (e.g., due to a discovered vulnerability).
4. Node A calls `respond(request, signature)`.
5. `assert_caller_is_attested_participant_and_protocol_active()` → `is_caller_an_attested_participant()` → finds the stored entry, checks `account_id` and `account_public_key` match → returns `Ok(())`.
6. `re_verify()` is never called. The expired, revoked-image attestation passes the check.
7. Node A's `respond()` call succeeds, and the signature is accepted by the contract, despite Node A running software that operators have explicitly revoked.

The existing test `test_re_verify_rejects_expired_attestation` confirms `reverify_participants()` would catch this, but `is_caller_an_attested_participant()` never invokes it: [8](#0-7)

    /// reverifies stored participant attestations.
    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

impl VerifiedAttestation {
    pub fn re_verify(
        &self,
        timestamp_seconds: u64,
        allowed_mpc_docker_image_hashes: &[NodeImageHash],
        allowed_launcher_docker_compose_hashes: &[LauncherDockerComposeHash],
        allowed_measurements: &[ExpectedMeasurements],
    ) -> Result<(), VerificationError> {
        match self {
            Self::Dstack(ValidatedDstackAttestation {
                mpc_image_hash,
                launcher_compose_hash,
                expiry_timestamp_seconds: expiration_timestamp_seconds,
                measurements,
            }) => {
                let attestation_has_expired = *expiration_timestamp_seconds < timestamp_seconds;

                if attestation_has_expired {
                    return Err(VerificationError::Custom(format!(
                        "The attestation expired at t = {:?}, time_now = {:?}",
                        expiration_timestamp_seconds, timestamp_seconds
                    )));
                }

                let () = verify_mpc_hash(mpc_image_hash, allowed_mpc_docker_image_hashes)?;
                let () = verify_launcher_compose_hash(
                    launcher_compose_hash,
                    allowed_launcher_docker_compose_hashes,
                )?;

                verify_measurements(measurements, allowed_measurements)?;

                Ok(())
            }
            Self::Mock(mock_attestation) => mock_attestation.verify_constraints(
                timestamp_seconds,
                allowed_mpc_docker_image_hashes,
                allowed_launcher_docker_compose_hashes,
                allowed_measurements,
            ),
        }
    }
