Looking at the production code carefully:

### Title
Unbacked eNear Minting via Direct `eNear.finaliseNearToEthTransfer` Call When `eNear.prover()` Is `FakeProver` — (`evm/src/eNear/contracts/ENearProxy.sol`)

---

### Summary

`ENearProxy.mint()` constructs crafted proof data and calls `eNear.finaliseNearToEthTransfer` directly. For this to succeed, `eNear.prover()` **must** be set to `FakeProver` — a contract that unconditionally returns `true` for any input. Because `eNear.finaliseNearToEthTransfer` is a public function with no caller restriction, any unprivileged attacker can call it directly (bypassing `ENearProxy` entirely) with crafted proof data and mint arbitrary eNear, since `FakeProver` approves every proof.

---

### Finding Description

`ENearProxy` has two distinct proof-verification paths:

**Path 1 — `ENearProxy.mint()` (privileged, MINTER_ROLE):** [1](#0-0) 

This constructs `fakeProofData` with a crafted receipt ID, amount, and recipient, then calls `eNear.finaliseNearToEthTransfer(fakeProofData, 0)` directly — **no real proof, no prover check at the ENearProxy level**. For this call to succeed, `eNear.prover()` must accept the fake data, which is only possible if `eNear.prover()` is `FakeProver`.

**Path 2 — `ENearProxy.finaliseNearToEthTransfer()` (public, legacy):** [2](#0-1) 

This verifies against `ENearProxy.prover` (the real prover) before forwarding to `eNear`. This is the intended security gate for legacy transfers.

**`FakeProver`:** [3](#0-2) 

`FakeProver.proveOutcome` ignores all inputs and returns `true` unconditionally.

**The mismatch:** `ENearProxy.prover` = real prover (security gate). `eNear.prover()` = `FakeProver` (no security). Since `eNear.finaliseNearToEthTransfer` is publicly callable (no `msg.sender` restriction in the interface or the original eNear contract design), an attacker can call it directly, bypassing `ENearProxy` entirely. `FakeProver` approves the call, and eNear mints tokens.

The proof data format is fully known from `ENearProxy.mint()`: [4](#0-3) 

An attacker replicates this structure with any unused receipt ID, any amount, and their own address.

---

### Impact Explanation

An unprivileged attacker mints arbitrary amounts of eNear with no NEAR backing. This directly violates the 1:1 backing invariant of the bridge, inflates eNear supply, and allows the attacker to redeem or sell unbacked tokens — draining real NEAR from the NEAR-side connector or crashing the eNear peg.

**Impact: Critical** — unauthorized creation of wrapped bridge assets through verification failure.

---

### Likelihood Explanation

The attack requires:
1. Knowing `eNear.prover()` is `FakeProver` (inferable from `ENearProxy.mint()` existing and working).
2. Knowing the proof data format (fully visible in `ENearProxy.mint()`).
3. Picking an unused receipt ID (trivially enumerable on-chain).
4. Calling a public function directly.

No privileged access, no key compromise, no external dependency. Fully executable by any on-chain actor.

---

### Recommendation

1. **Restrict `eNear.finaliseNearToEthTransfer` to only be callable by `ENearProxy`** — add a caller check (`require(msg.sender == ENearProxy_address)`) in the eNear contract via `adminSstore` or an upgrade.
2. **Alternatively**, replace `FakeProver` with a prover that only accepts calls from `ENearProxy` (i.e., checks `msg.sender`), so direct calls to `eNear.finaliseNearToEthTransfer` from other callers fail proof verification.
3. **Do not rely on `ENearProxy` as the sole access-control layer** when the underlying `eNear` contract is publicly callable and its prover is `FakeProver`.

---

### Proof of Concept

```solidity
// Attacker contract — no special privileges required
contract Exploit {
    IENear eNear = IENear(0x85F17Cf997934a597031b2E18a9aB6ebD4B9f6a4);

    function attack(uint256 unusedReceiptId) external {
        // Replicate ENearProxy.mint() proof data format
        // with attacker address and max amount
        uint128 amount = type(uint128).max;
        bytes memory nearConnector = eNear.nearConnector();

        bytes memory craftedProof = bytes.concat(
            new bytes(72),
            hex"01000000",
            abi.encodePacked(uint64(unusedReceiptId)),  // fresh receipt ID
            new bytes(24),
            abi.encodePacked(Borsh.swapBytes4(uint32(nearConnector.length))),
            abi.encodePacked(nearConnector),
            hex"022500000000",
            abi.encodePacked(Borsh.swapBytes16(amount)),
            abi.encodePacked(address(this)),            // attacker recipient
            new bytes(280)
        );

        // Bypass ENearProxy entirely — call eNear directly
        // eNear.prover() == FakeProver → proveOutcome() returns true for any input
        eNear.finaliseNearToEthTransfer(craftedProof, 0);

        // Attacker now holds `amount` unbacked eNear
        assert(eNear.balanceOf(address(this)) == amount);
    }
}
```
