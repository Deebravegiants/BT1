Audit Report

## Title
Missing `limitDailyMint` Enforcement in `RSETHPoolV2NBA.deposit` Allows Unbounded Daily `wrsETH` Minting - (File: contracts/pools/RSETHPoolV2NBA.sol)

## Summary
`RSETHPoolV2NBA.deposit` calls `wrsETH.mint` with no daily cap check, while every other pool variant in the codebase (`RSETHPoolV2`, `RSETHPoolV2ExternalBridge`, `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, `RSETHPoolV3WithNativeChainBridge`) enforces the `limitDailyMint` modifier on the same minting path. The contract also declares none of the required state variables (`dailyMintLimit`, `dailyMintAmount`, `lastMintDay`, `startTimestamp`). Any depositor can mint an unbounded amount of `wrsETH` through this pool in a single day, bypassing the protocol's intended daily issuance control.

## Finding Description
`RSETHPoolV2NBA.deposit` is defined as:

```solidity
function deposit(string memory referralId) external payable nonReentrant whenNotPaused {
    ...
    wrsETH.mint(msg.sender, rsETHAmount);
    ...
}
``` [1](#0-0) 

The contract's storage layout contains no daily-limit variables whatsoever. [2](#0-1) 

By contrast, `RSETHPoolV2.deposit` applies `limitDailyMint(msg.value)`: [3](#0-2) 

The `limitDailyMint` modifier in `RSETHPoolV2` tracks `dailyMintAmount` against `dailyMintLimit` and reverts with `DailyMintLimitExceeded` when the cap is exceeded: [4](#0-3) 

The same modifier and state variables are present in `RSETHPoolV2ExternalBridge`: [5](#0-4) [6](#0-5) 

A grep across all pool contracts confirms `limitDailyMint` appears in every pool except `RSETHPoolV2NBA`. The only guards present in `RSETHPoolV2NBA.deposit` are `nonReentrant` and `whenNotPaused`, neither of which constrains mint volume.

## Impact Explanation
The protocol's daily issuance cap — the primary on-chain risk-management control — is entirely absent for this pool. A depositor can call `RSETHPoolV2NBA.deposit` with an arbitrarily large ETH value repeatedly within the same day and receive proportionally large `wrsETH` mints with no ceiling. Because deposited ETH backs the minted `wrsETH` at the oracle rate, no direct value is lost under normal oracle conditions. This matches the allowed impact: **Low — Contract fails to deliver promised returns, but doesn't lose value.**

## Likelihood Explanation
No special role, no front-running, no oracle manipulation, and no privileged access is required. Any externally-owned account can call `deposit` with any ETH amount at any time the contract is unpaused. The condition is trivially and repeatably reachable by any depositor.

## Recommendation
Add the `limitDailyMint` mechanism to `RSETHPoolV2NBA` consistent with the other pool variants:
1. Introduce `dailyMintLimit`, `dailyMintAmount`, `lastMintDay`, and `startTimestamp` storage variables.
2. Implement the `limitDailyMint(uint256 amount)` modifier mirroring the logic in `RSETHPoolV2`.
3. Apply `limitDailyMint(msg.value)` to `deposit`.
4. Add a `reinitializer` function (and a `setDailyMintLimit` admin function) to set the initial limit and start timestamp, mirroring the pattern in `RSETHPoolV2.reinitialize`. [7](#0-6) 

## Proof of Concept
1. Deploy `RSETHPoolV2NBA` with a valid `wrsETH` and oracle.
2. Call `RSETHPoolV2NBA.deposit{value: 1000 ether}("")` — `wrsETH.mint(attacker, rsETHAmount)` executes with no cap check.
3. Immediately call `RSETHPoolV2NBA.deposit{value: 1000 ether}("")` again in the same block/day — succeeds again with no revert.
4. Repeat N times; total minted `wrsETH` is unbounded within a single day.
5. Differential: the identical sequence on `RSETHPoolV2` reverts with `DailyMintLimitExceeded` after the daily quota is exhausted. [8](#0-7)

```text
    modifier limitDailyMint(uint256 amount) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        // Calculate the amount of rsETH that will be minted
        (uint256 rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```
