### Title
TEE Attestation Validity Not Re-Verified at Signing/Key-Event Execution Time, Allowing Expired-TEE Nodes to Participate in Signing — (File: `crates/contract/src/lib.rs`)

---

### Summary

The contract checks TEE attestation **validity** (expiry, docker-image hash, OS measurements) only at `vote_new_parameters` time via `reverify_and_cleanup_participants`. All subsequent signing and key-event operations (`respond`, `vote_reshared`, `vote_pk`, `start_keygen_instance`, `start_reshare_instance`, `vote_abort_key_event_instance`) gate access through `assert_caller_is_attested_participant_and_protocol_active`, which calls `is_caller_an_attested_participant` — a function that only checks **attestation existence**, not validity. A node whose TEE attestation expires (or whose docker image is removed from the allowed list) between two `vote_new_parameters` calls retains full signing and key-event participation rights on-chain, breaking the TEE security invariant.

---

### Finding Description

`assert_caller_is_attested_participant_and_protocol_active` is the single gate for all node-facing mutating operations:

```rust

    let participants = self.protocol_state.active_participants();
    Self::assert_caller_is_signer();
    let attestation_check = self
        .tee_state
        .is_caller_an_attested_participant(participants);
    assert_matches::assert_matches!(
        attestation_check,
        Ok(()),
        "Caller must be an attested participant"
    );
}

`is_caller_an_attested_participant` only verifies that a stored entry exists and that the `account_id` / `account_public_key` fields match the signer. It never calls `reverify_participants`:

---

### Impact Explanation

The TEE model's core invariant is: **only nodes running inside a currently-trusted TEE environment may participate in signing and key events**. The desync breaks this invariant. A node whose attestation has expired — meaning its certificate is stale, its docker image has been rotated out of the allowed list, or its OS measurements are no longer accepted — can still:

- Submit signatures via `respond()`, completing signing rounds on-chain.
- Vote on resharing completion via `vote_reshared()`, advancing the key-event state machine.
- Vote on generated public keys via `vote_pk()`, influencing which key the network adopts.
- Start key-event instances via `start_keygen_instance()` / `start_reshare_instance()`.

Off-chain, such a node participates in the P2P MPC protocol (threshold signing, key resharing) while the contract accepts its on-chain submissions without detecting the expired TEE. This breaks the production safety invariant that all active signers are running in a currently-attested, trusted execution environment — matching the **Medium** impact class: *participant-state or contract execution-flow manipulation that breaks production safety/accounting invariants*.

---

### Likelihood Explanation

TEE attestation certificates have finite validity periods. In a long-running network where `vote_new_parameters` is infrequent (only triggered by governance), the window between consecutive TEE re-verifications can span weeks or months. Any node whose attestation expires during this window — whether due to certificate expiry or a docker-image rotation that removes the old hash from the allowed list — retains full on-chain signing rights until the next resharing or until `clean_invalid_attestations` is called externally. This is a normal operational condition, not an exotic edge case.

---

### Recommendation

Re-verify attestation validity inside `assert_caller_is_attested_participant_and_protocol_active` (or inside `is_caller_an_attested_participant`) by calling `reverify_participants` in addition to the existence check. If the re-verification fails, the call should be rejected with the same error path used for `AttestationNotFound`. This closes the desync window and ensures the TEE validity invariant is enforced at every signing and key-event boundary, not only at governance-vote time.

---

### Proof of Concept

1. Node A is a current participant; its TEE attestation is valid and stored in `tee_state.stored_attestations`.
2. Governance calls `vote_new_parameters` — `reverify_and_cleanup_participants` runs and confirms Node A's attestation is valid at this moment.
3. Time passes. Node A's attestation certificate expires, or its docker image hash is removed from `allowed_docker_image_hashes` via a `vote_code_hash` governance action.
4. No resharing occurs, so `clean_invalid_attestations` is not triggered.
5. Node A calls `respond(request, signature)` with a cryptographically valid signature produced by participating in the off-chain MPC round.
6. The contract executes `assert_caller_is_attested_participant_and_protocol_active` → `is_caller_an_attested_participant`. The stale entry is still present in `stored_attestations`; the existence check passes; `Ok(())` is returned.
7. The contract resolves the pending yield and delivers the signature to the user — signed by a node whose TEE is no longer trusted. [9](#0-8) [10](#0-9)

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub fn reverify_and_cleanup_participants(
        &mut self,
        participants: &Participants,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeValidationResult {
        self.allowed_docker_image_hashes
            .cleanup_expired_hashes(tee_upgrade_deadline_duration);

        let participants_with_valid_attestation: Vec<_> = participants
            .participants()
            .iter()
            .filter(|(_, _, participant_info)| {
                // Use the stored NodeId (keyed by TLS public key) so the real
                // `account_public_key` participates in re-verification. If
                // there is no stored attestation for this TLS key, the
                // participant is invalid.
                let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
                else {
                    return false;
                };

                let tee_status =
                    self.reverify_participants(&node_id, tee_upgrade_deadline_duration);

                matches!(tee_status, TeeQuoteStatus::Valid)
            })
            .cloned()
            .collect();

        if participants_with_valid_attestation.len() != participants.len() {
            let participants_with_valid_attestation =
                Participants::init(participants.next_id(), participants_with_valid_attestation);

            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            }
        } else {
            TeeValidationResult::Full
        }
    }

    pub fn clean_invalid_attestations(
        &mut self,
        tee_upgrade_deadline_duration: Duration,
        max_scan: usize,
    ) -> u32 {
        let has_invalid_attestation = |node_id: &NodeId| {
            !matches!(
                self.reverify_participants(node_id, tee_upgrade_deadline_duration),
                TeeQuoteStatus::Valid
            )
        };

        // Materialize candidates before any mutation to avoid iterator invalidation.
        let invalid_tls_keys: Vec<Ed25519PublicKey> = self
            .stored_attestations
            .iter()
            .take(max_scan)
            .filter(|(_, node_attestation)| has_invalid_attestation(&node_attestation.node_id))
            .map(|(tls_pk, _)| tls_pk.clone())
            .collect();

        let removed = u32::try_from(invalid_tls_keys.len())
            .expect("u32 should always be convertible from usize on wasm32");

        for tls_pk in invalid_tls_keys {
            self.stored_attestations.remove(&tls_pk);
        }
        removed
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
