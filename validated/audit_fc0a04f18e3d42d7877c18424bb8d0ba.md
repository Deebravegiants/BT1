### Title
Sub-transaction native script phase-1 validation bypass via top-level witness set cross-sharing — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Utxow.hs`)

---

### Summary

`dijkstraUtxowTransition` runs `validateFailedBabbageScripts` only against `topScriptHashesNeeded` (top-level scripts), while `babbageMissingScripts` is satisfied by `witnessScripts` aggregated across all levels. A sub-transaction's needed native script that is placed exclusively in the top-level witness set satisfies the aggregated presence check but is never phase-1 validated at either level, allowing the sub-transaction to spend a UTxO locked by a failing native script.

---

### Finding Description

**Step 1 — UTXOW-level aggregated presence check passes.**

In `dijkstraUtxowTransition`:

```haskell
let witnessScripts =
      Map.keysSet (tx ^. witsTxL . scriptTxWitsL)
        <> foldMap (Map.keysSet . (^. witsTxL . scriptTxWitsL)) subTxs
    ...
runTest $ Babbage.babbageMissingScripts pp allScriptHashesNeeded refScripts witnessScripts
```

`witnessScripts` is the union of script hashes from the top-level witness set and all sub-tx witness sets. `allScriptHashesNeeded` is the union of script hashes needed across all levels. If native script H is needed by the sub-tx and is placed in the **top-level** witness set, `babbageMissingScripts` passes because H ∈ `witnessScripts`. [1](#0-0) 

**Step 2 — UTXOW-level phase-1 validation does not check H.**

```haskell
runTest $ Babbage.validateFailedBabbageScripts tx scriptsProvided topScriptHashesNeeded
```

`topScriptHashesNeeded` contains only the top-level tx's needed script hashes. H is needed by the sub-tx, not the top-level tx, so H ∉ `topScriptHashesNeeded`. `validateFailedBabbageScripts` only fails a script if `scriptHash `Set.member` neededHashes`, so H is silently skipped. [2](#0-1) [3](#0-2) 

**Step 3 — Sub-tx's `scriptsProvided` does not contain H.**

`scriptsProvidedDijkstraStAnnTx` for a `StAnnTx SubTx era` returns `dsastScriptsProvided`, which is computed via `getBabbageScriptsProvided utxo subTx`:

```haskell
getBabbageScriptsProvided utxo tx = ScriptsProvided ans
  where
    ins = (txBody ^. referenceInputsTxBodyL) `Set.union` (txBody ^. inputsTxBodyL)
    ans = getReferenceScripts utxo ins `Map.union` (tx ^. witsTxL . scriptTxWitsL)
```

This includes only the sub-tx's own witness scripts and reference scripts from UTxO entries pointed to by the sub-tx's inputs. It does **not** include scripts from the top-level tx's witness set. Since H is only in the top-level witness set, H ∉ sub-tx's `scriptsProvided`. [4](#0-3) [5](#0-4) 

**Step 4 — SUBUTXOW-level phase-1 validation does not check H.**

```haskell
let scriptsProvided = scriptsProvidedStAnnTx stAnnTx  -- sub-tx's own scripts only
    ...
runTest $ Babbage.validateFailedBabbageScripts tx scriptsProvided scriptHashesNeeded
```

`validateFailedBabbageScripts` iterates over `scriptsProvided`. Since H ∉ sub-tx's `scriptsProvided`, the function never encounters H and never validates it. H is needed (`scriptHashesNeeded` includes H) but is absent from `scriptsProvided`, so the filter produces no failure. [6](#0-5) 

**Step 5 — No `babbageMissingScripts` check at SUBUTXOW level.**

`dijkstraSubUtxowTransition` has no call to `babbageMissingScripts`. The only script-presence enforcement is the aggregated check at UTXOW level, which already passed in Step 1. [7](#0-6) 

**Step 6 — SUBUTXO proceeds to update UTxO state.**

`dijkstraSubUtxoTransition` calls `Shelley.updateUTxOStateNoFees`, which removes the spent UTxO from the UTxO set without re-checking script authorization. The sub-tx's spend of the UTxO locked by H is committed. [8](#0-7)

### Impact Explanation

An unprivileged attacker can spend any UTxO locked by a native script by placing the script in the top-level witness set of a batch transaction and spending the UTxO from a sub-transaction. The native script is never executed. This constitutes a direct, unauthorized transfer of ADA or native assets — a Critical impact under the allowed scope.

---

### Likelihood Explanation

The attack requires only:
1. Knowledge of the target native script (scripts are public once submitted to the chain).
2. Construction of a valid batch transaction with a sub-tx spending the target UTxO.
3. Placing the script in the top-level witness set.

No privileged access, governance majority, or key compromise is required. The attack is fully constructible by any transaction author.

---

### Recommendation

`dijkstraUtxowTransition` must run `validateFailedBabbageScripts` against `allScriptHashesNeeded` (not just `topScriptHashesNeeded`), using a `scriptsProvided` map that correctly resolves each script hash to the level at which it was provided, and validates it against the tx at that level. Alternatively, `dijkstraSubUtxowTransition` must add a `babbageMissingScripts`-style check that ensures every script hash needed by the sub-tx is present in the sub-tx's own `scriptsProvided` (not just in the aggregated top-level set), so that `validateFailedBabbageScripts` at SUBUTXOW level can actually encounter and validate the script.

The root asymmetry to fix: the aggregated presence check (`babbageMissingScripts` at UTXOW) must not be satisfied by scripts that are not also validated at the level where they are needed.

---

### Proof of Concept

1. Create UTxO `u` locked by native script `H = RequireAnyOf []` (always fails). `hash(H) = h`.
2. Construct batch tx:
   - Top-level tx body: no inputs locked by scripts; witness set contains `{h ↦ H}`.
   - Sub-tx body: `inputs = {u}`, output redirecting funds to attacker address.
3. Submit. Trace:
   - UTXOW: `allScriptHashesNeeded = {h}`, `witnessScripts = {h}` → `babbageMissingScripts` passes.
   - UTXOW: `topScriptHashesNeeded = {}` → `validateFailedBabbageScripts` skips H.
   - SUBUTXOW: sub-tx `scriptsProvided = {}` (H not in sub-tx witness set, not a ref script) → `validateFailedBabbageScripts` skips H.
   - SUBUTXO: `updateUTxOStateNoFees` removes `u` from UTxO, adds attacker output.
4. Result: `u` is spent without `H` ever being evaluated. Funds transferred to attacker.

validateFailedBabbageScripts tx (ScriptsProvided scriptsProvided) neededHashes =
  let failedScripts =

          ( \scriptHash script ->

                Nothing -> False
