### Title
`try_sync` Mutates `l2_gas_price` Before Block-Info Validation, Leaving Corrupted Gas Price on Sync Failure — (`crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs`)

---

### Summary

`SequencerConsensusContext::try_sync` unconditionally overwrites `self.l2_gas_price` from the synced block's `next_l2_gas_price` field **before** performing block-info validation. When the subsequent validation fails (invalid timestamp or block number), the function returns `false` — signalling that sync did not occur — but the in-memory `l2_gas_price` has already been permanently mutated. Every subsequent proposal build and proposal validation uses this corrupted value, causing the validator to reject proposals whose `l2_gas_price_fri` was computed from the correct (pre-corruption) price.

---

### Finding Description

In `try_sync`:

```
self.l2_gas_price = max(                          // line 1056 — mutation
    sync_block.block_header_without_hash.next_l2_gas_price,
    VersionedConstants::latest_constants().min_gas_price,
);
// … then …
if !(block_number == height                        // line 1067 — validation
    && timestamp.0 >= last_block_timestamp
    && timestamp.0 <= now + self.config.static_config.block_timestamp_window_seconds)
{
    warn!(…);
    return false;                                  // line 1080 — returns false, but l2_gas_price already changed
}
```

The mutation at line 1056 is unconditional. The block-info guard at lines 1067–1080 is the only check that can cause the function to return `false` after the block has been fetched. If that guard fires, the function signals failure to the caller but leaves `self.l2_gas_price` permanently set to whatever `next_l2_gas_price` the rejected block carried. [1](#0-0) 

The corrupted `l2_gas_price` then flows into two critical paths:

**Validator path** — `validate_proposal` constructs `ProposalInitValidation` with `l2_gas_price_fri: self.l2_gas_price` and `is_proposal_init_valid` enforces an exact equality check:

```rust
if !(init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri …) {
    return Err(ValidateProposalError::InvalidProposalInit(…));
}
``` [2](#0-1) [3](#0-2) 

**Proposer path** — `build_proposal` sets `l2_gas_price_fri: args.l2_gas_price` directly in the outgoing `ProposalInit`: [4](#0-3) 

A node whose `l2_gas_price` was silently corrupted by a failed `try_sync` will either (a) emit proposals with the wrong `l2_gas_price_fri` that honest validators reject, or (b) reject honest proposals whose `l2_gas_price_fri` was computed from the correct price.

---

### Impact Explanation

**High — proposal validation rejects valid proposals before sequencing.**

A validator node that has undergone a failed `try_sync` will hold a `l2_gas_price` that differs from the network-agreed value. The exact-equality check in `is_proposal_init_valid` then causes it to reject every subsequent proposal from an honest proposer. This maps to: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing"* — the proposal-validation gate is the admission point for consensus-level block content.

If the corrupted value propagates to the proposer side instead, the proposer broadcasts a `ProposalInit` with a wrong `l2_gas_price_fri`, which honest validators reject, stalling consensus at that height.

---

### Likelihood Explanation

`try_sync` is called by the consensus manager on every height where the node has not yet reached a local decision. The trigger requires the state-sync client to return a block whose `next_l2_gas_price` differs from the current value **and** whose timestamp or block number fails the guard. This can occur:

1. **Clock skew** — a legitimately committed block whose timestamp is slightly ahead of the local wall clock (within the `block_timestamp_window_seconds` tolerance on the proposer side but outside it on the validator side) causes `try_sync` to return `false` after mutating `l2_gas_price`.
2. **Malicious or faulty P2P sync peer** — a peer that serves a block with a crafted `next_l2_gas_price` and an out-of-window timestamp can silently corrupt the field on any node that calls `try_sync`.

Neither path requires any privileged on-chain operation.

---

### Recommendation

Move the `l2_gas_price` assignment to **after** all validation passes, or stage it in a local variable and commit only on success:

```rust
// Validate first
if !(block_number == height
    && timestamp.0 >= last_block_timestamp
    && timestamp.0 <= now + self.config.static_config.block_timestamp_window_seconds)
{
    warn!(…);
    return false;
}
// Only mutate state after validation succeeds
self.l2_gas_price = max(
    sync_block.block_header_without_hash.next_l2_gas_price,
    VersionedConstants::latest_constants().min_gas_price,
);
```

This mirrors the explicit-state-change principle recommended in the original report: state should only be updated when the operation is known to succeed.

---

### Proof of Concept

1. Node V is a validator at height H. `try_sync(H)` is called.
2. The state-sync client returns a block for height H with `next_l2_gas_price = P_bad` and `timestamp = now + window_seconds + 1` (one second past the allowed window).
3. `try_sync` executes line 1056: `self.l2_gas_price = max(P_bad, min_price)`. Node V's gas price is now `P_bad`.
4. The timestamp check at line 1067 fails. `try_sync` returns `false`. Consensus continues.
5. The honest proposer P builds a proposal using its own correct `l2_gas_price = P_good` and broadcasts `ProposalInit { l2_gas_price_fri: P_good, … }`.
6. Node V constructs `ProposalInitValidation { l2_gas_price_fri: P_bad, … }` and calls `is_proposal_init_valid`.
7. The equality check `P_good == P_bad` fails → `ValidateProposalError::InvalidProposalInit` is returned.
8. Node V sends a `nil` prevote. If a quorum of validators are in the same state, consensus stalls at height H. [5](#0-4) [6](#0-5) [3](#0-2)

    async fn try_sync(&mut self, height: BlockNumber) -> bool {
        let sync_block = match self.deps.state_sync_client.get_block(height).await {
            Err(StateSyncClientError::StateSyncError(StateSyncError::BlockNotFound(_))) => {
                return false;
            }
            Err(e) => {
                error!("Sync returned an error: {e:?}");
                return false;
            }
            Ok(block) => block,
        };
        // May be default for blocks older than 0.14.0, ensure min gas price is met.
        self.l2_gas_price = max(
            sync_block.block_header_without_hash.next_l2_gas_price,
            VersionedConstants::latest_constants().min_gas_price,
        );

        // TODO(Asmaa): validate starknet_version and parent_hash when they are stored.
        let block_number = sync_block.block_header_without_hash.block_number;
        let timestamp = sync_block.block_header_without_hash.timestamp;
        let last_block_timestamp =
            self.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
        let now: u64 = self.deps.clock.unix_now();
        if !(block_number == height
            && timestamp.0 >= last_block_timestamp
            && timestamp.0 <= now + self.config.static_config.block_timestamp_window_seconds)
        {
            warn!(
                "Invalid block info: expected block number {}, got {}, expected timestamp range \
                 [{}, {}], got {}",
                height,
                block_number,
                last_block_timestamp,
                now + self.config.static_config.block_timestamp_window_seconds,
                timestamp.0,
            );
            return false;
        }
        self.record_fee_proposal(height, sync_block.block_header_without_hash.fee_proposal_fri);
        self.previous_proposal_init =
            Some(previous_proposal_init_from_block_header(&sync_block.block_header_without_hash));
        self.interrupt_active_proposal().await;

        info!(
            "Adding sync block to Batcher for height {}",
            sync_block.block_header_without_hash.block_number,
        );
        if let Err(e) = self.deps.batcher.add_sync_block(sync_block).await {
            error!("Failed to add sync block to Batcher: {e:?}");
            return false;
        }

        true
    }
