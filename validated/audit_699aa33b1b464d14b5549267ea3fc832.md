### Title
`StakePoolVoter` Witness Not Required in `voterWitnesses`, Allowing Unauthorized Governance Votes - (File: `eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs`)

---

### Summary

The `voterWitnesses` function in `eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs` silently discards the `KeyHash StakePool` for `StakePoolVoter` entries and returns `Nothing`, meaning no key-hash witness is ever added to the required-witness set for stake-pool votes. Because `voterWitnesses` is the sole mechanism by which voter key-hash witnesses are collected into the UTXOW required-witness check, any transaction can include a voting procedure claiming to be from any registered stake pool without providing a signature from that pool's operator key.

---

### Finding Description

`getConwayWitsVKeyNeeded` assembles the complete set of key-hash witnesses that a Conway transaction must supply:

```haskell
getConwayWitsVKeyNeeded utxo txBody =
  getShelleyWitsVKeyNeededNoGov utxo txBody
    `Set.union` Set.map asWitness (txBody ^. reqSignerHashesTxBodyG)
    `Set.union` voterWitnesses txBody
``` [1](#0-0) 

The `voterWitnesses` helper iterates over every `VotingProcedure` in the transaction body and attempts to add the voter's key hash to the required-witness set:

```haskell
voterWitnesses txb =
  Map.foldrWithKey' accum mempty (unVotingProcedures (txb ^. votingProceduresTxBodyL))
  where
    accum voter _ khs =
      maybe khs (`Set.insert` khs) $
        case voter of
          CommitteeVoter cred -> credKeyHashWitness cred
          DRepVoter cred      -> credKeyHashWitness cred
          StakePoolVoter _    -> Nothing          -- ← key hash silently discarded
``` [2](#0-1) 

For `CommitteeVoter` and `DRepVoter`, `credKeyHashWitness` extracts the key hash from the credential and returns `Just (asWitness kh)`. For `StakePoolVoter kh`, the pool's `KeyHash StakePool` is bound to `_` and discarded; `Nothing` is returned. The `maybe khs (Set.insert khs)` wrapper therefore leaves the accumulated set unchanged, so the pool operator's key hash is never added to the required-witness set.

The result flows directly into `validateNeededWitnesses` inside the UTXOW transition rule:

```haskell
validateNeededWitnesses witsKeyHashes certState utxo txBody =
  let needed = getWitsVKeyNeeded certState utxo txBody
      missingWitnesses = Set.difference needed witsKeyHashes
   in failureOnNonEmptySet missingWitnesses MissingVKeyWitnessesUTXOW
``` [3](#0-2) 

Because the pool's key hash is never in `needed`, `missingWitnesses` is always empty for stake-pool votes, and the check always passes regardless of whether the pool operator signed the transaction.

The correct fix is:

```haskell
StakePoolVoter kh -> Just (asWitness kh)
```

This is structurally identical to the external report's root cause: a conditional guard checks the wrong entity's state (here, no entity at all instead of the pool operator), so the guard is always satisfied and the intended authorization is never enforced.

---

### Impact Explanation

Stake-pool operators vote in Conway governance using their cold key (`KeyHash StakePool`). Their votes are weighted by the live stake delegated to the pool and count toward ratification thresholds for governance actions including treasury withdrawals, protocol-parameter changes, hard-fork initiations, committee updates, and constitutional changes. Because no witness from the pool operator is required, an unprivileged attacker can submit a transaction containing `VotingProcedure` entries for arbitrarily many registered stake pools, casting "Yes" (or any other) votes on any open governance action without possessing any pool operator key. If the attacker targets enough pools to collectively exceed the applicable ratification threshold, a governance action of their choosing—including a treasury withdrawal to an attacker-controlled address or a malicious protocol-parameter update—can be ratified and enacted through an otherwise-valid ledger state transition.

This matches the allowed Critical impact: **Unauthorized governance, treasury, protocol-parameter, committee, constitution, or hard-fork action is enacted.**

---

### Likelihood Explanation

The entry path requires only the ability to submit a normal transaction. No privileged access, no key compromise, no consensus majority, and no social engineering are needed. The attacker must know the `KeyHash StakePool` values of registered pools (these are public on-chain data) and must submit votes before the governance action's voting deadline. The attack is fully deterministic and repeatable. The only practical constraint is that the attacker must target enough pools to meet the ratification threshold, but because pool key hashes are public and the transaction can include votes for many pools simultaneously, this is a realistic scenario.

---

### Recommendation

Change the `StakePoolVoter` branch in `voterWitnesses` from:

```haskell
StakePoolVoter _ -> Nothing
```

to:

```haskell
StakePoolVoter kh -> Just (asWitness kh)
``` [4](#0-3) 

`asWitness :: HasKeyRole f => f r -> f Witness` is already used for the same coercion elsewhere in the file (e.g., `Set.map asWitness (txBody ^. reqSignerHashesTxBodyG)`), so no new imports are required. The same fix must be verified in the Dijkstra era, which reuses `getConwayWitsVKeyNeeded` directly: [5](#0-4)

### Proof of Concept

1. Alice registers a stake pool with cold key `kh_pool` and accumulates delegated stake.
2. Bob (the attacker, who does not hold `kh_pool`) constructs a transaction `tx` containing:
   - A `VotingProcedure` with voter `StakePoolVoter kh_pool` voting `VoteYes` on a live `TreasuryWithdrawals` governance action that sends funds to Bob's address.
   - No witness from `kh_pool` in `tx ^. witsTxL . addrTxWitsL`.
3. Bob submits `tx`. The UTXOW rule calls `getConwayWitsVKeyNeeded`, which calls `voterWitnesses`. `voterWitnesses` returns `Nothing` for `StakePoolVoter kh_pool`, so `kh_pool` is absent from `needed`. `validateNeededWitnesses` finds no missing witnesses and the transaction is accepted.
4. Alice's pool's vote is now recorded as `VoteYes` without Alice's knowledge or consent. If enough pools are similarly impersonated to meet the stake-pool ratification threshold, the treasury withdrawal is ratified and enacted, transferring ADA to Bob's address.
