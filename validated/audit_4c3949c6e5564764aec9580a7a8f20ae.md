Audit Report

## Title
Daily Fee Mint Limit Revert Permanently Freezes Unclaimed Yield and Stales Cross-Chain Rate βÄî (`contracts/LRTOracle.sol`, `contracts/cross-chain/RSETHRateProvider.sol`)

## Summary

`LRTOracle._updateRsETHPrice()` calls `_checkAndUpdateDailyFeeMintLimit()` at line 303 before writing `rsETHPrice` at line 313. When the fee amount to mint exceeds `maxFeeMintAmountPerDay`, the call reverts with `DailyFeeMintLimitExceeded`, leaving `rsETHPrice` permanently stale. `RSETHRateProvider.getLatestRate()` reads this stale storage value directly, and `CrossChainRateProvider.updateRate()` broadcasts it cross-chain, freezing unclaimed yield for all downstream consumers.

## Finding Description

`updateRSETHPrice()` is an unrestricted public function callable by any address when the contract is not paused: [1](#0-0) 

Inside `_updateRsETHPrice()`, when TVL has grown, a protocol fee is computed and `_checkAndUpdateDailyFeeMintLimit` is called at line 303 βÄî **before** the price write at line 313: [2](#0-1) 

`_checkAndUpdateDailyFeeMintLimit` hard-reverts when the cumulative fee amount would exceed `maxFeeMintAmountPerDay`: [3](#0-2) 

Because the revert unwinds the entire call, `rsETHPrice = newRsETHPrice` (line 313) is never executed. The storage variable `rsETHPrice` retains its pre-yield value.

`RSETHRateProvider.getLatestRate()` reads `rsETHPrice` directly from storage: [4](#0-3) 

`CrossChainRateProvider.updateRate()` then fetches this stale value and broadcasts it cross-chain via LayerZero: [5](#0-4) 

**Cascading self-reinforcement:** Because `rsETHPrice` is stale, `previousTVL = rsethSupply Ο½ rsETHPrice` is underestimated on every subsequent call: [6](#0-5) 

This inflates `rewardAmount`, inflates `protocolFeeInETH`, and inflates `rsethAmountToMintAsProtocolFee` further. Each new period starts with an even larger fee amount, making the limit even harder to satisfy βÄî the freeze compounds across periods without admin action.

`updateRSETHPriceAsManager()` also calls `_updateRsETHPrice()` and is subject to the same revert βÄî the manager cannot bypass the limit through this path either: [7](#0-6) 

## Impact Explanation

**Medium βÄî Permanent freezing of unclaimed yield.** `rsETHPrice` is frozen at its pre-yield value. All yield accrued since the last successful update is invisible to the cross-chain rate. `RSETHRateProvider.getLatestRate()` / `getRate()` return the stale rate to any integrator. The freeze persists across daily periods due to the cascading inflation of the fee amount, until the manager calls `setMaxFeeMintAmountPerDay()` with a sufficiently large value.

## Likelihood Explanation

`maxFeeMintAmountPerDay` is a live, intentionally configurable parameter. Any non-zero setting creates the precondition. A single large deposit or EigenLayer reward accrual can push the fee amount over the remaining daily limit. No privileged role is needed to trigger the revert βÄî any public caller (or a keeper bot) calling `updateRSETHPrice()` will hit it. The cascading inflation of the fee amount means the window of exploitability grows over time, not shrinks.

## Recommendation

Do not revert inside `_updateRsETHPrice()` when the daily fee limit is exceeded. Instead, either:

1. **Cap the fee mint** to the remaining daily allowance and proceed with the price update (partial fee collection is better than no price update), or
2. **Skip the fee mint** for that call (emit an event) and still write `rsETHPrice`, so the price is always updated regardless of fee-minting state.

The price update and the fee mint are logically independent; coupling them with a hard revert is the root cause.

## Proof of Concept

```solidity
function test_dailyLimitFreezesYield() public {
    // 1. Set maxFeeMintAmountPerDay to 1 wei
    vm.prank(manager);
    lrtOracle.setMaxFeeMintAmountPerDay(1);

    // 2. Simulate TVL increase so protocolFeeInETH > 0
    _simulateTVLIncrease(1000 ether);

    // 3. Record current rsETHPrice
    uint256 priceBefore = lrtOracle.rsETHPrice();

    // 4. Public caller triggers updateRSETHPrice() βÄî expect DailyFeeMintLimitExceeded
    vm.expectRevert(
        abi.encodeWithSelector(ILRTOracle.DailyFeeMintLimitExceeded.selector)
    );
    lrtOracle.updateRSETHPrice();

    // 5. Assert rsETHPrice is unchanged (stale)
    assertEq(lrtOracle.rsETHPrice(), priceBefore);

    // 6. Assert RSETHRateProvider returns the stale value
    assertEq(rsETHRateProvider.getLatestRate(), priceBefore);

    // 7. Warp to next period βÄî cascading: fee amount is now even larger
    vm.warp(block.timestamp + 1 days);
    vm.expectRevert(
        abi.encodeWithSelector(ILRTOracle.DailyFeeMintLimitExceeded.selector)
    );
    lrtOracle.updateRSETHPrice(); // still reverts βÄî freeze persists
}
```
