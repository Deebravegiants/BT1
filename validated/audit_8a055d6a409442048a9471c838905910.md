### Title
Unvalidated `builder` Field in `ProposalInit` Allows a Legitimate Proposer to Redirect Block Fees and Corrupt the Sequencer Address in Committed Block Headers — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates many fields of `ProposalInit` (height, L1/L2 gas prices, starknet version, timestamp, `version_constant_commitment`, `fee_proposal_fri`) but never checks `init_proposed.builder` against any locally-trusted expected value. The `builder` field flows directly into `sequencer_address` in the `BlockInfo` passed to the batcher, which determines where transaction fees are collected during execution and what value is stored in the committed block header. A legitimate proposer (any validator whose turn it is to propose) can set `builder` to an arbitrary address, causing fees to be redirected and the block header to carry a wrong sequencer address — both of which are committed to the chain and accepted by all validators.

### Finding Description

**Root cause.** `ProposalInit.builder` is a proposer-supplied field that is never validated by the receiving validator node.

`ProposalInitValidation` (the struct that carries the validator's locally-trusted reference values) has no `expected_builder` field: [1](#0-0) 

`is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`, but contains no check on `init_proposed.builder`: [2](#0-1) 

**Downstream use.** `convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in the `BlockInfo` that is forwarded to the batcher: [3](#0-2) 

This `BlockInfo` is passed to `batcher.validate_block` (and `batcher.propose_block`) as `ValidateBlockInput.block_info`: [4](#0-3) 

The same `init.builder` is also stored as the `sequencer` field in `BlockHeaderWithoutHash` that is committed to state sync: [5](#0-4) 

**Why the `ProposalFinMismatch` check does not catch this.** The validator re-executes the block using the proposer-supplied `builder` value (via `convert_to_sn_api_block_info`). The batcher therefore computes `PartialBlockHash` with the malicious `sequencer_address`. The proposer also computed the commitment with the same malicious `builder`. Both sides agree, so the `built_block != received_fin.proposal_commitment` check passes: [6](#0-5) 

**Why the `proposer` check does not help.** The consensus layer validates `init.proposer` against the committee: [7](#0-6) 

But `init.builder` is a separate, independent field that is never cross-checked against any committee-derived or locally-configured expected value.

**Exploit path.**
1. Attacker is a legitimate validator whose turn it is to propose at height H, round R.
2. Attacker constructs a `ProposalInit` with `builder = attacker_address` (any address they control).
3. Attacker broadcasts the proposal. All validators call `is_proposal_init_valid`, which passes because `builder` is not checked.
4. Each validator calls `convert_to_sn_api_block_info(init)` → `sequencer_address = attacker_address` → passes this to `batcher.validate_block`.
5. The batcher executes all transactions with `sequencer_address = attacker_address`, so all fee transfers go to the attacker's address.
6. The batcher computes `PartialBlockHash` with `sequencer_address = attacker_address`. The proposer's commitment matches. `ProposalFinMismatch` does not trigger.
7. Consensus reaches decision. `update_state_sync_with_new_block` stores `sequencer = attacker_address` in `BlockHeaderWithoutHash`.
8. The block is committed with the wrong sequencer address and all fees redirected.

### Impact Explanation

- **Incorrect fee / balance with economic impact (Critical):** All transaction fees in the block are transferred to the attacker-controlled `builder` address instead of the legitimate sequencer. This is a direct theft of sequencer revenue.
- **Wrong state committed (Critical):** The `sequencer` field in the committed `BlockHeaderWithoutHash` is the attacker's address. Since `sequencer_address` is part of Starknet's block hash computation, the `PartialBlockHash` (and therefore the `ProposalCommitment`) is computed over the wrong sequencer address, and this wrong value is permanently committed to the chain and propagated to all nodes.
- **Wrong RPC view (High):** Any RPC query for the block header will return the attacker's address as the sequencer.

### Likelihood Explanation

Any validator in the active committee can trigger this during their proposer turn. No special privilege beyond being a staking validator is required. The attack is silent — it produces no error, no log warning, and no consensus disagreement, because all validators independently re-derive the same wrong commitment from the same unvalidated `builder` value.

### Recommendation

Add `expected_builder: ContractAddress` to `ProposalInitValidation`, populated from the node's static config (`self.config.static_config.builder_address`) when constructing the validation criteria in `validate_proposal` in `sequencer_consensus_context.rs`. Then add a check in `is_proposal_init_valid`:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
``` [8](#0-7) 

### Proof of Concept

1. Node A is the legitimate proposer for height 5, round 0. Its `builder_address` in static config is `0xLEGIT`.
2. Node A constructs `ProposalInit { builder: 0xATTACKER, ..rest_valid }`.
3. Node A broadcasts the proposal. All peer validators receive it.
4. Each peer calls `is_proposal_init_valid` — passes (no `builder` check).
5. Each peer calls `convert_to_sn_api_block_info` → `sequencer_address = 0xATTACKER` → `batcher.validate_block(block_info)`.
6. Batcher executes 100 transactions, each paying 1 STRK fee → 100 STRK transferred to `0xATTACKER`.
7. Batcher returns `PartialBlockHash` computed with `sequencer_address = 0xATTACKER`. Node A's `ProposalFin.proposal_commitment` was also computed with `0xATTACKER`. They match.
8. Consensus commits the block. `BlockHeaderWithoutHash.sequencer = 0xATTACKER` is stored in state sync.
9. 100 STRK is permanently at `0xATTACKER`. The block header on-chain records `0xATTACKER` as the sequencer.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
