### Title
Single Byzantine Participant Can Drop All Pending Requests via Unvalidated IndexerHeightMessage — (`crates/node/src/network/indexer_heights.rs`, `crates/node/src/requests/queue.rs`)

---

### Summary

A single TLS-authenticated, bidirectionally-connected Byzantine participant can send an `IndexerHeightMessage` with an arbitrarily inflated height value. Because `set_height` applies no upper-bound validation and is a monotonic ratchet, and because `eligible_leaders_and_maximum_height` derives `maximum_height` as the raw `max()` over all alive participants' stored heights, the resulting `cutoff_block` becomes large enough to classify every in-flight request as `RequestStatus::Drop(RequestTimedOut)`, permanently removing them from the queue.

---

### Finding Description

**`set_height` — no validation, monotonic ratchet** [1](#0-0) 

The function accepts any `u64` value and stores it if it exceeds the current value. There is no cap, no sanity check against the local indexer height, and no check against the on-chain block height. Once a large value is stored it can never decrease.

**`eligible_leaders_and_maximum_height` — raw `max()` over alive participants** [2](#0-1) 

`maximum_height` is the unclamped maximum of every alive participant's stored height. If the Byzantine node is in `alive_participants` with a fabricated height `H`, then `maximum_height = H`.

**`cutoff_block` derivation** [3](#0-2) 

```
cutoff_block = (H.saturating_sub(200) + 1)
```

With `REQUEST_EXPIRATION_BLOCKS = 200`: [4](#0-3) 

If `H = current_block_height + 200`, then `cutoff_block = current_block_height + 1`. Every request indexed at or below `current_block_height` satisfies `is_older_than(cutoff_block)` and is dropped.

**`is_older_than` — the drop trigger** [5](#0-4) 

**`alive_participants` inclusion — the precondition** [6](#0-5) 

A peer is included in `alive_participants` when `my_height <= peer_height + MAX_INDEXER_HEIGHT_DIFF` AND bidirectionally connected. If the Byzantine node sends `H = current_height + 200`, the condition is trivially satisfied for any non-negative `MAX_INDEXER_HEIGHT_DIFF`. The node does **not** need to send `u64::MAX` — any value exceeding `current_block_height + 199` is sufficient and avoids any potential overflow in the alive-participants check.

**Note on the `u64::MAX` specific claim:** If `MAX_INDEXER_HEIGHT_DIFF > 0`, sending exactly `u64::MAX` causes `peer_height + MAX_INDEXER_HEIGHT_DIFF` to wrap in release mode, potentially excluding the Byzantine node from `alive_participants`. The attack is more reliably executed with `H = current_block_height + 200` (or any moderately inflated value), which avoids overflow entirely.

**`STALE_PARTICIPANT_THRESHOLD` does not protect against drops** [7](#0-6) 

The staleness filter only affects `eligible_leaders`. The `is_older_than(cutoff_block)` check runs **before** the leader check in `process()`: [8](#0-7) 

Requests are dropped before the leader check is ever reached.

**Permanent ratchet effect**

Once `set_height` stores `H`, it cannot be lowered. Even if the Byzantine node disconnects immediately after sending the message, the stored height persists. Any subsequent reconnection with a real height `< H` will not update the tracker.

---

### Impact Explanation

All pending `SignatureRequest`, `CKDRequest`, and `VerifyForeignTxRequest` entries in the queue are permanently removed with `DropReason::RequestTimedOut`. Users receive signing failures. The on-chain yield-resume callbacks will eventually time out, but the MPC nodes will not attempt to fulfill them — the requests are gone from the queue and will not be retried. This breaks the request-lifecycle invariant that requests should only be discarded after the network has genuinely advanced past their expiration window.

---

### Likelihood Explanation

The precondition is one authenticated participant behaving maliciously — strictly below the signing threshold. No threshold collusion is required. The attack is a single message. The `IndexerHeightMessage` path has no rate limiting or height-bound validation. The monotonic ratchet means a single message is sufficient; the attacker does not need to maintain a persistent connection.

---

### Recommendation

1. **Clamp incoming heights**: In `set_height`, reject any height that exceeds the local node's own indexer height by more than `REQUEST_EXPIRATION_BLOCKS` (or a configurable bound).
2. **Use median instead of max**: Derive `maximum_height` from the median (or a threshold-weighted percentile) of alive participants' heights rather than the raw maximum, so a single outlier cannot dominate.
3. **Validate in `run_receive_message`**: Before calling `set_height` for a peer's reported height, compare it against the local height and drop the message if it is implausibly far ahead.

### Proof of Concept

```rust
// Deterministic unit test sketch using TestNetworkAPI
let (mut pending_requests, mut setup) = TestSetup::new();

// Index a request at a normal block height (~100)
let req = setup.add_request_leader();
setup.update(&mut pending_requests);

// Byzantine participant 0 reports a height 200 blocks ahead
// (participant 0 is bidirectionally connected and in alive_participants)
let byzantine = setup.participant_ids[0];
setup.network_api.set_height(byzantine, 100 + REQUEST_EXPIRATION_BLOCKS);
// All other participants remain at height 100

// get_requests_to_attempt computes:
//   maximum_height = 300 (from Byzantine node)
//   cutoff_block   = 300 - 200 + 1 = 101
//   request.block_height = 100 < 101 → Drop(RequestTimedOut)
let to_attempt = pending_requests.get_requests_to_attempt();
assert_eq!(to_attempt.len(), 0, "all requests were dropped by Byzantine height claim");
```

    pub(super) fn eligible_leaders_and_maximum_height(&self) -> (HashSet<ParticipantId>, u64) {
        // Collect the indexer heights and alive participants. Calculate maximum available height
        // from the alive nodes. Then, filter out the participants that are not alive or are too
        // stale.
        let indexer_heights = self.network_api.indexer_heights();
        let alive_participants = self.network_api.alive_participants();
        let maximum_height = alive_participants
            .iter()
            .map(|p| indexer_heights.get(p).copied().unwrap_or(0))
            .max()
            .unwrap_or(0);
        let eligible_leaders = self
            .all_participants
            .iter()
            .filter(|p| {
                alive_participants.contains(p)
                    && indexer_heights.get(p).copied().unwrap_or(0) + STALE_PARTICIPANT_THRESHOLD
                        >= maximum_height
            })
            .copied()
            .collect::<HashSet<_>>();
        (eligible_leaders, maximum_height)
    }

        let my_height = *indexer_heights.get(&self.my_participant_id()).unwrap_or(&0);
        for participant in self.all_participant_ids() {

                    .connectivity(participant)
                    .is_bidirectionally_connected()

                result.push(participant);
            }
        }
        result.push(self.my_participant_id());
