### Title
Unguarded `condition.at("rrrff")` in `get_metadata_and_phs` crashes wallet NFT sync on memo-less CREATE_COIN — (`chia/wallet/nft_wallet/nft_puzzle_utils.py`)

### Summary
`get_metadata_and_phs` unconditionally navigates the CLVM path `rrrff` on every CREATE_COIN condition with amount=1. When the memos list is absent, the path traversal calls `.first()` on nil, raising `EvalError`. Because the CLVM NFT puzzle does not enforce memo presence at the consensus layer, an attacker who owns an NFT can craft a valid on-chain spend to a victim's address with no memos, permanently preventing the victim's wallet from recording the child NFT coin.

### Finding Description
In `nft_puzzle_utils.py` line 256:
```python
memo = bytes32(condition.at("rrrff").as_atom())
```
The path `rrrff` assumes the structure `(51 puzzle_hash amount (memo . rest))`. When memos are absent, the structure is `(51 puzzle_hash amount)` and `rrr` resolves to `nil` (an atom). The subsequent `f` step calls `.first()` on an atom, which raises `clvm.EvalError`. No try/except wraps this call, and no pre-check verifies that `condition.at("rrr")` is a pair before descending further.

### Impact Explanation
The `EvalError` propagates out of `get_metadata_and_phs` into `puzzle_solution_received`. The child NFT coin is never written to the wallet database. Because the on-chain spend is valid and already confirmed, the wallet cannot recover by replaying the block — every subsequent sync attempt will re-raise the same exception for the same spend. The NFT is permanently lost from the victim's wallet state.

### Likelihood Explanation
Any NFT owner can craft a spend omitting memos. The CLVM ownership layer and transfer program enforce royalties and puzzle-hash correctness, not memo presence. The `CreateCoin` condition type itself marks memos as optional. The attack requires only that the attacker own an NFT and submit a spend to the victim's address — no key compromise or privileged access needed.

### Recommendation
Guard the memo access before navigating `rrrff`:
```python
memos_node = condition.at("rrr")
if memos_node == Program.NIL or not memos_node.listp():
    continue  # or log and skip
memo_atom = memos_node.first().first().as_atom()
if len(memo_atom) != 32:
    continue
memo = bytes32(memo_atom)
```
Alternatively, wrap the block in `try/except (EvalError, ValueError)` and skip conditions with malformed or absent memos.

```python
    def at(self, position: str) -> Program:
        """
        Take a string of only `f` and `r` characters and follow the corresponding path.

        Example:

        `assert Program.to(17) == Program.to([10, 20, 30, [15, 17], 40, 50]).at("rrrfrf")`

        """
        v = self
        for c in position.lower():
            if c == "f":
                v = v.first()
            elif c == "r":
                v = v.rest()
            else:
                raise ValueError(f"`at` got illegal character `{c}`. Only `f` & `r` allowed")
        return v
```

```python
class CreateCoin(Condition):
    puzzle_hash: bytes32
    amount: uint64
    memos: list[bytes] | None = None
    memo_blob: Program | None = None

    def __post_init__(self) -> None:
        if self.memos is not None and self.memo_blob is not None:
            raise ValueError("Cannot have both memos and memo_blob")
        return super().__post_init__()

    def to_program(self) -> Program:
        condition_args = [ConditionOpcode.CREATE_COIN, self.puzzle_hash, self.amount]
        if self.memos is not None:
            condition_args.append(self.memos)
        elif self.memo_blob is not None:
            condition_args.append(self.memo_blob)
        condition: Program = Program.to(condition_args)
        return condition
```
