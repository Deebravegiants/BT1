### Title
Unrestricted `sendETHToL1ViaBridge` Enables Block Stuffing on Unichain — (`contracts/bridges/UnichainMessenger.sol`)

### Summary
`UnichainMessenger.sendETHToL1ViaBridge` has no access control and no minimum value threshold. Any address can call it with 1 wei repeatedly, initiating many low-cost bridge operations that collectively saturate Unichain block space and delay the `BRIDGER_ROLE`'s legitimate `bridgeAssetsViaNativeBridge` transaction.

### Finding Description
`UnichainMessenger.sendETHToL1ViaBridge` is a fully public function with a single guard: `msg.value == value`. [1](#0-0) 

There is no `onlyRole`, no minimum ETH threshold, and no per-sender rate limit. Compare this to `LineaMessenger`, which at least enforces `value > minimumFeeInWei`: [2](#0-1) 

The legitimate caller — `RSETHPoolNoWrapper.bridgeAssetsViaNativeBridge()` — is correctly gated behind `onlyRole(BRIDGER_ROLE)`: [3](#0-2) 

But the messenger itself enforces nothing. An attacker bypasses the pool entirely and calls the messenger directly.

### Impact Explanation
Each attacker call with 1 wei triggers `IUnichainMessenger(l2bridge).bridgeETHTo{value: 1}(...)` with `DEFAULT_GAS_LIMIT = 200_000`: [4](#0-3) 

On Unichain (OP Stack, ~30M block gas limit), each call consumes ~50k–100k gas. An attacker can fill a block with ~300–600 such calls. During the stuffing window, the `BRIDGER_ROLE`'s `bridgeAssetsViaNativeBridge` transaction is excluded, temporarily freezing the ETH held in `RSETHPoolNoWrapper` that cannot be bridged to `L1Vault`.

**Impact: Low — Block stuffing.**

### Likelihood Explanation
- The function is public with no role check.
- Cost per call is 1 wei + gas; on Unichain L2, gas is cheap.
- No protocol-level defense exists in the messenger contract.
- The same pattern exists in `BaseMessenger`, `OptimismMessenger`, `ArbitrumMessenger`, and `ScrollMessenger` — all lack access control on `sendETHToL1ViaBridge`. [5](#0-4) [6](#0-5) 

### Recommendation
Add an access control modifier (e.g., `onlyRole(BRIDGER_ROLE)`) to `sendETHToL1ViaBridge` in all messenger contracts, or enforce a meaningful minimum ETH threshold (similar to `LineaMessenger`'s `minimumFeeInWei` check) to make spam economically infeasible.

### Proof of Concept
```solidity
// Fuzz test: call sendETHToL1ViaBridge with value in [1, 1e9 wei] across N calls
// Assert total gas consumed exceeds Unichain block gas limit before BRIDGER_ROLE tx is included
for (uint i = 0; i < N; i++) {
    unichainMessenger.sendETHToL1ViaBridge{value: 1}(l2bridge, target, 1);
}
// BRIDGER_ROLE's bridgeAssetsViaNativeBridge() is excluded from the block
```
