### Title
Single Failing Asset Price Oracle Permanently Blocks `updateRSETHPrice()` and Freezes Protocol Fee Minting — (`File: contracts/LRTOracle.sol`)

### Summary

`LRTOracle._getTotalEthInProtocol()` iterates over every supported asset and calls `getAssetPrice(asset)` for each one. If any single asset's price oracle reverts (e.g., the oracle contract is paused, deprecated, or otherwise broken), the entire `updateRSETHPrice()` call reverts. No error-handling or skip logic exists. This is the direct structural analog of the Ion Protocol withdrawal-queue bug: iterating over a list and allowing one failing element to abort the whole operation.

---

### Finding Description

`LRTOracle._getTotalEthInProtocol()` is the private helper that computes the total ETH value of all protocol assets. It is called unconditionally by `_updateRsETHPrice()`, which is in turn called by the public `updateRSETHPrice()`:

```solidity
// contracts/LRTOracle.sol  _getTotalEthInProtocol()
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);          // <-- reverts if oracle fails
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    unchecked { ++assetIdx; }
}
```

`getAssetPrice(asset)` delegates to an external `IPriceFetcher` contract:

```solidity
function getAssetPrice(address asset) public view onlySupportedOracle(asset) returns (uint256) {
    return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
}
```

If the `IPriceFetcher` for **any** supported asset reverts — because that oracle contract is paused, has been deprecated, or encounters an internal error — the entire `_getTotalEthInProtocol()` call reverts, which propagates up through `_updateRsETHPrice()` and out of `updateRSETHPrice()`. There is no `try/catch`, no skip-on-failure, and no fallback.

The same `_updateRsETHPrice()` path is responsible for:
1. Computing and minting protocol fee rsETH to the treasury.
2. Updating the stored `rsETHPrice` used by all deposits and withdrawals.
3. Triggering the downside-protection auto-pause when the price drops beyond the configured threshold.

All three are blocked for as long as any single oracle in the list is broken.

---

### Impact Explanation

**Permanent freezing of unclaimed yield (Medium).**

Protocol fees are minted inside `_updateRsETHPrice()`:

```solidity
if (protocolFeeInETH > 0) {
    uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);
    _checkAndUpdateDailyFeeMintLimit(rsethAmountToMintAsProtocolFee);
    IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
}
```

If `updateRSETHPrice()` cannot execute, no fees are ever minted to the treasury. If the oracle failure is permanent (e.g., the oracle contract is deprecated and never replaced), the accrued yield is permanently frozen. Even in the temporary case, every block during which the oracle is broken is a block during which yield that has accrued to the protocol is not captured.

Additionally, the stored `rsETHPrice` becomes stale, causing all subsequent deposits and withdrawal-unlock operations to use an incorrect exchange rate, which can silently mis-price rsETH minting and asset unlocking for all users.

---

### Likelihood Explanation

The protocol supports multiple LST assets (stETH, ETHx, sfrxETH, etc.), each with its own `IPriceFetcher`. Any one of these oracle contracts can be paused by its own administrator (e.g., Chainlink's circuit-breaker, or the protocol's own `PAUSER_ROLE` applied to a custom oracle wrapper). The `LRTConfig.pauseAll()` function already demonstrates that individual contracts in the system can be paused independently. The likelihood is **low** in normal operation but becomes **realistic** during incident response, oracle upgrades, or when an integrated LST protocol is itself paused — exactly the scenarios in which the Ion Protocol team acknowledged their analogous bug was most likely to occur.

---

### Recommendation

Wrap the per-asset oracle call in a `try/catch` block and skip assets whose oracle reverts, or accumulate a flag and revert with a descriptive error only after the loop completes. This mirrors the Ion Protocol team's own stated fix: "return zero values … when the pool is paused rather than allow reverts."

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    try IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset) returns (uint256 assetER) {
        uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
        totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    } catch {
        // skip asset with broken oracle; emit an event for observability
        emit AssetOracleSkipped(asset);
    }
    unchecked { ++assetIdx; }
}
```

---

### Proof of Concept

1. Protocol has two supported assets: stETH and ETHx.
2. The stETH price oracle contract is paused (by its own admin or by the protocol's `PAUSER_ROLE`).
3. Anyone calls `LRTOracle.updateRSETHPrice()`.
4. Execution enters `_updateRsETHPrice()` → `_getTotalEthInProtocol()`.
5. The loop reaches stETH, calls `getAssetPrice(stETH)` → `IPriceFetcher(stETHOracle).getAssetPrice(stETH)` → **reverts** (oracle paused).
6. The entire transaction reverts. ETHx's oracle is never consulted.
7. `rsETHPrice` is not updated. Protocol fees are not minted. The auto-pause downside-protection cannot trigger.
8. This persists for every block until the stETH oracle is unpaused — or permanently if the oracle is deprecated.
