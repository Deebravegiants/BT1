### Title
Malicious Harvester Can Permanently Divert Farmer Block Rewards to Zero Puzzle Hash via Unvalidated `farmer_reward_address_override` - (File: chia/farmer/farmer_api.py)

---

### Summary

In `FarmerAPI._process_respond_signatures`, the farmer unconditionally accepts any non-`None` `bytes32` value supplied by a connected harvester as `farmer_reward_address_override` — including `bytes32.zeros` — without any validity check. A malicious third-party harvester (as enabled by CHIP-22) can set this field to the all-zeros puzzle hash, causing the farmer's XCH block reward to be permanently sent to an unspendable address.

---

### Finding Description

CHIP-22 (`chia/protocols/harvester_protocol.py`) introduced `farmer_reward_address_override: bytes32 | None` in both `NewProofOfSpace` and `RespondSignatures`. When a harvester sets this field to a non-`None` value, the farmer is expected to use it as the `farmer_puzzle_hash` in `DeclareProofOfSpace` instead of its own configured `farmer_target`. [1](#0-0) 

In `_process_respond_signatures`, the farmer applies the override with no guard against the zero puzzle hash:

```python
farmer_reward_address = self.farmer.farmer_target
if response.farmer_reward_address_override is not None:
    farmer_reward_address = response.farmer_reward_address_override   # ← no zero-check
    include_source_signature_data = True

return farmer_protocol.DeclareProofOfSpace(
    ...
    farmer_reward_address,   # ← passed directly to full node
    ...
)
``` [2](#0-1) 

The full node then uses `request.farmer_puzzle_hash` verbatim when constructing the block's foliage: [3](#0-2) 

There is no downstream consensus rule that rejects a zero `farmer_reward_puzzle_hash` for non-genesis blocks. Block header validation only checks the genesis pre-farm hash for the genesis block: [4](#0-3) 

The `notify_farmer_reward_taken_by_harvester_as_fee` helper logs a warning when the fee threshold is suspicious, but it does **not** reject or sanitize the override address, and it is called before the address is used: [5](#0-4) 

---

### Impact Explanation

When a malicious harvester sets `farmer_reward_address_override = bytes32(b'\x00' * 32)`, the farmer's XCH block reward coin is created with puzzle hash `0x0000…0000`. No known private key controls that puzzle hash, so the reward is permanently unspendable — effectively burned. This constitutes **unauthorized reward diversion affecting XCH**, matching the Critical/High impact scope.

---

### Likelihood Explanation

CHIP-22 explicitly enables third-party harvesters (e.g., DrPlotter, GPU harvesters) to set `farmer_reward_address_override` to redirect a portion of rewards to themselves as a fee. Farmers routinely connect to such harvesters. Any such harvester — or any peer that obtains a valid TLS harvester connection — can supply `bytes32.zeros` as the override. The farmer has no mechanism to detect or reject this before the block is submitted. [6](#0-5) 

---

### Recommendation

In `_process_respond_signatures`, add a zero-address guard before accepting the override:

```python
if response.farmer_reward_address_override is not None:
    if response.farmer_reward_address_override == bytes32.zeros:
        self.farmer.log.error("Harvester supplied zero puzzle hash as farmer_reward_address_override; ignoring.")
        return None
    farmer_reward_address = response.farmer_reward_address_override
    include_source_signature_data = True
``` [7](#0-6) 

---

### Proof of Concept

1. A malicious harvester establishes a valid TLS connection to the farmer (standard CHIP-22 third-party harvester setup).
2. When the farmer sends `RequestSignatures` for a winning proof of space, the harvester responds with `RespondSignatures` where `farmer_reward_address_override = bytes32(b'\x00' * 32)`.
3. `_process_respond_signatures` passes the `is not None` check and sets `farmer_reward_address = bytes32.zeros`.
4. The farmer sends `DeclareProofOfSpace` to the full node with `farmer_puzzle_hash = bytes32.zeros`.
5. The full node constructs and propagates a valid block whose farmer reward coin has puzzle hash `0x0000…0000`.
6. The farmer's XCH block reward is permanently unspendable. [2](#0-1) [8](#0-7)

    def notify_farmer_reward_taken_by_harvester_as_fee(
        self, sp: farmer_protocol.NewSignagePoint, proof_of_space: harvester_protocol.NewProofOfSpace
    ) -> None:
        """
        Apply a fee quality convention (see CHIP-22: https://github.com/Chia-Network/chips/pull/88)
        given the proof and signage point. This will be tested against the fee threshold reported
        by the harvester (if any), and logged.
        """
        assert proof_of_space.farmer_reward_address_override is not None

        challenge_str = str(sp.challenge_hash)

        ph_prefix = self.config["network_overrides"]["config"][self.config["selected_network"]]["address_prefix"]
        farmer_reward_puzzle_hash = encode_puzzle_hash(proof_of_space.farmer_reward_address_override, ph_prefix)

        self.log.info(
            f"Farmer reward for challenge '{challenge_str}' "
            + f"taken by harvester for reward address '{farmer_reward_puzzle_hash}'"
        )

        fee_quality = calculate_harvester_fee_quality(proof_of_space.proof.proof, sp.challenge_hash)
        fee_quality_rate = float(fee_quality) / float(0xFFFFFFFF) * 100.0

        if proof_of_space.fee_info is not None:
            fee_threshold = proof_of_space.fee_info.applied_fee_threshold
            fee_threshold_rate = float(fee_threshold) / float(0xFFFFFFFF) * 100.0

            if fee_quality <= fee_threshold:
                self.log.info(
                    f"Fee threshold passed for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
            else:
                self.log.warning(
                    f"Invalid fee threshold for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
                self.log.warning(
                    "Harvester illegitimately took a fee reward that "
                    + "did not belong to it or it incorrectly applied the fee convention."
                )
        else:
            self.log.warning(
                "Harvester illegitimately took reward by failing to provide its fee rate "
                + f"for challenge '{challenge_str}'. "
                + f"Fee quality was {fee_quality_rate:.3f}% ({fee_quality} or 0x{fee_quality:08x})"
            )
