### Title
LP Principal Silently Burned to Zero on `removeLiquidity` When Proportional Scaled Balance Falls Below `TOKEN_0_SCALE_MULTIPLIER` — (`metric-core/contracts/libraries/LiquidityLib.sol`)

### Summary

`LiquidityLib.removeLiquidity` computes each bin's proportional scaled amount with floor division, decrements the bin's stored balance and `binTotals`, then converts the total to external units with a second floor division. When the proportional scaled amount is nonzero but less than `TOKEN_0_SCALE_MULTIPLIER` (e.g. `1e12` for USDC), the external conversion truncates to zero, no tokens are transferred, yet the LP's shares are fully burned and the bin accounting is permanently decremented. The stranded scaled value migrates into the pool's surplus and is later collected as spread fees.

### Finding Description

**Step 1 — per-bin floor division** [1](#0-0) 

```solidity
uint256 amount0Scaled = _checkedMul(binState.token0BalanceScaled, sharesToRemove) / binTotalSharesVal;
```

Floor division. If `token0BalanceScaled * sharesToRemove / binTotalSharesVal` yields, say, `5e11` (for a USDC pool where `TOKEN_0_SCALE_MULTIPLIER = 1e12`), `amount0Scaled = 5e11`.

**Step 2 — bin state and global accounting decremented** [2](#0-1) 

```solidity
binState.token0BalanceScaled -= uint104(amount0Scaled);   // 5e11 removed from bin
binTotals.scaledToken0 -= totalToken0ToRemoveScaled;       // global tracking decremented
```

**Step 3 — second floor division in `_deltasScaledToExternal`** [3](#0-2) 

```solidity
deltaAmount0 = scaledDeltaAmount0 / ctx.token0ScaleMultiplier;
// 5e11 / 1e12 = 0
```

**Step 4 — conditional transfer skipped** [4](#0-3) 

```solidity
if (amount0Removed > 0) {
    IERC20(ctx.token0).safeTransfer(owner, amount0Removed);
}
```

`amount0Removed = 0`, so no transfer occurs. The LP's shares are burned, the bin balance is decremented, but the LP receives nothing.

**Step 5 — stranded value becomes surplus, collected as fees**

The surplus formula in `collectFees` is: [5](#0-4) 

```solidity
uint256 surplus0Scaled =
    balance0() * TOKEN_0_SCALE_MULTIPLIER - uint256(binTotals.scaledToken0) - notionalFee0AmountScaled;
```

Because `binTotals.scaledToken0` was decremented but no actual token0 left the contract, `surplus0Scaled` grows by exactly `amount0Scaled`. This surplus is then split between protocol and admin as spread fees.

### Impact Explanation

The LP suffers a direct, irreversible loss of up to `TOKEN_0_SCALE_MULTIPLIER − 1` scaled units of token0 (and independently token1) per `removeLiquidity` call. For USDC (`TOKEN_0_SCALE_MULTIPLIER = 1e12`), the maximum loss per call is just under 1 USDC. For WBTC (`TOKEN_0_SCALE_MULTIPLIER = 1e10`), the maximum loss per call is just under 0.01 WBTC (~$1,000 at current prices). The lost principal is not destroyed — it is silently redirected to the protocol/admin fee surplus.

### Likelihood Explanation

The condition is reachable through normal pool operation:

1. An LP adds shares to a bin (paying real token0 via `ceilDiv` on add).
2. Swaps partially drain the bin's `token0BalanceScaled` to a residual below `TOKEN_0_SCALE_MULTIPLIER`.
3. The LP removes all their shares — `amount0Scaled` is nonzero but `< TOKEN_0_SCALE_MULTIPLIER`, so `amount0Removed = 0`.

No privileged access, malicious setup, or non-standard token behavior is required. The `minimalMintableLiquidity` guard only prevents leaving a dust *share* position; it does not guard against receiving zero tokens on a full exit. [6](#0-5) 

### Recommendation

After computing `amount0Scaled` and `amount1Scaled`, add a guard that reverts (or skips the decrement) when the scaled amount is nonzero but would convert to zero external units:

```solidity
// In removeLiquidity, after computing amount0Scaled:
if (amount0Scaled > 0 && amount0Scaled < ctx.token0ScaleMultiplier) {
    // Either round up to 1 external unit, or revert, or skip the decrement
    // Simplest safe fix: use ceilDiv for the external conversion on remove as well,
    // or require amount0Scaled >= ctx.token0ScaleMultiplier when nonzero.
}
```

Alternatively, change `_deltasScaledToExternal` for the remove path to use `Math.Rounding.Ceil` (matching the add path), which ensures the LP always receives at least 1 external unit when their scaled entitlement is nonzero. This is the symmetric fix to the intentional `Ceil` used on `addLiquidity`. [7](#0-6)
