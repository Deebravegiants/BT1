Now I have enough context. Let me look at the `builder` field usage in fee charging and the `ProposalInitValidation` struct construction to confirm the finding.

### Title
`ProposalInit.builder` (sequencer address) is not validated in `is_proposal_init_valid`, allowing a malicious proposer to redirect all block fees to an arbitrary address — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates ten fields of `ProposalInit` against locally-trusted references but never checks the `builder` field. `builder` is passed verbatim to `convert_to_sn_api_block_info` as `sequencer_address`, which is then used both as the fee-transfer recipient for every transaction in the block and as the `sequencer` component of `PartialBlockHashComponents` (and therefore the block hash). A Byzantine proposer can set `builder` to any address, causing all block fees to be credited to that address and the committed block hash to encode the wrong sequencer.

---

### Finding Description

**Validated fields in `is_proposal_init_valid`:**

| Field | Check |
|---|---|
| `height` | exact equality |
| `l1_da_mode` | exact equality |
| `l2_gas_price_fri` | exact equality |
| `starknet_version` | exact equality |
| `version_constant_commitment` | exact equality |
| `timestamp` | within window |
| `l1_gas_price_fri/wei`, `l1_data_gas_price_fri/wei` | within margin |
| `fee_proposal_fri` | within margin |
| **`builder`** | **never checked** | [1](#0-0) 

The `ProposalInitValidation` struct, constructed in `validate_proposal` inside `sequencer_consensus_context.rs`, carries `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `fee_actual`, and timestamp parameters — but no `builder_address`, even though `self.config.static_config.builder_address` is available at that call site. [2](#0-1) [3](#0-2) 

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in `BlockInfo`: [4](#0-3) 

`PartialBlockHashComponents::new` then copies `block_info.sequencer_address` into the `sequencer` field, which is chained into the Poseidon block hash: [5](#0-4) [6](#0-5) 

`execute_fee_transfer` uses `block_context.block_info.sequencer_address` as the ERC-20 transfer recipient for every transaction fee: [7](#0-6) 

The `ProposalFin` commitment comparison does **not** guard against this. Both the proposer and the validator derive `PartialBlockHash` from the same `builder` value supplied in `ProposalInit`, so their computed commitments agree regardless of what address was used: [8](#0-7) 

---

### Impact Explanation

**Critical — Incorrect fee/balance with economic impact.**

Every transaction fee in the block is transferred to the attacker-controlled address instead of the legitimate sequencer. The state diff written to storage reflects fee token balance changes for the wrong address. The committed block hash encodes the wrong sequencer address, corrupting the authoritative on-chain record. All of this is accepted by every honest validator because the `ProposalFin` commitment check passes (both sides used the same spoofed `builder`).

---

### Likelihood Explanation

Any validator node becomes the proposer in its assigned round. No special privilege beyond consensus participation is required. The attack requires only crafting a `ProposalInit` with a modified `builder` field — a single-field change that passes all existing validation gates.

---

### Recommendation

Add `builder_address: ContractAddress` to `ProposalInitValidation` and enforce an exact-equality check in `is_proposal_init_valid`:


### Proof of Concept

1. A Byzantine validator node waits for its proposer turn at height `H`.
2. In `initiate_build`, it overrides `builder` with an attacker-controlled address `ATTACKER_ADDR` instead of `args.builder_address`.
3. It streams `ProposalInit { builder: ATTACKER_ADDR, ... }` to all peers.
4. Each honest validator calls `is_proposal_init_valid` — no check on `builder`, so validation passes.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: ATTACKER_ADDR, ... }`.
6. The batcher executes all transactions; every `execute_fee_transfer` call transfers fees to `ATTACKER_ADDR`.
7. `BlockExecutionArtifacts::new` computes `PartialBlockHashComponents { sequencer: ATTACKER_ADDR, ... }` and derives `PartialBlockHash`.
8. The validator's `batcher_block_commitment` matches `received_fin.proposal_commitment` (both used `ATTACKER_ADDR`), so `ProposalFinMismatch` is not triggered.
9. Consensus reaches decision; `decision_reached` commits the block. All fees for height `H` are in `ATTACKER_ADDR`'s balance; the legitimate sequencer receives nothing. [9](#0-8) [10](#0-9)

```rust
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,

        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }

        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),

                init_proposed.version_constant_commitment

        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)

            init_proposed.clone(),
            proposal_init_validation.clone(),

        ));
    }

    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

            l1_data_gas_price_fri,

            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,

                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \

    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
```

```rust
        args.deps.state_sync_client.clone(),
        &init,
        args.deps.clock.as_ref(),
        args.retrospective_block_hash_deadline,
        args.retrospective_block_hash_retry_interval_millis,
        args.compare_retrospective_block_hash,
    )
    .await?;

    let build_proposal_input = ProposeBlockInput {
        proposal_id: args.proposal_id,
        deadline: args.batcher_deadline,
        retrospective_block_hash,
        block_info: convert_to_sn_api_block_info(&init)?,
        proposal_round: args.proposal_round,
    };
    debug!("Initiating build proposal: {build_proposal_input:?}");
    args.deps.batcher.propose_block(build_proposal_input.clone()).await.map_err(|err| {
        BuildProposalError::Batcher(
            format!("Failed to initiate build proposal {build_proposal_input:?}."),
            err,
        )
    })?;
    Ok(init)
```
