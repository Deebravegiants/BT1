### Title
Unvalidated `ProposalInit.builder` Used as `sequencer_address` Enables Fee Theft and Wrong Block State — (`File: crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`ProposalInit.builder` is a proposer-supplied field that is passed directly as `sequencer_address` into the block's `BlockInfo` by `convert_to_sn_api_block_info`. The validator path calls the same function with the same unvalidated `init.builder` value. Because `is_proposal_init_valid` never checks `builder`, a malicious committee proposer can set it to any arbitrary address. Every validator will re-execute the block with that address as the sequencer, reach the same commitment, and accept the block. All transaction fees for the block are then transferred to the attacker-controlled address, and the committed block header permanently records the wrong sequencer address.

---

### Finding Description

`ProposalInit` carries two identity fields:

- `proposer: ContractAddress` — validated in `handle_proposal` against the committee-derived expected proposer.
- `builder: ContractAddress` — **never validated anywhere**. [1](#0-0) 

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address`: [2](#0-1) 

This function is called on **both** the proposer path (`initiate_build`) and the validator path (`initiate_validation`): [3](#0-2) [4](#0-3) 

`is_proposal_init_valid` validates timestamp, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, and `fee_proposal_fri` — but contains **no check on `init.builder`**: [5](#0-4) 

The consensus manager validates `init.proposer` against the committee but leaves `init.builder` unchecked: [6](#0-5) 

---

### Impact Explanation

`sequencer_address` is the address to which all transaction fees are transferred during block execution. Because every validator independently calls `convert_to_sn_api_block_info` with the same proposer-supplied `init.builder`, they all execute the block with the attacker's address as the sequencer, produce the same `ProposalCommitment`, and vote to commit the block. The result is:

1. **Wrong fee balances in committed state** — all fees for every transaction in the block are credited to the attacker-controlled address instead of the legitimate sequencer.
2. **Wrong block header** — `sequencer_address` is part of the block header and enters the block hash commitment, so the committed block permanently records the forged address.

This matches the impact class: *Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact* and *Wrong state … storage value … from blockifier/syscall/execution logic*.

---

### Likelihood Explanation

The attacker must be a current-round committee proposer. In a BFT system with `n` validators, up to `f < n/3` can be Byzantine. A single malicious proposer turn is sufficient to execute the attack. No external user interaction is required; the malicious proposer simply sets `builder` to any address before broadcasting the proposal. All honest validators will accept the block because they re-derive the same commitment using the same unvalidated `builder` value.

---

### Recommendation

In `is_proposal_init_valid`, add a check that `init_proposed.builder` equals the locally-expected builder address for the current height and round (analogous to how `init.proposer` is checked against the committee in `handle_proposal`). If the builder address is deterministically derived from the proposer identity or a fixed per-height configuration, compare against that. At minimum, reject any `builder` value that does not match the node's own configured `builder_address`.

```rust
// In is_proposal_init_valid:
if init_proposed.builder != expected_builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("builder mismatch: expected={:?}, got={:?}",
            expected_builder_address, init_proposed.builder),
    ));
}
```

`ProposalInitValidation` should carry the expected `builder` address so the validator can enforce it.

---

### Proof of Concept

1. A malicious node wins the proposer slot for height `H`, round `R`.
2. It constructs `ProposalInit { builder: attacker_address, proposer: self_address, … }` with all other fields valid.
3. It broadcasts the proposal. `handle_proposal` in every peer checks `init.proposer == expected_proposer` — this passes.
4. Every peer calls `validate_proposal` → `is_proposal_init_valid` — `builder` is never checked, so this passes.
5. Every peer calls `initiate_validation` → `convert_to_sn_api_block_info(&init)` → `sequencer_address = attacker_address`.
6. The blockifier executes all transactions with `sequencer_address = attacker_address`; all fee transfers credit `attacker_address`.
7. Every peer computes the same `ProposalCommitment` (because they all used the same `builder`), the `ProposalFin` comparison passes, and the block is committed.
8. The committed block header records `sequencer_address = attacker_address`; the attacker has stolen all sequencer fees for block `H`. [7](#0-6) [8](#0-7)

        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {

                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
