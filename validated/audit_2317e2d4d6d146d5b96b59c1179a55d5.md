### Title
Witness Count Not Validated Against Input Count in `updateUTxOTxWitness` — zip-truncation allows unwitnessed inputs to be spent — (`eras/byron/ledger/impl/src/Cardano/Chain/UTxO/Validation.hs`)

---

### Summary

`updateUTxOTxWitness` uses Haskell's `zip` to pair input addresses with witnesses before calling `validateWitness`. Because `zip` silently truncates to the shorter list, a transaction with N inputs and M < N witnesses causes only M witness checks to run. The subsequent `updateUTxOTx` call then consumes all N inputs unconditionally, spending the unwitnessed inputs without any authorization check.

---

### Finding Description

The vulnerable code is in `updateUTxOTxWitness`:

```haskell
addresses <-
  mapM (`UTxO.lookupAddress` utxo) (NE.toList $ txInputs tx)
    `wrapError` UTxOValidationUTxOError

mapM_
  (uncurry $ validateWitness pmi sigData)
  (zip addresses (V.toList witness))   -- truncates to min(N, M)
  `wrapError` UTxOValidationTxValidationError
``` [1](#0-0) 

`addresses` is derived from `NE.toList $ txInputs tx` and has exactly N elements (one per input). `V.toList witness` has M elements. Haskell's `zip` produces `min(N, M)` pairs — the remaining `N - M` inputs are silently dropped from the witness-check loop.

No length-equality guard exists anywhere in the validation path:
- `validateTx` checks inputs exist in the UTxO and network magic — no witness count check. [2](#0-1) 
- `validateTxAux` checks fee, size, and balance — no witness count check. [3](#0-2) 

After the truncated witness loop, `updateUTxOTx` is called unconditionally and applies the full UTxO state change for all N inputs:

```haskell
updateUTxOTx env utxo aTx
``` [4](#0-3) 

Inside `updateUTxOTx`, the UTxO update removes all N inputs and adds the new outputs:

```haskell
UTxO.union (S.fromList (NE.toList (txInputs tx)) </| utxo) (txOutputUTxO tx)
``` [5](#0-4) 

The entire witness-validation block is gated on `whenTxValidation`, which is active when `txValidationMode == TxValidation` — the standard mode for block validation. [6](#0-5) 

The Byron executable spec (`UTXOWS` rule) requires `length wits == length ins`. This production implementation violates that invariant.

---

### Impact Explanation

**Critical — Direct loss of ADA through an invalid ledger state transition.**

An attacker who:
1. Knows the `TxIn` references of UTxO entries at addresses they do not control (public blockchain data)
2. Holds a key for at least one address (their own)

can craft an `ATxAux` with N inputs (including victims' inputs at indices 1..N-1) and only 1 witness (for their own input at index 0). The `zip` produces exactly 1 pair; only the attacker's own witness is validated. `updateUTxOTx` then removes all N UTxO entries and credits the outputs to the attacker. ADA belonging to the unwitnessed addresses is transferred without authorization.

---

### Likelihood Explanation

The exploit requires only:
- Knowledge of victim UTxO entries (public)
- One valid key (attacker's own address)
- Ability to submit a block containing the crafted transaction

No privileged access, governance majority, or key leakage is required. The attack is fully constructible offline and submittable by any block producer.

**Practical constraint:** Byron is a historical era; the Cardano mainnet is currently in the Conway era and does not accept new Byron-era transactions. The vulnerability is exploitable in any Byron-era deployment (private networks, testnets, or chain-replay infrastructure running Byron validation).

---

### Recommendation

Add an explicit length-equality check before the `zip` in `updateUTxOTxWitness`, consistent with the `UTXOWS` spec rule:

```haskell
let nInputs   = length (NE.toList (txInputs tx))
    nWitnesses = V.length witness
when (nWitnesses /= nInputs) $
  throwError $ UTxOValidationTxValidationError
             $ TxValidationWitnessCountMismatch nInputs nWitnesses
```

This requires adding a `TxValidationWitnessCountMismatch` constructor to `TxValidationError`. [7](#0-6) 

---

### Proof of Concept

```haskell
-- Construct a UTxO with 3 entries at distinct addresses addrA, addrB, addrC
-- Attacker holds keyA only.
let tx = UnsafeTx
      { txInputs    = txInA :| [txInB, txInC]  -- 3 inputs
      , txOutputs   = attackerOut :| []
      , txAttributes = mkAttributes ()
      }
    witness = V.fromList [VKWitness vkA (sign keyA sigData)]  -- only 1 witness

    ta = mkTxAux tx witness

-- Run updateUTxOTxWitness in TxValidation mode:
-- zip [addrA, addrB, addrC] [witnessA]  =>  [(addrA, witnessA)]
-- mapM_ validateWitness [(addrA, witnessA)]  =>  OK (only addrA checked)
-- updateUTxOTx removes txInA, txInB, txInC from UTxO  =>  addrB and addrC stolen
```

The resulting UTxO has all three inputs consumed and the attacker's output credited, despite no witness being provided for `addrB` or `addrC`.

validateTxAux ::
  MonadError TxValidationError m =>
  Environment ->
  UTxO ->
  ATxAux ByteString ->
  m ()
validateTxAux env utxo (ATxAux (Annotated tx _) _ txBytes) = do
  -- Check that the size of the transaction is less than the maximum
  txSize
    <= maxTxSize
    `orThrowError` TxValidationTxTooLarge txSize maxTxSize

  -- Calculate the minimum fee from the 'TxFeePolicy'
  minFee <-
    if isRedeemUTxO inputUTxO
      then pure $ mkKnownLovelace @0
      else calculateMinimumFee feePolicy

  -- Calculate the balance of the output 'UTxO'
  balanceOut <-
    balance (txOutputUTxO tx)
      `wrapError` TxValidationLovelaceError "Output Balance"

  -- Calculate the balance of the restricted input 'UTxO'
  balanceIn <-
    balance inputUTxO
      `wrapError` TxValidationLovelaceError "Input Balance"

  -- Calculate the 'fee' as the difference of the balances
  fee <-
    subLovelace balanceIn balanceOut
      `wrapError` TxValidationLovelaceError "Fee"

  -- Check that the fee is greater than the minimum
  (minFee <= fee) `orThrowError` TxValidationFeeTooSmall tx minFee fee
  where
    Environment {protocolParameters} = env

    maxTxSize = ppMaxTxSize protocolParameters
    feePolicy = ppTxFeePolicy protocolParameters

    txSize :: Natural
    txSize = fromIntegral $ BS.length txBytes

    inputUTxO = S.fromList (NE.toList (txInputs tx)) <| utxo

    calculateMinimumFee ::
      MonadError TxValidationError m => TxFeePolicy -> m Lovelace
    calculateMinimumFee = \case
      TxFeePolicyTxSizeLinear txSizeLinear ->
        calculateTxSizeLinear txSizeLinear txSize
          `wrapError` TxValidationLovelaceError "Minimum Fee"
