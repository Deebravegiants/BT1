### Title
Unbounded `fee_proposal_fri` During Startup Window Allows Malicious Proposer to Permanently Skew L2 Gas Price - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

The `fee_proposal_fri` bounds check inside `is_proposal_init_valid` is gated on `fee_actual` being `Some`. During the first `window_size` blocks after genesis (or after a protocol upgrade to V0_14_3), `fee_actual` is always `None`, so the check is silently skipped. A malicious proposer can set `fee_proposal_fri` to any value — including `u128::MAX` or `1` — during this unconstrained window. Those extreme values are committed into the block header, recorded in `fee_proposals_window`, and eventually dominate the median (`fee_actual`) used to compute `next_l2_gas_price` for all subsequent blocks.

---

### Finding Description

In `is_proposal_init_valid`, the bounds check for `fee_proposal_fri` is wrapped in an `if let` that only fires when both `fee_actual` and `fee_proposal` are `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

`fee_actual` is computed by `compute_fee_actual`, which returns `None` whenever `height < window_size` **or** whenever any block in the sliding window has `fee_proposal_fri = None` (pre-V0_14_3 blocks):

```rust
Some(None) | None => {
    warn!("Cannot compute fee_actual for height {height}: ...");
    return None;
}
``` [2](#0-1) 

When `fee_actual` is `None`, the validator accepts **any** `fee_proposal_fri` value from the proposer. The accepted value is then:

1. Bound into the `ProposalCommitment` as `Poseidon(partial_block_hash, fee_proposal_fri)`: [3](#0-2) 

2. Recorded into `fee_proposals_window` at `decision_reached` via `record_fee_proposal`: [4](#0-3) 

3. Used to compute `fee_actual` (median of window) for future blocks, which feeds into `next_l2_gas_price`: [5](#0-4) 

The `fee_proposals_window` is an in-memory `BTreeMap` initialized from state-sync at startup and updated on every committed block: [6](#0-5) 

The validator constructs `ProposalInitValidation.fee_actual` from its own local window at the time `validate_proposal` is called: [7](#0-6) 

Because both proposer and validator independently compute `fee_actual = None` during the startup window, they agree that no bounds check is needed — so the malicious value passes validation on every honest node.

---

### Impact Explanation

**Impact: Critical — Incorrect fee/gas price with economic impact.**

A malicious proposer who controls blocks during the startup window (first `window_size` blocks) can set `fee_proposal_fri = u128::MAX`. Once those values fill the window, `compute_fee_actual` returns an extreme median. `compute_proposer_fee_proposal` then clamps the oracle's `fee_target` to `[u128::MAX / (1+margin), u128::MAX]`, and `calculate_next_l2_gas_price_for_fin` propagates this extreme value as `next_l2_gas_price` into every subsequent block header. All transaction fees for future blocks are computed against this corrupted base price.

The corrupted field is `next_l2_gas_price` in `BlockHeaderWithoutHash`, which is stored in state-sync and used by the blockifier for fee charging: [8](#0-7) 

---

### Likelihood Explanation

The startup window occurs at genesis and after every protocol upgrade that introduces V0_14_3. A malicious validator who controls more than half of the `window_size` proposer slots during this window can set the median to an extreme value. With a small window (e.g., 2–5 blocks) or with a coalition of malicious validators, this threshold is reachable. Even without controlling the median, injecting a single extreme value shifts the median toward the extreme on each subsequent block where the malicious proposer is selected, creating a slow-drift attack.

The `fee_actual = None` condition also re-triggers after any protocol downgrade or state-sync gap that reintroduces pre-V0_14_3 blocks into the window, since `compute_fee_actual` returns `None` for any `None` entry: [9](#0-8) 

---

### Recommendation

1. **Enforce absolute bounds unconditionally.** Even when `fee_actual` is `None`, require `fee_proposal_fri` to lie within a protocol-defined absolute range (e.g., `[min_gas_price, max_gas_price]` from `VersionedConstants`). The honest proposer already falls back to `self.l2_gas_price` when `fee_actual` is `None`; the validator can apply the same fallback as the reference for bounds checking.

2. **Use `l2_gas_price` as the reference during startup.** When `fee_actual` is `None`, substitute `l2_gas_price` (the current EIP-1559 base fee) as the reference for `fee_proposal_bounds`. This preserves the existing margin logic while closing the unconstrained window.

3. **Add a test for the startup-window bypass.** The existing test suite covers `fee_actual = Some(...)` cases but has no test asserting that an extreme `fee_proposal_fri` is rejected when `fee_actual` is `None`. [10](#0-9) 

---

### Proof of Concept

1. Network starts at genesis; `fee_proposals_window` is empty; `compute_fee_actual` returns `None` for all heights `< window_size`.

2. Malicious validator is selected as proposer for block 0. It constructs `ProposalInit` with `fee_proposal_fri = Some(GasPrice(u128::MAX))`.

3. Every honest validator calls `is_proposal_init_valid`. Because `proposal_init_validation.fee_actual = None`, the `if let (Some(fee_actual), Some(fee_proposal))` arm does not fire. The proposal passes validation. [1](#0-0) 

4. `validate_proposal` calls `handle_proposal_part` with `fee_proposal = Some(u128::MAX)`. The validator computes `batcher_block_commitment = proposal_commitment_from(partial_block_hash, Some(u128::MAX))` and compares it to `fin.proposal_commitment`. They match (the proposer computed the same hash). The proposal is accepted. [11](#0-10) 

5. `decision_reached` is called. `finalize_decision` calls `record_fee_proposal(BlockNumber(0), Some(GasPrice(u128::MAX)))`, inserting the extreme value into `fee_proposals_window`. [4](#0-3) 

6. The malicious validator repeats for blocks 1 through `window_size/2`. Honest validators fill the remaining slots with `l2_gas_price` (the fallback when `fee_actual` is `None`).

7. At block `window_size`, `compute_fee_actual` computes the median of the window. If the malicious validator controlled `> window_size/2` slots, the median is `u128::MAX`. [12](#0-11) 

8. `compute_proposer_fee_proposal` for block `window_size` receives `fee_actual = Some(u128::MAX)`. `fee_proposal_bounds` returns `(u128::MAX / (1+margin), u128::MAX)`. The oracle target is clamped into this range. `next_l2_gas_price` is set to an extreme value and written into the block header for all future blocks. [13](#0-12)

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

    async fn compute_proposer_fee_proposal(
        &self,
        fee_actual: Option<GasPrice>,
        timestamp: u64,

    ) -> GasPrice {
        SNIP35_FEE_TARGET_ATTO_USD.set_lossy(target_atto_usd_per_l2_gas);
        let Some(fee_actual) = fee_actual else {
            warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");

            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        SNIP35_FEE_PROPOSAL_FRI.set_lossy(proposal.0);
        proposal
    }
