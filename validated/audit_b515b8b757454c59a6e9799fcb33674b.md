### Title
`whenNotPaused` on `completeWithdrawal` Freezes User rsETH After `initiateWithdrawal` Commits Funds - (File: `contracts/LRTWithdrawalManager.sol`)

---

### Summary

`LRTWithdrawalManager` implements a two-step withdrawal flow. In step 1, `initiateWithdrawal` transfers rsETH from the user into the contract. In step 2, `completeWithdrawal` returns the corresponding ETH/LST to the user. Both functions carry the `whenNotPaused` modifier. If the contract is paused between the two steps, the user's rsETH is locked inside the contract with no mechanism to recover it while the pause is active.

---

### Finding Description

`initiateWithdrawal` pulls rsETH from the caller at line 166 and records a withdrawal request: [1](#0-0) 

`completeWithdrawal`, the only user-callable function to finalize the request and receive assets, is gated by `whenNotPaused`: [2](#0-1) 

The operator-assisted path `completeWithdrawalForUser` is equally blocked: [3](#0-2) 

`unlockQueue`, the prerequisite operator step that must run before any completion is possible, also carries `whenNotPaused`: [4](#0-3) 

The contract exposes no function that allows a user to cancel a pending withdrawal request and reclaim their rsETH while the contract is paused. `sweepRemainingAssets` routes funds to the treasury, not back to users: [5](#0-4) 

The `pauseAll` helper in `LRTConfig` pauses `LRTWithdrawalManager` as part of a single call, meaning a protocol-wide emergency pause immediately creates this condition for every in-flight withdrawal: [6](#0-5) 

---

### Impact Explanation

Any user who has called `initiateWithdrawal` and had their rsETH transferred to the contract before a pause event cannot complete the withdrawal or recover their rsETH for the entire duration of the pause. The rsETH balance is held by the contract with no user-accessible exit path. This constitutes a **temporary freezing of funds** (Medium). In the degenerate case where the contract is never unpaused, it becomes a **permanent freezing of funds** (Critical).

---

### Likelihood Explanation

Protocol-wide pauses are a normal operational event (security incidents, oracle anomalies, upgrade windows). The `PAUSER_ROLE` can trigger a pause at any time via `LRTConfig.pauseAll()`. Given that the withdrawal delay is set to 8 days by default (`withdrawalDelayBlocks = 8 days / 12 seconds`), there is a large window during which a pause can occur between `initiateWithdrawal` and `completeWithdrawal`. Any user with an in-flight withdrawal at the time of a pause is affected. [7](#0-6) 

---

### Recommendation

Remove `whenNotPaused` from `completeWithdrawal` and `completeWithdrawalForUser`. These functions only disburse assets that are already committed and owed to users; blocking them during a pause does not protect the protocol and directly harms users. Alternatively, add a `cancelWithdrawal` function (callable even when paused) that returns the user's rsETH and removes the pending request, restoring the user to their pre-initiation state.

---

### Proof of Concept

1. User calls `initiateWithdrawal(asset, amount, referralId)`. rsETH is transferred from the user to `LRTWithdrawalManager` at line 166. A `WithdrawalRequest` is recorded.
2. `PAUSER_ROLE` calls `LRTConfig.pauseAll()`, which pauses `LRTWithdrawalManager`.
3. After the withdrawal delay passes, the user calls `completeWithdrawal(asset, referralId)`. The call reverts at the `whenNotPaused` modifier (line 183).
4. The user has no alternative path: `completeWithdrawalForUser` is also blocked (line 199), `unlockQueue` is blocked (line 279), and no cancellation function exists.
5. The user's rsETH remains locked in the contract for the entire duration of the pause, with no recourse.

        IPausable rsETHContract = IPausable(rsETH);

        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!lrtWithdrawalManager.paused()) lrtWithdrawalManager.pause();
        if (!lrtOracle.paused()) lrtOracle.pause();
        if (!rsETHContract.paused()) rsETHContract.pause();

        address[] memory nodeDelegatorQueue = ILRTDepositPool(address(lrtDepositPool)).getNodeDelegatorQueue();
        uint256 nodeDelegatorCount = nodeDelegatorQueue.length;

        for (uint256 i = 0; i < nodeDelegatorCount;) {
            IPausable nodeDelegator = IPausable(nodeDelegatorQueue[i]);
            if (!nodeDelegator.paused()) nodeDelegator.pause();
            unchecked {
                ++i;
            }
        }

        emit PausedAll(msg.sender);
    }
