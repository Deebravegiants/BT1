Audit Report

## Title
Missing Staleness Validation in `ChainlinkPriceOracle.getAssetPrice()` Enables Stale-Price-Driven rsETH Minting Dilution and False Emergency Pause — (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `updatedAt`, `answeredInRound`, and `roundId`, accepting any price regardless of age or round completeness. The same project already implements `answeredInRound < roundID`, `timestamp == 0`, and `price <= 0` guards in `ChainlinkOracleForRSETHPoolCollateral`, confirming the omission is an inconsistency rather than a design choice. A stale price propagates into rsETH minting amounts (diluting existing holders) and into the TVL computation that drives the emergency-pause trigger.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` (L49–55):

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values are available; only `answer` is consumed. No check on:
- `updatedAt < block.timestamp - MAX_STALENESS` (price age)
- `answeredInRound < roundId` (round completeness)
- `price <= 0` (invalid price)

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` (L26–37) performs all three checks, proving the project is aware of the pattern.

**Exploit path A — rsETH minting dilution:**
1. A Chainlink LST/ETH feed goes stale (network congestion, oracle downtime). Last reported price is inflated vs. current market.
2. Attacker (or any user) calls `LRTDepositPool.depositAsset(stETH, amount)`.
3. `getRsETHAmountToMint()` (L519–521) computes `(amount * lrtOracle.getAssetPrice(stETH)) / lrtOracle.rsETHPrice()` using the stale inflated price.
4. Depositor receives more rsETH than their deposit is worth; existing rsETH holders are diluted.

**Exploit path B — false emergency pause (temporary fund freeze):**
1. A Chainlink feed goes stale with a price lower than the current market.
2. Any caller invokes `LRTOracle.updateRSETHPrice()` (L87–89, public, no access control beyond `whenNotPaused`).
3. `_getTotalEthInProtocol()` (L336–343) aggregates stale low prices → artificially depressed TVL.
4. `newRsETHPrice` drops below `highestRsethPrice` by more than `pricePercentageLimit` → `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` are triggered (L277–281), freezing deposits and withdrawals.

Existing guards are insufficient: the `pricePercentageLimit` check (L273–274) is designed to catch genuine price drops, not to substitute for staleness validation; a stale price that falls just within the limit passes silently.

## Impact Explanation

- **Path A:** Theft of unclaimed yield (High). Existing rsETH holders lose yield/value proportional to the minting overage caused by the stale inflated price. Repeated exploitation across multiple deposits compounds the dilution.
- **Path B:** Temporary freezing of funds (Medium). A stale deflated price can trigger the emergency pause, halting all deposits and withdrawals until an admin manually unpauses.

## Likelihood Explanation

Chainlink feeds go stale during L2 sequencer outages, network congestion, or oracle node downtime — documented real-world events. `updateRSETHPrice()` is callable by any unprivileged address with no rate limiting. `depositAsset()` is the normal user deposit flow. No special access, no front-running, and no governance capture is required; the attacker need only deposit while a feed is stale, or call the public price-update function.

## Recommendation

Add staleness and validity checks in `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
uint256 public constant MAX_STALENESS = 1 hours; // tune per feed heartbeat

function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (price <= 0) revert InvalidPrice();
    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt < block.timestamp - MAX_STALENESS) revert PriceOutdated();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

`MAX_STALENESS` should be set per-feed based on the Chainlink heartbeat (e.g., 1 hour for ETH/USD, 24 hours for some LST feeds).
