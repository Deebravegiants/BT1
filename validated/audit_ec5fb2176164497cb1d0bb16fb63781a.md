Audit Report

## Title
Hardcoded `gasLimit=0` in `ScrollMessenger.sendETHToL1ViaBridge` Causes Deterministic L1 Relay Failure, Temporarily Freezing Bridged ETH — (`contracts/bridges/ScrollMessenger.sol`)

## Summary
`ScrollMessenger.sendETHToL1ViaBridge` passes `gasLimit=0` to Scroll's `sendMessage`, based on an incorrect assumption that `0` means "use default." Scroll's protocol treats `gasLimit` as a literal execution gas cap for the L1 relay call. With `gasLimit=0`, the L1 relay call to `l1VaultETHForL2Chain` receives only the 2300-gas value-transfer stipend, which is insufficient for any non-trivial `receive()` logic, causing `FailedRelayedMessage` on every invocation and temporarily freezing the bridged ETH in the L1 messenger until a manual `replayMessage` is performed.

## Finding Description
In `contracts/bridges/ScrollMessenger.sol` line 23, the call is:
```solidity
IScrollMessenger(l2bridge).sendMessage{ value: value }(target, value, "", 0, msg.sender);
```
The NatSpec comment on line 19 claims `gasLimit=0` uses a "default gas limit," but `contracts/interfaces/L2/IScrollMessenger.sol` line 62 explicitly documents the parameter as:
> `@param gasLimit Gas limit required to complete the message relay on corresponding chain.`

This is a required execution gas cap, not a sentinel. Scroll's L1ScrollMessenger stores the queued `gasLimit` and uses it literally in the relay call: `_to.call{value: _value, gas: _gasLimit}(_message)`. With `_gasLimit=0` and `_value>0`, the EVM forwards only the 2300-gas value-transfer stipend to `l1VaultETHForL2Chain` (an `L1VaultETH` contract). Any `receive()` logic beyond a bare ETH accept (e.g., event emission, accounting updates) exceeds 2300 gas, causing the relay to fail and emit `FailedRelayedMessage`.

Every other messenger in the codebase correctly uses a non-zero constant. `contracts/bridges/BaseMessenger.sol` line 15 defines `uint32 public constant DEFAULT_GAS_LIMIT = 200_000` and passes it at line 25. `ScrollMessenger` is the sole deviation.

The full call chain confirmed in the codebase:
`RSETHPoolV2.bridgeAssets()` (line 286–298) → `IL2Messenger(messenger).sendETHToL1ViaBridge` → `ScrollMessenger.sendETHToL1ViaBridge` → `sendMessage(..., 0, ...)` → L1 relay fails.

## Impact Explanation
Every `bridgeAssets()` call on the Scroll pool successfully dequeues ETH from the pool and locks it in the Scroll L2 bridge contract, but the corresponding L1 relay deterministically fails. The ETH is temporarily frozen in the L1ScrollMessenger until a manual `replayMessage` with a sufficient `gasLimit` is submitted. This matches **Medium — Temporary freezing of funds**.

## Likelihood Explanation
This is a 100%-reproducible, deterministic failure on every bridge call through `ScrollMessenger`. No attacker action is required. The `BRIDGER_ROLE` is an operational role expected to call `bridgeAssets()` regularly as part of normal protocol operations; the bug fires on every such legitimate invocation. The only remediation path without a contract upgrade is a manual `replayMessage` call per failed bridge transaction.

## Recommendation
Replace the hardcoded `0` with a sufficient gas limit constant, consistent with `BaseMessenger`:
```solidity
uint32 public constant DEFAULT_GAS_LIMIT = 200_000;

function sendETHToL1ViaBridge(address l2bridge, address target, uint256 value) external payable nonReentrant {
    if (msg.value != value) revert MismatchedMsgValue();
    IScrollMessenger(l2bridge).sendMessage{ value: value }(target, value, "", DEFAULT_GAS_LIMIT, msg.sender);
}
```

## Proof of Concept
1. Fork Scroll mainnet at a recent block.
2. Impersonate `BRIDGER_ROLE` on the deployed `RSETHPoolV2` (Scroll).
3. Ensure the pool holds a non-zero ETH balance minus fees.
4. Call `RSETHPoolV2.bridgeAssets()`.
5. Observe the `SentMessage` event on L2 with `gasLimit=0`.
6. On the L1 fork, call `L1ScrollMessenger.relayMessageWithProof(target, value, "", nonce, proof)`.
7. Assert `FailedRelayedMessage` is emitted (not `RelayedMessage`).
8. Confirm ETH is locked in `L1ScrollMessenger` until `replayMessage` is called with `gasLimit >= 200_000`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
