Audit Report

## Title
Stale `rsETHPrice` Broadcast Cross-Chain After Auto-Pause Skips Price Update — (`contracts/cross-chain/RSETHRateProvider.sol`)

## Summary
When a price drop beyond `pricePercentageLimit` triggers the auto-pause in `LRTOracle._updateRsETHPrice()`, the function returns at line 281 before reaching `rsETHPrice = newRsETHPrice` at line 313, leaving the storage variable frozen at the pre-pause (inflated) value. Because `CrossChainRateProvider.updateRate()` has no access control and no check for whether `LRTOracle` is paused, any unprivileged caller can immediately broadcast the stale inflated rate cross-chain via LayerZero.

## Finding Description
In `LRTOracle._updateRsETHPrice()`, when `isPriceDecreaseOffLimit` is true, the function pauses the protocol and returns early: [1](#0-0) 

The assignment `rsETHPrice = newRsETHPrice` at line 313 is never reached, so `rsETHPrice` remains at the pre-pause higher value: [2](#0-1) 

`RSETHRateProvider.getLatestRate()` reads `rsETHPrice` directly from storage without any staleness or pause check: [3](#0-2) 

`CrossChainRateProvider.updateRate()` is `external payable nonReentrant` with no role guard and no check that `LRTOracle` is paused. It reads `getLatestRate()` and immediately sends the result via LayerZero: [4](#0-3) 

Note also that `updateRSETHPrice()` carries `whenNotPaused`, so the stale price cannot be corrected through the normal public path after auto-pause — only a manager calling `updateRSETHPriceAsManager()` can fix it, but the stale rate can be broadcast at any time before that correction occurs: [5](#0-4) 

## Impact Explanation
Cross-chain receivers receive an inflated rsETH/ETH rate that does not reflect actual protocol collateral backing after a slashing or TVL drop event. Any protocol or pool on the destination chain that prices rsETH using this rate will overvalue it relative to what the L1 protocol can redeem. This matches **Low — contract fails to deliver promised returns, but doesn't lose value** within the allowed impact scope.

## Likelihood Explanation
The auto-pause is an intentional safety mechanism that fires whenever a significant TVL drop occurs (e.g., EigenLayer slashing). The `updateRate()` call requires only ETH for LayerZero gas fees. Any observer watching for the `Paused` event on `LRTOracle` can immediately call `updateRate()` to lock in the stale rate cross-chain before the admin can unpause and correct it. The condition is realistic and repeatable.

## Recommendation
Add a pause check in `RSETHRateProvider` overriding `updateRate()`, or add it directly in `CrossChainRateProvider.updateRate()`:

```solidity
function updateRate() external payable nonReentrant {
    require(!ILRTOracle(rsETHPriceOracle).paused(), "Oracle paused");
    // ... rest of function
}
```

Alternatively, update `rsETHPrice` to `newRsETHPrice` before calling `_pause()` and `return` in `_updateRsETHPrice()`, so the stored price always reflects the latest computed (lower) value regardless of pause state. This is arguably the more correct fix, as it ensures the stored price accurately reflects the actual state of the protocol at the moment of pause.

## Proof of Concept

```solidity
// 1. pricePercentageLimit is set (e.g., 5e16 = 5%)
// 2. TVL drops >5% (e.g., EigenLayer slashing event)
//    → updateRSETHPrice() is called
//    → _updateRsETHPrice() computes newRsETHPrice < highestRsethPrice by >5%
//    → isPriceDecreaseOffLimit = true
//    → lrtDepositPool.pause(), withdrawalManager.pause(), _pause() called
//    → return at line 281 — rsETHPrice NOT updated, frozen at pre-slash value

// 3. Attacker (any address) observes Paused event, then calls:
uint256 lzFee = /* estimated LayerZero fee */;
rsETHRateProvider.updateRate{value: lzFee}();

// 4. getLatestRate() returns ILRTOracle.rsETHPrice() — the stale pre-slash value
// 5. LayerZero broadcasts stale inflated rate to destination chain

// Assertion:
uint256 broadcastRate = rsETHRateProvider.rate();
uint256 actualRate = totalETHInProtocol / rsETH.totalSupply(); // post-slash
assert(broadcastRate > actualRate); // stale rate exceeds actual backing
```
