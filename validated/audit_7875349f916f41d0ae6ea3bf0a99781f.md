### Title
Aave Reserve Pause Freezes User ETH Withdrawals With No Protocol Escape Hatch — (`contracts/LRTWithdrawalManager.sol`)

### Summary

`LRTWithdrawalManager` deposits unlocked ETH into Aave v3 to earn yield. When a user calls `completeWithdrawal`, the contract pulls ETH back from Aave if its own balance is insufficient. If Aave's WETH reserve is paused (a standard Aave governance action), the `_withdrawFromAave` call reverts with no fallback, freezing all pending ETH withdrawals. Critically, the only admin escape hatch — `setAaveIntegrationEnabled(false)` — also calls `_withdrawFromAave` internally, so it too reverts while Aave is paused. There is no code path that allows the protocol to disable the Aave integration or service user withdrawals while Aave is unavailable.

---

### Finding Description

**Root cause — `_processWithdrawalCompletion` has no fallback when Aave reverts:** [1](#0-0) 

When `isAaveIntegrationEnabled` is `true` and the contract's ETH balance is below the requested amount, `_withdrawFromAave` is called unconditionally. There is no `try/catch` and no alternative code path. If `aaveWETHGateway.withdrawETH` reverts (e.g., because Aave has paused the WETH reserve), the entire `completeWithdrawal` call reverts.

**Root cause — `setAaveIntegrationEnabled(false)` requires a successful Aave withdrawal before it can disable the integration:** [2](#0-1) 

When the manager attempts to disable the Aave integration, the code first calls `_collectInterestToTreasury()` and then `_withdrawFromAave(aaveBalance)`. Both of these call `aaveWETHGateway.withdrawETH`, which reverts if the reserve is paused. The `isAaveIntegrationEnabled` flag is never flipped to `false`, so the integration remains active and the withdrawal path remains broken.

**Root cause — `emergencyWithdrawFromAave` also calls `_withdrawFromAave`:** [3](#0-2) 

The PAUSER_ROLE emergency function calls `_collectInterestToTreasury()` and then `_withdrawFromAave(amount)`, both of which revert when Aave is paused. There is no code path that bypasses the Aave gateway.

**Contrast with deposit path — which correctly uses `try/catch`:** [4](#0-3) 

Deposits to Aave silently fail and leave ETH in the contract. Withdrawals from Aave have no equivalent resilience, creating a one-way ratchet: ETH flows into Aave successfully even when Aave is degraded, but cannot flow back out.

### Impact Explanation

When Aave's WETH reserve is paused:

1. **Users cannot complete ETH withdrawals** — `completeWithdrawal` and `completeWithdrawalForUser` both revert.
2. **The manager cannot disable the Aave integration** — `setAaveIntegrationEnabled(false)` reverts before flipping the flag.
3. **The PAUSER_ROLE emergency function fails** — `emergencyWithdrawFromAave` reverts.
4. **All ETH deposited to Aave is inaccessible** for the duration of the pause, with no protocol-level escape hatch.

This constitutes a **temporary freeze of user funds** (Medium impact per the allowed scope). If the Aave pause is indefinite or the reserve is deprecated, it escalates to a permanent freeze (Critical).

---

### Likelihood Explanation

Aave governance has paused individual reserves and the entire pool during past security incidents (e.g., the Aave v2 CRV incident, the Aave v3 emergency mode). The `LRTWithdrawalManager` holds aWETH on behalf of rsETH withdrawers; any Aave WETH reserve pause directly triggers this freeze. No attacker action is required — the trigger is a standard, documented Aave governance operation.

---

### Recommendation

1. **Add `try/catch` in `_processWithdrawalCompletion`** around the `_withdrawFromAave` call. If Aave withdrawal fails, revert with a descriptive error (e.g., `AaveWithdrawalFailed`) so the state is not corrupted, but also add a separate admin function to service the withdrawal from contract ETH balance directly when Aave is unavailable.

2. **Separate the "disable" action from the "withdraw" action** in `setAaveIntegrationEnabled`. Allow the flag to be set to `false` independently, and provide a separate `withdrawAllFromAave` function. This ensures the integration can be disabled even when Aave is paused.

3. **Add a force-disable function** callable by PAUSER_ROLE that sets `isAaveIntegrationEnabled = false` without attempting any Aave interaction, so that `completeWithdrawal` can fall back to contract ETH balance immediately.

---

### Proof of Concept

1. Manager enables Aave integration and calls `unlockQueue` for ETH. The unlocked ETH is deposited to Aave via `depositToAaveExternal`. `totalETHDepositedToAave` is now > 0 and `address(this).balance ≈ 0`.
2. Aave governance pauses the WETH reserve (a documented Aave governance action).
3. A user whose withdrawal was unlocked in step 1 calls `completeWithdrawal(ETH_TOKEN, ...)`.
4. `_processWithdrawalCompletion` checks `address(this).balance < request.expectedAssetAmount` → true.
5. `_withdrawFromAave(amountNeeded)` is called → `aaveWETHGateway.withdrawETH(...)` reverts because the reserve is paused.
6. The entire `completeWithdrawal` call reverts. The user's rsETH has already been burned (at `initiateWithdrawal` time) and their withdrawal request is locked in the queue with no way to complete it.
7. Manager calls `setAaveIntegrationEnabled(false)` → reverts at `_collectInterestToTreasury` → `aaveWETHGateway.withdrawETH` reverts.
8. PAUSER_ROLE calls `emergencyWithdrawFromAave` → reverts at `_withdrawFromAave` → same reason.
9. All user ETH withdrawals are frozen for the duration of the Aave pause with no protocol escape hatch.

    function _withdrawFromAave(uint256 amount) internal returns (uint256 withdrawnAmount) {
        if (amount == 0) return 0;

        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        if (aaveBalance == 0) revert InsufficientAaveBalance();

        // Only withdraw up to the principal amount (don't use accrued interest for user withdrawals)
        uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave ? aaveBalance : totalETHDepositedToAave;

        withdrawnAmount = amount > withdrawablePrincipal ? withdrawablePrincipal : amount;
        if (withdrawnAmount == 0) return 0;

        aaveWETHGateway.withdrawETH(aavePool, withdrawnAmount, address(this));
        totalETHDepositedToAave -= withdrawnAmount;

        emit ETHWithdrawnFromAave(withdrawnAmount, totalETHDepositedToAave);
    }
