### Title
Stale Chainlink Price Accepted in `getAssetPrice` Enables Deposit-at-Inflated-Price Attack, Diluting Existing rsETH Holders — (`contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice` consumes `latestRoundData` with no staleness (`updatedAt`) or `answeredInRound` validation. When a Chainlink LST feed is stale at a price higher than the true current price (e.g., after a slashing event that the feed has not yet reflected), an attacker can deposit at the inflated stale price, receive excess rsETH, wait for the feed to update and `updateRSETHPrice()` to be called, then withdraw — extracting ETH value from existing rsETH holders.

---

### Finding Description

**Root cause — `ChainlinkPriceOracle.getAssetPrice`:**

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52-54
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`updatedAt`, `roundId`, and `answeredInRound` are all discarded. No `block.timestamp - updatedAt > heartbeat` guard exists. [1](#0-0) 

**Deposit minting uses this price directly:**

```solidity
// LRTDepositPool.getRsETHAmountToMint L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [2](#0-1) 

**`updateRSETHPrice` recomputes TVL using the same oracle, now at the corrected lower price:**

```solidity
// LRTOracle._getTotalEthInProtocol L339
uint256 assetER = getAssetPrice(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
``` [3](#0-2) 

**Withdrawal uses the stored (now-lower) `rsETHPrice`:**

```solidity
// LRTWithdrawalManager.getExpectedAssetAmount L593
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
``` [4](#0-3) 

**Payout at unlock takes `min(expectedAssetAmount, currentReturn)`:**

```solidity
// LRTWithdrawalManager._calculatePayoutAmount L833-834
uint256 currentReturn = (request.rsETHUnstaked * rsETHPrice) / assetPrice;
return (request.expectedAssetAmount < currentReturn) ? request.expectedAssetAmount : currentReturn;
``` [5](#0-4) 

Since the attacker initiates withdrawal *after* `updateRSETHPrice()` has already set the lower `rsETHPrice`, both `expectedAssetAmount` and `currentReturn` reflect the same post-correction prices, so the minimum equals the inflated amount.

---

### Impact Explanation

**Concrete numerical example:**

| Variable | Value |
|---|---|
| True LST price | 1.00 ETH |
| Stale Chainlink price | 1.10 ETH (+10%) |
| rsETHPrice before attack | 1.00 ETH |
| Protocol TVL (100 LST) | 100 ETH |
| rsETH supply | 100 |

1. **Attacker deposits 10 LST** (true value = 10 ETH) at stale price 1.10:
   - rsETH minted = `10 × 1.10 / 1.00` = **11 rsETH** (fair share = 10)
   - New supply = 111, new TVL (true) = 110 ETH

2. **Chainlink updates; `updateRSETHPrice()` called:**
   - `totalETHInProtocol` = 110 ETH (true price)
   - `newRsETHPrice` = 110 / 111 ≈ **0.9910 ETH**

3. **Attacker initiates withdrawal of 11 rsETH:**
   - `expectedAssetAmount` = `11 × 0.9910 / 1.00` ≈ **10.90 LST**

4. **Attacker receives 10.90 LST (10.90 ETH) — profit: 0.90 ETH**

5. **Existing 100 rsETH holders now hold** `100 × 0.9910` = **99.10 ETH** (loss: 0.90 ETH)

This is direct theft of existing user funds through share dilution.

---

### Likelihood Explanation

- Chainlink LST feeds (e.g., stETH/ETH) have heartbeat intervals of up to 24 hours and deviation thresholds of 0.5–1%. During this window, the on-chain price can lag the true price.
- A stale-high price occurs when the LST loses value (slashing event, depeg) and the feed has not yet updated. Slashing events on major LST protocols (Lido, Rocket Pool) have occurred historically.
- `updateRSETHPrice()` is a public, permissionless function — anyone can call it after the Chainlink feed updates.
- The attacker does not need to front-run any specific transaction; they only need to observe that the feed is stale and deposit before it updates.
- The `pricePercentageLimit` downside-pause only triggers if the price drop exceeds the configured threshold. A drop within the limit (e.g., < 1%) allows the attack to proceed without triggering a pause. [6](#0-5) 

---

### Recommendation

Add a staleness check in `ChainlinkPriceOracle.getAssetPrice`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

require(answeredInRound >= roundId, "Stale round");
require(block.timestamp - updatedAt <= MAX_STALENESS, "Stale price");
require(price > 0, "Non-positive price");
```

`MAX_STALENESS` should be set per-feed based on the Chainlink heartbeat (e.g., 3600 seconds for 1-hour feeds, 86 400 seconds for 24-hour feeds).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Fork mainnet at a block where the stETH/ETH Chainlink feed is near its
// heartbeat boundary (price hasn't updated for ~23h).

contract StalePricePoC {
    ILRTDepositPool depositPool = ILRTDepositPool(DEPOSIT_POOL);
    ILRTWithdrawalManager withdrawalManager = ILRTWithdrawalManager(WITHDRAWAL_MANAGER);
    ILRTOracle oracle = ILRTOracle(LRT_ORACLE);
    IERC20 stETH = IERC20(STETH);
    IERC20 rsETH = IERC20(RSETH);

    function attack() external {
        // 1. Observe: Chainlink stETH/ETH feed is stale at 1.10 ETH
        //    True price (from secondary source) is 1.00 ETH
        uint256 stalePriceReported = oracle.getAssetPrice(address(stETH));
        // stalePriceReported == 1.10e18

        // 2. Deposit 10 stETH at stale-high price → receive 11 rsETH
        stETH.approve(address(depositPool), 10e18);
        depositPool.depositAsset(address(stETH), 10e18, 0, "");
        // rsETH balance = 11e18

        // 3. Simulate Chainlink feed update (warp time / use mock)
        //    Now latestRoundData returns 1.00 ETH

        // 4. Call updateRSETHPrice() — public, permissionless
        oracle.updateRSETHPrice();
        // rsETHPrice drops from 1.00 to ~0.991

        // 5. Initiate withdrawal of 11 rsETH
        rsETH.approve(address(withdrawalManager), 11e18);
        withdrawalManager.initiateWithdrawal(address(stETH), 11e18, "");
        // expectedAssetAmount ≈ 10.9 stETH recorded

        // 6. Wait withdrawalDelayBlocks (~8 days), operator calls unlockQueue()
        // 7. completeWithdrawal() → receive ~10.9 stETH

        // Profit: ~0.9 stETH extracted from existing rsETH holders
    }
}
```
