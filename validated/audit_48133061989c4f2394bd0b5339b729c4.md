Audit Report

## Title
Stale `rsETHPrice` vs. Live `swETH.getRate()` Asymmetry in `instantWithdrawal` Enables Over-Extraction via Block Stuffing — (`contracts/LRTWithdrawalManager.sol`, `contracts/LRTOracle.sol`, `contracts/oracles/SwETHPriceOracle.sol`)

## Summary

`LRTOracle.rsETHPrice` is a cached storage variable updated only when `updateRSETHPrice()` is explicitly called, while `SwETHPriceOracle.getAssetPrice()` reads `swETH.getRate()` live on every invocation. `getExpectedAssetAmount` divides the stale numerator by the live denominator, so a drop in `swETH.getRate()` (e.g., a slashing event) that has not yet been reflected in `rsETHPrice` causes `instantWithdrawal` to disburse more swETH than the burned rsETH is worth. An attacker can extend this staleness window by block-stuffing to prevent the keeper from landing `updateRSETHPrice()`, then drain the inflated payout from `LRTUnstakingVault`.

## Finding Description

**Root cause — price source asymmetry**

`LRTOracle.rsETHPrice` is a plain `uint256` storage slot with no accompanying freshness timestamp: [1](#0-0) 

It is written only inside `_updateRsETHPrice()`, which is reached exclusively through the public `updateRSETHPrice()` (or the manager-gated `updateRSETHPriceAsManager()`): [2](#0-1) 

`SwETHPriceOracle.getAssetPrice()`, by contrast, delegates directly to `swETH.getRate()` on every call — no caching, no staleness guard: [3](#0-2) 

**Vulnerable formula**

`getExpectedAssetAmount` combines both sources: [4](#0-3) 

```
underlyingToReceive = amount * rsETHPrice(stale) / swETH.getRate()(live)
```

When `swETH.getRate()` falls (slashing) but `rsETHPrice` has not yet been refreshed, the denominator shrinks while the numerator stays inflated, yielding a larger `underlyingToReceive` than the burned rsETH actually represents.

**`instantWithdrawal` consumes this value without any price-band guard**

The computed `assetAmountUnlocked` is used directly to burn rsETH and redeem from the vault: [5](#0-4) 

No minimum/maximum price bounds are checked — unlike `unlockQueue`, which enforces `_validatePrices` with caller-supplied `minimumAssetPrice` / `maximumAssetPrice` / `minimumRsEthPrice` / `maximumRsEthPrice`: [6](#0-5) 

**Block stuffing extends the staleness window**

`updateRSETHPrice()` is permissionless and is normally invoked by a keeper bot. An attacker fills every block with their own transactions to prevent the keeper from landing `updateRSETHPrice()`. During that window, `rsETHPrice` remains at its pre-slash value while `swETH.getRate()` reflects the post-slash rate. The attacker calls `instantWithdrawal(swETH, rsETHAmount)` and collects the inflated payout.

**Why the downside-protection mechanism does not help**

`_updateRsETHPrice()` contains a circuit-breaker that pauses the protocol if the computed new price falls beyond `pricePercentageLimit`: [7](#0-6) 

This protection only fires when `updateRSETHPrice()` is actually called. Block stuffing prevents that call, so the protocol remains unpaused and the stale price persists indefinitely.

## Impact Explanation

The attacker redeems rsETH at a stale high rsETH/ETH rate while paying a live low swETH/ETH rate, extracting more swETH from `LRTUnstakingVault` than the burned rsETH is worth. The surplus is borne by other vault depositors and queued withdrawers. This matches the allowed impact class **Low — Block stuffing**: the attack is only economically rational when the profit from the rate divergence exceeds the sustained block-stuffing cost, bounding both likelihood and magnitude.

## Likelihood Explanation

Two simultaneous conditions are required: (1) a meaningful drop in `swETH.getRate()` (a slashing event on Swell) and (2) the attacker successfully stuffing enough blocks to delay `updateRSETHPrice()`. Slashing events are rare; block stuffing on Ethereum mainnet is expensive. The attack is economically rational only when the rate divergence is large enough to cover block-stuffing costs, making exploitation uncommon but not impossible.

## Recommendation

1. **Add a staleness timestamp to `rsETHPrice`** and revert in `getExpectedAssetAmount` (and therefore `instantWithdrawal`) if the stored price is older than a configurable heartbeat (e.g., 1 hour).
2. **Alternatively**, compute `rsETHPrice` on-the-fly inside `instantWithdrawal` by calling `_updateRsETHPrice()` inline (or re-deriving it from `_getTotalEthInProtocol`) so both prices are evaluated at the same block.
3. **Add a price-band guard to `instantWithdrawal`** analogous to the `minimumAssetPrice` / `maximumAssetPrice` / `minimumRsEthPrice` / `maximumRsEthPrice` bounds already enforced in `unlockQueue`.

## Proof of Concept

```solidity
// Fork test against a mainnet fork where swETH is a supported asset.

// 1. Record pre-slash prices.
uint256 rsETHPriceBefore = lrtOracle.rsETHPrice();       // cached, will go stale
uint256 swETHRateBefore  = ISwETH(swETH).getRate();

// 2. Simulate block stuffing: do NOT call updateRSETHPrice().
//    Mock a 5% swETH slash so getRate() returns a lower value.
vm.mockCall(
    swETH,
    abi.encodeWithSelector(ISwETH.getRate.selector),
    abi.encode(swETHRateBefore * 95 / 100)
);

// 3. rsETHPrice is still the pre-slash value; swETH.getRate() is now 5% lower.
uint256 rsETHAmount   = 1 ether;
uint256 expectedSwETH = lrtWithdrawalManager.getExpectedAssetAmount(swETH, rsETHAmount);
// expectedSwETH = rsETHAmount * rsETHPriceBefore / (swETHRateBefore * 0.95)
//               > rsETHAmount * rsETHPriceBefore / swETHRateBefore  (fair value)

// 4. Call instantWithdrawal — burns rsETHAmount, receives inflated expectedSwETH.
lrtWithdrawalManager.instantWithdrawal(swETH, rsETHAmount, "");

// 5. Assert over-extraction.
uint256 swETHRateNow   = ISwETH(swETH).getRate();        // still mocked at 95%
uint256 rsETHPriceNow  = lrtOracle.rsETHPrice();         // still stale (pre-slash)
assertGt(
    expectedSwETH * swETHRateNow,   // ETH value of swETH received
    rsETHAmount   * rsETHPriceNow,  // ETH value of rsETH burned (at stale price)
    "attacker extracted more ETH value than rsETH burned"
);
// Passes because swETHRateNow < swETHRateBefore while rsETHPriceNow == rsETHPriceBefore.
```
