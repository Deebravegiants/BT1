### Title
Missing Incomplete-Round Guard in `ChainlinkPriceOracle.getAssetPrice` Enables Any Caller to Trigger Protocol Auto-Pause — (`contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice` accepts a Chainlink `latestRoundData()` response without validating `updatedAt` or `answer`. When a feed is in an incomplete round (`updatedAt=0, answer=0`), the function silently returns `0`. This zero price propagates through `LRTOracle._getTotalEthInProtocol()`, deflates the computed `newRsETHPrice`, and — because `updateRSETHPrice()` is a permissionless `public` function — any caller can trigger the downside-protection auto-pause of `LRTDepositPool` and `LRTWithdrawalManager`, temporarily freezing all user fund flows.

---

### Finding Description

**Root cause — missing validation in `ChainlinkPriceOracle.getAssetPrice`:**

`ChainlinkPriceOracle.getAssetPrice` (lines 49–55) calls `latestRoundData()` and immediately casts the raw `answer` to `uint256`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

There is no check for:
- `updatedAt == 0` (incomplete round)
- `answer <= 0` (invalid/zero price)
- `answeredInRound < roundId` (stale round)

By contrast, the sibling oracle `ChainlinkOracleForRSETHPoolCollateral.getRate` (lines 30–32) explicitly guards all three cases and reverts:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

**Propagation through `_getTotalEthInProtocol`:**

`LRTOracle._getTotalEthInProtocol` iterates over all supported assets and accumulates `totalAssetAmt.mulWad(assetER)`. If any asset's oracle returns `0`, that asset's entire ETH contribution is zeroed out, deflating `totalETHInProtocol`. [3](#0-2) 

**Auto-pause trigger in `_updateRsETHPrice`:**

The deflated `totalETHInProtocol` produces a `newRsETHPrice` well below `highestRsethPrice`. The downside-protection block at lines 270–281 then fires:

```solidity
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;
    }
``` [4](#0-3) 

**Permissionless entrypoint:**

`updateRSETHPrice()` carries no role restriction — it is `public whenNotPaused`, callable by any EOA or contract: [5](#0-4) 

---

### Impact Explanation

When triggered, `LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()` are called, halting all deposits, withdrawals, and claims. Unpausing requires `onlyLRTAdmin` intervention. Until admin acts, all user fund flows are frozen — matching the **Medium: Temporary freezing of funds** impact scope.

---

### Likelihood Explanation

Chainlink incomplete rounds (`updatedAt=0`) are a documented, non-attacker-controlled Chainlink infrastructure event. They are rare but real. The attacker's role is minimal: observe the feed state and call the public `updateRSETHPrice()`. No special permissions, no front-running, no governance capture is required. Likelihood is **Low** (rare external condition) but the path is fully concrete and locally testable.

---

### Recommendation

Add the same three guards present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
``` [6](#0-5)
