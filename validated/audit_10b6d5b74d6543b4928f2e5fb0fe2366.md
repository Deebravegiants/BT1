Audit Report

## Title
Fee Baseline Uses Depressed `rsETHPrice` Instead of `highestRsethPrice`, Charging Fees on Slashing Recovery — (`contracts/LRTOracle.sol`)

## Summary

`_updateRsETHPrice()` computes the protocol fee baseline as `previousTVL = rsethSupply × rsETHPrice` using the last stored price. When a price drop falls within `pricePercentageLimit` (no pause triggered), `rsETHPrice` is written down to the depressed value at line 313. On subsequent recovery, the fee is charged on the full delta from the depressed baseline — including the recovery of previously-slashed value — rather than only on genuine new yield above the prior peak. This causes excess rsETH to be minted to the treasury, diluting all existing rsETH holders.

## Finding Description

In `_updateRsETHPrice()`, the fee baseline is computed at line 234:

```solidity
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);
```

`rsETHPrice` is the last stored price, unconditionally overwritten at line 313 on every non-pausing call. `highestRsethPrice` is tracked separately and updated only upward (lines 294–296), but is **never used** in the fee calculation.

The downside-pause guard at lines 270–281 only fires when `pricePercentageLimit > 0` and the drop exceeds `pricePercentageLimit × highestRsethPrice`. When it fires, the function returns early at line 281 **without** updating `rsETHPrice`, preserving the baseline. But when the drop is within the limit (or `pricePercentageLimit == 0`), execution falls through to line 313 and `rsETHPrice` is written to the depressed value.

On the subsequent recovery call:
- `previousTVL = rsethSupply × depressedPrice` (line 234)
- `totalETHInProtocol` = TVL at recovered price (line 231)
- `rewardAmount = totalETHInProtocol − previousTVL` (line 245) — inflated by the recovery portion
- `protocolFeeInETH = rewardAmount × protocolFeeInBPS / 10_000` (line 246)

The fee is therefore charged on both the recovery of slashed value **and** genuine new yield above the old peak. `highestRsethPrice` is never consulted for the fee baseline despite being maintained precisely for this purpose. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

## Impact Explanation

Excess rsETH is minted to the treasury at lines 304–307. Because rsETH is a share token, minting extra shares to the treasury dilutes every existing holder's proportional claim on the underlying ETH. The dilution equals the fee taken on the recovery portion — a direct, quantifiable transfer of value from rsETH holders to the treasury. This constitutes **theft of unclaimed yield** (High severity). [5](#0-4) 

## Likelihood Explanation

- `updateRSETHPrice()` is `public whenNotPaused` — callable by any unprivileged address with no special role or deposit required.
- `protocolFeeInBPS > 0` is the normal operational state (manager-configurable, capped at 1500 BPS per `LRTConfig`).
- EigenLayer slashing or LST de-pegging events can cause small, within-limit price drops; `pricePercentageLimit` is admin-set and may be larger than the actual drop, or set to zero.
- No attacker-controlled input is required; the condition arises from normal market events followed by a public call to `updateRSETHPrice()`.
- The condition is repeatable: every slashing-and-recovery cycle that stays within the limit re-triggers the inflated fee. [6](#0-5) 

## Recommendation

Replace the fee baseline with the maximum of the last stored price and the all-time-high price, so fees are only charged on genuine new yield above the previous peak:

```solidity
uint256 feeBaselinePrice = rsETHPrice > highestRsethPrice ? rsETHPrice : highestRsethPrice;
uint256 previousTVL = rsethSupply.mulWad(feeBaselinePrice);
```

This anchors `previousTVL` to the peak TVL, ensuring `rewardAmount` only captures growth that is genuinely new yield.

## Proof of Concept

**State setup:**
```
rsethSupply        = 1000e18
rsETHPrice         = 1.1e18   (highestRsethPrice = 1.1e18)
protocolFeeInBPS   = 1000     (10%)
pricePercentageLimit = 0.20e18  (20%)
```

**Step 1 — price drops to 0.9e18 (18% drop, within 20% limit, no pause):**
```
totalETHInProtocol = 900e18
previousTVL        = 1000 × 1.1 = 1100e18
900 < 1100 → no fee
rsETHPrice updated to 0.9e18
highestRsethPrice  stays at 1.1e18
```

**Step 2 — price recovers to 1.2e18 (anyone calls updateRSETHPrice()):**
```
totalETHInProtocol = 1200e18
previousTVL        = 1000 × 0.9 = 900e18   ← depressed baseline
rewardAmount       = 1200 - 900 = 300e18
protocolFeeInETH   = 300e18 × 10% = 30e18

Correct fee (yield above peak only):
  rewardAmount_correct = 1200 - 1100 = 100e18
  protocolFeeInETH_correct = 10e18

Excess fee minted to treasury: 20e18 ETH-equivalent in rsETH
→ treasury receives 3× the legitimate fee, stealing yield from holders
```

**Foundry test assertion:**
```solidity
// After Step 2, assert rsethAmountToMintAsProtocolFee == 10e18 / newRsETHPrice
// not 30e18 / newRsETHPrice
assertEq(
    rsETH.balanceOf(treasury),
    uint256(10e18).divWad(newRsETHPrice),
    "fee should only cover yield above peak"
);
```
