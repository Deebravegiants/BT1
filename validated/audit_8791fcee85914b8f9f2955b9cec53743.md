### Title
Oracle Auto-Pause Freezes User Assets After rsETH Is Already Burned in Withdrawal Queue - (File: contracts/LRTOracle.sol, contracts/LRTWithdrawalManager.sol)

### Summary

`LRTOracle.updateRSETHPrice()` is a permissionless public function that automatically pauses `LRTWithdrawalManager` when the rsETH price drops beyond `pricePercentageLimit`. Because `completeWithdrawal()` enforces `whenNotPaused`, users whose rsETH has already been irreversibly burned by `unlockQueue()` cannot retrieve their owed assets until an admin manually unpauses ŌĆö creating a state-triggered temporary fund freeze analogous to the Absorber recovery-mode lock described in the reference report.

---

### Finding Description

The withdrawal lifecycle in `LRTWithdrawalManager` has three sequential steps:

1. **`initiateWithdrawal()`** ŌĆö user's rsETH is transferred into the contract (`whenNotPaused`).
2. **`unlockQueue()`** ŌĆö operator burns the held rsETH (`burnFrom`) and redeems the corresponding assets from `LRTUnstakingVault` into the withdrawal manager (`whenNotPaused`).
3. **`completeWithdrawal()`** ŌĆö user receives the redeemed assets (`whenNotPaused`).

After step 2, the user's rsETH is permanently destroyed and the owed ETH/LST sits inside `LRTWithdrawalManager`. At this exact moment the user is in an irrecoverable intermediate state: they hold neither rsETH nor the underlying asset.

`LRTOracle.updateRSETHPrice()` is declared `public` with no access restriction. Its internal logic computes a new rsETH price and, if the price has fallen more than `pricePercentageLimit` below the all-time high, it calls `withdrawalManager.pause()` and returns without updating the stored price. This auto-pause can be triggered by any external caller at any time the on-chain price satisfies the threshold condition (e.g., immediately after a slashing event is reflected in EigenLayer strategy balances).

Once `LRTWithdrawalManager` is paused, `completeWithdrawal()` reverts unconditionally. The user's assets remain locked in the contract until an admin calls `LRTWithdrawalManager.unpause()`.

---

### Impact Explanation

**Temporary freezing of funds (Medium).** After `unlockQueue` burns a user's rsETH, the user has no rsETH to reclaim and cannot call `completeWithdrawal()` while the contract is paused. The ETH or LST owed to the user is stranded in `LRTWithdrawalManager`. The freeze lasts until admin intervention. There is no on-chain mechanism that automatically unpauses the contract or allows withdrawal completion to bypass the pause guard in this post-burn state.

---

### Likelihood Explanation

**Medium.** A real rsETH price drop (e.g., from EigenLayer slashing) is required to satisfy the threshold. However, once the price condition is met, the trigger is fully permissionless ŌĆö any address can call `updateRSETHPrice()` to atomically pause the withdrawal manager. The window between `unlockQueue` (rsETH burned) and `completeWithdrawal` (assets transferred) is non-zero in practice because the two calls are made in separate transactions, often by different actors (operator vs. user).

---

### Recommendation

- **Short term:** Remove `whenNotPaused` from `completeWithdrawal()` (and `completeWithdrawalForUser()`). Requests that have already been unlocked (rsETH burned, assets redeemed) represent a firm obligation; blocking their completion during a pause does not protect the protocol and only harms users who have already surrendered their rsETH.
- **Long term:** Separate the "deposit protection" pause (blocking new `initiateWithdrawal` calls) from the "completion" path. A two-tier pause ŌĆö one that stops new entries and one that stops completions ŌĆö would allow the protocol to halt new exposure while still honouring existing obligations.

---

```
// Step 1 ŌĆō Alice initiates withdrawal (rsETH leaves Alice's wallet)
lrtWithdrawalManager.initiateWithdrawal(ETH_TOKEN, 1 ether, "ref");

// Step 2 ŌĆō Operator unlocks the queue; Alice's rsETH is burned, ETH sits in WithdrawalManager
lrtWithdrawalManager.unlockQueue(ETH_TOKEN, nextNonce, minAP, minRP, maxAP, maxRP);
// At this point: Alice.rsETH == 0, WithdrawalManager.balance += expectedETH

// Step 3 ŌĆō EigenLayer slashing event reduces totalETHInProtocol
// newRsETHPrice < highestRsethPrice * (1 - pricePercentageLimit)

// Step 4 ŌĆō Anyone triggers the auto-pause
lrtOracle.updateRSETHPrice();
// LRTOracle._updateRsETHPrice() executes:
//   withdrawalManager.pause();   ŌåÉ LRTWithdrawalManager is now paused
//   _pause();  return;

// Step 5 ŌĆō Alice tries to collect her ETH
lrtWithdrawalManager.completeWithdrawal(ETH_TOKEN, "ref");
// Ōå│ reverts: "Pausable: paused"
// Alice's rsETH is gone; her ETH is frozen in the contract.
```
