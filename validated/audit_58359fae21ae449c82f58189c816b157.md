Audit Report

## Title
Protocol Fee Permanently Lost When `updateRSETHPrice()` Is Called While `LRTDepositPool` or `LRTWithdrawalManager` Is Paused - (File: contracts/LRTOracle.sol)

## Summary
`updateRSETHPrice()` is publicly callable whenever `LRTOracle.paused == false`, but `_updateRsETHPrice()` internally skips fee minting when `lrtDepositPool.paused() || withdrawalManager.paused()` is true. Despite skipping the fee, `rsETHPrice` is unconditionally advanced to the current TVL-derived value, permanently moving the fee-accounting baseline forward. All yield accrued during the partial-pause window escapes the protocol fee mechanism entirely.

## Finding Description
`updateRSETHPrice()` passes its only access check — `whenNotPaused` — whenever `LRTOracle.paused == false` (L87–89). Inside `_updateRsETHPrice()`, a broader `protocolPaused` flag is computed at L240 that also covers `lrtDepositPool.paused()` and `withdrawalManager.paused()`. When `protocolPaused == true`, the fee calculation at L244–247 produces `protocolFeeInETH = 0`. However, `newRsETHPrice` is computed at L250 as `(totalETHInProtocol - 0).divWad(rsethSupply)` — fully reflecting the yield earned during the pause — and is written unconditionally to `rsETHPrice` at L313. The only code path that skips the L313 write is the downside-protection early return at L281, which is irrelevant here since yield accrual increases the price. On the next call after unpausing, `previousTVL = rsethSupply.mulWad(rsETHPrice)` already incorporates the paused-window yield, so `totalETHInProtocol ≈ previousTVL` and `protocolFeeInETH` is again zero. The fee for the entire pause window is irrecoverably lost.

## Impact Explanation
The protocol treasury permanently forfeits the rsETH fee that should have been minted on yield accrued while `lrtDepositPool` or `withdrawalManager` was paused. The fee is not deferred — the baseline is advanced before the fee is collected, making recovery impossible. This constitutes **High — Theft of unclaimed yield** from the protocol treasury.

## Likelihood Explanation
`LRTDepositPool` and `LRTWithdrawalManager` are paused independently of `LRTOracle` during routine security responses or EigenLayer-related incidents; `LRTOracle` is intentionally kept active so the price feed remains available. During any such window, `updateRSETHPrice()` is callable by any EOA or contract with no special privileges, making the trigger trivially reachable. The condition recurs on every partial-pause event, and the loss scales with both pause duration and yield rate. Likelihood is **Medium**.

## Recommendation
When `protocolPaused == true`, return early before writing `rsETHPrice`, preserving the fee baseline for collection after unpausing:

```solidity
if (protocolPaused) {
    return; // do not advance rsETHPrice; fee will be collected after unpause
}
```

Alternatively, revert so callers are explicitly informed the update cannot proceed while the protocol is partially paused.

## Proof of Concept
**Setup:** `rsethSupply = 1 000 rsETH`, `rsETHPrice = 1.05 ETH`, `protocolFeeInBPS = 1000` (10%), `previousTVL = 1 050 ETH`.

1. `lrtDepositPool` is paused; `LRTOracle.paused = false`.
2. EigenLayer rewards push `totalETHInProtocol` to `1 060 ETH` (10 ETH yield).
3. Any EOA calls `updateRSETHPrice()`:
   - `whenNotPaused` passes.
   - `protocolPaused = true` → `protocolFeeInETH = 0`.
   - `newRsETHPrice = 1 060 / 1 000 = 1.06 ETH`.
   - `rsETHPrice` written as `1.06 ETH` (L313).
4. `lrtDepositPool` is unpaused.
5. Next `updateRSETHPrice()` call: `previousTVL = 1 000 × 1.06 = 1 060 ETH`; `totalETHInProtocol ≈ 1 060 ETH`; `protocolFeeInETH = 0`.

**Result:** 10 ETH of yield generates **0 protocol fee** instead of the expected 1 ETH worth of rsETH minted to the treasury. A Foundry fork test can confirm this by (a) deploying the contracts, (b) pausing `lrtDepositPool`, (c) simulating yield accrual via mock asset price increase, (d) calling `updateRSETHPrice()` as an unprivileged address, (e) unpausing, (f) calling `updateRSETHPrice()` again, and (g) asserting that the treasury received zero rsETH across both calls despite a positive TVL delta.
