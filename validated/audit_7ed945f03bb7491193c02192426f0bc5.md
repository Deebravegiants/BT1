### Title
Missing Qi_self-to-Q Consistency Check in `attach_private_scalar` Allows Accepted Invalid Key Blob — (`src/cbmpc/api/schnorr2pc.cpp`)

---

### Summary

`coinbase::api::schnorr_2p::attach_private_scalar` verifies that the caller-supplied `x_fake * G == Qi_fake` but never verifies that `Qi_fake` is consistent with the blob's stored global public key `Q_compressed`. Because the `schnorr_2p` `key_blob_v1_t` does not persist `Qi_self`, an attacker who supplies a self-consistent but wrong `(x_fake, Qi_fake)` pair receives `SUCCESS` and obtains a structurally valid blob whose `x_share` is cryptographically inconsistent with `Q`. Signing with that blob defers the failure to the BIP340 final-verify step.

---

### Finding Description

**Blob format — what is and is not stored**

`key_blob_v1_t` in `src/cbmpc/api/schnorr2pc.cpp` stores only the global public key and the private scalar:

```
Q_compressed   ← Q = Q1 + Q2  (global key)
x_share        ← this party's private scalar
``` [1](#0-0) 

`Qi_self` (this party's individual share point) is **not** stored in the blob. It is computed on-the-fly in `get_public_share_compressed` as `x_share * G`. [2](#0-1) 

**What `attach_private_scalar` checks**

The function performs two checks:

1. `x * G == Qi_self` (caller-supplied point) — line 221
2. `Q` from the blob is a valid curve point — lines 224–226 [3](#0-2) 

**What it does not check**

There is no check that `Qi_self + Q_other == Q`. Because `Q_other` is not stored in the blob, the function has no way to enforce this invariant. An attacker who supplies a fresh, self-consistent `(x_fake, Qi_fake = x_fake * G)` pair satisfies check (1) trivially, while the blob's `Q_compressed` remains the legitimate global key.

**Contrast with `schnorr_mp`**

The multi-party variant stores `Qis_compressed` (a map of every party's share point) in the blob, so its `attach_private_scalar` can explicitly verify that the supplied `public_share_compressed` matches the stored `Qi_self`:

> "This validates that x matches the key blob by checking: `public_share_compressed` matches the blob's Qi_self, and x*G == Qi_self." [4](#0-3) [5](#0-4) 

The `schnorr_2p` header comment makes no such claim — it only promises `x*G == public_share_compressed`: [6](#0-5) 

---

### Impact Explanation

`attach_private_scalar` returns `SUCCESS` and writes `x_fake` into a blob whose `Q_compressed` encodes the legitimate global key `Q = Q1 + Q2`. The resulting blob is structurally valid and passes all blob-parsing checks (`deserialize_key_blob`, `blob_to_key`). When `sign` is called with this blob, the protocol runs to completion but produces a partial signature computed under `x_fake` rather than the real `x1`; the BIP340 final-verify step then rejects the signature. The failure is silently deferred from key-restore time to signing time.

This is a **public-API-reachable validation bypass that produces accepted invalid cryptographic output** — matching the "High" impact tier in the scope definition. [7](#0-6) 

---

### Likelihood Explanation

The attack requires the adversary to:
1. Possess a legitimate public blob (output of `detach_private_scalar`).
2. Call `attach_private_scalar` with a self-chosen `(x_fake, Qi_fake)` pair.

Both are reachable by an unprivileged API caller who has obtained the public blob (which is explicitly designed to be safe to persist and share). No threshold collusion, no host-memory compromise, and no secret knowledge is required.

---

### Recommendation

Store `Qi_self` explicitly in `key_blob_v1_t` (mirroring the `schnorr_mp` design), and add a check in `attach_private_scalar` that the caller-supplied `public_share_compressed` equals the stored `Qi_self` before accepting the scalar. This closes the gap without changing the external API signature. [1](#0-0) 

---

### Proof of Concept

```
1. Run DKG for P1 and P2 → key_blob_1, key_blob_2.
2. Call detach_private_scalar(key_blob_1) → public_1, x1_fixed.
3. Generate a fresh random scalar x_fake (≠ x1).
4. Compute Qi_fake = x_fake * G (a valid secp256k1 point).
5. Call attach_private_scalar(public_1, x_fake_fixed, Qi_fake_compressed, out_blob).
   → Expect SUCCESS (x_fake * G == Qi_fake passes; no Q-consistency check).
6. Call sign(job1, out_blob, msg, sig1) with P2 using key_blob_2.
   → Expect FAILURE at BIP340 verify (x_fake ≠ x1, so Q1_fake + Q2 ≠ Q).
```

This confirms `attach_private_scalar` accepts a cryptographically invalid blob, silently deferring the failure to signing time. [8](#0-7)

error_t attach_private_scalar(mem_t public_key_blob, mem_t private_scalar_fixed, mem_t public_share_compressed,
                              buf_t& out_key_blob) {
  if (const error_t rv = coinbase::api::detail::validate_mem_arg_max_size(public_key_blob, "public_key_blob",
                                                                          coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))
    return rv;
  key_blob_v1_t pub;
  error_t rv = coinbase::convert(pub, public_key_blob);
  if (rv) return rv;
  if (pub.version != key_blob_version_v1) return coinbase::error(E_FORMAT, "unsupported key blob version");
  if (pub.role > 1) return coinbase::error(E_FORMAT, "invalid key blob role");
  if (static_cast<curve_id>(pub.curve) != curve_id::secp256k1)
    return coinbase::error(E_FORMAT, "invalid key blob curve");
  if (pub.Q_compressed.empty()) return coinbase::error(E_FORMAT, "invalid key blob");

  const auto curve = coinbase::crypto::curve_secp256k1;
  const coinbase::crypto::mod_t& q = curve.order();
  const int order_size = q.get_bin_size();
  if (order_size <= 0) return coinbase::error(E_GENERAL, "invalid curve order size");

  if (const error_t rvm = coinbase::api::detail::validate_mem_arg(private_scalar_fixed, "private_scalar_fixed"))
    return rvm;
  if (private_scalar_fixed.size != order_size) return coinbase::error(E_BADARG, "private_scalar_fixed wrong size");

  if (const error_t rvp = coinbase::api::detail::validate_mem_arg(public_share_compressed, "public_share_compressed"))
    return rvp;

  coinbase::crypto::ecc_point_t Qi_self(curve);
  if (rv = Qi_self.from_bin(curve, public_share_compressed))
    return coinbase::error(rv, "invalid public_share_compressed");
  if (rv = curve.check(Qi_self)) return coinbase::error(rv, "invalid public_share_compressed");

  const coinbase::crypto::bn_t x = coinbase::crypto::bn_t::from_bin(private_scalar_fixed) % q;
  if (!q.is_in_range(x)) return coinbase::error(E_FORMAT, "invalid private_scalar_fixed");

  const auto& G = curve.generator();
  if (x * G != Qi_self) return coinbase::error(E_FORMAT, "x_share mismatch key blob");

  // Validate and normalize global public key encoding.
  coinbase::crypto::ecc_point_t Q(curve);
  if (rv = Q.from_bin(curve, pub.Q_compressed)) return coinbase::error(rv, "invalid key blob");
  if (rv = curve.check(Q)) return coinbase::error(rv, "invalid key blob");

  pub.x_share = x;
  pub.Q_compressed = Q.to_compressed_bin();
  out_key_blob = coinbase::convert(pub);
  return SUCCESS;
}
