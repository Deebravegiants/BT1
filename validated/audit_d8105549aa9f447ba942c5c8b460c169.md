Audit Report

## Title
`getAssetUnstaking` Uses Raw `scaledShares` Without Slashing Adjustment, Overstating TVL and Inflating `rsETH` Price - (File: contracts/NodeDelegator.sol)

## Summary

`NodeDelegator.getAssetUnstaking()` reads `scaledShares` from EigenLayer's withdrawal queue and passes them directly to `strategy.sharesToUnderlyingView()`. EigenLayer's own interface documentation explicitly states these values must be multiplied by the operator's current `maxMagnitude * beaconChainScalingFactor` to obtain actual withdrawable shares. When slashing occurs during the withdrawal delay, the function overstates recoverable assets, inflating `rsETH` price and enabling rsETH holders to redeem at a rate exceeding the protocol's real backing.

## Finding Description

`NodeDelegator.getAssetUnstaking()` calls `_getDelegationManager().getQueuedWithdrawals(address(this))` and uses the returned `withdrawalShares` array directly:

```solidity
uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
amount += strategyAsset == LRTConstants.ETH_TOKEN
    ? sharesToUnstake
    : strategy.sharesToUnderlyingView(sharesToUnstake);
```

The `Withdrawal` struct in `IDelegationManager.sol` explicitly documents that this field contains `scaledShares`, not actual withdrawable shares:

> *"Note that these scaledShares need to be multiplied by the operator's maxMagnitude and beaconChainScalingFactor at completion to include slashing occurring during the queue withdrawal delay."*

The relationship is:
```
scaledShares = depositShares / (maxMagnitude_at_queue * beaconChainScalingFactor_at_queue)
actual_withdrawable = scaledShares * maxMagnitude_at_completion * beaconChainScalingFactor_at_completion
```

If slashing reduces `maxMagnitude` after a withdrawal is queued, `actual_withdrawable < scaledShares`, but the function returns `sharesToUnderlyingView(scaledShares)` — the pre-slash value.

This inflated value propagates through the entire TVL and pricing stack:

1. `getAssetUnstaking` → overstated `assetUnstakingFromEigenLayer` in `LRTDepositPool.getAssetDistributionData`
2. `LRTDepositPool.getTotalAssetDeposits` → overstated total assets
3. `LRTOracle._getTotalEthInProtocol` → overstated ETH backing
4. `LRTOracle.rsETHPrice` → inflated `rsETH` price
5. `LRTWithdrawalManager.getAvailableAssetAmount` → overstated available liquidity

The active-strategy path in `NodeDelegatorHelper.getAssetBalance` correctly calls `DelegationManager.getWithdrawableShares()`, which returns slashing-adjusted shares. The queued-withdrawal path has no equivalent correction.

## Impact Explanation

**Critical — Protocol insolvency / direct theft of user funds.**

When slashing reduces actual recoverable assets, the `rsETH` price remains inflated. Any rsETH holder who redeems at this inflated price receives more underlying assets than the protocol will actually recover from the withdrawal queue. Once the withdrawal completes and the real (lower) asset amount arrives, the protocol has paid out more than it received, leaving remaining holders with under-backed rsETH. With sufficient slashing magnitude or multiple concurrent queued withdrawals, the protocol can be rendered insolvent. `LRTWithdrawalManager.getExpectedAssetAmount` locks in the payout at the time of the withdrawal request using the stored `rsETHPrice`, and `updateRSETHPrice()` is a public function requiring no privileged access.

## Likelihood Explanation

**Medium.** EigenLayer's slashing mechanism is live and operators can be slashed for AVS misbehavior. The withdrawal delay is at least `minWithdrawalDelayBlocks` (7 days on mainnet), providing a window during which slashing can occur after a withdrawal is queued. The Kelp DAO protocol actively delegates to EigenLayer operators and regularly initiates unstaking. The public `updateRSETHPrice()` entry point means no privileged access is required to trigger the price update at the inflated value. The attacker only needs to be an existing rsETH holder.

## Recommendation

Replace the raw `scaledShares` read in `getAssetUnstaking` with a slashing-adjusted calculation. The correct withdrawable shares for a queued withdrawal are:

```solidity
actual_shares = scaledShares * currentMaxMagnitude * currentBeaconChainScalingFactor
```

Alternatively, mirror the approach already used in `NodeDelegatorHelper.getAssetBalance`, which calls `DelegationManager.getWithdrawableShares()`. For queued withdrawals, EigenLayer exposes `getSlashableSharesInQueue` and the `depositScalingFactor` per staker/strategy, which can be used to compute the slashing-adjusted value.

## Proof of Concept

1. NDC is delegated to operator O and holds 1000 stETH in the EigenLayer strategy.
2. Operator calls `initiateUnstaking` — 1000 stETH worth of `depositShares` are queued. EigenLayer stores `scaledShares = depositShares / slashingFactor_at_queue` in the `Withdrawal` struct.
3. During the 7-day delay, operator O is slashed 10% — `maxMagnitude` drops from `1e18` to `0.9e18`.
4. `getAssetUnstaking` reads `scaledShares` and returns `sharesToUnderlyingView(scaledShares)` ≈ 1000 stETH (pre-slash value). The actual recoverable amount is ≈ 900 stETH.
5. Attacker calls `LRTOracle.updateRSETHPrice()` (public) — price is computed using the overstated 1000 stETH, inflating `rsETH` price by ~11%.
6. Attacker (existing rsETH holder) calls `LRTWithdrawalManager.initiateWithdrawal`, locking in `expectedAssetAmount` at the inflated price.
7. Withdrawal completes — only 900 stETH arrives. The protocol pays out the locked-in inflated amount, leaving a 100 stETH deficit socialized across remaining rsETH holders.

**Foundry fork test plan:** Fork mainnet post-slashing-upgrade. Deploy NDC delegated to a test operator. Queue a withdrawal. Use `vm.store` to reduce the operator's `maxMagnitude` in the AllocationManager (simulating a slash). Call `getAssetUnstaking` and assert the returned value exceeds the actual withdrawable amount returned by completing the withdrawal. Then call `updateRSETHPrice`, initiate a withdrawal at the inflated price, complete the EigenLayer withdrawal, and assert the protocol deficit.
