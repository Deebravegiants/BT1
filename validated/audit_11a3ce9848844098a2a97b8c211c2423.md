### Title
Unprivileged Caller Can Front-Run `init` to Seize Full MPC Participant Control - (File: `crates/contract/src/lib.rs`)

### Summary
The `init` function that sets the initial participant set and threshold parameters of the MPC contract lacks the `#[private]` access-control attribute. Because the documented deployment procedure deploys the contract **without** an init call and then calls `init` in a separate transaction, any external account can race to call `init` first and install themselves as the sole participant with threshold 1, gaining complete unauthorized signing authority over the MPC network.

### Finding Description
In `crates/contract/src/lib.rs`, the two initialization entry points are declared as follows:

```rust
// Line 1924-1926 — NO #[private]
#[handle_result]
#[init]
pub fn init(
    parameters: dtos::ThresholdParameters,
    init_config: Option<dtos::InitConfig>,
) -> Result<Self, Error> { … }

// Line 1976-1979 — HAS #[private]
#[private]
#[init]
#[handle_result]
pub fn init_running(…) -> Result<Self, Error> { … }
```

`init_running` is correctly guarded with `#[private]`, which in NEAR restricts the caller to `predecessor_account_id == current_account_id` (i.e., only the contract itself). `init`, however, carries only `#[init]`, which merely prevents re-initialization once state exists — it places **no restriction on who may call it**.

The official deployment procedure (localnet, testnet, and the production TEE cluster script) always deploys the contract in a first transaction with `without-init-call`, then calls `init` in a separate subsequent transaction:

```bash
# Step 1 — deploy (no init)
near contract deploy mpc-contract.test.near use-file "$MPC_CONTRACT_PATH" without-init-call …

# Step 2 — init (separate tx, open window)
near contract call-function as-transaction mpc-contract.test.near init file-args /tmp/init_args.json …
```

Between these two transactions the contract has no state, so `#[init]` will accept the first caller. Any account that submits a valid `init` call before the legitimate operator does will win the race.

### Impact Explanation
An attacker who wins the race supplies their own `ThresholdParameters` — for example, a single participant (themselves) with `threshold = 1`. The contract transitions immediately to `ProtocolContractState::Running` with the attacker as the only recognized participant. From that point:

- Every governance vote (`vote_add_domains`, `vote_new_parameters`, `vote_code_hash`, `vote_tee_verifier_change`, etc.) requires only the attacker's single vote to pass.
- The attacker can add domains, trigger key generation, and become the sole signer for every domain.
- All threshold signatures issued by the MPC network are then under the attacker's unilateral control, enabling unauthorized signing of arbitrary foreign-chain transactions (Bitcoin, Ethereum, Solana, etc.) and unauthorized confidential key derivation (CKD/BLS12-381).
- The legitimate operator cannot recover without redeploying the contract, because `#[init]` prevents a second `init` call once state is written.

This satisfies **Critical — Unauthorized transaction execution, threshold signature issuance, or confidential key derivation output without the required participant authorization**.

### Likelihood Explanation
The window is small (one block gap between deploy and init) but the attack is fully automatable: an attacker monitors the NEAR mempool or block stream for a `DeployContract` action targeting the known MPC contract account, then immediately submits their own `init` call. NEAR block times are ~1 second and transactions are publicly visible before finalization, making front-running straightforward. No privileged access, leaked key, or TEE bypass is required — only a funded NEAR account and knowledge of the contract ABI.

### Recommendation
Add `#[private]` to `init`, mirroring the protection already applied to `init_running`:

```rust
#[private]          // ← add this
#[handle_result]
#[init]
pub fn init(
    parameters: dtos::ThresholdParameters,
    init_config: Option<dtos::InitConfig>,
) -> Result<Self, Error> { … }
```

With `#[private]`, only a call whose `predecessor_account_id == current_account_id` (i.e., a self-call from the contract account) is accepted, which is exactly the pattern used for `init_running`. Alternatively, deploy and initialize atomically in a single transaction using NEAR's `with-init-call` flag, eliminating the open window entirely.

### Proof of Concept

1. Operator broadcasts: `near contract deploy mpc-contract.near use-file mpc.wasm without-init-call …`
2. Attacker observes the deploy transaction in the mempool/block and immediately submits:
   ```json
   near contract call-function as-transaction mpc-contract.near init \
     json-args '{
       "parameters": {
         "threshold": 1,
         "participants": {
           "next_id": 1,
           "participants": [["attacker.near", 0, {"tls_public_key": "ed25519:<attacker_key>", "url": "https://attacker.example"}]]
         }
       }
     }' sign-as attacker.near …
   ```
3. Because `init` has no `#[private]` guard, the call succeeds. The contract enters `Running` state with `attacker.near` as the sole participant and threshold 1.
4. The operator's subsequent legitimate `init` call fails with "contract already initialized."

async fn init_running_rejects_external_callers_pre_initialization() {
    let (worker, contract) = init().await;
    let number_of_participants = 2;
    let (accounts, participants) = gen_accounts(&worker, number_of_participants).await;

    let threshold_parameters = ThresholdParameters::new(
        participants.clone(),
        Threshold::new(number_of_participants as u64),
    )
    .unwrap();

    let init_running_args = serde_json::json!({
            "domains": [],
            "next_domain_id": 0,
            "keyset": Keyset::new(EpochId::new(2), vec![]),
            "parameters": threshold_parameters,
    });

    let execution_error = accounts[0]
        .call(contract.id(), method_names::INIT_RUNNING)
        .max_gas()
        .args_json(init_running_args)
        .transact()
        .await
        .unwrap()
        .into_result()
        .expect_err("method is private and not callable from participant account.");

    let error_message = format!("{:?}", execution_error);

    let expected_error_message = "Smart contract panicked: Method init_running is private";

    assert!(
        error_message.contains(expected_error_message),
        "init_running call was accepted by external caller. expected method to be private. {:?}",
        error_message
    )
