### Title
O(assets × NDCs) Unbounded Gas in `updateRSETHPrice` Call Chain — (`contracts/LRTOracle.sol`, `contracts/LRTDepositPool.sol`)

---

### Summary

`updateRSETHPrice()` is a public, permissionless function. Its internal call chain traverses a nested loop: for every supported asset it calls `getTotalAssetDeposits`, which calls `getAssetDistributionData`, which iterates every NDC and makes multiple external EigenLayer calls per iteration. There is no hard upper bound on either dimension enforced in contract code, making total gas cost O(assets × NDCs × EigenLayer-calls-per-NDC). As the protocol grows legitimately, this function can exceed the block gas limit, permanently freezing the rsETH price and blocking all deposits and withdrawals that depend on it.

---

### Finding Description

**Call chain:**

```
updateRSETHPrice()          [public, no role check]
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ for each asset in supportedAssetList:          ← outer loop (A)
                 getTotalAssetDeposits(asset)
                   └─ getAssetDistributionData(asset)
                        └─ for each NDC in nodeDelegatorQueue: ← inner loop (N)
                             IERC20(asset).balanceOf(ndc)
                             INodeDelegator(ndc).getAssetBalance(asset)
                               └─ NodeDelegatorHelper.getAssetBalance()
                                    └─ delegationManager.getWithdrawableShares()  ← EL call
                             INodeDelegator(ndc).getAssetUnstaking(asset)
                               └─ delegationManager.getQueuedWithdrawals()        ← EL call
                                    └─ for each queued withdrawal × strategies:   ← 3rd loop
``` [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

**Dimension 1 — NDC count:** `maxNodeDelegatorLimit` is initialized to `10` but has no contract-enforced ceiling. `onlyLRTAdmin` can raise it to any value via `updateMaxNodeDelegatorLimit`. [5](#0-4) [6](#0-5) 

**Dimension 2 — Asset count:** `addNewSupportedAsset` is gated by `TIME_LOCK_ROLE` but is otherwise unbounded. The protocol starts with 2 assets and can grow. [7](#0-6) 

**The protocol's own code acknowledges the multiplicative cost** in a comment in `LRTUnstakingVault.sol`:

> `// Need buffer for theoretical operator forced undelegations (ndc count * asset count = 15)`

This confirms the team is aware of the O(N×A) structure and has designed around small numbers — but has not enforced those limits in contract code. [8](#0-7) 

**Per-NDC-per-asset gas cost is not trivial.** Each inner iteration makes:
1. `IERC20.balanceOf` (~2,500 gas)
2. `getAssetBalance` → `delegationManager.getWithdrawableShares` (~15,000–25,000 gas, EigenLayer SLOAD-heavy)
3. `getAssetUnstaking` → `delegationManager.getQueuedWithdrawals` + inner loop over queued withdrawals × strategies (~20,000–60,000 gas, unbounded by queued withdrawal count)

At 10 NDCs × 5 assets × ~50,000 gas/iteration = ~2.5M gas (safe). At 30 NDCs × 10 assets × ~50,000 gas/iteration = ~15M gas (approaching limit). At 50 NDCs × 15 assets = ~37.5M gas (exceeds 30M block limit).

---

### Impact Explanation

If `updateRSETHPrice()` reverts with out-of-gas:
- `rsETHPrice` is never updated (stale price)
- `depositAsset` / `depositETH` call `getRsETHAmountToMint` which reads `rsETHPrice` — minting uses a stale rate
- `_beforeDeposit` calls `_checkIfDepositAmountExceedesCurrentLimit` → `getTotalAssetDeposits` — this also OOGs, reverting all deposits
- `LRTWithdrawalManager.getExpectedAssetAmount` reads `lrtOracle.rsETHPrice()` — withdrawals use stale price or fail
- The oracle's price-deviation guard (`pricePercentageLimit`) can auto-pause the protocol if price drops, but cannot unpause if `updateRSETHPrice` itself OOGs

The result is a permanent freeze of price updates and effective DoS of deposits and withdrawals — matching **Medium: Unbounded gas consumption** and **Medium: Temporary (potentially permanent) freezing of funds**. [9](#0-8) [10](#0-9) 

---

### Likelihood Explanation

The protocol is designed to grow: more LSTs are added as EigenLayer supports them, and more NDCs are added to distribute operator risk. The admin can raise `maxNodeDelegatorLimit` without any contract-level ceiling. The comment `ndc count * asset count = 15` shows the team expects growth but relies on off-chain discipline rather than on-chain enforcement. As the protocol scales to 20–30 NDCs and 8–10 assets (plausible for a maturing restaking protocol), the gas cost crosses the block limit without any malicious action required.

---

### Recommendation

1. **Enforce hard caps in contract code** for both `maxNodeDDelegatorLimit` (e.g., `require(maxNodeDelegatorLimit_ <= 15)`) and the number of supported assets.
2. **Cache per-NDC data** off-chain and use a push-based oracle pattern (operator submits pre-computed TVL with a merkle proof or signed attestation) rather than computing it on-chain in a single transaction.
3. **Split `_getTotalEthInProtocol` into paginated calls** so gas cost per transaction is bounded, with a multi-step commit pattern for price updates.
4. **Add a gas-estimation check** in CI/tests that fails if `updateRSETHPrice` gas exceeds a threshold at max configured dimensions.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
