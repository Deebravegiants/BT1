### Title
`RequireGuard (KeyHashObj kh)` Native Script Does Not Enforce Key Signature — (`File: eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

The Dijkstra era introduces a new `RequireGuard` native script primitive that supports both `ScriptHashObj` and `KeyHashObj` credentials. For `ScriptHashObj` guards, the script must be provided and validated (enforced via `getDijkstraScriptsNeeded`). However, for `KeyHashObj` guards, there is no corresponding enforcement that the key hash signed the transaction. Dijkstra's `getWitsVKeyNeeded` is set to `getConwayWitsVKeyNeeded` without modification, which has no knowledge of the Dijkstra-specific `guards` field. As a result, `RequireGuard (KeyHashObj kh)` can be satisfied by any transaction that merely includes `kh` in the `guards` set — without possessing the private key for `kh`. This allows an attacker to spend UTxOs locked by scripts containing `RequireGuard (KeyHashObj kh)` without authorization.

---

### Finding Description

**Vulnerability class:** Incomplete authorization check — a function that validates authorization for an action checks one authorization mechanism (script witnesses) but omits the complementary mechanism (key witnesses) for a newly introduced credential type.

The Dijkstra era introduces the `guards` field (`guardsTxBodyL`) in the transaction body, an `OSet (Credential Guard)`. The new `DijkstraRequireGuard` native script constructor evaluates to `True` if a credential is present in this set:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Scripts.hs, line 575
RequireGuard cred -> cred `OSet.member` guards
```

For `ScriptHashObj` guards, `getDijkstraScriptsNeeded` correctly adds them to `scriptsNeeded`, triggering script witness validation:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs, lines 173-176
guardingScriptsNeeded = AlonzoScriptsNeeded $
  catMaybes $
    zipAsIxItem (txb ^. guardsTxBodyL) $
      \(AsIxItem idx cred) -> (\sh -> (GuardingPurpose (AsIxItem idx sh), sh)) <$> credScriptHash cred
```

`credScriptHash cred` returns `Nothing` for `KeyHashObj` credentials, so key hash guards are silently dropped from `scriptsNeeded`. This is correct — key hashes are not scripts. However, the complementary check — adding key hash guards to `witsVKeyNeeded` — is entirely absent.

The root cause is at line 139 of the same file:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs, line 139
getWitsVKeyNeeded _ = getConwayWitsVKeyNeeded
```

Dijkstra reuses Conway's `getWitsVKeyNeeded` without modification. Conway's implementation (`getConwayWitsVKeyNeeded`) has no knowledge of the Dijkstra-specific `guards` field and therefore never adds key hash guards to the required witness set. No other code path adds them either.

The CDDL specification comment confirms the intended security model:

```
; Dijkstra adds guard scripts for enhanced security.
; A guard script requires a credential to authorize execution.
script_require_guard = (6, credential)
```

"Requires a credential to authorize execution" — for a `KeyHashObj` credential, authorization means signing. The implementation does not enforce this.

The `validateGuardDatums` function in `SubUtxow.hs` (lines 210–235) further confirms the design intent: it explicitly distinguishes key hash guards (datum must be `SNothing`) from Plutus script guards (datum must be `SJust`), treating them as a distinct authorization path. Yet no signature check is wired up for that path.

---

### Impact Explanation

An unprivileged attacker can spend any UTxO locked by a native script containing `RequireGuard (KeyHashObj kh)` without possessing the private key for `kh`. The attacker constructs a transaction that:

1. Spends the target UTxO.
2. Includes `KeyHashObj kh` in the `guards` set of the transaction body.
3. Provides no signature from `kh`.

The ledger accepts this transaction because:
- `getDijkstraScriptsNeeded` does not add `kh` to `scriptsNeeded` (correct — it is not a script).
- `getWitsVKeyNeeded` does not add `kh` to required witnesses (incorrect — the key hash guard is never enforced).
- `evalDijkstraNativeScript` evaluates `RequireGuard (KeyHashObj kh)` as `True` because `kh` is in the guards set.

This constitutes **direct loss of ADA or native assets through an invalid ledger state transition** — a Critical impact. Any UTxO whose locking script relies on `RequireGuard (KeyHashObj kh)` for authorization is fully unprotected.

---

### Likelihood Explanation

The vulnerability is reachable by any unprivileged transaction sender. The attacker needs only:
- Knowledge of the key hash `kh` used in the guard (publicly visible on-chain in the script hash).
- The ability to construct and submit a standard Dijkstra transaction.

No privileged access, leaked keys, or governance majority is required. The attack is deterministic and requires no brute force. The existing test in `UtxowSpec.hs` (lines 30–38) demonstrates the behavior directly: a UTxO locked by `RequireGuard (KeyHashObj kh)` is successfully spent by including `kh` in the guards set, with no signature from `kh` required.

---

### Recommendation

Override `getWitsVKeyNeeded` for Dijkstra to include key hash guards in the required witnesses. Specifically, for each `KeyHashObj kh` in `txBody ^. guardsTxBodyL`, add `asWitness kh` to the set returned by `getWitsVKeyNeeded`. This mirrors the pattern used for other key-hash-based authorization requirements (e.g., withdrawal authors, certificate authors) throughout the Shelley/Conway witness rules.

---

### Proof of Concept

The existing test at `eras/dijkstra/impl/testlib/Test/Cardano/Ledger/Dijkstra/Imp/UtxowSpec.hs` lines 30–38 demonstrates the behavior:

```haskell
it "Spending inputs locked by script requiring a keyhash guard" $ do
  guardKeyHash <- KeyHashObj <$> freshKeyHash
  scriptHash <- impAddNativeScript (RequireGuard guardKeyHash)
  txIn <- produceScript scriptHash
  let tx = mkBasicTx (mkBasicTxBody & inputsTxBodyL .~ [txIn])
  submitFailingTx
    tx
    [injectFailure $ Conway.ScriptWitnessNotValidatingUTXOW $ NES.singleton scriptHash]
  submitTx_ $ tx & bodyTxL . guardsTxBodyL .~ [guardKeyHash]
```

The test creates a `freshKeyHash` credential, locks a UTxO with `RequireGuard (KeyHashObj kh)`, and successfully spends it by placing `kh` in the guards set — with no signature from `kh`. The ImpTest framework's `submitTx_` auto-fixup would add a signature if `getWitsVKeyNeeded` required one; the fact that the transaction succeeds without any such fixup confirms the missing enforcement.

To confirm the vulnerability without auto-fixup, a test using `withNoFixup` that omits the signature from `kh` while including `kh` in the guards set should succeed (demonstrating the bug) rather than fail with `MissingVKeyWitnessesUTXOW`.

```haskell
evalDijkstraNativeScript keyHashes (ValidityInterval txStart txExp) guards = go
  where
    -- the evaluation will stop as soon as it reaches the required number of valid scripts
    isValidMOf n SSeq.Empty = n <= 0
    isValidMOf n (ts SSeq.:<| tss) =
      n <= 0 || if go ts then isValidMOf (n - 1) tss else isValidMOf n tss
    go = \case
      RequireTimeStart lockStart -> lockStart `lteNegInfty` txStart
      RequireTimeExpire lockExp -> txExp `ltePosInfty` lockExp
      RequireSignature hash -> hash `Set.member` keyHashes
      RequireAllOf xs -> all go xs
      RequireAnyOf xs -> any go xs
      RequireMOf m xs -> isValidMOf m xs
      RequireGuard cred -> cred `OSet.member` guards
      _ -> error "Impossible: All NativeScripts should have been accounted for"
```

```haskell
validateGuardDatums (ScriptsProvided scripts) txBody =
  failureOnNonEmptySet malformed SubMalformedGuardDatums
  where
    malformed =
      Map.foldlWithKey' accum mempty (txBody ^. requiredTopLevelGuardsL)
    accum acc cred mbDatum =
      case credScriptHash cred of
        Nothing ->
          -- Key hash: datum must be SNothing
          case mbDatum of
            SNothing -> acc
            SJust _ -> Set.insert cred acc
        Just scriptHash ->
          case Map.lookup scriptHash scripts of
            Just script
              | isNativeScript script ->
                  -- Native script: datum must be SNothing
                  case mbDatum of
                    SNothing -> acc
                    SJust _ -> Set.insert cred acc
              | otherwise ->
                  -- Plutus script: datum must be SJust
                  case mbDatum of
                    SJust _ -> acc
                    SNothing -> Set.insert cred acc
            Nothing -> acc
```
