### Title
Missing Chainlink `latestRoundData()` Return Value Validation Allows Invalid Price to Corrupt `rsETHPrice` — (`contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice` discards every return value from `latestRoundData()` except `answer`. No check is made on `updatedAt`, `answeredInRound`, or the sign of `answer`. A feed that returns `updatedAt = 0` (uninitialized / broken round) with an arbitrarily small `answer` is silently accepted, the resulting price flows into `LRTOracle._updateRsETHPrice()`, and the corrupted `rsETHPrice` is stored on-chain and used for all subsequent minting.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice` (line 52) destructures `latestRoundData()` keeping only `price`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol  line 52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

The four ignored fields are `roundId`, `startedAt`, `updatedAt`, and `answeredInRound`. None of the following standard Chainlink guards are present:

| Guard | Missing |
|---|---|
| `updatedAt > 0` (round completed) | ✓ |
| `updatedAt >= block.timestamp - staleness` | ✓ |
| `answeredInRound >= roundId` | ✓ |
| `price > 0` | ✓ |

The corrupted price propagates through the full pricing stack:

1. `LRTOracle._getTotalEthInProtocol()` calls `getAssetPrice(asset)` for every supported asset and multiplies by total deposits. [2](#0-1) 

2. `_updateRsETHPrice()` divides the resulting ETH total by `rsethSupply` to produce `newRsETHPrice`, then writes it to `rsETHPrice`. [3](#0-2) 

3. `LRTDepositPool.getRsETHAmountToMint()` uses the stored `rsETHPrice` for every deposit. [4](#0-3) 

4. `updateRSETHPrice()` is `public whenNotPaused` — any EOA can trigger a price update. [5](#0-4) 

The only partial guard is `pricePercentageLimit`, but its default value is `0` (Solidity zero-initialisation; `setPricePercentageLimit` must be called explicitly by admin). When it is `0`, the condition `pricePercentageLimit > 0 && ...` is always false, so the downside-pause and upside-revert branches are both skipped entirely. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

If a supported asset's Chainlink feed returns `updatedAt = 0` with `answer = 1` (1 wei):

- `getAssetPrice` returns `1 * 1e18 / 1e8 = 1e10` (for an 8-decimal feed) instead of ~`1e18`.
- `_getTotalEthInProtocol()` collapses to near-zero.
- `newRsETHPrice` collapses to near-zero.
- Any depositor calling `depositAsset` / `depositETH` immediately after `updateRSETHPrice()` receives `amount * getAssetPrice(asset) / rsETHPrice` rsETH — an astronomically inflated amount.
- The minted rsETH represents claims on real underlying assets, draining the protocol.

This is **protocol insolvency**: depositors mint rsETH far in excess of their contributed value, and existing holders' rsETH is diluted to near-worthlessness.

---

### Likelihood Explanation

- Chainlink feeds can return `updatedAt = 0` for a newly deployed / uninitialized aggregator, or during certain sequencer / L2 outage scenarios.
- `updateRSETHPrice()` is permissionless — any address can call it at any time.
- `pricePercentageLimit` defaults to `0`, so no automatic circuit-breaker fires.
- No off-chain keeper is required; the attacker calls `updateRSETHPrice()` themselves immediately after the feed enters the bad state, then deposits.

---

### Recommendation

Add the following checks inside `getAssetPrice`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

require(price > 0,                        "Chainlink: non-positive price");
require(updatedAt > 0,                    "Chainlink: round not complete");
require(answeredInRound >= roundId,       "Chainlink: stale round");
require(block.timestamp - updatedAt <= STALENESS_THRESHOLD, "Chainlink: stale price");
```

Additionally, ensure `pricePercentageLimit` is set to a non-zero value during deployment/initialisation so the circuit-breaker in `_updateRsETHPrice` is always active.
