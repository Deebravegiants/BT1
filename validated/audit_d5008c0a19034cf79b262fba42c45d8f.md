Audit Report

## Title
Router's Stranded Native ETH Consumed to Settle Attacker's Zero-Cost WETH-Input Swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`PeripheryPayments.pay()` settles WETH-input swaps by wrapping the router's entire native ETH balance without verifying that the ETH originated from the current caller's `msg.value`. If a prior user sent excess ETH and omitted `refundETH`, an attacker can call `exactInputSingle{value: 0}` with `tokenIn = WETH` and have the router wrap the victim's stranded ETH to pay the pool, receiving real output tokens at zero cost.

## Finding Description
In `pay()`, when `token == WETH` and `payer != address(this)`, the function reads `address(this).balance` (line 74) and, if that balance covers the required `value`, wraps and forwards the router's own ETH without ever pulling from `payer` (lines 75–77). There is no check that this ETH was deposited by the current caller.

The entry point `exactInputSingle` is `payable` but enforces no minimum `msg.value`. An attacker calls it with `value: 0` and `tokenIn = WETH`. The pool executes the swap and calls `metricOmmSwapCallback` → `_justPayCallback` → `pay(WETH, attacker, pool, amountIn)`. Because `address(this).balance >= amountIn` (from the victim's stranded ETH), the first branch is taken: the router wraps its own ETH and transfers WETH to the pool. The pool's `IncorrectDelta` guard only verifies that the pool received the correct token amount; it does not care about the source of funds. The attacker receives output tokens; the victim's ETH is permanently consumed.

The `receive()` guard (`if (msg.sender != WETH) revert NotWETH()`) prevents arbitrary ETH pushes but does not prevent ETH accumulation from `msg.value` in payable calls, so the router legitimately accumulates ETH whenever users over-send and skip `refundETH`.

## Impact Explanation
Direct, permanent loss of a prior user's ETH principal. The attacker receives real pool output tokens (token0 or token1) without transferring any asset. Loss magnitude equals `min(router.balance, attacker's amountIn)` and is bounded only by how much ETH the victim left in the router. This is a direct theft of user funds meeting Critical/High Sherlock thresholds.

## Likelihood Explanation
The multicall + `refundETH` pattern is the documented safe path, but `exactInputSingle` is independently callable and payable. Users routinely over-send ETH to guarantee swap success. Omitting `refundETH` is a realistic user error. An attacker can monitor `address(router).balance` on-chain and exploit it atomically in a single transaction with zero upfront cost. No special privileges are required.

## Recommendation
Inside `pay()`, when `token == WETH`, only consume native ETH up to the amount that belongs to the current call context. The cleanest fix is to pass `msg.value` of the originating entry point into `pay()` (or store it in transient storage at entry) and cap native ETH usage to that value. Any shortfall must be pulled from `payer` via `safeTransferFrom`. Alternatively, enforce `address(this).balance == 0` at the start of every non-multicall payable entry point, or require callers to always pair WETH-input swaps with `refundETH` inside a multicall and enforce this as a hard invariant (e.g., by reverting if `msg.value > 0` outside multicall).

## Proof of Concept
```
1. Victim calls router.exactInputSingle{value: 10_000}(tokenIn=WETH, amountIn=1_000, ...)
   → swap uses 1_000 ETH worth of WETH; 9_000 ETH remains in router.
   → Victim does NOT call refundETH.

2. Attacker calls router.exactInputSingle{value: 0}(tokenIn=WETH, amountIn=9_000, ...)
   → Pool executes swap, calls metricOmmSwapCallback on router.
   → Router calls _justPayCallback → pay(WETH, attacker, pool, 9_000).
   → nativeBalance = address(this).balance = 9_000 >= value = 9_000 → first branch taken.
   → Router wraps its own 9_000 ETH and safeTransfers WETH to pool.
   → Pool's IncorrectDelta check passes (pool received correct WETH amount).
   → Attacker receives pool output tokens.

Result: Attacker paid 0 ETH; victim lost 9_000 ETH.
```
Foundry test: deploy router with WETH, fund victim's swap leaving surplus ETH in router, then call `exactInputSingle{value: 0}` as attacker with `amountIn = router.balance` and assert attacker received output tokens and `router.balance == 0`.
