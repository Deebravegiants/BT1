### Title
ERC1155 Callback Reentrancy in `initTransfer1155` Allows Unbacked `InitTransfer` Events — (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

### Summary

`initTransfer1155` makes an external call to an attacker-controlled ERC1155 token contract via `safeTransferFrom` before completing its state finalization (event emission). A malicious ERC1155 token can reenter `initTransfer1155` during that call, causing multiple `InitTransfer` events to be emitted with distinct nonces while zero tokens are actually locked in the bridge. The NEAR side treats every emitted `InitTransfer` event as proof of locked collateral and will mint/release the corresponding wrapped tokens.

### Finding Description

`initTransfer1155` accepts any arbitrary ERC1155 token address supplied by the caller. The function increments `currentOriginNonce` and then makes an external call to the attacker-supplied token contract: [1](#0-0) 

The sequence is:
1. `currentOriginNonce += 1` (nonce = N)
2. `IERC1155(tokenAddress).safeTransferFrom(msg.sender, address(this), tokenId, amount, "")` — **external call to attacker-controlled contract**
3. `initTransferExtension(...)` — Wormhole message published
4. `emit InitTransfer(...)` with nonce N

There is no reentrancy guard on `initTransfer1155`. During step 2, the malicious token's `safeTransferFrom` can call back into `initTransfer1155`:

- Reentrant call: nonce becomes N+1, malicious token's inner `safeTransferFrom` returns without reverting (and without transferring tokens), `emit InitTransfer(...)` with nonce N+1 fires.
- Outer call resumes: `emit InitTransfer(...)` with nonce N fires.

Two events are emitted; zero tokens are locked.

The `onERC1155Received` hook on the bridge does check `operator != address(this)`: [2](#0-1) 

This check is bypassed because during `initTransfer1155` the bridge itself is the caller of `safeTransferFrom`, making `operator == address(this)` true. The check was designed to block direct ERC1155 sends, not to guard against reentrancy from within `initTransfer1155`. The malicious token can also simply skip calling `onERC1155Received` entirely and just reenter then return — the bridge does not verify the callback was invoked.

The CLAUDE.md security invariant "Event–transfer atomicity: `InitTransfer` must only be emitted in a code path where tokens have already been burned/locked" is violated: [3](#0-2) 

### Impact Explanation

The NEAR side reads `InitTransfer` events as the sole proof that tokens are held on EVM. Each event carries a unique `originNonce`. For every unbacked event the attacker generates, NEAR will mint or release the corresponding wrapped token amount to the attacker's specified recipient. This is unauthorized creation of wrapped bridge assets with no collateral — a direct bridge drain.

### Likelihood Explanation

The attack requires only:
1. Deploying a malicious ERC1155 contract (permissionless, no privileged access).
2. Calling `initTransfer1155` with that contract as `tokenAddress` (public entrypoint, no whitelist).

No admin keys, MPC threshold signers, or external dependency compromise are needed. Any unprivileged user can execute this.

### Recommendation

1. **Add a reentrancy guard** (`nonReentrant` modifier from OpenZeppelin) to `initTransfer1155` and `initTransfer`.
2. **Follow strict CEI**: emit the `InitTransfer` event and call `initTransferExtension` *before* the external `safeTransferFrom` call, or verify the actual token balance delta after the call (check `balanceOf(address(this))` before and after).
3. **Consider a token allowlist** for ERC1155 tokens accepted by the bridge, analogous to how `isBridgeToken` gates ERC20 paths.

### Proof of Concept

```solidity
contract MaliciousERC1155 {
    OmniBridge bridge;
    bool reentered;

    function safeTransferFrom(
        address from, address to, uint256 id, uint256 amount, bytes calldata
    ) external {
        if (!reentered) {
            reentered = true;
            // Reenter: emits InitTransfer with nonce N+1, no tokens locked
            bridge.initTransfer1155(address(this), id, uint128(amount), 0, 0, "attacker.near", "");
        }
        // Return without reverting, without transferring any tokens
        // Bridge proceeds to emit InitTransfer with nonce N — also no tokens locked
    }

    function balanceOf(address, uint256) external pure returns (uint256) { return 1e18; }
    // ... minimal ERC1155 stubs
}
```

Attacker calls `bridge.initTransfer1155(malicious, tokenId, amount, 0, 0, "attacker.near", "")`. Result: two `InitTransfer` events emitted (nonces N and N+1), zero ERC1155 tokens held by the bridge. NEAR mints `2 × amount` wrapped tokens to the attacker.
