### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Inject Arbitrary `sequencer_address` into Block Execution and Hash - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit.builder` is a proposer-supplied wire field that is mapped directly to `BlockInfo.sequencer_address` and used in block hash computation, transaction execution context, and state-sync block headers. The validator's `is_proposal_init_valid` function checks every other security-sensitive field in `ProposalInit` (height, l1_da_mode, l2_gas_price_fri, all four L1 gas prices, starknet_version, version_constant_commitment, fee_proposal_fri, timestamp) but never checks `builder`. `ProposalInitValidation` — the struct that carries locally-trusted reference values — has no `builder` field at all. A malicious designated proposer can therefore set `builder` to any arbitrary `ContractAddress`, and every validator node will accept the proposal and commit a block whose `sequencer_address` is attacker-controlled.

---

### Finding Description

`ProposalInit` carries a `builder` field:

> "Address of the one who builds/sequences the block." [1](#0-0) 

`convert_to_sn_api_block_info` maps it directly to `BlockInfo.sequencer_address` with no validation: [2](#0-1) 

That `BlockInfo` is passed to the batcher in `initiate_validation` → `ValidateBlockInput.block_info`: [3](#0-2) 

The batcher uses `BlockInfo.sequencer_address` in two ways:

1. **Block hash**: `PartialBlockHashComponents::new` stores it as `sequencer: SequencerContractAddress(block_info.sequencer_address)`, which is then chained into the Poseidon block hash: [4](#0-3) [5](#0-4) 

2. **Transaction execution context**: `BlockContext` is constructed from `BlockInfo`, and every transaction's `get_block_info()` syscall returns `sequencer_address` from that context: [6](#0-5) 

After consensus, `update_state_sync_with_new_block` stores `init.builder` as `SequencerContractAddress` in the committed block header: [7](#0-6) 

Now look at `is_proposal_init_valid`. It validates timestamp, starknet_version, version_constant_commitment, height, l1_da_mode, l2_gas_price_fri, all four L1 gas prices, and fee_proposal_fri. `builder` is never mentioned: [8](#0-7) 

`ProposalInitValidation` — the struct of locally-trusted reference values — has no `builder` field: [9](#0-8) 

When `validate_proposal` is called, the `ProposalInitValidation` is constructed from the node's own state, but `builder` is simply absent: [10](#0-9) 

---

### Impact Explanation

A malicious designated proposer sets `init.builder = <attacker_address>` in the `ProposalInit` it broadcasts. Every validator node calls `is_proposal_init_valid`, which passes (no `builder` check), then calls `initiate_validation` → `convert_to_sn_api_block_info`, which maps `init.builder` to `BlockInfo.sequencer_address`. The batcher executes all transactions under a `BlockContext` whose `sequencer_address` is attacker-controlled.

Concrete corrupted values:

- **Wrong `get_block_info().sequencer_address` syscall result** for every transaction in the block. Any contract that branches on `sequencer_address` (e.g., fee-payment logic, access-control guards, or sequencer-tip routing) will execute incorrectly.
- **Wrong block hash**: `sequencer_address` is a direct input to the Poseidon hash chain in `calculate_block_hash`. The committed block hash encodes the attacker-chosen address.
- **Wrong block header in state sync**: `BlockHeaderWithoutHash.sequencer` is set from `init.builder`, so the stored header is permanently wrong.
- **Fee redirection**: In Starknet, transaction fees are paid to the sequencer address. With an attacker-controlled `builder`, all fees in the block flow to the attacker's address.

---

### Likelihood Explanation

The attacker must be the designated proposer for at least one consensus round. In a permissioned validator set this is a normal rotation — every honest validator is a proposer periodically, and a single compromised or malicious validator can exploit this on its assigned rounds. No special privilege beyond being a validator is required. The attack requires only setting one field in the `ProposalInit` message before broadcasting; no other node detects or rejects it.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and check it in `is_proposal_init_valid`. The locally-trusted reference value is the node's own configured `builder_address` (already available in `SequencerConsensusContext`). The check should be an exact equality:

```rust
// In ProposalInitValidation:
pub builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder
        ),
    ));
}
```

Populate `proposal_init_validation.builder` from `self.builder_address` (the same value used in `initiate_build`) when constructing `ProposalInitValidation` inside `validate_proposal` in `sequencer_consensus_context.rs`.

---

### Proof of Concept

1. A malicious validator is the designated proposer for round R at height H.
2. It constructs a `ProposalInit` with `builder = ContractAddress(0xdeadbeef)` (attacker-controlled address) and all other fields valid.
3. It broadcasts this `ProposalInit` followed by a valid transaction batch and `ProposalFin`.
4. Every validator node calls `validate_proposal` → `is_proposal_init_valid`. The function checks height, l1_da_mode, l2_gas_price_fri, L1 gas prices, starknet_version, version_constant_commitment, fee_proposal_fri — all pass. `builder` is never checked.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which sets `sequencer_address = ContractAddress(0xdeadbeef)`.
6. The batcher executes all transactions with `BlockContext.sequencer_address = 0xdeadbeef`. Any contract calling `get_block_info()` receives `sequencer_address = 0xdeadbeef`.
7. `PartialBlockHashComponents::new` stores `sequencer = SequencerContractAddress(0xdeadbeef)`, which is hashed into the block hash.
8. Consensus reaches decision; `update_state_sync_with_new_block` stores `sequencer = SequencerContractAddress(0xdeadbeef)` in the committed block header.
9. All transaction fees for block H are paid to `0xdeadbeef`.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
