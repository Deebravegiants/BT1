### Title
`respond_verify_foreign_tx` Accepts Caller-Supplied `payload_hash` Without Cross-Validating Against the Stored Request — (`File: crates/contract/src/lib.rs`)

---

### Summary

`respond_verify_foreign_tx` verifies that `response.signature` is a valid ECDSA signature over `response.payload_hash`, but never checks that `response.payload_hash` was actually derived from the stored `VerifyForeignTransactionRequest`. A single Byzantine attested MPC participant can substitute a legitimately-obtained signature from a *different* `verify_foreign_transaction` request to resolve any pending request, delivering a forged `payload_hash` to the caller and permanently draining the pending-request slot.

---

### Finding Description

**Creation phase** — `verify_foreign_transaction()` stores the full `VerifyForeignTransactionRequest` (containing `tx_id`, `extractors`, `domain_id`, `payload_version`) in `pending_verify_foreign_tx_requests`, keyed by the request struct itself.

**Confirmation phase** — `respond_verify_foreign_tx()` receives a caller-supplied `(request, response)` pair. The contract performs two checks:

1. `request` must be a key in `pending_verify_foreign_tx_requests` (map lookup).
2. `response.signature` must be a valid ECDSA signature over `response.payload_hash` under the ForeignTx domain root key. [1](#0-0) 

What is **never checked**: that `response.payload_hash` equals `SHA-256(borsh(ForeignTxSignPayload { request: stored_request, values: … }))`. The contract has the stored `request` available as the map key, but it never recomputes or bounds-checks the hash against it.

The `payload_hash` is supposed to commit to both the original request and the extracted on-chain values:

```
payload_hash = SHA-256(borsh(ForeignTxSignPayload::V1 { request, values }))
``` [2](#0-1) 

Because `values` are not included in the response DTO and the contract cannot recompute the hash without them, the contract passes `response.payload_hash` through to the caller verbatim: [3](#0-2) 

The SDK-side verifier (`ForeignChainSignatureVerifier::verify_signature`) does perform this cross-check, but it is an off-chain helper for callers — it is not enforced by the contract: [4](#0-3) 

---

### Impact Explanation

A single Byzantine attested MPC participant can execute the following attack:

1. Submit a legitimate `verify_foreign_transaction(tx_A)` for a real, finalized foreign transaction.
2. Observe the honest network's response `{payload_hash_A, sig_A}` on-chain (it is public).
3. Wait for a victim to submit `verify_foreign_transaction(tx_B)` for a *different* transaction (e.g., a non-existent deposit, an unfinalized transfer, or a transaction on a different chain).
4. Immediately call `respond_verify_foreign_tx(request_B, {payload_hash_A, sig_A})`.
5. The contract accepts: `sig_A` is a valid signature over `payload_hash_A` under the ForeignTx domain key ✓; `request_B` is in the pending map ✓.
6. The pending slot for `request_B` is permanently drained; honest nodes can no longer respond.
7. The victim's bridge contract receives `{payload_hash_A, sig_A}` as the "verified" result for `tx_B`.

Any bridge contract that does not independently recompute the expected `payload_hash` from its own known `(request_B, expected_values)` and compare it to `response.payload_hash` will accept this as proof that `tx_B` was verified. This enables minting tokens for a non-existent deposit, double-spending, or other invalid bridge execution — a direct financial loss.

The attack requires only **one** Byzantine attested participant, well below the signing threshold.

---

### Likelihood Explanation

- **Entry path is open to any single attested participant.** `respond_verify_foreign_tx` requires only `assert_caller_is_attested_participant_and_protocol_active()` — no threshold agreement.
- **Pre-obtaining a valid signature is trivial.** The Byzantine node submits any real foreign transaction, waits for the honest network to sign it, and reads the response from the NEAR blockchain.
- **Racing honest nodes is easy.** The Byzantine node skips the foreign-chain RPC step entirely and can respond in the same block the request appears.
- **Callers may not validate `payload_hash`.** The SDK helper exists but is not mandatory; bridge contracts that trust the contract's signature check alone are fully exploitable.

Likelihood: **Medium** (requires one Byzantine TEE participant; the signature recycling step is straightforward).

---

### Recommendation

Include `extracted_values` in the response submitted to the contract, and have `respond_verify_foreign_tx` recompute and assert the expected hash before accepting the response:

```rust
// In respond_verify_foreign_tx, after verifying the signature:
let expected_payload = ForeignTxSignPayload::V1(ForeignTxSignPayloadV1 {
    request: request.request.clone(),   // from the stored/caller-supplied request
    values: response.extracted_values,  // newly added field in the response DTO
});
let expected_hash = expected_payload.compute_msg_hash()?;
if expected_hash != response.payload_hash {
    return Err(RespondError::PayloadHashMismatch.into());
}
```

This mirrors the fix applied in the referenced report: cross-validate the confirmation-time data against the creation-time stored request before committing the state change.

verify_foreign_transaction({ tx_id: TX_A, extractors: [BlockHash], domain_id: FTX_DOMAIN })

// Step 3 – Byzantine node B immediately calls respond with TX_A's response.

    #[handle_result]
    pub fn respond_verify_foreign_tx(
        &mut self,
        request: VerifyForeignTransactionRequest,
        response: VerifyForeignTransactionResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!(
            "respond_verify_foreign_tx: signer={}, request={:?}",
            &signer,
            &request
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain.0.into())?;

        let signature_is_valid = match (&response.signature, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");

                let payload_hash: [u8; 32] = response.payload_hash.0;

                // Check the signature is correct against the root public key
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    &payload_hash,
                    &secp_pk,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_verify_foreign_tx_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )

pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
