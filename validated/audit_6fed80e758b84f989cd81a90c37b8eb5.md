### Title
Unvalidated `fee_proposal_fri` During Bootstrap Allows Arbitrary Fee-Market Seeding — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` deliberately skips all bounds enforcement on `fee_proposal_fri` when `fee_actual` is `None` (the bootstrap window of the first `window_size` blocks). An honest proposer falls back to `l2_gas_price` during this period, but the validator never checks that the proposed value matches that fallback. A malicious proposer can therefore publish any `fee_proposal_fri` — including `u128::MAX` or `0` — and every validator will accept it, record it in `fee_proposals_window`, and use it to compute `fee_actual` for all subsequent blocks.

---

### Finding Description

**Root cause — the unguarded bootstrap branch:**

In `is_proposal_init_valid`, the fee-proposal bounds check is wrapped in an `if let` that silently does nothing when `fee_actual` is `None`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(...);
    }
}
``` [1](#0-0) 

The only other check on `fee_proposal_fri` is that it is `Some` (present) for `starknet_version >= V0_14_3`: [2](#0-1) 

No check verifies that the value is close to `l2_gas_price`, which is the honest proposer's bootstrap fallback:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [3](#0-2) 

**How `fee_proposal_fri` propagates into the commitment and the fee-market state:**

1. `proposal_commitment_from` hashes `fee_proposal_fri` into the `ProposalCommitment` that consensus votes on:

```rust
ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
``` [4](#0-3) 

2. `finalize_decision` records the accepted `fee_proposal_fri` into `fee_proposals_window`:

```rust
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [5](#0-4) 

3. `compute_fee_actual` computes the median of `fee_proposals_window` to produce `fee_actual` for future blocks: [6](#0-5) 

4. `fee_actual` is then passed to `calculate_next_l2_gas_price_for_fin` (proposer side) and bounds future `fee_proposal_fri` values via `fee_proposal_bounds`: [7](#0-6) 

**The analog to the external bug:**

In the external report, platform fees accrue on minters (peripheral) rather than the core `SoundEditionV1_2` contract, so an edition owner can substitute a fee-free minter. Here, the fee-market invariant (`fee_proposal_fri ≈ l2_gas_price` during bootstrap) is enforced only at the consensus-orchestrator layer and only when `fee_actual` is `Some`. During bootstrap the core invariant is entirely absent: the blockifier never sees `fee_proposal_fri`, and the orchestrator's guard is a no-op. A malicious proposer substitutes an extreme value exactly as an edition owner substitutes a fee-free minter.

---

### Impact Explanation

A malicious proposer that controls even a simple majority of the `window_size` bootstrap blocks can seed `fee_proposals_window` with `u128::MAX` (or `0`). After the window fills, `compute_fee_actual` returns an extreme median. Every subsequent honest proposer's `fee_proposal_fri` is then bounded by `fee_proposal_bounds(extreme_fee_actual, margin_ppt)`, which is itself extreme. Because `fee_actual` feeds `calculate_next_l2_gas_price_for_fin`, the L2 gas price for all future blocks is driven to an extreme value that all validators agree on (they all recorded the same poisoned window). Users pay extreme fees or transactions become economically unexecutable. The `proposal_commitment` for every bootstrap block also encodes the attacker-chosen value, corrupting the `parent_proposal_commitment` chain forwarded to the centralized cende pipeline.

This matches: **Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

---

### Likelihood Explanation

`window_size` is read from `VersionedConstants::latest_constants().fee_proposal_window_size`. During the initial deployment of a new chain (or after a hard reset), the bootstrap window is the first `window_size` blocks. Any validator that is selected as proposer during this window — a normal, non-privileged role in BFT consensus — can exploit the gap. In a small validator set the probability of being selected for a majority of bootstrap blocks is non-negligible.

---

### Recommendation

During bootstrap (`fee_actual is None`), validate that `fee_proposal_fri` is within a configured margin of `l2_gas_price` (the honest proposer's documented fallback). Concretely, reuse `fee_proposal_bounds` with `l2_gas_price` as the reference:

```rust
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    let reference = match proposal_init_validation.fee_actual {
        Some(fa) => fa,
        None => proposal_init_validation.l2_gas_price

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
