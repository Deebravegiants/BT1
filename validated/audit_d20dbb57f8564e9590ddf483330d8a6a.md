Audit Report

## Title
Excess ETH Sent to `updateRate()` Is Permanently Locked — No Withdrawal Mechanism Exists - (File: contracts/cross-chain/MultiChainRateProvider.sol)

## Summary
`MultiChainRateProvider.updateRate()` is `payable` and forwards exactly the on-chain `estimatedFee` to the LayerZero endpoint per receiver. Any `msg.value` exceeding the sum of those fees accumulates in the contract with no `receive()`, `fallback()`, or withdrawal function to recover it. ETH over-provisioned by callers is permanently frozen.

## Finding Description
In `updateRate()` (L108–137), the loop queries `estimateFees()` on-chain and sends exactly that amount to the LayerZero endpoint:

```solidity
(uint256 estimatedFee,) = ILayerZeroEndpoint(layerZeroEndpoint)
    .estimateFees(dstChainId, address(this), _payload, false, bytes(""));

ILayerZeroEndpoint(layerZeroEndpoint).send{ value: estimatedFee }(
    dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
);
```

The endpoint receives exactly `estimatedFee` — not a slice of `msg.value` — so it has nothing to refund to `msg.sender`. Any `msg.value` above `∑estimatedFee` remains in `MultiChainRateProvider`. The contract inherits only `Ownable` and `ReentrancyGuard`, neither of which defines a `receive()` or `fallback()`. No `rescueETH`, `withdraw`, or sweep function exists anywhere in the contract. The stranded ETH is irrecoverable.

## Impact Explanation
This constitutes **permanent freezing of funds** (Critical). Callers who over-provision ETH — a standard practice to guard against fee fluctuations between off-chain estimation and on-chain execution — permanently lose the excess. The ETH is locked in the contract with no recovery path for any party, including the owner.

## Likelihood Explanation
`updateRate()` has no access control; any address may call it. The `estimateTotalFee()` helper exists but fee estimates are inherently approximate and can shift between the view call and the actual transaction. Callers are explicitly directed to estimate fees off-chain (L105–107 NatDoc), making over-provisioning the expected behavior. With multiple `rateReceivers`, cumulative estimation error grows, increasing the stranded amount per call. The condition is triggered on every call where `msg.value > ∑estimatedFee`.

## Recommendation
Refund unspent ETH to `msg.sender` at the end of `updateRate()`:

```solidity
uint256 remaining = address(this).balance;
if (remaining > 0) {
    (bool ok,) = payable(msg.sender).call{value: remaining}("");
    require(ok, "refund failed");
}
```

Alternatively, add an `onlyOwner` `rescueETH()` function, or enforce `msg.value == estimateTotalFee()` and revert otherwise.

## Proof of Concept
1. Deploy a concrete implementation of `MultiChainRateProvider` with two `rateReceivers` on a local fork.
2. Call `estimateTotalFee()` → returns `1 ether`.
3. Call `updateRate{ value: 1.01 ether }()`.
4. The loop sends exactly `estimatedFee` (~0.5 ether each) to the LayerZero endpoint for each receiver; the endpoint refunds nothing because it received exactly what it was owed.
5. Assert `address(multiChainRateProvider).balance == 0.01 ether` after the call.
6. Attempt any withdrawal — no function exists; confirm the ETH is permanently stuck. [1](#0-0) [2](#0-1)

```text
    function updateRate() external payable nonReentrant {
        uint256 latestRate = getLatestRate();

        rate = latestRate;

        lastUpdated = block.timestamp;

        bytes memory _payload = abi.encode(latestRate);

        uint256 rateReceiversLength = rateReceivers.length;

        for (uint256 i; i < rateReceiversLength;) {
            uint16 dstChainId = uint16(rateReceivers[i]._chainId);

            bytes memory remoteAndLocalAddresses = abi.encodePacked(rateReceivers[i]._contract, address(this));

            (uint256 estimatedFee,) = ILayerZeroEndpoint(layerZeroEndpoint)
                .estimateFees(dstChainId, address(this), _payload, false, bytes(""));

            ILayerZeroEndpoint(layerZeroEndpoint).send{ value: estimatedFee }(
                dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
            );

            unchecked {
                ++i;
            }
        }

        emit RateUpdated(rate);
    }
```
