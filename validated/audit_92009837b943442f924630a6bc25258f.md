**Forgery feasibility with a torsion-point public key:**

Ed25519 verification (RFC 8032 cofactor form) checks: `[8][S]B = [8]R + [8][k]A`. If `A` is a torsion point of order dividing 8 (e.g., the order-2 point), then `[8][k]A = O` for all `k`. The equation reduces to `[8][S]B = [8]R`. Choosing `S = 1` and `R = B` (the generator) satisfies this for **any message** `M`, since `k = H(R‖A‖M)` is irrelevant when `[8][k]A` vanishes. OpenSSL's `ossl_ed25519_verify` does not independently check subgroup membership of the supplied public key bytes.

### Title
Missing subgroup check in `ecurve_ed_t::pub_from_der` allows torsion-point public key to pass signature verification — (`src/cbmpc/crypto/base_eddsa.cpp`)

### Summary
`ecurve_ed_t::pub_from_der` accepts any on-curve Ed25519 point, including torsion points, because it calls `from_bin` without a subsequent subgroup check. A caller that then passes the resulting `ecc_pub_key_t` to `verify` will invoke `ED25519_verify` with a torsion-point public key, for which universal signature forgery is trivial.

### Finding Description
`pub_from_der` validates only the DER prefix bytes and total size, then delegates to `from_bin`, which decodes any valid curve point. The `ec25519_core::from_bin` implementation solves the curve equation `x² = (y²−1)/(dy²+1)` and returns `SUCCESS` for any point on the Ed25519 curve, including the 8 torsion points (e.g., the order-2 point `(0, −1)` with compressed encoding `0xEC FF…FF 7F`). No call to `is_in_subgroup()` or `curve.check()` follows. The `verify` method then re-serializes the stored point and passes it directly to `ED25519_verify` without any subgroup guard.

### Impact Explanation
For a torsion point `A` of order dividing 8, the cofactor verification equation `[8][S]B = [8]R + [8][k]A` reduces to `[8][S]B = [8]R` because `[8][k]A = O`. Setting `S = 1` and `R = B` satisfies this for every message `M`, since `k = H(R‖A‖M)` is irrelevant. Any caller that loads a public key via `pub_from_der` and then calls `verify` will accept this forged `(R=G, S=1)` signature for any message, constituting a complete signature forgery.

### Likelihood Explanation
The attacker needs only to supply a 44-byte DER blob (12-byte `x509_prefix` + 32-byte torsion-point encoding) to any code path that calls `pub_from_der` followed by `verify`. The torsion-point encoding is a fixed, publicly known constant. No secret material or privileged access is required.

### Recommendation
Add a subgroup check immediately after `from_bin` in `pub_from_der`:

```cpp
error_t ecurve_ed_t::pub_from_der(ecc_pub_key_t& P, mem_t der) const {
  if (der.size != ed25519::x509_prefix.size + ed25519::pub_compressed_bin_size())
    return coinbase::error(E_FORMAT);
  if (0 != memcmp(ed25519::x509_prefix.data, der.data, ed25519::x509_prefix.size))
    return coinbase::error(E_FORMAT);
  error_t rv = from_bin(P, der.skip(ed25519::x509_prefix.size));
  if (rv) return rv;
  // Reject torsion points and infinity
  if (error_t rv2 = curve_ed25519.check(P)) return rv2;
  return SUCCESS;
}
```

Alternatively, add the subgroup check inside `verify` before calling `ED25519_verify`, consistent with how other API entry points (e.g., `attach_private_scalar`) already call `curve.check()` and `is_in_subgroup()`. [8](#0-7)

static error_t from_bin(point_t& R, mem_t bin) {
  if (bin.size != 32) return coinbase::error(E_FORMAT);

  buf_t buf = bin.rev();
  uint8_t neg = buf[0] >> 7;
  buf[0] &= 0x7f;
  fe_t y = fe_t::to_fe(uint256_t::from_bin(buf));

  // x² = (y² - 1) / (dy² + 1)

  fe_t u, v, w, vxx, check;

  u = y * y;
  v = u * formula_t::get_d();
  u -= fe_t::one();       // u = y^2-1
  v += fe_t::one();       // v = dy^2+1
  w = u * v;              // w = u*v
  fe_t x = w.pow22523();  // x = w^((q-5)/8)
  x *= u;                 // x = u * w^((q-5)/8)

  vxx = x * x;
  vxx *= v;
  check = vxx - u;  // vx^2-u
  if (!check.is_zero()) {
    check = vxx + u;  // vx^2+u
    if (!check.is_zero()) {
      return coinbase::error(E_CRYPTO);
    }
    static const fe_t sqrtm1 =
        fe_t::from_bn(bn_t::from_hex("2b8324804fc1df0b2b4d00993dfbd7a72f431806ad2fe478c4ee1b274a0ea0b0"));
    x *= sqrtm1;
  }

  uint256_t x_val = x.from_fe();
  if (neg != (x_val.w0 & 1)) x = -x;

  R.x = x;
  R.y = y;
  R.z = fe_t::one();
  return 0;
}
