### Title
Publicly Known Default Passphrase Encrypts Private Key Keyring When No User Passphrase Is Set — (`File: chia/util/keyring_wrapper.py`)

### Summary
When a user does not set a master passphrase, Chia's `FileKeyring` encrypts the `keyring.yaml` file containing all private key entropy using a hardcoded, publicly known default passphrase. Any attacker who obtains the keyring file — through an unlocked workstation, backup exposure, or cloud sync — can immediately decrypt it using the well-known string and extract all private keys, enabling full coin control without any brute-force effort.

### Finding Description

Chia stores private key entropy in a flat YAML file (`~/.chia_keys/keyring.yaml`) encrypted with ChaCha20Poly1305, where the symmetric key is derived via PBKDF2 from a master passphrase. The master passphrase is **optional** by design: [1](#0-0) 

When no user passphrase is set, `KeyringWrapper._get_initial_cached_passphrase()` falls back to a hardcoded constant: [2](#0-1) 

That constant is: [3](#0-2) 

`"$ chia passphrase set # all the cool kids are doing it!"` — a publicly known string committed to the open-source repository. The keyring backend is **always** `FileKeyring` (a flat file), never the OS native credential store for the actual keys: [4](#0-3) 

The OS credential stores (`MacKeyring`, `WinKeyring`) are only used to optionally persist the *master passphrase itself*, not the private keys. On Linux, `supports_os_passphrase_storage()` returns `False`, so no OS credential store is used at all: [5](#0-4) 

The private key entropy is stored inside the encrypted YAML blob: [6](#0-5) 

When `remove_master_passphrase` is called, the keyring is re-encrypted back to the default passphrase: [7](#0-6) 

### Impact Explanation

An attacker who obtains `keyring.yaml` (via unlocked workstation, backup, cloud sync, or any file-read path) can decrypt it immediately using the publicly known default passphrase — no brute force required. The decrypted entropy bytes directly reconstruct the BLS12-381 master private key via `AugSchemeMPL.key_gen(mnemonic_to_seed(...))`, giving the attacker full signing authority over all XCH, CATs, NFTs, and pool wallets controlled by those keys. This meets the **High** impact threshold: bypass of keychain authorization enabling unauthorized signing and coin control. [8](#0-7) 

### Likelihood Explanation

The master passphrase is explicitly optional (`"is_optional": True`). Users who never run `chia passphrase set` — a common case — have their keys protected only by the default passphrase. The default passphrase is in the public repository and is trivially known to any attacker. The keyring file itself is created with `0o600` permissions, but this does not protect against: the same-user session (unlocked workstation), backup tools that copy files before permission checks, cloud sync clients, or any other process running as the same OS user. [9](#0-8) 

### Recommendation

- **Short term:** Treat the absence of a user-set passphrase as an unencrypted state and warn users prominently at startup. Do not silently fall back to a publicly known default passphrase.
- **Long term:** Store private key entropy in the OS-native secure credential store (macOS Keychain, Windows Credential Manager, Linux Secret Service/libsecret) rather than a flat file, mirroring the pattern already used for the master passphrase itself. On Linux, require a user passphrase with no silent default.

```python
    def _get_initial_cached_passphrase(self) -> str:
        """
        Grab the saved passphrase from the OS credential store (if available), otherwise
        use the default passphrase
        """
        from chia.util.keychain import supports_os_passphrase_storage

        passphrase: str | None = None

        if supports_os_passphrase_storage():

        if passphrase is None:
            passphrase = DEFAULT_PASSPHRASE_IF_NO_MASTER_PASSPHRASE
```

```python
def symmetric_key_from_passphrase(passphrase: str, salt: bytes) -> bytes:
    return pbkdf2_hmac("sha256", passphrase.encode(), salt, HASH_ITERS)


def encrypt_data(input_data: bytes, key: bytes, nonce: bytes) -> bytes:
    encryptor = ChaCha20Poly1305(key)
    data = encryptor.encrypt(nonce, CHECKBYTES_VALUE + input_data, None)
    return data


def decrypt_data(input_data: bytes, key: bytes, nonce: bytes) -> bytes:
    decryptor = ChaCha20Poly1305(key)
    output = decryptor.decrypt(nonce, input_data, None)
    if CHECKBYTES_VALUE != output[: len(CHECKBYTES_VALUE)]:
        raise ValueError("decryption failure (checkbytes)")
    return output[len(CHECKBYTES_VALUE) :]
```
