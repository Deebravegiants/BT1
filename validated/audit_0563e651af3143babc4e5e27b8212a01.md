### Title
Missing `curve.check()` in `get_public_key_compressed` allows infinity-point public key to be returned without error — (`src/cbmpc/api/ecdsa_mp.cpp`)

### Summary
`ecdsa_mp::get_public_key_compressed` parses `Q_compressed` from an attacker-controlled blob via `Q.from_bin()`, which explicitly accepts the infinity point encoding, then serializes the result back without calling `curve.check(Q)`. The function returns `SUCCESS` with `pub_key` set to the compressed encoding of the infinity point.

### Finding Description
`get_public_key_compressed` performs only three checks on the blob: size limit, version field, and curve ID. It does not call `deserialize_key_blob` (which enforces `x_share·G == Qi_self` and `Q == ΣQi`). After `Q.from_bin()` succeeds on an all-zero 33-byte input, no validity guard is applied before `Q.to_compressed_bin()` is called and the result is returned to the caller.

The missing guard is `curve.check(Q)`, which is present in every other point-deserialization path in the codebase (`convert_fixed_curve`, `attach_private_scalar`, `ossl_ecdsa_verify`, etc.).

### Impact Explanation
The caller of `get_public_key_compressed` receives a compressed encoding of the infinity point as the public key, with no error indication. Any downstream code that trusts this return value — e.g., storing it, comparing it, or passing it to a signature verifier — operates on a degenerate key. This constitutes invalid cryptographic output accepted from a public API with no error signal.

### Likelihood Explanation
The attacker only needs to craft a syntactically valid key blob (correct version, valid curve ID, all-zero `Q_compressed` of the right size). No cryptographic knowledge or threshold participation is required. The function is a public API entry point.

### Recommendation
Add `if (rv = curve.check(Q)) return coinbase::error(rv, "invalid key blob");` immediately after the `Q.from_bin(...)` call in `get_public_key_compressed`, consistent with every other point-deserialization site in the codebase.

### Proof of Concept
1. Serialize a `key_blob_v1_t` with `version=1`, `curve=secp256k1`, `party_name="p"`, `Q_compressed` = 33 zero bytes, `Qis_compressed={"p": <33 zero bytes>}`, `x_share=0`.
2. Call `ecdsa_mp::get_public_key_compressed(blob, pub_key)`.
3. Assert the return value is `SUCCESS` and `pub_key` is 33 zero bytes (infinity encoding).
4. Without the fix, the assertion passes; with `curve.check(Q)` inserted, it returns an error.
