### Title
`rollback_to_block` does not clear `finish_exiting_height`, causing `_finish_leaving_pool` to call `exit_waiting_room` on a non-exiting PlotNFT after a reorg, permanently looping `ValueError` on every new peak — (`chia/wallet/plotnft_wallet/plotnft_store.py`, `plotnft_wallet.py`)

### Summary
After a chain reorg that rolls back the exit-to-waiting-room spend, `finish_exiting_height` remains set while `plotnft2s` reverts to the pre-exit row. Once the unconfirmed-transaction guard clears (transaction dropped from mempool), every `new_peak` call attempts `_finish_leaving_pool`, which calls `exit_waiting_room` on a non-exiting PlotNFT, raises `ValueError`, closes the delivering peer, and repeats indefinitely.

### Finding Description
`PlotNFTStore.rollback_to_block` deletes rows from `plotnft2s` and `pool_reward2s` but leaves `finish_exiting_height` and `finish_exiting_info` intact. After a reorg that undoes the exit-to-waiting-room spend:

1. `get_latest_plotnft` returns the pre-exit (non-exiting) PlotNFT.
2. `get_exiting_height` still returns the old target height.
3. Once the unconfirmed exit transaction is dropped, `new_peak` calls `_finish_leaving_pool`.
4. `exit_waiting_room` raises `ValueError("Cannot exit waiting room while not in it")`.
5. The exception propagates uncaught through `WalletStateManager.new_peak`, is caught by `_process_new_subscriptions`, which closes the peer.
6. `finish_exiting_height` is never cleared, so the cycle repeats on every new peak.

### Impact Explanation
The wallet permanently fails to finish leaving the pool and repeatedly disconnects from every peer that delivers a new block, degrading sync to a loop of connect → `ValueError` → disconnect. This is a long-lived inability to process sync updates and pool actions.

### Likelihood Explanation
Requires (a) a reorg that rolls back the exit-to-waiting-room spend (natural 1-block reorgs are common) and (b) the exit transaction being dropped from the mempool before re-confirmation (possible when the wallet is briefly offline or the mempool is full). Neither condition requires privileged access; once both hold, any peer advancing the height past `finish_exiting_height + 2` triggers the loop.

### Recommendation
In `PlotNFTStore.rollback_to_block`, also clear `finish_exiting_height` and `finish_exiting_info` for any wallet whose exiting PlotNFT row is being deleted:

```python
async def rollback_to_block(self, *, height: int) -> None:
    async with self.db_wrapper.writer_maybe_transaction() as conn:
        # Find wallet_ids whose exiting row is being rolled back
        rows = await conn.execute_fetchall(
            "SELECT DISTINCT wallet_id FROM finish_exiting_height feh "
            "JOIN plotnft2s p ON p.exiting = 1 AND p.created_height > ?",
            (height,),
        )
        for (wallet_id,) in rows:
            await conn.execute("DELETE FROM finish_exiting_info WHERE wallet_id = ?", (wallet_id,))
            await conn.execute("DELETE FROM finish_exiting_height WHERE wallet_id = ?", (wallet_id,))
        await conn.execute("DELETE FROM plotnft2s WHERE created_height > ?", (height,))
        await conn.execute("DELETE FROM pool_reward2s WHERE height > ?", (height,))
        await conn.execute("UPDATE pool_reward2s SET spent_height = NULL WHERE spent_height > ?", (height,))
```

Additionally, add a guard in `_finish_leaving_pool` asserting `plotnft.exiting` before calling `exit_waiting_room`.

### Proof of Concept
1. Create a PlotNFT wallet in pooling state at block N.
2. Call `leave_pool`; confirm the exit-to-waiting-room spend at block N+1. Assert `finish_exiting_height` is set.
3. Call `plotnft2_store.rollback_to_block(height=N)`. Assert the exiting row is gone but `finish_exiting_height` is still set.
4. Clear the tx_store of unconfirmed transactions (simulating a dropped mempool entry).
5. Call `wallet.new_peak(finish_exiting_height + 3)`.
6. Observe `ValueError("Cannot exit waiting room while not in it")` raised from `exit_waiting_room`.
7. Assert `finish_exiting_height` is still set; repeat step 5 — same error every time.
