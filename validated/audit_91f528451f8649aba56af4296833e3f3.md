### Title
Stale `rsETHPrice` After Downside-Pause Early Return Causes Protocol to Miss Fees on Post-Pause Yield — (`contracts/LRTOracle.sol`)

### Summary
When `_updateRsETHPrice()` triggers the downside-protection pause (`isPriceDecreaseOffLimit`), it returns early at line 281 **without writing `newRsETHPrice` to `rsETHPrice`**. The stored `rsETHPrice` is therefore left at the pre-drop (inflated) value. On the next call after unpausing, `previousTVL` is computed from this stale price, so any genuine yield accrued while the protocol was paused is invisible to the fee mechanism.

### Finding Description

Inside `_updateRsETHPrice()`:

**Step 1 — baseline TVL is anchored to the stored (stale) price:** [1](#0-0) 

```solidity
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);   // uses stored rsETHPrice
```

**Step 2 — a new (lower) price is computed:** [2](#0-1) 

**Step 3 — if the drop exceeds `pricePercentageLimit`, the protocol pauses and returns early:** [3](#0-2) 

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;          // ← exits before rsETHPrice = newRsETHPrice (line 313)
}
```

**Step 4 — the write that never happens:** [4](#0-3) 

```solidity
rsETHPrice = newRsETHPrice;   // never reached when early return fires
```

**Concrete scenario:**

| State | `rsETHPrice` (stored) | Actual TVL | `previousTVL` computed |
|---|---|---|---|
| Before drop | 1.05 ETH | 1050 ETH | — |
| Drop event (e.g. slashing) | **still 1.05 ETH** (stale) | 900 ETH | — |
| Pause fires, early return | **still 1.05 ETH** | 900 ETH | — |
| Yield accrues during pause | **still 1.05 ETH** | 950 ETH | — |
| Unpause → `updateRSETHPrice()` | **still 1.05 ETH** | 950 ETH | `1000 × 1.05 = 1050 ETH` |

At the post-unpause call:
- `totalETHInProtocol = 950 ETH`
- `previousTVL = 1050 ETH` (stale baseline)
- `950 < 1050` → `protocolFeeInETH = 0`

The 50 ETH of genuine yield accrued during the pause (900 → 950 ETH) generates **zero protocol fee**. [5](#0-4) 

This persists across every subsequent call until `totalETHInProtocol` climbs back above the stale `previousTVL` of 1050 ETH — which may never happen if the slashing event was permanent.

### Impact Explanation
The protocol treasury fails to collect fees on all yield accrued between the pause event and the point (if ever) where TVL recovers past the stale baseline. No user funds are lost, but the protocol fails to deliver its promised fee revenue. This matches **Low — Contract fails to deliver promised returns, but doesn't lose value**.

### Likelihood Explanation
The trigger condition (`isPriceDecreaseOffLimit`) is an intended production path. Any slashing event or significant LST de-peg that moves the price beyond `pricePercentageLimit` will activate it. Yield continues to accrue on remaining assets during the pause. The subsequent fee-miss is automatic and requires no attacker action.

### Recommendation
Before the early return, write the computed `newRsETHPrice` to `rsETHPrice` so the baseline is accurate when the protocol resumes:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
+   rsETHPrice = newRsETHPrice;   // anchor baseline to actual drop level
    _pause();
    return;
}
```

This ensures that after unpausing, `previousTVL` reflects the true post-drop state and fees are correctly collected on all subsequent yield.
