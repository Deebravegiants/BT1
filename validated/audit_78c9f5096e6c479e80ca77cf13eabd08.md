### Title
Gateway Admission Uses Stale Previous-Block L2 Gas Price Instead of `next_l2_gas_price` for Fee Threshold Validation — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price from the **previous committed block** (`get_block_info().gas_prices.strk_gas_prices.l2_gas_price`) to enforce the minimum gas-price threshold, while the correct reference price for the block being built is `next_l2_gas_price` stored in the same block header. The same stale block info is forwarded into `run_validate_entry_point`, where only the block number is bumped to `unchecked_next()` but the gas prices are left at the previous block's values. An acknowledged TODO comment in the source code confirms the wrong field is being used.

### Finding Description

`SyncStateReaderFactory::get_blockifier_state_reader_and_gateway_fixed_block_from_latest_block` snapshots `latest_block_number` at the moment a transaction arrives and constructs a `GatewayFixedBlockSyncStateClient` pinned to that number. [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info` fetches and permanently caches the header of that pinned block via `OnceCell`. [2](#0-1) 

`validate_resource_bounds` then reads `l2_gas_price` from that cached header and uses it as the threshold for the **next** block: [3](#0-2) 

The inline comment `// TODO(Arni): getnext_l2_gas_price from the block header.` explicitly acknowledges that `next_l2_gas_price` (the EIP-1559-style price the consensus layer will actually use for the next block) should be read instead. [4](#0-3) 

The same stale `BlockInfo` is reused in `run_validate_entry_point`, where only the block number is incremented while all gas prices remain from the previous block: [5](#0-4) 

The consensus orchestrator confirms that `next_l2_gas_price` from the block header is the authoritative price for the next block: [6](#0-5) 

### Impact Explanation

Under EIP-1559-style pricing, `next_l2_gas_price` can be materially higher than the current block's `l2_gas_price` during periods of high network usage. A transaction whose `max_price_per_unit` satisfies the stale threshold but falls below the real next-block threshold will:

1. **Pass gateway admission** — `validate_resource_bounds` accepts it because the threshold is computed from the wrong (lower) price.
2. **Pass the blockifier's fee-sufficiency check** — `run_validate_entry_point` builds a `BlockContext` with the previous block's gas prices, so `verify_can_pay_committed_bounds` also uses the wrong price.
3. **Be admitted to the mempool** and forwarded to the batcher.
4. **Be rejected or reverted during block building** — the batcher uses the correct `next_l2_gas_price`, so the transaction's resource bounds are insufficient.

The net effect is that the gateway/mempool admission layer accepts transactions that are invalid for the block they will be sequenced into, satisfying the "High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing" impact criterion.

### Likelihood Explanation

The condition is triggered whenever the L2 gas price rises between consecutive blocks, which is the normal EIP-1559 behaviour during any period of elevated demand. No privileged access or special network position is required; any user submitting a transaction with a gas price calibrated to the previous block's price will trigger the path. The TODO comment confirms the developers are aware the wrong field is used, meaning the defect is present in every gateway validation today.

### Recommendation

1. In `validate_resource_bounds`, replace the read of `l2_gas_price` with `next_l2_gas_price` from the block header:

```rust
// Before (wrong generation):
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;

// After (correct generation):
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_block_l2_gas_price()  // new method reading next_l2_gas_price
    .await?;
```

2. Extend `GatewayFixedBlockStateReader` with a `get_next_block_l2_gas_price()` method that reads `block_header_without_hash.next_l2_gas_price` from the pinned block.

3. In `run_validate_entry_point`, update the gas prices in `block_info` to reflect `next_l2_gas_price` before constructing `BlockContext`, so the fee-sufficiency check is also performed against the correct generation.

### Proof of Concept

1. Observe the current L2 gas price `P_curr` from the latest committed block header.
2. Observe `P_next = block_header.next_l2_gas_price > P_curr` (e.g., during a traffic spike).
3. Submit an `InvokeV3` transaction with `l2_gas.max_price_per_unit = P_curr * min_gas_price_percentage / 100` (just at the stale threshold).
4. The gateway calls `validate_resource_bounds`, computes `threshold = P_curr * percentage / 100`, and accepts the transaction.
5. The batcher builds the next block with `l2_gas_price = P_next`; the transaction's `max_price_per_unit` is below `P_next * percentage / 100` and is rejected/reverted.
6. The transaction was admitted through the gateway and mempool despite being invalid for the block it was sequenced into. [7](#0-6) [8](#0-7)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
