Audit Report

## Title
`ChainlinkPriceOracle.getAssetPrice()` Accepts Stale Chainlink Data With No Validation, Enabling rsETH Dilution and Yield Theft — (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` discards all validation fields from `latestRoundData()`, accepting stale prices without any staleness, completeness, or sign check. A stale low price for any LST asset (e.g., stETH, ETHx) propagates through the permissionless `updateRSETHPrice()` into the stored `rsETHPrice`, which is then used as the denominator when minting rsETH for depositors. An attacker who deposits ETH immediately after forcing a deflated `rsETHPrice` receives more rsETH than their deposit warrants, extracting value from existing holders when the price recovers.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` reads only the `answer` field:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No check on `updatedAt`, `answeredInRound >= roundId`, `price > 0`, or any heartbeat window. The same codebase already implements the correct pattern in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L27-32
(uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
    AggregatorV3Interface(oracle).latestRoundData();
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The stale price propagates as follows:

1. `LRTOracle.updateRSETHPrice()` (public, no access control) → `_updateRsETHPrice()`
2. `_updateRsETHPrice()` calls `_getTotalEthInProtocol()`, which calls `getAssetPrice(asset)` for every supported LST
3. A stale low price for any LST undervalues `totalETHInProtocol`
4. `newRsETHPrice = (totalETHInProtocol - fee).divWad(rsethSupply)` is set below true value and stored in `rsETHPrice`

When a depositor (or attacker) then calls `depositETH()` or `depositAsset()`, the mint calculation is:

```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

For an ETH deposit, `getAssetPrice(ETH) = 1e18` (unaffected by the stale LST feed), but `rsETHPrice` is the deflated stored value — so the attacker receives more rsETH than their deposit is worth at the true rate.

## Impact Explanation

**High — Theft of unclaimed yield.** When the Chainlink LST/ETH feed recovers and `rsETHPrice` is updated to its true value, the attacker's over-minted rsETH redeems for more ETH than was deposited. The surplus is extracted from the proportional share of existing rsETH holders (their yield is diluted). This is a direct, concrete transfer of value from existing holders to the attacker, matching the "Theft of unclaimed yield" impact class.

Secondary impact: if the stale price drop exceeds `pricePercentageLimit`, `_updateRsETHPrice()` pauses both `LRTDepositPool` and `LRTWithdrawalManager`, temporarily freezing all user funds until an admin manually unpauses — matching "Temporary freezing of funds" (Medium).

## Likelihood Explanation

Chainlink LST/ETH feeds (e.g., stETH/ETH on mainnet) have 24-hour heartbeat intervals and can lag during periods of low L1 activity, sequencer downtime, or network congestion. The attacker requires no privileged access: they only need to monitor the feed's `updatedAt` off-chain, observe a stale price below the true market rate, and execute two permissionless transactions (`updateRSETHPrice()` followed by `depositETH()`). No governance capture, no flash loan, and no brute force is required. The condition (stale feed at a below-market price) is realistic and has occurred historically on Chainlink feeds.

## Recommendation

Apply the same staleness guards already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optional: per-asset configurable heartbeat
    // if (block.timestamp - updatedAt > maxStaleness[asset]) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, consider a configurable per-asset maximum staleness window so feeds that have not updated within their expected heartbeat interval are rejected rather than silently accepted.
