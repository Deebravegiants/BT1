Audit Report

## Title
Undistributed Reward Tokens Permanently Stuck When `totalKernelStaked` Drops to Zero During Active Reward Period - (File: contracts/KERNEL/KernelDepositPool.sol)

## Summary
When all stakers call `initiateWithdrawal()` during an active reward period, `totalKernelStaked` immediately drops to zero, causing `rewardPerToken()` to freeze and all future reward emissions to be unearnable. The remaining reward tokens are permanently locked in the contract with no recovery path, as `notifyRewardAmount` also reverts with `NoStakedTokens` when `totalKernelStaked == 0`, and no `recoverERC20` function exists.

## Finding Description
**Root cause — `rewardPerToken()` short-circuits at zero supply:** [1](#0-0) 

When `totalKernelStaked == 0`, the function returns `rewardPerTokenStored` unchanged. The `rewardRate` remains non-zero (set by a prior `notifyRewardAmount` call), but no address accumulates those emissions.

**`initiateWithdrawal()` immediately decrements `totalKernelStaked` at initiation, not at claim time:** [2](#0-1) 

If the last staker calls `initiateWithdrawal()` while `block.timestamp < finishAt`, `totalKernelStaked` becomes 0 mid-period and all remaining reward emissions are lost.

**`notifyRewardAmount` blocks any restart:** [3](#0-2) 

Even after `finishAt` passes, the admin cannot call `notifyRewardAmount` to restart distribution because the guard reverts when `totalKernelStaked == 0`.

**No recovery function exists.** The complete set of admin functions is `setRewardsDuration`, `notifyRewardAmount`, `setWithdrawalDelay`, and `setMaxNumberOfWithdrawalsPerUser` — none can extract stuck `rewardsToken` balance. [4](#0-3) 

The contract's own NatSpec acknowledges this behavior and states the mitigation is purely operational ("ensuring there are always some tokens staked"), not enforced by code: [5](#0-4) 

## Impact Explanation
**Medium. Permanent freezing of unclaimed yield.**

The `rewardsToken` balance equal to `rewardRate × (finishAt − T)` (where T is the timestamp when `totalKernelStaked` reached 0) is permanently locked in the contract. There is no admin or user path to recover it. This exactly matches the allowed impact: *"Medium. Permanent freezing of unclaimed yield."*

## Likelihood Explanation
**Low-Medium.** `initiateWithdrawal()` is a normal, permissionless user action callable at any time. The scenario requires `totalKernelStaked` to reach zero during an active reward period. This can be triggered by a single large staker holding 100% of stake, a coordinated exit, or a loss-of-confidence event. The `maxNumberOfWithdrawalsPerUser` cap does not prevent this; it only limits concurrent pending withdrawals per user, not the ability to withdraw the full staked balance. [6](#0-5) 

## Recommendation
Add an admin-restricted `recoverERC20` function that allows recovering `rewardsToken` in excess of what has been earned but not yet claimed (i.e., `rewardsToken.balanceOf(address(this)) - sum(rewards[user])`). Alternatively, when `totalKernelStaked` drops to 0 mid-period, record the remaining undistributed rewards (`rewardRate × (finishAt − block.timestamp)`) and allow the admin to reclaim them after `finishAt`. A simpler mitigation is to remove the `NoStakedTokens` guard from `notifyRewardAmount` so that new stakers can re-enter and absorb the stuck balance in a subsequent period.

## Proof of Concept
1. Admin calls `notifyRewardAmount(1_000e18)` with `duration = 30 days`. `rewardRate = 1_000e18 / 30 days`.
2. On day 3, the sole staker (or all stakers) call `initiateWithdrawal()`. `totalKernelStaked → 0`.
3. `rewardPerToken()` now returns `rewardPerTokenStored` unchanged for the remaining 27 days.
4. Approximately `rewardRate × 27 days ≈ 900e18` reward tokens are never earned by anyone.
5. After day 30, `finishAt` passes. The `rewardsToken` balance of the contract still holds ~900e18.
6. Admin attempts `notifyRewardAmount(...)` → reverts with `NoStakedTokens`.
7. No other function can extract the stuck `rewardsToken` balance. Funds are permanently frozen.

**Foundry test plan:**
```solidity
function test_rewardsFrozenWhenTotalStakedZero() public {
    // Setup: stake, notify reward, fast-forward 3 days, withdraw all
    vm.prank(admin);
    pool.notifyRewardAmount(1_000e18); // duration = 30 days

    vm.warp(block.timestamp + 3 days);
    vm.prank(staker);
    pool.initiateWithdrawal(stakerBalance); // totalKernelStaked → 0

    // Fast-forward past finishAt
    vm.warp(block.timestamp + 28 days);

    // Assert ~900e18 rewardsToken stuck in contract
    assertGt(rewardsToken.balanceOf(address(pool)), 800e18);

    // Assert admin cannot recover via notifyRewardAmount
    vm.prank(admin);
    vm.expectRevert(KernelDepositPool.NoStakedTokens.selector);
    pool.notifyRewardAmount(1e18);
}
```

```text
    function initiateWithdrawal(uint256 _amount) external nonReentrant updateReward(msg.sender) {
        if (_amount == 0) revert AmountZero();
        if (balanceOf[msg.sender] < _amount) revert InsufficientStakedBalance();
        if (userWithdrawalIds[msg.sender].length >= maxNumberOfWithdrawalsPerUser) revert WithdrawalLimitReached();

        balanceOf[msg.sender] -= _amount;
        totalKernelStaked -= _amount;

        // Create a withdrawal record
        uint256 withdrawalId = ++withdrawalCounter;
        uint256 unlockTime = block.timestamp + withdrawalDelay;

        withdrawals[withdrawalId] = Withdrawal({
            user: msg.sender, amount: _amount, unlockTime: unlockTime, claimed: false, withdrawalId: withdrawalId
        });
        userWithdrawalIds[msg.sender].push(withdrawalId);

        emit WithdrawalInitiated(msg.sender, _amount, withdrawalId, unlockTime);
    }
```

```text
    /**
     * @notice Sets the duration for rewards distribution
     * @param _duration The duration in seconds
     */
    function setRewardsDuration(uint256 _duration) external onlyRole(DEFAULT_ADMIN_ROLE) {
        if (finishAt >= block.timestamp) revert RewardDurationNotFinished();
        if (_duration == 0) revert InvalidDuration();
        duration = _duration;
        emit RewardsDurationUpdated(_duration);
    }

    /**
     * @notice Notifies the contract about a new reward amount
     * @dev Uses a transfer-in pattern to determine the exact reward amount received.
     *      Also, to avoid undistributed rewards when no one is staked, this function reverts if totalKernelStaked is
     *      zero.
     * @param _amount The amount of reward tokens to add
     */
    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }

    /**
     * @notice Updates the withdrawal delay
     * @param _withdrawalDelay The new withdrawal delay
     */
    function setWithdrawalDelay(uint256 _withdrawalDelay) external onlyRole(DEFAULT_ADMIN_ROLE) {
        if (_withdrawalDelay == 0) revert InvalidWithdrawalDelay();
        if (_withdrawalDelay > MAX_WITHDRAWAL_DELAY) revert MaximumWithdrawalDelayExceeded();

        withdrawalDelay = _withdrawalDelay;
        emit WithdrawalDelayUpdated(_withdrawalDelay);
    }

    /**
     * @notice Updates the maximum number of withdrawals per user
     * @param _maxNumberOfWithdrawalsPerUser The new maximum number of withdrawals per user
     */
    function setMaxNumberOfWithdrawalsPerUser(uint256 _maxNumberOfWithdrawalsPerUser)
        external
        onlyRole(DEFAULT_ADMIN_ROLE)
    {
        if (_maxNumberOfWithdrawalsPerUser == 0 || _maxNumberOfWithdrawalsPerUser > MAX_WITHDRAWALS_PER_USER) {
            revert InvalidMaxNumberOfWithdrawalsPerUser();
        }

        maxNumberOfWithdrawalsPerUser = _maxNumberOfWithdrawalsPerUser;
        emit MaxNumberOfWithdrawalsPerUserUpdated(_maxNumberOfWithdrawalsPerUser);
    }
```
