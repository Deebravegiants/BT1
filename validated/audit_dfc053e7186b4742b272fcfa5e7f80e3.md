### Title
Unvalidated `farmer_reward_address_override` Allows Connected Harvester to Redirect Farmer Block Rewards to Arbitrary Puzzle Hash — (`chia/farmer/farmer_api.py`)

### Summary

A connected third-party harvester can set `farmer_reward_address_override` in a `RespondSignatures` message to any arbitrary 32-byte puzzle hash. The farmer accepts this value unconditionally and uses it as the `farmer_reward_puzzle_hash` in `DeclareProofOfSpace`, redirecting the farmer's block reward (1/8 XCH + transaction fees) to the attacker-controlled address. The CHIP-22 fee-quality threshold check exists only as a log warning and is never enforced.

### Finding Description

`RespondSignatures` carries an optional `farmer_reward_address_override: bytes32 | None` field: [1](#0-0) 

In `_process_respond_signatures`, the farmer unconditionally replaces its configured `farmer_target` with whatever the harvester supplied: [2](#0-1) 

The resulting `farmer_reward_address` is placed directly into `DeclareProofOfSpace` and broadcast to the full node: [3](#0-2) 

The only guard that exists is `notify_farmer_reward_taken_by_harvester_as_fee`, which is called earlier when `NewProofOfSpace` arrives: [4](#0-3) 

That function computes a fee-quality score and emits log warnings when the threshold is not met, but **never blocks or rejects the override**: [5](#0-4) 

There is no whitelist of permitted override addresses, no enforcement of the fee-quality threshold, and no check that the override in `RespondSignatures` matches what was declared in `NewProofOfSpace`. Any puzzle hash — including one the harvester operator controls — is accepted silently.

### Impact Explanation

Every time the farmer wins a block while connected to a malicious third-party harvester, the farmer's 1/8-XCH reward (plus all transaction fees in that block) is paid to the harvester-controlled puzzle hash instead of the farmer's configured `xch_target_address`. The farmer receives no reward for the block. This is a direct, irreversible loss of XCH.

### Likelihood Explanation

CHIP-22 explicitly encourages farmers to connect to third-party harvester services. Any such service can silently activate this redirection for every block the farmer wins. The farmer sees only a log warning (if `fee_info` is absent or the threshold is exceeded) but the block reward is already gone. No key compromise, no admin access, and no on-chain action by the farmer is required.

### Recommendation

1. **Enforce the fee-quality threshold**: if `fee_quality > fee_threshold` (or `fee_info` is absent), set `farmer_reward_address_override = None` before constructing `DeclareProofOfSpace` rather than merely logging a warning.
2. **Whitelist permitted override addresses**: allow farmers to configure a set of puzzle hashes that harvesters are permitted to redirect rewards to, and reject any override not in that set.
3. **Verify consistency**: confirm that the `farmer_reward_address_override` in `RespondSignatures` matches the value (if any) that was declared in the corresponding `NewProofOfSpace` message.

### Proof of Concept

1. Operate a third-party harvester service that farmers connect to via CHIP-22.
2. In the harvester's `request_signatures` handler, return a `RespondSignatures` with `farmer_reward_address_override` set to an attacker-controlled puzzle hash and `fee_info=None` (or any threshold value).
3. The farmer's `_process_respond_signatures` replaces `self.farmer.farmer_target` with the attacker's puzzle hash (line 918) and broadcasts `DeclareProofOfSpace` to the full node with that address as `farmer_reward_puzzle_hash`.
4. The full node creates the farmer-reward coin at the attacker's puzzle hash. The farmer receives nothing. [6](#0-5) [7](#0-6) [1](#0-0)

    def notify_farmer_reward_taken_by_harvester_as_fee(
        self, sp: farmer_protocol.NewSignagePoint, proof_of_space: harvester_protocol.NewProofOfSpace
    ) -> None:
        """
        Apply a fee quality convention (see CHIP-22: https://github.com/Chia-Network/chips/pull/88)
        given the proof and signage point. This will be tested against the fee threshold reported
        by the harvester (if any), and logged.
        """
        assert proof_of_space.farmer_reward_address_override is not None

        challenge_str = str(sp.challenge_hash)

        ph_prefix = self.config["network_overrides"]["config"][self.config["selected_network"]]["address_prefix"]
        farmer_reward_puzzle_hash = encode_puzzle_hash(proof_of_space.farmer_reward_address_override, ph_prefix)

        self.log.info(
            f"Farmer reward for challenge '{challenge_str}' "
            + f"taken by harvester for reward address '{farmer_reward_puzzle_hash}'"
        )

        fee_quality = calculate_harvester_fee_quality(proof_of_space.proof.proof, sp.challenge_hash)
        fee_quality_rate = float(fee_quality) / float(0xFFFFFFFF) * 100.0

        if proof_of_space.fee_info is not None:
            fee_threshold = proof_of_space.fee_info.applied_fee_threshold
            fee_threshold_rate = float(fee_threshold) / float(0xFFFFFFFF) * 100.0

            if fee_quality <= fee_threshold:
                self.log.info(
                    f"Fee threshold passed for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
            else:
                self.log.warning(
                    f"Invalid fee threshold for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
                self.log.warning(
                    "Harvester illegitimately took a fee reward that "
                    + "did not belong to it or it incorrectly applied the fee convention."
                )
        else:
            self.log.warning(
                "Harvester illegitimately took reward by failing to provide its fee rate "
                + f"for challenge '{challenge_str}'. "
                + f"Fee quality was {fee_quality_rate:.3f}% ({fee_quality} or 0x{fee_quality:08x})"
            )
