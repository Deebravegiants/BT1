### Title
`respond_ckd` Skips CKD Output Verification for `AppPublicKey` Variant, Enabling a Single Malicious Participant to Deliver a Forged Confidential Key - (File: crates/contract/src/lib.rs)

### Summary
`respond_ckd` in `crates/contract/src/lib.rs` calls `ckd_output_check` only for the `CKDAppPublicKey::AppPublicKeyPV` variant. For the `CKDAppPublicKey::AppPublicKey` variant the match arm is empty, so the verification result is never computed and the response is accepted unconditionally. A single malicious attested participant can call `respond_ckd` with any forged `CKDResponse`; the contract passes it directly to `resolve_yields_for`, resolving all pending yields and delivering an invalid confidential key to the requesting user.

For `AppPublicKey` requests, the error indicator is never computed and the invalid response is used directly, exactly as the external report describes callers ignoring the error code from `getBuyInfo()`.
