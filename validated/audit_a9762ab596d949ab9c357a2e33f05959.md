### Title
Wrong Rounding Direction in `tierRefScriptFee` Allows Underpayment of Reference Script Minimum Fee - (File: `eras/conway/impl/src/Cardano/Ledger/Conway/Tx.hs`)

### Summary

`tierRefScriptFee` uses `floor` to convert the rational fee accumulator to a `Coin` integer. Because this function computes a **minimum fee** component, it should round **up** (ceiling) ‚Äî exactly as the analogous `txscriptfee` (Plutus execution-unit fee) does in the formal spec and implementation. Using `floor` allows a transaction submitter to pay 1 lovelace less than the true minimum reference-script fee whenever the rational result is non-integer, modifying fees outside design parameters.

---

### Finding Description

`tierRefScriptFee` is the Conway-era function that computes the tiered minimum fee for reference scripts. Its final rounding step is:

```haskell
go !acc !curTierPrice !n
  | n < sizeIncrement =
      Coin $ floor (acc + toRational n * curTierPrice)   -- ‚Üê rounds DOWN
``` [1](#0-0) 

`curTierPrice` is a `Rational` (derived from `ppMinFeeRefScriptCostPerByteL` and the growth multiplier), so `acc + toRational n * curTierPrice` is generally non-integer. `floor` truncates toward zero, yielding a result that is potentially 1 lovelace below the true minimum.

This is directly inconsistent with the formal specification and implementation of `txscriptfee`, the analogous Plutus execution-unit fee, which explicitly uses `ceiling`:

> "The prices are expressed as rational numbers, so we take the ceiling of the result of the calculation to get an integer-valued Coin." [2](#0-1) 

```haskell
txscriptfee Prices {prMem, prSteps} ExUnits {exUnitsMem = m, exUnitsSteps = s} =
  Coin $ ceiling $ ...
``` [3](#0-2) 

`tierRefScriptFee` is called from `getConwayMinFeeTx`, which is the `getMinFeeTx` implementation for `ConwayEra`:

```haskell
getConwayMinFeeTx pp tx refScriptsSize =
  alonzoMinFeeTx pp tx <+> refScriptsFee
  where
    refScriptsFee = tierRefScriptFee ...
``` [4](#0-3) 

The resulting `minFee` is then enforced in `feesOK` via:

```haskell
failureUnless (minFee <= theFee) (FeeTooSmallUTxO ...)
``` [5](#0-4) 

Because `minFee` is 1 lovelace too low, the check passes for a transaction that pays 1 lovelace less than the true minimum.

---

### Impact Explanation

**Medium** ‚Äî Attacker-controlled transactions modify fees outside design parameters.

Every transaction that uses reference scripts and whose rational fee result is non-integer can be submitted with a fee 1 lovelace below the true minimum. The fee pot (and ultimately the reward pot and treasury) receives 1 lovelace less per such transaction than the protocol intends. This is a systematic, per-transaction deviation from the fee schedule introduced in Conway.

---

### Likelihood Explanation

**High** ‚Äî The `ppMinFeeRefScriptCostPerByteL` parameter is a `NonNegativeInterval` (rational), and the tiered multiplier is also rational. For any reference-script size that is not an exact multiple of the stride, the accumulated rational will be non-integer, triggering the rounding discrepancy on every such transaction.

---

### Recommendation

Replace `floor` with `ceiling` in the final accumulation step of `tierRefScriptFee`, consistent with `txscriptfee`:

```haskell
-- Before
Coin $ floor (acc + toRational n * curTierPrice)

-- After
Coin $ ceiling (acc + toRational n * curTierPrice)
``` [1](#0-0) 

---

### Proof of Concept

Let `ppMinFeeRefScriptCostPerByte = 15`, stride = 25600, multiplier = 1.2, and total reference-script size = 1 byte (well within the first tier).

- True minimum fee = `ceiling(1 * 15) = 15` lovelace (exact integer, no difference here).

Now let size = 3 bytes and `ppMinFeeRefScriptCostPerByte = 10 % 3` (a rational value, which is valid since the parameter type is `NonNegativeInterval`):

- Rational result = `3 * (10/3) = 10.0` ‚Äî exact, no difference.

Let size = 1 byte and `ppMinFeeRefScriptCostPerByte = 2 % 3`:

- Rational result = `1 * (2/3) = 0.666...`
- `floor(0.666) = 0` lovelace ‚Äî computed minimum fee = 0
- `ceiling(0.666) = 1` lovelace ‚Äî correct minimum fee = 1

A transaction with 1 byte of reference scripts and this parameter setting passes `feesOK` with `txFee = 0`, paying 1 lovelace less than the protocol-intended minimum. The attacker entry path is simply submitting a Conway-era transaction with any reference input whose UTxO output contains a reference script. [6](#0-5) [7](#0-6)
