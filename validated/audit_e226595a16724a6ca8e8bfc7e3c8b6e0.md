### Title
Missing CKD Output Verification for `AppPublicKey` Variant Allows Single Byzantine Node to Substitute Arbitrary Key Derivation Output — (`crates/contract/src/lib.rs`)

### Summary

`respond_ckd` performs the pairing-based `ckd_output_check` only for the `AppPublicKeyPV` variant. For the `AppPublicKey` variant the match arm is an explicit no-op. Because `resolve_yields_for` resolves all queued yields immediately upon a single call, one attested-but-Byzantine participant can submit any `(big_y, big_c)` pair for any pending `AppPublicKey` CKD request and the contract will accept and deliver those attacker-chosen values to the caller.

### Finding Description

In `respond_ckd` the output-validity gate is:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}          // ← no check
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
``` [1](#0-0) 

`ckd_output_check` verifies the pairing equation `e(big_c, g2) = e(big_y, app_pk2) · e(hash_point, public_key)`, which cryptographically binds the response to the network's master public key and the caller's app key. This check requires the G2 component (`pk2`) that is only present in `AppPublicKeyPV`. [2](#0-1) 

`AppPublicKey` carries only a G1 point, so the pairing check is structurally impossible for that variant — but the contract still accepts and resolves the response without any substitute verification. [3](#0-2) 

`resolve_yields_for` has no threshold aggregation: the first single call to `respond_ckd` that matches a pending request key drains the entire yield queue and delivers the response to all waiting callers. [4](#0-3) 

### Impact Explanation

A single attested participant (one Byzantine node, well below the signing threshold) can:

1. Observe a pending `AppPublicKey` CKD request.
2. Call `respond_ckd` with `big_y = G1_identity` and `big_c = g1 * t` for an attacker-chosen scalar `t`.
3. The contract accepts the response unconditionally and resolves the yield.
4. The caller computes `big_c − app_sk · big_y = g1·t − app_sk·identity = g1·t`.
5. The attacker knows `t`, so they know the caller's derived private key.

This enables the attacker to forge signatures on behalf of the caller for any `AppPublicKey` CKD request, matching the critical impact: *unauthorized access to secret material that materially enables forgery or secret recovery*. [5](#0-4) 

### Likelihood Explanation

`AppPublicKey` is a supported, non-deprecated variant accepted by the live contract. Any single attested participant can exploit this — no threshold collusion, no key leakage, and no special privilege beyond being an attested node is required. The attack is deterministic and requires only one malicious `respond_ckd` call. [6](#0-5) 

### Recommendation

Either:

- **Deprecate and reject `AppPublicKey` requests** at the `request_app_private_key` entry point, forcing all callers to use `AppPublicKeyPV` which supports on-chain output verification; or
- **Add an equivalent output check for `AppPublicKey`**: since only a G1 key is available, a threshold-aggregated proof (e.g., requiring `t-of-n` nodes to submit matching responses before resolving) must replace the pairing check, because a single-G1-key variant cannot support the pairing equation.

The empty match arm `AppPublicKey(_) => {}` must not remain in production code while the variant is still accepted. [1](#0-0)

pub(crate) fn ckd_output_check(
    app_id: &dtos::CkdAppId,
    output: &CKDResponse,
    app_public_key: &dtos::CKDAppPublicKeyPV,
    public_key: &dtos::Bls12381G2PublicKey,
) -> bool {
    let big_c = env::bls12381_p1_decompress(&output.big_c);
    let big_y = env::bls12381_p1_decompress(&output.big_y);
    let pk2 = env::bls12381_p2_decompress(&app_public_key.pk2);
    let pk = env::bls12381_p2_decompress(public_key);
    let hash_point = hash_app_id_with_pk(public_key.as_slice(), app_id.as_ref());

    let pairing_input = [
        big_c.as_slice(),
        MINUS_G2_GENERATOR_UNCOMPRESSED.as_slice(),
        big_y.as_slice(),
        pk2.as_slice(),
        hash_point.as_slice(),
        pk.as_slice(),
    ]
    .concat();
    env::bls12381_pairing_check(&pairing_input)
}

pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
