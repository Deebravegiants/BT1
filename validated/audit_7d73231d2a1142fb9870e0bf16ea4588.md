### Title
Proposer-Controlled ETH/FRI Rate Skew via Independent Gas-Price Margin Validation Corrupts `l2_gas_price_wei` in BlockInfo — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, `crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`is_proposal_init_valid` checks `l1_gas_price_fri` and `l1_gas_price_wei` independently against the validator's oracle readings, each within a symmetric `l1_gas_price_margin_percent` band. It never validates their **ratio**. `convert_to_sn_api_block_info` then back-calculates the ETH→FRI rate from those two proposer-supplied values and uses it to derive `l2_gas_price_wei` for the executed block's `BlockInfo`. A malicious proposer can set FRI at the top of its margin and WEI at the bottom (or vice-versa), both passing validation, while skewing the back-calculated rate by up to `(1+m)/(1-m) − 1 ≈ 2m` (≈ 22 % for the 10 % margin). The resulting `l2_gas_price_wei` embedded in every transaction's execution context is wrong, causing ETH-denominated fees to be systematically under- or over-charged for the entire block.

---

### Finding Description

**Step 1 – Independent margin checks in `is_proposal_init_valid`** [1](#0-0) 

The validator computes its own reference values from the oracle and checks each of the four prices independently:

```
within_margin(l1_gas_price_fri_proposed,  l1_gas_price_fri,  margin%)
within_margin(l1_gas_price_wei_proposed,  l1_gas_price_wei,  margin%)
```

No check is made on the ratio `l1_gas_price_fri / l1_gas_price_wei`.

**Step 2 – Back-calculation of ETH→FRI rate from proposer-supplied values** [2](#0-1) 

```rust
eth_to_fri_rate = l1_gas_price_fri * WEI_PER_ETH / l1_gas_price_wei
```

Both inputs come directly from the proposer's `ProposalInit`.

**Step 3 – Rate used to derive `l2_gas_price_wei` in BlockInfo** [3](#0-2) 

```rust
let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;
let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)?;
```

`l2_gas_price_wei` is then placed into `BlockInfo.eth_gas_prices.l2_gas_price` and used by the blockifier for every ETH-denominated fee calculation in the block.

**Step 4 – Manipulation arithmetic**

With `margin_percent = 10` (confirmed by test comments):

| Proposer sets | Resulting rate | `l2_gas_price_wei` |
|---|---|---|
| FRI = 1.1×ref, WEI = 0.9×ref | 1.222 × true rate | **0.818 × true** (users underpay ETH) |
| FRI = 0.9×ref, WEI = 1.1×ref | 0.818 × true rate | **1.222 × true** (users overpay ETH) |

Both proposals pass `is_proposal_init_valid` because each individual price is within its 10 % band.

**Step 5 – Fallback persistence** [4](#0-3) 

When the oracle fails in a subsequent block, `get_l1_prices_in_fri_and_wei_and_conversion_rate` falls back to the previous block's stored prices and calls `calculate_eth_to_fri_rate` on them. A manipulated previous block therefore poisons the fallback rate for all oracle-down blocks that follow.

---

### Impact Explanation

Every transaction in the manipulated block has its ETH-denominated L2 gas price (`l2_gas_price_wei`) wrong by up to ≈ 22 %. Users paying fees in ETH are systematically over- or under-charged. During oracle outages the wrong rate propagates to subsequent blocks. This is a direct, quantifiable economic impact on fee correctness — matching the "Incorrect fee … with economic impact" criterion.

---

### Likelihood Explanation

Any consensus validator that wins a proposal round can trigger this. No special privilege beyond being a proposer is required. The manipulation is invisible to other validators because each individual price passes the margin check. The proposer does not need to sacrifice funds (unlike the VToken donation attack); they simply choose values at opposite ends of the allowed band.

---

### Recommendation

Replace the four independent margin checks with a single check that also validates the **ratio** of the FRI and WEI prices against the locally-trusted ETH→FRI rate:

```rust
let local_rate = calculate_eth_to_fri_rate(&local_prices)?;
let proposed_rate = calculate_eth_to_fri_rate(&proposed_prices)?;
if !within_margin_u128(proposed_rate, local_rate, l1_gas_price_margin_percent) {
    return Err(InvalidProposalInit(..., "ETH/FRI rate out of margin"));
}
```

Anchoring the rate check to the locally-derived `local_rate` (not the proposer-supplied value) prevents the band from being widened by the proposer's own input — the same principle already documented in the `within_margin` comment. [5](#0-4) 

---

### Proof of Concept

```
Assume: l1_gas_price_margin_percent = 10
        oracle reference: l1_gas_price_fri = 1_000_000 FRI, l1_gas_price_wei = 1_000 WEI
        true ETH/FRI rate = 1_000_000 * 10^18 / 1_000 = 10^21

Proposer sends:
  l1_gas_price_fri = 1_100_000   (within 10% upper bound → passes within_margin)
  l1_gas_price_wei =   900       (within 10% lower bound → passes within_margin)

calculate_eth_to_fri_rate:
  rate = 1_100_000 * 10^18 / 900 = 1.222 * 10^21   (22% above true rate)

l2_gas_price_fri = 7_000_000_000 (validator-enforced, correct)
l2_gas_price_wei = 7_000_000_000 / (1.222 * 10^21) * 10^18
                 ≈ 5_728 WEI     (should be 7_000 WEI — 18% undercharge)

All validators accept the proposal (each individual price is within margin).
Every ETH-paying transaction in this block is charged 18% less than correct.
``` [2](#0-1) [6](#0-5) [7](#0-6)

/// Returns whether `proposed` is within `margin_percent` of the locally-trusted `reference`,
/// i.e. within the symmetric band `[reference*(1-m), reference*(1+m)]`.
///
/// The band is anchored to `reference` (the node's own L1 oracle read), not to the
/// proposer-supplied `proposed`: anchoring to `proposed` would let a malicious proposer scale the
/// band width with its own input and widen it in its favor.
fn within_margin(proposed: GasPrice, reference: GasPrice, margin_percent: u128) -> bool {
    // For small numbers (e.g., less than 10 wei, if margin is 10%), even an off-by-one
    // error might be bigger than the margin, even if it is just a rounding error.
    // We make an exception for such mismatch, and don't bother checking percentages
    // if the difference in price is only one wei.
    if proposed.0.abs_diff(reference.0) <= GAS_PRICE_ABS_DIFF_MARGIN {
        return true;
    }
    // Saturate: `reference.0 * margin_percent` can overflow u128 on large WEI prices.
    let margin = reference.0.saturating_mul(margin_percent) / 100;
    proposed.0.abs_diff(reference.0) <= margin
}

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
