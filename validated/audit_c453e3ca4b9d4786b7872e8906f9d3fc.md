### Title
Unauthorized Gas Key Balance Burn via Cross-Contract `DeleteKey` Action - (File: runtime/runtime/src/access_keys.rs)

### Summary
The gas key feature burns a gas key's NEAR balance when the key is deleted. However, `delete_gas_key` performs no check that the action's `predecessor_id` equals the `receiver_id` (account owner). Because NEAR's runtime allows any contract to create a receipt targeting any account and append a `DeleteKey` action, an unprivileged attacker can permanently destroy another account's gas key balance without the victim's consent.

### Finding Description
When a `DeleteKeyAction` targets a gas key, the runtime calls `action_delete_key` → `delete_gas_key`. Inside `delete_gas_key`, the gas key's stored balance is added to `result.tokens_burnt` and the key is removed from state — permanently destroying the NEAR tokens held in that gas key. [1](#0-0) 

The `account_id` passed to this function is always `receipt.receiver_id()`. There is no check that `receipt.predecessor_id() == receipt.receiver_id()` (i.e., that the action was initiated by the account owner). The runtime dispatches the action identically regardless of who sent the receipt: [2](#0-1) 

The `promise_batch_action_delete_key` host function is available to all contracts and accepts any `promise_idx`, including one created with `promise_batch_create` targeting an arbitrary account: [3](#0-2) 

The designers explicitly recognized that some gas key operations must be restricted to transactions only. The `WithdrawFromGasKeyAction` carries a comment stating it "must only be available via transactions, not via contract execution (there is no corresponding promise batch action host function)": [4](#0-3) 

No equivalent restriction was applied to `DeleteKey` when it targets a gas key, leaving the balance-burning path reachable from any contract.

### Impact Explanation
An attacker deploys a contract, calls `promise_batch_create("victim.near")`, then `promise_batch_action_delete_key(promise_idx, victim_gas_key_pubkey)`. When the resulting receipt executes on `victim.near`, `delete_gas_key` burns the entire gas key balance. The corrupted protocol value is the victim's gas key balance — NEAR tokens are permanently destroyed from the victim's account without authorization. This is a direct, irreversible loss of user funds.

### Likelihood Explanation
The attack requires only: (1) knowing the victim's gas key public key (observable on-chain via `view_access_keys`), and (2) deploying a contract and calling it. No privileged role is needed. The `promise_batch_action_delete_key` host function is universally available. Any funded gas key on any account is at risk.

### Recommendation
Apply the same restriction used for `WithdrawFromGasKeyAction`: when `DeleteKeyAction` targets a gas key, verify that `receipt.predecessor_id() == receipt.receiver_id()` before burning the balance. Alternatively, instead of burning the gas key balance on deletion, return it to the account's main balance — eliminating the destructive consequence of unauthorized deletion.

### Proof of Concept
1. Victim account `victim.near` has a gas key with public key `GK_PK` and balance 0.1 NEAR.
2. Attacker deploys `attacker_contract.near` with the following logic:
   ```
   let p = promise_batch_create("victim.near");
   promise_batch_action_delete_key(p, GK_PK);
   ```
3. Attacker calls `attacker_contract.near`.
4. The resulting receipt executes on `victim.near`: `action_delete_key` → `delete_gas_key` is called with `account_id = "victim.near"`, `gas_key_info.balance = 0.1 NEAR`.
5. `result.tokens_burnt += 0.1 NEAR` — the balance is burned.
6. The gas key is removed from state.
7. Victim's 0.1 NEAR is permanently destroyed; no refund is issued. [5](#0-4)

fn delete_gas_key(
    config: &RuntimeConfig,
    state_update: &mut TrieUpdate,
    account: &mut Account,
    result: &mut ActionResult,
    account_id: &AccountId,
    public_key: &PublicKey,
    access_key: &AccessKey,
    gas_key_info: &GasKeyInfo,
) -> Result<(), RuntimeError> {
    if gas_key_info.balance > GasKeyInfo::MAX_BALANCE_TO_BURN {
        result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh {
            account_id: account_id.clone(),
            public_key: Some(Box::new(public_key.clone())),
            balance: gas_key_info.balance,
        }
        .into());
        return Ok(());
    }
    result.tokens_burnt =
        result.tokens_burnt.checked_add(gas_key_info.balance).ok_or(IntegerOverflowError)?;
    let num_nonces = gas_key_info.num_nonces as usize;
    for i in 0..gas_key_info.num_nonces {
        remove_gas_key_nonce(state_update, account_id.clone(), public_key.clone(), i);
    }
    let nonce_key_len = gas_key_nonce_key_len(account_id, &public_key.into());
    let nonce_remove_compute = storage_removes_compute(
        &config.wasm_config.ext_costs,
        num_nonces,
        nonce_key_len * num_nonces,
        AccessKey::NONCE_VALUE_LEN * num_nonces,
    );
    result.compute_usage = safe_add_compute(result.compute_usage, nonce_remove_compute)?;
    remove_access_key(state_update, account_id.clone(), public_key.clone());
    account.set_storage_usage(account.storage_usage().saturating_sub(gas_key_storage_cost(
        &config.fees,
        public_key,
        access_key,
        gas_key_info.num_nonces,
    )));
    Ok(())
