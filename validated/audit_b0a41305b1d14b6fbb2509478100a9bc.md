### Title
Underestimated Minimum Gas for Declare Transactions Admits Insufficient-Resource-Bound Txs — (`File: crates/blockifier/src/fee/gas_usage.rs`)

### Summary

`estimate_minimal_gas_vector` omits `n_compiled_class_hash_updates: 1` from the `StateChangesCount` it builds for Declare (V2/V3) transactions. The missing component is the on-chain DA cost of writing the `(class_hash, compiled_class_hash)` pair, which every V2/V3 Declare transaction actually produces. Because the lower-bound estimate is too low, the pre-validation resource-bounds check accepts Declare transactions whose resource bounds are below the true minimum cost, causing them to fail during block execution with `InsufficientResourceBounds`.

---

### Finding Description

In `estimate_minimal_gas_vector` the `StateChangesCount` for a `Declare` transaction is:

```rust
// crates/blockifier/src/fee/gas_usage.rs  lines 168-174
Transaction::Declare(_) => StateChangesCount {
    n_storage_updates: 1,
    n_class_hash_updates: 0,
    // TODO(Yoni): BLOCKIFIER-RESET: should be 1.
    n_compiled_class_hash_updates: 0,   // ← BUG: should be 1
    n_modified_contracts: 1,
},
``` [1](#0-0) 

The TODO comment is the developer's own acknowledgement that this value is wrong. The correct value is confirmed by the test helper `declare_expected_state_changes_count`, which sets `n_compiled_class_hash_updates: 1` for V2/V3 Declare transactions:

```rust
// crates/blockifier/src/transaction/transactions_test.rs  lines 1846-1852
} else if version == TransactionVersion::TWO || version == TransactionVersion::THREE {
    StateChangesCount {
        n_storage_updates: 1,
        n_modified_contracts: 1,
        n_compiled_class_hash_updates: 1, // Also set compiled class hash.
        ..StateChangesCount::default()
    }
``` [2](#0-1) 

`get_onchain_data_segment_length` adds `n_compiled_class_hash_updates * 2` felts to the DA segment:

```rust
// crates/blockifier/src/fee/gas_usage.rs  lines 33-34
// For each compiled class updated (through declare): class_hash, compiled_class_hash
onchain_data_segment_length += state_changes_count.n_compiled_class_hash_updates * 2;
``` [3](#0-2) 

With `n_compiled_class_hash_updates = 0`, the DA segment length is underestimated by **2 felts**, and `get_da_gas_cost` therefore returns a gas vector that is too low by:

- **KZG/blob mode**: `2 × DATA_GAS_PER_FIELD_ELEMENT` blob-gas units
- **Calldata mode**: `2 × SHARP_GAS_PER_DA_WORD` L1-gas units (minus the standard discount) [4](#0-3) 

`estimate_minimal_gas_vector` feeds this underestimated vector into the pre-validation resource-bounds check (`check_fee_bounds` / `verify_can_pay_committed_bounds`), which is called during gateway stateful validation and at the start of block execution: [5](#0-4) 

---

### Impact Explanation

A user can craft a Declare V3 transaction whose `l1_data_gas` (blob mode) or `l1_gas` (calldata mode) resource bound is set to exactly the underestimated minimum — i.e., 2 blob-gas units (or the calldata equivalent) below the true minimum. The pre-validation check passes because the estimate is too low. The transaction is admitted to the mempool and forwarded to the batcher. During block execution the actual DA cost exceeds the resource bound, and the transaction fails with `InsufficientResourceBounds`. The sequencer has accepted and processed an invalid transaction, wasting block space and charging the sender a fee for a reverted transaction.

Impact category: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

---

### Likelihood Explanation

Any unprivileged user can submit a Declare V3 transaction. The window of "too-low but accepted" resource bounds is small (2 felts of DA), but it is deterministic and reproducible. A user who queries `starknet_estimateFee` and then subtracts exactly 2 blob-gas units from the returned `l1_data_gas` bound will reliably trigger this path. The bug is present in all production code paths that call `estimate_minimal_gas_vector` for Declare transactions.

---

### Recommendation

Change `n_compiled_class_hash_updates` from `0` to `1` in the `Declare` arm of `estimate_minimal_gas_vector`, as the TODO comment already prescribes:

```rust
// crates/blockifier/src/fee/gas_usage.rs
Transaction::Declare(_) => StateChangesCount {
    n_storage_updates: 1,
    n_class_hash_updates: 0,
    n_compiled_class_hash_updates: 1,  // Fix: was 0
    n_modified_contracts: 1,
},
``` [1](#0-0) 

After the fix, update the regression tests in `gas_usage_test.rs` and `transactions_test.rs` that snapshot the minimal gas vector for Declare transactions to reflect the corrected (higher) value.

### Proof of Concept

1. Obtain the true minimum DA gas for a Declare V3 transaction in KZG mode:
   - `n_compiled_class_hash_updates = 1` → segment length contribution = 2 felts
   - `true_min_blob_gas = 2 × DATA_GAS_PER_FIELD_ELEMENT`

2. Craft a Declare V3 transaction with:
   ```
   l1_data_gas.max_amount = true_min_blob_gas - 1
   ```
   (all other resource bounds set to their true minimums)

3. Submit to the gateway. The stateful validator calls `estimate_minimal_gas_vector`, which returns `n_compiled_class_hash_updates = 0`, so the estimated minimum blob gas is `0`. The check `max_amount >= estimated_min` passes.

4. The transaction is admitted to the mempool and reaches the batcher.

5. During block execution, the actual DA cost is `true_min_blob_gas`, which exceeds `max_amount`. The transaction fails with `InsufficientResourceBounds` — an invalid transaction was accepted and consumed sequencer resources. [5](#0-4) [6](#0-5) [4](#0-3)

pub fn get_onchain_data_segment_length(state_changes_count: &StateChangesCount) -> usize {
    // For each newly modified contract:
    // contract address (1 word).
    // + 1 word with the following info: A flag indicating whether the class hash was updated, the
    // number of entry updates, and the new nonce.
    let mut onchain_data_segment_length = state_changes_count.n_modified_contracts * 2;
    // For each class updated (through a deploy or a class replacement).
    onchain_data_segment_length +=
        state_changes_count.n_class_hash_updates * constants::CLASS_UPDATE_SIZE;
    // For each modified storage cell: key, new value.
    onchain_data_segment_length += state_changes_count.n_storage_updates * 2;
    // For each compiled class updated (through declare): class_hash, compiled_class_hash
    onchain_data_segment_length += state_changes_count.n_compiled_class_hash_updates * 2;

    onchain_data_segment_length
}

/// Returns the gas cost of data availability on L1.
pub fn get_da_gas_cost(state_changes_count: &StateChangesCount, use_kzg_da: bool) -> GasVector {
    let onchain_data_segment_length = get_onchain_data_segment_length(state_changes_count);

    let (l1_gas, blob_gas) = if use_kzg_da {
        (
            0_u8.into(),
            u64_from_usize(
                onchain_data_segment_length * eth_gas_constants::DATA_GAS_PER_FIELD_ELEMENT,
            )
            .into(),
        )
    } else {
        // TODO(Yoni, 1/5/2024): count the exact amount of nonzero bytes for each DA entry.
        let naive_cost = onchain_data_segment_length * eth_gas_constants::SHARP_GAS_PER_DA_WORD;

        // For each modified contract, the expected non-zeros bytes in the second word are:
        // 1 bytes for class hash flag; 2 for number of storage updates (up to 64K);
        // 3 for nonce update (up to 16M).
        let modified_contract_cost = eth_gas_constants::get_calldata_word_cost(1 + 2 + 3);
        let modified_contract_discount =
            eth_gas_constants::GAS_PER_MEMORY_WORD - modified_contract_cost;
        let mut discount = state_changes_count.n_modified_contracts * modified_contract_discount;

        // Up to balance of 8*(10**10) ETH.
        let fee_balance_value_cost = eth_gas_constants::get_calldata_word_cost(12);
        discount += eth_gas_constants::GAS_PER_MEMORY_WORD - fee_balance_value_cost;

        // Cost must be non-negative after discount.
        let gas = naive_cost.saturating_sub(discount);

        (u64_from_usize(gas).into(), 0_u8.into())
    };

    GasVector { l1_gas, l1_data_gas: blob_gas, ..Default::default() }
}

/// Returns an estimated lower bound for the gas required by the given account transaction.
pub fn estimate_minimal_gas_vector(
    block_context: &BlockContext,
    tx: &AccountTransaction,
    gas_usage_vector_computation_mode: &GasVectorComputationMode,
) -> GasVector {
    // TODO(Dori, 1/8/2023): Give names to the constant VM step estimates and regression-test them.
    let BlockContext { block_info, versioned_constants, .. } = block_context;
    let state_changes_by_account_tx = match &tx.tx {
        // We consider the following state changes: sender balance update (storage update) + nonce
        // increment (contract modification) (we exclude the sequencer balance update and the ERC20
        // contract modification since it occurs for every tx).
        Transaction::Declare(_) => StateChangesCount {
            n_storage_updates: 1,
            n_class_hash_updates: 0,
            // TODO(Yoni): BLOCKIFIER-RESET: should be 1.
            n_compiled_class_hash_updates: 0,
            n_modified_contracts: 1,
        },
        Transaction::Invoke(_) => StateChangesCount {
            n_storage_updates: 1,
            n_class_hash_updates: 0,
            n_compiled_class_hash_updates: 0,
            n_modified_contracts: 1,
        },
        // DeployAccount also updates the address -> class hash mapping.
        Transaction::DeployAccount(_) => StateChangesCount {
            n_storage_updates: 1,
            n_class_hash_updates: 1,
            n_compiled_class_hash_updates: 0,
            n_modified_contracts: 1,
        },
    };

    // TODO(Yoni): BLOCKIFIER-RESET: reuse TransactionReceipt code.
    let data_segment_length = get_onchain_data_segment_length(&state_changes_by_account_tx);
    let os_steps_for_type = versioned_constants
        .os_resources_for_tx_type(&tx.tx_type(), tx.extended_calldata_length())
        .n_steps
        + versioned_constants.os_kzg_da_resources(data_segment_length).n_steps;

    let resources = ExtendedExecutionResources {
        vm_resources: ExecutionResources { n_steps: os_steps_for_type, ..Default::default() },
        ..Default::default()
    };
    let da_gas_cost = get_da_gas_cost(&state_changes_by_account_tx, block_info.use_kzg_da);
    let vm_resources_cost = get_extended_vm_resources_cost(
        versioned_constants,
        &resources,
        0,
        gas_usage_vector_computation_mode,
    );
    da_gas_cost.checked_add(vm_resources_cost).unwrap_or_else(|| {
        panic!(
            "Overflow in minimal gas estimation; attempted to add {da_gas_cost:?} to \
             {vm_resources_cost:?}"
        )
    })
}
