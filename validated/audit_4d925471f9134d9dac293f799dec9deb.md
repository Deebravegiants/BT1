### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Redirect Sequencer Fees and Corrupt Block Header — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries a `builder` field (the sequencer/block-builder address). The validator's `is_proposal_init_valid` function checks `height`, `timestamp`, `starknet_version`, `version_constant_commitment`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, and `fee_proposal_fri` — but never checks `builder`. The proposer-supplied `builder` value is forwarded verbatim to the batcher as `BlockInfo.sequencer_address`, stored in the committed block header, and used by the blockifier to collect transaction fees. A malicious-but-legitimate proposer can set `builder` to any address, redirecting all block fees and corrupting the committed `sequencer_address` / block hash.

---

### Finding Description

`ProposalInit.builder` is defined as "Address of the one who builds/sequences the block."

**Proposer side** (`crates/apollo_consensus_orchestrator/src/build_proposal.rs`, line 174):
```rust
builder: args.builder_address,   // taken from static config
```
A TODO comment at line 806–807 of `sequencer_consensus_context.rs` explicitly acknowledges this is not yet committee-derived:
```rust
// TODO(Asmaa): Get it from committee once we have it.
builder_address: self.config.static_config.builder_address,
```

**Validator side** (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, lines 252–418): `is_proposal_init_valid` validates `height`, `timestamp`, `starknet_version`, `version_constant_commitment`, `l1_da_mode`, `l2_gas_price_fri`, four L1 gas price fields, and `fee_proposal_fri`. The `builder` field is never read or compared.

The unvalidated `builder` is then forwarded to the batcher via `convert_to_sn_api_block_info(&init)` (called at line 466 in `initiate_validation` and line 552 in `finalize_decision`), which maps `init.builder` → `BlockInfo.sequencer_address`.

At decision time (`sequencer_consensus_context.rs`, line 397):
```rust
let sequencer = SequencerContractAddress(init.builder);
let block_header_without_hash = BlockHeaderWithoutHash {
    ...
    sequencer,
    ...
};
```

Because the validator's batcher also receives the proposer-supplied `builder` via `ValidateBlockInput.block_info`, both sides compute the same `partial_block_hash` with the attacker-controlled `sequencer_address`. The `ProposalFinMismatch` guard (`built_block != received_fin.proposal_commitment`) therefore does **not** fire — both sides agree on the wrong value.

---

### Impact Explanation

1. **Fee theft (Critical — economic impact):** The blockifier pays transaction fees to `BlockInfo.sequencer_address`. A malicious proposer sets `builder` to an address they control; all fees for that block are transferred there instead of the legitimate sequencer.

2. **Wrong committed block header / block hash (Critical — state commitment):** `sequencer_address` is a component of the Starknet block header hash. A forged `builder` produces a block hash that does not correspond to the legitimate sequencer, corrupting the L1-anchored state commitment for that height.

---

### Likelihood Explanation

Any validator that is legitimately selected as proposer for a round can exploit this. No external attacker capability is required beyond being a committee member. The validator nodes will unanimously accept the proposal because `builder` is not part of any validation criterion.

---

### Recommendation

Add a check inside `is_proposal_init_valid` that rejects any `ProposalInit` whose `builder` field does not match the locally-configured (or committee-derived) expected builder address, mirroring the pattern already used for `version_constant_commitment`:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder),
    ));
}
```

`ProposalInitValidation` should carry `expected_builder`, populated from `self.config.static_config.builder_address` (or the committee once available, per the existing TODO).

---

### Proof of Concept

1. A legitimate validator is selected as proposer for height H.
2. The validator modifies `builder_address` in its static config (or patches `initiate_build`) to set `ProposalInit.builder` to an attacker-controlled address `ATTACKER_ADDR`.
3. The proposal is broadcast. Every other validator calls `is_proposal_init_valid`; `builder` is never checked; validation passes.
4. `initiate_validation` calls `convert_to_sn_api_block_info(&init)`, producing `BlockInfo { sequencer_address: ATTACKER_ADDR, ... }`.
5. The batcher executes the block with `sequencer_address = ATTACKER_ADDR`; all fee-transfer syscalls credit `ATTACKER_ADDR`.
6. `finish_proposal` returns `partial_block_hash` computed over the execution with `ATTACKER_ADDR`. `proposal_commitment_from(partial_block_hash, fee_proposal)` matches the proposer's `ProposalFin.proposal_commitment`. The `ProposalFinMismatch` guard passes.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
