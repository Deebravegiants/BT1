Audit Report

## Title
Missing Staleness Check on `CrossChainRateReceiver.getRate()` Causes Over-Minting of agETH in `AGETHPoolV3.deposit()` — (`contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary

`CrossChainRateReceiver` stores `lastUpdated` when a rate is received via LayerZero but never checks it in `getRate()`. Because `updateRate()` on `AGETHMultiChainRateProvider` is permissionless and requires the caller to supply ETH for LayerZero fees, the L2 receiver can hold a stale (lower) rate indefinitely. Any user calling `AGETHPoolV3.deposit()` during a staleness window receives more agETH than the current backing rate justifies, diluting the backing ratio for all existing agETH holders.

## Finding Description

The full rate propagation chain is confirmed in the code:

1. `AGETHMultiChainRateProvider.getLatestRate()` reads the live rate from `agETHPriceOracle`. [1](#0-0) 

2. `MultiChainRateProvider.updateRate()` is `external payable` with no access control — anyone can call it, but must supply ETH for LayerZero fees. It pushes the rate to all registered `CrossChainRateReceiver` contracts. [2](#0-1) 

3. `CrossChainRateReceiver.lzReceive()` stores the received value in `rate` and records `lastUpdated`. [3](#0-2) 

4. `CrossChainRateReceiver.getRate()` returns `rate` with **no freshness check**. `lastUpdated` is stored but never validated against any maximum age. [4](#0-3) 

5. `AGETHPoolV3.getRate()` delegates directly to `IOracle(agETHOracle).getRate()`, which resolves to the stale receiver value. [5](#0-4) 

6. `viewSwapAgETHAmountAndFee` computes `agETHAmount = amountAfterFee * 1e18 / agETHToETHrate` using that stale rate, and `deposit()` mints the result unconditionally. [6](#0-5) 

**Arithmetic:** If the live rate has moved from `R` to `R * 1.1` but the receiver still holds `R`, the deposit mints `amountAfterFee * 1e18 / R` instead of `amountAfterFee * 1e18 / (R * 1.1)` — approximately 9.09% more agETH than the deposited ETH justifies. No guard in either `AGETHPoolV3` or `CrossChainRateReceiver` prevents this.

## Impact Explanation

Users depositing ETH into `AGETHPoolV3` during a staleness window receive more agETH than the current backing rate justifies. The deposited ETH is retained by the pool, but the excess agETH minted dilutes the backing ratio for all existing agETH holders. This matches **Low: Contract fails to deliver promised returns, but doesn't lose value** — the protocol's exchange-rate invariant is violated without direct ETH loss.

## Likelihood Explanation

`updateRate()` requires the caller to pay LayerZero cross-chain fees in ETH. There is no on-chain mechanism that enforces a maximum staleness period or incentivizes timely updates. Operational delays, LayerZero congestion, or simply no one paying the fee creates a realistic and repeatable staleness window. Because agETH accrues staking rewards continuously, any non-trivial delay produces a measurable rate divergence. The condition is triggered by normal user deposits — no privileged access or special attacker capability is required.

## Recommendation

Add a maximum staleness check in `CrossChainRateReceiver.getRate()`:

```solidity
uint256 public constant MAX_RATE_AGE = 24 hours;

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= MAX_RATE_AGE, "Rate is stale");
    return rate;
}
```

This causes `AGETHPoolV3.deposit()` to revert rather than mint at a stale rate, protecting the backing invariant. Alternatively, the check can be placed in `AGETHPoolV3.getRate()` to scope the protection to the pool without affecting other consumers of the receiver.

## Proof of Concept

```solidity
// Local fork test — no public mainnet testing
function testStalePriceMinting() public {
    // T=0: push rate R to receiver via LayerZero mock
    mockRateProvider.setRate(1.05e18);
    provider.updateRate{value: lzFee}();
    // receiver.rate == 1.05e18, lastUpdated == block.timestamp

    // Advance time; agETH appreciates on L1
    vm.warp(block.timestamp + 2 days);
    mockRateProvider.setRate(1.155e18); // R * 1.1

    // updateRate() is NOT called — receiver.rate still == 1.05e18

    uint256 balBefore = agETH.balanceOf(user);
    vm.prank(user);
    pool.deposit{value: 1 ether}("ref");
    uint256 minted = agETH.balanceOf(user) - balBefore;

    uint256 correctAmount = 1 ether * 1e18 / 1.155e18; // ~0.8658 agETH
    uint256 staleAmount   = 1 ether * 1e18 / 1.05e18;  // ~0.9524 agETH

    assertApproxEqRel(minted, staleAmount, 1e15);
    assertGt(minted, correctAmount); // ~9.1% over-mint confirmed
}
```
