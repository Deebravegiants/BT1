### Title
Unbounded Gas Consumption in Public `updateRSETHPrice()` via Deeply Nested Cross-Contract Loops — (File: contracts/LRTOracle.sol)

---

### Summary

`LRTOracle.updateRSETHPrice()` is an unrestricted public function that triggers a deeply nested, multi-contract loop chain whose gas cost scales with the number of supported assets, node delegators, and queued EigenLayer withdrawals. Any unprivileged caller can invoke it repeatedly at minimal cost, consuming block gas. As the protocol grows, the function can exceed the block gas limit and become permanently uncallable, freezing the price oracle.

---

### Finding Description

`updateRSETHPrice()` carries only a `whenNotPaused` guard — no role check, no fee, no rate limit: [1](#0-0) 

It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which iterates over every supported asset: [2](#0-1) 

For each asset it calls `ILRTDepositPool.getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)`, which loops over every entry in `nodeDelegatorQueue` and, for each NDC, calls `getAssetUnstaking(asset)`: [3](#0-2) 

`NodeDelegator.getAssetUnstaking()` fetches **all** queued EigenLayer withdrawals for that NDC and then iterates over every strategy inside each withdrawal: [4](#0-3) 

The total computation is therefore:

```
O(assets × NDCs × queued_withdrawals_per_NDC × strategies_per_withdrawal)
```

All four dimensions grow with normal protocol operation (more LSTs added, more NDCs deployed, more EigenLayer unstaking rounds queued). There is no cap on queued withdrawals per NDC.

---

### Impact Explanation

**Medium — Unbounded gas consumption / temporary freeze of price updates.**

1. **Repeated griefing**: Any address can call `updateRSETHPrice()` in a tight loop, filling blocks with expensive computation at the cost of only gas, exhausting block capacity for legitimate transactions.
2. **Permanent DoS of price oracle**: Once the accumulated protocol state causes the function to exceed the block gas limit, `updateRSETHPrice()` becomes permanently uncallable. The stored `rsETHPrice` becomes stale. Downstream functions that read `lrtOracle.rsETHPrice()` — including `unlockQueue` in `LRTWithdrawalManager` and `getRsETHAmountToMint` in `LRTDepositPool` — will operate on an outdated rate, leading to mispriced deposits and withdrawals (temporary freeze / incorrect accounting). [1](#0-0) 

---

### Likelihood Explanation

**Medium.** The function is unconditionally public. No ETH deposit, no role, and no rate limit is required to call it. The protocol is designed to support multiple LSTs and multiple NDCs, and EigenLayer unstaking rounds accumulate naturally during normal operation. The gas cost therefore grows deterministically with protocol usage, not with any special attacker action.

---

### Recommendation

1. **Short term**: Add an access-control modifier (e.g., `onlyLRTOperator` or `onlyLRTManager`) to `updateRSETHPrice()` so that only trusted callers can trigger the expensive computation path.
2. **Long term**: Refactor `_getTotalEthInProtocol()` to avoid the nested cross-contract loop. Cache per-asset TVL snapshots that are updated incrementally (on deposit/withdrawal events) rather than recomputed in full on every price update. Alternatively, bound the number of queued withdrawals that `getAssetUnstaking()` iterates over, or move the EigenLayer withdrawal accounting off-chain with on-chain attestation.

### Proof of Concept

```
Attacker (EOA)
  │
  └─► LRTOracle.updateRSETHPrice()          [public, no guard]
        └─► _getTotalEthInProtocol()
              └─► for each asset (N assets):
                    └─► LRTDepositPool.getTotalAssetDeposits(asset)
                          └─► getAssetDistributionData(asset)
                                └─► for each NDC (M NDCs):
                                      └─► NodeDelegator.getAssetUnstaking(asset)
                                            └─► DelegationManager.getQueuedWithdrawals()
                                                  └─► for each withdrawal (W):
                                                        └─► for each strategy (S):
                                                              [external call + computation]
```

Total external calls per invocation: `N × M × (1 + W × S)`.

With, for example, 5 assets, 10 NDCs, and 20 queued withdrawals of 2 strategies each, a single call executes ≈ 5 × 10 × (1 + 20 × 2) = **2,050 external calls**. An attacker submitting this repeatedly in the same block saturates block gas at negligible personal cost beyond the gas fee itself.

        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)

            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
