Audit Report

## Title
`highestRsethPrice` Never Reset on Unpause Causes Repeated Auto-Pause Loop — (`contracts/LRTOracle.sol`)

## Summary
When `_updateRsETHPrice()` triggers the downside auto-pause, it returns early at line 281 without updating `rsETHPrice` or `highestRsethPrice`. After an admin calls `unpause()`, the stale `highestRsethPrice` remains unchanged, so any subsequent public call to `updateRSETHPrice()` re-triggers the auto-pause if the price has only partially recovered. This creates a loop that admin `unpause()` alone cannot break, temporarily freezing deposits and withdrawals.

## Finding Description

**Auto-pause path (lines 270–281):** [1](#0-0) 

When `isPriceDecreaseOffLimit` is true, the function pauses `lrtDepositPool`, `withdrawalManager`, and `LRTOracle` via `_pause()`, then returns early. Neither `rsETHPrice` nor `highestRsethPrice` is written.

**`unpause()` (lines 143–146):** [2](#0-1) 

`unpause()` only sets `paused = false`. It does not reset `highestRsethPrice`, does not unpause `lrtDepositPool` or `withdrawalManager`, and provides no mechanism to adjust the peak price reference.

**`highestRsethPrice` update path (lines 294–296):** [3](#0-2) 

`highestRsethPrice` is only ever updated upward. There is no admin function to lower or reset it independently.

**Re-pause condition (lines 273–274):** [4](#0-3) 

The condition `diff > pricePercentageLimit.mulWad(highestRsethPrice)` uses the same stale `highestRsethPrice` on every call after unpause.

**`updateRSETHPriceAsManager()` does not help (lines 94–96):** [5](#0-4) 

The manager-only bypass skips only the upside revert at line 264. The downside auto-pause block at lines 277–281 has no role check and fires for any caller, including the manager. [6](#0-5) 

**Exploit flow:**
1. Price drops → `newRsETHPrice` falls below `highestRsethPrice` by more than `pricePercentageLimit` → auto-pause fires, all three contracts paused, function returns early.
2. Admin calls `unpause()` on all three contracts. `highestRsethPrice` is unchanged.
3. Price partially recovers but remains more than `pricePercentageLimit` below `highestRsethPrice`.
4. Any unprivileged user calls `updateRSETHPrice()` (public, `whenNotPaused` now passes) → auto-pause fires again immediately.
5. Loop repeats from step 2 indefinitely.

The only admin escape hatches are calling `setPricePercentageLimit(0)` to disable the protection entirely, or waiting for the price to fully recover above `highestRsethPrice`.

## Impact Explanation

**Medium — Temporary freezing of funds.** Deposits and withdrawals are blocked for the duration of the loop. `lrtDepositPool` and `withdrawalManager` are paused by the auto-pause trigger and are not unpaused by `LRTOracle.unpause()`. Even after the admin unpauses all three contracts individually, the next public `updateRSETHPrice()` call re-pauses them if the price is still below `highestRsethPrice` by more than `pricePercentageLimit`. User funds are inaccessible until either the price fully recovers above `highestRsethPrice` or the admin disables the price limit entirely.

## Likelihood Explanation

This is realistic in any scenario where a price drop triggers the auto-pause but the price only partially recovers — a common real-world scenario for LST price fluctuations. No privileged access is required to trigger the re-pause: any unprivileged external user calling the public `updateRSETHPrice()` function after admin unpause is sufficient. The condition is deterministic and repeatable as long as the price remains below the threshold.

## Recommendation

Reset `highestRsethPrice` to the current `rsETHPrice` inside `unpause()`, so that the post-unpause price baseline reflects the actual current price rather than the pre-crash peak:

```solidity
function unpause() external whenPaused onlyLRTAdmin {
    paused = false;
    highestRsethPrice = rsETHPrice; // reset peak to current price
    emit Unpaused(msg.sender);
}
```

Alternatively, add a dedicated admin function `resetHighestRsethPrice()` to allow explicit resetting without coupling it to the unpause flow.

## Proof of Concept

**State-test sequence:**

```
Setup:
  highestRsethPrice    = 1.05e18
  rsETHPrice           = 1.05e18
  pricePercentageLimit = 1e16  (1%)

Step 1 — Trigger auto-pause:
  Underlying oracle drops → newRsETHPrice = 1.02e18
  diff = 0.03e18 > 0.01 * 1.05e18 = 0.0105e18 → isPriceDecreaseOffLimit = true
  → lrtDepositPool.paused = true, withdrawalManager.paused = true, LRTOracle.paused = true
  → return (rsETHPrice stays 1.05e18, highestRsethPrice stays 1.05e18)

Step 2 — Admin unpauses all three contracts:
  LRTOracle.unpause(), lrtDepositPool.unpause(), withdrawalManager.unpause()
  highestRsethPrice → still 1.05e18 (unchanged)

Step 3 — Price partially recovers:
  newRsETHPrice = 1.035e18
  diff = 1.05e18 - 1.035e18 = 0.015e18
  threshold = 0.01 * 1.05e18 = 0.0105e18
  0.015e18 > 0.0105e18 → isPriceDecreaseOffLimit = true

Step 4 — Unprivileged user calls updateRSETHPrice():
  whenNotPaused passes (paused = false after step 2)
  → auto-pause fires again at lines 277-281
  → lrtDepositPool.paused = true, withdrawalManager.paused = true, LRTOracle.paused = true

Assert: protocol is paused again despite admin unpause and partial price recovery.
Loop: repeat from Step 2 indefinitely until price > 1.05e18 or admin calls setPricePercentageLimit(0).
```
