### Title
Unrestricted `claim` Caller Allows Any Address to Force Fee-Bearing Claims on Behalf of Any User - (File: `contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

---

### Summary

`MerkleDistributor.claim` accepts an arbitrary `account` parameter with no restriction that `account == msg.sender`. Any unprivileged caller can supply a valid merkle proof and trigger a claim for any victim, causing the victim's tokens to be transferred to them minus a protocol fee of up to 10%. The victim permanently loses that fee with no recourse.

---

### Finding Description

The `claim` function in `MerkleDistributor` takes `account` as a caller-supplied parameter and performs no check that `msg.sender == account`:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
    // No check: account == msg.sender
    ...
    uint256 fee = (claimableAmount * feeInBPS) / 10_000;
    uint256 amountToSend = claimableAmount - fee;

    IERC20(token).safeTransfer(account, amountToSend);
    IERC20(token).safeTransfer(protocolTreasury, fee);
}
``` [1](#0-0) 

The merkle proof is public on-chain (or derivable from the merkle tree data published off-chain). An attacker can reconstruct any user's valid `(index, account, cumulativeAmount, merkleProof)` tuple and call `claim` on their behalf at any time. The contract deducts `feeInBPS` (up to `MAX_FEE_IN_BPS = 1000`, i.e. 10%) from the claimable amount before sending to `account`. [2](#0-1) [3](#0-2) 

By contrast, `KernelMerkleDistributor._processClaim` correctly enforces `account == msg.sender`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [4](#0-3) 

`MerkleDistributor` is missing this guard entirely.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

An attacker forces a claim for any victim at any time. The victim receives `claimableAmount * (1 - feeInBPS/10000)` tokens instead of the full `claimableAmount`. The fee (up to 10%) is permanently redirected to `protocolTreasury`. The victim cannot undo the claim (`isClaimed` returns true afterward) and cannot recover the lost fee. This constitutes direct, irreversible theft of unclaimed yield from any user whose merkle data is publicly available.

---

### Likelihood Explanation

**High.** The merkle tree data (index, account, cumulativeAmount, proof) is necessarily published off-chain for users to claim. Any observer can reconstruct valid call parameters for any user. No special privilege, capital, or timing is required. The attacker pays only gas. The attack is profitable whenever `feeInBPS > 0` and can be automated to target all users in a single block.

---

### Recommendation

Add a caller restriction identical to the one in `KernelMerkleDistributor`:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
    if (account != msg.sender) revert Unauthorized();
    // ... rest of logic
}
```

This ensures only the rightful owner can trigger their own claim and decide when to incur the fee.

---

### Proof of Concept

1. Protocol publishes merkle tree. Alice's leaf: `(index=5, account=Alice, cumulativeAmount=1000e18)`.
2. Attacker reconstructs Alice's merkle proof from the published tree.
3. Attacker calls `MerkleDistributor.claim(5, Alice, 1000e18, aliceProof)`.
4. Contract verifies proof (valid), computes `fee = 1000e18 * feeInBPS / 10000`.
5. Alice receives `1000e18 - fee`; `fee` goes to `protocolTreasury`.
6. `userClaims[Alice].lastClaimedIndex = 5` — Alice's claim is permanently marked as used.
7. Alice calls `claim` herself later → reverts with `AlreadyClaimed`.
8. Alice has permanently lost `fee` tokens she was entitled to.

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
