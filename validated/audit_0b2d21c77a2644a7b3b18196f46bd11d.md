### Title
`is_caller_an_attested_participant` Does Not Re-Verify Attestation Validity, Allowing Expired/Revoked TEE Nodes to Submit Threshold Signatures - (`File: crates/contract/src/tee/tee_state.rs`)

---

### Summary

The `is_caller_an_attested_participant` function, used to gate every `respond*` and key-event voting call, only checks that a stored attestation **exists** and that the account/key identity matches. It never calls `reverify_participants` to check whether the stored attestation is still **valid** (not expired, docker-image hash still whitelisted, launcher hash still allowed, OS measurements still accepted). The batch re-verification path (`reverify_and_cleanup_participants`, called by `verify_tee`) applies all of those validity checks. The two paths are therefore inconsistent: the per-call gate accepts a node whose attestation has since expired or been revoked, while the batch path would correctly reject it.

---

### Finding Description

`is_caller_an_attested_participant` (tee_state.rs lines 469–498) performs only identity checks:

1. Caller is in the current participant set.
2. A `NodeAttestation` entry exists for the caller's TLS key.
3. The stored `account_id` matches `signer_account_id`.
4. The stored `account_public_key` matches `signer_account_pk`. [1](#0-0) 

It never calls `reverify_participants`, which performs the full re-verification: [2](#0-1) 

`reverify_participants` checks expiry timestamp, docker-image hash whitelist, launcher compose hash whitelist, and OS measurement whitelist — none of which `is_caller_an_attested_participant` touches.

`assert_caller_is_attested_participant_and_protocol_active` (called in `respond`, `respond_ckd`, `respond_verify_foreign_tx`, `start_keygen_instance`, `vote_pk`, `start_reshare_instance`, `vote_reshared`, `vote_abort_key_event_instance`) delegates to `is_caller_an_attested_participant`. [3](#0-2) [4](#0-3) [5](#0-4) 

The contract-wide `accept_requests` flag is set by `verify_tee`, which calls `reverify_and_cleanup_participants`. However, `accept_requests` is a **global** flag, not per-node. When `verify_tee` finds that some (but not all) participants have invalid attestations and the threshold is still met, it sets `accept_requests = true` and initiates resharing to eventually remove the invalid nodes. [6](#0-5) 

During that resharing window — and in any window before `verify_tee` is called — a node whose attestation has expired or whose docker-image hash has been revoked can still call `respond*` and have its signature accepted, because `is_caller_an_attested_participant` never re-verifies the stored attestation.

---

### Impact Explanation

The TEE attestation is the contract's mechanism for ensuring that every signing participant is running verified, unmodified code inside a trusted execution environment. A node whose attestation has expired or whose image hash has been revoked is no longer verified to be running trusted code. Allowing such a node to call `respond`, `respond_ckd`, or `respond_verify_foreign_tx` means threshold signatures can be produced with participation from a node that the contract's own validity logic (`reverify_participants`) would reject. This breaks the production safety invariant that only currently-attested TEE nodes may contribute to signing, and constitutes participant-state/contract-execution-flow manipulation that undermines the TEE security model without requiring network-level DoS or operator misconfiguration.

**Allowed impact matched:** Medium — participant-state and contract execution-flow manipulation that breaks production safety/accounting invariants.

---

### Likelihood Explanation

TEE attestations expire on a schedule (controlled by the attestation's `expiry_timestamp_seconds`). Docker-image hashes are revoked via governance votes (`vote_code_hash`, `vote_remove_launcher_hash`). In both cases there is a window — between the expiry/revocation event and the next successful `verify_tee` call that completes resharing — during which the affected node retains full `respond*` access. `verify_tee` is called by participants, not automatically, so the window can be arbitrarily long. The entry path is a direct, unprivileged call to `respond` from the affected node's NEAR account.

---

### Recommendation

`is_caller_an_attested_participant` should call `reverify_participants` on the stored attestation before returning `Ok(())`, mirroring the logic already present in `reverify_and_cleanup_participants`. This ensures that the per-call gate and the batch re-verification path apply identical validity criteria, eliminating the inconsistency.

Alternatively, `assert_caller_is_attested_participant_and_protocol_active` can be extended to call `reverify_participants` inline, so that every `respond*` and key-event call re-checks attestation validity at the point of use.

---

### Proof of Concept

1. The contract is `Running` with nodes A, B, C (threshold = 2). All have valid attestations. `accept_requests = true`.
2. Node A's attestation expires (or its docker-image hash is removed via `vote_remove_launcher_hash` reaching the all-participant threshold).
3. A user calls `sign(...)`. The contract enqueues a yield.
4. `verify_tee` is called. It finds Node A's attestation invalid. Since threshold (2) is still met with B and C, it sets `accept_requests = true` and initiates resharing.
5. **Before resharing completes**, Node A calls `respond(request, signature)`.
6. `assert_caller_is_attested_participant_and_protocol_active` calls `is_caller_an_attested_participant`. Node A is still in the participant set (resharing has not concluded), its `NodeAttestation` entry still exists, and the account/key identity matches. The function returns `Ok(())`.
7. `is_running_or_resharing()` returns `true` (resharing is active). `accept_requests` is `true`.
8. The contract accepts Node A's signature and resolves the yield — despite Node A's attestation being invalid under `reverify_participants`. [1](#0-0) [2](#0-1) [7](#0-6)

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
