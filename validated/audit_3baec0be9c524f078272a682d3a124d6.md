### Title
Missing Zero-Address Validation for `ProposalInit.builder` in Proposal Validation Allows Fee Misdirection — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates timestamp, starknet version, gas prices, and L1 DA mode from a received `ProposalInit`, but performs **no validation whatsoever on `init.builder`** — the field that becomes `sequencer_address` (fee recipient) for the entire block. A malicious proposer can set `builder` to `ContractAddress(0)` or any arbitrary address; every validator will accept the proposal, and all transaction fees for that block will be transferred to the wrong address.

### Finding Description

`ProposalInit.builder` is documented as "Address of the one who builds/sequences the block." [1](#0-0) 

`convert_to_sn_api_block_info` maps it directly to `sequencer_address` in the `BlockInfo` that is handed to the blockifier for every transaction in the block: [2](#0-1) 

`execute_fee_transfer` then uses `block_context.block_info.sequencer_address` as the ERC-20 transfer recipient for every fee charge: [3](#0-2) 

`is_proposal_init_valid` — the sole gate that a validator runs before accepting a proposal — checks height, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices (within margin), `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`. It never inspects `init.builder`: [4](#0-3) 

Gas prices receive an explicit non-zero guard via `NonzeroGasPrice::new()` inside `convert_to_sn_api_block_info`: [5](#0-4) 

No equivalent guard exists for `builder`. The protobuf deserializer only checks that the field is present (`ok_or(missing("builder"))`), not that it is non-zero: [6](#0-5) 

`ContractAddress::validate()` defines the valid range as `(0x1, upper_bound)`, explicitly excluding zero: [7](#0-6) 

but `validate()` is never called on `init.builder` anywhere in the proposal validation path.

### Impact Explanation

Every transaction fee in the affected block is transferred to the address the proposer placed in `builder`. If that address is `ContractAddress(0)`, fees are burned. If it is an attacker-controlled address, fees are stolen. Because `is_proposal_init_valid` is the only check validators run before committing to a proposal, all honest validators will accept and finalize the block, permanently recording the wrong `sequencer_address` in the committed `BlockHeaderWithoutHash`: [8](#0-7) 

This matches the **Critical** impact category: *Incorrect fee/balance effect with economic impact*.

### Likelihood Explanation

Any consensus participant whose turn it is to propose can trigger this. In a decentralized validator set, proposer rotation is unprivileged — no special key or admin role is required beyond being a scheduled proposer. The attack is a single-field substitution in the outbound `ProposalInit` message and requires no cooperation from other nodes.

### Recommendation

Add an explicit non-zero (and optionally range) check on `init.builder` inside `is_proposal_init_valid`, mirroring the `NonzeroGasPrice` guard already applied to every gas-price field:

```rust
// In is_proposal_init_valid, before the gas-price checks:
if init_proposed.builder == ContractAddress::default() {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        "builder address must not be zero".to_string(),
    ));
}
// Optionally also call init_proposed.builder.validate() to enforce the
// [0x2, L2_ADDRESS_UPPER_BOUND) range already defined in starknet_api.
```

### Proof of Concept

1. A malicious proposer overrides `args.builder_address` (or patches the outbound `ProposalInit`) to `ContractAddress::default()` before calling `build_proposal`.
2. The `ProposalInit` with `builder = 0x0` is streamed to all validators.
3. Each validator calls `is_proposal_init_valid` — no check on `builder` exists, so the function returns `Ok(())`.
4. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which sets `sequencer_address = ContractAddress(0)` in the `BlockInfo` passed to the batcher.
5. For every transaction in the block, `execute_fee_transfer` issues an ERC-20 `transfer(recipient=0x0, …)`.
6. Consensus reaches decision; `finalize_decision` commits the block with `sequencer = SequencerContractAddress(init.builder) = 0x0`.
7. All fees for the block are permanently lost (burned to address 0) with no on-chain recourse. [9](#0-8) [10](#0-9) [11](#0-10)

async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
