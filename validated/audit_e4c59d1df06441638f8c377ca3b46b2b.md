### Title
Unvalidated `builder` Field in `ProposalInit` Allows Malicious Proposer to Redirect Sequencer Fees and Corrupt Block State — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates most fields of the network-supplied `ProposalInit` but silently accepts the `builder` field without any check. `builder` is directly mapped to `sequencer_address` in `convert_to_sn_api_block_info` and passed to the batcher for block execution. A malicious-but-authenticated proposer can set `builder` to an arbitrary address; the validator re-executes the block with that address as the sequencer, both sides compute the same `partial_block_hash`, the `ProposalFinMismatch` guard passes, and the block is committed with the wrong sequencer address — redirecting all transaction fees and corrupting the `get_execution_info` syscall result for every transaction in the block.

---

### Finding Description

`ProposalInit` carries a `builder` field (the block-builder / fee-recipient address): [1](#0-0) 

`convert_to_sn_api_block_info` maps it directly to `sequencer_address` in the `BlockInfo` that is handed to the batcher: [2](#0-1) 

`is_proposal_init_valid` validates `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, and `fee_proposal_fri`. The `builder` field is never mentioned: [3](#0-2) 

`ProposalInitValidation` — the struct that carries the validator's reference values — has no `builder` slot, so there is no reference value to compare against even if a check were added: [4](#0-3) 

The only downstream guard is the `ProposalFinMismatch` check: [5](#0-4) 

This check compares `batcher_block_commitment` (computed by the validator's own batcher) against `received_fin.proposal_commitment` (sent by the proposer). Because the validator feeds the proposer-supplied `builder` value into its own batcher call via `initiate_validation` → `convert_to_sn_api_block_info`, both sides execute with the same (attacker-chosen) `sequencer_address` and produce the same `partial_block_hash`. The guard therefore passes unconditionally regardless of what `builder` contains: [6](#0-5) 

The `proposal_commitment_from` function hashes only `partial_block_hash` and `fee_proposal_fri`; it does not independently bind `builder`: [7](#0-6) 

---

### Impact Explanation

**Fee redirection (Critical — economic impact).** Every transaction fee in the block is paid to `sequencer_address`. With `builder` set to an attacker-controlled address, all fees for the committed block flow to that address instead of the legitimate sequencer.

**Wrong `get_execution_info` syscall result (Critical — wrong execution result).** Contracts that call `get_execution_info` receive `sequencer_address = attacker_address`. Any contract that uses the sequencer address for access control, routing, or accounting will behave incorrectly for every transaction in the block.

**Wrong block header commitment (Critical — wrong state commitment).** The Starknet block hash includes the sequencer address. The committed block header permanently records the attacker's address as the sequencer, corrupting the canonical chain state visible to L1 and to any verifier.

---

### Likelihood Explanation

The attack requires the node to be the elected proposer for a round, which is a normal, rotating role in the Tendermint-based consensus. Any validator node that becomes proposer can trigger this. No special privilege beyond being an active consensus participant is needed. In a network with `f` Byzantine validators out of `3f+1`, any one of them can be the proposer for some round.

---

### Recommendation

1. Add an `expected_builder: ContractAddress` field to `ProposalInitValidation`, populated from the node's own configured `builder_address` (or from a committee-level mapping of proposer → builder).
2. In `is_proposal_init_valid`, assert `init_proposed.builder == proposal_init_validation.expected_builder` and return `InvalidProposalInit` on mismatch.
3. If the protocol intentionally allows the proposer to choose an arbitrary builder (MEV-relay style), document this explicitly and ensure the economic model accounts for it; otherwise the check above is mandatory.

---

### Proof of Concept

```
1. Malicious node is elected proposer for round R at height H.

2. In initiate_build, the node constructs ProposalInit with:
     builder = <attacker_wallet_address>   // arbitrary, not the node's own address
     (all other fields are valid and pass is_proposal_init_valid)

3. The node's own batcher executes the block with
     sequencer_address = attacker_wallet_address
   and returns partial_block_hash_A.

4. The proposer streams ProposalPart::Init (with builder=attacker) and
   ProposalPart::Fin (with proposal_commitment = Poseidon(partial_block_hash_A,
   fee_proposal_fri)).

5. Each validator receives the Init, calls is_proposal_init_valid — builder is
   never checked — then calls initiate_validation → convert_to_sn_api_block_info,
   which sets sequencer_address = attacker_wallet_address in the ValidateBlockInput.

6. Each validator's batcher re-executes the same transactions with
   sequencer_address = attacker_wallet_address and produces the same
   partial_block_hash_A.

7. proposal_commitment_from(partial_block_hash_A, fee_proposal_fri) matches
   received_fin.proposal_commitment → ProposalFinMismatch check passes.

8. Consensus reaches quorum; decision_reached commits the block.
   The committed block header contains sequencer_address = attacker_wallet_address.
   All transaction fees for height H are credited to attacker_wallet_address.
   Every get_execution_info syscall in the block returned attacker_wallet_address
   as the sequencer.
``` [8](#0-7) [9](#0-8) [6](#0-5)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
