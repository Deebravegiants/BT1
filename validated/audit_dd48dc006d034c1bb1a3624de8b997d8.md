### Title
Precommit Vote Signature Does Not Bind `chain_id`, `height`, or `round` — (`File: crates/apollo_signature_manager/src/signature_manager.rs`)

### Summary

`build_precommit_vote_message_digest` constructs the signed payload for consensus precommit votes using only a domain separator and the `block_hash`. It omits `chain_id`, `height`, `round`, and `voter`. This is the direct Sequencer analog of the external bug: a signed message that lacks sufficient context to prevent cross-context replay.

### Finding Description

`sign_precommit_vote` calls `build_precommit_vote_message_digest`, which constructs the signed message as:

```
blake2s(b"PRECOMMIT_VOTE" || block_hash.to_bytes_be())
``` [1](#0-0) 

The `Vote` struct that carries this signature over the network contains `height`, `round`, `voter`, and `proposal_commitment` as separate, unsigned fields: [2](#0-1) 

None of those fields are covered by the signature. The full block hash itself is computed as:

```
Poseidon(BLOCK_HASH_VERSION, block_number, state_root, sequencer_address, block_timestamp, …)
``` [3](#0-2) 

`chain_id` is absent from `calculate_block_hash` and from `PartialBlockHashComponents`: [4](#0-3) 

Because `chain_id` is not in the block hash and not in the signed message, a precommit signature produced on one chain is cryptographically valid on any other chain that happens to produce the same block hash value. Additionally, because `height` and `round` are not signed, a captured `Vote` can have those fields replaced while the signature remains valid against the same `block_hash`.

The verification counterpart mirrors the same omission: [5](#0-4) 

### Impact Explanation

**Impact: High — Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.**

A validator's precommit signature is not bound to a specific chain, height, or round. An attacker who observes a valid precommit `(validator V, block_hash X, height H, round R)` on chain A can:

1. Reuse the same `(signature, voter)` pair in a crafted `Vote` with a different `height` or `round` on chain A (if the same `block_hash` reappears — structurally possible since `block_hash` does not include `chain_id`).
2. Replay the signature verbatim on chain B (testnet/devnet) if the same `block_hash` value is produced there, making it appear that validator V precommitted for a block on chain B.

Either path allows forged consensus votes to be injected, potentially contributing to a false quorum or corrupting the consensus decision at a given height.

### Likelihood Explanation

**Medium.** The `block_hash` does include `block_number`, `state_root`, `sequencer_address`, and `timestamp`, making accidental collision within a single chain very unlikely. However, the structural absence of `chain_id` from both the block hash and the signed message is a direct, reachable code path with no compensating guard. Any deployment that runs the same validator key on multiple networks (e.g., a validator participating in both a testnet and a staging network) is immediately exposed to cross-chain replay.

### Recommendation

Include `chain_id`, `height`, and `round` in the precommit vote message digest:

```rust
fn build_precommit_vote_message_digest(
    block_hash: BlockHash,
    chain_id: &ChainId,
    height: BlockNumber,
    round: Round,
) -> MessageDigest {
    let mut message = Vec::new();
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash.to_bytes_be());
    message.extend_from_slice(chain_id.as_hex().as_bytes());
    message.extend_from_slice(&height.0.to_be_bytes());
    message.extend_from_slice(&round.to_be_bytes());
    MessageDigest(blake2s_to_felt(&message))
}
```

Update `sign_precommit_vote` and `verify_precommit_vote_signature` accordingly, and propagate the new parameters through all call sites.

### Proof of Concept

1. Validator V signs a precommit on **chain A** for `block_hash = X` at `height = 100, round = 0`. The signed bytes are `blake2s(b"PRECOMMIT_VOTE" || X)`.
2. On **chain B** (different `chain_id`, same validator set), the sequencer produces a block whose hash also equals `X` (possible because `chain_id` is absent from `calculate_block_hash`).
3. An attacker constructs a `Vote { vote_type: Precommit, height: 100, round: 0, proposal_commitment: Some(X_partial), voter: V, signature: <V's signature from chain A> }` and broadcasts it on chain B.
4. `verify_precommit_vote_signature(X, signature, V.public_key)` returns `true` — the signature is valid on chain B.
5. If this forged vote contributes to a quorum on chain B, the consensus engine accepts a block that validator V never actually endorsed on that chain. [6](#0-5) [1](#0-0)

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
