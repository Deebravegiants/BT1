### Title
Unrestricted `clean_invalid_attestations` Enables Adversarial Removal of Participant Attestations, Temporarily Blocking Threshold-Signing Responses — (File: `crates/contract/src/lib.rs`)

---

### Summary

The `clean_invalid_attestations` public endpoint carries no access-control guard and is explicitly documented as callable by anyone. An unprivileged adversary can invoke it at any moment to evict attestation records from `tee_state.stored_attestations`. Because `is_caller_an_attested_participant` — the gate used by every `respond*` callback — checks only for the *existence* of a stored attestation rather than its current validity, removing an entry (even one that just expired) immediately prevents the owning node from submitting threshold-signature responses until it re-attests. This is the direct analog of the `destroyInstance()` unrestricted-deletion class: a state-destructive function callable by any wallet.

---

### Finding Description

**Root cause — no access control on a state-mutating endpoint** [1](#0-0) 

The function carries no `#[private]` attribute, no participant check, and no self-call restriction. Any NEAR account may call it at any time while the contract is in `Running` state.

**What it deletes**

Internally it calls `TeeState::clean_invalid_attestations`, which iterates `stored_attestations` and removes every entry whose `reverify_participants` result is not `TeeQuoteStatus::Valid` (expired timestamp or stale docker/launcher/measurement hash): [2](#0-1) 

**The inconsistency that makes removal harmful**

`assert_caller_is_attested_participant_and_protocol_active`, which gates every `respond*` callback, delegates to `is_caller_an_attested_participant`. That check only verifies that an entry *exists* in `stored_attestations` — it does **not** re-verify the attestation's validity: [3](#0-2) 

Consequence: a node whose attestation has just expired can still call `respond_signature` / `respond_ckd` as long as the record remains in storage. The moment an adversary calls `clean_invalid_attestations` and the record is deleted, the same node is locked out — `AttestationNotFound` is returned and the call panics — until the node successfully re-attests.

**Affected callers of `assert_caller_is_attested_participant_and_protocol_active`** (all gated during `Running` state): [4](#0-3) 

The comment at line 2420–2422 explicitly lists `respond*` callbacks as going through this path.

---

### Impact Explanation

During the window between attestation expiry and successful re-attestation, an adversary can call `clean_invalid_attestations` to delete one or more participant records. Each affected node immediately loses the ability to submit `respond_signature` or `respond_ckd` receipts. If enough nodes are simultaneously in this expired-but-not-yet-renewed state, no threshold of responses can be collected, and all in-flight signature or CKD requests time out. This is a **request-lifecycle manipulation that breaks the production signing invariant** (threshold responses must be collectable from attested participants) without requiring any network-level capability.

The sandbox test confirms that `clean_invalid_attestations` is intentionally reachable by any account and that it evicts participant-owned entries: [5](#0-4) 

---

### Likelihood Explanation

Attestations carry a 7-day expiry stamped by the node at submission time. All nodes in a cluster that was deployed or last re-attested together will have correlated expiry windows. An adversary who monitors `get_tee_accounts` can observe when multiple participants' attestations are about to expire and time the call to `clean_invalid_attestations` to hit the gap before `monitor_attestation_removal` on each node fires and re-submits. The attack requires no privileged role, no collusion, and no key material — only a single on-chain transaction.

---

### Recommendation

1. **Restrict the caller**: Apply the same guard used by `remove_non_participant_update_votes` — allow only the contract itself (self-call) or a current participant to invoke `clean_invalid_attestations`. This eliminates the unprivileged entry path while preserving the post-reshare cleanup promise.
2. **Alternatively, add validity check to `is_caller_an_attested_participant`**: Make the gate re-verify the stored attestation's current validity so that an expired-but-present record no longer grants access. This removes the inconsistency that makes deletion harmful.

---

### Proof of Concept

```
1. Observe that participant P's attestation expires at block T
   (via get_tee_accounts / stored expiry_timestamp_seconds).

2. At block T+1, before P's node calls submit_participant_info again,
   submit the following transaction from any unprivileged account:

     near call <mpc_contract> clean_invalid_attestations
       '{"max_scan": 1000}' --accountId attacker.near

3. P's entry is removed from stored_attestations.

4. P's node attempts to call respond_signature for a pending request.
   assert_caller_is_attested_participant_and_protocol_active →
   is_caller_an_attested_participant → stored_attestations.get(&tls_key)
   returns None → AttestationNotFound → call panics.

5. All in-flight sign() requests that required P's response time out
   (fail_on_timeout fires), and deposited funds are not returned to
   callers until the timeout receipt executes.

6. Repeat step 2 each time P re-attests to sustain the disruption
   across multiple signing rounds.
```

    /// Prunes up to `max_scan` stored attestations that fail re-verification (expired or
    /// referencing stale whitelists). Returns the number of entries removed. Callable by
    /// anyone while the protocol is in `Running`.
    #[handle_result]
    pub fn clean_invalid_attestations(&mut self, max_scan: u32) -> Result<u32, Error> {
        log!(
            "clean_invalid_attestations: signer={}, max_scan={}",
            env::signer_account_id(),
            max_scan
        );
        // Running-only: keygen / resharing may reference attestations that have not yet
        // been activated, so cleanup is off-limits during those phases.
        if !matches!(self.protocol_state, ProtocolContractState::Running(_)) {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }
        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);
        Ok(self
            .tee_state
            .clean_invalid_attestations(tee_upgrade_deadline_duration, max_scan as usize))
    }

    pub fn clean_invalid_attestations(
        &mut self,
        tee_upgrade_deadline_duration: Duration,
        max_scan: usize,
    ) -> u32 {
        let has_invalid_attestation = |node_id: &NodeId| {
            !matches!(
                self.reverify_participants(node_id, tee_upgrade_deadline_duration),
                TeeQuoteStatus::Valid
            )
        };

        // Materialize candidates before any mutation to avoid iterator invalidation.
        let invalid_tls_keys: Vec<Ed25519PublicKey> = self
            .stored_attestations
            .iter()
            .take(max_scan)
            .filter(|(_, node_attestation)| has_invalid_attestation(&node_attestation.node_id))
            .map(|(tls_pk, _)| tls_pk.clone())
            .collect();

        let removed = u32::try_from(invalid_tls_keys.len())
            .expect("u32 should always be convertible from usize on wasm32");

        for tls_pk in invalid_tls_keys {
            self.stored_attestations.remove(&tls_pk);
        }
        removed

    /// Returns Ok(()) if the caller has at least one participant entry
    /// whose TLS key matches an attested node belonging to the caller account.
    ///
    /// Handles multiple participants per account and supports legacy mock nodes.
    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

/// **Test that `clean_invalid_attestations()` prunes expired attestations end-to-end via
/// the public endpoint**, including attestations that belong to current participants.
/// Restores the cleanup-path coverage that lived in the old `clean_tee_status` test.
#[test]
fn clean_invalid_attestations__should_remove_expired_entries() {
    // Given
    const PARTICIPANT_COUNT: usize = 2;
    const THRESHOLD: u64 = 2;
    const EXPIRY_SECONDS: u64 = 1_000;
    const NOW_NS: u64 = 5_000 * NANOS_IN_SECOND;

    testing_env!(
        VMContextBuilder::new()
            .attached_deposit(NearToken::from_near(1))
            .block_timestamp(0)
            .build()
    );

    let mut setup = TestSetupBuilder::new()
        .with_partcipant_count(PARTICIPANT_COUNT)
        .with_threshold(THRESHOLD)
        .build();

    let expiring_attestation = Attestation::Mock(MockAttestation::WithConstraints {
        mpc_docker_image_hash: None,
        launcher_docker_compose_hash: None,
        expiry_timestamp_seconds: Some(EXPIRY_SECONDS),
        expected_measurements: None,
    });

    // init_running seeds one mock `Valid` attestation per participant. Overwrite the
    // first participant's entry with an expiring one, and add a brand-new entry for an
    // outsider account.
    let participant_node = {
        let (account_id, _, info) = &setup.participants_list[0];
        NodeId {
            account_id: account_id.clone(),
            tls_public_key: info.tls_public_key.clone(),
            account_public_key: bogus_ed25519_public_key(),
        }
    };
    setup.submit_attestation_for_node(&participant_node, expiring_attestation.clone());

    let stale_node = NodeId {
        account_id: "stale.near".parse().unwrap(),
        tls_public_key: bogus_ed25519_public_key(),
        account_public_key: bogus_ed25519_public_key(),
    };
    setup.submit_attestation_for_node(&stale_node, expiring_attestation);

    const EXPECTED_STORED: usize = PARTICIPANT_COUNT + 1; // original mocks + outsider entry
    assert_eq!(setup.contract.get_tee_accounts().len(), EXPECTED_STORED);

    // When: time advances past the expiry and cleanup runs with a generous max_scan.
    set_system_time(NOW_NS);
    let removed = setup.contract.clean_invalid_attestations(100).unwrap();

    // Then: both entries with `expiry_timestamp_seconds` in the past are evicted; the
    // second participant's un-overwritten `Valid` mock remains.
    const EXPECTED_REMOVED: u32 = 2;
    assert_eq!(removed, EXPECTED_REMOVED);
    assert_eq!(
        setup.contract.get_tee_accounts().len(),
        EXPECTED_STORED - EXPECTED_REMOVED as usize
    );
}
