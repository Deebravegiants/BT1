### Title
Missing Consensus Vote Signature Verification Allows Any Peer to Forge Validator Votes — (`File: crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

The `handle_vote` function in `SingleHeightConsensus` accepts incoming `Vote` messages from the P2P network and counts them toward quorum without ever verifying the cryptographic signature. A `Vote` carries a `voter` field (claimed validator address) and a `signature` field, but the signature is never checked against the claimed voter's public key. Any connected peer can send a vote claiming to be any committee member, and the vote will be accepted and counted.

The only guards applied before the vote is forwarded to the state machine are:
1. Height check
2. Committee membership check (`vote.voter` is in the committee — but this is the *claimed* voter, not a verified one)
3. Duplicate/conflict detection [2](#0-1) 

The `Vote` protobuf message carries both a `voter` address and a `signature` field: [3](#0-2) 

The signature verification infrastructure exists — `verify_precommit_vote_signature` and `build_precommit_vote_message_digest` are fully implemented in `apollo_signature_manager` — but they are **only called in tests**, never in the production vote-handling path: [4](#0-3) [5](#0-4) 

This is the direct Sequencer analog of the external report's pattern: a signature verification function exists and is correct, but is unused in the production code path, leaving the critical check absent.

### Impact Explanation

Any network peer can craft a `Vote` message with:
- `voter` = any validator address present in the current committee
- `signature` = arbitrary bytes (or empty)
- `proposal_commitment` = any block hash

The vote passes all existing guards and is counted toward quorum. An attacker who can send P2P messages can forge prevotes and precommits from any validator identity. If the attacker forges votes representing ≥ 2/3 of the committee weight, `upon_decision` fires and `DecisionReached` is emitted for an attacker-chosen block commitment: [6](#0-5) 

This leads to the sequencer committing a wrong block, producing wrong state, wrong receipts, and wrong block hash — a consensus-level integrity break.

### Likelihood Explanation

The trigger requires only the ability to send P2P messages to a sequencer node, which is available to any connected peer in the network. No privileged access is needed. The attacker must forge votes totaling ≥ 2/3 of stake weight, but since each validator's identity is only a claimed address field with no cryptographic binding, a single attacker can forge votes for every committee member simultaneously.

### Recommendation

Call `verify_precommit_vote_signature` (or the equivalent prevote verification) inside `handle_vote` before forwarding the vote to the state machine. The signing and verification infrastructure already exists in `apollo_signature_manager`; it only needs to be wired into the production vote-handling path. The `// TODO(Asmaa): verify the signature` comment should be resolved before the consensus layer is used in a security-sensitive context.

### Proof of Concept

1. Connect a peer to the sequencer's consensus P2P endpoint.
2. Obtain the current committee for height `H` (available from the network or RPC).
3. For each validator `V_i` in the committee, craft a `Vote { vote_type: Precommit, height: H, round: 0, proposal_commitment: <attacker_chosen_hash>, voter: V_i.address, signature: <arbitrary> }`.
4. Send all forged votes to the sequencer.
5. `handle_vote` passes the height check, the committee membership check (the claimed address is valid), and the duplicate check (first occurrence), then forwards each vote to the state machine.
6. `upon_decision` accumulates the forged precommit weights; once ≥ 2/3 threshold is reached, `DecisionReached` fires with the attacker-chosen block commitment. [7](#0-6)

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }

fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash);

    MessageDigest(blake2s_to_felt(&message))
}
