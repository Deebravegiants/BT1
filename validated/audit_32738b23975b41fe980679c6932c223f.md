### Title
Unbounded Nested Loop in `_getTotalEthInProtocol()` Can Permanently Block `updateRSETHPrice()` — (File: `contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a public function that calls `_getTotalEthInProtocol()`, which contains a deeply nested loop structure: it iterates over all supported assets, and for each asset calls `LRTDepositPool.getTotalAssetDeposits()`, which in turn iterates over all NodeDelegators and calls `NodeDelegator.getAssetUnstaking()` on each. `getAssetUnstaking()` itself contains a nested loop over all queued EigenLayer withdrawals and all strategies per withdrawal. As the protocol scales, this compound iteration can exceed the block gas limit, permanently preventing price updates, blocking protocol fee minting, and freezing the rsETH price oracle.

---

### Finding Description

The call chain rooted at `updateRSETHPrice()` is:

```
updateRSETHPrice()                          [public, no auth]
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ for each asset (N):
                 └─ getTotalAssetDeposits(asset)
                      └─ getAssetDistributionData(asset)
                           └─ for each NDC (M):
                                └─ getAssetUnstaking(asset)   [external call]
                                     └─ for each queued withdrawal (P):
                                          └─ for each strategy (Q):
                                               └─ strategy.sharesToUnderlyingView()  [external call]
```

**`LRTOracle._getTotalEthInProtocol()`** iterates over every supported asset and calls `getTotalAssetDeposits` for each: [1](#0-0) 

**`LRTDepositPool.getAssetDistributionData()`** iterates over every NDC and calls `getAssetUnstaking` on each: [2](#0-1) 

**`NodeDelegator.getAssetUnstaking()`** contains a nested loop over all queued EigenLayer withdrawals and all strategies per withdrawal, with an external call to `strategy.sharesToUnderlyingView()` inside the inner loop: [3](#0-2) 

The total iteration count is **O(N\_assets × M\_NDCs × P\_withdrawals × Q\_strategies)**, each step involving external calls. The protocol's own comments acknowledge up to 80 uncompleted withdrawals and up to 10 NDCs: [4](#0-3) 

With 10 assets, 10 NDCs, 80 queued withdrawals distributed across NDCs, and multiple strategies per withdrawal, the gas cost can easily exceed Ethereum's 30M block gas limit.

---

### Impact Explanation

If `updateRSETHPrice()` reverts due to gas exhaustion:

1. **Theft of unclaimed yield (High):** Protocol fee minting via `IRSETH.mint()` is blocked. The fee is computed and minted inside `_updateRsETHPrice()` — if the function cannot complete, the treasury never receives its fee share of accrued yield. [5](#0-4) 

2. **Permanent freezing of unclaimed yield (Medium):** The rsETH price becomes permanently stale. New depositors receive rsETH at an outdated rate, and the price diverges from actual TVL indefinitely. [6](#0-5) 

3. **Unbounded gas consumption (Medium):** The function is callable by any unprivileged user and will revert on-chain once the iteration count crosses the gas limit threshold, with no recovery path short of a contract upgrade.

---

### Likelihood Explanation

`updateRSETHPrice()` is `public` with no access control — any depositor or external caller can trigger it: [7](#0-6) 

The protocol is designed to scale: `maxNodeDelegatorLimit` starts at 10 and is admin-adjustable upward, the supported asset list grows over time, and `maxUncompletedWithdrawalCount` can be set up to 80. As the protocol matures and all three dimensions grow simultaneously, the gas cost of `_getTotalEthInProtocol()` grows multiplicatively. This is not a theoretical edge case — it is the expected production state of a mature deployment.

---

### Recommendation

Decouple the per-NDC `getAssetUnstaking()` aggregation from the price update path. Options include:

1. Cache the unstaking amounts in storage (updated lazily when withdrawals are queued/completed) so `getAssetUnstaking()` becomes an O(1) read instead of an O(P×Q) loop.
2. Accept an explicit NDC index parameter in `getAssetDistributionData()` (analogous to the `_tokenIndex` fix in the reference report) so callers can paginate across NDCs.
3. Remove `getAssetUnstaking()` from the on-chain price computation path and account for unstaking amounts via a separate keeper-updated storage variable.

---

### Proof of Concept

A realistic production scenario:
- 8 supported assets (ETH + 7 LSTs)
- 10 NodeDelegators
- 80 queued EigenLayer withdrawals distributed across NDCs, each with 2 strategies

`_getTotalEthInProtocol()` will execute:
- 8 outer iterations (assets)
- × 10 NDC calls per asset = 80 `getAssetUnstaking()` calls
- × 80 withdrawal iterations per NDC call
- × 2 strategy iterations per withdrawal, each calling `strategy.sharesToUnderlyingView()` (external)

= **12,800 external calls** inside a single `updateRSETHPrice()` transaction. At ~2,100 gas per external call minimum, this alone exceeds 26M gas — approaching the 30M block gas limit before accounting for the rest of the execution. Once this threshold is crossed, `updateRSETHPrice()` permanently reverts, fee minting halts, and the rsETH price oracle is frozen.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
