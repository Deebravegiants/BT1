### Title
`combine_ac` accepts attacker-substituted ciphertext without verifying `Q`, allowing output of arbitrary scalars — (`src/cbmpc/api/pve_batch_ac.cpp`)

### Summary
`combine_ac` calls `aggregate_to_restore_row` with `skip_verify=true` and performs no independent validation of the `Q` values embedded in the deserialized ciphertext. When the challenge bit for the target row is 1, `x_bin` is read directly from the serialized (attacker-controlled) `row.x_bin`. The final check `x[j]*G == Q[j]` uses the attacker-controlled `Q[j]`, making it trivially satisfiable. An attacker who can substitute the ciphertext blob can cause `combine_ac` to output arbitrary scalars as recovered private key shares.

### Finding Description
`combine_ac` in `src/cbmpc/api/pve_batch_ac.cpp` (lines 314–367) deserializes an `ec_pve_ac_t` from the caller-supplied `ciphertext` blob and immediately calls `aggregate_to_restore_row` with `skip_verify=true`:

```cpp
rv = pve_ct.aggregate_to_restore_row(bridge, ac_internal, attempt_index, label, quorum_bn, xs_bn,
                                     /*skip_verify=*/true, pk_ptrs);
```

Inside `aggregate_to_restore_row`, the `skip_verify=true` path skips the only call to `verify()`, which is the function that checks `this->Q` against a caller-supplied trusted `Q` and validates the cut-and-choose proof. After skipping verification:

- `b` (the challenge bitmask) is taken from the deserialized ciphertext.
- When `b.get_bit(row_index) == 1`, `x_bin` is assigned from `row.x_bin` — a raw deserialized field.
- `seed` comes from AES-GCM decryption of `row.c`, but the attacker controls `row.quorum_c`, so honest parties' `partial_decrypt_ac_attempt` calls produce shares of the attacker's chosen `K`, giving the attacker full knowledge of `seed`.
- The final check `x[j]*G != Q[j]` uses `Q[j]` from the deserialized ciphertext, which the attacker also controls.

The attacker crafts a ciphertext where:
1. `b` has bit=1 at `attempt_index`.
2. `quorum_c` encrypts shares of a chosen `K`.
3. `row.c` is a valid AES-GCM ciphertext under `K` of a chosen seed.
4. `x_bin` encodes `x1[j] = x_target[j] - x0[j]` where `x0[j] = drbg(seed).gen_bn(q)`.
5. `Q[j] = x_target[j] * G`.

`combine_ac` then outputs `x_target` as the recovered scalars.

### Impact Explanation
Critical. An attacker who can substitute the ciphertext blob passed to `combine_ac` (without the caller having called `verify_ac` first) can cause `combine_ac` to output attacker-chosen scalars as the recovered private key shares. This completely breaks the confidentiality guarantee of the PVE scheme for any caller that does not separately invoke `verify_ac`.

### Likelihood Explanation
The `combine_ac` public API provides no built-in verification step and no warning at the call site. The `verify_ac` function is entirely separate. Any caller that omits `verify_ac` — a plausible omission given the API design — is fully exposed. The ciphertext is a serialized blob typically transmitted over a network, making substitution a realistic attacker capability.

### Recommendation
`combine_ac` should either:
1. Require the caller to pass the expected `Q` values (trusted public keys) and validate them against `this->Q` before proceeding, or
2. Call `verify()` internally (requiring `all_ac_pks` to be passed through `combine_ac`), removing the `skip_verify=true` shortcut from the public API path.

At minimum, the API documentation must make it a hard precondition that `verify_ac` is called before `combine_ac`, and the internal check should assert this invariant.

### Proof of Concept
```
1. Choose target scalar x_target and K.
2. Compute quorum shares of K; encrypt each share under the corresponding leaf public key → quorum_c.
3. Derive k_aes, iv from hash(K, L); AES-GCM encrypt a chosen seed → row.c.
4. Compute x0 = drbg(seed).gen_bn(q); set x1 = x_target - x0 mod q; encode x1 → row.x_bin.
5. Set Q[0] = x_target * G; set b with bit=1 at attempt_index.
6. Serialize the crafted ec_pve_ac_t into a ciphertext blob.
7. Call partial_decrypt_ac_attempt on the crafted ciphertext for each leaf → honest parties return shares of K.
8. Call combine_ac(crafted_ciphertext, attempt_index, label, honest_shares) → out_xs[0] == x_target. ✓
```
