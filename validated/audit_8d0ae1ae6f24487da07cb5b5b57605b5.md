### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Redirect All Block Fees to Arbitrary Address - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates many fields of `ProposalInit` (timestamp, starknet_version, l1_da_mode, l2_gas_price_fri, all four L1 gas prices, version_constant_commitment, fee_proposal_fri) but never validates the `builder` field. The `builder` address is passed directly to `convert_to_sn_api_block_info` and becomes the `sequencer_address` in `BlockInfo`, which is used during execution to collect fees and is committed into the block hash via `PartialBlockHashComponents`. A malicious consensus proposer can set `builder` to any address; every validator will accept the proposal and compute the same commitment over the attacker-chosen sequencer address, finalizing a block that routes all transaction fees to the attacker.

### Finding Description

`ProposalInit` carries a `builder` field defined as "Address of the one who builds/sequences the block." [1](#0-0) 

`is_proposal_init_valid` exhaustively checks every other economically-sensitive field but contains no check for `builder`: [2](#0-1) 

The unchecked `builder` value is then forwarded verbatim to `initiate_validation` via `convert_to_sn_api_block_info(init)`, which maps it to `BlockInfo.sequencer_address`: [3](#0-2) 

`BlockInfo.sequencer_address` is consumed in two ways:

1. **Fee collection during execution** — the sequencer address is the recipient of all transaction fees charged during block execution.
2. **Block hash commitment** — `PartialBlockHashComponents::new` stores it as `sequencer`, which is hashed into the final `PartialBlockHash` and therefore into `ProposalCommitment`: [4](#0-3) [5](#0-4) 

Because both the proposer and every validator derive the commitment from the proposer-supplied `builder`, they all agree on the same `ProposalCommitment`. The `ProposalFinMismatch` guard only compares `built_block` against `received_fin.proposal_commitment`: [6](#0-5) 

No other guard in the validation path checks `builder`. The proposer sets it freely in `initiate_build`: [7](#0-6) 

### Impact Explanation

A malicious consensus proposer sets `ProposalInit.builder` to an attacker-controlled address. Every validator runs `is_proposal_init_valid`, finds no violation, calls `initiate_validation` with the attacker address as `sequencer_address`, executes the block, and computes a `ProposalCommitment` that commits to the attacker address. Consensus reaches agreement on this commitment. The finalized block has the wrong sequencer address, and all transaction fees for that block are credited to the attacker rather than the legitimate sequencer. The corrupted value is `BlockInfo.sequencer_address` → fee balance of the attacker's account, and the committed `PartialBlockHash` / `ProposalCommitment` that all validators sign over. This matches: **Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

### Likelihood Explanation

The attacker must be a consensus validator selected as the round proposer. Within a single malicious proposer turn, the attack is zero-cost and undetectable until the block is finalized. No additional capability beyond being a scheduled proposer is required.

### Recommendation

Add a `builder` field to `ProposalInitValidation` (populated from the node's own configured builder address, the same source used in `initiate_build`). In `is_proposal_init_valid`, reject any proposal whose `init_proposed.builder` does not equal the locally-expected builder address:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder),
    ));
}
```

### Proof of Concept

1. Malicious validator is selected as proposer for height H.
2. In `initiate_build`, override `args.builder_address` with attacker-controlled address `0xDEAD`.
3. `ProposalInit { builder: 0xDEAD, ... }` is broadcast.
4. Each validator calls `is_proposal_init_valid` — passes (no `builder` check).
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)` → `BlockInfo { sequencer_address: 0xDEAD, ... }`.
6. Batcher executes all transactions with `sequencer_address = 0xDEAD`; fees accumulate at `0xDEAD`.
7. `PartialBlockHashComponents { sequencer: 0xDEAD, ... }` → `ProposalCommitment` matches proposer's value.
8. `ProposalFinMismatch` check passes; `fin_sender.send(proposal_commitment)` succeeds.
9. Consensus decides on the commitment; block is finalized with `sequencer_address = 0xDEAD`.
10. All transaction fees for block H are in the attacker's account. [8](#0-7) [9](#0-8) [10](#0-9)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
