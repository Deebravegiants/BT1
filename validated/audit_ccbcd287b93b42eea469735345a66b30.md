### Title
Unvalidated `ProposalInit.builder` Address Flows Into Block Hash and Execution Context — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit.builder` is a proposer-supplied `ContractAddress` that is accepted verbatim by the validator without any whitelist or equality check. It is forwarded directly into `convert_to_sn_api_block_info()` as `sequencer_address`, which then enters `PartialBlockHashComponents` (and therefore the committed block hash) and the execution `BlockInfo` visible to contracts via the `get_block_info()` syscall. A malicious proposer can set `builder` to any arbitrary address, causing every validator to execute and commit a block whose `sequencer_address` is wrong — producing a wrong block hash and wrong syscall results for all contracts that read `get_block_info().sequencer_address`.

---

### Finding Description

`is_proposal_init_valid()` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `timestamp`, `starknet_version`, `version_constant_commitment`, all four L1 gas-price fields, and `fee_proposal_fri`. It does **not** validate `builder`. [1](#0-0) 

After `is_proposal_init_valid()` passes, `initiate_validation()` calls `convert_to_sn_api_block_info(&init)`: [2](#0-1) 

Inside `convert_to_sn_api_block_info()`, `init.builder` is placed verbatim into `sequencer_address`: [3](#0-2) 

This `BlockInfo` is passed to `batcher.validate_block()` and subsequently to `PartialBlockHashComponents::new()`: [4](#0-3) 

`sequencer` is a direct input to `calculate_block_hash()`: [5](#0-4) 

The `ProposalInitValidation` struct — the only reference data the validator holds — contains no `builder` field: [6](#0-5) 

Because both proposer and validator derive their `batcher_block_commitment` from the same `init.builder`-derived `block_info`, the `ProposalFinMismatch` guard does not fire: [7](#0-6) 

The block is committed with the attacker-chosen `sequencer_address` baked into the `PartialBlockHashComponents` stored in the batcher: [8](#0-7) 

---

### Impact Explanation

**Wrong state / wrong block hash from accepted input (Critical scope).**

1. **Wrong `get_block_info().sequencer_address` syscall result.** Every contract that calls `get_block_info()` during execution of this block receives the attacker-chosen address instead of the legitimate sequencer address. Any contract logic that branches on `sequencer_address` (e.g., fee-routing, access control, tip distribution) produces wrong storage writes, wrong events, and wrong receipts — all committed permanently.

2. **Wrong block hash committed to storage.** `sequencer` is a direct input to `calculate_block_hash()`. The committed `PartialBlockHashComponents` carries the wrong `sequencer` value, so the final block hash is wrong. All subsequent blocks chain off this wrong hash, and any L1 anchoring or proof that covers the block hash is invalidated.

---

### Likelihood Explanation

Requires a consensus participant to win the proposer election for a round and broadcast a `ProposalInit` with a crafted `builder` field. In a permissionless or semi-permissionless validator set this is a realistic threat. The attack is silent — no existing guard rejects it, and the `ProposalFinMismatch` check passes because all validators use the same wrong `builder` value.

---

### Recommendation

Add a `builder` field to `ProposalInitValidation` (populated from the node's own configured `builder_address`) and enforce equality in `is_proposal_init_valid()`:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(...));
}
```

Alternatively, derive `sequencer_address` from the node's own local configuration rather than from the network-supplied `ProposalInit.builder`, analogous to how `l2_gas_price_fri` is locally enforced rather than blindly accepted.

---

### Proof of Concept

1. Attacker controls a consensus validator that wins the proposer slot for block N.
2. Attacker constructs `ProposalInit { builder: ContractAddress(0xdead), ... }` with all other fields valid (passing all existing checks).
3. Validator nodes receive the `ProposalInit`, call `is_proposal_init_valid()` — passes (no `builder` check).
4. `initiate_validation()` → `convert_to_sn_api_block_info()` → `sequencer_address = 0xdead`.
5. Batcher executes all transactions with `block_info.sequencer_address = 0xdead`; contracts calling `get_block_info()` see `0xdead`.
6. `PartialBlockHashComponents { sequencer: 0xdead, ... }` is stored; `calculate_block_hash()` produces a hash that encodes `0xdead`.
7. `batcher_block_commitment` matches `received_fin.proposal_commitment` (both derived from the same wrong `builder`); `ProposalFinMismatch` does not fire.
8. Block N is committed with wrong `sequencer_address` and wrong block hash. [9](#0-8) [10](#0-9) [11](#0-10)

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
