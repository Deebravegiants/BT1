### Title
Uninitialized `previous_proposal_init` Defaults to Timestamp 0, Bypassing Lower-Bound Timestamp Validation in Proposal Acceptance — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`SequencerConsensusContext` initializes `previous_proposal_init` to `None` at construction and only populates it after `decision_reached` or `try_sync`. The `is_proposal_init_valid` function derives `last_block_timestamp` via `.map_or(0, ...)`, silently defaulting to `0` whenever no previous block info is present. This disables the lower-bound timestamp check for any validator that has just restarted at height > 0, allowing a malicious proposer to submit a `ProposalInit` with `timestamp = 0` (or any arbitrarily old value) that passes all timestamp guards on those validators.

---

### Finding Description

**Root cause — uninitialized field:**

`SequencerConsensusContext::new()` sets `previous_proposal_init: None`. [1](#0-0) 

The field is only written in two places:
- `decision_reached` (after a block is committed locally): [2](#0-1) 

- `try_sync` (after a sync block is accepted): [3](#0-2) 

There is no code path that pre-populates `previous_proposal_init` from persistent storage before the consensus loop begins. A validator that restarts at height N > 0 therefore enters the validation path with `previous_proposal_init = None`.

**Defective guard in `is_proposal_init_valid`:** [4](#0-3) 

When `previous_proposal_init` is `None`, `last_block_timestamp` is `0`. The guard `init_proposed.timestamp < last_block_timestamp` becomes `timestamp < 0`, which is always `false` for a `u64`. The only remaining constraint is the upper-bound check (`timestamp <= now + window`), so any timestamp in `[0, now + window]` — including `0` — passes unconditionally.

The identical pattern appears in `try_sync`: [5](#0-4) 

**Propagation to block execution:**

A proposal that passes `is_proposal_init_valid` proceeds to `initiate_validation`, which calls `batcher.validate_block` with the `block_info` derived directly from the accepted `ProposalInit` (including its `timestamp`): [6](#0-5) 

The `convert_to_sn_api_block_info` function embeds `init.timestamp` as `BlockTimestamp` into the `BlockInfo` passed to the blockifier: [7](#0-6) 

If the proposal is accepted by ≥ 2/3 validators, `decision_reached` commits the block with that timestamp, and `update_state_sync_with_new_block` persists it.

---

### Impact Explanation

If a block with `timestamp = 0` (Unix epoch, 1970-01-01) is committed:

1. **Wrong syscall result** — every contract calling `get_block_timestamp()` receives `0`, corrupting any time-dependent logic (vesting, auctions, expiry checks). This is a wrong state value from blockifier execution.
2. **Wrong block hash** — `timestamp` is a component of `PartialBlockHashComponents`; the committed `partial_block_hash` and all downstream `ProposalCommitment` values are incorrect.
3. **Wrong fee / gas price** — `get_l1_prices_in_fri_and_wei` queries the L1 gas price provider with `timestamp = 0`; the provider will fail to find data for that epoch and fall back to stale or default prices, producing incorrect fee accounting for every transaction in the block.
4. **Broken monotonicity** — all subsequent blocks must have `timestamp >= 0`, so the invariant is trivially satisfied but the chain's real-time ordering is destroyed.

---

### Likelihood Explanation

The exploit requires:

1. A coordinated restart (or simultaneous first-start) of ≥ 2/3 of the validator set at height > 0, before any of them has called `decision_reached` or successfully completed `try_sync` for the preceding height.
2. A malicious (or compromised) proposer that sends `ProposalInit { timestamp: 0, … }` during that window.
3. The restarted validators receiving the proposal before receiving a sync block that would populate `previous_proposal_init`.

This window is narrow in steady-state operation but is realistic during a planned upgrade, a coordinated rolling restart, or a network partition that forces a large fraction of validators to restart simultaneously. The proposer role rotates through the validator set, so any validator that becomes proposer during the window can trigger the condition unilaterally for its own proposal.

---

### Recommendation

Before the consensus loop begins (e.g., inside `initialize_fee_proposals_window` or a new `initialize_previous_proposal_init` call), read the committed block header for `height - 1` from `state_sync_client` and populate `previous_proposal_init` from it. This mirrors the existing `initialize_fee_proposals_window` pattern: [8](#0-7) 

Alternatively, add an explicit guard in `is_proposal_init_valid` that rejects any proposal with `timestamp == 0` when `height > 0`, regardless of whether `previous_proposal_init` is populated.

---

### Proof of Concept

```
1. Network is at height N = 1000 (last committed block has timestamp T_999 ≈ 1_700_000_000).
2. 2/3+ validators restart simultaneously (e.g., rolling upgrade).
   → Each restarts with previous_proposal_init = None.
3. Before any validator calls try_sync or decision_reached, the proposer
   (which did NOT restart) broadcasts:
     ProposalInit {
       height: 1000,
       timestamp: 0,          // Unix epoch
       l2_gas_price_fri: <valid>,
       l1_da_mode: <valid>,
       …
     }
4. Each restarted validator calls is_proposal_init_valid:
     last_block_timestamp = None.map_or(0, …) = 0
     check: 0 < 0  → false  (no error)
     check: 0 > now + window → false (no error)
   → Proposal passes timestamp validation.
5. Validators execute the block with BlockTimestamp(0) and vote Precommit.
6. ≥ 2/3 Precommits → decision_reached → block committed with timestamp = 0.
7. All get_block_timestamp() syscalls in that block return 0.
   PartialBlockHash is computed over timestamp=0, producing a wrong commitment.
```

    pub async fn initialize_fee_proposals_window(
        &mut self,
        start_height: BlockNumber,
    ) -> StateSyncClientResult<()> {
        const STATE_SYNC_RETRY_INTERVAL: Duration = Duration::from_millis(500);
        let window_size = VersionedConstants::latest_constants().fee_proposal_window_size;
        let window_end_height = start_height.0;
        let window_start_height = window_end_height.saturating_sub(window_size);
        let mut pending_heights: VecDeque<BlockNumber> =
            (window_start_height..window_end_height).map(BlockNumber).collect();
        while let Some(block_number) = pending_heights.pop_front() {
            match self.deps.state_sync_client.get_block(block_number).await {
                Ok(block) => self.record_fee_proposal(
                    block_number,
                    block.block_header_without_hash.fee_proposal_fri,
                ),
                Err(StateSyncClientError::StateSyncError(StateSyncError::BlockNotFound(_))) => {
                    warn!(
                        "State sync not ready for height {block_number}; re-queueing after \
                         {STATE_SYNC_RETRY_INTERVAL:?}"
                    );
                    pending_heights.push_back(block_number);
                    tokio::time::sleep(STATE_SYNC_RETRY_INTERVAL).await;
                }
                Err(e) => return Err(e),
            }
        }
        Ok(())
    }

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
