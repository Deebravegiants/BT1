### Title
Lido Share-Rate Drop Leaves Phantom `ethValueInWithdrawal`, Permanently Inflating rsETH Price — (`contracts/LRTConverter.sol`)

### Summary
When stETH is moved to `LRTConverter` via `transferAssetFromDepositPool`, `ethValueInWithdrawal` is credited at the current oracle price. If Lido's withdrawal queue finalizes at a lower share rate (due to slashing), the actual ETH received is less than the credited amount. `_sendEthToDepositPool` reduces `ethValueInWithdrawal` by the actual ETH received, leaving a non-zero phantom remainder. This phantom value is permanently included in `_getTotalEthInProtocol()` via `getETHDistributionData()`, causing `rsETHPrice * rsETH.totalSupply() > actual ETH in protocol`.

### Finding Description

`transferAssetFromDepositPool` credits `ethValueInWithdrawal` at the oracle price at transfer time:

```solidity
// LRTConverter.sol:140
ethValueInWithdrawal += (_amount * lrtOracle.getAssetPrice(_asset)) / 1e18;
```

`getAssetDistributionData` for stETH explicitly zeroes `assetLyingInConverter` and delegates to `getETHDistributionData`:

```solidity
// LRTDepositPool.sol:460
assetLyingInConverter = 0; // assets in converter are accounted in their eth value => getETHDistributionData()
```

```solidity
// LRTDepositPool.sol:498-499
address lrtConverter = lrtConfig.getContract(LRTConstants.LRT_CONVERTER);
ethLyingInConverter = ILRTConverter(lrtConverter).ethValueInWithdrawal();
```

After `claimStEth` → `_sendEthToDepositPool(actualETH)`:

```solidity
// LRTConverter.sol:255-258
if (ethValueInWithdrawal > _amount) {
    ethValueInWithdrawal -= _amount;   // phantom = oracle_value - actual_ETH
} else {
    ethValueInWithdrawal = 0;
}
```

If `oracle_price > finalization_rate`, `ethValueInWithdrawal` retains `(oracle_price - finalization_rate) * stETH_amount` as phantom ETH with no backing. This phantom is then used in `_updateRsETHPrice`:

```solidity
// LRTOracle.sol:250
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
```

### Impact Explanation

The rsETH price is permanently overstated. Every subsequent `updateRSETHPrice` call bakes in the phantom ETH, meaning `rsETHPrice * rsETH.totalSupply() > actual redeemable ETH`. New depositors receive fewer rsETH tokens than they should (subsidizing existing holders), and the protocol cannot fully honour all rsETH redemptions. This is **Protocol insolvency (Critical)**.

The downside-protection circuit breaker in `_updateRsETHPrice` (lines 270–281) does not help: it only triggers on a price *decrease* beyond `pricePercentageLimit`. The phantom value causes a price *increase* (or prevents a correct decrease), so the circuit breaker never fires.

### Likelihood Explanation

Lido's `WithdrawalQueue.finalize` uses `min(current_share_rate, rate_at_request_time)`. A drop in share rate requires a validator slashing event. While historically rare, Lido's share rate has decreased during mass-slashing incidents. The stETH oracle price used by the protocol (typically 1.02–1.05 ETH/stETH) is always above 1.0, so any finalization at a rate below the oracle price at transfer time creates a phantom. The likelihood is **Low** but the impact is permanent and unrecoverable without an upgrade.

### Recommendation

In `_sendEthToDepositPool`, always zero out `ethValueInWithdrawal` after a claim cycle rather than subtracting the received amount:

```solidity
// Option A: zero out on each claim (requires tracking per-request credited value)
// Option B: add an admin function to manually correct ethValueInWithdrawal
// Option C: re-read actual stETH balance in LRTConverter before crediting,
//           or credit at finalization rate rather than oracle price
```

The safest fix is to track `ethValueInWithdrawal` per withdrawal request (keyed by `requestId`) and zero out exactly the credited amount for that request upon claim, regardless of actual ETH received. Any shortfall should be recognised as a loss and reflected in a lower rsETH price rather than silently inflating it.

### Proof of Concept

```solidity
// Fork test (Mainnet fork, no public-mainnet state changes)
function test_phantomEthValueAfterLidoSlash() external {
    // 1. Operator transfers 1000 stETH to LRTConverter at oracle price 1.02
    //    => ethValueInWithdrawal = 1020e18
    vm.prank(assetTransferRole);
    lrtConverter.transferAssetFromDepositPool(stETH, 1000e18);
    assertEq(lrtConverter.ethValueInWithdrawal(), 1020e18);

    // 2. Operator submits Lido withdrawal request
    vm.prank(operator);
    lrtConverter.unstakeStEth(1000e18);

    // 3. Simulate Lido finalization at 1.0 ETH/stETH (slash scenario)
    //    finalize() with maxShareRate = 1e27 (1.0 in ray)
    vm.prank(lidoFinalizationRole);
    withdrawalQueue.finalize{value: 1000e18}(requestId, 1e27);

    // 4. Claim — LRTConverter receives 1000 ETH
    vm.prank(operator);
    lrtConverter.claimStEth(requestId, hint);

    // 5. Phantom value remains
    assertEq(lrtConverter.ethValueInWithdrawal(), 20e18); // 20 ETH phantom

    // 6. rsETH price is inflated
    lrtOracle.updateRSETHPrice();
    uint256 rsETHPrice = lrtOracle.rsETHPrice();
    uint256 supply = rsETH.totalSupply();
    uint256 actualETH = address(lrtDepositPool).balance; // + other real ETH

    // Invariant violated: price * supply > actual ETH
    assertGt(rsETHPrice * supply / 1e18, actualETH);
}
```
