### Title
Unvalidated `ProposalInit.builder` Field Allows Arbitrary `sequencer_address` in Block Hash and Execution Context — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates many `ProposalInit` fields (height, timestamp, l1_da_mode, l2_gas_price_fri, starknet_version, version_constant_commitment, all four L1 gas prices, fee_proposal_fri) but never checks `init.builder`. That field is passed verbatim as `sequencer_address` into `BlockInfo`, which is then committed into the block hash via `PartialBlockHashComponents.sequencer` and used as the fee-recipient address during transaction execution. A malicious proposer can set `builder` to any address, causing every validator to execute and commit a block with a wrong `sequencer_address`.

---

### Finding Description

**Data-flow chain:**

1. **Proposer** constructs `ProposalInit` in `initiate_build` (`build_proposal.rs:174`):
   ```rust
   builder: args.builder_address,   // taken from static config
   ```
   This value is transmitted over the network to all validators.

2. **Validator** calls `is_proposal_init_valid` (`validate_proposal.rs:153–160`). The function checks height, timestamp bounds, starknet_version, version_constant_commitment, l1_da_mode, l2_gas_price_fri, all four L1 gas prices, and fee_proposal_fri. **`init.builder` is never compared against any locally-trusted reference.** `ProposalInitValidation` does not even carry a `builder` field.

3. After passing validation, `initiate_validation` calls `convert_to_sn_api_block_info(init)` (`utils.rs:332`):
   ```rust
   sequencer_address: init.builder,   // proposer-supplied, unvalidated
   ```

4. The resulting `BlockInfo` is passed to the batcher's `validate_block`. Inside the batcher, `PartialBlockHashComponents::new` (`block_hash_calculator.rs:231`) reads:
   ```rust
   sequencer: SequencerContractAddress(block_info.sequencer_address),
   ```

5. `calculate_block_hash` (`block_hash_calculator.rs:258`) commits `sequencer` into the Poseidon block hash:
   ```rust
   .chain(&partial_block_hash_components.sequencer.0)
   ```

6. The same `sequencer_address` is the fee-recipient for every transaction in the block and is returned by the `get_sequencer_address()` syscall.

**The invariant broken:** every validator independently trusts its own configuration for which address is the legitimate block builder/sequencer. Because `builder` is never validated, a malicious proposer can substitute any address and all validators will accept it, execute with it, and commit it.

---

### Impact Explanation

- **Wrong block hash / state commitment.** The `sequencer_address` is a direct input to `calculate_block_hash`. A spoofed `builder` produces a block hash that encodes the wrong sequencer, corrupting the authoritative on-chain commitment for that block.
- **Fee redirection.** Transaction fees are paid to `sequencer_address`. A malicious proposer sets `builder` to an address they control and collects all fees from every transaction in the block.
- **Wrong `get_sequencer_address()` syscall result.** Any contract that calls this syscall (e.g., for access control or fee-routing logic) receives the attacker-supplied address instead of the legitimate one, potentially bypassing contract-level guards.

---

### Likelihood Explanation

Requires the attacker to be a scheduled proposer in the current consensus round — a role that rotates among the validator set. Any validator that gains proposer rights can trigger this with a single modified config value (`builder_address`). No external user interaction or privileged system access beyond normal proposer duties is needed.

---

### Recommendation

Add `builder_address: ContractAddress` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`:

```rust
// In ProposalInitValidation
pub builder_address: ContractAddress,

// In is_proposal_init_valid
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

Populate `builder_address` from `self.config.static_config.builder_address` when constructing `ProposalInitValidation` in `validate_proposal` (`sequencer_consensus_context.rs:880–900`), mirroring how `starknet_version` and `l1_da_mode` are already sourced from local config.

---

### Proof of Concept

1. Attacker controls a validator node that is scheduled as proposer for block N.
2. Attacker sets `builder_address = <attacker_wallet>` in their node's static config.
3. `initiate_build` constructs `ProposalInit { builder: <attacker_wallet>, … }` and streams it to all peers.
4. Each peer's `validate_proposal` calls `is_proposal_init_valid` — no check on `builder` — and proceeds.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: <attacker_wallet>, … }`.
6. The batcher executes all transactions with `sequencer_address = <attacker_wallet>`; all fees are credited to the attacker.
7. `PartialBlockHashComponents { sequencer: <attacker_wallet>, … }` is hashed into the `ProposalCommitment` and ultimately into the canonical block hash.
8. Consensus reaches agreement on this block; the wrong `sequencer_address` is permanently committed to the chain.

```rust
#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
```

```rust
pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
```
