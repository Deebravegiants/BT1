### Title
Unvalidated `builder` Field in `ProposalInit` Allows Arbitrary Sequencer Address Injection into Block Hash and Execution Context - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

### Summary

The `builder` field of `ProposalInit` is accepted verbatim by every validator without any check against a locally-trusted reference value. `convert_to_sn_api_block_info` maps `init.builder` directly to `BlockInfo.sequencer_address`, which flows into both the block hash (`PartialBlockHashComponents.sequencer`) and the blockifier execution context. A malicious proposer can set `builder` to any address, causing all validators to execute the block with an attacker-controlled sequencer address, redirecting transaction fees and corrupting the `get_sequencer_address` syscall result for every transaction in the block.

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` validates the following `ProposalInit` fields: `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields (within margin), and `fee_proposal_fri` (presence and bounds). The `builder` field is absent from `ProposalInitValidation` and is never checked. [1](#0-0) 

`convert_to_sn_api_block_info` (called from both `initiate_build` and `initiate_validation`) maps `init.builder` directly to `sequencer_address`: [2](#0-1) 

That `sequencer_address` is then embedded in `PartialBlockHashComponents`: [3](#0-2) 

And hashed into the block hash at position 4 of the Poseidon chain: [4](#0-3) 

The proposer sets `builder` from its own static config: [5](#0-4) 

The TODO comment confirms no committee-derived constraint exists yet: [6](#0-5) 

### Impact Explanation

**Incorrect fee/balance with economic impact (Critical):** In Starknet, transaction fees are transferred from the user's account to `sequencer_address` during execution. A malicious proposer sets `builder` to an attacker-controlled address; all validators accept the proposal and execute every transaction in the block with that address as the sequencer, directing all block fees to the attacker.

**Wrong syscall result (Critical):** Every contract that calls `get_sequencer_address` during execution of that block receives the attacker-supplied address instead of the legitimate sequencer address. Any contract using the sequencer address for access control or routing will behave incorrectly.

**Wrong block hash committed to chain (High):** The sequencer address is a direct input to `calculate_block_hash`. The finalized block hash stored on-chain encodes the attacker's address, permanently corrupting the canonical chain record for that block.

### Likelihood Explanation

Any validator that wins a consensus round becomes the proposer. In a permissionless or semi-permissioned BFT network, this is reachable by any participating node. No special privilege beyond being a scheduled proposer is required. The attack is silent — validators log the `builder` address but never reject it.

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce an exact-match check in `is_proposal_init_valid`, analogous to the existing `height` / `l1_da_mode` / `l2_gas_price_fri` check:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(...));
}
```

`expected_builder` should be populated from `self.config.static_config.builder_address` at the call sites in `sequencer_consensus_context.rs` (lines 880–900 and 1178–1198) where `ProposalInitValidation` is constructed. Once the committee-derived builder mechanism is implemented, the reference value should come from there instead. [7](#0-6) 

### Proof of Concept

```
1. Attacker controls validator node V.
2. V wins a proposer slot at height H.
3. V constructs ProposalInit with builder = ATTACKER_ADDRESS (any address).
4. V streams the proposal to all other validators.
5. Each validator calls is_proposal_init_valid — builder is never checked, validation passes.
6. initiate_validation calls convert_to_sn_api_block_info(init):
     sequencer_address = ATTACKER_ADDRESS
7. The batcher executes all transactions with sequencer_address = ATTACKER_ADDRESS.
   - All fee transfers credit ATTACKER_ADDRESS.
   - get_sequencer_address syscall returns ATTACKER_ADDRESS.
8. BlockExecutionArtifacts::new computes PartialBlockHashComponents with
     sequencer = ATTACKER_ADDRESS, producing a block hash that encodes ATTACKER_ADDRESS.
9. All validators agree on this commitment; the block is finalized with the wrong sequencer.
``` [8](#0-7) [9](#0-8) [10](#0-9) [11](#0-10)

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
