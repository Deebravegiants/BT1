Audit Report

## Title
Stale agETH/ETH Rate in `CrossChainRateReceiver.getRate()` Enables Over-Minting of agETH — (`contracts/agETH/AGETHRateReceiver.sol` / `contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary
`CrossChainRateReceiver.getRate()` returns the stored `rate` with no check against `lastUpdated`, so any LayerZero delivery delay leaves a stale-low rate in place. `AGETHPoolV3.deposit(token, amount)` divides fresh collateral value by this stale rate, minting more agETH than the deposited collateral backs. The excess agETH is unbacked and dilutes all existing agETH holders' redemption value.

## Finding Description
`CrossChainRateReceiver.getRate()` unconditionally returns `rate` with no staleness enforcement:

```solidity
// contracts/cross-chain/CrossChainRateReceiver.sol L103-105
function getRate() external view returns (uint256) {
    return rate;   // lastUpdated is recorded but never checked here
}
```

`AGETHPoolV3.getRate()` delegates directly to this oracle:

```solidity
// contracts/agETH/AGETHPoolV3.sol L104-106
function getRate() public view returns (uint256) {
    return IOracle(agETHOracle).getRate();
}
```

`viewSwapAgETHAmountAndFee(amount, token)` uses both rates to compute the mint amount:

```solidity
// contracts/agETH/AGETHPoolV3.sol L188-194
uint256 agETHToETHrate = getRate();                                       // potentially stale-low
uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate();  // fresh
agETHAmount = amountAfterFee * tokenToETHRate / agETHToETHrate;
```

Because agETH is yield-bearing, its true rate grows monotonically. Any gap between `lastUpdated` and the current block means `rate < R_true`, making the denominator smaller and `agETHAmount` larger than the collateral justifies. The inflated amount is minted unconditionally:

```solidity
// contracts/agETH/AGETHPoolV3.sol L147-151
(uint256 agETHAmount, uint256 fee) = viewSwapAgETHAmountAndFee(amount, token);
feeEarnedInToken[token] += fee;
agETH.mint(msg.sender, agETHAmount);
```

No cap, slippage limit, or staleness guard exists anywhere in the deposit path. The same flaw applies to the ETH deposit path via `viewSwapAgETHAmountAndFee(amount)` at L160-168.

## Impact Explanation
Every unit of excess agETH minted is unbacked by pool collateral. The total agETH supply grows beyond the pool's holdings, reducing the per-token redemption value for all existing holders. This constitutes **direct theft of at-rest user funds** (Critical), matching the allowed impact scope. The attacker can bridge the excess agETH to mainnet and redeem at the true rate for a risk-free profit proportional to deposit size and staleness gap.

## Likelihood Explanation
agETH accrues yield continuously, so `R_true` drifts above any cached `R_stale` within hours of a missed or delayed LayerZero message. No privileged access, key compromise, or victim mistake is required — the attacker only needs to observe that `lastUpdated` is old and call `deposit`. LayerZero liveness is not guaranteed; network congestion or outages create the necessary window. The attack is permissionless and repeatable until the rate is refreshed.

## Recommendation
Enforce a maximum staleness window in `CrossChainRateReceiver.getRate()`:

```solidity
uint256 public constant MAX_RATE_AGE = 24 hours;

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= MAX_RATE_AGE, "Rate is stale");
    return rate;
}
```

Alternatively, add a staleness guard in `AGETHPoolV3.getRate()` that reverts when the agETH oracle's `lastUpdated` is beyond an acceptable threshold, preventing deposits when the rate cannot be trusted.

## Proof of Concept
```solidity
// Fork test on the chain where AGETHPoolV3 is deployed
function testStaleRateOverMint() public {
    uint256 R_true = agETHRateReceiver.rate();

    // Simulate staleness: write a lower rate directly to storage slot 0
    uint256 R_stale = R_true * 99 / 100; // 1% below true (conservative)
    vm.store(address(agETHRateReceiver), bytes32(uint256(0)), bytes32(R_stale));

    uint256 T_fresh = IOracle(agETHPool.supportedTokenOracle(wstETH)).getRate();
    uint256 amount = 1e18;
    deal(wstETH, attacker, amount);

    vm.startPrank(attacker);
    IERC20(wstETH).approve(address(agETHPool), amount);
    agETHPool.deposit(wstETH, amount, "");
    vm.stopPrank();

    uint256 minted = agETH.balanceOf(attacker);

    // Invariant: minted * R_true must not exceed amount * T_fresh
    // (i.e., agETH received must not exceed collateral value at true rate)
    assertGt(
        minted * R_true,
        amount * T_fresh,
        "Invariant violated: excess agETH minted due to stale rate"
    );
}
```

The assertion passes whenever `R_stale < R_true`, confirming the over-mint. Profit scales linearly with deposit size and the staleness gap.
