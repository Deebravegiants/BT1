### Title
Unconstrained `fee_proposal_fri` During Startup Window Allows Arbitrary Fee Market Manipulation — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

When `fee_actual` is `None` (the first `window_size` blocks after V0_14_3 activation, before the sliding window accumulates enough entries), `is_proposal_init_valid` skips the `fee_proposal_fri` bounds check entirely. A malicious proposer can set `fee_proposal_fri` to any value — including `u128::MAX` or `0` — and every honest validator will accept the proposal without objection. The injected value is permanently recorded in `BlockHeaderWithoutHash.fee_proposal_fri`, populates the `fee_proposals_window`, and directly skews every future `fee_actual` (median) computation for the lifetime of the window.

---

### Finding Description

In `is_proposal_init_valid`, the bounds check is guarded by a combined `if let`:

```rust
// crates/apollo_consensus_orchestrator/src/validate_proposal.rs  lines 396-416
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    ...
}
``` [1](#0-0) 

When `proposal_init_validation.fee_actual` is `None`, the entire body is skipped. The proposer side correctly freezes at `l2_gas_price` as the fallback:

```rust
// crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs  lines 478-481
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [2](#0-1) 

But the validator never enforces that the received `fee_proposal_fri` equals (or is within any margin of) the validator's own `l2_gas_price`. The two sides are asymmetric: the proposer is constrained by convention; the validator enforces nothing.

`fee_actual` is computed and passed into `ProposalInitValidation` at both call sites: [3](#0-2) [4](#0-3) 

Both call `compute_fee_actual`, which returns `None` whenever the window is incomplete (height < `window_size`, or any window entry is `None`/missing): [5](#0-4) 

The accepted `fee_proposal_fri` is then bound into the `ProposalCommitment` via `proposal_commitment_from`: [6](#0-5) 

and stored in `BuiltProposals` keyed by that commitment: [7](#0-6) 

On `decision_reached`, the `fee_proposal_fri` from the winning `ProposalInit` is recorded into `fee_proposals_window` via `record_fee_proposal`: [8](#0-7) 

and is later read back from `BlockHeaderWithoutHash.fee_proposal_fri` when re-initializing the window after a restart: [9](#0-8) 

---

### Impact Explanation

The `fee_proposals_window` is the sole input to `compute_fee_actual`, which computes the median used as the reference for all subsequent `fee_proposal` bounds. An attacker who controls the proposer role for more than ⌊`window_size`/2⌋ + 1 blocks during the startup window can set `fee_actual` to any value — including `u128::MAX` — for every block that follows. Even controlling a minority of startup slots injects persistent outliers that shift the median. The corrupted `fee_actual` then propagates: every honest proposer's `compute_fee_proposal` clamps to bounds derived from the corrupted median, and every validator's bounds check (once `fee_actual` becomes `Some`) enforces those corrupted bounds. The result is a permanently manipulated L2 gas price with direct economic impact on all users.

---

### Likelihood Explanation

The attack window is the first `window_size` blocks after V0_14_3 activation (or after a fresh genesis). Any validator selected as proposer during that window can exploit this without any additional privilege. In Tendermint, proposer selection rotates deterministically through the validator set, so a single malicious validator is guaranteed at least one proposer slot during a sufficiently long startup window. The attack requires no special tooling — only setting `fee_proposal_fri` to an extreme value in the `ProposalInit` message.

---

### Recommendation

When `fee_actual` is `None`, the validator should enforce that the received `fee_proposal_fri` equals the validator's own `l2_gas_price` (within a small absolute tolerance, e.g., `GAS_PRICE_ABS_DIFF_MARGIN`). Concretely, add a branch to `is_proposal_init_valid`:

```rust
// When fee_actual is None (startup), the honest proposer freezes at l2_gas_price.
// Enforce the same constraint on the validator side.
if fee_proposal_required {
    if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
        let expected = proposal_init_validation.l2_gas_price_fri; // validator's fallback
        if fee_proposal.0.abs_diff(expected.0) > GAS_PRICE_ABS_DIFF_MARGIN {
            return Err(ValidateProposalError::InvalidProposalInit(...));
        }
    }
}
```

This closes the asymmetry between proposer and validator during the startup window.

---

### Proof of Concept

1. Network activates Starknet V0_14_3 (or starts from genesis). `fee_proposals_window` is empty; `compute_fee_actual` returns `None` for all heights `< window_size`.
2. Malicious validator is selected as proposer for block `N` (where `N < window_size`).
3. Proposer constructs `ProposalInit` with `fee_proposal_fri = Some(GasPrice(u128::MAX))` instead of the honest fallback `l2_gas_price`.
4. Honest validators call `is_proposal_init_valid`. `proposal_init_validation.fee_actual = None` → the `if let` at line 398 does not match → bounds check is skipped entirely. Proposal is accepted.
5. `decision_reached` is called. `record_fee_proposal(N, Some(GasPrice(u128::MAX)))` inserts the extreme value into `fee_proposals_window`.
6. After `window_size` blocks (with the malicious proposer controlling ≥ ⌊`window_size`/2⌋ + 1 slots), `compute_fee_actual` returns `Some(GasPrice(u128::MAX))`.
7. All subsequent `fee_proposal` bounds are computed relative to `u128::MAX`. Honest proposers are forced to propose near-`u128::MAX` fees; all user transactions become prohibitively expensive. [1](#0-0) [10](#0-9) [11](#0-10)

    /// `fee_actual`. When `fee_actual` is `None` (window incomplete), freeze at `l2_gas_price`; the
    /// validator derives the same fallback so both sides agree.
    async fn compute_proposer_fee_proposal(
        &self,
        fee_actual: Option<GasPrice>,
        timestamp: u64,
        target_atto_usd_per_l2_gas: u128,
    ) -> GasPrice {
        SNIP35_FEE_TARGET_ATTO_USD.set_lossy(target_atto_usd_per_l2_gas);
        let Some(fee_actual) = fee_actual else {
            warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
            SNIP35_FEE_PROPOSAL_FRI.set_lossy(self.l2_gas_price.0);
            return self.l2_gas_price;
        };
        SNIP35_FEE_ACTUAL_FRI.set_lossy(fee_actual.0);

        let fee_target = self.resolve_fee_target(timestamp, target_atto_usd_per_l2_gas).await;

        let proposal = compute_fee_proposal(
            fee_target,
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        SNIP35_FEE_PROPOSAL_FRI.set_lossy(proposal.0);
        proposal
    }

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
