### Title
Sub-transactions in Dijkstra era bypass value conservation check, enabling ADA creation - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxo.hs`)

---

### Summary

The Dijkstra era's `SUBUTXO` rule (`dijkstraSubUtxoTransition`) does not enforce value conservation for sub-transactions. The top-level `UTXO` rule (`dijkstraUtxoTransition`) calls `validateValueNotConservedUTxO` only against the top-level `txBody`, which is blind to sub-transaction inputs and outputs. An unprivileged transaction sender can craft a sub-transaction whose outputs exceed its inputs in ADA value, creating ADA out of thin air.

---

### Finding Description

The Dijkstra era introduces nested sub-transactions (`subTransactionsTxBodyL`). The top-level UTXO rule enforces value conservation:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Utxo.hs, line 380-381
{- consumed pp utxo₀ txb = produced pp certState txb -}
runTest $ Shelley.validateValueNotConservedUTxO pp originalUtxo certState txBody
```

Here `txBody` is `tx ^. bodyTxL` — the **top-level** transaction body only. The `consumed` function reads `txBody ^. inputsTxBodyL` (top-level spend inputs) and `txBody ^. withdrawalsTxBodyL` (top-level withdrawals). The `produced` function reads `txBody ^. outputsTxBodyL` (top-level outputs) and `txBody ^. feeTxBodyL`. Neither function inspects `txBody ^. subTransactionsTxBodyL`. [1](#0-0) 

The sub-transaction processing path (`dijkstraSubUtxoTransition`) performs these checks: [2](#0-1) 

Comparing the two rule bodies, `dijkstraSubUtxoTransition` is **missing** `validateValueNotConservedUTxO`. The mapping function `dijkstraUtxoToDijkstraSubUtxoPredFailure` explicitly marks `ValueNotConservedUTxO` as impossible for `SUBUTXO`:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxo.hs, line 333
ValueNotConservedUTxO _ -> error "Impossible: `ValueNotConservedUTxO` for SUBUTXO"
``` [3](#0-2) 

When `isValid` is `True`, `dijkstraSubUtxoTransition` calls `Shelley.updateUTxOStateNoFees`, which mechanically removes sub-tx inputs from the UTxO and adds sub-tx outputs — with no conservation check: [4](#0-3) 

Sub-transaction inputs are validated to exist in `originalUtxo`: [5](#0-4) 

But there is no check that the sub-transaction's output value ≤ its input value. The `DijkstraSubTxBodyRaw` record confirms sub-transactions carry their own `dstbrOutputs` and `dstbrWithdrawals` fields that are never included in the top-level conservation equation: [6](#0-5) 

The `validateBatchWithdrawals` check in the top-level rule does aggregate withdrawals across sub-transactions, demonstrating that batch-level aggregation is possible — but no analogous aggregation exists for UTxO value conservation: [7](#0-6) 

---

### Impact Explanation

**Critical. Direct creation of ADA through an invalid ledger state transition.**

A sub-transaction can consume inputs worth X lovelace from `originalUtxo` and produce outputs worth X + Y lovelace (Y > 0). The top-level conservation check passes because it only balances the top-level body. The sub-transaction's outputs are committed to the UTxO state via `updateUTxOStateNoFees` without any conservation enforcement. The net ledger effect is Y lovelace created from nothing, violating the fundamental preservation-of-value invariant that underpins all Cardano ledger eras.

---

### Likelihood Explanation

Any unprivileged transaction sender can submit a Dijkstra-era transaction containing sub-transactions. No special role, key, or governance threshold is required. The attacker only needs:
1. A UTxO input to use as the sub-transaction's spend input (to satisfy `validateBadInputsUTxO`).
2. A crafted sub-transaction body whose `dstbrOutputs` total exceeds the value of `dstbrSpendInputs`.

The top-level transaction remains independently balanced (paying fees normally), so it passes all top-level checks. The sub-transaction passes all its own checks (valid inputs, minimum output size, network ID, etc.) while silently inflating the UTxO.

---

### Recommendation

Add a value conservation check inside `dijkstraSubUtxoTransition`, analogous to the top-level check, using the sub-transaction body and the UTxO state at the time of sub-transaction processing:

```haskell
-- In dijkstraSubUtxoTransition, after validateBadInputsUTxO:
runTest $ Shelley.validateValueNotConservedUTxO pp originalUtxo certState txBody
```

Alternatively, modify the top-level `validateValueNotConservedUTxO` call to aggregate consumed and produced values across all sub-transactions, similar to how `validateBatchWithdrawals` aggregates withdrawals across the batch.

---

### Proof of Concept

1. Attacker holds a UTxO entry `u₀` worth 2 ADA (enough to satisfy `validateBadInputsUTxO` and `validateOutputTooSmallUTxO`).
2. Attacker constructs a top-level Dijkstra transaction `txTop` that:
   - Spends a separate fee-covering input.
   - Embeds one sub-transaction `txSub` with:
     - `dstbrSpendInputs = {u₀}` (2 ADA)
     - `dstbrOutputs = [TxOut attacker_addr 1_000_000_000_000]` (1,000,000 ADA)
3. `dijkstraUtxoTransition` runs `validateValueNotConservedUTxO` on `txTop`'s body — passes, because `txTop` itself is balanced.
4. `dijkstraSubUtxoTransition` runs for `txSub`: all checks pass (`u₀` exists in `originalUtxo`, output meets minimum ADA, network IDs match). No conservation check is performed.
5. `updateUTxOStateNoFees` removes `u₀` and inserts the 1,000,000 ADA output into the UTxO.
6. Attacker now controls 1,000,000 ADA created from 2 ADA, with ~999,998 ADA conjured from nothing.

dijkstraSubUtxoTransition = do
  TRC (SubUtxoEnv slot pp certState originalUtxo (IsValid isValid), utxoState, stAnnTx) <-
    judgmentContext
  let tx = stAnnTx ^. txStAnnTxG

  let txBody = tx ^. bodyTxL

  runTest $ Allegra.validateOutsideValidityIntervalUTxO slot txBody

  sysSt <- liftSTS $ asks systemStart
  ei <- liftSTS $ asks epochInfo
  runTest $ Alonzo.validateOutsideForecast ei slot sysSt tx

  let allSizedOutputs = txBody ^. allSizedOutputsTxBodyF
  let allOutputs = fmap sizedValue allSizedOutputs
  runTest $ Alonzo.validateOutputTooBigUTxO pp allOutputs

  runTest $ Shelley.validateInputSetEmptyUTxO txBody

  let inputs = txBody ^. inputsTxBodyL
  let refInputs = txBody ^. referenceInputsTxBodyL
  runTest $ Shelley.validateBadInputsUTxO originalUtxo (inputs `Set.union` refInputs)
  runTest $ Shelley.validateBadInputsUTxO (utxosUtxo utxoState) inputs

  runTestOnSignal $ Shelley.validateOutputBootAddrAttrsTooBig allOutputs

  runTestOnSignal $ Babbage.validateOutputTooSmallUTxO pp allSizedOutputs

  netId <- liftSTS $ asks networkId
  runTestOnSignal $ Shelley.validateWrongNetwork netId allOutputs
  runTestOnSignal $ Shelley.validateWrongNetworkWithdrawal netId txBody
  runTestOnSignal $ validateWrongNetworkInDirectDeposit netId txBody
  runTestOnSignal $ Alonzo.validateWrongNetworkInTxBody netId txBody

  if isValid
    then do
      newState <-
        Shelley.updateUTxOStateNoFees
          pp
          utxoState
          txBody
          certState
          (utxosGovState utxoState)
          (tellEvent . TotalDeposits (hashAnnotated txBody))
          (\a b -> tellEvent $ TxUTxODiff a b)
      pure $ newState & utxosDonationL <>~ txBody ^. treasuryDonationTxBodyL
    else
      pure utxoState
