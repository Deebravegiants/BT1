### Title
Protocol Fee Incorrectly Charged on Rewards Accrued During Pause Period - (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle._updateRsETHPrice()` uses a `!protocolPaused` guard to skip protocol fee extraction when the system is paused. However, because `updateRSETHPrice()` is blocked by `whenNotPaused` during a pause, the TVL at pause time is never snapshotted into `rsETHPrice`. When the protocol is unpaused and `updateRSETHPrice()` is called, `protocolPaused` is now `false`, so the fee is charged on the **entire** TVL increase since the last price update — including all rewards that accrued during the pause period — contrary to the stated design intent.

### Finding Description

`LRTOracle._updateRsETHPrice()` computes the protocol fee as follows:

```solidity
bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

uint256 protocolFeeInETH = 0;
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
``` [1](#0-0) 

`previousTVL` is derived from the stored `rsETHPrice`:

```solidity
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);
``` [2](#0-1) 

`updateRSETHPrice()` — the public entry point — carries `whenNotPaused`, so it cannot be called while any of the three contracts is paused:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [3](#0-2) 

Neither `LRTDepositPool.pause()` nor `LRTWithdrawalManager.pause()` nor `LRTOracle._pause()` calls `_updateRsETHPrice()` before setting the paused flag:

```solidity
// LRTDepositPool
function pause() external onlyRole(LRTConstants.PAUSER_ROLE) {
    _pause();
}
``` [4](#0-3) 

```solidity
// LRTOracle internal pause (called on automatic price-drop trigger)
function _pause() internal {
    if (paused) return;
    paused = true;
    emit Paused(msg.sender);
}
``` [5](#0-4) 

The automatic pause path is particularly clear: when a price drop exceeds `pricePercentageLimit`, `_updateRsETHPrice()` pauses all three contracts and **returns early without updating `rsETHPrice`**:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
``` [6](#0-5) 

**Consequence**: `rsETHPrice` is frozen at its pre-pause value. Underlying EigenLayer positions continue earning staking rewards during the pause, so `totalETHInProtocol` grows. On the first `updateRSETHPrice()` call after unpausing, `protocolPaused = false`, and the fee is charged on `totalETHInProtocol − previousTVL`, which spans the entire interval from the last price update through the pause period and beyond.

### Impact Explanation

The protocol fee is minted as rsETH to the treasury, diluting all existing rsETH holders. By charging the fee on rewards that accrued during the pause — a period the code explicitly intends to be fee-free — rsETH holders lose yield they are entitled to. This constitutes **theft of unclaimed yield** (High impact per the allowed scope).

### Likelihood Explanation

Pauses are a normal and expected protocol event: they are triggered automatically by the oracle's downside-protection logic whenever the rsETH price drops beyond `pricePercentageLimit`, and can also be triggered manually by the `PAUSER_ROLE`. EigenLayer staking rewards accrue continuously regardless of the pause state. Every pause-then-unpause cycle silently mis-charges fees on the paused-period rewards. Likelihood: **Medium**.

### Recommendation

Mirror the Tapioca fix: call `_updateRsETHPrice()` (or the manager-accessible `updateRSETHPriceAsManager()`) **before** setting the paused flag in each `pause()` entry point. This snapshots `rsETHPrice` to the current value at pause time, so that `previousTVL` on the next post-unpause call reflects only rewards earned after the pause ended.

```diff
// LRTDepositPool
function pause() external onlyRole(LRTConstants.PAUSER_ROLE) {
+   // Snapshot the price so post-unpause fee is only on post-unpause rewards
+   address lrtOracleAddress = lrtConfig.getContract(LRTConstants.LRT_ORACLE);
+   ILRTOracle(lrtOracleAddress).updateRSETHPriceAsManager();
    _pause();
}
```

Similarly, in `LRTOracle._pause()` (called by the automatic price-drop path), invoke `_updateRsETHPrice()` before setting `paused = true` (guarding against re-entrancy with the existing `if (paused) return` check).

### Proof of Concept

1. Protocol is live; `updateRSETHPrice()` is called at block B0, setting `rsETHPrice = P0`.
2. At block B1, the rsETH price drops beyond `pricePercentageLimit`. `_updateRsETHPrice()` pauses all three contracts and returns **without** updating `rsETHPrice`. `rsETHPrice` remains `P0`.
3. During blocks B1–B2 (pause period), EigenLayer staking rewards accrue. `totalETHInProtocol` grows by `R_pause`.
4. At block B2, admin unpauses all three contracts.
5. At block B3, anyone calls `updateRSETHPrice()`:
   - `protocolPaused = false`
   - `previousTVL = rsethSupply × P0`
   - `totalETHInProtocol = previousTVL + R_pause + R_post`
   - `rewardAmount = R_pause + R_post`
   - Protocol fee = `(R_pause + R_post) × feeBPS / 10_000`
6. The fee on `R_pause` is charged even though the protocol was paused during that entire period. rsETH holders are diluted by `R_pause × feeBPS / 10_000` worth of rsETH that should not have been minted to the treasury.
