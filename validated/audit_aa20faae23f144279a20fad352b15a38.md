Audit Report

## Title
Unbounded Nested Iteration Over NDCs and EigenLayer Queued Withdrawals Causes Deposit DoS - (File: contracts/LRTDepositPool.sol, contracts/NodeDelegator.sol)

## Summary
Every call to `depositETH()` or `depositAsset()` triggers a nested loop through all registered `NodeDelegator` contracts, each of which makes an external call to EigenLayer's `DelegationManager.getQueuedWithdrawals()` and iterates over all returned withdrawals and their strategies. As the protocol scales, the gas cost per deposit grows unboundedly, eventually causing all deposits to revert when the block gas limit is exceeded.

## Finding Description
The full call chain is confirmed in code:

1. `depositETH()` / `depositAsset()` → `_beforeDeposit()` [1](#0-0)  → `_checkIfDepositAmountExceedesCurrentLimit()` [2](#0-1)  → `getTotalAssetDeposits()` [3](#0-2)  → `getAssetDistributionData()` / `getETHDistributionData()`. [4](#0-3) 

2. Both distribution functions loop over every NDC and call `getAssetUnstaking()` on each: [5](#0-4)  and [6](#0-5) 

3. `NodeDelegator.getAssetUnstaking()` makes an external call to `DelegationManager.getQueuedWithdrawals(address(this))` and then runs a nested loop over all returned `Withdrawal[]` and their `strategies[]`. [7](#0-6) 

There are no guards preventing this from consuming unbounded gas. `maxNodeDelegatorLimit` is initialized to 10 and is admin-settable upward. [8](#0-7)  EigenLayer's withdrawal queue is bounded only by `maxUncompletedWithdrawalCount`, which is a separate per-NDC limit that can accumulate across many NDCs simultaneously. No caching or snapshotting of the TVL computation exists in the deposit hot path.

## Impact Explanation
The total gas per deposit scales as **O(NDC_count × queued_withdrawals_per_NDC × strategies_per_withdrawal)**. Once this product is large enough to exceed the block gas limit (~30M gas on Ethereum), every call to `depositETH()` and `depositAsset()` reverts. This constitutes **Medium — Temporary freezing of funds**: deposits are blocked until operators complete enough pending EigenLayer withdrawals to reduce the queued withdrawal count below the gas threshold. The freeze is temporary because operators can resolve it, but it is fully reachable under normal protocol operation at scale.

## Likelihood Explanation
Any unprivileged depositor triggers this path on every deposit call — no special role or exploit is required. EigenLayer withdrawals have a mandatory delay of days to weeks before completion, so during periods of high unstaking activity, many withdrawals will be simultaneously queued across multiple NDCs. This is a normal operational state, not an edge case. The protocol is explicitly designed to support multiple NDCs, making the scaling condition expected rather than hypothetical.

## Recommendation
1. **Decouple TVL accounting from the deposit hot path**: Maintain an on-chain cached TVL variable updated only when withdrawals are queued or completed, rather than recomputing live across all NDCs and EigenLayer strategies on every deposit.
2. **Remove `getAssetUnstaking()` from deposit limit checks**: The deposit limit check in `_checkIfDepositAmountExceedesCurrentLimit` should use a snapshotted value rather than calling `getAssetUnstaking()` (which makes external calls and iterates over all queued withdrawals) on every deposit.
3. **Single-pass NDC iteration**: `getAssetDistributionData` and `getETHDistributionData` both iterate over all NDCs independently; a single pass accumulating all asset balances would halve the NDC iteration count.

## Proof of Concept
```
// Preconditions (normal protocol operation):
// - N NDCs registered (e.g., N=10, within maxNodeDelegatorLimit)
// - Each NDC has M queued EigenLayer withdrawals (e.g., M=50, within maxUncompletedWithdrawalCount)
// - Each withdrawal has S strategies (e.g., S=2)
//
// Trigger (any unprivileged user):
//   depositETH{value: 1 ether}(0, "")
//
// Gas consumed:
//   N * M external calls to DelegationManager.getQueuedWithdrawals()
//   + N * M * S iterations + strategy.sharesToUnderlyingView() calls
//   = 10 * 50 * 2 = 1000 external calls + loop overhead
//   ≈ 5M–30M gas depending on EigenLayer state size
//
// Result: transaction reverts with out-of-gas
//         all deposits frozen until operators complete pending withdrawals
//
// Foundry fork test plan:
//   1. Fork mainnet/testnet with EigenLayer deployed
//   2. Register N NodeDelegator contracts
//   3. Queue M withdrawals per NDC via requestWithdrawal()
//   4. Call depositETH{value: 1 ether}(0, "") and measure gas
//   5. Assert gas usage grows linearly with N*M and eventually exceeds block limit
```

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
