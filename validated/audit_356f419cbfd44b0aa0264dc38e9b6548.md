### Title
Missing `state_diff_commitment` Hash Verification in P2P Sync Client Allows Malicious Peer to Corrupt Node State Storage — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary
`StateDiffStreamBuilder::parse_data_for_block` validates only the *length* of received state diff chunks against the header's `state_diff_length` field. It never reads or verifies the `state_diff_commitment` (Poseidon hash) stored in the same header. A malicious P2P peer can serve state diff chunks whose total length matches `target_state_diff_len` but whose content is entirely different from the canonical state diff, causing the node to persist a fabricated state diff to storage with no error.

### Finding Description

`parse_data_for_block` opens the stored block header and extracts only `state_diff_length`: [1](#0-0) 

It then accumulates chunks until the length matches and performs three checks: length equality, empty-chunk rejection, and intra-diff key conflicts: [2](#0-1) 

The `state_diff_commitment` field — a Poseidon hash committed in the block header — is **never read** in this file (confirmed by grep: zero occurrences of `state_diff_commitment` in `state_diff.rs`). After passing the length check, the assembled `ThinStateDiff` is written unconditionally to storage: [3](#0-2) 

The header *does* carry `state_diff_commitment` — it is stored during header sync and used in block hash calculation: [4](#0-3) 

The hash function `calculate_state_diff_hash` is already available in the codebase and is used by the committer for exactly this purpose: [5](#0-4) 

### Impact Explanation

Once a fabricated state diff is written to storage, every RPC call that reads state for that block returns wrong values. `StateSync::get_storage_at` reads directly from the corrupted storage: [6](#0-5) 

This means `starknet_getStorageAt`, `starknet_getClassAt`, `starknet_call`, fee estimation, and simulation all operate on attacker-controlled state — returning authoritative-looking wrong values to clients. The corruption is permanent until the block is reverted.

### Likelihood Explanation

Any peer on the P2P network can serve state diff responses. The P2P sync client registers an unauthenticated SQMR protocol client for `Protocol::StateDiff`: [7](#0-6) 

No authentication, reputation gating, or commitment re-check prevents a malicious peer from substituting a same-length but different-content state diff. The attacker only needs to know the `state_diff_length` value for the target block (publicly available from the block header).

### Recommendation

After assembling `result` and before returning `Ok(Some((result, block_number)))`, read `state_diff_commitment` from the stored header and verify it:

```rust
let header = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("...");
let expected_commitment = header.state_diff_commitment
    .ok_or(P2pSyncClientError::OldHeaderInStorage {
        block_number,
        missing_field: "state_diff_commitment",
    })?;
let actual_commitment = calculate_state_diff_hash(&result);
if actual_commitment != expected_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffCommitment {
        block_number,
        expected: expected_commitment,
        actual: actual_commitment,
    }));
}
```

The `calculate_state_diff_hash` function already exists at `crates/starknet_api/src/block_hash/state_diff_hash.rs`: [8](#0-7) 

### Proof of Concept

1. Run the P2P sync client against a mock peer.
2. The mock peer sends a valid header for block N with `state_diff_length = K` and a real `state_diff_commitment = H`.
3. The mock peer then serves state diff chunks whose total `len()` equals `K` but whose storage diffs, nonces, or deployed contracts differ from the canonical state diff (so their Poseidon hash ≠ H).
4. Assert that `parse_data_for_block` returns `Ok(Some(...))` — it will, because only the length is checked.
5. Assert that the stored state diff in storage does not match the canonical one — it will match the attacker's fabricated version.
6. Call `get_storage_at` for a key modified by the attacker and observe the wrong value returned.
