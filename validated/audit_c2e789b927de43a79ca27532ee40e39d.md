Audit Report

## Title
Permissionless `updateRSETHPrice()` Allows Any Caller to Trigger Protocol-Wide Pause and Freeze Deposits/Withdrawals - (File: `contracts/LRTOracle.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is a public, permissionless function that internally invokes a downside-protection mechanism capable of pausing `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. Because no caller restriction exists beyond `whenNotPaused`, any external account can trigger a protocol-wide freeze at zero capital cost whenever the computed rsETH price falls below `highestRsethPrice` by more than `pricePercentageLimit`. The resulting pause blocks all user-facing deposits and withdrawals until an admin manually unpauses each contract.

## Finding Description

`updateRSETHPrice()` carries only a `whenNotPaused` guard and is callable by any address:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, the downside-protection block unconditionally pauses three contracts when the price-drop condition is met:

```solidity
// contracts/LRTOracle.sol L277-282
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

The condition is:

```solidity
// contracts/LRTOracle.sol L272-274
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
```

`highestRsethPrice` is a monotonically increasing all-time-high watermark, updated only when `newRsETHPrice > highestRsethPrice`:

```solidity
// contracts/LRTOracle.sol L293-296
if (newRsETHPrice > highestRsethPrice) {
    highestRsethPrice = newRsETHPrice;
}
```

`newRsETHPrice` is derived from live oracle prices and on-chain balances via `_getTotalEthInProtocol()`. Any real-world event that reduces protocol TVL — validator slashing, an LST oracle dip, or EigenLayer withdrawal delays — can push `newRsETHPrice` below the threshold.

`LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()` both require `PAUSER_ROLE`:

```solidity
// contracts/LRTDepositPool.sol L349-351
function pause() external onlyRole(LRTConstants.PAUSER_ROLE) {
    _pause();
}
// contracts/LRTWithdrawalManager.sol L347-349
function pause() external onlyRole(LRTConstants.PAUSER_ROLE) {
    _pause();
}
```

The `onlyRole` modifier resolves through `LRTConfigRoleChecker` against `lrtConfig`:

```solidity
// contracts/utils/LRTConfigRoleChecker.sol L19-25
modifier onlyRole(bytes32 role) {
    if (!IAccessControl(address(lrtConfig)).hasRole(role, msg.sender)) { ... }
    _;
}
```

For the auto-pause to function as designed, `LRTOracle` must hold `PAUSER_ROLE` in `lrtConfig`. This is the evident design intent — without it, the entire downside-protection block is dead code. The same role is used by `LRTConfig.pauseAll()`, confirming `PAUSER_ROLE` is the standard mechanism for cross-contract pausing. Given this, a production deployment is expected to grant `LRTOracle` the `PAUSER_ROLE`.

Once paused, `rsETHPrice` is **not updated** (the function returns early), and unpausing requires `onlyLRTAdmin` on each contract individually:

```solidity
// contracts/LRTOracle.sol L143-146
function unpause() external whenPaused onlyLRTAdmin {
    paused = false;
    emit Unpaused(msg.sender);
}
```

**Attack path:**
1. Protocol accumulates staking rewards; `highestRsethPrice` is set to e.g. `1.05e18`.
2. A validator slashing event or LST oracle dip causes `newRsETHPrice` to fall to `1.03e18` (~1.9% drop).
3. With `pricePercentageLimit = 1e16` (1%), the condition `diff (0.02e18) > threshold (1.05e16)` is satisfied.
4. Attacker calls `updateRSETHPrice()` — no role, no capital required.
5. `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` are all paused atomically.
6. All deposits (`depositETH`, `depositAsset`) and withdrawals (`initiateWithdrawal`, `completeWithdrawal`, `instantWithdrawal`) revert until admin manually unpauses each contract.

## Impact Explanation

All user-facing deposit and withdrawal functions on `LRTDepositPool` and `LRTWithdrawalManager` are blocked for the duration of the pause. Users with pending withdrawal requests cannot complete them; new depositors cannot enter. This constitutes **temporary freezing of funds**, matching the allowed Medium impact category. If admin response is delayed (multisig latency, key-holder unavailability), the freeze extends proportionally. The attack is repeatable: once unpaused, the attacker can call `updateRSETHPrice()` again the next time the price condition is met.

## Likelihood Explanation

The condition is realistic and requires no attacker capital. LST oracle prices fluctuate naturally, EigenLayer slashing events reduce TVL, and `highestRsethPrice` never decreases. An attacker monitoring on-chain state can call `updateRSETHPrice()` the moment the condition is satisfied. The tighter `pricePercentageLimit` is configured, the easier the condition is to satisfy. The attack costs only gas.

## Recommendation

1. **Restrict `updateRSETHPrice()`** to authorized callers (e.g., `onlyLRTManager` or a dedicated keeper role), mirroring the existing `updateRSETHPriceAsManager()` pattern. The auto-pause path must not be reachable by arbitrary external callers.
2. **Separate the pause trigger from the price update**: require an explicit, role-gated call to activate the pause rather than embedding it inside a public price-update function.
3. **Add a confirmation window or cooldown** before the pause fires, so a transient oracle dip does not immediately freeze the protocol.

## Proof of Concept

```
Preconditions:
  LRTOracle holds PAUSER_ROLE in lrtConfig (required for auto-pause to function)
  highestRsethPrice = 1.05e18
  pricePercentageLimit = 1e16 (1%)
  Current totalETHInProtocol reflects a slashing event → newRsETHPrice = 1.03e18

Attack:
  attacker.call(lrtOracle.updateRSETHPrice())

  Inside _updateRsETHPrice():
    diff = 1.05e18 - 1.03e18 = 0.02e18
    threshold = pricePercentageLimit.mulWad(highestRsethPrice)
              = 1e16 * 1.05e18 / 1e18 = 1.05e16
    isPriceDecreaseOffLimit = (0.02e18 > 1.05e16) → true

    lrtDepositPool.pause()       ← LRTDepositPool frozen
    withdrawalManager.pause()    ← LRTWithdrawalManager frozen
    _pause()                     ← LRTOracle frozen
    return                       ← rsETHPrice NOT updated

Result:
  depositETH(), depositAsset(), initiateWithdrawal(), completeWithdrawal(),
  instantWithdrawal() all revert until admin calls unpause() on each contract.
  Cost to attacker: gas only.

Foundry test plan:
  1. Fork mainnet; deploy/configure LRTOracle with PAUSER_ROLE granted to it.
  2. Set pricePercentageLimit = 1e16.
  3. Manipulate mock oracle prices to produce newRsETHPrice < highestRsethPrice * 0.99.
  4. Call lrtOracle.updateRSETHPrice() from an unprivileged address.
  5. Assert lrtDepositPool.paused() == true, withdrawalManager.paused() == true, lrtOracle.paused() == true.
  6. Assert depositETH() reverts with ContractPaused/Pausable: paused.
```
