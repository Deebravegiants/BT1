Audit Report

## Title
Missing Caller Authentication in `MerkleDistributor.claim()` Enables Forced Fee Extraction from Any User - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary
`MerkleDistributor.claim()` accepts an arbitrary `account` parameter but never verifies `msg.sender == account`. Any caller with a valid merkle proof (published off-chain) can trigger a claim on behalf of any user, forcing up to 10% of their unclaimed yield to be irrevocably transferred to `protocolTreasury` without the user's consent. The sibling contract `KernelMerkleDistributor` explicitly enforces `if (account != msg.sender) revert Unauthorized()`, confirming the omission is unintentional.

## Finding Description
`MerkleDistributor.claim()` performs no identity check on the caller:

```solidity
// contracts/utils/MerkleDistributor/MerkleDistributor.sol L97-L106
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
    // ← no msg.sender == account check
```

After proof verification, the fee is unconditionally deducted and sent to `protocolTreasury`:

```solidity
// L138-L144
uint256 fee = (claimableAmount * feeInBPS) / 10_000;
uint256 amountToSend = claimableAmount - fee;
IERC20(token).safeTransfer(account, amountToSend);
IERC20(token).safeTransfer(protocolTreasury, fee);
```

`MAX_FEE_IN_BPS = 1000` caps `feeInBPS` at 10%, adjustable by the owner via `setFeeInBPS()`. The fee is applied regardless of who calls `claim()`.

By contrast, `KernelMerkleDistributor._processClaim()` at L311-313 explicitly enforces:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

The exploit path is straightforward: merkle proofs are published off-chain for self-claiming. An attacker iterates the distribution list and calls `claim(index, victim, cumulativeAmount, proof)` for every eligible account. Each call deducts the fee from the victim's yield and sends it to `protocolTreasury`. The victim's `lastClaimedIndex` and `cumulativeAmount` are updated, preventing re-claim until the next root is published.

## Impact Explanation
Each forced claim extracts up to 10% of the victim's unclaimed yield and transfers it to `protocolTreasury` without the victim's consent. The victim receives only `claimableAmount - fee` instead of `claimableAmount`. This is a concrete, irreversible reduction in unclaimed yield for every user in the distribution. This maps to **High — Theft of unclaimed yield**.

## Likelihood Explanation
- Merkle proofs are public by design; no privileged access is required.
- No access control, rate limiting, or economic barrier prevents mass forced-claiming.
- The attacker needs only to iterate the published distribution list and submit one transaction per user.
- The attack is repeatable at every merkle root update.
- Likelihood is **High**.

## Recommendation
Add the same caller identity check already present in `KernelMerkleDistributor`:

```solidity
function claim(...) external override whenNotPaused {
    if (account != msg.sender) revert Unauthorized();
    ...
}
```

If claiming on behalf of others is intentionally supported (e.g., for relayers), the fee should be waived when `msg.sender != account`, or explicit on-chain delegation should be required.

## Proof of Concept
1. Protocol publishes merkle root with Alice's leaf: `(index=1, account=Alice, cumulativeAmount=1000e18)` and proof `P`. `feeInBPS = 1000` (10%).
2. Attacker calls:
   ```solidity
   merkleDistributor.claim(1, Alice, 1000e18, P);
   ```
3. Contract verifies proof (valid), computes `claimableAmount = 1000e18`, `fee = 100e18`, `amountToSend = 900e18`.
4. `900e18` tokens transferred to Alice; `100e18` tokens transferred to `protocolTreasury`.
5. `userClaims[Alice].lastClaimedIndex = 1`; Alice cannot re-claim until the next root.
6. Alice loses `100e18` tokens (10% of her yield) she never consented to pay as a fee.

**Foundry test plan:** Deploy `MerkleDistributor`, initialize with `feeInBPS = 1000`, construct a merkle tree with one leaf for `alice`, publish the root, then call `claim(index, alice, amount, proof)` from `attacker` address. Assert `alice` receives `amount * 0.9` and `protocolTreasury` receives `amount * 0.1`, confirming forced fee extraction by an unprivileged third party.
