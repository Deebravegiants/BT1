### Title
Transfer Initiation Without Destination-Chain Token Deployment Check Causes Irreversible Fund Lock — (`near/omni-bridge/src/lib.rs`, `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

Both the NEAR bridge's `init_transfer` and the EVM bridge's `initTransfer` accept and immediately consume user funds (burn or lock) without verifying that the transferred token is registered/deployed on the destination chain. If the token is absent on the destination side, the relayer's subsequent `sign_transfer` call permanently fails, and no cancel or refund path exists. Funds are irreversibly frozen.

---

### Finding Description

**NEAR → Foreign-chain path**

`init_transfer` (called via `ft_on_transfer`) accepts any token for any destination chain with only two checks: the recipient chain is not NEAR, and the fee is less than the amount. [1](#0-0) 

It then immediately calls `init_transfer_internal`, which burns the tokens (if they are bridge-deployed) or locks them (if NEAR-native) before any destination-chain validation occurs. [2](#0-1) 

Later, when the relayer calls `sign_transfer`, it attempts to resolve the token address on the destination chain: [3](#0-2) 

If the token has never been deployed on the destination chain (i.e., `deploy_token` was never called for that chain), `get_token_address` returns `None` and the call panics with `FailedToGetTokenAddress`. The transfer record remains in `pending_transfers` indefinitely.

There is no `cancel_transfer`, `refund_transfer`, or any user-accessible function to recover funds from a stuck pending transfer. The only removal paths are `remove_transfer_message` (called inside `sign_transfer_callback` when fee is zero, which requires a successful MPC signature) and `claim_fee_callback` (which also requires a completed destination-chain proof). Both are unreachable when the token is unregistered. [4](#0-3) 

**EVM → NEAR path**

`initTransfer` on the EVM bridge accepts any `tokenAddress` — including arbitrary ERC20s not registered in `nearToEthToken` — and locks or burns them immediately: [5](#0-4) 

When the relayer submits the proof to NEAR's `fin_transfer_callback`, the contract looks up token decimals: [6](#0-5) 

If the token is not registered on NEAR, this panics with `TokenDecimalsNotFound`. The EVM tokens remain locked in the bridge contract with no recovery path for the user.

---

### Impact Explanation

**For bridge-deployed (wrapped) tokens on NEAR:** `burn_tokens_if_needed` destroys the tokens immediately. They cannot be recovered even by the DAO, because the underlying supply is gone. [7](#0-6) 

**For NEAR-native tokens:** Tokens are transferred into the bridge contract via `ft_transfer_call` and the `locked_tokens` counter is incremented. The DAO could theoretically use `transfer_token_as_dao` to move them out, but no user-accessible recovery exists, and the `locked_tokens` accounting would be left inconsistent.

**For EVM-locked tokens:** Tokens sit in the `OmniBridge` contract with no user-callable exit. The `initTransfer` nonce is consumed, so the transfer cannot be re-submitted.

This matches the **Critical — Irreversible fund lock** impact category.

---

### Likelihood Explanation

The scenario is reachable whenever a token has been deployed on one chain but not yet on another — a normal operational state during any new token onboarding. A user who sees the token on chain A and attempts to bridge it to chain B (where `deploy_token` has not yet been called) will lose funds permanently. The protocol provides no UI-level or contract-level guard against this.

---

### Recommendation

1. **Pre-flight check in `init_transfer`:** Before burning or locking tokens, verify that `get_token_address(destination_chain, token_id)` returns `Some`. Revert (return the full amount) if the token is not registered on the destination chain.

2. **Pre-flight check in `initTransfer` (EVM):** Verify that `ethToNearToken[tokenAddress]` is non-empty before accepting funds.

3. **User-accessible cancellation:** Add a `cancel_transfer` function that allows the original sender to reclaim funds from a pending transfer that has not yet been signed, subject to a timeout or explicit relayer acknowledgement of failure.

---

### Proof of Concept

1. Token `foo.near` exists on NEAR but `deploy_token` has never been called for the Solana chain.
2. User calls `ft_transfer_call` on `foo.near` with `receiver_id = omni-bridge.near` and `msg = {"InitTransfer": {"recipient": "sol:...", "fee": "0", ...}}`.
3. `ft_on_transfer` → `init_transfer` → `init_transfer_internal`: tokens are locked in the bridge; `pending_transfers` entry is created; `U128(0)` is returned (tokens consumed).
4. Relayer calls `sign_transfer(transfer_id, ...)`.
5. `get_token_address(ChainKind::Sol, "foo.near")` returns `None` → panic `ERR_FAILED_TO_GET_TOKEN_ADDRESS`.
6. No further call can complete or cancel the transfer. Funds are permanently frozen. [3](#0-2) [8](#0-7)

    fn init_transfer_internal(
        &mut self,
        transfer_message: TransferMessage,
        storage_owner: AccountId,
    ) -> U128 {
        let required_storage_balance = self
            .add_transfer_message(transfer_message.clone(), storage_owner.clone())
            .saturating_add(NearToken::from_yoctonear(transfer_message.fee.native_fee.0));

        if self
            .try_update_storage_balance(
                storage_owner,
                required_storage_balance,
                NearToken::from_yoctonear(0),
            )
            .is_err()
        {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        if let OmniAddress::Near(token_id) = transfer_message.token.clone() {
            self.burn_tokens_if_needed(token_id.clone(), transfer_message.amount);

            self.lock_tokens_if_needed(
                transfer_message.get_destination_chain(),
                &token_id,
                transfer_message.amount.0,
            );
        } else {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        env::log_str(&OmniBridgeEvent::InitTransferEvent { transfer_message }.to_log_string());
        U128(0)
    }

    fn remove_transfer_message(&mut self, transfer_id: TransferId) -> TransferMessage {
        let storage_usage = env::storage_usage();
        let transfer = self
            .pending_transfers
            .remove(&transfer_id)
            .map(storage::TransferMessageStorage::into_main)
            .near_expect(BridgeError::TransferNotExist);

        let refund =
            env::storage_byte_cost().saturating_mul((storage_usage - env::storage_usage()).into());

        if let Some(mut storage) = self.accounts_balances.get(&transfer.owner) {
            storage.available = storage.available.saturating_add(refund);
            self.accounts_balances.insert(&transfer.owner, &storage);
        }

        transfer.message
    }

    function initTransfer(
        address tokenAddress,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
        currentOriginNonce += 1;
        if (fee >= amount) {
            revert InvalidFee();
        }

        uint256 extensionValue;
        if (tokenAddress == address(0)) {
            if (fee != 0) {
                revert InvalidFee();
            }
            extensionValue = msg.value - amount - nativeFee;
        } else {
            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    customMinters[tokenAddress],
                    amount
                );
                ICustomMinter(customMinters[tokenAddress]).burn(
                    tokenAddress,
                    amount
                );
            } else if (isBridgeToken[tokenAddress]) {
                BridgeToken(tokenAddress).burn(msg.sender, amount);
            } else {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    address(this),
                    amount
                );
            }
        }

        initTransferExtension(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message,
            extensionValue
        );

        emit BridgeTypes.InitTransfer(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message
        );
    }
