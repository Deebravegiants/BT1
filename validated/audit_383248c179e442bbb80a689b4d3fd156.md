### Title
Expired Attestation Not Checked in `is_caller_an_attested_participant`, Allowing Unverified Nodes to Participate in Critical Protocol Operations — (`crates/contract/src/tee/tee_state.rs`)

---

### Summary

`is_caller_an_attested_participant` only verifies that a stored attestation *exists* and that the account/key fields match the caller. It never calls `reverify_participants` (which checks expiry and allowlist membership). As a result, a node whose TEE attestation has expired can still pass the `assert_caller_is_attested_participant_and_protocol_active` gate and invoke every critical protocol method it guards — including `vote_reshared`, `vote_pk`, `respond*`, `start_keygen_instance`, and `start_reshare_instance` — without a valid, current TEE attestation.

---

### Finding Description

The contract enforces TEE participation through two distinct code paths:

**Path A — expiry-aware re-verification** (`reverify_participants` / `reverify_and_cleanup_participants`): [1](#0-0) 

This path calls `re_verify` on the stored `VerifiedAttestation`, which checks the expiry timestamp, the current allowed MPC image hash list, the launcher compose hash list, and the OS measurements.

**Path B — presence-only check** (`is_caller_an_attested_participant`): [2](#0-1) 

This path only confirms that:
1. The caller's account ID is in the participant set.
2. A stored attestation exists for that participant's TLS key.
3. The stored `account_id` and `account_public_key` match the caller.

It never calls `reverify_participants` or `re_verify`. Attestation expiry, image-hash revocation, and measurement changes are completely invisible to this check.

`assert_caller_is_attested_participant_and_protocol_active` exclusively uses Path B: [3](#0-2) 

And this guard is the sole TEE gate for every key-protocol method: [4](#0-3) 

The `re_verify` logic that Path A relies on correctly rejects expired attestations: [5](#0-4) 

The default attestation lifetime is `DEFAULT_EXPIRATION_DURATION_SECONDS` (7 days). After that window, `re_verify` returns an error, but `is_caller_an_attested_participant` never invokes it.

---

### Impact Explanation

A node whose Dstack TDX attestation has expired — meaning its TEE certificate chain is no longer valid — retains the ability to:

- **Cast `vote_reshared` votes** to complete a key resharing and influence the new participant set.
- **Call `respond` / `respond_ckd` / `respond_verify_foreign_tx`** to submit signing or CKD responses, contributing to threshold operations without a verified TEE.
- **Call `start_keygen_instance` / `start_reshare_instance`** as leader to initiate new protocol rounds.
- **Cast `vote_abort_key_event_instance`** to abort in-progress key events.

The TEE attestation requirement exists precisely to ensure that key-share material is only handled inside a verified confidential compute environment. Allowing an expired-attestation node to participate in `respond*` calls breaks this invariant: the node may be running outside a TEE (e.g., after a migration, a TEE failure, or a deliberate downgrade) and its key-share operations are no longer hardware-attested. This maps to the allowed impact class: **participant-state and contract execution-flow manipulation that breaks production safety/accounting invariants**.

---

### Likelihood Explanation

Attestations expire after 7 days. Any node that:
- Fails to renew its attestation (PCCS connectivity loss, TEE hardware failure, image-hash rotation that the node hasn't adopted),
- Is intentionally kept running past its attestation window, or
- Has its image hash removed from the allowlist (which causes `re_verify` to fail but `is_caller_an_attested_participant` to still pass),

…will silently retain full protocol participation rights. The `verify_tee` function can trigger resharing to evict such nodes, but it is a separate, externally-triggered call — not an inline check. Between `verify_tee` invocations, the expired node participates freely. The window is bounded only by how frequently any participant calls `verify_tee`.

---

### Recommendation

`is_caller_an_attested_participant` should call `reverify_participants` (or inline the equivalent `re_verify` call) after confirming the attestation exists, so that expiry and allowlist membership are checked at the same point as identity:

```rust
pub(crate) fn is_caller_an_attested_participant(
    &self,
    participants: &Participants,
) -> Result<(), AttestationCheckError> {
    // ... existing identity checks ...

    // Add: re-verify the stored attestation is still current
    let node_id = &attestation.node_id;
    match self.reverify_participants(node_id, tee_upgrade_deadline_duration) {
        TeeQuoteStatus::Valid => Ok(()),
        TeeQuoteStatus::Invalid(reason) => Err(AttestationCheckError::AttestationExpiredOrInvalid(reason)),
    }
}
```

This mirrors the pattern used by `reverify_and_cleanup_participants` and closes the gap between the two verification paths.

---

### Proof of Concept

1. Participant P holds a valid attestation at time T=0 (expiry at T=7d).
2. At T=7d+1s, P's attestation is expired. `re_verify` would return `Err(VerificationError::Custom("The attestation expired..."))`.
3. No one calls `verify_tee` to trigger resharing.
4. P calls `vote_reshared(key_event_id)`.
5. `assert_caller_is_attested_participant_and_protocol_active` calls `is_caller_an_attested_participant`.
6. `is_caller_an_attested_participant` finds P's stored attestation, confirms account ID and key match — **returns `Ok(())`** without checking expiry.
7. P's vote is counted toward the resharing threshold, despite P running with an unverified TEE.

The existing unit test `test_re_verify_rejects_expired_attestation` confirms that `reverify_participants` correctly rejects expired attestations: [6](#0-5) 

But no equivalent test exists for `is_caller_an_attested_participant` with an expired attestation, confirming the gap.

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,

        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);

            &allowed_measurements,

            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

impl VerifiedAttestation {
    pub fn re_verify(
        &self,
        timestamp_seconds: u64,
        allowed_mpc_docker_image_hashes: &[NodeImageHash],
        allowed_launcher_docker_compose_hashes: &[LauncherDockerComposeHash],
        allowed_measurements: &[ExpectedMeasurements],
    ) -> Result<(), VerificationError> {
        match self {
            Self::Dstack(ValidatedDstackAttestation {
                mpc_image_hash,
                launcher_compose_hash,
                expiry_timestamp_seconds: expiration_timestamp_seconds,
                measurements,
            }) => {
                let attestation_has_expired = *expiration_timestamp_seconds < timestamp_seconds;

                if attestation_has_expired {
                    return Err(VerificationError::Custom(format!(
                        "The attestation expired at t = {:?}, time_now = {:?}",
                        expiration_timestamp_seconds, timestamp_seconds
                    )));
                }

                let () = verify_mpc_hash(mpc_image_hash, allowed_mpc_docker_image_hashes)?;
                let () = verify_launcher_compose_hash(
                    launcher_compose_hash,
                    allowed_launcher_docker_compose_hashes,
                )?;

                verify_measurements(measurements, allowed_measurements)?;

                Ok(())
            }
            Self::Mock(mock_attestation) => mock_attestation.verify_constraints(
                timestamp_seconds,
                allowed_mpc_docker_image_hashes,
                allowed_launcher_docker_compose_hashes,
                allowed_measurements,
            ),
        }
    }
