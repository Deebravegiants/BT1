### Title
Sub-Transaction DRep Script Voter Bypasses Script Execution via Cross-Level Witness Aggregation — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxow.hs`)

---

### Summary

In the Dijkstra era, a DRep registered with a Plutus script credential can cast governance votes inside a sub-transaction (`DijkstraSubTx`) with an empty witness set (`dstWits = mempty`). The DRep script is never executed because the sub-level `SUBUTXOW` rule's `hasExactSetOfRedeemers` check only requires redeemers for scripts that appear in the sub-tx's own `scriptsProvided`, and the top-level `UTXOW` rule's `babbageMissingScripts` check is satisfied by placing the script in the top-level tx's `scriptTxWitsL`. No rule ever executes the script in the context of the vote.

---

### Finding Description

**Step 1 — `scriptsNeeded` for the sub-tx includes the DRep script.**

`getDijkstraScriptsNeeded` calls `getConwayScriptsNeeded`, which builds `votingScriptsNeeded` by iterating over `votingProceduresTxBodyL` and extracting `credScriptHash` for `DRepVoter (ScriptHashObj sh)`: [1](#0-0) 

So `dsastScriptsNeeded` for the sub-tx contains `(VotingPurpose ..., drepScript)`.

**Step 2 — `scriptsProvided` for the sub-tx is empty.**

`scriptsProvidedDijkstraStAnnTx` for a `DijkstraStAnnSubTx` returns `dsastScriptsProvided`, which is computed by `getBabbageScriptsProvided utxo subTx` — only the sub-tx's own witness set and reference inputs: [2](#0-1) 

With `dstWits = mempty` and no reference inputs containing `drepScript`, `dsastScriptsProvided` is empty.

**Step 3 — `hasExactSetOfRedeemers` in `SUBUTXOW` requires a redeemer only if the script is in `scriptsProvided`.**

The critical guard in `hasExactSetOfRedeemers`:

```haskell
redeemersNeeded =
  [ ...
  | (sp, sh) <- scriptsNeeded
  , Just script <- [Map.lookup sh scriptsProvided]   -- ← gating condition
  , not (isNativeScript script)
  ]
``` [3](#0-2) 

Because `drepScript` is not in the sub-tx's `scriptsProvided`, `Map.lookup drepScript {}` returns `Nothing`, so no redeemer is required and no Plutus execution is triggered.

**Step 4 — `SUBUTXOW` has no `babbageMissingScripts` check.**

`dijkstraSubUtxowTransition` runs `validateFailedBabbageScripts`, `validateNeededWitnesses`, `missingRequiredDatums`, `checkScriptIntegrityHash`, `hasExactSetOfRedeemers`, and `validateGuardDatums` — but **no** `babbageMissingScripts`: [4](#0-3) 

Compare with the top-level `dijkstraUtxowTransition`, which explicitly runs the aggregated check: [5](#0-4) 

**Step 5 — The top-level `babbageMissingScripts` is satisfied by placing `drepScript` in the top-level tx's `scriptTxWitsL`.**

`witnessScripts` is the union of scripts from the top-level tx AND all sub-txs: [6](#0-5) 

`allScriptHashesNeeded` includes `drepScript` (from the sub-tx's voting procedure). Since `drepScript` is in `witnessScripts` (from the top-level tx's `scriptTxWitsL`), `babbageMissingScripts` passes.

**Step 6 — The top-level `hasExactSetOfRedeemers` does not require a redeemer for `drepScript`.**

The top-level check uses `topScriptsNeeded` (only the top-level tx's own scripts needed), not `allScriptHashesNeeded`: [7](#0-6) 

Since the DRep vote is in the sub-tx, `drepScript` is not in `topScriptsNeeded`. No redeemer is required in the top-level tx.

**Step 7 — `validateNeededWitnesses` in `SUBUTXOW` does not catch script credentials.**

`getConwayWitsVKeyNeeded` / `voterWitnesses` only inserts a key-hash witness for `DRepVoter (KeyHashObj kh)`, not for `DRepVoter (ScriptHashObj sh)`: [8](#0-7) 

**Net result:** `drepScript` is present in the batch (satisfying `babbageMissingScripts`) but is never looked up in any `scriptsProvided` map at the sub-tx level, so `hasExactSetOfRedeemers` never demands a redeemer for it, and the Plutus evaluator is never invoked. The vote is recorded by `conwayGovTransition` without any script authorization.

---

### Impact Explanation

**Critical — Unauthorized governance action enacted.**

A DRep with a Plutus script credential can cast votes on any governance action (parameter changes, treasury withdrawals, hard-fork initiations, committee updates, constitutional changes) without the script's authorization logic ever being evaluated. The script may encode conditions such as "only vote Yes on proposals with a specific anchor hash" or "only vote when a multisig quorum approves" — all of these conditions are silently bypassed. The vote is permanently recorded in `gasDRepVotes` and counts toward ratification thresholds.

---

### Likelihood Explanation

The path is concrete and locally testable:

1. Register a DRep with `ScriptHashObj drepScript` (requires script execution once, at registration).
2. Construct a top-level tx with `drepScript` in `scriptTxWitsL` and no redeemers.
3. Embed a sub-tx with `DRepVoter (ScriptHashObj drepScript)` vote and `dstWits = mempty`.
4. Submit. All validation passes; the vote is recorded; the script is never run.

No privileged access, no key leakage, no governance majority, and no third-party compromise is required beyond having registered the DRep.

---

### Recommendation

`dijkstraSubUtxowTransition` must enforce that every script hash in `scriptHashesNeeded` for the sub-tx is either present in the sub-tx's own `scriptsProvided` (witness set or reference inputs) or explicitly delegated to the top-level tx via a declared mechanism. Concretely:

- Add a `babbageMissingScripts`-equivalent check inside `dijkstraSubUtxowTransition` scoped to the sub-tx's own `scriptsProvided` and reference inputs, **or**
- Change `hasExactSetOfRedeemers` in the sub-tx context to use the aggregated `scriptsProvided` (top-level + sub-tx) when determining which needed scripts are Plutus scripts requiring redeemers, while still requiring the redeemer to be present in the sub-tx's own redeemer map.

The root cause is the asymmetry: `babbageMissingScripts` is aggregated across levels (intentionally), but `hasExactSetOfRedeemers` uses only the sub-tx's own `scriptsProvided`, creating a gap where a needed Plutus script is "present" at the batch level but never executed at any level.

---

### Proof of Concept

```
1. Register drepScript (a Plutus V3 script that always fails) as a DRep:
     ConwayRegDRep (ScriptHashObj drepScript) deposit SNothing
   (This requires executing drepScript once, at registration.)

2. Build top-level tx:
     dtWits = mempty & scriptTxWitsL .~ Map.singleton drepScriptHash drepScript
     dtBody = mkBasicTxBody
                & subTransactionsTxBodyL .~ OMap.singleton subTxId subTx

3. Build sub-tx:
     dstWits = mempty   -- no scripts, no redeemers
     dstBody = mkBasicTxBody
                 & votingProceduresTxBodyL .~
                     VotingProcedures (Map.singleton
                       (DRepVoter (ScriptHashObj drepScriptHash))
                       (Map.singleton govActionId (VotingProcedure VoteYes SNothing)))

4. Submit top-level tx.

5. Assert: govActionId's gasDRepVotes contains ScriptHashObj drepScriptHash → VoteYes.
   Assert: drepScript was never evaluated (it always fails, yet no script failure occurred).
```

The vote is recorded. The always-failing script is never run.

dijkstraSubUtxowTransition = do
  TRC (env@(SubUtxoEnv _ pp certState originalUtxo _), utxoState, stAnnTx) <-
    judgmentContext
  let tx = stAnnTx ^. txStAnnTxG
      txBody = tx ^. bodyTxL
      witsKeyHashes = keyHashWitnessesTxWits (tx ^. witsTxL)
      scriptsProvided = scriptsProvidedStAnnTx stAnnTx

  {- ∀[ (vk , σ) ∈ vKeySigs ] isSigned vk (txidBytes txId) σ -}
  runTestOnSignal $ Shelley.validateVerifiedWits tx

  let scriptsNeeded = scriptsNeededStAnnTx stAnnTx
      scriptHashesNeeded = getScriptsHashesNeeded scriptsNeeded

  {- ∀[ s ∈ p1ScriptsNeeded ] validP1Script vKeyHashesProvided txVldt s -}
  runTest $ Babbage.validateFailedBabbageScripts tx scriptsProvided scriptHashesNeeded

  {- vKeyHashesNeeded ⊆ vKeyHashesProvided -}
  runTest $ Shelley.validateNeededWitnesses witsKeyHashes certState originalUtxo txBody

  {- dataHashesNeeded ⊆ mapˢ hash dataProvided -}
  runTest $ Alonzo.missingRequiredDatums scriptsProvided originalUtxo tx

  {- txADhash ≡ map hash txAuxData -}
  runTestOnSignal $ Shelley.validateMetadata pp tx

  let scriptIntegrity = mkScriptIntegrity pp tx (plutusLanguagesUsedStAnnTx stAnnTx)
  runTest $ Alonzo.checkScriptIntegrityHash tx pp scriptIntegrity

  runTest $ Alonzo.hasExactSetOfRedeemers tx scriptsProvided scriptsNeeded

  runTest $
    Babbage.validateScriptsWellFormedTxOuts
      pp
      (tx ^. witsTxL . scriptTxWitsL)
      (tx ^. bodyTxL . outputsTxBodyL)

  runTest $ validateGuardDatums scriptsProvided txBody

  trans @(EraRule "SUBUTXO" era) $ TRC (env, utxoState, stAnnTx)
