### Title
Tokens Permanently Locked When `normalize_amount` Rounds to Zero Before `sign_transfer` Validation - (`near/omni-bridge/src/lib.rs`)

### Summary

`init_transfer_internal` locks or burns user tokens before any decimal-normalization check is performed. The zero-amount guard exists only in `sign_transfer`, which is called later by a relayer. If a user sends an amount smaller than the decimal-normalization divisor (`10^(origin_decimals − decimals)`), their tokens are irreversibly locked/burned while `sign_transfer` will always revert, leaving the transfer permanently stuck with no cancel or refund path.

### Finding Description

The Omni Bridge normalizes token amounts when crossing chains with different decimal precisions. The normalization uses floor division:

```rust
fn normalize_amount(amount: u128, decimals: Decimals) -> u128 {
    let diff_decimals: u32 = (decimals.origin_decimals - decimals.decimals).into();
    amount / (10_u128.pow(diff_decimals))
}
``` [1](#0-0) 

The guard against a zero normalized amount exists only in `sign_transfer`:

```rust
let amount_to_transfer = Self::normalize_amount(
    transfer_message.amount_without_fee().near_expect(BridgeError::InvalidFee),
    decimals,
);
require!(
    amount_to_transfer > 0,
    BridgeError::InvalidAmountToTransfer.as_ref()
);
``` [2](#0-1) 

However, by the time `sign_transfer` is called, the tokens have already been locked or burned inside `init_transfer_internal`:

```rust
if let OmniAddress::Near(token_id) = transfer_message.token.clone() {
    self.burn_tokens_if_needed(token_id.clone(), transfer_message.amount);
    self.lock_tokens_if_needed(
        transfer_message.get_destination_chain(),
        &token_id,
        transfer_message.amount.0,
    );
}
``` [3](#0-2) 

`init_transfer_internal` returns `U128(0)` on success, which tells the NEP-141 `ft_transfer_call` machinery to keep the tokens in the bridge contract (no refund). The only refund paths inside `init_transfer_internal` are storage-balance failures, not decimal-normalization failures. [4](#0-3) 

There is no user-callable cancel or refund function for a stuck `TransferMessage`. The internal helper is named `remove_transfer_message_without_refund`, confirming no refund is issued.

### Impact Explanation

**Irreversible fund lock** — any user who initiates a NEAR→EVM transfer with an amount smaller than `10^(origin_decimals − decimals)` will have their tokens permanently locked or burned. For a token registered with `origin_decimals = 24` and `decimals = 18` (a common NEAR token configuration), any transfer amount below `1_000_000` (10^6 yocto-units) normalizes to zero. The transfer message is stored on-chain, `sign_transfer` will always revert with `InvalidAmountToTransfer`, and no recovery path exists.

This matches the **Irreversible fund lock / permanently unclaimable user value** impact category.

### Likelihood Explanation

The condition is reachable by any unprivileged user through the public `ft_transfer_call` entry point. No special role or key is required. A user who sends a "dust" amount — or who miscalculates the minimum bridgeable unit — will trigger the lock silently: `init_transfer` succeeds and emits an `InitTransferEvent`, giving no indication that the transfer can never be completed. The decimal gap between NEAR (24 decimals) and EVM tokens (commonly 18 decimals) makes this a realistic scenario for any NEAR-native token bridged to EVM.

### Recommendation

Add the normalization check inside `init_transfer` (or `init_transfer_internal`) **before** locking or burning tokens. The destination-chain token address and its registered decimals are accessible at that point via the same `get_token_address` + `token_decimals` lookup used in `sign_transfer`. If `normalize_amount(amount_without_fee, decimals) == 0`, return the full amount to trigger a refund via the NEP-141 `ft_transfer_call` protocol:

```rust
// In init_transfer_internal, before burn/lock:
let token_address = self.get_token_address(
    transfer_message.get_destination_chain(),
    self.get_token_id(&transfer_message.token),
);
if let Some(token_address) = token_address {
    if let Some(decimals) = self.token_decimals.get(&token_address) {
        let normalized = Self::normalize_amount(
            transfer_message.amount_without_fee()
                .unwrap_or(0),
            decimals,
        );
        if normalized == 0 {
            self.remove_transfer_message_without_refund(
                transfer_message.get_transfer_id()
            );
            return transfer_message.amount; // triggers ft_transfer_call refund
        }
    }
}
```

### Proof of Concept

1. A NEAR token is registered with `origin_decimals = 24`, `decimals = 18` (divisor = 10^6).
2. Alice calls `ft_transfer_call(bridge, amount=500_000, msg=InitTransfer{fee=0, recipient=eth:0x...})`.
3. `init_transfer_internal` succeeds: 500,000 yocto-tokens are burned/locked; `InitTransferEvent` is emitted; function returns `U128(0)` → no refund.
4. Relayer calls `sign_transfer(transfer_id, ...)`.
5. `normalize_amount(500_000, {origin=24, decimals=18})` = `500_000 / 1_000_000` = **0**.
6. `require!(amount_to_transfer > 0, ...)` panics with `InvalidAmountToTransfer`.
7. Alice's 500,000 tokens are permanently locked with no recovery path. [5](#0-4) [2](#0-1) [4](#0-3)

    fn init_transfer_internal(
        &mut self,
        transfer_message: TransferMessage,
        storage_owner: AccountId,
    ) -> U128 {
        let required_storage_balance = self
            .add_transfer_message(transfer_message.clone(), storage_owner.clone())
            .saturating_add(NearToken::from_yoctonear(transfer_message.fee.native_fee.0));

        if self
            .try_update_storage_balance(
                storage_owner,
                required_storage_balance,
                NearToken::from_yoctonear(0),
            )
            .is_err()
        {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        if let OmniAddress::Near(token_id) = transfer_message.token.clone() {
            self.burn_tokens_if_needed(token_id.clone(), transfer_message.amount);

            self.lock_tokens_if_needed(
                transfer_message.get_destination_chain(),
                &token_id,
                transfer_message.amount.0,
            );
        } else {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        env::log_str(&OmniBridgeEvent::InitTransferEvent { transfer_message }.to_log_string());
        U128(0)
    }
