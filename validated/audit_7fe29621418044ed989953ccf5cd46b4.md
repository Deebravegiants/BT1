### Title
Unconstrained `fee_proposal_fri` During Startup Window Allows Permanent Fee Market Manipulation — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

During the first `fee_proposal_window_size` (10) blocks, `is_proposal_init_valid` skips all bounds enforcement on `fee_proposal_fri` because `fee_actual` is `None`. A malicious proposer can set `fee_proposal_fri` to any arbitrary value (e.g., `u128::MAX`). These values are recorded verbatim in `fee_proposals_window` and become the sole input to `compute_fee_actual` once the window fills. If a majority of the first 10 blocks carry extreme `fee_proposal_fri` values, `fee_actual` becomes extreme, which forces `l2_gas_price` to an extreme value for all subsequent blocks — permanently breaking the fee market.

---

### Finding Description

In `is_proposal_init_valid`, the bounds check on `fee_proposal_fri` is guarded by an `if let` that only fires when **both** `fee_actual` is `Some` and `fee_proposal_fri` is `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

`fee_actual` is `None` whenever `compute_fee_actual` cannot fill a complete window of `window_size` prior heights:

After the window fills, `fee_actual` becomes the **floor** for `l2_gas_price` via `calculate_next_l2_gas_price_for_fin`:

calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [5](#0-4) 

An honest proposer freezes `fee_proposal` at `l2_gas_price` when `fee_actual` is `None`:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [6](#0-5) 

A malicious proposer is not constrained to do this. There is no floor check (e.g., `fee_proposal_fri >= min_gas_price`) anywhere in the validator path during startup.

The `l2_gas_price_fri` field (the block's execution gas price) is checked with exact equality, but `fee_proposal_fri` (the SNIP-35 fee-market signal) is a **separate** field with no analogous guard during startup:

```rust
if !(init_proposed.height == proposal_init_validation.height
    && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
    && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
``` [7](#0-6)

### Impact Explanation

If a malicious proposer controls ≥ 6 of the first 10 blocks and sets `fee_proposal_fri = u128::MAX` for each:

- `compute_fee_actual` at height 10 returns `u128::MAX` (median of a majority-`u128::MAX` window).
- `effective_min = max(config_min, u128::MAX) = u128::MAX`.
- `calculate_next_base_gas_price` returns `u128::MAX` as `l2_gas_price`.
- Every subsequent transaction whose `max_price_per_unit < u128::MAX` is rejected at pre-validation.
- The network's fee market is permanently broken until a governance intervention or chain revert.

Conversely, setting `fee_proposal_fri = 1` for a majority of the first 10 blocks drives `fee_actual` to near-zero. While `config_min` prevents `l2_gas_price` from dropping below the configured floor, the `fee_actual` value then becomes the reference for all future `fee_proposal` bounds (±0.2 % per block), permanently anchoring the fee market at a near-zero level and enabling spam at negligible cost.

This matches: **Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

---

### Likelihood Explanation

- Requires the attacker to be the proposer for ≥ 6 of the first 10 blocks.
- In a small or newly-launched validator set (the exact scenario where the startup window applies), a single validator with moderate weight can be selected as proposer for a majority of early blocks.
- No additional on-chain cost is incurred beyond the normal proposer role; the malicious `fee_proposal_fri` value is accepted by all validators without error.
- The attack window is exactly the first `window_size = 10` blocks — a narrow but deterministic and predictable target.

---

### Recommendation

Add a floor check on `fee_proposal_fri` that is enforced **regardless** of whether `fee_actual` is available. At minimum, require:

```rust
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    let min_price = VersionedConstants::latest_constants().min_gas_price;
    if fee_proposal < min_price {
        return Err(ValidateProposalError::InvalidProposalInit(..., "fee_proposal below min_gas_price"));
    }
    // Optionally add an upper cap (e.g., some multiple of min_gas_price or a hard maximum).
}
```

This mirrors the existing `min_gas_price` guard applied to `l2_gas_price` and closes the unconstrained startup window.

---

### Proof of Concept

```
Network launches at block 0. fee_proposal_window_size = 10.

Blocks 0–5 (malicious proposer):
  ProposalInit { fee_proposal_fri: Some(u128::MAX), l2_gas_price_fri: <valid>, ... }
  → is_proposal_init_valid: fee_actual = None → bounds check SKIPPED → accepted
  → finalize_decision: record_fee_proposal(height, Some(u128::MAX))

Blocks 6–9 (honest proposer):
  ProposalInit { fee_proposal_fri: Some(8_000_000_000), ... }
  → record_fee_proposal(height, Some(8_000_000_000))

Block 10:
  compute_fee_actual(window, height=10, window_size=10)
  window = [u128::MAX, u128::MAX, u128::MAX, u128::MAX, u128::MAX, u128::MAX,
            8e9, 8e9, 8e9, 8e9]  (sorted)
  median (even) = avg(window[4], window[5]) = avg(8e9, u128::MAX) ≈ u128::MAX / 2

  calculate_next_l2_gas_price_for_fin:
    effective_min = max(config_min, u128::MAX / 2) = u128::MAX / 2
    l2_gas_price = u128::MAX / 2

All transactions with max_price_per_unit < u128::MAX / 2 are rejected.
Fee market is broken for all subsequent blocks.
```

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
