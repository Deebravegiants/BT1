### Title
`respond_verify_foreign_tx` Accepts Caller-Supplied `payload_hash` Without Binding It to the Original Request - (File: crates/contract/src/lib.rs)

### Summary
The `respond_verify_foreign_tx` function verifies that a submitted signature is valid over a caller-supplied `payload_hash`, but never verifies that `payload_hash` is the canonical hash of the original pending foreign-transaction request. A single Byzantine participant below the signing threshold who acts as signing leader for any one request can reuse that request's valid MPC signature to forge the verification response for any other pending foreign-transaction request.
