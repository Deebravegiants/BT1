Audit Report

## Title
`RSETHPriceFeed.latestRoundData()` Returns ETH/USD Staleness Metadata While Serving a Frozen rsETH Price During Auto-Pause - (File: contracts/oracles/RSETHPriceFeed.sol)

## Summary
`RSETHPriceFeed.latestRoundData()` computes rsETH/USD by multiplying the ETH/USD Chainlink price by `LRTOracle.rsETHPrice`, but returns `updatedAt` exclusively from the ETH/USD feed. When `LRTOracle` auto-pauses due to a price drop exceeding `pricePercentageLimit`, `rsETHPrice` is frozen at its pre-drop value while `updatedAt` continues to reflect the ETH/USD feed's near-real-time updates, causing any consumer that checks `updatedAt` for staleness (e.g., Morpho) to accept a stale, inflated rsETH/USD price as current.

## Finding Description
`RSETHPriceFeed.latestRoundData()` (lines 63–70) fetches all five return values from `ETH_TO_USD.latestRoundData()` and then overwrites only `answer`:

```solidity
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
```

`RS_ETH_ORACLE.rsETHPrice()` resolves to `LRTOracle.rsETHPrice` (line 28), a stored state variable updated only inside `_updateRsETHPrice()`. The public entry point `updateRSETHPrice()` (lines 87–89) is gated by `whenNotPaused`. Inside `_updateRsETHPrice()`, when the computed new price falls more than `pricePercentageLimit` below `highestRsethPrice`, the function pauses the protocol and returns early (lines 277–281) **without writing the new price to `rsETHPrice`**:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;   // rsETHPrice is NOT updated
}
```

After this point, `updateRSETHPrice()` reverts for all callers (`whenNotPaused`). The manager escape hatch `updateRSETHPriceAsManager()` (lines 94–96) requires the `MANAGER` role and is not guaranteed to be called promptly. During the entire pause window, `RSETHPriceFeed.latestRoundData()` returns a fresh `updatedAt` (ETH/USD updates every ~1–2 minutes on mainnet) paired with the stale, pre-drop `rsETHPrice`. No on-chain mechanism exposes this discrepancy to consumers.

## Impact Explanation
**Low — Contract fails to deliver promised returns.** `RSETHPriceFeed` is deployed as a Chainlink-compatible feed for Morpho (per README: `RSETHPriceFeed (Morph)`). Its implicit contract is to provide an accurate, freshness-stamped rsETH/USD price. During an auto-pause triggered by a genuine rsETH price decline, the feed instead serves a stale (inflated) price with a fresh `updatedAt`, causing Morpho's staleness guard to pass on data that is materially incorrect. The contract does not deliver the promised return value.

## Likelihood Explanation
The auto-pause is a designed safety mechanism that activates automatically when any unprivileged caller invokes `updateRSETHPrice()` during a market stress event (slashing, depeg) that moves the rsETH price beyond `pricePercentageLimit`. No attacker capability or privileged access is required to enter the vulnerable state — it is the normal protocol response to the exact conditions (price decline) under which accurate pricing is most critical. The pause persists until an admin manually unpauses, which may take hours.

## Recommendation
Track the timestamp of the last successful `rsETHPrice` write in `LRTOracle` and expose it via the `IRSETHOracle` interface. In `RSETHPriceFeed.latestRoundData()`, return the minimum of the ETH/USD `updatedAt` and the rsETH price's last-updated timestamp:

```solidity
// LRTOracle.sol — add alongside rsETHPrice
uint256 public lastRsETHPriceUpdate;

// In _updateRsETHPrice(), just before `rsETHPrice = newRsETHPrice;`
lastRsETHPriceUpdate = block.timestamp;

// RSETHPriceFeed.latestRoundData()
updatedAt = Math.min(updatedAt, RS_ETH_ORACLE.lastRsETHPriceUpdate());
```

This ensures any consumer checking `updatedAt` against a heartbeat threshold will correctly detect staleness in the rsETH price component.
