### Title
Deprecated Class Body Not Hash-Verified During P2P Sync — Wrong Cairo 0 Bytecode Stored Under Valid Class Hash - (`crates/apollo_p2p_sync/src/client/class.rs`)

---

### Summary

During P2P class sync, a peer-supplied `DeprecatedContractClass` body is stored under a `class_hash` taken from the trusted state diff without ever verifying that `hash(deprecated_class_body) == class_hash`. A malicious peer can therefore poison the class manager with arbitrary Cairo 0 bytecode under any legitimately-declared class hash, causing every subsequent execution of that contract to run the wrong code.

---

### Finding Description

In `write_to_storage`, the loop over deprecated classes calls:

```rust
class_manager_client
    .add_deprecated_class(class_hash, deprecated_class.clone())
    .await
``` [1](#0-0) 

`class_hash` is taken from the already-validated state diff (trusted), but `deprecated_class` is the raw body received from the peer (untrusted). The only validation performed in `parse_data_for_block` is:

- `deprecated_declared_classes.contains(&class_hash)` — confirms the hash appears in the state diff.
- Duplicate-class check. [2](#0-1) 

Neither `parse_data_for_block` nor `write_to_storage` computes `hash(deprecated_class_body)` and compares it to `class_hash`.

`add_deprecated_class` in the class manager passes the body straight to `set_deprecated_class` with no hash recomputation: [3](#0-2) 

`set_deprecated_class` in `CachedClassStorage` stores the body as-is: [4](#0-3) 

The contrast with Sierra (Cairo 1) classes is instructive: `add_class` returns a `ClassHashes` struct containing the manager-computed hash, and there is already a developer TODO to verify it against the state-diff hash. For deprecated classes, `add_deprecated_class` returns `()` — there is no return path for the computed hash, and no TODO exists for this check. [5](#0-4) 

The hash algorithm for deprecated classes is well-defined and callable (`compute_deprecated_class_hash`): [6](#0-5) 

---

### Impact Explanation

Once a wrong body is stored, every call to `get_executable(class_hash)` for that Cairo 0 contract returns the attacker-chosen bytecode. The blockifier uses this to execute the contract, producing wrong state transitions, wrong events, wrong return values, and wrong fee accounting for all transactions that invoke the poisoned class. This maps directly to: **"Critical. Wrong compiled class / contract code selected for execution."**

---

### Likelihood Explanation

Any unauthenticated peer in the P2P sync network can trigger this. The syncing node connects to peers to fetch class bodies; there is no authentication or reputation gate that prevents a peer from sending a mismatched body. The bad data is not rejected — it is stored unconditionally (the retry loop only retries on transport errors, not on hash mismatches). A single malicious peer that serves one block's class data is sufficient.

---

### Recommendation

After calling `add_deprecated_class`, recompute the hash from the stored body and compare it to the state-diff `class_hash`. Because `add_deprecated_class` currently returns `()`, the simplest fix is to compute the hash **before** calling the class manager, using `compute_deprecated_class_hash`, and reject the peer (return `BadPeerError`) if it does not match. This mirrors the intent of the existing TODO for Sierra classes.

---

### Proof of Concept

1. Construct a `DeprecatedContractClass` body `B_evil` (e.g., a no-op program).
2. Choose any legitimately-declared `class_hash H` from a real block's state diff.
3. Serve `(DeprecatedContractClass(B_evil), H)` from a malicious peer during P2P sync.
4. `parse_data_for_block` accepts it because `H` is in `deprecated_declared_classes`.
5. `write_to_storage` calls `add_deprecated_class(H, B_evil)` — stored without hash check.
6. Assert `get_executable(H)` returns `B_evil`, not the legitimate bytecode.
7. Execute any transaction calling a contract deployed with class `H`; observe wrong execution result.

pub fn compute_deprecated_class_hash(
    contract_class: &ContractClass,
) -> Result<Felt, HintedClassHashError> {
    let hinted_class_hash = compute_cairo_hinted_class_hash(contract_class)?;
    let contract_definition_vec = serde_json::to_vec(contract_class)?;
    let contract_definition: CairoContractDefinition<'_> =
        serde_json::from_slice(&contract_definition_vec)?;

    let FlatEntryPointFelts { external, l1_handler, constructor } =
        get_flat_entry_point_felts(&contract_definition.entry_points_by_type);
    let builtins = ascii_strs_as_felts(&contract_definition.program.builtins);
    let bytecode = hex_strs_as_felts(&contract_definition.program.data);

    let mut hash_state = HashState::<Pedersen>::new();
    hash_state.update_single(&DEPRECATED_COMPILED_CLASS_VERSION);
    hash_state.update_with_hashchain(&external);
    hash_state.update_with_hashchain(&l1_handler);
    hash_state.update_with_hashchain(&constructor);
    hash_state.update_with_hashchain(&builtins);
    hash_state.update_single(&hinted_class_hash);
    hash_state.update_with_hashchain(&bytecode);
    Ok(hash_state.finalize())
}
