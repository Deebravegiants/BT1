### Title
Missing TEE Attestation Validity Check in `vote_cancel_resharing` Allows Participants with Expired/Revoked Attestations to Block Their Own Removal - (File: `crates/contract/src/lib.rs`, `crates/contract/src/state/resharing.rs`)

---

### Summary

`vote_cancel_resharing` enforces only a signer-identity check (`assert_caller_is_signer`) and a participant-membership check (via `AuthenticatedAccountId::new`), but **omits the TEE attestation validity check** that every other privileged node method requires. This is the direct analog of the ERC20 report's Issue 1: one authorization condition (participant membership) is verified while a second required condition (valid TEE attestation) is silently skipped. Participants whose attestations have expired or whose MPC image hash is no longer whitelisted can vote to cancel the very resharing that `verify_tee` triggered to remove them.

---

### Finding Description

`vote_cancel_resharing` in `lib.rs` calls only `Self::assert_caller_is_signer()` before delegating to `protocol_state.vote_cancel_resharing()`:

```rust
pub fn vote_cancel_resharing(&mut self) -> Result<(), Error> {
    Self::assert_caller_is_signer();                          // ← only identity check
    if let Some(new_state) = self.protocol_state.vote_cancel_resharing()? {
        self.protocol_state = new_state;
    }
    Ok(())
}
``` [1](#0-0) 

The inner `resharing.vote_cancel_resharing()` adds a participant-membership check via `AuthenticatedAccountId::new(previous_running_participants)`:

```rust
pub fn vote_cancel_resharing(&mut self) -> Result<Option<RunningContractState>, Error> {
    let previous_running_participants = self.previous_running_state.parameters.participants();
    let authenticated_candidate = AuthenticatedAccountId::new(previous_running_participants)?;
    self.cancellation_requests.insert(authenticated_candidate);
    ...
}
``` [2](#0-1) 

This checks only that `env::signer_account_id()` is in the previous participant set. It does **not** call `self.tee_state.is_caller_an_attested_participant(...)` or `reverify_participants(...)`.

Every other privileged node method — `vote_reshared`, `vote_pk`, `vote_abort_key_event_instance`, `start_keygen_instance`, `start_reshare_instance`, and all `respond*` callbacks — calls `assert_caller_is_attested_participant_and_protocol_active()`, which additionally verifies that a stored attestation exists and that the signer's account public key matches the attested key:

```rust
fn assert_caller_is_attested_participant_and_protocol_active(&self) {
    let participants = self.protocol_state.active_participants();
    Self::assert_caller_is_signer();
    let attestation_check = self
        .tee_state
        .is_caller_an_attested_participant(participants);
    assert_matches::assert_matches!(attestation_check, Ok(()), "Caller must be an attested participant");
}
``` [3](#0-2) 

`is_caller_an_attested_participant` itself does not re-verify attestation expiry or image-hash validity — it only checks that a stored entry exists and keys match: [4](#0-3) 

The `verify_tee` function is the mechanism that detects participants with expired or revoked attestations and triggers a resharing to remove them: [5](#0-4) 

Because `vote_cancel_resharing` skips the attestation check, the participants being removed by `verify_tee` can immediately call `vote_cancel_resharing` to undo the resharing.

The same missing check exists in `vote_cancel_keygen`: [6](#0-5) 

---

### Impact Explanation

**Bypassed safety invariant:** The TEE validation mechanism (`verify_tee` → resharing) is designed to guarantee that only participants running authorized, up-to-date MPC images remain in the network. By cancelling the resharing, invalid participants circumvent this guarantee.

**Concrete consequence:** After a successful cancellation, the invalid participants remain in the `RunningContractState` participant set. Because `is_caller_an_attested_participant` does not re-verify attestation validity (only checks stored-entry existence and key match), these participants can still call `respond`, `respond_ckd`, and `respond_verify_foreign_tx` to submit signatures for pending requests. Participants running an outdated or compromised MPC image thus continue to contribute partial signatures to the threshold computation.

This satisfies the allowed impact: *"participant-state or contract execution-flow manipulation that breaks production safety/accounting invariants."*

---

### Likelihood Explanation

The attack is reachable when:

1. **Some participants have invalid attestations** — e.g., their DCAP certificate has expired, or the MPC Docker image hash they attested to has been removed from the whitelist after a software upgrade.
2. **The governance threshold T satisfies T ≤ N − V**, where N is the total previous-epoch participant count and V is the count of participants with valid attestations. Concretely: if N = 5, T = 2, and 3 participants have invalid attestations, those 3 can accumulate 3 ≥ T = 2 cancellation votes.
3. **No operator intervention** before the cancellation votes are submitted — a realistic window during a rolling TEE image upgrade.

This scenario is realistic during any TEE image deprecation cycle where a subset of operators is slow to upgrade.

---

### Recommendation

Add a TEE attestation presence check inside `vote_cancel_resharing` (and `vote_cancel_keygen`) before accepting the vote. The check should verify that the signer has a stored attestation entry in `tee_state.stored_attestations` keyed by the participant's registered TLS public key. Optionally, call `reverify_participants` to also enforce that the stored attestation has not expired and still matches the current image-hash whitelist:

```rust
pub fn vote_cancel_resharing(&mut self) -> Result<(), Error> {
    Self::assert_caller_is_signer();
    // Add: verify the signer has a valid stored attestation
    let previous_participants = self.protocol_state
        .previous_running_participants()?;
    self.tee_state
        .is_caller_an_attested_participant(previous_participants)
        .map_err(|_| InvalidState::NotParticipant { ... })?;
    ...
}
```

### Proof of Concept

**Setup:** 5-participant network, governance threshold T = 2.

1. Three participants' DCAP certificates expire (or their MPC image hash is removed from the whitelist via `vote_code_hash`).
2. Any participant calls `verify_tee`. The contract detects 3 invalid participants, confirms 2 valid participants remain (≥ T = 2), and calls `transition_to_resharing_no_checks` to produce a `ResharingContractState` that proposes the 2 valid participants as the new set.
3. Each of the 3 invalid participants calls `vote_cancel_resharing`. The inner `AuthenticatedAccountId::new(previous_running_participants)` succeeds (they are still in the previous participant set). No attestation check is performed.
4. After 2 votes (≥ T = 2), `threshold_cancellation_votes_reached` is true and the contract reverts to the previous `RunningContractState` containing all 5 participants.
5. The 3 invalid participants remain in the active participant set. Their stored attestation entries still exist in `tee_state.stored_attestations` (cleanup only runs after a successful resharing). `is_caller_an_attested_participant` returns `Ok(())` for them.
6. The invalid participants can call `respond` to submit signatures for any pending sign requests, contributing to threshold-signature production despite running unauthorized or outdated MPC images. [2](#0-1) [1](#0-0) [4](#0-3)

    pub fn verify_tee(&mut self) -> Result<bool, Error> {
        log!("verify_tee: signer={}", env::signer_account_id());
        // Caller must be a participant (node or operator).
        self.voter_or_panic();
        let ProtocolContractState::Running(running_state) = &mut self.protocol_state else {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        };
        let current_params = running_state.parameters.clone();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        match self.tee_state.reverify_and_cleanup_participants(
            current_params.participants(),
            tee_upgrade_deadline_duration,
        ) {
            TeeValidationResult::Full => {
                self.accept_requests = true;
                log!("All participants have an accepted Tee status");
                Ok(true)
            }
            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            } => {
                let remaining = participants_with_valid_attestation.len();
                // Defense in depth: the surviving participant set must keep the full
                // threshold relation intact — the GovernanceThreshold must still sit
                // within its bounds for the smaller set (in particular it must not
                // exceed the remaining participant count or the upper cap) and must
                // remain at least every domain's ReconstructionThreshold (the kickout
                // keeps the existing per-domain thresholds). Otherwise we refuse and
                // wait for manual intervention.
                let max_reconstruction_threshold =
                    max_reconstruction_threshold(running_state.domains.domains());
                if let Err(err) = ThresholdParameters::validate_governance_against_reconstruction(
                    u64::try_from(remaining).expect("participant count fits in u64"),
                    current_params.threshold(),
                    max_reconstruction_threshold,
                ) {
                    log!(
                        "Kicking out participants with an invalid TEE status would break the threshold relation ({:?}); {} participants remain with a valid TEE status. This requires manual intervention. We will not accept new signature requests as a safety precaution.",
                        err,
                        remaining,
                    );
                    self.accept_requests = false;
                    return Ok(false);
                }

                // here, we set it to true, because at this point, we have at least `threshold`
                // number of participants with an accepted Tee status.
                self.accept_requests = true;

                // do we want to adjust the threshold?
                //let n_participants_new = new_participants.len();
                //let new_threshold = (3 * n_participants_new + 4) / 5; // minimum 60%
                //let new_threshold = new_threshold.max(2); // but also minimum 2
                let new_threshold = usize::try_from(current_params.threshold().value())
                    .expect("threshold value fits in usize");

                let threshold_parameters = ThresholdParameters::new(
                    participants_with_valid_attestation,
                    Threshold::new(new_threshold as u64),
                )
                .expect("Require valid threshold parameters"); // this should never happen.
                current_params.validate_incoming_proposal(&threshold_parameters)?;
                // This resharing only changes the participant set, so the
                // per-domain reconstruction-threshold updates map is empty.
                let proposed_parameters =
                    ProposedThresholdParameters::new(threshold_parameters, BTreeMap::new());
                let res = running_state.transition_to_resharing_no_checks(&proposed_parameters);
                if let Some(resharing) = res {
                    self.protocol_state = ProtocolContractState::Resharing(resharing);
                }

                Ok(true)
            }
        }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
