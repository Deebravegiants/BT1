### Title
Storage Deposit Paid via `submit_participant_info` Is Permanently Locked When Attestation Entry Is Cleaned Up — (File: `crates/contract/src/lib.rs`, `crates/contract/src/tee/tee_state.rs`)

---

### Summary

Any account that calls `submit_participant_info` and pays a storage deposit to register a TEE attestation entry permanently loses that deposit when the entry is later removed by `clean_invalid_attestations`. The freed storage cost is credited back to the contract's balance, not to the original depositor. No refund path exists.

---

### Finding Description

`submit_participant_info` measures the storage consumed by inserting a new `NodeAttestation` entry into `stored_attestations` and charges the caller exactly that cost:

```rust
// crates/contract/src/lib.rs ~L826-848
if attestation_storage_must_be_paid_by_caller {
    let storage_used = env::storage_usage().saturating_sub(initial_storage);
    let cost = env::storage_byte_cost().saturating_mul(storage_used as u128);
    let attached = env::attached_deposit();
    if attached < cost {
        return Err(...);
    }
    if let Some(diff) = attached.checked_sub(cost) && diff > NearToken::from_yoctonear(0) {
        Promise::new(account_id).transfer(diff).detach();   // only excess is refunded
    }
}
```

The condition `attestation_storage_must_be_paid_by_caller` is true for every new attestation submission and for any caller who is not yet a participant:

```rust
let attestation_storage_must_be_paid_by_caller =
    is_new_attestation || caller_is_not_participant;
```

After every successful resharing (`vote_reshared`), the contract automatically fires a detached promise to `clean_invalid_attestations`:

```rust
// crates/contract/src/lib.rs ~L1194-1205
Promise::new(env::current_account_id())
    .function_call(
        method_names::CLEAN_INVALID_ATTESTATIONS.to_string(),
        ...
        Gas::from_tgas(self.config.clean_invalid_attestations_tera_gas),
    )
    .detach();
```

`clean_invalid_attestations` removes every entry whose attestation no longer passes re-verification (expired certificate, revoked docker-image hash, changed OS measurement, etc.):

```rust
// crates/contract/src/tee/tee_state.rs ~L406-434
pub fn clean_invalid_attestations(...) -> u32 {
    ...
    for tls_pk in invalid_tls_keys {
        self.stored_attestations.remove(&tls_pk);   // storage freed → credited to contract
    }
    removed
}
```

On NEAR, freeing storage credits the **contract's** balance, not the account that originally paid the deposit. There is no code anywhere in the contract that records who paid for each attestation entry or that issues a refund when the entry is removed. The depositor's funds are permanently absorbed into the contract balance.

---

### Impact Explanation

Every prospective or current participant that calls `submit_participant_info` pays a storage deposit. When their attestation expires or is invalidated (e.g., after a docker-image hash rotation or certificate expiry), `clean_invalid_attestations` silently removes the entry and the deposit is irrecoverably locked in the contract. There is no withdrawal function, no refund callback, and no governance path to recover these funds. This breaks the accounting invariant that storage deposits should be returned to depositors when the corresponding storage is freed.

**Impact: Medium** — systematic, permanent loss of participant-paid storage deposits; breaks production accounting invariants.

---

### Likelihood Explanation

Attestation cleanup is triggered automatically after **every** resharing via a detached promise. Attestations become invalid whenever: (a) the allowed docker-image hash set is rotated (normal upgrade path), (b) a launcher image is removed, (c) OS measurements change, or (d) the attestation certificate expires. All of these are routine operational events. Any participant who submitted attestation info before such an event and is not re-included in the new epoch will lose their deposit with no recourse.

**Likelihood: Medium** — triggered by normal, expected protocol operations (resharing + image rotation).

---

### Recommendation

1. Record the depositor's `AccountId` alongside each `NodeAttestation` entry at insertion time.
2. In `clean_invalid_attestations` (and any other removal path), issue a `Promise::new(depositor).transfer(stored_deposit)` before removing the entry, so the freed storage cost is returned to the original payer.
3. Alternatively, track a per-entry `deposit_paid` field and refund it on removal, mirroring the pattern already used in `propose_update` for excess-deposit refunds.

---

### Proof of Concept

1. Account `alice.near` calls `submit_participant_info` with a valid attestation. The contract charges `env::storage_byte_cost() * storage_delta` NEAR and stores the entry in `stored_attestations` keyed by Alice's TLS public key.
2. Participants vote to rotate the allowed docker-image hash (`vote_code_hash` reaches threshold → `whitelist_tee_proposal`). Alice's stored attestation was tied to the old image hash.
3. Participants complete a resharing (`vote_reshared` reaches threshold). The contract automatically fires `clean_invalid_attestations`.
4. `clean_invalid_attestations` calls `self.stored_attestations.remove(&alice_tls_pk)`. NEAR credits the freed storage bytes back to the contract's balance.
5. Alice has no function to call to recover her deposit. The funds remain in the contract permanently. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

```rust
    pub fn clean_invalid_attestations(
        &mut self,
        tee_upgrade_deadline_duration: Duration,
        max_scan: usize,
    ) -> u32 {
        let has_invalid_attestation = |node_id: &NodeId| {
            !matches!(
                self.reverify_participants(node_id, tee_upgrade_deadline_duration),
                TeeQuoteStatus::Valid
            )
        };

        // Materialize candidates before any mutation to avoid iterator invalidation.
        let invalid_tls_keys: Vec<Ed25519PublicKey> = self
            .stored_attestations
            .iter()
            .take(max_scan)
            .filter(|(_, node_attestation)| has_invalid_attestation(&node_attestation.node_id))
            .map(|(tls_pk, _)| tls_pk.clone())
            .collect();

        let removed = u32::try_from(invalid_tls_keys.len())
            .expect("u32 should always be convertible from usize on wasm32");

        for tls_pk in invalid_tls_keys {
            self.stored_attestations.remove(&tls_pk);
        }
        removed
    }
```
