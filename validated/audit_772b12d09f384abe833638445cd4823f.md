Audit Report

## Title
Missing Caller Validation in `MerkleDistributor.claim()` Enables Forced Fee-Deducting Claims on Behalf of Any User - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary
`MerkleDistributor.claim()` accepts an arbitrary `account` parameter but performs no check that `msg.sender == account`. Any unprivileged caller can trigger a claim for any user, causing the protocol to permanently deduct up to 10% of the user's claimable allocation as a fee sent to `protocolTreasury`, without the user's consent. The sibling contract `KernelMerkleDistributor` explicitly guards against this with an `account != msg.sender` revert, confirming the omission is unintentional.

## Finding Description
`MerkleDistributor.claim()` accepts `account` as a caller-supplied parameter and processes the full claim flow — merkle proof verification, state update, fee deduction, and token transfer — without ever verifying that `msg.sender == account`. [1](#0-0) 

After proof verification, the contract deducts a fee and marks the claim consumed: [2](#0-1) 

The fee ceiling is `MAX_FEE_IN_BPS = 1000` (10%). [3](#0-2) 

The sibling contract `KernelMerkleDistributor._processClaim()` explicitly guards against this exact pattern: [4](#0-3) 

`MerkleDistributor` has no equivalent guard anywhere in its `claim()` function. [5](#0-4) 

## Impact Explanation
An attacker calls `claim(index, victim, cumulativeAmount, proof)` with a valid merkle proof for the victim. The contract: (1) marks the victim's claim as consumed via `userClaims[victim].lastClaimedIndex = index`, (2) deducts `feeInBPS / 10_000` of the claimable amount and sends it to `protocolTreasury`, and (3) sends the remainder to `victim`. The victim permanently loses the fee portion — up to 10% of their allocation — with no recourse. This is **theft of unclaimed yield** (High severity), which is an explicitly allowed impact in scope.

## Likelihood Explanation
The function is `external`, requires no special role, and is callable by any EOA or contract while the contract is unpaused. The only prerequisite is a valid merkle proof for the target account, which is either derivable from publicly published off-chain tree data or observable on-chain from prior claim transactions. The attack is repeatable across all eligible users and across every new merkle root epoch.

## Recommendation
Add a caller-identity check identical to the one already present in `KernelMerkleDistributor._processClaim()`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Insert this check in `MerkleDistributor.claim()` before the merkle proof verification (after the `isClaimed` check), mirroring the pattern at `KernelMerkleDistributor.sol` lines 311–313. [4](#0-3) 

## Proof of Concept
1. Alice has an unclaimed allocation of 1000 tokens in the merkle tree. `feeInBPS = 500` (5%).
2. Bob (attacker) obtains Alice's merkle proof from the published tree data.
3. Bob calls `MerkleDistributor.claim(index, alice, 1000, aliceProof)` from his own EOA.
4. The contract verifies the proof (valid, since it commits to `alice`'s address), sets `userClaims[alice].lastClaimedIndex = index`, transfers 950 tokens to Alice and 50 tokens to `protocolTreasury`.
5. Alice's claim is permanently consumed. She cannot reclaim the 50 tokens.
6. Bob spent only gas; Alice lost 50 tokens without consent.

**Foundry test plan:**
```solidity

    // Setup: fund distributor, set merkle root with Alice's leaf
    // Act: Bob calls claim(index, alice, 1000, aliceProof)

    /// @inheritdoc IMerkleDistributor
    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
