### Title
Stale Foreign-Chain Attestation Replay via `respond_verify_foreign_tx` - (File: `crates/contract/src/lib.rs`)

### Summary

`respond_verify_foreign_tx` verifies only that the submitted ECDSA signature is mathematically valid over the caller-supplied `payload_hash` using the root public key. It does not track which `(payload_hash, signature)` pairs have already been consumed. Because the `VerifyForeignTransactionRequest` map key contains no per-submission nonce or timestamp, a single malicious attested participant can replay a previously published valid response against any new pending entry for the same foreign transaction, causing downstream consumers to receive a stale or incorrect chain-state attestation.

### Finding Description

The `respond_verify_foreign_tx` function in `crates/contract/src/lib.rs` accepts a `VerifyForeignTransactionResponse` that contains a caller-supplied `payload_hash` and an ECDSA signature. The only on-chain check is that the signature is valid over `payload_hash` under the root public key: [1](#0-0) 

After passing this check, `resolve_yields_for` drains the pending-request queue for the matching `VerifyForeignTransactionRequest` key and removes the map entry: [2](#0-1) [3](#0-2) 

The `VerifyForeignTransactionRequest` key is composed solely of `(request, domain_id, payload_version)` — no per-submission nonce, no caller account, no timestamp: [4](#0-3) 

Likewise, the signed payload (`ForeignTxSignPayload::V1`) contains only `(request, extracted_values)` — no timestamp or block height: [5](#0-4) 

Once a legitimate MPC response is published on-chain (the `(payload_hash, signature)` pair is permanently visible in NEAR transaction history), any subsequent `verify_foreign_transaction` call for the same `tx_id` creates a fresh pending entry under the identical map key. A single malicious attested participant can immediately call `respond_verify_foreign_tx` with the old `(payload_hash, signature)` — no MPC protocol execution required — and the contract accepts it, because the signature is still mathematically valid over `payload_hash` under the root key.

The contract has no "already-used signature" registry and no mechanism to distinguish a freshly computed response from a replayed one.

### Impact Explanation

**High.** The `verify_foreign_transaction` flow is the trust anchor for the Omnibridge inbound path: NEAR contracts rely on the returned `VerifyForeignTransactionResponse` to confirm that a specific foreign-chain transaction finalized in a specific block. A replayed stale response delivers a signature attesting to an old `(tx_id, block_hash, extracted_values)` tuple. If the foreign chain has since reorganized (realistic for Bitcoin with low confirmation counts, or any chain before finality), the attested block hash no longer exists in the canonical chain. A bridge contract that acts on this stale attestation — e.g., minting tokens for a deposit that was reorganized away — suffers a double-spend or invalid bridge execution. This matches the allowed impact: *"Cross-chain replay, forged foreign-chain verification... that causes invalid bridge execution or double-spend conditions."*

### Likelihood Explanation

**Medium.** Three conditions must hold simultaneously:

1. A legitimate `verify_foreign_transaction(tx_id_X)` must have completed, publishing `(payload_hash_X, sig_X)` on-chain. This happens in normal operation.
2. A new `verify_foreign_transaction(tx_id_X)` must be submitted. This occurs naturally (e.g., after a timeout, or by a different user/bridge service).
3. A single malicious attested participant must call `respond_verify_foreign_tx` with the old response before honest nodes finish the MPC protocol. Since the attacker skips the MPC computation entirely (they already have the signature), they have a decisive timing advantage over honest nodes.

Only one node below the signing threshold needs to be compromised or act maliciously.

### Recommendation

1. **Track consumed signatures.** Maintain a `LookupSet<Hash256>` (or equivalent) of `payload_hash` values that have already been accepted by `respond_verify_foreign_tx`. Reject any response whose `payload_hash` appears in this set.

2. **Bind the response to the submission epoch.** Include a NEAR block height or a per-submission nonce in the `VerifyForeignTransactionRequest` key and in the signed payload. This makes each pending entry unique and prevents an old signature from satisfying a new entry.

3. **Include a freshness field in `ForeignTxSignPayload`.** Add the observed foreign-chain block height or timestamp to `ForeignTxSignPayloadV1` so that the signed attestation is inherently time-bound and a replayed signature from a different epoch is cryptographically distinguishable.

### Proof of Concept

```

// → sig_X is now permanently visible in NEAR transaction history

// Step 2 — Bitcoin reorg: tx_id_X is now in block_hash_B (block_hash_A is stale)

// Step 3 — New submission (e.g., bridge service retries after timeout)

// → new pending entry created under the same VerifyForeignTransactionRequest key

// Step 4 — Malicious attested node replays old response (no MPC protocol needed)

    response: { payload_hash: payload_hash_X,    // hash of stale (tx_id_X, block_hash_A)

})
// → contract accepts: signature check passes, resolve_yields_for drains the queue
// → user2 receives attestation for block_hash_A (reorganized away), not block_hash_B
// → bridge mints tokens for a deposit that no longer exists on the canonical chain
```

    pub fn respond_verify_foreign_tx(
        &mut self,
        request: VerifyForeignTransactionRequest,
        response: VerifyForeignTransactionResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!(
            "respond_verify_foreign_tx: signer={}, request={:?}",
            &signer,
            &request
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain.0.into())?;

        let signature_is_valid = match (&response.signature, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");

                let payload_hash: [u8; 32] = response.payload_hash.0;

                // Check the signature is correct against the root public key
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    &payload_hash,
                    &secp_pk,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_verify_foreign_tx_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
