### Title
`simulate_transactions` with `Tag::Latest` misclassifies class replacements as fresh deployments for contracts deployed in the latest block — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

---

### Summary

`convert_thin_state_diff` uses `block_number.prev()` as the "previous block" for all non-pending `block_id` values, including `Tag::Latest`. However, `simulate_transactions` always executes against `StateNumber::unchecked_right_after_block(block_number)` — the same state for both `Tag::Latest` and `Tag::Pending`. This creates an off-by-one: for `Tag::Latest`, the deployed-contract check queries one block too early, causing class replacements on contracts deployed in the latest block to be reported as fresh deployments in the simulation's `state_diff`.

---

### Finding Description

**Root cause — `get_accepted_block_number` collapses both tags to the same N:** [1](#0-0) 

Both `Tag::Latest` and `Tag::Pending` resolve to the same `block_number = N`.

**Simulation state is identical for both tags:** [2](#0-1) 

`state_number = StateNumber::unchecked_right_after_block(N)` — the execution context is state-after-block-N for both tags.

**`convert_thin_state_diff` branches asymmetrically:** [3](#0-2) 

| `block_id` | `prev_block_number` | `is_deployed` queries state at |
|---|---|---|
| `Tag::Pending` | `Some(N)` | `right_after_block(N)` ✓ |
| `Tag::Latest` | `N.prev() = N-1` | `right_after_block(N-1)` ✗ |

**Concrete failure path:**

Suppose contract A was deployed in block N (the latest accepted block). A caller submits `starknet_simulateTransactions` with `block_id = Tag::Latest` and a transaction that calls `replace_class` on A.

1. Execution runs against state-after-N, where A **is** deployed.
2. The blockifier emits A in `deployed_contracts` of the induced `ThinStateDiff` (the blockifier has no `replaced_classes` field; that distinction is made by `convert_thin_state_diff`).
3. `convert_thin_state_diff` calls `is_deployed(N-1, A)`.
4. At `right_after_block(N-1)`, A does **not** exist (it was deployed in block N).
5. `is_deployed` returns `false`; A is **not** moved to `replaced_classes`.
6. The returned `SimulatedTransaction.transaction_trace.state_diff` shows A in `deployed_contracts` — a fresh deployment — instead of `replaced_classes`. [4](#0-3) 

---

### Impact Explanation

`starknet_simulateTransactions` is a public, unauthenticated RPC endpoint. The returned `state_diff` inside `SimulatedTransaction.transaction_trace` is authoritative-looking: tooling, wallets, and indexers use it to understand what a transaction will do. Misclassifying a class replacement as a fresh deployment is a materially wrong value returned by the simulation RPC, matching the **High** impact tier: *"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."*

---

### Likelihood Explanation

The condition is narrow but fully attacker-controlled: the attacker only needs a contract deployed in the current latest block (which they can deploy themselves) and then immediately simulate a `replace_class` call against it at `Tag::Latest`. No privileged access, no special network position, and no timing race beyond the normal block cadence is required.

---

### Recommendation

`convert_thin_state_diff` is shared between `get_state_update` (where `block_number.prev()` is semantically correct — the diff *belongs to* block N, so the prior state is N-1) and `simulate_transactions` (where the simulation runs *against* state-after-N, so the prior state is also N). The fix is to pass the correct `prev_block_number` from the call site rather than deriving it from `block_id` inside the shared helper. For `simulate_transactions`, always pass `Some(block_number)` (i.e., treat it like `Tag::Pending`) regardless of whether `block_id` is `Tag::Latest` or `Tag::Pending`, since both execute against the same state.

---

### Proof of Concept

1. Ensure block N is the latest accepted block and contains a deploy of contract A with class hash `C0`.
2. Call `starknet_simulateTransactions` with `block_id = Tag::Latest` and a transaction that calls `replace_class(C1)` from within A.
3. Observe `SimulatedTransaction.transaction_trace.state_diff.deployed_contracts` contains `{address: A, class_hash: C1}`.
4. Observe `SimulatedTransaction.transaction_trace.state_diff.replaced_classes` is empty.
5. The correct result would have `replaced_classes = [{contract_address: A, class_hash: C1}]` and `deployed_contracts = []`.
6. Contrast: repeat with `block_id = Tag::Pending` — the result correctly shows A in `replaced_classes`.
