### Title
Hardcoded Consensus Signing Private Key Used in Production `create_signature_manager()` - (File: `crates/apollo_signature_manager/src/lib.rs`)

### Summary
The production `create_signature_manager()` factory function unconditionally instantiates `LocalKeyStore` with a hardcoded ECDSA private key (`0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133`). This key is used to sign consensus **precommit votes** (block hash signatures) and **peer identity challenges**. Because the key is embedded in public source code, any party can forge signatures that appear to originate from the sequencer's validator identity.

### Finding Description

`LocalKeyStore::new_for_testing()` in `signature_manager.rs` hardcodes both the private and public key:

```rust
// crates/apollo_signature_manager/src/signature_manager.rs, lines 101-111
pub(crate) const fn new_for_testing() -> Self {
    const PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
        "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
    ));
    const PUBLIC_KEY: PublicKey = PublicKey(Felt::from_hex_unchecked(
        "0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a",
    ));
    Self { private_key: PRIVATE_KEY, public_key: PUBLIC_KEY }
}
``` [1](#0-0) 

The production `lib.rs` exports `create_signature_manager()` — with **no `#[cfg(test)]` or `#[cfg(feature = "testing")]` gate** — and it calls directly into `new_for_testing()`:

```rust
// crates/apollo_signature_manager/src/lib.rs, lines 17-43

        Self(GenericSignatureManager::new(LocalKeyStore::new_for_testing()))

// ...
pub fn create_signature_manager() -> SignatureManager {
    SignatureManager::new()

``` [2](#0-1) 

The legitimate constructor `LocalKeyStore::_new(private_key: PrivateKey)` is dead code (underscore-prefixed, never called outside the file): [3](#0-2) 

`create_signature_manager()` is invoked from the production node startup in `crates/apollo_node/src/components.rs`, wiring the hardcoded key into the live `ConsensusManager` via `SharedSignatureManagerClient`. [4](#0-3) 

The `SignatureManager` uses this key for two operations:

- `sign_precommit_vote(block_hash)` — produces the ECDSA signature attached to every consensus precommit vote.
- `sign_identification(peer_id, challenge)` — authenticates the node's validator identity to peers. [5](#0-4) 

### Impact Explanation

The private key `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` is publicly visible in the repository. Any party can:

1. **Forge precommit votes** for an arbitrary block hash. Because the corresponding public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a` is what peers use to verify the sequencer's votes, a forged signature will pass `verify_precommit_vote_signature`. This allows an attacker to inject consensus votes that appear to originate from the legitimate sequencer validator, potentially driving consensus toward an attacker-chosen block hash.

2. **Impersonate the sequencer's peer identity** by forging `sign_identification` responses, bypassing the peer authentication handshake.

This maps to the allowed impact: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload"* and *"Wrong state, receipt, event … or revert result"* via a corrupted consensus finalization path.

### Likelihood Explanation

The key is in plain text in a public Rust source file. No special access, privilege, or network position is required. Any developer who clones the repository or reads the source has the key. The `_new` constructor being dead code confirms no runtime override path exists.

### Recommendation

1. **Remove `new_for_testing()` from production paths.** Gate it strictly with `#[cfg(any(feature = "testing", test))]`.
2. **Activate `_new(private_key)`** as the sole production constructor, rename it to `new`, and require the caller to supply a key loaded from a secrets manager (e.g., the Kubernetes `ExternalSecret` infrastructure already present in the repo).
3. **Rotate the exposed key** immediately; treat `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` as compromised.
4. Add a CI lint/secret-scan rule that rejects any `Felt::from_hex_unchecked` literal inside a non-test `KeyStore` implementation.

### Proof of Concept

```rust
// Attacker PoC — no special access needed, key is public
use starknet_core::crypto::ecdsa_sign;
use starknet_core::types::Felt;

let stolen_private_key = Felt::from_hex_unchecked(
    "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
);

// Forge a precommit vote for any block hash the attacker chooses
let target_block_hash: Felt = /* attacker-chosen value */;
let mut message = b"PRECOMMIT_VOTE".to_vec();
message.extend_from_slice(&target_block_hash.to_bytes_be());
let digest = blake2s_to_felt(&message);

let forged_signature = ecdsa_sign(&stolen_private_key, &digest).unwrap();
// forged_signature passes verify_precommit_vote_signature() on every peer
// because they all trust public key 0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a
```

The forged vote is indistinguishable from a legitimate sequencer precommit, satisfying the consensus quorum check and potentially finalizing an attacker-chosen block.
