### Title
Stale `parameters_votes` Not Cleared on Resharing Cancellation Enables Single-Participant Governance Bypass - (File: crates/contract/src/state/resharing.rs)

### Summary

When a resharing is cancelled via `vote_cancel_resharing`, the contract returns `previous_running_state.clone()` directly. If participants cast re-proposal votes (via `vote_new_parameters` during resharing) that did not reach consensus, those votes accumulate in `previous_running_state.parameters_votes` and are **never cleared** before the state is restored as the new Running state. A single Byzantine participant can then exploit these stale votes to trigger a new resharing transition without the required unanimous consent.

### Finding Description

`ResharingContractState` holds a `previous_running_state: RunningContractState` field. During resharing, participants may call `vote_new_parameters` to propose a re-proposal (a new participant set / threshold). This call routes through `self.previous_running_state.process_new_parameters_proposal(proposal)`, which mutates `previous_running_state.parameters_votes` by inserting the signer's vote. [1](#0-0) 

If the re-proposal does not reach unanimous consensus (e.g., only participants A and B vote for proposal P2, but not C), the votes remain in `previous_running_state.parameters_votes`.

When `vote_cancel_resharing` subsequently reaches threshold and transitions back to Running state, it returns `self.previous_running_state.clone()` verbatim: [2](#0-1) 

This clone carries A and B's stale votes for P2 into the new Running state. By contrast, `RunningContractState::new()` — which is called in every other transition path — explicitly resets `parameters_votes` to `ThresholdParametersVotes::default()`: [3](#0-2) 

Back in Running state, participant C calls `vote_new_parameters` with proposal P2 and the correct new epoch ID. Inside `process_new_parameters_proposal`, `ThresholdParametersVotes::vote` counts **all** stored votes for P2 — including A and B's stale entries — and returns `n_votes = 3`. Since `new_num_participants == 3`, the function returns `true` and the contract immediately transitions to Resharing: [4](#0-3) 

The epoch-ID guard only validates the `prospective_epoch_id` argument supplied by C; it does not invalidate the stale votes already stored in `parameters_votes`. [5](#0-4) 

### Impact Explanation

A single Byzantine participant (C) can force the contract into Resharing state using stale votes from participants who no longer consent. This breaks the governance invariant that **all** proposed participants must actively vote in the current Running-state context before a resharing transition is permitted. The disruption:

- Halts normal signing operations for the duration of the unwanted resharing.
- Allows C to drive the contract into a governance state that A and B explicitly cancelled, without their renewed consent.
- If P2 carries a lowered governance threshold, a successful resharing (requiring A and B's off-chain cooperation) would reduce the signing threshold, enabling future unauthorized signature issuance with fewer participants.

This matches: **Medium — contract execution-flow manipulation that breaks production safety/accounting invariants.**

### Likelihood Explanation

The preconditions are realistic in a Byzantine-participant model:

1. A resharing is initiated and then cancelled — a documented, tested flow.
2. During the resharing, at least one re-proposal vote is cast (participants routinely call `vote_new_parameters` during resharing to adjust parameters).
3. A single participant (C) survives into the restored Running state and submits the same proposal P2.

No privileged access, leaked keys, or network-level interference is required. The entire attack is executed through normal on-chain contract calls.

### Recommendation

In `vote_cancel_resharing`, reset `parameters_votes` before returning the previous running state, mirroring what `RunningContractState::new()` does:

```rust
// In ResharingContractState::vote_cancel_resharing
let running_state = if threshold_cancellation_votes_reached {
    let mut previous_running_state = self.previous_running_state.clone();
    previous_running_state.previously_cancelled_resharing_epoch_id =
        Some(prospective_epoch_id);
    // Clear stale re-proposal votes accumulated during resharing
    previous_running_state.parameters_votes = ThresholdParametersVotes::default();
    Some(previous_running_state)
} else {
    None
};
``` [6](#0-5) 

### Proof of Concept

**Setup**: Running state, participants A, B, C, governance threshold = 2, unanimous vote (all 3) required for resharing.

1. A, B, C each call `vote_new_parameters(epoch+1, P1)` → contract transitions to `ResharingContractState`. `previous_running_state.parameters_votes` is empty (created via `RunningContractState::new()`).

2. During resharing, A calls `vote_new_parameters(epoch+2, P2)` → `previous_running_state.parameters_votes` now contains `{A → P2}`.

3. B calls `vote_new_parameters(epoch+2, P2)` → `previous_running_state.parameters_votes` now contains `{A → P2, B → P2}`. Count = 2 < 3, no transition.

4. A and B call `vote_cancel_resharing()` → threshold (2) reached. Contract returns `previous_running_state.clone()` as new Running state. **`parameters_votes` still contains `{A → P2, B → P2}`.**

5. C calls `vote_new_parameters(epoch+3, P2)` in the restored Running state. `process_new_parameters_proposal` calls `parameters_votes.vote(P2, C)`, which counts `{A, B, C}` = 3 votes for P2. `new_num_participants == 3` → returns `true` → **contract transitions to Resharing with P2, using only C's active vote.** [7](#0-6) [8](#0-7)

    pub fn vote_cancel_resharing(&mut self) -> Result<Option<RunningContractState>, Error> {
        let previous_running_participants = self.previous_running_state.parameters.participants();
        let authenticated_candidate = AuthenticatedAccountId::new(previous_running_participants)?;
        self.cancellation_requests.insert(authenticated_candidate);

        let cancellation_votes_count = self.cancellation_requests.len() as u64;
        let previous_running_threshold = self.previous_running_state.parameters.threshold();

        let threshold_cancellation_votes_reached: bool =
            cancellation_votes_count >= previous_running_threshold.value();

        let running_state = if threshold_cancellation_votes_reached {
            let mut previous_running_state = self.previous_running_state.clone();
            let prospective_epoch_id = self.prospective_epoch_id();
            previous_running_state.previously_cancelled_resharing_epoch_id =
                Some(prospective_epoch_id);

            Some(previous_running_state)
        } else {
            None
        };

        Ok(running_state)

    pub fn vote(
        &mut self,
        proposal: &ProposedThresholdParameters,
        participant: AuthenticatedAccountId,
    ) -> u64 {
        if self
            .proposal_by_account
            .insert(participant, proposal.clone())
            .is_some()
        {
            log!("removed one vote for signer");
        }
        u64::try_from(
            self.proposal_by_account
                .values()
                .filter(|&prop| prop == proposal)
                .count(),
        )
        .expect("usize should never fail to convert on u64 on wasm32")
    }
