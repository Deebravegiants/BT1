### Title
Unprivileged Transaction Sender Can Arbitrarily Increase Any Registered Account Balance via Unguarded `directDeposits` Field - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Entities.hs`)

---

### Summary

The Dijkstra era introduces a new `directDeposits` transaction body field that allows any transaction sender to increase the balance of any registered stake account. The `ENTITIES` rule applies these deposits with no witness or authorization check from the recipient account owner. This directly parallels the `RM_UpdateReward` pattern: a balance-modifying operation that should be restricted to authorized callers is reachable by any unprivileged user.

---

### Finding Description

The Dijkstra era adds `DirectDeposits` as a new optional field in the transaction body (CDDL map key `25`):

```
direct_deposits = {+ reward_account => coin}
``` [1](#0-0) 

The `dijkstraEntitiesTransition` function in `ENTITIES` processes this field as follows:

```haskell
let directDeposits = tx ^. bodyTxL . directDepositsTxBodyL
    accountsAfterCerts = certStateAfterCerts ^. certDStateL . accountsL
failOnJust (directDepositsMissingAccounts directDeposits accountsAfterCerts) $
  injectFailure . DirectDepositsToMissingAccounts

pure $ certStateAfterCerts & certDStateL . accountsL %~ applyDirectDeposits directDeposits
``` [2](#0-1) 

The **only** validation performed is:
1. Network ID check (`validateWrongNetworkInDirectDeposit`)
2. That target accounts are registered (`directDepositsMissingAccounts`) [3](#0-2) 

There is **no witness requirement** from the recipient account owner. The `dijkstraUtxowTransition` calls `Shelley.validateNeededWitnesses`, which uses `getWitsVKeyNeeded`. That function collects witnesses for UTxO inputs, withdrawals, certificates, and voters — but **not** for direct deposit recipients: [4](#0-3) [5](#0-4) 

The same gap exists in `dijkstraSubEntitiesTransition` for sub-transactions: [6](#0-5) 

The `applyDirectDeposits` function itself explicitly documents that it performs no authorization checks:

```haskell
-- | Add each direct-deposit amount to the matching account balance.
--
-- /Note/ - There are no checks that direct deposits mention only registered accounts.
applyDirectDeposits :: EraAccounts era => DirectDeposits -> Accounts era -> Accounts era
``` [7](#0-6) 

---

### Impact Explanation

**Violated invariant**: The foundational Cardano design principle states that reward accounts "can not receive funds via transactions. Instead, their balance is only increased when rewards are distributed." [8](#0-7) 

Any unprivileged transaction sender can:

1. **Force-deposit coin into any registered account** without the account owner's knowledge or consent, bypassing the reward distribution mechanism entirely.

2. **Block stake credential deregistration**: The Conway `DELEG` rule enforces that an account with a non-zero balance cannot be deregistered (`StakeKeyHasNonZeroAccountBalanceDELEG`): [9](#0-8) 

   An attacker can send 1 lovelace to any registered account, preventing deregistration until the victim withdraws. For **script-based stake credentials** whose withdrawal script cannot be satisfied (e.g., a time-locked or always-failing script), this constitutes a **permanent freeze** of the account's deposit — recovery would require a hard fork.

3. **Manipulate governance/staking power**: Increasing an account's balance increases the stake delegated to its pool and the voting weight of its DRep delegation, outside the normal reward distribution mechanism.

This matches the allowed impact: *"Medium. Attacker-controlled transactions… modify fees, deposits, refunds, rewards, treasury donations, or withdrawals outside design parameters."* The permanent-freeze scenario for script accounts also reaches *"High. Permanent freezing of funds, deposits, rewards, or withdrawals where recovery requires a hard fork."*

---

### Likelihood Explanation

**High.** Any user who can submit a valid Dijkstra era transaction can include a `directDeposits` map targeting any registered account address. No special keys, governance role, or privileged access is required. The attacker only needs to know the target's stake credential (publicly observable on-chain) and pay the deposit amount plus transaction fees.

---

### Recommendation

Add a witness requirement for each recipient account address in `directDeposits`, analogous to how withdrawals require the stake credential witness:

```haskell
-- In getShelleyWitsVKeyNeededNoGov or the Dijkstra equivalent:
directDepositAuthors :: Set (KeyHash Witness)
directDepositAuthors = Map.foldrWithKey' accum Set.empty
    (unDirectDeposits $ txBody ^. directDepositsTxBodyL)
  where
    accum key _ !ans =
      let cred = key ^. accountAddressCredentialL
       in case credKeyHashWitness cred of
            Nothing -> ans
            Just vkeyWit -> Set.insert vkeyWit ans
```

This mirrors the existing `wdrlAuthors` pattern in `getShelleyWitsVKeyNeededNoGov`: [10](#0-9) 

Script-based accounts should require the corresponding script witness, mirroring the withdrawal script witness pattern.

---

### Proof of Concept

1. Alice registers a stake credential `cred_alice` (key-hash or script-hash).
2. Bob constructs a Dijkstra era transaction with:
   - Any valid UTxO input covering fees + 1 lovelace
   - `directDeposits = { AccountAddress Testnet (AccountId cred_alice) => Coin 1 }`
   - No witness for `cred_alice`
3. The transaction passes `dijkstraUtxowTransition` (no witness required for direct deposit recipients) and `dijkstraEntitiesTransition` (only checks that `cred_alice` is registered).
4. Alice's account balance is now 1 lovelace. She cannot deregister her stake credential (`StakeKeyHasNonZeroAccountBalanceDELEG`) until she withdraws — which, for a script credential whose script cannot be satisfied, is impossible without a hard fork.

The entry path is: unprivileged transaction sender → Dijkstra `LEDGER` → `UTXOW` (no direct-deposit witness check) → `ENTITIES` (`applyDirectDeposits` with no authorization) → account balance modified. [11](#0-10)

dijkstraEntitiesTransition = do
  TRC (EntitiesEnv legacyMode certsEnv, certState, certificates) <- judgmentContext
  let Conway.CertsEnv tx pp curEpoch _committee _committeeProposals = certsEnv
      withdrawals = tx ^. bodyTxL . withdrawalsTxBodyL
      accounts = certState ^. certDStateL . accountsL

  runTest $ Conway.validateWithdrawalsDelegated accounts tx

  network <- liftSTS $ asks networkId

  validateWithdrawals legacyMode network withdrawals accounts

  let certStateBeforeCerts =
        certState
          & Conway.updateDormantDRepExpiries tx curEpoch
          & Conway.updateVotingDRepExpiries tx curEpoch (pp ^. ppDRepActivityL)

    trans @(EraRule "CERTS" era) $ TRC (certsEnv, certStateBeforeCerts, certificates)

getShelleyWitsVKeyNeededNoGov ::
  EraTx era =>
  UTxO era ->
  TxBody l era ->
  Set (KeyHash Witness)
getShelleyWitsVKeyNeededNoGov utxo' txBody =
  certAuthors
    `Set.union` inputAuthors
    `Set.union` owners
    `Set.union` wdrlAuthors
  where
    inputAuthors :: Set (KeyHash Witness)
    inputAuthors = foldr' accum Set.empty (txBody ^. spendableInputsTxBodyF)
      where
        accum txin !ans =
          case txinLookup txin utxo' of
            Just txOut ->
              case txOut ^. addrTxOutL of
                Addr _ (KeyHashObj pay) _ -> Set.insert (asWitness pay) ans
                AddrBootstrap bootAddr ->
                  Set.insert (asWitness (bootstrapKeyHash bootAddr)) ans
                _ -> ans
            Nothing -> ans

    wdrlAuthors :: Set (KeyHash Witness)
    wdrlAuthors = Map.foldrWithKey' accum Set.empty (unWithdrawals (txBody ^. withdrawalsTxBodyL))
      where
        accum key _ !ans =
          let cred = key ^. accountAddressCredentialL
           in case credKeyHashWitness cred of
                Nothing -> ans
                Just vkeyWit -> Set.insert vkeyWit ans
    owners :: Set (KeyHash Witness)
    owners = foldr' accum Set.empty (txBody ^. certsTxBodyL)
      where
        accum (RegPoolTxCert pool) !ans =
          Set.union
            (Set.map asWitness (sppOwners pool))
            ans
        accum _cert ans = ans
    certAuthors :: Set (KeyHash Witness)
    certAuthors = foldr' accum Set.empty (txBody ^. certsTxBodyL)
      where
        accum cert !ans =
          case getVKeyWitnessTxCert cert of
            Nothing -> ans
