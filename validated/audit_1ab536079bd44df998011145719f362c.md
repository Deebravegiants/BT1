### Title
Missing Failure-Status Check Before Processing NEP-141 Events from Execution Outcomes - (File: chain/rosetta-rpc/src/adapters/nep141.rs)

### Summary

`collect_nep141_events` in the Rosetta RPC adapter iterates over all execution outcomes for a block and extracts `ft_transfer` EVENT_JSON log entries without first checking whether the execution outcome has a `Failure` status. Because NEAR records logs in `ExecutionOutcome` regardless of whether the receipt succeeded or failed, a malicious user can deploy a contract that emits fake NEP-141 `ft_transfer` EVENT_JSON logs inside a receipt that then panics, causing the Rosetta RPC to include fabricated fungible-token transfer operations in its block response.

### Finding Description

In `chain/rosetta-rpc/src/adapters/nep141.rs`, `collect_nep141_events` loops over every `ExecutionOutcomeWithIdView` in the block and calls `extract_events` on each one:

```rust
pub(crate) fn collect_nep141_events(
    receipt_execution_outcomes: &Vec<ExecutionOutcomeWithIdView>,
    ...
) -> crate::errors::Result<Vec<FungibleTokenEvent>> {
    let mut res = Vec::new();
    for outcome in receipt_execution_outcomes {
        let events = extract_events(outcome);   // ← no status check
        for event in events {
            res.extend(compose_rosetta_nep141_events(...)?);
        }
    }
    Ok(res)
}

### Impact Explanation

Any unprivileged user can deploy a NEAR smart contract that:
1. Emits one or more `EVENT_JSON:{"standard":"nep141","version":"1.0.0","event":"ft_transfer","data":[...]}` log lines naming arbitrary accounts and amounts.
2. Immediately panics, causing the receipt to finish with `ExecutionStatus::Failure`.

Because `collect_nep141_events` never checks `outcome.outcome.status`, the Rosetta RPC will include these fabricated `FungibleTokenEvent` entries in the block response. Exchanges, indexers, and custodians that rely on the Rosetta `/block` endpoint to track NEP-141 token balances will receive a corrupted view of which transfers occurred in that block. The concrete corrupted value is the `events` field of `ExecutionToReceipts` and the resulting Rosetta block transaction operations returned by the RPC.

### Likelihood Explanation

Medium. The Rosetta RPC must be enabled (it is an optional component), but when it is, any account holder can trigger this by submitting a single function-call transaction to a contract they control. No special privileges are required.

### Recommendation

In `collect_nep141_events` (or in `extract_events`), skip any outcome whose status is `ExecutionStatusView::Failure` or `ExecutionStatusView::Unknown` before extracting and composing events:

```rust
for outcome in receipt_execution_outcomes {
    if matches!(
        outcome.outcome.status,
        ExecutionStatusView::Failure(_) | ExecutionStatusView::Unknown
    ) {
        continue;
    }
    let events = extract_events(outcome);
    ...
}
```

### Proof of Concept

1. Deploy a contract with a method that logs `EVENT_JSON:{"standard":"nep141","version":"1.0.0","event":"ft_transfer","data":[{"old_owner_id":"victim.near","new_owner_id":"attacker.near","amount":"1000000000000000000000000"}]}` and then calls `env::panic_str("revert")`.
2. Call that method from any account.
3. The receipt lands on-chain with `ExecutionStatus::Failure`; the log is still stored in the `ExecutionOutcome`.
4. Query the Rosetta `/block` endpoint for that block with the relevant currency configured.
5. Observe that the response contains a `ft_transfer` operation crediting `attacker.near` with 1 NEAR-worth of tokens, even though no actual token transfer occurred.

pub(crate) fn extract_events(
    execution_outcome: &ExecutionOutcomeWithIdView,
) -> Vec<crate::models::Nep141Event> {
    let prefix = "EVENT_JSON:";
    execution_outcome
        .outcome
        .logs
        .iter()
        .filter_map(|untrimmed_log| {
            let log = untrimmed_log.trim();
            if !log.starts_with(prefix) {
                return None;
            }

            match serde_json::from_str::<'_, crate::models::Nep141Event>(log[prefix.len()..].trim())
            {
                Ok(result) => Some(result),
                Err(_err) => None,
            }
        })
        .collect()
}
