Audit Report

## Title
Mismatched Units Between Minimum Deposit and Minimum Withdrawal Thresholds Can Temporarily Freeze User rsETH - (`contracts/LRTWithdrawalManager.sol` / `contracts/LRTDepositPool.sol`)

## Summary
`LRTDepositPool.minAmountToDeposit` is denominated in ETH/asset units while `LRTWithdrawalManager.minRsEthAmountToWithdraw[asset]` is denominated in rsETH units. When an admin sets both to the same nominal value and the rsETH price exceeds 1 ETH/rsETH, a user depositing exactly the minimum receives fewer rsETH than the withdrawal minimum requires, blocking all withdrawal paths until admin intervention corrects the parameters.

## Finding Description
`LRTDepositPool._beforeDeposit()` at line 657 enforces:
```solidity
if (depositAmount == 0 || depositAmount < minAmountToDeposit) {
    revert InvalidAmountToDeposit();
}
```
Here `depositAmount` is in ETH or LST asset units.

`LRTWithdrawalManager.initiateWithdrawal()` at line 162 and `instantWithdrawal()` at line 224 both enforce:
```solidity
if (rsETHUnstaked == 0 || rsETHUnstaked < minRsEthAmountToWithdraw[asset]) {
    revert InvalidAmountToWithdraw();
}
```
Here `rsETHUnstaked` is in rsETH token units.

`getRsETHAmountToMint()` at line 520 computes:
```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```
When `rsETHPrice > 1e18` (i.e., rsETH has appreciated), depositing exactly `minAmountToDeposit` ETH yields strictly fewer rsETH than `minAmountToDeposit` in raw token units. If the admin sets both parameters to the same nominal value (e.g., `0.05e18`), the user's resulting rsETH balance falls below `minRsEthAmountToWithdraw[asset]`, causing both `initiateWithdrawal` and `instantWithdrawal` to revert. The two setters — `setMinAmountToDeposit` (line 282) and `setMinRsEthAmountToWithdraw` (line 330) — have no enforced relationship, making this misconfiguration silently possible.

## Impact Explanation
Affected users cannot call `initiateWithdrawal` or `instantWithdrawal` because their rsETH balance is below the withdrawal minimum. The admin can remediate by lowering `minRsEthAmountToWithdraw` or unpausing the deposit pool, so the freeze is **temporary** rather than permanent. This matches **Medium — Temporary freezing of funds**.

## Likelihood Explanation
Two concurrent conditions are required: (1) the admin sets both minimums to the same nominal value — a natural mistake given both parameters superficially represent "minimum amount" but are in different units; (2) `LRTDepositPool` is paused (via `PAUSER_ROLE`) or the per-asset deposit cap is exhausted, preventing the user from topping up. Both conditions are routine operational events. The rsETH price diverges from 1:1 as soon as EigenLayer rewards begin accruing, making the unit gap active early in the protocol's life. Likelihood is **medium**.

## Recommendation
Enforce a consistent relationship between the two minimums. The simplest fix is to validate in `setMinRsEthAmountToWithdraw` that the rsETH minimum, when converted to ETH at the current oracle price, does not exceed `minAmountToDeposit`. Alternatively, express `minRsEthAmountToWithdraw[asset]` in ETH-equivalent terms and convert at withdrawal time, or add a NatSpec warning clearly documenting the unit difference so operators set the withdrawal minimum conservatively below the deposit minimum in rsETH terms.

## Proof of Concept
```
Setup:
  minAmountToDeposit       = 0.05e18  (ETH units, set via LRTDepositPool.setMinAmountToDeposit)
  minRsEthAmountToWithdraw = 0.05e18  (rsETH units, set via LRTWithdrawalManager.setMinRsEthAmountToWithdraw)
  rsETHPrice               = 1.10e18  (ETH per rsETH, from LRTOracle after rewards accrue)

Step 1 — Alice deposits the protocol minimum:
  Alice calls LRTDepositPool.depositETH{value: 0.05e18}(0, "")
  rsethAmountToMint = (0.05e18 * 1e18) / 1.10e18 ≈ 0.04545e18 rsETH
  → Alice holds 0.04545e18 rsETH (below 0.05e18 withdrawal minimum)

Step 2 — LRTDepositPool is paused (normal operational event):
  PAUSER_ROLE calls LRTDepositPool.pause()

Step 3 — Alice tries to withdraw via initiateWithdrawal:
  0.04545e18 < 0.05e18 → revert InvalidAmountToWithdraw()

Step 4 — Alice tries instantWithdrawal:
  0.04545e18 < 0.05e18 → revert InvalidAmountToWithdraw()

Step 5 — Alice tries to top up to meet the withdrawal minimum:
  LRTDepositPool.depositETH{value: 0.01e18}(0, "")
  → revert "Pausable: paused"

Result: Alice's rsETH is frozen until admin lowers minRsEthAmountToWithdraw or unpauses the pool.
```

Foundry test plan: deploy `LRTDepositPool` and `LRTWithdrawalManager` with a mock oracle returning `rsETHPrice = 1.1e18`; set both minimums to `0.05e18`; have a test address deposit `0.05e18` ETH; pause the deposit pool; assert both `initiateWithdrawal` and `instantWithdrawal` revert with `InvalidAmountToWithdraw` for the full received rsETH balance.
