Audit Report

## Title
Permissionless `claim()` Allows Any Caller to Force Fee Deduction on Any User's Reward — (`contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

## Summary

`MerkleDistributor.claim()` accepts an arbitrary `account` parameter with no `msg.sender == account` guard. Any unprivileged caller who possesses a valid merkle proof for a victim (derivable from public off-chain data) can trigger the victim's claim, permanently deducting up to 10% as a non-refundable fee sent to `protocolTreasury`, without the victim's consent. The sibling contract `KernelMerkleDistributor` correctly blocks this with an `Unauthorized` revert, but `MerkleDistributor` has no such protection.

## Finding Description

`MerkleDistributor.claim()` is a permissionless `external` function: [1](#0-0) 

There is no check that `msg.sender == account`. After verifying the merkle proof, the function immediately deducts the fee and transfers it irrecoverably to `protocolTreasury`: [2](#0-1) 

The victim's claim state is then permanently updated: [3](#0-2) 

By contrast, `KernelMerkleDistributor._processClaim()` enforces caller identity before any state change: [4](#0-3) 

`MerkleDistributor` is missing this guard entirely. The `feeInBPS` can be up to `MAX_FEE_IN_BPS = 1000` (10%): [5](#0-4) 

## Impact Explanation

**High — Theft of unclaimed yield.**

A victim's claimable reward tokens are reduced by up to 10% (`feeInBPS / 10_000 × claimableAmount`) without their knowledge or consent. The fee is irrecoverably transferred to `protocolTreasury` with no refund path. If the owner intends to lower `feeInBPS`, an attacker can race to force the victim's claim at the current higher rate, permanently extracting more yield than the victim would have paid voluntarily. This matches the allowed impact "Theft of unclaimed yield."

## Likelihood Explanation

**High.** Merkle proofs are computed from publicly available off-chain data. Any external caller can reconstruct a valid `(index, account, cumulativeAmount, merkleProof)` tuple for any eligible address. No special privilege, leaked key, or oracle manipulation is required. The only precondition is that the contract is not paused and the victim has an unclaimed balance. The attack is repeatable across all eligible accounts in every distribution round.

## Recommendation

Add the same caller-identity check already present in `KernelMerkleDistributor._processClaim()` at the top of `MerkleDistributor.claim()`, before any state changes or transfers:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Also add the `Unauthorized` error to `IMerkleDistributor` in `MerkleDistributor.sol` to match. [4](#0-3) 

## Proof of Concept

**Foundry test plan:**

```solidity
function test_forcedClaimDeductsFeeWithoutConsent() public {
    // Setup: deploy MerkleDistributor with feeInBPS = 500 (5%)
    // Mint 1000e18 tokens to distributor, build merkle tree with alice's leaf
    // Set merkle root

    address alice = makeAddr("alice");
    address bob   = makeAddr("bob");

    // Bob calls claim() with Alice's valid proof
    vm.prank(bob);
    distributor.claim(index, alice, 1000e18, aliceProof);

    // Alice receives only 950e18; 50e18 sent to protocolTreasury
    assertEq(token.balanceOf(alice),           950e18);
    assertEq(token.balanceOf(protocolTreasury), 50e18);

    // Alice's claim is permanently consumed — she cannot reclaim
    assertTrue(distributor.isClaimed(index, alice));
}
```

Steps:
1. Protocol deploys `MerkleDistributor` with `feeInBPS = 500`.
2. Alice is entitled to `1000e18` reward tokens; her merkle proof is publicly derivable.
3. Bob calls `claim(index, alice, 1000e18, aliceProof)` — no privilege required.
4. Contract deducts `fee = 50e18`, sends `950e18` to Alice, `50e18` to `protocolTreasury`.
5. `userClaims[alice].cumulativeAmount` is permanently updated; Alice cannot reclaim.
6. If the owner had planned to call `setFeeInBPS(0)` the next block, Alice permanently loses the 50-token difference with no recourse. [6](#0-5)

```text
    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
