### Title
Stale Attestation Check in `is_caller_an_attested_participant` Allows Expired-TEE Node to Submit Threshold Signatures — (File: `crates/contract/src/tee/tee_state.rs`)

---

### Summary

`is_caller_an_attested_participant` only verifies that a stored attestation **exists** for the calling node; it never re-verifies whether that attestation is **currently valid** (not expired, not using a revoked image hash). This is the direct analog of the external report's `subDir[author][ref] != 0` check: both confirm a historical record rather than an active, live state. As a result, a participant node whose TEE attestation has expired (or whose MPC image hash has been revoked) can continue to call `respond()`, `respond_ckd()`, and `respond_verify_foreign_tx()` — the three functions that deliver threshold signatures and CKD outputs to the contract — for as long as it remains in the participant set and `accept_requests` is `true`.

---

### Finding Description

`assert_caller_is_attested_participant_and_protocol_active()` is the gate that protects every signing-response endpoint. It delegates the attestation check to `is_caller_an_attested_participant`: [1](#0-0) 

The function performs four checks:
1. Is the caller in the active participant list?
2. Is there a stored attestation keyed by the caller's TLS public key?
3. Does the stored `node_id.account_id` match the signer?
4. Does the stored `node_id.account_public_key` match the signer's Ed25519 key?

**What it never does** is call `reverify_participants` to confirm the stored `VerifiedAttestation` still passes `re_verify` at the current block timestamp: [2](#0-1) 

`re_verify` checks expiry, revoked image hashes, and revoked launcher compose hashes: [3](#0-2) 

The three response endpoints that rely on this gate are: [4](#0-3) [5](#0-4) [6](#0-5) 

The only other guard is the global `accept_requests` flag, which is only set to `false` by `verify_tee()` when **fewer than threshold** participants have valid attestations: [7](#0-6) 

If only one participant's attestation expires while the remaining set still meets threshold, `accept_requests` stays `true`, `verify_tee()` triggers a resharing (not an immediate ban), and the expired node remains in the participant set until resharing completes. During that entire window the expired node passes `is_caller_an_attested_participant` and can call `respond()`.

---

### Impact Explanation

A node whose TEE certificate has expired, or whose MPC image hash has been revoked by a governance vote, can continue to submit threshold signature responses (`respond`), confidential key derivation outputs (`respond_ckd`), and foreign-chain verification responses (`respond_verify_foreign_tx`) to the contract. The TEE attestation requirement exists precisely to guarantee that key shares are only exercised inside a currently-valid trusted execution environment. Accepting responses from a node that no longer satisfies this requirement breaks the production safety invariant of the TEE security model: the contract treats a historically-attested node as currently-attested, mirroring the external report's pattern of checking `subDir[author][ref] != 0` (ever subscribed) instead of `subDir[author][ref] > block.timestamp` (currently subscribed).

This maps to the **Medium** allowed impact: participant-state and contract execution-flow manipulation that breaks production safety/accounting invariants (the TEE attestation requirement for signing participation) without relying on network-level DoS or operator misconfiguration.

---

### Likelihood Explanation

TEE certificates have a finite validity window (`DEFAULT_EXPIRATION_DURATION_SECONDS`, currently 7 days per the design docs). Every production node must resubmit its attestation hourly. A node that fails to renew — whether due to a software fault, a revoked image hash after a governance vote, or deliberate inaction — will have an expired attestation in `stored_attestations` while remaining in the active participant set until `verify_tee()` is called and resharing completes. The resharing process itself takes multiple blocks and requires threshold cooperation. The window is therefore measured in minutes to hours on a live network, and the entry path requires no privileged access: any current participant can let its attestation lapse.

---

### Recommendation

Add a live re-verification call inside `is_caller_an_attested_participant` (or inside `assert_caller_is_attested_participant_and_protocol_active`) before returning `Ok(())`:

```rust
// After confirming account_id and account_public_key match, re-verify liveness:
let tee_upgrade_deadline_duration = /* read from config or pass as parameter */;
match self.reverify_participants(&attestation.node_id, tee_upgrade_deadline_duration) {
    TeeQuoteStatus::Valid => Ok(()),
    TeeQuoteStatus::Invalid(reason) => Err(AttestationCheckError::AttestationExpiredOrRevoked(reason)),
}
```

This mirrors the external report's recommended fix (`require(subDir[author][ref] > block.timestamp)`) by replacing a historical-existence check with a current-validity check, ensuring that only nodes whose TEE attestations are valid at the time of the `respond` call can submit signing responses.

---

### Proof of Concept

1. Deploy the contract with 3 participants (threshold = 2). All three submit valid attestations with `expiry_timestamp_seconds = now + 10`.
2. Advance block time past the expiry for participant 3 only. Do **not** call `verify_tee()`.
3. Participant 3 calls `respond(request, valid_signature)`. Because `is_caller_an_attested_participant` only checks `stored_attestations.get(&tls_key)` (which still returns the expired entry) and never calls `reverify_participants`, the call succeeds.
4. Observe that `accept_requests` is still `true` (threshold participants 1 and 2 are still valid), and the expired-attestation node's response is accepted and resolves the pending yield.

The existing unit test `test_re_verify_rejects_expired_attestation` confirms that `reverify_participants` correctly rejects expired attestations — it is simply never invoked on the hot path through `is_caller_an_attested_participant`. [8](#0-7)

    /// reverifies stored participant attestations.
    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

    pub fn re_verify(
        &self,
        timestamp_seconds: u64,
        allowed_mpc_docker_image_hashes: &[NodeImageHash],
        allowed_launcher_docker_compose_hashes: &[LauncherDockerComposeHash],
        allowed_measurements: &[ExpectedMeasurements],
    ) -> Result<(), VerificationError> {
        match self {
            Self::Dstack(ValidatedDstackAttestation {
                mpc_image_hash,
                launcher_compose_hash,
                expiry_timestamp_seconds: expiration_timestamp_seconds,
                measurements,
            }) => {
                let attestation_has_expired = *expiration_timestamp_seconds < timestamp_seconds;

                if attestation_has_expired {
                    return Err(VerificationError::Custom(format!(
                        "The attestation expired at t = {:?}, time_now = {:?}",
                        expiration_timestamp_seconds, timestamp_seconds
                    )));
                }

                let () = verify_mpc_hash(mpc_image_hash, allowed_mpc_docker_image_hashes)?;
                let () = verify_launcher_compose_hash(
                    launcher_compose_hash,
                    allowed_launcher_docker_compose_hashes,
                )?;

                verify_measurements(measurements, allowed_measurements)?;

                Ok(())

                allowed_mpc_docker_image_hashes,
