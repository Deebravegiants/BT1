Audit Report

## Title
Unbounded Gas Consumption in `updateRSETHPrice()` Due to Uncapped `supportedAssetList` Driving Nested External-Call Loops - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is publicly callable and internally executes a three-level nested loop whose outermost dimension — `supportedAssetList` — has no explicit cap and grows monotonically through normal governance. As the list grows, gas cost scales as O(N × D × W) across external calls, eventually causing an out-of-gas revert that permanently prevents price updates and can trigger the protocol's auto-pause circuit breaker.

## Finding Description
The call chain is fully verified in the codebase:

1. `updateRSETHPrice()` is `public` and `whenNotPaused`, callable by any address. It delegates to `_updateRsETHPrice()`, which calls `_getTotalEthInProtocol()`. [1](#0-0) 

2. `_getTotalEthInProtocol()` fetches `supportedAssets` from `lrtConfig.getSupportedAssetList()` — an unbounded array — and loops over every entry, calling `ILRTDepositPool.getTotalAssetDeposits(asset)` for each. [2](#0-1) 

3. `getTotalAssetDeposits` calls `getAssetDistributionData(asset)`, which loops over every NDC in `nodeDelegatorQueue` and calls `INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset)` for each. [3](#0-2) 

4. `getAssetUnstaking` calls EigenLayer's `getQueuedWithdrawals(address(this))` (one external call per NDC per asset) and then iterates over all returned withdrawals and their strategies, calling `strategy.sharesToUnderlyingView(...)` for each non-ETH share. [4](#0-3) 

5. `supportedAssetList` has no cap. `_addNewSupportedAsset` only checks for duplicates and a non-zero deposit limit; it pushes unconditionally. [5](#0-4) 

By contrast, `nodeDelegatorQueue` is bounded by `maxNodeDelegatorLimit` (initialized to 10) and queued withdrawals are bounded by `maxUncompletedWithdrawalCount`. No analogous cap exists for `supportedAssetList`. [6](#0-5) [7](#0-6) 

The total gas cost is therefore **O(N_assets × N_NDCs × N_withdrawals)** external calls. At N=20 assets, D=10 NDCs, W=80 withdrawals, this is 16,000 external calls; at N=50 it is 40,000 — well beyond the Ethereum block gas limit (~30M gas).

## Impact Explanation
`rsETHPrice` is the sole on-chain exchange rate for rsETH. If `updateRSETHPrice()` reverts due to gas exhaustion, the price becomes permanently stale. Additionally, `_updateRsETHPrice()` contains a downside circuit breaker: if the new price deviates too far from `highestRsethPrice`, it auto-pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself — freezing all deposits and withdrawals. [8](#0-7) 

**Impact: Medium — Unbounded gas consumption / Temporary freezing of funds**, both of which are in the allowed impact scope.

## Likelihood Explanation
No attacker action is required. The condition is reached through routine governance expansion of the supported LST list via `addNewSupportedAsset` (TIME_LOCK_ROLE). The protocol is explicitly designed to onboard additional LSTs over time. The degradation is gradual and deterministic: each new asset added increases gas cost proportionally. Any public caller can trigger the revert once the threshold is crossed.

## Recommendation
1. Introduce an explicit cap on `supportedAssetList` in `_addNewSupportedAsset`, analogous to `maxNodeDelegatorLimit` for NDCs.
2. Refactor `_getTotalEthInProtocol()` to avoid calling `getAssetUnstaking()` (which itself calls EigenLayer's `getQueuedWithdrawals`) inside the per-asset loop. Cache or pre-aggregate unstaking amounts off-chain and push them on-chain, or split the price update into per-asset batches callable by the manager.
3. Consider separating TVL aggregation from the price-update path so that gas-heavy accounting does not block the critical price-update function.

            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
