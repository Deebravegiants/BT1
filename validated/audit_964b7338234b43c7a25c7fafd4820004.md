Audit Report

## Title
Missing Chainlink Staleness Checks in `ChainlinkPriceOracle.getAssetPrice()` Allows Stale Prices to Drive rsETH Minting - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all staleness-related return values, accepting any price the feed returns regardless of age or validity. A stale inflated price flows directly into `LRTDepositPool.getRsETHAmountToMint()`, where it inflates the numerator against a separately-stored `rsETHPrice` denominator, causing depositors to receive excess rsETH and diluting the ETH-per-rsETH ratio for all existing holders.

## Finding Description
In `ChainlinkPriceOracle.getAssetPrice()`, the call to `latestRoundData()` captures only the `price` field:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

The three discarded fields — `roundId`, `updatedAt`, and `answeredInRound` — are precisely those needed to detect stale data. No check is made for `answeredInRound >= roundId`, `updatedAt != 0`, or `price > 0`.

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` in the same repository performs all three checks:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

The stale price propagates through the following call chain:

1. `ChainlinkPriceOracle.getAssetPrice(asset)` returns the stale inflated price. [3](#0-2) 
2. `LRTOracle.getAssetPrice(asset)` passes it through without additional checks. [4](#0-3) 
3. `LRTDepositPool.getRsETHAmountToMint()` uses the live stale price as the numerator against the separately-stored `rsETHPrice` denominator: `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`. [5](#0-4) 
4. Any unprivileged depositor triggers this via `depositAsset()` or `depositETH()`. [6](#0-5) 

Because `lrtOracle.getAssetPrice(asset)` is a live call while `lrtOracle.rsETHPrice()` is a stored value updated separately via `updateRSETHPrice()`, a stale inflated Chainlink price inflates the numerator without a corresponding inflation of the denominator, causing the depositor to receive more rsETH than the true ETH value of their deposit.

## Impact Explanation
When a Chainlink feed goes stale with an inflated price, any depositor calling `depositAsset()` or `depositETH()` receives excess rsETH. This excess rsETH represents a claim on protocol TVL not backed by a corresponding deposit, diluting the ETH-per-rsETH ratio for all existing holders. This constitutes **theft of unclaimed yield from existing rsETH holders**, matching the allowed High impact scope.

## Likelihood Explanation
Chainlink feeds do go stale during network congestion, oracle node outages, or circuit-breaker events. The Chainlink documentation explicitly warns that `answeredInRound < roundId` indicates stale data. No privileged access is required — any unprivileged depositor can exploit the window while the feed is stale. The developers were demonstrably aware of the staleness check pattern, having applied it in `ChainlinkOracleForRSETHPoolCollateral`, but omitted it from the core oracle used for minting. **Likelihood: Medium.**

## Recommendation
Apply the same staleness checks already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    require(answeredInRound >= roundId, "Stale price");
    require(updatedAt != 0, "Incomplete round");
    require(price > 0, "Invalid price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept

**Foundry fork test plan:**

1. Fork mainnet at a block where a supported LST Chainlink feed (e.g., stETH/ETH) has a recent `updatedAt`.
2. Warp `block.timestamp` forward by more than the feed's heartbeat (e.g., 25 hours) without triggering a new round, simulating a stale feed.
3. Record `lrtOracle.rsETHPrice()` (stored value, not yet updated).
4. Call `lrtDepositPool.depositAsset(stETH, amount, 0, "")` as an unprivileged address.
5. Assert that `rsethAmountToMint` exceeds `(amount * truePrice) / rsETHPrice` — i.e., the depositor received more rsETH than the true ETH value of their deposit.
6. Confirm that calling `updateRSETHPrice()` after the deposit shows a reduced ETH-per-rsETH ratio, proving dilution of existing holders.

The test requires no privileged access and is reproducible on any mainnet fork where a supported feed's `updatedAt` can be made stale by time-warping.
