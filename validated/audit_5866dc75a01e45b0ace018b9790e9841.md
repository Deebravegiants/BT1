### Title
Unvalidated `builder` Field in `ProposalInit` Allows Fee Redirection to Attacker-Controlled Address — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates many `ProposalInit` fields but never checks `init.builder`. Because `builder` is passed verbatim as `sequencer_address` into `BlockInfo` for execution, a malicious proposer can set it to any address they control, redirecting all transaction fees for that block to themselves. Both the proposer's and validator's batcher execute with the same attacker-supplied `builder`, so the `ProposalFinMismatch` check passes and the block is committed with the wrong sequencer address.

---

### Finding Description

`ProposalInit` carries two identity fields:

- `proposer` — the consensus validator who proposes the block
- `builder` — the entity that builds/sequences the block (becomes `sequencer_address` in execution) [1](#0-0) 

`proposer` is validated at the consensus layer before `validate_proposal` is ever called: [2](#0-1) 

`builder`, however, is **never validated**. `ProposalInitValidation` — the struct that carries all reference values for `is_proposal_init_valid` — has no `builder` field: [3](#0-2) 

`is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`, but never touches `init.builder`: [4](#0-3) 

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`: [5](#0-4) 

The batcher then executes every transaction in the block with this `sequencer_address`. All fee transfers go to `block_context.block_info.sequencer_address`: [6](#0-5) 

Because the validator also reads `builder` from the received `ProposalInit` and passes it to its own batcher, both sides compute the same `partial_block_hash`. The `ProposalFinMismatch` guard therefore does not fire: [7](#0-6) 

The block is committed with the attacker-supplied `builder` as the canonical sequencer address, and the corrupted fee state is persisted.

---

### Impact Explanation

**Critical — Incorrect fee accounting with direct economic impact.**

Every transaction fee in the block is transferred to the attacker-controlled `builder` address instead of the legitimate sequencer. The state diff, block header, and on-chain commitment all record the wrong sequencer address. The legitimate builder receives nothing for that block; the attacker receives the full fee revenue.

---

### Likelihood Explanation

**Medium.** Any validator that wins a proposal round can execute this attack with a single field change in `ProposalInit`. No special network position, no prior state, and no cryptographic material beyond normal validator credentials is required. The attack is silent: no error is logged, no metric fires, and consensus reaches finality normally.

---

### Recommendation

Add `builder: ContractAddress` to `ProposalInitValidation` and populate it from the locally-configured `builder_address` (the same value used in `initiate_build`). In `is_proposal_init_valid`, reject any proposal whose `init.builder` does not equal the expected value:

```rust
if init_proposed.builder != proposal_init_validation.builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder
        ),
    ));
}
```

This mirrors the pattern already used for `proposer` at the consensus layer and for `starknet_version`, `l1_da_mode`, and `l2_gas_price_fri` inside `is_proposal_init_valid`.

---

### Proof of Concept

1. Malicious validator **V** is elected proposer for height H, round R.
2. **V** constructs `ProposalInit` with `builder = attacker_address` (any address **V** controls) instead of the legitimate `builder_address` from config.
3. **V** broadcasts the proposal. Every validator node receives the `ProposalInit` and calls `validate_proposal` → `is_proposal_init_valid`.
4. `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, L1 prices, `starknet_version`, `version_constant_commitment`, `fee_proposal_fri` — all pass. `builder` is never read.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: attacker_address, … }`.
6. The batcher executes all transactions with `sequencer_address = attacker_address`; every `execute_fee_transfer` sends fees to `attacker_address`.
7. `finish_proposal` returns a `partial_block_hash` computed with `attacker_address` as sequencer. The proposer's batcher computed the same hash (same `builder`). `ProposalFinMismatch` check passes.
8. Consensus reaches quorum; `decision_reached` commits the block. The state diff records fee balances credited to `attacker_address`. The legitimate builder's balance is unchanged.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
