### Title
Unvalidated `builder` Field in `ProposalInit` Allows Malicious Proposer to Inject Arbitrary Sequencer Address into Block Execution and Block Hash - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary
`is_proposal_init_valid` validates many fields of the network-supplied `ProposalInit` but never checks `init.builder`. The `builder` field is consumed verbatim as `sequencer_address` in `convert_to_sn_api_block_info`, which feeds the batcher's block execution context and the `PartialBlockHashComponents` used for the final block hash. A malicious proposer can set `builder` to any address, causing the validator to execute and commit a block whose sequencer address, fee-collection target, and block hash are all attacker-controlled.

### Finding Description

`ProposalInit` carries two identity fields:

- `proposer` — the consensus committee member who proposed the block
- `builder` — the address of the node that sequences/builds the block (becomes `sequencer_address`) [1](#0-0) 

The consensus manager checks `init.proposer` against the committee-derived expected proposer before forwarding the proposal: [2](#0-1) 

However, `is_proposal_init_valid` — the function responsible for validating all `ProposalInit` fields before the batcher is invoked — validates `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri`, but **never checks `init.builder`**: [3](#0-2) 

After `is_proposal_init_valid` returns `Ok`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address` in the `BlockInfo` passed to the batcher: [4](#0-3) 

The same `init.builder` is later written into the stored `BlockHeaderWithoutHash` as the canonical sequencer address: [5](#0-4) 

And `sequencer` is a committed input to `calculate_block_hash` via `PartialBlockHashComponents`: [6](#0-5) 

Because the proposer controls `builder` and the validator uses it without checking it, both sides compute the same `partial_block_hash` (with the attacker-chosen sequencer address), so the `ProposalFinMismatch` guard at line 244 passes: [7](#0-6) 

### Impact Explanation

A malicious proposer sets `init.builder` to an attacker-controlled address. The validator accepts the proposal (all checked fields are valid), executes the block with `sequencer_address = attacker_address`, and commits:

1. **Wrong fee collection** — transaction fees paid to the sequencer during execution are credited to the attacker's address instead of the legitimate sequencer. This is a direct economic impact matching the "Incorrect fee … balance … with economic impact" criterion.
2. **Wrong block hash** — the committed block hash encodes the attacker's address as sequencer, permanently corrupting the canonical chain state for that block.
3. **Wrong stored block header** — `BlockHeaderWithoutHash.sequencer` is persisted with the attacker's address, corrupting the authoritative RPC view of the block.

### Likelihood Explanation

Any validator node that receives a proposal from a malicious (or compromised) proposer is affected. The proposer is a legitimate consensus participant (the `proposer` field passes the committee check), so no special privilege beyond being the elected proposer for a round is required. The attack is fully unprivileged from the perspective of the missing check.

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`. The expected value is the local node's configured `builder_address` (already available in `SequencerConsensusContext`). Alternatively, if the builder address is proposer-specific and not locally known, it must at minimum be bound to the `proposer` identity through a signed attestation or a registry lookup, so that an arbitrary address cannot be injected.

### Proof of Concept

1. Attacker is the elected proposer for height H, round R.
2. Attacker constructs `ProposalInit` with all fields matching the validator's expectations **except** `builder = attacker_wallet_address`.
3. Attacker builds a valid block using its own batcher (with `sequencer_address = attacker_wallet_address`), producing a correct `ProposalFin.proposal_commitment`.
4. Validator receives the proposal, calls `is_proposal_init_valid` — passes (builder is not checked).
5. Validator calls `initiate_validation` → `convert_to_sn_api_block_info` → batcher executes block with `sequencer_address = attacker_wallet_address`.
6. Batcher returns `partial_block_hash` computed with `sequencer = attacker_wallet_address`.
7. Validator computes `batcher_block_commitment` — matches `fin.proposal_commitment` (both used the same builder value).
8. `ProposalFinMismatch` check passes; block is committed.
9. All transaction fees for block H go to `attacker_wallet_address`; the block hash and stored header permanently record the wrong sequencer. [8](#0-7) [9](#0-8)

```rust
#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
```

```rust
pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
```
