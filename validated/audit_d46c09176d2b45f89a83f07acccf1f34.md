### Title
Bare `assert` Statements in `__get_cc_sub_slot` and `__validate_pospace` Crash Weight Proof Validation on Crafted Proof - (File: `chia/full_node/weight_proof.py`)

### Summary
Two internal weight-proof validation helpers — `__get_cc_sub_slot` and `__validate_pospace` — use bare Python `assert` statements instead of graceful error returns. A malicious peer can send a crafted `WeightProof` whose segment structure triggers these asserts, raising an unhandled `AssertionError` that propagates through the entire sync path and crashes the syncing node or wallet.

### Finding Description

`__get_cc_sub_slot` (lines 1532–1554) is called from `__validate_pospace` (line 1404) whenever the challenge block is not the genesis case. It iterates backwards over sub-slots looking for a slot-end entry:

```python
def __get_cc_sub_slot(sub_slots, idx, ses):
    sub_slot: SubSlotData | None = None
    for i in reversed(range(idx)):          # empty if idx == 0
        sub_slot = sub_slots[i]
        if sub_slot.cc_slot_end_info is not None:
            break

    assert sub_slot is not None             # ← AssertionError if idx == 0
    assert sub_slot.cc_slot_end_info is not None  # ← AssertionError if no slot-end found
    ...
``` [1](#0-0) 

When `idx == 0` the `reversed(range(0))` loop body never executes, `sub_slot` stays `None`, and `assert sub_slot is not None` raises `AssertionError`.

`__validate_pospace` (lines 1392–1443) calls `__get_cc_sub_slot` whenever the condition `first_in_sub_epoch and segment.sub_epoch_n == 0 and idx == 0` is **not** met — meaning any segment in `sub_epoch_n > 0` whose challenge block sits at `idx == 0` in `sub_slots` will reach the crashing assert:

```python
else:
    cc_sub_slot_hash = __get_cc_sub_slot(segment.sub_slots, idx, ses).get_hash()
``` [2](#0-1) 

`__validate_pospace` also contains two additional bare asserts that crash on crafted input:

```python
assert curr_slot.cc_slot_end_info          # line 1413 – None if attacker omits it
assert sub_slot_data.proof_of_space is not None  # line 1424 – None if attacker omits it
``` [3](#0-2) 

The prior SEC-614 fix hardened `__get_rc_sub_slot` to return `None` gracefully, and the test `test_weight_proof_validation_challenge_at_segment_start` confirms that path is safe. However, `__get_cc_sub_slot` and `__validate_pospace` were **not** similarly hardened and remain reachable via a different crafted segment structure. [4](#0-3) 

**Reachable path for `__get_cc_sub_slot` crash:**
1. Craft a `WeightProof` with `sub_epoch_n > 0`.
2. Ensure `segments[0]` of that sub-epoch has its challenge block at `idx > 0` so `__get_rc_sub_slot` succeeds and `_validate_sub_epoch_segments` does not short-circuit.
3. Craft a later segment (the one that will be sampled) so its challenge block is at `idx == 0` in `sub_slots`.
4. `_validate_segment` → `__validate_pospace` → `__get_cc_sub_slot(sub_slots, 0, ses)` → `AssertionError`. [5](#0-4) 

The `AssertionError` is not caught anywhere in the call stack:

```
_validate_sub_epoch_segments
  → _validate_segment
    → __validate_pospace
      → __get_cc_sub_slot   ← AssertionError raised here
```

propagates through `validate_weight_proof_inner` → `validate_weight_proof` → `WalletWeightProofHandler.validate_weight_proof` with no intervening `try/except`. [6](#0-5) 

### Impact Explanation

Weight proof validation is the critical path for long-sync. A malicious peer that serves a crafted `WeightProof` causes an `AssertionError` to propagate out of `WalletWeightProofHandler.validate_weight_proof` (or `WeightProofHandler.validate_weight_proof`), crashing the sync task. If the attacker controls the peer the victim connects to (e.g., via eclipse or by being the only reachable peer), the victim node or wallet is permanently unable to complete long-sync. This matches the **High** impact class: long-lived inability for honest nodes/wallets to process sync updates.

### Likelihood Explanation

Any unprivileged network peer can send a `WeightProof` message. The attacker only needs to take a legitimately-structured weight proof (obtainable from the public network), strip the leading slot-end entries from one sampled segment so its challenge block lands at `idx == 0`, and serve it to a syncing victim. No keys, admin access, or cryptographic breaks are required.

### Recommendation

Replace the bare `assert` statements in `__get_cc_sub_slot` and `__validate_pospace` with explicit checks that return `None` (matching the pattern already applied to `__get_rc_sub_slot`):

In `__get_cc_sub_slot`:
```python
if sub_slot is None or sub_slot.cc_slot_end_info is None:
    log.error("segment missing slot-end entry before challenge block")
    return None   # change return type to ChallengeChainSubSlot | None
```

In `__validate_pospace`, propagate the `None` return from `__get_cc_sub_slot` and replace bare asserts with guarded checks:
```python
cc_sub_slot = __get_cc_sub_slot(segment.sub_slots, idx, ses)
if cc_sub_slot is None:
    return None
cc_sub_slot_hash = cc_sub_slot.get_hash()
...
if curr_slot.cc_slot_end_info is None:
    log.error("overflow block missing cc_slot_end_info")
    return None
if sub_slot_data.proof_of_space is None:
    log.error("challenge block missing proof_of_space")
    return None
```

def __get_rc_sub_slot(
    constants: ConsensusConstants,
    segment: SubEpochChallengeSegment,
    summaries: list[SubEpochSummary],
    curr_ssi: uint64,
) -> RewardChainSubSlot | None:
    ses = summaries[uint32(segment.sub_epoch_n - 1)]
    # find first challenge in sub epoch
    first_idx = None
    first = None
    for idx, curr in enumerate(segment.sub_slots):
        if curr.cc_slot_end is None:
            first_idx = idx
            first = curr
            break

    if first_idx is None or first is None or first.signage_point_index is None:
        log.error("segment missing challenge block or signage_point_index")
        return None
    idx = first_idx
    slots = segment.sub_slots

    # number of slots to look for
    slots_n = 1
    if is_overflow_block(constants, first.signage_point_index):
        if idx >= 2 and slots[idx - 2].cc_slot_end is None:
            slots_n = 2

    new_diff = None if ses is None else ses.new_difficulty
    new_ssi = None if ses is None else ses.new_sub_slot_iters
    ses_hash: bytes32 | None = None if ses is None else ses.get_hash()
    overflow = is_overflow_block(constants, first.signage_point_index)
    if overflow:
        if idx >= 2 and slots[idx - 2].cc_slot_end is not None and slots[idx - 1].cc_slot_end is not None:
            ses_hash = None
            new_ssi = None
            new_diff = None

    sub_slot = slots[idx]
    while True:
        if sub_slot.cc_slot_end:
            slots_n -= 1
            if slots_n == 0:
                break
        idx -= 1
        if idx < 0:
            log.error("malformed segment: no slot-end entry before challenge block")
            return None
        sub_slot = slots[idx]

    @pytest.mark.anyio
    async def test_weight_proof_validation_challenge_at_segment_start(
        self, default_1000_blocks: list[FullBlock], blockchain_constants: ConsensusConstants
    ) -> None:
        """SEC-614: validation must not crash when a segment's challenge block
        is the first sub-slot entry (first_idx == 0).

        In legitimately-constructed proofs, segment creation always places at
        least one slot-end entry before the challenge block (first_idx >= 1).
        A malicious peer could send a crafted proof where first_idx == 0; the
        old ``assert first_idx`` rejected this with an AssertionError instead
        of cleanly failing validation.
        """
        blocks = default_1000_blocks
        header_cache, height_to_hash, sub_blocks, summaries = await load_blocks_dont_validate(
            blocks, blockchain_constants
        )
        wpf = WeightProofHandler(
            blockchain_constants, BlockchainMock(sub_blocks, header_cache, height_to_hash, summaries)
        )
        wp = await wpf.get_proof_of_weight(blocks[-1].header_hash)
        assert wp is not None

        # Find the first segment of sub_epoch > 0 and strip the leading
        # slot-end entries so the challenge block lands at index 0.
        modified_segments = list(wp.sub_epoch_segments)
        target_found = False
        for i, seg in enumerate(modified_segments):
            if seg.sub_epoch_n > 0:
                challenge_idx = next(
                    (j for j, ssd in enumerate(seg.sub_slots) if ssd.cc_slot_end is None),
                    None,
                )
                if challenge_idx is not None and challenge_idx > 0:
                    new_sub_slots = list(seg.sub_slots[challenge_idx:])
                    assert new_sub_slots[0].cc_slot_end is None
                    modified_segments[i] = seg.replace(sub_slots=new_sub_slots)
                    target_found = True
                    break

        assert target_found, "No segment found with leading slot-end entries to strip"

        modified_wp = dataclasses.replace(wp, sub_epoch_segments=modified_segments)

        # Pre-fix: AssertionError in __get_rc_sub_slot crashes validation.
        # Post-fix: the malformed segment causes a hash mismatch, returning
        # (False, 0) cleanly.
        wpf_verify = WeightProofHandler(
            blockchain_constants, BlockchainMock(sub_blocks, header_cache, height_to_hash, {})
        )
        valid, _fork_point = wpf_verify.validate_weight_proof_single_proc(modified_wp)
        assert not valid
