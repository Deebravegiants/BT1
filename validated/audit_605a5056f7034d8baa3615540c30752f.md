Audit Report

## Title
Permanent `stakedButUnverifiedNativeETH` Inflation via Pre-Verification Validator Exit Causes Double-Counted ETH in `getEffectivePodShares()` — (`contracts/NodeDelegator.sol`)

## Summary

`stakedButUnverifiedNativeETH` is incremented in `stake32Eth` and decremented exclusively in `verifyWithdrawalCredentials`. EigenLayer's `IEigenPod.verifyWithdrawalCredentials` enforces that validators **must not have an exit epoch set**, reverting with `ValidatorIsExitingBeaconChain`. If a validator exits the beacon chain (e.g., via slashing) before credentials are verified, the 32 ETH returned to the EigenPod is captured by a subsequent checkpoint and awarded as `withdrawableShare`, while `stakedButUnverifiedNativeETH` is permanently stuck at 32 ETH. `getEffectivePodShares()` then returns `stakedButUnverifiedNativeETH + withdrawableShare = 64 ETH` for a single 32 ETH validator, permanently inflating the rsETH price and causing protocol insolvency.

## Finding Description

**Root cause — `stake32Eth` increments the tracker unconditionally:**

`stakedButUnverifiedNativeETH` is incremented by 32 ETH on every validator stake. [1](#0-0) 

**Only decrement path is permanently blocked for exited validators:**

`verifyWithdrawalCredentials` is the sole function that decrements `stakedButUnverifiedNativeETH`. [2](#0-1) 

However, EigenLayer's interface explicitly states and enforces that validators proven via `verifyWithdrawalCredentials` **must not have an exit epoch set already**, reverting with `ValidatorIsExitingBeaconChain`. [3](#0-2) [4](#0-3) 

Once a validator exits the beacon chain before credential verification, calling `NodeDelegator.verifyWithdrawalCredentials` will revert at the EigenPod level. There is no other code path in `NodeDelegator.sol` that decrements `stakedButUnverifiedNativeETH`.

**Checkpoint awards shares for the returned ETH:**

When the validator exits, 32 ETH is returned to the EigenPod address. Because the validator was never verified, it remains `INACTIVE` in EigenPod — `activeValidatorCount = 0`. When `startCheckpoint` is called (a routine operator action), `podBalanceGwei` captures the 32 ETH sitting in the pod, `proofsRemaining = 0` (no ACTIVE validators to prove), so the checkpoint finalizes immediately and awards 32 ETH in `withdrawableShare`. [5](#0-4) 

**`getEffectivePodShares()` double-counts:**

```
getEffectivePodShares() = stakedButUnverifiedNativeETH + withdrawableShare
                        = 32 ETH + 32 ETH = 64 ETH   ← only 32 ETH exists
``` [6](#0-5) 

**Inflated value propagates to rsETH price:**

`getETHDistributionData` calls `getEffectivePodShares()` for each NDC and feeds into `getTotalAssetDeposits`. [7](#0-6) 

`_getTotalEthInProtocol` in `LRTOracle` aggregates this to compute `rsETHPrice` via the public `updateRSETHPrice()` function callable by anyone. [8](#0-7) [9](#0-8) 

## Impact Explanation

The rsETH price is permanently inflated by 32 ETH per affected validator. The protocol reports more ETH backing than actually exists. When rsETH holders redeem at the inflated price, the protocol cannot cover redemptions — this is **protocol insolvency** (Critical). The inflation is irreversible: there is no code path that can decrement `stakedButUnverifiedNativeETH` for an exited-but-unverified validator.

## Likelihood Explanation

The trigger condition — a validator exiting before `verifyWithdrawalCredentials` is called — is realistic and not operator-controlled:

- **Slashing**: A validator slashed on the beacon chain is forced to exit. Slashing is an external event that can occur in the window between `stake32Eth` and the operator calling `verifyWithdrawalCredentials`. This window is operationally unavoidable.
- **Voluntary early exit**: The operator may voluntarily exit a validator before completing the EigenLayer credential verification flow.
- **Beacon chain forced exits**: EIP-7002 and related mechanisms can force exits.

The subsequent `startCheckpoint` call is a routine operational action that operators are expected to perform regularly. No malicious intent is required from the operator — the state corruption arises from an external beacon chain event combined with normal protocol operations. Once corrupted, any user can call the public `updateRSETHPrice()` to lock in the inflated price.

## Recommendation

Add a mechanism to decrement `stakedButUnverifiedNativeETH` for validators that exited without credential verification. Introduce a `recoverUnverifiedStake(bytes calldata pubkey)` function that, given proof that the validator is `INACTIVE` in EigenPod (never verified) and the pod has received the corresponding ETH via checkpoint shares, decrements `stakedButUnverifiedNativeETH` by 32 ETH. This prevents double-counting without requiring the blocked `verifyWithdrawalCredentials` path.

Alternatively, in `getEffectivePodShares()`, subtract any portion of `withdrawableShare` that corresponds to ETH received from unverified validator exits, so that the two components are mutually exclusive.

## Proof of Concept

```solidity
// Fork test outline (Holesky or local fork with EigenLayer deployed)
// 1. Deploy NodeDelegator, create EigenPod
// 2. Call stake32Eth(pubkey, sig, root)
//    → stakedButUnverifiedNativeETH = 32 ETH
// 3. Simulate validator exit on beacon chain WITHOUT calling verifyWithdrawalCredentials
//    → 32 ETH arrives at EigenPod address (NonBeaconChainETHReceived emitted)
// 4. Call NodeDelegator.startCheckpoint(false)
//    → eigenPod.activeValidatorCount() == 0 (validator is INACTIVE, never verified)
//    → checkpoint.proofsRemaining == 0, finalizes immediately
//    → withdrawableShare = 32 ETH awarded for pod balance
// 5. Assert:
//    eigenPod.validatorStatus(pubkey) == INACTIVE  // never verified
//    stakedButUnverifiedNativeETH == 32 ETH         // never decremented
//    getWithdrawableShare(beaconChainETHStrategy) == 32 ETH
//    getEffectivePodShares() == 64 ETH              // BUG: should be 32 ETH
// 6. Attempt verifyWithdrawalCredentials → reverts ValidatorIsExitingBeaconChain
//    → confirms stakedButUnverifiedNativeETH can never be decremented
// 7. Call updateRSETHPrice() (public, no access control)
//    → rsETHPrice inflated permanently
```
