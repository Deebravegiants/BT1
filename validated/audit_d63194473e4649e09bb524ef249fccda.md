## Forgery Feasibility Under a Torsion Key

For the order-2 torsion point `A` (order 2, so `[2]A = ∞`), Ed25519 verification checks `[S]B = R + [k]A` where `k = H(R ∥ A ∥ msg)`. Since `[k]A ∈ {A, ∞}` depending on parity of `k`, an attacker with a chosen `S` can:

1. Compute `R₀ = [S]B`
2. Compute `k₀ = H(R₀ ∥ A ∥ msg)` — if even, output `(R₀, S)` ✓
3. Compute `k₁ = H(R₀ − A ∥ A ∥ msg)` — if odd, output `(R₀ − A, S)` ✓

Probability of success per attempt: **≥ 3/4**. OpenSSL's `ED25519_verify` does not perform its own subgroup check on the public key, so it accepts this.

### Title
Missing subgroup check in `ecurve_ed_t::pub_from_der` allows torsion-key acceptance and Ed25519 signature forgery — (`src/cbmpc/crypto/base_eddsa.cpp`)

### Summary
`ecurve_ed_t::pub_from_der` parses the 32-byte point payload via `from_bin`, which only validates the curve equation. It never calls `curve.check()` (which enforces subgroup membership). An attacker supplying a DER blob with the valid ed25519 X.509 prefix followed by the encoding of any torsion point receives `SUCCESS` and a live `ecc_pub_key_t` holding a torsion point. Subsequent calls to `verify` on that key delegate to `ED25519_verify` without a subgroup guard, enabling signature forgery.

### Finding Description
`ecurve_ed_t::pub_from_der` (lines 191–195 of `src/cbmpc/crypto/base_eddsa.cpp`) validates only the DER prefix bytes, then calls `from_bin`. `from_bin` calls `ec25519_core::from_bin`, which solves the curve equation and returns `SUCCESS` for any point on the curve — including the 8 torsion points of Ed25519 (cofactor 8). No call to `is_in_subgroup()` or `curve.check()` follows. The designated guard, `ecurve_t::check()`, is present and correct (it calls `is_in_subgroup()`), but is simply absent from the DER parsing path. The converter path (`ecc_point_t::convert_fixed_curve`) correctly calls `curve.check()` after `from_bin`, confirming the pattern is known. The test `RejectTorsionAndFixInfinityEq` explicitly documents that `from_bin` accepts the order-2 torsion point with `SUCCESS` while `curve.check()` rejects it.

### Impact Explanation
A caller who parses an attacker-supplied ed25519 public key via `pub_from_der` (or the `ecc_pub_key_t::from_der` wrapper that dispatches to it) and then calls `verify` will pass the torsion point's encoding to `ED25519_verify`. Because `[k]A ∈ {A, ∞}` for a torsion key `A`, the attacker can forge a valid `(R, S)` pair for any message with ≥ 3/4 probability per attempt, requiring on average ~2 trials. This constitutes a full signature forgery under an attacker-chosen key that the library accepted as valid.

### Likelihood Explanation
The DER blob is fully attacker-controlled (32 bytes of point data after a fixed prefix). The torsion point encoding is public knowledge (the test file even hardcodes it). No secret material is required. Any code path that calls `pub_from_der` on untrusted input and then calls `verify` without an intervening `curve.check()` is directly exploitable.

### Recommendation
Add a subgroup check at the end of `ecurve_ed_t::pub_from_der`:

```cpp
error_t ecurve_ed_t::pub_from_der(ecc_pub_key_t& P, mem_t der) const {
  if (der.size != ed25519::x509_prefix.size + ed25519::pub_compressed_bin_size())
    return coinbase::error(E_FORMAT);
  if (0 != memcmp(ed25519::x509_prefix.data, der.data, ed25519::x509_prefix.size))
    return coinbase::error(E_FORMAT);
  error_t rv = from_bin(P, der.skip(ed25519::x509_prefix.size));
  if (rv) return rv;
  // Reject torsion points and infinity
  if (!P.is_in_subgroup() || P.is_infinity())
    return coinbase::error(E_CRYPTO);
  return SUCCESS;
}
```

Equivalently, delegate to `ecurve_t(this).check(P)` after `from_bin`, consistent with the converter path.

static error_t from_bin(point_t& R, mem_t bin) {
  if (bin.size != 32) return coinbase::error(E_FORMAT);

  buf_t buf = bin.rev();
  uint8_t neg = buf[0] >> 7;
  buf[0] &= 0x7f;
  fe_t y = fe_t::to_fe(uint256_t::from_bin(buf));

  // x² = (y² - 1) / (dy² + 1)

  fe_t u, v, w, vxx, check;

  u = y * y;
  v = u * formula_t::get_d();
  u -= fe_t::one();       // u = y^2-1
  v += fe_t::one();       // v = dy^2+1
  w = u * v;              // w = u*v
  fe_t x = w.pow22523();  // x = w^((q-5)/8)
  x *= u;                 // x = u * w^((q-5)/8)

  vxx = x * x;
  vxx *= v;
  check = vxx - u;  // vx^2-u
  if (!check.is_zero()) {
    check = vxx + u;  // vx^2+u
    if (!check.is_zero()) {
      return coinbase::error(E_CRYPTO);
    }
    static const fe_t sqrtm1 =
        fe_t::from_bn(bn_t::from_hex("2b8324804fc1df0b2b4d00993dfbd7a72f431806ad2fe478c4ee1b274a0ea0b0"));
    x *= sqrtm1;
  }

  uint256_t x_val = x.from_fe();
  if (neg != (x_val.w0 & 1)) x = -x;

  R.x = x;
  R.y = y;
  R.z = fe_t::one();
  return 0;
}

TEST(CryptoEdDSA, RejectTorsionAndFixInfinityEq) {
  crypto::vartime_scope_t vartime_scope;
  ecurve_t curve = crypto::curve_ed25519;

  // Compressed encoding of the Ed25519 order-2 point (x=0, y=-1):
  // y = p-1 = 2^255-20, sign bit = 0.
  uint8_t order2[32];
  order2[0] = 0xec;
  for (int i = 1; i < 31; i++) order2[i] = 0xff;
  order2[31] = 0x7f;

  ecc_point_t P(curve);
  EXPECT_EQ(P.from_bin(curve, mem_t(order2, 32)), SUCCESS);
  EXPECT_TRUE(P.is_on_curve());
  EXPECT_FALSE(P.is_infinity());
  EXPECT_FALSE(P.is_in_subgroup());
  EXPECT_NE(curve.check(P), SUCCESS);

  // Sanity: infinity should not compare equal to the generator.
  const ecc_point_t G = curve.generator();
  const ecc_point_t I = curve.infinity();
  EXPECT_FALSE(G == I);
  EXPECT_TRUE(I.is_infinity());
}
