### Title
Unconstrained `fee_proposal_fri` During Startup Window Allows Malicious Proposer to Inflate L2 Gas Price — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` enforces a multiplicative margin check on `fee_proposal_fri` only when `fee_actual` is `Some`. When `fee_actual` is `None` — which is the case for every block in the first `fee_proposal_window_size` heights — the bounds check is entirely skipped. A malicious proposer during this startup window can set `fee_proposal_fri` to an arbitrarily large value. Validators accept it, the value is committed into the `ProposalCommitment`, recorded in `fee_proposals_window`, and eventually becomes the median `fee_actual` that drives all future L2 gas prices.

---

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` validates `fee_proposal_fri` in two steps:

**Step 1 — presence check** (always enforced for V0_14_3+): [1](#0-0) 

**Step 2 — bounds check** (only enforced when `fee_actual` is `Some`): [2](#0-1) 

The comment explicitly documents the bypass: *"During initiation (fee_actual is None, <window_size blocks), bounds are not enforced."*

`fee_actual` is computed as the median of `fee_proposals_window` over the last `fee_proposal_window_size` heights: [3](#0-2) 

It returns `None` whenever the window is incomplete — i.e., for every block at height `< fee_proposal_window_size`. The validator constructs `ProposalInitValidation` with this `None` value: [4](#0-3) 

When `fee_actual` is `None`, the `if let (Some(fee_actual), Some(fee_proposal))` guard at line 398 short-circuits, and any `fee_proposal_fri` value — including `u128::MAX` — passes validation.

The accepted `fee_proposal_fri` is then used to compute the `ProposalCommitment` that consensus votes on: [5](#0-4) 

`proposal_commitment_from` hashes `fee_proposal_fri` into the commitment: [6](#0-5) 

After the block is decided, the `fee_proposal_fri` from the `ProposalInit` is recorded in `fee_proposals_window`. Once `window_size` blocks have been decided, `compute_fee_actual` returns the median of those recorded values. That median becomes `fee_actual`, which is passed as a floor to `calculate_next_l2_gas_price_for_fin`: [7](#0-6) 

The resulting `l2_gas_price` is placed directly into the next `ProposalInit.l2_gas_price_fri`: [8](#0-7) 

And validated by peers against their own locally-computed `l2_gas_price`: [9](#0-8) 

**Attack chain:**

1. Malicious proposer is elected for ≥ ⌈`window_size`/2⌉ + 1 blocks during the startup window.
2. Each malicious proposal carries `fee_proposal_fri = u128::MAX` (or any large value) with a valid `l2_gas_price_fri` (normal value).
3. Validators skip the bounds check (fee_actual is None), compute `ProposalCommitment = Poseidon(partial_block_hash, u128::MAX)`, and vote for it.
4. The inflated values are recorded in `fee_proposals_window`.
5. After `window_size` blocks, `fee_actual = median(window) = u128::MAX`.
6. `calculate_next_l2_gas_price_for_fin` uses `fee_actual` as a floor → `l2_gas_price = u128::MAX`.
7. All future `ProposalInit` carry `l2_gas_price_fri = u128::MAX`; validators accept them because their own `l2_gas_price` is now `u128::MAX`.
8. The gateway's `StatelessTransactionValidator` rejects any transaction whose `l2_gas.max_price_per_unit < u128::MAX`: [10](#0-9) 

All user transactions become unsubmittable.

---

### Impact Explanation

**Incorrect fee/gas price effect with economic impact (Critical).** A malicious proposer controlling a majority of the startup-window rounds can permanently inflate the network-wide L2 gas price to `u128::MAX`. Once the poisoned values propagate through `fee_proposals_window` into `fee_actual`, every subsequent honest proposer computes the same inflated `l2_gas_price` and embeds it in their `ProposalInit`. The gateway then rejects all user transactions whose `max_price_per_unit` is below `u128::MAX`, effectively halting the network. The corrupted value is committed into the `ProposalCommitment` that consensus signs over, making it part of the canonical chain state.

---

### Likelihood Explanation

The startup window spans the first `fee_proposal_window_size` blocks (10 blocks in current constants). In Tendermint-style consensus the proposer rotates deterministically across the committee. A validator controlling ≥ ⌈`window_size`/2⌉ + 1 proposer slots during genesis — achievable by controlling a proportional stake share or via Sybil — can execute the attack. The window is narrow but occurs at the most sensitive moment: network launch or restart after a hard fork that activates V0_14_3. No existing guard caps `fee_proposal_fri` from above during this period.

---

### Recommendation

1. **Add an absolute upper bound on `fee_proposal_fri`** independent of `fee_actual`. A reasonable cap is a small multiple of the configured `min_gas_price` or a protocol-level constant (e.g., `MAX_FEE_PROPOSAL_FRI`). Reject any proposal whose `fee_proposal_fri` exceeds this cap even when `fee_actual` is `None`.

2. **Alternatively, seed `fee_proposals_window` at genesis** with `window_size` copies of the configured `min_gas_price` (or a genesis parameter). This makes `fee_actual` available from block 0, so the normal margin check is enforced from the first block.

3. **Validate `fee_proposal_fri` against `l2_gas_price`** as a fallback reference when `fee_actual` is `None`. The proposer's own fallback is already `l2_gas_price`; the validator can enforce the same margin around `l2_gas_price` during the startup window.

### Proof of Concept

```rust
// Demonstrates that a validator accepts fee_proposal_fri = u128::MAX
// during the startup window (fee_actual = None).
//
// Run: cargo test --package apollo_consensus_orchestrator \
//      -- validate_proposal::test::startup_window_accepts_arbitrary_fee_proposal --nocapture

#[tokio::test]
async fn startup_window_accepts_arbitrary_fee_proposal() {
    use crate::validate_proposal::{is_proposal_init_valid, ProposalInitValidation};
    use crate::utils::make_gas_price_params;
    use crate::test_utils::{create_test_and_network_deps, proposal_init, HEIGHT_0, ROUND_0};
    use starknet_api::block::GasPrice;

    let (deps, _network) = create_test_and_network_deps();

    // fee_actual = None: startup window, window not yet full.
    let validation = ProposalInitValidation {
        height: HEIGHT_0,
        block_timestamp_window_seconds: 60,
        previous_proposal_init: None,
        l1_da_mode: starknet_api::data_availability::L1DataAvailabilityMode::Blob,
        l2_gas_price_fri: starknet_api::block::GasPrice(8_000_000_000),
        starknet_version: starknet_api::block::StarknetVersion::LATEST,
        fee_actual: None,  // <-- startup window: bounds check is SKIPPED
    };

    let mut init = proposal_init(HEIGHT_0, ROUND_0);
    // Malicious proposer sets fee_proposal_fri to u128::MAX.
    init.fee_proposal_fri = Some(GasPrice(u128::MAX));

    let gas_price_params = make_gas_price_params(&Default::default());
    let result = is_proposal_init_valid(
        &validation,
        &init,
        deps.clock.as_ref(),
        std::sync::Arc::new(deps.l1_gas_price_provider),
        &gas_price_params,
    )
    .await;

    // PASSES: no bounds check when fee_actual is None.
    assert!(result.is_ok(), "Expected Ok, got: {result:?}");

    // The accepted fee_proposal_fri = u128::MAX is now committed into
    // ProposalCommitment = Poseidon(partial_block_hash, u128::MAX)
    // and will be recorded in fee_proposals_window, poisoning fee_actual
    // after window_size blocks.
}
```

The accepted `fee_proposal_fri = u128::MAX` flows through `proposal_commitment_from` into the `ProposalCommitment` that consensus votes on, then into `fee_proposals_window` via `record_fee_proposal`, and finally into `fee_actual` → `calculate_next_l2_gas_price_for_fin` → `l2_gas_price` for all future blocks.

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
