### Title
Consensus Votes Accepted Without Signature Verification, Allowing Any Peer to Forge Quorum — (`crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`SingleHeightConsensus::handle_vote` accepts prevotes and precommits from the P2P network without verifying the cryptographic signature in `vote.signature`. The `voter` field is attacker-controlled and only checked for committee membership. Any peer connected to the gossipsub network can forge votes attributed to any committee member, manufacture a false quorum, and drive the consensus state machine to a `DecisionReached` for an arbitrary `ProposalCommitment` that was never built or validated.

### Finding Description

In `handle_vote`, the only admission checks are height equality and committee membership:

```rust
// TODO(Asmaa): verify the signature          ← explicit unimplemented guard
if !self.committee.members().iter().any(|s| s.address == vote.voter) {
    debug!("Ignoring vote from non validator: vote={:?}", vote);
    return VecDeque::new();
}
``` [1](#0-0) 

The `vote.voter` field is a `ContractAddress` deserialized verbatim from the protobuf wire message. The `vote.signature` field is present in the struct but is never passed to any verification function. The `SignatureManager` already exposes `verify_precommit_vote_signature` for exactly this purpose, but it is never called on the inbound path. [2](#0-1) 

On the outbound side, self-votes are created with `signature: RawSignature::default()` (an all-zero byte array) and a matching TODO:

```rust
// TODO(Asmaa): sign the vote
signature: RawSignature::default(),
``` [3](#0-2) 

The `MultiHeightManager` code that receives votes from the network explicitly acknowledges the missing guard:

> "Since vote signatures are not yet verified, a peer can forge votes with arbitrary voter addresses; without this cap that would grow `future_votes` without bound and exhaust memory." [4](#0-3) 

The `Vote` struct's `voter` field is a `ContractAddress` deserialized from the protobuf `voter` field with no binding to the sending peer's identity: [5](#0-4) 

The duplicate/conflict guard uses `(round, voter)` as the key, so one forged vote per `(round, voter)` pair passes through. With a 4-validator committee requiring a Byzantine quorum of 3, an attacker needs only 3 distinct committee addresses.

Vote weight is looked up from the committee by `voter` address: [6](#0-5) 

So forging a vote from a high-weight validator yields that validator's full weight toward quorum.

### Impact Explanation

An attacker who can send gossipsub messages to any consensus node can:

1. Obtain committee member `ContractAddress` values from the public staking contract.
2. Craft `Vote` protobuf messages with `voter = <committee_member_i>`, `height = H`, `round = R`, `proposal_commitment = <arbitrary_felt>`, `signature = []`.
3. Send 3 such prevotes and 3 such precommits (for a 4-member committee with Byzantine quorum).
4. The state machine reaches `DecisionReached` for the attacker-chosen `ProposalCommitment`.

The decided `ProposalCommitment` is then passed to `decision_reached` in the context, which triggers block finalization with a commitment that was never produced by the batcher or validated by any real validator. This corrupts the canonical block hash, state root, and all downstream commitments.

**Impact class**: Critical — wrong state/block commitment accepted through forged consensus vote signatures.

### Likelihood Explanation

The attack requires only:
- Network connectivity to any consensus node (gossipsub is a broadcast protocol; any connected peer can publish to the vote topic).
- Knowledge of committee member addresses (publicly readable from the L2 staking contract).
- Ability to serialize a valid protobuf `Vote` message.

No privileged access, no key material, and no special timing is required. The codebase itself documents the missing check with two explicit `TODO` comments.

### Recommendation

1. **Sign outbound votes**: In `StateMachine::make_self_vote`, call `SignatureManager::sign_precommit_vote` (or an equivalent that covers both vote types) and store the result in `vote.signature` before broadcasting.
2. **Verify inbound votes**: In `SingleHeightConsensus::handle_vote`, after the committee membership check, look up the committee member's registered public key and call `verify_precommit_vote_signature(block_hash, vote.signature, public_key)`. Reject votes that fail verification.
3. **Reject zero/default signatures**: Add an explicit guard that rejects `RawSignature::default()` (all-zero) before attempting cryptographic verification, mirroring the `proof_block_hash == Felt::ZERO` guard already present in the blockifier. [7](#0-6) 

### Proof of Concept

```
// Attacker connects to the gossipsub vote topic.
// Committee for height H = [V0, V1, V2, V3], Byzantine quorum = 3.
// Target commitment = EVIL_HASH (never built by any batcher).

for voter in [V0, V1, V2]:
    send Vote {
        vote_type: Prevote,
        height: H,
        round: 0,
        proposal_commitment: Some(EVIL_HASH),
        voter: voter,          // real committee address, attacker-chosen
        signature: [],         // RawSignature::default(), never checked
    }

// handle_vote passes: height matches, voter in committee, not duplicate.
// StateMachine records 3 prevotes for EVIL_HASH → prevote quorum reached.

for voter in [V0, V1, V2]:
    send Vote {
        vote_type: Precommit,
        height: H,
        round: 0,
        proposal_commitment: Some(EVIL_HASH),
        voter: voter,
        signature: [],
    }

// StateMachine records 3 precommits for EVIL_HASH → precommit quorum reached.
// DecisionReached(EVIL_HASH) emitted → block finalized with attacker-chosen commitment.
```
