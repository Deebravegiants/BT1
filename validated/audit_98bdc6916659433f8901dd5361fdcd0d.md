### Title
Queue-Flooding Blocks `verify_foreign_transaction` Requests — (File: `crates/contract/src/lib.rs`, `crates/contract/src/pending_requests.rs`)

### Summary

`verify_foreign_transaction` stores pending requests under a key (`VerifyForeignTransactionRequest`) that does **not** include the caller's account ID. Any unprivileged account can pre-fill the 128-slot fan-out queue for any specific foreign-chain transaction, causing every subsequent legitimate submission for that same transaction to revert with `PendingRequestQueueFull`. This is a direct analog of the `addCandidate` hash-collision blocking pattern: the attacker exploits a caller-agnostic key to occupy all available slots and freeze a victim's bridge-release flow.

---

### Finding Description

`verify_foreign_transaction` converts the user-supplied `VerifyForeignTransactionRequestArgs` into a `VerifyForeignTransactionRequest` via `args_into_verify_foreign_tx_request`:

```rust
// crates/contract/src/dto_mapping.rs:840-848
pub fn args_into_verify_foreign_tx_request(
    args: dtos::VerifyForeignTransactionRequestArgs,
) -> dtos::VerifyForeignTransactionRequest {
    dtos::VerifyForeignTransactionRequest {
        domain_id: args.domain_id,
        request: args.request,
        payload_version: args.payload_version,
    }
}
```

The resulting struct contains only `{domain_id, request, payload_version}` — the caller's `predecessor_account_id` is **never included**. [1](#0-0) [2](#0-1) 

This key is used as the map key in `pending_verify_foreign_tx_requests`:

```rust
// crates/contract/src/lib.rs:229-239
fn add_verify_foreign_tx_request(
    &mut self,
    request: VerifyForeignTransactionRequest,
    data_id: CryptoHash,
) {
    pending_requests::push_pending_yield(
        &mut self.pending_verify_foreign_tx_requests,
        request,
        data_id,
    );
}
``` [3](#0-2) 

`push_pending_yield` enforces a hard cap of `MAX_PENDING_REQUEST_FAN_OUT = 128` and panics if exceeded:

```rust
// crates/contract/src/pending_requests.rs:50-58
let queue = requests.entry(request).or_default();
if queue.len() >= usize::from(MAX_PENDING_REQUEST_FAN_OUT) {
    env::panic_str(
        &RequestError::PendingRequestQueueFull {
            limit: MAX_PENDING_REQUEST_FAN_OUT,
        }
        .to_string(),
    );
}
``` [4](#0-3) 

By contrast, `sign` and `request_app_private_key` both pass `&predecessor` into their respective `::new` constructors, making their keys caller-specific and immune to this attack:

```rust
// crates/contract/src/lib.rs:379-384
let request = SignatureRequest::new(
    request.domain_id,
    request.payload,
    &predecessor,   // ← caller-bound
    &request.path,
);
``` [5](#0-4) 

`verify_foreign_transaction` makes no equivalent binding:

```rust
// crates/contract/src/lib.rs:549-556
let request = args_into_verify_foreign_tx_request(request);
let callback_args = serde_json::to_vec(&(&request,)).unwrap();
self.enqueue_yield_request(
    method_names::RETURN_VERIFY_FOREIGN_TX_AND_CLEAN_STATE_ON_SUCCESS,
    callback_args,
    callback_gas,
    move |this, id| this.add_verify_foreign_tx_request(request, id),
);
``` [6](#0-5) 

---

### Impact Explanation

`verify_foreign_transaction` is the entry point for bridge users to release funds locked on a foreign chain (Ethereum, Solana, Bitcoin, etc.). If an attacker pre-fills the 128-slot queue for a specific foreign-chain transaction ID, every subsequent call by the legitimate bridge user reverts. The attacker can continuously refill the queue after it drains (when MPC nodes respond or yields time out), permanently preventing the victim from completing the bridge release. This maps to **permanent freezing of funds** controlled by the bridge flow, matching the Critical/Medium allowed impact scope for request-lifecycle manipulation that breaks production safety invariants.

---

### Likelihood Explanation

The cost to the attacker is 128 × 1 yoctoNEAR per queue-fill cycle — effectively zero. No special privilege, key material, or threshold collusion is required. The attacker only needs to know the target foreign-chain transaction ID (publicly visible on the foreign chain and in NEAR indexer events). The attack is repeatable indefinitely.

---

### Recommendation

Include the caller's `predecessor_account_id` in the `VerifyForeignTransactionRequest` key, mirroring the pattern used by `sign` and `request_app_private_key`. Concretely, `args_into_verify_foreign_tx_request` should accept the predecessor and embed it in the returned struct, and `VerifyForeignTransactionRequest` should gain a `requester: AccountId` field. This makes each caller's queue slot independent, eliminating cross-user queue saturation.

---

### Proof of Concept

1. Alice submits `verify_foreign_transaction` for Ethereum tx `0xABC…` targeting domain `D`.
2. Eve observes the pending request on-chain (or predicts it from the foreign chain).
3. Eve submits 128 identical `verify_foreign_transaction` calls for the same `{request: 0xABC…, domain_id: D, payload_version: V1}`, each with 1 yoctoNEAR deposit. Total cost: ~128 yoctoNEAR.
4. The queue for that key is now full (`queue.len() == 128`).
5. Alice's next `verify_foreign_transaction` call hits `push_pending_yield`, finds `queue.len() >= 128`, and panics with `PendingRequestQueueFull`. Alice's transaction reverts.
6. When MPC nodes eventually respond and drain the queue, Eve immediately resubmits 128 more requests.
7. Alice's bridge funds remain locked indefinitely. [7](#0-6) [8](#0-7)

        log!(
            "verify_foreign_transaction: predecessor={:?}, request={:?}",
            env::predecessor_account_id(),
            request
        );

        self.check_request_preconditions(
            request.domain_id,
            DomainPurpose::ForeignTx,

        );

        let requested_chain = request.request.chain();
        let supported_chains = self.get_supported_foreign_chains();
        if !supported_chains.contains(&requested_chain) {
            env::panic_str(
                &InvalidParameters::ForeignChainNotSupported {
                    requested: requested_chain,
                }
                .to_string(),

```rust
pub(crate) fn push_pending_yield<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: K,
    data_id: CryptoHash,
) where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let queue = requests.entry(request).or_default();
    if queue.len() >= usize::from(MAX_PENDING_REQUEST_FAN_OUT) {
        env::panic_str(
            &RequestError::PendingRequestQueueFull {
                limit: MAX_PENDING_REQUEST_FAN_OUT,
            }
            .to_string(),
        );
    }
    queue.push(YieldIndex { data_id });
}
