### Title
P2P peer can corrupt stored `ThinStateDiff` by sending `DeclaredClass` with `compiled_class_hash=None`, misclassifying a Cairo 1 class as deprecated — (`crates/apollo_protobuf/src/converters/state_diff.rs`)

### Summary
The protobuf-to-`StateDiffChunk` converter uses the presence/absence of `compiled_class_hash` to distinguish Cairo 1 (`DeclaredClass`) from Cairo 0 (`DeprecatedDeclaredClass`). An unauthenticated p2p peer can omit `compiled_class_hash` for any Cairo 1 class hash H, causing the syncing node to store H in `ThinStateDiff::deprecated_declared_classes` instead of `class_hash_to_compiled_class_hash`. Because `parse_data_for_block` never verifies the assembled diff against the `state_diff_commitment` already stored in the block header, the corrupted diff is written to storage unconditionally.

### Finding Description
**Entry point**: unauthenticated p2p peer responding to a `StateDiffsRequest`.

**Step 1 — converter misroutes the chunk**:
`TryFrom<protobuf::StateDiffsResponse> for DataOrFin<StateDiffChunk>` at lines 51-59 branches on `declared_class.compiled_class_hash.as_ref()`. When `None`, it calls `declared_class.try_into()` targeting `DeprecatedDeclaredClass`, which only extracts `class_hash` and discards the compiled hash field entirely. [1](#0-0) 

**Step 2 — `unite_state_diffs` inserts into the wrong field**:
The `DeprecatedDeclaredClass` arm pushes `class_hash` into `state_diff.deprecated_declared_classes` (a `Vec`), not into `class_hash_to_compiled_class_hash` (an `IndexMap`). [2](#0-1) 

**Step 3 — no commitment cross-check**:
After the loop, `parse_data_for_block` calls only `validate_deprecated_declared_classes_non_conflicting` (a deduplication check) and then returns `Ok(Some((result, block_number)))`. It never reads `header.state_diff_commitment` and never calls `calculate_state_diff_hash`. The corrupted `ThinStateDiff` is written to storage via `append_state_diff`. [5](#0-4) 

**Step 4 — length check does not catch the swap**:
Both `StateDiffChunk::DeclaredClass` and `StateDiffChunk::DeprecatedDeclaredClass` return `len() == 1`, so the `state_diff_length` guard passes even with the substitution. [6](#0-5) 

### Impact Explanation
The node stores a `ThinStateDiff` where class H appears in `deprecated_declared_classes` instead of `class_hash_to_compiled_class_hash`. Consequences:

1. **Wrong state root**: any downstream Patricia trie update driven by this diff will compute a different global root than the canonical one.
2. **Wrong class type resolution**: the node treats H as a Cairo 0 class. Execution requests that load H will attempt to fetch a `DeprecatedContractClass` instead of a Sierra/CASM artifact, producing wrong execution results or panics.
3. **Wrong data served to peers**: the node re-serves the corrupted diff to other syncing nodes, propagating the corruption.
4. **Commitment mismatch detectable only externally**: the `state_diff_commitment` in the stored header is correct (it came from the header sync), but `calculate_state_diff_hash(stored_diff) ≠ header.state_diff_commitment`, so any later integrity check will fail or produce a wrong block hash.

### Likelihood Explanation
Any unauthenticated p2p peer can trigger this. The attacker needs only to respond to a `StateDiffsRequest` with a well-formed `DeclaredClass` message where `compiled_class_hash` is omitted. No keys, signatures, or operator access are required. The node will accept and store the corrupted diff as long as the total `state_diff_length` count matches the header.

### Recommendation
In `parse_data_for_block`, after assembling the full `ThinStateDiff`, read `header.state_diff_commitment` from storage and assert:
```rust
let expected = header.state_diff_commitment
    .ok_or(P2pSyncClientError::MissingCommitment { block_number })?;
let actual = calculate_state_diff_hash(&result);
if actual != expected {
    return Err(ParseDataError::BadPeer(BadPeerError::StateDiffCommitmentMismatch { ... }));
}
```
This mirrors the existing guard in `apollo_committer` (`verify_state_diff_hash` flag) and closes the gap entirely. [7](#0-6) 

### Proof of Concept
```rust
#[test]
fn declared_class_with_no_compiled_hash_misroutes_to_deprecated() {
    use apollo_protobuf::protobuf;
    use apollo_protobuf::sync::{DataOrFin, StateDiffChunk};
    use starknet_api::block_hash::state_diff_hash::calculate_state_diff_hash;
    use starknet_api::core::{ClassHash, CompiledClassHash};
    use starknet_api::state::ThinStateDiff;
    use indexmap::indexmap;

    let h = starknet_api::felt!("0xdeadbeef");

    // Build a DeclaredClass protobuf message with compiled_class_hash absent.
    let proto = protobuf::StateDiffsResponse {
        state_diff_message: Some(
            protobuf::state_diffs_response::StateDiffMessage::DeclaredClass(
                protobuf::DeclaredClass {
                    class_hash: Some(h.into()),
                    compiled_class_hash: None,   // <-- attacker omits this
                },
            ),
        ),
    };

    // Converter silently routes to DeprecatedDeclaredClass.
    let chunk: DataOrFin<StateDiffChunk> = proto.try_into().unwrap();
    assert!(matches!(
        chunk.0.unwrap(),
        StateDiffChunk::DeprecatedDeclaredClass(_)
    ));

    // Assemble the "corrupted" ThinStateDiff (H in deprecated_declared_classes).
    let corrupted = ThinStateDiff {
        deprecated_declared_classes: vec![ClassHash(h)],
        ..Default::default()
    };

    // Assemble the "correct" ThinStateDiff (H in class_hash_to_compiled_class_hash).
    let correct = ThinStateDiff {
        class_hash_to_compiled_class_hash: indexmap! {
            ClassHash(h) => CompiledClassHash(starknet_api::felt!("0xcafe"))
        },
        ..Default::default()
    };

    // The two state diff commitments differ — the node stores the wrong one.
    assert_ne!(
        calculate_state_diff_hash(&corrupted),
        calculate_state_diff_hash(&correct)
    );
}
```

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);

    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
