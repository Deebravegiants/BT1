### Title
Missing TLS Public Key Uniqueness Validation Allows Ghost Participant Injection - (File: `crates/contract/src/primitives/participants.rs`)

### Summary
`Participants::validate()` checks for duplicate `AccountId` and `ParticipantId` values but never checks for duplicate `tls_public_key` values across participants. Because the TEE state is keyed exclusively by `tls_public_key`, two participants sharing the same TLS key produce a ghost participant: one that passes TEE re-verification (by borrowing the other's stored attestation) yet can never register its own attestation or participate in P2P communication.

### Finding Description
`Participants::validate()` enforces two uniqueness invariants — no duplicate `ParticipantId`, no duplicate `AccountId` — but omits a third invariant that is equally load-bearing for the TEE and P2P subsystems: no duplicate `tls_public_key`. [1](#0-0) 

The `tls_public_key` field inside `ParticipantInfo` is the sole key used by `TeeState::stored_attestations` (an `IterableMap<Ed25519PublicKey, NodeAttestation>`). Every TEE lookup, re-verification, and cleanup operation resolves a participant through this key: [2](#0-1) 

`validate_incoming_proposal` calls `proposal.validate()` and then checks that existing participants have not changed their `ParticipantInfo`, but it performs no cross-participant TLS-key uniqueness check for newly added participants: [3](#0-2) 

`vote_new_parameters` feeds directly into this path: [4](#0-3) 

`add_participant` in `TeeState` does enforce that a TLS key cannot be re-registered by a *different* account, but this guard fires only at attestation-submission time, not at proposal-validation time: [5](#0-4) 

### Impact Explanation
When a participant set containing two entries sharing TLS key K is accepted:

1. Participant B (the first to call `submit_participant_info`) registers key K successfully.
2. Participant C (same key K, different `AccountId`) is permanently blocked by `TlsKeyOwnedByOtherAccount` and can never store its own attestation.
3. `reverify_and_cleanup_participants` resolves C's TLS key K and finds B's attestation, so C is counted as `TeeQuoteStatus::Valid` — the contract believes C is a legitimate, attested TEE node.
4. C is a ghost participant: it holds a slot in the threshold count, appears TEE-valid, but cannot participate in P2P signing rounds. The effective signing capacity is lower than the contract's recorded threshold.
5. If B and C are controlled by the same adversary, B can impersonate C in the P2P mesh (identical TLS identity), manipulating message routing and potentially biasing protocol execution below the honest-majority assumption.

This breaks the participant-state accounting invariant: the contract's recorded threshold no longer reflects the true number of independently-operating, independently-attested nodes.

### Likelihood Explanation
`vote_new_parameters` is callable by any existing participant. A single malicious participant can craft a proposal containing two new entries with the same `tls_public_key`. Honest co-voters inspecting only account IDs and participant IDs would not detect the collision; the contract itself performs no check. Once threshold votes accumulate on the identical proposal bytes, resharing proceeds with the broken set. No privileged access or key leakage is required — only the ability to submit a governance vote, which every participant already holds.

### Recommendation
Add a `tls_public_key` uniqueness check inside `Participants::validate()`, mirroring the existing `AccountId` and `ParticipantId` checks:

```rust
let mut tls_keys: BTreeSet<Ed25519PublicKey> = BTreeSet::new();
for (_, _, info) in &self.participants {
    tls_keys.insert(info.tls_public_key.clone());
}
if tls_keys.len() != self.len() {
    return Err(InvalidCandidateSet::DuplicateTlsPublicKeys.into());
}
```

Additionally, `validate_incoming_proposal` should verify that no new participant's `tls_public_key` collides with any existing participant's key, since the existing `ParticipantInfoChanged` guard only protects already-present entries.

### Proof of Concept

1. Running contract has participants `[A, B]` with threshold 2 (both honest).
2. Malicious participant A calls `vote_new_parameters` proposing `[A, B, C, D]` where C and D are new entries both carrying `tls_public_key = K`.
3. `validate_incoming_proposal` → `proposal.validate()` passes: no duplicate `AccountId`, no duplicate `ParticipantId`.
4. B votes for the same proposal (identical bytes); threshold is reached; contract enters resharing.
5. After resharing completes, C calls `submit_participant_info` with key K → succeeds, stores attestation under K with `account_id = C`.
6. D calls `submit_participant_info` with key K → rejected: `TlsKeyOwnedByOtherAccount`.
7. `reverify_and_cleanup_participants` for the new set `[A, B, C, D]`: D's TLS key K resolves to C's stored attestation → D is marked `TeeQuoteStatus::Valid`.
8. D is a ghost participant: counted in the threshold, appears TEE-valid, but cannot sign or communicate independently. The network operates as if it has 4 participants but effectively has 3 independent nodes. [1](#0-0) [6](#0-5) [7](#0-6)

    pub fn validate(&self) -> Result<(), Error> {
        let mut ids: BTreeSet<ParticipantId> = BTreeSet::new();
        let mut accounts: BTreeSet<AccountId> = BTreeSet::new();
        for (acc_id, pid, _) in &self.participants {
            accounts.insert(acc_id.clone());
            ids.insert(*pid);
            if self.next_id.get() <= pid.get() {
                return Err(InvalidCandidateSet::ParticipantIdNotLessThanNextId {
                    id: pid.get(),
                    next_id: self.next_id.get(),
                }
                .into());
            }
        }
        if ids.len() != self.len() {
            return Err(InvalidCandidateSet::DuplicateParticipantIds.into());
        }
        if accounts.len() != self.len() {
            return Err(InvalidCandidateSet::DuplicateAccountIds.into());
        }
        Ok(())
    }

        let tls_pk = node_id.tls_public_key.clone();

        // Authorization: a TLS key registered to one account must not be
        // overwritten by a submission from a different account. Without this,
        // any caller could replace any participant's stored attestation, since
        // the entry is keyed only by `tls_public_key`.
        if let Some(existing) = self.stored_attestations.get(&tls_pk)

        {
            return Err(AttestationSubmissionError::TlsKeyOwnedByOtherAccount);
        }

        let insertion = self.stored_attestations.insert(
            tls_pk,
            NodeAttestation {
                node_id,
                verified_attestation,
            },
        );

        Ok(match insertion {
            Some(_previous_attestation) => ParticipantInsertion::UpdatedExistingParticipant,
            None => ParticipantInsertion::NewlyInsertedParticipant,
        })
    }

    pub fn reverify_and_cleanup_participants(
        &mut self,
        participants: &Participants,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeValidationResult {
        self.allowed_docker_image_hashes
            .cleanup_expired_hashes(tee_upgrade_deadline_duration);

        let participants_with_valid_attestation: Vec<_> = participants
            .participants()
            .iter()
            .filter(|(_, _, participant_info)| {
                // Use the stored NodeId (keyed by TLS public key) so the real
                // `account_public_key` participates in re-verification. If
                // there is no stored attestation for this TLS key, the
                // participant is invalid.
                let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
                else {
                    return false;
                };

                let tee_status =
                    self.reverify_participants(&node_id, tee_upgrade_deadline_duration);

                matches!(tee_status, TeeQuoteStatus::Valid)
            })
            .cloned()
            .collect();

        if participants_with_valid_attestation.len() != participants.len() {
            let participants_with_valid_attestation =
                Participants::init(participants.next_id(), participants_with_valid_attestation);

            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            }
        } else {
            TeeValidationResult::Full
        }
    }

    pub fn validate_incoming_proposal(&self, proposal: &ThresholdParameters) -> Result<(), Error> {
        // ensure the proposed threshold parameters are valid:
        // if performance issue, inline and merge with loop below
        proposal.validate()?;
        let mut old_by_id: BTreeMap<ParticipantId, AccountId> = BTreeMap::new();
        let mut old_by_acc: BTreeMap<AccountId, (ParticipantId, ParticipantInfo)> = BTreeMap::new();
        for (acc, id, info) in self.participants().participants() {
            old_by_id.insert(*id, acc.clone());
            old_by_acc.insert(acc.clone(), (*id, info.clone()));
        }
        let new_participants = proposal.participants().participants();
        let mut new_min_id = u32::MAX;
        let mut new_max_id = 0u32;
        let mut n_old = 0u64;
        for (new_account, new_id, new_info) in new_participants {
            match old_by_acc.get(new_account) {
                Some((old_id, old_info)) => {
                    if new_id != old_id {
                        return Err(InvalidCandidateSet::ParticipantIdChanged {
                            account_id: new_account.clone(),
                            old_id: old_id.get(),
                            new_id: new_id.get(),
                        }
                        .into());
                    }
                    if *new_info != *old_info {
                        return Err(InvalidCandidateSet::ParticipantInfoChanged {
                            account_id: new_account.clone(),
                        }
                        .into());
                    }
                    n_old += 1;
                }
                None => {
                    if let Some(existing_account) = old_by_id.get(new_id) {
                        return Err(InvalidCandidateSet::NewParticipantReusesOldId {
                            account_id: new_account.clone(),
                            new_id: new_id.get(),
                            existing_account_id: existing_account.clone(),
                        }
                        .into());
                    }
                    new_min_id = std::cmp::min(new_min_id, new_id.get());
                    new_max_id = std::cmp::max(new_max_id, new_id.get());
                }
            }
        }
        // assert there are enough old participants
        if n_old < self.threshold().value() {
            return Err(InvalidCandidateSet::InsufficientOldParticipants.into());
        }
        // ensure the new ids are contiguous and unique
        let n_new = proposal.participants().len() as u64 - n_old;
        if n_new > 0 {
            if n_new - 1 != (new_max_id - new_min_id) as u64 {
                return Err(InvalidCandidateSet::NewParticipantIdsNotContiguous.into());
            }
            if new_min_id != self.participants().next_id().get() {
                return Err(InvalidCandidateSet::NewParticipantIdsNotContiguous.into());
            }
            if new_max_id + 1 != proposal.participants().next_id().get() {
                return Err(InvalidCandidateSet::NewParticipantIdsTooHigh.into());
            }
        }
        Ok(())
    }

    pub fn vote_new_parameters(
        &mut self,
        prospective_epoch_id: EpochId,
        proposal: dtos::ProposedThresholdParameters,
    ) -> Result<(), Error> {
        Self::assert_caller_is_signer();
        let proposal: ProposedThresholdParameters = proposal.try_into_contract_type()?;
        log!(
            "vote_new_parameters: signer={}, proposal={:?}",
            env::signer_account_id(),
            proposal,
        );

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        let validation_result = self.tee_state.reverify_and_cleanup_participants(
            proposal.participants(),
            tee_upgrade_deadline_duration,
        );

        let proposed_participants = proposal.participants();
        match validation_result {
            TeeValidationResult::Full => {
                if let Some(new_state) = self
                    .protocol_state
                    .vote_new_parameters(prospective_epoch_id, &proposal)?
                {
                    self.protocol_state = new_state;
                }
                Ok(())
            }
            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            } => {
                let invalid_participants: Vec<_> = proposed_participants
                    .participants()
                    .iter()
                    .filter(|(account_id, _, _)| {
                        !participants_with_valid_attestation
                            .is_participant_given_account_id(account_id)
                    })
                    .collect();

                Err(InvalidParameters::InvalidTeeRemoteAttestation {
                    reason: format!(
                        "The following participants have invalid TEE status: {:?}",
                        invalid_participants
                    ),
                }
                .into())
            }
        }
    }
