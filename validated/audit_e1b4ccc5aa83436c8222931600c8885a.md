### Title
Off-by-One in `valid_entries()` Permanently Includes Last-Expired Image Hash in Allowed Set — (`crates/contract/src/tee/proposal.rs`)

### Summary

`AllowedDockerImageHashes::valid_entries()` uses `rposition` to find the rightmost (last) expired entry and then returns `allowed_tee_proposals[cutoff_index..]`, which **includes** the expired entry at `cutoff_index` itself. As a result, the most-recently-expired image hash is never evicted from the valid set as long as at least one newer entry exists. A node running that expired hash passes every TEE attestation check — `submit_participant_info` and `reverify_participants` — indefinitely.

### Finding Description

The vector `allowed_tee_proposals` is sorted oldest-first. `valid_entries()` is supposed to return only entries whose grace period has not yet elapsed:

```rust
// crates/contract/src/tee/proposal.rs lines 173-192
let cutoff_index = self
    .allowed_tee_proposals
    .iter()
    .rposition(|allowed_docker_image| {
        ...
        grace_period_deadline < current_time   // true = expired
    })
    .unwrap_or(0);

self.allowed_tee_proposals
    .get(cutoff_index..)   // BUG: includes the expired entry at cutoff_index
    .unwrap_or(&[])
    .to_vec()
``` [1](#0-0) 

**Concrete trace with two entries:**

| Index | Hash | Deadline | State at `T1+grace < now < T2+grace` |
|-------|------|----------|--------------------------------------|
| 0 | `hash_A` | `T1 + grace` | **expired** → predicate = `true` |
| 1 | `hash_B` | `T2 + grace` | valid → predicate = `false` |

`rposition` scans right-to-left, finds index 0 as the last `true`, returns `Some(0)`.
`cutoff_index = 0`.
`allowed_tee_proposals[0..]` = `[hash_A, hash_B]` — the expired `hash_A` is included.

The correct slice should be `[cutoff_index + 1..]` = `[hash_B]`.

**Why the existing test does not catch this:**

`test_clean_expired` sets `first_entry_expiry_time_nanoseconds = second_entry_time_nano_seconds + grace + 1`, which simultaneously expires **both** entries. With both expired, `rposition` returns `Some(1)` and `[1..]` = `[entry_1]` — the test passes but never exercises the partial-expiry case. [2](#0-1) 

**Cleanup path is also broken:**

`cleanup_expired_hashes` calls `valid_entries` and assigns the result back. Since `valid_entries` returns the expired entry, `cleanup_expired_hashes` never removes it from storage either. [3](#0-2) 

### Impact Explanation

Both attestation enforcement paths consume the buggy list:

1. **`add_participant`** (called from `submit_participant_info`) passes `get_allowed_mpc_docker_image_hashes(...)` — which calls `valid_entries()` — to `attestation.verify_locally(...)`. A node presenting `hash_A` passes `verify_mpc_hash` because `hash_A` is in the returned slice. [4](#0-3) 

2. **`reverify_participants`** (called from `verify_tee`) uses the same path. The expired node's stored attestation re-verifies successfully, so it is never evicted. [5](#0-4) 

A node running the expired image hash therefore remains an attested participant indefinitely and can call `respond`/`respond_ckd`, contributing to threshold signatures and CKD outputs. This is an attestation authorization bypass enabling participation by a node whose image the governance participants explicitly intended to de-authorize after the grace period.

### Likelihood Explanation

The scenario is the normal upgrade path: threshold participants vote in a new image hash (`hash_B`), the old one (`hash_A`) enters its grace period, nodes upgrade. After the grace period elapses, `hash_A` should be rejected. Any operator who did not upgrade (or who deliberately keeps running `hash_A`) continues to pass attestation checks. No special privileges are required beyond being a current or prospective participant.

### Recommendation

Change the slice start from `cutoff_index` to `cutoff_index + 1` to exclude the expired entry, while preserving the "keep at least one" fallback for the all-expired case:

```rust
let cutoff_index = self
    .allowed_tee_proposals
    .iter()
    .rposition(|e| grace_period_deadline(e) < current_time)
    .map(|i| i.saturating_add(1))   // exclude the expired entry itself
    .unwrap_or(0);
```

When all entries are expired, `rposition` returns `Some(last)`, `saturating_add(1)` gives `last + 1`, and `[last+1..]` is empty — add an explicit fallback to return the last entry in that case to preserve the "always return at least one" invariant documented on `get()`.

Add a unit test that advances time past only the first entry's deadline (not the second's) and asserts the expired hash is absent from the result.

    fn valid_entries(&self, tee_upgrade_deadline_duration: Duration) -> Vec<AllowedMpcDockerImage> {
        let current_time = Timestamp::now();
        // get the index of the most recently enforced docker image
        let cutoff_index = self
            .allowed_tee_proposals
            .iter()
            .rposition(|allowed_docker_image| {
                let Some(grace_period_deadline) = allowed_docker_image
                    .added
                    .checked_add(tee_upgrade_deadline_duration)
                else {
                    log!("Error: timestamp overflowed when calculating grace_period_deadline.");
                    return true;
                };
                // if the grace period for this docker hash is in the past, then older hashes are no longer accepted
                grace_period_deadline < current_time
            })
            .unwrap_or(0);

        self.allowed_tee_proposals
            .get(cutoff_index..)
            .unwrap_or(&[])
            .to_vec()
    }
