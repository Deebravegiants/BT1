Audit Report

## Title
Stale `rsETHPrice` Enables Yield Sniping via Permissionless `FeeReceiver.sendFunds()` - (File: `contracts/LRTOracle.sol`)

## Summary
`LRTOracle.rsETHPrice` is a lazily-updated state variable that is only written when `updateRSETHPrice()` or `updateRSETHPriceAsManager()` is explicitly called. Because `FeeReceiver.sendFunds()` is permissionless and immediately increases the deposit pool's ETH balance (and therefore the true rsETH price), any depositor who acts between a reward push and the next price update receives more rsETH than their deposit warrants, extracting yield that belongs to existing holders.

## Finding Description
`FeeReceiver.sendFunds()` carries no access-control modifier and forwards the contract's entire ETH balance to `LRTDepositPool` via `receiveFromRewardReceiver()`, which is itself unrestricted.

```solidity
// contracts/FeeReceiver.sol L53-57
function sendFunds() external {
    uint256 balance = address(this).balance;
    ILRTDepositPool(depositPool).receiveFromRewardReceiver{ value: balance }();
    emit MevRewardsAddedToTVL(balance);
}
```

`getETHDistributionData()` immediately reflects the new balance:

```solidity
// contracts/LRTDepositPool.sol L480
ethLyingInDepositPool = address(this).balance;
```

`getRsETHAmountToMint()` divides by the **stored** (now stale) `rsETHPrice`:

```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

`_beforeDeposit()` is declared `private view`, so it cannot mutate state to refresh the price before computing the mint amount. No call to `updateRSETHPrice()` is made anywhere in the deposit path.

**Exploit sequence (single block):**
1. Attacker calls `FeeReceiver.sendFunds()` — 10 ETH of MEV rewards land in `LRTDepositPool`; `rsETHPrice` remains at the pre-reward value (e.g., 1.0 ETH).
2. Attacker calls `depositETH{value: 10 ether}(0, "")` — receives `10 / 1.0 = 10 rsETH` instead of the correct `10 / 1.1 ≈ 9.09 rsETH`.
3. Anyone calls `updateRSETHPrice()` — new price ≈ 1.09 ETH.
4. Attacker's 10 rsETH is now worth ≈ 10.9 ETH; original 100-rsETH holders receive ≈ 109 ETH instead of 110 ETH.

The `pricePercentageLimit` guard in `_updateRsETHPrice()` only throttles the price-update call itself; it does not protect the deposit mint calculation and does not prevent the attack.

## Impact Explanation
Existing rsETH holders permanently lose a portion of their accumulated staking/MEV yield to the attacker. This is a concrete, repeatable **theft of unclaimed yield** — a High-severity impact under the allowed scope. The loss is proportional to the reward amount and the attacker's deposit size, and is irreversible once `updateRSETHPrice()` is called.

## Likelihood Explanation
Both `FeeReceiver.sendFunds()` and `depositETH()` are callable by any unprivileged external account with no preconditions beyond having ETH. The attack requires no privileged access, no victim mistake, and no external protocol compromise. It is repeatable every reward cycle and can be executed atomically in a single transaction or block. Likelihood is **High**.

## Recommendation
Atomically refresh `rsETHPrice` before computing the mint amount. The simplest fix is to remove the `view` modifier from `_beforeDeposit` and call `updateRSETHPrice()` at its start:

```solidity
function _beforeDeposit(address asset, uint256 depositAmount, uint256 minRSETHAmountExpected)
    private
    returns (uint256 rsethAmountToMint)
{
    ILRTOracle(lrtConfig.getContract(LRTConstants.LRT_ORACLE)).updateRSETHPrice();
    // ... existing checks ...
    rsethAmountToMint = getRsETHAmountToMint(asset, depositAmount);
    if (rsethAmountToMint < minRSETHAmountExpected) revert MinimumAmountToReceiveNotMet();
}
```

Alternatively, `getRsETHAmountToMint()` can compute the live price on-the-fly as `_getTotalEthInProtocol() / rsethSupply` rather than reading the stored `rsETHPrice`.

## Proof of Concept
Foundry fork/unit test plan:
1. Deploy protocol; mint 100 rsETH to Alice at price 1.0 ETH (TVL = 100 ETH).
2. Send 10 ETH to `FeeReceiver`; assert `rsETHPrice` is still 1.0 ETH.
3. Call `FeeReceiver.sendFunds()` from attacker EOA; assert `LRTDepositPool.balance == 110 ETH`.
4. Attacker calls `depositETH{value: 10 ether}(0, "")`.
5. Assert attacker received 10 rsETH (not ≈ 9.09 rsETH).
6. Call `updateRSETHPrice()`; assert new price ≈ 1.09 ETH.
7. Assert attacker's 10 rsETH × 1.09 ≈ 10.9 ETH (profit ≈ 0.9 ETH).
8. Assert Alice's 100 rsETH × 1.09 = 109 ETH (lost ≈ 1 ETH of yield).

**File:** contracts/LRTOracle.sol (L214-313)
```text
    function _updateRsETHPrice() internal {
        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;
        }

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

        // only take fee if TVL increased and protocol is not paused
        uint256 protocolFeeInETH = 0;
        if (!protocolPaused && totalETHInProtocol > previousTVL) {
            uint256 rewardAmount = totalETHInProtocol - previousTVL;
            protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
        }

        // compute new rsETH price based on total ETH minus fee
        uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

        if (newRsETHPrice > highestRsethPrice) {
            // check if the price is above the threshold
            uint256 priceDifference = newRsETHPrice - highestRsethPrice;
            // pricePercentageLimit is in 1e18 precision (100% = 1e18, 1% = 1e16)
            bool isPriceIncreaseOffLimit =
                pricePercentageLimit > 0 && priceDifference > pricePercentageLimit.mulWad(highestRsethPrice);

            // check if the price difference is above the threshold
            if (isPriceIncreaseOffLimit) {
                // if sender has a manager role, this doesn't revert.
                // if not, it reverts as price went above the threshold
                if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) {
                    revert PriceAboveDailyThreshold();
                }
            }
        }

        // downside protection — pause if price drops too far
        if (newRsETHPrice < highestRsethPrice) {
            uint256 diff = highestRsethPrice - newRsETHPrice;
            // normalizing to 1e18
            bool isPriceDecreaseOffLimit =
                pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);

            // if price decrease is off limit, pause the protocol (unless it's already paused)
            if (isPriceDecreaseOffLimit) {
                if (!lrtDepositPool.paused()) lrtDepositPool.pause();
                if (!withdrawalManager.paused()) withdrawalManager.pause();
                _pause();
                return;
            }

            // if price has decreased compared to the previous price, emit an event to reflect that
            if (previousPrice > newRsETHPrice) {
                emit RsETHPriceDecrease(newRsETHPrice, previousPrice);
            }

            // emit an event to notify that the price is currently below the peak (all time high) price
            emit RsETHPriceBelowPeak(highestRsethPrice, newRsETHPrice);
        }

        // update highest price if new price exceeds it
        if (newRsETHPrice > highestRsethPrice) {
            highestRsethPrice = newRsETHPrice;
        }

        // mint protocol fee as rsETH if there's a fee to take
        if (protocolFeeInETH > 0) {
            // Calculate rsETH amount to mint as protocol fee
            uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);

            _checkAndUpdateDailyFeeMintLimit(rsethAmountToMintAsProtocolFee);
            if (rsethAmountToMintAsProtocolFee > 0) {
                address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
                IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
                emit FeeMinted(treasury, rsethAmountToMintAsProtocolFee);
            }
        } else {
            _checkAndUpdateDailyFeeMintLimit(0);
        }

        rsETHPrice = newRsETHPrice;
```
