Audit Report

## Title
`ChainlinkPriceOracle::getAssetPrice` Accepts Stale Chainlink Prices, Enabling Inflated rsETH Minting - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle::getAssetPrice` calls `latestRoundData()` but discards `updatedAt` and `answeredInRound`, accepting any non-reverting answer regardless of age. Because this function is the terminal price source for rsETH minting, a stale inflated price allows an attacker to mint more rsETH than their deposited assets are worth, diluting existing holders and pushing the protocol toward insolvency.

## Finding Description
In `contracts/oracles/ChainlinkPriceOracle.sol` at line 52, the destructuring discards all staleness-relevant return values:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
``` [1](#0-0) 

Neither `updatedAt` (heartbeat check: `block.timestamp - updatedAt > heartbeat`) nor `answeredInRound` (round completeness check: `answeredInRound < roundId`) is validated. The raw `price` is returned directly.

This contrasts with `contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`, which captures and validates both `answeredInRound < roundID` and `timestamp == 0`: [2](#0-1) 

The stale price propagates through the full deposit call chain:

- `LRTDepositPool::depositAsset` → `_beforeDeposit` → `getRsETHAmountToMint`
- `getRsETHAmountToMint` at line 520: `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()` [3](#0-2) 

- `LRTOracle::getAssetPrice` delegates directly to `ChainlinkPriceOracle::getAssetPrice`.

No existing guard in `_beforeDeposit` checks oracle freshness; the only check is `minRSETHAmountExpected`, which an attacker sets to 0. [4](#0-3) 

## Impact Explanation
An inflated stale price causes `rsethAmountToMint` to be computed higher than the deposited assets' true value. The attacker receives excess rsETH backed by insufficient collateral, directly diluting all existing rsETH holders and reducing the protocol's collateralization ratio. Repeated exploitation drives the protocol toward insolvency. This matches the allowed impact: **Critical — Protocol insolvency / Direct theft of user funds**.

## Likelihood Explanation
Chainlink LST/ETH feeds (e.g., stETH/ETH, rETH/ETH) have heartbeat windows of 1–24 hours and update only on price deviation. During L1 congestion, sequencer downtime (on L2 deployments), or low-volatility periods, feeds can remain at a stale price while the actual market price diverges. No privileged access or oracle operator compromise is required — the attacker only needs to call `depositAsset` during the staleness window, which is a normal public function. The `minRSETHAmountExpected` slippage parameter provides no protection because the attacker benefits from receiving *more* rsETH, not less.

## Recommendation
Capture and validate `updatedAt` and `answeredInRound` in `getAssetPrice`, and store a per-asset heartbeat:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (price <= 0) revert InvalidPrice();
    if (answeredInRound < roundId) revert StalePrice();
    if (block.timestamp - updatedAt > assetHeartbeat[asset]) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Add a `mapping(address => uint256) public assetHeartbeat` set by `onlyLRTManager`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`.

## Proof of Concept
1. The Chainlink stETH/ETH feed last updated 25 hours ago (within its 24-hour heartbeat, no deviation triggered). Actual stETH price has dropped 5% due to a slashing event.
2. Attacker calls `LRTDepositPool::depositAsset(stETH, 100e18, 0)`.
3. `getRsETHAmountToMint` calls `ChainlinkPriceOracle::getAssetPrice(stETH)`, which returns the stale (pre-slash, inflated) price.
4. `rsethAmountToMint = (100e18 * stalePrice) / rsETHPrice` — attacker receives ~5% more rsETH than their stETH is currently worth.
5. Attacker redeems rsETH at the correct (lower) price, extracting value from honest depositors.
6. Repeat until the protocol's collateral backing per rsETH is materially degraded.

**Foundry fork test plan**: Fork mainnet at a block where a Chainlink LST feed has not updated for >heartbeat seconds. Mock `latestRoundData` to return a stale `updatedAt`. Call `depositAsset` and assert that `rsethAmountToMint` exceeds the fair value computed with the current market price, confirming the over-issuance.
