### Title
Pending Signature Requests Permanently Blocked When `accept_requests` Is Set to `false` After Yield Enqueue — (`File: crates/contract/src/lib.rs`)

---

### Summary

The `respond()`, `respond_ckd()`, and `respond_verify_foreign_tx()` functions all gate on `self.accept_requests` before resolving already-enqueued yield requests. When `verify_tee()` sets `accept_requests = false` after a user's `sign()` call has already placed a yield in `pending_signature_requests`, the MPC nodes can no longer call `respond()` to resolve that yield. The yield eventually times out, the user's deposit is forfeited, and the cross-chain transaction is permanently abandoned. No recovery path exists for the already-queued yields.

This is the direct analog of the StakedlvlUSD bug: a state change (`accept_requests = false`) that occurs during a pending period (the yield/resume window) causes in-flight requests to be irrecoverably stuck, and the only administrative function (`verify_tee`) that could restore the flag does not account for — or rescue — the already-queued yields.

---

### Finding Description

**Step 1 — User enqueues a yield.**

A user calls `sign()`. The contract passes `check_request_preconditions`, which verifies `accept_requests == true` at that moment, then calls `enqueue_yield_request`, storing the yield index in `pending_signature_requests`. [1](#0-0) 

**Step 2 — `verify_tee()` sets `accept_requests = false`.**

Any participant (a Byzantine node strictly below the signing threshold qualifies) calls `verify_tee()`. If the number of participants with a valid TEE attestation has fallen below the threshold — a natural event requiring no privileged access — the contract sets `accept_requests = false` and returns `Ok(false)` without entering resharing. [2](#0-1) 

**Step 3 — `respond()` is blocked for the already-queued yield.**

When MPC nodes attempt to call `respond()` to deliver the computed signature and resolve the pending yield, the function checks `accept_requests` and returns `Err(TeeValidationFailed)` before ever reaching `resolve_yields_for`. [3](#0-2) 

The same guard exists in `respond_ckd()`: [4](#0-3) 

And in `respond_verify_foreign_tx()`: [5](#0-4) 

**Step 4 — The yield times out; the request is lost.**

NEAR's yield/resume mechanism has a protocol-level timeout. Once the timeout elapses, the callback `return_signature_and_clean_state_on_success` is invoked with `Err(PromiseError::Failed)`, the pending request entry is removed from `pending_signature_requests`, and the user's deposit is consumed by the contract with no refund path. [6](#0-5) 

**The missing recovery path.**

The `redistributeLockedAmount` analog in StakedlvlUSD failed to look at the silo balance. Here, `verify_tee()` — the only function that can restore `accept_requests = true` — does not drain or rescue the already-queued yields before blocking `respond()`. There is no administrative endpoint to force-resolve or refund pending yields while `accept_requests` is false. [7](#0-6) 

---

### Impact Explanation

All pending `sign`, `request_app_private_key`, and `verify_foreign_transaction` yields that were enqueued before `accept_requests` was set to `false` are irrecoverably blocked. MPC nodes cannot call any of the three `respond*` functions. When the NEAR yield timeout elapses, every affected request is silently dropped and the user's deposit is forfeited. If `accept_requests` remains `false` for longer than the yield window (e.g., because the TEE attestation situation requires manual operator intervention), the loss is permanent for those requests. This breaks the request-lifecycle accounting invariant: a request that was validly accepted and paid for cannot be fulfilled or refunded.

**Allowed impact class matched**: *Medium — request-lifecycle and contract execution-flow manipulation that breaks production safety/accounting invariants.*

---

### Likelihood Explanation

- TEE attestation expiry is a natural, time-driven event requiring no privileged access or collusion.
- `verify_tee()` is callable by any single participant (a Byzantine node strictly below the signing threshold), satisfying the prompt's attacker model.
- The window between a user's `sign()` call and the `verify_tee()` call that sets `accept_requests = false` is realistic in production: attestations expire on a schedule, and `verify_tee()` is called periodically by nodes.
- No threshold-level collusion, leaked keys, or physical TEE attacks are required.

---

### Recommendation

Separate the "accept new requests" gate from the "resolve existing requests" gate:

1. Keep the `accept_requests` check in `check_request_preconditions` (guards `sign`, `request_app_private_key`, `verify_foreign_transaction`) to block new submissions.
2. **Remove** the `accept_requests` check from `respond()`, `respond_ckd()`, and `respond_verify_foreign_tx()`. Nodes should always be permitted to resolve yields that were validly enqueued before the flag changed.
3. Alternatively, add an administrative endpoint that, when `accept_requests` is `false`, iterates over `pending_signature_requests`, `pending_ckd_requests`, and `pending_verify_foreign_tx_requests` and force-resumes each yield with a failure response so that users receive a clean error and their deposits are refunded.

---

### Proof of Concept

```
1. Contract is Running, accept_requests = true.
2. Alice calls sign() with 1 yoctoNEAR deposit.
   → pending_signature_requests[alice_request] = [yield_index_0]
3. Participant P1 (Byzantine, below threshold) calls verify_tee().
   → Two or more attestations have expired naturally.
   → accept_requests = false  (lib.rs:1737)
4. MPC nodes compute the signature and call respond(alice_request, sig).
   → respond() hits the guard at lib.rs:579-581 → Err(TeeValidationFailed)
   → yield_index_0 is never resumed.
5. NEAR yield timeout elapses.
   → return_signature_and_clean_state_on_success(alice_request, Err(Failed)) fires.
   → pending_signature_requests entry removed.
   → Alice's deposit is consumed; her cross-chain transaction is permanently lost.
6. No administrative function exists to rescue yield_index_0 while
   accept_requests == false.

    #[handle_result]
    pub fn respond(
        &mut self,
        request: SignatureRequest,
        response: dtos::SignatureResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!("respond: signer={}, request={:?}", &signer, &request);

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain)?;

        let signature_is_valid = match (&response, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                // generate the expected public key
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");
                let affine = *k256::PublicKey::try_from(&secp_pk)
                    .expect("stored key is always valid")
                    .as_affine();
                let expected_public_key =
                    derive_key_secp256k1(&affine, &request.tweak).map_err(RespondError::from)?;

                let payload_hash = request.payload.as_ecdsa().expect("Payload is not ECDSA");

                // Check the signature is correct
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    payload_hash,
                    &expected_public_key,
                )
                .is_ok()
            }
            (
                dtos::SignatureResponse::Ed25519 { signature },
                PublicKeyExtended::Ed25519 {
                    edwards_point: public_key_edwards_point,
                    ..
                },
            ) => {
                let derived_public_key_edwards_point = derive_public_key_edwards_point_ed25519(
                    &public_key_edwards_point,
                    &request.tweak,
                );
                let derived_public_key_32_bytes =
                    dtos::Ed25519PublicKey::from(derived_public_key_edwards_point.compress());

                let message = request.payload.as_eddsa().expect("Payload is not EdDSA");

                near_mpc_signature_verifier::verify_eddsa_signature(
                    signature,
                    message,
                    &derived_public_key_32_bytes,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_signature_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

    pub fn verify_tee(&mut self) -> Result<bool, Error> {
        log!("verify_tee: signer={}", env::signer_account_id());
        // Caller must be a participant (node or operator).
        self.voter_or_panic();
        let ProtocolContractState::Running(running_state) = &mut self.protocol_state else {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        };
        let current_params = running_state.parameters.clone();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        match self.tee_state.reverify_and_cleanup_participants(
            current_params.participants(),
            tee_upgrade_deadline_duration,
        ) {
            TeeValidationResult::Full => {
                self.accept_requests = true;
                log!("All participants have an accepted Tee status");
                Ok(true)
            }
            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            } => {
                let remaining = participants_with_valid_attestation.len();
                // Defense in depth: the surviving participant set must keep the full
                // threshold relation intact — the GovernanceThreshold must still sit
                // within its bounds for the smaller set (in particular it must not
                // exceed the remaining participant count or the upper cap) and must
                // remain at least every domain's ReconstructionThreshold (the kickout
                // keeps the existing per-domain thresholds). Otherwise we refuse and
                // wait for manual intervention.
                let max_reconstruction_threshold =
                    max_reconstruction_threshold(running_state.domains.domains());
                if let Err(err) = ThresholdParameters::validate_governance_against_reconstruction(
                    u64::try_from(remaining).expect("participant count fits in u64"),
                    current_params.threshold(),
                    max_reconstruction_threshold,
                ) {
                    log!(
                        "Kicking out participants with an invalid TEE status would break the threshold relation ({:?}); {} participants remain with a valid TEE status. This requires manual intervention. We will not accept new signature requests as a safety precaution.",
                        err,
                        remaining,
                    );
                    self.accept_requests = false;
                    return Ok(false);
                }

                // here, we set it to true, because at this point, we have at least `threshold`
                // number of participants with an accepted Tee status.
                self.accept_requests = true;

                // do we want to adjust the threshold?
                //let n_participants_new = new_participants.len();
                //let new_threshold = (3 * n_participants_new + 4) / 5; // minimum 60%
                //let new_threshold = new_threshold.max(2); // but also minimum 2
                let new_threshold = usize::try_from(current_params.threshold().value())
                    .expect("threshold value fits in usize");

                let threshold_parameters = ThresholdParameters::new(
                    participants_with_valid_attestation,
                    Threshold::new(new_threshold as u64),
                )
                .expect("Require valid threshold parameters"); // this should never happen.
                current_params.validate_incoming_proposal(&threshold_parameters)?;
                // This resharing only changes the participant set, so the
                // per-domain reconstruction-threshold updates map is empty.
                let proposed_parameters =
                    ProposedThresholdParameters::new(threshold_parameters, BTreeMap::new());
                let res = running_state.transition_to_resharing_no_checks(&proposed_parameters);
                if let Some(resharing) = res {
                    self.protocol_state = ProtocolContractState::Resharing(resharing);
                }

                Ok(true)
            }
        }
    }

/// Resume every yield queued for `request` with `response_bytes`, draining the
/// fan-out map in one pass. Returns `Err(RequestNotFound)` if the map held no entry.
///
/// Resuming a yield that has already timed out is a no-op at the SDK level.
pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
