### Title
Sub-Transaction Treasury Donation Bypasses Value Conservation, Enabling Arbitrary ADA Creation — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxo.hs`)

---

### Summary

`dijkstraSubUtxoTransition` unconditionally appends `txBody ^. treasuryDonationTxBodyL` to `utxosDonationL` after calling `updateUTxOStateNoFees`, but neither `SUBUTXO` nor any rule in its call chain enforces value conservation for sub-transactions. The only value-conservation check in the Dijkstra era operates exclusively on the top-level tx body and uses `originalUtxo`; it is structurally blind to sub-transaction treasury donations. An unprivileged user who controls any UTxO can therefore inflate `utxosDonation` by an arbitrary amount, creating ADA from nothing that is transferred to the treasury at the epoch boundary.

---

### Finding Description

**Rule execution order in `dijkstraLedgerTransition`:**

1. `SUBLEDGERS` processes all sub-transactions (modifying `UTxOState`).
2. `UTXOW` → `UTXO` processes the top-level tx, running `validateValueNotConservedUTxO` against `originalUtxo` and the top-level `txBody` only. [1](#0-0) 

**`SUBUTXO` applies the donation with no balance check:**

```haskell
-- SubUtxo.hs line 267-276
newState <-
  Shelley.updateUTxOStateNoFees
    pp utxoState txBody certState
    (utxosGovState utxoState)
    (tellEvent . TotalDeposits (hashAnnotated txBody))
    (\a b -> tellEvent $ TxUTxODiff a b)
pure $ newState & utxosDonationL <>~ txBody ^. treasuryDonationTxBodyL
``` [2](#0-1) 

There is no call to `validateValueNotConservedUTxO` anywhere in `dijkstraSubUtxoTransition`. The `grep_search` confirms `validateValueNotConservedUTxO` appears only in `Utxo.hs` (the top-level rule) within the Dijkstra era. [3](#0-2) 

**`validateTreasuryValue` in `SUBLEDGER` does not constrain the donation amount:**

`Conway.validateTreasuryValue` checks only the optional `currentTreasuryValueTxBodyL` field (a declaration of the current treasury balance), not `treasuryDonationTxBodyL`. If `currentTreasuryValue` is `SNothing`, the check is a no-op. [4](#0-3) [5](#0-4) 

**`updateUTxOStateNoFees` does not include the donation in the UTxO balance:**

The function removes inputs, adds outputs, and adjusts deposits — but leaves `utxosDonation` unchanged. The donation is appended afterward as a separate operation, with no accounting against the UTxO delta. [6](#0-5) 

**Top-level `producedTxBody` does not include sub-transaction donations:**

`producedTxBody` sums `proOutputs + proFees + proDeposits` for the top-level tx body only. Sub-transaction treasury donations are never included in the consumed/produced balance. [7](#0-6) 

---

### Impact Explanation

**Critical — Direct creation of ADA through an invalid ledger state transition.**

A sub-transaction with inputs worth X ADA and outputs worth X ADA (UTxO-balanced) can set `treasuryDonation = Y` for any positive `Y`. After processing:

- UTxO net change: 0 (inputs removed, equal outputs added)
- `utxosDonation` increase: +Y ADA

Y ADA is created from nothing. At the epoch boundary this inflated donation is transferred to the treasury, permanently increasing the circulating supply beyond the protocol-defined maximum of 45 billion ADA.

---

### Likelihood Explanation

Any user who controls at least one UTxO entry can construct this attack. No governance majority, privileged key, or special role is required. The attacker only needs to:

1. Own any UTxO.
2. Construct a sub-transaction spending that UTxO with equal-value outputs and an arbitrarily large `treasuryDonation`.
3. Embed it in a valid top-level transaction.

The top-level tx passes all checks because its own value conservation is satisfied. The sub-transaction passes all checks because `SUBUTXO` has no value conservation check.

---

### Recommendation

Add a value-conservation check inside `dijkstraSubUtxoTransition` that accounts for the treasury donation. Specifically, for a valid sub-transaction, enforce:

```
consumed(inputs) = produced(outputs) + depositChange + treasuryDonation
```

This mirrors the invariant enforced by `validateValueNotConservedUTxO` for top-level transactions, extended to include the donation field. Alternatively, restrict `treasuryDonationTxBodyL` in sub-transactions to always be zero (if donations from sub-transactions are not an intended feature). [2](#0-1) 

---

### Proof of Concept

```
1. Start with UTxO: { txIn_A → (addr_attacker, 100 ADA) }
2. Construct sub-tx:
     inputs  = { txIn_A }
     outputs = { (addr_attacker, 100 ADA) }   -- balanced
     treasuryDonation = 45_000_000_000_000_000  -- 45 billion ADA
     currentTreasuryValue = SNothing            -- skip validateTreasuryValue
3. Construct top-level tx:
     inputs  = { some other UTxO the attacker controls }
     outputs = { change output }
     fee     = minFee
     -- value conservation satisfied for top-level tx body
4. Submit. SUBLEDGERS processes sub-tx:
     - SUBUTXO: no validateValueNotConservedUTxO called
     - utxosDonation += 45_000_000_000_000_000
5. UTXOW processes top-level tx:
     - validateValueNotConservedUTxO passes (checks only top-level txBody)
6. At epoch boundary: treasury += 45_000_000_000_000_000 ADA
7. Assert: utxosDonation >> sum of all legitimate donations in the block
```

updateUTxOStateNoFees pp utxos txBody certState govState depositChangeEvent txUtxODiffEvent = do
  let UTxOState {utxosUtxo, utxosDeposited, utxosFees, utxosDonation} = utxos
      UTxO utxo = utxosUtxo
      !utxoAdd = txouts txBody -- These will be inserted into the UTxO
      {- utxoDel  = txins txb ◁ utxo -}
      !(utxoWithout, utxoDel) = extractKeys utxo (txBody ^. inputsTxBodyL)
      {- newUTxO = (txins txb ⋪ utxo) ∪ outs txb -}
      newUTxO = utxoWithout `Map.union` unUTxO utxoAdd
      deletedUTxO = UTxO utxoDel
      totalRefunds = certsTotalRefundsTxBody pp certState txBody
      totalDeposits = certsTotalDepositsTxBody pp certState txBody
      depositChange = totalDeposits <-> totalRefunds
  depositChangeEvent depositChange
  txUtxODiffEvent deletedUTxO utxoAdd
  pure $!
    UTxOState
      { utxosUtxo = UTxO newUTxO
      , utxosDeposited = utxosDeposited <> depositChange
      , utxosFees = utxosFees
      , utxosGovState = govState
      , utxosInstantStake =
          deleteInstantStake deletedUTxO (addInstantStake utxoAdd (utxos ^. instantStakeL))
      , utxosDonation = utxosDonation
      }
