Audit Report

## Title
Missing `msg.sender == account` Authorization in `MerkleDistributor.claim()` Allows Forced Claims with Involuntary Fee Deduction - (File: `contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

## Summary
`MerkleDistributor.claim()` accepts a caller-supplied `account` parameter but never verifies that `msg.sender == account`. Any unprivileged external caller can trigger a valid claim on behalf of any account, forcing an immediate fee deduction from the victim's claimable yield. When `feeInBPS > 0` (up to 10%), the fee is irrecoverably sent to `protocolTreasury`, permanently reducing the victim's entitled distribution without their consent.

## Finding Description
In `contracts/utils/MerkleDistributor/MerkleDistributor.sol`, the `claim` function at lines 97–147 accepts `account` as a caller-supplied parameter and verifies the Merkle proof against `(index, account, cumulativeAmount)`. There is no check that `msg.sender == account`. [1](#0-0) 

After proof verification, the contract computes a fee of up to 10% of `claimableAmount` and transfers it to `protocolTreasury`, with the remainder sent to `account`. [2](#0-1) 

The sibling contract `KernelMerkleDistributor._processClaim()` explicitly guards against this with `if (account != msg.sender) { revert Unauthorized(); }` at lines 311–313, but `MerkleDistributor` has no equivalent guard. [3](#0-2) 

Exploit path:
1. Owner sets `feeInBPS = 1000` (10%) and publishes a Merkle root.
2. Victim has a valid leaf `(index, account=0xVICTIM, cumulativeAmount)` and is waiting for the fee to be reduced before claiming.
3. Attacker derives the Merkle proof from off-chain distribution data or on-chain events.
4. Attacker calls `merkleDistributor.claim(index, 0xVICTIM, cumulativeAmount, victimProof)`.
5. Contract verifies the proof (valid), deducts 10% fee, transfers 90% to `0xVICTIM` and 10% to `protocolTreasury`.
6. `userClaims[0xVICTIM].lastClaimedIndex` is updated; the victim can never reclaim the fee portion.

## Impact Explanation
This is a **High** severity finding: **Theft of unclaimed yield**. The victim's entitled yield distribution is permanently reduced by the fee amount (`claimableAmount * feeInBPS / 10_000`) without their consent. The fee is irrecoverably transferred to `protocolTreasury`. The victim receives less than their full entitlement and has no recourse. [4](#0-3) 

## Likelihood Explanation
- No special role, key, or oracle access is required — any unprivileged external account can call `claim()`.
- Merkle proof parameters are either emitted in events (`Claimed`) or derivable from off-chain distribution data, making them publicly observable.
- `feeInBPS` is configurable up to `MAX_FEE_IN_BPS = 1000` (10%); when non-zero, every forced claim extracts yield from the victim. [4](#0-3) 
- The attack is repeatable across every new Merkle root index and every eligible account.

## Recommendation
Add a caller authorization check at the top of `claim()`, before Merkle proof verification, mirroring the pattern already established in `KernelMerkleDistributor._processClaim()`: [3](#0-2) 

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Insert this immediately after the `isClaimed` check in `MerkleDistributor.claim()`. [5](#0-4) 

## Proof of Concept
Foundry fork test outline:

```solidity
function test_forcedClaimExtractsYield() public {
    // Setup: owner sets feeInBPS = 1000, sets merkle root
    vm.prank(owner);
    distributor.setFeeInBPS(1000);
    vm.prank(owner);
    distributor.setMerkleRoot(merkleRoot);

    uint256 victimBalanceBefore = token.balanceOf(victim);
    uint256 treasuryBalanceBefore = token.balanceOf(protocolTreasury);

    // Attacker forces claim on behalf of victim
    vm.prank(attacker);
    distributor.claim(index, victim, cumulativeAmount, victimProof);

    // Victim received only 90% of entitlement
    assertEq(token.balanceOf(victim) - victimBalanceBefore, cumulativeAmount * 9000 / 10_000);
    // Treasury received 10% fee without victim's consent
    assertEq(token.balanceOf(protocolTreasury) - treasuryBalanceBefore, cumulativeAmount * 1000 / 10_000);
    // Victim's claim is now marked as used — cannot reclaim the fee
    assertTrue(distributor.isClaimed(index, victim));
}
``` [6](#0-5)
