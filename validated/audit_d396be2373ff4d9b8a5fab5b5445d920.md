## Verdict

### Title
Wallet Transaction Permanently Suppressed via Peer-Reported FAILED Mempool Responses — (`chia/wallet/wallet_transaction_store.py`: `increment_sent`)

### Summary
After `minimum_send_attempts` (6) `MempoolInclusionStatus.FAILED` responses with non-fee errors from full node peers, `increment_sent` marks a pending transaction `confirmed=True, confirmed_at_height=0`. The transaction is then permanently excluded from `get_not_sent` (`WHERE confirmed=0`) and immune to `rollback_to_block` (`WHERE confirmed_at_height>height` skips height 0), making the spend bundle unretryable without manual intervention.

### Finding Description
`increment_sent` calls `tx.is_valid()` after each peer response. `is_valid()` returns `False` when `len(sent_to) >= 6` and no entry is `SUCCESS` or a fee error. At that point, `increment_sent` replaces the record with `confirmed=True, confirmed_at_height=uint32(0)`.

The two recovery mechanisms both fail for this state:
- `get_not_sent`: `WHERE confirmed=0` — excludes the tx.
- `rollback_to_block(h)`: `WHERE confirmed_at_height > h` — `0 > 0` is false, so the tx survives any rollback.

The only way to trigger this is through the `transaction_ack` handler, which requires the ack to match an in-flight send (`_tx_messages_in_progress` guard). This means the attacker must be a peer the wallet actually sent the tx to — not an arbitrary unsolicited message.

### Impact Explanation
The spend bundle is permanently lost from the wallet's retry queue. The coins remain unspent on-chain, but the wallet displays the transaction as confirmed (at height 0), causing accounting divergence. The user cannot recover without manually inspecting the DB or resyncing the wallet. This fits **High: corruption of wallet sync state with direct security impact**.

### Likelihood Explanation
The attacker must control all 6 peers the wallet connects to (Sybil attack on peer connections). If even one honest peer returns `SUCCESS` or a fee error, `is_valid()` stays `True` and the attack fails. This is a meaningful precondition, but:
- Anyone can run a Chia full node.
- Wallets often connect to a small number of peers (sometimes as few as 1–3).
- A targeted attack against a specific wallet (e.g., by flooding peer discovery with attacker nodes) is realistic.

Full node peers are explicitly untrusted in the Chia model, so this does not require "trusted host compromise."

### Recommendation
1. Do not use `confirmed=True` to represent "permanently failed" — use a distinct state (e.g., a `failed` boolean or a `FAILED` `TransactionType`).
2. `rollback_to_block` should also reset transactions with `confirmed=True AND confirmed_at_height=0 AND sent_to` containing only FAILED entries back to unconfirmed.
3. `add_transaction_record` should remove the entry from `unconfirmed_txs` when `confirmed` transitions to `True`, mirroring `set_confirmed`.
4. Consider rate-limiting or requiring a minimum number of honest-peer confirmations before permanently suppressing a tx.

### Proof of Concept
```python
# 1. Add a pending tx
await store.add_transaction_record(tr1)  # confirmed=False

# 2. Six peers each return FAILED with a non-fee error
for i in range(6):
    await store.increment_sent(
        tr1.name, f"peer{i}", MempoolInclusionStatus.FAILED, Err.UNKNOWN
    )

# 3. tx is now confirmed=True, confirmed_at_height=0
tx = await store.get_transaction_record(tr1.name)
assert tx.confirmed is True
assert tx.confirmed_at_height == 0

# 4. get_not_sent returns nothing (WHERE confirmed=0)
assert await store.get_not_sent() == []

# 5. rollback_to_block(0) does NOT delete it (0 > 0 is false)
await store.rollback_to_block(0)
assert await store.get_transaction_record(tr1.name) is not None
# spend bundle is permanently lost from retry queue
```

    @metadata.request(peer_required=True)
    async def transaction_ack(self, ack: wallet_protocol.TransactionAck, peer: WSChiaConnection) -> None:
        """
        This is an ack for our previous SendTransaction call. This removes the transaction from
        the send queue if we have sent it to enough nodes.
        """
        async with self.wallet_node.wallet_state_manager.lock:
            assert peer.peer_node_id is not None
            name = peer.peer_node_id.hex()
            matched_in_flight_send = False
            if peer.peer_node_id in self.wallet_node._tx_messages_in_progress:
                in_progress = self.wallet_node._tx_messages_in_progress[peer.peer_node_id]
                if ack.txid in in_progress:
                    matched_in_flight_send = True
                    in_progress.remove(ack.txid)
                if in_progress == []:
                    del self.wallet_node._tx_messages_in_progress[peer.peer_node_id]

            # Ignore stale/unsolicited acknowledgements that don't correspond to an in-flight send.
            if not matched_in_flight_send:
                self.wallet_node.log.debug(
                    "Ignoring unsolicited transaction ack from peer %s for tx %s", name, ack.txid
                )
                return

            status = MempoolInclusionStatus(ack.status)
            try:
                wallet_state_manager = self.wallet_node.wallet_state_manager
            except RuntimeError as e:
                if "not assigned" in str(e):
                    return
                raise

            if status == MempoolInclusionStatus.SUCCESS:
                self.wallet_node.log.info(
                    f"SpendBundle has been received and accepted to mempool by the FullNode. {ack}"
                )
            elif status == MempoolInclusionStatus.PENDING:
                self.wallet_node.log.info(f"SpendBundle has been received (and is pending) by the FullNode. {ack}")
            else:
                if not self.wallet_node.is_trusted(peer) and ack.error == Err.NO_TRANSACTIONS_WHILE_SYNCING.name:
                    self.wallet_node.log.info(f"Peer {peer.get_peer_info()} is not synced, closing connection")
                    await peer.close()
                    return
                self.wallet_node.log.warning(f"SpendBundle has been rejected by the FullNode. {ack}")
            if ack.error is not None:
                await wallet_state_manager.remove_from_queue(ack.txid, name, status, Err[ack.error])
            else:
                await wallet_state_manager.remove_from_queue(ack.txid, name, status, None)

    def is_valid(self) -> bool:
        if len(self.sent_to) < minimum_send_attempts:
            # we haven't tried enough peers yet
            return True
        if any(x[1] == MempoolInclusionStatus.SUCCESS.value for x in self.sent_to):
            # we managed to push it to mempool at least once
            return True
        if any(x[2] in {Err.INVALID_FEE_LOW_FEE.name, Err.INVALID_FEE_TOO_CLOSE_TO_ZERO.name} for x in self.sent_to):
