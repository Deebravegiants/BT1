### Title
Unregistered ERC-20 Tokens Accepted by `initTransfer` Cause Permanent Fund Lock — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

### Summary
`OmniBridge.initTransfer` and `OmniBridge.initTransfer1155` accept any caller-supplied token address without verifying it is registered in the bridge's token mapping. Tokens that have no corresponding NEAR-side registration are transferred into the contract and permanently locked: the NEAR `fin_transfer_callback` panics on the missing decimals entry, and the EVM contract exposes no admin-withdrawal path to recover them.

### Finding Description

`initTransfer` branches on three conditions before accepting a token: [1](#0-0) 

```solidity
if (customMinters[tokenAddress] != address(0)) { … burn … }
else if (isBridgeToken[tokenAddress])           { … burn … }
else {
    IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
}
```

The `else` branch is reached for **any** ERC-20 that is neither a custom-minter token nor a deployed bridge token. There is no guard that checks `ethToNearToken[tokenAddress] != ""` — i.e., that the token has a registered NEAR counterpart. The tokens are pulled from the user and held by the contract, and `InitTransfer` is emitted. [2](#0-1) 

On the NEAR side, when a relayer submits the proof, `fin_transfer_callback` immediately looks up `token_decimals` for the token address extracted from the proof: [3](#0-2) 

```rust
let decimals = self
    .token_decimals
    .get(&init_transfer.token)
    .near_expect(BridgeError::TokenDecimalsNotFound);
```

Because the token was never registered on NEAR, this call panics unconditionally. The NEAR transaction reverts; the EVM tokens remain locked. The EVM contract contains no `removeFunds`, `emergencyWithdraw`, or equivalent admin function that could recover arbitrary locked ERC-20 balances. [2](#0-1) 

The identical pattern exists in the StarkNet bridge: `omni_bridge.cairo` `init_transfer` also falls through to an unrestricted `transfer_from` for any token not in `starknet_to_near_token`, with no recovery path. [4](#0-3) 

The Solana program is **not** affected: the `vault` account is a PDA seeded with `[VAULT_SEED, mint.key()]`, so Anchor's account-constraint system rejects any mint for which no vault PDA was previously created by an admin instruction — providing implicit registration enforcement. [5](#0-4) 

### Impact Explanation

Tokens deposited via the unguarded `else` branch are irreversibly locked in the EVM (or StarkNet) bridge contract. There is no automated or admin-accessible recovery path on-chain. This satisfies the **Critical** impact criterion: *Irreversible fund lock, frozen redemption path, or permanently unclaimable user value in bridge flows.*

### Likelihood Explanation

The entry point is fully public and requires no privilege. A user who mistakenly (or experimentally) calls `initTransfer` with a token that has not yet been registered via `deployToken` or `addCustomToken` will lose their funds. The `logMetadata` → `deployToken` prerequisite is not enforced anywhere in `initTransfer`, making accidental misuse realistic.

### Recommendation

Add a registration check at the top of `initTransfer` (and `initTransfer1155`) before any token transfer occurs:

```solidity
require(
    tokenAddress == address(0) ||
    bytes(ethToNearToken[tokenAddress]).length > 0 ||
    customMinters[tokenAddress] != address(0),
    "ERR_TOKEN_NOT_REGISTERED"
);
```

Apply the equivalent guard in `starknet/src/omni_bridge.cairo` `init_transfer`:

```cairo
assert(
    self.is_bridge_token(token_address) ||
    self.starknet_to_near_token.read(token_address).len() > 0,
    'ERR_TOKEN_NOT_REGISTERED'
);
```

### Proof of Concept

1. `USDT_FAKE` is an ERC-20 that was never passed through `logMetadata` + `deployToken` on the EVM bridge.
2. Attacker (or confused user) calls:
   ```solidity
   omniBridge.initTransfer(USDT_FAKE, 1_000e6, 0, 0, "near:victim.near", "");
   ```
3. The `else` branch executes `safeTransferFrom(msg.sender, address(this), 1_000e6)`. Tokens are now held by `OmniBridge`. `InitTransfer` event is emitted.
4. A relayer submits the EVM proof to NEAR `fin_transfer`. `fin_transfer_callback` calls `self.token_decimals.get(&init_transfer.token).near_expect(BridgeError::TokenDecimalsNotFound)` — panics. NEAR transaction reverts.
5. No further relayer attempt can succeed. The EVM contract has no withdrawal function. `USDT_FAKE` tokens are permanently locked. [6](#0-5) [7](#0-6)

        address tokenAddress,

        currentOriginNonce += 1;

            revert InvalidFee();

        uint256 extensionValue;
        if (tokenAddress == address(0)) {
            if (fee != 0) {
                revert InvalidFee();
            }
            extensionValue = msg.value - amount - nativeFee;

            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {
                IERC20(tokenAddress).safeTransferFrom(

                    amount
                );

                    msg.sender,
                    address(this),
                    amount
                );

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,
            recipient: ByteArray,
            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::InitTransfer(
                        InitTransfer {
                            sender: caller,
                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
                )
        }
