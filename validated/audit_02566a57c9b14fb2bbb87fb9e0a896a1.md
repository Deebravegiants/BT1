Audit Report

## Title
Stale `rsETHPrice` After Early-Return Pause Path Causes Protocol Fee Under-Collection - (File: contracts/LRTOracle.sol)

## Summary
In `LRTOracle._updateRsETHPrice()`, the downside-protection branch at line 277–282 calls `_pause()` and `return`s without first writing `newRsETHPrice` to the `rsETHPrice` state variable. After the protocol is unpaused, every subsequent fee calculation uses this stale, artificially-high `rsETHPrice` as the TVL baseline, causing the protocol treasury to collect fewer fees than it is entitled to — or none at all.

## Finding Description
`_updateRsETHPrice()` computes `newRsETHPrice` and then checks whether the price drop exceeds `pricePercentageLimit`:

```solidity
// contracts/LRTOracle.sol lines 270-282
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);

    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;   // ← exits WITHOUT executing rsETHPrice = newRsETHPrice
    }
}
```

The assignment `rsETHPrice = newRsETHPrice` at line 313 is never reached. `rsETHPrice` retains the pre-drop value.

On every subsequent call after unpausing, the fee baseline is computed as:

```solidity
// contracts/LRTOracle.sol lines 233-246
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);  // stale high price
...
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
```

Because `rsETHPrice` is stale-high, `previousTVL` is inflated above the actual TVL at the time of the pause. Any yield earned during and after the pause is measured against this inflated baseline, so `rewardAmount` is smaller than the true gain — or the condition `totalETHInProtocol > previousTVL` is never satisfied, and zero fees are collected even though TVL genuinely grew.

The existing `protocolPaused` guard correctly suppresses fees while the protocol is paused, but it provides no protection once the protocol is unpaused: the stale `rsETHPrice` persists indefinitely until a future price update overwrites it, and every fee calculation in that window is distorted.

## Impact Explanation
The protocol treasury (`PROTOCOL_TREASURY`) receives fewer rsETH fee tokens than it is entitled to. In the worst case — where TVL does not recover above the stale pre-drop price — all yield earned after the pause is permanently exempt from the protocol fee. This is a direct, quantifiable, permanent loss of unclaimed yield for the protocol, matching the allowed impact: **High — Theft of unclaimed yield**.

## Likelihood Explanation
`updateRSETHPrice()` is `public` and callable by any address with no access control. The downside-protection branch fires whenever a single price update shows a decline exceeding `pricePercentageLimit` relative to `highestRsethPrice` — a realistic scenario during any significant market drawdown or EigenLayer slashing event. No privileged access is required to trigger the root cause. The stale state persists automatically until the next successful price update that clears the condition, making the impact durable and repeatable.

## Recommendation
Update `rsETHPrice` to `newRsETHPrice` **before** calling `_pause()` and returning, so the baseline for future fee calculations always reflects the actual price at the time of the pause:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    rsETHPrice = newRsETHPrice;   // ← record the actual price before pausing
    _pause();
    return;
}
```

## Proof of Concept
1. Initial state: `rsETHPrice = 1.10e18`, `highestRsethPrice = 1.10e18`, `rsethSupply = 1000e18`, `pricePercentageLimit = 0.05e18` (5%).
2. A slashing event reduces TVL to 900 ETH. `newRsETHPrice = 0.90e18` — an 18% drop, exceeding the 5% limit.
3. Any address calls `updateRSETHPrice()`. The early-return path fires: protocol is paused, `rsETHPrice` stays at `1.10e18`.
4. Admin unpauses. TVL recovers to 1050 ETH through staking rewards.
5. `updateRSETHPrice()` is called. `previousTVL = 1000 × 1.10 = 1100 ETH`. Since `1050 < 1100`, the condition `totalETHInProtocol > previousTVL` is **false** — zero fees are charged.
6. The actual gain since the pause was `1050 − 900 = 150 ETH`. At a 10% protocol fee that is 15 ETH of fees the treasury never receives.
7. Even if TVL recovers to 1200 ETH, fees are charged only on `1200 − 1100 = 100 ETH` instead of the true gain of `1200 − 900 = 300 ETH`, permanently under-collecting fees on the 200 ETH earned during the pause.

**Foundry test plan**: Deploy `LRTOracle` with a mock config; set `rsETHPrice = 1.10e18`, `highestRsethPrice = 1.10e18`, `pricePercentageLimit = 0.05e18`; mock `_getTotalEthInProtocol()` to return 900e18; call `updateRSETHPrice()` as an unprivileged address; assert `paused == true` and `rsETHPrice == 1.10e18` (unchanged); unpause; mock TVL to 1050e18; call `updateRSETHPrice()`; assert `protocolFeeInETH == 0` and treasury rsETH balance is unchanged.
