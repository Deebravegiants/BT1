### Title
Expired DRep Can Revive Itself via `ConwayUpdateDRep` Certificate, Bypassing Governance Participation Expiry — (File: `eras/conway/impl/src/Cardano/Ledger/Conway/Rules/GovCert.hs`)

---

### Summary

The `GOVCERT` rule's handler for `ConwayUpdateDRep` certificates does not verify that the target DRep is still within its active (non-expired) window before resetting its expiry epoch. An expired DRep credential holder can submit a `ConwayUpdateDRep` certificate to unconditionally overwrite their stored `drepExpiry` with a future epoch, reviving themselves and re-entering governance ratification without paying a new registration deposit. This is the direct Cardano analog of the reported `extend_expiration()` post-grace-period bypass.

---

### Finding Description

In `conwayGovCertTransition`, the `ConwayUpdateDRep` branch performs exactly one guard:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Rules/GovCert.hs  lines 256-272
ConwayUpdateDRep cred mAnchor -> do
  Map.member cred (certState ^. certVStateL . vsDRepsL)
    ?! (injectFailure . ConwayDRepNotRegistered) cred
  pure $
    certState
      & certVStateL . vsDRepsL
          %~ Map.adjust
               ( \drepState ->
                   drepState
                     & drepExpiryL
                         .~ computeDRepExpiry
                              ppDRepActivity
                              cgceCurrentEpoch
                              (certState ^. certVStateL . vsNumDormantEpochsL)
                     & drepAnchorL .~ mAnchor
               )
               cred
``` [1](#0-0) 

The sole predicate is `Map.member cred vsDRepsL` — the DRep is registered. There is **no check** that the DRep's actual expiry (`drepExpiry + vsNumDormantEpochs`) has not already passed the current epoch.

The "actual expiry" concept is defined elsewhere. The test helper `isDRepExpired` computes:

```haskell
binOpEpochNo (+) (vState ^. vsNumDormantEpochsL) (drep' ^. drepExpiryL) < currentEpoch
``` [2](#0-1) 

And `updateDormantDRepExpiry` — the only other place that bulk-updates expiries — explicitly skips DReps whose actual expiry is already past:

```haskell
let actualExpiry = binOpEpochNo (+) numDormantEpochs currentExpiry
 in if actualExpiry < currentEpoch
      then currentExpiry   -- already expired: do NOT extend
      else actualExpiry
```
