### Title
Unvalidated `builder` Field in `ProposalInit` Allows Malicious Proposer to Redirect Fees and Corrupt Block Hash — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, `crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`ProposalInit.builder` is accepted verbatim from the network and used as `sequencer_address` in every block-execution context, but `is_proposal_init_valid` never checks it against any locally-trusted value. A legitimate-but-malicious proposer can set `builder` to an arbitrary address, redirecting all transaction fees for the block to an attacker-controlled account, corrupting the on-chain block hash (which commits `sequencer_address`), and returning a wrong value from the `get_sequencer_address` syscall.

---

### Finding Description

`ProposalInit` carries a `builder` field defined as "Address of the one who builds/sequences the block": [1](#0-0) 

`convert_to_sn_api_block_info` maps this field directly to `sequencer_address` in the `BlockInfo` that is handed to the blockifier for every transaction in the block: [2](#0-1) 

The proposer sets `builder` from its own static config: [3](#0-2) 

The validator, however, never checks this field. `is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri` — but `builder` is absent from both the check list and the `ProposalInitValidation` struct: [4](#0-3) [5](#0-4) 

The `proposer` field is validated against the committee in `handle_proposal`: [6](#0-5) 

But `builder` receives no equivalent check. The TODO comment in the proposer path acknowledges this gap: [3](#0-2) 

Because the validator's batcher receives the attacker-supplied `builder` value via `initiate_validation` → `convert_to_sn_api_block_info`, it executes all transactions with the wrong `sequencer_address`. The batcher then produces a `PartialBlockHash` that commits this wrong address: [7](#0-6) [8](#0-7) 

Since both proposer and validator use the same attacker-supplied `builder`, the `ProposalCommitment` computed by the batcher matches the one in `ProposalFin`, so the `ProposalFinMismatch` guard does not fire: [9](#0-8) 

The block is accepted by consensus and committed with the corrupted `sequencer_address`.

---

### Impact Explanation

**Fee theft (Critical — economic impact):** Every transaction fee in the block is transferred to `block_context.block_info.sequencer_address`, which is the attacker-controlled `builder` address. The fee transfer calldata uses this address as the recipient: [10](#0-9) 

**Wrong block hash committed on-chain (Critical — wrong state commitment):** `sequencer_address` is a direct input to `calculate_block_hash`. A wrong `builder` produces a wrong `PartialBlockHash`, which propagates into the `ProposalCommitment` that consensus votes on and into the L1-anchored block hash.

**Wrong `get_sequencer_address` syscall result (Critical — wrong execution result):** Contracts that call `get_sequencer_address` receive the attacker-controlled address, corrupting any contract logic that depends on knowing the real sequencer. [11](#0-10) 

---

### Likelihood Explanation

The trigger requires being the elected proposer for a consensus round — a role that rotates among committee members. Any validator node that is selected as proposer can exploit this without any additional privilege. In a network with multiple validators (including potentially adversarial ones), this is a realistic threat. The attack is silent: no error is logged, no metric fires, and the block passes all existing validation checks.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and check it in `is_proposal_init_valid` against the locally-configured `builder_address` (or, once the committee API is available, against the committee-derived value). The check should mirror the existing `proposer` check in `handle_proposal`:

```rust
// In ProposalInitValidation:
pub builder_address: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

`SequencerConsensusContext::validate_current_round_proposal` should populate `builder_address` from `self.config.static_config.builder_address` when constructing `ProposalInitValidation`.

---

### Proof of Concept

1. Attacker controls a validator node that is elected proposer for height H, round R.
2. Attacker constructs `ProposalInit` with all legitimate fields (height, gas prices, etc.) but sets `builder = ATTACKER_ADDRESS`.
3. Attacker streams the proposal to peer validators.
4. Each validator calls `is_proposal_init_valid` — passes, because `builder` is not checked.
5. Each validator calls `initiate_validation` → `convert_to_sn_api_block_info(&init)` → `sequencer_address = ATTACKER_ADDRESS`.
6. The batcher executes all transactions with `sequencer_address = ATTACKER_ADDRESS`; all fees are transferred to `ATTACKER_ADDRESS`.
7. The batcher computes `PartialBlockHash` committing `ATTACKER_ADDRESS` as sequencer.
8. `proposal_commitment_from(partial, fee_proposal)` produces a commitment that matches the attacker's `ProposalFin`.
9. `ProposalFinMismatch` check passes; the proposal is accepted and inserted into `valid_proposals`.
10. Consensus reaches decision; `decision_reached` commits the block with the wrong sequencer address, wrong block hash, and drained fees.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));

    partial_block_hash_components: &PartialBlockHashComponents,

    Ok(BlockHash(

            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())

            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)

                gas_prices_to_hash(
