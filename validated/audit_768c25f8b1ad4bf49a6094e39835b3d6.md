Audit Report

## Title
`stakedButUnverifiedNativeETH` Permanently Non-Decrementable After Full EigenLayer Slash Traps NodeDelegator in Queue - (`contracts/NodeDelegator.sol`, `contracts/LRTDepositPool.sol`)

## Summary
When ETH is staked via `stake32Eth()` but `verifyWithdrawalCredentials()` has not yet been called, `stakedButUnverifiedNativeETH` is incremented. If the delegated EigenLayer operator is subsequently slashed to `maxMagnitude = 0`, `verifyWithdrawalCredentials()` becomes permanently uncallable because EigenLayer's `increaseDelegatedShares()` reverts at `maxMagnitude = 0`. This leaves `stakedButUnverifiedNativeETH` permanently non-zero, causing `_checkResidueEthBalance()` to always revert with `NodeDelegatorHasETH`, permanently trapping the NodeDelegator in the queue with no admin recovery path.

## Finding Description

**Root cause — `getEffectivePodShares()` sums two independent quantities:**

`getEffectivePodShares()` at `contracts/NodeDelegator.sol:556-562` returns `stakedButUnverifiedNativeETH + withdrawableShare`. After a full slash, `withdrawableShare` (derived from EigenLayer's `DelegationManager.getWithdrawableShares()`) correctly drops to 0, but `stakedButUnverifiedNativeETH` is a local storage variable that EigenLayer's slashing mechanism never touches. [1](#0-0) 

**`stakedButUnverifiedNativeETH` has exactly two write sites:**

- Incremented in `stake32Eth()` at line 166. [2](#0-1) 
- Decremented in `verifyWithdrawalCredentials()` at line 240, immediately before calling `eigenPod.verifyWithdrawalCredentials()`. [3](#0-2) 

No admin setter, no override, and no other write path exists (confirmed by exhaustive grep across the repository).

**`eigenPod.verifyWithdrawalCredentials()` is permanently blocked after a full slash:**

EigenLayer's `IDelegationManager.increaseDelegatedShares()` is explicitly documented to revert when the operator's `maxMagnitude = 0`. The call chain `eigenPod.verifyWithdrawalCredentials()` → `EigenPodManager.recordBeaconChainETHBalanceUpdate()` → `DelegationManager.increaseDelegatedShares()` means that after a full slash, every call to `verifyWithdrawalCredentials()` reverts, making `stakedButUnverifiedNativeETH` permanently non-decrementable. [4](#0-3) 

**`_checkResidueEthBalance()` gates NDC removal with a strict `!= 0` check:**

`_removeNodeDelegatorContractFromQueue()` calls `_checkResidueEthBalance()`, which reverts with `NodeDelegatorHasETH` if `getEffectivePodShares() != 0`. Since `stakedButUnverifiedNativeETH` is permanently non-zero, this check always reverts. [5](#0-4) 

Both `removeNodeDelegatorContractFromQueue()` and `removeManyNodeDelegatorContractsFromQueue()` route through `_removeNodeDelegatorContractFromQueue()`, so both are permanently blocked. [6](#0-5) 

**Queue capacity is finite:** `addNodeDelegatorContractToQueue()` enforces `nodeDelegatorQueue.length + length > maxNodeDelegatorLimit`, so a permanently stuck NDC consuming a queue slot can block protocol migration to a new operator. [7](#0-6) 

## Impact Explanation

**Critical — Permanent freezing of funds and permanent freezing of the NodeDelegator.**

1. **ETH frozen in EigenPod**: After a full slash, the validator can exit the beacon chain and ETH lands in the EigenPod. However, `startCheckpoint()` → `completeCheckpoint()` also routes through `increaseDelegatedShares()`, which reverts at `maxMagnitude = 0`. The ETH in the EigenPod is irrecoverable through any EigenLayer mechanism.
2. **NDC permanently stuck**: `removeNodeDelegatorContractFromQueue()` always reverts with `NodeDelegatorHasETH`. The NDC cannot be dequeued.
3. **Protocol migration blocked**: If the NDC queue is at `maxNodeDelegatorLimit`, new NDCs cannot be added, preventing the protocol from migrating to a healthy operator.

This matches the allowed impact: **Critical — Permanent freezing of funds**.

## Likelihood Explanation

**Low-Medium.** Two preconditions must coincide:
1. ETH staked via `stake32Eth()` but `verifyWithdrawalCredentials()` not yet called — a normal operational window that can last hours to days.
2. The EigenLayer operator is slashed to `maxMagnitude = 0` by a registered AVS — a legitimate EigenLayer operation callable by any AVS against any operator that has allocated magnitude to it.

EigenLayer slashing is a designed protocol feature, not an external protocol compromise. An operator registering with an AVS that later slashes them (whether due to misbehavior or a malicious AVS) is a realistic risk in the EigenLayer ecosystem. The timing window (precondition 1) is always present during normal operations.

## Recommendation

1. **Add an admin function to reset `stakedButUnverifiedNativeETH`** (e.g., `onlyLRTAdmin`) for post-slash recovery, allowing the admin to zero out the variable when EigenLayer's slashing has made it irrecoverable.
2. **Alternatively, modify `_checkResidueEthBalance()`** to skip the `getEffectivePodShares()` check when the operator's `maxMagnitude == 0` (i.e., a full slash has occurred), treating `stakedButUnverifiedNativeETH` as irrecoverably burned.
3. **Consider a `forceRemoveNodeDelegator()` function** callable by admin that bypasses the ETH balance check when the operator has been fully slashed, to allow protocol migration without a proxy upgrade.

## Proof of Concept

```solidity
// Foundry fork test
function test_slashTrapsNDC() public {
    // 1. Stake 32 ETH, do NOT verify credentials
    vm.prank(operator);
    nodeDelegator.stake32Eth(pubkey, sig, root);
    assertEq(nodeDelegator.stakedButUnverifiedNativeETH(), 32 ether);

    // 2. AVS slashes operator to maxMagnitude = 0 via EigenLayer AllocationManager
    IAllocationManager.SlashingParams memory params = IAllocationManager.SlashingParams({
        operator: operator,
        operatorSetId: 1,
        strategies: beaconStrategies,
        wadsToSlash: [1e18], // 100%
        description: "slash"
    });
    vm.prank(avs);
    allocationManager.slashOperator(avs, params);

    // 3. getEffectivePodShares() returns stakedButUnverifiedNativeETH (withdrawableShare = 0)
    assertEq(nodeDelegator.getEffectivePodShares(), 32 ether);

    // 4. removeNodeDelegatorContractFromQueue reverts permanently
    vm.prank(admin);
    vm.expectRevert(ILRTDepositPool.NodeDelegatorHasETH.selector);
    lrtDepositPool.removeNodeDelegatorContractFromQueue(address(nodeDelegator));

    // 5. verifyWithdrawalCredentials also reverts (maxMagnitude = 0 blocks increaseDelegatedShares)
    vm.prank(operator);
    vm.expectRevert();
    nodeDelegator.verifyWithdrawalCredentials(ts, stateRootProof, indices, proofs, fields);

    // 6. stakedButUnverifiedNativeETH is permanently non-zero — no admin reset exists
    assertEq(nodeDelegator.stakedButUnverifiedNativeETH(), 32 ether);
}
```
