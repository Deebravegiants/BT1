Audit Report

## Title
Missing Chainlink Oracle Validity Checks Allow Stale or Zero Price to Corrupt rsETH Minting and Pricing — (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `roundId`, `updatedAt`, and `answeredInRound`, performing no staleness, completeness, or sign validation. The same protocol codebase applies all three guards in `ChainlinkOracleForRSETHPoolCollateral.sol`, confirming the omission is unintentional. A stale or zero price fed into `_updateRsETHPrice()` sets an incorrect `rsETHPrice`, enabling subsequent depositors to receive more rsETH than they should, diluting existing holders' accrued yield.

## Finding Description

In `contracts/oracles/ChainlinkPriceOracle.sol`, `getAssetPrice()` reads from a Chainlink feed as:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

All of `roundId`, `updatedAt`, and `answeredInRound` are silently discarded. No check is made for:
1. **Stale round** — `answeredInRound < roundId`
2. **Incomplete round** — `updatedAt == 0`
3. **Non-positive price** — `price <= 0`

By contrast, `ChainlinkOracleForRSETHPoolCollateral.sol` enforces all three:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

`ChainlinkPriceOracle` is the oracle wired into `LRTOracle` for all supported LST assets. Its output flows into `_getTotalEthInProtocol()`, which iterates every supported asset and accumulates `totalETHInProtocol`: [3](#0-2) 

This value directly determines `newRsETHPrice` in `_updateRsETHPrice()`: [4](#0-3) 

`updateRSETHPrice()` is a permissionless public function callable by any user: [5](#0-4) 

**Stale price path:** If a Chainlink LST/ETH feed stalls and the stale price is lower than the true market price, `_getTotalEthInProtocol()` returns a deflated TVL, `newRsETHPrice` is set below its true value, and any depositor calling `depositAsset()` receives more rsETH than they should — diluting all existing holders.

**Zero price path:** If `price == 0` is returned (e.g., during a Chainlink circuit-breaker event), `getAssetPrice` returns 0 for that asset. `_getTotalEthInProtocol()` omits that asset's contribution entirely, further deflating `rsETHPrice` and amplifying the dilution effect.

**Note on negative price:** For `price < 0`, `uint256(price)` wraps to a near-`type(uint256).max` value, but the subsequent multiplication `uint256(price) * 1e18` overflows and reverts under Solidity 0.8+ checked arithmetic. The negative-price path therefore causes a DoS on `updateRSETHPrice` and `depositAsset` rather than unbounded minting — the Critical theft-of-funds scenario described in the submission is not achievable. The High impact (theft of unclaimed yield) via stale/zero price remains valid.

The `pricePercentageLimit` guard in `_updateRsETHPrice` provides partial mitigation: if the price deviation exceeds the configured threshold, the protocol pauses or reverts. However, `pricePercentageLimit` defaults to `0` (no protection), and even when set, deviations within the threshold pass through silently. [6](#0-5) 

## Impact Explanation

**High — Theft of unclaimed yield.** When a Chainlink feed goes stale with a price lower than the true market price, or returns zero, `rsETHPrice` is set below its true value. Any depositor who calls `depositAsset()` or `depositETH()` during this window receives more rsETH than the deposited value warrants: [7](#0-6) 

The excess rsETH minted to the depositor is extracted from the accrued yield of all existing rsETH holders, constituting theft of unclaimed yield. This matches the allowed High impact scope.

## Likelihood Explanation

Chainlink feeds go stale during network congestion, sequencer downtime (on L2 deployments), or when the deviation threshold is not crossed for extended periods. No attacker action is required for the stale-price path — any depositor who transacts during a stale window is affected. `updateRSETHPrice()` is fully permissionless, so the corrupted price can be committed by anyone. The zero-price path is less frequent but has occurred historically during Chainlink circuit-breaker events. The entry points (`depositAsset`, `depositETH`, `updateRSETHPrice`) are all callable by unprivileged external users.

## Recommendation

Apply the same three guards already present in `ChainlinkOracleForRSETHPoolCollateral.sol` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

if (answeredInRound < roundId) revert StalePrice();
if (updatedAt == 0) revert IncompleteRound();
if (price <= 0) revert InvalidPrice();
```

Additionally, add a configurable per-feed `heartbeat` staleness window (e.g., `block.timestamp - updatedAt > heartbeat`) to catch feeds that are technically complete but temporally stale.

## Proof of Concept

1. A Chainlink LST/ETH feed stalls; `updatedAt` is 2 hours old and the true LST price has dropped 0.5% since the last update (within any configured `pricePercentageLimit`).
2. Any caller invokes `LRTOracle.updateRSETHPrice()` (permissionless).
3. `_getTotalEthInProtocol()` calls `ChainlinkPriceOracle.getAssetPrice(lst)`, which returns the stale lower price with no revert.
4. `newRsETHPrice` is computed using the deflated TVL, setting `rsETHPrice` 0.5% below its true value.
5. A depositor calls `depositAsset(lst, amount, 0)`. `getRsETHAmountToMint` divides by the deflated `rsETHPrice`, minting ~0.5% more rsETH than the deposit warrants.
6. The excess rsETH is extracted from the accrued yield of all existing rsETH holders.

**Foundry fork test outline:**
```solidity
// Fork mainnet at a block where a known LST/ETH feed has a stale round
// Mock latestRoundData to return updatedAt = block.timestamp - 3 hours, answeredInRound < roundId
// Call updateRSETHPrice() — observe rsETHPrice set below true value
// Deposit LST — observe rsethAmountToMint exceeds fair value
// Assert existing holder share value decreased
```
