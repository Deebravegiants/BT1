### Title
Unvalidated `builder` Field in `ProposalInit` Allows Arbitrary Sequencer Address Injection into Block Execution - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid()` validates many `ProposalInit` fields but never checks `init_proposed.builder`. This field is directly mapped to `sequencer_address` in the `BlockInfo` passed to the batcher for execution. A malicious proposer can set `builder` to any arbitrary address, causing all fee transfers in the block to be credited to an attacker-controlled address, producing wrong storage values, wrong `get_sequencer_address` syscall results, and a wrong committed state diff.

---

### Finding Description

`ProposalInit` carries a `builder` field defined as "Address of the one who builds/sequences the block": [1](#0-0) 

In `convert_to_sn_api_block_info()`, `init.builder` is unconditionally mapped to `sequencer_address` in the `starknet_api::block::BlockInfo` that is forwarded to the batcher: [2](#0-1) 

This `BlockInfo` is passed to the batcher via `ValidateBlockInput` inside `initiate_validation()`: [3](#0-2) 

The batcher then uses `sequencer_address` as the calldata recipient in every fee transfer executed during block validation: [4](#0-3) 

`is_proposal_init_valid()` validates `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, and `fee_proposal_fri` — but **never** `builder`: [5](#0-4) 

`ProposalInitValidation` does not carry a `builder_address` reference field, so there is no locally-trusted value to compare against: [6](#0-5) 

The proposer's own build path sets `builder` from `self.config.static_config.builder_address`, but nothing prevents a malicious proposer from emitting any other value: [7](#0-6) 

---

### Impact Explanation

`sequencer_address` is the fee-transfer recipient for every transaction in the block. An injected `builder` address causes:

1. **Wrong storage values**: All fee-transfer `ERC-20` storage writes credit the attacker-controlled address instead of the legitimate sequencer, producing a wrong state diff that is committed to the chain.
2. **Wrong `get_sequencer_address` syscall result**: Contracts that call `get_sequencer_address` (or `get_execution_info`) receive the attacker-supplied address, corrupting any contract logic that depends on it.
3. **Wrong concurrency fee-transfer path**: `is_sequencer_the_sender()` compares `sender_address` against `block_context.block_info.sequencer_address`; with a spoofed address the comparison is wrong, potentially selecting the wrong fee-transfer code path and producing a divergent execution result.

This matches: *"Critical. Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input."*

---

### Likelihood Explanation

Any validator that is the designated leader for a consensus round can propose a block. The consensus layer checks that the `proposer` identity matches the expected leader, but `builder` is a separate field that is never cross-checked. A single Byzantine validator in the committee can trigger this on every round it leads, with no additional preconditions.

---

### Recommendation

Add a `builder_address: ContractAddress` field to `ProposalInitValidation`, populate it from `self.config.static_config.builder_address` at the call sites in `validate_proposal()` and `set_height_and_round()`, and add a check inside `is_proposal_init_valid()`:

```rust
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

This mirrors the existing pattern used for `height`, `l1_da_mode`, `l2_gas_price_fri`, and `starknet_version`. [8](#0-7) 

---

### Proof of Concept

1. A malicious (or compromised) validator is the designated leader for round R at height H.
2. It builds a `ProposalInit` with `builder = attacker_address` (any arbitrary `ContractAddress`).
3. The proposal is streamed to peer validators.
4. Each validator calls `validate_proposal()` → `is_proposal_init_valid()`. The function checks timestamp, starknet_version, version_constant_commitment, height, l1_da_mode, l2_gas_price_fri, L1 gas prices, and fee_proposal_fri — all of which can be set correctly. `builder` is never checked. Validation passes.
5. `initiate_validation()` calls `convert_to_sn_api_block_info(&init)`, which sets `sequencer_address = attacker_address` in `BlockInfo`.
6. The batcher executes every transaction in the block with `sequencer_address = attacker_address`. Every `execute_fee_transfer` call uses `attacker_address` as the ERC-20 transfer recipient.
7. The resulting state diff credits all block fees to `attacker_address`. The `ProposalFin` commitment is computed over this wrong state.
8. Consensus reaches a decision; the block is committed with the wrong fee accounting permanently recorded on-chain. [9](#0-8) [10](#0-9) [11](#0-10)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

    fn execute_fee_transfer(
        state: &mut dyn State,
        tx_context: Arc<TransactionContext>,
        actual_fee: Fee,
    ) -> TransactionExecutionResult<CallInfo> {
        // The least significant 128 bits of the amount transferred.
        let lsb_amount = Felt::from(actual_fee.0);
        // The most significant 128 bits of the amount transferred.
        let msb_amount = Felt::ZERO;

        let TransactionContext { block_context, tx_info } = tx_context.as_ref();
        let storage_address = tx_context.fee_token_address();
        // The fee contains the cost of running this transfer, and the token contract is
        // well known to the sequencer, so there is no need to limit its run.
        let mut remaining_gas_for_fee_transfer =
            block_context.versioned_constants.os_constants.gas_costs.base.default_initial_gas_cost;
        let fee_transfer_call = CallEntryPoint {
            class_hash: None,
            code_address: None,
            entry_point_type: EntryPointType::External,
            entry_point_selector: selector_from_name(constants::TRANSFER_ENTRY_POINT_NAME),
            calldata: calldata![
                *block_context.block_info.sequencer_address.0.key(), // Recipient.
                lsb_amount,
                msb_amount
            ],
            storage_address,
            caller_address: tx_info.sender_address(),
            call_type: CallType::Call,

            initial_gas: remaining_gas_for_fee_transfer,
        };
        let mut context = EntryPointExecutionContext::new_invoke(
            tx_context,
            true,
            SierraGasRevertTracker::new(GasAmount(remaining_gas_for_fee_transfer)),
        );

        Ok(fee_transfer_call
            .execute(state, &mut context, &mut remaining_gas_for_fee_transfer)
            .map_err(|error| Box::new(TransactionFeeError::ExecuteFeeTransferError(error)))?)
    }
