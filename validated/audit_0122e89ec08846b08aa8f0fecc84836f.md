### Title
Sub-Transaction Native Script Validation Bypass via Cross-Level Witness Placement — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxow.hs`)

---

### Summary

In the Dijkstra era's `SUBUTXOW` rule, the per-sub-transaction phase-1 script validation check (`validateFailedBabbageScripts`) only considers scripts present in that sub-transaction's own `scriptsProvided`. The top-level `UTXOW` rule performs an **aggregated** `babbageMissingScripts` check across all levels, which passes as long as each needed script is provided *somewhere* in the batch. An attacker can exploit this asymmetry by placing a native script required by sub-transaction B into sub-transaction A's witness set. The aggregated check passes, but the script is never executed at any level, allowing the sub-transaction to spend a native-script-locked UTxO without the script being validated.

---

### Finding Description

**Vulnerability class**: witness/script validation bypass — analogous to `transferFrom` bypassing `onERC721Received`: a required validation hook is never triggered because the script is placed in the wrong transaction's witness set.
