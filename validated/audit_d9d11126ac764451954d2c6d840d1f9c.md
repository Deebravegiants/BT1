### Title
Missing `whenNotPaused` Modifier on `swapAssetToPremintedRsETH()` Allows Pause Bypass - (File: `contracts/pools/RSETHPoolV2ExternalBridge.sol`)

### Summary
`RSETHPoolV2ExternalBridge` implements a `whenNotPaused` modifier to halt user-facing operations during security incidents. The `deposit()` function is correctly gated. However, `swapAssetToPremintedRsETH()`, which is accessible to any address holding `OPERATOR_ROLE` or `WHITELISTED_USER_ROLE`, is missing the `whenNotPaused` modifier, allowing it to continue operating when the contract is paused.

### Finding Description
The contract defines a custom `paused` boolean and a `whenNotPaused` modifier: [1](#0-0) 

The `deposit()` function is correctly protected: [2](#0-1) 

However, `swapAssetToPremintedRsETH()` — which transfers ETH from the pool to the caller in exchange for rsETH — carries no `whenNotPaused` guard: [3](#0-2) 

Access is granted to any address with `OPERATOR_ROLE` **or** `WHITELISTED_USER_ROLE`: [4](#0-3) 

The function uses `viewSwapAssetToPremintedRsETH()`, which reads from the same oracle that may be the reason the contract was paused: [5](#0-4) 

The same class of missing modifier exists in `RSETHPoolV3.sol`, `RSETHPoolV3ExternalBridge.sol`, and `RSETHPoolV3WithNativeChainBridge.sol` on their respective `swapAssetToPremintedRsETH()` functions, though those restrict access to `OPERATOR_ROLE` only. [6](#0-5) 

### Impact Explanation
When the `PAUSER_ROLE` holder pauses the contract in response to a security incident (e.g., a discovered rate manipulation or oracle anomaly), `deposit()` is correctly blocked. However, any address holding `WHITELISTED_USER_ROLE` can still call `swapAssetToPremintedRsETH()`, which:

1. Reads the (potentially compromised) oracle rate via `viewSwapAssetToPremintedRsETH()`.
2. Pulls rsETH from the caller into the wrapper.
3. Sends ETH from the pool's balance to the caller.

This allows continued ETH extraction from the pool at a potentially incorrect rate during the exact window the pause was intended to protect. Impact: **Medium — Temporary freezing of funds bypass**; escalates to **High/Critical** if the oracle rate is manipulated at the time of the pause.

### Likelihood Explanation
`WHITELISTED_USER_ROLE` is a role that can be granted to multiple external users (not just protocol operators). Any such user who observes the pause transaction can immediately call `swapAssetToPremintedRsETH()` before the incident is resolved. The pause mechanism is the primary emergency safeguard, and its bypass is directly reachable by a class of non-admin users.

### Recommendation
Add `whenNotPaused` to `swapAssetToPremintedRsETH()` in `RSETHPoolV2ExternalBridge.sol` (and the analogous functions in `RSETHPoolV3.sol`, `RSETHPoolV3ExternalBridge.sol`, and `RSETHPoolV3WithNativeChainBridge.sol`):

```solidity
function swapAssetToPremintedRsETH(
    address rsETH,
    uint256 rsETHAmount
)
    external
    nonReentrant
    whenNotPaused                          // <-- add this
    onlyOperatorOrWhitelisted(msg.sender)
{
```

### Proof of Concept

1. The `PAUSER_ROLE` holder calls `pause()` on `RSETHPoolV2ExternalBridge` due to a detected oracle anomaly inflating the rsETH/ETH rate.
2. `deposit()` reverts with `ContractPaused` — new rsETH minting is halted.
3. A whitelisted user (holding `WHITELISTED_USER_ROLE`) calls:
   ```solidity
   swapAssetToPremintedRsETH(rsETHAddress, largeAmount)
   ```
4. The function executes without checking `paused`. It calls `viewSwapAssetToPremintedRsETH(largeAmount)`, which reads the inflated oracle rate and returns an inflated `ethAmount`.
5. The pool transfers `ethAmount` of ETH to the whitelisted user, draining the pool's ETH balance at the manipulated rate.
6. The pause, intended to freeze all pool interactions, has been bypassed for this function. [7](#0-6)

    function swapAssetToPremintedRsETH(
        address rsETH,
        uint256 rsETHAmount
    )
        external
        nonReentrant
        onlyOperatorOrWhitelisted(msg.sender)
    {
        UtilLib.checkNonZeroAddress(rsETH);

        IRsETHTokenWrapper wrapper = IRsETHTokenWrapper(address(wrsETH));

        if (!wrapper.allowedTokens(rsETH)) revert TokenNotAllowedInWrapper();
        if (rsETHAmount == 0) revert InvalidAmount();
        if (rsETHAmount > wrapper.maxAmountToDepositBridgerAsset(rsETH)) revert ExceedsMaxAmountToDepositInWrapper();

        // Get the amount of ETH to transfer to the user for the given amount of rsETH provided
        uint256 ethAmount = viewSwapAssetToPremintedRsETH(rsETHAmount);

        // Transfer rsETH from sender to the wrapper
        IERC20(rsETH).safeTransferFrom(msg.sender, address(wrapper), rsETHAmount);

        // Transfer the ETH from the pool to the sender
        if (getETHBalanceMinusFees() < ethAmount) revert InsufficientETHBalanceForReverseSwap();
        (bool success,) = payable(msg.sender).call{ value: ethAmount }("");
        if (!success) revert TransferFailed();

        emit ReverseSwapOccurred(msg.sender, rsETH, rsETHAmount, ethAmount);
    }
