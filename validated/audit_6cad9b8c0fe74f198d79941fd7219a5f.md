### Title
SNIP-35 `fee_proposal` Bounded-Range Manipulation Shifts `fee_actual` Median, Corrupting `next_l2_gas_price` in Committed Block Headers — (`crates/apollo_consensus_orchestrator/src/dynamic_gas_price/mod.rs`)

---

### Summary

The SNIP-35 fee-proposal mechanism clips each proposer's submitted `fee_proposal_fri` to a ±0.2 ppt (parts-per-thousand) geometric band around the current `fee_actual`. `fee_actual` is the median of the last `window_size` (10) `fee_proposal` values. Because the validator only checks that the proposed value is **within the band** — not that it moves toward the oracle target — a malicious block proposer can consistently submit at the extreme of the allowed range, gradually steering the median in any direction regardless of the STRK/USD oracle. The corrupted `fee_actual` is then used as the `effective_min` floor in `calculate_next_l2_gas_price_for_fin`, causing every subsequent block's `next_l2_gas_price` (stored in `BlockHeaderWithoutHash` and used for all transaction fee calculations) to be wrong.

This is the direct structural analog to the external report: the clip/bound prevents a single large jump but does not prevent sustained within-range manipulation of the time-weighted aggregate.

---

### Finding Description

**Step 1 — Bounds enforcement (the clip).**

`fee_proposal_bounds` computes the allowed range for a proposer's `fee_proposal_fri`:

```
upper = fee_actual * (1000 + 2) / 1000   // +0.2%
lower = fee_actual * 1000 / (1000 + 2)   // −0.2%
``` [1](#0-0) 

**Step 2 — Validator only checks the band, not the direction.**

`is_proposal_init_valid` rejects any `fee_proposal` outside `[lower_bound, upper_bound]` but accepts any value inside, including the extreme endpoints: [2](#0-1) 

There is no check that the proposer is moving toward the oracle's `fee_target`. A proposer whose oracle says "decrease" can submit at the upper bound and pass validation.

**Step 3 — `fee_actual` is the median of the window.**

`compute_fee_actual` takes the median of the last 10 `fee_proposal` values: [3](#0-2) 

A proposer who controls ≥6 of the last 10 blocks and consistently submits at the upper bound will shift the median upward by ~0.2% per controlled block. With `window_size = 10` and `margin_ppt = 2`, the median can be doubled in roughly 350 controlled blocks (~15 minutes at 2.6 s/block).

**Step 4 — Corrupted `fee_actual` becomes the minimum L2 gas price floor.**

`calculate_next_l2_gas_price_for_fin` uses `fee_actual` as `effective_min`:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [4](#0-3) 

When `fee_actual` is artificially inflated, the EIP-1559 mechanism cannot decrease the L2 gas price below it, even when gas usage is far below target.

**Step 5 — Corrupted price is committed to state and used for execution.**

`finalize_decision` calls `update_l2_gas_price` (which calls `calculate_next_l2_gas_price`) and then stores the result in `BlockHeaderWithoutHash.next_l2_gas_price` via `update_state_sync_with_new_block`: [5](#0-4) [6](#0-5) 

This `next_l2_gas_price` is the L2 gas price used for all transaction fee calculations in the following block. It is also forwarded to the cende blob pipeline as `FeeMarketInfo.next_l2_gas_price`. [7](#0-6) 

---

### Impact Explanation

**Corrupted value**: `BlockHeaderWithoutHash.next_l2_gas_price` (and `FeeMarketInfo.next_l2_gas_price` in the cende blob) is set to an artificially inflated (or deflated) value.

**Economic impact**: Every transaction in the next block pays fees computed against this wrong L2 gas price. Upward manipulation causes users to overpay; downward manipulation underprices the network, enabling spam or griefing. The effect compounds across blocks because the corrupted `fee_actual` feeds back into the next window.

This matches the Critical impact category: *"Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."*

---

### Likelihood Explanation

In the current single-sequencer deployment the sequencer controls all proposals, making this trivially triggerable (though it is self-harm in that context). In the multi-validator Tendermint deployment this code is designed for, any validator that wins proposer selection for a majority of the 10-block window can execute the attack. The `fee_proposal_margin_ppt = 2` and `window_size = 10` parameters are hardcoded in versioned constants: [8](#0-7) 

The convergence test confirms the manipulation rate: 75 gwei → 100 gwei (+33%) in ~800 blocks; 100 gwei → 60 gwei (−40%) in ~1,420 blocks. A proposer controlling 6/10 blocks can achieve the same movement in ~half that time. [9](#0-8) 

---

### Recommendation

1. **Direction enforcement**: In `is_proposal_init_valid`, after computing `fee_proposal_bounds`, also compute the validator's own `fee_target` from the oracle and verify that the proposed value moves toward (or stays at) `fee_target`. A proposer whose oracle says "decrease" should not be permitted to submit at the upper bound.

2. **Tighten the margin or widen the window**: Reducing `fee_proposal_margin_ppt` or increasing `window_size` slows the manipulation rate. These are versioned-constants parameters and can be adjusted without a code change.

3. **Median-of-signed-deltas**: Instead of taking the median of raw `fee_proposal` values, take the median of the signed percentage changes each proposer submitted relative to `fee_actual` at their block. This makes it harder to bias the aggregate by consistently submitting at one extreme.

---

### Proof of Concept

```
Window state (height h-10 … h-1): all entries = F (honest fee_actual = F)
fee_proposal_margin_ppt = 2, window_size = 10

Malicious proposer controls blocks h, h+1, …, h+9 (all 10 blocks):

  Block h:   submits fee_proposal = F * 1002/1000  (upper bound, passes is_proposal_init_valid)
  Block h+1: fee_actual = median([F*1002/1000, F, F, F, F, F, F, F, F, F]) = F
             submits fee_proposal = F * 1002/1000  (still upper bound)
  ...
  Block h+5: 6 entries are F*1002/1000, 4 are F
             fee_actual = median = F*1002/1000  (median crosses over)
  Block h+6: submits fee_proposal = (F*1002/1000) * 1002/1000 = F*(1002/1000)^2
  ...

After 350 controlled blocks: fee_actual ≈ 2F
effective_min in calculate_next_base_gas_price = 2F
next_l2_gas_price cannot decrease below 2F regardless of gas usage
All transactions in subsequent blocks pay fees computed at ≥ 2F instead of F
```

The corrupted `next_l2_gas_price` is stored in `BlockHeaderWithoutHash` at `update_state_sync_with_new_block` and propagated to the cende blob at `prepare_blob_for_next_height`, making the wrong price authoritative for the next block's execution context. [10](#0-9) [11](#0-10) [12](#0-11)

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

pub fn compute_fee_proposal(
    fee_target: Option<GasPrice>,
    fee_actual: GasPrice,
    margin_ppt: u128,
) -> GasPrice {
    let Some(fee_target) = fee_target else {
        return fee_actual;
    };
    let (lower, upper) = fee_proposal_bounds(fee_actual, margin_ppt);
    GasPrice(fee_target.0.clamp(lower, upper))
}

pub(crate) fn fee_proposal_bounds(fee_actual: GasPrice, margin_ppt: u128) -> (u128, u128) {
    let denom = U256::from(PPT_DENOMINATOR);
    let scaled = denom + U256::from(margin_ppt);
    let fee_actual_u256 = U256::from(fee_actual.0);
    let upper = u128::try_from(fee_actual_u256 * scaled / denom).unwrap_or(u128::MAX);
    let lower = u128::try_from(fee_actual_u256 * denom / scaled).unwrap_or(0);
    (lower, upper)
}

pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}

async fn test_compute_proposer_fee_proposal_converges_to_oracle_target() {
    // (strk_usd_rate, fee_target, n_blocks_until_convergence_with_buffer).
    // 75 gwei bootstrap -> 100 gwei target at +33% reaches by block ~795.
    // 100 gwei -> 60 gwei at -40% reaches by block ~1410.
    let phases: [(u128, GasPrice, u64); 2] = [
        (30_000_000_000_000_000, GasPrice(100_000_000_000), 800),
        (50_000_000_000_000_000, GasPrice(60_000_000_000), 1420),
    ];

    let (mut deps, _network) = create_test_and_network_deps();
    // Register per-phase strk_to_usd_rate expectations BEFORE setup_default_expectations so they
    // are not shadowed by the catch-all default (mockall matches oldest first).
    let mut seq = mockall::Sequence::new();
    for &(rate, _, n_blocks) in &phases {
        deps.l1_gas_price_provider
            .expect_get_strk_to_usd_rate()
            .times(usize::try_from(n_blocks).unwrap())
            .in_sequence(&mut seq)
            .returning(move |_| Ok(rate));
    }
    deps.setup_default_expectations();
    let mut context = deps.build_context();

    // Bootstrap the window with 75 gwei (the $0.04 target).
    let window_size = VersionedConstants::latest_constants().fee_proposal_window_size;
    for h in 0..window_size {
        context.record_fee_proposal(BlockNumber(h), Some(GasPrice(75_000_000_000)));
    }

    let mut height = window_size;
    for (phase_idx, (_, fee_target, n_blocks)) in phases.into_iter().enumerate() {
        for _ in 0..n_blocks {
            let h = BlockNumber(height);
            let fee_actual = compute_fee_actual(&context.fee_proposals_window, h, window_size)
                .expect("window stays complete across the loop");
            let proposal = context
                .compute_proposer_fee_proposal(Some(fee_actual), 0, TARGET_ATTO_USD_PER_L2_GAS)
                .await;
            context.record_fee_proposal(h, Some(proposal));
            height += 1;
        }
        let final_fee_actual =
            compute_fee_actual(&context.fee_proposals_window, BlockNumber(height), window_size)
                .expect("window stays complete across the loop");
        assert_eq!(
            final_fee_actual, fee_target,
            "phase {phase_idx}: fee_actual did not reach fee_target after {n_blocks} blocks",
        );
    }
