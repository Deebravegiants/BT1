### Title
Single-Participant `vote_abort` Enables Indefinite Keygen Abort, Freezing All Sign-Request Processing — (File: `crates/contract/src/state/key_event.rs`)

---

### Summary

Any single registered participant can call `vote_abort_key_event_instance` to immediately destroy an active keygen or resharing instance with no threshold requirement. A Byzantine participant strictly below the signing threshold can exploit this to abort every new instance the leader starts, trapping the contract in `Initializing` state indefinitely. While in `Initializing`, `domain_registry()` returns an error for every user-facing call, so no `sign`, `request_app_private_key`, or `verify_foreign_transaction` request can be processed — freezing the MPC network's signing capability.

---

### Finding Description

**Root cause — `KeyEvent::vote_abort` requires no threshold:** [1](#0-0) 

```rust
pub fn vote_abort(&mut self, key_event_id: KeyEventId) -> Result<(), Error> {
    let candidate = self.verify_vote(&key_event_id)?;
    if self.instance.as_ref().unwrap().completed.contains(&candidate) {
        return Err(VoteError::VoteAlreadySubmitted.into());
    }
    self.instance = None;   // ← single vote, immediate abort
    Ok(())
}
```

This is critically asymmetric with `vote_success`, which requires **all** participants to vote before the instance completes. [2](#0-1) 

The public contract method exposes this to any attested participant: [3](#0-2) 

**Why `Initializing` state freezes all signing:**

`domain_registry()` returns an error for `Initializing`, causing `env::panic_str` inside `check_request_preconditions` for every `sign` / `request_app_private_key` / `verify_foreign_transaction` call: [4](#0-3) [5](#0-4) 

**`Initializing` is entered from `Running` via `vote_add_domains`:** [6](#0-5) 

Once the contract enters `Initializing`, the `key_event_id` of the active instance is public contract state. The Byzantine participant:

1. Waits for the leader to call `start_keygen_instance`.
2. Immediately calls `vote_abort_key_event_instance` with the observed `key_event_id`.
3. The instance is set to `None`; the leader must start a new attempt.
4. Repeat indefinitely.

The only escape is `vote_cancel_keygen`, which requires **threshold** votes and permanently deletes the new domain: [7](#0-6) 

The same single-abort path exists for `Resharing` via `ResharingContractState::vote_abort`: [8](#0-7) 

---

### Impact Explanation

While the contract is stuck in `Initializing`, **no sign requests of any kind can be submitted or fulfilled**. A single Byzantine participant below the signing threshold can sustain this indefinitely, freezing the MPC network's ability to produce threshold signatures. Any bridge flow, omnibridge inbound, or chain-abstraction operation that depends on `sign` or `verify_foreign_transaction` is halted. This breaks the core threshold-security invariant: the system must remain live when fewer than `threshold` participants are Byzantine.

Impact class: **Medium** — contract execution-flow manipulation that breaks production safety/liveness invariants without requiring network-level DoS or threshold collusion.

---

### Likelihood Explanation

- Requires one registered, attested participant (not threshold-or-above collusion).
- The `key_event_id` is fully public; no privileged information is needed.
- The attack is cheap: one on-chain call per leader attempt.
- The attacker can sustain it indefinitely; the leader has no way to exclude the aborting participant from the current `Initializing` epoch.
- Motivation: a participant being rotated out, a competitor, or a participant whose TEE is compromised.

---

### Recommendation

1. **Require a threshold of abort votes** before destroying the instance (mirror `vote_cancel_keygen`'s quorum logic).
2. **Track per-participant abort history** and exclude repeat aborters from the current epoch's instances.
3. **Allow the leader to start a new instance without waiting** for the aborting participant, effectively demoting them for the current attempt.

---

### Proof of Concept

```
1. Contract is Running with existing domains.
2. Governance reaches quorum on vote_add_domains([new_domain])
   → contract transitions to Initializing.
3. All sign() calls now panic: domain_registry() → Err(ProtocolStateNotRunning).
4. Leader (lowest participant ID) calls start_keygen_instance(key_event_id_0).
5. Byzantine participant (any single participant, not the leader) reads
   key_event_id_0 from contract state and calls
   vote_abort_key_event_instance(key_event_id_0).
   → KeyEvent::vote_abort sets self.instance = None immediately.
6. Leader calls start_keygen_instance(key_event_id_1).
7. Byzantine participant calls vote_abort_key_event_instance(key_event_id_1).
8. Steps 6–7 repeat indefinitely.
9. Contract never reaches Running; no sign requests are processed.
10. Only escape: threshold participants coordinate vote_cancel_keygen,
    permanently deleting the new domain and returning to Running
    with only the pre-existing keys.
```

            .as_mut()
            .unwrap()
            .vote_success(candidate, public_key)?
        {
            VoteSuccessResult::Voted(count) => {
                if count == self.parameters.participants().len() {
                    Ok(true)
                } else {
                    Ok(false)
                }
            }
            VoteSuccessResult::PublicKeyDisagreement => {
                log!("Public key disagreement; aborting key event instance.");
                self.instance = None;
                Ok(false)
            }
        }
    }

    pub fn vote_cancel(
        &mut self,
        next_domain_id: u64,
    ) -> Result<Option<RunningContractState>, Error> {
        if next_domain_id != self.domains.next_domain_id() {
            return Err(InvalidParameters::NextDomainIdMismatch.into());
        }
        let participant = AuthenticatedParticipantId::new(
            self.generating_key.proposed_parameters().participants(),
        )?;
        let required_threshold = self
            .generating_key
            .proposed_parameters()
            .threshold()
            .value() as usize;
        if self.cancel_votes.insert(participant) && self.cancel_votes.len() >= required_threshold {
            let mut domains = self.domains.clone();
            domains.retain_domains(self.generated_keys.len());
            return Ok(Some(RunningContractState::new(
                domains,
                Keyset::new(self.epoch_id, self.generated_keys.clone()),
                self.generating_key.proposed_parameters().clone(),
                AddDomainsVotes::default(),
            )));
        }
        Ok(None)
    }
