### Title
Caller-Supplied `Attestation::Mock(MockAttestation::Valid)` Unconditionally Bypasses TEE Authenticity Verification in `submit_participant_info` - (File: crates/contract/src/lib.rs)

---

### Summary

The `submit_participant_info` function accepts a caller-supplied `Attestation` parameter. The `Attestation::Mock(MockAttestation::Valid)` variant always passes verification unconditionally — no cryptographic proof, no hardware quote, no image-hash check. Any NEAR account (including an existing participant acting as a Byzantine node below the signing threshold) can submit this self-certifying attestation and be stored in `tee_state.stored_attestations` as a fully "attested" participant. The contract's `is_caller_an_attested_participant` guard does not distinguish Mock from Dstack attestations, so the node can subsequently call `respond()` and other privileged node-only methods while operating entirely outside a TEE.

---

### Finding Description

**Root cause — `submit_participant_info` accepts any `Attestation` variant without rejecting `Mock`:** [1](#0-0) 

The function converts the caller-supplied DTO and immediately passes it to `tee_state.add_participant`. There is no guard that rejects `Attestation::Mock`.

**`MockAttestation::Valid` is unconditionally self-certifying:** [2](#0-1) 

`MockAttestation::Valid => Ok(())` — no report-data binding, no image-hash check, no timestamp, no hardware quote. The caller-supplied object certifies itself, exactly as the oracle wrapper in the reference report returns `msg.sender`.

**`add_participant` stores the result without distinguishing Mock from Dstack:** [3](#0-2) 

After `verify_locally` returns `Ok`, the `VerifiedAttestation::Mock(MockAttestation::Valid)` value is inserted into `stored_attestations` keyed by the caller's TLS public key.

**`is_caller_an_attested_participant` does not distinguish Mock from Dstack:** [4](#0-3) 

The guard only checks that a stored entry exists, that `account_id` matches, and that `account_public_key` matches `env::signer_account_pk()`. A participant who submitted `Attestation::Mock(MockAttestation::Valid)` via `submit_participant_info` has their real NEAR account key stored as `account_public_key` and passes this check identically to a genuine Dstack-attested node.

**`re_verify` also passes unconditionally for `MockAttestation::Valid`:** [5](#0-4) 

`verify_tee()` / `reverify_and_cleanup_participants` will never evict a participant whose stored attestation is `VerifiedAttestation::Mock(MockAttestation::Valid)`, so the bypass is permanent until the participant is explicitly removed by governance.

**An existing participant can downgrade from Dstack to Mock at any time:** [6](#0-5) 

The only ownership guard prevents a *different* account from overwriting a TLS key. The same account can freely overwrite its own Dstack attestation with `Attestation::Mock(MockAttestation::Valid)`.

---

### Impact Explanation

**Impact: Medium** — participant-state manipulation that breaks a production safety invariant without requiring threshold collusion.

The production safety invariant is that every active participant must be running approved software inside a genuine Intel TDX enclave, proven by a hardware-backed DCAP quote. A single Byzantine participant (strictly below the signing threshold) can:

1. Call `submit_participant_info(Attestation::Mock(MockAttestation::Valid), their_tls_key)` to replace their real Dstack attestation.
2. Continue to pass `is_caller_an_attested_participant` and call `respond()`, `respond_verify_foreign_tx()`, and all other node-only methods.
3. Never be evicted by `verify_tee()` or `clean_invalid_attestations()`.
4. Operate with their MPC key share entirely outside a TEE — the key share is no longer hardware-protected.

The `accept_requests` flag remains `true` because `reverify_and_cleanup_participants` sees all participants as valid. The contract's TEE-enforcement layer is silently neutralised for that participant.

This matches the Medium allowed impact: *"participant-state or contract execution-flow manipulation that breaks production safety/accounting invariants."*

---

### Likelihood Explanation

**High.** The attack requires only a single NEAR transaction from an existing participant account. No threshold collusion, no privileged role, no leaked key, and no network-level capability is needed. The `Attestation` enum and `MockAttestation::Valid` variant are part of the public ABI (exposed in the contract's JSON schema). [7](#0-6) 

Any participant who wishes to stop running inside a TEE (e.g., to reduce operational cost, or as a malicious insider) can do so unilaterally and invisibly.

---

### Recommendation

1. **Reject `Attestation::Mock` in `submit_participant_info` on mainnet.** Add a compile-time feature flag (e.g., `cfg(feature = "mock-attestation")`) or a runtime contract configuration field that causes `submit_participant_info` to return an error when the submitted attestation is `Attestation::Mock`.
2. **Add a `is_dstack` predicate** to `VerifiedAttestation` and enforce it in `is_caller_an_attested_participant` for production builds.
3. **Track the open TODO** (`#1087`) that acknowledges the placeholder Mock attestations seeded during initialization; resolve it so that the initialization path also uses real attestations.

    pub fn submit_participant_info(
        &mut self,
        proposed_participant_attestation: dtos::Attestation,
        tls_public_key: dtos::Ed25519PublicKey,
    ) -> Result<(), Error> {
        let proposed_participant_attestation =
            proposed_participant_attestation.try_into_contract_type()?;

        let account_key = env::signer_account_pk();
        let account_id = Self::assert_caller_is_signer();

        log!(
            "submit_participant_info: signer={}, proposed_participant_attestation={:?}, account_key={:?}",
            account_id,
            proposed_participant_attestation,
            account_key
        );

        // Save the initial storage usage to know how much to charge the proposer for the storage
        // used
        let initial_storage = env::storage_usage();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        // The node always signs submissions with an Ed25519 key
        // (`near_signer_key`), so the signer key here is Ed25519 in practice.
        // Reject non-Ed25519 signer keys rather than silently storing a value
        // we could never match against in `is_caller_an_attested_participant`.
        let account_public_key = dtos::Ed25519PublicKey::try_from(&account_key).map_err(|_| {
            InvalidParameters::InvalidTeeRemoteAttestation {
                reason: "signer account key must be Ed25519".to_string(),
            }
        })?;

        // Add the participant information to the contract state
        let attestation_insertion_result = self
            .tee_state
            .add_participant(
                NodeId {
                    account_id: account_id.clone(),
                    tls_public_key,
                    account_public_key,
                },
                proposed_participant_attestation,
                tee_upgrade_deadline_duration,
            )

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
