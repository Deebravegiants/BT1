Audit Report

## Title
Attacker Can Exhaust the Global `dailyMintLimit` to Temporarily Block All Deposits - (File: contracts/pools/RSETHPoolV3.sol)

## Summary
The `limitDailyMint` modifier in `RSETHPoolV3` and its sibling pool contracts maintains a single shared `dailyMintAmount` counter with no per-user sub-limit. Any unprivileged depositor can consume the entire daily quota in a single transaction, blocking all other users from depositing for up to 24 hours. The attacker's only cost is the `feeBps` fee, and they receive `wrsETH` in return.

## Finding Description
The `limitDailyMint` modifier in `RSETHPoolV3.sol` accumulates a global `dailyMintAmount` counter shared across all depositors. When a deposit is made, the modifier computes the rsETH equivalent of the deposit, checks that `dailyMintAmount + rsETHAmount <= dailyMintLimit`, and then unconditionally increments `dailyMintAmount`:

```solidity
// contracts/pools/RSETHPoolV3.sol lines 119-123
if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
    revert DailyMintLimitExceeded();
}
dailyMintAmount += rsETHAmount;
```

There is no per-user sub-limit, no whitelist bypass, and no mechanism to reclaim consumed quota within the same day. An attacker calls `deposit()` with an amount large enough to push `dailyMintAmount` to `dailyMintLimit`. After this single transaction, every subsequent `deposit()` call by any user reverts with `DailyMintLimitExceeded` until `getCurrentDay()` increments (up to 24 hours later).

The `deposit()` function in `RSETHPoolV3` is fully permissionless — it carries only `nonReentrant`, `whenNotPaused`, and `limitDailyMint` guards, none of which restrict who can call it:

```solidity
// contracts/pools/RSETHPoolV3.sol lines 246-251
function deposit(string memory referralId)
    external
    payable
    nonReentrant
    whenNotPaused
    limitDailyMint(msg.value, ETH_IDENTIFIER)
```

The same pattern is confirmed verbatim in:
- `contracts/pools/RSETHPoolV3ExternalBridge.sol` lines 130–158
- `contracts/pools/RSETHPoolV3WithNativeChainBridge.sol` lines 108–137
- `contracts/pools/RSETHPoolV2.sol` lines 72–93
- `contracts/pools/RSETHPoolV2ExternalBridge.sol` lines 104–126

## Impact Explanation
All L2 pool deposits are blocked for up to 24 hours per attack. Users who need to enter a position are denied access to the protocol for a bounded but meaningful period. The attacker does not lose funds — they receive `wrsETH` proportional to their deposit. This constitutes **temporary freezing of funds** (access to the deposit function is denied for all users for up to 24 hours), which matches the allowed impact class **Medium — Temporary freezing of funds**.

## Likelihood Explanation
The attack requires no special role, flash loan, or oracle manipulation. Any actor holding ETH (or a supported LST) equal to the `dailyMintLimit` value can execute it permissionlessly in a single transaction. The attacker pays only the `feeBps` fee (capped at 10% by `setFeeBps` in `RSETHPoolV3`, and 10% in `RSETHPoolV2`/`RSETHPoolV2ExternalBridge`, meaning the real cost is at most 10% of the deposited amount, and in practice far less given typical fee settings of 0.1–0.5%). The attack is repeatable every 24 hours. The `dailyMintLimit` is a finite, known on-chain value, making the required capital calculable in advance.

## Recommendation
Introduce a per-address deposit sub-limit so that no single depositor can consume the entire daily quota in one transaction. Alternatively, enforce a maximum deposit size per transaction (e.g., a fraction of `dailyMintLimit`). A simpler mitigation is to remove the L2 daily mint limit entirely and rely solely on the L1 deposit limit (`lrtConfig.depositLimitByAsset`) as the authoritative cap, since the L2 pool's daily limit provides only a weak rate-limit that is trivially bypassable by any sufficiently capitalized actor.

## Proof of Concept
1. `dailyMintLimit` is set to `X` rsETH-equivalent (e.g., 100 ETH worth).
2. Attacker calls `RSETHPoolV3.deposit{value: Y}("")` where `Y` is chosen so that `viewSwapRsETHAmountAndFee(Y).rsETHAmount == X`.
3. Inside `limitDailyMint`:
   - If `currentDay > lastMintDay`: `dailyMintAmount` resets to 0.
   - `0 + X <= X` → check passes.
   - `dailyMintAmount = X`.
4. Attacker receives `X` wrsETH (minus fee).
5. Any subsequent call by a legitimate user with any non-zero amount reverts: `dailyMintAmount + rsETHAmount > dailyMintLimit`.
6. State persists until `getCurrentDay()` increments (up to 24 hours later).

Foundry test plan:
```solidity
function test_exhaustDailyMintLimit() public {
    // Setup: deploy RSETHPoolV3 with dailyMintLimit = 100 ether equivalent
    // Warp past startTimestamp
    vm.warp(startTimestamp + 1);
    // Attacker deposits exactly dailyMintLimit worth
    vm.prank(attacker);
    pool.deposit{value: attackAmount}("");
    // Verify dailyMintAmount == dailyMintLimit
    assertEq(pool.dailyMintAmount(), pool.dailyMintLimit());
    // Legitimate user deposit reverts

    vm.expectRevert(RSETHPoolV3.DailyMintLimitExceeded.selector);
    pool.deposit{value: 1 ether}("");
}

### Citations

**File:** contracts/pools/RSETHPoolV3.sol (L96-125)
```text
    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```

**File:** contracts/pools/RSETHPoolV3.sol (L246-265)
```text
    function deposit(string memory referralId)
        external
        payable
        nonReentrant
        whenNotPaused
        limitDailyMint(msg.value, ETH_IDENTIFIER)
    {
        if (!isEthDepositEnabled) revert EthDepositDisabled();
        uint256 amount = msg.value;

        if (amount == 0) revert InvalidAmount();

        (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);

        feeEarnedInETH += fee;

        wrsETH.mint(msg.sender, rsETHAmount);

        emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
    }
```
