Audit Report

## Title
Unbounded Gas Consumption in `updateRSETHPrice()` via N_assets × N_NDCs External Calls to `getQueuedWithdrawals` — (File: `contracts/LRTOracle.sol`, `contracts/NodeDelegator.sol`, `contracts/LRTDepositPool.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is a public function with no access control. Its call chain causes `NodeDelegator.getAssetUnstaking()` to be invoked once per supported asset per NDC, each time making a full external call to EigenLayer's `getQueuedWithdrawals` and iterating over all queued withdrawals × strategies for that NDC. The developers' own gas bound (capped at 80 total withdrawals) understates the true cost by a factor of N_assets, because the same withdrawal set is re-traversed once per asset. With no hard cap on supported assets and an admin-adjustable NDC limit with no ceiling, gas consumption grows unboundedly.

## Finding Description

The call chain is confirmed in the code:

1. `updateRSETHPrice()` is `public whenNotPaused` with no role restriction.
2. It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which loops over every supported asset and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each.
3. `getTotalAssetDeposits` calls `getAssetDistributionData(asset)`. For non-ETH assets this loops over all NDCs and calls `INodeDelegator(ndc).getAssetUnstaking(asset)` per NDC. For ETH it calls `getETHDistributionData()`, which also calls `getAssetUnstaking(ETH_TOKEN)` per NDC.
4. `getAssetUnstaking(asset)` in `NodeDelegator` calls `_getDelegationManager().getQueuedWithdrawals(address(this))` — a full external call returning all queued withdrawals for that NDC — then iterates over every withdrawal × every strategy, calling `strategy.sharesToUnderlyingView(...)` for each matching strategy.

Total external `getQueuedWithdrawals` calls per price update = **N_assets × N_NDCs**.  
Total strategy iterations = **N_assets × total_withdrawals_across_all_NDCs × avg_strategies_per_withdrawal**.

The `maxUncompletedWithdrawalCount` cap of 80 bounds the total withdrawal count globally, but does not bound the N_assets multiplier. The developers' comment states *"120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price"* — this bound counts total withdrawals, not the N_assets × total_withdrawals product. With 5 supported assets, 10 NDCs, 80 total withdrawals (8 per NDC), and 2 strategies per withdrawal: 50 external `getQueuedWithdrawals` calls + 800 `sharesToUnderlyingView` calls occur per price update. If `maxNodeDelegatorLimit` is raised (no hard ceiling exists in `updateMaxNodeDelegatorLimit`) or more assets are added (no hard cap), the cost grows further. The function is callable by any unprivileged address at any time.

## Impact Explanation

If `updateRSETHPrice()` approaches or exceeds the block gas limit, the stored `rsETHPrice` becomes permanently stale. All deposits (`depositETH`, `depositAsset`) and withdrawals continue using the stale price, causing incorrect rsETH minting and redemption amounts. Additionally, the auto-pause safety mechanism in `_updateRsETHPrice()` — which pauses `LRTDepositPool` and `LRTWithdrawalManager` on excessive price drops — can never trigger, removing the protocol's primary downside protection. This matches the allowed impact: **Medium — Unbounded gas consumption**.

## Likelihood Explanation

No attacker action is required. The gas cost grows automatically as the protocol adds supported assets and NDCs through normal operational scaling. Any unprivileged caller can invoke `updateRSETHPrice()` at any time. The developers' own comment confirms awareness of gas sensitivity but uses an incomplete bound. The condition is reachable through legitimate protocol growth without any adversarial behavior.

## Recommendation

1. Cache the result of `getQueuedWithdrawals(ndc)` per NDC and reuse it across all asset iterations in a single price update, rather than calling it once per asset per NDC.
2. Introduce a `totalAssetUnstaking` mapping in `LRTUnstakingVault` updated incrementally on `initiateUnstaking` / `completeUnstaking`, eliminating the need to traverse EigenLayer queued withdrawals at price-update time.
3. Enforce a hard cap on the number of supported assets, or restructure `_getTotalEthInProtocol` to avoid the N_assets × N_NDCs traversal.
4. Enforce a hard ceiling on `maxNodeDelegatorLimit` in `updateMaxNodeDelegatorLimit`.

**File:** contracts/NodeDelegator.sol (L405-427)
```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
