### Title
Cross-Chain Replay of `DelegateAction` (Meta-Transaction) Signatures — (`File: core/primitives/src/action/delegate.rs`, `core/primitives/src/signable_message.rs`)

### Summary
`DelegateAction` (NEP-366 meta-transaction) signatures do not commit to any chain-specific value. The signed payload contains only `sender_id`, `receiver_id`, `actions`, `nonce`, `max_block_height`, and `public_key` — no chain ID, no genesis hash, no network identifier. A `SignedDelegateAction` created and broadcast on NEAR mainnet is cryptographically valid on NEAR testnet (or any NEAR fork) and can be submitted there by any unprivileged relayer, executing the inner actions against the victim's account on the unintended chain.

### Finding Description

`DelegateAction.get_nep461_hash()` computes the signed digest as:

```
SHA-256( borsh( MessageDiscriminant(NEP_366) || DelegateAction ) )
``` [1](#0-0) 

`SignableMessage` contains only a fixed NEP discriminant and the message body — no chain ID, no genesis hash, no network-specific value: [2](#0-1) [3](#0-2) 

The `DelegateAction` struct itself carries no chain-binding field: [4](#0-3) 

By contrast, a regular `Transaction` (V0/V1) includes `block_hash`, which is chain-specific — a mainnet block hash will not satisfy the `transaction_validity_period` check on testnet. `DelegateAction` uses only `max_block_height`, a plain integer that is equally valid on any chain at the same height: [5](#0-4) 

At execution time, `apply_delegate_action` checks: signature validity, `max_block_height`, sender match, and nonce — but never any chain identity: [6](#0-5) 

`validate_delegate_action_key` similarly checks only nonce ordering and access-key permissions, with no chain-binding check: [7](#0-6) 

### Impact Explanation

An attacker who obtains a `SignedDelegateAction` intended for mainnet (e.g., by acting as a relayer, intercepting off-chain relay traffic, or observing on-chain data) can wrap it in a fresh outer `Transaction` and submit it to testnet (or any NEAR fork) as a relayer. If the victim has the same account name and key on both networks — extremely common for developers and power users — the runtime will:

1. Verify the signature: **passes** (same key, same payload hash, no chain binding).
2. Check `max_block_height`: **passes** (block heights on testnet are comparable integers).
3. Check nonce: **passes** if the nonce has not yet been used on the target chain.
4. Execute the inner actions (`Transfer`, `AddKey`, `DeleteKey`, `FunctionCall`, etc.) against the victim's account on the unintended chain.

Corrupted protocol values: victim's access-key nonce on the target chain, victim's token balance, victim's access-key set (if `AddKey`/`DeleteKey` actions are inside the delegate), and any contract state touched by inner `FunctionCall` actions.

### Likelihood Explanation

- NEAR mainnet and testnet are both live networks. Many accounts share the same name and key material across both.
- `SignedDelegateAction` objects are transmitted off-chain to relayers; any party acting as a relayer on the target chain can submit the replayed action.
- No special privilege is required: submitting a transaction via public RPC is the only capability needed.
- The `DelegateActionV2` / `VersionedSignedDelegateAction` path (NEP-611) has the same missing chain binding: [8](#0-7) 

### Recommendation

Include the chain's genesis hash (or a dedicated chain ID) in the signed payload. The simplest fix is to add a `chain_id: CryptoHash` field to `DelegateAction` and `DelegateActionV2`, populated from the genesis block hash (already available in `ApplyState`), and verify it matches during `apply_delegate_action`. Alternatively, incorporate the genesis hash into the `SignableMessage` discriminant or as a mandatory prefix before the borsh-serialized body, mirroring how EIP-712 domain separators work.

### Proof of Concept

1. Alice signs a `DelegateAction` on NEAR mainnet to transfer 10 NEAR to Bob:
   - `sender_id = "alice.near"`, `receiver_id = "bob.near"`, `actions = [Transfer(10 NEAR)]`, `nonce = 5`, `max_block_height = 200_000_000`
   - Signed hash = `SHA-256(borsh(0x_NEP366_discriminant || DelegateAction_body))` — no chain ID.

2. Attacker (acting as relayer on testnet) wraps the same `SignedDelegateAction` in a testnet outer transaction:
   ```
   SignedTransaction {
     transaction: Transaction::V0 { signer_id: attacker, receiver_id: "alice.near",
                                    block_hash: <testnet block hash>, actions: [Delegate(signed_da)] },
     signature: <attacker's testnet key signature>
   }
   ```

3. Testnet runtime calls `apply_delegate_action`:
   - `signed_delegate_action.verify()` → **true** (same ed25519 key, same hash, no chain binding).
   - `block_height <= max_block_height` → **true**.
   - Nonce check → **passes** (nonce 5 not yet used on testnet).
   - Inner `Transfer(10 NEAR)` executes, draining Alice's testnet balance. [9](#0-8)

pub(crate) fn apply_delegate_action(
    state_update: &mut TrieUpdate,
    apply_state: &ApplyState,
    action_receipt: &VersionedActionReceipt,
    sender_id: &AccountId,
    signed_delegate_action: VersionedSignedDelegateActionRef<'_>,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    if !signed_delegate_action.verify() {
        result.result = Err(ActionErrorKind::DelegateActionInvalidSignature.into());
        return Ok(());
    }
    let delegate_action = signed_delegate_action.delegate_action();
    if apply_state.block_height > delegate_action.max_block_height() {
        result.result = Err(ActionErrorKind::DelegateActionExpired.into());
        return Ok(());
    }
    if delegate_action.sender_id().as_str() != sender_id.as_str() {
        result.result = Err(ActionErrorKind::DelegateActionSenderDoesNotMatchTxReceiver {
            sender_id: delegate_action.sender_id().clone(),
            receiver_id: sender_id.clone(),
        }
        .into());
        return Ok(());
    }

    validate_delegate_action_key(state_update, apply_state, delegate_action, result)?;
    if result.result.is_err() {
        // Validation failed. Need to return Ok() because this is not a runtime error.
        // "result.result" will be return to the User as the action execution result.
        return Ok(());
    }

fn validate_delegate_action_key(
    state_update: &mut TrieUpdate,
    apply_state: &ApplyState,
    delegate_action: VersionedDelegateActionRef<'_>,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let sender_id = delegate_action.sender_id();
    let public_key = delegate_action.public_key();
    // 'sender_id' account existence must be checked by a caller
    let mut access_key = match get_access_key(state_update, sender_id, public_key)? {
        Some(access_key) => access_key,
        None => {
            result.result = Err(ActionErrorKind::DelegateActionAccessKeyError(
                InvalidAccessKeyError::AccessKeyNotFound {
                    account_id: sender_id.clone(),
                    public_key: public_key.clone().into(),
                },
            )
            .into());
            return Ok(());
        }
    };

    // A plain nonce advances the single access_key.nonce and forbids gas keys;
    // a gas key nonce advances one of the gas key's nonces selected by
    // nonce_index.
    let delegate_nonce = delegate_action.nonce();
    let (current_nonce, nonce_update) = match delegate_nonce {
        TransactionNonce::Nonce { .. } => {
            if access_key.gas_key_info().is_some() {
                result.result = Err(ActionErrorKind::DelegateActionAccessKeyError(
                    InvalidAccessKeyError::DelegateActionRequiresNonGasKey,
                )
                .into());
                return Ok(());
            }
            (access_key.nonce, DelegateNonceUpdate::AccessKey)
        }
        TransactionNonce::GasKeyNonce { nonce_index, .. } => {
            let Some(gas_key_info) = access_key.gas_key_info() else {
                result.result = Err(ActionErrorKind::DelegateActionAccessKeyError(
                    InvalidAccessKeyError::DelegateActionRequiresGasKey,
                )
                .into());
                return Ok(());
            };
            if nonce_index >= gas_key_info.num_nonces {
                result.result = Err(ActionErrorKind::DelegateActionInvalidNonceIndex {
                    nonce_index,
                    num_nonces: gas_key_info.num_nonces,
                }
                .into());
                return Ok(());
            }
            // The index is range-checked above and gas keys initialize every
            // nonce row at creation, so a missing row is inconsistent state.
            let current_nonce =
                get_gas_key_nonce(state_update, sender_id, public_key, nonce_index)?.ok_or_else(
                    || {
                        StorageError::StorageInconsistentState(format!(
                            "gas key nonce row missing for {} {} at in-range index {nonce_index} (num_nonces {})",
                            sender_id, public_key, gas_key_info.num_nonces,
                        ))
                    },
                )?;
            (current_nonce, DelegateNonceUpdate::GasKey { nonce_index })
        }
    };

    if delegate_nonce.nonce() <= current_nonce {
        result.result = Err(ActionErrorKind::DelegateActionInvalidNonce {
            delegate_nonce: delegate_nonce.nonce(),
            ak_nonce: current_nonce,
        }
        .into());
        return Ok(());
    }
