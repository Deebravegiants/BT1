### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Redirect All Block Fee Revenue to Arbitrary Address — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, `crates/apollo_consensus_orchestrator/src/utils.rs`)

### Summary

`is_proposal_init_valid` validates many fields of a received `ProposalInit` (height, timestamp, L1/L2 gas prices, `starknet_version`, `version_constant_commitment`, `l1_da_mode`, `fee_proposal_fri`) but never validates the `builder` field. `convert_to_sn_api_block_info` then maps `init.builder` directly to `sequencer_address` in the `BlockInfo` passed to the blockifier. Because every validator node uses the proposer-supplied `builder` for execution, both sides compute the same `ProposalCommitment` over the wrong sequencer address, the commitment check passes, and the corrupted block is committed to state.

### Finding Description

`ProposalInit` carries a `builder` field described as "Address of the one who builds/sequences the block." [1](#0-0) 

`convert_to_sn_api_block_info` maps this field directly to `sequencer_address` in the `starknet_api::block::BlockInfo` that is forwarded to the batcher for both proposal building and validation: [2](#0-1) 

`is_proposal_init_valid` checks timestamp bounds, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, and `fee_proposal_fri`. It never checks `builder`: [3](#0-2) 

`ProposalInitValidation` — the struct that carries the validator's local reference values — has no `builder` or `expected_builder` field, so there is no local value to compare against: [4](#0-3) 

The `sequencer_address` derived from `builder` is used in two critical ways by the blockifier:

1. **Fee transfer recipient** — every transaction's fee is transferred to `block_context.block_info.sequencer_address`: [5](#0-4) 

2. **`get_sequencer_address` syscall** — contracts can read this value at runtime: [6](#0-5) 

Because every validator node calls `convert_to_sn_api_block_info(init)` with the same proposer-supplied `init.builder`, both proposer and validator execute the block with the identical wrong `sequencer_address`. The final `ProposalCommitment` comparison in `validate_proposal` therefore passes: [7](#0-6) 

The corrupted block — with fees credited to the attacker-controlled address and the wrong `sequencer_address` in the block header — is committed to storage.

### Impact Explanation

A scheduled proposer sets `ProposalInit.builder` to an address they control. All transaction fees in the block are transferred to that address instead of the legitimate sequencer. The committed block header contains the wrong `sequencer_address`, producing a wrong state root and wrong block hash. Contracts that call `get_sequencer_address()` during that block receive the attacker's address. The economic impact is the total fee revenue of the block; the state impact is a permanently incorrect sequencer balance and block header.

### Likelihood Explanation

Any node that is legitimately scheduled as proposer for a height/round can trigger this. The consensus manager verifies `init.proposer` matches the expected proposer for that slot, but does not constrain `init.builder`. No additional capability beyond normal proposer rotation is required. [8](#0-7) 

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`. The validator should compare `init_proposed.builder` against its locally configured sequencer/builder address (or a committee-derived expected value) and reject any proposal where they differ, analogous to how `l2_gas_price_fri` and `l1_da_mode` are enforced against local configuration.

### Proof of Concept

1. A node is selected as proposer for height H, round R (verified by the consensus manager via `get_proposer_for_height`).
2. In `initiate_build`, the node constructs `ProposalInit` with `builder: attacker_address` instead of its own sequencer address.
3. The `ProposalInit` is broadcast to all validators.
4. Each validator calls `is_proposal_init_valid` — all checked fields pass; `builder` is never inspected.
5. Each validator calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: attacker_address, ... }`.
6. The batcher executes all transactions with `sequencer_address = attacker_address`; every fee transfer credits `attacker_address`.
7. Both proposer and validators compute the same `ProposalCommitment` (both used `attacker_address`); the fin-mismatch check passes.
8. `decision_reached` commits the block; the state diff records fee token balances credited to `attacker_address`, and the block header stores `sequencer = attacker_address`. [9](#0-8) [10](#0-9)

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
