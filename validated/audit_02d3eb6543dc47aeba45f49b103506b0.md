### Title
Hardcoded Testing Private Key Used in Production Consensus Signing — (`File: crates/apollo_signature_manager/src/lib.rs`)

### Summary

The production `create_signature_manager()` function instantiates `SignatureManager` using `LocalKeyStore::new_for_testing()`, which contains a hardcoded, publicly visible ECDSA private key. This key is used to sign consensus precommit votes (`sign_precommit_vote`) and peer identity challenges (`sign_identification`). Because the private key is embedded in source code, any party can forge valid consensus signatures for arbitrary block hashes.

### Finding Description

In `crates/apollo_signature_manager/src/lib.rs`, the exported production factory function `create_signature_manager()` calls `SignatureManager::new()`, which in turn calls `LocalKeyStore::new_for_testing()`: [1](#0-0) 

`LocalKeyStore::new_for_testing()` contains a hardcoded private key literal: [2](#0-1) 

This key (`0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133`) is publicly visible in the repository. The `SignatureManager::sign()` method retrieves this key via `self.keystore.get_key()` and uses it directly with `ecdsa_sign` to produce consensus precommit vote signatures: [3](#0-2) 

The `sign_precommit_vote` path signs a `BlockHash` for Tendermint consensus: [4](#0-3) 

The `config_secrets_schema.json` lists P2P network keys as secrets but does **not** include the consensus signing key: [5](#0-4) 

A TODO comment in `lib.rs` explicitly acknowledges the production key management is unresolved: [1](#0-0) 

### Impact Explanation

Because all sequencer nodes instantiated via `create_signature_manager()` share the same hardcoded private key, and that key is publicly readable from source code:

1. An attacker can compute valid ECDSA signatures over any `BlockHash` using the known private key.
2. These forged signatures are indistinguishable from legitimate precommit votes.
3. If the consensus layer accepts precommit votes signed by this key as representing a quorum, an attacker can drive consensus to commit an attacker-chosen block hash.
4. Peer identity challenges (`sign_identification`) are equally forgeable, allowing impersonation of any validator peer.

This matches the impact: **High — Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload**, and escalates toward **Critical** if the consensus layer uses these signatures to finalize block commitments.

### Likelihood Explanation

The private key is in plain text in a public repository. No exploit tooling is required — the attacker reads the constant, constructs an ECDSA signature over a target block hash, and injects it into the consensus message flow. The `create_signature_manager()` function is the sole production constructor, so every deployed node is affected.

### Recommendation

- **Short term**: Remove `LocalKeyStore::new_for_testing()` from the production `create_signature_manager()` path. Gate it behind `#[cfg(test)]` or a `testing` feature flag. Add a compile-time or runtime assertion that prevents the hardcoded key from being used outside tests.
- **Medium term**: Implement a production `KeyStore` that loads the private key from a secrets management backend (e.g., GCP Secret Manager, AWS Secrets Manager, HashiCorp Vault) at startup, and add `"signature_manager_config.private_key"` (or equivalent) to `config_secrets_schema.json`.
- **Long term**: Migrate to a dedicated remote signing service (HSM or KMS) so the raw private key material never resides in the sequencer process, consistent with the external report's long-term recommendation.

### Proof of Concept

```
# 1. Read the hardcoded private key from source:
PRIVATE_KEY = 0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133

# 2. Choose a target block hash to forge a precommit vote for:
TARGET_BLOCK_HASH = <attacker-chosen value>

# 3. Compute the message digest as build_precommit_vote_message_digest does:
#    message = b"PRECOMMIT_VOTE" || TARGET_BLOCK_HASH.to_bytes_be()
#    digest  = blake2s_to_felt(message)

# 4. Sign with the known private key:
#    forged_signature = ecdsa_sign(PRIVATE_KEY, digest)

# 5. Inject forged_signature into the consensus precommit vote message
#    for TARGET_BLOCK_HASH. The verifier calls verify_precommit_vote_signature
#    with the corresponding public key (also derivable from PRIVATE_KEY),
#    which returns Ok(true).
```

Since every node uses the same key, a single attacker can produce a full quorum of forged precommit signatures for any block hash, satisfying any threshold check the consensus layer applies.

    async fn sign(&self, message_digest: MessageDigest) -> SignatureManagerResult<RawSignature> {
        let private_key = self.keystore.get_key().await?;
        let signature = ecdsa_sign(&private_key, &message_digest)
            .map_err(|e| SignatureManagerError::Sign(e.to_string()))?;

        Ok(signature.into())
    }
