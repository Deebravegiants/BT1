### Title
Unverified CKD Response Accepted for `AppPublicKey` Variant Allows Single Byzantine Participant to Corrupt Key Derivation Output - (File: `crates/contract/src/lib.rs`)

### Summary
In `respond_ckd`, the contract performs a full BLS12-381 pairing-based cryptographic verification of the CKD response only when the request used the `AppPublicKeyPV` variant. When the request used the legacy `AppPublicKey` variant, the verification branch is entirely empty — the contract accepts any `(big_y, big_c)` values without any check. A single Byzantine attested participant (strictly below the signing threshold) can race to call `respond_ckd` with a fabricated garbage response for any pending `AppPublicKey` CKD request. The contract accepts it, resolves and removes the pending yield, and delivers the garbage to the user. The user cannot decrypt the correct derived key, permanently destroying their access to any funds or secrets controlled by that key.

### Finding Description
In `respond_ckd` at `crates/contract/src/lib.rs` lines 675–682, the match on `request.app_public_key` has an asymmetric structure:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}   // ← empty: no verification
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
```

The `AppPublicKey(_) => {}` arm is a no-op. For `AppPublicKeyPV` requests, `ckd_output_check` enforces the pairing equation `e(big_c, g2) = e(big_y, app_pk2) · e(H(pk, app_id), msk_pk)`, which cryptographically binds the response to the master secret key and the user's ephemeral key. For `AppPublicKey` requests, no such binding is checked. Any `(big_y, big_c)` pair is accepted.

After the match, `pending_requests::resolve_yields_for` is called unconditionally, which drains the entire pending-request queue and delivers the (potentially garbage) response to all waiting callers. Once drained, the request is gone from state — honest nodes cannot re-respond.

The `AppPublicKey` variant is the legacy "privately verifiable" format, where the user supplies only `a·G1` (a G1 point). On-chain verification is structurally impossible for this variant because the contract does not know the user's scalar `a`. However, this design gap means the contract provides no protection against a malicious participant submitting a fabricated response for this variant.

### Impact Explanation
A single Byzantine attested participant (below the signing threshold) can:
1. Monitor the NEAR chain for `request_app_private_key` calls using the `AppPublicKey` variant.
2. Race to call `respond_ckd` with arbitrary garbage `(big_y, big_c)` values before honest nodes respond.
3. The contract accepts the garbage response (empty verification branch), resolves all queued yields with it, and removes the pending request from state.
4. The user receives garbage key material. Since `big_c ≠ H(pk, app_id)·msk + a·big_y`, the user cannot decrypt the correct derived key.
5. Any funds or secrets controlled by the derived key are permanently inaccessible.

This is permanent freezing of funds/key material controlled by the MPC network's key derivation output, triggered by a single participant below the signing threshold — matching the **Critical** impact tier (permanent freezing of funds controlled by the MPC network or chain-signature contract).

### Likelihood Explanation
- The `AppPublicKey` variant is the legacy default and is still accepted by the contract.
- `respond_ckd` requires only that the caller be a single attested participant; there is no threshold requirement on the respond call itself.
- The attack is a simple race: the malicious participant calls `respond_ckd` with `big_y = [0u8; 48]` and `big_c = [0u8; 48]` before honest nodes respond.
- The NEAR chain is public; any attested participant can observe pending CKD requests in real time.
- No collusion above threshold is required — one Byzantine participant suffices.

### Recommendation
1. **Deprecate and reject `AppPublicKey` requests** in `respond_ckd`. Since no on-chain verification is possible for this variant, the contract cannot enforce response integrity. Require all new CKD requests to use `AppPublicKeyPV`, which supports the full pairing check.
2. If backward compatibility must be maintained, add a threshold-based commit-reveal or multi-participant agreement mechanism before resolving `AppPublicKey` CKD yields, so that a single participant cannot unilaterally deliver a response.
3. At minimum, document the security gap prominently and warn users that `AppPublicKey` CKD responses are not verified on-chain.

### Proof of Concept
1. User Alice calls `request_app_private_key` with `AppPublicKey(a·G1)` on a BLS12-381 CKD domain.
2. Malicious attested participant Eve calls `respond_ckd` with:
   ```rust
   respond_ckd(
       ckd_request,  // matches Alice's pending request
       CKDResponse {
           big_y: Bls12381G1PublicKey([0u8; 48]),  // garbage
           big_c: Bls12381G1PublicKey([0u8; 48]),  // garbage
       }
   )
   ```
3. The contract executes the `AppPublicKey(_) => {}` branch — no check is performed.
4. `resolve_yields_for` drains Alice's pending yield with the garbage response and removes the request from `pending_ckd_requests`.
5. Alice's promise resolves with `(big_y=0, big_c=0)`. She cannot compute `big_c - a·big_y = H(pk, app_id)·msk` because the values are fabricated.
6. Alice's derived key is permanently unrecoverable. Any funds she had pre-funded to an address derived from that key are frozen.

The root cause is directly analogous to the reference report: a conditional branch (`AppPublicKey(_) => {}`) silently skips the critical action (cryptographic verification), allowing the operation to complete and state to be consumed without the required security guarantee being enforced.     #[handle_result]
    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

pub(crate) fn ckd_output_check(
    app_id: &dtos::CkdAppId,
    output: &CKDResponse,
    app_public_key: &dtos::CKDAppPublicKeyPV,
    public_key: &dtos::Bls12381G2PublicKey,
) -> bool {
    let big_c = env::bls12381_p1_decompress(&output.big_c);
    let big_y = env::bls12381_p1_decompress(&output.big_y);
    let pk2 = env::bls12381_p2_decompress(&app_public_key.pk2);
    let pk = env::bls12381_p2_decompress(public_key);
    let hash_point = hash_app_id_with_pk(public_key.as_slice(), app_id.as_ref());

    let pairing_input = [
        big_c.as_slice(),
        MINUS_G2_GENERATOR_UNCOMPRESSED.as_slice(),
        big_y.as_slice(),
        pk2.as_slice(),
        hash_point.as_slice(),
        pk.as_slice(),
    ]
    .concat();
    env::bls12381_pairing_check(&pairing_input)
}

pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
