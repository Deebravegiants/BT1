### Title
`ChainlinkPriceOracle.getAssetPrice()` Silently Accepts Zero Price, Enabling rsETH Price Distortion and Depositor Theft - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` does not validate that the Chainlink-returned price is positive before returning it. A zero price silently propagates through `LRTOracle._getTotalEthInProtocol()` into `_updateRsETHPrice()`, causing `rsETHPrice` to be set to a severely undervalued figure. Because `updateRSETHPrice()` is a public function callable by anyone, an attacker can commit the distorted price on-chain and then deposit assets to receive a disproportionate amount of rsETH, diluting existing holders.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` reads from Chainlink and returns the raw result with no positivity check:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

If `price == 0` (which Chainlink can return during circuit-breaker events, feed deprecation, or oracle failure), the function returns `0` without reverting.

This is in direct contrast to `ChainlinkOracleForRSETHPoolCollateral.getRate()`, which is used for L2 pool collateral and explicitly guards against this:

```solidity
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

The zero price then flows into `LRTOracle._getTotalEthInProtocol()`, where each asset's ETH value is computed as `totalAssetAmt.mulWad(assetER)`. A zero `assetER` silently zeroes out that asset's entire TVL contribution: [3](#0-2) 

The undervalued `totalETHInProtocol` is then used to compute the new rsETH price:

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
``` [4](#0-3) 

`updateRSETHPrice()` is a public, permissionless function: [5](#0-4) 

Once `rsETHPrice` is committed at the distorted (undervalued) level, `LRTDepositPool.getRsETHAmountToMint()` uses it directly in the mint calculation:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [6](#0-5) 

If `rsETHPrice` is halved (because one major LST's Chainlink feed returned 0), the attacker receives twice the rsETH for the same deposit, diluting all existing holders.

---

### Impact Explanation

**High — Theft of unclaimed yield / dilution of existing rsETH holders.**

If a supported LST asset's Chainlink feed returns 0 (e.g., stETH/ETH feed during a circuit-breaker event), `totalETHInProtocol` is undervalued by that asset's entire TVL share. The resulting `rsETHPrice` is proportionally lower. Any depositor who calls `depositAsset()` or `depositETH()` while this distorted price is stored receives more rsETH than the protocol's actual backing warrants, extracting value from all existing rsETH holders.

Additionally, if `pricePercentageLimit` is configured and the price drop exceeds the threshold, the protocol auto-pauses — causing a temporary freeze of all deposits and withdrawals. [7](#0-6) 

---

### Likelihood Explanation

**Low-Medium.** Chainlink feeds can return 0 during circuit-breaker activations, feed deprecation, or oracle infrastructure failures — all of which have occurred historically on mainnet. The attack requires no special privilege: `updateRSETHPrice()` is public, so any external actor can call it immediately after a zero-price round is published, before the Chainlink feed recovers. The window is narrow but real, and the attacker is incentivised by the profit from minting excess rsETH.

---

### Recommendation

Add a positivity check in `ChainlinkPriceOracle.getAssetPrice()`, mirroring the guard already present in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();
    if (price <= 0) revert InvalidPrice();
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
``` [8](#0-7) 

---

### Proof of Concept

1. Chainlink's stETH/ETH feed enters a circuit-breaker state and returns `price = 0` for one round.
2. Attacker calls `LRTOracle.updateRSETHPrice()` (public, no access control).
3. `_getTotalEthInProtocol()` computes stETH's contribution as `stETHBalance * 0 = 0`, undervaluing `totalETHInProtocol` by the entire stETH TVL.
4. `newRsETHPrice` is set to, e.g., `0.5e18` instead of `1.05e18`.
5. Attacker calls `LRTDepositPool.depositAsset(stETH, 1e18, 0, "")`.
6. `getRsETHAmountToMint` = `(1e18 * stETHPrice) / 0.5e18` ≈ `2.1e18` rsETH instead of the correct `~1e18`.
7. Chainlink feed recovers; attacker holds rsETH backed by only half the ETH it represents, having extracted yield from all prior holders. [9](#0-8) [10](#0-9)
