Audit Report

## Title
Unchecked Negative Chainlink Price Cast to `uint256` Causes Overflow Revert, Freezing Deposits and Price Updates - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` casts the `int256 price` returned by `latestRoundData()` directly to `uint256` without a non-positive guard. In Solidity 0.8.x, if `price` is negative, the cast produces `type(uint256).max`, and the subsequent multiplication by `1e18` triggers a checked-arithmetic overflow revert. This propagates up through `LRTOracle._getTotalEthInProtocol()` and blocks both `updateRSETHPrice()` and all deposit flows until the feed recovers or the oracle is replaced.

## Finding Description

In `contracts/oracles/ChainlinkPriceOracle.sol` at lines 52–54:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

There is no `price <= 0` check. If `price == -1`, then `uint256(int256(-1))` evaluates to `2^256 - 1`, and `(2^256 - 1) * 1e18` overflows under Solidity 0.8.x checked arithmetic, causing a revert.

The sibling contract `contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol` at line 32 applies the correct guard:

```solidity
if (ethPrice <= 0) revert InvalidPrice();
```

The call chain that propagates the revert:
1. `LRTDepositPool._beforeDeposit()` → `getRsETHAmountToMint()` → `lrtOracle.getAssetPrice(asset)` (line 520)
2. `LRTOracle.getAssetPrice()` → `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)` (line 157)
3. `LRTOracle._getTotalEthInProtocol()` → `getAssetPrice(asset)` (line 339), called from `_updateRsETHPrice()` (line 231)
4. `LRTOracle.updateRSETHPrice()` is `public` with no access restriction beyond `whenNotPaused` (line 87)

No existing check in `ChainlinkPriceOracle` or `LRTOracle` intercepts a negative price before the overflow.

## Impact Explanation

**Medium — Temporary freezing of funds.**

If any supported asset's Chainlink feed returns a negative price:
- All calls to `LRTDepositPool.depositETH()` and `depositAsset()` revert, blocking new deposits.
- `LRTOracle.updateRSETHPrice()` reverts, leaving the rsETH price stale.

The freeze persists until the feed recovers or the admin replaces the oracle via `updatePriceOracleFor`. Existing deposited funds are not lost, but the deposit flow is fully blocked, matching the "temporary freezing of funds" impact class.

## Likelihood Explanation

Chainlink's `latestRoundData()` returns `int256` precisely because negative values are a documented edge case (e.g., feed misconfiguration, aggregator anomaly, or extreme market conditions). The protocol team demonstrably considers this realistic — `ChainlinkOracleForRSETHPoolCollateral` explicitly guards against `ethPrice <= 0`. No attacker action is required; the condition arises from the external feed itself. Any user attempting a deposit or price update during such a feed state would trigger the revert.

## Recommendation

Add a non-positive price check before casting, consistent with the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

## Proof of Concept

1. Deploy a mock `AggregatorV3Interface` whose `latestRoundData()` returns `price = -1` and `decimals() = 8`.
2. Register it via `ChainlinkPriceOracle.updatePriceFeedFor(asset, mockFeed)`.
3. Call `ChainlinkPriceOracle.getAssetPrice(asset)`:
   - Executes `uint256(int256(-1)) * 1e18` = `(2^256 - 1) * 1e18` → Solidity 0.8.x overflow revert.
4. Call `LRTOracle.updateRSETHPrice()` → reverts at `_getTotalEthInProtocol()`.
5. Call `LRTDepositPool.depositAsset(asset, amount, 0, "")` → reverts at `_beforeDeposit()` → `getRsETHAmountToMint()` → `lrtOracle.getAssetPrice(asset)`.

Foundry test sketch:
```solidity
function test_negativePrice_freezesDeposits() public {
    MockBadFeed feed = new MockBadFeed(); // returns price = -1
    vm.prank(manager);
    chainlinkOracle.updatePriceFeedFor(asset, address(feed));
    vm.expectRevert(); // arithmetic overflow
    chainlinkOracle.getAssetPrice(asset);
}
```
