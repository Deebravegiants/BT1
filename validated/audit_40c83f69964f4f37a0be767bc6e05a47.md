### Title
Unbounded `fee_proposal_fri` During Startup Window Permanently Corrupts L2 Gas Price - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

### Summary

During the first `fee_proposal_window_size` (10) blocks after genesis or the V0_14_3 protocol upgrade, the `fee_proposal_fri` field in `ProposalInit` is accepted by validators without any bounds check. A malicious proposer can set `fee_proposal_fri` to an arbitrary extreme value (e.g., `u128::MAX`). This value is bound into the `ProposalCommitment`, accepted by all validators, recorded in `fee_proposals_window`, and — once the window fills — becomes the `fee_actual` floor that permanently drives the L2 gas price to near `u128::MAX` for all subsequent blocks.

### Finding Description

So a proposer who sets `fee_proposal_fri = u128::MAX` will produce a `ProposalFin.proposal_commitment = Poseidon(partial, u128::MAX)`, and the validator will independently compute the same value and accept it. The extreme value is then recorded in `fee_proposals_window` and propagates into `fee_actual` once the window fills.

`fee_actual` is used as a floor for the EIP-1559 L2 gas price:

```rust
fn calculate_next_l2_gas_price(&self, height: BlockNumber, l2_gas_used: GasAmount) -> GasPrice {
    let fee_actual = compute_fee_actual(
        &self.fee_proposals_window,
        height,
        VersionedConstants::latest_constants().fee_proposal_window_size,
    );
    calculate_next_l2_gas_price_for_fin(
        self.l2_gas_price, height, l2_gas_used,
        self.config.dynamic_config.override_l2_gas_price_fri,
        &self.config.dynamic_config.min_l2_gas_price_per_height,
        fee_actual,
    )
}

The `fee_proposal_margin_ppt` is 2 ppt (0.2%) across all deployed versioned-constants files: [7](#0-6) 

Once `fee_actual ≈ u128::MAX`, `fee_proposal_bounds` saturates: `upper = u128::MAX`, `lower ≈ u128::MAX`. All future honest proposers are forced to emit `fee_proposal_fri ≈ u128::MAX`, locking the L2 gas price floor permanently.

### Impact Explanation

**Critical — Incorrect fee/gas accounting with permanent economic impact.**

After the startup window, `fee_actual` is the median of the recorded `fee_proposal_fri` values. If a malicious proposer controls ≥6 of the first 10 proposal slots and sets `fee_proposal_fri = u128::MAX`, the median becomes `u128::MAX`. The L2 gas price floor is then `u128::MAX` for all future blocks, making every transaction require a near-infinite fee. The network becomes economically unusable. The `fee_proposal_margin_ppt = 2` ppt bound then prevents any honest proposer from ever reducing the fee back to a normal level — each block can only move the fee by 0.2%, and the window median will remain near `u128::MAX` for thousands of blocks.

### Likelihood Explanation

**Medium.** The attack window is the first `fee_proposal_window_size` (10) blocks after genesis or after the V0_14_3 protocol upgrade activates. The attacker must be selected as proposer by the Tendermint rotation for a majority of those 10 slots. This is not a privileged role — any consensus participant can be selected. The attack is a one-time, permanent action with no recovery path short of a hard fork.

### Recommendation

Do not skip the `fee_proposal_fri` bounds check when `fee_actual` is `None`. Instead, use a safe default reference value (e.g., `min_gas_price` or the current `l2_gas_price`) as the anchor for the bounds check during the startup window. This preserves the invariant that `fee_proposal_fri` can never deviate more than `fee_proposal_margin_ppt` from a known-safe reference, regardless of window completeness.

```rust
// Instead of skipping when fee_actual is None:
let reference = proposal_init_validation.fee_actual
    .unwrap_or(proposal_init_validation.l2_gas_price_fri);
let (lower_bound, upper_bound) = fee_proposal_bounds(
    reference,
    VersionedConstants::latest_constants().fee_proposal_margin_ppt,
);
if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
    return Err(...);
}
```
