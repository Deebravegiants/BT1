### Title
Gateway Admission Uses Stale Previous-Block L2 Gas Price Instead of `next_l2_gas_price` for Resource-Bounds Validation — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's `validate_resource_bounds` function computes its admission threshold against the **current committed block's** `l2_gas_price`, not the `next_l2_gas_price` that the EIP-1559 fee market has already computed for the block the transaction will actually be executed in. A developer TODO in the code explicitly acknowledges this. The mismatch causes the gateway to accept transactions whose `max_price_per_unit` is too low for the next block (they will revert on execution) and to reject transactions whose `max_price_per_unit` is correct for the next block but falls below the stale threshold.

---

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price via `gateway_fixed_block_state_reader.get_block_info()`:

```rust

    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` constructs a `BlockInfo` from `block_header.l2_gas_price` — the gas price **of** the committed block — and never reads `block_header.next_l2_gas_price`:

```rust
gas_prices: GasPrices {
    strk_gas_prices: GasPriceVector {
        ...
        l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
    },
},
``` [2](#0-1) 

The `BlockHeaderWithoutHash` struct carries a distinct `next_l2_gas_price` field — the EIP-1559-derived price for the **next** block — which is what the transaction will be charged at execution time:

```rust
pub struct BlockHeaderWithoutHash {
    pub l2_gas_price: GasPricePerToken,
    pub l2_gas_consumed: GasAmount,
    pub next_l2_gas_price: GasPrice,
    ...
}
``` [3](#0-2) 

The admission threshold is:

```rust
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
``` [4](#0-3) 

Because `calculate_next_l2_gas_price_for_fin` adjusts the price every block based on gas consumed vs. the EIP-1559 target, `next_l2_gas_price` diverges from `l2_gas_price` whenever the block is not exactly at the gas target: [5](#0-4) 

The `next_l2_gas_price` is written into the committed block header by `update_state_sync_with_new_block` using `self.l2_gas_price` (already updated to the next value by `update_l2_gas_price`): [6](#0-5) 

---

### Impact Explanation

**False acceptance (invalid transaction admitted):** When the network is congested and `next_l2_gas_price > l2_gas_price`, a transaction whose `max_price_per_unit` satisfies the stale threshold (`>= min_gas_price_percentage% × l2_gas_price`) but is below `next_l2_gas_price` passes gateway admission and enters the mempool. When the batcher executes it, blockifier's `perform_pre_validation_stage` checks `max_price_per_unit >= actual_gas_price` against the real next-block price and raises `InsufficientResourceBounds`, causing the transaction to revert. The gateway has accepted an invalid transaction.

**False rejection (valid transaction rejected):** When the network is under-congested and `next_l2_gas_price < l2_gas_price`, a transaction whose `max_price_per_unit` is correctly set for the next block (above `next_l2_gas_price`) but below `min_gas_price_percentage% × l2_gas_price` is rejected at the gateway even though it would execute successfully. The gateway has rejected a valid transaction.

Both outcomes fall under the allowed impact: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

---

### Likelihood Explanation

The EIP-1559 mechanism adjusts the price every block. With `gas_price_max_change_denominator` controlling the per-block delta, the gap between `l2_gas_price` and `next_l2_gas_price` is bounded per block but accumulates over sustained congestion or under-utilization. The false-acceptance path is reachable by any user who submits a transaction with `max_price_per_unit` set to the current block's gas price (a natural choice for wallets reading the latest block header). No privileged access is required; the trigger is ordinary transaction submission through the public gateway endpoint.

---

### Recommendation

Replace the `get_block_info()` call in `validate_resource_bounds` with a dedicated method on `GatewayFixedBlockStateReader` that returns `next_l2_gas_price` directly from `BlockHeaderWithoutHash.next_l2_gas_price`. The TODO comment in the code already identifies this fix:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
```

Concretely:
1. Add `async fn get_next_l2_gas_price(&self) -> StarknetResult<NonzeroGasPrice>` to `GatewayFixedBlockStateReader`.
2. Implement it in `GatewayFixedBlockSyncStateClient` by reading `block_header.next_l2_gas_price`.
3. Replace `previous_block_l2_gas_price` in `validate_resource_bounds` with the value returned by this new method.

---

### Proof of Concept

1. Read the latest committed block header; note `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (high-congestion scenario).
2. Construct a v3 `INVOKE` transaction with `AllResources` bounds where `l2_gas.max_price_per_unit = floor(min_gas_price_percentage/100 × P)` (just at the stale threshold) but `< P'`.
3. Submit to the gateway. `validate_resource_bounds` computes `threshold = min_gas_price_percentage% × P` and the transaction passes.
4. The transaction enters the mempool and is picked up by the batcher.
5. Blockifier's `perform_pre_validation_stage` evaluates `max_price_per_unit < actual_gas_price (= P')` and raises `InsufficientResourceBounds`, reverting the transaction — confirming the gateway admitted an invalid transaction. [7](#0-6) [8](#0-7) [9](#0-8)

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
