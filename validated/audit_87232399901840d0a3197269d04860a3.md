### Title
Unvalidated `builder` Field in `ProposalInit` Used as `sequencer_address` in Block Execution — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, `crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`is_proposal_init_valid` validates many fields of a network-received `ProposalInit` (height, l1_da_mode, l2_gas_price_fri, L1 gas prices, starknet_version, version_constant_commitment, fee_proposal_fri) but never validates the `builder` field. `convert_to_sn_api_block_info` then maps `init.builder` directly to `sequencer_address` in the `BlockInfo` passed to the batcher for execution. A malicious proposer can set `builder` to any arbitrary address; all validators will accept the proposal, execute the block with the attacker-controlled `sequencer_address`, and commit a block whose hash encodes the wrong sequencer — redirecting all transaction fees and corrupting the authoritative block hash.

---

### Finding Description

`ProposalInit` carries a `builder` field (the address of the block builder/sequencer): [1](#0-0) 

`convert_to_sn_api_block_info` maps this field directly to `sequencer_address` in the `BlockInfo` used for execution: [2](#0-1) 

`is_proposal_init_valid` — the sole gate that checks a received `ProposalInit` before the batcher is invoked — validates `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`, but contains **no check on `builder`**: [3](#0-2) [4](#0-3) 

The proposer sets `builder` from its own static config: [5](#0-4) 

A malicious proposer can substitute any address for `builder`. The validator calls `is_proposal_init_valid` (which passes), then `initiate_validation` → `convert_to_sn_api_block_info(init)` (which uses the attacker-supplied `builder` as `sequencer_address`): [6](#0-5) [7](#0-6) 

The `ProposalFinMismatch` check at line 244 does **not** protect against this. Both the proposer and the validator independently call `convert_to_sn_api_block_info` with the same (attacker-supplied) `init.builder`, so both batchers execute with `sequencer_address = attacker_address`, produce the same `partial_block_hash`, and the commitment comparison passes: [8](#0-7) 

The `partial_block_hash` is computed from `PartialBlockHashComponents`, which includes `sequencer` (derived from `block_info.sequencer_address`): [9](#0-8) 

---

### Impact Explanation

**Wrong state, receipt, and block hash from execution logic (Critical).**

1. **Fee theft**: Starknet transfers transaction fees to `sequencer_address` during execution. With `builder = attacker_address`, every fee in the block is credited to the attacker's account rather than the legitimate sequencer.

2. **Corrupted block hash**: `sequencer_address` is a direct input to `calculate_block_hash` via `PartialBlockHashComponents`. A wrong `builder` produces a wrong `partial_block_hash` and therefore a wrong final block hash, breaking retrospective block hash verification and any downstream consumer of the canonical block hash.

3. **Wrong `get_sequencer_address()` syscall result**: Contracts that call `get_sequencer_address()` during execution receive the attacker-controlled address, corrupting any contract logic that depends on it.

---

### Likelihood Explanation

Any consensus participant who is selected as proposer in a round can trigger this. The proposer role rotates among validators, so the attack surface is any validator in the active set. No external or privileged access beyond being a consensus participant is required. The attack is silent — no error is raised, no log distinguishes a tampered `builder` from a legitimate one.

---

### Recommendation

Add an explicit check for `builder` inside `is_proposal_init_valid`. The validator knows the expected builder address from its own static config (`self.config.static_config.builder_address`). This value should be threaded into `ProposalInitValidation` and compared against `init_proposed.builder` with an exact-match rejection, analogous to how `l1_da_mode` and `l2_gas_price_fri` are checked:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

The `ProposalInitValidation` struct should gain an `expected_builder: ContractAddress` field, populated from `self.config.static_config.builder_address` at the call sites in `sequencer_consensus_context.rs`. [10](#0-9) [11](#0-10) 

---

### Proof of Concept

1. A malicious validator is selected as proposer for height H.
2. It constructs `ProposalInit` with `builder = 0xATTACKER` (its own address) instead of the legitimate `builder_address` from config.
3. It broadcasts this `ProposalInit` to all peer validators.
4. Each validator calls `validate_proposal` → `is_proposal_init_valid`: all checked fields pass; `builder` is never inspected.
5. Each validator calls `initiate_validation` → `convert_to_sn_api_block_info(init)` → batcher executes with `sequencer_address = 0xATTACKER`.
6. All fee transfers in the block go to `0xATTACKER`.
7. The batcher returns `partial_block_hash` computed with `sequencer = 0xATTACKER`; `proposal_commitment_from(partial_block_hash, fee_proposal_fri)` matches the proposer's `ProposalFin`; `ProposalFinMismatch` check passes.
8. Consensus reaches decision; the block is committed with `sequencer_address = 0xATTACKER` in the canonical block hash.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
