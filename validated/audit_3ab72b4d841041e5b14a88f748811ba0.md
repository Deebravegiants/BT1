### Title
Unconstrained `fee_proposal_fri` During Bootstrap Window Permanently Poisons `fee_actual` and Corrupts `l2_gas_price` for All Subsequent Blocks — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

The SNIP-35 fee-proposal validation in `is_proposal_init_valid` intentionally skips the bounds check on `ProposalInit.fee_proposal_fri` when `fee_actual` is `None` — which is the case for the first `window_size` (10) blocks. A malicious proposer during this bootstrap window can set `fee_proposal_fri` to any value (e.g., `u128::MAX`) without rejection. Because every committed block's `fee_proposal_fri` is recorded in `fee_proposals_window` and `fee_actual` is the median of that window, poisoning 6 or more of the first 10 blocks drives `fee_actual` to `u128::MAX` at block 10. From that point, the bounds for block 11 are `[~u128::MAX, u128::MAX]`, locking `l2_gas_price` near `u128::MAX` permanently.

---

### Finding Description

**Root cause — skipped bounds check during bootstrap:**

In `is_proposal_init_valid`, the fee-proposal bounds check is guarded by an `if let` that requires both `fee_actual` and `fee_proposal` to be `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

`fee_actual` is computed as the median of `fee_proposals_window[height - window_size .. height - 1]`. For heights 0–9 (with `window_size = 10`), `compute_fee_actual` returns `None` because `height.0.checked_sub(window_size)` underflows:

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ({window_size})");
    return None;
};
``` [2](#0-1) 

The only check applied to `fee_proposal_fri` during bootstrap is that it is `Some` (required for V0_14_3+). No value constraint is enforced. An honest proposer freezes at `l2_gas_price`:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [3](#0-2) 

But a malicious proposer is not constrained to do so.

**Poisoning the window:**

Every committed block's `fee_proposal_fri` is recorded unconditionally in `finalize_decision`:

```rust
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [4](#0-3) 

```rust
fn record_fee_proposal(&mut self, height: BlockNumber, fee_proposal_fri: Option<GasPrice>) {
    self.fee_proposals_window.insert(height, fee_proposal_fri);
}
``` [5](#0-4) 

If a malicious proposer sets `fee_proposal_fri = u128::MAX` for blocks 0–9, then at block 10, `compute_fee_actual` returns `Some(u128::MAX)` (median of ten `u128::MAX` values). The bounds for block 11 become `[u128::MAX * 1000 / 1002, u128::MAX]` — both saturate near `u128::MAX` — locking the fee permanently.

**Downstream effect on `l2_gas_price`:**

`fee_actual` feeds `calculate_next_l2_gas_price_for_fin`, which updates `self.l2_gas_price`. This price is placed in `ProposalInit.l2_gas_price_fri` for every subsequent block and passed to the blockifier as `BlockInfo.gas_prices.l2_gas_price`. Validators enforce an exact match on `l2_gas_price_fri`:

```rust
if !(init_proposed.height == proposal_init_validation.height
    && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
    && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
{
    return Err(ValidateProposalError::InvalidProposalInit(...));
}
``` [6](#0-5) 

With `l2_gas_price ≈ u128::MAX`, every transaction's `max_price_per_unit` is below the block's gas price, causing all transactions to fail the pre-validation fee check:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow { ... }
    );
}
``` [7](#0-6) 

The gateway's `validate_tx_l2_gas_price_within_threshold` also rejects all V3 transactions whose `max_price_per_unit` is below the threshold derived from the corrupted `l2_gas_price`: [8](#0-7)

### Impact Explanation

A malicious proposer during the first 10 blocks (bootstrap window) can permanently corrupt `fee_actual` to `u128::MAX`. This causes:

1. **Incorrect fee/gas accounting with economic impact (Critical):** `l2_gas_price` is driven to `u128::MAX` for all subsequent blocks. Fee calculations overflow or produce astronomically wrong values.
2. **Mempool/gateway admission rejects all valid transactions (High):** Every V3 transaction is rejected at the gateway because its `max_price_per_unit` is below the corrupted threshold. The network becomes permanently unusable.
3. **Wrong `ProposalCommitment`:** The commitment is `Poseidon(partial_block_hash, fee_proposal_fri)`. Blocks committed with `fee_proposal_fri = u128::MAX` carry a commitment that encodes the corrupted value, which is then used as `parent_proposal_commitment` in the cende blob. [9](#0-8) 

---

### Likelihood Explanation

- **Trigger condition:** The attacker must be selected as proposer during at least 6 of the first 10 blocks. In a Tendermint validator set, proposer selection is round-robin by voting power. A validator with ≥60% voting power can guarantee this; a validator with lower power has a proportional probability.
- **No special privileges beyond being a validator:** Any validator in the set can be a proposer. The attack requires no external oracle manipulation, no flash loan, and no contract interaction — only setting a field in `ProposalInit` to an extreme value.
- **Permanent effect:** Once `fee_actual` is poisoned, the geometric bounds `[~u128::MAX, u128::MAX]` prevent recovery without a protocol upgrade or manual override via `override_l2_gas_price_fri`.

---

### Recommendation

Enforce a fallback constraint on `fee_proposal_fri` during the bootstrap window. When `fee_actual` is `None`, the validator should require that `fee_proposal_fri` equals the locally-computed `l2_gas_price` (within a small tolerance), rather than accepting any value:

```rust
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    match proposal_init_validation.fee_actual {
        Some(fee_actual) => {
            // existing bounds check
        }
        None => {
            // Bootstrap: enforce fee_proposal == l2_gas_price (within 1 unit or small margin)
            let expected = proposal_init_validation.l2_gas_price_fri;
            if fee_proposal != expected {
                return Err(ValidateProposalError::InvalidProposalInit(...));
            }
        }
    }
}
```

Alternatively, cap `fee_proposal_fri` at a protocol-defined maximum (e.g., `min_gas_price * MAX_MULTIPLIER`) regardless of whether `fee_actual` is available.

### Proof of Concept

**Pre-conditions:**
- Network is at genesis (height 0), `window_size = 10` (from `orchestrator_versioned_constants_0_14_3.json`).
- Attacker controls a validator that is selected as proposer for blocks 0–9 (or any 6+ of them).

**Attack steps:**

1. For each block in heights 0–9 where the attacker is proposer, construct `ProposalInit` with `fee_proposal_fri = Some(GasPrice(u128::MAX))`.
2. Broadcast the proposal. Validators call `is_proposal_init_valid`:
   - `fee_actual = compute_fee_actual(&window, height, 10)` returns `None` (height < 10).
   - The `if let (Some(fee_actual), Some(fee_proposal))` guard is not entered.
   - The proposal passes validation.
3. Consensus reaches decision on each block. `finalize_decision` calls `record_fee_proposal(height, Some(u128::MAX))`.
4. At height 10, `compute_fee_actual` returns `Some(u128::MAX)` (median of 10 × `u128::MAX`).
5. `update_l2_gas_price` sets `self.l2_gas_price ≈ u128::MAX`.
6. All subsequent `ProposalInit` messages carry `l2_gas_price_fri ≈ u128::MAX`.
7. The gateway rejects every V3 transaction with `GAS_PRICE_TOO_LOW`. The network is permanently halted.

**Corrupted value:** `l2_gas_price = u128::MAX` (18_446_744_073_709_551_615 FRI), versus the expected `min_gas_price = 0x1dcd65000` (8_000_000_000 FRI). [10](#0-9)
