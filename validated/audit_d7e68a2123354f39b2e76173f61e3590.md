### Title
Unchecked `RAND_bytes()` Return Value in EdDSA Scalar Signing Produces Uninitialized Nonce — (`File: src/cbmpc/crypto/ec25519_core.cpp`)

---

### Summary

`ED25519_sign_with_scalar` calls `RAND_bytes(nonce, 64)` without checking the return value. If `RAND_bytes` fails, the 64-byte `nonce` array remains uninitialized stack data. That uninitialized value is then used directly as the EdDSA signing nonce. The function unconditionally returns `1`, so callers cannot detect the failure. Nonce reuse or predictability in EdDSA allows full private scalar recovery from two signatures.

---

### Finding Description

In `src/cbmpc/crypto/ec25519_core.cpp`, `ED25519_sign_with_scalar` declares a 64-byte stack buffer and fills it via `RAND_bytes`:

```cpp
// src/cbmpc/crypto/ec25519_core.cpp:987-999
extern "C" int ED25519_sign_with_scalar(...) {
  uint8_t nonce[64];
  RAND_bytes(nonce, 64);          // ← return value never checked

  uint8_t az[32];
  for (int i = 0; i < 32; i++) az[i] = scalar_bin[31 - i];

  sign_with_nonce(out_sig, message, message_len, public_key, az, nonce);
  OPENSSL_cleanse(nonce, sizeof(nonce));
  OPENSSL_cleanse(az, sizeof(az));
  return 1;                       // ← always 1, failure is invisible
}
```

`RAND_bytes` returns `1` on success, `0` on failure, and `-1` if not supported (OpenSSL 3). The return value is silently discarded. If it fails, `nonce[64]` holds whatever was on the stack at that moment.

`sign_with_nonce` then reads all 64 bytes of that buffer to derive the nonce scalar:

```cpp
// src/cbmpc/crypto/ec25519_core.cpp:969-984
static void sign_with_nonce(..., const uint8_t nonce[32]) {
  bn_t nonce_bn = from_le_mod_q(mem_t(nonce, 64));  // reads 64 bytes
  point_t R;
  curve_t::mul_to_generator(nonce_bn, R);
  to_bin(R, signature);
  ...
  bn_t s = q.mul(hram_bn, az_bn);
  s = q.add(s, nonce_bn);
  ...
}
```

Contrast with the library's own `gen_random` wrapper, which properly asserts the result:

```cpp
// src/cbmpc/crypto/base.cpp:79-82
void gen_random(byte_ptr output, int size) {
  int res = RAND_bytes(output, size);
  cb_assert(res > 0);             // ← correct pattern
}
```

`ED25519_sign_with_scalar` bypasses this wrapper and calls `RAND_bytes` directly without the check.

---

### Impact Explanation

`ED25519_sign_with_scalar` is the signing path taken whenever the key is stored as a scalar (i.e., `K.ed_bin.empty()` is true), which is the MPC key-share path:

```cpp
// src/cbmpc/crypto/base_eddsa.cpp:212-224
buf_t ecurve_ed_t::sign(const ecc_prv_key_t& K, mem_t hash) const {
  ...
  if (K.ed_bin.empty()) {
    buf_t scalar = K.value().to_bin(ed25519::prv_bin_size());
    ED25519_sign_with_scalar(sig.data(), hash.data, hash.size,
                             pub_bin.data(), scalar.data());  // ← this path
  } else {
    ED25519_sign(...);
  }
  return sig;
}
```

This is reached from the public API:

- `coinbase::api::eddsa_2p::sign()` → `cbmpc_eddsa_2p_sign()` (C API)
- `coinbase::api::eddsa_mp::sign_additive()` → `cbmpc_eddsa_mp_sign_additive()` (C API)
- `coinbase::api::eddsa_mp::sign_ac()` → `cbmpc_eddsa_mp_sign_ac()` (C API)

If `RAND_bytes` fails and the nonce is uninitialized or repeated across two signing calls, an attacker who observes two signatures `(R, s1)` and `(R, s2)` over messages `m1` and `m2` can recover the private scalar `x` via:

```
s1 = r + H(R, pk, m1) * x  (mod q)
s2 = r + H(R, pk, m2) * x  (mod q)
x  = (s1 - s2) / (H(R, pk, m1) - H(R, pk, m2))  (mod q)
```

This is a complete private key share compromise.

---

### Likelihood Explanation

`RAND_bytes` failure is uncommon on well-seeded production systems but is not impossible: it can occur during early boot before the OS entropy pool is ready, in constrained/embedded environments, or when OpenSSL's internal DRBG is in an error state. The critical aggravating factor is that the function always returns `1`, so the failure is completely invisible to callers and to the protocol layer — no error is propagated, no retry occurs, and a signature is produced and accepted as valid.

---

### Recommendation

Replace the bare `RAND_bytes` call with the library's own checked wrapper, or add an explicit return-value check and abort:

```cpp
extern "C" int ED25519_sign_with_scalar(...) {
  uint8_t nonce[64];
  if (RAND_bytes(nonce, 64) != 1) {
    OPENSSL_cleanse(nonce, sizeof(nonce));
    return 0;  // propagate failure
  }
  ...
  return 1;
}
```

Callers (`ecurve_ed_t::sign`) should then check the return value and propagate an error rather than returning a potentially invalid signature.

---

### Proof of Concept

1. Trigger `RAND_bytes` failure (e.g., by exhausting OpenSSL entropy or patching it to return 0 in a test harness).
2. Call `coinbase::api::eddsa_2p::sign()` twice with the same scalar-based key blob and two different messages `m1`, `m2`.
3. Both calls proceed without error and return signatures `(R1, s1)` and `(R2, s2)`.
4. If the uninitialized stack nonce is identical across both calls (deterministic stack layout), `R1 == R2`.
5. Compute `x = (s1 - s2) * modular_inverse(H(R,pk,m1) - H(R,pk,m2), q) mod q` to recover the private scalar. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
