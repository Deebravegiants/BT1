Audit Report

## Title
Publicly Callable `updateRSETHPrice()` Allows Any Actor to Trigger Protocol-Wide Pause — (`contracts/LRTOracle.sol`)

## Summary
`LRTOracle.updateRSETHPrice()` is declared `public` with only a `whenNotPaused` guard. Its internal implementation unconditionally pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself whenever the computed rsETH price falls more than `pricePercentageLimit` below `highestRsethPrice`. Any unprivileged EOA can call this function the moment on-chain conditions satisfy the pause trigger, forcing a protocol-wide freeze before operators can assess or respond.

## Finding Description
`updateRSETHPrice()` at line 87 carries no caller restriction:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, lines 270–282 execute unconditionally for any caller:

```solidity
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);

    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;
    }
```

The privileged counterpart `updateRSETHPriceAsManager()` (line 94, `onlyLRTManager`) exists specifically to bypass the *upside* threshold revert (`PriceAboveDailyThreshold` at line 264) — it does **not** bypass the downside pause. However, operators may legitimately choose to delay calling the price update function during a transient dip (e.g., a temporary LST price anomaly) and wait for recovery before committing the new price on-chain. The public function removes that operational option entirely: any external actor can force the price update and lock in the pause the moment the threshold is crossed.

Existing guards are insufficient:
- `whenNotPaused` only prevents re-entrancy after the pause is set; it does not restrict who may initiate the update.
- There is no `onlyLRTOperator`, `onlyLRTManager`, or equivalent check on `updateRSETHPrice()`.
- `_pause()` at line 319 is idempotent (`if (paused) return`) but does not prevent the first caller from triggering it.

## Impact Explanation
**Medium — Temporary freezing of funds.**

Once paused, all user-facing entry points revert: `depositETH`, `depositAsset` (deposit pool), `initiateWithdrawal`, `completeWithdrawal`, `instantWithdrawal` (withdrawal manager). Funds are not lost but are inaccessible until an admin manually calls `unpause()` on each of the three contracts individually. This matches the allowed scope impact of *Temporary freezing of funds*.

## Likelihood Explanation
The trigger condition — `newRsETHPrice` dropping more than `pricePercentageLimit` below `highestRsethPrice` — is realistic after EigenLayer slashing events, large validator exits, or significant LST price dislocations. An attacker requires no capital, no prior setup, and no privileges. They need only monitor on-chain state and submit a single call to `updateRSETHPrice()`. The attack is repeatable: after each admin `unpause()`, if the price condition still holds, the attacker can immediately re-trigger the pause.

## Recommendation
Restrict `updateRSETHPrice()` to authorized callers (e.g., `onlyLRTOperator` or `onlyLRTManager`), mirroring the access control already applied to `updateRSETHPriceAsManager()`. If a permissionless price update is required for liveness, separate the pause-triggering path: allow public callers to update the price but gate the automatic pause invocation behind a privileged role check, so operators retain the ability to assess transient conditions before committing a protocol-wide freeze.

## Proof of Concept
1. Deploy/fork with `highestRsethPrice = 1.05e18`, `pricePercentageLimit = 1e16` (1%).
2. Simulate a slashing event reducing `_getTotalEthInProtocol()` such that `newRsETHPrice = 1.03e18` (≈1.9% drop, exceeding the 1% limit).
3. Unprivileged EOA calls `LRTOracle.updateRSETHPrice()`.
4. `_updateRsETHPrice()` computes `isPriceDecreaseOffLimit = true`.
5. `LRTDepositPool.pause()`, `LRTWithdrawalManager.pause()`, and `LRTOracle._pause()` are called.
6. Subsequent calls to `depositETH`, `initiateWithdrawal`, and `completeWithdrawal` revert with `ContractPaused` / `Pausable: paused`.
7. Protocol remains frozen until admin unpauses each contract.

**Foundry test sketch:**
```solidity
function test_anyoneCanTriggerPause() public {
    // arrange: set price conditions so newRsETHPrice < highestRsethPrice * (1 - limit)
    vm.prank(attacker); // unprivileged EOA
    lrtOracle.updateRSETHPrice();
    assertTrue(lrtDepositPool.paused());
    assertTrue(withdrawalManager.paused());
    assertTrue(lrtOracle.paused());
}
```
