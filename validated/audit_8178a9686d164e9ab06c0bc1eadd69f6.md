### Title
`vote_add_domains` Requires Unanimity Instead of Governance Threshold, Allowing a Single Sub-Threshold Participant to Permanently Block New Domain Additions — (`crates/contract/src/state/running.rs`)

### Summary

`vote_add_domains` in `RunningContractState` gates the domain-addition transition on **all** participants voting, not on the governance threshold. A single participant who withholds their vote permanently prevents any new signing domain from ever being added to the network, making the domain registry effectively immutable in practice — the direct analog of CVXStaker's inability to process newly added `extraRewards`.

### Finding Description

In `crates/contract/src/state/running.rs`, the transition condition for adding new domains is:

```rust
if self.parameters.participants().len() as u64 == n_votes {
``` [1](#0-0) 

This requires every single participant to cast an identical vote before the contract transitions to `Initializing` to generate keys for the new domains. Every other governance operation in the same contract uses the governance threshold (`Threshold`), not unanimity:

- `vote_new_parameters` (resharing) — threshold votes
- `vote_reshared` — threshold votes
- `vote_public_key` — threshold votes [2](#0-1) 

The `DomainRegistry` itself is a `Vec<DomainConfig>` that is only ever extended through this single code path: [3](#0-2) 

There is no alternative path to add domains. If the unanimity condition is never satisfied, the domain list is frozen at its initial state.

The contract's `check_request_preconditions` rejects any `sign()` call whose `domain_id` is not in the registry:

```rust
let Some(domain_config) = domains.get_domain_by_domain_id(domain_id) else {
    env::panic_str(&InvalidParameters::DomainNotFound { provided: domain_id }.to_string());
};
``` [4](#0-3) 

So any new domain that cannot be added is permanently unserviceable.

### Impact Explanation

A single participant (strictly below the signing threshold) who refuses to call `vote_add_domains` with the agreed proposal permanently prevents the domain registry from being extended. All `sign()` and `request_app_private_key()` calls targeting the new domain ID will be rejected with `DomainNotFound` indefinitely. The network loses the ability to expand its signing capabilities (e.g., adding a new EdDSA domain, a new CKD domain, or a new ForeignTx domain) without a full contract upgrade. This breaks the production safety invariant that governance decisions require only threshold-many votes, and permanently freezes the request lifecycle for any domain type not present at initialization.

**Allowed impact matched:** Medium — contract execution-flow manipulation that breaks production safety/accounting invariants without relying on network-level DoS or operator misconfiguration.

### Likelihood Explanation

The governance threshold is typically strictly less than the total participant count (e.g., threshold = 7 out of 10 participants). Any one of the 10 participants — including one that is Byzantine but below the signing threshold — can unilaterally veto all future domain additions by simply not submitting a matching `vote_add_domains` transaction. The attacker needs no special privilege, no key material, and no collusion. The entry path is the public `vote_add_domains` contract method, callable by any participant account.

### Recommendation

Replace the unanimity check with the governance threshold, consistent with every other governance operation:

```rust
// Before (unanimity):
if self.parameters.participants().len() as u64 == n_votes {

// After (threshold):
if self.parameters.threshold().value() <= n_votes {
``` [1](#0-0) 

### Proof of Concept

1. Network is `Running` with N = 10 participants, governance threshold T = 7.
2. Participants 1–9 all call `vote_add_domains` with an identical `DomainConfig` for a new EdDSA domain.
3. Participant 10 (Byzantine, below signing threshold) never calls `vote_add_domains`.
4. `n_votes = 9 < 10 = participants().len()` — the condition at line 237 is never satisfied.
5. The contract never transitions to `Initializing`; the new domain is never added to `DomainRegistry`.
6. Any user calling `sign({ domain_id: new_id, ... })` receives `DomainNotFound` permanently.
7. The network cannot add any new signing domain for the lifetime of this participant set without a full contract upgrade. [5](#0-4) [6](#0-5)

    pub fn vote_add_domains(
        &mut self,
        domains: Vec<DomainConfig>,
    ) -> Result<Option<InitializingContractState>, Error> {
        if domains.is_empty() {
            return Err(DomainError::AddDomainsMustAddAtLeastOneDomain.into());
        }
        let num_participants = u64::try_from(self.parameters.participants().len())
            .expect("participant count fits in u64");
        for domain in &domains {
            validate_domain_purpose(domain)?;
            validate_domain_threshold(domain, num_participants)?;
        }
        // Keep trust assumptions consistent: a domain must never require more shares to
        // reconstruct than the GovernanceThreshold demands to govern. Route through the
        // canonical helper so the cross-domain invariant has a single source of truth.
        ThresholdParameters::validate_governance_against_reconstruction(
            num_participants,
            self.parameters.threshold(),
            max_reconstruction_threshold(&domains),
        )?;
        let participant = AuthenticatedParticipantId::new(self.parameters.participants())?;
        let n_votes = self.add_domains_votes.vote(domains.clone(), &participant);
        if self.parameters.participants().len() as u64 == n_votes {
            let new_domains = self.domains.add_domains(domains.clone())?;
            Ok(Some(InitializingContractState {
                generated_keys: self.keyset.domains.clone(),
                domains: new_domains,
                epoch_id: self.keyset.epoch_id,
                generating_key: KeyEvent::new(
                    self.keyset.epoch_id,
                    domains[0].clone(),
                    self.parameters.clone(),
                ),
                cancel_votes: BTreeSet::new(),
            }))
        } else {
            Ok(None)
        }
    }
