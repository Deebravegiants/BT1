### Title
Single Byzantine Participant Can Forge CKD Response for `AppPublicKey` Variant, Bypassing Threshold Requirement — (`File: crates/contract/src/lib.rs`)

### Summary

`respond_ckd` performs cryptographic output verification only for the `AppPublicKeyPV` variant of a CKD request. For the legacy `AppPublicKey` variant, the verification block is an explicit no-op. Any single attested participant — strictly below the signing threshold — can call `respond_ckd` with an arbitrary `CKDResponse` for any pending `AppPublicKey` request, and the contract will accept it, drain the entire fan-out queue, and deliver the forged response to every waiting caller.

### Finding Description

In `respond_ckd` (lines 653–689 of `crates/contract/src/lib.rs`), after confirming the caller is an attested participant, the contract branches on the request's `app_public_key` variant:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}          // ← no check at all
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
``` [1](#0-0) 

For `AppPublicKeyPV`, `ckd_output_check` verifies that the response is cryptographically consistent with the BLS12-381 master public key and the user's ephemeral key pair — enforcing that the MPC network actually produced the output. For `AppPublicKey`, the arm is empty: the response bytes are accepted unconditionally.

After this branch, `resolve_yields_for` is called unconditionally, draining the entire pending fan-out queue and delivering whatever bytes were supplied to every caller who submitted the same request:

```rust
pending_requests::resolve_yields_for(
    &mut self.pending_ckd_requests,
    &request,
    serde_json::to_vec(&response).unwrap(),
)
``` [2](#0-1) 

The fan-out queue can hold up to `MAX_PENDING_REQUEST_FAN_OUT` (128) yields per request key, all of which are resolved in one call. [3](#0-2) 

### Impact Explanation

A single Byzantine attested participant — one node, well below the t-of-n signing threshold — can:

1. Observe any pending `AppPublicKey` CKD request on-chain.
2. Construct an arbitrary `CKDResponse` (`big_y`, `big_c` are free BLS12-381 G1 points with no constraint).
3. Call `respond_ckd` before the legitimate threshold-produced response arrives.
4. The contract accepts the forged response, removes the pending entry, and delivers the forged `big_y`/`big_c` to every queued caller.

The legitimate MPC response, when it eventually arrives, finds `RequestNotFound` and is silently discarded. Every caller who submitted that request receives attacker-controlled key material instead of the threshold-derived confidential key. The threshold invariant — that t-of-n participants must collaborate to produce a CKD output — is completely bypassed for the `AppPublicKey` variant.

This maps to the allowed impact: **Medium — request-lifecycle and contract execution-flow manipulation that breaks production safety/accounting invariants** (the threshold invariant for CKD) without requiring network-level DoS or operator misconfiguration.

### Likelihood Explanation

The `AppPublicKey` variant is described in the README as the "legacy" format and is still accepted. Any single current attested participant who turns Byzantine can exploit this. The attack requires no special knowledge beyond observing the mempool/chain for pending CKD requests, and the window is the entire lifetime of the pending yield (~200 blocks). Likelihood is **Medium**.

### Recommendation

Apply the same `ckd_output_check` (or an equivalent single-key variant) to the `AppPublicKey` arm of `respond_ckd`. If on-chain verification is not feasible for the legacy single-key format, the `AppPublicKey` variant should be deprecated and rejected at the `respond_ckd` gate, forcing callers to use `AppPublicKeyPV` which carries the verifiable key pair.

### Proof of Concept

```
1. Alice submits request_app_private_key with AppPublicKey variant (legacy single G1 point).
   → pending_ckd_requests[request_key] = [yield_id_alice]

2. Byzantine participant Eve (a single attested node, below threshold) observes the request on-chain.

3. Eve calls respond_ckd(request, CKDResponse { big_y: [0u8;48], big_c: [0u8;48] }).
   → AppPublicKey arm: no check, falls through.
   → resolve_yields_for drains the queue, resumes yield_id_alice with Eve's forged bytes.

4. Alice's yield-callback fires with Ok(forged_response).
   → Alice receives attacker-controlled big_y / big_c.
   → The legitimate threshold response (when it arrives) gets RequestNotFound.

5. The threshold invariant is bypassed: one participant produced the accepted CKD output.
``` [4](#0-3) [5](#0-4)

    #[handle_result]
    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
