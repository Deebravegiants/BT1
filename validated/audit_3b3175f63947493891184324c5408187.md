### Title
Anyone Can Call `claim` for Any Account in `MerkleDistributor`, Forcing Fee Deduction — (File: `contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

---

### Summary

The `claim` function in `MerkleDistributor.sol` accepts an arbitrary `account` address parameter but performs no check that `msg.sender == account`. Any external caller can trigger a claim on behalf of any user, causing the protocol fee to be permanently deducted from the victim's allocation and sent to the treasury. The victim receives fewer tokens than they would have received by claiming on their own terms.

---

### Finding Description

`MerkleDistributor.claim` is declared as:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
``` [1](#0-0) 

After verifying the merkle proof, the function deducts a fee and sends the remainder to `account`:

```solidity
uint256 fee = (claimableAmount * feeInBPS) / 10_000;
uint256 amountToSend = claimableAmount - fee;

IERC20(token).safeTransfer(account, amountToSend);
IERC20(token).safeTransfer(protocolTreasury, fee);
``` [2](#0-1) 

There is **no check** that `msg.sender == account`. The function processes the claim, marks it consumed, and deducts the fee regardless of who the caller is.

This is in direct contrast to `KernelMerkleDistributor.sol`, which explicitly guards against this pattern:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [3](#0-2) 

Because merkle proofs and cumulative amounts are derivable from public off-chain data (indexer events, Merkle tree snapshots), an attacker has everything needed to call `claim` for any victim at any time.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

The fee portion of every user's merkle allocation is permanently lost to the user when a third party forces the claim. The victim cannot reclaim the fee; it is irrevocably transferred to `protocolTreasury`. The user's `userClaims` state is updated to reflect the full `cumulativeAmount`, so they cannot re-claim the fee-deducted portion later. [4](#0-3) 

---

### Likelihood Explanation

**High.** The attack requires:
- No special role or privilege.
- Only public on-chain/off-chain data: the victim's address, their merkle index, cumulative amount, and proof.
- A single external call with no upfront capital.

Any unprivileged address can execute this against any unclaimed allocation at any time the contract is unpaused.

---

### Recommendation

Add a caller authorization check at the top of `claim`, mirroring the pattern already used in `KernelMerkleDistributor.sol`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [3](#0-2) 

---

### Proof of Concept

```solidity
// Attacker has no tokens and no special role.
// Merkle proof data is derived from public off-chain snapshot.
address victim = 0xVictim;
uint256 index = ...; // from public snapshot
uint256 cumulativeAmount = ...; // from public snapshot
bytes32[] memory proof = ...; // from public Merkle tree

// Attacker forces victim's claim, deducting fee from victim's allocation.
merkleDistributor.claim(index, victim, cumulativeAmount, proof);

// Result:
// - victim receives (cumulativeAmount - previouslyClaimed) * (1 - feeInBPS/10_000)
// - fee is sent to protocolTreasury — permanently lost to victim
// - victim's userClaims state is updated; they cannot reclaim the fee
``` [5](#0-4)

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
