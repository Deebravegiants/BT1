### Title
Unprivileged Actor Can Trigger Circuit-Breaker via Transient swETH Rate Dip, Temporarily Freezing All Deposits and Withdrawals — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is an unrestricted public function. Its internal circuit-breaker compares the freshly computed `newRsETHPrice` against `highestRsethPrice` (the all-time high). If the difference exceeds `pricePercentageLimit`, it immediately pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. Because `SwETHPriceOracle` passes through `swETH.getRate()` with no smoothing or staleness guard, any unprivileged caller who times their call to a block where swETH's rate has transiently dipped can fire the circuit-breaker and freeze the protocol until an admin manually unpauses.

---

### Finding Description

**Entry point — no access control:** [1](#0-0) 

`updateRSETHPrice()` is `public whenNotPaused` with no role restriction. Any EOA or contract can call it at any block.

**Oracle passes raw swETH rate with no smoothing:** [2](#0-1) 

`getAssetPrice()` returns `ISwETH(swETHAddress).getRate()` directly. swETH's `getRate()` is a spot value that can transiently dip during known rebase or rate-update windows.

**Circuit-breaker compares against all-time high, not previous price:** [3](#0-2) 

When `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit × highestRsethPrice`, the code calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` — all in the same transaction — then returns. There is no auto-unpause; recovery requires admin action.

**Asymmetric access control — price increase vs. price decrease:**

For a price *increase* above the threshold, a non-manager caller gets a revert (`PriceAboveDailyThreshold`): [4](#0-3) 

For a price *decrease* below the threshold, there is no such role check — the circuit-breaker fires unconditionally for any caller: [5](#0-4) 

This asymmetry is the root cause: the protocol protects against unprivileged actors pushing a high price through, but not against unprivileged actors triggering a downside pause.

**`highestRsethPrice` is the all-time high, never reset downward:** [6](#0-5) 

`highestRsethPrice` only ever increases. Over time, even a modest transient dip in swETH rate can push `newRsETHPrice` below `highestRsethPrice` by more than `pricePercentageLimit`, especially as the protocol accumulates yield and the all-time high grows.

---

### Impact Explanation

When the circuit-breaker fires:
- `LRTDepositPool` is paused → `depositETH` and `depositAsset` revert for all users.
- `LRTWithdrawalManager` is paused → `initiateWithdrawal` reverts for all users.
- `LRTOracle` itself is paused → `updateRSETHPrice()` is blocked until admin unpauses.

No funds are lost, but all deposits and withdrawals are frozen until an admin calls `unpause()` on each contract. This is **temporary freezing of funds** — a Medium-severity impact per the allowed scope.

---

### Likelihood Explanation

- swETH's `getRate()` is a spot value that can dip transiently during rebase windows (a documented property of swETH).
- `updateRSETHPrice()` requires zero privileges and zero capital.
- The attacker only needs to observe the swETH rate on-chain and submit a transaction at the right block — no front-running is required; any block where the rate is transiently low suffices.
- `pricePercentageLimit` is set by admin; even a 1% limit (`1e16`) is realistic and achievable by a transient dip.
- The attack is repeatable: after admin unpauses, the attacker can wait for the next rebase window and repeat.

---

### Recommendation

1. **Restrict `updateRSETHPrice()` to a privileged role** (e.g., `MANAGER` or a dedicated keeper role), or add a time-lock between successive public calls.
2. **Use a TWAP or rate-smoothing layer** in `SwETHPriceOracle` instead of passing `swETH.getRate()` raw.
3. **Add a role check before the downside pause**, mirroring the existing check for the upside case — only allow the circuit-breaker to fire when called by a manager, and emit an event (not a pause) for unprivileged callers.
4. **Consider comparing against `previousPrice` (not `highestRsethPrice`)** for the downside circuit-breaker, so that only a drop relative to the last recorded price — not the all-time high — triggers a pause.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Fork test at a historical block where swETH.getRate() was transiently low
// (e.g., immediately after a rebase event reduced the rate below the ATH by > pricePercentageLimit)

contract GriefTest is Test {
    LRTOracle oracle = LRTOracle(ORACLE_ADDR);
    LRTDepositPool pool = LRTDepositPool(POOL_ADDR);
    LRTWithdrawalManager wm = LRTWithdrawalManager(WM_ADDR);

    function testGrief() external {
        // Pre-condition: pricePercentageLimit is set (e.g. 1e16 = 1%)
        // Pre-condition: swETH.getRate() at this block is transiently below
        //                highestRsethPrice by > pricePercentageLimit

        assertFalse(pool.paused());
        assertFalse(wm.paused());

        // Unprivileged caller triggers the circuit-breaker
        vm.prank(address(1));
        oracle.updateRSETHPrice();

        // Protocol is now frozen
        assertTrue(pool.paused());
        assertTrue(wm.paused());
        assertTrue(oracle.paused());

        // Advance one block — rate recovers, but protocol stays paused (no auto-unpause)
        vm.roll(block.number + 1);
        assertTrue(pool.paused()); // still paused — requires admin action
    }
}
```

The test requires no special privileges, no capital, and no front-running — only knowledge of a block where swETH's spot rate is transiently below the protocol's all-time-high rsETH price by more than `pricePercentageLimit`.
