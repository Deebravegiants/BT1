### Title
Wallet Crash via Undersized NFT Memo in `get_metadata_and_phs` — (`chia/wallet/nft_wallet/nft_puzzle_utils.py`)

### Summary

`get_metadata_and_phs` unconditionally calls `bytes32(condition.at("rrrff").as_atom())` on the first memo of a `CREATE_COIN` (opcode 51, amount=1) condition. If the memo atom is fewer than 32 bytes, `bytes32()` raises `ValueError`. No length guard exists. Because CLVM does not enforce memo length, an NFT owner can submit a valid on-chain spend with a 31-byte memo, crashing any wallet that processes the spend via `handle_nft` or `puzzle_solution_received`.

### Impact Explanation

An attacker who owns an NFT can craft a spend bundle where the `CREATE_COIN` condition (amount=1) carries a 31-byte memo atom. The CLVM NFT puzzle does not enforce memo length — memos are wallet-layer hints — so the spend is consensus-valid and will be included in a block. The attacker sets the `CREATE_COIN` puzzle hash to a victim's known derivation address (puzzle hashes are public on-chain). The child NFT is created at the victim's address. When the victim's wallet detects the new coin and calls `handle_nft` → `get_metadata_and_phs` on the parent spend, `bytes32()` raises `ValueError`, crashing the wallet's sync path. With no exception guard, this corrupts the wallet's NFT tracking state and, depending on how far the exception propagates through the sync loop, can render the wallet unable to process further updates.

This maps to: **High — Corruption of wallet sync state / NFT tracking state with direct security impact**, and potentially **High — Long-lived inability for an honest wallet to process valid spend bundles**.

### Likelihood Explanation

- Requires only NFT ownership (no leaked keys, no admin access).
- Memo length is not enforced by consensus; the spend is unconditionally valid on-chain.
- Victim's puzzle hash is publicly observable.
- No existing guard in the code path.

### Recommendation

Add a length check before constructing `bytes32`:

```python
raw = condition.at("rrrff").as_atom()
if len(raw) != 32:
    continue   # or log and skip this condition
memo = bytes32(raw)
```

Wrap `handle_nft` and `puzzle_solution_received` call sites in `try/except Exception` so a malformed spend cannot crash the wallet's sync loop.
