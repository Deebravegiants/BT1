Audit Report

## Title
Missing Staleness Check in `ChainlinkPriceOracle.getAssetPrice` Enables Block-Stuffing-Triggered Spurious Auto-Pause — (`contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice` calls `latestRoundData()` but discards `updatedAt` and `answeredInRound`, accepting arbitrarily stale prices. A block-stuffing attacker can suppress Chainlink keeper updates long enough for the stale (lower) LST price to diverge from the real price by more than `pricePercentageLimit`. Any caller can then invoke the public `LRTOracle.updateRSETHPrice()`, causing the computed `newRsETHPrice` to fall below the threshold and unconditionally pausing `LRTDepositPool` and `LRTWithdrawalManager`.

## Finding Description

**Root cause:** `ChainlinkPriceOracle.getAssetPrice` (L49–55) ignores all staleness fields:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
``` [1](#0-0) 

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` (L27–32) correctly validates `answeredInRound < roundID` and `timestamp == 0`: [2](#0-1) 

The staleness guard exists in the codebase but was not applied to the oracle that feeds the rsETH price calculation.

**Public entry point:** `updateRSETHPrice()` is callable by any address: [3](#0-2) 

**TVL uses the stale price:** `_getTotalEthInProtocol` iterates all supported assets and calls `getAssetPrice(asset)`, which routes through the unchecked `ChainlinkPriceOracle`: [4](#0-3) 

**Auto-pause trigger:** If the stale-low price makes `newRsETHPrice` fall more than `pricePercentageLimit` below `highestRsethPrice`, the protocol pauses unconditionally — no admin action required: [5](#0-4) 

## Impact Explanation

**Low — Block stuffing.** All deposits and withdrawals are frozen until an admin manually unpauses. The pause is triggered by an artificial price artifact (block-stuffed stale data), not a genuine loss of value. The invariant that the auto-pause should only fire on real price drops is violated. This maps to the explicitly listed allowed impact: *Low. Block stuffing.*

## Likelihood Explanation

Block stuffing on Ethereum mainnet is expensive but has been executed in practice. The attacker must suppress Chainlink keeper updates long enough for the LST/ETH price to rise by more than `pricePercentageLimit` relative to the last on-chain update. LST prices accrue slowly (~4% APY), so the required stuffing window depends on how tightly `pricePercentageLimit` is configured. At 1% (1e16), the window is long and costly; at lower limits it becomes more feasible. The attacker gains no funds — only a temporary protocol freeze. The attack is repeatable after each admin unpause.

## Recommendation

Add a staleness check to `ChainlinkPriceOracle.getAssetPrice`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

if (answeredInRound < roundId) revert StalePrice();
if (updatedAt == 0) revert IncompleteRound();
if (block.timestamp - updatedAt > MAX_STALENESS) revert StalePrice();

return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`MAX_STALENESS` should be set per-feed based on the Chainlink heartbeat (e.g., 3600 seconds for a 1-hour heartbeat feed).

## Proof of Concept

```solidity
function test_blockStuffing_spuriousPause() external {
    // 1. Configure a 1% price drop limit
    lrtOracle.setPricePercentageLimit(1e16);

    // 2. Establish a peak price with a fresh Chainlink answer
    mockChainlink.setAnswer(1.05e18);
    lrtOracle.updateRSETHPrice();
    uint256 peak = lrtOracle.highestRsethPrice(); // 1.05e18

    // 3. Attacker stuffs blocks; Chainlink cannot post an update.
    //    Mock now returns the stale-low answer (>1% below peak).
    mockChainlink.setAnswer(1.03e18); // stale, ~1.9% below peak

    // 4. Any unprivileged caller triggers the price update
    vm.prank(address(0xdead));
    lrtOracle.updateRSETHPrice();

    // 5. Protocol is paused despite no real price drop
    assertTrue(lrtDepositPool.paused());
    assertTrue(withdrawalManager.paused());
}
```

The test passes on unmodified code, confirming the spurious pause is reachable without any privileged role.
