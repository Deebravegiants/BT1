Audit Report

## Title
Unvalidated Chainlink `int256` Price Cast to `uint256` Enables Temporary Fund Freeze - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` retrieves `int256 price` from Chainlink's `latestRoundData()` and casts it directly to `uint256` without a `price > 0` guard. A zero price causes division-by-zero in `LRTWithdrawalManager.getExpectedAssetAmount()`, freezing all withdrawal initiations for the affected asset, and causes `LRTOracle.updateRSETHPrice()` to compute an artificially low rsETH price that can trigger the automatic protocol-wide pause. A negative price wraps to near `type(uint256).max` and causes overflow reverts across all oracle-dependent paths under Solidity 0.8.x checked arithmetic.

## Finding Description
**Root cause — `contracts/oracles/ChainlinkPriceOracle.sol` L52–54:**
```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```
No `require(price > 0, ...)` exists before the cast. Chainlink's `latestRoundData()` returns `int256` and can legitimately return `0` (deprecated aggregator, circuit-breaker round) or a negative value.

**Downstream impact paths confirmed in code:**

**Path 1 — `price == 0` → withdrawal freeze (division by zero):**
`LRTWithdrawalManager.getExpectedAssetAmount()` at L593:
```solidity
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
```
`getAssetPrice(asset)` returns `0` → division by zero → revert. Every call to `initiateWithdrawal()` (L168) and `instantWithdrawal()` (L228) for the affected asset reverts, freezing all withdrawal initiations.

**Path 2 — `price == 0` → protocol-wide auto-pause:**
`LRTOracle._getTotalEthInProtocol()` at L339–343 sums `totalAssetAmt.mulWad(assetER)` for each asset. With `assetER = 0`, the affected asset's TVL contribution is omitted, causing `newRsETHPrice` to drop sharply. The publicly callable `updateRSETHPrice()` (L87) then hits the downside-protection check at L270–281:
```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```
This pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` simultaneously, freezing the entire protocol until an admin manually unpauses.

**Path 3 — `price < 0` → overflow revert across all paths:**
`uint256(price)` wraps to a value near `type(uint256).max`. The subsequent multiplication `uint256(price) * 1e18` overflows under Solidity 0.8.x checked arithmetic, causing every code path that calls `getAssetPrice` to revert.

**Existing checks are insufficient:**
`updatePriceOracleForValidated()` (L101–108 in `LRTOracle.sol`) validates the price at oracle setup time (`price > 1e16 && price < 1e19`), but this is a one-time check at configuration. It does not prevent the live feed from returning `0` or a negative value during normal operation. Additionally, `updatePriceOracleFor()` (L113) can be called directly by admin without any price validation.

## Impact Explanation
**Medium — Temporary freezing of funds.**

A zero price from any supported asset's Chainlink feed causes: (1) all withdrawal initiations for that asset to revert with division-by-zero, and (2) the publicly callable `updateRSETHPrice()` to trigger the automatic protocol-wide pause, locking both deposits and withdrawals until an admin manually unpauses. A negative price causes overflow reverts across all oracle-dependent paths, achieving the same freeze. Funds are not permanently lost — the freeze is lifted once the feed returns a valid price and an admin unpauses — but user access to funds is blocked for the duration.

## Likelihood Explanation
**Low-to-Medium.** Chainlink price feeds for major LSTs (stETH/ETH, ETHx/ETH) rarely return zero or negative values on mainnet. However, the condition is reachable in documented edge cases: a deprecated aggregator round or a Chainlink circuit-breaker activation. No attacker action is required; any unprivileged caller can trigger the protocol pause by calling the already-public `updateRSETHPrice()` after the feed returns an invalid value. The `pricePercentageLimit` must be set (non-zero) for the auto-pause path to trigger, but the withdrawal division-by-zero path is unconditional.

## Recommendation
Add a positivity check in `ChainlinkPriceOracle.getAssetPrice()` before casting:
```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
require(price > 0, "ChainlinkPriceOracle: invalid price");
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```
This ensures that a zero or negative Chainlink response reverts immediately at the oracle boundary rather than propagating silently into downstream arithmetic.

## Proof of Concept
1. Chainlink's `latestRoundData()` for a supported LST (e.g., stETH/ETH) returns `price = 0` (circuit-breaker or deprecated round).
2. `ChainlinkPriceOracle.getAssetPrice(stETH)` returns `0` — no revert, no check. (`ChainlinkPriceOracle.sol` L52–54)
3. Any user calls `LRTWithdrawalManager.initiateWithdrawal(stETH, amount, ...)`:
   - `getExpectedAssetAmount(stETH, amount)` → `amount * rsETHPrice / 0` → **revert** (division by zero). (`LRTWithdrawalManager.sol` L593) All stETH withdrawal initiations are frozen.
4. Any user calls `LRTOracle.updateRSETHPrice()` (public, no access control):
   - `_getTotalEthInProtocol()` returns TVL with stETH contribution = 0. (`LRTOracle.sol` L339–343)
   - `newRsETHPrice` drops sharply; if `pricePercentageLimit > 0`, `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` are called automatically. (`LRTOracle.sol` L277–281) **Entire protocol is temporarily frozen.**
