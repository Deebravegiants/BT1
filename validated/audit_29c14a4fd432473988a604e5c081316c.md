### Title
Missing Subgroup Check in `ecurve_ed_t::pub_from_der` Accepts Torsion Points as Valid Ed25519 Public Keys - (`src/cbmpc/crypto/base_eddsa.cpp`)

---

### Summary

`ecurve_ed_t::pub_from_der` validates the X.509 DER prefix then delegates entirely to `from_bin`, which only checks that the input bytes decode to a point on the curve. Neither function calls `is_in_subgroup()` or `curve.check()`. An attacker supplying a DER blob whose 32-byte payload encodes a torsion point (e.g., the order-2 point `(0, p-1)`) receives `SUCCESS` from `pub_from_der` and obtains an `ecc_pub_key_t` holding a torsion-contaminated point. Downstream `verify()` re-serializes that point and passes it directly to OpenSSL's `ED25519_verify`, which performs no subgroup check of its own, enabling signature forgery against the torsion key.

---

---

### Impact Explanation

Ed25519 has cofactor 8. The order-2 torsion point `T = (0, p-1)` is on the curve but not in the prime-order subgroup. OpenSSL's `ED25519_verify` uses the cofactor-less equation `[S]B = R + [H(R,A,M)]A`. With `A = T` (order 2), `[H]T` is either `T` or the identity depending on the parity of `H`. For any target message `M`:

1. Compute `H = H(R_candidate, T, M)`.
2. If `H` is even: pick any `S`, set `R = [S]B` — the equation `[S]B = R` holds trivially.
3. If `H` is odd: pick any `S`, set `R = [S]B − T`.

In both cases the attacker can produce a valid `(R, S)` pair for any message without knowledge of any private key. Any caller that parses an attacker-supplied DER blob via `pub_from_der` and then calls `verify` on the result accepts the forged signature.

---

### Likelihood Explanation

`ecc_pub_key_t::from_der` is declared in `include-internal/cbmpc/internal/crypto/base_ecc.h` and is part of the supported internal C++ API surface. [9](#0-8) 

Any caller that accepts an Ed25519 public key in DER/X.509 SubjectPublicKeyInfo format from an untrusted source (TLS peer certificate, key-exchange message, configuration input) and passes it to `pub_from_der` is directly exploitable. The attacker input is a single 44-byte blob (12-byte prefix + 32-byte torsion payload) that is trivially constructable.

---

### Recommendation

Add a subgroup and validity check immediately after `from_bin` succeeds in `pub_from_der`:

```cpp
error_t ecurve_ed_t::pub_from_der(ecc_pub_key_t& P, mem_t der) const {
  if (der.size != ed25519::x509_prefix.size + ed25519::pub_compressed_bin_size())
    return coinbase::error(E_FORMAT);
  if (0 != memcmp(ed25519::x509_prefix.data, der.data, ed25519::x509_prefix.size))
    return coinbase::error(E_FORMAT);
  error_t rv = from_bin(P, der.skip(ed25519::x509_prefix.size));
  if (rv) return rv;
  if (error_t rv2 = check(P)) return rv2;   // rejects infinity and non-subgroup points
  return SUCCESS;
}
```

This mirrors the pattern already used consistently in `eddsa2pc.cpp` and `eddsa_mp.cpp`.

### Proof of Concept

```cpp
// Compressed encoding of the Ed25519 order-2 torsion point (x=0, y=p-1):
// y = 2^255-20, little-endian, sign bit = 0
uint8_t order2[32];
order2[0] = 0xec;
for (int i = 1; i < 31; i++) order2[i] = 0xff;
order2[31] = 0x7f;

// Build DER blob: X.509 prefix || torsion payload
buf_t der(ed25519::x509_prefix.size + 32);
memmove(der.data(), ed25519::x509_prefix.data, ed25519::x509_prefix.size);
memmove(der.data() + ed25519::x509_prefix.size, order2, 32);

ecurve_ed_t ed_curve;
ecc_pub_key_t P(curve_ed25519.infinity());
error_t rv = ed_curve.pub_from_der(P, der);

// BUG: rv == SUCCESS, but P holds a torsion point
assert(rv == SUCCESS);           // passes — should have returned E_CRYPTO
assert(!P.is_in_subgroup());     // passes — confirms torsion point accepted
assert(ed_curve.check(P) != SUCCESS); // passes — curve.check would have caught it

// Forge a signature for any message against this torsion key:
// Pick S, compute R = [S]G, check H(R,T,M) parity, adjust R if needed.
// ED25519_verify(msg, sig=(R,S), pub=order2) returns 1.
```

  ecurve_t curve = crypto::curve_ed25519;

  // Compressed encoding of the Ed25519 order-2 point (x=0, y=-1):
  // y = p-1 = 2^255-20, sign bit = 0.
