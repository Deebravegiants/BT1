### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Redirect All Block Fees and Corrupt Block Hash — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates every field of `ProposalInit` that affects block execution except `builder`. The `builder` field is mapped directly to `sequencer_address` in `BlockInfo` and is then used as the fee-transfer recipient for every transaction in the block and as an input to the block hash. Any proposer can set `builder` to an arbitrary address; validators accept the proposal without objection, all fees are transferred to the attacker-controlled address, and the committed block hash encodes the wrong sequencer.

---

### Finding Description

`ProposalInit` carries a `builder` field (proto field 6) that identifies the address of the block builder/sequencer. [1](#0-0) 

`convert_to_sn_api_block_info` maps this field directly to `sequencer_address` in the `BlockInfo` that is handed to the batcher: [2](#0-1) 

That `BlockInfo` is forwarded to the batcher via `ValidateBlockInput` in `initiate_validation`: [3](#0-2) 

The batcher executes every transaction in the block with `sequencer_address = init.builder`. The blockifier's fee-transfer logic uses `block_context.block_info.sequencer_address` as the ERC-20 transfer recipient: [4](#0-3) 

The same `sequencer_address` is also committed into `PartialBlockHashComponents` and therefore into the final block hash: [5](#0-4) 

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices (within margin), `timestamp`, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`. It does **not** validate `builder`: [6](#0-5) 

`ProposalInitValidation` — the struct that carries the locally-trusted reference values — has no `builder` field at all: [7](#0-6) 

The final `ProposalFinMismatch` guard compares the batcher's computed `ProposalCommitment` against the proposer's `ProposalFin.proposal_commitment`: [8](#0-7) 

Because the validator's batcher executes with the same attacker-supplied `builder`, it produces the same `PartialBlockHash` the proposer already computed. The mismatch check therefore passes, and the proposal is accepted.

---

### Impact Explanation

**Fee theft (Critical — economic impact):** Every transaction fee in the block is transferred to `init.builder`. A malicious proposer sets `builder` to their own address and collects 100 % of block fees that should go to the legitimate sequencer.

**Wrong block hash / state commitment (Critical — wrong state):** `sequencer_address` is chained into `calculate_block_hash` via `PartialBlockHashComponents`. The committed block hash encodes the attacker's address, not the legitimate sequencer's address. Every downstream consumer of the block hash (L1 anchoring, sync, proving) receives a wrong value.

**Wrong `get_sequencer_address` syscall result (Critical — wrong execution result):** Contracts that call `get_sequencer_address` during the block receive the attacker's address. Any contract logic that branches on the sequencer address (e.g., fee-rebate contracts, governance contracts) executes incorrectly.

---

### Likelihood Explanation

The trigger requires no special privilege. Any validator that wins the proposer slot for a round — a normal, unprivileged network participant — can craft a `ProposalInit` with an arbitrary `builder`. The attack is fully self-consistent: the proposer computes the `ProposalFin` commitment using their chosen `builder`, the validator's batcher re-executes with the same `builder`, and the `ProposalFinMismatch` guard passes. No cryptographic material beyond the proposer's normal consensus key is needed.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`. The validator knows its own `builder_address` from static configuration; it should reject any proposal whose `init.builder` differs from the locally configured value:

```rust
// In ProposalInitValidation:
pub builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder
        ),
    ));
}
```

Populate `proposal_init_validation.builder` from `self.config.static_config.builder_address` at the call site in `validate_proposal` (analogous to how `l1_da_mode` and `l2_gas_price_fri` are populated from local config): [9](#0-8) 

---

### Proof of Concept

1. Attacker is selected as proposer for round R at height H.
2. Attacker constructs `ProposalInit` with all fields matching what validators expect **except** `builder = attacker_wallet_address`.
3. Attacker calls `build_proposal` locally with `builder_address = attacker_wallet_address`, producing a valid `PartialBlockHash` that encodes `attacker_wallet_address` as sequencer.
4. Attacker streams `ProposalPart::Init(init)` → `ProposalPart::Transactions(...)` → `ProposalPart::Fin(commitment_built_with_attacker_builder)` to the network.
5. Each validator calls `is_proposal_init_valid`: all checked fields pass; `builder` is not checked.
6. Each validator's batcher executes the block with `sequencer_address = attacker_wallet_address`; all fees are transferred to that address.
7. Each validator's batcher computes `PartialBlockHash` with `attacker_wallet_address` as sequencer; `proposal_commitment_from(partial, fee_proposal)` matches the proposer's `ProposalFin.proposal_commitment`.
8. `ProposalFinMismatch` check passes; proposal is accepted by consensus.
9. `decision_reached` commits the block. The block hash encodes `attacker_wallet_address` as sequencer. All fees for the block are in the attacker's account. [10](#0-9) [11](#0-10) [12](#0-11) [13](#0-12)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
