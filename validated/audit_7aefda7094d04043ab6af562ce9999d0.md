### Title
Metadata Signature Lacks Chain-ID Binding in `deployToken()`, Enabling Cross-Chain Replay — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

The `deployToken()` function in `OmniBridge.sol` verifies a NEAR-MPC-signed metadata payload that contains no destination chain identifier. Because the same `nearBridgeDerivedAddress` is used across all EVM deployments, any valid `deployToken()` signature observed on one EVM chain (e.g., Ethereum) can be replayed verbatim on every other EVM chain (e.g., Base, Arbitrum) to deploy an unauthorized wrapped token without NEAR explicitly authorizing that deployment on the target chain.

---

### Finding Description

`finTransfer()` correctly binds its signed payload to the destination chain by embedding `omniBridgeChainId` twice in the Borsh-encoded message: [1](#0-0) 

`deployToken()` constructs its signed payload with no chain identifier at all: [2](#0-1) 

The payload is only `PayloadType.Metadata | token | name | symbol | decimals`. Because the NEAR MPC key (`nearBridgeDerivedAddress`) is shared across all EVM deployments, the resulting ECDSA signature is equally valid on every chain that runs `OmniBridge`. The only replay guard inside `deployToken()` is the existence check: [3](#0-2) 

This check is per-chain state; it does not prevent the same signature from being submitted on a different chain where the token has not yet been deployed.

---

### Impact Explanation

An attacker who observes a successful `deployToken()` transaction on Chain A can extract the `(signatureData, metadata)` calldata and submit it to `OmniBridge` on Chain B. This:

1. Deploys a wrapped token on Chain B with the correct metadata, owned by the bridge, before NEAR has authorized or configured that chain for the token.
2. Permanently blocks the protocol from legitimately deploying the same token on Chain B later (`ERR_TOKEN_EXIST` will revert every future attempt).
3. Emits a `DeployToken` event on Chain B that NEAR's indexer may ingest, potentially causing NEAR to register the token for Chain B and begin signing `finTransfer` payloads for it — creating an unbacked wrapped supply path.

This falls under **Critical — unauthorized creation of wrapped bridge assets through deployment/verification failure** and **High — acceptance of insufficiently-bound signatures that bypass execution gates**.

---

### Likelihood Explanation

- `deployToken()` is a public, permissionless function — no role or stake required.
- The calldata (signature + metadata) is fully visible on-chain the moment it is used on Chain A.
- Replaying it on Chain B requires only copying the calldata and submitting a transaction; no cryptographic capability is needed.
- Multiple EVM chains (Ethereum, Base, Arbitrum, etc.) are all in scope and all share the same `nearBridgeDerivedAddress`.

Likelihood is **High**.

---

### Recommendation

Bind the metadata signature to the destination chain by including `omniBridgeChainId` in the Borsh-encoded payload that is hashed and signed, mirroring the pattern already used in `finTransfer()`:

```solidity
bytes memory borshEncoded = bytes.concat(
    bytes1(uint8(BridgeTypes.PayloadType.Metadata)),
+   bytes1(omniBridgeChainId),          // chain binding
    Borsh.encodeString(metadata.token),
    Borsh.encodeString(metadata.name),
    Borsh.encodeString(metadata.symbol),
    bytes1(metadata.decimals)
);
```

The NEAR MPC signing logic must be updated correspondingly to include the target chain ID when producing metadata signatures.

---

### Proof of Concept

1. NEAR MPC signs a metadata payload for token `"foo.near"` (name, symbol, decimals) — no chain ID in the message.
2. A relayer calls `OmniBridge.deployToken(sig, metadata)` on Ethereum. Transaction is public.
3. Attacker copies `sig` and `metadata` from the Ethereum transaction.
4. Attacker calls `OmniBridge.deployToken(sig, metadata)` on Base (same `nearBridgeDerivedAddress`, same `OmniBridge` contract).
5. `ECDSA.recover(keccak256(borshEncoded), sig) == nearBridgeDerivedAddress` passes — identical payload, identical signer.
6. `isBridgeToken[nearToEthToken["foo.near"]]` is `false` on Base — existence check passes.
7. A new `BridgeToken` proxy for `"foo.near"` is deployed on Base, `DeployToken` event is emitted, and the token is registered in Base's bridge mappings — without NEAR ever authorizing a Base deployment.
8. Any future legitimate `deployToken()` call for `"foo.near"` on Base reverts with `ERR_TOKEN_EXIST`. [4](#0-3)

```text
    function deployToken(
        bytes calldata signatureData,
        BridgeTypes.MetadataPayload calldata metadata
    ) external payable whenNotPaused(PAUSED_DEPLOY_TOKEN) returns (address) {
        if (tokenImplementationAddress == address(0)) {
            revert TokenImplementationNotSet();
        }
        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.Metadata)),
            Borsh.encodeString(metadata.token),
            Borsh.encodeString(metadata.name),
            Borsh.encodeString(metadata.symbol),
            bytes1(metadata.decimals)
        );
        bytes32 hashed = keccak256(borshEncoded);

        if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
            revert InvalidSignature();
        }

        require(
            !isBridgeToken[nearToEthToken[metadata.token]],
            "ERR_TOKEN_EXIST"
        );
        uint8 decimals = _normalizeDecimals(metadata.decimals);

        // slither-disable-next-line reentrancy-no-eth
        address bridgeTokenProxy = address(
            new ERC1967Proxy(
                tokenImplementationAddress,
                abi.encodeWithSelector(
                    BridgeToken.initialize.selector,
                    metadata.name,
                    metadata.symbol,
                    decimals
                )
            )
        );

        deployTokenExtension(
            metadata.token,
            bridgeTokenProxy,
            decimals,
            metadata.decimals
        );

        emit BridgeTypes.DeployToken(
            bridgeTokenProxy,
            metadata.token,
            metadata.name,
            metadata.symbol,
            decimals,
            metadata.decimals
        );

        isBridgeToken[address(bridgeTokenProxy)] = true;
        ethToNearToken[address(bridgeTokenProxy)] = metadata.token;
        nearToEthToken[metadata.token] = address(bridgeTokenProxy);

        return bridgeTokenProxy;
    }
```
