### Title
Unauthenticated Indexer Height Inflation Enables Byzantine Participant to Monopolize Leader Selection - (File: crates/node/src/network/indexer_heights.rs, crates/node/src/requests/queue.rs)

### Summary
A single Byzantine participant (strictly below the signing threshold) can broadcast a fabricated `IndexerHeightMessage` with an arbitrarily large block height over the authenticated P2P mesh. Because `IndexerHeightTracker::set_height` applies no upper-bound validation and the value is monotonically locked in, every honest node's `eligible_leaders_and_maximum_height` computation will set `maximum_height` to the fake value. All honest nodes then fail the `STALE_PARTICIPANT_THRESHOLD = 10` filter and are excluded from the eligible-leader set, leaving the Byzantine node as the sole leader for every pending signature, CKD, and foreign-tx-verification request.

### Finding Description

**Root cause — no validation of peer-supplied indexer height**

`IndexerHeightTracker::set_height` accepts any `u64` from any authenticated peer without checking whether the value is plausible relative to the current chain tip: [1](#0-0) 

The value is stored permanently (it only ever increases) and is immediately visible to the leader-selection logic.

**Delivery path — unauthenticated height claim over TLS P2P**

When the receive loop processes an `IndexerHeight` packet it calls `set_height` directly with the peer-supplied value and no further checks: [2](#0-1) 

Any registered participant can send this packet; the TLS handshake only proves identity, not height validity.

**Effect on leader selection — `STALE_PARTICIPANT_THRESHOLD = 10`**

Every second, `get_requests_to_attempt` calls `eligible_leaders_and_maximum_height`: [3](#0-2) 

`maximum_height` is the maximum height among alive participants. If the Byzantine node has injected `fake_height = real_height + 1000`, then every honest node whose actual height is `real_height` satisfies `real_height + 10 < fake_height`, so it is excluded from `eligible_leaders`. Only the Byzantine node passes the filter.

**Main loop — called on every iteration**

`get_requests_to_attempt` is invoked unconditionally in the `monitor_block_updates` loop for signatures, CKDs, and foreign-tx verifications: [4](#0-3) [5](#0-4) [6](#0-5) 

**Threshold constant** [7](#0-6) 

A gap of just 11 blocks above the honest cluster's real height is sufficient to exclude all honest nodes.

### Impact Explanation

Once all honest nodes are excluded from `eligible_leaders`, they each return `RequestStatus::Wait("no eligible leaders for this request")` or `RequestStatus::Wait("we are not leader")` for every queued request. The Byzantine node is the only node that will attempt to lead signing rounds. If it selectively refuses to initiate protocols, all pending signature requests time out after `REQUEST_EXPIRATION_BLOCKS = 200` blocks, breaking the liveness of the chain-signature service and causing users' yield-resume callbacks to expire. This is a participant-state manipulation that breaks the request-lifecycle safety invariant without requiring network-level DoS. [8](#0-7) 

### Likelihood Explanation

Any single registered MPC participant can execute this attack by sending one `IndexerHeightMessage` packet per connection. The TLS layer authenticates the sender but does not constrain the height value. The stored height never decreases, so a single inflated message permanently poisons the tracker until the node restarts. No threshold collusion, key material, or privileged access is required.

### Recommendation

1. **Bound incoming heights**: In `set_height`, reject any peer-supplied height that exceeds the node's own locally observed NEAR chain tip by more than a small configurable margin (e.g., `MAX_INDEXER_HEIGHT_DIFF`).
2. **Use local height as the reference**: Compute `maximum_height` in `eligible_leaders_and_maximum_height` relative to the node's own locally indexed height rather than trusting peer-reported values as the ceiling.
3. **Decay stale heights**: If a peer's reported height has not been updated within a timeout window, reset it to zero rather than keeping the last-seen maximum.

### Proof of Concept

1. Byzantine participant `B` (one of N registered participants, below threshold) establishes a normal TLS connection to all honest nodes.
2. `B` sends `IndexerHeightMessage { height: current_near_height + 1000 }` to every peer.
3. Each honest node's `IndexerHeightTracker` stores `B`'s height as `current_near_height + 1000`.
4. On the next call to `eligible_leaders_and_maximum_height` (within 1 second), `maximum_height = current_near_height + 1000`.
5. Every honest node has `own_height + 10 < current_near_height + 1000`, so it is excluded from `eligible_leaders`.
6. `B` is the only entry in `eligible_leaders` on every honest node.
7. `B` refuses to initiate any signing protocol. All pending signature requests accumulate until they exceed `REQUEST_EXPIRATION_BLOCKS = 200` blocks and are dropped with `DropReason::RequestTimedOut`.
8. Users' `sign()` yield-resume callbacks expire; the chain-signature service is effectively frozen for the duration of the attack.

    pub(super) fn eligible_leaders_and_maximum_height(&self) -> (HashSet<ParticipantId>, u64) {
        // Collect the indexer heights and alive participants. Calculate maximum available height
        // from the alive nodes. Then, filter out the participants that are not alive or are too
        // stale.
        let indexer_heights = self.network_api.indexer_heights();
        let alive_participants = self.network_api.alive_participants();
        let maximum_height = alive_participants
            .iter()
            .map(|p| indexer_heights.get(p).copied().unwrap_or(0))
            .max()
            .unwrap_or(0);
        let eligible_leaders = self
            .all_participants
            .iter()
            .filter(|p| {
                alive_participants.contains(p)
                    && indexer_heights.get(p).copied().unwrap_or(0) + STALE_PARTICIPANT_THRESHOLD
                        >= maximum_height
            })
            .copied()
            .collect::<HashSet<_>>();
        (eligible_leaders, maximum_height)
    }
