### Title
Gateway L2 Gas Price Admission Check Uses Stale Current-Block Price Instead of Next-Block Price — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` validates a transaction's L2 gas `max_price_per_unit` against the **current committed block's** `l2_gas_price`, but the transaction will be executed in the **next block** at `next_l2_gas_price`. This is the direct Sequencer analog of the external report's "wrong price used for post-trade margin check": the admission decision is made with a price that differs from the price the execution engine will enforce.

---

### Finding Description

In `validate_resource_bounds`, the gateway reads the reference price as:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
```

This is `block_N.l2_gas_price` — the price of the **already-committed** block. [1](#0-0) 

The admission threshold applied is:

```
tx.max_price_per_unit  >=  (min_gas_price_percentage / 100) * block_N.l2_gas_price
``` [2](#0-1) 

However, the transaction will be included in block N+1, whose L2 gas price is `block_N.next_l2_gas_price` — computed via EIP-1559 from block N's actual gas usage. The `BlockHeaderWithoutHash` struct carries this field:

```rust
pub next_l2_gas_price: GasPrice,
```

but `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` maps only `block_header.l2_gas_price` into the returned `BlockInfo`; `next_l2_gas_price` is silently dropped. [3](#0-2) 

The blockifier's `check_fee_bounds`, which runs during actual execution, enforces:

```
tx.max_price_per_unit  >=  block_N+1.l2_gas_price   (= block_N.next_l2_gas_price)
``` [4](#0-3) 

The two prices diverge whenever block N's gas usage differs from the EIP-1559 target. `calculate_next_base_gas_price` adjusts the price up or down by `price * |gas_used - gas_target| / (gas_target * denominator)` per block. [5](#0-4) 

The developer acknowledged the discrepancy with an inline TODO: `// TODO(Arni): getnext_l2_gas_price from the block header.` [6](#0-5) 

---

### Impact Explanation

**Case 1 — price rising (high congestion, `next_l2_gas_price > block_N.l2_gas_price`):**

A transaction with `max_price_per_unit` in the range `[threshold, next_l2_gas_price)` passes the gateway admission check but will be rejected by the blockifier with `MaxGasPriceTooLow`. The gateway admits an invalid transaction that cannot be sequenced.

**Case 2 — price falling (low congestion, `next_l2_gas_price < block_N.l2_gas_price`):**

A transaction with `max_price_per_unit` in the range `[next_l2_gas_price, threshold)` is rejected at the gateway even though the blockifier would accept it. A valid transaction is incorrectly refused.

Both cases match the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The EIP-1559 mechanism bounds the per-block price change to `price / gas_price_max_change_denominator`, so the gap between `block_N.l2_gas_price` and `block_N.next_l2_gas_price` is bounded. However, during sustained high or low congestion the gap accumulates across blocks and can become material. With `min_gas_price_percentage = 100` (full enforcement), every block where gas usage deviates from target creates a window of incorrectly admitted or rejected transactions. The condition is reachable by any user submitting a transaction during normal network operation — no privileged access is required.

---

### Recommendation

Expose `next_l2_gas_price` through `GatewayFixedBlockStateReader` (or a new dedicated accessor) and replace the reference price in `validate_resource_bounds` with `block_N.next_l2_gas_price`. The `BlockHeaderWithoutHash` already carries this field; `GatewayFixedBlockSyncStateClient` simply needs to surface it. [7](#0-6) 

---

### Proof of Concept

1. Observe that `calculate_next_l2_gas_price_for_fin` is called with `l2_gas_used = max_block_size` (full block), producing `next_l2_gas_price > current_l2_gas_price`. [8](#0-7) 

2. Submit a transaction with `max_price_per_unit = current_l2_gas_price` (exactly at the 100% threshold). The gateway computes `threshold = 1.0 * current_l2_gas_price` and admits the transaction. [2](#0-1) 

3. The batcher includes the transaction in block N+1 at `next_l2_gas_price > current_l2_gas_price`. The blockifier's `check_fee_bounds` finds `max_price_per_unit < next_l2_gas_price` and raises `MaxGasPriceTooLow`, reverting or dropping the transaction. [9](#0-8) 

4. The gateway has admitted a transaction that the execution engine rejects — the admission decision was made against the wrong (stale) price, exactly mirroring the external report's "wrong mark price used for margin check" pattern.

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
