### Title
Expired TEE Attestation Accepted for Signing and Key-Event Operations — (`crates/contract/src/tee/tee_state.rs`)

---

### Summary

`is_caller_an_attested_participant()` checks only that a stored attestation *exists* for the caller's TLS key; it never re-verifies whether that attestation has since expired or been invalidated by a docker-image / measurement whitelist change. Every critical node-facing method — `respond`, `respond_ckd`, `respond_verify_foreign_tx`, `vote_pk`, `vote_reshared`, `start_reshare_instance`, and `vote_abort_key_event_instance` — gates access through this stale check. A node whose TEE certificate has expired can therefore continue to submit signatures and drive key-event state transitions, bypassing the TEE enforcement invariant.

---

### Finding Description

`assert_caller_is_attested_participant_and_protocol_active()` delegates to `is_caller_an_attested_participant()`: [1](#0-0) 

`is_caller_an_attested_participant()` performs three checks — participant membership, attestation presence, and account-key binding — but **never calls `reverify_participants()`**: [2](#0-1) 

By contrast, `reverify_participants()` does invoke `re_verify()` with the current block timestamp, docker-image whitelist, and measurement whitelist: [3](#0-2) 

`reverify_and_cleanup_participants()` (which calls `reverify_participants()`) is invoked only in `verify_tee()` and `vote_new_parameters()`: [4](#0-3) [5](#0-4) 

The secondary guard `accept_requests` is set to `false` only when `verify_tee()` is explicitly called and finds fewer than threshold valid attestations: [6](#0-5) 

Between the moment an attestation expires and the moment a participant calls `verify_tee()`, `accept_requests` remains `true` and `is_caller_an_attested_participant()` still returns `Ok(())` for the expired node. All of the following methods are therefore reachable by a node with a stale attestation: [7](#0-6) [8](#0-7) [9](#0-8) 

---

### Impact Explanation

The TEE attestation is the mechanism that guarantees a node is running approved software inside a verified enclave. An expired attestation means that guarantee has lapsed — the node may be running a non-approved binary that could exfiltrate its key share or sign arbitrary payloads. Allowing such a node to call `respond()` and the key-event vote methods breaks the production safety invariant that **only currently-attested TEE nodes may participate in threshold signing and key-event state transitions**. This maps to the Medium allowed impact: *participant-state or contract execution-flow manipulation that breaks production safety/accounting invariants*.

---

### Likelihood Explanation

Attestations carry an explicit `expiry_timestamp_seconds` field and expire in normal operation. The `verify_tee()` call is not automatic — it must be triggered by a participant. Any gap between expiry and the next `verify_tee()` invocation is an open window. Because the expired node itself has no incentive to trigger `verify_tee()`, the window can persist indefinitely if other participants are slow or offline.

---

### Recommendation

`is_caller_an_attested_participant()` should call `reverify_participants()` (or an equivalent inline check) before returning `Ok(())`, analogous to how `reverify_and_cleanup_participants()` is already used in `verify_tee()` and `vote_new_parameters()`. Concretely:

```diff
 pub(crate) fn is_caller_an_attested_participant(
     &self,
     participants: &Participants,
+    tee_upgrade_deadline_duration: Duration,
 ) -> Result<(), AttestationCheckError> {
     // ... existing membership + key-binding checks ...

+    // Re-verify expiry and whitelist membership at call time.
+    let node_id = &attestation.node_id;
+    match self.reverify_participants(node_id, tee_upgrade_deadline_duration) {
+        TeeQuoteStatus::Valid => {}
+        TeeQuoteStatus::Invalid(reason) => {
+            return Err(AttestationCheckError::AttestationExpiredOrInvalid(reason));
+        }
+    }

     Ok(())
 }
```

`assert_caller_is_attested_participant_and_protocol_active()` would pass `self.config.tee_upgrade_deadline_duration_seconds` (already available at the call sites) to this function.

---

### Proof of Concept

1. Node A submits a valid attestation with `expiry_timestamp_seconds = T`.
2. Block time advances past `T`; no participant calls `verify_tee()`.
3. `accept_requests` remains `true`; `stored_attestations` still contains Node A's entry.
4. Node A calls `respond(request, signature)`.
5. `assert_caller_is_attested_participant_and_protocol_active()` → `is_caller_an_attested_participant()` finds the entry, checks account-key binding, returns `Ok(())` — **no expiry check performed**.
6. The contract accepts the response and resolves the pending yield, despite Node A's TEE guarantee having lapsed.

The same path applies to `vote_pk()` and `vote_reshared()`, allowing an expired-attestation node to drive DKG and resharing state transitions.

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
