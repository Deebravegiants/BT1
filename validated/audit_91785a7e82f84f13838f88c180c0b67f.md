### Title
Unvalidated `ProposalInit.builder` Field Allows Any Proposer to Redirect All Block Fee Revenue to an Arbitrary Address — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, `crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`ProposalInit` carries a `builder` field that is directly mapped to `BlockInfo.sequencer_address` — the address that receives every transaction fee in the block. The proposal-init validation function `is_proposal_init_valid` never checks `builder` against the locally-configured expected value. Any legitimate proposer can therefore set `builder` to an arbitrary address, redirecting all block fees to themselves while producing a `ProposalCommitment` that validators accept without objection.

---

### Finding Description

`ProposalInit` contains a `builder` field documented as "Address of the one who builds/sequences the block." [1](#0-0) 

`convert_to_sn_api_block_info` maps this field directly to `sequencer_address` in the `starknet_api::block::BlockInfo` that is handed to the batcher for both block building and block validation: [2](#0-1) 

`sequencer_address` is the recipient of every fee transfer executed by the blockifier: [3](#0-2) 

The validator node calls `is_proposal_init_valid` before forwarding the proposal to the batcher. That function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`. It does **not** check `builder`: [4](#0-3) 

The `ProposalInitValidation` struct that carries the locally-trusted reference values has no `builder` field at all: [5](#0-4) 

The proposer sets `builder` from its own local config (`args.builder_address`): [6](#0-5) 

Because the batcher uses the proposer-supplied `builder` as `sequencer_address` for both the proposer's own execution and the validator's re-execution, the resulting `ProposalCommitment` (partial block hash) is identical on both sides. The `ProposalFinMismatch` guard therefore passes: [7](#0-6) 

The block hash itself commits `sequencer_address` as a first-class field: [8](#0-7) 

---

### Impact Explanation

**Critical — Wrong state, receipt, event, and storage value; incorrect fee and balance effect with economic impact.**

1. **Fee theft**: Every transaction fee in the block is transferred to the attacker-controlled address instead of the legitimate sequencer. The `sequencer_address` field in `BlockInfo` is the sole recipient of all `execute_fee_transfer` calls.
2. **Wrong block hash committed to the chain**: `sequencer_address` is hashed into the partial block hash and ultimately into the Starknet block hash. A block committed with a forged `builder` produces a permanently incorrect block hash, corrupting the chain's state commitment.
3. **Wrong execution receipts and events**: Every fee-transfer event emitted during block execution records the attacker's address as the recipient, producing incorrect on-chain receipts.
4. **No detection by existing guards**: Because both the proposer and the validator derive `sequencer_address` from the same `init.builder` value, the `ProposalCommitment` computed by the batcher matches the `ProposalFin` commitment, so `ProposalFinMismatch` is never triggered.

---

### Likelihood Explanation

**High.** The trigger requires only that the attacker win a single proposer election — a normal, unprivileged event for any validator in the committee. No admin key, no special configuration, and no out-of-band access is needed. The attack is invisible to all current validation checks and produces a block that every honest validator will prevote and precommit for.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`:

```rust
// In ProposalInitValidation:
pub expected_builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

`expected_builder` should be populated from `ContextStaticConfig.builder_address` — the same value the proposer uses when constructing `ProposalInit`: [9](#0-8) 

---

### Proof of Concept

1. A malicious validator wins the proposer election for height H, round R.
2. In `initiate_build`, it overrides `builder` with `attacker_address` before calling `propose_block`.
3. The batcher builds the block with `sequencer_address = attacker_address`; all fee transfers go to `attacker_address`.
4. The proposer broadcasts `ProposalInit { builder: attacker_address, ... }` followed by the transaction batches and `ProposalFin { proposal_commitment: C }`.
5. Each validator node calls `validate_proposal` → `is_proposal_init_valid` (passes — `builder` is not checked) → `initiate_validation` → `convert_to_sn_api_block_info(init)` → `sequencer_address = attacker_address`.
6. The batcher re-executes the block with `sequencer_address = attacker_address`, producing the same `ProposalCommitment C`.
7. `built_block == received_fin.proposal_commitment` → `Ok(C)` — no mismatch detected.
8. Consensus reaches decision on the block; `decision_reached` commits it to storage with `sequencer_address = attacker_address` permanently encoded in the block hash. [10](#0-9) [11](#0-10)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
