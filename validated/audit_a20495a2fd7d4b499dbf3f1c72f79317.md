Audit Report

## Title
Unprivileged Caller Can Trigger Protocol-Wide Pause via Public `updateRSETHPrice()` - (File: contracts/LRTOracle.sol)

## Summary

`updateRSETHPrice()` is declared `public` with only a `whenNotPaused` modifier, allowing any address to invoke it. Its internal logic contains a downside-protection branch that, when the computed rsETH price drops more than `pricePercentageLimit` below `highestRsethPrice`, atomically pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle`. Because the trigger condition can be satisfied by natural oracle price fluctuations, an unprivileged attacker can freeze all user deposits and withdrawals with a single zero-value transaction.

## Finding Description

`updateRSETHPrice()` at line 87 of `contracts/LRTOracle.sol` carries no role restriction:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, the downside-protection branch at lines 270–282 fires unconditionally for any caller when the price condition is met:

```solidity
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;
    }
}
```

The asymmetry with the upside branch (lines 252–266) is telling: when the price rises above the threshold, an unprivileged caller receives a `PriceAboveDailyThreshold` revert. When the price falls below the threshold, the same unprivileged caller instead triggers a full protocol pause. `highestRsethPrice` is a monotonically increasing watermark (lines 293–296), so as the protocol accumulates staking rewards, the watermark grows and any temporary dip in LST oracle prices can satisfy the condition without any oracle manipulation.

Existing guards are insufficient: `whenNotPaused` only prevents calls when already paused; it does not restrict who may call the function. `updateRSETHPriceAsManager()` (line 94) demonstrates the developers knew how to restrict callers but did not apply that restriction to the public variant.

## Impact Explanation

When the branch fires, three contracts are paused atomically. `LRTDepositPool.depositETH()` and `depositAsset()` both carry `whenNotPaused` (lines 83, 107), blocking all new deposits. `LRTWithdrawalManager.initiateWithdrawal()`, `completeWithdrawal()`, and `instantWithdrawal()` all carry `whenNotPaused` (lines 158, 183, 219), blocking all withdrawal initiation and completion. Unpausing each contract requires a separate `onlyLRTAdmin` call: `LRTDepositPool.unpause()` (line 354), `LRTWithdrawalManager.unpause()` (line 352), and `LRTOracle.unpause()` (line 143). Until those three admin transactions are confirmed on-chain, all user funds in transit are frozen.

**Impact: Medium — Temporary freezing of funds.**

## Likelihood Explanation

The preconditions are realistic: `pricePercentageLimit` must be non-zero (the feature exists to be used and is configurable by admin via `setPricePercentageLimit`), and the oracle-reported rsETH price must temporarily dip below `highestRsethPrice` by more than the configured threshold. LST prices fluctuate due to Chainlink update cadence, brief depegs, or normal market volatility. The attacker requires no capital, no privileged role, and no oracle manipulation — only the ability to observe public on-chain state and submit a single transaction. The attack is repeatable: after admin unpauses, the attacker can trigger the pause again at the next qualifying price moment.

## Recommendation

Apply the same role restriction used by `updateRSETHPriceAsManager()` to the public entry point, or introduce a dedicated `KEEPER_ROLE` for price updates. Alternatively, separate the pause-triggering path from the public price-read path: keep a view-only public function for reading the price, and gate the state-changing (and pause-triggering) path behind a role check. The upside branch already demonstrates the correct pattern — replicate it for the downside branch by reverting for unprivileged callers instead of pausing.

## Proof of Concept

1. Protocol has been running; `highestRsethPrice = 1.05e18`. Admin sets `pricePercentageLimit = 1e16` (1%).
2. A Chainlink stETH/ETH price update causes `_getTotalEthInProtocol()` to return a value such that `newRsETHPrice = 1.034e18` (~1.5% below the watermark).
3. Attacker calls `LRTOracle.updateRSETHPrice()` — no role, no ETH value.
4. `diff = 1.05e18 - 1.034e18 = 0.016e18`; `pricePercentageLimit.mulWad(highestRsethPrice) = 0.01 * 1.05e18 = 0.0105e18`; `diff > threshold` → `isPriceDecreaseOffLimit = true`.
5. `lrtDepositPool.pause()`, `withdrawalManager.pause()`, `_pause()` execute.
6. All calls to `depositETH`, `depositAsset`, `initiateWithdrawal`, `completeWithdrawal`, and `instantWithdrawal` revert with `Paused` until admin manually unpauses all three contracts.

**Foundry test sketch:**
