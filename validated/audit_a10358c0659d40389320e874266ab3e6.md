### Title
Unvalidated `ProposalInit.builder` Field Flows Into Block Hash and Execution Context as Unchecked `sequencer_address` — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`ProposalInit` carries a `builder: ContractAddress` field that is accepted from the network and passed verbatim into `BlockInfo.sequencer_address` via `convert_to_sn_api_block_info`. This address is then committed into `PartialBlockHashComponents.sequencer`, which is hashed into the `PartialBlockHash` / `ProposalCommitment`, and is also visible to every contract via the `get_execution_info()` syscall. The `is_proposal_init_valid` function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri` — but never checks `builder`. Any legitimate proposer for a round can set `builder` to an arbitrary address and have the proposal accepted by all validators.

### Finding Description

`ProposalInit.builder` is defined as "Address of the one who builds/sequences the block." [1](#0-0) 

When building a proposal, the proposer sets this from its own static config: [2](#0-1) 

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in `BlockInfo`: [3](#0-2) 

This `BlockInfo` is forwarded to the batcher in both the build path (`propose_block`) and the validate path (`validate_block`): [4](#0-3) 

Inside the batcher, `PartialBlockHashComponents::new` reads `block_info.sequencer_address` and stores it as the `sequencer` field: [5](#0-4) 

`calculate_block_hash` chains `sequencer` into the Poseidon hash that becomes the final block hash: [6](#0-5) 

`is_proposal_init_valid` — the sole gate for `ProposalInit` content — checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri`, but contains **no check on `builder`**: [7](#0-6) 

`ProposalInitValidation` — the struct that carries the locally-trusted reference values — has no `builder_address` field at all: [8](#0-7) 

The consensus manager's `handle_proposal` only verifies `init.proposer` against the expected proposer; `init.builder` is never inspected: [9](#0-8) 

### Impact Explanation

A legitimate proposer for any round can set `init.builder` to an arbitrary `ContractAddress`. All validators call `convert_to_sn_api_block_info(init)` with the same manipulated `init`, so they all compute the same `PartialBlockHash` and agree on the same `ProposalCommitment`. Consensus reaches a decision normally. The committed block then carries:

1. **Wrong block hash** — `sequencer_address` is one of the fourteen fields hashed by `calculate_block_hash`. The on-chain block hash is permanently wrong relative to the expected builder address. This is a wrong state commitment.
2. **Wrong execution context** — every transaction executed in the block sees the manipulated address via `get_execution_info().block_info.sequencer_address`. Contracts that branch on the sequencer address (e.g., fee-token contracts, access-control logic) execute with the wrong identity.
3. **Fee redirection** — sequencer fees collected during execution are credited to `sequencer_address`. A proposer who sets `builder` to their own address or a contract they control diverts all sequencer fees for that block.

### Likelihood Explanation

In a rotating-proposer validator set, every validator becomes the legitimate proposer for some round. No special privilege beyond being a scheduled proposer is required. The trigger is a single-field modification in the outbound `ProposalInit` before it is streamed to peers. No existing guard in `is_proposal_init_valid`, `handle_proposal`, or `initiate_validation` rejects a mismatched `builder`.

### Recommendation

Add `builder_address: ContractAddress` to `ProposalInitValidation` and populate it from `self.config.static_config.builder_address` at the call sites in `validate_proposal` (both the `Ordering::Equal` branch and the queued-proposal path in `set_height_and_round`). In `is_proposal_init_valid`, reject any `init_proposed.builder != proposal_init_validation.builder_address`. This mirrors the existing pattern for `l1_da_mode`, `l2_gas_price_fri`, and `starknet_version`.

### Proof of Concept

1. A validator is the legitimate proposer for height H, round R.
2. It constructs `ProposalInit` normally but sets `builder = ContractAddress::from(ATTACKER_ADDRESS)` instead of `self.config.static_config.builder_address`.
3. It streams this `ProposalInit` to peers.
4. Each peer's `handle_proposal` checks `init.proposer == expected_proposer` — passes.
5. Each peer's `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, gas prices, `starknet_version`, `version_constant_commitment`, `timestamp`, `fee_proposal_fri` — all pass; `builder` is never read.
6. `initiate_validation` calls `convert_to_sn_api_block_info(init)` → `BlockInfo { sequencer_address: ATTACKER_ADDRESS, … }` → `batcher.validate_block(ValidateBlockInput { block_info, … })`.
7. The batcher executes all transactions with `sequencer_address = ATTACKER_ADDRESS`. Sequencer fees are credited to `ATTACKER_ADDRESS`. Contracts reading `get_execution_info()` see `ATTACKER_ADDRESS`.
8. `PartialBlockHashComponents::new` stores `sequencer = ATTACKER_ADDRESS`; `PartialBlockHash::from_partial_block_hash_components` hashes it. Both proposer and all validators compute the same `ProposalCommitment` (since all used the same `init.builder`). Consensus decides.
9. The committed block's hash permanently encodes `ATTACKER_ADDRESS` as the sequencer, diverging from the expected block hash.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
