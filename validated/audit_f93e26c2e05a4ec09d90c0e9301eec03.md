### Title
Missing `ppuWellFormed` Validation for New Dijkstra Protocol Parameters Allows Malformed Governance Proposals to Bypass Intended Limits - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/PParams.hs`)

---

### Summary

The `ConwayEraPParams DijkstraEra` instance's `ppuWellFormed` function omits validation checks for the four new Dijkstra-era protocol parameters (`maxRefScriptSizePerTx`, `maxRefScriptSizePerBlock`, `refScriptCostStride`, `refScriptCostMultiplier`). This is the direct analog of the Bond Protocol M-7 bug: a feature-controlling parameter lacks a setter/validation guard, leaving the feature broken or exploitable. Concretely, a governance proposal setting `maxRefScriptSizePerTx = 0` or `maxRefScriptSizePerBlock = 0` bypasses the `ppuWellFormed` gate and enters the governance system without being rejected as `MalformedProposal`.

---

### Finding Description

In the Conway era, the four reference-script cost/limit parameters are **hardcoded constants**, not protocol parameters:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/PParams.hs:980-983
ppMaxRefScriptSizePerTxG    = L.to . const $ 200 * 1024
ppMaxRefScriptSizePerBlockG = L.to . const $ 1024 * 1024
ppRefScriptCostMultiplierG  = L.to . const . fromJust $ boundRational 1.2
ppRefScriptCostStrideG      = L.to . const $ knownNonZeroBounded @25_600
``` [1](#0-0) 

In the Dijkstra era, these become **live, updatable protocol parameters** stored in `DijkstraPParams`:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/PParams.hs:151-157
dppMaxRefScriptSizePerBlock :: !(THKD ... f Word32)
dppMaxRefScriptSizePerTx    :: !(THKD ... f Word32)
dppRefScriptCostStride      :: !(THKD ... f (NonZero Word32))
dppRefScriptCostMultiplier  :: !(THKD ... f PositiveInterval)
``` [2](#0-1) 

They are registered with governance update tags 34–37 and have full `ppuLens` accessors: [3](#0-2) 

However, the `ppuWellFormed` implementation for `DijkstraEra` is a copy of the Conway checks and **adds no validation for the four new parameters**:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/PParams.hs:539-557
instance ConwayEraPParams DijkstraEra where
  ppuWellFormed _pv ppu =
    and
      [ isValid (/= 0) ppuMaxBBSizeL
      , isValid (/= 0) ppuMaxTxSizeL
      , isValid (/= 0) ppuMaxBHSizeL
      , isValid (/= 0) ppuMaxValSizeL
      , isValid (/= 0) ppuCollateralPercentageL
      , isValid (/= EpochInterval 0) ppuCommitteeMaxTermLengthL
      , isValid (/= EpochInterval 0) ppuGovActionLifetimeL
      , isValid (/= mempty) ppuPoolDepositL
      , isValid (/= zero) ppuGovActionDepositL
      , isValid (/= zero) ppuDRepDepositL
      , isValid ((/= CompactCoin 0) . unCoinPerByte) ppuCoinsPerUTxOByteL
      , ppu /= emptyPParamsUpdate
      , isValid (/= 0) ppuNOptL
      -- *** NO checks for ppuMaxRefScriptSizePerTxL,
      --     ppuMaxRefScriptSizePerBlockL,
      --     ppuRefScriptCostStrideL, ppuRefScriptCostMultiplierL ***
      ]
``` [4](#0-3) 

`ppuWellFormed` is the gate called by `actionWellFormed` in the GOV rule:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Gov.hs:397-399
isWellFormed = case ga of
  ParameterChange _ ppd _ -> ppuWellFormed pv ppd
  _ -> True
``` [5](#0-4) 

A `ParameterChange` proposal with `ppuMaxRefScriptSizePerTxL = SJust 0` passes this gate unchallenged and enters the governance system. `maxRefScriptSizePerTx` is typed `Word32`, so `0` is a syntactically valid value.

Once enacted, `validateAllRefScriptSize` in the Dijkstra LEDGER rule enforces the limit:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Ledger.hs:321-329
validateAllRefScriptSize pp utxo tx =
  let totalRefScriptSize = batchNonDistinctRefScriptsSize utxo tx
      maxRefScriptSizePerTx = fromIntegral @Word32 @Int $ pp ^. ppMaxRefScriptSizePerTxG
   in failureUnless (totalRefScriptSize <= maxRefScriptSizePerTx) $
        DijkstraTxRefScriptsSizeTooBig ...
``` [6](#0-5) 

With `maxRefScriptSizePerTx = 0`, the condition `totalRefScriptSize <= 0` fails for every transaction that references any script, permanently freezing all reference-script-based UTxOs.

The same applies to `maxRefScriptSizePerBlock = 0` in the BBODY rule.

Note: `refScriptCostStride` is typed `NonZero Word32` (zero excluded by type) and `refScriptCostMultiplier` is `PositiveInterval` (zero excluded by type), so those two parameters cannot be set to zero. The exploitable gap is specifically `maxRefScriptSizePerTx` and `maxRefScriptSizePerBlock`, both `Word32`.

---

### Impact Explanation

**Medium → escalates to High upon enactment.**

The immediate, attacker-controlled impact is that a governance proposal with `maxRefScriptSizePerTx = 0` bypasses the intended `ppuWellFormed` validation limit and enters the governance system — matching the Medium criterion: *"attacker-controlled proposals exceed intended validation limits."*

If the proposal is ratified and enacted, the impact escalates to **High: permanent freezing of funds** — every UTxO locked by a Plutus or native script that requires a reference script becomes unspendable, with no recovery path short of a hard fork.

---

### Likelihood Explanation

Any transaction sender can submit a `ParameterChange` governance action by paying the governance action deposit. No privileged access is required to submit the proposal. The missing `ppuWellFormed` check means the proposal is not rejected at submission time and enters the governance system. Enactment requires governance ratification, but the submission step — the one that bypasses the intended guard — is fully attacker-controlled and requires no special privilege.

---

### Recommendation

Add zero-value guards for the two `Word32` parameters in `ppuWellFormed` for `DijkstraEra`:

```haskell
instance ConwayEraPParams DijkstraEra where
  ppuWellFormed _pv ppu =
    and
      [ -- existing checks ...
      , isValid (/= 0) ppuMaxRefScriptSizePerTxL
      , isValid (/= 0) ppuMaxRefScriptSizePerBlockL
      -- refScriptCostStride: NonZero Word32 type already prevents 0
      -- refScriptCostMultiplier: PositiveInterval type already prevents 0
      ]
```

This mirrors the pattern already applied to `ppuMaxBBSizeL`, `ppuMaxTxSizeL`, etc., and is consistent with the Conway era's hardcoded non-zero values for these same limits.

---

### Proof of Concept

1. Construct a `ParameterChange` governance action whose `PParamsUpdate` sets `ppuMaxRefScriptSizePerTxL = SJust 0`.
2. Submit the transaction. The GOV rule calls `actionWellFormed`, which calls `ppuWellFormed`. Because `ppuWellFormed` for `DijkstraEra` has no check for `ppuMaxRefScriptSizePerTxL`, the proposal passes and is accepted into the governance system — no `MalformedProposal` failure is raised.
3. The proposal is now live in the governance system. Any subsequent ratification and enactment sets `maxRefScriptSizePerTx = 0` in the active protocol parameters.
4. From that point, `validateAllRefScriptSize` evaluates `totalRefScriptSize <= 0` for every transaction. Any transaction whose inputs or reference inputs contain a script (size > 0) fails with `DijkstraTxRefScriptsSizeTooBig`, permanently freezing all reference-script-locked funds.
