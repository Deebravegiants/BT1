### Title
Expired-Attestation Participants Can Cancel Their Own TEE-Triggered Removal via `vote_cancel_resharing` — (`crates/contract/src/state/resharing.rs`, `crates/contract/src/lib.rs`)

### Summary
`vote_cancel_resharing` performs no TEE attestation check. Participants whose attestations expired (and who triggered a `verify_tee`-initiated resharing to remove them) remain in `previous_running_state.parameters.participants()` and can cast valid cancellation votes. When threshold-many such participants coordinate, the resharing is cancelled and the contract reverts to the state that includes them, with `accept_requests` still `true`.

### Finding Description
`lib.rs::vote_cancel_resharing` calls only `assert_caller_is_signer()`. [1](#0-0) 

`resharing.rs::vote_cancel_resharing` authenticates the caller solely by checking membership in `previous_running_state.parameters.participants()` via `AuthenticatedAccountId::new`. [5](#0-4) 

`verify_tee` sets `accept_requests = true` and calls `transition_to_resharing_no_checks`, storing the full original participant set (including expired ones) as `previous_running_state`. [4](#0-3) 

The cancellation threshold check compares against `previous_running_threshold.value()`, which is the old epoch's threshold — the same threshold the expired participants can collectively meet. [6](#0-5) 

### Impact Explanation
When `2T <= N` (governance threshold ≤ half the participant count) and exactly T participants have expired attestations, those T participants can cancel the resharing triggered to remove them. The contract reverts to `previous_running_state` with `accept_requests = true`, reinstating the expired-attestation participants in the active signing set. This is a participant/attestation authorization bypass: the TEE-triggered participant-set change is invalidated and the removed participants are reinstated.

### Likelihood Explanation
Requires `2T <= N` (threshold ≤ N/2) and at least T expired-attestation participants coordinating before resharing completes. This is a realistic configuration — e.g., N=5, T=2. The expired participants have a strong incentive to act (they are being removed) and the window is the entire resharing duration.

### Recommendation
In `resharing.rs::vote_cancel_resharing`, after constructing `authenticated_candidate`, verify that the caller's TEE attestation is still valid (e.g., call `tee_state.is_caller_an_attested_participant(previous_running_participants)` at the `lib.rs` level before delegating). Alternatively, explicitly exclude participants that were the cause of the `verify_tee`-triggered resharing from casting cancellation votes.

### Proof of Concept
1. Construct a `RunningContractState` with N=5 participants, T=2.
2. Expire the attestations of participants P1 and P2 (K=2).
3. Call `verify_tee` from P0 (valid); it sees 3 valid participants ≥ T=2, sets `accept_requests = true`, and calls `transition_to_resharing_no_checks` with {P0, P3, P4} as the new set. `previous_running_state` contains all 5 participants.
4. Call `vote_cancel_resharing` from P1 (expired attestation): `assert_caller_is_signer()` passes; `AuthenticatedAccountId::new(previous_running_participants)` passes (P1 is in the old set); vote count = 1.
5. Call `vote_cancel_resharing` from P2 (expired attestation): vote count = 2 ≥ T=2; resharing cancelled; `protocol_state` reverts to `previous_running_state` containing P1 and P2.
6. Assert `accept_requests == true` and that P1, P2 are still active participants — demonstrating the bypass.

    pub fn vote_cancel_resharing(&mut self) -> Result<Option<RunningContractState>, Error> {
        let previous_running_participants = self.previous_running_state.parameters.participants();
        let authenticated_candidate = AuthenticatedAccountId::new(previous_running_participants)?;
        self.cancellation_requests.insert(authenticated_candidate);

        let cancellation_votes_count = self.cancellation_requests.len() as u64;
        let previous_running_threshold = self.previous_running_state.parameters.threshold();

        let threshold_cancellation_votes_reached: bool =
            cancellation_votes_count >= previous_running_threshold.value();

        let running_state = if threshold_cancellation_votes_reached {
            let mut previous_running_state = self.previous_running_state.clone();
            let prospective_epoch_id = self.prospective_epoch_id();
            previous_running_state.previously_cancelled_resharing_epoch_id =
                Some(prospective_epoch_id);

            Some(previous_running_state)
        } else {
            None
        };

        Ok(running_state)
    }
