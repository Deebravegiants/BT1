The attack path is concrete and traceable. Here is the full analysis:

### Title
Unauthenticated CATWallet-to-CRCATWallet Conversion via Attacker-Crafted CRCAT Coin Spend — (`chia/wallet/wallet_state_manager.py`)

### Summary

An unprivileged attacker who holds any CAT coins of the same `asset_id` as a victim's plain `CATWallet` can permanently convert that wallet to a `CRCATWallet` with attacker-controlled `authorized_providers` and `proofs_checker`, by submitting a single on-chain spend. No user consent is required. After conversion the victim's wallet can no longer spend their existing plain CAT coins and any future CRCAT spending requires a VC issued by the attacker's chosen providers.

### Finding Description

**Entry point:** `WalletStateManager._add_coin_states` → `determine_coin_type` → `handle_cat`

**Root cause:** `handle_cat` calls `CRCATWallet.convert_to_cr` using `authorized_providers` and `proofs_checker` extracted directly from the attacker's coin spend, with no user-consent gate and no check that the conversion was initiated by the wallet owner.

**Step-by-step trace:**

**Step 1 – Attacker crafts the spend.**
The attacker owns plain CAT coins of `asset_id X` (same as the victim's `CATWallet`). They spend one with an inner solution that:
- Emits a REMARK condition `(1 victim_inner_puzzle_hash attacker_providers attacker_proofs_checker)` — the format consumed by `CRCAT.get_next_from_coin_spend` for non-CRCAT parents.
- Emits a `CREATE_COIN` condition whose puzzle hash is `CAT(X, CR_LAYER(attacker_providers, attacker_proofs_checker, victim_inner_puzzle_hash))` and whose memos hint to `victim_inner_puzzle_hash`.

The victim's inner puzzle hashes are public (derived from the victim's public key, visible on-chain).

**Step 2 – Victim's wallet receives the coin state.**
The wallet subscribes to its own puzzle hashes as hints. The new CRCAT coin is hinted to `victim_inner_puzzle_hash`, so the full node delivers its `CoinState` to the victim's wallet via the hint subscription mechanism (`register_for_ph_updates` / `hint_store`). [1](#0-0) 

**Step 3 – `handle_cat` is invoked.**
`_add_coin_states` calls `determine_coin_type` → `handle_cat`. [2](#0-1) 

**Step 4 – Hint guard passes.**
`handle_cat` extracts the hint from the parent spend's memos and looks it up in the puzzle store. Because the hint is `victim_inner_puzzle_hash`, which the victim controls, `derivation_record` is non-`None` and execution continues. [3](#0-2) 

**Step 5 – Coin is identified as CRCAT.**
The coin's puzzle hash does not match a plain CAT puzzle, so the code calls `CRCAT.get_next_from_coin_spend`. Because the parent is a plain CAT (not a CRCAT), the non-CR branch fires: it reads `new_inner_puzzle_hash`, `authorized_providers`, and `proofs_checker` directly from the attacker's REMARK condition. [4](#0-3) 

**Step 6 – Inner-puzzle-hash guard passes.**
The code checks that `crcat.inner_puzzle_hash` is in the victim's puzzle store. Because the attacker set it to `victim_inner_puzzle_hash`, this check passes. [5](#0-4) 

**Step 7 – No existing CRCAT wallet found.**
The loop at lines 1292–1295 finds no CRCAT wallet for `asset_id X` (the victim only has a plain `CATWallet`), so execution falls through. [6](#0-5) 

**Step 8 – `convert_to_cr` is called with attacker-controlled parameters.**
The code finds the victim's plain `CATWallet` and calls `convert_to_cr` passing `crcat.authorized_providers` and `crcat.proofs_checker` — both fully attacker-controlled values from the crafted spend. [7](#0-6) 

`convert_to_cr` overwrites the wallet record in the database and replaces the in-memory wallet object with a `CRCATWallet` carrying the attacker's parameters. There is no user-consent check anywhere in this path. [8](#0-7) 

### Impact Explanation

- The victim's `CATWallet` is permanently converted to a `CRCATWallet` with attacker-chosen `authorized_providers` and `proofs_checker` stored in the wallet database.
- The victim's existing plain CAT coins (puzzle hash `CAT(X, victim_inner_puzzle)`) no longer match the CRCAT wallet's expected puzzle hash, so the wallet stops tracking and cannot spend them.
- All future CRCAT spending requires a VC issued by one of the attacker's `authorized_providers`. If the attacker sets an empty or unreachable provider list, the victim's CAT wallet is permanently bricked.
- This is **wallet sync state corruption with direct security impact** (High) and **unauthorized coin control** (High).

### Likelihood Explanation

- The attacker only needs to hold any amount of the same CAT (purchasable on any DEX).
- The victim's inner puzzle hashes are public.
- The attack requires a single on-chain transaction and triggers automatically on the victim's next wallet sync.
- No interaction from the victim is required beyond having a running wallet node.

### Recommendation

1. **Require explicit user consent** before calling `convert_to_cr`. The conversion should only be triggered by an explicit RPC/UI action, not automatically during coin sync.
2. **Validate that the conversion parameters match a known, user-approved configuration** before persisting the change.
3. As a defense-in-depth measure, consider adding a check that the `authorized_providers` and `proofs_checker` in the observed coin spend match what the wallet already expects (if any prior CRCAT configuration exists).

    @classmethod
    async def convert_to_cr(
        cls,
        cat_wallet: CATWallet,
        authorized_providers: list[bytes32],
        proofs_checker: ProofsChecker,
    ) -> None:
        replace_self = cls()
        replace_self.standard_wallet = cat_wallet.standard_wallet
        replace_self.log = logging.getLogger(cat_wallet.get_name())
        replace_self.log.info(f"Converting CAT wallet {cat_wallet.id()} to CR-CAT wallet")
        replace_self.wallet_state_manager = cat_wallet.wallet_state_manager
        replace_self.info = cls.wallet_info_type(
            cat_wallet.cat_info.limitations_program_hash, None, authorized_providers, proofs_checker
        )
        await cat_wallet.wallet_state_manager.user_store.update_wallet(
            WalletInfo(
                cat_wallet.id(), cat_wallet.get_name(), uint8(WalletType.CRCAT.value), bytes(replace_self.info).hex()
            )
        )
        updated_wallet_info = await cat_wallet.wallet_state_manager.user_store.get_wallet_by_id(cat_wallet.id())
        assert updated_wallet_info is not None
        replace_self.wallet_info = updated_wallet_info

        cat_wallet.wallet_state_manager.wallets[cat_wallet.id()] = replace_self
