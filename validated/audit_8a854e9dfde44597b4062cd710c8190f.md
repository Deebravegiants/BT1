### Title
Hardcoded Consensus Signing Private Key Used in Production Signature Manager — (`File: crates/apollo_signature_manager/src/lib.rs`)

### Summary

The production `SignatureManager` is instantiated via `create_signature_manager()`, which unconditionally calls `LocalKeyStore::new_for_testing()`. That function embeds a well-known, publicly-visible ECDSA private key directly in source code. This key is used to sign consensus **precommit votes** and **peer identity challenges**. Any party with access to the source repository can extract the key and forge valid signatures for any block hash, impersonating the sequencer's validator identity in the Tendermint consensus round.

### Finding Description

`create_signature_manager()` in `crates/apollo_signature_manager/src/lib.rs` is the sole production factory for the `SignatureManager` component:

```rust
pub fn create_signature_manager() -> SignatureManager {
    SignatureManager::new()   // calls LocalKeyStore::new_for_testing()
}
``` [1](#0-0) 

`LocalKeyStore::new_for_testing()` hard-codes both the private and public key:

```rust
const PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
    "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
));
const PUBLIC_KEY: PublicKey = PublicKey(Felt::from_hex_unchecked(
    "0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a",
));
``` [2](#0-1) 

This `SignatureManager` is wired into the live node in `crates/apollo_node/src/components.rs`:

```rust
Some(create_signature_manager())
``` [3](#0-2) 

The `SignatureManager` uses this key for two operations:

1. `sign_precommit_vote(block_hash)` — produces the ECDSA signature attached to every Tendermint precommit message. [4](#0-3) 
2. `sign_identification(peer_id, challenge)` — authenticates the node's identity to peers. [5](#0-4) 

The codebase itself acknowledges this is unfinished:

```
// TODO(Elin): understand how key store would look in production and better define
// the way the signature manager is created.
``` [6](#0-5) 

The analog to the ZecWallet bug is exact: ZecWallet stored a shared secret in plaintext Android SharedPreferences; here the consensus signing secret is stored as a plaintext constant in the binary, equally readable by anyone with repository access.

### Impact Explanation

Because the private key is publicly known, any attacker can:

- Compute `ecdsa_sign(PRIVATE_KEY, precommit_vote_digest(block_hash))` for an arbitrary `block_hash` and inject a forged precommit vote that is cryptographically indistinguishable from a legitimate one produced by the sequencer.
- Forge peer-identity signatures, allowing the attacker to impersonate the sequencer's validator identity during the `INIT_PEER_ID` handshake.

If the corresponding public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a` is registered in the validator set (which it must be for the node to participate in consensus), forged precommit votes for an attacker-chosen block hash will be accepted by every peer that verifies them with `verify_precommit_vote_signature`. This maps directly to the allowed impact: **"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."**

### Likelihood Explanation

The private key is a compile-time constant in a public repository. No special access, network position, or privileged operation is required. Any reader of the source code can extract it immediately. The trigger is unprivileged and requires no interaction with the running node.

### Recommendation

**Short term:** Remove `create_signature_manager()` from the production node startup path until a proper key-management solution exists. Gate the component on an explicit configuration flag that fails at startup if no real key is provided, preventing accidental deployment with the test key.

**Long term:** Implement a `KeyStore` backed by an HSM, OS keychain, or encrypted-at-rest key file (analogous to the `Sensitive<Vec<u8>>` wrapper already used for the network `secret_key`). The private key must never appear in source code or unencrypted configuration. Apply the same `Sensitive<T>` redaction pattern already present in `apollo_config` to prevent the key from leaking through logs or serialized config dumps. [7](#0-6) 

### Proof of Concept

```rust
// Attacker extracts the public constant from source and forges a precommit vote:
use starknet_core::crypto::ecdsa_sign;
use starknet_core::types::Felt;

let private_key = Felt::from_hex_unchecked(
    "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
);

// Choose any target block hash to forge a precommit for:
let target_block_hash: [u8; 32] = /* attacker-chosen value */;

// Replicate build_precommit_vote_message_digest:
let mut message = b"PRECOMMIT_VOTE".to_vec();
message.extend_from_slice(&target_block_hash);
let digest = blake2s_to_felt(&message);

// Produce a valid signature accepted by verify_precommit_vote_signature:
let forged_sig = ecdsa_sign(&private_key, &digest).unwrap();
// → forged_sig passes verify_precommit_vote_signature for any block hash
```

The forged signature passes `verify_precommit_vote_signature` because it is produced with the exact key whose public counterpart is registered as the validator's identity. [8](#0-7)

/// A wrapper for values that are considered **sensitive** (e.g. secrets, tokens, URLs).
#[derive(Clone, Deserialize)]
#[serde(transparent, bound(deserialize = "T: Deserialize<'de>"))]
pub struct Sensitive<T> {
    inner: T,
    #[serde(skip)]
    redactor: Option<Redactor<T>>,
}

impl<T> Sensitive<T> {
    /// Creates a new `Sensitive<T>` with no custom redactor.
    pub fn new(inner: T) -> Self {
        Self { inner, redactor: None }
    }

    /// Attaches a custom redactor function to this `Sensitive` value.
    pub fn with_redactor<F>(mut self, redactor: F) -> Self
    where
        F: Fn(&T) -> String + Send + Sync + 'static,
    {
        self.redactor = Some(Arc::new(redactor));
        self
    }

    /// Consumes the wrapper and returns the inner sensitive value.
    pub fn expose_secret(self) -> T {
        self.inner
    }

    /// Returns a reference to the inner sensitive value.
    pub fn peek_secret(&self) -> &T {
        &self.inner
    }
