### Title
Optional `network_id` in Alonzo+ transaction body omits network domain from signed witness data, enabling cross-network replay for output-free transactions - (File: `eras/alonzo/impl/src/Cardano/Ledger/Alonzo/Rules/Utxo.hs`)

---

### Summary
The `network_id` field (CBOR map key `15`) in the Alonzo+ transaction body is optional. When absent, `validateWrongNetworkInTxBody` unconditionally succeeds. Because witnesses sign the transaction body hash, and the body hash does not commit to the network when `network_id` is omitted, a transaction signed on one network is cryptographically valid on any other network that shares the same UTxO entry. For transactions that carry zero Shelley-format outputs and zero withdrawals, the only remaining network-binding checks (`validateWrongNetwork`, `validateWrongNetworkWithdrawal`) iterate over empty collections and also trivially pass. The signed transaction is therefore accepted on the replay network, consuming the UTxO inputs and destroying the ADA as fees.

---

### Finding Description

**Root cause — `validateWrongNetworkInTxBody` silently passes when `network_id` is absent:** [1](#0-0) 

```haskell
validateWrongNetworkInTxBody netId txBody =
  case txBody ^. networkIdTxBodyL of
    SNothing -> pure ()          -- ← no check; network is unconstrained
    SJust bid ->
      failureUnless (netId == bid) $
        WrongNetworkInTxBody Mismatch {mismatchSupplied = bid, mismatchExpected = netId}
```

The field is declared optional in every post-Shelley CDDL schema: [2](#0-1) 

```
, ? 15 : network_id               ; new
```

The same optional declaration appears in Babbage, Conway, and Dijkstra: [3](#0-2) [4](#0-3) [5](#0-4) 

**The two address-level network checks are also bypassable for output-free transactions:**

In Alonzo/Babbage/Conway/Dijkstra all three network checks are `runTestOnSignal`, meaning they only execute when `IsValid = True`. For a valid transaction with no outputs and no withdrawals, both `validateWrongNetwork` and `validateWrongNetworkWithdrawal` iterate over empty collections and pass: [6](#0-5) 

The same pattern is present in Babbage: [7](#0-6) 

And in Dijkstra (top-level and sub-transaction rules): [8](#0-7) [9](#0-8) 

**Contrast with Byron**, which mandates a `ProtocolMagicId`-based domain tag on every signature and rejects any mismatch: [10](#0-9) [11](#0-10) 

Shelley+ dropped the mandatory protocol-magic tag from the signing path and replaced it with address-embedded network IDs — but those checks are skipped when there are no outputs or withdrawals.

**`getNetwork` for `AddrBootstrap`** does map Byron addresses to a `Network` value, so Byron bootstrap addresses in outputs would also be checked. The gap is specifically the zero-output, zero-withdrawal case: [12](#0-11)

### Impact Explanation

When a new testnet is bootstrapped from mainnet state (a standard Cardano operational practice), the UTxO sets are initially identical. A user who signs a zero-output transaction on testnet — for example, a stake-key deregistration where the deposit refund is consumed entirely as fees, or an explicit "burn" — without including `network_id` produces a witness that is cryptographically identical on mainnet. An attacker who observes the testnet mempool can replay the transaction on mainnet, consuming the user's mainnet UTxO and destroying the ADA as fees. The ledger accepts the transition as valid.

**Impact: Medium** — attacker-controlled transaction modifies fees outside design parameters (all UTxO value consumed as fees on the unintended network), matching the allowed impact: *"Attacker-controlled transactions… modify fees, deposits, refunds, rewards, treasury donations, or withdrawals outside design parameters."*

---

### Likelihood Explanation

- New testnets bootstrapped from mainnet state are a recurring operational event in Cardano's history; at bootstrap the UTxO sets are identical.
- Most production wallets omit `network_id` from transaction bodies because the field is optional and adds bytes.
- Zero-output transactions are uncommon but valid and do occur (stake-key deregistration consuming the deposit as fees, explicit value-burn transactions).
- The attacker requires only passive observation of the testnet mempool and the ability to submit a transaction to mainnet — no privileged access.

**Likelihood: Low** — requires the coincidence of a testnet bootstrap from mainnet state AND a zero-output/zero-withdrawal transaction being signed during the window before the UTxO sets diverge.

---

### Recommendation

Make `network_id` mandatory in the transaction body for Alonzo+ eras, or at minimum enforce the check unconditionally rather than only when the field is present. The `SNothing` branch of `validateWrongNetworkInTxBody` should be treated as a predicate failure rather than a pass:

```haskell
validateWrongNetworkInTxBody netId txBody =
  case txBody ^. networkIdTxBodyL of
    SNothing ->
      failureUnless False $
        MissingNetworkId netId   -- require the field to be present
    SJust bid ->
      failureUnless (netId == bid) $
        WrongNetworkInTxBody Mismatch {mismatchSupplied = bid, mismatchExpected = netId}
```

This is a hard-fork change. As a non-breaking interim measure, document the cross-network replay risk and require wallet implementations to always include `network_id` in transaction bodies.

### Proof of Concept

1. Testnet is bootstrapped from mainnet state. UTxO entry `(txid_A, 0) → (addr, 100 ADA)` exists on both networks; `addr` is controlled by key `K`.

2. On testnet, construct transaction `T`:
   - `inputs = {(txid_A, 0)}`
   - `outputs = []` (empty)
   - `fee = 100 ADA`
   - `network_id` field: **absent** (`SNothing`)

3. Sign `T` with key `K` on testnet → produces witness `W` over `hash(body(T))`.

4. Submit `T + W` to testnet:
   - `validateBadInputsUTxO`: `(txid_A, 0)` ∈ testnet UTxO ✓
   - `validateWrongNetwork`: outputs = [] → trivially passes ✓
   - `validateWrongNetworkWithdrawal`: withdrawals = {} → trivially passes ✓
   - `validateWrongNetworkInTxBody`: `SNothing` → `pure ()` ✓
   - `validateValueNotConservedUTxO`: consumed = produced = 100 ADA ✓
   - Transaction accepted; UTxO consumed on testnet.

5. Submit the **identical** `T + W` to mainnet (same body bytes, same witness):
   - All checks pass for identical reasons.
   - `(txid_A, 0)` is consumed on mainnet; 100 ADA destroyed as fees.
   - User suffers irreversible ADA loss on mainnet without having signed any mainnet transaction.
