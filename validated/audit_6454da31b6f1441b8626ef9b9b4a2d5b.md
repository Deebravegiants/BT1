### Title
Cryptographic Domains Cannot Be Removed Once Added — (`crates/contract/src/lib.rs`, `crates/contract/src/state/running.rs`)

---

### Summary

The MPC contract exposes `vote_add_domains` to permanently append new cryptographic signing domains (e.g., a new Cait-Sith ECDSA or FROST EdDSA domain) to the protocol. There is no corresponding `vote_remove_domains` or equivalent mechanism. Once a domain is committed, it is irreversible through any on-chain governance path, including resharing.

---

### Finding Description

`vote_add_domains` in `lib.rs` collects unanimous participant votes and, on quorum, transitions the contract into `InitializingContractState` to generate keys for the new domains: [1](#0-0) 

The underlying state-machine implementation in `running.rs` confirms that once the vote count equals the full participant count, the new domains are appended and the state transitions — with no removal path: [2](#0-1) 

Critically, resharing (`vote_new_parameters`) does **not** remove domains. The `InitializingContractState` produced by `vote_add_domains` carries `generated_keys: self.keyset.domains.clone()`, preserving all previously generated domain keys across epoch boundaries: [3](#0-2) 

The contract ABI snapshot confirms there is no `vote_remove_domains` method exposed: [4](#0-3) 

By contrast, the TEE measurement system does implement a removal path (`vote_remove_os_measurement`), and the launcher hash system has `vote_remove_launcher_hash` (requiring unanimous votes). The domain system has no equivalent. [5](#0-4) 

---

### Impact Explanation

If a cryptographic vulnerability is discovered in a live domain's protocol (e.g., a flaw in the Cait-Sith ECDSA presignature generation, or a bias in the FROST nonce derivation), the protocol team has **no on-chain mechanism** to disable or remove that domain. The contract will continue to accept `sign()` requests targeting the compromised domain and queue them for processing. The only available mitigation is off-chain: coordinating all node operators to stop processing requests for that domain. This is slower, error-prone, and not enforceable by the contract — every second of delay while the domain remains active and nodes are still running exposes users to unauthorized signature issuance from the compromised domain.

This breaks the production safety invariant that only cryptographically sound domains should be available for signing, and maps to: **Medium — contract execution-flow manipulation that breaks production safety/accounting invariants without relying on network-level DoS or operator misconfiguration.**

---

### Likelihood Explanation

Cryptographic protocol vulnerabilities in threshold signature schemes are rare but not theoretical — the history of MPC protocols includes practical attacks on specific instantiations (e.g., nonce reuse in ECDSA, biased randomness in presignature generation). The NEAR MPC system supports multiple protocols (Cait-Sith, Robust ECDSA, FROST) across multiple domains, increasing the attack surface. The likelihood of needing emergency domain removal is low but non-negligible over the protocol's lifetime, and the consequence of having no removal path is permanent.

---

### Recommendation

Implement a `vote_remove_domain` function gated behind the same unanimous-participant threshold used by `vote_add_domains`. The function should:

1. Accept a `DomainId` to remove.
2. Require all current participants to vote.
3. On quorum, mark the domain as inactive in the contract state so new `sign()` requests targeting it are rejected.
4. Preserve existing key material (do not delete keyshares) to allow forensic analysis.

This mirrors the pattern already established for `vote_remove_os_measurement` and `vote_remove_launcher_hash`.

---

### Proof of Concept

1. Participants unanimously call `vote_add_domains` with a new `DomainId(N)` using protocol `CaitSith`.
2. Contract transitions through `Initializing` → `Running` with the new domain active.
3. A vulnerability is discovered in the Cait-Sith presignature protocol for this domain.
4. The protocol team attempts to disable the domain on-chain — **no such function exists**.
5. Users continue submitting `sign()` requests with `domain_id: N`.
6. The contract accepts and queues these requests.
7. Nodes that have not yet been manually stopped continue processing them, producing signatures from the compromised domain.
8. The only recourse is an emergency contract upgrade (slow, requires governance) or coordinated off-chain node shutdown (not enforceable by the contract). [1](#0-0) [6](#0-5)

    pub fn vote_add_domains(
        &mut self,
        domains: Vec<DomainConfig>,
    ) -> Result<Option<InitializingContractState>, Error> {
        if domains.is_empty() {
            return Err(DomainError::AddDomainsMustAddAtLeastOneDomain.into());
        }
        let num_participants = u64::try_from(self.parameters.participants().len())
            .expect("participant count fits in u64");
        for domain in &domains {
            validate_domain_purpose(domain)?;
            validate_domain_threshold(domain, num_participants)?;
        }
        // Keep trust assumptions consistent: a domain must never require more shares to
        // reconstruct than the GovernanceThreshold demands to govern. Route through the
        // canonical helper so the cross-domain invariant has a single source of truth.
        ThresholdParameters::validate_governance_against_reconstruction(
            num_participants,
            self.parameters.threshold(),
            max_reconstruction_threshold(&domains),
        )?;
        let participant = AuthenticatedParticipantId::new(self.parameters.participants())?;
        let n_votes = self.add_domains_votes.vote(domains.clone(), &participant);
        if self.parameters.participants().len() as u64 == n_votes {
            let new_domains = self.domains.add_domains(domains.clone())?;
            Ok(Some(InitializingContractState {
                generated_keys: self.keyset.domains.clone(),
                domains: new_domains,
                epoch_id: self.keyset.epoch_id,
                generating_key: KeyEvent::new(
                    self.keyset.epoch_id,
                    domains[0].clone(),
                    self.parameters.clone(),
                ),
                cancel_votes: BTreeSet::new(),
            }))
        } else {
            Ok(None)
        }
