### Title
Missing `now`-lower-bound on `ProposalInit.timestamp` lets a proposer embed stale L1 gas prices in committed blocks — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` enforces only a monotonicity lower-bound (`timestamp >= last_block_timestamp`) and a future upper-bound (`timestamp <= now + window`). There is no lower-bound against `now`. Because the validator's own L1-gas-price reference is computed by calling `get_l1_prices_in_fri_and_wei(init_proposed.timestamp, …)` — i.e., anchored to the proposer-supplied timestamp — a proposer who sets a past timestamp causes both sides to agree on historical L1 gas prices. The `within_margin` check passes, the proposal is accepted, and the committed block carries stale L1 gas prices and a past `block_timestamp`.

---

### Finding Description

In `is_proposal_init_valid` the timestamp window is:

```
last_block_timestamp  ≤  init_proposed.timestamp  ≤  now + block_timestamp_window_seconds
``` [1](#0-0) 

There is no guard of the form `init_proposed.timestamp >= now - lower_bound`. Immediately after the timestamp check, the validator queries L1 gas prices using the proposer-supplied timestamp:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,          // ← proposer-controlled past value
    proposal_init_validation.previous_proposal_init.as_ref(),
    gas_price_params,
).await;
``` [2](#0-1) 

`get_price_info` inside `L1GasPriceProvider` only rejects timestamps that are too far **in the future** relative to the newest sample (`timestamp > last_timestamp + max_time_gap_seconds`). It has no lower-bound guard against timestamps that are arbitrarily old: [3](#0-2) 

A past timestamp causes `get_price_info` to select an older window of L1 blocks for the rolling mean, returning historical (potentially lower) gas prices. Because the validator uses the same past timestamp as its reference, the `within_margin` check passes: [4](#0-3) 

The proposer also controls the `block_timestamp` field of `BlockInfo` forwarded to the batcher via `convert_to_sn_api_block_info(init)`: [5](#0-4) 

The same missing lower-bound exists in `try_sync`: [6](#0-5) 

---

### Impact Explanation

**L1 gas prices in committed blocks are wrong.** A proposer who sets `timestamp = last_block_timestamp + 1` (the minimum allowed) forces the validator to compute its reference from the same historical window. Both sides agree, the margin check passes, and the block is committed with L1 gas prices that reflect a past L1 state rather than the current one. This directly affects:

1. **Transaction fee calculation** — `l1_gas_price_fri` / `l1_data_gas_price_fri` embedded in `BlockInfo` are used by the blockifier to charge L1 gas fees. Artificially low historical prices reduce fees charged to users, causing economic loss to the protocol.
2. **`get_block_timestamp` syscall** — contracts that rely on block timestamps for time-locked logic (e.g., oracles, vesting, auctions) receive a past value, potentially causing incorrect execution results.

This matches the allowed impact: *"Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact"* (Critical) and *"Wrong state … or revert result from blockifier/syscall/execution logic"* (Critical).

---

### Likelihood Explanation

The exploitable window is `[last_block_timestamp, now)`. In normal operation (2–3 s block times) this window is small and the L1 gas price mean (300-block rolling average) barely shifts. However:

- After any network pause or slow-block period the window grows proportionally.
- At genesis (`previous_proposal_init = None`), `last_block_timestamp = 0`, so the proposer can set any timestamp from epoch 0 to `now + window`, potentially forcing the fallback to `min_l1_gas_price_wei` / `DEFAULT_ETH_TO_FRI_RATE`.
- The `max_time_gap_seconds` guard (900 s) only fires when the timestamp is too far **ahead** of the newest L1 sample, not when it is behind. [3](#0-2) 

---

### Recommendation

Add a lower-bound against `now` in `is_proposal_init_valid`, analogous to the Aera fix (`max(block.timestamp, startTime)`):

```rust
// Effective timestamp must not be more than N seconds in the past.
let effective_timestamp = init_proposed.timestamp.max(last_block_timestamp);
if effective_timestamp + proposal_init_validation.max_past_timestamp_seconds < now {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("Timestamp too far in the past: now={now}, proposed={}", init_proposed.timestamp),
    ));
}
```

Additionally, anchor the L1 gas price lookup to `max(init_proposed.timestamp, now - lag)` so that a past timestamp cannot select a historical price window that diverges from the current market:

```rust
let price_lookup_timestamp = init_proposed.timestamp.max(
    now.saturating_sub(gas_price_params.max_price_lookup_lag_seconds)
);
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    price_lookup_timestamp,   // clamped
    …
).await;
``` [7](#0-6)

### Proof of Concept

1. Network produces blocks normally; `last_block_timestamp = T_prev`, `now = T_prev + 2`.
2. Proposer constructs `ProposalInit` with `timestamp = T_prev` (minimum allowed).
3. Proposer queries `get_l1_prices_in_fri_and_wei(T_prev, …)` → receives prices from the window ending at `T_prev - lag_margin_seconds`.
4. Proposer embeds those prices in `ProposalInit.l1_gas_price_fri` / `l1_data_gas_price_fri`.
5. Validator calls `is_proposal_init_valid`:
   - `T_prev >= last_block_timestamp` ✓
   - `T_prev <= now + window` ✓
   - Validator calls `get_l1_prices_in_fri_and_wei(T_prev, …)` → same historical prices.
   - `within_margin(proposed, reference, margin)` passes because both are identical.
6. `initiate_validation` forwards `block_info` with `block_timestamp = T_prev` to the batcher.
7. Block is committed with `block_timestamp = T_prev` and historical L1 gas prices.
8. Contracts calling `get_block_timestamp()` receive `T_prev` instead of `now`; transaction fees are computed from historical L1 prices. [8](#0-7) [9](#0-8)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}
