### Title
Unverified CKD Response for `AppPublicKey` Variant Allows Single Byzantine Participant to Deliver Attacker-Controlled Key Material - (File: `crates/contract/src/lib.rs`)

### Summary

In `respond_ckd`, when the pending request uses the `CKDAppPublicKey::AppPublicKey` variant, the contract performs **zero cryptographic verification** of the submitted `CKDResponse` before resolving the user's yield. A single attested participant (strictly below the signing threshold) can submit an arbitrary, attacker-crafted `CKDResponse`, delivering key material the attacker controls to the requesting user.

### Finding Description

The `respond_ckd` function in `crates/contract/src/lib.rs` contains an asymmetric verification branch:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}          // ← NO VERIFICATION
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}

pending_requests::resolve_yields_for(
    &mut self.pending_ckd_requests,
    &request,
    serde_json::to_vec(&response).unwrap(),
)
``` [1](#0-0) 

For the `AppPublicKeyPV` variant, `ckd_output_check` is called to verify the response against the BLS12-381 master public key and the user's app public key. For the `AppPublicKey` variant, the arm is an empty no-op — the response is accepted unconditionally and immediately delivered to the user via `resolve_yields_for`.

The caller is only required to be an attested participant:

```rust
self.assert_caller_is_attested_participant_and_protocol_active();
``` [2](#0-1) 

No threshold of participants is required to agree on the response content. A single attested node is sufficient to resolve all queued yields for a given `AppPublicKey` CKD request.

### Impact Explanation

The CKD flow is designed to deliver a derived private key to the user, encrypted under their `app_public_key`. Because the contract does not verify the `CKDResponse` for the `AppPublicKey` variant, a single Byzantine attested participant can:

1. Generate an arbitrary key pair `(sk_evil, pk_evil)`.
2. Encrypt `sk_evil` under the user's `app_public_key` to form a valid-looking `CKDResponse`.
3. Call `respond_ckd` with the pending request and this crafted response.
4. The contract resolves the user's yield with the attacker's response.
5. The user decrypts the response and obtains `sk_evil` — a key the attacker already knows.

The attacker now possesses the user's "derived private key." Any data encrypted to the corresponding public key is readable by the attacker; any signature scheme using that key is forgeable by the attacker. This is a complete compromise of the confidential key derivation output without threshold authorization.

### Likelihood Explanation

Any single attested participant in the MPC network can exploit this. The attacker does not need to collude with other nodes, compromise TEE hardware, or perform any off-chain attack. The only prerequisite is holding a valid TEE attestation, which is the normal operational state of every MPC node. The attack is a single on-chain transaction.

### Recommendation

Apply the same cryptographic output check to the `AppPublicKey` variant that is already applied to `AppPublicKeyPV`. If a deterministic verification function analogous to `ckd_output_check` cannot be constructed for `AppPublicKey` (e.g., because the response is opaque ciphertext), then the contract must require a threshold of independent `respond_ckd` submissions agreeing on the same response before resolving the yield — mirroring the threshold guarantee that the signing flow provides.

### Proof of Concept

1. User calls `request_app_private_key` with `CKDAppPublicKey::AppPublicKey(user_app_pk)` and attaches 1 yoctoNEAR. The contract enqueues the request and yields.

2. Attacker (one attested participant) constructs a `CKDResponse` containing key material of their choice — e.g., a known private scalar encrypted under `user_app_pk`.

3. Attacker calls:
   ```
   respond_ckd(request = <matching CKDRequest>, response = <crafted CKDResponse>)
   ```

4. Inside `respond_ckd`:
   - `assert_caller_is_attested_participant_and_protocol_active()` passes (attacker is attested).
   - `match &request.app_public_key { AppPublicKey(_) => {} ... }` — the empty arm executes; no check runs.
   - `resolve_yields_for(...)` resumes the user's yield with the attacker's response.

5. The user's `request_app_private_key` call returns the attacker-crafted `CKDResponse`. The user decrypts it and uses a private key the attacker already knows. [3](#0-2) [4](#0-3)

    #[handle_result]
    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
