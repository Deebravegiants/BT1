### Title
Storage Deposit Permanently Locked After `propose_update` Execution — (File: `crates/contract/src/update.rs`, `crates/contract/src/lib.rs`)

### Summary
The `propose_update` function collects a storage deposit from the proposer to cover on-chain storage costs. When the update is later executed via `do_update`, all stored entries are cleared (storage freed), but the deposit is never returned to the proposer. The collected NEAR tokens accumulate permanently in the contract with no withdrawal path, directly mirroring the external report's pattern of "value tracked but never utilized or withdrawable."

### Finding Description

`propose_update` enforces a deposit equal to `required_deposit(bytes_used(update))`: [1](#0-0) 

The required deposit is computed as `storage_byte_cost * bytes_used`, where `bytes_used` includes the full contract binary plus overhead for up to 128 participant vote slots: [2](#0-1) 

Only the excess above the required amount is refunded to the proposer: [3](#0-2) 

When `vote_update` reaches threshold and calls `do_update`, the entry is removed from storage and **all** entries and votes are cleared: [4](#0-3) 

`do_update` never issues a `Promise::new(proposer).transfer(deposit)`. The storage is freed but the NEAR tokens remain locked in the contract balance with no function to recover them. There is no `withdraw_update_deposit` or equivalent anywhere in the contract.

### Impact Explanation

For a contract binary update, `bytes_used` = `size_of::<UpdateEntry>()` + `128 × size_of::<AccountId>()` + `code.len()`. A typical MPC contract binary of ~500 KB yields a required deposit of roughly 5 NEAR. Every successful upgrade cycle permanently destroys that deposit. Over the operational lifetime of the network (multiple upgrades), the cumulative locked amount grows without bound and is irrecoverable. This breaks the production accounting invariant that storage deposits are returned when storage is freed.

### Likelihood Explanation

Contract upgrades are a routine, expected governance operation. Every upgrade proposal that reaches threshold and executes triggers the loss. No adversarial action is required — the loss occurs through normal participant governance.

### Recommendation

In `do_update`, before clearing entries, iterate over stored `UpdateEntry` records and refund each proposer's `bytes_used`-derived deposit:

```rust
// Before entries.clear(), refund each proposer
for (_id, entry) in self.entries.iter() {
    let refund = required_deposit(entry.bytes_used);
    Promise::new(proposer_account).transfer(refund).detach();
}
```

Alternatively, record the proposer's account alongside the `UpdateEntry` at proposal time so the exact proposer can be refunded on execution or cancellation.

### Proof of Concept

1. Participant A calls `propose_update` with a 500 KB contract binary. Required deposit ≈ 5 NEAR. Excess is refunded; exactly 5 NEAR is retained by the contract.
2. Threshold participants call `vote_update(id)`. Threshold is reached; `do_update` is invoked.
3. `do_update` removes the entry and calls `self.entries.clear()` — storage is freed.
4. No transfer back to Participant A is issued anywhere in `do_update` or `vote_update`.
5. The 5 NEAR is permanently locked in the contract. Participant A has no recourse.
6. Repeat for each subsequent upgrade: deposits accumulate, none are ever returned. [5](#0-4) [6](#0-5)

    pub fn propose_update(
        &mut self,
        #[serializer(borsh)] args: ProposeUpdateArgs,
    ) -> Result<UpdateId, Error> {
        // Only voters can propose updates:
        let proposer = self.voter_or_panic();
        let update: Update = args.try_into()?;

        let attached = env::attached_deposit();
        let required = ProposedUpdates::required_deposit(&update);
        if attached < required {
            return Err(InvalidParameters::InsufficientDeposit {
                attached: attached.as_yoctonear(),
                required: required.as_yoctonear(),
            }
            .into());
        }

        let id = self.proposed_updates.propose(update);

        log!(
            "propose_update: signer={}, id={:?}",
            env::signer_account_id(),
            id,
        );

        // Refund the difference if the proposer attached more than required.
        if let Some(diff) = attached.checked_sub(required)
            && diff > NearToken::from_yoctonear(0)
        {
            Promise::new(proposer).transfer(diff).detach();
        }

        Ok(id)
    }

    pub fn do_update(&mut self, id: &UpdateId, gas: Gas) -> Option<Promise> {
        let entry = self.entries.remove(id)?;

        // Clear all entries as they might be no longer valid
        self.entries.clear();
        self.vote_by_participant.clear();

        let mut promise = Promise::new(env::current_account_id());
        match entry.update {
            Update::Contract(code) => {
                // deploy contract then do a `migrate` call to migrate state.
                promise = promise.deploy_contract(code).function_call(
                    method_names::MIGRATE,
                    Vec::new(),
                    NearToken::from_near(0),
                    gas,
                );
            }
            Update::Config(config) => {
                // If we vote for a new config, we should use
                // the value `contract_upgrade_deposit_tera_gas` from the config
                // as the new gas value
                let new_config_gas_value = Gas::from_tgas(config.contract_upgrade_deposit_tera_gas);
                promise = promise.function_call(
                    method_names::UPDATE_CONFIG,
                    serde_json::to_vec(&(&config,)).unwrap(),
                    NearToken::from_near(0),
                    new_config_gas_value,
                );
            }
        }
        Some(promise)
    }

fn bytes_used(update: &Update) -> u128 {
    let mut bytes_used = std::mem::size_of::<UpdateEntry>() as u128;

    // Assume a high max of 128 participant votes per update entry.
    bytes_used += 128 * std::mem::size_of::<AccountId>() as u128;

    match update {
        Update::Contract(code) => {
            bytes_used += code.len() as u128;
        }
        Update::Config(config) => {
            let bytes = serde_json::to_vec(&config).unwrap();
            bytes_used += bytes.len() as u128;
        }
    }

    bytes_used
}

fn required_deposit(bytes_used: u128) -> NearToken {
    env::storage_byte_cost().saturating_mul(bytes_used)
}
