Let me look at the exact validation logic and how `builder` flows into block execution and state commitment.

### Title
Unvalidated `ProposalInit.builder` Accepted by Validators Allows Malicious Proposer to Redirect Sequencer Fees and Corrupt Block Header State — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries a `builder` field (the sequencer/builder address). Every other security-sensitive field in `ProposalInit` — `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri` — is checked by `is_proposal_init_valid`. The `builder` field is never checked. It is accepted verbatim from the network and immediately used as the `sequencer_address` in both the blockifier `BlockInfo` (which governs fee collection) and the committed `BlockHeaderWithoutHash` (which is persisted to state sync). A legitimately selected proposer can set `builder` to any arbitrary address, causing all validators to execute and commit the block with the wrong sequencer address, redirecting block fees and corrupting the canonical block header.

---

### Finding Description

`ProposalInit.builder` is defined as:

> "Address of the one who builds/sequences the block." [1](#0-0) 

The `ProposalInitValidation` struct — the sole source of locally-trusted reference values used by `is_proposal_init_valid` — contains `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `block_timestamp_window_seconds`, `previous_proposal_init`, and `fee_actual`. It does **not** contain any expected `builder` address: [2](#0-1) 

`is_proposal_init_valid` validates all fields present in `ProposalInitValidation` but has no check for `builder` anywhere in its body: [3](#0-2) 

After `is_proposal_init_valid` returns `Ok`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address` in the `BlockInfo` passed to the batcher: [4](#0-3) 

On decision, `update_state_sync_with_new_block` also reads `init.builder` directly and writes it as the `sequencer` field of `BlockHeaderWithoutHash`, which is committed to state sync: [5](#0-4) 

By contrast, the `proposer` field **is** validated against the committee's expected proposer before the proposal is forwarded to the context: [6](#0-5) 

`builder` has no equivalent guard anywhere in the validation path.

---

### Impact Explanation

**Fee theft (Critical — economic impact):** The `sequencer_address` in `BlockInfo` is the address to which the blockifier credits sequencer fees for every transaction in the block. Because validators accept the proposer-supplied `builder` without checking it, a malicious proposer sets `builder = attacker_address`. Every validator's batcher executes the block with `sequencer_address = attacker_address`. All transaction fees for that block are credited to the attacker. The `ProposalFinMismatch` check does not catch this because both the proposer's and the validator's batchers use the same `attacker_address`, so they compute identical block hashes.

**Wrong block header committed to state (Critical — wrong state):** `update_state_sync_with_new_block` writes `SequencerContractAddress(init.builder)` into `BlockHeaderWithoutHash.sequencer`. This header is persisted to state sync and becomes the canonical record of the block. The committed block header permanently contains the wrong sequencer address, corrupting the authoritative state view for all downstream consumers (RPC, provers, L1 anchoring).

---

### Likelihood Explanation

Any node that is legitimately selected as a proposer by the committee can trigger this. Proposer selection is weighted-random over the staker set; no special privilege beyond being selected is required. The attack requires no front-running, no race condition, and no external dependency — the proposer simply sets `builder` to an arbitrary address in the `ProposalInit` it broadcasts. The only prerequisite is being scheduled as proposer for a round, which is a normal operational event.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`. The expected value should be derived from the committee configuration (e.g., the proposer's registered builder address, or the node's own `builder_address` if the protocol requires proposer == builder). At minimum, reject a `builder` of `ContractAddress::default()` (zero address). The fix mirrors the existing `proposer` check in `single_height_consensus.rs`: look up the expected builder for the height/round from a trusted local source and reject any `ProposalInit` whose `builder` does not match.

---

### Proof of Concept

**Step 1 — Proposer constructs a malicious `ProposalInit`:**

```rust

// A malicious proposer sets it to their own wallet:
let init = ProposalInit {
    builder: attacker_contract_address,  // arbitrary, unchecked by validators
    proposer: legitimate_proposer,       // must match committee; IS checked
    height: current_height,
    // ... all other fields set legitimately
};
``` [7](#0-6) 

**Step 2 — Validator receives `ProposalInit`, `is_proposal_init_valid` runs, `builder` is never checked:**

The entire body of `is_proposal_init_valid` checks `timestamp`, `starknet_version`, `version_constant_commitment`, `height`/`l1_da_mode`/`l2_gas_price_fri`, L1 gas prices, and `fee_proposal_fri`. No line references `init_proposed.builder`. [8](#0-7) 

**Step 3 — `convert_to_sn_api_block_info` maps `builder` → `sequencer_address`:**

```rust
sequencer_address: init.builder,  // attacker_contract_address flows here
``` [9](#0-8) 

**Step 4 — Batcher executes block with `sequencer_address = attacker_contract_address`; fees credited to attacker. Both proposer and validator compute identical block hash (both use `attacker_contract_address`), so `ProposalFinMismatch` does not fire.**

**Step 5 — On decision, `update_state_sync_with_new_block` commits the wrong sequencer to state:**

```rust
let sequencer = SequencerContractAddress(init.builder); // attacker_contract_address
// ... written into BlockHeaderWithoutHash.sequencer and persisted
``` [10](#0-9) 

The block is finalized with `sequencer_address = attacker_contract_address` in both the execution context and the canonical block header, with no error raised at any point in the pipeline.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
