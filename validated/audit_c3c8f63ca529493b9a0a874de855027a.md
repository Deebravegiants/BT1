### Title
Unvalidated `ProposalInit.builder` Field Allows Any Proposer to Corrupt Block Header Sequencer Address and Block Hash - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`ProposalInit` carries a `builder: ContractAddress` field ("Address of the one who builds/sequences the block") that is transmitted over the wire and accepted verbatim by every validator. The field is never checked against any locally-trusted value in `is_proposal_init_valid` or anywhere else in the validation path. Because `builder` is used directly as `sequencer_address` in both block execution (`BlockInfo`) and the block header (`BlockHeaderWithoutHash`), and because `sequencer_address` is a direct input to `calculate_block_hash`, any validator who is the round proposer can set `builder` to an arbitrary address and have the block accepted by the network with a corrupted sequencer address, wrong block hash, and misdirected fee collection.

### Finding Description

`ProposalInit` has two identity fields:

- `proposer: ContractAddress` — the consensus validator proposing the block
- `builder: ContractAddress` — the sequencer address that builds/sequences the block

The `proposer` field **is** validated. In `handle_proposal` in `manager.rs` and in `single_height_consensus.rs`, the proposer is looked up from the committee and compared against `init.proposer` before `validate_proposal` is ever called. [1](#0-0) 

The `builder` field is **never** validated. `ProposalInitValidation` — the struct that carries all locally-trusted reference values used by `is_proposal_init_valid` — has no `builder` field: [2](#0-1) 

`is_proposal_init_valid` validates `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, L1 gas prices, and `fee_proposal_fri` — but never `builder`: [3](#0-2) 

The proposer's own node sets `builder` from a locally-configured value: [4](#0-3) [5](#0-4) 

The config field is `ContextStaticConfig.builder_address`: [6](#0-5) 

But when a validator receives a proposal from the network, `init.builder` is taken straight off the wire and passed unchecked into `convert_to_sn_api_block_info`, which maps it to `sequencer_address` in `BlockInfo`: [7](#0-6) 

This `BlockInfo` is forwarded to the batcher via `validate_block` in `initiate_validation`: [8](#0-7) 

After consensus, `init.builder` is also used as `sequencer` in `BlockHeaderWithoutHash` when the block is committed to state sync: [9](#0-8) 

`sequencer` is a direct input to `PartialBlockHashComponents`, which is chained into `calculate_block_hash`: [10](#0-9) [11](#0-10) 

### Impact Explanation

A malicious validator who is the round proposer sets `builder` to an attacker-controlled address `X` in `ProposalInit`. Because no validator checks `init.builder` against any expected value, all validators execute the block with `sequencer_address = X`. The resulting effects are:

1. **Wrong block hash committed to L1.** `sequencer_address` is directly chained into `calculate_block_hash`. Every validator computes the same wrong hash (they all use the attacker-supplied `builder`), so the `ProposalFinMismatch` check does not fire and the block is accepted. The wrong hash is anchored on L1.

2. **Fee misdirection.** In Starknet, transaction fees are transferred to the sequencer address during execution. With `sequencer_address = X`, all fees for the block are credited to the attacker's address instead of the legitimate sequencer.

3. **Wrong `get_execution_info` syscall result.** Contracts that call `get_execution_info` receive `X` as the sequencer address. Any contract that uses the sequencer address for access control or logic (e.g., fee token contracts, bridge contracts) will observe the wrong value.

The `ProposalFinMismatch` guard at the end of `validate_proposal` does **not** catch this because both the proposer's batcher and the validator's batcher use the same attacker-supplied `builder` from `init`, so they compute identical (but wrong) commitments. [12](#0-11) 

### Likelihood Explanation

The attacker must be a legitimate validator in the committee and must be selected as the proposer for a round. In Tendermint-style consensus, proposer selection rotates among all validators, so any validator in the set can trigger this. No special privilege beyond committee membership is required. The attack is deterministic and requires no brute force.

### Recommendation

Add `builder_address: ContractAddress` to `ProposalInitValidation` and check `init_proposed.builder == proposal_init_validation.builder_address` inside `is_proposal_init_valid`, returning `InvalidProposalInit` on mismatch. The locally-trusted value is already available as `self.config.static_config.builder_address` in `SequencerConsensusContext::validate_proposal`. [2](#0-1) [13](#0-12) 

### Proof of Concept

1. Attacker is validator `V` in the committee. In round `R`, `V` is selected as proposer.
2. `V` calls `build_proposal` normally but overrides `init.builder` to attacker address `X` before broadcasting `ProposalPart::Init`.
3. All peer validators receive the `ProposalInit` with `builder = X`.
4. Each validator calls `is_proposal_init_valid` — no check on `builder`, so validation passes.
5. Each validator calls `initiate_validation` → `convert_to_sn_api_block_info(&init)` → `BlockInfo { sequencer_address: X, ... }` is sent to the batcher.
6. The batcher executes all transactions with `sequencer_address = X`. Fees are transferred to `X`. `get_execution_info` returns `X`.
7. `PartialBlockHashComponents { sequencer: X, ... }` is computed; `calculate_block_hash` chains `X` into the hash.
8. Both proposer and validators compute the same (wrong) `ProposalCommitment`; `ProposalFinMismatch` does not fire.
9. Consensus reaches decision; the block with `sequencer_address = X` and the wrong block hash is committed to storage and anchored on L1.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
