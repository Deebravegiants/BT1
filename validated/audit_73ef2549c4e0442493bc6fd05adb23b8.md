### Title
Governance Action Deposit Permanently Diverted to Treasury When `pProcReturnAddr` Is Unregistered During Conway Bootstrap Phase — (File: `eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Gov.hs`)

---

### Summary

During the Conway bootstrap phase (protocol version 9), the `GOV` rule skips the check that `pProcReturnAddr` in a governance proposal must correspond to a registered stake account. This creates a structural mismatch: the deposit instruction accepts any `AccountAddress` at submission time, but `returnProposalDeposits` at epoch boundary requires the credential to be registered to credit the refund. When the credential is absent, the deposit is silently routed to the treasury and is permanently inaccessible to the proposer without a governance-majority treasury-withdrawal action.

---

### Finding Description

**Root cause — missing validation at deposit time (bootstrap bypass):**

In `conwayGovTransition` (`Gov.hs`), the registration check for the return address is gated behind `unless (hardforkConwayBootstrapPhase ...)`:

```haskell
unless (hardforkConwayBootstrapPhase $ pp ^. ppProtocolVersionL) $ do
  isAccountRegistered (refundAddress ^. accountAddressCredentialL) (certDState ^. accountsL)
    ?! (injectFailure . ProposalReturnAccountDoesNotExist) refundAddress
``` [1](#0-0) 

`hardforkConwayBootstrapPhase` is `True` exactly when `pvMajor pv == natVersion @9`: [2](#0-1) 

So during protocol version 9, a transaction sender may supply any `AccountAddress` — including one whose staking credential has never been registered — as `pProcReturnAddr`, and the transaction is accepted. The governance action deposit is deducted from the proposer's UTxO.

**Deposit-return path — funds diverted to treasury:**

At every epoch boundary, `returnProposalDeposits` in `Epoch.hs` iterates over all removed proposals (expired, enacted, or superseded) and attempts to credit the deposit to the return-address credential:

```haskell
processProposal gas (!accounts, !unclaimed)
  | (Just _accountState, newAccounts) <-
        updateLookupAccountState addRefund cred accounts = (newAccounts, unclaimed)
  | otherwise = (accounts, Map.insert (gasId gas) (gasDeposit gas) unclaimed)
  where
    cred = gasReturnAddr gas ^. accountAddressCredentialL
``` [3](#0-2) 

If `cred` is not in the accounts map, the deposit is placed in `unclaimed`. The epoch transition then sweeps `unclaimed` into the treasury:

```haskell
& casTreasuryL <>~ (utxoState0 ^. utxosDonationL <> fold unclaimed)
``` [4](#0-3) 

The proposer's deposit is gone. The only recovery path is a `TreasuryWithdrawals` governance action ratified by a DRep/CC/SPO supermajority —
