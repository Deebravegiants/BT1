### Title
Prime-N Paillier Modulus Accepted by Both `paillier_t::convert`/`create_pub` and `valid_paillier_t` ZK Proof, Enabling Attacker Plaintext Recovery — (`src/cbmpc/crypto/base_paillier.cpp`, `src/cbmpc/zk/zk_paillier.cpp`)

---

### Summary

An attacker who controls a serialized Paillier key blob (or acts as a malicious MPC peer) can supply a 2048-bit **prime** N as their Paillier modulus. Both the deserialization path (`paillier_t::convert`) and the public-key constructor (`create_pub`) accept it without a primality check. Critically, the `valid_paillier_t` ZK proof — which is supposed to certify that N = p·q — also **passes** for a prime N due to a mathematical coincidence in the proof structure. An honest party that encrypts a secret under this prime N exposes the plaintext to the attacker, who knows φ(N) = N−1 and can run the standard Paillier decryption algorithm.

---

### Finding Description

**Deserialization / public-key construction — no primality check**

`paillier_t::convert` deserializes N via `mod_t::convert`, which enforces only: [1](#0-0) 

```
m > 1  &&  m.get_bits_count() <= MAX_MODULUS_BITS  &&  m.is_odd()
```

After that, `paillier_t::convert` adds only an upper-bound check: [2](#0-1) 

`create_pub` has the identical two-check sequence: [3](#0-2) 

Neither path checks that N is composite, that N has exactly `bit_size` bits, or that N = p·q for any p, q.

**`valid_paillier_t` ZK proof passes for prime N**

The non-interactive proof is supposed to certify that the prover knows φ(N), implying N is composite. The verifier checks: [4](#0-3) 

```cpp
if (sigma[i].pow_mod(N, N) != rho) return coinbase::error(E_CRYPTO);
```

The prover computes `sigma[i] = rho^{N_inv} mod N` where `N_inv = N^{-1} mod φ(N)`. [5](#0-4) 

For a **prime** N, φ(N) = N−1, so `N_inv = N^{-1} mod (N−1)` exists and the prover can compute it. The verifier equation `sigma^N ≡ rho (mod N)` then holds by Fermat's little theorem:

```
sigma^N = rho^(N · N_inv) mod N
N · N_inv ≡ 1 (mod N−1)  ⟹  N · N_inv = 1 + k(N−1)
rho^(1 + k(N−1)) = rho · (rho^(N−1))^k ≡ rho · 1^k = rho  (mod N)
```

The `check_integer_with_small_primes` guard also passes trivially for a large prime: [6](#0-5) 

**Attacker can decrypt ciphertexts encrypted under prime N**

Standard Paillier encryption `c = (1 + mN) · r^N mod N²` is well-formed for any odd N. For prime N, φ(N²) = N(N−1), so `r^{N(N−1)} ≡ 1 (mod N²)`. Decryption with φ(N) = N−1:

```
c^(N−1) mod N² = (1 + mN)^(N−1) · r^{N(N−1)} mod N²
               ≡ (1 − mN) · 1  (mod N²)   [binomial, higher terms ≡ 0 mod N²]
L(c^(N−1)) = −m
mu = L(g^(N−1))^{−1} = (−1)^{−1} = −1 ≡ N−1  (mod N)
m_recovered = (−m)(N−1) mod N = m  ✓
```

The attacker who chose N knows N−1 = φ(N) and can run this decryption on any ciphertext produced by an honest party.

---

### Impact Explanation

Any honest party that accepts the attacker's prime N as a Paillier public key — whether via `paillier_t::convert` during protocol message deserialization or via `create_pub` — and then encrypts a secret value (e.g., an ECDSA key share, an OT mask, a threshold secret) under that key exposes the plaintext to the attacker. The `valid_paillier_t` ZK proof, which is the intended guard against malformed keys, provides no protection here because it passes for prime N. This enables full plaintext recovery of any value encrypted under the attacker-supplied key.

---

### Likelihood Explanation

The attack requires only that the attacker act as a malicious MPC peer who supplies their own Paillier public key — a standard capability in any threshold or two-party protocol. Generating a 2048-bit prime is routine. The attacker must also be able to produce a valid `valid_paillier_t` proof for their prime N, which is straightforward since they know φ(N) = N−1. No threshold collusion or host compromise is required.

---

### Recommendation

1. **Add a compositeness check in `create_pub` and `paillier_t::convert`**: after the bit-size check, verify that N is not prime (e.g., via a Miller-Rabin test or by requiring that N passes a dedicated RSA-modulus validation).
2. **Strengthen the `valid_paillier_t` ZK proof**: the current proof (`sigma^N ≡ rho mod N`) is satisfied by prime N. A correct proof of RSA-modulus structure must additionally verify that N is composite — for example, by using a proof that N has no small factors **and** that N is not prime (e.g., Goldberg-Lindell or a dedicated composite-witness proof).
3. **Add a lower-bound bit-size check**: `paillier_t::convert` rejects `N.get_bits_count() > bit_size` but not `< bit_size`; add the symmetric check.

---

### Proof of Concept

```cpp
// 1. Generate a 2048-bit prime N_prime (attacker-controlled).
bn_t N_prime = bn_t::generate_prime(2048, false);

// 2. Honest party accepts it via create_pub — succeeds, no primality check.
paillier_t pub;
assert(pub.create_pub(N_prime) == SUCCESS);

// 3. Honest party encrypts a secret.
bn_t secret = 42;
bn_t cipher = pub.encrypt(secret);

// 4. Attacker decrypts using phi(N_prime) = N_prime - 1.
//    Construct a private paillier_t with p=N_prime, q=1 is not valid,
//    but attacker can implement the L-function directly:
//    c1 = cipher^(N_prime-1) mod N_prime^2
//    m  = L(c1) * (N_prime-1) mod N_prime  [since mu = N_prime-1]
bn_t phi = N_prime - 1;
mod_t NN(N_prime * N_prime);
bn_t c1;
MODULO(NN) c1 = cipher.pow(phi);
// L(c1) = (c1 - 1) / N_prime
bn_t Lc1 = (c1 - 1) / N_prime;
bn_t recovered;
MODULO(mod_t(N_prime)) recovered = Lc1 * phi;  // mu = phi = N-1
assert(recovered == secret);  // attacker recovers plaintext
```
