### Title
Unauthenticated P2P Peer Can Inject Forged `state_root` into Node Storage via Unverified `SignedBlockHeader` — (`crates/apollo_p2p_sync/src/client/header.rs`)

---

### Summary

The p2p sync client's `HeaderStreamBuilder::parse_data_for_block` performs no cryptographic verification of the received `SignedBlockHeader`. A forged `state_root` (protobuf field 6) passes through the deserialization layer, the parse step, and `write_to_storage` without any check, landing verbatim in `StorageBlockHeader::state_root`. The `starknet_getStateUpdate` RPC handler then reads that field directly as `new_root`, returning an attacker-controlled value to every caller.

---

### Finding Description

**Deserialization — no validation of `state_root`**

`TryFrom<protobuf::SignedBlockHeader> for SignedBlockHeader` maps field 6 directly to `GlobalRoot` with no range or consistency check:

```rust
let state_root = value
    .state_root
    .ok_or(missing("SignedBlockHeader::state_root"))?
    .try_into()
    .map(GlobalRoot)?;
``` [1](#0-0) 

**Parse step — only block number and signature count are checked**

`parse_data_for_block` rejects a header only if the block number is wrong or the signature vector length ≠ 1. There is no:
- cryptographic verification of the signature against the block hash,
- verification that `block_hash = calculate_block_hash(state_root, …)`,
- comparison of `state_root` against any trusted source. [2](#0-1) 

A grep across the entire `crates/apollo_p2p_sync/` tree returns zero hits for `verify_block_signature` or `calculate_block_hash`.

**Storage write — verbatim persistence**

`write_to_storage` calls `append_header` unconditionally: [3](#0-2) 

`append_header` copies `block_header_without_hash.state_root` straight into `StorageBlockHeader::state_root`: [4](#0-3) 

**RPC exposure — `starknet_getStateUpdate` returns the forged value**

`get_state_update` reads the stored header and returns `header.new_root` (which is `block_header_without_hash.state_root`) as the `new_root` field of `AcceptedStateUpdate`: [5](#0-4) 

---

### Impact Explanation

Any unauthenticated p2p peer can serve a `SignedBlockHeader` with an arbitrary `state_root`. The receiving node stores it and subsequently returns it as `new_root` in `starknet_getStateUpdate` responses — an authoritative-looking wrong value served to every downstream client, bridge, or prover that queries the node. The stored `block_hash` field is also peer-supplied and unverified, so the attacker can supply a self-consistent forged chain (forged `state_root` + matching forged `block_hash`) that passes all in-code checks.

Severity: **High** — RPC returns an authoritative-looking wrong value (fits the "High. RPC execution … returns an authoritative-looking wrong value" criterion). The claimed "Critical" label is overstated because the corrupted node is a syncing full node, not the sequencer's own block-production or consensus path.

---

### Likelihood Explanation

The p2p sync protocol is permissionless: any node can respond to a header query. No peer authentication or reputation gating is enforced at the sync-client layer. A single malicious peer that wins the race to respond to a query is sufficient to corrupt the target node's header storage for the queried block range.

---

### Recommendation

1. **Verify the block hash**: after deserializing a `SignedBlockHeader`, recompute `calculate_block_hash` from the received header fields and assert it equals the received `block_hash` field. Reject the peer on mismatch.
2. **Verify the sequencer signature**: call `verify_block_signature(sequencer_pub_key, &signature, &state_diff_commitment, &block_hash)` (the function already exists in `starknet_api::block`) before storing any header received over p2p.
3. **Verify chain linkage**: the existing TODO comment ("Check that parent_hash is the same as the previous block's hash") should be resolved — reject headers whose `parent_hash` does not match the stored previous block hash. [6](#0-5) 

---

### Proof of Concept

```rust
// Pseudocode — mirrors the existing test pattern in header_test.rs
let forged_state_root = GlobalRoot(felt!("0x1234"));
let signed_header = SignedBlockHeader {
    block_header: BlockHeader {
        block_hash: BlockHash(felt!("0xdeadbeef")), // arbitrary, unverified
        block_header_without_hash: BlockHeaderWithoutHash {
            block_number: BlockNumber(0),
            state_root: forged_state_root,
            ..Default::default()
        },
        state_diff_length: Some(0),
        ..Default::default()
    },
    signatures: vec![BlockSignature::default()], // length == 1, passes the only check
};

// write_to_storage stores it verbatim
storage_writer
    .begin_rw_txn().unwrap()
    .append_header(BlockNumber(0), &signed_header.block_header).unwrap()
    .commit().unwrap();

// starknet_getStateUpdate returns new_root == 0x1234
let update = rpc.get_state_update(BlockId::Number(0)).await.unwrap();
assert_eq!(update.new_root, forged_state_root); // passes — attacker wins
```
