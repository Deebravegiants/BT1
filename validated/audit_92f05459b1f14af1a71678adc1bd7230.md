### Title
Missing Vote Signature Verification Allows Any Network Peer to Forge Consensus Votes as Any Validator — (`File: crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`handle_vote()` in `SingleHeightConsensus` accepts prevotes and precommits from the network without verifying the cryptographic signature on the vote. The only admission check is that `vote.voter` is a member of the current committee. Because committee member addresses are public on-chain, any network peer can forge a vote claiming to be any validator by setting `voter` to a known committee address and leaving `signature` as the zero default. The `verify_precommit_vote_signature` function exists in `apollo_signature_manager` but is never called in production code.

### Finding Description

`handle_vote()` in `single_height_consensus.rs` contains an explicit `// TODO(Asmaa): verify the signature` comment at line 242, confirming that signature verification is intentionally deferred and not yet implemented. [1](#0-0) 

The only guard present is a membership check on `vote.voter`:

```rust
if !self.committee.members().iter().any(|s| s.address == vote.voter) {
    debug!("Ignoring vote from non validator: vote={:?}", vote);
    return VecDeque::new();
}
``` [2](#0-1) 

This check is trivially bypassed: a forger sets `vote.voter` to any known committee member address. The `Vote` struct carries a `signature: RawSignature` field that is never inspected during admission. [3](#0-2) 

The protobuf deserializer explicitly defaults the signature to `RawSignature::default()` (the zero value) when the wire field is absent:

```rust
let signature =
    value.signature.map(|hashes| hashes.try_into()).transpose()?.unwrap_or_default();
``` [4](#0-3) 

The node's own self-votes are also created with `signature: RawSignature::default()` and a matching `// TODO(Asmaa): sign the vote` comment, confirming the signing half is equally unimplemented: [5](#0-4) 

The verification functions `verify_precommit_vote_signature` and `verify_identity` exist in `apollo_signature_manager` but appear **only** in unit tests — `grep` across the entire repository finds zero production call sites for either function. [6](#0-5) 

### Impact Explanation

An attacker who is a reachable network peer and knows the committee composition (public on-chain data) can:

1. Craft a `Vote` message with `voter` set to any committee member's `ContractAddress`, `signature` set to the zero default, and `proposal_commitment` set to any `ProposalCommitment` value.
2. Broadcast it to the target validator node.
3. The node's `handle_vote()` passes the height check and the membership check, then feeds the forged vote directly into the state machine as a genuine prevote or precommit.

By repeating this for enough distinct committee member addresses to satisfy the quorum threshold, the attacker can force the state machine to emit `SMRequest::DecisionReached` for an arbitrary `ProposalCommitment`. This maps to a wrong block hash, wrong state root, wrong receipt/event commitments, and wrong L1 anchoring data being committed at that height.

Impact: **High** — consensus admission accepts forged votes, enabling an unprivileged network peer to manipulate the consensus decision and the resulting block/state commitment.

### Likelihood Explanation

- The attacker only needs to be a reachable p2p peer; no privileged access is required.
- Committee member addresses are public (derived from the L2 staking contract).
- The zero-default signature passes deserialization without error.
- No rate-limiting or proof-of-work guards the vote ingestion path.
- The bug is confirmed by two explicit TODO comments in production code, not a subtle logic error.

### Recommendation

1. **Implement signature verification in `handle_vote()`**: resolve the `// TODO(Asmaa): verify the signature` by calling `verify_precommit_vote_signature` (already implemented in `apollo_signature_manager`) against the public key associated with `vote.voter` before the vote is forwarded to the state machine.
2. **Implement vote signing in `make_self_vote()`**: resolve the `// TODO(Asmaa): sign the vote` by calling `sign_precommit_vote` via the `SignatureManager` client before broadcasting.
3. **Reject zero/default signatures at the deserialization boundary**: in `converters/consensus.rs`, treat an absent or all-zero signature as a conversion error rather than silently defaulting.
4. **Add a test** that sends a vote with a mismatched or zero signature and asserts it is rejected by `handle_vote()`.

### Proof of Concept

```
Attacker (any p2p peer):

1. Observe committee for height H:
   validators = [V1, V2, V3, V4]   // public on-chain

2. For each Vi needed to reach quorum, craft:
   Vote {
       vote_type: Precommit,
       height: H,
       round: 0,
       proposal_commitment: Some(ProposalCommitment(attacker_chosen_felt)),
       voter: Vi,                   // legitimate committee address
       signature: RawSignature::default(),  // zero — never checked
   }

3. Serialize each Vote to protobuf and broadcast to the target node.

4. Target node's handle_vote():
   - height check: passes (H == H)
   - membership check: passes (Vi ∈ committee)
   - signature check: SKIPPED (TODO comment)
   - vote fed to state machine

5. After quorum threshold is met, state machine emits:
   SMRequest::DecisionReached(Decision {
       block: attacker_chosen_felt,   // wrong ProposalCommitment
       ...
   })

6. Node commits the wrong block hash / state root at height H.
``` [7](#0-6)

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
