### Title
Missing Chainlink Staleness Check Enables Phantom-Yield Fee Minting and rsETH Dilution — (`contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` discards the `updatedAt` timestamp returned by `latestRoundData()`, accepting arbitrarily stale prices without any heartbeat validation. When a stale, inflated price flows into `LRTOracle._updateRsETHPrice()`, the protocol computes phantom yield, charges a fee on it, and mints excess rsETH to the treasury — permanently diluting every existing holder's backing ratio.

---

### Finding Description

**Root cause — `contracts/oracles/ChainlinkPriceOracle.sol` lines 49–55:**

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();   // updatedAt silently discarded
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
``` [1](#0-0) 

`latestRoundData()` returns `(roundId, answer, startedAt, updatedAt, answeredInRound)`. The destructuring pattern `(, int256 price,,,)` silently drops `updatedAt` and `answeredInRound`. No comparison against `block.timestamp - maxStaleness` is ever performed.

**Exploit path through `LRTOracle._updateRsETHPrice()`:**

1. `_getTotalEthInProtocol()` iterates every supported asset and calls `getAssetPrice(asset)`, which returns the stale inflated price. [2](#0-1) 

2. `totalETHInProtocol` is inflated. `rewardAmount = totalETHInProtocol - previousTVL` is phantom yield. [3](#0-2) 

3. `protocolFeeInETH` is computed on that phantom yield and `rsethAmountToMintAsProtocolFee` is minted to the treasury. [4](#0-3) 

**Existing guards and why they do not prevent this:**

| Guard | Why it fails |
|---|---|
| `pricePercentageLimit` | Defaults to `0`; the check is `pricePercentageLimit > 0 && …`, so it is a no-op until explicitly set. Even when set, the manager can bypass it via `updateRSETHPriceAsManager()`. |
| `maxFeeMintAmountPerDay` | Caps total daily fee minting but does not distinguish legitimate yield from phantom yield; a stale price within the cap still causes over-extraction. |
| `whenNotPaused` | Unrelated to price freshness. | [5](#0-4) [6](#0-5) 

`updateRSETHPrice()` is a **public, permissionless function** — any EOA can trigger it. [7](#0-6) 

---

### Impact Explanation

Excess rsETH is minted to the treasury backed by no real collateral. Every existing holder's share of the underlying ETH pool shrinks permanently. This is a direct, irreversible reduction in the backing ratio — **protocol insolvency through fee over-extraction**. The treasury profits at the expense of all depositors; the loss cannot be recovered without a separate remediation mint.

---

### Likelihood Explanation

Chainlink feeds have defined heartbeat intervals (e.g., 1 hour for stETH/ETH, 24 hours for some LST feeds). Network congestion, sequencer downtime (on L2), or a temporary oracle node outage can push `updatedAt` past the heartbeat without any price update. This is a well-documented, historically observed failure mode — not a theoretical one. Because `updateRSETHPrice()` is permissionless, an attacker can monitor for stale conditions and call it at the optimal moment.

---

### Recommendation

Add a configurable `maxStaleness` parameter per feed and validate `updatedAt` in `getAssetPrice`:

```solidity
mapping(address asset => uint256 maxStaleness) public assetMaxStaleness;

function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,, uint256 updatedAt,) = priceFeed.latestRoundData();
    if (block.timestamp - updatedAt > assetMaxStaleness[asset]) revert StalePriceFeed();
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Set `assetMaxStaleness` to slightly above each feed's documented heartbeat (e.g., heartbeat + 10 %).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Fork mainnet at a recent block, then:
// 1. Deploy / use existing ChainlinkPriceOracle + LRTOracle
// 2. Record rsETH totalSupply and treasury balance before
// 3. Warp time past the Chainlink heartbeat without triggering a feed update
// 4. Call updateRSETHPrice()
// 5. Assert treasury rsETH balance increased
// 6. Assert rsETH price (totalETH / supply) is lower than it would be with a fresh price

function testStaleOraclePhantomFee() public {
    // snapshot state
    uint256 supplyBefore = rsETH.totalSupply();
    uint256 treasuryBefore = rsETH.balanceOf(treasury);
    uint256 priceBefore = lrtOracle.rsETHPrice();

    // advance time past heartbeat (e.g., 25 hours for a 24h-heartbeat feed)
    // Chainlink feed is NOT updated — updatedAt stays in the past
    vm.warp(block.timestamp + 25 hours);

    // anyone can call this
    lrtOracle.updateRSETHPrice();

    uint256 supplyAfter = rsETH.totalSupply();
    uint256 treasuryAfter = rsETH.balanceOf(treasury);

    // treasury received rsETH minted on phantom yield
    assertGt(treasuryAfter, treasuryBefore, "treasury gained rsETH");
    // total supply increased while real collateral did not
    assertGt(supplyAfter, supplyBefore, "supply inflated");
    // per-token backing ratio decreased
    assertLt(lrtOracle.rsETHPrice(), priceBefore, "price diluted");
}
