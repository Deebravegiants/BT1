### Title
Unprivileged Cross-Contract Call Can Overwrite Any Account's `GlobalContractDeployMode::AccountId` Global Contract - (File: runtime/runtime/src/global_contracts.rs)

### Summary
`action_deploy_global_contract` stores a global contract under the **receiver** account's ID when `deploy_mode == AccountId`, but never verifies that the receipt's predecessor (the caller) is the same account as the receiver. Because any deployed contract can create a cross-contract receipt targeting an arbitrary account and append `DeployGlobalContractAction(AccountId)` to it via the `promise_batch_action_deploy_global_contract_by_account_id` host function, an unprivileged attacker can overwrite the global contract stored under any victim account's ID with attacker-controlled WASM code.

### Finding Description
`GlobalContractDeployMode::AccountId` is explicitly designed so that "the owner can update the contract for all its users." The identifier used to store and look up the code is `GlobalContractCodeIdentifier::AccountId(account_id)`, where `account_id` is the receipt's receiver.

In `action_deploy_global_contract` the only guard is a balance check on the receiver:

```rust
// runtime/runtime/src/global_contracts.rs  lines 39-58
let Some(updated_balance) = account.amount().checked_sub(storage_cost) else { … };
initiate_distribution(state_update, account_id.clone(), …, &deploy_contract.deploy_mode, …)?;
```

Inside `initiate_distribution` (lines 149-155):

```rust
let id = match deploy_mode {
    GlobalContractDeployMode::AccountId => {
        GlobalContractIdentifier::AccountId(account_id.clone())  // receiver, not signer
    }
    …
};
```

There is no check that `predecessor_id == receiver_id`. The host function that contracts use to schedule this action:

```rust
// runtime/near-vm-runner/src/logic/logic.rs  lines 2558-2570
pub fn promise_batch_action_deploy_global_contract_by_account_id(
    &mut self, promise_idx: u64, code_len: u64, code_ptr: u64,
) -> Result<()> {
    self.promise_batch_action_deploy_global_contract_impl(
        promise_idx, code_len, code_ptr,
        GlobalContractDeployMode::AccountId, …)
}
```

accepts any `promise_idx`, including one whose receipt targets a completely different account. The dispatch in `runtime/runtime/src/lib.rs` (lines 593-603) passes no actor constraint to `action_deploy_global_contract`.

### Impact Explanation
An attacker who controls any deployed contract can:
1. Call `promise_batch_create(victim_account_id)` to create a receipt targeting `victim.near`.
2. Call `promise_batch_action_deploy_global_contract_by_account_id` to attach malicious WASM to that receipt.
3. When the receipt executes, `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId("victim.near") }` is overwritten with the attacker's code.
4. Every account whose `AccountContract` is `GlobalByAccount("victim.near")` now executes the attacker's WASM on every function call — enabling balance theft, state corruption, or denial of service across all such accounts.

The corrupted protocol value is the global-contract trie entry `TrieKey::GlobalContractCode { identifier: AccountId(victim) }` and, transitively, the runtime execution outcome of every account that references it.

### Likelihood Explanation
The attacker only needs a funded account to deploy a one-function contract and submit a single transaction. No validator or operator privilege is required. The victim's account must have enough balance to cover the (potentially tiny) storage cost for the attacker's code; if not, the attacker can prepend a `Transfer` action to the same batch or use a near-zero-byte WASM stub. The `GlobalContractDeployMode::AccountId` feature is production-enabled and the host function is unconditionally exported.

### Recommendation
In `action_deploy_global_contract`, when `deploy_mode == AccountId`, assert that the receipt's `predecessor_id` equals `account_id` (the receiver). Alternatively, restrict `DeployGlobalContractAction(AccountId)` at the receipt-validation layer so it is only accepted when `predecessor_id == receiver_id`, mirroring the ownership invariant stated in the design documentation.

### Proof of Concept
1. Attacker deploys a contract on `attacker.near` containing:
   ```rust
   pub fn hijack_victim() {
       let p = env::promise_batch_create("victim.near");
       env::promise_batch_action_deploy_global_contract_by_account_id(
           p, MALICIOUS_WASM.len() as u64, MALICIOUS_WASM.as_ptr() as u64);
   }
   ```
2. Attacker submits a `FunctionCall` transaction calling `hijack_victim`.
3. The resulting action receipt targets `victim.near` with `DeployGlobalContractAction { code: MALICIOUS_WASM, deploy_mode: AccountId }`.
4. `action_deploy_global_contract` is called with `account_id = "victim.near"`, passes the balance check (storage cost deducted from victim), and calls `initiate_distribution` which stores `MALICIOUS_WASM` under `GlobalContractCodeIdentifier::AccountId("victim.near")`.
5. All accounts whose contract is `AccountContract::GlobalByAccount("victim.near")` now execute `MALICIOUS_WASM`.

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())
