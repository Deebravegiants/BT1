### Title
Fee Amount Permanently Locked in `locked_tokens[destination]` on Fast-Transfer Finalization Path — (`near/omni-bridge/src/lib.rs`)

### Summary

In `process_fin_transfer_to_other_chain`, `lock_tokens_if_needed(destination_chain, token, fee.fee)` is called **unconditionally** before the fast-transfer branch check. When a fast transfer exists, the code sends only `amount_without_fee` to the relayer and marks the transfer as finalised — but never unlocks or forwards the `fee` amount. The fee tokens remain in the contract with no claim path, and `locked_tokens[destination][token]` is permanently inflated by `fee`.

---

### Finding Description

The execution path is:

```
fin_transfer()
  → fin_transfer_callback()
    → process_fin_transfer_to_other_chain()
```

Inside `process_fin_transfer_to_other_chain`:

**Step 1 — Unconditional fee lock (lines 2007–2011):** [1](#0-0) 

`lock_tokens_if_needed(destination_chain, token, fee.fee)` increments `locked_tokens[destination][token]` by `fee.fee` regardless of whether a fast transfer exists.

**Step 2 — Fast-transfer branch (lines 2033–2045):** [2](#0-1) 

When a fast transfer is found, `send_tokens(token, relayer, amount_without_fee)` is called and the transfer is marked finalised. **No `unlock_tokens_if_needed` is called for the fee, and no transfer message is added** (the `add_transfer_message` call at line 2048 is in the `else` branch only).

**Step 3 — No-fast-transfer branch (lines 2021–2050):** [3](#0-2) 

In the normal path, `lock_tokens_if_needed(destination, token, amount_without_fee)` is also called and `add_transfer_message` creates a pending transfer — providing a future claim path for both fee and principal. This path is **not taken** when a fast transfer exists.

**`lock_tokens_if_needed` guard:** [4](#0-3) 

The guard only skips locking when `token_origin_chain == destination_chain` or `amount == 0`. For any cross-chain wrapped token with a non-zero fee, the lock is applied.

**`unlock_tokens` / `revert_lock_actions`:** [5](#0-4) 

`revert_lock_actions` is only called in error/revert callbacks. There is no public or protocol-level function that can unlock the orphaned fee entry for the fast-transfer path.

---

### Impact Explanation

- The `fee.fee` token amount is held by the NEAR bridge contract and is **permanently unclaimable** — no transfer message exists, no pending claim, no public unlock path.
- `locked_tokens[destination][token]` is inflated by `fee` per fast-transfer finalization. The invariant `locked_tokens[chain][token] == Σ pending_transfer.amount` is violated.
- Inflated `locked_tokens` allows future `unlock_tokens` calls to succeed against a counter that is higher than the true pending sum, which can cause accounting divergence that grows with each fast-transfer finalization.
- Accumulates silently across all fast-transfer finalizations for any non-native token with a non-zero fee.

**Impact category:** Critical — Irreversible fund lock / permanently unclaimable protocol value.

---

### Likelihood Explanation

Fast transfers are a core, advertised feature of the bridge. Any relayer finalizing an inbound transfer to a non-NEAR chain where a fast transfer was previously executed will trigger this path. The condition is:

1. Token origin chain ≠ destination chain (i.e., wrapped token — the common case).
2. `fee.fee > 0` (standard for any incentivized relay).
3. A fast transfer exists for the transfer ID.

All three conditions are routine in normal bridge operation.

---

### Recommendation

Move `lock_tokens_if_needed(destination_chain, token, fee.fee)` **inside the `else` branch** (no fast transfer), alongside the `amount_without_fee` lock and `add_transfer_message`. In the fast-transfer path, the fee should either be sent directly to `predecessor_account_id` (the fin-transfer relayer) or explicitly excluded from the lock entirely, since no pending outbound transfer exists to justify it.

```rust
// Fast transfer path: fee goes to fin-transfer relayer directly
if let Some(relayer) = recipient {
    self.send_tokens(token.clone(), relayer, U128(amount_without_fee), "").detach();
    self.send_tokens(token, predecessor_account_id, U128(fee.fee), "").detach();
    self.mark_fast_transfer_as_finalised(&fast_transfer.id());
} else {
    // Normal path: lock fee + principal, add pending transfer
    self.lock_tokens_if_needed(destination_chain, &token, fee.fee.into());
    self.lock_tokens_if_needed(destination_chain, &token, amount_without_fee);
    required_balance = self.add_transfer_message(...).saturating_add(required_balance);
}
```

### Proof of Concept

Stateful invariant test (pseudocode):

```
1. Register token with origin_chain = EVM, deploy on NEAR.
2. Simulate inbound transfer: origin=EVM, destination=EVM, amount=1000, fee=50.
3. Call fast_transfer() to register a fast transfer for this transfer_id.
4. Call fin_transfer() → fin_transfer_callback() → process_fin_transfer_to_other_chain().
5. Assert: locked_tokens[EVM][token] == 0  // all accounting should net to zero
   Actual: locked_tokens[EVM][token] == 50  // fee permanently inflated
6. Assert: no pending transfer exists for transfer_id  // confirmed, fast transfer was finalised
7. Assert: fee tokens (50) are unclaimable — no public function can retrieve them.
```

The invariant `locked_tokens[chain][token] == Σ pending_transfer.amount_without_fee` fails by exactly `fee.fee` after every fast-transfer finalization involving a non-native token with a non-zero fee.

    pub(crate) fn lock_tokens_if_needed(
        &mut self,
        chain_kind: ChainKind,
        token_id: &AccountId,
        amount: u128,
    ) -> LockAction {
        if self.get_token_origin_chain(token_id) == chain_kind || amount == 0 {
            return LockAction::Unchanged;
        }

        self.lock_tokens(chain_kind, token_id, amount)
    }

    pub fn revert_lock_actions(&mut self, lock_actions: &[LockAction]) {
        for lock_action in lock_actions {
            match lock_action {
                LockAction::Locked {
                    chain_kind,
                    token_id,
                    amount,
                } => {
                    self.unlock_tokens(*chain_kind, token_id, *amount);
                }
                LockAction::Unlocked {
                    chain_kind,
                    token_id,
                    amount,
                } => {
                    self.lock_tokens(*chain_kind, token_id, *amount);
                }
                LockAction::Unchanged => {}
            }
        }
