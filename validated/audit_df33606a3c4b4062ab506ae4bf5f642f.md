### Title
Missing Vote Signature Verification Allows Forged Quorum — (`crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary
`SingleHeightConsensus::handle_vote` accepts votes from any network peer without verifying the `Vote.signature` field. The only guard is a committee membership check on the `voter` address, which is public information. An attacker with a valid libp2p identity can forge votes for any or all committee members, accumulate a fraudulent quorum, and drive a node to commit a block that the legitimate validators never voted for.

### Finding Description
In `single_height_consensus.rs::handle_vote` (line 242), there is an explicit `// TODO(Asmaa): verify the signature` comment. No call to `verify_precommit_vote_signature` (or any equivalent) exists anywhere in the production vote-handling path. The protobuf converter silently defaults a missing or empty `signature` field to `RawSignature::default()` via `unwrap_or_default()`. The state machine itself also generates self-votes with `signature: RawSignature::default()` (state_machine.rs:255), confirming the signing infrastructure is not yet wired up end-to-end.

The only checks performed before a vote is counted are:
1. Height match
2. `voter` address is in the committee (public information)
3. Duplicate/conflict detection (keyed on `(round, voter)` — easily avoided by using each validator address once)

### Impact Explanation
An attacker who can connect to the gossipsub network (any internet-accessible node) can:
1. Observe the current height/round and the proposer's `proposal_commitment` from broadcast proposals.
2. Craft `Vote` messages with `voter` set to each committee member's address and `signature` zeroed.
3. Broadcast these via gossipsub (gossipsub-level `MessageAuthenticity::Signed` only verifies the libp2p sender identity, not the `voter` field inside the payload).
4. Each forged vote passes all checks in `handle_vote` and is fed into the state machine.
5. Once 2/3+1 forged precommits accumulate, `DecisionReached` is emitted and the node commits the block without any legitimate validator having actually voted.

### Likelihood Explanation
The committee addresses are public. Connecting to the gossipsub network requires only a valid libp2p keypair (trivially generated). The attack requires no privileged access. The TODO comment confirms this is a known gap, not an obscure edge case.

### Recommendation
Call `verify_precommit_vote_signature` (or the equivalent prevote verifier) inside `handle_vote` before passing the vote to the state machine. The committee member's public key must be retrieved from the committee roster and used to verify `vote.signature` against the canonical vote digest. Votes failing verification must be dropped and the peer reported.

### Proof of Concept
Inject a `Vote` with:
- `vote_type = Precommit`
- `height = current_height`
- `round = current_round`
- `proposal_commitment = observed_commitment`
- `voter = known_committee_member_address`
- `signature = RawSignature::default()` (all zeros)

Trace: gossipsub receive → `manager::handle_vote` (manager.rs:996 calls `shc.handle_vote(message)`) → `SingleHeightConsensus::handle_vote` (line 249 passes committee check, line 270 passes as `VoteStatus::New`) → `state_machine.handle_event(StateMachineEvent::Precommit(vote))`. Repeat for 2/3+1 distinct committee addresses. Assert `DecisionReached` is emitted with no legitimate validator having signed.

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
