### Title
State Modified Before Deposit Check in `submit_participant_info` Allows Free Storage Consumption - (File: `crates/contract/src/lib.rs`)

---

### Summary

`submit_participant_info` inserts a new attestation entry into `tee_state.stored_attestations` **before** verifying that the caller attached sufficient deposit to cover the resulting storage cost. In NEAR SDK Rust, `#[handle_result]` functions that `return Err(...)` do **not** revert state — only a panic does. As a result, any caller with a valid TEE attestation can permanently write to contract storage without paying for it.

---

### Finding Description

The function `submit_participant_info` in `crates/contract/src/lib.rs` follows this sequence:

1. Records `initial_storage = env::storage_usage()`.
2. Calls `self.tee_state.add_participant(...)`, which inserts a `NodeAttestation` entry into the `stored_attestations` `IterableMap` — a **committed state mutation**.
3. Only then computes `storage_used`, derives `cost`, and checks `attached < cost`.
4. If the deposit is insufficient, executes `return Err(InvalidParameters::InsufficientDeposit { ... })`. [1](#0-0) [2](#0-1) 

Because `submit_participant_info` is annotated `#[handle_result]` and `#[payable]`, returning `Err` serialises the error as the transaction's return value but **does not roll back the storage write** performed by `add_participant`. The NEAR runtime only reverts state on a panic; a graceful `Err` return leaves all mutations in place. [3](#0-2) 

The `stored_attestations` map is the authoritative registry consulted by `is_caller_an_attested_participant` (used to gate every node-facing method such as `respond`, `vote_pk`, `start_keygen_instance`, `vote_reshared`) and by `reverify_and_cleanup_participants` during governance resharing votes. [4](#0-3) 

---

### Impact Explanation

**Medium — breaks the storage-accounting invariant without requiring operator misconfiguration.**

An attacker who possesses a valid TEE attestation can call `submit_participant_info` with zero (or minimal) attached deposit. The attestation entry is written to `stored_attestations` and the NEAR storage cost is silently borne by the contract account. Repeated calls inflate the contract's on-chain storage footprint at no cost to the attacker, degrading the contract's storage budget and potentially interfering with future legitimate attestation submissions or governance operations that scan `stored_attestations`.

Additionally, once an entry exists in `stored_attestations`, the node satisfies the TEE-attestation prerequisite checked during `vote_new_parameters` resharing proposals, meaning the free-storage path also bypasses the economic barrier intended to gate participation in governance. [5](#0-4) 

---

### Likelihood Explanation

Moderate. The precondition is a valid TEE attestation that passes `verify_locally` (correct docker-image hash, launcher hash, measurements, and unexpired certificate). In production this requires running inside a genuine Intel TDX enclave with an approved image. However, the barrier is purely technical, not economic — once an operator controls a valid TEE node, they can exploit this path repeatedly at negligible cost. [6](#0-5) 

---

### Recommendation

Move the deposit check **before** the call to `add_participant`, or replace the `return Err(...)` path with `env::panic_str(...)` so that the NEAR runtime rolls back the storage write on failure. The simplest fix is to pre-compute the expected storage cost (e.g. by measuring a dry-run or using a fixed upper bound), verify the deposit upfront, and only then mutate state — mirroring the pattern already used in `require_deposit` for sign requests. [7](#0-6) 

---

### Proof of Concept

1. Attacker operates a genuine TDX node with an approved MPC image, obtaining a valid `Attestation`.
2. Attacker calls `submit_participant_info(attestation, tls_public_key)` attaching **0 yoctoNEAR**.
3. `add_participant` succeeds and inserts the `NodeAttestation` into `stored_attestations`.
4. The deposit check computes `cost > 0`, finds `attached == 0`, and executes `return Err(InsufficientDeposit)`.
5. The NEAR runtime commits the transaction (no panic occurred); the entry persists in `stored_attestations`.
6. The attacker's storage cost is borne by the contract account. Repeating with fresh TLS keys inflates storage unboundedly. [8](#0-7)

fn require_deposit(minimum_deposit: NearToken, predecessor: &AccountId) {
    let deposit = env::attached_deposit();
    match deposit.checked_sub(minimum_deposit) {
        None => {
            env::panic_str(
                &InvalidParameters::InsufficientDeposit {
                    attached: deposit.as_yoctonear(),
                    required: minimum_deposit.as_yoctonear(),
                }
                .to_string(),
            );
        }
        Some(diff) => {
            if diff > NearToken::from_yoctonear(0) {
                log!("refund excess deposit {diff} to {predecessor}");
                Promise::new(predecessor.clone()).transfer(diff).detach();
            }
        }
    }
}

                    AttestationSubmissionError::InvalidAttestation(_) => {
                        format!("TeeQuoteStatus is invalid: {err}")
                    }
                    AttestationSubmissionError::TlsKeyOwnedByOtherAccount => err.to_string(),
                };
                InvalidParameters::InvalidTeeRemoteAttestation { reason }
            })?;

        let caller_is_not_participant = self.voter_account().is_err();
        let is_new_attestation = matches!(
            attestation_insertion_result,
            ParticipantInsertion::NewlyInsertedParticipant

        let attestation_storage_must_be_paid_by_caller =
            is_new_attestation || caller_is_not_participant;

        if attestation_storage_must_be_paid_by_caller {
            // `saturating_sub`: if a re-submission shrinks the entry, charge nothing
            // rather than underflow. Intentional asymmetry: we do not refund freed bytes
            // either — the caller already paid for the larger entry, and we'd rather
            // accept that asymmetry than open a refund path for payload-shrinking games.
            let storage_used = env::storage_usage().saturating_sub(initial_storage);
            let cost = env::storage_byte_cost().saturating_mul(storage_used as u128);
            let attached = env::attached_deposit();

            if attached < cost {
                return Err(InvalidParameters::InsufficientDeposit {
                    attached: attached.as_yoctonear(),
                    required: cost.as_yoctonear(),
                }
                .into());
            }

            // Refund the difference if the proposer attached more than required
            if let Some(diff) = attached.checked_sub(cost)
                && diff > NearToken::from_yoctonear(0)
            {
                Promise::new(account_id).transfer(diff).detach();
            }
        }

        Ok(())
    }

    pub(crate) fn add_participant(
        &mut self,
        node_id: NodeId,
        attestation: Attestation,
        tee_upgrade_deadline_duration: Duration,
    ) -> Result<ParticipantInsertion, AttestationSubmissionError> {
        let expected_report_data: ReportData = ReportDataV1::new(
            *node_id.tls_public_key.as_bytes(),
            *node_id.account_public_key.as_bytes(),
        )
        .into();

        let accepted_measurements = self.get_accepted_measurements();
        // TODO(#3264): run DCAP in the verifier contract (Promise + callback) and
        // do the post-DCAP checks here, instead of verifying locally in-WASM.
        let AcceptedAttestation {
            attestation: verified_attestation,
            advisory_ids,
        } = attestation.verify_locally(
            expected_report_data.into(),
            Self::current_time_seconds(),
            &self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration),
            &self.get_allowed_launcher_compose_hashes(),
            &accepted_measurements,
        )?;

        log_informational_advisory_ids(&advisory_ids);

        let tls_pk = node_id.tls_public_key.clone();

        // Authorization: a TLS key registered to one account must not be
        // overwritten by a submission from a different account. Without this,
        // any caller could replace any participant's stored attestation, since
        // the entry is keyed only by `tls_public_key`.
        if let Some(existing) = self.stored_attestations.get(&tls_pk)
            && existing.node_id.account_id != node_id.account_id
        {
            return Err(AttestationSubmissionError::TlsKeyOwnedByOtherAccount);
        }

        let insertion = self.stored_attestations.insert(
            tls_pk,
            NodeAttestation {
                node_id,
                verified_attestation,
            },
        );

        Ok(match insertion {
            Some(_previous_attestation) => ParticipantInsertion::UpdatedExistingParticipant,
            None => ParticipantInsertion::NewlyInsertedParticipant,
        })
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
