Audit Report

## Title
Unprotected `updateRSETHPrice()` Allows Any Caller to Trigger Protocol-Wide Pause - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is declared `public whenNotPaused` with no access control. When the computed rsETH price has fallen more than `pricePercentageLimit` below `highestRsethPrice`, any external address can call this function to automatically pause `LRTDepositPool`, `LRTWithdrawalManager`, and the oracle itself, freezing all user deposits and withdrawals until an admin manually unpauses each contract.

## Finding Description
The contract exposes two entry points into `_updateRsETHPrice()`:

```solidity
// L87 — no access control
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}

// L94 — restricted to manager
function updateRSETHPriceAsManager() external onlyLRTManager {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, the upside branch (L260–266) explicitly gates on the `MANAGER` role before allowing a price increase above the threshold:

```solidity
if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) {
    revert PriceAboveDailyThreshold();
}
```

The downside branch (L270–282) applies no equivalent caller check:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

When `newRsETHPrice < highestRsethPrice` by more than `pricePercentageLimit`, any EOA can call `updateRSETHPrice()`, reach this branch, and pause all three contracts. The asymmetry is the root cause: the protocol correctly restricts privileged price-increase overrides but omits the same restriction for the pause-triggering downside path.

## Impact Explanation
**Medium — Temporary freezing of funds.** Once paused, `LRTDepositPool` rejects all `depositETH`/`depositAsset` calls and `LRTWithdrawalManager` rejects all withdrawal claims. Unpausing each contract requires `onlyLRTAdmin` action. Until the admin responds, all user funds are inaccessible. The attacker spends only gas and gains no direct financial benefit, but the disruption is real and repeatable after each unpause as long as the price condition persists.

## Likelihood Explanation
**Medium.** The triggering condition — rsETH price falling more than `pricePercentageLimit` below `highestRsethPrice` — is a realistic market event (EigenLayer operator slashing, LST de-peg, reduction in underlying strategy balances). An attacker monitoring on-chain state can detect the condition and call `updateRSETHPrice()` before the protocol team acts. No privileged access, no capital, and no oracle manipulation is required.

## Recommendation
Apply the same access control to the downside pause branch that is already applied to the upside threshold branch. Either restrict the public entry point:

```solidity
function updateRSETHPrice() public whenNotPaused onlyLRTOperator {
    _updateRsETHPrice();
}
```

Or gate the pause trigger on an authorized role, mirroring the upside pattern at L263:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) {
        revert PriceBelowDailyThreshold();
    }
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

## Proof of Concept
1. Deploy a local fork with `pricePercentageLimit > 0` and `highestRsethPrice` set.
2. Simulate a slashing event or LST de-peg so that `_getTotalEthInProtocol()` returns a value that makes `newRsETHPrice < highestRsethPrice` by more than `pricePercentageLimit`.
3. From any unprivileged EOA, call `LRTOracle.updateRSETHPrice()`.
4. Observe that `_updateRsETHPrice()` enters the `isPriceDecreaseOffLimit` branch at L277, calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()`.
5. Confirm that subsequent `depositAsset` and withdrawal claim calls revert with paused errors.
6. Confirm that only `onlyLRTAdmin` can unpause, and that the attack is repeatable after each unpause while the price condition persists. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
