Audit Report

## Title
Missing Chainlink Staleness and Validity Checks in `ChainlinkPriceOracle.getAssetPrice()` Enables Stale Price Acceptance - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all validation return fields, accepting zero, stale, or incomplete-round prices without reverting. Because `LRTOracle.updateRSETHPrice()` is a public, permissionless function, any external caller can invoke it while a Chainlink feed is in a stale or zero-price state, causing the downside-protection logic to pause `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle`, freezing all user deposits and withdrawals until admin intervention.

## Finding Description
In `contracts/oracles/ChainlinkPriceOracle.sol` L52, `getAssetPrice()` captures only `price` from `latestRoundData()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No check is performed that `price > 0`, `updatedAt != 0`, or `answeredInRound >= roundId`. If `price` is `0` (returned during an incomplete or corrupted round), `uint256(0) * 1e18 / decimals` silently returns `0` without reverting.

This contrasts directly with `contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol` L30-32, which the same codebase ships with all three guards:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

**Exploit path:**

1. A Chainlink LST feed (e.g., stETH/ETH) enters a state where `answeredInRound < roundId` or `updatedAt == 0`, returning `price = 0`.
2. Any unprivileged external caller invokes `LRTOracle.updateRSETHPrice()` (public, no access control, L87-89).
3. `_updateRsETHPrice()` calls `_getTotalEthInProtocol()`, which calls `getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns `0`.
4. `totalETHInProtocol` is underestimated by the full stETH TVL contribution.
5. `newRsETHPrice` is computed as `(totalETHInProtocol - fee) / rsethSupply`, which drops sharply.
6. At L273-274, `diff > pricePercentageLimit.mulWad(highestRsethPrice)` evaluates to `true` (assuming `pricePercentageLimit > 0`).
7. L278-280: `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` are called.
8. All deposits and withdrawals are frozen until an admin manually unpauses.

Existing checks are insufficient: `_beforeDeposit` only checks `depositAmount > 0` and deposit limits; `getRsETHAmountToMint` performs no oracle sanity check; `_getTotalEthInProtocol` performs no sanity check on the returned `assetER`.

## Impact Explanation
**Medium — Temporary freezing of funds.**

A stale/zero Chainlink price for any single registered LST asset causes `_getTotalEthInProtocol()` to undercount protocol ETH value. The resulting `newRsETHPrice` drop triggers the downside-protection auto-pause at L277-281 of `LRTOracle.sol`, freezing all user deposits and withdrawals across `LRTDepositPool` and `LRTWithdrawalManager` until an admin manually calls `unpause()`. This is a temporary (not permanent) freeze, matching the Medium impact class.

## Likelihood Explanation
Chainlink price feeds are documented to return stale data during network congestion, when a heartbeat update is delayed, or when a new round has not yet been answered (`answeredInRound < roundId`). These are normal operational conditions, not oracle operator compromise. `updateRSETHPrice()` is a public function callable by any address, so no privileged access is required to trigger the pause. Any transient staleness event on any one of the registered LST feeds is sufficient. Likelihood is low-to-medium, but the consequence (protocol-wide pause requiring admin intervention) is disproportionate.

## Recommendation
Apply the same three guards already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    require(price > 0, "Chainlink price <= 0");
    require(updatedAt != 0, "Incomplete round");
    require(answeredInRound >= roundId, "Stale price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Optionally, add a `block.timestamp - updatedAt <= MAX_STALENESS` heartbeat check tuned to each feed's expected update frequency.

## Proof of Concept

1. Fork mainnet at a block where a registered LST Chainlink feed has `answeredInRound < roundId` or mock the feed to return `(roundId=2, price=0, startedAt=0, updatedAt=0, answeredInRound=1)`.
2. Set `pricePercentageLimit` to a non-zero value (e.g., `1e16` for 1%).
