### Title
`RequireGuard` Key-Hash Credential Bypass: Guards Field Requires No Witness for Key-Hash Credentials - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Scripts.hs`)

---

### Summary

In the Dijkstra era, the new `RequireGuard` native script evaluates by checking whether a credential appears in the transaction body's `guards` field. For key-hash credentials, the ledger never requires a corresponding signature for credentials listed in `guards`. Any unprivileged attacker can place an arbitrary key-hash credential in the `guards` field without possessing the private key, causing `RequireGuard(keyHashCredential)` to evaluate to `True` and allowing unauthorized spending of any UTxO, minting policy, or certificate locked by such a script.

---

### Finding Description

**Root cause — `evalDijkstraNativeScript` checks the declared `guards` set, not actual witnesses:**

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Scripts.hs
evalDijkstraNativeScript keyHashes (ValidityInterval txStart txExp) guards = go
  where
    go = \case
      RequireSignature hash -> hash `Set.member` keyHashes   -- checks actual witnesses
      ...
      RequireGuard cred     -> cred `OSet.member` guards     -- checks declared body field only
``` [1](#0-0) 

`RequireSignature` checks `keyHashes`, which is the set of key hashes extracted from the actual cryptographic witnesses (`tx ^. witsTxL`). `RequireGuard` checks `guards`, which is the `OSet (Credential Guard)` read directly from the transaction body field `guardsTxBodyL`. The transaction body is committed to by the fee-payer's signature, but the individual credentials listed inside `guards` carry no authentication requirement of their own.

**The UTXOW rule never adds `guards` key-hash credentials to the required-witness set:**

The Dijkstra UTXOW transition calls `Shelley.validateNeededWitnesses`, which delegates to `getWitsVKeyNeeded` → `getShelleyWitsVKeyNeededNoGov`. That function collects witnesses only for input payment keys, withdrawal authors, certificate authors, and pool owners. The `guards` field is never consulted. [2](#0-1) [3](#0-2) 

**Contrast with script credentials in `guards`:** When a `ScriptHashObj` credential appears in `guards`, the UTXOW script-completeness check (`babbageMissingScripts`) forces the script to be provided and phase-1 validated. Key-hash credentials receive no analogous enforcement.

**The test confirms the asymmetry explicitly:**

```haskell
-- eras/dijkstra/impl/testlib/Test/Cardano/Ledger/Dijkstra/Imp/UtxowSpec.hs
it "Spending inputs locked by script requiring a keyhash guard" $ do
  guardKeyHash <- KeyHashObj <$> freshKeyHash
  scriptHash   <- impAddNativeScript (RequireGuard guardKeyHash)
  txIn         <- produceScript scriptHash
  let tx = mkBasicTx (mkBasicTxBody & inputsTxBodyL .~ [txIn])
  -- fails without the guard declared
  submitFailingTx tx [injectFailure $ Conway.ScriptWitnessNotValidatingUTXOW ...]
  -- succeeds with the guard declared — no signature for guardKeyHash is added
  submitTx_ $ tx & bodyTxL . guardsTxBodyL .~ [guardKeyHash]
``` [4](#0-3) 

The transaction succeeds with only the fee-payer's signature. No witness for `guardKeyHash` is required or checked.

**Attacker-controlled entry path:**

1. Attacker identifies a UTxO (or minting policy, or certificate credential) locked by a native script containing `RequireGuard(victimKeyHash)`.
2. Attacker constructs a transaction spending/using that output.
3. Attacker sets `bodyTxL . guardsTxBodyL .~ [victimKeyHash]` — no private key for `victimKeyHash` is needed.
4. Attacker signs only with their own fee-paying key.
5. `evalDijkstraNativeScript` evaluates `RequireGuard victimKeyHash` → `victimKeyHash ∈ guards` → `True`.
6. The UTXOW rule finds no missing witnesses (guards are not in the required-witness set).
7. The transaction is accepted; funds are stolen.

---

### Impact Explanation

**Critical — Direct loss of ADA or native assets through an invalid ledger state transition.**

Any UTxO, minting policy, or staking/governance credential protected solely or partially by `RequireGuard(keyHashCredential)` can be spent, minted against, or exercised by any unprivileged attacker without possessing the corresponding private key. The ledger accepts the transaction as valid, producing a permanent, irreversible state transition that transfers or destroys the victim's assets.

---

### Likelihood Explanation

**High.** The Dijkstra era is new and `RequireGuard` is its headline native-script feature, explicitly marketed as "enhanced security" in the CDDL comments. Script authors will naturally use it to gate UTxOs on key-hash authorization. The attack requires no special access: any node participant can submit a crafted transaction. The exploit is deterministic and requires only knowledge of the target script hash and the credential embedded in it (both are on-chain). [5](#0-4) 

---

### Recommendation

Add key-hash credentials listed in `guards` to the required-witness set inside `getWitsVKeyNeeded` (or its Dijkstra override). Concretely, for every `KeyHashObj kh` in `txBody ^. guardsTxBodyL`, add `asWitness kh` to the set returned by the witness-needed computation, mirroring how `RequireSignature` already enforces signatures. Script-hash credentials in `guards` already require the script to be provided; key-hash credentials must require the corresponding signature symmetrically.

---

### Proof of Concept

```
1. Deploy a UTxO locked by:
     script = RequireGuard (KeyHashObj victimKH)
   where victimKH is any key hash the attacker does not control.

2. Construct a spending transaction:
     tx = mkBasicTx (mkBasicTxBody
           & inputsTxBodyL  .~ [lockedTxIn]
           & outputsTxBodyL .~ [attackerOutput]
           & guardsTxBodyL  .~ [KeyHashObj victimKH])   -- no signature for victimKH

3. Sign tx with attacker's own fee-paying key only.

4. Submit tx.

Result:
  - evalDijkstraNativeScript evaluates RequireGuard victimKH
      → victimKH ∈ {KeyHashObj victimKH} → True
  - validateNeededWitnesses finds no missing witnesses
      (guards are absent from getShelleyWitsVKeyNeededNoGov)
  - Transaction accepted; attacker receives the locked funds.
``` [6](#0-5) [7](#0-6) [8](#0-7)

evalDijkstraNativeScript ::
  (DijkstraEraScript era, NativeScript era ~ DijkstraNativeScript era) =>
  Set.Set (KeyHash Witness) ->
  ValidityInterval ->
  OSet (Credential Guard) ->
  NativeScript era ->
  Bool
evalDijkstraNativeScript keyHashes (ValidityInterval txStart txExp) guards = go
  where
    -- the evaluation will stop as soon as it reaches the required number of valid scripts
    isValidMOf n SSeq.Empty = n <= 0
    isValidMOf n (ts SSeq.:<| tss) =
      n <= 0 || if go ts then isValidMOf (n - 1) tss else isValidMOf n tss
    go = \case
      RequireTimeStart lockStart -> lockStart `lteNegInfty` txStart
      RequireTimeExpire lockExp -> txExp `ltePosInfty` lockExp
      RequireSignature hash -> hash `Set.member` keyHashes
      RequireAllOf xs -> all go xs
      RequireAnyOf xs -> any go xs
      RequireMOf m xs -> isValidMOf m xs
      RequireGuard cred -> cred `OSet.member` guards
      _ -> error "Impossible: All NativeScripts should have been accounted for"
```

getShelleyWitsVKeyNeededNoGov ::
  EraTx era =>
  UTxO era ->
  TxBody l era ->
  Set (KeyHash Witness)
getShelleyWitsVKeyNeededNoGov utxo' txBody =
  certAuthors
    `Set.union` inputAuthors
    `Set.union` owners
    `Set.union` wdrlAuthors
  where
    inputAuthors :: Set (KeyHash Witness)
    inputAuthors = foldr' accum Set.empty (txBody ^. spendableInputsTxBodyF)
      where
        accum txin !ans =
          case txinLookup txin utxo' of
            Just txOut ->
              case txOut ^. addrTxOutL of
                Addr _ (KeyHashObj pay) _ -> Set.insert (asWitness pay) ans
                AddrBootstrap bootAddr ->
                  Set.insert (asWitness (bootstrapKeyHash bootAddr)) ans
                _ -> ans
            Nothing -> ans

    wdrlAuthors :: Set (KeyHash Witness)
    wdrlAuthors = Map.foldrWithKey' accum Set.empty (unWithdrawals (txBody ^. withdrawalsTxBodyL))
      where
        accum key _ !ans =
          let cred = key ^. accountAddressCredentialL
           in case credKeyHashWitness cred of
                Nothing -> ans
                Just vkeyWit -> Set.insert vkeyWit ans
    owners :: Set (KeyHash Witness)
    owners = foldr' accum Set.empty (txBody ^. certsTxBodyL)
      where
        accum (RegPoolTxCert pool) !ans =
          Set.union
            (Set.map asWitness (sppOwners pool))
            ans
        accum _cert ans = ans
    certAuthors :: Set (KeyHash Witness)
    certAuthors = foldr' accum Set.empty (txBody ^. certsTxBodyL)
      where
        accum cert !ans =
          case getVKeyWitnessTxCert cert of
            Nothing -> ans
            Just vkeyWit -> Set.insert vkeyWit ans
