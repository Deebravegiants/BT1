### Title
Missing Validation of `ProposalInit.builder` (Sequencer Address) Allows Zero-Address Block Commitment - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates timestamp, starknet version, gas prices, and fee proposals, but never checks that `ProposalInit.builder` — the sequencer address embedded in every block hash — is a valid non-zero address. A malicious proposer can set `builder = ContractAddress::default()` (zero), causing all validators to execute and commit a block whose `PartialBlockHashComponents.sequencer` is zero, misdirecting all sequencer fee payments to the zero address and producing a permanently wrong block hash.

### Finding Description

`ProposalInit.builder` is the field that becomes `BlockInfo.sequencer_address` for the entire block execution. It flows through two critical paths:

**Path 1 — Block execution / fee accounting:**
`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in `BlockInfo` with no range check: [1](#0-0) 

The blockifier then uses `sequencer_address` to compute the storage keys where sequencer fees are deposited: [2](#0-1) 

If `sequencer_address = 0x0`, all fee transfers write to storage slots under address zero — a valid storage location that no real contract owns — permanently destroying the fees.

**Path 2 — Block hash / commitment:**
`PartialBlockHashComponents::new` copies `sequencer_address` into the `sequencer` field: [3](#0-2) 

`calculate_block_hash` chains this value directly into the Poseidon hash: [4](#0-3) 

A zero `builder` therefore produces a `PartialBlockHash` — and thus a `ProposalCommitment` — that encodes the wrong sequencer, which is then stored permanently.

**The missing guard:**
`is_proposal_init_valid` checks timestamp, starknet version, `version_constant_commitment`, height, L1 DA mode, L2 gas price, L1 gas prices (within margin), and `fee_proposal_fri`, but contains no check on `init.builder`: [5](#0-4) 

`ContractAddress::validate()` already exists and enforces the range `[0x2, L2_ADDRESS_UPPER_BOUND)`: [6](#0-5) 

It is called for transaction `sender_address` in the gateway: [7](#0-6) 

But it is never called on `init.builder` anywhere in the proposal validation path.

The codebase itself acknowledges zero is invalid for contract addresses: [8](#0-7) 

### Impact Explanation

A malicious proposer sets `ProposalInit.builder = ContractAddress::default()` (zero). The validator's `is_proposal_init_valid` passes without error. `initiate_validation` calls `convert_to_sn_api_block_info` which constructs a `BlockInfo` with `sequencer_address = 0x0` and passes it to the batcher. The batcher executes the block, crediting all sequencer fees to address zero. The batcher computes `PartialBlockHashComponents` with `sequencer = 0x0`, producing a `ProposalCommitment` that both proposer and validator agree on (since both derive it from the same `init.builder`). Consensus reaches decision, `decision_reached` commits the block. The committed block hash permanently encodes a zero sequencer address, and all fee revenue for that block is lost.

This matches: **Critical — Incorrect fee, gas, resource accounting, balance with economic impact** and **Wrong state, receipt, or storage value from blockifier/syscall/execution logic**.

### Likelihood Explanation

In the current single-sequencer Starknet deployment the proposer is trusted. In the multi-validator decentralized deployment this codebase targets (Tendermint consensus, multiple `ContractAddress` validators), any validator selected as proposer for a round can trigger this with a single crafted `ProposalInit`. No special privilege beyond being selected as proposer is required. The attack is trivially constructable: set `builder` to `Felt::ZERO` in the outbound `ProposalInit`.

### Recommendation

Add a call to `init.builder.validate()` (or an equivalent non-zero check) inside `is_proposal_init_valid` in `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, immediately after the existing gas-price checks:

```rust
init.builder.validate().map_err(|e| {
    ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("Invalid builder address: {e}"),
    )
})?;
```

Optionally, also verify that `init.builder` matches the locally-configured `builder_address` (from `ContextStaticConfig`) so that a proposer cannot redirect fees to an arbitrary but non-zero address.

### Proof of Concept

1. A validator is selected as proposer for round R.
2. It constructs `ProposalInit { builder: ContractAddress::default(), .. }` (all other fields valid).
3. It streams this `ProposalInit` to peer validators.
4. Each peer calls `validate_proposal` → `is_proposal_init_valid`: all checks pass (builder is not checked).
5. `initiate_validation` → `convert_to_sn_api_block_info` → `BlockInfo { sequencer_address: ContractAddress(0), .. }` is sent to the batcher.
6. The batcher executes the block; `get_sequencer_balance_keys(ContractAddress(0))` computes storage slots under address zero; fee transfers write there.
7. `PartialBlockHashComponents { sequencer: SequencerContractAddress(ContractAddress(0)), .. }` is hashed; the resulting `ProposalCommitment` matches the proposer's `ProposalFin`.
8. `validate_proposal` returns `Ok(commitment)`; consensus votes `PREVOTE`/`PRECOMMIT`; `decision_reached` commits the block.
9. All sequencer fees for the block are permanently lost; the committed block hash encodes a zero sequencer address.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
