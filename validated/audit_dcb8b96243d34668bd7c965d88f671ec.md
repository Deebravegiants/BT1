### Title
NFT Royalty Payment Silently Zeroed via Integer Division Truncation in `compute_royalty_amount` — (`chia/wallet/nft_wallet/nft_wallet.py`)

### Summary
The `compute_royalty_amount` function uses sequential integer floor division that silently truncates to zero for small offered amounts. When the royalty truncates to zero, downstream code in `make_nft1_offer` also filters the NFT's `trade_prices_list` to empty, removing on-chain royalty enforcement conditions entirely. An unprivileged buyer can deliberately offer a small fungible amount to pay zero royalty to the NFT creator, bypassing offer settlement accounting for NFT royalties.

### Finding Description

`compute_royalty_amount` in `chia/wallet/nft_wallet/nft_wallet.py` computes royalty as:

```python
amount = abs(offered_amount) // royalty_split * percentage // MAX_ROYALTY_BASIS_POINTS
``` [1](#0-0) 

`MAX_ROYALTY_BASIS_POINTS = 10000`. Because Python integer division truncates toward zero, any `offered_amount` where `abs(offered_amount) // royalty_split * percentage < 10000` produces `royalty = 0`. The existing test suite explicitly confirms and accepts this behavior:

```python
def test_small_amount_truncates_to_zero() -> None:
    result = compute_royalty_amount(offered_amount=-50, royalty_split=1, percentage=100)
    assert result == uint64(0)
``` [2](#0-1) 

When `compute_royalty_amount` returns 0, two downstream effects compound the bypass:

**Effect 1 — `trade_prices_list` is emptied (on-chain enforcement removed):**

In `make_nft1_offer`, the `trade_prices_list` passed to the NFT coin spend is filtered:

```python
trade_prices_list=[
    list(price)
    for price in trade_prices
    if price[0] * offered_royalty_percentages[asset] // MAX_ROYALTY_BASIS_POINTS != 0
],
``` [3](#0-2) 

If the royalty on the trade price is zero, the entry is excluded. An empty `trade_prices_list` is passed to the CLVM `NFT_TRANSFER_PROGRAM_DEFAULT` puzzle, which then generates no royalty payment conditions on-chain.

**Effect 2 — Royalty coin spend is skipped entirely:**

```python
if sum(p.amount for _, p in payments) == 0:
    continue
``` [4](#0-3) 

No royalty coin is created in the spend bundle. The NFT transfer settles on-chain with zero royalty paid to the creator.

### Impact Explanation

An unprivileged buyer (offer maker) can craft an offer with a fungible amount small enough to cause `compute_royalty_amount` to return 0. The resulting spend bundle contains no royalty payment coin and no on-chain royalty enforcement conditions in the NFT transfer program solution. The NFT creator's entitled royalty is silently zeroed out in the settled offer. This is an unauthorized accounting change affecting NFT offer settlement — the royalty recipient (a third party to the trade) loses funds they are entitled to without any on-chain mechanism to detect or prevent it.

### Likelihood Explanation

The threshold for zero royalty is reachable with normal mojo-denominated amounts. For a 1% royalty (100 basis points), any offer below 10,000 mojos produces zero royalty. For a 0.01% royalty (1 basis point), any offer below 10,000 mojos also produces zero. In CAT markets where amounts are denominated in smaller units, this threshold is easily reached. The attacker only needs to create a valid offer — no special privileges, keys, or access are required.

### Recommendation

Reorder the arithmetic to multiply before dividing, preventing premature truncation:

```python
amount = abs(offered_amount) * percentage // royalty_split // MAX_ROYALTY_BASIS_POINTS
```

Additionally, enforce a minimum royalty of 1 mojo when `percentage > 0` and `offered_amount > 0`, so that a non-zero royalty percentage always results in at least 1 mojo paid to the creator.

MAX_ROYALTY_BASIS_POINTS = 10000


def compute_royalty_amount(offered_amount: int, royalty_split: int, percentage: int) -> uint64:
    """Compute royalty using integer arithmetic, validating against overflow and excessive percentage."""
    if percentage > MAX_ROYALTY_BASIS_POINTS:
        raise ValueError(f"NFT royalty percentage {percentage} exceeds 100% ({MAX_ROYALTY_BASIS_POINTS} basis points)")
    amount = abs(offered_amount) // royalty_split * percentage // MAX_ROYALTY_BASIS_POINTS
    royalty = uint64(amount)
    if royalty >= abs(offered_amount):
        raise ValueError("Royalty amount meets or exceeds the offered amount")
    return royalty
```
