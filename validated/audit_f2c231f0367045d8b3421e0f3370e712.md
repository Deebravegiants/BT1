### Title
Single Reverting Asset Price Oracle Permanently Blocks All rsETH Price Updates — (File: `contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle._getTotalEthInProtocol()` iterates over every supported asset and calls `getAssetPrice(asset)` inside the loop. If any one asset's price oracle reverts — due to a Chainlink circuit breaker, stale-price guard, sequencer downtime, or the oracle address being zeroed — the entire `updateRSETHPrice()` call reverts. No asset's price contribution can be summed, and the rsETH exchange rate cannot be updated until an admin manually fixes the broken oracle.

---

### Finding Description

`_getTotalEthInProtocol()` is the private aggregation function called by every code path that updates the rsETH price:

```
updateRSETHPrice() (public)
updateRSETHPriceAsManager() (manager-only)
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ for each supportedAsset:
                 getAssetPrice(asset)   ← can revert
```

The loop in question:

```solidity
// LRTOracle.sol lines 336–348
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);          // ← hard revert if oracle fails
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    unchecked { ++assetIdx; }
}
```

`getAssetPrice` is guarded by `onlySupportedOracle`, which reverts when `assetPriceOracle[asset] == address(0)`:

```solidity
// LRTOracle.sol lines 40–45
modifier onlySupportedOracle(address asset) {
    if (assetPriceOracle[asset] == address(0)) {
        revert AssetOracleNotSupported();
    }
    _;
}
```

And the underlying `IPriceFetcher.getAssetPrice(asset)` call can itself revert (e.g., Chainlink `latestRoundData` reverts on sequencer downtime or when the aggregator is deprecated).

There is no `try/catch`, no `continue`, and no skip logic. A single failing oracle for any one of the supported assets causes the entire loop — and therefore the entire price update — to revert.

---

### Impact Explanation

When `updateRSETHPrice()` is blocked:

1. **Permanent freezing of unclaimed yield** — Protocol fees are minted inside `_updateRsETHPrice()` via `IRSETH.mint(treasury, rsethAmountToMintAsProtocolFee)`. If the function can never complete, accrued yield is never minted to the treasury.
2. **Contract fails to deliver promised returns** — The stored `rsETHPrice` becomes stale. All depositors and withdrawers transact at an incorrect exchange rate for as long as the oracle is broken.
3. **Price-drop auto-pause cannot trigger** — The downside protection that pauses `LRTDepositPool` and `LRTWithdrawalManager` on a large price drop also lives inside `_updateRsETHPrice()` and is therefore also blocked.

Impact classification: **Medium — Permanent freezing of unclaimed yield** and **Low — Contract fails to deliver promised returns**.

---

### Likelihood Explanation

The protocol supports multiple LST assets (stETH, ETHx, etc.), each with its own price oracle adapter. Chainlink feeds are known to revert under several real-world conditions: L2 sequencer downtime, deprecated aggregators, and stale-price circuit breakers. With N supported assets, the probability that at least one oracle is temporarily unavailable grows with N. No attacker action is required; ordinary market or infrastructure conditions are sufficient to trigger the block.

---

### Recommendation

Wrap the `getAssetPrice` call in a `try/catch` and skip (or use a fallback price for) any asset whose oracle reverts, so the remaining assets can still be aggregated and the price update can proceed:

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    try this.getAssetPrice(asset) returns (uint256 assetER) {
        uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
        totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    } catch {
        // emit an event or revert with a more informative error;
        // at minimum, do not block the entire update
    }
    unchecked { ++assetIdx; }
}
```

Alternatively, revert the entire call only when the failing asset has a non-negligible TVL contribution, and skip it otherwise — mirroring the recommended fix in the referenced AMM report.

---

### Proof of Concept

1. Protocol has two supported assets: `stETH` (oracle healthy) and `ETHx` (oracle reverts — e.g., Chainlink aggregator deprecated on L2 after sequencer restart).
2. Anyone calls `updateRSETHPrice()`.
3. `_getTotalEthInProtocol()` enters the loop, processes `stETH` successfully, then calls `getAssetPrice(ETHx)`.
4. The `ETHx` oracle reverts. The entire transaction reverts.
5. `rsETHPrice` is never updated. Protocol fees are never minted. The price-drop auto-pause cannot fire.
6. This persists until an admin calls `updatePriceOracleFor(ETHx, newOracle)` — but during that window all price-dependent protocol functions operate on a stale rate.
