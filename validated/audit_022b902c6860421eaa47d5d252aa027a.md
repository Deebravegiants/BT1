### Title
Stale TEE Attestation Passes `is_caller_an_attested_participant` Without Re-Verification — (`File: crates/contract/src/tee/tee_state.rs`)

---

### Summary

`is_caller_an_attested_participant` checks only that a stored attestation *exists* and matches the caller's identity. It never re-verifies whether the stored attestation is still valid under the current time, Docker-image whitelist, or TEE-measurement whitelist. A participant whose attestation has since expired, or whose image hash has been removed from the allowed set, continues to pass this gate and can cast governance votes — directly analogous to the BoostController bug where a delegation persists after the backing veRAAC is withdrawn.

---

### Finding Description

When a node joins the protocol it calls `add_participant`, which runs full attestation verification at that moment: [1](#0-0) 

The `AcceptedAttestation` is stored in `stored_attestations` keyed by TLS public key. From that point on, every governance call that requires TEE authorization goes through `is_caller_an_attested_participant`: [2](#0-1) 

This function performs three checks: the caller is a known participant, a stored attestation exists for their TLS key, and the stored `account_public_key` matches the signer's key. It does **not** call `reverify_participants` and does **not** check:

- whether the attestation's expiry timestamp has passed,
- whether the Docker image hash embedded in the attestation is still in `allowed_docker_image_hashes`,
- whether the TEE measurements are still in `allowed_measurements`.

The contract *does* provide `reverify_and_cleanup_participants` and `clean_invalid_attestations` for periodic cleanup: [3](#0-2) [4](#0-3) 

But these are opt-in, externally triggered calls. They are **not** invoked inside `is_caller_an_attested_participant`, so between cleanup invocations the stale attestation remains effective.

---

### Impact Explanation

**Medium — participant-state manipulation that breaks production safety/accounting invariants.**

The protocol's safety invariant is: only nodes whose TEE attestation is currently valid (unexpired, running an approved image, matching approved measurements) may participate in governance. A node whose image hash was removed from the whitelist (e.g., after a security patch mandates an upgrade) or whose attestation certificate has expired retains full voting rights until an explicit cleanup call is made. That node can:

- vote on `vote_new_parameters` (resharing to a new participant set),
- vote on `vote_add_domains` / `vote_cancel_keygen`,
- vote on TEE whitelist changes (`vote`, `vote_launcher`, `vote_measurement`).

If the threshold is low (e.g., 2-of-3) and the stale node's vote is the deciding one, governance decisions are made by a node the protocol has effectively de-authorized. This breaks the accounting invariant that vote weight reflects currently-authorized TEE participants.

---

### Likelihood Explanation

Realistic. TEE attestation certificates have finite lifetimes (typically 30–90 days for DCAP). Between the moment an attestation expires and the moment an operator calls `clean_invalid_attestations`, the stale node can freely cast governance votes. Similarly, when the governance removes an old Docker image hash (routine during upgrades), nodes still running the old image pass `is_caller_an_attested_participant` until cleanup runs. Neither scenario requires privileged access or collusion — it is a natural consequence of the upgrade/rotation lifecycle.

---

### Recommendation

Inside `is_caller_an_attested_participant`, after confirming the attestation exists and the identity matches, call `reverify_participants` on the retrieved `NodeId` and return `AttestationCheckError` if the result is `TeeQuoteStatus::Invalid`. This mirrors the pattern already used in `reverify_and_cleanup_participants`:

```rust
// after identity checks pass:
let node_id = attestation.node_id.clone();
match self.reverify_participants(&node_id, tee_upgrade_deadline_duration) {
    TeeQuoteStatus::Valid => Ok(()),
    TeeQuoteStatus::Invalid(reason) => Err(AttestationCheckError::AttestationExpiredOrRevoked(reason)),
}
```

This ensures every governance call is gated on the *current* validity of the attestation, not just its existence at registration time — eliminating the snapshot-at-creation gap.

---

### Proof of Concept

1. **Node registers** — calls `add_participant` with a valid DCAP attestation tied to Docker image hash `H1`. Attestation stored with expiry `T_exp`.
2. **Governance removes `H1`** — participants vote to remove the old image hash from `allowed_docker_image_hashes` (e.g., after a security patch). `clean_invalid_attestations` is not called immediately.
3. **Stale node votes** — the node (still running image `H1`) calls `vote_new_parameters` to initiate resharing to a participant set that includes a malicious node. `is_caller_an_attested_participant` checks: participant exists ✓, attestation stored ✓, keys match ✓ — returns `Ok(())`. The vote is counted.
4. **Threshold reached** — if the stale node's vote is the deciding one, resharing begins with a participant set that includes the attacker's node, granting it future signing capability.
5. **No re-verification occurred** — `reverify_participants` was never called during step 3; the stale attestation's image hash `H1` is no longer in `allowed_docker_image_hashes` but the check was never made. [2](#0-1) [5](#0-4)

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub fn reverify_and_cleanup_participants(
        &mut self,
        participants: &Participants,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeValidationResult {
        self.allowed_docker_image_hashes
            .cleanup_expired_hashes(tee_upgrade_deadline_duration);

        let participants_with_valid_attestation: Vec<_> = participants
            .participants()
            .iter()
            .filter(|(_, _, participant_info)| {
                // Use the stored NodeId (keyed by TLS public key) so the real
                // `account_public_key` participates in re-verification. If
                // there is no stored attestation for this TLS key, the
                // participant is invalid.
                let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
                else {
                    return false;
                };

                let tee_status =
                    self.reverify_participants(&node_id, tee_upgrade_deadline_duration);

                matches!(tee_status, TeeQuoteStatus::Valid)
            })
            .cloned()
            .collect();

        if participants_with_valid_attestation.len() != participants.len() {
            let participants_with_valid_attestation =
                Participants::init(participants.next_id(), participants_with_valid_attestation);

            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            }
        } else {
            TeeValidationResult::Full
        }
    }

    pub fn clean_invalid_attestations(
        &mut self,
        tee_upgrade_deadline_duration: Duration,
        max_scan: usize,
    ) -> u32 {
        let has_invalid_attestation = |node_id: &NodeId| {
            !matches!(
                self.reverify_participants(node_id, tee_upgrade_deadline_duration),
                TeeQuoteStatus::Valid
            )
        };

        // Materialize candidates before any mutation to avoid iterator invalidation.
        let invalid_tls_keys: Vec<Ed25519PublicKey> = self
            .stored_attestations
            .iter()
            .take(max_scan)
            .filter(|(_, node_attestation)| has_invalid_attestation(&node_attestation.node_id))
            .map(|(tls_pk, _)| tls_pk.clone())
            .collect();

        let removed = u32::try_from(invalid_tls_keys.len())
            .expect("u32 should always be convertible from usize on wasm32");

        for tls_pk in invalid_tls_keys {
            self.stored_attestations.remove(&tls_pk);
        }
        removed
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
