Audit Report

## Title
`recoverFrozenFunds()` Misses rsETH Locked in the Withdrawal Queue, Allowing Blocked Users to Receive Underlying Assets - (File: contracts/RSETH.sol)

## Summary
`RSETH.recoverFrozenFunds()` only seizes the rsETH balance held directly in the blocked user's wallet. When a user has already called `initiateWithdrawal()`, their rsETH resides in `LRTWithdrawalManager`, making `balanceOf(from)` return zero. Because `completeWithdrawal()` contains no block-enforcement check and transfers LST/ETH (not rsETH), a blocked user can still complete their pending withdrawal and receive the full underlying asset value, defeating the freeze mechanism entirely.

## Finding Description
When `initiateWithdrawal()` is called, rsETH is pulled from the user into `LRTWithdrawalManager`:

```solidity
// LRTWithdrawalManager.sol L166
IERC20(lrtConfig.rsETH()).safeTransferFrom(msg.sender, address(this), rsETHUnstaked);
```

After `blockUserTransfers()` is applied and `recoverFrozenFunds()` is called, the recovery reads only the user's wallet balance:

```solidity
// RSETH.sol L215-219
uint256 accountBalance = balanceOf(from);
super._transfer(from, custodyAddress, accountBalance);
emit FrozenFundsRecovered(from, custodyAddress, accountBalance);
```

Since the rsETH is held by `LRTWithdrawalManager`, `balanceOf(from) == 0` and nothing is recovered.

`_processWithdrawalCompletion()` (called by both `completeWithdrawal()` and `completeWithdrawalForUser()`) has no `_enforceNotBlocked` check anywhere in its execution path. It calls `_transferAsset(asset, user, request.expectedAssetAmount)`, which transfers LST or ETH — not rsETH — so the rsETH transfer hook that would invoke `_enforceNotBlocked` is never reached.

During `unlockQueue()`, rsETH is burned from `address(this)` (the manager):

```solidity
// LRTWithdrawalManager.sol L305
if (rsETHBurned != 0) IRSETH(lrtConfig.rsETH()).burnFrom(address(this), rsETHBurned);
```

`burnFrom` calls `_enforceNotBlocked(address(this))` — the manager is not blocked, so the burn succeeds and the underlying asset is allocated to the blocked user's request. The blocked user then calls `completeWithdrawal()` with no impediment.

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

The `blockUserTransfers()` + `recoverFrozenFunds()` mechanism is designed to allow the protocol to fully freeze and seize a user's rsETH (e.g., for sanctions compliance). When a user has a pending withdrawal, `recoverFrozenFunds()` silently recovers zero, and the user can still call `completeWithdrawal()` to receive the full underlying LST/ETH value. No third-party funds are lost, but the freeze mechanism fails to deliver its stated guarantee.

## Likelihood Explanation
Any user who called `initiateWithdrawal()` before being blocked is in this state. The withdrawal delay is 8 days (`withdrawalDelayBlocks = 8 days / 12 seconds`), providing a substantial window during which a pending withdrawal can exist when a block is applied. No front-running is required — the user need not anticipate the block. The condition is naturally reachable by any user with an active withdrawal request.

## Recommendation
1. `recoverFrozenFunds()` should also cancel and recover rsETH held in `LRTWithdrawalManager` on behalf of the blocked user — e.g., by iterating `userAssociatedNonces` for all assets, burning the queued rsETH from the manager on behalf of the blocked user's requests, and crediting the committed assets back to `assetsCommitted`.
2. `_processWithdrawalCompletion()` should check whether the recipient (`user`) is currently blocked via `_enforceNotBlocked(user)` and revert if so, consistent with the intent of the freeze mechanism.

## Proof of Concept
1. Alice holds 100 rsETH and calls `LRTWithdrawalManager.initiateWithdrawal(stETH, 100e18, "")`. Her 100 rsETH is transferred to `LRTWithdrawalManager`; `RSETH.balanceOf(Alice) == 0`.
2. Manager calls `RSETH.blockUserTransfers([Alice])`. `transfersBlockedUntil[Alice] = block.timestamp + 1 days`.
3. Admin calls `RSETH.recoverFrozenFunds(Alice)`. `balanceOf(Alice) == 0`, so `accountBalance = 0` and nothing is recovered. `FrozenFundsRecovered` is emitted with amount 0.
4. After the 8-day withdrawal delay, operator calls `unlockQueue(stETH, ...)`. `burnFrom(address(LRTWithdrawalManager), 100e18)` succeeds (manager is not blocked), and stETH is allocated to Alice's request.
5. Alice calls `completeWithdrawal(stETH, "")`. No block check is performed; `_transferAsset(stETH, Alice, expectedAmount)` executes and Alice receives the full stETH amount.

Alice successfully receives her underlying assets despite being blocked, and the admin's `recoverFrozenFunds()` call recovered nothing.

    function _processWithdrawalCompletion(address asset, address user, string calldata referralId) internal {
        if (userAssociatedNonces[asset][user].empty()) {
            revert NoWithdrawalRequests(user, asset);
        }

        // Retrieve and remove the oldest withdrawal request for the user.
        uint256 usersFirstWithdrawalRequestNonce = userAssociatedNonces[asset][user].popFront();
        // Ensure the request is already unlocked.
        if (usersFirstWithdrawalRequestNonce >= nextLockedNonce[asset]) revert WithdrawalLocked();

        bytes32 requestId = getRequestId(asset, usersFirstWithdrawalRequestNonce);
        WithdrawalRequest memory request = withdrawalRequests[requestId];

        delete withdrawalRequests[requestId];

        // Check that the withdrawal delay has passed since the request's initiation.
        if (block.number < request.withdrawalStartBlock + withdrawalDelayBlocks) revert WithdrawalDelayNotPassed();

        unlockedWithdrawalsCount[asset]--;

        // If Aave integration is enabled and asset is ETH, withdraw from Aave if needed
        if (isAaveIntegrationEnabled && asset == LRTConstants.ETH_TOKEN) {
            uint256 contractBalance = address(this).balance;
            if (contractBalance < request.expectedAssetAmount) {
                uint256 amountNeeded = request.expectedAssetAmount - contractBalance;
                _withdrawFromAave(amountNeeded);

                // Verify we have sufficient balance after withdrawal
                uint256 balanceAfter = address(this).balance;
                if (balanceAfter < request.expectedAssetAmount) {
                    revert InsufficientLiquidityForWithdrawal();
                }
            }
        }

        _transferAsset(asset, user, request.expectedAssetAmount);

        emit ReferralIdEmitted(referralId);
        emit AssetWithdrawalFinalized(user, asset, request.rsETHUnstaked, request.expectedAssetAmount);
    }
