### Title
Sub-Transaction Plutus Script Execution Bypass via Missing `babbageMissingScripts` Check in `SUBUTXOW` Rule — (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxow.hs`)

---

### Summary

In the Dijkstra era, the `SUBUTXOW` rule (`dijkstraSubUtxowTransition`) omits the `babbageMissingScripts` check that is present in the top-level `UTXOW` rule. The top-level `UTXOW` performs an aggregated missing-script check across all transaction levels, which can be satisfied by placing a required script in the top-level transaction's witnesses. Because sub-transaction script execution only draws from the sub-transaction's own witnesses and reference scripts, a Plutus script needed by a sub-transaction can be absent from the sub-transaction's witness set, causing the script to never be evaluated while the script-locked UTxO is still consumed.

---

### Finding Description

**Root cause — missing check in `dijkstraSubUtxowTransition`:**

The top-level `dijkstraUtxowTransition` (Dijkstra `UTXOW`) performs an aggregated `babbageMissingScripts` check: [1](#0-0) 

It builds `allScriptHashesNeeded` as the union of scripts needed by the top-level transaction and every sub-transaction, and `witnessScripts` as the union of witness scripts from all levels. `babbageMissingScripts` then enforces that every needed script is covered by either a witness or a reference script — across the entire bundle.

The sub-transaction rule `dijkstraSubUtxowTransition` (`SUBUTXOW`) performs several checks: [2](#0-1) 

It calls `validateFailedBabbageScripts` (line 267), `validateNeededWitnesses` (line 270), `missingRequiredDatums` (line 273), `checkScriptIntegrityHash` (line 279), `hasExactSetOfRedeemers` (line 281), and `validateScriptsWellFormedTxOuts` (lines 283–287). **`babbageMissingScripts` is never called.**

**Why the aggregated top-level check does not protect sub-transaction execution:**

`babbageMissingScripts` in the top-level `UTXOW` is satisfied as long as the script hash appears in *any* level's witness set: [3](#0-2) 

If script A is placed in the top-level transaction's witnesses, `witnessScripts` contains it, and `missing` is empty — the check passes.

However, when the sub-transaction is processed through `SUBUTXO → UTXOS`, `collectTwoPhaseScriptInputs` only collects scripts where `scriptHash ↦ script ∈ txscripts (txwits subTx)`. The sub-transaction's `txscripts` is derived from its own witnesses and reference scripts — it does **not** include scripts from the top-level transaction's witnesses. [4](#0-3) 

Because script A is absent from the sub-transaction's `txscripts`, `collectTwoPhaseScriptInputs` does not collect it, `evalScripts` never evaluates it, and the script-locked UTxO is consumed without any script execution.

**`hasExactSetOfRedeemers` does not close the gap:**

`hasExactSetOfRedeemers` in `SUBUTXOW` (line 281) requires redeemers only for scripts that are both in `scriptsNeeded` *and* in the sub-transaction's `txscripts`. Since script A is absent from the sub-transaction's `txscripts`, no redeemer is required, and the check passes silently. [5](#0-4) 

**Ledger execution order confirms the gap:**

The `dijkstraLedgerTransition` processes sub-transactions via `SUBLEDGERS` first, then calls the top-level `UTXOW`: [6](#0-5) 

The top-level `UTXOW` runs after sub-transactions have already been applied to the UTxO state, so its aggregated check cannot retroactively enforce script execution within the already-processed sub-transactions.

---

### Impact Explanation

**Critical — Direct loss of ADA or native assets through an invalid ledger state transition.**

Any UTxO locked by a Plutus script that is not deployed as a reference script can be spent by a sub-transaction without executing the script. This completely bypasses the script-based access control model: the Plutus script's logic (e.g., multi-sig threshold, time-lock, DeFi invariant) is never evaluated. An attacker who can construct a Dijkstra-era transaction with a sub-transaction can drain any such UTxO unconditionally.

---

### Likelihood Explanation

**Medium.** The Dijkstra era is not yet active on mainnet, but it is the production implementation path in this repository. Once the era activates, the attack requires only the ability to submit a well-formed transaction — no privileged access, no key compromise, no governance majority. The attacker must know the script (to include it in the top-level witnesses to satisfy the aggregated check) and must target a UTxO not protected by a reference-script deployment. Both conditions are routinely met in practice.

---

### Recommendation

Add a per-sub-transaction `babbageMissingScripts` call inside `dijkstraSubUtxowTransition`, checking that all scripts needed by the sub-transaction are present in the sub-transaction's own witness set or accessible as reference scripts — mirroring the check already present in `babbageUtxowTransition` and `dijkstraUtxowTransition`. The aggregated top-level check is insufficient because it conflates witness availability across levels with per-level script executability.

### Proof of Concept

```
-- Attacker constructs a Dijkstra-era top-level transaction T containing:
--   sub-transaction S that spends UTxO U locked by Plutus script A
--   (script A is NOT a reference script in the UTxO)
--
-- S.witnesses.scripts  = {}          -- script A absent from sub-tx witnesses
-- S.witnesses.redeemers = {}         -- no redeemer needed (script not in txscripts)
-- T.witnesses.scripts  = { A }       -- script A present in top-level witnesses only
--
-- Validation path:
--
-- 1. SUBLEDGERS → SUBUTXOW for S:
--    - validateFailedBabbageScripts: passes (A not in S.scriptsProvided)
--    - hasExactSetOfRedeemers:       passes (A not in S.txscripts → no redeemer required)
--    - babbageMissingScripts:        NOT CALLED  ← gap
--    → S is accepted; UTxO U is consumed; script A never executed
--
-- 2. UTXOW for T:
--    - allScriptHashesNeeded = {A}   (from S.scriptsNeeded)
--    - witnessScripts        = {A}   (from T.witnesses)
--    - babbageMissingScripts:        passes (A ∈ witnessScripts)
--    → T is accepted
--
-- Result: UTxO U spent; Plutus script A never evaluated.

babbageMissingScripts _ sNeeded sRefs sReceived =
  sequenceA_
    [ failureOnNonEmptySet extra Shelley.ExtraneousScriptWitnessesUTXOW
    , failureOnNonEmptySet missing Shelley.MissingScriptWitnessesUTXOW
    ]
  where
    neededNonRefs = sNeeded `Set.difference` sRefs
    missing = neededNonRefs `Set.difference` sReceived
    extra = sReceived `Set.difference` neededNonRefs
