### Title
Missing Time-Based Staleness Check in `getRate()` Allows Stale Collateral Price to Over-Mint wrsETH — (`contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`)

---

### Summary

`ChainlinkOracleForRSETHPoolCollateral.getRate()` validates Chainlink data with only `answeredInRound < roundID` and `timestamp == 0`, but **never checks `block.timestamp - updatedAt > STALENESS_THRESHOLD`**. A Chainlink feed that has not been updated for an arbitrarily long time (e.g., 48 h) but whose last round was self-answered (`answeredInRound == roundID`, `timestamp != 0`) passes all guards and returns the stale price. When that stale price is inflated relative to the current market price of the collateral token, any caller of `deposit(token, amount, referralId)` on `RSETHPoolV3` / `RSETHPoolV3ExternalBridge` receives more wrsETH than the deposited collateral is worth.

---

### Finding Description

`getRate()` performs three checks:

```solidity
if (answeredInRound < roundID) revert StalePrice();   // round-based only
if (timestamp == 0)            revert IncompleteRound();
if (ethPrice <= 0)             revert InvalidPrice();
``` [1](#0-0) 

There is no check of the form:

```solidity
if (block.timestamp - timestamp > MAX_DELAY) revert StalePrice();
```

The `answeredInRound == roundID` condition only confirms the answer was computed in the same round it was opened — it says nothing about how long ago that round closed. A feed that last updated 48 hours ago with `answeredInRound == roundID` passes every guard and returns the old price.

This oracle is used as the `supportedTokenOracle` for collateral tokens in `RSETHPoolV3` and `RSETHPoolV3ExternalBridge`:

```solidity
uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate();
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
``` [2](#0-1) [3](#0-2) 

An inflated `tokenToETHRate` directly inflates `rsETHAmount`, causing `wrsETH.mint(msg.sender, rsETHAmount)` to mint more shares than the deposited collateral backs. [4](#0-3) 

---

### Impact Explanation

**Critical — Direct theft of user/protocol funds.**

If the collateral token's real market price has fallen while the Chainlink feed is stale (e.g., a liquid-staking token depegs), an attacker deposits the now-cheaper token and receives wrsETH priced at the old, higher rate. The attacker can immediately redeem or sell the excess wrsETH, extracting value that was never deposited. The protocol's wrsETH supply becomes under-collateralised, harming all existing holders.

---

### Likelihood Explanation

Chainlink feeds can go stale during network congestion, keeper failures, or low-volatility periods where the heartbeat is not triggered. The attacker only needs to:

1. Monitor the `updatedAt` field of the relevant Chainlink feed.
2. Wait for a period where the feed is stale **and** the real market price has moved downward.
3. Call `deposit(token, amount, referralId)` — a fully permissionless, public function.

No privileged role, no governance capture, and no oracle operator compromise is required. The vulnerability is entirely in the production contract's missing guard.

---

### Recommendation

Add a configurable maximum staleness delay and enforce it in `getRate()`:

```solidity
uint256 public constant MAX_STALENESS = 3600; // e.g. 1 hour

function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID) revert StalePrice();
    if (timestamp == 0) revert IncompleteRound();
    if (block.timestamp - timestamp > MAX_STALENESS) revert StalePrice(); // ← ADD THIS
    if (ethPrice <= 0) revert InvalidPrice();

    return uint256(ethPrice) * 1e18 / 10 ** uint256(AggregatorV3Interface(oracle).decimals());
}
``` [5](#0-4) 

The threshold should be set per-feed based on its documented heartbeat interval.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";

// Minimal mock that satisfies answeredInRound == roundID, timestamp != 0,
// but updatedAt is 48 hours in the past with an inflated price.
contract MockStaleAggregator {
    function decimals() external pure returns (uint8) { return 8; }
    function latestRoundData() external view returns (
        uint80 roundId, int256 answer, uint256 startedAt,
        uint256 updatedAt, uint80 answeredInRound
    ) {
        roundId        = 1;
        answeredInRound = 1;                          // passes answeredInRound < roundID check
        updatedAt      = block.timestamp - 48 hours;  // 48 h stale — NOT caught
        startedAt      = updatedAt;
        answer         = 2_000e8;                     // inflated: real price is now 1_000e8
    }
}

contract StalePriceTest is Test {
    function test_staleOraclePassesAllChecks() public {
        MockStaleAggregator agg = new MockStaleAggregator();
        ChainlinkOracleForRSETHPoolCollateral oracle =
            new ChainlinkOracleForRSETHPoolCollateral(address(agg));

        // Should revert with StalePrice — but it does NOT
        uint256 rate = oracle.getRate();

        // rate == 2_000e18 (inflated), real market rate is 1_000e18
        assertEq(rate, 2_000e18);

        // Attacker deposits 1 unit of collateral (real value: 1_000 ETH-equivalent)
        // and receives rsETHAmount based on 2_000 ETH-equivalent → 2× over-mint
    }
}
```

The test demonstrates that `getRate()` returns the stale inflated price without reverting, satisfying the precondition for the over-minting attack through `deposit(token, amount, referralId)`. [5](#0-4)
