### Title
Missing Block-Height Visibility Check for Cairo 0 Classes in `ExecutionStateReader::get_compiled_class` — (`crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary

`ExecutionStateReader::get_compiled_class` enforces a `state_number`-bounded visibility check for `ContractClass::V1` (Sierra/CASM) classes fetched from the class manager, but unconditionally returns `ContractClass::V0` (Cairo 0 / deprecated) classes without any such check. A developer TODO comment at the exact location explicitly acknowledges this gap. An unprivileged user can exploit this to make RPC execution (`starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`) succeed with a Cairo 0 class that was not yet declared at the requested block height, producing a wrong authoritative-looking execution result.

### Finding Description

In `get_compiled_class`, when `class_manager_handle` is `Some`, the code fetches the compiled class from the class manager and branches on its type:

- **V1 path** (lines 122–134): calls `is_contract_class_declared(storage_reader, class_hash, state_number)` and returns `UndeclaredClassHash` if the class was not declared at or before `state_number`.
- **V0 path** (lines 138–140): returns the class unconditionally, with only a TODO comment explaining the omission. [1](#0-0) 

The fallback storage path (lines 144–160) is correct: it calls `get_contract_class`, which uses `get_deprecated_class_definition_at(state_number, class_hash)` — a function that enforces block-height visibility for Cairo 0 classes. [2](#0-1) 

The class manager is a shared, persistent cache populated as blocks are processed. Once a Cairo 0 class is declared in block N+1, it is stored in the class manager and returned by `get_executable` for any subsequent query, regardless of the `state_number` of the requesting `ExecutionStateReader`.

### Impact Explanation

Any RPC endpoint that triggers blockifier execution with an `ExecutionStateReader` that has `class_manager_handle = Some` is affected. During execution, the blockifier calls `get_compiled_class(class_hash)` on the state reader whenever it needs to load a class — including for `library_call` syscalls, where the class hash is supplied directly by the executing contract's code.

If a contract deployed at block N issues a `library_call` with a class hash H that corresponds to a Cairo 0 class declared only at block N+1, the class manager returns the class without a block-height check, and the execution succeeds. The correct behavior is `StateError::UndeclaredClassHash(H)`. The RPC caller receives a wrong, authoritative-looking execution result.

This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

### Likelihood Explanation

The attack requires only public, unprivileged operations:

1. Pre-compute the class hash H of a Cairo 0 class C (deterministic from the class definition).
2. Deploy a contract at block N whose code issues `library_call(H, selector, calldata)`.
3. Submit a `declare` transaction for C at block N+1 (populates the class manager cache).
4. Call `starknet_call` / `starknet_estimateFee` / `starknet_simulateTransactions` at block N.

No validator, sequencer operator, or privileged access is required. The class manager cache is shared across all RPC requests, so step 3 permanently enables the bypass for any future query at block N.

### Recommendation

Apply the same `is_contract_class_declared` guard to the V0 branch. The underlying issue cited in the TODO — that `get_class_definition_block_number` does not cover deprecated classes — is already solved: `get_deprecated_class_definition_block_number` exists in storage and is used correctly by `is_class_declared_at` in `StateSync`. [3](#0-2) 

The fix is to call `is_contract_class_declared` (or an equivalent that checks `deprecated_declared_classes_block_table`) for the V0 branch before returning the class, mirroring the V1 guard at lines 123–134. [4](#0-3) 

### Proof of Concept

```
Setup:
  - class_manager returns ContractClass::V0(C) for class_hash H
  - storage has H declared in block N+1 (deprecated_declared_classes_block_table)
  - ExecutionStateReader.state_number = right_after_block(N)  [i.e., before N+1]
  - ExecutionStateReader.class_manager_handle = Some(...)

Call:
  exec_state_reader.get_compiled_class(H)

Expected: Err(StateError::UndeclaredClassHash(H))
Actual:   Ok(RunnableCompiledClass::V0(...))   ← wrong
```

This mirrors the existing test structure in `state_reader_test.rs` (lines 304–392), which already tests the V1 case (`class_not_declared_but_in_class_manager` → `UndeclaredClassHash`) but has no equivalent test for V0. [5](#0-4)

#[test]
fn get_compiled_class() {
    let casm_contract_class = CasmContractClass {
        compiler_version: "0.0.0".to_string(),
        prime: Default::default(),
        bytecode: Default::default(),
        bytecode_segment_lengths: Default::default(),
        hints: Default::default(),
        pythonic_hints: Default::default(),
        entry_points_by_type: Default::default(),
    };
    let expected_result = casm_contract_class.clone();

    let mut mock_class_manager_client = MockClassManagerClient::new();
    mock_class_manager_client.expect_get_executable().returning(move |_| {
        Ok(Some(ContractClass::V1((casm_contract_class.clone(), SierraVersion::default()))))
    });
    let shared_mock_class_manager_client = Arc::new(mock_class_manager_client);

    // Set execution state reader with state_number to be right after block 0.
    let (mut exec_state_reader, mut storage_writer) =
        set_execution_state_reader(shared_mock_class_manager_client.clone(), 0);

    // Prepare storage so class with hash 0x2 is declared in block 0 and class with hash 0x3 is
    // declared in block 1.
    let class_hash_0x2 = class_hash!("0x2");
    let class_hash_0x3 = class_hash!("0x3");

    storage_writer
        .begin_rw_txn()
        .unwrap()
        .append_header(BlockNumber(0), &BlockHeader::default())
        .unwrap()
        .append_state_diff(
            BlockNumber(0),
            ThinStateDiff {
                class_hash_to_compiled_class_hash: indexmap!(class_hash_0x2 => compiled_class_hash!(1_u8)),
                ..Default::default()
            },
        )
        .unwrap()
        .append_header(
            BlockNumber(1),
            &BlockHeader { block_hash: BlockHash(felt!(1_u128)), ..Default::default() },
        )
        .unwrap()
        .append_state_diff(
            BlockNumber(1),
            ThinStateDiff {
                class_hash_to_compiled_class_hash: indexmap!(class_hash_0x3 => compiled_class_hash!(2_u8)),
                ..Default::default()
            },
        )
        .unwrap()
        .commit()
        .unwrap();

    // At state_number right after block 0, class with hash 0x2 is already declared.
    let result = exec_state_reader.get_compiled_class(class_hash_0x2).unwrap();
    assert_eq!(
        result,
        RunnableCompiledClass::V1(
            (expected_result.clone(), SierraVersion::default()).try_into().unwrap()
        )
    );

    // Class with hash 0x3 is not yet declared.
    let result = exec_state_reader.get_compiled_class(class_hash_0x3);
    assert_matches!(result, Err(StateError::UndeclaredClassHash(hash)) if hash == class_hash_0x3);

    // Adjust execution state reader state_number to be right after block 1.
    exec_state_reader.state_number = StateNumber::unchecked_right_after_block(BlockNumber(1));

    // Class with hash 0x2 is still declared.
    let result = exec_state_reader.get_compiled_class(class_hash_0x2).unwrap();
    assert_eq!(
        result,
        RunnableCompiledClass::V1(
            (expected_result.clone(), SierraVersion::default()).try_into().unwrap()
        )
    );

    // And class with hash 0x3 is declared too.
    let result = exec_state_reader.get_compiled_class(class_hash_0x3).unwrap();
    assert_eq!(
        result,
        RunnableCompiledClass::V1((expected_result, SierraVersion::default()).try_into().unwrap())
    );
}
