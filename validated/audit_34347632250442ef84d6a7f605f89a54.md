### Title
Unconstrained `fee_proposal_fri` During Startup Window Allows Arbitrary L2 Gas Price Manipulation — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

During the first `fee_proposal_window_size` (10) blocks of a new chain, the `fee_proposal_fri` bounds check inside `is_proposal_init_valid` is explicitly skipped when `fee_actual` is `None`. A malicious proposer controlling a majority of the first 10 blocks can set `fee_proposal_fri` to an arbitrary value (e.g., `u128::MAX`), which is accepted by all validators without rejection, stored in `fee_proposals_window`, and then used to compute a corrupted `fee_actual` for all subsequent blocks — permanently skewing the L2 gas price.

---

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` enforces that a proposer's `fee_proposal_fri` lies within a geometric band around `fee_actual` (the median of the last `window_size` blocks):

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    // bounds check ...
}
``` [1](#0-0) 

The guard is a pattern-match that silently passes when `proposal_init_validation.fee_actual` is `None`. `fee_actual` is `None` whenever `compute_fee_actual` cannot produce a median — specifically when `height < window_size` (10):

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ({window_size})");
    return None;
};
``` [2](#0-1) 

`fee_actual` is passed into `ProposalInitValidation` at both the `validate_proposal` and `set_height_and_round` call sites:

```rust
fee_actual: compute_fee_actual(
    &self.fee_proposals_window,
    init.height,
    VersionedConstants::latest_constants().fee_proposal_window_size,
),
``` [3](#0-2) 

For a brand-new chain, `initialize_fee_proposals_window` is called with `start_height = 0`. The range `(0..0)` is empty, so the window is never populated and `fee_actual` remains `None` for all of blocks 0–9:

```rust
let window_start_height = window_end_height.saturating_sub(window_size);
let mut pending_heights: VecDeque<BlockNumber> =
    (window_start_height..window_end_height).map(BlockNumber).collect();
``` [4](#0-3) 

`initialize_fee_proposals_window` is called once in `ConsensusManager::run` before `run_consensus`:

```rust
consensus_context
    .initialize_fee_proposals_window(current_height.height)
    .await
    .expect("Failed to bootstrap fee_proposals window");
``` [5](#0-4) 

---

### Impact Explanation

Each accepted `fee_proposal_fri` value is stored in `fee_proposals_window` and later used to compute `fee_actual` (the median of the window), which in turn drives `calculate_next_l2_gas_price_for_fin` and is written as `next_l2_gas_price` into every block header:

```rust
fn calculate_next_l2_gas_price(&self, height: BlockNumber, l2_gas_used: GasAmount) -> GasPrice {
    let fee_actual = compute_fee_actual(
        &self.fee_proposals_window, height,
        VersionedConstants::latest_constants().fee_proposal_window_size,
    );
    calculate_next_l2_gas_price_for_fin(
        self.l2_gas_price, height, l2_gas_used,
        self.config.dynamic_config.override_l2_gas_price_fri,
        &self.config.dynamic_config.min_l2_gas_price_per_height,
        fee_actual,
    )
}
``` [6](#0-5) 

If a malicious proposer controls ≥ 5 of the first 10 blocks and sets `fee_proposal_fri = u128::MAX` in each, the median at block 10 becomes `u128::MAX`. From block 11 onward, the `fee_proposal` bounds check enforces that every proposer must stay within `fee_actual * (1 ± 2/1000)` ≈ `u128::MAX`, permanently locking the L2 gas price at an astronomically high value and making all user transactions prohibitively expensive.

This matches: **Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

---

### Likelihood Explanation

- Applies only to a brand-new chain (genesis); existing chains with `start_height ≥ window_size` are unaffected because `initialize_fee_proposals_window` pre-populates the window.
- Requires the attacker to control ≥ 5 of the first 10 block proposals (majority of the startup window). In a small or newly-launched validator set this is feasible.
- No special admin key or out-of-band access is needed — any validator in the set can propose blocks.

---

### Recommendation

When `fee_actual` is `None` (startup window), bound `fee_proposal_fri` against a locally-known reference instead of skipping the check entirely. The natural candidate is `self.l2_gas_price` (the EIP-1559 base fee already in use), which is always available:

```rust
let reference = proposal_init_validation.fee_actual
    .unwrap_or(proposal_init_validation.l2_gas_price_fri); // fallback to current base fee
let (lower_bound, upper_bound) = fee_proposal_bounds(
    reference,
    VersionedConstants::latest_constants().fee_proposal_margin_ppt,
);
if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
    return Err(...);
}
```

This mirrors the proposer-side fallback already implemented in `compute_proposer_fee_proposal`:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [7](#0-6) 

The proposer already freezes at `l2_gas_price` when `fee_actual` is unavailable; the validator should apply the same reference to enforce the same bound.

---

### Proof of Concept

1. Deploy a new Starknet chain (`start_height = 0`). `initialize_fee_proposals_window` produces an empty window; `fee_actual = None` for blocks 0–9.
2. A malicious validator controls ≥ 5 of the first 10 proposer slots (feasible in a small validator set).
3. For each controlled slot, the malicious proposer sets `fee_proposal_fri = u128::MAX` in `ProposalInit`.
4. `is_proposal_init_valid` reaches the fee-proposal check:
   ```rust
   if let (Some(fee_actual), Some(fee_proposal)) =
       (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
   ```
   `fee_actual` is `None`, so the `if let` does not match and the check is silently skipped. The proposal is accepted.
5. After 10 blocks, `compute_fee_actual` computes `median([u128::MAX × 5, normal × 5]) = u128::MAX`.
6. From block 11 onward, every honest proposer's `fee_proposal` is rejected unless it is within `2/1000` of `u128::MAX`. The L2 gas price is permanently corrupted. [1](#0-0) [8](#0-7) [9](#0-8)

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

    pub async fn initialize_fee_proposals_window(
        &mut self,
        start_height: BlockNumber,
    ) -> StateSyncClientResult<()> {
        const STATE_SYNC_RETRY_INTERVAL: Duration = Duration::from_millis(500);
        let window_size = VersionedConstants::latest_constants().fee_proposal_window_size;
        let window_end_height = start_height.0;
        let window_start_height = window_end_height.saturating_sub(window_size);
        let mut pending_heights: VecDeque<BlockNumber> =
            (window_start_height..window_end_height).map(BlockNumber).collect();
        while let Some(block_number) = pending_heights.pop_front() {
            match self.deps.state_sync_client.get_block(block_number).await {
                Ok(block) => self.record_fee_proposal(
                    block_number,
                    block.block_header_without_hash.fee_proposal_fri,
                ),
                Err(StateSyncClientError::StateSyncError(StateSyncError::BlockNotFound(_))) => {
                    warn!(
                        "State sync not ready for height {block_number}; re-queueing after \
                         {STATE_SYNC_RETRY_INTERVAL:?}"
                    );
                    pending_heights.push_back(block_number);
                    tokio::time::sleep(STATE_SYNC_RETRY_INTERVAL).await;
                }
                Err(e) => return Err(e),
            }
        }
        Ok(())
    }
