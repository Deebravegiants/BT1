### Title
Gateway `check_declare_permissions` Not Enforced in Consensus Validator Path Allows Unauthorized Declare Transactions - (File: `crates/apollo_gateway/src/gateway.rs`)

### Summary

The gateway enforces two declare-transaction restrictions — `block_declare` (a global kill-switch) and `authorized_declarer_accounts` (an allowlist) — via `check_declare_permissions()` called inside `add_tx_inner()`. The consensus validator path (`handle_proposal_part` → `convert_consensus_tx_to_internal_consensus_tx`) never calls this check. A Byzantine proposer can craft a declare transaction from an unauthorized sender, inject it directly into a proposal, and every honest validator will execute and commit it without ever consulting the operator's restrictions.

### Finding Description

The gateway's `add_tx_inner()` calls `check_declare_permissions()` before any other processing:

```rust
if let RpcTransaction::Declare(ref declare_tx) = tx {
    if let Err(e) = self.check_declare_permissions(declare_tx) {
        ...
        return Err(e);
    }
}
``` [1](#0-0) 

`check_declare_permissions` enforces two operator-configured restrictions:

```rust
fn check_declare_permissions(&self, declare_tx: &RpcDeclareTransaction) -> Result<(), StarknetError> {
    if self.config.static_config.block_declare {
        return Err(...); // global kill-switch
    }
    if !self.config.is_authorized_declarer(&declare_v3_tx.sender_address) {
        return Err(...); // allowlist
    }
    Ok(())
}
``` [2](#0-1) 

The consensus validator path processes incoming proposal transactions in `handle_proposal_part`, which calls `transaction_converter.convert_consensus_tx_to_internal_consensus_tx(tx)`: [3](#0-2) 

`TransactionConverter::convert_consensus_tx_to_internal_consensus_tx` dispatches to `convert_rpc_tx_to_internal` for `RpcTransaction` payloads. That function performs class compilation and hash calculation but **never calls `check_declare_permissions`**: [4](#0-3)

### Impact Explanation

When `authorized_declarer_accounts` is set (e.g., to a single trusted deployer address), the operator's intent is that **no other address may declare a contract class on-chain**. A Byzantine proposer can:

1. Construct a `RpcDeclareTransaction::V3` from any unauthorized `sender_address`.
2. Wrap it in a `ConsensusTransaction::RpcTransaction` and stream it as a `ProposalPart::Transactions` batch.
3. Every honest validator processes it through `convert_consensus_tx_to_internal_consensus_tx` → `convert_rpc_tx_to_internal`, which compiles the Sierra class and computes the hash — no permission check occurs.
4. The batcher executes the declare, the class hash is committed to state, and the block is finalized.

The result is an unauthorized class declared on-chain, violating the operator's allowlist invariant. The same applies when `block_declare = true`: a Byzantine proposer can still land declare transactions despite the global kill-switch.

This maps to the allowed impact: **High — admission accepts invalid transactions before sequencing**, because the consensus validator is the last admission gate before the batcher executes and commits the block.

### Likelihood Explanation

Any consensus participant whose turn it is to propose (round-robin or weighted selection) can trigger this. No external user access is required; one Byzantine validator in the committee suffices. The `authorized_declarer_accounts` feature is explicitly production-configured (it appears in `config_schema.json` and has gateway tests), so operators who rely on it are directly exposed.

### Recommendation

Apply the same `check_declare_permissions` logic inside the consensus validator's transaction-conversion path. Concretely, `handle_proposal_part` (or `convert_consensus_tx_to_internal_consensus_tx`) should reject any `RpcDeclareTransaction` whose `sender_address` is not in `authorized_declarer_accounts`, and reject all declares when `block_declare = true`. Alternatively, move the enforcement into the blockifier's `perform_pre_validation_stage` so it is invariant across all ingestion paths.

### Proof of Concept

```
Setup:
  gateway_config.static_config.authorized_declarer_accounts = ["0xAUTH"]
  (only 0xAUTH may declare)

Attack:
  1. Byzantine proposer P constructs:
       tx = RpcDeclareTransaction::V3 { sender_address: 0xUNAUTH, contract_class: <any Sierra>, ... }
       consensus_tx = ConsensusTransaction::RpcTransaction(tx)

  2. P streams ProposalPart::Transactions([consensus_tx]) to all validators.

  3. Each honest validator calls:
       handle_proposal_part(...)
         → convert_consensus_tx_to_internal_consensus_tx(consensus_tx)
           → convert_rpc_tx_to_internal(RpcTransaction::Declare(...))
             → class_manager_client.add_class(contract_class)   // compiles Sierra
             → InternalRpcDeclareTransactionV3 { sender_address: 0xUNAUTH, class_hash, ... }
             // ← check_declare_permissions() is NEVER called

  4. Validator sends the internal tx to the batcher via send_txs_for_proposal.

  5. Batcher executes the declare; class_hash is written to state.

  6. Block is committed. 0xUNAUTH's class is now live on-chain despite the allowlist.

        &self,
        declare_tx: &RpcDeclareTransaction,
    ) -> Result<(), StarknetError> {
        // TODO(noamsp): Return same error as in Python gateway.
        if self.config.static_config.block_declare {
            return Err(StarknetError {

                message: "Transaction type is temporarily blocked.".to_string(),

        let RpcDeclareTransaction::V3(declare_v3_tx) = declare_tx;

                code: StarknetErrorCode::KnownErrorCode(
                    KnownStarknetErrorCode::UnauthorizedDeclare,
                ),
                message: format!(
                    "Account address {} is not allowed to declare contracts.",
                    &declare_v3_tx.sender_address
                ),
            });
        }
        Ok(())

                let (internal_tx, proof_data) = self.convert_rpc_tx_to_internal(tx).await?;

                    self.spawn_verify_and_store_proof(proof_facts, proof)
                });

                let internal_tx = self.convert_consensus_l1_handler_to_internal_l1_handler(tx)?;
                Ok((InternalConsensusTransaction::L1Handler(internal_tx), None))
            }
        }
    }

    async fn convert_rpc_tx_to_internal(
        &self,
        tx: RpcTransaction,
    ) -> TransactionConverterResult<(InternalRpcTransaction, Option<(ProofFacts, Proof)>)> {
        let (tx_without_hash, proof_data) = match tx {
            RpcTransaction::Invoke(RpcInvokeTransaction::V3(tx)) => {
                let proof_data = if tx.proof_facts.is_empty() {
                    None
                } else {
                    Some((tx.proof_facts.clone(), tx.proof.clone()))
                };
                (InternalRpcTransactionWithoutTxHash::Invoke(tx.into()), proof_data)
            }
            RpcTransaction::Declare(RpcDeclareTransaction::V3(tx)) => {
                let ClassHashes { class_hash, executable_class_hash_v2 } =
                // TODO(Dori): Make this async and spawn a task to compile and add it to the class manager.
                    self.class_manager_client.add_class(tx.contract_class).await?;
                // TODO(Aviv): Ensure that we do not want to
                // allow declare with compiled class hash v1.
                if tx.compiled_class_hash != executable_class_hash_v2 {
                    return Err(TransactionConverterError::ValidateCompiledClassHashError(
                        ValidateCompiledClassHashError::CompiledClassHashMismatch {
                            computed_class_hash: executable_class_hash_v2,
                            supplied_class_hash: tx.compiled_class_hash,
                        },
                    ));
                }
                (
                    InternalRpcTransactionWithoutTxHash::Declare(InternalRpcDeclareTransactionV3 {
                        sender_address: tx.sender_address,
                        compiled_class_hash: tx.compiled_class_hash,
                        signature: tx.signature,
                        nonce: tx.nonce,
                        class_hash,
                        resource_bounds: tx.resource_bounds,
                        tip: tx.tip,
                        paymaster_data: tx.paymaster_data,
                        account_deployment_data: tx.account_deployment_data,
                        nonce_data_availability_mode: tx.nonce_data_availability_mode,
                        fee_data_availability_mode: tx.fee_data_availability_mode,
                    }),
                    None,
                )
            }
            RpcTransaction::DeployAccount(RpcDeployAccountTransaction::V3(tx)) => {
                let contract_address = tx.calculate_contract_address()?;
                (
                    InternalRpcTransactionWithoutTxHash::DeployAccount(
                        InternalRpcDeployAccountTransaction {
                            tx: RpcDeployAccountTransaction::V3(tx),
                            contract_address,
                        },
                    ),
                    None,
                )
            }
        };
        let tx_hash = tx_without_hash.calculate_transaction_hash(&self.chain_id)?;
        Ok((InternalRpcTransaction { tx: tx_without_hash, tx_hash }, proof_data))
    }
