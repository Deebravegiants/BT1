### Title
Unguarded `fee_proposal_fri` During Startup Window Bypasses Bounds Enforcement, Corrupting Future L2 Gas Price — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

In `is_proposal_init_valid`, the bounds check for `fee_proposal_fri` is gated on `fee_actual` being `Some`. When `fee_actual` is `None` — which is the documented, reachable state for the first `window_size` blocks after Starknet V0_14_3 activation (or during the pre-V0_14_3 → V0_14_3 transition window) — the entire bounds check is silently skipped. A malicious proposer selected during this window can publish any `fee_proposal_fri` value (including `u128::MAX`), validators accept it without objection, and the value is permanently stored in `fee_proposals_window`, poisoning future `fee_actual` (median) calculations and therefore the L2 gas price for all subsequent blocks.

---

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` enforces that `fee_proposal_fri` lies within `[fee_actual / (1+m), fee_actual * (1+m)]`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    ...bounds check...
}
``` [1](#0-0) 

The `if let` pattern silently drops the check when `fee_actual` is `None`. `fee_actual` is `None` whenever `compute_fee_actual` returns `None`:

```rust
pub fn compute_fee_actual(...) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!("Cannot compute fee_actual for height {height}: height is below window_size ...");
        return None;
    };
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => { ... return None; }
        }
    }
    ...
}
``` [2](#0-1) 

This `None` state is reached for every block during the first `window_size` heights after V0_14_3 activation, and for every block during the transition period where any prior block in the window recorded `fee_proposal_fri = None` (pre-V0_14_3 era).

The honest proposer path freezes at `l2_gas_price` when `fee_actual` is `None`:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [3](#0-2) 

But the validator never checks that the proposer honored this constraint. It simply skips the check entirely.

At decision time, the accepted `fee_proposal_fri` is unconditionally stored in the window:

```rust
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [4](#0-3) 

And the window feeds `fee_actual`, which feeds the next L2 gas price:

```rust
fn calculate_next_l2_gas_price(&self, height: BlockNumber, l2_gas_used: GasAmount) -> GasPrice {
    let fee_actual = compute_fee_actual(
        &self.fee_proposals_window, height,
        VersionedConstants::latest_constants().fee_proposal_window_size,
    );
    calculate_next_l2_gas_price_for_fin(self.l2_gas_price, height, l2_gas_used, ..., fee_actual)
}
``` [5](#0-4) 

The `ProposalCommitment` binds `fee_proposal_fri` via Poseidon hash, but the validator computes the commitment using the proposer-supplied value, so any value the proposer sends will produce a matching commitment:

```rust
let batcher_block_commitment = proposal_commitment_from(
    finished_info.proposal_commitment.partial_block_hash,
    fee_proposal,   // ← proposer-supplied, unchecked during startup
);
``` [6](#0-5) 

---

### Impact Explanation

A malicious proposer selected during the startup window can set `fee_proposal_fri` to an extreme value (e.g., `u128::MAX`). Validators accept it. The value enters `fee_proposals_window`. If the attacker controls a majority of proposals during the `window_size`-block startup period, `fee_actual` becomes extreme, driving the L2 gas price to `u128::MAX` and making all subsequent transactions economically unexecutable. Even a minority of poisoned entries shifts the median, causing incorrect fee accounting for all users on all future blocks. This matches the "Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact" criterion.

---

### Likelihood Explanation

The startup window is bounded (first `window_size` blocks after V0_14_3 activation, or during the pre-V0_14_3 transition). Any proposer legitimately selected by consensus during that window can trigger this without any special privilege. The transition window is particularly relevant at mainnet upgrade time, when the window will be fully open for `window_size` consecutive blocks.

---

### Recommendation

When `fee_actual` is `None`, the validator should enforce that `fee_proposal_fri` equals (or is within a small margin of) the locally-derived `l2_gas_price` fallback — the same value the honest proposer is required to use. Concretely, in `is_proposal_init_valid`, add an `else` branch to the existing `if let`:

```rust
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    // existing bounds check against fee_actual
} else if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    // fee_actual unavailable: proposer must freeze at l2_gas_price
    // enforce fee_proposal == l2_gas_price (within a small abs margin)
    if fee_proposal != proposal_init_validation.l2_gas_price_fri {
        return Err(InvalidProposalInit(..., "fee_proposal must equal l2_gas_price during startup"));
    }
}
```

`ProposalInitValidation` already carries `l2_gas_price_fri`, so no new fields are needed. [7](#0-6) 

---

### Proof of Concept

1. Network activates Starknet V0_14_3. `fee_proposals_window` is empty; `compute_fee_actual` returns `None` for the first `window_size` heights.
2. Attacker is selected as proposer at height `H < window_size`.
3. Attacker constructs `ProposalInit` with `fee_proposal_fri = Some(GasPrice(u128::MAX))`.
4. Validator calls `is_proposal_init_valid`. The `if let (Some(fee_actual), Some(fee_proposal))` pattern evaluates to `(None, Some(...))`, so the entire bounds block is skipped. Validation returns `Ok(())`.
5. Validator computes `batcher_block_commitment = proposal_commitment_from(partial_block_hash, Some(GasPrice(u128::MAX)))`. Proposer sent the same commitment. `built_block == received_fin.proposal_commitment` → proposal accepted.
6. At `decision_reached`, `record_fee_proposal(H, Some(GasPrice(u128::MAX)))` stores the poisoned value.
7. Once `window_size` such blocks are committed, `compute_fee_actual` returns `Some(GasPrice(u128::MAX))`.
8. `calculate_next_l2_gas_price_for_fin` sets `l2_gas_price = u128::MAX`. All future transactions require fees of `u128::MAX * gas_used`, making the chain economically unusable.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
