Audit Report

## Title
Nested Unbounded Loops in `updateRSETHPrice()` Bypass Protocol Gas Cap via EigenLayer Forced Undelegations — (File: `contracts/LRTOracle.sol`, `contracts/LRTDepositPool.sol`, `contracts/NodeDelegator.sol`)

## Summary
`LRTOracle.updateRSETHPrice()` is a public function that triggers a deeply nested multi-contract loop whose gas cost scales with the number of supported assets (N), node delegators (M), and EigenLayer queued withdrawals (Q). The protocol attempts to cap Q at 80 via `maxUncompletedWithdrawalCount`, but `NodeDelegator.getAssetUnstaking()` queries EigenLayer's `getQueuedWithdrawals()` directly — bypassing the protocol's internal counter — meaning EigenLayer-forced undelegations can push the actual queued withdrawal count above the cap, causing `updateRSETHPrice()` to exceed the block gas limit and permanently stale the rsETH price.

## Finding Description

The full call chain is confirmed in code:

`updateRSETHPrice()` (public, `LRTOracle.sol:87`) → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` (`LRTOracle.sol:331-349`) loops over all `supportedAssets` (N), calling `ILRTDepositPool.getTotalAssetDeposits(asset)` per asset. [1](#0-0) 

`getTotalAssetDeposits()` calls `getAssetDistributionData()` (`LRTDepositPool.sol:426-462`), which loops over all `nodeDelegatorQueue` entries (M), calling `INodeDelegator.getAssetUnstaking(asset)` per NDC per asset. [2](#0-1) 

`getAssetUnstaking()` (`NodeDelegator.sol:405-427`) calls `_getDelegationManager().getQueuedWithdrawals(address(this))` — a live EigenLayer query — and iterates over all returned withdrawals and their strategies, calling `strategy.sharesToUnderlyingView()` per non-ETH strategy. [3](#0-2) 

**Root cause — the mitigation is incomplete:** `LRTUnstakingVault.setMaxUncompletedWithdrawalCount()` caps the protocol's internal `uncompletedWithdrawalCount` at 80. The comment explicitly acknowledges: *"120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price."* [4](#0-3) 

However, `getAssetUnstaking()` does not read `uncompletedWithdrawalCount` — it calls `getQueuedWithdrawals(address(this))` directly on EigenLayer's `DelegationManager`. [5](#0-4) 

EigenLayer's `DelegationManager.undelegate()` can be called by the operator to whom an NDC is delegated, force-undelegating the NDC and creating new queued withdrawals on EigenLayer's side without incrementing the protocol's `uncompletedWithdrawalCount`. With M=10 NDCs each holding N=10 asset strategies, a forced undelegation of all NDCs creates up to 100 additional EigenLayer-side withdrawals beyond the protocol's tracked cap. The `undelegate()` function in `NodeDelegator.sol` checks the cap only for protocol-initiated undelegations, not for operator-forced ones. [6](#0-5) 

The same nested loop is also triggered on every user deposit via `_checkIfDepositAmountExceedesCurrentLimit` → `getTotalAssetDeposits`. [7](#0-6) 

## Impact Explanation

If `updateRSETHPrice()` reverts out-of-gas, `rsETHPrice` in `LRTOracle` becomes permanently stale. All downstream consumers — `getRsETHAmountToMint` (deposit minting), `getExpectedAssetAmount` (withdrawal sizing), and withdrawal queue unlocking — operate on an incorrect price. This constitutes **Medium — Unbounded gas consumption**, with a secondary risk of **Medium — Temporary freezing of funds** if the stale price causes deposit or withdrawal operations to revert on limit checks. The protocol's own comment confirms the gas ceiling is real and was calibrated by the developers.

## Likelihood Explanation

Normal protocol operation with 10 NDCs, 10 supported assets, and 80 tracked withdrawals already generates ~1,700 external calls per `updateRSETHPrice()` invocation. An EigenLayer operator force-undelegating all NDCs (a legitimate EigenLayer action, not an attack) pushes the actual queued withdrawal count above 120 — the protocol's own acknowledged gas limit — causing the function to revert. No unprivileged attacker action is required; the gas exhaustion emerges from normal protocol scaling combined with a realistic EigenLayer operator action. Likelihood is **Medium**.

## Recommendation

1. **Decouple `getAssetUnstaking` from live EigenLayer enumeration**: Maintain an incremental `assetUnstakingAmount` mapping updated on `initiateUnstaking`/`completeUnstaking` events rather than re-enumerating `getQueuedWithdrawals` on every price update.
2. **Bound `getQueuedWithdrawals` iteration**: Add a hard iteration cap in `getAssetUnstaking` equal to `maxUncompletedWithdrawalCount`, reverting or returning a partial result if exceeded, to prevent silent OOG failures.
3. **Handle forced undelegations**: In `setUncompletedWithdrawalCount()` (the manager-callable resync function), also enforce that the live EigenLayer count does not exceed `maxUncompletedWithdrawalCount`, and pause price updates if it does.
4. **Add a gas guard** in `updateRSETHPrice()` that checks the product of NDC count × total queued withdrawals against a safe threshold before entering the loop.

## Proof of Concept

```
// Normal operation (no attacker needed):
// State: 10 supported assets, 10 NDCs, 80 total queued withdrawals (at protocol cap)
// Then: EigenLayer operator calls DelegationManager.undelegate(NDC_address) for each NDC
//       → creates up to 100 additional EigenLayer-side withdrawals (10 NDCs × 10 strategies)
//       → protocol's uncompletedWithdrawalCount is NOT incremented (forced, not protocol-initiated)

// Anyone calls:
LRTOracle.updateRSETHPrice()
  → _getTotalEthInProtocol()
    → for i in [0..9] (10 assets):
        LRTDepositPool.getTotalAssetDeposits(asset[i])
          → getAssetDistributionData(asset[i])
            → for j in [0..9] (10 NDCs):
                NodeDelegator[j].getAssetUnstaking(asset[i])
                  → DelegationManager.getQueuedWithdrawals(NDC[j])
                    // returns 80+100=180 withdrawals (above the 120 gas-safe limit)
                    → for k in [0..179] (180 withdrawals):
                        for s in [0..1] (2 strategies):
                          strategy.sharesToUnderlyingView(...)
  // Total external calls: 100 (getQueuedWithdrawals) + 10×180×2 (sharesToUnderlyingView) = 3,700
  // Estimated gas: >30M → REVERT (out of gas)
  // rsETHPrice permanently stale
```

```text
    function undelegate() external whenNotPaused onlyLRTManager {
        if (elOperatorDelegatedTo() == address(0)) {
            revert CantUndelegate();
        }

        bytes32[] memory withdrawalRoots = _getDelegationManager().undelegate(address(this));

        if (
            _getUnstakingVault().uncompletedWithdrawalCount() + withdrawalRoots.length
                > _getUnstakingVault().maxUncompletedWithdrawalCount()
        ) {
            revert MaxUncompletedWithdrawalsReached();
        }

        for (uint256 i; i < withdrawalRoots.length; i++) {
            _getUnstakingVault().increaseUncompletedWithdrawalCount();

            // NOTE: For legacy event emission we emit single withdrawal roots
            bytes32[] memory singleWithdrawal = new bytes32[](1);
            singleWithdrawal[0] = withdrawalRoots[i];
            emit WithdrawalQueued(_getNonce() - withdrawalRoots.length + i, address(this), singleWithdrawal);
        }

        emit Undelegated();
    }
```

**File:** contracts/NodeDelegator.sol (L405-427)

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
