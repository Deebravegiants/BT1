### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Inject Arbitrary Sequencer Address into Block Hash and Execution Context - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

### Summary

The `ProposalInit.builder` field — which becomes the `sequencer_address` in every block's execution context and is hashed into the canonical block hash — is never validated by the validator node. A malicious-but-legitimate proposer can set `builder` to any arbitrary address. Validators accept the proposal, execute all transactions with the attacker-controlled sequencer address, and commit a block whose hash and stored header embed the wrong sequencer. Contracts that call `get_sequencer_address()` during execution receive the forged value.

### Finding Description

`ProposalInit` carries two authority-related addresses:

- `proposer` — the consensus identity of the block proposer
- `builder` — "Address of the one who builds/sequences the block" [1](#0-0) 

The consensus manager validates `proposer` against the committee before forwarding the proposal to the orchestrator: [2](#0-1) [3](#0-2) 

However, `is_proposal_init_valid()` — the function that performs all semantic checks on `ProposalInit` fields — checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`, but **never checks `builder`**: [4](#0-3) 

`ProposalInitValidation` — the struct that carries the locally-trusted reference values used for comparison — does not even contain a `builder` field: [5](#0-4) 

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address` in the `BlockInfo` sent to the batcher: [6](#0-5) 

This `sequencer_address` then propagates into three critical downstream paths:

**1. Block hash computation.** `PartialBlockHashComponents::new()` reads `block_info.sequencer_address` into the `sequencer` field, which `calculate_block_hash()` chains into the Poseidon hash: [7](#0-6) [8](#0-7) 

**2. Block header storage.** `update_state_sync_with_new_block()` stores `SequencerContractAddress(init.builder)` in the committed block header: [9](#0-8) 

**3. Execution syscall.** The `get_sequencer_address` syscall returns `block_info.sequencer_address` to any contract that calls it during transaction execution: [10](#0-9) 

Because the proposer also controls the `ProposalFin.proposal_commitment` it sends, it can compute a commitment that is consistent with the forged `builder` address. The validator's batcher executes with the same forged address and produces the same `PartialBlockHash`, so the final `built_block == received_fin.proposal_commitment` check passes: [11](#0-10) 

The block is committed with the attacker-chosen sequencer address embedded in the block hash, block header, and execution results.

### Impact Explanation

- **Wrong block hash**: The canonical block hash includes the forged `sequencer_address`. Every subsequent block that chains off this hash inherits the corruption.
- **Wrong stored block header**: The `sequencer` field in the committed block header is the attacker-controlled address, causing incorrect RPC responses for `starknet_getBlockWithTxs` and related calls.
- **Wrong syscall result**: Any contract that calls `get_sequencer_address()` during execution of transactions in this block receives the forged address. Contracts that gate logic on the sequencer address (e.g., fee-collection contracts, privileged admin contracts) can be manipulated.
- **Wrong `PartialBlockHash`**: The `ProposalCommitment` voted on by the network encodes the forged sequencer, so the committed partial block hash is permanently wrong.

This matches the allowed impact: *"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input"* and *"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."*

### Likelihood Explanation

Any validator that wins a proposer slot for a round can execute this attack. In a decentralized Tendermint network, every validator rotates through the proposer role. The attack requires no special capability beyond being the legitimate proposer for one round. The forged `builder` value passes all existing validation checks silently.

### Recommendation

Add a `builder` field to `ProposalInitValidation` populated from the node's own configured `builder_address`, and add a check in `is_proposal_init_valid` that rejects any `ProposalInit` where `init.builder != proposal_init_validation.builder`. The reference value should come from the same local configuration that the proposer uses when constructing `ProposalInit` (i.e., `args.builder_address` in `initiate_build`), not from the network message.

```rust
// In ProposalInitValidation:

// In is_proposal_init_valid:

    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder

    ));
```

### Proof of Concept

1. Attacker is the legitimate proposer for height H, round R.
2. Attacker constructs `ProposalInit` with `builder = ATTACKER_ADDRESS` (any arbitrary address).
3. Attacker computes `PartialBlockHashComponents` using `ATTACKER_ADDRESS` as `sequencer`, derives `PartialBlockHash`, and sends it as `ProposalFin.proposal_commitment`.
4. Validator calls `is_proposal_init_valid()` — passes, because `builder` is never checked.
5. Validator calls `convert_to_sn_api_block_info(init)` → `sequencer_address = ATTACKER_ADDRESS`.
6. Batcher executes all transactions with `ATTACKER_ADDRESS` as sequencer; `get_sequencer_address()` syscall returns `ATTACKER_ADDRESS` to all contracts.
7. Batcher computes `PartialBlockHash` with `ATTACKER_ADDRESS` → matches proposer's `ProposalFin.proposal_commitment`.
8. `built_block == received_fin.proposal_commitment` check passes.
9. Block is committed: block hash, block header, and all execution results embed `ATTACKER_ADDRESS` as the sequencer.

```rust
#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
```
