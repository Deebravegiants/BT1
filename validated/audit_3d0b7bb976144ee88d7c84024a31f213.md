### Title
`LRTConverter.ethValueInWithdrawal` Uses Stale Entry-Time Price, Under-Reporting Total ETH in Protocol - (File: contracts/LRTConverter.sol)

### Summary
`LRTConverter` tracks the ETH value of LST assets (e.g., stETH) sent to it for unstaking via the manually maintained `ethValueInWithdrawal` variable. This value is stamped at the oracle price at the moment of transfer and is never refreshed. Because yield-bearing LSTs continuously appreciate in ETH terms, `ethValueInWithdrawal` chronically under-reports the true ETH value of assets sitting in the converter, causing `_getTotalEthInProtocol()` in `LRTOracle` to return a deflated figure and `rsETHPrice` to be set lower than it should be.

### Finding Description
When an operator calls `transferAssetFromDepositPool`, the converter records the ETH value of the transferred LST at the current oracle price:

```solidity
// LRTConverter.sol
ethValueInWithdrawal += (_amount * lrtOracle.getAssetPrice(_asset)) / 1e18;
```

This value is consumed by `LRTDepositPool.getETHDistributionData()`:

```solidity
// LRTDepositPool.sol
address lrtConverter = lrtConfig.getContract(LRTConstants.LRT_CONVERTER);
ethLyingInConverter = ILRTConverter(lrtConverter).ethValueInWithdrawal();
```

which feeds `LRTOracle._getTotalEthInProtocol()`:

```solidity
uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
```

and ultimately drives `rsETHPrice`:

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
```

stETH is a rebasing token whose ETH exchange rate increases continuously. Once stETH is transferred to the converter, `ethValueInWithdrawal` is frozen at the entry-time price and is only reduced (never increased) when ETH is eventually claimed and forwarded to the deposit pool via `_sendEthToDepositPool`. During the entire unstaking window (which can span days to weeks on Lido), the growing ETH value of the stETH held in the converter is invisible to the oracle.

### Impact Explanation
`rsETHPrice` is computed from a deflated `totalETHInProtocol`. A lower `rsETHPrice` causes `getRsETHAmountToMint` to mint more rsETH per unit of deposited asset than is warranted:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

Depositors who enter while stETH is sitting in the converter receive a slightly inflated rsETH allocation, diluting the claims of existing rsETH holders. The protocol under-delivers on the promised exchange rate to existing holders for the duration of the unstaking window.

**Impact level:** Low — contract fails to deliver promised returns to existing rsETH holders; no direct fund theft.

### Likelihood Explanation
stETH accrues rewards continuously (~3–4 % APR). Any time the operator routes stETH through the converter for Lido withdrawal (a routine operational step), the discrepancy is active for the full withdrawal queue duration (typically 1–7 days). This is a normal, recurring operational path, making the condition highly likely to occur in practice.

### Recommendation
Replace the static entry-time ETH value with a live calculation of the current ETH value of LST tokens held in the converter. For example, instead of storing `ethValueInWithdrawal` as a fixed ETH amount, store the raw LST token amounts and compute their current ETH value on-the-fly using the oracle:

```solidity
// Instead of:
ethValueInWithdrawal += (_amount * lrtOracle.getAssetPrice(_asset)) / 1e18;

// Track raw token balances and compute live:
mapping(address => uint256) public tokenAmountsInWithdrawal;
tokenAmountsInWithdrawal[_asset] += _amount;

// In ethValueInWithdrawal() view:
for each asset: total += tokenAmountsInWithdrawal[asset] * oracle.getAssetPrice(asset) / 1e18;
```

This mirrors the fix recommended in H-03: use the live, comprehensive balance rather than a stale snapshot.

### Proof of Concept
1. Operator calls `transferAssetFromDepositPool(stETH, 1000e18)` when stETH price = 1.00 ETH.
   - `ethValueInWithdrawal` = 1000 ETH.
2. Seven days pass; stETH accrues ~0.077% yield; price = 1.00077 ETH.
   - Actual ETH value of 1000 stETH in converter = 1000.77 ETH.
   - `ethValueInWithdrawal` still = 1000 ETH (stale).
3. `_getTotalEthInProtocol()` under-counts by 0.77 ETH.
4. `updateRSETHPrice()` is called; `rsETHPrice` is set ~0.077% lower than correct.
5. A depositor calling `depositAsset` during this window receives ~0.077% more rsETH than warranted, diluting all existing rsETH holders by that margin. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
