Audit Report

## Title
Missing `msg.sender == account` Validation Enables Forced Claims with Permanent Fee Loss — (`contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

## Summary

`MerkleDistributor.claim` accepts a caller-supplied `account` parameter but never verifies that `msg.sender == account`. Any unprivileged caller who possesses a valid merkle proof for a victim can trigger a claim on the victim's behalf. When `feeInBPS > 0`, the fee is permanently deducted from the victim's allocation and sent to `protocolTreasury` without the victim's consent, causing irreversible loss of unclaimed yield. The sibling contract `KernelMerkleDistributor` explicitly guards against this with an `Unauthorized` revert, confirming the protocol is aware of the pattern but failed to apply it consistently.

## Finding Description

`MerkleDistributor.claim` (L97–147) accepts `account` as a caller-supplied parameter and verifies a merkle proof for `(index, account, cumulativeAmount)`, but contains no check that `msg.sender == account`: [1](#0-0) 

After proof verification, the function computes and permanently deducts a fee of up to `MAX_FEE_IN_BPS = 1000` (10%), sending it to `protocolTreasury`: [2](#0-1) 

By contrast, `KernelMerkleDistributor._processClaim` explicitly enforces caller identity before proceeding: [3](#0-2) 

The maximum fee is capped at 10%: [4](#0-3) 

**Exploit flow:**
1. Owner sets `feeInBPS = 1000` and publishes a merkle root.
2. Off-chain distribution publishes Alice's proof: `(index=1, account=Alice, cumulativeAmount=1000e18, proof=[...])`.
3. Attacker (Bob) calls `merkleDistributor.claim(1, Alice, 1000e18, aliceProof)`.
4. Proof verifies. Contract computes `fee = 100e18`, `amountToSend = 900e18`.
5. `900e18` tokens sent to Alice; `100e18` tokens sent to `protocolTreasury`.
6. `userClaims[Alice]` is updated; Alice can never reclaim the `100e18` fee.
7. Bob spent only gas. Alice permanently lost 10% of her allocation — yield she could have preserved by waiting for the fee to be reduced or set to zero.

The victim has no recourse: once `userClaims[account].lastClaimedIndex` and `cumulativeAmount` are updated, the claim is permanently marked and the fee is gone. [5](#0-4) 

## Impact Explanation

**High — Theft of unclaimed yield.**

Every user whose merkle proof is known (i.e., all users, since proofs are distributed publicly for claiming) can be force-claimed at any time while `feeInBPS > 0`. At the maximum 10% fee, up to 10% of every user's token allocation is permanently redirected to `protocolTreasury` without the user's consent. A victim who intended to wait until the fee was reduced to zero before claiming is permanently deprived of that option. This matches the allowed impact: **High — Theft of unclaimed yield**.

## Likelihood Explanation

**High.** Merkle proofs for distributor contracts are always published off-chain (otherwise users cannot claim). No special privilege, role, capital, or victim interaction is required. Any EOA can execute the attack for any victim in a single transaction, at any time after the merkle root is set and while `feeInBPS > 0`.

## Recommendation

Add a caller-identity check at the top of `claim`, mirroring the pattern already used in `KernelMerkleDistributor._processClaim`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Also add the `Unauthorized` error to `IMerkleDistributor` in `MerkleDistributor.sol`, as it is currently only defined in the `KernelMerkleDistributor` interface.

## Proof of Concept

**Foundry test (unit):**

```solidity
function test_forcedClaimWithFee() public {
    // Setup: owner sets 10% fee and merkle root
    vm.prank(owner);
    distributor.setFeeInBPS(1000);
    vm.prank(owner);
    distributor.setMerkleRoot(merkleRoot);

    // Alice's proof is public; Bob force-claims on Alice's behalf
    uint256 aliceBalanceBefore = token.balanceOf(alice);
    uint256 treasuryBalanceBefore = token.balanceOf(protocolTreasury);

    vm.prank(bob); // Bob is the caller, not Alice
    distributor.claim(1, alice, 1000e18, aliceProof);

    // Alice received only 900e18 (10% fee deducted without her consent)
    assertEq(token.balanceOf(alice) - aliceBalanceBefore, 900e18);
    // Treasury received 100e18 from Alice's allocation
    assertEq(token.balanceOf(protocolTreasury) - treasuryBalanceBefore, 100e18);
    // Alice's claim is permanently marked; she cannot reclaim the 100e18
    assertTrue(distributor.isClaimed(1, alice));
}
```

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }
