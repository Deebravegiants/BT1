### Title
Unbounded Loop Over `supportedAssets` in `LRTOracle._getTotalEthInProtocol()` Causes Permanent DoS on Price Updates - (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle.updateRSETHPrice()` is publicly callable and internally executes a nested unbounded loop across `supportedAssets` and `nodeDelegatorQueue`. As the `supportedAssets` array has no cap, a sufficient number of supported assets can push the gas cost of `updateRSETHPrice()` past the block gas limit, permanently preventing rsETH price updates and breaking the protocol's core accounting.

### Finding Description
`LRTOracle._getTotalEthInProtocol()` iterates over every entry in `lrtConfig.getSupportedAssetList()` (the `supportedAssets` array in `LRTConfig`). [1](#0-0) 

For each asset it calls `ILRTDepositPool.getTotalAssetDeposits(asset)`, which resolves to `getAssetDistributionData()`: [2](#0-1) 

That inner loop calls `INodeDelegator.getAssetUnstaking(asset)` on every NDC, which itself iterates over all queued EigenLayer withdrawals: [3](#0-2) 

The outer driver, `updateRSETHPrice()`, carries no access control — it is `public whenNotPaused`: [4](#0-3) 

`LRTConfig.addNewSupportedAsset()` pushes to `supportedAssetList` with no upper-bound check: [5](#0-4) 

By contrast, `nodeDelegatorQueue` is bounded by `maxNodeDelegatorLimit` (default 10), but `supportedAssets` has no analogous cap. The resulting triple-nested call chain — `supportedAssets × nodeDelegatorQueue × queuedWithdrawals` — grows without bound as assets are added.

### Impact Explanation
When the gas cost of `updateRSETHPrice()` exceeds the block gas limit, the rsETH price stored in `rsETHPrice` can no longer be refreshed. Every downstream consumer of `lrtOracle.rsETHPrice()` — including `getRsETHAmountToMint()` used in `depositETH`/`depositAsset`, and `getExpectedAssetAmount()` used in `initiateWithdrawal`/`instantWithdrawal` — operates on a permanently stale price. This constitutes **unbounded gas consumption** leading to a permanent inability to update the protocol's core exchange rate.

**Impact: Medium — Unbounded gas consumption / permanent stale price.**

### Likelihood Explanation
Adding a new supported asset requires the `TIME_LOCK_ROLE`, which is an admin-level privilege. A large number of assets must be added before the gas limit is breached. This is unlikely under normal operation but becomes a realistic risk if the protocol expands its supported asset set over time, or if a governance/admin key is compromised.

**Likelihood: Low.**

### Recommendation
1. Introduce an explicit cap on `supportedAssetList` length in `_addNewSupportedAsset()`, analogous to `maxNodeDelegatorLimit` in `LRTDepositPool`.
2. Consider caching or pre-computing per-asset TVL off-chain and providing it as a parameter to `updateRSETHPrice()`, rather than recomputing it on-chain in a single transaction.
3. Alternatively, allow `updateRSETHPrice()` to accept a subset of assets to process incrementally.

### Proof of Concept
1. Admin (via `TIME_LOCK_ROLE`) calls `LRTConfig.addNewSupportedAsset()` repeatedly, adding N assets to `supportedAssetList`.
2. Each asset has a corresponding strategy and NDCs with queued EigenLayer withdrawals.
3. Any caller invokes `LRTOracle.updateRSETHPrice()`.
4. Execution enters `_getTotalEthInProtocol()`, which loops N times; each iteration calls `getTotalAssetDeposits()` → `getAssetDistributionData()` → loops over `nodeDelegatorQueue` → calls `getAssetUnstaking()` → loops over all queued withdrawals.
5. At sufficient N, the transaction reverts with out-of-gas, and `rsETHPrice` is never updated again. [6](#0-5) [7](#0-6)

        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    /// @param asset Asset address
    /// @param depositLimit Deposit limit for the asset
    function _addNewSupportedAsset(address asset, uint256 depositLimit) private {
        UtilLib.checkNonZeroAddress(asset);
        if (depositLimit == 0) {
            revert InvalidDepositLimit();
        }
        if (isSupportedAsset[asset]) {
            revert AssetAlreadySupported();
        }
        isSupportedAsset[asset] = true;
        supportedAssetList.push(asset);
        depositLimitByAsset[asset] = depositLimit;
        emit AddedNewSupportedAsset(asset, depositLimit);
    }
