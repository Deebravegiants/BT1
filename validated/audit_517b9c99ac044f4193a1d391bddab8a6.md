### Title
Native Fee Tokens Permanently Locked in Starknet OmniBridge With No Withdrawal Path - (File: starknet/src/omni_bridge.cairo)

### Summary

The Starknet `OmniBridge` contract collects `native_fee` STRK tokens from users during `init_transfer` and holds them in the contract itself, but exposes no function to withdraw or distribute those accumulated fees. The funds are permanently locked until a contract upgrade adds a withdrawal path.

### Finding Description

In `init_transfer`, when `native_fee > 0`, the contract pulls STRK tokens from the caller directly into `get_contract_address()` (the bridge contract itself): [1](#0-0) 

The `IOmniBridge` interface and the full `OmniBridge` module expose no function to withdraw, sweep, or distribute these accumulated STRK fee tokens: [2](#0-1) 

The only privileged operation available is `upgrade`, which replaces the class hash but does not transfer tokens: [3](#0-2) 

Every call to `init_transfer` with a non-zero `native_fee` permanently deposits STRK into the contract with no redemption path.

### Impact Explanation

All `native_fee` STRK tokens collected across every `init_transfer` call are irreversibly locked in the contract. This matches the **Critical** impact category: *Irreversible fund lock, permanently unclaimable protocol value in bridge fee flows.* The protocol cannot recover its own fee revenue without a contract upgrade, exactly mirroring the M-12 judgment that "funds are stuck in the contract until an upgrade."

### Likelihood Explanation

`native_fee` is a standard, documented parameter of `init_transfer`. Any user bridging tokens and paying a native fee triggers the lock. The path is reachable by any unprivileged caller through the public `init_transfer` entrypoint with no special preconditions.

### Recommendation

Add a privileged `withdraw_native_fee` (or equivalent) function to the `IOmniBridge` interface and `OmniBridge` implementation that transfers accumulated STRK fee tokens to a designated fee recipient or treasury address, restricted to `DEFAULT_ADMIN_ROLE`.

### Proof of Concept

1. Alice calls `init_transfer(token_address, 1000, 10, 50, "recipient.near", "")` with `native_fee = 50`.
2. The contract executes `IERC20Dispatcher { contract_address: strk_token }.transfer_from(Alice, get_contract_address(), 50)`.
3. 50 STRK are now held by the `OmniBridge` contract.
4. No function in the contract allows the admin or anyone else to retrieve these 50 STRK.
5. Repeat for every bridging user who pays a native fee — all STRK accumulates with no exit path. [4](#0-3)

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,
            recipient: ByteArray,
            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::InitTransfer(
                        InitTransfer {
                            sender: caller,
                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
                )
        }
