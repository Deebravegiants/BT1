### Title
Phase-2 Invalid Transactions Bypass Per-Transaction Reference Script Size Limit via Sub-Transactions in Dijkstra Era - (File: eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Ledger.hs)

---

### Summary

In the Dijkstra era, `validateAllRefScriptSize` — the only check that enforces `maxRefScriptSizePerTx` across the top-level transaction **and all its sub-transactions** — is gated behind `tx ^. isValidTxL == IsValid True`. Phase-2 invalid transactions (`IsValid False`) skip this check entirely. Simultaneously, the block-level check (`validateBodyRefScriptsSizeTooBig`) only counts top-level transaction reference scripts and never recurses into sub-transactions. An unprivileged sender can therefore craft a phase-2 invalid top-level transaction whose sub-transactions collectively reference scripts far exceeding `maxRefScriptSizePerTx`, with no ledger rule enforcing the limit.

---

### Finding Description

**Root cause — per-tx limit gated on `IsValid True`:**

In `dijkstraLedgerTransition`, the reference-script size check is inside the `IsValid True` branch:

```haskell
(utxoStateBeforeUtxow, certStateFinal) <-
    if tx ^. isValidTxL == IsValid True
      then do
        ...
        runTest $ validateAllRefScriptSize pp originalUtxo tx   -- ← only here
        ...
      else pure (utxoStateAfterSubLedgers, certStateAfterSubLedgers)
``` [1](#0-0) 

`validateAllRefScriptSize` calls `batchNonDistinctRefScriptsSize`, which sums reference-script bytes across the top-level transaction **and every sub-transaction**:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum (foldMap' (Sum . txNonDistinctRefScriptsSize utxo)
                       (tx ^. bodyTxL . subTransactionsTxBodyL))
``` [2](#0-1) 

When `IsValid False`, this entire aggregated check is skipped.

**Root cause — block-level check does not cover sub-transactions:**

`validateBodyRefScriptsSizeTooBig` (called in both the Conway and Dijkstra BBODY rules) delegates to `totalRefScriptSizeInBlock`, which iterates only over top-level `Tx TopTx era` objects and calls `txNonDistinctRefScriptsSize` on each:

```haskell
totalRefScriptSizeInBlock protVer txs (UTxO utxo)
  | otherwise = snd $ F.foldl' accum (utxo, 0) txs
  where
    accum (!accUtxo, !accSum) tx =
      ...
       in (updatedUtxo, accSum + txNonDistinctRefScriptsSize (UTxO accUtxo) tx)
``` [3](#0-2) 

`txNonDistinctRefScriptsSize` only inspects `referenceInputsTxBodyL` and `inputsTxBodyL` of the **top-level** transaction body; it never descends into `subTransactionsTxBodyL`:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [4](#0-3) 

**Sub-transactions are still processed for `IsValid False` top-level transactions:**

`dijkstraLedgerTransition` unconditionally dispatches sub-transactions to `SUBLEDGERS` before the validity branch: [5](#0-4) 

`SUBLEDGERS → SUBLEDGER → SUBUTXO` validates each sub-transaction (checks reference-input membership, output sizes, network IDs, etc.) but contains no reference-script size check: [6](#0-5) 

**Fee calculation also excludes sub-transaction reference scripts:**

`getConwayMinFeeTxUtxo` (used by Dijkstra) passes only `txNonDistinctRefScriptsSize utxo tx` (top-level only) to `getMinFeeTx`: [7](#0-6) 

Sub-transaction reference-script bytes are therefore not priced into the minimum fee, making the attack cheaper.

---

### Impact Explanation

`maxRefScriptSizePerTx` is a protocol parameter (`dppMaxRefScriptSizePerTx` in Dijkstra) whose purpose is to bound the per-transaction reference-script deserialization cost and prevent the DDoS class of attack documented in ADR-009. By submitting a phase-2 invalid top-level transaction whose sub-transactions collectively reference scripts exceeding this limit, an attacker forces every validating node to look up and measure (and, for script-execution purposes, deserialize) reference scripts beyond the intended cap — without any ledger rule rejecting the transaction. The block-level cap (`maxRefScriptSizePerBlock`) does not compensate because it never counts sub-transaction reference scripts.

This matches the **Medium** allowed impact: *Attacker-controlled transactions exceed intended validation limits.*

---

### Likelihood Explanation

The attacker requires:
1. Sufficient ADA to create UTxO entries carrying large reference scripts (normal UTxO deposits).
2. The ability to construct a Dijkstra-era transaction with `isValid = False` and multiple sub-transactions referencing those UTxO entries — all standard, unprivileged operations.
3. A block producer willing to include the transaction (standard mempool submission).

No privileged role, key compromise, or consensus majority is required. The cost scales with the number of large reference-script UTxOs created, but the attack can be repeated across blocks as long as those UTxOs remain unspent.

---

### Recommendation

Apply `validateAllRefScriptSize` unconditionally in `dijkstraLedgerTransition`, removing the `IsValid True` guard for this specific check:

```haskell
-- Always enforce the per-tx ref-script size limit, regardless of phase-2 validity
runTest $ validateAllRefScriptSize pp originalUtxo tx

(utxoStateBeforeUtxow, certStateFinal) <-
    if tx ^. isValidTxL == IsValid True
      then do ...
      else pure (utxoStateAfterSubLedgers, certStateAfterSubLedgers)
```

Additionally, update `totalRefScriptSizeInBlock` (or add a Dijkstra-specific override) to recurse into sub-transactions so the block-level cap also accounts for sub-transaction reference scripts.

Finally, update `getMinFeeTxUtxo` for Dijkstra to include sub-transaction reference-script sizes in the minimum-fee calculation so that economic deterrence is correctly priced.

---

### Proof of Concept

1. **Setup**: Create `N` UTxO entries, each holding a large Plutus reference script of size `S` bytes (e.g., `S = 50 000` bytes, `N = 10`).

2. **Craft attack transaction**:
   - Top-level transaction: `isValid = False` (declare a phase-2 script failure; any always-failing Plutus script suffices). Include valid collateral.
   - Embed `M` sub-transactions (e.g., `M = 10`), each with `referenceInputsTxBodyL` pointing to all `N` large-script UTxOs.

3. **Resulting reference-script load per transaction**:
   - Top-level: 0 bytes counted by `validateAllRefScriptSize` (check skipped).
   - Sub-transactions: `M × N × S = 10 × 10 × 50 000 = 5 000 000` bytes — 24× the Conway hard cap of 204 800 bytes and far beyond any reasonable `maxRefScriptSizePerTx` setting.
   - Block-level check: counts only top-level reference scripts → 0 bytes → passes.

4. **Fee paid**: minimum fee based on top-level transaction size + top-level reference-script cost only; sub-transaction reference-script cost is not priced in.

5. **Effect**: Every validating node processes `5 MB` of reference-script lookups per transaction with no ledger rule rejecting it, bypassing the `maxRefScriptSizePerTx` limit entirely. [8](#0-7) [2](#0-1) [9](#0-8) [6](#0-5)

totalRefScriptSizeInBlock ::
  (AlonzoEraTx era, BabbageEraTxBody era) => ProtVer -> StrictSeq (Tx TopTx era) -> UTxO era -> Int
totalRefScriptSizeInBlock protVer txs (UTxO utxo)
  | pvMajor protVer <= natVersion @10 =
      getSum $ foldMap' (Monoid.Sum . txNonDistinctRefScriptsSize (UTxO utxo)) txs
  | otherwise =
      snd $ F.foldl' accum (utxo, 0) txs

       in (updatedUtxo, accSum + txNonDistinctRefScriptsSize (UTxO accUtxo) tx)

dijkstraSubUtxoTransition = do
  TRC (SubUtxoEnv slot pp certState originalUtxo (IsValid isValid), utxoState, stAnnTx) <-
    judgmentContext
  let tx = stAnnTx ^. txStAnnTxG

  let txBody = tx ^. bodyTxL

  runTest $ Allegra.validateOutsideValidityIntervalUTxO slot txBody

  sysSt <- liftSTS $ asks systemStart
  ei <- liftSTS $ asks epochInfo
  runTest $ Alonzo.validateOutsideForecast ei slot sysSt tx

  let allSizedOutputs = txBody ^. allSizedOutputsTxBodyF
  let allOutputs = fmap sizedValue allSizedOutputs
  runTest $ Alonzo.validateOutputTooBigUTxO pp allOutputs

  runTest $ Shelley.validateInputSetEmptyUTxO txBody

  let inputs = txBody ^. inputsTxBodyL
  let refInputs = txBody ^. referenceInputsTxBodyL
  runTest $ Shelley.validateBadInputsUTxO originalUtxo (inputs `Set.union` refInputs)
  runTest $ Shelley.validateBadInputsUTxO (utxosUtxo utxoState) inputs

  runTestOnSignal $ Shelley.validateOutputBootAddrAttrsTooBig allOutputs

  runTestOnSignal $ Babbage.validateOutputTooSmallUTxO pp allSizedOutputs

  netId <- liftSTS $ asks networkId
  runTestOnSignal $ Shelley.validateWrongNetwork netId allOutputs
  runTestOnSignal $ Shelley.validateWrongNetworkWithdrawal netId txBody
  runTestOnSignal $ validateWrongNetworkInDirectDeposit netId txBody
  runTestOnSignal $ Alonzo.validateWrongNetworkInTxBody netId txBody

  if isValid
    then do
      newState <-
        Shelley.updateUTxOStateNoFees
          pp
          utxoState
          txBody
          certState
          (utxosGovState utxoState)
          (tellEvent . TotalDeposits (hashAnnotated txBody))
          (\a b -> tellEvent $ TxUTxODiff a b)
      pure $ newState & utxosDonationL <>~ txBody ^. treasuryDonationTxBodyL
