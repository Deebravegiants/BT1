### Title
Vote Signature Verification Skipped in Consensus — (`crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`SingleHeightConsensus::handle_vote()` accepts peer votes without verifying their cryptographic signatures. A dedicated `verify_precommit_vote_signature` library function exists in `apollo_signature_manager` but is never called on the production vote-handling path. Any network peer can forge votes attributed to legitimate committee validators, accumulate a fraudulent quorum, and force consensus to commit an attacker-chosen block.

### Finding Description

`handle_vote()` in `crates/apollo_consensus/src/single_height_consensus.rs` performs three checks before accepting a vote: height match, committee membership, and duplicate/conflict detection. The function carries an explicit `// TODO(Asmaa): verify the signature` comment at line 242, acknowledging that the cryptographic check is absent. [1](#0-0) 

The `Vote` struct carries a `signature: RawSignature` field (populated with `RawSignature::default()` in tests, meaning the field exists but is never validated in production). [2](#0-1) 

The verification library is fully implemented in `apollo_signature_manager`:

```rust
pub fn verify_precommit_vote_signature(
    block_hash: BlockHash,
    signature: RawSignature,
    public_key: PublicKey,
) -> SignatureVerificationResult<bool> { ... }
``` [3](#0-2) 

This function returns `Result<bool, SignatureVerificationError>` — `Ok(true)` for a valid signature, `Ok(false)` for an invalid one, and `Err(...)` for a malformed input. The ecrecover2 analog is exact: just as `ecrecover2` called the precompile but ignored the status flag (accepting garbage output as a valid address), `handle_vote()` receives a signed vote but ignores the signature field entirely, treating any vote from a known validator address as authentic.

The function is only exercised in unit tests: [4](#0-3) 

It is never imported or called from any production consensus path. `grep` for `verify_precommit_vote_signature` returns only `signature_manager.rs` (definition) and `signature_manager_test.rs` (tests).

Votes arrive via the `BroadcastVoteChannel` P2P channel and are dispatched to `shc.handle_vote(message)` in `manager.rs`: [5](#0-4) 

### Impact Explanation

An attacker who can send messages on the consensus gossip network can:

1. Observe the current committee (public information).
2. Craft `Vote` messages with `voter` set to any legitimate validator address and `signature` set to arbitrary bytes (or `RawSignature::default()`).
3. Send enough forged votes to satisfy the Byzantine quorum threshold (≥ 2/3 of total weight).
4. The state machine reaches `DecisionReached` for an attacker-chosen `ProposalCommitment`, causing `context.decision_reached()` to commit a fraudulent block.

This maps to the **Critical** impact category: *Invalid or unauthorized Starknet transaction accepted through account validation, signature, nonce, chain id, fee/resource bound, paymaster, or account-deployment logic* — and more directly to the consensus commitment path where a wrong `ProposalCommitment` is finalized.

### Likelihood Explanation

The consensus network is a broadcast channel reachable by any node that can establish a libp2p connection. The committee addresses are public. No privileged access is required beyond network connectivity. The TODO comment confirms the gap is known but unimplemented, not a deliberate design choice.

### Recommendation

Call `verify_precommit_vote_signature` (or the equivalent for prevotes) inside `handle_vote()` before passing the vote to the state machine. The public key for each validator must be resolved from the committee (e.g., via `self.committee.members()` keyed on `vote.voter`). Reject votes where the signature is absent, malformed, or does not verify against the voter's registered public key. The existing `SignatureVerificationError` type and `verify_precommit_vote_signature` API are already suitable for this purpose.

    proposal_commitment: Some(attacker_chosen_commitment),

// handle_vote() passes all three guards:
// 1. vote.height == height          ✓
// 2. committee.members() contains voter ✓
// 3. received_vote() returns New    ✓
// 4. signature check                ← MISSING (TODO)
//

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
