### Title
Missing On-Chain Response Verification for `AppPublicKey` CKD Variant Allows Malicious Participant to Inject Arbitrary Derived Key - (File: crates/contract/src/lib.rs)

---

### Summary

The `respond_ckd` function in the MPC contract applies cryptographic on-chain verification of the CKD response **only** for the `AppPublicKeyPV` (publicly verifiable) variant. For the `AppPublicKey` (privately verifiable) variant, the response is accepted with **no verification whatsoever**. Any single malicious attested participant can call `respond_ckd` with a fabricated `CKDResponse` for any pending non-PV CKD request, delivering an attacker-controlled key to the requesting user.

---

### Finding Description

The `respond_ckd` function contains an asymmetric validation branch:

```rust
// crates/contract/src/lib.rs:675-682
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}   // ← NO CHECK
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
``` [1](#0-0) 

For `AppPublicKeyPV`, `ckd_output_check` performs a BLS12-381 pairing check that cryptographically binds the response `(big_c, big_y)` to the network public key and the `app_id`:

```rust
// crates/contract/src/primitives/ckd.rs:80-101
pub(crate) fn ckd_output_check(...) -> bool {
    // e(big_c, g2) = e(big_y, app_pk2) . e(hash_point, public_key)
    env::bls12381_pairing_check(&pairing_input)
}
``` [2](#0-1) 

For `AppPublicKey`, the branch is an empty arm — the response bytes are serialized and delivered to the user's yield callback without any cryptographic check.

By contrast, the analogous `respond` function for threshold signatures **always** verifies the signature against the derived public key before accepting it:

```rust
// crates/contract/src/lib.rs:642-644
if !signature_is_valid {
    return Err(RespondError::InvalidSignature.into());
}
``` [3](#0-2) 

The `CKDRequest` (including `app_id`, `app_public_key`, and `domain_id`) is stored in the public contract state and is readable by any observer. Any attested participant can therefore:

1. Read a pending `AppPublicKey` CKD request from `pending_ckd_requests`.
2. Construct an arbitrary `CKDResponse` with chosen `big_c` and `big_y` values (any valid G1 points).
3. Call `respond_ckd` before the honest leader does.
4. The contract resolves the yield and delivers the fabricated response to the user. [4](#0-3) 

The `CKDRequest` key used for lookup is derived from `(app_public_key, app_id, domain_id)`:

    #[handle_result]
    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

pub(crate) fn ckd_output_check(
    app_id: &dtos::CkdAppId,
    output: &CKDResponse,
    app_public_key: &dtos::CKDAppPublicKeyPV,
    public_key: &dtos::Bls12381G2PublicKey,
) -> bool {
    let big_c = env::bls12381_p1_decompress(&output.big_c);
    let big_y = env::bls12381_p1_decompress(&output.big_y);
    let pk2 = env::bls12381_p2_decompress(&app_public_key.pk2);
    let pk = env::bls12381_p2_decompress(public_key);
    let hash_point = hash_app_id_with_pk(public_key.as_slice(), app_id.as_ref());

    let pairing_input = [
        big_c.as_slice(),
        MINUS_G2_GENERATOR_UNCOMPRESSED.as_slice(),
        big_y.as_slice(),
        pk2.as_slice(),
        hash_point.as_slice(),
        pk.as_slice(),
    ]
    .concat();
    env::bls12381_pairing_check(&pairing_input)
}
