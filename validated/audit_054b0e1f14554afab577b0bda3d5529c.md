### Title
Missing `curve.check()` after `from_bin` in `blob_to_keyset` allows Ed25519 infinity point as accepted root public key — (`src/cbmpc/api/hd_keyset_eddsa_2p.cpp`)

### Summary
`blob_to_keyset` in the EdDSA-2P HD keyset API deserializes `root_Q_compressed` via `from_bin` but omits the mandatory `curve.check()` call present in every other comparable deserialization path. The Ed25519 infinity encoding (`y=1, x=0`) passes `from_bin` without error, so a crafted keyset blob causes `extract_root_public_key_compressed` (and `refresh`/`derive_eddsa_2p_keys`) to return `SUCCESS` while accepting an invalid root public key.

### Finding Description
In `src/cbmpc/api/hd_keyset_eddsa_2p.cpp`, `blob_to_keyset` (lines 63–81) calls `keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed)` and propagates any error, but never calls `keyset.curve.check(keyset.root.Q)`. The analogous functions in `eddsa2pc.cpp`, `eddsa_mp.cpp`, and `ecdsa2pc.cpp` all call `curve.check()` immediately after `from_bin`.

The Ed25519 compressed encoding of the identity/infinity point is `[0x01, 0x00, ..., 0x00]` (32 bytes, y=1 little-endian, sign bit 0). `ec25519_core::from_bin` processes this as: y=1 → u=0 → x=0 → check=0 → returns `SUCCESS`, storing the projective point (X=0, Y=1, Z=1). `ecurve_t::check` would reject this via its `is_infinity()` guard, but that guard is never invoked.

All three public API entry points that call `deserialize_keyset_blob` are affected: `extract_root_public_key_compressed`, `refresh`, and `derive_eddsa_2p_keys`.

### Impact Explanation
A caller that invokes `extract_root_public_key_compressed` with a crafted blob receives `SUCCESS` and a 32-byte output encoding the infinity point. Any downstream system that trusts this output as a valid Ed25519 public key (e.g., for signature verification, key agreement, or HD derivation) operates on an invalid key. For `derive_eddsa_2p_keys`, the infinity root public key propagates into the MPC derivation protocol, producing derived key blobs whose public key component is cryptographically invalid. This constitutes accepted invalid cryptographic output at a public API boundary.

### Likelihood Explanation
The attacker only needs to craft a syntactically valid keyset blob (correct version, role, curve fields, and scalar shares in range) with `root_Q_compressed = [0x01, 0x00, ..., 0x00]`. No protocol participation or threshold collusion is required. The blob format is documented implicitly by the serialization code and is straightforward to reverse-engineer.

### Recommendation
Add `curve.check()` immediately after `from_bin` for both `root_Q` and `root_K` in `blob_to_keyset` (`src/cbmpc/api/hd_keyset_eddsa_2p.cpp`), mirroring the pattern used in `eddsa2pc.cpp` and `eddsa_mp.cpp`:

```cpp
error_t rv = keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed);
if (rv) return rv;
if (keyset.curve.check(keyset.root.Q)) return coinbase::error(E_FORMAT, "invalid keyset blob");
rv = keyset.root.K.from_bin(keyset.curve, blob.root_K_compressed);
if (rv) return rv;
if (keyset.curve.check(keyset.root.K)) return coinbase::error(E_FORMAT, "invalid keyset blob");
```

### Proof of Concept
Construct a keyset blob with:
- `version = 1`, `role = 0`, `curve = ed25519`
- `root_Q_compressed = [0x01, 0x00, ..., 0x00]` (32 bytes — Ed25519 infinity encoding)
- `root_K_compressed` = any valid Ed25519 point encoding
- `x_share`, `k_share` = any values in `[0, q)`

Call `coinbase::api::hd_keyset_eddsa_2p::extract_root_public_key_compressed(blob, out)`. The function returns `SUCCESS` and `out` contains `[0x01, 0x00, ..., 0x00]` — the infinity point — rather than an error.

static error_t from_bin(point_t& R, mem_t bin) {
  if (bin.size != 32) return coinbase::error(E_FORMAT);

  buf_t buf = bin.rev();
  uint8_t neg = buf[0] >> 7;
  buf[0] &= 0x7f;
  fe_t y = fe_t::to_fe(uint256_t::from_bin(buf));

  // x² = (y² - 1) / (dy² + 1)

  fe_t u, v, w, vxx, check;

  u = y * y;
  v = u * formula_t::get_d();
  u -= fe_t::one();       // u = y^2-1
  v += fe_t::one();       // v = dy^2+1
  w = u * v;              // w = u*v
  fe_t x = w.pow22523();  // x = w^((q-5)/8)
  x *= u;                 // x = u * w^((q-5)/8)

  vxx = x * x;
  vxx *= v;
  check = vxx - u;  // vx^2-u
  if (!check.is_zero()) {
    check = vxx + u;  // vx^2+u
    if (!check.is_zero()) {
      return coinbase::error(E_CRYPTO);
    }
    static const fe_t sqrtm1 =
        fe_t::from_bn(bn_t::from_hex("2b8324804fc1df0b2b4d00993dfbd7a72f431806ad2fe478c4ee1b274a0ea0b0"));
    x *= sqrtm1;
  }

  uint256_t x_val = x.from_fe();
  if (neg != (x_val.w0 & 1)) x = -x;

  R.x = x;
  R.y = y;
  R.z = fe_t::one();
  return 0;
}
