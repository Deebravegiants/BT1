### Title
Unguarded Assertion on Peer-Supplied Coin-State Count Permanently Drops Pool Wallet Singleton Sync State — (File: `chia/wallet/wallet_state_manager.py`)

---

### Summary

Inside `WalletStateManager._add_coin_states`, when processing a spent `POOLING_WALLET` singleton coin, the code walks the singleton's spend chain by repeatedly calling `wallet_node.get_coin_state` against the connected peer. At line 2122 the result is checked with a bare `assert len(new_coin_state) == 1`. If the peer returns an empty list — either because it is malicious or because a reorg occurred between the initial coin-state push and this follow-up query — the assertion raises `AssertionError`. The outer exception handler classifies this as a non-retryable error and permanently removes the coin state from the retry store, leaving the pool wallet's singleton tip permanently stale until a full manual resync.

---

### Finding Description

`_add_coin_states` in `chia/wallet/wallet_state_manager.py` processes batches of `CoinState` objects received from a peer. For a `POOLING_WALLET` coin that has been spent, it enters a loop to walk the singleton's spend chain:

```python
# chia/wallet/wallet_state_manager.py  lines 2089-2123

    curr_coin_state = new_coin_state[0]
```

The `get_coin_state` call is an external RPC to the connected peer. If the peer returns `[]` (empty list), the `assert` raises `AssertionError`. This propagates to the outer handler:

```python
# lines 2227-2235
except Exception as e:
    self.log.exception(f"Failed to add coin_state: {coin_state}, error: {e}")
    if rollback_wallets is not None:
        self.wallets = rollback_wallets
    if isinstance(e, (PeerRequestException, aiosqlite.Error)):
        await self.retry_store.add_state(coin_state, peer.peer_node_id, fork_height)
    else:
        await self.retry_store.remove_state(coin_state)   # ← permanent drop
    continue
```

`AssertionError` is not a `PeerRequestException` or `aiosqlite.Error`, so the coin state is **permanently removed from the retry store** rather than queued for retry. The DB transaction is rolled back, the in-memory wallet state is restored, and the loop moves on. The pool wallet's singleton tip is now stale and will not be automatically corrected. ---

### Impact Explanation

**Impact class:** High — Corruption of pool membership state and wallet sync state with direct security impact.

Once the coin state is dropped from the retry store, the pool wallet's view of the singleton is frozen at the pre-transition state. Concretely:

- `PoolWallet.get_current_state()` reads from `pool_store`, which was never updated (DB rolled back). It returns the stale tip.
- `PoolWallet.new_peak()` and `generate_travel_transactions()` operate on the stale tip, so any subsequent pool-state transitions (join, leave, absorb rewards) will construct spends against a coin that no longer exists on-chain. These spends will be rejected by the mempool.
- Pool reward coins accumulate at the `p2_singleton_puzzle_hash` but cannot be absorbed because the wallet does not know the current singleton tip coin.
- The wallet will not automatically recover: the retry store no longer holds the coin state, and no background task re-requests it. Recovery requires a manual full resync. [3](#0-2) [4](#0-3) [5](#0-4) 

---

### Likelihood Explanation

**Attacker path (malicious peer):** Any unprivileged actor can run a Chia full node. When a wallet connects to it, the malicious node:
1. Sends a legitimate `CoinState` update for the victim's pool wallet singleton (obtained by observing the chain).
2. When the wallet follows up with `get_coin_state([new_singleton_name])`, the malicious node returns `[]`.
3. The assertion fires, the coin state is permanently dropped.

**Non-malicious trigger (reorg):** A reorg between the moment the peer pushes the original `CoinState` and the moment the wallet issues the follow-up `get_coin_state` query can cause the new singleton coin to temporarily disappear from the peer's view, producing the same empty-list response and the same permanent drop. [6](#0-5) [7](#0-6) 

---

### Recommendation

Replace the bare `assert` with an explicit guard that treats an empty response as a retryable peer error:

```python
if len(new_coin_state) != 1:
    raise PeerRequestException(
        f"Expected 1 coin state for singleton {new_singleton_name.hex()}, "
        f"got {len(new_coin_state)}"
    )
curr_coin_state = new_coin_state[0]
```

`PeerRequestException` is already handled by the retry path (`retry_store.add_state`), so the coin state will be retried on the next cycle or against a different peer rather than being silently discarded. [8](#0-7) 

---

### Proof of Concept

1. Run a patched full node that intercepts `respond_coin_state` for any request whose coin name matches a known pool wallet singleton's child coin and returns an empty `RespondCoinState`.
2. Connect a wallet that has an active `POOLING_WALLET` to this node.
3. Mine a block that spends the pool wallet singleton (e.g., an absorb-rewards transaction).
4. The full node pushes the `CoinState` for the spent singleton to the wallet.
5. The wallet enters the `while curr_coin_state.spent_height is not None:` loop, fetches the spend, applies the state transition, then calls `get_coin_state([new_singleton_name])`.
6. The malicious node returns `[]`.
7. `assert len(new_coin_state) == 1` raises `AssertionError`.
8. The exception handler removes the coin state from the retry store.
9. `pw_status` now returns the stale pre-transition pool state indefinitely; subsequent `pw_absorb_rewards` calls construct invalid spends and are rejected. [9](#0-8) [10](#0-9)

```python
    async def apply_state_transition(
        self, new_state: CoinSpend, block_height: uint32, action_scope: WalletActionScope
    ) -> bool:
        """
        Updates the Pool state (including DB) with new singleton spends.
        The DB must be committed after calling this method. All validation should be done here. Returns True iff
        the spend is a valid transition spend for the singleton, False otherwise.
        """
        tip: tuple[uint32, CoinSpend] = await self.get_tip()
        tip_spend = tip[1]

        tip_coin: Coin | None = get_most_recent_singleton_coin_from_coin_spend(tip_spend)
        assert tip_coin is not None
        spent_coin_name: bytes32 = tip_coin.name()

        if spent_coin_name != new_state.coin.name():
            history: list[tuple[uint32, CoinSpend]] = await self.get_spend_history()
            if new_state.coin.name() in [sp.coin.name() for _, sp in history]:
                self.log.info(f"Already have state transition: {new_state.coin.name().hex()}")
            else:
                self.log.warning(
                    f"Failed to apply state transition. tip: {tip_coin} new_state: {new_state} height {block_height}"
                )
            return False

        await self.wallet_state_manager.pool_store.add_spend(self.wallet_id, new_state, block_height)
        tip_spend = (await self.get_tip())[1]
        self.log.info(f"New PoolWallet singleton tip_coin: {tip_spend} farmed at height {block_height}")

        # If we have reached the target state, resets it to None. Loops back to get current state
        for _, added_spend in reversed(
            await self.wallet_state_manager.pool_store.get_spends_for_wallet(self.wallet_id)
        ):
            latest_state: PoolState | None = solution_to_pool_state(added_spend)
            if latest_state is not None:
                if self.target_state == latest_state:
                    self.target_state = None
                    self.next_transaction_fee = uint64(0)
                    self.next_tx_config = DEFAULT_TX_CONFIG
                break

        await self.update_pool_config(action_scope)
        return True
```

```python
    async def new_peak(self, peak_height: uint32) -> None:
        # This gets called from the WalletStateManager whenever there is a new peak

        pool_wallet_info: PoolWalletInfo = await self.get_current_state()
        tip_height, tip_spend = await self.get_tip()

        if self.target_state is None:
            return
        if self.target_state == pool_wallet_info.current:
            self.target_state = None
            raise ValueError(f"Internal error. Pool wallet {self.wallet_id} state: {pool_wallet_info.current}")

        if (
            self.target_state.state in {FARMING_TO_POOL.value, SELF_POOLING.value}
            and pool_wallet_info.current.state == LEAVING_POOL.value
        ):
            leave_height = tip_height + pool_wallet_info.current.relative_lock_height

            # Add some buffer (+2) to reduce chances of a reorg
            if peak_height > leave_height + 2:
                unconfirmed: list[
                    TransactionRecord
                ] = await self.wallet_state_manager.tx_store.get_unconfirmed_for_wallet(self.wallet_id)
                next_tip: Coin | None = get_most_recent_singleton_coin_from_coin_spend(tip_spend)
                assert next_tip is not None

                if any([rem.name() == next_tip.name() for tx_rec in unconfirmed for rem in tx_rec.removals]):
                    self.log.info("Already submitted second transaction, will not resubmit.")
                    return

                self.log.info(f"Attempting to leave from\n{pool_wallet_info.current}\nto\n{self.target_state}")
                assert self.target_state.version == POOL_PROTOCOL_VERSION
                assert pool_wallet_info.current.state == LEAVING_POOL.value
                assert self.target_state.target_puzzle_hash is not None

                if self.target_state.state == SELF_POOLING.value:
                    assert self.target_state.relative_lock_height == 0
                    assert self.target_state.pool_url is None
                elif self.target_state.state == FARMING_TO_POOL.value:
                    assert self.target_state.relative_lock_height >= self.MINIMUM_RELATIVE_LOCK_HEIGHT
                    assert self.target_state.pool_url is not None

                async with self.wallet_state_manager.new_action_scope(self.next_tx_config, push=True) as action_scope:
                    await self.generate_travel_transactions(self.next_transaction_fee, action_scope)
```

```python
async def fetch_coin_spend(height: uint32, coin: Coin, peer: WSChiaConnection) -> CoinSpend:
    solution_response = await peer.call_api(
        FullNodeAPI.request_puzzle_solution, RequestPuzzleSolution(coin.name(), height)
    )
    if solution_response is None or not isinstance(solution_response, RespondPuzzleSolution):
        raise PeerRequestException(f"Was not able to obtain solution {solution_response}")
    coin_id = coin.name()
    if solution_response.response.puzzle.get_tree_hash() != coin.puzzle_hash:
        raise PeerRequestException(f"Peer returned wrong puzzle hash for coin {coin_id}")
    if solution_response.response.coin_name != coin_id:
        raise PeerRequestException(f"Peer returned wrong coin name in puzzle solution for coin {coin_id}")

    return make_spend(
        coin,
        solution_response.response.puzzle,
        solution_response.response.solution,
    )
```
