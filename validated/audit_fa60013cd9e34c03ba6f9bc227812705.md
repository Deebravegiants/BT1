### Title
Zero-Rate Broadcast via Unguarded `updateRate()` Before Oracle Initialization — (`contracts/cross-chain/CrossChainRateProvider.sol`)

### Summary

`CrossChainRateProvider.updateRate()` has no guard against broadcasting a zero rate. If called before `LRTOracle.updateRSETHPrice()` has ever been invoked, it propagates `rsETHPrice == 0` to the L2 `CrossChainRateReceiver`, which also stores it without validation. Any subsequent call to `RSETHPoolV2.deposit()` then reverts with a division-by-zero, freezing deposits until a valid rate is re-broadcast.

### Finding Description

**Root cause — no zero-check in `updateRate()`:**

`CrossChainRateProvider.updateRate()` unconditionally reads `getLatestRate()` and sends it over LayerZero:

```solidity
function updateRate() external payable nonReentrant {
    uint256 latestRate = getLatestRate();   // may return 0
    rate = latestRate;
    // ...
    ILayerZeroEndpoint(layerZeroEndpoint).send{ value: msg.value }(
        dstChainId, remoteAndLocalAddresses, _payload, ...
    );
}
``` [1](#0-0) 

`RSETHRateProvider.getLatestRate()` directly reads the storage variable `rsETHPrice` from `LRTOracle`:

```solidity
function getLatestRate() public view override returns (uint256) {
    return ILRTOracle(rsETHPriceOracle).rsETHPrice();
}
``` [2](#0-1) 

`LRTOracle.rsETHPrice` is a plain `uint256` state variable initialized to `0` by the EVM. It is only set inside `_updateRsETHPrice()`, which is called by `updateRSETHPrice()` or `updateRSETHPriceAsManager()`. Before either of those is ever called, `rsETHPrice == 0`. [3](#0-2) 

**Root cause — no zero-check in `lzReceive()`:**

`CrossChainRateReceiver.lzReceive()` stores whatever rate it receives, including 0:

```solidity
uint256 _rate = abi.decode(_payload, (uint256));
rate = _rate;
``` [4](#0-3) 

`CrossChainRateReceiver.getRate()` then returns this stored zero: [5](#0-4) 

**Division-by-zero in `RSETHPoolV2`:**

`RSETHPoolV2.viewSwapRsETHAmountAndFee()` divides by the oracle rate with no zero guard:

```solidity
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;  // panics if 0
``` [6](#0-5) 

`deposit()` invokes this through the `limitDailyMint` modifier before any state change, so every deposit reverts: [7](#0-6) 

### Impact Explanation

Deposits into `RSETHPoolV2` revert with a Solidity division-by-zero panic for as long as the receiver holds `rate == 0`. No funds are lost, but the pool fails to deliver its promised swap service. The state is correctable by calling `updateRate()` again once `rsETHPrice` is properly set, matching the **Low — contract fails to deliver promised returns, but doesn't lose value** scope.

### Likelihood Explanation

The precondition (`rsETHPrice == 0`) exists only during the deployment window before `updateRSETHPrice()` is first called. On a live mainnet deployment this window is narrow, but the code contains no guard to prevent it. Any caller (no role required) can trigger `updateRate()` during this window. The `updateRate()` function is intentionally public and payable, so no special access is needed.

### Recommendation

Add a zero-rate guard in `updateRate()` and/or `lzReceive()`:

```solidity
// In CrossChainRateProvider.updateRate():
uint256 latestRate = getLatestRate();
require(latestRate != 0, "Rate cannot be zero");
```

Optionally, add a symmetric guard in `CrossChainRateReceiver.lzReceive()`:

```solidity
require(_rate != 0, "Received zero rate");
```

### Proof of Concept

1. Deploy `LRTOracle` via proxy; do **not** call `updateRSETHPrice()` → `rsETHPrice == 0`.
2. Deploy `RSETHRateProvider` pointing to that oracle.
3. Deploy `RSETHRateReceiver` (L2 side, or mock `lzReceive` directly in a fork test).
4. Any EOA calls `RSETHRateProvider.updateRate{value: fee}()`.
5. `getLatestRate()` returns `0`; LayerZero delivers the payload; `RSETHRateReceiver.rate` becomes `0`.
6. Call `RSETHPoolV2.deposit{value: 1 ether}("")` → `viewSwapRsETHAmountAndFee` executes `1e18 / 0` → EVM panic revert.
7. Assert: `receiver.getRate() == 0` and `deposit` reverts.
