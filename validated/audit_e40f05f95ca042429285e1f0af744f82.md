### Title
Hardcoded `ADDRESS_REGISTRAR_ACCOUNT_ID` Is Mainnet-Only and Breaks Testnet EOA Base-Token Transfers - (`runtime/near-wallet-contract/implementation/wallet-contract/src/lib.rs`)

---

### Summary

The wallet contract bakes `ADDRESS_REGISTRAR_ACCOUNT_ID = "address-map.near"` at compile time. The build system (`build.rs`) correctly parameterises `CHAIN_ID` per network (397 / 398 / 399), but performs no equivalent substitution for `ADDRESS_REGISTRAR_ACCOUNT_ID`. Every compiled variant — mainnet, testnet, and localnet — therefore calls the same `address-map.near` account. On NEAR testnet, top-level accounts end in `.testnet`, so `address-map.near` is not the testnet address registrar. Any testnet `rlp_execute` call that reaches the address-check branch will issue a cross-contract call to a non-existent (or wrong) account, the promise will fail, and the transaction type is permanently broken for testnet users.

---

### Finding Description

**Hardcoded constant** [1](#0-0) 

```
const ADDRESS_REGISTRAR_ACCOUNT_ID: &str = std::include_str!("ADDRESS_REGISTRAR_ACCOUNT_ID");
```

The included file contains: [2](#0-1) 

```
address-map.near
```

**Build system handles `CHAIN_ID` per network but not `ADDRESS_REGISTRAR_ACCOUNT_ID`**

`build.rs` reads the current `CHAIN_ID` file, overwrites it with the correct per-network value (397 / 398 / 399), builds the WASM, then restores the file: [3](#0-2) 

No analogous step exists for `ADDRESS_REGISTRAR_ACCOUNT_ID`. All three build variants (`wallet_contract_mainnet.wasm`, `wallet_contract_testnet.wasm`, `wallet_contract_localnet.wasm`) embed the same `address-map.near` string.

**Execution path that fails**

Inside `inner_rlp_execute`, when the transaction kind is `EOABaseTokenTransfer { address_check: Some(address), .. }`, the contract issues a cross-contract call to the registrar: [4](#0-3) 

On testnet, `address-map.near` does not resolve to the testnet address registrar (NEAR testnet top-level accounts end in `.testnet`). The promise fails, and `address_check_callback` returns: [5](#0-4) 

```rust
PromiseResult::Failed => {
    return PromiseOrValue::Value(ExecuteResponse {
        success: false,
        error: Some("Call to Address Registrar contract failed".into()),
    });
}
```

Critically, the nonce was **not** incremented before the registrar call (the increment is deferred to `address_check_callback`), so the nonce is never advanced for this transaction type. The user can retry, but every retry hits the same broken registrar and fails identically — permanently blocking this transaction class.

---

### Impact Explanation

Any testnet eth-implicit account user who submits an Ethereum-emulated base-token transfer to another eth-implicit account (the `EOABaseTokenTransfer { address_check: Some(_) }` branch) will receive a permanent failure. The nonce is never incremented for this path, so the wallet is not bricked entirely, but this specific transaction type is rendered non-functional on testnet. If the testnet address registrar account happens to be owned by a third party, that party could also return arbitrary lookup results, potentially misleading the relayer-honesty check.

---

### Likelihood Explanation

The `address_check` branch is triggered whenever the Ethereum transaction payload is parseable as a supported emulation and the `target` is another eth-implicit account — a common and intended use case for the wallet contract. Any testnet user attempting this will encounter the failure. The root cause is a straightforward build-system omission: `CHAIN_ID` is parameterised per network but `ADDRESS_REGISTRAR_ACCOUNT_ID` is not.

---

### Recommendation

Extend `build_contract` in `build.rs` to accept a per-network `address_registrar_account_id` string, write it to `ADDRESS_REGISTRAR_ACCOUNT_ID` before the Docker build (mirroring the `CHAIN_ID` pattern), and restore the file afterwards: [6](#0-5) 

Concretely:
- Mainnet → `address-map.near`
- Testnet → the actual testnet registrar account (e.g. `address-map.testnet`)
- Localnet → a locally-deployed registrar account

---

### Proof of Concept

1. Deploy the testnet wallet contract (`wallet_contract_testnet.wasm`) to a testnet eth-implicit account.
2. Craft a signed Ethereum transaction whose payload is parseable as an ERC-20 or base-token emulation, with `to` set to another eth-implicit address not registered in the registrar.
3. Call `rlp_execute(target, tx_bytes_b64)` where `target` is the eth-implicit account corresponding to `to`.
4. The contract enters the `EOABaseTokenTransfer { address_check: Some(address) }` branch and calls `address-map.near::lookup(address)` on testnet.
5. The call fails (account does not exist or is not the registrar); `address_check_callback` returns `success: false, error: "Call to Address Registrar contract failed"`.
6. The nonce remains unchanged; every subsequent attempt produces the same result, permanently blocking this transaction type for the user.

fn build_contract(
    dir: &str,
    contract_name: &str,
    output: &str,
    chain_id: u64,
) -> anyhow::Result<()> {
    let wasm_target_path = format!("./res/{}.wasm", output);
    if Path::new(&wasm_target_path).exists() {
        // Skip building if an artifact is already present
        return Ok(());
    }

    let absolute_dir = Path::new(dir).canonicalize()?;

    let chain_id_path = absolute_dir.join("wallet-contract/src/CHAIN_ID");
    let chain_id_content = std::fs::read(&chain_id_path).context("Failed to read CHAIN_ID file")?;

    // Update the chain id before building
    std::fs::write(&chain_id_path, chain_id.to_string().into_bytes())?;
    docker_build(absolute_dir.to_str().expect("path should be valid UTF-8"))?;

    // Restore chain id file to original value after building
    std::fs::write(&chain_id_path, chain_id_content)?;

    let build_artifact_path =
        format!("target/wasm32-unknown-unknown/release/{}.wasm", contract_name);
    let src = absolute_dir.join(build_artifact_path);

    std::fs::copy(&src, &wasm_target_path)
        .with_context(|| format!("Failed to copy `{}` to `{}`", src.display(), wasm_target_path))?;
    Ok(())
}
