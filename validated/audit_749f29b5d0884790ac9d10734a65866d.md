Audit Report

## Title
Undistributed Reward Tokens Permanently Stranded in `KernelDepositPool` When All Stakers Exit Before Reward Period Ends - (File: contracts/KERNEL/KernelDepositPool.sol)

## Summary

When all stakers exit during an active reward period, the remaining undistributed reward tokens accumulate in the contract with no on-chain path to recovery. The `NoStakedTokens` guard blocks the admin from rolling over the stranded balance, and once the period expires, the subsequent `notifyRewardAmount` call ignores the pre-existing balance entirely, permanently freezing those tokens.

## Finding Description

`notifyRewardAmount` uses a transfer-in pattern to compute `receivedAmount` as `balanceAfter - balanceBefore`. When the previous period has expired (`block.timestamp >= finishAt`), the new rate is set solely from `receivedAmount`:

```solidity
// L579-580
if (block.timestamp >= finishAt) {
    rewardRate = receivedAmount / duration;
```

Any tokens already sitting in the contract from the prior period are captured in `balanceBefore` and are therefore invisible to the new rate calculation. They are never scheduled for distribution.

The `NoStakedTokens` guard at L570 prevents the admin from calling `notifyRewardAmount` while the pool is empty, closing the only administrative path to roll over the stranded balance before `finishAt`. No `rescue`, `recover`, or `sweep` function exists anywhere in the contract (confirmed by full-contract search).

The contract's own NatDoc at L17–22 explicitly acknowledges this behavior:

> *"If `totalKernelStaked` ever hits zero during a reward distribution window, any remaining rewards for that period will stay locked in the contract."*

The stated mitigation is purely operational ("ensuring there are always some tokens staked"), with no on-chain enforcement. Normal user withdrawals via `initiateWithdrawal` can reduce `totalKernelStaked` to zero at any time, defeating the operational assumption.

**Exploit path:**
1. Admin calls `notifyRewardAmount(1000e18)` → `rewardRate` set, `finishAt = T + 7 days`.
2. At `T + 3 days`, all stakers call `initiateWithdrawal()` → `totalKernelStaked = 0`. ~571e18 tokens remain unallocated.
3. Admin attempts `notifyRewardAmount(...)` to rescue → reverts: `NoStakedTokens()` (L570).
4. `finishAt` passes. A new staker deposits.
5. Admin calls `notifyRewardAmount(1000e18)` → `rewardRate = 1000e18 / duration`. The 571e18 stranded tokens are in `balanceBefore`; `receivedAmount = 1000e18` only. 571e18 tokens are permanently locked.

## Impact Explanation

**Medium — Permanent freezing of unclaimed yield.** Reward tokens legitimately deposited by the admin for staker distribution become permanently locked in the contract. No staker can ever claim them, and no admin function can recover them. The contract's own documentation confirms the tokens "will stay locked."

## Likelihood Explanation

The trigger is entirely composed of normal, unprivileged user actions: stakers calling `initiateWithdrawal`. No coordination, special privilege, or external dependency is required. Mass exits are a realistic market event (price drop, better yield elsewhere). The admin has no on-chain recourse once `totalKernelStaked` hits zero during an active period.

## Recommendation

When starting a new reward period after the previous one has ended, include any excess token balance already held by the contract in the new `rewardRate` calculation:

```solidity
if (block.timestamp >= finishAt) {
    uint256 leftover = rewardsToken.balanceOf(address(this)) - receivedAmount;
    rewardRate = (receivedAmount + leftover) / duration;
} else {
    uint256 remaining = (finishAt - block.timestamp) * rewardRate;
    rewardRate = (receivedAmount + remaining) / duration;
}
```

Alternatively, add an admin-callable function to re-queue stranded tokens into the next reward period, or remove the `NoStakedTokens` guard when the previous period has already expired (so the admin can roll over the balance before a new staker is required).

## Proof of Concept

Foundry test plan:

```solidity
function test_strandedRewards() public {
    // 1. Admin notifies 1000e18 rewards, duration = 7 days
    vm.prank(admin);
    pool.notifyRewardAmount(1000e18);

    // 2. At T+3 days, all stakers withdraw
    vm.warp(block.timestamp + 3 days);
    vm.prank(staker);
    pool.initiateWithdrawal(stakerBalance);
    assertEq(pool.totalKernelStaked(), 0);

    // 3. Admin cannot rescue — reverts NoStakedTokens
    vm.prank(admin);
    vm.expectRevert(KernelDepositPool.NoStakedTokens.selector);
    pool.notifyRewardAmount(1000e18);

    // 4. Period expires, new staker deposits
    vm.warp(block.timestamp + 4 days + 1);
    vm.prank(newStaker);
    pool.stake(1e18);

    // 5. Admin starts new period with 1000e18
    uint256 contractBalanceBefore = rewardsToken.balanceOf(address(pool));
    vm.prank(admin);
    pool.notifyRewardAmount(1000e18);
    uint256 contractBalanceAfter = rewardsToken.balanceOf(address(pool));

    // Stranded tokens are still in the contract, not included in rewardRate
    // rewardRate = 1000e18 / duration, not (1000e18 + ~571e18) / duration
    assertGt(contractBalanceBefore, 0); // stranded tokens present
    // Verify rewardRate only accounts for new deposit
    assertEq(pool.rewardRate(), 1000e18 / pool.duration());
}
```

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
