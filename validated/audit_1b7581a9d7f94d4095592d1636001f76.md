### Title
Sub-transaction treasury donation creates ADA without UTxO deduction — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxo.hs`)

### Summary
`dijkstraSubUtxoTransition` applies `utxosDonationL <>~ txBody ^. treasuryDonationTxBodyL` for every valid sub-transaction without any value-conservation check. Because `updateUTxOStateNoFees` does not deduct the donation from the UTxO, and no other rule in the sub-tx processing chain enforces `inputs_value = outputs_value + donation`, an attacker can set `dstbrTreasuryDonation` to an arbitrarily large `Coin` in a sub-tx whose inputs and outputs are otherwise balanced, inflating `utxosDonation` by that amount without removing any ADA from the UTxO.

### Impact Explanation
An attacker submits a top-level Dijkstra transaction containing a sub-transaction whose inputs and outputs are balanced (so the UTxO is unaffected) but whose `dstbrTreasuryDonation` is set to an arbitrarily large value. After processing, `utxosDonation` is inflated by that amount with no corresponding ADA removed from the UTxO. At the next epoch boundary this inflated donation is moved into the treasury, permanently increasing the total ADA supply. This is direct, unconditional ADA creation — a Critical impact.

### Likelihood Explanation
The Dijkstra era is a new era under active development. Any user who can submit a valid top-level Dijkstra transaction (an unprivileged operation) can embed sub-transactions. The sub-tx body format already supports `dstbrTreasuryDonation` and the CBOR decoder accepts any positive `Coin` value for it. No governance, key, or threshold is required. [10](#0-9) 

### Recommendation
Add a value-conservation check inside `dijkstraSubUtxoTransition` that enforces:

```
inputs_value = outputs_value + deposit_change + treasury_donation
```

This mirrors how the top-level UTXO rule enforces conservation. Alternatively, prohibit `treasuryDonationTxBodyL` in sub-transactions entirely (set it to zero in the sub-tx body type or reject non-zero values in SUBUTXO), since the economic rationale for sub-tx donations is unclear and the field is the sole source of the imbalance.

### Proof of Concept
1. Construct a top-level Dijkstra tx whose own inputs/outputs/fees satisfy `validateValueNotConservedUTxO`.
2. Embed a sub-tx with:
   - `dstbrSpendInputs` = some UTxO entry worth X ADA
   - `dstbrOutputs` = one output worth X ADA (balanced)
   - `dstbrTreasuryDonation` = 1,000,000,000 ADA
3. Submit the top-level tx. All predicate checks pass (no conservation check in SUBUTXO, `validateTreasuryValue` only checks `currentTreasuryValue`).
4. Observe `utxosDonation` has increased by 1,000,000,000 ADA while the UTxO total is unchanged.
5. After the epoch boundary, the treasury balance has increased by 1,000,000,000 ADA beyond what any legitimate donation or expansion accounts for.
