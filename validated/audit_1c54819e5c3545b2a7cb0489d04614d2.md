### Title
Proposer-Supplied `builder` Address Accepted Without Validation — Wrong `sequencer_address` Committed to Block Execution (`crates/apollo_consensus_orchestrator/src/utils.rs`)

### Summary

`ProposalInit.builder` is a proposer-supplied wire field that is passed directly as `sequencer_address` into every block's `BlockInfo` by the validator without any check against a locally-trusted expected value. A malicious-but-legitimately-selected proposer can set `builder` to an arbitrary address, causing all fee payments in that block to be credited to the attacker-controlled address. Every other validator node accepts the proposal because `is_proposal_init_valid` never inspects `builder`, and both sides execute with the same `builder` value, so the `ProposalCommitment` still matches.

### Finding Description

`ProposalInit` carries a `builder` field ("Address of the one who builds/sequences the block"): [1](#0-0) 

When the proposer builds a block, `builder` is set from the node's own static config: [2](#0-1) 

The validator receives `ProposalInit` over the wire and converts it to `starknet_api::block::BlockInfo` via `convert_to_sn_api_block_info`, which maps `init.builder` directly to `sequencer_address`: [3](#0-2) 

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas price fields, and `fee_proposal_fri`. It never checks `builder`: [4](#0-3) 

`ProposalInitValidation` — the struct that carries all locally-trusted reference values — has no `builder` field at all: [5](#0-4) 

Because both proposer and validator derive `sequencer_address` from the same `init.builder` wire value, they execute the block identically and produce the same `partial_block_hash`. The `ProposalCommitment` comparison therefore passes: [6](#0-5) 

The committed block carries the attacker-chosen `sequencer_address` in its `BlockInfo`, and all fee transfers during execution credit that address.

### Impact Explanation

`sequencer_address` is the fee-recipient address used throughout block execution. Every transaction fee paid in the block is transferred to this address. A malicious proposer who sets `builder` to an attacker-controlled address redirects the entire block's fee revenue. The committed block header also records the wrong sequencer address, corrupting the authoritative on-chain state. This matches the allowed impact: **"Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact" — Critical.**

### Likelihood Explanation

Any validator node that is legitimately selected as proposer for a round can execute this attack with a single-field modification to `ProposalInit`. No external precondition (allowance, prior approval, failed transaction) is required — the attacker only needs to be the current-round proposer, which is a normal protocol role. The attack is silent: no error is raised, no commitment mismatch occurs, and the block is finalized normally.

### Recommendation

Add `expected_builder: ContractAddress` to `ProposalInitValidation`, populated from `self.config.static_config.builder_address` (or from the committee once builder addresses are committee-derived), and reject any `ProposalInit` whose `builder` field does not match:

```rust
// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("builder mismatch: expected={:?}, got={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder),
    ));
}
```

This mirrors the existing pattern used for `l1_da_mode`, `l2_gas_price_fri`, and `starknet_version`.

### Proof of Concept

1. Attacker controls validator node `V_attacker` and is selected as proposer for height `H`, round `R`.
2. `V_attacker` calls `initiate_build` but overrides `builder` in the constructed `ProposalInit` to `attacker_wallet_address` instead of `self.config.static_config.builder_address`.
3. `V_attacker` streams the modified `ProposalInit` followed by transaction batches and `ProposalFin`.
4. Every honest validator calls `validate_proposal` → `is_proposal_init_valid`: `builder` is never checked; validation passes.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which sets `sequencer_address = attacker_wallet_address`.
6. The batcher executes all transactions with `sequencer_address = attacker_wallet_address`; all fee transfers credit the attacker.
7. Both proposer and validators compute the same `partial_block_hash` (both used `attacker_wallet_address`), so `ProposalCommitment` matches and consensus finalizes the block.
8. The committed block permanently records `sequencer_address = attacker_wallet_address` and the attacker has received all block fees. [7](#0-6) [8](#0-7)

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
