Audit Report

## Title
rsETH Transfers Unrestricted When RSETH Contract Is Paused, Enabling Sale of Non-Redeemable Tokens - (File: contracts/RSETH.sol)

## Summary
`RSETH._transfer` does not enforce `whenNotPaused`, so rsETH tokens remain freely transferable even when the RSETH contract itself is paused. Because `burnFrom` (used by the withdrawal path) is correctly guarded by `whenNotPaused`, a holder can sell rsETH on a DEX after a pause while buyers receive tokens they cannot redeem until the protocol is unpaused. A second, automated trigger path exists: `LRTOracle._updateRsETHPrice()` pauses `LRTDepositPool` and `LRTWithdrawalManager` but never pauses the RSETH token, creating an asymmetry with `LRTConfig.pauseAll()` that leaves transfers unrestricted while both withdrawal entry points are blocked.

## Finding Description

**Root cause — missing pause check in `_transfer`:**

`RSETH.mint` and `RSETH.burnFrom` both carry `whenNotPaused`: [1](#0-0) 

The overridden `_transfer` only enforces the per-address block list: [2](#0-1) 

No `whenNotPaused` check is present. `transfer()` and `transferFrom()` therefore succeed regardless of the RSETH pause state.

**Automated trigger — oracle circuit breaker:**

`LRTOracle._updateRsETHPrice()` fires automatically when the price drops beyond `pricePercentageLimit`. It pauses `LRTDepositPool` and `LRTWithdrawalManager` and pauses itself, but never touches the RSETH token contract: [3](#0-2) 

In contrast, `LRTConfig.pauseAll()` explicitly pauses RSETH: [4](#0-3) 

**Withdrawal entry points are blocked during any pause:**

Both `initiateWithdrawal` and `instantWithdrawal` carry `whenNotPaused`: [5](#0-4) [6](#0-5) 

**Exploit flow:**
1. A large market move causes `updateRSETHPrice()` (public, callable by anyone) to fire the circuit breaker, pausing `LRTDepositPool` and `LRTWithdrawalManager`.
2. The malicious rsETH holder calls `rsETH.transfer(dex, amount)` or routes through a DEX — succeeds because `_transfer` has no pause check.
3. A victim buys rsETH on the DEX.
4. Victim calls `initiateWithdrawal` or `instantWithdrawal` — both revert with `Paused`.
5. Victim's rsETH is locked in a non-redeemable state until an admin unpauses `LRTWithdrawalManager`. No on-chain indication of when (or whether) that will occur.

The same flow applies when RSETH itself is paused (e.g., via `pauseAll()`): `burnFrom` is blocked so redemption is impossible, yet `_transfer` still succeeds, allowing the holder to sell.

## Impact Explanation

Buyers acquire rsETH that cannot be converted back to ETH or LSTs for the duration of the pause. Their underlying economic value is temporarily inaccessible. This is a concrete **Medium — Temporary freezing of funds**. The freeze is bounded by the pause duration but has no on-chain upper bound; there is no forced unpause mechanism.

## Likelihood Explanation

`updateRSETHPrice()` is a public, permissionless function. Any sufficiently large market move triggers the circuit breaker without admin involvement. A sophisticated rsETH holder who monitors oracle state can sell rsETH on a DEX immediately after the pause takes effect (no front-running required — transfers remain open throughout the pause). No privileged key compromise or governance capture is needed.

## Recommendation

1. Add `whenNotPaused` to `_transfer` in `RSETH.sol` (or override `_beforeTokenTransfer` to enforce the pause check). Ensure `LRTWithdrawalManager` is added to `isPermanentlyExempt` so that `initiateWithdrawal`'s `safeTransferFrom` call can still receive rsETH even during a pause, consistent with the existing exemption mechanism.

2. Update `LRTOracle._updateRsETHPrice()` to also pause the RSETH token contract when the price-drop circuit breaker fires, making it symmetric with `LRTConfig.pauseAll()`.

## Proof of Concept

```solidity
// Foundry fork test outline
function test_transferDuringPause() public {
    // 1. Trigger oracle circuit breaker (simulate price drop beyond threshold)
    //    updateRSETHPrice() is public — call it after manipulating oracle prices
    lrtOracle.updateRSETHPrice();
    assertTrue(lrtWithdrawalManager.paused());
    assertFalse(rsETH.paused()); // RSETH is NOT paused by oracle path

    // 2. Malicious holder transfers rsETH to victim (succeeds — no pause check)
    vm.prank(maliciousHolder);
    rsETH.transfer(victim, amount); // succeeds

    // 3. Victim cannot redeem
    vm.prank(victim);
    vm.expectRevert("Pausable: paused");
    lrtWithdrawalManager.initiateWithdrawal(asset, amount, "");

    vm.prank(victim);
    vm.expectRevert("Pausable: paused");
    lrtWithdrawalManager.instantWithdrawal(asset, amount, "");

    // 4. Repeat with RSETH itself paused (via pauseAll)
    vm.prank(pauser);
    lrtConfig.pauseAll();
    assertTrue(rsETH.paused());

    vm.prank(maliciousHolder2);
    rsETH.transfer(victim2, amount); // still succeeds — _transfer has no whenNotPaused
}
```

    function pauseAll() external onlyRole(LRTConstants.PAUSER_ROLE) {
        IPausable lrtDepositPool = IPausable(getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable lrtWithdrawalManager = IPausable(getContract(LRTConstants.LRT_WITHDRAW_MANAGER));
        IPausable lrtOracle = IPausable(getContract(LRTConstants.LRT_ORACLE));
        IPausable rsETHContract = IPausable(rsETH);

        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!lrtWithdrawalManager.paused()) lrtWithdrawalManager.pause();
        if (!lrtOracle.paused()) lrtOracle.pause();
        if (!rsETHContract.paused()) rsETHContract.pause();

        address[] memory nodeDelegatorQueue = ILRTDepositPool(address(lrtDepositPool)).getNodeDelegatorQueue();
        uint256 nodeDelegatorCount = nodeDelegatorQueue.length;

        for (uint256 i = 0; i < nodeDelegatorCount;) {
            IPausable nodeDelegator = IPausable(nodeDelegatorQueue[i]);
            if (!nodeDelegator.paused()) nodeDelegator.pause();
            unchecked {
                ++i;
            }
        }

        emit PausedAll(msg.sender);
    }
