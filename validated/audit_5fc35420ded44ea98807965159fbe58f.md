### Title
Unvalidated `builder` Field in `ProposalInit` Allows Malicious Proposer to Redirect Block Fees - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

### Summary

`is_proposal_init_valid` validates many fields of a received `ProposalInit` (height, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`) but never checks the `builder` field. The `builder` address is passed directly to `convert_to_sn_api_block_info` and becomes the sequencer address used by the blockifier for fee collection. A malicious proposer can set `builder` to an attacker-controlled address; validators accept the proposal without objection, execute the block with the attacker's address as sequencer, and commit a block in which all transaction fees flow to the attacker.

### Finding Description

`ProposalInit` carries a `builder: ContractAddress` field that identifies the address that builds/sequences the block and receives transaction fees. [1](#0-0) 

When the legitimate proposer builds a block, `builder` is set from the node's own configuration: [2](#0-1) 

The validator receives the proposer's `ProposalInit` over the network and calls `is_proposal_init_valid`. That function checks height, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`: [3](#0-2) 

The `ProposalInitValidation` struct — the reference object the validator uses — contains no `builder` or `expected_builder` field: [4](#0-3) 

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which converts the proposer-supplied `init.builder` into the sequencer address that the blockifier uses for fee collection: [5](#0-4) 

Because the validator re-executes the block using the proposer's own `init` (including the attacker-chosen `builder`), both sides compute the same `ProposalCommitment`, the `ProposalFinMismatch` guard does not fire, and the block is committed with the attacker's address as sequencer. [6](#0-5) 

The consensus-layer proposer check in `manager.rs` only verifies that `init.proposer` matches the expected committee member; it does not constrain `init.builder`: [7](#0-6) 

### Impact Explanation

The `builder` address is the sequencer address for the block. All transaction fees for every transaction in the block are credited to this address. A malicious validator-turn proposer can set `builder` to any address they control, siphoning the entire fee revenue of one block per proposal round they win. Because the validator re-executes with the attacker's `builder`, the resulting `ProposalCommitment` is identical on both sides and consensus reaches agreement on a block with wrong fee accounting. This matches the impact category: **Critical — Incorrect fee/balance effect with economic impact**.

### Likelihood Explanation

Any node that wins a proposal round can exploit this. In a permissioned validator set the attacker must be a registered validator, but no external capability beyond being a proposer is required. The attack is silent (no error, no log anomaly on the validator side) and repeatable every time the attacker is selected as proposer.

### Recommendation

Add an `expected_builder: ContractAddress` field to `ProposalInitValidation` (populated from the node's own `builder_address` config) and add a check inside `is_proposal_init_valid`:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

This mirrors the existing pattern used for `height`, `l1_da_mode`, and `l2_gas_price_fri` at line 312–321 of `validate_proposal.rs`. [8](#0-7) 

### Proof of Concept

1. Attacker controls a validator node that wins a proposal round.
2. In `initiate_build`, instead of using `args.builder_address` (the node's legitimate address), the attacker patches the code to set `builder: attacker_wallet_address`.
3. The malicious `ProposalInit` is streamed to all validators.
4. Each validator calls `is_proposal_init_valid`; all checked fields (height, gas prices, version, etc.) match, so validation passes.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)` with `init.builder = attacker_wallet_address`, making `attacker_wallet_address` the sequencer address for this block.
6. The blockifier executes all transactions and credits fees to `attacker_wallet_address`.
7. `proposal_commitment_from(partial_block_hash, fee_proposal_fri)` produces the same hash on both proposer and validator sides (both used the same `init`), so `ProposalFinMismatch` is not triggered.
8. Consensus commits the block; `attacker_wallet_address` receives all transaction fees. [2](#0-1) [9](#0-8) [10](#0-9)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
