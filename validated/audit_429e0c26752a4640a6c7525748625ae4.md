### Title
Permissionless `updateRSETHPrice()` Allows Any Caller to Trigger Protocol-Wide Pause and Freeze Deposits/Withdrawals - (File: `contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a public, permissionless function. Its internal logic contains a downside-protection mechanism that automatically pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself whenever the computed rsETH price falls below `highestRsethPrice` by more than `pricePercentageLimit`. Because any external caller can invoke this function, an attacker can trigger a permanent, protocol-wide freeze at any moment the price condition is satisfied — with zero cost.

---

### Finding Description

`updateRSETHPrice()` carries only a `whenNotPaused` guard:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [1](#0-0) 

Inside `_updateRsETHPrice()`, the downside-protection block is:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
``` [2](#0-1) 

The condition is evaluated as:

```solidity
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
``` [3](#0-2) 

`highestRsethPrice` is a monotonically increasing watermark — it is updated only when `newRsETHPrice > highestRsethPrice`: [4](#0-3) 

`newRsETHPrice` is derived from `_getTotalEthInProtocol()` (which reads live Chainlink/protocol oracle prices and on-chain balances) divided by `rsETH.totalSupply()`. Any real-world event that reduces the protocol's ETH-denominated TVL — validator slashing, a supported LST depeg, or a temporary oracle price dip — can push `newRsETHPrice` below the threshold. [5](#0-4) 

When the pause fires, `rsETHPrice` is **not updated** (the function returns early), so the stale price persists. Unpausing requires `onlyLRTAdmin`:

```solidity
function unpause() external whenPaused onlyLRTAdmin {
    paused = false;
    ...
}
``` [6](#0-5) 

`LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()` are both gated by `PAUSER_ROLE`, which `LRTOracle` must hold for the auto-pause to succeed. Once triggered, all user-facing entry points on both contracts revert until an admin manually calls `unpause()` on each. [7](#0-6) [8](#0-7) 

**Attack path:**

1. The protocol accumulates staking rewards; `highestRsethPrice` is updated to, say, `1.05 ETH`.
2. A validator slashing event (or a brief LST oracle dip) causes `newRsETHPrice` to fall to `1.03 ETH` — a `~1.9%` drop.
3. If `pricePercentageLimit` is set to `1%` (1e16), the condition `diff > pricePercentageLimit * highestRsethPrice` is satisfied.
4. The attacker calls `updateRSETHPrice()` at zero cost.
5. `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` are all paused atomically.
6. All deposits (`depositETH`, `depositAsset`) and all withdrawals (`initiateWithdrawal`, `completeWithdrawal`, `instantWithdrawal`) revert until an admin manually unpauses each contract.

The attacker incurs only gas cost. No capital is required.

---

### Impact Explanation

All user-facing deposit and withdrawal functions on `LRTDepositPool` and `LRTWithdrawalManager` are blocked for the duration of the pause. Users with pending withdrawal requests cannot complete them. New depositors cannot enter. This constitutes a **temporary freezing of funds** — matching the "Medium: Temporary freezing of funds" impact category. If the admin response is slow (e.g., key-holder unavailability, multisig delay), the freeze can extend significantly.

---

### Likelihood Explanation

The condition is realistic: LST oracle prices fluctuate, EigenLayer slashing events reduce TVL, and `highestRsethPrice` is a strict all-time-high watermark that never decreases. Any attacker monitoring on-chain state can call `updateRSETHPrice()` the moment the condition is met, at negligible cost. The tighter `pricePercentageLimit` is set, the easier the condition is to satisfy.

---

### Recommendation

1. **Restrict `updateRSETHPrice()`** to authorized callers (e.g., `onlyLRTManager` or a keeper role), mirroring the existing `updateRSETHPriceAsManager()` pattern. The auto-pause path should not be reachable by arbitrary external callers.
2. **Separate the pause trigger from the price update**: require an explicit, role-gated call to pause rather than embedding it in a public price-update function.
3. **Add a cooldown or confirmation window** before the pause fires, so a transient oracle dip does not immediately freeze the protocol.

---

### Proof of Concept

```
Preconditions:
  highestRsethPrice = 1.05e18  (set after prior reward accrual)
  pricePercentageLimit = 1e16  (1%)
  Current totalETHInProtocol reflects a slashing event → newRsETHPrice = 1.03e18

Attack:
  attacker.call(lrtOracle.updateRSETHPrice())

  Inside _updateRsETHPrice():
    diff = 1.05e18 - 1.03e18 = 0.02e18
    threshold = pricePercentageLimit.mulWad(highestRsethPrice)
              = 1e16 * 1.05e18 / 1e18 = 1.05e16
    isPriceDecreaseOffLimit = (0.02e18 > 1.05e16) → true

    lrtDepositPool.pause()       ← LRTDepositPool frozen
    withdrawalManager.pause()    ← LRTWithdrawalManager frozen
    _pause()                     ← LRTOracle frozen
    return                       ← rsETHPrice NOT updated

Result:
  All deposits and withdrawals revert until admin manually unpauses.
  Cost to attacker: gas only.
```

    function _updateRsETHPrice() internal {
        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;
        }

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

        // only take fee if TVL increased and protocol is not paused
        uint256 protocolFeeInETH = 0;
        if (!protocolPaused && totalETHInProtocol > previousTVL) {
            uint256 rewardAmount = totalETHInProtocol - previousTVL;
            protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
        }

        // compute new rsETH price based on total ETH minus fee
        uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

```
