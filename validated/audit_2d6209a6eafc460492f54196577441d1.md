Audit Report

## Title
Blacklisted Token Recipient Causes Permanent Freezing of Unclaimed Yield — (`contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

## Summary

`MerkleDistributor.claim()` unconditionally transfers yield to the `account` address encoded in the merkle leaf with no alternative-recipient or rescue mechanism. If the configured token is a blacklist-capable ERC20 (e.g., USDC) and `account` is blacklisted after the merkle root is set, every future `claim()` invocation for that account reverts at the `safeTransfer` call, permanently locking the allocated yield inside the contract with no on-chain recovery path.

## Finding Description

The `claim()` function at [1](#0-0)  always sends tokens to `account` — the address baked into the merkle leaf — regardless of who calls the function. If the token contract (e.g., USDC) has blacklisted `account`, `safeTransfer` reverts on every call. The merkle leaf encodes `account` as the recipient permanently [2](#0-1) , so there is no way to redirect the allocation to a different address.

The contract exposes no `rescueTokens`, `emergencyWithdraw`, or alternative-recipient mechanism. The only admin functions are `setMerkleRoot`, `setProtocolTreasury`, `setToken`, `setFeeInBPS`, `pause`, and `unpause` [3](#0-2)  — none of which can recover or redirect a stuck allocation. `setToken()` only affects future distributions and does not unblock the frozen yield for the blacklisted account. [4](#0-3) 

The contract is explicitly described as a **generic** distributor whose `token` field is configurable post-deployment, making USDC or USDT a realistic and supported token choice. [5](#0-4) 

## Impact Explanation

The yield allocated to a blacklisted `account` is permanently locked inside the `MerkleDistributor` contract. No admin function can recover or redirect it. This concretely matches the allowed impact: **Medium — Permanent freezing of unclaimed yield**.

## Likelihood Explanation

- The `token` field is configurable via `setToken()` and can be set to any ERC20, including USDC or USDT. [4](#0-3) 
- USDC/USDT blacklisting is an operational reality (OFAC sanctions, exchange compliance). Any user whose address is sanctioned after a merkle root is published triggers this path.
- The exploit requires no privileged access by the attacker: `setMerkleRoot` (owner, normal operation) → `token.blacklist(account)` (token issuer, independent external action) → `claim(...)` (anyone, public function). The SECURITY.md exclusion for "privileged addresses" refers to this protocol's privileged roles, not external token issuers. [6](#0-5) 
- Likelihood is low-to-medium: blacklisting is rare but operationally real and non-hypothetical.

## Recommendation

1. **Add an alternative-recipient parameter**: Allow the claimant (verified via `msg.sender == account`) to specify a different destination address, enabling a blacklisted account to redirect yield to a clean address.
2. **Add a token rescue function**: An `onlyOwner` `rescueTokens(address token, address to, uint256 amount)` function would allow recovery of stuck tokens as a last resort.
3. **Document token compatibility**: If the distributor is intended only for non-blacklistable tokens, enforce this in `setToken()` or document it clearly.

## Proof of Concept

```solidity
// Fork mainnet. Deploy MerkleDistributor with USDC as token.
// 1. Owner calls setMerkleRoot(root)
//    root encodes leaf: keccak256(abi.encodePacked(uint256(1), VICTIM, 1000e6))
// 2. USDC blacklister calls USDC.blacklist(VICTIM)
// 3. Anyone calls distributor.claim(1, VICTIM, 1000e6, proof)
//    → safeTransfer(VICTIM, ...) reverts: "Blacklistable: account is blacklisted"
// 4. No rescueTokens() exists on MerkleDistributor
//    → 1000e6 USDC permanently stuck in the contract

function testBlacklistedAccountFreezesYield() public {
    address victim = address(0xBEEF);
    uint256 amount = 1000e6;

    bytes32 leaf = keccak256(abi.encodePacked(uint256(1), victim, amount));
    bytes32 root = leaf;
    bytes32[] memory proof = new bytes32[](0);

    vm.prank(owner);
    distributor.setMerkleRoot(root);

    vm.prank(usdcBlacklister);
    IFiatToken(USDC).blacklist(victim);

    vm.expectRevert("Blacklistable: account is blacklisted");
    distributor.claim(1, victim, amount, proof);

    // Confirm no recovery path: no rescueTokens() selector exists
    assertEq(IERC20(USDC).balanceOf(address(distributor)), amount);
}
```

    /// @dev Set the merkle root for the given index.
    /// @dev only called by the owner.
    /// @param _merkleRootToSet The merkle root to set.
    function setMerkleRoot(bytes32 _merkleRootToSet) external onlyOwner {
        if (_merkleRootToSet == bytes32(0)) {
            revert ZeroValueProvided();
        }

        currentMerkleRoot = _merkleRootToSet;

        currentMerkleRootIndex++;
        currentIndex++;

        emit MerkleRootSet(currentMerkleRootIndex, currentMerkleRoot);
    }

    /// @dev Set the protocol treasury address.
    /// @dev only called by the owner.
    /// @param _protocolTreasury The address of the protocol treasury.
    function setProtocolTreasury(address _protocolTreasury) external onlyOwner {
        if (_protocolTreasury == address(0)) {
            revert ZeroValueProvided();
        }

        protocolTreasury = _protocolTreasury;

        emit ProtocolTreasuryUpdated(_protocolTreasury);
    }

    /// @dev Set the token address.
    /// @dev only called by the owner.
    /// @param _token The address of the token.
    function setToken(address _token) external onlyOwner {
        if (_token == address(0)) {
            revert ZeroValueProvided();
        }

        token = _token;

        emit TokenUpdated(_token);
    }

    /// @dev Set the fee in BPS.

            revert InvalidFeeInBPS();

    }
