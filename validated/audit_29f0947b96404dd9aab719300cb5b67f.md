### Title
Unvalidated `ProposalInit.builder` (sequencer/fee-recipient address) allows a malicious proposer to redirect all block fees to an attacker-controlled address - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

---

### Summary

`is_proposal_init_valid` validates many `ProposalInit` fields but never checks `init.builder`. Because `init.builder` is used verbatim as `sequencer_address` in `BlockInfo`, and `sequencer_address` is the fee-transfer recipient for every transaction in the block, a malicious proposer can set `builder` to any address and redirect all block fees to themselves. Honest validators accept the proposal because the `proposal_commitment` (which does bind `sequencer_address` through `partial_block_hash`) is self-consistent: both sides compute the hash using the proposer-supplied `builder`, so the commitment matches even with the wrong address.

---

### Finding Description

**Root cause — missing check in `is_proposal_init_valid`**

`is_proposal_init_valid` enforces `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas-price fields (within margin), and `fee_proposal_fri` (presence + bounds). It does **not** check `init.builder`. [1](#0-0) 

**`init.builder` becomes the fee-transfer recipient**

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in `BlockInfo`: [2](#0-1) 

`execute_fee_transfer` uses `block_context.block_info.sequencer_address` as the sole recipient of every fee transfer: [3](#0-2) 

**Why the commitment does not protect against this**

`PartialBlockHashComponents` includes `sequencer` (derived from `block_info.sequencer_address`): [4](#0-3) 

`calculate_block_hash` chains `sequencer.0` into the Poseidon hash: [5](#0-4) 

`proposal_commitment_from` wraps this `partial_block_hash`: [6](#0-5) 

Because the validator feeds the proposer-supplied `init.builder` into the batcher (via `convert_to_sn_api_block_info`), the batcher computes a `partial_block_hash` that already encodes the attacker's address. The resulting `batcher_block_commitment` matches `received_fin.proposal_commitment`, so the `ProposalFinMismatch` guard passes: [7](#0-6) 

The commitment is self-consistent with the wrong address; no guard catches it.

**The proposer sets `builder` from static config — no committee check yet**

The proposer sets `builder` from `self.config.static_config.builder_address` with an explicit TODO to derive it from the committee: [8](#0-7) 

Until that TODO is resolved and the validator enforces the expected value, any proposer can supply an arbitrary `builder`.

---

### Impact Explanation

Every transaction fee in the block is transferred to `sequencer_address` = `init.builder`. A malicious proposer sets `builder` to an attacker-controlled address; all STRK fees from every transaction in that block flow to the attacker instead of the legitimate sequencer. The block is also committed to state sync with the wrong `sequencer` field in `BlockHeaderWithoutHash`: [9](#0-8) 

This is a **Critical** economic impact: incorrect fee/balance effect — all block fees are permanently misdirected.

---

### Likelihood Explanation

Requires a malicious validator to be selected as proposer for a round. In the current Tendermint-based multi-validator design this is a normal rotation. The attack requires no special privilege beyond being a committee member and being scheduled as proposer. The manipulation is a single field change in `ProposalInit` with no cryptographic barrier.

---

### Recommendation

Add a check inside `is_proposal_init_valid` that `init_proposed.builder` equals the locally-known expected builder address. Once the committee-derived builder address is available (per the existing TODO), derive the expected value from the committee. Until then, use the node's own `builder_address` config as the reference, mirroring the pattern already used for `l2_gas_price_fri`, `l1_da_mode`, and `height`:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

Add `expected_builder: ContractAddress` to `ProposalInitValidation` and populate it from `self.config.static_config.builder_address` at the call site in `validate_current_round_proposal`.

---

### Proof of Concept

1. A malicious validator is scheduled as proposer for height H, round R.
2. In `initiate_build`, it overrides `builder` in the constructed `ProposalInit` to `ATTACKER_ADDRESS` instead of its configured `builder_address`.
3. It streams `ProposalPart::Init(init)` with `init.builder = ATTACKER_ADDRESS` to all validators.
4. Each validator calls `is_proposal_init_valid` — no check on `builder` — and proceeds.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: ATTACKER_ADDRESS, … }`.
6. The batcher executes all transactions; `execute_fee_transfer` transfers every fee to `ATTACKER_ADDRESS`.
7. `PartialBlockHashComponents::new` encodes `ATTACKER_ADDRESS` as `sequencer`; `partial_block_hash` and `proposal_commitment` are computed consistently on both sides.
8. `built_block == received_fin.proposal_commitment` passes; the proposal is accepted and committed.
9. All STRK fees from block H are permanently held by `ATTACKER_ADDRESS`; the block header stored in state sync records `ATTACKER_ADDRESS` as the sequencer.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
