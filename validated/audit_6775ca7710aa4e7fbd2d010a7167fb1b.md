Audit Report

## Title
Stale Pre-Pause `rsETHPrice` Propagated to L2 Pool After L1 Oracle Auto-Pause Causes Depositors to Receive Fewer rsETH Tokens - (File: contracts/cross-chain/RSETHRateReceiver.sol)

## Summary

When `LRTOracle._updateRsETHPrice()` triggers its downside-protection auto-pause, it returns early without writing the new (lower) price, leaving `rsETHPrice` frozen at the pre-drop (inflated) value. Because `CrossChainRateProvider.updateRate()` has no access control and no oracle-pause guard, any caller can immediately push this stale inflated rate to L2. `RSETHRateReceiver.getRate()` returns it unconditionally, and `RSETHPoolV2.deposit()` uses it to compute the rsETH mint amount, causing depositors to receive fewer rsETH tokens than the true rate entitles them to.

## Finding Description

**Root cause — `rsETHPrice` frozen at inflated value on auto-pause**

In `LRTOracle._updateRsETHPrice()`, when the computed `newRsETHPrice` drops beyond `pricePercentageLimit`, the function calls `_pause()` and returns without ever writing `rsETHPrice = newRsETHPrice`: [1](#0-0) 

`rsETHPrice` is a plain public state variable with no read-time pause guard: [2](#0-1) 

**Rate provider — no pause check before reading `rsETHPrice`**

`RSETHRateProvider.getLatestRate()` reads `rsETHPrice` directly: [3](#0-2) 

**Rate provider — `updateRate()` is permissionless**

`CrossChainRateProvider.updateRate()` has only a `nonReentrant` guard; any EOA can call it and bridge the stale rate to L2: [4](#0-3) 

**L2 receiver — `getRate()` returns cached rate unconditionally**

`CrossChainRateReceiver` stores `lastUpdated` but never uses it in `getRate()`: [5](#0-4) 

`RSETHRateReceiver` adds no pause mechanism on top of this: [6](#0-5) 

**Pool — uses the stale rate to compute mint amount**

`RSETHPoolV2.deposit()` calls `viewSwapRsETHAmountAndFee()`, which divides by the stale (inflated) rate: [7](#0-6) 

The pool's own `paused` flag is independent of the L1 oracle state and is not automatically set when the L1 oracle pauses. [8](#0-7) 

## Impact Explanation

When the L1 oracle auto-pauses, `rsETHPrice` is frozen at the pre-drop value (e.g., 1.0 ETH/rsETH) while the true rate has fallen (e.g., 0.95 ETH/rsETH). A user depositing 1 ETH on L2 receives `1e18 * 1e18 / 1.0e18 = 1.0 rsETH` instead of the correct `1e18 * 1e18 / 0.95e18 ≈ 1.053 rsETH`. The protocol retains the full ETH deposit but the depositor is shortchanged on rsETH. This matches: **Low — Contract fails to deliver promised returns, but doesn't lose value.**

## Likelihood Explanation

The auto-pause path is a normal operational event triggered whenever the price moves beyond `pricePercentageLimit`. The window between L1 oracle auto-pause and a manual L2 pool pause is non-zero and can span multiple blocks. During this window, any unprivileged user can call `RSETHRateProvider.updateRate{value: lzFee}()` to actively re-push the stale rate to L2, and any depositor calling `RSETHPoolV2.deposit()` will be shortchanged. No special privileges or victim mistakes are required.

## Recommendation

1. In `RSETHRateProvider.getLatestRate()` (and `RSETHMultiChainRateProvider.getLatestRate()`), check `ILRTOracle(rsETHPriceOracle).paused()` and revert if the oracle is paused, preventing stale rate propagation at the source.
2. Add an access control modifier (e.g., `onlyOwner`) to `CrossChainRateProvider.updateRate()` so the rate cannot be permissionlessly pushed to L2.
3. Add a staleness check in `CrossChainRateReceiver.getRate()` using the stored `lastUpdated` timestamp, reverting if the rate is older than a configurable threshold.
4. Optionally, add a `pause()`/`unpause()` mechanism to `CrossChainRateReceiver` with a `whenNotPaused` guard on `getRate()` and `lzReceive()`.

## Proof of Concept

```solidity
// Foundry fork test
function test_stalePriceAfterAutoPause() public {
    // 1. Trigger auto-pause on L1 by simulating a price drop beyond pricePercentageLimit
    //    (e.g., manipulate underlying asset price oracle to drop > limit)
    vm.prank(address(lrtOracle)); // simulate internal call
    lrtOracle.updateRSETHPrice(); // triggers _pause() + return, rsETHPrice NOT updated
    assertTrue(lrtOracle.paused());

    uint256 stalePrice = lrtOracle.rsETHPrice(); // pre-drop inflated value

    // 2. Anyone pushes the stale rate to L2 (permissionless)
    uint256 lzFee = rsETHRateProvider.estimateFees();
    vm.deal(attacker, lzFee);
    vm.prank(attacker);
    rsETHRateProvider.updateRate{value: lzFee}(); // no revert

    // 3. L2 receiver holds the stale rate
    assertEq(rsETHRateReceiver.getRate(), stalePrice);

    // 4. User deposits on L2 at the stale (inflated) rate
    uint256 trueRate = /* current true rate after drop */;
    uint256 expectedAtTrueRate = 1 ether * 1e18 / trueRate;
    uint256 rsETHBefore = wrsETH.balanceOf(user);
    vm.prank(user);
    rsETHPoolV2.deposit{value: 1 ether}("ref");
    uint256 rsETHAfter = wrsETH.balanceOf(user);

    // User receives fewer rsETH than the true rate would give
    assertLt(rsETHAfter - rsETHBefore, expectedAtTrueRate);
}
```
