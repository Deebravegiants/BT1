### Title
Cairo 0 Class Declaration Skips State-Number Check in `ExecutionStateReader::get_compiled_class`, Allowing Undeclared Class Execution at Historical Blocks - (`crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`ExecutionStateReader::get_compiled_class` enforces a declaration check against `self.state_number` for Cairo 1 classes but explicitly skips the same check for Cairo 0 (deprecated) classes when the `class_manager_handle` is set. An identical asymmetry exists in `StateReaderAndContractManager::get_compiled_from_class_manager`. Because the class manager is a global, state-agnostic cache, any Cairo 0 class that was ever declared is retrievable regardless of the queried block. RPC execution calls (`starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_traceTransaction`) at a historical block before a Cairo 0 class was declared will silently succeed using the undeclared class instead of returning `UndeclaredClassHash`, producing authoritative-looking wrong values.

---

### Finding Description

In `ExecutionStateReader::get_compiled_class`, when `class_manager_handle` is `Some`, the function dispatches on the returned `ContractClass` variant:

```
ContractClass::V1(casm_contract_class) => {
    let is_declared = is_contract_class_declared(
        &self.storage_reader.begin_ro_txn()...,
        &class_hash,
        self.state_number,   // ← checked against the queried block
    )?;
    if is_declared { Ok(...) } else { Err(StateError::UndeclaredClassHash(class_hash)) }
}
// TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.
ContractClass::V0(deprecated_contract_class) => {
    Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    // ← NO declaration check; class returned unconditionally
}
``` [1](#0-0) 

The Cairo 1 arm calls `is_contract_class_declared` and rejects the class if it was not declared at `self.state_number`. The Cairo 0 arm returns the class unconditionally. The TODO comment is an explicit acknowledgement that the check is missing and deferred.

The same asymmetry appears in `StateReaderAndContractManager::get_compiled_from_class_manager`, which is the batcher-side path. When a Cairo 0 class is found in the contract-class cache, the code takes the empty match arm and skips `is_declared`:

```rust
match &runnable_class {
    RunnableCompiledClass::V0(_) => {}   // ← no is_declared call
    _ => {
        // The Cairo1 class is cached; verify it is declared,
        // since existence in the cache does not guarantee that
        // (it might contain a declared class from a reverted block, for example).
        if !self.state_reader.is_declared(class_hash)? {
            return Err(StateError::UndeclaredClassHash(class_hash));
        }
    }
}
``` [2](#0-1) 

The comment explicitly names the reorg scenario as the motivation for the Cairo 1 check, yet Cairo 0 is left unguarded.

The class manager is a global cache keyed only by `ClassHash`, not by block number or state number: [3](#0-2) 

Once a Cairo 0 class is declared at any block, `class_manager_client.get_executable(class_hash)` returns it for every subsequent call, regardless of the `state_number` the reader was constructed with.

---

### Impact Explanation

**Primary (High — RPC):** Any caller of `starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_traceTransaction` can supply a `block_id` that precedes the block at which a Cairo 0 class was declared. The RPC server constructs an `ExecutionStateReader` with `state_number` set to that historical block, but `get_compiled_class` returns the class anyway. The execution proceeds as if the class were declared, producing a wrong return value, wrong fee estimate, or wrong trace instead of the correct `CLASS_HASH_NOT_FOUND` / `UndeclaredClassHash` error. This is an authoritative-looking wrong value from an RPC execution endpoint.

**Secondary (Critical — batcher, reorg):** After a chain reorganisation, the contract-class cache may contain Cairo 0 classes declared in reverted blocks. `StateReaderAndContractManager` will serve those classes to the blockifier without checking `is_declared`, potentially executing transactions against undeclared classes and producing wrong state diffs, receipts, and storage roots.

---

### Likelihood Explanation

The RPC path is reachable by any unprivileged user with no special setup: submit a `starknet_call` at any block number before a known Cairo 0 class declaration. The class manager is populated as a side-effect of normal sequencer operation. No mempool manipulation, front-running, or privileged access is required. The batcher path requires a reorg, which is a lower-probability event but is explicitly anticipated by the existing Cairo 1 guard.

---

### Recommendation

Apply the same `is_contract_class_declared` / `is_declared` guard to the Cairo 0 branch in both locations:

1. **`crates/apollo_rpc_execution/src/state_reader.rs`** — in the `ContractClass::V0` arm, call `is_contract_class_declared` against `self.state_number` and return `Err(StateError::UndeclaredClassHash(class_hash))` if the check fails. The existing TODO comment already identifies this as the intended fix once `get_class_definition_block_number` is corrected.

2. **`crates/blockifier/src/state/state_reader_and_contract_manager.rs`** — in the `RunnableCompiledClass::V0(_)` cache-hit arm, call `self.state_reader.is_declared(class_hash)` and return `Err(StateError::UndeclaredClassHash(class_hash))` on failure, mirroring the Cairo 1 arm.

---

### Proof of Concept

1. Declare a Cairo 0 class `C` at block `N` on a running sequencer node.
2. Issue `starknet_call` (or `starknet_estimateFee`) targeting a contract whose class hash is `C`, with `block_id = N - 1`.
3. **Expected:** `CLASS_HASH_NOT_FOUND` error, because `C` was not declared at block `N - 1`.
4. **Actual:** The `ExecutionStateReader` is constructed with `state_number = right_after_block(N-1)`. `get_compiled_class(C)` reaches the `class_manager_handle` branch, retrieves `C` as `ContractClass::V0`, skips the declaration check, and returns `Ok(RunnableCompiledClass::V0(...))`. Execution proceeds and returns a result as if `C` were declared, producing a wrong authoritative RPC response.

The code path is:

```
starknet_call (RPC)
  → ExecutionStateReader::get_compiled_class(C)
      → class_manager_client.get_executable(C)  // returns Some(V0)
      → ContractClass::V0 arm                   // no is_contract_class_declared call
      → Ok(RunnableCompiledClass::V0(...))       // wrong: should be UndeclaredClassHash
``` [4](#0-3) [5](#0-4)
