### Title
`is_caller_an_attested_participant()` Does Not Check Attestation Expiry, Allowing Expired-TEE Nodes to Participate in Threshold Signing and Protocol Governance - (File: `crates/contract/src/tee/tee_state.rs`)

---

### Summary

`TeeState::is_caller_an_attested_participant()` verifies that a caller has a stored attestation entry and that the stored keys match, but it **never re-verifies whether the attestation has expired**. The expiry check exists in `TeeState::reverify_participants()`, which is called by `reverify_and_cleanup_participants()` and `clean_invalid_attestations()`, but is absent from the hot-path authorization gate used by every critical protocol method. As a result, a node whose TEE attestation has expired can continue to call `respond()`, `vote_pk()`, `vote_reshared()`, `start_keygen_instance()`, and `start_reshare_instance()` — all of which rely on `assert_caller_is_attested_participant_and_protocol_active()` as their sole TEE authorization check.

---

### Finding Description

`is_caller_an_attested_participant()` performs three checks:

1. The signer account is in the active participant set.
2. A stored attestation entry exists for the participant's TLS key.
3. The stored `account_id` and `account_public_key` match the caller. [1](#0-0) 

It does **not** call `re_verify()` with the current block timestamp, so an attestation that has passed its `expiry_timestamp_seconds` is treated as valid.

By contrast, `reverify_participants()` explicitly calls `re_verify()` with `Self::current_time_seconds()`, which rejects expired attestations: [2](#0-1) 

`assert_caller_is_attested_participant_and_protocol_active()` — the single authorization gate for all critical node-facing methods — delegates entirely to `is_caller_an_attested_participant()`: [3](#0-2) 

This gate is used by:
- `respond()` — submits threshold signatures [4](#0-3) 
- `respond_ckd()` — submits CKD responses [5](#0-4) 
- `respond_verify_foreign_tx()` — submits foreign-chain verification signatures [6](#0-5) 
- `vote_pk()`, `vote_reshared()`, `start_keygen_instance()`, `start_reshare_instance()`, `vote_abort_key_event_instance()` [7](#0-6) 

---

### Impact Explanation

The TEE attestation is the contract's mechanism for ensuring that only nodes running in a certified Trusted Execution Environment can participate in threshold signing and protocol governance. Attestation expiry is the signal that the TEE environment's certification has lapsed and the node should be excluded.

Because `is_caller_an_attested_participant()` does not check expiry, a node whose attestation has expired — and whose TEE environment is therefore no longer certified — can:

1. **Submit threshold signatures via `respond()`**: The contract accepts the signature and resolves the pending yield, delivering a signature to the end user. The signature is cryptographically valid (the node still holds its key share), but it was produced by a node that is no longer authorized by the TEE security model.
2. **Vote on resharing completion via `vote_reshared()`**: An expired-attestation node can cast the deciding vote that transitions the contract from `Resharing` to `Running`, finalizing a new participant set and keyset — a critical governance action.
3. **Start keygen/reshare instances**: An expired-attestation node can act as leader and initiate new protocol instances.

This breaks the production safety invariant that only currently-attested TEE nodes may influence threshold signing and key governance. It maps to: **participant/attestation authorization bypass that causes invalid bridge execution or double-spend conditions** (High), or at minimum **participant-state or contract execution-flow manipulation that breaks production safety/accounting invariants** (Medium).

---

### Likelihood Explanation

Attestation expiry is a normal operational event — attestations have finite lifetimes. The window between expiry and the next `verify_tee()` call (which triggers resharing to remove the expired node) can be arbitrarily long if no participant calls `verify_tee()`. During this window, the expired node continues to pass the `assert_caller_is_attested_participant_and_protocol_active()` check on every call. No special attacker capability is required beyond simply not renewing the attestation and continuing to operate.

---

### Recommendation

Add an expiry re-verification step inside `is_caller_an_attested_participant()`. After confirming the attestation entry exists and the keys match, call `re_verify()` (or the equivalent `reverify_participants()` helper) with the current block timestamp and reject the caller if the attestation has expired:

```rust
// After key-match checks, add:
let tee_upgrade_deadline_duration = /* pass from contract config */;
match attestation.verified_attestation.re_verify(
    Self::current_time_seconds(),
    &self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration),
    &self.get_allowed_launcher_compose_hashes(),
    &self.get_accepted_measurements(),
) {
    Ok(()) => Ok(()),
    Err(e) => Err(AttestationCheckError::AttestationExpiredOrInvalid(e.to_string())),
}
```

This mirrors the fix described in the external report: the check that exists in the "action" path (`reverify_participants` / `swap`) must also be present in the "validation" path (`is_caller_an_attested_participant` / `validate`).

### Proof of Concept

1. Node A is a legitimate participant with a valid attestation that expires at block timestamp `T`.
2. At timestamp `T+1`, Node A's attestation is expired. `reverify_participants()` would return `TeeQuoteStatus::Invalid`.
3. No participant has yet called `verify_tee()`, so the contract is still in `Running` state with Node A in the participant set.
4. Node A calls `respond(request, signature)`. The contract executes `assert_caller_is_attested_participant_and_protocol_active()`, which calls `is_caller_an_attested_participant()`. This function finds Node A's entry in `stored_attestations`, confirms the key match, and returns `Ok(())` — **without checking expiry**.
5. The contract accepts the signature and resolves the pending yield, delivering a threshold signature produced with participation from a node whose TEE certification has lapsed.
6. Similarly, Node A can call `vote_reshared()` to cast the deciding vote on a resharing round, finalizing a new keyset with an uncertified node's participation. [1](#0-0) [8](#0-7) [3](#0-2) [4](#0-3) [9](#0-8)

    /// reverifies stored participant attestations.
    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

    /// Casts a vote for `public_key` for the attempt identified by `key_event_id`.
    ///
    /// The effect of this method is either:
    ///  - Returns error (which aborts with no changes), if there is no active key generation
    ///    attempt (including if the attempt timed out), if the signer is not a participant, or if
    ///    the key_event_id corresponds to a different domain, different epoch, or different attempt
    ///    from the current key generation attempt.
    ///  - Returns Ok(()), with one of the following changes:
    ///    - A vote has been collected but we don't have enough votes yet.
    ///    - This vote is for a public key that disagrees from an earlier voted public key, causing
    ///      the attempt to abort; another call to `start` is then necessary.
    ///    - Everyone has now voted for the same public key; the state transitions into generating a
    ///      key for the next domain.
    ///    - Same as the last case, except that all domains have a generated key now, and the state
    ///      transitions into Running with the newly generated keys.
    #[handle_result]
    pub fn vote_pk(
        &mut self,
        key_event_id: KeyEventId,
        public_key: dtos::PublicKey,
    ) -> Result<(), Error> {
        log!(
            "vote_pk: signer={}, key_event_id={:?}, public_key={:?}",
            env::signer_account_id(),
            key_event_id,
            public_key,
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        let extended_key =
            public_key
                .try_into()
                .map_err(|err: PublicKeyExtendedConversionError| {
                    InvalidParameters::MalformedPayload {
                        reason: err.to_string(),
                    }
                })?;

        if let Some(new_state) = self.protocol_state.vote_pk(key_event_id, extended_key)? {
            self.protocol_state = new_state;
        }

        Ok(())
    }
