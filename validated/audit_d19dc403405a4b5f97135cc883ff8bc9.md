### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Redirect All Block Fee Income to an Arbitrary Address — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, `crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`ProposalInit` carries a `builder: ContractAddress` field that is broadcast by the proposer over the P2P network and consumed by every validator. The field is passed verbatim as `sequencer_address` into the `BlockInfo` that drives blockifier execution. Because `is_proposal_init_valid` never checks `builder` against any locally-trusted reference, a legitimate consensus proposer can set `builder` to any address they control, redirecting all transaction fee payments for that block to themselves and committing a block whose on-chain `sequencer_address` is wrong.

---

### Finding Description

`ProposalInit` is defined in `crates/apollo_protobuf/src/consensus.rs` with two distinct address fields:

- `proposer: ContractAddress` — "Address of the one who proposed the block in consensus."
- `builder: ContractAddress` — "Address of the one who builds/sequences the block." [1](#0-0) 

The `proposer` field is validated against the committee in both `apollo_consensus/src/manager.rs` and `single_height_consensus.rs`: [2](#0-1) [3](#0-2) 

The `builder` field, however, is **never validated**. `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri` — but not `builder`: [4](#0-3) [5](#0-4) 

The `ProposalInitValidation` struct, which carries the locally-trusted reference values used by `is_proposal_init_valid`, has no `builder` field at all: [6](#0-5) 

After passing `is_proposal_init_valid`, the raw `init` is forwarded to `convert_to_sn_api_block_info`, which maps `init.builder` directly to `sequencer_address` in the `BlockInfo` that is handed to the batcher for execution: [7](#0-6) 

In blockifier execution, `sequencer_address` is the address that receives fee payments from every transaction in the block. A wrong `sequencer_address` means all fees are credited to the attacker-controlled address instead of the legitimate sequencer.

On the proposer side, `builder_address` is correctly sourced from local static config: [8](#0-7) 

But validators never compare the received `init.builder` against their own `builder_address` config or any committee-derived value.

---

### Impact Explanation

Every validator calls `initiate_validation` → `convert_to_sn_api_block_info(init)` → `batcher.validate_block(input)` with the proposer-supplied `builder` as `sequencer_address`: [9](#0-8) 

The batcher executes all transactions in the block using this `sequencer_address`. Fee token balances are updated accordingly: the attacker's address receives all sequencer fees, and the legitimate sequencer receives nothing. The committed block header stores the wrong `sequencer_address`, corrupting the authoritative on-chain state and every RPC view of that block's sequencer.

This matches two Critical impact categories:
- **Wrong state / balance**: fee token storage at the legitimate sequencer address is not credited; the attacker's address is credited instead.
- **Incorrect fee / balance with economic impact**: all transaction fees for the block are stolen.

---

### Likelihood Explanation

The attacker must be the designated consensus proposer for a given height/round — a role that rotates among all staking validators. Any validator node in the committee can exploit this when it is their turn to propose. No external unprivileged trigger is required beyond being selected as proposer, which is a normal, expected event for every validator.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and check it in `is_proposal_init_valid`, comparing `init_proposed.builder` against the locally-configured `builder_address` (or a committee-derived value once available):

```rust
// In ProposalInitValidation:
pub builder_address: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

Populate `builder_address` in `validate_proposal` from `self.config.static_config.builder_address`, mirroring how the proposer sets it in `build_proposal`.

---

### Proof of Concept

1. Validator node A is selected as the consensus proposer for block N.
2. A constructs `ProposalInit` with `builder = attacker_address` (any address A controls) instead of the legitimate `self.config.static_config.builder_address`.
3. A broadcasts the proposal. Every other validator calls `validate_proposal` → `is_proposal_init_valid`. None of the checks in `is_proposal_init_valid` inspect `builder`; the proposal passes.
4. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: attacker_address, ... }`.
5. The batcher executes all transactions with `sequencer_address = attacker_address`. Fee token balances are updated: `attacker_address` receives all fees.
6. Consensus reaches decision; the block is committed with `sequencer_address = attacker_address` in the block header.
7. The legitimate sequencer receives zero fees for block N; the attacker receives all of them.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
