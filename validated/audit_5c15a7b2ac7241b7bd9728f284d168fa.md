### Title
Unbounded `fee_proposal_fri` Accepted During Startup Window Enables Persistent L2 Gas Price Manipulation - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

---

### Summary

The SNIP-35 `fee_proposal_fri` bounds check in `is_proposal_init_valid` is explicitly skipped whenever `fee_actual` is `None` — which is the case for the first `window_size` (10) blocks of the network. A malicious proposer who gets to propose even one block during this startup window can commit an arbitrary `fee_proposal_fri` value (e.g., `u128::MAX`) that validators accept without any range enforcement. That value is stored in the sliding window, poisons the median (`fee_actual`) once the window fills, and forces the L2 gas price floor to an attacker-controlled level for hundreds of subsequent blocks.

---

### Finding Description

`compute_fee_actual` computes the median of the last `window_size` (= 10) `fee_proposal_fri` values and returns `None` when the window is not yet full (i.e., `height < window_size`). [1](#0-0) 

The validator's `is_proposal_init_valid` function uses an `if let (Some(...), Some(...))` pattern to gate the bounds check on `fee_actual` being `Some`. When `fee_actual` is `None` the entire check is silently skipped: [2](#0-1) 

The comment on line 397 confirms this is intentional: *"During initiation (fee_actual is None, <window_size blocks), bounds are not enforced."*

The `fee_proposal_window_size` is 10 across all deployed versioned-constants files: [3](#0-2) 

During those first 10 blocks, any value in `[0, u128::MAX]` passes validation for `fee_proposal_fri`. The accepted value is then:

1. **Committed to the block header** via `BlockHeaderWithoutHash.fee_proposal_fri`: [4](#0-3) 

2. **Stored in `fee_proposals_window`** via `record_fee_proposal`: [5](#0-4) 

3. **Bound into the `ProposalCommitment`** hash so consensus signs over it: [6](#0-5) 

Once the window fills at block 10, `compute_fee_actual` returns the median of the 10 stored values. With `window_size = 10` (even), the median is the average of the 5th and 6th sorted values. A malicious proposer who controls ≥ 6 of the first 10 blocks can set the median to any value. Even controlling just 5 blocks and setting them all to `u128::MAX` yields a median of `(honest_5th + u128::MAX) / 2`.

That poisoned `fee_actual` is then used as the **floor** for the EIP-1559 L2 gas price calculation: [7](#0-6) 

With `fee_proposal_margin_ppt = 2` (0.2% per block), the convergence test shows it takes ~800–1400 blocks to recover from a poisoned starting value: [8](#0-7) 

---

### Impact Explanation

A malicious proposer who wins even a majority of the first 10 block slots can set `fee_proposal_fri` to `u128::MAX` in each of those blocks. Validators accept all of them (no bounds check). Once block 10 is reached, `fee_actual` becomes `u128::MAX` (or near it). `calculate_next_l2_gas_price_for_fin` uses `fee_actual` as `effective_min`, so the L2 gas price floor is set to `u128::MAX`. Every transaction in every subsequent block is priced at `u128::MAX` FRI, making the network economically unusable for hundreds of blocks until the 0.2%-per-block margin slowly walks the price back down. This is a **wrong fee/gas accounting value with direct economic impact** committed to authoritative block headers and signed over by consensus.

---

### Likelihood Explanation

The attack window is exactly the first 10 blocks of the network (genesis). In a Tendermint-style round-robin proposer schedule, a single malicious validator with ≥ 50% stake weight (or who is selected as proposer for ≥ 6 of the first 10 rounds) can execute this. The `weighted_quorum_single_heavy_staker_reaches_quorum` test confirms a single heavy staker can dominate: [9](#0-8) 

The opportunity is one-time (genesis only), but the damage persists for hundreds of blocks. Likelihood is **Medium** for a well-resourced attacker with significant stake at genesis.

---

### Recommendation

1. **Enforce a minimum-sample absolute bound during startup**: even when `fee_actual` is `None`, reject `fee_proposal_fri` values outside a hard absolute range (e.g., `[min_gas_price, some_max_startup_price]`). This mirrors the external report's recommendation to enforce a minimum sample size or return the sample size to callers.

2. **Alternatively, pre-seed the window from a trusted genesis configuration**: populate `fee_proposals_window` with a canonical bootstrap value for all `window_size` slots before block 0 is proposed, so `fee_actual` is never `None` and the bounds check always fires.

3. **At minimum, add a `min_gas_price` floor check on `fee_proposal_fri`** unconditionally (regardless of `fee_actual`), so the value can never be `0` or `u128::MAX` even during startup.

---

### Proof of Concept

```
Startup window: blocks 0–9, window_size = 10, fee_proposal_margin_ppt = 2.

1. Malicious proposer wins blocks 0, 2, 4, 6, 8 (5 of 10 slots).
   Sets fee_proposal_fri = u128::MAX in each.

2. Honest proposers win blocks 1, 3, 5, 7, 9.
   Set fee_proposal_fri = 80_000_000_000 (80 gwei, normal value).

3. is_proposal_init_valid for each of blocks 0–9:
   - proposal_init_validation.fee_actual = None  (height < 10)
   - The `if let (Some(fee_actual), Some(fee_proposal))` arm is NOT entered.
   - All fee_proposal_fri values accepted unconditionally.

4. At block 10, compute_fee_actual over heights [0,9]:
   Sorted: [80G, 80G, 80G, 80G, 80G, MAX, MAX, MAX, MAX, MAX]
   Median (even window) = avg(window[4], window[5])
                        = avg(80_000_000_000, u128::MAX)
                        ≈ u128::MAX / 2  (≈ 1.7 × 10^38 FRI)

5. calculate_next_l2_gas_price_for_fin at block 10:
   effective_min = max(config_min, fee_actual) = u128::MAX / 2
   → next L2 gas price = u128::MAX / 2

6. All transactions in blocks 10+ are priced at ~u128::MAX / 2 FRI.
   At 0.2% per block, recovery takes ~800 blocks (~35 minutes at 2.6s/block).
```

The root cause is in `is_proposal_init_valid`: [2](#0-1) 
with `fee_actual` sourced from: [10](#0-9)

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}

async fn test_compute_proposer_fee_proposal_converges_to_oracle_target() {
    // (strk_usd_rate, fee_target, n_blocks_until_convergence_with_buffer).
    // 75 gwei bootstrap -> 100 gwei target at +33% reaches by block ~795.
    // 100 gwei -> 60 gwei at -40% reaches by block ~1410.
    let phases: [(u128, GasPrice, u64); 2] = [
        (30_000_000_000_000_000, GasPrice(100_000_000_000), 800),
        (50_000_000_000_000_000, GasPrice(60_000_000_000), 1420),
    ];

    let (mut deps, _network) = create_test_and_network_deps();
    // Register per-phase strk_to_usd_rate expectations BEFORE setup_default_expectations so they
    // are not shadowed by the catch-all default (mockall matches oldest first).
    let mut seq = mockall::Sequence::new();
    for &(rate, _, n_blocks) in &phases {
        deps.l1_gas_price_provider
            .expect_get_strk_to_usd_rate()
            .times(usize::try_from(n_blocks).unwrap())
            .in_sequence(&mut seq)
            .returning(move |_| Ok(rate));
    }
    deps.setup_default_expectations();
    let mut context = deps.build_context();

    // Bootstrap the window with 75 gwei (the $0.04 target).
    let window_size = VersionedConstants::latest_constants().fee_proposal_window_size;
    for h in 0..window_size {
        context.record_fee_proposal(BlockNumber(h), Some(GasPrice(75_000_000_000)));
    }

    let mut height = window_size;
    for (phase_idx, (_, fee_target, n_blocks)) in phases.into_iter().enumerate() {
        for _ in 0..n_blocks {
            let h = BlockNumber(height);
            let fee_actual = compute_fee_actual(&context.fee_proposals_window, h, window_size)
                .expect("window stays complete across the loop");
            let proposal = context
                .compute_proposer_fee_proposal(Some(fee_actual), 0, TARGET_ATTO_USD_PER_L2_GAS)
                .await;
            context.record_fee_proposal(h, Some(proposal));
            height += 1;
        }
        let final_fee_actual =
            compute_fee_actual(&context.fee_proposals_window, BlockNumber(height), window_size)
                .expect("window stays complete across the loop");
        assert_eq!(
            final_fee_actual, fee_target,
            "phase {phase_idx}: fee_actual did not reach fee_target after {n_blocks} blocks",
        );
    }
