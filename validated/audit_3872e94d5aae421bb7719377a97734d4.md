Audit Report

## Title
Missing Chainlink Price Feed Staleness Validation Allows Stale Price Exploitation - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all return values except `answer` (price), performing zero staleness or validity checks. A stale Chainlink price can be used to mint excess rsETH on deposit at the expense of existing protocol participants. The same protocol already implements the correct checks in `ChainlinkOracleForRSETHPoolCollateral.getRate()`, making the omission in the primary oracle unambiguous.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` at line 52 reads only `price` from `latestRoundData()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`roundId`, `startedAt`, `updatedAt`, and `answeredInRound` are all silently discarded. No checks are performed for:
- `answeredInRound >= roundId` (stale round detection)
- `updatedAt != 0` (incomplete round detection)
- `price > 0` (invalid/negative price detection)
- `block.timestamp - updatedAt <= heartbeat` (feed freshness)

The exploit path is fully public and permissionless:
1. A Chainlink LST/ETH feed becomes stale at a price above the true market price (e.g., due to network congestion or keeper failure).
2. Attacker calls `LRTDepositPool.depositAsset()` → `_beforeDeposit()` → `getRsETHAmountToMint()` → `lrtOracle.getAssetPrice(asset)` → `ChainlinkPriceOracle.getAssetPrice()`.
3. The stale inflated price flows into `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`, minting more rsETH than the deposit is worth.
4. Attacker holds or redeems excess rsETH, extracting value from existing depositors whose rsETH is now backed by less ETH per token.

The `pricePercentageLimit` guard in `LRTOracle._updateRsETHPrice()` is irrelevant here — it only fires when the manager explicitly calls `updateRSETHPrice()`, not at deposit time.

The contrast with `ChainlinkOracleForRSETHPoolCollateral.getRate()` (lines 30–32) is direct: that contract checks `answeredInRound < roundID`, `timestamp == 0`, and `ethPrice <= 0` before returning — the exact checks absent from the primary oracle.

## Impact Explanation
**Critical — Direct theft of at-rest user funds.**

When a stale inflated price is accepted, the rsETH supply is inflated relative to the protocol's actual ETH backing. Every existing rsETH holder's redemption value is permanently diluted. The attacker's excess rsETH represents a claim on ETH that was deposited by prior users. This is direct, permanent theft of at-rest user funds, matching the Critical impact class.

## Likelihood Explanation
**Medium.**

Chainlink LST/ETH feeds on Ethereum mainnet have 24-hour heartbeats and 0.5% deviation thresholds. Staleness windows arise during prolonged network congestion, Chainlink keeper outages, or feed deprecation events. The presence of `contracts/L2/` in the repository also indicates L2 deployment, where sequencer downtime can cause extended staleness. No on-chain mechanism prevents a deposit during a staleness window. The attacker requires no special privileges — only the ability to call `depositAsset()`.

## Recommendation
Mirror the validation pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optionally: if (block.timestamp - updatedAt > HEARTBEAT) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept
**Foundry fork test plan:**

1. Fork Ethereum mainnet at a block where a supported LST/ETH Chainlink feed is live.
2. Deploy or reference the existing `ChainlinkPriceOracle` with the stETH/ETH feed.
3. Use `vm.mockCall` to make `latestRoundData()` return a stale round: set `updatedAt = block.timestamp - 25 hours`, `answeredInRound = roundId - 1`, and `answer` at 5% above the true price.
4. Call `LRTDepositPool.depositAsset(stETH, amount, 0, "")` as an unprivileged attacker.
5. Assert that `rsethAmountToMint` is inflated by ~5% relative to a deposit using a fresh price.
6. Assert that existing rsETH holders' `rsETHPrice()` is diluted after `updateRSETHPrice()` is called.
