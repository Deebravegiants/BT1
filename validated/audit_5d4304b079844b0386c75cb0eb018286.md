### Title
Gateway Stateful Validator Checks Only L2 Gas Price Threshold, Leaving L1 Gas and L1 Data Gas Prices Unchecked at Admission — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`validate_tx_l2_gas_price_within_threshold` enforces a minimum L2 gas price threshold at gateway admission but explicitly skips the check for `ValidResourceBounds::L1Gas` legacy transactions and applies no threshold at all to `l1_gas` or `l1_data_gas` price fields in `AllResources` transactions. A TODO comment in the code acknowledges this gap. As a result, any unprivileged user can submit transactions with below-market L1 gas prices that pass all gateway and mempool admission checks, are inserted into the priority queue, and are only rejected later inside the blockifier's `perform_pre_validation_stage`. This is the direct Sequencer analog of `minLoanSize = 0`: a missing minimum threshold at the admission layer allows economically unviable transactions to accumulate and consume sequencer resources.

### Finding Description

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` is the only gateway-level gas price floor check. Its implementation contains two structural gaps:

**Gap 1 — `L1Gas` legacy transactions are entirely exempt:**

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:385-387
ValidResourceBounds::L1Gas(_) => {
    // No validation required for legacy transactions.
}
```

Any transaction carrying `ValidResourceBounds::L1Gas` (a V3 transaction converted to the legacy single-resource form) bypasses the threshold check unconditionally.

**Gap 2 — `AllResources` transactions: only L2 gas price is checked:**

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:358-384
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(...) {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // ... threshold check on l2_gas only ...
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
}
```

The developer-acknowledged TODO confirms that `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never checked against any threshold at the gateway level.

The stateless validator's `min_gas_price` check only covers `l2_gas.max_price_per_unit`:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator.rs:71-76
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

`l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are not checked here either.

**Attack path:**

1. Attacker submits `AllResources` transactions with `l2_gas.max_price_per_unit = min_gas_price` (8 Gwei, the stateless floor) but `l1_gas.max_price_per_unit = 1` and `l1_data_gas.max_price_per_unit = 1`.
2. Stateless validator passes: `l2_gas` price meets `min_gas_price`; total fee is non-zero.
3. Stateful validator passes: `validate_tx_l2_gas_price_within_threshold` only checks `l2_gas`.
4. Mempool admits the transaction into the priority queue (keyed on `max_l2_gas_price`).
5. Batcher selects the transaction for execution.
6. Blockifier's `check_fee_bounds` in `perform_pre_validation_stage` detects `l1_gas.max_price_per_unit < actual_l1_gas_price` and rejects the transaction — **after** sequencer resources have been consumed.

### Impact Explanation

The gateway and mempool accept transactions that are invalid at execution time (below-market L1 gas price). These transactions enter the priority queue, are dequeued by the batcher, and fail inside `perform_pre_validation_stage`. This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."**

At scale, an attacker can flood the priority queue with such transactions, causing the batcher to repeatedly attempt and discard them, degrading block-building throughput. The mempool's capacity limits slow but do not prevent this because each transaction individually passes all admission checks.

### Likelihood Explanation

The attack requires no privileged access. Any user with a funded account can craft `AllResources` transactions with a valid L2 gas price and a near-zero L1 gas price. The stateless `min_gas_price` check (8 Gwei) applies only to `l2_gas`, so the cost to the attacker is the minimum L2 gas fee per transaction. The gap is acknowledged by the developers themselves via the TODO comment.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to enforce a minimum threshold on `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` relative to the previous block's L1 gas prices, mirroring the existing L2 check. Apply the same logic to `ValidResourceBounds::L1Gas` transactions using the L1 gas price from the previous block. This closes the gap acknowledged by the TODO at line 358.

### Proof of Concept

1. Obtain the current block's L2 gas price `P_l2` and L1 gas price `P_l1`.
2. Construct an `AllResources` invoke transaction with:
   - `l2_gas.max_price_per_unit = P_l2` (meets the stateless `min_gas_price` floor)
   - `l1_gas.max_price_per_unit = 1` (far below `P_l1`)
   - `l1_data_gas.max_price_per_unit = 1`
3. Submit via the HTTP gateway. The stateless validator passes (L2 price ≥ `min_gas_price`; total fee > 0). The stateful validator passes (`validate_tx_l2_gas_price_within_threshold` only checks L2 price). The transaction is inserted into the mempool priority queue.
4. Repeat at high volume. Each transaction is dequeued by the batcher and fails in `check_fee_bounds` inside `perform_pre_validation_stage` with `ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas, ... }`, wasting batcher execution cycles without producing valid block content.

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier

                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {

                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)

            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
