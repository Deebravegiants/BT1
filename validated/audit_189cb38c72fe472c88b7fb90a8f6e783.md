Audit Report

## Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped Per-Bin Additional Fees — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

## Summary
`setPoolBinAdditionalFees` forwards `addFeeBuyE6` and `addFeeSellE6` (both `uint16`, max 65,535 ≈ 6.55%) directly to the pool with no comparison against `maxAdminSpreadFeeE6`. Because these per-bin fees are added on top of the oracle-derived base fee in every swap, the pool admin can charge traders more than the factory owner's governance cap permits, breaking the admin-boundary invariant the protocol explicitly enforces on the parallel `setPoolAdminFees` path.

## Finding Description

**Capped path — `setPoolAdminFees`:** [1](#0-0) 

Both `newAdminSpreadFeeE6` and `newAdminNotionalFeeE8` are rejected if they exceed the factory-owner-set caps before any pool state is touched.

**Uncapped path — `setPoolBinAdditionalFees`:** [2](#0-1) 

No cap check exists. The call passes directly to `setBinAdditionalFees` on the pool, which stores the raw `uint16` values with no validation beyond the bin-index range check. [3](#0-2) 

**Per-bin fees are applied in every swap:**

For the buy-token-0 direction: [4](#0-3) 

For the sell-token-0 direction: [5](#0-4) 

In both cases `binState.addFeeBuyE6` / `addFeeSellE6` is added to `params.baseFeeX64` (the oracle-spread base fee) and applied to every unit of input in the affected bin. A `uint16` value of 65,535 represents ≈6.55%, which can exceed any `maxAdminSpreadFeeE6` value the factory owner has configured below that threshold (the hard ceiling is 200,000 = 20%, set in `HARD_MAX_SPREAD_FEE_E6`). [6](#0-5) 

**Exploit flow:**
1. Factory owner sets `maxAdminSpreadFeeE6 = 10_000` (1%) via `setFeeCaps`.
2. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)` — no revert.
3. Every swap through bin 0 now pays base fee + 6.55% additional, far exceeding the 1% governance cap.
4. The excess fee accrues to the pool's LP balances (not separately tracked), effectively extracting value from traders beyond what governance permits.

## Impact Explanation
This is a direct admin-boundary break: the pool admin exceeds the `maxAdminSpreadFeeE6` cap set by the factory owner, which is the exact invariant the protocol's governance model is designed to enforce. Traders in the affected bin pay materially higher fees than the protocol governance model permits, constituting a direct loss of user principal on every swap routed through that bin. This meets the "Admin-boundary break: pool admin exceeds caps" allowed impact and qualifies as Medium severity (pool admin is semi-trusted and bounded per the contest Q&A; bypassing caps is explicitly called out as a valid finding).

## Likelihood Explanation
The pool admin is a single role with no additional preconditions required. The call is a single transaction with no timelock. It can be applied to any bin at any time and is immediately effective on the next swap. The bypass is repeatable and persistent until the factory owner intervenes (which has no direct mechanism to reset per-bin fees).

## Recommendation
Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool:

```solidity
if (uint24(addFeeBuyE6) > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (uint24(addFeeSellE6) > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
```

Alternatively, enforce a combined cap (global admin spread fee + per-bin additional fee ≤ `maxAdminSpreadFeeE6`) to prevent the two paths from being used additively to exceed the cap.

## Proof of Concept

```solidity
// Foundry test sketch
function test_binFeeBypassesCap() public {
    // Factory owner sets a 1% admin spread cap
    factory.setFeeCaps(200_000, 10_000, 1_000_000, 1_000_000);

    address pool = _createPool(); // adminSpreadFeeE6 = 0 at creation

    // Pool admin sets per-bin fee to uint16.max (≈6.55%), bypassing the 1% cap
    vm.prank(admin);
    factory.setPoolBinAdditionalFees(pool, 0, 65_535, 65_535); // no revert

    // Verify storage
    (,,, uint16 buyFee, uint16 sellFee) = PoolStateLibrary._binState(pool, 0);
    assertEq(buyFee, 65_535); // 6.55% >> 1% cap
    assertEq(sellFee, 65_535);

    // Any swap through bin 0 now pays base fee + 6.55%
}
```
