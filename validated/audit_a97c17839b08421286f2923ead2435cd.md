### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Redirect All Block Fees and Corrupt Block Hash - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, L1 gas prices, `fee_proposal_fri`, and `timestamp` — but never validates `ProposalInit.builder`. The `builder` field is passed verbatim as `sequencer_address` into `BlockInfo` via `convert_to_sn_api_block_info`, which is then used for fee transfers and included in the partial block hash. A legitimate-but-malicious proposer can set `builder` to any arbitrary address; every validator re-executes the block with that address, computes the same `PartialBlockHash`, and accepts the proposal — permanently committing a block whose sequencer address, fee balances, and block hash are all wrong.

### Finding Description

`ProposalInit` carries two identity fields:

- `proposer` — validated in `handle_proposal` against the committee's expected proposer for the round.
- `builder` — **never validated anywhere**. [1](#0-0) 

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in `BlockInfo`: [2](#0-1) 

`is_proposal_init_valid` checks many `ProposalInit` fields but has no check on `builder`: [3](#0-2) 

The `ProposalInitValidation` struct — the only source of locally-trusted reference values passed to `is_proposal_init_valid` — does not include a `builder_address` field at all: [4](#0-3) 

The validator's own `builder_address` is available in `ContextStaticConfig`: [5](#0-4) 

but it is never threaded into `ProposalInitValidation` or compared against `init.builder`.

`PartialBlockHashComponents::new` stores `block_info.sequencer_address` (i.e., `init.builder`) as the `sequencer` field: [6](#0-5) 

`calculate_block_hash` chains `sequencer` into the Poseidon hash: [7](#0-6) 

Because both proposer and validator call `convert_to_sn_api_block_info` with the same proposer-supplied `init.builder`, they compute identical `PartialBlockHash` values. The `ProposalFinMismatch` guard therefore does **not** catch the manipulation: [8](#0-7) 

### Impact Explanation

`sequencer_address` is the fee-transfer destination for every transaction in the block. Setting `builder` to an attacker-controlled address redirects all block fees to that address (wrong storage values in the fee-token contract). The committed block hash also encodes the wrong sequencer address, and any contract calling `get_block_info()` during execution sees the attacker's address as the sequencer. All three effects are permanent once the block is decided by consensus.

This matches:
- *Critical. Wrong state … storage value … from blockifier/syscall/execution logic for accepted input.*
- *Critical. Incorrect fee … balance … with economic impact.*

### Likelihood Explanation

The attacker must be a legitimate committee member scheduled as proposer for the target round. No external unprivileged trigger exists. However, in a permissioned validator set any single compromised or malicious validator can exploit this every time it is the designated proposer, with no other validators able to detect or reject the manipulation.

### Recommendation

Add `builder_address: ContractAddress` to `ProposalInitValidation` and populate it from `self.config.static_config.builder_address` in `validate_current_round_proposal`. Then add the following check inside `is_proposal_init_valid`:

```rust
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

This mirrors the existing `proposer` check in `handle_proposal` and closes the gap. [9](#0-8) 

### Proof of Concept

1. Attacker controls a validator node that is the scheduled proposer for height H, round R.
2. Attacker modifies `builder_address` in `ContextStaticConfig` to `ATTACKER_ADDRESS` before calling `build_proposal`.
3. `initiate_build` constructs `ProposalInit { builder: ATTACKER_ADDRESS, … }` and streams it to all peers.
4. Each validator peer calls `validate_proposal` → `is_proposal_init_valid` (no `builder` check) → `initiate_validation` → `convert_to_sn_api_block_info` → `BlockInfo { sequencer_address: ATTACKER_ADDRESS }`.
5. The batcher executes all transactions with `sequencer_address = ATTACKER_ADDRESS`; all fees are credited to `ATTACKER_ADDRESS` in the fee-token contract.
6. `PartialBlockHashComponents::new` stores `ATTACKER_ADDRESS` as `sequencer`; both sides compute the same `PartialBlockHash`; `ProposalFinMismatch` is not triggered.
7. Consensus reaches decision; the block is committed with `sequencer = ATTACKER_ADDRESS`, wrong fee balances, and a block hash that encodes the attacker's address. [10](#0-9) [11](#0-10) [12](#0-11)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
