### Title
Reentrancy in `OmniBridge.initTransfer()` via Malicious ERC20 Enables Duplicate Cross-Chain Settlement and Unbacked Wrapped Token Minting — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.initTransfer()` has no reentrancy guard and performs an external `safeTransferFrom` call on an arbitrary ERC20 token before the function fully completes. A malicious ERC20 token can re-enter `initTransfer` during that call, causing two distinct `InitTransfer` events to be emitted with different origin nonces — without the bridge actually receiving the corresponding tokens. Both events constitute valid proofs on NEAR, resulting in duplicate cross-chain settlement and unbacked wrapped token minting.

---

### Finding Description

In `OmniBridge.initTransfer()`, the function first increments `currentOriginNonce`, then — for non-bridge, non-custom-minter tokens — calls `IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount)`: [1](#0-0) 

There is no `nonReentrant` modifier and no reentrancy guard anywhere in the inheritance chain (`UUPSUpgradeable`, `AccessControlUpgradeable`, `SelectivePausableUpgradable`). The `currentOriginNonce` increment at line 381 happens before the external call, so a re-entrant invocation of `initTransfer` will consume a fresh nonce (N+1) while the outer call holds nonce N.

Attack sequence with a malicious ERC20 whose `transferFrom` re-enters `initTransfer`:

1. Outer call: `currentOriginNonce` → N; calls `safeTransferFrom` on malicious token.
2. Malicious token's `transferFrom` re-enters `initTransfer`.
3. Re-entrant call: `currentOriginNonce` → N+1; calls `safeTransferFrom` again (token does not actually move funds); emits `InitTransfer(nonce=N+1)`.
4. Re-entrant call returns; outer call resumes; emits `InitTransfer(nonce=N)`.

Both events are emitted with distinct, valid nonces. The bridge holds zero (or one) token deposit but two `InitTransfer` proofs exist on-chain. [2](#0-1) 

The same pattern exists in `initTransfer1155`, where `currentOriginNonce` is incremented before `IERC1155.safeTransferFrom`: [3](#0-2) 

---

### Impact Explanation

On the NEAR side, `fin_transfer` / `fin_transfer_callback` verifies the emitter address against registered factories and checks `finalised_transfers` for replay, but each of the two re-entrant events carries a **distinct** `(origin_chain, origin_nonce)` pair: [4](#0-3) 

Because the nonces differ, neither event is flagged as a replay. Both are processed as independent, valid transfers. The NEAR bridge mints wrapped tokens for both, while the EVM bridge holds at most one token deposit — breaking the 1:1 backing guarantee and producing unbacked wrapped supply.

This maps directly to the **High** allowed impact: *"Replayable, non-unique, or duplicate cross-chain settlement … that produces double-credit or unbacked supply."*

---

### Likelihood Explanation

The attack requires only:
1. Deploying a malicious ERC20 (permissionless).
2. Registering it via the public `logMetadata` → `deployToken` flow (no privileged role needed).
3. Calling `initTransfer` with the malicious token.

No privileged role, leaked key, or external dependency compromise is required. Any unprivileged user can execute this end-to-end.

---

### Recommendation

Add OpenZeppelin's `ReentrancyGuard` (or `ReentrancyGuardUpgradeable`) to `OmniBridge` and apply the `nonReentrant` modifier to `initTransfer` and `initTransfer1155`:

```solidity
import {ReentrancyGuardUpgradeable} from
    "@openzeppelin/contracts-upgradeable/utils/ReentrancyGuardUpgradeable.sol";

contract OmniBridge is ..., ReentrancyGuardUpgradeable {
    function initTransfer(...) external payable nonReentrant whenNotPaused(PAUSED_INIT_TRANSFER) { ... }
    function initTransfer1155(...) external payable nonReentrant whenNotPaused(PAUSED_INIT_TRANSFER) { ... }
}
```

Alternatively, restructure to strict CEI: record the transfer (store nonce + token + amount in a mapping) **before** any external call, and verify the mapping entry is fresh on re-entry.

    IOmniBridge bridge;

    // Standard ERC20 stubs omitted for brevity; balanceOf/allowance return large values.
