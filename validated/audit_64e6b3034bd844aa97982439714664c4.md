Audit Report

## Title
Silent Zero Return in `_withdrawFromAave` When `totalETHDepositedToAave == 0` With Residual aWETH Balance Causes Temporary ETH Withdrawal Freeze — (`contracts/LRTWithdrawalManager.sol`)

## Summary
When `totalETHDepositedToAave == 0` but `aaveAWETH.balanceOf(address(this)) > 0` (an interest residual left after full principal withdrawal), `_withdrawFromAave` silently returns 0 instead of reverting, causing `_processWithdrawalCompletion` to revert with `InsufficientLiquidityForWithdrawal` for all ETH withdrawal completions. This state is reachable in normal protocol operation whenever interest accrues between the last `collectInterestToTreasury` call and the final principal withdrawal. All monitoring view functions (`getAaveWithdrawableLiquidity`, `getAaveBalance`, `aaveHealthCheck`) return positive/healthy values in this state, masking the broken condition and delaying operator diagnosis.

## Finding Description
**Root cause:** `_withdrawFromAave` computes `withdrawablePrincipal = min(aaveBalance, totalETHDepositedToAave)` at line 912. When `totalETHDepositedToAave == 0`, this evaluates to 0 regardless of `aaveBalance`. The guard at line 909 only reverts when `aaveBalance == 0`, so a non-zero residual passes through. Line 914–915 then sets `withdrawnAmount = 0` and silently returns 0.

**Reachable precondition (normal operation):**
1. 100 ETH deposited → `totalETHDepositedToAave = 100`, `aaveBalance ≈ 100`
2. Interest accrues → `aaveBalance = 100.5`
3. User withdrawals drain principal: `_withdrawFromAave(100)` → `withdrawablePrincipal = min(100.5, 100) = 100`, withdraws 100, sets `totalETHDepositedToAave = 0`; `aaveBalance = 0.5` remains as orphaned interest residual

**Exploit path:**
- Any user calls `completeWithdrawal` for an ETH request while the contract holds insufficient ETH
- `_processWithdrawalCompletion` (line 720–731) calls `_withdrawFromAave(amountNeeded)`, receives 0
- `balanceAfter < request.expectedAssetAmount` → reverts with `InsufficientLiquidityForWithdrawal`
- All state changes (including `delete withdrawalRequests[requestId]` at line 712) are rolled back; the request remains queued but unserviceable

**Misleading view functions in this state:**
- `getAaveWithdrawableLiquidity()` (line 688): returns `IERC20(WETH_ADDRESS).balanceOf(address(aaveAWETH))` — the Aave pool's total WETH liquidity, always large and unrelated to the contract's position
- `_checkAaveHealth()` (line 931): `principal(0) > aaveBalance(0.5)` is false → returns `true`

**`emergencyWithdrawFromAave` also broken in this state (lines 551–563):**
- Line 554–555: `aaveBalance = 0.5 > 0` → passes the guard
- Line 558: `_collectInterestToTreasury()` drains the 0.5 ETH residual to treasury, leaving `aaveBalance = 0`
- Line 560: `_withdrawFromAave(amount)` → `aaveBalance == 0` → reverts with `InsufficientAaveBalance`
- The emergency escape hatch itself fails in the exact state it is needed

## Impact Explanation
**Medium — Temporary freezing of funds.** All ETH withdrawal completions revert while the contract is in this state. User funds are not lost (requests remain in queue), but users cannot access their ETH until an operator diagnoses and resolves the state. The freeze is not permanent: the operator can call `collectInterestToTreasury()` (succeeds because `_checkAaveHealth()` returns `true`) then `setAaveIntegrationEnabled(false)`, after which withdrawals proceed normally. However, the misleading view functions delay diagnosis, and `emergencyWithdrawFromAave` — the intended recovery tool — also fails, leaving only the indirect two-step operator path.

## Likelihood Explanation
Medium. No adversarial action is required. The state arises in normal protocol operation whenever Aave interest accrues between the last interest collection and the final principal withdrawal — a routine occurrence for any protocol with active Aave integration and ongoing user withdrawals. The condition is not edge-case; it will be reached repeatedly over the protocol's lifetime.

## Recommendation
1. **`_withdrawFromAave`**: When `totalETHDepositedToAave == 0` and `aaveBalance > 0`, either revert with a descriptive error (e.g., `OnlyInterestResidualRemaining`) or treat the residual as withdrawable by collecting it as interest inline before returning.
2. **`emergencyWithdrawFromAave`**: Re-read `aaveBalance` after `_collectInterestToTreasury()` and skip `_withdrawFromAave` if the balance is now 0, or restructure to withdraw principal before collecting interest.
3. **`getAaveWithdrawableLiquidity`**: Return `min(WETH_balance_of_aWETH_contract, totalETHDepositedToAave)` to reflect the contract's actual withdrawable principal rather than pool-level liquidity.
4. **`_checkAaveHealth`**: Return `false` when `totalETHDepositedToAave == 0` and `aaveBalance > 0` to correctly flag the orphaned residual state.
