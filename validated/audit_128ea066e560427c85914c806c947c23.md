### Title
Removed-Tombstone Source-Budget Exhaustion Allows Suppression of Active Edges in NextHopTable — (`chain/network/src/routing/graph/mod.rs`)

### Summary

The per-source edge budget (`max_edges_per_source`) can be fully consumed by valid `Removed`-state tombstones for peer pairs that have never been `Active` in the victim node's graph. Because `update_edge` increments `source_edge_count` for any new key regardless of edge state, an attacker who has legitimately connected to and disconnected from `N` peers can send `N` Removed tombstones to exhaust their budget, after which any subsequent `Active` edges they introduce are silently dropped. This causes the victim's `NextHopTable` to be missing reachable peers, misrouting chunk and witness messages.

### Finding Description

**Root cause — `Inner::update_edge` (lines 67–110):** [1](#0-0) 

The function checks `is_new_key` and enforces the per-source cap before branching on edge state. For a `Removed` edge whose key has never appeared in `Inner::edges`:

1. `is_new_key = true` — cap check passes if budget is available.
2. `EdgeState::Removed => self.graph.remove_edge(&key.0, &key.1)` — this is a **no-op** in `bfs::Graph` because the pair was never added to the BFS graph.
3. The `is_new_key` block then runs unconditionally, inserting the key into `edge_source` and incrementing `source_edge_count`.
4. The tombstone is inserted into `Inner::edges`. [2](#0-1) 

The BFS graph no-op is confirmed in `bfs::Graph::remove_edge`: [3](#0-2) 

**Signature validity of Removed edges:**

`Edge::verify()` for `Removed` state requires `signature0` and `signature1` to verify against `prev_hash()` (the hash of the preceding Active edge at `nonce-1`), plus a removal signature from one party: [4](#0-3) 

An attacker controlling peer A can legitimately obtain valid Removed edges for `(A, B1)`, `(A, B2)`, ..., `(A, Bn)` by actually connecting to and disconnecting from those peers. Both parties sign during the handshake and removal, so the attacker holds all required signatures.

**Budget exhaustion path in `add_edges`:** [5](#0-4) 

The pre-check at lines 257–263 uses `provisional_new_edges` to skip verification once the cap is reached. Once the budget is exhausted by tombstones, subsequent `Active` edges from the same source are dropped at the pre-check stage (or in `update_edge`) without being added to the BFS graph or `Inner::edges`.

### Impact Explanation

The `NextHopTable` is computed exclusively from the BFS graph, which only contains `Active` edges. Tombstones stored in `Inner::edges` contribute nothing to reachability. When the attacker's source budget is exhausted by tombstones, any `Active(A, C)` edge the attacker later sends is rejected. If peer C is only reachable through A's advertisement (e.g., a newly connected peer not yet gossiped by others), the victim's `NextHopTable` will be missing C, causing chunk and witness messages destined for C to be misrouted or dropped.

### Likelihood Explanation

**Preconditions the attacker must satisfy:**
1. Control one peer key (peer A) — trivial.
2. Have previously connected to and disconnected from `max_edges_per_source` distinct peers (B1…Bn) to obtain valid Removed edges — requires real network activity proportional to the cap value.
3. The tombstones must persist: B1…Bn must remain reachable via other Active edges so `prune_unreachable_peers` does not reclaim the budget. [6](#0-5) 

If B1…Bn are well-connected peers (validators, boot nodes), they will remain in `next_hops` and their `peer_reachable_at` will be refreshed, keeping the tombstones alive indefinitely.

**Mitigating factors:**
- Active edges are gossiped by multiple peers; `Active(A, C)` sent by C or other peers would be attributed to a different source and accepted.
- `prune_edges_after` (if configured) eventually evicts old tombstones, recovering the budget.
- The attacker's budget is bounded by `max_edges_per_source`; the default production value (referenced as `DEFAULT_ROUTING_GRAPH_MAX_EDGES_PER_SOURCE` in config) determines the cost of the attack. [7](#0-6) 

### Recommendation

In `update_edge`, do not count a `Removed` tombstone as a new key against the source budget when the corresponding `Active` edge was never present in `Inner::edges`. One approach: only increment `source_edge_count` when `EdgeState::Active` and `is_new_key`, or when `EdgeState::Removed` and the key was already tracked (i.e., the Active edge was previously accepted). Alternatively, maintain a separate tombstone budget distinct from the active-edge budget.

    fn update_edge(&mut self, edge: Edge, source: Option<&PeerId>) -> bool {
        if has(&self.edges, &edge) {
            return false;
        }
        let key = edge.key();
        let is_new_key = !self.edges.contains_key(key);
        if is_new_key {
            // Global edge cap (applies to local AND remote).
            if self.edges.len() >= self.config.max_total_edges {
                metrics::EDGE_DROPPED.inc();
                return false;
            }
            // Per-source cap (remote only).
            if let Some(source) = source {
                let count = self.source_edge_count.get(source).copied().unwrap_or(0);
                if count >= self.config.max_edges_per_source {
                    metrics::EDGE_DROPPED.inc();
                    return false;
                }
            }
        }
        match edge.edge_type() {
            EdgeState::Active => {
                if !self.graph.add_edge(&key.0, &key.1) {
                    // BFS peer cap exceeded.
                    metrics::EDGE_DROPPED.inc();
                    return false;
                }
            }
            EdgeState::Removed => self.graph.remove_edge(&key.0, &key.1),
        }
        // Pin edges to their original introducing source. We intentionally
        // don't reassign attribution on nonce updates so that an attacker cannot recycle
        // budget across multiple colluding peers by replaying edges with bumped
        // nonces from a fresh source.
        if is_new_key {
            if let Some(source) = source {
                self.edge_source.insert(key.clone(), source.clone());
                *self.source_edge_count.entry(source.clone()).or_insert(0) += 1;
            }
        }
        self.edges.insert(key.clone(), edge);
        true
    }

    fn prune_unreachable_peers(&mut self, unreachable_since: time::Instant) {
        // Select peers to prune.
        let mut peers = HashSet::new();
        for k in self.edges.keys() {
            for peer_id in [&k.0, &k.1] {
                if self
                    .peer_reachable_at
                    .get(peer_id)
                    .map(|t| t < &unreachable_since)
                    .unwrap_or(true)
                {
                    peers.insert(peer_id.clone());
                }
            }
        }
        if peers.is_empty() {
            return;
        }

        // Prune peers from peer_reachable_at.
        for peer_id in &peers {
            self.peer_reachable_at.remove(&peer_id);
        }

        // Prune edges from graph.
        self.remove_adjacent_edges(&peers);
    }

    pub fn remove_edge(&mut self, peer0: &PeerId, peer1: &PeerId) {
        assert_ne!(peer0, peer1);
        if self.contains_edge(peer0, peer1) {
            let id0 = self.get_id(peer0);
            let id1 = self.get_id(peer1);

            self.adjacency[id0 as usize].retain(|&x| x != id1);
            self.adjacency[id1 as usize].retain(|&x| x != id0);

            self.remove_if_unused(id0);
            self.remove_if_unused(id1);

            self.total_active_edges -= 1;
        }
