### Title
`EvmFinality::Latest` Hardcoded for Abstract Chain in MPC Prover Allows Reorg-Based Unbacked Token Minting on NEAR - (File: near/omni-prover/mpc-omni-prover/src/lib.rs)

---

### Summary
`MpcOmniProver::init()` hardcodes `EvmFinality::Latest` as the required finality level for the Abstract chain. `Latest` is the weakest EVM finality tag — it corresponds to the most recently produced block with zero confirmation depth. Because the MPC network attests to a transaction's existence at this non-finalized block, any chain reorg that removes the source block after the MPC signature is produced but before the NEAR transfer is settled leaves a valid, irrevocable MPC-signed proof for a transaction that no longer exists on the canonical Abstract chain. A trusted relayer can then finalize the transfer on NEAR, minting unbacked wrapped tokens.

---

### Finding Description
In `MpcOmniProver::init()`, the finality map is populated with:

```rust
finalities.insert(ChainKind::Abs, MpcFinality::Evm(EvmFinality::Latest));
``` [1](#0-0) 

`verify_proof()` enforces that the caller-supplied `sign_payload` carries a finality tag that exactly matches the stored value:

```rust
require!(
    Self::request_matches_finality(&payload_v1.request, finality),
    ProverError::FinalityMismatch.as_ref()
);
``` [2](#0-1) 

`request_matches_finality` performs a strict equality check between the request's `finality` field and the stored `MpcFinality`:

```rust
(ForeignChainRpcRequest::Ethereum(args) | ForeignChainRpcRequest::Abstract(args),
 MpcFinality::Evm(finality)) => args.finality == *finality,
``` [3](#0-2) 

Because the stored value is `EvmFinality::Latest`, only proofs carrying `EvmFinality::Latest` are accepted — `Safe` or `Finalized` proofs are rejected. The test fixture confirms this is the intended production value for Abstract chain:

```rust
fn abs_testnet_evm_request() -> EvmRpcRequest {
    EvmRpcRequest { ..., finality: EvmFinality::Latest }
}
``` [4](#0-3) 

After the MPC network signs the payload, `verify_callback` checks only that `SHA-256(borsh(sign_payload)) == response.payload_hash` — there is no second on-chain check of source-chain finality at settlement time:

```rust
if expected_hash != mpc_response.payload_hash {
    return Err(ProverError::InvalidPayloadHash.to_string());
}
``` [5](#0-4) 

The resulting `ProverResult` is consumed by `fin_transfer_callback` in the omni-bridge, which mints or releases tokens unconditionally once the prover returns success: [6](#0-5) 

---

### Impact Explanation
If a reorg removes the Abstract chain block that contained the `InitTransfer` event after the MPC network has already signed the proof, the MPC signature remains cryptographically valid. A trusted relayer can submit it to `fin_transfer`, which calls `verify_proof` → MPC prover → `fin_transfer_callback`, and the bridge releases or mints tokens on NEAR for a source deposit that no longer exists on the canonical Abstract chain. This produces unbacked wrapped supply — a direct unauthorized creation of bridge assets through verification failure.

---

### Likelihood Explanation
Abstract chain is a ZKsync-based EVM L2 with a centralized sequencer. Natural reorgs are infrequent but not impossible (sequencer restarts, network partitions). The window of exposure is the latency between MPC signing and NEAR transaction finality (seconds to low minutes). A sequencer operator who is also a bridge user can deliberately trigger a reorg to exploit this deterministically. Even without malicious sequencer behavior, the `EvmFinality::Latest` setting guarantees that every single Abstract-chain bridge proof is attested against a potentially-reorgable block, making the bridge structurally unsound for that chain.

---

### Recommendation
Change the hardcoded finality for `ChainKind::Abs` from `EvmFinality::Latest` to `EvmFinality::Safe` (or `EvmFinality::Finalized` if Abstract chain's RPC supports it). `Safe` corresponds to the checkpoint-validated head, which has negligible reorg probability. The `set_finality` function already exists for post-deployment correction:

```rust
// init():
finalities.insert(ChainKind::Abs, MpcFinality::Evm(EvmFinality::Safe));
``` [7](#0-6) 

---

### Proof of Concept

1. **User** calls the Abstract chain bridge contract `initTransfer(token, amount, recipient="near:alice")`. The event is included in block `N` (latest).
2. **Trusted relayer** observes the event and immediately calls `fin_transfer` on the NEAR omni-bridge with `chain_kind = Abs` and a `MpcVerifyProofArgs` whose embedded `EvmRpcRequest` carries `finality: EvmFinality::Latest` and `tx_id` pointing to the Abstract chain transaction.
3. NEAR omni-bridge calls `MpcOmniProver::verify_proof`. The finality check passes (`Latest == Latest`). The prover calls `mpc_contract.verify_foreign_transaction` with the request.
4. The MPC network nodes query the Abstract chain at `latest`, observe the transaction in block `N`, and return a signed `VerifyForeignTransactionResponse`. `verify_callback` validates the payload hash and returns `ProverResult::InitTransfer`.
5. **A reorg occurs** on Abstract chain (natural or sequencer-induced), replacing block `N` with a fork that does not contain the `initTransfer` transaction. The user's tokens are never actually locked.
6. `fin_transfer_callback` on NEAR has already received the valid `ProverResult` and mints/releases tokens to `alice` on NEAR.
7. Alice holds real NEAR-side tokens; the Abstract chain lock never existed. The bridge supply is unbacked by the amount of the transfer.

    pub fn fin_transfer_callback(
        &mut self,
        #[serializer(borsh)] storage_deposit_actions: &Vec<StorageDepositAction>,
        #[serializer(borsh)] predecessor_account_id: AccountId,
    ) -> PromiseOrValue<Nonce> {
        let Ok(ProverResult::InitTransfer(init_transfer)) = Self::decode_prover_result(0) else {
            env::panic_str(BridgeError::InvalidProofMessage.to_string().as_str())
        };
        require!(
            self.factories
                .get(&init_transfer.emitter_address.get_chain())
                == Some(init_transfer.emitter_address),
            BridgeError::UnknownFactory.as_ref()
        );

        let decimals = self
            .token_decimals
            .get(&init_transfer.token)
            .near_expect(BridgeError::TokenDecimalsNotFound);

        let destination_nonce =
            self.get_next_destination_nonce(init_transfer.recipient.get_chain());
        let transfer_message = TransferMessage {
            origin_nonce: init_transfer.origin_nonce,
            token: init_transfer.token,
            amount: Self::denormalize_amount(init_transfer.amount.0, decimals).into(),
            recipient: init_transfer.recipient,
            fee: Self::denormalize_fee(&init_transfer.fee, decimals),
            sender: init_transfer.sender,
            msg: init_transfer.msg,
            destination_nonce,
            origin_transfer_id: None,
        };

        if let OmniAddress::Near(recipient) = transfer_message.recipient.clone() {
            self.process_fin_transfer_to_near(
                recipient,
                &predecessor_account_id,
                transfer_message,
                storage_deposit_actions,
            )
            .into()
        } else {
            self.process_fin_transfer_to_other_chain(predecessor_account_id, transfer_message);
            PromiseOrValue::Value(destination_nonce)
        }
    }
