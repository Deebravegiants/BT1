### Title
Missing `curve.check()` after `from_bin` in `blob_to_keyset` allows Ed25519 infinity point to be accepted as a valid root public key — (`src/cbmpc/api/hd_keyset_eddsa_2p.cpp`)

### Summary
`blob_to_keyset` calls `ecc_point_t::from_bin` on the attacker-controlled `root_Q_compressed` field but never calls `curve.check()` afterward. The Ed25519 `from_bin` implementation successfully decodes the infinity encoding (y=1, x=0) and returns `SUCCESS`. As a result, `extract_root_public_key_compressed` returns `SUCCESS` with `out_Q_compressed` encoding the infinity point, and `derive_eddsa_2p_keys` proceeds into the 2PC derivation protocol with an invalid root key.

### Finding Description
`keyset_blob_v1_t` stores `root_Q_compressed` and `root_K_compressed` as raw `buf_t` fields. When `deserialize_keyset_blob` calls `coinbase::convert(blob, in)`, these fields are deserialized as opaque byte buffers — the `ecc_point_t::convert_fixed_curve` path (which calls `curve.check()`) is bypassed entirely. `blob_to_keyset` then calls `keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed)` and checks only the `error_t` return value. Since `ec25519_core::from_bin` accepts the infinity encoding (y=1 yields u=0, x=0, no error branch is taken), the call returns `SUCCESS` and the infinity point is stored in `keyset.root.Q`. No `curve.check()` is ever invoked, so the infinity rejection guard is completely absent from this path.

The same missing check applies to `root_K_compressed` (line 80) and to the analogous `deserialize_keyset_blob` call in `derive_eddsa_2p_keys`.

### Impact Explanation
Any caller of `extract_root_public_key_compressed` that trusts the `SUCCESS` return to mean the output is a valid, non-degenerate public key will receive the infinity point and may use it for signature verification (where it could accept arbitrary signatures depending on the verifier) or as a trust anchor for derived keys. More critically, `derive_eddsa_2p_keys` uses the same `deserialize_keyset_blob` path: a crafted blob with infinity as `root_Q` causes the 2PC derivation protocol to run with a degenerate root key, enabling public key substitution in derived `eddsa_2p` key blobs.

### Likelihood Explanation
The `keyset_blob` is an opaque byte string that callers persist and pass back to the API. Any party that can supply a crafted blob — including a storage-layer attacker or a Byzantine participant who generates their own blob — can trigger this with a single crafted 32-byte field. No threshold collusion is required; the attack is single-party and requires no network interaction for `extract_root_public_key_compressed`.

### Recommendation
Add `curve.check()` calls after each `from_bin` in `blob_to_keyset`:

```cpp
// src/cbmpc/api/hd_keyset_eddsa_2p.cpp, blob_to_keyset
error_t rv = keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed);
if (rv) return rv;
if (rv = keyset.curve.check(keyset.root.Q)) return coinbase::error(E_FORMAT, "invalid root Q");

rv = keyset.root.K.from_bin(keyset.curve, blob.root_K_compressed);
if (rv) return rv;
if (rv = keyset.curve.check(keyset.root.K)) return coinbase::error(E_FORMAT, "invalid root K");
```

This mirrors the existing guard in `ecc_point_t::convert_fixed_curve`.

### Proof of Concept
Craft a valid-looking `keyset_blob_v1_t` with:
- `version = 1`, `role = 0`, `curve = ed25519`
- `root_Q_compressed = [0x01, 0x00, ..., 0x00]` (32 bytes — Ed25519 infinity encoding)
- `root_K_compressed` = any valid non-infinity point
- `x_share`, `k_share` = any values in `[0, q)`

Serialize and call:
```cpp
buf_t out;
error_t rv = coinbase::api::hd_keyset_eddsa_2p::extract_root_public_key_compressed(crafted_blob, out);
assert(rv == SUCCESS);           // passes — should fail
assert(out == infinity_encoding); // out_Q_compressed is the infinity point
```

static error_t from_bin(point_t& R, mem_t bin) {
  if (bin.size != 32) return coinbase::error(E_FORMAT);

  buf_t buf = bin.rev();
  uint8_t neg = buf[0] >> 7;
  buf[0] &= 0x7f;
  fe_t y = fe_t::to_fe(uint256_t::from_bin(buf));

  // x² = (y² - 1) / (dy² + 1)

  fe_t u, v, w, vxx, check;

  u = y * y;
  v = u * formula_t::get_d();
  u -= fe_t::one();       // u = y^2-1
  v += fe_t::one();       // v = dy^2+1
  w = u * v;              // w = u*v
  fe_t x = w.pow22523();  // x = w^((q-5)/8)
  x *= u;                 // x = u * w^((q-5)/8)

  vxx = x * x;
  vxx *= v;
  check = vxx - u;  // vx^2-u
  if (!check.is_zero()) {
    check = vxx + u;  // vx^2+u
    if (!check.is_zero()) {
      return coinbase::error(E_CRYPTO);
    }
    static const fe_t sqrtm1 =
        fe_t::from_bn(bn_t::from_hex("2b8324804fc1df0b2b4d00993dfbd7a72f431806ad2fe478c4ee1b274a0ea0b0"));
    x *= sqrtm1;
  }

  uint256_t x_val = x.from_fe();
  if (neg != (x_val.w0 & 1)) x = -x;

  R.x = x;
  R.y = y;
  R.z = fe_t::one();
  return 0;
}
