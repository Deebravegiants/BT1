Audit Report

## Title
`ChainlinkPriceOracle.getAssetPrice()` Accepts Stale Prices, Enabling Excess rsETH Minting via Depressed `rsETHPrice` - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` discards all validity metadata from `latestRoundData()`, accepting stale or incomplete Chainlink rounds without any check. Because `LRTOracle.updateRSETHPrice()` is publicly callable with no role restriction and `pricePercentageLimit` defaults to `0`, an attacker can commit a stale-depressed `rsETHPrice` on-chain and immediately deposit ETH (whose price is always `1e18`) to receive more rsETH than their deposit warrants, diluting all existing rsETH holders.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` reads only the raw `price` field:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L49-55
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

It silently ignores `roundId`, `updatedAt`, and `answeredInRound`. No check exists for `answeredInRound < roundId` (stale round), `updatedAt == 0` (incomplete round), `price <= 0` (invalid price), or time-based staleness.

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` performs all three structural checks:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The stale price propagates through the following confirmed call chain:

1. `LRTOracle._getTotalEthInProtocol()` (L339) calls `getAssetPrice(asset)` for each supported LST, returning the stale low price.
2. `_updateRsETHPrice()` (L250) computes `newRsETHPrice = totalETHInProtocol / rsethSupply` using the understated TVL.
3. `updateRSETHPrice()` (L87) is `public whenNotPaused` — any address can commit the stale price on-chain.
4. The `pricePercentageLimit` guard (L256-257) is gated on `pricePercentageLimit > 0`; the variable is declared as `uint256 public pricePercentageLimit` and defaults to `0`, meaning the circuit-breaker is inactive by default.
5. The downside-protection pause (L273-274) is similarly gated on `pricePercentageLimit > 0` and therefore also inactive by default.
6. `LRTDepositPool.getRsETHAmountToMint()` (L520) uses the now-incorrect `rsETHPrice` as the denominator: `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`.

## Impact Explanation
**High — Theft of unclaimed yield (dilution of existing rsETH holders).**

If a Chainlink LST/ETH feed (e.g., stETH/ETH) goes stale at a price 5% below the true market price:

- `_getTotalEthInProtocol()` underestimates protocol TVL.
- `rsETHPrice` is stored ~5% below its true value.
- An attacker depositing ETH (whose price is always `1e18` via `OneETHPriceOracle`, unaffected by the stale LST feed) receives `amount × 1e18 / depressedRsETHPrice` rsETH — more than the true exchange rate entitles them to.
- The excess rsETH represents a proportional claim on the protocol's TVL that was not contributed by the attacker, directly diluting the share of all existing rsETH holders. This constitutes theft of accrued yield from existing holders.

## Likelihood Explanation
**Medium.** Chainlink feeds go stale during network congestion, oracle node failures, or extreme market events. No special privilege is required: `updateRSETHPrice()` is publicly callable. An attacker monitoring Chainlink round data can detect a stale round and atomically call `updateRSETHPrice()` followed by `depositETH()` in the same block, locking in the favorable rate before the oracle recovers. The attack is repeatable for as long as the feed remains stale.

## Recommendation
Apply the same validation pattern already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    if (block.timestamp - updatedAt > MAX_STALENESS_PERIOD) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, ensure `pricePercentageLimit` is set to a non-zero value at deployment so the `_updateRsETHPrice()` circuit-breaker and downside-protection pause are active.

## Proof of Concept
1. A Chainlink stETH/ETH feed goes stale — `answeredInRound < roundId` — with the last committed price 5% below the true market price.
2. Attacker calls `LRTOracle.updateRSETHPrice()` (no access control, `public whenNotPaused`). Because `pricePercentageLimit == 0`, neither the upside threshold nor the downside pause triggers.
3. `_getTotalEthInProtocol()` calls `getAssetPrice(stETH)` via `ChainlinkPriceOracle`, which returns the stale low price without reverting.
4. `rsETHPrice` is set ~5% below its true value and stored on-chain.
5. Attacker calls `LRTDepositPool.depositETH{value: amount}(0, "")`. ETH price is always `1e18` (via `OneETHPriceOracle`), unaffected by the stale stETH feed.
6. `getRsETHAmountToMint = (amount × 1e18) / depressedRsETHPrice` yields ~5.26% more rsETH than the true exchange rate entitles the attacker to.
7. Attacker holds or sells the excess rsETH; when the oracle recovers and `rsETHPrice` is corrected upward, existing holders' per-token TVL claim is permanently diluted by the excess supply minted in step 6.

**Foundry fork test outline:**
- Fork mainnet at a block where a stETH/ETH Chainlink round is stale (`answeredInRound < roundId`).
- Deploy or reference the live `ChainlinkPriceOracle` and `LRTOracle`.
- Record `rsETHPrice` before and after calling `updateRSETHPrice()`.
- Assert `rsETHPrice` decreased relative to the true price.
- Deposit ETH as the attacker and assert `rsethAmountToMint > amount * 1e18 / trueRsETHPrice`.
