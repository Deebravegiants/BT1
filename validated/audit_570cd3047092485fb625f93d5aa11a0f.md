### Title
Missing State Diff Commitment Verification Allows Malicious P2P Peer to Corrupt Syncing Node State — (`crates/apollo_p2p_sync/src/client/state_diff.rs`, `crates/apollo_protobuf/src/converters/state_diff.rs`)

---

### Summary

The p2p sync client assembles a `ThinStateDiff` from peer-supplied `StateDiffChunk` messages and writes it to storage, but **never verifies the assembled diff against the `state_diff_commitment` stored in the block header**. The only guard is a `state_diff_length` count check. Because both `DeclaredClass` and `DeprecatedDeclaredClass` chunks each count as length 1, a malicious peer can substitute a `DeclaredClass{class_hash=X, compiled_class_hash=Some(ZERO)}` for a block whose canonical state has `deprecated_declared_classes=[X]`. The length check passes, the wrong diff is committed to storage, and the node permanently diverges from canonical state.

---

### Finding Description

**Step 1 — Protobuf routing is purely `Some`/`None` on `compiled_class_hash`.** [1](#0-0) 

Any non-`None` value — including `Some(ZERO)` — routes to `StateDiffChunk::DeclaredClass`. Zero is a valid `Felt`, so `try_into()` succeeds and `CompiledClassHash(ZERO)` is accepted.

**Step 2 — Both chunk variants have identical length.** [2](#0-1) 

`StateDiffChunk::DeclaredClass(_) => 1` and `StateDiffChunk::DeprecatedDeclaredClass(_) => 1`. The length-based guard in `parse_data_for_block` cannot distinguish them.

**Step 3 — `parse_data_for_block` only checks `state_diff_length`, not `state_diff_commitment`.** [3](#0-2) 

The function reads `state_diff_length` from the stored header and loops until the count matches. After the loop it calls `validate_deprecated_declared_classes_non_conflicting` (a duplicate-check only) and returns. There is no call to `calculate_state_diff_hash` and no comparison against `header.state_diff_commitment`.

**Step 4 — The wrong diff is written directly to storage.** [4](#0-3) 

`write_to_storage` calls `append_state_diff` unconditionally. No commitment check occurs here either.

**Step 5 — `calculate_state_diff_hash` hashes `class_hash_to_compiled_class_hash` and `deprecated_declared_classes` into completely different positions.** [5](#0-4) 

`chain_declared_classes` contributes `[count, class_hash, compiled_class_hash, ...]` while `chain_deprecated_declared_classes` contributes `[count, class_hash, ...]`. A diff with `{class_hash_to_compiled_class_hash: {X: ZERO}}` produces a provably different Poseidon hash than `{deprecated_declared_classes: [X]}`.

**Step 6 — The `state_diff_commitment` in the block header is never used to validate p2p-received state diffs.**

The `verify_state_diff_hash` guard exists only in the batcher/committer path: [6](#0-5) 

That path is not exercised during p2p sync. The p2p sync client has no equivalent check.

---

### Impact Explanation

A syncing node stores `class_hash_to_compiled_class_hash={(X, ZERO)}` instead of `deprecated_declared_classes=[X]`. This is a wrong class-type mapping: the node believes class X is a Cairo 1 Sierra class with compiled hash ZERO, while the canonical chain treats it as a Cairo 0 deprecated class. Any subsequent contract execution that looks up class X will use the wrong class type, producing wrong execution results. The node permanently diverges from canonical state with no self-healing mechanism, because the bad diff is committed to storage and the commitment mismatch is never detected.

---

### Likelihood Explanation

Any unauthenticated p2p peer can send crafted `StateDiffsResponse` messages. The attacker only needs to know which blocks contain deprecated class declarations (publicly visible on-chain) and send the corresponding `DeclaredClass` message with `compiled_class_hash=Some(ZERO)`. No privileged access is required.

---

### Recommendation

In `parse_data_for_block` (or immediately before `write_to_storage`), after assembling the full `ThinStateDiff`, compute `calculate_state_diff_hash(&result)` and compare it against `header.state_diff_commitment`. If they differ, return `ParseDataError::BadPeer(...)` and report the peer. This is the same pattern already used in the committer: [7](#0-6)

### Proof of Concept

```rust
use indexmap::indexmap;
use starknet_api::block_hash::state_diff_hash::calculate_state_diff_hash;
use starknet_api::core::{ClassHash, CompiledClassHash};
use starknet_api::state::ThinStateDiff;
use starknet_types_core::felt::Felt;

#[test]
fn deprecated_vs_declared_zero_hash_differ() {
    let x = ClassHash(Felt::from(0xdeadu64));

    // What the honest block contains
    let canonical = ThinStateDiff {
        deprecated_declared_classes: vec![x],
        ..Default::default()
    };

    // What a malicious peer injects (compiled_class_hash = Some(ZERO))
    let malicious = ThinStateDiff {
        class_hash_to_compiled_class_hash: indexmap! { x => CompiledClassHash(Felt::ZERO) },
        ..Default::default()
    };

    // Both have len() == 1, so the length guard passes
    assert_eq!(canonical.len(), malicious.len());

    // But the commitments differ — the node stores the wrong one
    assert_ne!(
        calculate_state_diff_hash(&canonical),
        calculate_state_diff_hash(&malicious),
        "Commitments must differ; the malicious diff corrupts state"
    );
}
```

`canonical.len() == malicious.len() == 1` confirms the length guard is bypassed, while the differing `StateDiffCommitment` values confirm the stored state diverges from the canonical block.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
