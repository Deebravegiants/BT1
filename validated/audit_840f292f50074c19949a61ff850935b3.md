### Title
Missing `forceResumeReceive` Enables Permanent Channel Block Freezing Rate Updates — (`contracts/cross-chain/CrossChainRateReceiver.sol`)

---

### Summary

`CrossChainRateReceiver` implements only `ILayerZeroReceiver` and omits `ILayerZeroUserApplicationConfig`. Under LayerZero v1's blocking delivery model, any revert inside `lzReceive` causes the endpoint to store the failed payload and block all subsequent messages on that channel. Without `forceResumeReceive`, there is no on-chain path to clear the stored payload, permanently freezing rate updates and all yield accrual that depends on them.

---

### Finding Description

`CrossChainRateReceiver` is declared as:

```solidity
abstract contract CrossChainRateReceiver is ILayerZeroReceiver, Ownable {
``` [1](#0-0) 

It does **not** implement `ILayerZeroUserApplicationConfig`. The `forceResumeReceive` function exists only in the interface file and is never implemented anywhere in the receiver hierarchy: [2](#0-1) 

The `lzReceive` implementation contains three `require` guards that can revert:

```solidity
require(msg.sender == layerZeroEndpoint, "Sender should be lz endpoint");
require(_srcChainId == srcChainId, "Src chainId must be correct");
require(srcAddress == rateProvider, "Src address must be provider");
``` [3](#0-2) 

Additionally, `abi.decode(_payload, (uint256))` at line 93 will revert on a malformed payload, and an out-of-gas condition during delivery will also cause a revert. [4](#0-3) 

Under LZ v1's blocking model, any such revert causes the endpoint to call `storedPayload[srcChainId][srcAddress] = ...` and block all future deliveries on that channel. The only recovery mechanism is for the UA to call `ILayerZeroEndpoint.forceResumeReceive(srcChainId, srcAddress)` — which requires the UA to expose a function that proxies this call. `CrossChainRateReceiver` exposes no such function, leaving the channel permanently blocked with no on-chain recovery.

This affects all concrete receiver deployments: `RSETHRateReceiver` and `AGETHRateReceiver`, both of which inherit `CrossChainRateReceiver` without adding `forceResumeReceive`. [5](#0-4) 

---

### Impact Explanation

The `rate` stored in `CrossChainRateReceiver` is consumed by pool contracts (`RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, `RSETHPoolV3WithNativeChainBridge`) to compute swap amounts:

```solidity
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
``` [6](#0-5) 

A frozen rate means the pool permanently uses a stale rsETH/ETH exchange rate. Users minting rsETH on destination chains receive incorrect amounts — either overpaying or underpaying — and all yield accrual reflected in the rate (staking rewards accumulating on mainnet) is never propagated to destination chains. This matches the **Medium — Permanent freezing of unclaimed yield** impact.

---

### Likelihood Explanation

A revert in `lzReceive` can occur via:

1. **Out-of-gas**: If the relayer forwards insufficient gas (e.g., gas estimation error or deliberate griefing), the call reverts with OOG. This is a known LZ v1 risk and has occurred in production deployments of similar contracts.
2. **Misconfiguration during upgrade**: If `srcChainId` or `rateProvider` is updated on one side but not the other (e.g., during a provider migration), the `require` checks at lines 90–91 will revert on the next delivery, blocking the channel.

Both scenarios are realistic and require no privileged access beyond what is already part of normal operation or relayer behavior.

---

### Recommendation

Implement `ILayerZeroUserApplicationConfig` in `CrossChainRateReceiver` and add a `forceResumeReceive` function that proxies to the LZ endpoint, restricted to `onlyOwner`:

```solidity
import { ILayerZeroUserApplicationConfig } from ".../ILayerZeroUserApplicationConfig.sol";

abstract contract CrossChainRateReceiver is
    ILayerZeroReceiver,
    ILayerZeroUserApplicationConfig,
    Ownable
{
    function forceResumeReceive(
        uint16 _srcChainId,
        bytes calldata _srcAddress
    ) external override onlyOwner {
        ILayerZeroEndpoint(layerZeroEndpoint).forceResumeReceive(
            _srcChainId,
            _srcAddress
        );
    }
    // setConfig, setSendVersion, setReceiveVersion can be no-ops or also proxied
}
```

This matches the pattern recommended in the LZ v1 documentation and used by `NonblockingLzApp` in the official LZ SDK.
