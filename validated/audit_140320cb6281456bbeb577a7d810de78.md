Audit Report

## Title
Missing Chainlink `latestRoundData` Staleness and Validity Checks Allow Stale/Invalid Prices to Corrupt rsETH Exchange Rate - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `roundId`, `updatedAt`, and `answeredInRound`, performing no staleness, incomplete-round, or non-positive price validation. A stale or zero Chainlink price for any supported LST asset propagates through `LRTOracle._getTotalEthInProtocol()` into `rsETHPrice`, corrupting the exchange rate used for all deposits and withdrawals. The same repository already applies the correct validation pattern in `ChainlinkOracleForRSETHPoolCollateral`, confirming the omission is unintentional.

## Finding Description
In `contracts/oracles/ChainlinkPriceOracle.sol` at L52, `getAssetPrice()` discards all validation fields:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No checks are performed for:
- `price > 0` — a zero price silently zeroes the asset's TVL contribution; a negative price causes `uint256(negativeValue)` to wrap to a huge number, overflowing `mulWad` and reverting `updateRSETHPrice()` entirely (DoS).
- `updatedAt != 0` — an incomplete round returns `updatedAt == 0` with a potentially stale `answer`.
- `answeredInRound >= roundId` — a carried-over answer from a prior round indicates a stale price.

The call chain is:
1. `ChainlinkPriceOracle.getAssetPrice()` (L49–55) — returns stale/invalid price.
2. `LRTOracle.getAssetPrice()` (L156–158) — delegates to the above.
3. `LRTOracle._getTotalEthInProtocol()` (L336–348) — accumulates `totalAssetAmt.mulWad(assetER)` for all supported assets using the corrupted price.
4. `LRTOracle._updateRsETHPrice()` (L250) — computes `newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply)`.
5. `rsETHPrice = newRsETHPrice` (L313) — the corrupted value is stored.

The entry point `updateRSETHPrice()` (L87) is `public whenNotPaused`, callable by any unprivileged address.

By contrast, `contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol` at L30–32 correctly validates all three conditions:
```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The existing `pricePercentageLimit` downside-protection check (L270–282) does not mitigate this: if `pricePercentageLimit` is zero (unset), no pause is triggered and the corrupted price is written unconditionally. Even when set, a stale price within the threshold window is written silently.

## Impact Explanation
**High — Theft of unclaimed yield.**

A stale Chainlink price for any supported LST (stETH, cbETH, rETH) causes `_getTotalEthInProtocol()` to return an incorrect TVL:
- **Inflated stale price**: `rsETHPrice` is overstated. New depositors receive fewer rsETH shares than owed (value extracted from depositors). Existing holders' shares are diluted in reverse — theft of unclaimed yield from new depositors.
- **Deflated stale price**: `rsETHPrice` is understated. New depositors receive more rsETH than owed, draining value from existing holders — theft of unclaimed yield from existing holders.
- **Zero price** (incomplete round): the asset's entire TVL contribution is zeroed, collapsing `rsETHPrice` and, if `pricePercentageLimit > 0`, triggering the downside-protection pause — temporary freezing of funds.

## Likelihood Explanation
Chainlink feeds go stale during network congestion, sequencer downtime (on L2s), or feed deprecation. No special privileges are required: any external caller can invoke the public `updateRSETHPrice()` at any time. The vulnerability is passively exploitable — a caller need only invoke the function during a period of feed staleness. This is a known, recurring real-world condition, not a theoretical one.

## Recommendation
Apply the same validation pattern already present in `ChainlinkOracleForRSETHPoolCollateral.sol`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    require(price > 0, "Chainlink price <= 0");
    require(updatedAt != 0, "Incomplete round");
    require(answeredInRound >= roundId, "Stale price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, add a heartbeat staleness check: `require(block.timestamp - updatedAt <= MAX_STALENESS, "Price too stale")`, with `MAX_STALENESS` tuned per feed (e.g., 3600s for 1-hour heartbeat feeds).

## Proof of Concept
**Step 1 — Setup**: Deploy or fork with a supported LST asset (e.g., stETH) whose Chainlink feed is configured in `ChainlinkPriceOracle`.

**Step 2 — Simulate staleness**: In a Foundry fork test, mock `latestRoundData()` on the price feed to return `answeredInRound < roundId` (stale) or `updatedAt == 0` (incomplete round) with a price deviating from the true market price by, say, 5%.

**Step 3 — Trigger**: Call `LRTOracle.updateRSETHPrice()` as an unprivileged address.

**Step 4 — Observe**: Read `rsETHPrice` before and after. Confirm it reflects the stale/incorrect price. Simulate a deposit at the corrupted rate and verify the rsETH minted differs from the correct amount, quantifying the yield theft.

**Foundry test sketch**:
```solidity
function test_stalePrice_corruptsRsETHPrice() public {
    // Mock feed to return stale answeredInRound
    vm.mockCall(priceFeed, abi.encodeWithSelector(AggregatorV3Interface.latestRoundData.selector),
        abi.encode(uint80(10), int256(0.95e18), 0, block.timestamp - 2 hours, uint80(9)));
    
    uint256 priceBefore = lrtOracle.rsETHPrice();
    lrtOracle.updateRSETHPrice(); // public, no privilege needed
    uint256 priceAfter = lrtOracle.rsETHPrice();
    
    assertNotEq(priceBefore, priceAfter); // price corrupted by stale feed
}
```
