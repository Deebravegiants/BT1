### Title
Vote Signature Never Verified in Consensus — Any Peer Can Forge Validator Votes to Manipulate Quorum - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`SingleHeightConsensus::handle_vote` accepts every inbound prevote and precommit without verifying the cryptographic signature on the `Vote` struct. The only admission check is that `vote.voter` appears in the committee member list — a public set. Any network peer can craft a `Vote` with `voter` set to any known validator address and `signature` set to `RawSignature::default()` (the all-zero sentinel that the state machine itself uses when signing is not yet implemented), and the vote will be counted toward quorum as if it came from that validator.

### Finding Description

`handle_vote` in `SingleHeightConsensus` contains an explicit unimplemented guard:

```rust

    // TODO(Asmaa): verify the signature   ← guard is absent
    ...

        return VecDeque::new();

    // vote is accepted and forwarded to the state machine

The only guard is a membership check on `vote.voter`, which is a public field that any sender can set to any known validator address. The `signature` field is never read after deserialization.

The state machine's own `make_self_vote` confirms that signing is also unimplemented — it writes `signature: RawSignature::default()` into every outbound vote:

```rust

The `verify_precommit_vote_signature` and `verify_identity` functions exist in `apollo_signature_manager` and are tested in isolation, but they are **never called** from the consensus vote-handling path — confirmed by a grep showing zero call sites outside the signature manager's own test file. ### Impact Explanation

An attacker who can send broadcast messages on the consensus vote channel (any connected peer) can:

1. Enumerate the public committee member addresses for the current height.
2. Craft `Vote` structs with `voter = <any_validator_address>`, `signature = RawSignature::default()`, and `proposal_commitment = <target_commitment>`.
3. Inject enough forged prevotes and precommits to satisfy the Byzantine quorum threshold (>2/3 of total weight).
4. The local node's state machine reaches `DecisionReached` for the attacker-chosen `proposal_commitment` and calls `context.decision_reached(height, round, commitment, ...)`.

This maps to the **High** impact category: consensus admission accepts invalid (forged) votes before sequencing, and can cause a node to commit a block commitment that was never legitimately agreed upon by the real validator set.

### Likelihood Explanation

The committee member addresses are public (derived from staking state). The `RawSignature::default()` value is the exact value the node itself uses for outbound votes, so no cryptographic material is needed. Any peer reachable on the consensus broadcast topic can trigger this. The `TODO(Asmaa): verify the signature` comment confirms the gap is known but unguarded.

### Recommendation

Before forwarding a vote to the state machine, call `verify_precommit_vote_signature` (or the equivalent for prevotes) using the public key associated with `vote.voter` from the committee roster. Reject and report the peer if verification fails or returns `false`. The infrastructure (`verify_precommit_vote_signature`, `verify_identity`, `PublicKey` in committee members) already exists; it only needs to be wired into `handle_vote`.

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(

                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }

            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
