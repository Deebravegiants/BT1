### Title
Missing RSA Modulus Compositeness Check Allows Trivial Private-Key Recovery - (`src/cbmpc/api/pve_base_pke.cpp`)

### Summary

`base_pke_rsa_ek_from_modulus` accepts any 256-byte non-zero integer as an RSA modulus without verifying it is composite. An attacker who supplies a 2048-bit prime as the modulus receives a valid ek blob, can compute the RSA private exponent trivially (since φ(p) = p−1 for a prime p), and recovers any scalar `x` encrypted under that ek without the HSM private key.

### Finding Description

`base_pke_rsa_ek_from_modulus` performs only two checks on the caller-supplied modulus: [1](#0-0) 

1. Byte length is exactly 256 (RSA-2048 size).
2. Value is not zero.

No compositeness test, no minimum-bit-length check, and no check that the modulus is a product of two large primes is performed. After these two checks pass, the raw BIGNUM is handed directly to `rsa_pub_key_t::set`: [2](#0-1) 

`rsa_pub_key_t::set` calls `EVP_PKEY_fromdata` with only `n` and `e`: [3](#0-2) 

OpenSSL's `EVP_PKEY_fromdata` for a public key does not perform a primality or compositeness check on `n`. The resulting ek blob is indistinguishable from a legitimate one and is accepted by `encrypt` without further modulus validation: [4](#0-3) 

### Impact Explanation

If an attacker supplies a 2048-bit prime `p` as the modulus:

- φ(p) = p − 1 (trivially computable since p is prime)
- d = e⁻¹ mod (p − 1) where e = 65537 is computable in milliseconds
- The attacker now holds the full RSA private key and can OAEP-decrypt any ciphertext produced by `encrypt` under that ek, recovering the encrypted scalar `x`

This directly satisfies the Critical impact scope: a shipped API path lets an attacker recover a private scalar without the HSM private key.

### Likelihood Explanation

The function is explicitly designed for the HSM integration path where the modulus is an externally supplied byte string. Any caller who controls the bytes passed as `modulus` — including a malicious integration layer, a compromised HSM driver, or a Byzantine participant who provides the ek to an honest encryptor — can trigger this path. The API is public and documented in both the C++ and C headers. [5](#0-4) [6](#0-5) 

### Recommendation

Add a compositeness check inside `base_pke_rsa_ek_from_modulus` before constructing the key blob. OpenSSL provides `BN_check_prime` (or `BN_is_prime_fasttest_ex`) for this purpose. The check should:

1. Reject the modulus if it is prime (i.e., `BN_check_prime` returns 1).
2. Reject the modulus if its actual bit length is less than 2048 (leading-zero padding in the 256-byte encoding would produce a shorter effective modulus).
3. Optionally, reject if the modulus is even (trivially factorable).

The analogous ECIES path already performs a curve-point validity check (`curve_p256.check(pk)`) before accepting the public key: [7](#0-6) 

The RSA path should apply the same principle.

// 5. Attacker RSA-OAEP decrypts the KEM ciphertext inside `ciphertext` using d.

  if (const error_t rv = coinbase::api::detail::validate_mem_arg(label, "label")) return rv;
  if (const error_t rv = coinbase::api::detail::validate_mem_arg(x, "x")) return rv;

  const auto icurve = to_internal_curve(curve);
  if (!icurve.valid()) return coinbase::error(E_BADARG, "unsupported curve");

  // Defensive check: avoid large attacker-controlled allocations when converting `x` into a bignum.
  const int max_x_size = icurve.order().get_bin_size();
  if (x.size > max_x_size) return coinbase::error(E_RANGE, "x too large");

  base_pke_bridge_t bridge(base_pke);
  coinbase::mpc::ec_pve_t pve_ct;

  const coinbase::mem_t ek_mem(ek.data, ek.size);
  const coinbase::crypto::bn_t x_bn = coinbase::crypto::bn_t::from_bin(x);

  out_ciphertext.free();
  rv = pve_ct.encrypt(bridge, coinbase::mpc::pve_keyref(ek_mem), label, icurve, x_bn);
  if (rv) return rv;

  pve_ciphertext_blob_v1_t blob;
  blob.ct = coinbase::convert(pve_ct);
  out_ciphertext = coinbase::convert(blob);

  scoped_ptr_t<EVP_PKEY_CTX> ctx = EVP_PKEY_CTX_new_from_name(NULL, "RSA", NULL);

  OSSL_PARAM_BLD_free(param_bld);
