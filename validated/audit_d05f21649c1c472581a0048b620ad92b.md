### Title
Integer Rounding in `translate_threshold` Causes Robust ECDSA Signing to Always Fail for Even Governance Thresholds - (File: crates/node/src/providers/robust_ecdsa.rs)

### Summary

`translate_threshold` converts the governance threshold to a `MaxMalicious` value via integer division `(num_signers - 1) / 2`, which rounds down. Presignature generation selects exactly `threshold` participants, but the signing protocol enforces a strict equality `participants.len() == 2 * max_malicious + 1`. For any even governance threshold, these two values differ by exactly 1, causing every robust ECDSA signing attempt to fail unconditionally.

### Finding Description

`translate_threshold` in `crates/node/src/providers/robust_ecdsa.rs` computes:

```rust
Ok(MaxMalicious::from((number_of_signers - 1) / 2))
``` [1](#0-0) 

`get_number_of_signers` returns `threshold` unchanged, so `num_signers = threshold`. [2](#0-1) 

Presignature generation then selects exactly `num_signers = threshold` participants: [3](#0-2) 

At signing time, the threshold-signatures `sign` function enforces a **strict equality**:

```rust
if participants.len() != robust_ecdsa_threshold {
    return Err(InitializationError::BadParameters(
        "the number of participants during signing must be exactly 2*max_malicious+1 ...".to_string(),
    ));
}
``` [4](#0-3) 

where `robust_ecdsa_threshold = 2 * max_malicious + 1`.

**Arithmetic trace for even threshold (e.g., `threshold = 6`):**

| Step | Value |
|---|---|
| `num_signers` | 6 |
| `max_malicious = (6 - 1) / 2` | **2** (rounds down) |
| `2 * max_malicious + 1` | **5** |
| `participants.len()` | **6** |
| Equality check `6 == 5` | **FAILS** |

The intermediate `compute_thresholds` guard only checks `2 * max_malicious + 1 <= num_signers` (i.e., `5 <= 6`), which passes, so the mismatch is not caught before signing. [5](#0-4) 

### Impact Explanation

For any even governance threshold, every robust ECDSA signing attempt returns `BadParameters` from the threshold-signatures layer. Presignatures are consumed (removed from storage) before the signing call, so each failed attempt permanently wastes a presignature. Cross-chain operations that depend on a robust ECDSA signature cannot complete, and any funds locked pending such a signature are permanently frozen. This breaks the core production safety invariant that a validly configured MPC network can produce signatures.

### Likelihood Explanation

Even governance thresholds are valid and reachable. The minimum governance threshold is `ceil(0.6 * n)`:

- `n = 9` → min threshold = **6** (even) → signing always fails
- `n = 10` → min threshold = **6** (even) → signing always fails
- `n = 13` → min threshold = **8** (even) → signing always fails

No privileged access is required; the failure is triggered by any normal signature request once the network is configured with an even threshold. The attacker-controlled entry point is the public `sign` contract method, which enqueues a request that the MPC nodes will attempt and fail to fulfill.

### Recommendation

Replace the floor division in `translate_threshold` with a selection that guarantees `2 * max_malicious + 1 == num_signers`. The simplest fix is to derive `max_malicious` from the desired signer count rather than from the governance threshold directly, and to select `2 * max_malicious + 1` participants during presignature generation instead of `threshold` participants. Alternatively, add a validation in `compute_thresholds` that rejects even thresholds for robust ECDSA domains, forcing operators to choose an odd threshold when deploying a DamgardEtAl domain.

### Proof of Concept

```rust
// Reproduces the rounding mismatch for threshold = 6
fn poc_even_threshold_rounding() {
    let threshold: usize = 6; // valid governance threshold for n=9 or n=10
    let number_of_participants: usize = 9;

    // translate_threshold path
    let num_signers = threshold; // get_number_of_signers returns threshold unchanged
    let max_malicious = (num_signers - 1) / 2; // = 2 (rounds down)

    // presignature generation selects num_signers participants
    let presig_participant_count = num_signers; // = 6

    // signing check: participants.len() must equal 2 * max_malicious + 1
    let required = 2 * max_malicious + 1; // = 5

    assert_ne!(
        presig_participant_count, required,
        "signing will always fail: {} participants selected but {} required",
        presig_participant_count, required
    );
    // assertion passes: 6 != 5, confirming the signing error is unconditional
}
```

fn compute_thresholds(
    governance_threshold: u64,
    num_running_participants: usize,
) -> anyhow::Result<(usize, MaxMalicious)> {
    let governance_threshold: usize = governance_threshold.try_into()?;
    let num_signers = get_number_of_signers(governance_threshold, num_running_participants)?;
    let robust_ecdsa_threshold =
        translate_threshold(governance_threshold, num_running_participants)?;
    anyhow::ensure!(
        robust_ecdsa_threshold
            .value()
            .checked_mul(2)
            .and_then(|v| v.checked_add(1))
            .is_some_and(|v| v <= num_signers)
    );
    Ok((num_signers, robust_ecdsa_threshold))
}
