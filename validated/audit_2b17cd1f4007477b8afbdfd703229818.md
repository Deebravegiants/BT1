Audit Report

## Title
`setPoolAdminFeeDestination` Redirects Accrued Admin Fees to New Destination Without Prior Flush — (File: metric-core/contracts/MetricOmmPoolFactory.sol)

## Summary
`MetricOmmPoolFactory.setPoolAdminFeeDestination` overwrites `poolAdminFeeDestination[pool]` without first calling `collectFees` to flush pending admin fees to the old destination. Every other fee-parameter mutator (`setPoolAdminFees`, `setPoolProtocolFee`) explicitly collects fees before mutating state. After the overwrite, any call to the permissionless `collectPoolFees` routes all previously accrued admin fees — both spread surplus and `notionalFeeToken0Scaled`/`notionalFeeToken1Scaled` accumulators — to the new destination, causing a direct, irreversible loss for the old fee destination.

## Finding Description
`setPoolAdminFeeDestination` at L438-447 performs a bare storage write with no pre-collection:

```solidity
poolAdminFeeDestination[pool] = newAdminFeeDestination;  // L445 — no flush
```

By contrast, `setPoolAdminFees` (L418-425) and `setPoolProtocolFee` (L328-335) both call `IMetricOmmPoolCollectFees(pool).collectFees(...)` with the old destination before any state mutation.

`collectPoolFees` (L379-389) is permissionless and passes the current `poolAdminFeeDestination[pool]` directly to `collectFees`. Inside `collectFees` (L416-421), all admin-share tokens (spread surplus split + notional fee accumulators) are transferred to `adminFeeDestination_`. The notional accumulators (`notionalFeeToken0Scaled`, `notionalFeeToken1Scaled`) are then zeroed (L429-430), making the misdirection permanent and irreversible.

Exploit path:
1. Pool operates with `adminFeeDestination = Alice`; swaps accumulate `notionalFeeToken0Scaled`/`notionalFeeToken1Scaled` and spread surplus.
2. Pool admin calls `setPoolAdminFeeDestination(pool, Bob)`. No fees are collected; `poolAdminFeeDestination[pool]` is overwritten to `Bob`.
3. Anyone (keeper, bot, attacker) calls `collectPoolFees(pool)`. Factory reads `poolAdminFeeDestination[pool] == Bob` and passes it to `collectFees`.
4. All admin-share tokens are transferred to `Bob`; accumulators are zeroed. Alice receives nothing.

The two-step admin transfer (`proposePoolAdminTransfer` / `acceptPoolAdmin`) makes this especially reachable: a new admin accepting the role can immediately call `setPoolAdminFeeDestination` to capture all fees accrued under the previous admin's tenure.

## Impact Explanation
Direct, irreversible loss of real token balances (token0 and token1) for the old admin fee destination. The magnitude equals the full admin share of all spread surplus and notional fee accumulators outstanding at the time of the destination change. This is a direct loss of owed fee assets, matching the allowed impact gate for Critical/High direct loss of protocol fees above Sherlock thresholds.

## Likelihood Explanation
Fee destination changes are routine operations (treasury rotation, multisig migration, admin handover). The two-step admin transfer flow makes it especially likely that a new admin immediately sets a new fee destination, capturing all prior accruals. `collectPoolFees` is permissionless, so any third party can trigger the misdirected payout at any time after the destination is changed, requiring no further privileged action.

## Recommendation
Add a `collectFees` call at the top of `setPoolAdminFeeDestination`, mirroring the pattern in `setPoolAdminFees` and `setPoolProtocolFee`:

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();

    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool).collectFees(
        c.protocolSpreadFeeE6,
        c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8,
        c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]   // old destination
    );

    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

## Proof of Concept
1. Deploy pool with `adminFeeDestination = Alice`.
2. Execute multiple swaps to accumulate `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` and spread surplus.
3. Pool admin calls `setPoolAdminFeeDestination(pool, Bob)` — no fees collected, destination overwritten.
4. Call `collectPoolFees(pool)` (permissionless). Factory passes `Bob` to `collectFees`.
5. Assert: `Bob` receives all admin-share tokens; `Alice` receives zero; `notionalFeeToken0Scaled == 0` and `notionalFeeToken1Scaled == 0`.

Foundry test: snapshot balances of Alice and Bob before step 3, call `setPoolAdminFeeDestination`, call `collectPoolFees`, assert `Alice.balance == snapshot` (no change) and `Bob.balance > snapshot` (received all accrued admin fees).
