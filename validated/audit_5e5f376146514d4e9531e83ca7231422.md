Audit Report

## Title
`RSETHPriceFeed.latestRoundData()` Returns ETH/USD Staleness Metadata for a Composite rsETH/USD Price ‚Äî (`contracts/oracles/RSETHPriceFeed.sol`)

## Summary

`RSETHPriceFeed` implements `AggregatorV3Interface` and computes a composite rsETH/USD price by multiplying the ETH/USD Chainlink price by `RS_ETH_ORACLE.rsETHPrice()`. However, the `updatedAt` and `answeredInRound` fields returned by `latestRoundData()` are sourced exclusively from the ETH/USD Chainlink feed and carry no information about when the rsETH component was last updated. Any downstream integrator performing a standard Chainlink staleness check will be misled into treating a stale rsETH/USD price as fresh.

## Finding Description

`RSETHPriceFeed` declares itself as an `AggregatorV3Interface` implementor: [1](#0-0) 

`latestRoundData()` fetches all five return values from the ETH/USD Chainlink feed and then overwrites only `answer` with the composite rsETH/USD price: [2](#0-1) 

`updatedAt` is passed through unchanged from the ETH/USD feed. The `IRSETHOracle` interface exposes only `rsETHPrice()` with no timestamp: [3](#0-2) 

`LRTOracle.rsETHPrice` is a stored value updated only on explicit calls to `updateRSETHPrice()` / `_updateRsETHPrice()`: [4](#0-3) [5](#0-4) 

No `lastUpdated` timestamp is stored or exposed by `LRTOracle`. There is therefore no mechanism for `RSETHPriceFeed` to reflect the age of the rsETH component in the metadata it returns.

Additionally, `getRoundData(_roundId)` fetches the ETH/USD price for a historical round but multiplies it by the *current* rsETH/ETH rate, producing a synthetic price that never existed at that round: [6](#0-5) 

## Impact Explanation

**Low ‚Äî Contract fails to deliver promised returns, but doesn't lose value.**

`RSETHPriceFeed` advertises full `AggregatorV3Interface` compliance, including accurate staleness metadata. When the rsETH oracle price is stale (no `updateRSETHPrice()` call for an extended period), `latestRoundData()` returns a recent `updatedAt` sourced from the ETH/USD feed, causing any integrator performing a standard staleness check (`block.timestamp - updatedAt < heartbeat`) to accept a stale rsETH/USD price as valid. The LRT-rsETH protocol itself does not directly lose value; harm materializes in downstream integrators (lending markets, structured products) that rely on this feed for collateral valuation.

## Likelihood Explanation

`updateRSETHPrice()` is a keeper-driven, non-automatic call. Any gap between ETH/USD feed updates (every few minutes) and rsETH price updates (periodic) creates a window where `RSETHPriceFeed` returns a stale rsETH component with a fresh-looking `updatedAt`. This is a realistic and recurring condition in normal protocol operation, requiring no privileged access or attacker action to trigger ‚Äî it arises passively whenever the keeper is delayed.

## Recommendation

`LRTOracle` should store and expose a `lastUpdated` timestamp alongside `rsETHPrice`:

```solidity
uint256 public rsETHPriceLastUpdated;
// in _updateRsETHPrice():
rsETHPriceLastUpdated = block.timestamp;
```

`IRSETHOracle` should expose this value, and `RSETHPriceFeed.latestRoundData()` should return the minimum of the two timestamps:

```solidity
uint256 rsETHLastUpdated = RS_ETH_ORACLE.lastUpdated();
if (rsETHLastUpdated < updatedAt) updatedAt = rsETHLastUpdated;
```

`getRoundData` should revert with `NotSupported()` since historical rsETH prices are not stored and the current implementation returns a price that never existed at the queried round.

## Proof of Concept

1. Deploy `RSETHPriceFeed` pointing to a live ETH/USD Chainlink feed and `LRTOracle`.
2. Call `LRTOracle.updateRSETHPrice()` to set an initial rsETH price (e.g., `1.05e18`).
3. Advance time 24 hours without calling `updateRSETHPrice()` again. The ETH/USD Chainlink feed continues to update normally every few minutes.
4. Call `RSETHPriceFeed.latestRoundData()`.
   - `updatedAt` is recent (e.g., 5 minutes ago, from the ETH/USD feed).
   - `answer` uses the 24-hour-old rsETH price of `1.05e18`.
5. A lending protocol checking `block.timestamp - updatedAt < 3600` passes the staleness check and accepts the stale rsETH/USD price as valid collateral valuation.

Foundry fork test outline:
```solidity
function test_staleRsETHPriceWithFreshUpdatedAt() public {
    // snapshot rsETH price after initial update
    lrtOracle.updateRSETHPrice();
    // warp 24 hours ‚Äî ETH/USD feed still ticks, rsETH price does not
    vm.warp(block.timestamp + 24 hours);
    (, int256 answer,, uint256 updatedAt,) = rsETHPriceFeed.latestRoundData();
    // updatedAt is recent (ETH/USD heartbeat), answer uses 24h-old rsETH price
    assertGt(block.timestamp - updatedAt, 23 hours); // FAILS ‚Äî updatedAt is fresh
    // demonstrates the mismatch
}
```
