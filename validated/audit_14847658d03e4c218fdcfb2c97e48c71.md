### Title
Gateway Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Wrong Admission Decisions — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's `StatefulTransactionValidator::validate_resource_bounds` checks a transaction's `max_price_per_unit` against the `l2_gas_price` field of the latest finalized block. However, the next block to be built uses `next_l2_gas_price` (already computed and stored in the same block header). When the network is under load, `next_l2_gas_price > l2_gas_price`. Transactions with `max_price_per_unit` in the range `[l2_gas_price, next_l2_gas_price)` pass gateway admission but are rejected by the blockifier's pre-validation during execution with `MaxGasPriceTooLow`. The code itself contains a TODO acknowledging this exact defect.

---

### Finding Description

**Stale price read at admission:**

`validate_resource_bounds` reads the gas price from the latest finalized block via `get_block_info()`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The admission threshold is `min_gas_price_percentage * l2_gas_price / 100`. With the default `min_gas_price_percentage = 100`, the threshold equals `l2_gas_price` exactly. [2](#0-1) 

**`next_l2_gas_price` is already computed but not used:**

`StorageBlockHeader` stores both `l2_gas_price` (the price used in the current block) and `next_l2_gas_price` (the EIP-1559-adjusted price for the next block): [3](#0-2) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` constructs a `BlockInfo` that maps `block_header.l2_gas_price` but **drops** `block_header.next_l2_gas_price`: [4](#0-3) 

**Execution rejects what admission accepted:**

The blockifier's pre-validation in `AccountTransaction` checks `resource_bounds.max_price_per_unit >= actual_gas_price` where `actual_gas_price` is the block's `l2_gas_price` — which equals the previous block's `next_l2_gas_price`: [5](#0-4) 

**EIP-1559 price update path:**

`calculate_next_base_gas_price` computes `next_l2_gas_price` from gas usage. When `gas_used > gas_target`, the price increases by up to `price / gas_price_max_change_denominator` (1/48 ≈ 2.1% per block): [6](#0-5) 

With `gas_price_max_change_denominator = 48`, a single heavily-loaded block raises `next_l2_gas_price` by up to ~2.1% above `l2_gas_price`.

---

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions before sequencing.**

Any transaction submitted with `max_price_per_unit` satisfying:

```
l2_gas_price  ≤  max_price_per_unit  <  next_l2_gas_price
```

passes the gateway's `validate_tx_l2_gas_price_within_threshold` check (using the stale `l2_gas_price`) but is rejected by the blockifier's `check_resource_bounds` during execution (using the correct `next_l2_gas_price`). The gateway issues a false "valid" verdict. The transaction enters the mempool, consumes sequencer resources, and is ultimately reverted or dropped, with the user paying fees for a failed transaction.

The `validate_resource_bounds` flag is `true` by default in production config: [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

**Likelihood: High** — The condition `next_l2_gas_price > l2_gas_price` occurs whenever any block is more than half-full (i.e., `gas_used > gas_target`). Under normal network load this is routine. The discrepancy window is up to ~2.1% per block and is predictable from public block data. Any user who sets `max_price_per_unit` to exactly the current block's `l2_gas_price` (a natural choice when following the "current price") will hit this window during periods of rising demand.

---

### Recommendation

Replace the stale `l2_gas_price` read with `next_l2_gas_price` from the block header, exactly as the in-code TODO requests:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
```

Concretely:
1. Expose `next_l2_gas_price` through `GatewayFixedBlockStateReader::get_block_info` (or add a dedicated `get_next_l2_gas_price` method).
2. In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, map `block_header.next_l2_gas_price` into the returned value.
3. In `validate_resource_bounds`, use `next_l2_gas_price` as the admission threshold instead of `l2_gas_price`.

This ensures the gateway's admission decision is consistent with the price the blockifier will enforce during execution.

---

### Proof of Concept

**Setup:** Network is under load; latest finalized block N has:
- `l2_gas_price = 3_000_000_000` FRI (used in block N)
- `next_l2_gas_price = 3_062_500_000` FRI (≈ 2.1% increase, stored in block N's header, will be used in block N+1)

**Step 1 — User submits transaction:**
```
max_price_per_unit = 3_000_000_000  // exactly current block's l2_gas_price
```

**Step 2 — Gateway admission (`validate_resource_bounds`):**
```
threshold = min_gas_price_percentage(100) * l2_gas_price(3_000_000_000) / 100
          = 3_000_000_000
tx_l2_gas_price(3_000_000_000) >= threshold(3_000_000_000)  → ADMITTED ✓
``` [9](#0-8) 

**Step 3 — Block N+1 is built with `l2_gas_price = next_l2_gas_price = 3_062_500_000`.**

**Step 4 — Blockifier pre-validation:**
```
resource_bounds.max_price_per_unit(3_000_000_000) < actual_gas_price(3_062_500_000)
→ MaxGasPriceTooLow  → EXECUTION FAILS ✗
``` [5](#0-4) 

**Result:** The gateway issued a valid admission verdict for a transaction that the blockifier rejects. The invariant "gateway-admitted transactions are valid for sequencing" is broken. The user pays fees for a failed transaction; the mempool is polluted with transactions that cannot execute.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
