### Title
Hardcoded Private Key Used in Production Consensus Signature Manager - (File: `crates/apollo_signature_manager/src/lib.rs`)

---

### Summary

The production `create_signature_manager()` factory function instantiates a `SignatureManager` backed by `LocalKeyStore::new_for_testing()`, which embeds a well-known private key committed to the public repository. Any party with repository read access can derive the sequencer's consensus signing identity and forge valid precommit vote signatures over arbitrary block hashes.

---

### Finding Description

`crates/apollo_signature_manager/src/lib.rs` re-exports `LocalKeyStoreSignatureManager` as the canonical `SignatureManager` type and provides `create_signature_manager()` as the production factory:

```rust
pub use LocalKeyStoreSignatureManager as SignatureManager;

pub fn create_signature_manager() -> SignatureManager {
    SignatureManager::new()   // calls LocalKeyStore::new_for_testing()
}
``` [1](#0-0) 

`LocalKeyStore::new_for_testing()` in `signature_manager.rs` hard-codes both the private and public key:

```rust
pub(crate) const fn new_for_testing() -> Self {
    const PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
        "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
    ));
    const PUBLIC_KEY: PublicKey = PublicKey(Felt::from_hex_unchecked(
        "0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a",
    ));
    Self { private_key: PRIVATE_KEY, public_key: PUBLIC_KEY }
}
``` [2](#0-1) 

This same key is used to sign two security-critical operations:

- **Consensus precommit votes** — `sign_precommit_vote(block_hash)` signs the block hash that the node commits to in the consensus round.
- **Peer identity challenges** — `sign_identification(peer_id, challenge)` authenticates the node's p2p identity. [3](#0-2) 

The owning TODO comment in `lib.rs` explicitly acknowledges the key store is not production-ready:

```
// TODO(Elin): understand how key store would look in production and better define
// the way the signature manager is created.
``` [4](#0-3) 

---

### Impact Explanation

**Critical / High.** Because the private key is a compile-time constant in a public repository, any observer can:

1. Reconstruct the exact ECDSA private key.
2. Call `ecdsa_sign(private_key, precommit_vote_digest)` for any block hash of their choice.
3. Inject a forged precommit vote that is cryptographically indistinguishable from a legitimate one produced by the sequencer node.

If the consensus layer accepts forged precommit votes, an attacker can steer the committed block hash to an arbitrary value, directly corrupting the block commitment path (`ProposalCommitment` / `ProposalFin` comparison). This maps to:

- **Critical**: Wrong state/receipt/block hash commitment from consensus logic.
- **High**: Transaction conversion or signature/hash logic binds the wrong signer or hash.

Additionally, because every node running this codebase shares the same key, the attacker can impersonate *all* sequencer nodes simultaneously, not just one.

---

### Likelihood Explanation

**High.** The repository is public. The private key is a `const` value visible in plain text. No special access, privilege, or network position is required — a read of the source file is sufficient to extract the key and begin forging signatures.

---

### Recommendation

1. **Remove `new_for_testing()` from the production code path.** `create_signature_manager()` must never call a function named `new_for_testing()`.
2. **Load the private key from a secrets manager or environment variable at runtime**, analogous to how `EthereumBaseLayerConfig` wraps endpoint URLs in `Sensitive<Url>` and marks them `ParamPrivacyInput::Private`. [5](#0-4) 
3. **Rotate the exposed key immediately.** Since the key is already public, any deployment that used it must be considered compromised.
4. **Gate `new_for_testing()` behind `#[cfg(test)]`** so it cannot be referenced from production binaries.

### Proof of Concept

```python
from starknet_crypto import sign   # or any STARK-curve ECDSA library

PRIVATE_KEY = 0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133

# Domain separator from signature_manager.rs: b"PRECOMMIT_VOTE"
# Attacker chooses an arbitrary target block hash:
target_block_hash = 0xdeadbeef...

# Compute the same blake2s digest the sequencer would compute:
import hashlib
msg = b"PRECOMMIT_VOTE" + target_block_hash.to_bytes(32, "big")
digest = blake2s_to_felt(msg)   # matches build_precommit_vote_message_digest()

# Sign with the publicly known key:
forged_r, forged_s = sign(private_key=PRIVATE_KEY, msg_hash=digest)

# Submit forged_r, forged_s as a precommit vote for target_block_hash.
# The sequencer's verify_precommit_vote_signature() will accept it as valid.

fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash);

    MessageDigest(blake2s_to_felt(&message))
}
