### Title
Missing `Q == sum(Qis)` re-validation in `attach_private_scalar` enables aggregate public-key substitution in EdDSA-MP AC signing — (`src/cbmpc/api/eddsa_mp.cpp`)

### Summary
`attach_private_scalar` accepts a tampered `public_key_blob` whose `Q_compressed` field has been replaced with an attacker-chosen point `P_evil`. Because neither `attach_private_scalar` nor the no-job `deserialize_ac_key_blob` (called by `sign_ac`) re-derives or re-validates `Q` against `sum(Qis)`, the poisoned `Q_compressed` propagates silently into the signing key. All subsequent `sign_ac` calls produce Ed25519 signatures that are valid under `P_evil`, not the real aggregate key.

### Finding Description

`key_blob_v1_t` stores three independent fields: `Q_compressed` (the aggregate public key), `Qis_compressed` (per-party public shares), and `x_share` (the private scalar). [6](#0-5) 

`attach_private_scalar` validates only the self-share binding (`x*G == Qi_self`) and the `public_share_compressed` argument, but never checks that `Q_compressed == sum(Qis_compressed)`: [7](#0-6) 

The no-job `deserialize_ac_key_blob` (used exclusively by `sign_ac`) repeats the same omission — it validates `x_share * G == Qi_self` but copies `Q` from the blob without verifying it equals `sum(Qis)`: [2](#0-1) 

`sign_ac` then passes this key (with `key.Q = P_evil`) through `to_additive_share` and into `eddsampc::sign`, which uses `key.Q` as the Ed25519 public key in the Fiat-Shamir hash `H(R || Q || msg)`: [8](#0-7) 

The internal self-verification inside `sign_batch` also uses `key.Q`: [9](#0-8) 

Because the hash and the internal verify both use the same `P_evil`, the internal check passes and the function returns a well-formed signature `(R, s)` that verifies under `P_evil`.

### Impact Explanation

An attacker who can supply a tampered `public_key_blob` to `attach_private_scalar` (e.g., by corrupting blob storage or acting as a malicious serialized-input provider) causes all parties that restore from that blob to silently sign under `P_evil` instead of the real aggregate key. `get_public_key_compressed` will also report `P_evil` as the key. This is a complete aggregate public-key substitution: the honest parties believe they are signing under the legitimate key, but the output signatures are valid only under the attacker-chosen key.

### Likelihood Explanation

The `detach_private_scalar` / `attach_private_scalar` API is explicitly designed for backup/restore workflows where the public blob is persisted separately from the private scalar. Any storage layer, backup system, or transit path that is not integrity-protected is a viable injection point. The attacker needs only to flip 32 bytes in the blob; no knowledge of the private scalar is required.

### Recommendation

In `attach_private_scalar`, after deserializing the blob, recompute `Q_sum = sum(Qis_compressed)` and reject the blob if `Q_compressed != Q_sum`. Apply the same guard in the no-job `deserialize_ac_key_blob`. This mirrors the check already present in the additive `deserialize_key_blob`: [3](#0-2) 

### Proof of Concept

```
1. Run dkg_ac (n=2, threshold=2) → key_blobs[0], key_blobs[1]
2. detach_private_scalar(key_blobs[0]) → public_blob, x_fixed
3. Deserialize public_blob; replace Q_compressed with P_evil (any valid Ed25519 point ≠ Q1+Q2)
4. Re-serialize → tampered_public_blob
5. get_public_share_compressed(key_blobs[0]) → Qi_self
6. attach_private_scalar(tampered_public_blob, x_fixed, Qi_self) → merged_blob
   // Returns SUCCESS — x*G == Qi_self passes; Q == sum(Qis) is never checked
7. get_public_key_compressed(merged_blob) → P_evil  // confirms substitution
8. sign_ac(job, merged_blob, ...) → sig
   // sig verifies under P_evil, not the real aggregate key
9. assert ed25519_verify(P_evil, msg, sig) == OK
10. assert ed25519_verify(real_Q, msg, sig) == FAIL
```
