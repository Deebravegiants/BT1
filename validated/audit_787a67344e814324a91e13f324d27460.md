Audit Report

## Title
Protocol Fee Permanently Lost When `updateRSETHPrice()` Is Called While `LRTDepositPool` or `LRTWithdrawalManager` Is Paused - (File: contracts/LRTOracle.sol)

## Summary
`updateRSETHPrice()` is gated only by `LRTOracle.paused`, but the internal fee-skip logic also checks `lrtDepositPool.paused()` and `withdrawalManager.paused()`. When either of those contracts is paused while `LRTOracle` is not, any unprivileged caller can advance `rsETHPrice` to the current TVL without minting any protocol fee, permanently erasing the fee baseline for all yield accrued during the pause window.

## Finding Description
The `whenNotPaused` modifier at line 47–50 checks only `LRTOracle.paused`. This means `updateRSETHPrice()` (line 87–89) is callable by any EOA whenever `LRTOracle.paused == false`, regardless of the state of `lrtDepositPool` or `withdrawalManager`.

Inside `_updateRsETHPrice()`, the broader pause flag is computed at line 240:
```solidity
bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;
```
Fee minting is skipped when `protocolPaused == true` (lines 243–247):
```solidity
uint256 protocolFeeInETH = 0;
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
```
However, `rsETHPrice` is unconditionally written at line 313:
```solidity
rsETHPrice = newRsETHPrice;
```
There is no early return or guard on `protocolPaused` before this assignment. The fee baseline for the next call is `previousTVL = rsethSupply.mulWad(rsETHPrice)`. Because `rsETHPrice` was already advanced to reflect the yield earned during the pause, the next call after unpausing sees `totalETHInProtocol ≈ previousTVL` and collects zero fee. The protocol fee for the entire pause window is permanently lost.

## Impact Explanation
Every invocation of `updateRSETHPrice()` during a partial-pause window (only `lrtDepositPool` or `withdrawalManager` paused) causes the protocol treasury to permanently forfeit the rsETH fee that should have been minted on the yield accrued during that window. The fee is not deferred — it is gone. This constitutes **High — Theft of unclaimed yield**, which is an explicitly listed valid impact.

## Likelihood Explanation
`LRTDepositPool` and `LRTWithdrawalManager` are paused independently of `LRTOracle` during routine security responses, maintenance, or EigenLayer-related incidents. `LRTOracle` is intentionally kept active during such pauses so that the price feed remains available. During any such pause window, `updateRSETHPrice()` is publicly callable by any EOA or contract with no special privileges, making the trigger trivially reachable. Likelihood is **Medium**.

## Recommendation
When `protocolPaused == true`, do not update `rsETHPrice`. Add an early return after computing `protocolPaused` and before computing `newRsETHPrice`:

```solidity
if (protocolPaused) {
    return; // preserve fee baseline; fee will be collected after unpause
}
```

This ensures that when the protocol is unpaused, the full yield accrued during the pause is subject to the normal fee calculation on the next `updateRSETHPrice()` call.

## Proof of Concept
**Setup:**
- `rsethSupply = 1 000 rsETH`, `rsETHPrice = 1.05 ETH`, `protocolFeeInBPS = 1000` (10%)
- `previousTVL = 1 000 × 1.05 = 1 050 ETH`

**Step 1 — Partial pause:** `lrtDepositPool` is paused. `LRTOracle.paused = false`.

**Step 2 — Yield accrues:** EigenLayer rewards push `totalETHInProtocol` to `1 060 ETH` (10 ETH yield).

**Step 3 — Unprivileged call:** Any user calls `updateRSETHPrice()`.
- `whenNotPaused` passes (`LRTOracle.paused = false`).
- `protocolPaused = lrtDepositPool.paused() = true`.
- `protocolFeeInETH = 0` (fee skipped, line 244).
- `newRsETHPrice = 1 060 / 1 000 = 1.06 ETH` (line 250).
- `rsETHPrice` written as `1.06 ETH` (line 313).

**Step 4 — Unpause:** `lrtDepositPool` is unpaused.

**Step 5 — Next price update:** `updateRSETHPrice()` is called.
- `previousTVL = 1 000 × 1.06 = 1 060 ETH`.
- `totalETHInProtocol ≈ 1 060 ETH` (no new yield since Step 3).
- `totalETHInProtocol ≤ previousTVL` → `protocolFeeInETH = 0`.

**Result:** The 10 ETH of yield earned during the pause generated **0 protocol fee** instead of the expected `10 × 10% = 1 ETH` worth of rsETH minted to the treasury. The loss is permanent and scales with both the duration of the pause and the yield rate.
