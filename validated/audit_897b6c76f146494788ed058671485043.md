Audit Report

## Title
Same-block oracle price update enables risk-free arbitrage against LP positions - (File: metric-core/contracts/MetricOmmPool.sol)

## Summary
`MetricOmmPool.swap()` fetches a live oracle price on every invocation with no per-block or per-transaction price cache. Because `ChainlinkOracle.updateReport()` is permissionless, an attacker can atomically execute a first swap at the stale on-chain price, push a newer DON-signed report to overwrite the oracle, and execute a reverse swap at the updated price — all within a single transaction — extracting the price delta from LP principal with zero directional risk.

## Finding Description
Every call to `swap()` immediately invokes `_getBidAndAskPriceX64()`:

```solidity
// MetricOmmPool.sol:228
(uint128 bidPriceX64, uint128 askPriceX64) = _getBidAndAskPriceX64();
```

`_getBidAndAskPriceX64()` calls the price provider live on every invocation with no block-level or transaction-level cache:

```solidity
// MetricOmmPool.sol:804-813
function _getBidAndAskPriceX64() internal returns (uint128 bidPriceX64, uint128 askPriceX64) {
    address activePriceProvider = _resolvedPriceProvider();
    try IPriceProvider(activePriceProvider).getBidAndAskPrice() returns (uint128 bid, uint128 ask) {
        ...
    }
}
```

The oracle's ingestion path is fully permissionless — any caller holding a valid DON-signed report can push it:

```solidity
// ChainlinkOracle.sol:68-70
function updateReport(bytes calldata fullReport) external {
    _store(_verifyReport(fullReport));
}
```

`_store` overwrites the stored price whenever the incoming report's timestamp is strictly newer:

```solidity
// ChainlinkOracle.sol:91-94
if (d.timestampMs.isAfter(oracleData[feedId].timestampMs)) {
    oracleData[feedId] = d;
    emit ReportStored(feedId, d.price, d.spread0, d.timestampMs);
}
```

The only timestamp guard (`revertIfAfterBlockTimeWithDrift`) rejects reports with timestamps *after* `block.timestamp + drift` — it does not prevent pushing a report whose observation timestamp is newer than the currently stored one but still in the past. Two signed reports R₁ (timestamp T) and R₂ (timestamp T+10, both in the past) can both be pushed in the same block.

The pool's `nonReentrant(PoolActions.SWAP)` modifier uses transient storage that is cleared by `_nonReentrantAfter()` immediately after each `swap()` call returns:

```solidity
// MetricReentrancyGuardTransient.sol:40-42
function _nonReentrantAfter() internal {
    TransientSlot.tstore(TransientSlot.asUint256(_reentrancyGuardStorageSlot()), 0);
}
```

This prevents only *nested* reentrant calls within a single swap execution. It imposes no constraint on two sequential, independent `swap()` calls in the same transaction. After the first swap returns, the transient slot is 0 and a second swap proceeds normally.

**Exploit call sequence (single transaction):**
1. `pool.swap(recipient, false, exactIn(X_USDC), MAX_PRICE)` → pool reads oracle → bid/ask from P₁ → attacker receives token0 at P₁
2. `oracle.updateReport(R₂)` → `_store` accepts R₂ (newer timestamp) → oracle now holds P₂ > P₁
3. `pool.swap(recipient, true, exactIn(token0_amount), 0)` → pool reads oracle → bid/ask from P₂ → attacker receives token1 at P₂

The attacker captures `(P₂ − P₁) × amount − fees` with zero directional risk. The loss is borne entirely by LPs whose deposited assets are sold at P₁ and repurchased at P₂.

## Impact Explanation
Direct loss of LP principal. The pool's token balances after the round-trip are lower than before by the price delta minus fees. This constitutes a real token balance loss above Sherlock thresholds whenever the price delta exceeds the spread fee (e.g., a 2% price move with a 1% spread yields ~$500 profit per $50k round trip, scaling linearly with position size and price move). This matches the "bad-price execution" and "direct loss of user principal / owed LP assets" allowed impact categories.

## Likelihood Explanation
Chainlink Data Streams publishes signed price reports off-chain that anyone can push on-chain via the permissionless `updateReport`. A searcher monitoring the off-chain feed can observe a price move, hold the newer signed report, execute the first swap at the stale price, push the report, and execute the reverse swap — all within a single atomic transaction. No privileged access is required. MEV infrastructure (flashbots bundles, private mempools) makes this straightforward on any EVM chain where the protocol is deployed. The attack is repeatable on every block where a price move exceeds the spread fee.

## Recommendation
Cache the oracle bid/ask at the first price read within a transaction and reuse it for all subsequent reads in the same transaction. EIP-1153 transient storage (already used by the pool for the reentrancy guard) is the natural fit — it auto-clears at end of each transaction, requiring no manual cleanup:

```solidity
uint128 transient private _cachedBidX64;
uint128 transient private _cachedAskX64;

function _getBidAndAskPriceX64() internal returns (uint128, uint128) {
    uint128 cached = _cachedBidX64;
    if (cached != 0) return (cached, _cachedAskX64);
    // ... existing fetch logic ...
    _cachedBidX64 = bid;
    _cachedAskX64 = ask;
    return (bid, ask);
}
```

This ensures that within any single transaction, all swap calls see the same oracle price, eliminating the intra-transaction price manipulation vector.

## Proof of Concept
**Setup**: Pool with WBTC/USDC, spread fee = 1%. Off-chain Chainlink Data Streams has two valid DON-signed reports: R₁ (WBTC = $50,000, timestamp T) and R₂ (WBTC = $51,000, timestamp T+10). On-chain oracle currently holds R₁.

**Attack contract (single tx):**
```solidity
function attack(address pool, address oracle, bytes calldata r2) external {
    // Step 1: swap at stale price P1 = $50,000
    pool.swap(address(this), false, exactIn(50_000e6), type(uint128).max, "", "");
    // Step 2: push newer report → oracle now holds P2 = $51,000
    ChainlinkOracle(oracle).updateReport(r2);
    // Step 3: reverse swap at new price P2 = $51,000
    pool.swap(address(this), true, exactIn(1e8), 0, "", "");
    // Net: ~$1,000 profit minus ~$1,010 in fees → profitable above ~2% move at 1% spread
}
```

**Foundry test plan**: Deploy pool with `MockOracle`, call `setBidAndAskPrice` to P₁ before swap 1 and P₂ before swap 2 (simulating the `updateReport` step), assert LP token balances decreased and attacker token balances increased by the price delta minus fees.
