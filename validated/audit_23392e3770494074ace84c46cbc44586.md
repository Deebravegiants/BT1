### Title
Unauthenticated SQMR Peer Can Store Arbitrary `TransactionHash` for L1HandlerTransactions, Corrupting RPC Transaction Lookups — (`crates/apollo_p2p_sync/src/client/transaction.rs`)

---

### Summary

The P2P sync client's `parse_data_for_block` accepts `transaction_hash` values from unauthenticated SQMR peers and stores them directly without recomputing or validating them against the transaction body. For `L1HandlerTransaction`, the nonce, calldata, contract address, and entry point selector are decoded from the protobuf body, but the `transaction_hash` field is taken verbatim from a separate protobuf field. A malicious peer can supply any arbitrary hash, causing the node to index the transaction under the wrong key and making it unreachable via `starknet_getTransactionByHash` with the canonical hash.

---

### Finding Description

In `parse_data_for_block`, the sync client reads `FullTransaction { transaction, transaction_output, transaction_hash }` from the peer and pushes `transaction_hash` directly into `block_body.transaction_hashes` with an explicit developer acknowledgment that this is unvalidated: [1](#0-0) 

The `FullTransaction` is deserialized in `TryFrom<protobuf::TransactionInBlock> for (Transaction, TransactionHash)`. The `tx_hash` is extracted from the protobuf `transaction_hash` field independently of the transaction body: [2](#0-1) 

For the `L1Handler` variant specifically, the body (`nonce`, `address`, `entry_point_selector`, `calldata`) is decoded via `TryFrom<protobuf::L1HandlerV0>`: [3](#0-2) 

At no point is the canonical hash recomputed from the body fields and compared against the peer-supplied `transaction_hash`. The canonical hash formula for L1 handler transactions includes `(L1_HANDLER_PREFIX, version, contract_address, entry_point_selector, calldata_hash, chain_id, nonce)`: [4](#0-3) 

The `write_to_storage` path then commits the `BlockBody` (including the corrupted `transaction_hashes` vec) directly to storage: [5](#0-4) 

The storage layer indexes transactions by the stored hash via `TransactionHashToIdxTable`: [6](#0-5) 

---

### Impact Explanation

The RPC handler for `starknet_getTransactionByHash` looks up the transaction index using the stored hash: [7](#0-6) 

If the stored hash is attacker-controlled, then:
- `starknet_getTransactionByHash(canonical_hash)` → `TXN_HASH_NOT_FOUND` (the real L1 handler transaction is invisible)
- `starknet_getTransactionByHash(attacker_hash)` → returns the transaction body under a fabricated hash

Similarly, `starknet_getTransactionReceipt` and `starknet_getTransactionByBlockIdAndIndex` (which calls `get_transaction_hash_by_idx`) will return the wrong hash in the response, causing the receipt's `transaction_hash` field to be wrong.

**Clarification on `message_hash`**: The `message_hash` field in the L1 handler receipt is computed from the transaction body fields via `calc_msg_hash()` (nonce, contract_address, entry_point_selector, calldata), not from the stored `TransactionHash`: [8](#0-7) 

So the keccak L1↔L2 message hash itself is not directly corrupted by a hash-only mismatch. However, if the attacker also manipulates the body fields (nonce, calldata) to not correspond to any real L1 message, the `message_hash` in the receipt will also be wrong, fully corrupting L1-to-L2 message tracking.

---

### Likelihood Explanation

The SQMR protocol is unauthenticated — any peer can connect and serve transaction data. The sync client only validates `n_transactions` against the stored header but does **not** validate transaction hashes against the `transaction_commitment` in the block header. The developer-acknowledged TODO at line 88 confirms this is a known gap, not an intentional design choice.

---

### Recommendation

After decoding the `L1HandlerTransaction` body from the protobuf, recompute the canonical hash using `L1HandlerTransaction::calculate_transaction_hash(chain_id, &version)` (as done in `ExecutableL1HandlerTransaction::create`) and compare it against the peer-supplied `transaction_hash`. A mismatch should be treated as a `BadPeerError` and the peer should be reported/disconnected. This pattern already exists for the trusted path: [9](#0-8) 

---

### Proof of Concept

1. Stand up a malicious SQMR peer that responds to transaction sync queries.
2. For a block containing an `L1HandlerTransaction`, serve the correct body fields (`nonce`, `address`, `entry_point_selector`, `calldata`) but set `transaction_hash` to an arbitrary value `0xdeadbeef...`.
3. The syncing node stores the transaction indexed under `0xdeadbeef...`.
4. Call `starknet_getTransactionByHash(canonical_hash)` → `TXN_HASH_NOT_FOUND`.
5. Call `starknet_getTransactionByHash(0xdeadbeef...)` → returns the L1 handler transaction body.
6. Assert that `canonical_hash != 0xdeadbeef...` and that the stored hash does not satisfy `get_l1_handler_transaction_hash(body, chain_id, version)`.

    chain_id: &ChainId,
    version: L1HandlerVersions,

    Ok(TransactionHash(

                Some(*INVOKE)
            } else {
                Some(*L1_HANDLER)
            }
        })
        .chain_if_fn(|| {
            if version > L1HandlerVersions::V0Deprecated {
                Some(transaction_version.0)
            } else {
                None
            }

        .chain(&transaction.entry_point_selector.0)
        .chain(&HashChain::new().chain_iter(transaction.calldata.0.iter()).get_pedersen_hash())

            if version > L1HandlerVersions::V0Deprecated {
                Some(Felt::ZERO)

                None
            }

        .chain_if_fn(|| {
            if version > L1HandlerVersions::AsInvoke {
                Some(transaction.nonce.0)
            } else {
                None
            }
        })

    async fn get_transaction_by_hash(
        &self,
        transaction_hash: TransactionHash,
    ) -> RpcResult<TransactionWithHash> {
        verify_storage_scope(&self.storage_reader)?;

        let txn = self.storage_reader.begin_ro_txn().map_err(internal_server_error)?;

        if let Some(transaction_index) =
            txn.get_transaction_idx_by_hash(&transaction_hash).map_err(internal_server_error)?
        {
            let transaction = txn
                .get_transaction(transaction_index)
                .map_err(internal_server_error)?
                .ok_or_else(|| ErrorObjectOwned::from(TRANSACTION_HASH_NOT_FOUND))?;

            Ok(TransactionWithHash { transaction: transaction.try_into()?, transaction_hash })
        } else {
            // The transaction is not in any non-pending block. Search for it in the pending block
            // and if it's not found, return error.
            let client_transaction = read_pending_data(&self.pending_data, &txn)
                .await?
                .block
                .transactions()
                .iter()
                .find(|transaction| transaction.transaction_hash() == transaction_hash)
                .ok_or_else(|| ErrorObjectOwned::from(TRANSACTION_HASH_NOT_FOUND))?
                .clone();

            let starknet_api_transaction: StarknetApiTransaction =
                client_transaction.try_into().map_err(internal_server_error)?;
            return Ok(TransactionWithHash {
                transaction: starknet_api_transaction.try_into()?,
                transaction_hash,
            });
        }
    }
