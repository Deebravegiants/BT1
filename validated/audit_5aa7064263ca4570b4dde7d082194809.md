Audit Report

## Title
Stale Rate Served Indefinitely by `getRate()` Enables Mispriced rsETH Minting on L2 Pools — (`contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary
`CrossChainRateReceiver.getRate()` returns the cached `rate` without consulting `lastUpdated`, so if the LayerZero bridge stops delivering updates, all downstream L2 pool contracts continue minting rsETH at an arbitrarily old rate. Because rsETH is a monotonically appreciating yield-bearing token, a stale (lower) rate causes depositors to receive more rsETH than they are entitled to, diluting existing holders' accrued yield.

## Finding Description
`CrossChainRateReceiver` stores `rate` and `lastUpdated` as separate state variables. [1](#0-0) 

`lzReceive` updates both on every successful bridge message. [2](#0-1) 

`getRate()` returns only `rate` with no staleness check against `lastUpdated`. [3](#0-2) 

Every L2 pool contract (`RSETHPoolV2`, `RSETHPoolV2NBA`, `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, `RSETHPoolV3WithNativeChainBridge`) delegates its own `getRate()` directly to `IOracle(rsETHOracle).getRate()` with no additional freshness guard. [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6) 

The minting formula in every pool divides by the stale rate. [8](#0-7) 

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` explicitly reverts on stale data, a pattern that `CrossChainRateReceiver` does not follow despite tracking `lastUpdated`. [9](#0-8) 

**Exploit path:**
1. A valid `lzReceive` delivers rate `R` at time `T0`.
2. The LayerZero bridge stalls (relayer gas exhaustion, outage, or simply no one calling `updateRate()`).
3. Time advances; the true rsETH/ETH rate grows to `R' > R`.
4. Any depositor calls `deposit()` on any L2 pool. `getRate()` returns the stale `R`.
5. `rsETHAmount = amountAfterFee * 1e18 / R` — because `R < R'`, the depositor receives more rsETH than the current rate warrants.
6. The surplus rsETH represents a claim on ETH that was never deposited, diluting all existing holders.

The daily mint limit present in V2/V3 pools bounds the per-day magnitude but does not prevent the attack; it resets every 24 hours and the bridge can remain stale for days. [10](#0-9) 

## Impact Explanation
**High — Theft of unclaimed yield.** Existing rsETH holders continuously accrue staking yield reflected in the rising rsETH/ETH rate. When deposits occur at a stale lower rate, new holders receive rsETH representing more ETH than they deposited. This proportionally shrinks every existing holder's claim on the underlying ETH pool — their accrued yield is transferred to the new depositors. The effect is permanent and cumulative for the duration of the bridge outage.

## Likelihood Explanation
LayerZero bridge liveness depends on off-chain relayers and gas funding. The `updateRate()` call on the provider side is permissionless but requires the caller to supply ETH for cross-chain gas; if no one calls it or the relayer stalls, the receiver silently serves a stale rate indefinitely. No on-chain circuit breaker exists. The trigger is a normal, unprivileged `deposit()` call — no attacker coordination is required; ordinary depositors unknowingly exploit the condition. The receiver is deployed on Arbitrum, Optimism, Base, Scroll, Blast, Mode, Linea, zkSync, Zircuit, and other chains, making the attack surface broad.

## Recommendation
Add a configurable `maxStaleness` threshold and revert in `getRate()` if the rate is too old, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
uint256 public maxStaleness; // e.g. 86400 (24 hours)

error StaleRate();

function getRate() external view returns (uint256) {
    if (block.timestamp - lastUpdated > maxStaleness) revert StaleRate();
    return rate;
}
```

`maxStaleness` should be owner-configurable and initialized to a value consistent with the expected bridge update frequency.

## Proof of Concept

```solidity
// Fork test on any L2 where RSETHRateReceiver is deployed as rsETHOracle
function test_staleRateMispricing() external {
    // 1. Deliver a valid rate at T0
    vm.prank(layerZeroEndpoint);
    receiver.lzReceive(
        srcChainId,
        abi.encodePacked(rateProvider, address(0)),
        0,
        abi.encode(1.05e18) // rate = 1.05 ETH per rsETH
    );
    assertEq(receiver.getRate(), 1.05e18);

    // 2. Warp 48 hours — bridge delivers no updates
    vm.warp(block.timestamp + 48 hours);

    // 3. getRate() still returns the stale rate — no revert
    assertEq(receiver.getRate(), 1.05e18);
    // true rate after 48h of staking yield would be ~1.0506e18

    // 4. Depositor mints rsETH at stale rate
    // 1 ETH → rsETHAmount = 1e18 * 1e18 / 1.05e18 ≈ 952380952380952380
    // at true rate 1.0506e18 → should be ≈ 951817057...
    // depositor receives ~563895... extra rsETH per ETH
    // at 10,000 ETH deposited during outage → ~5.6 ETH of yield stolen from existing holders
    vm.deal(attacker, 10_000 ether);
    vm.prank(attacker);
    pool.deposit{value: 10_000 ether}("ref");
}
```
