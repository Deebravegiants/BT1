Audit Report

## Title
Unrestricted `claim()` Caller Allows Forced Fee Extraction Against Any Account - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary
`MerkleDistributor.claim()` accepts an arbitrary `account` address with no `msg.sender == account` guard, allowing any external caller to force a token claim on behalf of any eligible account. When `feeInBPS > 0`, this permanently reduces the victim's received yield by the fee amount, which flows irrecoverably to `protocolTreasury`. The sibling contract `KernelMerkleDistributor` explicitly closes this gap with an `Unauthorized` revert.

## Finding Description
In `contracts/utils/MerkleDistributor/MerkleDistributor.sol`, the public `claim` function at L97–147 accepts `account` as a caller-supplied parameter and performs no check that `msg.sender == account`: [1](#0-0) 

The fee deduction and transfer execute unconditionally against the supplied `account`: [2](#0-1) 

Once processed, `userClaims[account].lastClaimedIndex` is updated to `index`, so `isClaimed` returns `true` and the victim cannot re-claim: [3](#0-2) 

The sibling contract `KernelMerkleDistributor._processClaim()` explicitly prevents this at L311–313: [4](#0-3) 

`MerkleDistributor` has no equivalent guard. All merkle proofs and eligible accounts are derivable from the publicly distributed merkle tree, so no privileged access is required to construct a valid call.

## Impact Explanation
**High — Theft of unclaimed yield.**

When `feeInBPS > 0` (up to `MAX_FEE_IN_BPS = 1000`, i.e. 10%), an attacker can force a claim for any eligible account and cause the fee to be permanently extracted from the victim's allocation. The victim receives `claimableAmount - fee` instead of the full `claimableAmount` they would have received by claiming themselves — particularly after a pending fee reduction. The lost tokens flow to `protocolTreasury` and are irrecoverable by the victim. This matches the allowed impact: **High — Theft of unclaimed yield**. [5](#0-4) 

## Likelihood Explanation
**Medium.** The exploit requires no capital, no special role, and no victim mistake — only gas. Merkle-eligible accounts and their proofs are public. An attacker can:
1. Enumerate eligible accounts from the published merkle tree.
2. Watch the mempool for a `setFeeInBPS` transaction that lowers the fee.
3. Front-run it by calling `claim(index, victim, cumulativeAmount, proof)` at the current higher fee rate.

Even without a pending fee reduction, an attacker can force claims at any time at the current fee rate, denying victims the ability to time their own claims. [6](#0-5) 

## Recommendation
Add a caller restriction identical to `KernelMerkleDistributor._processClaim()`:

```solidity
function claim(...) external override whenNotPaused {
    if (account != msg.sender) revert Unauthorized();
    // ...
}
```

If third-party claiming is intentional (e.g., for gas relayers), introduce an explicit per-account opt-in delegation mechanism rather than allowing unrestricted third-party claims. [4](#0-3) 

## Proof of Concept
1. Deploy `MerkleDistributor` with `feeInBPS = 500` (5%).
2. Alice has a valid merkle proof entitling her to 1,000 tokens; she intends to wait for a fee reduction.
3. Admin submits `setFeeInBPS(0)` to the mempool.
4. Attacker front-runs by calling:
   ```solidity
   merkleDistributor.claim(aliceIndex, alice, 1000e18, aliceProof);
   ```
5. Alice's claim executes at 5% fee: she receives 950 tokens; 50 tokens go to `protocolTreasury`.
6. Admin's `setFeeInBPS(0)` lands — but `userClaims[alice].lastClaimedIndex >= aliceIndex`, so `isClaimed` returns `true` and Alice cannot re-claim.
7. Alice permanently loses 50 tokens she would have received under the zero-fee regime.

**Foundry test plan:**
- Deploy `MerkleDistributor` with `feeInBPS = 500`.
- Construct a merkle tree with Alice's allocation; set root.
- As `attacker` (not Alice), call `claim(index, alice, amount, proof)`.
- Assert Alice received `amount * 9500 / 10000` and `protocolTreasury` received the fee.
- Assert `isClaimed(index, alice)` returns `true`.
- Assert Alice cannot call `claim` again (reverts with `AlreadyClaimed`). [7](#0-6)
