### Title
`NonCanonicalBlock` Misclassified as Non-Transient in `ForeignChainInspectionError::is_transient()`, Causing Spurious Permanent Failure of `verify_foreign_transaction` Requests - (File: `crates/foreign-chain-inspector/src/lib.rs`)

### Summary

`ForeignChainInspectionError::is_transient()` omits `NonCanonicalBlock` from its transient-error list, even though the Bitcoin inspector's own inline documentation explicitly states that a reorg landing between its two sequential RPC calls "could in principle yield a spurious `NonCanonicalBlock`" and that "The caller is expected to retry." The `FanOut` inspector uses `is_transient()` to decide whether an error is a substantive verdict or a tolerable hiccup. Because `NonCanonicalBlock` is classified as non-transient, a race-condition-induced spurious result is treated as a terminal, substantive verdict, causing the `verify_foreign_transaction` signing request to fail permanently rather than being retried.

### Finding Description

**Root cause — `is_transient()` omits `NonCanonicalBlock`:** [1](#0-0) 

Only four variants are considered transient: `ClientError`, `RpcRequestFailed`, `NotFinalized`, and `NotEnoughBlockConfirmations`. `NonCanonicalBlock` is absent.

**The Bitcoin inspector explicitly documents the race condition:** [2](#0-1) 

The two sequential RPC calls (`getblockheader` then `getblockhash`) are inherently racy: a reorg landing between them causes `getblockhash` to return a different canonical hash than the one `getblockheader` returned, producing a `NonCanonicalBlock` error that is entirely spurious. The comment says "The caller is expected to retry" — but `is_transient()` returns `false` for this variant.

The EVM inspector has the same structural race: `verify_block_is_canonical` makes two sequential calls (`eth_getTransactionReceipt` then `eth_getBlockByNumber`), and a reorg between them yields the same spurious `NonCanonicalBlock`. [3](#0-2) 

**How `FanOut` propagates the misclassification:** [4](#0-3) 

Every error for which `is_transient()` returns `false` is pushed into `non_transient_errors`. If all inspectors agree on `NonCanonicalBlock` (which they will during a reorg, since all of them hit the same chain state), the FanOut propagates it as a substantive, terminal verdict rather than tolerating it and retrying. [5](#0-4) 

**Downstream effect on `verify_foreign_transaction`:**

A `verify_foreign_transaction` request parks a NEAR yield that must be resolved by an MPC node calling `respond_verify_foreign_tx`. If the node's foreign-chain inspection returns a non-transient error, the node does not produce a signature share, the request is never resolved, and the on-chain yield times out. [6](#0-5) 

### Impact Explanation

A legitimate `verify_foreign_transaction` request — for a transaction that is genuinely on the canonical chain — fails permanently whenever a reorg races with the two sequential RPC calls inside `verify_block_is_canonical`. The pending NEAR yield times out, the user's request is discarded, and any downstream bridge action (e.g., releasing funds on the foreign chain contingent on the MPC signature) is blocked until the user resubmits. This breaks the request-lifecycle invariant: a valid request should fail only when the foreign transaction is genuinely absent or non-canonical, not due to a transient race condition. Impact: **Medium** — request-lifecycle and contract execution-flow invariant violation.

### Likelihood Explanation

Reorgs are routine on all supported chains (Bitcoin, EVM chains). The race window is narrow (between two sequential RPC calls), but it is non-zero and grows with RPC latency. Any reorg of depth ≥ 1 that lands during an active `verify_foreign_transaction` attempt can trigger this. No privileged access is required; the condition arises from normal blockchain operation.

### Recommendation

Add `NonCanonicalBlock` (and, for consistency, `InconsistentRpcResponse`, which can also arise from a mid-reorg RPC inconsistency) to the transient set in `is_transient()`:

```rust
pub fn is_transient(&self) -> bool {
    matches!(
        self,
        Self::ClientError(_)
            | Self::RpcRequestFailed(_)
            | Self::NotFinalized
            | Self::NotEnoughBlockConfirmations { .. }
            | Self::NonCanonicalBlock { .. }       // spurious during reorg race
            | Self::InconsistentRpcResponse { .. } // spurious during reorg race
    )
}
```

This aligns the implementation with the Bitcoin inspector's own inline documentation ("The caller is expected to retry") and ensures the `FanOut` tolerates these race-condition errors rather than treating them as terminal verdicts.

### Proof of Concept

1. A user submits `verify_foreign_transaction` for a confirmed Bitcoin transaction.
2. An MPC node begins verification; `BitcoinInspector::extract` calls `getblockheader` (returns block at height H with hash A), then a reorg occurs, then `getblockhash(H)` returns hash B ≠ A.
3. `verify_block_is_canonical` returns `Err(NonCanonicalBlock { ... })`.
4. `FanOut::extract` sees `is_transient() == false`, pushes it into `non_transient_errors`, and propagates it as a terminal verdict.
5. All nodes experience the same chain state; all return `NonCanonicalBlock`; the FanOut on every node returns the same non-transient error.
6. No node produces a signature share; `respond_verify_foreign_tx` is never called; the NEAR yield times out; the user's request is permanently discarded. [7](#0-6) [8](#0-7)

/// All inner inspectors are queried concurrently. The fan-out treats every
/// non-transient outcome (success or non-transient error, see
/// [`ForeignChainInspectionError::is_transient`]) as a substantive verdict that must

/// * All substantive verdicts are `Ok` with the same extracted values → returns those values.

///   error variants, or two different success values) → returns

///
/// Variant-level comparison is used for non-transient errors, so inspectors that all report

    /// Checks that the receipt's block is on the canonical chain by resolving its height via
    /// `getblockheader` and then asking the RPC for the canonical hash at that height via
    /// `getblockhash`. `getblockhash` only ever returns canonical blocks, so a mismatch means
    /// the `getrawtransaction` response was anchored to a side block (stale tx index,
    /// partially-applied reorg, divergent RPC backend, etc.).
    ///
    /// The two RPC calls are necessarily sequential — `getblockhash`'s height parameter
    /// depends on `getblockheader`'s response — so a reorg landing between them could in
    /// principle yield a spurious `NonCanonicalBlock`. The caller is expected to retry.
    ///
    /// Failures from the RPCs themselves ("Block not found" / "block height out of range")
    /// surface as `ClientError` rather than `NonCanonicalBlock`; mapping those error
    /// messages to a more specific variant is left to a follow-up.
    async fn verify_block_is_canonical(
        &self,
        receipt_blockhash: TransportBitcoinBlockHash,
    ) -> Result<(), ForeignChainInspectionError> {
        let get_block_header_args = GetBlockHeaderArgs {
            blockhash: receipt_blockhash,
            verbose: VERBOSE_RESPONSE,
        };
        let block: GetBlockHeaderVerboseResponse = self
            .client
            .request(GET_BLOCK_HEADER_METHOD, &get_block_header_args)
            .await?;

        // Defensive: `getblockheader` looks the header up *by hash*, so a well-behaved backend
        // always echoes back the hash we queried
        if block.hash != receipt_blockhash {
            return Err(ForeignChainInspectionError::InconsistentRpcResponse {
                requested_hash: (*receipt_blockhash).to_vec().into(),
                returned_hash: (*block.hash).to_vec().into(),
            });
        }

        let get_block_hash_args = GetBlockHashArgs {
            height: block.height,
        };
        let canonical_blockhash: TransportBitcoinBlockHash = self
            .client
            .request(GET_BLOCK_HASH_METHOD, &get_block_hash_args)
            .await?;

        if canonical_blockhash != receipt_blockhash {
            return Err(non_canonical_block_error(
                block.height,
                receipt_blockhash,
                canonical_blockhash,
            ));
        }
        Ok(())
    }

    async fn verify_block_is_canonical(
        &self,
        receipt_block_number: U64,
        receipt_block_hash: H256,
    ) -> Result<(), ForeignChainInspectionError> {
        let args = GetBlockByNumberArgs::new(
            BlockNumberOrTag::Number(receipt_block_number),
            ReturnFullTransactionObjects::from(false),
        );
        let canonical: GetBlockByNumberResponse = self
            .client
            .request(GET_BLOCK_BY_NUMBER_METHOD, &args)
            .await?;

        let hash_matches = canonical.hash == receipt_block_hash;
        let height_matches = canonical.number == receipt_block_number;
        if !hash_matches || !height_matches {
            return Err(ForeignChainInspectionError::NonCanonicalBlock {
                block_number: receipt_block_number.as_u64(),
                receipt_hash: receipt_block_hash.into(),
                canonical_hash: canonical.hash.into(),
            });
        }
        Ok(())
    }

    #[handle_result]
    pub fn respond_verify_foreign_tx(
        &mut self,
        request: VerifyForeignTransactionRequest,
        response: VerifyForeignTransactionResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!(
            "respond_verify_foreign_tx: signer={}, request={:?}",
            &signer,
            &request
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain.0.into())?;

        let signature_is_valid = match (&response.signature, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");

                let payload_hash: [u8; 32] = response.payload_hash.0;

                // Check the signature is correct against the root public key
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    &payload_hash,
                    &secp_pk,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_verify_foreign_tx_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }
