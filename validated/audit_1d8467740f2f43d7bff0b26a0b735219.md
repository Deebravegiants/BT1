### Title
Unvalidated `builder` Field in `ProposalInit` Allows Malicious Proposer to Redirect Transaction Fees via Arbitrary Sequencer Address — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates many fields of `ProposalInit` (height, timestamp, starknet version, gas prices, fee proposal) but never validates the `builder` field. `convert_to_sn_api_block_info` maps `init.builder` directly to `BlockInfo::sequencer_address`, which the blockifier uses as the fee-recipient address for every transaction in the block and which is hashed into the partial block hash. A malicious proposer can set `builder` to any address, redirecting all transaction fees in that block to an attacker-controlled address and producing a block hash that encodes the wrong sequencer.

---

### Finding Description

`ProposalInit` carries a `builder: ContractAddress` field described as "Address of the one who builds/sequences the block." [1](#0-0) 

`convert_to_sn_api_block_info` converts a received `ProposalInit` into a `BlockInfo` and unconditionally assigns `init.builder` to `sequencer_address`: [2](#0-1) 

This `BlockInfo` is passed to the batcher in `initiate_validation`: [3](#0-2) 

The batcher uses `sequencer_address` as the fee-recipient for every transaction it executes. It is also captured in `PartialBlockHashComponents`: [4](#0-3) 

and hashed into the final block hash: [5](#0-4) 

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`, but contains **no check on `builder`**: [6](#0-5) [7](#0-6) 

`ProposalInitValidation` — the struct that carries the validator's reference values — has no `builder` or `sequencer_address` field at all: [8](#0-7) 

The same unvalidated `builder` value is also written into `BlockHeaderWithoutHash::sequencer` when the block is committed to state sync: [9](#0-8) 

---

### Impact Explanation

Every transaction fee paid in the block is transferred to `sequencer_address`. Because `builder` is proposer-supplied and unvalidated, a malicious proposer can set it to an attacker-controlled address. All fees for every transaction in that block are then credited to the attacker rather than the legitimate sequencer. Additionally, the partial block hash encodes the wrong sequencer address, producing a permanently incorrect on-chain block hash for that height. This matches the allowed impact: **"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."**

---

### Likelihood Explanation

The attacker must be a staked validator selected as the proposer for at least one block. In a rotating-proposer Tendermint design this is a normal, scheduled event for every participant. No external precondition beyond being a consensus participant is required; the malicious `builder` value is accepted by every honest validator without any check.

---

### Recommendation

Add `builder` (or an equivalent `expected_sequencer_address`) to `ProposalInitValidation` and reject any `ProposalInit` whose `builder` does not match the locally configured expected sequencer address for the current height. Concretely, in `is_proposal_init_valid`:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder,
            init_proposed.builder
        ),
    ));
}
```

`expected_builder` should be populated from the same source as `args.builder_address` used by the proposer in `initiate_build`. [10](#0-9) 

---

### Proof of Concept

1. Attacker is a staked validator and is selected as proposer for block N.
2. In `initiate_build`, attacker sets `builder: attacker_address` instead of the node's configured `builder_address`.
3. The `ProposalInit` with `builder = attacker_address` is broadcast to all validators.
4. Each validator calls `is_proposal_init_valid` — no check on `builder` — and proceeds.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: attacker_address, … }`.
6. The batcher executes all transactions with `sequencer_address = attacker_address`; every fee transfer credits the attacker.
7. `PartialBlockHashComponents` encodes `attacker_address` as the sequencer; the block hash is permanently wrong.
8. `ProposalFin` comparison passes because both proposer and all validators computed the commitment from the same (attacker-supplied) `builder` value.
9. The block is finalized; the attacker has received all transaction fees for block N.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
