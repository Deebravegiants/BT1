### Title
Users' Pending ETH Withdrawals Earn Aave Interest That Is Silently Diverted to the Treasury - (File: contracts/LRTWithdrawalManager.sol)

---

### Summary

When `unlockQueue` is called, users' ETH is redeemed from `LRTUnstakingVault` and immediately deposited into Aave v3. The Aave interest that accrues on this ETH during the withdrawal delay window is explicitly excluded from user payouts and is instead routed to the protocol treasury. Users receive only their locked `expectedAssetAmount` (principal), losing all yield earned on their ETH while it sits in Aave.

---

### Finding Description

The withdrawal lifecycle in `LRTWithdrawalManager` has three stages:

**Stage 1 – `initiateWithdrawal`:** The user's rsETH is transferred to the contract and `expectedAssetAmount` is locked at the current oracle price.

**Stage 2 – `unlockQueue` (operator-triggered):** rsETH is burned, ETH is redeemed from `LRTUnstakingVault`, and — if Aave integration is enabled — the entire unlocked ETH amount is deposited into Aave v3: [1](#0-0) 

**Stage 3 – `completeWithdrawal`:** When the user claims, `_withdrawFromAave` is called with an explicit cap that prevents withdrawing any accrued interest: [2](#0-1) 

The comment `// Only withdraw up to the principal amount (don't use accrued interest for user withdrawals)` confirms this is intentional. The interest is later swept to the treasury via `_collectInterestToTreasury`: [3](#0-2) 

The user's ETH — which belongs to the user from the moment `unlockQueue` burns their rsETH and redeems the asset — earns Aave yield during the entire withdrawal delay period (default 8 days, up to 16 days), and none of that yield is returned to the user.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Every ETH withdrawal processed through `unlockQueue` when Aave integration is enabled results in the user's ETH earning Aave interest that is diverted to the treasury. The user's rsETH is already burned at `unlockQueue` time, so the ETH is unambiguously the user's asset. The protocol extracts yield from it without disclosure or consent. At Aave WETH supply rates (historically 1–5% APY) over an 8-day delay, each user loses approximately 0.02–0.11% of their withdrawal amount. Across all withdrawals this constitutes a systematic, ongoing extraction of user yield.

---

### Likelihood Explanation

**Medium.** The Aave integration must be enabled by the manager (`setAaveIntegrationEnabled`), which is a privileged but routine operational action. Once enabled, every ETH withdrawal processed through `unlockQueue` is affected automatically. No special attacker action is required — the loss occurs passively for every withdrawing user. [4](#0-3) 

---

### Recommendation

When processing `completeWithdrawal`, withdraw the user's `expectedAssetAmount` from Aave without capping at `totalETHDepositedToAave`. Alternatively, track each user's deposited principal individually and return any pro-rata interest earned on their specific ETH to them at claim time. If the protocol intends to keep Aave yield, this should be disclosed as an explicit fee and bounded.

---

### Proof of Concept

1. Alice calls `initiateWithdrawal(ETH, 10 rsETH)`. Her rsETH is transferred to the contract; `expectedAssetAmount = 10 ETH` is locked.
2. Operator calls `unlockQueue(ETH, ...)`. Alice's rsETH is burned, 10 ETH is redeemed from `LRTUnstakingVault`, and `_depositToAave(10 ETH)` is called — `totalETHDepositedToAave += 10 ETH`.
3. Over 8 days, Aave accrues ~0.002 ETH interest. `aaveAWETH.balanceOf(address(this))` = 10.002 ETH.
4. Alice calls `completeWithdrawal(ETH)`. Inside `_processWithdrawalCompletion`, `_withdrawFromAave(10 ETH)` is called. The cap `withdrawablePrincipal = min(10.002, 10) = 10 ETH` limits the withdrawal to 10 ETH.
5. Alice receives exactly 10 ETH. The 0.002 ETH interest remains in Aave.
6. Operator later calls `collectInterestToTreasury()`. The 0.002 ETH is withdrawn from Aave and sent to the treasury. [5](#0-4) [6](#0-5)

    function _withdrawFromAave(uint256 amount) internal returns (uint256 withdrawnAmount) {
        if (amount == 0) return 0;

        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        if (aaveBalance == 0) revert InsufficientAaveBalance();

        // Only withdraw up to the principal amount (don't use accrued interest for user withdrawals)
        uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave ? aaveBalance : totalETHDepositedToAave;

        withdrawnAmount = amount > withdrawablePrincipal ? withdrawablePrincipal : amount;
        if (withdrawnAmount == 0) return 0;

        aaveWETHGateway.withdrawETH(aavePool, withdrawnAmount, address(this));
        totalETHDepositedToAave -= withdrawnAmount;

        emit ETHWithdrawnFromAave(withdrawnAmount, totalETHDepositedToAave);
    }

    function _collectInterestToTreasury() internal returns (uint256 interestAmount) {
        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        uint256 principal = totalETHDepositedToAave;

        // Return 0 if no interest or balance is less than principal (accounting for rounding)
        if (aaveBalance <= principal) return 0;

        interestAmount = aaveBalance - principal;

        aaveWETHGateway.withdrawETH(aavePool, interestAmount, address(this));

        address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        (bool sent,) = payable(treasury).call{ value: interestAmount }("");
        if (!sent) revert TreasuryTransferFailed();
