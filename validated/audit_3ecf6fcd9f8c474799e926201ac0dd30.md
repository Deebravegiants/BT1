Audit Report

## Title
`KernelMerkleDistributor.claimAndStake` and `KernelTop100MerkleDistributor.claimAndStake` Unconditionally Revert Due to Missing `STAKE_FOR_ROLE` Grant - (File: contracts/KERNEL/KernelMerkleDistributor.sol, contracts/KERNEL/KernelTop100MerkleDistributor.sol)

## Summary

`KernelMerkleDistributor.claimAndStake` calls `IKernelDepositPool(kernelDepositPool).stakeFor(account, amountToStake)` at line 282. `KernelDepositPool.stakeFor` is protected by `onlyRole(STAKE_FOR_ROLE)`. Neither `KernelMerkleDistributor.initialize` nor `KernelDepositPool.initialize` ever grants `STAKE_FOR_ROLE` to the distributor, and no function inside the distributor can acquire it. Every call to `claimAndStake` therefore reverts unconditionally. The same defect exists in `KernelTop100MerkleDistributor.claimAndStake` at line 371.

## Finding Description

`KernelDepositPool` defines `STAKE_FOR_ROLE = keccak256("STAKE_FOR_ROLE")` and guards `stakeFor` with `onlyRole(STAKE_FOR_ROLE)`: [1](#0-0) [2](#0-1) 

`KernelDepositPool.initialize` only grants `DEFAULT_ADMIN_ROLE` to `_admin`; no distributor address receives `STAKE_FOR_ROLE`: [3](#0-2) 

`KernelMerkleDistributor.initialize` sets a token approval for the pool but never calls `grantRole`: [4](#0-3) 

`claimAndStake` then calls `stakeFor` without holding the role: [5](#0-4) 

The same pattern exists in `KernelTop100MerkleDistributor`: [6](#0-5) 

No function anywhere in either distributor contract calls `grantRole` on `KernelDepositPool`, and neither distributor inherits `AccessControl`. The only path to fix this is an out-of-band admin call to `KernelDepositPool.grantRole(STAKE_FOR_ROLE, address(distributor))` — a step that is absent from the initialization flow and undocumented.

## Impact Explanation

Every invocation of `claimAndStake` by any user reverts at the `stakeFor` call. The atomic claim-and-stake flow advertised by both contracts is permanently non-functional in the as-deployed state. No funds are lost (the revert rolls back all state changes), but the contracts fail to deliver their promised single-transaction claim-and-stake functionality. This matches the allowed impact: **Low — Contract fails to deliver promised returns, but doesn't lose value**.

## Likelihood Explanation

The failure is unconditional and immediate: any user who calls `claimAndStake` with a valid Merkle proof will receive a revert. No special attacker capability is required; the revert is triggered by normal user interaction with a publicly advertised function.

## Recommendation

After deploying `KernelMerkleDistributor` (and `KernelTop100MerkleDistributor`), the `DEFAULT_ADMIN_ROLE` holder of `KernelDepositPool` must call:

```solidity
KernelDepositPool.grantRole(STAKE_FOR_ROLE, address(kernelMerkleDistributor));
KernelDepositPool.grantRole(STAKE_FOR_ROLE, address(kernelTop100MerkleDistributor));
```

To make this explicit and auditable, add a dedicated admin function inside each distributor (e.g., `grantStakeForRole`) that calls `KernelDepositPool.grantRole`, or perform the grant atomically inside `initialize` if the deployer controls both contracts.

## Proof of Concept

1. Deploy `KernelDepositPool` with admin `A`; deploy `KernelMerkleDistributor` pointing at the pool.
2. Confirm: `KernelDepositPool.hasRole(STAKE_FOR_ROLE, address(distributor))` returns `false`.
3. Admin sets a Merkle root; user `U` has a valid leaf `(index, U, amount)`.
4. `U` calls `claimAndStake(index, U, amount, proof)`.
5. `_processClaim` succeeds; execution reaches line 282: `IKernelDepositPool(kernelDepositPool).stakeFor(U, amountToStake)`.
6. `KernelDepositPool.stakeFor` executes `onlyRole(STAKE_FOR_ROLE)` — distributor does not hold the role → revert with `AccessControl` error.
7. Entire transaction reverts. `U` receives nothing. `claimAndStake` is non-functional for all callers.

    function claimAndStake(uint256 amount, bytes32[] calldata merkleProof) external nonReentrant whenNotPaused {
        address user = msg.sender;

        // Verify merkle proof and update user claim data
        _verifyClaimProof(user, amount, merkleProof);

        // Get claimable amount
        uint256 claimableAmount = _getUnclaimedVestedAmount(user, amount);

        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim data
        userClaims[user].lastClaimTimestamp = block.timestamp;
        userClaims[user].amountClaimed += claimableAmount;

        // Calculate fee
        uint256 fee = (claimableAmount * feeInBPS) / FEE_DENOMINATOR;
        uint256 amountToStake = claimableAmount - fee;

        // Transfer fee and stake tokens
        if (fee > 0) {
            kernel.safeTransfer(protocolTreasury, fee);
        }

        kernelDepositPool.stakeFor(user, amountToStake);

        emit ClaimedAndStaked(user, amountToStake);
    }
