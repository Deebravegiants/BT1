### Title
Expired-Attestation Participant Bypasses TEE Authorization in `respond_verify_foreign_tx` via Bounded Scan Gap — (`crates/contract/src/tee/tee_state.rs`)

### Summary

`is_caller_an_attested_participant` never re-verifies attestation validity (expiry, image hash, measurements). It only checks that a stored entry exists and that the account/key fields match. The automatic post-reshare cleanup (`clean_invalid_attestations`) is bounded to 100 entries, so an expired entry positioned beyond that window survives indefinitely. A current participant whose attestation has expired can therefore continue to call `respond_verify_foreign_tx` and have the contract accept their responses.

### Finding Description

**Root cause — `is_caller_an_attested_participant` performs no re-verification:** [1](#0-0) 

The function checks only: (1) caller is in `participants`, (2) a stored attestation exists for their TLS key, (3) `account_id` matches, (4) `account_public_key` matches. It never calls `reverify_participants`, never checks expiry, never checks the image hash or measurements. An entry that would fail `re_verify` is indistinguishable from a valid one at this gate.

**Contributing factor — bounded scan leaves expired entries intact:** [2](#0-1) 

The automatic post-reshare sweep is invoked with `max_scan = RESHARE_CLEAN_INVALID_ATTESTATIONS_MAX_SCAN = 100`: [3](#0-2) [4](#0-3) 

Any expired entry at iteration position > 100 is not touched by the automatic sweep. Because `clean_invalid_attestations` is a public endpoint callable by anyone with an arbitrary `max_scan`, a determined observer *could* call it with a larger window — but there is no guarantee this happens, and the contract itself never enforces it.

**Call path to impact:** [5](#0-4) [6](#0-5) 

`respond_verify_foreign_tx` → `assert_caller_is_attested_participant_and_protocol_active` → `is_caller_an_attested_participant`. The only attestation gate on this path is the existence/key-match check; no validity re-check occurs.

The response must also pass the MPC-root-key signature check: [7](#0-6) 

A node with an expired attestation still holds its MPC key shares (key shares do not expire with the attestation), so it can produce a signature that passes this check.

### Impact Explanation

A participant node running an expired or revoked TEE image — one that should no longer be trusted — can submit `respond_verify_foreign_tx` calls that the contract accepts as authoritative. The contract resolves the pending yield and returns the response to the original caller as if it came from a trusted TEE. This enables forged foreign-chain verification and invalid bridge execution.

### Likelihood Explanation

Attestations expire naturally (certificate lifetimes, image-hash rotation). The window between expiry and cleanup is non-zero in every deployment. The bounded scan makes the window indefinitely long when `stored_attestations` exceeds 100 entries. The attacker does not need to do anything unusual beyond letting their attestation expire while remaining in the active participant set.

The constraint that the attacker must be a **current active participant** (not a fully external party) reduces likelihood somewhat, but a Byzantine participant below the signing threshold is explicitly within scope.

### Recommendation

1. **Re-verify in `is_caller_an_attested_participant`**: call `reverify_participants` (or an equivalent inline check) and return `Err` if the stored attestation fails re-verification. This closes the root cause regardless of cleanup timing.
2. **Or cache a validity bit**: store a `valid_until` timestamp alongside each `NodeAttestation` and reject callers whose cached expiry has passed, refreshed on each successful `submit_participant_info`.
3. **Increase or remove the scan cap**: if re-verification at call time is not adopted, the automatic sweep should cover the entire map (or be called in a loop until convergence) so no expired entry can outlast a resharing.

### Proof of Concept

```rust
// Insert 101 expired attestations so the target lands at position 101.
// Call clean_invalid_attestations(max_scan=100).
// Assert target entry survives.
// Call is_caller_an_attested_participant for the target.
// Assert Ok(()) is returned despite the expired attestation.

let expired = Attestation::Mock(MockAttestation::WithConstraints {
    expiry_timestamp_seconds: Some(1_000), // expires at t=1000
    ..
});
for i in 0..101 {
    tee_state.add_participant(make_node(i), expired.clone(), Duration::ZERO).unwrap();
}
// advance clock past expiry
set_block_timestamp(5_000 * 1_000_000_000);

// automatic post-reshare sweep
tee_state.clean_invalid_attestations(Duration::ZERO, 100);

// target at position 101 survives
assert!(tee_state.stored_attestations.contains_key(&target_tls_key));

// is_caller_an_attested_participant returns Ok(()) — no expiry check
set_signer(&target_account_id, &target_account_pk);
assert_matches!(
    tee_state.is_caller_an_attested_participant(&participants_containing_target),
    Ok(())
);
```

```rust
    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
```
