Audit Report

## Title
Stale Rate Used for Swap Pricing Due to Missing Freshness Check in `CrossChainRateReceiver.getRate()` - (File: `contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary
`CrossChainRateReceiver.getRate()` returns the cached `rate` storage variable without consulting `lastUpdated`, which is recorded on every `lzReceive()` call. All pool contracts route swap pricing through `IOracle(rsETHOracle).getRate()`, which resolves to this function when the oracle is a `CrossChainRateReceiver`. If no cross-chain update arrives for an extended period, every subsequent deposit is priced at an arbitrarily old rate, causing the pool to over- or under-pay depositors relative to the true current rsETH/ETH exchange rate.

## Finding Description
`CrossChainRateReceiver` stores `rate` and `lastUpdated` as public state variables. Both are written atomically in `lzReceive()`:

```solidity
// CrossChainRateReceiver.sol L95-97
rate = _rate;
lastUpdated = block.timestamp;
```

However, `getRate()` ignores `lastUpdated` entirely:

```solidity
// CrossChainRateReceiver.sol L102-105
function getRate() external view returns (uint256) {
    return rate;
}
```

`RSETHPool.getRate()` delegates unconditionally to this function:

```solidity
// RSETHPool.sol L254-256
function getRate() public view returns (uint256) {
    return IOracle(rsETHOracle).getRate();
}
```

The returned value is used directly in `viewSwapRsETHAmountAndFee()`, which drives both ETH and ERC-20 deposit paths:

```solidity
// RSETHPool.sol L316-319
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
```

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` explicitly validates freshness before returning a price (`answeredInRound < roundID` and `timestamp == 0` checks). No equivalent guard exists in `CrossChainRateReceiver`. The `updateRate()` function on the provider side is permissionless but requires the caller to supply ETH for LayerZero fees; if the keeper stops or is underfunded, the receiver's `rate` silently ages with no on-chain detection.

## Impact Explanation
rsETH is a yield-bearing token whose ETH value increases monotonically. A stale (lower-than-current) `rate` causes `rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate` to yield more rsETH per ETH than the current exchange rate warrants, overpaying depositors at the pool's expense. A stale rate that is higher than current (e.g., after a slashing event) underpays depositors. In both cases the pool fails to deliver the exchange rate it promises. This matches the allowed impact: **Low — Contract fails to deliver promised returns, but doesn't lose value.**

## Likelihood Explanation
Any normal depositor calling `deposit()` triggers the vulnerable path. No attacker action is required; the condition arises passively whenever the off-chain keeper fails to call `updateRate()` (infrastructure outage, insufficient ETH for LZ fees, relayer halt). This is a realistic operational scenario, not an admin compromise or governance attack. The vulnerability is repeatable for every deposit made while the rate is stale.

## Recommendation
Introduce a configurable maximum staleness threshold and revert in `getRate()` when the rate is too old:

```solidity
uint256 public maxStaleness; // e.g., 24 hours, set by owner

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= maxStaleness, "Rate is stale");
    return rate;
}
```

`maxStaleness` should be settable by the owner and initialized to a value consistent with the expected update frequency (e.g., 24 hours).

## Proof of Concept
```solidity
// Foundry fork test (Arbitrum fork, CrossChainRateReceiver as rsETHOracle)
function testStaleRate() external {
    // 1. Simulate a rate update at T=0
    vm.prank(layerZeroEndpoint);
    rateReceiver.lzReceive(srcChainId, srcAddressEncoded, 0, abi.encode(1.05e18));

    uint256 rateAtT0    = rateReceiver.getRate();      // 1.05e18
    uint256 updatedAtT0 = rateReceiver.lastUpdated();  // block.timestamp

    // 2. Advance time by 7 days — no lzReceive called
    vm.warp(block.timestamp + 7 days);

    // 3. getRate() still returns the old value — no revert
    assertEq(rateReceiver.getRate(), rateAtT0);
    assertEq(rateReceiver.lastUpdated(), updatedAtT0);

    // 4. Pool uses stale rate for swap
    (uint256 rsETHAmount,) = pool.viewSwapRsETHAmountAndFee(1 ether);

    // 5. Compare against current on-chain price (higher after 7 days of yield)
    uint256 currentRate   = ILRTOracle(rsETHPriceOracle).rsETHPrice(); // e.g. 1.06e18
    uint256 expectedAmount = 1 ether * 1e18 / currentRate;

    // rsETHAmount > expectedAmount — pool overpays depositors
    assertGt(rsETHAmount, expectedAmount);
}
```
