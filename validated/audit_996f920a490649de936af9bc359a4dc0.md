### Title
Missing Consistency Guards in `get_public_key_compressed` Allow Attacker-Chosen Public Key Substitution — (`src/cbmpc/api/ecdsa_mp.cpp`)

---

### Summary

`coinbase::api::ecdsa_mp::get_public_key_compressed` deserializes `Q_compressed` from an attacker-supplied blob and returns it as the authoritative public key without verifying that `Q == sum(Qis)` or that `x_share * G == Qi_self`. Both guards are present in `deserialize_key_blob` (used for signing) but are entirely absent in the public-key-extraction path. Any caller who can supply a crafted blob receives an attacker-chosen curve point as the "real" public key.

---

### Finding Description

`get_public_key_compressed` performs only three checks before returning `Q`: [1](#0-0) 

1. Size limit (line 460–462)
2. Version is `key_blob_version_v1` (1) or `ac_key_blob_version_v1` (2) (line 466–467)
3. Curve is valid and not ed25519 (lines 469–472)
4. `Q.from_bin` succeeds — i.e., the bytes decode to a point on the curve (line 475)

It then immediately returns `Q.to_compressed_bin()` (line 477).

By contrast, `deserialize_key_blob` — the path used for every signing call — enforces two additional invariants after parsing the same fields: [2](#0-1) 

- **`Q == sum(Qis)`** (line 174): the global public key must equal the sum of all per-party share public keys.
- **`x_share * G == Qi_self`** (line 179): the private scalar share must be consistent with the self share public key.

Neither guard appears anywhere in `get_public_key_compressed`. An attacker who constructs a blob with:
- `version = 1` (or `2`)
- a valid `curve` field
- `Q_compressed` = encoding of any attacker-chosen curve point `Q'`
- arbitrary (or zeroed) `Qis_compressed` and `x_share`

will receive `Q'` back from the API with `SUCCESS`, with no indication that the blob is inconsistent.

The same gap exists in the C-level wrapper `cbmpc_ecdsa_mp_get_public_key_compressed`: [3](#0-2) 

---

### Impact Explanation

An application that calls `get_public_key_compressed` to derive a blockchain address or to display/verify the shared public key will receive the attacker-substituted point `Q'` as the authoritative answer. This enables:

- **Address substitution**: funds sent to the address derived from `Q'` are unrecoverable by the honest parties (whose actual shared key is a different point).
- **Key-identity confusion**: any downstream check that compares the returned public key against an expected value (e.g., a certificate, a smart-contract address, a BIP-32 derivation root) will silently accept the wrong key.

Signing is unaffected because `sign` calls `deserialize_key_blob`, which does enforce both guards. The vulnerability is scoped to the public-key-extraction API path.

---

### Likelihood Explanation

The blob format is opaque bytes (`mem_t`). Any caller of the public C API (`cbmpc_ecdsa_mp_get_public_key_compressed`) or the C++ API (`coinbase::api::ecdsa_mp::get_public_key_compressed`) can supply arbitrary bytes. No authentication of the blob is required. Constructing a passing blob requires only: a valid version byte, a valid curve identifier, and a compressed encoding of any curve point — all trivially achievable without any cryptographic secret.

---

### Recommendation

Apply the same two consistency checks that `deserialize_key_blob` already enforces. Specifically, inside `get_public_key_compressed`, after parsing `Q` and all `Qis`:

1. Compute `Q_sum = sum(Qis)` and reject if `Q != Q_sum`.
2. Reject if `blob.x_share * G != Qis[blob.party_name]`.

Alternatively, refactor `get_public_key_compressed` to call `deserialize_key_blob` (which requires a `job_mp_t`) or extract the shared validation logic into a helper used by both paths, eliminating the divergence.

---

### Proof of Concept

```cpp
// Build a blob where Q_compressed encodes an attacker-chosen point A.
// All other fields are set to pass the minimal checks in get_public_key_compressed.

key_blob_v1_t crafted;
crafted.version = 1;                          // passes version check
crafted.curve   = (uint32_t)curve_id::secp256k1; // passes curve check
// Set Q_compressed to the generator G (or any other point the attacker controls).
crafted.Q_compressed = curve_secp256k1.generator().to_compressed_bin();
// x_share and Qis_compressed can be anything; they are never read by
// get_public_key_compressed.
crafted.x_share = 0;

buf_t serialized = coinbase::convert(crafted);

buf_t pub_key;
error_t rv = coinbase::api::ecdsa_mp::get_public_key_compressed(mem_t(serialized), pub_key);

// rv == SUCCESS
// pub_key == encoding of G  (attacker-chosen point, not the real shared key)
assert(rv == SUCCESS);
assert(pub_key == curve_secp256k1.generator().to_compressed_bin());
```

The test passes without error, and `pub_key` contains the attacker-chosen point rather than the legitimate shared public key established during DKG.

error_t get_public_key_compressed(mem_t key_blob, buf_t& pub_key) {
  if (const error_t rv = coinbase::api::detail::validate_mem_arg_max_size(key_blob, "key_blob",
                                                                          coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))
    return rv;
  key_blob_v1_t blob;
  error_t rv = coinbase::convert(blob, key_blob);
  if (rv) return rv;
  if (blob.version != key_blob_version_v1 && blob.version != ac_key_blob_version_v1)
    return coinbase::error(E_FORMAT, "unsupported key blob version");

  const auto cid = static_cast<curve_id>(blob.curve);
  if (cid == curve_id::ed25519) return coinbase::error(E_FORMAT, "invalid key blob curve");
  const auto curve = to_internal_curve(cid);
  if (!curve.valid()) return coinbase::error(E_FORMAT, "invalid key blob curve");

  coinbase::crypto::ecc_point_t Q(curve);
  if (rv = Q.from_bin(curve, blob.Q_compressed)) return coinbase::error(rv, "invalid key blob");

  pub_key = Q.to_compressed_bin();
  return SUCCESS;
}
