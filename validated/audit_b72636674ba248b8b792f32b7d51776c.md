### Title
Donation of 1 wei permanently blocks `removeSupportedToken`, preventing admin from removing compromised or deprecated tokens - (File: `contracts/pools/RSETHPoolV3.sol`, `contracts/pools/RSETHPoolV3ExternalBridge.sol`)

### Summary
Both `RSETHPoolV3` and `RSETHPoolV3ExternalBridge` guard `removeSupportedToken` with a strict zero-balance check on the live ERC-20 balance. Because any external actor can donate a single wei of a supported token directly to the contract, this check can be made to revert permanently, preventing the `TIMELOCK_ROLE` from ever removing a token from the supported list.

### Finding Description
`removeSupportedToken` in both pool contracts contains the following guard:

```solidity
if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();
``` [1](#0-0) [2](#0-1) 

The check reads the raw on-chain ERC-20 balance of the contract, which is not controlled by any internal accounting variable. Any address can call `IERC20(token).transfer(pool, 1)` to donate 1 wei of a supported token to the pool. After that transfer, `balanceOf(pool) >= 1`, so the guard always reverts and the token can never be removed.

The `feeEarnedInToken[token]` accounting variable is never consulted here; only the raw balance is checked. There is no sweep or recovery path that resets the balance to zero without also requiring the token to already be removable. [3](#0-2) 

### Impact Explanation
The `TIMELOCK_ROLE` is permanently unable to remove a supported token from `supportedTokenList` and `supportedTokenOracle`. Concrete consequences:

- If a token's oracle becomes stale or is manipulated, the admin cannot remove the token to halt further deposits at incorrect exchange rates.
- If a token is deprecated or its contract is compromised, it cannot be delisted; users continue to be able to deposit it and receive `wrsETH` minted against a potentially incorrect rate.
- The `supportedTokenList` array and `supportedTokenOracle` mapping are permanently polluted.

The admin retains the ability to update the oracle via `setSupportedTokenOracle` and to pause the contract, but cannot fully delist the token. This matches **Low – Contract fails to deliver promised returns, but doesn't lose value**.

### Likelihood Explanation
Trivially high. The attacker needs only to call `token.transfer(pool, 1)` (cost: 1 wei + gas). The action is irreversible, requires no special role, and can be repeated for every supported token. There is no economic barrier.

### Recommendation
Replace the raw-balance check with an internal accounting variable that tracks only protocol-deposited tokens (excluding direct transfers). For example, maintain a `mapping(address => uint256) public tokenDeposited` that is incremented on `deposit` and decremented on `moveAssetsForBridging`/`withdrawFees`, and check that variable instead of `balanceOf`. Alternatively, add a permissioned `sweepDonatedTokens` function that transfers any balance exceeding `feeEarnedInToken[token]` to the treasury before the removal check is evaluated.

### Proof of Concept
1. `RSETHPoolV3` (or `RSETHPoolV3ExternalBridge`) has token `T` in its `supportedTokenList`.
2. Attacker calls `T.transfer(address(pool), 1)` — costs 1 wei + gas.
3. Admin (TIMELOCK_ROLE) calls `pool.removeSupportedToken(T, index)`.
4. Execution reaches `if (IERC20(T).balanceOf(address(pool)) != 0) revert TokenBalanceNotZero()` — `balanceOf` returns at least 1, so the call reverts.
5. Step 3 will revert for every future call as long as the donated wei remains in the contract, which is permanent because there is no mechanism to drain it without first removing the token. [1](#0-0) [2](#0-1)

    function removeSupportedToken(address token, uint256 tokenIndex) external onlyRole(TIMELOCK_ROLE) {
        UtilLib.checkNonZeroAddress(token);
        if (supportedTokenList[tokenIndex] != token) revert TokenNotFoundError();
        if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();

        delete supportedTokenOracle[token];
        supportedTokenList[tokenIndex] = supportedTokenList[supportedTokenList.length - 1];
        supportedTokenList.pop();
        emit RemovedSupportedToken(token);
    }
```
