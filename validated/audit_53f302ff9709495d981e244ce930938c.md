### Title
Single Byzantine Participant Can Permanently Freeze All Foreign-Chain Verification Requests via Wrong Chain-Support Lookup - (`File: crates/contract/src/lib.rs`)

---

### Summary

`verify_foreign_transaction` gates on `get_supported_foreign_chains()`, which uses an all-participant intersection rule. A single active participant below the signing threshold can register an empty chain list, collapsing the intersection to the empty set and causing every `verify_foreign_transaction` call to panic-reject with `ForeignChainNotSupported`. The correct function — `get_available_foreign_chains()`, which requires only `signing_threshold` participants to cover a chain — is already implemented but is not used at the call site.

---

### Finding Description

`verify_foreign_transaction` performs a chain-support check at:

```rust
let requested_chain = request.request.chain();
let supported_chains = self.get_supported_foreign_chains();   // ← wrong function
if !supported_chains.contains(&requested_chain) {
    env::panic_str(&InvalidParameters::ForeignChainNotSupported { ... }.to_string());
}
``` [1](#0-0) 

`get_supported_foreign_chains()` computes the **strict intersection** of every active participant's registered chain set:

```rust
let all_active_nodes_supports_chain =
    nodes_supporting_chain.is_superset(&active_participant_account_ids);
``` [2](#0-1) 

This reads from `node_foreign_chain_support`, which is populated by `register_foreign_chain_support`. Any active participant can call that method with an empty `SupportedForeignChains`:

```rust
pub fn register_foreign_chain_support(
    &mut self,
    foreign_chain_support: dtos::SupportedForeignChains,
) -> Result<(), Error> {
    let account_id = self.voter_or_panic();
    self.node_foreign_chain_support
        .foreign_chain_support_by_node
        .insert(account_id, foreign_chain_support);
``` [3](#0-2) 

Once one participant's entry is empty, the intersection across all active participants is empty, so `get_supported_foreign_chains()` returns `{}`. Every subsequent `verify_foreign_transaction` call panics regardless of which chain is requested.

The correct function — `get_available_foreign_chains()` — is already implemented and uses the threshold-based cache in `ForeignChainsMetadata.available_foreign_chains`, which requires only `signing_threshold` participants to cover a chain:

```rust
self.available_foreign_chains = chain_to_supporter_count
    .into_iter()
    .filter_map(|(chain, count)| (count >= threshold).then_some(chain))
    .collect::<BTreeSet<_>>()
    .into();
``` [4](#0-3) 

The design documentation explicitly identifies this as the required fix:

> `verify_foreign_transaction(C)` is **rejected unless `C` is available** … The legacy `get_supported_foreign_chains()` (the intersection rule) is **to be deprecated** in favour of the two views above.



The two storage paths are entirely separate: `register_foreign_chain_support` → `node_foreign_chain_support` (old, account-keyed) feeds `get_supported_foreign_chains()`; `register_foreign_chains_config` → `foreign_chains` (new, TLS-key-keyed) feeds `get_available_foreign_chains()`. The call site in `verify_foreign_transaction` reads from the wrong one.

---

### Impact Explanation

Every `verify_foreign_transaction` call panics with `ForeignChainNotSupported` for every chain, permanently freezing the foreign-chain verification flow. Bridge inbound flows (foreign chain → NEAR) that depend on MPC-attested foreign-transaction proofs cannot complete. No funds can be released or attested until the Byzantine participant re-registers a non-empty chain list. This is a request-lifecycle invariant break: a chain that `signing_threshold` honest participants cover is incorrectly treated as unsupported.

**Allowed impact matched**: Medium — request-lifecycle manipulation that breaks production safety/accounting invariants without relying on network-level DoS or operator misconfiguration.

---

### Likelihood Explanation

Any single active participant (strictly below the signing threshold) can trigger this with one on-chain call. No collusion, no key material, no special tooling is required. The call is authenticated only by participant membership (`voter_or_panic()`), which every active node satisfies. The attack is persistent until the attacker voluntarily re-registers a non-empty set.

---

### Recommendation

Replace the call to `get_supported_foreign_chains()` in `verify_foreign_transaction` with `get_available_foreign_chains()`:

```rust
// Before (wrong — all-participant intersection):
let supported_chains = self.get_supported_foreign_chains();

// After (correct — threshold-based cache):
let supported_chains = self.get_available_foreign_chains();
``` [1](#0-0) 

This is consistent with the stated design intent in `docs/design/calculating-supported-foreign-chains.md` and with the already-implemented `recompute_available_foreign_chains` / `update_available_chains_config_cache` infrastructure. [5](#0-4) 

---

### Proof of Concept

1. Network is `Running` with 4 participants, signing threshold 3. All 4 participants have previously registered Bitcoin support via `register_foreign_chain_support`. `verify_foreign_transaction(Bitcoin)` succeeds.

2. Participant P1 (one of the 4, below threshold) calls:
   ```
   register_foreign_chain_support({ foreign_chain_support: [] })
   ```
   This inserts `P1 → {}` into `node_foreign_chain_support`.

3. `get_supported_foreign_chains()` now computes: for Bitcoin, `supporters = {P2, P3, P4}`, but `active_participants = {P1, P2, P3, P4}`. `{P2,P3,P4}.is_superset({P1,P2,P3,P4})` is `false`. Bitcoin is excluded. Result: `{}`.

4. Any caller submits `verify_foreign_transaction(Bitcoin, ...)`. The contract panics: `"Requested foreign chain, Bitcoin, is not supported."` — even though 3 of 4 participants (≥ threshold) cover Bitcoin.

5. The correct `get_available_foreign_chains()` would return `{Bitcoin}` because `count(Bitcoin) = 3 ≥ threshold = 3`, but it is never consulted. [6](#0-5) [7](#0-6)

    pub fn get_supported_foreign_chains(&self) -> dtos::SupportedForeignChains {
        let active_participant_account_ids = self
            .protocol_state
            .active_participants()
            .participants()
            .iter()
            .map(|(account_id, _, _)| account_id.clone())
            .collect::<BTreeSet<_>>();

        let mut foreign_chain_to_node_mapping: BTreeMap<
            &dtos::ForeignChain,
            BTreeSet<dtos::AccountId>,
        > = BTreeMap::new();

        for (account_id, chains) in self
            .node_foreign_chain_support
            .foreign_chain_support_by_node
            .iter()
        {
            for chain in chains.iter() {
                foreign_chain_to_node_mapping
                    .entry(chain)
                    .or_default()
                    .insert(account_id.clone());
            }
        }

        foreign_chain_to_node_mapping
            .into_iter()
            .filter_map(|(foreign_chain, nodes_supporting_chain)| {
                let all_active_nodes_supports_chain =
                    nodes_supporting_chain.is_superset(&active_participant_account_ids);

                if all_active_nodes_supports_chain {
                    Some(foreign_chain)
                } else {
                    None
                }
            })
            .cloned()
            .collect::<BTreeSet<dtos::ForeignChain>>()
            .into()

    pub(crate) fn update_available_chains_config_cache(
        &mut self,
        active_tls_keys: &BTreeSet<dtos::Ed25519PublicKey>,
        threshold: u64,
    ) {
        let mut chain_to_supporter_count: std::collections::BTreeMap<dtos::ForeignChain, u64> =
            std::collections::BTreeMap::new();
        for tls_key in active_tls_keys {
            let Some(chains) = self.foreign_chains_configs.get(tls_key) else {
                continue;
            };
            for chain in chains.iter() {
                if self.rpc_whitelist.entries.is_whitelisted(chain) {
                    let count = chain_to_supporter_count.entry(*chain).or_default();
                    *count = count
                        .checked_add(1)
                        .expect("supporter count bounded by participant set size");
                }
            }
        }
        self.available_foreign_chains = chain_to_supporter_count
            .into_iter()
            .filter_map(|(chain, count)| (count >= threshold).then_some(chain))
            .collect::<BTreeSet<_>>()
            .into();
    }
