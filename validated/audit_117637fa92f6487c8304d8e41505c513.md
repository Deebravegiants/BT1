Audit Report

## Title
`RSETHPriceFeed` Returns ETH/USD-Only Staleness Metadata for Composite rsETH/USD Price - (File: contracts/oracles/RSETHPriceFeed.sol)

## Summary
`RSETHPriceFeed` computes rsETH/USD by multiplying `RS_ETH_ORACLE.rsETHPrice()` (rsETH/ETH) with the ETH/USD Chainlink answer, but the returned `updatedAt` and `answeredInRound` values are sourced exclusively from the ETH/USD feed. `LRTOracle` stores no last-update timestamp for `rsETHPrice`, so `RSETHPriceFeed` has no mechanism to reflect the rsETH component's staleness. Any consumer applying a standard Chainlink staleness guard will pass the check even when the rsETH/ETH rate is hours or days old.

## Finding Description
In `latestRoundData()` (L63–70) and `getRoundData()` (L53–61) of `contracts/oracles/RSETHPriceFeed.sol`, all five return values are first populated from `ETH_TO_USD.latestRoundData()` / `ETH_TO_USD.getRoundData()`. Only `answer` is then overwritten with the composite price:

```solidity
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
```

`updatedAt` and `answeredInRound` remain those of the ETH/USD feed. `LRTOracle` (`contracts/LRTOracle.sol` L28–29) stores `rsETHPrice` as a plain `uint256` with no accompanying timestamp. `_updateRsETHPrice()` (L313–315) sets `rsETHPrice = newRsETHPrice` and emits an event but records no `lastUpdatedAt`. `updateRSETHPrice()` (L87–89) is a public function callable by anyone when not paused, but in practice it is called on a periodic (daily or less frequent) schedule. The ETH/USD Chainlink heartbeat is typically 1 hour or less. During any window where `updateRSETHPrice()` has not been called but the ETH/USD feed has ticked, `latestRoundData()` returns a recent `updatedAt` while serving a stale rsETH/ETH rate. No existing check in `RSETHPriceFeed` guards against this mismatch.

## Impact Explanation
`RSETHPriceFeed` is a Chainlink-compatible price feed whose stated purpose is to provide an accurate rsETH/USD price with correct staleness metadata. It fails to deliver this: `updatedAt` reflects only the ETH/USD component's freshness, not the composite price's true age. Any external protocol (lending market, AMM, structured product) integrating this feed with a standard staleness guard (`require(block.timestamp - updatedAt < heartbeat)`) will consume a stale rsETH/USD price as if it were fresh. This matches the allowed Low impact: **Contract fails to deliver promised returns, but doesn't lose value** for LRT-rsETH depositors directly.

## Likelihood Explanation
No attacker action is required. The condition arises in normal operation whenever the ETH/USD Chainlink feed updates (every few minutes to 1 hour) while `rsETHPrice` has not been refreshed (daily or less). Any caller of `latestRoundData()` during such a window receives misleading metadata. The window is a routine operating state, not an edge case.

## Recommendation
Add a `lastPriceUpdateTime` state variable to `LRTOracle`, set it in `_updateRsETHPrice()` alongside `rsETHPrice = newRsETHPrice`. Expose it via the `IRSETHOracle` interface. In `RSETHPriceFeed.latestRoundData()` and `getRoundData()`, return `min(ethToUsdUpdatedAt, rsETHOracleLastUpdatedAt)` as `updatedAt`, and set `answeredInRound` to reflect the more stale of the two sources. This ensures staleness metadata accurately represents the composite price's true freshness.

## Proof of Concept
```solidity
// Fork test setup:
// 1. Deploy LRTOracle + RSETHPriceFeed pointing to a live ETH/USD Chainlink feed
// 2. Call updateRSETHPrice() to set rsETHPrice; record block.timestamp as T0
// 3. Warp forward 25 hours (simulating no rsETH oracle update)
// 4. The ETH/USD Chainlink feed has updated naturally (heartbeat ~1h)

(, int256 answer,, uint256 updatedAt,) = rsETHPriceFeed.latestRoundData();

// updatedAt == recent ETH/USD feed timestamp (e.g., block.timestamp - 30 seconds)
// answer    == rsETHPrice_from_T0 * current_ETH_USD / 1e18  (25-hour-old rsETH/ETH rate)

// Standard staleness guard used by integrators:
require(block.timestamp - updatedAt < 3600, "stale"); // PASSES — 30s < 1h
// Integrator proceeds with a 25-hour-old rsETH/USD composite price
```
Foundry fork test: deploy against mainnet fork, `vm.warp(block.timestamp + 25 hours)` without calling `updateRSETHPrice()`, then assert `updatedAt > block.timestamp - 3600` (passes) while `rsETHPrice` is 25 hours old.
