The grep results are conclusive:

### Title
Missing `state_diff_commitment` Cross-Check in P2P State Diff Sync Allows Unauthenticated Peer to Store Fabricated `ThinStateDiff` — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary
`StateDiffStreamBuilder::parse_data_for_block` assembles a `ThinStateDiff` from peer-supplied `StateDiffChunk` messages and writes it to storage without ever verifying the result against the `state_diff_commitment` field stored in the `BlockHeader`. Because the header itself is also stored without cryptographic signature verification, an unauthenticated P2P peer controls both the `state_diff_length` sentinel and the chunk content, and can cause the node to persist an arbitrary `ThinStateDiff` for any block number.

### Finding Description

**Step 1 — Fake header accepted without signature verification.**

`HeaderStreamBuilder::parse_data_for_block` in `header.rs` performs only two checks before storing a `SignedBlockHeader`:

- Block number ordering. [1](#0-0) 
- Signature vector length equals 1. [2](#0-1) 

`verify_block_signature` (which performs the actual Poseidon/ECDSA check against the sequencer's public key) is defined in `starknet_api/src/block.rs` but is **never called** in the P2P sync crate. The signed header — including its `state_diff_length` and `state_diff_commitment` fields — is written verbatim to storage. [3](#0-2) 

**Step 2 — `target_state_diff_len` is read from the peer-controlled stored header.**

`StateDiffStreamBuilder::parse_data_for_block` opens a read-only transaction and reads `state_diff_length` from the just-stored header as the authoritative loop bound: [4](#0-3) 

**Step 3 — Chunks are consumed until the peer-controlled length is satisfied.**

The loop accumulates `StateDiffChunk` messages until `current_state_diff_len == target_state_diff_len`. The only structural checks are: no empty chunk, no duplicate keys within a chunk, and no duplicate deprecated class hashes in the final result. [5](#0-4) 

**Step 4 — No commitment cross-check before storage.**

After the loop, `parse_data_for_block` calls `validate_deprecated_declared_classes_non_conflicting` and immediately returns `Ok(Some((result, block_number)))`. The `state_diff_commitment` field that was stored in the header is **never read** in this function, and `calculate_state_diff_hash` is **never called** anywhere in `crates/apollo_p2p_sync/`. [6](#0-5) 

The assembled `ThinStateDiff` is then written directly to storage: [7](#0-6) 

### Impact Explanation
A syncing node stores a `ThinStateDiff` whose `deployed_contracts`, `storage_diffs`, `nonces`, and declared classes are entirely attacker-controlled. Any subsequent RPC call (`starknet_getStorageAt`, `starknet_getClassHashAt`, `starknet_call`, fee estimation, etc.) that reads state derived from this block will return attacker-chosen values. The node's view of the state root diverges silently from the canonical chain.

### Likelihood Explanation
The P2P sync endpoint is unauthenticated by design — any node on the network can act as a peer. No rate-limiting, allowlist, or proof-of-work guards the header or state-diff query responses. The attack requires only two sequential query responses (one `SignedBlockHeader`, then N `StateDiffChunk` messages) and no privileged access.

### Recommendation
1. **Verify the block signature** in `HeaderStreamBuilder::parse_data_for_block` against the known sequencer public key before storing the header. Call `verify_block_signature` from `starknet_api::block`. [8](#0-7) 
2. **Verify the assembled state diff** in `StateDiffStreamBuilder::parse_data_for_block` before returning. After the loop, call `calculate_state_diff_hash(&result)` and compare it against `block_header.state_diff_commitment`. If they differ, return `Err(ParseDataError::BadPeer(...))`. [9](#0-8) 

### Proof of Concept

```
1. Attacker node connects to the victim's P2P sync port.

2. Victim sends a header query for block N.

3. Attacker responds with:
     SignedBlockHeader {
       block_header: BlockHeader {
         block_number: N,
         state_diff_length: Some(2),          // K = 2
         state_diff_commitment: Some(<any>),  // never verified
         block_hash: <any>,
         ...
       },
       signatures: vec![BlockSignature::default()],  // length == 1, passes check
     }
   The header is stored. state_diff_length = 2 is now in storage.

4. Victim sends a state diff query for block N.

5. Attacker responds with exactly 2 length-units of StateDiffChunk:
     StateDiffChunk::ContractDiff(ContractDiff {
       contract_address: VICTIM_CONTRACT,
       class_hash: Some(ATTACKER_CLASS_HASH),  // fabricated deployment
       storage_diffs: {SLOT: ATTACKER_VALUE},
       nonce: None,
     })
   current_state_diff_len (= 2) == target_state_diff_len (= 2) → loop exits.

6. validate_deprecated_declared_classes_non_conflicting passes (no deprecated classes).

7. ThinStateDiff { deployed_contracts: {VICTIM_CONTRACT → ATTACKER_CLASS_HASH},
                   storage_diffs: {VICTIM_CONTRACT → {SLOT → ATTACKER_VALUE}}, ... }
   is written to storage for block N.

8. starknet_getClassHashAt(VICTIM_CONTRACT, block=N) → ATTACKER_CLASS_HASH  ✓ (wrong)
   starknet_getStorageAt(VICTIM_CONTRACT, SLOT, block=N) → ATTACKER_VALUE    ✓ (wrong)
```
