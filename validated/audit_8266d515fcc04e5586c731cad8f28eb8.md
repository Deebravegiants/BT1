Audit Report

## Title
Stale Chainlink Price Accepted Without Staleness Validation Enables Over-Minting of rsETH - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards `updatedAt`, `answeredInRound`, and `roundId`, accepting stale or incomplete round data without any validation. This stale price propagates through `LRTOracle.getAssetPrice()` into `LRTDepositPool.getRsETHAmountToMint()`, allowing any unprivileged depositor to mint excess rsETH when a feed stalls at an above-market price, directly diluting the redemption value of all existing rsETH holders.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` (L52) captures only the `price` field from `latestRoundData()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No check is performed on `updatedAt` (last update timestamp), `answeredInRound` (must be ≥ `roundId` for a complete round), or `roundId`. A stale or incomplete round is silently accepted. [1](#0-0) 

The same codebase's `ChainlinkOracleForRSETHPoolCollateral.getRate()` correctly validates all three conditions:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

The stale price propagates through the following confirmed call chain:

1. `LRTOracle.getAssetPrice(asset)` delegates directly to `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)`, which resolves to `ChainlinkPriceOracle`. [3](#0-2) 

2. `LRTDepositPool.getRsETHAmountToMint()` computes `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`, using the stale price directly in the numerator. [4](#0-3) 

3. `depositAsset()` is fully permissionless (guarded only by `whenNotPaused` and `onlySupportedERC20Token`), so any external caller can trigger this path. [5](#0-4) 

No existing guard in `_beforeDeposit` or `getRsETHAmountToMint` checks oracle freshness. [6](#0-5) 

## Impact Explanation
**High — Theft of unclaimed yield.**

When a Chainlink LST/ETH feed stalls at a price above the current market price (e.g., within its 24-hour heartbeat window during a market stress event), the numerator of the rsETH minting formula is inflated. A depositor receives more rsETH shares than the deposited assets are worth. Because rsETH is a share token backed by a fixed ETH-denominated pool, over-minting directly reduces the redemption value per share for all existing rsETH holders — equivalent to theft of their accrued yield. This matches the allowed impact: **High. Theft of unclaimed yield.**

## Likelihood Explanation
**Medium.** Chainlink LST/ETH feeds on mainnet operate with 24-hour heartbeat intervals. During periods of high network congestion or rapid price movement, feeds can lag significantly within that window without triggering a deviation-based update. The `depositAsset()` entry point requires no privileged role — any external account holding a supported LST can exploit the stale price the moment it diverges from market. The attack is repeatable for the entire duration the feed remains stale.

## Recommendation
Add staleness and round-completeness checks to `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    require(answeredInRound >= roundId, "Stale price");
    require(updatedAt != 0, "Incomplete round");
    require(price > 0, "Invalid price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, enforce a per-feed maximum age threshold (e.g., `block.timestamp - updatedAt <= MAX_STALENESS`) calibrated to each feed's documented heartbeat interval.

## Proof of Concept
1. The stETH/ETH Chainlink feed (24-hour heartbeat) stalls at `1.05e18` while the actual market price falls to `1.00e18`.
2. Attacker calls `LRTDepositPool.depositAsset(stETH, 100e18, 0, "")`.
3. `_beforeDeposit` calls `getRsETHAmountToMint(stETH, 100e18)`.
4. `LRTOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` returns stale `1.05e18`.
5. `rsethAmountToMint = (100e18 * 1.05e18) / rsETHPrice` — 5% more rsETH than the deposited assets are worth.
6. All existing rsETH holders' redemption value is diluted by the 5 ETH discrepancy.

**Foundry fork test plan:**
- Fork mainnet at a block where a stETH/ETH feed update is overdue.
- Deploy or mock `ChainlinkPriceOracle` pointing to the live feed.
- Call `depositAsset` and assert `rsethAmountToMint > (depositAmount * trueMarketPrice) / rsETHPrice`.
- Alternatively, mock `latestRoundData` to return `answeredInRound < roundId` and `updatedAt = block.timestamp - 25 hours`, confirm no revert occurs and excess rsETH is minted.
