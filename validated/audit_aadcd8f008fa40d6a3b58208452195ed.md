### Title
Unauthenticated `AddGasPrice`/`Initialize` on `RemoteL1GasPriceServer` Allows Arbitrary L1 Gas Price Injection into Block Proposals — (`crates/apollo_l1_gas_price/src/communication.rs`)

---

### Summary

The `L1GasPriceProvider` component exposes a `RemoteComponentServer` over HTTP/2 that handles `L1GasPriceRequest::AddGasPrice` and `L1GasPriceRequest::Initialize` with no authentication, no caller identity check, and no access control of any kind. Any network-reachable process can reset the price buffer and inject arbitrary `base_fee_per_gas` / `blob_fee` values. Those values flow directly into the `l1_gas_price_fri`, `l1_data_gas_price_fri`, `l1_gas_price_wei`, and `l1_data_gas_price_wei` fields of every subsequent `ProposalInit`, corrupting the gas prices embedded in every block header and every fee calculation performed by the blockifier.

---

### Finding Description

**Root cause — no access control on the write path of `L1GasPriceProvider`**

`L1GasPriceProvider::add_price_info` is the sole write entry-point for L1 gas price data. Its only guard is a sequential block-number check: [1](#0-0) 

There is no check on who is calling it, no signature, no token, no IP allowlist. The `price_info` fields (`base_fee_per_gas`, `blob_fee`) are accepted verbatim.

`initialize()` resets the ring buffer entirely and also has no access control: [2](#0-1) 

Both operations are exposed as `L1GasPriceRequest` variants handled unconditionally: [3](#0-2) 

**The `RemoteComponentServer` adds no authentication layer**

When the node runs in distributed mode (`LocalExecutionWithRemoteEnabled`), a `RemoteL1GasPriceServer` is created and bound to `0.0.0.0` (all interfaces) by default: [4](#0-3) [5](#0-4) 

The `remote_component_server_handler` deserializes the incoming binary body and forwards it to the local client with no credential check: [6](#0-5) 

`RemoteServerConfig` has no `allowed_ips`, no token field, no TLS client-certificate field: [7](#0-6) 

**Attack path**

1. Attacker reaches the `RemoteL1GasPriceServer` TCP port (default: `0.0.0.0:<configured port>`).
2. Sends a serialized `L1GasPriceRequest::Initialize` — resets the ring buffer, bypassing the sequential block-number guard.
3. Sends `L1GasPriceRequest::AddGasPrice(GasPriceData { block_number: N, timestamp: T, price_info: PriceInfo { base_fee_per_gas: X, blob_fee: Y } })` with attacker-chosen `X` and `Y`. The current L1 block number `N` is publicly observable on Ethereum.
4. Repeats step 3 for subsequent block numbers to fill the ring buffer with manipulated samples.
5. The provider's `get_price_info` now returns the attacker-controlled mean.

**How the corrupted price reaches block proposals**

The consensus orchestrator calls `get_l1_prices_in_fri_and_wei_and_conversion_rate`, which queries the provider: [8](#0-7) 

The result is placed directly into `ProposalInit.l1_gas_price_fri`, `l1_data_gas_price_fri`, `l1_gas_price_wei`, `l1_data_gas_price_wei`: [9](#0-8) 

These fields are then used by the blockifier as the authoritative gas prices for every transaction in the block.

**Why the validator's margin check does not fully mitigate this**

The validator re-queries its own provider and rejects proposals outside `l1_gas_price_margin_percent`: [10](#0-9) 

However:
- If the attacker injects into the shared provider (distributed deployment where provider is a separate microservice), both proposer and validator read the same corrupted data, so the margin check passes.
- Even if each node has its own provider, the attacker can inject prices that are within the margin (e.g., 5–9% above true prices if the margin is 10%), making the manipulation undetectable by the validator while still causing measurable economic harm to users.

---

### Impact Explanation

**Critical — Incorrect fee, gas, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

Injected `base_fee_per_gas` and `blob_fee` values flow into every transaction's fee calculation for the duration the corrupted samples remain in the ring buffer (up to `storage_limit` blocks, default 3000). Users are overcharged or undercharged for L1 data availability costs. The L1 gas price fields in the block header are permanently wrong for those blocks, affecting any downstream system (provers, bridges, fee refund logic) that reads them.

---

### Likelihood Explanation

**Medium.** The `RemoteL1GasPriceServer` is an internal microservice port, but it binds to `0.0.0.0` by default with no authentication. In a Kubernetes deployment without strict NetworkPolicy rules, any pod in the cluster can reach it. An external attacker who achieves any foothold inside the cluster (or who can reach the port via a misconfigured ingress or NodePort) can exploit this immediately. The L1 block number needed to pass the sequential check is publicly observable on Ethereum.

---

### Recommendation

1. **Authenticate the write path.** Add a shared secret (HMAC token in a request header, or mutual TLS) to the `RemoteComponentServer` / `RemoteComponentClient` pair, checked before dispatching any `L1GasPriceRequest`.
2. **Restrict `AddGasPrice` and `Initialize` to the scraper identity.** At minimum, add an IP allowlist for the `RemoteL1GasPriceServer` so only the `L1GasPriceScraper`'s address is accepted.
3. **Bind to localhost by default.** Change `DEFAULT_BIND_IP` from `Ipv4Addr::UNSPECIFIED` to `Ipv4Addr::LOCALHOST` for all internal component servers, requiring explicit opt-in to wider exposure.
4. **Validate price bounds inside `add_price_info`.** Reject samples whose `base_fee_per_gas` or `blob_fee` exceed a configurable sanity ceiling, providing defense-in-depth even if authentication is bypassed.

---

### Proof of Concept

```python
import httpx, struct, cbor2  # or any HTTP/2 client + the project's serde format

TARGET = "http://<sequencer-node>:<l1_gas_price_provider_port>/"

# Step 1: Reset the ring buffer (no auth required)
initialize_request = serialize_l1_gas_price_request("Initialize")
httpx.post(TARGET, content=initialize_request, headers={"content-type": "application/octet-stream"})

# Step 2: Inject 300 blocks of inflated gas prices
# Current L1 block number is publicly known (e.g., from etherscan)
current_l1_block = 21_000_000
for i in range(300):
    add_request = serialize_l1_gas_price_request("AddGasPrice", {
        "block_number": current_l1_block + i,
        "timestamp":    <current_unix_ts + i * 12>,
        "price_info": {
            "base_fee_per_gas": 999_999_999_999,   # ~1000 gwei, far above true value
            "blob_fee":         999_999_999_999,
        }
    })
    httpx.post(TARGET, content=add_request, headers={"content-type": "application/octet-stream"})

# Result: the next block proposal will embed l1_gas_price ~1000 gwei,
# causing every transaction in that block to be charged ~1000x the true L1 cost.
```

The `serialize_l1_gas_price_request` function must produce the same binary encoding used by `SerdeWrapper` in `RemoteComponentClient`. No credentials are required at any step.

        Ok(())

    #[instrument(skip_all, fields(request_id = %request_id, remote_addr = %client_peer))]
    async fn remote_component_server_handler(
        http_request: HyperRequest<Incoming>,
        request_id: RequestId,
        client_peer: SocketAddr,
        local_client: LocalComponentClient<Request, Response>,
        metrics: &'static RemoteServerMetrics,
        max_request_body_bytes: usize,
    ) -> Result<HyperResponse<Full<Bytes>>, hyper::Error> {
        trace!("Received HTTP request: {http_request:?}");
        let body_bytes =
            match Limited::new(http_request.into_body(), max_request_body_bytes).collect().await {
                Ok(collected) => collected.to_bytes(),
                Err(err) => {
                    warn!("Request body too large: {err}");
                    let server_error = ServerError::RequestBodyTooLarge(err.to_string());
                    return Ok(HyperResponse::builder()
                        .status(StatusCode::PAYLOAD_TOO_LARGE)
                        .header(CONTENT_TYPE, APPLICATION_OCTET_STREAM)
                        .body(Full::new(Bytes::from(
                            SerdeWrapper::new(server_error)
                                .wrapper_serialize()
                                .expect("Server error serialization should succeed"),
                        )))
                        .expect("Response building should succeed"));
                }
            };
        trace!("Extracted {} bytes from HTTP request body", body_bytes.len());

        metrics.increment_total_received();

        let http_response = match SerdeWrapper::<Request>::wrapper_deserialize(&body_bytes)
            .map_err(|err| ClientError::ResponseDeserializationFailure(err.to_string()))
        {
            Ok(request) => {
                trace!(
                    remote_addr = %client_peer,
                    request_id = %request_id,
                    request_type = request.request_label(),
                    "remote component request",
                );
                trace!("Successfully deserialized request: {request:?}");
                metrics.increment_valid_received();

                // Wrap the send operation in a tokio::spawn as it is NOT a cancel-safe operation.
                // Even if the current task is cancelled, the inner task will continue to run.
                // Note: this creates a new request ID for the local client.
                let response = tokio::spawn(async move { local_client.send(request).await })
                    .await
                    .expect("Should be able to extract value from the task");

                metrics.increment_processed();

                match response {
                    Ok(response) => {
                        trace!("Local client processed request successfully: {response:?}");
                        HyperResponse::builder()
                            .status(StatusCode::OK)
                            .header(CONTENT_TYPE, APPLICATION_OCTET_STREAM)
                            .body(Full::new(Bytes::from(
                                SerdeWrapper::new(response)
                                    .wrapper_serialize()
                                    .expect("Response serialization should succeed"),
                            )))
                    }
                    Err(error) => {
                        panic!(
                            "Remote server failed sending with its local client. Error: {error:?}"
                        );
                    }
                }
            }
            Err(error) => {
                error!("Failed to deserialize request: {error:?}");
                let server_error = ServerError::RequestDeserializationFailure(error.to_string());
                HyperResponse::builder().status(StatusCode::BAD_REQUEST).body(Full::new(
                    Bytes::from(
                        SerdeWrapper::new(server_error)
                            .wrapper_serialize()
                            .expect("Server error serialization should succeed"),
                    ),
                ))
            }
        }
        .expect("Response building should succeed");
        trace!("Built HTTP response: {http_response:?}");

        Ok(http_response)
    }
