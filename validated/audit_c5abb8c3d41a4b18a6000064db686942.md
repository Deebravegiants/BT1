### Title
Sub-Transaction Reference Script Fee Not Included in Minimum Fee Calculation - (`File: eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Utxo.hs`)

### Summary

In the Dijkstra era, the minimum fee validation for a top-level transaction uses `getConwayMinFeeTxUtxo`, which only accounts for reference scripts in the top-level transaction's own inputs. Reference scripts used by embedded sub-transactions are excluded from the fee calculation, allowing an attacker to use arbitrarily large reference scripts in sub-transactions while paying only the base size-proportional fee — bypassing the exponential reference-script fee protection introduced in Conway.

### Finding Description

The Conway era introduced a tiered, exponentially-growing reference-script fee (`tierRefScriptFee`) to deter DDoS attacks via large Plutus scripts used as reference scripts (see ADR 009). This fee is computed by `getConwayMinFeeTxUtxo`, which calls `txNonDistinctRefScriptsSize` to measure the total reference-script bytes touched by a transaction.

The Dijkstra era introduces nested ("sub") transactions. A top-level `DijkstraTxBodyRaw` embeds sub-transactions in `dtbrSubTransactions`. Each sub-transaction can itself reference UTxO entries that carry reference scripts.

The Dijkstra `EraUTxO` instance sets:

```haskell
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
``` [1](#0-0) 

`getConwayMinFeeTxUtxo` is:

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
``` [2](#0-1) 

And `txNonDistinctRefScriptsSize` only inspects the top-level transaction's own inputs and reference inputs:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [3](#0-2) 

The codebase already contains `batchNonDistinctRefScriptsSize`, which correctly aggregates reference-script sizes across the top-level transaction **and all sub-transactions**:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
``` [4](#0-3) 

However, `batchNonDistinctRefScriptsSize` is **never called** during fee validation. The Dijkstra UTXO rule validates the fee as:

```haskell
{- minfee pp txTop utxo₀ ≤ txfee txb -}
runTest $ Shelley.validateFeeTooSmallUTxO pp tx originalUtxo
``` [5](#0-4) 

`validateFeeTooSmallUTxO` calls `getMinFeeTxUtxo`, which resolves to `getConwayMinFeeTxUtxo` → `txNonDistinctRefScriptsSize` (top-level only). Sub-transactions are processed by `SUBUTXO`, which explicitly uses `updateUTxOStateNoFees` and performs no fee check at all:

```haskell
  if isValid
    then do
      newState <-
        Shelley.updateUTxOStateNoFees
          pp utxoState txBody certState ...
``` [6](#0-5) 

The comment on `updateUTxOStateNoFees` confirms this is intentional for sub-transactions:

```haskell
-- | Like 'updateUTxOState', but does not collect fees. This is used for sub-transactions
-- where fees are not applicable.
``` [7](#0-6) 

The design intent is that the top-level transaction's fee covers the entire batch — but the fee calculation omits the sub-transactions' reference-script component.

### Impact Explanation

**Medium. Attacker-controlled transactions exceed intended validation limits or modify fees outside design parameters.**

An attacker constructs a top-level Dijkstra transaction whose sub-transactions reference UTxO entries carrying large Plutus scripts. The top-level fee only pays the tiered reference-script fee for scripts touched by the top-level transaction's own inputs. The sub-transactions' reference scripts — potentially hundreds of kilobytes — are deserialized and validated by every node at zero marginal reference-script fee cost. This directly undermines the DDoS deterrent introduced by ADR 009 and the Conway hard fork, allowing the same class of resource-exhaustion attack that occurred on June 25, 2024 to be replicated in the Dijkstra era at a fraction of the intended cost.

### Likelihood Explanation

The Dijkstra era is protocol version 12 and is not yet deployed on mainnet. However, the vulnerability is structurally present in the production code: `batchNonDistinctRefScriptsSize` exists but is not wired into `getMinFeeTxUtxo`. Any transaction author can exploit this once the era activates, with no special privileges required — only the ability to submit a valid top-level transaction containing sub-transactions with reference inputs pointing to large-script UTxOs.

### Recommendation

Replace the `getMinFeeTxUtxo` implementation for `DijkstraEra` with one that uses `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
getDijkstraMinFeeTxUtxo :: (EraTx era, BabbageEraTxBody era, DijkstraEraTxBody era)
  => PParams era -> Tx l era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx
```

and set `getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo` in the `EraUTxO DijkstraEra` instance.

### Proof of Concept

1. Create a UTxO entry `u` containing a large Plutus reference script (e.g., 100 KiB).
2. Construct a sub-transaction `subTx` with `referenceInputsTxBodyL = {u}` and any valid spend input.
3. Construct a top-level Dijkstra transaction `topTx` with `subTransactionsTxBodyL = {subTx}` and set `feeTxBodyL` to the minimum fee computed by `getConwayMinFeeTxUtxo` (which ignores `u`'s script size).
4. Submit `topTx`. The ledger accepts it: `validateFeeTooSmallUTxO` passes because `getMinFeeTxUtxo` only measures the top-level transaction's reference scripts. The sub-transaction's 100 KiB script is deserialized by every node at no additional reference-script fee cost.

The discrepancy is directly observable by comparing:
- `getConwayMinFeeTxUtxo pp topTx utxo` (used in validation, ignores sub-tx scripts)
- `getMinFeeTx pp topTx (batchNonDistinctRefScriptsSize utxo topTx)` (correct batch fee, includes sub-tx scripts) [1](#0-0) [4](#0-3) [5](#0-4) [8](#0-7)
