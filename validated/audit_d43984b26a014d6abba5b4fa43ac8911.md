### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Inject Arbitrary Sequencer Address, Corrupting Block Hash and Misdirecting Fees - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates every security-critical field of `ProposalInit` (timestamp, starknet version, L1/L2 gas prices, `fee_proposal_fri`, `l1_da_mode`, `height`) but never checks `init.builder`. That field is then passed verbatim as `sequencer_address` into the `BlockInfo` used for both block execution and block hash computation. A malicious proposer can set `builder` to any arbitrary address, causing all sequencer fees for the block to be redirected to an attacker-controlled address and the committed block hash to encode the wrong sequencer.

---

### Finding Description

`ProposalInit` carries a `builder` field — the address of the entity that builds/sequences the block:

```rust
/// Address of the one who builds/sequences the block.
pub builder: ContractAddress,
``` [1](#0-0) 

`convert_to_sn_api_block_info` maps this field directly to `sequencer_address` in the `BlockInfo` that is handed to the batcher:

```rust
Ok(starknet_api::block::BlockInfo {
    ...
    sequencer_address: init.builder,
    ...
})
``` [2](#0-1) 

This `BlockInfo` is used in two places:

1. **Block execution** — passed to `batcher.validate_block(ValidateBlockInput { block_info, ... })`, which drives the blockifier. The blockifier uses `block_context.block_info.sequencer_address` to determine who receives transaction fees. [3](#0-2) 

2. **Block hash computation** — `PartialBlockHashComponents::new` reads `block_info.sequencer_address` and includes it in the Poseidon hash that becomes the canonical block hash: [4](#0-3) [5](#0-4) 

3. **State sync storage** — `update_state_sync_with_new_block` stores `SequencerContractAddress(init.builder)` in `BlockHeaderWithoutHash.sequencer`: [6](#0-5) 

`is_proposal_init_valid` validates every other security-critical field but has no check for `builder`: [7](#0-6) 

The proposer sets `builder` from its own static config with no committee-derived constraint:

```rust
// TODO(Asmaa): Get it from committee once we have it.
builder_address: self.config.static_config.builder_address,
``` [8](#0-7) 

---

### Impact Explanation

**Fee misdirection (Critical — economic impact):** Every transaction fee charged in the block is credited to `sequencer_address`. A malicious proposer sets `builder` to an attacker-controlled address; all sequencer revenue for that block is permanently redirected there.

**Wrong block hash committed (Critical — wrong state commitment):** The sequencer address is a direct input to `calculate_block_hash`. Because all validators receive the same `ProposalInit` and all call `convert_to_sn_api_block_info` with the same `init.builder`, they all compute the same (wrong) commitment and reach consensus on it. The block hash stored in state and eventually anchored to L1 encodes the attacker's address as the sequencer, not the legitimate one.

**Wrong state sync record:** `BlockHeaderWithoutHash.sequencer` is persisted with the attacker-supplied address, corrupting the authoritative view returned by the RPC layer for that block.

---

### Likelihood Explanation

Exploiting this requires the attacker to be a scheduled proposer in the validator committee for at least one round — a privileged but reachable position in a permissioned or semi-permissioned validator set. No additional capability is needed: the attacker simply sets `builder` to an arbitrary address in their node config before their proposer turn. The consensus layer checks `init.proposer` against the committee but never checks `init.builder`. [9](#0-8) 

---

### Recommendation

Add a check for `init.builder` inside `is_proposal_init_valid` in `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`. The expected value should be derived from the committee (the TODO at line 806 of `sequencer_consensus_context.rs` already acknowledges this). Until committee-derived builder addresses are available, the validator should at minimum reject any `init.builder` that differs from the locally configured `builder_address`, mirroring how `init.proposer` is checked against the committee-derived proposer identity.

---

### Proof of Concept

1. A validator node is scheduled as proposer for block N.
2. Before its turn, the attacker sets `builder_address` in `ContextStaticConfig` to an attacker-controlled contract address `0xDEAD`.
3. The node calls `build_proposal`, which sets `init.builder = 0xDEAD` and broadcasts `ProposalInit` to all peers.
4. Each peer calls `validate_proposal` → `is_proposal_init_valid`. No check on `builder` exists; validation passes.
5. Each peer calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: 0xDEAD, ... }`.
6. The batcher executes the block with `sequencer_address = 0xDEAD`; all transaction fees are credited to `0xDEAD`.
7. `PartialBlockHashComponents::new` uses `sequencer_address = 0xDEAD`; the resulting block hash encodes the wrong sequencer.
8. Consensus reaches agreement on this commitment; the block is finalized with the corrupted hash and misdirected fees.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
