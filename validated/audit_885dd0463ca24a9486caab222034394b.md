Audit Report

## Title
Stale Chainlink Price Accepted Without Staleness Validation in `ChainlinkPriceOracle.getAssetPrice()` - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all return values except `answer`, performing no check on `answeredInRound`, `updatedAt`, or price sign. A stale or inflated Chainlink price propagates directly into rsETH minting via `LRTDepositPool.depositAsset()` and into the rsETH/ETH rate via `LRTOracle.updateRSETHPrice()`, allowing a depositor to receive rsETH at a stale inflated price and dilute the accrued yield of existing rsETH holders.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` reads from Chainlink as:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values of `latestRoundData()` are available but only `answer` is used. No check is made on:
- `answeredInRound >= roundId` (stale round detection)
- `updatedAt != 0` (incomplete round detection)
- `price > 0` (zero/negative answer detection)
- `block.timestamp - updatedAt` against a heartbeat threshold

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` — used in the L2 pool oracle path — performs all three guards and reverts with `StalePrice()`, `IncompleteRound()`, or `InvalidPrice()`:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L27-32
(uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
    AggregatorV3Interface(oracle).latestRoundData();
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The unprotected `ChainlinkPriceOracle` is the oracle registered for supported LST assets on L1. Its output is consumed in two critical paths:

**Path 1 — rsETH minting:** `LRTDepositPool.depositAsset()` → `_beforeDeposit()` → `getRsETHAmountToMint()`:
```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

**Path 2 — rsETH/ETH price update:** `LRTOracle.updateRSETHPrice()` → `_updateRsETHPrice()` → `_getTotalEthInProtocol()`:
```solidity
// contracts/LRTOracle.sol L339
uint256 assetER = getAssetPrice(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
```

Existing guards do not mitigate this: the `pricePercentageLimit` check in `_updateRsETHPrice()` only triggers on `updateRSETHPrice()` calls and compares against `highestRsethPrice`, not the actual market price. The `minRSETHAmountExpected` parameter in `depositAsset()` is attacker-controlled and can be set to zero.

## Impact Explanation
When a Chainlink feed for a supported LST (e.g., stETH/ETH) is stale at a price higher than the current market price, a depositor calling `depositAsset()` receives `rsethAmountToMint = amount * stalePriceHigh / rsETHPrice_stale`. Because `rsETHPrice` was computed from the same stale feed at a prior update, the depositor receives rsETH representing a claim on more ETH than their deposit is actually worth at the current market price. When the price corrects, the total ETH backing is lower than the rsETH supply implies, and existing holders' accrued yield (the ETH appreciation they had earned) is diluted. This is a concrete **High — Theft of unclaimed yield**: existing rsETH holders lose the yield they had accrued from LST appreciation to the depositor who entered at the stale inflated rate.

## Likelihood Explanation
Chainlink feeds have documented heartbeat intervals (e.g., 24 hours for stETH/ETH on mainnet) and deviation thresholds. During network congestion, oracle node downtime, or rapid price movement that has not yet triggered a deviation update, the feed can remain stale for minutes to hours. No privileged access or governance action is required — any unprivileged depositor can call `depositAsset()` at any time, including during a period of feed staleness. The condition is historically observed and requires no victim mistake.

## Recommendation
Apply the same staleness guards already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optionally: if (block.timestamp - updatedAt > HEARTBEAT) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept
1. At T=0: stETH/ETH = 1.05, `rsETHPrice` is updated to 1.05 (100 stETH deposited, 100 rsETH outstanding).
2. At T=1: stETH actual price drops to 0.98, but the Chainlink feed has not triggered a deviation update and remains stale at 1.05.
3. Attacker calls `LRTDepositPool.depositAsset(stETH, 100e18, 0, "")`.
4. `getRsETHAmountToMint(stETH, 100e18)` → `lrtOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → `latestRoundData()` returns stale `price = 1.05e18`. No revert.
5. `rsethAmountToMint = (100e18 * 1.05e18) / 1.05e18 = 100e18` rsETH minted to attacker.
6. Protocol now holds 200 stETH and has 200 rsETH outstanding.
7. When price corrects to 0.98: `totalETH = 200 * 0.98 = 196 ETH`, `rsETHPrice = 196/200 = 0.98`.
8. Original holders: 100 rsETH × 0.98 = 98 ETH — they lost the 5 ETH of yield they had accrued from stETH appreciating from 1.0 to 1.05.
9. Attacker: 100 rsETH × 0.98 = 98 ETH — received rsETH worth 98 ETH for 100 stETH worth 98 ETH, having entered at the stale inflated rate and captured the yield dilution.

**Foundry fork test plan:** Fork mainnet, mock the stETH/ETH Chainlink aggregator to return a stale `updatedAt` and inflated `answer`, call `depositAsset` as an unprivileged address, then call `updateRSETHPrice` and assert that original holder rsETH value in ETH has decreased relative to their pre-deposit entitlement.
