Audit Report

## Title
`emergencyWithdrawFromAave` Leaves `isAaveIntegrationEnabled = true` After Draining Aave, Causing `completeWithdrawal` to Permanently Revert - (File: contracts/LRTWithdrawalManager.sol)

## Summary

`emergencyWithdrawFromAave` withdraws ETH from Aave but never sets `isAaveIntegrationEnabled = false`. After the emergency drains Aave's balance to zero, every subsequent `completeWithdrawal` call for ETH that cannot be fully covered by the contract's idle balance enters the Aave withdrawal branch and hard-reverts with `InsufficientAaveBalance`. Affected users' funds are frozen until a manager separately calls `setAaveIntegrationEnabled(false)`.

## Finding Description

`emergencyWithdrawFromAave` is callable by `PAUSER_ROLE` and is the intended response to an Aave emergency: [1](#0-0) 

After this call completes, `isAaveIntegrationEnabled` remains `true`. `_processWithdrawalCompletion` (invoked by every `completeWithdrawal`) unconditionally enters the Aave branch when the flag is set and the contract's idle ETH is insufficient: [2](#0-1) 

`_withdrawFromAave` hard-reverts when `aaveBalance == 0`: [3](#0-2) 

There is no `try/catch` around this call in `_processWithdrawalCompletion`, unlike the deposit path in `unlockQueue`: [4](#0-3) 

`_checkAaveHealth` does not detect the broken state: after a full emergency withdrawal both `totalETHDepositedToAave` and `aaveBalance` are `0`, so `principal > aaveBalance` evaluates to `0 > 0 = false`, and the function incorrectly returns `true`: [5](#0-4) 

## Impact Explanation

**Medium — Temporary freezing of user ETH withdrawal funds.**

Any unlocked ETH withdrawal request whose `expectedAssetAmount` exceeds the contract's idle ETH balance after the emergency withdrawal will revert on every `completeWithdrawal` call. The funds are not lost but are inaccessible to the user until a manager calls `setAaveIntegrationEnabled(false)`. This matches the allowed impact class "Medium. Temporary freezing of funds."

## Likelihood Explanation

**Low.** Requires: (a) a genuine emergency prompting a `PAUSER_ROLE` holder to call `emergencyWithdrawFromAave`, and (b) at least one pending unlocked withdrawal whose amount exceeds the contract's remaining idle ETH after the emergency. Both conditions are realistic in an Aave exploit scenario but are not the normal operating path. The `PAUSER_ROLE` is acting in good faith — this is not collusion.

## Recommendation

`emergencyWithdrawFromAave` should disable the integration immediately after withdrawing, mirroring the cleanup already present in `setAaveIntegrationEnabled(false)`:

```solidity
function emergencyWithdrawFromAave(uint256 amount) external nonReentrant onlyRole(LRTConstants.PAUSER_ROLE) {
    if (!isAaveIntegrationEnabled) revert AaveIntegrationNotEnabled();
    uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
    if (aaveBalance == 0) revert InsufficientAaveBalance();
    _collectInterestToTreasury();
    uint256 withdrawnAmount = _withdrawFromAave(amount);
+   isAaveIntegrationEnabled = false;
+   _revokeApprovalToAaveWETHGateway();
    emit EmergencyWithdrawFromAave(withdrawnAmount, address(this));
+   emit AaveIntegrationEnabled(false);
}
```

Alternatively, `_processWithdrawalCompletion` should guard the `_withdrawFromAave` call with an `aaveBalance` pre-check and skip (or revert with a clearer error) when Aave is empty.

## Proof of Concept

1. Aave integration active; 100 ETH deposited (`totalETHDepositedToAave = 100`).
2. `unlockQueue` unlocks two requests: Alice 60 ETH, Bob 60 ETH — both claimable.
3. Pauser calls `emergencyWithdrawFromAave(type(uint256).max)`:
   - `_withdrawFromAave` withdraws 100 ETH → `totalETHDepositedToAave = 0`, `aaveBalance = 0`.
   - `isAaveIntegrationEnabled` **remains `true`**.
   - Contract ETH balance = 100 ETH.
4. Alice calls `completeWithdrawal(ETH)` → `100 >= 60` → succeeds. Contract balance = 40 ETH.
5. Bob calls `completeWithdrawal(ETH)`:
   - `isAaveIntegrationEnabled == true` → enters Aave branch.
   - `40 < 60` → calls `_withdrawFromAave(20)`.
   - `aaveAWETH.balanceOf(address(this)) == 0` → **`revert InsufficientAaveBalance()`**.
6. Bob's 60 ETH is frozen. No user-callable path unblocks it; only a manager calling `setAaveIntegrationEnabled(false)` restores normal operation. [1](#0-0) [2](#0-1) [3](#0-2)
