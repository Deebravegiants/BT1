### Title
Any Unprivileged Caller Can Trigger Protocol-Wide Pause via `updateRSETHPrice()` During Price Drops, Temporarily Freezing User Withdrawals — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a public, permissionless function. When the computed rsETH price has dropped more than `pricePercentageLimit` below `highestRsethPrice`, calling it unconditionally pauses both `LRTDepositPool` and `LRTWithdrawalManager`. Any unprivileged user can trigger this during periods of market stress — exactly when users most need to withdraw — temporarily freezing all deposits and withdrawals until an admin manually unpauses.

---

### Finding Description

`LRTOracle.updateRSETHPrice()` is callable by any address when the oracle is not paused:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, a downside-protection branch unconditionally pauses the entire protocol when the new price falls beyond the configured threshold:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

The condition `isPriceDecreaseOffLimit` is:

```solidity
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
```

There is **no access control** on `updateRSETHPrice()`. Any externally owned account can call it at any time. If the oracle-reported rsETH price (derived from live Chainlink LST prices and EigenLayer TVL) has dropped beyond `pricePercentageLimit` relative to `highestRsethPrice`, the call will:

1. Pause `LRTDepositPool` — blocking `depositETH()` and `depositAsset()`.
2. Pause `LRTWithdrawalManager` — blocking `initiateWithdrawal()`, `completeWithdrawal()`, and `instantWithdrawal()`.
3. Pause `LRTOracle` itself — preventing any further price updates until admin unpauses.

The `unpause()` functions on both `LRTDepositPool` and `LRTWithdrawalManager` require `onlyLRTAdmin`, meaning recovery depends entirely on a privileged actor responding in time.

---

### Impact Explanation

**Temporary freezing of funds (Medium).** Once the pause is triggered:

- Users with pending withdrawal requests cannot call `completeWithdrawal()` to retrieve their assets.
- Users cannot initiate new withdrawals via `initiateWithdrawal()` or `instantWithdrawal()`.
- The `LRTWithdrawalManager.whenNotPaused` modifier gates all three user-facing withdrawal paths.

The freeze persists until the admin calls `unpause()` on each contract. During volatile market conditions — the exact scenario where users most urgently need to exit — the protocol becomes inaccessible to all users.

---

### Likelihood Explanation

**Medium.** The trigger condition (rsETH price dropping beyond `pricePercentageLimit`) is realistic during:

- EigenLayer slashing events that reduce the ETH value backing rsETH.
- Temporary LST depeg events that lower `_getTotalEthInProtocol()` via Chainlink price feeds.
- Large coordinated withdrawals that reduce TVL faster than the stored `highestRsethPrice` reflects.

Once the price condition is met, **any user** — including a griefing actor who monitors oracle prices off-chain — can call `updateRSETHPrice()` to atomically pause the protocol. No capital, no special role, and no front-running is required beyond a single public function call.

---

### Recommendation

Restrict `updateRSETHPrice()` to authorized callers (e.g., `onlyLRTOperator` or `onlyLRTManager`) so that the auto-pause mechanism cannot be triggered by arbitrary external accounts. Alternatively, separate the price-update logic from the auto-pause logic: emit an event or set a flag when the threshold is breached, and require a privileged actor to explicitly trigger the pause in a separate transaction.

---

### Proof of Concept

1. Admin sets `pricePercentageLimit = 1e16` (1%) via `setPricePercentageLimit`.
2. `highestRsethPrice` is recorded at `1.05 ETH` after a period of rewards.
3. An EigenLayer slashing event reduces total ETH in protocol; the new computed price is `1.034 ETH` — a drop of ~1.5%, exceeding the 1% limit.
4. Attacker (any EOA) calls `LRTOracle.updateRSETHPrice()`.
5. Inside `_updateRsETHPrice()`: `diff = 1.05e18 - 1.034e18 = 0.016e18`; `pricePercentageLimit.mulWad(highestRsethPrice) = 0.01 * 1.05e18 = 0.0105e18`; `isPriceDecreaseOffLimit = true`.
6. `lrtDepositPool.pause()` is called → `LRTDepositPool` paused.
7. `withdrawalManager.pause()` is called → `LRTWithdrawalManager` paused.
8. `_pause()` is called → `LRTOracle` paused.
9. All users attempting `completeWithdrawal()`, `initiateWithdrawal()`, or `instantWithdrawal()` receive `Pausable: paused` reverts.
10. Protocol remains frozen until admin calls `unpause()` on each contract.

    function instantWithdrawal(
        address asset,
        uint256 rsETHUnstaked,
        string calldata referralId
    )
        external
        nonReentrant
        whenNotPaused
        onlySupportedAsset(asset)
        onlySupportedStrategy(asset)
        onlyInstantWithdrawalAllowed(asset)
    {
        if (rsETHUnstaked == 0 || rsETHUnstaked < minRsEthAmountToWithdraw[asset]) {
            revert InvalidAmountToWithdraw();
        }
        if (IERC20(lrtConfig.rsETH()).balanceOf(msg.sender) < rsETHUnstaked) revert NotEnoughRsETH();
        uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
        IRSETH(lrtConfig.rsETH()).burnFrom(address(msg.sender), rsETHUnstaked);
        ILRTUnstakingVault unstakingVault = ILRTUnstakingVault(lrtConfig.getContract(LRTConstants.LRT_UNSTAKING_VAULT));
        if (assetAmountUnlocked > unstakingVault.getAssetsAvailableForInstantWithdrawal(asset)) {
            revert CantInstantWithdrawMoreThanAvailable();
        }

        unstakingVault.redeem(asset, assetAmountUnlocked);

        uint256 fee = (assetAmountUnlocked * instantWithdrawalFee) / 10_000;
        uint256 userAmount = assetAmountUnlocked - fee;

        address feeRecipient = instantWithdrawalFeeRecipient;
        if (feeRecipient == address(0)) {
            // Backwards-compatible default: send fees to the protocol treasury
            feeRecipient = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        }
        if (fee > 0) {
            _transferAsset(asset, feeRecipient, fee);
            emit InstantWithdrawalFeeCollected(msg.sender, asset, fee);
        }

        _transferAsset(asset, msg.sender, userAmount);
        emit ReferralIdEmitted(referralId);
        emit AssetWithdrawalFinalized(msg.sender, asset, rsETHUnstaked, userAmount);
    }
