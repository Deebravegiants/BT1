### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Redirect All Block Fees and Corrupt the Committed Block Hash — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates every security-sensitive field of `ProposalInit` received from the network **except** `builder`. That field is passed verbatim as `sequencer_address` into `BlockInfo`, which drives both fee collection during execution and the `sequencer` slot of the Poseidon block-hash preimage. A validator selected as proposer can set `builder` to any address, causing every validator to execute the block with an attacker-controlled sequencer address, redirecting all transaction fees and producing a committed block hash that encodes the wrong sequencer.

---

### Finding Description

`ProposalInit` carries a `builder: ContractAddress` field described as "Address of the one who builds/sequences the block." [1](#0-0) 

`convert_to_sn_api_block_info` maps this field directly to `sequencer_address` in `BlockInfo` without any guard: [2](#0-1) 

That `BlockInfo` is forwarded to the batcher via `ValidateBlockInput` in `initiate_validation`: [3](#0-2) 

`is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices (within a margin), `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri`. It never checks `builder`: [4](#0-3) 

`PartialBlockHashComponents::new` then captures `block_info.sequencer_address` (i.e. the attacker-supplied `builder`) into the `sequencer` slot: [5](#0-4) 

`calculate_block_hash` chains `sequencer` into the Poseidon preimage at position 4: [6](#0-5) 

Because every validator derives `BlockInfo` from the same `ProposalInit.builder`, they all compute the same `partial_block_hash` and the same `batcher_block_commitment`. The final check: [7](#0-6) 

passes, so the proposal is accepted and the block is committed with the attacker-chosen sequencer address.

---

### Impact Explanation

**Fee theft (Critical economic impact).** During blockifier execution, transaction fees are transferred to `block_context.block_info.sequencer_address`. With `builder` set to an attacker-controlled address, every fee payment in the block is credited to that address instead of the legitimate sequencer. The state diff written to storage reflects these wrong balance changes.

**Wrong committed block hash (Critical state commitment).** The `sequencer_address` slot is a direct input to the Poseidon block-hash preimage. A block committed with an attacker-chosen sequencer address produces a different block hash than the one honest nodes would compute from the configured `builder_address`, permanently corrupting the chain's canonical hash chain.

---

### Likelihood Explanation

In a multi-validator BFT deployment the proposer role rotates among validators. Any validator that is legitimately selected as proposer for a round can craft a `ProposalInit` with an arbitrary `builder` value. No special privilege beyond being the round's proposer is required. The attack is silent: the proposal passes all existing checks and is indistinguishable from an honest proposal to every other validator.

---

### Recommendation

Add a `builder` field to `ProposalInitValidation` (populated from the node's own `static_config.builder_address`) and enforce exact equality inside `is_proposal_init_valid`, mirroring the existing check for `l1_da_mode` and `l2_gas_price_fri`:

### Proof of Concept

1. Attacker node is selected as proposer for round R at height H.
2. It constructs `ProposalInit` with `builder = attacker_wallet_address` (any address it controls).
3. It broadcasts the proposal stream to all validators.
4. Each validator calls `is_proposal_init_valid` — passes, because `builder` is never checked.
5. Each validator calls `initiate_validation` → `convert_to_sn_api_block_info` → `sequencer_address = attacker_wallet_address`.
6. The batcher executes all transactions; every fee transfer credits `attacker_wallet_address`.
7. `PartialBlockHashComponents` encodes `attacker_wallet_address` as `sequencer`.
8. `batcher_block_commitment` (validator-computed) equals `received_fin.proposal_commitment` (attacker-computed) — both used the same `builder`.
9. `validate_proposal` returns `Ok(built_block)`, consensus decides, block is committed.
10. All fees from height H are in `attacker_wallet_address`; the canonical block hash encodes the wrong sequencer. [9](#0-8) [10](#0-9)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
