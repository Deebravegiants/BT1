### Title
Node with Expired or Revoked TEE Attestation Bypasses Attestation Validity Check and Participates in Signing and Governance - (File: `crates/contract/src/tee/tee_state.rs`)

---

### Summary

`is_caller_an_attested_participant` verifies only that a stored attestation **exists** for the calling node, but never calls `reverify_participants` to confirm the attestation is still **valid** (not expired, not tied to a revoked image hash). This is the direct structural analog of the external report: the UStb contract checked `hasRole(WHITELISTED_ROLE)` without checking `!hasRole(BLACKLISTED_ROLE)`, while the MPC contract checks `stored_attestations.get(tls_key).is_some()` without checking `re_verify(…).is_ok()`.

---

### Finding Description

`TeeState::is_caller_an_attested_participant` (called from `assert_caller_is_attested_participant_and_protocol_active`, which gates `respond`, `vote_pk`, `vote_reshared`, `start_keygen_instance`, `start_reshare_instance`, and `vote_abort_key_event_instance`) performs four checks:

1. Caller is in the active `Participants` set.
2. A `NodeAttestation` entry exists in `stored_attestations` keyed by the caller's TLS public key.
3. The stored `account_id` matches the signer.
4. The stored `account_public_key` matches the signer's Ed25519 key. [1](#0-0) 

It does **not** call `reverify_participants`, which is the function that re-runs the full attestation validity pipeline (expiry timestamp, allowed MPC docker image hash, allowed launcher compose hash, allowed OS measurements): [2](#0-1) 

A node's stored attestation can become invalid after it was accepted because:
- The attestation's expiry timestamp passes.
- The MPC docker image hash the attestation was tied to is removed from the allowlist (e.g., after a vulnerability is found in that image version).
- The launcher compose hash or OS measurement is removed from the allowlist.

`clean_invalid_attestations` is only triggered as a deferred promise after resharing completes, leaving a window during which a node with an invalid attestation remains in `stored_attestations` and passes `is_caller_an_attested_participant`. [3](#0-2) 

The `reverify_and_cleanup_participants` path (called during `vote_new_parameters`) does perform validity checks, but this is only on the proposed participant set for governance proposals, not on the caller of `respond` or other node-facing methods. [4](#0-3) 

---

### Impact Explanation

A node running a revoked or expired TEE image can:

1. **Call `respond()`** — submit a threshold signature to the contract. The contract verifies the signature cryptographically, but the node's TEE may be running untrusted code (the reason its image hash was revoked). The contract's own invariant — that only nodes with currently valid TEE attestations may submit responses — is broken. [5](#0-4) 

2. **Cast governance votes** (`vote_pk`, `vote_reshared`, `start_keygen_instance`, `start_reshare_instance`, `vote_abort_key_event_instance`) — a node whose TEE is no longer trusted participates in key-generation and resharing governance, breaking the production safety invariant that only nodes with valid attestations may influence the MPC key lifecycle. [6](#0-5) [7](#0-6) 

This maps to the allowed Medium impact: **participant-state manipulation that breaks production safety invariants** — specifically, the invariant that only TEE-attested nodes participate in the MPC protocol.

---

### Likelihood Explanation

- TEE attestations carry expiry timestamps. Any node that does not refresh its attestation before expiry enters this state automatically.
- When participants vote to revoke an image hash (`vote_code_hash` whitelisting a new hash implicitly makes the old one expire after the upgrade deadline), all nodes still running the old image have their attestations invalidated, but remain in `stored_attestations` until `clean_invalid_attestations` runs.
- `clean_invalid_attestations` is only triggered as a deferred promise after resharing; it is bounded (`max_scan`) and may not clear all invalid entries in one call.
- The window between attestation invalidation and cleanup is therefore non-trivial and predictable. [8](#0-7) 

---

### Recommendation

Add a `reverify_participants` call inside `is_caller_an_attested_participant` (or inside `assert_caller_is_attested_participant_and_protocol_active`) after confirming the attestation exists, and return an error if `TeeQuoteStatus::Invalid` is returned. The `tee_upgrade_deadline_duration` is already available on the `MpcContract` via `self.config.tee_upgrade_deadline_duration_seconds`.

```rust
// After the existing account_id / account_public_key checks:
let tee_status = self.reverify_participants(&attestation.node_id, tee_upgrade_deadline_duration);
if !matches!(tee_status, TeeQuoteStatus::Valid) {
    return Err(AttestationCheckError::AttestationInvalid);
}
```

---

### Proof of Concept

1. Node A submits a valid attestation via `submit_participant_info`; it is accepted and stored in `stored_attestations`.
2. Participants vote to whitelist a new MPC docker image hash via `vote_code_hash`. The old image hash's upgrade deadline passes, making Node A's attestation invalid under `reverify_participants`.
3. `clean_invalid_attestations` has not yet run (no resharing has completed since the hash change).
4. Node A calls `respond(request, valid_signature)`. The contract calls `assert_caller_is_attested_participant_and_protocol_active`, which calls `is_caller_an_attested_participant`. Node A is still in `stored_attestations` and passes all four checks. The contract accepts the response despite Node A running a revoked image.
5. Node A also calls `vote_pk(key_event_id, public_key)` and casts a governance vote, despite its TEE no longer being trusted. [1](#0-0) [2](#0-1)

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub fn reverify_and_cleanup_participants(
        &mut self,
        participants: &Participants,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeValidationResult {
        self.allowed_docker_image_hashes
            .cleanup_expired_hashes(tee_upgrade_deadline_duration);

        let participants_with_valid_attestation: Vec<_> = participants
            .participants()
            .iter()
            .filter(|(_, _, participant_info)| {
                // Use the stored NodeId (keyed by TLS public key) so the real
                // `account_public_key` participates in re-verification. If
                // there is no stored attestation for this TLS key, the
                // participant is invalid.
                let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
                else {
                    return false;
                };

                let tee_status =
                    self.reverify_participants(&node_id, tee_upgrade_deadline_duration);

                matches!(tee_status, TeeQuoteStatus::Valid)
            })
            .cloned()
            .collect();

        if participants_with_valid_attestation.len() != participants.len() {
            let participants_with_valid_attestation =
                Participants::init(participants.next_id(), participants_with_valid_attestation);

            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            }
        } else {
            TeeValidationResult::Full
        }
    }

    pub fn clean_invalid_attestations(
        &mut self,
        tee_upgrade_deadline_duration: Duration,
        max_scan: usize,
    ) -> u32 {
        let has_invalid_attestation = |node_id: &NodeId| {
            !matches!(
                self.reverify_participants(node_id, tee_upgrade_deadline_duration),
                TeeQuoteStatus::Valid
            )
        };

        // Materialize candidates before any mutation to avoid iterator invalidation.
        let invalid_tls_keys: Vec<Ed25519PublicKey> = self
            .stored_attestations
            .iter()
            .take(max_scan)
            .filter(|(_, node_attestation)| has_invalid_attestation(&node_attestation.node_id))
            .map(|(tls_pk, _)| tls_pk.clone())
            .collect();

        let removed = u32::try_from(invalid_tls_keys.len())
            .expect("u32 should always be convertible from usize on wasm32");

        for tls_pk in invalid_tls_keys {
            self.stored_attestations.remove(&tls_pk);
        }
        removed
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
