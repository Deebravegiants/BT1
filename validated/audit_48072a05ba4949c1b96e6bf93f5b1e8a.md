### Title
Attacker-Controlled `authorized_providers` Injected via On-Chain CRCAT Spend Irreversibly Converts Victim's CATWallet — (`chia/wallet/wallet_state_manager.py`, `chia/wallet/vc_wallet/cr_cat_wallet.py`)

### Summary

An unprivileged attacker who holds any amount of a CAT token (for a TAIL hash the victim also holds as a plain `CATWallet`) can submit a valid on-chain spend bundle that converts those tokens into a CRCAT with attacker-chosen `authorized_providers` and `proofs_checker`, hinted to the victim's puzzle hash. The victim's wallet sync path (`handle_cat` → `CRCATWallet.convert_to_cr`) unconditionally overwrites the victim's `CATWallet` DB record with the attacker-supplied CRCAT parameters, permanently converting the wallet type and locking the victim's existing CAT coins behind an attacker-controlled VC access policy.

### Finding Description

**Step 1 — `get_next_from_coin_spend` reads `authorized_providers` from unverified on-chain data**

When the parent spend is a plain CAT (not already a CRCAT), `CRCAT.get_next_from_coin_spend` extracts `authorized_providers` and `proofs_checker` from a REMARK condition embedded in the inner solution: [1](#0-0) 

The attacker fully controls this REMARK condition because they are spending their own CAT coins. There is no signature or trusted-source validation on these values.

**Step 2 — `handle_cat` calls `convert_to_cr` with the attacker-supplied values**

In `WalletStateManager.handle_cat`, after detecting the coin is a CRCAT, the code checks whether the victim already has a CRCAT wallet for this asset ID (lines 1292–1295). If not, it falls through to the CAT wallet conversion block and calls `CRCATWallet.convert_to_cr` directly with `crcat.authorized_providers` and `crcat.proofs_checker` — both sourced from the attacker's spend: [2](#0-1) 

The only guard before this point (lines 1280–1289) checks that the CRCAT's `inner_puzzle_hash` either matches a derivation record or equals `construct_pending_approval_state(hinted_coin.hint, amount).get_tree_hash()`. The attacker satisfies the second branch by constructing the CRCAT output with exactly that inner puzzle hash — a value they can compute from the victim's public puzzle hash (the hint). [3](#0-2) 

**Step 3 — `convert_to_cr` performs no guard on existing wallet coins**

`CRCATWallet.convert_to_cr` unconditionally overwrites the wallet record in the DB with the attacker-supplied `authorized_providers` and `proofs_checker`, and replaces the in-memory wallet object: [4](#0-3) 

By contrast, the analogous `RCATWallet.convert_to_revocable` explicitly guards against this by refusing to convert if the wallet already holds coins: [5](#0-4) 

No such guard exists in `convert_to_cr`.

### Impact Explanation

- The victim's `CATWallet` is permanently converted to a `CRCATWallet` whose `authorized_providers` and `proofs_checker` are entirely attacker-controlled.
- All existing plain CAT coins the victim holds become inaccessible through the wallet: the wallet now treats them as CRCAT coins and will attempt to spend them with a VC from the attacker's providers, which the victim does not possess.
- The victim cannot spend their coins through normal wallet operations until the wallet record is manually repaired at the DB level.
- This constitutes **corruption of wallet coin-control state with direct security impact** (High impact per scope rules).

### Likelihood Explanation

- The attacker needs only to acquire a non-zero amount of the target CAT token — a permissionless market action for any publicly traded CAT.
- No leaked keys, admin access, or cryptographic breaks are required.
- The attack is triggered passively by the victim's wallet sync; the victim takes no action.
- The attack is repeatable and affects any victim who holds a plain `CATWallet` for a publicly traded TAIL.

### Recommendation

1. **Validate `authorized_providers` against a trusted source** before calling `convert_to_cr`. The wallet should only accept a CRCAT-triggered conversion if the `authorized_providers` match a user-configured or previously trusted list.
2. **Add an existing-coin guard to `convert_to_cr`**, mirroring the guard in `convert_to_revocable`:
   ```python
   if not await cat_wallet.lineage_store.is_empty():
       cat_wallet.log.error("Received a CRCAT to a CAT wallet that already has CATs")
       return
   ```
3. **Do not auto-convert** a `CATWallet` to `CRCATWallet` based solely on an inbound on-chain coin. Require explicit user confirmation before any irreversible wallet-type change.

### Proof of Concept

1. Victim has a `CATWallet` for TAIL hash `T` with existing plain CAT coins.
2. Attacker acquires any amount of CAT tokens with TAIL hash `T`.
3. Attacker computes `victim_ph` (victim's puzzle hash, publicly visible from prior transactions or hints).
4. Attacker constructs a spend bundle:
   - Spends their CAT coin with inner solution containing REMARK condition: `(1 pending_approval_state_hash [attacker_provider_id] attacker_proofs_checker_program)`
   - where `pending_approval_state_hash = construct_pending_approval_state(victim_ph, amount).get_tree_hash()`
   - The output coin is hinted to `victim_ph`
5. Attacker submits the spend bundle; it is valid and accepted by consensus.
6. Victim's wallet receives the coin state update (hinted to their puzzle hash).
7. `handle_cat` is called → detects CRCAT → inner puzzle hash matches pending approval state → no existing CRCAT wallet found → calls `CRCATWallet.convert_to_cr(victim_cat_wallet, [attacker_provider_id], attacker_proofs_checker)`.
8. Victim's `CATWallet` DB record is overwritten with `WalletType.CRCAT` and attacker-controlled parameters.
9. Assert: victim's wallet type is now `CRCAT`; `authorized_providers` equals `[attacker_provider_id]`; victim's existing plain CAT coins are no longer spendable through the wallet.

    @classmethod
    async def convert_to_cr(
        cls,
        cat_wallet: CATWallet,
        authorized_providers: list[bytes32],
        proofs_checker: ProofsChecker,
    ) -> None:
        replace_self = cls()
        replace_self.standard_wallet = cat_wallet.standard_wallet
        replace_self.log = logging.getLogger(cat_wallet.get_name())
        replace_self.log.info(f"Converting CAT wallet {cat_wallet.id()} to CR-CAT wallet")
        replace_self.wallet_state_manager = cat_wallet.wallet_state_manager
        replace_self.info = cls.wallet_info_type(
            cat_wallet.cat_info.limitations_program_hash, None, authorized_providers, proofs_checker
        )
        await cat_wallet.wallet_state_manager.user_store.update_wallet(
            WalletInfo(
                cat_wallet.id(), cat_wallet.get_name(), uint8(WalletType.CRCAT.value), bytes(replace_self.info).hex()
            )
        )
        updated_wallet_info = await cat_wallet.wallet_state_manager.user_store.get_wallet_by_id(cat_wallet.id())
        assert updated_wallet_info is not None
        replace_self.wallet_info = updated_wallet_info

        cat_wallet.wallet_state_manager.wallets[cat_wallet.id()] = replace_self
