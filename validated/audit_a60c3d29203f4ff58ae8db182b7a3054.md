### Title
Gateway Admits `AllResources` Transactions With Zero L1/L1-Data Gas Price That Inevitably Fail at Blockifier Pre-Validation — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's resource-bounds validation only checks the transaction's `l2_gas.max_price_per_unit` against the previous block's L2 gas price. It never validates `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The blockifier's `check_fee_bounds`, however, enforces all three gas prices at execution time. Any `AllResources` transaction with a zero (or below-market) L1 or L1-data gas price therefore passes every gateway and mempool check, is admitted to the mempool, and then fails with a `TransactionPreValidationError` the moment the batcher hands it to the blockifier — wasting sequencer resources and enabling systematic mempool pollution with transactions that can never be sequenced.

---

### Finding Description

**Stateless validator — only L2 gas price is checked**

`StatelessTransactionValidator::validate_resource_bounds` performs two checks:

1. `max_possible_fee(Tip::ZERO) == Fee(0)` — rejects only if the *total* fee across all resources is zero.
2. `l2_gas.max_price_per_unit < min_gas_price` — rejects only if the L2 gas price is below a configured floor. [1](#0-0) 

Neither `l1_gas.max_price_per_unit` nor `l1_data_gas.max_price_per_unit` is compared against any minimum. A transaction with `l2_gas.max_price_per_unit = valid`, `l1_gas.max_price_per_unit = 0`, `l1_data_gas.max_price_per_unit = 0` passes both checks because `max_possible_fee` is non-zero (contributed by L2 gas alone).

**Stateful validator — same gap, with an explicit TODO**

`StatefulTransactionValidator::validate_resource_bounds` reads the previous block's L2 gas price and calls `validate_tx_l2_gas_price_within_threshold`. The function is prefixed with the comment:

```
// TODO(Arni): Consider running this validation for all gas prices.
``` [2](#0-1) 

Inside `validate_tx_l2_gas_price_within_threshold`, only `l2_gas.max_price_per_unit` is extracted and compared; `l1_gas` and `l1_data_gas` prices are never touched. [3](#0-2) 

**Blockifier enforces all three prices at execution time**

`AccountTransaction::check_fee_bounds` builds a tuple of `(resource, resource_bounds, minimal_gas_amount, actual_gas_price)` for **all three** resources — L1Gas, L1DataGas, and L2Gas — and rejects the transaction if `resource_bounds.max_price_per_unit < actual_gas_price.get()` for any of them. [4](#0-3) 

This check runs inside `perform_pre_validation_stage`, which is called before any state mutation. [5](#0-4) 

**Mempool also tracks only L2 gas price**

The mempool's `validate_tx` and `add_tx` log and filter on `max_l2_gas_price` only; there is no L1 gas price filter at the mempool layer either. [6](#0-5) 

---

### Impact Explanation

Any `AllResources` transaction with `l1_gas.max_price_per_unit = 0` (or below the current L1 gas price) passes the full gateway pipeline — stateless validation, stateful validation, and mempool admission — and is stored in the mempool. When the batcher dequeues it and the blockifier calls `check_fee_bounds`, the transaction fails with `TransactionPreValidationError(InsufficientResourceBounds { MaxGasPriceTooLow { L1Gas } })` and is discarded without being included in any block. The nonce is not permanently incremented (pre-validation errors are rolled back), but the batcher and blockifier have already consumed CPU and I/O to process the transaction. An attacker can continuously flood the mempool with such transactions, degrading sequencer throughput.

This matches the allowed impact: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

---

### Likelihood Explanation

The trigger is fully unprivileged. Any user with a funded account can submit a V3 (`AllResources`) transaction with a valid L2 gas price and zero L1/L1-data gas prices. The gateway returns a success response, and the transaction is stored in the mempool. No special role, key, or network position is required. The gap is acknowledged in the source with a TODO comment, confirming it is not an intentional design choice.

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or introduce a parallel function) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK prices, using the same percentage-threshold logic already applied to L2 gas. The stateless validator should similarly extend its `min_gas_price` floor check to cover all three resource types. This mirrors the completeness already present in `check_fee_bounds` inside the blockifier.

---

### Proof of Concept

1. Obtain a funded account on the network.
2. Construct an `AllResources` (V3) invoke transaction with:
   - `l2_gas.max_price_per_unit` = current L2 gas price (passes gateway threshold)
   - `l2_gas.max_amount` = any non-zero value
   - `l1_gas.max_price_per_unit` = `0`
   - `l1_data_gas.max_price_per_unit` = `0`
3. Submit via the gateway RPC endpoint.
4. **Stateless check** (`validate_resource_bounds`): `max_possible_fee(Tip::ZERO)` is non-zero (L2 contribution) → passes; `l2_gas.max_price_per_unit >= min_gas_price` → passes. L1 prices are never examined. [7](#0-6) 
5. **Stateful check** (`validate_tx_l2_gas_price_within_threshold`): only `l2_gas.max_price_per_unit` is compared against the previous block price → passes. [3](#0-2) 
6. Transaction is admitted to the mempool. Gateway returns success.
7. Batcher dequeues the transaction; blockifier calls `perform_pre_validation_stage` → `check_fee_bounds`.
8. For `AllResources`, `l1_gas.max_price_per_unit (0) < actual_l1_gas_price (> 0)` → `ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas }` → `TransactionPreValidationError`. [8](#0-7) 
9. Transaction is discarded; no block inclusion, no state change, but sequencer resources are consumed.
10. Repeat at scale to degrade sequencer throughput.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier

                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {

                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
