### Title
Pre-reveal size check ordering in `agree_random` allows malicious P2 to cause state divergence in `key_share_2p_t::refresh` — (`src/cbmpc/protocol/agree_random.cpp`)

### Summary
In `agree_random`, P1 unconditionally sends `r1` and `com.rand` to P2 (line 25) before the size check on the received `r2` (line 32). A malicious P2 who sends a wrong-length `r2` causes P1 to abort after revealing `r1`, while P2 successfully completes the protocol. When called from `key_share_2p_t::refresh`, this leaves P1 holding the old key share and P2 holding an updated share derived from a scalar P1 never accepted, permanently destroying the shared key.

### Finding Description
The `agree_random` function implements a commit-then-reveal coin-flip:

1. P1 commits to `r1`, sends `com.msg`
2. P2 sends `r2`
3. P1 reveals `r1` and `com.rand`
4. Both verify and XOR

The size validation of `r2` (`r2.size() != bits_to_bytes(bitlen)`) occurs at line 32, **after** P1 has already transmitted `r1` and `com.rand` at line 25. A Byzantine P2 can send a `buf_t` of any non-matching length. The `p2_to_p1` call at line 24 performs no length pre-check — it simply deserializes whatever bytes arrive. P1 then proceeds to line 25 and sends `r1` unconditionally. Only at line 32 does P1 detect the malformed `r2` and abort.

On P2's side, `r2` is P2's own correctly-sized value, so line 32 passes for P2. P2 also received a valid `r1` and verified the commitment at line 28. P2 computes `out = r1 XOR r2` and returns `SUCCESS`. [4](#0-3) 

### Impact Explanation
In `key_share_2p_t::refresh`, `agree_random` is the sole source of the refresh scalar `r`:

```cpp
if (rv = agree_random(job, q.get_bits_count() + SEC_P_STAT, rand_bits)) return rv;
bn_t r = bn_t::from_bin(rand_bits) % q;
``` [5](#0-4) 

P1 aborts and retains `x1_old`. P2 stores `x2_old - r`. The invariant `x1 + x2 = x` is broken. The key pair is permanently inconsistent — neither party can detect this without an out-of-band check, and any subsequent signing attempt will produce invalid signatures or fail. This constitutes accepted invalid cryptographic output (P2 accepts a new key share that does not correspond to the actual private key) caused by a validation bypass in the refresh protocol, matching the High impact criterion.

### Likelihood Explanation
P2 is a direct protocol peer with full control over the bytes it sends in round 2. No threshold assumption protects against a single malicious P2 in a 2-party setting. The attack requires only that P2 send a `buf_t` of the wrong length — a trivial transport-layer manipulation.

### Recommendation
Move the size check on `r2` to **before** P1 sends `r1` and `com.rand`. Specifically, after `job.p2_to_p1(r2)` returns, P1 should validate `r2.size() == bits_to_bytes(bitlen)` and abort immediately if it fails, before executing `job.p1_to_p2(r1, com.rand)`. This ensures P1 never reveals its decommitment to a peer that has already deviated.

```cpp
if (rv = job.p2_to_p1(r2)) return rv;
// Guard: validate r2 length before revealing r1
if (r2.size() != coinbase::bits_to_bytes(bitlen)) return coinbase::error(E_CRYPTO);
if (rv = job.p1_to_p2(r1, com.rand)) return rv;
```

### Proof of Concept
1. Construct a fake `data_transport_i` for P2 that intercepts round 2 and substitutes a `buf_t` of length `bits_to_bytes(bitlen) + 1`.
2. Run P1's `agree_random` in one thread and P2's in another, connected through the fake transport.
3. Assert P1 returns a non-`SUCCESS` error code.
4. Assert P2 returns `SUCCESS` and `out` is populated with `r1 XOR r2`.
5. Assert that calling `key_share_2p_t::refresh` under the same transport leaves P1's `new_key` unmodified (old share) and P2's `new_key.x_share == x2_old - r`.
6. Assert `(P1's x_share) + (P2's new x_share) != x` (verify against the public key `Q`).

error_t agree_random(job_2p_t& job, int bitlen, buf_t& out) {
  error_t rv = UNINITIALIZED_ERROR;
  buf_t r1, r2;
  const crypto::mpc_pid_t& sender_pid = job.get_pid(party_t::p1);
  coinbase::crypto::commitment_t com(sender_pid);

  if (job.is_p1()) {
    r1 = crypto::gen_random_bitlen(bitlen);
    com.gen(r1);
  }

  if (rv = job.p1_to_p2(com.msg)) return rv;

  if (job.is_p2()) {
    r2 = crypto::gen_random_bitlen(bitlen);
  }

  if (rv = job.p2_to_p1(r2)) return rv;
  if (rv = job.p1_to_p2(r1, com.rand)) return rv;

  if (job.is_p2()) {
    if (rv = com.open(r1)) return rv;
  }

  if (r1.size() != coinbase::bits_to_bytes(bitlen)) return coinbase::error(E_CRYPTO);
  if (r2.size() != coinbase::bits_to_bytes(bitlen)) return coinbase::error(E_CRYPTO);

  out = mem_t(r1) ^ mem_t(r2);
  return SUCCESS;
}

error_t key_share_2p_t::refresh(job_2p_t& job, const key_share_2p_t& key, key_share_2p_t& new_key) {
  error_t rv = UNINITIALIZED_ERROR;
  new_key.role = key.role;
  new_key.curve = key.curve;
  new_key.Q = key.Q;

  const mod_t& q = key.curve.order();
  buf_t rand_bits;
  if (rv = agree_random(job, q.get_bits_count() + SEC_P_STAT, rand_bits)) return rv;
  bn_t r = bn_t::from_bin(rand_bits) % q;

  if (job.is_p1()) {
    MODULO(q) { new_key.x_share = key.x_share + r; }
  }

  if (job.is_p2()) {
    MODULO(q) { new_key.x_share = key.x_share - r; }
  }

  return SUCCESS;
}
