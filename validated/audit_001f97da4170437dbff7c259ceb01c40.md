### Title
Sub-Transactions in Dijkstra Era Bypass Value Conservation, Enabling Direct ADA Creation — (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxo.hs`)

---

### Summary

The Dijkstra era's `SUBUTXO` rule processes sub-transactions without enforcing the preservation-of-value property (`validateValueNotConservedUTxO`). The top-level `UTXO` rule enforces value conservation only for the top-level transaction body against the pre-batch UTxO (`originalUtxo`), and does not account for sub-transaction inputs or outputs. An unprivileged transaction author can embed a sub-transaction that spends a small UTxO input and creates arbitrarily large outputs, with no ledger rule rejecting the imbalance.

---

### Finding Description

The Dijkstra era introduces nested transactions (`SubTx`) embedded inside a top-level transaction (`TopTx`). The processing pipeline is:

```
LEDGER → SUBLEDGERS → SUBLEDGER → SUBUTXOW → SUBUTXO
```

**Root cause — `SUBUTXO` omits value conservation:**

`dijkstraSubUtxoTransition` in `SubUtxo.hs` performs the following checks for a sub-transaction:

- Validity interval (`validateOutsideValidityIntervalUTxO`)
- Forecast range (`validateOutsideForecast`)
- Non-empty input set (`validateInputSetEmptyUTxO`)
- Inputs exist in `originalUtxo` and current `utxoState` (`validateBadInputsUTxO`)
- Output size/value limits
- Network ID checks

It then calls `Shelley.updateUTxOStateNoFees` to apply the state change. **`validateValueNotConservedUTxO` is never called.** This is structurally confirmed by the error-mapping function `dijkstraUtxoToDijkstraSubUtxoPredFailure`, which explicitly marks both `ValueNotConservedUTxO` and `FeeTooSmallUTxO` as runtime `error "Impossible"` — meaning the `SUBUTXO` rule is architecturally incapable of producing these failures.

**Root cause — top-level check does not cover sub-transactions:**

`dijkstraUtxoTransition` in `Utxo.hs` does call `validateValueNotConservedUTxO`, but only for the top-level transaction body:

```haskell
runTest $ Shelley.validateValueNotConservedUTxO pp originalUtxo certState txBody
```

`originalUtxo` is the UTxO snapshot taken **before** any sub-transactions are processed. `txBody` is the top-level transaction body only. Sub-transaction inputs and outputs are invisible to this check. By the time the top-level UTXOW rule runs, the UTxO state passed to it (`utxoStateBeforeUtxow`) already reflects all sub-transaction mutations, but the value conservation predicate was evaluated against the pre-mutation snapshot and the top-level body alone.

**Processing order in `dijkstraLedgerTransition`:**

```
1. SUBLEDGERS processes all sub-transactions → utxoStateAfterSubLedgers
2. ENTITIES processes top-level certificates
3. GOV processes top-level governance
4. UTXOW/UTXO processes top-level tx with:
     - environment: originalUtxo (pre-sub-tx snapshot)
     - state:       utxoStateAfterSubLedgers (post-sub-tx)
   → validateValueNotConservedUTxO uses originalUtxo + top-level txBody only
```

The sub-transaction's value imbalance is already committed to the ledger state before the only value-conservation check runs, and that check is scoped to the top-level transaction.

---

### Impact Explanation

**Critical — Direct creation of ADA through an invalid ledger state transition.**

An attacker can embed a sub-transaction that:
- Spends one or more UTxO inputs totalling `V_in` lovelace
- Declares outputs totalling `V_out >> V_in` lovelace

No ledger rule rejects this. The surplus `V_out − V_in` lovelace is permanently added to the UTxO, violating the global preservation-of-value invariant. This constitutes direct, attacker-controlled creation of ADA without any corresponding destruction of existing value. All honest nodes applying the same block would accept the same invalid state, causing no divergence but permanently inflating the ADA supply.

---

### Likelihood Explanation

Any user who can submit a valid Dijkstra-era transaction can exploit this. No privileged role, governance majority, or leaked key is required. The attacker only needs to:
1. Hold any UTxO output (even dust)
2. Construct a top-level Dijkstra transaction with a sub-transaction body that spends that output and declares inflated outputs
3. Provide valid witnesses for the top-level transaction

The sub-transaction body (`DijkstraSubTxBodyRaw`) has no `fee` field and no value-conservation enforcement. The exploit is mechanically straightforward once the Dijkstra era is active.

---

### Recommendation

Add value conservation validation to `dijkstraSubUtxoTransition`, analogous to the check in `dijkstraUtxoTransition`:

```haskell
-- In dijkstraSubUtxoTransition, before the updateUTxOStateNoFees call:
runTest $ Shelley.validateValueNotConservedUTxO pp (utxosUtxo utxoState) certState txBody
```

Note that the sub-transaction should use the **current** `utxoState` (not `originalUtxo`) as its reference UTxO, since earlier sub-transactions in the same batch may have already consumed some inputs. Alternatively, the top-level `validateValueNotConservedUTxO` call should be extended to account for the aggregate consumed/produced values across all sub-transactions in the batch.

The `dijkstraUtxoToDijkstraSubUtxoPredFailure` mapping should also be updated to handle `ValueNotConservedUTxO` rather than panicking.

dijkstraLedgerTransition = do
  TRC (Shelley.LedgerEnv slot mbCurEpochNo txIx pp chainAccountState, ledgerState, stAnnTx) <-
    judgmentContext
  let tx = stAnnTx ^. txStAnnTxG

  -- Capture the original UTxO before any subtransaction processing.
  -- This is passed through the environment to UTXOW
  -- and SUBLEDGERS, and used for all witness/validation lookups.
  let originalUtxo = utxosUtxo (ledgerState ^. lsUTxOStateL)
      subStAnnTxs = subTransactionsStAnnTx stAnnTx

  -- Process all subtransactions first
  LedgerState utxoStateAfterSubLedgers certStateAfterSubLedgers <-
    trans @(EraRule "SUBLEDGERS" era) $
      TRC
        ( SubLedgerEnv
            slot
            mbCurEpochNo
            txIx
            pp
            chainAccountState
            originalUtxo
            (tx ^. isValidTxL)
        , ledgerState
        , subStAnnTxs
        )

  curEpochNo <- maybe (liftSTS $ epochFromSlot slot) pure mbCurEpochNo

  (utxoStateBeforeUtxow, certStateFinal) <-
    if tx ^. isValidTxL == IsValid True
      then do
        let txBody = tx ^. bodyTxL
        runTest $ Conway.validateTreasuryValue txBody (chainAccountState ^. casTreasuryL)
        runTest $ validateAllRefScriptSize pp originalUtxo tx

        let govState = utxoStateAfterSubLedgers ^. utxosGovStateL
            committee = govState ^. committeeGovStateL
            proposals = govState ^. proposalsGovStateL
            committeeProposals = proposalsWithPurpose grCommitteeL proposals

        certStateAfterENTITIES <-
          trans @(EraRule "ENTITIES" era) $
            TRC
              ( EntitiesEnv
                  (stAnnTx ^. plutusLegacyModeStAnnTxG)
                  (Conway.CertsEnv tx pp curEpochNo committee committeeProposals)
              , certStateAfterSubLedgers
              , StrictSeq.fromStrict $ txBody ^. certsTxBodyL
              )

        let govSignal =
              Conway.GovSignal
                { Conway.gsVotingProcedures = txBody ^. votingProceduresTxBodyL
                , Conway.gsProposalProcedures = txBody ^. proposalProceduresTxBodyL
                , Conway.gsCertificates = txBody ^. certsTxBodyL
                }
        proposalsState <-
          trans @(EraRule "GOV" era) $
            TRC
              ( Conway.GovEnv
                  (txIdTxBody txBody)
                  curEpochNo
                  pp
                  (govState ^. constitutionGovStateL . constitutionGuardrailsScriptHashL)
                  certStateAfterENTITIES
                  (govState ^. committeeGovStateL)
              , proposals
              , govSignal
              )
        pure
          ( utxoStateAfterSubLedgers
              & utxosGovStateL . proposalsGovStateL .~ proposalsState
          , certStateAfterENTITIES
          )
      else pure (utxoStateAfterSubLedgers, certStateAfterSubLedgers)

  -- Call UTXOW with DijkstraUtxoEnv, passing the original UTxO and original certState
  utxoStateFinal <-
    trans @(EraRule "UTXOW" era) $
      TRC
        ( DijkstraUtxoEnv slot pp (lsCertState ledgerState) originalUtxo
        , utxoStateBeforeUtxow
        , stAnnTx
        )
  pure $ LedgerState utxoStateFinal certStateFinal
