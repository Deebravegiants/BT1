### Title
Unvalidated `ProposalInit.builder` Allows Proposer to Redirect All Block Fee Transfers to an Arbitrary Address â€” (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`ProposalInit.builder` is the network-transmitted address that becomes `BlockInfo.sequencer_address` â€” the recipient of every fee transfer in a block. The validator's `is_proposal_init_valid` function checks timestamp, starknet version, version constant commitment, height, L1 DA mode, L2 gas price, L1 gas prices, and fee proposal, but never validates `builder`. A malicious proposer can set `builder` to zero or any attacker-controlled address; the validator accepts the proposal, executes all transactions with the wrong fee recipient, and commits a block header containing the wrong sequencer address.

### Finding Description

`ProposalInit` carries a `builder` field defined as "Address of the one who builds/sequences the block": [1](#0-0) 

`convert_to_sn_api_block_info` maps `init.builder` directly to `BlockInfo.sequencer_address` with no guard: [2](#0-1) 

`BlockInfo.sequencer_address` is then used as the calldata recipient in every fee transfer: [3](#0-2) 

The validator's `is_proposal_init_valid` checks seven distinct fields but has no branch that reads or constrains `init.builder`: [4](#0-3) 

`ProposalInitValidation` â€” the struct that carries the locally-trusted reference values â€” does not include an expected builder address, so there is no reference value to compare against: [5](#0-4) 

The proposer sets `builder` from `args.builder_address` during `initiate_build`: [6](#0-5) 

Because the validator never checks this field, a proposer can substitute any address â€” including `ContractAddress::default()` (zero) â€” and the validator will call `initiate_validation` â†’ `convert_to_sn_api_block_info` with the tampered value, passing it to the batcher as the authoritative `sequencer_address` for the block.

### Impact Explanation

Every account transaction in the block calls `execute_fee_transfer` with `block_context.block_info.sequencer_address` as the ERC-20 transfer recipient. If `builder` is zero, all fees are burned. If `builder` is an attacker address, all fees are stolen. Additionally, `sequencer_address` is part of `PartialBlockHashComponents` and therefore part of the committed block hash, so the wrong `builder` also corrupts the authoritative block commitment accepted by consensus.

This matches the **Critical** impact category: *Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.*

### Likelihood Explanation

The trigger is a single field substitution in a `ProposalInit` message sent over the p2p network. Any node that wins a proposer slot can exploit this. No privileged access beyond being a consensus participant is required. The validator has no existing guard and no reference value to compare against.

### Recommendation

1. Add `expected_builder: ContractAddress` to `ProposalInitValidation`, populated from the node's own configured sequencer/builder address.
2. In `is_proposal_init_valid`, reject any `init_proposed.builder != proposal_init_validation.expected_builder`.
3. Add a non-zero check: reject `init_proposed.builder == ContractAddress::default()`.
4. Add a test analogous to the existing `invalid_proposal_init` test that verifies a tampered `builder` is rejected.

### Proof of Concept

1. A malicious node wins a proposer slot at height H.
2. It constructs `ProposalInit` with all legitimate fields except `builder: ContractAddress::default()` (zero address).
3. It streams this `ProposalInit` to validators.
4. Validators call `is_proposal_init_valid` â€” all checks pass because `builder` is never read.
5. Validators call `initiate_validation` â†’ `convert_to_sn_api_block_info(init)` â†’ `BlockInfo { sequencer_address: ContractAddress::default(), â€¦ }`.
6. The batcher executes all transactions; every `execute_fee_transfer` sends fees to address `0x0`.
7. `validate_proposal` returns `Ok(built_block)`, consensus reaches decision, and the block with `sequencer_address = 0x0` is committed.
8. All transaction fees for block H are permanently lost; the committed block header contains the wrong sequencer address. [7](#0-6) [8](#0-7)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
