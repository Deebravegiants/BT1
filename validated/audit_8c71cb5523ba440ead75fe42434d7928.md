### Title
Single Byzantine Attested Participant Can Inject Arbitrary CKD Output for `AppPublicKey` Requests — (`crates/contract/src/lib.rs`, `crates/contract/src/primitives/ckd.rs`)

---

### Summary

`respond_ckd` performs **no cryptographic verification** of the response when the request uses the `AppPublicKey` variant. A single attested participant — strictly below the signing threshold — can call `respond_ckd` with an arbitrary fabricated `CKDResponse` and the contract will accept it, resolving the victim's yield with attacker-controlled key material.

---

### Finding Description

`CKDRequest` derives `Eq/Ord/PartialEq/PartialOrd` over all three fields including `app_public_key`: [1](#0-0) 

Two requests with the same `(predecessor_id, derivation_path, domain_id)` but different `app_public_key` values are therefore stored as **separate entries** in `pending_ckd_requests`.

`respond_ckd` gates on a single attested participant check, then branches on the key variant: [2](#0-1) 

- `AppPublicKeyPV` arm: calls `ckd_output_check`, a pairing-equation check that cryptographically binds the response to the network public key and the app key — a single Byzantine node **cannot** forge a passing value.
- `AppPublicKey` arm: **empty body** — no verification whatsoever. Any `CKDResponse` is accepted unconditionally.

`assert_caller_is_attested_participant_and_protocol_active` requires exactly **one** attested participant; there is no threshold count: [3](#0-2) 

`resolve_yields_for` is a first-writer-wins drain: the first `respond_ckd` call removes the map entry and resumes all queued yields; subsequent calls return `RequestNotFound`: [4](#0-3) 

**Attack path (two-request variant as described):**

1. Victim calls `request_app_private_key({AppPublicKey(pk_A), path, domain})` → entry stored under key `K_A`.
2. Victim calls `request_app_private_key({AppPublicKey(pk_B), path, domain})` → entry stored under separate key `K_B` (different `app_public_key` → different map key).
3. Attacker (single attested participant) front-runs honest nodes and calls `respond_ckd(K_A, fabricated_response)`.
4. Contract accepts: no check on `AppPublicKey` arm, `resolve_yields_for` drains `K_A` with attacker-controlled bytes.
5. Honest nodes later call `respond_ckd(K_B, correct_response)` — succeeds for `K_B` unaffected.
6. Victim's TEE app for `pk_A` receives the attacker-chosen secret.

The two-request framing is not required; the simpler single-request variant works identically: any pending `AppPublicKey` CKD request can be resolved by one Byzantine participant with arbitrary output.

---

### Impact Explanation

The victim's TEE application receives an attacker-chosen BLS12-381 scalar as its "derived confidential key." Any data encrypted under the honest derived key becomes undecryptable; any data the attacker later presents encrypted under the injected key can be decrypted by the attacker. This is unauthorized confidential key derivation output delivered to the victim — matching the Critical impact tier.

---

### Likelihood Explanation

The attacker must be a registered, attested MPC participant. TEE attestation is required to pass `is_caller_an_attested_participant`: [5](#0-4) 

However, the contract does **not** verify that the `respond_ckd` transaction was produced by the TEE software itself — it only checks that the signing key belongs to an attested node. A participant who has legitimately passed attestation can call `respond_ckd` directly (outside the TEE process) with fabricated data. No TEE compromise is required; Byzantine behavior by one participant suffices. With `n` participants and threshold `t > 1`, any single participant can mount this attack unilaterally.

---

### Recommendation

Apply the same `ckd_output_check` pairing verification to the `AppPublicKey` arm, or require threshold-many independent `respond_ckd` calls before resolving the yield. The `AppPublicKeyPV` path already demonstrates the correct pattern: [6](#0-5) 

If `AppPublicKey` is intentionally unverifiable (e.g., for use cases where the derived key is intentionally public), document this explicitly and warn callers that a single Byzantine participant can inject arbitrary output. Consider deprecating the variant in favor of `AppPublicKeyPV`.

---

### Proof of Concept

```rust
// 1. Victim submits AppPublicKey CKD request
let pk_a = CKDAppPublicKey::AppPublicKey(Bls12381G1PublicKey([0xAA; 48]));
contract.request_app_private_key(CKDRequestArgs {
    derivation_path: "secret/path".to_string(),
    app_public_key: pk_a.clone(),
    domain_id: DomainId::default(),
});

let ckd_request_a = CKDRequest::new(pk_a, DomainId::default(), &victim_id, "secret/path");
assert!(contract.get_pending_ckd_request(&ckd_request_a).is_some());

// 2. Attacker (single attested participant) injects fabricated response
with_active_participant_and_attested_context(&contract);
let fabricated = CKDResponse {
    big_y: Bls12381G1PublicKey([0xDE; 48]),  // attacker-chosen
    big_c: Bls12381G1PublicKey([0xAD; 48]),  // attacker-chosen
};
// No AppPublicKey verification → accepted unconditionally
contract.respond_ckd(ckd_request_a, fabricated).expect("injected");

// 3. Yield resolved with attacker-controlled bytes; victim TEE receives wrong key
```

The `AppPublicKey` arm at line 676 is an empty no-op, so the fabricated response passes straight through to `resolve_yields_for` and is delivered to the victim's yield callback. [7](#0-6)

pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
