### Title
Missing Reference Script Fee Accounting for Subtransactions in Dijkstra Era — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

The Dijkstra era introduces subtransactions (`subTransactionsTxBodyL`) embedded inside a top-level transaction. The minimum fee calculation for a Dijkstra transaction uses `getConwayMinFeeTxUtxo`, which only measures reference script sizes from the **top-level** transaction's inputs. Reference scripts referenced exclusively through subtransaction inputs are entirely excluded from the fee calculation. An unprivileged submitter can craft a transaction with negligible top-level reference scripts but arbitrarily large reference scripts in subtransactions, paying a fee far below the true validation cost imposed on every node.

---

### Finding Description

The Dijkstra era `EraUTxO` instance delegates `getMinFeeTxUtxo` directly to the Conway implementation:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs, line 141
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
``` [1](#0-0) 

`getConwayMinFeeTxUtxo` computes the reference script size by calling `txNonDistinctRefScriptsSize`, which only inspects the top-level transaction's `inputsTxBodyL` and `referenceInputsTxBodyL`:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs, lines 174-187
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx

txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [2](#0-1) 

The Dijkstra codebase already contains the correct aggregate function `batchNonDistinctRefScriptsSize`, which sums reference script sizes across the top-level transaction **and all subtransactions**:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs, lines 264-277
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
``` [3](#0-2) 

This function is **never called** from `getMinFeeTxUtxo`. The `feesOK` predicate in the UTXO rule calls `getMinFeeTxUtxo` to enforce the minimum fee, so the ledger accepts Dijkstra transactions whose subtransactions carry arbitrarily large reference scripts without charging for them.

The analogous gap was already exploited in the Conway era (June 2024 DDoS), documented in the ADR:

> "Overhead of using reference scripts was not properly accounted for when this feature was introduced in the Babbage era, which is now fixed in the Conway era." [4](#0-3) 

The Dijkstra era repeats the same class of omission for its new subtransaction dimension.

---

### Impact Explanation

**Medium — Attacker-controlled transactions modify fees outside design parameters.**

Every node must deserialize and validate all reference scripts in every subtransaction. The tiered `tierRefScriptFee` pricing was specifically designed to make large-script batches expensive. By placing large reference scripts only in subtransactions, an attacker pays only the base size-linear fee while imposing the full deserialization cost on the network. This is a fee undercharge that falls outside the intended design parameters of the reference script pricing mechanism. [5](#0-4) 

---

### Likelihood Explanation

**High.** Any unprivileged transaction submitter can construct a valid Dijkstra top-level transaction with one or more subtransactions whose `referenceInputsTxBodyL` or `inputsTxBodyL` point to UTxO entries containing large Plutus scripts. No special role, key, or governance action is required. The entry path is a single serialized transaction submitted to the mempool.

---

### Recommendation

Override `getMinFeeTxUtxo` in the `DijkstraEra` `EraUTxO` instance with a Dijkstra-specific implementation that uses `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs
getDijkstraMinFeeTxUtxo ::
  ( EraTx era
  , DijkstraEraTxBody era
  , BabbageEraTxBody era
  ) =>
  PParams era -> Tx TopTx era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx

-- In the EraUTxO DijkstraEra instance:
getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo   -- replaces getConwayMinFeeTxUtxo
``` [6](#0-5) 

---

### Proof of Concept

1. Deploy a UTxO entry containing a large Plutus script (e.g., 25 KiB) as a reference script.
2. Construct a Dijkstra top-level transaction with:
   - No reference inputs at the top level (so `txNonDistinctRefScriptsSize` returns 0).
   - One subtransaction whose `referenceInputsTxBodyL` includes the UTxO from step 1.
3. Compute the minimum fee using `getMinFeeTxUtxo` (which calls `getConwayMinFeeTxUtxo` → `txNonDistinctRefScriptsSize`). The reference script fee component is **zero**.
4. Submit the transaction. The `feesOK` check passes because the enforced minimum fee does not include the subtransaction's reference script overhead.
5. Every validating node must still deserialize the 25 KiB script from the subtransaction, paying the full CPU/memory cost, while the submitter paid only the base byte-linear fee. [7](#0-6) [3](#0-2)

getConwayMinFeeTxUtxo ::
  ( EraTx era
  , BabbageEraTxBody era
  ) =>
  PParams era ->
  Tx l era ->
  UTxO era ->
  Coin
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx

-- | Calculate the total size of reference scripts used by the transactions. Duplicate
-- scripts will be counted as many times as they occur, since there is never a reason to
-- include an input with the same reference script.
--
-- Any input that appears in both regular inputs and reference inputs of a transaction is
-- only used once in this computation.
txNonDistinctRefScriptsSize :: (EraTx era, BabbageEraTxBody era) => UTxO era -> Tx l era -> Int
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
