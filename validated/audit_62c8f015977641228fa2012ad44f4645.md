### Title
Attacker Can Exhaust the Daily Mint Limit to Temporarily Block All Deposits - (File: contracts/pools/RSETHPoolV3.sol)

---

### Summary

The `limitDailyMint` modifier in `RSETHPoolV3.sol`, `RSETHPoolV3ExternalBridge.sol`, and `RSETHPoolV3WithNativeChainBridge.sol` tracks `dailyMintAmount` as a monotonically increasing counter that is never decremented within a day. An unprivileged depositor with sufficient capital can exhaust the entire `dailyMintLimit` in a single transaction, blocking all other users from depositing for up to 24 hours. Because the attacker receives `wrsETH` in return for their ETH, their capital is not lost — only gas fees are incurred.

---

### Finding Description

The `limitDailyMint` modifier enforces a per-day cap on the total rsETH minted:

```solidity
// contracts/pools/RSETHPoolV3.sol lines 119–124
if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
    revert DailyMintLimitExceeded();
}
dailyMintAmount += rsETHAmount;
```

`dailyMintAmount` is incremented on every successful deposit and is only reset when `currentDay > lastMintDay` (i.e., after a full 24-hour period elapses). There is no corresponding decrement path available to unprivileged users — `swapAssetToPremintedRsETH` (the reverse-swap) is gated behind `OPERATOR_ROLE`.

An attacker can call `deposit()` with an ETH amount large enough to push `dailyMintAmount` to `dailyMintLimit` in a single transaction. After that, every subsequent `deposit()` call from any user reverts with `DailyMintLimitExceeded` until the next day boundary. The attacker receives `wrsETH` — a liquid token redeemable for the underlying ETH value — so no capital is destroyed; only gas is spent.

The same root cause exists identically in:
- `contracts/pools/RSETHPoolV3ExternalBridge.sol` (lines 130–158)
- `contracts/pools/RSETHPoolV3WithNativeChainBridge.sol` (lines 108–137)

The `setDailyMintLimit` admin function can update the limit, but this requires the `DEFAULT_ADMIN_ROLE` to react, and the attack can be re-executed at the start of every new day.

---

### Impact Explanation

**Medium. Temporary freezing of funds.**

All users are blocked from depositing ETH or supported LSTs into any affected pool for up to 24 hours per attack cycle. The deposit path is the sole entry point for minting `wrsETH`/`rsETH` on L2. A sustained attacker repeating the attack each day can indefinitely deny the deposit service, preventing users from entering the protocol and locking out time-sensitive depositors (e.g., those responding to yield opportunities).

---

### Likelihood Explanation

**Medium.**

The attacker must hold capital equal to the `dailyMintLimit` in ETH or a supported LST. However:
- The attacker receives `wrsETH` in return (fair-value liquid token), so no capital is destroyed.
- The only real cost is L2 gas, which is negligible on chains like Optimism, Base, or Arbitrum.
- The attack is repeatable every 24 hours with no additional capital requirement (the attacker simply holds `wrsETH` and re-deposits the next day if they can redeem it, or uses fresh capital).
- If `feeBps == 0` (which is a valid configuration since `setFeeBps` permits 0), the attack is entirely free except for gas.

---

### Recommendation

1. **Emergency reset**: Add a function callable by `PAUSER_ROLE` or `DEFAULT_ADMIN_ROLE` (without timelock) to reset `dailyMintAmount` to zero, allowing rapid recovery from a DOS attempt.
2. **Shorter time windows**: Consider using a rolling window (e.g., per-hour or per-block) rather than a fixed 24-hour period to reduce the maximum DOS duration.
3. **Minimum deposit fee**: Enforce a non-zero minimum `feeBps` to impose a real capital cost on exhausting the daily limit.
4. **Decrement on reverse-swap**: If a public reverse-swap path is ever added, decrement `dailyMintAmount` proportionally to disincentivize cycling.

---

### Proof of Concept

**Setup**: `dailyMintLimit = 1000 ether` (in rsETH terms), `feeBps = 0`, pool is live on an L2.

**Steps**:

1. Attacker calls `deposit{value: X}("")` where `X` is chosen so that `viewSwapRsETHAmountAndFee(X).rsETHAmount == dailyMintLimit`.
2. Inside `limitDailyMint`: `dailyMintAmount += 1000e18` → `dailyMintAmount == dailyMintLimit`.
3. Attacker receives `1000e18` `wrsETH` tokens (full fair-value equivalent of their ETH).
4. Any subsequent call by any user to `deposit()` hits:
   ```solidity
   if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
       revert DailyMintLimitExceeded();  // always true for any rsETHAmount > 0
   }
   ```
5. All deposits are blocked for up to 24 hours.
6. Next day: attacker repeats with the same `wrsETH` (if redeemable) or fresh ETH. Cost per day: gas only.

**File:** contracts/pools/RSETHPoolV3.sol (L96-125)
```text
    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```

**File:** contracts/pools/RSETHPoolV3.sol (L246-265)
```text
    function deposit(string memory referralId)
        external
        payable
        nonReentrant
        whenNotPaused
        limitDailyMint(msg.value, ETH_IDENTIFIER)
    {
        if (!isEthDepositEnabled) revert EthDepositDisabled();
        uint256 amount = msg.value;

        if (amount == 0) revert InvalidAmount();

        (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);

        feeEarnedInETH += fee;

        wrsETH.mint(msg.sender, rsETHAmount);

        emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
    }
```
