### Title
Hardcoded `gasLimit = 0` in `ScrollMessenger` May Cause ETH to Be Permanently Stuck in Bridge - (`contracts/bridges/ScrollMessenger.sol`)

---

### Summary

`ScrollMessenger.sol` hardcodes the `gasLimit` parameter to `0` when calling Scroll's native bridge `sendMessage`. This is the direct analog of the `dstGasForCall = 0` pattern in the external report: a hardcoded zero gas parameter passed to a cross-chain bridge causes the destination-side execution to fail, leaving funds stuck.

---

### Finding Description

`ScrollMessenger.sendETHToL1ViaBridge` is the Scroll-specific implementation of the `IL2Messenger` interface used by all Scroll-deployed pool contracts (`RSETHPool`, `RSETHPoolNoWrapper`, `RSETHPoolV3ExternalBridge`) to bridge ETH from Scroll L2 to Ethereum L1.

The call is:

```solidity
IScrollMessenger(l2bridge).sendMessage{ value: value }(target, value, "", 0, msg.sender);
//                                                                        ^
//                                                              gasLimit hardcoded to 0
```

The developer comment reads: *"Gas limit is set to 0 to use the default gas limit"* — but Scroll's bridge does **not** fall back to a default when `gasLimit = 0`. The `gasLimit` field in Scroll's `sendMessage` is the gas budget allocated for executing the message on L1. When set to 0, the L1 relay transaction will revert with out-of-gas, and the ETH will remain locked in the Scroll bridge with no automatic recovery path. [1](#0-0) 

The affected pool functions that invoke this path: [2](#0-1) [3](#0-2) [4](#0-3) 

---

### Impact Explanation

When `BRIDGER_ROLE` calls `bridgeAssetsViaNativeBridge()` on any Scroll-deployed pool:

1. ETH is deducted from the pool and sent to Scroll's L2 messenger (`sendMessage` with `gasLimit = 0`).
2. The message is stored in the L2 messenger and ETH is burned/locked on L2.
3. When a relayer attempts to execute the message on L1 via `relayMessageWithProof`, the execution fails because `gasLimit = 0` provides no gas budget for the L1 call (even a bare ETH transfer to `l1VaultETHForL2Chain` requires ≥ 21,000 gas).
4. ETH is stuck in the Scroll bridge. The `ScrollMessenger` provides no retry mechanism with a corrected gas limit.

**Impact**: Temporary (potentially permanent without manual intervention) freezing of user-deposited ETH that was pooled for bridging. Matches the **Temporary freezing of funds** impact class.

---

### Likelihood Explanation

`bridgeAssetsViaNativeBridge()` is a routine operational call made by the `BRIDGER_ROLE` to move accumulated ETH from Scroll L2 to L1. It is called in the normal course of protocol operations, not in edge cases. Every invocation on Scroll will trigger this code path with `gasLimit = 0`.

---

### Recommendation

Replace the hardcoded `0` with a configurable per-chain gas limit, following the same pattern recommended in the external report:

```diff
contract ScrollMessenger is IL2Messenger, ReentrancyGuard {
+   uint32 public constant DEFAULT_GAS_LIMIT = 168_000; // Scroll L1 execution gas

    function sendETHToL1ViaBridge(address l2bridge, address target, uint256 value) external payable nonReentrant {
        if (msg.value != value) revert MismatchedMsgValue();
-       IScrollMessenger(l2bridge).sendMessage{ value: value }(target, value, "", 0, msg.sender);
+       IScrollMessenger(l2bridge).sendMessage{ value: value }(target, value, "", DEFAULT_GAS_LIMIT, msg.sender);
    }
}
```

The gas limit should be calibrated to cover the L1 `receive()` execution of `l1VaultETHForL2Chain` and should be configurable by an admin to accommodate future changes.

---

### Proof of Concept

1. Pool accumulates ETH on Scroll L2 from user deposits.
2. `BRIDGER_ROLE` calls `RSETHPool.bridgeAssetsViaNativeBridge()`.
3. `ScrollMessenger.sendETHToL1ViaBridge()` is called, which calls `IScrollMessenger(l2bridge).sendMessage{value: ethBalance}(l1VaultETHForL2Chain, ethBalance, "", 0, msg.sender)`.
4. ETH is burned on L2; message stored in Scroll L2 messenger with `gasLimit = 0`.
5. Relayer calls `L1ScrollMessenger.relayMessageWithProof(...)` on Ethereum — execution reverts with out-of-gas because `gasLimit = 0`.
6. ETH is stuck; `l1VaultETHForL2Chain` never receives the funds; rsETH minting from this ETH is blocked. [5](#0-4)

```text
contract ScrollMessenger is IL2Messenger, ReentrancyGuard {
    /**
     * @notice Bridge ETH from Scroll L2 to Ethereum Mainnet
     * @param l2bridge The address of the L2 bridge on Scroll
     * @param target The address of the target contract on L1
     * @param value The amount of ETH to send
     * @dev Gas limit is set to 0 to use the default gas limit
     */
    function sendETHToL1ViaBridge(address l2bridge, address target, uint256 value) external payable nonReentrant {
        if (msg.value != value) revert MismatchedMsgValue();
        IScrollMessenger(l2bridge).sendMessage{ value: value }(target, value, "", 0, msg.sender);
    }
```
