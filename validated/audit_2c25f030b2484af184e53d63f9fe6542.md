Audit Report

## Title
Chainlink Price Feed Not Validated for Staleness in `ChainlinkPriceOracle.getAssetPrice` - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `priceFeed.latestRoundData()` but discards all staleness-related return values (`roundId`, `updatedAt`, `answeredInRound`), accepting any price — including stale ones — as valid. A stale inflated price flows directly into rsETH minting calculations, allowing a depositor to receive more rsETH than their deposit is worth, diluting existing holders' unclaimed yield. A stale deflated price can trigger the permissionless `updateRSETHPrice()` to pause the protocol, temporarily freezing all user funds.

## Finding Description

In `contracts/oracles/ChainlinkPriceOracle.sol` at line 52, `getAssetPrice()` destructures only the `price` field from `latestRoundData()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

No check is made on `answeredInRound >= roundId` (round completeness) or `updatedAt > 0` (round not incomplete) or `price > 0` (valid price). This contrasts directly with `ChainlinkOracleForRSETHPoolCollateral.getRate()`, which performs all three checks: [2](#0-1) 

The stale price propagates through the following confirmed call chain:

1. `LRTOracle.getAssetPrice(asset)` delegates to `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)` — which resolves to `ChainlinkPriceOracle.getAssetPrice()`. [3](#0-2) 

2. `LRTDepositPool.getRsETHAmountToMint()` uses `lrtOracle.getAssetPrice(asset)` live at deposit time: [4](#0-3) 

3. `LRTOracle._getTotalEthInProtocol()` uses `getAssetPrice(asset)` for every supported LST to compute total protocol TVL: [5](#0-4) 

4. `_updateRsETHPrice()` uses that TVL to compute `rsETHPrice` and triggers the downside-protection pause if the computed price drops too far: [6](#0-5) 

5. `updateRSETHPrice()` is public and permissionless (only `whenNotPaused` guard): [7](#0-6) 

## Impact Explanation

**Primary impact — Theft of unclaimed yield (High):**

`getRsETHAmountToMint` computes `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`. If the Chainlink feed is stale at a price higher than the current actual price (e.g., stETH/ETH stale at 1.05 while actual is 1.02), a depositor receives more rsETH than their deposit is worth. The excess minted rsETH dilutes all existing rsETH holders' proportional claim on protocol TVL — a direct theft of unclaimed yield from existing holders.

**Secondary impact — Temporary freezing of funds (Medium):**

`updateRSETHPrice()` is permissionless. If a stale low price causes `newRsETHPrice` to fall below `highestRsethPrice` by more than `pricePercentageLimit`, both `lrtDepositPool` and `withdrawalManager` are paused, freezing all user deposits and withdrawals until an admin manually unpauses. [8](#0-7) 

## Likelihood Explanation

Chainlink LST/ETH feeds (stETH/ETH, rETH/ETH, etc.) use heartbeat-based updates with intervals up to 24 hours. During any such window where the feed has not updated, the last reported price remains valid from the contract's perspective. No special privileges are required — `depositAsset` and `depositETH` are open to any user. An attacker can monitor on-chain feed timestamps and act opportunistically whenever `block.timestamp - updatedAt` exceeds the heartbeat threshold. The condition is observable on-chain and repeatable.

## Recommendation

Apply the same staleness validation already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    require(answeredInRound >= roundId, "Stale price");
    require(updatedAt > 0, "Incomplete round");
    require(price > 0, "Invalid price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept

1. The Chainlink stETH/ETH feed goes stale; its last reported price is 1.05e18 (actual current rate: 1.02e18). On-chain, `updatedAt` is more than 24 hours old and `answeredInRound < roundId`.
2. Alice calls `LRTDepositPool.depositAsset(stETH, 100e18, minRSETH, "")`.
3. `_beforeDeposit` calls `getRsETHAmountToMint(stETH, 100e18)`.
4. `getRsETHAmountToMint` calls `lrtOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns stale `1.05e18` with no staleness revert.
5. `rsethAmountToMint = (100e18 * 1.05e18) / rsETHPrice` — Alice receives ~2.9% more rsETH than her deposit is worth.
6. Existing rsETH holders are diluted by the excess minted rsETH.

**Foundry fork test plan:**
- Fork mainnet at a block where a Chainlink LST/ETH feed's `updatedAt` is stale (or mock `latestRoundData` to return a stale round where `answeredInRound < roundId`).
- Call `depositAsset` with the stale price active.
- Assert that `rsethAmountToMint` exceeds the fair value computed at the correct current price.
- Assert that no revert occurs in `ChainlinkPriceOracle.getAssetPrice()`.
