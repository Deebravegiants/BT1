Audit Report

## Title
Aave Interest Accrued on Users' Pending ETH Withdrawals Is Systematically Diverted to the Treasury - (File: contracts/LRTWithdrawalManager.sol)

## Summary
When `unlockQueue` is called, users' rsETH is burned and the redeemed ETH is deposited into Aave v3. During the withdrawal delay window (up to 16 days), Aave accrues interest on this ETH. `_withdrawFromAave` explicitly caps withdrawals at `totalETHDepositedToAave` (the deposited principal), preventing any accrued interest from reaching users. That interest is later swept to the protocol treasury via `_collectInterestToTreasury`. Every user who withdraws ETH while Aave integration is enabled loses the yield earned on their own ETH during the delay.

## Finding Description
**Stage 1 – `initiateWithdrawal` (L150–178):** The user's rsETH is transferred to the contract and `expectedAssetAmount` is fixed at the current oracle price.

**Stage 2 – `unlockQueue` (L299–319):** rsETH is burned (`burnFrom`, L305), ETH is redeemed from `LRTUnstakingVault` (L307), and the full `assetAmountUnlocked` is deposited into Aave (L310–317), incrementing `totalETHDepositedToAave` (L898).

**Stage 3 – `completeWithdrawal` / `_processWithdrawalCompletion` (L719–734):** When the user claims, `_withdrawFromAave(amountNeeded)` is called. Inside `_withdrawFromAave` (L905–921), the withdrawal is hard-capped:

```solidity
// Only withdraw up to the principal amount (don't use accrued interest for user withdrawals)
uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave
    ? aaveBalance : totalETHDepositedToAave;
withdrawnAmount = amount > withdrawablePrincipal ? withdrawablePrincipal : amount;
```

The user receives exactly `request.expectedAssetAmount` (L734). Any `aaveAWETH.balanceOf(address(this))` above `totalETHDepositedToAave` is never accessible to users.

**Treasury sweep – `_collectInterestToTreasury` (L945–961):** The surplus (`aaveBalance - totalETHDepositedToAave`) is withdrawn from Aave and sent to the treasury address. This function is called on every `collectInterestToTreasury` public call and also on `setAaveIntegrationEnabled(false)`.

The root cause is the deliberate accounting separation: `totalETHDepositedToAave` tracks only principal, and `_withdrawFromAave` enforces that users can never receive more than principal, while the surplus is routed to the treasury.

## Impact Explanation
**High — Theft of unclaimed yield.**

At the moment `unlockQueue` burns the user's rsETH and redeems ETH from `LRTUnstakingVault`, the ETH is unambiguously earmarked for the user. Any yield earned on that ETH during the delay period is yield the user would have received had the ETH been returned immediately. Instead, it is systematically extracted and sent to the treasury. At Aave WETH supply rates of 1–5% APY over an 8-day delay, each user loses approximately 0.02–0.11% of their withdrawal amount. Across all withdrawals this is a continuous, compounding extraction of user yield.

## Likelihood Explanation
The Aave integration is enabled by the LRT manager via `setAaveIntegrationEnabled(true)` — a routine operational action, not an attack. Once enabled, the yield diversion requires no further privileged action; it occurs automatically for every ETH withdrawal processed through `unlockQueue`. No special attacker capability is needed. The affected path (`completeWithdrawal`) is a standard public user function.

## Recommendation
Track each user's deposited principal individually (e.g., `mapping(bytes32 requestId => uint256) aaveDeposited`) at `unlockQueue` time, and return the user's pro-rata share of accrued interest at `completeWithdrawal` time. Alternatively, withdraw `expectedAssetAmount` from Aave without capping at `totalETHDepositedToAave`, allowing users to receive their proportional yield. If the protocol intends to retain Aave yield as a protocol fee, this must be explicitly disclosed to users and bounded as a named fee parameter.

## Proof of Concept
1. Alice calls `initiateWithdrawal(ETH, 10e18)`. `expectedAssetAmount = 10 ETH` is stored; rsETH is held by the contract.
2. Operator calls `unlockQueue(ETH, ...)`. `burnFrom(address(this), rsETH)` executes (L305); `unstakingVault.redeem(ETH, 10e18)` executes (L307); `depositToAaveExternal(10e18)` executes (L311), setting `totalETHDepositedToAave = 10e18`.
3. After 8 days, Aave has accrued 0.002 ETH interest. `aaveAWETH.balanceOf(address(this)) = 10.002e18`.
4. Alice calls `completeWithdrawal(ETH)`. `_processWithdrawalCompletion` calls `_withdrawFromAave(10e18)`. Inside: `withdrawablePrincipal = min(10.002e18, 10e18) = 10e18`; `withdrawnAmount = 10e18`. Alice receives exactly 10 ETH (L734).
5. The 0.002 ETH remains as aWETH in the contract.
6. Operator calls `collectInterestToTreasury()`. `_collectInterestToTreasury` computes `interestAmount = 10.002e18 - 10e18 = 0.002e18`, withdraws it from Aave, and sends it to the treasury (L954–957).
