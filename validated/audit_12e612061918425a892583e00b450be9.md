Audit Report

## Title
Missing Chainlink `latestRoundData()` Validity Checks in `ChainlinkPriceOracle.getAssetPrice()` Allows Stale Prices to Corrupt rsETH Exchange Rate - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all validity fields, performing no staleness, incomplete-round, or non-positive price checks. The same repository already implements all three checks in `ChainlinkOracleForRSETHPoolCollateral.getRate()`, making the omission a concrete inconsistency. Because `updateRSETHPrice()` is a public function, an attacker can trigger a price update at the exact moment a Chainlink feed is stale, permanently writing a corrupted `rsETHPrice` to storage and causing incorrect rsETH minting for depositors and incorrect payout calculations for withdrawers.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` fetches the price as:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52-54
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values of `latestRoundData()` are available but only `answer` is used. No check is made for:
- `answeredInRound < roundId` (stale round)
- `updatedAt == 0` (incomplete round)
- `price <= 0` (invalid/negative price)

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` in the same repository performs all three checks:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L27-32
(uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
    AggregatorV3Interface(oracle).latestRoundData();
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The exploit path is fully reachable by an unprivileged caller:

1. `updateRSETHPrice()` is public with no access control beyond `whenNotPaused` (L87-89).
2. It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(asset)` → `ChainlinkPriceOracle.getAssetPrice()`, which returns the stale price without reverting.
3. `totalETHInProtocol` is computed using the stale price (L231), and `newRsETHPrice` is written to storage (L250, L313).

The `pricePercentageLimit` guard (L252-266) provides partial mitigation only when: (a) `pricePercentageLimit != 0` (it defaults to 0), and (b) the stale price deviation exceeds the configured threshold. A stale price within the threshold passes through silently. Additionally, `LRTWithdrawalManager._createUnlockParams()` reads `lrtOracle.getAssetPrice(asset)` directly (L848), bypassing `_updateRsETHPrice()` entirely and thus bypassing even the partial price-limit guard.

## Impact Explanation
**Theft of unclaimed yield (High):** A stale-low asset price causes `totalETHInProtocol` to be underestimated, setting `rsETHPrice` artificially low. Subsequent depositors receive more rsETH than they are entitled to, diluting existing rsETH holders and constituting theft of their accrued yield. A stale-high price causes depositors to receive fewer rsETH tokens than owed. A zero price (Chainlink circuit-breaker) causes `uint256(0)` to propagate, setting `rsETHPrice` to near-zero and enabling unbounded rsETH minting. The withdrawal path is also directly affected: `_calculatePayoutAmount` uses `assetPrice` from `_createUnlockParams()` (which reads the same vulnerable oracle) to determine how much asset a withdrawer receives, causing over- or under-payment.

## Likelihood Explanation
Chainlink feeds are documented to return stale data during network congestion, sequencer downtime (on L2), or when a feed's heartbeat has not been met. `updateRSETHPrice()` is a public function requiring no privileged access, so any external caller — including a bot — can invoke it at the precise moment a feed is stale. This is a realistic, externally triggerable condition. The attacker needs no capital, no flash loan, and no privileged role.

## Recommendation
Apply the same validity checks already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, consider adding a per-feed configurable heartbeat check (`block.timestamp - updatedAt > heartbeat`) to guard against feeds that are technically in a valid round but have not been updated within their expected interval.

## Proof of Concept
**Foundry fork test outline:**

1. Fork mainnet at a block where a supported LST/ETH Chainlink feed has `answeredInRound < roundId` (or mock `latestRoundData()` to return such a state).
2. Call `LRTOracle.updateRSETHPrice()` as an unprivileged EOA.
3. Assert that the call succeeds (no revert) and that `rsETHPrice` is written with the stale-derived value.
4. Compute the expected `rsETHPrice` using the correct (non-stale) price and assert the two values differ.
5. Call `depositAsset()` as a depositor and assert the minted rsETH amount differs from the correct amount, demonstrating dilution of existing holders.
6. Call `_createUnlockParams` path via `unlockQueue()` and assert `_calculatePayoutAmount` returns an incorrect asset amount.

**Minimal call sequence (no fork):**
1. Deploy `ChainlinkPriceOracle` with a mock `AggregatorV3Interface` that returns `(roundId=2, price=X, startedAt=T, updatedAt=T, answeredInRound=1)` (stale: `answeredInRound < roundId`).
2. Call `getAssetPrice(asset)` — observe it returns `uint256(X) * 1e18 / decimals` with no revert.
3. Contrast with `ChainlinkOracleForRSETHPoolCollateral.getRate()` using the same mock — observe it reverts with `StalePrice()`.
