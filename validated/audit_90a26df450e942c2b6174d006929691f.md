Audit Report

## Title
Zero `rsETHAmount` Minted on Dust Deposits Due to Integer Division Truncation - (File: contracts/pools/RSETHPoolV3ExternalBridge.sol)

## Summary
The `deposit()` functions in `RSETHPoolV3ExternalBridge.sol`, `RSETHPoolV3.sol`, `RSETHPoolV3WithNativeChainBridge.sol`, and `RSETHPoolNoWrapper.sol` guard only against `amount == 0` but not against `rsETHAmount == 0`. Because `viewSwapRsETHAmountAndFee` uses integer division (`amountAfterFee * 1e18 / rsETHToETHrate`), any dust deposit where `amountAfterFee * 1e18 < rsETHToETHrate` truncates to `rsETHAmount = 0`. The depositor's ETH is absorbed into the pool while `wrsETH.mint(msg.sender, 0)` (or `rsETH.safeTransfer(msg.sender, 0)`) executes silently, leaving the depositor with no receipt tokens and no recourse.

## Finding Description
In `RSETHPoolV3ExternalBridge.sol` the ETH deposit path is:

```solidity
// lines 373‚Äì383
uint256 amount = msg.value;
if (amount == 0) revert InvalidAmount();          // guards amount, not rsETHAmount
(uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
feeEarnedInETH += fee;
wrsETH.mint(msg.sender, rsETHAmount);             // rsETHAmount can be 0
```

`viewSwapRsETHAmountAndFee` (lines 418‚Äì427) computes:

```solidity
fee = amount * feeBps / 10_000;
uint256 amountAfterFee = amount - fee;
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;   // integer division
```

With `rsETHToETHrate = 1.05e18` (realistic post-yield value) and `amount = 1 wei`:
- `fee = 0`, `amountAfterFee = 1`
- `rsETHAmount = 1 * 1e18 / 1.05e18 = 0` (truncated)

Standard OpenZeppelin ERC20 `_mint` does not revert when `amount == 0`; it emits a zero-value `Transfer` event and returns. The depositor's 1 wei ETH is credited to the pool's bridgeable balance (`feeEarnedInETH` is unchanged; the raw ETH balance increases), permanently inaccessible to the depositor.

The identical pattern exists in:
- `RSETHPoolV3.sol` lines 254‚Äì264, 299‚Äì308
- `RSETHPoolV3WithNativeChainBridge.sol` lines 289‚Äì300, 335‚Äì344
- `RSETHPoolNoWrapper.sol` lines 231‚Äì244, 277‚Äì286 (uses `rsETH.safeTransfer(msg.sender, 0)` which also succeeds silently)

The token deposit path in `RSETHPoolV3ExternalBridge.sol` lines 442‚Äì452 has the same issue: `rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate` truncates to 0 when `tokenToETHRate < rsETHToETHrate` and `amountAfterFee` is small.

## Impact Explanation
A depositor who sends a dust ETH amount (e.g., 1 wei) receives 0 wrsETH/rsETH. The deposited ETH is retained by the pool and eventually bridged to L1, but the depositor holds no claim on it. This constitutes a permanent loss of the deposited amount for the user. **Impact: Low** ‚Äî the contract fails to deliver the promised receipt tokens for a valid non-zero deposit, and the deposited value is unrecoverable by the depositor.

## Likelihood Explanation
Any unprivileged external caller can trigger this by calling `deposit{value: 1}("")` on any of the affected contracts while the pool is unpaused. No special role, precondition, or economic setup is required beyond the rsETH rate exceeding 1e18 (which is the normal post-yield state). The condition is deterministic and reproducible on every affected L2 pool variant. **Likelihood: Low** ‚Äî the monetary loss per call is negligible (1 wei), but the code path is unconditionally reachable.

## Recommendation
Add a post-computation zero-check on `rsETHAmount` in every `deposit()` function, immediately after the `viewSwapRsETHAmountAndFee` call:

```solidity
(uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
if (rsETHAmount == 0) revert InvalidAmount();
```

Apply this to both the ETH and token deposit paths in `RSETHPoolV3ExternalBridge.sol`, `RSETHPoolV3.sol`, `RSETHPoolV3WithNativeChainBridge.sol`, `RSETHPoolNoWrapper.sol`, `RSETHPoolV2ExternalBridge.sol`, and `RSETHPoolV2NBA.sol`.

## Proof of Concept
1. Deploy `RSETHPoolV3ExternalBridge` on a local fork where `getRate()` returns `1.05e18` (realistic post-yield rsETH/ETH rate).
2. Call `deposit{value: 1}("")` from an unprivileged EOA ‚Äî `amount = 1`, passes the `amount == 0` guard.
3. `viewSwapRsETHAmountAndFee(1)` computes: `fee = 0`, `amountAfterFee = 1`, `rsETHAmount = 1 * 1e18 / 1.05e18 = 0`.
4. `wrsETH.mint(msg.sender, 0)` executes without revert; depositor's wrsETH balance is unchanged.
5. Pool's ETH balance increases by 1 wei; `getETHBalanceMinusFees()` increases by 1; depositor has no receipt token and no withdrawal path.

Foundry fuzz test sketch:
```solidity
function testFuzz_dustDepositZeroMint(uint256 dustAmount) public {
    dustAmount = bound(dustAmount, 1, getRate() / 1e18 - 1); // amounts that truncate to 0
    uint256 balBefore = wrsETH.balanceOf(address(this));
    pool.deposit{value: dustAmount}("");
    assertEq(wrsETH.balanceOf(address(this)), balBefore); // receives 0 wrsETH
}
```
