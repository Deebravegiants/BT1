### Title
Unvalidated `ProposalInit.builder` Allows Any Proposer to Redirect All Block Fees to an Arbitrary Sequencer Address — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates every security-critical field of `ProposalInit` (height, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, `fee_proposal_fri`) but never checks the `builder` field. That field is forwarded verbatim as `sequencer_address` into every transaction executed in the block. Because the Starknet protocol routes all transaction fees to `sequencer_address`, any validator who wins a proposal round can silently redirect the entire block's fee revenue to an address they control, and every honest validator will accept the proposal because the `ProposalCommitment` still matches.

### Finding Description

**Root cause — missing field check in `is_proposal_init_valid`**

`is_proposal_init_valid` in `crates/apollo_consensus_orchestrator/src/validate_proposal.rs` (lines 252–418) checks eight fields of `ProposalInit` but has no branch for `builder`:

```
timestamp ✓  starknet_version ✓  version_constant_commitment ✓
height ✓  l1_da_mode ✓  l2_gas_price_fri ✓
l1_gas_price_{fri,wei} ✓  l1_data_gas_price_{fri,wei} ✓
fee_proposal_fri ✓
builder  ✗  (never read)
``` [1](#0-0) 

**How `builder` becomes `sequencer_address`**

Immediately after `is_proposal_init_valid` returns `Ok`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`:

```rust
sequencer_address: init.builder,
``` [2](#0-1) 

That `BlockInfo` is sent to the batcher via `validate_block`, so every transaction in the block is executed with the proposer-supplied `builder` as the sequencer address. [3](#0-2) 

**`builder` is also persisted in the block header**

`update_state_sync_with_new_block` stores `init.builder` as `SequencerContractAddress` in `BlockHeaderWithoutHash.sequencer`, so the wrong address is committed to storage and propagated to sync peers:

```rust
let sequencer = SequencerContractAddress(init.builder);
``` [4](#0-3) 

**Why the commitment check does not catch this**

The `ProposalCommitment` is `Poseidon(partial_block_hash, fee_proposal_fri)`. The partial block hash is computed by the batcher from the block it actually executed — using the attacker-supplied `builder` as `sequencer_address`. The proposer pre-computed the same commitment with the same `builder`, so `built_block == received_fin.proposal_commitment` holds and the proposal is accepted: [5](#0-4) 

**The proposer's own `builder_address` is never compared**

When building a proposal the node uses `self.config.static_config.builder_address` (with a TODO noting it should eventually come from the committee). That value is never consulted during validation:

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
