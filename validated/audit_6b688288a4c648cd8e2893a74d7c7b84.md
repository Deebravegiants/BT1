### Title
`OmniBridgeWormhole.initializeWormhole()` Always Reverts Due to Nested `initializer` Modifier Conflict, Enabling Front-Run Takeover of `nearBridgeDerivedAddress` — (File: `evm/src/omni-bridge/contracts/OmniBridgeWormhole.sol`)

---

### Summary

`OmniBridgeWormhole.initializeWormhole()` is decorated with OpenZeppelin's `initializer` modifier and internally calls `OmniBridge.initialize()`, which is also decorated with `initializer`. The nested call always reverts because OZ's `initializer` sets `_initialized = 1` and `_initializing = true` before the body executes; when `initialize()` is then entered, neither the `initialSetup` nor the `construction` branch is satisfiable, causing an unconditional `InvalidInitialization()` revert. The intended initializer is permanently broken. The only reachable initialization path is calling `OmniBridge.initialize()` directly — a `public` function with no access control beyond the `initializer` guard — which any unprivileged attacker can front-run to install a malicious `nearBridgeDerivedAddress`. That address is the sole signature-verification key for both `finTransfer()` and `deployToken()`, so the attacker gains the ability to mint arbitrary wrapped bridge tokens.

---

### Finding Description

**Root cause — conflicting `initializer` modifiers:**

`OmniBridgeWormhole.initializeWormhole()` carries the `initializer` modifier: [1](#0-0) 

Its first action is to call `OmniBridge.initialize()`, which also carries `initializer`: [2](#0-1) 

When `initializeWormhole()` is the top-level call, OZ's `initializer` modifier sets `_initialized = 1` and `_initializing = true` before the body runs. When `initialize()` is then entered, the modifier evaluates:

```
isTopLevelCall  = !_initializing  = false
initialized     = _initialized    = 1
initialSetup    = (initialized == 0 && isTopLevelCall) = false
construction    = (initialized == 1 && address(this).code.length == 0)
               → false  (proxy has deployed bytecode)
```

Neither branch is true → `revert InvalidInitialization()`. `initializeWormhole()` **always reverts** and can never complete.

**Forced fallback to the unprotected `initialize()`:**

Because `initializeWormhole()` is permanently broken, the only callable initializer is `OmniBridge.initialize()`, which is `public` with no role guard: [3](#0-2) 

The proxy is deployed with empty init-data (`bytes("")`), so `_initialized` starts at 0 and any external caller can invoke `initialize()` before the admin.

**Attacker-controlled `nearBridgeDerivedAddress`:**

`nearBridgeDerivedAddress` is the only signer checked in both `finTransfer()` and `deployToken()`: [4](#0-3) [5](#0-4) 

An attacker who front-runs `initialize()` with their own address as `nearBridgeDerivedAddress_` becomes the sole trusted signer. They can then:

1. Call `finTransfer()` with a self-signed payload to mint arbitrary amounts of any registered bridge token to any recipient.
2. Call `deployToken()` with a self-signed metadata payload to register and deploy new bridge tokens under attacker-controlled parameters.

**Secondary impact — `_wormhole` permanently unset:**

Even without front-running, if the admin calls `initialize()` directly (the only non-reverting path), `_wormhole` and `_consistencyLevel` are never written. Every call to `deployTokenExtension`, `logMetadataExtension`, `finTransferExtension`, and `initTransferExtension` will call `publishMessage` on `address(0)`, reverting all cross-chain operations: [6](#0-5) 

---

### Impact Explanation

- **Critical — Unauthorized creation of wrapped bridge assets.** An attacker who front-runs `initialize()` controls `nearBridgeDerivedAddress` and can forge valid ECDSA signatures accepted by `finTransfer()` and `deployToken()`, minting unbacked wrapped tokens to arbitrary recipients.
- **Critical — Irreversible frozen redemption path.** Even without front-running, `_wormhole = address(0)` causes every bridge operation to revert, permanently freezing all cross-chain transfers through the Wormhole path.

---

### Likelihood Explanation

The proxy is deployed with empty calldata, leaving a window between deployment and admin initialization. Front-running is a standard mempool attack requiring no special privilege. The `initialize()` function is `public` with no access control. The attacker needs only to observe the deployment transaction and submit a competing `initialize()` call with higher gas.

---

### Recommendation

1. Change `OmniBridge.initialize()` from `public initializer` to `internal` and annotate it with `onlyInitializing` so it can be safely called from child initializers:

```solidity
// OmniBridge.sol
function initialize(
    address tokenImplementationAddress_,
    address nearBridgeDerivedAddress_,
    uint8 omniBridgeChainId_
) internal onlyInitializing {
    ...
}
```

2. Keep `OmniBridgeWormhole.initializeWormhole()` as the sole `external initializer` entry point. This eliminates the nested-modifier conflict and removes the unguarded public `initialize()` surface.

3. If `OmniBridge` must remain independently deployable, expose a separate `public initializer` in `OmniBridge` and a distinct `public initializer` in `OmniBridgeWormhole` that calls the internal `__OmniBridge_init()` helper — the standard OZ upgradeable inheritance pattern.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.24;

import "forge-std/Test.sol";
import {ERC1967Proxy} from "@openzeppelin/contracts/proxy/ERC1967/ERC1967Proxy.sol";
import {OmniBridgeWormhole} from "src/omni-bridge/contracts/OmniBridgeWormhole.sol";
import {BridgeToken} from "src/omni-bridge/contracts/BridgeToken.sol";

contract OmniBridgeWormholeInitTest is Test {
    OmniBridgeWormhole logic;
    OmniBridgeWormhole bridge;
    BridgeToken tokenLogic;

    address admin   = address(0xAD);
    address attacker = address(0xBAD);
    address wormhole = address(0xBEEF);

    function setUp() public {
        vm.startPrank(admin);
        logic     = new OmniBridgeWormhole();
        tokenLogic = new BridgeToken();
        // Proxy deployed with empty init data — _initialized == 0
        bridge = OmniBridgeWormhole(
            address(new ERC1967Proxy(address(logic), bytes("")))
        );
        vm.stopPrank();
    }

    // 1. Demonstrate initializeWormhole() always reverts
    function test_initializeWormhole_alwaysReverts() public {
        vm.prank(admin);
        vm.expectRevert(); // InvalidInitialization from nested initializer
        bridge.initializeWormhole(
            address(tokenLogic),
            vm.addr(1),   // intended nearBridgeDerivedAddress
            1,
            wormhole,
            1
        );
    }

    // 2. Demonstrate attacker front-runs initialize() with malicious signer
    function test_attacker_frontRuns_initialize() public {
        address maliciousSigner = vm.addr(0xDEAD);

        // Attacker calls initialize() before admin
        vm.prank(attacker);
        bridge.initialize(address(tokenLogic), maliciousSigner, 1);

        // Admin's initializeWormhole() now reverts (already initialized)
        vm.prank(admin);
        vm.expectRevert();
        bridge.initializeWormhole(
            address(tokenLogic), vm.addr(1), 1, wormhole, 1
        );

        // Attacker controls nearBridgeDerivedAddress
        assertEq(bridge.nearBridgeDerivedAddress(), maliciousSigner);
        // Attacker can now sign finTransfer / deployToken payloads
    }
}
```
