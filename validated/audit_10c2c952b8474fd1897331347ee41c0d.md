### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped Per-Bin Additional Fees — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

The pool admin can call `setPoolBinAdditionalFees` to set per-bin swap fees (`addFeeBuyE6`, `addFeeSellE6`) up to `uint16.max = 65 535 E6 ≈ 6.55 %` with **no cap enforcement**, while the factory owner's `maxAdminSpreadFeeE6` cap — which is supposed to bound all admin-controlled fee extraction — is never consulted for this path. This lets the pool admin silently exceed the factory-owner-imposed ceiling and extract more from every swap than the protocol governance model permits.

---

### Finding Description

The factory owner controls the maximum admin fee via `maxAdminSpreadFeeE6` (and `maxAdminNotionalFeeE8`). When the pool admin calls `setPoolAdminFees`, the factory enforces the cap:

```solidity
// MetricOmmPoolFactory.sol – setPoolAdminFees
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
``` [1](#0-0)

```solidity
