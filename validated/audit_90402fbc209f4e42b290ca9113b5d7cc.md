### Title
No Mechanism to Remove a Whitelisted Foreign Chain from `AllowedProviders` ŌĆö (File: `crates/contract/src/foreign_chain_rpc.rs`)

---

### Summary

The `vote_update_foreign_chain_providers` endpoint allows threshold-agreeing participants to add or update a foreign chain's RPC provider entry in `AllowedProviders`. However, there is no corresponding mechanism to **remove** a foreign chain from the whitelist once it has been added. Because `ChainEntry` mandates a `NonEmptyBTreeMap` of providers, participants cannot even vote in a zero-provider entry to effectively nullify a chain. A whitelisted chain is therefore permanent and irremovable on-chain, breaking the network's ability to respond to a security incident on a foreign chain.

---

### Finding Description

`AllowedProviders` is the on-chain structure that records which foreign chains the MPC network trusts for transaction verification. It is backed by an `IterableMap` and exposes only three methods:

- `replace` ŌĆö inserts or overwrites a chain entry
- `snapshot` ŌĆö read-only view
- `is_whitelisted` ŌĆö read-only predicate [1](#0-0) 

There is no `remove` method. The sole mutation path through `ForeignChainRpcWhitelist::vote` always calls `self.entries.replace(chain, entry)` and never removes a chain slot: [2](#0-1) 

Additionally, `ChainEntry` enforces a `NonEmptyBTreeMap` for its provider list, so participants cannot vote in a zero-provider entry to indirectly disable a chain: [3](#0-2) 

The `TryFrom<dtos::ChainEntry>` conversion explicitly rejects a quorum of zero and a quorum exceeding the provider count, but there is no path that accepts an empty provider map: [4](#0-3) 

The contract's availability gate (`verify_foreign_transaction`) rejects a chain only when it is not in the **available** set (whitelisted AND covered by Ōēź threshold nodes). Operators can make a chain temporarily unavailable by removing it from their local config, but the chain remains **whitelisted** on-chain and can become available again the moment enough nodes re-register it ŌĆö there is no permanent on-chain removal path. [5](#0-4) 

---

### Impact Explanation

**Medium.** If a whitelisted foreign chain is compromised (e.g., a 51% attack, a critical RPC-layer vulnerability, or a chain deprecation), the MPC network has no on-chain mechanism to permanently remove it from the whitelist. The chain remains a valid target for `verify_foreign_transaction`, and any threshold of nodes that re-registers it (even accidentally after a config rollout) would make it available again. This breaks the production safety invariant that the network can authoritatively and permanently disable a compromised chain, and it creates a window in which forged foreign-chain transactions could be submitted and verified against a chain the operators intended to decommission.

---

### Likelihood Explanation

**Low-to-Medium.** Foreign chains are not expected to be compromised frequently, but the scenario is realistic: chains have undergone 51% attacks (Ethereum Classic, Bitcoin Gold), RPC providers have been compromised, and chains are deprecated. The design document explicitly acknowledges the need to remove compromised providers: [6](#0-5) 

Provider-level removal is addressed (by voting in a new `ChainEntry` without the compromised provider), but chain-level removal is not. The gap is therefore a known design blind spot, not a purely theoretical concern.

---

### Recommendation

Add a `vote_remove_foreign_chain` endpoint (or extend `vote_update_foreign_chain_providers` to accept a sentinel "remove" variant) that, upon threshold agreement, calls a new `AllowedProviders::remove(chain)` method. The removal should also trigger a recomputation of the available-chain cache (analogous to the existing `recompute_available_foreign_chains` call) and clear any pending provider votes for the removed chain via `ProviderVotes::retain`.

---

### Proof of Concept

1. Threshold of participants call `vote_update_foreign_chain_providers` for `ForeignChain::Ethereum`, whitelisting it with a valid `ChainEntry`. The entry is stored via `AllowedProviders::replace`.
2. Ethereum undergoes a 51% attack; the network operators decide to stop serving Ethereum signing requests.
3. Operators attempt to remove Ethereum from the whitelist. No such endpoint exists ŌĆö `AllowedProviders` has no `remove` method and `ForeignChainRpcWhitelist::vote` only calls `replace`.
4. Operators attempt to vote in a zero-provider `ChainEntry` to nullify the slot. The `NonEmptyBTreeMap` constraint in `ChainEntry` rejects this at deserialization time.
5. Operators remove Ethereum from their local `foreign_chains.yaml`, making it temporarily unavailable. However, the chain remains whitelisted on-chain.
6. After a config rollout or operator error, enough nodes re-register Ethereum coverage; the chain becomes available again without any on-chain vote, and `verify_foreign_transaction` resumes accepting Ethereum requests ŌĆö against the operators' intent. [7](#0-6) [8](#0-7)

impl TryFrom<dtos::ChainEntry> for ChainEntry {
    type Error = ChainEntryValidationError;

    fn try_from(entry: dtos::ChainEntry) -> Result<Self, Self::Error> {
        let dtos::ChainEntry { providers, quorum } = entry;
        if quorum == 0 {
            return Err(ChainEntryValidationError::ZeroQuorum);
        }
        let providers_len = u64::try_from(providers.len()).map_err(|e| {
            ChainEntryValidationError::ProvidersLenOverflow {
                len: providers.len(),
                reason: e.to_string(),
            }
        })?;
        if quorum > providers_len {
            return Err(ChainEntryValidationError::QuorumExceedsProviders {
                quorum,
                providers_len,
            });
        }
        for (id, config) in providers.iter() {
            if let ChainRouting::PathSegment { segment } = &config.chain_routing
                && segment.contains('/')
            {
                return Err(ChainEntryValidationError::PathSegmentContainsSlash {
                    provider_id: id.0.clone(),
                });
            }
            if let (
                ChainRouting::QueryParam {
                    name: routing_name, ..
                },
                dtos::AuthScheme::Query { name: auth_name },
            ) = (&config.chain_routing, &config.auth_scheme)
                && routing_name == auth_name
            {
                return Err(ChainEntryValidationError::QueryParamCollidesWithAuth {
                    provider_id: id.0.clone(),
                    name: auth_name.clone(),
                });
            }
        }
        Ok(ChainEntry { providers, quorum })
    }

#[derive(Debug)]
pub(crate) struct AllowedProviders {
    entries: IterableMap<ForeignChain, ChainEntry>,
}

impl Default for AllowedProviders {
    fn default() -> Self {
        Self {
            entries: IterableMap::new(StorageKey::AllowedForeignChainProvidersV1),
        }
    }
}

impl AllowedProviders {
    fn replace(&mut self, chain: ForeignChain, entry: ChainEntry) {
        self.entries.insert(chain, entry);
    }

    /// Owned clone of the whitelist as DTOs; required by `allowed_foreign_chain_providers`,
    /// which borsh-serializes the result across the contract boundary.
    pub fn snapshot(&self) -> BTreeMap<ForeignChain, dtos::ChainEntry> {
        self.entries
            .iter()
            .map(|(c, e)| (*c, e.clone().into()))
            .collect()
    }

    /// Whether `chain` is currently whitelisted (has a voted-in `ChainEntry`).
    pub fn is_whitelisted(&self, chain: &ForeignChain) -> bool {
        self.entries.contains_key(chain)
    }
}

    pub fn vote(
        &mut self,
        participant: AuthenticatedParticipantId,
        votes: NonEmptyBTreeMap<ForeignChain, dtos::ChainEntry>,
        threshold_parameters: &ThresholdParameters,
    ) -> Result<Vec<ForeignChain>, Error> {
        let mut applied: Vec<ForeignChain> = Vec::new();
        let votes: BTreeMap<ForeignChain, dtos::ChainEntry> = votes.into();
        for (chain, entry) in votes {
            let entry: ChainEntry = entry.try_into()?;
            let hash = ProposalHash::from(entry.clone());
            if self
                .votes
                .vote(chain, hash, participant.clone(), threshold_parameters)?
            {
                self.entries.replace(chain, entry);
                applied.push(chain);
            }
        }
        Ok(applied)
    }
