### Title
Unsigned Absorb Spend While in LEAVING_POOL State Resets `relative_lock_height` Timer, Griefing Farmer's Pool Exit — (`chia/pools/pool_puzzles.py`)

---

### Summary

The pool singleton's waiting-room (LEAVING\_POOL) inner puzzle allows an absorb ("claim-p2-nft") spend path that requires **no owner signature**. When this spend is executed, the current singleton coin is consumed and a new one is created at the current block height. Because `ASSERT_HEIGHT_RELATIVE` is measured from the **creation height of the coin being spent**, each unsigned absorb resets the `relative_lock_height` countdown. An unprivileged attacker who observes a farmer entering the LEAVING\_POOL state can repeatedly absorb the farmer's legitimate p2\_singleton farming rewards — at zero cost when fee is 0 — to perpetually reset the exit timer and prevent the farmer from ever completing their pool exit.

---

### Finding Description

**Pool singleton state machine recap**

A farmer leaving a pool transitions through:
1. `FARMING_TO_POOL` → `LEAVING_POOL` (waiting room) — requires owner signature
2. `LEAVING_POOL` → `SELF_POOLING` — requires owner signature **and** `ASSERT_HEIGHT_RELATIVE(relative_lock_height)`

The `ASSERT_HEIGHT_RELATIVE` condition is evaluated against the coin being spent. If the singleton coin was created at block height H, the exit spend is only valid at height ≥ H + `relative_lock_height`.

**The unsigned absorb path**

The waiting-room inner puzzle (`POOL_WAITING_ROOM_MOD`) exposes two spend paths:

| Path | Signature required? | Effect |
|---|---|---|
| exit-waiting-room | Yes (owner key) | Transitions to SELF\_POOLING; gated by `ASSERT_HEIGHT_RELATIVE` |
| claim-p2-nft (absorb) | **No** | Spends current singleton, creates new singleton at current height |

`create_absorb_spend` in `pool_puzzles.py` constructs the absorb bundle with an empty `G2Element()` signature: [1](#0-0) 

The wallet code itself documents this: [2](#0-1) 

The lifecycle test confirms the absorb executes with no signature while in the waiting room: [3](#0-2) 

**Timer reset mechanism**

After the absorb, the new singleton coin is created at the absorb block height H\_absorb. The farmer's exit spend now requires height ≥ H\_absorb + `relative_lock_height`. If H\_absorb > H\_entry (the original waiting-room entry height), the farmer must wait longer than intended. An attacker who absorbs at height H\_entry + `relative_lock_height` − 1 (just before the farmer can exit) forces the farmer to wait another full `relative_lock_height` period.

**Attacker requirements**

- Know the farmer's singleton launcher\_id (public on-chain)
- Derive the p2\_singleton puzzle hash (deterministic from launcher\_id)
- Find an unspent p2\_singleton farming reward coin at that address (public blockchain data)
- Submit the absorb spend bundle with `G2Element()` (no key needed)

The p2\_singleton coins are legitimate farming rewards; the attacker cannot fabricate them. However, any farmer actively farming will continuously accumulate new rewards at the p2\_singleton address, giving the attacker a perpetual supply of absorb opportunities. [4](#0-3) 

---

### Impact Explanation

**High — Permanent or long-lived inability for farmers to complete pool actions.**

A farmer who has initiated a pool exit (`LEAVING_POOL`) can be indefinitely prevented from completing it. The attacker absorbs each new farming reward as it appears, resetting the `relative_lock_height` countdown each time. Since the farmer is still farming (otherwise they would not be trying to leave — they want to self-pool), new rewards keep arriving. The attack costs the attacker only the transaction fee, which is 0 when no fee is attached to the absorb.

The farmer's rewards are not stolen (they still go to `target_puzzle_hash`), but the farmer is locked into the pool indefinitely against their will. This constitutes unauthorized singleton mutation and a long-lived inability to complete a pool action. [5](#0-4) 

---

### Likelihood Explanation

- The attack requires no special privileges, no keys, and no funds beyond a small transaction fee (optionally zero).
- All information needed (launcher\_id, p2\_singleton address, unspent reward coins) is publicly visible on-chain.
- A pool operator who wants to retain a farmer, or a competitor who wants to harm a farmer, has clear economic motivation.
- The attack is repeatable as long as the farmer continues to farm.

---

### Recommendation

The waiting-room absorb spend should assert the owner's signature (or at minimum assert a minimum coin age for the resulting singleton), so that only the farmer can trigger an absorb while in the LEAVING\_POOL state. Alternatively, the `ASSERT_HEIGHT_RELATIVE` for the exit spend should be anchored to the **original waiting-room entry height** (stored in the puzzle or solution), not the creation height of the most recently spent singleton coin.

---

### Proof of Concept

1. Farmer calls `pw_self_pool` → singleton enters `LEAVING_POOL` at block height H. `relative_lock_height` = 1000.
2. Farmer must wait until height H + 1000 to complete exit.
3. At height H + 999, attacker observes an unspent p2\_singleton reward coin at the farmer's p2\_singleton address.
4. Attacker calls `create_absorb_spend(last_coin_spend, target_pool_state, launcher_coin, reward_height, ...)` and submits the bundle with `G2Element()` — accepted by the network.
5. New singleton coin is created at height H + 999. Farmer now must wait until (H + 999) + 1000 = H + 1999.
6. Attacker repeats at height H + 1998, forcing the farmer to wait until H + 2998.
7. This continues indefinitely as long as new farming rewards appear. [6](#0-5) [7](#0-6)

def create_absorb_spend(
    last_coin_spend: CoinSpend,
    current_state: PoolState,
    launcher_coin: Coin,
    height: uint32,
    genesis_challenge: bytes32,
    delay_time: uint64,
    delay_ph: bytes32,
) -> list[CoinSpend]:
    inner_puzzle: Program = pool_state_to_inner_puzzle(
        current_state, launcher_coin.name(), genesis_challenge, delay_time, delay_ph
    )
    reward_amount: uint64 = calculate_pool_reward(height)
    if is_pool_member_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol: Program = Program.to([reward_amount, height])
    elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, destination_puzhash, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol = Program.to([0, reward_amount, height])
    else:
        raise ValueError
    # full sol = (parent_info, my_amount, inner_solution)
    coin: Coin | None = get_most_recent_singleton_coin_from_coin_spend(last_coin_spend)
    assert coin is not None

    if coin.parent_coin_info == launcher_coin.name():
        parent_info: Program = Program.to([launcher_coin.parent_coin_info, launcher_coin.amount])
    else:
        p = Program.from_bytes(bytes(last_coin_spend.puzzle_reveal))
        last_coin_spend_inner_puzzle: Program | None = get_inner_puzzle_from_puzzle(p)
        assert last_coin_spend_inner_puzzle is not None
        parent_info = Program.to(

                last_coin_spend.coin.parent_coin_info,
                last_coin_spend_inner_puzzle.get_tree_hash(),
                last_coin_spend.coin.amount,
            ]
        )
    full_solution: SerializedProgram = SerializedProgram.to([parent_info, last_coin_spend.coin.amount, inner_sol])
    full_puzzle: SerializedProgram = create_full_puzzle(inner_puzzle, launcher_coin.name()).to_serialized()
    assert coin.puzzle_hash == full_puzzle.get_tree_hash()

    p2_singleton_puzzle = create_p2_singleton_puzzle(
        SINGLETON_MOD_HASH, launcher_coin.name(), delay_time, delay_ph

    p2_singleton_solution = SerializedProgram.to([inner_puzzle.get_tree_hash(), reward_coin.name()])

    assert full_puzzle.get_tree_hash() == coin.puzzle_hash
    assert get_inner_puzzle_from_puzzle(Program.from_bytes(bytes(full_puzzle))) is not None

    coin_spends = [
        CoinSpend(coin, full_puzzle, full_solution),
        CoinSpend(reward_coin, p2_singleton_puzzle, p2_singleton_solution),
    ]
    return coin_spends
