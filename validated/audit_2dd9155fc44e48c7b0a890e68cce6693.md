### Title
Missing Minimum `BlockConfirmations` Validation in `verify_foreign_transaction` Enables Attestation of Insufficiently-Confirmed Transactions - (File: `crates/contract/src/lib.rs`)

### Summary

The `verify_foreign_transaction` contract endpoint accepts a user-supplied `BlockConfirmations` value (including `0`) with no on-chain minimum enforcement. Because the node-side inspector uses a `<=` comparison (`threshold <= actual_confirmations`), setting `confirmations: 0` causes the finality check to be trivially satisfied for any transaction that has at least one confirmation and a valid blockhash. An unprivileged caller can obtain a valid MPC threshold signature attesting to a minimally-confirmed (easily-reorganizable) Bitcoin transaction, enabling a double-spend against any bridge contract that trusts these attestations.

### Finding Description

`verify_foreign_transaction` in `crates/contract/src/lib.rs` accepts a `VerifyForeignTransactionRequestArgs` that includes a `ForeignChainRpcRequest::Bitcoin(BitcoinRpcRequest { confirmations, ... })`. The `confirmations` field is typed as `BlockConfirmations(pub u64)` — a plain `u64` wrapper with no minimum constraint. [1](#0-0) 

The contract performs no validation on the `confirmations` value before enqueuing the request: [2](#0-1) 

The ABI schema confirms `BlockConfirmations` allows `minimum: 0.0`: [3](#0-2) 

On the node side, `BitcoinInspector::extract` enforces the threshold with:

```rust
let enough_block_confirmations =
    block_confirmations_threshold <= transaction_block_confirmation;
``` [4](#0-3) 

When `block_confirmations_threshold = 0`, the condition `0 <= actual_confirmations` is always `true` for any transaction that has been mined (≥ 1 confirmation), bypassing the finality check entirely. The subsequent `verify_block_is_canonical` call succeeds because a mined transaction has a valid `blockhash`. The nodes then produce a threshold signature over the `ForeignTxSignPayload` and call `respond_verify_foreign_tx`, which the contract accepts after verifying the signature: [5](#0-4) 

The `BlockConfirmations` type definition has no minimum: [6](#0-5) 

### Impact Explanation

A bridge contract (e.g., an omnibridge inbound flow) that calls `verify_foreign_transaction` and trusts the returned MPC signature to release NEAR-side funds will release those funds upon receiving a valid signature for a 1-confirmation Bitcoin transaction. Bitcoin transactions with 1 confirmation are trivially reorganizable by a miner with sufficient hashrate, or via a selfish-mining attack. The attacker:

1. Sends a Bitcoin transaction (1 confirmation).
2. Calls `verify_foreign_transaction` with `confirmations: 0`.
3. Receives a valid MPC threshold signature attesting to the transaction.
4. Presents the signature to the bridge contract to claim NEAR-side funds.
5. Executes a Bitcoin double-spend (reorganization), recovering the Bitcoin.

This constitutes a **forged foreign-chain verification that causes invalid bridge execution and double-spend conditions**, matching the High impact tier.

### Likelihood Explanation

The entry path requires only a 1 yoctoNEAR deposit and a valid NEAR account — no privileged access. The `confirmations` field is fully attacker-controlled. The only external requirement is the ability to mine or bribe a miner for a short reorganization, which is a known and economically viable attack on Bitcoin for high-value targets. The contract's lack of any minimum confirmation guard makes this straightforwardly exploitable.

### Recommendation

**Short term:** Add an on-chain minimum confirmation check inside `verify_foreign_transaction` before enqueuing the request. Reject any `BitcoinRpcRequest` with `confirmations < MINIMUM_BITCOIN_CONFIRMATIONS` (e.g., 6):

```rust
const MINIMUM_BITCOIN_CONFIRMATIONS: u64 = 6;

if let ForeignChainRpcRequest::Bitcoin(ref btc_req) = request.request {
    if btc_req.confirmations.0 < MINIMUM_BITCOIN_CONFIRMATIONS {
        env::panic_str("Bitcoin confirmations below minimum required");
    }
}
```

**Long term:** Encode per-chain finality minimums as part of the on-chain `ChainEntry` governance configuration (voted in by participants), so operators can adjust the minimum confirmation threshold without a contract upgrade. This mirrors the existing `quorum` validation in `ChainEntry::try_from`: [7](#0-6) 

### Proof of Concept

```json
// Call verify_foreign_transaction with confirmations: 0
{
  "request": {
    "domain_id": 0,
    "payload_version": "V1",
    "request": {
      "Bitcoin": {
        "tx_id": "<hex of a 1-confirmation Bitcoin tx>",
        "confirmations": 0,
        "extractors": ["BlockHash"]
      }
    }
  }
}
```

1. Attacker broadcasts Bitcoin tx `T`, waits for 1 confirmation.
2. Attacker calls `verify_foreign_transaction` with `confirmations: 0` and `tx_id = T`.
3. MPC nodes call `BitcoinInspector::extract(T, BlockConfirmations(0), [BlockHash])`.
4. Node check: `0 <= 1` → `true` → confirmation gate bypassed.
5. `verify_block_is_canonical` succeeds (tx is in a block).
6. Nodes produce threshold signature; `respond_verify_foreign_tx` accepts it.
7. Attacker presents signature to bridge contract, receives NEAR-side funds.
8. Attacker double-spends `T` on Bitcoin via reorganization. [8](#0-7) [1](#0-0)

    pub fn respond_verify_foreign_tx(
        &mut self,
        request: VerifyForeignTransactionRequest,
        response: VerifyForeignTransactionResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!(
            "respond_verify_foreign_tx: signer={}, request={:?}",
            &signer,
            &request
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain.0.into())?;

        let signature_is_valid = match (&response.signature, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");

                let payload_hash: [u8; 32] = response.payload_hash.0;

                // Check the signature is correct against the root public key
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    &payload_hash,
                    &secp_pk,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_verify_foreign_tx_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )

    async fn extract(
        &self,
        transaction: BitcoinTransactionHash,
        block_confirmations_threshold: BlockConfirmations,
        extractors: Vec<BitcoinExtractor>,
    ) -> Result<Vec<BitcoinExtractedValue>, ForeignChainInspectionError> {
        let request_parameters = GetRawTransactionArgs {
            transaction_hash: TransportBitcoinTransactionHash::from(*transaction),
            verbose: VERBOSE_RESPONSE,
        };

        // TODO(#1978): add retry mechanism if the error from the request is transient
        let rpc_response: GetRawTransactionVerboseResponse = self
            .client
            .request(GET_RAW_TRANSACTION_METHOD, &request_parameters)
            .await?;

        let transaction_block_confirmation = rpc_response.confirmations.into();
        let enough_block_confirmations =
            block_confirmations_threshold <= transaction_block_confirmation;

        if !enough_block_confirmations {
            return Err(ForeignChainInspectionError::NotEnoughBlockConfirmations {
                expected: block_confirmations_threshold,
                got: transaction_block_confirmation,
            });
        }

        self.verify_block_is_canonical(rpc_response.blockhash)
            .await?;

        let extracted_values = extractors
            .iter()
            .map(|extractor| extractor.extract_value(&rpc_response))
            .collect();

        Ok(extracted_values)
    }

impl TryFrom<dtos::ChainEntry> for ChainEntry {
    type Error = ChainEntryValidationError;

    fn try_from(entry: dtos::ChainEntry) -> Result<Self, Self::Error> {
        let dtos::ChainEntry { providers, quorum } = entry;
        if quorum == 0 {
            return Err(ChainEntryValidationError::ZeroQuorum);
        }
        let providers_len = u64::try_from(providers.len()).map_err(|e| {
            ChainEntryValidationError::ProvidersLenOverflow {
                len: providers.len(),
                reason: e.to_string(),
            }
        })?;
        if quorum > providers_len {
            return Err(ChainEntryValidationError::QuorumExceedsProviders {
                quorum,
                providers_len,
            });
        }
        for (id, config) in providers.iter() {
            if let ChainRouting::PathSegment { segment } = &config.chain_routing
                && segment.contains('/')
            {
                return Err(ChainEntryValidationError::PathSegmentContainsSlash {
                    provider_id: id.0.clone(),
                });
            }
            if let (
                ChainRouting::QueryParam {
                    name: routing_name, ..
                },
                dtos::AuthScheme::Query { name: auth_name },
            ) = (&config.chain_routing, &config.auth_scheme)
                && routing_name == auth_name
            {
                return Err(ChainEntryValidationError::QueryParamCollidesWithAuth {
                    provider_id: id.0.clone(),
                    name: auth_name.clone(),
                });
            }
        }
        Ok(ChainEntry { providers, quorum })
    }
