### Title
ECDSA Signature Malleability in `ossl_ecdsa_verify` — Missing Lower-S Normalization Allows Malleable Signature Acceptance - (File: `src/cbmpc/crypto/base_ecc.cpp`)

---

### Summary

The custom `ossl_ecdsa_verify` function used by both `ecurve_secp256k1_t::verify` and `ecurve_ossl_t::verify` — and therefore by the public `ecc_pub_key_t::verify` API — only checks that `s ∈ (0, q)` but never enforces `s ≤ q/2`. Any caller of the public verification API will accept both `(r, s)` and `(r, q−s)` as valid signatures over the same message and key, giving a trivially computable second valid signature for every signature the MPC protocols produce.

---

### Finding Description

`ossl_ecdsa_verify` in `src/cbmpc/crypto/base_ecc.cpp` performs the following range check on the signature scalars:

```cpp
if (r <= 0 || r >= q || s <= 0 || s >= q)
    return coinbase::error(E_CRYPTO, "ossl_ecdsa_verify: invalid scalar");
``` [1](#0-0) 

There is no `s > q/2` rejection. The function then proceeds to compute the standard ECDSA verification equation and returns `SUCCESS` for any `s` in `(0, q)`.

This function is the sole verification back-end for both curve families:

- `ecurve_secp256k1_t::verify` (secp256k1) delegates directly to it: [2](#0-1) 

- `ecurve_ossl_t::verify` (P-256, P-384, P-521) delegates directly to it: [3](#0-2) 

Both are reached through the public `ecc_pub_key_t::verify` method: [4](#0-3) 

The MPC signing protocols do normalize `s` before outputting a signature. In `ecdsa_2p.cpp`:

```cpp
bn_t q_minus_s = q - s;
if (q_minus_s < s) s = q_minus_s;
``` [5](#0-4) 

And in `ecdsa_mp.cpp`: [6](#0-5) 

So the protocols produce lower-S signatures and their internal self-check passes. However, the public `ecc_pub_key_t::verify` API — which is the verification entry point documented and exposed to all callers — accepts the high-S malleable counterpart `(r, q−s)` equally.

The vendored secp256k1 library's own `secp256k1_ecdsa_verify` explicitly rejects high-S signatures:

```c
return (!secp256k1_scalar_is_high(&s) && ...);
``` [7](#0-6) 

cb-mpc's own secp256k1 verification path bypasses this library function entirely and uses `ossl_ecdsa_verify` instead, losing the lower-S enforcement.

---

### Impact Explanation

Any system that calls `ecc_pub_key_t::verify` (the shipped public C++ API, or the C API wrappers `cbmpc_ecdsa_2p_sign` / `cbmpc_ecdsa_mp_sign_additive` / `cbmpc_ecdsa_mp_sign_ac` whose callers verify the returned DER blob) will accept a malleable signature `(r, q−s)` as valid for the same message and public key. Concretely:

- A caller that uses the returned DER signature as a unique token for replay protection, deduplication, or authorization will accept both the original and its malleated form, enabling double-spend or double-authorization attacks.
- A caller that stores the signature as a commitment and later checks "was this exact signature seen before?" will be bypassed.
- The public `ecc_pub_key_t::verify` is the documented verification entry point for MPC-produced signatures (used in tests and demos as the canonical check), so callers have no indication they must add their own lower-S check. [8](#0-7) [9](#0-8) 

---

### Likelihood Explanation

The attack requires no cryptographic capability: given any valid lower-S signature `(r, s)` output by the MPC protocol, the malleable form `(r, q−s)` is computed by a single modular subtraction. No key material, threshold participation, or protocol interaction is needed. The only precondition is that the caller uses `ecc_pub_key_t::verify` in a context where signature uniqueness is a security invariant, which is the natural use case for MPC-produced blockchain or authentication signatures.

---

### Recommendation

Add a lower-S rejection to `ossl_ecdsa_verify` immediately after the existing range check:

```cpp
if (r <= 0 || r >= q || s <= 0 || s >= q)
    return coinbase::error(E_CRYPTO, "ossl_ecdsa_verify: invalid scalar");
// Enforce lower-S normalization (prevents signature malleability)
bn_t half_q = (q.value() + 1) / 2;
if (s > half_q)
    return coinbase::error(E_CRYPTO, "ossl_ecdsa_verify: high-S signature rejected");
```

This aligns `ossl_ecdsa_verify` with the behavior of the vendored `secp256k1_ecdsa_verify` and with OpenZeppelin's ECDSA library (the fix applied in the reference report).

---

### Proof of Concept

```cpp
// 1. Run the 2-party MPC signing protocol to obtain a lower-S DER signature.
buf_t sig_der;  // produced by ecdsa2pc::sign(...)

// 2. Parse the signature and compute the malleable counterpart.
ecdsa_signature_t sig;
sig.from_der(curve_secp256k1, sig_der);
bn_t r = sig.get_r();
bn_t s = sig.get_s();
const mod_t& q = curve_secp256k1.order();
bn_t s_high = q.value() - s;   // s_high > q/2 by construction (s was lower-S)

ecdsa_signature_t malleable_sig(curve_secp256k1, r, s_high);
buf_t malleable_der = malleable_sig.to_der();

// 3. Verify the malleable signature against the same public key and message.
ecc_pub_key_t pub(key.Q);
error_t rv = pub.verify(msg_hash, malleable_der);
// rv == SUCCESS  ← ossl_ecdsa_verify accepts high-S; no lower-S check present
assert(rv == SUCCESS);

// 4. Both sig_der and malleable_der now pass verification for the same (msg, key).

error_t ossl_ecdsa_verify(const EC_GROUP* group, EC_POINT* point, mem_t hash, mem_t signature) {
  error_t rv = UNINITIALIZED_ERROR;
  ecurve_t curve = ecurve_t::find(group);
  if (!curve) return coinbase::error(E_CRYPTO, "ossl_ecdsa_verify: unsupported curve");

  ecdsa_signature_t sig;
  if (rv = sig.from_der(curve, signature)) return rv;

  const mod_t& q = curve.order();
  const bn_t& r = sig.get_r();
  const bn_t& s = sig.get_s();
  if (r <= 0 || r >= q || s <= 0 || s >= q) return coinbase::error(E_CRYPTO, "ossl_ecdsa_verify: invalid scalar");

  int curve_size = curve.size();
  if (hash.size >= curve_size) hash.size = curve_size;
  const bn_t e = bn_t::from_bin(hash);

  buf_t oct(curve.point_bin_size());
  cb_assert(EC_POINT_point2oct(group, point, POINT_CONVERSION_UNCOMPRESSED, oct.data(), oct.size(),
                               bn_t::thread_local_storage_bn_ctx()) > 0);

  ecc_point_t Q;
  if (rv = Q.from_oct(curve, oct)) return rv;
  if (rv = curve.check(Q)) return rv;

  bn_t u1, u2;
  MODULO(q) {
    const bn_t w = q.inv(s);
    u1 = e * w;
    u2 = r * w;
  }

  vartime_scope_t vartime_scope;
  const ecc_point_t R = curve.mul_add(u1, Q, u2);
  if (R.is_infinity()) return coinbase::error(E_CRYPTO, "ossl_ecdsa_verify: R is infinity");
  if (q.mod(R.get_x()) != r) return coinbase::error(E_CRYPTO, "ossl_ecdsa_verify: invalid signature");
  return SUCCESS;
}
