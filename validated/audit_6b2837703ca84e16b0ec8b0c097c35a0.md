### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Inject Arbitrary Sequencer Address into Block Execution — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit.builder` is a proposer-supplied field that is forwarded verbatim as `sequencer_address` into the `BlockInfo` used for block execution. The validator's `is_proposal_init_valid` function never checks `init.builder` against any locally-trusted reference. A single malicious proposer can set `builder` to any arbitrary `ContractAddress`, causing every validating node to execute the block with the wrong sequencer address, redirect all transaction fees to an attacker-controlled address, corrupt the `get_sequencer_address()` syscall result seen by every contract in the block, and commit a block hash that encodes the wrong sequencer.

---

### Finding Description

`ProposalInit` carries a `builder` field (the sequencer/block-builder address) alongside the gas prices, timestamp, and other block-metadata fields. [1](#0-0) 

When a validator receives a proposal, `is_proposal_init_valid` is called to check the incoming `ProposalInit`. It enforces `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas-price fields (within a margin), `starknet_version`, `version_constant_commitment`, `fee_proposal_fri`, and `timestamp`. It does **not** check `init.builder` at all. [2](#0-1) 

After passing `is_proposal_init_valid`, the unchecked `init` is forwarded to `convert_to_sn_api_block_info`, which maps `init.builder` directly to `sequencer_address` in the `BlockInfo` that drives execution: [3](#0-2) 

That `BlockInfo` is passed to `batcher.validate_block` via `initiate_validation`: [4](#0-3) 

The batcher executes all transactions with the attacker-supplied `sequencer_address`, computes `PartialBlockHashComponents` (which includes `sequencer`) from that same `BlockInfo`, and returns a `PartialBlockHash` that encodes the attacker's address: [5](#0-4) [6](#0-5) 

The proposer, who chose `builder`, also computed `ProposalFin.proposal_commitment` with the same address, so the final commitment comparison in `validate_proposal` passes: [7](#0-6) 

The proposal is accepted, stored in `valid_proposals`, and eventually committed. The `ProposalInitValidation` struct that carries the validator's locally-trusted reference values has no `builder` field, confirming there is no path by which the validator could enforce a correct value: [8](#0-7) 

---

### Impact Explanation

**Critical — Wrong state and economic impact.**

1. **Fee theft**: All transaction fees in the block are credited to `sequencer_address`. With an attacker-controlled `builder`, every fee in the block is transferred to the attacker's address, not the legitimate sequencer's address. This is a direct, irreversible economic loss.

2. **Wrong `get_sequencer_address()` syscall result**: Every Cairo contract that calls the `get_sequencer_address` syscall during this block receives the attacker-supplied address. Any contract logic that branches on the sequencer address (access control, fee routing, oracle guards) will execute incorrectly.

3. **Wrong block hash committed on-chain**: `sequencer` is a named field in `PartialBlockHashComponents` and is hashed into the `PartialBlockHash` / final `BlockHash`. The committed block hash encodes the wrong sequencer address permanently. [9](#0-8) 

---

### Likelihood Explanation

Any validator that is scheduled as the proposer for a round can exploit this. No additional privilege is required. The attack requires only setting `init.builder` to an arbitrary address when constructing `ProposalInit` in `initiate_build`. All other validators will accept the proposal because `is_proposal_init_valid` never checks `builder`. The attack is deterministic and requires no brute-force or timing. [10](#0-9) 

---

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`. Each validator node already knows its own configured `builder_address`; that value should be injected into `ProposalInitValidation` at construction time (in `validate_proposal` / `validate_current_round_proposal`) and compared with `init_proposed.builder` using an exact-match check, the same way `height`, `l1_da_mode`, and `l2_gas_price_fri` are enforced. [11](#0-10) 

---

### Proof of Concept

1. A malicious validator node is scheduled as the proposer for height H.
2. In `initiate_build`, it sets `builder: ContractAddress::from(ATTACKER_ADDRESS)` instead of its legitimate `builder_address`.
3. It broadcasts `ProposalPart::Init(init)` with the tampered `builder` field.
4. Every honest validator calls `is_proposal_init_valid` — the function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, L1 gas prices, `starknet_version`, `version_constant_commitment`, `fee_proposal_fri`, and `timestamp`. None of these checks involve `builder`. Validation passes.
5. `convert_to_sn_api_block_info` maps `init.builder` → `sequencer_address: ATTACKER_ADDRESS` and passes it to `batcher.validate_block`.
6. The batcher executes all transactions with `sequencer_address = ATTACKER_ADDRESS`. All fees are credited to `ATTACKER_ADDRESS`. All `get_sequencer_address()` syscalls return `ATTACKER_ADDRESS`.
7. The batcher computes `PartialBlockHashComponents { sequencer: ATTACKER_ADDRESS, ... }` and returns a `PartialBlockHash` that encodes `ATTACKER_ADDRESS`.
8. The proposer's `ProposalFin.proposal_commitment` was computed with the same `ATTACKER_ADDRESS`, so `built_block == received_fin.proposal_commitment` — the commitment check passes.
9. The proposal is accepted by all honest validators, reaches BFT decision, and is committed. The block is finalized with `ATTACKER_ADDRESS` as the sequencer, all fees transferred to `ATTACKER_ADDRESS`, and the block hash encoding `ATTACKER_ADDRESS`.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
