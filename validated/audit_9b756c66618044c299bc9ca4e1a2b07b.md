### Title
`GlobalContractDistributionReceipt::V1` Bypasses Nonce Freshness Check, Enabling Stale Contract Code Overwrite Across Shards - (File: `runtime/runtime/src/global_contracts.rs`)

### Summary
`GlobalContractDistributionReceipt::V1` receipts carry no nonce field and return a hardcoded `0` from `nonce()`. The `check_and_update_nonce` function rejects a receipt only when `incoming_nonce < stored_nonce`. Because both the V1 receipt nonce and the initial stored nonce are `0`, the condition `0 < 0` is always false, so every V1 receipt passes the freshness check unconditionally. After processing, `set_nonce` writes `0` back, leaving `stored_nonce = 0` permanently. Any number of V1 receipts for the same contract identifier can therefore all be applied in sequence, with no staleness protection.

### Finding Description
The nonce-based idempotency system was added to prevent stale `GlobalContractDistributionReceipt` receipts from overwriting newer contract code. The system has two receipt versions:

- **V1** (`GlobalContractDistributionReceiptV1`): no nonce field; `nonce()` returns the hardcoded constant `0`.
- **V2** (`GlobalContractDistributionReceiptV2`): carries an explicit `nonce` field set by `increment_nonce` at deploy time. [1](#0-0) 

The freshness gate in `apply_distribution_current_shard` is: [2](#0-1) 

For a V1 receipt:
- `incoming_nonce = global_contract_data.nonce()` → `0`
- `stored_nonce = get_nonce(...)` → `0` (initial state; `get_nonce` returns `0` when no key exists)
- Guard: `0 < 0` → **false** → receipt is accepted
- `set_nonce(state_update, nonce_key, 0)` → stored nonce remains `0`

Every subsequent V1 receipt for the same identifier repeats this cycle: `stored_nonce` never advances past `0`, so the guard never fires. This is the exact analog of the deprecated Chainlink `latestAnswer` API: a "freshness" field that is always `0` and is never checked against a meaningful threshold. [3](#0-2) 

The nonce upgrade was introduced to fix stale-overwrite races. The CHANGELOG confirms V1 receipts predate this fix: [4](#0-3) 

V1 receipts created before the upgrade can still reside in the delayed-receipt queue on any shard. When they drain, the freshness check is silently bypassed.

### Impact Explanation
An unprivileged user who deployed the same `AccountId`-mode global contract multiple times before the nonce upgrade will have multiple V1 `GlobalContractDistributionReceipt` entries in the delayed-receipt queue. Because `forward_distribution_next_shard` is called unconditionally after `apply_distribution_current_shard` (even when the nonce check returns stale), each V1 receipt propagates hop-by-hop across all shards: [5](#0-4) 

Different shards drain their delayed queues independently and may process the V1 receipts in different orders. The result is that `TrieKey::GlobalContractCode` for the same `AccountId` identifier ends up holding **different bytecode on different shards**. Accounts on shard A execute contract version V1; accounts on shard B execute version V2. Cross-shard calls that depend on a consistent global contract produce divergent execution outcomes, corrupting the per-shard state root for the `GlobalContractCode` trie entry.

### Likelihood Explanation
The global contracts feature shipped with V1 receipts before the nonce idempotency fix. Any mainnet or testnet deployment that processed `DeployGlobalContract` actions in `AccountId` mode before the V2 upgrade has V1 receipts potentially still in the delayed queue. No special privilege is required; the receipts were created by ordinary user transactions and are processed automatically by the runtime.

### Recommendation
In `check_and_update_nonce`, treat V1 receipts (identified by `global_contract_data.maybe_nonce() == None`) as unconditionally stale if `stored_nonce > 0` (a V2 receipt has already been applied), and as a one-time install if `stored_nonce == 0` but immediately advance `stored_nonce` to a sentinel value (e.g., `u64::MAX` or a dedicated "V1 applied" marker) so that no second V1 receipt can pass. Alternatively, gate the entire `apply_distribution_current_shard` path on the receipt being V2 when the `GlobalContractNonce` feature is active.

### Proof of Concept
1. Before the nonce upgrade, user submits two `DeployGlobalContract` (AccountId mode) transactions for the same account, producing V1 receipts R1 (code_v1) and R2 (code_v2) in the delayed queue on shard S.
2. After the upgrade, shard S drains its queue. R1 is processed: `stored_nonce=0`, `incoming_nonce=0`, `0 < 0` is false → accepted, `GlobalContractCode` set to `code_v1`, `stored_nonce` written as `0`.
3. R2 is processed: same check, same result → accepted, `GlobalContractCode` overwritten with `code_v2`, `stored_nonce` still `0`.
4. Meanwhile, on shard T, the forwarded copies arrive in reverse order (R2 then R1): `GlobalContractCode` ends up as `code_v1`.
5. Shard S has `code_v2`; shard T has `code_v1`. Accounts on both shards reference the same `AccountId`-mode global contract but execute different bytecode, producing divergent state roots for `TrieKey::GlobalContractCode`. [6](#0-5) [7](#0-6)

fn apply_distribution_current_shard(
    receipt: &Receipt,
    global_contract_data: &GlobalContractDistributionReceipt,
    apply_state: &ApplyState,
    state_update: &mut TrieUpdate,
) -> Result<Compute, RuntimeError> {
    let identifier = match &global_contract_data.id() {
        GlobalContractIdentifier::CodeHash(hash) => GlobalContractCodeIdentifier::CodeHash(*hash),
        GlobalContractIdentifier::AccountId(account_id) => {
            GlobalContractCodeIdentifier::AccountId(account_id.clone())
        }
    };

    let is_nonce_fresh = check_and_update_nonce(global_contract_data, &identifier, state_update)?;
    if !is_nonce_fresh {
        return Ok(0);
    }

    let config = apply_state.config.wasm_config.clone();
    let trie_key = TrieKey::GlobalContractCode { identifier };
    let code_len = global_contract_data.code().len() as u64;
    state_update.set(trie_key, global_contract_data.code().to_vec());
    state_update.commit(StateChangeCause::ReceiptProcessing { receipt_hash: receipt.get_hash() });
    let code_hash = match global_contract_data.id() {
        GlobalContractIdentifier::CodeHash(hash) => Some(*hash),
        GlobalContractIdentifier::AccountId(_) => None,
    };
    precompile_contract_with_warming(
        &ContractCode::new(global_contract_data.code().to_vec(), code_hash),
        config,
        apply_state.next_wasm_config.clone(),
        apply_state.cache.as_deref(),
    );
    near_vm_runner::report_metrics(apply_state.shard_id, "global_contract");
    let fees = &apply_state.config.fees;
    let per_byte_total = fees
        .deploy_global_contract_execution_per_byte
        .checked_mul(code_len)
        .ok_or(IntegerOverflowError)?;
    let compute = fees
        .deploy_global_contract_execution_base
        .checked_add(per_byte_total)
        .ok_or(IntegerOverflowError)?;
    Ok(compute)
}

fn check_and_update_nonce(
    global_contract_data: &GlobalContractDistributionReceipt,
    identifier: &GlobalContractCodeIdentifier,
    state_update: &mut TrieUpdate,
) -> Result<bool, RuntimeError> {
    let nonce_key = TrieKey::GlobalContractNonce { identifier: identifier.clone() };
    let stored_nonce = get_nonce(state_update, &nonce_key)?;
    let incoming_nonce = global_contract_data.nonce();

    // Allow the same nonce since the nonce is updated immediately when
    // initiating distribution to prevent multiple distributions with the same
    // nonce from being initiated.
    if incoming_nonce < stored_nonce {
        return Ok(false);
    }

    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)
}

// Retrieves the stored nonce for the given global contract identifier. If no
// nonce is stored, returns 0.
fn get_nonce(state_update: &TrieUpdate, nonce_key: &TrieKey) -> Result<u64, RuntimeError> {
    let stored_nonce = state_update.get(nonce_key, AccessOptions::DEFAULT)?;
    let Some(stored_nonce) = stored_nonce else {
        return Ok(0);
    };
    let stored_nonce: [u8; 8] =
        stored_nonce.try_into().expect("GlobalContractNonce should be 8 bytes");
    let stored_nonce = u64::from_le_bytes(stored_nonce);
    Ok(stored_nonce)
}
