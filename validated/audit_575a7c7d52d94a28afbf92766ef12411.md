### Title
Unelected Committee Member Votes Accepted by GOV Ledger Rule Before Protocol Version 11 — (File: `eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Gov.hs`)

---

### Summary

Before protocol version 11, the `GOV` ledger rule does not enforce the ban on unelected committee member votes. That check exists only in the `MEMPOOL` rule, which block producers can bypass by injecting transactions directly into a block. An unelected committee member who has authorized a hot key (via a pending `UpdateCommittee` proposal) can therefore have their vote recorded and counted toward governance action ratification, potentially enacting unauthorized governance actions.

---

### Finding Description

The vulnerability class from the external report is **access-control placed in the wrong layer**: the enforcement exists in one path (L2 / mempool) but is absent from the authoritative path (L1 / ledger rule), so the restriction can be bypassed by going directly to the authoritative path.

The exact same pattern appears in the Conway `GOV` rule.

**GOV rule — check is gated behind a hardfork flag (absent before PV 11):**

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Gov.hs
when (hardforkConwayDisallowUnelectedCommitteeFromVoting $ pp ^. ppProtocolVersionL) $
  failOnNonEmpty
    (unelectedCommitteeVoters committee committeeState gsVotingProcedures)
    (injectFailure . UnelectedCommitteeVoters)
``` [1](#0-0) 

**MEMPOOL rule — complementary check present only before PV 11:**

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Mempool.hs
unless (hardforkConwayDisallowUnelectedCommitteeFromVoting protVer) $
  -- This check can completely be removed once mainnet switches to protocol
  -- version 11, since the same check has been implemented in the GOV rule.
  ...
  failOnNonEmpty
    ( unelectedCommitteeVoters ... )
    (ConwayMempoolFailure ...)
``` [2](#0-1) 

The two flags are **complementary inverses**: the mempool rejects unelected votes when the GOV rule does not, and vice versa. This means the ledger rule has no protection before PV 11.

**Why unelected members can pass all other GOV checks:**

The `VotersDoNotExist` check uses `authorizedHotCommitteeCredentials committeeState` (all hot credentials, including those of potential future members), not `authorizedElectedHotCommitteeCredentials`:

```haskell
knownCommitteeMembers = authorizedHotCommitteeCredentials committeeState
...
CommitteeVoter hotCred -> CommitteeVoter <$> internSet hotCred knownCommitteeMembers
``` [3](#0-2) [4](#0-3) 

`checkVotersAreValid` calls `isCommitteeVotingAllowed`, which checks action-type eligibility and term expiry, but does **not** check whether the member is elected:

```haskell
CommitteeVoter {} -> isCommitteeVotingAllowed currentEpoch committeeState (gasAction gas)
``` [5](#0-4) 

**How an unelected member obtains an authorized hot key:**

`ConwayAuthCommitteeHotKey` is accepted for both current members and *potential future members* (cold credentials appearing in any pending `UpdateCommittee` proposal):

```haskell
isCurrentMember || isPotentialFutureMember
  ?! (injectFailure . ConwayCommitteeIsUnknown) coldCred
``` [6](#0-5) 

Once the `AuthCommitteeHotKey` certificate is processed, the hot credential is stored in `CommitteeState` and is returned by `authorizedHotCommitteeCredentials`, making the unelected member indistinguishable from an elected one to all GOV checks except the gated `unelectedCommitteeVoters` check.

---

### Impact Explanation

An unelected committee member's vote is recorded in `gasCommitteeVotes` and counted by the `committeeAccepted` / `committeeAcceptedRatio` logic in the `RATIFY` rule. If the unelected member's vote tips the committee acceptance ratio above the threshold, a governance action that would otherwise fail ratification is enacted. Affected action types include `HardForkInitiation`, `TreasuryWithdrawals`, `ParameterChange`, `NewConstitution`, and `UpdateCommittee`.

**Impact class:** Critical — unauthorized governance, treasury, protocol-parameter, committee, constitution, or hard-fork action enacted.

---

### Likelihood Explanation

The attack requires two cooperating parties:

1. **An unelected committee member** — any cold credential appearing in a pending `UpdateCommittee` proposal can submit `AuthCommitteeHotKey` and obtain a recognized hot credential.
2. **A block producer** — any block producer can include a transaction directly in a block, bypassing the mempool. No majority or consensus threshold is required; a single block producer suffices.

The block producer does not need to be the same entity as the unelected committee member. The unelected member broadcasts a signed voting transaction; any block producer who receives it (or is colluding) can include it. This is a realistic, low-barrier scenario on any Conway-era network running below PV 11.

---

### Recommendation

Remove the `hardforkConwayDisallowUnelectedCommitteeFromVoting` gate from the `GOV` rule's `unelectedCommitteeVoters` check, or apply it unconditionally from the first Conway epoch. The mempool-only enforcement is insufficient because block producers are not bound by mempool rules. The comment in `Mempool.hs` ("This check can completely be removed once mainnet switches to protocol version 11") confirms the design intent, but the transition window leaves the ledger rule unprotected.

---

### Proof of Concept

1. Submit an `UpdateCommittee` proposal adding cold credential `ccCold` as a new member.
2. `ccCold` submits `AuthCommitteeHotKey ccCold ccHot` — accepted because `ccCold` is a potential future member.
3. `ccHot` signs a voting transaction `VoteYes` on a `TreasuryWithdrawals` governance action.
4. A block producer (bypassing the mempool) includes this transaction directly in a block.
5. The `GOV` rule processes the transaction: `ccHot` is in `authorizedHotCommitteeCredentials`, passes `VotersDoNotExist`; `isCommitteeVotingAllowed` passes (treasury withdrawals allow committee votes); the `UnelectedCommitteeVoters` check is skipped (PV < 11).
6. The vote is recorded in `gasCommitteeVotes`.
7. At epoch boundary, `committeeAcceptedRatio` counts the unelected member's vote, the threshold is met, and the treasury withdrawal is enacted.
