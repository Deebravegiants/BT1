Audit Report

## Title
`RSETHPriceFeed.getRoundData` Returns Fabricated Historical Price by Mixing Historical ETH/USD with Current rsETH/ETH Rate - (File: contracts/oracles/RSETHPriceFeed.sol)

## Summary
`RSETHPriceFeed` implements `AggregatorV3Interface` and its `getRoundData` function fetches a historical ETH/USD price for a specific Chainlink round but multiplies it by the **current** `RS_ETH_ORACLE.rsETHPrice()` value. Because `LRTOracle` stores only a single scalar `rsETHPrice` with no historical snapshot mechanism, the function structurally cannot reconstruct the rsETH/ETH rate that was valid at the requested round, returning a price that never existed at that point in time.

## Finding Description
In `contracts/oracles/RSETHPriceFeed.sol` at lines 53–61, `getRoundData` calls `ETH_TO_USD.getRoundData(_roundId)` to retrieve the historical ETH/USD price for a past round, then immediately overwrites `answer` with `int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18`. `RS_ETH_ORACLE.rsETHPrice()` resolves to `LRTOracle.rsETHPrice`, declared at `contracts/LRTOracle.sol` line 28 as a plain `uint256` that is overwritten on every price update (line 313: `rsETHPrice = newRsETHPrice`). There is no mapping, array, or checkpoint structure keyed by round ID or timestamp. Consequently, for any round other than the most recent one, the rsETH/ETH multiplier applied is the present-day value, not the value that existed when that round was recorded. `latestRoundData` (lines 63–70) is correct because both components are current, but `getRoundData` is broken by design.

## Impact Explanation
The contract fails to deliver the historical price data it promises to implement per `AggregatorV3Interface`. Any external protocol on Morph that integrates this feed and calls `getRoundData` for historical price verification — circuit-breaker checks, price-deviation guards, liquidation baselines — will receive a fabricated answer. Because rsETH/ETH monotonically increases over time (staking reward accrual), the error is consistently inflationary: historical prices are always overstated relative to what they actually were. This matches the allowed impact: **Low — Contract fails to deliver promised returns, but doesn't lose value directly within LRT-rsETH itself.**

## Likelihood Explanation
`RSETHPriceFeed` is deployed on Morph and is publicly callable by any address without preconditions. Any external protocol that integrates it and calls `getRoundData` is immediately affected. No privileged role, special setup, or attacker action is required — the incorrect behavior is triggered by a standard read call to a public view function. The magnitude of the error grows over time as rsETH/ETH appreciates.

## Recommendation
Since `LRTOracle` has no historical rsETH price snapshots, `getRoundData` cannot be correctly implemented. The safest fix is to revert explicitly, preventing consumers from relying on incorrect data:

```solidity
function getRoundData(uint80 /*_roundId*/)
    external
    pure
    override
    returns (uint80, int256, uint256, uint256, uint80)
{
    revert("RSETHPriceFeed: historical round data unavailable");
}
```

Alternatively, add a historical rsETH/ETH price snapshot mechanism to `LRTOracle` (keyed by Chainlink round ID or block timestamp) so `getRoundData` can apply the correct rsETH/ETH rate for the requested round.

## Proof of Concept
1. At time T0, `LRTOracle.rsETHPrice` = 1.02e18 (rsETH/ETH). Chainlink ETH/USD round R0 records ETH = $3000. Correct rsETH/USD at R0 = 1.02 × $3000 = $3060.
2. Time passes; staking rewards accrue. `LRTOracle.rsETHPrice` is updated to 1.05e18. Chainlink advances to round R1.
3. A consumer calls `RSETHPriceFeed.getRoundData(R0)`:
   - `ETH_TO_USD.getRoundData(R0)` returns `answer = 3000e8` (correct historical ETH/USD).
   - Line 60 computes: `answer = int256(1.05e18) * 3000e8 / 1e18` = **3150e8**.
4. The returned historical price is **$3150**, not the correct **$3060** — a 3% inflation.
5. Any protocol comparing the current rsETH/USD price against this inflated historical baseline will underestimate true price appreciation, potentially suppressing circuit-breaker triggers or mispricing collateral.
