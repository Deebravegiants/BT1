### Title
Unbacked wrsETH from Pool Minting Drains Wrapper's altRsETH Reserves, Causing Protocol Insolvency - (File: contracts/L2/RsETHTokenWrapper.sol)

---

### Summary

`RsETHTokenWrapper.mint()` is callable by any `MINTER_ROLE` holder (e.g., `RSETHPoolV2`, `RSETHPoolV3`, `RSETHPoolV2ExternalBridge`, `RSETHPoolV3ExternalBridge`) and mints wrsETH without depositing any altRsETH collateral into the wrapper. Because `_withdraw()` imposes no solvency check — it only verifies the caller holds enough wrsETH to burn — a user who received wrsETH from a pool deposit can immediately redeem it for altRsETH that was deposited by other users, leaving those users with unbacked wrsETH they can never redeem.

---

### Finding Description

**Minting path (no collateral deposited into wrapper):**

Every pool's `deposit()` function calls `wrsETH.mint(msg.sender, rsETHAmount)` directly: [1](#0-0) [2](#0-1) 

The ETH (or ERC-20 token) goes to the pool contract. Nothing is transferred to `RsETHTokenWrapper`. The wrapper's `mint()` function has no collateral requirement: [3](#0-2) 

**Withdrawal path (no solvency check):**

`_withdraw()` only checks that the asset is allowed and burns the caller's wrsETH, then transfers altRsETH out of the wrapper: [4](#0-3) 

There is no check that `balanceOf(altRsETH) >= totalSupply()` after the withdrawal, nor any check that the caller's wrsETH was originally minted against deposited altRsETH.

**The invariant that breaks:**

`maxAmountToDepositBridgerAsset()` reveals the design intent — the wrapper is supposed to be fully collateralized (`balance >= totalSupply`): [5](#0-4) 

But because wrsETH is a fungible ERC-20, there is no on-chain mechanism to distinguish "pool-minted wrsETH" (backed by ETH in the pool, pending bridging) from "wrapper-deposited wrsETH" (backed by altRsETH already in the wrapper). A pool-minted wrsETH holder can freely redeem for altRsETH that was deposited by regular users.

---

### Impact Explanation

**Critical — Protocol insolvency / direct theft of user funds.**

Concrete state trace:

| Step | Action | Wrapper altRsETH balance | wrsETH totalSupply |
|------|--------|--------------------------|-------------------|
| 0 | User B calls `wrapper.deposit(altRsETH, 1000)` | 1000 | 1000 |
| 1 | User A calls `pool.deposit{value: X}()` → pool calls `wrsETH.mint(A, 500)` | 1000 | 1500 |
| 2 | User A calls `wrapper.withdraw(altRsETH, 500)` | 500 | 1000 |
| 3 | User B calls `wrapper.withdraw(altRsETH, 1000)` | **REVERTS** | — |

After step 2, User B holds 1000 wrsETH but the wrapper only holds 500 altRsETH. User B's 500 altRsETH has been permanently stolen by User A. The bridger's `depositBridgerAssets()` call (step 3 in the intended flow) cannot restore User B's funds because User A already drained them.

---

### Likelihood Explanation

**High.** The attack requires no special role, no governance capture, and no oracle manipulation. Any user who receives wrsETH via a pool deposit (a standard, publicly accessible function) can immediately call `wrapper.withdraw()` to drain altRsETH deposited by other users. The attack is profitable whenever the wrapper holds any altRsETH balance from regular `deposit()` calls, which is the normal operating state of the system.

---

### Recommendation

Enforce a collateralization invariant in `_withdraw()`. Before transferring altRsETH out, verify that the wrapper's remaining balance of that asset is sufficient to back all remaining wrsETH supply:

```solidity
function _withdraw(address _asset, address _to, uint256 _amount) internal {
    if (!allowedTokens[_asset]) revert TokenNotAllowed();
    _burn(msg.sender, _amount);
    // Ensure wrapper remains solvent after withdrawal
    uint256 remainingBalance = ERC20Upgradeable(_asset).balanceOf(address(this)) - _amount;
    if (remainingBalance < totalSupply()) revert InsufficientCollateral();
    ERC20Upgradeable(_asset).safeTransfer(_to, _amount);
    emit Withdraw(_asset, msg.sender, _to, _amount);
}
```

Alternatively, track per-asset collateral separately from pool-minted supply, or require the bridger to fully collateralize the wrapper before any withdrawals are permitted.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

import "forge-std/Test.sol";
import {RsETHTokenWrapper} from "contracts/L2/RsETHTokenWrapper.sol";
import {RSETHPoolV2} from "contracts/pools/RSETHPoolV2.sol";
import {MockAltRsETH} from "./mocks/MockAltRsETH.sol"; // standard ERC20 mock

contract DrainWrapperTest is Test {
    RsETHTokenWrapper wrapper;
    RSETHPoolV2 pool;
    MockAltRsETH altRsETH;

    address admin   = address(0xA);
    address bridger = address(0xB);
    address userA   = address(0x1); // attacker (uses pool)
    address userB   = address(0x2); // victim (deposited altRsETH)

    function setUp() public {
        altRsETH = new MockAltRsETH();
        wrapper  = new RsETHTokenWrapper();
        wrapper.initialize(admin, bridger, address(altRsETH));

        // Deploy pool with MINTER_ROLE on wrapper
        pool = new RSETHPoolV2();
        pool.initialize(admin, bridger, address(wrapper), 0, address(mockOracle));
        vm.prank(admin);
        wrapper.grantRole(wrapper.MINTER_ROLE(), address(pool));

        // Fund userB with altRsETH
        altRsETH.mint(userB, 1000e18);
    }

    function testDrainAltRsETH() public {
        // Step 1: userB deposits 1000 altRsETH into wrapper
        vm.startPrank(userB);
        altRsETH.approve(address(wrapper), 1000e18);
        wrapper.deposit(address(altRsETH), 1000e18);
        vm.stopPrank();

        assertEq(altRsETH.balanceOf(address(wrapper)), 1000e18);
        assertEq(wrapper.totalSupply(), 1000e18);

        // Step 2: userA deposits ETH into pool → pool mints 500 wrsETH for userA
        vm.deal(userA, 1 ether);
        vm.prank(userA);
        pool.deposit{value: 1 ether}("ref"); // mints ~500e18 wrsETH (oracle-dependent)

        uint256 userABalance = wrapper.balanceOf(userA);
        assertGt(userABalance, 0);

        // Step 3: userA drains altRsETH from wrapper
        vm.prank(userA);
        wrapper.withdraw(address(altRsETH), userABalance);

        // Step 4: userB can no longer redeem their altRsETH
        vm.prank(userB);
        vm.expectRevert(); // ERC20: transfer amount exceeds balance
        wrapper.withdraw(address(altRsETH), 1000e18);

        // Assert insolvency: wrapper balance < totalSupply
        assertLt(
            altRsETH.balanceOf(address(wrapper)),
            wrapper.totalSupply(),
            "Wrapper is insolvent: altRsETH balance < wrsETH totalSupply"
        );
    }
}
```

    function maxAmountToDepositBridgerAsset(address _asset) public view returns (uint256) {
        if (!allowedTokens[_asset]) return 0;

        // get totalSupply of wrsETH minted
        uint256 wrsETHSupply = totalSupply();
        // balance of _asset with the contract
