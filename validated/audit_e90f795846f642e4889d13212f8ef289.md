### Title
Single Reverting Price Oracle in `_getTotalEthInProtocol()` Loop Permanently Blocks `updateRSETHPrice()` — (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle._getTotalEthInProtocol()` iterates over every supported asset and makes an external call to each asset's price oracle. If any single oracle reverts (e.g., due to being paused by external governance, deprecated, or returning out-of-range data), the entire `updateRSETHPrice()` call reverts. Because `updateRSETHPrice()` is the sole mechanism for updating the rsETH exchange rate and minting protocol fee yield, a single broken oracle permanently freezes protocol fee accrual and stales the rsETH price until admin intervention.

### Finding Description
In `LRTOracle._getTotalEthInProtocol()`, all supported assets are iterated and `getAssetPrice(asset)` is called for each one:

```solidity
// LRTOracle.sol L336-L348
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);          // external call to IPriceFetcher
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    unchecked { ++assetIdx; }
}
```

`getAssetPrice` delegates to the registered `IPriceFetcher` for each asset:

```solidity
// LRTOracle.sol L156-L158
function getAssetPrice(address asset) public view onlySupportedOracle(asset) returns (uint256) {
    return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
}
```

The registered price fetchers include `ChainlinkPriceOracle` (calls `latestRoundData()` on a Chainlink aggregator), `EthXPriceOracle` (calls `IETHXStakePoolsManager.getExchangeRate()`), and `SwETHPriceOracle` (calls `ISwETH.getRate()`). Any of these external calls can revert if the underlying feed is deprecated, the external protocol is paused by its own governance, or the aggregator is decommissioned.

`_getTotalEthInProtocol()` is called exclusively by `_updateRsETHPrice()`, which is called by both the public `updateRSETHPrice()` and the manager-only `updateRSETHPriceAsManager()`. There is no per-asset error handling or skip logic. A single reverting oracle causes the entire price update to revert.

Consequences of a blocked `updateRSETHPrice()`:
1. **Protocol fee minting is frozen.** The fee-minting logic inside `_updateRsETHPrice()` never executes, so accrued yield is never converted to rsETH for the treasury.
2. **rsETH price becomes stale.** All deposits (`depositAsset`, `depositETH`) and withdrawal initiations (`initiateWithdrawal`) use the stored `rsETHPrice` value. A stale price causes incorrect rsETH minting and incorrect asset-to-rsETH conversion for withdrawals.
3. **Price-drop auto-pause is disabled.** The downside-protection logic that auto-pauses the protocol on a large price drop cannot trigger.

While the admin can fix this by calling `updatePriceOracleFor()` or `removeSupportedAsset()`, there is a window of vulnerability during which yield is frozen and the price is stale.

### Impact Explanation
**Medium.** Protocol fee yield accrual is permanently frozen until admin intervention. The stored `rsETHPrice` becomes stale, causing all deposits and withdrawal initiations to use an incorrect exchange rate. This maps to "Permanent freezing of unclaimed yield" and "Contract fails to deliver promised returns."

### Likelihood Explanation
**Low.** The price oracles are set by LRT admin. However, the underlying feeds (Chainlink aggregators, Stader's `ETHXStakePoolsManager`, Swell's `SwETH`) are governed by external protocols. Any of these can be paused, deprecated, or decommissioned without LRT-rsETH governance action, triggering the DoS without any admin compromise on the LRT-rsETH side.

### Recommendation
1. Wrap each `getAssetPrice(asset)` call in a `try/catch` inside `_getTotalEthInProtocol()` and skip or use a fallback price for assets whose oracle reverts, similar to the fix applied in the referenced PR 401 (low-level call with gas cap).
2. Alternatively, allow the oracle loop to be called with a subset of assets so a single broken oracle can be excluded while the price update proceeds for the rest.
3. Emit an event when an oracle call fails so operators are alerted immediately.

### Proof of Concept

1. Admin adds `assetX` with oracle `OracleX` (e.g., a Chainlink feed).
2. The external Chainlink feed for `assetX` is deprecated; `latestRoundData()` begins reverting.
3. Any call to `updateRSETHPrice()` (public, callable by anyone) now reverts at `_getTotalEthInProtocol()` → `getAssetPrice(assetX)` → `OracleX.getAssetPrice(assetX)` → `priceFeed.latestRoundData()` (reverts).
4. `rsETHPrice` is never updated; protocol fee rsETH is never minted.
5. All new deposits and withdrawal initiations use the stale `rsETHPrice`, producing incorrect rsETH amounts.
6. The price-drop auto-pause cannot trigger regardless of actual TVL changes.
7. State remains frozen until admin calls `updatePriceOracleFor(assetX, newOracle)` or `removeSupportedAsset(assetX, idx)`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

```text
    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```
