Audit Report

## Title
`SfrxETHPriceOracle` Returns sfrxETH/frxETH Rate Instead of sfrxETH/ETH, Inflating rsETHPrice and Bypassing Downside Protection — (`contracts/oracles/SfrxETHPriceOracle.sol`)

## Summary

`SfrxETHPriceOracle.getAssetPrice()` returns `ISfrxETH.pricePerShare()`, which is the ERC-4626 vault share price denominated in its underlying asset frxETH — not in ETH. The interface comment confirms this: "How much frxETH is 1E18 sfrxETH worth." When frxETH trades below ETH parity, the oracle overstates sfrxETH's ETH value, inflating `rsETHPrice` and silently bypassing the protocol's downside-protection pause mechanism, which only triggers on a computed price *decrease* — a condition that never fires when the oracle inflates TVL upward.

## Finding Description

`SfrxETHPriceOracle.getAssetPrice()` at `contracts/oracles/SfrxETHPriceOracle.sol` L35–41 returns the raw vault share price:

```solidity
return ISfrxETH(sfrxETHContractAddress).pricePerShare();
```

The local `ISfrxETH` interface at L8–11 documents the unit: "How much frxETH is 1E18 sfrxETH worth." This is the sfrxETH→frxETH conversion rate. The parenthetical "Price is in ETH, not USD" is the Frax team's embedded assumption that frxETH ≡ 1 ETH — it does not mean the function queries an ETH market price.

This value is consumed without adjustment in `LRTOracle._getTotalEthInProtocol()` at `contracts/LRTOracle.sol` L339–343:

```solidity
uint256 assetER = getAssetPrice(asset);          // sfrxETH/frxETH, not sfrxETH/ETH
uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
```

The inflated `totalETHInProtocol` flows into `_updateRsETHPrice()` at L250:

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
```

**Downside-protection bypass.** The pause guard at L270–281 only fires when `newRsETHPrice < highestRsethPrice`. When frxETH depegs, the oracle inflates TVL, so `newRsETHPrice` appears to *increase* relative to `highestRsethPrice`. The guard never triggers; the protocol continues operating with a false price.

**Timing-window exploit.** `rsETHPrice` is a stored value updated lazily by `updateRSETHPrice()`. Between a frxETH depeg event and the next price update call, `getAssetPrice(sfrxETH)` already reflects the inflated rate while `rsETHPrice` still holds the old correct value. `LRTDepositPool.getRsETHAmountToMint()` at `contracts/LRTDepositPool.sol` L520 computes:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

With inflated numerator and correct denominator, a depositor receives more rsETH than the true ETH value of their sfrxETH warrants, diluting all existing rsETH holders.

No privileged role is required. `depositAsset()` is a public, permissionless function callable by any address.

## Impact Explanation

**Critical — Protocol Insolvency.**

Two concrete harms:

1. **Timing-window theft from existing holders.** Any depositor who calls `depositAsset(sfrxETH, …)` after frxETH depegs but before `updateRSETHPrice()` is called receives excess rsETH. The excess is backed by no real ETH, diluting every existing rsETH holder's redemption value.

2. **Persistent rsETHPrice inflation + downside-protection bypass.** Once `updateRSETHPrice()` is called post-depeg, `rsETHPrice` is set to an inflated value. The downside-protection mechanism — which would otherwise pause deposits and withdrawals — never fires because the computed price appears to have risen. The protocol continues accepting deposits and processing withdrawals against a price that overstates true ETH backing, compounding insolvency with each subsequent `updateRSETHPrice()` call while frxETH remains depegged.

## Likelihood Explanation

frxETH is not a USD stablecoin; it is an ETH-pegged LST whose peg is maintained by Curve pool liquidity and Frax redemption mechanics. It has traded below 0.99 ETH during periods of Curve pool imbalance and broader DeFi stress. No attacker action is required to cause the depeg — the vulnerability activates automatically whenever frxETH trades below parity. The timing-window exploit requires only that a depositor act before the next `updateRSETHPrice()` call, which is a public function with no access control. The SECURITY.md exclusion for "depegging of an external stablecoin" does not apply: frxETH is an LST, not a stablecoin.

## Recommendation

Replace the single `pricePerShare()` call with a two-step calculation in `SfrxETHPriceOracle.getAssetPrice()`:

1. Obtain the sfrxETH/frxETH rate: `uint256 sfrxEthToFrxEth = ISfrxETH(sfrxETHContractAddress).pricePerShare();`
2. Obtain the frxETH/ETH market rate from a Chainlink `frxETH/ETH` feed or a Curve TWAP: `uint256 frxEthToEth = IChainlinkAggregator(frxEthEthFeed).latestAnswer();`
3. Return `sfrxEthToFrxEth * frxEthToEth / 1e18`.

This mirrors the fix recommended for the analogous `SfrxEth` derivative in the Asymmetry protocol audit.

## Proof of Concept

**Preconditions:**
- frxETH/ETH market rate = 0.95 (Curve pool imbalance; no attacker action required)
- sfrxETH `pricePerShare()` = 1.05e18 (5% staking rewards accrued)
- Protocol holds 1 000 sfrxETH; rsETH supply = 1 000; `rsETHPrice` = 1.0e18 (last stored value, not yet updated)

**Step 1 — Attacker calls `depositAsset(sfrxETH, 100e18, 0, "")`:**
- `getAssetPrice(sfrxETH)` → 1.05e18 (inflated; frxETH depeg not reflected)
- `rsETHPrice` → 1.0e18 (stale correct value)
- `rsethAmountToMint = 100e18 * 1.05e18 / 1.0e18 = 105e18`
- True ETH value of 100 sfrxETH = 100 × 0.9975 = 99.75 ETH
- True rsETH due at correct rsETHPrice (0.9975) = 99.75 / 0.9975 = 100 rsETH
- **Attacker receives 5 excess rsETH backed by no real ETH.**

**Step 2 — Anyone calls `updateRSETHPrice()`:**
- Protocol now holds 1 100 sfrxETH; rsETH supply = 1 105
- Oracle TVL = 1 100 × 1.05 = 1 155 ETH (inflated); true TVL = 1 100 × 0.9975 = 1 097.25 ETH
- `newRsETHPrice` = 1 155 / 1 105 ≈ 1.0453e18 (inflated > `highestRsethPrice`)
- Downside-protection branch (`newRsETHPrice < highestRsethPrice`) is **never entered**; no pause occurs
- `rsETHPrice` is set to 1.0453e18; true backing per rsETH ≈ 0.9929 ETH — protocol is insolvent
