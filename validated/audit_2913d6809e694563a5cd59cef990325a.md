### Title
Unbounded `fee_proposal_fri` Accepted During Startup Window Poisons `fee_actual` and Future L2 Gas Price — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` explicitly skips the `fee_proposal_fri` bounds check when `fee_actual` is `None` (the first `window_size` blocks after V0_14_3 activation). A malicious proposer selected during this window can set `fee_proposal_fri` to any value — including `u128::MAX` or `0` — without rejection. That value is committed to consensus, stored in `fee_proposals_window`, and becomes an input to the median that determines `fee_actual` for all subsequent blocks, permanently distorting the L2 gas price.

---

### Finding Description

In `is_proposal_init_valid`, the bounds check on `fee_proposal_fri` is gated on `fee_actual` being `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

When `fee_actual` is `None` — which is the case for every block in the first `window_size`-block window after V0_14_3 activates — the entire `if let` body is skipped. No lower or upper bound is applied to the proposer-supplied `fee_proposal_fri`.

`fee_actual` is computed by `compute_fee_actual`, which returns `None` whenever the sliding window has not yet accumulated `window_size` entries: [2](#0-1) 

The `ProposalInitValidation` struct passed to the validator carries this `None` directly: [3](#0-2) 

After the proposal is accepted, `finalize_decision` unconditionally records the proposer-supplied value into the window: [4](#0-3) 

`compute_fee_actual` then computes the median of all stored values, including the injected extreme: [5](#0-4) 

The corrupted `fee_actual` is then used as the reference for all future `fee_proposal` bounds checks and as an input to `calculate_next_l2_gas_price_for_fin`, which sets the L2 gas price for every subsequent block.

Additionally, the proposer-supplied `fee_proposal_fri` is forwarded verbatim to the centralized cende pipeline: [6](#0-5) 

---

### Impact Explanation

**Corrupted value**: `fee_actual` for all blocks after the startup window fills, computed as the median of a window that contains one or more attacker-injected extreme values.

**Economic impact path**:
1. Attacker sets `fee_proposal_fri = u128::MAX` (or `0`) in `ProposalInit` during the startup window.
2. `is_proposal_init_valid` skips the bounds check; the proposal is accepted by all validators.
3. `record_fee_proposal(height, Some(u128::MAX))` stores the extreme value in `fee_proposals_window`.
4. Once the window fills, `fee_actual = median(window)` is inflated (or deflated to zero).
5. All future `fee_proposal` bounds are anchored to this corrupted `fee_actual` via `fee_proposal_bounds`, so the distortion persists for at least `window_size` additional blocks even with fully honest subsequent proposers.
6. `calculate_next_l2_gas_price_for_fin` uses `fee_actual` as a floor, propagating the inflated price into every block's `l2_gas_price`.

Setting `fee_proposal_fri = u128::MAX` makes all L2 transactions prohibitively expensive (economic DoS). Setting it to `0` collapses the base fee, enabling fee-free transaction flooding.

This matches the allowed impact: **Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

---

### Likelihood Explanation

The startup window is a one-time event when V0_14_3 activates. Any validator selected as proposer during those `window_size` blocks can trigger this. Proposer selection is a normal consensus role — no special privilege beyond validator membership is required. A single malicious proposer block is sufficient to shift the median if the window is small; controlling a minority of blocks in the window is sufficient for a larger window. The window length is determined by `VersionedConstants::latest_constants().fee_proposal_window_size`, which the convergence tests suggest is on the order of hundreds of blocks, giving an attacker a wide opportunity.

---

### Recommendation

**Short term**: During the startup window, validate `fee_proposal_fri` against a locally-derived fallback reference instead of skipping the check entirely. The honest proposer already uses `l2_gas_price` as its fallback when `fee_actual` is `None`: [7](#0-6) 

The validator should apply the same fallback: when `fee_actual` is `None`, check `fee_proposal_fri` against `[lower_bound(l2_gas_price), upper_bound(l2_gas_price)]` rather than accepting any value unconditionally.

**Long term**: Enforce a hard absolute cap on `fee_proposal_fri` (e.g., reject values above `MAX_REASONABLE_GAS_PRICE`) regardless of whether `fee_actual` is available, to prevent catastrophic poisoning of the fee window.

---

### Proof of Concept

1. V0_14_3 activates at block `N`. `fee_proposals_window` is empty; `fee_actual = None` for all blocks `N` through `N + window_size - 1`.
2. Malicious validator is selected as proposer for block `N` (normal consensus rotation).
3. Proposer constructs `ProposalInit { fee_proposal_fri: Some(GasPrice(u128::MAX)), starknet_version: V0_14_3, ... }`.
4. Validator node calls `is_proposal_init_valid`. `proposal_init_validation.fee_actual` is `None`. The `if let (Some(fee_actual), Some(fee_proposal))` arm does not match; no bounds check executes. Validation returns `Ok(())`. [1](#0-0) 

5. `initiate_validation` sends `ValidateBlockInput` to the batcher. The batcher does not receive or validate `fee_proposal_fri`.
6. `handle_proposal_part` computes `batcher_block_commitment = proposal_commitment_from(partial_block_hash, Some(u128::MAX))`. The proposer's `ProposalFin.proposal_commitment` was computed with the same value, so the equality check passes. [8](#0-7) 

7. `validate_proposal` returns `Ok(batcher_block_commitment)`. Consensus votes on this commitment.
8. `decision_reached` is called. `finalize_decision` calls `record_fee_proposal(N, Some(GasPrice(u128::MAX)))`. [4](#0-3) 

9. At block `N + window_size`, `compute_fee_actual` computes the median of the window. With even one entry of `u128::MAX` and the rest at the honest `l2_gas_price`, the median is significantly elevated. With a majority of attacker-controlled blocks, the median equals `u128::MAX`.
10. All subsequent blocks use this inflated `fee_actual` as the reference for `fee_proposal` bounds and as the floor for `l2_gas_price`, making the network economically unusable.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
