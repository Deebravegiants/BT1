### Title
MPC Key Share and Storage Encryption Key Exposed as Positional CLI Arguments - (File: `crates/node/src/cli.rs`)

### Summary
`ImportKeyshareCmd` and `ExportKeyshareCmd` accept the raw MPC key share JSON (containing `private_share`) and the local AES storage encryption key as **positional command-line arguments** with no environment-variable fallback. On Linux, `/proc/<pid>/cmdline` is world-readable by default, so any unprivileged user on the same host can read both secrets while the process is running.

### Finding Description

`ImportKeyshareCmd` is defined in `crates/node/src/cli.rs` with two sensitive fields declared as bare positional arguments — no `long`, no `env(...)`:

```rust
// crates/node/src/cli.rs lines 215-223
#[arg(
    help = "JSON string with the keyshare in format: {\"epoch\":1,\"private_share\":\"...\",\"public_key\":\"...\"}"
)]
pub keyshare_json: String,          // ← raw key share, positional

#[arg(help = "Hex-encoded 16 byte AES key for local storage encryption")]
pub local_encryption_key_hex: String,  // ← AES key, positional
``` [1](#0-0) 

`ExportKeyshareCmd` has the same problem for `local_encryption_key_hex`:

```rust
// crates/node/src/cli.rs lines 232-234
#[arg(help = "Hex-encoded 16 byte AES key for local storage encryption")]
pub local_encryption_key_hex: String,
``` [2](#0-1) 

Because neither field carries an `env(...)` attribute, clap has no environment-variable path; the values **must** be supplied on the command line. The invocation therefore looks like:

```bash
mpc-node import-keyshare \
    --home-dir /data \
    '{"epoch":1,"private_share":"<SECRET_SCALAR>","public_key":"..."}' \
    0123456789ABCDEF0123456789ABCDEF
```

Both secrets land verbatim in `/proc/<pid>/cmdline` for the lifetime of the process.

Contrast this with `StartCmd`, where secrets are correctly gated behind `env(...)` so they are never required on the command line:

```rust
// crates/node/src/cli.rs lines 80-81
#[arg(env("MPC_SECRET_STORE_KEY"))]
pub secret_store_key_hex: String,
``` [3](#0-2) 

The `SecretsStartConfig` struct even implements a custom `Debug` that redacts these values, showing the developers are aware of the sensitivity:

```rust
// crates/node-config/src/start.rs lines 131-143
impl std::fmt::Debug for SecretsStartConfig {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        f.debug_struct("SecretsStartConfig")
            .field("secret_store_key_hex", &"[REDACTED]")
            ...
    }
}
``` [4](#0-3) 

That same care was not applied to the import/export CLI arguments.

### Impact Explanation

**Critical — Unauthorized access to MPC key share material.**

`keyshare_json` contains the `private_share` scalar — the participant's actual secret share of the distributed MPC key. Reading it from `/proc/<pid>/cmdline` gives an attacker the raw key share without any authorization. Combined with shares from other compromised nodes (or in a low-threshold deployment), this directly enables threshold-signature forgery or full private-key reconstruction.

`local_encryption_key_hex` is the AES-128 key that encrypts the keyshare at rest. An attacker who captures it from the process list can decrypt the stored keyshare file on disk at any future time, providing persistent, silent access to the key share even after the import process exits.

### Likelihood Explanation

**Medium.** The `import-keyshare` command is a documented, operator-facing migration step (see `docs/node-migration-guide.md`). It is run on the same host as the MPC node. Any unprivileged user account on that host — including a compromised service account, a co-tenant in a shared VM, or a container with a shared PID namespace — can read `/proc/<pid>/cmdline` without any special privileges on a default Linux installation. The window is short (the process lifetime of the import command), but tools like `pspy` poll `/proc` continuously and will capture it reliably. [5](#0-4) 

### Recommendation

1. Add `env("MPC_KEYSHARE_JSON")` and `env("MPC_LOCAL_ENCRYPTION_KEY_HEX")` attributes to both fields in `ImportKeyshareCmd` and `ExportKeyshareCmd`, mirroring the pattern used in `StartCmd`. Operators then supply secrets via environment variables, which are not visible in `/proc/<pid>/cmdline` (they are only readable from `/proc/<pid>/environ` by the process owner or root).
2. Alternatively, accept the keyshare JSON from a file path argument or from stdin (e.g., `--keyshare-file`), so the secret material never appears in the argument list.
3. Document that the `MPC_LOCAL_ENCRYPTION_KEY_HEX` environment variable must be used and that passing the key as a positional argument is insecure.

### Proof of Concept

On a Linux host where an operator runs:

```bash
mpc-node import-keyshare \
    --home-dir /data \
    '{"epoch":5,"private_share":"aabbcc...","public_key":"..."}' \
    deadbeefdeadbeefdeadbeefdeadbeef
```

An unprivileged attacker on the same host runs:

```bash
# pspy or a simple loop:
while true; do
    for pid in /proc/[0-9]*/cmdline; do
        content=$(cat "$pid" 2>/dev/null | tr '\0' ' ')
        echo "$content" | grep -q 'import-keyshare' && echo "CAPTURED: $content"
    done
done
```

The output will contain the full `private_share` scalar and the AES key in plaintext. The attacker can then use the AES key to decrypt the keyshare stored at `$MPC_HOME_DIR/permanent_keys/` at any time. [6](#0-5)

```rust
impl ImportKeyshareCmd {
    pub async fn run(&self) -> anyhow::Result<()> {
        let runtime = tokio::runtime::Runtime::new()?;
        runtime.block_on(async {
            println!("Importing keyshare to local storage...");

            // Parse the encryption key
            let encryption_key_bytes = <[u8; 16]>::from_hex(&self.local_encryption_key_hex)
                .map_err(|_| {
                    anyhow::anyhow!("Invalid encryption key: must be 32 hex characters (16 bytes)")
                })?;

            let keyshare: PermanentKeyshareData = serde_json::from_str(&self.keyshare_json)
                .map_err(|e| anyhow::anyhow!("Failed to parse keyshare JSON: {}", e))?;

            println!("Parsed keyshare for epoch {}", keyshare.epoch_id());

            // Create the local storage and store the keyshare
            let home_dir = PathBuf::from(&self.home_dir);

            // Ensure the directory exists
            if !home_dir.exists() {
                std::fs::create_dir_all(&home_dir).map_err(|e| {
                    anyhow::anyhow!("Failed to create directory {}: {}", home_dir.display(), e)
                })?;
            }

            let storage =
                LocalPermanentKeyStorageBackend::new(home_dir.clone(), encryption_key_bytes)
                    .await?;

            // Check for existing keyshare
            if storage.load().await?.is_some() {
                anyhow::bail!("Refusing to overwrite existing local keyshare");
            }

            // Store the keyshare
            storage
                .store(&serde_json::to_vec(&keyshare)?, "imported")
                .await?;
            println!("Successfully imported keyshare to {}", home_dir.display());

            Ok(())
        })
    }
