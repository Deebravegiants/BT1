### Title
Unprivileged Caller Can Trigger Full Protocol Pause via Public `updateRSETHPrice()` When swETH Rate Is Transiently Low — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a `public` function with no access-control guard. Its internal logic contains an auto-pause branch that calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` on the oracle itself whenever the freshly computed rsETH price falls below `highestRsethPrice` by more than `pricePercentageLimit`. Because `SwETHPriceOracle.getAssetPrice()` passes `ISwETH.getRate()` through without any validation, a transient dip in the swETH exchange rate is sufficient to satisfy the trigger condition. Any unprivileged address can call the function at that moment and freeze all user deposits and withdrawals.

---

### Finding Description

**Entry point — no access control:**

`updateRSETHPrice()` carries only the `whenNotPaused` modifier; there is no role check. [1](#0-0) 

**Oracle passes rate through without validation:**

`SwETHPriceOracle.getAssetPrice()` returns `ISwETH(swETHAddress).getRate()` verbatim — no floor, no staleness check, no bounds check. [2](#0-1) 

**Auto-pause branch in `_updateRsETHPrice()`:**

When `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit × highestRsethPrice`, the function unconditionally pauses the deposit pool, the withdrawal manager, and the oracle itself, then returns. [3](#0-2) 

**`highestRsethPrice` is the all-time high:**

`highestRsethPrice` is only ever updated upward. Any transient dip in any supported asset's oracle rate — including swETH — causes `newRsETHPrice` to fall below it. [4](#0-3) 

**`LRTDepositPool.pause()` requires `PAUSER_ROLE`:**

For the auto-pause calls to succeed, `LRTOracle` must hold `PAUSER_ROLE` on both `LRTDepositPool` and `LRTWithdrawalManager`. This is a deployment requirement for the feature to function at all — the code is clearly designed to work this way. [5](#0-4) 

**Unpause requires admin action on each contract separately:**

`LRTOracle.unpause()` is `onlyLRTAdmin`; `LRTDepositPool.unpause()` and `LRTWithdrawalManager.unpause()` are also `onlyLRTAdmin`. All three must be called individually to restore normal operation. [6](#0-5) 

---

### Impact Explanation

All user deposits (`depositETH`, `depositAsset`) and all withdrawals are gated by `whenNotPaused` on `LRTDepositPool` and `LRTWithdrawalManager`. Once the auto-pause fires, every user interaction is blocked until an admin manually unpauses all three contracts. This constitutes **temporary freezing of funds** — a Medium-severity impact.

---

### Likelihood Explanation

The swETH rate (`ISwETH.getRate()`) is updated periodically by the Swell protocol. Between updates, the on-chain rate can lag behind the true value, and a rate update that corrects downward (e.g., after a slashing event, a rebase, or a protocol accounting adjustment) is a realistic, non-adversarial condition. Because `highestRsethPrice` is the all-time high and `pricePercentageLimit` can be set to a small value (e.g., 1% = `1e16`), even a modest transient dip satisfies the trigger. The attacker needs only to monitor the swETH rate and call the public function at the right block — no capital, no privilege, and no external compromise required.

---

### Recommendation

1. **Restrict `updateRSETHPrice()` to authorized callers** (e.g., `onlyLRTManager` or a keeper role), or add a cooldown/rate-limit so it cannot be called arbitrarily.
2. **Decouple the auto-pause from the public price-update path.** The auto-pause should only be triggerable by a privileged role, not as a side-effect of a permissionless call.
3. **Add a sanity bound to `SwETHPriceOracle.getAssetPrice()`** — reject rates that deviate more than a configurable percentage from the last known value, or use a time-weighted average.
