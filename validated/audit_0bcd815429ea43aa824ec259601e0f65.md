Audit Report

## Title
Pool Self-Recipient in `swap` Silently Converts Trader Output Into Swept Protocol Fees — (`metric-core/contracts/MetricOmmPool.sol`)

## Summary
`MetricOmmPool.swap` accepts an arbitrary `recipient` address with no guard against `recipient == address(this)`. When the pool address is passed as `recipient`, the output-token transfer is a self-transfer leaving `balance1()` unchanged, while `binTotals.scaledToken1` has already been decremented. The resulting phantom surplus is later swept as spread fees via `collectFees`, permanently destroying the trader's output tokens.

## Finding Description
The exploit path is as follows:

1. `_executeSwap` decrements `binTotals.scaledToken1` unconditionally for a `zeroForOne` swap at line 739:
   ```solidity
   binTotals.scaledToken1 = uint128(uint256(binTotals.scaledToken1) - uint256(-amount1DeltaScaled));
   ```

2. `transferToken1(recipient, uint256(-amount1Delta))` is called at line 254 **before** the callback. `transferToken1` is a thin wrapper around `IERC20(TOKEN1).safeTransfer(to, amount)` (line 570). When `to == address(this)`, this is a no-op on the pool's ERC-20 balance — `balance1()` (line 562: `IERC20(TOKEN1).balanceOf(address(this))`) is unchanged.

3. The `IncorrectDelta` guard at lines 261–263 only checks that the **input** token (token0) arrived:
   ```solidity
   if (amount0Delta > 0 && balance0Before + uint256(amount0Delta) > balance0()) {
       revert IncorrectDelta();
   }
   ```
   It never verifies that the output token actually left the pool.

4. After the swap, the invariant `balance1() * TOKEN_1_SCALE_MULTIPLIER == binTotals.scaledToken1 + notionalFeeToken1Scaled` is broken. `collectFees` at lines 385–388 computes:
   ```solidity
   uint256 surplus1Scaled =
     balance1() * TOKEN_1_SCALE_MULTIPLIER - uint256(binTotals.scaledToken1) - notionalFee1AmountScaled;
   ```
   The phantom surplus equals exactly the trader's expected output and is distributed to `adminFeeDestination` and the factory as spread fees.

There is no `require(recipient != address(this))` anywhere in `swap` (lines 217–225).

## Impact Explanation
Direct loss of 100% of the trader's expected output principal. The trader pays full token0 input (enforced by the callback check) and receives zero token1 output. The stolen tokens are permanently reclassified as protocol/admin spread-fee revenue once `collectFees` is called. Additionally, `binTotals` no longer covers actual LP claims for the output token, constituting pool insolvency for that token. This meets the Critical/High direct loss of user principal threshold.

## Likelihood Explanation
Any caller of `swap` controls `recipient` directly; no router or factory validation prevents `recipient = pool`. The `MetricOmmSimpleRouter.exactInput` (lines 103–112) already passes `address(this)` as intermediate recipient in multi-hop swaps — a one-off error in a similar custom integrator computing `recipient` programmatically would trivially produce `recipient = pool` and trigger this silently. The exploit is repeatable, requires no special privilege, and leaves no on-chain trace distinguishing it from a normal swap.

## Recommendation
Add a self-recipient guard at the top of `swap`, before any state mutation:
```solidity
require(recipient != address(this), SelfRecipient());
```
This costs a single comparison and fully closes the attack surface.

## Proof of Concept
```
Setup:
  pool  = MetricOmmPool(token0=USDC, token1=WETH)
  alice = trader (implements metricOmmSwapCallback)

1. Alice calls pool.swap(
       recipient       = address(pool),   // ← self-address
       zeroForOne      = true,
       amountSpecified = 1_000e6,         // exact-in 1 000 USDC
       priceLimitX64   = 0,
       callbackData    = "",
       extensionData   = ""
   );

2. _executeSwap computes amount1DeltaScaled = -X (e.g. -0.5e18 WETH).
   binTotals.scaledToken1 -= X  ← accounting says WETH left (line 739).

3. transferToken1(address(pool), 0.5e18) → safeTransfer to self,
   balance1() unchanged.

4. Callback fires; Alice pays 1 000 USDC. IncorrectDelta check passes
   (only checks token0 input, lines 261-263).

5. State after swap:
     balance1() * TOKEN_1_SCALE_MULTIPLIER  =  binTotals.scaledToken1 + X
     surplus1Scaled = X  (Alice's 0.5 WETH)

6. Anyone calls factory.collectPoolFees(pool).
   surplus1Scaled = X → swept to adminFeeDestination / FACTORY (lines 385-388, 420-421).

Alice lost 1 000 USDC and received 0 WETH.
```
