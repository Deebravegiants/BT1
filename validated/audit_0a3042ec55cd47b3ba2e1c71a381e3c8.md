Audit Report

## Title
Missing Chainlink Price Feed Staleness Check Enables Inflated rsETH Minting - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards the `updatedAt` timestamp, performing no staleness check. A stale price is accepted as valid and fed directly into `LRTDepositPool.getRsETHAmountToMint()`, which determines how much rsETH to mint per deposit. During an LST depeg event where the Chainlink feed has not yet updated, an attacker can deposit depegged LST at the stale inflated price and receive excess rsETH, diluting all existing rsETH holders.

## Finding Description
In `ChainlinkPriceOracle.getAssetPrice()`, the five return values of `latestRoundData()` are destructured as:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
``` [1](#0-0) 

The `updatedAt` field (4th return value) is discarded. No maximum age threshold is enforced anywhere in this function or its callers. The returned price is consumed unconditionally by `getRsETHAmountToMint()`:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [2](#0-1) 

This is called from `_beforeDeposit()`, which is invoked by both `depositAsset()` and `depositETH()` — both public, unprivileged entry points. [3](#0-2) 

The only guards in `_beforeDeposit()` are a minimum deposit amount check and a deposit limit check — neither validates oracle freshness. [4](#0-3) 

## Impact Explanation
**High — Theft of unclaimed yield from existing rsETH holders.**

When a supported LST depegs while the Chainlink feed is stale, an attacker deposits the depegged LST at the pre-depeg price and receives more rsETH than the deposited assets are worth. When the oracle eventually updates, `rsETHPrice()` adjusts downward, and all existing holders bear the loss of the over-minted supply. This is a concrete, quantifiable extraction of value from existing rsETH holders, matching the "Theft of unclaimed yield" impact class.

## Likelihood Explanation
**Medium.** Chainlink feeds operate on heartbeat intervals (e.g., 1 hour for ETH/stETH) and deviation thresholds. During high network congestion, oracle node downtime, or rapid price movement that has not yet crossed the deviation threshold, feeds can remain stale for minutes to hours. LST depeg events are historically correlated with exactly these conditions (e.g., stETH during the Merge). An attacker monitoring mempool and oracle state requires no special privileges to time the exploit.

## Recommendation
Add a configurable `maxStaleness` parameter and validate `updatedAt` in `getAssetPrice()`:

```solidity
(, int256 price,, uint256 updatedAt,) = priceFeed.latestRoundData();
if (block.timestamp - updatedAt > maxStaleness) {
    revert StalePriceFeed(asset, updatedAt);
}
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`maxStaleness` should be settable per asset (or globally) by the LRT manager, initialized to a value slightly above the feed's documented heartbeat interval.

## Proof of Concept
1. Chainlink stETH/ETH feed has a 1-hour heartbeat. At `T=0`, feed reports `1.0 ETH` per stETH (`updatedAt = T`).
2. At `T+30min`, stETH depegs to `0.95 ETH`. The Chainlink feed has not yet updated (heartbeat not triggered, deviation threshold not crossed).
3. Attacker calls `LRTDepositPool.depositAsset(stETH, 1000e18, 0, "")`.
4. `ChainlinkPriceOracle.getAssetPrice(stETH)` returns the stale `1.0 ETH` price — `updatedAt` is never checked.
5. `getRsETHAmountToMint` mints rsETH as if 1000 stETH = 1000 ETH worth of value.
6. Actual value deposited is 950 ETH. The 50 ETH difference is extracted from existing rsETH holders when the oracle updates and rsETH price adjusts downward.

**Foundry fork test plan:**
- Fork mainnet at a block where a Chainlink feed's `updatedAt` is near its heartbeat boundary.
- Warp `block.timestamp` forward past the heartbeat without triggering a feed update (mock the aggregator to return a stale `updatedAt`).
- Call `depositAsset` with the depegged LST amount.
- Assert that `rsethAmountToMint` exceeds the fair value, confirming over-minting.
