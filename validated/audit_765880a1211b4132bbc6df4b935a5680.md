### Title
Unenforced Fee-Quality Check Allows Any Connected Harvester to Redirect Farmer Block Rewards to Arbitrary Address - (`chia/farmer/farmer_api.py`)

---

### Summary

CHIP-22 introduced `farmer_reward_address_override` in the harvester→farmer protocol so that third-party harvesters (e.g. DrPlotter) can take a fee by redirecting the farmer's block reward to their own address when a proof meets a negotiated quality threshold. The farmer is supposed to enforce this threshold before honouring the override. In practice the threshold check is **only logged**; the override is accepted unconditionally. Any connected harvester can set `farmer_reward_address_override` to any puzzle hash and redirect 100 % of the farmer's block reward on every block won.

---

### Finding Description

`NewProofOfSpace` carries an optional `farmer_reward_address_override` field: [1](#0-0) 

When the farmer receives this message and the proof is good enough to make a block, it calls `notify_farmer_reward_taken_by_harvester_as_fee`: [2](#0-1) 

That function computes the fee quality and compares it to the harvester-supplied threshold, but **only emits log messages** regardless of the result — it never returns a value, raises an exception, or sets any flag that would block the override: [3](#0-2) 

After this purely advisory call, execution continues normally. Later, in `_process_respond_signatures`, the override is applied without any quality check: [4](#0-3) 

The resulting `DeclareProofOfSpace` carries the attacker-controlled `farmer_reward_address` to the full node, which writes it into the block's foliage and pays the reward there: [5](#0-4) 

---

### Impact Explanation

Every time the farmer wins a block, the attacker-controlled harvester can redirect the entire farmer reward (currently 0.25 XCH per block) to an arbitrary puzzle hash. The farmer receives nothing. This is a direct, permanent theft of XCH from the farmer with no on-chain recourse. The impact matches the allowed scope: **unauthorized payout redirection of XCH controlled by a singleton-adjacent farming reward mechanism**.

---

### Likelihood Explanation

The attacker must be a harvester that the farmer has connected to — the exact scenario CHIP-22 was designed for (third-party GPU/ASIC harvester services). A farmer who delegates plot-proving to any external harvester service is fully exposed. The attacker needs no keys, no admin access, and no cryptographic break; they only need to set one field in a protocol message they are already authorised to send.

---

### Recommendation

`notify_farmer_reward_taken_by_harvester_as_fee` must return a boolean (or raise) and the caller in `new_proof_of_space` must abort processing when the fee quality does not satisfy the threshold. Concretely:

1. In `farmer.py`, change `notify_farmer_reward_taken_by_harvester_as_fee` to return `False` (and log a warning) when `fee_quality > fee_threshold` or when `fee_info` is absent.
2. In `farmer_api.py` `new_proof_of_space`, check the return value and `return None` (drop the proof) when the override is present but the quality check fails.
3. Apply the same guard in `_process_respond_signatures` before line 918: if `farmer_reward_address_override` is set but the stored proof's fee quality does not pass the threshold, discard the response.

---

### Proof of Concept

1. Attacker operates a third-party harvester service and the victim farmer connects to it.
2. The farmer's plots produce a valid proof of space for a signage point.
3. The harvester sends `NewProofOfSpace` with `farmer_reward_address_override = <attacker_puzzle_hash>` and any `fee_info.applied_fee_threshold` value (or `fee_info = None`).
4. `new_proof_of_space` calls `notify_farmer_reward_taken_by_harvester_as_fee`, which logs a warning but does **not** stop execution. [6](#0-5) 
5. The farmer sends `RequestSignatures` to the harvester; the harvester responds with `RespondSignatures` also carrying `farmer_reward_address_override = <attacker_puzzle_hash>`.
6. `_process_respond_signatures` unconditionally sets `farmer_reward_address = response.farmer_reward_address_override`. [7](#0-6) 
7. `DeclareProofOfSpace` is sent to the full node with the attacker's address; the block is farmed and the reward (0.25 XCH) is paid to the attacker. The farmer receives 0 XCH for the block they won.
