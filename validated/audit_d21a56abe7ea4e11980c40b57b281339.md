### Title
Unbounded `fee_proposal_fri` During Startup Window Enables Permanent L2 Gas Price Manipulation — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

During the first `fee_proposal_window_size` (10) blocks after genesis or restart, the `fee_proposal_fri` field in `ProposalInit` is accepted by validators without any bounds check. A malicious elected proposer can set this field to an arbitrary value. These values are recorded in `fee_proposals_window` and, once the window fills, `compute_fee_actual` computes their median as `fee_actual` — the reference floor for all future L2 gas price calculations. The bounds check that would normally prevent this manipulation is explicitly disabled during the startup window.

---

### Finding Description

The `is_proposal_init_valid` function in `validate_proposal.rs` enforces that `fee_proposal_fri` lies within a geometric band around `fee_actual`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(...);
    }
}

### Impact Explanation

**Impact: Critical — Incorrect fee/gas accounting with economic impact.**

A malicious proposer who controls proposals during the first 10 blocks (e.g., at genesis, after a revert, or after a restart) can set `fee_proposal_fri` to `u128::MAX` (or `1`) in each block. After block 10, `compute_fee_actual` returns the median of these values as `fee_actual`. Because future `fee_proposal` bounds are anchored to `fee_actual` with only a 2 ppt margin, all subsequent proposers are forced to also propose near `u128::MAX` (or `1`), or their proposals are rejected. This permanently sets the L2 gas price floor to an extreme value:

- **Inflated `fee_actual` (`u128::MAX`):** All L2 transactions fail with out-of-gas / fee-too-high errors; the network becomes unusable.
- **Deflated `fee_actual` (`1`):** L2 transactions are priced near zero, enabling spam and breaking fee-market economics.

The corrupted `fee_proposal_fri` values are also forwarded to the centralized cende pipeline as `FeeProposalInfo`, persisting the manipulation in off-chain storage.

---

### Likelihood Explanation

**Likelihood: Medium.**

The attacker must be an elected proposer during the startup window. In Tendermint-style consensus, the proposer is selected from the validator committee by a deterministic rotation — any validator can be elected. An attacker who controls even a majority of the first 10 proposer slots (e.g., by controlling multiple validators or by being the sole initial validator at genesis) can fully control `fee_actual`. Genesis and post-revert restarts are realistic operational events.

---

### Recommendation

1. **Enforce a minimum bound on `fee_proposal_fri` during the startup window.** When `fee_actual` is `None`, validate that `fee_proposal_fri` lies within a reasonable absolute range (e.g., `[min_gas_price, max_gas_price]`) rather than accepting any value.

2. **Alternatively, seed `fee_proposals_window` from committed on-chain state before consensus begins.** The `initialize_fee_proposals_window` function already does this for restarts; ensure it is always called and that the window is never empty when consensus starts.

3. **Add a hard cap on `fee_proposal_fri` independent of `fee_actual`.** Even when bounds are enforced, a cap prevents the reference from drifting to pathological values over time.

---

### Proof of Concept

1. Network starts at genesis (height 0). `fee_proposals_window` is empty. `compute_fee_actual` returns `None` for all heights < `window_size` (10).

2. A malicious validator is elected proposer for heights 0–9 (or controls a majority of those slots).

3. For each of those blocks, the proposer sets `fee_proposal_fri = GasPrice(u128::MAX)` in `ProposalInit`.

4. In `is_proposal_init_valid`, the check at line 398 evaluates `(None, Some(u128::MAX))` — the `if let` does not match, so the bounds check is skipped. The proposal is accepted.

5. At `finalize_decision`, `record_fee_proposal(height, Some(GasPrice(u128::MAX)))` is called for each of the 10 blocks.

6. At height 10, `compute_fee_actual` returns `Some(GasPrice(u128::MAX))` (median of 10 identical values).

7. `calculate_next_l2_gas_price_for_fin` receives `fee_actual = u128::MAX` as the floor and sets `self.l2_gas_price = u128::MAX`.

8. All subsequent L2 transactions require `max_price_per_unit >= u128::MAX` in their resource bounds, which is impossible to satisfy. The network is permanently halted for user transactions.

9. Any honest proposer at height 11 who tries to set `fee_proposal_fri` to a sane value is rejected: `fee_proposal_bounds(u128::MAX, 2)` returns `lower ≈ u128::MAX`, so only values near `u128::MAX` are accepted. [1](#0-0) [10](#0-9) [5](#0-4) [7](#0-6)

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
