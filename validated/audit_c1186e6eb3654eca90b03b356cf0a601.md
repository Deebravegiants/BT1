Audit Report

## Title
`rsETHPrice` Baseline Advanced Unconditionally While Fee Is Skipped During Partial Pause — (`contracts/LRTOracle.sol`)

## Summary

In `LRTOracle._updateRsETHPrice()`, the protocol-fee calculation is skipped whenever `protocolPaused` is `true`, but `rsETHPrice` — the variable that serves as the fee-accounting baseline for the next call — is written unconditionally at line 313. When the deposit pool or withdrawal manager is paused (but the oracle is not), any public caller can advance the price baseline without triggering fee collection, permanently excluding the accrued rewards from the fee mechanism.

## Finding Description

`_updateRsETHPrice()` computes `protocolPaused` as:

```solidity
// LRTOracle.sol L240
bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;
``` [1](#0-0) 

When `protocolPaused == true`, the fee is zeroed out:

```solidity
// LRTOracle.sol L243-247
uint256 protocolFeeInETH = 0;
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
``` [2](#0-1) 

`newRsETHPrice` is then computed from the full `totalETHInProtocol` (since `protocolFeeInETH == 0`), and written unconditionally:

```solidity
// LRTOracle.sol L250, L313
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
// ...
rsETHPrice = newRsETHPrice;
``` [3](#0-2) [4](#0-3) 

The public entry point is gated only by the oracle's own `paused` flag:

```solidity
// LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [5](#0-4) 

The oracle's `paused` is independent of `lrtDepositPool.paused()` and `withdrawalManager.paused()`. The downside-protection auto-pause only fires when the oracle itself detects an out-of-limit price drop and calls `_pause()` internally. [6](#0-5)  Routine operational pauses of the deposit pool or withdrawal manager (upgrades, EigenLayer queue management) do not pause the oracle, leaving `updateRSETHPrice()` callable by anyone.

On the next call after the pause is lifted, `previousTVL = rsethSupply.mulWad(rsETHPrice)` is computed from the already-advanced `rsETHPrice`, so the rewards that accrued during the pause window are never subject to the fee. [7](#0-6) 

## Impact Explanation

**High — Theft of unclaimed yield.**

The protocol treasury is the intended recipient of `protocolFeeInBPS` of all staking rewards. Every time `updateRSETHPrice()` is called while the deposit pool or withdrawal manager is paused (but the oracle is not), the entire reward delta for that period is permanently excluded from fee collection. The treasury receives zero fee on those rewards; rsETH holders retain the full price appreciation. The loss is irreversible once `rsETHPrice` has been advanced and scales with the reward delta and fee rate.

## Likelihood Explanation

**Medium.** The deposit pool and withdrawal manager are paused regularly for operational reasons. The oracle is not co-paused in these scenarios. Any rsETH holder is economically incentivized to call `updateRSETHPrice()` during such a pause: doing so advances the price baseline without the fee dilution that would otherwise reduce their per-share value. No special privileges are required — the function is public.

## Recommendation

Do not advance `rsETHPrice` when the fee calculation was skipped. Two options:

1. **Guard the price write** — only commit `rsETHPrice = newRsETHPrice` when `!protocolPaused` (i.e., when the fee was actually computed).
2. **Defer the baseline** — when `protocolPaused`, compute `newRsETHPrice` for display/event purposes but do not update the stored `rsETHPrice`. This also prevents `highestRsethPrice` from ratcheting upward during a pause, which has additional implications for the downside-protection threshold.

Option 2 is safer as it closes both the fee-bypass and the `highestRsethPrice` ratchet issues simultaneously.

## Proof of Concept

1. Protocol live: `rsETHPrice = 1.05e18`, `rsethSupply = 1_000_000e18`, `protocolFeeInBPS = 1000` (10%).
2. Admin pauses `LRTDepositPool` for an upgrade. Oracle is **not** paused.
3. EigenLayer staking rewards accrue; `totalETHInProtocol` grows by `1_000e18`.
4. An rsETH holder calls `updateRSETHPrice()`:
   - `protocolPaused = true` (deposit pool paused, oracle not paused).
   - `protocolFeeInETH = 0` (fee branch skipped).
   - `newRsETHPrice = (1_050_000e18 + 1_000e18) / 1_000_000e18 = 1.051e18`.
   - `rsETHPrice = 1.051e18` ← baseline advanced with no fee collected.
5. Admin unpauses `LRTDepositPool`.
6. Next `updateRSETHPrice()` call: `previousTVL = 1_000_000e18 * 1.051e18 = 1_051_000e18`. The 1_000 ETH reward from step 3 is already baked in; `totalETHInProtocol - previousTVL = 0` (assuming no new rewards), so no fee is ever collected on it.
7. Treasury permanently loses `1_000 * 10% = 100 ETH` worth of fee.

**Foundry test plan:**
- Fork or mock `LRTDepositPool`, `withdrawalManager`, `LRTConfig`, and `RSETH`.
- Set initial state as above.
- Call `lrtDepositPool.pause()` as admin.
- Call `lrtOracle.updateRSETHPrice()` as an unprivileged address.
- Assert `rsETHPrice == 1.051e18` and treasury rsETH balance unchanged.
- Call `lrtDepositPool.unpause()`.
- Call `lrtOracle.updateRSETHPrice()` again (no new rewards).
- Assert fee minted to treasury is still 0 and `rsETHPrice` reflects no fee deduction for the 1_000 ETH reward period.
