Audit Report

## Title
Missing Time-Based Staleness Check in `ChainlinkOracleForRSETHPoolCollateral.getRate()` Enables Stale-Price Over-Payment — (`contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`)

## Summary

`ChainlinkOracleForRSETHPoolCollateral.getRate()` validates round completeness and non-zero timestamp but performs no check that `block.timestamp - updatedAt` is within the feed's heartbeat window. Because Chainlink feeds only push updates when a deviation threshold is crossed or a heartbeat elapses, a price that is many hours old can pass all three existing guards. Any caller of the permissionless `RSETHPool.deposit(address,uint256,string)` during a staleness window where the oracle price is above the true market price receives proportionally more `wrsETH` than the deposited token is worth, draining the pool's `wrsETH` inventory.

## Finding Description

`ChainlinkOracleForRSETHPoolCollateral.getRate()` (lines 26–37) calls `latestRoundData()` and applies exactly three guards:

```solidity
if (answeredInRound < roundID) revert StalePrice();   // round completeness
if (timestamp == 0)            revert IncompleteRound();
if (ethPrice <= 0)             revert InvalidPrice();
```

There is no guard of the form `if (block.timestamp - timestamp > heartbeat) revert StalePrice()`. Chainlink feeds update only when the price moves beyond a deviation threshold (e.g. 0.5%) **or** a heartbeat elapses (e.g. 24 h on Arbitrum LST feeds). Between those events `answeredInRound == roundID` and `timestamp != 0`, so all three checks pass even when the price is many hours old.

`RSETHPool.viewSwapRsETHAmountAndFee(uint256,address)` (lines 326–347) consumes this rate directly:

```solidity
uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate();
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
```

`RSETHPool.deposit(address,uint256,string)` (lines 284–305) then transfers `rsETHAmount` of `wrsETH` from the pool's own balance to the caller. If `tokenToETHRate` is stale-high, `rsETHAmount` is proportionally inflated and the pool over-pays in `wrsETH` for every deposit during the staleness window.

## Impact Explanation

**High — Theft of unclaimed yield.** The pool's `wrsETH` balance is its inventory of accrued protocol value. An attacker who deposits a supported token while its oracle price is stale-high receives more `wrsETH` than the token's current ETH value warrants. The excess `wrsETH` is permanently removed from the pool's inventory; the pool cannot recover it. This matches the allowed impact "Theft of unclaimed yield." The SECURITY.md exclusion for "incorrect data supplied by third-party oracles" does not apply here: the oracle is functioning as designed; the vulnerability is the contract's failure to validate data freshness.

## Likelihood Explanation

Chainlink LST/ETH feeds on Arbitrum (e.g. wstETH/ETH) have 24-hour heartbeats and 0.5% deviation thresholds. During periods of low volatility the price can be many hours old while all three existing guards pass. No privileged access is required; `deposit(address,uint256,string)` is a public, permissionless function protected only by `nonReentrant` and `whenNotPaused`. A sophisticated attacker monitoring on-chain oracle state can identify the staleness window and execute the deposit atomically in a single transaction.

## Recommendation

Add a configurable heartbeat staleness check in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
uint256 public immutable heartbeat; // e.g. 86_400 + 300 for a 24-hour feed

constructor(address _oracle, uint256 _heartbeat) {
    oracle = _oracle;
    heartbeat = _heartbeat;
}

function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID) revert StalePrice();
    if (timestamp == 0) revert IncompleteRound();
    if (block.timestamp - timestamp > heartbeat) revert StalePrice();
    if (ethPrice <= 0) revert InvalidPrice();

    return uint256(ethPrice) * 1e18 / 10 ** uint256(AggregatorV3Interface(oracle).decimals());
}
```

Set `heartbeat` to slightly above the feed's documented update interval (e.g. `86_400 + 300` seconds for a 24-hour feed).
