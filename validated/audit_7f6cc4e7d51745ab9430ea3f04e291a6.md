Audit Report

## Title
Public `updateRSETHPrice()` Triggers Deeply Nested Cross-Contract Loop With No Caller Restriction — (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is an unrestricted public function protected only by `whenNotPaused`. Any external caller can invoke it to trigger a multi-level cross-contract loop whose gas cost scales with the number of supported assets, node delegators, and queued EigenLayer withdrawals per NDC. As protocol state grows, the function's gas cost grows proportionally, and any caller can repeatedly invoke it at the cost of only gas fees.

## Finding Description
The call chain is confirmed in the codebase:

**Step 1** — `updateRSETHPrice()` has no role check: [1](#0-0) 

**Step 2** — `_getTotalEthInProtocol()` loops over every supported asset and calls `getTotalAssetDeposits(asset)` for each: [2](#0-1) 

**Step 3** — `getTotalAssetDeposits` calls `getAssetDistributionData`, which loops over every NDC and calls `getAssetUnstaking(asset)` per NDC: [3](#0-2) 

**Step 4** — `NodeDelegator.getAssetUnstaking()` fetches all queued EigenLayer withdrawals for that NDC via `getQueuedWithdrawals()` and iterates over every strategy in each withdrawal: [4](#0-3) 

The `ETH_TOKEN` path in `getETHDistributionData()` also calls `getAssetUnstaking(LRTConstants.ETH_TOKEN)` per NDC, doubling the withdrawal-fetch calls for the ETH asset: [5](#0-4) 

**Existing mitigations reviewed and found insufficient:**

- `maxNodeDelegatorLimit` is initialized to 10 but is admin-adjustable upward with no hard ceiling: [6](#0-5) 

- `maxUncompletedWithdrawalCount` in `LRTUnstakingVault` caps the *total* uncompleted withdrawals across all NDCs, but is also admin-configurable with no hard ceiling, starts at 0 (unset), and the cap is shared — a single NDC can hold all slots: [7](#0-6) 

- `whenNotPaused` only prevents calls when the oracle is paused; it provides no rate-limiting or caller restriction during normal operation.

The total gas per call is `O(assets × NDCs × queued_withdrawals_per_NDC × strategies_per_withdrawal)`. All four dimensions grow with normal protocol operation and are bounded only by admin-set parameters that have no on-chain ceiling.

## Impact Explanation
**Medium — Unbounded gas consumption.**

Any unprivileged caller can invoke `updateRSETHPrice()` repeatedly, consuming block gas at the cost of only the gas fee itself. As the protocol adds more LSTs, more NDCs, and accumulates more EigenLayer unstaking rounds, the per-call gas cost grows. If the accumulated state causes the function to approach or exceed the block gas limit, `updateRSETHPrice()` becomes uncallable, freezing the stored `rsETHPrice`. Downstream functions that read `lrtOracle.rsETHPrice()` — including `getRsETHAmountToMint` in `LRTDepositPool` — will operate on a stale rate: [8](#0-7) 

## Likelihood Explanation
**Medium.** The function requires no ETH, no role, and no special state to call. The gas cost grows deterministically with normal protocol usage (more assets, more NDCs, more EigenLayer unstaking rounds). No special attacker action is needed beyond calling the public function. The `maxUncompletedWithdrawalCount` and `maxNodeDelegatorLimit` parameters are admin-configurable and have no on-chain ceiling, so the effective bound depends entirely on operational choices.

## Recommendation
1. **Short term**: Add an access-control modifier (e.g., `onlyLRTOperator` or `onlyLRTManager`) to `updateRSETHPrice()` so only trusted callers can trigger the expensive computation path. The existing `updateRSETHPriceAsManager()` pattern already demonstrates this approach.
2. **Long term**: Refactor `_getTotalEthInProtocol()` to avoid the nested cross-contract loop. Cache per-asset TVL snapshots updated incrementally on deposit/withdrawal events rather than recomputing in full on every price update. Alternatively, enforce a strict on-chain ceiling for `maxUncompletedWithdrawalCount` and `maxNodeDelegatorLimit` to bound worst-case gas.

## Proof of Concept
```
Attacker (EOA)
  └─► LRTOracle.updateRSETHPrice()          [public, whenNotPaused only]
        └─► _updateRsETHPrice()
              └─► _getTotalEthInProtocol()
                    └─► for each asset (N):
                          └─► LRTDepositPool.getTotalAssetDeposits(asset)
                                └─► getAssetDistributionData(asset)
                                      └─► for each NDC (M):
                                            └─► NodeDelegator.getAssetUnstaking(asset)
                                                  └─► DelegationManager.getQueuedWithdrawals(ndc)
                                                        └─► for each withdrawal (W):
                                                              └─► for each strategy (S):
                                                                    [external call + sharesToUnderlyingView]
```

**Foundry fork test plan:**
1. Fork mainnet/testnet with the deployed contracts.
2. Add 5 supported assets, 10 NDCs, and queue 20 withdrawals of 2 strategies each across the NDCs via `initiateUnstaking`.
3. Call `updateRSETHPrice()` from an unprivileged EOA and measure gas via `vm.expectCall` counts or `gasleft()` snapshots.
4. Assert gas consumed exceeds a safe operational threshold (e.g., 5M gas) with realistic protocol parameters.
5. Increase `maxUncompletedWithdrawalCount` and repeat to demonstrate linear gas growth.

```text
            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
