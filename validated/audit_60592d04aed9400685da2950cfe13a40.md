### Title
`ChainlinkPriceOracle::getAssetPrice` uses Chainlink circuit-breaker-clamped price, enabling over-minting of rsETH when an LST depegs - (File: contracts/oracles/ChainlinkPriceOracle.sol)

### Summary
`ChainlinkPriceOracle.getAssetPrice` calls `latestRoundData()` and uses the returned answer without checking whether it has been clamped to the aggregator's built-in `minAnswer` floor. If a supported LST asset (stETH, rETH, ETHx, etc.) depegs below that floor, the oracle silently returns the inflated minimum price. Any depositor can then mint rsETH at the false price, stealing value from all existing rsETH holders.

### Finding Description
Chainlink's off-chain aggregator contracts enforce a `minAnswer` / `maxAnswer` circuit breaker. When the true market price of an asset falls below `minAnswer`, the aggregator continues to report `minAnswer` rather than the real price. The protocol's `ChainlinkPriceOracle` does not validate the returned answer against these bounds.

`getAssetPrice` in `contracts/oracles/ChainlinkPriceOracle.sol`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

There is no check of the form `require(price > minAnswer && price < maxAnswer)`. The raw answer is returned and propagated directly into rsETH minting math.

`LRTOracle.getAssetPrice` delegates unconditionally to this oracle:

```solidity
return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
``` [2](#0-1) 

`LRTDepositPool.getRsETHAmountToMint` uses the result to compute how many rsETH tokens to mint:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [3](#0-2) 

If `getAssetPrice(asset)` is inflated (clamped to `minAnswer` while the real price is lower), the numerator is artificially large and the depositor receives more rsETH than the deposited asset is worth.

`ChainlinkOracleForRSETHPoolCollateral.getRate()` has the same omission — it checks `ethPrice <= 0` but not the `minAnswer`/`maxAnswer` bounds:

```solidity
if (ethPrice <= 0) revert InvalidPrice();
``` [4](#0-3) 

### Impact Explanation
**Critical — direct theft of user funds / protocol insolvency.**

When an LST depegs below `minAnswer`, an attacker deposits the depegged asset and receives rsETH priced as if the asset were still at `minAnswer`. The excess rsETH represents a claim on the protocol's real ETH reserves that was never backed by equivalent value. Every existing rsETH holder is diluted; if the attack is large enough the protocol becomes insolvent (rsETH redeemable value < outstanding supply).

### Likelihood Explanation
LST depeg events are not hypothetical — the stETH/ETH depeg of May 2022 and various smaller events demonstrate that LSTs can trade significantly below their peg. Chainlink's `minAnswer` for LST/ETH feeds is typically set well above zero (e.g., 0.1 ETH for stETH/ETH). Any depositor who observes the depeg on-chain can immediately exploit this without any special privilege.

### Recommendation
Extend `AggregatorV3Interface` (or use `AggregatorV2V3Interface`) to expose `minAnswer` / `maxAnswer`, then revert if the returned price is at or outside those bounds:

```solidity
interface AggregatorV3Interface {
    function minAnswer() external view returns (int192);
    function maxAnswer() external view returns (int192);
    // ... existing functions
}

function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();

    if (price <= int256(priceFeed.minAnswer()) || price >= int256(priceFeed.maxAnswer())) {
        revert PriceOutOfAggregatorRange();
    }

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Apply the same fix to `ChainlinkOracleForRSETHPoolCollateral.getRate()`.

### Proof of Concept
1. Assume stETH is a supported asset with a Chainlink stETH/ETH feed whose `minAnswer` = 0.5e18 (50% of ETH).
2. A black-swan event causes stETH to trade at 0.1e18 on the open market.
3. Chainlink's aggregator clamps the answer at `minAnswer` = 0.5e18.
4. Attacker acquires 1000 stETH at market price (cost: ~100 ETH worth).
5. Attacker calls `LRTDepositPool.depositAsset(stETH, 1000e18, 0)`.
6. `getRsETHAmountToMint` computes: `1000e18 * 0.5e18 / rsETHPrice`. If `rsETHPrice` ≈ 1e18, attacker receives ~500 rsETH.
7. True value deposited: ~100 ETH. rsETH received redeemable for ~500 ETH.
8. Attacker extracts ~400 ETH of value from existing rsETH holders, causing protocol insolvency. [5](#0-4) [6](#0-5)
