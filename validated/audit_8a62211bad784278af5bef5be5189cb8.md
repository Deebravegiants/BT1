Audit Report

## Title
Unpermissioned `updateRSETHPrice()` Allows Any Caller to Trigger Protocol-Wide Auto-Pause — (`contracts/LRTOracle.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is declared `public whenNotPaused` with no role restriction. When the freshly computed rsETH price falls below `highestRsethPrice` by more than `pricePercentageLimit`, the internal `_updateRsETHPrice()` immediately pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. Because the entry point is unrestricted, any EOA can race to call it the moment on-chain conditions satisfy the threshold, freezing all deposits and withdrawal claims until an `onlyLRTAdmin` manually unpauses each contract.

## Finding Description

`updateRSETHPrice()` carries only `whenNotPaused` — no role check: [1](#0-0) 

It delegates to `_updateRsETHPrice()`, which contains the downside-protection branch. When `isPriceDecreaseOffLimit` is true, the function pauses all three contracts and returns early without updating `rsETHPrice`: [2](#0-1) 

The asymmetry is the root cause. The upside guard explicitly checks for `MANAGER` role before allowing a price push above the threshold: [3](#0-2) 

No equivalent role check exists on the downside path. `updateRSETHPriceAsManager()` exists precisely to let a manager bypass the upside revert, but there is no gating on who can trigger the downside pause via the public path: [4](#0-3) 

Unpausing each contract requires `onlyLRTAdmin`, meaning recovery is gated on a governance action: [5](#0-4) 

## Impact Explanation

**Medium — Temporary freezing of funds.**

Once the three-contract pause is triggered, `LRTDepositPool` rejects all new deposits and `LRTWithdrawalManager` rejects all withdrawal claims. Every user's in-flight withdrawal and every pending deposit is frozen until an `onlyLRTAdmin` call unpauses each of the three contracts. This matches the explicitly allowed impact "Temporary freezing of funds."

## Likelihood Explanation

The attacker requires no capital and no privileged role. Two conditions must hold simultaneously:

1. `pricePercentageLimit > 0` — set by admin and expected to be non-zero in production (the variable exists precisely to enforce a daily price-movement cap).
2. The live rsETH price computed from current EigenLayer TVL is below `highestRsethPrice` by more than `pricePercentageLimit * highestRsethPrice / 1e18`.

Condition 2 arises from ordinary events: an LST oracle price dip, a slashing event on an EigenLayer operator, or a temporary imbalance in the underlying asset mix. The attacker does not manufacture the price drop — they only observe it and call `updateRSETHPrice()` before any legitimate keeper does. The function is public and costs only gas, so the attacker can front-run any keeper bot. The attack is repeatable: after admin unpauses, the attacker can trigger it again whenever conditions are met.

## Recommendation

Add a caller restriction to `updateRSETHPrice()` that mirrors the upside guard. Two concrete options:

1. **Restrict to a keeper/manager role** — add `onlyLRTManager` (or a dedicated `KEEPER_ROLE`) to `updateRSETHPrice()`. Expose a separate, fully view-only function for off-chain price reads.
2. **Move the auto-pause trigger into the role-gated path** — keep a permissionless read path but move the downside-pause logic into `updateRSETHPriceAsManager()`, which already requires `onlyLRTManager`. The public path would revert (like the upside case) instead of pausing when the threshold is exceeded.

## Proof of Concept

```
Preconditions:
  - pricePercentageLimit = 1e16  (1%, set by admin)
  - highestRsethPrice = 1.05e18
  - Current EigenLayer TVL implies newRsETHPrice = 1.03e18
    → diff = 0.02e18 > 1e16 * 1.05e18 / 1e18 = 1.05e16  ✓ threshold exceeded

Attack:
  1. Attacker calls LRTOracle.updateRSETHPrice() from any EOA.
  2. _updateRsETHPrice() computes newRsETHPrice = 1.03e18.
  3. isPriceDecreaseOffLimit = true  (line 273-274).
  4. lrtDepositPool.pause() executes  → deposits frozen.
  5. withdrawalManager.pause() executes → withdrawal claims frozen.
  6. LRTOracle._pause() executes → oracle paused.
  7. Function returns early; rsETHPrice is NOT updated.

Result:
  - All user deposits and withdrawal claims are blocked until
    an LRTAdmin manually unpauses each of the three contracts.
  - Attacker spent only gas; no capital at risk.
  - Attack is repeatable after each admin unpause.

Foundry test plan:
  - Fork mainnet (or local deployment).
  - Set pricePercentageLimit to 1e16 via setPricePercentageLimit().
  - Set highestRsethPrice by calling updateRSETHPrice() once at a
    high-price state.
  - Manipulate mock LST oracle to return a price 2% below peak.
  - Call updateRSETHPrice() from address(0xdead) (unprivileged EOA).
  - Assert lrtDepositPool.paused() == true,
         withdrawalManager.paused() == true,
         lrtOracle.paused() == true.
  - Assert that a subsequent deposit call reverts with ContractPaused.
```
