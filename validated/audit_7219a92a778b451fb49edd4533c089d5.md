### Title
Pause Bypass via `swapAssetToPremintedRsETH` Accessible to `WHITELISTED_USER_ROLE` During Contract Pause - (File: contracts/pools/RSETHPoolV2ExternalBridge.sol)

---

### Summary
`RSETHPoolV2ExternalBridge.sol` implements a custom pause mechanism that guards `deposit()` with `whenNotPaused`, but the `swapAssetToPremintedRsETH()` function — which is reachable by any address holding `WHITELISTED_USER_ROLE` — carries no such guard. When the contract is paused, whitelisted users can continue to drain ETH from the pool by submitting rsETH, while regular depositors are locked out of all interactions. This is a direct analog to the Ion Protocol pause bypass: one class of user-facing operations is blocked by the pause while another succeeds, creating asymmetric access and undermining the security guarantee the pause is meant to provide.

---

### Finding Description

`RSETHPoolV2ExternalBridge` maintains a `bool public paused` flag and a custom `whenNotPaused` modifier:

```solidity
modifier whenNotPaused() {
    if (paused) revert ContractPaused();
    _;
}
```

The `deposit()` function is correctly guarded:

```solidity
function deposit(string memory referralId) external payable nonReentrant whenNotPaused limitDailyMint(msg.value) {
```

However, `swapAssetToPremintedRsETH()` — which transfers ETH out of the pool to the caller in exchange for rsETH — is **not** guarded by `whenNotPaused`:

```solidity
function swapAssetToPremintedRsETH(
    address rsETH,
    uint256 rsETHAmount
)
    external
    nonReentrant
    onlyOperatorOrWhitelisted(msg.sender)   // ← no whenNotPaused
{
    ...
    uint256 ethAmount = viewSwapAssetToPremintedRsETH(rsETHAmount);
    IERC20(rsETH).safeTransferFrom(msg.sender, address(wrapper), rsETHAmount);
    (bool success,) = payable(msg.sender).call{ value: ethAmount }("");
    ...
}
```

The access modifier `onlyOperatorOrWhitelisted` grants entry to **both** `OPERATOR_ROLE` and `WHITELISTED_USER_ROLE`:

```solidity
modifier onlyOperatorOrWhitelisted(address account) {
    if (!hasRole(OPERATOR_ROLE, account) && !hasRole(WHITELISTED_USER_ROLE, account)) {
        revert NotOperatorOrWhitelisted();
    }
    _;
}
```

`WHITELISTED_USER_ROLE` is a distinct, non-operator role designed to be granted to regular protocol participants:

```solidity
bytes32 public constant WHITELISTED_USER_ROLE = keccak256("WHITELISTED_USER_ROLE");
```

The result: when the pool is paused, whitelisted users can still call `swapAssetToPremintedRsETH` and receive ETH from the pool, while ordinary depositors are completely locked out.

---

### Impact Explanation

During a pause — which is typically triggered in response to a security incident, rate anomaly, or emergency — the pool's ETH balance can be drained by whitelisted users calling `swapAssetToPremintedRsETH`. This:

1. **Undermines the pause guarantee**: The pause is intended to halt all asset movement, but one user-accessible path remains open.
2. **Creates asymmetric access**: Whitelisted users can exit the pool (receive ETH) while regular users cannot deposit or otherwise interact.
3. **Reduces bridgeable ETH**: The pool's ETH is meant to be bridged to L1 via Stargate/LayerZero. Draining it during a pause reduces the amount available for bridging, potentially disrupting L1 settlement.

**Impact classification: Low — Contract fails to deliver promised returns, but doesn't lose value.** The pause does not deliver its promised security invariant (all operations halted) for whitelisted users.

---

### Likelihood Explanation

`WHITELISTED_USER_ROLE` is a named, persistent on-chain role granted to protocol participants as part of normal operation. Any holder of this role who observes the pause event (emitted on-chain) can immediately call `swapAssetToPremintedRsETH` without any further admin action. The entry path is direct and requires no oracle manipulation, front-running, or external dependency. Likelihood is **Medium**.

---

### Recommendation

Add the `whenNotPaused` modifier to `swapAssetToPremintedRsETH` in `RSETHPoolV2ExternalBridge.sol`:

```solidity
function swapAssetToPremintedRsETH(
    address rsETH,
    uint256 rsETHAmount
)
    external
    nonReentrant
    whenNotPaused                          // ← add this
    onlyOperatorOrWhitelisted(msg.sender)
{
```

This mirrors the fix applied in Ion Protocol (PR #36), which added a single unified pause covering all liquidation paths.

---

### Proof of Concept

1. Admin calls `pause()` on `RSETHPoolV2ExternalBridge` — `paused = true` is set, `Paused` event emitted.
2. Regular user calls `deposit()` → reverts with `ContractPaused()`. ✓ Pause works.
3. Whitelisted user (holding `WHITELISTED_USER_ROLE`) calls `swapAssetToPremintedRsETH(rsETH, amount)`:
   - `onlyOperatorOrWhitelisted` passes (user has the role).
   - No `whenNotPaused` check exists → execution continues.
   - `ethAmount = viewSwapAssetToPremintedRsETH(rsETHAmount)` computes ETH owed.
   - `rsETH` transferred from caller to wrapper.
   - ETH transferred from pool to caller.
   - Transaction succeeds despite the pause.

```text
    function swapAssetToPremintedRsETH(
        address rsETH,
        uint256 rsETHAmount
    )
        external
        nonReentrant
        onlyOperatorOrWhitelisted(msg.sender)
    {
        UtilLib.checkNonZeroAddress(rsETH);

        IRsETHTokenWrapper wrapper = IRsETHTokenWrapper(address(wrsETH));

        if (!wrapper.allowedTokens(rsETH)) revert TokenNotAllowedInWrapper();
        if (rsETHAmount == 0) revert InvalidAmount();
        if (rsETHAmount > wrapper.maxAmountToDepositBridgerAsset(rsETH)) revert ExceedsMaxAmountToDepositInWrapper();

        // Get the amount of ETH to transfer to the user for the given amount of rsETH provided
        uint256 ethAmount = viewSwapAssetToPremintedRsETH(rsETHAmount);

        // Transfer rsETH from sender to the wrapper
        IERC20(rsETH).safeTransferFrom(msg.sender, address(wrapper), rsETHAmount);

        // Transfer the ETH from the pool to the sender
        if (getETHBalanceMinusFees() < ethAmount) revert InsufficientETHBalanceForReverseSwap();
        (bool success,) = payable(msg.sender).call{ value: ethAmount }("");
        if (!success) revert TransferFailed();

        emit ReverseSwapOccurred(msg.sender, rsETH, rsETHAmount, ethAmount);
    }
```
