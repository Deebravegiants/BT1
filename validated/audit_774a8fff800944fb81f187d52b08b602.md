### Title
GovernanceThreshold Upper Bound Set to 100% Enables Permanent Protocol Freeze via Single Participant Failure - (File: `crates/contract/src/primitives/thresholds.rs`)

### Summary
The `governance_threshold_upper_relative_bound` function returns `n` (100% of participants), allowing the governance threshold to be set to require unanimous agreement. Once set to `t = n`, the loss of any single participant permanently freezes all signing and blocks every governance recovery path, with no on-chain escape hatch.

### Finding Description
`governance_threshold_upper_relative_bound(n)` in `crates/contract/src/primitives/thresholds.rs` returns `n` unconditionally, making the effective upper cap on the governance threshold equal to 100% of participants. The code itself acknowledges this is a known risk:

> "Currently set to 100% of participants but would be a discussion subject to drop this upper bound down not to have problems with smart contract being locked if t = n and if an operator stops voting"

Because `validate_governance_against_reconstruction` requires `governance >= max(reconstruction_threshold)`, setting governance threshold to `n` also permits setting every domain's reconstruction threshold to `n`. Both are accepted by `ThresholdParameters::new` and `process_new_parameters_proposal`.

Once the state `governance_threshold = n` and `reconstruction_threshold = n` is committed on-chain, the freeze is triggered through the `verify_tee` path:

1. Any participant's TEE attestation expires (natural hardware lifecycle, network partition, etc.).
2. `verify_tee` is called (callable by any current participant).
3. `TeeValidationResult::Partial` is returned with `remaining = n - 1` valid participants.
4. `ThresholdParameters::validate_governance_against_reconstruction(n-1, Threshold(n), ...)` fails because `n > n-1`.
5. The contract sets `self.accept_requests = false` and returns `Ok(false)` — no resharing is initiated.
6. All subsequent `sign`, `respond`, `respond_ckd`, and `respond_verify_foreign_tx` calls are rejected with `TeeError::TeeValidationFailed`.
7. `vote_new_parameters` to lower the threshold requires `n` votes, but only `n-1` participants are reachable — the vote can never pass.

There is no emergency override, no privileged recovery function, and no way to re-enable `accept_requests` without a new contract deployment.

### Impact Explanation
**Critical — permanent freezing of funds controlled by the MPC network.**

With reconstruction threshold = `n` and `accept_requests = false`, the MPC network can no longer produce threshold signatures for any foreign chain. All assets whose only spending authority is a derived MPC key (Bitcoin UTXOs, EVM accounts, etc.) become permanently inaccessible. The contract must be redeployed and a new key generation ceremony performed, but the old keys — and the funds they control — are irrecoverable.

### Likelihood Explanation
**Low.** Reaching the dangerous state requires the current governance quorum (≥60% of participants) to vote for `t = n` via `vote_new_parameters`. This is an unlikely deliberate governance decision. However, the code explicitly flags it as a known risk, and the trigger (a single TEE attestation expiry) is a routine operational event that will eventually occur in any long-running deployment. Once the threshold is set to `n`, the freeze is deterministic and irreversible.

### Recommendation
Lower `governance_threshold_upper_relative_bound` to a value strictly less than `n`, for example `ceil(0.8 * n)` or `n - 1`. This ensures that even if the governance threshold is set to its maximum, the loss of one participant still leaves enough voters to pass a recovery proposal. The analogous fix in the external report is to set a minimum allowed value for the dangerous parameter; here the symmetric fix is to set a maximum that preserves a one-participant safety margin.

```rust
// Before (effectively no upper cap):
pub(crate) fn governance_threshold_upper_relative_bound(n: u64) -> u64 {
    n
}

// After (preserves one-participant recovery margin):
pub(crate) fn governance_threshold_upper_relative_bound(n: u64) -> u64 {
    // Allow at most n-1 so that losing one participant never makes
    // governance permanently unreachable.
    n.saturating_sub(1).max(governance_threshold_lower_relative_bound(n))
}
```

Additionally, `vote_new_parameters` should reject any proposal that would set `reconstruction_threshold = n` for any domain, since that independently causes permanent signing failure on a single-node loss.

### Proof of Concept

**Step 1 — Set dangerous parameters (requires current governance quorum, ≥60%):**
```
vote_new_parameters({
  participants: [P1, P2, ..., Pn],   // all n current participants
  threshold: n,                       // governance threshold = 100%
  per_domain_thresholds: { domain_0: n }  // reconstruction threshold = 100%
})
```
This passes `validate_threshold` (n ≤ n, n ≥ ceil(0.6n), n ≤ upper_bound=n) and `validate_governance_against_reconstruction` (n ≥ n). After `n` votes the resharing completes and the new parameters are committed.

**Step 2 — Wait for any single TEE attestation to expire (routine operational event):**
Participant `Pk`'s TEE attestation expires after `tee_upgrade_deadline_duration_seconds`.

**Step 3 — Call `verify_tee` (callable by any participant):**
```
verify_tee()
```
Inside `verify_tee`:
- `reverify_and_cleanup_participants` returns `TeeValidationResult::Partial { participants_with_valid_attestation: [P1..P_{k-1}, P_{k+1}..Pn] }` (n-1 participants).
- `validate_governance_against_reconstruction(n-1, Threshold(n), Some(ReconstructionThreshold(n)))` returns `Err(BelowReconstructionThreshold)` because `n > n-1`.
- `self.accept_requests = false` is set.
- Returns `Ok(false)`.

```

    fn validate_threshold(n_shares: u64, k: Threshold) -> Result<(), Error> {
        if k.value() > n_shares {
            return Err(InvalidThreshold::MaxRequirementFailed {
                max: n_shares,
                found: k.value(),
            }
            .into());
        }
        if k.value() < MIN_THRESHOLD_ABSOLUTE {
            return Err(InvalidThreshold::MinAbsRequirementFailed.into());
        }
        let lower_relative_bound = governance_threshold_lower_relative_bound(n_shares);
        if k.value() < lower_relative_bound {
            return Err(InvalidThreshold::MinRelRequirementFailed {
                required: lower_relative_bound,
                found: k.value(),
            }
            .into());
        }
        let upper_relative_bound = governance_threshold_upper_relative_bound(n_shares);
        if k.value() > upper_relative_bound {
            return Err(InvalidThreshold::MaxRelRequirementFailed {
                max: upper_relative_bound,
                found: k.value(),
            }
            .into());
        }
        Ok(())
    }

    pub fn validate_governance_against_reconstruction(
        num_participants: u64,
        governance: Threshold,
        max_reconstruction_threshold: Option<ReconstructionThreshold>,
    ) -> Result<(), Error> {
        Self::validate_threshold(num_participants, governance)?;
        if let Some(max_reconstruction_threshold) = max_reconstruction_threshold
            && governance.value() < max_reconstruction_threshold.inner()
        {
            return Err(InvalidThreshold::BelowReconstructionThreshold {
                reconstruction_threshold: max_reconstruction_threshold.inner(),
                governance_threshold: governance.value(),
            }
            .into());
        }
        Ok(())
    }
