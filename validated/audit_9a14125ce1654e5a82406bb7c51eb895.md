### Title
Missing State Diff Commitment Verification in P2P Sync Allows Attacker-Controlled Peer to Corrupt Node State — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary
An attacker-controlled p2p peer can serve a `SignedBlockHeader` with an arbitrary `state_diff_commitment` and `state_diff_length=N`, then serve `StateDiffChunk`s whose content does not hash to that commitment. Because neither the block signature nor the assembled state diff hash is ever verified against the stored commitment, the wrong `ThinStateDiff` is written to storage for block N.

### Finding Description

**Step 1 — Header accepted without signature verification.**

`HeaderStreamBuilder::parse_data_for_block` in `crates/apollo_p2p_sync/src/client/header.rs` performs only two checks on a peer-supplied `SignedBlockHeader`:

- Block number ordering (line 104–113)
- Signatures vector length == 1 (line 115–118) [1](#0-0) 

`verify_block_signature` (which hashes `block_hash || state_diff_commitment` and checks the sequencer's ECDSA signature) is **never called** in the p2p sync client. It exists only in `crates/starknet_api/src/block.rs` and its own unit test. [2](#0-1) 

The header — including its attacker-chosen `state_diff_commitment` and `state_diff_length` — is written to storage unconditionally. [3](#0-2) 

**Step 2 — State diff accepted without commitment verification.**

`StateDiffStreamBuilder::parse_data_for_block` reads `target_state_diff_len` from the already-stored (attacker-controlled) header, accumulates `StateDiffChunk`s until `current_state_diff_len == target_state_diff_len`, and returns the assembled `ThinStateDiff` for storage. There is no call to `calculate_state_diff_hash` and no comparison against `header.state_diff_commitment`. [4](#0-3) 

The only structural checks performed are:
- Length accounting (lines 72–103)
- No duplicate deprecated declared classes (line 106)
- No conflicting keys within a single chunk (inside `unite_state_diffs`) [5](#0-4) 

**Step 3 — Wrong `ThinStateDiff` committed to storage.**

`BlockData::write_to_storage` for `(ThinStateDiff, BlockNumber)` calls `append_state_diff` directly with the unverified result. [6](#0-5) 

The function `calculate_state_diff_hash` that would perform the correct check is well-defined and used in the batcher/committer path, but is absent from the p2p sync client path. [7](#0-6) 

### Impact Explanation
Any node syncing via p2p can have its storage state corrupted for any block. Corrupted storage means:
- Wrong contract storage values, nonces, deployed contracts, and declared classes returned by RPC (`starknet_getStorageAt`, `starknet_getNonce`, `starknet_getClassAt`, etc.)
- If the node is also used for execution or fee estimation, wrong execution results follow from the wrong state root

This matches the explicitly scoped impact: *"Unprivileged-user-triggered … p2p sync path causes a node to anchor to the wrong … state diff."*

### Likelihood Explanation
Any peer on the p2p network can serve headers and state diff chunks. No stake, key, or operator privilege is required. The attacker only needs to be the peer that the victim node queries — achievable via eclipse attack, Sybil nodes, or by being the first peer to respond to a query. The attack is silent: the node logs no error and advances its state marker normally.

### Recommendation
In `StateDiffStreamBuilder::parse_data_for_block`, after the length check passes and before returning `Ok(Some((result, block_number)))`, compute and compare:

```rust
let stored_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("header must exist")
    .state_diff_commitment
    .ok_or(/* OldHeaderInStorage */)?;

let computed = calculate_state_diff_hash(&result);
if computed != stored_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::StateDiffHashMismatch { ... }));
}
```

Additionally, `HeaderStreamBuilder::parse_data_for_block` should call `verify_block_signature` with the network's known sequencer public key so that the `state_diff_commitment` embedded in the header cannot be forged in the first place.

### Proof of Concept
1. Stand up a mock p2p server peer.
2. Serve `SignedBlockHeader` for block 0 with `state_diff_commitment = X` (arbitrary Poseidon hash) and `state_diff_length = 1`.
3. Serve one `StateDiffChunk::ContractDiff` whose storage value differs from what `X` commits to (i.e., `calculate_state_diff_hash` of this chunk ≠ X).
4. Send `DataOrFin(None)` to close the session.
5. Assert via `storage_reader.begin_ro_txn().get_state_diff(BlockNumber(0))` that the stored `ThinStateDiff` contains the attacker-chosen storage value, not the one committed by `X`.

The existing test harness in `crates/apollo_p2p_sync/src/client/state_diff_test.rs` already provides all scaffolding (`run_test`, `Action::SendHeader`, `Action::SendStateDiff`, `Action::CheckStorage`) needed to implement this proof directly. [8](#0-7)

fn unite_state_diffs(
    state_diff: &mut ThinStateDiff,
    state_diff_chunk: StateDiffChunk,
) -> Result<(), BadPeerError> {
    match state_diff_chunk {
        StateDiffChunk::ContractDiff(contract_diff) => {
            if let Some(class_hash) = contract_diff.class_hash {
                if state_diff
                    .deployed_contracts
                    .insert(contract_diff.contract_address, class_hash)
                    .is_some()
                {
                    return Err(BadPeerError::ConflictingStateDiffParts);
                }
            }
            if let Some(nonce) = contract_diff.nonce {
                if state_diff.nonces.insert(contract_diff.contract_address, nonce).is_some() {
                    return Err(BadPeerError::ConflictingStateDiffParts);
                }
            }
            if !contract_diff.storage_diffs.is_empty() {
                match state_diff.storage_diffs.get_mut(&contract_diff.contract_address) {
                    Some(storage_diffs) => {
                        for (k, v) in contract_diff.storage_diffs {
                            if storage_diffs.insert(k, v).is_some() {
                                return Err(BadPeerError::ConflictingStateDiffParts);
                            }
                        }
                    }
                    None => {
                        state_diff
                            .storage_diffs
                            .insert(contract_diff.contract_address, contract_diff.storage_diffs);
                    }
                }
            }
        }
        StateDiffChunk::DeclaredClass(declared_class) => {
            if state_diff
                .class_hash_to_compiled_class_hash
                .insert(declared_class.class_hash, declared_class.compiled_class_hash)
                .is_some()
            {
                return Err(BadPeerError::ConflictingStateDiffParts);
            }
        }
        StateDiffChunk::DeprecatedDeclaredClass(deprecated_declared_class) => {
            state_diff.deprecated_declared_classes.push(deprecated_declared_class.class_hash);
        }
    }
    Ok(())
