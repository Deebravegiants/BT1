Audit Report

## Title
Missing Chainlink Staleness Validation Enables Stale Price Minting of rsETH — (`contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `updatedAt` and `answeredInRound`, accepting arbitrarily stale prices without revert. Because this price feeds directly into rsETH minting arithmetic in `LRTDepositPool`, any depositor can mint excess rsETH during a stale-feed window when the underlying LST has depegged, directly extracting value from all existing rsETH holders and driving the protocol toward insolvency.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` silently drops all staleness-relevant return values from `latestRoundData()`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L49-55
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

Neither `updatedAt` (feed freshness) nor `answeredInRound` (round completeness) is checked. The stale price propagates through `LRTOracle.getAssetPrice()`:

```solidity
// contracts/LRTOracle.sol L156-158
function getAssetPrice(address asset) public view onlySupportedOracle(asset) returns (uint256) {
    return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
}
``` [2](#0-1) 

...directly into rsETH minting in `LRTDepositPool.getRsETHAmountToMint()`:

```solidity
// contracts/LRTDepositPool.sol L519-520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [3](#0-2) 

The secondary oracle `ChainlinkOracleForRSETHPoolCollateral` adds partial guards (`answeredInRound < roundID`, `timestamp == 0`) but still omits the critical `block.timestamp - timestamp > stalePeriod` check: [4](#0-3) 

This secondary oracle is used for pool collateral rate queries, not for the primary deposit minting path, so its partial guards do not mitigate the `ChainlinkPriceOracle` bug.

## Impact Explanation
**Critical — Direct theft of user funds / Protocol insolvency.**

When a Chainlink LST/ETH feed goes stale while the underlying LST has depegged (e.g., stETH drops to 0.97 ETH), the stale inflated price (1.00 ETH) is used to compute `rsethAmountToMint`. The depositor receives ~3% excess rsETH per deposit. Upon redemption or secondary-market sale, this excess rsETH transfers real ETH-equivalent value away from all existing rsETH holders. Repeated or large-scale exploitation directly causes protocol insolvency: rsETH supply becomes backed by less ETH-equivalent collateral than its price implies. This is direct theft of existing user funds, not merely a best-practice gap. [5](#0-4) 

## Likelihood Explanation
Chainlink LST/ETH feeds (stETH/ETH, rETH/ETH) have 24-hour heartbeat intervals and 0.5% deviation thresholds. During network congestion, keeper failures, or rapid LST price movement, the feed can lag by hours while the on-chain price diverges from market. No special privilege is required: any address can call `depositAsset()`. The stale-feed window persists until the Chainlink keeper self-heals or an admin manually pauses the protocol. The attack is passive — the attacker simply deposits during the window. [6](#0-5) 

## Recommendation
Add a configurable `stalePeriod` mapping to `ChainlinkPriceOracle` and enforce all three staleness checks:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();
require(answeredInRound >= roundId, "Stale round");
require(updatedAt != 0, "Incomplete round");
require(block.timestamp - updatedAt <= stalePeriod[asset], "Stale price");
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Apply the same `block.timestamp - timestamp <= stalePeriod` guard to `ChainlinkOracleForRSETHPoolCollateral.getRate()`, which currently only checks `answeredInRound < roundID` and `timestamp == 0`. [7](#0-6) 

## Proof of Concept
**Foundry fork test outline:**

1. Fork mainnet at a block where the stETH/ETH Chainlink feed was last updated >24h ago (or mock `latestRoundData()` to return `updatedAt = block.timestamp - 25 hours` with `price = 1e18` while actual stETH market price is 0.97 ETH).
2. Call `LRTDepositPool.depositAsset(stETH, 1000e18, 0, "")` as an unprivileged EOA.
3. Assert that `rsethAmountToMint` equals `(1000e18 × 1.00e18) / rsETHPrice` — approximately 3% more than `(1000e18 × 0.97e18) / rsETHPrice`.
4. Redeem or sell the excess rsETH; assert that the ETH extracted exceeds the ETH-equivalent value of the deposited stETH at current market price.
5. Assert that `rsETHPrice` (after `updateRSETHPrice()`) has decreased, confirming dilution of existing holders.

No admin action, no privileged role, no front-running required — the stale price is returned by the on-chain feed itself. [7](#0-6) [6](#0-5)
