### Title
Unvalidated `builder` Field in `ProposalInit` Allows Proposer to Redirect All Block Fees to Arbitrary Address — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates timestamp, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, and `fee_proposal_fri`, but never checks the `builder` field. `convert_to_sn_api_block_info` maps `init.builder` directly to `block_info.sequencer_address`, which is the address that receives all transaction fees and is committed into the block hash. A committee member who is the current-round proposer can set `builder` to any address, causing every validator to execute the block with an attacker-controlled `sequencer_address`, permanently redirecting all block fees and producing a wrong block hash that the entire committee will agree on.

### Finding Description

`ProposalInit` carries a `builder` field defined as "Address of the one who builds/sequences the block." [1](#0-0) 

`convert_to_sn_api_block_info` converts this field directly into `sequencer_address` in the `BlockInfo` that is handed to the blockifier: [2](#0-1) 

`PartialBlockHashComponents::new` then stores that address as the `sequencer` field committed into the partial block hash: [3](#0-2) 

`calculate_block_hash` chains `sequencer.0` into the Poseidon hash: [4](#0-3) 

`is_proposal_init_valid` validates many `ProposalInit` fields but contains no check on `builder`: [5](#0-4) 

The `ProposalInitValidation` struct, which carries every value the validator checks against, has no `builder` member: [6](#0-5) 

`initiate_validation` passes the unchecked `builder` straight to the batcher as `block_info`: [7](#0-6) 

The batcher executes the block with the attacker-supplied `sequencer_address`, computes the `ProposalCommitment` from the resulting `PartialBlockHashComponents`, and returns it. The validator then compares that commitment with `ProposalFin.proposal_commitment`: [8](#0-7) 

Because every validator re-executes with the same attacker-supplied `builder`, they all derive the same (wrong) commitment and reach consensus on it.

### Impact Explanation

**Impact: Critical — Incorrect fee/balance with economic impact.**

The `sequencer_address` is the recipient of all transaction fees in every block. By setting `builder` to an address they control, the proposer redirects 100 % of block fees to themselves (or any target). Additionally, the committed `sequencer` field in the block hash is wrong, producing a permanently incorrect on-chain state root commitment for every block the attacker proposes.

### Likelihood Explanation

**Likelihood: Medium.**

The attacker must be a committee member and must be selected as the proposer for a given round. Both conditions are satisfied in the normal course of consensus — every committee member proposes blocks in rotation. No external exploit primitive is required beyond crafting a `ProposalInit` with a modified `builder` field, which is a trivial wire-level change.

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`. Each validator node knows its own expected `builder_address` from `ContextStaticConfig`: [9](#0-8) 

The check should be:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

This mirrors the existing pattern for `height`, `l1_da_mode`, and `l2_gas_price_fri` at line 312–321 of `validate_proposal.rs`.

### Proof of Concept

1. Attacker is a committee member and is selected as proposer for round R at height H.
2. Attacker calls `build_proposal` normally but, before streaming `ProposalPart::Init`, replaces `init.builder` with `attacker_address`.
3. Every validator receives the `ProposalInit` with `builder = attacker_address`.
4. Each validator calls `is_proposal_init_valid` — no check on `builder` exists, so validation passes.
5. Each validator calls `initiate_validation` → `convert_to_sn_api_block_info(init)` → `block_info.sequencer_address = attacker_address`.
6. The batcher executes all transactions with `sequencer_address = attacker_address`; all fee transfers credit `attacker_address`.
7. `BlockExecutionArtifacts::new` → `PartialBlockHashComponents::new` → `sequencer = attacker_address` → `ProposalCommitment` is computed with the wrong sequencer.
8. All validators compute the same commitment and send `Precommit`; consensus is reached.
9. `decision_reached` commits the block; `attacker_address` now holds all block fees, and the on-chain block hash encodes the wrong sequencer address permanently. [10](#0-9) [11](#0-10) [12](#0-11)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
