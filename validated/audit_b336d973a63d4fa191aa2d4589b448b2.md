## Attack Feasibility

The attacker:
1. Creates a parent coin on-chain with any puzzle P (e.g., the trivial `(1)` puzzle that returns its solution as conditions, or any standard wallet puzzle).
2. Spends that parent coin to create a child coin with `create_mirror_puzzle().get_tree_hash()` — a real, confirmed mirror coin on-chain.
3. Operates a malicious full node that the victim wallet connects to.
4. When the victim wallet calls `fetch_coin_spend` for the parent coin, the malicious peer returns:
   - The correct puzzle P (hash-validated, so it must match the on-chain puzzle hash — the attacker chose P, so they know it).
   - A **crafted solution** S such that running P with S produces a `CREATE_COIN` condition targeting the mirror puzzle hash with memos `[victim_tracked_launcher_id, attacker_url]`.
5. `get_mirror_info` extracts `(victim_tracked_launcher_id, [attacker_url])`.
6. `is_launcher_tracked(victim_tracked_launcher_id)` returns `True` (victim is subscribed to that store).
7. `add_mirror` records the attacker URL as a mirror for the victim's tracked store.

For a puzzle like `(1)`, the crafted solution is simply a list of CLVM conditions — trivially constructed. For a standard wallet puzzle, the delegated puzzle in the solution can be crafted to emit any conditions. No signature is checked by `get_mirror_info`.

The entry point is explicitly a **peer/sync path**, which is in scope per the rules.

### Title
Malicious peer injects arbitrary mirror record via unvalidated parent coin solution in `DataLayerWallet.coin_added` — (`chia/data_layer/data_layer_wallet.py`)

### Summary
`coin_added` fetches the parent coin's spend from the connected peer and trusts the peer-supplied solution to determine the `launcher_id` and URLs for a new mirror record. Because `fetch_coin_spend` only validates the puzzle hash (not the solution), a malicious peer can serve a crafted solution that causes `get_mirror_info` to return an attacker-chosen `launcher_id` and URL list, resulting in a spurious mirror record being written for any store the victim is already tracking.

### Finding Description
In `DataLayerWallet.coin_added`, when a coin with the mirror puzzle hash is detected, the wallet fetches the parent coin's spend via `fetch_coin_spend`. That function validates only that the returned puzzle's hash matches the on-chain coin's puzzle hash; the solution is accepted verbatim from the peer. `get_mirror_info` then executes `run_with_cost(parent_puzzle, INFINITE_COST, parent_solution)` on the peer-supplied data and reads `launcher_id` and URLs from the memos of the first matching `CREATE_COIN` condition. There is no cross-check that the solution matches what was actually broadcast on-chain.

### Impact Explanation
The attacker writes a mirror record associating an attacker-controlled URL with a `launcher_id` the victim is subscribed to. `DataLayer.update_subscriptions_from_wallet` subsequently includes that URL in the set of servers queried for store data, causing the victim's DataLayer node to fetch (and potentially accept) data from an attacker-controlled server for a store the attacker does not own — a direct data-poisoning path for subscribed DataLayer stores. async def fetch_coin_spend(height: uint32, coin: Coin, peer: WSChiaConnection) -> CoinSpend:
    solution_response = await peer.call_api(
        FullNodeAPI.request_puzzle_solution, RequestPuzzleSolution(coin.name(), height)
    )
    if solution_response is None or not isinstance(solution_response, RespondPuzzleSolution):
        raise PeerRequestException(f"Was not able to obtain solution {solution_response}")
    coin_id = coin.name()
    if solution_response.response.puzzle.get_tree_hash() != coin.puzzle_hash:
        raise PeerRequestException(f"Peer returned wrong puzzle hash for coin {coin_id}")
    if solution_response.response.coin_name != coin_id:
        raise PeerRequestException(f"Peer returned wrong coin name in puzzle solution for coin {coin_id}")

    return make_spend(
        coin,
        solution_response.response.puzzle,
        solution_response.response.solution,
    )

def get_mirror_info(
    parent_puzzle: Program | SerializedProgram, parent_solution: Program | SerializedProgram
) -> tuple[bytes32, list[bytes]]:
    assert type(parent_puzzle) is type(parent_solution)
    _, conditions = run_with_cost(parent_puzzle, INFINITE_COST, parent_solution)
    for condition in conditions.as_iter():
        if (
            condition.first().as_python() == ConditionOpcode.CREATE_COIN
            and condition.at("rf").as_python() == create_mirror_puzzle().get_tree_hash()
        ):
            memos: list[bytes] = condition.at("rrrf").as_python()
            launcher_id = bytes32(memos[0])
            return launcher_id, [url for url in memos[1:]]
    raise ValueError("The provided puzzle and solution do not create a mirror coin")
