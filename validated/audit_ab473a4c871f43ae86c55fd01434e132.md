### Title
Proposer-Controlled `init.builder` Accepted as Unvalidated `sequencer_address` in Block Context — (`crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`ProposalInit.builder` is a proposer-supplied field that is transmitted over the wire and decoded without any validation against a locally-trusted value. It is used verbatim as the `sequencer_address` in the `BlockInfo` that the batcher uses to execute every transaction in the block. A malicious-but-legitimate proposer can set `init.builder` to any arbitrary address, causing all block fees to be credited to that address and the wrong `sequencer_address` to be committed to the block hash and visible to contracts via `get_sequencer_address()`.

---

### Finding Description

`convert_to_sn_api_block_info` in `crates/apollo_consensus_orchestrator/src/utils.rs` converts a `ProposalInit` received from the network into a `starknet_api::block::BlockInfo`:

```rust
// utils.rs line 332
sequencer_address: init.builder,
``` [1](#0-0) 

`is_proposal_init_valid` validates many fields of `ProposalInit` against locally-trusted values — `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices (within margin), `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri` — but **`init.builder` is never checked**: [2](#0-1) 

The `ProposalInit` struct carries `builder` as a plain `ContractAddress` field that is decoded directly from the protobuf wire format: [3](#0-2) [4](#0-3) 

On the proposer side, `builder` is set to the node's own configured address:

```rust
// build_proposal.rs line 174
builder: args.builder_address,
``` [5](#0-4) 

But on the validator side, `init.builder` is passed straight through to `initiate_validation` → `convert_to_sn_api_block_info` → `ValidateBlockInput.block_info.sequencer_address` with no cross-check against any locally-known builder address. [6](#0-5) 

The `sequencer_address` in `BlockInfo` is the address to which the blockifier credits transaction fees and the value returned by the `get_sequencer_address()` syscall. It is also committed to in the block header and therefore in the block hash.

---

### Impact Explanation

**Incorrect fee accounting with economic impact (Critical):** A malicious proposer sets `init.builder` to an attacker-controlled address. Every transaction fee in the block is credited to that address instead of the legitimate sequencer. Because all validators accept the same `ProposalInit` and execute with the same `sequencer_address`, the block reaches consensus and is finalized with the wrong fee recipient baked into the committed state.

**Wrong execution context value (Critical):** Any contract that calls `get_sequencer_address()` during the block receives the attacker-controlled address. Smart contracts that gate logic on the sequencer address (e.g., fee-market contracts, privileged admin checks) will observe a wrong value, potentially enabling further exploitation.

**Wrong block hash commitment (High):** The `sequencer_address` is part of the block header that feeds into the block hash. The finalized block hash commits to the attacker-controlled address, producing a permanently incorrect on-chain record.

---

### Likelihood Explanation

Any legitimate committee member whose turn it is to propose a block can trigger this. No external attacker capability is required beyond holding a validator key. In a decentralized sequencer with rotating proposers, every validator is a potential trigger. The attack is silent — the proposal passes all existing validation checks and reaches consensus normally.

---

### Recommendation

Add `init.builder` to the fields validated in `is_proposal_init_valid`. The validator node knows its own builder address (the same value used in `ProposalBuildArguments.builder_address`). Reject any proposal whose `init.builder` does not match the expected builder address for the current height/round, analogous to how `init.proposer` is checked against the committee in `handle_proposal`:

```rust
// In is_proposal_init_valid, add:
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder),
    ));
}
```

`ProposalInitValidation` should carry the locally-configured `expected_builder` address, populated from the same source as `ProposalBuildArguments.builder_address`. [7](#0-6) 

---

### Proof of Concept

1. A malicious proposer (legitimate committee member) constructs a `ProposalInit` with `builder` set to an attacker-controlled address `ATTACKER_ADDR`.
2. The proposal is streamed to all validators. `handle_proposal` in `manager.rs` checks `init.proposer` against the committee — this passes because the proposer is legitimate. `init.builder` is not checked here. [8](#0-7) 
3. `is_proposal_init_valid` runs all its checks — none of them touch `init.builder`. The proposal init is accepted. [9](#0-8) 
4. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which sets `sequencer_address = ATTACKER_ADDR` in the `BlockInfo` passed to the batcher. [10](#0-9) 
5. The batcher executes all transactions with `sequencer_address = ATTACKER_ADDR`. All fees are credited to `ATTACKER_ADDR`. All `get_sequencer_address()` syscalls return `ATTACKER_ADDR`.
6. The batcher returns a `proposal_commitment` (partial block hash) that commits to the block executed with `ATTACKER_ADDR` as sequencer. The validator sends this commitment to consensus.
7. Consensus reaches a decision on the block. The finalized block has `ATTACKER_ADDR` as its sequencer address, with all fees credited there and the wrong address committed to the block hash.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}

impl TryFrom<protobuf::ProposalInit> for ProposalInit {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ProposalInit) -> Result<Self, Self::Error> {
        let height = BlockNumber(value.height);
        let round = value.round;
        let valid_round = value.valid_round;
        let proposer = value.proposer.ok_or(missing("proposer"))?.try_into()?;
        let timestamp = value.timestamp;
        let builder = value.builder.ok_or(missing("builder"))?.try_into()?;
        let l1_da_mode = enum_int_to_l1_data_availability_mode(value.l1_da_mode)?;
        let l2_gas_price_fri =
            GasPrice(value.l2_gas_price_fri.ok_or(missing("l2_gas_price_fri"))?.into());
        let l1_gas_price_fri =
            GasPrice(value.l1_gas_price_fri.ok_or(missing("l1_gas_price_fri"))?.into());
        let l1_data_gas_price_fri =
            GasPrice(value.l1_data_gas_price_fri.ok_or(missing("l1_data_gas_price_fri"))?.into());
        let l1_gas_price_wei =
            GasPrice(value.l1_gas_price_wei.ok_or(missing("l1_gas_price_wei"))?.into());
        let l1_data_gas_price_wei =
            GasPrice(value.l1_data_gas_price_wei.ok_or(missing("l1_data_gas_price_wei"))?.into());
        let starknet_version = match StarknetVersion::try_from(value.starknet_version.to_owned()) {
            Ok(version) => version,
            Err(_) => {
                return Err(ProtobufConversionError::OutOfRangeValue {
                    type_description: "starknet version",
                    value_as_str: value.starknet_version,
                });

            .version_constant_commitment

            .try_into()?;

        Ok(ProposalInit {
            height,
            round,
            valid_round,
            proposer,
            timestamp,
            builder,
            l1_da_mode,
            l2_gas_price_fri,
            l1_gas_price_fri,
            l1_data_gas_price_fri,
            l1_gas_price_wei,

            starknet_version,
            version_constant_commitment,
