### Title
Malicious Proposer Can Set Arbitrary `fee_proposal_fri` During Startup/Transition Window, Corrupting the Fee Market for All Future Blocks — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

The `is_proposal_init_valid` function enforces that `fee_proposal_fri` lies within a geometric margin of `fee_actual`. However, this bounds check is guarded by an `if let (Some(fee_actual), Some(fee_proposal)) = ...` pattern that silently skips enforcement whenever `fee_actual` is `None`. `fee_actual` is `None` during the first `window_size` blocks of a new chain **and** during the entire V0_14_3 upgrade transition (while the sliding window still contains pre-V0_14_3 blocks whose `fee_proposal_fri` is `None`). During this window, a malicious proposer can publish any `fee_proposal_fri` value — including `u128::MAX` — and every honest validator will accept it. The corrupted values are recorded in `fee_proposals_window`, and once the window fills, `fee_actual` is computed as the median of those extreme values, permanently skewing the L2 gas price for all subsequent blocks.

---

### Finding Description

**Invariant that should hold:** For Starknet version ≥ V0_14_3, every accepted `fee_proposal_fri` must lie within the geometric band `[fee_actual / (1 + margin), fee_actual * (1 + margin)]`. This is the SNIP-35 rate-limiting invariant that prevents any single proposer from moving the base fee more than `margin_ppt` per block.

**Where the invariant is broken:**

In `is_proposal_init_valid` (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`, lines 396–416):

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    ...bounds check...
}
```

When `fee_actual` is `None`, the entire bounds check is skipped. The validator only verifies that `fee_proposal_fri` is `Some` (required for V0_14_3+); it does **not** verify that the value equals the honest fallback (`l2_gas_price`), nor does it apply any other constraint.

**When `fee_actual` is `None`:**

`fee_actual` is computed by `compute_fee_actual` (`crates/apollo_consensus_orchestrator/src/dynamic_gas_price/mod.rs`, lines 56–92). It returns `None` in two production-reachable cases:

1. **Genesis / new chain**: `height < window_size` (the subtraction underflows).
2. **V0_14_3 upgrade transition**: Any height in `[height - window_size, height - 1]` is a pre-V0_14_3 block (recorded as `None` in `fee_proposals_window`). `compute_fee_actual` returns `None` on the first `None` entry it encounters.

Both cases are explicitly handled in `SequencerConsensusContext::validate_proposal` (lines 895–899), which passes `fee_actual: compute_fee_actual(...)` directly into `ProposalInitValidation`. When that returns `None`, the bounds check in `is_proposal_init_valid` is entirely bypassed.

**What an honest proposer does vs. what a malicious proposer can do:**

An honest proposer calls `compute_proposer_fee_proposal` (`sequencer_consensus_context.rs`, lines 471–494), which freezes at `self.l2_gas_price` when `fee_actual` is `None`. A malicious proposer ignores this convention and sets `fee_proposal_fri` to any value (e.g., `u128::MAX`). The validator has no check that enforces `fee_proposal_fri == l2_gas_price` during startup.

**Propagation of the corrupted value:**

After the proposal is accepted, `decision_reached` calls `record_fee_proposal` (`sequencer_consensus_context.rs`, line 308–310), inserting the malicious `fee_proposal_fri` into `fee_proposals_window`. Once `window_size` blocks have been produced with extreme values, `compute_fee_actual` returns the median of those extreme values. All future `fee_proposal_fri` values are then bounded relative to this extreme `fee_actual`, permanently anchoring the fee market at the attacker's chosen level.

The corrupted `fee_proposal_fri` is also bound into the `ProposalCommitment` via `proposal_commitment_from(partial, fee_proposal)` = `Poseidon(partial.0, fee_proposal_fri)` (`dynamic_gas_price/mod.rs`, lines 163–171), so it is committed to consensus and cannot be rolled back without a chain reorg.

---

### Impact Explanation

A malicious proposer who is selected during the startup/transition window can:

1. Set `fee_proposal_fri = u128::MAX` (or `0`) in their `ProposalInit`.
2. Every honest validator accepts the proposal because `fee_actual` is `None` and the bounds check is skipped.
3. The extreme value is recorded in `fee_proposals_window` and committed to the block header.
4. After `window_size` blocks, `fee_actual` = median of extreme values ≈ `u128::MAX`.
5. All future proposers must publish `fee_proposal_fri` within a small margin of `u128::MAX`, making the L2 gas price effectively unbounded.
6. Alternatively, setting `fee_proposal_fri = 0` drives `fee_actual` to zero, making L2 gas free.

This is a **Critical** impact: incorrect fee/gas/resource accounting with direct economic impact on all Starknet users and the protocol's fee revenue.

---

### Likelihood Explanation

- **New chain (genesis)**: The attack window is exactly `window_size` blocks. Any validator selected as proposer during this period can exploit it. No special privilege is required — proposer selection is part of normal Tendermint consensus rotation.
- **V0_14_3 upgrade transition**: The attack window lasts until all `window_size` blocks in the sliding window are V0_14_3+ blocks. During this transition, any proposer can exploit the gap.
- **Attacker precondition**: Being selected as proposer at least once during the window. In a permissioned validator set this requires being a registered validator; in a permissionless set it requires no special access.

---

### Recommendation

When `fee_actual` is `None`, the validator should enforce that `fee_proposal_fri` equals the honest fallback value (`l2_gas_price_fri` from `ProposalInitValidation`), rather than accepting any value. Concretely, replace the silent skip with an explicit equality (or tight-margin) check:

```rust
match (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri) {
    (Some(fee_actual), Some(fee_proposal)) => {
        // existing bounds check
    }
    (None, Some(fee_proposal)) => {
        // Startup/transition: enforce fee_proposal == l2_gas_price (the honest fallback)
        if fee_proposal != proposal_init_validation.l2_gas_price_fri {
            return Err(ValidateProposalError::InvalidProposalInit(...));
        }
    }
    _ => {}
}
```

This ensures that even during the startup window, a malicious proposer cannot deviate from the expected fallback value.

---

### Proof of Concept

**Setup**: A new Starknet chain at genesis, Starknet version V0_14_3. `window_size = N` (e.g., 100). `fee_proposals_window` is empty, so `compute_fee_actual` returns `None` for all heights `< N`.

**Attack**:
1. Attacker is selected as proposer at height 0 (or any height `< N`).
2. Attacker constructs a `ProposalInit` with `fee_proposal_fri = Some(GasPrice(u128::MAX))`.
3. Attacker broadcasts the proposal to the network.
4. Each honest validator calls `validate_proposal` → `is_proposal_init_valid`.
5. In `is_proposal_init_valid`, `proposal_init_validation.fee_actual` is `None` (window empty).
6. The `if let (Some(fee_actual), Some(fee_proposal)) = ...` pattern does **not** match (because `fee_actual` is `None`).
7. The bounds check is skipped entirely. The proposal passes validation.
8. Consensus reaches decision; `record_fee_proposal(height, Some(GasPrice(u128::MAX)))` is called.
9. Attacker repeats for heights 1 through `N-1` (or bribes/controls enough proposers).
10. At height `N`, `compute_fee_actual` returns `Some(GasPrice(u128::MAX))`.
11. All future `fee_proposal_fri` values must be within `margin_ppt` of `u128::MAX`.
12. The L2 gas price is permanently anchored at an extreme value.

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

    /// Compute the proposer's fee_proposal: clamp the oracle's `fee_target` to a margin around
    /// `fee_actual`. When `fee_actual` is `None` (window incomplete), freeze at `l2_gas_price`; the
    /// validator derives the same fallback so both sides agree.
    async fn compute_proposer_fee_proposal(
        &self,
        fee_actual: Option<GasPrice>,
        timestamp: u64,
        target_atto_usd_per_l2_gas: u128,
    ) -> GasPrice {
        SNIP35_FEE_TARGET_ATTO_USD.set_lossy(target_atto_usd_per_l2_gas);
        let Some(fee_actual) = fee_actual else {
            warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
            SNIP35_FEE_PROPOSAL_FRI.set_lossy(self.l2_gas_price.0);
            return self.l2_gas_price;
        };
        SNIP35_FEE_ACTUAL_FRI.set_lossy(fee_actual.0);

        let fee_target = self.resolve_fee_target(timestamp, target_atto_usd_per_l2_gas).await;

        let proposal = compute_fee_proposal(
            fee_target,
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        SNIP35_FEE_PROPOSAL_FRI.set_lossy(proposal.0);
        proposal
    }
