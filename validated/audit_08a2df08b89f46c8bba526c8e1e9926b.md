### Title
Single-Participant Key Event Instance Abort Enables Indefinite DKG/Resharing DoS — (`crates/contract/src/state/key_event.rs`)

### Summary

A single Byzantine participant, strictly below the signing threshold, can indefinitely prevent distributed key generation (DKG) or key resharing from completing by repeatedly calling `vote_abort_key_event_instance`. Because `vote_abort` immediately nullifies the active key event instance on a single participant's vote — with no threshold requirement — the attacker can abort every attempt as soon as the leader starts one, keeping the contract permanently stuck in `Initializing` or `Resharing` state and blocking all new signing requests.

### Finding Description

The `vote_abort` function in `KeyEvent` immediately destroys the active key event instance on a single participant's call:

```rust
pub fn vote_abort(&mut self, key_event_id: KeyEventId) -> Result<(), Error> {
    let candidate = self.verify_vote(&key_event_id)?;
    if self.instance.as_ref().unwrap().completed.contains(&candidate) {
        return Err(VoteError::VoteAlreadySubmitted.into());
    }
    self.instance = None;   // ← entire instance wiped by one vote
    Ok(())
}
``` [1](#0-0) 

The only guard is that the candidate must not already appear in `completed` (the set of participants who voted *success*). Because each new attempt starts with a fresh, empty `completed` set, the attacker's abort eligibility is fully restored on every attempt. [2](#0-1) 

The public entry point `vote_abort_key_event_instance` gates on `assert_caller_is_attested_participant_and_protocol_active()`, which only requires the caller to be an attested participant — not a threshold quorum. [3](#0-2) 

The leader must call `start_keygen_instance` / `start_reshare_instance` to open a new attempt, but the attacker can immediately abort it again. This cycle repeats indefinitely at the cost of one NEAR transaction per attempt. [4](#0-3) 

### Impact Explanation

While the contract is stuck in `Initializing` or `Resharing` state, it cannot process new signing requests. A resharing that never completes means the participant set cannot be updated, and the network cannot return to `Running` state through normal means. The only escape is `vote_cancel_resharing`, which itself requires a threshold quorum of the *previous* participant set — a quorum the attacker's presence does not block, but which requires coordinated operator action. Until that cancellation is executed, all new signature requests are frozen.

This matches the allowed impact: **Medium — contract execution-flow manipulation that breaks production safety invariants without relying on network-level DoS or operator misconfiguration.** [5](#0-4) 

### Likelihood Explanation

Any single participant in the proposed resharing or initializing set can execute this attack. The cost is one NEAR transaction per aborted attempt (a few cents). No cryptographic material, threshold collusion, or privileged access is required beyond holding a valid TEE attestation as a participant. The attack is fully on-chain and repeatable.

### Recommendation

**Short term:** Require a threshold quorum (or at minimum more than one participant) to abort a key event instance, mirroring the threshold requirement already applied to `vote_cancel_resharing`. A single-participant abort should at most mark that participant's own contribution as failed, not destroy the entire instance.

**Long term:** Add invariant tests that verify a key event instance cannot be aborted by fewer than `threshold` participants, and that DKG/resharing always completes within a bounded number of blocks regardless of individual participant behavior.

### Proof of Concept

1. Network enters `Resharing` state (e.g., after `vote_new_parameters` reaches threshold).
2. Leader (lowest participant ID) calls `start_reshare_instance(key_event_id)` — instance is created.
3. Attacker (any single participant in the proposed set) immediately calls `vote_abort_key_event_instance(key_event_id)` — `self.instance = None`.
4. Leader must call `start_reshare_instance` with the next `attempt_id` — attacker aborts again.
5. Repeat indefinitely. The contract never transitions back to `Running`; all `sign()` calls are blocked. [1](#0-0) [3](#0-2) [6](#0-5)

    /// The leader is always the participant with the lowest participant ID.
    pub fn start(&mut self, key_event_id: KeyEventId, timeout_blocks: u64) -> Result<(), Error> {
        self.cleanup_if_timed_out();
        if self.instance.is_some() {
            return Err(KeyEventError::ActiveKeyEvent.into());
        }
        let expected_key_event_id =
            KeyEventId::new(self.epoch_id, self.domain.id, self.next_attempt_id);

            return Err(KeyEventError::KeyEventIdMismatch.into());
        }
        self.verify_leader()?;
        self.instance = Some(KeyEventInstance::new(self.next_attempt_id, timeout_blocks));
        self.next_attempt_id = self.next_attempt_id.next();
        Ok(())
    }

    pub fn vote_cancel_resharing(&mut self) -> Result<Option<RunningContractState>, Error> {
        let previous_running_participants = self.previous_running_state.parameters.participants();
        let authenticated_candidate = AuthenticatedAccountId::new(previous_running_participants)?;
        self.cancellation_requests.insert(authenticated_candidate);

        let cancellation_votes_count = self.cancellation_requests.len() as u64;
        let previous_running_threshold = self.previous_running_state.parameters.threshold();

        let threshold_cancellation_votes_reached: bool =
            cancellation_votes_count >= previous_running_threshold.value();

        let running_state = if threshold_cancellation_votes_reached {
            let mut previous_running_state = self.previous_running_state.clone();
            let prospective_epoch_id = self.prospective_epoch_id();
            previous_running_state.previously_cancelled_resharing_epoch_id =
                Some(prospective_epoch_id);

            Some(previous_running_state)
        } else {
            None
        };

        Ok(running_state)
