### Title
Uncaught `ValueError` in `singleton_removed` when hint field is not 32 bytes permanently breaks DL wallet singleton tracking — (`chia/data_layer/data_layer_wallet.py`)

### Summary
`singleton_removed` in `DataLayerWallet` catches only `IndexError` when parsing `CREATE_COIN` hint fields via `bytes32()`. Because `bytes32` raises `ValueError` for non-32-byte inputs, and because on-chain hint fields are not consensus-validated for length, an attacker can submit a valid singleton spend with a malformed hint to crash the wallet's singleton tracking permanently.

### Finding Description
In `singleton_removed` (called during wallet sync when a tracked DL singleton coin is spent), the code runs the parent puzzle/solution via `run_with_cost`, iterates conditions, and for the odd-amount `CREATE_COIN` condition attempts:

```python
root = bytes32(condition[3][1])
inner_puzzle_hash = bytes32(condition[3][2])
``` [2](#0-1) 

wrapped in `except IndexError` only. [3](#0-2) 

`bytes32(x)` raises `ValueError` when `len(x) != 32`. The `CREATE_COIN` hint list (`condition[3]`) is arbitrary CLVM data — consensus imposes no 32-byte constraint on hint atoms. An attacker who controls a DL singleton spend (or who can submit any valid spend of a tracked singleton) can set the second hint element to a 31-byte (or any non-32-byte) atom. The resulting `ValueError` is uncaught and propagates out of `singleton_removed`.

`match_dl_singleton` at lines 43–56 only checks the puzzle structure (mod hashes and curry args), not the solution or hint content, so it returns `True` for any structurally valid DL singleton puzzle regardless of what hints the solution produces. [4](#0-3) 

### Impact Explanation
When `singleton_removed` throws, the wallet fails to record the new `SingletonRecord` (lines 845–861) and fails to register interest in the child coin (line 862). The wallet's view of the DL singleton's lineage is permanently broken: it never learns the new coin ID, root, or inner puzzle hash, and all subsequent state updates for that DL store are lost. This matches the High impact category: corruption of Data Layer root/store state and permanent inability to process Data Layer updates. [5](#0-4) 

### Likelihood Explanation
Any party who can spend a tracked DL singleton on-chain (i.e., the singleton's authorized spender, or anyone who can front-run or replace a spend in the mempool) can trigger this. The spend is consensus-valid — hint fields are not length-checked by the chain. The wallet will attempt to process every confirmed spend of a tracked singleton, so the crash is deterministic and repeatable on every sync attempt.

### Recommendation
Change the `except` clause to catch both `IndexError` and `ValueError`:

```python
except (IndexError, ValueError):
    self.log.warning(
        f"Parent {parent_name} with launcher {singleton_record.launcher_id} "
        "did not hint its child properly"
    )
    return
``` [3](#0-2) 

### Proof of Concept
1. Wallet tracks a DL singleton with launcher ID `L`.
2. Attacker (or the singleton's authorized spender) submits a spend of the singleton's current coin where the inner puzzle produces a `CREATE_COIN` condition with odd amount and a hint list whose second element is a 31-byte atom (e.g., `b'\x00' * 31`).
3. The spend is accepted by consensus (hint length is not validated on-chain).
4. Wallet sync calls `singleton_removed` for the spent coin.
5. `match_dl_singleton` returns `True` (puzzle structure is valid).
6. `run_with_cost` returns the conditions including the malformed `CREATE_COIN`.
7. `bytes32(condition[3][1])` raises `ValueError("bad bytes32 initializer")`.
8. Exception propagates uncaught; `singleton_removed` exits abnormally.
9. No `SingletonRecord` is written; the child coin is never tracked.
10. Assert: wallet has no record of the new singleton coin; all future DL store updates are invisible to the wallet.

def match_dl_singleton(puzzle: Program | SerializedProgram) -> tuple[bool, Iterator[Program]]:
    """
    Given a puzzle test if it's a CAT and, if it is, return the curried arguments
    """
    mod, singleton_curried_args = uncurry(puzzle)
    if mod == SINGLETON_TOP_LAYER_MOD:
        mod, dl_curried_args = uncurry(singleton_curried_args.at("rf"))
        if mod == NFT_STATE_LAYER_MOD and dl_curried_args.at("rrf") == ACS_MU_PH:
            launcher_id = singleton_curried_args.at("frf")
            root = dl_curried_args.at("rff")
            innerpuz = dl_curried_args.at("rrrf")
            return True, iter((innerpuz, root, launcher_id))

    return False, iter(())
