Audit Report

## Title
Unbounded Nested Gas Loop in `LRTOracle._getTotalEthInProtocol()` Can Permanently Block Price Updates — (File: `contracts/LRTOracle.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is publicly callable with no access control. It calls `_getTotalEthInProtocol()`, which iterates over the uncapped `supportedAssets` array and for each asset calls `LRTDepositPool.getAssetDistributionData()`, which loops over `nodeDelegatorQueue` and calls `NodeDelegator.getAssetUnstaking()` per NDC — itself invoking EigenLayer's `getQueuedWithdrawals()` and iterating over all queued withdrawal structs. As the protocol adds supported assets through normal governance, the combined gas cost grows multiplicatively and can exceed the block gas limit, permanently freezing the rsETH price.

## Finding Description

**Entry point — no privilege required:**

`updateRSETHPrice()` is `public whenNotPaused` with no role restriction: [1](#0-0) 

It unconditionally calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`.

**Outer loop — no cap on `supportedAssets`:**

`_getTotalEthInProtocol()` fetches the full `supportedAssetList` and iterates over every entry: [2](#0-1) 

`addNewSupportedAsset()` in `LRTConfig` pushes to `supportedAssetList` with no upper-bound check: [3](#0-2) 

**Inner loop — per-asset NDC traversal:**

For each asset, `getAssetDistributionData()` loops over every NDC in `nodeDelegatorQueue` and calls `getAssetUnstaking()` on each: [4](#0-3) 

**Innermost loop — EigenLayer queued withdrawal iteration:**

`getAssetUnstaking()` calls `getQueuedWithdrawals(address(this))` (an external EigenLayer storage read) and then iterates over all returned withdrawal structs and their strategy arrays: [5](#0-4) 

**Combined complexity:** `O(supportedAssets × nodeDelegatorQueue × queuedWithdrawals × strategies)`. The outer dimension (`supportedAssets`) has no cap. `nodeDelegatorQueue` is bounded by `maxNodeDelegatorLimit` (default 10, admin-settable upward). Queued withdrawals are bounded by `maxUncompletedWithdrawalCount` (also admin-settable). As assets grow, `getQueuedWithdrawals()` is called `assets × NDCs` times — each an external EigenLayer storage read — causing gas to grow without bound.

**Existing guards are insufficient:** `maxNodeDelegatorLimit` and `maxUncompletedWithdrawalCount` bound the inner dimensions but do nothing to cap the outer `supportedAssets` loop. No analogous `maxSupportedAssets` limit exists anywhere in `LRTConfig`.

## Impact Explanation

If `_getTotalEthInProtocol()` exhausts the block gas limit, every call to `updateRSETHPrice()` (public) and `updateRSETHPriceAsManager()` (manager) reverts. Consequences:

- `rsETHPrice` is permanently frozen; all subsequent deposits and withdrawals use a stale price, causing incorrect rsETH minting and incorrect asset-out calculations.
- Protocol fee minting inside `_updateRsETHPrice()` halts permanently.
- The automatic downside-protection pause (triggered when `newRsETHPrice < highestRsethPrice` beyond threshold) can never fire.

**Impact: Medium — Unbounded gas consumption**, matching the allowed impact scope.

## Likelihood Explanation

No malicious actor is required. Normal governance (`TIME_LOCK_ROLE` calling `addNewSupportedAsset()`) expands `supportedAssets` as the protocol integrates additional LSTs (rETH, cbETH, sfrxETH, osETH, etc.). With 20 assets, 10 NDCs, and 10 queued withdrawals per NDC, `getQueuedWithdrawals()` is called 200 times per `updateRSETHPrice()` invocation, each reading EigenLayer storage. At higher asset counts or elevated `maxNodeDelegatorLimit`, the 30M gas block limit is reachable without any attacker involvement.

## Recommendation

1. **Cap `supportedAssets`:** Add a `maxSupportedAssets` state variable in `LRTConfig` and enforce it in `_addNewSupportedAsset()`, analogous to `maxNodeDelegatorLimit` in `LRTDepositPool`.
2. **Cache TVL incrementally:** Maintain a running TVL accumulator updated on deposit/withdrawal events rather than recomputing the full nested loop on every price update call.
3. **Batch TVL computation:** Allow `_getTotalEthInProtocol()` to be called per-asset in separate transactions and aggregate results, so no single transaction must traverse the entire array.

## Proof of Concept

**Call sequence (no privilege required):**

```
anyone → LRTOracle.updateRSETHPrice()          [public, whenNotPaused]
           └─ _updateRsETHPrice()
                └─ _getTotalEthInProtocol()
                     └─ lrtConfig.getSupportedAssetList()   ← N assets, no cap
                     └─ for each asset (N):
                          └─ LRTDepositPool.getTotalAssetDeposits(asset)
                               └─ getAssetDistributionData(asset)
                                    └─ for each NDC (M, bounded by maxNodeDelegatorLimit):
                                         └─ INodeDelegator.getAssetUnstaking(asset)
                                              └─ DelegationManager.getQueuedWithdrawals(NDC)
                                                   └─ for each withdrawal (K) × strategy (S)
```

**Foundry fork test plan:**

1. Fork mainnet; deploy protocol with `N` supported assets (e.g., 15–25).
2. Add `M` NDCs (e.g., 10) and queue `K` withdrawals per NDC (e.g., 10).
3. Call `LRTOracle.updateRSETHPrice()` from an unprivileged EOA and measure gas via `vm.expectRevert` or `gasleft()` assertions.
4. Increase `N` incrementally and record gas until the call reverts with out-of-gas, demonstrating the unbounded growth.

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    /// @param asset Asset address
    /// @param depositLimit Deposit limit for the asset
    function _addNewSupportedAsset(address asset, uint256 depositLimit) private {
        UtilLib.checkNonZeroAddress(asset);
        if (depositLimit == 0) {
            revert InvalidDepositLimit();
        }
        if (isSupportedAsset[asset]) {
            revert AssetAlreadySupported();
        }
        isSupportedAsset[asset] = true;
        supportedAssetList.push(asset);
        depositLimitByAsset[asset] = depositLimit;
        emit AddedNewSupportedAsset(asset, depositLimit);
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
