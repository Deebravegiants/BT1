### Title
`batch_count` / `Q.size()` Invariant Not Enforced in `combine_ac`, Allowing Wrong-Count Output on SUCCESS — (`src/cbmpc/api/pve_batch_ac.cpp`)

### Summary
`combine_ac` parses `blob.batch_count` (N) from the outer blob but never validates it against `pve_ct.get_Q().size()` (M). An attacker who controls the ciphertext blob can set `batch_count = N` while embedding an inner `ec_pve_ac_t` with M ≠ N Q-points. `combine_ac` returns SUCCESS with M scalars in `out_xs`, while `get_ac_batch_count` on the same blob returns N. The API layer and protocol layer disagree about batch size.

### Finding Description
`pve_ac_ciphertext_blob_v1_t` contains two independent sources of batch size: the outer `batch_count` field and the inner `ec_pve_ac_t::Q` vector. `parse_ac_ciphertext` validates only the outer field. `combine_ac` uses the outer field for nothing after parsing, and `aggregate_to_restore_row` derives `batch_size = int(Q.size())` from the inner field. No guard of the form `pve_ct.get_Q().size() == blob.batch_count` exists anywhere in the `combine_ac` path.

An attacker:
1. Legitimately encrypts M scalars → valid `ec_pve_ac_t` with M Q-points, `blob.batch_count = M`
2. Patches `blob.batch_count` to N ≠ M in the serialized blob
3. Provides valid quorum shares for the M-scalar ciphertext
4. Calls `combine_ac` → returns SUCCESS with `out_xs.size() == M`

Meanwhile `get_ac_batch_count` on the same blob returns N.

### Impact Explanation
A caller who uses `get_ac_batch_count` to determine expected output size and then calls `combine_ac` without checking `out_xs.size()` will silently process M scalars instead of N. If M < N, key material is silently truncated; if M > N, attacker-controlled scalars are silently appended. In either case `combine_ac` returns SUCCESS, giving no indication of the mismatch. This fits the Medium scope: a public-API-reachable serialization invariant break causes unsafe state acceptance with security impact on recovered key material.

### Likelihood Explanation
Any party that can supply the ciphertext blob (network peer, storage provider, malicious sender) can trivially patch the 4-byte `batch_count` field. No cryptographic capability is required. The inner `ec_pve_ac_t` remains fully valid, so all cryptographic checks inside `aggregate_to_restore_row` pass.

### Recommendation
Add a consistency check in `combine_ac` immediately after deserializing `pve_ct`:

```cpp
if (pve_ct.get_Q().size() != static_cast<size_t>(blob.batch_count))
    return coinbase::error(E_FORMAT, "batch_count mismatch");
```

The same check should be added in `partial_decrypt_ac_attempt` for symmetry. `parse_ac_ciphertext` is the natural place to enforce this invariant once the inner `ec_pve_ac_t` is deserialized.

### Proof of Concept
```
1. Encrypt M=3 scalars with a valid 2-of-3 AC → serialize to buf_t ct_valid
2. Deserialize pve_ac_ciphertext_blob_v1_t from ct_valid
3. Patch blob.batch_count = 5 (N=5 != M=3)
4. Re-serialize to buf_t ct_patched
5. Call get_ac_batch_count(ct_patched, &n) → assert n == 5
6. Call partial_decrypt_ac_attempt for two leaves → get valid shares
7. Call combine_ac(..., ct_patched, ..., quorum, out_xs) → assert rv == SUCCESS
8. assert out_xs.size() == 3  // M, not N=5
9. // SUCCESS returned, but caller expected 5 scalars and got 3
``` [7](#0-6) [8](#0-7)

    return rv;

  error_t rv = UNINITIALIZED_ERROR;

  const auto icurve = to_internal_curve(curve);
  if (!icurve.valid()) return coinbase::error(E_BADARG, "unsupported curve");

  coinbase::crypto::ss::ac_owned_t ac_internal;

  if (rv = to_internal_ac_and_leaves(ac, icurve, ac_internal, leaf_names)) return rv;

  if (rv = parse_ac_ciphertext(ciphertext, blob)) return rv;

  coinbase::mpc::ec_pve_ac_t pve_ct;

  for (const auto& q : pve_ct.get_Q()) {

  }

  const int expected_share_size = icurve.order().get_bin_size();
  for (const auto& [name_view, share_bytes] : quorum_shares) {

  }

  std::map<std::string, coinbase::crypto::bn_t> quorum_bn;
  for (const auto& [name_view, share_bytes] : quorum_shares) {
    quorum_bn.emplace(std::string(name_view), coinbase::crypto::bn_t::from_bin(share_bytes));
  }

  detail::base_pke_bridge_t bridge(base_pke);
  coinbase::mpc::ec_pve_ac_t::pks_t pk_ptrs;

  std::vector<coinbase::crypto::bn_t> xs_bn;
  rv = pve_ct.aggregate_to_restore_row(bridge, ac_internal, attempt_index, label, quorum_bn, xs_bn,

  if (rv) {
    out_xs.clear();
    return rv;
  }

  std::vector<buf_t> out_local;
  out_local.resize(xs_bn.size());
  const int out_len = icurve.order().get_bin_size();
  for (size_t i = 0; i < xs_bn.size(); i++) out_local[i] = xs_bn[i].to_bin(out_len);
  out_xs = std::move(out_local);
  return SUCCESS;

error_t ec_pve_ac_t::aggregate_to_restore_row(const pve_base_pke_i& base_pke, const ss::ac_t& ac, int row_index,
                                              mem_t label, const std::map<std::string, bn_t>& quorum_decrypted,
                                              std::vector<bn_t>& x, bool skip_verify, const pks_t& all_ac_pks) const {
  error_t rv = UNINITIALIZED_ERROR;
  if (row_index < 0 || row_index >= kappa) return coinbase::error(E_RANGE);
  if (Q.empty()) return coinbase::error(E_BADARG);

  if (!skip_verify) {
    if (all_ac_pks.empty()) {
      return coinbase::error(E_BADARG, "all_ac_pks is required when skip_verify is false");
    }
    if (rv = verify(base_pke, ac, all_ac_pks, Q, label)) return rv;
  }

  const row_t& row = rows[row_index];

  int batch_size = int(Q.size());
  ecurve_t curve = Q[0].get_curve();
  int curve_size = curve.size();
  const auto& G = curve.generator();
  const mod_t& q = curve.order();

  buf_t L = crypto::sha256_t::hash(label, Q);

  bn_t K;
  if (rv = ac.reconstruct(q, quorum_decrypted, K)) return rv;

  buf_t k_and_iv = crypto::ro::hash_string(K, L).bitlen(256 + iv_bitlen);
  mem_t k_aes = k_and_iv.take(32);
  mem_t iv = k_and_iv.skip(32);

  buf_t decrypted_data;
  if (rv = crypto::aes_gcm_t::decrypt(k_aes, iv, L, tag_size, row.c, decrypted_data)) return rv;

  mem_t seed;
  mem_t x_bin;

  bool bit = b.get_bit(row_index);
  if (bit) {
    x_bin = row.x_bin;
    seed = decrypted_data;
  } else {
    x_bin = decrypted_data;
    if (row.r.size() != 32) return coinbase::error(E_CRYPTO);
    seed = row.r.take(16);
  }

  if (x_bin.size != batch_size * curve_size) return coinbase::error(E_CRYPTO);
  crypto::drbg_aes_ctr_t drbg(seed);
  x.resize(batch_size);
  for (int j = 0; j < batch_size; j++) {
    bn_t x0 = drbg.gen_bn(q);
    bn_t x1 = bn_t::from_bin(x_bin.range(j * curve_size, curve_size));
    MODULO(q) x[j] = x0 + x1;
    if (x[j] * G != Q[j]) return coinbase::error(E_CRYPTO);
  }

  return SUCCESS;
}
