Audit Report

## Title
Unpause Does Not Reset `highestRsethPrice`, Allowing Any Caller to Immediately Re-Trigger Auto-Pause — (`contracts/LRTOracle.sol`)

## Summary

`unpause()` resets only `paused = false` without updating `highestRsethPrice`. Because `updateRSETHPrice()` is public and permissionless (guarded only by `whenNotPaused`), any external caller can immediately re-trigger the auto-pause after an admin unpause by calling it while oracle prices remain below the historical peak, keeping the protocol frozen indefinitely.

## Finding Description

`_updateRsETHPrice()` contains a downside circuit breaker at lines 270–282:

```solidity
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;  // early return — highestRsethPrice unchanged
    }
}
``` [1](#0-0) 

When this fires, `highestRsethPrice` is left at its peak value. `unpause()` at lines 143–146 only flips the flag:

```solidity
function unpause() external whenPaused onlyLRTAdmin {
    paused = false;
    emit Unpaused(msg.sender);
}
``` [2](#0-1) 

After `unpause()`, the state is identical to the moment before the auto-pause fired — `highestRsethPrice` still holds the historical peak. `updateRSETHPrice()` is public with only a `whenNotPaused` guard: [3](#0-2) 

So any caller can immediately invoke it, recompute the same depressed `newRsETHPrice`, satisfy `isPriceDecreaseOffLimit`, and re-trigger `_pause()`.

The manager bypass at lines 260–266 only covers the **upside** threshold (`isPriceIncreaseOffLimit`): [4](#0-3) 

There is no analogous bypass for the downside path. `updateRSETHPriceAsManager()` (line 94) lacks `whenNotPaused` so it can be called while paused, but it still executes `_updateRsETHPrice()` and will re-trigger the downside auto-pause if prices remain depressed — it does not help break the cycle. [5](#0-4) 

The only admin escape hatch is calling `setPricePercentageLimit(0)` to disable the circuit breaker entirely, which is an unacceptable operational trade-off.

## Impact Explanation

While the oracle is paused, `lrtDepositPool` and `withdrawalManager` are also paused (lines 278–279), freezing all deposits, withdrawals, and claims. Any unprivileged EOA can nullify every admin unpause at zero cost (gas only), extending the freeze indefinitely for as long as oracle prices remain below `highestRsethPrice * (1 - pricePercentageLimit)`. This constitutes **temporary (but indefinitely extendable) freezing of user funds**, matching the **Medium** impact scope. [6](#0-5) 

## Likelihood Explanation

- `pricePercentageLimit` is a configurable admin parameter expected to be non-zero in production.
- The auto-pause fires precisely when prices are depressed; recovery is not instantaneous, so the condition persists across the unpause window.
- The attacker requires no funds, no role, and no front-running — a single public call to `updateRSETHPrice()` suffices.
- The attack is repeatable at zero cost beyond gas, with no rate limiting.

## Recommendation

In `unpause()`, reset `highestRsethPrice` to the current `rsETHPrice` so the next call to `updateRSETHPrice()` uses the post-recovery baseline rather than the stale historical peak:

```solidity
function unpause() external whenPaused onlyLRTAdmin {
    paused = false;
    highestRsethPrice = rsETHPrice; // reset baseline to current price
    emit Unpaused(msg.sender);
}
```

Alternatively, add a manager-role bypass symmetric to the upside bypass in the downside auto-pause path, so `updateRSETHPriceAsManager()` can update `highestRsethPrice` without re-triggering the pause.

## Proof of Concept

```solidity
// Precondition: pricePercentageLimit = 1e16 (1%)
// Precondition: oracle prices set so newRsETHPrice < highestRsethPrice by >1%

// Step 1: auto-pause fires
oracle.updateRSETHPrice(); // anyone calls
assert(oracle.paused() == true);
assert(oracle.highestRsethPrice() == PEAK); // unchanged

// Step 2: admin unpauses
vm.prank(admin);
oracle.unpause();
assert(oracle.paused() == false);
assert(oracle.highestRsethPrice() == PEAK); // still unchanged

// Step 3: attacker immediately re-triggers — same oracle prices, no state change needed
vm.prank(attacker);
oracle.updateRSETHPrice();
assert(oracle.paused() == true); // re-paused; admin unpause nullified
assert(oracle.highestRsethPrice() == PEAK); // still unchanged

// Step 4: repeat from Step 2 indefinitely at gas cost only
```
