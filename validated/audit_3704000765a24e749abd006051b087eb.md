### Title
Gateway L2 Gas Price Admission Check Uses Current Block's Price Instead of Next Block's Price, Causing Incorrect Admission Decisions — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `validate_tx_l2_gas_price_within_threshold` function in the gateway's stateful transaction validator computes its admission threshold from the **current block's** L2 gas price (`strk_gas_prices.l2_gas_price`). Transactions admitted through the gateway will be executed in the **next block**, which carries a different L2 gas price computed by the EIP-1559 `calculate_next_base_gas_price` mechanism. The code itself acknowledges the wrong field is being read via an open TODO. The mismatch causes the gateway to accept transactions that the blockifier will reject (when price is rising) and to reject transactions that the blockifier would accept (when price is falling).

---

### Finding Description

`validate_resource_bounds` reads the reference price for the admission threshold from `get_block_info().gas_prices.strk_gas_prices.l2_gas_price`: [1](#0-0) 

The comment on line 229 explicitly flags the wrong field:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
```

`BlockHeaderWithoutHash` carries two distinct L2 gas price fields: [2](#0-1) 

- `l2_gas_price: GasPricePerToken` — the price **used in the current block**
- `next_l2_gas_price: GasPrice` — the EIP-1559-adjusted price **for the next block**

The gateway reads `l2_gas_price` (current block) but transactions will be sequenced into the next block at `next_l2_gas_price`. The EIP-1559 adjustment is computed in `calculate_next_base_gas_price`: [3](#0-2) 

The price can move up or down by up to `price / gas_price_max_change_denominator` per block. The threshold applied at the gateway is therefore anchored to the wrong block.

The threshold computation in `validate_tx_l2_gas_price_within_threshold`: [4](#0-3) 

The blockifier's own fee check, which runs at execution time against the **actual next-block price**, is: [5](#0-4) 

These two checks use different reference prices, creating a window of divergence.

---

### Impact Explanation

**Case A — price rising (high block utilization):**
`next_l2_gas_price > current_l2_gas_price`. A transaction with `max_price_per_unit` in the band `[current * threshold, next * threshold)` passes the gateway check but fails `check_fee_bounds` in the blockifier with `MaxGasPriceTooLow`. The gateway admits a transaction that is invalid for the block it will be sequenced into.

**Case B — price falling (low block utilization):**
`next_l2_gas_price < current_l2_gas_price`. A transaction with `max_price_per_unit` in the band `[next * threshold, current * threshold)` is rejected by the gateway with `GAS_PRICE_TOO_LOW` even though it satisfies the actual next-block price. The gateway rejects a valid transaction before sequencing.

Both cases match the allowed impact: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

---

### Likelihood Explanation

The L2 gas price changes every block under normal EIP-1559 operation. The discrepancy is always present whenever the network is not at exactly the gas target. The width of the divergence band is bounded by `price / gas_price_max_change_denominator` per block, but with `min_gas_price_percentage = 100` (the default) the full band is exposed. No privileged access is required; any user submitting a V3 (`AllResources`) transaction with a gas price in the divergence band triggers the incorrect decision.

---

### Recommendation

Replace the `l2_gas_price` field read with `next_l2_gas_price` from the block header. The `GatewayFixedBlockStateReader` should expose `next_l2_gas_price` (already stored in `BlockHeaderWithoutHash`) and `validate_resource_bounds` should use it as the reference for the admission threshold. This directly resolves the open TODO and aligns the gateway's admission decision with the price the blockifier will enforce.

---

### Proof of Concept

Assume `min_gas_price_percentage = 100`, `gas_price_max_change_denominator = 8` (EIP-1559 default), and the current block's L2 gas price is `1000 FRI` with the block fully utilized.

1. `next_l2_gas_price = 1000 + 1000/8 = 1125 FRI` (computed by `calculate_next_base_gas_price`).
2. Gateway threshold (wrong): `1000 * 100% = 1000 FRI`.
3. Blockifier threshold (correct, next block): `1125 FRI`.
4. Submit a V3 transaction with `l2_gas.max_price_per_unit = 1050 FRI`.
5. Gateway check: `1050 >= 1000` → **admitted**.
6. Blockifier `check_fee_bounds`: `1050 < 1125` → `MaxGasPriceTooLow` → **rejected at execution**.

The transaction is accepted by the gateway and enters the mempool, but the batcher's blockifier rejects it during pre-validation, wasting sequencer resources and producing a failed transaction receipt for the user.

For Case B (price falling), reverse the scenario: current price `1000 FRI`, block empty, `next_l2_gas_price = 875 FRI`. A transaction with `max_price_per_unit = 900 FRI` is rejected by the gateway (`900 < 1000`) but would have been valid for the next block (`900 >= 875`).

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }
