### Title
Unvalidated `builder` Field in `ProposalInit` Allows Proposer to Inject Arbitrary Sequencer Address into Committed Block — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` performs an incomplete field-by-field check of the received `ProposalInit` struct. The `builder` field — which is directly mapped to `sequencer_address` in the executed block — is never validated against any expected value. A malicious proposer can set `builder` to an arbitrary address; the validator accepts the proposal, executes the block with that address as the sequencer, and commits a block whose sequencer address, fee-collection target, and partial block hash are all attacker-controlled.

---

### Finding Description

`ProposalInit` carries 14 fields. `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, `timestamp`, `l1_gas_price_fri/wei`, `l1_data_gas_price_fri/wei`, and `fee_proposal_fri`. The `builder` field is never checked. [1](#0-0) 

`is_proposal_init_valid` checks only these fields: [2](#0-1) 

`builder` is absent from `ProposalInitValidation` entirely: [3](#0-2) 

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`: [4](#0-3) 

The resulting `BlockInfo` (with the attacker-supplied `sequencer_address`) is passed to the batcher for block execution: [5](#0-4) 

The batcher then builds `PartialBlockHashComponents` from that `BlockInfo`, where `sequencer` is a committed field: [6](#0-5) 

`sequencer` is chained into the Poseidon block hash: [7](#0-6) 

Because the proposer also used the same `builder` value when building the block, the `batcher_block_commitment` computed by the validator matches `received_fin.proposal_commitment`, so the final mismatch check passes: [8](#0-7) 

The block is committed with the attacker-chosen `sequencer_address`.

---

### Impact Explanation

**Wrong state / wrong block hash**: The committed block's `sequencer_address` (and therefore its partial block hash) is attacker-controlled. Every downstream consumer — state sync, RPC `starknet_getBlockWithTxHashes`, proof system — sees the wrong sequencer address.

**Economic impact / fee redirection**: In Starknet execution, transaction fees are transferred from the user's account to `sequencer_address` during block execution. With `builder` set to an attacker-controlled address, all fees for that block are redirected to the attacker.

**Wrong contract execution context**: Any contract that reads `get_execution_info().sequencer_address` during that block receives the attacker-supplied address, corrupting any logic that depends on it (e.g., fee-market contracts, access-control checks keyed on the sequencer).

---

### Likelihood Explanation

Exploiting this requires being the elected proposer for at least one consensus round. In a rotating-proposer BFT protocol, any validator can become proposer. A single malicious validator can exploit this for every block they propose. No other precondition is needed: the attack is a simple field substitution in the `ProposalInit` message sent over the P2P stream.

---

### Recommendation

Add `builder` (and `proposer`) to `ProposalInitValidation` and enforce them in `is_proposal_init_valid`. The validator already knows the expected proposer for the current round from the consensus state machine; it should also know the expected builder address (either equal to the proposer, or from a separate registry). The check should be an exact equality, not a margin:

```rust
// In ProposalInitValidation:
pub expected_builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(...));
}
```

This mirrors the fix applied to the JBSplitsStore bug: add the missing field to the sameness check so the struct cannot be partially overwritten.

---

### Proof of Concept

1. Attacker controls a validator node that becomes the proposer for round R.
2. Attacker sets `builder = ATTACKER_WALLET` in the `ProposalInit` sent to peers.
3. Attacker builds the block with `sequencer_address = ATTACKER_WALLET` (consistent with the `ProposalInit`).
4. Validator nodes receive the `ProposalInit`, call `is_proposal_init_valid` — passes (no `builder` check).
5. Validator nodes call `convert_to_sn_api_block_info(init)` → `sequencer_address = ATTACKER_WALLET`.
6. Batcher executes the block; all transaction fees are transferred to `ATTACKER_WALLET`.
7. Batcher computes `partial_block_hash` with `sequencer = ATTACKER_WALLET`; validator's `batcher_block_commitment` matches the proposer's `ProposalFin.proposal_commitment`.
8. Proposal is accepted; block is committed with `sequencer_address = ATTACKER_WALLET` and all fees redirected.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
