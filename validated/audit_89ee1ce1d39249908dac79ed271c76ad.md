Audit Report

## Title
Missing Caller Identity Check in `claim()` Enables Forced Fee-Deducted Claims on Any Account - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary
`MerkleDistributor.claim()` accepts an arbitrary `account` parameter but never verifies that `msg.sender == account`. Any unprivileged caller can supply a victim's valid merkle proof tuple and force the victim's allocation to be disbursed with the protocol fee permanently deducted. The sibling contract `KernelMerkleDistributor` enforces `account == msg.sender` in `_processClaim()`, confirming the omission in `MerkleDistributor` is a defect.

## Finding Description
`MerkleDistributor.claim()` is a fully public, permissionless function:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
``` [1](#0-0) 

The function validates the merkle proof against `(index, account, cumulativeAmount)` and then unconditionally deducts a fee before transferring to `account`:

```solidity
uint256 fee = (claimableAmount * feeInBPS) / 10_000;
uint256 amountToSend = claimableAmount - fee;
IERC20(token).safeTransfer(account, amountToSend);
IERC20(token).safeTransfer(protocolTreasury, fee);
``` [2](#0-1) 

There is no `msg.sender == account` check anywhere in the function or its call path. Merkle proofs are derivable from public off-chain distribution data. An attacker can supply any victim's valid `(index, account, cumulativeAmount, merkleProof)` tuple and force the claim to execute, with the fee permanently extracted from the victim's allocation and sent to `protocolTreasury`. The victim's `userClaims` state is updated, preventing any future claim for the same index. [3](#0-2) 

By contrast, `KernelMerkleDistributor._processClaim()` explicitly enforces caller identity:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [4](#0-3) 

## Impact Explanation
**High — Theft of unclaimed yield.** The fee (`feeInBPS`, up to `MAX_FEE_IN_BPS = 1000`, i.e. 10%) is deducted from the victim's claimable amount and sent to `protocolTreasury`. The victim is entitled to their full allocation but permanently receives less. The fee portion of their unclaimed yield is irrecoverably extracted without their consent. The victim's claim state is finalized, preventing any future claim for the same index. [5](#0-4) 

## Likelihood Explanation
**High.** The entry point is fully permissionless — no role, no capital, and no prior interaction with the protocol is required. Merkle proofs are public inputs derivable from off-chain distribution data. The attack can be executed against any account with an unclaimed allocation at any time after the merkle root is set, and is repeatable across all eligible accounts. [1](#0-0) 

## Recommendation
Add a caller identity check at the top of `claim()`, mirroring the guard already present in `KernelMerkleDistributor._processClaim()`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [4](#0-3) 

## Proof of Concept
1. Protocol sets a merkle root. Alice has a valid leaf `(index=1, account=Alice, cumulativeAmount=1000e18)` and her proof is derivable from public distribution data.
2. Attacker Bob calls:
   ```solidity
   MerkleDistributor.claim(1, Alice, 1000e18, aliceProof);
   ```
3. Contract verifies the proof (valid), computes `fee = 1000e18 * feeInBPS / 10_000` (e.g. 100e18 at 10%), sends `900e18` to Alice and `100e18` to `protocolTreasury`.
4. `userClaims[Alice].lastClaimedIndex = 1` is set — Alice cannot claim index 1 again.
5. Alice permanently loses 100e18 tokens she was entitled to receive in full. Bob expended only gas.

**Foundry test plan:** Deploy `MerkleDistributor`, initialize with `feeInBPS = 1000`, set a merkle root for Alice's leaf, call `claim()` from a separate `attacker` address supplying Alice's valid proof, assert Alice received `900e18` and `protocolTreasury` received `100e18`, assert `isClaimed(1, Alice) == true`. [6](#0-5)

    /// @inheritdoc IMerkleDistributor
    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
