The key finding: `getMinFeeTxUtxo` for `DijkstraEra` is bound to `getConwayMinFeeTxUtxo`, which calls `txNonDistinctRefScriptsSize` — a function that only inspects the **top-level** transaction's inputs. ### Title
Sub-transaction reference scripts excluded from fee and size validation in Dijkstra batch transactions - (`File: eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`, `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubLedger.hs`)

### Summary

In the Dijkstra era, the minimum-fee check for a batch transaction uses `getConwayMinFeeTxUtxo`, which calls `txNonDistinctRefScriptsSize` on the top-level transaction only. Reference scripts referenced by sub-transactions are never counted toward the tiered reference-script fee surcharge, and the `maxRefScriptSizePerTx` cap is explicitly not applied to sub-transactions. An unprivileged user can craft a batch transaction whose top-level body references no large scripts (paying minimal fees) while embedding sub-transactions that reference arbitrarily large Plutus scripts, bypassing the fee and size controls introduced to prevent the June 2024 DDoS attack.

### Finding Description

`EraUTxO DijkstraEra` binds `getMinFeeTxUtxo` to `getConwayMinFeeTxUtxo`:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
```

`getConwayMinFeeTxUtxo` computes the reference-script surcharge using `txNonDistinctRefScriptsSize`, which only inspects the top-level transaction's `inputsTxBodyL` and `referenceInputsTxBodyL`:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx

txNonDistinctRefScriptsSize utxo tx = ...
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
```

A correct batch-aware version already exists but is never called in fee validation:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum (foldMap' (Sum . txNonDistinctRefScriptsSize utxo)
                       (tx ^. bodyTxL . subTransactionsTxBodyL))
```

The Dijkstra UTXO rule enforces the fee check only against the top-level transaction:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Utxo.hs (line 373)
runTest $ Shelley.validateFeeTooSmallUTxO pp tx originalUtxo
```

The sub-ledger rule explicitly marks the ref-script size failure as impossible for sub-transactions:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubLedger.hs (line 375)
Conway.ConwayTxRefScriptsSizeTooBig _ -> error "Impossible: `ConwayTxRefScriptsSizeTooBig` for SUBLEDGER"
```

### Impact Explanation

This matches the **Medium** allowed impact: *attacker-controlled transactions exceed intended validation limits or modify fees outside design parameters.*

The reference-script fee and size cap were introduced specifically to prevent DDoS attacks where cheap-to-submit transactions are expensive to validate (deserialization cost). By routing large reference scripts through sub-transactions, an attacker can submit a batch that:

1. Pays fees as if no large reference scripts are used (fee below the intended minimum for the actual computational work).
2. Exceeds the `maxRefScriptSizePerTx` limit without triggering a rejection.

This re-opens the DDoS vector that the Conway-era `minFeeRefScriptCostPerByte` parameter was designed to close, now applicable to any Dijkstra batch transaction.

### Likelihood Explanation

Any unprivileged user can submit a Dijkstra batch transaction. No special role, key, or governance majority is required. The attacker only needs to:
- Construct a top-level transaction with minimal or no reference-script inputs.
- Embed one or more sub-transactions whose inputs or reference inputs point to UTxOs containing large Plutus scripts.

The attack is directly reachable through the normal transaction submission path.

### Recommendation

Replace the call to `getConwayMinFeeTxUtxo` in `EraUTxO DijkstraEra` with a Dijkstra-specific implementation that uses `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
getMinFeeTxUtxo pp tx utxo =
  getMinFeeTx pp tx (batchNonDistinctRefScriptsSize utxo tx)
```

Additionally, apply the `validateRefScriptSize` check (currently only in the Conway LEDGER rule) to each sub-transaction in the Dijkstra SUBLEDGER or SUBUTXO rule, using the per-sub-transaction ref-script size against `maxRefScriptSizePerTx`.

### Proof of Concept

1. Deploy a UTxO containing a large Plutus script as a reference script (e.g., near the 200 KiB limit).
2. Construct a Dijkstra top-level transaction with no reference inputs and a minimal fee (computed without any ref-script surcharge).
3. Embed a sub-transaction whose `referenceInputsTxBodyL` points to the large-script UTxO.
4. Submit the batch. The UTXO rule calls `validateFeeTooSmallUTxO` → `getConwayMinFeeTxUtxo` → `txNonDistinctRefScriptsSize` on the top-level body only, finding zero reference-script bytes and accepting the minimal fee.
5. The node must deserialize and validate the large reference script for the sub-transaction, incurring the full computational cost while the submitter paid near-zero ref-script fees.
