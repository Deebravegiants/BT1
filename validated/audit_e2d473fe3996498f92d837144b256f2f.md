### Title
`SyncStateReader::get_compiled_class_hash` Is a `todo!()` Stub That Panics in Production Gateway Validation — (File: `crates/apollo_gateway/src/sync_state_reader.rs`)

### Summary

The production `SyncStateReader` — the state reader used by the gateway's stateful transaction validator — implements `get_compiled_class_hash` with a bare `todo!()` macro. In Rust, `todo!()` unconditionally panics at runtime. Any transaction that causes the blockifier's `CachedState` to fall through to the underlying state reader for a compiled-class-hash lookup will crash the gateway task and cause the transaction to be rejected, even if it is fully valid.

### Finding Description

`SyncStateReader` implements the `BlockifierStateReader` (`StateReader`) trait for the gateway. Every method in the trait has a real implementation that queries the state-sync client — except `get_compiled_class_hash`: [1](#0-0) 

```rust
fn get_compiled_class_hash(&self, _class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    todo!()
}
```

The `StateReader` trait documents the expected contract for this method: [2](#0-1) 

> Returns the compiled class hash of the given class hash. Returns `CompiledClassHash::default()` if no v1_class is found for the given class hash.

Other production implementations (e.g., `ApolloReader`, `ExecutionStateReader`) fully implement this by querying storage: [3](#0-2) [4](#0-3) 

`SyncStateReader` is wrapped in `SyncOrGenesisStateReader`, which delegates directly to it: [5](#0-4) 

`StateReaderAndContractManager` — the blockifier-facing wrapper — also delegates straight through: [6](#0-5) 

So the call chain is:

```
CachedState::get_compiled_class_hash
  → SyncOrGenesisStateReader::get_compiled_class_hash
    → SyncStateReader::get_compiled_class_hash
      → todo!()  ← PANIC
```

The `VersionedStateProxy` (used in concurrent execution) also falls through to the initial state reader when the value is not cached: [7](#0-6) 

### Impact Explanation

`get_compiled_class_hash` is called by the blockifier during `Declare` transaction pre-validation to determine whether a class is already declared (a non-zero compiled class hash means the class exists). When the gateway's `StatefulValidator::perform_validations` processes a `Declare` transaction and the compiled-class-hash cache misses, the call reaches `SyncStateReader::get_compiled_class_hash`, which panics. The gateway task aborts and returns an error to the caller. A valid `Declare` transaction is rejected before it can enter the mempool.

**Impact category:** High — Mempool/gateway admission rejects valid transactions before sequencing.

### Likelihood Explanation

Every `Declare` transaction submitted through the HTTP gateway triggers stateful validation via `SyncStateReader`. The compiled-class-hash cache in `CachedState` starts empty for each validation run, so the first lookup for any class hash will miss and fall through to the underlying reader. The panic is therefore triggered deterministically by any `Declare` transaction processed by the gateway in production.

### Recommendation

Implement `get_compiled_class_hash` in `SyncStateReader` by querying the state-sync client, mirroring the pattern used by `get_nonce_at`, `get_storage_at`, and `get_class_hash_at`:

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    // Query state_sync_client for the compiled class hash at self.block_number.
    // Return CompiledClassHash::default() if not found (Cairo 0 / undeclared).
}
```

The `StateSyncClient` trait already exposes the necessary primitives (e.g., `is_class_declared_at`). The implementation should follow the same `runtime.block_on(...)` pattern used by the other methods in `SyncStateReader`. [8](#0-7) 

### Proof of Concept

1. Start the sequencer with the gateway enabled.
2. Submit a valid `Declare` transaction (v2 or v3) via the HTTP gateway.
3. The gateway calls `StatefulValidator::perform_validations`.
4. The blockifier's `CachedState` calls `get_compiled_class_hash` on the underlying `SyncStateReader`.
5. `todo!()` fires; the gateway task panics with `"not yet implemented"`.
6. The transaction is rejected with an internal error, even though it is valid.

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }

        let state = state_opt.inner_mut()?;
        match state.compiled_class_hashes.read(self.tx_index, class_hash) {
            Some(value) => Ok(value),
            None => {
