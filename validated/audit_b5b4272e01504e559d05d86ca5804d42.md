### Title
Third-Party Harvester Can Unconditionally Redirect Farmer Block Rewards to Arbitrary Address — (`File: chia/farmer/farmer_api.py`, `chia/farmer/farmer.py`)

### Summary
The CHIP-22 third-party harvester fee convention allows a connected harvester to redirect the farmer's block reward (`farmer_reward_address_override`) to any arbitrary address. The farmer's fee-quality validation in `notify_farmer_reward_taken_by_harvester_as_fee()` is **purely advisory** — it only emits log warnings and never rejects or blocks the override. A malicious harvester can unconditionally steal 100% of the farmer's block rewards (base farmer reward + transaction fees) by setting `applied_fee_threshold = 0xFFFFFFFF` or by omitting `fee_info` entirely.

### Finding Description

The CHIP-22 protocol (implemented in this codebase) allows a third-party harvester to take the farmer's block reward as a fee by setting `farmer_reward_address_override` in `NewProofOfSpace` and `RespondSignatures`. The intended guard is a "fee quality" check: the harvester is supposed to only take the reward when `fee_quality <= applied_fee_threshold`, where `fee_quality` is a deterministic hash-derived value.

**Missing enforcement in `FarmerAPI.new_proof_of_space()`:**

```python
# chia/farmer/farmer_api.py lines 128-129
if new_proof_of_space.farmer_reward_address_override is not None:
    self.farmer.notify_farmer_reward_taken_by_harvester_as_fee(sp, new_proof_of_space)
```

The call to `notify_farmer_reward_taken_by_harvester_as_fee()` returns `None` in all cases — whether the threshold passes, fails, or `fee_info` is absent. The farmer then unconditionally proceeds to use the override. [1](#0-0) 

**Missing enforcement in `_process_respond_signatures()`:**

```python
# chia/farmer/farmer_api.py lines 916-919
farmer_reward_address = self.farmer.farmer_target
if response.farmer_reward_address_override is not None:
    farmer_reward_address = response.farmer_reward_address_override
    include_source_signature_data = True
```

There is no check here against any farmer-configured maximum threshold. Any non-`None` value from the harvester is accepted unconditionally and placed into `DeclareProofOfSpace.farmer_reward_address`. [2](#0-1) 

**The advisory-only guard in `notify_farmer_reward_taken_by_harvester_as_fee()`:**

The function checks `fee_quality <= fee_threshold` and logs a warning if the check fails, but **never returns an error, raises an exception, or signals the caller to abort**. When `fee_info` is `None`, it also only logs a warning and returns. [3](#0-2) 

**The `fee_quality` check is trivially bypassed:**

`calculate_harvester_fee_quality()` produces a `uint32` value. A harvester can set `applied_fee_threshold = uint32(0xFFFFFFFF)` (the maximum `uint32` value), guaranteeing `fee_quality <= fee_threshold` is always true, making the check log "Fee threshold passed" for every block. [4](#0-3) 

**Protocol message definitions confirm the override is fully attacker-controlled:**

`RespondSignatures.farmer_reward_address_override` is an optional `bytes32` field with no protocol-level constraint. [5](#0-4) 

### Impact Explanation

A malicious third-party harvester can redirect the farmer's entire block reward — comprising `calculate_base_farmer_reward(height)` (1/8 of block reward) plus all transaction fees — to any arbitrary puzzle hash on every block the farmer wins. This is a permanent, on-chain XCH payout redirection with no recourse once the block is confirmed. The farmer's configured `farmer_target` is silently bypassed. [6](#0-5) 

### Likelihood Explanation

Any harvester connected to the farmer can execute this attack. Third-party harvesters are an explicitly supported and documented use case (CHIP-22). A malicious harvester operator needs only to set `farmer_reward_address_override` to their own address and `applied_fee_threshold = 0xFFFFFFFF` in every `NewProofOfSpace` and `RespondSignatures` message. No cryptographic break, key compromise, or admin access is required. The farmer has no configuration option to reject or cap the override.

### Recommendation

The farmer must enforce a farmer-configured maximum fee threshold and **reject** (not merely log) any `farmer_reward_address_override` when:
1. `fee_info` is `None` (harvester did not provide a threshold), or
2. `fee_quality > applied_fee_threshold` (harvester claimed a fee it was not entitled to), or
3. `applied_fee_threshold` exceeds a farmer-configured maximum acceptable fee rate.

`notify_farmer_reward_taken_by_harvester_as_fee()` should return a boolean indicating whether the override is legitimate, and `new_proof_of_space()` and `_process_respond_signatures()` must refuse to use the override when the check fails.

### Proof of Concept

1. Farmer connects to a malicious third-party harvester.
2. Harvester finds a valid proof of space and sends `NewProofOfSpace` with:
   - `farmer_reward_address_override = attacker_puzzle_hash`
   - `fee_info = ProofOfSpaceFeeInfo(applied_fee_threshold=uint32(0xFFFFFFFF))`
3. Farmer calls `notify_farmer_reward_taken_by_harvester_as_fee()` — logs "Fee threshold passed" (since `fee_quality <= 0xFFFFFFFF` is always true) and returns `None`. Processing continues.
4. Farmer sends `RequestSignatures` to harvester.
5. Harvester responds with `RespondSignatures` containing `farmer_reward_address_override = attacker_puzzle_hash`.
6. Farmer executes lines 917–918: `farmer_reward_address = response.farmer_reward_address_override` — no check performed.
7. Farmer broadcasts `DeclareProofOfSpace` with `farmer_reward_address = attacker_puzzle_hash`.
8. Full node creates a block; the farmer coin (`calculate_base_farmer_reward(height) + fees`) is minted to `attacker_puzzle_hash` instead of the farmer's configured address. [7](#0-6) [8](#0-7)

    def notify_farmer_reward_taken_by_harvester_as_fee(
        self, sp: farmer_protocol.NewSignagePoint, proof_of_space: harvester_protocol.NewProofOfSpace
    ) -> None:
        """
        Apply a fee quality convention (see CHIP-22: https://github.com/Chia-Network/chips/pull/88)
        given the proof and signage point. This will be tested against the fee threshold reported
        by the harvester (if any), and logged.
        """
        assert proof_of_space.farmer_reward_address_override is not None

        challenge_str = str(sp.challenge_hash)

        ph_prefix = self.config["network_overrides"]["config"][self.config["selected_network"]]["address_prefix"]
        farmer_reward_puzzle_hash = encode_puzzle_hash(proof_of_space.farmer_reward_address_override, ph_prefix)

        self.log.info(
            f"Farmer reward for challenge '{challenge_str}' "
            + f"taken by harvester for reward address '{farmer_reward_puzzle_hash}'"
        )

        fee_quality = calculate_harvester_fee_quality(proof_of_space.proof.proof, sp.challenge_hash)
        fee_quality_rate = float(fee_quality) / float(0xFFFFFFFF) * 100.0

        if proof_of_space.fee_info is not None:
            fee_threshold = proof_of_space.fee_info.applied_fee_threshold
            fee_threshold_rate = float(fee_threshold) / float(0xFFFFFFFF) * 100.0

            if fee_quality <= fee_threshold:
                self.log.info(
                    f"Fee threshold passed for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
            else:
                self.log.warning(
                    f"Invalid fee threshold for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
                self.log.warning(
                    "Harvester illegitimately took a fee reward that "
                    + "did not belong to it or it incorrectly applied the fee convention."
                )
        else:
            self.log.warning(
                "Harvester illegitimately took reward by failing to provide its fee rate "
                + f"for challenge '{challenge_str}'. "
                + f"Fee quality was {fee_quality_rate:.3f}% ({fee_quality} or 0x{fee_quality:08x})"
            )

def calculate_harvester_fee_quality(proof: bytes, challenge: bytes32) -> uint32:
    """
    This calculates the 'fee quality' given a convention between farmers and third party harvesters.
    See CHIP-22: https://github.com/Chia-Network/chips/pull/88
    """
    return uint32(int.from_bytes(std_hash(proof + challenge)[32 - 4 :], byteorder="big", signed=False))
