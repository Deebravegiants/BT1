### Title
Missing RSA Modulus Validity Checks in `base_pke_rsa_ek_from_modulus` Allows Encryption Under Weak/Invalid Modulus - (`src/cbmpc/api/pve_base_pke.cpp`)

### Summary
`base_pke_rsa_ek_from_modulus` (and its C API wrapper `cbmpc_pve_base_pke_rsa_ek_from_modulus`) accepts any 256-byte non-zero value as an RSA-2048 modulus without checking that it is odd, has the correct bit-length (MSB set), or is composite. An attacker who controls the modulus input can produce a valid ek blob from a trivially factorable modulus (e.g., an even number), then call `cbmpc_pve_encrypt` successfully, obtaining a PVE ciphertext encrypted under a weak key.

### Finding Description

`base_pke_rsa_ek_from_modulus` in `src/cbmpc/api/pve_base_pke.cpp` performs only two checks on the caller-supplied modulus: [1](#0-0) 

- Size must be exactly 256 bytes.
- Value must not be zero (`BN_is_zero`).

It then calls `pk.set(n_bn, e_bn)` unconditionally: [2](#0-1) 

`rsa_pub_key_t::set` calls `EVP_PKEY_fromdata` with `EVP_PKEY_PUBLIC_KEY`: [3](#0-2) 

OpenSSL 3.x does not validate modulus oddness or compositeness at `fromdata` time for a public-key-only import; those checks are deferred to `EVP_PKEY_public_check`. So an even modulus (e.g., `0xFF…FE`) is accepted without error.

The resulting ek blob is then usable with `cbmpc_pve_encrypt` / `coinbase::api::pve::encrypt`, which calls `pve_ct.encrypt(bridge, pve_keyref(ek_mem), ...)`: [4](#0-3) 

This reaches `kem_policy_rsa_oaep_t::encapsulate`, which calls `pub_key.encrypt_oaep_with_seed`: [5](#0-4) 

`encrypt_oaep_with_seed` pads the KEM shared secret and calls `encrypt_raw`, which performs `EVP_PKEY_encrypt` with `RSA_NO_PADDING` — a raw modular exponentiation `m^e mod n`: [6](#0-5) [7](#0-6) 

For an even `n`, this computation succeeds and produces a well-formed-looking 256-byte ciphertext. The entire `cbmpc_pve_encrypt` call returns `CBMPC_SUCCESS`.

By contrast, the ECIES path in the same file calls `curve_p256.check(pk)` to validate the public key: [8](#0-7) 

No equivalent guard exists for the RSA path.

### Impact Explanation

A caller who provides an even 256-byte modulus (e.g., `0xFF…FE`) obtains a valid ek blob and can then encrypt a PVE secret under it. The resulting ciphertext is encrypted under a modulus that is trivially factorable (divide by 2 repeatedly). Any party who observes the ek blob can factor `n`, compute `d = e^{-1} mod λ(n)`, and decrypt the PVE ciphertext, recovering the protected scalar. The same applies to a prime modulus (where `d = e^{-1} mod (n-1)` is publicly computable). This constitutes accepted invalid cryptographic output from a public API path.

### Likelihood Explanation

The function is explicitly designed for the HSM use case where the modulus is imported from an external device. A malicious or compromised HSM, a man-in-the-middle on the HSM export channel, or a direct API caller can supply a crafted modulus. The missing oddness check (`BN_is_odd`) is a single-line fix, making the absence notable. The ECIES sibling function demonstrates the intended pattern (explicit key validation) that was not applied here.

### Recommendation

Add the following checks in `base_pke_rsa_ek_from_modulus` immediately after the `BN_is_zero` check:

1. **Oddness**: `BN_is_odd(n_bn)` — reject even moduli.
2. **Bit-length**: `BN_num_bits(n_bn) == coinbase::crypto::RSA_KEY_LENGTH` — reject moduli with leading zero bytes (effective bit-length < 2048).
3. **Optionally**: a Miller-Rabin primality check to reject prime moduli, or at minimum document that primality is not verified and callers must ensure the modulus is composite.

  constexpr unsigned long kDefaultExponent = 65537;

  BIGNUM* n_bn = BN_bin2bn(modulus.data, modulus.size, nullptr);

    BN_free(n_bn);

  coinbase::crypto::rsa_pub_key_t pk;

  BN_free(e_bn);

  ek_blob.key_type = static_cast<uint32_t>(base_pke_key_type_v1::rsa_oaep_2048);

  return SUCCESS;

error_t rsa_pub_key_t::encrypt_raw(mem_t in, buf_t& out) const {
  int n_size = size();
  if (n_size != in.size) return coinbase::error(E_CRYPTO);

  scoped_ptr_t<EVP_PKEY_CTX> ctx = EVP_PKEY_CTX_new(ptr, NULL);
  if (EVP_PKEY_encrypt_init(ctx) <= 0) return openssl_error("RSA encrypt RAW error");
  if (EVP_PKEY_CTX_set_rsa_padding(ctx, RSA_NO_PADDING) <= 0) return openssl_error("RSA encrypt RAW error");
  size_t outlen = n_size;
  if (EVP_PKEY_encrypt(ctx, out.alloc(n_size), &outlen, in.data, in.size) <= 0)
    return openssl_error("RSA encrypt RAW error");
  return SUCCESS;
}
