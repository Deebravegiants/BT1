### Title
Single Participant's Expired TEE Attestation Blocks All `vote_new_parameters` Calls — (File: `crates/contract/src/lib.rs`)

---

### Summary

`vote_new_parameters` validates **every** participant in the proposed set before recording any vote. If even one participant's TEE attestation is invalid or expired, the entire call is rejected with `InvalidTeeRemoteAttestation`. This is the direct analog of the `Voter.distribute` bug: one failing item in a batch check causes the whole operation to fail, and a Byzantine participant strictly below the signing threshold can exploit this to freeze governance when the network operates at exactly the threshold size.

---

### Finding Description

In `vote_new_parameters` [1](#0-0) , the contract calls `reverify_and_cleanup_participants` before recording any vote:

```rust
let validation_result = self.tee_state.reverify_and_cleanup_participants(
    proposal.participants(),
    tee_upgrade_deadline_duration,
);
```

`reverify_and_cleanup_participants` [2](#0-1)  iterates over **all** proposed participants and filters to those whose stored attestation re-verifies as `TeeQuoteStatus::Valid`:

```rust
let participants_with_valid_attestation: Vec<_> = participants
    .participants()
    .iter()
    .filter(|(_, _, participant_info)| {
        let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
        else { return false; };
        let tee_status = self.reverify_participants(&node_id, tee_upgrade_deadline_duration);
        matches!(tee_status, TeeQuoteStatus::Valid)
    })
    .cloned()
    .collect();

if participants_with_valid_attestation.len() != participants.len() {
    TeeValidationResult::Partial { participants_with_valid_attestation }
} else {
    TeeValidationResult::Full
}
``` [3](#0-2) 

Back in `vote_new_parameters`, a `Partial` result causes an immediate hard error — **no vote is recorded**:

```rust
TeeValidationResult::Partial { participants_with_valid_attestation } => {
    ...
    Err(InvalidParameters::InvalidTeeRemoteAttestation {
        reason: format!("The following participants have invalid TEE status: {:?}", invalid_participants),
    }.into())
}
``` [4](#0-3) 

The sandbox test `vote_new_parameters_errors_if_new_participant_is_missing_valid_attestation` confirms this behavior in production: every participant's vote fails when even one proposed participant lacks a valid attestation. [5](#0-4) 

**Attack path (Byzantine participant below signing threshold):**

1. Network runs with N participants at threshold T (e.g., N = T = 3).
2. One participant deliberately allows their TEE attestation to expire (certificate expiry, or their docker image hash is no longer on the allowlist).
3. Operators need to perform a key refresh (same participant set, same threshold).
4. Every call to `vote_new_parameters` with the full participant set returns `InvalidTeeRemoteAttestation`.
5. Because N = T, the operators cannot propose a reduced set (dropping below threshold). `verify_tee` also refuses to kick anyone out when doing so would drop below threshold. [6](#0-5) 
6. The network is frozen: no resharing, no participant rotation, no threshold change — until the Byzantine participant voluntarily re-submits a valid attestation.

---

### Impact Explanation

This is a **contract execution-flow manipulation that breaks the production safety invariant** that the network can always perform governance operations (resharing) when needed. In the minimum-viable configuration (N = threshold), a single Byzantine participant below the signing threshold can permanently freeze all governance transitions — adding/removing participants, changing the threshold, or performing a key refresh — without any collusion or privileged access. The network continues signing (key shares are unaffected) but cannot evolve its participant set, which is a critical safety property for long-term operation.

---

### Likelihood Explanation

TEE attestations expire naturally (X.509 certificate lifetimes, docker image hash rotation after security patches). A participant who wishes to block a governance change need only stop renewing their attestation. No special capability is required — any single participant strictly below the signing threshold can trigger this. The scenario is realistic in production deployments where the network is bootstrapped at the minimum threshold size.

---

### Recommendation

1. **Decouple attestation validation from vote recording.** Allow a vote to be recorded even when some proposed participants have invalid attestations, provided the valid subset still meets the governance threshold. Only block the final state transition (into Resharing) if the full proposed set is not valid.
2. **Alternatively**, add a variant of `vote_new_parameters` that accepts a subset of the proposed participants (analogous to the report's recommended `distribute(uint256 _start, uint256 _finish)` and `distribute(address[] memory _gauges)` overloads), so operators can vote for a reduced set that excludes the participant with the expired attestation.
3. **Ensure `verify_tee` is always callable as a prerequisite** to kick out participants with expired attestations before a resharing vote, even when the network is at exactly the threshold size (currently blocked by the safety guard).

    pub fn vote_new_parameters(
        &mut self,
        prospective_epoch_id: EpochId,
        proposal: dtos::ProposedThresholdParameters,
    ) -> Result<(), Error> {
        Self::assert_caller_is_signer();
        let proposal: ProposedThresholdParameters = proposal.try_into_contract_type()?;
        log!(
            "vote_new_parameters: signer={}, proposal={:?}",
            env::signer_account_id(),
            proposal,
        );

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        let validation_result = self.tee_state.reverify_and_cleanup_participants(
            proposal.participants(),
            tee_upgrade_deadline_duration,
        );

        let proposed_participants = proposal.participants();
        match validation_result {
            TeeValidationResult::Full => {
                if let Some(new_state) = self
                    .protocol_state
                    .vote_new_parameters(prospective_epoch_id, &proposal)?
                {
                    self.protocol_state = new_state;
                }
                Ok(())
            }
            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            } => {
                let invalid_participants: Vec<_> = proposed_participants
                    .participants()
                    .iter()
                    .filter(|(account_id, _, _)| {
                        !participants_with_valid_attestation
                            .is_participant_given_account_id(account_id)
                    })
                    .collect();

                Err(InvalidParameters::InvalidTeeRemoteAttestation {
                    reason: format!(
                        "The following participants have invalid TEE status: {:?}",
                        invalid_participants
                    ),
                }
                .into())
            }
        }
    }

```rust
    pub fn reverify_and_cleanup_participants(
        &mut self,
        participants: &Participants,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeValidationResult {
        self.allowed_docker_image_hashes
            .cleanup_expired_hashes(tee_upgrade_deadline_duration);

        let participants_with_valid_attestation: Vec<_> = participants
            .participants()
            .iter()
            .filter(|(_, _, participant_info)| {
                // Use the stored NodeId (keyed by TLS public key) so the real
                // `account_public_key` participates in re-verification. If
                // there is no stored attestation for this TLS key, the
                // participant is invalid.
                let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
                else {
                    return false;
                };

                let tee_status =
                    self.reverify_participants(&node_id, tee_upgrade_deadline_duration);

                matches!(tee_status, TeeQuoteStatus::Valid)
            })
            .cloned()
            .collect();

        if participants_with_valid_attestation.len() != participants.len() {
            let participants_with_valid_attestation =
                Participants::init(participants.next_id(), participants_with_valid_attestation);

            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            }
        } else {
            TeeValidationResult::Full
        }
    }

#[tokio::test]
async fn vote_new_parameters_errors_if_new_participant_is_missing_valid_attestation() {
    let SandboxTestSetup {
        worker,
        contract,
        mut mpc_signer_accounts,
        ..
    } = SandboxTestSetup::builder()
        .with_protocols(ALL_PROTOCOLS)
        .build()
        .await;

    let state = get_state(&contract).await;
    let ProtocolContractState::Running(ref running_state) = state else {
        panic!("expected running state");
    };
    let threshold = running_state.parameters.threshold;
    let epoch_id = running_state.keyset.epoch_id;
    let mut proposed_participants: mpc_contract::primitives::participants::Participants =
        (&running_state.parameters.participants).into_contract_type();

    let (new_account, new_account_id) = gen_account(&worker).await;
    let new_participant_info = gen_participant_info();

    // Add the new participant to the participant set, and propose this to the contract.
    proposed_participants
        .insert(new_account_id.clone(), new_participant_info)
        .unwrap();

    let threshold_parameters =
        ThresholdParameters::new(proposed_participants, Threshold::new(threshold.0 + 1)).unwrap();

    mpc_signer_accounts.push(new_account.clone());

    // Vote to transition to resharing state
    for account in &mpc_signer_accounts {
        let call_result = account
            .call(contract.id(), method_names::VOTE_NEW_PARAMETERS)
            .max_gas()
            .args_json(json!({
                "prospective_epoch_id": dtos::EpochId(epoch_id.0 + 1),
                "proposal": ProposedThresholdParameters::new(
                    threshold_parameters.clone(),
                    BTreeMap::new(),
                ),
            }))
            .transact()
            .await
            .unwrap()
            .into_result()
            .expect_err("calling `vote_new_parameters` must fail when one participant has invalid TEE status.");

        let error_message = call_result.to_string();
        let expected_prefix =
            mpc_contract::errors::InvalidParameters::InvalidTeeRemoteAttestation {
                reason: String::new(),
            }
            .to_string();
        assert!(
            error_message.contains(&expected_prefix),
            "Expected InvalidTeeRemoteAttestation error, got: {error_message}"
        );
    }
