### Title
Non-Atomic Eviction in `try_make_space` Allows Zero-Cost Permanent Eviction of Legitimate Mempool Transactions — (File: `crates/apollo_mempool/src/mempool.rs`)

---

### Summary

The mempool's capacity-overflow handler evicts gap-account transactions **before** confirming the incoming transaction can be admitted. If the freed space is still insufficient, the incoming transaction is rejected with `MempoolError::MempoolFull`, but the already-evicted legitimate transactions are **never restored**. Because the triggering transaction is rejected by the mempool and never executed, no on-chain fee is charged to the attacker. This enables a zero-cost, repeatable eviction of valid pending transactions from the mempool — a direct sequencer-layer analog to the M-8 DOS.

---

### Finding Description

**Root cause — `handle_capacity_overflow` / `try_make_space` in `crates/apollo_mempool/src/mempool.rs`**

When a new transaction overflows the mempool's `capacity_in_bytes` limit, `add_tx_validations` calls `handle_capacity_overflow`:

```rust
// mempool.rs:1031-1052
fn handle_capacity_overflow(...) -> Result<(), MempoolError> {
    let creating_gap = (account_has_gap || !account_has_txs) && !closing_gap;
    let required_space = tx.total_bytes().saturating_sub(freed_bytes);
    if !creating_gap && self.try_make_space(required_space) {
        return Ok(());
    }
    Err(MempoolError::MempoolFull)   // ← incoming tx rejected
}
```

`try_make_space` iterates over random gap-accounts and removes their transactions from `tx_pool` until `required_space` bytes are freed:

```rust
// mempool.rs:992-1029
pub fn try_make_space(&mut self, required_space: u64) -> bool {
    ...
    while total_space_freed < required_space {
        let Some(address) = self.get_evictable_account() else { break; };
        let txs = self.tx_pool.account_txs_sorted_by_nonce(address).copied().collect();
        for tx_ref in txs.iter().rev() {
            let tx = self.tx_pool.remove(tx_ref.tx_hash).expect("...");
            total_space_freed += tx.total_bytes();
            evicted_txs.push(*tx_ref);
            ...
        }
        ...
    }
    self.tx_queue.remove_txs(&evicted_txs);
    total_space_freed >= required_space   // ← may return false
}
```

If `try_make_space` returns `false` (the attacker's transaction is larger than the total bytes freed), `handle_capacity_overflow` returns `MempoolError::MempoolFull` and the incoming transaction is rejected. **The evicted transactions are not restored.** The test suite explicitly documents and asserts this behaviour:

```rust
// fee_mempool_test.rs:1936-1938
// Transactions tx1 and tx2 are evicted regardless of whether large_tx is accepted or rejected.
// We do not revert the eviction attempt even if adding large_tx ultimately fails.
assert!(!mempool.tx_pool.contains_account(contract_address!("0x1")));
```

**Attack path**

1. Attacker deploys a custom account contract whose `__validate__` entry point accepts any signature (one-time setup).
2. Attacker crafts an invoke transaction with `nonce == account_nonce` (no gap, so `creating_gap = false`) and a large signature (e.g. 20+ `Felt` elements), making `tx.total_bytes()` exceed the combined size of the gap transactions they wish to evict.
3. The transaction passes full gateway validation: stateless size/resource-bounds checks, stateful `verify_can_pay_committed_bounds` (attacker holds balance but never spends it), and the custom `__validate__` entry point.
4. The gateway calls `mempool_client.add_tx`. The mempool is full with legitimate gap transactions from other users.
5. `handle_capacity_overflow` calls `try_make_space(required_space)`. Gap-account transactions are permanently removed from `tx_pool` and `tx_queue`.
6. `try_make_space` returns `false` (attacker's transaction is still too large). `MempoolError::MempoolFull` is returned to the gateway.
7. The attacker's transaction is **never executed** → **no fee is charged**.
8. Legitimate gap transactions are permanently gone from the mempool.
9. Repeat from step 2 with a fresh nonce.

The attacker's only recurring cost is the CPU time the gateway spends on stateful validation — a cost borne by the sequencer, not the attacker.

---

### Impact Explanation

**High — Mempool admission rejects valid transactions before sequencing.**

Valid transactions that were legitimately admitted to the mempool (gap transactions awaiting gap-fill, e.g. the `invoke` half of a `deploy_account + invoke` pair) are permanently evicted without being sequenced. The attacker pays zero on-chain fees per eviction cycle. Repeated attacks keep the mempool cleared of any gap transactions, preventing those accounts from ever having their transactions sequenced until the attack stops.

---

### Likelihood Explanation

**Medium.** The attacker must:
- Control an account whose `__validate__` accepts an oversized signature (achievable with a custom account contract).
- Craft a transaction whose serialised byte size exceeds the combined size of the gap transactions targeted for eviction.
- Maintain sufficient STRK balance to pass `verify_can_pay_committed_bounds` (balance is never spent).

The test case `rejects_or_accepts_tx_based_on_freed_space` demonstrates the attack is reachable with a signature of only 20 `Felt` elements (~640 bytes), well within any practical size limit. No privileged access is required.

---

### Recommendation

Make eviction **atomic** with admission. Before calling `try_make_space`, snapshot the transactions to be evicted. If `try_make_space` returns `false` and the incoming transaction is ultimately rejected, re-insert the snapshotted transactions into `tx_pool` and `tx_queue`. Alternatively, perform a dry-run capacity check (count bytes of evictable gap accounts) before committing any removal, and only proceed with eviction if the check confirms the incoming transaction will fit.

```rust
        // The replaced transaction is still pooled, so its bytes still count toward
        // `size_in_bytes()`. Credit what its removal will free: a same-size bump nets to zero (no

        // with how a fresh next-nonce transaction is treated. The removal happens only after
        // capacity is confirmed below, so a rejected incoming transaction never strands the
        // account.

                .get_by_tx_hash(reference.tx_hash)
                .expect("Replacement target from pool must exist.")
                .total_bytes()

            self.handle_capacity_overflow(tx, account_nonce, freed_bytes)?;

        // transaction is inserted by the caller.
```

```rust
    pub fn try_make_space(&mut self, required_space: u64) -> bool {
        let mut total_space_freed = 0;
        let mut evicted_txs = Vec::new();

        while total_space_freed < required_space {
            let Some(address) = self.get_evictable_account() else {
                break;
            };

            let txs: Vec<_> = self.tx_pool.account_txs_sorted_by_nonce(address).copied().collect();
            for tx_ref in txs.iter().rev() {
                let tx = self
                    .tx_pool
                    .remove(tx_ref.tx_hash)
                    .expect("Transaction must exist in the pool.");
                total_space_freed += tx.total_bytes();
                evicted_txs.push(*tx_ref);
                metric_count_evicted_txs(1);
                self.decrement_stuck_txs_if_gap_account(address, 1);
                if total_space_freed >= required_space {
                    break;
                }
            }

            // Clean up if account is now empty.
            if !self.tx_pool.contains_account(address) {
                self.accounts_with_gap.swap_remove(&address);
            }
        }

        // Keep the queue consistent with the pool: the evicted txs were removed from the pool, so
        // drop their (now-orphaned) queue references too. In fee mode gap accounts are never
        // queued, so this is a no-op there; in Echonet/FIFO mode they are, and skipping this leaves
        // a dangling reference that panics the next `get_txs`.
        self.tx_queue.remove_txs(&evicted_txs);

        total_space_freed >= required_space
    }
```

```rust
    fn handle_capacity_overflow(
        &mut self,
        tx: &InternalRpcTransaction,
        account_nonce: Nonce,
        freed_bytes: u64,
    ) -> Result<(), MempoolError> {
        let address = tx.contract_address();

        let account_has_gap = self.accounts_with_gap.contains(&address);
        let account_has_txs = self.tx_pool.contains_account(address);
        let closing_gap = tx.nonce() == account_nonce;
        let creating_gap = (account_has_gap || !account_has_txs) && !closing_gap;

        // Only the net growth must be evicted: an accompanying replacement removal frees
        // `freed_bytes` (0 when there is no replacement).
        let required_space = tx.total_bytes().saturating_sub(freed_bytes);
        if !creating_gap && self.try_make_space(required_space) {
            return Ok(());
        }

        Err(MempoolError::MempoolFull)
    }
```
