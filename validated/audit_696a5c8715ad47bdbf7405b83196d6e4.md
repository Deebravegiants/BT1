### Title
Passphrase-knowledge guard bypassed via `current_passphrase=None` in `KeyringWrapper.set_master_passphrase` ŌĆö (`chia/util/keyring_wrapper.py`)

### Summary
The three-clause AND guard that enforces proof-of-knowledge of the current master passphrase short-circuits to `False` when `current_passphrase=None` is supplied, allowing any caller to invoke `set_master_passphrase` on a populated keyring without knowing the existing passphrase.

### Finding Description
In `KeyringWrapper.set_master_passphrase`, the guard:

```python
if (
    self.has_master_passphrase()
    and current_passphrase is not None          # ŌåÉ bypass here
    and not self.master_passphrase_is_valid(current_passphrase)
):
    raise KeychainCurrentPassphraseIsInvalid
``` [4](#0-3) 

is intended to require a valid current passphrase when the keyring has content. The `current_passphrase is not None` clause was likely added to handle the no-passphrase case, but it creates a bypass: supplying `None` makes the entire condition `False` regardless of `has_master_passphrase()`. The cached passphrase is then immediately overwritten with the attacker-controlled value at line 258 before any further check. [5](#0-4) 

### Impact Explanation
If the daemon's `set_keyring_passphrase` RPC handler forwards a caller-supplied `None` to this function (unverified but plausible given the pattern), a local attacker can re-encrypt the keyring to an attacker-controlled passphrase, then decrypt it to extract all stored private keys. This is unauthorized key use / keychain authorization bypass ŌĆö a High-severity impact.

### Likelihood Explanation
Requires local access to the daemon WebSocket (same-user or same-machine attacker with access to the SSL credentials in `~/.chia/`). Not remotely exploitable without those credentials.

### Recommendation
Fix the guard to treat `None` as an invalid passphrase rather than skipping validation:

```python
if self.has_master_passphrase() and not self.master_passphrase_is_valid(current_passphrase or ""):
    raise KeychainCurrentPassphraseIsInvalid
``` [1](#0-0)
