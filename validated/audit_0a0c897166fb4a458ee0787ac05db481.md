### Title
Unvalidated `builder` Field in `ProposalInit` Allows Proposer to Redirect Block Fees and Corrupt Committed Sequencer Address — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` never checks the `builder` field of an incoming `ProposalInit`. Because `builder` is passed verbatim as `sequencer_address` into `BlockInfo` and then into `PartialBlockHashComponents`, a malicious-but-legitimately-elected proposer can set `builder` to any address. All validators accept the proposal (the batcher-computed commitment matches because it also uses the proposer-supplied `builder`), the block is committed with the attacker-controlled sequencer address, and all transaction fees in that block are credited to the attacker's address.

### Finding Description

`ProposalInit` carries a `builder` field — the address of the block builder/sequencer. [1](#0-0) 

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in `BlockInfo`: [2](#0-1) 

`PartialBlockHashComponents::new` then stores that address as `sequencer`: [3](#0-2) 

`sequencer` is chained into the Poseidon block hash: [4](#0-3) 

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, timestamp, L1 gas prices, and `fee_proposal_fri` — but **never** `builder`: [5](#0-4) 

`ProposalInitValidation` has no `builder` field at all: [6](#0-5) 

`initiate_validation` calls `convert_to_sn_api_block_info(init)` with the unvalidated `init`, so the batcher executes the block with the attacker-supplied `sequencer_address`: [7](#0-6) 

The batcher computes `partial_block_hash` from `PartialBlockHashComponents` (which includes the attacker's `sequencer`), and the validator computes `batcher_block_commitment = proposal_commitment_from(partial_block_hash, fee_proposal)`. The proposer's `ProposalFin` carries the same commitment (built with the same attacker-controlled `builder`), so the final comparison passes: [8](#0-7) 

### Impact Explanation

**Fee theft (Critical — economic impact):** The blockifier credits transaction fees to `block_context.block_info.sequencer_address`. Because that address is `init.builder`, a proposer who sets `builder` to their own address receives all fees from every transaction in the block instead of the legitimate sequencer.

**Wrong `get_sequencer_address` syscall result (Critical — wrong execution result):** The deprecated syscall handler returns `block_info.sequencer_address`: [9](#0-8) 

Any contract that gates logic on the sequencer address (e.g., admin checks, fee-routing contracts) will observe the attacker-controlled address and may execute incorrect branches.

**Wrong block hash committed (Critical — wrong state commitment):** The `sequencer` field is part of `PartialBlockHashComponents` stored in storage and used to compute the final `BlockHash`. The committed block hash is permanently wrong. [10](#0-9) 

### Likelihood Explanation

Low-to-moderate. The attacker must be a legitimate consensus participant elected as proposer for a round. In a permissioned validator set this means a malicious validator; in a more open deployment any elected proposer can trigger it. No special privileges beyond being elected proposer are required — the attack is a single-field modification to the `ProposalInit` message.

### Recommendation

Add `builder` to `ProposalInitValidation` and check it in `is_proposal_init_valid`. The expected `builder` address is already known to each validator node via `self.config.static_config.builder_address` (the same field the proposer uses when building): [11](#0-10) 

The validator should reject any `ProposalInit` whose `builder` does not equal the locally-configured `builder_address` (or the committee-derived builder address once that is available).

### Proof of Concept

1. Attacker is elected as proposer for height H, round R.
2. Attacker constructs `ProposalInit` with all valid fields **except** `builder`, which is set to `ATTACKER_ADDRESS` instead of the legitimate sequencer address.
3. Attacker computes `block_info` with `sequencer_address = ATTACKER_ADDRESS`, executes the block, obtains `partial_block_hash` (which commits `ATTACKER_ADDRESS` as sequencer), and sends `ProposalFin` with `proposal_commitment = proposal_commitment_from(partial_block_hash, fee_proposal)`.
4. Each validator calls `is_proposal_init_valid` — passes (no `builder` check).
5. Each validator calls `initiate_validation` → `convert_to_sn_api_block_info(init)` → batcher executes with `sequencer_address = ATTACKER_ADDRESS`.
6. Batcher computes the same `partial_block_hash` → same `batcher_block_commitment` → matches `received_fin.proposal_commitment` → `validate_proposal` returns `Ok`.
7. Consensus reaches decision; `decision_reached` commits the block. All fees in block H are credited to `ATTACKER_ADDRESS`. `get_sequencer_address` syscall returns `ATTACKER_ADDRESS` for all transactions in the block. The committed `PartialBlockHashComponents` in storage contains `sequencer = ATTACKER_ADDRESS`. [12](#0-11) [13](#0-12) [14](#0-13)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
