### Title
Missing Cross-Map Class Hash Validation in P2P State Diff and Class Sync Allows Malicious Peer to Inject Cairo 0 Class Under a Cairo 1 Hash — (`crates/apollo_p2p_sync/src/client/state_diff.rs`, `crates/apollo_p2p_sync/src/client/class.rs`)

### Summary
A malicious unauthenticated P2P peer can craft a state diff that places the same class hash X in both `class_hash_to_compiled_class_hash` (Cairo 1) and `deprecated_declared_classes` (Cairo 0). No existing validation rejects this. The class sync then accepts a `DeprecatedContractClass` payload for X (because X is in `deprecated_declared_classes`) and a `ContractClass` payload for X (because X is in `class_hash_to_compiled_class_hash`), calling both `add_deprecated_class(X, malicious_cairo0)` and `add_class(any_sierra_with_hash_Z)`. Because `add_class` stores the Sierra under its actual hash Z ≠ X, `get_executable(X)` subsequently returns the attacker-controlled Cairo 0 class. Any contract deployed under class X will execute the wrong code.

### Finding Description

**Gap 1 — `unite_state_diffs`:** `StateDiffChunk::DeprecatedDeclaredClass` is pushed to `deprecated_declared_classes` with no check against `class_hash_to_compiled_class_hash`. [10](#0-9) 

**Gap 2 — `validate_deprecated_declared_classes_non_conflicting`:** Only checks for intra-list duplicates; never checks whether any hash in `deprecated_declared_classes` also appears in `class_hash_to_compiled_class_hash`. [2](#0-1) 

**Gap 3 — `parse_data_for_block` (class.rs):** `is_declared` for `DeprecatedContractClass` checks only `deprecated_declared_classes`; `is_declared` for `ContractClass` checks only `declared_classes`. There is no cross-map guard preventing the same hash from being accepted as both types in the same block. [4](#0-3) 

**Gap 4 — `write_to_storage` (class.rs):** `add_class` is called with the Sierra class content; the class manager computes the storage key from the class hash of that content, not from the peer-supplied `class_hash`. The mismatch is undetected (acknowledged by a TODO comment). [6](#0-5) 

### Impact Explanation
`get_executable(X)` returns an attacker-controlled Cairo 0 class for a hash the state diff records as Cairo 1. Any contract whose class hash is X will execute the wrong bytecode, producing wrong state transitions, wrong events, wrong return values, and wrong fee accounting — all of which are Critical-scope impacts ("Wrong compiled class, CASM/native artifact, class hash, or contract code selected for execution").

### Likelihood Explanation
The attack requires only that the victim node connects to a single malicious P2P peer during sync. No privileged access is needed. The peer is unauthenticated and can send arbitrary `StateDiffChunk` and `ApiContractClass` messages. The two missing validations are independent and both absent, so no partial fix closes the hole.

### Recommendation
1. In `unite_state_diffs`, when processing `StateDiffChunk::DeprecatedDeclaredClass`, reject if `state_diff.class_hash_to_compiled_class_hash.contains_key(&class_hash)`.
2. In `validate_deprecated_declared_classes_non_conflicting` (or as a separate post-merge check), verify that `deprecated_declared_classes` and `class_hash_to_compiled_class_hash` are disjoint.
3. In `parse_data_for_block` (class.rs), when accepting a `DeprecatedContractClass` for `class_hash`, additionally assert `!declared_classes.contains_key(&class_hash)`, and vice versa.
4. Implement the acknowledged TODO: after `add_class` returns, verify that the returned `class_hash` matches the expected hash from the state diff; treat a mismatch as a `BadPeerError`.

        let sierra_class = SierraContractClass::try_from(&class)?;
        let class_hash = sierra_class.calculate_class_hash();
        if let Ok(Some(executable_class_hash_v2)) =
            self.classes.get_executable_class_hash_v2(class_hash)
        {
            // Class already exists.
            return Ok(ClassHashes { class_hash, executable_class_hash_v2 });
        }

        let compilation_start_time = Instant::now();
        let (raw_executable_class, executable_class_hash_v2) =
            self.compiler.compile(class.clone()).await.map_err(|err| match err {
                SierraCompilerClientError::SierraCompilerError(error) => {
                    ClassManagerError::SierraCompiler { class_hash, error }
                }
                SierraCompilerClientError::ClientError(error) => {
                    ClassManagerError::Client(error.to_string())
                }
            })?;
        debug!(
            %class_hash,
            compiled_class_hash = %executable_class_hash_v2,
            compilation_elapsed_ms = compilation_start_time.elapsed().as_millis(),
            class_size_bytes =
                class.size().map_or("Failed to get class size".to_owned(), |size| size.to_string()),
            casm_size_bytes =
                raw_executable_class.size().map_or("Failed to get casm size".to_owned(), |size| size.to_string()),
            "Finished compiling class."
        );

        self.validate_class_length(&raw_executable_class)?;
        Self::validate_class_version(&sierra_class)?;
        self.classes.set_class(
            class_hash,
            class,
            executable_class_hash_v2,
            raw_executable_class,
        )?;

        let class_hashes = ClassHashes { class_hash, executable_class_hash_v2 };
        Ok(class_hashes)
    }
