Audit Report

## Title
Missing Staleness, Round Completeness, and Validity Checks on Chainlink `latestRoundData()` Allows Stale Price to Corrupt rsETH Exchange Rate - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards `roundId`, `updatedAt`, and `answeredInRound`, performing no staleness, round-completeness, or sign/zero check on the returned price. A stale or zero price propagates directly into `LRTOracle._getTotalEthInProtocol()`, corrupting the rsETH/ETH exchange rate stored on-chain. Because `updateRSETHPrice()` is a public, permissionless function, an attacker can time the price update to exploit the stale value for direct theft of user funds or theft of yield.

## Finding Description

**Root cause â€” `ChainlinkPriceOracle.getAssetPrice()` (L49â€“55):**

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();   // roundId, updatedAt, answeredInRound all dropped
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All three staleness guards are absent: no `answeredInRound >= roundId` check, no `updatedAt != 0` check, no `block.timestamp - updatedAt <= MAX_STALENESS` check, and no `price > 0` check. The sibling contract `ChainlinkOracleForRSETHPoolCollateral.getRate()` (L27â€“32) applies all three guards, confirming the pattern is known and intentionally used elsewhere in the same repository.

**Propagation path:**

1. `LRTOracle._getTotalEthInProtocol()` (L336â€“343) calls `getAssetPrice(asset)` for every supported LST collateral and accumulates `totalETHInProtocol += totalAssetAmt.mulWad(assetER)`.
2. `_updateRsETHPrice()` (L250) computes `newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply)` and persists it to `rsETHPrice`.
3. `updateRSETHPrice()` (L87â€“89) is `public whenNotPaused` with no role restriction â€” any external caller can trigger it.

**Why existing guards are insufficient:**

The `pricePercentageLimit` circuit-breaker (L252â€“283) does not compensate for this:
- It defaults to `0`, in which case the check is entirely skipped (`pricePercentageLimit > 0 &&` at L257 and L273).
- Even when non-zero, the upside check only fires when `newRsETHPrice > highestRsethPrice` (L252). A stale inflated price that is still below the historical peak bypasses the check entirely, leaving the full stale-price window exploitable.
- The downside pause (L270â€“283) only triggers if the price drops beyond the threshold relative to the all-time high, not relative to the true current market price.

## Impact Explanation

**Scenario A â€” Stale price is lower than actual (deflated):**
`totalETHInProtocol` is understated â†’ `rsETHPrice` is deflated. An attacker calls `updateRSETHPrice()` to lock in the deflated price, then deposits LST collateral and receives more rsETH shares than the true value warrants. When Chainlink updates, the attacker redeems at the corrected rate, extracting value from existing holders. **Impact: Theft of unclaimed yield (High).**

**Scenario B â€” Stale price is higher than actual (inflated):**
`rsETHPrice` is inflated. An attacker calls `updateRSETHPrice()` to lock in the inflated price, then queues a withdrawal priced at the inflated rate, receiving more underlying ETH than their rsETH shares are worth. **Impact: Direct theft of user funds (Critical).**

## Likelihood Explanation

Chainlink LST/ETH feeds operate on heartbeats of up to 24 hours plus a deviation threshold. During the heartbeat window the on-chain price can diverge from the true market price by the full deviation band without triggering an update â€” a known, recurring condition. `updateRSETHPrice()` is permissionless, so an attacker needs only to monitor off-chain prices and call the function at the moment the stale price is most favorable. No special privileges, no victim mistakes, and no external protocol compromise are required.

## Recommendation

1. **Short term:** Add staleness and validity guards to `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:
   ```solidity
   (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
       priceFeed.latestRoundData();
   require(answeredInRound >= roundId, "Stale price");
   require(updatedAt != 0, "Incomplete round");
   require(price > 0, "Invalid price");
   require(block.timestamp - updatedAt <= MAX_STALENESS, "Price too old");
   ```
   `MAX_STALENESS` should be set per-feed to match its published heartbeat (e.g., 3600 s for 1-hour feeds, 86 400 s for 24-hour feeds) plus a small buffer.

2. **Long term:** Restrict `updateRSETHPrice()` to a trusted keeper role, or add a circuit-breaker that refuses to persist a new `rsETHPrice` if any individual feed is stale, preventing opportunistic timing attacks.

## Proof of Concept

1. Chainlink's stETH/ETH feed has a 24-hour heartbeat. At hour 23, the true stETH price has dropped 1.5 % but the on-chain feed still reports the old (higher) price. `pricePercentageLimit` is either 0 or set to 2 %, and `highestRsethPrice` equals the current stale-inflated level, so neither circuit-breaker fires.
2. Attacker observes the discrepancy off-chain.
3. Attacker calls `LRTOracle.updateRSETHPrice()` (no access control). `_getTotalEthInProtocol()` calls `ChainlinkPriceOracle.getAssetPrice(stETH)`, which returns the stale inflated price. `rsETHPrice` is set above its true value.
4. Attacker immediately calls `LRTWithdrawalManager` to queue a withdrawal, receiving a claim priced at the inflated `rsETHPrice`.
5. When Chainlink updates at hour 24, the true price is reflected. The attacker's withdrawal claim was locked in at the inflated rate, extracting excess ETH from the pool at the expense of remaining rsETH holders.

**Foundry fork test outline:**
- Fork mainnet at a block where the stETH/ETH feed is near its heartbeat boundary.
- Deploy or point to the live `ChainlinkPriceOracle` and `LRTOracle`.
- `vm.warp` to a timestamp within the heartbeat window but after the true price has moved (simulate via `vm.mockCall` on `latestRoundData` returning an old `updatedAt`).
- Call `updateRSETHPrice()` as an unprivileged address; assert `rsETHPrice` diverges from the true rate.
- Execute a withdrawal and assert the attacker receives more ETH than the true rsETH value.
