### Title
Unchecked CKD Response for `AppPublicKey` Variant Allows Single Byzantine Participant to Deliver Forged Key Material - (File: crates/contract/src/lib.rs)

### Summary

`respond_ckd` in `MpcContract` performs a cryptographic output check on the CKD response only when the request uses the `CKDAppPublicKey::AppPublicKeyPV` variant. When the request uses `CKDAppPublicKey::AppPublicKey`, the check is silently skipped and any `CKDResponse` is accepted without verification. A single Byzantine attested participant — strictly below the signing threshold — can call `respond_ckd` with arbitrary key material and the contract will resolve the pending yield, delivering attacker-controlled key output to the requesting user.

### Finding Description

In `respond_ckd`, after confirming the caller is an attested participant and the protocol is active, the contract branches on the `app_public_key` field of the request:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}          // ← no check, response accepted as-is
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}

pending_requests::resolve_yields_for(
    &mut self.pending_ckd_requests,
    &request,
    serde_json::to_vec(&response).unwrap(),
)
``` [1](#0-0) 

For `AppPublicKeyPV`, `ckd_output_check` verifies the MPC-produced response is consistent with the expected application public key and the domain's root key. For `AppPublicKey`, the arm is a no-op (`{}`): the public key embedded in the variant is ignored, the response bytes are never validated, and `resolve_yields_for` immediately resumes all queued yields with whatever `response` the caller supplied. [2](#0-1) 

The analog to the Frax bug is exact: just as `collateral_token.transferFrom` returning `false` was never checked, the `CKDResponse` for `AppPublicKey` requests is never checked — the operation proceeds as if it succeeded.

### Impact Explanation

A single Byzantine attested participant (one node, below threshold) can:

1. Observe a pending `request_app_private_key` request whose `app_public_key` field is `AppPublicKey(pk)`.
2. Construct an arbitrary `CKDResponse` containing key material the attacker controls or knows.
3. Call `respond_ckd(request, forged_response)` — the contract passes all guards (attested participant, protocol running, BLS domain check) and reaches the match arm, which is a no-op.
4. `resolve_yields_for` resumes the user's yield with the forged response bytes.

The user's contract or application receives attacker-chosen key material as the output of the confidential key derivation. Because the attacker chose the key, they can sign arbitrary messages under the derived key, impersonating the user for any downstream protocol that trusts the CKD output. This constitutes **confidential key derivation output delivered without the required participant authorization** — the threshold MPC protocol is bypassed entirely for this request class.

### Likelihood Explanation

The attacker must be an attested participant (pass TEE attestation). In production this requires running in Intel TDX. However, only **one** such participant is needed — the threshold is irrelevant because the contract imposes no quorum on `respond_ckd` calls. Any single participant who turns Byzantine after attestation, or whose TEE is compromised, can exploit this for every `AppPublicKey` CKD request in flight. The `AppPublicKey` variant is a valid, documented request path exposed in the public ABI. [3](#0-2) 

### Recommendation

Apply `ckd_output_check` (or an equivalent verification) unconditionally for both `AppPublicKey` and `AppPublicKeyPV` variants. The public key embedded in `AppPublicKey(_)` should be extracted and used to verify the response, mirroring the `AppPublicKeyPV` branch:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
```

If `AppPublicKey` intentionally omits the check for a design reason, that reason must be documented and the security implication (single-participant forgery) must be explicitly accepted. If no such reason exists, the variant should be removed or unified with `AppPublicKeyPV`.

### Proof of Concept

1. Deploy the contract in running state with one attested participant `alice`.
2. User `bob` calls `request_app_private_key` with `app_public_key: AppPublicKey(legitimate_pk)` and attaches the required deposit. The request enters `pending_ckd_requests`.
3. `alice` (Byzantine) constructs `forged_response` containing a BLS key she controls.
4. `alice` calls `respond_ckd(bob_request, forged_response)`.
5. The contract executes the `AppPublicKey(_) => {}` arm — no check runs.
6. `resolve_yields_for` resumes `bob`'s yield with `forged_response` bytes.
7. `bob`'s callback receives `forged_response` as the legitimate CKD output.
8. `alice` can now sign under the forged key, impersonating `bob` in any protocol that trusts the CKD output. [4](#0-3) [5](#0-4)

    #[handle_result]
    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,

        )

pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
