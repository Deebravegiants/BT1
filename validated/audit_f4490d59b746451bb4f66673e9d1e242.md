### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Redirect All Block Fees and Corrupt Block Hash — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries a `builder` field (the sequencer/block-builder address). The validator's `is_proposal_init_valid` function checks every other economically-sensitive field (`height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, `fee_proposal_fri`) but **never checks `builder`**. `convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in the `BlockInfo` that drives execution. Because `sequencer_address` is the fee-transfer recipient and is committed into the block hash, a legitimate-but-malicious proposer can set `builder` to any address, redirect all transaction fees from the block to that address, and produce a block hash that diverges from what honest validators would compute for the same transactions.

---

### Finding Description

`ProposalInit.builder` is defined as "Address of the one who builds/sequences the block": [1](#0-0) 

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`. The `builder` field is **absent from every check**: [2](#0-1) 

`ProposalInitValidation` — the struct that carries the locally-trusted reference values — has no `builder_address` field at all: [3](#0-2) 

After `is_proposal_init_valid` passes, `convert_to_sn_api_block_info` maps `init.builder` verbatim to `sequencer_address`: [4](#0-3) 

This `BlockInfo` is forwarded to the batcher via `ValidateBlockInput`: [5](#0-4) 

Inside the batcher, `PartialBlockHashComponents::new` captures `block_info.sequencer_address` as a committed field: [6](#0-5) 

`calculate_block_hash` chains `sequencer` into the Poseidon hash: [7](#0-6) 

`execute_fee_transfer` sends all transaction fees to `block_context.block_info.sequencer_address`: [8](#0-7) 

`update_state_sync_with_new_block` also stores `init.builder` as the block's sequencer in the header committed to state sync: [9](#0-8) 

---

### Impact Explanation

A legitimate proposer (whose `proposer` address is verified by the consensus manager) can set `builder` to any arbitrary address in the `ProposalInit` they broadcast. Because validators never check `builder`:

1. **Fee theft**: Every fee-paying transaction in the block transfers its fee to the attacker-controlled `builder` address instead of the honest sequencer. This is a direct, per-block economic loss.
2. **Wrong block hash**: `sequencer_address` is a committed field in `PartialBlockHashComponents` and therefore in the final block hash. The committed block hash will differ from what any honest node would compute for the same transaction set with the correct sequencer address.
3. **Wrong state sync header**: The block header stored to state sync carries the attacker-supplied `builder` as `sequencer`, corrupting the authoritative view of the chain.

Matching allowed impacts:
- *Critical — Incorrect fee/balance with economic impact* (all block fees redirected).
- *Critical — Wrong state/receipt/event from blockifier execution* (wrong fee-transfer call info, wrong sequencer in state diff).
- *High — RPC execution/simulation returns authoritative-looking wrong value* (block hash, sequencer field in block header).

---

### Likelihood Explanation

Any validator node in the committee can become the legitimate proposer for a round. No external capability is required beyond being scheduled as proposer. The malicious proposer simply sets `builder` to a chosen address before broadcasting `ProposalInit`. All other validators will accept the proposal because `is_proposal_init_valid` does not check `builder`. The attack is silent, requires no special tooling, and can be repeated every round the attacker is scheduled to propose.

---

### Recommendation

Add `builder_address` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`:

```rust
// In ProposalInitValidation
pub builder_address: ContractAddress,

// In is_proposal_init_valid
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

Populate `builder_address` in `validate_proposal` from `self.config.static_config.builder_address` (the same source used by the proposer in `initiate_build`): [10](#0-9) 

---

### Proof of Concept

1. Attacker is the scheduled proposer for height H, round R.
2. Attacker constructs `ProposalInit` with all fields matching what validators expect **except** `builder = ATTACKER_WALLET`.
3. Attacker broadcasts the proposal stream.
4. Each validator calls `is_proposal_init_valid` — all checks pass because `builder` is never examined.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)` → `sequencer_address = ATTACKER_WALLET`.
6. The batcher executes all transactions; every `execute_fee_transfer` sends fees to `ATTACKER_WALLET`.
7. `BlockExecutionArtifacts::new` → `PartialBlockHashComponents::new` → `calculate_block_hash` commits `ATTACKER_WALLET` as `sequencer` into the block hash.
8. `ProposalFin` comparison passes (the batcher's commitment matches the proposer's because both used `ATTACKER_WALLET`).
9. Consensus reaches decision; the block with wrong sequencer address and redirected fees is finalized. [11](#0-10) [12](#0-11) [13](#0-12)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
