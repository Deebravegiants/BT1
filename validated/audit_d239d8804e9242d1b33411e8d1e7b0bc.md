### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Redirect All Block Fees and Corrupt Block Hash - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates many fields of a received `ProposalInit` (height, L1/L2 gas prices, DA mode, starknet version, fee proposal) but never checks the `builder` field. Because `convert_to_sn_api_block_info` maps `init.builder` directly to `BlockInfo.sequencer_address`, a malicious proposer can set `builder` to any arbitrary address. Every validator re-executes the block with that address as the sequencer, routing all transaction fees to the attacker-controlled address and embedding the wrong sequencer address in the block hash.

---

### Finding Description

**Step 1 – `builder` is proposer-supplied and flows into `BlockInfo.sequencer_address`.**

In `convert_to_sn_api_block_info` the mapping is unconditional:

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs  line 332
sequencer_address: init.builder,
``` [1](#0-0) 

This `BlockInfo` is passed to the batcher for both the proposer path (`initiate_build` → `propose_block`) and the validator path (`initiate_validation` → `validate_block`). [2](#0-1) 

**Step 2 – `is_proposal_init_valid` never checks `builder`.**

The function validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, timestamp bounds, all four L1 gas prices (within margin), and `fee_proposal_fri`. The `builder` field is absent from every check. [3](#0-2) 

**Step 3 – `sequencer_address` is the fee-transfer recipient for every transaction in the block.**

`execute_fee_transfer` hard-codes `block_context.block_info.sequencer_address` as the ERC-20 transfer recipient:

```rust
// crates/blockifier/src/transaction/account_transaction.rs  line 572
calldata: calldata![
    *block_context.block_info.sequencer_address.0.key(), // Recipient.
    lsb_amount,
    msb_amount
],
``` [4](#0-3) 

`add_fee_to_sequencer_balance` also writes the fee credit to the storage slot keyed by `block_context.block_info.sequencer_address`. [5](#0-4) 

**Step 4 – `sequencer_address` is committed into the block hash.**

`PartialBlockHashComponents::new` copies `block_info.sequencer_address` into the `sequencer` field:

```rust
// crates/starknet_api/src/block_hash/block_hash_calculator.rs  line 231
sequencer: SequencerContractAddress(block_info.sequencer_address),
``` [6](#0-5) 

`calculate_block_hash` chains this value directly into the Poseidon hash: [7](#0-6) 

**Step 5 – The codebase itself acknowledges `builder` is not yet committee-derived.**

```rust
// crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs  line 806
// TODO(Asmaa): Get it from committee once we have it.
builder_address: self.config.static_config.builder_address,
``` [8](#0-7) 

This confirms that `builder` is intended to be committee-validated in the future but is currently unchecked, leaving the door open for a proposer to supply any value.

---

### Impact Explanation

A malicious proposer sets `ProposalInit.builder` to an attacker-controlled address `A`. Every validator calls `is_proposal_init_valid`, which passes (no `builder` check), then calls `initiate_validation` → `convert_to_sn_api_block_info`, which sets `sequencer_address = A`. The batcher executes all transactions with `A` as the sequencer:

1. **Fee theft (Critical economic impact):** Every `execute_fee_transfer` call in the block transfers fees to `A`. The state diff written to storage reflects `A`'s balance increasing, not the legitimate sequencer's.
2. **Wrong block hash committed:** `PartialBlockHashComponents.sequencer = A` is hashed into `PartialBlockHash`, which becomes the `ProposalCommitment`. The committed block hash is permanently wrong.
3. **Wrong `get_sequencer_address` syscall:** Any contract that calls `get_sequencer_address` during execution in that block receives `A` instead of the real sequencer, potentially corrupting contract-level logic that depends on the sequencer identity.

Because all validators execute with the same `init.builder`, they all compute the same (corrupted) `ProposalCommitment`, so consensus reaches agreement on the malicious block.

---

### Likelihood Explanation

The attacker must be selected as the proposer for a round, which requires committee membership (staking). Once selected, the attack requires only setting one field in the `ProposalInit` message before broadcasting. No additional capability is needed. In a permissioned or semi-permissioned validator set this is a realistic threat from any compromised or malicious validator node.

---

### Recommendation

Add a `builder` check inside `is_proposal_init_valid`. Until the committee-derived builder address is available (per the TODO), validators should reject any proposal whose `builder` does not match the locally-configured `builder_address`:

```rust
// In is_proposal_init_valid, after the existing height/da_mode/l2_gas_price check:
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

`ProposalInitValidation` should carry an `expected_builder: ContractAddress` field populated from `self.config.static_config.builder_address` at the call sites in `sequencer_consensus_context.rs`.

---

### Proof of Concept

1. Attacker controls validator node `V` that is selected as proposer for height `H`, round `R`.
2. `V` constructs `ProposalInit` with all legitimate fields **except** `builder: attacker_address` (any address the attacker controls).
3. `V` broadcasts the proposal stream to all peers.
4. Each peer calls `is_proposal_init_valid` — passes, because `builder` is not checked.
5. Each peer calls `initiate_validation` → `convert_to_sn_api_block_info` → `BlockInfo { sequencer_address: attacker_address, ... }`.
6. The batcher executes all transactions; every `execute_fee_transfer` sends fees to `attacker_address`.
7. `PartialBlockHashComponents { sequencer: attacker_address, ... }` is hashed into `ProposalCommitment`.
8. All validators agree on this commitment; consensus decides the block.
9. All transaction fees for block `H` are credited to `attacker_address` in the canonical state.

async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
