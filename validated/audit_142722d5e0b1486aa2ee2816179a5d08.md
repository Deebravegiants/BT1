### Title
Unvalidated `builder` Address in `ProposalInit` Allows Malicious Proposer to Redirect All Block Transaction Fees to an Arbitrary Address — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` enforces tight bounds on every economically-sensitive field in `ProposalInit` (L1/L2 gas prices, `l1_da_mode`, `l2_gas_price_fri`, `fee_proposal_fri`, `starknet_version`, `version_constant_commitment`) but never checks the `builder` field. `builder` is passed verbatim as `sequencer_address` into `BlockInfo` and is the sole recipient of every fee transfer executed in the block. A malicious proposer can set `builder` to any address, causing all validator nodes to execute the block with that address as the fee recipient, permanently redirecting all transaction fees and producing a block hash that commits to the wrong sequencer address.

### Finding Description

**Root cause — missing field in `ProposalInitValidation` and `is_proposal_init_valid`**

`ProposalInitValidation` carries the validator's locally-derived reference values for every field it enforces: [1](#0-0) 

`builder` is absent from this struct. `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas-price fields, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`, but never reads or compares `init_proposed.builder`: [2](#0-1) 

**Propagation — `builder` becomes `sequencer_address` in `BlockInfo`**

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address`: [3](#0-2) 

This `BlockInfo` is forwarded to the batcher via `ValidateBlockInput`: [4](#0-3) 

**Economic impact — `sequencer_address` is the fee recipient**

Every fee transfer in the blockifier sends tokens to `block_context.block_info.sequencer_address`: [5](#0-4) 

**Commitment impact — `sequencer_address` is hashed into the block hash**

`PartialBlockHashComponents::new` includes `sequencer_address`: [6](#0-5) 

`calculate_block_hash` chains it into the Poseidon hash: [7](#0-6) 

Because both the proposer and every validator derive `sequencer_address` from the same unchecked `ProposalInit.builder`, they all produce the same `partial_block_hash` and the same `ProposalCommitment`. The `ProposalFinMismatch` guard therefore never fires: [8](#0-7) 

### Impact Explanation

**Critical — Incorrect fee accounting with direct economic impact.**

A malicious proposer sets `builder` to an attacker-controlled address. Every validator node executes the block with that address as `sequencer_address`. All transaction fees for the entire block are transferred to the attacker's address. The legitimate sequencer receives nothing. The committed block hash encodes the attacker's address as the sequencer, so the on-chain record is permanently wrong. The state diff written to storage reflects the attacker's fee-token balance increase, not the legitimate sequencer's.

This matches the allowed impact: *"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."*

### Likelihood Explanation

**Medium.** The attacker must be a validator node that wins a proposer slot. In a small, permissioned validator set this is a realistic threat. The attack requires no external interaction, no race condition, and no special transaction — only a single modified `ProposalInit` field. The proposer can also set `builder` to the zero address or a burn address to deny fees to everyone, not just redirect them.

### Recommendation

1. Add `expected_builder: ContractAddress` to `ProposalInitValidation`, populated from `self.config.static_config.builder_address` at the call site in `validate_proposal` / `set_height_and_round`.
2. In `is_proposal_init_valid`, reject any `init_proposed.builder` that differs from `proposal_init_validation.expected_builder`, analogous to the existing `height` / `l1_da_mode` / `l2_gas_price_fri` check.
3. Add a test case to `validate_proposal_test.rs` that sets `init.builder` to a different address and asserts `InvalidProposalInit` is returned.

### Proof of Concept

1. Malicious validator node wins a proposer slot for block N.
2. In `initiate_build`, it overrides `builder: args.builder_address` with `builder: attacker_address` before constructing `ProposalInit`.
3. The modified `ProposalInit` is broadcast to all validators.
4. Each validator calls `is_proposal_init_valid` — no check on `builder`, so validation passes.
5. Each validator calls `initiate_validation` → `convert_to_sn_api_block_info(init)` → `sequencer_address = attacker_address`.
6. The batcher executes all transactions; every `execute_fee_transfer` sends fees to `attacker_address`.
7. `PartialBlockHashComponents::new` hashes `attacker_address` as `sequencer`; the resulting `partial_block_hash` matches the proposer's `ProposalFin.proposal_commitment`.
8. `built_block == received_fin.proposal_commitment` — the proposal is accepted.
9. Block N is committed with `sequencer_address = attacker_address`; all fees for that block are permanently credited to the attacker.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
