### Title
Missing `ppuWellFormed` Bounds Validation for Dijkstra-Era Reference Script Fee Parameters Allows Fee Undercharge via Governance - (`File: eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/PParams.hs`)

---

### Summary

The `ppuWellFormed` implementation for `DijkstraEra` does not validate the four new Dijkstra-specific protocol parameters (`maxRefScriptSizePerBlock`, `maxRefScriptSizePerTx`, `refScriptCostStride`, `refScriptCostMultiplier`) that directly govern the reference-script fee calculation and the per-transaction/per-block size caps. A governance actor can enact a `ParameterChange` proposal that sets `maxRefScriptSizePerTx = 0` (disabling the per-tx size cap entirely) or `refScriptCostMultiplier` to its type minimum (`positiveIntervalEpsilon ≈ 1e-19`), causing the tiered fee to collapse to zero lovelace for any reference-script payload. This lets an unprivileged transaction sender attach up to the block-level limit of reference scripts at effectively zero marginal cost, recreating the exact DDoS vector the tiered pricing was designed to prevent.

---

### Finding Description

In `DijkstraEra`, the `ConwayEraPParams` instance's `ppuWellFormed` function validates only the parameters inherited from Conway:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/PParams.hs:539-557
instance ConwayEraPParams DijkstraEra where
  ppuWellFormed _pv ppu =
    and
      [ isValid (/= 0) ppuMaxBBSizeL
      , isValid (/= 0) ppuMaxTxSizeL
      ...
      , isValid (/= 0) ppuNOptL
      ]
```

The four Dijkstra-specific parameters — `ppuMaxRefScriptSizePerBlockL`, `ppuMaxRefScriptSizePerTxL`, `ppuRefScriptCostStrideL`, `ppuRefScriptCostMultiplierL` — are **absent from this list**. [1](#0-0) 

By contrast, the Conway era's `ppuWellFormed` correctly validates its own security-critical parameters (e.g., `ppuPoolDepositL /= 0`, `ppuGovActionDepositL /= 0`), but the Dijkstra override does not extend this to the new parameters. [2](#0-1) 

The `ppuWellFormed` result is the sole gate used by `actionWellFormed` in the `GOV` rule to reject malformed `ParameterChange` proposals:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Gov.hs:393-399
actionWellFormed pv ga = failureUnless isWellFormed $ MalformedProposal ga
  where
    isWellFormed = case ga of
      ParameterChange _ ppd _ -> ppuWellFormed pv ppd
      _ -> True
``` [3](#0-2) 

The fee calculation in `getConwayMinFeeTx` (used for all Conway-and-later eras including Dijkstra) reads `ppRefScriptCostMultiplierG` and `ppRefScriptCostStrideG` directly from the protocol parameters at transaction validation time:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Tx.hs:103-112
getConwayMinFeeTx pp tx refScriptsSize =
  alonzoMinFeeTx pp tx <+> refScriptsFee
  where
    refScriptCostPerByte = unboundRational (pp ^. ppMinFeeRefScriptCostPerByteL)
    refScriptsFee =
      tierRefScriptFee
        (unboundRational $ pp ^. ppRefScriptCostMultiplierG)
        (fromIntegral @Word32 @Int . unNonZero $ pp ^. ppRefScriptCostStrideG)
        refScriptCostPerByte
        refScriptsSize
``` [4](#0-3) 

The per-tx size cap is enforced in `validateAllRefScriptSize` using `ppMaxRefScriptSizePerTxG`:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Ledger.hs:321-329
validateAllRefScriptSize pp utxo tx =
  let totalRefScriptSize = batchNonDistinctRefScriptsSize utxo tx
      maxRefScriptSizePerTx = fromIntegral @Word32 @Int $ pp ^. ppMaxRefScriptSizePerTxG
   in failureUnless (totalRefScriptSize <= maxRefScriptSizePerTx) $ ...
``` [5](#0-4) 

Setting `maxRefScriptSizePerTx = 0` via a governance update makes `totalRefScriptSize <= 0` always false for any non-empty reference script payload, **blocking all transactions that use reference scripts** — a permanent freeze of DApp functionality. Alternatively, setting `refScriptCostMultiplier` to `positiveIntervalEpsilon` (the type's minimum, `1/10^19`) causes `tierRefScriptFee` to return `Coin 0` for any realistic script size, eliminating the fee deterrent entirely. [6](#0-5) 

The `PositiveInterval` type's minimum bound is `positiveIntervalEpsilon = 1 % 10^19`, which is positive and therefore passes the type-level constraint, but is economically equivalent to zero for any script size below `10^19` bytes. [7](#0-6) 

---

### Impact Explanation

**Medium — attacker-controlled governance proposal modifies fees and size limits outside design parameters.**

- Setting `refScriptCostMultiplier` to `positiveIntervalEpsilon` collapses the tiered reference-script fee to zero lovelace for all practical script sizes, allowing an attacker to submit transactions with large reference scripts at near-zero marginal cost. This directly recreates the DDoS vector documented in ADR-009 and the June 2024 mainnet attack.
- Setting `maxRefScriptSizePerTx = 0` permanently freezes all transactions that use reference scripts (since `totalRefScriptSize > 0` always fails the check), which constitutes permanent freezing of DApp functionality requiring a hard fork to recover.

Both outcomes fall within the allowed impact scope: the first is "attacker-controlled transactions exceed intended validation limits or modify fees outside design parameters"; the second is "permanent freezing of funds/withdrawals where recovery requires a hard fork."

---

### Likelihood Explanation

Requires a successful on-chain governance `ParameterChange` action in Dijkstra era, which needs DRep/SPO/CC ratification. This is not trivially achievable by a single unprivileged actor, but the governance threshold is a known, finite barrier — not a trusted-role or key-leak requirement. The absence of the `ppuWellFormed` check means the proposal is not rejected at submission time, so a sufficiently motivated coalition (or a governance participant who misunderstands the parameter semantics) can enact it. The analogous Conway-era parameters (`poolDeposit`, `govActionDeposit`) are protected by `ppuWellFormed` precisely because their misuse has protocol-level consequences; the omission here is a direct parallel to the original report's "inadequate validation" class.

---

### Recommendation

Extend `ppuWellFormed` in the `ConwayEraPParams DijkstraEra` instance to validate the four new parameters:

```haskell
-- Add to the `and [...]` list in ppuWellFormed:
, isValid (/= 0) ppuMaxRefScriptSizePerBlockL
, isValid (/= 0) ppuMaxRefScriptSizePerTxL
-- refScriptCostStride is NonZero Word32, already type-safe
-- refScriptCostMultiplier: enforce a meaningful minimum, e.g. >= 1
, isValid (>= fromJust (boundRational 1.0)) ppuRefScriptCostMultiplierL
```

The `refScriptCostMultiplier` should be validated to be `>= 1` (i.e., non-decreasing tiers), since a value below 1 causes the fee to *decrease* with each tier, inverting the exponential deterrent. The `maxRefScriptSizePerTx` and `maxRefScriptSizePerBlock` should be validated as non-zero to prevent the size-cap freeze.

---

### Proof of Concept

1. In Dijkstra era, a governance participant submits a `ParameterChange` proposal with:
   ```
   ppuRefScriptCostMultiplierL = SJust positiveIntervalEpsilon  -- ≈ 1e-19
   ```
2. `actionWellFormed` calls `ppuWellFormed _pv ppu`. The Dijkstra instance checks only the inherited Conway parameters; `ppuRefScriptCostMultiplierL` is not checked. `ppuWellFormed` returns `True`. The proposal is accepted on-chain.
3. After ratification and enactment, `pp ^. ppRefScriptCostMultiplierG` returns `positiveIntervalEpsilon`.
4. Any transaction with reference scripts calls `getConwayMinFeeTx`, which calls `tierRefScriptFee (1e-19) stride baseFee size`. For any `size < 204800` bytes, the result is `Coin (floor (size * baseFee * 1e-19)) = Coin 0`.
5. The attacker submits transactions with up to 200 KiB of reference scripts paying zero reference-script fee, saturating block validation capacity at minimal cost — the exact DDoS scenario from June 2024. [8](#0-7) [9](#0-8) [10](#0-9)

```haskell
tierRefScriptFee ::
  HasCallStack =>
  -- | Growth factor or step multiplier
  Rational ->
  -- | Increment size in which price grows linearly according to the price
  Int ->
  -- | Base fee. Currently this is customizable by `ppMinFeeRefScriptCostPerByteL`
  Rational ->
  -- | Total RefScript size in bytes
  Int ->
  Coin
tierRefScriptFee multiplier sizeIncrement
  | multiplier <= 0 || sizeIncrement <= 0 = error "Size increment and multiplier must be positive"
  | otherwise = go 0
  where
    go !acc !curTierPrice !n
      | n < sizeIncrement =
          Coin $ floor (acc + toRational n * curTierPrice)
      | otherwise =
          go (acc + sizeIncrementRational * curTierPrice) (multiplier * curTierPrice) (n - sizeIncrement)
    sizeIncrementRational = toRational sizeIncrement
```
