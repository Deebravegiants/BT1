### Title
Unvalidated User-Supplied Token Address in `init_transfer` Enables Unbacked Cross-Chain Minting — (File: `starknet/src/omni_bridge.cairo`)

### Summary
`init_transfer` on the Starknet bridge (and `initTransfer` on the EVM bridge) accepts any caller-supplied `token_address` without validating it against a whitelist or factory registry. For non-bridge tokens the function only asserts the boolean return value of `transfer_from`. An attacker can deploy a "poisoned" ERC-20 that returns `true` from `transfer_from` without moving any tokens, call the permissionless `log_metadata` to register it on NEAR, and then call `init_transfer` to emit a fully-formed `InitTransfer` event with an arbitrary amount — causing NEAR to mint unbacked bridge tokens.

### Finding Description

**Step 1 — permissionless token registration.**
`log_metadata` on Starknet has no access control:

```cairo
fn log_metadata(ref self: ContractState, token: ContractAddress) {
    // no role check
    ...
    self.emit(Event::LogMetadata(LogMetadata { address: token, name, symbol, decimals }))
}
``` [1](#0-0) 

Any caller can pass an arbitrary `token` address. The bridge calls `name()`, `symbol()`, `decimals()` on it and emits `LogMetadata`. A trusted relayer then submits the proof to NEAR, which registers the token in `token_decimals` and `token_id_to_address`.

**Step 2 — unvalidated token address in `init_transfer`.**
`init_transfer` on Starknet performs no check that `token_address` is a known/registered token:

```cairo
if self.is_bridge_token(token_address) {
    IBridgeTokenDispatcher { contract_address: token_address }
        .burn(caller, amount.into());
} else {
    let success = IERC20Dispatcher { contract_address: token_address }
        .transfer_from(caller, get_contract_address(), amount.into());
    assert(success, 'ERR_TRANSFER_FROM_FAILED');
}
``` [2](#0-1) 

`is_bridge_token` only returns `true` for tokens deployed by the bridge itself:

```cairo
fn is_bridge_token(self: @ContractState, token_address: ContractAddress) -> bool {
    self.starknet_to_near_token.read(token_address).len() > 0
}
``` [3](#0-2) 

A poisoned ERC-20 deployed by the attacker is not in `starknet_to_near_token`, so execution falls into the `else` branch. The only guard is `assert(success, ...)`. A contract that returns `true` from `transfer_from` without moving tokens passes this assertion. The `InitTransfer` event is then emitted with the attacker-chosen `amount`.

**Step 3 — NEAR mints without backing.**
`fin_transfer_callback` on NEAR verifies only that the emitter is the known Starknet factory and that the token is in `token_decimals`:

```rust
require!(
    self.factories.get(&init_transfer.emitter_address.get_chain())
        == Some(init_transfer.emitter_address),
    BridgeError::UnknownFactory.as_ref()
);
let decimals = self
    .token_decimals
    .get(&init_transfer.token)
    .near_expect(BridgeError::TokenDecimalsNotFound);
``` [4](#0-3) 

Both conditions are satisfied: the event was genuinely emitted by the Starknet bridge (known factory), and the token was registered in step 1. NEAR proceeds to mint tokens for the attacker with no real assets locked on Starknet.

**Same root cause exists on EVM.**
`initTransfer` in `OmniBridge.sol` has the identical pattern for native tokens:

```solidity
} else {
    IERC20(tokenAddress).safeTransferFrom(
        msg.sender,
        address(this),
        amount
    );
}
``` [5](#0-4) 

`SafeERC20.safeTransferFrom` checks the return value but not the actual balance delta. The permissionless `logMetadata` on EVM enables the same registration step. [6](#0-5) 

### Impact Explanation
An attacker can mint an arbitrary quantity of any bridge-wrapped token on NEAR without locking any real assets on Starknet or EVM. This breaks the 1:1 backing guarantee of the bridge, inflates the wrapped token supply, and allows the attacker to redeem the unbacked tokens for real assets held by the bridge on other chains — a direct, irreversible theft of protocol funds.

### Likelihood Explanation
The entry points (`log_metadata`, `init_transfer`) are fully public with no role restriction. Deploying a poisoned ERC-20 on Starknet or EVM is trivial. Trusted relayers process all events that pass proof verification; they have no mechanism to detect that a token's `transfer_from` is a no-op. The attack requires no privileged access, no leaked keys, and no collusion.

### Recommendation
1. **Whitelist or factory-gate `init_transfer`**: reject `token_address` values that are not registered in the bridge's token registry (`starknet_to_near_token` / `nearToEthToken`) before attempting any token interaction.
2. **Balance-delta check**: after calling `transfer_from`, verify that `balanceOf(bridge)` increased by at least `amount`.
3. **Gate `log_metadata`**: require that the token address is already known to the bridge (e.g., deployed by a trusted factory or pre-approved by governance) before emitting a `LogMetadata` event that triggers automatic NEAR-side registration.

### Proof of Concept

```
// 1. Deploy PoisonToken on Starknet:
//    transfer_from(from, to, amount) → returns true, transfers nothing
//    name() → "Legit Token", symbol() → "LGT", decimals() → 18

// 2. Register on NEAR (permissionless):
starknet_bridge.log_metadata(poison_token_address);
// Relayer submits LogMetadata proof to NEAR → token registered in token_decimals

// 3. Mint unbacked tokens:
starknet_bridge.init_transfer(
    token_address = poison_token_address,
    amount        = 1_000_000 * 10^18,   // arbitrary large amount
    fee           = 0,
    native_fee    = 0,
    recipient     = "attacker.near",
    message       = ""
);
// PoisonToken.transfer_from returns true, no tokens move
// InitTransfer event emitted with amount = 1_000_000e18
// Relayer submits InitTransfer proof to NEAR
// NEAR mints 1_000_000 wrapped tokens to attacker.near with zero backing
```

        fn log_metadata(ref self: ContractState, token: ContractAddress) {
            // There are two possible metadata standards in use.
            // 1. Old style: name and symbol are felt252 values.
            // 2. New style: name and symbol are ByteArray values (ERC20 ABI).
            // We are using low-level contract calls to determine the type.

            let call_data: Array<felt252> = array![];
            let mut res = syscalls::call_contract_syscall(
                token, selector!("name"), call_data.span(),
            )
                .unwrap_syscall();

            let name = if res.len() == 1 {
                // Old standard (felt252)
                let name = OptionTrait::expect(
                    Serde::<felt252>::deserialize(ref res), 'Could not deserialize name',
                );
                utils::felt252_to_string(name)
            } else {
                // New standard (ByteArray)
                OptionTrait::expect(
                    Serde::<ByteArray>::deserialize(ref res), 'Could not deserialize name',
                )
            };

            let mut res = syscalls::call_contract_syscall(
                token, selector!("symbol"), call_data.span(),
            )
                .unwrap_syscall();

            let symbol = if res.len() == 1 {
                // Old standard (felt252)
                let symbol = OptionTrait::expect(
                    Serde::<felt252>::deserialize(ref res), 'Could not deserialize symbol',
                );
                utils::felt252_to_string(symbol)
            } else {
                // New standard (ByteArray)
                OptionTrait::expect(
                    Serde::<ByteArray>::deserialize(ref res), 'Could not deserialize symbol',
                )
            };

            let decimals = {
                let mut res = syscalls::call_contract_syscall(
                    token, selector!("decimals"), call_data.span(),
                )
                    .unwrap_syscall();

                let decimals = OptionTrait::expect(
                    Serde::<u8>::deserialize(ref res), 'Could not deserialize decimals',
                );
                decimals
            };

            self.emit(Event::LogMetadata(LogMetadata { address: token, name, symbol, decimals }))
