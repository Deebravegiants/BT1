### Title
`MockAttestation::Valid` Bypasses TEE Attestation Verification in Production — (`crates/contract/src/tee/tee_state.rs`, `crates/contract/src/lib.rs`)

---

### Summary

The `submit_participant_info` endpoint accepts `Attestation::Mock(MockAttestation::Valid)` in production with no feature-flag or runtime guard. `MockAttestation::Valid` unconditionally passes all verification checks — no TEE quote, no image-hash check, no measurement check, no expiry — allowing any existing participant to register as "TEE-attested" without running inside a genuine Trusted Execution Environment.

---

### Finding Description

The `Attestation` enum exposed in the public contract ABI contains a `Mock(MockAttestation)` variant alongside the real `Dstack` variant. [1](#0-0) 

`MockAttestation::Valid` always returns `Ok(())` from `verify_constraints`, bypassing every security gate: [2](#0-1) 

The DTO-to-internal mapping in `dto_mapping.rs` converts `dtos::MockAttestation::Valid` → `MockAttestation::Valid` with no production guard: [3](#0-2) 

`submit_participant_info` then calls `tee_state.add_participant(...)` with the converted attestation. Because `MockAttestation::Valid` always passes `attestation.verify(...)`, the entry is stored in `stored_attestations` as a fully valid, never-expiring attestation: [4](#0-3) 

Once stored, `is_caller_an_attested_participant` will accept this participant for every attestation-gated method: [5](#0-4) 

This gates the following critical operations:

- `vote_pk` / `vote_reshared` / `start_keygen_instance` / `start_reshare_instance` — key-generation and resharing governance
- `respond` / `respond_ckd` / `respond_verify_foreign_tx` — threshold-signature submission [6](#0-5) 

Additionally, because `MockAttestation::Valid` never expires, the participant is immune to the periodic `verify_tee` eviction mechanism that removes participants with stale attestations.

---

### Impact Explanation

A single malicious or compromised participant can submit `Attestation::Mock(MockAttestation::Valid)` instead of a genuine Dstack quote. Their node is then treated as TEE-verified by the contract for all subsequent operations. Key shares distributed to this node during DKG or resharing are held outside any TEE boundary, violating the foundational security invariant that key material is only ever accessible inside attested hardware. The mock attestation also never expires, so the standard eviction path (`verify_tee` → resharing) cannot remove the participant.

This breaks the production safety invariant — all active participants must be running inside genuine TEEs — without requiring any network-level DoS or operator misconfiguration.

---

### Likelihood Explanation

Any existing participant can exploit this unilaterally. No collusion with other participants is required: the attacker simply calls `submit_participant_info` with `Attestation::Mock(MockAttestation::Valid)` instead of their real Dstack quote. The contract's public ABI exposes the `Mock` variant, and the sandbox end-to-end tests confirm it is accepted on a live contract: [7](#0-6) 

---

### Recommendation

1. **Reject `Mock` attestations in production at the contract boundary.** In `submit_participant_info`, check the attestation variant before calling `tee_state.add_participant` and return an error if the variant is `Mock`.
2. **Gate the `Mock` variant behind a compile-time feature flag** (e.g., `#[cfg(feature = "mock-attestation")]`) so it is physically absent from production WASM builds.
3. **Remove `MockAttestation` from the public ABI** exposed to external callers; keep it only in test-only crate features.

---

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
