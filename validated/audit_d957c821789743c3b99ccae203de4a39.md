### Title
Missing Protocol Feature Gate for `DeployGlobalContract`, `UseGlobalContract`, and `DeterministicStateInit` Actions Allows Pre-Feature Execution via Contract-Emitted Receipts - (File: runtime/runtime/src/action_validation.rs)

### Summary
`validate_action_with_mode` lacks protocol-feature guards for `Action::DeployGlobalContract`, `Action::UseGlobalContract`, and `Action::DeterministicStateInit`. Newer action types introduced in the same codebase (`DelegateV2`, `TransferToGasKey`, `WithdrawFromGasKey`) are all gated with `require_protocol_feature`. The three global-contract/deterministic-account actions are not. Because receipt-level validation is the last enforcement point for actions emitted by contracts via host functions (those receipts never pass tx-admission), an unprivileged user can deploy a contract that calls the corresponding host functions on a pre-feature protocol version, causing those actions to be executed and committed to state before the protocol version that authorises them is active.

### Finding Description
`validate_action_with_mode` dispatches per-action validation:

```rust
Action::DeployGlobalContract(a) => validate_deploy_global_contract_action(limit_config, a),
Action::UseGlobalContract(a)    => validate_use_global_contract_action(a),
Action::DeterministicStateInit(a) => validate_deterministic_state_init(limit_config, a, receiver)
```

None of these arms call `require_protocol_feature`. [1](#0-0) 

By contrast, every other recently-added action type is explicitly gated:

```rust
Action::DelegateV2(a) => {
    require_protocol_feature(ProtocolFeature::DelegateV2, "DelegateV2", current_protocol_version)?;
    ...
}
Action::TransferToGasKey(_) => validate_transfer_to_gas_key_action(current_protocol_version)
// which calls require_protocol_feature(ProtocolFeature::GasKeys, ...)
``` [2](#0-1) 

The code's own comment explains why this gate is load-bearing:

> "Centralized post-quantum gate … is load-bearing for actions emitted by contracts via host functions: those actions create new receipts that never go through tx admission, so on a pre-feature protocol they must be rejected here." [3](#0-2) 

The host-function implementations for the three ungated actions also carry no protocol-version check: [4](#0-3) [5](#0-4) 

`validate_deploy_global_contract_action` and `validate_use_global_contract_action` only check size/account-id format, never the protocol version: [6](#0-5) 

`validate_deterministic_state_init` only checks the derived receiver id and key/value lengths: [7](#0-6) 

### Impact Explanation
During the protocol-upgrade window — after validators have deployed new node software but before the epoch boundary at which the protocol version is bumped — an unprivileged user can deploy a WASM contract that calls `promise_batch_action_deploy_global_contract`, `promise_batch_action_use_global_contract`, or the state-init host function. The resulting action receipts pass `validate_receipt` → `validate_action_receipt` → `validate_actions_with_mode` → `validate_action_with_mode` without any protocol-version rejection. The runtime then executes `action_deploy_global_contract` / `action_use_global_contract` / `action_deterministic_state_init`, writing global-contract trie keys and modifying account state under a protocol version that is not supposed to support those operations. The corrupted state root is committed to `DBCol::ChunkExtra` and propagated to all validators, making the corruption permanent and consensus-wide. [8](#0-7) [9](#0-8) 

### Likelihood Explanation
The attack window is the upgrade period between node-software rollout and the epoch-boundary protocol-version bump. This window is predictable and publicly observable on-chain. An attacker only needs to: (1) deploy a contract that calls the ungated host functions, and (2) trigger a call to that contract before the epoch boundary. No privileged access is required. The host functions are unconditionally registered in the WASM runtime once the node binary is upgraded, so any deployed contract can invoke them.

### Recommendation
Add `require_protocol_feature` calls for the three ungated action arms, mirroring the pattern used for `DelegateV2` and `GasKeys`:

```rust
Action::DeployGlobalContract(a) => {
    require_protocol_feature(ProtocolFeature::GlobalContracts, "GlobalContracts", current_protocol_version)?;
    validate_deploy_global_contract_action(limit_config, a)
}
Action::UseGlobalContract(a) => {
    require_protocol_feature(ProtocolFeature::GlobalContracts, "GlobalContracts", current_protocol_version)?;
    validate_use_global_contract_action(a)
}
Action::DeterministicStateInit(a) => {
    require_protocol_feature(ProtocolFeature::DeterministicStateInit, "DeterministicStateInit", current_protocol_version)?;
    validate_deterministic_state_init(limit_config, a, receiver)
}
```

The same guard should be added inside the host-function implementations so that contract-level calls are rejected at gas-charging time rather than only at receipt-validation time.

### Proof of Concept
1. Observe that the network is in the upgrade window (new node binary deployed, protocol version not yet bumped).
2. Deploy a WASM contract whose `call` method invokes `promise_batch_action_deploy_global_contract` targeting itself.
3. Submit a `FunctionCall` transaction to that contract.
4. The runtime executes the function call, which appends a `DeployGlobalContract` action receipt.
5. `validate_receipt` → `validate_action_with_mode` reaches `Action::DeployGlobalContract(a) => validate_deploy_global_contract_action(limit_config, a)` — no protocol-version check fires.
6. `action_deploy_global_contract` writes a `TrieKey::GlobalContractCode` entry into the state trie under the pre-feature protocol version.
7. The resulting state root, committed to `DBCol::ChunkExtra`, reflects global-contract state that the protocol version does not authorise, corrupting the canonical chain state. [10](#0-9) [11](#0-10)

fn validate_deterministic_state_init(
    limit_config: &LimitConfig,
    action: &DeterministicStateInitAction,
    receiver_id: &AccountId,
) -> Result<(), ActionsValidationError> {
    validate_global_contract_identifier(action.state_init.code())?;

    let derived_id = derive_near_deterministic_account_id(&action.state_init);

    if derived_id != *receiver_id {
        return Err(ActionsValidationError::InvalidDeterministicStateInitReceiver {
            derived_id,
            receiver_id: receiver_id.clone(),
        });
    }

    // State init entries must not violate limits of individual state keys and values.
    for (key, value) in action.state_init.data() {
        if key.len() as u64 > limit_config.max_length_storage_key {
            return Err(ActionsValidationError::DeterministicStateInitKeyLengthExceeded {
                length: key.len() as u64,
                limit: limit_config.max_length_storage_key,
            }
            .into());
        }

        if value.len() as u64 > limit_config.max_length_storage_value {
            return Err(ActionsValidationError::DeterministicStateInitValueLengthExceeded {
                length: value.len() as u64,
                limit: limit_config.max_length_storage_value,
            }
            .into());
        }
    }

    Ok(())
}

fn promise_batch_action_deploy_global_contract_impl(
    ctx: &mut Ctx,
    memory: &mut [u8],
    promise_idx: u64,
    code_len: u64,
    code_ptr: u64,
    mode: GlobalContractDeployMode,
    method_name: &str,
) -> Result<()> {
    ctx.result_state.gas_counter.pay_base(base)?;
    if ctx.context.is_view() {
        return Err(HostError::ProhibitedInView { method_name: method_name.to_owned() }.into());
    }
    let code = get_memory_or_register(
        &mut ctx.result_state.gas_counter,
        memory,
        &ctx.registers,
        code_ptr,
        code_len,
    )?;
    let code_len = code.len() as u64;
    let limit = ctx.config.limit_config.max_contract_size;
    if code_len > limit {
        return Err(HostError::ContractSizeExceeded { size: code_len, limit }.into());
    }
    let code = code.into();

    let (receipt_idx, sir) = promise_idx_to_receipt_idx_with_sir(ctx, promise_idx)?;

    pay_action_base(
        &mut ctx.result_state.gas_counter,
        &ctx.fees_config,
        ActionCosts::deploy_global_contract_base,
        sir,
    )?;
    pay_action_per_byte(
        &mut ctx.result_state.gas_counter,
        &ctx.fees_config,
        ActionCosts::deploy_global_contract_byte,
        code_len,
        sir,
    )?;

    ctx.ext.append_action_deploy_global_contract(receipt_idx, code, mode);
    Ok(())

pub(crate) fn use_global_contract(
    state_update: &mut TrieUpdate,
    account_id: &AccountId,
    account: &mut Account,
    contract_identifier: &GlobalContractIdentifier,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let key = TrieKey::GlobalContractCode { identifier: contract_identifier.clone().into() };
    if !state_update.contains_key(&key, AccessOptions::DEFAULT)? {
        result.result = Err(ActionErrorKind::GlobalContractDoesNotExist {
            identifier: contract_identifier.clone(),
        }
        .into());
        return Ok(());
    }
    clear_account_contract_storage_usage(state_update, account_id, account)?;
    if account.contract().is_local() {
        state_update.remove(TrieKey::ContractCode { account_id: account_id.clone() });
    }
    let contract = match contract_identifier {
        GlobalContractIdentifier::CodeHash(code_hash) => AccountContract::Global(*code_hash),
        GlobalContractIdentifier::AccountId(id) => AccountContract::GlobalByAccount(id.clone()),
    };
    account.set_storage_usage(
        account.storage_usage().checked_add(contract_identifier.len() as u64).ok_or_else(|| {
            StorageError::StorageInconsistentState(format!(
                "Storage usage integer overflow for account {}",
                account_id
            ))
        })?,
    );
    account.set_contract(contract);
    Ok(())
}

pub(crate) fn action_deterministic_state_init(
    state_update: &mut TrieUpdate,
    apply_state: &ApplyState,
    maybe_account: &mut Option<Account>,
    account_id: &AccountId,
    receipt: &Receipt,
    action: &DeterministicStateInitAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    // See https://github.com/near/NEPs/blob/master/neps/nep-0616.md#account-state
    // for the detailed description around deterministic account state.
    let storage_usage_config = &apply_state.config.fees.storage_usage_config;
    let account = match maybe_account {
        Some(account) => account,
        None => {
            // cspell:ignore nonexist
            // `nonexist` -> `uninit` account state transition
            // Create with zero balance now and check later how much of the
            // provided deposit is needed.
            let new_account = create_deterministic_account(Balance::ZERO, storage_usage_config);
            maybe_account.insert(new_account)
        }
    };
    if account.contract().is_none() {
        // `uninit` -> `active` account state transition
        deploy_deterministic_account(
            state_update,
            account,
            account_id,
            &action.state_init,
            result,
            storage_usage_config,
        )?;
    }
    if result.result.is_err() {
        return Ok(());
    }

    // Use attached deposit to satisfy storage staking requirements and refund
    // the rest.
    let deposit_refund = match check_storage_stake(account, account.amount(), &apply_state.config) {
        Ok(_) => {
            // no additional storage needed, refunding all
            action.deposit
        }
        Err(StorageStakingError::LackBalanceForStorageStaking(missing_amount)) => {
            if missing_amount <= action.deposit {
                // use exactly as much as needed and refund the rest
                let new_balance = safe_add_balance(account.amount(), missing_amount)?;
                account.set_amount(new_balance);
                action
                    .deposit
                    .checked_sub(missing_amount)
                    .expect("just checked missing_amount <= action.deposit")
            } else {
                result.result = Err(ActionErrorKind::LackBalanceForState {
                    account_id: account_id.clone(),
                    amount: missing_amount,
                }
                .into());
                return Ok(());
            }
        }
        Err(StorageStakingError::StorageError(err)) => {
            return Err(RuntimeError::StorageError(StorageError::StorageInconsistentState(err)));
        }
    };

    if deposit_refund > Balance::ZERO {
        result
            .new_receipts
            .push(Receipt::new_balance_refund(receipt.balance_refund_receiver(), deposit_refund));
    }

    Ok(())
