### Title
Protocol fee is assessed on yield that accrued during a pause period upon first unpause price update — (File: contracts/LRTOracle.sol)

### Summary

`LRTOracle._updateRsETHPrice()` contains a `protocolPaused` guard that skips fee minting while the protocol is paused. However, because `updateRSETHPrice()` is blocked by `whenNotPaused` during a pause, the stored `rsETHPrice` is never updated during the pause window. On the first call after unpause, the fee is computed against the stale pre-pause price, so the entire TVL increase — including yield that accrued while users were locked out — is subject to the protocol fee.

### Finding Description

`LRTOracle._updateRsETHPrice()` computes the protocol fee as follows:

```solidity
// contracts/LRTOracle.sol lines 240-247
bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

uint256 protocolFeeInETH = 0;
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
``` [1](#0-0) 

`previousTVL` is derived from the stored `rsETHPrice`:

```solidity
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);
``` [2](#0-1) 

The public entry point is gated:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [3](#0-2) 

While the manager bypass exists:

```solidity
function updateRSETHPriceAsManager() external onlyLRTManager {
    _updateRsETHPrice();
}
``` [4](#0-3) 

The oracle also has an auto-pause path that returns without updating `rsETHPrice`:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;   // rsETHPrice is NOT updated here
}
``` [5](#0-4) 

**Sequence of events that triggers the bug:**

1. Protocol is running. `rsETHPrice = P₀`. `updateRSETHPrice()` is called regularly.
2. Protocol is paused (PAUSER\_ROLE or auto-pause on price drop). `updateRSETHPrice()` is blocked. `rsETHPrice` remains `P₀`.
3. During the pause, underlying LSTs (stETH, ETHx) continue accruing staking rewards. TVL grows from `TVL₀` to `TVL₁`.
4. Protocol unpauses. `updateRSETHPrice()` is called.
5. `previousTVL = rsethSupply × P₀ = TVL₀`.
6. `totalETHInProtocol = TVL₁`.
7. `protocolPaused = false` (protocol is now unpaused).
8. `rewardAmount = TVL₁ − TVL₀` — **this includes all yield that accrued during the pause**.
9. `protocolFeeInETH = rewardAmount × protocolFeeInBPS / 10_000` — fee is taken on the full amount.

The `protocolPaused` guard only prevents fee minting during a call made *while* the protocol is paused. It does not prevent fee minting on yield that accumulated *during* a past pause window. The only way to avoid this is for a manager to proactively call `updateRSETHPriceAsManager()` during the pause (which updates `rsETHPrice` without taking a fee), but this is not enforced by the protocol.

### Impact Explanation

rsETH holders lose a portion of their staking yield — specifically the protocol fee percentage applied to all rewards that accrued during the pause — for a period when they were unable to withdraw or otherwise exit their position. The `protocolPaused` check was clearly designed to prevent fee extraction during a pause, but the implementation leaves a gap: the fee is still effectively charged on paused-period yield, just deferred to the first post-unpause price update. This constitutes theft of unclaimed yield.

### Likelihood Explanation

The oracle contains an automatic pause trigger (`isPriceDecreaseOffLimit`) that fires without any manual intervention whenever the rsETH price drops beyond the configured threshold. After such an auto-pause, the protocol must be investigated and manually unpaused by an admin. During that investigation window — which could span hours or days — staking rewards continue to accrue. There is no on-chain enforcement requiring a manager to call `updateRSETHPriceAsManager()` before unpausing. This scenario is realistic and repeatable.

### Recommendation

On unpause (or as part of the unpause transaction), call `_updateRsETHPrice()` internally with fee minting suppressed, so that `rsETHPrice` is brought up to the current TVL before the public `updateRSETHPrice()` is re-enabled. Alternatively, record the TVL at the moment of pausing and use it as the fee baseline on the first post-unpause update, rather than the pre-pause stored price.

### Proof of Concept

```
State before pause:
  rsETHPrice = 1.05 ETH/rsETH
  rsethSupply = 1,000,000 rsETH
  previousTVL = 1,050,000 ETH

Protocol paused for 7 days (auto-pause on price drop).
  updateRSETHPrice() blocked — rsETHPrice stays at 1.05.
  Staking rewards accrue: +700 ETH (≈4% APY / 52 weeks).

Protocol unpauses:
  totalETHInProtocol = 1,050,700 ETH
  protocolPaused = false
  rewardAmount = 700 ETH  ← includes 7 days of paused-period yield
  protocolFeeInETH = 700 × 1000 / 10000 = 70 ETH  (assuming 10% fee)

Users receive 630 ETH worth of yield instead of 700 ETH.
The 70 ETH fee is minted as rsETH to the treasury,
diluting all rsETH holders for yield they earned while locked out.
``` [6](#0-5)

    function _updateRsETHPrice() internal {
        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;
        }

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

        // only take fee if TVL increased and protocol is not paused
        uint256 protocolFeeInETH = 0;
        if (!protocolPaused && totalETHInProtocol > previousTVL) {
            uint256 rewardAmount = totalETHInProtocol - previousTVL;
            protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
        }

        // compute new rsETH price based on total ETH minus fee
        uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

        if (newRsETHPrice > highestRsethPrice) {
            // check if the price is above the threshold
            uint256 priceDifference = newRsETHPrice - highestRsethPrice;
            // pricePercentageLimit is in 1e18 precision (100% = 1e18, 1% = 1e16)
            bool isPriceIncreaseOffLimit =
                pricePercentageLimit > 0 && priceDifference > pricePercentageLimit.mulWad(highestRsethPrice);

            // check if the price difference is above the threshold
            if (isPriceIncreaseOffLimit) {
                // if sender has a manager role, this doesn't revert.
                // if not, it reverts as price went above the threshold
                if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) {
                    revert PriceAboveDailyThreshold();
                }
            }
        }

        // downside protection — pause if price drops too far
        if (newRsETHPrice < highestRsethPrice) {
            uint256 diff = highestRsethPrice - newRsETHPrice;
            // normalizing to 1e18
            bool isPriceDecreaseOffLimit =
                pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);

            // if price decrease is off limit, pause the protocol (unless it's already paused)
            if (isPriceDecreaseOffLimit) {
                if (!lrtDepositPool.paused()) lrtDepositPool.pause();
                if (!withdrawalManager.paused()) withdrawalManager.pause();
                _pause();
                return;
            }

            // if price has decreased compared to the previous price, emit an event to reflect that
            if (previousPrice > newRsETHPrice) {
                emit RsETHPriceDecrease(newRsETHPrice, previousPrice);
            }

            // emit an event to notify that the price is currently below the peak (all time high) price
            emit RsETHPriceBelowPeak(highestRsethPrice, newRsETHPrice);
        }

        // update highest price if new price exceeds it
        if (newRsETHPrice > highestRsethPrice) {
            highestRsethPrice = newRsETHPrice;
        }

        // mint protocol fee as rsETH if there's a fee to take
        if (protocolFeeInETH > 0) {
            // Calculate rsETH amount to mint as protocol fee
            uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);

            _checkAndUpdateDailyFeeMintLimit(rsethAmountToMintAsProtocolFee);
            if (rsethAmountToMintAsProtocolFee > 0) {
                address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
                IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
                emit FeeMinted(treasury, rsethAmountToMintAsProtocolFee);
            }
        } else {
            _checkAndUpdateDailyFeeMintLimit(0);
        }

        rsETHPrice = newRsETHPrice;

        emit RsETHPriceUpdate(rsETHPrice, previousPrice);
    }
