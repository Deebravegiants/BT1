Audit Report

## Title
Missing Caller Identity Check Allows Forced Fee Deduction on Any User's Claim - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary

`MerkleDistributor.claim()` accepts an arbitrary `account` parameter but never verifies that `msg.sender == account`. Any unprivileged caller can trigger a claim for any user, causing a fee of up to 10% to be permanently deducted from the victim's entitled token allocation and sent to `protocolTreasury`. The victim's cumulative claim record is updated to the full amount, making the fee portion irrecoverable.

## Finding Description

`MerkleDistributor.claim()` is a public, permissionless function: [1](#0-0) 

There is no check that `msg.sender == account` anywhere in the function body. The fee is computed and deducted unconditionally: [2](#0-1) 

The victim's state is updated to the full `cumulativeAmount` before the fee-reduced transfer: [3](#0-2) 

This means the victim can never re-claim the fee portion — their `userClaims[account].cumulativeAmount` already reflects the full entitlement, so any future call would compute `claimableAmount = 0` and revert with `NoTokensToClaim`.

All parameters required to construct a valid call (`index`, `account`, `cumulativeAmount`, `merkleProof`) are either derivable from the publicly distributed merkle tree or observable from `Claimed` events emitted for other users.

The sister contract `KernelMerkleDistributor._processClaim()` explicitly guards against this exact pattern: [4](#0-3) 

The absence of this guard in `MerkleDistributor` is a clear omission, not a design choice.

## Impact Explanation

**High — Theft of unclaimed yield.**

`feeInBPS` is configurable up to `MAX_FEE_IN_BPS = 1000` (10%): [5](#0-4) 

When an attacker forces a claim for a victim, the fee is permanently transferred to `protocolTreasury` and the victim's claim record is finalized. The victim receives `claimableAmount - fee` instead of `claimableAmount`. The lost fee is irrecoverable. A user who intended to wait for the owner to lower `feeInBPS` via `setFeeInBPS()` is permanently deprived of the fee portion of their yield. This matches the allowed impact: **High. Theft of unclaimed yield.**

## Likelihood Explanation

**High.** The attack requires no special privileges. All parameters needed to construct a valid call are publicly available from the merkle tree distribution or on-chain `Claimed` events. The attacker pays only gas. The attack is repeatable for every user in the distribution and can be executed immediately after a new merkle root is set via `setMerkleRoot()`. [6](#0-5) 

## Recommendation

Add a caller identity check at the top of `claim()`, mirroring the pattern already used in `KernelMerkleDistributor._processClaim()`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Also add the `Unauthorized` error to `IMerkleDistributor` to match the interface in `KernelMerkleDistributor`.

## Proof of Concept

1. Deploy `MerkleDistributor` with `feeInBPS = 1000` (10%).
2. Owner calls `setMerkleRoot()` with a root covering Alice at `(index=1, account=Alice, cumulativeAmount=1000e18)`.
3. Attacker obtains Alice's `(index, account, cumulativeAmount, merkleProof)` from the public merkle tree.
4. Attacker calls `MerkleDistributor.claim(1, Alice, 1000e18, merkleProof)` — no special role required.
5. Contract computes `fee = 100e18`, transfers `900e18` to Alice and `100e18` to `protocolTreasury`.
6. `userClaims[Alice].cumulativeAmount` is set to `1000e18` — Alice can never reclaim the `100e18` fee.
7. Alice permanently loses 10% of her entitled token allocation without her consent.

**Foundry test sketch:**
```solidity
function test_forcedClaimFeeDeduction() public {
    // Setup: deploy distributor with feeInBPS=1000, set merkle root for Alice
    // Attacker (not Alice) calls claim(index, alice, amount, proof)
    vm.prank(attacker);
    distributor.claim(index, alice, cumulativeAmount, proof);
    // Assert: alice received only 90% of entitlement
    assertEq(token.balanceOf(alice), cumulativeAmount * 9000 / 10_000);
    // Assert: alice cannot claim again
    vm.expectRevert(IMerkleDistributor.AlreadyClaimed.selector);
    vm.prank(alice);
    distributor.claim(index, alice, cumulativeAmount, proof);
}
```
