### Title
Dijkstra Era Fee Check Omits Sub-Transaction Reference Script Costs - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

The `EraUTxO DijkstraEra` instance delegates `getMinFeeTxUtxo` to `getConwayMinFeeTxUtxo`, which computes the reference-script fee component using only the top-level transaction's inputs. In the Dijkstra era, a top-level transaction may embed sub-transactions, each of which can reference UTxO outputs carrying large scripts. The codebase already defines `batchNonDistinctRefScriptsSize` to aggregate reference-script sizes across the whole batch, but `getMinFeeTxUtxo` never calls it. As a result, the minimum-fee check in `dijkstraUtxoTransition` accepts a fee that is systematically lower than the protocol-intended amount whenever sub-transactions use reference scripts.

---

### Finding Description

**Vulnerable instance** — `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`, line 141:

```haskell
instance EraUTxO DijkstraEra where
  ...
  getMinFeeTxUtxo = getConwayMinFeeTxUtxo   -- ← reuses Conway logic unchanged
```

`getConwayMinFeeTxUtxo` is defined in `eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs`:

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
```

`txNonDistinctRefScriptsSize` only inspects the top-level transaction's own inputs and reference inputs:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL)
          `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
```

It does **not** traverse `subTransactionsTxBodyL`.

The Dijkstra module itself provides the correct aggregation function (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`, lines 264–277):

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
        ( foldMap'
            (Sum . txNonDistinctRefScriptsSize utxo)
            (tx ^. bodyTxL . subTransactionsTxBodyL)
        )
```

This function is exported but never wired into `getMinFeeTxUtxo`.

The fee enforcement point in `dijkstraUtxoTransition` (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Utxo.hs`, line 373):

```haskell
{- minfee pp txTop utxo₀ ≤ txfee txb -}
runTest $ Shelley.validateFeeTooSmallUTxO pp tx originalUtxo
```

calls `getMinFeeTxUtxo pp tx originalUtxo`, which resolves to `getConwayMinFeeTxUtxo` and therefore silently omits the sub-transaction reference-script fee component.

---

### Impact Explanation

The reference-script fee was introduced specifically to prevent DDoS attacks where large scripts are loaded cheaply (documented in `docs/adr/2024-08-14_009-refscripts-fee-change.md`; an actual attack occurred on 25 June 2024). In the Dijkstra era, an attacker can embed sub-transactions that each reference UTxO outputs carrying large Plutus scripts. The top-level fee check ignores those sub-transaction reference-script sizes entirely, so the attacker pays only the size-linear fee for the bytes of the batch, not the tiered exponential reference-script surcharge that `tierRefScriptFee` is designed to impose. This allows the attacker to force nodes to deserialize and validate large scripts at a cost far below the protocol-intended price, recreating the pre-Conway DDoS vector in the new era.

**Impact class:** Medium — attacker-controlled transactions modify fees outside design parameters.

---

### Likelihood Explanation

Any unprivileged user can submit a Dijkstra top-level transaction. No special key, governance role, or majority is required. The attacker only needs to craft a top-level transaction whose sub-transactions reference UTxO outputs that hold large scripts. The fee shortfall scales with the total size of sub-transaction reference scripts, so the attack is economically rational whenever the omitted fee exceeds the cost of constructing the batch.

---

### Recommendation

Override `getMinFeeTxUtxo` in the `EraUTxO DijkstraEra` instance to use `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
instance EraUTxO DijkstraEra where
  ...
  getMinFeeTxUtxo pp tx utxo =
    getMinFeeTx pp tx (batchNonDistinctRefScriptsSize utxo tx)
```

This mirrors the pattern already used by `getConwayMinFeeTxUtxo` but substitutes the batch-aware size function that the Dijkstra module already exports.

---

### Proof of Concept

1. Deploy a UTxO output containing a large Plutus script (e.g., 50 KiB) as a reference script.
2. Construct a Dijkstra top-level transaction with:
   - A minimal top-level body (no reference inputs at the top level).
   - One or more sub-transactions whose `referenceInputsTxBodyL` point to the large-script UTxO.
3. Set `feeTxBodyL` to the value returned by `getConwayMinFeeTxUtxo` (which ignores sub-transaction reference scripts).
4. Submit the transaction.

The `validateFeeTooSmallUTxO` check passes because it calls `getMinFeeTxUtxo = getConwayMinFeeTxUtxo`, which computes the fee without the sub-transaction reference-script surcharge. The node accepts the transaction and processes the large reference scripts at below-protocol cost.
