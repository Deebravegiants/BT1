### Title
Missing Subgroup Check in `blob_to_keyset` Allows Non-Prime-Order Root Public Key Acceptance ‚Äî (`src/cbmpc/api/hd_keyset_eddsa_2p.cpp`)

### Summary

`blob_to_keyset` in `hd_keyset_eddsa_2p.cpp` deserializes `root_Q_compressed` via `from_bin` but never calls `curve.check()` (which performs the `is_in_subgroup` test). Every analogous deserialization path in the same codebase does call `curve.check()`. An attacker who supplies a crafted keyset blob containing a valid-on-curve but non-prime-order Ed25519 point causes `extract_root_public_key_compressed` to return `SUCCESS` and emit that point as the root public key.

### Finding Description

`blob_to_keyset` performs the following validation on the public key field:

```cpp
error_t rv = keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed);
if (rv) return rv;
return keyset.root.K.from_bin(keyset.curve, blob.root_K_compressed);
``` [1](#0-0) 

`from_bin` delegates to `ec25519_core::from_bin`, which decodes the compressed point and verifies it lies on the Ed25519 curve, but does **not** verify prime-order subgroup membership. [2](#0-1) 

The analogous function `blob_to_key` in `eddsa2pc.cpp` adds the required guard immediately after `from_bin`:

```cpp
error_t rv = key.Q.from_bin(key.curve, blob.Q_compressed);
if (rv) return coinbase::error(rv, "invalid key blob");
if (key.curve.check(key.Q)) return coinbase::error(E_FORMAT, "invalid key blob");
``` [3](#0-2) 

`extract_Q_from_key_blob` in `eddsa_mp.cpp` also applies the same guard:

```cpp
rv = Q.from_bin(curve, blob.Q_compressed);
if (rv) return coinbase::error(rv, "invalid key blob");
if (curve.check(Q)) return coinbase::error(E_FORMAT, "invalid key blob");
``` [4](#0-3) 

`curve.check()` internally calls `is_in_subgroup`, which is defined for Ed25519 as:

```cpp
bool ecurve_ed_t::is_in_subgroup(const ecc_point_t& P) const {
  if (!is_on_curve(P)) return false;
  return ec25519_core::is_in_subgroup(P.storage);
}
``` [5](#0-4) 

Because `blob_to_keyset` omits this call, a point with a non-trivial torsion component (Ed25519 cofactor = 8 means there are 7 non-identity low-order points) passes deserialization. `extract_root_public_key_compressed` then re-serializes the unchecked point and returns `SUCCESS`:

```cpp
error_t extract_root_public_key_compressed(mem_t keyset_blob, buf_t& out_Q_compressed) {
  ...
  const error_t rv = deserialize_keyset_blob(keyset_blob, keyset);
  if (rv) return rv;
  out_Q_compressed = keyset.root.Q.to_compressed_bin();
  return SUCCESS;
}
``` [6](#0-5) 

The same missing check also affects `refresh` and `derive_eddsa_2p_keys`, both of which call `deserialize_keyset_blob` ‚Üí `blob_to_keyset` without any subsequent subgroup validation. [7](#0-6) [8](#0-7) 

### Impact Explanation

Any caller of `extract_root_public_key_compressed` that trusts the returned 32-byte value as a valid Ed25519 public key will accept a point outside the prime-order subgroup. Downstream operations (signature verification, HD derivation consistency checks, key agreement) that assume the public key is in the prime-order subgroup may produce incorrect or forgeable results. This is a public API reachable validation bypass that causes accepted invalid cryptographic output, matching the High severity criterion.

### Likelihood Explanation

The attacker only needs to supply a crafted `keyset_blob` byte string to the public API. No privileged access, threshold collusion, or protocol participation is required. The 7 non-identity low-order Ed25519 points are publicly known constants, making crafting trivial.

### Recommendation

Add the subgroup check in `blob_to_keyset` immediately after each `from_bin` call, mirroring the pattern used in `blob_to_key` (`eddsa2pc.cpp`) and `extract_Q_from_key_blob` (`eddsa_mp.cpp`):

```cpp
error_t rv = keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed);
if (rv) return rv;
if (keyset.curve.check(keyset.root.Q)) return coinbase::error(E_FORMAT, "invalid keyset blob");
rv = keyset.root.K.from_bin(keyset.curve, blob.root_K_compressed);
if (rv) return rv;
if (keyset.curve.check(keyset.root.K)) return coinbase::error(E_FORMAT, "invalid keyset blob");
``` [9](#0-8) 

### Proof of Concept

1. Take any of the 7 known non-identity low-order Ed25519 points (e.g., the point of order 2: `(0, p-1)` encoded as `ecffffffffffffffffffffffffffffffffffffffffffffffffffffffffffff7f`).
2. Construct a `keyset_blob_v1_t` with `root_Q_compressed` set to that 32-byte encoding, valid `role`/`curve` fields, and in-range scalar shares.
3. Serialize it and call `extract_root_public_key_compressed`.
4. Assert the return value is `SUCCESS`.
5. Deserialize the returned 32-byte point and call `curve_ed25519.is_in_subgroup()` on it ‚Äî it returns `false`, confirming the output is not a valid Ed25519 public key.

static error_t blob_to_keyset(const keyset_blob_v1_t& blob, coinbase::mpc::key_share_eddsa_hdmpc_2p_t& keyset) {
  if (blob.role > 1) return coinbase::error(E_FORMAT, "invalid keyset blob role");
  if (static_cast<curve_id>(blob.curve) != curve_id::ed25519)
    return coinbase::error(E_FORMAT, "invalid keyset blob curve");

  keyset.party_index = static_cast<coinbase::mpc::party_idx_t>(blob.role);
  keyset.curve = coinbase::crypto::curve_ed25519;

  const coinbase::crypto::mod_t& q = keyset.curve.order();
  if (!q.is_in_range(blob.x_share)) return coinbase::error(E_FORMAT, "invalid keyset blob");
  if (!q.is_in_range(blob.k_share)) return coinbase::error(E_FORMAT, "invalid keyset blob");

  keyset.root.x_share = blob.x_share;
  keyset.root.k_share = blob.k_share;

  error_t rv = keyset.root.Q.from_bin(keyset.curve, blob.root_Q_compressed);
  if (rv) return rv;
  return keyset.root.K.from_bin(keyset.curve, blob.root_K_compressed);
}
