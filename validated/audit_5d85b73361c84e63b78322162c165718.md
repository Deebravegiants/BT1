### Title
Missing `promise_batch_action_withdraw_from_gas_key` Host Function Creates Irreversible Gas Key Deposit via Contract Execution — (File: `runtime/near-vm-runner/src/logic/logic.rs`)

---

### Summary

The NEAR VM exposes `promise_batch_action_transfer_to_gas_key` so that contracts can fund gas key balances, but provides no corresponding `promise_batch_action_withdraw_from_gas_key` host function. Funds moved into a gas key balance via contract execution cannot be programmatically returned to the account balance. For contract accounts that hold no full-access key — a common production pattern — those funds are permanently locked with no on-chain recovery path.

---

### Finding Description

`promise_batch_action_transfer_to_gas_key` is fully implemented and exposed as a host function: [1](#0-0) 

The `WithdrawFromGasKeyAction` struct exists and its runtime handler `action_withdraw_from_gas_key` is implemented: [2](#0-1) 

However, the action definition carries an explicit comment acknowledging the asymmetry: [3](#0-2) 

> "This action must only be available via transactions, not via contract execution (there is no corresponding promise batch action host function)."

Searching `runtime/near-vm-runner/src/imports.rs` and `runtime/near-vm-runner/src/logic/logic.rs` confirms there is no `promise_batch_action_withdraw_from_gas_key` import or implementation. The `imports!` macro that registers every callable host function contains `promise_batch_action_transfer_to_gas_key` but no withdraw counterpart: [4](#0-3) 

The action-level validator for `WithdrawFromGasKey` only gates on protocol version, not on whether the action originates from a transaction or a contract-emitted receipt: [5](#0-4) 

This means the withdrawal path is structurally unreachable from contract execution: no host function exists to append a `WithdrawFromGasKey` action to a promise batch, so any NEAR deposited into a gas key via `promise_batch_action_transfer_to_gas_key` can only be recovered by a direct signed transaction carrying `WithdrawFromGasKey`.

---

### Impact Explanation

**Corrupted protocol value:** The gas key balance (stored in the trie as part of the `AccessKey` entry) and the account's `amount` field are the concrete protocol values affected. When a contract deposits NEAR into a gas key via the host function, the account's `amount` is debited and the gas key's `balance` is credited. There is no host function to reverse this, so the account balance is permanently reduced from the contract's perspective.

For contract accounts that have no full-access key — a standard hardened deployment pattern — no party can sign a direct `WithdrawFromGasKey` transaction. The gas key balance becomes permanently inaccessible: it cannot be spent as a deposit (gas key balance only covers gas, not deposits), cannot be transferred out, and cannot be reclaimed by the account. The NEAR is effectively burned from the account's usable balance.

**Impact: Medium** — funds locked in gas key balance are not lost to the protocol (they still exist in the trie), but they are permanently inaccessible to the account owner if no full-access key exists, reducing the account's spendable balance irreversibly.

---

### Likelihood Explanation

**Likelihood: Medium** — The `promise_batch_action_transfer_to_gas_key` host function is intentionally exposed for contracts to manage gas key funding programmatically (confirmed by the test at `test-loop-tests/src/tests/gas_keys.rs` line 635). Any contract that funds gas keys on behalf of users, or that manages its own gas key balance, will encounter this asymmetry. Contract accounts without full-access keys are common in production. A user interacting with such a contract via a public RPC transaction can trigger the deposit path; the missing withdraw path then leaves their funds stranded. [6](#0-5) 

---

### Recommendation

Add a `promise_batch_action_withdraw_from_gas_key` host function in `runtime/near-vm-runner/src/logic/logic.rs` mirroring the structure of `promise_batch_action_transfer_to_gas_key`. Register it in the `imports!` macro in `runtime/near-vm-runner/src/imports.rs`. Expose the corresponding wasmtime shim in `runtime/near-vm-runner/src/wasmtime_runner/logic.rs`. Gate the new host function on `ProtocolFeature::GasKeys` (already used by the deposit path). This restores symmetry and allows contracts to programmatically recover gas key balances.

---

### Proof of Concept

1. Deploy a contract account `vault.near` with no full-access key (only a function-call key or gas key).
2. The contract calls `promise_batch_action_transfer_to_gas_key` targeting a gas key on `vault.near`, depositing e.g. 10 NEAR from the contract's account balance into the gas key balance.
3. The gas key balance increases by 10 NEAR; the account `amount` decreases by 10 NEAR — both trie values are updated.
4. The contract attempts to recover the funds: no `promise_batch_action_withdraw_from_gas_key` host function exists, so no receipt carrying `WithdrawFromGasKey` can be constructed from within contract execution.
5. Since `vault.near` has no full-access key, no external party can sign a direct `WithdrawFromGasKey` transaction either.
6. The 10 NEAR is permanently locked in the gas key balance. The account's spendable `amount` is permanently reduced by 10 NEAR with no on-chain recovery path.

    /// Appends `TransferToGasKey` action to the batch of actions for the given promise pointed by
    /// `promise_idx`.
    ///
    /// # Errors
    ///
    /// * If `promise_idx` does not correspond to an existing promise returns `InvalidPromiseIndex`.
    /// * If the promise pointed by the `promise_idx` is an ephemeral promise created by
    /// `promise_and` returns `CannotAppendActionToJointPromise`.
    /// * If the given public key is not a valid (e.g. wrong length) returns `InvalidPublicKey`.
    /// * If `amount_ptr + 16` or `public_key_len + public_key_ptr` points outside the memory of the
    /// guest or host returns `MemoryAccessViolation`.
    /// * If called as view function returns `ProhibitedInView`.
    ///
    /// # Cost
    ///
    /// `burnt_gas := base + dispatch action base fee + dispatch action per byte fee * num bytes + cost of reading public key and u128 from memory`
    /// `used_gas := burnt_gas + exec action base fee + exec action per byte fee * num bytes`
    pub fn promise_batch_action_transfer_to_gas_key(
        &mut self,
        promise_idx: u64,
        public_key_len: u64,
        public_key_ptr: u64,
        amount_ptr: u64,
    ) -> Result<()> {
        self.result_state.gas_counter.pay_base(base)?;
        if self.context.is_view() {
            return Err(HostError::ProhibitedInView {
                method_name: "promise_batch_action_transfer_to_gas_key".to_string(),
            }
            .into());
        }
        let public_key = self.get_public_key(public_key_ptr, public_key_len)?;
        let amount = Balance::from_yoctonear(
            self.memory.get_u128(&mut self.result_state.gas_counter, amount_ptr)?,
        );
        let (receipt_idx, sir) = self.promise_idx_to_receipt_idx_with_sir(promise_idx)?;
        let receiver_id = self.ext.get_receipt_receiver(receipt_idx);
        let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
        let exec = gas_key_transfer_exec_fee(
            &self.fees_config,
            receiver_id.len(),
            public_key_len as usize,
        );
        let burn_base = send.base;
        let use_base =
            burn_base.gas.checked_add(exec.base.gas).ok_or(HostError::IntegerOverflow)?;
        self.result_state.gas_counter.pay_action_accumulated(
            burn_base,
            use_base,
            ActionCosts::gas_key_transfer_base,
        )?;
        let burn_byte = send.per_byte;
        let use_byte =
            burn_byte.gas.checked_add(exec.per_byte.gas).ok_or(HostError::IntegerOverflow)?;
        self.result_state.gas_counter.pay_action_accumulated(
            burn_byte,
            use_byte,
            ActionCosts::gas_key_byte,
        )?;
        self.result_state.deduct_balance(amount)?;
        self.ext.append_action_transfer_to_gas_key(receipt_idx, public_key.decode()?, amount);
        Ok(())
    }

pub(crate) fn action_withdraw_from_gas_key(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    result: &mut ActionResult,
    account_id: &AccountId,
    action: &WithdrawFromGasKeyAction,
) -> Result<(), RuntimeError> {
    let Some(mut access_key) = get_access_key(state_update, account_id, &action.public_key)? else {
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };
    let Some(gas_key_info) = access_key.gas_key_info_mut() else {
        // Key exists but is not a gas key
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };

    let Some(updated_balance) = gas_key_info.balance.checked_sub(action.amount) else {
        result.result = Err(ActionErrorKind::InsufficientGasKeyBalance {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
            balance: gas_key_info.balance,
            required: action.amount,
        }
        .into());
        return Ok(());
    };
    gas_key_info.balance = updated_balance;
    set_access_key(state_update, account_id.clone(), action.public_key.clone(), &access_key);

    let new_account_balance = account.amount().checked_add(action.amount).ok_or_else(|| {
        RuntimeError::StorageError(StorageError::StorageInconsistentState(
            "Account balance integer overflow".to_string(),
        ))
    })?;
    account.set_amount(new_account_balance);
    Ok(())
