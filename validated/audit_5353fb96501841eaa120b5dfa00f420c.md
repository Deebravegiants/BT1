Audit Report

## Title
Stale `CrossChainRateReceiver` Rate Used in L2 Pool Mints Excess wrsETH, Undercollateralizing the Wrapper - (File: contracts/cross-chain/CrossChainRateReceiver.sol)

## Summary
`CrossChainRateReceiver.getRate()` returns the last LayerZero-pushed rate with no staleness enforcement. Both `RSETHPoolV3` and `RSETHPoolV3ExternalBridge` use this rate to mint wrsETH to depositors, while the L1 mints rsETH at the current (higher) rate after bridging. Because rsETH accrues staking rewards monotonically, a stale rate is always lower than the current L1 rate, causing the L2 pool to mint more wrsETH per ETH than the rsETH that will back it, leaving the `RsETHTokenWrapper` undercollateralized and preventing some holders from withdrawing.

## Finding Description

**Rate storage with no staleness guard:**

`CrossChainRateReceiver.lzReceive()` stores the pushed rate and a timestamp but `getRate()` returns the raw stored value unconditionally:

```solidity
// CrossChainRateReceiver.sol:93-97
uint256 _rate = abi.decode(_payload, (uint256));
rate = _rate;
lastUpdated = block.timestamp;

// CrossChainRateReceiver.sol:103-105
function getRate() external view returns (uint256) {
    return rate;
}
```

No check on `block.timestamp - lastUpdated` exists anywhere in the deposit path.

**L2 minting path uses the stale rate:**

`RSETHPoolV3.deposit()` calls `viewSwapRsETHAmountAndFee()`, which divides by the oracle rate and mints wrsETH directly via the MINTER_ROLE — without depositing any rsETH into the wrapper:

```solidity
// RSETHPoolV3.sol:304-307
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;

// RSETHPoolV3.sol:262
wrsETH.mint(msg.sender, rsETHAmount);
```

`RSETHPoolV3ExternalBridge.deposit()` follows the identical pattern. The `RsETHTokenWrapper.mint()` (line 190-192) is a privileged mint that issues wrsETH without requiring any rsETH collateral to be deposited at that moment.

**L1 mints rsETH at the current rate:**

After the bridger moves ETH to L1, `L1Vault.depositETHForL1VaultETH()` mints rsETH at the live L1 rate:

```solidity
// L1Vault.sol:151-158
uint256 rsETHAmountToMint = lrtDepositPool.getRsETHAmountToMint(ETH_IDENTIFIER, balanceOfETH);
lrtDepositPool.depositETH{ value: balanceOfETH }(rsETHAmountToMint, "");
```

**Wrapper withdrawal is 1:1:**

`RsETHTokenWrapper._withdraw()` burns wrsETH and transfers rsETH 1:1 (line 123-125). If the wrapper holds fewer rsETH tokens than wrsETH outstanding, withdrawals revert for the last holders.

**The accounting gap:**

| Step | Rate | wrsETH/rsETH for 1 ETH |
|---|---|---|
| L2 wrsETH mint | `R_stale` (lower) | `1e18 / R_stale` (larger) |
| L1 rsETH mint | `R_current` (higher) | `1e18 / R_current` (smaller) |

The difference `(1/R_stale − 1/R_current) × deposit` is wrsETH issued with no rsETH backing.

**Existing mitigations are insufficient:**

The `limitDailyMint` modifier caps total daily wrsETH minted but does not prevent the rate mismatch from occurring within that cap. The `paused` flag requires privileged action. `updateRate()` on the provider is permissionless but requires the caller to pay LayerZero gas; no on-chain mechanism forces it to be called before deposits.

## Impact Explanation

**High — Theft of unclaimed yield:** An attacker who deposits when the rate is stale receives wrsETH representing yield that has not yet been propagated to L2. When they later call `withdraw()`, they redeem more rsETH (in ETH value) than they deposited, extracting value from existing holders whose rsETH backing is diluted.

**Medium — Temporary freezing of funds:** Because the wrapper holds fewer rsETH than wrsETH outstanding, the last holders to attempt `withdraw()` will find the wrapper's rsETH balance insufficient and their transactions will revert until the deficit is externally covered.

## Likelihood Explanation

The rate becomes stale naturally whenever `updateRate()` is not called. rsETH accrues staking rewards continuously (~3–5% APY), so even 24–48 hours of staleness creates a measurable discrepancy (~0.01–0.03%). An attacker requires no privileged access: they monitor `CrossChainRateReceiver.lastUpdated` on-chain, wait for inactivity, and call `RSETHPoolV3.deposit()` or `RSETHPoolV3ExternalBridge.deposit()` with the maximum ETH allowed by the daily mint limit. The attack is repeatable each day up to `dailyMintLimit`.

## Recommendation

Add a staleness guard in `viewSwapRsETHAmountAndFee()` in both pool contracts, or enforce it inside `CrossChainRateReceiver.getRate()` itself so all consumers are protected:

```solidity
uint256 public constant MAX_RATE_AGE = 24 hours;

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= MAX_RATE_AGE, "Rate is stale");
    return rate;
}
```

Alternatively, integrate a keeper or Chainlink Automation job that calls `updateRate()` on a fixed cadence and pause deposits automatically if the rate has not been refreshed within `MAX_RATE_AGE`.

## Proof of Concept

1. L1 rsETH price is `1.05e18` (current, after 48 h of staking rewards).
2. `CrossChainRateReceiver.rate` is `1.00e18` (stale, `lastUpdated` is 48 h ago).
3. Attacker calls `RSETHPoolV3.deposit{value: 1 ether}("")`.
   - `viewSwapRsETHAmountAndFee(1e18)` → `rsETHAmount = 1e18 * 1e18 / 1.00e18 = 1.000e18`.
   - `wrsETH.mint(attacker, 1.000e18)` — no rsETH deposited into wrapper.
   - Correct amount at current rate: `1e18 * 1e18 / 1.05e18 ≈ 0.952e18`.
   - Excess wrsETH minted: `≈ 0.048e18`.
4. Bridger calls `moveAssetsForBridging()` → 1 ETH sent to L1Vault.
5. Manager calls `L1Vault.depositETHForL1VaultETH()` → `≈ 0.952e18` rsETH minted at current rate.
6. Manager calls `L1Vault.bridgeRsETHToL2()` → `0.952e18` rsETH bridged to L2 wrapper.
7. Bridger calls `RsETHTokenWrapper.depositBridgerAssets(rsETH, 0.952e18)`.
8. Wrapper now holds `0.952e18` rsETH but has `1.000e18` wrsETH outstanding.
9. Attacker calls `withdraw(rsETH, 1.000e18)` → receives `1.000e18` rsETH worth `1.05 ETH` at current rate, having deposited `1 ETH`. The `0.048e18` rsETH shortfall falls on existing holders, some of whom cannot withdraw.
