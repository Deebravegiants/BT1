### Title
Single Depositor Can Exhaust Entire Daily Mint Limit in One Transaction, Freezing Pool Deposits for All Other Users for Up to 24 Hours - (File: contracts/pools/RSETHPoolV2ExternalBridge.sol)

---

### Summary

The `limitDailyMint` modifier in `RSETHPoolV2ExternalBridge` enforces a global `dailyMintLimit` with no per-user cap. Any single depositor with sufficient ETH can consume the entire daily allowance in a single `deposit()` call, causing all subsequent deposit attempts by other users to revert with `DailyMintLimitExceeded` until the next day boundary is crossed.

---

### Finding Description

The `deposit()` function applies the `limitDailyMint(msg.value)` modifier, which tracks a single global counter `dailyMintAmount` against `dailyMintLimit`. [1](#0-0) 

Inside the modifier, there is no per-address accounting — only a global running total: [2](#0-1) 

The day resets only when `getCurrentDay()` exceeds `lastMintDay`: [3](#0-2) 

`getCurrentDay()` is computed as: [4](#0-3) 

Because `deposit()` has no role restriction (any EOA can call it) and no per-user deposit cap, a single actor can send enough ETH in one transaction to push `dailyMintAmount` to exactly `dailyMintLimit`. After that, every subsequent deposit — even for 1 wei — will revert because `dailyMintAmount + rsETHAmount > dailyMintLimit`.

The freeze persists for up to 24 hours, until `block.timestamp` crosses the next day boundary relative to `startTimestamp`.

---

### Impact Explanation

**Medium — Temporary freezing of funds.**

The pool's deposit functionality is completely blocked for all other users for up to 24 hours per attack. The attacker receives rsETH in return (no capital loss, only opportunity cost), making the attack economically viable for a well-capitalized actor. The daily limit is a protocol-level security control; exhausting it in a single transaction defeats its purpose and denies service to all other depositors for the remainder of the day.

---

### Likelihood Explanation

**Medium.** The attack requires capital equal to the ETH needed to mint `dailyMintLimit` rsETH. Since the attacker receives rsETH back (the deposit is not a loss), the only cost is the opportunity cost of locked capital and gas. The `deposit()` function is permissionless — no role, whitelist, or KYC gate prevents any address from calling it. A motivated whale or competitor can execute this repeatedly, once per day, at low net cost.

---

### Recommendation

Introduce a per-address deposit cap within the daily window, or enforce a maximum single-transaction deposit size relative to the remaining daily limit. For example:

```solidity
uint256 public maxSingleDepositRatio; // e.g., 1000 = 10% of dailyMintLimit

modifier limitDailyMint(uint256 amount) {
    ...
    uint256 maxSingleDeposit = dailyMintLimit * maxSingleDepositRatio / 10_000;
    if (rsETHAmount > maxSingleDeposit) revert ExceedsMaxSingleDeposit();
    ...
}
```

Alternatively, track per-user daily minted amounts and cap each user's contribution to a fraction of `dailyMintLimit`.

```solidity
// Foundry test (local fork or mock setup)
// Assumes: dailyMintLimit = 100 ether (in rsETH terms), rate = 1:1, feeBps = 0

function testSingleDepositorExhaustsDailyLimit() public {
    // Setup: dailyMintLimit set to 100e18 rsETH, rate = 1e18 (1:1), feeBps = 0
    pool.setDailyMintLimit(100 ether);

    address whale = address(0xA);
    address victim = address(0xB);
    vm.deal(whale, 100 ether);
    vm.deal(victim, 1 ether);

    // Step 1: Whale deposits exactly enough ETH to exhaust the daily limit
    vm.prank(whale);
    pool.deposit{value: 100 ether}("ref");

    // dailyMintAmount == dailyMintLimit == 100e18

    // Step 2: Victim tries to deposit any amount — reverts
    vm.prank(victim);
    vm.expectRevert(RSETHPoolV2ExternalBridge.DailyMintLimitExceeded.selector);
    pool.deposit{value: 1 ether}("ref");

    // Step 3: Confirm freeze persists until next day boundary
    vm.warp(block.timestamp + 23 hours);
    vm.prank(victim);
    vm.expectRevert(RSETHPoolV2ExternalBridge.DailyMintLimitExceeded.selector);
    pool.deposit{value: 1 ether}("ref");

    // Step 4: After 24h boundary, deposit succeeds
    vm.warp(block.timestamp + 2 hours); // crosses day boundary
    vm.prank(victim);
    pool.deposit{value: 1 ether}("ref"); // succeeds
}
```

```text
    modifier limitDailyMint(uint256 amount) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        // Calculate the amount of rsETH that will be minted
        (uint256 rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```
