### Title
Byzantine Participant Can Permanently Freeze Signing by Advertising an Arbitrarily Large Indexer Height — (`crates/node/src/network/indexer_heights.rs`, `crates/node/src/network.rs`, `crates/node/src/requests/queue.rs`)

---

### Summary

A single Byzantine participant (below the signing threshold) can permanently freeze all signing and CKD request processing on every honest node by sending a single crafted `IndexerHeight` P2P message with a near-maximal `u64` height value. Because `set_height` enforces only monotone-increase with no validity ceiling, and because `eligible_leaders_and_maximum_height` derives `maximum_height` directly from the unauthenticated advertised value, every honest node is permanently excluded from the eligible-leader set.

---

### Finding Description

**Root cause 1 — `set_height` has no upper bound** [1](#0-0) 

`set_height` only enforces monotone-increase (`if height > current`). There is no validity ceiling. Any value up to `u64::MAX` is accepted and stored permanently for the lifetime of the process.

**Root cause 2 — `run_receive_message` passes the height directly with no validation** [2](#0-1) 

When a `PeerMessage::IndexerHeight` arrives, the height field is forwarded verbatim to `set_height`. The TLS layer authenticates the *sender identity* (`message.from` is the real participant ID), but it does not validate the *content* of the height field. A Byzantine participant can therefore set its own stored height to any `u64` value.

**Root cause 3 — `all_alive_participant_ids` uses unchecked addition** [3](#0-2) 

The liveness condition is:
```
my_height <= peer_height + MAX_INDEXER_HEIGHT_DIFF   // MAX_INDEXER_HEIGHT_DIFF = 50
```

If the Byzantine peer sends `height = u64::MAX - 51`, then `peer_height + 50 = u64::MAX - 1`. The condition `my_height <= u64::MAX - 1` is `true` for any real block height, so the Byzantine peer is **included** in `alive_participants`. (Sending `u64::MAX` itself would cause wrapping overflow to 49, which would exclude the peer — the attacker must choose a value in the range `[0, u64::MAX - 50]`.)

**Root cause 4 — `eligible_leaders_and_maximum_height` derives `maximum_height` from the unauthenticated value** [4](#0-3) 

`STALE_PARTICIPANT_THRESHOLD = 10`. [5](#0-4) 

With the Byzantine peer in `alive_participants` at height `u64::MAX - 51`:
- `maximum_height = u64::MAX - 51`
- For every honest node with real height `H`: `H + 10 >= u64::MAX - 51` → **false**
- Honest nodes are excluded from `eligible_leaders`
- Only the Byzantine peer is an eligible leader; it does not process requests
- `get_requests_to_attempt` returns zero attempts for all pending requests [6](#0-5) 

**Persistence:** Because `set_height` is monotone-increasing and the tracker is in-memory, the inflated value persists for the process lifetime. On restart the tracker resets to 0, but the Byzantine peer can immediately re-send the inflated height over the already-authenticated TLS connection, reinstating the freeze within milliseconds.

---

### Impact Explanation

A single Byzantine participant below the signing threshold can freeze all signing and CKD request processing on every honest node indefinitely. No threshold collusion is required. The attack targets the request-lifecycle invariant: the set of eligible leaders must reflect only chain-verified heights; an unauthenticated height advertisement must not permanently alter participant liveness state.

---

### Likelihood Explanation

Any participant in the MPC network (even one that is not a signer) can send this message. The TLS transport authenticates the sender identity but not the height value. The attack requires sending a single P2P message and maintaining a bidirectional TCP connection — both trivially achievable by any participant node.

---

### Recommendation

1. **Clamp incoming heights to a plausible ceiling.** In `set_height`, reject or clamp any height that exceeds the local node's own height by more than a configurable maximum (e.g., `MAX_INDEXER_HEIGHT_DIFF * 2`).
2. **Use saturating/checked arithmetic in `all_alive_participant_ids`.** Replace `peer_height + MAX_INDEXER_HEIGHT_DIFF` with `peer_height.saturating_add(MAX_INDEXER_HEIGHT_DIFF)` to prevent wrapping.
3. **Exclude the Byzantine peer's height from `maximum_height` if it is implausibly large.** `eligible_leaders_and_maximum_height` should filter out heights that deviate from the local node's height by more than a bounded threshold before computing `maximum_height`.

---

### Proof of Concept

```
1. Network: 3 participants (P0=honest, P1=honest, P2=Byzantine), threshold=2.
2. P0 and P1 are at real block height ~1000.
3. P2 sends IndexerHeight { height: u64::MAX - 51 } to P0 and P1.
4. set_height stores u64::MAX - 51 for P2 on both P0 and P1.
5. all_alive_participant_ids on P0:
     my_height=1000, peer_height(P2)=u64::MAX-51
     1000 <= (u64::MAX-51)+50 = u64::MAX-1  → true  → P2 included
6. eligible_leaders_and_maximum_height on P0:
     maximum_height = u64::MAX - 51
     P0: 1000 + 10 >= u64::MAX - 51 → false → excluded
     P1: 1000 + 10 >= u64::MAX - 51 → false → excluded
     P2: (u64::MAX-51) + 10 overflows → wraps, but P2 is alive → included
     eligible_leaders = {P2} only
7. get_requests_to_attempt returns [] for all pending sign/CKD requests.
8. P2 does not process requests. Signing is frozen.
9. On P0/P1 restart, P2 immediately re-sends the inflated height → freeze reinstated.
```

    pub(super) fn eligible_leaders_and_maximum_height(&self) -> (HashSet<ParticipantId>, u64) {
        // Collect the indexer heights and alive participants. Calculate maximum available height
        // from the alive nodes. Then, filter out the participants that are not alive or are too
        // stale.
        let indexer_heights = self.network_api.indexer_heights();
        let alive_participants = self.network_api.alive_participants();
        let maximum_height = alive_participants
            .iter()
            .map(|p| indexer_heights.get(p).copied().unwrap_or(0))
            .max()
            .unwrap_or(0);
        let eligible_leaders = self
            .all_participants
            .iter()
            .filter(|p| {
                alive_participants.contains(p)
                    && indexer_heights.get(p).copied().unwrap_or(0) + STALE_PARTICIPANT_THRESHOLD
                        >= maximum_height
            })
            .copied()
            .collect::<HashSet<_>>();
        (eligible_leaders, maximum_height)
