### Title
Consensus Vote Signature Never Verified — Any Peer Can Forge Validator Votes to Manufacture Quorum - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`handle_vote` in `SingleHeightConsensus` accepts any incoming `Vote` whose `voter` field names a committee member, without ever verifying the cryptographic signature carried in `vote.signature`. A single network peer can therefore send one forged vote per committee member, manufacture a 2/3 precommit quorum for an arbitrary `ProposalCommitment`, and drive the consensus engine to `DecisionReached` for a block that was never legitimately voted on.

### Finding Description

The `Vote` wire message carries both a `voter: ContractAddress` and a `signature: RawSignature` field. [1](#0-0) 

The protobuf definition confirms the signature field is present on the wire. [2](#0-1) 

When a vote arrives, `handle_vote` in `SingleHeightConsensus` performs two checks: (1) the height matches, and (2) `vote.voter` is a member of the committee. The signature is explicitly skipped with a `TODO`: [3](#0-2) 

The state machine itself acknowledges signatures are not yet produced — self-votes are emitted with `RawSignature::default()`: [4](#0-3) 

The verification infrastructure exists in `apollo_signature_manager` (`verify_precommit_vote_signature`) but is never called on the inbound vote path: [5](#0-4) 

Because the only guard is a committee-membership check on the self-reported `voter` address, an attacker can send `N` votes — one per committee member — each with a different `voter` value drawn from the committee list and an arbitrary (or empty) signature. The duplicate-vote check (`received_vote`) keys on `(round, voter)`, so each forged vote for a distinct `voter` passes as `VoteStatus::New` and is accepted into the state machine's vote maps. [6](#0-5) 

Once the forged precommit weight exceeds the 2/3 Byzantine quorum threshold, `upon_decision` fires and emits `SMRequest::DecisionReached` for the attacker-chosen `ProposalCommitment`. [7](#0-6) 

### Impact Explanation

A single unprivileged network peer can drive every honest validator node to `DecisionReached` for a `ProposalCommitment` that was never legitimately voted on by the real committee. The resulting `Decision` is passed to `context.decision_reached`, which triggers block finalization, state-sync, and L1 anchoring for a block whose consensus was entirely fabricated. This corrupts the canonical chain state, all receipts, events, and storage roots derived from that block.

This matches the allowed impact: **High — Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload** (the `voter` field is accepted as authoritative without cryptographic binding to the actual sender).

### Likelihood Explanation

Any peer connected to the consensus gossip channel can send `Vote` messages. No privileged access is required. The attacker only needs to know the committee member addresses for the target height (publicly derivable from the committee provider). The attack requires sending at most `|committee|` messages.

### Recommendation

Before accepting a vote into the state machine, verify `vote.signature` against the public key associated with `vote.voter` in the committee. The `verify_precommit_vote_signature` function in `apollo_signature_manager` already implements the required ECDSA check and should be called inside `handle_vote` after the committee-membership check, rejecting votes with invalid or missing signatures.

### Proof of Concept

1. Attacker connects to the consensus P2P broadcast topic.
2. For height `H`, round `R`, attacker obtains the committee member addresses `[V1, V2, V3, ...]`.
3. Attacker sends `Vote { vote_type: Precommit, height: H, round: R, proposal_commitment: Some(X), voter: V1, signature: RawSignature::default() }` for each `Vi`.
4. Each vote passes the height check and the committee-membership check at line 249 of `single_height_consensus.rs`.
5. Each vote is recorded as `VoteStatus::New` (distinct `voter` keys).
6. Once accumulated weight exceeds 2/3, `upon_decision` emits `DecisionReached(Decision { block: X, round: R, precommits: [...] })`.
7. The honest node finalizes block `X` without any legitimate validator having voted for it. [8](#0-7)

    pub(crate) fn received_vote(&self, vote: &Vote) -> VoteStatus {
        let determine_status = |old: &Vote, new: &Vote| {
            if old.proposal_commitment == new.proposal_commitment {
                VoteStatus::Duplicate
            } else {
                VoteStatus::Conflict(old.clone(), new.clone())
            }
        };

        // Check Map
        let key = (vote.round, vote.voter);
        let map_entry = match vote.vote_type {
            VoteType::Prevote => self.prevotes.get(&key),
            VoteType::Precommit => self.precommits.get(&key),
        };

        if let Some((old_vote, _)) = map_entry {
            return determine_status(old_vote, vote);
        }

        // Check Queue
        for event in &self.events_queue {
            let queued_vote = match (event, vote.vote_type) {
                (StateMachineEvent::Prevote(v), VoteType::Prevote) => v,
                (StateMachineEvent::Precommit(v), VoteType::Precommit) => v,
                _ => continue,
            };

            if queued_vote.round == vote.round && queued_vote.voter == vote.voter {
                return determine_status(queued_vote, vote);
            }
        }

        VoteStatus::New
    }
