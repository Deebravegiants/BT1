### Title
Incorrect `public_key_len` (wire length) Used Instead of `trie_id_len` in Gas-Key Host Functions, Causing Wrong Gas Fee for ML-DSA-65 Keys - (File: `runtime/near-vm-runner/src/logic/logic.rs`, `runtime/near-vm-runner/src/wasmtime_runner/logic.rs`)

### Summary

The host functions `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, and `promise_batch_action_add_gas_key_with_function_call` pass the raw borsh wire length (`public_key_len as usize`) to the gas-key fee helpers, while the transaction-conversion path passes `public_key.trie_id_len()`. For ML-DSA-65 keys the two values diverge by ~59× (1953 vs 33 bytes), causing a massive overcharge in gas fees when these host functions are called with ML-DSA-65 keys.

The `gas_key_transfer_send_fee` per-byte component multiplies `gas_key_byte.send_fee` by the supplied length: [10](#0-9) 

With `public_key_len = 1953` instead of `33`, the per-byte send fee is overcharged by a factor of ~59. The exec fee compounds this through `access_key_key_len(account_id_len, public_key_len)`, which adds the full 1953-byte wire length to the trie-key length estimate instead of the 33-byte hash. The fee-parity integration test only exercises ED25519 keys, so the discrepancy is not caught: [13](#0-12) 

### Impact Explanation

The corrupted protocol value is **gas balance**: the gas charged for a `TransferToGasKey` or `AddGasKey` action executed via a host function is incorrect when an ML-DSA-65 public key is supplied. Concretely:

- `gas_key_byte.send_fee` = 59,357,464 gas/byte; overcharge for send = (1953 − 33) × 59,357,464 ≈ **113.9 Ggas**
- `gas_key_byte.exec_fee` = 101,435,400 gas/byte; overcharge for exec ≈ **194.8 Ggas**
- Combined overcharge per `TransferToGasKey` call: **~309 Ggas**
- For `AddGasKey` with 3 nonces the exec overcharge is multiplied by 3: **~584 Ggas**

A contract that calls `promise_batch_action_transfer_to_gas_key` or `promise_batch_action_add_gas_key_with_full_access` with an ML-DSA-65 key will burn hundreds of Ggas more than the equivalent transaction action, potentially exhausting its prepaid gas and failing with `GasExceeded`. This creates a protocol invariant violation: the same on-chain action costs a different amount of gas depending on whether it is initiated via a signed transaction or via a host function call.

### Likelihood Explanation

The `PostQuantumSignatures` protocol feature (version 85) must be active, and a contract must call one of the affected host functions with an ML-DSA-65 public key. Both conditions are reachable by any unprivileged user once the feature is enabled. Any contract that manages gas keys programmatically (e.g., a gas-key factory or a relayer) and accepts caller-supplied ML-DSA-65 keys would trigger the overcharge. Likelihood is **Low-Medium** given the novelty of ML-DSA-65 adoption, but the entry path is fully unprivileged.

### Recommendation

Replace `public_key_len as usize` with `public_key.trie_id_len()` (obtained after decoding the key) in all three host functions in both `logic.rs` and `wasmtime_runner/logic.rs`. Extend the `test_gas_key_fee_parity` test to cover ML-DSA-65 keys to prevent regression.

### Proof of Concept

1. Deploy a contract that calls `promise_batch_action_add_gas_key_with_full_access` with a borsh-encoded ML-DSA-65 public key (1953 bytes) and `num_nonces = 3`.
2. Observe that `gas_burnt` on the receipt is ~584 Ggas higher than the equivalent `AddKey(GasKeyFullAccess)` transaction action.
3. Confirm the discrepancy matches `(1953 − 33) × gas_key_byte.exec_fee × (nonce_key_len_factor) × 3`.
4. The existing `test_gas_key_fee_parity` test passes only because it uses ED25519 keys (where `len() == trie_id_len() == 33`); substituting an ML-DSA-65 key causes the assertion `add_a_outcome.gas_burnt == add_b_outcome.gas_burnt` to fail.

```rust
/// Send fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the public key length (what the sender sees).
pub fn gas_key_transfer_send_fee(
    cfg: &RuntimeFeesConfig,
    sender_is_receiver: bool,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).send_fee(sender_is_receiver);
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .send_fee(sender_is_receiver)
        .checked_mul(public_key_len as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

/// Exec fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}
```

```rust
pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =
        access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((nonce_key_len + AccessKey::NONCE_VALUE_LEN) as u64)
        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
    GasKeyAddFee { base, per_byte }
```

```rust
/// Verify that adding, funding, and deleting a gas key via transaction vs host function
/// produces identical gas_burnt and tokens_burnt on the action execution receipt.
fn test_gas_key_fee_parity(mode: GasKeyKind) {
    let mut setup = setup_host_function_test();
    let account = setup.account.clone();

    let num_nonces: NonceIndex = 4;
    let gas_key_a_signer: Signer =
        InMemorySigner::from_seed(account.clone(), KeyType::ED25519, "gas_key_tx").into();
    let gas_key_b_signer: Signer =
        InMemorySigner::from_seed(account.clone(), KeyType::ED25519, "gas_key_host_fn").into();

    let public_key_b_base64 = near_primitives_core::serialize::to_base64(
        &borsh::to_vec(&gas_key_b_signer.public_key()).unwrap(),
    );

    // Add gas key A via transaction, B via host function
    let add_a_outcome = setup.run_actions(vec![Action::AddKey(Box::new(AddKeyAction {
        public_key: gas_key_a_signer.public_key(),
        access_key: mode.access_key(num_nonces, &account),
    }))]);
    let add_b_outcome = setup.run_call_promise(serde_json::json!([
        {"batch_create": {"account_id": account.as_str()}, "id": 0},
        mode.add_action_json(num_nonces, &account, &public_key_b_base64),
    ]));
    assert_eq!(add_a_outcome.gas_burnt, add_b_outcome.gas_burnt);
    assert_eq!(add_a_outcome.tokens_burnt, add_b_outcome.tokens_burnt);
```
