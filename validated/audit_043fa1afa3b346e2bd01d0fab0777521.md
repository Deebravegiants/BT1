### Title
Missing Starknet-Level Vote Signature Verification Allows Unprivileged Peer to Impersonate Any Validator — (`crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`SingleHeightConsensus::handle_vote` accepts any `Vote` whose `voter` field matches a committee member's `ContractAddress`, without ever verifying the Starknet ECDSA signature carried in `vote.signature`. The verification function exists (`verify_precommit_vote_signature`) and the committee's `Staker.public_key` field is populated, but neither is called in the production vote path. Any peer connected to the gossipsub network can forge votes from any validator address and have them counted toward quorum.

---

### Finding Description

`handle_vote` in `single_height_consensus.rs` performs exactly two checks before accepting a vote into the state machine:

1. Height match (`vote.height != height`)
2. Committee membership by address (`self.committee.members().iter().any(|s| s.address == vote.voter)`)

The function body contains an explicit unimplemented marker:

```rust
// TODO(Asmaa): verify the signature
``` [1](#0-0) 

After those two checks pass, the vote is fed directly into the state machine as a `StateMachineEvent::Prevote` or `StateMachineEvent::Precommit` with full voting weight applied: [2](#0-1) 

The `Vote` struct carries a `signature: RawSignature` field: [3](#0-2) 

The `Staker` struct carries a `public_key: Felt` field explicitly described as "used to verify the staker's identity": [4](#0-3) 

The verification function `verify_precommit_vote_signature` is fully implemented in `apollo_signature_manager`: [5](#0-4) 

A `grep` across the entire `crates/apollo_consensus/` tree for `verify_precommit_vote_signature`, `sign_precommit_vote`, and `vote.signature` returns **zero matches**. The verification infrastructure exists but is never wired into the consensus vote path.

The libp2p gossipsub layer uses `MessageAuthenticity::Signed(keypair)`, which only authenticates the transport-layer sender's libp2p `PeerId`. It has no knowledge of the protobuf payload fields `voter` or `proposer`, which are application-level identities entirely independent of the libp2p key. A peer with any valid libp2p keypair can sign the gossipsub envelope while embedding any `ContractAddress` in `vote.voter`.

---

### Impact Explanation

An unprivileged peer connected to the gossipsub network can:

1. Observe the current committee (public, derived from the staking contract).
2. Craft `Vote` messages with `voter` set to any committee member's `ContractAddress` and any `proposal_commitment`.
3. Broadcast them; they pass all transport-layer checks (libp2p signature on the envelope) and all application-layer checks in `handle_vote` (height match + committee membership by address).
4. The state machine counts these votes with the impersonated validator's full staking weight.

By injecting enough forged votes to reach the 2/3 prevote and precommit quorum thresholds, the attacker can cause a node to emit `DecisionReached` for a proposal commitment of the attacker's choosing, without any real validator having voted. This is a direct consensus integrity violation: the node commits a block that the actual validator set never agreed to.

---

### Likelihood Explanation

The entry point is any peer that can connect to the gossipsub topic — no validator key, no stake, no privileged access required. The committee is public. The forged votes require only knowledge of committee member addresses and construction of a valid protobuf `Vote` message. The missing check is confirmed by an in-code TODO comment, meaning it has never been active in any version of this codebase.

---

### Recommendation

In `handle_vote`, after the committee membership check, retrieve the `Staker.public_key` for `vote.voter` and call `verify_precommit_vote_signature` (or an equivalent prevote verifier) against `vote.signature`. Reject the vote and report the peer if verification fails. The signing side (`sign_precommit_vote` in `SignatureManager`) and the verification function are already implemented; the gap is solely the call site in `handle_vote`.

---

### Proof of Concept

```rust
// Non-validator peer constructs a forged vote:
let forged_vote = Vote {
    vote_type: VoteType::Precommit,
    height: current_height,
    round: 0,
    proposal_commitment: Some(real_proposal_commitment),
    voter: known_committee_member_address, // any address from the public committee
    signature: RawSignature::default(),    // empty / garbage — never checked
};

// Broadcast via gossipsub with the attacker's own libp2p keypair.
// handle_vote passes:
//   1. height check: passes (correct height)
//   2. committee check: passes (voter is a real committee address)
//   3. signature check: SKIPPED (TODO comment, no call to verify_precommit_vote_signature)
// Vote is accepted into the state machine with full validator weight.
```

Repeat for enough committee members (by weight) to reach the 2/3 quorum threshold. The node emits `DecisionReached` without any real validator having signed.
