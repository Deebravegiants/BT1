Audit Report

## Title
Unbacked `MINTER_ROLE` Mint Enables Drain of altAgETH Collateral via `_withdraw` — (`contracts/agETH/AGETHTokenWrapper.sol`)

## Summary
`AGETHTokenWrapper` exposes two independent minting paths: a collateralised `_deposit` (altAgETH in → wrapper agETH out) and an uncollateralised `mint` restricted to `MINTER_ROLE`. The `_withdraw` function redeems wrapper agETH for altAgETH held in the contract without distinguishing between backed and unbacked tokens. When `AGETHPoolV3` holds `MINTER_ROLE` and mints wrapper agETH in exchange for ETH, those tokens carry no altAgETH backing, allowing any holder to drain altAgETH deposited by legitimate users and permanently break the 1:1 backing invariant.

## Finding Description
`AGETHTokenWrapper._withdraw` (lines 111–119) burns wrapper agETH from the caller and transfers an equal amount of altAgETH out of the contract. The only guard is `allowedTokens[_asset]`; there is no check that the burned tokens were originally minted with altAgETH collateral: [1](#0-0) 

`AGETHTokenWrapper.mint` (lines 165–167) is gated solely by `MINTER_ROLE` and calls `_mint` with zero altAgETH deposited: [2](#0-1) 

`AGETHPoolV3.deposit` (lines 115–128) calls `agETH.mint(msg.sender, agETHAmount)` directly upon ETH receipt, requiring `MINTER_ROLE` on the `agETH` address (i.e., `AGETHTokenWrapper`) to function at all: [3](#0-2) 

The `IERC20AgETH` interface used by `AGETHPoolV3` matches `AGETHTokenWrapper.mint` exactly: [4](#0-3) 

`maxAmountToDepositBridgerAsset` (lines 90–101) acknowledges the invariant `totalSupply() <= balanceOfAssetInWrapper` but is a view-only helper; it does not gate `_withdraw`: [5](#0-4) 

**Exploit sequence:**

| Step | `totalSupply` | `wrapper.balance(altAgETH)` |
|------|--------------|----------------------------|
| User A: `deposit(altAgETH, 100)` | 100 | 100 |
| Attacker: `AGETHPoolV3.deposit{value: X}()` → `mint(attacker, 50)` | 150 | 100 |
| Attacker: `withdraw(altAgETH, 50)` | 100 | 50 |
| User A: `withdraw(altAgETH, 100)` | **REVERTS** | 50 |

The attacker's ETH remains in `AGETHPoolV3`; the altAgETH belonging to User A is permanently stolen.

## Impact Explanation
**Critical — Direct theft of user funds and protocol insolvency.** Every altAgETH deposited via `deposit()` is redeemable 1:1. After the attack, the wrapper holds fewer altAgETH than outstanding wrapper agETH, making some depositors unable to redeem. The shortfall is permanent until the bridger manually re-collateralises via `depositBridgerAssets()`, which is itself rate-limited by `maxAmountToDepositBridgerAsset` and requires off-chain coordination. [6](#0-5) 

## Likelihood Explanation
The precondition — `AGETHPoolV3` holding `MINTER_ROLE` on `AGETHTokenWrapper` — is the intended production deployment: `AGETHPoolV3` cannot call `agETH.mint` without it, and both contracts reside in the same `contracts/agETH/` directory sharing the same token interface. Depositing ETH into `AGETHPoolV3` is a public, permissionless action (`deposit(string)` has no access control). Any attacker who can send ETH can execute this immediately after deployment, and the attack is repeatable as long as altAgETH collateral remains in the wrapper. [7](#0-6) 

## Recommendation
1. **Track backed supply separately.** Maintain a `uint256 backedSupply` that increments only in `_deposit` and `depositBridgerAssets`, and decrements in `_withdraw`. Gate `_withdraw` so it cannot burn more than `backedSupply`.
2. **Separate the token.** Give `AGETHPoolV3` its own mintable agETH token distinct from `AGETHTokenWrapper`. The wrapper should only ever mint via `_deposit` (altAgETH in → wrapper agETH out) and should not expose a free `mint()` to external minters. [8](#0-7) 

## Proof of Concept
```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

function testDrainAltAgETH() public {
    // Precondition: AGETHPoolV3 has MINTER_ROLE on AGETHTokenWrapper (intended deployment)
    wrapper.grantRole(MINTER_ROLE, address(pool));

    // User A deposits 100 altAgETH into the wrapper
    altAgETH.approve(address(wrapper), 100e18);
    wrapper.deposit(address(altAgETH), 100e18);
    assertEq(wrapper.totalSupply(), 100e18);
    assertEq(altAgETH.balanceOf(address(wrapper)), 100e18);

    // Attacker deposits ETH to AGETHPoolV3 → mints 50 wrapper agETH (no altAgETH deposited)
    vm.prank(attacker);
    pool.deposit{value: ethEquivalentOf50AgETH}("ref");
    assertEq(wrapper.totalSupply(), 150e18);
    assertEq(altAgETH.balanceOf(address(wrapper)), 100e18); // still 100

    // Attacker withdraws 50 altAgETH using unbacked wrapper agETH
    vm.prank(attacker);
    wrapper.withdraw(address(altAgETH), 50e18);
    assertEq(altAgETH.balanceOf(address(wrapper)), 50e18); // drained

    // User A's 100 wrapper agETH can now only redeem 50 altAgETH → REVERTS
    vm.expectRevert();
    wrapper.withdraw(address(altAgETH), 100e18);

    // Insolvency confirmed
    assert(altAgETH.balanceOf(address(wrapper)) < wrapper.totalSupply());
}
```
