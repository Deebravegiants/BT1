### Title
`attach_private_scalar` Accepts Attacker-Substituted Global Public Key `Q_compressed` Without Cross-Validation ‚Äî (`src/cbmpc/api/schnorr2pc.cpp`)

### Summary

`attach_private_scalar` validates that the provided scalar `x` satisfies `x*G == Qi_self` (the party's own share point), and that `pub.Q_compressed` decodes to a valid curve point. However, it performs **no check** that the global public key `Q` from the blob is consistent with `Qi_self` or any externally-known reference. An attacker who can write to the storage location of the public blob (which the API explicitly describes as "safe to persist as public-only material") can substitute any valid secp256k1 point for `Q_compressed`. `attach_private_scalar` will return `SUCCESS` and emit a key blob carrying the attacker's chosen `Q` paired with the legitimate `x_share`.

### Finding Description

`detach_private_scalar` produces a public blob containing `Q_compressed` (the global public key) with `x_share` zeroed out to the curve order: [1](#0-0) 

`attach_private_scalar` then reconstructs the full key blob. Its validation chain is:

1. Parses `public_key_blob` into `pub`, checks version/role/curve/non-empty `Q_compressed` ‚Äî structural only.
2. Parses `public_share_compressed` into `Qi_self`, checks it is on curve.
3. Parses `private_scalar_fixed` into `x`, checks range.
4. **Checks `x * G == Qi_self`** ‚Äî binds the scalar to the party's own share point.
5. Parses `pub.Q_compressed` into `Q`, checks it is on curve ‚Äî **only a curve-membership check**.
6. Writes `pub.x_share = x` and `pub.Q_compressed = Q.to_compressed_bin()` into the output blob. [2](#0-1) 

Step 5 is the missing guard. There is no check that `Q` equals `Qi_self + Qj` for any known peer share, nor any check against an externally-supplied expected public key, nor any MAC/signature over the blob. Any valid secp256k1 point passes step 5.

The `key_blob_v1_t` struct has no integrity field: [3](#0-2) 

### Impact Explanation

The output key blob is accepted with `SUCCESS` and carries the attacker's `Q_attacker` as the global public key. Downstream consequences:

- `get_public_key_compressed` / `extract_public_key_xonly` return `Q_attacker`, causing any application that derives addresses, registers the public key, or verifies signatures to operate against the wrong key.
- The signing protocol (`sign_batch`) uses `key.Q` directly in the BIP340 challenge hash and in the final signature self-verification: [4](#0-3) [5](#0-4) 

If the attacker can also tamper with P2's public blob (both blobs are described as "safe to persist as public-only material" and may reside in the same untrusted store), both parties would use `Q_attacker`, the signing protocol would complete successfully, and the resulting signature would verify under `Q_attacker` ‚Äî an attacker-chosen key ‚Äî rather than the legitimate key established during DKG.

Even in the single-blob case (only P1's blob tampered), the accepted key blob causes P1 to compute the BIP340 challenge under `Q_attacker` while P2 uses the legitimate `Q`, producing a protocol failure that leaks the disagreement and disrupts availability.

### Likelihood Explanation

The public blob is explicitly documented as safe for untrusted storage (backup servers, cloud storage, PVE batch backup flows as shown in the demo). Any storage-layer attacker ‚Äî a compromised backup service, a malicious storage provider, or a man-in-the-middle on the retrieval path ‚Äî can flip `Q_compressed` to any valid point. The modification requires no knowledge of the scalar and passes all existing validation in `attach_private_scalar`.

### Recommendation

`attach_private_scalar` should require the caller to supply the expected global public key as an additional `mem_t expected_Q_compressed` parameter and reject any blob whose `Q_compressed` does not match it byte-for-byte after normalization. Alternatively, the public blob should be integrity-protected (e.g., HMAC under a caller-supplied key, or an authenticated encryption wrapper) so that any tampering is detected before the blob is used.

### Proof of Concept

```
// After DKG, P1 holds key_blob_1.
buf_t pub_blob, x_fixed;
ASSERT_EQ(detach_private_scalar(key_blob_1, pub_blob, x_fixed), SUCCESS);

buf_t Qi;
ASSERT_EQ(get_public_share_compressed(key_blob_1, Qi), SUCCESS);

// Attacker: deserialize pub_blob, replace Q_compressed with an arbitrary
// valid secp256k1 point Q_evil, re-serialize into tampered_pub.
key_blob_v1_t evil;
coinbase::convert(evil, pub_blob);
evil.Q_compressed = Q_evil_compressed;   // any valid curve point
buf_t tampered_pub = coinbase::convert(evil);

// attach_private_scalar accepts the tampered blob.
buf_t out;
error_t rv = attach_private_scalar(tampered_pub, x_fixed, Qi, out);
ASSERT_EQ(rv, SUCCESS);   // passes ‚Äî no Q vs Qi cross-check

// The resulting blob reports Q_evil as the global public key.
buf_t reported_Q;
get_public_key_compressed(out, reported_Q);
ASSERT_EQ(reported_Q, Q_evil_compressed);   // attacker's point, not the DKG key
```
