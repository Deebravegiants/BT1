### Title
Unauthenticated SQMR Peer Can Inject Arbitrary Transaction Hashes Into Storage, Corrupting `starknet_getTransactionByHash` RPC Results — (`crates/apollo_p2p_sync/src/client/transaction.rs`)

---

### Summary

`TransactionStreamFactory::parse_data_for_block` accepts `transaction_hash` from a peer-supplied `FullTransaction` and stores it directly into the `transaction_hash_to_idx` table without verifying it against the Poseidon/Pedersen hash of the transaction body. Any unauthenticated SQMR peer can inject an arbitrary `Felt` as the canonical hash for any transaction in any synced block, permanently corrupting the node's transaction lookup index and causing `starknet_getTransactionByHash` to return authoritative-looking wrong results.

---

### Finding Description

In `parse_data_for_block`, the `transaction_hash` field of each peer-supplied `FullTransaction` is pushed directly into `block_body.transaction_hashes` with no integrity check: [1](#0-0) 

The `BlockBody` is then committed to storage via `append_body`, which calls `write_transactions`. That function inserts the peer-supplied hash as the canonical key in `transaction_hash_to_idx_table` and as the `tx_hash` field of `TransactionMetadata`: [2](#0-1) 

The block header synced earlier carries a `transaction_commitment` (a Patricia root over the real transaction hashes): [3](#0-2) 

However, **no code path verifies the synced transaction hashes against this commitment** before or after storage. The TODO at line 88 of `transaction.rs` explicitly acknowledges the gap: [4](#0-3) 

---

### Impact Explanation

The RPC handler `get_transaction_by_hash` resolves transactions exclusively through `get_transaction_idx_by_hash`, which queries the corrupted `transaction_hash_to_idx` table: [5](#0-4) 

After a malicious peer injects fake hash `F` for a transaction whose real hash is `R`:

- `starknet_getTransactionByHash(R)` → `TRANSACTION_HASH_NOT_FOUND` (real hash absent from index)
- `starknet_getTransactionByHash(F)` → returns the transaction body paired with `F`, an authoritative-looking but cryptographically invalid response

This is a persistent, storage-level corruption: the fake hash survives node restarts and is served to all downstream RPC consumers.

---

### Likelihood Explanation

The P2P sync client is the default sync mode for sequencer nodes not acting as proposers. The `StateSyncRunner` runs both the P2P sync client and the RPC server in the same process: [6](#0-5) 

Running a peer node in the Starknet P2P network is permissionless. An attacker needs only to operate a node that the victim connects to via SQMR and respond to a `TransactionQuery` with a `FullTransaction` carrying a crafted `transaction_hash`. No validator, operator, or privileged credentials are required.

---

### Recommendation

Before pushing `transaction_hash` into `block_body.transaction_hashes`, recompute the hash from the transaction body and reject (and penalize) the peer if it does not match:

```rust
let recomputed = transaction.calculate_hash(/* chain_id */);
if recomputed != transaction_hash {
    return Err(ParseDataError::BadPeer(BadPeerError::InvalidTransactionHash { ... }));
}
block_body.transaction_hashes.push(transaction_hash);
```

Additionally, after all transaction hashes for a block are collected, verify the resulting transaction commitment against the value stored in the block header (`header.transaction_commitment`) before writing to storage.

---

### Proof of Concept

Using the existing `run_test` / `MockClientResponsesManager` harness in `crates/apollo_p2p_sync/src/client/test_utils.rs`: [7](#0-6) 

1. Sync a header for block 0 with `n_transactions = 1`.
2. Send a `FullTransaction` where `transaction_hash = Felt::from(0xdeadbeef)` but the transaction body is a valid `InvokeTransaction` whose real hash is `R ≠ 0xdeadbeef`.
3. Send `DataOrFin(None)` to close the session.
4. Read storage:
   - `get_transaction_idx_by_hash(R)` → `None` (real hash not indexed)
   - `get_transaction_idx_by_hash(Felt::from(0xdeadbeef))` → `Some(TransactionIndex(0, 0))` (fake hash indexed)
   - `get_block_transaction_hashes(BlockNumber(0))` → `[Felt::from(0xdeadbeef)]`
5. Assert `stored_hash != recomputed_hash` — the invariant is violated.
