### Title
Pool Singleton Absorb Spend in LEAVING_POOL State Resets Lock Height, Preventing Pool Exit - (File: `chia/pools/pool_wallet.py`, `chia/pools/pool_puzzles.py`)

### Summary

`create_absorb_spend` in `pool_puzzles.py` is valid for both `FARMING_TO_POOL` and `LEAVING_POOL` (waiting room) singleton states, and requires no signature. When an absorb spend is confirmed on-chain while the singleton is in `LEAVING_POOL` state, `apply_state_transition` records the new spend at the current block height, advancing `tip_height`. Because `new_peak` computes the unlock height as `tip_height + relative_lock_height`, each absorb spend effectively resets the cooldown window. A malicious pool operator can repeatedly submit absorb spends using accumulated farming rewards to keep the lock height perpetually in the future, permanently preventing the farmer from completing the pool-exit transition.

### Finding Description

Chia's pool-exit flow is a two-step singleton state machine:

1. **Step 1** – Farmer calls `self_pool()` / `join_pool()`, which transitions the singleton from `FARMING_TO_POOL` → `LEAVING_POOL` (the "waiting room"). The block height of this spend is recorded as `tip_height`.
2. **Step 2** – After `relative_lock_height` blocks pass, `new_peak` automatically submits the second travel transaction to complete the exit.

`new_peak` computes the unlock condition as:

```python
leave_height = tip_height + pool_wallet_info.current.relative_lock_height
if peak_height > leave_height + 2:
    await self.generate_travel_transactions(...)
``` [1](#0-0) 

`tip_height` comes from `get_tip()`, which returns the height of the **most recent singleton spend** stored by `apply_state_transition`:

```python
await self.wallet_state_manager.pool_store.add_spend(self.wallet_id, new_state, block_height)
``` [2](#0-1) 

`create_absorb_spend` explicitly handles the waiting-room (LEAVING_POOL) inner puzzle:

```python
elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
    inner_sol = Program.to([0, reward_amount, height])
``` [3](#0-2) 

The absorb spend bundle is submitted with an **empty aggregate signature** (`G2Element()`), requiring no owner authorization:

```python
claim_spend = WalletSpendBundle(all_spends, G2Element())
# If fee is 0, no signatures are required to absorb
``` [4](#0-3) 

`claim_pool_rewards` performs **no check on the current singleton state** before building and submitting absorb spends: [5](#0-4) 

The same lock-height check in `self_pool` and `join_pool` also reads `history[-1][0]` (the most recent spend height), so it is equally affected: [6](#0-5) 

### Impact Explanation

Every absorb spend confirmed while the singleton is in `LEAVING_POOL` state advances `tip_height` to the current block height, pushing `leave_height` forward by `relative_lock_height` blocks. A pool operator who continuously submits absorb spends (using any accumulated p2_singleton farming rewards, which are public and require no farmer cooperation) keeps the unlock condition permanently unsatisfied. The farmer can never complete Step 2 and is locked into the pool indefinitely. This constitutes **permanent inability for a farmer to perform a valid pool action (pool exit)** and **corruption of pool membership state with direct security impact** — the farmer's XCH block rewards continue flowing to the pool's `target_puzzle_hash` with no recourse.

### Likelihood Explanation

The attack requires only:
- Knowledge of the singleton's current on-chain state (fully public).
- Existence of at least one unspent p2_singleton farming reward coin (guaranteed to accumulate over time for any active farmer).
- No signature from the farmer.

Any pool operator motivated to retain a farmer (e.g., to continue collecting fees) can execute this continuously at negligible cost.

### Recommendation

`claim_pool_rewards` (and the underlying `create_absorb_spend` path) should refuse to build or submit an absorb spend when the singleton is in `LEAVING_POOL` state. Add a state guard at the top of `claim_pool_rewards`:

```python
current_state: PoolWalletInfo = await self.get_current_state()
if current_state.current.state == LEAVING_POOL.value:
    raise ValueError("Cannot absorb rewards while leaving pool; doing so would reset the exit lock height.")
```

Additionally, the lock-height reference point should be anchored to the block height of the **LEAVING_POOL transition spend** specifically (the first spend that set the state to `LEAVING_POOL`), not the most recent singleton spend, so that subsequent absorb spends cannot shift it.

### Proof of Concept

1. Farmer joins pool with `relative_lock_height = 32`.
2. Farmer calls `pw_self_pool` → singleton transitions to `LEAVING_POOL` at block height H. `tip_height = H`.
3. Pool operator observes the on-chain state. At block H+10, a farming reward coin exists at the p2_singleton puzzle hash.
4. Pool operator constructs and broadcasts an absorb spend bundle (no signature needed) using `create_absorb_spend` with the waiting-room inner puzzle path. The spend is confirmed at block H+10. `tip_height` is now H+10.
5. `new_peak` now requires `peak_height > (H+10) + 32 + 2 = H+44` before submitting the exit transaction.
6. Pool operator repeats step 4 every time a new farming reward accumulates. `tip_height` is always near the current chain tip.
7. The condition `peak_height > leave_height + 2` is never satisfied. The farmer is permanently trapped in `LEAVING_POOL` state. [7](#0-6) [8](#0-7)

    async def new_peak(self, peak_height: uint32) -> None:
        # This gets called from the WalletStateManager whenever there is a new peak

        pool_wallet_info: PoolWalletInfo = await self.get_current_state()
        tip_height, tip_spend = await self.get_tip()

        if self.target_state is None:
            return
        if self.target_state == pool_wallet_info.current:
            self.target_state = None
            raise ValueError(f"Internal error. Pool wallet {self.wallet_id} state: {pool_wallet_info.current}")

        if (
            self.target_state.state in {FARMING_TO_POOL.value, SELF_POOLING.value}
            and pool_wallet_info.current.state == LEAVING_POOL.value
        ):
            leave_height = tip_height + pool_wallet_info.current.relative_lock_height

            # Add some buffer (+2) to reduce chances of a reorg
            if peak_height > leave_height + 2:
                unconfirmed: list[
                    TransactionRecord
                ] = await self.wallet_state_manager.tx_store.get_unconfirmed_for_wallet(self.wallet_id)
                next_tip: Coin | None = get_most_recent_singleton_coin_from_coin_spend(tip_spend)
                assert next_tip is not None

                if any([rem.name() == next_tip.name() for tx_rec in unconfirmed for rem in tx_rec.removals]):
                    self.log.info("Already submitted second transaction, will not resubmit.")
                    return

                self.log.info(f"Attempting to leave from\n{pool_wallet_info.current}\nto\n{self.target_state}")
                assert self.target_state.version == POOL_PROTOCOL_VERSION
                assert pool_wallet_info.current.state == LEAVING_POOL.value
                assert self.target_state.target_puzzle_hash is not None

                if self.target_state.state == SELF_POOLING.value:
                    assert self.target_state.relative_lock_height == 0
                    assert self.target_state.pool_url is None
                elif self.target_state.state == FARMING_TO_POOL.value:
                    assert self.target_state.relative_lock_height >= self.MINIMUM_RELATIVE_LOCK_HEIGHT
                    assert self.target_state.pool_url is not None

                async with self.wallet_state_manager.new_action_scope(self.next_tx_config, push=True) as action_scope:
                    await self.generate_travel_transactions(self.next_transaction_fee, action_scope)

def create_absorb_spend(
    last_coin_spend: CoinSpend,
    current_state: PoolState,
    launcher_coin: Coin,
    height: uint32,
    genesis_challenge: bytes32,
    delay_time: uint64,
    delay_ph: bytes32,
) -> list[CoinSpend]:
    inner_puzzle: Program = pool_state_to_inner_puzzle(
        current_state, launcher_coin.name(), genesis_challenge, delay_time, delay_ph
    )
    reward_amount: uint64 = calculate_pool_reward(height)
    if is_pool_member_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol: Program = Program.to([reward_amount, height])
    elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, destination_puzhash, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol = Program.to([0, reward_amount, height])
    else:
        raise ValueError
    # full sol = (parent_info, my_amount, inner_solution)
    coin: Coin | None = get_most_recent_singleton_coin_from_coin_spend(last_coin_spend)
    assert coin is not None

    if coin.parent_coin_info == launcher_coin.name():
        parent_info: Program = Program.to([launcher_coin.parent_coin_info, launcher_coin.amount])
    else:
        p = Program.from_bytes(bytes(last_coin_spend.puzzle_reveal))
        last_coin_spend_inner_puzzle: Program | None = get_inner_puzzle_from_puzzle(p)
        assert last_coin_spend_inner_puzzle is not None
        parent_info = Program.to(
            [
                last_coin_spend.coin.parent_coin_info,
                last_coin_spend_inner_puzzle.get_tree_hash(),
                last_coin_spend.coin.amount,
            ]
        )
    full_solution: SerializedProgram = SerializedProgram.to([parent_info, last_coin_spend.coin.amount, inner_sol])
    full_puzzle: SerializedProgram = create_full_puzzle(inner_puzzle, launcher_coin.name()).to_serialized()
    assert coin.puzzle_hash == full_puzzle.get_tree_hash()

    reward_parent: bytes32 = pool_parent_id(height, genesis_challenge)
    p2_singleton_puzzle = create_p2_singleton_puzzle(
        SINGLETON_MOD_HASH, launcher_coin.name(), delay_time, delay_ph
    ).to_serialized()
    reward_coin: Coin = Coin(reward_parent, p2_singleton_puzzle.get_tree_hash(), reward_amount)
    p2_singleton_solution = SerializedProgram.to([inner_puzzle.get_tree_hash(), reward_coin.name()])
    assert p2_singleton_puzzle.get_tree_hash() == reward_coin.puzzle_hash
    assert full_puzzle.get_tree_hash() == coin.puzzle_hash
    assert get_inner_puzzle_from_puzzle(Program.from_bytes(bytes(full_puzzle))) is not None

    coin_spends = [
        CoinSpend(coin, full_puzzle, full_solution),
        CoinSpend(reward_coin, p2_singleton_puzzle, p2_singleton_solution),
    ]
    return coin_spends
