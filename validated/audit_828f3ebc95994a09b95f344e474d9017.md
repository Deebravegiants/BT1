### Title
`RSETH._transfer` Override Omits `whenNotPaused` Guard, Allowing rsETH Transfers to Bypass the Emergency Pause — (File: `contracts/RSETH.sol`)

---

### Summary

`RSETH.sol` overrides `_transfer` to enforce per-address transfer blocks but omits the `whenNotPaused` modifier. As a result, any rsETH holder can freely call `transfer` or `transferFrom` while the contract is paused, bypassing the emergency stop that correctly gates `mint` and `burnFrom`.

---

### Finding Description

`RSETH` inherits `ERC20Upgradeable` and `PausableUpgradeable` separately — it does **not** inherit `ERC20PausableUpgradeable`, so no automatic pause hook is wired into the ERC-20 transfer path. The contract overrides `_transfer` to enforce the per-address block list:

```solidity
// contracts/RSETH.sol L287-291
function _transfer(address from, address to, uint256 amount) internal override {
    _enforceNotBlocked(from);
    _enforceNotBlocked(to);
    super._transfer(from, to, amount);   // no whenNotPaused check
}
```

Meanwhile, every privileged mutation correctly gates on the pause state:

```solidity
// mint — L233-239
function mint(...) external onlyRole(LRTConstants.MINTER_ROLE) whenNotPaused checkDailyMintLimit(amount) {
    _enforceNotBlocked(to);
    _mint(to, amount);
}

// burnFrom — L245-248
function burnFrom(...) external onlyRole(LRTConstants.BURNER_ROLE) whenNotPaused {
    _enforceNotBlocked(account);
    _burn(account, amount);
}
```

The asymmetry is the root cause: `_transfer` enforces one security control (`_enforceNotBlocked`) but silently skips the other (`whenNotPaused`). This is structurally identical to the reference report's `_update()` checking only the recipient blacklist and not the sender blacklist — an incomplete guard in the token-movement hook.

There is no `_beforeTokenTransfer` override in `RSETH` that could compensate, confirmed by grep returning no matches for that hook in the file.

---

### Impact Explanation

When the rsETH contract is paused (e.g., via `LRTConfig.pauseAll()` during an incident):

- `mint` is correctly blocked — no new rsETH can enter circulation.
- `burnFrom` is correctly blocked — withdrawals halt.
- **`transfer` / `transferFrom` are NOT blocked** — any holder can freely move rsETH to arbitrary addresses.

The most concrete harm: the protocol's `recoverFrozenFunds` mechanism requires the target address to hold a balance and be actively blocked. If an admin pauses the contract to investigate suspicious activity and then attempts to block a user, that user can front-run or race the `blockUserTransfers` call by transferring their rsETH to a fresh address during the pause window. The original address ends up with zero balance; `recoverFrozenFunds` recovers nothing; the funds escape custody.

**Impact: Low — Contract fails to deliver promised returns, but doesn't lose value.** The pause mechanism's security guarantee (halt all token movements) is not upheld for the `transfer`/`transferFrom` path.

---

### Likelihood Explanation

Any rsETH holder can trigger this with a standard ERC-20 `transfer` call. No special role, no oracle manipulation, no external dependency — just a wallet and a paused contract. The window is the entire duration of any pause event, which can last hours or days during an incident response.

---

### Recommendation

Add `whenNotPaused` to the `_transfer` override so that the pause state is enforced uniformly across all token-movement paths:

```solidity
function _transfer(address from, address to, uint256 amount) internal override whenNotPaused {
    _enforceNotBlocked(from);
    _enforceNotBlocked(to);
    super._transfer(from, to, amount);
}
```

Alternatively, override `_beforeTokenTransfer` to call `require(!paused(), ...)`, which is the standard OpenZeppelin pattern used by `ERC20PausableUpgradeable`.

---

### Proof of Concept

```solidity
// Foundry test sketch
function testTransferBypassesPause() public {
    // Setup: mint rsETH to user
    vm.prank(minter);
    rsETH.mint(user, 1e18);

    // Admin pauses the contract
    vm.prank(pauser);
    rsETH.pause();
    assertTrue(rsETH.paused());

    // mint is correctly blocked
    vm.expectRevert("Pausable: paused");
    vm.prank(minter);
    rsETH.mint(user, 1e18);

    // burnFrom is correctly blocked
    vm.expectRevert("Pausable: paused");
    vm.prank(burner);
    rsETH.burnFrom(user, 1e18);

    // transfer is NOT blocked — succeeds while paused
    vm.prank(user);
    rsETH.transfer(user2, 1e18);          // should revert, but doesn't
    assertEq(rsETH.balanceOf(user2), 1e18); // funds moved during pause
}
```
