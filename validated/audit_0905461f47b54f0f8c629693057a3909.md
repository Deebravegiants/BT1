### Title
Pusher-delegation consent signature is replayable before deadline, allowing creator to reinstate revoked oracle write authority — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` verifies a pusher's EIP-191 consent signature but binds it only to `(chainid, address(this), deadline, pusher, creator)`. There is no nonce and no "action" field. The deadline prevents replay *after* it expires, but the creator can replay the identical signature at any time *before* the deadline to re-establish a delegation the pusher already revoked via `revokePusher()`. This is the direct Metric OMM analog of the `confidentialApprove` replay/inversion bug: the revocation state is not tied to the signed payload, and there is no used-signature registry.

### Finding Description

`allowPushers` in `CompressedOracle.sol` constructs the signed digest as:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The signed payload contains no nonce and no boolean "allow/revoke" action field. The code comment acknowledges the partial mitigation:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [2](#0-1) 

However, the deadline only blocks replay *after* it expires. Before expiry, the creator can call `allowPushers` again with the same signature bytes to overwrite `namespaceRemapping[pusher]` back to `msg.sender`, undoing the pusher's self-revocation:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);   // cleared here
    emit PusherRevoked(msg.sender, creator);
}
``` [3](#0-2) 

There is no nonce, no used-signature mapping, and no on-chain record that the pusher revoked. The creator holds the original signature bytes off-chain and can replay them at will until the deadline timestamp passes.

### Impact Explanation

The `CompressedOracle` feeds bid/ask prices into Metric OMM pools via `_getBidAndAskPriceX64()`, which drives every live swap execution. If a pusher revokes because their signing key was compromised, the creator can replay the original consent signature to re-establish the delegation. The compromised pusher can then push arbitrary slot words into the creator's namespace. Those values decode into `mid` and `spread` prices consumed by `SwapMath.midAndSpreadFeeX64FromBidAsk`, producing bad-price execution: traders receive more output than the oracle permits, or the pool receives less input than owed, directly draining LP principal. This satisfies the "Bad-price execution" and "Pool insolvency" impact gates. [4](#0-3) 

### Likelihood Explanation

- The creator is a semi-trusted party who holds the original signature bytes off-chain (they submitted them in the original `allowPushers` call, which is public on-chain).
- The pusher's revocation is a self-service action requiring no creator cooperation, so the creator has a clear motive to replay when the pusher revokes against their wishes.
- The window is the full deadline duration (up to any value the creator chose when calling `allowPushers`; nothing caps it).
- No special privileges beyond being the original creator are required.

### Recommendation

**Short term:** Add a per-pusher nonce to the signed digest and increment it on every successful `allowPushers` call. Track the current nonce in storage so a replayed old signature recovers a stale nonce and fails verification.

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender, pusherNonce[pusher]   // <-- add nonce
    ))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
pusherNonce[pusher]++;                             // <-- increment
namespaceRemapping[pusher] = msg.sender;
```

**Long term:** Bind the "action" (allow vs. revoke) into the signed payload so a consent signature cannot be repurposed as a re-delegation after revocation, mirroring the recommendation in the reference report.

### Proof of Concept

```
1. Pusher signs consent for Creator with deadline = block.timestamp + 365 days.
   digest = keccak256(abi.encode(chainid, oracle, deadline, pusher, creator))
   sig = pusher.sign(digest)

2. Creator calls allowPushers(deadline, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓

3. Pusher's key is compromised; pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓

4. Creator (or attacker who observed the original tx) calls
   allowPushers(deadline, [pusher], [sig])  ← same sig, same deadline
   → _ensureDeadline passes (deadline is still in the future)
   → ECDSA.recover returns pusher  ← sig is still valid
   → namespaceRemapping[pusher] = creator  ← revocation undone ✗

5. Compromised pusher pushes manipulated slot words into creator's namespace.
   Pool reads manipulated bid/ask → bad-price swap execution → LP loss.
``` [5](#0-4)

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
