### Title
Chainlink `latestRoundData()` minAnswer/maxAnswer Circuit Breaker Not Checked — (`contracts/oracles/ChainlinkPriceOracle.sol`)

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` with no validation of the returned `answer` against the aggregator's `minAnswer`/`maxAnswer` circuit breaker bounds, no staleness check, and no zero/negative check. If a supported LST asset crashes below the aggregator's `minAnswer`, the oracle silently returns the floor price instead of the real price, inflating the protocol's computed TVL and rsETH price, enabling depositors of the crashed asset to steal value from all existing rsETH holders.

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` is the price source for every supported LST asset in the protocol: [1](#0-0) 

The call at line 52 discards all returned fields except `price`. There is:
- No check that `price > 0` (a negative `int256` silently wraps to a huge `uint256`)
- No staleness check (`updatedAt`, `answeredInRound`)
- No check against the aggregator's `minAnswer`/`maxAnswer` circuit breaker

This price is consumed by `LRTOracle._getTotalEthInProtocol()`, which sums `assetAmount × assetPrice` across all supported assets to compute the protocol's total ETH value: [2](#0-1) 

That total is then used to compute `newRsETHPrice`: [3](#0-2) 

Chainlink aggregators have a built-in circuit breaker: if an asset's market price falls below `minAnswer`, the aggregator continues returning `minAnswer` rather than the true price. This is exactly what happened with LUNA on Venus/BSC.

### Impact Explanation

**Critical — theft of funds from existing rsETH holders.**

If a supported LST (e.g., stETH, rETH) crashes below its Chainlink aggregator `minAnswer`:

1. `ChainlinkPriceOracle.getAssetPrice()` returns `minAnswer` (inflated relative to true price).
2. `_getTotalEthInProtocol()` overstates the protocol's ETH TVL.
3. `rsETHPrice` is set above its true value.
4. An attacker deposits the crashed asset (worth near zero on the open market) and receives rsETH priced at the inflated rate.
5. The attacker redeems rsETH for healthy assets (ETH, other LSTs), draining value from all other holders.

This is direct theft of funds from existing rsETH holders and constitutes protocol insolvency.

### Likelihood Explanation

Moderate. Chainlink's circuit breaker is a known, documented behavior. The LUNA/Venus incident is a real precedent. The LRT-rsETH protocol holds multiple LST assets, each with its own Chainlink feed and `minAnswer`. A de-peg or exploit in any one of those LSTs (e.g., a slashing event, smart contract exploit, or liquidity crisis) could trigger this path. No privileged access is required — any depositor can exploit it once the circuit breaker activates.

### Recommendation

Fetch the aggregator's `minAnswer` and `maxAnswer` from the `AggregatorV2V3Interface` and revert if the returned price falls outside those bounds:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    if (price <= 0) revert InvalidPrice();
    if (updatedAt == 0 || answeredInRound < roundId) revert StalePrice();

    // Circuit breaker bounds check
    int192 minAnswer = AggregatorV2V3Interface(assetPriceFeed[asset]).minAnswer();
    int192 maxAnswer = AggregatorV2V3Interface(assetPriceFeed[asset]).maxAnswer();
    if (price <= minAnswer || price >= maxAnswer) revert PriceOutOfBounds();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Alternatively, integrate a secondary oracle (e.g., Uniswap TWAP) as a sanity check when the Chainlink price is near its bounds.

### Proof of Concept

1. Assume `stETH` is a supported asset with a Chainlink feed whose `minAnswer` = 0.1 ETH.
2. A black-swan event causes stETH to de-peg to 0.001 ETH on the open market.
3. Chainlink's circuit breaker activates; `latestRoundData()` returns `minAnswer` = 0.1 ETH.
4. `ChainlinkPriceOracle.getAssetPrice(stETH)` returns `0.1e18` (100× the real value).
5. `_getTotalEthInProtocol()` overstates TVL by the full stETH balance × 99× difference.
6. `updateRSETHPrice()` sets `rsETHPrice` to an inflated value.
7. Attacker buys stETH cheaply on the open market (at 0.001 ETH each), deposits into `LRTDepositPool`, and receives rsETH priced at the inflated rate.
8. Attacker redeems rsETH for ETH or other healthy LSTs, extracting value from all other rsETH holders.
