### Title
Missing Zero-Address Check on `rateReceiver` in `CrossChainRateProvider.updateRate()` Allows ETH Loss and Rate Delivery Failure — (`contracts/cross-chain/CrossChainRateProvider.sol`)

---

### Summary

`CrossChainRateProvider.updateRate()` encodes `rateReceiver` directly into the LayerZero destination path without checking whether it is `address(0)`. Because `rateReceiver` is never initialized in the constructor and `updateRateReceiver()` also lacks a zero-address guard, any caller can invoke `updateRate()` while `rateReceiver == address(0)`, burning ETH on a LZ message that is routed to a non-existent destination and leaving the legitimate `RSETHRateReceiver.rate` permanently stale until the owner intervenes.

---

### Finding Description

`RSETHRateProvider` inherits from `CrossChainRateProvider`. The constructor sets `rsETHPriceOracle`, `rateInfo`, `dstChainId`, and `layerZeroEndpoint`, but **never sets `rateReceiver`**. [1](#0-0) 

`rateReceiver` therefore defaults to `address(0)` at deployment. [2](#0-1) 

`updateRate()` is `external payable` with no access control and no zero-address guard on `rateReceiver`. It encodes `rateReceiver` directly as the remote destination: [3](#0-2) 

Specifically, line 88 packs `rateReceiver` (potentially `address(0)`) as the first 20 bytes of the LZ path: [4](#0-3) 

And line 96–98 sends the message, consuming all `msg.value` as the LZ fee: [5](#0-4) 

`updateRateReceiver()` also lacks a zero-address check, meaning the owner can accidentally reset `rateReceiver` to `address(0)` at any time: [6](#0-5) 

On the receiver side, `lzReceive()` validates `srcAddress == rateProvider` but cannot be reached at all if the LZ message was sent to `address(0)` — the message is simply dropped or delivered to a null address on the destination chain: [7](#0-6) 

---

### Impact Explanation

- ETH paid as the LZ fee is permanently lost (burned into the LZ endpoint for a message that reaches no valid receiver).
- `RSETHRateReceiver.rate` on the destination chain remains `0` or stale for the entire window during which `rateReceiver` is unset or reset to `address(0)`.
- Any pool or protocol on the destination chain that reads `RSETHRateReceiver.getRate()` will receive `0` or a stale value, which can cause incorrect pricing, failed swaps, or temporary freezing of rate-dependent operations.

Impact: **Medium — Temporary freezing of funds / ETH loss**.

---

### Likelihood Explanation

- The window between contract deployment and the owner calling `updateRateReceiver()` is always present.
- `updateRate()` has no access control — any external actor (including bots or automated keepers) can call it immediately after deployment.
- `updateRateReceiver()` has no zero-address guard, so an owner mistake can re-open the window at any time post-deployment.
- No on-chain mechanism prevents or reverts the call when `rateReceiver == address(0)`.

---

### Recommendation

1. Add a zero-address check in `updateRate()`:
   ```solidity
   require(rateReceiver != address(0), "rateReceiver not set");
   ```
2. Add a zero-address check in `updateRateReceiver()`:
   ```solidity
   require(_rateReceiver != address(0), "zero address");
   ```
3. Consider initializing `rateReceiver` in the `RSETHRateProvider` constructor, or enforcing that `updateRate()` is only callable after `rateReceiver` is set.

---

### Proof of Concept

```solidity
// Local fork test (no mainnet)
function testUpdateRateWithZeroReceiver() public {
    // Deploy without calling updateRateReceiver()
    RSETHRateProvider provider = new RSETHRateProvider(
        address(mockOracle),   // returns a nonzero rsETHPrice
        dstChainId,
        address(mockLZEndpoint)
    );

    // rateReceiver is address(0) at this point
    assertEq(provider.rateReceiver(), address(0));

    // Anyone can call updateRate() — ETH is consumed
    uint256 fee = 0.01 ether;
    vm.deal(attacker, fee);
    vm.prank(attacker);
    provider.updateRate{value: fee}();

    // LZ endpoint received a send() call with address(0) as destination
    // (assert via mockLZEndpoint.lastRemotePath() encodes address(0))
    bytes memory path = mockLZEndpoint.lastRemotePath();
    address encodedReceiver;
    assembly { encodedReceiver := mload(add(path, 20)) }
    assertEq(encodedReceiver, address(0));

    // Legitimate receiver rate is still 0
    assertEq(rateReceiver.rate(), 0);
}
```
