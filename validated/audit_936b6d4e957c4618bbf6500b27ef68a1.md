Looking at the sequencer codebase, I need to find an analog to "missing validation of critical parameters in constructors/initialization." The bug class maps to: **critical fields in `ProposalInit` accepted without validation, leading to wrong block execution state**.

Let me trace the exact path.

In `is_proposal_init_valid`, the following fields are validated: `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields (via `within_margin`), and `fee_proposal_fri`. Gas prices are additionally enforced non-zero via `NonzeroGasPrice::new()` in `convert_to_sn_api_block_info`.

The `builder` field — which becomes `sequencer_address` in `BlockInfo` — is **never validated**.

The default config sets `builder_address: ContractAddress::default()` = `0x0`.

---

### Title
`ProposalInit.builder` Accepted Without Zero-Address Validation, Corrupting Sequencer Address in Block Execution — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary
`is_proposal_init_valid` validates every economically significant field of `ProposalInit` except `builder`. The `builder` field is passed directly as `sequencer_address` into `BlockInfo` and from there into block execution, the block hash, and the `get_sequencer_address()` syscall. The default static config sets `builder_address` to `ContractAddress::default()` (`0x0`). No guard in the validation path rejects a zero or otherwise invalid sequencer address, so any proposer node running with default configuration silently commits blocks with `sequencer_address = 0x0`.

### Finding Description

`is_proposal_init_valid` in `crates/apollo_consensus_orchestrator/src/validate_proposal.rs` performs the following checks: [1](#0-0) 

It checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`. The `builder` field is absent from every check.

`convert_to_sn_api_block_info` then maps `init.builder` directly to `sequencer_address`: [2](#0-1) 

Gas prices are protected by `NonzeroGasPrice::new()` which returns an error on zero, propagating rejection. No analogous guard exists for `builder`.

### Impact Explanation

When a proposer node runs with default config (`builder_address = 0x0`) or deliberately sets `builder` to zero:

1. **Wrong fee accounting (Critical):** `sequencer_address = 0x0` in `BlockInfo` means all transaction fees collected during block execution are credited to address `0x0` — effectively burned. This is a direct economic loss for the sequencer operator.

2. **Wrong syscall result (Critical/High):** Every contract that calls `get_sequencer_address()` during that block receives `0x0` instead of the real sequencer address. Contracts that gate logic on the sequencer address (e.g., fee-market contracts, privileged admin checks) will behave incorrectly.

3. **Wrong block hash (Critical):** `PartialBlockHashComponents.sequencer` is `0x0`, so the `ProposalCommitment` / `PartialBlockHash` is computed over a wrong sequencer field. The block hash committed to consensus and eventually to L1 is permanently wrong for that block.

### Likelihood Explanation

The default `builder_address` is `0x0`. Any operator who deploys a sequencer node without explicitly overriding `static_config.builder_address` will propose blocks with zero sequencer address. Because `is_proposal_init_valid` never checks `builder`, all validators will accept and finalize such proposals. This is a misconfiguration that requires no malicious intent and no special privilege beyond being selected as proposer in the normal Tendermint rotation.

### Recommendation

1. Add a non-zero check for `init.builder` inside `is_proposal_init_valid`, analogous to the `NonzeroGasPrice::new()` guards already applied to gas prices:
   ```rust
   if init.builder == ContractAddress::default() {
       return Err(ValidateProposalError::InvalidProposalInit(
           init_proposed.clone(),
           proposal_init_validation.clone(),
           "builder address must not be zero".to_string(),
       ));
   }
   ```
2. Add a `#[validate]` attribute or custom validator on `ContextStaticConfig.builder_address` that rejects `0x0` at config-load time, so misconfigured nodes fail fast before ever proposing.

### Proof of Concept

1. Node A starts with default config; `builder_address = ContractAddress(0x0)`.
2. Tendermint selects Node A as proposer for height H, round 0.
3. `initiate_build` constructs `ProposalInit { builder: ContractAddress(0x0), ... }` and broadcasts it.
4. Validator Node B receives the proposal; `is_proposal_init_valid` runs — no check on `builder`, returns `Ok(())`.
5. `initiate_validation` calls `convert_to_sn_api_block_info`, producing `BlockInfo { sequencer_address: 0x0, ... }`.
6. Batcher executes the block: all fees credited to `0x0`; `get_sequencer_address()` returns `0x0`.
7. `PartialBlockHashComponents { sequencer: SequencerContractAddress(0x0), ... }` is hashed into `ProposalCommitment`.
8. Consensus finalizes the block; the wrong sequencer address is permanently committed to the block hash and L1 state update.

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
