### Title
`dl_verify_proof` Accepts Any Coin With a Matching Puzzle Hash, Enabling Forged DL Proof Responses — (`chia/data_layer/data_layer_util.py`)

### Summary
`dl_verify_proof` verifies only that a coin with the supplied `coin_id` exists on-chain and that its `puzzle_hash` equals the expected DL singleton puzzle hash. It performs no singleton lineage check (odd amount, launcher parent, or lineage proof). An attacker who can create any coin with a DL-shaped puzzle hash — which is trivially possible by spending any coin to create an output with the computed `puzzle_hash` — can obtain a `VerifyProofResponse(success=True)` for entirely fabricated store contents.

`create_host_fullpuz` is a pure deterministic function — it curries `SINGLETON_TOP_LAYER_MOD` with a struct built from the attacker-chosen `genesis_id` (store_id) and `SINGLETON_LAUNCHER_PUZZLE_HASH`, then wraps the DL layer: [3](#0-2) 

Because the puzzle hash is fully determined by attacker-controlled inputs, the attacker can pre-compute it and create a coin with that exact `puzzle_hash` at any amount (including even amounts, which are not valid singletons). No lineage check, no odd-amount check, and no launcher-parent check is ever performed.

By contrast, the wallet's own DL tracking code in `match_dl_launcher` explicitly enforces `amount % 2 == 0` rejection and full puzzle hash verification against the launcher spend: [4](#0-3) 

That guard is entirely absent from `dl_verify_proof`.

### Impact Explanation

An attacker can:
1. Choose arbitrary `inner_puz_hash`, `root`, `store_id`.
2. Compute `target_ph = create_host_fullpuz(inner_puz_hash, root, store_id).get_tree_hash_precalc(inner_puz_hash)`.
3. Spend any coin to create an output with `puzzle_hash = target_ph` (any amount, even amount — not a singleton).
4. Build a valid Merkle tree for chosen key-value pairs under `root`.
5. Submit `DLProof(coin_id=<that coin>, inner_puzzle_hash=inner_puz_hash, store_proofs=StoreProofsHashes(store_id=store_id, proofs=[valid_proof]))` to either the `/dl_verify_proof` wallet RPC or the `/verify_proof` data layer RPC.
6. Receive `VerifyProofResponse(success=True, current_root=True, verified_clvm_hashes=<attacker-chosen k/v>)`.

This allows fraudulent DL proof claims: any relying party (e.g., a DL-offer counterparty, an application verifying store membership before releasing funds or access) is deceived into believing attacker-chosen key-value pairs are committed in a legitimate DL singleton store. This is a corruption of Data Layer root/store state integrity with direct security impact on any system that uses `dl_verify_proof` as a trust anchor.

### Likelihood Explanation

Creating a coin with an arbitrary `puzzle_hash` is a standard Chia operation requiring only a small amount of XCH. The computation of the target puzzle hash is straightforward using the public `create_host_fullpuz` function. The attack requires no privileged access, no leaked keys, and no cryptographic breaks. Any party that can submit an RPC request can execute this.

### Recommendation

After fetching `coin_states[0]`, add the following checks before calling `dl_verify_proof_internal`:

1. **Odd amount**: `coin_states[0].coin.amount % 2 != 0` — enforces the singleton invariant.
2. **Singleton lineage**: Fetch the parent coin spend and verify the parent is either the singleton launcher (`puzzle_hash == SINGLETON_LAUNCHER_PUZZLE_HASH`) or a prior DL singleton (using `match_dl_singleton`). This mirrors the logic already present in `DataLayerWallet.match_dl_launcher` and `singleton_removed`.
3. **Launcher existence**: Confirm the launcher coin (identified by `store_proofs.store_id`) was spent as a valid DL launcher, using `get_launcher_coin_state` logic already in `DataLayerWallet`.

```python
def dl_verify_proof_internal(dl_proof: DLProof, puzzle_hash: bytes32) -> list[KeyValueHashes]:
    """Verify a proof of inclusion for a DL singleton"""

    verified_keys: list[KeyValueHashes] = []

    for reference_proof in dl_proof.store_proofs.proofs:
        inner_puz_hash = dl_proof.inner_puzzle_hash
        host_fullpuz_program = create_host_fullpuz(
            inner_puz_hash, reference_proof.root(), dl_proof.store_proofs.store_id
        )
        expected_puzzle_hash = host_fullpuz_program.get_tree_hash_precalc(inner_puz_hash)

        if puzzle_hash != expected_puzzle_hash:
            raise ProofIntegrityError(
                "Invalid Proof: incorrect puzzle hash: expected:"
                f"{expected_puzzle_hash.hex()} received: {puzzle_hash.hex()}"
            )

        proof = ProofOfInclusion(
            node_hash=reference_proof.node_hash,
            layers=[
                ProofOfInclusionLayer(
                    other_hash_side=Side(layer.other_hash_side),
                    other_hash=layer.other_hash,
                    combined_hash=layer.combined_hash,
                )
                for layer in reference_proof.layers
            ],
        )

        leaf_hash = internal_hash(left_hash=reference_proof.key_clvm_hash, right_hash=reference_proof.value_clvm_hash)
        if leaf_hash != proof.node_hash:
            raise ProofIntegrityError("Invalid Proof: node hash does not match key and value")

        if not proof.valid():
            raise ProofIntegrityError("Invalid Proof: invalid proof of inclusion found")

        verified_keys.append(
            KeyValueHashes(key_clvm_hash=reference_proof.key_clvm_hash, value_clvm_hash=reference_proof.value_clvm_hash)
        )

    return verified_keys
```

```python
async def dl_verify_proof(
    dlproof: DLProof,
    wallet_node: WalletNode,
    peer: WSChiaConnection,
) -> VerifyProofResponse:
    """Verify a proof of inclusion for a DL singleton"""

    coin_id = dlproof.coin_id
    coin_states = await wallet_node.get_coin_state([coin_id], peer=peer)
    if len(coin_states) == 0:
        raise ProofIntegrityError(f"Invalid Proof: No DL singleton found at coin id: {coin_id.hex()}")

    verified_keys = dl_verify_proof_internal(dlproof, coin_states[0].coin.puzzle_hash)

    response = VerifyProofResponse(
        verified_clvm_hashes=ProofResultInclusions(dlproof.store_proofs.store_id, verified_keys),
        success=True,
        current_root=coin_states[0].spent_height is None,
    )
    return response
```
