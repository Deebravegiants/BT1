### Title
`is_proposal_init_valid` validates FRI and WEI gas prices independently without checking their mutual consistency (implied ETH/STRK rate) — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` checks each of the four L1 gas price fields (`l1_gas_price_fri`, `l1_data_gas_price_fri`, `l1_gas_price_wei`, `l1_data_gas_price_wei`) independently against the validator's own reference values, but never verifies that the ratio `proposed_fri / proposed_wei` is consistent with the oracle's ETH/STRK rate. A malicious proposer can set FRI prices at the top of the allowed margin and WEI prices at the bottom (or vice versa), embedding a systematically skewed implied ETH/STRK conversion rate into every accepted block. This rate is then used to derive `l2_gas_price_wei` in `convert_to_sn_api_block_info`, and the raw WEI prices are passed directly into the blockifier's `eth_gas_prices`, causing wrong ETH-denominated fee accounting for all transactions in the block.

### Finding Description

The validator's reference FRI and WEI prices are always derived from a single consistent oracle call in `get_l1_prices_in_fri_and_wei`:

```
prices_in_fri = convert_from_wei(prices_in_wei, eth_to_fri_rate)
```

So the validator's own reference values satisfy `fri_ref / wei_ref ≈ eth_to_fri_rate`. However, the check in `is_proposal_init_valid` only enforces:

```rust
within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, margin)
&& within_margin(l1_data_gas_price_fri_proposed, l1_data_gas_price_fri, margin)
&& within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, margin)
&& within_margin(l1_data_gas_price_wei_proposed, l1_data_gas_price_wei, margin)
```

Each field is checked against its own reference independently. There is no check that `proposed_fri / proposed_wei ≈ eth_to_fri_rate`. A proposer can therefore pass all four checks while embedding an implied rate of:

```
implied_rate = proposed_fri * WEI_PER_ETH / proposed_wei
             = (fri_ref * (1+m)) * WEI_PER_ETH / (wei_ref * (1-m))
             = true_rate * (1+m)/(1-m)
```

With the 10% margin (`l1_gas_price_margin_percent`), this gives an implied rate up to ~22% above or below the true oracle rate.

This skewed implied rate is then used in `convert_to_sn_api_block_info` to derive `l2_gas_price_wei`:

```rust
let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;
// eth_to_fri_rate = l1_gas_price_fri * WEI_PER_ETH / l1_gas_price_wei  (proposer-controlled)
let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)?;
```

Additionally, the proposer-supplied `l1_gas_price_wei` and `l1_data_gas_price_wei` are passed directly into `eth_gas_prices` in the `BlockInfo` given to the blockifier, so ETH-denominated L1 gas fees are also affected.

### Impact Explanation

**Impact: High — wrong fee/gas accounting with economic impact.**

- `eth_gas_prices.l2_gas_price` (used for ETH-denominated L2 gas fees) is derived from the inconsistent implied rate and can be up to ~22% wrong.
- `eth_gas_prices.l1_gas_price` and `eth_gas_prices.l1_data_gas_price` are the proposer-supplied WEI values directly, which can be set to the bottom of the margin while FRI prices are at the top.
- A malicious proposer can systematically undercharge ETH-paying users and overcharge STRK-paying users (or vice versa) every block, creating a persistent economic imbalance.
- The wrong `l2_gas_price_wei` is committed into the block's `BlockInfo` and used for all fee calculations in that block.

### Likelihood Explanation

**Likelihood: Medium.**

- Requires a legitimate consensus proposer (validator role).
- The manipulation is bounded by `l1_gas_price_margin_percent` (currently 10%), giving a maximum implied-rate skew of ~22%.
- No external trigger is needed beyond the proposer crafting a `ProposalInit` with FRI and WEI prices at opposite ends of their respective margins.
- The manipulation can be applied every block the attacker is proposer, making it systematic rather than one-off.

### Recommendation

After the four independent margin checks pass, add a cross-check that the implied ETH/STRK rate derived from the proposed prices is within margin of the validator's own oracle rate:

```rust
// After the four within_margin checks:
let proposed_implied_rate = calculate_eth_to_fri_rate(&PreviousProposalInitInfo {
    l1_prices_fri: L1PricesInFri { l1_gas_price: l1_gas_price_fri_proposed, .. },
    l1_prices_wei: L1PricesInWei { l1_gas_price: l1_gas_price_wei_proposed, .. },
    ..
})?;
let reference_rate = calculate_eth_to_fri_rate_from(l1_gas_price_fri, l1_gas_price_wei);
if !within_margin(proposed_implied_rate, reference_rate, l1_gas_price_margin_percent) {
    return Err(InvalidProposalInit(..., "Implied ETH/STRK rate out of margin"));
}
```

Alternatively, derive the expected WEI prices from the proposed FRI prices using the validator's own oracle rate, and check those derived values instead of the raw proposed WEI values.

### Proof of Concept

Let `margin = 10%`, `true_rate = 1000 STRK/ETH`, `l1_gas_price_fri_ref = 1000`, `l1_gas_price_wei_ref = 1`.

A malicious proposer submits:
- `l1_gas_price_fri = 1100` (top of 10% margin → passes `within_margin`)
- `l1_gas_price_wei = 0.9` → rounded to `1` (bottom of margin → passes `within_margin`)

In `convert_to_sn_api_block_info`:
```
eth_to_fri_rate = 1100 * WEI_PER_ETH / 0.9 ≈ 1222 * WEI_PER_ETH / 1
```
(~22% above true rate)

```
l2_gas_price_wei = l2_gas_price_fri / 1222  ← 18% lower than correct value
```

All ETH-denominated L2 gas fees in this block are 18% lower than they should be. The proposer can repeat this every block they are selected, systematically subsidizing ETH-paying users at the protocol's expense.

fn calculate_eth_to_fri_rate(
    proposal_init_info: &PreviousProposalInitInfo,
) -> Result<u128, StarknetApiError> {
    let eth_to_fri_rate = proposal_init_info
        .l1_prices_fri
        .l1_gas_price
        .0
        .checked_mul(WEI_PER_ETH)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Fri should be small enough to multiply by WEI_PER_ETH. Previous \
                 proposal init info: {:?}",
                proposal_init_info
            ))
        })?
        .checked_div(proposal_init_info.l1_prices_wei.l1_gas_price.0)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Wei should be non-zero. Previous proposal init info: {:?}",
                proposal_init_info
            ))
        })?;
    if eth_to_fri_rate == 0 {
        return Err(StarknetApiError::GasPriceConversionError(format!(
            "Eth to fri rate is zero. Previous proposal init info: {:?}",
            proposal_init_info
        )));
    }
    Ok(eth_to_fri_rate)
