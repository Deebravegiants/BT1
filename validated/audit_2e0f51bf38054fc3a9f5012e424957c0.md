### Title
Missing Oracle Answer Validation Allows Zero/Negative Price Propagation - (`contracts/oracles/RSETHPriceFeed.sol`)

### Summary
`RSETHPriceFeed.latestRoundData()` and `getRoundData()` perform no validation on the `answer` returned by the upstream `ETH_TO_USD` Chainlink feed. If that feed returns `answer ≤ 0`, the contract silently computes and returns a zero or negative RSETH/USD price, violating the invariant expected by downstream consumers.

### Finding Description
The entire body of `latestRoundData()` is:

```solidity
// contracts/oracles/RSETHPriceFeed.sol, lines 63-70
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
``` [1](#0-0) 

There is no guard of the form `require(answer > 0, "invalid ETH/USD price")` before the multiplication. The identical omission exists in `getRoundData()`: [2](#0-1) 

When `ETH_TO_USD.latestRoundData()` returns `answer = 0` (e.g., an uninitialized or deprecated aggregator, or a Chainlink feed that has not yet received its first answer), the arithmetic `int256(rsETHPrice) * 0 / 1e18 = 0` is computed and returned without any revert. When `answer < 0` (possible if the aggregator's stored value is negative due to a bug or misconfiguration), the result is also negative. Neither case triggers a revert.

Missing validations present in the contract:
- No `answer > 0` check on the ETH/USD feed response
- No staleness check (`updatedAt == 0` or `block.timestamp - updatedAt > threshold`)
- No `answeredInRound >= roundId` check

### Impact Explanation
Any downstream protocol (Morpho, Aave, Compound-style markets) that consumes `RSETHPriceFeed` as a Chainlink-compatible oracle and enforces `require(answer > 0)` on the returned price will treat `answer = 0` as a broken feed sentinel. This causes all fund-access paths — borrows, withdrawals, liquidations — to revert, freezing user funds in those markets for as long as the bad price persists. The duration depends on how quickly the upstream feed recovers or governance intervenes; in a worst-case deprecation scenario this can be indefinite.

### Likelihood Explanation
Chainlink aggregators can return `answer = 0` in several realistic, non-adversarial scenarios:
- A newly deployed or not-yet-seeded aggregator
- A deprecated feed whose proxy is pointed at a zeroed-out implementation
- An aggregator whose `minAnswer`/`maxAnswer` circuit breaker is set to 0 by misconfiguration

No privileged actor or key compromise is required; any caller of `latestRoundData()` observes the propagated zero.

### Recommendation
Add explicit validation immediately after fetching the ETH/USD answer:

```solidity
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
require(answer > 0, "RSETHPriceFeed: invalid ETH/USD answer");
require(updatedAt != 0 && block.timestamp - updatedAt <= MAX_STALENESS, "RSETHPriceFeed: stale ETH/USD price");
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
require(answer > 0, "RSETHPriceFeed: non-positive RSETH/USD price");
```

Apply the same guards in `getRoundData()`.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";
import "../contracts/oracles/RSETHPriceFeed.sol";

contract MockETHUSD {
    function decimals() external pure returns (uint8) { return 8; }
    function description() external pure returns (string memory) { return "ETH/USD"; }
    function version() external pure returns (uint256) { return 1; }
    function latestRoundData() external pure returns (
        uint80, int256, uint256, uint256, uint80
    ) {
        // Simulate circuit-breaker / uninitialized aggregator returning 0
        return (1, 0, block.timestamp, block.timestamp, 1);
    }
    function getRoundData(uint80 r) external pure returns (
        uint80, int256, uint256, uint256, uint80
    ) { return (r, 0, 0, 0, r); }
}

contract MockRSETHOracle {
    function rsETHPrice() external pure returns (uint256) {
        return 1.05e18; // 1.05 ETH per rsETH
    }
}

contract RSETHPriceFeedTest is Test {
    RSETHPriceFeed feed;

    function setUp() public {
        feed = new RSETHPriceFeed(
            address(new MockETHUSD()),
            address(new MockRSETHOracle()),
            "RSETH/USD"
        );
    }

    function test_zeroPricePropagates() public {
        (, int256 answer,,,) = feed.latestRoundData();
        // Asserts answer == 0 with NO revert — invariant broken
        assertEq(answer, 0);
    }
}
```

Running this test against the unmodified contract passes, confirming that a zero ETH/USD answer propagates silently through `RSETHPriceFeed.latestRoundData()`. [3](#0-2)
