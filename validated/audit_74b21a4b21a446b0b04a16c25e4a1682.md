### Title
`handle_spice_contract_code_response()` Accepts Unauthenticated Contract Code Responses from Any Validator Peer — (File: chain/client/src/spice/chunk_validator_actor.rs)

### Summary
`handle_spice_contract_code_response()` in the SPICE chunk validator actor processes `SpiceContractCodeResponse` messages without verifying the sender's identity or any cryptographic signature. The `SpiceContractCodeResponse` type only has a V1 variant that carries no signature field, and the handler performs no sender check. Any validator peer can send a `SpiceContractCodeResponse` and have it accepted, bypassing the invariant that only the specifically requested chunk producer should supply contract code. This is the direct nearcore analog of the `atlasSolverCall()` bug: a function that should only be triggered by a trusted source instead accepts data from any caller.

### Finding Description

The function performs no sender identity check and no signature verification. The `SpiceContractCodeResponse` type only has a V1 variant with no signature field:

This contrasts sharply with the non-SPICE `ContractCodeResponse`, which added a signed V2 variant (`SignedContractCodeResponse` protocol feature) and `validate_witness_contract_code_response_signature()` precisely to authenticate the responder: [3](#0-2) [4](#0-3) 

The SPICE code path never received this authentication upgrade. Additionally, the `chunk_id` field inside the response is never used to scope which pending chunks are resolved — the handler iterates over **all** pending chunks and resolves any hash that matches, regardless of which chunk the response claims to be for. [5](#0-4) 

The request side (`handle_spice_contract_code_request`) does verify the requester's signature and role: [6](#0-5) 

But the response side has no equivalent check.

### Impact Explanation
A malicious validator peer (any participant with T1 network access) can send a `SpiceContractCodeResponse` containing the correct contract bytes for a pending chunk. Contract bytes are public on-chain data derivable from trie storage. The chunk validator will:

1. Hash the received bytes and match them against `trusted.missing` (the hash check provides integrity but not authenticity).
2. Remove the matched hashes from `trusted.missing` and push the bytes into `entry.contracts`.
3. Trigger `try_assemble_and_validate_chunk`, producing a chunk endorsement based on data supplied by an unauthorized source.

The broken invariant is: **only the chunk producer that was explicitly requested should supply contract code**. The validator's trust model is violated — it accepts and acts on data from any validator peer, not the specifically requested producer. The `chunk_id` in the response is also ignored, so a single spoofed response can resolve pending contracts across multiple unrelated chunks simultaneously.

The concrete corrupted protocol value is the **chunk endorsement decision**: the validator endorses a chunk based on contract code whose provenance has not been authenticated, bypassing the sender-verification boundary that the non-SPICE path enforces.

### Likelihood Explanation
Any validator with T1 network connectivity to a chunk validator can send a `SpiceContractCodeResponse`. Contract bytes are recoverable from on-chain trie storage. The attack requires no special cryptographic material beyond validator network access, which is obtainable by any staked validator. The SPICE feature is gated by `protocol_feature_spice`, so impact is scoped to networks where that feature is enabled.

### Recommendation
Add a signature field to `SpiceContractCodeResponse` (mirroring `ContractCodeResponseV2`) and verify it in `handle_spice_contract_code_response()` before processing, checking that the responder is a chunk producer for the relevant shard and epoch. Additionally, filter resolved contracts by the `chunk_id` carried in the response rather than resolving against all pending chunks indiscriminately.

### Proof of Concept
1. Observe a chunk validator waiting for contract code: it has sent a `SpiceContractCodeRequest` to chunk producer P for chunk C with missing hash H.
2. As any other validator V (not P), retrieve the contract bytes B where `hash(B) == H` from on-chain trie storage.
3. Construct `SpiceContractCodeResponse::V1(SpiceContractCodeResponseV1 { chunk_id: C, compressed_contracts: encode([B]) })`.
4. Send this message to the chunk validator over the T1 network connection.
5. `handle_spice_contract_code_response()` decompresses B, computes `CodeHash(hash(B)) == H`, finds H in `trusted.missing`, removes it, pushes B into `entry.contracts`, and calls `try_assemble_and_validate_chunk`.
6. The chunk validator produces an endorsement for chunk C based on contract code supplied by unauthorized validator V — no signature or sender check was performed at any point. [7](#0-6) [8](#0-7)

    fn handle_spice_contract_code_response(
        &mut self,
        response: SpiceContractCodeResponse,
    ) -> Result<(), Error> {
        let contracts = response.decompress_contracts()?;

        // Map code_hash -> bytes for the received contracts.
        let mut received: HashMap<CodeHash, CodeBytes> = HashMap::new();
        for contract in contracts {
            let hash = CodeHash(hash(&contract.0));
            received.insert(hash, contract);
        }

        // Resolve across all pending chunks that are waiting on any of these contracts.
        let mut maybe_ready: Vec<SpiceChunkId> = Vec::new();
        for (chunk_id, entry) in &mut self.partial_chunk_data {
            let Some(trusted) = &mut entry.trusted else {
                continue;
            };

            let mut resolved_any = false;
            for (hash, bytes) in &received {
                if trusted.missing.remove(hash) {
                    entry.contracts.push(bytes.clone());
                    resolved_any = true;
                }
            }
            if resolved_any && trusted.missing.is_empty() {
                maybe_ready.push(chunk_id.clone());
            }
        }

        let signer = self
            .validator_signer
            .get()
            .ok_or_else(|| Error::NotAValidator("no signer".to_owned()))?;
        for chunk_id in maybe_ready {
            self.try_assemble_and_validate_chunk(&chunk_id, signer.clone())?;
        }
        Ok(())
    }

pub enum SpiceContractCodeResponse {
    V1(SpiceContractCodeResponseV1) = 0,
}

impl SpiceContractCodeResponse {
    pub fn encode(chunk_id: SpiceChunkId, contracts: &Vec<CodeBytes>) -> std::io::Result<Self> {
        SpiceContractCodeResponseV1::encode(chunk_id, contracts).map(|v1| Self::V1(v1))
    }

    pub fn chunk_id(&self) -> &SpiceChunkId {
        match self {
            Self::V1(v1) => &v1.chunk_id,
        }
    }

    pub fn decompress_contracts(&self) -> std::io::Result<Vec<CodeBytes>> {
        let compressed_contracts = match self {
            Self::V1(v1) => &v1.compressed_contracts,
        };
        compressed_contracts.decode().map(|(data, _size)| data)
    }
}

        let (compressed_contracts, _size) = CompressedContractCode::encode(contracts)?;
        Ok(Self { chunk_id, compressed_contracts })
    }
