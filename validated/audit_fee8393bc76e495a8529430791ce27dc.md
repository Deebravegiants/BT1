### Title
Stale Governance Threshold in `vote_update()` Enables Malicious Contract Upgrade via Resharing-Induced Threshold Reduction — (File: `crates/contract/src/lib.rs`)

---

### Summary

`vote_update()` evaluates the *current* governance threshold at execution time without invalidating pending proposals when resharing reduces that threshold. Because `process_new_parameters_proposal()` requires only unanimous consent from the *proposed* participant set (not threshold consent from the *current* set), a coalition of fewer-than-threshold malicious participants can (1) vote for a malicious contract upgrade, (2) self-propose a resharing to a smaller set with a lower threshold, and (3) after resharing completes, their previously insufficient votes satisfy the new lower threshold, triggering the malicious upgrade.

---

### Finding Description

**Root cause — `vote_update()` uses the live threshold with no epoch/version guard:** [1](#0-0) 

```rust
let threshold = self.threshold()?;          // always the *current* threshold
// …
let valid_votes_count = running_state
    .parameters.participants().participants().iter()
    .filter(|(account_id, _, _)| {
        self.proposed_updates.vote_by_participant
            .get(account_id)
            .is_some_and(|voted_id| *voted_id == id)
    })
    .count();

if (valid_votes_count as u64) < threshold.value() {
    return Ok(false);
}
```

There is no check that the threshold in force when the proposal was created equals the threshold in force at execution time, and no mechanism to invalidate pending proposals when resharing changes the threshold.

**Enabling condition — resharing can be initiated with fewer than threshold votes:**

`process_new_parameters_proposal()` transitions to resharing when the vote count equals the *proposed* participant count, not the *current* threshold: [2](#0-1) 

```rust
let n_votes = self.parameters_votes.vote(proposal, candidate);
Ok(new_num_participants == n_votes)
```

The threshold-gate only applies to *new* (non-current) participants being admitted: [3](#0-2) 

```rust
if AuthenticatedAccountId::new(self.parameters.participants()).is_err() {
    let n_votes = self.parameters_votes
        .n_votes(proposal, self.parameters.participants());
    if n_votes < self.parameters.threshold().value() {
        return Err(VoteError::VoterPending.into());
    }
}
```

If all proposed participants are already current participants, no threshold-sized quorum from the current set is required to start resharing.

**`proposed_updates` survive resharing and are not epoch-gated:**

The post-resharing cleanup only removes votes from accounts that are *no longer* participants: [4](#0-3) 

Proposals created in epoch N remain executable in epoch N+1 with no epoch check.

---

### Impact Explanation

A successful exploit delivers an **unauthorized contract upgrade**. The attacker-controlled WASM replaces the live MPC contract, enabling:

- Arbitrary redirection of `respond()` to accept forged signatures without threshold participation — **unauthorized threshold signature issuance**.
- Draining of NEAR deposits held in pending yield callbacks — **direct loss of funds**.
- Permanent corruption of the `protocol_state`, freezing all future signing — **permanent freezing of funds**.

This matches the *Critical* impact tier: unauthorized transaction execution / threshold signature issuance without required participant authorization.

---

### Likelihood Explanation

**Attacker profile:** Byzantine participants strictly below the signing threshold. With a 7-participant, threshold-5 network, only 3 colluding nodes are required — well below the threshold.

**Preconditions:**
1. The 3 malicious nodes must hold valid TEE attestations (they are running nodes, so this is satisfied by construction).
2. They must be able to call `propose_update`, `vote_update`, and `vote_new_parameters` — all are unprivileged participant calls.
3. They must complete the resharing key ceremony among themselves — feasible since they control all 3 nodes in the proposed set.

No external oracle, leaked key, or network-level DoS is required.

---

### Recommendation

Mirror the fix described in the external report: **invalidate all pending `vote_update` proposals whenever resharing changes the governance threshold or participant set.** Concretely:

1. Record the `epoch_id` at the time each update is proposed (store it alongside the `UpdateEntry`).
2. In `vote_update()`, reject execution if the stored epoch differs from the current epoch:
   ```rust
   if proposal_epoch != current_epoch_id {
       return Err(InvalidParameters::StaleProposal.into());
   }
   ```
3. Alternatively, clear `proposed_updates` entirely in `vote_reshared()` when the new `RunningContractState` is constructed, rather than relying on the best-effort cleanup promise.

---

### Proof of Concept

**Setup:** 7 participants `{P1…P7}`, governance threshold `T = 5`.

1. **P1, P2, P3** call `propose_update` with malicious WASM → `UpdateId = 42`.
2. **P1, P2, P3** each call `vote_update(42)` → `valid_votes_count = 3 < 5`; returns `false`. Votes are stored in `proposed_updates.vote_by_participant`.
3. **P1, P2, P3** call `vote_new_parameters` proposing `{P1, P2, P3}` with `threshold = 2`, `prospective_epoch_id = current + 1`. All three are current participants, so each vote is accepted immediately. After the third vote, `n_votes (3) == new_num_participants (3)` → resharing starts.
4. P1, P2, P3 complete the key resharing ceremony and call `vote_reshared`. Contract transitions to `Running` with `participants = {P1, P2, P3}`, `threshold = 2`. The cleanup promise removes votes only for removed participants (P4–P7); P1–P3 votes for `UpdateId 42` remain.
5. **P1** calls `vote_update(42)`. `self.threshold()` returns `2`. The filter counts P1, P2, P3 as current participants with `voted_id == 42` → `valid_votes_count = 3 ≥ 2`. `do_update` fires the malicious WASM deployment promise. **Upgrade executes.** [5](#0-4) [6](#0-5)

    pub fn vote_update(&mut self, id: UpdateId) -> Result<bool, Error> {
        log!(
            "vote_update: signer={}, id={:?}",
            env::signer_account_id(),
            id,
        );

        let ProtocolContractState::Running(running_state) = &self.protocol_state else {
            env::panic_str("protocol must be in running state");
        };

        let threshold = self.threshold()?;

        let voter = self.voter_or_panic();
        if self.proposed_updates.vote(&id, voter).is_none() {
            return Err(InvalidParameters::UpdateNotFound.into());
        }

        // Filter votes to only count current participants voting for this specific update.
        // This ensures correctness even if the cleanup promise in MpcContract::vote_reshared() fails.
        let valid_votes_count = running_state
            .parameters
            .participants()
            .participants()
            .iter()
            .filter(|(account_id, _, _)| {
                self.proposed_updates
                    .vote_by_participant
                    .get(account_id)
                    .is_some_and(|voted_id| *voted_id == id)
            })
            .count();

        // Not enough votes from current participants, wait for more.
        if (valid_votes_count as u64) < threshold.value() {
            return Ok(false);
        }

        let update_gas_deposit = Gas::from_tgas(self.config.contract_upgrade_deposit_tera_gas);

        let Some(_promise) = self.proposed_updates.do_update(&id, update_gas_deposit) else {
            return Err(InvalidParameters::UpdateNotFound.into());
        };

        Ok(true)
    }

    /// Casts a vote for `proposal`, removing any previous votes by `env::signer_account_id()`.
    /// Fails if the proposal is invalid or the signer is not a proposed participant.
    /// Returns true if all participants of the proposed parameters voted for it.
    pub(super) fn process_new_parameters_proposal(
        &mut self,
        proposal: &ProposedThresholdParameters,
    ) -> Result<bool, Error> {
        // ensure the proposal is valid against the current parameters
        self.parameters
            .validate_incoming_proposal(proposal.parameters())?;

        // Validate effective per-domain thresholds (updates override, absent
        // domains keep theirs) against the proposed participant count.
        let new_num_participants =
            u64::try_from(proposal.participants().len()).expect("participant count fits in u64");
        let threshold_updates = proposal.per_domain_thresholds();
        // Reject unknown domain IDs: the loop below iterates existing domains, so
        // an unknown ID would otherwise be silently ignored here (it's caught at
        // the resharing transition, but we fail fast at vote acceptance).
        for id in threshold_updates.keys() {
            if self.domains.get_domain_by_domain_id(*id).is_none() {
                return Err(DomainError::UnknownDomainInProposal { domain_id: *id }.into());
            }
        }
        let effective_domains: Vec<DomainConfig> = self
            .domains
            .domains()
            .iter()
            .map(|domain| {
                let effective_threshold = threshold_updates
                    .get(&domain.id)
                    .copied()
                    .unwrap_or(domain.reconstruction_threshold);
                DomainConfig {
                    reconstruction_threshold: effective_threshold,
                    ..domain.clone()
                }
            })
            .collect();
        for domain in &effective_domains {
            validate_domain_threshold(domain, new_num_participants)?;
        }

        // The GovernanceThreshold must dominate every domain's effective ReconstructionThreshold;
        // enforced here so the state transition is self-contained (single source of truth).
        ThresholdParameters::validate_governance_against_reconstruction(
            new_num_participants,
            proposal.threshold(),
            max_reconstruction_threshold(&effective_domains),
        )?;

        // ensure the signer is a proposed participant
        let candidate = AuthenticatedAccountId::new(proposal.participants())?;

        // If the signer is not a participant of the current epoch, they can only vote after
        // `threshold` participant of the current epoch have casted their vote to admit them.
        if AuthenticatedAccountId::new(self.parameters.participants()).is_err() {
            let n_votes = self
                .parameters_votes
                .n_votes(proposal, self.parameters.participants());
            if n_votes < self.parameters.threshold().value() {
                return Err(VoteError::VoterPending.into());
            }
        }

        // finally, vote.
        let n_votes = self.parameters_votes.vote(proposal, candidate);
        Ok(new_num_participants == n_votes)
