### Title
Unvalidated `ProposalInit.builder` Address Flows Into Block Execution as `sequencer_address` — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates many fields of a proposer-supplied `ProposalInit` (height, `l1_da_mode`, `l2_gas_price_fri`, all L1 gas prices, `starknet_version`, `version_constant_commitment`, `fee_proposal_fri`) but never checks `init.builder`. That field is the block-builder/sequencer address. It is passed verbatim to `convert_to_sn_api_block_info` as `sequencer_address` in the `BlockInfo` used for every transaction executed in the block. A malicious-but-legitimately-scheduled proposer can set `builder` to any address — including zero or an attacker-controlled contract — and every validator will accept the proposal, execute all transactions against the wrong sequencer address, and commit the corrupted block header to state.

### Finding Description

`ProposalInit` carries a `builder: ContractAddress` field, documented as "Address of the one who builds/sequences the block." [1](#0-0) 

`is_proposal_init_valid` constructs a `ProposalInitValidation` that contains `height`, `block_timestamp_window_seconds`, `previous_proposal_init`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, and `fee_actual`. There is no `builder` field and no check against it anywhere in the function. [2](#0-1) 

The full validation logic (lines 260–418) checks timestamp bounds, `starknet_version`, `version_constant_commitment`, the three-field combined check (`height`, `l1_da_mode`, `l2_gas_price_fri`), all four L1 gas prices within margin, and `fee_proposal_fri` bounds — but `init.builder` is never read. [3](#0-2) 

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address` in the `starknet_api::block::BlockInfo` sent to the batcher: [4](#0-3) 

The batcher executes every transaction in the proposal against this `BlockInfo`. The `sequencer_address` is visible to Cairo contracts via `get_execution_info()` and is included in the block header commitment that becomes the canonical block hash. Because both the proposer's batcher and the validator's batcher receive the same `init.builder`, they compute the same `partial_block_hash`, so the `ProposalFinMismatch` guard does not fire. [5](#0-4) 

The corrupted `builder` address is then persisted through `finalize_decision` → `update_state_sync_with_new_block` → `cende_block_info`: [6](#0-5) 

### Impact Explanation

Any contract that reads `get_execution_info().block_info.sequencer_address` during execution (e.g., fee-token contracts, access-control contracts, contracts that route payments to the sequencer) will observe the attacker-chosen address instead of the real sequencer. The block header stored in state and returned by RPC will carry the wrong `sequencer_address`. Fee accounting that credits the sequencer address is misdirected. All of this is committed to the canonical chain with full BFT finality because every honest validator independently re-executes against the same attacker-supplied `builder` value and arrives at the same (wrong) block commitment.

This matches: **Critical — Wrong state/receipt/execution result from blockifier/syscall logic for accepted input** and **High — RPC view returns an authoritative-looking wrong value**.

### Likelihood Explanation

The attack requires the adversary to be the legitimately scheduled proposer for a round (selected by the committee). Any validator reaches this role in rotation. No special admin key or out-of-band privilege is needed beyond holding a validator slot. The proposer simply sets `builder` to an arbitrary `ContractAddress` when constructing `ProposalInit` in `initiate_build`; no other change is required. [7](#0-6) 

### Recommendation

Add `builder: ContractAddress` to `ProposalInitValidation` and check it in `is_proposal_init_valid`, analogous to how `l1_da_mode` and `l2_gas_price_fri` are checked:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

Each validator node knows its own builder address from static configuration. That value should be the sole accepted value for `init.builder` when the local node is the expected proposer, and the known builder address of the scheduled proposer otherwise (sourced from the validator set / staking registry).

### Proof of Concept

1. Validator A is the scheduled proposer for height H, round R.
2. A constructs `ProposalInit` with `builder: ContractAddress::from(0x1337_u64)` (arbitrary attacker address) instead of its real sequencer address.
3. A broadcasts the `ProposalInit` to all peers.
4. Each peer calls `validate_proposal` → `is_proposal_init_valid`. The function checks height, `l1_da_mode`, `l2_gas_price_fri`, gas prices, `starknet_version`, `version_constant_commitment`, `fee_proposal_fri` — all pass because A set them correctly. `builder` is never read.
5. Each peer calls `initiate_validation` → `convert_to_sn_api_block_info(init)` → `sequencer_address = ContractAddress(0x1337)`.
6. The batcher executes all transactions with `sequencer_address = 0x1337`. Contracts calling `get_execution_info()` see `0x1337`. Fee tokens directed to the sequencer go to `0x1337`.
7. The batcher returns `partial_block_hash` computed over the block header containing `sequencer_address = 0x1337`. Both A's batcher and every validator's batcher produce the same hash (same inputs), so `built_block == received_fin.proposal_commitment` passes.
8. The block is finalized and stored with `sequencer_address = 0x1337` in the canonical chain. [8](#0-7) [9](#0-8)

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
