### Title
Unrestricted `claim` Caller Allows Anyone to Force-Claim on Behalf of Any User, Causing Permanent Fee-Based Yield Loss ‚Äî (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

---

### Summary

The `claim` function in `MerkleDistributor.sol` does not enforce that `msg.sender == account`. Any external caller who possesses a valid merkle proof for a target user can trigger that user's claim at any time, causing the victim to permanently lose a portion of their entitled tokens to the protocol fee ‚Äî without their knowledge or consent.

---

### Finding Description

The `claim` function accepts an arbitrary `account` parameter but performs no authorization check binding the caller to that account: [1](#0-0) 

After verifying the merkle proof for `account`, the contract deducts a fee and transfers the remainder to `account`: [2](#0-1) 

The fee is calculated as:

```
fee = claimableAmount * feeInBPS / 10_000
```

and is sent to `protocolTreasury`, not to the caller. The victim receives only `claimableAmount - fee`. Their `lastClaimedIndex` and `cumulativeAmount` are updated, permanently consuming the claim.

By contrast, `KernelMerkleDistributor._processClaim` explicitly guards against this with: [3](#0-2) 

`MerkleDistributor.sol` has no equivalent guard.

---

### Impact Explanation

**High ‚Äî Theft of unclaimed yield.**

With `MAX_FEE_IN_BPS = 1000` (10%), an attacker can force-claim on behalf of a user entitled to 1,000 tokens, causing that user to permanently lose 100 tokens to the treasury. The attacker pays only gas. The loss is irreversible: the claim is marked used and the victim cannot reclaim the fee. This can be applied to every eligible address in the merkle tree in a single block. [4](#0-3) 

---

### Likelihood Explanation

**High.** Merkle trees and their proofs are computed from public off-chain data and are routinely published by protocols. Any external caller can reconstruct valid proofs for any leaf. No privileged role, leaked key, or governance action is required. The attack path is a single public function call with no preconditions beyond knowing the victim's index, address, and cumulative amount ‚Äî all of which are public.

---

### Recommendation

Add an authorization check identical to the one in `KernelMerkleDistributor`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Insert this check at the top of `claim`, before the merkle proof verification. Additionally, add `nonReentrant` to `claim` for defense-in-depth, as it is currently absent unlike in `KernelMerkleDistributor`. [5](#0-4) 

---

### Proof of Concept

```solidity
// Attacker observes Alice's merkle proof from the public tree.
// Alice is entitled to 1000 tokens; feeInBPS = 1000 (10%).

merkleDistributor.claim(aliceIndex, alice, 1000e18, aliceMerkleProof);

// Result:
//   Alice receives: 900e18 tokens (90%)
//   Protocol treasury receives: 100e18 tokens (10%)
//   Alice's claim is permanently consumed ‚Äî she cannot reclaim the 100e18 fee.
//   Attacker cost: gas only.

// Attacker repeats for every address in the merkle tree.
```

The analog to the referral report is direct: just as the referral contract imposed no restriction on *who* could trigger the registration reward loop, `MerkleDistributor` imposes no restriction on *who* can trigger a claim ‚Äî enabling an unprivileged external caller to repeatedly exploit the fee mechanism across all eligible accounts.

```text
    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
