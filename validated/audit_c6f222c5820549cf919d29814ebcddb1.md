Audit Report

## Title
LP Principal Silently Burned to Zero on `removeLiquidity` When Proportional Scaled Balance Falls Below Scale Multiplier — (`metric-core/contracts/libraries/LiquidityLib.sol`)

## Summary

`LiquidityLib.removeLiquidity` computes each bin's proportional scaled amount with floor division, decrements the bin's stored balance and `binTotals`, then converts the total to external units with a second floor division. When the resulting scaled amount is nonzero but less than `token0ScaleMultiplier` (e.g. `1e12` for USDC), the external conversion truncates to zero, no tokens are transferred, yet the LP's shares are fully burned and bin accounting is permanently decremented. The stranded scaled value migrates into the pool's surplus and is later collected as spread fees.

## Finding Description

**Step 1 — per-bin floor division:** [1](#0-0) 

`amount0Scaled = binState.token0BalanceScaled * sharesToRemove / binTotalSharesVal` uses integer floor division. If the result is e.g. `5e11` for a USDC pool where `token0ScaleMultiplier = 1e12`, `amount0Scaled = 5e11`.

**Step 2 — bin state and global accounting decremented, shares burned:** [2](#0-1) 

The bin balance, `binTotalShares`, and `positionBinShares` are all updated before any transfer occurs.

**Step 3 — second floor division in `_deltasScaledToExternal`:** [3](#0-2) [4](#0-3) 

`removeLiquidity` passes `Math.Rounding.Floor`, so `deltaAmount0 = 5e11 / 1e12 = 0`.

**Step 4 — conditional transfer skipped:** [5](#0-4) 

`amount0Removed = 0`, so no transfer occurs. The LP receives nothing despite shares being burned.

**Step 5 — stranded value becomes surplus, collected as fees:** [6](#0-5) 

Because `binTotals.scaledToken0` was decremented but no actual token0 left the contract, `surplus0Scaled` grows by exactly `amount0Scaled`, which is then split between protocol and admin as spread fees.

**Existing guard is insufficient:** [7](#0-6) 

The `minimalMintableLiquidity` guard only reverts when `newUserShares > 0 && newUserShares < minimalMintableLiquidity`. A full exit (`newUserShares = 0`) bypasses it entirely.

**Asymmetry with `addLiquidity`:** The add path uses `Math.Rounding.Ceil` at line 142, ensuring the pool always receives at least 1 external unit when the scaled amount is nonzero. The remove path uses `Math.Rounding.Floor`, creating a structural asymmetry that permanently disadvantages LPs. [8](#0-7) 

## Impact Explanation

Direct, irreversible loss of LP principal up to `TOKEN_0_SCALE_MULTIPLIER − 1` scaled units per `removeLiquidity` call. For USDC (`1e12`), up to just under 1 USDC per call. For WBTC (`1e10`), up to just under 0.01 WBTC (~$1,000 at current prices). The lost principal is not destroyed — it is silently redirected to the protocol/admin fee surplus. This constitutes a direct loss of user principal meeting Sherlock High/Medium thresholds, and a broken core liquidity withdrawal flow.

## Likelihood Explanation

Reachable through normal, unprivileged pool operation: (1) LP adds shares to a bin above current price (token0 only); (2) swaps partially drain the bin's `token0BalanceScaled` to a residual below `token0ScaleMultiplier`; (3) LP calls `removeLiquidity` for all shares — `amount0Scaled` is nonzero but `< token0ScaleMultiplier`, so `amount0Removed = 0`. No privileged access, malicious setup, or non-standard ERC20 behavior is required. The condition is repeatable and worsens with higher-value tokens like WBTC.

## Recommendation

After computing `amount0Scaled` and `amount1Scaled`, add a guard that reverts when the scaled amount is nonzero but would convert to zero external units:

```solidity
if (amount0Scaled > 0 && amount0Scaled < ctx.token0ScaleMultiplier) {
    revert DustRemoval(); // or round up to 1 external unit
}
```

Alternatively, change `_deltasScaledToExternal` for the remove path to use `Math.Rounding.Ceil` (matching the add path), ensuring the LP always receives at least 1 external unit when their scaled entitlement is nonzero. This is the symmetric fix to the intentional `Ceil` used on `addLiquidity`.

## Proof of Concept

```solidity
// USDC pool: token0ScaleMultiplier = 1e12
// 1. LP adds 1000 shares to bin 4 (above current price, token0 only)
//    addLiquidity uses ceilDiv → LP pays e.g. 1 USDC (1e18 scaled)
//    binState.token0BalanceScaled = 1e18, binTotalShares[4] = 1000

// 2. Swaps drain bin 4 token0 down to 5e11 scaled units
//    binState.token0BalanceScaled = 5e11

// 3. LP calls removeLiquidity for all 1000 shares
//    amount0Scaled = 5e11 * 1000 / 1000 = 5e11
//    binState.token0BalanceScaled -= 5e11  → 0
//    binTotals.scaledToken0 -= 5e11
//    _deltasScaledToExternal(5e11, Floor) = 5e11 / 1e12 = 0
//    amount0Removed = 0 → no transfer
//    LP receives 0 USDC, shares burned

// 4. surplus0Scaled increases by 5e11 → collected as fees by protocol/admin
//    assertEq(amount0Removed, 0);          // LP gets nothing
//    assertEq(pool.token0BalanceScaled(bin4), 0); // bin drained
```
