Audit Report

## Title
Unbounded `supportedAssetList` Creates Multiplicative Gas Growth in `updateRSETHPrice()` — (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is a public function whose internal call chain traverses a nested loop over all supported assets, all node delegators, and all EigenLayer queued withdrawals per delegator. The `supportedAssetList` in `LRTConfig` has no enforced maximum length, making the total gas cost grow multiplicatively as the protocol legitimately adds more LST assets. Once the cross-product of assets × NDCs × queued withdrawals × strategies exceeds the block gas limit, every call to `updateRSETHPrice()` reverts, freezing price updates and protocol fee accrual.

## Finding Description

`updateRSETHPrice()` is public with no access control: [1](#0-0) 

It calls `_getTotalEthInProtocol()`, which iterates over every entry in `supportedAssetList` — an array with **no enforced maximum length**: [2](#0-1) 

For each asset, `ILRTDepositPool.getTotalAssetDeposits()` is called, which calls `getAssetDistributionData()`, which loops over the entire `nodeDelegatorQueue`: [3](#0-2) 

For each NDC, `getAssetUnstaking()` is called on `NodeDelegator`, which fetches **all** queued withdrawals from EigenLayer's `DelegationManager` and iterates over strategies per withdrawal with external calls: [4](#0-3) 

The `_addNewSupportedAsset()` internal function pushes to `supportedAssetList` with no length check: [5](#0-4) 

The `nodeDelegatorQueue` is capped at `maxNodeDelegatorLimit` (default 10) and `maxUncompletedWithdrawalCount` is capped at 80. The protocol's own comment explicitly acknowledges the gas sensitivity of this design: [6](#0-5) 

The total gas cost is **O(assets × NDCs × queued_withdrawals × strategies_per_withdrawal)**. With `supportedAssetList` being the only uncapped dimension, each new LST asset added through the legitimate `TIME_LOCK_ROLE` governance path multiplies the gas cost across all NDCs and all their queued withdrawals. `updateRSETHPriceAsManager()` calls the same `_updateRsETHPrice()` and would also revert under the same conditions. [7](#0-6) 

## Impact Explanation

When `updateRSETHPrice()` permanently reverts due to out-of-gas: (1) `rsETHPrice` becomes stale, breaking all downstream price-dependent operations; (2) protocol fee minting inside `_updateRsETHPrice()` halts entirely, constituting permanent freezing of unclaimed yield; (3) the downside price-protection auto-pause mechanism stops functioning. The impact matches **Medium — Unbounded gas consumption** and **Medium — Permanent freezing of unclaimed yield** from the allowed scope.

## Likelihood Explanation

`addNewSupportedAsset()` is gated by `TIME_LOCK_ROLE` — a legitimate governance path, not a malicious attack vector. Kelp DAO's stated protocol goal is to expand supported LSTs. With the existing caps of 10 NDCs and 80 queued withdrawals, even a modest increase to 15–20 supported assets combined with a near-full NDC queue and near-maximum queued withdrawals can push the function past the ~30M block gas limit given the multiple external calls per iteration (ERC20 `balanceOf`, EigenLayer strategy `sharesToUnderlyingView`, etc.). Likelihood is **Low** in isolation but rises as the protocol scales normally.

## Recommendation

1. Enforce an explicit maximum on `supportedAssetList` length inside `_addNewSupportedAsset()` in `LRTConfig.sol`, consistent with the gas budget already calculated for the NDC and withdrawal caps.
2. Cache per-asset total ETH values incrementally (updated on deposit/withdrawal events) rather than recomputing the full sum on every `updateRSETHPrice()` call.
3. Alternatively, split `_getTotalEthInProtocol()` into a paginated or per-asset update pattern so no single transaction must traverse the full cross-product.

## Proof of Concept

Call path triggered by any unprivileged address after legitimate protocol growth:

```
updateRSETHPrice()                          // public, no access control
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ for each asset in supportedAssetList (NO CAP):
                 └─ LRTDepositPool.getTotalAssetDeposits(asset)
                      └─ getAssetDistributionData(asset)
                           └─ for each NDC in nodeDelegatorQueue (≤10):
                                └─ NodeDelegator.getAssetUnstaking(asset)
                                     └─ DelegationManager.getQueuedWithdrawals()  // external call
                                          └─ for each withdrawal (≤80):
                                               └─ for each strategy in withdrawal:
                                                    └─ strategy.sharesToUnderlyingView()  // external call
```

**Foundry fork test plan**: Deploy against a mainnet fork. Use `TIME_LOCK_ROLE` to add 15–20 supported assets. Add 10 NDCs. Queue 80 withdrawals across NDCs via `initiateUnstaking()`. Call `updateRSETHPrice()` and assert it reverts with out-of-gas. Measure gas with `vm.expectRevert()` or `gasleft()` assertions to confirm the block gas limit is exceeded.

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
