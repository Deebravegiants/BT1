### Title
Hardcoded Private Key in Production Consensus Signing Path — (`File: crates/apollo_signature_manager/src/lib.rs`)

### Summary

The production factory function `create_signature_manager()` unconditionally instantiates a `SignatureManager` using `LocalKeyStore::new_for_testing()`, which embeds a fixed, publicly-known ECDSA private key. This key is used to sign consensus **precommit votes** (binding a `BlockHash`) and **peer identity challenges**. Any party with repository access can extract the key and forge valid precommit signatures for arbitrary block hashes, directly corrupting the consensus decision path.

### Finding Description

`crates/apollo_signature_manager/src/lib.rs` exports the public type alias `SignatureManager` and the production factory:

```rust
pub fn create_signature_manager() -> SignatureManager {
    SignatureManager::new()
}
```

`SignatureManager::new()` calls `LocalKeyStore::new_for_testing()`:

```rust
pub(crate) const fn new_for_testing() -> Self {
    const PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
        "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
    ));
    ...
    Self { private_key: PRIVATE_KEY, public_key: PUBLIC_KEY }
}
```

Neither `create_signature_manager` nor `LocalKeyStoreSignatureManager::new` is gated by `#[cfg(test)]`. The private key `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` is therefore compiled into every production binary.

The `SignatureManager` exposes two signing operations:

- `sign_precommit_vote(block_hash)` — signs the consensus precommit message that binds a validator's vote to a specific `BlockHash`.
- `sign_identification(peer_id, challenge)` — signs the peer identity challenge used in P2P authentication.

### Impact Explanation

Because the private key is a compile-time constant embedded in the production binary and visible in the repository, any observer can:

1. Extract `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` from the source.
2. Reconstruct the corresponding `PublicKey` (`0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a`).
3. Call `ecdsa_sign(private_key, build_precommit_vote_message_digest(target_block_hash))` to produce a valid precommit signature for **any** block hash of their choosing.
4. Inject that forged precommit vote into the consensus network, causing peers that trust this validator's public key to accept a vote for a wrong or attacker-chosen block.

This directly corrupts the consensus decision: the wrong block hash is committed, leading to an incorrect state root, wrong receipts/events, and an invalid L1 anchor.

The impact matches: **High — Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload**, and escalates toward **Critical — Wrong state, receipt, event, or block commitment** if the forged vote achieves a quorum. [4](#0-3) [5](#0-4) 

### Likelihood Explanation

The private key is in plain text in a public (or privately shared) repository. No exploit tooling is required — a single `grep` or source read suffices. The only prerequisite is network access to the consensus P2P layer to inject the forged vote, which is a standard peer connection.

### Recommendation

1. **Remove the hardcoded key immediately.** `LocalKeyStore::new_for_testing()` must be restricted to `#[cfg(test)]` and must not be reachable from `create_signature_manager()`.
2. **Implement a production `KeyStore`** that reads the private key from an environment variable, a secrets manager (e.g., GCP Secret Manager, HashiCorp Vault), or an HSM — never from a compile-time constant.
3. **Rotate the exposed key.** Treat `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` as compromised; any validator identity or precommit vote signed with it must be considered forgeable.
4. **Audit git history** for any prior commits that may have contained other plaintext keys (the `testing_secrets.json` pattern of `0x0101...0101` for network `secret_key` fields is a secondary instance of the same class). [6](#0-5) [7](#0-6)

fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash);

    MessageDigest(blake2s_to_felt(&message))
}
