### Title
Stale TEE Attestation State Allows Expired-Attestation Participants to Submit Signatures — (File: crates/contract/src/tee/tee_state.rs)

### Summary
The `is_caller_an_attested_participant()` function does not check whether the caller's stored attestation has expired. Because `verify_tee()` — the only function that performs expiry checks and sets `accept_requests = false` — must be called explicitly by a participant, there is a window between attestation expiry and the next `verify_tee()` invocation during which a participant whose TEE environment may be compromised can still submit signatures via `respond()`, `respond_ckd()`, and `respond_verify_foreign_tx()`.

### Finding Description

`is_caller_an_attested_participant()` in `crates/contract/src/tee/tee_state.rs` checks only four things:

1. The caller is in the participants list
2. A stored attestation exists for the caller's TLS key
3. The attestation's `account_id` matches the signer
4. The attestation's `account_public_key` matches the signer's key [1](#0-0) 

It does **not** call `reverify_participants()` to check whether the attestation has expired. The expiry check lives exclusively in `reverify_and_cleanup_participants()`: [2](#0-1) 

…which is only invoked from `verify_tee()`: [3](#0-2) 

`verify_tee()` must be called explicitly by a participant. Until it is called, `accept_requests` remains `true` and `is_caller_an_attested_participant()` continues to pass for participants with expired attestations.

The `respond()` function calls `assert_caller_is_attested_participant_and_protocol_active()`: [4](#0-3) 

…which calls `is_caller_an_attested_participant()`: [5](#0-4) 

The `accept_requests` flag is checked in `respond()`, but it is only set to `false` by `verify_tee()`. Between attestation expiry and the next `verify_tee()` call, both guards pass for a participant with an expired attestation.

The analog to the external report is direct:

| StakingRouter | NEAR MPC |
|---|---|
| `availableValidatorsCount` not updated until oracle finalizes | `accept_requests` not updated until `verify_tee()` is called |
| Deposits allocated to stuck-validator NO before update | Signatures accepted from expired-attestation participant before update |
| Fix: flag preventing deposits until update finalized | Fix: check expiry inside `is_caller_an_attested_participant()` |

### Impact Explanation

This breaks the production safety invariant that only nodes with valid TEE attestations can participate in the signing process. A participant whose TEE environment may be compromised (expired attestation) can still call `respond()` and have their signature accepted during the window between attestation expiry and `verify_tee()` being called.

For `respond_verify_foreign_tx()`, this is particularly relevant: a compromised node could submit a false verification of a foreign-chain transaction, potentially enabling unauthorized bridge execution or double-spend conditions. The signature submitted to `respond()` must be cryptographically valid (the contract verifies it), but the TEE guarantee — that the node computed it inside a trusted enclave running approved software — is no longer enforced during the stale window.

This matches the allowed impact: **Medium — participant-state manipulation that breaks production safety/accounting invariants without relying on network-level DoS or operator misconfiguration.**

### Likelihood Explanation

`verify_tee()` has no automatic caller; it must be triggered explicitly by a participant. The node's `TeeContext::verify_tee()` submits the call on a periodic cadence, but there is no on-chain enforcement of that cadence. If the calling node is offline, slow, or adversarially delayed, the window can extend indefinitely. Attestation expiry timestamps are configurable and can be short (the test suite uses 5 seconds; production uses hours), but the gap between expiry and the next `verify_tee()` transaction landing on-chain is always nonzero and can be large. [6](#0-5) 

### Recommendation

Check attestation expiry inside `is_caller_an_attested_participant()` by calling `reverify_participants()` on the caller's stored attestation before returning `Ok(())`. This mirrors the check already performed in `reverify_and_cleanup_participants()` and closes the window without requiring a separate `verify_tee()` call.

Alternatively, add an on-chain timestamp guard: record the last `verify_tee()` block timestamp and reject `respond*` calls if it is older than the attestation expiry window.

### Proof of Concept

1. Participant P has a valid attestation that expires at block timestamp T.
2. No participant calls `verify_tee()` between T and T + Δ (e.g., the calling node is temporarily offline).
3. During [T, T + Δ], P's TEE environment may be compromised (certificate expired, enclave potentially running unapproved code).
4. P participates in the off-chain threshold protocol and, as leader, calls `respond(request, signature)` on-chain.
5. `assert_caller_is_attested_participant_and_protocol_active()` → `is_caller_an_attested_participant()` passes: P is in the participants list and has a stored attestation; no expiry check is performed.
6. `accept_requests` is still `true` (only `verify_tee()` sets it to `false`).
7. The cryptographic signature check passes (P computed a valid share using its still-valid key material).
8. The signature is accepted and delivered to the original caller — produced by a node whose TEE validity guarantee has lapsed. [7](#0-6) [8](#0-7) [9](#0-8)

    pub fn reverify_and_cleanup_participants(
        &mut self,
        participants: &Participants,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeValidationResult {
        self.allowed_docker_image_hashes
            .cleanup_expired_hashes(tee_upgrade_deadline_duration);

        let participants_with_valid_attestation: Vec<_> = participants
            .participants()
            .iter()
            .filter(|(_, _, participant_info)| {
                // Use the stored NodeId (keyed by TLS public key) so the real
                // `account_public_key` participates in re-verification. If
                // there is no stored attestation for this TLS key, the
                // participant is invalid.
                let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
                else {
                    return false;
                };

                let tee_status =
                    self.reverify_participants(&node_id, tee_upgrade_deadline_duration);

                matches!(tee_status, TeeQuoteStatus::Valid)
            })
            .cloned()
            .collect();

        if participants_with_valid_attestation.len() != participants.len() {
            let participants_with_valid_attestation =
                Participants::init(participants.next_id(), participants_with_valid_attestation);

            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            }
        } else {
            TeeValidationResult::Full
        }
    }

    /// whose TLS key matches an attested node belonging to the caller account.
    ///
    /// Handles multiple participants per account and supports legacy mock nodes.
    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

    pub fn verify_tee(&mut self) -> Result<bool, Error> {
        log!("verify_tee: signer={}", env::signer_account_id());
        // Caller must be a participant (node or operator).
        self.voter_or_panic();
        let ProtocolContractState::Running(running_state) = &mut self.protocol_state else {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        };
        let current_params = running_state.parameters.clone();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        match self.tee_state.reverify_and_cleanup_participants(
            current_params.participants(),
            tee_upgrade_deadline_duration,
        ) {
            TeeValidationResult::Full => {
                self.accept_requests = true;
                log!("All participants have an accepted Tee status");
                Ok(true)
            }
            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            } => {
                let remaining = participants_with_valid_attestation.len();
                // Defense in depth: the surviving participant set must keep the full
                // threshold relation intact — the GovernanceThreshold must still sit
                // within its bounds for the smaller set (in particular it must not
                // exceed the remaining participant count or the upper cap) and must
                // remain at least every domain's ReconstructionThreshold (the kickout
                // keeps the existing per-domain thresholds). Otherwise we refuse and
                // wait for manual intervention.
                let max_reconstruction_threshold =
                    max_reconstruction_threshold(running_state.domains.domains());
                if let Err(err) = ThresholdParameters::validate_governance_against_reconstruction(
                    u64::try_from(remaining).expect("participant count fits in u64"),
                    current_params.threshold(),
                    max_reconstruction_threshold,
                ) {
                    log!(
                        "Kicking out participants with an invalid TEE status would break the threshold relation ({:?}); {} participants remain with a valid TEE status. This requires manual intervention. We will not accept new signature requests as a safety precaution.",
                        err,
                        remaining,
                    );
                    self.accept_requests = false;
                    return Ok(false);
                }

                // here, we set it to true, because at this point, we have at least `threshold`
                // number of participants with an accepted Tee status.
                self.accept_requests = true;

                // do we want to adjust the threshold?
                //let n_participants_new = new_participants.len();
                //let new_threshold = (3 * n_participants_new + 4) / 5; // minimum 60%
                //let new_threshold = new_threshold.max(2); // but also minimum 2
                let new_threshold = usize::try_from(current_params.threshold().value())
                    .expect("threshold value fits in usize");

                let threshold_parameters = ThresholdParameters::new(
                    participants_with_valid_attestation,
                    Threshold::new(new_threshold as u64),
                )
                .expect("Require valid threshold parameters"); // this should never happen.
                current_params.validate_incoming_proposal(&threshold_parameters)?;
                // This resharing only changes the participant set, so the
                // per-domain reconstruction-threshold updates map is empty.
                let proposed_parameters =
                    ProposedThresholdParameters::new(threshold_parameters, BTreeMap::new());
                let res = running_state.transition_to_resharing_no_checks(&proposed_parameters);
                if let Some(resharing) = res {
                    self.protocol_state = ProtocolContractState::Resharing(resharing);
                }

                Ok(true)
            }
        }
