### Title
Attached NEAR Deposit Permanently Frozen in `submit_participant_info` When Caller Is Existing Participant Re-Submitting Attestation - (File: `crates/contract/src/lib.rs`)

---

### Summary

`submit_participant_info` is `#[payable]` and accepts an attached NEAR deposit, but only refunds excess tokens in the branch where `attestation_storage_must_be_paid_by_caller` is `true`. When an existing participant re-submits their attestation (e.g., to refresh a TEE quote), the condition evaluates to `false` and any attached deposit is silently absorbed by the contract with no refund path.

---

### Finding Description

The function `submit_participant_info` computes a boolean gate:

```rust
let attestation_storage_must_be_paid_by_caller =
    is_new_attestation || caller_is_not_participant;

if attestation_storage_must_be_paid_by_caller {
    // ... storage cost check and excess refund
    if let Some(diff) = attached.checked_sub(cost)
        && diff > NearToken::from_yoctonear(0)
    {
        Promise::new(account_id).transfer(diff).detach();
    }
}
// ← no else branch: deposit is silently kept when condition is false
``` [1](#0-0) 

When `is_new_attestation` is `false` **and** `caller_is_not_participant` is `false` — i.e., an already-registered participant re-submits their attestation — the entire `if` block is skipped. Any NEAR tokens attached to the call are permanently frozen in the contract balance. There is no `else` branch to return `msg.value` (the NEAR deposit) to the caller.

This is structurally identical to the Derby M-2 pattern: a `payable` function that conditionally uses the attached value and silently discards it when the condition is not met.

The scenario is realistic because:
1. The function is `#[payable]`, so the NEAR runtime accepts any deposit.
2. On first submission, a participant **must** attach tokens to cover storage; tooling or scripts that automate re-submissions may continue attaching a deposit.
3. Nodes periodically re-submit attestations to refresh their TEE quote status (e.g., after a TCB update). Each such re-submission with any attached deposit permanently loses those tokens. [2](#0-1) 

---

### Impact Explanation

Any NEAR tokens attached to a re-submission call by an existing participant are permanently frozen in the MPC contract balance. There is no sweep, withdrawal, or governance mechanism to recover them. This breaks the production accounting invariant that a `#[payable]` function must either consume or return the attached deposit.

Matches allowed impact: **Medium** — balance/accounting invariant broken without requiring network-level DoS or operator misconfiguration.

---

### Likelihood Explanation

Existing participants must periodically re-submit attestations to keep their TEE status valid (e.g., after TCB upgrades or attestation expiry). Automated node tooling that always attaches a small deposit (mirroring the first-submission flow) will silently lose funds on every re-submission. The entry path requires no special privilege beyond being an existing participant.

---

### Recommendation

Add an `else` branch that refunds the full attached deposit when storage payment is not required:

```rust
if attestation_storage_must_be_paid_by_caller {
    // existing storage cost check and refund logic
} else {
    let attached = env::attached_deposit();
    if attached > NearToken::from_yoctonear(0) {
        Promise::new(account_id).transfer(attached).detach();
    }
}
```

Alternatively, remove `#[payable]` from `submit_participant_info` entirely and split the function into two entry points — one for new participants (payable, charges storage) and one for re-submissions (non-payable, rejects any deposit at the NEAR runtime level). [3](#0-2) 

---

### Proof of Concept

1. Participant `alice.near` calls `submit_participant_info` for the first time, attaching 0.1 NEAR to cover storage. The `is_new_attestation = true` branch fires; excess is refunded correctly.
2. Alice's TEE quote expires. She calls `submit_participant_info` again with the same 0.1 NEAR attached (same tooling script).
3. Now `is_new_attestation = false` (entry already exists) and `caller_is_not_participant = false` (Alice is a registered participant).
4. `attestation_storage_must_be_paid_by_caller = false || false = false`.
5. The `if` block is skipped entirely. Alice's 0.1 NEAR is absorbed by the contract with no refund.
6. The contract balance increases by 0.1 NEAR; Alice's balance decreases by 0.1 NEAR. No recovery is possible. [4](#0-3)

    #[payable]
    #[handle_result]
    pub fn submit_participant_info(
        &mut self,
        proposed_participant_attestation: dtos::Attestation,
        tls_public_key: dtos::Ed25519PublicKey,
    ) -> Result<(), Error> {
        let proposed_participant_attestation =
            proposed_participant_attestation.try_into_contract_type()?;

        let account_key = env::signer_account_pk();
        let account_id = Self::assert_caller_is_signer();

        log!(
            "submit_participant_info: signer={}, proposed_participant_attestation={:?}, account_key={:?}",
            account_id,
            proposed_participant_attestation,
            account_key
        );

        // Save the initial storage usage to know how much to charge the proposer for the storage
        // used
        let initial_storage = env::storage_usage();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        // The node always signs submissions with an Ed25519 key
        // (`near_signer_key`), so the signer key here is Ed25519 in practice.
        // Reject non-Ed25519 signer keys rather than silently storing a value
        // we could never match against in `is_caller_an_attested_participant`.
        let account_public_key = dtos::Ed25519PublicKey::try_from(&account_key).map_err(|_| {
            InvalidParameters::InvalidTeeRemoteAttestation {
                reason: "signer account key must be Ed25519".to_string(),
            }
        })?;

        // Add the participant information to the contract state
        let attestation_insertion_result = self
            .tee_state
            .add_participant(
                NodeId {
                    account_id: account_id.clone(),
                    tls_public_key,
                    account_public_key,
                },
                proposed_participant_attestation,
                tee_upgrade_deadline_duration,
            )
            .map_err(|err| {
                let reason = match &err {
                    AttestationSubmissionError::InvalidAttestation(_) => {
                        format!("TeeQuoteStatus is invalid: {err}")
                    }
                    AttestationSubmissionError::TlsKeyOwnedByOtherAccount => err.to_string(),
                };
                InvalidParameters::InvalidTeeRemoteAttestation { reason }
            })?;

        let caller_is_not_participant = self.voter_account().is_err();
        let is_new_attestation = matches!(
            attestation_insertion_result,
            ParticipantInsertion::NewlyInsertedParticipant
        );

        let attestation_storage_must_be_paid_by_caller =
            is_new_attestation || caller_is_not_participant;

        if attestation_storage_must_be_paid_by_caller {
            // `saturating_sub`: if a re-submission shrinks the entry, charge nothing
            // rather than underflow. Intentional asymmetry: we do not refund freed bytes
            // either — the caller already paid for the larger entry, and we'd rather
            // accept that asymmetry than open a refund path for payload-shrinking games.
            let storage_used = env::storage_usage().saturating_sub(initial_storage);
            let cost = env::storage_byte_cost().saturating_mul(storage_used as u128);
            let attached = env::attached_deposit();

            if attached < cost {
                return Err(InvalidParameters::InsufficientDeposit {
                    attached: attached.as_yoctonear(),
                    required: cost.as_yoctonear(),
                }
                .into());
            }

            // Refund the difference if the proposer attached more than required
            if let Some(diff) = attached.checked_sub(cost)
                && diff > NearToken::from_yoctonear(0)
            {
                Promise::new(account_id).transfer(diff).detach();
            }
        }

        Ok(())
    }
