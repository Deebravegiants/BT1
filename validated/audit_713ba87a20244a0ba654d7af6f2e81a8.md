Audit Report

## Title
Share Inflation Attack via `rsETHPrice` Manipulation on First Deposit - (File: `contracts/LRTOracle.sol`, `contracts/LRTDepositPool.sol`)

## Summary
When `rsethSupply` is 1 wei (first depositor state), an attacker can directly transfer a large ERC20 balance to `LRTDepositPool`, then call the public `updateRSETHPrice()` to commit an astronomically inflated `rsETHPrice`. Because `pricePercentageLimit` is never set in `initialize()` and defaults to `0`, the price-increase guard is permanently disabled for unprivileged callers. Subsequent depositors who pass `minRSETHAmountExpected = 0` receive 0 rsETH while their assets are absorbed into the TVL, which the attacker redeems in full via their 1-wei rsETH position.

## Finding Description

**Root cause 1 — `updateRSETHPrice()` is public and permissionless:** [1](#0-0) 

Any external caller can commit a new `rsETHPrice` at any time the contract is not paused.

**Root cause 2 — `pricePercentageLimit` is never initialized, permanently disabling the price-increase guard:** [2](#0-1) 

`initialize()` never sets `pricePercentageLimit`, so it remains `0`. The guard at line 256 short-circuits: [3](#0-2) 

`pricePercentageLimit > 0` is always `false`, so `isPriceIncreaseOffLimit` is always `false` and the `PriceAboveDailyThreshold` revert is never reached for any caller.

**Root cause 3 — TVL reads live `balanceOf`, making it donation-inflatable:** [4](#0-3) 

A direct ERC20 transfer to `LRTDepositPool` immediately inflates `getTotalAssetDeposits`, which feeds into `_getTotalEthInProtocol()`: [5](#0-4) 

**Root cause 4 — Minting uses the stored (manipulable) `rsETHPrice`:** [6](#0-5) 

When `rsETHPrice` is inflated to `X >> amount * assetPrice`, integer division rounds down to 0.

**Root cause 5 — No non-zero output enforcement:** [7](#0-6) 

`0 < 0` is `false`, so a victim passing `minRSETHAmountExpected = 0` receives 0 rsETH with no revert. Their ERC20 tokens are transferred in at line 114 before `_mintRsETH(0)` is called. [8](#0-7) 

## Impact Explanation

**Critical — Direct theft of user funds.** The victim's deposited LST is permanently absorbed into the protocol TVL. The attacker's 1-wei rsETH represents 100% of the rsETH supply and therefore 100% of TVL. Upon withdrawal, the attacker redeems at the inflated price and extracts the victim's full deposit. The attack is repeatable against every subsequent depositor.

## Likelihood Explanation

**Medium-to-High.** The attacker must be the first depositor (or act when `rsethSupply` is near zero). The attack is front-runnable: the attacker monitors the mempool for the first legitimate deposit, front-runs with steps 1–3, and back-runs with the withdrawal. No privilege is required. The only natural mitigation (`pricePercentageLimit`) is unset by default and requires an admin action to activate. The donation cost is recoverable from the victim's deposit.

## Recommendation

1. **Set `pricePercentageLimit` to a non-zero value in `initialize()`** (e.g., `1e16` for 1%) so that a sudden price spike from a donation triggers `PriceAboveDailyThreshold` for non-manager callers.
2. **Seed the protocol with a non-trivial initial rsETH mint to `address(0)`** on first deployment so `rsethSupply` is never 1 wei.
3. **Enforce a non-zero minimum rsETH output** in `_beforeDeposit`: revert if `rsethAmountToMint == 0` regardless of `minRSETHAmountExpected`.
4. **Snapshot TVL at deposit time** or use a time-weighted price rather than reading live `balanceOf` to prevent single-block manipulation.

## Proof of Concept

```
Initial state:
  rsETHPrice = 1e18 (set by updateRSETHPrice() when rsethSupply == 0)
  rsethSupply = 0

Step 1 — Attacker deposits 1 wei stETH via depositAsset():
  rsethAmountToMint = (1 * 1e18) / 1e18 = 1 wei rsETH
  rsethSupply = 1

Step 2 — Attacker directly transfers 1,000,000e18 stETH to LRTDepositPool:
  IERC20(stETH).transfer(address(depositPool), 1_000_000e18)
  getTotalAssetDeposits(stETH) = 1_000_000e18 + 1

Step 3 — Attacker calls updateRSETHPrice() (public, no access control):
  totalETHInProtocol ≈ 1_000_000e18
  newRsETHPrice = 1_000_000e18 * 1e18 / 1 = 1e42
  pricePercentageLimit == 0 → isPriceIncreaseOffLimit = false → no revert
  rsETHPrice = 1e42

Step 4 — Victim calls depositAsset(stETH, 1_000_000e18, 0, ""):
  rsethAmountToMint = (1_000_000e18 * 1e18) / 1e42 = 0
  0 >= minRSETHAmountExpected (0) → no revert
  Victim's 1_000_000e18 stETH transferred in; 0 rsETH minted.

Step 5 — Attacker redeems 1 wei rsETH (= 100% of supply):
  Entitled to 100% of TVL ≈ 2_000_000e18 stETH
  Attacker profit ≈ 1_000_000e18 stETH (victim's deposit minus donation cost)
```

**Foundry test plan:** Deploy `LRTOracle` + `LRTDepositPool` on a local fork. Call `initialize()` without setting `pricePercentageLimit`. Execute steps 1–5 above. Assert victim's rsETH balance is 0 and attacker's redeemable TVL share equals the victim's deposit amount.
