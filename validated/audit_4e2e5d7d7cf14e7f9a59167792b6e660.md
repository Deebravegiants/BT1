### Title
Consensus Vote Signature Never Verified — Any Peer Can Forge Votes from Committee Members to Force Quorum - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`SingleHeightConsensus::handle_vote` checks that `vote.voter` is a committee member (identity/membership check) but never verifies the cryptographic `signature` field on the `Vote`. The code contains an explicit `// TODO(Asmaa): verify the signature` comment acknowledging the gap. Because `voter` is a self-reported field in the wire message, any network peer can craft a `Vote` with `voter` set to any legitimate committee member's address and have it counted toward quorum. This is the direct Sequencer analog of the Gorples `mint_authority` bug: membership in a set is checked, but actual signing authority is never verified.

### Impact Explanation

An attacker who can inject messages into the consensus vote broadcast channel can:

1. Observe a legitimate `ProposalInit` that has been validated and recorded in the state machine's `proposals` map.
2. Craft `Vote` messages (prevote and precommit) with `voter` set to any committee member's `ContractAddress` and `signature` set to `RawSignature::default()`.
3. Accumulate enough forged votes to satisfy the 2/3 quorum threshold (`VotesThreshold`).
4. Trigger `upon_decision`, which emits `SMRequest::DecisionReached` with the attacker-chosen `proposal_commitment`.

The `upon_decision` path in the state machine: [6](#0-5) 

The resulting `DecisionReached` event is committed via `context.decision_reached`, which calls `batcher.decision_reached` and `state_sync_client.add_new_block`, anchoring the forged decision into the node's state. This bypasses the Byzantine fault-tolerance guarantee: a single attacker with network access can force a consensus decision without any genuine validator participation, leading to wrong block commitments, wrong state roots, and wrong receipts/events being committed.

### Likelihood Explanation

Any peer that can send messages to the `BroadcastVoteChannel` (the gossip/broadcast topic for consensus votes) can exploit this. The network layer does not authenticate the consensus-layer `voter` field against the transport-layer peer identity. The `// TODO(Asmaa): verify the signature` comment confirms this is a known, unimplemented control, not a design choice. Likelihood is **High** for any deployment where the consensus vote channel is reachable by untrusted peers.

### Recommendation

1. **Verify the vote signature in `handle_vote`** before accepting the vote. Use the existing `verify_precommit_vote_signature` (or a generalized equivalent) from `apollo_signature_manager`, keyed to the committee member's registered public key.
2. **Sign votes before broadcasting** in `make_self_vote` using `SignatureManager::sign_precommit_vote` (or a prevote equivalent), replacing `RawSignature::default()`.
3. **Reject votes with invalid or missing signatures** at the `SingleHeightConsensus::handle_vote` boundary, before they reach the state machine.
4. Ensure the committee member's public key is retrievable from the staking/committee data so it can be used for verification.

### Proof of Concept

A malicious peer sends the following forged precommit votes (pseudocode):

```rust
// Attacker observes a validated proposal with commitment C at height H, round R.
// Attacker knows committee members: [V1, V2, V3, V4] with total_weight = 4.
// Quorum = 3 (2/3 of 4, rounded up).

for voter in [V1, V2, V3] {
    let forged_vote = Vote {
        vote_type: VoteType::Precommit,
        height: H,
        round: R,
        proposal_commitment: Some(C),
        voter,                          // legitimate committee address, self-reported
        signature: RawSignature::default(), // zero bytes, never checked
    };
    broadcast(forged_vote); // injected into the consensus vote channel
}
// Result: upon_decision fires, DecisionReached(C) is emitted,
// batcher.decision_reached is called, block is committed.
```

The `handle_vote` membership check passes because `voter` is a real committee address. The `signature` field is never verified. The state machine accumulates 3 forged precommit votes, meets quorum, and emits `DecisionReached` — identical to the Gorples case where passing a valid-looking `mint_authority` address (without signing) was sufficient to mint unlimited tokens.
