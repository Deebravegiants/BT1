### Title
Consensus Vote Signature Never Verified — Any Peer Can Impersonate Validators to Forge Quorum - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`SingleHeightConsensus::handle_vote` accepts network votes by checking only that `vote.voter` is a committee member, but never verifies the cryptographic `signature` field against that claimed voter address. An explicit `// TODO(Asmaa): verify the signature` comment confirms the omission. Any unprivileged network peer can broadcast forged votes claiming to be any validator, manufacture a false quorum, and drive consensus to decide on an attacker-chosen `ProposalCommitment`.

### Finding Description

`handle_vote` in `crates/apollo_consensus/src/single_height_consensus.rs` performs two checks before accepting a vote:

1. Height match (line 245)
2. Committee membership of `vote.voter` (line 249)

The `Vote` struct carries both `voter: ContractAddress` (claimed identity) and `signature: RawSignature` (cryptographic proof): [2](#0-1) 

The signature field is populated over the wire: [3](#0-2) 

But it is never verified. On the signing side, `make_self_vote` in `state_machine.rs` also carries the same TODO and emits an empty default signature: [4](#0-3) 

The verification infrastructure exists (`verify_precommit_vote_signature` in `apollo_signature_manager`) but is never called in the vote-handling path: [5](#0-4) 

This is the direct sequencer analog of the LlamaCore `castApprovalBySig` bug: just as that code used `msg.sender` (the relayer) instead of `policyholder` (the actual authorized party) for the permission check, here `vote.voter` (the claimed identity embedded in the message) is trusted without any cryptographic proof that the network sender controls that address.

### Impact Explanation

A single malicious network peer can:

1. Construct `Vote` messages with `vote.voter` set to any legitimate validator address and `signature` left empty/default.
2. Broadcast enough such forged votes to reach the 2/3+ quorum threshold.
3. The state machine in `handle_precommit` / `handle_prevote` accumulates these votes by `(round, voter)` key and triggers quorum logic: [6](#0-5) 

4. Consensus decides on the attacker-chosen `ProposalCommitment`, committing a wrong block to the chain — wrong state, wrong receipts, wrong events, wrong storage roots for all subsequent heights.

This matches the allowed impact: **wrong state committed from consensus decision driven by forged validator votes**.

### Likelihood Explanation

The gossipsub broadcast channel for votes is reachable by any peer that connects to the p2p network. No privileged access is required. The `Vote` protobuf message is fully attacker-controlled. The only guard — committee membership of `vote.voter` — is bypassable by simply setting `vote.voter` to any known validator address, which is public information from the staking contract.

### Recommendation

In `handle_vote`, after the committee membership check, look up the validator's public key from the committee staker set and call `verify_precommit_vote_signature` (or an equivalent that covers both prevote and precommit) before accepting the vote:

```rust
let staker = self.committee.members().iter()
    .find(|s| s.address == vote.voter)
    .ok_or(...)?;
let valid = verify_precommit_vote_signature(
    vote.proposal_commitment_as_block_hash(),
    vote.signature.clone(),
    staker.public_key,
)?;
if !valid {
    warn!("Invalid signature on vote from {:?}", vote.voter);
    return VecDeque::new();
}
```

The signing side (`make_self_vote`) must also be updated to produce a real signature using `SignatureManager::sign_precommit_vote`.

### Proof of Concept

```
1. Attacker connects to the sequencer p2p network as any peer.
2. Attacker reads the current committee from the staking contract to obtain
   validator addresses V1, V2, V3 (enough for 2/3 quorum).
3. For each Vi, attacker constructs:
     Vote {
       vote_type: Precommit,
       height: H,
       round: R,
       proposal_commitment: Some(<attacker-chosen commitment>),
       voter: Vi,
       signature: RawSignature::default(),   // empty — never checked
     }
4. Attacker broadcasts all forged votes via the gossipsub vote topic.
5. handle_vote passes the committee membership check for each Vi.
6. The signature TODO is skipped; votes are fed into the state machine.
7. Quorum is reached; consensus calls decision_reached with the
   attacker-chosen ProposalCommitment, committing a wrong block.
``` [7](#0-6)

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
