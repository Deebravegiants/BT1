Audit Report

## Title
Excess ETH Permanently Locked in `updateRate()` With No Refund or Recovery Path - (File: contracts/cross-chain/MultiChainRateProvider.sol)

## Summary
`MultiChainRateProvider::updateRate()` is `payable` and accepts ETH to cover LayerZero messaging fees. It computes each receiver's fee on-chain via `ILayerZeroEndpoint.estimateFees()` and forwards exactly that amount per send. Any ETH in `msg.value` beyond the sum of all per-receiver fees remains in the contract permanently, as there is no refund logic, no `receive()`/`fallback()` function, and no owner-rescue function.

## Finding Description
Inside `updateRate()` (lines 108–137 of `contracts/cross-chain/MultiChainRateProvider.sol`), the loop computes `estimatedFee` on-chain for each receiver and calls:

```solidity
ILayerZeroEndpoint(layerZeroEndpoint).send{ value: estimatedFee }(
    dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
);
```

The `_refundAddress` parameter (`payable(msg.sender)`) is LayerZero's internal per-send refund: per the `ILayerZeroEndpoint` interface comment (line 12–13 of `contracts/external/layerzero/interfaces/ILayerZeroEndpoint.sol`), it only refunds ETH if LayerZero itself consumes *less than the `estimatedFee` forwarded to that single `send()` call*. It does not refund the difference between `msg.value` and `Σ estimatedFee_i`.

After the loop completes, any surplus (`msg.value − Σ estimatedFee_i`) sits in the `MultiChainRateProvider` contract. The contract:
- Has no `receive()` or `fallback()` function
- Has no `withdraw` or ETH-rescue function
- Inherits non-upgradeable `Ownable` (line 4, 13)

There is no on-chain path to recover stranded ETH.

## Impact Explanation
Any ETH sent in excess of the exact total LayerZero fee is permanently frozen in the contract with zero recovery path. This satisfies **Critical – Permanent freezing of funds**: the contract is non-upgradeable, has no sweep function, and the ETH cannot be retrieved by any party.

## Likelihood Explanation
`updateRate()` has no access control — any external caller can invoke it. Callers (including the protocol's own keeper/operator) routinely add a safety buffer when paying for cross-chain messaging fees because the off-chain estimate from `estimateTotalFee()` may differ from the on-chain execution context. Any overpayment is irrecoverable. Likelihood is **Medium**.

## Recommendation
Add an explicit refund of unused ETH at the end of `updateRate()`:

```solidity
uint256 remaining = address(this).balance;
if (remaining > 0) {
    (bool ok,) = payable(msg.sender).call{value: remaining}("");
    require(ok, "ETH refund failed");
}
```

Alternatively, enforce `msg.value == estimateTotalFee()` with a strict equality check, or add an owner-only ETH rescue function.

## Proof of Concept
1. Deploy a concrete implementation of `MultiChainRateProvider` with two `rateReceivers`.
2. Call `estimateTotalFee()` → returns `0.01 ETH`.
3. Call `updateRate{value: 0.1 ETH}()` (caller adds a 10× buffer).
4. Inside the loop, `estimatedFee` for each receiver is computed on-chain; only `0.01 ETH` total is forwarded to LayerZero via `send{ value: estimatedFee }`.
5. `0.09 ETH` remains in the `MultiChainRateProvider` contract.
6. No function exists to withdraw it; the ETH is permanently locked.

Foundry test sketch:
```solidity
function test_excessEthLocked() public {
    uint256 totalFee = provider.estimateTotalFee();
    uint256 overpay = totalFee * 10;
    provider.updateRate{value: overpay}();
    assertEq(address(provider).balance, overpay - totalFee); // surplus stuck
}

    function updateRate() external payable nonReentrant {
        uint256 latestRate = getLatestRate();

        rate = latestRate;

        lastUpdated = block.timestamp;

        bytes memory _payload = abi.encode(latestRate);

        uint256 rateReceiversLength = rateReceivers.length;

        for (uint256 i; i < rateReceiversLength;) {
            uint16 dstChainId = uint16(rateReceivers[i]._chainId);

            bytes memory remoteAndLocalAddresses = abi.encodePacked(rateReceivers[i]._contract, address(this));

            (uint256 estimatedFee,) = ILayerZeroEndpoint(layerZeroEndpoint)
                .estimateFees(dstChainId, address(this), _payload, false, bytes(""));

            ILayerZeroEndpoint(layerZeroEndpoint).send{ value: estimatedFee }(
                dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
            );

            unchecked {
                ++i;
            }
        }

        emit RateUpdated(rate);
    }
