### Title
Unvalidated Chainlink `latestRoundData()` Return Values Allow Stale/Negative Price to Corrupt rsETH Minting Math - (File: contracts/oracles/ChainlinkPriceOracle.sol)

### Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `updatedAt` and `answeredInRound`, and blindly casts the `int256 price` to `uint256` without checking for non-positive values. This stale or negative price propagates directly into rsETH mint calculations and the protocol-wide TVL computation, enabling share mis-accounting that harms depositors.

### Finding Description
In `contracts/oracles/ChainlinkPriceOracle.sol`, `getAssetPrice()` fetches the Chainlink price as follows:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Three validations are absent:
1. **No staleness check** — `updatedAt` is silently discarded; a feed that stopped updating hours or days ago returns without revert.
2. **No incomplete-round check** — `answeredInRound < roundId` is never tested.
3. **No non-positive price check** — `price` is cast directly to `uint256`; a negative `int256` wraps to a value near `2^256`, making the returned "price" astronomically large.

By contrast, the sibling contract `ChainlinkOracleForRSETHPoolCollateral` performs all three checks (`answeredInRound < roundID`, `timestamp == 0`, `ethPrice <= 0`), confirming the team is aware of the pattern but did not apply it to `ChainlinkPriceOracle`. [1](#0-0) [2](#0-1) 

The corrupted price flows through two critical paths:

**Path 1 — rsETH mint amount (deposit flow)**
`LRTDepositPool.getRsETHAmountToMint()` calls `lrtOracle.getAssetPrice(asset)` directly, which delegates to `ChainlinkPriceOracle.getAssetPrice()`. A stale-low price means the depositor receives fewer rsETH than their assets are worth; a negative-wrapped price means the numerator overflows and the depositor receives near-zero rsETH. [3](#0-2) 

**Path 2 — rsETH price update (public function)**
`LRTOracle.updateRSETHPrice()` is an unrestricted public function. It calls `_getTotalEthInProtocol()`, which iterates over all supported assets and sums `assetER * totalAssetAmt` using the same `getAssetPrice()`. A corrupted asset price inflates or deflates `totalETHInProtocol`, producing a wrong `newRsETHPrice` that is then stored in `rsETHPrice` and used for all subsequent mints. [4](#0-3) [5](#0-4) 

### Impact Explanation
**High.** A negative price from a malfunctioning feed causes `uint256(price)` to wrap to ~`2^256`, making `totalETHInProtocol` astronomically large. `newRsETHPrice` becomes enormous, so every subsequent depositor receives near-zero rsETH for their assets — a direct, permanent loss of deposited value. Even a merely stale price (e.g., a feed frozen at a value 10% below true market) causes depositors to receive systematically incorrect rsETH amounts, constituting theft of value from depositors or from existing rsETH holders depending on the direction of the error.

### Likelihood Explanation
**Low.** Chainlink feeds are generally reliable, but documented incidents of feeds going stale exist (e.g., during the Optimism outage, during extreme volatility). The negative-price scenario is rarer but possible during feed misconfiguration or circuit-breaker events. The public `updateRSETHPrice()` entry point means no privileged access is needed to trigger the bad state once the feed is stale.

### Recommendation

```diff
- (, int256 price,,,) = priceFeed.latestRoundData();
+ (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
+     priceFeed.latestRoundData();

+ if (price <= 0) revert InvalidPrice();
+ if (answeredInRound < roundId) revert StalePrice();
+ if (updatedAt < block.timestamp - 24 hours) revert StalePrice();

  return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

The staleness threshold should be set per-feed based on its documented heartbeat (e.g., 1 hour for ETH/USD, 24 hours for less-frequent feeds).

### Proof of Concept

1. Assume a supported LST asset (e.g., stETH) has its Chainlink feed go stale or return a negative `answer`.
2. Any address calls `LRTOracle.updateRSETHPrice()` (no access control).
3. `_getTotalEthInProtocol()` calls `ChainlinkPriceOracle.getAssetPrice(stETH)`.
4. `latestRoundData()` returns `price = -1` (negative); `uint256(-1)` = `2^256 - 1`.
5. `totalETHInProtocol` overflows to a near-maximum value.
6. `newRsETHPrice = totalETHInProtocol / rsethSupply` becomes enormous.
7. `rsETHPrice` is updated to this enormous value.
8. A depositor calls `depositAsset(stETH, 1e18)`: `rsethAmountToMint = (1e18 * getAssetPrice(stETH)) / rsETHPrice ≈ 0`.
9. The depositor's 1 stETH is accepted but they receive 0 (or dust) rsETH — funds are effectively stolen. [6](#0-5) [7](#0-6)

        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
