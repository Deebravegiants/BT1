### Title
Unbounded `fee_proposal_fri` Accepted During Startup Window Enables Persistent L2 Gas Price Manipulation — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

During the startup/near-genesis period (first `window_size` blocks, or after any restart/revert that empties the sliding window), `fee_actual` is `None` and the bounds check on `fee_proposal_fri` inside `is_proposal_init_valid` is completely skipped. A malicious proposer can embed an arbitrary `fee_proposal_fri` value in `ProposalInit`, validators accept it unconditionally, the value is committed into the `ProposalCommitment` hash, stored in `fee_proposals_window`, and then poisons the `fee_actual` median used to price every subsequent block once the window fills.

---

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` guards the `fee_proposal_fri` value with an `if let (Some(fee_actual), Some(fee_proposal))` pattern:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

When `fee_actual` is `None` the entire block is a no-op. `fee_actual` becomes `None` whenever `compute_fee_actual` cannot fill the window:

But nothing in the validator path enforces that the proposer used this honest fallback. The proposer can write any `u128` into `fee_proposal_fri` in the wire `ProposalInit` message: [4](#0-3) 

After the proposal is decided, the manipulated value is committed into the `ProposalCommitment` via Poseidon:

```rust

    partial: PartialBlockHash,
    fee_proposal: Option<GasPrice>,
) -> ProposalCommitment {
    let Some(fee_proposal) = fee_proposal else {
        return ProposalCommitment(partial.0);
    };
    ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
}
``` [5](#0-4) 

It is then stored in `fee_proposals_window`:

```rust
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [6](#0-5) 

And forwarded to the centralized cende pipeline:

```rust
fee_proposal_info: FeeProposalInfo { fee_proposal_fri: init.fee_proposal_fri },
``` [7](#0-6) 

Once the window accumulates `window_size` entries, `compute_fee_actual` returns the median of those stored values. If the startup-window entries were poisoned with `u128::MAX`, the median is `u128::MAX`, which becomes the floor for the EIP-1559 L2 gas price calculation for every subsequent block.

---

### Impact Explanation

**Incorrect fee / gas price with economic impact (Critical scope).**

- The `fee_actual` derived from the poisoned window becomes the floor passed to `calculate_next_l2_gas_price_for_fin`, driving the L2 gas price to an attacker-chosen value for all blocks after the window fills.
- The manipulated `fee_proposal_fri` is hashed into `ProposalCommitment` (`Poseidon(partial_block_hash, fee_proposal_fri)`), so the wrong commitment is what validators sign and what the parent-commitment chain carries forward.
- The value is also written into the cende blob sent to the centralized recorder, corrupting the off-chain fee record.

---

### Likelihood Explanation

The startup window is a deterministic, reachable condition: it applies to the first `window_size` blocks after genesis and to any node that restarts or reverts and re-enters consensus before the window repopulates. Any validator selected as proposer during this window can trigger the issue without any additional privilege beyond normal validator participation. The `initialize_fee_proposals_window` function re-queries state sync on startup, but a gap or pre-V0_14_3 entry in that range also produces `fee_actual = None`, extending the attack surface beyond genesis. [8](#0-7) 

---

### Recommendation

Add an absolute bounds check on `fee_proposal_fri` that is enforced regardless of whether `fee_actual` is `None`. At minimum, reject any `fee_proposal_fri` outside `[min_l2_gas_price, some_configured_maximum]`. This mirrors the recommendation in the external report: pass a hard limit as a parameter so that the price cannot be set to an unexpected value even when the reference anchor (here, `fee_actual`) is unavailable.

---

### Proof of Concept

1. Network launches; `fee_proposals_window` is empty; `compute_fee_actual` returns `None` for all heights `< window_size`.
2. A malicious validator is selected as proposer for block 1. It constructs a `ProposalInit` with `fee_proposal_fri = Some(GasPrice(u128::MAX))`.
3. Validators call `is_proposal_init_valid`. Because `proposal_init_validation.fee_actual` is `None`, the `if let (Some(fee_actual), Some(fee_proposal))` guard is false and the bounds check is skipped entirely.
4. The proposal passes all other checks (timestamp, L1 gas price margin, starknet version, etc.) and is accepted.
5. `finalize_decision` calls `self.record_fee_proposal(height, Some(GasPrice(u128::MAX)))`, storing the poisoned value.
6. The attacker repeats for as many startup-window blocks as they are selected proposer for.
7. Once `height >= window_size`, `compute_fee_actual` returns the median of the poisoned window — potentially `u128::MAX` — which becomes the floor for `calculate_next_l2_gas_price_for_fin` for all subsequent blocks.
8. Every user transaction is now priced against an attacker-controlled L2 gas price floor, with no on-chain mechanism to correct it until the poisoned entries age out of the window (which itself takes `window_size` honest blocks, each bounded by the now-extreme `fee_actual`). ```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

    pub async fn initialize_fee_proposals_window(
        &mut self,
        start_height: BlockNumber,
    ) -> StateSyncClientResult<()> {
        const STATE_SYNC_RETRY_INTERVAL: Duration = Duration::from_millis(500);
        let window_size = VersionedConstants::latest_constants().fee_proposal_window_size;
        let window_end_height = start_height.0;
        let window_start_height = window_end_height.saturating_sub(window_size);
        let mut pending_heights: VecDeque<BlockNumber> =
            (window_start_height..window_end_height).map(BlockNumber).collect();
        while let Some(block_number) = pending_heights.pop_front() {
            match self.deps.state_sync_client.get_block(block_number).await {
                Ok(block) => self.record_fee_proposal(
                    block_number,
                    block.block_header_without_hash.fee_proposal_fri,
                ),
                Err(StateSyncClientError::StateSyncError(StateSyncError::BlockNotFound(_))) => {
                    warn!(
                        "State sync not ready for height {block_number}; re-queueing after \
                         {STATE_SYNC_RETRY_INTERVAL:?}"
                    );
                    pending_heights.push_back(block_number);
                    tokio::time::sleep(STATE_SYNC_RETRY_INTERVAL).await;
                }
                Err(e) => return Err(e),
            }
        }
        Ok(())
    }
