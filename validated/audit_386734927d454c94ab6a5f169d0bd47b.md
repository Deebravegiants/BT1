### Title
Attacker-Controlled `EvmFinality::Latest` Bypasses Finality Enforcement in `extract`, Enabling Signatures Over Reorg-Vulnerable Transactions — (File: `crates/foreign-chain-inspector/src/evm/inspector.rs`)

---

### Summary

The `EvmFinality` level is fully attacker-controlled in `verify_foreign_transaction`. No minimum finality is enforced at the contract, node, or inspector layer. An unprivileged caller can submit `EvmFinality::Latest`, causing `extract` to sign over a transaction that is in the most-recent (non-finalized) EVM block. If the chain reorgs after signing, the MPC signature remains valid but attests to a transaction that no longer exists on the canonical chain.

---

### Finding Description

`EvmFinality` is a caller-supplied field inside `EvmRpcRequest`, which is part of `ForeignChainRpcRequest`, which is part of `VerifyForeignTransactionRequestArgs`. The contract accepts all three variants without restriction: [1](#0-0) 

The contract entry point performs no finality-level validation: [2](#0-1) 

The node-side conversion accepts `Latest` without error: [3](#0-2) 

Inside `extract`, the attacker-supplied finality is forwarded directly to `verify_finality_level`: [4](#0-3) 

`verify_finality_level` with `EthereumFinality::Latest` queries `eth_getBlockByNumber("latest")` — the chain tip — and only checks `head.number >= receipt_block_number`. Any transaction already included in any block trivially passes this check: [5](#0-4) 

`verify_block_is_canonical` then checks the block hash at the moment of the RPC call. This check is valid only at that instant; it provides no protection against a reorg that occurs after the check completes and before (or after) the signature is issued: [6](#0-5) 

Once both checks pass, the MPC network signs `ForeignTxSignPayloadV1 { request, values }` where `request` contains `finality: EvmFinality::Latest`. The signature is permanent and irrevocable.

---

### Impact Explanation

The MPC network is the trust anchor for foreign-chain verification. Downstream consumers (e.g., Omnibridge) verify the MPC signature and act on the attested values without independently re-verifying finality. A signature over a `Latest`-finality transaction is indistinguishable from a `Finalized`-finality signature to any consumer that does not explicitly gate on the finality field embedded in the payload. If the underlying EVM transaction is reorged out after signing, the attacker holds a valid MPC attestation for a transaction that no longer exists on the canonical chain, enabling invalid bridge execution or double-spend conditions.

---

### Likelihood Explanation

Ethereum mainnet reorgs of 1–2 blocks occur regularly. An attacker submits a transaction, immediately calls `verify_foreign_transaction` with `EvmFinality::Latest`, and races the MPC signing window (which completes in seconds) against the reorg window. On chains with faster block times or weaker finality (e.g., BNB, Polygon, Abstract, HyperEVM — all supported), the reorg window is wider and the attack is easier to execute. No special privilege is required; any account with 1 yoctoNEAR deposit can submit the request.

---

### Recommendation

Enforce a minimum finality level at the contract or node layer. The simplest fix is to reject any `EvmRpcRequest` where `finality != EvmFinality::Finalized` inside `verify_foreign_transaction` or inside the node's signing handler. Alternatively, add a per-chain minimum-finality policy to the on-chain foreign-chain configuration so operators can set the floor. `EthereumFinality::Latest` and `EthereumFinality::Safe` should not be accepted for bridge-critical signing flows.

---

### Proof of Concept

1. Submit a transaction `T` on an EVM chain (e.g., BNB) that is included in block `N` (the current latest block).
2. Immediately call `verify_foreign_transaction` with:
   ```json
   {
     "request": {
       "Bnb": {
         "tx_id": "<hash of T>",
         "finality": "Latest",
         "extractors": ["BlockHash"]
       }
     },
     "domain_id": <foreign_tx_domain>,
     "payload_version": "V1"
   }
   ```
3. MPC nodes call `eth_getTransactionReceipt` → receipt at block `N`. [7](#0-6) 
4. `verify_finality_level` queries `eth_getBlockByNumber("latest")` → returns block `N` (or `N+1`). Check passes. [8](#0-7) 
5. `verify_block_is_canonical` queries `eth_getBlockByNumber(N)` → hash matches. Check passes. [9](#0-8) 
6. MPC signs and returns `VerifyForeignTransactionResponse` with a valid signature over `(request, [BlockHash(block_N_hash)])`.
7. A 1-block reorg occurs; block `N` is replaced by `N'`. Transaction `T` no longer exists on the canonical chain.
8. Attacker presents the MPC signature to the Omnibridge contract to claim bridge funds for a transaction that was never finalized and no longer exists.

    async fn verify_finality_level(
        &self,
        receipt_block_number: U64,
        finality: EthereumFinality,
    ) -> Result<(), ForeignChainInspectionError> {
        let finality_tag = match finality {
            EthereumFinality::Finalized => FinalityTag::Finalized,
            EthereumFinality::Safe => FinalityTag::Safe,
            EthereumFinality::Latest => FinalityTag::Latest,
        };
        let args = GetBlockByNumberArgs::new(
            BlockNumberOrTag::Tag(finality_tag),
            ReturnFullTransactionObjects::from(false),
        );
        let head: GetBlockByNumberResponse = self
            .client
            .request(GET_BLOCK_BY_NUMBER_METHOD, &args)
            .await?;

        if head.number < receipt_block_number {
            return Err(ForeignChainInspectionError::NotFinalized);
        }
        Ok(())
    }

    async fn verify_block_is_canonical(
        &self,
        receipt_block_number: U64,
        receipt_block_hash: H256,
    ) -> Result<(), ForeignChainInspectionError> {
        let args = GetBlockByNumberArgs::new(
            BlockNumberOrTag::Number(receipt_block_number),
            ReturnFullTransactionObjects::from(false),
        );
        let canonical: GetBlockByNumberResponse = self
            .client
            .request(GET_BLOCK_BY_NUMBER_METHOD, &args)
            .await?;

        let hash_matches = canonical.hash == receipt_block_hash;
        let height_matches = canonical.number == receipt_block_number;
        if !hash_matches || !height_matches {
            return Err(ForeignChainInspectionError::NonCanonicalBlock {
                block_number: receipt_block_number.as_u64(),
                receipt_hash: receipt_block_hash.into(),
                canonical_hash: canonical.hash.into(),
            });
        }
        Ok(())
    }
