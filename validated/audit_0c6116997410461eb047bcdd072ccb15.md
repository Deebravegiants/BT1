### Title
Owner Can Sweep Protocol's KERNEL Distribution Tokens, Bypassing Vesting and Merkle Claim Mechanism - (File: contracts/KERNEL/KernelTop100MerkleDistributor.sol)

### Summary

`KernelTop100MerkleDistributor.withdrawTokens()` imposes no restriction on which token the owner may withdraw. The owner can call it with `_token = address(kernel)` to drain the entire KERNEL balance held in trust for users, bypassing the vesting schedule and merkle-proof verification entirely.

### Finding Description

`KernelTop100MerkleDistributor` holds KERNEL tokens that are meant to be distributed to eligible users according to a 30-day vesting schedule, verified by merkle proofs. The contract exposes an unrestricted token-recovery function: [1](#0-0) 

```solidity
function withdrawTokens(address _token, uint256 _amount, address _recipient) external onlyOwner {
    UtilLib.checkNonZeroAddress(_token);
    UtilLib.checkNonZeroAddress(_recipient);
    if (_amount == 0) { revert ZeroValueProvided(); }
    IERC20(_token).safeTransfer(_recipient, _amount);
    emit TokensWithdrawn(_token, _amount, _recipient);
}
```

There is no guard preventing `_token` from being `address(kernel)`. [2](#0-1) 

The `kernel` state variable is the sole token the contract is designed to distribute. The normal claim path requires:

1. A valid merkle proof (`_verifyClaimProof`) [3](#0-2) 
2. Elapsed vesting time (`_getUnclaimedVestedAmount`) [4](#0-3) 
3. A per-user `amountClaimed` accounting update [5](#0-4) 

`withdrawTokens()` bypasses all three of these controls.

### Impact Explanation

The owner can call `withdrawTokens(address(kernel), kernel.balanceOf(address(this)), ownerAddress)` to drain every KERNEL token held in the distributor before any user has claimed. All users in the merkle tree lose their entire entitled allocation. This is direct theft of unclaimed yield — **High severity**.

### Likelihood Explanation

The function is callable by the `owner` role at any time with no time-lock or multi-sig requirement visible in the contract. The missing check is a single line of code (`if (_token == address(kernel)) revert();`), making it a straightforward oversight rather than a complex attack. The owner role is the only prerequisite; no external conditions are needed.

### Recommendation

Add an explicit guard in `withdrawTokens()` that reverts if the caller attempts to withdraw the primary distribution token:

```solidity
function withdrawTokens(address _token, uint256 _amount, address _recipient) external onlyOwner {
    if (_token == address(kernel)) revert CannotWithdrawKernelToken();
    // ... rest of function
}
```

This mirrors the fix applied in AeraVaultV1 PR #132, which added a check preventing the Treasury from sweeping BPT tokens that are critical to the protocol's withdrawal flow.

### Proof of Concept

1. Deploy `KernelTop100MerkleDistributor` and fund it with 1,000,000 KERNEL tokens for user distribution.
2. Users are in the merkle tree and vesting has begun.
3. Owner calls:
   ```solidity
   distributor.withdrawTokens(
       address(kernel),
       kernel.balanceOf(address(distributor)),
       ownerAddress
   );
   ```
4. All 1,000,000 KERNEL tokens are transferred to the owner.
5. Every user's subsequent `claim()` or `claimAndStake()` call reverts due to insufficient balance, permanently losing their entitled yield. [1](#0-0) [6](#0-5)

```text
    function _getUnclaimedVestedAmount(address user, uint256 userTotalClaimableAmount) internal view returns (uint256) {
        UserClaim storage userClaim = userClaims[user];

        // If user has claimed everything, return 0
        if (userClaim.amountClaimed >= userTotalClaimableAmount) {
            return 0;
        }

        // Calculate vesting end time
        uint256 vestingEndTime = vestingStartTimestamp + VESTING_DURATION;

        // Calculate start and end times for the period
        uint256 startTime = userClaim.lastClaimTimestamp > 0 ? userClaim.lastClaimTimestamp : vestingStartTimestamp;

        // Cap current time at vesting end time
        uint256 currentTime = block.timestamp;
        if (currentTime > vestingEndTime) {
            currentTime = vestingEndTime;
        }

        // If current time is before start time or vesting hasn't started yet, nothing to claim
        if (currentTime <= startTime || currentTime <= vestingStartTimestamp) {
            return 0;
        }

        // Calculate total vested amount based on time elapsed since vesting start
        uint256 totalElapsedTime = currentTime - vestingStartTimestamp;
        uint256 totalVestedAmount = (userTotalClaimableAmount * totalElapsedTime) / VESTING_DURATION;

        // Cap at total amount
        if (totalVestedAmount > userTotalClaimableAmount) {
            totalVestedAmount = userTotalClaimableAmount;
        }

        // Calculate unclaimed amount
        uint256 unclaimedAmount = totalVestedAmount - userClaim.amountClaimed;

        return unclaimedAmount;
```

```text
    function _verifyClaimProof(address user, uint256 amount, bytes32[] calldata merkleProof) internal view {
        UtilLib.checkNonZeroAddress(user);

        if (merkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (amount == 0) {
            revert ZeroValueProvided();
        }

        // Verify the merkle proof
        bytes32 leaf = keccak256(abi.encodePacked(user, amount));
        bool isValid = MerkleProofUpgradeable.verify(merkleProof, merkleRoot, leaf);

        if (!isValid) {
            revert InvalidMerkleProof();
        }
    }
```

```text
    function claim(uint256 amount, bytes32[] calldata merkleProof) external nonReentrant whenNotPaused {
        address user = msg.sender;

        // Verify merkle proof and update user claim data
        _verifyClaimProof(user, amount, merkleProof);

        // Get claimable amount
        uint256 claimableAmount = _getUnclaimedVestedAmount(user, amount);

        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim data
        userClaims[user].lastClaimTimestamp = block.timestamp;
        userClaims[user].amountClaimed += claimableAmount;

        // Calculate fee
        uint256 fee = (claimableAmount * feeInBPS) / FEE_DENOMINATOR;
        uint256 amountToSend = claimableAmount - fee;

        // Transfer tokens
        if (fee > 0) {
            kernel.safeTransfer(protocolTreasury, fee);
        }
        kernel.safeTransfer(user, amountToSend);

        emit Claimed(user, amountToSend);
    }
```
