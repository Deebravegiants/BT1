### Title
Malformed Weight Proof Segment Causes `AssertionError` Crash in `__get_cc_sub_slot`, Blocking Wallet Sync - (File: `chia/full_node/weight_proof.py`)

### Summary

`__get_cc_sub_slot` in `chia/full_node/weight_proof.py` contains hard `assert` statements that fire when a weight proof segment has its challenge block at sub-slot index 0 for any sub-epoch `> 0`. An unprivileged peer can craft and send such a weight proof, causing an unhandled `AssertionError` that crashes weight proof validation instead of returning `(False, 0)` cleanly. This blocks the wallet node from syncing.

### Finding Description

During weight proof validation, `_validate_sub_epoch_segments` calls `_validate_segment` for each segment. When a segment is sampled, `_validate_segment` calls `__validate_pospace`, which in turn calls `__get_cc_sub_slot` for any segment where `not (first_in_sub_epoch and segment.sub_epoch_n == 0 and idx == 0)`: [1](#0-0) 

`__get_cc_sub_slot` iterates backwards from `idx` to find a slot-end entry: [2](#0-1) 

When `idx == 0`, `reversed(range(0))` is empty, so `sub_slot` remains `None`. The hard `assert sub_slot is not None` at line 1539 then fires an `AssertionError`. Similarly, `assert sub_slot.cc_slot_end_info is not None` at line 1540 fires if all preceding slots lack `cc_slot_end_info`.

A second hard assert exists in `__validate_pospace` itself at line 1413: [3](#0-2) 

This fires when an overflow block is at `idx >= 1` but `slots[idx-1].cc_slot_end_info` is `None`.

Neither `_validate_segment`, `_validate_sub_epoch_segments`, nor `validate_weight_proof_single_proc` wraps these calls in `try/except`: [4](#0-3) [5](#0-4) [6](#0-5) 

The SEC-614 fix addressed the analogous assert in `__get_rc_sub_slot` (replacing `assert first_idx` with a graceful `return None`), and the test explicitly documents the pre-fix crash behavior: [7](#0-6) 

However, `__get_cc_sub_slot` and the inline assert in `__validate_pospace` were not fixed.

### Impact Explanation

An `AssertionError` propagates unhandled through `__validate_pospace` â†’ `_validate_segment` â†’ `_validate_sub_epoch_segments` â†’ `validate_weight_proof_single_proc` (or `validate_weight_proof_inner`), crashing the wallet node's sync process. A malicious peer can repeatedly send crafted weight proofs to prevent the wallet from ever completing a sync, constituting a long-lived denial of sync.

### Likelihood Explanation

Any unprivileged peer can send a `RespondProofOfWeight` message. The attacker controls the weight proof content entirely, so they can include a single segment per sub-epoch (guaranteeing it is the sampled one) with `sub_epoch_n > 0` and the challenge block placed at `sub_slots[0]` (i.e., `cc_slot_end is None` at index 0). No keys, admin access, or cryptographic breaks are required.

### Recommendation

Replace the hard `assert` statements in `__get_cc_sub_slot` with graceful error returns, mirroring the SEC-614 fix applied to `__get_rc_sub_slot`:

```python
# __get_cc_sub_slot
if sub_slot is None or sub_slot.cc_slot_end_info is None:
    log.error("segment missing cc_slot_end_info before challenge block")
    return None  # caller must handle None
```

Change the return type to `ChallengeChainSubSlot | None` and update `__validate_pospace` to propagate `None` upward. Similarly, replace `assert curr_slot.cc_slot_end_info` at line 1413 with a conditional `return None`.

### Proof of Concept

1. Obtain a valid weight proof from a synced node.
2. Find any segment with `sub_epoch_n > 0` that has leading slot-end entries before the challenge block.
3. Strip all leading slot-end entries so the challenge block lands at `sub_slots[0]` (exactly as done in `test_weight_proof_validation_challenge_at_segment_start` for `__get_rc_sub_slot`).
4. Ensure this segment is the only segment for its sub-epoch (so `sampled_seg_index == 0` and it is always sampled).
5. Send the modified weight proof to a victim wallet node during sync.
6. `__validate_pospace` is called with `idx == 0` for `sub_epoch_n > 0`, `__get_cc_sub_slot` is called, `reversed(range(0))` yields nothing, `sub_slot` is `None`, `assert sub_slot is not None` raises `AssertionError`, crashing validation. [2](#0-1) [8](#0-7)

def __get_cc_sub_slot(sub_slots: list[SubSlotData], idx: int, ses: SubEpochSummary | None) -> ChallengeChainSubSlot:
    sub_slot: SubSlotData | None = None
    for i in reversed(range(idx)):
        sub_slot = sub_slots[i]
        if sub_slot.cc_slot_end_info is not None:
            break

    assert sub_slot is not None
    assert sub_slot.cc_slot_end_info is not None
