### Title
`is_caller_an_attested_participant()` Does Not Check Attestation Validity (Expiry / Image-Hash Revocation), Allowing Nodes with Stale or Revoked Attestations to Participate in Threshold Signing — (`File: crates/contract/src/tee/tee_state.rs`)

---

### Summary

`TeeState::is_caller_an_attested_participant()` only verifies that a stored attestation *exists* and that the account/key bindings match. It never calls `re_verify()` to confirm the attestation is still valid (not expired, image hash still on the allowlist, measurements still accepted). As a result, every critical signing-path gate that delegates to `assert_caller_is_attested_participant_and_protocol_active()` — including `respond`, `vote_reshared`, `vote_pk`, `start_keygen_instance`, and `start_reshare_instance` — can be passed by a node whose TEE attestation has expired or whose image hash has been governance-revoked.

---

### Finding Description

`is_caller_an_attested_participant()` performs three checks:

1. The caller's account ID is in the active participant set.
2. A `NodeAttestation` entry exists in `stored_attestations` for the participant's TLS key.
3. The stored `account_id` and `account_public_key` match the transaction signer. [1](#0-0) 

It does **not** call `re_verify()` on the stored `VerifiedAttestation`. The `re_verify()` function is the authoritative validity check: it rejects entries whose `expiry_timestamp_seconds` has passed and entries whose `mpc_image_hash` or `launcher_compose_hash` is no longer on the governance allowlist. [2](#0-1) 

By contrast, `reverify_participants()` — used by `verify_tee()` and `reverify_and_cleanup_participants()` — correctly calls `re_verify()` with the current block timestamp and the live allowlists: [3](#0-2) 

The two code paths diverge: governance-facing checks (`verify_tee`, `vote_new_parameters`) use the full validity check, while the per-call signing-path gate (`assert_caller_is_attested_participant_and_protocol_active`) uses only the existence/binding check. [4](#0-3) 

The comment in `lib.rs` explicitly lists the methods gated by this weaker check: [5](#0-4) 

`DEFAULT_EXPIRATION_DURATION_SECONDS` is 1 day: [6](#0-5) 

The `expiry_timestamp_seconds` is stamped off-chain by the node at submission time and is not re-derived by the contract: [7](#0-6) 

---

### Impact Explanation

A node whose attestation has expired (TEE certificate lapsed, node failed to resubmit) or whose `mpc_image_hash` has been removed from the governance allowlist (image revoked because it was found to be compromised) retains the ability to:

- Call `respond` / `respond_ckd` / `respond_verify_foreign_tx` — submitting signature shares that are accepted into threshold signing operations.
- Call `vote_reshared` — casting a vote that advances or completes a key resharing.
- Call `vote_pk` — casting a vote that completes distributed key generation.
- Call `start_keygen_instance` / `start_reshare_instance` — initiating new key-event attempts as the leader.

The security invariant of the system is that only nodes running in a verified, governance-approved TEE image may participate in signing. This invariant is broken: a node running a revoked image (one removed from the allowlist precisely because it was found unsafe) can still contribute signature shares to every pending signing request that was enqueued before `verify_tee()` was called, and can do so indefinitely if `verify_tee()` is never invoked (it is not called automatically).

This matches the **Medium** allowed impact: participant-state and contract execution-flow manipulation that breaks a production safety invariant (TEE-gated signing participation) without requiring network-level DoS.

---

### Likelihood Explanation

The window opens whenever:

1. A node's attestation expires (1-day TTL; nodes resubmit hourly under normal conditions, but PCCS outages or connectivity issues can prevent renewal).
2. Governance removes an image hash from the allowlist (the intended response to a discovered compromise).

In scenario (2) the window is permanent until `verify_tee()` is called and the node is kicked out. `verify_tee()` is a participant-initiated call with no automatic trigger; in a degraded or adversarial environment it may not be called promptly. The node retains its NEAR account key and participant-list membership, so the only missing gate is the attestation validity check that `is_caller_an_attested_participant()` omits.

---

### Recommendation

Replace the existence-only check in `is_caller_an_attested_participant()` with a full validity check by calling `re_verify()` on the stored `VerifiedAttestation` before returning `Ok(())`. The required inputs (current block timestamp, allowed image hashes, allowed launcher compose hashes, accepted measurements) are already available on `TeeState` and are used identically in `reverify_participants()`. The fix aligns the per-call signing gate with the governance-facing validity check, closing the gap between the two code paths.

---

### Proof of Concept

1. Deploy the contract with participants A, B, C (threshold 2). All three submit valid Dstack attestations.
2. Governance votes to remove participant A's `mpc_image_hash` from the allowlist (`vote_code_hash` / removal path). Participant A can no longer submit a new valid attestation.
3. Do **not** call `verify_tee()`. Participant A's entry remains in `stored_attestations` with the now-invalid `mpc_image_hash`.
4. A user calls `sign()`. The request is enqueued (or `accept_requests` is still `true` because `verify_tee()` was not called).
5. Participant A calls `respond(signature_share, ...)`. The contract calls `assert_caller_is_attested_participant_and_protocol_active()`, which calls `is_caller_an_attested_participant()`. The function finds A's entry in `stored_attestations`, confirms the account/key binding, and returns `Ok(())` — **without calling `re_verify()`**. A's signature share is accepted.
6. With A's share plus one honest node's share, the threshold is met and the signature is produced — with a contribution from a node running a governance-revoked image.

The same path applies with an expired attestation: substitute step 2 with "advance block time past A's `expiry_timestamp_seconds`" and omit calling `verify_tee()`. [1](#0-0) [3](#0-2) [8](#0-7) [4](#0-3)

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

impl VerifiedAttestation {
    pub fn re_verify(
        &self,
        timestamp_seconds: u64,
        allowed_mpc_docker_image_hashes: &[NodeImageHash],
        allowed_launcher_docker_compose_hashes: &[LauncherDockerComposeHash],
        allowed_measurements: &[ExpectedMeasurements],
    ) -> Result<(), VerificationError> {
        match self {
            Self::Dstack(ValidatedDstackAttestation {
                mpc_image_hash,
                launcher_compose_hash,
                expiry_timestamp_seconds: expiration_timestamp_seconds,
                measurements,
            }) => {
                let attestation_has_expired = *expiration_timestamp_seconds < timestamp_seconds;

                if attestation_has_expired {
                    return Err(VerificationError::Custom(format!(
                        "The attestation expired at t = {:?}, time_now = {:?}",
                        expiration_timestamp_seconds, timestamp_seconds
                    )));
                }

                let () = verify_mpc_hash(mpc_image_hash, allowed_mpc_docker_image_hashes)?;
                let () = verify_launcher_compose_hash(
                    launcher_compose_hash,
                    allowed_launcher_docker_compose_hashes,
                )?;

                verify_measurements(measurements, allowed_measurements)?;

                Ok(())
            }
            Self::Mock(mock_attestation) => mock_attestation.verify_constraints(
                timestamp_seconds,
                allowed_mpc_docker_image_hashes,
                allowed_launcher_docker_compose_hashes,
                allowed_measurements,
            ),
        }
    }
