Audit Report

## Title
Stale `rsETHPrice` Enables Depositors to Capture Accrued Yield at Existing Holders' Expense - (File: contracts/LRTDepositPool.sol)

## Summary
`LRTOracle` stores `rsETHPrice` as a state variable updated only via explicit calls to `updateRSETHPrice()`. Neither `depositETH()` nor `depositAsset()` refreshes this price before computing the mint amount, so any depositor can deposit during the staleness window at a lower-than-true price, receive excess rsETH, and profit after the price is updated — directly diluting existing holders' accrued yield.

## Finding Description
`getRsETHAmountToMint()` computes the rsETH to mint as `(amount * assetPrice) / lrtOracle.rsETHPrice()`, reading the stored `rsETHPrice` state variable directly: [1](#0-0) 

Both `depositETH()` and `depositAsset()` delegate to `_beforeDeposit()`, which calls `getRsETHAmountToMint()` without first invoking `updateRSETHPrice()`: [2](#0-1) [3](#0-2) [4](#0-3) 

`updateRSETHPrice()` is public and permissionless (within the `whenNotPaused` guard), meaning anyone can call it, but it is never called atomically with deposits: [5](#0-4) 

`_updateRsETHPrice()` computes the true price as `(totalETHInProtocol - fee) / rsethSupply`, where `totalETHInProtocol` reflects current on-chain balances including accrued staking rewards: [6](#0-5) 

When rewards have accrued since the last `updateRSETHPrice()` call, `rsETHPrice` is lower than the true price. Because `rsETHPrice` is the denominator in the mint formula, a stale (lower) value inflates `rsethAmountToMint`. The attacker:
1. Observes that on-chain TVL (EigenLayer strategy balances + deposit pool balances) exceeds `rsethSupply * rsETHPrice`.
2. Calls `depositETH()` or `depositAsset()` — receives excess rsETH at the stale price.
3. Waits for any caller (keeper, bot, or themselves if within `pricePercentageLimit`) to call `updateRSETHPrice()`.
4. Holds rsETH worth more than the ETH deposited; existing holders' per-token value is reduced by the dilution.

The `pricePercentageLimit` guard only blocks non-manager callers when the price increase exceeds the configured threshold: [7](#0-6) 

For normal continuous reward accrual (e.g., ~4–5% APY), each update period's price increase is well within any reasonable daily threshold, so the attacker can call `updateRSETHPrice()` themselves to complete the attack. Even if the threshold is exceeded, the manager will eventually update the price, and the attacker's inflated position benefits regardless.

## Impact Explanation
This is **High — Theft of unclaimed yield**. Accrued staking rewards are embedded in the rising TVL and belong to existing rsETH holders. By depositing at the stale price, the attacker mints more rsETH than the true exchange rate warrants. After the price update, the new equilibrium price is lower than it would have been without the attacker's deposit, transferring a portion of the accrued yield from existing holders to the attacker. The magnitude scales with deposit size and oracle staleness duration.

## Likelihood Explanation
Staking rewards accrue continuously and `updateRSETHPrice()` is never called automatically in any deposit path, so a staleness window always exists between reward accrual and the next oracle update. The attack requires no special role, no flash loan, and no victim mistake — only monitoring on-chain balances (e.g., EigenLayer strategy shares) against the stored `rsETHPrice`. It is repeatable on every oracle update cycle.

## Recommendation
Call `_updateRsETHPrice()` (or an internal equivalent) at the beginning of `depositETH()` and `depositAsset()` — before `_beforeDeposit()` computes `rsethAmountToMint` — so that `rsETHPrice` always reflects the current TVL before any new rsETH is minted.

## Proof of Concept
1. Deploy on a mainnet fork. Record `rsETHPrice = P` and `totalETHInProtocol = T` where `T > rsethSupply * P` (rewards have accrued).
2. Call `LRTDepositPool.depositETH{value: X}(0, "")`. Observe `rsethAmountToMint = X / P` (inflated vs. true `X / (T / rsethSupply)`).
3. Call `LRTOracle.updateRSETHPrice()`. Observe `rsETHPrice` rises to `T' / rsethSupply'` where `rsethSupply'` includes the attacker's minted tokens.
4. Assert attacker's rsETH value > X ETH and existing holders' per-token value < pre-attack true price — confirming yield transfer.
