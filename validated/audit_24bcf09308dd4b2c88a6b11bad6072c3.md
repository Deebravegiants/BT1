### Title
Gateway Admission Uses Stale `l2_gas_price` While Execution Uses EIP-1559-Adjusted `next_l2_gas_price` — Dual-Price Inconsistency Causes Invalid Admission and Spurious Rejection - (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's `validate_resource_bounds` checks a transaction's `max_l2_gas_price` against the **current committed block's `l2_gas_price`**, while the blockifier executes the transaction against the **next block's `l2_gas_price`** (the EIP-1559-adjusted `next_l2_gas_price` stored in the same block header). These two values diverge every time the network is not at exactly the gas target. The result is a structural dual-price window: the gateway admits transactions that the blockifier will reject, and rejects transactions the blockifier would accept.

---

### Finding Description

**Admission path — gateway reads `l2_gas_price` of the last committed block:**

In `validate_resource_bounds`, the gateway fetches block info via `GatewayFixedBlockSyncStateClient::get_block_info()`, which maps `block_header.l2_gas_price.price_in_fri` into `strk_gas_prices.l2_gas_price`: [1](#0-0) [2](#0-1) 

The code itself carries a `TODO` acknowledging the wrong field is used:

> `// TODO(Arni): getnext_l2_gas_price from the block header.` [3](#0-2) 

**Execution path — batcher uses `next_l2_gas_price` of the last committed block:**

After each `decision_reached`, the consensus orchestrator calls `update_l2_gas_price`, which runs `calculate_next_l2_gas_price_for_fin` (EIP-1559 formula) and stores the result in `self.l2_gas_price`. This value is then placed in `ProposalInit.l2_gas_price_fri` and forwarded to the batcher as the gas price for the **current** block being built: [4](#0-3) [5](#0-4) 

The blockifier's pre-validation then enforces `tx.max_price_per_unit >= block_info.l2_gas_price` (the EIP-1559-adjusted price): [6](#0-5) 

**The two prices are structurally different:**

| Layer | Value used | Source field |
|---|---|---|
| Gateway admission | `committed_block.l2_gas_price` | `BlockHeaderWithoutHash::l2_gas_price` |
| Blockifier execution | `committed_block.next_l2_gas_price` | `BlockHeaderWithoutHash::next_l2_gas_price` (via `ProposalInit`) |

`next_l2_gas_price` is computed by `calculate_next_base_gas_price` and can differ from `l2_gas_price` by up to `1/gas_price_max_change_denominator` per block. With the default `min_gas_price_percentage = 100`, the admission threshold equals `committed_block.l2_gas_price` exactly, while execution requires `committed_block.next_l2_gas_price`. [7](#0-6) 

---

### Impact Explanation

**Case A — price rose (congested previous block):**
`next_l2_gas_price > l2_gas_price`. A transaction with `max_l2_gas_price = l2_gas_price` passes gateway admission but is rejected by the blockifier with `MaxGasPriceTooLow`. The transaction is admitted to the mempool, consumes sequencer resources, and is never included in a block. This is **admission of an invalid transaction**.

**Case B — price fell (uncongested previous block):**
`next_l2_gas_price < l2_gas_price`. A transaction with `max_l2_gas_price` in the range `[next_l2_gas_price, threshold)` is rejected by the gateway but would succeed execution. This is **rejection of a valid transaction before sequencing**.

Both cases match the allowed impact: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

EIP-1559 adjusts the L2 gas price every block whenever actual gas usage differs from the target. Any block that is not exactly at the gas target produces a divergence between `l2_gas_price` and `next_l2_gas_price`. Under normal network load this divergence is present continuously. The window is not a race condition — it is a permanent structural mismatch that affects every block where usage ≠ target.

---

### Recommendation

Replace the read of `block_header.l2_gas_price` in `validate_resource_bounds` with `block_header.next_l2_gas_price`, which is the price the batcher will actually use for the next block. The `BlockHeaderWithoutHash` struct already carries this field: [8](#0-7) 

Concretely, `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` should populate `strk_gas_prices.l2_gas_price` from `block_header.next_l2_gas_price` (in FRI) rather than `block_header.l2_gas_price.price_in_fri`. The `GatewayFixedBlockStateReader` trait may need a dedicated accessor or the `BlockInfo` struct may need a separate field to carry `next_l2_gas_price` without conflating it with the current block's price.

---

### Proof of Concept

1. Observe the last committed block header: `l2_gas_price = P`, `next_l2_gas_price = P'` where `P' > P` (block was above gas target).
2. Submit an `InvokeV3` transaction with `l2_gas.max_price_per_unit = P` (exactly at the gateway threshold with `min_gas_price_percentage = 100`).
3. Gateway calls `validate_resource_bounds` → threshold = `100% * P = P` → `P >= P` → **admitted**.
4. Transaction enters the mempool and is picked up by the batcher for the next block, which uses `l2_gas_price = P'`.
5. Blockifier pre-validation: `max_price_per_unit (P) < actual_gas_price (P')` → `MaxGasPriceTooLow` → **rejected at execution**.
6. Transaction is never included in a block despite passing gateway admission.

The magnitude of the divergence is bounded by `1/gas_price_max_change_denominator` per block (EIP-1559 cap), but at sustained high load the price can drift arbitrarily far above the admission threshold over multiple blocks, making the window larger over time. [9](#0-8)

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
