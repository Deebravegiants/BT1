Audit Report

## Title
Unbounded O(n × m) External-Call Loop in `_getTotalEthInProtocol()` Can Permanently Stale `rsETHPrice` — (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle._getTotalEthInProtocol()` performs a nested external-call loop over every supported asset × every node delegator, executing up to 3 external calls per pair. This function is invoked by the unrestricted public `updateRSETHPrice()`. As the protocol adds assets and node delegators through normal governance, the transaction gas cost grows without bound, eventually reverting and permanently freezing the `rsETHPrice` storage variable.

## Finding Description
`updateRSETHPrice()` is callable by any account with no access restriction beyond `whenNotPaused`:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

`_updateRsETHPrice()` calls `_getTotalEthInProtocol()`, which iterates over every entry in `supportedAssetList` and for each asset makes an external call to `ILRTDepositPool.getTotalAssetDeposits()`:

```solidity
// contracts/LRTOracle.sol L331-349
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    uint256 assetER = getAssetPrice(asset);                                          // 1 external call per asset
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset); // triggers inner loop
    ...
}
```

`getTotalAssetDeposits()` delegates to `getAssetDistributionData()`, which itself loops over the full `nodeDelegatorQueue` making 3 external calls per NDC per asset:

```solidity
// contracts/LRTDepositPool.sol L446-456
uint256 ndcsCount = nodeDelegatorQueue.length;
for (uint256 i; i < ndcsCount;) {
    assetLyingInNDCs += IERC20(asset).balanceOf(nodeDelegatorQueue[i]);
    assetStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset);
    assetUnstakingFromEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset);
    ...
}
```

The total external calls per `updateRSETHPrice()` invocation is `assets × NDCs × 3 + assets`. Both dimensions grow through normal governance: `addNewSupportedAsset` (TIME_LOCK_ROLE) and `addNodeDelegatorContractToQueue` (admin). `maxNodeDelegatorLimit` starts at 10 but has no on-chain upper bound:

```solidity
// contracts/LRTDepositPool.sol L290-296
function updateMaxNodeDelegatorLimit(uint256 maxNodeDelegatorLimit_) external onlyLRTAdmin {
    if (maxNodeDelegatorLimit_ < nodeDelegatorQueue.length) {
        revert InvalidMaximumNodeDelegatorLimit();
    }
    maxNodeDelegatorLimit = maxNodeDelegatorLimit_;
    ...
}
```

No guard exists that caps the product `supportedAssets.length × nodeDelegatorQueue.length`. Once the transaction reverts due to out-of-gas, `rsETHPrice` is never updated again (the write at `rsETHPrice = newRsETHPrice` on L313 is never reached), and all subsequent mints use the stale stored value:

```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

## Impact Explanation
This matches the allowed impact **Medium — Unbounded gas consumption**. Once the gas cost of `_getTotalEthInProtocol()` exceeds the block gas limit, `updateRSETHPrice()` permanently reverts for any caller. The stored `rsETHPrice` becomes stale, causing all subsequent `getRsETHAmountToMint()` calls to compute incorrect rsETH mint amounts — either over-minting (diluting existing holders) or under-minting (cheating depositors), depending on the direction of price drift since the freeze.

## Likelihood Explanation
The trigger (`updateRSETHPrice()`) is callable by any unprivileged external account. The preconditions — a moderate number of supported assets and node delegators — are achieved through normal, legitimate protocol governance operations, not through any adversarial action. At 30 assets × 30 NDCs the loop executes 2,700 external calls; at cold-call costs (~2,500 gas each) that is ~6.75M gas for calls alone, and with warm-storage reads, EigenLayer strategy lookups, and arithmetic overhead the total approaches the 30M block gas limit well within a plausible operational scale. The protocol explicitly supports multiple LSTs and is designed to scale its EigenLayer delegation infrastructure.

## Recommendation
1. Enforce a hard on-chain cap on `supportedAssets.length × nodeDelegatorQueue.length` that accounts for the nested gas cost, rather than capping each dimension independently.
2. Refactor `_getTotalEthInProtocol()` to maintain an incrementally updated `totalETHInProtocol` accumulator updated on deposit/withdrawal/rebalance events, eliminating the full recomputation on every price update.
3. Alternatively, introduce a paginated TVL aggregation pattern where per-asset totals are stored lazily and aggregated off-chain, with the oracle consuming a pre-committed aggregate.

## Proof of Concept
1. Governance adds N supported LST assets via `LRTConfig.addNewSupportedAsset` (TIME_LOCK_ROLE).
2. Admin raises `maxNodeDelegatorLimit` to M via `LRTDepositPool.updateMaxNodeDelegatorLimit` and adds M NDCs via `addNodeDelegatorContractToQueue`.
3. Any unprivileged user calls `LRTOracle.updateRSETHPrice()`.
4. `_getTotalEthInProtocol()` executes `N × M × 3 + N` external calls in a single transaction.
5. At sufficiently large N × M (e.g., 30 × 30 = 2,700 external calls), the transaction exceeds the 30M block gas limit and reverts.
6. `rsETHPrice` is never written; all subsequent calls to `getRsETHAmountToMint()` divide by the stale stored price, producing incorrect rsETH mint amounts for all depositors. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

```text
    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```
