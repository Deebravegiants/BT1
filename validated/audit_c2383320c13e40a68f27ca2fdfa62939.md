### Title
Uninitialized `starknet_version` in `BlockHeaderWithoutHash` Sent to State Sync Produces Wrong Authoritative RPC Block Metadata — (`File: crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs`)

### Summary

`update_state_sync_with_new_block` constructs a `BlockHeaderWithoutHash` using `..Default::default()` for all fields not explicitly listed. The `starknet_version` field — directly available as `init.starknet_version` — is silently left at `StarknetVersion::default()` (the zero/earliest version). Every block committed through the consensus path is stored in state sync with the wrong protocol version. RPC methods that read from state sync return this wrong value as authoritative block metadata, and any node that later syncs from this state sync will feed the wrong `starknet_version` into the batcher's `add_sync_block` path, where it gates the block-hash computation branch.
