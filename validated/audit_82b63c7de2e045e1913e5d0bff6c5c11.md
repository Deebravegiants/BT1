### Title
Caller Identity Omitted from `verify_foreign_transaction` Request Key Enables Unauthorized Signature Acquisition - (File: `crates/contract/src/dto_mapping.rs`)

### Summary

The `verify_foreign_transaction` contract method does not bind the pending request to the caller's NEAR account ID. Unlike `sign` and `request_app_private_key`, which both include `predecessor_account_id` in the request key, `verify_foreign_transaction` stores only `(domain_id, request, payload_version)`. Any unprivileged account that observes a legitimate user's call on-chain can submit an identical request and receive the same MPC threshold signature over the foreign chain attestation payload.

---

### Finding Description

In `crates/contract/src/lib.rs`, the three user-facing request methods handle caller binding differently:

**`sign`** — includes `predecessor` in the request key:
```rust
let request = SignatureRequest::new(
    request.domain_id,
    request.payload,
    &predecessor,   // ← caller bound here
    &request.path,
);
```

**`request_app_private_key`** — includes `predecessor` in the request key:
```rust
let request = CKDRequest::new(
    request.app_public_key,
    domain_id,
    &predecessor,   // ← caller bound here
    &request.derivation_path,
);
```

**`verify_foreign_transaction`** — does **not** include `predecessor`:
```rust
let request = args_into_verify_foreign_tx_request(request);
// predecessor is never passed
```

`args_into_verify_foreign_tx_request` in `crates/contract/src/dto_mapping.rs` (lines 840–848) constructs the key as:

```rust
pub fn args_into_verify_foreign_tx_request(
    args: dtos::VerifyForeignTransactionRequestArgs,
) -> dtos::VerifyForeignTransactionRequest {
    dtos::VerifyForeignTransactionRequest {
        domain_id: args.domain_id,
        request: args.request,
        payload_version: args.payload_version,
        // predecessor_account_id is absent
    }
}
```

The contract stores pending requests in `pending_verify_foreign_tx_requests: LookupMap<VerifyForeignTransactionRequest, Vec<YieldIndex>>`. Because the key does not include the caller's identity, two different callers submitting identical `(domain_id, request, payload_version)` tuples share the same map entry and the same `Vec<YieldIndex>`. When MPC nodes call `respond_verify_foreign_tx`, `resolve_yields_for` resumes **all** yield promises stored under that key, delivering the same signed attestation to every caller who submitted the same request. ### Impact Explanation

**Impact: High** — Cross-chain replay / forged foreign-chain verification / unauthorized signature issuance.

The MPC network produces a threshold ECDSA signature over `ForeignTxSignPayload`, which encodes the verified foreign chain transaction data and extracted values. This signed payload is the authoritative on-chain attestation that a specific foreign chain event occurred. An attacker who obtains this signature without being the original requester can:

1. Present the signed attestation as if they were the authorized requester, breaking the access-control assumption of the `verify_foreign_transaction` flow.
2. Use the signature in any downstream protocol or bridge contract that accepts MPC-signed foreign chain proofs, enabling double-spend or unauthorized bridge execution if the downstream contract does not independently verify the original requester's identity.
3. Drain the deposit paid by the legitimate user (the attacker pays only the minimum 1 yoctoNEAR deposit but receives the same signed output).

The signed payload is verified against the **root public key** (not a per-user derived key) in `respond_verify_foreign_tx`, so the signature is universally valid and not scoped to any particular user. [6](#0-5) [7](#0-6) 

---

### Likelihood Explanation

**Likelihood: High.**

All NEAR transactions are public. An attacker needs only to:
1. Watch the NEAR chain for `verify_foreign_transaction` calls.
2. Extract the `(domain_id, request, payload_version)` arguments from the observed transaction.
3. Submit an identical call before or after the legitimate user's call is processed.

No privileged access, key material, or threshold collusion is required. The attack is fully permissionless and can be automated with a simple NEAR indexer.

---

### Recommendation

Include `predecessor_account_id` in the `VerifyForeignTransactionRequest` key, exactly as `sign` and `request_app_private_key` do. Modify `args_into_verify_foreign_tx_request` (or the call site in `verify_foreign_transaction`) to capture and embed the caller's account ID:

```rust
// In verify_foreign_transaction, after check_request_preconditions returns predecessor:
let (_, predecessor) = self.check_request_preconditions(...);

// Pass predecessor into the request construction:
let request = VerifyForeignTransactionRequest {
    domain_id: args.domain_id,
    request: args.request,
    payload_version: args.payload_version,
    predecessor_account_id: predecessor,  // bind to caller
};
```

This ensures that two different callers submitting the same foreign chain transaction parameters produce distinct pending request keys, so each caller's yield promise is resolved independently and the signed attestation is scoped to the original requester. [1](#0-0) [8](#0-7) 

---

### Proof of Concept

1. **Legitimate user** (`alice.near`) calls `verify_foreign_transaction` with:
   ```json
   {
     "domain_id": 0,
     "request": { "Bitcoin": { "tx_id": "0xABCD...", "confirmations": 6, "extractors": [...] } },
     "payload_version": "V1"
   }
   ```
   The contract stores `YieldIndex_Alice` under key `VerifyForeignTransactionRequest { domain_id: 0, request: Bitcoin{...}, payload_version: V1 }`.

2. **Attacker** (`eve.near`) observes Alice's transaction on-chain and immediately submits the identical call. The contract finds the existing key and appends `YieldIndex_Eve` to the same `Vec<YieldIndex>`.

3. MPC nodes observe one pending request (deduplicated by key). They verify the Bitcoin transaction, compute `ForeignTxSignPayload`, produce a threshold ECDSA signature, and call `respond_verify_foreign_tx`.

4. `resolve_yields_for` iterates the `Vec<YieldIndex>` and resumes **both** `YieldIndex_Alice` and `YieldIndex_Eve` with the same `VerifyForeignTransactionResponse { payload_hash, signature }`.

5. Eve now holds a valid MPC-signed attestation of Alice's Bitcoin transaction, obtained without Alice's consent and without performing the foreign chain verification herself. [9](#0-8) [10](#0-9)

    fn check_request_preconditions(
        &self,
        domain_id: DomainId,
        expected_purpose: DomainPurpose,
        minimum_gas: Gas,
        minimum_deposit: NearToken,
    ) -> (DomainConfig, AccountId) {
        // 1. Look up the domain and check its purpose.
        let domains = match self.protocol_state.domain_registry() {
            Ok(domains) => domains,
            Err(err) => env::panic_str(&err.to_string()),
        };
        let Some(domain_config) = domains.get_domain_by_domain_id(domain_id) else {
            env::panic_str(
                &InvalidParameters::DomainNotFound {
                    provided: domain_id,
                }
                .to_string(),
            );
        };
        if domain_config.purpose != expected_purpose {
            env::panic_str(
                &InvalidParameters::WrongDomainPurpose {
                    domain_id: domain_config.id,
                    expected: expected_purpose,
                    actual: domain_config.purpose,
                }
                .to_string(),
            );
        }
        let domain_config = domain_config.clone();

        // 2. Make sure the call will not run out of gas doing yield/resume logic.
        let prepaid_gas = env::prepaid_gas();
        if prepaid_gas < minimum_gas {
            env::panic_str(
                &InvalidParameters::InsufficientGas {
                    provided: prepaid_gas.as_gas(),
                    required: minimum_gas.as_gas(),
                }
                .to_string(),
            );
        }

        // 3. Require the minimum deposit and refund any excess.
        let predecessor = env::predecessor_account_id();
        require_deposit(minimum_deposit, &predecessor);

        // 4. Refuse the request if the contract is not currently accepting requests
        //    (e.g. because TEE validation has failed).
        if !self.accept_requests {
            env::panic_str(&TeeError::TeeValidationFailed.to_string())
        }

        (domain_config, predecessor)
    }

    pub fn respond_verify_foreign_tx(
        &mut self,
        request: VerifyForeignTransactionRequest,
        response: VerifyForeignTransactionResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!(
            "respond_verify_foreign_tx: signer={}, request={:?}",
            &signer,
            &request
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain.0.into())?;

        let signature_is_valid = match (&response.signature, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");

                let payload_hash: [u8; 32] = response.payload_hash.0;

                // Check the signature is correct against the root public key
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    &payload_hash,
                    &secp_pk,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_verify_foreign_tx_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

fn build_signature_request(
    request: &VerifyForeignTxRequest,
    foreign_tx_payload: &dtos::ForeignTxSignPayload,
) -> anyhow::Result<SignatureRequest> {
    let payload_hash: [u8; ECDSA_PAYLOAD_SIZE_BYTES] =
        foreign_tx_payload.compute_msg_hash()?.into();
    let payload_bytes: BoundedVec<u8, ECDSA_PAYLOAD_SIZE_BYTES, ECDSA_PAYLOAD_SIZE_BYTES> =
        payload_hash.into();

    Ok(SignatureRequest {
        id: request.id,
        receipt_id: request.receipt_id,
        payload: Payload::Ecdsa(payload_bytes),
        tweak: Tweak::new([0u8; 32]),
        entropy: request.entropy,
        timestamp_nanosec: request.timestamp_nanosec,
        domain: request.domain_id,
    })
}
