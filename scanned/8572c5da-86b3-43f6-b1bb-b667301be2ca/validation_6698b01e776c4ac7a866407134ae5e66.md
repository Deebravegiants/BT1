### Title
Reward Tokens Permanently Stuck When `totalKernelStaked` Drops to Zero During Active Reward Period - (File: contracts/KERNEL/KernelDepositPool.sol)

### Summary
Any staker can call `initiateWithdrawal` to immediately reduce `totalKernelStaked` to zero during an active reward period. When `totalKernelStaked == 0`, `rewardPerToken()` freezes and reward tokens that should have been distributed during the zero-staking window are permanently locked in `KernelDepositPool` with no recovery path.

### Finding Description

`KernelDepositPool` is a Synthetix-style staking rewards contract. When `notifyRewardAmount` is called, reward tokens are transferred into the contract and a `rewardRate` is set for the `duration` window.

The `rewardPerToken()` function has a guard:

```solidity
function rewardPerToken() public view returns (uint256) {
    if (totalKernelStaked == 0) {
        return rewardPerTokenStored;  // frozen — no accumulation
    }
    return rewardPerTokenStored + (rewardRate * (lastTimeRewardApplicable() - updatedAt) * DECIMAL_PRECISION)
        / totalKernelStaked;
}
```

When `totalKernelStaked == 0`, `rewardPerTokenStored` does not increase. The `updatedAt` timestamp is also frozen at the last `lastTimeRewardApplicable()` snapshot (set by the `updateReward` modifier). This means that for every second `totalKernelStaked` is zero, `rewardRate` tokens per second are silently lost — they remain in the contract balance but can never be attributed to any staker.

`initiateWithdrawal` immediately decrements `totalKernelStaked`:

```solidity
function initiateWithdrawal(uint256 _amount) external nonReentrant updateReward(msg.sender) {
    ...
    balanceOf[msg.sender] -= _amount;
    totalKernelStaked -= _amount;   // immediate reduction
    ...
}
```

There is no check that a reward period is active, and no mechanism to recover stuck reward tokens. The contract's own NatSpec acknowledges this:

> *"If `totalKernelStaked` ever hits zero during a reward distribution window, any remaining rewards for that period will stay locked in the contract."*

But this is only mitigated by an off-chain deployment assumption — it is not enforced on-chain.

### Impact Explanation

Reward tokens (`rewardsToken`) deposited via `notifyRewardAmount` that correspond to any time interval where `totalKernelStaked == 0` are permanently frozen in `KernelDepositPool`. There is no `sweep`, `recoverERC20`, or admin rescue function for `rewardsToken`. The stuck amount equals `rewardRate × (duration of zero-staking period)` in reward token units.

**Impact: Medium — Permanent freezing of unclaimed yield.**

### Likelihood Explanation

Any staker can trigger this unilaterally by calling `initiateWithdrawal` for their full balance during an active reward period. In a scenario with a single large staker (or coordinated exit), `totalKernelStaked` drops to zero and the remaining reward budget for the period is lost. This is reachable by an unprivileged depositor with no special role.

### Recommendation

Add a recovery function allowing the admin to reclaim undistributed reward tokens after a reward period ends, or enforce that `totalKernelStaked > 0` is maintained during an active reward window (e.g., by reverting `initiateWithdrawal` if it would zero out `totalKernelStaked` while `block.timestamp < finishAt`). Alternatively, track the "lost" reward amount when `totalKernelStaked == 0` and roll it into the next reward period via `notifyRewardAmount`.

### Proof of Concept

1. Admin calls `notifyRewardAmount(1_000e18)` with `duration = 30 days`. `rewardRate = 1_000e18 / 30 days`.
2. Alice is the only staker with `balanceOf[Alice] = 1_000e18`, `totalKernelStaked = 1_000e18`.
3. After 10 days, Alice calls `initiateWithdrawal(1_000e18)`. The `updateReward(Alice)` modifier snapshots her earned rewards correctly for the first 10 days. Then `totalKernelStaked` is set to 0.
4. For the remaining 20 days, `rewardPerToken()` returns the frozen `rewardPerTokenStored`. No staker accumulates rewards.
5. Reward tokens for 20 days (`rewardRate × 20 days ≈ 666e18`) remain in the contract permanently.

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
