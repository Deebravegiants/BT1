### Title
Missing ETH Recovery Mechanism in L1Vault/L1VaultV2 Causes Bridged ETH to Be Frozen When LRTDepositPool Is Unavailable - (File: contracts/L1Vault.sol, contracts/L1VaultV2.sol)

### Summary
`L1Vault` and `L1VaultV2` accept ETH from L2 bridges via `receive()` but provide no emergency ETH withdrawal path. The sole mechanism to consume that ETH — `depositETHForL1VaultETH()` — calls `lrtDepositPool.depositETH`, which can revert due to a pause, a reached deposit cap, or an oracle failure. When any of those conditions hold, all ETH sitting in the vault is frozen with no recovery route.

### Finding Description
Both `L1Vault` and `L1VaultV2` expose a bare `receive()` that accepts ETH forwarded by the L2 bridge infrastructure:

```solidity
// L1Vault.sol line 368 / L1VaultV2.sol line 563
receive() external payable { }
```

The only function that moves this ETH onward is `depositETHForL1VaultETH()`:

```solidity
// L1Vault.sol lines 150-161
function depositETHForL1VaultETH() external payable nonReentrant onlyRole(MANAGER_ROLE) {
    uint256 balanceOfETH = address(this).balance;
    uint256 rsETHAmountToMint = lrtDepositPool.getRsETHAmountToMint(ETH_IDENTIFIER, balanceOfETH);

    if (rsETHAmountToMint == 0) {
        revert InvalidMinRSETHAmountExpected();
    }

    lrtDepositPool.depositETH{ value: balanceOfETH }(rsETHAmountToMint, "");
    ...
}
```

`LRTDepositPool.depositETH` carries a `whenNotPaused` guard and a deposit-limit check:

```solidity
// LRTDepositPool.sol lines 76-93
function depositETH(...) external payable nonReentrant whenNotPaused onlySupportedAsset(LRTConstants.ETH_TOKEN) {
    uint256 rsethAmountToMint = _beforeDeposit(LRTConstants.ETH_TOKEN, msg.value, minRSETHAmountExpected);
    _mintRsETH(rsethAmountToMint);
    ...
}
```

`_beforeDeposit` reverts with `MaximumDepositLimitReached` when the cap is hit, and the oracle path reverts or returns 0 when the oracle is broken. In all these cases `depositETHForL1VaultETH()` reverts, and because neither `L1Vault` nor `L1VaultV2` contains any other ETH-egress function, the vault balance is frozen.

Neither contract has a `withdrawETH`, `rescueETH`, or equivalent function.

### Impact Explanation
ETH bridged from L2 pools (via Stargate/LayerZero or native bridges) accumulates in `L1Vault`/`L1VaultV2`. If `LRTDepositPool` is paused, its deposit cap is reached, or its oracle returns zero, the manager cannot call `depositETHForL1VaultETH()` and there is no alternative path to move the ETH. The funds are frozen for the duration of the blocking condition.

**Impact: Medium — Temporary freezing of funds.**

### Likelihood Explanation
`LRTDepositPool` has an explicit `pause()` function callable by the `PAUSER_ROLE`. Deposit limits are a normal operational parameter. Oracle disruptions are a known risk in any price-feed-dependent system. Any of these routine operational states causes the freeze. The L2 bridge continues to forward ETH to the vault regardless, so the frozen amount grows over time.

### Recommendation
Add an access-restricted ETH recovery function to both `L1Vault` and `L1VaultV2`, analogous to the recommendation in the reference report:

```solidity
function withdrawETH(address payable recipient, uint256 amount)
    external
    nonReentrant
    onlyRole(TIMELOCK_ROLE)
{
    UtilLib.checkNonZeroAddress(recipient);
    (bool success,) = recipient.call{ value: amount }("");
    if (!success) revert EthTransferFailed();
}
```

This allows the protocol to recover bridged ETH when the deposit pool is temporarily unavailable, without requiring a contract upgrade.

### Proof of Concept

1. User deposits ETH on L2 via `RSETHPoolV3ExternalBridge.deposit()`.
2. Bridger calls `bridgeAssets()` or `bridgeAssetsViaNativeBridge()`; ETH arrives at `L1Vault` via `receive()`.
3. Protocol admin pauses `LRTDepositPool` (e.g., for an upgrade or incident response).
4. Manager calls `L1Vault.depositETHForL1VaultETH()` → internally calls `lrtDepositPool.depositETH` → reverts with `Pausable: paused`.
5. No other function in `L1Vault` can move the ETH out.
6. ETH remains frozen in `L1Vault` for the entire duration of the pause, with no recovery path.
