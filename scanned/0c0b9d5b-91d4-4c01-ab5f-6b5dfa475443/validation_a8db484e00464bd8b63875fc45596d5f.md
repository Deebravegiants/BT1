### Title
Nested O(assets × NDCs × withdrawals) Loop in `updateRSETHPrice()` Causes Unbounded Gas Consumption, Risking Permanent Oracle Freeze — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a publicly callable function whose gas cost scales as O(numSupportedAssets × numNDCs × numQueuedEigenLayerWithdrawals). As the protocol accumulates queued EigenLayer withdrawals, the function's gas cost grows proportionally. The protocol's own comments acknowledge that beyond ~120 total queued withdrawals the function fails. If the gas cost exceeds the block gas limit, `updateRSETHPrice()` becomes permanently uncallable, freezing the rsETH price oracle and halting protocol fee minting.

---

### Finding Description

`LRTOracle.updateRSETHPrice()` is `public whenNotPaused` with no role restriction — any address can call it. [1](#0-0) 

It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which loops over every supported asset: [2](#0-1) 

For each asset it calls `ILRTDepositPool.getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)`, which loops over every NDC and calls `getAssetUnstaking(asset)` on each one: [3](#0-2) 

`NodeDelegator.getAssetUnstaking()` makes a fresh external call to EigenLayer's `getQueuedWithdrawals` and then iterates over every queued withdrawal and every strategy within it: [4](#0-3) 

The result is that `getQueuedWithdrawals(NDC_i)` is called **numAssets × numNDCs** times in a single transaction — once per (asset, NDC) pair — even though the result is identical for all assets sharing the same NDC. Total work is O(numAssets × numNDCs × numQueuedWithdrawalsPerNDC), with each iteration performing external EigenLayer calls.

The protocol itself acknowledges the gas concern in `LRTUnstakingVault.setMaxUncompletedWithdrawalCount`: [5](#0-4) 

The comment reads: *"120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price."* The cap is set to 80 as a buffer, but this does not account for the multiplicative effect of the asset count. With 5 assets and 10 NDCs, `getQueuedWithdrawals` is invoked 50 times per `updateRSETHPrice()` call. Each cold external call costs ≥2,100 gas plus EigenLayer's internal computation; at 50 calls with 8 withdrawals each, the cumulative gas easily reaches several million units and grows further as assets or NDCs are added.

Additionally, `setUncompletedWithdrawalCount` (callable by manager to resync the counter after forced undelegations) itself loops over all NDCs and fetches all queued withdrawals: [6](#0-5) 

If forced undelegations cause the actual EigenLayer queued-withdrawal count to exceed the tracked `uncompletedWithdrawalCount`, the soft cap no longer prevents the gas blowup.

---

### Impact Explanation

If `updateRSETHPrice()` exceeds the block gas limit it becomes permanently uncallable:

1. **Permanent freezing of unclaimed yield** — protocol fee minting (rsETH minted to treasury) halts entirely, because fee minting only occurs inside `_updateRsETHPrice()`. [7](#0-6) 

2. **Stale oracle price** — `unlockQueue` in `LRTWithdrawalManager` reads `lrtOracle.rsETHPrice()` to compute payout amounts. A frozen price causes incorrect (potentially under-valued) payouts to withdrawers, constituting a contract failure to deliver promised returns. [8](#0-7) 

**Impact rating: Medium** — Permanent freezing of unclaimed yield; contract fails to deliver promised returns.

---

### Likelihood Explanation

- `updateRSETHPrice()` is publicly callable with no access control.
- The protocol's own comment confirms the gas ceiling is real and was already hit at 120 withdrawals in testing.
- The multiplicative factor (assets × NDCs) means the effective ceiling is lower than the raw withdrawal count suggests. With 10 NDCs and 5 assets, the function performs 50 external EigenLayer calls per invocation.
- Forced undelegations (an acknowledged operational scenario) can push the actual queued-withdrawal count above the soft cap without triggering the `maxUncompletedWithdrawalCount` guard.

**Likelihood rating: Medium.**

---

### Recommendation

Refactor `getAssetUnstaking` calls out of the per-asset loop. Cache `getQueuedWithdrawals(NDC_i)` once per NDC and reuse the result across all assets, reducing external calls from O(assets × NDCs) to O(NDCs). Alternatively, introduce a dedicated view function in `NodeDelegator` that returns unstaking amounts for all supported assets in a single `getQueuedWithdrawals` call, and call it once per NDC inside `getAssetDistributionData`. This mirrors the fix recommended in the reference report: batch the work and commit/flush incrementally rather than accumulating everything in one unbounded pass.

---

### Proof of Concept

Call trace for `updateRSETHPrice()` with 5 assets and 10 NDCs each holding 8 queued withdrawals:

```
updateRSETHPrice()                          // public, no role check
└─ _updateRsETHPrice()
   └─ _getTotalEthInProtocol()
      └─ for asset in [ETH, stETH, ETHx, sfrxETH, X]:   // 5 iterations
            getTotalAssetDeposits(asset)
            └─ getAssetDistributionData(asset)
               └─ for NDC in [NDC0..NDC9]:               // 10 iterations
                     getAssetUnstaking(asset)             // called 50 times total
                     └─ getQueuedWithdrawals(NDC_i)       // 50 external EigenLayer calls
                        └─ for withdrawal in [0..7]:      // 8 × 50 = 400 inner iterations
                              for strategy in withdrawal.strategies: ...
```

Total: 50 external calls + 400+ inner loop iterations per `updateRSETHPrice()` invocation. As assets or NDCs increase, or as forced undelegations push queued withdrawals above the soft cap, this crosses the block gas limit and the oracle is permanently frozen.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
