### Title
Aave WETH Reserve Pause Permanently Freezes All ETH Deposited to Aave — No Functional Emergency Escape Path Exists (`contracts/LRTWithdrawalManager.sol`)

---

### Summary

When the Aave WETH reserve is paused (a normal Aave governance/guardian action), every code path in `LRTWithdrawalManager` that could recover ETH from Aave reverts unconditionally. There is no function that can set `isAaveIntegrationEnabled = false` or return ETH to users without first successfully calling `aaveWETHGateway.withdrawETH`. The result is a permanent, irrecoverable freeze of all ETH held in Aave.

---

### Finding Description

**Root cause — `setAaveIntegrationEnabled(false)` atomically withdraws before disabling:** [1](#0-0) 

When `enabled = false`, the function calls `_collectInterestToTreasury()` and then `_withdrawFromAave(aaveBalance)`. Both ultimately call `aaveWETHGateway.withdrawETH(...)`: [2](#0-1) [3](#0-2) 

If the Aave WETH reserve is paused, both calls revert. The entire `setAaveIntegrationEnabled(false)` transaction reverts, `isAaveIntegrationEnabled` remains `true`, and the flag can never be cleared.

**Root cause — `emergencyWithdrawFromAave` is not a true emergency path:** [4](#0-3) 

The function unconditionally calls `_collectInterestToTreasury()` before attempting the principal withdrawal. If there is any accrued interest (`aaveBalance > principal`), `_collectInterestToTreasury()` calls `aaveWETHGateway.withdrawETH` and reverts when the pool is paused. Even if there is no interest, `_withdrawFromAave(amount)` itself calls `withdrawETH` and reverts. There is no try/catch and no way to skip the interest collection step.

**Root cause — `completeWithdrawal` for ETH also reverts:** [5](#0-4) 

When `isAaveIntegrationEnabled == true` and the contract's native ETH balance is insufficient, `_withdrawFromAave` is called and reverts. The user's withdrawal request is deleted from storage at line 712 before this check, meaning the request is consumed but the ETH is never delivered. [6](#0-5) 

**No administrative override exists.** There is no function that sets `isAaveIntegrationEnabled = false` without first withdrawing, and no function that bypasses `aaveWETHGateway.withdrawETH` entirely.

---

### Impact Explanation

All ETH deposited to Aave via `_depositToAave` is permanently inaccessible:

- `setAaveIntegrationEnabled(false)` → reverts (cannot disable integration)
- `emergencyWithdrawFromAave(amount)` → reverts (calls `withdrawETH` internally)
- `completeWithdrawal(ETH)` → reverts with `InsufficientLiquidityForWithdrawal` (or deletes the request and fails to pay)
- `configureAaveIntegration(...)` → also calls `_collectInterestToTreasury` + `_withdrawFromAave` before reconfiguring, same revert

**Impact: Critical — Permanent freezing of funds.** The full `totalETHDepositedToAave` balance is locked with no recovery path.

---

### Likelihood Explanation

Aave v3 reserves can be paused by the Aave Guardian (a multisig) or by governance without any compromise of the LRT-rsETH protocol. This is a documented, exercised feature of Aave (e.g., the March 2023 CRV incident triggered reserve pauses). The precondition — ETH deposited to Aave with `isAaveIntegrationEnabled = true` — is the normal operating state of the integration. No attacker action is required; a routine Aave guardian pause is sufficient.

---

### Recommendation

1. **Decouple the flag from the withdrawal in `setAaveIntegrationEnabled(false)`**: Allow the flag to be set to `false` independently of whether the withdrawal succeeds. Wrap the withdrawal in a try/catch or add a separate `forceDisableAaveIntegration()` that only sets the flag.

2. **Fix `emergencyWithdrawFromAave`**: Wrap `_collectInterestToTreasury()` in a try/catch so that a failed interest collection does not block the principal emergency withdrawal.

3. **Fix `_processWithdrawalCompletion`**: If `_withdrawFromAave` reverts (pool paused), either revert before deleting the request, or wrap in try/catch and fall back gracefully.
