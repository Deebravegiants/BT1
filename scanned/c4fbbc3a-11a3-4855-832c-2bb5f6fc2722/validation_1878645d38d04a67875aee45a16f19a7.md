### Title
Permanent Freezing of ETH Withdrawal Funds for Non-ETH-Receiving Contract Users - (File: contracts/LRTWithdrawalManager.sol)

### Summary
If a user initiates an ETH withdrawal from a contract address that cannot receive ETH (e.g., a smart wallet, multisig, or contract without a `receive()` function), the withdrawal becomes permanently frozen after rsETH is burned in `unlockQueue`. There is no mechanism to redirect, cancel, or recover the stuck ETH.

### Finding Description
The withdrawal lifecycle in `LRTWithdrawalManager` proceeds in two operator-gated phases:

**Phase 1 — `initiateWithdrawal`:** The user's rsETH is transferred into the contract and a withdrawal request is recorded.

**Phase 2 — `unlockQueue` (operator-only):** rsETH held by the contract is burned and the corresponding ETH is pulled from `LRTUnstakingVault` into `LRTWithdrawalManager`. [1](#0-0) 

Only after both phases does the user call `completeWithdrawal`, which internally calls `_processWithdrawalCompletion`: [2](#0-1) 

The final step is `_transferAsset`, which for ETH performs a low-level call: [3](#0-2) 

If `payable(to).call{ value: amount }("")` fails (because `to` is a contract that rejects ETH), the function reverts with `EthTransferFailed()`. Because Solidity reverts roll back all state changes, the `popFront()` and `delete withdrawalRequests[requestId]` are also rolled back — the withdrawal record is restored. However, **the rsETH was already burned in Phase 2 (`unlockQueue`)**, and there is no function to:
- Redirect the withdrawal to a different address,
- Cancel the withdrawal and return rsETH,
- Or recover the ETH while the user's withdrawal remains in the unlocked queue.

The `sweepRemainingAssets` function explicitly blocks execution while any unlocked withdrawal exists: [4](#0-3) 

Both `completeWithdrawal` (sends to `msg.sender`) and `completeWithdrawalForUser` (sends to the original `user` address) route ETH to the same non-receiving address, so even operator intervention cannot unblock the withdrawal. [5](#0-4) 

### Impact Explanation
**Critical — Permanent freezing of funds.**

The user's rsETH is burned (irreversible) during `unlockQueue`. The corresponding ETH is allocated to the user's withdrawal in `LRTWithdrawalManager` but can never be claimed. There is no cancellation, redirection, or admin recovery path. The ETH is permanently frozen in the contract.

### Likelihood Explanation
**Low-Medium.** Smart contracts that interact with DeFi protocols are common (smart wallets, multisigs, protocol-owned treasuries, aggregators). Many such contracts do not implement a `receive()` function or intentionally revert on unexpected ETH receipt. Any such contract that calls `initiateWithdrawal` for ETH will have its funds permanently frozen once the operator runs `unlockQueue`.

### Recommendation
1. Add a `cancelWithdrawal` function that allows users to reclaim their rsETH before `unlockQueue` is called.
2. After `unlockQueue`, allow users to specify an alternative recipient address for their ETH withdrawal.
3. Alternatively, if ETH transfer fails, wrap the ETH as WETH and send WETH to the user instead, ensuring the transfer always succeeds.

### Proof of Concept
1. A smart contract `VictimContract` (no `receive()` function) calls `initiateWithdrawal(ETH_TOKEN, rsETHAmount, "")`. rsETH is transferred to `LRTWithdrawalManager`.
2. Operator calls `unlockQueue(ETH_TOKEN, ...)`. rsETH held by the manager is burned; ETH is moved from `LRTUnstakingVault` to `LRTWithdrawalManager`.
3. `VictimContract` calls `completeWithdrawal(ETH_TOKEN, "")`. Inside `_processWithdrawalCompletion`, `payable(VictimContract).call{ value: amount }("")` returns `(false, ...)`. The function reverts with `EthTransferFailed()`.
4. Every subsequent call to `completeWithdrawal` or `completeWithdrawalForUser` for this user also reverts.
5. rsETH is burned (gone). ETH is stuck in `LRTWithdrawalManager`. `sweepRemainingAssets` cannot be called because `hasUnlockedWithdrawals` returns `true`. Funds are permanently frozen.

```text
    function _processWithdrawalCompletion(address asset, address user, string calldata referralId) internal {
        if (userAssociatedNonces[asset][user].empty()) {
            revert NoWithdrawalRequests(user, asset);
        }

        // Retrieve and remove the oldest withdrawal request for the user.
        uint256 usersFirstWithdrawalRequestNonce = userAssociatedNonces[asset][user].popFront();
        // Ensure the request is already unlocked.
        if (usersFirstWithdrawalRequestNonce >= nextLockedNonce[asset]) revert WithdrawalLocked();

        bytes32 requestId = getRequestId(asset, usersFirstWithdrawalRequestNonce);
        WithdrawalRequest memory request = withdrawalRequests[requestId];

        delete withdrawalRequests[requestId];

        // Check that the withdrawal delay has passed since the request's initiation.
        if (block.number < request.withdrawalStartBlock + withdrawalDelayBlocks) revert WithdrawalDelayNotPassed();

        unlockedWithdrawalsCount[asset]--;

        // If Aave integration is enabled and asset is ETH, withdraw from Aave if needed
        if (isAaveIntegrationEnabled && asset == LRTConstants.ETH_TOKEN) {
            uint256 contractBalance = address(this).balance;
            if (contractBalance < request.expectedAssetAmount) {
                uint256 amountNeeded = request.expectedAssetAmount - contractBalance;
                _withdrawFromAave(amountNeeded);

                // Verify we have sufficient balance after withdrawal
                uint256 balanceAfter = address(this).balance;
                if (balanceAfter < request.expectedAssetAmount) {
                    revert InsufficientLiquidityForWithdrawal();
                }
            }
        }

        _transferAsset(asset, user, request.expectedAssetAmount);

        emit ReferralIdEmitted(referralId);
        emit AssetWithdrawalFinalized(user, asset, request.rsETHUnstaked, request.expectedAssetAmount);
    }
```

```text
    function _transferAsset(address asset, address to, uint256 amount) internal {
        if (asset == LRTConstants.ETH_TOKEN) {
            (bool sent,) = payable(to).call{ value: amount }("");
            if (!sent) revert EthTransferFailed();
        } else {
            IERC20(asset).safeTransfer(to, amount);
        }
    }
```
