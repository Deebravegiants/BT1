### Title
Permanent Freezing of Unclaimed Yield via `highestRsethPrice` One-Way Ratchet With No Reset Path — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle._updateRsETHPrice()` maintains `highestRsethPrice` as a strictly monotone-increasing ratchet. `SwETHPriceOracle.getAssetPrice()` forwards `swETH.getRate()` with no staleness or bounds check. A transient spike in `swETH.getRate()` permanently elevates `highestRsethPrice`; once the rate normalises every subsequent public and manager-level price update computes `newRsETHPrice < highestRsethPrice` by more than `pricePercentageLimit`, triggering the auto-pause cascade. No admin function exists to reset `highestRsethPrice`, so the protocol cannot self-recover without a contract upgrade.

---

### Finding Description

**`SwETHPriceOracle.getAssetPrice()` — no staleness or bounds check**

```solidity
// contracts/oracles/SwETHPriceOracle.sol  L34-40
function getAssetPrice(address asset) external view returns (uint256) {
    if (asset != swETHAddress) revert InvalidAsset();
    return ISwETH(swETHAddress).getRate();   // raw, unchecked
}
```

The return value is consumed directly by `_getTotalEthInProtocol()`, which feeds `newRsETHPrice`. [1](#0-0) 

**`highestRsethPrice` ratchet — upward only, no reset**

```solidity
// contracts/LRTOracle.sol  L294-296
if (newRsETHPrice > highestRsethPrice) {
    highestRsethPrice = newRsETHPrice;   // only ever increases
}
``` [2](#0-1) 

**Auto-pause on downside breach — returns early, never lowers `highestRsethPrice`**

```solidity
// contracts/LRTOracle.sol  L270-281
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;   // exits without updating highestRsethPrice
    }
    ...
}
``` [3](#0-2) 

**`updateRSETHPriceAsManager()` does not bypass the ratchet**

The manager escape-hatch calls the same `_updateRsETHPrice()` and hits the same early-return path; it cannot lower `highestRsethPrice` either. [4](#0-3) 

**`unpause()` only unpauses `LRTOracle`**

```solidity
// contracts/LRTOracle.sol  L143-146
function unpause() external whenPaused onlyLRTAdmin {
    paused = false;
    emit Unpaused(msg.sender);
}
```

It does not unpause `lrtDepositPool` or `withdrawalManager`, and even if all three are manually unpaused, the very next `updateRSETHPrice()` call re-triggers the cascade because `highestRsethPrice` is still inflated. [5](#0-4) 

No function in `LRTOracle` accepts a new value for `highestRsethPrice`. Recovery requires a proxy upgrade. [6](#0-5) 

---

### Impact Explanation

Once `highestRsethPrice` is set to the spike value:

- Every call to `updateRSETHPrice()` (public) hits the auto-pause branch and returns early — fee distributions and yield accrual stop permanently.
- `updateRSETHPriceAsManager()` produces the same outcome.
- Pending withdrawal yield and protocol fee minting are frozen indefinitely.
- Recovery requires a governance-approved proxy upgrade, which is not a self-healing path.

Impact: **Medium — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

The trigger is a transient spike in `swETH.getRate()`. This does not require an active attacker: a bug in Swell's internal rate accounting (e.g., a miscalculated rebase, a rounding error in their oracle, or a short-lived misconfiguration) is sufficient. The absence of any staleness or bounds check in `SwETHPriceOracle` means the raw value is accepted unconditionally. The same risk applies to any other asset oracle registered in the protocol.

The root cause is not solely the external oracle — it is the combination of the unchecked oracle read **and** the irreversible ratchet design inside LRT-rsETH's own contracts.

Likelihood: **Low** (requires an oracle anomaly), but the consequence is permanent and unrecoverable without an upgrade.

---

### Recommendation

1. **Add a staleness / bounds check in `SwETHPriceOracle.getAssetPrice()`** — reject any rate that deviates more than a configurable percentage from the last accepted value or that is older than a heartbeat threshold.

2. **Add an admin reset for `highestRsethPrice`** — expose a `setHighestRsethPrice(uint256)` guarded by `onlyLRTAdmin` so the protocol can recover without a proxy upgrade.

3. **Separate the ratchet update from the auto-pause path** — consider updating `highestRsethPrice` only after a multi-block TWAP confirmation rather than on a single price observation.

---

### Proof of Concept

```solidity
// State-test (local fork or mock environment)
// 1. Deploy LRTOracle with swETH oracle pointing to a mock ISwETH.
// 2. Set pricePercentageLimit = 5e16 (5%).
// 3. Seed rsETH supply and initial price so highestRsethPrice = 1.04e18.

// 4. Mock swETH.getRate() returns 1.5e18 (spike).
mockSwETH.setRate(1.5e18);
lrtOracle.updateRSETHPrice();
// highestRsethPrice is now ~1.44e18 (inflated by the spike)

// 5. Mock swETH.getRate() returns to normal 1.04e18.
mockSwETH.setRate(1.04e18);
lrtOracle.updateRSETHPrice();
// diff = 1.44e18 - 1.00e18 ≈ 28% > 5% → auto-pause fires, returns early
// Assert: lrtDepositPool.paused() == true
// Assert: withdrawalManager.paused() == true
// Assert: lrtOracle.paused() == true
// Assert: lrtOracle.highestRsethPrice() == <spike value> (unchanged)

// 6. Admin unpauses all three contracts manually.
// 7. Call updateRSETHPrice() again.
// → Same auto-pause fires immediately. Protocol is permanently frozen.
// Assert: lrtOracle.paused() == true  (re-paused)
```
