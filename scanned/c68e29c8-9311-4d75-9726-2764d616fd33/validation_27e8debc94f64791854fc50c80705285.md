### Title
Immutable Chainlink Feed Address in `RSETHPriceFeed` Cannot Be Updated If Feed Is Deprecated — (File: contracts/oracles/RSETHPriceFeed.sol)

---

### Summary

`RSETHPriceFeed.sol` stores both the Chainlink ETH/USD aggregator (`ETH_TO_USD`) and the rsETH oracle (`RS_ETH_ORACLE`) as `public immutable` variables in a non-upgradeable, ownerless contract. There are no setter functions and no upgrade path. If Chainlink deprecates the ETH/USD feed (a documented historical practice), the contract is permanently broken with no on-chain recovery mechanism.

---

### Finding Description

`RSETHPriceFeed` is a non-upgradeable contract that implements the `AggregatorV3Interface` to expose an rsETH/USD price feed for external DeFi integrations (e.g., Morph, per the README deployment table).

Both critical external addresses are locked at construction:

```solidity
// contracts/oracles/RSETHPriceFeed.sol
AggregatorV3Interface public immutable ETH_TO_USD;   // line 28
IRSETHOracle          public immutable RS_ETH_ORACLE; // line 31
``` [1](#0-0) 

They are set once in the constructor and can never be changed:

```solidity
constructor(address ethToUSDAggregatorAddress, address rsETHOracle, string memory description_) {
    ETH_TO_USD = AggregatorV3Interface(ethToUSDAggregatorAddress);
    RS_ETH_ORACLE = IRSETHOracle(rsETHOracle);
    ...
}
``` [2](#0-1) 

Both `latestRoundData()` and `getRoundData()` delegate entirely to these two immutable addresses:

```solidity
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
``` [3](#0-2) 

The contract has no `owner`, no `admin`, no `Ownable`, no `AccessControl`, and no upgrade proxy. There is no function to update either address. [4](#0-3) 

By contrast, every other contract in the protocol that stores external addresses (pool contracts, `L1VaultV2`, etc.) provides explicit setter functions protected by `TIMELOCK_ROLE` or `onlyOwner`. For example, `RSETHPoolV3` has `setRSETHOracle`, `setStargatePool`, `setL1VaultETHForL2Chain`, etc. [5](#0-4) 

---

### Impact Explanation

Chainlink has a documented history of deprecating price feed contracts and deploying replacements at new addresses. If the ETH/USD Chainlink feed pointed to by `ETH_TO_USD` is deprecated:

- `latestRoundData()` on the old feed will either revert or return a permanently stale answer.
- `RSETHPriceFeed.latestRoundData()` and `getRoundData()` will consequently revert or return a wrong rsETH/USD price.
- Any external DeFi protocol (e.g., on Morph) that has integrated `RSETHPriceFeed` as its Chainlink-compatible oracle for rsETH will receive broken price data.
- rsETH holders using those external protocols (as collateral, in AMMs, etc.) will be exposed to mispricing — potentially triggering incorrect liquidations or preventing legitimate operations.

There is no on-chain recovery path. The only fix is to deploy a new `RSETHPriceFeed` contract and coordinate all external integrators to migrate — an off-chain, social coordination problem with no guarantee of success.

**Impact classification**: Low — Contract fails to deliver promised returns (accurate rsETH/USD price), but no funds are directly lost within the LRT-rsETH protocol itself.

---

### Likelihood Explanation

Chainlink has deprecated ETH/USD feeds on multiple networks in the past. The ETH/USD feed is high-priority and unlikely to be deprecated on mainnet in the near term, but the risk is non-zero over the protocol's lifetime. The `RS_ETH_ORACLE` immutable is lower risk since it points to the `LRTOracle` upgradeable proxy whose address is stable. The primary risk vector is `ETH_TO_USD`.

---

### Recommendation

Replace both `immutable` declarations with mutable state variables and add owner-gated setter functions:

```solidity
AggregatorV3Interface public ethToUSD;
IRSETHOracle public rsETHOracle;

address public owner;

modifier onlyOwner() {
    require(msg.sender == owner, "Not owner");
    _;
}

function setEthToUSD(address _ethToUSD) external onlyOwner {
    ethToUSD = AggregatorV3Interface(_ethToUSD);
    emit EthToUSDUpdated(_ethToUSD);
}

function setRsETHOracle(address _rsETHOracle) external onlyOwner {
    rsETHOracle = IRSETHOracle(_rsETHOracle);
    emit RsETHOracleUpdated(_rsETHOracle);
}
```

Alternatively, convert `RSETHPriceFeed` to an upgradeable proxy pattern consistent with the rest of the protocol.

---

### Proof of Concept

1. `RSETHPriceFeed` is deployed with `ETH_TO_USD = <Chainlink ETH/USD feed>` and `RS_ETH_ORACLE = <LRTOracle proxy>`. [2](#0-1) 
2. Chainlink deprecates the ETH/USD feed at that address and deploys a new one.
3. Any caller (external protocol, keeper, user) calls `RSETHPriceFeed.latestRoundData()`.
4. The call delegates to the deprecated `ETH_TO_USD.latestRoundData()`, which either reverts or returns a stale `updatedAt` timestamp.
5. The external protocol receives a broken or stale rsETH/USD price.
6. No admin function exists in `RSETHPriceFeed` to point to the new feed. The contract is permanently broken. [6](#0-5)
