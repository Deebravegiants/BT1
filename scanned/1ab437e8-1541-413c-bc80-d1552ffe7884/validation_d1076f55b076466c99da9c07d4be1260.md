### Title
Permanent `stakedButUnverifiedNativeETH` Overstatement When Beacon Chain Validator Is Slashed Before Credential Verification — (`contracts/NodeDelegator.sol`)

---

### Summary

`stakedButUnverifiedNativeETH` can only be decremented via `verifyWithdrawalCredentials`. If a validator is slashed on the beacon chain and exits before that call is made, `verifyWithdrawalCredentials` reverts (EigenLayer rejects proofs for exited validators), leaving the counter permanently inflated. `getEffectivePodShares` then returns phantom ETH, which propagates into `getTotalAssetDeposits` → `_getTotalEthInProtocol` → `rsETHPrice`, inflating the price and making the protocol insolvent.

---

### Finding Description

**Increment path — `stake32Eth`:** [1](#0-0) 

`stakedButUnverifiedNativeETH += 32 ether` is written unconditionally when the operator stakes.

**Sole decrement path — `verifyWithdrawalCredentials`:** [2](#0-1) 

The counter is only reduced here. There is no admin setter, no emergency reset, and no alternative decrement path anywhere in the codebase.

**`getEffectivePodShares` adds both terms:** [3](#0-2) 

`withdrawableShare` is fetched from `DelegationManager.getWithdrawableShares`. Before `verifyWithdrawalCredentials` is called, EigenLayer has no record of the validator, so this returns 0. The function therefore returns `stakedButUnverifiedNativeETH + 0`.

**`IShareManager.stakerDepositShares` explicitly returns 0 for negative/zero shares:** [4](#0-3) 

**TVL propagation:**

`getEffectivePodShares` feeds `getETHDistributionData`: [5](#0-4) 

which feeds `getTotalAssetDeposits` → `_getTotalEthInProtocol`: [6](#0-5) 

which drives `rsETHPrice = totalETHInProtocol / rsethSupply`.

**Scenario:**

1. Operator calls `stake32Eth` → `stakedButUnverifiedNativeETH = 32 ether`.
2. Validator is slashed on the beacon chain and exits (exit epoch set) before credentials are verified.
3. Operator attempts `verifyWithdrawalCredentials` — EigenLayer's `EigenPod` reverts because the validator's exit epoch is no longer `FAR_FUTURE_EPOCH` (active validators only).
4. `stakedButUnverifiedNativeETH` is permanently stuck at 32 ether.
5. `getEffectivePodShares` returns 32 ether of phantom ETH.
6. `updateRSETHPrice` inflates `rsETHPrice` by the phantom 32 ETH.

---

### Impact Explanation

The inflated `rsETHPrice` means:
- New depositors receive fewer rsETH tokens than they are entitled to (they pay the inflated price).
- Existing rsETH holders can redeem at the inflated price, draining real ETH that backs other holders.
- The last withdrawers cannot be made whole — the protocol is insolvent by exactly 32 ETH per slashed-before-verification validator.

The downside-protection circuit breaker in `LRTOracle._updateRsETHPrice` (lines 270–281) only triggers on price *decreases*; an inflated price passes through unchecked. [7](#0-6) 

---

### Likelihood Explanation

Beacon chain validator slashings are rare but real on-chain events. The window between `stake32Eth` and `verifyWithdrawalCredentials` can span hours to days (off-chain proof generation). Any validator slashed during that window triggers this permanently unrecoverable state. No attacker action is required — the slashing is a standard beacon chain event.

---

### Recommendation

Add an operator-callable (or admin-callable) function to manually correct `stakedButUnverifiedNativeETH` for validators that have been slashed and exited before credential verification, e.g.:

```solidity
function decreaseStakedButUnverifiedNativeETH(uint256 amount) external onlyLRTManager {
    if (amount > stakedButUnverifiedNativeETH) revert InsufficientStakedBalance();
    stakedButUnverifiedNativeETH -= amount;
}
```

Alternatively, integrate a checkpoint-based proof path that can mark a pre-verification validator as slashed and zero out its contribution to `stakedButUnverifiedNativeETH`.

---

### Proof of Concept

```solidity
// Fork test (Holesky or local EigenLayer fork)
function test_phantomETH_slashedBeforeVerification() public {
    // 1. Operator stakes 32 ETH
    vm.prank(operator);
    nodeDelegator.stake32Eth(pubkey, sig, depositDataRoot);
    assertEq(nodeDelegator.stakedButUnverifiedNativeETH(), 32 ether);

    // 2. Simulate beacon chain slash: validator exits (exit epoch set)
    //    verifyWithdrawalCredentials now reverts for this validator
    vm.prank(operator);
    vm.expectRevert(); // EigenPod: validator must be active
    nodeDelegator.verifyWithdrawalCredentials(ts, srp, indices, vfProofs, vFields);

    // 3. stakedButUnverifiedNativeETH is still 32 ether
    assertEq(nodeDelegator.stakedButUnverifiedNativeETH(), 32 ether);

    // 4. getEffectivePodShares returns phantom 32 ETH
    assertEq(nodeDelegator.getEffectivePodShares(), 32 ether);

    // 5. rsETH price is inflated by 32 ETH of phantom backing
    uint256 priceBefore = lrtOracle.rsETHPrice();
    lrtOracle.updateRSETHPrice();
    assertGt(lrtOracle.rsETHPrice(), actualFairPrice);
}
```
