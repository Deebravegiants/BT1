### Title
Missing Chainlink `latestRoundData()` Output Validation Enables Stale/Invalid Price Acceptance - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all return values except `price`, performing no staleness, completeness, or sign checks. A stale or negative price propagates directly into `LRTOracle._updateRsETHPrice()`, corrupting the global `rsETHPrice` used to mint rsETH for every depositor.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches the Chainlink price as follows:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

Three validations are absent:

1. **No staleness check** — `answeredInRound >= roundId` is never verified. A price carried over from a prior round is silently accepted.
2. **No incomplete-round check** — `updatedAt != 0` is never verified.
3. **No sign check** — `price > 0` is never verified. In Solidity 0.8.x, an explicit `int256 → uint256` cast does **not** revert; `uint256(-1)` wraps to `2^256 - 1`, and the subsequent `* 1e18` arithmetic overflow reverts the entire `updateRSETHPrice()` call, causing a DoS.

By contrast, the team already applies all three checks correctly in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

The unvalidated price flows through:

- `LRTOracle.getAssetPrice()` → `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)` [3](#0-2) 
- `LRTOracle._getTotalEthInProtocol()` → iterates all supported assets, accumulating `totalETHInProtocol` [4](#0-3) 
- `LRTOracle._updateRsETHPrice()` → computes `newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply)` and writes it to `rsETHPrice` [5](#0-4) 
- `LRTDepositPool.getRsETHAmountToMint()` → `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()` [6](#0-5) 

`updateRSETHPrice()` is a public, permissionless function callable by any address. [7](#0-6) 

---

### Impact Explanation

**Stale price scenario (High — theft of unclaimed yield):**
If a Chainlink feed enters a stale state (e.g., during network congestion or a sequencer outage), the last reported price may be significantly lower than the true current price. Any caller can invoke `updateRSETHPrice()` with the stale price, setting `rsETHPrice` artificially low. Depositors who then call `depositAsset()` / `depositETH()` receive more rsETH than the true exchange rate warrants, diluting the holdings of all existing rsETH holders — a direct theft of their accrued yield.

**Negative/zero price scenario (Medium — temporary DoS):**
A Chainlink feed returning `price <= 0` causes `uint256(price)` to wrap to a near-`2^256` value; the subsequent `* 1e18` overflows and reverts. Every call to `updateRSETHPrice()` reverts until the feed recovers, freezing the protocol's ability to update its exchange rate.

---

### Likelihood Explanation

Chainlink feeds have historically returned stale data during network congestion, sequencer downtime (on L2s), or oracle disruptions. The `updateRSETHPrice()` function is public and permissionless, so any actor — including one who benefits from a stale price — can trigger the update at the worst possible moment. No special access is required.

---

### Recommendation

Apply the same three-check pattern already used in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    require(answeredInRound >= roundId, "Chainlink: stale price");
    require(updatedAt != 0,            "Chainlink: incomplete round");
    require(price > 0,                 "Chainlink: invalid price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

---

### Proof of Concept

1. A Chainlink LST/ETH feed (e.g., stETH/ETH) enters a stale state; `answeredInRound < roundId` and `updatedAt` is old.
2. Attacker (or any user) calls `LRTOracle.updateRSETHPrice()` (public, no access control).
3. `_getTotalEthInProtocol()` calls `ChainlinkPriceOracle.getAssetPrice(stETH)`.
4. `latestRoundData()` returns the stale, artificially low price; no check rejects it.
5. `totalETHInProtocol` is underestimated; `newRsETHPrice` is set below the true value and written to `rsETHPrice`. [8](#0-7) 
6. Attacker immediately calls `LRTDepositPool.depositAsset()`. `getRsETHAmountToMint()` divides by the deflated `rsETHPrice`, minting more rsETH than the deposited assets are worth. [6](#0-5) 
7. Existing rsETH holders are diluted; the attacker has extracted yield that belonged to them.
