### Title
`removeSupportedToken` Blocked by Dust Token Transfer — (`contracts/pools/RSETHPoolV3.sol`, `contracts/pools/RSETHPoolV3ExternalBridge.sol`)

### Summary
Both `RSETHPoolV3.removeSupportedToken` and `RSETHPoolV3ExternalBridge.removeSupportedToken` gate token delisting on an exact `balanceOf(address(this)) != 0` check. Because any external actor can transfer a dust amount of the token to the pool contract, this check can be made to revert permanently, preventing the protocol from ever delisting a compromised or depegged asset.

### Finding Description
`RSETHPoolV3.removeSupportedToken` contains the following guard:

```solidity
if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();
```

The intent is to ensure no residual balance remains before delisting. However, `balanceOf` reflects the live on-chain balance, which any token holder can inflate by sending even 1 wei to the pool. The same pattern is present verbatim in `RSETHPoolV3ExternalBridge.removeSupportedToken`.

There is no mechanism in either contract to drain an arbitrary token balance to zero before calling `removeSupportedToken`. The `moveAssetsForBridging` and `bridgeTokens` functions only move the balance *minus* the tracked fee (`getTokenBalanceMinusFees`), so a dust donation that exceeds the fee accounting is not reachable through normal operational paths. The `withdrawFees` function only withdraws the tracked fee amount. No sweep function exists for arbitrary token residue.

### Impact Explanation
If a supported LST (e.g., wstETH on a pool that supports it) becomes compromised, depegged, or must be emergency-delisted, the protocol's only removal path is `removeSupportedToken`. An attacker who holds even 1 wei of that token can permanently block this call. While the token remains listed, users can continue depositing it and receiving freshly minted `wrsETH` at the oracle-quoted rate. If the token's true value has collapsed, every such deposit dilutes the backing of all outstanding `wrsETH`, leading to protocol insolvency for existing holders.

**Impact: Critical — Protocol insolvency / permanent inability to delist a compromised asset.**

### Likelihood Explanation
Any holder of the token (including the token's own deployer or a malicious actor who acquired dust on the open market) can execute this with a single ERC-20 `transfer` call. No special role or privilege is required. The attacker can monitor the mempool for a `removeSupportedToken` transaction and front-run it, or simply pre-donate before the admin acts. The cost is negligible (1 wei of the token + gas).

### Recommendation
Replace the live `balanceOf` check with a storage-tracked variable that records only protocol-received deposits (analogous to the short-term fix in the reference report). Alternatively, add a privileged `sweepToken(address token, address recipient)` function that allows the admin to drain any residual balance before delisting, so the `balanceOf` check can be satisfied regardless of external donations.

### Proof of Concept
1. `wstETH` is a supported token in `RSETHPoolV3ExternalBridge` (added via `addSupportedToken`).
2. The token becomes depegged; the protocol calls `removeSupportedToken(wstETH, index)`.
3. Eve front-runs with `IERC20(wstETH).transfer(poolAddress, 1)`.
4. Inside `removeSupportedToken`, `IERC20(wstETH).balanceOf(address(this))` returns `1 != 0`, triggering `revert TokenBalanceNotZero()`.
5. The call reverts. Eve repeats step 3 on every subsequent attempt.
6. `wstETH` remains a supported deposit token indefinitely; users continue minting `wrsETH` against a worthless asset, draining protocol solvency.

### Citations

**File:** contracts/pools/RSETHPoolV3.sol (L559-568)
```text
    function removeSupportedToken(address token, uint256 tokenIndex) external onlyRole(TIMELOCK_ROLE) {
        UtilLib.checkNonZeroAddress(token);
        if (supportedTokenList[tokenIndex] != token) revert TokenNotFoundError();
        if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();

        delete supportedTokenOracle[token];
        supportedTokenList[tokenIndex] = supportedTokenList[supportedTokenList.length - 1];
        supportedTokenList.pop();
        emit RemovedSupportedToken(token);
    }
```
