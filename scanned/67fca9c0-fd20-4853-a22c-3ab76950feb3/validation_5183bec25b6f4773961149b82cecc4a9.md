### Title
Immutable `i_maxSupply` Cap in `WrappedRSETH` Permanently Blocks CCIP Yield Minting When Supply Is Exhausted — (`contracts/ccip/WrappedRSETH.sol`)

---

### Summary

`WrappedRSETH` enforces an immutable supply cap (`i_maxSupply`) set at deployment. Once `totalSupply()` reaches this cap, every call to `mint()` reverts with `MaxSupplyExceeded`. Because `i_maxSupply` is declared `immutable`, it can never be raised. Any in-flight CCIP message that calls `mint()` to deliver bridged rsETH (including accrued yield) will fail on execution and, since the cap cannot be lifted, will remain permanently unexecutable.

---

### Finding Description

`WrappedRSETH.mint()` enforces the cap unconditionally:

```solidity
// contracts/ccip/WrappedRSETH.sol  line 138
if (i_maxSupply != 0 && totalSupply() + amount > i_maxSupply)
    revert MaxSupplyExceeded(totalSupply() + amount);
``` [1](#0-0) 

`i_maxSupply` is stored as an `immutable` state variable set once in the constructor and never writable again:

```solidity
uint256 internal immutable i_maxSupply;   // line 41
// ...
i_maxSupply = maxSupply_;                 // line 54
``` [2](#0-1) [3](#0-2) 

There is no `setMaxSupply`, no emergency bypass, and no owner-callable override anywhere in the contract. [4](#0-3) 

On the L1 side, `L1VaultV2.bridgeRsETHToL2UsingCCIP()` sends rsETH tokens via the Chainlink CCIP router. On the L2 side, the CCIP OffRamp calls `mint()` on `WrappedRSETH` to deliver the tokens (including any yield component embedded in the rsETH exchange rate). [5](#0-4) 

If `totalSupply()` is at `i_maxSupply` when the OffRamp attempts execution, the call reverts. Chainlink CCIP allows manual re-execution of failed messages, but every retry hits the same immutable cap and reverts identically. The only theoretical escape is for existing holders to burn tokens first — but the yield recipient has no authority to compel that, and no on-chain mechanism exists to coordinate it.

---

### Impact Explanation

- **Scope:** Medium — Permanent freezing of unclaimed yield.
- All wrsETH yield that the CCIP bridge attempts to mint after the cap is reached is permanently undeliverable.
- Affected users cannot receive their accrued rsETH yield on L2 regardless of how long they wait or how many retries are attempted.
- The `i_maxSupply` immutability means no admin action can unblock the situation without deploying an entirely new token contract and migrating all state.

---

### Likelihood Explanation

- Precondition: `WrappedRSETH` is deployed with a finite `maxSupply_` (non-zero). The contract explicitly supports this path and the README lists both "CCIP (Chainlink) RSETH (Old)" and "CCIP (Chainlink) RSETH (New)" deployments, meaning the constructor has been called in production.
- Trigger: Organic bridging activity (no attacker required) fills `totalSupply()` to `i_maxSupply`. An attacker can accelerate this by bridging a large amount themselves.
- No special role, leaked key, or governance capture is needed. Any address with rsETH on L1 can bridge via `L1VaultV2.bridgeRsETHToL2UsingCCIP()`.

---

### Recommendation

1. **Replace `immutable` with a governable cap.** Store `maxSupply` in a regular storage variable and add an owner-only setter so the cap can be raised if needed.
2. **Alternatively, deploy with `maxSupply_ = 0` (unlimited).** The contract already treats `0` as "no cap" — this is the safest option for a bridge token where supply must track L1 bridged amounts indefinitely.
3. **Add a CCIP execution fallback.** If the cap is ever hit, provide a mechanism (e.g., temporarily pausing new bridges while the cap is raised) so in-flight messages can be retried successfully.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";
import "contracts/ccip/WrappedRSETH.sol";

contract MaxSupplyPoC is Test {
    WrappedRSETH token;
    address minter = address(0xBEEF);
    address user   = address(0xCAFE);

    function setUp() public {
        // Deploy with a finite cap of 1000 tokens
        token = new WrappedRSETH("wrsETH", "wrsETH", 18, 1000e18, address(this));
        token.grantMintRole(minter);
    }

    function test_supplyCapBlocksYieldMint() public {
        // Step 1: fill supply to cap via normal bridging
        vm.prank(minter);
        token.mint(user, 1000e18);
        assertEq(token.totalSupply(), 1000e18);

        // Step 2: CCIP OffRamp attempts to mint yield — reverts permanently
        vm.prank(minter);
        vm.expectRevert(
            abi.encodeWithSelector(WrappedRSETH.MaxSupplyExceeded.selector, 1000e18 + 1)
        );
        token.mint(address(0xDEAD), 1); // yieldRecipient, yieldAmount

        // Step 3: No recovery path — i_maxSupply is immutable, cannot be raised
        assertEq(token.maxSupply(), 1000e18); // unchanged
    }
}
