### Title
`dailyMintAmount` is not decremented in `swapAssetToPremintedRsETH()`, causing the daily mint limit to be artificially exhausted - (File: contracts/pools/RSETHPoolV3.sol)

---

### Summary

`RSETHPoolV3` tracks daily rsETH minting via `dailyMintAmount`, which is incremented in the `limitDailyMint` modifier applied to both `deposit()` overloads. However, `swapAssetToPremintedRsETH()` — the functional inverse of `deposit()` — does not decrement `dailyMintAmount` when rsETH is returned to the system. This causes the daily mint counter to overstate actual net minting, artificially exhausting the daily limit and blocking legitimate user deposits for the remainder of the day.

---

### Finding Description

`RSETHPoolV3` enforces a daily minting cap through the `limitDailyMint` modifier:

```solidity
// contracts/pools/RSETHPoolV3.sol lines 119-123
if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
    revert DailyMintLimitExceeded();
}
dailyMintAmount += rsETHAmount;
```

This modifier is applied to both `deposit(string)` (ETH path, line 251) and `deposit(address,uint256,string)` (token path, line 280). Every deposit increases `dailyMintAmount`.

`swapAssetToPremintedRsETH()` (lines 414–450) is the explicit reverse operation: it accepts rsETH from the caller, transfers it into the wrapper, and returns ETH or a supported token from the pool. The function comment states: *"Functionally, it works as the opposite of `deposit`, but it does not charge any fees."*

Despite being the inverse of `deposit()`, `swapAssetToPremintedRsETH()` never decrements `dailyMintAmount`:

```solidity
// contracts/pools/RSETHPoolV3.sol lines 414-450
function swapAssetToPremintedRsETH(
    address rsETH,
    address token,
    uint256 rsETHAmount
)
    external
    nonReentrant
    onlySupportedTokenOrEth(token)
    onlyRole(OPERATOR_ROLE)
{
    // ... transfers rsETH in, transfers ETH/token out
    // dailyMintAmount is never decremented here
    emit ReverseSwapOccurred(msg.sender, rsETH, token, rsETHAmount, tokenAmount);
}
```

The result is that `dailyMintAmount` accumulates gross minting without accounting for rsETH returned via reverse swaps, causing the effective daily capacity to be understated.

---

### Impact Explanation

**Medium — Temporary freezing of funds.**

When the operator legitimately calls `swapAssetToPremintedRsETH()` during a day where deposits have already consumed a significant portion of `dailyMintLimit`, the counter is not reduced. Subsequent user `deposit()` calls revert with `DailyMintLimitExceeded` even though the net rsETH minted for the day is well below `dailyMintLimit`. Users are locked out of depositing until the next day resets `dailyMintAmount` to zero (via the `currentDay > lastMintDay` branch in `limitDailyMint`). This is a temporary but complete denial of the deposit function for all users for the remainder of that day.

---

### Likelihood Explanation

**Medium.** `swapAssetToPremintedRsETH()` is a documented, intended protocol operation performed by the `OPERATOR_ROLE` as part of normal pool management (e.g., rebalancing). No malicious or compromised actor is required. Any day on which the operator executes a reverse swap after significant deposit volume will trigger the condition. The larger the reverse swap relative to remaining daily capacity, the more deposits are blocked.

---

### Recommendation

Decrement `dailyMintAmount` in `swapAssetToPremintedRsETH()` by the rsETH amount returned, mirroring the increment done in `limitDailyMint`:

```solidity
// After computing tokenAmount and before the transfers:
if (getCurrentDay() == lastMintDay) {
    if (dailyMintAmount >= rsETHAmount) {
        dailyMintAmount -= rsETHAmount;
    } else {
        dailyMintAmount = 0;
    }
}
```

This ensures `dailyMintAmount` reflects net minting for the day, consistent with the intent of the daily limit mechanism.

---

### Proof of Concept

1. `dailyMintLimit = 1000 rsETH`, current day begins.
2. Users call `deposit()` with ETH → `dailyMintAmount = 900`.
3. Operator calls `swapAssetToPremintedRsETH()` returning 600 rsETH to the wrapper → `dailyMintAmount` remains `900` (not decremented).
4. Net rsETH minted for the day = 900 − 600 = **300 rsETH**, well below the 1000 limit.
5. A new user attempts `deposit()` for 200 rsETH worth of ETH → modifier checks `900 + 200 = 1100 > 1000` → **reverts with `DailyMintLimitExceeded`**.
6. All deposits are blocked for the rest of the day despite only 300 rsETH net being outstanding.

**Root cause line**: `swapAssetToPremintedRsETH()` at [1](#0-0)  never touches `dailyMintAmount`, while `limitDailyMint` at [2](#0-1)  unconditionally increments it on every deposit.
