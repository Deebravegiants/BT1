### Title
Dust Stakers Permanently Earn Zero Rewards Due to Integer Truncation in `earned()` — (`contracts/KERNEL/KernelDepositPool.sol`)

---

### Summary

The `earned()` function divides by `DECIMAL_PRECISION` (1e18) after multiplying `balanceOf[_account]` by the per-token reward delta. For stakers whose balance is small relative to `totalKernelStaked`, this division truncates to zero every update cycle, permanently denying them any reward tokens. The truncated micro-rewards have no recovery path and remain locked in the contract.

---

### Finding Description

`rewardPerToken()` accumulates a global index scaled by `DECIMAL_PRECISION`:

```solidity
// KernelDepositPool.sol:412-413
rewardPerTokenStored + (rewardRate * (lastTimeRewardApplicable() - updatedAt) * DECIMAL_PRECISION)
    / totalKernelStaked;
```

`earned()` then reverses the scaling:

```solidity
// KernelDepositPool.sol:422-423
return (balanceOf[_account] * (rewardPerToken() - userRewardPerTokenPaid[_account]) / DECIMAL_PRECISION)
    + rewards[_account];
``` [1](#0-0) 

The division by `DECIMAL_PRECISION` truncates to zero whenever:

```
balanceOf[_account] * delta_rewardPerToken < DECIMAL_PRECISION
```

i.e., when `balanceOf[_account] * rewardRate * elapsed < totalKernelStaked`.

**Concrete example:**
- `totalKernelStaked = 1e24` (1 million KERNEL, 18 decimals)
- `rewardRate = 1e18` (1 reward token/second)
- `duration = 7 days = 604,800 s`
- `delta_rewardPerToken = 1e18 × 604800 × 1e18 / 1e24 = 6.048 × 10¹⁷`
- For `balanceOf = 1` wei: `earned = 1 × 6.048e17 / 1e18 = 0`

The `updateReward` modifier advances `userRewardPerTokenPaid[_account]` to the current `rewardPerTokenStored` on every interaction:

```solidity
// KernelDepositPool.sol:237-238
rewards[_account] = earned(_account);          // stores 0
userRewardPerTokenPaid[_account] = rewardPerTokenStored;  // checkpoint advanced
``` [2](#0-1) 

This means the truncated delta is discarded on every call to `stake`, `withdraw`, or `getReward`. The dust staker's checkpoint is advanced even though nothing was credited, so the truncated portion is irrecoverably lost.

There is no `sweep`, `recover`, or admin rescue function in the contract, so the excess reward balance has no exit path. [3](#0-2) 

---

### Impact Explanation

Dust stakers receive zero reward tokens despite staking for a full reward period. Their proportional share of rewards — though sub-wei in absolute terms — is permanently stranded in the contract with no recovery mechanism. This violates the invariant that every staker accrues nonzero rewards proportional to their stake over any sufficiently long period.

**Impact: Low — Contract fails to deliver promised returns, but doesn't lose value.**

The absolute value of locked rewards per dust staker is sub-wei of the reward token, so there is no material financial loss. However, the behavioral invariant is broken.

---

### Likelihood Explanation

Any user who stakes fewer tokens than `totalKernelStaked / (rewardRate × duration)` is affected. With large pools and moderate reward rates this threshold can be non-trivial (e.g., staking fewer than ~1,000 wei KERNEL in the example above). The path is fully permissionless via `stake()`.

---

### Recommendation

Accumulate truncated remainders rather than discarding them. One standard approach is to track the remainder at the `rewardPerToken` level and carry it forward, or to enforce a minimum stake amount that guarantees `balanceOf * rewardRate * duration >= totalKernelStaked` (i.e., at least 1 wei of reward is earned per period). Alternatively, add an admin `recoverExcessRewards()` function so stranded dust can be reclaimed and redistributed.

---

### Proof of Concept

```solidity
// Pseudocode — run on a local fork or Foundry test
function testDustStakerEarnsZero() public {
    // Setup: 1 million KERNEL already staked by whale
    uint256 whaleStake = 1_000_000e18;
    kernelToken.mint(whale, whaleStake);
    vm.prank(whale); pool.stake(whaleStake);

    // Admin starts reward period: 1 reward token/second for 7 days
    rewardsToken.mint(admin, 604_800e18);
    vm.prank(admin); pool.setRewardsDuration(7 days);
    vm.prank(admin); pool.notifyRewardAmount(604_800e18);

    // Dust staker stakes 1 wei
    kernelToken.mint(dustStaker, 1);
    vm.prank(dustStaker); pool.stake(1);

    // Fast-forward full reward period
    vm.warp(block.timestamp + 7 days);

    // Claim rewards
    vm.prank(dustStaker); pool.getReward();

    // Assert: dust staker received nothing
    assertEq(rewardsToken.balanceOf(dustStaker), 0);
    assertEq(pool.rewards(dustStaker), 0);

    // delta_rewardPerToken = 1e18 * 604800 * 1e18 / 1e24 = 6.048e17 < 1e18
    // earned = 1 * 6.048e17 / 1e18 = 0  ✓
}
```

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
