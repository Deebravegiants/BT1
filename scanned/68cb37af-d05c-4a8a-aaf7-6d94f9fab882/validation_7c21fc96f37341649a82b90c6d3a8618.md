### Title
`KernelMerkleDistributor.claimAndStake` Calls Role-Restricted `stakeFor` Without Holding `STAKE_FOR_ROLE`, Making the Function Permanently Unusable - (File: contracts/KERNEL/KernelMerkleDistributor.sol)

---

### Summary

`KernelMerkleDistributor.claimAndStake` calls `KernelDepositPool.stakeFor`, which is gated by `onlyRole(STAKE_FOR_ROLE)`. `KernelMerkleDistributor` is never granted this role during initialization and has no mechanism to acquire it, so every call to `claimAndStake` reverts unconditionally.

---

### Finding Description

`KernelMerkleDistributor.claimAndStake` is a public user-facing function that is supposed to verify a Merkle proof, deduct a fee, and atomically stake the remaining KERNEL tokens into `KernelDepositPool` on behalf of the caller. [1](#0-0) 

At line 282 it calls:

```solidity
IKernelDepositPool(kernelDepositPool).stakeFor(account, amountToStake);
```

`KernelDepositPool.stakeFor` is protected by `onlyRole(STAKE_FOR_ROLE)`: [2](#0-1) 

`STAKE_FOR_ROLE` is defined as: [3](#0-2) 

`KernelMerkleDistributor.initialize` sets a token approval for `KernelDepositPool` but never grants `STAKE_FOR_ROLE` to itself: [4](#0-3) 

`KernelDepositPool.initialize` only grants `DEFAULT_ADMIN_ROLE` to `_admin` and does not pre-grant `STAKE_FOR_ROLE` to any distributor: [5](#0-4) 

There is no function anywhere in `KernelMerkleDistributor` to acquire or request `STAKE_FOR_ROLE`. The interface comment in `KernelMerkleDistributor` itself acknowledges the restriction ("Only STAKE_FOR_ROLE can call this function") but the contract never satisfies it: [6](#0-5) 

The same defect exists in `KernelTop100MerkleDistributor.claimAndStake`, which also calls `kernelDepositPool.stakeFor(user, amountToStake)` without holding the role: [7](#0-6) 

---

### Impact Explanation

Every call to `claimAndStake` reverts at the `stakeFor` call. The promised atomic claim-and-stake flow is permanently broken. Users who attempt it receive no tokens and no staked position. They must fall back to `claim()` and then manually stake — a two-transaction path that the contract explicitly promises to collapse into one. No funds are lost (the revert rolls back all state changes including the fee transfer inside `_processClaim`), but the contract fails to deliver its advertised functionality.

**Impact: Low — Contract fails to deliver promised returns, but doesn't lose value.**

---

### Likelihood Explanation

The failure is unconditional: `STAKE_FOR_ROLE` is never granted during initialization, so `claimAndStake` reverts for every caller on every call. Any user who reads the interface and calls `claimAndStake` will experience the revert immediately.

---

### Recommendation

During `KernelMerkleDistributor.initialize`, after deploying or receiving the `KernelDepositPool` address, the admin should grant `STAKE_FOR_ROLE` to the `KernelMerkleDistributor` contract address via `KernelDepositPool.grantRole(STAKE_FOR_ROLE, address(kernelMerkleDistributor))`. The same fix applies to `KernelTop100MerkleDistributor`. Alternatively, add a dedicated `grantStakeForRole` admin function inside each distributor that calls `KernelDepositPool.grantRole` so the setup is explicit and auditable.

---

### Proof of Concept

1. Deploy `KernelDepositPool` with admin `A`.
2. Deploy `KernelMerkleDistributor` pointing at the pool. Note: `initialize` never calls `KernelDepositPool.grantRole(STAKE_FOR_ROLE, address(this))`.
3. Admin sets a Merkle root; user `U` has a valid leaf.
4. `U` calls `claimAndStake(index, U, amount, proof)`.
5. `_processClaim` succeeds internally; execution reaches line 282: `IKernelDepositPool(kernelDepositPool).stakeFor(U, amountToStake)`.
6. `KernelDepositPool.stakeFor` checks `onlyRole(STAKE_FOR_ROLE)` — `KernelMerkleDistributor` does not hold this role → revert.
7. Entire transaction reverts. `U` receives nothing and holds no staked position. `claimAndStake` is permanently non-functional.

    function claimAndStake(uint256 amount, bytes32[] calldata merkleProof) external nonReentrant whenNotPaused {
        address user = msg.sender;

        // Verify merkle proof and update user claim data
        _verifyClaimProof(user, amount, merkleProof);

        // Get claimable amount
        uint256 claimableAmount = _getUnclaimedVestedAmount(user, amount);

        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim data
        userClaims[user].lastClaimTimestamp = block.timestamp;
        userClaims[user].amountClaimed += claimableAmount;

        // Calculate fee
        uint256 fee = (claimableAmount * feeInBPS) / FEE_DENOMINATOR;
        uint256 amountToStake = claimableAmount - fee;

        // Transfer fee and stake tokens
        if (fee > 0) {
            kernel.safeTransfer(protocolTreasury, fee);
        }

        kernelDepositPool.stakeFor(user, amountToStake);

        emit ClaimedAndStaked(user, amountToStake);
