### Title
Reward Tokens Permanently Locked When `totalKernelStaked` Drops to Zero Mid-Period — (`contracts/KERNEL/KernelDepositPool.sol`)

---

### Summary

`KernelDepositPool` guards `notifyRewardAmount` against a zero-staked start, but provides no protection against `totalKernelStaked` falling to zero **during** an active reward window. When that happens, `rewardPerToken()` freezes and all undistributed reward tokens for the remainder of the period are permanently locked in the contract with no recovery path.

---

### Finding Description

`notifyRewardAmount` contains a guard at line 570:

```solidity
if (totalKernelStaked == 0) revert NoStakedTokens();
``` [1](#0-0) 

This prevents starting a reward period with no stakers. However, `initiateWithdrawal` immediately decrements `totalKernelStaked` at line 326 with no check on whether a reward period is currently active:

```solidity
balanceOf[msg.sender] -= _amount;
totalKernelStaked -= _amount;
``` [2](#0-1) 

Once `totalKernelStaked` reaches zero, `rewardPerToken()` short-circuits and returns the frozen stored value:

```solidity
function rewardPerToken() public view returns (uint256) {
    if (totalKernelStaked == 0) {
        return rewardPerTokenStored;
    }
    ...
}
``` [3](#0-2) 

From that moment forward, `rewardRate` continues to be set and `finishAt` is in the future, but no new `rewardPerToken` accumulates. All reward tokens deposited for the remaining period are stranded. The contract has no `rescueTokens`, no refund path, and no admin function that can recover them. The contract's own NatSpec acknowledges this gap:

> *"If `totalKernelStaked` ever hits zero during a reward distribution window, any remaining rewards for that period will stay locked in the contract."* [4](#0-3) 

The stated mitigation is purely operational ("ensuring there are always some tokens staked"), which is not enforced on-chain.

---

### Impact Explanation

Any reward tokens transferred into the contract via `notifyRewardAmount` that have not yet been distributed at the moment `totalKernelStaked` hits zero are permanently inaccessible. This constitutes **permanent freezing of unclaimed yield** (Medium severity per scope).

---

### Likelihood Explanation

The scenario is realistic at protocol launch when the staker set is small. A single legitimate staker who is also the last remaining staker can trigger this unintentionally by calling `initiateWithdrawal` for any reason (e.g., to exit the protocol). No coordination or malicious intent is required. The 30-day withdrawal delay means the staker has a strong incentive to exit before a new reward period begins, but the timing is not enforced.

---

### Recommendation

1. **On-chain enforcement:** In `initiateWithdrawal`, revert if the withdrawal would bring `totalKernelStaked` to zero while `block.timestamp < finishAt`.
2. **Recovery path:** Add an admin-only `rescueStrandedRewards()` function that can only be called after `finishAt` and only when `totalKernelStaked == 0`, transferring the unearned balance back to the treasury.
3. **Minimum stake:** Enforce a protocol-owned minimum stake (e.g., 1e18 tokens held by the contract itself) that can never be withdrawn, guaranteeing `totalKernelStaked > 0` for the entire reward window.

---

### Proof of Concept

```
Preconditions:
  - duration = 7 days
  - Alice is the only staker: 1000e18 KERNEL

Step 1: Admin calls notifyRewardAmount(1000e18)
  → totalKernelStaked = 1000e18 > 0, check passes
  → rewardRate = 1000e18 / 604800

Step 2: After 1 day, Alice calls initiateWithdrawal(1000e18)
  → updateReward(alice) runs:
       rewardPerTokenStored updated to reflect 1 day of accrual
       rewards[alice] ≈ 142e18 (1/7 of total, correctly saved)
  → totalKernelStaked = 0

Step 3: rewardPerToken() now returns frozen rewardPerTokenStored
  → No further accumulation for the remaining 6 days

Step 4: After 30 days Alice claims her KERNEL and her ~142e18 reward tokens.

Result: ~857e18 reward tokens (6/7 of the notified amount) are permanently
        locked in KernelDepositPool with no recovery mechanism.
``` [5](#0-4) [6](#0-5) [7](#0-6)

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
```
