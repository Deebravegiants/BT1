### Title
Unbounded Gas Consumption in `getAssetUnstaking()` Blocks `updateRSETHPrice()` and User Deposits - (File: contracts/NodeDelegator.sol)

### Summary
`NodeDelegator.getAssetUnstaking()` fetches **all** queued EigenLayer withdrawals via `getQueuedWithdrawals()` and iterates over them in a nested loop. This function is called in the critical paths of both `LRTOracle.updateRSETHPrice()` and `LRTDepositPool.depositAsset()`/`depositETH()`. As the protocol scales toward its configured limits, the cumulative gas cost of these nested external calls and loops can exceed the block gas limit, making those functions permanently uncallable.

### Finding Description

`NodeDelegator.getAssetUnstaking()` is implemented as:

```solidity
function getAssetUnstaking(address asset) external view returns (uint256 amount) {
    (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
        _getDelegationManager().getQueuedWithdrawals(address(this));

    for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
        IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];
        for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
            ...
        }
    }
}
``` [1](#0-0) 

This function is called inside `LRTDepositPool.getAssetDistributionData()` and `getETHDistributionData()`, once per NDC per asset: [2](#0-1) [3](#0-2) 

`getAssetDistributionData()` is called by `getTotalAssetDeposits()`, which is called by:

1. `LRTOracle._getTotalEthInProtocol()` — once per supported asset, inside the public `updateRSETHPrice()`: [4](#0-3) 

2. `LRTDepositPool._checkIfDepositAmountExceedesCurrentLimit()` — called inside every `depositAsset()` and `depositETH()`: [5](#0-4) 

The total number of `getAssetUnstaking()` calls per `updateRSETHPrice()` invocation is **S × N** (supported assets × NDCs). Each call makes one external call to `getQueuedWithdrawals()` and iterates over up to W queued withdrawals × T strategies each.

The developers themselves acknowledge the gas concern in a comment: [6](#0-5) 

> "120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price"

At protocol limits (S=5 assets, N=10 NDCs, W=80 withdrawals/NDC, T=2 strategies/withdrawal):
- **50 external calls** to `getQueuedWithdrawals()` (each returning up to 80 withdrawal structs with large calldata)
- **8,000 inner loop iterations**

This can easily exceed Ethereum's 30M block gas limit.

### Impact Explanation

**Medium — Unbounded gas consumption.**

When the protocol operates near its configured limits, `updateRSETHPrice()` (public, no access control) and `depositAsset()`/`depositETH()` (user-facing) become uncallable due to OOG revert. Consequences:
- rsETH price cannot be updated → price staleness, fee minting halted, downside-protection auto-pause cannot trigger.
- User deposits revert → temporary freezing of the deposit path. [7](#0-6) [8](#0-7) 

### Likelihood Explanation

The protocol caps `maxNodeDelegatorLimit` (default 10, admin-adjustable) and `maxUncompletedWithdrawalCount` (≤ 80). These are not small numbers. During periods of high withdrawal activity — which is normal protocol operation — all NDCs can accumulate many queued withdrawals simultaneously. No attacker action is required; ordinary protocol usage at scale triggers the condition. The developers' own comment confirms they are aware the gas budget is tight.

### Recommendation

- Cache the result of `getQueuedWithdrawals()` per NDC per block, or restructure `getAssetUnstaking()` to avoid fetching all queued withdrawals on every call.
- Maintain an on-chain running total of `assetUnstaking` per NDC per asset (updated on `initiateUnstaking` / `completeUnstaking`) instead of recomputing it by iterating EigenLayer's withdrawal queue on every read.
- Decouple the deposit limit check from the full TVL computation; use a cached/snapshotted value for deposit gating.

### Proof of Concept

Call chain for `updateRSETHPrice()` with 5 supported assets and 10 NDCs each having 80 queued withdrawals:

```
updateRSETHPrice()
  └─ _getTotalEthInProtocol()                          // loops over 5 assets
       └─ getTotalAssetDeposits(asset)                 // per asset
            └─ getAssetDistributionData(asset)
                 └─ [loop: 10 NDCs]
                      └─ getAssetUnstaking(asset)      // 50 total calls
                           └─ getQueuedWithdrawals()   // external call, returns 80 withdrawals
                                └─ [loop: 80 × 2]     // 8,000 inner iterations
```

At these numbers the transaction exceeds 30M gas and reverts, making `updateRSETHPrice()` and all user deposits permanently uncallable until the withdrawal queue drains. [9](#0-8) [10](#0-9)

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }

        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

            unchecked {
                ++assetIdx;
            }
        }
    }
