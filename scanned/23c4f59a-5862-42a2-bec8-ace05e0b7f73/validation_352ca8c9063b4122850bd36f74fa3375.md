### Title
`ChainlinkPriceOracle.getAssetPrice()` Does Not Check for Stale Chainlink Price - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards all freshness-related return values (`updatedAt`, `answeredInRound`, `roundId`). A stale price is then used to compute rsETH mint amounts for every depositor, which can cause the protocol to mint more rsETH than the deposited assets are worth, leading to insolvency.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches the Chainlink price as follows:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values of `latestRoundData()` are `(roundId, answer, startedAt, updatedAt, answeredInRound)`. The code discards `roundId`, `startedAt`, `updatedAt`, and `answeredInRound` entirely. There is no check that:
- `updatedAt` is within an acceptable heartbeat window (`block.timestamp - updatedAt <= maxDelay`)
- `answeredInRound >= roundId` (round completeness)
- `price > 0`

This price is consumed by `LRTOracle.getAssetPrice()`, which is called by `LRTDepositPool.getRsETHAmountToMint()`:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

If the Chainlink feed for any supported LST (stETH, rETH, swETH, etc.) becomes stale — due to network congestion, sequencer downtime, or oracle node failure — the protocol will continue minting rsETH using the last reported (outdated) price.

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` in the same codebase does perform partial staleness checks (`answeredInRound < roundID`, `timestamp == 0`), demonstrating that the team is aware of the pattern but did not apply it to the primary oracle. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

**Protocol insolvency (Critical).** If a supported LST's Chainlink feed goes stale while the asset's true market price has fallen, `getAssetPrice()` returns the old, inflated price. Every depositor during the stale window receives more rsETH than their deposited assets are worth. The rsETH supply becomes undercollateralized relative to the actual ETH value held, causing bad debt for the protocol and losses for existing rsETH holders. [4](#0-3) 

---

### Likelihood Explanation

Chainlink feeds can go stale during periods of network congestion, L1/L2 sequencer downtime, or when price deviation thresholds are not breached (heartbeat-only updates). The `ChainlinkPriceOracle` is the live production oracle for all LST assets (stETH, rETH, swETH, ETHx, sfrxETH) on mainnet. Any depositor calling `depositAsset()` or `depositETH()` during a stale window triggers the vulnerable path with no special permissions required. [5](#0-4) 

---

### Recommendation

Add staleness and round-completeness checks in `ChainlinkPriceOracle.getAssetPrice()`, consistent with what `ChainlinkOracleForRSETHPoolCollateral` already does:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    require(price > 0, "invalid price");
    require(answeredInRound >= roundId, "stale price: incomplete round");
    require(updatedAt != 0, "stale price: round not complete");
    require(block.timestamp - updatedAt <= MAX_PRICE_AGE, "stale price: heartbeat exceeded");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

`MAX_PRICE_AGE` should be set per-asset based on the Chainlink feed's documented heartbeat (e.g., 3600 seconds for 1-hour heartbeat feeds). [1](#0-0) 

---

### Proof of Concept

1. Chainlink's stETH/ETH feed stops updating (e.g., heartbeat window passes without a price deviation trigger).
2. The last reported price is `1.05 ETH` per stETH; the true market price drops to `0.90 ETH`.
3. A depositor calls `LRTDepositPool.depositAsset(stETH, 100e18, ...)`.
4. `getRsETHAmountToMint()` calls `lrtOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns stale `1.05e18`.
5. `rsethAmountToMint = (100e18 * 1.05e18) / rsETHPrice` — the depositor receives rsETH valued at 105 ETH worth of backing, but only deposited assets worth 90 ETH.
6. The protocol is now undercollateralized by 15 ETH per 100 stETH deposited during the stale window. [1](#0-0) [6](#0-5)
