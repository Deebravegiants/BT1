### Title
Stale Rate in `CrossChainRateReceiver.getRate()` Causes Pool to Overpay rsETH to Depositors, Depleting Inventory — (`contracts/cross-chain/CrossChainRateReceiver.sol`)

---

### Summary

`CrossChainRateReceiver` stores `lastUpdated` but never enforces a freshness check in `getRate()`. Both `RSETHPool` and `RSETHPoolNoWrapper` call `getRate()` unconditionally to price every deposit. When the LayerZero update is delayed and the true L1 rsETH/ETH rate has risen, the stale (lower) rate causes the pool to transfer more rsETH per ETH than the deposited ETH will generate when bridged back to L1, gradually depleting the pool's rsETH inventory.

---

### Finding Description

`CrossChainRateReceiver.getRate()` returns the raw stored `rate` with no staleness guard:

```solidity
// CrossChainRateReceiver.sol line 103-105
function getRate() external view returns (uint256) {
    return rate;
}
``` [1](#0-0) 

`lastUpdated` is written on every `lzReceive` call but is never read anywhere in the contract or by callers. [2](#0-1) 

Both pool contracts delegate pricing entirely to this oracle:

```solidity
// RSETHPool.sol line 316-319
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
``` [3](#0-2) 

```solidity
// RSETHPoolNoWrapper.sol line 282-285
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
``` [4](#0-3) 

The pool then transfers pre-existing rsETH from its own balance — it does **not** mint: [5](#0-4) [6](#0-5) 

**Arithmetic of the overpayment:**

| Scenario | Rate used | rsETH out for 1 ETH |
|---|---|---|
| Stale (stale = 1.05e18) | 1.05e18 | `1e18 / 1.05e18` ≈ **0.9524 rsETH** |
| Fresh (true = 1.10e18) | 1.10e18 | `1e18 / 1.10e18` ≈ **0.9091 rsETH** |
| **Overpayment per ETH** | | **≈ 0.0433 rsETH** |

The 1 ETH bridged to L1 will only generate ≈ 0.9091 rsETH at the true rate. The pool already paid out 0.9524, so it is short ≈ 0.0433 rsETH per ETH deposited during the stale window. This shortfall comes directly from the pool's pre-loaded rsETH inventory.

---

### Impact Explanation

The pool's rsETH inventory is depleted faster than the bridged ETH replenishes it. No new unbacked rsETH is minted (the pool transfers pre-existing tokens), but the pool systematically gives away rsETH at a discount relative to the true L1 rate. Once inventory is exhausted, subsequent depositors cannot be served. The protocol absorbs the spread loss silently with no on-chain alert.

This matches **Low — Contract fails to deliver promised returns, but doesn't lose value** (the rsETH given out was already minted and backed on L1; the pool's own inventory is the loss vector, not user funds).

---

### Likelihood Explanation

LayerZero message delays are a known operational reality (network congestion, relayer downtime). rsETH accrues yield continuously, so even a few hours of staleness during a high-accrual period creates a measurable spread. No attacker action is required beyond depositing during the stale window; any user benefits automatically. The pool has no circuit-breaker for stale rates.

---

### Recommendation

Add a configurable maximum staleness threshold and revert in `getRate()` if exceeded:

```solidity
uint256 public maxStaleness; // e.g. 24 hours

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= maxStaleness, "Rate is stale");
    return rate;
}
```

Alternatively, expose `lastUpdated` to callers and let the pool contracts enforce the check before pricing a deposit.

    // 1. Deploy pool pointing at a mock CrossChainRateReceiver

    oracle.setRate(1.05e18);          // stale rate
    oracle.setLastUpdated(block.timestamp - 2 days);

    RSETHPoolNoWrapper pool = /* deploy with oracle */;
    // pre-load pool with rsETH
    rsETH.transfer(address(pool), 10e18);

    // 2. User deposits 1 ETH at stale rate
    uint256 balBefore = rsETH.balanceOf(user);
    pool.deposit{value: 1e18}("ref");
    uint256 staleOut = rsETH.balanceOf(user) - balBefore;

    // 3. Simulate rate update to true value
    oracle.setRate(1.10e18);
    oracle.setLastUpdated(block.timestamp);

    uint256 freshOut = rsETH.balanceOf(user2) - balBefore2;
    // freshOut ≈ 0.9091e18

    // 5. Assert overpayment
    assertGt(staleOut, freshOut);
    // Difference ≈ 0.0433e18 rsETH per ETH — pool inventory shortfall
}
