### Title
Lack of Pause Functionality on `bridgeRsETHToL2()` Allows rsETH to be Permanently Locked Without Release on L2 - (File: contracts/L1Vault.sol, contracts/L1VaultV2.sol)

### Summary
`L1Vault` and `L1VaultV2` implement no pause mechanism on `bridgeRsETHToL2()`. If the L2 OFT endpoint or the destination chain's receiving contract becomes paused or broken, the `MANAGER_ROLE` can still invoke `bridgeRsETHToL2()`, causing rsETH to be locked/burned inside the OFT Adapter on L1 without being released on L2. The only available mitigation requires `TIMELOCK_ROLE` — a higher-privilege role — creating a dangerous response-time gap.

### Finding Description
`L1Vault` and `L1VaultV2` both expose `bridgeRsETHToL2()`, which calls `oftAdapter.send()` to transfer rsETH from L1 to L2 via LayerZero. [1](#0-0) [2](#0-1) 

Neither contract inherits from `PausableUpgradeable` or implements any pause mechanism: [3](#0-2) [4](#0-3) 

The `IRSETH_OFTAdapter` interface also exposes no pause function: [5](#0-4) 

The only administrative levers to stop bridging are:
- Revoking `MANAGER_ROLE` — requires `DEFAULT_ADMIN_ROLE`
- Replacing the OFT Adapter via `setOFTAdapter()` — requires `TIMELOCK_ROLE` [6](#0-5) 

Both require a higher-privilege role than a hypothetical `PAUSER` role, and both involve governance delay (timelock). By contrast, the L2 pool contracts (e.g., `RSETHPoolV2ExternalBridge`, `RSETHPoolV3ExternalBridge`) do implement a `PAUSER_ROLE` with a `whenNotPaused` modifier on their deposit functions, demonstrating the protocol's awareness of the pattern — but this protection is absent on the L1 bridging side. [7](#0-6) 

### Impact Explanation
If the L2 LayerZero endpoint or the `RsETHTokenWrapper.mint()` function on L2 is paused or broken, any call to `bridgeRsETHToL2()` will lock rsETH inside the OFT Adapter on L1 with no corresponding release on L2. Depending on whether the OFT Adapter uses lock-and-mint or burn-and-mint semantics, the rsETH is either stuck (temporary freeze) or destroyed (permanent freeze). LayerZero's retry mechanism can recover stuck messages only if the destination endpoint eventually becomes functional; a permanently broken L2 endpoint results in permanent loss.

**Impact: Medium — Temporary freezing of funds (escalating to Critical — Permanent freezing of funds if the L2 endpoint is permanently broken).**

### Likelihood Explanation
LayerZero endpoints are pausable by their operators for maintenance or security incidents. The `RsETHTokenWrapper.mint()` function on L2 is access-controlled and could be paused or have its `MINTER_ROLE` revoked. In such scenarios, the `MANAGER_ROLE` on L1 — acting in good faith — may call `bridgeRsETHToL2()` without knowing the remote chain is broken, since there is no on-chain check of the remote endpoint's health before sending. The absence of a low-privilege emergency stop makes this a realistic operational risk.

### Recommendation
1. Inherit from `PausableUpgradeable` in both `L1Vault` and `L1VaultV2`.
2. Add a `PAUSER_ROLE` with a `pause()` function.
3. Apply `whenNotPaused` to `bridgeRsETHToL2()` (and `bridgeRsETHToL2UsingCCIP()` in `L1VaultV2`).
4. This mirrors the pattern already used in the L2 pool contracts and allows a lower-privilege role to halt bridging immediately when the remote chain has issues, without requiring a timelock-gated governance action.

### Proof of Concept
1. The L2 LayerZero endpoint or `RsETHTokenWrapper.mint()` is paused due to a security incident on L2.
2. `MANAGER_ROLE` on L1 calls `L1Vault.bridgeRsETHToL2(amount, minAmount, nativeFee)`.
3. `oftAdapter.send()` is called at line 254 of `L1Vault.sol`; rsETH is locked/burned in the OFT Adapter on L1.
4. The LayerZero message arrives at the L2 endpoint, but the endpoint is paused — the message cannot be executed.
5. rsETH is stuck in the OFT Adapter on L1 with no corresponding release on L2.
6. The only remediation path — calling `setOFTAdapter()` or revoking `MANAGER_ROLE` — requires `TIMELOCK_ROLE`, introducing governance delay during which additional bridging calls may compound the loss. [8](#0-7)

```text
    function bridgeRsETHToL2(
        uint256 amount,
        uint256 minAmount,
        uint256 nativeFee
    )
        external
        payable
        nonReentrant
        onlyRole(MANAGER_ROLE)
    {
        if (rsETH.balanceOf(address(this)) < amount) {
            revert InsufficientRsETHBalance();
        }

        if (minAmount > amount || minAmount == 0) {
            revert InvalidMinAmount();
        }

        if (msg.value != nativeFee) {
            revert IncorrectNativeFee();
        }

        IERC20(address(rsETH)).safeIncreaseAllowance(address(oftAdapter), amount);

        SendParam memory sendParam = SendParam({
            dstEid: dstLzChainId,
            to: getReceiver(),
            amountLD: amount,
            minAmountLD: minAmount,
            extraOptions: bytes(""),
            composeMsg: bytes(""),
            oftCmd: bytes("")
        });

        MessagingFee memory fee = MessagingFee({ nativeFee: nativeFee, lzTokenFee: 0 });

        oftAdapter.send{ value: nativeFee }(sendParam, fee, msg.sender);

        emit BridgedRsETHToL2(dstLzChainId, l2Receiver, amount, minAmount);
    }
```
