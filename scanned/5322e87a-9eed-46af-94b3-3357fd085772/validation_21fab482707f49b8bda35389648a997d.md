### Title
`rewardRate` Truncation in `notifyRewardAmount()` Permanently Locks Reward Dust - (File: contracts/KERNEL/KernelDepositPool.sol)

### Summary
In `KernelDepositPool.notifyRewardAmount()`, the `rewardRate` is computed via integer division `receivedAmount / duration`. The remainder (`receivedAmount % duration`) is silently discarded and permanently locked in the contract, never distributed to stakers. This is the direct analog of the `_getReserveRatioScalar()` rounding-down issue: a division truncates a scalar that governs how much value is distributed, causing the protocol to deliver less than the deposited reward amount.

### Finding Description
`notifyRewardAmount()` computes the per-second reward rate as:

```solidity
// contracts/KERNEL/KernelDepositPool.sol:579-584
if (block.timestamp >= finishAt) {
    rewardRate = receivedAmount / duration;
} else {
    uint256 remaining = (finishAt - block.timestamp) * rewardRate;
    rewardRate = (receivedAmount + remaining) / duration;
}
``` [1](#0-0) 

Both branches perform integer division by `duration`. The truncated remainder — up to `duration - 1` wei per call — is transferred into the contract via `safeTransferFrom` but is never accounted for in `rewardRate`. Since `rewardRate` drives all reward accrual through `rewardPerToken()` and `earned()`, the dust is permanently stranded:

```solidity
// contracts/KERNEL/KernelDepositPool.sol:412-413
return rewardPerTokenStored + (rewardRate * (lastTimeRewardApplicable() - updatedAt) * DECIMAL_PRECISION)
    / totalKernelStaked;
``` [2](#0-1) 

There is no sweep, recovery, or rollover mechanism for the truncated dust. The contract comment acknowledges the Synthetix origin but does not address this precision loss. [3](#0-2) 

### Impact Explanation
Every call to `notifyRewardAmount()` permanently locks up to `duration - 1` wei of the rewards token. Over repeated reward periods this accumulates. The locked tokens are irrecoverable: there is no admin sweep function, no `emergencyWithdraw`, and no mechanism to reclaim stranded reward dust. Stakers receive strictly less than the deposited reward amount. This maps to **Medium — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
`notifyRewardAmount()` is called by the admin as routine protocol operation at the start of every reward period. The truncation occurs on every single call without exception. The magnitude per call is bounded by `duration - 1` wei; for a 7-day duration (604 800 seconds) this is up to 604 799 wei of the rewards token per call. For tokens with fewer decimals (e.g., 6-decimal stablecoins used as rewards) this becomes material over many periods.

### Recommendation
Track and roll over the truncated dust into the next reward period, or explicitly account for it:

```solidity
if (block.timestamp >= finishAt) {
    rewardRate = receivedAmount / duration;
    // roll leftover into next period or emit event
    uint256 leftover = receivedAmount - rewardRate * duration;
    // store leftover for next notifyRewardAmount call
} else {
    uint256 remaining = (finishAt - block.timestamp) * rewardRate;
    rewardRate = (receivedAmount + remaining) / duration;
}
```

Alternatively, require that `receivedAmount` is an exact multiple of `duration`, or use a higher-precision accumulator (scale `rewardRate` by `1e18` before dividing).

### Proof of Concept
1. Admin sets `duration = 604800` (7 days).
2. Admin calls `notifyRewardAmount(1_000_000e6)` (1 million USDC, 6 decimals).
3. `rewardRate = 1_000_000e6 / 604800 = 1653439` (truncated).
4. Total distributed over the period: `1653439 * 604800 = 999_999_187_200`.
5. Dust locked: `1_000_000_000_000 - 999_999_187_200 = 812_800` wei ≈ **0.81 USDC permanently frozen per call**.
6. Over 100 reward periods: ~81 USDC permanently locked, with no recovery path. [4](#0-3)

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
```
