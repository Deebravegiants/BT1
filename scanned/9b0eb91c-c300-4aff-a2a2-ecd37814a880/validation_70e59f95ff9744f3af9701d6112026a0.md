### Title
Publicly Callable `updateRSETHPrice()` Allows Any Actor to Trigger Protocol-Wide Pause — (`contracts/LRTOracle.sol`)

---

### Summary
`LRTOracle.updateRSETHPrice()` carries no caller restriction beyond `whenNotPaused`. Its internal logic automatically pauses the deposit pool, withdrawal manager, and oracle whenever the computed rsETH price falls below a configured threshold relative to `highestRsethPrice`. Because the function is public, any unprivileged actor can invoke it the moment on-chain conditions satisfy the pause trigger, forcing a protocol-wide freeze before operators have a chance to respond.

---

### Finding Description
`updateRSETHPrice()` is declared `public` with only the `whenNotPaused` modifier:

```solidity
// contracts/LRTOracle.sol line 87
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, after computing `newRsETHPrice` from live on-chain data, the following block executes unconditionally for any caller:

```solidity
// contracts/LRTOracle.sol lines 270-282
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);

    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;
    }
```

When `isPriceDecreaseOffLimit` is true, the function calls `pause()` on `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. All three contracts enforce `whenNotPaused` on their user-facing entry points (`depositETH`, `depositAsset`, `initiateWithdrawal`, `completeWithdrawal`, `instantWithdrawal`). After the pause, every user interaction with the protocol reverts until an admin calls `unpause()` on each contract individually.

A privileged counterpart exists — `updateRSETHPriceAsManager()` (line 94) — which is restricted to `onlyLRTManager`. Its existence confirms the protocol designers intended privileged actors to control sensitive price updates, yet the public variant retains the same pause-triggering code path.

---

### Impact Explanation
Any unprivileged depositor, withdrawer, or rsETH holder can call `updateRSETHPrice()` the instant the computed price crosses the downside threshold (e.g., after a slashing event or a large queued withdrawal reduces `_getTotalEthInProtocol()`). The result is an immediate, protocol-wide pause that blocks:
- New ETH/LST deposits (`depositETH`, `depositAsset`)
- New withdrawal requests (`initiateWithdrawal`)
- Withdrawal completions (`completeWithdrawal`, `instantWithdrawal`)

Funds are not lost, but they are inaccessible until the admin team detects the pause and manually unpauses each contract. This constitutes **temporary freezing of funds** (Medium impact per the allowed scope).

---

### Likelihood Explanation
The trigger condition — `newRsETHPrice` falling more than `pricePercentageLimit` below `highestRsethPrice` — is realistic whenever EigenLayer slashing occurs, a large batch of validators exits, or a significant LST depeg event reduces `_getTotalEthInProtocol()`. An attacker monitoring on-chain state can detect this condition in the same block it arises and submit `updateRSETHPrice()` before any operator transaction. No special privileges, capital, or prior setup are required. Likelihood is **medium**.

---

### Recommendation
Restrict `updateRSETHPrice()` to authorized callers (e.g., `onlyLRTOperator` or `onlyLRTManager`), or separate the pause-triggering logic from the public price-update path. If a permissionless price update is desired for liveness, the automatic pause should only be executable by privileged roles, mirroring the pattern already established by `updateRSETHPriceAsManager()`.

---

### Proof of Concept
1. Protocol accumulates staking rewards; `highestRsethPrice` = 1.05 ETH, `pricePercentageLimit` = 1% (1e16).
2. A slashing event reduces total ETH in protocol; `newRsETHPrice` computes to 1.03 ETH — a 1.9% drop, exceeding the limit.
3. Attacker (any EOA) calls `LRTOracle.updateRSETHPrice()`.
4. `_updateRsETHPrice()` evaluates `isPriceDecreaseOffLimit = true`.
5. `LRTDepositPool.pause()`, `LRTWithdrawalManager.pause()`, and `LRTOracle._pause()` are called.
6. All subsequent `depositETH`, `initiateWithdrawal`, and `completeWithdrawal` calls revert with `Pausable: paused`.
7. Protocol remains frozen until admin manually unpauses each contract. [1](#0-0) [2](#0-1) [3](#0-2)
