### Title
Unbounded Gas Consumption in `depositETH`/`depositAsset` via Full `nodeDelegatorQueue` Iteration — (`contracts/LRTDepositPool.sol`)

---

### Summary

Every call to `depositETH` or `depositAsset` unconditionally iterates the entire `nodeDelegatorQueue` and issues multiple external calls per NDC. As the queue grows (via legitimate admin scaling), gas consumption grows linearly with queue length — and super-linearly when each NDC has accumulated queued EigenLayer withdrawals — eventually causing deposits to revert out-of-gas.

---

### Finding Description

**Call chain:**

```
depositETH / depositAsset
  └─ _beforeDeposit
       └─ _checkIfDepositAmountExceedesCurrentLimit
            └─ getTotalAssetDeposits
                 └─ getAssetDistributionData / getETHDistributionData
                      └─ for i in nodeDelegatorQueue:
                           ├─ IERC20(asset).balanceOf(ndc[i])          [external]
                           ├─ INodeDelegator(ndc[i]).getAssetBalance()  [external]
                           └─ INodeDelegator(ndc[i]).getAssetUnstaking() [external]
                                └─ DelegationManager.getQueuedWithdrawals(ndc)
                                     └─ for each withdrawal:
                                          └─ for each strategy: ...
```

**Layer 1 — outer loop over `nodeDelegatorQueue`:**

In `getAssetDistributionData`, for each NDC the code executes three external calls: [1](#0-0) 

In `getETHDistributionData`, for each NDC: [2](#0-1) 

**Layer 2 — nested loop inside `getAssetUnstaking`:**

Each `getAssetUnstaking()` call on a NodeDelegator fetches **all** queued EigenLayer withdrawals for that NDC and iterates over every withdrawal and every strategy within it: [3](#0-2) 

This makes the total gas cost **O(NDCs × queued_withdrawals × strategies_per_withdrawal)** — a product of three unbounded quantities.

**Layer 3 — no cap on `maxNodeDelegatorLimit`:**

The admin can raise `maxNodeDelegatorLimit` to any value and fill the queue: [4](#0-3) [5](#0-4) 

The initial value is 10, but there is no on-chain upper bound preventing it from being set to 100, 500, or higher.

**The deposit path has no bypass:** `_beforeDeposit` always calls `_checkIfDepositAmountExceedesCurrentLimit`, which always calls `getTotalAssetDeposits`, which always calls the full loop — there is no short-circuit or cached value: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

Once `nodeDelegatorQueue` is large enough (e.g., 50–100 NDCs each with several queued EigenLayer withdrawals), every call to `depositETH` or `depositAsset` will consume gas approaching or exceeding the 30M block gas limit, causing all user deposits to revert with out-of-gas. This is a **temporary freezing of all user deposits** (ETH and LST) until the admin manually removes NDCs from the queue — which itself requires those NDCs to have zero residual balances, a non-trivial operational requirement.

---

### Likelihood Explanation

The precondition is a **legitimate operational action**: the protocol scales by adding more NDCs to distribute restaking load. An admin raising `maxNodeDelegatorLimit` to 50 and populating the queue is expected behavior, not a compromise. Any user calling `depositETH` with any nonzero value after this point triggers the condition. No attacker capability is required beyond submitting a standard deposit transaction.

---

### Recommendation

1. **Cache `getTotalAssetDeposits` results** using a storage variable updated lazily (e.g., on each deposit/withdrawal) rather than recomputing on every deposit.
2. **Impose a hard on-chain cap** on `maxNodeDelegatorLimit` (e.g., ≤ 20) to bound worst-case gas.
3. **Separate accounting from live iteration**: maintain running totals for `assetLyingInNDCs`, `assetStakedInEigenLayer`, and `assetUnstakingFromEigenLayer` in storage, updated incrementally when assets move, rather than querying all NDCs on every deposit.
4. **Limit `getAssetUnstaking` scope**: instead of fetching all queued withdrawals from EigenLayer on every deposit, maintain a protocol-side accounting of pending unstaking amounts updated at queue/complete time.

```solidity
// Foundry fork test (mainnet fork)

    LRTDepositPool pool = LRTDepositPool(DEPOSIT_POOL_ADDRESS);

    function testGasExhaustion() external {
        // Simulate admin raising limit and adding N NDCs (done in setUp via prank)
        // Each NDC has K queued EigenLayer withdrawals

        uint256 gasStart = gasleft();
        vm.deal(address(this), 1 ether);
        pool.depositETH{value: 1 ether}(0, "");
        uint256 gasUsed = gasStart - gasleft();

        // Assert gas used scales with nodeDelegatorQueue.length
        // At N=50 NDCs × K=10 queued withdrawals each, gasUsed approaches 30M
        assertLt(gasUsed, block.gaslimit, "deposit exceeds block gas limit");
    }
}
```

At N=50 NDCs, each `getAssetUnstaking()` call invokes `DelegationManager.getQueuedWithdrawals()` (a storage-heavy read returning dynamic arrays), making the total gas cost easily exceed 30M for a fully-populated queue with active EigenLayer positions.

```text
    function _checkIfDepositAmountExceedesCurrentLimit(address asset, uint256 amount) internal view returns (bool) {
        uint256 totalAssetDeposits = getTotalAssetDeposits(asset);
        if (asset == LRTConstants.ETH_TOKEN) {
            return (totalAssetDeposits > lrtConfig.depositLimitByAsset(asset));
```

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
