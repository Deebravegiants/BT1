### Title
No Staleness Check on Chainlink Price Feed Allows Stale Asset Prices to Be Used for rsETH Minting - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls Chainlink's `latestRoundData()` but discards all return values except `price`, performing zero staleness validation. This stale price flows directly into rsETH minting math for every LST deposit.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches the Chainlink price as follows:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

The `updatedAt` timestamp and `answeredInRound` are both silently discarded. There is no check of the form `block.timestamp - updatedAt <= maxAge` and no check that `answeredInRound >= roundId`. The contract declares the full return signature in its interface but ignores four of the five values. [2](#0-1) 

This oracle is the `IPriceFetcher` implementation registered in `LRTOracle.assetPriceOracle` for supported LST assets. `LRTOracle.getAssetPrice()` delegates directly to it:

```solidity
return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
``` [3](#0-2) 

`LRTDepositPool.getRsETHAmountToMint()` uses this price to compute how many rsETH tokens to mint per deposited LST:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [4](#0-3) 

This value is computed inside `_beforeDeposit`, which is called by the public `depositAsset()` function reachable by any user. [5](#0-4) 

For contrast, `ChainlinkOracleForRSETHPoolCollateral` (used in the pools path) does at least check `answeredInRound < roundID`, but still omits a time-based staleness bound: [6](#0-5) 

`ChainlinkPriceOracle` has neither check.

---

### Impact Explanation

If a supported LST asset depegs (e.g., a slashing event, liquidity crisis, or smart contract exploit causes its market price to drop) but the Chainlink feed has not yet updated — due to high gas costs, a heartbeat delay, or a deviation threshold not yet being crossed — the protocol continues to accept deposits at the last stale (inflated) price. Depositors receive more rsETH than the deposited assets are worth, directly diluting all existing rsETH holders and reducing the backing ratio of the token. Sustained stale-price deposits can push the protocol toward insolvency.

**Impact: High — Theft of unclaimed yield / protocol insolvency via inflated rsETH minting.**

---

### Likelihood Explanation

Chainlink feeds on Ethereum mainnet have heartbeat intervals of 1 hour to 24 hours depending on the feed. During periods of network congestion or when an asset's price moves slowly (staying within the deviation threshold), the feed can go many hours without updating. Any depositor — no special role required — can call `depositAsset()` at any time. The window of exposure is the entire duration between Chainlink updates, which is a normal operating condition, not an edge case.

---

### Recommendation

Add a configurable maximum age check in `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

require(answeredInRound >= roundId, "Stale round");
require(block.timestamp - updatedAt <= maxStaleness, "Stale price");
require(price > 0, "Invalid price");
```

`maxStaleness` should be configurable per asset (e.g., 1 hour for ETH/USD feeds, 24 hours for less-liquid LST feeds) and settable only by the admin role, mirroring the fix applied in the referenced Unitas Protocol report.

---

### Proof of Concept

1. Chainlink's stETH/ETH feed last updated 23 hours ago (within its 24-hour heartbeat, no update triggered).
2. stETH has depegged 5% on secondary markets due to a slashing event, but the on-chain Chainlink price still reflects the pre-depeg value.
3. Attacker calls `LRTDepositPool.depositAsset(stETH, amount, 0, "")`.
4. `getRsETHAmountToMint` calls `lrtOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns the stale inflated price.
5. Attacker receives ~5% more rsETH than the deposited stETH is currently worth.
6. Attacker immediately redeems or sells rsETH, extracting value from existing holders. [7](#0-6) [8](#0-7)
