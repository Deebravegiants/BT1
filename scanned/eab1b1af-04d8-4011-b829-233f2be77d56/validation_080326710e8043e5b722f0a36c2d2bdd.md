### Title
`LRTConfig.pauseAll()` Unable to Pause Protocol Contracts Due to Wrong Privilege Check - (File: contracts/LRTConfig.sol)

### Summary

`LRTConfig.pauseAll()` calls `pause()` on each protocol contract (`LRTDepositPool`, `LRTWithdrawalManager`, `LRTOracle`, `RSETH`, and all `NodeDelegator` instances). However, each `pause()` function enforces `onlyRole(LRTConstants.PAUSER_ROLE)` via `LRTConfigRoleChecker`, which checks `lrtConfig.hasRole(PAUSER_ROLE, msg.sender)`. When `LRTConfig` is the caller, `msg.sender` is `address(LRTConfig)` — not the original PAUSER_ROLE holder — so the call reverts unless `LRTConfig` itself has been explicitly granted `PAUSER_ROLE` in its own registry.

### Finding Description

`LRTConfig.pauseAll()` is the protocol's single-transaction emergency pause mechanism: [1](#0-0) 

It calls `lrtDepositPool.pause()`, `lrtWithdrawalManager.pause()`, `lrtOracle.pause()`, `rsETHContract.pause()`, and each `nodeDelegator.pause()`. Each of those `pause()` functions is guarded by `onlyRole(LRTConstants.PAUSER_ROLE)` from `LRTConfigRoleChecker`: [2](#0-1) [3](#0-2) [4](#0-3) 

`LRTConfigRoleChecker.onlyRole` resolves the role check against `lrtConfig.hasRole(role, msg.sender)`. When `LRTConfig` makes the cross-contract call, `msg.sender` inside each `pause()` is `address(LRTConfig)`, not the original EOA that called `pauseAll()`. For the call to succeed, `LRTConfig` itself must hold `PAUSER_ROLE` in its own AccessControl registry — which is not a standard deployment step and is not enforced anywhere in the codebase.

This is the direct analog of M-7: just as `OracleFactory.update()` calls `oracle.update()` but `oracle.update()` requires `msg.sender == factory().owner()` (not `factory()` itself), `LRTConfig.pauseAll()` calls `pause()` on each contract but those `pause()` functions require `msg.sender` to hold `PAUSER_ROLE` — not `LRTConfig` itself.

### Impact Explanation

`LRTConfig.pauseAll()` silently reverts on every call (unless `LRTConfig` was manually granted `PAUSER_ROLE`). The protocol's single-transaction emergency pause is non-functional. During an active exploit, the PAUSER_ROLE holder must call `pause()` on each contract individually across multiple transactions, leaving a window where some contracts remain unpaused and user funds can continue to be drained or frozen.

**Impact: Low — Contract fails to deliver promised returns (emergency pause broken); escalates to Medium (temporary freezing of funds) if the multi-step pause window is exploited.**

### Likelihood Explanation

The bug is triggered every time `pauseAll()` is called, which is the intended emergency response path. No special attacker action is needed — the function simply does not work as designed. Likelihood is high given any emergency scenario.

### Recommendation

Change the `pause()` functions in `LRTDepositPool`, `LRTWithdrawalManager`, `LRTOracle`, `RSETH`, and `NodeDelegator` to also accept a call from `address(lrtConfig)` (the factory/registry), or grant `LRTConfig` the `PAUSER_ROLE` during initialization. The cleanest fix mirrors the M-7 recommendation:

```solidity
// In LRTConfigRoleChecker or each pause() function:
modifier onlyPauserRoleOrLRTConfig() {
    require(
        lrtConfig.hasRole(LRTConstants.PAUSER_ROLE, msg.sender)
            || msg.sender == address(lrtConfig),
        "Not authorized"
    );
    _;
}
```

Alternatively, grant `PAUSER_ROLE` to `address(lrtConfig)` in the initializer of `LRTConfig`.

### Proof of Concept

1. Deploy the protocol normally. `LRTConfig` does **not** hold `PAUSER_ROLE` in its own registry.
2. Call `LRTConfig.pauseAll()` from an address that holds `PAUSER_ROLE`.
3. `LRTConfig` passes its own `onlyRole(PAUSER_ROLE)` check (the caller has the role).
4. `LRTConfig` calls `lrtDepositPool.pause()`. Inside `LRTDepositPool`, `msg.sender == address(LRTConfig)`.
5. `LRTConfigRoleChecker.onlyRole` checks `lrtConfig.hasRole(PAUSER_ROLE, address(LRTConfig))` → `false`.
6. The call reverts with an access-control error. `pauseAll()` fails entirely. [5](#0-4) [2](#0-1)

        if (!rsETHContract.paused()) rsETHContract.pause();

        address[] memory nodeDelegatorQueue = ILRTDepositPool(address(lrtDepositPool)).getNodeDelegatorQueue();

            }
        }

    }
