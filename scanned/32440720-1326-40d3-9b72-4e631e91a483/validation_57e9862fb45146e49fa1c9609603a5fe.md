### Title
Unbounded Gas Consumption in Publicly Callable `updateRSETHPrice()` via Nested Cross-Contract Loops - (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle.updateRSETHPrice()` is a public, permissionless function with no rate limiting or cooldown. Its internal execution traverses nested loops across all supported assets, all node delegators, and all queued EigenLayer withdrawals per NDC. As the protocol scales, the cumulative gas cost of this function grows unboundedly, eventually making it uncallable and permanently breaking the price oracle, fee minting, and the automatic safety-pause mechanism.

### Finding Description

`updateRSETHPrice()` at `LRTOracle.sol:87` is `public` with only a `whenNotPaused` guard — any unprivileged caller can invoke it at any time with no throttle:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Internally, `_updateRsETHPrice()` calls `_getTotalEthInProtocol()`, which iterates over every supported asset:

```solidity

    uint256 assetER = getAssetPrice(asset);
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
    ...
}
```

`getTotalAssetDeposits()` → `getAssetDistributionData()` in `LRTDepositPool.sol:447` then iterates over every NDC:

```solidity

    assetStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset);
```

`getAssetUnstaking()` in `NodeDelegator.sol:405–427` calls EigenLayer's `getQueuedWithdrawals()` and iterates over every pending withdrawal:

```solidity

    _getDelegationManager().getQueuedWithdrawals(address(this));

for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
    for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
        ...
    }

```

The total gas complexity is **O(assets × NDCs × queued\_withdrawals\_per\_NDC)**, with each inner iteration performing multiple cold external calls (`balanceOf`, `getAssetBalance`, `getAssetUnstaking`, `sharesToUnderlyingView`).

### Impact Explanation

If the cumulative gas cost of `updateRSETHPrice()` approaches or exceeds the Ethereum block gas limit (30M gas), the function becomes permanently uncallable. This causes:

1. **Stale rsETH price**: `rsETHPrice` is never updated. All deposits (`depositETH`, `depositAsset`) and withdrawals (`initiateWithdrawal`, `instantWithdrawal`) use the stale price, causing users to receive incorrect rsETH or asset amounts — a direct financial loss.
2. **Fee minting halted**: Protocol fees are never minted to treasury.
3. **Safety-pause mechanism broken**: The automatic protocol pause triggered when price drops below `pricePercentageLimit` (lines 277–281 of `LRTOracle.sol`) can never fire, removing a critical safety backstop.

Impact: **Medium — Unbounded gas consumption / Temporary freezing of protocol functionality.**

### Likelihood Explanation

The protocol initializes `maxNodeDelegatorLimit = 10` and supports several LSTs (stETH, ETHx, ETH, etc.). Each NDC can accumulate queued EigenLayer withdrawals up to `maxUncompletedWithdrawalCount`. With 5 assets × 10 NDCs × N queued withdrawals, each inner iteration performing 3–4 cold external calls (~5,000 gas each), the gas cost at N=50 pending withdrawals per NDC already reaches ~7.5M gas — and grows linearly with each new withdrawal. No cooldown, nonce, or block-based rate limit exists on the public entry point.

### Recommendation

1. Add a block-based cooldown to `updateRSETHPrice()` (e.g., enforce a minimum interval between calls).
2. Cache `getTotalAssetDeposits()` results within a single block to avoid redundant cross-contract traversals.
3. Decouple the `getAssetUnstaking()` loop from the price-update hot path; maintain a running tally updated on each `initiateUnstaking`/`completeUnstaking` call instead of recomputing from EigenLayer on every price update.

```text
            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
