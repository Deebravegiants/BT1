### Title
`tokenFeeBps` Permanently Defaults to Zero for Newly Added Tokens, Enabling Fee-Free Token Deposits - (File: contracts/pools/RSETHPool.sol)

### Summary
`RSETHPool.sol` maintains a per-token fee mapping (`tokenFeeBps`) that is never initialized when a token is added via `addSupportedToken`. It silently defaults to `0`, meaning every token deposit incurs zero protocol fee until an admin separately calls `setTokenFeeBps`. Any depositor who acts during this window receives more rsETH than the protocol intends, stealing fee yield.

### Finding Description
`RSETHPool.sol` uses two distinct fee variables:
- `feeBps` — a global rate applied to ETH deposits, set at initialization.
- `tokenFeeBps[token]` — a per-token rate applied to ERC-20 deposits, **never set when a token is added**.

`addSupportedToken` writes `supportedTokenOracle[token]` and `tokenBridge[token]` but leaves `tokenFeeBps[token]` at the Solidity default of `0`: [1](#0-0) 

The fee calculation for token deposits reads directly from this uninitialized slot: [2](#0-1) 

When `tokenFeeBps[token] == 0`, `fee = amount * 0 / 10_000 = 0`, so `amountAfterFee == amount`. The depositor receives rsETH computed on the full token amount with no fee deducted. The protocol's `feeEarnedInToken[token]` accumulates nothing.

The setter that can fix this requires `DEFAULT_ADMIN_ROLE` and is a separate, optional call: [3](#0-2) 

There is no enforcement that `setTokenFeeBps` must be called atomically with `addSupportedToken`, and `addSupportedToken` accepts no fee parameter.

### Impact Explanation
Every token deposit made between the moment a token is added and the moment `setTokenFeeBps` is called yields zero fee revenue. Depositors receive rsETH minted on the full token value rather than `value − fee`. This constitutes **theft of unclaimed yield** (protocol fee revenue) at the severity of a High finding per the allowed impact scope.

### Likelihood Explanation
`addSupportedToken` is gated by `TIMELOCK_ROLE`, so the pending call is publicly visible on-chain during the timelock delay. Sophisticated depositors can monitor the mempool, identify the exact block the token becomes active, and immediately deposit large amounts at 0% fee before the admin issues a separate `setTokenFeeBps` transaction. Even without front-running, any organic deposit volume between token activation and fee configuration is lost revenue. The admin has no on-chain reminder or enforcement to set the fee; the omission is structurally silent.

### Recommendation
Add a mandatory `feeBps` parameter to `addSupportedToken` and set `tokenFeeBps[token]` atomically inside the same call:

```solidity
function addSupportedToken(
    address token,
    address oracle,
    address bridge,
    uint256 feeBps_   // new required parameter
) external onlyRole(TIMELOCK_ROLE) {
    ...
    tokenFeeBps[token] = feeBps_;
    ...
}
```

Alternatively, add a `require(tokenFeeBps[token] != 0 || feeBps == 0)` guard inside `viewSwapRsETHAmountAndFee` to revert deposits until the fee is explicitly configured.

### Proof of Concept
1. Admin calls `addSupportedToken(wstETH, oracle, bridge)` — `tokenFeeBps[wstETH]` is `0`.
2. Attacker (or any user) immediately calls `deposit(wstETH, 1_000e18, "")`.
3. `viewSwapRsETHAmountAndFee(1_000e18, wstETH)` executes:
   - `feeBpsForToken = tokenFeeBps[wstETH] = 0`
   - `fee = 1_000e18 * 0 / 10_000 = 0`
   - `amountAfterFee = 1_000e18`
4. Attacker receives rsETH computed on the full `1_000e18` wstETH — zero fee collected.
5. `feeEarnedInToken[wstETH]` remains `0`; protocol earns nothing.
6. Admin later calls `setTokenFeeBps(wstETH, 30)` — but all deposits made before this call were fee-free. [4](#0-3)

```text
    function viewSwapRsETHAmountAndFee(
        uint256 amount,
        address token
    )
        public
        view
        onlySupportedToken(token)
        returns (uint256 rsETHAmount, uint256 fee)
    {
        uint256 feeBpsForToken = tokenFeeBps[token];
        fee = amount * feeBpsForToken / 10_000;
        uint256 amountAfterFee = amount - fee;

        // rate of rsETH in ETH
        uint256 rsETHToETHrate = getRate();

        // rate of token in ETH
        uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate();

        // Calculate the final rsETH amount
        rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
    }
```

```text
    function addSupportedToken(address token, address oracle, address bridge) external onlyRole(TIMELOCK_ROLE) {
        UtilLib.checkNonZeroAddress(token);
        UtilLib.checkNonZeroAddress(oracle);
        UtilLib.checkNonZeroAddress(bridge);

        if (supportedTokenOracle[token] != address(0)) {
            revert AlreadySupportedToken();
        }
        if (tokenBridge[token] != address(0)) {
            revert AlreadySupportedToken();
        }
        if (IOracle(oracle).getRate() == 0) {
            revert UnsupportedOracle();
        }
        supportedTokenList.push(token);
        supportedTokenOracle[token] = oracle;
        tokenBridge[token] = bridge;

        emit AddSupportedToken(token, oracle, bridge);
    }
```
