### Title
Unbounded Nested Loop in `NodeDelegator.getAssetUnstaking()` Blocks Deposits and Price Updates When Queued Withdrawals Accumulate - (File: contracts/NodeDelegator.sol)

---

### Summary

`NodeDelegator.getAssetUnstaking()` contains a nested loop over all EigenLayer-queued withdrawals with no gas guard. This function is called for every NDC for every supported asset inside the deposit and price-update critical paths. When queued withdrawals accumulate — especially via forced undelegations that bypass the vault's soft counter — the aggregate gas cost can exceed the block gas limit, permanently freezing user deposits and rsETH price updates.

---

### Finding Description

`NodeDelegator.getAssetUnstaking()` fetches all pending withdrawals from EigenLayer's `DelegationManager` and iterates over them in a nested loop:

    _getDelegationManager().getQueuedWithdrawals(address(this));

for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
    IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];
    for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
        ...
    }
}
``` [1](#0-0) 

This function is called inside `LRTDepositPool.getAssetDistributionData()` and `getETHDistributionData()`, once per NDC per loop iteration:

```solidity
// contracts/LRTDepositPool.sol lines 447–456
for (uint256 i; i < ndcsCount;) {
    assetLyingInNDCs += IERC20(asset).balanceOf(nodeDelegatorQueue[i]);
    assetStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset);
    assetUnstakingFromEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset);
    ...
}
``` [2](#0-1) 

`getTotalAssetDeposits()` calls `getAssetDistributionData()`, and is itself called from two critical paths:

**Path 1 — User deposit:**
`depositETH()` / `depositAsset()` → `_beforeDeposit()` → `_checkIfDepositAmountExceedesCurrentLimit()` → `getTotalAssetDeposits()`. [3](#0-2) 

**Path 2 — Public price update:**
`updateRSETHPrice()` → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getTotalAssetDeposits()` for each supported asset. [4](#0-3) 

The total gas complexity is **O(assets × NDCs × withdrawals × strategies\_per\_withdrawal)**. With up to 10 NDCs and 5 supported assets, `getQueuedWithdrawals()` is called up to 50 times per transaction, each returning the full pending withdrawal list for that NDC.

The protocol attempts to bound this via `maxUncompletedWithdrawalCount` (capped at 80 in `LRTUnstakingVault`): [5](#0-4) 

However, the code's own comment reveals the mitigation is incomplete:

```solidity
// 120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price
// Need buffer for theoretical operator forced undelegations (ndc count * asset count = 15)
if (_maxUncompletedWithdrawalCount > 80) {
``` [6](#0-5) 

EigenLayer operators can force-undelegate an NDC at any time, creating new queued withdrawals that are **not** tracked by `uncompletedWithdrawalCount`. The comment acknowledges up to 15 extra withdrawals from forced undelegations (NDC count × asset count). If the vault counter is already near 80 and forced undelegations push the real EigenLayer count above 120, the gas threshold is breached. The `setUncompletedWithdrawalCount()` resync function is manager-only and cannot be called atomically during a deposit. [7](#0-6) 

---

### Impact Explanation

When the aggregate queued-withdrawal count across all NDCs exceeds the gas-safe threshold, every call to `depositETH()`, `depositAsset()`, and `updateRSETHPrice()` reverts with out-of-gas. Deposits are frozen and the rsETH price cannot be updated. Because the condition is triggered by the state of EigenLayer's withdrawal queue (not by a single transaction), recovery requires operators to complete enough withdrawals to reduce the count — a process that takes at least the EigenLayer withdrawal delay (currently 7 days). During this window, all user deposits are frozen.

**Impact: Medium — Temporary freezing of funds / Unbounded gas consumption.**

---

### Likelihood Explanation

The protocol operates multiple NDCs, each capable of accumulating queued withdrawals from `initiateUnstaking()` and `undelegate()` calls. Forced undelegations by EigenLayer operators are a documented, non-adversarial protocol mechanism. The developers themselves sized the cap at 80 specifically because they computed that 120 is the gas-safe ceiling and wanted a 40-unit buffer — but the buffer is consumed by forced undelegations (up to 15 per the comment). A period of high unstaking activity combined with one or more forced undelegations is a realistic operational scenario, not a theoretical one.

---

### Recommendation

1. **Cache `getQueuedWithdrawals()` results**: In `_getTotalEthInProtocol()`, avoid calling `getAssetUnstaking()` once per asset per NDC. Instead, fetch each NDC's queued withdrawals once and compute all asset amounts in a single pass.

2. **Add a gas guard** analogous to the Compound recommendation: before each NDC iteration in `getAssetDistributionData()` / `getETHDistributionData()`, check `gasleft()` and revert or return early with a partial result if gas is insufficient.

3. **Store unstaking amounts as cached state**: Update a per-NDC per-asset `assetUnstaking` mapping whenever `initiateUnstaking()` / `completeUnstaking()` / `undelegate()` are called, and read from that mapping instead of re-iterating EigenLayer's withdrawal queue on every deposit.

---

### Proof of Concept

1. Protocol has 5 supported assets, 3 NDCs, and `maxUncompletedWithdrawalCount = 80`.
2. Operators call `initiateUnstaking()` across NDCs until `uncompletedWithdrawalCount = 78`.
3. An EigenLayer operator force-undelegates one NDC, creating 5 new queued withdrawals (one per strategy) directly in EigenLayer's `DelegationManager`, bypassing the vault counter. Real count = 83.
4. A user calls `depositETH()`. The call chain reaches `getAssetDistributionData()`, which calls `getAssetUnstaking()` for each of the 3 NDCs, for each of the 5 assets = 15 calls to `getQueuedWithdrawals()`, each iterating over up to 28 withdrawals × multiple strategies.
5. The transaction runs out of gas and reverts. All subsequent `depositETH()`, `depositAsset()`, and `updateRSETHPrice()` calls also revert until operators complete enough withdrawals to reduce the EigenLayer-side count below the gas-safe threshold — a process gated by the 7-day EigenLayer withdrawal delay.     function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }

        ILRTDepositPool lrtDepositPool = ILRTDepositPool(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IDelegationManager delegationManager =
            IDelegationManager(lrtConfig.getContract(LRTConstants.EIGEN_DELEGATION_MANAGER));
        address[] memory nodeDelegatorQueue = lrtDepositPool.getNodeDelegatorQueue();
        uint256 totalQueued;
        for (uint256 i = 0; i < nodeDelegatorQueue.length; i++) {
            address nodeDelegator = nodeDelegatorQueue[i];
            (IDelegationManager.Withdrawal[] memory queuedWithdrawals,) =
                delegationManager.getQueuedWithdrawals(nodeDelegator);
            totalQueued += queuedWithdrawals.length;
        }

        uncompletedWithdrawalCount = totalQueued;

        emit UncompletedWithdrawalCountSet(totalQueued);
    }
