### Title
Permanent Freezing of Unclaimed Yield via Non-Payable Treasury Blocking `emergencyWithdrawFromAave` — (`contracts/LRTWithdrawalManager.sol`)

### Summary
`emergencyWithdrawFromAave` unconditionally calls `_collectInterestToTreasury()` before withdrawing principal. If the protocol treasury is a non-payable contract (or any contract that rejects raw ETH), the treasury transfer inside `_collectInterestToTreasury` reverts, causing the entire emergency withdrawal to revert. Because every code path that can drain the interest portion goes through `_collectInterestToTreasury`, the accrued yield becomes permanently unrecoverable.

### Finding Description

`emergencyWithdrawFromAave` is gated to `PAUSER_ROLE` and is documented as the last-resort recovery path for Aave funds:

```solidity
// contracts/LRTWithdrawalManager.sol  line 551-563
function emergencyWithdrawFromAave(uint256 amount) external nonReentrant onlyRole(LRTConstants.PAUSER_ROLE) {
    if (!isAaveIntegrationEnabled) revert AaveIntegrationNotEnabled();

    uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
    if (aaveBalance == 0) revert InsufficientAaveBalance();

    // First collect any accrued interest to treasury
    _collectInterestToTreasury();          // ← unconditional, no try/catch

    uint256 withdrawnAmount = _withdrawFromAave(amount);
    emit EmergencyWithdrawFromAave(withdrawnAmount, address(this));
}
``` [1](#0-0) 

Inside `_collectInterestToTreasury`, after withdrawing the interest from Aave into the contract, the ETH is forwarded to the treasury with a raw call:

```solidity
// contracts/LRTWithdrawalManager.sol  line 954-958
aaveWETHGateway.withdrawETH(aavePool, interestAmount, address(this));

address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
(bool sent,) = payable(treasury).call{ value: interestAmount }("");
if (!sent) revert TreasuryTransferFailed();
``` [2](#0-1) 

If the treasury address is a contract without a `receive()` or `fallback()` function, `sent` is `false`, `TreasuryTransferFailed` is thrown, and the entire transaction (including the `_withdrawFromAave` call on line 560) reverts atomically.

Every other administrative path that could drain the interest suffers the same fate:

| Function | Also calls `_collectInterestToTreasury`? |
|---|---|
| `setAaveIntegrationEnabled(false)` | Yes — line 490 |
| `configureAaveIntegration(...)` | Yes — line 442 |
| `collectInterestToTreasury()` | Yes — line 544 | [3](#0-2) [4](#0-3) 

The only path that bypasses `_collectInterestToTreasury` is `_withdrawFromAave`, which is called from `_processWithdrawalCompletion` (user withdrawals). However, `_withdrawFromAave` caps its withdrawal at `totalETHDepositedToAave` (the principal):

```solidity
// line 912
uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave ? aaveBalance : totalETHDepositedToAave;
``` [5](#0-4) 

The interest portion (`aaveBalance - totalETHDepositedToAave`) is never touched by this path and remains permanently locked in Aave.

### Impact Explanation
The accrued Aave yield (interest) is permanently unrecoverable. No privileged role can extract it because every administrative function that handles the interest reverts when the treasury is non-payable. This matches **Medium — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The treasury address is set via `lrtConfig` and is never validated to be ETH-payable anywhere in the codebase. A Gnosis Safe multisig configured without a fallback handler, a pure ERC20 treasury contract, or any contract upgraded to remove ETH acceptance would trigger this condition. The protocol has no on-chain guard preventing such a configuration.

### Recommendation
Wrap the `_collectInterestToTreasury()` call inside `emergencyWithdrawFromAave` (and optionally in `setAaveIntegrationEnabled`/`configureAaveIntegration`) with a `try/catch` or a boolean return, so that a treasury transfer failure does not block principal recovery:

```solidity
function emergencyWithdrawFromAave(uint256 amount) external nonReentrant onlyRole(LRTConstants.PAUSER_ROLE) {
    if (!isAaveIntegrationEnabled) revert AaveIntegrationNotEnabled();
    uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
    if (aaveBalance == 0) revert InsufficientAaveBalance();

    // Best-effort interest collection; do not block principal recovery on failure
    try this.collectInterestToTreasuryExternal() { } catch { }

    uint256 withdrawnAmount = _withdrawFromAave(amount);
    emit EmergencyWithdrawFromAave(withdrawnAmount, address(this));
}
```

Alternatively, add a separate `rescueStuckInterest(address recipient)` function callable by admin that bypasses the treasury address, or validate at configuration time that the treasury can accept ETH.

### Proof of Concept

```solidity
// Fork test (Mainnet fork, Foundry)
function test_emergencyWithdraw_permanentlyFrozenYield() public {
    // 1. Deploy a non-payable treasury contract
    NonPayableTreasury treasury = new NonPayableTreasury();
    vm.prank(admin);
    lrtConfig.setContract(LRTConstants.PROTOCOL_TREASURY, address(treasury));

    // 2. Deposit ETH to Aave via unlockQueue (normal flow)
    vm.prank(operator);
    withdrawalManager.unlockQueue(...);

    // 3. Warp forward to accrue Aave interest
    vm.roll(block.number + 100_000);

    // 4. Confirm interest has accrued
    uint256 interest = withdrawalManager.getAccruedInterest();
    assertGt(interest, 0);

    // 5. PAUSER_ROLE calls emergencyWithdrawFromAave — always reverts
    vm.prank(pauser);
    vm.expectRevert(LRTWithdrawalManager.TreasuryTransferFailed.selector);
    withdrawalManager.emergencyWithdrawFromAave(type(uint256).max);

    // 6. Interest remains in Aave — permanently frozen
    assertEq(withdrawalManager.getAccruedInterest(), interest);
}

contract NonPayableTreasury {
    // No receive() or fallback() — rejects all ETH
}
```
