### Title
In-Flight Message Dropped on Endpoint Migration Causes Rate Staleness — (`contracts/cross-chain/CrossChainRateReceiver.sol`)

---

### Summary

`updateLayerZeroEndpoint` performs an immediate, atomic overwrite of `layerZeroEndpoint`. Any LayerZero message already committed to the old endpoint but not yet delivered will fail the `msg.sender` guard in `lzReceive` and be silently dropped. The receiver's `rate` then remains frozen at its pre-migration value until the owner manually triggers a fresh `updateRate()` through the new endpoint.

---

### Finding Description

`lzReceive` enforces a strict caller check:

```solidity
// CrossChainRateReceiver.sol line 83
require(msg.sender == layerZeroEndpoint, "Sender should be lz endpoint");
``` [1](#0-0) 

`updateLayerZeroEndpoint` replaces the stored address in a single storage write with no transition period:

```solidity
// CrossChainRateReceiver.sol lines 54-58
function updateLayerZeroEndpoint(address _layerZeroEndpoint) external onlyOwner {
    layerZeroEndpoint = _layerZeroEndpoint;
    emit LayerZeroEndpointUpdated(_layerZeroEndpoint);
}
``` [2](#0-1) 

LayerZero guarantees ordered, exactly-once delivery per channel, but delivery is asynchronous. A message submitted to the old endpoint before the migration call is finalized will be delivered by the **old** endpoint address. After the migration, `msg.sender` is the old endpoint while `layerZeroEndpoint` is the new one — the `require` reverts, the message is dropped, and `rate` / `lastUpdated` are never written.

There is no fallback path: the contract has no `retryMessage`, no secondary-endpoint allowlist, and no owner-callable `setRate` escape hatch. The only recovery is for the owner to call `updateRate()` on the provider side again, which sends a fresh message through the new endpoint.

---

### Impact Explanation

During the migration window the stale `rate` is served to every downstream consumer. Pool contracts such as `RSETHPoolV2ExternalBridge` and `RSETHPoolV2NBA` call `IOracle(rsETHOracle).getRate()` directly for every deposit/swap:

```solidity
// RSETHPoolV2NBA.sol lines 129-132
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
``` [3](#0-2) 

If the oracle in use is `RSETHRateReceiver` (or `AGETHRateReceiver`), users receive rsETH amounts computed from the frozen pre-migration rate. No funds are lost, but the contract fails to deliver the promised up-to-date exchange rate — matching the **Low** scope: *"Contract fails to deliver promised returns, but doesn't lose value."*

---

### Likelihood Explanation

Endpoint migration is a routine, documented admin operation (the function exists precisely for this purpose). The admin has no on-chain signal that a message is in-flight; LayerZero message finality is off-chain. The race condition is therefore plausible during any real migration and does not require any malicious actor.

---

### Recommendation

Add a two-step migration or a grace-period allowlist so the old endpoint can still deliver during the transition:

```solidity
address public pendingLayerZeroEndpoint;

function proposeLayerZeroEndpoint(address _new) external onlyOwner {
    pendingLayerZeroEndpoint = _new;
}

function acceptLayerZeroEndpoint() external onlyOwner {
    layerZeroEndpoint = pendingLayerZeroEndpoint;
    pendingLayerZeroEndpoint = address(0);
    emit LayerZeroEndpointUpdated(layerZeroEndpoint);
}

// in lzReceive:
require(
    msg.sender == layerZeroEndpoint || msg.sender == pendingLayerZeroEndpoint,
    "Sender should be lz endpoint"
);
```

Alternatively, expose an owner-callable `forceSetRate(uint256)` so the owner can manually push a fresh rate if a message is dropped.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";
import "contracts/cross-chain/RSETHRateReceiver.sol";

contract MigrationDropTest is Test {
    RSETHRateReceiver receiver;
    address oldEndpoint = address(0xAA);
    address newEndpoint = address(0xBB);
    address provider   = address(0xCC);
    uint16  srcChain   = 101;

    function setUp() public {
        receiver = new RSETHRateReceiver(srcChain, provider, oldEndpoint);
    }

    function test_inFlightMessageDropped() public {
        // 1. Simulate old endpoint delivering a rate update (succeeds before migration)
        bytes memory srcAddr = abi.encodePacked(provider);
        bytes memory payload = abi.encode(uint256(1.05e18));

        vm.prank(oldEndpoint);
        receiver.lzReceive(srcChain, srcAddr, 0, payload);
        assertEq(receiver.rate(), 1.05e18);

        // 2. Owner migrates to new endpoint
        receiver.updateLayerZeroEndpoint(newEndpoint);

        // 3. Old endpoint tries to deliver an in-flight message — REVERTS
        vm.prank(oldEndpoint);
        vm.expectRevert("Sender should be lz endpoint");
        receiver.lzReceive(srcChain, srcAddr, 1, abi.encode(uint256(1.06e18)));

        // 4. Rate is frozen at pre-migration value; new endpoint has not yet sent anything
        assertEq(receiver.rate(), 1.05e18); // stale — 1.06e18 was never applied
    }
}
