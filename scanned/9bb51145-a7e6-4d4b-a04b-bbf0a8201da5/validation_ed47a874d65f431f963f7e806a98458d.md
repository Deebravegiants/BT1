### Title
Missing Staleness Check on Chainlink `latestRoundData()` Enables Stale Price to Inflate rsETH Price and Steal Protocol Yield - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards the `updatedAt` timestamp, accepting any price regardless of age. Because `updateRSETHPrice()` is a public, permissionless function, any caller can trigger rsETH price recalculation using a stale inflated asset price, causing the protocol to mint fee tokens to the treasury based on phantom yield.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches the Chainlink price as:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

The five return values of `latestRoundData()` are `(roundId, answer, startedAt, updatedAt, answeredInRound)`. The `updatedAt` field — the timestamp of the last oracle update — is completely ignored. There is no check of the form `block.timestamp - updatedAt <= MAX_STALENESS`. [1](#0-0) 

This price is consumed by `LRTOracle.getAssetPrice()`, which feeds into `_getTotalEthInProtocol()`, which sums `totalAssetAmt * assetER` across all supported LST assets. [2](#0-1) 

`_getTotalEthInProtocol()` is called inside `_updateRsETHPrice()`, which is triggered by the public, permissionless `updateRSETHPrice()`. [3](#0-2) 

Inside `_updateRsETHPrice()`, if `totalETHInProtocol > previousTVL`, the delta is treated as yield and a protocol fee is minted to the treasury: [4](#0-3) 

```solidity
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
```

The fee is then minted as rsETH to the treasury: [5](#0-4) 

```solidity
uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);
...
IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
```

Contrast this with `ChainlinkOracleForRSETHPoolCollateral.sol`, which at least checks `answeredInRound < roundID` and `timestamp == 0`, but still lacks a `block.timestamp - timestamp > threshold` guard. [6](#0-5) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

When a Chainlink LST/ETH feed is stale and its last reported price is higher than the true current price (e.g., during low-volatility periods where the feed has not heartbeat-updated), an attacker calls the public `updateRSETHPrice()`. The inflated `totalETHInProtocol` exceeds `previousTVL`, the delta is classified as yield, and the protocol mints rsETH fee tokens to the treasury. This rsETH is backed by no real ETH, diluting all existing rsETH holders. The attacker does not need to hold any tokens or provide any capital; they only need to time the call to coincide with a stale feed.

A secondary scenario: a stale low price deflates `totalETHInProtocol` below `previousTVL`, triggering the downside-protection pause and temporarily freezing all deposits and withdrawals (Medium — temporary freezing of funds). [7](#0-6) 

---

### Likelihood Explanation

Chainlink feeds operate on a heartbeat (e.g., 24 h for ETH/ETH-correlated pairs) and a deviation threshold. During low-volatility periods, feeds routinely go many hours without updating. The attack requires no capital, no privileged role, and no flashloan — only a public call to `updateRSETHPrice()` at the right moment. This is a realistic, low-effort condition.

---

### Recommendation

Add a staleness guard in `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

require(answeredInRound >= roundId, "Stale round");
require(updatedAt != 0, "Incomplete round");
require(block.timestamp - updatedAt <= MAX_STALENESS, "Stale price");
require(price > 0, "Non-positive price");
```

`MAX_STALENESS` should be set per feed based on its documented heartbeat (e.g., 25 hours for a 24 h heartbeat feed). Apply the same pattern to `RSETHPriceFeed.latestRoundData()`. [8](#0-7) 

---

### Proof of Concept

1. Chainlink's ETH-correlated LST feed (e.g., rETH/ETH) has not updated for 20 hours; its last reported price is 1.05 ETH per rETH, while the true current price is 1.04 ETH.
2. Attacker calls `LRTOracle.updateRSETHPrice()` (no role required). [3](#0-2) 
3. `_getTotalEthInProtocol()` multiplies the rETH balance by the stale 1.05 rate, inflating `totalETHInProtocol`. [9](#0-8) 
4. `totalETHInProtocol > previousTVL` → `rewardAmount` is non-zero → `protocolFeeInETH` is computed. [10](#0-9) 
5. `rsethAmountToMintAsProtocolFee` is minted to the treasury, diluting all rsETH holders with no corresponding real ETH backing. [11](#0-10) 
6. `rsETHPrice` is updated to the inflated value, which also affects subsequent deposits and withdrawals.
