### Title
Stale `rsETHPrice` Used in Deposit Mint Calculation Allows Depositors to Receive Excess rsETH at Existing Holders' Expense - (File: `contracts/LRTDepositPool.sol`)

---

### Summary

`LRTDepositPool::depositAsset` and `depositETH` compute the rsETH mint amount using `LRTOracle::rsETHPrice`, which is a **stored state variable** that must be explicitly refreshed by calling `updateRSETHPrice()`. Neither deposit function refreshes the price before minting. When staking rewards have accrued but the price has not yet been updated, the stored price is stale (lower than the true value), causing depositors to receive more rsETH than they are entitled to, at the expense of existing rsETH holders.

---

### Finding Description

`LRTOracle` stores the rsETH/ETH exchange rate as a plain state variable: [1](#0-0) 

This value is only updated when `updateRSETHPrice()` (public, `whenNotPaused`) or `updateRSETHPriceAsManager()` is explicitly called: [2](#0-1) 

`LRTDepositPool::getRsETHAmountToMint` reads this stored value directly: [3](#0-2) 

`_beforeDeposit` calls `getRsETHAmountToMint` without first refreshing the price: [4](#0-3) 

Both `depositETH` and `depositAsset` invoke `_beforeDeposit` with no price update step: [5](#0-4) 

The mint formula is:

```
rsethAmountToMint = (depositAmount × assetPrice) / rsETHPrice
```

When `rsETHPrice` is stale (lower than the true current value), the denominator is too small, so `rsethAmountToMint` is inflated.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Staking rewards (stETH rebases, EigenLayer ETH rewards) continuously increase the true rsETH/ETH rate. Between successive `updateRSETHPrice()` calls, the stored `rsETHPrice` lags behind reality. A depositor who deposits during this window receives more rsETH than their deposit is worth at the true rate. When `updateRSETHPrice()` is eventually called, the price jumps to reflect the accrued rewards — but the depositor already holds the excess rsETH, permanently diluting all existing holders. The yield that should have accrued to existing holders is instead captured by the late depositor. [6](#0-5) 

---

### Likelihood Explanation

**Medium.** `updateRSETHPrice()` is called by off-chain bots/operators on a periodic schedule. Any gap between reward accrual and the next price update creates the exploitable window. An attacker can passively monitor `rsETHPrice` on-chain, compare it against the live TVL (computable from `_getTotalEthInProtocol`), and deposit precisely when the stored price is most stale. No privileged access is required; the entry path is the standard public `depositAsset`/`depositETH` interface. [7](#0-6) 

---

### Recommendation

Call `updateRSETHPrice()` (or its internal equivalent `_updateRsETHPrice()`) at the start of `depositAsset`, `depositETH`, and `initiateWithdrawal` before any amount calculation, mirroring the pattern used in the referenced fix. Alternatively, expose an internal `_updateRsETHPrice` call inside `_beforeDeposit` so every deposit atomically refreshes the price before minting. [4](#0-3) 

---

### Proof of Concept

1. At `t=0`, `rsETHPrice = 1.00 ETH` (stored). True TVL-implied price is also `1.00 ETH`.
2. Staking rewards accrue. True price rises to `1.01 ETH`. `updateRSETHPrice()` has not been called yet; stored `rsETHPrice` remains `1.00 ETH`.
3. Attacker calls `depositETH{value: 1 ETH}(0, "")`.
4. `getRsETHAmountToMint` computes: `(1e18 × 1e18) / 1.00e18 = 1.00 rsETH`.
   - At the true price of `1.01 ETH`, the correct mint would be: `1e18 / 1.01e18 ≈ 0.9901 rsETH`.
   - Attacker receives `≈ 0.0099 rsETH` excess.
5. Operator calls `updateRSETHPrice()`. `rsETHPrice` updates to `1.01 ETH`.
6. Attacker's `1.00 rsETH` is now worth `1.01 ETH` — a `0.01 ETH` gain extracted from existing holders' accrued yield.

The gap scales with the size of the deposit and the duration since the last price update. During high-reward periods or bot outages, the window and magnitude both increase. [3](#0-2) [1](#0-0)

    function _updateRsETHPrice() internal {
        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;
        }

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

        // only take fee if TVL increased and protocol is not paused
        uint256 protocolFeeInETH = 0;
        if (!protocolPaused && totalETHInProtocol > previousTVL) {
            uint256 rewardAmount = totalETHInProtocol - previousTVL;
            protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
        }

        // compute new rsETH price based on total ETH minus fee
        uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    function getRsETHAmountToMint(
        address asset,
        uint256 amount
    )
        public
        view
        override
        returns (uint256 rsethAmountToMint)
    {
        // setup oracle contract
        address lrtOracleAddress = lrtConfig.getContract(LRTConstants.LRT_ORACLE);
        ILRTOracle lrtOracle = ILRTOracle(lrtOracleAddress);

        // calculate rseth amount to mint based on asset amount and asset exchange rate
        rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
    }
