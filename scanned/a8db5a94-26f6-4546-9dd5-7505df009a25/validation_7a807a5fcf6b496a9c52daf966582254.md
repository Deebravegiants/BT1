### Title
Missing Staleness Check in `ChainlinkPriceOracle.getAssetPrice()` Allows Stale Price to Deflate `rsETHPrice` and Over-Mint rsETH - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards the `updatedAt` and `answeredInRound` return values, accepting any price regardless of age. Because `LRTOracle.updateRSETHPrice()` is a public, permissionless function, an attacker can wait for a Chainlink feed to go stale at a lower-than-actual price, trigger a price update that deflates the stored `rsETHPrice`, and then deposit ETH to mint rsETH at a discount — stealing value from existing rsETH holders.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches the latest Chainlink round data but only reads the `price` field:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52
(, int256 price,,,) = priceFeed.latestRoundData();
```

The `updatedAt` timestamp and `answeredInRound` counter — the two standard Chainlink staleness indicators — are completely ignored. [1](#0-0) 

By contrast, the protocol's own `ChainlinkOracleForRSETHPoolCollateral` (used for L2 pool collateral) explicitly validates both fields before accepting a price:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
``` [2](#0-1) 

`ChainlinkPriceOracle` is the oracle registered in `LRTOracle.assetPriceOracle` for supported LSTs (stETH, rETH, etc.). Its output flows into two critical paths:

**Path 1 — `rsETHPrice` update (public, permissionless):**
`LRTOracle.updateRSETHPrice()` → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(asset)` for every supported LST → stale price accepted → `totalETHInProtocol` is under-counted → `rsETHPrice` is deflated and written to storage. [3](#0-2) [4](#0-3) 

**Path 2 — rsETH minting:**
`LRTDepositPool.getRsETHAmountToMint()` divides by the stored `rsETHPrice`:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [5](#0-4) 

A deflated `rsETHPrice` in the denominator directly inflates the rsETH amount minted per unit of ETH deposited.

---

### Impact Explanation

When a Chainlink feed for a supported LST goes stale at a price lower than the true market price:

1. `_getTotalEthInProtocol()` under-counts the protocol's ETH value.
2. `rsETHPrice = totalETHInProtocol / rsethSupply` is deflated.
3. A depositor calling `depositETH()` receives `ETH_deposited * 1e18 / deflatedRsETHPrice` rsETH — more rsETH than the deposited ETH is actually worth.
4. When the oracle price corrects and `rsETHPrice` is updated upward, the attacker's rsETH is redeemable for more ETH than they deposited.

This constitutes **theft of yield from existing rsETH holders** via dilution: the attacker's over-minted rsETH claims a larger share of the protocol's total ETH, reducing the redemption value for all other holders.

**Impact: High — theft of unclaimed yield.**

---

### Likelihood Explanation

Chainlink price feeds operate on a deviation threshold (typically 0.5%) plus a heartbeat (typically 24 hours). During periods of low volatility, a feed can remain at its last reported price for up to 24 hours without triggering an update. Network congestion, sequencer downtime (on L2), or oracle node failures can also cause feeds to go stale. The attack requires no special privileges: `updateRSETHPrice()` is public and `depositETH()` / `depositAsset()` are open to any user.

**Likelihood: Low-Medium** — requires a Chainlink feed to go stale at a lower price, but this is a realistic on-chain condition, not a theoretical one.

---

### Recommendation

Add staleness validation to `ChainlinkPriceOracle.getAssetPrice()`, mirroring the checks already present in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optionally: if (block.timestamp - updatedAt > MAX_STALENESS) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

---

### Proof of Concept

1. Assume stETH/ETH Chainlink feed last reported `1.05e18` (true value) but has gone stale; it still returns `1.02e18` (3% below actual).
2. Protocol holds 1000 stETH. True `totalETHInProtocol` = `1050 ETH`. Stale `totalETHInProtocol` = `1020 ETH`.
3. With `rsethSupply = 1000`, true `rsETHPrice = 1.05e18`. Stale `rsETHPrice = 1.02e18`.
4. Attacker calls `LRTOracle.updateRSETHPrice()` — stored `rsETHPrice` drops to `1.02e18`.
5. Attacker calls `LRTDepositPool.depositETH{value: 10 ether}()`:
   - `rsethAmountToMint = 10e18 * 1e18 / 1.02e18 ≈ 9.804 rsETH`
   - Fair amount at true price: `10e18 / 1.05e18 ≈ 9.524 rsETH`
   - Excess: `≈ 0.28 rsETH` (~2.9% over-mint)
6. When the Chainlink feed updates and `rsETHPrice` corrects to `1.05e18`, the attacker's `9.804 rsETH` is redeemable for `≈ 10.29 ETH` — a profit of `≈ 0.29 ETH` at the expense of existing holders. [1](#0-0) [6](#0-5) [7](#0-6)

        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;
        }

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

        // only take fee if TVL increased and protocol is not paused
        uint256 protocolFeeInETH = 0;
        if (!protocolPaused && totalETHInProtocol > previousTVL) {
            uint256 rewardAmount = totalETHInProtocol - previousTVL;
            protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
        }

        // compute new rsETH price based on total ETH minus fee
        uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
