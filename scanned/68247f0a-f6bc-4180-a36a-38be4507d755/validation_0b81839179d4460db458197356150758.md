`RsETHTokenWrapper` does **not** inherit `Recoverable` and contains no sweep or emergency-recovery function. The vulnerability is real and concrete. Here is the full analysis:

---

### Title
`removeAllowedToken` Permanently Freezes Token Balances With No Recovery Path — (`contracts/L2/RsETHTokenWrapper.sol`)

---

### Summary

`RsETHTokenWrapper.removeAllowedToken` sets `allowedTokens[_asset] = false` with no check on the contract's current balance of that token. After removal, every code path that returns tokens to users — `withdraw` and `withdrawTo` — unconditionally reverts with `TokenNotAllowed`. The contract inherits no recovery mechanism (`Recoverable` exists in the codebase but is not used here), leaving any balance of the de-listed token permanently frozen inside the wrapper.

---

### Finding Description

`removeAllowedToken` (restricted to `TIMELOCK_ROLE`) performs only a "token must currently be allowed" guard before flipping the flag:

```solidity
// contracts/L2/RsETHTokenWrapper.sol  lines 180-185
function removeAllowedToken(address _asset) external onlyRole(TIMELOCK_ROLE) {
    if (!allowedTokens[_asset]) revert TokenNotAllowed();
    allowedTokens[_asset] = false;
    emit TokenRemoved(_asset);
}
``` [1](#0-0) 

There is no `require(ERC20Upgradeable(_asset).balanceOf(address(this)) == 0)` guard.

Both user-facing withdrawal paths delegate to `_withdraw`, whose first line is:

```solidity
// contracts/L2/RsETHTokenWrapper.sol  lines 120-121
function _withdraw(address _asset, address _to, uint256 _amount) internal {
    if (!allowedTokens[_asset]) revert TokenNotAllowed();
``` [2](#0-1) 

Once the flag is cleared, every call to `withdraw` or `withdrawTo` for that asset reverts unconditionally.

The contract's full inheritance chain is `Initializable`, `AccessControlUpgradeable`, `ERC20Upgradeable`, `ERC20PermitUpgradeable` — none of which provide a token-rescue function. [3](#0-2) 

The codebase does contain a `Recoverable` base contract with `recoverTokens`, but `RsETHTokenWrapper` does not inherit it. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Every alt-rsETH token held by the wrapper as backing for outstanding wrsETH is irretrievable once the token is de-listed. Holders of wrsETH cannot burn their shares for the underlying asset; the 1:1 backing is destroyed. This affects both principal deposited by users and any yield accrued by the alt-rsETH token itself while held in the wrapper.

---

### Likelihood Explanation

**Medium.** The `TIMELOCK_ROLE` is a governance/timelock actor, not a compromised key. Token migrations are a routine governance action on L2 deployments (e.g., replacing one bridge's alt-rsETH with another). A governance proposal to remove a deprecated token is entirely plausible and would not require malicious intent — only the absence of a balance check in the current implementation. The timelock delay provides a window for detection, but the contract offers no in-protocol remedy once the transaction executes.

---

### Recommendation

1. **Add a balance guard in `removeAllowedToken`:**
   ```solidity
   function removeAllowedToken(address _asset) external onlyRole(TIMELOCK_ROLE) {
       if (!allowedTokens[_asset]) revert TokenNotAllowed();
       if (ERC20Upgradeable(_asset).balanceOf(address(this)) != 0)
           revert TokenBalanceNonZero();
       allowedTokens[_asset] = false;
       emit TokenRemoved(_asset);
   }
   ```
2. **Alternatively (or additionally), inherit `Recoverable`** so that `DEFAULT_ADMIN_ROLE` can always rescue stranded tokens, consistent with the pattern already used elsewhere in the codebase.
3. **Or provide a dedicated migration path** that atomically swaps the old token for a new allowed token before de-listing.

### Proof of Concept

pragma solidity 0.8.27;

import "forge-std/Test.sol";
import "contracts/L2/RsETHTokenWrapper.sol";
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract MockAltRsETH is ERC20 {
    constructor() ERC20("altRsETH", "altRsETH") { _mint(msg.sender, 1000e18); }
}

contract FreezePoC is Test {
    RsETHTokenWrapper wrapper;
    MockAltRsETH altToken;
    address admin   = address(0xA);
    address timelock = address(0xB);
    address user    = address(0xC);

    function setUp() public {
        altToken = new MockAltRsETH();
        wrapper  = new RsETHTokenWrapper();
        wrapper.initialize(admin, address(0), address(altToken));

        vm.prank(admin);
        wrapper.grantRole(wrapper.TIMELOCK_ROLE(), timelock);

        // Fund user and deposit 100 tokens
        altToken.transfer(user, 100e18);
        vm.startPrank(user);
        altToken.approve(address(wrapper), 100e18);
        wrapper.deposit(address(altToken), 100e18);
        vm.stopPrank();
    }

    function test_permanentFreeze() public {
        // Precondition: wrapper holds 100 altToken
        assertEq(altToken.balanceOf(address(wrapper)), 100e18);

        // TIMELOCK_ROLE removes the token (no balance check)
        vm.prank(timelock);
        wrapper.removeAllowedToken(address(altToken));

        // User attempts to withdraw — reverts with TokenNotAllowed
        vm.prank(user);
        vm.expectRevert(RsETHTokenWrapper.TokenNotAllowed.selector);
        wrapper.withdraw(address(altToken), 100e18);

        // Tokens are permanently frozen; no recovery path exists
        assertEq(altToken.balanceOf(address(wrapper)), 100e18);
        assertEq(wrapper.balanceOf(user), 100e18); // wrsETH still held, worthless
    }
}
