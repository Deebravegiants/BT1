### Title
O(assets × NDCs × queued-withdrawals) Unbounded Gas in `updateRSETHPrice` Call Chain - (File: contracts/LRTOracle.sol / contracts/LRTDepositPool.sol)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a public function with no access control beyond `whenNotPaused`. Its internal call chain traverses a multiplicative nested loop: for every supported asset, it iterates every NDC, and for every NDC it calls `getAssetUnstaking()` which itself iterates all queued EigenLayer withdrawals for that NDC. There are no hard caps on any of these three dimensions, and as the protocol grows legitimately, the function can permanently exceed the block gas limit, freezing the rsETH price.

---

### Finding Description

The full call chain is:

```
updateRSETHPrice()                          [public, whenNotPaused only]
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ for each asset in supportedAssetList:          [A iterations]
                 └─ getTotalAssetDeposits(asset)
                      └─ getAssetDistributionData(asset)
                           └─ for each NDC in nodeDelegatorQueue:  [N iterations]
                                ├─ IERC20(asset).balanceOf(ndc)
                                ├─ INodeDelegator(ndc).getAssetBalance(asset)
                                │    └─ EigenLayer getWithdrawableShares + sharesToUnderlyingView
                                └─ INodeDelegator(ndc).getAssetUnstaking(asset)
                                     └─ DelegationManager.getQueuedWithdrawals(ndc)
                                          └─ for each queued withdrawal:  [W iterations]
                                               └─ for each strategy in withdrawal
```

This is O(A × N × W) external calls with no hard ceiling on any dimension.

**Dimension 1 — Assets (A):** `supportedAssetList` in `LRTConfig` grows via `addNewSupportedAsset()` (requires `TIME_LOCK_ROLE`). No maximum is enforced. [1](#0-0) 

**Dimension 2 — NDCs (N):** `maxNodeDelegatorLimit` starts at 10 but `updateMaxNodeDelegatorLimit()` has no upper bound check — an admin can raise it to any value. [2](#0-1) 

**Dimension 3 — Queued withdrawals (W):** `getAssetUnstaking()` calls `DelegationManager.getQueuedWithdrawals(address(this))` and iterates all returned withdrawals and their strategies. The `maxUncompletedWithdrawalCount` cap is 80 per NDC. [3](#0-2) 

The inner loop in `getAssetDistributionData` fires three external calls per (asset, NDC) pair: [4](#0-3) 

The outer loop in `_getTotalEthInProtocol` calls `getTotalAssetDeposits` (which calls `getAssetDistributionData`) once per asset: [5](#0-4) 

The protocol itself acknowledges the multiplicative relationship in a comment: [6](#0-5)

### Impact Explanation

If `updateRSETHPrice()` permanently exceeds the block gas limit:

- `rsETHPrice` is never updated; it becomes permanently stale.
- Protocol fees (minted as rsETH in `_updateRsETHPrice`) can never be collected.
- Users depositing receive rsETH amounts calculated from a frozen, incorrect price, violating the core accounting invariant.
- The `updateRSETHPriceAsManager()` path (`onlyLRTManager`) shares the same `_updateRsETHPrice()` internal function and would also OOG.

**Impact: Medium — Unbounded gas consumption** leading to permanent price staleness and broken fee accrual.

---

### Likelihood Explanation

Both the asset list and NDC queue are admin-controlled and grow through legitimate protocol operation (not attacker action). The current deployment is small (~3 assets, ~5 NDCs per the protocol's own comment). However:

- `maxNodeDelegatorLimit` has no hard ceiling and can be raised arbitrarily by admin.
- `supportedAssetList` has no cap.
- `getAssetUnstaking` adds a third multiplicative factor (up to 80 queued withdrawals per NDC).

A rough gas estimate: each `getAssetUnstaking` call with 80 queued withdrawals costs ~30,000–50,000 gas. At 20 NDCs × 10 assets = 200 pairs × 50,000 gas = **10M gas** just for `getAssetUnstaking` calls, plus overhead from `getAssetBalance` (EigenLayer strategy reads) and `balanceOf` calls. At 30 NDCs × 15 assets the function approaches or exceeds 30M gas.

**Likelihood: Low** at current scale, but increases as the protocol adds assets and NDCs over time with no architectural guard preventing it.

---

### Recommendation

1. **Hard-cap `maxNodeDelegatorLimit`** at a value (e.g., 15) that keeps `A × N` within safe gas bounds, enforced in `updateMaxNodeDelegatorLimit`.
2. **Hard-cap `supportedAssetList`** length in `_addNewSupportedAsset`.
3. **Cache `getQueuedWithdrawals` results** or restructure `getAssetUnstaking` to avoid re-fetching all queued withdrawals for every (asset, NDC) pair — consider a push-based accounting model instead.
4. **Add a gas guard** in `_getTotalEthInProtocol` (e.g., `require(gasleft() > MIN_GAS_RESERVE)`) to fail gracefully rather than silently OOG.

```solidity
// Foundry fork test (local fork, no mainnet)
// 1. Deploy LRTConfig, LRTDepositPool, LRTOracle, N NodeDelegator contracts
// 2. Add A supported assets to LRTConfig
// 3. Add N NDCs to nodeDelegatorQueue (raise maxNodeDelegatorLimit first)
// 4. Queue W withdrawals in EigenLayer for each NDC
// 5. Measure gas for updateRSETHPrice()

function testGasScaling() public {
    uint256 A = 15; // supported assets
    uint256 N = 20; // NDCs
    // setup: add A assets, N NDCs, W queued withdrawals per NDC
    // ...
    uint256 gasBefore = gasleft();
    lrtOracle.updateRSETHPrice();
    uint256 gasUsed = gasBefore - gasleft();
    // At A=15, N=20, W=80: gasUsed approaches or exceeds 30_000_000
    assertLt(gasUsed, 30_000_000, "OOG: price update permanently broken");
}
```

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
