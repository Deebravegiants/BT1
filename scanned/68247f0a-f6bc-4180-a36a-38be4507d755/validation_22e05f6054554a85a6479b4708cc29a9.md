### Title
Unrestricted Public `updateRSETHPrice()` With No Minimum Interval Enables Deposit-Before-Update Sandwich to Steal Yield From Existing rsETH Holders - (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle.updateRSETHPrice()` is a `public` function with no access control and no minimum cooldown between calls. The stored `rsETHPrice` is the sole input used by `LRTDepositPool.getRsETHAmountToMint()` to determine how many rsETH tokens to mint per deposit. When staking rewards accrue in EigenLayer and the stored price has not yet been updated, an attacker can atomically deposit at the stale (below-true-value) price and then call `updateRSETHPrice()` in the same transaction, minting excess rsETH and diluting existing holders' yield.

### Finding Description
`LRTOracle.updateRSETHPrice()` is declared `public whenNotPaused` with no timestamp guard, no cooldown variable, and no role restriction:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

The stored `rsETHPrice` state variable is only updated when this function (or `updateRSETHPriceAsManager()`) is called. Between calls, the price is stale. `LRTDepositPool.getRsETHAmountToMint()` reads this stale stored value directly:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

Because `rsETHPrice` is in the denominator, a stale (lower-than-true) price causes `rsethAmountToMint` to be inflated — the depositor receives more rsETH than the true exchange rate warrants.

The absence of any minimum interval between `updateRSETHPrice()` calls means an attacker can, within a single atomic transaction via a smart contract:
1. Call `LRTDepositPool.depositETH()` or `depositAsset()` while `rsETHPrice` is stale → receive excess rsETH
2. Call `LRTOracle.updateRSETHPrice()` → price is corrected upward, fee is minted to treasury

The attacker's excess rsETH represents yield that should have accrued to existing holders but is instead captured through dilution.

### Impact Explanation
Existing rsETH holders suffer dilution of their yield. When the attacker mints rsETH at a stale (below-true) price, the total rsETH supply increases by more than the proportional TVL contribution. After the price update, the attacker holds rsETH worth more than what they deposited, at the expense of all other holders whose share of the TVL is reduced. This constitutes theft of unclaimed yield (High severity).

### Likelihood Explanation
EigenLayer staking rewards accrue continuously, meaning `rsETHPrice` is always at least slightly stale between updates. The attack requires no special permissions — only the ability to deploy a smart contract that calls `depositETH()` followed by `updateRSETHPrice()` atomically. The profit scales with the size of the deposit and the magnitude of the stale price gap. The attack is repeatable every time rewards accrue.

### Recommendation
Introduce a minimum interval between permissionless `updateRSETHPrice()` calls, for example:

```solidity
uint256 public lastPriceUpdateTimestamp;
uint256 public constant MIN_UPDATE_INTERVAL = 1 hours;

function updateRSETHPrice() public whenNotPaused {
    require(block.timestamp >= lastPriceUpdateTimestamp + MIN_UPDATE_INTERVAL, "Too soon");
    lastPriceUpdateTimestamp = block.timestamp;
    _updateRsETHPrice();
}
```

Privileged `updateRSETHPriceAsManager()` can remain unrestricted for emergency use. Additionally, consider computing the rsETH mint amount using a freshly computed price rather than the stored stale value, or require that `updateRSETHPrice()` is called before any deposit in the same block.

### Proof of Concept
```solidity
contract SandwichAttack {
    ILRTDepositPool depositPool;
    ILRTOracle oracle;

    function attack() external payable {
        // Step 1: Deposit at stale (below-true) rsETHPrice
        // rsETHPrice is stale because rewards have accrued since last update
        // rsethAmountToMint = (msg.value * assetPrice) / rsETHPrice (stale, lower)
        // → attacker receives MORE rsETH than deserved
        depositPool.depositETH{value: msg.value}(0, "");

        // Step 2: Update price in the same transaction
        // rsETHPrice is now corrected upward
        // Attacker holds rsETH minted at the old lower price → worth more than deposited
        oracle.updateRSETHPrice();

        // Attacker's rsETH balance now represents a claim on more ETH than deposited,
        // diluting all existing rsETH holders' yield
    }
}
```
