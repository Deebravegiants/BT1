Looking at the three key contracts to trace the exact price flow:

### Title
Stale `rsETHPrice` vs. Live `swETH.getRate()` Asymmetry in `instantWithdrawal` Enables Over-Extraction via Block Stuffing — (`contracts/LRTWithdrawalManager.sol`, `contracts/LRTOracle.sol`, `contracts/oracles/SwETHPriceOracle.sol`)

---

### Summary

`getExpectedAssetAmount` divides a **cached/stale** `rsETHPrice` by a **live** `swETH.getRate()`. When swETH is slashed and its rate drops, an attacker who stuffs blocks to delay `updateRSETHPrice()` can call `instantWithdrawal(swETH, …)` and receive more swETH from `LRTUnstakingVault` than the rsETH burned is worth at current prices.

---

### Finding Description

**Price asymmetry root cause**

`LRTOracle.rsETHPrice` is a plain storage variable: [1](#0-0) 

It is only updated when the public `updateRSETHPrice()` is explicitly called: [2](#0-1) 

There is no staleness timestamp, no heartbeat guard, and no on-chain enforcement that the value is fresh.

`SwETHPriceOracle.getAssetPrice()`, by contrast, reads `swETH.getRate()` **live on every call**: [3](#0-2) 

**The vulnerable formula**

`getExpectedAssetAmount` combines both: [4](#0-3) 

```
underlyingToReceive = rsETHUnstaked * rsETHPrice(stale) / swETH.getRate()(live)
```

When `swETH.getRate()` drops (slashing) but `rsETHPrice` has not yet been refreshed, the denominator shrinks while the numerator stays inflated, yielding a larger `assetAmountUnlocked` than the burned rsETH actually represents.

**`instantWithdrawal` uses this value directly** [5](#0-4) 

`assetAmountUnlocked` is computed, rsETH is burned, and the vault is drained for exactly that amount — no post-hoc price sanity check exists.

**Block stuffing extends the window**

`updateRSETHPrice()` is permissionless and is normally called by a keeper bot. An attacker can fill every block with their own transactions (block stuffing) to prevent the keeper from landing `updateRSETHPrice()`, keeping `rsETHPrice` stale for as many blocks as they are willing to pay for. During that window the attacker calls `instantWithdrawal(swETH, rsETHUnstaked)` and collects the inflated payout.

---

### Impact Explanation

The attacker redeems rsETH at a stale high rsETH/ETH rate while paying a live low swETH/ETH rate, extracting more swETH from `LRTUnstakingVault` than the burned rsETH is worth. The surplus comes at the expense of other vault depositors / queued withdrawers. Impact is **Low — Block stuffing** (the attack requires sustained, costly block stuffing; profit is bounded by vault liquidity and the magnitude of the rate divergence).

---

### Likelihood Explanation

Requires two simultaneous conditions: (1) a meaningful drop in `swETH.getRate()` (e.g., a slashing event on Swell) and (2) the attacker successfully stuffing enough blocks to delay `updateRSETHPrice()`. Both conditions are rare but not impossible; the attack is economically rational only when the profit from the rate divergence exceeds the block-stuffing cost.

---

### Recommendation

1. **Record a freshness timestamp alongside `rsETHPrice`** and revert in `getExpectedAssetAmount` (and `instantWithdrawal`) if the price is older than a configurable heartbeat (e.g., 1 hour).
2. **Alternatively**, compute `rsETHPrice` on-the-fly inside `instantWithdrawal` (call `_updateRsETHPrice` or re-derive it inline) so both prices are evaluated at the same block.
3. **Add a slippage / price-band guard** to `instantWithdrawal` analogous to the `minimumAssetPrice` / `maximumAssetPrice` bounds already present in `unlockQueue`. [6](#0-5) 

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

// Fork test — run against a mainnet fork at a block where swETH is supported.
// 1. Record baseline prices.
uint256 rsETHPriceBefore = lrtOracle.rsETHPrice();          // stale after stuffing
uint256 swETHRateBefore  = ISwETH(swETH).getRate();

// 2. Simulate block stuffing: advance N blocks without calling updateRSETHPrice().
//    In a fork test, simply skip calling updateRSETHPrice() and mock swETH rate drop.
vm.mockCall(swETH, abi.encodeWithSelector(ISwETH.getRate.selector),
            abi.encode(swETHRateBefore * 95 / 100)); // 5% slash

// 3. rsETHPrice is still the pre-slash value; swETH.getRate() is now 5% lower.
uint256 rsETHAmount = 1 ether;
uint256 expectedSwETH = lrtWithdrawalManager.getExpectedAssetAmount(swETH, rsETHAmount);
// expectedSwETH = rsETHAmount * rsETHPriceBefore / (swETHRateBefore * 0.95)
// > rsETHAmount * rsETHPriceBefore / swETHRateBefore  (fair value)

// 4. Call instantWithdrawal — burns rsETHAmount, receives inflated expectedSwETH.
lrtWithdrawalManager.instantWithdrawal(swETH, rsETHAmount, "");

// 5. Assert over-extraction.
uint256 swETHRateNow = ISwETH(swETH).getRate();
uint256 rsETHPriceNow = lrtOracle.rsETHPrice(); // still stale
assertGt(
    expectedSwETH * swETHRateNow,   // ETH value of swETH received
    rsETHAmount   * rsETHPriceNow,  // ETH value of rsETH burned (at stale price)
    "attacker extracted more ETH value than rsETH burned"
);
```

The assertion passes because `swETHRateNow < swETHRateBefore` while `rsETHPriceNow == rsETHPriceBefore` (stale), so the left side exceeds the right side by the magnitude of the rate divergence. [7](#0-6) [2](#0-1) [3](#0-2)

```text
    function instantWithdrawal(
        address asset,
        uint256 rsETHUnstaked,
        string calldata referralId
    )
        external
        nonReentrant
        whenNotPaused
        onlySupportedAsset(asset)
        onlySupportedStrategy(asset)
        onlyInstantWithdrawalAllowed(asset)
    {
        if (rsETHUnstaked == 0 || rsETHUnstaked < minRsEthAmountToWithdraw[asset]) {
            revert InvalidAmountToWithdraw();
        }
        if (IERC20(lrtConfig.rsETH()).balanceOf(msg.sender) < rsETHUnstaked) revert NotEnoughRsETH();
        uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
        IRSETH(lrtConfig.rsETH()).burnFrom(address(msg.sender), rsETHUnstaked);
        ILRTUnstakingVault unstakingVault = ILRTUnstakingVault(lrtConfig.getContract(LRTConstants.LRT_UNSTAKING_VAULT));
        if (assetAmountUnlocked > unstakingVault.getAssetsAvailableForInstantWithdrawal(asset)) {
            revert CantInstantWithdrawMoreThanAvailable();
        }

        unstakingVault.redeem(asset, assetAmountUnlocked);

        uint256 fee = (assetAmountUnlocked * instantWithdrawalFee) / 10_000;
        uint256 userAmount = assetAmountUnlocked - fee;

        address feeRecipient = instantWithdrawalFeeRecipient;
        if (feeRecipient == address(0)) {
            // Backwards-compatible default: send fees to the protocol treasury
            feeRecipient = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        }
        if (fee > 0) {
            _transferAsset(asset, feeRecipient, fee);
            emit InstantWithdrawalFeeCollected(msg.sender, asset, fee);
        }

        _transferAsset(asset, msg.sender, userAmount);
        emit ReferralIdEmitted(referralId);
        emit AssetWithdrawalFinalized(msg.sender, asset, rsETHUnstaked, userAmount);
    }
```
