### Title
Fee-on-Transfer Token Insolvency in `stake()` / `stakeFor()` Due to Missing Balance Check - (File: contracts/KERNEL/KernelDepositPool.sol)

### Summary
`KernelDepositPool.stake()` and `stakeFor()` credit `balanceOf` and `totalKernelStaked` with the caller-supplied `_amount` without verifying the actual tokens received. If `kernelToken` carries a transfer fee, the contract records more staked balance than it holds, making it impossible for all stakers to reclaim their tokens.

### Finding Description
In `stake()` and `stakeFor()`, the internal accounting is updated with `_amount` before (and without verifying) the actual transfer result:

```solidity
// stake()
balanceOf[msg.sender] += _amount;       // line 284
totalKernelStaked += _amount;           // line 285
kernelToken.safeTransferFrom(msg.sender, address(this), _amount);  // line 286
```

The same pattern appears in `stakeFor()` (lines 309–311). No `balanceOf` snapshot is taken before/after the transfer to compute the real received amount.

By contrast, `notifyRewardAmount()` — in the same contract — explicitly guards against fee-on-transfer tokens for `rewardsToken`:

```solidity
uint256 balanceBefore = rewardsToken.balanceOf(address(this));   // line 573
rewardsToken.safeTransferFrom(msg.sender, address(this), _amount); // line 574
uint256 balanceAfter  = rewardsToken.balanceOf(address(this));   // line 575
uint256 receivedAmount = balanceAfter - balanceBefore;            // line 577
```

This inconsistency confirms the developers understood the risk for reward tokens but did not apply the same protection to the staking token.

### Impact Explanation
If `kernelToken` is a fee-on-transfer token, every `stake()` call inflates `balanceOf[user]` and `totalKernelStaked` beyond the tokens actually held. When users later call `initiateWithdrawal()` + `claimWithdrawal()`, the contract attempts `kernelToken.safeTransfer(msg.sender, withdrawal.amount)` for the full credited amount. The last stakers to withdraw will find the contract insolvent — it cannot cover their recorded balances. This is protocol insolvency for the KERNEL staking pool.

### Likelihood Explanation
The `kernelToken` is set once at `initialize()` and cannot be changed. If the KERNEL token deployed at that address implements a transfer fee (now or via an upgrade), every stake call silently over-credits users. The entry path requires no privilege — any user calling `stake()` with a non-zero amount triggers the accounting divergence.

### Recommendation
Apply the same balance-snapshot pattern used in `notifyRewardAmount()` to `stake()` and `stakeFor()`:

```solidity
uint256 balanceBefore = kernelToken.balanceOf(address(this));
kernelToken.safeTransferFrom(msg.sender, address(this), _amount);
uint256 received = kernelToken.balanceOf(address(this)) - balanceBefore;

balanceOf[msg.sender] += received;
totalKernelStaked     += received;
```

### Proof of Concept
1. Deploy `KernelDepositPool` with a fee-on-transfer `kernelToken` (e.g., 1 % fee).
2. Alice calls `stake(1000)`.
   - `balanceOf[Alice]` → 1000, `totalKernelStaked` → 1000.
   - Contract actually receives 990 tokens.
3. Bob calls `stake(1000)`.
   - `balanceOf[Bob]` → 1000, `totalKernelStaked` → 2000.
   - Contract actually receives 990 more tokens → total held = 1980.
4. Alice calls `initiateWithdrawal(1000)` then `claimWithdrawal()` → receives 1000 tokens. Contract holds 980.
5. Bob calls `initiateWithdrawal(1000)` then `claimWithdrawal()` → `safeTransfer` reverts; Bob cannot recover his stake. Contract is insolvent. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
