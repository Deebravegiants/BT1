### Title
System-Wide Pause Permanently Blocks Users from Completing Pending Withdrawals - (File: contracts/LRTWithdrawalManager.sol)

### Summary
When `LRTWithdrawalManager` is paused (via `PAUSER_ROLE` or `LRTConfig.pauseAll()`), users who have already transferred rsETH into the contract via `initiateWithdrawal` cannot call `completeWithdrawal` to recover their funds. There is no cancel/refund path, so rsETH is frozen in the contract for the entire duration of the pause.

### Finding Description
`initiateWithdrawal` transfers the user's rsETH from their wallet into `LRTWithdrawalManager` and records a pending request:

```solidity
IERC20(lrtConfig.rsETH()).safeTransferFrom(msg.sender, address(this), rsETHUnstaked);
``` [1](#0-0) 

Both `completeWithdrawal` and `instantWithdrawal` carry the `whenNotPaused` modifier:

```solidity
function completeWithdrawal(address asset, string calldata referralId) external nonReentrant whenNotPaused {
``` [2](#0-1) 

```solidity
function instantWithdrawal(...) external nonReentrant whenNotPaused ...
``` [3](#0-2) 

`LRTConfig.pauseAll()` pauses `LRTWithdrawalManager` atomically alongside all other protocol contracts:

```solidity
if (!lrtWithdrawalManager.paused()) lrtWithdrawalManager.pause();
``` [4](#0-3) 

The contract exposes no `cancelWithdrawal` or refund path. Once rsETH is transferred in and the contract is paused, the user has no on-chain mechanism to recover their tokens until the admin calls `unpause`.

### Impact Explanation
Users' rsETH is held by `LRTWithdrawalManager` for the entire pause duration with no recovery path. This constitutes a **temporary freezing of funds** (Medium severity per the allowed impact scope). If the pause is prolonged or the admin key is unavailable, the freeze extends indefinitely.

### Likelihood Explanation
The `PAUSER_ROLE` is a low-threshold role explicitly designed for rapid emergency response. `pauseAll()` in `LRTConfig` is a single-transaction operation that pauses all contracts simultaneously. Any legitimate emergency pause — e.g., an oracle manipulation event or a deposit-pool exploit — will simultaneously freeze all pending withdrawal completions for every user who has already committed rsETH. This is a routine operational scenario, not an edge case. [5](#0-4) 

### Recommendation
Add a `cancelWithdrawal` function (callable even when paused) that returns the user's rsETH and removes their pending request, or modify `completeWithdrawal` to be callable when paused so that users with already-committed rsETH can always recover their funds. This mirrors the original report's recommendation: privileged recovery paths should bypass the global pause.

### Proof of Concept
1. Alice calls `initiateWithdrawal(stETH, 1e18, "")`. Her 1 rsETH is transferred to `LRTWithdrawalManager`. A withdrawal request is recorded.
2. The `PAUSER_ROLE` calls `LRTConfig.pauseAll()`, which pauses `LRTWithdrawalManager`.
3. Alice calls `completeWithdrawal(stETH, "")`. The call reverts with `"Pausable: paused"` due to `whenNotPaused`.
4. Alice has no `cancelWithdrawal` function to call. Her 1 rsETH remains locked in the contract.
5. Alice cannot recover her rsETH until the admin calls `unpause()` on `LRTWithdrawalManager`. [6](#0-5) [7](#0-6)

    function pauseAll() external onlyRole(LRTConstants.PAUSER_ROLE) {
        IPausable lrtDepositPool = IPausable(getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable lrtWithdrawalManager = IPausable(getContract(LRTConstants.LRT_WITHDRAW_MANAGER));
        IPausable lrtOracle = IPausable(getContract(LRTConstants.LRT_ORACLE));
        IPausable rsETHContract = IPausable(rsETH);

        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!lrtWithdrawalManager.paused()) lrtWithdrawalManager.pause();
        if (!lrtOracle.paused()) lrtOracle.pause();
        if (!rsETHContract.paused()) rsETHContract.pause();

        address[] memory nodeDelegatorQueue = ILRTDepositPool(address(lrtDepositPool)).getNodeDelegatorQueue();
        uint256 nodeDelegatorCount = nodeDelegatorQueue.length;

        for (uint256 i = 0; i < nodeDelegatorCount;) {
            IPausable nodeDelegator = IPausable(nodeDelegatorQueue[i]);
            if (!nodeDelegator.paused()) nodeDelegator.pause();
            unchecked {
                ++i;
            }
        }

        emit PausedAll(msg.sender);
