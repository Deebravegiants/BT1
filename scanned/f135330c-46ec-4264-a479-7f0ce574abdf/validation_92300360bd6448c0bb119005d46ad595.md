### Title
Unverified `account` Parameter in `MerkleDistributor.claim()` Allows Anyone to Force-Claim Any User's Yield, Extracting Protocol Fee - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

---

### Summary

`MerkleDistributor.claim()` accepts a caller-supplied `account` address without verifying that `msg.sender == account`. Because Merkle proofs are public data, any attacker can call `claim()` on behalf of any victim at any time, forcing the protocol fee to be deducted from the victim's yield at an attacker-chosen moment. The victim loses the ability to time their own claim and is forced to pay the fee even if they were waiting for a fee reduction or preferred to accumulate more before claiming.

---

### Finding Description

`MerkleDistributor.claim()` takes `account` as a caller-supplied parameter and verifies only that the Merkle proof is valid for that `(index, account, cumulativeAmount)` tuple. It never checks that `msg.sender == account`.

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
    // ... no msg.sender == account check ...

    bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
    if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
        revert InvalidMerkleProof();
    }

    uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;
    uint256 fee = (claimableAmount * feeInBPS) / 10_000;
    uint256 amountToSend = claimableAmount - fee;

    IERC20(token).safeTransfer(account, amountToSend);
    IERC20(token).safeTransfer(protocolTreasury, fee);
}
``` [1](#0-0) 

The fee is configurable up to `MAX_FEE_IN_BPS = 1000` (10%). [2](#0-1) 

By contrast, `KernelMerkleDistributor._processClaim()` correctly enforces `if (account != msg.sender) { revert Unauthorized(); }`, demonstrating the protocol's own awareness that this check is required. [3](#0-2) 

`MerkleDistributor.sol` has no equivalent guard.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

An attacker who observes the public Merkle tree can call `claim(index, victimAddress, cumulativeAmount, merkleProof)` for any victim at any time. The protocol fee (up to 10%) is immediately deducted from the victim's claimable yield and sent to `protocolTreasury`. The victim:

1. Loses up to 10% of their yield as a fee, forced at the attacker's chosen moment.
2. Loses the ability to time their own claim — e.g., if the admin was about to call `setFeeInBPS(0)` to reduce the fee, the attacker can front-run that and force the victim to pay the current high fee, permanently extracting yield the victim would otherwise have kept.
3. Has their claim state permanently updated (`userClaims[account].lastClaimedIndex` and `cumulativeAmount`), preventing any re-claim of the same index. [4](#0-3) 

---

### Likelihood Explanation

**High.** Merkle proofs are derived from public off-chain data (the distribution tree). Any attacker can reconstruct valid `(index, account, cumulativeAmount, merkleProof)` tuples for any eligible address. No privileged access, leaked keys, or oracle manipulation is required. The call is permissionless and costs only gas.

---

### Recommendation

Add a caller identity check identical to the one already present in `KernelMerkleDistributor`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Insert this check at the top of `claim()`, before any state changes or token transfers. [5](#0-4) 

---

### Proof of Concept

1. Admin sets `feeInBPS = 1000` (10%) and publishes a Merkle root. Alice is entitled to 1000 tokens at `index = 1`.
2. Admin announces they will call `setFeeInBPS(0)` in the next block to remove the fee.
3. Attacker reads the public Merkle tree, reconstructs Alice's proof, and calls:
   ```solidity
   merkleDistributor.claim(1, alice, 1000e18, aliceProof);
   ```
4. The contract transfers 900 tokens to Alice and 100 tokens to `protocolTreasury` as fee.
5. Alice's `userClaims[alice].lastClaimedIndex` is set to 1 — she cannot re-claim index 1.
6. When the admin reduces the fee to 0%, Alice has already been forced to pay 100 tokens she would have kept.

Alice loses 100 tokens (10% of her yield) that she would have retained had she been allowed to claim after the fee reduction. The attacker spent only gas. [6](#0-5)

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
