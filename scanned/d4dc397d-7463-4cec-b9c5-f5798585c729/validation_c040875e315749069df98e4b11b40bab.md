### Title
`ChainlinkPriceOracle.getAssetPrice` Ignores Chainlink `latestRoundData` Validity Indicators, Enabling Stale-Price rsETH Minting - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice` calls Chainlink's `latestRoundData()` but discards every validity field (`roundId`, `startedAt`, `updatedAt`, `answeredInRound`). A stale or incomplete round price is silently accepted and propagated into rsETH mint calculations, while the sibling oracle `ChainlinkOracleForRSETHPoolCollateral` in the same repository performs all three required checks.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice` reads only the raw `answer` from Chainlink:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol  line 52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

No check is made for:
- `answeredInRound < roundId` (stale round)
- `updatedAt == 0` (incomplete round)
- `price <= 0` (invalid/negative answer)

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate`, used for L2 pool collateral pricing in the same codebase, performs all three guards:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol  lines 30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

The stale price returned by `ChainlinkPriceOracle.getAssetPrice` flows through the following call chain:

1. `LRTOracle.getAssetPrice` delegates to `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)` [3](#0-2) 
2. `LRTOracle._getTotalEthInProtocol` accumulates per-asset ETH value using that price [4](#0-3) 
3. `LRTOracle._updateRsETHPrice` derives the new rsETH/ETH price from the total [5](#0-4) 
4. `LRTDepositPool.getRsETHAmountToMint` uses `lrtOracle.getAssetPrice(asset)` and `lrtOracle.rsETHPrice()` to compute how many rsETH tokens to mint per deposit [6](#0-5) 
5. Both `depositETH` and `depositAsset` call `_beforeDeposit → getRsETHAmountToMint` and then mint rsETH [7](#0-6) 

---

### Impact Explanation

When a Chainlink feed goes stale (e.g., during L1 network congestion, a sequencer outage, or a feed deprecation), `ChainlinkPriceOracle.getAssetPrice` silently returns the last recorded price. If that stale price is **inflated** relative to the true market price, every depositor calling `depositAsset` or `depositETH` receives more rsETH than the deposited assets are worth. This directly dilutes existing rsETH holders and, if exploited at scale, leads to **protocol insolvency** — the total rsETH supply becomes unbacked. If the stale price is **deflated**, depositors are under-minted rsETH, constituting a loss of promised returns.

Impact classification: **Critical — protocol insolvency / share mis-accounting**.

---

### Likelihood Explanation

Chainlink feeds do go stale: heartbeat windows are typically 1–24 hours, and during periods of low volatility or network stress the `updatedAt` timestamp can lag significantly. The `answeredInRound < roundId` condition triggers whenever a round fails to complete. These are well-documented, real-world conditions. No privileged access is required; any user calling `depositAsset` or `depositETH` during a stale-price window triggers the issue.

---

### Recommendation

Apply the same three guards already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Optionally add a configurable `stalePriceThreshold` (e.g., `block.timestamp - updatedAt > threshold`) to enforce a maximum age on the price.

---

### Proof of Concept

1. A Chainlink LST/ETH feed used by `ChainlinkPriceOracle` goes stale (last update was 25 hours ago). The feed's `answeredInRound` is less than the current `roundId`, or `updatedAt` is far in the past.
2. An attacker (or any user) calls `LRTDepositPool.depositAsset(lstToken, amount, 0, "")`.
3. `_beforeDeposit` calls `getRsETHAmountToMint`, which calls `lrtOracle.getAssetPrice(lstToken)`.
4. `LRTOracle.getAssetPrice` calls `ChainlinkPriceOracle.getAssetPrice`, which calls `priceFeed.latestRoundData()` and returns the stale, inflated price without reverting.
5. The inflated asset price causes `getRsETHAmountToMint` to return a larger rsETH amount than the deposit warrants.
6. `_mintRsETH` mints the excess rsETH to the depositor.
7. The depositor redeems the excess rsETH for more underlying assets than they deposited, draining the protocol.
