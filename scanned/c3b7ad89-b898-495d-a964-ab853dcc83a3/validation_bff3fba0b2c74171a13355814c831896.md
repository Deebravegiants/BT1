### Title
Stale rsETH/ETH Rate Masked by Fresh ETH/USD `updatedAt` in `RSETHPriceFeed.latestRoundData()` ŌĆö (File: contracts/oracles/RSETHPriceFeed.sol)

### Summary
`RSETHPriceFeed.latestRoundData()` computes a composite rsETH/USD answer by multiplying a cached `rsETHPrice` (from `LRTOracle`) by a live ETH/USD Chainlink price, but returns only the ETH/USD feed's `updatedAt` timestamp. Because `rsETHPrice` is a stored state variable updated only when `updateRSETHPrice()` is called, the returned `updatedAt` does not reflect the freshness of the rsETH/ETH component. Any consumer applying a standard Chainlink staleness check on `updatedAt` will be deceived into treating a stale composite answer as fresh.

### Finding Description

`RSETHPriceFeed.latestRoundData()` is:

```solidity
function latestRoundData()
    external view
    returns (uint80 roundId, int256 answer, uint256 startedAt, uint256 updatedAt, uint80 answeredInRound)
{
    (roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData(); // ŌåÉ all metadata from ETH/USD
    answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;                             // ŌåÉ rsETH rate injected with no timestamp
}
``` [1](#0-0) 

`RS_ETH_ORACLE.rsETHPrice()` resolves to `LRTOracle.rsETHPrice`, a public state variable: [2](#0-1) 

This value is only updated when `updateRSETHPrice()` (a public, permissionless function) is successfully executed: [3](#0-2) 

The `updatedAt` returned by `latestRoundData()` is sourced exclusively from `ETH_TO_USD.latestRoundData()` ŌĆö it reflects when Chainlink last updated the ETH/USD price, not when `rsETHPrice` was last computed. There is no timestamp stored alongside `rsETHPrice` in `LRTOracle`, and `RSETHPriceFeed` performs no staleness check on the rsETH component.

**Block stuffing attack path**: An attacker on a low-basefee chain (e.g., an L2 where `RSETHPriceFeed` is deployed) can fill consecutive blocks with high-gas transactions to prevent `updateRSETHPrice()` from being included. During this window, `rsETHPrice` remains frozen at its last value while the ETH/USD Chainlink feed continues to update normally. Every call to `latestRoundData()` returns a `updatedAt` equal to the ETH/USD feed's latest update ŌĆö which may be seconds old ŌĆö while the rsETH/ETH component is arbitrarily stale. A consumer checking `block.timestamp - updatedAt < threshold` will pass the staleness check and consume the mispriced composite answer.

### Impact Explanation

The composite rsETH/USD answer can be arbitrarily stale in its rsETH/ETH component while appearing fresh to any standard Chainlink consumer pattern. This violates the invariant that `updatedAt` reflects the age of the returned `answer`. Downstream protocols (e.g., lending markets) using `RSETHPriceFeed` as a price source and relying on `updatedAt` for staleness detection cannot distinguish a fresh answer from a stale one. The impact is classified as **Low ŌĆö Block stuffing** per the allowed scope: the attacker uses block stuffing to freeze `rsETHPrice` while the feed continues to report a fresh `updatedAt`.

### Likelihood Explanation

Block stuffing is expensive but feasible on chains with low block gas costs. The `updateRSETHPrice()` function is public and permissionless, so the only way to prevent it from executing is to exclude it from blocks. The staleness issue itself (mismatched `updatedAt`) exists unconditionally ŌĆö even without block stuffing, a natural delay between `updateRSETHPrice()` calls creates a window where the composite answer is stale but `updatedAt` appears fresh.

### Recommendation

1. Store a `rsETHPriceUpdatedAt` timestamp in `LRTOracle` alongside `rsETHPrice` and update it every time `_updateRsETHPrice()` runs.
2. In `RSETHPriceFeed.latestRoundData()`, return `min(ethToUSD_updatedAt, rsETHPriceUpdatedAt)` as `updatedAt` so that consumers see the true age of the composite answer.
3. Optionally, add an explicit staleness revert in `latestRoundData()` if `rsETHPriceUpdatedAt` is older than a configurable threshold.

### Proof of Concept

Invariant fuzz test (Foundry):

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";

contract RSETHPriceFeedStalenessTest is Test {
    MockLRTOracle oracle;
    MockETHUSDFeed ethFeed;
    RSETHPriceFeed feed;

    function setUp() public {
        oracle  = new MockLRTOracle(1.05e18);   // rsETH/ETH = 1.05
        ethFeed = new MockETHUSDFeed(3000e8, block.timestamp);
        feed    = new RSETHPriceFeed(address(ethFeed), address(oracle), "rsETH/USD");
    }

    /// @dev Advance time by G seconds without calling updateRSETHPrice().
    ///      updatedAt must reflect rsETH staleness, but it doesn't.
    function invariant_updatedAtReflectsRsETHFreshness(uint256 G) public {
        G = bound(G, 1 hours, 30 days);
        skip(G);
        // ETH/USD feed updates to a new timestamp
        ethFeed.setUpdatedAt(block.timestamp);

        (, , , uint256 updatedAt, ) = feed.latestRoundData();

        // updatedAt equals the ETH/USD feed's fresh timestamp,
        // even though rsETHPrice has not been updated for G seconds.
        // This assertion FAILS ŌĆö proving the invariant is broken.
        assertLe(block.timestamp - updatedAt, 1 hours,
            "updatedAt appears fresh despite stale rsETH component");
    }
}
```
