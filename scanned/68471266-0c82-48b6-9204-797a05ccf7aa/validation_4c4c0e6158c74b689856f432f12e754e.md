### Title
Linea Bridge Fee Deduction Causes L1 Vault to Receive Less ETH Than Pool Accounting Claims — (`contracts/bridges/LineaMessenger.sol`)

---

### Summary

`LineaMessenger.sendETHToL1ViaBridge` passes the full `value` as `msg.value` to `ILineaMessageService.sendMessage` but only `minimumFee` as the `_fee` parameter. The Linea bridge delivers `msg.value - _fee` to the L1 recipient, meaning the L1 vault receives `value - minimumFee`, not `value`. The calling pool emits `BridgedETHToL1ViaNativeBridge(l1VaultETHForL2Chain, ethBalanceMinusFees)` claiming the full amount was bridged, creating a permanent per-call accounting gap of `minimumFee`.

---

### Finding Description

**Step 1 — Pool calls the messenger with the full balance:**

In `RSETHPool.bridgeAssetsViaNativeBridge()`, the pool computes `ethBalanceMinusFees` (pool balance minus protocol fees) and forwards it entirely to the messenger: [1](#0-0) 

```solidity
uint256 ethBalanceMinusFees = getETHBalanceMinusFees();

IL2Messenger(messenger).sendETHToL1ViaBridge{ value: ethBalanceMinusFees }(
    l2Bridge, l1VaultETHForL2Chain, ethBalanceMinusFees
);

emit BridgedETHToL1ViaNativeBridge(l1VaultETHForL2Chain, ethBalanceMinusFees);
```

The event records `ethBalanceMinusFees` as the amount bridged.

**Step 2 — LineaMessenger deducts the bridge fee silently:** [2](#0-1) 

```solidity
uint256 minimumFee = ILineaMessageService(l2bridge).minimumFeeInWei();
if (value <= minimumFee) revert InsufficientAmountForBridge();

ILineaMessageService(l2bridge).sendMessage{ value: value }(target, minimumFee, bytes(""));
```

`sendMessage` is called with `msg.value = value` but `_fee = minimumFee`. Per Linea's bridge semantics, the bridge delivers `msg.value - _fee` = `value - minimumFee` to the L1 recipient and retains `minimumFee` as the relayer fee.

**The gap:** The pool's event claims `ethBalanceMinusFees` was bridged; the L1 vault actually receives `ethBalanceMinusFees - minimumFee`. The pool's `getETHBalanceMinusFees()` only subtracts the protocol's own `feeEarnedInETH`, never the Linea bridge fee: [3](#0-2) 

```solidity
function getETHBalanceMinusFees() public view returns (uint256) {
    return address(this).balance - feeEarnedInETH;
}
```

---

### Impact Explanation

Every call to `bridgeAssetsViaNativeBridge` on a Linea-connected pool underfunds the L1 vault by exactly `minimumFee`. The `minimumFee` is not lost to an attacker — it is paid to the Linea relayer — but the L1 vault's actual receipts are permanently lower than what the pool's accounting and emitted events claim. This matches **Low: Contract fails to deliver promised returns, but doesn't lose value**.

---

### Likelihood Explanation

This triggers under normal operating conditions whenever `minimumFeeInWei() > 0`, which is always true on the live Linea bridge. No special actor, privilege, or race condition is required. Every bridge call exhibits the discrepancy.

---

### Recommendation

Subtract `minimumFee` from the bridged amount before emitting the event, or pass `value - minimumFee` as the amount to the bridge and emit that reduced figure. For example:

```solidity
uint256 minimumFee = ILineaMessageService(l2bridge).minimumFeeInWei();
uint256 amountDelivered = value - minimumFee;
ILineaMessageService(l2bridge).sendMessage{ value: value }(target, minimumFee, bytes(""));
emit ETHSentViaLineaBridge(l2bridge, target, amountDelivered, minimumFee);
```

The pool should similarly emit `BridgedETHToL1ViaNativeBridge(l1VaultETHForL2Chain, ethBalanceMinusFees - minimumFee)` to accurately reflect what the L1 vault will receive.

---

### Proof of Concept

1. Deploy a mock `ILineaMessageService` that records `msg.value` and `_fee` separately.
2. Deploy `LineaMessenger` pointing to the mock.
3. Call `sendETHToL1ViaBridge` with `value = 1 ether` and `minimumFeeInWei() = 0.001 ether`.
4. Assert: mock recorded `msg.value = 1 ether`, `_fee = 0.001 ether`, delivery = `0.999 ether`.
5. Assert: the pool's `BridgedETHToL1ViaNativeBridge` event emitted `1 ether`, not `0.999 ether`.
6. The mismatch between the emitted amount and the delivered amount is the bug. [2](#0-1) [4](#0-3)
