### Title
Bridger-Deposited Collateral Permanently Frozen After `removeAllowedToken` — (`contracts/L2/RsETHTokenWrapper.sol`)

### Summary
`depositBridgerAssets` accepts altRsETH collateral from the bridger without minting any wrsETH receipt. Once `TIMELOCK_ROLE` calls `removeAllowedToken` on that asset, every outbound path reverts with `TokenNotAllowed`, and no recovery function exists in the contract. The deposited collateral is irrecoverable under the current code.

### Finding Description
`depositBridgerAssets` is described as a "legacy function" that lets the bridger deposit altRsETH to back already-minted wrsETH on L2. [1](#0-0) 

Unlike `_deposit`, it transfers tokens in **without** minting wrsETH to the caller, so the bridger holds no receipt token that could later be burned to reclaim the collateral. [2](#0-1) 

`removeAllowedToken` simply flips the mapping to `false` with no check on whether any collateral of that token is currently held by the contract. [3](#0-2) 

The sole outbound path, `_withdraw`, unconditionally reverts for any non-allowed token. [4](#0-3) 

The contract inherits only `Initializable`, `AccessControlUpgradeable`, `ERC20Upgradeable`, and `ERC20PermitUpgradeable` — it does **not** inherit `Recoverable` and contains no `emergencyWithdraw`, `sweep`, or `recoverTokens` function. [5](#0-4) 

### Impact Explanation
Any altRsETH deposited via `depositBridgerAssets` before the token is removed from the allowlist becomes permanently locked in the wrapper. The bridger received no wrsETH for the deposit and therefore has no burn-path to recover the tokens. wrsETH holders also cannot redeem against that collateral. The funds are unrecoverable without a contract upgrade.

**Impact: Critical — Permanent freezing of funds.**

### Likelihood Explanation
The scenario is realistic during normal bridge lifecycle management: TIMELOCK removes a deprecated bridge token (e.g., migrating from one canonical bridge to another). The governance process does not enforce a pre-removal check on the contract's token balance. No on-chain guard prevents `removeAllowedToken` from being called while collateral is present.

**Likelihood: Medium.**

### Recommendation
Add a balance guard in `removeAllowedToken` that reverts if the contract still holds any of the token being removed:

```solidity
function removeAllowedToken(address _asset) external onlyRole(TIMELOCK_ROLE) {
    if (!allowedTokens[_asset]) revert TokenNotAllowed();
    if (ERC20Upgradeable(_asset).balanceOf(address(this)) > 0)
        revert TokenBalanceNonZero();
    allowedTokens[_asset] = false;
    emit TokenRemoved(_asset);
}
```

Alternatively, add an admin-gated `recoverToken(address asset, address to, uint256 amount)` function (similar to the existing `Recoverable` utility) so that collateral can always be extracted regardless of allowlist state. [6](#0-5)

import "forge-std/Test.sol";
import "contracts/L2/RsETHTokenWrapper.sol";
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract MockAltRsETH is ERC20 {
    constructor() ERC20("altRsETH", "altRsETH") {
        _mint(msg.sender, 1000 ether);
    }
}

contract FreezePoC is Test {
    RsETHTokenWrapper wrapper;
    MockAltRsETH altRsETH;

    address admin    = address(0xA);
    address timelock = address(0xB);
    address bridger  = address(0xC);
    address minter   = address(0xD);

    function setUp() public {
        altRsETH = new MockAltRsETH();
        wrapper  = new RsETHTokenWrapper();
        wrapper.initialize(admin, bridger, address(altRsETH));

        // Grant TIMELOCK_ROLE and MINTER_ROLE
        vm.prank(admin);
        wrapper.grantRole(wrapper.TIMELOCK_ROLE(), timelock);
        vm.prank(admin);
        wrapper.grantRole(wrapper.MINTER_ROLE(), minter);

        // Fund bridger
        altRsETH.transfer(bridger, 100 ether);
    }

    function test_permanentFreeze() public {
        // Step 1: Mint wrsETH so maxAmountToDepositBridgerAsset > 0
        vm.prank(minter);
        wrapper.mint(address(this), 100 ether);

        // Step 2: Bridger deposits 50 altRsETH as collateral (no wrsETH minted to bridger)
        vm.startPrank(bridger);
        altRsETH.approve(address(wrapper), 50 ether);
        wrapper.depositBridgerAssets(address(altRsETH), 50 ether);
        vm.stopPrank();

        assertEq(altRsETH.balanceOf(address(wrapper)), 50 ether);

        // Step 3: TIMELOCK removes the token
        vm.prank(timelock);
        wrapper.removeAllowedToken(address(altRsETH));

        // Step 4: No path can recover the 50 altRsETH
        vm.expectRevert(RsETHTokenWrapper.TokenNotAllowed.selector);
        wrapper.withdraw(address(altRsETH), 50 ether);

        vm.expectRevert(RsETHTokenWrapper.TokenNotAllowed.selector);
        wrapper.withdrawTo(address(altRsETH), address(this), 50 ether);

        // Collateral is permanently frozen
        assertEq(altRsETH.balanceOf(address(wrapper)), 50 ether);
    }
}
