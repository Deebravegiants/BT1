### Title
`emergencyWithdrawFromAave` silently withdraws less than requested without reverting when `aaveBalance < totalETHDepositedToAave` — (`contracts/LRTWithdrawalManager.sol`)

---

### Summary

`emergencyWithdrawFromAave(uint256 amount)` is designed to recover a specific amount of ETH from Aave in an emergency. However, when `aaveBalance <= totalETHDepositedToAave` (e.g., due to Aave rounding), the function silently withdraws less than `amount` with no revert, emitting an event that reports `withdrawnAmount < amount`. Operators have no on-chain signal that the full recovery failed.

---

### Finding Description

**Step 1 — `_collectInterestToTreasury()` silently returns 0** [1](#0-0) 

When `aaveBalance <= totalETHDepositedToAave`, the function returns 0 immediately with no revert and no event. This is intentional for the interest-collection path, but it means the post-condition of `emergencyWithdrawFromAave` is not enforced.

**Step 2 — `_withdrawFromAave` caps at `withdrawablePrincipal`** [2](#0-1) 

When `aaveBalance < totalETHDepositedToAave`, `withdrawablePrincipal = aaveBalance`. The function then sets `withdrawnAmount = min(amount, withdrawablePrincipal)`, which is strictly less than `amount`. It does **not** revert.

**Step 3 — `emergencyWithdrawFromAave` emits a misleading event and returns** [3](#0-2) 

The emitted `EmergencyWithdrawFromAave(withdrawnAmount, address(this))` reports the actual (reduced) amount. There is no check that `withdrawnAmount == amount` and no revert. The caller (PAUSER_ROLE) receives a success signal even though the full requested amount was not recovered.

**Concrete trigger condition:**

The `_checkAaveHealth()` function explicitly allows up to 2 wei of rounding difference: [4](#0-3) 

So a state where `totalETHDepositedToAave - aaveBalance == 1` is a normal, expected operating condition. In this state, calling `emergencyWithdrawFromAave(totalETHDepositedToAave)` withdraws `totalETHDepositedToAave - 1` with no revert.

---

### Impact Explanation

The function fails to deliver its promised behavior: withdraw exactly `amount` from Aave or revert. Instead it silently under-delivers. An operator who calls `emergencyWithdrawFromAave(X)` to recover all ETH before disabling Aave or taking corrective action will believe the contract now holds `X` ETH, when it actually holds `X - delta`. If pending user withdrawals depend on that ETH, subsequent `completeWithdrawal` calls may encounter a shortfall. No ETH is destroyed (it remains in Aave), but the contract's ETH balance is less than the operator expected, violating the invariant that emergency withdrawal must recover the full requested amount or revert.

**Scope match:** Low — Contract fails to deliver promised returns, but doesn't lose value.

---

### Likelihood Explanation

- The triggering condition (`aaveBalance < totalETHDepositedToAave` by even 1 wei) is explicitly acknowledged as a normal operating state by `_checkAaveHealth()`.
- No attacker action is required; this is a pure logic flaw reachable by the PAUSER_ROLE under normal Aave rounding.
- The PAUSER_ROLE is a legitimate, non-compromised actor following the documented emergency procedure.

---

### Recommendation

Add an explicit check in `emergencyWithdrawFromAave` after `_withdrawFromAave` returns, and revert if the withdrawn amount is less than requested (or less than the available Aave balance at the time of the call):

```solidity
uint256 withdrawnAmount = _withdrawFromAave(amount);
// Revert if we could not recover the full requested amount
uint256 effectiveRequest = amount > aaveBalance ? aaveBalance : amount;
if (withdrawnAmount < effectiveRequest) revert InsufficientAaveBalance();
emit EmergencyWithdrawFromAave(withdrawnAmount, address(this));
```

Alternatively, document explicitly that `emergencyWithdrawFromAave` is best-effort and may withdraw less than `amount`, and update monitoring tooling to compare `amount` vs `withdrawnAmount` in the emitted event.

---

### Proof of Concept

```solidity
// Setup:
// totalETHDepositedToAave = 100 ether
// Mock aaveAWETH.balanceOf(address(withdrawalManager)) = 100 ether - 1 wei

// Call:
withdrawalManager.emergencyWithdrawFromAave(100 ether);

// Assertions:
// 1. No revert occurred
// 2. EmergencyWithdrawFromAave event emitted with withdrawnAmount = 100 ether - 1 wei
// 3. address(withdrawalManager).balance increased by (100 ether - 1 wei), not 100 ether
// 4. totalETHDepositedToAave is now 1 wei (not 0)
// 5. Operator believes full 100 ether was recovered; contract is 1 wei short
```

```text
    function emergencyWithdrawFromAave(uint256 amount) external nonReentrant onlyRole(LRTConstants.PAUSER_ROLE) {
        if (!isAaveIntegrationEnabled) revert AaveIntegrationNotEnabled();

        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        if (aaveBalance == 0) revert InsufficientAaveBalance();

        // First collect any accrued interest to treasury
        _collectInterestToTreasury();

        uint256 withdrawnAmount = _withdrawFromAave(amount);

        emit EmergencyWithdrawFromAave(withdrawnAmount, address(this));
```

```text
    function _withdrawFromAave(uint256 amount) internal returns (uint256 withdrawnAmount) {
        if (amount == 0) return 0;

        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        if (aaveBalance == 0) revert InsufficientAaveBalance();

        // Only withdraw up to the principal amount (don't use accrued interest for user withdrawals)
        uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave ? aaveBalance : totalETHDepositedToAave;

        withdrawnAmount = amount > withdrawablePrincipal ? withdrawablePrincipal : amount;
        if (withdrawnAmount == 0) return 0;

        aaveWETHGateway.withdrawETH(aavePool, withdrawnAmount, address(this));
        totalETHDepositedToAave -= withdrawnAmount;

        emit ETHWithdrawnFromAave(withdrawnAmount, totalETHDepositedToAave);
    }
```

```text
    function _checkAaveHealth() internal view returns (bool healthy) {
        if (!isAaveIntegrationEnabled) return false;
        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        uint256 principal = totalETHDepositedToAave;
        // Allow small rounding differences (up to 2 wei)
        // Check if balance is significantly less than principal
        if (principal > aaveBalance && principal - aaveBalance > 2) return false;
        return true;
    }
```
