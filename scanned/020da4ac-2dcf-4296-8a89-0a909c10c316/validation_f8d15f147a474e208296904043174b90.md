### Title
Nested Unbounded Loop in `getAssetUnstaking()` Causes Growing Gas Cost in Public Deposit and Price-Update Functions - (File: contracts/NodeDelegator.sol)

### Summary
`NodeDelegator.getAssetUnstaking()` contains nested loops over all queued EigenLayer withdrawals and their strategies. This function is called for every NDC, for every supported asset, inside the public functions `depositETH()`, `depositAsset()`, `updateRSETHPrice()`, and `initiateWithdrawal()`. As the number of queued withdrawals and supported assets grows through normal protocol operation, the gas cost of these public functions grows multiplicatively, eventually making them prohibitively expensive or impossible to execute.

### Finding Description

`NodeDelegator.getAssetUnstaking()` fetches all queued withdrawals from EigenLayer and iterates over them with a nested loop: [1](#0-0) 

This function is called inside `LRTDepositPool.getAssetDistributionData()` and `getETHDistributionData()`, once per NDC in the `nodeDelegatorQueue`: [2](#0-1) [3](#0-2) 

`getTotalAssetDeposits()` calls `getAssetDistributionData()` / `getETHDistributionData()`, and is itself called from the public user-facing functions: [4](#0-3) 

`LRTOracle._getTotalEthInProtocol()` calls `getTotalAssetDeposits()` once per supported asset in a loop: [5](#0-4) 

`_getTotalEthInProtocol()` is called by the public `updateRSETHPrice()`: [6](#0-5) 

The total gas cost scales as: **`supportedAssets.length × nodeDelegatorQueue.length × queuedWithdrawals_per_NDC × strategies_per_withdrawal`**. Each `getAssetUnstaking()` call also makes an external call to EigenLayer's `getQueuedWithdrawals()`, which reads storage. With 5 supported assets and 10 NDCs, this is already 50 external EigenLayer calls per `updateRSETHPrice()` invocation.

The developers themselves acknowledge the gas concern in `LRTUnstakingVault.setMaxUncompletedWithdrawalCount()`: [7](#0-6) 

The comment states "120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price," confirming that beyond a threshold the function fails. The cap is set at 80 to leave a buffer, but this buffer does not account for the multiplicative effect of iterating over all supported assets — each asset causes a full re-traversal of all NDC queued withdrawals.

There is no caching of the `getAssetUnstaking()` result; it is recomputed from scratch on every call, for every asset.

### Impact Explanation

As the protocol operates normally — operators call `initiateUnstaking()` and new assets are added — the gas cost of `depositETH()`, `depositAsset()`, `updateRSETHPrice()`, and `initiateWithdrawal()` grows multiplicatively. At the limit, these public functions revert out-of-gas, temporarily freezing deposits, withdrawals, and price updates. This constitutes **temporary freezing of funds** (users cannot deposit or withdraw) and **medium unbounded gas consumption** on public entry points.

### Likelihood Explanation

This is triggered by normal protocol growth: adding supported assets (via `addNewSupportedAsset()`) and queuing withdrawals (via `initiateUnstaking()`). No malicious actor is required. The developers' own comment confirms awareness that the gas limit is reachable. The `maxUncompletedWithdrawalCount` cap (≤ 80) does not account for the multiplicative factor introduced by iterating over all supported assets in `_getTotalEthInProtocol()`.

### Recommendation

Cache the per-NDC unstaking amount in a storage variable (updated on `initiateUnstaking()` and `completeUnstaking()`) rather than recomputing it via a live EigenLayer loop on every call. This mirrors the mitigation recommended in the reference report: store the computed value and update it incrementally on state changes.

### Proof of Concept

Call chain for `updateRSETHPrice()` (publicly callable by anyone):

```
updateRSETHPrice()
  → _updateRsETHPrice()
    → _getTotalEthInProtocol()                          // loops over N supported assets
      → getTotalAssetDeposits(asset)                    // called N times
        → getAssetDistributionData(asset) / getETHDistributionData()
          → for each NDC (M NDCs):                      // loops over M NDCs
              getAssetUnstaking(asset)                  // called N×M times
                → getQueuedWithdrawals(NDC)             // external EigenLayer call, N×M times
                → for each withdrawal (K):              // nested loop
                    for each strategy (S):              // nested loop
```

With N=5 assets, M=10 NDCs, K=8 withdrawals/NDC, S=3 strategies: **1,200 inner iterations + 50 external EigenLayer storage calls** per single `updateRSETHPrice()` invocation. Adding more supported assets (each requiring `TIME_LOCK_ROLE`, a normal governance action) linearly increases this cost. The same path is hit on every `depositETH()` and `depositAsset()` call by any user. [1](#0-0) [5](#0-4) [2](#0-1)

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```

```text
    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```
