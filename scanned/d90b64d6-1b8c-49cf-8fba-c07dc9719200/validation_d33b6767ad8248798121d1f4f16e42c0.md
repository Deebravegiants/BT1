### Title
Unbounded Nested Loop in `updateRSETHPrice()` Causes Permanent DOS of Price Updates and Fee Minting - (File: contracts/LRTOracle.sol)

### Summary
`updateRSETHPrice()` is a public, permissionless function that internally executes a nested unbounded loop across all supported assets, all NodeDelegators, and all queued EigenLayer withdrawals per NodeDelegator. As the protocol scales, this function's gas cost grows without bound, eventually exceeding the block gas limit and permanently reverting, freezing the rsETH price and blocking fee minting.

### Finding Description

The call chain is:

1. `updateRSETHPrice()` (public, no access control) calls `_updateRsETHPrice()`, which calls `_getTotalEthInProtocol()`.

2. `_getTotalEthInProtocol()` loops over every entry in `supportedAssets`: [1](#0-0) 

3. For each asset, it calls `ILRTDepositPool.getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)`, which loops over every entry in `nodeDelegatorQueue`: [2](#0-1) 

4. For each NodeDelegator, it calls `getAssetUnstaking(asset)`, which fetches **all** queued EigenLayer withdrawals and iterates over them with a nested loop: [3](#0-2) 

The total gas cost is **O(supportedAssets × NDCs × queuedEigenLayerWithdrawals)**. None of these three dimensions is bounded by a protocol-level cap that prevents the product from exceeding the block gas limit. Queued EigenLayer withdrawals accumulate naturally during normal protocol operations (each user withdrawal request that flows through EigenLayer adds one entry) and are only removed when `completeQueuedWithdrawal()` is called. During periods of high withdrawal demand, many withdrawals can be in-flight simultaneously across multiple NDCs.

### Impact Explanation

When `updateRSETHPrice()` permanently reverts:

- The stored `rsETHPrice` is frozen at its last value. New deposits via `getRsETHAmountToMint` and withdrawals via `getExpectedAssetAmount` use this stale price.
- Protocol fee minting (`_checkAndUpdateDailyFeeMintLimit` + `IRSETH.mint`) inside `_updateRsETHPrice()` is permanently blocked, freezing all unclaimed yield owed to the treasury.
- The automatic price-drop protection (auto-pause of `LRTDepositPool` and `LRTWithdrawalManager`) can no longer trigger, removing a critical safety mechanism.

**Impact: Medium — Unbounded gas consumption leading to permanent freezing of unclaimed yield.** [4](#0-3) [5](#0-4) 

### Likelihood Explanation

`updateRSETHPrice()` has no access control — any external caller can attempt it. The growth of queued EigenLayer withdrawals is a direct consequence of normal protocol operations: every user withdrawal request that the operator processes through EigenLayer adds an entry to the queue. With a large user base and high withdrawal demand, many withdrawals will be in-flight concurrently across multiple NDCs. No adversarial action is required; organic protocol growth is sufficient to trigger the DOS. [4](#0-3) [6](#0-5) 

### Recommendation

Decouple the per-NDC `getAssetUnstaking()` call from the price update path. Options:

1. **Cache the unstaking amount**: Maintain a storage variable in `NodeDelegator` that is updated incrementally when withdrawals are queued and completed, rather than recomputing it by iterating all queued withdrawals on every price update.
2. **Limit loop depth**: Introduce a maximum cap on the number of queued EigenLayer withdrawals per NDC that are iterated, or paginate the computation.
3. **Separate accounting**: Track `assetUnstakingFromEigenLayer` as a running counter updated at queue/complete time, analogous to how `assetsCommitted` is tracked in `LRTWithdrawalManager`. [7](#0-6) 

### Proof of Concept

```
Scenario: 3 supported assets, 5 NDCs, 500 queued EigenLayer withdrawals per NDC

Call: updateRSETHPrice()
  → _getTotalEthInProtocol()
    → loop 3 assets
      → getTotalAssetDeposits(asset)
        → getAssetDistributionData(asset)
          → loop 5 NDCs
            → getAssetUnstaking(asset)   // fetches all 500 queued withdrawals from EigenLayer
              → outer loop: 500 withdrawals
                → inner loop: strategies per withdrawal

Total iterations: 3 × 5 × 500 = 7,500 storage reads from EigenLayer's DelegationManager,
each involving external calls and memory allocation.

At ~2,100 gas per SLOAD and external call overhead, this easily exceeds the 30M block gas limit.
Result: updateRSETHPrice() always reverts. rsETH price is permanently frozen.
        Fee minting is permanently blocked.
``` [8](#0-7) [9](#0-8)

        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
