### Title
Missing Upper Bound on `_duration` in `setRewardsDuration` Permanently Freezes Staking Rewards - (File: contracts/KERNEL/KernelDepositPool.sol)

### Summary
`KernelDepositPool.setRewardsDuration` accepts any non-zero `_duration` with no upper bound. If the admin fat-fingers an excessively large duration (e.g., 1000 years), the subsequent `notifyRewardAmount` call locks all deposited reward tokens in the contract for that entire period with no recovery path, permanently freezing unclaimed yield for all stakers.

### Finding Description
`setRewardsDuration` validates only that `_duration != 0`: [1](#0-0) 

There is no upper bound. When `notifyRewardAmount` is subsequently called, it sets: [2](#0-1) 

With `duration = 31_536_000_000` (1000 years in seconds):
- `rewardRate = receivedAmount / 31_536_000_000` — non-zero for any realistic reward amount, so the `RewardRateZero` guard does not trigger.
- `finishAt = block.timestamp + 31_536_000_000` — set 1000 years into the future.

All reward tokens are now locked in the contract, dripping out at a negligible rate. The admin cannot call `setRewardsDuration` again until `finishAt` passes (1000 years), because of the guard: [3](#0-2) 

Calling `notifyRewardAmount` again does not help — it recalculates `remaining` using the still-enormous `(finishAt - block.timestamp) * rewardRate` and resets `finishAt` to yet another 1000 years in the future, compounding the lock. There is no emergency reward-withdrawal function in the contract. [4](#0-3) 

### Impact Explanation
All KERNEL staking reward tokens deposited via `notifyRewardAmount` become permanently inaccessible to stakers for the misconfigured duration. Stakers who have locked their KERNEL tokens (subject to `withdrawalDelay`) and are waiting for rewards receive effectively zero yield for the entire period. This matches **Medium — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
Low. Requires the `DEFAULT_ADMIN_ROLE` holder to fat-finger the duration value (e.g., entering seconds instead of days, or adding extra digits). No malicious intent is required — an honest configuration mistake is sufficient. This is the same likelihood class as the reference report.

### Recommendation
Add an upper bound check in `setRewardsDuration`, analogous to the lower bound already present:

```solidity
uint256 public constant MAX_REWARDS_DURATION = 365 days;

function setRewardsDuration(uint256 _duration) external onlyRole(DEFAULT_ADMIN_ROLE) {
    if (finishAt >= block.timestamp) revert RewardDurationNotFinished();
    if (_duration == 0) revert InvalidDuration();
    if (_duration > MAX_REWARDS_DURATION) revert InvalidDuration(); // add upper bound
    duration = _duration;
    emit RewardsDurationUpdated(_duration);
}
```

Note that `KernelDepositPool` already defines `MAX_WITHDRAWAL_DELAY = 30 days` as a ceiling for the withdrawal delay: [5](#0-4) 

The same pattern should be applied to `duration`.

### Proof of Concept
1. Admin calls `setRewardsDuration(31_536_000_000)` (1000 years in seconds). Passes all checks since `31_536_000_000 != 0` and `finishAt < block.timestamp`.
2. Admin calls `notifyRewardAmount(1_000_000e18)` (1M reward tokens).
   - `rewardRate = 1_000_000e18 / 31_536_000_000 = 31_709_791_983_764_586` (non-zero, no revert).
   - `finishAt = block.timestamp + 31_536_000_000`.
3. Stakers call `getReward()` — they receive only `rewardRate * elapsed_seconds / totalKernelStaked` per second, which is negligible.
4. Admin attempts `setRewardsDuration(7 days)` to correct the mistake — reverts with `RewardDurationNotFinished` because `finishAt` is 1000 years away.
5. Admin attempts `notifyRewardAmount(0)` — reverts with `AmountZero`.
6. All 1M reward tokens remain locked in `KernelDepositPool` for ~1000 years with no recovery path. [6](#0-5)

```text
    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
```
