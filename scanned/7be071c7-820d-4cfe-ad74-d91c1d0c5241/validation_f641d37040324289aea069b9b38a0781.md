### Title
Missing Chainlink `minAnswer`/`maxAnswer` Circuit Breaker Check Allows Inflated Price Consumption During LST Depeg - (File: contracts/oracles/ChainlinkPriceOracle.sol)

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` with no validation whatsoever — no staleness check, no negative-price guard, and critically no check against the Chainlink aggregator's built-in `minAnswer`/`maxAnswer` circuit breakers. If a supported LST asset depegs severely, the Chainlink aggregator will clamp its return value at `minAnswer` rather than reporting the true crashed price. The protocol will then accept the depegged asset at the artificially inflated `minAnswer` price, allowing an attacker to mint excess rsETH and extract value from existing holders.

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` is the price source for all LST assets (stETH, cbETH, etc.) deposited into the LRT protocol: [1](#0-0) 

The function calls `latestRoundData()` and immediately casts the result to `uint256` with no guards:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

There is no check that `price > 0`, no staleness check on `updatedAt`, no `answeredInRound >= roundId` check, and — the core issue — no check that `price` is within the aggregator's `[minAnswer, maxAnswer]` range.

Chainlink aggregators have hardcoded `minAnswer` and `maxAnswer` values accessible one level below the proxy (on the underlying aggregator). When a real-world price crashes below `minAnswer`, the aggregator stops reporting the true price and instead returns `minAnswer`. This is exactly what happened with LUNA and has been documented for other assets.

This price is consumed directly in `LRTDepositPool.getRsETHAmountToMint()`: [2](#0-1) 

And in withdrawal processing via `LRTWithdrawalManager._createUnlockParams()`: [3](#0-2) 

### Impact Explanation

**Critical — Direct theft of user funds.**

If a supported LST (e.g., stETH, cbETH) depegs severely:
1. The Chainlink aggregator clamps its return at `minAnswer` (e.g., 0.95 ETH) while the true market price is far lower (e.g., 0.10 ETH).
2. An attacker buys the depegged LST cheaply on the open market.
3. The attacker calls `LRTDepositPool.depositAsset()` with the depegged LST.
4. `getRsETHAmountToMint()` uses the inflated `minAnswer` price, minting far more rsETH than the deposited asset is worth.
5. The attacker redeems or sells the excess rsETH, extracting ETH value from all existing rsETH holders.

This directly dilutes and steals from existing rsETH holders' backing, constituting direct theft of funds at rest. [4](#0-3) 

### Likelihood Explanation

LST depeg events are realistic and have historical precedent (stETH depeg in 2022, LUNA collapse). The Chainlink `minAnswer` circuit breaker behavior is well-documented. Any supported LST asset in the protocol is a potential trigger. The attack requires no special permissions — any unprivileged depositor can execute it by calling `depositAsset()`.

### Recommendation

Retrieve `minAnswer` and `maxAnswer` from the underlying Chainlink aggregator (one level below the proxy via `AggregatorV2V3Interface`) and revert if the returned price is at or near either bound:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    require(price > 0, "Invalid price");
    require(updatedAt != 0, "Incomplete round");
    require(answeredInRound >= roundId, "Stale price");

    // Retrieve circuit breaker bounds from the underlying aggregator
    int192 minAnswer = AggregatorV2V3Interface(address(priceFeed)).minAnswer();
    int192 maxAnswer = AggregatorV2V3Interface(address(priceFeed)).maxAnswer();
    require(price > minAnswer, "Price at lower circuit breaker bound");
    require(price < maxAnswer, "Price at upper circuit breaker bound");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

### Proof of Concept

1. Assume stETH is a supported asset with a Chainlink feed whose `minAnswer` = 0.95e18 (95% of ETH).
2. A black-swan event causes stETH to trade at 0.05e18 on the open market.
3. Chainlink aggregator hits `minAnswer` and freezes at 0.95e18.
4. Attacker acquires 1000 stETH for ~50 ETH worth of value.
5. Attacker calls `LRTDepositPool.depositAsset(stETH, 1000e18, minRSETH, "")`.
6. `getRsETHAmountToMint()` computes: `(1000e18 * 0.95e18) / rsETHPrice` — yielding rsETH worth ~950 ETH.
7. Attacker redeems rsETH, extracting ~900 ETH net profit from existing rsETH holders. [5](#0-4)
