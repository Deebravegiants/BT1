### Title
Fee Truncation to Zero for Low-Decimal Tokens in Pool Token Deposits - (`contracts/pools/RSETHPoolV3.sol`, `contracts/pools/RSETHPool.sol`, `contracts/pools/RSETHPoolNoWrapper.sol`, `contracts/pools/RSETHPoolV3ExternalBridge.sol`, `contracts/agETH/AGETHPoolV3.sol`)

---

### Summary

Every pool contract in the LRT-rsETH suite computes the protocol fee as `amount * feeBps / 10_000`. Because Solidity integer division truncates toward zero, any deposit whose `amount * feeBps` product is less than `10_000` produces a fee of exactly `0`. For tokens with low decimal precision (e.g., EURS at 2 decimals, USDC at 6 decimals) that are added as supported tokens via `addSupportedToken`, a depositor can make economically meaningful deposits and pay zero protocol fees.

---

### Finding Description

All pool contracts share the same fee calculation pattern in their `viewSwapRsETHAmountAndFee` (or `viewSwapAgETHAmountAndFee`) functions:

```solidity
fee = amount * feeBps / 10_000;
uint256 amountAfterFee = amount - fee;
``` [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

The denominator is `10_000`. For the fee to be non-zero, `amount * feeBps >= 10_000` must hold. With the minimum meaningful `feeBps = 1` (0.01%):

| Token | Decimals | Threshold for zero fee | Real-world value |
|---|---|---|---|
| ETH / wrsETH | 18 | `amount < 10,000 wei` | ~0.00000001 ETH (dust) |
| USDC | 6 | `amount < 10,000` units | `< 0.01 USDC` |
| EURS | 2 | `amount < 10,000` units | `< 100 EURS` |

For EURS (2 decimals), a depositor sending 9,999 units (≈ 99.99 EURS) with `feeBps = 1` computes:
```
fee = 9999 * 1 / 10_000 = 0
```
The depositor receives the full rsETH/agETH equivalent of 99.99 EURS while the protocol collects nothing.

With a higher `feeBps` (e.g., `feeBps = 10`, i.e., 0.1%), the threshold drops to `amount < 1,000` units (< 10 EURS), but the truncation still applies.

The `addSupportedToken` function in all pool contracts accepts any ERC20 token with a valid oracle, making low-decimal token support a realistic deployment scenario. [6](#0-5)

### Impact Explanation

Protocol fees are the treasury's yield from pool operations. When `fee` truncates to `0`, the protocol receives no revenue from the deposit while the depositor receives the full rsETH/agETH mint. This constitutes **theft of unclaimed yield** (protocol fee revenue). The depositor can repeat this for every deposit below the truncation threshold, systematically draining all fee revenue for that token.

**Impact: High** — Theft of unclaimed yield (protocol fees).

---

### Likelihood Explanation

The pools already support arbitrary ERC20 tokens via `addSupportedToken`. Stablecoins with 2–6 decimals (EURS, USDC, USDT) are common bridged assets on L2 chains where these pools are deployed. If any such token is added, any depositor can exploit the truncation without any special privileges. The attacker only needs to call `deposit(token, amount, referralId)` with `amount` below the truncation threshold.

**Likelihood: Medium** — Requires a low-decimal token to be added as a supported token, which is a realistic operational decision.

---

### Recommendation

Add a minimum fee check after computing the fee, or use a higher-precision intermediate calculation:

```solidity
// Option 1: Revert if fee rounds to zero (mirrors the Rubicon mitigation)
fee = amount * feeBps / 10_000;
require(fee > 0 || feeBps == 0, "Fee truncated to zero");

// Option 2: Scale up before dividing to preserve precision
fee = (amount * feeBps + 9_999) / 10_000; // round up
```

Alternatively, enforce a minimum deposit amount per token that guarantees `amount * feeBps >= 10_000` for any configured `feeBps`.

---

### Proof of Concept

Consider `AGETHPoolV3` with a supported EURS token (2 decimals) and `feeBps = 5` (0.05%):

1. Attacker calls `deposit(EURS, 1_999, "ref")` (≈ 19.99 EURS).
2. `viewSwapAgETHAmountAndFee(1_999, EURS)` computes:
   - `fee = 1_999 * 5 / 10_000 = 9_995 / 10_000 = 0` (truncated)
   - `amountAfterFee = 1_999 - 0 = 1_999`
3. `feeEarnedInToken[EURS] += 0` — no fee recorded.
4. Attacker receives agETH for the full 1,999 EURS with zero fee paid.
5. Repeating this across many deposits, the protocol collects zero fees for all EURS deposits below 2,000 units. [9](#0-8) [10](#0-9)
