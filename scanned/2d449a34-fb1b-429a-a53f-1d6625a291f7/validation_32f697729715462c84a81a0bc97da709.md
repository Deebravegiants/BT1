### Title
Permissionless `sendFunds()` + `updateRSETHPrice()` Enables Flash-Deposit Yield Extraction via `instantWithdrawal` — (`contracts/FeeReceiver.sol`, `contracts/LRTOracle.sol`, `contracts/LRTWithdrawalManager.sol`)

---

### Summary

An attacker can atomically: (1) deposit ETH at the stale rsETH price, (2) call the permissionless `FeeReceiver.sendFunds()` to push MEV/execution-layer rewards into `LRTDepositPool`, (3) call the permissionless `LRTOracle.updateRSETHPrice()` to reflect the higher TVL in the rsETH price, and (4) call `LRTWithdrawalManager.instantWithdrawal()` which prices the exit at the newly inflated rsETH price — extracting yield that was earned by existing stakers without having held rsETH for any meaningful time.

---

### Finding Description

**Step 1 — Permissionless reward injection**

`FeeReceiver.sendFunds()` carries no access control:

```solidity
function sendFunds() external {
    uint256 balance = address(this).balance;
    ILRTDepositPool(depositPool).receiveFromRewardReceiver{ value: balance }();
    emit MevRewardsAddedToTVL(balance);
}
``` [1](#0-0) 

Any caller can push accumulated MEV/EL rewards from `FeeReceiver` into `LRTDepositPool`, immediately increasing `address(this).balance` of the deposit pool and therefore the value returned by `getTotalAssetDeposits(ETH_TOKEN)`.

**Step 2 — Permissionless price update**

`LRTOracle.updateRSETHPrice()` is also open to any caller:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [2](#0-1) 

Inside `_updateRsETHPrice()`, the new price is computed as:

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
``` [3](#0-2) 

where `totalETHInProtocol` now includes the reward ETH just pushed by `sendFunds()`. The protocol fee (`protocolFeeInETH`) is deducted and minted as rsETH to the treasury, but the **remaining yield is fully reflected in the new rsETH price** and stored in `rsETHPrice`.

**Step 3 — `instantWithdrawal` uses the live price**

`getExpectedAssetAmount` reads the freshly updated `rsETHPrice`:

```solidity
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
``` [4](#0-3) 

`instantWithdrawal` calls this directly and redeems from `LRTUnstakingVault`:

```solidity
uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
IRSETH(lrtConfig.rsETH()).burnFrom(address(msg.sender), rsETHUnstaked);
...
unstakingVault.redeem(asset, assetAmountUnlocked);
``` [5](#0-4) 

**The math:**

Let T = TVL before rewards, R = reward amount, S = rsETH supply, P₀ = T/S (old price).

After `sendFunds()` + `updateRSETHPrice()`:
- `P₁ = (T + R − R·feeInBPS/10000) / S`

Attacker deposited D ETH at P₀, receiving `D·S/T` rsETH. On instant withdrawal:
- `assetAmountUnlocked = (D·S/T) · P₁ = D · (T + R·(1 − feeInBPS/10000)) / T`
- After `instantWithdrawalFee`: `userAmount = assetAmountUnlocked · (1 − instantWithdrawalFee/10000)`

**Net gain = `userAmount − D > 0`** whenever the yield fraction `R·(1−protocolFee)/T` exceeds the `instantWithdrawalFee` rate. For typical MEV rewards (small relative to TVL) and low instant-withdrawal fees, this is satisfied.

The ETH the attacker receives comes from `LRTUnstakingVault` (pre-funded for other users' queued withdrawals), while the reward ETH remains in `LRTDepositPool` — the attacker extracts value from the vault at an inflated price.

---

### Impact Explanation

Existing rsETH holders' unclaimed yield (MEV/EL rewards sitting in `FeeReceiver`) is partially captured by a flash depositor who contributes nothing to the protocol's staking activity. The ETH is drained from `LRTUnstakingVault`, reducing the pool available for legitimate queued withdrawals. Impact: **High — theft of unclaimed yield**.

---

### Likelihood Explanation

- Both `sendFunds()` and `updateRSETHPrice()` are permissionless public functions callable by anyone.
- The attack can be executed atomically in a single transaction.
- The only meaningful guard is `pricePercentageLimit`: if set and the reward causes a price increase above the threshold, `updateRSETHPrice()` reverts for non-managers. [6](#0-5) 
  However: (a) `pricePercentageLimit` defaults to 0 (no limit), (b) even when set, typical MEV rewards are small relative to TVL and stay within any reasonable daily threshold, and (c) the check only triggers when `newRsETHPrice > highestRsethPrice` — if the current price is below the all-time high, the guard is bypassed entirely.
- `instantWithdrawalFee` reduces but does not eliminate profit unless it exceeds the per-unit yield.
- ETH must be present in `LRTUnstakingVault` — a realistic precondition given normal protocol operation.

Likelihood: **Medium-High**.

---

### Recommendation

1. **Make `sendFunds()` access-controlled** (e.g., `onlyRole(LRTConstants.MANAGER)`) so rewards can only be injected by trusted operators who coordinate with the oracle update.
2. **Alternatively, snapshot the rsETH price at deposit time** and use the lesser of the deposit-time price and the current price when computing instant-withdrawal amounts — preventing a depositor from benefiting from a price increase that occurred after their deposit.
3. **Enforce a minimum holding period** before `instantWithdrawal` is available to a given rsETH balance, preventing atomic deposit→price-update→withdraw sequences.
4. **Ensure `pricePercentageLimit` is always set** to a non-zero value in production to limit the magnitude of any single price update.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

// Fork test (Foundry) — run against a local fork with FeeReceiver holding accumulated rewards
// and LRTUnstakingVault holding ETH available for instant withdrawal.

function testYieldExtractionViaInstantWithdrawal() public {
    address attacker = makeAddr("attacker");
    uint256 depositAmount = 10 ether;
    vm.deal(attacker, depositAmount);

    // 1. Record attacker's starting balance
    uint256 startBalance = attacker.balance; // 10 ETH

    // 2. Attacker deposits ETH at current (pre-reward) rsETH price P0
    vm.prank(attacker);
    lrtDepositPool.depositETH{value: depositAmount}(0, "");
    uint256 rsETHBalance = rsETH.balanceOf(attacker);

    // 3. Anyone calls sendFunds() — pushes MEV rewards into LRTDepositPool
    feeReceiver.sendFunds(); // permissionless

    // 4. Anyone calls updateRSETHPrice() — price rises to P1
    lrtOracle.updateRSETHPrice(); // permissionless

    // 5. Attacker instantly withdraws at P1
    vm.startPrank(attacker);
    rsETH.approve(address(lrtWithdrawalManager), rsETHBalance);
    lrtWithdrawalManager.instantWithdrawal(LRTConstants.ETH_TOKEN, rsETHBalance, "");
    vm.stopPrank();

    // 6. Assert net gain
    uint256 endBalance = attacker.balance;
    assertGt(endBalance, startBalance, "Attacker extracted yield");
}
```
