### Title
Unprotected `initialize2()` Reinitializer Allows Any Caller to Front-Run Upgrade Initialization - (File: contracts/NodeDelegator.sol)

### Summary
`NodeDelegator.initialize2()` carries no access control modifier, allowing any external caller to invoke it before the admin during a contract upgrade, permanently blocking the admin's own initialization call and leaving `lastNonce` in an attacker-influenced state.

### Finding Description
`NodeDelegator.initialize2()` is a reinitializer that sets the private `lastNonce` variable:

```solidity
function initialize2() external reinitializer(2) {
    lastNonce = _getNonce();
}
``` [1](#0-0) 

The `reinitializer(2)` modifier from OpenZeppelin's `Initializable` guarantees the function executes exactly once (when `_initialized < 2`), but it places **no restriction on who may call it**. Every other reinitializer in the codebase that touches sensitive state carries an explicit role check:

- `LRTWithdrawalManager.initialize2()` requires `onlyRole(LRTConstants.UNLOCKED_WITHDRAWAL_INITIALIZER)` [2](#0-1) 

- `LRTWithdrawalManager.initialize3()` requires `onlyLRTManager` [3](#0-2) 

- All `RSETHPoolV3ExternalBridge.reinitialize()` overloads require `onlyRole(DEFAULT_ADMIN_ROLE)` [4](#0-3) 

`NodeDelegator.initialize2()` is the sole reinitializer in the codebase that omits any access guard.

The analog to the external report is direct: just as `nodeIntegration: true` exposes privileged Node.js primitives to untrusted JavaScript, the missing access control on `initialize2()` exposes a privileged upgrade step to any untrusted external caller.

### Impact Explanation
An attacker who front-runs the admin's `initialize2()` call achieves two effects:

1. **Permanent lock-out of the admin's call.** Once the attacker's transaction executes, `_initialized` is set to `2`. Every subsequent admin call to `initialize2()` reverts with `InvalidInitialization`. The admin must redeploy a new implementation with `reinitializer(3)` to recover, incurring delay and operational cost.

2. **Potentially incorrect `lastNonce` state.** `lastNonce` is set to whatever `_getNonce()` returns at the attacker's chosen block. If `_getNonce()` is time- or state-sensitive (e.g., reads an EigenLayer nonce counter that advances with each queued withdrawal), the value recorded may differ from what the admin intended, causing subsequent nonce-based EigenLayer operations in `NodeDelegator` to behave incorrectly.

Impact: **Low** — contract fails to deliver promised upgrade behavior; potential for incorrect nonce state affecting EigenLayer interactions during the recovery window.

### Likelihood Explanation
The attack requires only that an adversary monitor the public mempool for the admin's pending `initialize2()` call and submit the same call with a higher gas price. MEV infrastructure on Ethereum mainnet makes this straightforward. Likelihood: **Low-Medium** (requires active mempool monitoring but no special privileges or capital).

### Recommendation
Add the same access control used by every other reinitializer in the codebase:

```solidity
function initialize2() external reinitializer(2) onlyLRTAdmin {
    lastNonce = _getNonce();
}
```

`onlyLRTAdmin` is already defined in `LRTConfigRoleChecker`, which `NodeDelegator` inherits via `LRTConfigRoleChecker`. [5](#0-4) 

### Proof of Concept
1. Admin deploys a new `NodeDelegator` implementation containing `initialize2()`.
2. Admin calls `upgradeTo(newImpl)` on the proxy (does not call `initialize2()`).
3. Admin broadcasts a follow-up transaction calling `initialize2()`.
4. Attacker observes the pending transaction in the mempool and submits an identical `initialize2()` call with higher gas.
5. Attacker's transaction is mined first; `lastNonce = _getNonce()` executes under the attacker's chosen block context; `_initialized` advances to `2`.
6. Admin's transaction reverts (`InvalidInitialization`).
7. `NodeDelegator` now holds an attacker-timed `lastNonce` value; the admin must deploy `reinitializer(3)` to correct state, during which EigenLayer nonce-dependent operations may fail.
