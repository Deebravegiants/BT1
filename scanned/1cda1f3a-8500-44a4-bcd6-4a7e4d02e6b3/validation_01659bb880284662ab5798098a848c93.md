### Title
Stale Chainlink Price Accepted Without Validation in `ChainlinkPriceOracle` Enables Incorrect rsETH Price and Fee Theft — (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards every validation field (`updatedAt`, `answeredInRound`, `roundId`), accepting stale, incomplete, or zero prices without revert. This price flows directly into `LRTOracle._getTotalEthInProtocol()` and `LRTDepositPool.getRsETHAmountToMint()`. When a Chainlink feed goes stale, the rsETH price update and fee-minting calculation are both corrupted, enabling theft of yield from rsETH holders or a temporary protocol freeze.

---

### Finding Description

`contracts/oracles/ChainlinkPriceOracle.sol` line 52 reads:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values of `latestRoundData()` are available (`roundId`, `answer`, `startedAt`, `updatedAt`, `answeredInRound`), but only `answer` is used. There is no check that:
- `updatedAt` is recent (staleness guard),
- `answeredInRound >= roundId` (completeness guard), or
- `price > 0` (validity guard).

The protocol already knows how to do this correctly. `contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol` — used for L2 pool collateral — applies all three guards:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The unvalidated price from `ChainlinkPriceOracle` propagates through two critical paths:

**Path 1 — rsETH price update and fee minting:**
`LRTOracle._getTotalEthInProtocol()` (line 339) calls `getAssetPrice(asset)` for every supported LST. The result feeds `_updateRsETHPrice()` (line 214), which computes `totalETHInProtocol`, derives `newRsETHPrice`, and mints protocol fee rsETH to the treasury proportional to the apparent TVL increase (lines 244–307).

**Path 2 — per-deposit rsETH minting:**
`LRTDepositPool.getRsETHAmountToMint()` (line 520) computes:
```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```
A stale asset price directly inflates or deflates the rsETH issued per deposit.

---

### Impact Explanation

**Theft of yield (High):** When a Chainlink feed for a supported LST (e.g., stETH/ETH) goes stale at an inflated value, `_getTotalEthInProtocol()` overcounts TVL. `_updateRsETHPrice()` then sees a phantom TVL increase, computes `rewardAmount = totalETHInProtocol - previousTVL` as artificially large, and mints excess fee rsETH to the treasury. This dilutes all existing rsETH holders — their share of the underlying pool shrinks without any real yield having been earned. The dilution is permanent once the fee mint is executed.

**Temporary fund freeze (Medium):** If a feed returns 0 (deprecated or broken), `getAssetPrice` returns 0, `_getTotalEthInProtocol()` undercounts TVL, `newRsETHPrice` drops below `highestRsethPrice` by more than `pricePercentageLimit`, and the downside-protection logic at lines 270–281 auto-pauses `LRTDepositPool` and `LRTWithdrawalManager`, freezing all user deposits and withdrawals until an admin manually unpauses.

---

### Likelihood Explanation

Chainlink feeds go stale during network congestion, sequencer outages (on L2), or when a feed is deprecated and replaced. These are documented, recurring events. The protocol already treats this risk as real enough to guard against it in `ChainlinkOracleForRSETHPoolCollateral`, yet the same guard is absent in `ChainlinkPriceOracle`, which is the oracle used for the core L1 deposit and price-update paths. No privileged actor or oracle operator compromise is required — the condition arises from normal Chainlink infrastructure behaviour.

---

### Recommendation

Apply the same validation pattern already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0)            revert IncompleteRound();
    if (price <= 0)                revert InvalidPrice();
    if (block.timestamp - updatedAt > MAX_STALENESS_PERIOD) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

### Proof of Concept

1. A Chainlink stETH/ETH feed goes stale (e.g., `updatedAt` is 25 hours old, last reported price 1.06 ETH per stETH while the true rate has since dropped to 1.04).
2. Any caller (unprivileged) invokes `LRTOracle.updateRSETHPrice()`.
3. `_getTotalEthInProtocol()` calls `LRTOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns the stale inflated value 1.06e18 with no revert.
4. `totalETHInProtocol` is overcounted; `totalETHInProtocol > previousTVL` evaluates true.
5. `rewardAmount` and `protocolFeeInETH` are computed on the phantom gain.
6. `IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee)` executes, minting excess rsETH to the treasury.
7. All existing rsETH holders are diluted by the spurious fee mint; the yield that was never earned has been permanently extracted.
