### Title
`getClaimableAmount` Returns Non-Zero Value When Contract Is Paused, Misrepresenting Actual Claimable Amount — (File: `contracts/KERNEL/KernelTop100MerkleDistributor.sol`)

---

### Summary

`getClaimableAmount()` in `KernelTop100MerkleDistributor` returns a non-zero vested amount even when the contract is paused, at which point `claim()` and `claimAndStake()` are both blocked by `whenNotPaused`. The view function therefore misrepresents the true claimable amount, which is effectively zero while paused.

---

### Finding Description

`getClaimableAmount` is a public view function that delegates directly to `_getUnclaimedVestedAmount`:

```solidity
function getClaimableAmount(address user, uint256 amount) external view returns (uint256) {
    return _getUnclaimedVestedAmount(user, amount);
}
``` [1](#0-0) 

`_getUnclaimedVestedAmount` computes the vested-but-unclaimed token amount based purely on time elapsed and the user's prior claim history: [2](#0-1) 

It contains no check for the contract's paused state. However, both execution paths that actually transfer tokens — `claim()` and `claimAndStake()` — are gated by `whenNotPaused`: [3](#0-2) [4](#0-3) 

When the contract is paused, `getClaimableAmount` continues to return a positive vested amount, but any attempt to actually claim that amount will revert. The view function's return value is therefore inaccurate with respect to what is actually redeemable at that moment.

---

### Impact Explanation

**Low. Contract fails to deliver promised returns, but doesn't lose value.**

Any external protocol (e.g., a lending or collateral-valuation system) that calls `getClaimableAmount` to determine how much KERNEL a user can claim and uses that as a basis for credit or collateral will receive an inflated figure during a pause. The actual claimable amount is zero while paused, but the view function reports a non-zero value. This can cause external integrators to extend credit or compute collateral against tokens that cannot currently be retrieved, mirroring the exact DeFi composability risk described in the Sablier report.

---

### Likelihood Explanation

The contract is `PausableUpgradeable` and the owner can pause at any time via `pause()`. [5](#0-4) 

Pauses are a normal operational event (e.g., during upgrades or incident response). Any external protocol that integrates `getClaimableAmount` as a live oracle for claimable collateral will be exposed during every pause window. The likelihood is **medium** given that pauses are a routine admin action and the function is explicitly public for external consumption.

---

### Recommendation

Add a paused-state check inside `getClaimableAmount` so it returns zero when claims are blocked:

```solidity
function getClaimableAmount(address user, uint256 amount) external view returns (uint256) {
    if (paused()) return 0;
    return _getUnclaimedVestedAmount(user, amount);
}
```

This mirrors the Sablier fix: the view function should reflect the actual redeemable amount under current on-chain conditions, not just the mathematical vesting calculation.

---

### Proof of Concept

1. Owner calls `pause()` on `KernelTop100MerkleDistributor`.
2. An external lending protocol calls `getClaimableAmount(alice, 1000e18)` — it returns, say, `400e18` (40% vested).
3. The lending protocol extends credit to Alice against `400e18` KERNEL.
4. Alice (or the lending protocol) calls `claim(1000e18, proof)` — it reverts with `Pausable: paused`.
5. The `400e18` KERNEL cannot be retrieved while paused; the lending protocol's collateral assumption is incorrect.
6. Owner unpauses eventually, but during the pause window the external protocol held a position based on a false claimable amount. [1](#0-0) [3](#0-2) [4](#0-3)

    function _getUnclaimedVestedAmount(address user, uint256 userTotalClaimableAmount) internal view returns (uint256) {
        UserClaim storage userClaim = userClaims[user];

        // If user has claimed everything, return 0
        if (userClaim.amountClaimed >= userTotalClaimableAmount) {
            return 0;
        }

        // Calculate vesting end time
        uint256 vestingEndTime = vestingStartTimestamp + VESTING_DURATION;

        // Calculate start and end times for the period
        uint256 startTime = userClaim.lastClaimTimestamp > 0 ? userClaim.lastClaimTimestamp : vestingStartTimestamp;

        // Cap current time at vesting end time
        uint256 currentTime = block.timestamp;
        if (currentTime > vestingEndTime) {
            currentTime = vestingEndTime;
        }

        // If current time is before start time or vesting hasn't started yet, nothing to claim
        if (currentTime <= startTime || currentTime <= vestingStartTimestamp) {
            return 0;
        }

        // Calculate total vested amount based on time elapsed since vesting start
        uint256 totalElapsedTime = currentTime - vestingStartTimestamp;
        uint256 totalVestedAmount = (userTotalClaimableAmount * totalElapsedTime) / VESTING_DURATION;

        // Cap at total amount
        if (totalVestedAmount > userTotalClaimableAmount) {
            totalVestedAmount = userTotalClaimableAmount;
        }

        // Calculate unclaimed amount
        uint256 unclaimedAmount = totalVestedAmount - userClaim.amountClaimed;

        return unclaimedAmount;
    }
```
