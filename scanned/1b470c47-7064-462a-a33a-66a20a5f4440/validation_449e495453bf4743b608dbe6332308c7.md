### Title
Off-by-One in `setFeeBps` Guard Allows 100% Fee, Permanently Locking Depositor ETH as Fees with Zero wrsETH Minted - (File: `contracts/pools/RSETHPoolV2NBA.sol`)

### Summary
`RSETHPoolV2NBA.setFeeBps` uses a strict `>` comparison instead of `>=`, allowing `feeBps = 10_000` (100%) to be set. When this value is active, any ETH deposited via `deposit()` is entirely consumed as fee, `rsETHAmount` computes to zero, `wrsETH.mint(msg.sender, 0)` is called without revert, and the depositor's ETH is permanently recorded in `feeEarnedInETH` with no user-facing recovery path.

### Finding Description

**Root cause — off-by-one in the fee guard:** [1](#0-0) 

```solidity
function setFeeBps(uint256 _feeBps) external onlyRole(DEFAULT_ADMIN_ROLE) {
    if (_feeBps > 10_000) revert InvalidFeeAmount();   // ← allows 10_000 (100%)
    feeBps = _feeBps;
    emit FeeBpsSet(_feeBps);
}
```

The guard `_feeBps > 10_000` does **not** revert when `_feeBps == 10_000`. A value of 10 000 basis points equals 100%, which is a semantically invalid fee for a swap pool.

**Fee math when `feeBps = 10_000`:** [2](#0-1) 

```
fee          = amount * 10_000 / 10_000  = amount
amountAfterFee = amount - fee            = 0
rsETHAmount  = 0 * 1e18 / rate          = 0
```

**Deposit path — no zero-amount guard on `rsETHAmount`:** [3](#0-2) 

```solidity
function deposit(string memory referralId) external payable nonReentrant whenNotPaused {
    uint256 amount = msg.value;
    if (amount == 0) revert InvalidAmount();          // only guards msg.value, not rsETHAmount
    (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
    feeEarnedInETH += fee;                            // += full ETH amount
    wrsETH.mint(msg.sender, rsETHAmount);             // mint(msg.sender, 0) — succeeds silently
    emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
}
```

The only zero-check is on `msg.value`, not on the computed `rsETHAmount`. `wrsETH.mint(address, 0)` is a no-op that does not revert. The depositor's ETH is fully absorbed into `feeEarnedInETH` with no wrsETH issued.

**No user recovery path:** `withdrawFees` is restricted to `BRIDGER_ROLE` and sends ETH to an arbitrary `receiver`, not back to depositors. [4](#0-3) 

### Impact Explanation

Every deposit made while `feeBps == 10_000` results in:
- Depositor loses 100% of their ETH.
- Depositor receives 0 wrsETH.
- ETH is locked in `feeEarnedInETH` and can only be recovered by a `BRIDGER_ROLE` actor sending it to an arbitrary address — there is no on-chain mechanism to refund individual depositors.

This matches **Medium — Temporary freezing of funds**: funds are locked until an admin manually intervenes and compensates affected users off-chain.

### Likelihood Explanation

`setFeeBps` is callable by `DEFAULT_ADMIN_ROLE` without a timelock in this contract (unlike `RSETHPoolV2.sol` and `RSETHPoolNoWrapper.sol`, which gate `setFeeBps` behind `TIMELOCK_ROLE`). [5](#0-4) 

The off-by-one means the contract's own validation presents `10_000` as a valid input. An admin performing routine fee configuration (e.g., setting a temporary 100% fee to halt deposits as an emergency measure) would not be warned by the contract. Any deposit landing in the same block or before the admin corrects the value is silently drained.

### Recommendation

Change the guard from strict-greater-than to greater-than-or-equal:

```solidity
// Before
if (_feeBps > 10_000) revert InvalidFeeAmount();

// After
if (_feeBps >= 10_000) revert InvalidFeeAmount();
```

Additionally, add a zero-amount guard in `deposit`:

```solidity
(uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
if (rsETHAmount == 0) revert InvalidAmount();
```

### Proof of Concept

```solidity
// 1. Admin sets fee to 10_000 (passes the > guard, does not revert)
pool.setFeeBps(10_000);

// 2. User deposits 1 ether
pool.deposit{value: 1 ether}("ref");

// 3. Assertions
assert(wrsETH.balanceOf(user) == 0);          // user received nothing
assert(pool.feeEarnedInETH() == 1 ether);     // full ETH locked as fees

// 4. Boundary check
// setFeeBps(10_001) reverts with InvalidFeeAmount
// setFeeBps(10_000) succeeds — confirms off-by-one
```

```text
    function viewSwapRsETHAmountAndFee(uint256 amount) public view returns (uint256 rsETHAmount, uint256 fee) {
        fee = amount * feeBps / 10_000;
        uint256 amountAfterFee = amount - fee;

        // rate of rsETH in ETH
        uint256 rsETHToETHrate = getRate();

        // Calculate the final rsETH amount
        rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
    }
```
