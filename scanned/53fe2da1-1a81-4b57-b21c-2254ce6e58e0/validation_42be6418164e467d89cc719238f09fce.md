### Title
Nested Unbounded Loops with External Calls in `LRTOracle.updateRSETHPrice()` Can Permanently Brick Price Updates - (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle.updateRSETHPrice()` is a public function that internally executes a deeply nested chain of loops, each making external calls. As the protocol grows (more supported assets, more NodeDelegators, more pending EigenLayer withdrawals), the cumulative gas cost can exceed the block gas limit, permanently preventing any rsETH price update.

### Finding Description
`updateRSETHPrice()` calls `_getTotalEthInProtocol()`, which iterates over every entry in `supportedAssetList`. For each asset it calls `ILRTDepositPool.getTotalAssetDeposits(asset)`, which calls `getAssetDistributionData(asset)`, which in turn loops over every address in `nodeDelegatorQueue` and calls three external functions per NDC:

```
IERC20(asset).balanceOf(nodeDelegatorQueue[i])          // external call
INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset)   // external call
INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset) // external call
```

`getAssetUnstaking()` itself contains a **nested double loop** over all queued EigenLayer withdrawals and their strategies, calling `strategy.underlyingToken()` and `strategy.sharesToUnderlyingView()` per iteration.

The total external-call count scales as:

```
O(supportedAssets × nodeDelegators × queuedWithdrawals × strategiesPerWithdrawal)
```

The developers themselves acknowledge this concern in `LRTUnstakingVault.setMaxUncompletedWithdrawalCount()`:

> "120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price"

This comment confirms the gas ceiling is a known constraint, but the limit is enforced only on `uncompletedWithdrawalCount` — not on the product of all three dimensions simultaneously. With `maxNodeDelegatorLimit` raiseable by admin and `supportedAssetList` growable via `TIME_LOCK_ROLE`, the combined gas cost can still exceed the block gas limit under normal protocol expansion.

### Impact Explanation
If `updateRSETHPrice()` becomes uncallable:
- The stored `rsETHPrice` is permanently stale.
- All new deposits via `depositETH` / `depositAsset` use the stale price to mint rsETH, breaking the exchange rate invariant.
- Protocol fee minting (`_checkAndUpdateDailyFeeMintLimit` + `IRSETH.mint`) is permanently blocked — yield theft via fee suppression.
- The price-deviation circuit-breaker (auto-pause on large price drops) stops functioning.

**Impact: Medium — unbounded gas consumption leading to permanent freezing of the price-update mechanism and permanent freezing of unclaimed yield (protocol fees).**

### Likelihood Explanation
The protocol is actively expanding: new LSTs are added as supported assets, new NodeDelegators are deployed, and EigenLayer withdrawal queues fill up during normal operations. The developers' own comment caps `maxUncompletedWithdrawalCount` at 80 specifically to avoid this, but the cap applies per-NDC-count and per-asset-count independently, not to their product. As the protocol scales to more assets and more NDCs, the effective limit shrinks. No attacker action is required — normal protocol growth triggers the condition.

### Recommendation
1. **Decouple price computation from on-chain enumeration.** Store per-asset TVL snapshots updated incrementally (on deposit/withdrawal events) rather than recomputing the full sum on every `updateRSETHPrice()` call.
2. **Bound the loop product explicitly.** Enforce a hard cap on `supportedAssets.length × nodeDelegatorQueue.length × maxUncompletedWithdrawalCount` and revert `addNewSupportedAsset` / `updateMaxNodeDelegatorLimit` / `setMaxUncompletedWithdrawalCount` if the product would exceed a safe gas budget.
3. **Allow partial iteration.** Accept a `startIndex` / `endIndex` parameter in `_getTotalEthInProtocol` so the computation can be split across multiple transactions and aggregated off-chain.

### Proof of Concept

Call path from any EOA:

```
// Step 1: anyone calls the public function
LRTOracle.updateRSETHPrice()
  └─ _getTotalEthInProtocol()                          // LRTOracle.sol:336
       └─ for each asset in supportedAssetList:        // LRTOracle.sol:336-348
            getAssetPrice(asset)                       // external oracle call
            ILRTDepositPool.getTotalAssetDeposits(asset)
              └─ getAssetDistributionData(asset)       // LRTDepositPool.sol:447-456
                   └─ for each NDC in nodeDelegatorQueue:
                        IERC20(asset).balanceOf(ndc)   // external ERC20 call
                        INodeDelegator(ndc).getAssetBalance(asset)   // external call
                        INodeDelegator(ndc).getAssetUnstaking(asset) // external call
                          └─ for each queuedWithdrawal:              // NodeDelegator.sol:409-427
                               for each strategy in withdrawal:
                                 strategy.underlyingToken()          // external call
                                 strategy.sharesToUnderlyingView()   // external call
```

With `N` assets, `D` NDCs, and `W` queued withdrawals per NDC, the number of external calls is `O(N × D × W)`. At `N=5`, `D=10`, `W=80` (all within documented limits), this is 4,000 external calls — well above what fits in a single block at ~30M gas. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
