### Title
Unprivileged Caller Can Trigger Circuit-Breaker Pause via Spot Oracle Dip — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a public function with no access control. Its internal circuit-breaker compares a freshly computed spot-price against the all-time-high `highestRsethPrice` and, if the drop exceeds `pricePercentageLimit`, immediately pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. Because `SwETHPriceOracle` forwards the live `ISwETH.getRate()` with no smoothing, any transient dip in that rate — however brief — is sufficient for an unprivileged actor to freeze all deposits and withdrawals until an admin manually unpauses.

---

### Finding Description

**Entrypoint — no access control on `updateRSETHPrice()`**

`LRTOracle.updateRSETHPrice()` is declared `public` with only the `whenNotPaused` modifier: [1](#0-0) 

Any EOA or contract can call it at any time.

**Spot oracle — no TWAP, no smoothing**

`SwETHPriceOracle.getAssetPrice()` returns the raw, instantaneous `ISwETH.getRate()`: [2](#0-1) 

This value is consumed directly inside `_getTotalEthInProtocol()` → `getAssetPrice()` → `_updateRsETHPrice()` with no time-weighted averaging.

**Circuit-breaker fires on any spot dip below `highestRsethPrice`**

Inside `_updateRsETHPrice()`, the computed `newRsETHPrice` is compared against the stored all-time-high `highestRsethPrice`. If the drop exceeds `pricePercentageLimit`, the protocol is immediately paused: [3](#0-2) 

The pause calls `lrtDepositPool.pause()` and `withdrawalManager.pause()` (both require `PAUSER_ROLE`, which `LRTOracle` must hold for this code path to execute), then sets `LRTOracle.paused = true`.

**No auto-unpause — admin action required**

`LRTOracle.unpause()` is restricted to `onlyLRTAdmin`: [4](#0-3) 

`LRTDepositPool.unpause()` is similarly restricted to `onlyLRTAdmin`: [5](#0-4) 

Once triggered, the freeze persists until an admin responds.

---

### Impact Explanation

All user-facing deposit and withdrawal operations are gated by `whenNotPaused` on `LRTDepositPool` and `LRTWithdrawalManager`. A successful trigger freezes every deposit and withdrawal until admin intervention. This matches the **Medium — Temporary freezing of funds** impact scope.

---

### Likelihood Explanation

The preconditions are:
1. `pricePercentageLimit > 0` (admin-configured, intended to be set in production).
2. swETH TVL is large enough that a transient rate dip moves `newRsETHPrice` past the threshold.
3. The attacker observes a block where `ISwETH.getRate()` is transiently lower than its recent value (e.g., during a swETH rebase window, a rate-update transaction, or any slashing event).

Condition 3 does not require front-running — the attacker simply monitors the swETH rate off-chain and submits `updateRSETHPrice()` in the same block where the dip is visible. No privileged access, no oracle compromise, and no admin collusion is needed.

---

### Recommendation

1. **Restrict `updateRSETHPrice()` to a privileged role** (e.g., `onlyLRTManager` or a keeper role), or add a minimum call interval (e.g., once per epoch).
2. **Use a TWAP or rate-smoothing layer** in `SwETHPriceOracle` instead of forwarding the raw spot `getRate()`.
3. **Decouple the pause trigger from the price-update caller**: require a separate, role-gated `triggerCircuitBreaker()` function so that an unprivileged price update cannot directly pause the protocol.

---

### Proof of Concept

```solidity
// Fork test at a block where swETH.getRate() is transiently below its recent peak.
// Precondition: pricePercentageLimit is set (e.g., 1e16 = 1%) and LRTOracle holds PAUSER_ROLE.

function testGriefPause() external {
    // 1. Record pre-state
    assertFalse(lrtDepositPool.paused());
    assertFalse(withdrawalManager.paused());

    // 2. Unprivileged caller triggers updateRSETHPrice at the dip block
    vm.prank(address(0xdead));
    lrtOracle.updateRSETHPrice(); // no role check — succeeds

    // 3. Circuit-breaker fired: both contracts are now paused
    assertTrue(lrtDepositPool.paused());
    assertTrue(withdrawalManager.paused());
    assertTrue(lrtOracle.paused());

    // 4. Advance one block — rate recovers, but protocol remains paused (no auto-unpause)
    vm.roll(block.number + 1);
    assertTrue(lrtDepositPool.paused()); // still paused, requires admin action

    // 5. Normal user deposit reverts
    vm.expectRevert("Pausable: paused");
    lrtDepositPool.depositETH{value: 1 ether}(0, "");
}
```
