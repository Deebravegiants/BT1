### Title
Nested External Calls in Unbounded Loops in `getAssetDistributionData`/`getETHDistributionData` Render `updateRSETHPrice` Prohibitively Expensive - (File: contracts/LRTDepositPool.sol, contracts/NodeDelegator.sol)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a public function that internally calls `_getTotalEthInProtocol()`, which loops over all supported assets and for each asset calls `LRTDepositPool.getTotalAssetDeposits()`. That function calls `getAssetDistributionData()` / `getETHDistributionData()`, which loop over all NDCs in `nodeDelegatorQueue` and make multiple external calls per iteration — including `INodeDelegator.getAssetUnstaking()`. That function in turn calls EigenLayer's `getQueuedWithdrawals()` and loops over the returned array, making additional external calls per withdrawal entry. This creates a three-level nested loop with external calls at every level, all reachable from a public, permissionless entry point.

---

### Finding Description

**Level 1 — `LRTOracle._getTotalEthInProtocol()`** loops over `supportedAssets`:

```solidity
// contracts/LRTOracle.sol:336-348
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);                          // external call
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr)
        .getTotalAssetDeposits(asset);                               // external call → triggers Level 2
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    unchecked { ++assetIdx; }
}
```

**Level 2 — `LRTDepositPool.getAssetDistributionData()`** loops over `nodeDelegatorQueue`:

```solidity
// contracts/LRTDepositPool.sol:447-456
for (uint256 i; i < ndcsCount;) {
    assetLyingInNDCs += IERC20(asset).balanceOf(nodeDelegatorQueue[i]);   // external call
    assetStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i])
        .getAssetBalance(asset);                                           // external call
    assetUnstakingFromEigenLayer += INodeDelegator(nodeDelegatorQueue[i])
        .getAssetUnstaking(asset);                                         // external call → triggers Level 3
    unchecked { ++i; }
}
```

Similarly, `getETHDistributionData()` loops over `nodeDelegatorQueue` and calls `getEffectivePodShares()` and `getAssetUnstaking(LRTConstants.ETH_TOKEN)` per NDC:

```solidity
// contracts/LRTDepositPool.sol:484-493
for (uint256 i; i < ndcsCount;) {
    ethLyingInNDCs += nodeDelegatorQueue[i].balance;
    ethStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getEffectivePodShares();  // external call
    ethUnstakingFromEigenLayer += INodeDelegator(nodeDelegatorQueue[i])
        .getAssetUnstaking(LRTConstants.ETH_TOKEN);                                          // external call
    unchecked { ++i; }
}
```

**Level 3 — `NodeDelegator.getAssetUnstaking()`** fetches all queued withdrawals from EigenLayer and loops over them:

```solidity

for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {

        address strategyAsset = ...
            : address(strategy.underlyingToken());                 // external call per strategy
        ...
        amount += ... strategy.sharesToUnderlyingView(sharesToUnstake);  // external call per strategy

```

The total gas cost of `updateRSETHPrice()` scales as:

> O(supportedAssets × NDCs × queuedWithdrawals_per_NDC × strategies_per_withdrawal)

All four dimensions grow as the protocol operates normally. `supportedAssets` is admin-controlled but grows over time. `nodeDelegatorQueue` is bounded by `maxNodeDelegatorLimit` (default 10, admin-adjustable). `queuedWithdrawals` per NDC is bounded by `maxUncompletedWithdrawalCount` (admin-adjustable), but EigenLayer's `getQueuedWithdrawals()` returns the full on-chain state for that NDC address.

---

### Impact Explanation

If the cumulative gas cost of `updateRSETHPrice()` grows to approach or exceed the block gas limit:

1. **Protocol fee minting is permanently blocked.** `_updateRsETHPrice()` is the only path that mints protocol fees via `IRSETH.mint()`. If it reverts OOG, fees are never collected — this is theft of unclaimed yield.
2. **rsETH price becomes permanently stale.** The stored `rsETHPrice` is only updated inside `_updateRsETHPrice()`. A stale price causes all subsequent deposits to mint rsETH at an incorrect rate, leading to share/asset mis-accounting.
3. **Downside protection is disabled.** The automatic pause triggered when price drops too far (`isPriceDecreaseOffLimit`) also lives inside `_updateRsETHPrice()`. If the function cannot execute, the circuit-breaker never fires.

Impact: **Medium — Unbounded gas consumption** leading to **High — Theft of unclaimed yield** (protocol fees) and **Medium — Temporary/permanent freezing of protocol price update**.

---

### Likelihood Explanation

`updateRSETHPrice()` is a public, permissionless function (only gated by `whenNotPaused`). No attacker action is needed to trigger the gas growth — it is a natural consequence of protocol operation:

- Operators add NDCs via `addNodeDelegatorContractToQueue()`.
- Operators call `initiateUnstaking()` repeatedly, filling EigenLayer's queued withdrawal list for each NDC.
- Admins add new supported LST assets over time.

As the protocol scales to its designed capacity (e.g., 10 NDCs × 5 assets × 20 queued withdrawals per NDC = 1,000 external calls per `updateRSETHPrice()` invocation), the function becomes prohibitively expensive. This is a realistic operational scenario, not a theoretical edge case.

---

### Recommendation

1. **Cache `getQueuedWithdrawals()` results off-chain and use a push model.** Instead of computing `assetUnstaking` on-the-fly by querying EigenLayer, maintain an on-chain accounting variable that is updated when withdrawals are queued and completed.
2. **Separate price update from TVL computation.** Store `totalAssetDeposits` per asset as a cached value updated lazily, rather than recomputing it from scratch on every `updateRSETHPrice()` call.
3. **Bound the loop depth explicitly.** Enforce a hard cap on `maxUncompletedWithdrawalCount` per NDC at a value that keeps the worst-case gas of `updateRSETHPrice()` well below the block gas limit, and document this constraint.
4. **Split `updateRSETHPrice()` into paginated steps** if full recomputation is required.

---

### Proof of Concept

Call trace for `updateRSETHPrice()` with 3 supported assets, 5 NDCs, and 10 queued withdrawals per NDC (each with 2 strategies):

```
updateRSETHPrice()                                          [public, no auth]
└── _updateRsETHPrice()
    └── _getTotalEthInProtocol()
        └── for asset in supportedAssets (×3):
            ├── getAssetPrice(asset)                        [external call ×3]
            └── ILRTDepositPool.getTotalAssetDeposits(asset) [external call ×3]
                └── getAssetDistributionData(asset)
                    └── for ndc in nodeDelegatorQueue (×5):
                        ├── IERC20.balanceOf(ndc)           [external call ×15]
                        ├── INodeDelegator.getAssetBalance() [external call ×15]
                        │   └── IStrategy.sharesToUnderlyingView() [external call ×15]
                        └── INodeDelegator.getAssetUnstaking() [external call ×15]
                            └── DelegationManager.getQueuedWithdrawals() [external call ×15]
                                └── for withdrawal (×10):
                                    └── for strategy (×2):
                                        ├── strategy.underlyingToken() [external call ×300]
                                        └── strategy.sharesToUnderlyingView() [external call ×300]
```

Total external calls: ~690 per `updateRSETHPrice()` invocation at moderate protocol scale. At full capacity (10 NDCs, 5 assets, 20 withdrawals, 3 strategies each), this exceeds 3,000 external calls, making the function impossible to execute within the block gas limit.     function getAssetUnstaking(address asset) external view returns (uint256 amount) {
```text
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
