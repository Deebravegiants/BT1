### Title
Missing Minimum Output Check in L2 Pool Token Deposit Functions Allows Depositor Token Loss - (File: contracts/pools/RSETHPoolNoWrapper.sol, contracts/pools/RSETHPool.sol)

### Summary
The `deposit(address token, ...)` functions in `RSETHPoolNoWrapper` and `RSETHPool` lack any minimum output validation. When the computed `rsETHAmount` rounds to zero due to integer division, the user's deposited ERC20 tokens are permanently transferred into the pool while the user receives nothing in return. No slippage guard or zero-output revert exists to prevent this outcome.

### Finding Description
In both `RSETHPoolNoWrapper.deposit(address token, uint256 amount, string memory referralId)` and `RSETHPool.deposit(address token, uint256 amount, string memory referralId)`, the flow is:

1. User's tokens are pulled from them via `safeTransferFrom`.
2. The rsETH output is computed via `viewSwapRsETHAmountAndFee(amount, token)`.
3. The computed `rsETHAmount` is transferred (or minted) to the user — even if it is zero.

The output calculation in `RSETHPoolNoWrapper` is:

```solidity
fee = amount * feeBps / 10_000;
uint256 amountAfterFee = amount - fee;
uint256 rsETHToETHrate = getRate();
uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate();
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
```

When `amountAfterFee * tokenToETHRate < rsETHToETHrate`, Solidity integer division truncates `rsETHAmount` to zero. The subsequent `rsETH.safeTransfer(msg.sender, 0)` succeeds silently (ERC20 zero-transfers are valid), and the user's deposited tokens are permanently stranded in the pool with no recovery path.

The same pattern exists in `RSETHPool.deposit(address token, ...)` using `tokenFeeBps[token]` per-token fee, and in `RSETHPoolV3ExternalBridge.deposit(address token, ...)` where `wrsETH.mint(msg.sender, 0)` is called instead.

The analog to the external report is direct: in the report, a minimum-amount check is gated behind `if (amount > 0)` and skipped when the output is zero; here, no such check exists at all, producing the identical outcome — user assets consumed, zero value returned.
