### Title
Zero-Rate Broadcast via Unguarded `updateRate()` Before First Oracle Initialization — (`contracts/cross-chain/MultiChainRateProvider.sol`)

### Summary

`MultiChainRateProvider.updateRate()` has no access control and no zero-rate guard. `LRTOracle.rsETHPrice` defaults to `0` after deployment and is only set by `_updateRsETHPrice()`. Any caller can invoke `updateRate()` before the first `updateRSETHPrice()` call, broadcasting a zero rate via LayerZero to all registered receivers, which causes division-by-zero reverts in downstream pool deposits.

---

### Finding Description

**Entrypoint — no access control:**

`MultiChainRateProvider.updateRate()` is `external payable nonReentrant` with no role check: [1](#0-0) 

**Rate source — defaults to zero:**

`RSETHMultiChainRateProvider.getLatestRate()` reads directly from `ILRTOracle.rsETHPrice()`: [2](#0-1) 

`rsETHPrice` is a plain `uint256` storage variable in `LRTOracle`, initialized to `0` by the EVM default: [3](#0-2) 

`LRTOracle.initialize()` does **not** call `_updateRsETHPrice()`, so `rsETHPrice` remains `0` until an explicit `updateRSETHPrice()` call: [4](#0-3) 

**No zero-rate guard before broadcast:**

`updateRate()` encodes and sends whatever `getLatestRate()` returns — including `0` — with no validation: [5](#0-4) 

**Downstream impact — division by zero:**

On destination chains, `RSETHPoolV2.viewSwapRsETHAmountAndFee()` divides by the received rate: [6](#0-5) 

`RSETHPoolV3` has the identical pattern: [7](#0-6) 

When `rsETHToETHrate == 0`, the division `amountAfterFee * 1e18 / rsETHToETHrate` reverts with a panic, making all deposits on destination chains non-functional until a valid rate update is relayed.

---

### Impact Explanation

The cross-chain rate invariant — that receivers always hold a valid non-zero rsETH/ETH exchange rate — is broken. All `RSETHPoolV2`/`RSETHPoolV3` contracts on destination chains that call `getRate()` on the receiver receive `0`, causing every `deposit()` call to revert via division by zero. No funds are lost; the state is recoverable by sending a valid rate update. This matches **Low: Contract fails to deliver promised returns, but doesn't lose value**.

---

### Likelihood Explanation

The vulnerable window is the deployment phase: after `LRTOracle` and `RSETHMultiChainRateProvider` are deployed but before the first `updateRSETHPrice()` is called. An attacker can monitor the mempool and front-run the first oracle update, or simply call `updateRate()` immediately after the provider is deployed. The attack requires only ETH for LayerZero fees and is permissionless.

---

### Recommendation

Add a zero-rate guard at the top of `updateRate()`:

```solidity
function updateRate() external payable nonReentrant {
    uint256 latestRate = getLatestRate();
    require(latestRate != 0, "MultiChainRateProvider: zero rate");
    // ... rest of function
}
```

Alternatively, restrict `updateRate()` to an authorized keeper role, or ensure `updateRSETHPrice()` is called atomically during deployment before any receivers are added.

---

### Proof of Concept

```solidity
// Local fork test (no mainnet)
function testZeroRateBroadcast() public {
    // 1. Deploy mock LRTOracle whose rsETHPrice() returns 0 (default state)
    MockLRTOracle oracle = new MockLRTOracle(); // rsETHPrice = 0

    // 2. Deploy RSETHMultiChainRateProvider pointing to mock oracle
    RSETHMultiChainRateProvider provider = new RSETHMultiChainRateProvider(
        address(oracle), address(mockLZEndpoint)
    );

    // 3. Add one receiver
    provider.addRateReceiver(101, address(mockReceiver));

    // 4. Unprivileged caller invokes updateRate() with ETH for LZ fees
    vm.prank(attacker);
    provider.updateRate{value: 0.01 ether}();

    // 5. Assert: provider.rate == 0
    assertEq(provider.rate(), 0);

    // 6. Assert: LZ send payload decodes to 0
    bytes memory payload = mockLZEndpoint.lastPayload();
    uint256 sentRate = abi.decode(payload, (uint256));
    assertEq(sentRate, 0);

    // 7. Assert: RSETHPoolV2 deposit reverts (division by zero)
    // Simulate receiver storing rate = 0
    mockReceiver.setRate(0);
    vm.expectRevert(); // division by zero panic
    pool.deposit{value: 1 ether}("ref");
}
```
