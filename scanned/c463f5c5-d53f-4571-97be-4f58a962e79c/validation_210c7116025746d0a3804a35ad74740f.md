### Title
Daily Mint Limit Exhausted at Near-Zero Cost Blocks Other Users' Deposits — (`contracts/pools/RSETHPoolV3.sol`)

---

### Summary

`RSETHPoolV3` enforces a `dailyMintLimit` on rsETH minting. An attacker can deposit ETH up to the full daily limit, receive `wrsETH`, immediately sell `wrsETH` on an L2 DEX to recover ETH, and repeat the next day. Because `wrsETH` is a yield-bearing token that appreciates over time, the net cost to the attacker is only the `feeBps` deposit fee — which can be as low as 0. This exhausts the daily limit every day, permanently preventing other users from depositing.

---

### Finding Description

`RSETHPoolV3.deposit()` applies the `limitDailyMint` modifier, which increments `dailyMintAmount` by the rsETH amount to be minted: [1](#0-0) 

Once `dailyMintAmount >= dailyMintLimit`, all further deposits revert with `DailyMintLimitExceeded()`: [2](#0-1) 

There is no mechanism to decrement `dailyMintAmount` when `wrsETH` is transferred or sold. The only function that returns ETH to a caller in exchange for rsETH is `swapAssetToPremintedRsETH`, but it is gated behind `OPERATOR_ROLE` and is therefore inaccessible to unprivileged users: [3](#0-2) 

However, `wrsETH` is a standard ERC-20 token that trades on L2 DEXes. An attacker can:

1. Call `deposit{value: X}()` — receives `wrsETH`, `dailyMintAmount` increases by the minted amount.
2. Sell `wrsETH` on a DEX for ETH — recovers capital. `dailyMintAmount` is **not** decremented.
3. Wait for the next day (`getCurrentDay() > lastMintDay` resets `dailyMintAmount = 0`).
4. Repeat.

The deposit fee is bounded by `feeBps ≤ 1000` (10%), and can be set to 0: [4](#0-3) 

Since `wrsETH` accrues restaking yield, the attacker's net cost per cycle is `feeBps × deposit_amount − yield_earned_while_holding_wrsETH`, which approaches zero or turns profitable at low fee settings.

---

### Impact Explanation

Every day, the attacker exhausts `dailyMintLimit` before legitimate users can deposit. All honest `deposit()` calls revert with `DailyMintLimitExceeded()` for the remainder of that day. This constitutes a **temporary freezing of funds** — users cannot convert ETH to `wrsETH` through the protocol's intended path. Repeated daily, it becomes a sustained denial-of-service on the deposit function.

**Severity: Medium** — Temporary freezing of funds (deposit functionality blocked per day).

---

### Likelihood Explanation

- The attacker needs capital equal to `dailyMintLimit` in ETH, which is a configurable protocol parameter.
- L2 DEX liquidity for `wrsETH/ETH` exists on chains where `RSETHPoolV3` is deployed (e.g., Base, Optimism).
- No special role or privileged access is required — only a standard `deposit()` call.
- The attack is profitable when `feeBps` is low or zero, and `wrsETH` yield offsets the fee.

**Likelihood: Medium** — Requires capital and DEX liquidity, but no protocol access beyond a standard deposit.

---

### Recommendation

1. **Reduce `dailyMintAmount` on reverse swap**: If `swapAssetToPremintedRsETH` is ever made user-accessible, decrement `dailyMintAmount` accordingly.
2. **Per-address deposit cap**: Limit how much of the daily quota a single address can consume in one day.
3. **Non-zero minimum fee**: Enforce `feeBps >= MIN_FEE` (e.g., 5–10 bps) so cycling always has a real cost that exceeds `wrsETH` yield.
4. **Cooldown per depositor**: Introduce a per-address cooldown between deposits to raise the cost of rapid cycling.

---

### Proof of Concept

```
Preconditions:
  - dailyMintLimit = 1000 rsETH
  - feeBps = 0
  - wrsETH/ETH DEX pool exists on the L2

Attack (repeated daily):
  1. attacker calls RSETHPoolV3.deposit{value: X_ETH}("")
     → wrsETH minted to attacker
     → dailyMintAmount = dailyMintLimit (exhausted)

  2. attacker sells wrsETH on DEX for ~X_ETH
     → dailyMintAmount unchanged (still exhausted)

  3. Any honest user calls deposit{value: Y}("")
     → reverts: DailyMintLimitExceeded()

  4. Next day: attacker repeats from step 1
     → dailyMintAmount reset to 0, immediately re-exhausted
```

The attacker's only cost is the DEX swap fee and any `feeBps` charged, both of which can be near-zero. Legitimate users are blocked from depositing every day at negligible attacker cost. [5](#0-4) [6](#0-5)

    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
