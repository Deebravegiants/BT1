### Title
User-Controlled Asset Parameter in `_withdraw` Enables Cross-Token Fund Theft When Multiple Tokens Are Allowed - (File: contracts/L2/RsETHTokenWrapper.sol)

---

### Summary
`RsETHTokenWrapper._withdraw` validates only that the caller-supplied `_asset` is in the `allowedTokens` mapping, but never enforces that the token being withdrawn matches the token the caller originally deposited. Because the contract explicitly supports multiple allowed tokens (via `addAllowedToken` and `reinitialize`), any holder of `wrsETH` can burn their shares and receive whichever allowed token they choose â€” including a more valuable one they never deposited.

---

### Finding Description

`_withdraw` in `RsETHTokenWrapper` is:

```solidity
function _withdraw(address _asset, address _to, uint256 _amount) internal {
    if (!allowedTokens[_asset]) revert TokenNotAllowed();   // only check
    _burn(msg.sender, _amount);
    ERC20Upgradeable(_asset).safeTransfer(_to, _amount);
    emit Withdraw(_asset, msg.sender, _to, _amount);
}
``` [1](#0-0) 

The sole guard is `allowedTokens[_asset]`. There is no check that the contract's balance of `_asset` was contributed by this caller, nor that `_asset` is the same token the caller deposited. The contract is explicitly designed to hold multiple allowed tokens simultaneously:

- `initialize` adds the first token via `_addAllowedToken`.
- `reinitialize` (callable by `DEFAULT_ADMIN_ROLE`) adds a second token.
- `addAllowedToken` (callable by `TIMELOCK_ROLE`) adds further tokens at any time. [2](#0-1) [3](#0-2) 

Once two tokens are allowed, the `wrsETH` supply represents the aggregate collateral of all allowed tokens. A caller who deposited the cheaper token can call `withdraw(expensiveToken, amount)` and drain the more valuable token from the contract.

The public entry points are unrestricted:

```solidity
function withdraw(address asset, uint256 _amount) external {
    _withdraw(asset, msg.sender, _amount);
}
function withdrawTo(address asset, address _to, uint256 _amount) external {
    _withdraw(asset, _to, _amount);
}
``` [4](#0-3) 

---

### Impact Explanation

**Critical â€” direct theft of user funds.**

When two allowed tokens trade at different prices (e.g., due to a depeg, bridge liquidity imbalance, or market conditions), an attacker can:

1. Deposit `N` units of the cheaper alt-rsETH token â†’ receive `N` wrsETH.
2. Call `withdraw(expensiveAltRsETH, N)` â†’ burn `N` wrsETH, receive `N` units of the more expensive token.

The attacker profits the price difference per unit, and the remaining depositors are left holding only the cheaper token. Repeated at scale this drains the entire balance of the more valuable token from the contract.

---

### Likelihood Explanation

**Medium.**

- Requires at least two tokens to be added to `allowedTokens`. This is a supported and documented feature of the contract (`reinitialize`, `addAllowedToken`).
- Requires a price discrepancy between the two allowed tokens. Alt-rsETH tokens on different L2 chains routinely trade at small but non-zero spreads relative to each other due to bridge latency and liquidity differences.
- No special role or permission is needed to execute the attack once two tokens are allowed â€” any `wrsETH` holder can do it.

---

### Recommendation

Track which token each unit of `wrsETH` was minted against, or restrict withdrawal to a single canonical token per wrsETH issuance. Alternatively, enforce that the contract's per-token balance never falls below the amount attributable to that token's depositors by maintaining per-token accounting (e.g., `mapping(address => uint256) public tokenReserves`) and decrementing it on withdrawal, preventing cross-token substitution.

---

### Proof of Concept

**Setup**: Two alt-rsETH tokens are allowed â€” `tokenA` (trading at 0.99 rsETH) and `tokenB` (trading at 1.00 rsETH). The contract holds 1000 `tokenB` deposited by the bridger.

**Attack**:
```
1. Attacker acquires 1000 tokenA (cost: ~990 USD equivalent).
2. Attacker calls deposit(tokenA, 1000) â†’ receives 1000 wrsETH.
3. Attacker calls withdraw(tokenB, 1000) â†’ burns 1000 wrsETH, receives 1000 tokenB (worth ~1000 USD).
4. Net profit: ~10 USD per 1000 units. Scales linearly.
```

The only revert condition in `_withdraw` is `!allowedTokens[tokenB]`, which is `false` (tokenB is allowed), so the call succeeds. [1](#0-0) [5](#0-4)
