### Title
Dead Fee-Split State: Treasury Never Receives Configured Fee Share - (File: `contracts/FeeReceiver.sol`)

### Summary

`FeeReceiver.initialize()` accepts, validates, and stores a treasury address and fee BPS, but `sendFunds()` unconditionally forwards 100% of the contract's ETH balance to `depositPool`, never reading those stored values.

### Finding Description

`initialize()` enforces non-zero values for `_protocolTreasury` and `_protocolFeePercentInBPS`: [1](#0-0) 

Both values are stored in state: [2](#0-1) 

`sendFunds()` then sends the entire balance to `depositPool` without referencing either field: [3](#0-2) 

There are no other functions in the contract that read `_legacyProtocolTreasury` or `_legacyProtocolFeePercentInBPS`. The only other manager function, `setDepositPool`, also does not touch them: [4](#0-3) 

The `_legacy` prefix on the variable names suggests these fields were intentionally deprecated from an earlier version, but `initialize()` still validates and requires non-zero values for them, creating a misleading contract interface that promises a fee split it never delivers.

### Impact Explanation

Every call to `sendFunds()` sends 100% of MEV/execution-layer rewards to `depositPool`. The treasury address configured at initialization receives 0 ETH, permanently. The configured fee percentage is never applied. This matches the scoped impact: **Low â€” contract fails to deliver promised returns, but doesn't lose value** (funds reach the deposit pool rather than being lost).

### Likelihood Explanation

This is unconditional â€” no special attacker action is required. Every invocation of `sendFunds()` reproduces the broken behavior. The treasury will never receive its share for the lifetime of this contract deployment unless the contract is upgraded.

### Recommendation

Either:
1. Remove the `_protocolTreasury` and `_protocolFeePercentInBPS` parameters from `initialize()` entirely (and the corresponding storage variables) if the fee split is intentionally deprecated, or
2. Implement the fee split in `sendFunds()`:
   ```solidity
   uint256 feeAmount = balance * _legacyProtocolFeePercentInBPS / 10_000;
   (bool ok,) = _legacyProtocolTreasury.call{value: feeAmount}("");
   require(ok);
   ILRTDepositPool(depositPool).receiveFromRewardReceiver{value: balance - feeAmount}();
   ```
