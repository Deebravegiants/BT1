### Title
Broken Price Oracle for Any Supported Asset Permanently Blocks `updateRSETHPrice()`, Freezing Protocol Fee Yield — (File: `contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle._getTotalEthInProtocol()` iterates over every supported asset and calls `getAssetPrice(asset)` for each, with no error handling. If any single external price oracle (Chainlink, Stader's `getExchangeRate()`, Frax's `pricePerShare()`, RocketPool's `getExchangeRate()`) reverts, the entire `updateRSETHPrice()` call reverts. This permanently blocks protocol fee minting and disables the automatic downside-protection pause mechanism until an admin manually replaces the broken oracle.

---

### Finding Description

`_getTotalEthInProtocol()` is a private function called by both the public `updateRSETHPrice()` and the manager-only `updateRSETHPriceAsManager()`: [1](#0-0) 

For every asset in `lrtConfig.getSupportedAssetList()`, it calls:

```solidity
uint256 assetER = getAssetPrice(asset);
```

`getAssetPrice` delegates to the registered external oracle: [2](#0-1) 

Each oracle makes an external call to a third-party protocol:

- `EthXPriceOracle` → `IETHXStakePoolsManager.getExchangeRate()` [3](#0-2) 
- `SfrxETHPriceOracle` → `ISfrxETH.pricePerShare()` [4](#0-3) 
- `RETHPriceOracle` → `IrETH.getExchangeRate()` [5](#0-4) 
- `ChainlinkPriceOracle` → `AggregatorV3Interface.latestRoundData()` [6](#0-5) 

There is no `try/catch`, no fallback, and no per-asset skip logic. A single reverting oracle causes the entire loop to revert.

`updateRSETHPrice()` is the sole mechanism for:
1. Minting protocol fee rsETH to the treasury [7](#0-6) 
2. Triggering the automatic downside-protection pause (auto-pausing `LRTDepositPool` and `LRTWithdrawalManager` when price drops beyond `pricePercentageLimit`) [8](#0-7) 

Even `updateRSETHPriceAsManager()` (manager-only) calls the same `_updateRsETHPrice()` path and is equally blocked. [9](#0-8) 

The admin can call `updatePriceOracleFor` to replace the broken oracle, but cannot set it to `address(0)` for a supported asset (enforced by `UtilLib.checkNonZeroAddress`), meaning a working replacement must be available immediately. [10](#0-9) 

---

### Impact Explanation

**Medium — Permanent freezing of unclaimed yield.**

If any single supported asset's external oracle permanently reverts (e.g., Stader pauses their contract, a Chainlink feed is deprecated without replacement, or Frax's `sfrxETH` contract is upgraded with a breaking change):

- `updateRSETHPrice()` is permanently blocked.
- Protocol fee rsETH minting to the treasury is permanently frozen — all accrued yield that would have been minted as fees is lost until the oracle is replaced.
- The automatic downside-protection pause cannot trigger. If the real rsETH value drops significantly during this window, the protocol cannot auto-pause, and users continue depositing at a stale (inflated) price, worsening insolvency risk.

---

### Likelihood Explanation

**Low-Medium.** The protocol integrates with multiple external protocols (Stader, Frax, RocketPool, Chainlink). Any one of them experiencing a contract pause, upgrade with breaking interface, or deprecation of a price feed is a realistic scenario over the protocol's lifetime. The Chainlink oracle path is particularly relevant — deprecated feeds return stale data or revert. The impact is triggered by a single asset's oracle failure, not all of them simultaneously.

---

### Recommendation

Add per-asset `try/catch` error handling inside `_getTotalEthInProtocol()` so that a single broken oracle does not block the entire price update. For example:

```solidity
try IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset) returns (uint256 price) {
    totalETHInProtocol += totalAssetAmt.mulWad(price);
} catch {
    // emit event, use last known price, or revert with specific asset info
}
```

Alternatively, allow the admin to set a static fallback price per asset that is used when the live oracle reverts, ensuring `updateRSETHPrice()` can always complete.

---

### Proof of Concept

1. Protocol has three supported assets: ETH, stETH, ETHx.
2. Stader upgrades their `StakePoolsManager` contract, breaking the `getExchangeRate()` interface.
3. `EthXPriceOracle.getAssetPrice(ethx)` now reverts on every call.
4. Anyone calls `updateRSETHPrice()`.
5. `_getTotalEthInProtocol()` reaches the ETHx iteration and calls `getAssetPrice(ethx)` → reverts.
6. The entire `updateRSETHPrice()` transaction reverts.
7. Protocol fee minting is blocked indefinitely.
8. If the real rsETH price drops below `highestRsethPrice * (1 - pricePercentageLimit)`, the auto-pause cannot trigger.
9. Admin must deploy a new working `EthXPriceOracle` and call `updatePriceOracleFor` before any recovery is possible. [11](#0-10) [3](#0-2)

```text
    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```
