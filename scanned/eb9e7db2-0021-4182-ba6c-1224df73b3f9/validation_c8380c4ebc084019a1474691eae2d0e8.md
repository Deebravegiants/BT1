### Title
Unbounded Gas Consumption in `updateRSETHPrice()` Due to Nested Iteration Over Supported Assets, NDCs, and EigenLayer Queued Withdrawals - (File: contracts/LRTOracle.sol)

---

### Summary

`updateRSETHPrice()` is a public, permissionless function in `LRTOracle.sol` that internally executes a triple-nested unbounded iteration: over all supported assets, over all Node Delegator Contracts (NDCs), and over all EigenLayer queued withdrawals per NDC. As the protocol scales, this function's gas cost grows without bound and can exceed the block gas limit, permanently freezing the rsETH price oracle.

---

### Finding Description

The public function `updateRSETHPrice()` calls `_updateRsETHPrice()`, which calls `_getTotalEthInProtocol()`. The call chain produces a deeply nested unbounded loop:

**Step 1 — `_getTotalEthInProtocol()` iterates over all supported assets:**


**Step 2 — `getTotalAssetDeposits()` → `getAssetDistributionData()` iterates over all NDCs:**


The total gas cost scales as **O(assets × NDCs × queued\_withdrawals\_per\_NDC)**, all of which are unbounded. The same nested pattern applies to `getETHDistributionData()` for the ETH asset path.

---

### Impact Explanation

If `updateRSETHPrice()` exceeds the block gas limit, the rsETH price oracle is permanently frozen at its last stored value. Consequences include:

- **Stale price used for all deposits**: `_beforeDeposit()` calls `getRsETHAmountToMint()` which reads `lrtOracle.rsETHPrice()` (the stored stale value), causing depositors to receive incorrect rsETH amounts.
- **Withdrawal processing at incorrect rates**: `_unlockWithdrawalRequests()` in `LRTWithdrawalManager` uses the rsETH price fetched from the oracle; a stale price causes incorrect payout calculations.
- **Price protection mechanisms disabled**: The downside protection logic in `_updateRsETHPrice()` (pause on price drop) becomes permanently non-functional.
- **Protocol fee minting halted**: Fee accrual via rsETH minting stops.

**Impact: Medium — Unbounded gas consumption leading to temporary/permanent freezing of the price update mechanism.**

---

### Likelihood Explanation

The EigenLayer queued withdrawal count per NDC is entirely unbounded and grows organically as users initiate unstaking. During periods of high withdrawal demand, each NDC can accumulate dozens to hundreds of pending EigenLayer withdrawals. Combined with multiple supported assets and up to `maxNodeDelegatorLimit` NDCs (initialized to 10, admin-adjustable upward), the gas cost of `updateRSETHPrice()` can realistically exceed Ethereum's ~30M block gas limit. No privileged action is required — normal protocol usage (deposits followed by unstaking) drives the growth. Any unprivileged caller (or automated keeper) invoking `updateRSETHPrice()` will trigger the OOG revert once the threshold is crossed.

---

### Recommendation

1. **Cap the number of EigenLayer queued withdrawals per NDC** that `getAssetUnstaking()` iterates over, or restructure it to use an incremental/cached accounting pattern rather than re-reading all queued withdrawals on every call.
2. **Cache `totalAssetDeposits` per asset** and update it incrementally on deposit/withdrawal events rather than recomputing it from scratch on every price update.
3. **Enforce a hard cap on `maxNodeDelegatorLimit`** and document the maximum safe number of NDCs and supported assets relative to the block gas limit.
4. Consider splitting `updateRSETHPrice()` into a two-phase pattern (accumulate TVL off-chain or in a separate preparatory call, then finalize the price update) to avoid unbounded on-chain computation.

---

### Proof of Concept

**Entry path (unprivileged):**
1. Any address calls `LRTOracle.updateRSETHPrice()`.
2. This calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`.
3. For each of N supported assets, `getTotalAssetDeposits(asset)` is called.
4. For each of M NDCs, `getAssetUnstaking(asset)` is called on each NDC.
5. Each `getAssetUnstaking()` call fetches all K queued EigenLayer withdrawals via `getQueuedWithdrawals()` and iterates over them.
6. Total external calls: O(N × M × K). With N=3 assets, M=10 NDCs, K=50 queued withdrawals each, this is 1,500 external storage reads plus EigenLayer delegation manager calls — sufficient to approach or exceed the block gas limit. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

```text
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```

```text
        if (asset == LRTConstants.ETH_TOKEN) {
            return (totalAssetDeposits > lrtConfig.depositLimitByAsset(asset));
        }
        return (totalAssetDeposits + amount > lrtConfig.depositLimitByAsset(asset));
    }
```
