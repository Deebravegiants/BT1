### Title
Oracle Substitution to `OneETHPriceOracle` Triggers False Auto-Pause via Stale `highestRsethPrice` — (`contracts/LRTOracle.sol`)

---

### Summary

`highestRsethPrice` is a monotonically increasing watermark that is never reset when asset price oracles are changed. When an admin legitimately assigns `OneETHPriceOracle` (which always returns `1e18`) to one or more assets whose prior oracle reported a higher rate, the computed `newRsETHPrice` drops relative to `highestRsethPrice`. If the apparent drop exceeds `pricePercentageLimit`, the downside-protection logic auto-pauses the deposit pool, withdrawal manager, and oracle — temporarily freezing all user funds — even though no actual loss occurred.

---

### Finding Description

**`OneETHPriceOracle` always returns `1e18`:** [1](#0-0) 

**`highestRsethPrice` is only ever written upward and is never reset on oracle changes:** [2](#0-1) 

**`updatePriceOracleFor` / `updatePriceOracleForValidated` do not touch `highestRsethPrice`:** [3](#0-2) 

**The downside-protection block compares `newRsETHPrice` against the stale `highestRsethPrice` watermark and auto-pauses if the difference exceeds `pricePercentageLimit`:** [4](#0-3) 

**The pause cascades to `lrtDepositPool`, `withdrawalManager`, and `LRTOracle` itself:** [5](#0-4) 

**Concrete arithmetic:**

| Variable | Value |
|---|---|
| `highestRsethPrice` (accumulated yield) | `1.1e18` |
| `pricePercentageLimit` (5%) | `5e16` |
| Threshold = `5e16 * 1.1e18 / 1e18` | `5.5e16` |
| `newRsETHPrice` (all assets at `1e18`) | `1.0e18` |
| `diff` = `1.1e18 − 1.0e18` | `1e17` |
| `diff > threshold`? | **YES** → auto-pause |

---

### Impact Explanation

All three contracts (`LRTDepositPool`, `LRTWithdrawManager`, `LRTOracle`) are paused simultaneously. User deposits and withdrawals are frozen until an admin manually calls `unpause()` on each. The pause is triggered by an oracle configuration change, not by any real loss of funds or attack.

**Impact: Medium — Temporary freezing of funds.**

---

### Likelihood Explanation

`OneETHPriceOracle` is a production contract in scope, explicitly designed to be assigned to assets via `updatePriceOracleFor`. The admin role is not compromised; the admin is performing a routine, supported configuration action. `highestRsethPrice` naturally drifts above `1e18` over time as staking rewards accrue. Any admin oracle rotation after a period of yield accumulation can silently cross the threshold and trigger the pause. The public `updateRSETHPrice()` can be called by anyone immediately after the oracle assignment, making the trigger permissionless once the precondition is set.

---

### Recommendation

Reset or recalculate `highestRsethPrice` inside `updatePriceOracleFor` / `updatePriceOracleForValidated` after an oracle change, or recompute it as the current `rsETHPrice` at the time of the oracle update. Alternatively, add a dedicated admin function to manually reset `highestRsethPrice` that must be called atomically with oracle reassignments.

---

### Proof of Concept

```solidity
// Invariant test (local fork or unit test, no mainnet)
function test_falseAutoPause_oracleSubstitution() public {
    // 1. Simulate accumulated yield: rsETH price has risen to 1.1e18
    //    (achieved by depositing assets whose oracle returns 1.1e18,
    //     then calling updateRSETHPrice() once)
    vm.prank(manager);
    lrtOracle.updateRSETHPriceAsManager();
    // highestRsethPrice is now 1.1e18

    // 2. Admin sets pricePercentageLimit to 5%
    vm.prank(admin);
    lrtOracle.setPricePercentageLimit(5e16);

    // 3. Admin legitimately assigns OneETHPriceOracle to all supported assets
    address oneEthOracle = address(new OneETHPriceOracle());
    address[] memory assets = lrtConfig.getSupportedAssetList();
    for (uint i = 0; i < assets.length; i++) {
        vm.prank(admin);
        lrtOracle.updatePriceOracleFor(assets[i], oneEthOracle);
    }

    // 4. Anyone calls updateRSETHPrice() — permissionless
    lrtOracle.updateRSETHPrice();

    // 5. Assert: protocol is now paused despite no real loss
    assertEq(lrtOracle.paused(), true);
    assertEq(lrtDepositPool.paused(), true);
    assertEq(lrtWithdrawManager.paused(), true);

    // 6. Assert: diff = 0.1e18 > threshold = 5e16 * 1.1e18 / 1e18 = 5.5e16
    uint256 diff = lrtOracle.highestRsethPrice() - 1e18; // 0.1e18
    uint256 threshold = (5e16 * lrtOracle.highestRsethPrice()) / 1e18; // 5.5e16
    assertGt(diff, threshold);
}
```

        // Sanity check: oracle price must have precision between 1e16 and 1e19
        uint256 price = IPriceFetcher(priceOracle).getAssetPrice(asset);

    /// @param asset asset address for which oracle price needs to be added/updated

        if (lrtConfig.isSupportedAsset(asset)) {
            UtilLib.checkNonZeroAddress(priceOracle);

        assetPriceOracle[asset] = priceOracle;
