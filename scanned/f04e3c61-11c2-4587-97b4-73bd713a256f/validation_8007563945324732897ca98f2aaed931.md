### Title
Unbounded Nested Loop Gas Consumption in Publicly Callable `updateRSETHPrice()` - (File: contracts/LRTOracle.sol, contracts/NodeDelegator.sol)

### Summary
`LRTOracle.updateRSETHPrice()` is publicly callable with no access control. It internally executes a chain of nested loops whose combined iteration count is the product of `supportedAssets.length`, `nodeDelegatorQueue.length`, and the number of queued EigenLayer withdrawals per NodeDelegator. As the protocol grows, this multiplicative gas cost can cause the function to revert due to block gas limit exhaustion, temporarily freezing all deposits and price updates.

### Finding Description
`updateRSETHPrice()` carries only a `whenNotPaused` modifier — any external caller can invoke it. [1](#0-0) 

Internally it calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which iterates over every supported asset: [2](#0-1) 

For each asset it calls `ILRTDepositPool.getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)`, which iterates over every NodeDelegator in `nodeDelegatorQueue`: [3](#0-2) 

For each NodeDelegator it calls `getAssetUnstaking(asset)`, which fetches **all** queued EigenLayer withdrawals and iterates over them with a second nested loop: [4](#0-3) 

The total iteration count is therefore:

```
supportedAssets.length
  × nodeDelegatorQueue.length
  × queuedWithdrawals.length (per NDC)
  × withdrawal.strategies.length (per withdrawal)
```

None of these dimensions has a hard cap enforced at the `updateRSETHPrice` call site. `supportedAssets` has no explicit maximum in `LRTConfig`. `maxNodeDelegatorLimit` defaults to 10 but is freely adjustable by admin. The EigenLayer queued-withdrawal count is bounded by `maxUncompletedWithdrawalCount` per NDC, but that value is also admin-settable and applies per-NDC, so the aggregate across all NDCs is unbounded.

The same nested-loop path is also triggered on every user deposit through `_beforeDeposit` → `_checkIfDepositAmountExceedesCurrentLimit` → `getTotalAssetDeposits`: [5](#0-4) 

### Impact Explanation
If the product of the four loop dimensions grows large enough to exceed the block gas limit, every call to `updateRSETHPrice()` reverts. Because `rsETHPrice` is stale, any deposit that checks `getRsETHAmountToMint` will use an outdated price, and the oracle update itself becomes permanently uncallable until the state is reduced (e.g., withdrawals completed). This constitutes a **temporary freezing of funds** for depositors and withdrawers who depend on a live price.

**Impact: Medium — Temporary freezing of funds / unbounded gas consumption.**

### Likelihood Explanation
The protocol is designed to scale: more LSTs are added as supported assets, more NodeDelegators are deployed to handle growing TVL, and EigenLayer withdrawal queues grow as the protocol processes large redemptions. All three dimensions grow monotonically during normal operation. No attacker action is required; the condition is reached through ordinary protocol growth. Likelihood is **Medium**.

### Recommendation
1. **Cap `supportedAssets`**: Enforce a maximum length for `supportedAssetList` in `LRTConfig.addNewSupportedAsset`, analogous to `maxNodeDelegatorLimit`.
2. **Aggregate `getAssetUnstaking` off-chain or cache it**: Replace the per-call EigenLayer `getQueuedWithdrawals` loop with a cached/accumulated counter that is updated incrementally when withdrawals are initiated and completed, similar to how `uncompletedWithdrawalCount` is already tracked.
3. **Access-restrict `updateRSETHPrice`**: Restrict it to `OPERATOR_ROLE` or `MANAGER` so that the gas cost is borne by a trusted party who can size the transaction appropriately.
4. **Bound `maxUncompletedWithdrawalCount` globally**: Enforce a protocol-wide ceiling across all NDCs, not just per-NDC.

### Proof of Concept
Call trace triggered by any unprivileged address:

```
attacker → LRTOracle.updateRSETHPrice()          // public, no role check
  → _updateRsETHPrice()
    → _getTotalEthInProtocol()
      for each asset in supportedAssets:          // loop 1 (no cap)
        → LRTDepositPool.getTotalAssetDeposits(asset)
          → getAssetDistributionData(asset)
            for each ndc in nodeDelegatorQueue:   // loop 2 (maxNodeDelegatorLimit, admin-settable)
              → NodeDelegator.getAssetUnstaking(asset)
                → DelegationManager.getQueuedWithdrawals(ndc)
                for each withdrawal:              // loop 3 (maxUncompletedWithdrawalCount × NDC count)
                  for each strategy:             // loop 4
                    accumulate amount
```

With, e.g., 5 supported assets, 10 NDCs, 20 queued withdrawals per NDC, and 3 strategies per withdrawal, the innermost body executes 5 × 10 × 20 × 3 = **3,000 times** per `updateRSETHPrice()` call, each involving external `staticcall`s to EigenLayer. As these numbers grow with protocol scale, the call will exceed the block gas limit. [1](#0-0) [6](#0-5) [3](#0-2) [7](#0-6)

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
