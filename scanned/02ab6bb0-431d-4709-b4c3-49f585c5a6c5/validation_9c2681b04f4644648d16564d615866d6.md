### Title
Missing Daily Mint Cap in `RSETHPoolNoWrapper.deposit()` Allows Single-Transaction Drain of Pool rsETH Balance — (File: contracts/pools/RSETHPoolNoWrapper.sol)

---

### Summary

`RSETHPoolNoWrapper` is the deposit pool for chains without a wrapper contract (e.g., Unichain). Its `deposit(string)` and `deposit(address,uint256,string)` functions are missing the `limitDailyMint` modifier that every other pool variant enforces. Any unprivileged depositor can drain the entire pre-funded rsETH balance of the contract in a single transaction, temporarily blocking all other depositors until the protocol manually refills the pool.

---

### Finding Description

Every other L2 pool variant enforces a per-day cap on how much rsETH can leave the pool:

- `RSETHPoolV3.deposit()` — `limitDailyMint(msg.value, ETH_IDENTIFIER)` [1](#0-0) 
- `RSETHPoolV3ExternalBridge.deposit()` — `limitDailyMint(msg.value, ETH_IDENTIFIER)` [2](#0-1) 
- `RSETHPoolV3WithNativeChainBridge.deposit()` — `limitDailyMint(msg.value, ETH_IDENTIFIER)` [3](#0-2) 

`RSETHPoolNoWrapper` has **no such modifier** on either deposit path:

```solidity
function deposit(string memory referralId) external payable nonReentrant whenNotPaused {
    if (!isEthDepositEnabled) revert EthDepositDisabled();
    uint256 amount = msg.value;
    if (amount == 0) revert InvalidAmount();
    (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
    feeEarnedInETH += fee;
    rsETH.safeTransfer(msg.sender, rsETHAmount);   // ← no cap
    emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
}
``` [4](#0-3) 

```solidity
function deposit(address token, uint256 amount, string memory referralId)
    external nonReentrant whenNotPaused onlySupportedToken(token)
{
    if (amount == 0) revert InvalidAmount();
    IERC20(token).safeTransferFrom(msg.sender, address(this), amount);
    (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount, token);
    feeEarnedInToken[token] += fee;
    rsETH.safeTransfer(msg.sender, rsETHAmount);   // ← no cap
    emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
}
``` [5](#0-4) 

Unlike the minting pools, `RSETHPoolNoWrapper` holds a finite pre-funded rsETH OFT balance and simply transfers it out. [6](#0-5) 

The `limitDailyMint` modifier in the other contracts resets a per-day counter and reverts if the cumulative rsETH distributed that day would exceed `dailyMintLimit`. [7](#0-6) 

`RSETHPoolNoWrapper` declares none of those state variables (`dailyMintLimit`, `dailyMintAmount`, `lastMintDay`, `startTimestamp`) and has no equivalent guard. [8](#0-7) 

---

### Impact Explanation

**Medium — Temporary freezing of funds.**

A single depositor can call `deposit{value: X}("")` where `X` is large enough to receive the entire rsETH balance of the pool. After that call succeeds, `rsETH.balanceOf(pool) == 0`, and every subsequent `deposit` call by any other user reverts with an ERC-20 insufficient-balance error. The pool is effectively bricked for all depositors until the protocol operator manually refills it with rsETH. The attacker pays fair-value ETH, so no direct theft occurs, but the deposit service is temporarily frozen for all other users.

---

### Likelihood Explanation

**High.** The `deposit` functions are public and payable with no access control beyond `whenNotPaused`. Any EOA or contract can call them with an arbitrarily large ETH value in a single transaction. No special knowledge, flash loan, or multi-step setup is required.

---

### Recommendation

Add the `limitDailyMint` modifier (and its supporting state variables) to both `deposit` overloads in `RSETHPoolNoWrapper`, mirroring the pattern used in `RSETHPoolV3`:

```solidity
modifier limitDailyMint(uint256 amount, address token) {
    // ... same logic as RSETHPoolV3 ...
    _;
}

function deposit(string memory referralId)
    external payable nonReentrant whenNotPaused
    limitDailyMint(msg.value, ETH_IDENTIFIER)   // ← add
{ ... }

function deposit(address token, uint256 amount, string memory referralId)
    external nonReentrant whenNotPaused onlySupportedToken(token)
    limitDailyMint(amount, token)               // ← add
{ ... }
```

---

### Proof of Concept

1. Protocol deploys `RSETHPoolNoWrapper` on Unichain and pre-funds it with 1 000 rsETH (≈ 1 000 ETH at current rate).
2. Attacker calls `deposit{value: 1000 ether}("")`.
3. `viewSwapRsETHAmountAndFee(1000 ether)` returns `rsETHAmount ≈ 1000e18` (minus fee).
4. `rsETH.safeTransfer(attacker, rsETHAmount)` succeeds; pool rsETH balance drops to ~0.
5. All subsequent `deposit` calls by legitimate users revert because `rsETH.balanceOf(pool) < rsETHAmount`.
6. Deposit functionality is frozen until the protocol operator sends more rsETH to the pool.

    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
