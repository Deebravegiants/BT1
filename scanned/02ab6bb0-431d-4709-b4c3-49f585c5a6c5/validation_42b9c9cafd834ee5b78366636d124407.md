### Title
Chainlink Oracle Deprecation Causes Unhandled Revert in `ChainlinkPriceOracle.getAssetPrice()`, Freezing Deposits and Withdrawals - (File: contracts/oracles/ChainlinkPriceOracle.sol)

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `priceFeed.latestRoundData()` with no error handling. When Chainlink deprecates a feed by setting its aggregator to the zero address, `latestRoundData()` reverts with empty data. This propagates through `LRTOracle.getAssetPrice()` and `_getTotalEthInProtocol()`, freezing all protocol functions that depend on asset pricing: deposits, withdrawal initiation, instant withdrawals, and the withdrawal unlock queue.

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` directly calls `priceFeed.latestRoundData()` with no `try/catch` or error handling:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
``` [1](#0-0) 

When Chainlink deprecates a feed (by setting the underlying aggregator to `address(0)`), `latestRoundData()` reverts with **empty revert data** (no error string, no custom error selector). This bare revert propagates upward uncaught through the entire call stack.

`LRTOracle.getAssetPrice()` delegates directly to the price fetcher with no protection: [2](#0-1) 

`LRTOracle._getTotalEthInProtocol()` iterates over **all** supported assets and calls `getAssetPrice()` for each. A single deprecated feed causes the entire loop to revert: [3](#0-2) 

This revert propagates into `_updateRsETHPrice()` → `updateRSETHPrice()`.

The following user-facing functions all call `lrtOracle.getAssetPrice()` and will revert:

- **`LRTDepositPool.getRsETHAmountToMint()`** (called by `depositETH` and `depositAsset`): [4](#0-3) 

- **`LRTWithdrawalManager.getExpectedAssetAmount()`** (called by `initiateWithdrawal` and `instantWithdrawal`): [5](#0-4) 

- **`LRTWithdrawalManager._createUnlockParams()`** (called by `unlockQueue`): [6](#0-5) 

### Impact Explanation

**Medium — Temporary freezing of funds.**

If any Chainlink feed registered in `ChainlinkPriceOracle` is deprecated, the following user operations revert entirely until an admin replaces the oracle:

- `depositETH` / `depositAsset` — users cannot deposit
- `initiateWithdrawal` / `instantWithdrawal` — users cannot initiate withdrawals
- `unlockQueue` — pending withdrawal requests cannot be processed, blocking `completeWithdrawal` for users already in the queue

Users who have already submitted withdrawal requests and are waiting in the queue have their funds temporarily frozen because `unlockQueue` cannot execute. The freeze persists until the admin calls `updatePriceOracleFor()` to replace the deprecated feed.

### Likelihood Explanation

Chainlink has a documented and observed practice of deprecating feeds by setting the aggregator to `address(0)`. This was confirmed on Polygon mainnet (January 2023) and is a normal part of Chainlink's lifecycle management. The LRT-rsETH protocol uses `ChainlinkPriceOracle` as the price source for LST assets (e.g., stETH, ETHx, rETH). Any one of these feeds being deprecated triggers the freeze. No attacker action is required — this is a passive, externally-triggered condition.

### Recommendation

Wrap the `latestRoundData()` call in a `try/catch` in `ChainlinkPriceOracle.getAssetPrice()`. On failure, either revert with a descriptive custom error (so callers can distinguish oracle failure from other errors) or return a sentinel value that upstream logic can handle gracefully. For example:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    try priceFeed.latestRoundData() returns (uint80, int256 price, uint256, uint256, uint80) {
        return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
    } catch {
        revert OracleCallFailed();
    }
}
```

Additionally, `LRTOracle._getTotalEthInProtocol()` could wrap each per-asset `getAssetPrice()` call in a `try/catch` so that a single deprecated feed does not freeze pricing for all other assets.

### Proof of Concept

1. Chainlink deprecates the stETH/ETH feed by setting its aggregator to `address(0)`.
2. Any call to `ChainlinkPriceOracle.getAssetPrice(stETH)` triggers `latestRoundData()` on the deprecated feed, which reverts with empty data.
3. `LRTOracle._getTotalEthInProtocol()` calls `getAssetPrice(stETH)` inside its loop and reverts.
4. `LRTOracle.updateRSETHPrice()` reverts.
5. `LRTDepositPool.depositETH()` → `_beforeDeposit()` → `getRsETHAmountToMint()` → `lrtOracle.getAssetPrice()` reverts. All deposits are blocked.
6. `LRTWithdrawalManager.initiateWithdrawal()` → `getExpectedAssetAmount()` → `lrtOracle.getAssetPrice()` reverts. All withdrawal initiations are blocked.
7. `LRTWithdrawalManager.unlockQueue()` → `_createUnlockParams()` → `lrtOracle.getAssetPrice()` reverts. Existing queued withdrawals cannot be processed; users with pending requests cannot call `completeWithdrawal()`.
