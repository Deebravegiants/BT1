### Title
`NodeDelegator.stake32EthValidated` Can Only Be Executed by LRT Operators Due to Inner `onlyLRTOperator` Check - (File: contracts/NodeDelegator.sol)

### Summary
`stake32EthValidated` is declared `external` with **no access control modifier**, indicating it was designed to be callable by a broader set of callers than `stake32Eth`. However, it internally calls `stake32Eth`, which carries the `onlyLRTOperator` modifier. Because `msg.sender` is checked inside `stake32Eth`, `stake32EthValidated` is silently restricted to LRT Operators only ŌĆö the same restriction as the inner function ŌĆö making the outer function's open signature misleading and its intended broader access unreachable.

### Finding Description
`stake32EthValidated` is the "safe" variant of `stake32Eth`: it first validates the on-chain deposit root matches the caller-supplied `expectedDepositRoot`, then delegates to `stake32Eth`.

```solidity
// contracts/NodeDelegator.sol:186-200
function stake32EthValidated(
    bytes calldata pubkey,
    bytes calldata signature,
    bytes32 depositDataRoot,
    bytes32 expectedDepositRoot
)
    external          // ŌåÉ no access-control modifier
{
    IETHPOSDeposit depositContract = _getEigenPodManager().ethPOS();
    bytes32 actualDepositRoot = depositContract.get_deposit_root();
    if (expectedDepositRoot != actualDepositRoot) {
        revert InvalidDepositRoot(expectedDepositRoot, actualDepositRoot);
    }
    stake32Eth(pubkey, signature, depositDataRoot);   // ŌåÉ calls onlyLRTOperator function
}
```

`stake32Eth` enforces `onlyLRTOperator`:

```solidity
// contracts/NodeDelegator.sol:150-175
function stake32Eth(
    bytes calldata pubkey,
    bytes calldata signature,
    bytes32 depositDataRoot
)
    public
    whenNotPaused
    onlyLRTOperator   // ŌåÉ strict gate on msg.sender
{
    ...
}
```

Because `stake32EthValidated` calls `stake32Eth` as a regular internal call (not via `this.`), `msg.sender` is preserved. The `onlyLRTOperator` check inside `stake32Eth` therefore fires against the original external caller. Any caller who is not an LRT Operator will revert at that inner check, even though `stake32EthValidated` itself has no guard.

This is the exact structural analog to the reported Golom bug: an outer function with broader (or absent) access control calls an inner function with a stricter `msg.sender` check, making the outer function's effective permission identical to the inner function's ŌĆö contrary to its declared interface.

### Impact Explanation
**Low ŌĆö Contract fails to deliver promised returns, but doesn't lose value.**

The `stake32EthValidated` function's open signature promises that any caller can invoke the deposit-root-validated staking path. In practice, every non-operator call reverts inside `stake32Eth`. The validated staking path is therefore inaccessible to any caller who is not already an LRT Operator, defeating the purpose of providing a separate, publicly-callable safe variant. No funds are lost, but the contract does not deliver the functionality its interface implies.

### Likelihood Explanation
Certain. Any non-operator who calls `stake32EthValidated` will always revert at the `onlyLRTOperator` check inside `stake32Eth`. The mismatch is structural and unconditional.

### Recommendation
Make the access control on `stake32EthValidated` explicit and consistent with `stake32Eth`. The simplest fix is to add the same `onlyLRTOperator` modifier to `stake32EthValidated`:

```solidity
function stake32EthValidated(
    bytes calldata pubkey,
    bytes calldata signature,
    bytes32 depositDataRoot,
    bytes32 expectedDepositRoot
)
    external
    onlyLRTOperator   // ŌåÉ add explicit guard
{
    ...
}
```

Alternatively, if the intent is to allow any caller to trigger validated staking, the `onlyLRTOperator` modifier must be removed from `stake32Eth` (or a separate internal helper without the guard must be introduced), and appropriate safety analysis must be performed.

### Proof of Concept
1. Alice (not an LRT Operator) calls `stake32EthValidated(pubkey, sig, dataRoot, expectedRoot)`.
2. The deposit-root check passes (correct `expectedRoot` supplied).
3. `stake32EthValidated` calls `stake32Eth(pubkey, sig, dataRoot)`.
4. Inside `stake32Eth`, the `onlyLRTOperator` modifier evaluates `IAccessControl(lrtConfig).hasRole(OPERATOR_ROLE, msg.sender)` ŌĆö Alice's address ŌĆö which returns `false`.
5. The transaction reverts with `CallerNotLRTConfigOperator`, even though `stake32EthValidated` has no access control of its own. [1](#0-0) [2](#0-1) [3](#0-2)
