### Title
Missing Chainlink Price Feed Staleness Check Enables Inflated rsETH Minting - (File: contracts/oracles/ChainlinkPriceOracle.sol)

### Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all return values except `price`, performing no staleness validation. A stale LST/ETH price is silently accepted and propagated into the rsETH minting calculation, allowing a depositor to receive more rsETH than the real value of their collateral warrants.

### Finding Description
In `contracts/oracles/ChainlinkPriceOracle.sol`, `getAssetPrice()` fetches the Chainlink price as:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

The `updatedAt` timestamp and `answeredInRound` / `roundId` fields are completely ignored. [1](#0-0) 

This contrasts with `contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`, which at minimum checks `answeredInRound < roundID` and `timestamp == 0` before accepting a price. [2](#0-1) 

`LRTOracle.getAssetPrice()` delegates directly to the registered `IPriceFetcher`, which for Chainlink-backed LSTs is `ChainlinkPriceOracle`. [3](#0-2) 

That price is consumed in two critical paths:

1. **Minting path** — `LRTDepositPool.getRsETHAmountToMint()` divides `amount * lrtOracle.getAssetPrice(asset)` by `lrtOracle.rsETHPrice()` to determine how many rsETH tokens to issue. [4](#0-3) 

2. **rsETH price update path** — `LRTOracle._getTotalEthInProtocol()` multiplies each asset's total deposit amount by `getAssetPrice(asset)` to compute the protocol TVL, which then sets `rsETHPrice`. [5](#0-4) 

### Impact Explanation
If a Chainlink LST/ETH feed goes stale (e.g., during network congestion or a Chainlink node outage) and the last reported price is higher than the real current price, an attacker can:

- Deposit the LST at the inflated stale price and receive more rsETH than the real ETH value of their deposit.
- This dilutes all existing rsETH holders, constituting theft of their proportional share of the protocol's TVL.

Conversely, if `updateRSETHPrice()` is called while the stale price is inflated, the computed `rsETHPrice` rises above its true value, allowing existing holders to redeem more underlying assets than they are entitled to.

**Impact: High — Theft of unclaimed yield / dilution of existing rsETH holders.**

### Likelihood Explanation
Chainlink feeds have documented heartbeat intervals (e.g., 1 hour for ETH/stETH). During L1 congestion events or Chainlink infrastructure incidents, feeds can lag significantly. The deposit function is permissionless and callable by any address, so no privileged access is required to exploit the window. The attacker only needs to monitor the on-chain `updatedAt` value and act when it exceeds the heartbeat threshold.

### Recommendation
Add a time-based staleness check in `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

require(answeredInRound >= roundId, "Stale round");
require(updatedAt != 0, "Incomplete round");
require(block.timestamp - updatedAt <= STALENESS_THRESHOLD, "Stale price");
require(price > 0, "Invalid price");
```

`STALENESS_THRESHOLD` should be set per feed based on its documented heartbeat (e.g., 3600 seconds for a 1-hour heartbeat feed).

### Proof of Concept

1. Assume `stETH/ETH` Chainlink feed last updated 2 hours ago at `1.05e18` (real current value: `1.02e18`).
2. Attacker calls `LRTDepositPool.depositAsset(stETH, 100e18)`.
3. `getRsETHAmountToMint` calls `lrtOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns stale `1.05e18`.
4. rsETH minted = `100e18 * 1.05e18 / rsETHPrice` — approximately 2.9% more rsETH than the real collateral value justifies.
5. Attacker immediately redeems the excess rsETH, extracting value from existing depositors.

No admin access, no governance capture, and no external protocol compromise is required — only a permissionless `depositAsset` call during a stale feed window. [6](#0-5) [7](#0-6)
