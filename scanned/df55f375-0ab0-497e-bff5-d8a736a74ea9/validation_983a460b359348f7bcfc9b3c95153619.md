### Title
Missing Non-Negative Validation of Chainlink `int256` Price Before `uint256` Cast Causes Deposit/Withdrawal Freeze - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` reads `int256 price` from Chainlink's `latestRoundData()` and casts it directly to `uint256` without verifying the value is positive. If Chainlink returns a zero or negative price (a known circuit-breaker behavior during extreme market events), the unchecked cast produces a value ≥ 2²⁵⁵, and the subsequent multiplication `uint256(price) * 1e18` overflows and reverts under Solidity 0.8's checked arithmetic. This causes every downstream call that depends on `getAssetPrice` — deposits, withdrawals, and rsETH price updates — to revert, temporarily freezing the affected asset's operations.

---

### Finding Description

In `contracts/oracles/ChainlinkPriceOracle.sol`, the `getAssetPrice` function fetches the Chainlink price and immediately casts the `int256` result to `uint256`:

```solidity
// ChainlinkPriceOracle.sol L52-54
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

There is no guard checking that `price > 0` before the cast. If Chainlink returns `price = -1`, then `uint256(-1) = 2²⁵⁶ − 1`. The multiplication `(2²⁵⁶ − 1) * 1e18` overflows and reverts in Solidity ≥ 0.8. Any non-positive `int256` value produces a `uint256` ≥ 2²⁵⁵, guaranteeing overflow on the `* 1e18` step.

The sister contract in the same repository, `ChainlinkOracleForRSETHPoolCollateral`, explicitly guards against this:

```solidity
// ChainlinkOracleForRSETHPoolCollateral.sol L32
if (ethPrice <= 0) revert InvalidPrice();
```

`ChainlinkPriceOracle` is missing this guard entirely.

---

### Impact Explanation

`LRTOracle.getAssetPrice()` delegates directly to `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)`, which resolves to `ChainlinkPriceOracle.getAssetPrice()` for Chainlink-backed assets. A revert there propagates to:

- **`LRTDepositPool.getRsETHAmountToMint()`** → `lrtOracle.getAssetPrice(asset)` → all `depositAsset()` calls for the affected asset revert.
- **`LRTWithdrawalManager.getExpectedAssetAmount()`** → `lrtOracle.getAssetPrice(asset)` → withdrawal amount calculation reverts, blocking withdrawal unlocking.
- **`LRTOracle._updateRsETHPrice()`** → `_getTotalEthInProtocol()` → rsETH price update reverts.

The result is a **temporary freeze of deposits and withdrawals** for any LST asset whose Chainlink feed returns a non-positive price. This matches the "Temporary freezing of funds" impact category (Medium).

---

### Likelihood Explanation

Chainlink price feeds have historically returned 0 during circuit-breaker events (e.g., the LUNA collapse in May 2022, where several feeds hit their `minAnswer` floor and returned stale or zero values). Chainlink's `AggregatorV3Interface` returns `int256`, and the specification does not guarantee positivity. No oracle operator compromise is required — the condition can arise from extreme market volatility or feed misconfiguration. The likelihood is low-to-medium but the scenario is realistic and precedented.

---

### Recommendation

Add a positivity check immediately after reading the Chainlink price, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

---

### Proof of Concept

1. Chainlink's ETH/stETH (or any supported LST) feed returns `price = -1` due to a circuit-breaker event.
2. `ChainlinkPriceOracle.getAssetPrice(stETH)` executes:
   - `uint256(-1)` = `115792089237316195423570985008687907853269984665640564039457584007913129639935`
   - `115792...935 * 1e18` → arithmetic overflow → **revert**
3. `LRTOracle.getAssetPrice(stETH)` reverts.
4. Any user calling `LRTDepositPool.depositAsset(stETH, ...)` receives a revert — deposits are frozen.
5. `LRTWithdrawalManager` calls to `getExpectedAssetAmount(stETH, ...)` revert — withdrawal unlocking is frozen.
6. `LRTOracle.updateRSETHPrice()` reverts — rsETH price cannot be updated.

All of this persists until the Chainlink feed recovers to a positive value, with no in-protocol mechanism to bypass the frozen state for the affected asset.
