### Title
Unbounded O(assets × NDCs) Complexity in `updateRSETHPrice` Enables Block Stuffing to Deposit at Stale Price — (`contracts/LRTOracle.sol`, `contracts/LRTDepositPool.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a public, permissionless function whose gas cost scales as O(supported assets × NDCs). Because deposits read the cached `rsETHPrice` with no staleness check, an attacker can stuff blocks to prevent price updates and then deposit at a stale, favorable rate.

---

### Finding Description

**Call chain and complexity:**

`updateRSETHPrice()` → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` loops over every supported asset: [1](#0-0) 

For each asset, `getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)` loops over every NDC: [2](#0-1) 

Per NDC per asset, three external calls are made: `balanceOf`, `getAssetBalance` (into EigenLayer), and `getAssetUnstaking`. For ETH, `getEffectivePodShares()` is also called per NDC: [3](#0-2) 

`maxNodeDelegatorLimit` is initialized to 10: [4](#0-3) 

With 10 NDCs and several supported assets, `updateRSETHPrice` can require 100–200+ external calls, making it a high-gas target for block stuffing.

**No staleness check on deposit:**

`getRsETHAmountToMint` reads the stored `rsETHPrice` directly with no freshness guard: [5](#0-4) 

`rsETHPrice` is only updated when `updateRSETHPrice` is successfully included in a block: [6](#0-5) 

---

### Impact Explanation

If `rsETHPrice` is stale (lower than the current fair value), a depositor receives more rsETH than their proportional share of TVL. When the price is eventually updated upward, the attacker's rsETH is redeemable for more ETH than was deposited, diluting existing holders. The scoped impact is **Low. Block stuffing** (the O(assets × NDCs) complexity is the specific protocol property that makes the function a viable block-stuffing target).

---

### Likelihood Explanation

Block stuffing on Ethereum mainnet is expensive (attacker must fill entire blocks at the prevailing base fee). The attack is economically rational only when the expected profit from the stale-price deposit exceeds the cost of stuffing enough blocks to keep `updateRSETHPrice` out. With a large TVL and a meaningful price lag, this threshold can be crossed. The function is permissionless, so no privileged access is required.

---

### Recommendation

1. **Add a staleness guard in the deposit path.** Record `lastPriceUpdateTimestamp` in `LRTOracle` and revert in `getRsETHAmountToMint` (or `_beforeDeposit`) if the price is older than an acceptable window (e.g., 1 hour).
2. **Reduce per-call gas cost.** Cache intermediate TVL values or batch NDC queries to lower the gas cost of `updateRSETHPrice`, making block stuffing less economically viable.
3. **Require a price update before deposit.** Alternatively, call `updateRSETHPrice` atomically inside `depositETH`/`depositAsset` (or require it to have been called in the same block).

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.0;

// Fork-test outline (Foundry, local fork)
// 1. Fork mainnet at block B where rsETHPrice = P_stale (e.g., 1.05e18)
// 2. Simulate TVL growth so fair price P_fair > P_stale (e.g., 1.06e18)
// 3. Attacker stuffs blocks: submit high-gas txs filling each block,
//    keeping updateRSETHPrice() out of the mempool for N blocks.
// 4. Attacker calls depositETH{value: X}(0, "") — minted rsETH = X * 1e18 / P_stale
// 5. Stop stuffing; anyone calls updateRSETHPrice() → rsETHPrice = P_fair
// 6. Attacker initiates withdrawal; rsETH redeems at P_fair
// 7. Assert: ETH received > X  (attacker extracted principal from other holders)

function testBlockStuffingStaleDeposit() public {
    uint256 staledPrice = lrtOracle.rsETHPrice(); // e.g. 1.05e18, not yet updated
    uint256 depositAmt  = 100 ether;

    // rsETH minted uses stale price
    uint256 rsethMinted = depositPool.getRsETHAmountToMint(ETH_TOKEN, depositAmt);
    // rsethMinted = depositAmt * 1e18 / staledPrice  (more than fair share)

    vm.prank(attacker);
    depositPool.depositETH{value: depositAmt}(0, "");

    // Now let price update
    lrtOracle.updateRSETHPrice();
    uint256 updatedPrice = lrtOracle.rsETHPrice(); // e.g. 1.06e18

    // Fair rsETH at updated price
    uint256 fairRseth = depositAmt * 1e18 / updatedPrice;

    assertGt(rsethMinted, fairRseth, "attacker received excess rsETH at stale price");
}
```
