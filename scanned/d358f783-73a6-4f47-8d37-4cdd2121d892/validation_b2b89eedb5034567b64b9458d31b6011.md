### Title
Any Token Holder Can Permanently Block Token Removal from Pool - (`contracts/pools/RSETHPoolV3ExternalBridge.sol`, `contracts/pools/RSETHPool.sol`, `contracts/pools/RSETHPoolNoWrapper.sol`)

---

### Summary

The `removeSupportedToken` function in three pool contracts guards removal with an exact-zero balance check (`balanceOf(address(this)) != 0`). Any token holder can send a dust amount of the token directly to the pool contract, permanently preventing the `TIMELOCK_ROLE` from ever removing that token from the supported list.

---

### Finding Description

In `RSETHPoolV3ExternalBridge.removeSupportedToken`, `RSETHPool.removeSupportedToken`, and `RSETHPoolNoWrapper.removeSupportedToken`, the guard condition is:

```solidity
if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();
```

This checks that the pool holds **exactly zero** of the token before allowing removal. Because any ERC-20 holder can call `transfer(poolAddress, 1)` at any time — including before the removal transaction is mined — any token holder can permanently block the `TIMELOCK_ROLE` from removing a supported token.

The pool contracts are deployed at known, fixed addresses. An attacker only needs to hold 1 wei of the token and transfer it to the pool. The check will then always revert, regardless of how many times the admin retries.

Note that `feeEarnedInToken[token]` is tracked separately and `getTokenBalanceMinusFees` subtracts it, but `removeSupportedToken` uses the raw `balanceOf`, not the net balance. Even a 1-wei donation that is entirely outside the fee accounting permanently blocks removal.

---

### Impact Explanation

The `TIMELOCK_ROLE` loses the ability to remove a supported token from the pool. This matters when:

- A token's oracle becomes stale or is compromised (the oracle address can be updated, but the token cannot be delisted).
- A token is deprecated or has a security vulnerability.
- The protocol needs to wind down support for a token.

While the token can still be deposited and its oracle/bridge can be updated, the inability to delist it means the pool is permanently forced to accept deposits of a token the protocol no longer wants to support. This constitutes a contract failing to deliver its promised administrative functionality.

**Impact**: Low — Contract fails to deliver promised returns (administrative removal), but does not directly lose value on its own.

---

### Likelihood Explanation

Extremely easy to execute. Any holder of 1 wei of any supported token (e.g., wstETH) can call `token.transfer(poolAddress, 1)` at any time. The cost is negligible. The attack is permanent and cannot be undone by the admin without a contract upgrade.

---

### Recommendation

Replace the exact-zero check with a sweep-then-remove pattern, or change the guard to allow removal when the only balance present is dust that was not deposited through the pool's normal flow. For example:

```solidity
// Instead of reverting on non-zero balance, sweep any remaining balance to treasury first
uint256 remaining = IERC20(token).balanceOf(address(this));
if (remaining > 0) {
    IERC20(token).safeTransfer(treasury, remaining);
}
```

Or, analogously to the fix described in the external report, change the guard to check only the **net** balance (i.e., `getTokenBalanceMinusFees(token) != 0`) so that externally donated dust does not block removal.

---

### Proof of Concept

1. Protocol deploys `RSETHPoolV3ExternalBridge` at address `0xPool`.
2. wstETH is added as a supported token via `addSupportedToken`.
3. Attacker holds 1 wei of wstETH and calls `wstETH.transfer(0xPool, 1)`.
4. `TIMELOCK_ROLE` calls `removeSupportedToken(wstETH, index)`.
5. The check `IERC20(wstETH).balanceOf(address(this)) != 0` evaluates to `true` (balance = 1 wei).
6. Transaction reverts with `TokenBalanceNotZero`.
7. The attacker can repeat step 3 after any future attempt, permanently blocking removal. [1](#0-0) [2](#0-1) [3](#0-2)

### Citations

**File:** contracts/pools/RSETHPoolV3ExternalBridge.sol (L769-779)
```text
    function removeSupportedToken(address token, uint256 tokenIndex) external onlyRole(TIMELOCK_ROLE) {
        UtilLib.checkNonZeroAddress(token);
        if (supportedTokenList[tokenIndex] != token) revert TokenNotFoundError();
        if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();

        delete supportedTokenOracle[token];
        delete tokenBridge[token];
        supportedTokenList[tokenIndex] = supportedTokenList[supportedTokenList.length - 1];
        supportedTokenList.pop();
        emit RemovedSupportedToken(token);
    }
```
