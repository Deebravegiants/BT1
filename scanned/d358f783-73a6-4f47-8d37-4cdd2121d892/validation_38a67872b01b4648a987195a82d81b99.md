### Title
Silent Aave Deposit Failure in `unlockQueue` Leaves ETH Idle Without Yield Until Manual Operator Intervention - (`contracts/LRTWithdrawalManager.sol`)

### Summary
When the Aave integration is enabled and the Aave WETH supply cap is reached, the `try/catch` in `unlockQueue` silently swallows the revert from `depositToAaveExternal`. The unlocked ETH remains in `LRTWithdrawalManager` earning zero yield, with no automatic retry, until an operator manually calls `depositIdleETHToAave`.

### Finding Description
In `unlockQueue`, after `unstakingVault.redeem` transfers ETH to the contract, the code attempts an Aave deposit via an external self-call:

```solidity
// contracts/LRTWithdrawalManager.sol:309-317
if (isAaveIntegrationEnabled && asset == LRTConstants.ETH_TOKEN && assetAmountUnlocked > 0) {
    try this.depositToAaveExternal(assetAmountUnlocked) { }
    catch (bytes memory reason) {
        emit AaveDepositFailed(assetAmountUnlocked, reason);
        // Silently fail if Aave deposit fails (e.g., pool at max capacity)
        // Funds remain in contract for withdrawals
    }
}
```

If `aaveWETHGateway.depositETH` reverts (e.g., Aave WETH supply cap reached), the catch block only emits an event. `totalETHDepositedToAave` is never incremented, and the ETH balance of the contract increases by `assetAmountUnlocked` with no yield accruing. The only recovery is `depositIdleETHToAave`, which is `onlyLRTOperator` and has no automatic scheduling. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation
The Aave integration is explicitly enabled to earn yield on ETH held pending withdrawal. When the deposit silently fails, the protocol fails to deliver the promised Aave yield on that ETH for the entire duration until an operator intervenes. No funds are lost — users can still withdraw — but the yield that the integration was designed to generate is not earned. This matches **Low: Contract fails to deliver promised returns, but doesn't lose value**.

### Likelihood Explanation
Aave v3 WETH supply caps on mainnet are a known, recurring constraint that have been hit in production. The condition (`isAaveIntegrationEnabled && asset == ETH_TOKEN`) is the normal operating state of the protocol. Any `unlockQueue` call during a period when the Aave WETH pool is at capacity triggers this path. No attacker action is required — it is a passive environmental condition.

### Recommendation
After a failed deposit, schedule or flag the idle ETH for retry. Options include:
- Checking `getAaveAvailableCapacity()` before attempting the deposit and skipping the try/catch if capacity is zero (avoids the silent failure entirely).
- Storing a `pendingAaveDeposit` amount and having `depositIdleETHToAave` automatically pick it up, or emitting a more actionable alert that off-chain monitoring can act on immediately.
- Alternatively, document clearly that `AaveDepositFailed` events must be monitored and `depositIdleETHToAave` called promptly, and enforce this with an off-chain keeper.

    function depositIdleETHToAave(uint256 amount) external nonReentrant whenNotPaused onlyLRTOperator {
        if (!isAaveIntegrationEnabled) revert AaveIntegrationNotEnabled();

        uint256 idleBalance = address(this).balance;
        if (idleBalance == 0) revert AmountMustBeGreaterThanZero();

        uint256 depositAmount = amount;
        if (amount == type(uint256).max || amount > idleBalance) {
            depositAmount = idleBalance;
        }

        if (depositAmount > 0) {
            _depositToAave(depositAmount);
        }
    }

    /// @notice Get available capacity in Aave for WETH deposits
    /// @return availableCapacity The amount of WETH that can still be deposited to Aave
    /// @dev Returns 0 if Aave is not configured or if supply cap is reached
    function getAaveAvailableCapacity() external view returns (uint256 availableCapacity) {
        if (address(aaveAWETH) == address(0) || address(aaveDataProvider) == address(0)) {
            revert InvalidAaveConfiguration();
        }

        // Get supply cap from Aave Data Provider
        (, uint256 supplyCap) = aaveDataProvider.getReserveCaps(WETH_ADDRESS);

        // If supply cap is 0, it means unlimited capacity
        if (supplyCap == 0) return type(uint256).max;

        // Get current total supply of aWETH
        uint256 totalSupply = aaveAWETH.totalSupply();

        // Convert supply cap to wei and calculate available capacity
        uint256 supplyCapWei = supplyCap * 1e18;
        if (supplyCapWei > totalSupply) {
            availableCapacity = supplyCapWei - totalSupply;
        }
        // else returns 0 (pool is at or over capacity)
    }
