### Title
Stale `rsETHPrice` Silently Passes Freshness Checks in `RSETHPriceFeed.latestRoundData()` - (File: contracts/oracles/RSETHPriceFeed.sol)

### Summary
`RSETHPriceFeed.latestRoundData()` computes rsETH/USD by multiplying a live Chainlink ETH/USD price by the stored `LRTOracle.rsETHPrice` (rsETH/ETH). It returns the `updatedAt` timestamp exclusively from the ETH/USD Chainlink feed, with no accounting for when `rsETHPrice` was last written. Any downstream consumer that checks `updatedAt` for staleness will see a fresh timestamp even when the rsETH/ETH component is arbitrarily old.

### Finding Description
`RSETHPriceFeed` is a Chainlink-compatible price feed that derives rsETH/USD by combining two independent price sources:

1. **ETH/USD** — a live Chainlink aggregator (`ETH_TO_USD`), which carries its own `updatedAt`.
2. **rsETH/ETH** — the stored state variable `LRTOracle.rsETHPrice`, updated only when someone calls `updateRSETHPrice()` or `updateRSETHPriceAsManager()`.

In `latestRoundData()`:

```solidity
// contracts/oracles/RSETHPriceFeed.sol  lines 63-70
function latestRoundData()
    external view
    returns (uint80 roundId, int256 answer, uint256 startedAt, uint256 updatedAt, uint80 answeredInRound)
{
    (roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
    answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
}
```

The destructured `updatedAt` comes entirely from `ETH_TO_USD`. The freshness of `RS_ETH_ORACLE.rsETHPrice()` is never queried, tracked, or reflected in the return value. `LRTOracle` stores no `lastUpdatedAt` field alongside `rsETHPrice`, so there is no on-chain way for `RSETHPriceFeed` to surface it even if it tried. [1](#0-0) 

`rsETHPrice` is a plain storage variable in `LRTOracle`: [2](#0-1) 

It is written only at the end of `_updateRsETHPrice()`: [3](#0-2) 

`updateRSETHPrice()` is public but gated by `whenNotPaused`. When the protocol is paused (e.g., triggered automatically by a price-drop circuit-breaker), `rsETHPrice` freezes at its last value while `ETH_TO_USD` continues to update. `RSETHPriceFeed.latestRoundData()` will keep returning a fresh `updatedAt` from ETH/USD, masking the staleness of the rsETH/ETH leg entirely. ### Impact Explanation
Any external protocol (lending market, DEX, structured product) that integrates `RSETHPriceFeed` as a Chainlink-compatible oracle and applies a standard time-based staleness guard on `updatedAt` will accept a stale rsETH/USD price as current. Depending on the direction of price drift:

- **Stale price too high** (rsETH appreciated but `rsETHPrice` not yet updated): collateral is over-valued → borrowers can extract more debt than their collateral supports → bad debt / insolvency in the lending protocol.
- **Stale price too low** (rsETH depreciated but `rsETHPrice` not yet updated): collateral is under-valued → unnecessary liquidations of healthy positions.

Within the LRT-rsETH scope itself, the `RSETHPriceFeed` is the canonical published price feed for rsETH/USD; delivering a misleading `updatedAt` means the contract fails to deliver its promised return (a fresh, trustworthy composite price), which falls under **Low — contract fails to deliver promised returns**.

### Likelihood Explanation
Low. The condition requires `rsETHPrice` to become stale while `ETH_TO_USD` remains live. The most realistic trigger is the automatic pause fired by the price-drop circuit-breaker in `_updateRsETHPrice()`, which halts further updates to `rsETHPrice` while Chainlink ETH/USD continues ticking. This is a realistic operational scenario, not a theoretical one.

### Recommendation
`RSETHPriceFeed.latestRoundData()` should track and expose the staleness of both price components. Since `LRTOracle` does not store a `lastUpdatedAt` for `rsETHPrice`, the fix requires two coordinated changes:

1. **`LRTOracle`**: add a `uint256 public rsETHPriceUpdatedAt` storage variable and write `rsETHPriceUpdatedAt = block.timestamp` alongside `rsETHPrice = newRsETHPrice` in `_updateRsETHPrice()`.
2. **`RSETHPriceFeed`**: after fetching both timestamps, return `min(updatedAt_from_ETH_USD, rsETHPriceUpdatedAt)` as the composite `updatedAt`, analogous to the recommended fix for `DivReducer`.

### Proof of Concept
1. Protocol operates normally; `rsETHPrice` is current.
2. An EigenLayer slashing event causes `newRsETHPrice` to drop beyond `pricePercentageLimit`. `_updateRsETHPrice()` fires the circuit-breaker: `lrtDepositPool.pause()`, `withdrawalManager.pause()`, `_pause()` — and **returns early without writing `rsETHPrice`**.
3. `updateRSETHPrice()` is now blocked by `whenNotPaused`; `rsETHPrice` is frozen at its pre-slashing value.
4. Meanwhile, ETH/USD on Chainlink continues to update every heartbeat.
5. A lending protocol calls `RSETHPriceFeed.latestRoundData()`. It receives:
   - `answer` = stale (pre-slashing) rsETH/ETH × current ETH/USD — an inflated rsETH/USD price.
   - `updatedAt` = Chainlink ETH/USD's latest timestamp — appears fresh.
6. The lending protocol's staleness check passes. It accepts the inflated price, allowing rsETH holders to borrow against over-valued collateral. [6](#0-5) [7](#0-6)
