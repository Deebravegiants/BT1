### Title
Missing Heartbeat Staleness Check in `ChainlinkOracleForRSETHPoolCollateral.getRate()` Enables Stale-Rate Exploitation, Draining Protocol rsETH Reserves — (`contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`)

---

### Summary

`ChainlinkOracleForRSETHPoolCollateral.getRate()` validates only the round-completeness condition (`answeredInRound < roundID`) but omits a time-based heartbeat check (`block.timestamp - updatedAt > MAX_STALENESS`). When a supported collateral token's market price drops but the Chainlink feed has not yet published a new round (within its deviation/heartbeat window), the oracle returns a stale-high `tokenToETHRate`. Because the pool fee is computed in raw token units — not ETH-adjusted — the fee does not scale with the inflated rsETH output, so the protocol systematically gives away more rsETH value than the fee compensates for.

---

### Finding Description

**Root cause — `ChainlinkOracleForRSETHPoolCollateral.getRate()`:**

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L26-36
function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID) revert StalePrice();   // ← only round-completeness check
    if (timestamp == 0) revert IncompleteRound();
    if (ethPrice <= 0) revert InvalidPrice();
    // ← NO: block.timestamp - timestamp > MAX_STALENESS check
    ...
}
```

The `answeredInRound < roundID` guard catches an *unanswered* round but does **not** catch a *heartbeat-stale* round — i.e., a round that was answered but whose `updatedAt` timestamp is arbitrarily old. Chainlink feeds update either on a deviation threshold (e.g. 0.5 %) or a heartbeat (e.g. 24 h). If the token price drops by less than the deviation threshold, the feed will not publish a new round until the next heartbeat, leaving `answeredInRound == roundID` while the price is materially stale.

**Fee accounting mismatch — `RSETHPool.viewSwapRsETHAmountAndFee(amount, token)`:**

```solidity
// contracts/pools/RSETHPool.sol L335-346
uint256 feeBpsForToken = tokenFeeBps[token];
fee = amount * feeBpsForToken / 10_000;          // fee in token units, not ETH-adjusted
uint256 amountAfterFee = amount - fee;

uint256 rsETHToETHrate = getRate();
uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate(); // ← stale-high

rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;          // inflated
```

The same pattern appears in `RSETHPoolV3.viewSwapRsETHAmountAndFee` and `RSETHPoolNoWrapper.viewSwapRsETHAmountAndFee`.

The fee is a flat percentage of the raw token deposit (`amount * feeBps / 10_000`). It does not scale with `tokenToETHRate`. When `tokenToETHRate` is stale-high by a factor `k > 1`, `rsETHAmount` is inflated by `k`, but `fee` is unchanged. The protocol receives tokens worth `fairRate / staleRate < 1` of what it pays out in rsETH, and the fee revenue in tokens cannot cover the shortfall.

---

### Impact Explanation

Let `staleRate = 1.1e18`, `fairRate = 0.9e18`, `rsETHToETHrate = 1.05e18`, `feeBps = 30`, `amount = 1e18` tokens.

| Quantity | Value |
|---|---|
| Fee (tokens) | `1e18 * 30 / 10_000 = 3e15` tokens |
| amountAfterFee | `9.97e17` tokens |
| rsETHAmount (stale) | `9.97e17 * 1.1e18 / 1.05e18 ≈ 1.044e18` rsETH |
| rsETH value at fair price | `1.044 * 1.05 ≈ 1.096 ETH` |
| Protocol receives (tokens → ETH) | `1e18 * 0.9e18 / 1e18 = 0.9 ETH` |
| Protocol loss per deposit | `≈ 0.196 ETH` |
| Fee revenue (tokens → ETH) | `3e15 * 0.9e18 / 1e18 = 0.0027 ETH` |

The fee covers ~1.4 % of the loss. Repeated deposits during the staleness window drain the pool's rsETH reserves. `feeEarnedInToken[token]` accumulates at the nominal token rate while the protocol has given away rsETH worth far more than the fee compensates for, violating the invariant `feeEarnedInToken[token] * fairTokenRate >= excess_rsETH_minted * rsETHRate`.

---

### Likelihood Explanation

- No oracle operator compromise is required. Chainlink feeds for LST collateral (e.g. wstETH/ETH) have deviation thresholds of 0.5 %–1 % and heartbeats of 1–24 h. A 10–20 % market drop within a heartbeat window is realistic during volatile periods.
- The attacker only needs to monitor the on-chain `updatedAt` timestamp and deposit when `block.timestamp - updatedAt` is large relative to the expected heartbeat.
- No special role, governance access, or private key is needed — `deposit(token, amount, referralId)` is a public, permissionless function.
- The `limitDailyMint` guard in V3 variants caps per-day exposure but does not prevent the attack; it only bounds the magnitude per day.

---

### Recommendation

Add a configurable maximum staleness period to `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
uint256 public immutable maxStaleness; // e.g. 3600 for 1-hour feeds

constructor(address _oracle, uint256 _maxStaleness) {
    oracle = _oracle;
    maxStaleness = _maxStaleness;
}

function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 updatedAt, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (ethPrice <= 0) revert InvalidPrice();
    if (block.timestamp - updatedAt > maxStaleness) revert StalePrice(); // ← add this
    ...
}
```

Set `maxStaleness` to slightly above the feed's documented heartbeat (e.g. heartbeat + 10 %).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Invariant test (Foundry) — run on a local fork with a mock stale Chainlink feed

import "forge-std/Test.sol";

contract MockStaleChainlinkFeed {
    // Returns a price that was last updated 25 hours ago (stale for a 24h heartbeat feed)
    // answeredInRound == roundID so the existing StalePrice check passes
    function latestRoundData() external view returns (
        uint80 roundId, int256 answer, uint256 startedAt, uint256 updatedAt, uint80 answeredInRound
    ) {
        roundId = 100;
        answeredInRound = 100;                          // passes answeredInRound < roundID check
        answer = 1.1e8;                                 // stale-high: 1.1 ETH per token (8 decimals)
        updatedAt = block.timestamp - 25 hours;         // 25h stale, no time check in contract
        startedAt = updatedAt;
    }
    function decimals() external pure returns (uint8) { return 8; }
}

contract StaleOracleInvariantTest is Test {
    // Deploy ChainlinkOracleForRSETHPoolCollateral with the mock stale feed
    // Deploy RSETHPool with feeBps=30, rsETHOracle returning 1.05e18
    // Perform N deposits
    // Assert: feeEarnedInToken[token] * fairRate < excess_rsETH_minted * rsETHRate
    // (invariant breaks — fee revenue does not cover excess rsETH given away)
    function testStaleOracleBreaksInvariant() public {
        MockStaleChainlinkFeed staleFeed = new MockStaleChainlinkFeed();
        // staleRate = 1.1e18, fairRate = 0.9e18, rsETHRate = 1.05e18, feeBps = 30
        // After 1 deposit of 1e18 tokens:
        //   fee = 3e15 tokens
        //   rsETHAmount = (1e18 - 3e15) * 1.1e18 / 1.05e18 ≈ 1.044e18
        //   fair rsETHAmount = (1e18 - 3e15) * 0.9e18 / 1.05e18 ≈ 0.854e18
        //   excess rsETH = 1.044e18 - 0.854e18 = 0.190e18
        //   fee in ETH = 3e15 * 0.9e18 / 1e18 = 2.7e15 ETH
        //   excess rsETH in ETH = 0.190e18 * 1.05e18 / 1e18 = 0.1995e18 ETH
        //   2.7e15 < 1.995e17 → invariant broken
        uint256 fairRate = 0.9e18;
        uint256 rsETHRate = 1.05e18;
        uint256 feeBps = 30;
        uint256 amount = 1e18;
        uint256 staleRate = 1.1e18;

        uint256 fee = amount * feeBps / 10_000;
        uint256 amountAfterFee = amount - fee;
        uint256 rsETHMinted = amountAfterFee * staleRate / rsETHRate;
        uint256 fairRsETHMinted = amountAfterFee * fairRate / rsETHRate;
        uint256 excessRsETH = rsETHMinted - fairRsETHMinted;

        uint256 feeInETH = fee * fairRate / 1e18;
        uint256 excessInETH = excessRsETH * rsETHRate / 1e18;

        // Invariant: feeInETH >= excessInETH
        assertGe(feeInETH, excessInETH, "INVARIANT BROKEN: fee does not cover excess rsETH");
        // This assertion FAILS: 2.7e15 < 1.995e17
    }
}
```

Running this test will fail the `assertGe`, confirming the invariant breaks under a stale-high oracle with realistic parameters.
