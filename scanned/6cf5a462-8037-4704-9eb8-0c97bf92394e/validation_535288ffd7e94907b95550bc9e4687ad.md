### Title
Unbounded Gas Consumption in `getETHDistributionData()` Due to Uncapped `maxNodeDelegatorLimit` Blocks All Price Updates — (`contracts/LRTDepositPool.sol`, `contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a permissionless `public` function. Its internal call chain reaches `LRTDepositPool.getETHDistributionData()`, which makes **two external EigenLayer calls per NDC** in an unbounded loop. Because `updateMaxNodeDelegatorLimit()` enforces no upper bound, a legitimately large NDC queue causes `updateRSETHPrice()` to run out of gas, permanently blocking all price updates.

---

### Finding Description

**Call chain:**

```
updateRSETHPrice()                          [public, no role check]
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ getTotalAssetDeposits(ETH_TOKEN)
                 └─ getAssetDistributionData(ETH_TOKEN)
                      └─ getETHDistributionData()
                           └─ for i in 0..ndcsCount:
                                nodeDelegatorQueue[i].balance          // BALANCE opcode
                                NDC[i].getEffectivePodShares()         // external call → EigenLayer.getWithdrawableShares()
                                NDC[i].getAssetUnstaking(ETH_TOKEN)    // external call → EigenLayer.getQueuedWithdrawals()
```

`updateRSETHPrice()` carries no access control: [1](#0-0) 

The ETH distribution loop makes two external EigenLayer calls per NDC: [2](#0-1) 

`getEffectivePodShares()` calls `DelegationManager.getWithdrawableShares()`: [3](#0-2) 

`getAssetUnstaking()` calls `DelegationManager.getQueuedWithdrawals()` and then iterates over every pending withdrawal: [4](#0-3) 

`updateMaxNodeDelegatorLimit()` only checks the new limit is ≥ the current queue length — **no upper bound**: [5](#0-4) 

The protocol itself acknowledges gas sensitivity in `LRTUnstakingVault`, capping `maxUncompletedWithdrawalCount` at 80 with the comment "120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price": [6](#0-5) 

Yet no analogous cap exists for the NDC count, creating an asymmetry: the withdrawal count is bounded, but the NDC count that drives the outer loop is not.

---

### Impact Explanation

If `maxNodeDelegatorLimit` is raised and a large number of NDCs are registered (both legitimate admin operations), every call to `updateRSETHPrice()` — including the public entry point — will exceed the block gas limit. This permanently blocks rsETH price updates, which in turn blocks:

- All deposits (price is read in `getRsETHAmountToMint`)
- All withdrawals (price is read in `getExpectedAssetAmount`)
- Protocol fee accrual

This matches the **Medium: Unbounded gas consumption** scope.

---

### Likelihood Explanation

The default `maxNodeDelegatorLimit` is 10, which is safe. However:

1. The limit is freely raisable by admin with no on-chain ceiling.
2. Each new EigenLayer operator relationship requires a new NDC, so the queue naturally grows as the protocol scales.
3. `updateRSETHPrice()` is public — once the queue is large enough, any caller (including a griefing bot) triggers OOG on every invocation.
4. The protocol's own comment in `LRTUnstakingVault` demonstrates awareness that gas is a binding constraint, yet the NDC loop has no corresponding guard.

---

### Recommendation

1. **Enforce an absolute upper bound** in `updateMaxNodeDelegatorLimit()`:
   ```solidity
   uint256 public constant MAX_NDC_HARD_LIMIT = 20; // tune to gas budget
   function updateMaxNodeDelegatorLimit(uint256 maxNodeDelegatorLimit_) external onlyLRTAdmin {
       if (maxNodeDelegatorLimit_ > MAX_NDC_HARD_LIMIT) revert LimitTooHigh();
       if (maxNodeDelegatorLimit_ < nodeDelegatorQueue.length) revert InvalidMaximumNodeDelegatorLimit();
       maxNodeDelegatorLimit = maxNodeDelegatorLimit_;
   }
   ```
2. **Add a gas-budget test** that measures `updateRSETHPrice()` gas at `maxNodeDelegatorLimit` NDCs and asserts it stays well below 15 million gas.
3. Consider caching per-NDC ETH balances off-chain and using a push model (NDCs report their balance on state change) rather than pulling from all NDCs on every price update.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Foundry fork test (local Anvil fork, no mainnet)
contract UnboundedGasPoC is Test {
    uint256 constant M = 50; // raise above current maxNodeDelegatorLimit

    function test_updateRSETHPrice_OOG() public {
        // 1. Admin raises limit
        vm.prank(admin);
        depositPool.updateMaxNodeDelegatorLimit(M);

        // 2. Deploy M mock NDCs, each with non-trivial getEffectivePodShares / getAssetUnstaking
        for (uint256 i; i < M; i++) {
            MockNDC ndc = new MockNDC(eigenLayerMock);
            vm.prank(admin);
            address[] memory arr = new address[](1);
            arr[0] = address(ndc);
            depositPool.addNodeDelegatorContractToQueue(arr);
        }

        // 3. Measure gas — expect linear growth; assert OOG at M > safe threshold
        uint256 gasBefore = gasleft();
        lrtOracle.updateRSETHPrice();
        uint256 gasUsed = gasBefore - gasleft();

        // At M=50 with real EigenLayer call costs (~30k gas each × 2 × 50 = 3M+)
        // plus outer loop overhead, this approaches or exceeds 15M block gas limit
        assertGt(gasUsed, 10_000_000, "gas should grow linearly with NDC count");
    }
}
```

Gas grows as `O(M × cost_of_getWithdrawableShares + M × cost_of_getQueuedWithdrawals)`. At realistic EigenLayer call costs (~20–50k gas each), 50+ NDCs push the transaction past the 30M gas block limit.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
