### Title
Per-Token Cap in `depositBridgerAssets` Allows Aggregate Collateral to Exceed `totalSupply` — (`contracts/L2/RsETHTokenWrapper.sol`)

---

### Summary

`depositBridgerAssets` enforces a per-token cap computed as `totalSupply - balance[_asset]`. Because `depositBridgerAssets` transfers tokens in **without minting wrsETH**, `totalSupply()` is unchanged between calls. With two or more allowed tokens, the BRIDGER_ROLE can deposit up to `N × totalSupply` aggregate collateral while each individual call passes its guard, permanently locking the excess tokens in the contract.

---

### Finding Description

`maxAmountToDepositBridgerAsset` computes the cap independently for each token: [1](#0-0) 

It reads `totalSupply()` and only the balance of the specific `_asset`, ignoring all other allowed tokens. `depositBridgerAssets` uses this value as its sole guard: [2](#0-1) 

Critically, `depositBridgerAssets` does **not** call `_mint`, so `totalSupply()` is unchanged after the first call. When the BRIDGER_ROLE calls the function for a second allowed token, `totalSupply - balance[tokenB]` is still the full `totalSupply`, allowing a second full-supply deposit to pass.

The only exit path for collateral is `_withdraw`, which requires burning wrsETH: [3](#0-2) 

With only `S` wrsETH in circulation, at most `S` tokens can ever be withdrawn. Any collateral beyond `S` is permanently irrecoverable — there is no admin rescue or sweep function.

---

### Impact Explanation

With two allowed tokens and `totalSupply = S`:

| Step | Action | `balance[tokenA]` | `balance[tokenB]` | `totalSupply` |
|------|--------|-------------------|-------------------|---------------|
| 0 | Initial | 0 | 0 | S |
| 1 | `depositBridgerAssets(tokenA, S)` — cap = S−0 = S ✓ | S | 0 | S |
| 2 | `depositBridgerAssets(tokenB, S)` — cap = S−0 = S ✓ | S | S | S |

Aggregate collateral = `2S`, but only `S` wrsETH exists to redeem it. The extra `S` tokens of whichever asset is redeemed last are permanently stuck. This violates the intended invariant `Σ balance[token] ≤ totalSupply` and causes a permanent loss of the excess bridger-deposited collateral.

Impact: **Low — contract fails to deliver promised returns (proper collateral bounding), excess collateral is irrecoverable.**

---

### Likelihood Explanation

Requires two preconditions, both reachable through normal protocol operation:

1. A second token is added via `addAllowedToken` (requires `TIMELOCK_ROLE`, a legitimate governance action): [4](#0-3) 

2. The BRIDGER_ROLE calls `depositBridgerAssets` for both tokens — its intended use case. No compromise or malicious intent is required; a well-intentioned bridger following the per-token cap logic would trigger this.

---

### Recommendation

Replace the per-token cap with an aggregate cap. In `maxAmountToDepositBridgerAsset`, subtract the **sum of all allowed token balances** from `totalSupply`, not just the balance of the queried token. Alternatively, track a single `totalCollateral` storage variable that is incremented in `depositBridgerAssets` and decremented in `_withdraw`, and enforce `totalCollateral + _amount ≤ totalSupply()`.

---

//         tokenA and tokenB are both in allowedTokens

// balance[tokenA] = 1000e18, totalSupply = 1000e18 (unchanged)

// Step 2: maxAmountToDepositBridgerAsset(tokenB) = 1000e18 - 0 = 1000e18

// balance[tokenB] = 1000e18, totalSupply = 1000e18 (unchanged)

// Invariant broken: balance[tokenA] + balance[tokenB] = 2000e18 > totalSupply = 1000e18
// Only 1000e18 wrsETH exists; 1000e18 of one token is permanently locked.
assert(
    tokenA.balanceOf(address(wrapper)) + tokenB.balanceOf(address(wrapper))
    > wrapper.totalSupply()
); // passes — invariant violated
```

    function maxAmountToDepositBridgerAsset(address _asset) public view returns (uint256) {
        if (!allowedTokens[_asset]) return 0;

        // get totalSupply of wrsETH minted
        uint256 wrsETHSupply = totalSupply();
        // balance of _asset with the contract
        uint256 balanceOfAssetInWrapper = ERC20Upgradeable(_asset).balanceOf(address(this));

        if (balanceOfAssetInWrapper > wrsETHSupply) return 0;

        return wrsETHSupply - balanceOfAssetInWrapper;
    }
```
