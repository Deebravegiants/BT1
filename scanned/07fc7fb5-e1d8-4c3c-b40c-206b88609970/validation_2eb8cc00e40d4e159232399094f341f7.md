### Title
Missing Chainlink Staleness Check Enables Unprivileged Caller to Trigger Protocol-Wide Auto-Pause — (`contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` fetches `latestRoundData()` but discards all freshness fields (`updatedAt`, `answeredInRound`). When a Chainlink feed goes stale, the last reported price — which for LSTs is always lower than the current price — is silently accepted. Because `LRTOracle.updateRSETHPrice()` is a public, permissionless function, any caller can invoke it while the feed is stale, causing `_updateRsETHPrice()` to compute a depressed `newRsETHPrice` that breaches `pricePercentageLimit` relative to `highestRsethPrice`, triggering an automatic pause of `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle`.

---

### Finding Description

**Root cause — missing staleness validation in `ChainlinkPriceOracle.getAssetPrice()`:**

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol  lines 49-55
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();   // updatedAt / answeredInRound silently dropped
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
``` [1](#0-0) 

The same codebase's `ChainlinkOracleForRSETHPoolCollateral` correctly validates both fields:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol  lines 27-32
(uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
    AggregatorV3Interface(oracle).latestRoundData();
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
``` [2](#0-1) 

The omission in `ChainlinkPriceOracle` is therefore a code deficiency, not an intentional design choice.

**Permissionless trigger — `updateRSETHPrice()` has no role guard:**

```solidity
// contracts/LRTOracle.sol  line 87
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [3](#0-2) 

**Auto-pause logic in `_updateRsETHPrice()`:**

```solidity
// contracts/LRTOracle.sol  lines 270-282
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;
    }
``` [4](#0-3) 

**Call chain:**

`updateRSETHPrice()` → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `LRTOracle.getAssetPrice(asset)` → `ChainlinkPriceOracle.getAssetPrice(asset)` → stale `latestRoundData()` price accepted → depressed `newRsETHPrice` → auto-pause fires. [5](#0-4) 

---

### Impact Explanation

When the auto-pause fires, `LRTDepositPool.pause()`, `LRTWithdrawalManager.pause()`, and `LRTOracle._pause()` are all called atomically. All user deposits and withdrawals are blocked until an admin manually calls `unpause()` on each contract. This constitutes **temporary freezing of funds** with no loss of principal, matching the Medium impact scope.

---

### Likelihood Explanation

- LST/LRT Chainlink feeds (stETH/ETH, rETH/ETH, ETHx/ETH) have heartbeats of 24 hours and deviation thresholds of 0.5–1%. A feed that misses even one heartbeat returns a price that is days-old.
- LST prices monotonically increase (staking rewards accrue continuously), so any stale price is necessarily lower than the current `highestRsethPrice`.
- If `pricePercentageLimit` is set to 1% (1e16 — the documented example), a feed stale by ~3–7 days is sufficient to breach the threshold for typical LST appreciation rates (~4–5% APY).
- The attacker needs no capital, no approvals, and no privileged role — only the ability to call a public function at the right moment.
- Chainlink feed staleness is a known, non-negligible operational risk (network congestion, node outages).

---

### Recommendation

Add staleness and validity checks to `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();
require(answeredInRound >= roundId, "Stale price");
require(updatedAt != 0, "Incomplete round");
require(block.timestamp - updatedAt <= STALENESS_THRESHOLD, "Price too old");
require(price > 0, "Non-positive price");
```

The `STALENESS_THRESHOLD` should be set per-feed based on its documented heartbeat (e.g., 25 hours for a 24-hour heartbeat feed).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";

interface ILRTOracle { function updateRSETHPrice() external; }
interface IPausable   { function paused() external view returns (bool); }

contract MockStaleChainlinkFeed {
    // Returns a price 2% below the current live price — simulates a stale feed
    function latestRoundData() external pure returns (
        uint80, int256, uint256, uint256, uint80
    ) {
        // answeredInRound == roundId (passes the answeredInRound check if it existed)
        // updatedAt is 48 hours ago — stale
        return (1, 0.98e8, 0, block.timestamp - 48 hours, 1);
    }
    function decimals() external pure returns (uint8) { return 8; }
}

contract StalePriceAutoFreezeTest is Test {
    // Fork mainnet / testnet, deploy mock feed, wire it into ChainlinkPriceOracle,
    // ensure highestRsethPrice is set to the live price, then:

    function test_staleChainlinkTriggersAutoFreeze() public {
        // 1. Deploy mock stale feed returning 0.98e18 (2% below highestRsethPrice of 1e18)
        MockStaleChainlinkFeed staleFeed = new MockStaleChainlinkFeed();

        // 2. (Setup) wire staleFeed into ChainlinkPriceOracle for the target asset
        //    vm.prank(manager); chainlinkOracle.updatePriceFeedFor(asset, address(staleFeed));

        // 3. Call updateRSETHPrice() as an unprivileged address
        vm.prank(address(0xdead));
        ILRTOracle(lrtOracleAddr).updateRSETHPrice();

        // 4. Assert all three contracts are paused
        assertTrue(IPausable(lrtDepositPoolAddr).paused(),       "DepositPool not paused");
        assertTrue(IPausable(lrtWithdrawalManagerAddr).paused(), "WithdrawalManager not paused");
        assertTrue(IPausable(lrtOracleAddr).paused(),            "LRTOracle not paused");
    }
}
```

The test requires no privileged role for the caller (`address(0xdead)`). The only setup step — wiring the mock feed — is performed by the manager role as a one-time configuration, which is within normal operational scope and does not represent an additional trust assumption for the attacker.
