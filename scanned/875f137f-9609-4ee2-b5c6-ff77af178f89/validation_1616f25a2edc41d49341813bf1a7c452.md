### Title
Reward Token Dust Permanently Stranded Per Epoch Due to Integer Division Truncation in `notifyRewardAmount` - (File: `contracts/KERNEL/KernelDepositPool.sol`)

---

### Summary

`KernelDepositPool.notifyRewardAmount` computes `rewardRate` via integer division (`receivedAmount / duration`), permanently stranding `receivedAmount % duration` reward tokens in the contract each epoch with no recovery path.

---

### Finding Description

In `notifyRewardAmount`, the reward rate is set by integer-dividing the received token amount by the distribution duration: [1](#0-0) 

```solidity
if (block.timestamp >= finishAt) {
    rewardRate = receivedAmount / duration;
} else {
    uint256 remaining = (finishAt - block.timestamp) * rewardRate;
    rewardRate = (receivedAmount + remaining) / duration;
}
```

Solidity integer division truncates. The total tokens that will ever be distributed is `rewardRate * duration`, which equals `receivedAmount - (receivedAmount % duration)`. The remainder — `receivedAmount % duration` tokens — is transferred into the contract but never enters the `rewardRate` accounting and is therefore never distributed to stakers.

The contract contains no `recoverERC20` or equivalent admin rescue function across its entire implementation (lines 1–621), so these tokens are permanently locked. [2](#0-1) 

Crucially, subsequent calls to `notifyRewardAmount` do **not** recapture prior-epoch dust. The `remaining` term on line 582 only accounts for `rewardRate * timeLeft` from the current active period — it does not include any previously stranded dust sitting in the contract balance.

---

### Impact Explanation

Every reward epoch permanently strands up to `duration - 1` tokens (in the reward token's smallest unit). For a 7-day duration (`604800` seconds) with a 6-decimal token (e.g., USDC), up to `604799` units (~$0.60) are stranded per epoch. Over many epochs this accumulates. Stakers collectively receive strictly less than the full reward amount deposited by the admin, violating the invariant that all deposited reward tokens are eventually distributed.

Impact: **Low — Contract fails to deliver promised returns, but doesn't lose value.**

---

### Likelihood Explanation

This triggers on every single call to `notifyRewardAmount` where `receivedAmount % duration != 0`, which is the overwhelming majority of real-world inputs. No special conditions, attacker, or privileged compromise is required — it is a structural property of the arithmetic.

---

### Recommendation

Scale `rewardRate` by a precision multiplier (e.g., `1e18`) and carry the fractional component forward, or explicitly add any unspent balance from the previous epoch back into the new epoch's distributable amount:

```solidity
// Option A: include leftover balance from prior epochs
uint256 leftover = rewardsToken.balanceOf(address(this)) - (rewardRate * (finishAt - block.timestamp));
rewardRate = (receivedAmount + leftover) / duration;

// Option B: after computing rewardRate, verify no dust is lost
require(rewardRate * duration == receivedAmount, "dust would be stranded");
```

Option A is the standard fix used in audited Synthetix forks.

// Setup: duration = 7 (seconds), receivedAmount = 10
// rewardRate = 10 / 7 = 1 (truncated)
// Total distributed = 1 * 7 = 7 tokens
// Stranded = 10 % 7 = 3 tokens

// Fuzz sketch (Foundry):
function testDustStranded(uint256 amount, uint256 dur) public {
    vm.assume(dur > 0 && dur < 1e6);
    vm.assume(amount > dur && amount < 1e24);
    vm.assume(amount % dur != 0); // dust condition

    pool.setRewardsDuration(dur);
    rewardToken.mint(admin, amount);
    vm.prank(admin);
    rewardToken.approve(address(pool), amount);
    vm.prank(admin);
    pool.notifyRewardAmount(amount);

    vm.warp(block.timestamp + dur + 1);
    vm.prank(staker);
    pool.getReward();

    // Stranded dust remains in contract
    assertEq(rewardToken.balanceOf(address(pool)), amount % dur);
    // Staker received less than deposited
    assertLt(rewardToken.balanceOf(staker), amount);
}

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
