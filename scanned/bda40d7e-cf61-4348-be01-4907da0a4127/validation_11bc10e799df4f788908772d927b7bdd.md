### Title
Single Global `minAmountToDeposit` Applied Across All Assets Without Exchange-Rate Conversion - (File: contracts/LRTDepositPool.sol)

### Summary
`LRTDepositPool._beforeDeposit()` enforces a single global `minAmountToDeposit` threshold against `depositAmount` expressed in the native units of whichever asset is being deposited. Because different supported LSTs (stETH, ETHx, sfrxETH) trade at different exchange rates relative to ETH, the same raw-token minimum represents a different ETH-equivalent value for each asset. This is the same unit-mismatch class as TRST-M-1: a threshold calibrated in one unit is compared directly against an amount in a different unit, with no conversion.

### Finding Description
`LRTDepositPool` stores a single global minimum:

```solidity
uint256 public minAmountToDeposit;
```

Both `depositETH` and `depositAsset` funnel through `_beforeDeposit`, which enforces:

```solidity
if (depositAmount == 0 || depositAmount < minAmountToDeposit) {
    revert InvalidAmountToDeposit();
}
```

`depositAmount` is `msg.value` for ETH, or the raw ERC-20 token amount for LSTs. `minAmountToDeposit` is a single `uint256` with no per-asset denomination. The admin sets it once via `setMinAmountToDeposit`, presumably calibrated against one asset (e.g., ETH wei). When the same value is then compared against ETHx or sfrxETH amounts — both of which accumulate staking rewards and therefore trade at a growing premium to ETH — the effective minimum in ETH-equivalent terms is different for each asset. There is no oracle call or exchange-rate conversion at the point of the minimum check; the oracle is only consulted later to compute `rsethAmountToMint`.

### Impact Explanation
The minimum deposit threshold is inconsistently enforced across supported assets:

- If `minAmountToDeposit` is calibrated in ETH wei (e.g., `1e15` = 0.001 ETH), then depositing `1e15` raw ETHx tokens passes the check, but ETHx is worth more than ETH per token (e.g., 1 ETHx ≈ 1.05 ETH and growing), so the effective minimum in ETH value is higher for ETHx depositors than for ETH depositors.
- Conversely, if the admin calibrates the minimum for ETHx, ETH depositors face a lower effective minimum in ETH value.
- As sfrxETH and ETHx continue to appreciate relative to ETH over time, the discrepancy grows, making the minimum increasingly inconsistent.

A user depositing a premium LST may be blocked from depositing an amount that is economically above the intended minimum (or allowed to deposit below it), depending on which asset the admin calibrated the threshold for. This matches the impact class: **contract fails to deliver promised returns, but doesn't lose value** (Low).

### Likelihood Explanation
This condition is always present whenever `minAmountToDeposit > 0` and more than one asset with a non-1:1 ETH exchange rate is supported. ETHx and sfrxETH are both supported and both trade at a premium. The discrepancy grows monotonically as staking rewards accumulate. Any user calling `depositAsset` with ETHx or sfrxETH is affected.

### Recommendation
Replace the single global `minAmountToDeposit` with a per-asset mapping, or convert `depositAmount` to its ETH-equivalent value (using `lrtOracle.getAssetPrice(asset)`) before comparing against the minimum threshold. For example:

```solidity
uint256 depositValueInEth = depositAmount * lrtOracle.getAssetPrice(asset) / 1e18;
if (depositValueInEth == 0 || depositValueInEth < minAmountToDeposit) {
    revert InvalidAmountToDeposit();
}
```

This mirrors how `getRsETHAmountToMint` already uses the oracle to normalize asset amounts.

### Proof of Concept

1. Admin sets `minAmountToDeposit = 1e15` (intending "minimum 0.001 ETH equivalent").
2. ETHx exchange rate is 1.05 ETH per ETHx (realistic after months of staking rewards).
3. User A calls `depositETH` with `msg.value = 1e15` (0.001 ETH). Check: `1e15 >= 1e15` → passes. ETH value: 0.001 ETH.
4. User B calls `depositAsset(ETHx, 1e15, ...)`. Check: `1e15 >= 1e15` → passes. ETH value: 0.00105 ETH — the minimum is effectively 5% higher for ETHx depositors.
5. Over time, as ETHx appreciates to 1.10 ETH per token, the same `1e15` raw ETHx represents 0.0011 ETH — a 10% higher effective minimum, growing indefinitely.
