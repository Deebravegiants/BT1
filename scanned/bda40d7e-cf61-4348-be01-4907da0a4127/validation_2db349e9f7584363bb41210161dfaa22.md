### Title
Missing `whenNotPaused` on `swapAssetToPremintedRsETH` Allows ETH Drain During Emergency Pause - (File: contracts/pools/RSETHPoolV2ExternalBridge.sol)

### Summary
`RSETHPoolV2ExternalBridge.swapAssetToPremintedRsETH()` transfers ETH from the pool to the caller in exchange for rsETH, is accessible to any address holding `WHITELISTED_USER_ROLE`, but does not apply the `whenNotPaused` modifier. When the contract is paused in response to an emergency, this function remains fully operational, allowing whitelisted users to continue draining the pool's ETH balance while new deposits are blocked.

### Finding Description
`RSETHPoolV2ExternalBridge` implements a custom pause mechanism via `bool public paused` and a `whenNotPaused` modifier. The `deposit()` function correctly applies this modifier. However, `swapAssetToPremintedRsETH()` — which transfers ETH from the pool to the caller in exchange for rsETH — does not apply `whenNotPaused`.

The function is gated by `onlyOperatorOrWhitelisted(msg.sender)`, which grants access to any address holding `WHITELISTED_USER_ROLE`. This role is explicitly distinct from the admin/operator roles and is designed to be granted to a broader set of pool users. [1](#0-0) 

During a pause triggered by `PAUSER_ROLE` (e.g., in response to an oracle anomaly, exploit, or other emergency), `deposit()` is correctly blocked: [2](#0-1) 

But `swapAssetToPremintedRsETH()` remains callable with no pause check: [3](#0-2) 

The `onlyOperatorOrWhitelisted` modifier confirms the dual-role access path: [4](#0-3) 

The function computes `ethAmount` from the oracle rate and transfers ETH from the pool to the caller: [5](#0-4) 

### Impact Explanation
The pool's ETH balance (deposited by users) can be drained by whitelisted users even while the contract is paused. This defeats the purpose of the pause mechanism: `deposit()` is blocked, preventing new ETH from entering, while `swapAssetToPremintedRsETH()` continues to allow ETH to exit. The pool can be left empty when the pause is lifted, preventing the protocol from resuming normal operations and temporarily freezing the ability of depositors to interact with a functioning pool. This maps to **Medium — Temporary freezing of funds**.

### Likelihood Explanation
Likelihood is **Low**. The caller must hold `WHITELISTED_USER_ROLE` (granted by admin), must possess rsETH to provide in exchange, and the per-call amount is bounded by `maxAmountToDepositBridgerAsset`. However, `WHITELISTED_USER_ROLE` is a user-level role explicitly designed to be granted to a broader set of pool participants, making it a realistic attack surface during a pause event.

### Recommendation
Add the `whenNotPaused` modifier to `swapAssetToPremintedRsETH()`:

```solidity
function swapAssetToPremintedRsETH(
    address rsETH,
    uint256 rsETHAmount
)
    external
    nonReentrant
    whenNotPaused   // add this
    onlyOperatorOrWhitelisted(msg.sender)
{
```

### Proof of Concept
1. Admin grants `WHITELISTED_USER_ROLE` to address `Alice`.
2. Users deposit ETH via `deposit()`, accumulating ETH in the pool.
3. An emergency occurs (e.g., oracle anomaly); `PAUSER_ROLE` holder calls `pause()`.
4. `deposit()` now reverts for all callers due to `whenNotPaused`.
5. Alice calls `swapAssetToPremintedRsETH(rsETH, rsETHAmount)` — no pause check, call succeeds.
6. ETH is transferred from the pool to Alice; the pool's ETH balance is reduced.
7. Steps 5–6 can be repeated (within `maxAmountToDepositBridgerAsset` per call) until the pool is drained.
8. When the pause is lifted, the pool has no ETH to service normal operations.

```text
    function swapAssetToPremintedRsETH(
        address rsETH,
        uint256 rsETHAmount
    )
        external
        nonReentrant
        onlyOperatorOrWhitelisted(msg.sender)
    {
        UtilLib.checkNonZeroAddress(rsETH);

        IRsETHTokenWrapper wrapper = IRsETHTokenWrapper(address(wrsETH));

        if (!wrapper.allowedTokens(rsETH)) revert TokenNotAllowedInWrapper();
        if (rsETHAmount == 0) revert InvalidAmount();
        if (rsETHAmount > wrapper.maxAmountToDepositBridgerAsset(rsETH)) revert ExceedsMaxAmountToDepositInWrapper();

        // Get the amount of ETH to transfer to the user for the given amount of rsETH provided
        uint256 ethAmount = viewSwapAssetToPremintedRsETH(rsETHAmount);

        // Transfer rsETH from sender to the wrapper
        IERC20(rsETH).safeTransferFrom(msg.sender, address(wrapper), rsETHAmount);

        // Transfer the ETH from the pool to the sender
        if (getETHBalanceMinusFees() < ethAmount) revert InsufficientETHBalanceForReverseSwap();
        (bool success,) = payable(msg.sender).call{ value: ethAmount }("");
        if (!success) revert TransferFailed();

        emit ReverseSwapOccurred(msg.sender, rsETH, rsETHAmount, ethAmount);
    }
```
