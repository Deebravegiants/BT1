### Title
`dailyMintAmount` Not Decreased During `swapAssetToPremintedRsETH`, Causing Daily Mint Limit to Appear Inflated - (File: `contracts/pools/RSETHPoolV3.sol`)

---

### Summary

The `dailyMintAmount` state variable is incremented every time a user deposits (mints wrsETH) via the `limitDailyMint` modifier, but it is never decremented when the operator calls `swapAssetToPremintedRsETH` (the reverse-swap path). This mirrors the original report's pattern exactly: a shared accounting variable is updated on one side of a paired operation but not the other, causing the tracked value to diverge from the true net state.

---

### Finding Description

In `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, and `RSETHPoolV3WithNativeChainBridge`, every deposit goes through the `limitDailyMint` modifier, which increments `dailyMintAmount` by the rsETH amount minted: [1](#0-0) 

The reverse operation, `swapAssetToPremintedRsETH`, is explicitly described as "the opposite of `deposit`." It accepts rsETH from the caller, deposits it into the wrapper, and returns ETH or tokens from the pool: [2](#0-1) 

However, `swapAssetToPremintedRsETH` contains no corresponding decrement of `dailyMintAmount`. After the reverse swap, the pool has given out ETH/tokens and received rsETH into the wrapper — effectively unwinding a prior deposit — but `dailyMintAmount` still reflects the original gross minted amount, not the net.

The same pattern exists in `RSETHPoolV3ExternalBridge`: [3](#0-2) 

And in `RSETHPoolV3WithNativeChainBridge`: [4](#0-3) 

---

### Impact Explanation

After `swapAssetToPremintedRsETH` is called for an amount `Y`, the true net wrsETH minted today is `dailyMintAmount - Y`, but the contract still records `dailyMintAmount` as the full gross value. The daily mint check: [5](#0-4) 

will reject new user deposits that would have been valid against the true net figure. Users are blocked from depositing for up to the remainder of the current day — a **temporary freezing of the deposit function** — even though the actual net minted rsETH is below the configured limit. This maps to **Medium: Temporary freezing of funds**.

---

### Likelihood Explanation

`swapAssetToPremintedRsETH` is a routine operational function (reverse swap) called by the `OPERATOR_ROLE` as part of normal protocol lifecycle — not a compromise or malicious act. Any time the operator performs a reverse swap on a day with significant deposit activity, the residual `dailyMintAmount` inflation can push the apparent daily total over the limit, blocking all further user deposits until midnight resets the counter.

---

### Recommendation

Decrement `dailyMintAmount` inside `swapAssetToPremintedRsETH` by the rsETH amount being returned, symmetrically with how `limitDailyMint` increments it on deposit. Apply the same fix to `RSETHPoolV3ExternalBridge` and `RSETHPoolV3WithNativeChainBridge`. Guard against underflow (clamp to zero) in case of cross-day accounting edge cases.

---

### Proof of Concept

1. Daily mint limit is set to 100 rsETH. `dailyMintAmount = 0`.
2. Users deposit, minting 90 rsETH. `dailyMintAmount = 90`.
3. Operator calls `swapAssetToPremintedRsETH` for 50 rsETH — pool gives out 50 ETH, receives 50 rsETH into wrapper. `dailyMintAmount` remains `90` (not decremented to `40`).
4. A user attempts to deposit, which would mint 15 rsETH. The check `90 + 15 > 100` reverts with `DailyMintLimitExceeded`.
5. The true net minted today is `90 - 50 = 40`. The deposit of 15 rsETH would bring the net to 55, well within the 100 limit. The user is incorrectly blocked until the next day's reset. [6](#0-5) [2](#0-1)

```text
    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```

**File:** contracts/pools/RSETHPoolV3.sol (L414-450)
```text
    function swapAssetToPremintedRsETH(
        address rsETH,
        address token,
        uint256 rsETHAmount
    )
        external
        nonReentrant
        onlySupportedTokenOrEth(token)
        onlyRole(OPERATOR_ROLE)
    {
        UtilLib.checkNonZeroAddress(rsETH);

        IRsETHTokenWrapper wrapper = IRsETHTokenWrapper(address(wrsETH));
        IERC20 tokenContract = IERC20(token);

        if (!wrapper.allowedTokens(rsETH)) revert TokenNotAllowedInWrapper();
        if (rsETHAmount == 0) revert InvalidAmount();
        if (rsETHAmount > wrapper.maxAmountToDepositBridgerAsset(rsETH)) revert ExceedsMaxAmountToDepositInWrapper();

        // Get the amount of token to transfer to the user for the given amount of rsETH provided
        uint256 tokenAmount = viewSwapAssetToPremintedRsETH(token, rsETHAmount);

        // Transfer rsETH from sender to the wrapper
        IERC20(rsETH).safeTransferFrom(msg.sender, address(wrapper), rsETHAmount);

        // Transfer the token from the pool to the sender
        if (token == ETH_IDENTIFIER) {
            if (getETHBalanceMinusFees() < tokenAmount) revert InsufficientETHBalanceForReverseSwap();
            (bool success,) = payable(msg.sender).call{ value: tokenAmount }("");
            if (!success) revert TransferFailed();
        } else {
            if (getTokenBalanceMinusFees(token) < tokenAmount) revert InsufficientAssetBalanceForReverseSwap();
            tokenContract.safeTransfer(msg.sender, tokenAmount);
        }

        emit ReverseSwapOccurred(msg.sender, rsETH, token, rsETHAmount, tokenAmount);
    }
```
