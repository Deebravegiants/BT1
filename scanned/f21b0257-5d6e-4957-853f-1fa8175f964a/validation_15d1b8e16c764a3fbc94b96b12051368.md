### Title
`LRTDepositPool._removeNodeDelegatorContractFromQueue` Can Be Permanently Blocked by Donating Supported LST Tokens to NodeDelegator — (`contracts/LRTDepositPool.sol`)

---

### Summary

`LRTDepositPool._removeNodeDelegatorContractFromQueue` enforces a residual-balance check before dequeuing a `NodeDelegator`. Because this check reads the raw ERC-20 `balanceOf` the `NodeDelegator` address, any unprivileged attacker can permanently block the removal of any `NodeDelegator` by donating a trivial amount of a supported LST token directly to that address.

---

### Finding Description

`_removeNodeDelegatorContractFromQueue` calls `_checkResidueLSTBalance` before it removes a `NodeDelegator` from the queue: [1](#0-0) 

`_checkResidueLSTBalance` iterates over every supported asset and reverts if the combined balance exceeds `maxNegligibleAmount`: [2](#0-1) 

The first term of `assetBalance` is a plain `IERC20.balanceOf(nodeDelegatorAddress)` call: [3](#0-2) 

Because standard LST tokens (stETH, cbETH, rETH, etc.) are freely transferable, any attacker can call `transfer(nodeDelegatorAddress, amount)` on any supported LST to inflate the raw balance of the `NodeDelegator` above `maxNegligibleAmount`. The revert at line 643 then fires unconditionally: [4](#0-3) 

`maxNegligibleAmount` is a storage variable initialized to zero: [5](#0-4) 

With `maxNegligibleAmount == 0`, even a single wei of any supported LST is sufficient to trigger the revert. Even if the value is raised, the attacker can always donate enough to exceed it.

An identical griefing path exists through the ETH balance branch of `_checkResidueEthBalance`: [6](#0-5) 

An attacker can force-send ETH via a `selfdestruct` call to push `address(nodeDelegatorAddress).balance` above `maxNegligibleAmount`.

---

### Impact Explanation

`removeNodeDelegatorContractFromQueue` is the only mechanism to decommission a `NodeDelegator`. If it is permanently blocked, the protocol cannot retire a compromised, misbehaving, or deprecated `NodeDelegator`. The `NodeDelegator` remains in the active queue indefinitely, and the protocol cannot migrate its EigenLayer delegation away from it. This constitutes a **permanent freezing of the management/migration path** for that `NodeDelegator` and all funds it holds.

**Severity: Medium** — Temporary (griefable to permanent) freezing of a critical protocol management operation.

---

### Likelihood Explanation

The attack requires only a standard ERC-20 `transfer` call with a dust amount of any supported LST token. No special permissions, flash loans, or complex setup are needed. Any unprivileged external actor can execute this at negligible cost. The attacker can front-run any admin recovery attempt to maintain the block indefinitely.

---

### Recommendation

Replace the raw `balanceOf` check with a tracked internal accounting variable (similar to how `getAssetBalance` and `getAssetUnstaking` already track EigenLayer-side balances). Only count tokens that entered the `NodeDelegator` through protocol-controlled deposit paths, not the raw on-chain balance. Alternatively, add a permissioned `sweepDust` function on `NodeDelegator` that allows the manager to transfer any unaccounted token balance back to the deposit pool before removal, and ensure this sweep cannot be front-run by making the removal atomic with the sweep.

---

### Proof of Concept

1. Protocol admin initiates removal of `NodeDelegator` at address `NDC`.
2. Attacker observes the pending `removeNodeDelegatorContractFromQueue(NDC)` transaction in the mempool.
3. Attacker front-runs with: `stETH.transfer(NDC, 1)` (1 wei of stETH, costing ~$0).
4. `_checkResidueLSTBalance(NDC)` computes `assetBalance = 1 + 0 + 0 = 1 > maxNegligibleAmount (0)` and reverts with `NodeDelegatorHasAssetBalance`.
5. The admin's removal transaction reverts. The `NodeDelegator` remains in the queue permanently.
6. Any subsequent removal attempt can be front-run identically, making the block permanent at negligible cost.

```text
    function _removeNodeDelegatorContractFromQueue(address nodeDelegatorAddress) internal {
        // 1. check if node delegator contract is in queue and find Index
        uint256 ndcIndex = _getNDCIndex(nodeDelegatorAddress);

        // 2. revert if node delegator contract has any asset balances.
        // 2.1 check if NDC has native ETH balance in eigen layer or/and in itself.
        _checkResidueEthBalance(nodeDelegatorAddress);
        // 2.2  check if NDC has LST balance
        _checkResidueLSTBalance(nodeDelegatorAddress);

        // 3. remove node delegator contract from queue
        // 3.1 remove from isNodeDelegator mapping
        isNodeDelegator[nodeDelegatorAddress] = 0;
        // 3.2 remove from nodeDelegatorQueue
        nodeDelegatorQueue[ndcIndex] = nodeDelegatorQueue[nodeDelegatorQueue.length - 1];
        nodeDelegatorQueue.pop();

        emit NodeDelegatorRemovedFromQueue(nodeDelegatorAddress);
    }
```

```text
    function _checkResidueEthBalance(address nodeDelegatorAddress) internal view {
        if (

                || address(nodeDelegatorAddress).balance > maxNegligibleAmount
                || INodeDelegator(nodeDelegatorAddress).getAssetUnstaking(LRTConstants.ETH_TOKEN) > 0

            revert NodeDelegatorHasETH();
        }
```

```text
    function _checkResidueLSTBalance(address nodeDelegatorAddress) internal view {
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetsLength = supportedAssets.length;

        uint256 assetBalance;
        for (uint256 i; i < supportedAssetsLength; ++i) {
            if (supportedAssets[i] == LRTConstants.ETH_TOKEN) {
                // this function only checks for residual LST balance
                continue;
            }

            assetBalance = IERC20(supportedAssets[i]).balanceOf(nodeDelegatorAddress)
                + INodeDelegator(nodeDelegatorAddress).getAssetBalance(supportedAssets[i]);
            assetBalance += INodeDelegator(nodeDelegatorAddress).getAssetUnstaking(supportedAssets[i]);

            if (assetBalance > maxNegligibleAmount) {
                revert NodeDelegatorHasAssetBalance(supportedAssets[i], assetBalance);
            }
        }
```
