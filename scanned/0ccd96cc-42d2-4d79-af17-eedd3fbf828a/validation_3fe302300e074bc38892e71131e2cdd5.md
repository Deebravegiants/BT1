### Title
Griefing via Direct Token Transfer Permanently Blocks `removeSupportedToken` - (File: contracts/pools/RSETHPoolNoWrapper.sol)

### Summary
Multiple pool contracts check that their token balance is exactly zero before allowing a supported token to be removed. Because any ERC-20 token can be transferred directly to a contract address by anyone, an unprivileged attacker can permanently prevent the `TIMELOCK_ROLE` from ever removing a supported token by sending as little as 1 wei of that token to the pool.

### Finding Description
`removeSupportedToken` in `RSETHPoolNoWrapper`, `RSETHPool`, `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, and `RSETHPoolV3WithNativeChainBridge` all share the same guard:

```solidity
// contracts/pools/RSETHPoolNoWrapper.sol  line 599
if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();
``` [1](#0-0) [2](#0-1) 

The check is intended to ensure no user funds are stranded when a token is delisted. However, `balanceOf(address(this))` reflects the *total* on-chain balance of the contract, including any tokens sent directly (i.e., not through the `deposit` flow). Because ERC-20 `transfer` / `transferFrom` cannot be blocked by the receiving contract, any external party can trivially keep the balance non-zero at negligible cost.

Even if the operator first calls `bridgeTokens` or `moveAssetsForBridging` to drain the legitimate balance to zero, the attacker can front-run (or simply re-send after the drain) to restore a non-zero balance, making the removal permanently impossible.

### Impact Explanation
The `removeSupportedToken` function is the only mechanism to delist a token. If it is permanently blocked:

- A token whose oracle becomes stale or manipulated cannot be removed; users continue to receive rsETH priced against a bad rate, leading to protocol insolvency or theft of rsETH reserves.
- A token whose bridge is compromised cannot be delisted; deposits keep flowing into a broken bridge path.
- The protocol loses the ability to respond to emergencies involving any supported LST/token.

This maps to **Medium – Temporary (effectively permanent) freezing of funds / contract fails to deliver promised governance capability**, with a realistic path to higher impact if the blocked token's oracle or bridge is later exploited.

### Likelihood Explanation
- Requires no privilege whatsoever — any EOA or contract can call `IERC20(token).transfer(poolAddress, 1)`.
- Cost is 1 wei of the target token plus gas.
- The attacker can repeat the transfer every block, making any admin drain-then-remove attempt trivially defeated.
- All five pool variants are affected, broadening the attack surface.

### Recommendation
Remove the strict zero-balance guard from `removeSupportedToken`. Instead, transfer any remaining balance to a designated recovery address (or to the caller) before deleting the token's metadata, or track only the *protocol-owned* balance (i.e., `balanceOf(this) - feeEarnedInToken[token]`) and allow removal when that tracked amount is zero. This mirrors the pattern already used in `getTokenBalanceMinusFees`. [3](#0-2) 

### Proof of Concept
1. Protocol deploys `RSETHPoolNoWrapper`; `wstETH` is added as a supported token.
2. Attacker calls `wstETH.transfer(address(pool), 1)` — costs ~1 wei + gas.
3. Admin (TIMELOCK_ROLE) calls `pool.removeSupportedToken(wstETH, idx)`.
4. Line 599 evaluates `IERC20(wstETH).balanceOf(address(pool)) != 0` → `true` → reverts with `TokenBalanceNotZero`.
5. Admin drains the pool via `bridgeTokens`; attacker immediately re-sends 1 wei.
6. Step 3 reverts again. The token can never be removed. [4](#0-3)

    function removeSupportedToken(address token, uint256 tokenIndex) external onlyRole(TIMELOCK_ROLE) {
        UtilLib.checkNonZeroAddress(token);
        if (supportedTokenList[tokenIndex] != token) revert TokenNotFoundError();
        if (IERC20(token).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();

        delete supportedTokenOracle[token];
        delete tokenBridge[token];
        supportedTokenList[tokenIndex] = supportedTokenList[supportedTokenList.length - 1];
        supportedTokenList.pop();
        emit RemovedSupportedToken(token);
    }
