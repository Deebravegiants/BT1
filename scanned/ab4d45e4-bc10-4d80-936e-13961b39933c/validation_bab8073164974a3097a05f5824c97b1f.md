Looking at the vulnerability class from H-7 — **any caller can act on another user's position without ownership verification, causing yield theft** — I need to find an analog in LRT-rsETH where an unprivileged caller can trigger an action on another user's account, resulting in loss of yield.

Let me examine the `MerkleDistributor` and `KernelMerkleDistributor` contracts closely, as they handle reward/token distribution.

From the search results, I can see:

- `KernelMerkleDistributor._processClaim` (line 311) explicitly checks `if (account != msg.sender) { revert Unauthorized(); }` — **protected**.
- `KernelTop100MerkleDistributor.claim` uses `address user = msg.sender` directly — **protected**.
- `MerkleDistributor.claim` (lines 97–147) takes an `account` parameter with **no `msg.sender == account` check** — **vulnerable**.

---

### Title
Any Caller Can Force-Claim Another User's Tokens, Causing Permanent Fee Loss — (File: `contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

### Summary

`MerkleDistributor.claim` accepts an arbitrary `account` parameter without verifying `msg.sender == account`. Any unprivileged caller who possesses a valid merkle proof for a victim can trigger the victim's claim. When `feeInBPS > 0`, the fee is deducted from the victim's claimable amount and permanently sent to the protocol treasury, stealing a portion of the victim's unclaimed yield.

### Finding Description

The `claim` function accepts an `account` parameter but never enforces that the caller is the beneficiary:

```solidity
// contracts/utils/MerkleDistributor/MerkleDistributor.sol lines 97–147
function claim(
    uint256 index,
    address account,          // ← arbitrary, no msg.sender check
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
    // ... validation ...
    uint256 fee = (claimableAmount * feeInBPS) / 10_000;
    uint256 amountToSend = claimableAmount - fee;

    IERC20(token).safeTransfer(account, amountToSend);       // victim gets reduced amount
    IERC20(token).safeTransfer(protocolTreasury, fee);       // fee permanently lost
}
``` [1](#0-0) 

The merkle proof inputs (`index`, `account`, `cumulativeAmount`, `merkleProof`) are all derived from the publicly available off-chain merkle tree. Any attacker can reconstruct them for any victim and call `claim` on their behalf.

The fee can be up to 10% (`MAX_FEE_IN_BPS = 1000`): [2](#0-1) 

This is structurally identical to H-7: just as any user could withdraw an NFT from another user's `Depositor` (manipulating their position without consent), here any user can trigger a claim for another user's `account` (consuming their claimable slot with a fee deduction they did not authorize).

The contrast with the newer `KernelMerkleDistributor` is telling — it explicitly guards against exactly this pattern:

```solidity
// contracts/KERNEL/KernelMerkleDistributor.sol line 311
if (account != msg.sender) {
    revert Unauthorized();
}
``` [3](#0-2) 

The generic `MerkleDistributor` was never updated with this guard.

### Impact Explanation

**High — Theft of unclaimed yield.**

When `feeInBPS > 0`, an attacker force-claiming for a victim causes the fee portion to be permanently redirected to the protocol treasury instead of the victim. At the maximum fee of 10%, a victim with 1,000 tokens claimable loses 100 tokens they were entitled to. The claim slot is then marked used, so the victim cannot re-claim the lost fee. The loss is irreversible. [4](#0-3) 

### Likelihood Explanation

**High.** All inputs required to call `claim` on behalf of a victim — `index`, `account`, `cumulativeAmount`, and `merkleProof` — are derived from the publicly posted off-chain merkle tree. No privileged access, leaked keys, or oracle manipulation is required. Any EOA can execute this attack permissionlessly against any eligible claimant.

### Recommendation

Add a caller-identity check identical to the one already present in `KernelMerkleDistributor._processClaim`:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
+   if (account != msg.sender) revert Unauthorized();
    // ... rest of logic
}
``` [5](#0-4) 

### Proof of Concept

1. Alice is entitled to 1,000 tokens in `MerkleDistributor`; `feeInBPS = 1000` (10%).
2. Bob reads Alice's `(index, cumulativeAmount, merkleProof)` from the public merkle tree off-chain.
3. Bob calls `MerkleDistributor.claim(index, alice, 1000, merkleProof)`.
4. The contract deducts `fee = 100` tokens → sends to `protocolTreasury`.
5. Alice receives only 900 tokens; her claim slot is marked used.
6. Alice has permanently lost 100 tokens of yield she was entitled to, with no recourse. [6](#0-5)

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
