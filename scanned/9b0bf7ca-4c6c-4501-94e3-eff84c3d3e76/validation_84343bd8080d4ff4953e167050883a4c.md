### Title
Unprivileged ETH Injection via Unguarded `receiveFrom*` Functions Triggers Spurious Fee rsETH Minting Against Non-Genuine Rewards — (`contracts/LRTDepositPool.sol`)

---

### Summary

`receiveFromRewardReceiver()`, `receiveFromLRTConverter()`, and `receiveFromNodeDelegator()` in `LRTDepositPool` carry no caller access control. Any external account can call them with arbitrary `msg.value`, inflating `address(this).balance`. Because `getETHDistributionData()` reads `address(this).balance` directly, and `_updateRsETHPrice()` uses that figure to compute `totalETHInProtocol`, the oracle treats the injected ETH as genuine protocol yield and mints fee rsETH to the treasury against it. Existing rsETH holders receive proportionally less yield than the protocol promises.

---

### Finding Description

**Step 1 — No access control on the receive functions** [1](#0-0) 

All three functions are bare `external payable` with no modifier, no role check, and no caller validation. Any EOA or contract can call them with any `msg.value`.

**Step 2 — `getETHDistributionData` reads raw contract balance** [2](#0-1) 

`ethLyingInDepositPool = address(this).balance` — injected ETH is immediately reflected here.

**Step 3 — `_getTotalEthInProtocol` aggregates the inflated balance** [3](#0-2) 

For the ETH asset, `getTotalAssetDeposits` → `getAssetDistributionData` → `getETHDistributionData`, so the injected ETH flows directly into `totalETHInProtocol`.

**Step 4 — Fee is computed on the injected ETH as if it were a genuine reward** [4](#0-3) 

`rewardAmount = totalETHInProtocol - previousTVL` includes the injected ETH. `protocolFeeInETH` is a percentage of that amount.

**Step 5 — Fee rsETH is minted to the treasury** [5](#0-4) 

The treasury receives rsETH minted against externally injected ETH, not genuine protocol-earned rewards.

**Partial mitigations that do not prevent the attack:**

- `pricePercentageLimit` reverts `updateRSETHPrice()` if the price increase exceeds the configured threshold — but only for large injections; small injections within the limit succeed. [6](#0-5) 
- `maxFeeMintAmountPerDay` caps total daily fee minting but does not prevent the spurious fee from being taken within that cap. [7](#0-6) 

---

### Impact Explanation

Existing rsETH holders receive less yield than the protocol promises. When fee rsETH is minted to the treasury against externally injected ETH, the new rsETH price is computed as `(totalETHInProtocol - protocolFeeInETH) / rsethSupply`. Holders' share of the injected ETH is reduced by the fee percentage, even though the injected ETH was not a protocol-earned reward. The protocol does not lose value (the injected ETH remains in the pool), but holders receive less yield than they are entitled to. This matches **Low — Contract fails to deliver promised returns, but doesn't lose value**.

---

### Likelihood Explanation

The attack requires only a public call with ETH attached — no role, no approval, no special state. The economic cost to the attacker is the injected ETH itself (which is donated to the pool). A griever willing to spend ETH can repeatedly trigger this. Likelihood is **Medium** given the direct, permissionless path and the concrete economic incentive for a competitor or griever.

---

### Recommendation

Add caller validation to all three receive functions. Only the designated `FeeReceiver`/`RewardReceiver`, `LRTConverter`, and registered `NodeDelegator` contracts should be permitted to call them:

```solidity
function receiveFromRewardReceiver() external payable {
    if (msg.sender != lrtConfig.getContract(LRTConstants.LRT_REWARD_RECEIVER))
        revert InvalidCaller();
}

function receiveFromLRTConverter() external payable {
    if (msg.sender != lrtConfig.getContract(LRTConstants.LRT_CONVERTER))
        revert InvalidCaller();
}

function receiveFromNodeDelegator() external payable {
    if (isNodeDelegator[msg.sender] == 0)
        revert InvalidCaller();
}
```

### Proof of Concept

```solidity
// Foundry test (local fork or mock setup)
function test_spuriousFeeMintViaUnguardedReceive() public {
    // Baseline: record treasury rsETH balance and current rsETH price
    uint256 treasuryBalanceBefore = rsETH.balanceOf(treasury);
    uint256 priceBefore = lrtOracle.rsETHPrice();

    // Attacker injects ETH via the unguarded function
    vm.deal(attacker, 10 ether);
    vm.prank(attacker);
    lrtDepositPool.receiveFromRewardReceiver{value: 10 ether}();

    // Trigger price update (public, no role required)
    lrtOracle.updateRSETHPrice();

    // Assert treasury received fee rsETH minted against the injected ETH
    uint256 treasuryBalanceAfter = rsETH.balanceOf(treasury);
    assertGt(treasuryBalanceAfter, treasuryBalanceBefore, "treasury gained fee rsETH");

    // Assert price increased (injected ETH counted as TVL)
    uint256 priceAfter = lrtOracle.rsETHPrice();
    assertGt(priceAfter, priceBefore, "price inflated by injected ETH");

    // The fee taken reduces what existing holders would have received
    // without the fee mechanism being triggered on non-genuine rewards
}
```
