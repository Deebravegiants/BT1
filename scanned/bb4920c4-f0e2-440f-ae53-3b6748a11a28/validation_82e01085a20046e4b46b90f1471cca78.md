### Title
Unprivileged Caller Can Trigger Protocol-Wide Auto-Pause via `updateRSETHPrice()` During Transient LST Price Dip — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a `public` function with no role check. Its internal implementation `_updateRsETHPrice()` contains a downside-protection branch that pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself whenever the computed `newRsETHPrice` falls below `highestRsethPrice` by more than `pricePercentageLimit`. Any unprivileged address can call `updateRSETHPrice()` at a moment when a transient LST price dip (e.g., stETH temporarily depegs) satisfies the threshold condition, causing a protocol-wide pause that freezes all deposits and withdrawals until an admin manually unpauses.

---

### Finding Description

**Entry point — no access control:** [1](#0-0) 

`updateRSETHPrice()` carries only the `whenNotPaused` modifier. Any EOA or contract can call it at any time.

**Auto-pause branch — triggered by price drop:** [2](#0-1) 

When `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit.mulWad(highestRsethPrice)`, the function immediately pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself, then returns. There is no role check on the caller before this branch executes.

**`highestRsethPrice` is the all-time high:** [3](#0-2) 

`highestRsethPrice` is only ever updated upward. Any transient dip below the historical peak by more than `pricePercentageLimit` (e.g., 1% = `1e16`) satisfies the condition.

**Price is read from external oracles:** [4](#0-3) 

`_getTotalEthInProtocol()` reads live asset prices from `IPriceFetcher` oracles (e.g., Chainlink). A transient stETH depeg or any LST price dip that moves the computed rsETH price below the historical peak by the threshold is sufficient.

**Attack path:**
1. Attacker monitors on-chain oracle prices for supported LSTs.
2. When a transient depeg causes `newRsETHPrice` to fall below `highestRsethPrice` by more than `pricePercentageLimit`, attacker calls `updateRSETHPrice()` from any address.
3. `_updateRsETHPrice()` computes the dip, enters the `isPriceDecreaseOffLimit` branch, and calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()`.
4. All deposits and withdrawals are frozen. Only an admin can unpause.

**Contrast with the upside branch:** [5](#0-4) 

For price *increases* above the threshold, the code reverts for non-managers. For price *decreases*, there is no such caller check — the pause executes unconditionally regardless of who called.

---

### Impact Explanation

All user deposits (`LRTDepositPool`) and all withdrawals (`LRTWithdrawalManager`) are frozen until an admin calls `unpause()`. This matches **Medium — Temporary freezing of funds**. The freeze duration depends on admin response time and is not bounded by the contract.

---

### Likelihood Explanation

- stETH has historically depegged (e.g., May 2022 LUNA crash: ~6% depeg). With `pricePercentageLimit = 1e16` (1%), even minor transient dips qualify.
- The attacker needs no capital, no special role, and no oracle manipulation — only the ability to observe oracle prices and submit a transaction at the right block.
- The attack is repeatable: after admin unpauses, the attacker can trigger it again during the next transient dip.

---

### Recommendation

Add a role check to `updateRSETHPrice()` (restrict to `MANAGER` or `OPERATOR_ROLE`), or separate the auto-pause logic so that it can only be triggered by a privileged caller. Alternatively, introduce a cooldown or require the price to be sustained below the threshold for multiple consecutive updates before pausing.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

// Foundry fork test (local fork, no public mainnet)
// Setup: deploy LRTOracle, LRTDepositPool, LRTWithdrawalManager with mocks.
// Set pricePercentageLimit = 1e16 (1%).
// Set highestRsethPrice = 1.05e18 (simulate prior ATH).
// Mock oracle returns assetPrice = 1.03e18 (transient ~1.9% dip from ATH).

function test_unprivilegedCallerTriggersAutoPause() public {
    // Arrange: price dip satisfies threshold
    mockOracle.setPrice(stETH, 1.03e18); // dip from ATH of 1.05e18 => ~1.9% > 1%
    
    // Act: call as unprivileged address
    vm.prank(address(1)); // no role
    lrtOracle.updateRSETHPrice();
    
    // Assert: protocol is paused
    assertTrue(lrtDepositPool.paused(),       "DepositPool should be paused");
    assertTrue(withdrawalManager.paused(),    "WithdrawalManager should be paused");
    assertTrue(lrtOracle.paused(),            "LRTOracle should be paused");
}
```
