### Title
Unrestricted `updateRSETHPrice()` Allows Any Caller to Trigger Protocol Fee Minting and Dilute rsETH Holders — (File: `contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is declared `public whenNotPaused` with no role restriction, allowing any unprivileged address to invoke the full fee-minting and price-update logic at will. Each call can mint rsETH fee tokens to the treasury (diluting all rsETH holders' unclaimed yield) and, under adverse price conditions, trigger a protocol-wide pause that freezes deposits and withdrawals. A privileged sibling function `updateRSETHPriceAsManager()` exists with `onlyLRTManager`, confirming the protocol intended access control on this path but failed to apply it to the public entry point.

---

### Finding Description

`LRTOracle.sol` exposes two entry points into `_updateRsETHPrice()`:

```solidity
// Line 87 — no role check, callable by anyone
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}

// Line 94 — correctly restricted
function updateRSETHPriceAsManager() external onlyLRTManager {
    _updateRsETHPrice();
}
```

`_updateRsETHPrice()` performs three security-sensitive operations on every call:

**1. Protocol fee minting (yield extraction)**
```solidity
// Lines 244–246
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
// Lines 301–307
uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);
_checkAndUpdateDailyFeeMintLimit(rsethAmountToMintAsProtocolFee);
IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
```

Any caller can trigger this whenever `totalETHInProtocol > previousTVL` (i.e., whenever rewards have accrued). The minted rsETH dilutes all existing holders by transferring a share of their accrued yield to the treasury.

**2. Protocol-wide pause trigger**
```solidity
// Lines 273–281
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

If the computed price falls below `highestRsethPrice * (1 - pricePercentageLimit)`, the function pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. Any caller can trigger this whenever on-chain conditions (e.g., a slashing event or temporary oracle dip) produce a qualifying price drop.

**3. rsETH price update**
```solidity
// Line 313
rsETHPrice = newRsETHPrice;
```

`rsETHPrice` is the denominator in `LRTDepositPool.getRsETHAmountToMint()` (line 520), so any caller can force a price snapshot that immediately affects how many rsETH tokens depositors receive.

---

### Impact Explanation

**Primary — High: Theft of unclaimed yield.**
Rewards that accrue in the protocol (ETH staking rewards, EigenLayer rewards) are reflected as an increase in `totalETHInProtocol`. Until `updateRSETHPrice()` is called, this yield belongs to rsETH holders as price appreciation. Any caller can invoke `updateRSETHPrice()` to force fee extraction: a portion of that yield is minted as rsETH to the treasury, permanently diluting holders. An attacker can time calls to coincide with large reward accruals (e.g., immediately after a validator reward sweep) to maximize the fee extracted per call, up to the `maxFeeMintAmountPerDay` ceiling. Across multiple daily periods, this allows systematic extraction of the maximum daily fee by any external actor, not just the authorized manager.

**Secondary — Medium: Temporary freezing of funds.**
If on-chain conditions produce a price drop exceeding `pricePercentageLimit`, any caller can trigger a full protocol pause by calling `updateRSETHPrice()`. This freezes all deposits and withdrawals until an admin calls `unpause()`, constituting a temporary freeze of user funds.

---

### Likelihood Explanation

The entry point is `public` with no authentication. Any EOA or contract can call it at zero cost beyond gas. No special role, token balance, or privileged position is required. The fee-minting path is triggered whenever rewards have accrued since the last call, which is a routine on-chain condition. Likelihood is **High**.

---

### Recommendation

Apply the same access control used on `updateRSETHPriceAsManager()` to `updateRSETHPrice()`, or introduce a keeper/bot role that is the only address permitted to call the public price update. If permissionless price updates are desired for liveness, separate the fee-minting logic from the price-read path so that fee extraction requires an authorized caller while the price snapshot remains publicly readable.

---

### Proof of Concept

1. Rewards accrue in the protocol (e.g., EigenLayer strategy yields increase `totalETHInProtocol` above `rsethSupply * rsETHPrice`).
2. Attacker (any EOA) calls `LRTOracle.updateRSETHPrice()`.
3. `_updateRsETHPrice()` computes `rewardAmount = totalETHInProtocol - previousTVL > 0`.
4. `protocolFeeInETH = rewardAmount * protocolFeeInBPS / 10_000` is computed.
5. `rsethAmountToMintAsProtocolFee = protocolFeeInETH / newRsETHPrice` rsETH is minted to the treasury.
6. All existing rsETH holders are diluted; their share of the underlying ETH decreases.
