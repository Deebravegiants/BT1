### Title
Missing Staleness and Validity Checks on Chainlink `latestRoundData()` Return Values - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards all validation fields (`roundId`, `updatedAt`, `answeredInRound`). A stale or zero price propagates directly into `LRTOracle._updateRsETHPrice()`, which is publicly callable, allowing any actor to lock in a manipulated rsETH price that governs every subsequent deposit and withdrawal.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches the LST/ETH price as follows:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values of `latestRoundData()` are available, but four are thrown away with no checks:

| Field | Checked? |
|---|---|
| `roundId` | No |
| `price` | No (`price <= 0` not rejected) |
| `startedAt` | No |
| `updatedAt` | No (no heartbeat / staleness window) |
| `answeredInRound` | No (`answeredInRound < roundId` not rejected) |

Contrast this with `ChainlinkOracleForRSETHPoolCollateral.getRate()`, which correctly validates all three conditions before using the price:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The unvalidated price from `ChainlinkPriceOracle` flows into `LRTOracle._getTotalEthInProtocol()`, which sums `assetPrice × totalAssetDeposits` for every supported LST. That sum is then used by `_updateRsETHPrice()` to set the global `rsETHPrice` storage variable, which governs how many rsETH tokens are minted per unit of deposited asset.

`updateRSETHPrice()` is a **public, permissionless function** — any externally owned account can call it at any time. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

---

### Impact Explanation

**Impact: High — Theft of unclaimed yield / share mis-accounting**

Two concrete scenarios:

1. **Inflated stale price (asset price has fallen, feed not yet updated):** `_getTotalEthInProtocol()` overstates TVL → `rsETHPrice` is set too high → new depositors receive fewer rsETH tokens than they deserve. Existing holders are not harmed, but depositors are systematically shortchanged.

2. **Deflated stale price (asset price has risen, feed not yet updated):** TVL is understated → `rsETHPrice` is set too low → new depositors receive more rsETH than they should → dilution of existing holders' share of the pool, equivalent to theft of accrued yield.

In either case, the attacker's lever is the public `updateRSETHPrice()` call, which they can time to coincide with a known stale feed window (e.g., immediately after a Chainlink heartbeat miss or during high gas conditions that delay feed updates). [5](#0-4) [6](#0-5) 

---

### Likelihood Explanation

**Likelihood: Low**

Chainlink feeds do occasionally go stale — during network congestion, sequencer downtime on L2 deployments, or when a feed is deprecated. The attacker does not need to cause the staleness; they only need to observe it and call `updateRSETHPrice()` at the right moment. No privileged access is required. The window is narrow but real and has been exploited in other protocols. [3](#0-2) 

---

### Recommendation

Apply the same validation pattern already used in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
uint256 private constant STALENESS_THRESHOLD = 3600; // 1 hour; tune per feed heartbeat

function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (price <= 0) revert InvalidPrice();
    if (answeredInRound < roundId) revert StalePrice();
    if (block.timestamp > updatedAt + STALENESS_THRESHOLD) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, apply equivalent validation inside `RSETHPriceFeed.latestRoundData()`, which also discards all validation fields from its inner `ETH_TO_USD.latestRoundData()` call. [1](#0-0) [7](#0-6) 

---

### Proof of Concept

1. Chainlink's ETH/stETH (or any supported LST/ETH) feed misses its heartbeat update — `updatedAt` is now `block.timestamp - 2 hours` while the true price has moved 2%.

2. Attacker observes the stale feed on-chain.

3. Attacker calls `LRTOracle.updateRSETHPrice()` (no access control).

4. Inside `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice()` returns the stale price with no revert.

5. `rsETHPrice` is written to storage with the stale-price-derived value.

6. Any user who calls `LRTDepositPool.depositAsset()` immediately after receives rsETH minted at the incorrect price, either over- or under-minting relative to the true asset value.

7. When the feed updates and `updateRSETHPrice()` is called again, the price corrects — but the damage (dilution or shortchanging) is already recorded in the minted rsETH supply. [3](#0-2) [1](#0-0) [8](#0-7)

    function _updateRsETHPrice() internal {
        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;
        }

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

        // only take fee if TVL increased and protocol is not paused
        uint256 protocolFeeInETH = 0;
        if (!protocolPaused && totalETHInProtocol > previousTVL) {
            uint256 rewardAmount = totalETHInProtocol - previousTVL;
            protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
        }

        // compute new rsETH price based on total ETH minus fee
        uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
