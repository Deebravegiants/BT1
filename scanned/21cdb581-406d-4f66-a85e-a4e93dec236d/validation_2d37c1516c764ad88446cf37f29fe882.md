### Title
Any Caller Can Force Fee Deduction on Another User's Merkle Claim - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

### Summary
`MerkleDistributor.claim()` accepts an `account` parameter but never verifies that `msg.sender == account`. Any unprivileged caller can trigger a claim on behalf of any other user, causing the protocol fee to be deducted from that user's claimable token amount without their consent. The analog to the zNS bug is exact: a check passes (valid merkle proof for `account`) but the function does not verify the caller has authority over the specific resource being acted upon (the `account`'s pending claim).

### Finding Description

`MerkleDistributor.claim()` is a public function that accepts an arbitrary `account` address:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
    ...
    uint256 fee = (claimableAmount * feeInBPS) / 10_000;
    uint256 amountToSend = claimableAmount - fee;

    IERC20(token).safeTransfer(account, amountToSend);
    IERC20(token).safeTransfer(protocolTreasury, fee);
    ...
}
```

There is no `msg.sender == account` guard. Any external caller who possesses a valid merkle proof for a victim's `account` (all proofs are public off-chain data) can call `claim()` on the victim's behalf, forcing the fee deduction. The victim receives `claimableAmount - fee` instead of `claimableAmount`, permanently losing up to `MAX_FEE_IN_BPS = 1000` basis points (10%) of their entitled tokens.

The developers recognized this exact risk in `KernelMerkleDistributor._processClaim()` and added the guard there:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

But the base `MerkleDistributor` contract is missing this check entirely.

The attacker-controlled entry path is:
1. Observe the merkle tree off-chain to find a victim `account` with a pending unclaimed balance.
2. Construct the valid merkle proof for `(index, account, cumulativeAmount)`.
3. Call `MerkleDistributor.claim(index, account, cumulativeAmount, merkleProof)` from any EOA.
4. The contract deducts `fee = claimableAmount * feeInBPS / 10_000` and sends it to `protocolTreasury`.
5. The victim receives only `claimableAmount - fee`, permanently losing the fee portion.

### Impact Explanation

The victim permanently loses up to 10% of their entitled token distribution. The fee is irrecoverably transferred to `protocolTreasury`; the victim cannot reclaim it. This constitutes **theft of unclaimed yield** — the user's pending reward allocation is reduced without their consent by an unprivileged third party. Severity: **High**.

### Likelihood Explanation

Likelihood is **high**. The attacker requires no privileged role, no capital, and no special on-chain state. Merkle proofs are public (derivable from the off-chain distribution data). The attack is a single transaction callable by any EOA. The only precondition is that the victim has not yet claimed.

### Recommendation

Add a caller ownership check identical to the one already present in `KernelMerkleDistributor`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Alternatively, replace the `account` parameter with `msg.sender` directly, as done in `KernelTop100MerkleDistributor.claim()`.

### Proof of Concept

1. Protocol deploys `MerkleDistributor` with `feeInBPS = 500` (5%).
2. Alice has `cumulativeAmount = 1000e18` tokens claimable at `index = 5`. Her proof is public.
3. Bob (unprivileged) calls `MerkleDistributor.claim(5, alice, 1000e18, aliceProof)`.
4. Contract computes `fee = 50e18`, sends `950e18` to Alice and `50e18` to `protocolTreasury`.
5. Alice permanently loses 50 tokens she was entitled to. Bob spent only gas. [1](#0-0) [2](#0-1) [3](#0-2)

```text
    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
