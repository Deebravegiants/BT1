### Title
Automatic Pause Triggered by Public `updateRSETHPrice()` Freezes Already-Unlocked Withdrawals — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is callable by any address. When the computed `newRsETHPrice` drops more than `pricePercentageLimit` below `highestRsethPrice`, the function automatically calls `withdrawalManager.pause()`. Because `LRTWithdrawalManager.completeWithdrawal` (and `instantWithdrawal`) carry `whenNotPaused`, users whose rsETH has already been burned during `unlockQueue` cannot retrieve the assets they are owed until an admin manually calls `unpause()`.

---

### Finding Description

**Entrypoint — public, no role check:**

`LRTOracle.updateRSETHPrice()` is `public whenNotPaused` with no role restriction. [1](#0-0) 

**Downside-protection branch — automatic pause:**

Inside `_updateRsETHPrice()`, when `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit × highestRsethPrice`, the function calls `withdrawalManager.pause()` (line 279) and `_pause()` on the oracle itself, then returns without updating `rsETHPrice`. [2](#0-1) 

**Price is derived from live on-chain state:**

`_getTotalEthInProtocol()` reads `ILRTDepositPool.getTotalAssetDeposits(asset)` for every supported asset. An EigenLayer slashing event that reduces NDC balances will lower this value, directly reducing `newRsETHPrice` on the next call to `updateRSETHPrice()`. [3](#0-2) 

**`completeWithdrawal` is gated by `whenNotPaused`:**

Once `withdrawalManager.pause()` succeeds, every user-facing withdrawal path reverts. [4](#0-3) [5](#0-4) 

**rsETH is burned before the pause, not after:**

In `unlockQueue`, rsETH is burned from the contract and assets are redeemed from the vault *before* any user calls `completeWithdrawal`. After `unlockQueue` runs, the user's rsETH is gone and the owed assets sit inside `LRTWithdrawalManager`. A subsequent pause leaves those users unable to claim assets they are already owed. [6](#0-5) 

---

### Impact Explanation

Users who have passed through `unlockQueue` (rsETH burned, assets redeemed into the contract) cannot call `completeWithdrawal` while the contract is paused. Their funds are frozen inside `LRTWithdrawalManager` until an admin calls `unpause()`. This is **temporary freezing of funds** — a Medium-severity impact per the allowed scope.

---

### Likelihood Explanation

- `updateRSETHPrice()` requires no privilege; any EOA or bot can call it at any time.
- EigenLayer slashing is a documented, realistic risk for LST-backed protocols.
- `pricePercentageLimit` is a finite threshold set by admin; a sufficiently large slashing event will cross it.
- No front-running or brute-force is required — the attacker simply calls the public function after slashing has already reduced `getTotalAssetDeposits`.

---

### Recommendation

Introduce a separate, non-paused path for completing *already-unlocked* withdrawals. One approach: add a `completeUnlockedWithdrawal` function that does **not** carry `whenNotPaused` but verifies `usersFirstWithdrawalRequestNonce < nextLockedNonce[asset]` (i.e., the request was unlocked before the pause). Alternatively, emit an event and allow `completeWithdrawal` to proceed for requests whose nonce is below `nextLockedNonce[asset]`, since those users' rsETH has already been burned and the assets are already in the contract.

---

### Proof of Concept

```solidity
// Fork test (Foundry) — no mainnet interaction required
function test_pauseFreezesUnlockedWithdrawals() public {
    // 1. User initiates withdrawal; operator calls unlockQueue
    //    → rsETH burned, assets redeemed into LRTWithdrawalManager
    vm.prank(operator);
    withdrawalManager.unlockQueue(asset, type(uint256).max, 0, type(uint256).max, 0, type(uint256).max);

    // 2. Simulate EigenLayer slashing: reduce NDC balance so
    //    getTotalAssetDeposits drops enough to push newRsETHPrice
    //    below (highestRsethPrice - pricePercentageLimit * highestRsethPrice)
    _simulateSlashing(slashAmount);

    // 3. Any unprivileged caller triggers the price update
    vm.prank(address(0xdead));
    lrtOracle.updateRSETHPrice();

    // 4. Withdrawal manager is now paused
    assertTrue(withdrawalManager.paused());

    // 5. User cannot complete their already-unlocked withdrawal
    vm.prank(user);
    vm.expectRevert("Pausable: paused");
    withdrawalManager.completeWithdrawal(asset, "");
}
```
