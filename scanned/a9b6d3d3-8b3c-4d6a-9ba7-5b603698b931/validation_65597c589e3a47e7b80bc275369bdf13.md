### Title
`CompressedOracleV1.fallback()` Push Calldata First 4 Bytes Not Reserved — Accidental Function-Selector Collision Silently Corrupts Oracle State - (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` uses a `fallback()` function as its primary price-push path. Pushers send N × 32-byte slot words directly as calldata. The first 4 bytes of the first word are not reserved, so they can accidentally match a function selector in the contract (or its inherited `OracleBase` / `AccessControl` parents), causing the EVM dispatcher to route the call to that function instead of the fallback. The most dangerous collision is with `revokePusher()`, which takes no arguments, succeeds silently, and permanently revokes the pusher's namespace delegation — leaving the oracle serving stale data to every pool that depends on it.

---

### Finding Description

`CompressedOracleV1` inherits from `OracleBase` (compressed variant) which inherits from OpenZeppelin `AccessControl`. The combined contract exposes roughly 30+ named functions. The EVM dispatch rule is unconditional: if `calldatasize() >= 4` and the first 4 bytes match any function selector, that function is called; the `fallback` is only reached when no selector matches.

The push path is:

```solidity
// CompressedOracle.sol line 311
fallback() override external {
    ...
    if (end == 0 || end % 32 != 0) revert BadCalldataLength();

    for (uint256 ptr = 0; ptr < end; ptr += 32) {
        uint256 word;
        assembly ("memory-safe") { word := calldataload(ptr) }
        uint8 slotId = uint8(word);          // LSB
        TimeMs timestampMs = toTimeMs(word >> 8 & X56);
        ...
    }
}
```

Each 32-byte word is laid out as:

```
[oracle0:6 bytes][oracle1:6 bytes][oracle2:6 bytes][oracle3:6 bytes][timestamp:7 bytes][slotId:1 byte]
```

The first 4 bytes of the first word are the high bits of `oracle0`'s compressed price/spread data. They are **not reserved** and can take any value depending on the live market price and spread indices being pushed.

The most impactful collision is with `revokePusher()`:

```solidity
// CompressedOracle.sol line 238
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

`revokePusher()` accepts no arguments — any calldata beyond the 4-byte selector is silently ignored by the ABI decoder. If the first 4 bytes of a push word match `keccak256("revokePusher()")[:4]`, the call:

1. Succeeds without reverting (assuming the pusher has an active delegation).
2. Sets `namespaceRemapping[pusher] = address(0)`, permanently revoking the delegation.
3. Does **not** update any oracle slot.
4. Emits `PusherRevoked`, which may go unnoticed in high-frequency push monitoring.

After the collision, all subsequent pushes from that wallet land in the pusher's **own** namespace instead of the creator's namespace. The creator's namespace receives no further updates and serves stale data indefinitely until the delegation is manually re-established.

Additional dangerous collisions exist with:

- **`acceptStateGuardRole(bytes32)`** — if the first 32 bytes decode to a `feedId` for which `pendingStateGuard[feedId] == msg.sender`, the pusher silently becomes the state guard for that feed, gaining the ability to set price guards and block legitimate guard transitions.
- **`allowPushers(uint256, address[], bytes[])`** / **`removePushers(address[])`** — could remap or remove pusher delegations if the calldata happens to ABI-decode correctly.
- Any view function (e.g., `getOracleData(bytes32)`) — causes a silent no-op revert, dropping the price update.

---

### Impact Explanation

The `CompressedOracleV1` is the price source for `MetricOmmPool`. During every swap, `_getBidAndAskPriceX64()` calls `IPriceProvider(activePriceProvider).getBidAndAskPrice()`, which ultimately reads from the oracle. If the oracle's namespace is stale (because the pusher's delegation was silently revoked), the pool executes swaps at an outdated bid/ask, causing:

- Traders to receive incorrect token amounts (bad-price execution).
- LPs to suffer losses from mispriced swaps against a stale oracle.
- The pool to remain in a degraded state until the off-chain operator detects the stale timestamp and re-authorizes the pusher.

This matches the allowed impact gate: **"Bad-price execution: stale, inverted, unbounded, or unclamped bid/ask quote reaches a pool swap."**

---

### Likelihood Explanation

With ~30 named selectors across `CompressedOracleV1`, `OracleBase`, and `AccessControl`, the per-push collision probability is approximately `30 / 2^32 ≈ 7 × 10^-9`. At a 5-second push cadence, a collision is expected roughly once every ~22 years for a single pusher. However:

- Multiple pushers multiply the exposure linearly.
- The `revokePusher()` collision is uniquely dangerous because it **succeeds silently** — no revert, no obvious on-chain signal beyond a `PusherRevoked` event — making it easy to miss in monitoring.
- Unlike WooFi (where the protocol can resend the failed tx), a `revokePusher()` collision permanently changes state; recovery requires a new `allowPushers` transaction with a fresh signed consent.

---

### Recommendation

Reserve the first 4 bytes of every push word as a fixed magic value (e.g., `0x00000000`) that does not collide with any function selector, and validate it at the top of `fallback()`:

```diff

    uint256 end;
    uint256 namespace;

    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0)) creator = msg.sender;

    assembly ("memory-safe") {
        end := calldatasize()
        namespace := shl(96, creator)
    }

+   if (end < 4 || end % 32 != 4) revert BadCalldataLength();

+   if (magic != 0x00000000) revert BadCalldataLength();

```

Alternatively, use a dedicated named function (e.g., `pushSlots(bytes calldata)`) instead of `fallback()`, eliminating the dispatch ambiguity entirely.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from
    "../../contracts/oracles/compressed/CompressedOracle.sol";

contract CompressedOracleSelectorCollisionTest is Test {
    CompressedOracleV1 oracle;
    address pusher = address(0xBEEF);
    address creator = address(0xCAFE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 60_000);

        // Creator authorizes pusher with a valid signature (simplified: pusher == creator for test)
        // In practice, use allowPushers with a real signature.
        // For the PoC, we test the collision directly.
    }

    function testCollisionWithRevokePusher() public {
        // selector of "revokePusher()": keccak256("revokePusher()")[:4]
        bytes4 sel = bytes4(keccak256("revokePusher()"));

        // Simulate a pusher that has an active delegation
        // (set up via vm.store or allowPushers in a real test)
        vm.store(
            address(oracle),
            keccak256(abi.encode(pusher, uint256(/* namespaceRemapping slot */ 0))),
            bytes32(uint256(uint160(creator)))
        );

        assertEq(oracle.namespaceRemapping(pusher), creator);

        // Craft a 32-byte push word whose first 4 bytes match revokePusher()
        bytes memory payload = abi.encodePacked(
            sel,                    // first 4 bytes = revokePusher() selector
            bytes28(0)              // remaining 28 bytes = arbitrary slot data
        );
        assertEq(payload.length, 32);

        vm.prank(pusher);
        (bool success,) = address(oracle).call(payload);

        // The call succeeds — revokePusher() was called instead of fallback
        assertTrue(success);
        // Pusher's delegation is now revoked
        assertEq(oracle.namespaceRemapping(pusher), address(0));
        // Oracle slot was NOT updated — stale data will be served to the pool
    }
}
```

```text
    ///         [data0:6][data1:6][data2:6][data3:6][ts:7][slotId:1]
    ///         The sender pushes into `namespaceRemapping[msg.sender]`, falling back to
    ///         its OWN namespace — a creator needs zero setup transactions to start
    ///         pushing. Each word carries its own timestamp (monotonicity is the only
    ///         freshness gate), so there is no deadline prefix.
    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);

            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
```
