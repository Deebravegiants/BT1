### Title
NodeDelegator Removal Permanently Blocked by Sending 1 Wei of ETH or LST Token - (File: contracts/LRTDepositPool.sol)

### Summary
An attacker can permanently block the removal of any `NodeDelegator` from the queue by sending 1 wei of ETH or a supported LST token to the `NodeDelegator` contract. This exploits the `_checkResidueEthBalance` and `_checkResidueLSTBalance` checks in `LRTDepositPool._removeNodeDelegatorContractFromQueue`, which revert if the `NodeDelegator` holds any residual assets above `maxNegligibleAmount` (default: 0).

### Finding Description
`_removeNodeDelegatorContractFromQueue` (called by both `removeNodeDelegatorContractFromQueue` and `removeManyNodeDelegatorContractsFromQueue`) performs two residue checks before allowing removal:

**Check 1 — ETH balance:** [1](#0-0) 

The condition `address(nodeDelegatorAddress).balance > maxNegligibleAmount` uses the raw ETH balance of the `NodeDelegator`. Because `NodeDelegator` exposes a public `receive()` function: [2](#0-1) 

any external account can send ETH to it without permission.

**Check 2 — LST balance:** [3](#0-2) 

`IERC20(supportedAssets[i]).balanceOf(nodeDelegatorAddress)` is the raw ERC20 balance. ERC20 transfers require no approval from the recipient, so any attacker can send 1 wei of stETH, ETHx, rETH, or sfrxETH directly to the `NodeDelegator`.

`maxNegligibleAmount` is a `uint256` storage variable initialized to zero by default: [4](#0-3) 

With `maxNegligibleAmount == 0`, a balance of even 1 wei satisfies `assetBalance > maxNegligibleAmount`, causing an immediate revert with `NodeDelegatorHasAssetBalance` or `NodeDelegatorHasETH`. The attacker can repeat the transfer after every operator sweep, making the block persistent.

The full removal flow that is blocked: [5](#0-4) 

### Impact Explanation
The admin cannot remove a `NodeDelegator` from the queue. If the `NodeDelegator` must be decommissioned (e.g., due to a security incident, operator key compromise, or migration), the protocol is unable to do so. Assets delegated through that `NodeDelegator` remain locked in EigenLayer strategies under an operator the protocol can no longer rotate away from, and the `nodeDelegatorQueue` slot cannot be reclaimed. This constitutes a **temporary (potentially indefinite) freezing of funds** routed through the targeted `NodeDelegator`, matching the "Medium: Temporary freezing of funds" impact tier.

### Likelihood Explanation
The attack is fully permissionless. The attacker needs only 1 wei of ETH (or any supported LST) and no approvals. It can be executed preemptively against every `NodeDelegator` in the queue at negligible cost, without needing to observe or frontrun any specific admin transaction. The attack is self-renewing: every time the operator sweeps the residual balance into EigenLayer, the attacker can re-send 1 wei.

### Recommendation
Replace raw `balanceOf` / `address.balance` checks with a pull-based sweep-then-check pattern: before the removal check, allow the operator to atomically transfer any residual balance to the `LRTDepositPool` or into an EigenLayer strategy, then verify the balance is zero. Alternatively, introduce a configurable `maxNegligibleAmount` that is set to a realistic dust threshold (e.g., 1000 wei) so that 1-wei griefing attacks are absorbed, and document that the admin must set this value on deployment.

### Proof of Concept
1. Protocol has `NodeDelegator` at address `NDC` with zero balance; admin prepares to call `removeNodeDelegatorContractFromQueue(NDC)`.
2. Attacker calls `(bool ok,) = payable(NDC).call{value: 1}("")` — costs 1 wei of ETH.
3. Admin's call reaches `_checkResidueEthBalance(NDC)`:
   - `address(NDC).balance == 1 > 0 == maxNegligibleAmount` → `revert NodeDelegatorHasETH()`.
4. Operator sweeps the 1 wei by calling `depositAssetIntoStrategy` or similar; attacker immediately re-sends 1 wei.
5. The `NodeDelegator` can never be removed as long as the attacker continues to send dust.

    function _removeNodeDelegatorContractFromQueue(address nodeDelegatorAddress) internal {
        // 1. check if node delegator contract is in queue and find Index
        uint256 ndcIndex = _getNDCIndex(nodeDelegatorAddress);

        // 2. revert if node delegator contract has any asset balances.
        // 2.1 check if NDC has native ETH balance in eigen layer or/and in itself.
        _checkResidueEthBalance(nodeDelegatorAddress);
        // 2.2  check if NDC has LST balance
        _checkResidueLSTBalance(nodeDelegatorAddress);

        // 3. remove node delegator contract from queue
        // 3.1 remove from isNodeDelegator mapping
        isNodeDelegator[nodeDelegatorAddress] = 0;
        // 3.2 remove from nodeDelegatorQueue
        nodeDelegatorQueue[ndcIndex] = nodeDelegatorQueue[nodeDelegatorQueue.length - 1];
        nodeDelegatorQueue.pop();

        emit NodeDelegatorRemovedFromQueue(nodeDelegatorAddress);
    }

    function _checkResidueEthBalance(address nodeDelegatorAddress) internal view {
        if (
            INodeDelegator(nodeDelegatorAddress).getEffectivePodShares() != 0
                || address(nodeDelegatorAddress).balance > maxNegligibleAmount
                || INodeDelegator(nodeDelegatorAddress).getAssetUnstaking(LRTConstants.ETH_TOKEN) > 0
        ) {
            revert NodeDelegatorHasETH();
        }
    }
