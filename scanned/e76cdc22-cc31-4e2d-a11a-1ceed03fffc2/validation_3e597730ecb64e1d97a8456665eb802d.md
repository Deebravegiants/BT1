### Title
Missing Chainlink Heartbeat Staleness Check Enables Stale-Price Minting of rsETH at a Discount — (`contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`)

---

### Summary

`ChainlinkOracleForRSETHPoolCollateral.getRate()` validates Chainlink data with only a round-completeness check (`answeredInRound < roundID`) and a zero-timestamp guard. It never checks `block.timestamp - updatedAt` against the feed's heartbeat. When a collateral feed goes silent for longer than its heartbeat (e.g., 24 h for ETH/USD), the contract continues to return the last stale price without reverting. Any depositor — including an attacker who simply monitors the mempool — can call `RSETHPoolV3.deposit(token, amount, referralId)` while the stale (inflated) price is live and receive more `wrsETH` than the deposited collateral is worth at the true market price.

---

### Finding Description

**Root cause — `ChainlinkOracleForRSETHPoolCollateral.getRate()` (lines 26–37)** [1](#0-0) 

The function fetches `(roundID, ethPrice, , timestamp, answeredInRound)` from `latestRoundData()` and applies three guards:

| Check | What it catches |
|---|---|
| `answeredInRound < roundID` | Round started but not yet answered |
| `timestamp == 0` | Incomplete round |
| `ethPrice <= 0` | Negative/zero price |

**Not present**: `block.timestamp - timestamp > HEARTBEAT` — the standard guard that detects a feed that has simply stopped updating.

A Chainlink feed can satisfy all three existing checks while being hours or days stale: `answeredInRound == roundID`, `timestamp != 0`, and `ethPrice > 0` are all true for the last valid round even if that round closed 30 hours ago.

**Exploitation path through `RSETHPoolV3.deposit(token, amount, referralId)`** [2](#0-1) 

The token deposit calls `viewSwapRsETHAmountAndFee(amount, token)`: [3](#0-2) 

The minted amount is:

```
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate
```

`tokenToETHRate` is read directly from `ChainlinkOracleForRSETHPoolCollateral.getRate()`. [4](#0-3) 

If the collateral token's market price has fallen since the last oracle update (e.g., stETH drops 2 % during a 26-hour feed silence), the stale `tokenToETHRate` is inflated relative to reality. The attacker deposits collateral worth `X` ETH and receives `wrsETH` worth `X + ΔX` ETH. The surplus `ΔX` is extracted from the protocol's backing, diluting all existing `wrsETH` holders.

The same oracle wrapper is consumed by `RSETHPoolV3ExternalBridge`, `RSETHPoolV3WithNativeChainBridge`, and `RSETHPool` via the same `supportedTokenOracle[token]` mapping, so the surface is broad.

**The sandwich framing is not required.** The attacker does not need to front-run anything. They only need to observe that the feed is stale (public on-chain data) and call `deposit()` before the feed resumes. Front-running a pending Chainlink keeper transaction is one way to maximise the window, but the vulnerability is exploitable at any point during the staleness period.

---

### Impact Explanation

The attacker mints `wrsETH` tokens whose face value (in ETH) exceeds the ETH-equivalent of the collateral they deposited. They can immediately transfer or bridge those tokens. The shortfall is borne by the protocol's collateral backing, reducing the redemption value for all other `wrsETH` holders — constituting direct theft of funds in-motion. The daily mint cap (`limitDailyMint`) bounds per-day loss but does not prevent the attack; it merely limits its magnitude per epoch. [5](#0-4) 

---

### Likelihood Explanation

Chainlink feeds do occasionally miss their heartbeat during network congestion or keeper outages. The stale window for ETH/USD is 24 h; for LST/ETH feeds it can be 24 h or longer. An attacker monitoring `latestRoundData().updatedAt` on-chain can detect the condition permissionlessly and act within the same block. No privileged role, no governance capture, and no oracle operator compromise is required — only a publicly observable on-chain state.

---

### Recommendation

Add a configurable heartbeat constant and enforce it in `getRate()`:

```solidity
uint256 public immutable heartbeat; // e.g. 86400 for 24 h

function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID) revert StalePrice();
    if (timestamp == 0) revert IncompleteRound();
    if (block.timestamp - timestamp > heartbeat) revert StalePrice(); // ← add this
    if (ethPrice <= 0) revert InvalidPrice();

    return uint256(ethPrice) * 1e18 / 10 ** uint256(AggregatorV3Interface(oracle).decimals());
}
```

Set `heartbeat` per-feed at construction time (e.g., `86400` for a 24 h feed, `3600` for a 1 h feed).

```solidity
pragma solidity 0.8.27;

// Fork mainnet at a recent block.
// 1. Deploy a mock AggregatorV3 that returns a fixed stale timestamp
//    (block.timestamp - 25 hours) and a price 2% above current market.
// 2. Deploy ChainlinkOracleForRSETHPoolCollateral pointing at the mock.
// 3. Set it as supportedTokenOracle[stETH] in RSETHPoolV3.
// 4. Call deposit(stETH, 1e18, "") — observe rsETHAmount_stale.
// 5. Update mock to return block.timestamp and the true lower price.
// 6. Call deposit(stETH, 1e18, "") — observe rsETHAmount_fresh.
// 7. Assert rsETHAmount_stale > rsETHAmount_fresh.
//    The difference is extractable value per 1 stETH deposited.

function testStalePriceMint() external {
    uint256 stalePrice  = 1.02e18; // stale: stETH = 1.02 ETH
    uint256 freshPrice  = 1.00e18; // true:  stETH = 1.00 ETH
    uint256 staleTs     = block.timestamp - 25 hours;

    MockAggregator agg = new MockAggregator(stalePrice, staleTs);
    ChainlinkOracleForRSETHPoolCollateral oracle =
        new ChainlinkOracleForRSETHPoolCollateral(address(agg));

    // getRate() succeeds — no heartbeat check fires
    uint256 rateStale = oracle.getRate(); // returns 1.02e18

    agg.setPrice(freshPrice, block.timestamp);
    uint256 rateFresh = oracle.getRate(); // returns 1.00e18

    // rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate
    // With rsETHToETHrate = 1.05e18 (example), feeBps = 0:
    uint256 rsETHStale = 1e18 * rateStale / 1.05e18; // ~0.9714 rsETH
    uint256 rsETHFresh = 1e18 * rateFresh / 1.05e18; // ~0.9524 rsETH

    assert(rsETHStale > rsETHFresh);
    // Δ ≈ 0.019 rsETH per 1 stETH deposited — extractable value
}
```
