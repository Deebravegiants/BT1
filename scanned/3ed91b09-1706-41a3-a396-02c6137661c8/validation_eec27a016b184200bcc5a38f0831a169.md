### Title
`NodeDelegator.getAssetUnstaking()` Unbounded Nested Loop Causes `LRTOracle.updateRSETHPrice()` to Run Out of Gas — (File: `contracts/NodeDelegator.sol`)

---

### Summary

`NodeDelegator.getAssetUnstaking()` iterates over every queued EigenLayer withdrawal for a given NDC. This function is invoked `numSupportedAssets × numNDCs` times inside a single `LRTOracle.updateRSETHPrice()` call. As the total number of queued withdrawals approaches the protocol's own cap (80), the cumulative gas cost of `updateRSETHPrice()` grows to the point where the transaction can no longer fit in one block, permanently blocking price updates and stalling the entire protocol.

---

### Finding Description

`LRTOracle.updateRSETHPrice()` is a public function that calls `_getTotalEthInProtocol()`, which iterates over every supported asset and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each one. [1](#0-0) 

`getTotalAssetDeposits()` delegates to `getAssetDistributionData()` (or `getETHDistributionData()`), both of which loop over every NDC in `nodeDelegatorQueue` and call `INodeDelegator(ndc).getAssetUnstaking(asset)` for each. [2](#0-1) [3](#0-2) 

`getAssetUnstaking()` itself makes an external call to `delegationManager.getQueuedWithdrawals(address(this))` and then runs a **nested loop** over every queued withdrawal and every strategy inside each withdrawal: [4](#0-3) 

The total work performed in one `updateRSETHPrice()` call is therefore:

```
numSupportedAssets × numNDCs × (external getQueuedWithdrawals calls)
+ numSupportedAssets × Σ(queued withdrawals per NDC) × (strategies per withdrawal)
```

The protocol's own comment acknowledges the ceiling: *"120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price"*, and the hard cap is set to 80. [5](#0-4) 

However, the cap is a **total** count across all NDCs, while the gas cost scales as `numAssets × totalWithdrawals`. With 5 supported assets, 10 NDCs, and 80 total queued withdrawals, the function performs ≥ 400 withdrawal-level iterations plus 50 external `getQueuedWithdrawals` calls — all inside a single transaction. As the number of supported assets grows, the multiplier grows with it, and the effective gas ceiling is lower than the raw withdrawal count suggests.

Additionally, `setUncompletedWithdrawalCount()` can resync the counter to the actual EigenLayer state (e.g., after forced undelegations), potentially allowing the real count to exceed the soft cap before operators can react. [6](#0-5) 

---

### Impact Explanation

If `updateRSETHPrice()` reverts with out-of-gas:

- The stored `rsETHPrice` in `LRTOracle` becomes permanently stale.
- All new deposits use the stale price, minting incorrect rsETH amounts.
- `unlockQueue()` in `LRTWithdrawalManager` uses `lrtOracle.rsETHPrice()`, so withdrawal payouts are computed from a stale rate.
- Protocol fee minting (inside `_updateRsETHPrice()`) is blocked, constituting permanent freezing of unclaimed yield.
- `updateRSETHPriceAsManager()` calls the same `_updateRsETHPrice()` path and is equally blocked.

**Impact: Medium — Permanent freezing of unclaimed yield / temporary freezing of funds.**

---

### Likelihood Explanation

The protocol operates with up to 10 NDCs and a growing set of supported assets. Operators routinely call `initiateUnstaking()` and `undelegate()`, filling the withdrawal queue toward the 80-withdrawal cap. No attacker action is required; normal protocol operation drives the queue toward the gas limit. The risk increases monotonically as the protocol scales.

---

### Recommendation

1. **Cache queued withdrawals per NDC once per `updateRSETHPrice()` call** rather than re-fetching them for every asset. A single `getQueuedWithdrawals()` call per NDC can be used to compute `getAssetUnstaking()` for all assets simultaneously.
2. **Decouple asset accounting from the price update path**: store per-asset unstaking amounts in storage and update them lazily (e.g., when withdrawals are completed), rather than recomputing them on every price update.
3. **Reduce the `maxUncompletedWithdrawalCount` cap** to account for the `numAssets` multiplier, or enforce a per-NDC cap rather than a global total.

---

### Proof of Concept

Call chain that exhausts gas:

```
LRTOracle.updateRSETHPrice()                          // public, no auth
  └─ _getTotalEthInProtocol()
       └─ for each asset (×5):
            LRTDepositPool.getTotalAssetDeposits(asset)
              └─ getAssetDistributionData(asset)
                   └─ for each NDC (×10):
                        NodeDelegator.getAssetUnstaking(asset)
                          └─ delegationManager.getQueuedWithdrawals(ndc)  // external call ×50
                               └─ for each withdrawal (×8 avg):
                                    for each strategy:
                                      sharesToUnderlyingView(...)          // external call
```

With 5 assets × 10 NDCs × 8 withdrawals × 2 strategies = **800 inner iterations** plus **50 external `getQueuedWithdrawals` calls** and up to **800 `sharesToUnderlyingView` calls**, the transaction gas cost approaches or exceeds the block gas limit, causing `updateRSETHPrice()` to revert and leaving `rsETHPrice` permanently stale.             totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }

    function setUncompletedWithdrawalCount() external onlyLRTManager {
        ILRTDepositPool lrtDepositPool = ILRTDepositPool(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IDelegationManager delegationManager =
            IDelegationManager(lrtConfig.getContract(LRTConstants.EIGEN_DELEGATION_MANAGER));
        address[] memory nodeDelegatorQueue = lrtDepositPool.getNodeDelegatorQueue();
        uint256 totalQueued;
        for (uint256 i = 0; i < nodeDelegatorQueue.length; i++) {
            address nodeDelegator = nodeDelegatorQueue[i];
            (IDelegationManager.Withdrawal[] memory queuedWithdrawals,) =
                delegationManager.getQueuedWithdrawals(nodeDelegator);
            totalQueued += queuedWithdrawals.length;
        }

        uncompletedWithdrawalCount = totalQueued;

        emit UncompletedWithdrawalCountSet(totalQueued);
    }
