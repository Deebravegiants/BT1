### Title
`swapAssetToPremintedRsETH` Bypasses Pause State for Whitelisted Users - (File: contracts/pools/RSETHPoolV2ExternalBridge.sol)

### Summary
`RSETHPoolV2ExternalBridge` exposes a reverse-swap path (`swapAssetToPremintedRsETH`) that is callable by any address holding `WHITELISTED_USER_ROLE` regardless of the contract's paused state. While `deposit()` is correctly gated by `whenNotPaused`, the reverse-swap function carries no such guard, allowing whitelisted users to continue draining ETH from the pool even after an emergency pause.

### Finding Description
`RSETHPoolV2ExternalBridge` implements a custom pause mechanism via the `paused` boolean and `whenNotPaused` modifier. The user-facing `deposit()` function correctly enforces this:

```solidity
// contracts/pools/RSETHPoolV2ExternalBridge.sol:289
function deposit(string memory referralId) external payable nonReentrant whenNotPaused limitDailyMint(msg.value) {
```

However, `swapAssetToPremintedRsETH()` — which transfers ETH out of the pool to the caller in exchange for rsETH — carries no `whenNotPaused` guard:

```solidity
// contracts/pools/RSETHPoolV2ExternalBridge.sol:418-425
function swapAssetToPremintedRsETH(
    address rsETH,
    uint256 rsETHAmount
)
    external
    nonReentrant
    onlyOperatorOrWhitelisted(msg.sender)   // ← no whenNotPaused
{
```

The access modifier `onlyOperatorOrWhitelisted` grants entry to both `OPERATOR_ROLE` **and** `WHITELISTED_USER_ROLE`:

```solidity
// contracts/pools/RSETHPoolV2ExternalBridge.sol:130-135
modifier onlyOperatorOrWhitelisted(address account) {
    if (!hasRole(OPERATOR_ROLE, account) && !hasRole(WHITELISTED_USER_ROLE, account)) {
        revert NotOperatorOrWhitelisted();
    }
    _;
}
```

`WHITELISTED_USER_ROLE` is a distinct, non-admin role intended for regular counterparties such as market makers or arbitrageurs. Any address holding this role can invoke `swapAssetToPremintedRsETH()` while the contract is paused, bypassing the emergency stop entirely.

The function's execution path:
1. Validates rsETH is allowed in the wrapper and amount is within limits.
2. Calls `viewSwapAssetToPremintedRsETH()` to compute ETH output at the current oracle rate.
3. Pulls rsETH from the caller into the wrapper.
4. Sends ETH from the pool's balance to the caller.

```solidity
// contracts/pools/RSETHPoolV2ExternalBridge.sol:435-443
uint256 ethAmount = viewSwapAssetToPremintedRsETH(rsETHAmount);
IERC20(rsETH).safeTransferFrom(msg.sender, address(wrapper), rsETHAmount);
if (getETHBalanceMinusFees() < ethAmount) revert InsufficientETHBalanceForReverseSwap();
(bool success,) = payable(msg.sender).call{ value: ethAmount }("");
```

This is structurally identical to the reported vulnerability: an operation that should be blocked during a restricted protocol state (pause ≡ liquidation) proceeds without the required state check.

### Impact Explanation
When the contract is paused — typically in response to an oracle anomaly, a discovered bug, or an active exploit — the protocol's intent is to halt all asset movement. The missing `whenNotPaused` on `swapAssetToPremintedRsETH()` means:

- ETH deposited by users can be withdrawn from the pool by whitelisted counterparties during the pause window.
- If the pause was triggered precisely because the oracle is returning an inflated rsETH/ETH rate, a whitelisted user can exploit that rate to extract more ETH than the rsETH they provide is worth, constituting direct theft of user-deposited funds.
- Even under a correct oracle rate, the pool's ETH liquidity is reduced during a period when the protocol has explicitly signalled that operations should stop, undermining the emergency response.

**Impact: Medium — Temporary freezing / theft of user funds during a pause event.**

### Likelihood Explanation
`WHITELISTED_USER_ROLE` is a non-admin role designed to be granted to external counterparties. Any such counterparty monitoring on-chain events will observe the `Paused` event and can immediately call `swapAssetToPremintedRsETH()` before the pause is lifted. The attacker path requires no admin compromise — only possession of a role that is routinely granted to market participants. Pauses are a known operational event in LRT protocols, making the window of exploitation realistic.

**Likelihood: Medium.**

### Recommendation
Add `whenNotPaused` to `swapAssetToPremintedRsETH()`:

```diff
 function swapAssetToPremintedRsETH(
     address rsETH,
     uint256 rsETHAmount
 )
     external
     nonReentrant
+    whenNotPaused
     onlyOperatorOrWhitelisted(msg.sender)
 {
```

The same missing guard exists in `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, and `RSETHPoolV3WithNativeChainBridge` for their `swapAssetToPremintedRsETH()` variants, though those are restricted to `OPERATOR_ROLE` only. The fix should be applied uniformly across all pool variants.

### Proof of Concept
1. Admin grants `WHITELISTED_USER_ROLE` to address `W` (a market maker).
2. An oracle anomaly causes rsETH to appear overvalued; the `PAUSER_ROLE` calls `pause()` — `deposit()` is now blocked.
3. `W` holds rsETH tokens and calls `swapAssetToPremintedRsETH(rsETH, amount)`.
4. The function executes: rsETH is transferred to the wrapper; ETH is sent to `W` at the inflated oracle rate.
5. `W` receives more ETH than the rsETH is actually worth; user-deposited ETH is drained from the pool while the protocol is in an emergency-paused state. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
