### Title
`MetricOmmPoolFactory.setPoolBinAdditionalFees` Allows Per-Bin Fees to Bypass Factory Caps, Enabling Total Effective Swap Fees to Exceed the Protocol Hard Cap — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6` / `addFeeSellE6` directly to the pool with no validation against any factory-level cap. The global admin spread fee is bounded by `maxAdminSpreadFeeE6` (≤ `HARD_MAX_SPREAD_FEE_E6 = 200_000`, i.e. 20%), but the per-bin additional fees are `uint16` values (max 65 535 = 6.5535% in E6 units) and are added on top of the global fee during every swap through the affected bin. A pool admin can therefore push the total effective spread fee for a specific bin above the protocol's hard ceiling, causing traders to pay more than the protocol's maximum-allowed fee.

---

### Finding Description

**Factory cap system for the global spread fee** — `setPoolAdminFees` validates:

```solidity
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
``` [1](#0-0) 

**Per-bin setter has no analogous check** — `setPoolBinAdditionalFees` simply forwards the caller-supplied values:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool-level `setBinAdditionalFees` only validates the bin index, not the fee magnitudes: [3](#0-2) 

**How the per-bin fee enters swap execution** — during every swap the per-bin fee is added to the base fee before computing the required input:

```solidity
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)   // buy token0
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)  // sell token0
``` [4](#0-3) [5](#0-4) 

**Arithmetic of the breach** — `uint16` max = 65 535 → 6.5535 % in E6 units. If the global `spreadFeeE6` is already at the hard cap of 200 000 (20 %), adding a per-bin fee of 65 535 yields an effective bin fee of 26.5535 %, exceeding `HARD_MAX_SPREAD_FEE_E6` by 32.7 %. [6](#0-5) 

---

### Impact Explanation

Every swap routed through the affected bin charges the trader a higher-than-permitted fee. The excess fee accrues to LPs (it is part of the LP fee, not the protocol fee), so it is a direct, real-money loss to traders. The pool admin can selectively target the bin where the current price sits (`curBinIdx`) to maximise the impact on active trading. This is an admin-boundary break: the factory owner's ability to cap fees via `maxAdminSpreadFeeE6` is fully circumvented for per-bin fees.

---

### Likelihood Explanation

The pool admin is a semi-trusted role whose fee-setting power is explicitly bounded by the factory's cap system. Any pool admin — including one that turns adversarial after deployment — can call `setPoolBinAdditionalFees` at any time with no timelock, no cooldown, and no cap check. The call requires only `onlyPoolAdmin(pool)` access, which is a normal operational privilege.

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool, mirroring the pattern used in `setPoolAdminFees`:

```diff
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
+   if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
+   if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce a separate hard cap constant for per-bin fees (e.g. `HARD_MAX_BIN_FEE_E6`) so that the sum of global spread fee and per-bin fee can never exceed `HARD_MAX_SPREAD_FEE_E6`.

---

### Proof of Concept

```solidity
// Pool is created with maxAdminSpreadFeeE6 = 200_000 (20 %, the hard cap).
// Global spreadFeeE6 is already at 200_000.
// Pool admin calls:
factory.setPoolBinAdditionalFees(pool, curBinIdx, 65_535, 65_535);
// No revert — addFeeBuyE6 = 65_535 (6.5535 %) is stored unchecked.

// Next swap through curBinIdx:
//   effectiveBuyFeeX64  = baseFeeX64(20 %) + 6.5535 % = 26.5535 %
//   effectiveSellFeeX64 = baseFeeX64(20 %) + 6.5535 % = 26.5535 %
// Trader pays 32.7 % more fee than the protocol hard cap permits.
// assertGt(effectiveFee, HARD_MAX_SPREAD_FEE_E6); // passes
```
