### Title
Immutable Chainlink Oracle in `ChainlinkOracleForRSETHPoolCollateral` Causes Permanent DOS of L2 Pool Token Deposits and Reverse Swaps - (File: contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol)

### Summary
`ChainlinkOracleForRSETHPoolCollateral` stores the underlying Chainlink aggregator address as `immutable`. If the Chainlink feed is deprecated or paused by Chainlink governance, `getRate()` will revert with no recovery path within the oracle contract itself, permanently blocking all token deposits and reverse swaps in every L2 pool that uses this oracle as a `supportedTokenOracle`.

### Finding Description
`ChainlinkOracleForRSETHPoolCollateral` wraps a Chainlink `AggregatorV3Interface` to provide collateral token/ETH rates to L2 pool contracts. The aggregator address is stored as `immutable`:

```solidity
address public immutable oracle;
```

The `getRate()` function calls `latestRoundData()` on this immutable address with no `try/catch` and no fallback:

```solidity
function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();
    if (answeredInRound < roundID) revert StalePrice();
    if (timestamp == 0) revert IncompleteRound();
    if (ethPrice <= 0) revert InvalidPrice();
    ...
}
```

Because `oracle` is `immutable`, there is no setter inside `ChainlinkOracleForRSETHPoolCollateral` to point to a replacement feed. If the Chainlink feed reverts for any reason (deprecation, Chainlink admin pause, logical flaw), every call to `getRate()` reverts permanently.

This `getRate()` is called inside `viewSwapRsETHAmountAndFee(amount, token)` in all L2 pool variants:

```solidity
uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate();
```

which is called by both `deposit(token, amount, referralId)` and `swapAssetToPremintedRsETH(...)`.

### Impact Explanation
When the underlying Chainlink feed reverts:
- All calls to `deposit(token, amount, referralId)` revert → users cannot deposit collateral tokens into the L2 pool.
- All calls to `swapAssetToPremintedRsETH(rsETH, token, rsETHAmount)` revert → users who hold pre-minted rsETH cannot swap back to the collateral token.

This constitutes **temporary freezing of funds** (Medium). The pool admin holds a `setSupportedTokenOracle` escape hatch, but it requires deploying a new oracle contract and executing a timelock-gated transaction, creating a meaningful window of DOS. During this window, users with rsETH in the wrapper cannot redeem it for the collateral token via the pool.

### Likelihood Explanation
Chainlink feeds are actively maintained infrastructure. Feeds have been deprecated on multiple chains historically. The ETH/LST feeds used for collateral pricing in these pools are live Chainlink aggregators subject to Chainlink's operational decisions. The probability is non-trivial over the protocol's lifetime, and the code provides no in-contract mitigation.

### Recommendation
1. Remove `immutable` from `oracle` and add an owner-controlled setter so the aggregator address can be updated without redeploying the wrapper contract.
2. Alternatively, wrap the `latestRoundData()` call in a `try/catch` and return a safe fallback value (or revert with a clear error that allows the pool to distinguish oracle failure from invalid price).

### Proof of Concept

1. `ChainlinkOracleForRSETHPoolCollateral` is deployed with a specific Chainlink aggregator address baked in as `immutable`. [1](#0-0) 

2. The Chainlink aggregator is called unconditionally with no try/catch in `getRate()`. [2](#0-1) 

3. This oracle is registered as `supportedTokenOracle[token]` in L2 pools. Any token deposit calls `viewSwapRsETHAmountAndFee`, which calls `IOracle(supportedTokenOracle[token]).getRate()`. [3](#0-2) 

4. The same path is hit in `swapAssetToPremintedRsETH` via `viewSwapAssetToPremintedRsETH`. [4](#0-3) 

5. If the Chainlink feed is deprecated or paused, `latestRoundData()` reverts, propagating the revert through `getRate()` → `viewSwapRsETHAmountAndFee` → `deposit` / `swapAssetToPremintedRsETH`, permanently blocking all token-denominated deposits and reverse swaps until a new oracle contract is deployed and the timelock-gated `setSupportedTokenOracle` is executed.
