### Title
`ethValueInWithdrawal` Permanently Inflated by Oracle Price Drift Between `transferAssetFromDepositPool` / `transferAssetToDepositPool` Calls — (`contracts/LRTConverter.sol`)

---

### Summary

`ethValueInWithdrawal` is credited at the oracle price at the time assets leave the deposit pool, but debited at the oracle price at the time assets return. When the LST price falls between the two calls, a permanent residual remains in `ethValueInWithdrawal`, inflating the ETH value reported by `getETHDistributionData()` and therefore overstating `rsETHPrice` in `LRTOracle`.

---

### Finding Description

`transferAssetFromDepositPool` adds to `ethValueInWithdrawal` using the oracle price at call time P1: [1](#0-0) 

`transferAssetToDepositPool` subtracts using the oracle price at call time P2: [2](#0-1) 

If P2 < P1 (a natural LST price decline between the two operations), the subtraction removes only `amount × P2 / 1e18`, leaving a residual of `amount × (P1 − P2) / 1e18` permanently in `ethValueInWithdrawal`. There is no mechanism to correct or reset this residual.

`getETHDistributionData()` in `LRTDepositPool` reads this value directly as `ethLyingInConverter`: [3](#0-2) 

`LRTOracle._getTotalEthInProtocol()` sums all asset deposits including ETH, which routes through `getETHDistributionData()`: [4](#0-3) 

The inflated `totalETHInProtocol` then feeds directly into `rsETHPrice` computation: [5](#0-4) 

---

### Impact Explanation

`rsETHPrice` is overstated by the residual ETH value. New depositors calling `depositAsset` or `depositETH` receive fewer rsETH tokens than they are entitled to, because `getRsETHAmountToMint` divides by the inflated `rsETHPrice`: [6](#0-5) 

No funds are lost from the protocol — the backing assets are still present — but depositors receive a smaller share than the true backing ratio warrants. This matches **Low: Contract fails to deliver promised returns, but doesn't lose value**.

---

### Likelihood Explanation

Both functions require `onlyAssetTransferRole`, so this is not triggerable by an unprivileged caller. However, the Asset Transfer Role holder performs these operations routinely (moving LSTs to the converter for unstaking, then returning unused amounts). LST oracle prices fluctuate continuously. No malicious intent is required: any round-trip where the price at return time is lower than at departure time silently inflates `ethValueInWithdrawal`. The residual accumulates across every such round-trip and is never corrected. [7](#0-6) [8](#0-7) 

---

### Recommendation

Track the **token amount** in the converter rather than its ETH value, and convert to ETH at query time using the current oracle price. Alternatively, on `transferAssetToDepositPool`, subtract the ETH value that was originally credited for those tokens (stored per-transfer), not the current oracle value of the returned amount.

---

### Proof of Concept

```solidity
// Fork-safe fuzz sketch (local/private testnet only)
// 1. Asset Transfer Role calls transferAssetFromDepositPool(stETH, 1000e18)
//    Oracle price P1 = 1.05e18 => ethValueInWithdrawal += 1050e18

// 2. stETH oracle price drops to P2 = 1.00e18 (natural market move)

// 3. Asset Transfer Role calls transferAssetToDepositPool(stETH, 1000e18)
//    assetValue = 1000e18 * 1.00e18 / 1e18 = 1000e18
//    ethValueInWithdrawal = 1050e18 - 1000e18 = 50e18  (residual, never zero)

// 4. Assert: ethValueInWithdrawal == 0  => FAILS (50e18 residual)
// 5. Assert: rsETHPrice == trueBacking  => FAILS (overstated by residual / rsethSupply)
```

The residual `50e18` is permanently included in `getETHDistributionData().ethLyingInConverter` and therefore in every subsequent `rsETHPrice` update, shortchanging all future depositors until the contract is upgraded or the residual is manually corrected.

    )
        external
        onlySupportedERC20Token(_asset)
        onlyAssetTransferRole
    {
        address lrtDepositPoolAddress = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address lrtOracleAddress = lrtConfig.getContract(LRTConstants.LRT_ORACLE);
        ILRTOracle lrtOracle = ILRTOracle(lrtOracleAddress);
        uint256 assetValue = (_amount * lrtOracle.getAssetPrice(_asset)) / 1e18;

        // Set to 0 if assetValue exceeds ethValueInWithdrawal, otherwise subtract assetValue
        ethValueInWithdrawal = ethValueInWithdrawal > assetValue ? ethValueInWithdrawal - assetValue : 0;

        IERC20(_asset).safeTransfer(lrtDepositPoolAddress, _amount);
    }

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
