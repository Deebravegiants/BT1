### Title
Unbounded Loop in `_checkResidueLSTBalance` Can Cause OOG Revert, Permanently Blocking NDC Removal — (`contracts/LRTDepositPool.sol`)

---

### Summary

`LRTDepositPool._checkResidueLSTBalance()` iterates over every entry in `lrtConfig.getSupportedAssetList()` and makes **three external calls per asset**. There is no cap on the length of `supportedAssetList`. As the protocol adds LST assets over time via `TIME_LOCK_ROLE`, this loop grows unboundedly, eventually causing `removeNodeDelegatorContractFromQueue` to revert out-of-gas, permanently blocking NDC dequeuing.

---

### Finding Description

`_removeNodeDelegatorContractFromQueue` calls `_checkResidueLSTBalance`, which fetches the full supported-asset array and loops over it: [1](#0-0) 

For each non-ETH asset the loop executes:

1. `IERC20(supportedAssets[i]).balanceOf(nodeDelegatorAddress)` — external call to the LST token contract.
2. `INodeDelegator(nodeDelegatorAddress).getAssetBalance(supportedAssets[i])` — which internally calls `getDelegationManager().getWithdrawableShares()` on EigenLayer.
3. `INodeDelegator(nodeDelegatorAddress).getAssetUnstaking(supportedAssets[i])` — which internally calls `getDelegationManager().getQueuedWithdrawals()` on EigenLayer, itself iterating over all queued withdrawals. [2](#0-1) 

`getAssetUnstaking` re-fetches the full queued-withdrawal list from EigenLayer on every iteration. With M supported assets and W queued withdrawals, the gas cost scales as O(M × W).

New assets are added via `addNewSupportedAsset`, gated only by `TIME_LOCK_ROLE`: [3](#0-2) 

There is no maximum length enforced on `supportedAssetList`: [4](#0-3) 

The entry point for NDC removal is: [5](#0-4) 

which unconditionally calls `_checkResidueLSTBalance` before dequeuing: [6](#0-5) 

---

### Impact Explanation

Once the gas cost of `_checkResidueLSTBalance` exceeds the block gas limit (~30 M gas), every call to `removeNodeDelegatorContractFromQueue` and `removeManyNodeDelegatorContractsFromQueue` reverts with OOG. NDCs cannot be dequeued, so:

- Funds held in an NDC that must be retired (e.g., after a slashing event or operator change) cannot be recovered via the normal removal path.
- The invariant that "NodeDelegators can always be cleanly dequeued" is broken.

This maps to **Medium — Temporary freezing of funds** (and **Medium — Unbounded gas consumption**). It is "temporary" only if the admin can reduce `supportedAssetList` via `removeSupportedAsset`, but that function itself requires `getTotalAssetDeposits(asset) <= maxNegligibleAmount`, which may not hold, creating a potential deadlock.

---

### Likelihood Explanation

`addNewSupportedAsset` is a routine governance action. The protocol already starts with two assets and is designed to onboard additional LSTs. Each EigenLayer `getQueuedWithdrawals` call is non-trivial in gas. With ~20–30 supported assets and several queued withdrawals per NDC, the cumulative cost becomes significant; with 50+ assets it can approach or exceed block limits. This is a realistic long-term protocol state, not a theoretical edge case.

---

### Recommendation

1. **Cap `supportedAssetList` length** in `addNewSupportedAsset` (e.g., `require(supportedAssetList.length < MAX_ASSETS)`).
2. **Refactor `_checkResidueLSTBalance`** to accept a caller-supplied asset subset rather than fetching the full list, or cache the list length and enforce a hard limit.
3. **Decouple `getAssetUnstaking` from `getQueuedWithdrawals`**: cache queued-withdrawal data at the NDC level rather than re-querying EigenLayer per asset per call.

---

    function _checkResidueLSTBalance(address nodeDelegatorAddress) internal view {
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetsLength = supportedAssets.length;

        uint256 assetBalance;
        for (uint256 i; i < supportedAssetsLength; ++i) {
            if (supportedAssets[i] == LRTConstants.ETH_TOKEN) {
                // this function only checks for residual LST balance
                continue;
            }

            assetBalance = IERC20(supportedAssets[i]).balanceOf(nodeDelegatorAddress)
                + INodeDelegator(nodeDelegatorAddress).getAssetBalance(supportedAssets[i]);
            assetBalance += INodeDelegator(nodeDelegatorAddress).getAssetUnstaking(supportedAssets[i]);

            if (assetBalance > maxNegligibleAmount) {
                revert NodeDelegatorHasAssetBalance(supportedAssets[i], assetBalance);
            }
        }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
