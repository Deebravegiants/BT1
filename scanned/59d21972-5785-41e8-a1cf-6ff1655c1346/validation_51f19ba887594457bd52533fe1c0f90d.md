### Title
Permanent Freezing of Reward Tokens When `totalKernelStaked` Drops to Zero During Active Reward Period - (File: contracts/KERNEL/KernelDepositPool.sol)

### Summary

`KernelDepositPool.sol` implements a Synthetix-style staking rewards contract where `initiateWithdrawal()` immediately reduces `totalKernelStaked`. If this value reaches zero during an active reward period, all remaining reward tokens for that period become permanently locked in the contract. There is no sweep or recovery function, and `notifyRewardAmount()` itself reverts when `totalKernelStaked == 0`, making the situation unrecoverable.

### Finding Description

`KernelDepositPool` distributes reward tokens over a fixed `duration` window set by `notifyRewardAmount()`. The reward accrual rate is computed in `rewardPerToken()`:

```solidity
function rewardPerToken() public view returns (uint256) {
    if (totalKernelStaked == 0) {
        return rewardPerTokenStored;  // accrual frozen
    }
    return rewardPerTokenStored + (rewardRate * (lastTimeRewardApplicable() - updatedAt) * DECIMAL_PRECISION)
        / totalKernelStaked;
}
```

When `totalKernelStaked == 0`, `rewardPerToken()` returns the stored value unchanged — reward accrual halts entirely. The reward tokens corresponding to the remaining window (`rewardRate * (finishAt - block.timestamp)`) stay in the contract but are never distributed to anyone.

`initiateWithdrawal()` immediately subtracts from `totalKernelStaked`:

```solidity
function initiateWithdrawal(uint256 _amount) external nonReentrant updateReward(msg.sender) {
    ...
    balanceOf[msg.sender] -= _amount;
    totalKernelStaked -= _amount;
    ...
}
```

If all stakers call `initiateWithdrawal()`, `totalKernelStaked` hits zero while `finishAt` is still in the future. The admin cannot restart distribution because `notifyRewardAmount()` enforces:

```solidity
if (totalKernelStaked == 0) revert NoStakedTokens();
```

There is no `sweep`, `emergencyWithdraw`, or `recoverTokens` function anywhere in `KernelDepositPool.sol`. The contract's own NatSpec acknowledges this at lines 18–22 but treats it as an operational concern rather than enforcing a guard:

> *"If `totalKernelStaked` ever hits zero during a reward distribution window, any remaining rewards for that period will stay locked in the contract."*

### Impact Explanation

Reward tokens deposited via `notifyRewardAmount()` for the current distribution window are permanently frozen in `KernelDepositPool` whenever `totalKernelStaked` reaches zero mid-period. No user can claim them, no admin function can recover them, and the contract cannot be restarted until someone stakes again — but even then, the already-elapsed zero-staked interval's reward tokens are irrecoverable. This is a **permanent freezing of unclaimed yield**.

### Likelihood Explanation

The `withdrawalDelay` can be set up to `MAX_WITHDRAWAL_DELAY = 30 days`. During a market stress event, coordinated exit, or loss of confidence in the protocol, all active stakers may call `initiateWithdrawal()` within a short window. Because the delay is up to 30 days, `totalKernelStaked` can remain at zero for an extended period while a reward window is still active. This is a realistic scenario requiring no privileged access — any staker can call `initiateWithdrawal()`.

### Recommendation

1. **Add a sweep/recovery function** restricted to the admin that allows recovering reward tokens when `totalKernelStaked == 0` and the reward period has ended (or been abandoned):
   ```solidity
   function recoverRewardTokens(uint256 amount) external onlyRole(DEFAULT_ADMIN_ROLE) {
       require(block.timestamp > finishAt || totalKernelStaked == 0, "Active period");
       rewardsToken.safeTransfer(msg.sender, amount);
   }
   ```
2. **Alternatively**, when `totalKernelStaked` drops to zero, automatically freeze `finishAt` to `block.timestamp` so the unallocated reward tokens are accounted for and can be rolled into the next period via `notifyRewardAmount()`.
3. **Enforce the operational assumption** in `initiateWithdrawal()` by reverting if the withdrawal would bring `totalKernelStaked` to zero during an active reward period, or by requiring a minimum staked amount to remain.

### Proof of Concept

1. Admin calls `notifyRewardAmount(1_000_000e18)` with `duration = 30 days`. `rewardRate = 1_000_000e18 / 30 days`. `finishAt = block.timestamp + 30 days`.
2. Alice is the only staker with `balanceOf[Alice] = 1000e18`, `totalKernelStaked = 1000e18`.
3. After 1 day, Alice calls `initiateWithdrawal(1000e18)`. `totalKernelStaked` becomes `0`. `withdrawalDelay = 30 days`, so Alice cannot claim her KERNEL for 30 days.
4. `rewardPerToken()` now returns `rewardPerTokenStored` unchanged for the remaining 29 days.
5. Approximately `rewardRate * 29 days ≈ 966_666e18` reward tokens are permanently locked in the contract.
6. Admin attempts `notifyRewardAmount(...)` → reverts with `NoStakedTokens`.
7. No sweep function exists. Tokens are irrecoverable. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
