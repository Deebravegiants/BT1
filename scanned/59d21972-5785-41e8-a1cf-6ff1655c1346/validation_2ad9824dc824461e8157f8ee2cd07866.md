### Title
`updateRSETHPriceAsManager()` Bypasses Pause Check, Allowing rsETH Price Update While Oracle Is Paused - (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle` defines a `whenNotPaused` modifier and applies it to the public `updateRSETHPrice()` function, but the privileged `updateRSETHPriceAsManager()` function calls the same internal `_updateRsETHPrice()` without the `whenNotPaused` guard. When the oracle is paused — including via the automatic downside-protection pause — the LRT manager can still invoke `updateRSETHPriceAsManager()` to overwrite the frozen `rsETHPrice`, defeating the purpose of the pause.

### Finding Description
`LRTOracle` has two public entry points into `_updateRsETHPrice()`:

```solidity
// contracts/LRTOracle.sol:87-96
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}

function updateRSETHPriceAsManager() external onlyLRTManager {
    _updateRsETHPrice();
}
```

`updateRSETHPrice()` is correctly gated by `whenNotPaused`, but `updateRSETHPriceAsManager()` carries no such guard. The oracle's own `_updateRsETHPrice()` contains an automatic downside-protection path that pauses the oracle (and the deposit pool and withdrawal manager) when the price drops beyond `pricePercentageLimit`:

```solidity
// contracts/LRTOracle.sol:277-281
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

After this auto-pause fires, `rsETHPrice` is frozen at the last safe value. However, the LRT manager can immediately call `updateRSETHPriceAsManager()` — which has no `whenNotPaused` check — to re-run `_updateRsETHPrice()` and overwrite `rsETHPrice` with the new (potentially still-anomalous) price.

### Impact Explanation
`rsETHPrice` is the denominator used by `LRTDepositPool.getRsETHAmountToMint()` to compute how many rsETH tokens a depositor receives:

```solidity
// contracts/LRTDepositPool.sol:520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

If the oracle is paused manually (not via the auto-pause path), the deposit pool may remain active. In that scenario, the manager can update `rsETHPrice` to an anomalous value while the oracle is nominally paused, causing depositors to receive an incorrect rsETH amount. Even in the auto-pause scenario, the price freeze — the primary safety invariant of the pause — is rendered ineffective because the manager can overwrite it at will.

**Impact: Low** — the contract fails to deliver its promised invariant (price is frozen while paused), but no direct loss of principal occurs absent further malicious action.

### Likelihood Explanation
Exploitation requires the holder of the `MANAGER` role to act contrary to protocol intent. The role is trusted, so likelihood is low. However, the missing modifier is a structural design flaw: the pause mechanism is advertised as a safety stop, yet one privileged path silently bypasses it.

### Recommendation
Add `whenNotPaused` to `updateRSETHPriceAsManager()`:

```solidity
function updateRSETHPriceAsManager() external whenNotPaused onlyLRTManager {
    _updateRsETHPrice();
}
```

If the intent is specifically to allow the manager to override the *price-increase threshold* (not the pause), the bypass logic should be scoped to that check inside `_updateRsETHPrice()` rather than omitting the pause guard entirely.

### Proof of Concept
1. `pricePercentageLimit` is set to 1 % (1e16).
2. TVL drops 2 %, triggering the auto-pause path in `_updateRsETHPrice()`: deposit pool, withdrawal manager, and oracle are all paused; `rsETHPrice` is frozen.
3. LRT manager calls `updateRSETHPriceAsManager()` — no `whenNotPaused` check fires.
4. `_updateRsETHPrice()` executes again; `rsETHPrice` is overwritten with the new (still-depressed) price.
5. If the deposit pool was paused only by the oracle's auto-pause and the manager subsequently unpauses it, depositors now interact with the anomalous price that the pause was meant to freeze.
