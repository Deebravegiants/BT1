### Title
Aave V3 Pool Pause Permanently Blocks ETH Withdrawal Completion and Disables All Recovery Paths - (File: contracts/LRTWithdrawalManager.sol)

### Summary
When the Aave V3 integration is enabled in `LRTWithdrawalManager`, a pause of the Aave V3 pool causes every ETH `completeWithdrawal()` call to revert. Critically, every administrative recovery path — `emergencyWithdrawFromAave()`, `setAaveIntegrationEnabled(false)`, and `configureAaveIntegration()` — also calls `_withdrawFromAave()` internally, so none of them can succeed while Aave is paused. Users with unlocked ETH withdrawal requests are frozen with no protocol-level escape hatch.

### Finding Description

`LRTWithdrawalManager` integrates with Aave V3 to earn yield on idle ETH held between `unlockQueue()` and `completeWithdrawal()`. When `isAaveIntegrationEnabled` is `true`, the `_processWithdrawalCompletion()` function checks whether the contract's native ETH balance covers the user's request; if not, it calls `_withdrawFromAave()`:

```solidity
// LRTWithdrawalManager.sol lines 720-731
if (isAaveIntegrationEnabled && asset == LRTConstants.ETH_TOKEN) {
    uint256 contractBalance = address(this).balance;
    if (contractBalance < request.expectedAssetAmount) {
        uint256 amountNeeded = request.expectedAssetAmount - contractBalance;
        _withdrawFromAave(amountNeeded);

        uint256 balanceAfter = address(this).balance;
        if (balanceAfter < request.expectedAssetAmount) {
            revert InsufficientLiquidityForWithdrawal();
        }
    }
}
```

`_withdrawFromAave()` unconditionally calls `aaveWETHGateway.withdrawETH()`:

```solidity
// LRTWithdrawalManager.sol line 917
aaveWETHGateway.withdrawETH(aavePool, withdrawnAmount, address(this));
```

The Aave V3 `Pool.withdraw()` function carries a `whenNotPaused` modifier. When the Aave guardian pauses the pool, this call reverts, propagating the revert up through `_processWithdrawalCompletion()` → `completeWithdrawal()` / `completeWithdrawalForUser()`.

All three administrative recovery paths share the same fatal dependency:

1. **`emergencyWithdrawFromAave()`** (line 560): calls `_withdrawFromAave(amount)` directly — reverts when Aave is paused.
2. **`setAaveIntegrationEnabled(false)`** (lines 486–496): calls `_collectInterestToTreasury()` then `_withdrawFromAave(aaveBalance)` before setting the flag — reverts before `isAaveIntegrationEnabled` is ever set to `false`.
3. **`configureAaveIntegration()`** (lines 438–448): calls `_withdrawFromAave(aaveBalance)` when reconfiguring — also reverts.

Because `isAaveIntegrationEnabled` can only be set to `false` inside `setAaveIntegrationEnabled()` after the withdrawal succeeds, there is no way to flip the flag while Aave is paused. The contract is stuck in a state where every ETH withdrawal completion reverts.

### Impact Explanation

All users who have initiated ETH withdrawals and whose requests have been unlocked via `unlockQueue()` cannot call `completeWithdrawal()`. Their rsETH has already been burned (in `unlockQueue()`) and the ETH is locked in Aave. The freeze persists for the entire duration of the Aave pause. This constitutes a **temporary freezing of funds** for all affected ETH withdrawers.

### Likelihood Explanation

Aave V3 has a guardian role that can pause the pool in response to security incidents; this has occurred on mainnet. The Aave integration in `LRTWithdrawalManager` is explicitly designed for mainnet ETH (the `WETH_ADDRESS` constant is hardcoded to the mainnet WETH address). Any Aave V3 pause event — however brief — blocks all in-flight ETH withdrawals with no protocol-level workaround available.

### Recommendation

1. In `_processWithdrawalCompletion()`, wrap the `_withdrawFromAave()` call in a `try/catch`. If the Aave withdrawal fails, fall back to serving the user from the contract's idle ETH balance if sufficient, or revert with a descriptive error that does not permanently block the queue.
2. Decouple the `isAaveIntegrationEnabled = false` flag-flip from the withdrawal step in `setAaveIntegrationEnabled()`. Set the flag first, then attempt the withdrawal. This allows the flag to be cleared even when Aave is paused, unblocking future `completeWithdrawal()` calls.
3. Add a dedicated emergency flag-reset function (callable by `PAUSER_ROLE`) that sets `isAaveIntegrationEnabled = false` without attempting any Aave interaction, mirroring the recommendation in the reference report to introduce a pause-bypass at the integration layer.
4. Optionally, check `IPool(aavePool).paused()` before calling `_withdrawFromAave()` and skip the Aave path when the pool is paused.

### Proof of Concept

1. Operator enables Aave integration: `setAaveIntegrationEnabled(true)`.
2. Operator calls `unlockQueue(ETH_TOKEN, ...)` — ETH is pulled from `LRTUnstakingVault` into `LRTWithdrawalManager` and auto-deposited to Aave via `depositToAaveExternal`. `isAaveIntegrationEnabled` is `true`, `totalETHDepositedToAave > 0`, contract idle ETH balance is 0.
3. User calls `initiateWithdrawal(ETH_TOKEN, rsETHAmount, ...)` — rsETH is transferred to the contract, request is queued.
4. Aave V3 guardian pauses the pool.
5. User calls `completeWithdrawal(ETH_TOKEN, ...)`:
   - `_processWithdrawalCompletion()` is entered.
   - `address(this).balance < request.expectedAssetAmount` → true (all ETH is in Aave).
   - `_withdrawFromAave(amountNeeded)` is called → `aaveWETHGateway.withdrawETH(aavePool, ...)` reverts with Aave's `RESERVE_PAUSED` error.
   - Transaction reverts. User cannot receive ETH.
6. `PAUSER_ROLE` calls `emergencyWithdrawFromAave(type(uint256).max)` → same revert.
7. Manager calls `setAaveIntegrationEnabled(false)` → `_withdrawFromAave()` is called before the flag is set → same revert. Flag remains `true`.

```text
    function _withdrawFromAave(uint256 amount) internal returns (uint256 withdrawnAmount) {
        if (amount == 0) return 0;

        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        if (aaveBalance == 0) revert InsufficientAaveBalance();

        // Only withdraw up to the principal amount (don't use accrued interest for user withdrawals)
        uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave ? aaveBalance : totalETHDepositedToAave;

        withdrawnAmount = amount > withdrawablePrincipal ? withdrawablePrincipal : amount;
        if (withdrawnAmount == 0) return 0;

        aaveWETHGateway.withdrawETH(aavePool, withdrawnAmount, address(this));
        totalETHDepositedToAave -= withdrawnAmount;

        emit ETHWithdrawnFromAave(withdrawnAmount, totalETHDepositedToAave);
    }
```
