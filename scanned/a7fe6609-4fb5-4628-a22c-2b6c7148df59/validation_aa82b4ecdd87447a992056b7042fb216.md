### Title
`AnchoredPriceProvider` Uses L1-Only Staleness Check With No Future Tolerance, Causing `FeedStalled` DoS on L2 Deployments — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider._isStale()` unconditionally treats any oracle `refTime > block.timestamp` as stale. On L2 chains (Arbitrum, Blast, etc.) the sequencer's clock can run slightly ahead of the EVM `block.timestamp`, so a freshly published oracle price with a refTime a few seconds in the future is rejected as stale. Every swap through a pool backed by this provider reverts with `FeedStalled()`. The protocol already recognises this problem for `PriceProvider` and `ProtectedPriceProvider` — both have dedicated L2 variants (`PriceProviderL2`, `ProtectedPriceProviderL2`) that add a `FUTURE_TOLERANCE` window — but `AnchoredPriceProvider`, described as "the one standard provider for public pools," has no L2 variant and no tolerance parameter.

---

### Finding Description

`AnchoredPriceProvider._isStale()` is explicitly labelled an L1 check:

```solidity
/// @dev Pure staleness check (L1). Any future refTime is stale.
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← hard rejection of any future refTime
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

This is called in `_readLeg()`, which is the only path to a valid bid/ask quote:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [2](#0-1) 

When `_readLeg` returns `ok = false`, `_getBidAndAskPrice` returns the sentinel `(0, type(uint128).max)`, and the public entry point reverts:

```solidity
function getBidAndAskPrice() external override returns (uint128 bid, uint128 ask) {
    (bid, ask) = _getBidAndAskPrice();
    if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
}
``` [3](#0-2) 

The L2-aware variants of the other providers solve this with a `FUTURE_TOLERANCE` immutable:

```solidity
/// @dev L2 sequencer timestamp can lag behind oracle publication time.
///      Allows refTime up to FUTURE_TOLERANCE seconds ahead of block.timestamp.
uint256 public immutable FUTURE_TOLERANCE;
``` [4](#0-3) 

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) {
        return (refTime - nowTs) > futureTol;   // ← tolerates small future skew
    }
    return (nowTs - refTime) > maxDelta;
}
``` [5](#0-4) 

`AnchoredPriceProvider` has no such parameter and no L2 sibling. `AnchoredProviderFactory` (which deploys it) likewise has no L2 variant, while `PriceProviderFactoryL2` exists for the other pair. [6](#0-5) 

---

### Impact Explanation

Every swap through a `MetricOmmPool` whose `priceProvider` is an `AnchoredPriceProvider` calls `getBidAndAskPrice()`. If the oracle's `refTime` is even one second ahead of the L2 `block.timestamp`, the call reverts with `FeedStalled()`. Because the sequencer on chains like Arbitrum can set block timestamps up to 1 hour in the future relative to the L1 anchor, this condition is not rare — it can persist for extended periods. The result is a complete DoS of all swaps (and any extension logic that depends on a live price) for every public pool backed by this provider. LPs cannot exit via swap; traders cannot execute. This matches the "broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows" impact gate.

---

### Likelihood Explanation

`AnchoredPriceProvider` is the designated standard provider for public pools. The protocol is explicitly multi-chain (L2 deployment scripts exist, `PriceProviderL2` and `ProtectedPriceProviderL2` exist). Pyth Lazer and Chainlink Data Streams publish timestamps derived from off-chain clocks that routinely run a few seconds ahead of the L2 sequencer's `block.timestamp`. The condition `refTime > block.timestamp` is therefore a normal operating state on L2, not an edge case.

---

### Recommendation

Introduce an `AnchoredPriceProviderL2` (or add an optional `futureTolerance` constructor parameter to `AnchoredPriceProvider`) and update `_isStale` to mirror the logic already present in `PriceProviderL2._isStale`:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Cap `futureTolerance` at construction (e.g. `<= 1 hours`) as `PriceProviderL2` does, and validate it in the factory envelope. Deploy a matching `AnchoredProviderFactoryL2` that creates L2-aware instances.

---

### Proof of Concept

1. Deploy `AnchoredProviderFactory` + `AnchoredPriceProvider` on Arbitrum (or any L2 with sequencer clock skew).
2. The Pyth Lazer oracle publishes a price update with `refTime = block.timestamp + 3` (3 seconds ahead of the current L2 block).
3. A user calls `MetricOmmPool.swap(...)`.
4. The pool calls `priceProvider.getBidAndAskPrice()`.
5. `_readLeg` calls `_isStale(refTime=T+3, nowTs=T, MAX_REF_STALENESS)`.
6. `refTime > nowTs` → `_isStale` returns `true` → `_readLeg` returns `ok = false`.
7. `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
8. `getBidAndAskPrice` reverts with `FeedStalled()`.
9. The swap reverts. All swaps revert for as long as the oracle refTime leads the L2 block timestamp.

        // refTime in the past or equal: check age
