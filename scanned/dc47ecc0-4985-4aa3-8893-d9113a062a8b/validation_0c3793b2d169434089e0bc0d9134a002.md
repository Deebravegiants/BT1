### Title
Missing Chainlink Heartbeat Staleness Check Allows Excess wrsETH Minting — (`contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`)

---

### Summary

`ChainlinkOracleForRSETHPoolCollateral.getRate()` omits the standard `block.timestamp - updatedAt > heartbeat` staleness check. A Chainlink feed that has not been updated beyond its heartbeat window (e.g., 24 h for stETH/ETH) but still satisfies `answeredInRound >= roundID` and `timestamp != 0` will pass all three guards and return a stale price. When this oracle is assigned as `supportedTokenOracle[token]` in `RSETHPoolV3` or `RSETHPoolV3WithNativeChainBridge`, any caller can invoke `deposit(token, amount, referralId)` and receive excess wrsETH, diluting the restaking yield accrued by existing holders.

---

### Finding Description

`ChainlinkOracleForRSETHPoolCollateral.getRate()` performs exactly three checks:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol  lines 30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0)            revert IncompleteRound();
if (ethPrice <= 0)             revert InvalidPrice();
``` [1](#0-0) 

None of these checks verify that `block.timestamp - updatedAt <= heartbeat`. A feed that last updated 25+ hours ago will still satisfy all three conditions and return the stale price.

This oracle is consumed by `RSETHPoolV3.viewSwapRsETHAmountAndFee(amount, token)`:

```solidity
// contracts/pools/RSETHPoolV3.sol  lines 331-334
uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate();
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
``` [2](#0-1) 

The same pattern exists in `RSETHPoolV3WithNativeChainBridge`: [3](#0-2) 

If `tokenToETHRate` is stale and inflated relative to the true current price, `rsETHAmount` is over-computed and `wrsETH.mint(msg.sender, rsETHAmount)` mints excess tokens: [4](#0-3) 

No other guard in the deposit path corrects for this. The `limitDailyMint` modifier calls the same `viewSwapRsETHAmountAndFee` with the same stale oracle, so it only caps the total inflated amount — it does not prevent the over-mint per unit of collateral. [5](#0-4) 

---

### Impact Explanation

The excess wrsETH minted represents a claim on more rsETH than the deposited collateral is worth at the true current price. When the attacker later unwraps wrsETH → rsETH, they extract rsETH that was backed by yield accrued by existing holders. This dilutes the rsETH/ETH rate for all existing wrsETH/rsETH holders, constituting **theft of unclaimed yield** (the restaking yield embedded in the growing rsETH/ETH rate is redistributed to the attacker).

---

### Likelihood Explanation

Chainlink feeds go stale in practice during periods of low price volatility (the feed does not trigger a deviation-threshold update) combined with network congestion or Chainlink node issues that delay the heartbeat update. The stETH/ETH feed has a 24 h heartbeat and a 0.5% deviation threshold; during stable periods it can approach the heartbeat boundary. No privileged access is required — any EOA can call `deposit(token, amount, referralId)` on a non-paused pool with a supported token.

---

### Recommendation

Add a heartbeat parameter to `ChainlinkOracleForRSETHPoolCollateral` and enforce it in `getRate()`:

```solidity
uint256 public immutable heartbeat;

constructor(address _oracle, uint256 _heartbeat) {
    oracle    = _oracle;
    heartbeat = _heartbeat;
}

function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 updatedAt, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID)              revert StalePrice();
    if (updatedAt == 0)                         revert IncompleteRound();
    if (block.timestamp - updatedAt > heartbeat) revert StalePrice();   // ← add this
    if (ethPrice <= 0)                          revert InvalidPrice();

    return uint256(ethPrice) * 1e18 / 10 ** uint256(AggregatorV3Interface(oracle).decimals());
}
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";

interface IAggregatorV3 {
    function latestRoundData() external view
        returns (uint80, int256, uint256, uint256, uint80);
    function decimals() external view returns (uint8);
}

contract MockStaleAggregator {
    function decimals() external pure returns (uint8) { return 18; }
    function latestRoundData() external view
        returns (uint80 roundId, int256 answer, uint256 startedAt,
                 uint256 updatedAt, uint80 answeredInRound)
    {
        roundId        = 1;
        answeredInRound = 1;                          // passes answeredInRound >= roundID
        answer         = 0.9999e18;                   // stale inflated stETH/ETH price
        updatedAt      = block.timestamp - 25 hours;  // 25 h ago — past 24 h heartbeat
        startedAt      = updatedAt;
    }
}

contract StalePricePoC is Test {
    function test_staleOracleAccepted() public {
        MockStaleAggregator agg = new MockStaleAggregator();

        // Deploy the oracle wrapper (no heartbeat check)
        ChainlinkOracleForRSETHPoolCollateral oracle =
            new ChainlinkOracleForRSETHPoolCollateral(address(agg));

        // getRate() succeeds and returns the 25-hour-old price — no revert
        uint256 rate = oracle.getRate();
        assertEq(rate, 0.9999e18, "stale price accepted");

        // In the real pool: rsETHAmount = amountAfterFee * 0.9999e18 / rsETHToETHrate
        // vs fair:          rsETHAmount = amountAfterFee * truePriceNow / rsETHToETHrate
        // Attacker receives (0.9999 - truePrice) / rsETHToETHrate * amountAfterFee
        // extra wrsETH per unit of collateral deposited.
    }
}
```

The test deploys `ChainlinkOracleForRSETHPoolCollateral` against a mock aggregator returning `updatedAt = block.timestamp - 25 hours` with `answeredInRound == roundID`. `getRate()` returns the stale price without reverting, confirming the missing guard. In a full fork test, calling `RSETHPoolV3.deposit(stETH, largeAmount, "")` with this oracle assigned would mint measurably more wrsETH than the fair-value baseline.
