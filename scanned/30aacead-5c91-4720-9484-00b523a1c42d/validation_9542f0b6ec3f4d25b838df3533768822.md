### Title
Missing Chainlink `updatedAt` Staleness Check Causes Under-Minting of rsETH for stETH Deposits - (`contracts/oracles/ChainlinkPriceOracle.sol`)

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all return values except `price`, including `updatedAt`. No staleness threshold is enforced. When the stETH/ETH feed is stale and stETH has appreciated, the L1Vault mints fewer rsETH than the deposited stETH value justifies.

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` destructures `latestRoundData()` as:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
``` [1](#0-0) 

`updatedAt`, `roundId`, and `answeredInRound` are all silently dropped. There is no check of the form `require(block.timestamp - updatedAt <= MAX_STALENESS)`.

The stale price flows directly into rsETH minting via:

1. `L1Vault.unwrapWstETH()` produces stETH from wstETH. [2](#0-1) 

2. `L1Vault.depositAssetForL1Vault(stETH)` calls `lrtDepositPool.getRsETHAmountToMint(stETH, balance)` and uses the result as both the expected mint amount and the `minRSETHAmountExpected` floor passed to `depositAsset`. [3](#0-2) 

3. `LRTDepositPool.getRsETHAmountToMint()` computes `(amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`. [4](#0-3) 

4. `LRTOracle.getAssetPrice()` delegates to `ChainlinkPriceOracle.getAssetPrice()`, which returns the stale price. [5](#0-4) 

Because `rsETHAmountToMint` is derived from the stale price and then used as the minimum, the `depositAsset` call succeeds without reverting — it simply mints fewer rsETH than the stETH is currently worth.

### Impact Explanation

The L1Vault (and the L2 users whose bridged assets it represents) receive fewer rsETH than the fair value of the deposited stETH. The stETH is not lost — it enters the protocol — but the rsETH shortfall is permanent: the under-minted rsETH is never corrected. This matches the **Low** scope: *contract fails to deliver promised returns, but doesn't lose value*.

### Likelihood Explanation

The Chainlink stETH/ETH feed has a 24-hour heartbeat and a 0.5% deviation threshold. Periods of low volatility where the feed goes many hours without an update are routine. The manager calling `unwrapWstETH()` followed by `depositAssetForL1Vault(stETH)` is the intended operational flow, so no adversarial action is required — the bug triggers under normal conditions whenever the feed is stale and stETH has accrued yield.

### Recommendation

Add a staleness check in `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(, int256 price,, uint256 updatedAt,) = priceFeed.latestRoundData();
require(block.timestamp - updatedAt <= MAX_STALENESS, "Stale price");
```

`MAX_STALENESS` should be set per-feed (e.g., 3600 seconds for stETH/ETH). Alternatively, expose a per-asset configurable heartbeat and revert if exceeded.

### Proof of Concept

Fork test outline (no mainnet execution):

```solidity
// 1. Deploy/fork with ChainlinkPriceOracle pointing at a mock AggregatorV3
// 2. Set mock to return: answer = 0.99e18 (stale, stETH below current fair value)
//    updatedAt = block.timestamp - 2 hours  (stale)
// 3. Call L1Vault.unwrapWstETH() → produces stETH
// 4. Call L1Vault.depositAssetForL1Vault(stETH)
//    → getRsETHAmountToMint uses 0.99e18 price
//    → rsETHMinted = stETHAmount * 0.99e18 / rsETHPrice
// 5. Advance mock to return answer = 1.00e18 (current fair value)
// 6. Assert: rsETHMinted_stale < rsETHMinted_fresh
//    → shortfall = stETHAmount * (1.00e18 - 0.99e18) / rsETHPrice > 0
// 7. Confirm transaction did NOT revert (minRSETHAmountExpected was also stale)
```

The assertion at step 6 will hold on unmodified code, confirming the invariant break: `getAssetPrice(stETH)` must revert or return 0 if `updatedAt > MAX_STALENESS`, but it does neither.

        UtilLib.checkNonZeroAddress(token);

        uint256 tokenBalance = IERC20(token).balanceOf(address(this));
        uint256 rsETHAmountToMint = lrtDepositPool.getRsETHAmountToMint(token, tokenBalance);

        if (rsETHAmountToMint == 0) {
            revert InvalidMinRSETHAmountExpected();
        }

        // Approve the LRT deposit pool to transfer the token
        IERC20(token).safeIncreaseAllowance(address(lrtDepositPool), tokenBalance);

        lrtDepositPool.depositAsset(token, tokenBalance, rsETHAmountToMint, "");

        emit AssetDepositForL1Vault(token, tokenBalance, rsETHAmountToMint);
    }
