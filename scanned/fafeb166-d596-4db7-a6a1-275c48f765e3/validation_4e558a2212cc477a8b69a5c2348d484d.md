### Title
Unbounded Gas in `_checkResidueLSTBalance` Scales O(A × EigenLayer-calls) with No Asset Cap — (`contracts/LRTDepositPool.sol`)

---

### Summary

`removeNodeDelegatorContractFromQueue` calls `_checkResidueLSTBalance`, which iterates every supported asset and makes multiple deep external calls per asset — including into EigenLayer — with no cap on the number of supported assets in `LRTConfig`. As the asset list grows through legitimate `TIME_LOCK_ROLE` governance, the function's gas cost grows without bound and can permanently exceed the block gas limit.

---

### Finding Description

`_checkResidueLSTBalance` fetches the full supported asset list and, for each non-ETH asset, makes three top-level external calls: [1](#0-0) 

Each of those three calls fans out further:

- `getAssetBalance` (NodeDelegator.sol line 551) delegates to `NodeDelegatorHelper.getAssetBalance`, which calls `lrtConfig.assetStrategy(asset)`, then `DelegationManager.getWithdrawableShares(...)`, then `IStrategy.sharesToUnderlyingView(...)` — **3 additional external calls per asset**. [2](#0-1) 

- `getAssetUnstaking` (NodeDelegator.sol lines 405-427) calls `DelegationManager.getQueuedWithdrawals(address(this))` and then, for every queued withdrawal entry, calls `strategy.underlyingToken()` and `strategy.sharesToUnderlyingView(...)` — **unbounded additional calls proportional to pending withdrawals**. [3](#0-2) 

`LRTConfig._addNewSupportedAsset` has **no cap** on the number of assets that can be added: [4](#0-3) 

`addNewSupportedAsset` is gated only by `TIME_LOCK_ROLE`, a legitimate governance role whose normal operation is to add new LST assets over time: [5](#0-4) 

The call chain is:

```
removeNodeDelegatorContractFromQueue
  └─ _removeNodeDelegatorContractFromQueue
       └─ _checkResidueLSTBalance
            └─ for each asset A:
                 ├─ IERC20.balanceOf(ndc)                          [1 call]
                 ├─ INodeDelegator.getAssetBalance(asset)          [→ 3 EL calls]
                 └─ INodeDelegator.getAssetUnstaking(asset)        [→ 1 + 2×W EL calls]
```

Total external calls ≈ `A × (4 + 2W)` where W = queued withdrawals. At Ethereum's 30 M gas block limit, with cold-call costs (~2 100 gas each) plus EigenLayer storage reads, the function becomes unexecutable well before A reaches 50.

---

### Impact Explanation

`removeNodeDelegatorContractFromQueue` and `removeManyNodeDelegatorContractsFromQueue` become permanently reverting once the asset list is large enough. The NDC cannot be dequeued, its funds cannot be formally separated from the pool accounting, and the admin loses the ability to restructure the delegator queue. This is **Medium — Unbounded gas consumption** per the allowed impact scope. [6](#0-5) 

---

### Likelihood Explanation

The precondition is ordinary protocol growth: `TIME_LOCK_ROLE` adding new LST assets over time is the intended use of `addNewSupportedAsset`. No key compromise or malicious actor is required. The protocol currently supports a small number of assets, so the issue is not immediately triggered, but there is no on-chain safeguard preventing it.

---

### Recommendation

1. **Cap the supported asset list** in `LRTConfig._addNewSupportedAsset` (e.g., `require(supportedAssetList.length < MAX_ASSETS)`).
2. **Refactor `_checkResidueLSTBalance`** to accept a caller-supplied asset subset rather than fetching the entire list, or cache the list length and revert early if it exceeds a safe threshold.
3. Alternatively, remove the balance-check precondition from `removeNodeDelegatorContractFromQueue` and enforce it off-chain, relying on the existing `NodeDelegatorHasAssetBalance` revert only when balances are actually non-zero.

---

// Foundry fork test — run against a local/private fork, not mainnet
import "forge-std/Test.sol";
import "../contracts/LRTConfig.sol";
import "../contracts/LRTDepositPool.sol";

contract UnboundedGasTest is Test {
    LRTConfig config;
    LRTDepositPool pool;

    function setUp() public { /* deploy & wire contracts */ }

    function testGasScalesWithAssets(uint8 numAssets) public {
        vm.assume(numAssets > 1 && numAssets <= 50);

        // TIME_LOCK_ROLE adds numAssets mock ERC-20s
        for (uint i = 0; i < numAssets; i++) {
            address mockToken = address(new MockERC20());
            vm.prank(timeLockRole);
            config.addNewSupportedAsset(mockToken, 100 ether);
        }

        address ndc = address(new MockNDC());
        vm.prank(admin);
        pool.addNodeDelegatorContractToQueue(toArray(ndc));

        uint256 gasBefore = gasleft();
        vm.prank(admin);
        pool.removeNodeDelegatorContractFromQueue(ndc);
        uint256 gasUsed = gasBefore - gasleft();

        // Assert gas grows linearly and eventually exceeds 5 M
        emit log_named_uint("gasUsed for A assets", gasUsed);
        // At A=50 this will revert with out-of-gas on mainnet block limit
    }
}

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
