### Title
L2 Pool Deposit Functions Lack Minimum rsETH Output Protection - (File: contracts/pools/RSETHPoolV3ExternalBridge.sol, contracts/pools/RSETHPoolV3.sol, contracts/pools/RSETHPool.sol, contracts/pools/RSETHPoolNoWrapper.sol, contracts/pools/RSETHPoolV2ExternalBridge.sol, contracts/pools/RSETHPoolV3WithNativeChainBridge.sol)

### Summary

Every L2 pool `deposit()` function mints wrsETH/rsETH based on the oracle rate at execution time, with no `minRsETHAmount` parameter. The L1 `LRTDepositPool` explicitly provides this protection; the L2 pools do not. If the oracle rate is updated between when a user signs a transaction and when it is executed, the user receives fewer tokens than expected with no recourse.

### Finding Description

The L1 `LRTDepositPool.depositETH()` and `depositAsset()` accept a `minRSETHAmountExpected` parameter and revert via `MinimumAmountToReceiveNotMet` if the minted amount falls below it: [1](#0-0) [2](#0-1) 

Every L2 pool `deposit()` function is missing this parameter entirely. The minted amount is computed solely from the live oracle rate at execution time:

### Impact Explanation

**Low — Contract fails to deliver promised returns, but doesn't lose value.**

When a user previews the deposit off-chain (e.g., via `viewSwapRsETHAmountAndFee`), they observe a specific oracle rate. If the oracle is updated before their transaction is mined, the rate increases (rsETH appreciates monotonically as staking rewards accrue), and the user receives fewer wrsETH/rsETH tokens than they expected. The user's ETH/LST is fully consumed; they simply receive less of the output token than the rate they saw at signing time. There is no mechanism to cancel or bound the deviation.

### Likelihood Explanation

The rsETH oracle rate on L2s is pushed periodically from L1 via cross-chain messages. Rate updates are routine protocol operations, not adversarial. Any user whose transaction is pending during an oracle update — including during normal network congestion on L2s — is exposed. The per-update magnitude is small (staking rewards are ~4–5% APY), but the exposure is structural and affects every depositor on every L2 pool version. The L1 contract's explicit `minRSETHAmountExpected` guard confirms the protocol is aware of this risk class and chose to protect L1 depositors but not L2 depositors.

### Recommendation

Add a `uint256 minRsETHAmount` parameter to all L2 pool `deposit()` functions and revert if the computed mint amount falls below it, mirroring the L1 `LRTDepositPool` pattern:

```solidity
// Example fix for RSETHPoolV3ExternalBridge.deposit(string)
function deposit(string memory referralId, uint256 minRsETHAmount)
    external payable nonReentrant whenNotPaused limitDailyMint(msg.value, ETH_IDENTIFIER)
{
    uint256 amount = msg.value;
    if (amount == 0) revert InvalidAmount();
    (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
    if (rsETHAmount < minRsETHAmount) revert MinimumAmountToReceiveNotMet();
    feeEarnedInETH += fee;
    wrsETH.mint(msg.sender, rsETHAmount);
    emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
}
```

Apply the same pattern to all token `deposit()` overloads across all L2 pool contracts.

### Proof of Concept

1. User calls `viewSwapRsETHAmountAndFee(1 ether)` on `RSETHPoolV3ExternalBridge` and observes `rsETHAmount = 0.952 wrsETH` at rate `1.05 ETH/rsETH`.
2. User submits `deposit{value: 1 ether}("ref")`.
3. Before the transaction is mined, the oracle is updated: rate becomes `1.06 ETH/rsETH`.
4. Transaction executes: `rsETHAmount = 1e18 * 1e18 / 1.06e18 = 0.943 wrsETH`.
5. User receives `0.943 wrsETH` instead of the expected `0.952 wrsETH` — a ~0.9% shortfall — with no ability to reject the outcome.
6. The L1 equivalent (`LRTDepositPool.depositETH`) would have reverted with `MinimumAmountToReceiveNotMet` under the same conditions.

    function _beforeDeposit(
        address asset,
        uint256 depositAmount,
        uint256 minRSETHAmountExpected
    )
        private
        view
        returns (uint256 rsethAmountToMint)
    {
        if (depositAmount == 0 || depositAmount < minAmountToDeposit) {
            revert InvalidAmountToDeposit();
        }

        if (_checkIfDepositAmountExceedesCurrentLimit(asset, depositAmount)) {
            revert MaximumDepositLimitReached();
        }

        rsethAmountToMint = getRsETHAmountToMint(asset, depositAmount);

        if (rsethAmountToMint < minRSETHAmountExpected) {
            revert MinimumAmountToReceiveNotMet();
        }
    }

    function viewSwapRsETHAmountAndFee(uint256 amount) public view returns (uint256 rsETHAmount, uint256 fee) {
        fee = amount * feeBps / 10_000;
        uint256 amountAfterFee = amount - fee;

        // rate of rsETH in ETH
        uint256 rsETHToETHrate = getRate();

        // Calculate the final rsETH amount
        rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
    }
