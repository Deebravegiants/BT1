### Title
Unguarded `sendETHToL1ViaBridge` with Unchecked `l2bridge` Enables Block Stuffing at Near-Zero ETH Cost — (`contracts/bridges/UnichainMessenger.sol`)

---

### Summary

`UnichainMessenger.sendETHToL1ViaBridge` has no caller access control and accepts an arbitrary, unvalidated `l2bridge` address. An attacker can supply a malicious contract that burns near-maximum gas then reverts. Because the revert propagates up with no `try/catch`, the entire transaction reverts, returning the attacker's ETH while consuming block gas up to the revert point. The attacker's only cost is gas fees, enabling sustained block stuffing on Unichain at negligible ETH cost.

---

### Finding Description

`UnichainMessenger.sendETHToL1ViaBridge` is declared `external payable nonReentrant` with no role check or caller whitelist: [1](#0-0) 

The `l2bridge` parameter is passed directly to `IUnichainMessenger(l2bridge).bridgeETHTo{value: value}(...)` with no address validation: [2](#0-1) 

The legitimate callers — `RSETHPoolNoWrapper.bridgeAssetsViaNativeBridge` and similar pool functions — enforce `onlyRole(BRIDGER_ROLE)` on their own side, but that restriction does not propagate to the messenger itself: [3](#0-2) 

Because there is no `try/catch` around the `bridgeETHTo` call, any revert in the malicious `l2bridge` propagates up and reverts the entire transaction. On revert, all EVM state changes — including the `ReentrancyGuard` storage slot and the ETH transfer — are rolled back. The attacker recovers their ETH; only gas consumed up to the revert point is permanently spent.

The `DEFAULT_GAS_LIMIT = 200_000` constant is passed as a *parameter* to `bridgeETHTo`, not as the EVM gas stipend for the call. The actual gas available to the malicious contract is bounded only by the 63/64 forwarding rule, so the attacker can craft a contract that burns ~190,000–200,000 gas before reverting. [4](#0-3) 

The same pattern exists in `BaseMessenger`, `ArbitrumMessenger`, and `ScrollMessenger`, all of which share the same missing access control: [5](#0-4) 

---

### Impact Explanation

**Low — Block stuffing.**

On Unichain (OP Stack), blocks are produced every ~2 seconds with a 30 M gas limit. Each malicious call consumes ~200,000+ gas and costs only the L2 gas fee (fractions of a cent). To fill one block requires ~150 such transactions. The attacker recovers all ETH on every revert, so the sustained cost is purely gas fees. This can delay or prevent legitimate bridge operations, user deposits, and protocol transactions from being included in blocks.

---

### Likelihood Explanation

**Medium.** The function is publicly callable with no preconditions beyond supplying `msg.value == value`. Deploying a gas-burning reverting contract is trivial. The attack is repeatable indefinitely and requires no privileged access, no front-running, and no protocol state setup.

---

### Recommendation

Add a caller access control modifier to `sendETHToL1ViaBridge` in all messenger contracts, restricting calls to the authorized pool contract(s) (e.g., an `onlyAuthorizedCaller` modifier backed by an immutable or admin-set address). Additionally, validate that `l2bridge` matches a protocol-configured address rather than accepting arbitrary input.

```solidity
address public immutable authorizedPool;

modifier onlyAuthorizedPool() {
    require(msg.sender == authorizedPool, "Unauthorized");
    _;
}

function sendETHToL1ViaBridge(address l2bridge, address target, uint256 value)
    external payable nonReentrant onlyAuthorizedPool { ... }
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

contract MaliciousBridge {
    // Burns ~190_000 gas then reverts — ETH is returned to caller
    function bridgeETHTo(address, uint32, bytes memory) external payable {
        uint256 i;
        while (gasleft() > 10_000) { unchecked { ++i; } }
        revert("burn");
    }
}

contract BlockStuffingPoC {
    UnichainMessenger messenger; // deployed UnichainMessenger
    MaliciousBridge  bridge;

    constructor(address _messenger) payable {
        messenger = UnichainMessenger(_messenger);
        bridge    = new MaliciousBridge();
    }

    // Call repeatedly; each call consumes ~200k gas, ETH is refunded
    function stuff() external payable {
        try messenger.sendETHToL1ViaBridge{value: 1}(address(bridge), address(this), 1) {}
        catch {}
        // ETH returned; only gas fees paid
    }
}
```

Deploy on a local Unichain fork, call `stuff()` in a loop within one block, and measure `block.gasUsed` vs. ETH balance change. The block gas will be consumed while the attacker's ETH balance remains unchanged (minus gas fees only).
