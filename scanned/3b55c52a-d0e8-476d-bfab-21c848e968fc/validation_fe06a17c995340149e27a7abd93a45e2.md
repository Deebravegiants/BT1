### Title
Single Broken or Reverting Asset Price Oracle Blocks Entire rsETH Price Update, Freezing Fee Minting and Price-Drop Protection — (File: `contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle._getTotalEthInProtocol()` iterates over every supported asset and makes an unbounded external call to each asset's price oracle. If any single oracle call reverts — due to the underlying external protocol being paused, the Chainlink feed reverting, or any other transient/permanent failure — the entire `updateRSETHPrice()` call reverts. This permanently blocks rsETH price updates, freezes protocol fee minting (unclaimed yield), and disables the automatic price-drop protection mechanism.

---

### Finding Description

`LRTOracle._getTotalEthInProtocol()` is the private function that computes total ETH value across all supported assets:

```solidity
// contracts/LRTOracle.sol lines 331-349
function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
    address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
    address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
    uint256 supportedAssetCount = supportedAssets.length;

    for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
        address asset = supportedAssets[assetIdx];
        uint256 assetER = getAssetPrice(asset);   // <-- external call, no try-catch
        uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
        totalETHInProtocol += totalAssetAmt.mulWad(assetER);
        unchecked { ++assetIdx; }
    }
}
```

`getAssetPrice(asset)` delegates to an external `IPriceFetcher` contract:

```solidity
// contracts/LRTOracle.sol line 157
return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
```

The concrete price fetchers each make external calls to third-party protocols:

- `EthXPriceOracle` → `IETHXStakePoolsManager.getExchangeRate()` (Stader protocol, pausable)
- `RETHPriceOracle` → `IrETH.getExchangeRate()` (Rocket Pool contract)
- `SfrxETHPriceOracle` → `ISfrxETH.pricePerShare()` (Frax protocol)
- `SwETHPriceOracle` → `ISwETH.getRate()` (Swell protocol)
- `ChainlinkPriceOracle` → `AggregatorV3Interface.latestRoundData()` (Chainlink feed)

There is no `try-catch` around any of these calls. If any one of them reverts for any reason, `_getTotalEthInProtocol()` reverts, which causes `_updateRsETHPrice()` to revert, which causes both `updateRSETHPrice()` and `updateRSETHPriceAsManager()` to revert.

---

### Impact Explanation

**Permanent freezing of unclaimed yield (Medium):** Protocol fee minting occurs inside `_updateRsETHPrice()`. If price updates are blocked, `_checkAndUpdateDailyFeeMintLimit` and the `IRSETH.mint(treasury, ...)` call at lines 304–307 can never execute. All accrued protocol fees are permanently frozen.

**Price-drop protection disabled (Medium):** The auto-pause logic at lines 270–282 — which pauses `LRTDepositPool` and `LRTWithdrawalManager` when the rsETH price drops beyond `pricePercentageLimit` — can never trigger. A slashing event that should auto-pause the protocol will go undetected.

**Stale rsETH price causes share dilution (Medium):** The stored `rsETHPrice` is frozen at its last value. As the protocol continues to accrue yield, the true price rises above the stored price. New depositors calling `depositAsset()` or `depositETH()` receive more rsETH than they are entitled to (computed via the stale stored price), diluting existing rsETH holders' share of the TVL.

---

### Likelihood Explanation

The protocol supports multiple LST assets, each with its own external price oracle. Any of the following realistic events triggers the freeze:

- A Chainlink price feed for any supported asset becomes temporarily unavailable or reverts (e.g., sequencer downtime on L2, feed deprecation).
- The Stader `ETHXStakePoolsManager`, Rocket Pool `rETH`, Frax `sfrxETH`, or Swell `swETH` contracts are paused for an emergency upgrade or exploit response.
- Any supported asset's underlying protocol is temporarily halted.

The protocol currently supports at least five distinct LST assets, each with an independent external dependency. The probability that at least one of these external calls reverts at some point during the protocol's lifetime is non-trivial.

---

### Recommendation

Wrap each `getAssetPrice(asset)` call inside `_getTotalEthInProtocol()` in a `try-catch` block. On failure, either skip the asset (using its last known price or zero) or emit an event and continue:

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    try IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset) returns (uint256 assetER) {
        uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
        totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    } catch {
        emit AssetPriceFetchFailed(asset);
        // Optionally revert the whole update, or use a cached price
    }
    unchecked { ++assetIdx; }
}
```

Alternatively, maintain a per-asset cached price that is updated on success and used as a fallback on failure.

---

### Proof of Concept

1. Protocol has two supported assets: `stETH` (oracle: `OneETHPriceOracle`, always returns `1e18`) and `ETHx` (oracle: `EthXPriceOracle`, calls `IETHXStakePoolsManager.getExchangeRate()`).
2. Stader's `ETHXStakePoolsManager` is paused for an emergency upgrade. Its `getExchangeRate()` reverts.
3. Any caller invokes `LRTOracle.updateRSETHPrice()`.
4. `_updateRsETHPrice()` calls `_getTotalEthInProtocol()`.
5. The loop reaches `ETHx`, calls `EthXPriceOracle.getAssetPrice(ETHx)`, which calls `IETHXStakePoolsManager.getExchangeRate()` — this reverts.
6. The revert propagates up: `_getTotalEthInProtocol()` reverts → `_updateRsETHPrice()` reverts → `updateRSETHPrice()` reverts.
7. `rsETHPrice` is frozen. Fee minting is blocked. Price-drop auto-pause cannot trigger. New depositors receive inflated rsETH amounts at the expense of existing holders.
