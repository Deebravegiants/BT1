### Title
Stale `rsETHPrice` Exploitable for Discounted rsETH Minting When Checkpoint Reward Exceeds `pricePercentageLimit` — (`contracts/LRTOracle.sol`)

---

### Summary

When an EigenLayer checkpoint credits consensus rewards large enough to push `newRsETHPrice` above `highestRsethPrice * (1 + pricePercentageLimit)`, the public `updateRSETHPrice()` reverts for all non-manager callers. The stored `rsETHPrice` remains stale. Because `LRTDepositPool.getRsETHAmountToMint()` divides directly by the stale `lrtOracle.rsETHPrice()`, any depositor can mint rsETH at the old, lower price during this window, diluting existing holders.

---

### Finding Description

`_updateRsETHPrice()` computes `newRsETHPrice` and then checks: [1](#0-0) 

If the price increase exceeds the limit and the caller is not a manager, the function reverts with `PriceAboveDailyThreshold`. The `rsETHPrice` state variable is **not updated** — it stays at its previous (lower) value.

The deposit pool mints rsETH using that stored price: [2](#0-1) 

There is no freshness check on `rsETHPrice` in the deposit path. Deposits remain open (the deposit pool is not paused by this condition — only the oracle's price-decrease branch triggers a pause). An attacker can therefore call `depositETH()` or `depositAsset()` and receive rsETH calculated at the stale lower price. [3](#0-2) 

The manager eventually calls `updateRSETHPriceAsManager()`: [4](#0-3) 

Once the price is updated, the attacker's rsETH tokens are worth more ETH than was deposited, at the direct expense of pre-existing holders whose share of the protocol's ETH is diluted.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Let `T` = total ETH in protocol, `S` = rsETH supply, `P_actual` = true price, `P_stale` = stale price (`P_stale < P_actual`), `D` = attacker deposit.

- Attacker mints `D / P_stale` rsETH instead of the correct `D / P_actual`.
- After the manager updates the price, the attacker redeems for more ETH than deposited.
- The surplus comes from existing holders, whose per-token ETH entitlement is reduced because the rsETH supply was inflated beyond what the deposited ETH warranted.

---

### Likelihood Explanation

- `pricePercentageLimit` is set by the admin to a small value (the code comments give 1% = `1e16` as the reference unit).
- EigenLayer checkpoints accumulate consensus rewards over time. A checkpoint that has not been finalized for several weeks can easily credit rewards exceeding 1% of the current price in a single update.
- The deposit pool remains open during the stale-price window; no additional privilege is required to exploit it.
- The attacker does not need to front-run any specific transaction — the window persists until the manager acts, which may span multiple blocks.

---

### Recommendation

1. **Enforce price freshness at deposit time**: record a `lastPriceUpdateTimestamp` in `_updateRsETHPrice()` and revert in `getRsETHAmountToMint()` (or `_beforeDeposit()`) if the price is older than a configurable staleness threshold.
2. **Pause deposits when the price update is blocked**: if `isPriceIncreaseOffLimit` is true and the caller is not a manager, also pause the deposit pool (mirroring the behavior already implemented for the price-decrease branch at lines 277–281).
3. **Raise `pricePercentageLimit` conservatively**: set it high enough that a single realistic checkpoint cannot exceed it, or implement a multi-step update mechanism that allows the price to catch up in increments without leaving deposits open at a stale price.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Fork test (Foundry) — run against a mainnet/testnet fork
// Preconditions:
//   - pricePercentageLimit = 1e16 (1%)
//   - A large EigenLayer checkpoint has just been verified, crediting >1% rewards

function testStaleDepositExploit() public {
    // 1. Confirm public updateRSETHPrice() reverts
    vm.expectRevert(ILRTOracle.PriceAboveDailyThreshold.selector);
    lrtOracle.updateRSETHPrice();

    uint256 stalePriceBeforeUpdate = lrtOracle.rsETHPrice();

    // 2. Attacker deposits at stale price
    uint256 attackerDeposit = 10 ether;
    vm.deal(attacker, attackerDeposit);
    vm.prank(attacker);
    lrtDepositPool.depositETH{value: attackerDeposit}(0, "");

    uint256 attackerRsETH = rsETH.balanceOf(attacker);
    // attackerRsETH > attackerDeposit / truePrice  (minted at stale lower price)

    // 3. Manager updates price
    vm.prank(manager);
    lrtOracle.updateRSETHPriceAsManager();

    uint256 updatedPrice = lrtOracle.rsETHPrice();
    assertGt(updatedPrice, stalePriceBeforeUpdate);

    // 4. Attacker's rsETH is now worth more than deposited
    uint256 attackerETHValue = attackerRsETH * updatedPrice / 1e18;
    assertGt(attackerETHValue, attackerDeposit); // profit at existing holders' expense
}
```
