### Title
Unvalidated Zero Price from Chainlink Feed Triggers Protocol-Wide Automatic Pause — (`contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` does not validate that the Chainlink `answer` is greater than zero. If a feed returns `0` (e.g., circuit-breaker activation or a depegged asset), the zero price propagates into `LRTOracle._getTotalEthInProtocol()`, artificially deflating `totalETHInProtocol`. This causes `newRsETHPrice` to drop below `highestRsethPrice` by more than `pricePercentageLimit`, triggering the automatic pause of `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` — freezing all user deposits and withdrawals until an admin manually unpauses.

---

### Finding Description

**Root cause — missing price validation in `ChainlinkPriceOracle.getAssetPrice()`:** [1](#0-0) 

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

There is no check that `price > 0`. If the feed returns `0`, the function returns `0` without reverting.

**Propagation into `_getTotalEthInProtocol()`:** [2](#0-1) 

For each supported asset, `getAssetPrice(asset)` is called. A zero return zeroes out that asset's entire ETH contribution (`totalAssetAmt.mulWad(0) == 0`), deflating `totalETHInProtocol`.

**Automatic pause trigger in `_updateRsETHPrice()`:** [3](#0-2) 

```solidity
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;
    }
```

If the deflated `newRsETHPrice` falls more than `pricePercentageLimit` below `highestRsethPrice`, all three contracts are paused atomically.

**Public entrypoint — anyone can trigger this:** [4](#0-3) 

`updateRSETHPrice()` is `public whenNotPaused`, so any account can call it once the feed returns 0.

---

### Impact Explanation

**Impact: Medium — Temporary freezing of funds.**

All user deposits (`LRTDepositPool`) and withdrawals (`LRTWithdrawalManager`) are frozen until an admin calls `unpause()` on each contract. The freeze is not permanent (admin can recover), but it is unilaterally triggerable by any caller whenever a Chainlink feed transiently returns 0. The question's claim of "Critical. Protocol insolvency" is overstated — no funds are lost or stolen, and the price is not actually written to storage when the pause branch is taken (the function returns early at line 281).

---

### Likelihood Explanation

Chainlink feeds can return `answer = 0` in documented edge cases: newly deployed feeds before the first round, circuit-breaker conditions, or depegged assets. The missing `price > 0` guard is a well-known Chainlink integration anti-pattern. Once the feed transiently returns 0, any public caller can race to invoke `updateRSETHPrice()` and freeze the protocol. Likelihood is **Low-Medium** — it requires an external feed anomaly, but no privileged access.

---

### Recommendation

Add a non-zero price guard in `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Additionally, consider adding a staleness check (`updatedAt`) and a minimum price floor to harden the oracle against other Chainlink edge cases.

---

### Proof of Concept

```solidity
// Local fork or mock test — no mainnet required
function test_zeroPricePausesProtocol() public {
    // 1. Deploy mock Chainlink feed that returns answer = 0
    MockChainlinkFeed mockFeed = new MockChainlinkFeed(0);
    // 2. Set the feed for a supported asset (e.g., stETH)
    vm.prank(lrtManager);
    chainlinkOracle.updatePriceFeedFor(stETH, address(mockFeed));
    // 3. Set pricePercentageLimit to 1% (1e16)
    vm.prank(lrtAdmin);
    lrtOracle.setPricePercentageLimit(1e16);
    // 4. Anyone calls updateRSETHPrice
    lrtOracle.updateRSETHPrice();
    // 5. Assert all three contracts are paused
    assertTrue(lrtDepositPool.paused());
    assertTrue(lrtWithdrawalManager.paused());
    assertTrue(lrtOracle.paused());
    // 6. Deposits revert
    vm.expectRevert();
    lrtDepositPool.depositAsset(stETH, 1 ether, 0, "");
}
```
