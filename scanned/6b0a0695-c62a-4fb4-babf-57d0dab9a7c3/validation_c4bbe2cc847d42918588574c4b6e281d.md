### Title
Missing `limitDailyMint` Safety Mechanism in `RSETHPoolNoWrapper` Allows Unlimited Daily Deposits — (File: contracts/pools/RSETHPoolNoWrapper.sol)

---

### Summary

`RSETHPoolNoWrapper` completely omits the `limitDailyMint` safety mechanism that every other L2 pool variant enforces on its `deposit()` functions. Any unprivileged depositor can drain the pool's entire rsETH balance in a single transaction, temporarily preventing all other users from depositing until the pool is manually refilled.

---

### Finding Description

The protocol explicitly implements a daily mint limit safety mechanism across its L2 pool family. `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, and `RSETHPoolV3WithNativeChainBridge` all declare the `limitDailyMint` modifier and apply it to both their ETH and token `deposit()` entry points: [1](#0-0) [2](#0-1) [3](#0-2) 

`RSETHPoolV3ExternalBridge` mirrors this pattern identically: [4](#0-3) [5](#0-4) 

`RSETHPoolNoWrapper`, however, declares **none** of the required state variables (`dailyMintLimit`, `dailyMintAmount`, `lastMintDay`, `startTimestamp`) and applies **no** daily limit modifier to either `deposit()` overload: [6](#0-5) [7](#0-6) 

The ETH deposit path in `RSETHPoolNoWrapper` transfers the full computed rsETH amount directly from the pool's pre-loaded balance with no per-day cap: [8](#0-7) 

This is the exact asymmetric-validation pattern from the reference report: one code path (all `RSETHPoolV3` variants) enforces a protective bound; the parallel path (`RSETHPoolNoWrapper`) does not, allowing the bound to be bypassed entirely.

---

### Impact Explanation

**Medium — Temporary freezing of funds.**

A single depositor with sufficient ETH can call `RSETHPoolNoWrapper.deposit()` in one transaction and receive the pool's entire rsETH balance. After that, every subsequent `deposit()` call from any other user reverts because `rsETH.safeTransfer()` fails on an empty balance. The pool is inoperable until the protocol operator manually bridges and reloads rsETH, which is an off-chain, non-atomic operation. During that window all L2 deposits on the affected chain are frozen.

---

### Likelihood Explanation

High. The entry point is fully public, requires no role, no whitelist, and no special token. Any user holding enough ETH (or a supported LST) on the deployed chain (Unichain, Arbitrum) can trigger this in a single call. The pool's rsETH balance is finite and observable on-chain, so the exact drain amount is trivially computable before the transaction is sent.

---

### Recommendation

Add the same `limitDailyMint` modifier — and the four backing state variables (`dailyMintLimit`, `dailyMintAmount`, `lastMintDay`, `startTimestamp`) — to `RSETHPoolNoWrapper`, mirroring the implementation in `RSETHPoolV3`: [9](#0-8) 

Apply the modifier to both `deposit(string)` and `deposit(address,uint256,string)` in `RSETHPoolNoWrapper`, and expose a `reinitialize` function (as done in `RSETHPoolV3`) to set the limit and start timestamp without redeploying.

---

### Proof of Concept

1. `RSETHPoolNoWrapper` is deployed on Unichain with, say, 1 000 rsETH pre-loaded.
2. Attacker observes the balance via `rsETH.balanceOf(poolAddress)`.
3. Attacker calls `RSETHPoolNoWrapper.deposit{value: X}("")` where `X` is chosen so that `viewSwapRsETHAmountAndFee(X).rsETHAmount == 1 000 rsETH`.
4. `rsETH.safeTransfer(attacker, 1000e18)` succeeds; pool balance drops to 0.
5. Any subsequent user calling `deposit()` receives a revert from `safeTransfer` (insufficient balance).
6. All L2 deposits on that chain are frozen until the protocol operator bridges new rsETH into the pool.

No privileged access, no oracle manipulation, and no multi-block setup is required.

**File:** contracts/pools/RSETHPoolV3.sol (L96-125)
```text
    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```

    function deposit(string memory referralId)
        external
        payable
        nonReentrant
        whenNotPaused
        limitDailyMint(msg.value, ETH_IDENTIFIER)
    {
        if (!isEthDepositEnabled) revert EthDepositDisabled();
        uint256 amount = msg.value;

        if (amount == 0) revert InvalidAmount();

        (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);

        feeEarnedInETH += fee;

        wrsETH.mint(msg.sender, rsETHAmount);

        emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
    }
