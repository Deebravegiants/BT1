### Title
Zero `totalETHInProtocol` with Non-Zero `rsethSupply` Triggers Spurious Auto-Pause — (`contracts/LRTOracle.sol`)

### Summary

`_updateRsETHPrice()` has no guard for the case where `totalETHInProtocol == 0` while `rsethSupply > 0`. This causes `newRsETHPrice = 0`, which is guaranteed to exceed any configured `pricePercentageLimit` downside threshold, triggering the auto-pause and freezing deposits, withdrawals, and the oracle itself — with no actual slashing having occurred.

---

### Finding Description

**Root cause — `_updateRsETHPrice()`, `contracts/LRTOracle.sol`**

The function guards only the `rsethSupply == 0` case: [1](#0-0) 

When `rsethSupply > 0` it proceeds unconditionally to compute: [2](#0-1) 

If `totalETHInProtocol == 0`, `newRsETHPrice` becomes `0`. The downside-protection block then compares it to `highestRsethPrice`: [3](#0-2) 

A 100% drop always exceeds any finite `pricePercentageLimit`, so `isPriceDecreaseOffLimit` is `true` and the protocol is paused.

**How `totalETHInProtocol == 0` with `rsethSupply > 0` is reached**

`_getTotalEthInProtocol()` sums `getTotalAssetDeposits(asset)` for every supported asset: [4](#0-3) 

`getTotalAssetDeposits` covers the deposit pool, all NDCs, EigenLayer staked + unstaking, converter, and unstaking vault: [5](#0-4) 

The realistic path that produces `totalETHInProtocol == 0` while `rsethSupply > 0`:

1. Protocol accumulates staking rewards; `updateRSETHPrice()` is called while `totalETHInProtocol > previousTVL`.
2. Protocol fee rsETH is minted to the treasury: [6](#0-5) 
3. All non-treasury users initiate and complete withdrawals, burning their rsETH. The treasury's fee rsETH is **never burned** during user withdrawals.
4. After all user withdrawals settle: every asset location (pool, NDCs, EigenLayer, unstaking vault) holds zero. `rsethSupply = treasury_fee_rsETH > 0`.
5. Any caller invokes the public `updateRSETHPrice()`: [7](#0-6) 
6. `newRsETHPrice = 0`, `diff = highestRsethPrice`, `isPriceDecreaseOffLimit = true` → auto-pause fires, freezing `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle`.

A secondary (simpler) path: rounding in the rsETH burn calculation leaves 1 wei of rsETH outstanding after the last withdrawal, with zero backing assets.

---

### Impact Explanation

**Medium — Temporary freezing of funds.**

Once the auto-pause fires, `LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()` are called, blocking all deposits and withdrawals. `LRTOracle` itself is also paused, preventing any price update. Recovery requires an admin to call `unpause()` on each contract. During the freeze, users with pending withdrawal requests cannot claim their assets.

---

### Likelihood Explanation

Moderate-low. The scenario requires the treasury to hold fee rsETH (which happens whenever the protocol earns rewards and `updateRSETHPrice()` is called with a TVL increase) and for all user-held rsETH to be burned through withdrawals. This is an extreme but reachable state — particularly relevant during protocol wind-down or after a large coordinated exit. No admin compromise, front-running, or external oracle manipulation is required. `updateRSETHPrice()` is `public` and callable by any EOA.

---

### Recommendation

Add an explicit guard at the top of `_updateRsETHPrice()` after the `rsethSupply == 0` check:

```solidity
if (totalETHInProtocol == 0) {
    // No assets in protocol; skip price update to avoid spurious pause.
    return;
}
```

Alternatively, treat `totalETHInProtocol == 0 && rsethSupply > 0` as a revert rather than a silent return, so operators are alerted. Also consider whether treasury-held fee rsETH should be excluded from `rsethSupply` when computing the price, or whether the treasury should periodically redeem its rsETH to keep the backing ratio consistent.

```solidity
// Pseudocode — wire up real mocks for a local Foundry test

function test_zeroPriceTriggersPause() public {
    // 1. Setup: rsETHPrice = 1 ether, highestRsethPrice = 1 ether
    //    rsethSupply = 1 wei (treasury fee rsETH minted in a prior update)
    //    All asset balances across DepositPool / NDCs / EigenLayer / UnstakingVault = 0

    // Preconditions
    assertEq(rseth.totalSupply(), 1);          // 1 wei rsETH in treasury
    assertEq(oracle.highestRsethPrice(), 1 ether);
    // getTotalAssetDeposits returns 0 for all supported assets

    // 2. Anyone calls updateRSETHPrice()
    oracle.updateRSETHPrice();

    // 3. Assert auto-pause fired
    assertTrue(oracle.paused());
    assertTrue(depositPool.paused());
    assertTrue(withdrawalManager.paused());
}
```

The test is locally reproducible on unmodified contracts with mocked asset balances set to zero and `rsethSupply` set to 1 wei via a prior fee-mint call.
