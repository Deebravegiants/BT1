### Title
`RSETHPoolV2NBA` on Blast Does Not Configure Gas Yield Nor ETH Native Yield Mode - (File: contracts/pools/RSETHPoolV2NBA.sol)

### Summary
The `RSETHPoolV2NBA` contract is deployed on Blast L2 and accepts ETH from users, yet its `initialize()` function never calls `IBlast.configureClaimableGas()` or `IBlast.configureClaimableYield()`. As a result, all gas yield accrued from user interactions and all native ETH yield on the contract's ETH balance are permanently unclaimable and lost to the protocol.

### Finding Description
On Blast L2, every contract that holds ETH or receives gas fees can opt into claimable yield modes by calling the Blast precompile's `configureClaimableGas()` and `configureClaimableYield()` during initialization. Without this configuration, the contract defaults to `VOID` mode for both gas and ETH yield — meaning all accrued yield is permanently discarded rather than being claimable by the protocol.

`RSETHPoolV2NBA` is confirmed deployed on Blast at `0x1558959f1a032F83f24A14Ff539944A926C51bdf`. Its `initialize()` function sets up roles, the oracle, and fee parameters, but contains no call to any Blast yield configuration: [1](#0-0) 

The contract accumulates ETH in two ways:
1. User deposits via `deposit()` (payable), where ETH sits in the contract until bridged.
2. Fee accumulation via `feeEarnedInETH`. [2](#0-1) 

The only Blast-specific configuration in the entire codebase is in `MerkleBlastPointsDistributor`, which only configures the Blast **Points** operator — not gas yield or ETH yield: [3](#0-2) 

A grep across all production contracts for `configureClaimableGas`, `configureClaimableYield`, `IBlast`, `BlastYield`, `claimGas`, `claimYield`, and `YieldMode` returns zero results outside of `MerkleBlastPointsDistributor`, confirming no gas or ETH yield is configured anywhere.

### Impact Explanation
**Medium — Permanent freezing of unclaimed yield.**

- All ETH native yield (Blast's rebasing ETH) accrued on the contract's ETH balance since deployment is permanently lost in `VOID` mode and cannot be recovered.
- All gas yield from every user call to `deposit()`, `withdrawFees()`, and `moveAssetsForBridging()` is permanently discarded.
- There is no admin function to retroactively configure yield mode or claim accrued yield, so the loss is irreversible for the period since deployment.

### Likelihood Explanation
**High.** The contract is already deployed and actively used on Blast. Every ETH deposit and every gas-consuming transaction permanently discards yield. No attacker action is required — the loss is automatic and continuous from the moment of deployment.

### Recommendation
In the `initialize()` function of `RSETHPoolV2NBA`, add calls to the Blast precompile to configure claimable gas and ETH yield modes:

```solidity
interface IBlast {
    function configureClaimableGas() external;
    function configureClaimableYield() external;
    function claimAllGas(address contractAddress, address recipient) external returns (uint256);
    function claimAllYield(address contractAddress, address recipient) external returns (uint256);
}

// In initialize():
IBlast(0x4300000000000000000000000000000000000002).configureClaimableGas();
IBlast(0x4300000000000000000000000000000000000002).configureClaimableYield();
```

Additionally, expose an admin-only function to claim accrued gas and ETH yield to the protocol treasury. Also evaluate whether `RsETHTokenWrapper` (also deployed on Blast) requires the same treatment for any ERC20 rebasing tokens it may hold.

### Proof of Concept

1. `RSETHPoolV2NBA` is deployed on Blast at `0x1558959f1a032F83f24A14Ff539944A926C51bdf` (confirmed in README).
2. A user calls `deposit{value: 1 ether}("ref")` — ETH enters the contract, gas is consumed, wrsETH is minted.
3. Because `initialize()` never called `configureClaimableGas()` or `configureClaimableYield()`, the Blast precompile treats both as `VOID`.
4. The ETH native yield on the 1 ETH balance accrues to the Blast protocol, not to `RSETHPoolV2NBA`.
5. The gas yield from the `deposit()` call is similarly discarded.
6. There is no function in `RSETHPoolV2NBA` to claim either yield type, making the loss permanent. [4](#0-3)
