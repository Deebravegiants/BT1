### Title
`collectInterestToTreasury` Callable When `LRTWithdrawalManager` Is Paused, Allowing Yield Extraction While Users Are Locked Out - (File: contracts/LRTWithdrawalManager.sol)

---

### Summary

`LRTWithdrawalManager.collectInterestToTreasury()` lacks the `whenNotPaused` modifier. When the contract is paused, all user-facing withdrawal functions are blocked, but the LRT operator can still call `collectInterestToTreasury()` to extract Aave-accrued interest to the protocol treasury. This is a direct analog to the BakerFi M-06 finding: a privileged operational function bypasses the pause, allowing yield/fee extraction while users are frozen.

---

### Finding Description

`LRTWithdrawalManager` inherits `PausableUpgradeable` and correctly gates all user-facing withdrawal functions with `whenNotPaused`:

- `initiateWithdrawal` — `whenNotPaused` [1](#0-0) 
- `completeWithdrawal` — `whenNotPaused` [2](#0-1) 
- `instantWithdrawal` — `whenNotPaused` [3](#0-2) 
- `unlockQueue` — `whenNotPaused` [4](#0-3) 
- `depositIdleETHToAave` — `whenNotPaused` [5](#0-4) 

However, `collectInterestToTreasury()` has only `nonReentrant` and `onlyLRTOperator` — **no `whenNotPaused`**:

```solidity
function collectInterestToTreasury() external nonReentrant onlyLRTOperator returns (uint256 interestAmount) {
    if (!_checkAaveHealth()) revert AaveHealthCheckFailed();
    return _collectInterestToTreasury();
}
``` [6](#0-5) 

The internal `_collectInterestToTreasury()` withdraws the excess aWETH balance above the tracked principal from Aave and sends it directly to the protocol treasury:

```solidity
function _collectInterestToTreasury() internal returns (uint256 interestAmount) {
    uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
    uint256 principal = totalETHDepositedToAave;
    if (aaveBalance <= principal) return 0;
    interestAmount = aaveBalance - principal;
    aaveWETHGateway.withdrawETH(aavePool, interestAmount, address(this));
    address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
    (bool sent,) = payable(treasury).call{ value: interestAmount }("");
    ...
}
``` [7](#0-6) 

The ETH deposited to Aave originates from user withdrawal requests that have been unlocked via `unlockQueue`. This ETH is held in trust to service pending user withdrawals. The Aave interest accruing on it represents yield on user-committed funds.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

When `LRTWithdrawalManager` is paused (e.g., due to a security incident, oracle anomaly, or price deviation triggering the auto-pause in `LRTOracle._updateRsETHPrice()`), users cannot initiate or complete withdrawals. Their rsETH is locked and their committed ETH sits in Aave earning interest. Despite the pause, the LRT operator can call `collectInterestToTreasury()` to extract that Aave interest to the protocol treasury. The yield accruing on user-committed funds is siphoned to the treasury while users have no ability to exit their positions.

---

### Likelihood Explanation

**Medium.** The `PAUSER_ROLE` and the LRT operator are distinct roles. A pause event (whether manual or auto-triggered by the oracle's price-deviation circuit breaker) does not prevent the operator from performing routine yield collection. The operator may call `collectInterestToTreasury()` during a pause without any collusion — simply as a routine operation unaware of or indifferent to the paused state. The Aave integration is an active feature with a dedicated operator workflow, making this call realistic during any pause period.

---

### Recommendation

Add the `whenNotPaused` modifier to `collectInterestToTreasury()`:

```solidity
function collectInterestToTreasury()
    external
    nonReentrant
    whenNotPaused          // <-- add this
    onlyLRTOperator
    returns (uint256 interestAmount)
{
    if (!_checkAaveHealth()) revert AaveHealthCheckFailed();
    return _collectInterestToTreasury();
}
``` [6](#0-5) 

---

### Proof of Concept

1. Aave integration is enabled; ETH from unlocked user withdrawals has been deposited to Aave via `unlockQueue` → `depositToAaveExternal`.
2. A security event triggers a pause: `PAUSER_ROLE` calls `LRTWithdrawalManager.pause()`.
3. Users attempt `completeWithdrawal` — reverts with `Pausable: paused`.
4. LRT operator calls `collectInterestToTreasury()` — **succeeds** because no `whenNotPaused` check exists.
5. `_collectInterestToTreasury()` withdraws `aaveBalance - totalETHDepositedToAave` from Aave and transfers it to the protocol treasury.
6. Users remain locked out; the yield on their committed ETH has been extracted to the treasury. [8](#0-7) [2](#0-1) [6](#0-5) [7](#0-6)
