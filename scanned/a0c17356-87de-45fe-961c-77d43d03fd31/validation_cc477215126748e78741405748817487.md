### Title
Missing Daily Mint Limit in `RSETHPoolNoWrapper` Allows Unlimited Pool Drainage — (File: contracts/pools/RSETHPoolNoWrapper.sol)

---

### Summary
`RSETHPoolNoWrapper` is the only pool variant that lacks the `limitDailyMint` safety mechanism present in every other pool. An unprivileged depositor can drain the pool's entire pre-loaded rsETH balance in a single transaction, temporarily freezing deposit access for all other users until an operator manually refills the pool.

---

### Finding Description
Every other pool variant — `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, `RSETHPoolV3WithNativeChainBridge`, and `RSETHPoolV2ExternalBridge` — declares the storage variables `dailyMintLimit`, `dailyMintAmount`, `lastMintDay`, and `startTimestamp`, and enforces the `limitDailyMint` modifier on both `deposit()` overloads.

`RSETHPoolNoWrapper` declares none of these variables and applies no such modifier. Both `deposit()` functions are completely unbounded:

```solidity
// contracts/pools/RSETHPoolNoWrapper.sol
function deposit(string memory referralId) external payable nonReentrant whenNotPaused {
    if (!isEthDepositEnabled) revert EthDepositDisabled();
    uint256 amount = msg.value;
    if (amount == 0) revert InvalidAmount();
    (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
    feeEarnedInETH += fee;
    rsETH.safeTransfer(msg.sender, rsETHAmount);   // no daily cap
    emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
}

---

### Impact Explanation
Once the pool's rsETH balance reaches zero, every subsequent `deposit()` call reverts at `rsETH.safeTransfer(msg.sender, rsETHAmount)` because the pool holds no rsETH to send. All other users are locked out of the deposit path until an operator manually bridges more rsETH into the pool. This is a **temporary freezing of the deposit functionality** — matching the "Medium. Temporary freezing of funds" impact tier — because legitimate users' ability to exchange ETH/LSTs for rsETH through this pool is suspended for an indeterminate period.

---

### Likelihood Explanation
The attack requires no special role or privilege. Any externally-owned account with sufficient ETH (or a supported LST) can call `deposit()` in a single transaction. The pool is deployed on chains such as Arbitrum and Unichain where gas costs are low, making large single-transaction deposits economically trivial. The attacker receives rsETH at the fair oracle rate, so the cost is only the spread/fee, not a net loss — making the attack economically rational as a griefing vector.

---

### Recommendation
Introduce the same `dailyMintLimit` mechanism used by all other pool variants into `RSETHPoolNoWrapper`:

1. Add storage variables: `dailyMintLimit`, `dailyMintAmount`, `lastMintDay`, `startTimestamp`.
2. Implement a `limitDailyMint` modifier that resets the counter each day and reverts when the cap is exceeded.
3. Apply the modifier to both `deposit(string)` and `deposit(address, uint256, string)`.
4. Add a `reinitializer` to set the initial `dailyMintLimit` and `startTimestamp`, and a setter guarded by `DEFAULT_ADMIN_ROLE`.

---

### Proof of Concept

1. `RSETHPoolNoWrapper` is deployed on Arbitrum with a pre-loaded balance of 500 rsETH.
2. Attacker observes the pool balance and the current oracle rate via `getRate()`.
3. Attacker calls `deposit{value: V}("")` where `V` is chosen so that `viewSwapRsETHAmountAndFee(V).rsETHAmount == 500 rsETH` (minus fee).
4. The pool transfers all 500 rsETH to the attacker; pool rsETH balance is now 0.
5. Any subsequent user calling `deposit{value: 1 ether}("")` receives a revert from `rsETH.safeTransfer` (ERC20 insufficient balance).
6. The deposit path is frozen until the operator bridges additional rsETH into the pool.

```text
    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```
