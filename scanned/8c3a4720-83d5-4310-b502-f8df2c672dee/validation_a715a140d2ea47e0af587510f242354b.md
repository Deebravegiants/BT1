### Title
Non-payable `PROTOCOL_TREASURY` permanently blocks Aave reconfiguration and freezes accrued yield — (`contracts/LRTWithdrawalManager.sol`)

---

### Summary

`configureAaveIntegration` atomically bundles an ETH push-transfer to `PROTOCOL_TREASURY` inside `_collectInterestToTreasury`. If the treasury address is a non-payable contract (e.g., a governance timelock, a DAO vault, or any contract without a `receive()` function), the transfer reverts and the entire reconfiguration reverts. Because every administrative exit path for the Aave integration calls the same `_collectInterestToTreasury` helper, the integration becomes permanently stuck: no reconfiguration, no disable, no emergency withdrawal, and no interest collection is possible.

---

### Finding Description

`_collectInterestToTreasury` performs two sequential, non-atomic operations in a single internal call:

1. Withdraws accrued interest from Aave via `aaveWETHGateway.withdrawETH` (ETH lands in the contract).
2. Pushes that ETH to `PROTOCOL_TREASURY` via a low-level `.call`. [1](#0-0) 

If the `.call` returns `false` (treasury is non-payable), the function reverts with `TreasuryTransferFailed`. Because the whole transaction reverts, the Aave withdrawal is also rolled back — the interest stays in Aave.

`_collectInterestToTreasury` is called unconditionally from **every** administrative path that touches a live Aave position:

| Function | Line |
|---|---|
| `configureAaveIntegration` | 442 |
| `setAaveIntegrationEnabled(false)` | 490 |
| `emergencyWithdrawFromAave` | 558 |
| `collectInterestToTreasury` (public) | 544 | [2](#0-1) [3](#0-2) [4](#0-3) 

Once any interest has accrued (`aaveAWETH.balanceOf > totalETHDepositedToAave`) and the treasury rejects ETH, all four paths revert. There is no code path that allows the manager to update Aave addresses, disable the integration, or drain the position without first successfully pushing ETH to the treasury.

---

### Impact Explanation

**Medium — Permanent freezing of unclaimed yield.**

- Accrued Aave interest cannot be collected; it is permanently locked in the Aave pool.
- The Aave integration cannot be reconfigured (e.g., to migrate to a new Aave pool version) or disabled.
- `emergencyWithdrawFromAave` (the PAUSER_ROLE escape hatch) is equally blocked.
- The principal deposited in Aave is also inaccessible via the normal `_withdrawFromAave` path because `configureAaveIntegration` and `setAaveIntegrationEnabled(false)` both revert before reaching `_withdrawFromAave`. [5](#0-4) 

---

### Likelihood Explanation

`PROTOCOL_TREASURY` is read dynamically from `lrtConfig` at call time. [6](#0-5) 

Protocol treasuries are commonly governance timelocks, DAO vaults, or multisigs that do not implement `receive()`. Any such address triggers the revert. No attacker action is required — the condition arises from a standard, legitimate treasury configuration. Interest accrues automatically over time in Aave, so the `aaveBalance > principal` precondition is met without any special setup.

---

### Recommendation

Decouple interest collection from reconfiguration. Two concrete options:

1. **Pull pattern**: Instead of pushing ETH to the treasury inside `_collectInterestToTreasury`, accumulate it in a `pendingTreasuryBalance` storage variable and provide a separate `claimTreasuryBalance()` function. Reconfiguration and disabling proceed regardless of whether the treasury can receive ETH.

2. **Graceful fallback**: If the `.call` to treasury fails, leave the withdrawn ETH in the contract and emit an event. Do not revert. Add a separate `sweepTreasuryETH()` function callable by the manager to retry the transfer later.

Either approach removes the hard dependency between treasury ETH delivery and Aave address management.

    }
}
```

The test demonstrates that with a non-payable treasury and accrued interest, `configureAaveIntegration` reverts and the old Aave addresses remain unchanged, permanently blocking reconfiguration and freezing the yield.

    function _collectInterestToTreasury() internal returns (uint256 interestAmount) {
        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        uint256 principal = totalETHDepositedToAave;

        // Return 0 if no interest or balance is less than principal (accounting for rounding)
        if (aaveBalance <= principal) return 0;

        interestAmount = aaveBalance - principal;

        aaveWETHGateway.withdrawETH(aavePool, interestAmount, address(this));

        address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        (bool sent,) = payable(treasury).call{ value: interestAmount }("");
        if (!sent) revert TreasuryTransferFailed();

        emit InterestCollectedToTreasury(interestAmount, treasury);
    }
```
