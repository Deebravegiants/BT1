### Title
`removeAllowedToken` Permanently Freezes Bridger-Deposited altAgETH When Outstanding Minted Wrapper Tokens Exist - (File: `contracts/agETH/AGETHTokenWrapper.sol`)

---

### Summary

`removeAllowedToken` sets `allowedTokens[asset] = false` with no check on outstanding wrapper supply or contract balance. Because `_withdraw` gates every redemption on `allowedTokens[asset]`, and because there is no `addAllowedToken` or rescue function, any altAgETH held in the contract at the time of removal becomes permanently unrecoverable.

---

### Finding Description

The contract has two distinct minting paths:

1. **`_deposit`** (line 125–132): user deposits altAgETH → wrapper minted 1:1. Supply and balance grow together.
2. **`mint`** (line 165–167): `MINTER_ROLE` mints wrapper tokens with **no corresponding altAgETH deposit**. Supply grows; balance stays flat.

`maxAmountToDepositBridgerAsset` (line 90–101) returns `totalSupply - balanceOfAssetInWrapper`. After `mint(K)` inflates supply by K without adding collateral, the bridger cap opens by exactly K, allowing `depositBridgerAssets` to pull K altAgETH into the contract. [1](#0-0) 

At this point the contract holds `S + K` altAgETH against `S + K` wrapper tokens. The S deposit-backed holders can still withdraw normally. After they do, the contract holds K altAgETH against K wrapper tokens that were minted by the bridge (not by depositors).

`removeAllowedToken` then sets the flag to false: [2](#0-1) 

Every subsequent call to `_withdraw` for that asset hits the guard at line 112 and reverts: [3](#0-2) 

There is no `addAllowedToken` (the comment at line 153 explicitly forbids it), no `rescue`/`emergencyWithdraw`, and no upgrade path that could unblock the tokens without a proxy upgrade. The K altAgETH is permanently locked. [4](#0-3) 

---

### Impact Explanation

K altAgETH deposited by the bridger as yield collateral for already-circulating wrapper tokens becomes permanently unrecoverable. Holders of the K bridge-minted wrapper tokens cannot redeem them. This matches **Medium – Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

The scenario is realistic during a token migration or deprecation: an admin calls `removeAllowedToken` to retire an old altAgETH variant without first verifying that all bridge-minted wrapper tokens have been redeemed. The MINTER_ROLE and BRIDGER_ROLE operations are normal bridge lifecycle actions, not adversarial. The only trigger is the admin's `removeAllowedToken` call while K > 0 wrapper tokens remain outstanding.

---

### Recommendation

Add a balance/supply invariant check inside `removeAllowedToken`:

```solidity
function removeAllowedToken(address _asset) external onlyRole(DEFAULT_ADMIN_ROLE) {
    uint256 balance = ERC20Upgradeable(_asset).balanceOf(address(this));
    require(balance == 0, "Outstanding collateral: drain before removal");
    allowedTokens[_asset] = false;
    emit TokenRemoved(_asset);
}
```

Alternatively, add a privileged `rescueToken` function so that if removal is needed urgently, the admin can first sweep the collateral to a safe address before disabling the token.
