### Title
Undistributed Reward Tokens Permanently Stuck When `totalKernelStaked` Drops to Zero During Active Reward Period - (File: contracts/KERNEL/KernelDepositPool.sol)

---

### Summary

In `KernelDepositPool.sol`, when all stakers call `initiateWithdrawal()` during an active reward period, `totalKernelStaked` immediately drops to zero. From that point, `rewardPerToken()` stops accumulating, and the remaining reward tokens for the period are never earned by anyone. No recovery function exists to retrieve these stuck tokens, and `notifyRewardAmount` itself reverts with `NoStakedTokens` when `totalKernelStaked == 0`, making the freeze permanent.

---

### Finding Description

**Root cause — `rewardPerToken()` freezes when `totalKernelStaked == 0`:** [1](#0-0) 

```solidity
function rewardPerToken() public view returns (uint256) {
    if (totalKernelStaked == 0) {
        return rewardPerTokenStored;
    }
    return rewardPerTokenStored + (rewardRate * (lastTimeRewardApplicable() - updatedAt) * DECIMAL_PRECISION)
        / totalKernelStaked;
}
```

When `totalKernelStaked` is zero, the function short-circuits and returns the stored value unchanged. The `rewardRate` continues to be non-zero (it was set by `notifyRewardAmount`) but no one accumulates those rewards.

**`initiateWithdrawal()` immediately decrements `totalKernelStaked`:** [2](#0-1) 

```solidity
function initiateWithdrawal(uint256 _amount) external nonReentrant updateReward(msg.sender) {
    ...
    balanceOf[msg.sender] -= _amount;
    totalKernelStaked -= _amount;
    ...
}
```

The staked balance is removed from `totalKernelStaked` at initiation time, not at claim time. If the last staker calls `initiateWithdrawal()` while a reward period is active (`block.timestamp < finishAt`), `totalKernelStaked` becomes 0 and all future reward emissions are lost.

**`notifyRewardAmount` blocks recovery:** [3](#0-2) 

```solidity
function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
    ...
    if (totalKernelStaked == 0) revert NoStakedTokens();
    ...
}
```

Even after the reward period ends, the admin cannot call `notifyRewardAmount` to restart distribution (which would at least allow new stakers to earn the stuck tokens), because the guard reverts when `totalKernelStaked == 0`.

**No `recoverERC20` or emergency withdrawal function exists** in the contract for the `rewardsToken`. The admin functions are limited to `setRewardsDuration`, `notifyRewardAmount`, `setWithdrawalDelay`, and `setMaxNumberOfWithdrawalsPerUser` — none of which can retrieve stuck reward tokens. [4](#0-3) 

---

### Impact Explanation

**Impact: Medium — Permanent freezing of unclaimed yield.**

The `rewardsToken` balance corresponding to `rewardRate × (finishAt − T)` (where T is the timestamp when `totalKernelStaked` reached 0) is permanently locked in the contract. There is no path — admin or otherwise — to recover it. This matches the allowed impact scope: *"Medium. Permanent freezing of unclaimed yield."*

---

### Likelihood Explanation

**Likelihood: Low-Medium.**

Any staker can call `initiateWithdrawal()` at any time — this is a normal, permissionless user action. The scenario requires all stakers to withdraw during an active reward period. This can occur naturally during a coordinated exit, a loss of confidence event, or if a single large staker holds the majority of stake and withdraws. The `MAX_WITHDRAWALS_PER_USER` limit does not prevent this; it only caps concurrent pending withdrawals per user. [5](#0-4) 

---

### Recommendation

Add a `recoverERC20` function restricted to the admin role that allows recovering the `rewardsToken` balance in excess of what has been earned (i.e., `rewardsToken.balanceOf(address(this)) - totalEarnedButUnclaimed`). Alternatively, when `totalKernelStaked` drops to 0 mid-period, record the remaining undistributed rewards and allow the admin to reclaim them after `finishAt`.

---

### Proof of Concept

1. Admin calls `notifyRewardAmount(1_000e18)` with `duration = 30 days`. `rewardRate = 1_000e18 / 30 days`.
2. On day 3, all stakers call `initiateWithdrawal()`. `totalKernelStaked` → 0.
3. `rewardPerToken()` now returns `rewardPerTokenStored` unchanged for the remaining 27 days.
4. Approximately `rewardRate × 27 days ≈ 900e18` reward tokens are never earned by anyone.
5. After day 30, `finishAt` passes. The `rewardsToken` balance of the contract still holds ~900e18.
6. Admin attempts `notifyRewardAmount(...)` → reverts with `NoStakedTokens`.
7. No other function can extract the stuck `rewardsToken` balance. Funds are permanently frozen. [1](#0-0) [6](#0-5)

        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
