### Title
Missing Chainlink Price Staleness Check in `ChainlinkPriceOracle` Leads to Incorrect rsETH Price Calculation - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards the `updatedAt` timestamp and `answeredInRound` values, accepting stale prices without any freshness validation. This stale price propagates directly into `LRTOracle._getTotalEthInProtocol()`, corrupting the rsETH price used for all deposits, withdrawals, and fee minting.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches the Chainlink price as follows:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values of `latestRoundData()` — `roundId`, `answer`, `startedAt`, `updatedAt`, `answeredInRound` — are available, but only `price` is used. The `updatedAt` timestamp (which indicates when the price was last updated) and `answeredInRound` (which detects incomplete rounds) are both discarded. [1](#0-0) 

This oracle is the price source for every supported LST asset. `LRTOracle._getTotalEthInProtocol()` iterates over all supported assets, calls `getAssetPrice(asset)` for each, and multiplies by the asset's total deposit amount to compute total ETH in the protocol. [2](#0-1) 

That total ETH value is then used in `_updateRsETHPrice()` to compute `newRsETHPrice = (totalETHInProtocol - protocolFeeInETH) / rsethSupply`. [3](#0-2) 

The protocol is demonstrably aware of this risk: `ChainlinkOracleForRSETHPoolCollateral.sol`, used for pool collateral pricing, explicitly validates both `answeredInRound < roundID` and `timestamp == 0` before returning a price. [4](#0-3) 

The same protection is absent from the main `ChainlinkPriceOracle` that feeds the core rsETH price.

---

### Impact Explanation

**Temporary freezing of funds (Medium):** If a Chainlink feed returns a stale, lower price for any LST asset, `_getTotalEthInProtocol()` underestimates TVL. The computed `newRsETHPrice` drops. If the drop exceeds `pricePercentageLimit` relative to `highestRsethPrice`, `_updateRsETHPrice()` automatically pauses both `LRTDepositPool` and `LRTWithdrawalManager`, freezing all user deposits and withdrawals until an admin manually unpauses. [5](#0-4) 

**Contract fails to deliver promised returns (Low):** If a stale, higher price is accepted, `newRsETHPrice` is inflated. Depositors calling `getRsETHAmountToMint()` receive fewer rsETH tokens than they are entitled to, since `rsethAmountToMint = (amount * assetPrice) / rsETHPrice` — a higher denominator reduces the mint amount. [6](#0-5) 

---

### Likelihood Explanation

`updateRSETHPrice()` is a public, permissionless function callable by any address. [7](#0-6) 

Chainlink feeds go stale during L2 sequencer downtime, network congestion, or feed deprecation. The protocol is deployed on multiple chains, increasing exposure. An attacker can call `updateRSETHPrice()` at any moment a feed is stale to trigger the incorrect price update. The root cause is the missing validation in the contract code itself — not merely external behavior.

---

### Recommendation

Apply the same staleness checks already present in `ChainlinkOracleForRSETHPoolCollateral.sol` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

---

### Proof of Concept

1. A Chainlink LST/ETH feed (e.g., stETH/ETH) goes stale — its `updatedAt` timestamp is hours old.
2. Any unprivileged caller invokes `LRTOracle.updateRSETHPrice()`.
3. `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)`.
4. `ChainlinkPriceOracle` returns the stale (lower) price with no revert.
5. `_getTotalEthInProtocol()` underestimates total ETH.
6. `newRsETHPrice` is computed as lower than `highestRsethPrice`.
7. If the difference exceeds `pricePercentageLimit`, `LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()` are called automatically.
8. All user deposits and withdrawals are frozen until an admin manually unpauses.

        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
