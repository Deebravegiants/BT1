### Title
Removing an Allowed Token Permanently Blocks Bridger Collateral Deposit for Outstanding wrsETH — (`contracts/L2/RsETHTokenWrapper.sol`)

### Summary

`removeAllowedToken` sets `allowedTokens[_asset] = false` with no check for outstanding uncollateralized wrsETH backed by that asset. Once removed, `depositBridgerAssets` is permanently blocked for that asset because `maxAmountToDepositBridgerAsset` returns `0` for non-allowed tokens, causing the `CannotDeposit` revert. Any wrsETH minted via `mint()` (MINTER_ROLE) that was awaiting bridger collateral for that token becomes permanently undercollateralized until the token is re-added.

### Finding Description

The `mint()` function allows MINTER_ROLE to mint wrsETH on L2 without any immediate collateral deposit: [1](#0-0) 

The `depositBridgerAssets()` function is the designated mechanism for the BRIDGER_ROLE to later deposit the bridged L1 rsETH to back that already-minted wrsETH: [2](#0-1) 

It gates on `maxAmountToDepositBridgerAsset`, which returns `0` immediately if the token is not in the allowed list: [3](#0-2) 

`removeAllowedToken` performs no check for outstanding uncollateralized wrsETH before disabling the token: [4](#0-3) 

The result: after `removeAllowedToken(tokenA)` executes, any call to `depositBridgerAssets(tokenA, amount)` will revert with `CannotDeposit` regardless of how much wrsETH was minted against tokenA and is awaiting collateral. The `_withdraw` path is also blocked for the same token: [5](#0-4) 

### Impact Explanation

The wrsETH `totalSupply` can exceed the sum of all allowed-token balances held by the wrapper. The contract fails to deliver its core promise — that every unit of wrsETH is backed 1:1 by a deposited alt-rsETH token — without any loss of the bridger's funds (the bridger still holds the tokens, they simply cannot be deposited). This matches **Low: Contract fails to deliver promised returns, but doesn't lose value**.

### Likelihood Explanation

The TIMELOCK_ROLE removing a token is a routine governance action (e.g., deprecating a bridge route or responding to a security incident). The BRIDGER_ROLE's deposit is an off-chain-triggered, asynchronous operation. There is no on-chain coordination mechanism between the two roles, so the ordering conflict can occur without any malicious intent — a legitimate token removal can race with or precede a pending bridger deposit. No key compromise or collusion is required.

### Recommendation

Before removing a token, `removeAllowedToken` should verify that the wrapper holds at least as many tokens as the current wrsETH `totalSupply` attributable to that token, or more practically, revert if `maxAmountToDepositBridgerAsset(_asset) > 0` (i.e., there is still uncollateralized wrsETH outstanding for that asset):

```solidity
function removeAllowedToken(address _asset) external onlyRole(TIMELOCK_ROLE) {
    if (!allowedTokens[_asset]) revert TokenNotAllowed();
    // Ensure no outstanding uncollateralized wrsETH for this asset
    if (maxAmountToDepositBridgerAsset(_asset) > 0) revert CannotRemoveWithOutstandingDebt();
    allowedTokens[_asset] = false;
    emit TokenRemoved(_asset);
}
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

import "forge-std/Test.sol";
import "contracts/L2/RsETHTokenWrapper.sol";
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract MockAltRsETH is ERC20 {
    constructor() ERC20("altRsETH", "altRsETH") { _mint(msg.sender, 1000 ether); }
}

contract RsETHWrapperTest is Test {
    RsETHTokenWrapper wrapper;
    MockAltRsETH altRsETH;

    address admin    = address(1);
    address timelock = address(2);
    address minter   = address(3);
    address bridger  = address(4);

    function setUp() public {
        altRsETH = new MockAltRsETH();
        wrapper  = new RsETHTokenWrapper();
        wrapper.initialize(admin, bridger, address(altRsETH));

        vm.startPrank(admin);
        wrapper.grantRole(wrapper.TIMELOCK_ROLE(), timelock);
        wrapper.grantRole(wrapper.MINTER_ROLE(),   minter);
        vm.stopPrank();

        // Fund bridger with altRsETH to simulate L1→L2 bridge arrival
        altRsETH.transfer(bridger, 100 ether);
        vm.prank(bridger);
        altRsETH.approve(address(wrapper), 100 ether);
    }

    function test_removeAllowedToken_blocks_bridger_deposit() public {
        // Step 1: MINTER_ROLE mints 100 wrsETH on L2 (no collateral yet)
        vm.prank(minter);
        wrapper.mint(address(5), 100 ether);

        // Invariant: wrsETH supply > collateral held
        assertEq(wrapper.totalSupply(), 100 ether);
        assertEq(altRsETH.balanceOf(address(wrapper)), 0);

        // Step 2: TIMELOCK_ROLE removes the token (legitimate governance action)
        vm.prank(timelock);
        wrapper.removeAllowedToken(address(altRsETH));

        // Step 3: BRIDGER_ROLE tries to deposit collateral — reverts
        vm.prank(bridger);
        vm.expectRevert(RsETHTokenWrapper.CannotDeposit.selector);
        wrapper.depositBridgerAssets(address(altRsETH), 100 ether);

        // wrsETH totalSupply still 100, collateral still 0 → undercollateralized
        assertEq(wrapper.totalSupply(), 100 ether);
        assertEq(altRsETH.balanceOf(address(wrapper)), 0);
    }
}
```
