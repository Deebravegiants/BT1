### Title
rsETH Permanently Locked in Withdrawal Manager Without User-Accessible Cancel Mechanism - (File: contracts/LRTWithdrawalManager.sol)

### Summary
Users who call `initiateWithdrawal()` have their rsETH transferred into `LRTWithdrawalManager`, but there is no mechanism for users to cancel the request or recover their rsETH. Completion of any withdrawal is gated behind an operator-only `unlockQueue()` call. If the operator fails to advance `nextLockedNonce[asset]`, the rsETH is frozen in the contract indefinitely with no user-accessible recovery path.

### Finding Description

In `LRTWithdrawalManager.initiateWithdrawal()`, the user's rsETH is pulled from their wallet into the contract:

```solidity
IERC20(lrtConfig.rsETH()).safeTransferFrom(msg.sender, address(this), rsETHUnstaked);
```

A withdrawal request is then recorded and placed in a locked queue. The request nonce is assigned from `nextUnusedNonce[asset]`, while `nextLockedNonce[asset]` tracks the boundary of unlocked requests. [1](#0-0) 

When the user later calls `completeWithdrawal()`, the internal `_processWithdrawalCompletion()` enforces:

```solidity
if (usersFirstWithdrawalRequestNonce >= nextLockedNonce[asset]) revert WithdrawalLocked();
``` [2](#0-1) 

The only way to advance `nextLockedNonce[asset]` is via `unlockQueue()`, which is restricted to `onlyAssetTransferOrOperatorRole`: [3](#0-2) 

The contract exposes no `cancelWithdrawal()` function. The `instantWithdrawal()` path is also unavailable to users who have already initiated a withdrawal, because their rsETH is already held by the contract — not in their wallet — so `burnFrom(msg.sender, ...)` would fail. [4](#0-3) 

The full set of user-callable withdrawal functions is:
- `initiateWithdrawal` — locks rsETH in contract
- `completeWithdrawal` — blocked by `WithdrawalLocked` until operator acts
- `instantWithdrawal` — requires rsETH in user's wallet (not applicable post-initiation)

There is no path for a user to recover their rsETH without operator intervention. [5](#0-4) 

### Impact Explanation

Once `initiateWithdrawal()` is called, the user's rsETH is transferred to the contract and cannot be recovered by the user through any on-chain mechanism. The rsETH remains frozen until the operator calls `unlockQueue()`. If the operator is unavailable due to operational failure, key loss, or a protocol upgrade that changes the operator address, the rsETH is permanently frozen. This constitutes **temporary (potentially permanent) freezing of funds**.

### Likelihood Explanation

Medium. The operator role is a trusted, regularly active role expected to call `unlockQueue()` as part of normal protocol operations. However, the structural absence of a cancel mechanism means any disruption to operator availability — operational failure, key rotation errors, or a protocol upgrade — leaves all pending withdrawal rsETH irrecoverable by users. The risk is not theoretical: the protocol already has multiple `reinitializer` upgrade paths, any of which could disrupt operator continuity. [6](#0-5) 

### Recommendation

Add a `cancelWithdrawal(address asset)` function that allows a user to cancel their oldest pending withdrawal request **only if it has not yet been unlocked** (i.e., its nonce is still `>= nextLockedNonce[asset]`), returning the original `rsETHUnstaked` amount to the user and decrementing `assetsCommitted[asset]` accordingly. This mirrors the pattern used in EigenLayer's own withdrawal queue, which allows stakers to re-queue or cancel pending withdrawals.

### Proof of Concept

1. User holds 10 rsETH and calls `initiateWithdrawal(stETH, 10e18)`.
2. Contract pulls 10 rsETH from user. Request is assigned nonce `N`, where `N >= nextLockedNonce[stETH]`. [7](#0-6) 
3. User waits for `withdrawalDelayBlocks` (~57,600 blocks / 8 days) to pass.
4. User calls `completeWithdrawal(stETH)` → reverts with `WithdrawalLocked` because `N >= nextLockedNonce[stETH]`. [2](#0-1) 
5. Operator never calls `unlockQueue(stETH, ...)` (e.g., due to key loss or upgrade).
6. User's 10 rsETH is permanently locked in `LRTWithdrawalManager` with no on-chain recovery path. [8](#0-7)

    function unlockQueue(
        address asset,
        uint256 firstExcludedIndex,
        uint256 minimumAssetPrice,
        uint256 minimumRsEthPrice,
        uint256 maximumAssetPrice,
        uint256 maximumRsEthPrice
    )
        external
        nonReentrant
        onlySupportedAsset(asset)
        whenNotPaused
        onlyAssetTransferOrOperatorRole
        returns (uint256 rsETHBurned, uint256 assetAmountUnlocked)
    {
        ILRTOracle lrtOracle = ILRTOracle(lrtConfig.getContract(LRTConstants.LRT_ORACLE));
        ILRTUnstakingVault unstakingVault = ILRTUnstakingVault(lrtConfig.getContract(LRTConstants.LRT_UNSTAKING_VAULT));

        UnlockParams memory params = _createUnlockParams(lrtOracle, unstakingVault, asset);

        _validatePrices(
            params.rsETHPrice,
            params.assetPrice,
            minimumRsEthPrice,
            maximumRsEthPrice,
            minimumAssetPrice,
            maximumAssetPrice
        );

        if (params.totalAvailableAssets == 0) revert AmountMustBeGreaterThanZero();

        // Updates and unlocks withdrawal requests up to a specified upper limit or until allocated assets are fully
        // utilized.
        (rsETHBurned, assetAmountUnlocked) = _unlockWithdrawalRequests(
            asset, params.totalAvailableAssets, params.rsETHPrice, params.assetPrice, firstExcludedIndex
        );

        if (rsETHBurned != 0) IRSETH(lrtConfig.rsETH()).burnFrom(address(this), rsETHBurned);
        //Take the amount to distribute from vault
        unstakingVault.redeem(asset, assetAmountUnlocked);

        // If Aave integration is enabled and asset is ETH, deposit to Aave
        if (isAaveIntegrationEnabled && asset == LRTConstants.ETH_TOKEN && assetAmountUnlocked > 0) {
            try this.depositToAaveExternal(assetAmountUnlocked) { }
            catch (bytes memory reason) {
                emit AaveDepositFailed(assetAmountUnlocked, reason);
                // Silently fail if Aave deposit fails (e.g., pool at max capacity)
                // Funds remain in contract for withdrawals
            }
        }

        emit AssetUnlocked(asset, rsETHBurned, assetAmountUnlocked, params.rsETHPrice, params.assetPrice);
    }
