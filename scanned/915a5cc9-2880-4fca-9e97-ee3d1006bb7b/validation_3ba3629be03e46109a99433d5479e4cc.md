### Title
Pool Admin Bypasses Factory Fee Cap via Uncapped `setPoolBinAdditionalFees` — (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary
`MetricOmmPoolFactory` enforces `maxAdminSpreadFeeE6` to bound how much the pool admin can charge traders. However, `setPoolBinAdditionalFees` forwards `addFeeBuyE6`/`addFeeSellE6` directly to the pool with **no cap validation**, allowing the pool admin to impose per-bin fees up to `uint16` max (65,535 E6 = 6.5535%) regardless of what the factory owner has set as the admin spread cap.

### Finding Description

The factory maintains explicit fee caps to limit pool-admin power:

```
maxAdminSpreadFeeE6   ≤ HARD_MAX_SPREAD_FEE_E6 (200_000 = 20%)
maxAdminNotionalFeeE8 ≤ HARD_MAX_NOTIONAL_FEE_E8 (1_000_000 = 1%)
```

`setPoolAdminFees` correctly enforces these: [1](#0-0) 

But `setPoolBinAdditionalFees` passes values straight through with **zero cap check**: [2](#0-1) 

The pool's `setBinAdditionalFees` only validates the bin index, not the fee magnitudes: [3](#0-2) 

`addFeeBuyE6` and `addFeeSellE6` are `uint16` fields in `BinState`, so they accept values up to 65,535 E6 (6.5535%): [4](#0-3) 

These per-bin fees are applied **on top of** the global spread fee during every swap through that bin. The factory owner's intent to cap admin spread revenue is fully bypassed.

### Impact Explanation

A pool admin can set `addFeeBuyE6 = 65_535` and `addFeeSellE6 = 65_535` on every bin. Every trader swapping through those bins pays an extra ≈6.5% fee that the factory owner's `maxAdminSpreadFeeE6` cap was designed to prevent. If the factory owner sets `maxAdminSpreadFeeE6 = 0` to eliminate admin spread entirely, the pool admin can still extract up to 6.5535% per bin via this path. The excess fee is a direct, real-token loss for every trader.

### Likelihood Explanation

The pool admin is a semi-trusted role — the factory's cap system exists precisely because the pool admin is **not** fully trusted. A malicious or compromised pool admin can exploit this immediately after pool creation with a single transaction, with no timelock or governance step required.

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

### Proof of Concept

1. Factory owner deploys factory with `maxAdminSpreadFeeE6 = 0` (no admin spread allowed).
2. Pool admin calls `factory.setPoolAdminFees(pool, 0, 0)` — passes cap check.
3. Pool admin calls `factory.setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)` — **no cap check**, succeeds.
4. Every trader swapping through bin 0 now pays an extra 6.5535% fee routed to the admin, violating the factory owner's zero-admin-spread policy. [5](#0-4) [3](#0-2)
