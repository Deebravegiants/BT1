### Title
`bridgeKernelToBSC()` Missing `whenNotPaused` Modifier Bypasses Emergency Pause ŌĆö (`contracts/KERNEL/KernelVaultETH.sol`)

### Summary
`KernelVaultETH` inherits `PausableUpgradeable` and applies `whenNotPaused` to all user-facing deposit functions, but omits it from the operator-facing `bridgeKernelToBSC()`. An operator can therefore execute a cross-chain token transfer while the contract is paused, violating the invariant that a pause halts all state-changing operations.

### Finding Description
`depositKernel()` and `depositKernelFor()` both carry `whenNotPaused`: [1](#0-0) [2](#0-1) 

`bridgeKernelToBSC()` carries only `nonReentrant` and `onlyRole(OPERATOR_ROLE)` ŌĆö no `whenNotPaused`: [3](#0-2) 

The bridge call at line 264 executes unconditionally when the contract is paused: [4](#0-3) 

### Impact Explanation
The pause mechanism is the protocol's primary emergency stop. When the admin pauses (e.g., after detecting an anomaly or after updating the `receiver` address via `setReceiver()`), the operator can still drain the vault's entire KERNEL balance to BSC. No funds are stolen in the classical sense ŌĆö tokens reach the configured `receiver` ŌĆö but the contract fails to deliver its promised invariant: a paused contract halts all state-changing operations including cross-chain transfers. This maps to **Low ŌĆö Contract fails to deliver promised returns, but doesn't lose value**.

### Likelihood Explanation
The admin and operator are distinct roles (separate `DEFAULT_ADMIN_ROLE` and `OPERATOR_ROLE` addresses). No compromise or collusion is required: the admin pauses for a legitimate emergency; the operator, unaware or acting on a pre-queued transaction, calls `bridgeKernelToBSC()`. The path is reachable on unmodified production code with no preconditions beyond normal role assignment.

### Recommendation
Add `whenNotPaused` to `bridgeKernelToBSC()`:

```solidity
function bridgeKernelToBSC(
    uint256 amount,
    uint256 minAmount,
    uint256 nativeFee,
    address refundAddress
)
    external
    payable
    nonReentrant
    whenNotPaused          // <-- add this
    onlyRole(OPERATOR_ROLE)
{
```
