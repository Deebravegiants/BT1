### Title
Undistributed Rewards Permanently Locked When All Stakers Exit During Active Reward Period - (File: `contracts/KERNEL/KernelDepositPool.sol`)

### Summary
`KernelDepositPool` distributes `rewardsToken` to stakers over a fixed period. When `totalKernelStaked` drops to zero mid-period, `rewardPerToken()` stops accumulating, and the remaining reward tokens already transferred into the contract become permanently unclaimable. No admin recovery function exists.

### Finding Description
`notifyRewardAmount` transfers reward tokens into the contract and sets a `rewardRate` for the duration. The accumulation logic in `rewardPerToken()` short-circuits to `rewardPerTokenStored` whenever `totalKernelStaked == 0`: [1](#0-0) 

Once all stakers have exited via `initiateWithdrawal` + `claimWithdrawal`, `totalKernelStaked` is zero. The `rewardRate` is still positive and `finishAt` is still in the future, but no rewards accumulate for anyone. The reward tokens corresponding to the remaining period sit in the contract with no path out:

- `getReward()` pays `rewards[msg.sender]`, which is zero for any address that was not staked.
- `notifyRewardAmount` cannot be called again until `finishAt` passes (`RewardDurationNotFinished`), and even then it only adds *new* rewards on top of the already-stranded balance.
- There is no `recoverERC20`, `sweep`, or emergency-withdrawal function anywhere in the contract. [2](#0-1) 

### Impact Explanation
Reward tokens transferred into `KernelDepositPool` for a distribution period are permanently frozen whenever the staked supply reaches zero before `finishAt`. This matches **Medium – Permanent freezing of unclaimed yield**.

### Likelihood Explanation
Any staker can call `initiateWithdrawal` permissionlessly. A single large staker (or a coordinated set) exiting during an active reward window is a realistic, low-effort path. The withdrawal delay (`withdrawalDelay`, capped at 30 days) only postpones the trigger; it does not prevent it. [3](#0-2) 

### Recommendation
Add an admin-only recovery function that can reclaim undistributed rewards after the reward period ends (i.e., when `block.timestamp > finishAt`). Compute the stranded amount as `rewardRate * (finishAt - lastTimeRewardApplicable())` at the time of the last zero-supply moment, or simply allow the admin to withdraw any `rewardsToken` balance in excess of what is owed to current stakers once `finishAt` has passed.

### Proof of Concept
1. Admin calls `notifyRewardAmount(1_000e18)` with `duration = 7 days`; `rewardRate = 1_000e18 / 7 days`.
2. Alice is the sole staker (`totalKernelStaked = 100e18`).
3. After 1 day Alice calls `initiateWithdrawal(100e18)`.
4. After `withdrawalDelay` Alice calls `claimWithdrawal`; `totalKernelStaked = 0`.
5. For the remaining 6 days `rewardPerToken()` returns the frozen `rewardPerTokenStored`; `≈857e18` reward tokens accumulate in the contract with no claimant.
6. After `finishAt` the admin attempts `notifyRewardAmount` again but the stranded `857e18` tokens remain in the contract forever — no function can move them out. [4](#0-3)

    function initiateWithdrawal(uint256 _amount) external nonReentrant updateReward(msg.sender) {
        if (_amount == 0) revert AmountZero();
        if (balanceOf[msg.sender] < _amount) revert InsufficientStakedBalance();
        if (userWithdrawalIds[msg.sender].length >= maxNumberOfWithdrawalsPerUser) revert WithdrawalLimitReached();

        balanceOf[msg.sender] -= _amount;
        totalKernelStaked -= _amount;

        // Create a withdrawal record
        uint256 withdrawalId = ++withdrawalCounter;
        uint256 unlockTime = block.timestamp + withdrawalDelay;

        withdrawals[withdrawalId] = Withdrawal({
            user: msg.sender, amount: _amount, unlockTime: unlockTime, claimed: false, withdrawalId: withdrawalId
        });
        userWithdrawalIds[msg.sender].push(withdrawalId);

        emit WithdrawalInitiated(msg.sender, _amount, withdrawalId, unlockTime);
    }

    function getReward() external nonReentrant updateReward(msg.sender) {
        uint256 rewardAmount = rewards[msg.sender];

        if (rewardAmount > 0) {
            rewards[msg.sender] = 0;
            rewardsToken.safeTransfer(msg.sender, rewardAmount);
            emit RewardsClaimed(msg.sender, rewardAmount);
        }
    }
