### Title
Unbounded Gas Growth in Public Deposit and Withdrawal Paths via Nested Loop Over Queued EigenLayer Withdrawals — (File: `contracts/NodeDelegator.sol`, `contracts/LRTDepositPool.sol`)

---

### Summary
Every public deposit (`depositETH`, `depositAsset`) and withdrawal initiation (`initiateWithdrawal`) triggers a call chain that ends in `NodeDelegator.getAssetUnstaking()`, which performs a nested loop over all queued EigenLayer withdrawals for each NDC. As the protocol accumulates queued withdrawals through normal operation, the gas cost of these user-facing functions grows proportionally. With enough queued withdrawals across multiple NDCs, these functions can exceed the block gas limit, permanently freezing deposits and withdrawal initiations.

---

### Finding Description

`NodeDelegator.getAssetUnstaking()` fetches all queued withdrawals from EigenLayer's `DelegationManager` and iterates over them in a nested loop:

```solidity
// contracts/NodeDelegator.sol L405-427
function getAssetUnstaking(address asset) external view returns (uint256 amount) {
    (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
        _getDelegationManager().getQueuedWithdrawals(address(this));

    for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
        IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];
        for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
            ...
            amount += ... strategy.sharesToUnderlyingView(sharesToUnstake);
        }
    }
}
```

This function is called inside `LRTDepositPool.getAssetDistributionData()` for **every NDC** in the queue:

```solidity
// contracts/LRTDepositPool.sol L446-456
uint256 ndcsCount = nodeDelegatorQueue.length;
for (uint256 i; i < ndcsCount;) {
    assetLyingInNDCs += IERC20(asset).balanceOf(nodeDelegatorQueue[i]);
    assetStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset);
    assetUnstakingFromEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset); // ← nested loop
    ...
}
```

The full call chain from a public user entry point is:

```
depositETH() / depositAsset()
  → _beforeDeposit()
    → _checkIfDepositAmountExceedesCurrentLimit()
      → getTotalAssetDeposits()
        → getAssetDistributionData()
          → [loop over NDCs] getAssetUnstaking()
            → getQueuedWithdrawals() [EigenLayer external call]
              → [loop over withdrawals]
                → [loop over strategies] sharesToUnderlyingView() [external call per strategy]
```

The same chain is triggered by `initiateWithdrawal()` via `getAvailableAssetAmount()` → `getTotalAssetDeposits()`.

The total work is **O(NDCs × queued\_withdrawals\_per\_NDC × strategies\_per\_withdrawal)**, with each inner iteration making cold external calls (≥2100 gas each). With `maxNodeDelegatorLimit` NDCs and `maxUncompletedWithdrawalCount` queued withdrawals per NDC — both of which grow with normal protocol operation — the gas cost of every deposit and withdrawal initiation grows without a hard ceiling relative to the block gas limit.

---

### Impact Explanation

As the protocol scales (more NDCs added, more EigenLayer withdrawals queued), the gas cost of `depositETH()`, `depositAsset()`, and `initiateWithdrawal()` increases proportionally. Once the cumulative gas cost of the nested loops across all NDCs approaches the block gas limit (~30M gas on Ethereum mainnet), these functions become uncallable. This results in:

- **Temporary or permanent freezing of deposits**: users cannot deposit ETH or LSTs.
- **Temporary or permanent freezing of withdrawal initiations**: users cannot queue new withdrawals.

This matches the **Medium — Unbounded gas consumption** and **Medium — Temporary freezing of funds** impact categories.

---

### Likelihood Explanation

The protocol is designed to operate with multiple NDCs and to regularly queue EigenLayer withdrawals as part of normal restaking operations. The `maxUncompletedWithdrawalCount` is a protocol-controlled parameter with no hard upper bound enforced relative to the block gas limit. As the protocol matures and processes more validator exits or LST unstaking cycles, the number of simultaneously queued withdrawals grows. No attacker action is required — normal protocol operation is sufficient to trigger the condition. Any user attempting a deposit or withdrawal initiation at that point will have their transaction revert out-of-gas.

---

### Recommendation

1. **Cache `getAssetUnstaking()` results off-chain** and pass them as calldata parameters to deposit/withdrawal functions, rather than recomputing them on every call.
2. **Decouple the deposit limit check from the full distribution data computation**: use a lightweight cached total rather than recomputing across all NDCs and all queued withdrawals on every deposit.
3. **Bound `maxUncompletedWithdrawalCount` per NDC** to a value that keeps the worst-case gas cost of the nested loop well below the block gas limit, accounting for the number of NDCs and strategies.
4. **Introduce a gas-bounded iteration** with a maximum iteration count, reverting or skipping if the count is exceeded, rather than iterating over an unbounded external array.

---

### Proof of Concept

1. Protocol has `N = 10` NDCs (at `maxNodeDelegatorLimit`).
2. Each NDC has `M = 50` queued EigenLayer withdrawals (within `maxUncompletedWithdrawalCount`).
3. Each withdrawal involves `S = 3` strategies.
4. A user calls `depositETH(minRSETHAmountExpected, referralId)`.
5. The call chain reaches `getAssetUnstaking()` for each NDC, performing `10 × 50 × 3 = 1500` inner iterations, each involving at least one cold external call to EigenLayer strategy contracts (~2100 gas cold CALL + strategy logic).
6. Estimated gas for the nested loops alone: `1500 × ~5000 gas = 7,500,000 gas`, plus the outer NDC loop overhead and `getQueuedWithdrawals()` data loading.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
