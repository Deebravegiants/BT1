### Title
Unbounded Gas Consumption in Publicly Callable `updateRSETHPrice()` Due to Deeply Nested Loops Over Unbounded Protocol State - (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle.updateRSETHPrice()` is a public function with no access control that executes a deeply nested chain of loops over unbounded protocol state. As the protocol scales (more supported assets, more NodeDelegators, more EigenLayer queued withdrawals), the gas cost grows without bound. Any unprivileged caller can invoke this function at any time, and eventually the accumulated state causes the function to exceed the block gas limit, permanently preventing oracle price updates.

### Finding Description
`updateRSETHPrice()` is declared `public` with only a `whenNotPaused` guard — no role restriction. [1](#0-0) 

It calls `_updateRsETHPrice()`, which calls `_getTotalEthInProtocol()`. That function iterates over every entry in `supportedAssetList`: [2](#0-1) 

For each asset it calls `ILRTDepositPool.getTotalAssetDeposits(asset)`, which calls `getAssetDistributionData()`, which loops over every entry in `nodeDelegatorQueue`: [3](#0-2) 

For each NodeDelegator it calls `getAssetUnstaking(asset)`, which fetches all queued EigenLayer withdrawals and iterates over them with a **nested** inner loop over strategies per withdrawal: [4](#0-3) 

The total gas cost is therefore **O(supportedAssets × nodeDelegators × queuedWithdrawals × strategiesPerWithdrawal)**. None of these dimensions are capped in a way that bounds the gas of a single `updateRSETHPrice()` call. There is no cooldown, no per-block call limit, and no maximum on any of the iterated arrays that would prevent the function from eventually exceeding the block gas limit.

### Impact Explanation
When `updateRSETHPrice()` exceeds the block gas limit it becomes permanently uncallable. The stored `rsETHPrice` in `LRTOracle` then becomes stale. All downstream consumers — `getRsETHAmountToMint` in `LRTDepositPool` and `getExpectedAssetAmount` in `LRTWithdrawalManager` — read this stale price directly: [5](#0-4) [6](#0-5) 

Additionally, the fee-minting path inside `_updateRsETHPrice()` — which mints rsETH to the treasury — also becomes permanently blocked, constituting permanent freezing of unclaimed yield. **Impact: Medium — Unbounded gas consumption / Permanent freezing of unclaimed yield.**

### Likelihood Explanation
The protocol is designed to grow: more LSTs are added as supported assets, more NodeDelegators are added to scale EigenLayer restaking, and operators regularly queue EigenLayer withdrawals via `initiateUnstaking`. This growth is a normal operational expectation, not an attack. Any unprivileged address can call `updateRSETHPrice()` at any time with no cost beyond gas, making the trigger path trivially reachable.

### Recommendation
1. **Cap the number of supported assets and NodeDelegators** at values that keep `updateRSETHPrice()` within a safe gas budget.
2. **Paginate or batch** the asset/NDC iteration so that price updates can be split across multiple transactions.
3. **Cache intermediate values** (e.g., per-NDC asset balances) that are updated lazily rather than recomputed in full on every call.
4. **Add an access-control guard** (e.g., `onlyLRTOperator`) to `updateRSETHPrice()` so that the function cannot be called by arbitrary external actors, reducing the attack surface to trusted callers who can manage call frequency.

### Proof of Concept
1. Protocol is deployed with N supported assets, M NodeDelegators, and K queued EigenLayer withdrawals per NDC (each with S strategies).
2. Any EOA calls `LRTOracle.updateRSETHPrice()`.
3. Execution path: `updateRSETHPrice` → `_updateRsETHPrice` → `_getTotalEthInProtocol` → N iterations → each calls `getTotalAssetDeposits` → `getAssetDistributionData` → M iterations → each calls `getAssetUnstaking` → K × S iterations.
4. Total external calls and storage reads scale as N × M × K × S. At realistic mainnet values (e.g., N=5, M=10, K=50, S=2), the gas cost already approaches several million gas. As K grows with normal protocol operation, the function eventually exceeds the 30M block gas limit.
5. Once uncallable, `rsETHPrice` is frozen at its last value; fee minting stops permanently; the `updateRSETHPriceAsManager()` path (which calls the same `_updateRsETHPrice`) is equally blocked. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
