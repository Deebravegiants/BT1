### Title
No Staleness Check on Chainlink `latestRoundData()` Allows Stale LST Prices to Corrupt rsETH Exchange Rate - (File: contracts/oracles/ChainlinkPriceOracle.sol)

### Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards all freshness fields (`updatedAt`, `answeredInRound`). A stale LST price propagates into `LRTOracle._getTotalEthInProtocol()`, corrupting the stored `rsETHPrice` and causing depositors to receive incorrect rsETH amounts.

### Finding Description
`ChainlinkPriceOracle.getAssetPrice()` retrieves the Chainlink price using `latestRoundData()` but ignores every staleness indicator the API provides:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

The five return values of `latestRoundData()` are `(roundId, answer, startedAt, updatedAt, answeredInRound)`. The contract binds only `price` (the second slot) and throws away `updatedAt` and `answeredInRound`, making it impossible to detect a stale round. By contrast, the sibling contract `ChainlinkOracleForRSETHPoolCollateral` correctly validates both fields:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
``` [2](#0-1) 

`ChainlinkPriceOracle` is the oracle registered for every supported LST (stETH, rETH, swETH, etc.) in `LRTOracle`. When `updateRSETHPrice()` is called, it invokes `_getTotalEthInProtocol()`, which loops over all supported assets and calls `getAssetPrice()` on each one: [3](#0-2) 

The resulting `totalETHInProtocol` value is used to compute and store `rsETHPrice`, which is then consumed by every subsequent deposit to determine how many rsETH tokens to mint. [4](#0-3) 

### Impact Explanation
If a Chainlink LST/ETH feed becomes stale (e.g., during L1 congestion, a sequencer outage, or a Chainlink node incident), `updateRSETHPrice()` — a public, permissionless function — will accept the stale price and write an incorrect `rsETHPrice` to storage. Depositors who call `depositAsset()` or `depositETH()` immediately after will receive rsETH amounts calculated from the corrupted rate. A stale price that is lower than the true price causes over-minting of rsETH, directly diluting existing holders and moving the protocol toward insolvency. A stale price that is higher causes under-minting, depriving depositors of their entitled yield.

**Impact: Low** — Contract fails to deliver promised returns; under specific conditions (stale price within `pricePercentageLimit` tolerance) it can escalate toward protocol insolvency.

### Likelihood Explanation
Chainlink feeds can go stale during network congestion or sequencer downtime. `updateRSETHPrice()` is public and callable by any address. The `pricePercentageLimit` guard provides partial protection only when the stale deviation exceeds the configured threshold; if the threshold is zero or the deviation is small, the stale price is accepted silently. **Likelihood: 2/5** — requires a Chainlink feed to go stale, which is uncommon but has occurred historically.

### Recommendation
Add staleness validation in `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optionally: if (block.timestamp - updatedAt > MAX_DELAY) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

### Proof of Concept
1. A Chainlink LST/ETH feed (e.g., stETH/ETH) stops updating — `updatedAt` is now hours old.
2. Any address calls `LRTOracle.updateRSETHPrice()`.
3. `_getTotalEthInProtocol()` calls `ChainlinkPriceOracle.getAssetPrice(stETH)`.
4. `latestRoundData()` returns the stale answer; no revert occurs because `updatedAt` and `answeredInRound` are never inspected.
5. `totalETHInProtocol` is computed with the stale stETH price, producing an incorrect `newRsETHPrice`.
6. If the stale price is below the true price and within `pricePercentageLimit`, `rsETHPrice` is written with the deflated value.
7. A depositor calling `depositAsset(stETH, amount, ...)` immediately after receives more rsETH than the protocol's actual backing warrants, diluting all existing rsETH holders. [5](#0-4) [6](#0-5) [7](#0-6)

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
