### Title
`LRTConfig.setProtocolFeeBps` Should Settle Accumulated Rewards Before Changing the Fee Rate - (File: `contracts/LRTConfig.sol`, `contracts/LRTOracle.sol`)

---

### Summary

`LRTConfig.setProtocolFeeBps` allows the MANAGER role to update `protocolFeeInBPS` at any time without first triggering `LRTOracle.updateRSETHPrice()`. Because `LRTOracle._updateRsETHPrice()` applies the **current** `protocolFeeInBPS` to the **entire reward delta** accumulated since the last price update, a fee-rate change retroactively reprices rewards that accrued under the old rate. When the fee is raised, rsETH holders (LPs) lose yield they had already earned; when the fee is lowered, the protocol loses fee revenue it had already accrued.

---

### Finding Description

`LRTOracle._updateRsETHPrice()` computes the protocol fee as follows:

```
uint256 rewardAmount = totalETHInProtocol - previousTVL;          // line 245
protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000; // line 246
```

`previousTVL` is reconstructed from `rsETHPrice * rsethSupply`, where `rsETHPrice` is the price stored at the **last** call to `updateRSETHPrice()`. The entire reward delta since that last call is therefore charged at whatever `protocolFeeInBPS` is **at the moment of the next call**, not at the rate that was in effect while the rewards were accumulating.

`setProtocolFeeBps` in `LRTConfig` simply overwrites the stored value with no settlement step:

```solidity
function setProtocolFeeBps(uint256 _protocolFeeInBPS) external onlyRole(LRTConstants.MANAGER) {
    if (_protocolFeeInBPS > 1500) revert ProtocolFeeExceedsLimit();
    protocolFeeInBPS = _protocolFeeInBPS;   // line 198 — no prior updateRSETHPrice() call
    emit UpdateFee(_protocolFeeInBPS);
}
```

`updateRSETHPrice()` is a public function callable by anyone, so the settlement could be enforced, but it is not.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

When `protocolFeeInBPS` is raised before the next `updateRSETHPrice()` call, the higher fee is applied to rewards that accrued entirely under the old, lower rate. rsETH holders (LPs) receive less yield than they were entitled to for that period. The difference is silently redirected to the protocol treasury, constituting theft of unclaimed yield from users.

Example:
- Old fee: 5 % (500 BPS). TVL grows by 10 ETH while fee is 5 %.
- Admin calls `setProtocolFeeBps(1000)` (10 %) without settling first.
- Next `updateRSETHPrice()` charges 1 ETH fee on the 10 ETH reward.
- Correct fee at 5 % would have been 0.5 ETH; LPs lose 0.5 ETH of yield.

The inverse (fee decrease) causes the protocol to under-collect, permanently losing fee revenue it had already accrued — Medium impact (permanent freezing of unclaimed yield for the protocol).

---

### Likelihood Explanation

Fee changes are routine governance operations. `updateRSETHPrice()` is not called atomically with `setProtocolFeeBps`, and there is no on-chain enforcement requiring it. Any gap between the last price update and the fee change — which can span hours or days depending on oracle cadence — creates the window. The MANAGER role is a lower-privilege role than DEFAULT_ADMIN, making fee changes more frequent. Likelihood is **Medium**.

---

### Recommendation

Call `updateRSETHPrice()` (or its internal equivalent) inside `setProtocolFeeBps` before overwriting `protocolFeeInBPS`, so that all rewards accumulated under the old rate are settled at that rate first:

```solidity
function setProtocolFeeBps(uint256 _protocolFeeInBPS) external onlyRole(LRTConstants.MANAGER) {
    if (_protocolFeeInBPS > 1500) revert ProtocolFeeExceedsLimit();
    // Settle accumulated rewards at the current fee rate before changing it
    ILRTOracle(getContract(LRTConstants.LRT_ORACLE)).updateRSETHPrice();
    protocolFeeInBPS = _protocolFeeInBPS;
    emit UpdateFee(_protocolFeeInBPS);
}
```

---

### Proof of Concept

1. `rsETHPrice = 1.0 ETH`, `rsethSupply = 100`, `protocolFeeInBPS = 500` (5 %).
2. EigenLayer rewards accrue; `totalETHInProtocol` rises to 110 ETH. No one calls `updateRSETHPrice()` yet.
3. MANAGER calls `LRTConfig.setProtocolFeeBps(1000)` (10 %).
4. Anyone calls `LRTOracle.updateRSETHPrice()`.
   - `previousTVL = 100 * 1.0 = 100 ETH`
   - `rewardAmount = 110 - 100 = 10 ETH`
   - `protocolFeeInETH = 10 * 1000 / 10_000 = 1 ETH` ← charged at new 10 % rate
5. Correct fee at 5 % = 0.5 ETH. LPs lose 0.5 ETH of yield they earned under the old rate.
