### Title
Treasury ETH Transfer Failure Permanently Blocks `setAaveIntegrationEnabled(false)` — (`contracts/LRTWithdrawalManager.sol`)

---

### Summary

When the LRT manager calls `setAaveIntegrationEnabled(false)`, the function unconditionally calls `_collectInterestToTreasury()` before withdrawing principal. Inside `_collectInterestToTreasury()`, accrued interest is first pulled from Aave and then forwarded to `PROTOCOL_TREASURY` via a bare `.call{value: ...}`. If the treasury is a non-payable contract (or any contract that reverts on ETH receipt), the entire transaction reverts — including the state change `isAaveIntegrationEnabled = enabled`. Because every exit path that touches Aave funds goes through the same helper, the integration can never be disabled and the accrued yield is frozen in Aave indefinitely.

---

### Finding Description

**Exact call chain:**

```
setAaveIntegrationEnabled(false)          [L469, onlyLRTManager]
  └─ aaveBalance = aaveAWETH.balanceOf(address(this))   [L487]
     └─ if aaveBalance > 0:
          _collectInterestToTreasury()                  [L490]
            ├─ interestAmount = aaveBalance - principal  [L952]
            ├─ aaveWETHGateway.withdrawETH(...)          [L954]  ← ETH lands in contract
            ├─ (bool sent,) = payable(treasury).call{value: interestAmount}("")  [L957]
            └─ if (!sent) revert TreasuryTransferFailed()  [L958]  ← REVERTS ENTIRE TX
``` [1](#0-0) [2](#0-1) 

The revert unwinds the Aave withdrawal too, so the interest remains locked as `aWETH` in the contract. The `isAaveIntegrationEnabled` flag is never flipped to `false`.

**No alternative escape path exists:**

- `emergencyWithdrawFromAave()` (callable by `PAUSER_ROLE`) also calls `_collectInterestToTreasury()` first and is equally blocked. [3](#0-2) 

- `collectInterestToTreasury()` (the external operator wrapper) also delegates to the same internal function and reverts identically. [4](#0-3) 

- `configureAaveIntegration()` (reconfiguration path) also calls `_collectInterestToTreasury()` before switching pools, so it is blocked as well. [5](#0-4) 

**Treasury address:**

`PROTOCOL_TREASURY` is a generic contract slot in `LRTConfig.contractMap`, set by `DEFAULT_ADMIN_ROLE` with no validation that the address can receive ETH. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

When `PROTOCOL_TREASURY` is a non-payable contract and Aave has accrued any interest (`aaveAWETH.balanceOf(withdrawalManager) > totalETHDepositedToAave`):

- `setAaveIntegrationEnabled(false)` is permanently bricked for the manager.
- `emergencyWithdrawFromAave()` is permanently bricked for the pauser.
- All accrued yield remains locked as `aWETH` in the `LRTWithdrawalManager` indefinitely.
- The only recovery requires the `DEFAULT_ADMIN_ROLE` to call `LRTConfig.setContract(PROTOCOL_TREASURY, payableAddress)` and then retry — an out-of-band governance action not guaranteed to be available quickly.

**Scoped impact: Medium — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

- Protocol treasuries are frequently multisig contracts or governance contracts. While Gnosis Safe has a `receive()` fallback, other common treasury patterns (e.g., a `TimelockController`, a custom accounting contract, or a contract that conditionally reverts) do not.
- No validation is performed when `PROTOCOL_TREASURY` is set in `LRTConfig`, so a non-payable address is a valid and reachable configuration.
- The condition `aaveBalance > totalETHDepositedToAave` is guaranteed to be true after any non-zero time with ETH deposited to Aave (interest accrues every block).
- Likelihood: **Low-Medium** (depends on treasury implementation, but the code provides zero protection against it).

---

### Recommendation

Decouple the interest collection from the integration-disabling logic. Two options:

1. **Wrap the treasury transfer in a try/catch or skip-on-failure:** If the treasury transfer fails, leave the interest as ETH in the contract (recoverable later) but still allow the integration to be disabled.

2. **Separate concerns entirely:** Remove `_collectInterestToTreasury()` from `setAaveIntegrationEnabled` and `emergencyWithdrawFromAave`. Let the operator collect interest independently; the disable/emergency path should only withdraw principal back to the contract.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

// Fork test (Foundry) — run against mainnet fork
// NonPayableTreasury: a contract with no receive() / fallback()
contract NonPayableTreasury { }

contract AuditPoC is Test {
    LRTWithdrawalManager wm;
    LRTConfig config;

    function setUp() public {
        // ... deploy/configure protocol with Aave integration enabled
        // Set PROTOCOL_TREASURY to NonPayableTreasury
        config.setContract(LRTConstants.PROTOCOL_TREASURY, address(new NonPayableTreasury()));
    }

    function test_setAaveIntegrationEnabled_reverts_nonPayableTreasury() public {
        // Deposit ETH to Aave
        vm.prank(operator);
        wm.depositIdleETHToAave(type(uint256).max);

        // Warp forward to accrue interest
        vm.warp(block.timestamp + 365 days);

        // Confirm interest has accrued
        uint256 aaveBalance = IAToken(wm.aaveAWETH()).balanceOf(address(wm));
        assertGt(aaveBalance, wm.totalETHDepositedToAave());

        // Manager attempts to disable Aave integration — must revert
        vm.prank(manager);
        vm.expectRevert(LRTWithdrawalManager.TreasuryTransferFailed.selector);
        wm.setAaveIntegrationEnabled(false);

        // Integration is still enabled — yield is frozen
        assertTrue(wm.isAaveIntegrationEnabled());
    }
}
```

    function _collectInterestToTreasury() internal returns (uint256 interestAmount) {
        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        uint256 principal = totalETHDepositedToAave;

        // Return 0 if no interest or balance is less than principal (accounting for rounding)
        if (aaveBalance <= principal) return 0;

        interestAmount = aaveBalance - principal;

        aaveWETHGateway.withdrawETH(aavePool, interestAmount, address(this));

        address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        (bool sent,) = payable(treasury).call{ value: interestAmount }("");
        if (!sent) revert TreasuryTransferFailed();

        emit InterestCollectedToTreasury(interestAmount, treasury);
    }
