### Title
Daily Mint Limit Can Be Doubled at Day Boundaries - (`contracts/pools/RSETHPoolV3.sol`)

### Summary
The `limitDailyMint` modifier across all L2 pool contracts uses a day-index comparison (`currentDay > lastMintDay`) to reset the daily mint counter. Because the day index is computed with floor division from a fixed `startTimestamp`, a depositor can mint up to `dailyMintLimit` wrsETH in the final second of one day and another full `dailyMintLimit` in the first second of the next day — effectively doubling the intended per-day cap within a two-second window.

### Finding Description
`getCurrentDay()` returns `(block.timestamp - startTimestamp) / 1 days` (integer floor division). The `limitDailyMint` modifier resets `dailyMintAmount` to zero whenever `currentDay > lastMintDay`:

```solidity
uint256 currentDay = getCurrentDay();
if (currentDay > lastMintDay) {
    lastMintDay = currentDay;
    dailyMintAmount = 0;
}
if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
    revert DailyMintLimitExceeded();
}
dailyMintAmount += rsETHAmount;
```

Because the reset fires the instant `currentDay` increments, a depositor can:

1. At `block.timestamp = startTimestamp + N * 1 days - 1` (last second of day N−1): deposit the full `dailyMintLimit` worth of ETH/tokens.
2. At `block.timestamp = startTimestamp + N * 1 days` (first second of day N): `currentDay` becomes `N > lastMintDay`, the counter resets to zero, and the depositor can immediately deposit another full `dailyMintLimit`.

Two transactions one second apart consume `2 × dailyMintLimit` of wrsETH minting capacity. This pattern is identical to the Teller M-8 root cause: a period-boundary calculation that rounds in the attacker's favour, letting them consume two periods' worth of a rate-limited resource in a negligible time window.

The same modifier is duplicated verbatim in:
- `contracts/pools/RSETHPoolV3.sol`
- `contracts/pools/RSETHPoolV3ExternalBridge.sol`
- `contracts/pools/RSETHPoolV3WithNativeChainBridge.sol`
- `contracts/pools/RSETHPoolV2ExternalBridge.sol`
- `contracts/pools/RSETHPoolV2.sol`

### Impact Explanation
The daily mint limit is the protocol's primary rate-control mechanism on L2 pools. Bypassing it allows a single depositor to mint twice the intended daily cap of wrsETH in a two-second window. This weakens the risk-control guarantee the protocol promises to operators and governance.
