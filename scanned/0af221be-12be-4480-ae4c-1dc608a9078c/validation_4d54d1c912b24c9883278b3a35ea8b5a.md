### Title
`_calculatePayoutAmount` min-cap silently strips accrued yield from withdrawing users — (`contracts/LRTWithdrawalManager.sol`)

---

### Summary

`_calculatePayoutAmount` in `LRTWithdrawalManager` returns `min(expectedAssetAmount, currentReturn)`. Because rsETH is a restaking token that continuously accrues rewards, its ETH-denominated price normally rises between the time a user initiates a withdrawal and the time the operator calls `unlockQueue`. In that common case `currentReturn > expectedAssetAmount`, so the cap fires, the user's full rsETH is burned, but they receive only the initiation-time value. The yield that accrued on their locked rsETH during the waiting period is silently retained by the vault and redistributed to remaining rsETH holders.

---

### Finding Description

**Flow**

1. `initiateWithdrawal` — user transfers `rsETHUnstaked` to the contract; `expectedAssetAmount` is snapshotted at the current oracle price and stored in `WithdrawalRequest`. [1](#0-0) 

2. Time passes (days to weeks, the EigenLayer withdrawal delay). rsETH accrues restaking rewards; `rsETHPrice` rises.

3. Operator calls `unlockQueue` → `_unlockWithdrawalRequests` → `_calculatePayoutAmount`:

```solidity
uint256 currentReturn = (request.rsETHUnstaked * rsETHPrice) / assetPrice;
return (request.expectedAssetAmount < currentReturn)
    ? request.expectedAssetAmount   // ← cap fires; user loses appreciation
    : currentReturn;
``` [2](#0-1) 

4. `payoutAmount = expectedAssetAmount` (the smaller, stale value). The full `rsETHUnstaked` is burned, but only `expectedAssetAmount` assets are pulled from the vault and credited to the user. [3](#0-2) 

5. `completeWithdrawal` transfers `request.expectedAssetAmount` to the user — the stale, lower amount. [4](#0-3) 

The delta `(currentReturn − expectedAssetAmount)` is never distributed; it remains in the vault and inflates the share value for all remaining rsETH holders at the expense of the withdrawing user.

---

### Impact Explanation

Every user who initiates a standard (non-instant) withdrawal loses the yield that accrued on their rsETH during the waiting period. Because rsETH is a restaking token whose price is designed to increase monotonically with EigenLayer rewards, the cap fires on virtually every normal withdrawal. The lost amount scales with the waiting period length and the reward rate. This constitutes **theft of unclaimed yield** (High).

---

### Likelihood Explanation

The condition `currentReturn > expectedAssetAmount` is the normal, expected state for any withdrawal that completes after rewards have accrued. The EigenLayer withdrawal delay is measured in days; even a modest annualised restaking yield of 4–6 % produces a measurable gap over that window. No special attacker action is required — the loss is automatic for every withdrawing user.

---

### Recommendation

At `unlockQueue` time, recalculate the payout using the **current** oracle price rather than capping at the stale initiation-time value. The slashing-protection intent (preventing over-payment when rsETH price drops) is preserved by keeping the `min` only for the downside case, while the upside case should pay `currentReturn`:

```solidity
// current behaviour — strips yield on the upside
return (request.expectedAssetAmount < currentReturn)
    ? request.expectedAssetAmount
    : currentReturn;

// corrected — only cap on the downside (slashing)
return currentReturn;   // always pay current fair value; vault solvency is
                        // already enforced by the availableAssetAmount check
```

Alternatively, if the protocol intentionally locks in the initiation-time price, the rsETH should **not** be burned until `completeWithdrawal` is called, so the user retains the economic exposure to their rsETH during the waiting period.

---

### Proof of Concept

**Setup**: rsETH price = 1.05 ETH, stETH price = 1.02 ETH at initiation.

1. Alice calls `initiateWithdrawal(stETH, 1e18 rsETH)`.
   - `expectedAssetAmount = 1e18 * 1.05e18 / 1.02e18 ≈ 1.0294 stETH`
   - `assetsCommitted[stETH] += 1.0294 stETH`
   - Alice's 1 rsETH is held by the contract.

2. Two weeks pass. rsETH price rises to 1.08 ETH (restaking rewards accrued).

3. Operator calls `unlockQueue(stETH, ...)`.
   - `currentReturn = 1e18 * 1.08e18 / 1.02e18 ≈ 1.0588 stETH`
   - `payoutAmount = min(1.0294, 1.0588) = 1.0294 stETH` ← cap fires
   - Alice's 1 rsETH is burned; `assetAmountToUnlock = 1.0294 stETH`.

4. Alice calls `completeWithdrawal(stETH)` and receives **1.0294 stETH**.

5. Alice's 1 rsETH was worth **1.0588 stETH** at unlock time. She lost **0.0294 stETH** (~2.8 %) of yield that accrued during the waiting period. That value remains in the vault. [5](#0-4) [6](#0-5)
