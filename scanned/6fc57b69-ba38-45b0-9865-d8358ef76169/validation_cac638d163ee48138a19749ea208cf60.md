### Title
Unrestricted Third-Party Claim Execution Forces Fee Deduction on Victims - (File: `contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

---

### Summary

`MerkleDistributor.claim()` accepts an arbitrary `account` parameter but never verifies that `msg.sender == account`. Because merkle proofs are public off-chain data, any third party can trigger a claim for any user. When a non-zero fee is configured (up to 10%), the victim's claimable yield is permanently reduced by the fee amount, which is irrecoverably sent to `protocolTreasury`.

---

### Finding Description

`MerkleDistributor.claim()` performs merkle proof verification and fee deduction but contains no caller-identity check:

```solidity
function claim(
    uint256 index,
    address account,       // ← caller-supplied, never validated against msg.sender
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
    ...
    uint256 fee = (claimableAmount * feeInBPS) / 10_000;
    uint256 amountToSend = claimableAmount - fee;

    IERC20(token).safeTransfer(account, amountToSend);
    IERC20(token).safeTransfer(protocolTreasury, fee);   // fee permanently lost to victim
}
``` [1](#0-0) 

The sibling contract `KernelMerkleDistributor._processClaim()` explicitly guards against this with:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [2](#0-1) 

`MerkleDistributor` is missing this guard entirely. The fee ceiling is `MAX_FEE_IN_BPS = 1000` (10%), and the fee is set by the owner at deploy/upgrade time. [3](#0-2) 

The unintended execution path is: **attacker → `claim(index, victim, amount, proof)` → victim's yield is claimed with fee deducted, without victim's consent.**

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Any user's pending yield can be forcibly claimed by a third party. The fee (up to 10% of the claimable increment) is permanently transferred to `protocolTreasury` and is unrecoverable by the victim. The victim receives only `claimableAmount - fee` instead of the full `claimableAmount`. This is a direct, irreversible reduction of the user's entitled yield, triggered entirely by an unprivileged external caller.

---

### Likelihood Explanation

**Medium.** Merkle proofs for distributor contracts are standard public off-chain data (published by the protocol for users to self-claim). Any attacker can read the proof tree, construct a valid `claim()` call for any victim, and submit it. No privileged access, no leaked keys, and no front-running dependency is required. The only precondition is that the victim has an unclaimed balance and the fee is non-zero.

---

### Recommendation

Add the same caller-identity guard present in `KernelMerkleDistributor` to `MerkleDistributor.claim()`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Insert this check immediately after the `isClaimed` check, before any state mutation or token transfer. [4](#0-3) 

---

### Proof of Concept

1. Protocol deploys `MerkleDistributor` with `feeInBPS = 1000` (10%).
2. Alice is entitled to 1,000 tokens; her merkle proof is published in the protocol's public distribution data.
3. Attacker Bob calls:
   ```solidity
   merkleDistributor.claim(currentIndex, alice, 1000e18, aliceProof);
   ```
4. The function passes all checks (valid proof, not yet claimed, non-zero amount).
5. `fee = 1000e18 * 1000 / 10_000 = 100e18` tokens are sent to `protocolTreasury`.
6. Alice receives only `900e18` tokens — 100 tokens of her yield are permanently lost without her consent.
7. Alice's `userClaims[alice].lastClaimedIndex` is updated, so she cannot re-claim the same index. [5](#0-4)

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
