### Title
Uncapped per-bin additional fees allow pool admin to charge traders beyond the global fee cap - (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary
`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6` and `addFeeSellE6` to the pool with no cap validation, while the global admin spread fee is explicitly bounded by `maxAdminSpreadFeeE6`. A pool admin can set per-bin additional fees to the `uint16` maximum (65535 in E6 units = 6.5535%), charging traders significantly more than the global fee cap implies and extracting funds beyond the protocol's intended admin-fee ceiling.

### Finding Description

The factory enforces explicit caps on the global admin spread fee inside `setPoolAdminFees`:

```solidity
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
