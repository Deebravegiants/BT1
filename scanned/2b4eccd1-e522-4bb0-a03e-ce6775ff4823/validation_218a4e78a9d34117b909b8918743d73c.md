### Title
Stale `rsETHPrice` Used in `instantWithdrawal()` Allows Users to Drain More Assets Than Current Rate Warrants - (File: contracts/LRTWithdrawalManager.sol)

### Summary
`LRTWithdrawalManager.instantWithdrawal()` computes the asset payout using `lrtOracle.rsETHPrice()`, a stored/cached state variable that is only updated when `updateRSETHPrice()` is explicitly called. If the true current rsETH price has decreased since the last oracle update (e.g., due to an EigenLayer slashing event within the `pricePercentageLimit` threshold), an unprivileged user can call `instantWithdrawal()` with the stale, inflated price and receive more underlying assets than the current rate warrants, at the expense of other protocol participants.

### Finding Description

`LRTOracle.rsETHPrice` is a stored state variable updated only when `updateRSETHPrice()` is called: [1](#0-0) [2](#0-1) 

`LRTWithdrawalManager.getExpectedAssetAmount()` reads this cached value directly: [3](#0-2) 

`instantWithdrawal()` calls `getExpectedAssetAmount()` and immediately transfers the result to the user with no freshness check and no minimum-of-current-vs-stored guard: [4](#0-3) 

Contrast this with the normal `initiateWithdrawal()` + `unlockQueue()` path, which applies `_calculatePayoutAmount()` at unlock time and takes the **minimum** of the stored expected amount and the current-price-derived amount: [5](#0-4) 

`instantWithdrawal()` has no equivalent protection. The payout is computed once, using the stale `rsETHPrice`, and transferred directly.

The `_updateRsETHPrice()` logic pauses the protocol only when the price drop exceeds `pricePercentageLimit`: [6](#0-5) 

For drops **within** the configured threshold (e.g., ≤1%), the protocol remains unpaused and `instantWithdrawal()` remains callable with the stale, higher price.

### Impact Explanation

An attacker who holds rsETH can call `instantWithdrawal()` immediately after a loss event (before `updateRSETHPrice()` is called) and receive more underlying assets than the current rsETH/asset exchange rate entitles them to. The excess assets come from the `LRTUnstakingVault`, effectively stealing value from other protocol participants. This is a direct theft of user funds.

**Impact level:** Critical — direct theft of funds at rest in the unstaking vault.

### Likelihood Explanation

- EigenLayer slashing is a known, documented risk for the protocol's node delegators.
- A small slashing event (within `pricePercentageLimit`) does not trigger an automatic pause.
- `updateRSETHPrice()` is not called atomically with every withdrawal; it is a separate public transaction.
- An attacker can monitor on-chain slashing events and front-run the price update by calling `instantWithdrawal()` in the same block or the next block before any keeper calls `updateRSETHPrice()`.
- `isInstantWithdrawalEnabled` must be set for the target asset, but this is a live operational feature with a dedicated setter.

**Likelihood level:** Medium — realistic trigger condition (slashing within threshold) combined with a straightforward front-running opportunity.

### Recommendation

Before computing `assetAmountUnlocked` in `instantWithdrawal()`, force a price update:

```solidity
function instantWithdrawal(...) external ... {
    // Force price to be current before computing payout
    ILRTOracle(lrtConfig.getContract(LRTConstants.LRT_ORACLE)).updateRSETHPrice();
    uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
    ...
}
```

Alternatively, compute the rsETH price on-the-fly inside `getExpectedAssetAmount()` rather than reading the cached `rsETHPrice` storage slot, mirroring how `_getTotalEthInProtocol()` always reads live asset prices via `getAssetPrice()`.

### Proof of Concept

1. Protocol state: `rsETHPrice = 1.05e18` (last updated 12 hours ago). True current price after a small slashing event: `1.04e18`. Drop is 0.95%, within `pricePercentageLimit = 1%`, so no pause is triggered.
2. Attacker holds `100e18` rsETH. Calls `instantWithdrawal(ETH_TOKEN, 100e18, "")`.
3. `getExpectedAssetAmount` computes: `100e18 * 1.05e18 / 1e18 = 105 ETH`.
4. True fair payout at current price: `100e18 * 1.04e18 / 1e18 = 104 ETH`.
5. Attacker receives `105 ETH` (minus fee), extracting `~1 ETH` more than entitled.
6. At scale (e.g., 10,000 rsETH), the attacker extracts `~100 ETH` of excess value from the vault before `updateRSETHPrice()` is called. [7](#0-6) [8](#0-7)

        address asset,
        uint256 rsETHUnstaked,
        string calldata referralId
    )
        external
        nonReentrant
        whenNotPaused
        onlySupportedAsset(asset)
        onlySupportedStrategy(asset)
        onlyInstantWithdrawalAllowed(asset)
    {
        if (rsETHUnstaked == 0 || rsETHUnstaked < minRsEthAmountToWithdraw[asset]) {
            revert InvalidAmountToWithdraw();
        }
        if (IERC20(lrtConfig.rsETH()).balanceOf(msg.sender) < rsETHUnstaked) revert NotEnoughRsETH();
        uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
        IRSETH(lrtConfig.rsETH()).burnFrom(address(msg.sender), rsETHUnstaked);
        ILRTUnstakingVault unstakingVault = ILRTUnstakingVault(lrtConfig.getContract(LRTConstants.LRT_UNSTAKING_VAULT));
        if (assetAmountUnlocked > unstakingVault.getAssetsAvailableForInstantWithdrawal(asset)) {
            revert CantInstantWithdrawMoreThanAvailable();
        }

        unstakingVault.redeem(asset, assetAmountUnlocked);

        uint256 fee = (assetAmountUnlocked * instantWithdrawalFee) / 10_000;
        uint256 userAmount = assetAmountUnlocked - fee;

        address feeRecipient = instantWithdrawalFeeRecipient;
        if (feeRecipient == address(0)) {
            // Backwards-compatible default: send fees to the protocol treasury
            feeRecipient = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        }
        if (fee > 0) {
            _transferAsset(asset, feeRecipient, fee);
            emit InstantWithdrawalFeeCollected(msg.sender, asset, fee);
        }

        _transferAsset(asset, msg.sender, userAmount);
        emit ReferralIdEmitted(referralId);
        emit AssetWithdrawalFinalized(msg.sender, asset, rsETHUnstaked, userAmount);
    }

    function getExpectedAssetAmount(
        address asset,
        uint256 amount
    )
        public
        view
        override
        returns (uint256 underlyingToReceive)
    {
        // setup oracle contract
        ILRTOracle lrtOracle = ILRTOracle(lrtConfig.getContract(LRTConstants.LRT_ORACLE));

        // calculate underlying asset amount to receive based on rsETH amount and asset exchange rate
        underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
    }
