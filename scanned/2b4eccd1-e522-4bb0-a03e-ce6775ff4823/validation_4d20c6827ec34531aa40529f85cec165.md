### Title
Unbounded Gas Consumption in `updateRSETHPrice()` Due to Nested Loops Over Supported Assets and Node Delegators - (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle.updateRSETHPrice()` is a public function that internally executes a nested loop structure: an outer loop over all supported assets and an inner loop over all node delegators (NDCs), issuing multiple external calls per pair. As the protocol scales, this function can exceed the block gas limit and become permanently uncallable, freezing rsETH price updates and halting fee minting.

---

### Finding Description

`updateRSETHPrice()` (line 87) is public and calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`. Inside `_getTotalEthInProtocol()` (lines 331–349), there is an outer loop over every entry in `supportedAssets`:

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);                          // external call
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr)
        .getTotalAssetDeposits(asset);                               // triggers inner loop
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    ...
}
```

`getTotalAssetDeposits()` calls `getAssetDistributionData()` (lines 446–456), which itself loops over every NDC in `nodeDelegatorQueue` and makes **three external calls per NDC per asset**:

```solidity
for (uint256 i; i < ndcsCount;) {
    assetLyingInNDCs += IERC20(asset).balanceOf(nodeDelegatorQueue[i]);
    assetStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset);
    assetUnstakingFromEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset);
    ...
}
```

For ETH specifically, `getETHDistributionData()` (lines 484–493) also loops over all NDCs with two external calls each (EigenPod shares + unstaking).

The total external call count scales as **O(assets × NDCs)**, with each call costing at minimum 2,100 gas (cold CALL opcode). With `N` supported assets and `M` NDCs, the function issues `3·N·M + 2·M` external calls. At `N=10, M=10` this is already 320 external calls, and `maxNodeDelegatorLimit` is admin-adjustable upward with no hard ceiling enforced in the code.

---

### Impact Explanation

If `updateRSETHPrice()` exceeds the block gas limit:

1. The rsETH price stored in `rsETHPrice` becomes permanently stale.
2. Fee minting via `_checkAndUpdateDailyFeeMintLimit` / `IRSETH.mint` inside `_updateRsETHPrice()` stops — **permanent freezing of unclaimed yield** (protocol fees).
3. Deposits via `getRsETHAmountToMint()` and withdrawals via `getExpectedAssetAmount()` both read `lrtOracle.rsETHPrice()`, so they operate on a stale rate — **contract fails to deliver promised returns**.
4. The price-deviation circuit breaker (pause logic at lines 270–282) can no longer trigger, removing a key safety mechanism.

Impact: **Medium — Unbounded gas consumption / Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

The protocol is designed to support multiple LSTs (ETH, stETH, ETHx, sfrxETH, etc.) and multiple NDCs. Both arrays grow through normal, legitimate protocol operation. `maxNodeDelegatorLimit` starts at 10 but is adjustable upward by admin with no on-chain cap. As the protocol adds more assets and NDCs over time, the gas cost of `updateRSETHPrice()` grows quadratically. No attacker action is required — organic protocol growth is sufficient to trigger the condition.

---

### Recommendation

1. Cache intermediate per-NDC values off-chain or in storage and update them incrementally rather than recomputing the full sum on every price update.
2. Introduce a hard cap on `maxNodeDelegatorLimit` × `supportedAssets.length` product, or split `_getTotalEthInProtocol()` into a paginated/batched function callable across multiple transactions.
3. Consider storing a running `totalETHInProtocol` accumulator that is updated on deposit/withdrawal events rather than recomputed from scratch on every oracle update.

---

### Proof of Concept

Call path for gas exhaustion:

1. Any address calls `LRTOracle.updateRSETHPrice()` (public, no access control).
2. → `_updateRsETHPrice()` → `_getTotalEthInProtocol()`.
3. Outer loop iterates over `supportedAssets` (length `N`).
4. For each asset, `ILRTDepositPool.getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)` iterates over `nodeDelegatorQueue` (length `M`), issuing 3 external calls per NDC.
5. Total external calls: `3·N·M`. At `N=10, M=20` → 600 external calls. At 2,100 gas each (cold) → 1,260,000 gas for calls alone, before computation, storage reads, or EigenLayer strategy logic inside `getAssetBalance()`.
6. As `M` grows beyond the initial limit of 10, the function crosses the 30M block gas limit and becomes permanently uncallable. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
