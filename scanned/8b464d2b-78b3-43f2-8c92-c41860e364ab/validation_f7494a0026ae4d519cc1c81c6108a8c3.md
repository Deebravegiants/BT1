### Title
`bridgeKernelToBSC()` Missing `whenNotPaused` Modifier Allows Cross-Chain Transfers During Emergency Pause — (`contracts/KERNEL/KernelVaultETH.sol`)

### Summary
`KernelVaultETH` inherits `PausableUpgradeable` and applies `whenNotPaused` to all user-facing deposit functions, but omits it from `bridgeKernelToBSC()`. An operator can therefore execute a cross-chain LayerZero transfer while the contract is paused, violating the invariant that a pause halts all state-changing operations.

### Finding Description
`depositKernel()` and `depositKernelFor()` both carry `whenNotPaused`: [1](#0-0) [2](#0-1) 

`bridgeKernelToBSC()` carries only `nonReentrant` and `onlyRole(OPERATOR_ROLE)`: [3](#0-2) 

The function proceeds to call `kernelOftAdapter.send()` and update `lastBridgedDepositId` with no pause check: [4](#0-3) 

### Impact Explanation
The pause mechanism fails to deliver its stated guarantee. When the admin pauses the contract — typically as an emergency stop — the operator can still bridge the entire KERNEL balance held in the vault to BSC. If the admin had just called `setReceiver(newAddr)` before pausing (e.g., to update the BSC receiver while verifying it is safe), the operator can bridge to that new, potentially unverified address before the admin has a chance to intervene. No user funds are directly stolen (tokens go to the configured `receiver`), but the contract fails to deliver the promised "all operations halted" guarantee of a pause. **Scope: Low — contract fails to deliver promised returns, but doesn't lose value.** [5](#0-4) 

### Likelihood Explanation
The operator is a trusted role, so this is not an external attacker path. However, the admin and operator are distinct roles and may act independently. An admin pausing for an emergency does not prevent an operator who is unaware of the pause (or acting on a pre-scheduled basis) from calling `bridgeKernelToBSC()`. The inconsistency is a concrete code defect, not a theoretical one.

### Recommendation
Add `whenNotPaused` to `bridgeKernelToBSC()`:

```solidity
function bridgeKernelToBSC(
    uint256 amount,
    uint256 minAmount,
    uint256 nativeFee,
    address refundAddress
)
    external
    payable
    nonReentrant
    whenNotPaused          // <-- add this
    onlyRole(OPERATOR_ROLE)
{
```
