### Title
Anyone Can Force a Claim for Any User, Causing Fee Loss on Unclaimed Yield — (`contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

---

### Summary

`MerkleDistributor.sol`'s `claim()` function accepts an arbitrary `account` parameter but performs no check that `msg.sender == account`. Any unprivileged caller who possesses (or can reconstruct) a valid merkle proof for a victim can trigger the victim's claim at any time, forcing the fee deduction at the current `feeInBPS` rate even if the victim was deliberately waiting for the fee to be lowered.

---

### Finding Description

`MerkleDistributor.claim()` is a public, permissionless function:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
    ...
    IERC20(token).safeTransfer(account, amountToSend);   // fee already deducted
    IERC20(token).safeTransfer(protocolTreasury, fee);
    ...
}
``` [1](#0-0) 

There is no guard of the form `if (account != msg.sender) revert Unauthorized()`. The merkle tree data (indices, accounts, cumulative amounts, and proofs) is published off-chain so that users can construct their own claims; this means any third party can reconstruct a valid proof for any leaf and call `claim()` on behalf of any victim.

By contrast, `KernelMerkleDistributor._processClaim()` explicitly enforces this check:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [2](#0-1) 

`MerkleDistributor.sol` is missing this protection entirely.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

`feeInBPS` is owner-controlled and can be set up to `MAX_FEE_IN_BPS = 1000` (10%). [3](#0-2) 

A user who observes that the current fee is high (e.g., 10%) may deliberately defer their claim, waiting for the owner to reduce `feeInBPS`. An attacker can front-run or simply call `claim()` for that user while the fee is still high, forcing the fee deduction before the user can act. The fee is permanently transferred to `protocolTreasury`; the user cannot recover it. On a 10 000 KERNEL claim at 10% fee, the user loses 1 000 KERNEL they would have retained had they been allowed to time their own claim.

---

### Likelihood Explanation

**Medium.**

- Merkle proofs are public by design (users need them to claim); any observer can reconstruct a valid proof for any leaf.
- The attacker needs no special role, no capital, and no privileged access — only the ability to call a public function.
- The attack is only profitable (in terms of harm to the victim) when `feeInBPS > 0` and the victim is deferring their claim in anticipation of a fee reduction, which is a realistic user behaviour given the owner's ability to change the fee at any time.

---

### Recommendation

Add a caller check identical to the one already present in `KernelMerkleDistributor`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

This ensures only the rightful recipient can trigger their own claim and choose the timing (and thus the fee rate) that applies to them.

---

### Proof of Concept

1. Owner sets `feeInBPS = 1000` (10%).
2. Alice holds a valid merkle leaf `(index=1, account=Alice, cumulativeAmount=10_000e18)` and decides to wait for the fee to drop.
3. Bob (unprivileged) reconstructs Alice's proof from the published merkle tree data.
4. Bob calls `MerkleDistributor.claim(1, Alice, 10_000e18, aliceProof)`.
5. The contract deducts `fee = 1_000e18` and sends it to `protocolTreasury`; Alice receives only `9_000e18`.
6. Alice can no longer claim the 1 000 tokens she would have retained by waiting — they are permanently gone to the treasury. [4](#0-3)

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
