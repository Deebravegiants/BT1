### Title
Reward Tokens Permanently Locked When `totalKernelStaked` Drops to Zero During an Active Reward Period - (File: contracts/KERNEL/KernelDepositPool.sol)

### Summary
`KernelDepositPool` distributes reward tokens to KERNEL stakers using a Synthetix-style per-token accumulator. When `totalKernelStaked` reaches zero mid-period, `rewardPerToken()` freezes the accumulator and stops advancing it, while `rewardRate` continues to imply token emission. The reward tokens that should have been distributed during the zero-staked window are never credited to any account and become permanently irrecoverable from the contract. The contract's own NatSpec acknowledges this but relies entirely on an off-chain operational assumption that is not enforced on-chain.

### Finding Description

`rewardPerToken()` contains the following guard:

```solidity
function rewardPerToken() public view returns (uint256) {
    if (totalKernelStaked == 0) {
        return rewardPerTokenStored;   // accumulator frozen
    }
    return rewardPerTokenStored
        + (rewardRate * (lastTimeRewardApplicable() - updatedAt) * DECIMAL_PRECISION)
        / totalKernelStaked;
}
``` [1](#0-0) 

When the last staker calls `initiateWithdrawal`, the `updateReward` modifier snapshots `rewardPerTokenStored` and `updatedAt` at that moment, then the function body sets `totalKernelStaked` to zero. [2](#0-1) [3](#0-2) 

From that point forward, every call to `rewardPerToken()` returns the frozen `rewardPerTokenStored`. The elapsed time `(lastTimeRewardApplicable() - updatedAt)` is never consumed; the reward tokens that `rewardRate` implies for that window are never added to any user's `earned()` balance. When a new staker eventually arrives, `updateReward` sets `updatedAt = lastTimeRewardApplicable()` (current time), silently discarding the entire zero-staked interval. The reward tokens for that interval remain in the contract with no path to recovery.

The contract's own NatSpec explicitly acknowledges this:

> *"If `totalKernelStaked` ever hits zero during a reward distribution window, any remaining rewards for that period will stay locked in the contract."* [4](#0-3) 

The stated mitigation — "ensuring there are always some tokens staked" — is a purely off-chain operational assumption. No on-chain guard prevents `totalKernelStaked` from reaching zero during an active period.

`notifyRewardAmount` does block starting a new period when `totalKernelStaked == 0`, but this does not protect against the mid-period drain: [5](#0-4) 

### Impact Explanation

Any reward tokens emitted by `rewardRate` during the zero-staked window are permanently locked inside `KernelDepositPool`. They cannot be claimed by any user (no staker exists to accumulate them) and there is no admin rescue function. This constitutes **permanent freezing of unclaimed yield**.

### Likelihood Explanation

Any staker can call `initiateWithdrawal` followed by `claimWithdrawal` (after `withdrawalDelay`). If all stakers exit during an active reward period — a realistic scenario during market stress, loss of confidence, or a coordinated migration — `totalKernelStaked` reaches zero and the remaining period's rewards are locked. No privileged action is required; ordinary user withdrawals are the sole trigger.

### Recommendation

When `totalKernelStaked` drops to zero, the contract should stop advancing the reward clock rather than silently discarding elapsed time. Two complementary fixes:

1. **In `rewardPerToken()`**: when `totalKernelStaked == 0`, also update `updatedAt` to `lastTimeRewardApplicable()` so the zero-staked interval is not silently consumed when stakers return. This prevents the accumulator gap but does not recover already-locked tokens.

2. **Add a recovery function**: allow the admin to reclaim undistributed reward tokens (i.e., `rewardRate * remainingTime` when `totalKernelStaked == 0`), analogous to the Velodrome fix of returning `claimable` to the Minter when a gauge is killed.

### Proof of Concept

1. Admin calls `notifyRewardAmount(1_000_000e18)` with `duration = 7 days`. `rewardRate = 1_000_000e18 / 604800 ≈ 1653 tokens/sec`. `finishAt = now + 7 days`.
2. Alice is the only staker with `balanceOf[Alice] = 1000e18`, `totalKernelStaked = 1000e18`.
3. After 3 days, Alice calls `initiateWithdrawal(1000e18)`. `updateReward` snapshots `rewardPerTokenStored` and `updatedAt = now`. `totalKernelStaked` becomes 0.
4. The remaining 4 days of rewards (`≈ 4 * 86400 * 1653 ≈ 571,392 tokens`) are now in limbo: `rewardPerToken()` returns the frozen `rewardPerTokenStored` for every subsequent call.
5. After `withdrawalDelay`, Alice calls `claimWithdrawal` and recovers her KERNEL principal. She calls `getReward()` and receives her 3-day share. The 4-day remainder stays locked.
6. Bob stakes 1 wei on day 5. `updateReward` sets `updatedAt = now` (day 5), discarding days 3–5. Bob earns only from day 5 onward. The days 3–5 reward tokens (~285,696 tokens) are permanently unclaimable. [1](#0-0) [5](#0-4)

    function notifyRewardAmount(uint256 _amount) external onlyRole(DEFAULT_ADMIN_ROLE) updateReward(address(0)) {
        if (_amount == 0) revert AmountZero();

        // Prevent starting a reward period when no tokens are staked to avoid unallocated rewards
        if (totalKernelStaked == 0) revert NoStakedTokens();

        // Transfer reward tokens into the contract
        uint256 balanceBefore = rewardsToken.balanceOf(address(this));
        rewardsToken.safeTransferFrom(msg.sender, address(this), _amount);
        uint256 balanceAfter = rewardsToken.balanceOf(address(this));
        // Calculate the actual amount of tokens received in case of a transfer fee (tax)
        uint256 receivedAmount = balanceAfter - balanceBefore;

        if (block.timestamp >= finishAt) {
            rewardRate = receivedAmount / duration;
        } else {
            uint256 remaining = (finishAt - block.timestamp) * rewardRate;
            rewardRate = (receivedAmount + remaining) / duration;
        }

        if (rewardRate == 0) revert RewardRateZero();

        finishAt = block.timestamp + duration;
        updatedAt = block.timestamp;

        emit NotifyRewardAmount(receivedAmount, finishAt);
    }
