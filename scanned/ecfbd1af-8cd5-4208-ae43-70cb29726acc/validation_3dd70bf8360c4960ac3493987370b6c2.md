### Title
Missing Staleness Check on Chainlink Price Feed Allows Stale Prices to Drive Deposits and Withdrawals - (File: contracts/oracles/ChainlinkPriceOracle.sol)

### Summary

`ChainlinkPriceOracle.getAssetPrice` calls `latestRoundData()` but silently discards the `updatedAt` timestamp, meaning a Chainlink feed that has stopped updating is treated as live. Every deposit and withdrawal in the protocol is priced through this oracle, so a stale feed directly controls how much rsETH is minted or how many assets are returned.

### Finding Description

`ChainlinkPriceOracle.getAssetPrice` at line 52 destructures the return of `latestRoundData()` but captures only `price`, ignoring `updatedAt`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No maximum-age check (e.g., `require(block.timestamp - updatedAt <= MAX_STALENESS)`) is present anywhere in the function. The `updatedAt` field — the on-chain equivalent of the LPDA `endTime` — is returned by Chainlink but never read.

This price is consumed by `LRTOracle.getAssetPrice`, which is in turn called by:

- `LRTDepositPool.getRsETHAmountToMint` → `depositAsset` / `depositETH` (rsETH minting)
- `LRTWithdrawalManager.getExpectedAssetAmount` → `initiateWithdrawal` (withdrawal sizing)
- `LRTConverter.transferAssetFromDepositPool` / `transferAssetToDepositPool` (TVL accounting)

The call chain for an unprivileged depositor is:

```
depositAsset()
  → _beforeDeposit()
    → getRsETHAmountToMint()
      → lrtOracle.getAssetPrice(asset)          // LRTOracle.sol
        → IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)
          → priceFeed.latestRoundData()          // ChainlinkPriceOracle.sol:52
            // updatedAt is discarded — no staleness check
```

### Impact Explanation

If a supported LST depegs or is slashed and its Chainlink feed lags (circuit-breaker pause, oracle node failure, network congestion), the stale price remains higher than the true market price. An attacker can:

1. Deposit the now-devalued LST at the inflated stale price.
2. Receive more rsETH than the deposited LST is worth.
3. Redeem or sell the rsETH, extracting value from existing rsETH holders.

This constitutes direct dilution of all rsETH holders — equivalent to theft of their proportional share of the protocol's TVL. In the reverse direction (stale price lower than actual), withdrawers receive fewer assets than owed, causing the protocol to fail to deliver promised returns.

**Impact: High** — Theft of value from existing rsETH holders proportional to the price deviation and deposit size.

### Likelihood Explanation

Chainlink feeds can become stale during:
- Network congestion preventing oracle node transactions
- Chainlink circuit-breaker events (deviation threshold not met for extended periods)
- Oracle node failures or key rotation delays

LST assets (stETH, ETHx) have historically experienced price deviations during market stress. The window between a real price move and a Chainlink update can be minutes to hours, which is sufficient for a monitoring attacker to act.

**Likelihood: Medium** — Requires a Chainlink feed to go stale, which is an uncommon but documented real-world event.

### Recommendation

Add a configurable staleness threshold and validate `updatedAt` in `getAssetPrice`:

```solidity
uint256 public constant MAX_STALENESS = 1 hours;

function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,, uint256 updatedAt,) = priceFeed.latestRoundData();
    require(block.timestamp - updatedAt <= MAX_STALENESS, "Stale price");
    require(price > 0, "Invalid price");
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

### Proof of Concept

1. Chainlink stETH/ETH feed stops updating (e.g., circuit breaker). Last reported price: 1.0 ETH per stETH. True market price after slashing event: 0.85 ETH per stETH.
2. Attacker acquires 1000 stETH at 0.85 ETH each (850 ETH cost).
3. Attacker calls `LRTDepositPool.depositAsset(stETH, 1000e18, 0, "")`.
4. `getRsETHAmountToMint` calls `lrtOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice` → returns stale 1.0e18.
5. Attacker receives rsETH priced as if 1000 ETH was deposited, not 850 ETH.
6. Attacker redeems or sells rsETH, extracting ~150 ETH of value from existing holders.
