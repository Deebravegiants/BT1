### Title
Missing `setMerkleRoot()` Admin Function Permanently Locks All Claims on Wrong-Root Deployment — (`contracts/KERNEL/KernelTop100MerkleDistributor.sol`)

### Summary
`KernelTop100MerkleDistributor` sets `merkleRoot` once at initialization with no owner function to update it. If the wrong root is committed at deploy time, every `claim()` and `claimAndStake()` call reverts with `InvalidMerkleProof` for all users, with no on-chain recovery path short of a proxy implementation upgrade.

### Finding Description
`initialize()` accepts `_merkleRoot` and writes it to `merkleRoot` (line 216). No subsequent admin function can overwrite this value. `_verifyClaimProof()` (lines 281–298) verifies every claim against `merkleRoot`; a mismatch unconditionally reverts. The sibling contract `KernelMerkleDistributor.sol` exposes `setMerkleRoot(bytes32) onlyOwner` (line 402), demonstrating the pattern exists in the codebase but was omitted from `KernelTop100MerkleDistributor`.

### Impact Explanation
If the merkle root committed at initialization does not match the off-chain distribution tree (e.g., due to a deployment script error, a stale snapshot, or a tree rebuild after deployment), **all** Top-100 recipients are permanently unable to claim. Tokens remain in the contract (owner can `withdrawTokens`), so there is no direct value loss, but the entire promised distribution is inaccessible without a proxy upgrade. This matches **Low — Contract fails to deliver promised returns, but doesn't lose value**.

### Likelihood Explanation
Deployment errors (wrong argument ordering, stale tree hash, regenerated tree post-deploy) are realistic operational risks, especially for a one-shot `initializer`. The absence of a correction mechanism converts a recoverable mistake into a permanent failure requiring a contract upgrade.

### Recommendation
Add an `onlyOwner` setter mirroring the sibling contract:

```solidity
function setMerkleRoot(bytes32 _merkleRoot) external onlyOwner {
    if (_merkleRoot == bytes32(0)) revert ZeroValueProvided();
    merkleRoot = _merkleRoot;
    emit MerkleRootSet(_merkleRoot);
}
```

    function _verifyClaimProof(address user, uint256 amount, bytes32[] calldata merkleProof) internal view {
        UtilLib.checkNonZeroAddress(user);

        if (merkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (amount == 0) {
            revert ZeroValueProvided();
        }

        // Verify the merkle proof
        bytes32 leaf = keccak256(abi.encodePacked(user, amount));
        bool isValid = MerkleProofUpgradeable.verify(merkleProof, merkleRoot, leaf);

        if (!isValid) {
            revert InvalidMerkleProof();
        }
