### Title
Missing Chainlink Staleness Check Enables Phantom-Yield Fee Extraction and rsETH Holder Dilution - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` discards all freshness fields from `latestRoundData()`. A feed that has gone stale at an elevated price is accepted as valid, inflating `_getTotalEthInProtocol()`, which causes `_updateRsETHPrice()` to compute a phantom `rewardAmount`, extract a protocol fee on it, and mint excess rsETH to the treasury — diluting every existing holder's backing ratio.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards `updatedAt` and `answeredInRound`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol  line 52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

There is no check of the form `require(updatedAt >= block.timestamp - heartbeat)` or `require(answeredInRound >= roundId)`. Any price that was last pushed before the heartbeat deadline is returned as if it were current.

This stale price propagates through the entire fee-computation path:

1. `LRTOracle._getTotalEthInProtocol()` calls `getAssetPrice(asset)` for every supported asset and multiplies by on-chain deposit balances. [2](#0-1) 

2. `_updateRsETHPrice()` computes `previousTVL = rsethSupply * rsETHPrice` (using the last *stored* price, which was correct) and then compares it against the freshly fetched (stale, inflated) `totalETHInProtocol`. [3](#0-2) 

3. The positive delta is treated as real yield; a protocol fee is extracted from it and converted to rsETH, which is minted to the treasury. [4](#0-3) 

---

### Impact Explanation

Every rsETH token minted against phantom yield increases total supply without a corresponding increase in real collateral. The new `rsETHPrice` stored after the call is computed as `(totalETHInProtocol - protocolFeeInETH) / rsethSupply`; because `totalETHInProtocol` was inflated and the fee was taken from that inflated figure, the stored price is also inflated relative to real collateral. When the Chainlink feed eventually refreshes to the true (lower) market price, the next `updateRSETHPrice()` call will see a *drop* in TVL, potentially triggering the downside-protection pause — but the excess rsETH already minted to the treasury is not burned. All existing holders permanently hold tokens backed by less real ETH than the stored price implies.

**Impact: Critical — Protocol insolvency through fee over-extraction.**

---

### Likelihood Explanation

Chainlink feeds go stale routinely during periods of low volatility (the feed only updates when the price moves beyond the deviation threshold *or* the heartbeat fires). For assets like stETH/ETH or cbETH/ETH, the heartbeat is 24 hours; a network congestion event or a brief oracle outage is sufficient. `updateRSETHPrice()` is a public, permissionless function — anyone can call it at any time, including immediately after the heartbeat window expires while the feed has not yet been refreshed. [5](#0-4) 

The `pricePercentageLimit` guard only blocks non-manager callers when the computed `newRsETHPrice` exceeds `highestRsethPrice` by more than the configured threshold. A moderate staleness (within the threshold) passes the guard for any caller. Even a large staleness is bypassed by a manager calling `updateRSETHPriceAsManager()`. [6](#0-5) 

The `maxFeeMintAmountPerDay` cap limits the *magnitude* of a single extraction but does not prevent it; the fee is still minted on phantom yield up to the daily cap. [7](#0-6) 

---

### Recommendation

Add staleness and round-completeness validation inside `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound)
    = priceFeed.latestRoundData();

require(answeredInRound >= roundId, "Stale round");
require(updatedAt >= block.timestamp - maxStaleness[asset], "Stale price");
require(price > 0, "Non-positive price");
```

Store a per-asset `maxStaleness` value (set to slightly above the feed's heartbeat) and revert if the condition is not met. This prevents any stale price from entering the TVL and fee computation.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Foundry fork test — run with:
// forge test --match-test test_staleOraclePhantomFee --fork-url $MAINNET_RPC -vvvv

import "forge-std/Test.sol";
import "../contracts/LRTOracle.sol";
import "../contracts/RSETH.sol";

contract StaleOraclePoC is Test {
    LRTOracle oracle = LRTOracle(<DEPLOYED_ORACLE>);
    RSETH rseth = RSETH(<DEPLOYED_RSETH>);

    function test_staleOraclePhantomFee() external {
        uint256 supplyBefore = rseth.totalSupply();
        uint256 priceBefore  = oracle.rsETHPrice();

        // Advance time past the Chainlink heartbeat (e.g. 25 hours for a 24h feed)
        // without triggering a Chainlink update — the feed's `updatedAt` stays in the past
        vm.warp(block.timestamp + 25 hours);

        // Anyone can call this — no role required
        oracle.updateRSETHPrice();

        uint256 supplyAfter = rseth.totalSupply();
        uint256 priceAfter  = oracle.rsETHPrice();

        // rsETH supply increased (fee minted to treasury) while real collateral did not
        assertGt(supplyAfter, supplyBefore, "excess rsETH minted on phantom yield");

        // The stored price is now inflated relative to real collateral
        assertGt(priceAfter, priceBefore, "price inflated by stale oracle");
    }
}
```

The test forks mainnet, warps past the heartbeat without touching the Chainlink feed (so `latestRoundData()` still returns the old, elevated price), calls the public `updateRSETHPrice()`, and asserts that rsETH supply grew while no new collateral entered the protocol.
