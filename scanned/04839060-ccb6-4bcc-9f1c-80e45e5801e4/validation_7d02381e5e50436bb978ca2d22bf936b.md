### Title
Missing Positive-Price Bound Check in `ChainlinkPriceOracle.getAssetPrice()` Allows Zero-Price to Collapse rsETH Rate and Freeze Protocol - (File: contracts/oracles/ChainlinkPriceOracle.sol)

---

### Summary

`ChainlinkPriceOracle.getAssetPrice()` fetches `int256 price` from Chainlink's `latestRoundData()` and casts it directly to `uint256` with no check that `price > 0`. This is the direct analog of the original report's missing modulus bound check: a computed/fetched value is used without validating it falls within the required domain. A zero price silently propagates through `LRTOracle._getTotalEthInProtocol()`, collapses `totalETHInProtocol`, and causes `_updateRsETHPrice()` to compute a drastically reduced rsETH price, triggering the downside-protection auto-pause and temporarily freezing all deposits and withdrawals.

---

### Finding Description

`ChainlinkPriceOracle.getAssetPrice()` performs no positivity check on the raw Chainlink answer:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

If `price == 0` (a valid Chainlink return during a circuit-breaker or feed misconfiguration), `uint256(0)` is returned silently. This zero flows into `LRTOracle._getTotalEthInProtocol()`:

```solidity
uint256 assetER = getAssetPrice(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
``` [2](#0-1) 

A zero `assetER` for any supported LST asset zeroes out that asset's entire ETH contribution, causing `totalETHInProtocol` to be severely understated. `_updateRsETHPrice()` then computes:

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
``` [3](#0-2) 

The resulting `newRsETHPrice` is far below `highestRsethPrice`. The downside-protection branch fires:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
``` [4](#0-3) 

This auto-pauses `LRTDepositPool` and `LRTWithdrawalManager`, freezing all user deposits and withdrawals until an admin manually unpauses.

Additionally, `ChainlinkPriceOracle` also omits a staleness check (`updatedAt` is never validated), meaning a stale-but-nonzero price can silently persist and produce the same undervaluation effect over time. [5](#0-4) 

By contrast, the newer `ChainlinkOracleForRSETHPoolCollateral` wrapper used in L2 pool collateral oracles does enforce `ethPrice <= 0` revert and staleness checks:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [6](#0-5) 

The L1 `ChainlinkPriceOracle` lacks these same guards entirely.

---

### Impact Explanation

**Medium — Temporary freezing of funds.**

A zero or stale Chainlink price for any supported LST asset causes `_updateRsETHPrice()` to compute a collapsed rsETH price, triggering the auto-pause of both `LRTDepositPool` and `LRTWithdrawalManager`. All user deposits and withdrawals are frozen until an admin manually unpauses. No funds are permanently lost, but users cannot access their assets or make new deposits during the freeze window.

---

### Likelihood Explanation

Chainlink feeds can return `0` or stale data during:
- Circuit-breaker events (price moves beyond deviation threshold)
- Feed deprecation or misconfiguration
- L1 network congestion causing missed heartbeat updates

`updateRSETHPrice()` is a public, permissionless function — any external caller (including a bot or attacker) can invoke it at any time, including immediately after a Chainlink feed returns a zero/stale answer. The window between a bad Chainlink answer and a triggered auto-pause is a single transaction.

---

### Recommendation

Add a positivity and staleness check in `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

---

### Proof of Concept

1. Chainlink's ETH-denominated LST feed (e.g., stETH/ETH) returns `price = 0` due to a circuit-breaker event.
2. Any address calls `LRTOracle.updateRSETHPrice()` (public, no access control).
3. `getAssetPrice(stETH)` returns `0`.
4. `_getTotalEthInProtocol()` omits stETH's entire TVL contribution.
5. `newRsETHPrice` is computed far below `highestRsethPrice`.
6. `isPriceDecreaseOffLimit` evaluates to `true` (assuming `pricePercentageLimit > 0`).
7. `LRTDepositPool.pause()`, `LRTWithdrawalManager.pause()`, and `LRTOracle._pause()` are all called.
8. All user deposits and withdrawals revert with `Pausable: paused` until admin intervention.
