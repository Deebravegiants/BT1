### Title
Stale Merkle Proof Invalidation Causes Temporary Freezing of Claims — (`contracts/utils/MerkleBlastPointsDistributor.sol`)

### Summary
`claim()` accepts any historical index `[1, currentIndex]` but verifies exclusively against `currentMerkleRoot`. Every `setMerkleRoot()` call silently invalidates all outstanding proofs for prior roots, temporarily freezing any user who holds a proof for a non-current root.

### Finding Description
`setMerkleRoot()` overwrites `currentMerkleRoot` without archiving the previous root: [1](#0-0) 

`claim()` then verifies the caller's proof exclusively against this single stored root: [2](#0-1) 

Yet the `InvalidIndex` guard permits any index from 1 to `currentIndex`: [3](#0-2) 

This creates a structural contradiction: the index range implies historical claims are valid, but the single-root verification means only proofs for the **latest** root will pass. Any user who received a proof for root K and has not yet claimed before root N is set will receive `InvalidMerkleProof` on every attempt using their existing proof.

The same flaw exists identically in `MerkleDistributor.sol`: [4](#0-3) 

### Impact Explanation
Users who hold proofs for any root other than `currentMerkleRoot` are temporarily unable to claim. They remain frozen until the off-chain distribution system regenerates and delivers a proof valid for the current root. If the off-chain system is delayed, unavailable, or does not re-issue proofs promptly after each root update, the freeze extends. No funds are permanently lost, but the claim entitlement is inaccessible for an unbounded window.

**Impact: Medium — Temporary freezing of funds.**

### Likelihood Explanation
This triggers on every routine `setMerkleRoot()` call. Any user who has not claimed between two consecutive root updates is affected. No attacker action is required — normal protocol operation is sufficient to trigger the freeze.

### Recommendation
Store a mapping from index to root, and verify the proof against the root corresponding to the claimed index:

```solidity
mapping(uint256 => bytes32) public merkleRoots;

function setMerkleRoot(bytes32 _merkleRootToSet) external onlyOwner {
    currentIndex++;
    merkleRoots[currentIndex] = _merkleRootToSet;
    currentMerkleRoot = _merkleRootToSet; // keep for compatibility
    emit MerkleRootSet(currentIndex, _merkleRootToSet);
}

// In claim():
bytes32 rootForIndex = merkleRoots[index];
if (!MerkleProofUpgradeable.verify(merkleProof, rootForIndex, node)) {
    revert InvalidMerkleProof();
}
```

This allows users to claim at any historical index using the proof they originally received.
