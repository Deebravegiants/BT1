### Title
Missing `msg.sender == account` Guard Enables Griefing of Victim's Claim State — (`contracts/utils/MerkleDistributor/MerkleBlastPointsDistributor.sol`)

### Summary

`MerkleBlastPointsDistributor.claim()` accepts an arbitrary `account` parameter with no check that `msg.sender == account`. Any caller who possesses a valid Merkle proof for a victim can execute the victim's claim, advancing the victim's `lastClaimedIndex` and storing their cumulative amounts — without the victim's consent. If the next Merkle root does not increase the victim's cumulative allocation, the victim is permanently unable to advance their claim state.

### Finding Description

The `claim` function at lines 86–131 of `MerkleBlastPointsDistributor.sol` performs no caller-identity check:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeBlastPointAmount,
    uint256 cumulativeBlastGoldAmount,
    bytes32[] calldata merkleProof
)
    external
    override
    whenNotPaused
{
    // No msg.sender == account check anywhere
    ...
    userClaims[account].lastClaimedIndex = index;
    userClaims[account].cumulativeBlastPointAmount = cumulativeBlastPointAmount;
    userClaims[account].cumulativeBlastGoldAmount = cumulativeBlastGoldAmount;
``` [1](#0-0) 

The sibling contract `KernelMerkleDistributor._processClaim()` correctly enforces this guard:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [2](#0-1) 

The `isClaimed` check uses `lastClaimedIndex >= index`, so once an attacker sets `lastClaimedIndex = currentIndex` for the victim, the victim's own transaction reverts with `AlreadyClaimed`. [3](#0-2) 

At the next root, if the victim's cumulative amounts are unchanged (no new points earned in that period), the delta computation yields zero and the function reverts with `NoPointsToClaim`:

```solidity
uint256 claimableBlastPoints = cumulativeBlastPointAmount - userClaims[account].cumulativeBlastPointAmount;
uint256 claimableBlastGold   = cumulativeBlastGoldAmount  - userClaims[account].cumulativeBlastGoldAmount;
if (claimableBlastPoints == 0 && claimableBlastGold == 0) {
    revert NoPointsToClaim();
}
``` [4](#0-3) 

The victim is then permanently unable to advance their claim state until a future root increases their cumulative allocation.

### Impact Explanation

**Impact: Medium — Permanent freezing of unclaimed yield.**

The `claim` function does not transfer any ERC20 tokens; it only updates internal accounting state and emits a `Claimed` event. Blast Points are distributed off-chain by the operator. Therefore, no on-chain "funds" in the traditional sense are frozen. However, the victim's claim state is corrupted: they cannot record a claim at the current index, and if the next root carries the same cumulative totals, they cannot claim at the next index either. This permanently blocks the victim from advancing their on-chain claim record, which is the prerequisite for receiving future incremental allocations through this contract. This fits "Medium. Permanent freezing of unclaimed yield."

The "Critical. Permanent freezing of funds" classification in the question is overstated because no token transfer occurs inside `claim`.

### Likelihood Explanation

**Likelihood: Medium.**

- Merkle proofs for a given index are deterministic and publicly derivable from off-chain distribution data (or observable from a pending mempool transaction).
- The attack requires no special role, no capital, and no complex setup — just a valid proof and a higher-priority transaction.
- The "permanent" condition requires the next root to carry unchanged cumulative amounts for the victim, which is realistic for users who are inactive in a given period.
- The attacker gains nothing financially; this is a pure griefing vector.

### Recommendation

Add a caller-identity check at the top of `claim`, mirroring `KernelMerkleDistributor`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [5](#0-4)

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeBlastPointAmount,
        uint256 cumulativeBlastGoldAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node =
            keccak256(abi.encodePacked(index, account, cumulativeBlastPointAmount, cumulativeBlastGoldAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableBlastPoints = cumulativeBlastPointAmount - userClaims[account].cumulativeBlastPointAmount;
        uint256 claimableBlastGold = cumulativeBlastGoldAmount - userClaims[account].cumulativeBlastGoldAmount;

        // Ensure there is something to claim
        if (claimableBlastPoints == 0 && claimableBlastGold == 0) {
            revert NoPointsToClaim();
        }

        // Update user claim info
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeBlastPointAmount = cumulativeBlastPointAmount;
        userClaims[account].cumulativeBlastGoldAmount = cumulativeBlastGoldAmount;

        emit Claimed(index, account, claimableBlastPoints, claimableBlastGold);
    }
```
