### Title
Stale Rate in `CrossChainRateReceiver` Allows Over-Minting of agETH, Stealing Yield from Existing Holders — (`contracts/agETH/AGETHRateReceiver.sol`, `contracts/cross-chain/CrossChainRateReceiver.sol`, `contracts/agETH/AGETHPoolV3.sol`)

---

### Summary

`CrossChainRateReceiver` stores `lastUpdated` but never enforces a maximum staleness age. `AGETHPoolV3.deposit()` consumes the stale rate directly to mint agETH. When the L2 rate lags behind the true L1 rate (which rises continuously as staking yield accrues), any depositor receives more agETH than their ETH warrants, diluting existing holders' unclaimed yield.

---

### Finding Description

`CrossChainRateReceiver.getRate()` returns the stored `rate` unconditionally: [1](#0-0) 

`lastUpdated` is written on every `lzReceive` call but is never read back or compared against any maximum age anywhere in the contract: [2](#0-1) [3](#0-2) 

`AGETHPoolV3.deposit()` calls `viewSwapAgETHAmountAndFee`, which fetches the rate and computes the mint amount: [4](#0-3) 

The formula is:

```
agETHAmount = amountAfterFee * 1e18 / agETHToETHrate
```

Because agETH is a yield-bearing token, its ETH-denominated rate rises monotonically over time. Between LayerZero updates, the L2 `rate` is always lower than the true L1 rate. A depositor who acts during this window receives:

```
excess_agETH = amountAfterFee * 1e18 * (1/R_stale - 1/R_true)
```

This excess agETH is unbacked — the ETH bridged to L1 via `moveAssetsForBridging` is only what was deposited, not what the inflated agETH supply implies. Existing holders' share of the backing pool is diluted. [5](#0-4) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Every deposit made while the L2 rate is stale extracts yield that belongs to existing agETH holders. The attacker bridges the excess agETH to L1 and redeems it for more ETH than they deposited. The shortfall is borne by the existing holder pool. The magnitude scales with deposit size and the degree of staleness (time since last LayerZero update × yield rate).

---

### Likelihood Explanation

**Medium-High.** The L2 rate is *always* stale to some degree — it can only be as fresh as the last LayerZero message. Rate updates are push-based and periodic; they are not triggered on every block. A sophisticated depositor can monitor the L1 rate (e.g., via the agETH rate provider contract) and deposit on L2 immediately after yield accrues but before the next update arrives. No special role, key, or governance action is required.

---

### Recommendation

Add a configurable `maxStaleness` parameter to `CrossChainRateReceiver` and revert in `getRate()` if `block.timestamp - lastUpdated > maxStaleness`:

```solidity
uint256 public maxStaleness; // e.g. 24 hours

function getRate() external view returns (uint256) {
    require(
        lastUpdated != 0 && block.timestamp - lastUpdated <= maxStaleness,
        "Rate is stale"
    );
    return rate;
}
```

`AGETHPoolV3.deposit()` will then revert when the oracle is stale, preventing over-minting until a fresh rate arrives.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Fork test (local fork of L2 deployment)
// 1. Deploy a mock AGETHRateReceiver that returns a rate N% below the true L1 rate
// 2. Deploy AGETHPoolV3 pointing to the mock receiver
// 3. Record trueRate from L1 oracle
// 4. Deposit 1 ETH via AGETHPoolV3.deposit()
// 5. Assert: mintedAgETH * trueRate / 1e18 > 1 ETH (deposited)

contract StaleRatePoC {
    uint256 constant TRUE_RATE  = 1.05e18; // true L1 rate: 1 agETH = 1.05 ETH
    uint256 constant STALE_RATE = 1.00e18; // stale L2 rate: 1 agETH = 1.00 ETH

    function test_staleRateOverMint(AGETHPoolV3 pool, IERC20 agETH) external payable {
        // pool.agETHOracle() returns STALE_RATE
        uint256 depositAmount = 1 ether;
        uint256 agETHBefore = agETH.balanceOf(address(this));

        pool.deposit{value: depositAmount}("poc");

        uint256 minted = agETH.balanceOf(address(this)) - agETHBefore;
        // minted = 1e18 * 1e18 / 1.00e18 = 1e18 agETH
        // worth at true rate: 1e18 * 1.05e18 / 1e18 = 1.05 ETH
        // profit = 0.05 ETH per 1 ETH deposited (5% staleness)
        assert(minted * TRUE_RATE / 1e18 > depositAmount);
    }
}
```

The assertion confirms that the minted agETH, valued at the true L1 rate, exceeds the deposited ETH — quantifying the yield extracted per basis point of staleness.
