### Title
Manager Calling `depositAssetForL1Vault(wstETH)` Before `unwrapWstETH()` Always Reverts, Leaving wstETH Idle — (`contracts/L1VaultV2.sol`)

---

### Summary

`L1VaultV2.depositAssetForL1Vault` accepts any token address and attempts to deposit it directly into `LRTDepositPool`. Because wstETH is never registered as a supported asset in `LRTConfig`, passing wstETH will always revert. The correct operational sequence — `unwrapWstETH()` first, then `depositAssetForL1Vault(stETH)` — is not enforced by the contract, so a manager calling the wrong function or the wrong token leaves wstETH sitting idle in the vault and delays rsETH minting for L2 users.

---

### Finding Description

**Entrypoint:** `L1VaultV2.depositAssetForL1Vault(address token)` — callable only by `MANAGER_ROLE`.

**Intended flow for wstETH bridged from L2:**
1. wstETH arrives in `L1VaultV2` from the L2 bridge.
2. Manager calls `unwrapWstETH()` → converts wstETH → stETH.
3. Manager calls `depositAssetForL1Vault(stETH)` → deposits stETH into `LRTDepositPool` → mints rsETH.

**Broken flow:**

If the manager calls `depositAssetForL1Vault(wstETH)` (step 3 before step 2, or passing the wrong token):

```
depositAssetForL1Vault(wstETH)
  → tokenBalance = IERC20(wstETH).balanceOf(this)   // > 0, vault holds wstETH
  → lrtDepositPool.getRsETHAmountToMint(wstETH, ...)
      → lrtOracle.getAssetPrice(wstETH)              // reverts: no oracle / onlySupportedAsset check
  OR
  → rsETHAmountToMint == 0 → revert InvalidMinRSETHAmountExpected
  OR (if oracle somehow returns a value)
  → lrtDepositPool.depositAsset(wstETH, ...)
      → onlySupportedERC20Token(wstETH)
          → lrtConfig.isSupportedAsset(wstETH) == false
          → revert AssetNotSupported
```

At every possible path, the call reverts. wstETH remains in `L1VaultV2` and no rsETH is minted.

**Root cause:** `depositAssetForL1Vault` has no guard preventing the manager from passing wstETH as the token, and the contract does not enforce the required call order (`unwrapWstETH` → `depositAssetForL1Vault(stETH)`).

**Relevant code:**

`L1VaultV2.depositAssetForL1Vault` — accepts any token, no wstETH guard: [1](#0-0) 

`LRTDepositPool.depositAsset` — gated by `onlySupportedERC20Token`: [2](#0-1)

---

### Impact Explanation

wstETH sits idle in `L1VaultV2`. L2 users who bridged wstETH receive no rsETH until the manager discovers the mistake and calls `unwrapWstETH()` followed by `depositAssetForL1Vault(stETH)`. No funds are permanently lost — the wstETH is recoverable — but the contract fails to deliver the promised conversion to rsETH during the delay. This matches **Low: Contract fails to deliver promised returns, but doesn't lose value**.

---

### Likelihood Explanation

The manager is a trusted role, so this is an operational mistake rather than an adversarial attack. However, the risk is realistic because:
- `depositAssetForL1Vault` accepts any address with no wstETH-specific guard.
- The required two-step sequence (`unwrapWstETH` → `depositAssetForL1Vault(stETH)`) is not enforced on-chain.
- A manager unfamiliar with the flow, or one who passes `wstETH` instead of `stETH` after unwrapping, will silently fail (revert) with no rsETH minted.

Likelihood is **Low** (trusted role, recoverable), but the path is concrete and locally testable.

---

### Recommendation

Add an explicit guard in `depositAssetForL1Vault` that reverts if `token == wstETH`:

```solidity
function depositAssetForL1Vault(address token) external nonReentrant onlyRole(MANAGER_ROLE) {
    UtilLib.checkNonZeroAddress(token);
    if (token == wstETH) revert UseUnwrapWstETHFirst(); // <-- add this
    ...
}
```

Alternatively, add a combined helper `unwrapAndDepositWstETH()` that atomically calls `unwrapWstETH()` and then `depositAssetForL1Vault(stETH)`, eliminating the ordering risk entirely.

### Proof of Concept

Fork test outline (Mainnet fork, no public-mainnet state changes):

```solidity
function test_depositWstETHBeforeUnwrapReverts() public {
    // Setup: send wstETH to L1VaultV2 (simulating L2 bridge delivery)
    deal(wstETH, address(l1VaultV2), 10 ether);

    // Precondition: wstETH is NOT a supported asset in LRTConfig
    assertFalse(lrtConfig.isSupportedAsset(wstETH));

    // Manager calls depositAssetForL1Vault(wstETH) WITHOUT calling unwrapWstETH() first
    vm.prank(manager);
    vm.expectRevert(); // AssetNotSupported or InvalidMinRSETHAmountExpected
    l1VaultV2.depositAssetForL1Vault(wstETH);

    // wstETH is still sitting idle in the vault
    assertEq(IERC20(wstETH).balanceOf(address(l1VaultV2)), 10 ether);

    // Correct sequence: unwrap first, then deposit stETH
    vm.prank(manager);
    l1VaultV2.unwrapWstETH();

    address stETH = lrtConfig.getLSTToken(LRTConstants.ST_ETH_TOKEN);
    vm.prank(manager);
    l1VaultV2.depositAssetForL1Vault(stETH); // succeeds
}
```

    function depositAssetForL1Vault(address token) external nonReentrant onlyRole(MANAGER_ROLE) {
        UtilLib.checkNonZeroAddress(token);

        uint256 tokenBalance = IERC20(token).balanceOf(address(this));
        uint256 rsETHAmountToMint = lrtDepositPool.getRsETHAmountToMint(token, tokenBalance);

        if (rsETHAmountToMint == 0) {
            revert InvalidMinRSETHAmountExpected();
        }

        // Approve the LRT deposit pool to transfer the token
        IERC20(token).safeIncreaseAllowance(address(lrtDepositPool), tokenBalance);

        lrtDepositPool.depositAsset(token, tokenBalance, rsETHAmountToMint, "");

        emit AssetDepositForL1Vault(token, tokenBalance, rsETHAmountToMint);
    }

    function unwrapWstETH() external nonReentrant onlyRole(MANAGER_ROLE) {
        uint256 wstETHBalance = IERC20(wstETH).balanceOf(address(this));

        if (wstETHBalance == 0) {
            revert NoWstETHBalance();
        }

        // Unwrap wstETH to stETH

        emit WstETHUnwrapped(stETHAmount);
    }
