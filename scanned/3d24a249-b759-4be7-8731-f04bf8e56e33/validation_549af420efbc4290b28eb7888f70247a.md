### Title
Nested Unbounded Loop in `NodeDelegator.getAssetUnstaking()` Propagates to User-Facing `initiateWithdrawal()` and Public `updateRSETHPrice()` — (File: `contracts/NodeDelegator.sol`)

---

### Summary

`NodeDelegator.getAssetUnstaking()` iterates over all queued EigenLayer withdrawals (with a nested loop over strategies per withdrawal). This function is called for every NDC inside `LRTDepositPool.getAssetDistributionData()`, which is itself called from the user-facing `LRTWithdrawalManager.initiateWithdrawal()` and the public `LRTOracle.updateRSETHPrice()`. As the number of NDCs, queued withdrawals, and supported assets grows through normal protocol operation, these call chains can exhaust the block gas limit, temporarily freezing user withdrawals and price updates.

---

### Finding Description

`NodeDelegator.getAssetUnstaking()` fetches all queued withdrawals from EigenLayer's `DelegationManager` and iterates over them with a nested loop: [1](#0-0) 

This function is called for **every NDC** inside `LRTDepositPool.getAssetDistributionData()`: [2](#0-1) 

`getAssetDistributionData()` is called by `getTotalAssetDeposits()`: [3](#0-2) 

`getTotalAssetDeposits()` is called in two critical paths:

**Path 1 — User withdrawal initiation:**
`LRTWithdrawalManager.initiateWithdrawal()` → `getAvailableAssetAmount()` → `lrtDepositPool.getTotalAssetDeposits(asset)`: [4](#0-3) 

**Path 2 — Public price update:**
`LRTOracle.updateRSETHPrice()` → `_getTotalEthInProtocol()` iterates over every supported asset and calls `getTotalAssetDeposits(asset)` for each: [5](#0-4) 

The total gas cost scales as **O(N\_assets × N\_NDCs × N\_queued\_withdrawals × N\_strategies\_per\_withdrawal)**. Each dimension grows through normal protocol operation:

- `supportedAssetList` has **no explicit cap** — `addNewSupportedAsset` (TIME\_LOCK\_ROLE) can add assets indefinitely: [6](#0-5) 
- `nodeDelegatorQueue` is bounded by `maxNodeDelegatorLimit` (default 10, admin-adjustable upward): [7](#0-6) 
- Queued withdrawals per NDC are bounded globally by `maxUncompletedWithdrawalCount`, but each `getQueuedWithdrawals` call to EigenLayer returns all withdrawals for that specific NDC, and the nested loop over strategies per withdrawal adds further cost.

---

### Impact Explanation

- **Temporary freezing of user withdrawals (Medium):** `initiateWithdrawal()` is a user-facing function that calls the full nested loop chain. If the gas cost exceeds the block limit, every call to `initiateWithdrawal()` reverts, preventing users from queuing withdrawals.
- **Inability to update rsETH price (Medium):** `updateRSETHPrice()` is public and callable by anyone. If it OOGs, the stored `rsETHPrice` becomes stale, causing all subsequent mints and withdrawals to use an incorrect exchange rate.

---

### Likelihood Explanation

As the protocol scales — more supported assets added by governance, more NDCs deployed, and more concurrent EigenLayer unstaking operations in flight — the gas cost of the nested loop grows multiplicatively. This is a realistic operational state, not a theoretical edge case. Operators legitimately call `initiateUnstaking` repeatedly as part of normal restaking management, accumulating queued withdrawals.

---

### Recommendation

1. Cache the result of `getAssetUnstaking()` per NDC per block, or restructure accounting so that unstaking amounts are tracked in protocol storage (updated on `initiateUnstaking`/`completeUnstaking`) rather than recomputed on every read.
2. Introduce an explicit cap on `supportedAssetList` length in `addNewSupportedAsset`.
3. Consider separating the price-update path from the full asset-distribution computation, so `updateRSETHPrice()` does not need to call `getAssetUnstaking()` on every NDC for every asset on every invocation.

---

### Proof of Concept

1. Protocol has 5 supported assets, 10 NDCs, and 20 total queued EigenLayer withdrawals each with 2 strategies.
2. A user calls `initiateWithdrawal(asset, rsETHAmount, referralId)`.
3. Internally: `getAvailableAssetAmount(asset)` → `getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)` → for each of 10 NDCs: `getAssetUnstaking(asset)` → `getQueuedWithdrawals(ndcAddr)` (external call) + loop over up to 20 withdrawals × 2 strategies.
4. For `updateRSETHPrice()`, the same inner loop runs for **each of the 5 assets**, multiplying the cost by 5.
5. With realistic gas costs per SLOAD and external call, the combined gas for `updateRSETHPrice()` can approach or exceed the 30M block gas limit as these parameters grow, causing the transaction to revert and temporarily freezing withdrawal initiation and price updates.

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```

```text
    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```
