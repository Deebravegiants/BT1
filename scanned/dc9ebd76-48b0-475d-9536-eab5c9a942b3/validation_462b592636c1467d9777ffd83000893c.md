### Title
Stale `rsETHPrice` Allows Depositors to Capture Unclaimed Yield Before Price Update — (`contracts/LRTOracle.sol`, `contracts/LRTDepositPool.sol`)

---

### Summary

`getRsETHAmountToMint` divides by the **stored** `rsETHPrice` state variable, which is only updated on explicit calls to `updateRSETHPrice()`. Because stETH is a rebasing token, the protocol's total ETH backing grows automatically between price updates. Any depositor who deposits during this window receives more rsETH than their fair share, capturing yield that belongs to existing holders.

---

### Finding Description

`getRsETHAmountToMint` computes the rsETH mint amount as:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [1](#0-0) 

The numerator uses `getAssetPrice(asset)`, which reads live from Chainlink and returns the current stETH/ETH exchange rate (~1e18, unchanged by a rebase). The denominator uses `lrtOracle.rsETHPrice()`, which reads the **cached** state variable `rsETHPrice` in `LRTOracle`:

```solidity
uint256 public override rsETHPrice;
``` [2](#0-1) 

This value is only updated when `updateRSETHPrice()` / `_updateRsETHPrice()` is explicitly called:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [3](#0-2) 

`_updateRsETHPrice()` computes the new price from `_getTotalEthInProtocol()`, which sums `totalAssetAmt * assetER` across all supported assets:

```solidity
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
``` [4](#0-3) 

When stETH rebases, `IERC20(stETH).balanceOf(...)` increases across the deposit pool, NDCs, and unstaking vault — so `getTotalAssetDeposits(stETH)` grows — but `rsETHPrice` does **not** update automatically. The stored price is therefore lower than the true backing ratio.

`_beforeDeposit` performs no freshness check on `rsETHPrice`:

```solidity
rsethAmountToMint = getRsETHAmountToMint(asset, depositAmount);
if (rsethAmountToMint < minRSETHAmountExpected) {
    revert MinimumAmountToReceiveNotMet();
}
``` [5](#0-4) 

`minRSETHAmountExpected` is caller-controlled and provides no protection against over-minting.

---

### Impact Explanation

When `rsETHPrice` is stale (lower than true backing), the formula `amount * assetPrice / rsETHPrice` over-mints rsETH for the depositor. After `updateRSETHPrice()` is called and the price rises to reflect the rebase, the attacker's rsETH represents more ETH than they deposited. The excess comes directly from the yield that should have accrued to existing holders. This is **theft of unclaimed yield** (High impact per scope).

The protocol does not become insolvent — total backing still covers total supply — but yield is permanently redistributed from existing rsETH holders to the attacker. The magnitude per attack is bounded by the yield accrued since the last price update (stETH APY ≈ 3–4% / 365 ≈ 0.01% per day of TVL), but it is repeatable every update cycle.

---

### Likelihood Explanation

- `updateRSETHPrice()` is public but not called atomically with deposits; there is always a window between yield accrual and price update.
- stETH rebases daily; the window is predictable and observable on-chain.
- No special role, flashloan, or governance action is required — any EOA can exploit this.
- The attacker sets `minRSETHAmountExpected` to the inflated amount they expect, so the slippage guard does not block them.

---

### Recommendation

Before computing `rsethAmountToMint`, call `updateRSETHPrice()` (or an internal equivalent) to ensure `rsETHPrice` reflects the current backing. Alternatively, compute the mint amount on-the-fly from `_getTotalEthInProtocol()` and `rsethSupply` rather than from the cached `rsETHPrice`, so deposits always use a fresh ratio.

---

### Proof of Concept

```solidity
// Fork mainnet at block B (stETH has just rebased, updateRSETHPrice not yet called)
// rsETHPrice = P_stale  (< true backing P_true)

// Step 1: Attacker deposits stETH
uint256 staleRsETH = depositPool.getRsETHAmountToMint(stETH, DEPOSIT);
// staleRsETH = DEPOSIT * stETHPrice / P_stale  >  DEPOSIT * stETHPrice / P_true

depositPool.depositAsset(stETH, DEPOSIT, 0, "");
// attacker receives staleRsETH rsETH

// Step 2: Anyone calls updateRSETHPrice — price rises to P_true
lrtOracle.updateRSETHPrice();
// rsETHPrice = P_true > P_stale

// Step 3: Attacker initiates withdrawal
// Each rsETH is now redeemable at P_true ETH
// attacker redeems staleRsETH * P_true > DEPOSIT * stETHPrice
// profit = staleRsETH * (P_true - P_stale) = yield stolen from existing holders

assert(attackerETHOut > DEPOSIT * stETHPrice / 1e18); // profitable round-trip
```

The invariant `unclaimed yield cannot be stolen by a public caller` is broken: the stale `rsETHPrice` denominator in `getRsETHAmountToMint` allows any EOA to capture accrued-but-unrecorded yield by depositing before `updateRSETHPrice()` is called.
