### Title
Interest Collection Blocks Emergency Principal Recovery When Treasury Cannot Receive ETH — (`contracts/LRTWithdrawalManager.sol`)

### Summary
`emergencyWithdrawFromAave` unconditionally calls `_collectInterestToTreasury` before `_withdrawFromAave`. If Aave interest has accrued and the treasury address is a contract that cannot receive ETH, the treasury transfer reverts the entire transaction, permanently blocking the emergency path for yield collection and temporarily blocking principal recovery until the treasury is fixed.

### Finding Description

`emergencyWithdrawFromAave` is the sole privileged escape hatch for the `PAUSER_ROLE` to pull funds out of Aave in an emergency: [1](#0-0) 

It unconditionally calls `_collectInterestToTreasury` first. Inside that function, if `aaveBalance > totalETHDepositedToAave` (i.e., interest has accrued), it:
1. Withdraws the interest from Aave to the contract (`aaveWETHGateway.withdrawETH`)
2. Attempts to push ETH to the treasury via a low-level call
3. Reverts with `TreasuryTransferFailed` if the push fails [2](#0-1) 

Because the revert unwinds the entire transaction, `_withdrawFromAave` on line 560 is never reached.

Every other admin path that withdraws from Aave (`setAaveIntegrationEnabled(false)`, `configureAaveIntegration`) also calls `_collectInterestToTreasury` first, so they are equally blocked: [3](#0-2) [4](#0-3) 

The only code path that calls `_withdrawFromAave` without first going through `_collectInterestToTreasury` is `_processWithdrawalCompletion` (user withdrawal flow). However, that path caps withdrawals at `withdrawablePrincipal = min(aaveBalance, totalETHDepositedToAave)`, so the interest portion remains in Aave regardless: [5](#0-4) 

The treasury address is stored in `LRTConfig` and can be updated by `DEFAULT_ADMIN_ROLE` via `LRTConfig.setContract`: [6](#0-5) 

This means the principal freeze is **temporary** (admin can update the treasury address to unblock the emergency path). However, the **yield/interest** is permanently frozen in Aave for as long as the treasury cannot receive ETH, because there is no code path that collects interest without pushing it to the treasury.

### Impact Explanation

- **Unclaimed yield (interest)**: Permanently frozen in Aave if the treasury is permanently unable to receive ETH. Even if the treasury is only temporarily broken, the yield accrued during that window cannot be retroactively collected without fixing the treasury first.
- **Principal**: Temporarily frozen via the emergency path, but recoverable through user withdrawals or by the admin updating the treasury address. Not permanently frozen.

The scoped impact is **Medium — Permanent freezing of unclaimed yield**.

### Likelihood Explanation

The treasury is expected to be a smart contract (multisig, DAO treasury, or similar). Such contracts can temporarily or permanently lose the ability to receive ETH (e.g., a Gnosis Safe with a broken fallback, a contract upgrade that removes `receive()`, or a contract that runs out of gas on receipt). This is a realistic operational scenario, not a theoretical one. The precondition (interest accrued in Aave) is guaranteed to be true after any meaningful period of operation.

### Recommendation

Decouple interest collection from principal recovery in the emergency path. One approach: if `_collectInterestToTreasury` fails (wrap in try/catch or check treasury receivability), allow `emergencyWithdrawFromAave` to proceed with principal withdrawal anyway, leaving the interest in Aave for later collection. Alternatively, add a separate `emergencyWithdrawPrincipalOnly` function that skips interest collection entirely.

```text
    function _collectInterestToTreasury() internal returns (uint256 interestAmount) {
        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        uint256 principal = totalETHDepositedToAave;

        // Return 0 if no interest or balance is less than principal (accounting for rounding)
        if (aaveBalance <= principal) return 0;

        interestAmount = aaveBalance - principal;

        aaveWETHGateway.withdrawETH(aavePool, interestAmount, address(this));

        address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        (bool sent,) = payable(treasury).call{ value: interestAmount }("");
        if (!sent) revert TreasuryTransferFailed();

        emit InterestCollectedToTreasury(interestAmount, treasury);
    }
```
