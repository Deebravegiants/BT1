### Title
Malfunctioning Asset Price Oracle Permanently Freezes `updateRSETHPrice()`, Causing Theft of Unclaimed Yield - (File: contracts/LRTOracle.sol)

---

### Summary

`LRTOracle._getTotalEthInProtocol()` iterates over every supported asset and calls `getAssetPrice(asset)` for each one. There is no mechanism to disable or skip a broken oracle. If any single asset's external price oracle permanently reverts (e.g., the underlying protocol is hacked or killed), `updateRSETHPrice()` is permanently DOSed, the rsETH price is frozen at a stale value, protocol fee minting is blocked, and new depositors receive more rsETH than they deserve — diluting and stealing yield from existing rsETH holders.

---

### Finding Description

`LRTOracle._getTotalEthInProtocol()` loops through the full `supportedAssetList` and calls `getAssetPrice(asset)` for each entry:

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);          // external oracle call
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    unchecked { ++assetIdx; }
}
```

`getAssetPrice` delegates to the registered `IPriceFetcher` for each asset:

```solidity
function getAssetPrice(address asset) public view onlySupportedOracle(asset) returns (uint256) {
    return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
}
```

The price fetchers call into external protocols:
- `EthXPriceOracle` → `IETHXStakePoolsManager.getExchangeRate()`
- `SfrxETHPriceOracle` → `ISfrxETH.pricePerShare()`
- `SwETHPriceOracle` → `ISwETH.getRate()`
- `ChainlinkPriceOracle` → `AggregatorV3Interface.latestRoundData()`

If any one of these external calls permanently reverts (e.g., the underlying protocol is paused, killed, or hacked), `_getTotalEthInProtocol()` reverts, which causes `_updateRsETHPrice()` to revert, which causes both `updateRSETHPrice()` (public) and `updateRSETHPriceAsManager()` to revert.

There is no mechanism in `LRTOracle` or `LRTConfig` to disable or skip a broken oracle in the loop. `LRTConfig.removeSupportedAsset()` exists but requires `getTotalAssetDeposits(asset) <= maxNegligibleAmount`, which will not be satisfied for an asset with significant TVL.

---

### Impact Explanation

Once `updateRSETHPrice()` is permanently DOSed:

1. **Theft of unclaimed yield (High)**: `rsETHPrice` is frozen at its last value. As rewards continue to accrue in EigenLayer, the actual per-share value of rsETH rises above the stale cached price. New depositors calling `depositAsset` or `depositETH` compute `rsethAmountToMint = (amount * assetPrice) / rsETHPrice` using the stale (too-low) `rsETHPrice`, receiving more rsETH than they are entitled to. This dilutes existing rsETH holders, effectively stealing their accrued yield.

2. **Permanent freeze of unclaimed yield (Medium)**: Protocol fee minting inside `_updateRsETHPrice()` is permanently blocked. The treasury never receives its share of accrued rewards.

3. **Price protection disabled**: The auto-pause logic that triggers when `newRsETHPrice` drops below `highestRsethPrice * pricePercentageLimit` can never fire, removing a key safety mechanism.

---

### Likelihood Explanation

The supported LST ecosystem (ETHx via Stader, sfrxETH via Frax, swETH via Swell) each depend on external protocol contracts. Any of these protocols being exploited, paused, or killed — a realistic scenario given the history of DeFi exploits — would trigger the revert. The Chainlink oracle path can also revert if the feed is deprecated or the aggregator is paused. This is a medium-likelihood event given the number of external dependencies in the loop.

---

### Recommendation

Add a per-asset `oracleEnabled` flag to `LRTOracle` (callable by admin/manager) that allows skipping a broken oracle in `_getTotalEthInProtocol()`. When an oracle is disabled, its contribution to TVL should either be omitted or use a last-known-good cached value. This mirrors the enable/disable derivative mitigation applied to SafEth.

Alternatively, wrap each `getAssetPrice(asset)` call in a `try/catch` inside `_getTotalEthInProtocol()` and fall back to the last cached price for that asset, emitting an event to alert operators.

---

### Proof of Concept

1. `swETH` is a supported asset. Its oracle is `SwETHPriceOracle`, which calls `ISwETH(swETHAddress).getRate()`.
2. The Swell protocol is exploited and its `getRate()` function is bricked (permanently reverts).
3. Any call to `LRTOracle.updateRSETHPrice()` now reverts: `updateRSETHPrice()` → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(swETH)` → `SwETHPriceOracle.getAssetPrice()` → `ISwETH.getRate()` → **revert**.
4. `rsETHPrice` is frozen at its last value. EigenLayer rewards continue to accrue, so the true per-share value rises above the stale price.
5. Alice deposits stETH. `getRsETHAmountToMint` uses the stale (too-low) `rsETHPrice`, minting Alice more rsETH than she deserves. Bob, an existing rsETH holder, has his share diluted — his yield is stolen.
6. The admin cannot remove swETH from `supportedAssetList` because `getTotalAssetDeposits(swETH) > maxNegligibleAmount`, so `removeSupportedAsset` reverts with `CannotRemoveAssetWithDeposits`.

```text
    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```
