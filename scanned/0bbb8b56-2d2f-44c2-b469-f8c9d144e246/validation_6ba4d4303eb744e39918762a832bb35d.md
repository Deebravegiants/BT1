### Title
`_withdrawFromAave` Principal Cap Causes `completeWithdrawal` to Revert When Accrued Interest Bridges the Gap — (`contracts/LRTWithdrawalManager.sol`)

---

### Summary

`_withdrawFromAave` deliberately caps withdrawals at `min(aaveAWETH.balanceOf, totalETHDepositedToAave)` to reserve accrued interest for the treasury. When interest has accrued, a user's `expectedAssetAmount` can fall in the range `(totalETHDepositedToAave, aaveAWETH.balanceOf]`, causing `_processWithdrawalCompletion` to withdraw only the principal, fail the post-withdrawal balance check, and revert with `InsufficientLiquidityForWithdrawal` — even though the full aWETH balance would cover the request.

---

### Finding Description

`_withdrawFromAave` computes a `withdrawablePrincipal` ceiling:

```solidity
uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave
    ? aaveBalance
    : totalETHDepositedToAave;
withdrawnAmount = amount > withdrawablePrincipal ? withdrawablePrincipal : amount;
``` [1](#0-0) 

When `aaveBalance > totalETHDepositedToAave` (interest has accrued), `withdrawablePrincipal = totalETHDepositedToAave`. Any requested `amount` larger than `totalETHDepositedToAave` is silently capped.

`_processWithdrawalCompletion` then checks the post-withdrawal ETH balance:

```solidity
uint256 amountNeeded = request.expectedAssetAmount - contractBalance;
_withdrawFromAave(amountNeeded);
uint256 balanceAfter = address(this).balance;
if (balanceAfter < request.expectedAssetAmount) {
    revert InsufficientLiquidityForWithdrawal();
}
``` [2](#0-1) 

If `request.expectedAssetAmount > totalETHDepositedToAave` but `<= aaveAWETH.balanceOf`, the withdrawal is capped short, `balanceAfter < request.expectedAssetAmount`, and the call reverts.

---

### Impact Explanation

The user's rsETH was transferred to the contract at `initiateWithdrawal` time and burned at `unlockQueue` time. The user cannot reclaim their rsETH and cannot complete the withdrawal. The contract holds sufficient aWETH to honour the request but refuses to use the interest portion, temporarily freezing the user's ETH redemption. No funds are permanently lost, but the contract fails to deliver the promised return on demand.

Scoped impact: **Low — Contract fails to deliver promised returns, but doesn't lose value.**

---

### Likelihood Explanation

- Aave integration is a supported production feature; interest accrues continuously and passively.
- No attacker action is required — normal time passage creates the gap between `totalETHDepositedToAave` and `aaveAWETH.balanceOf`.
- Any user whose `expectedAssetAmount` (set at initiation time using the rsETH price) falls in the range `(totalETHDepositedToAave, aaveAWETH.balanceOf]` will be affected.
- Calling `collectInterestToTreasury()` before `completeWithdrawal` does not fix the issue: it withdraws the interest to treasury, leaving `aaveBalance = totalETHDepositedToAave`, which is still less than `request.expectedAssetAmount`.

---

### Recommendation

In `_processWithdrawalCompletion`, when withdrawing from Aave to cover a user request, allow the full aWETH balance to be used (not just the principal). The interest-reservation logic should be relaxed when user redemptions require it. One approach: in `_withdrawFromAave`, when called from the withdrawal completion path, use `aaveBalance` as the ceiling instead of `totalETHDepositedToAave`, and adjust `totalETHDepositedToAave` accordingly (flooring at zero). Alternatively, track and update `totalETHDepositedToAave` correctly when interest is implicitly consumed.

    function _withdrawFromAave(uint256 amount) internal returns (uint256 withdrawnAmount) {
        if (amount == 0) return 0;

        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        if (aaveBalance == 0) revert InsufficientAaveBalance();

        // Only withdraw up to the principal amount (don't use accrued interest for user withdrawals)
        uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave ? aaveBalance : totalETHDepositedToAave;

        withdrawnAmount = amount > withdrawablePrincipal ? withdrawablePrincipal : amount;
        if (withdrawnAmount == 0) return 0;

        aaveWETHGateway.withdrawETH(aavePool, withdrawnAmount, address(this));
        totalETHDepositedToAave -= withdrawnAmount;

        emit ETHWithdrawnFromAave(withdrawnAmount, totalETHDepositedToAave);
    }
