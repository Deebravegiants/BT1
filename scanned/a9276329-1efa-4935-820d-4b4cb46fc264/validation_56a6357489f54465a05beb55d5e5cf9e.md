### Title
Donation Attack via Direct ETH/LST Transfer Inflates `getTotalAssetDeposits()`, Triggering Deposit Limit DoS - (File: contracts/LRTDepositPool.sol)

### Summary
A malicious user can transfer ETH or LST tokens directly to `LRTDepositPool` without going through the official deposit flow. Because `getTotalAssetDeposits()` reads the raw on-chain balance of the contract, this inflates the protocol's reported TVL. If the inflated total exceeds the configured per-asset deposit limit, all subsequent legitimate deposits for that asset are blocked with `MaximumDepositLimitReached`, causing a temporary denial of service.

### Finding Description

`LRTDepositPool` exposes a plain `receive()` function and accepts ETH from any caller:

```solidity
receive() external payable { }
```

The ETH accounting function reads the raw contract balance:

```solidity
function getETHDistributionData() public view override returns (...) {
    ethLyingInDepositPool = address(this).balance;   // ← raw balance, inflatable by donation
    ...
}
```

For LST assets the same pattern applies:

```solidity
assetLyingInDepositPool = IERC20(asset).balanceOf(address(this));  // ← raw balance
```

Both values feed directly into `getTotalAssetDeposits()`:

```solidity
return (assetLyingInDepositPool + assetLyingInNDCs + effectiveAssetWithEigenLayer
        + assetLyingInConverter + assetLyingUnstakingVault);
```

`_beforeDeposit()` calls `_checkIfDepositAmountExceedesCurrentLimit()`, which compares this inflated total against the admin-configured cap:

```solidity
function _checkIfDepositAmountExceedesCurrentLimit(address asset, uint256 amount) internal view returns (bool) {
    uint256 totalAssetDeposits = getTotalAssetDeposits(asset);
    if (asset == LRTConstants.ETH_TOKEN) {
        return (totalAssetDeposits > lrtConfig.depositLimitByAsset(asset));
    }
    return (totalAssetDeposits + amount > lrtConfig.depositLimitByAsset(asset));
}
```

If the donation pushes `totalAssetDeposits` past `depositLimitByAsset`, every call to `depositETH` or `depositAsset` reverts:

```solidity
if (_checkIfDepositAmountExceedesCurrentLimit(asset, depositAmount)) {
    revert MaximumDepositLimitReached();
}
```

### Impact Explanation

All new ETH (or LST) deposits are blocked for as long as the donated balance remains in the contract and the deposit limit is not raised by an admin. This is a **temporary freezing of user funds** (new depositors cannot enter the protocol) and maps to **Medium – Temporary freezing of funds** in the allowed impact scope. The donated assets are not lost; they are counted as protocol TVL and inflate the rsETH price, benefiting existing holders at the expense of new depositors who are locked out.

### Likelihood Explanation

The attack requires no special permissions. `LRTDepositPool.receive()` is unconditionally open, and any ERC-20 transfer to the contract address is accepted. The attacker must commit real ETH or LSTs, but the amount needed is only the gap between the current TVL and the deposit cap — which can be small when the protocol is near its limit. The attack is therefore realistic and cheap relative to the DoS it produces.

### Recommendation

Decouple the deposit-limit accounting from the raw contract balance. Maintain an explicit `depositedAmount[asset]` storage variable that is incremented only through the official `depositETH` / `depositAsset` paths and decremented through the withdrawal path. Use this variable instead of `IERC20(asset).balanceOf(address(this))` and `address(this).balance` in `getTotalAssetDeposits()` for the purpose of the deposit-limit check. Alternatively, add a `sweepDonations()` function that moves unsolicited balances to a NodeDelegator so they are properly accounted for and cannot be used to grief the limit.

### Proof of Concept

```
Initial state:
  depositLimitByAsset[ETH] = 1000 ETH
  getTotalAssetDeposits(ETH) = 990 ETH  (10 ETH headroom)

Attack:
  attacker.call{value: 11 ether}(address(lrtDepositPool), "");
  // receive() accepts it; address(this).balance now includes 11 ETH donation

After donation:
  getETHDistributionData().ethLyingInDepositPool = 990 + 11 = 1001 ETH
  getTotalAssetDeposits(ETH) = 1001 ETH  > 1000 ETH limit

Victim calls depositETH(1 ether, ...):
  _checkIfDepositAmountExceedesCurrentLimit → 1001 > 1000 → true
  → revert MaximumDepositLimitReached()   ← DoS achieved
