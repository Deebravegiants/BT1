### Title
Chainlink `latestRoundData()` Missing Staleness and Validity Checks in `ChainlinkPriceOracle` - (File: contracts/oracles/ChainlinkPriceOracle.sol)

### Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all return values except `price`. No staleness check (`updatedAt`), no incomplete-round check (`answeredInRound < roundId`), and no non-positive price guard are applied. A stale or zero price propagates directly into rsETH price computation via the publicly callable `updateRSETHPrice()`, enabling any unprivileged caller to corrupt the protocol exchange rate.

### Finding Description
In `contracts/oracles/ChainlinkPriceOracle.sol` at line 52, the call to `latestRoundData()` silently drops every validity signal:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

The discarded fields are:
- `updatedAt` — timestamp of the last price update; needed to detect a stale feed.
- `answeredInRound` — must be `>= roundId` to confirm the round is complete.
- `roundId` — needed for the above comparison.

Additionally, `price` is cast directly to `uint256` without checking `price > 0`, so a zero or negative answer (possible during a Chainlink circuit-breaker event) silently becomes a valid price.

By contrast, the sibling contract `ChainlinkOracleForRSETHPoolCollateral` applies all three guards:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

`ChainlinkPriceOracle` is the oracle registered for supported LST assets (stETH, cbETH, etc.) in `LRTOracle`. Its output flows through `LRTOracle.getAssetPrice()` → `_getTotalEthInProtocol()` → `_updateRsETHPrice()`: [3](#0-2) 

`updateRSETHPrice()` has no access control — any address may call it: [4](#0-3) 

### Impact Explanation
A stale or zero asset price fed into `_getTotalEthInProtocol()` produces an incorrect `totalETHInProtocol`, which directly sets `rsETHPrice`. An artificially low price (e.g., zero during a Chainlink outage) causes depositors to receive far more rsETH than their deposit is worth, draining value from existing holders — protocol insolvency. An artificially high price causes withdrawers to receive more ETH than they are entitled to, also draining the pool. The `highestRsethPrice` downside-protection guard only triggers if the new price is below the stored peak by more than `pricePercentageLimit`; a zero price would trigger it, but a moderately stale price (e.g., 5–10% off) may not.

**Impact: Critical — Protocol insolvency / direct theft of user funds.**

### Likelihood Explanation
Chainlink feeds do go stale during network congestion, sequencer downtime (L2), or when the asset price moves outside the deviation threshold. The entry point (`updateRSETHPrice`) is permissionless, so any user can trigger the price update at the worst possible moment (e.g., immediately after detecting a stale feed). No special privileges are required.

### Recommendation
Apply the same guards already present in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optional: add a heartbeat check, e.g. require(block.timestamp - updatedAt <= MAX_DELAY)

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
``` [5](#0-4) 

### Proof of Concept
1. Chainlink's ETH/stETH feed stops updating (network congestion or circuit-breaker).
2. `latestRoundData()` returns the last known (stale) price — e.g., 0.95 ETH — while the true price is 1.00 ETH.
3. Any address calls `LRTOracle.updateRSETHPrice()`.
4. `_getTotalEthInProtocol()` calls `ChainlinkPriceOracle.getAssetPrice(stETH)`, which returns the stale 0.95 value with no revert.
5. `rsETHPrice` is set lower than the true value.
6. A depositor calls `LRTDepositPool.depositAsset(stETH, amount)` and receives more rsETH than the true ETH value of their deposit, extracting value from all existing rsETH holders. [4](#0-3) [6](#0-5)
