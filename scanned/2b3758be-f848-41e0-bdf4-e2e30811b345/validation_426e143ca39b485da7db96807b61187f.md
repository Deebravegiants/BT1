### Title
Unbounded Loop in `pauseAll()` Over `nodeDelegatorQueue` Can Render Emergency Pause Uncallable - (`contracts/LRTConfig.sol`)

### Summary
`LRTConfig.pauseAll()` iterates over the entire `nodeDelegatorQueue` with no gas cap or partial-pause fallback. Because `maxNodeDelegatorLimit` has no upper bound and is admin-adjustable, the loop's gas cost grows linearly and unboundedly with the number of registered `NodeDelegator` contracts. At sufficient queue depth the transaction will exceed the block gas limit, permanently disabling the emergency pause mechanism.

### Finding Description

`LRTConfig.pauseAll()` performs the following unbounded loop: [1](#0-0) 

For each entry it makes **three external calls**:
1. `nodeDelegator.paused()` — cold external call (~2,600 gas) + SLOAD (~2,100 gas)
2. `nodeDelegator.pause()` — cold external call (~2,600 gas), which internally executes:
   - `onlyRole(PAUSER_ROLE)` → `IAccessControl(lrtConfig).hasRole(...)` — another cold external call (~2,600 gas) + SLOAD (~2,100 gas)
   - `_pause()` → SSTORE of the paused flag (~20,000 gas first write)

Per unpaused NDC: ≈ **32,000–35,000 gas**. At Ethereum's ~30 M gas block limit this becomes uncallable at roughly **850–950 NDCs**.

The `maxNodeDelegatorLimit` is initialized to 10 but has **no upper cap**: [2](#0-1) 

`addNodeDelegatorContractToQueue` only enforces the current limit value: [3](#0-2) 

There is no partial-pause fallback, no index-range parameter, and no gas-remaining check anywhere in `pauseAll()`.

Note: the question's premise about `IPauserRegistry.isPauser()` via EigenLayer's pausability stack is incorrect for `NodeDelegator`. The `pause()` function uses `onlyRole(LRTConstants.PAUSER_ROLE)` resolved through `LRTConfigRoleChecker`, which calls `lrtConfig.hasRole()`: [4](#0-3) [5](#0-4) 

The gas cost per iteration is still substantial and the unbounded-loop root cause is unchanged.

### Impact Explanation
If `pauseAll()` reverts due to gas exhaustion during an emergency (exploit in progress, oracle manipulation, etc.), **none of the protocol's pausable contracts can be atomically halted**. Deposits, withdrawals, and oracle updates remain live. The PAUSER_ROLE holder would have to manually call `pause()` on each contract individually, introducing latency during which funds can be drained. This matches the allowed scope: **Medium — Unbounded gas consumption** and **Medium — Temporary freezing of funds**.

### Likelihood Explanation
The current default limit is 10 NDCs, which is safe. However, the admin can raise `maxNodeDelegatorLimit` to any value with a single transaction and no timelock constraint visible in the code. As the protocol scales (more validators, more operators), adding hundreds of NDCs is operationally plausible. The vulnerability is latent today and becomes exploitable through normal protocol growth, not through any attacker action.

### Recommendation
1. **Add a hard cap** on `maxNodeDelegatorLimit` (e.g., 50 or 100) that is enforced in `updateMaxNodeDelegatorLimit`.
2. **Refactor `pauseAll()`** to accept an optional index range `(uint256 start, uint256 end)` so the caller can pause NDCs in batches if the queue is large.
3. Alternatively, emit an event and let individual NDC `pause()` calls be driven off-chain, removing the loop from `pauseAll()` entirely.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

// Local fork test (Hardhat/Foundry, no public mainnet)
// 1. Deploy LRTConfig, LRTDepositPool, N MockNodeDelegator contracts.
// 2. Admin calls updateMaxNodeDelegatorLimit(N) then addNodeDelegatorContractToQueue([...N addresses...]).
// 3. Grant PAUSER_ROLE to `pauser`.
// 4. Measure gas for pauser.pauseAll() as N increases.

contract GasTest is Test {
    function test_pauseAll_gasGrowth() public {
        uint256[] memory sizes = new uint256[](4);
        sizes[0] = 10; sizes[1] = 100; sizes[2] = 500; sizes[3] = 900;

        for (uint256 s; s < sizes.length; s++) {
            uint256 n = sizes[s];
            // deploy n MockNodeDelegators, register them, call pauseAll()
            uint256 gasBefore = gasleft();
            lrtConfig.pauseAll();
            uint256 gasUsed = gasBefore - gasleft();
            // assert gasUsed grows linearly; at n≈900 gasUsed > 30_000_000
            emit log_named_uint("NDCs", n);
            emit log_named_uint("gasUsed", gasUsed);
        }
    }
}
```

Expected output: gas consumed ≈ `35_000 * n`. At `n = 900`, gas ≈ 31.5 M, exceeding the block gas limit and causing `pauseAll()` to revert on every attempt, permanently disabling the atomic emergency pause.
