### Title
Arbitrum Address Aliasing Blocks Pauser/Admin from Calling `pause()` and Safety Functions When Sequencer is Down — (`contracts/pools/RSETHPoolV2ExternalBridge.sol`)

---

### Summary

The LRT-rsETH L2 pool contracts deployed on Arbitrum (`RSETHPoolV2`, `RSETHPoolV2ExternalBridge`, `RSETHPoolV3`) use OpenZeppelin `AccessControl` with `msg.sender`-based role checks. When the Arbitrum sequencer is down, state changes can still be submitted via the Delayed Inbox. However, Arbitrum aliases the `msg.sender` of any L1 **contract** (e.g., a multisig) calling into L2 via the Delayed Inbox:

```
L2_Alias = L1_Contract_Address + 0x1111000000000000000000000000000000001111
```

Because the aliased address does not hold `PAUSER_ROLE`, `DEFAULT_ADMIN_ROLE`, `BRIDGER_ROLE`, or `OPERATOR_ROLE`, every privileged safety function — including `pause()`, `unpause()`, `bridgeAssets()`, and `swapAssetToPremintedRsETH()` — is completely blocked for the protocol's multisig operators during sequencer downtime.

---

When the Arbitrum sequencer is down, users can still submit deposits via the Delayed Inbox (the `deposit()` function has no role restriction, only `whenNotPaused`): [7](#0-6) 

If the protocol's multisig (a contract on L1) attempts to call `pause()` via the Delayed Inbox, the `msg.sender` on L2 becomes the aliased address, which holds no role. The `onlyRole(PAUSER_ROLE)` check reverts. The same applies to `unpause()` (`DEFAULT_ADMIN_ROLE`), `bridgeAssets()` (`BRIDGER_ROLE`), and `swapAssetToPremintedRsETH()` (`OPERATOR_ROLE`): [8](#0-7) [9](#0-8) 

---

### Impact Explanation

During sequencer downtime, the `rsETHOracle` rate used by the pool becomes stale (it cannot be updated). Users can still deposit ETH via the Delayed Inbox at the stale rate, minting wrsETH at an incorrect exchange rate. The pauser/admin multisig cannot call `pause()` to halt deposits because the aliased `msg.sender` fails the role check. This results in:

- **Temporary freezing of safety controls**: The protocol cannot be paused in an emergency during sequencer downtime.
- **Incorrect minting**: Users depositing via the Delayed Inbox receive wrsETH at a stale rate, potentially over- or under-minting relative to the true rsETH/ETH rate.
- **Bridger blocked**: `bridgeAssets()` cannot be called to move accumulated ETH to L1, leaving funds stranded in the L2 pool during the outage.

This maps to **Medium — Temporary freezing of funds** and **Low — Contract fails to deliver promised returns**.

---

### Likelihood Explanation

- Arbitrum sequencer downtime is rare but has occurred historically.
- Protocol multisigs are contracts (not EOAs), so aliasing applies to them. EOAs are not aliased, but protocol admins are virtually always multisigs.
- The window of risk is the entire duration of sequencer downtime, during which users can still interact via the Delayed Inbox.

---

### Recommendation

1. **Grant aliased roles at initialization**: Pre-compute the Arbitrum alias of each privileged address (`alias = address + 0x1111000000000000000000000000000000001111`) and grant it the same roles. This allows the aliased sender to pass role checks when calling via the Delayed Inbox.

2. **Alternatively**, implement a dedicated emergency pause path that accepts calls from the aliased address of the pauser, using `ArbSys.wasMyCallersAddressAliased()` and `ArbSys.myCallersAddressWithoutAliasing()` to recover the true L1 sender before checking roles — following the pattern in OpenZeppelin's `LibArbitrumL2`: [1](#0-0) 

---

### Proof of Concept

1. Deploy `RSETHPoolV2ExternalBridge` on Arbitrum. Grant `PAUSER_ROLE` to a Gnosis Safe multisig at L1 address `0xABCD...`.
2. The Arbitrum alias of this multisig is `0xABCD... + 0x1111000000000000000000000000000000001111`.
3. The Arbitrum sequencer goes down. A security incident begins (e.g., oracle rate becomes stale).
4. Users submit `deposit()` calls via the Delayed Inbox at the stale rate, minting wrsETH at incorrect amounts.
5. The multisig attempts to call `pause()` via the Delayed Inbox from L1.
6. On L2, `msg.sender` = aliased address. `hasRole(PAUSER_ROLE, aliasedAddress)` returns `false`.
7. The call reverts with `ContractPaused` check never reached — the contract cannot be paused.
8. Deposits continue at the stale rate for the entire duration of the outage.
