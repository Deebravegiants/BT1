### Title
Unbounded `rateReceivers` array in `MultiChainRateProvider.updateRate()` causes gas exhaustion — (`contracts/cross-chain/MultiChainRateProvider.sol`)

### Summary
`addRateReceiver()` appends to `rateReceivers` without any length cap. `updateRate()` iterates the entire array in a single transaction, making two external calls per entry. As the protocol expands to more chains, the transaction will eventually exceed the block gas limit, permanently breaking rate propagation to all receivers.

### Finding Description
`addRateReceiver()` is `onlyOwner` and unconditionally pushes a new `RateReceiver` struct: [1](#0-0) 

`updateRate()` then iterates every entry, calling `estimateFees` and `send` on the LayerZero endpoint for each: [2](#0-1) 

Each loop iteration performs two external calls. With N receivers, that is 2N external calls in one transaction. There is no alternative single-receiver dispatch path — `updateRate()` is the only function that pushes rates to destination chains. [3](#0-2) 

### Impact Explanation
Once `rateReceivers.length` grows large enough that the cumulative gas of 2N external calls exceeds the block gas limit (~30M on Ethereum), every call to `updateRate()` reverts. No rate update can be dispatched to **any** receiver. All destination-chain rate receivers become permanently stale until the owner manually removes entries via `removeRateReceiver()`. During that window, any DeFi protocol on a destination chain relying on the rsETH/agETH rate (collateral pricing, AMM weights, etc.) operates on a stale oracle, which can freeze user positions or cause mispricing.

### Likelihood Explanation
The protocol is already deployed on 15+ chains (Arbitrum, Optimism, Polygon zkEVM, Blast, Mode, Scroll, Base, Linea, X Layer, Zircuit, zkSync, Unichain, TAC, Avalanche, Sonic, Ink, Plasma, MegaETH, Mantle, Stable — see README). [4](#0-3) 

Each new chain integration requires one more `addRateReceiver()` call. The owner acts in good faith; no compromise is required. The threshold at which gas exhaustion occurs depends on LayerZero endpoint gas costs per call, but with 2 external calls per receiver and 15+ already registered, the margin is finite and shrinks with every new chain.

### Recommendation
- Add a maximum cap constant (e.g., `uint256 public constant MAX_RECEIVERS = 30`) and enforce it in `addRateReceiver()`.
- Alternatively, add a `updateRateForReceiver(uint256 index)` function that dispatches to a single receiver, so the owner can split updates across multiple transactions when the array is large.
- Consider batching with a start/end index parameter in `updateRate(uint256 from, uint256 to)`.

```solidity
// Invariant fuzz test (Foundry)
function testGasExhaustion(uint8 N) public {
    vm.assume(N > 10 && N < 200);
    for (uint256 i = 0; i < N; i++) {
        vm.prank(owner);
        provider.addRateReceiver(uint16(i + 1), address(uint160(i + 1)));
    }
    uint256 gasBefore = gasleft();
    vm.prank(caller);
    provider.updateRate{value: N * 0.01 ether}();
    uint256 gasUsed = gasBefore - gasleft();
    // Assert gas used grows linearly with N and eventually exceeds block limit
    assertLt(gasUsed, 30_000_000, "Exceeded block gas limit");
}
```

The test will fail (revert OOG) once N crosses the threshold determined by the LayerZero endpoint's per-call cost, demonstrating the invariant break.

```text
    function updateRate() external payable nonReentrant {
        uint256 latestRate = getLatestRate();

        rate = latestRate;

        lastUpdated = block.timestamp;

        bytes memory _payload = abi.encode(latestRate);

        uint256 rateReceiversLength = rateReceivers.length;

        for (uint256 i; i < rateReceiversLength;) {
            uint16 dstChainId = uint16(rateReceivers[i]._chainId);

            bytes memory remoteAndLocalAddresses = abi.encodePacked(rateReceivers[i]._contract, address(this));

            (uint256 estimatedFee,) = ILayerZeroEndpoint(layerZeroEndpoint)
                .estimateFees(dstChainId, address(this), _payload, false, bytes(""));

            ILayerZeroEndpoint(layerZeroEndpoint).send{ value: estimatedFee }(
                dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
            );

            unchecked {
                ++i;
            }
        }

        emit RateUpdated(rate);
    }
```
