Looking at the exact code paths:

### Title
Uncollectable Aave Yield Permanently Frozen When `withdrawETH` Reverts — (`contracts/LRTWithdrawalManager.sol`)

---

### Summary

Every path that disables Aave integration or performs an emergency withdrawal first calls `_collectInterestToTreasury()`, which makes an unguarded external call to `aaveWETHGateway.withdrawETH`. If that call reverts (e.g., Aave pool is paused, or WETH liquidity is exhausted), all three disable/recovery paths revert atomically, leaving `isAaveIntegrationEnabled = true` and the accrued interest permanently locked in the Aave pool with no reachable recovery path.

---

### Finding Description

`_collectInterestToTreasury()` performs a bare external call with no try/catch: [1](#0-0) 

This function is called unconditionally (without error handling) from all three operator-accessible paths that could disable the integration or recover funds:

**Path 1 — `setAaveIntegrationEnabled(false)`** (LRTManager role): [2](#0-1) 

**Path 2 — `emergencyWithdrawFromAave`** (PAUSER_ROLE): [3](#0-2) 

**Path 3 — `configureAaveIntegration`** (LRTManager role, reconfiguration branch): [4](#0-3) 

In all three cases, if `aaveWETHGateway.withdrawETH` reverts, the entire transaction reverts. `isAaveIntegrationEnabled` is never set to `false` (it is only written at line 503, after the revert point), and the accrued interest remains locked in Aave with no reachable path to recover it. [5](#0-4) 

---

### Impact Explanation

The accrued Aave yield (the `aaveBalance - totalETHDepositedToAave` surplus) is permanently inaccessible:

- `collectInterestToTreasury()` (operator path) also calls `_collectInterestToTreasury()` and reverts for the same reason.
- There is no function that sets `isAaveIntegrationEnabled = false` directly, bypasses interest collection, or withdraws only principal without first collecting interest.
- The yield cannot be recovered until Aave's `withdrawETH` succeeds again — which may never happen if the pool is permanently broken or the WETH reserve is durably illiquid.

**Impact: Medium — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

Aave v3 pools have a guardian-controlled pause mechanism that has been exercised on mainnet (e.g., during the March 2023 Euler exploit response). When a pool is paused, all `withdraw` calls revert. Additionally, if the WETH reserve utilization reaches 100%, `withdrawETH` reverts due to insufficient liquidity. Both conditions are realistic and have historical precedent. The vulnerability requires no attacker — it is triggered by normal Aave operational states.

---

### Recommendation

Wrap the `_collectInterestToTreasury()` call in a try/catch (or use a low-level call) inside `setAaveIntegrationEnabled(false)` and `emergencyWithdrawFromAave`, so that a failure to collect interest does not block disabling the integration. Emit an event on failure so the uncollected interest amount is observable off-chain. For example:

```solidity
// In setAaveIntegrationEnabled(false) and emergencyWithdrawFromAave:
try this._collectInterestToTreasuryExternal() { }
catch (bytes memory reason) {
    emit InterestCollectionFailed(reason);
    // Continue — disabling integration must not be blocked by interest collection
}
```

Alternatively, add a separate admin function that sets `isAaveIntegrationEnabled = false` directly without touching Aave, for use when the pool is in a broken state.

    vm.prank(lrtManager);

The root cause is the absence of error handling around `aaveWETHGateway.withdrawETH` at line 954 of `contracts/LRTWithdrawalManager.sol`, combined with the architectural requirement that interest collection must succeed before integration can be disabled. [6](#0-5)

    function _collectInterestToTreasury() internal returns (uint256 interestAmount) {
        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        uint256 principal = totalETHDepositedToAave;

        // Return 0 if no interest or balance is less than principal (accounting for rounding)
        if (aaveBalance <= principal) return 0;

        interestAmount = aaveBalance - principal;

        aaveWETHGateway.withdrawETH(aavePool, interestAmount, address(this));

        address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        (bool sent,) = payable(treasury).call{ value: interestAmount }("");
        if (!sent) revert TreasuryTransferFailed();

        emit InterestCollectedToTreasury(interestAmount, treasury);
    }
