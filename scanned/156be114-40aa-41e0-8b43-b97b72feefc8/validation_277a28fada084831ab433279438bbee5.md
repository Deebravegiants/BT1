### Title
Stale `rsETHPrice` After Early-Return Pause Path Causes Protocol Fee Under-Collection - (File: contracts/LRTOracle.sol)

### Summary
In `LRTOracle._updateRsETHPrice()`, when the downside-protection branch triggers an emergency pause, the function calls `_pause()` and immediately `return`s **without writing the newly computed `newRsETHPrice` back to the `rsETHPrice` state variable**. On every subsequent price update the fee baseline (`previousTVL`) is computed from this stale, artificially-high price, causing the protocol to collect far less performance fee than it is entitled to — or to collect none at all.

### Finding Description

`_updateRsETHPrice()` computes a new price and, if the price has fallen beyond `pricePercentageLimit` relative to `highestRsethPrice`, executes the following early-exit path:

```solidity
// contracts/LRTOracle.sol  lines 270-282
if (newRsETHPrice < highestRsethPrice) {
    ...
    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;          // ← exits WITHOUT executing `rsETHPrice = newRsETHPrice`
    }
    ...
}
```

The assignment `rsETHPrice = newRsETHPrice` at line 313 is never reached. `rsETHPrice` therefore retains the pre-drop value.

The protocol fee is computed on every subsequent call as:

```solidity
// lines 233-246
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);   // uses stale high price
...
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
```

Because `rsETHPrice` is stale-high, `previousTVL` is inflated. Any real yield earned during and after the pause period is measured against this inflated baseline, so the computed `rewardAmount` is smaller than the true gain — or the condition `totalETHInProtocol > previousTVL` is never satisfied at all, and **zero fees are collected** even though the TVL genuinely grew.

### Impact Explanation

The protocol treasury (`PROTOCOL_TREASURY`) receives fewer rsETH fee tokens than it is entitled to. In the worst case (TVL does not recover above the stale price), **all yield earned after the pause is permanently exempt from the protocol fee**. This is a direct, quantifiable loss of unclaimed yield for the protocol.

Impact: **High — theft of unclaimed yield.**

### Likelihood Explanation

`updateRSETHPrice()` is a `public` function callable by any address. The downside-protection branch fires whenever a single price update shows a decline exceeding `pricePercentageLimit` relative to `highestRsethPrice`. This is a realistic scenario during any significant market drawdown or EigenLayer slashing event. No privileged access is required to trigger the root cause; the stale state persists automatically until the next successful price update that clears the condition.

### Recommendation

Update `rsETHPrice` to `newRsETHPrice` **before** calling `_pause()` and returning, so the baseline for future fee calculations always reflects the actual price at the time of the pause:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    rsETHPrice = newRsETHPrice;   // ← record the actual price
    _pause();
    return;
}
```

### Proof of Concept

1. Initial state: `rsETHPrice = 1.10 ETH`, `highestRsethPrice = 1.10 ETH`, `rsethSupply = 1 000 rsETH`, `pricePercentageLimit = 0.05e18` (5 %).
2. A slashing event reduces TVL to 900 ETH. `newRsETHPrice = 0.90 ETH` — a 18 % drop, exceeding the 5 % limit.
3. `_updateRsETHPrice()` hits the early-return path: pauses the protocol and returns. `rsETHPrice` stays at `1.10 ETH`.
4. Admin unpauses. TVL recovers to 1 050 ETH through staking rewards.
5. `updateRSETHPriceAsManager()` is called. `previousTVL = 1 000 × 1.10 = 1 100 ETH`. Since `1 050 < 1 100`, the condition `totalETHInProtocol > previousTVL` is **false** — **zero fees are charged**.
6. The actual gain since the pause was `1 050 − 900 = 150 ETH`. At a 10 % protocol fee that is 15 ETH of fees the treasury never receives.
7. Even if TVL recovers to 1 200 ETH, fees are charged only on `1 200 − 1 100 = 100 ETH` instead of the true gain of `1 200 − 900 = 300 ETH`, permanently under-collecting fees on the 200 ETH earned during the pause. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
