### Title
Accumulated Rounding Error in Fee-Mint Causes Persistent rsETHPrice Drift - (File: contracts/LRTOracle.sol)

### Summary
In `LRTOracle._updateRsETHPrice()`, the protocol fee is computed and minted as rsETH shares using two sequential integer-division (`divWad`) operations. Due to truncation in both divisions, the stored `rsETHPrice` after each update is not exactly consistent with `totalETHInProtocol / newTotalSupply`. This small per-call discrepancy accumulates across every `updateRSETHPrice()` invocation, causing the protocol's internal accounting of TVL to drift from the true value, which in turn causes every subsequent fee calculation to be slightly biased.

### Finding Description

In `_updateRsETHPrice()`:

```solidity
// Line 234
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

// Line 250
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

// Line 301
uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);

// Line 306
IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);

// Line 313
rsETHPrice = newRsETHPrice;
```

Mathematically, if division were exact:

```
newRsETHPrice = (T - F) / S
feeShares     = F / newRsETHPrice = F * S / (T - F)
newSupply     = S + feeShares = S * T / (T - F)
truePrice     = T / newSupply = (T - F) / S  ✓  (equals newRsETHPrice)
```

With integer truncation (`divWad` = `floor(a * 1e18 / b)`):

- `newRsETHPrice = floor((T - F) * 1e18 / S)` — truncates down by up to 1 unit
- `rsethAmountToMintAsProtocolFee = floor(F * 1e18 / newRsETHPrice)` — because `newRsETHPrice` is slightly low, this rounds up slightly

After minting, `newTotalSupply * newRsETHPrice / 1e18 ≠ T`. The stored `rsETHPrice` is slightly inconsistent with the actual supply and TVL. On the next call, `previousTVL = rsethSupply.mulWad(rsETHPrice)` uses this stale price, causing `rewardAmount` and therefore `protocolFeeInETH` to be computed from a slightly incorrect baseline. The error compounds with every call to `updateRSETHPrice()`. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation

The accumulated drift in `rsETHPrice` causes:

1. **Deposit mis-accounting**: `getRsETHAmountToMint` uses `lrtOracle.rsETHPrice()` as the denominator. A slightly incorrect price means depositors receive marginally more or fewer rsETH shares than they are entitled to, diluting or under-serving existing holders. [4](#0-3) 

2. **Withdrawal mis-accounting**: `getExpectedAssetAmount` uses `lrtOracle.rsETHPrice()` as the numerator. A drifted price means withdrawers receive marginally more or fewer underlying assets than they are entitled to. [5](#0-4) 

3. **Fee over/under-charging**: Each subsequent fee calculation uses `previousTVL = rsethSupply.mulWad(rsETHPrice)`. Because `rsETHPrice` is systematically slightly low (floor division), `previousTVL` is slightly underestimated, `rewardAmount` is slightly overestimated, and the protocol takes slightly more fee than it should. [6](#0-5) 

The magnitude per call is at most a few wei, but it accumulates monotonically with every price update. This matches **Low — Contract fails to deliver promised returns, but doesn't lose value**.

### Likelihood Explanation

`updateRSETHPrice()` is a public, permissionless function callable by anyone when the oracle is not paused. [7](#0-6)  It is expected to be called frequently (e.g., by keepers or any user). Every call that results in a non-zero fee mint introduces a rounding residual. Likelihood is **High** that the error accumulates over the protocol's lifetime.

### Recommendation

After computing `rsethAmountToMintAsProtocolFee`, recalculate `rsETHPrice` using the **post-mint** total supply so the stored price is exactly consistent with the new supply and TVL:

```solidity
uint256 newSupply = rsethSupply + rsethAmountToMintAsProtocolFee;
rsETHPrice = totalETHInProtocol.divWad(newSupply);
```

This ensures `newSupply * rsETHPrice / 1e18 ≈ totalETHInProtocol` and prevents the systematic drift from compounding across updates.

### Proof of Concept

Concrete numeric example (simplified, 1e18 = 1 ETH):

```
rsethSupply          = 1_000_000e18  (1M rsETH)
rsETHPrice           = 1.05e18       (1.05 ETH/rsETH)
totalETHInProtocol   = 1_060_000e18  (1.06M ETH, i.e. 10k ETH reward)
protocolFeeInBPS     = 1000          (10%)
protocolFeeInETH     = 1_000e18      (10% of 10k ETH reward)

newRsETHPrice = floor((1_060_000e18 - 1_000e18) * 1e18 / 1_000_000e18)
              = floor(1_059_000e18 * 1e18 / 1_000_000e18)
              = 1_059_000_000_000_000_000  (= 1.059 ETH, exact here)

feeShares = floor(1_000e18 * 1e18 / 1_059_000_000_000_000_000)
          = floor(1e39 / 1.059e18)
          = 944_287_063_267_...  (truncated)

True feeShares (exact) = 1_000e18 / 1.059e18 = 944_287_063_267_...xxx

After mint:
  newSupply = 1_000_000e18 + feeShares
  Implied price = 1_060_000e18 / newSupply
               ≠ newRsETHPrice  (differs by rounding residual)

On next call:
  previousTVL = newSupply * newRsETHPrice / 1e18
              < 1_060_000e18  (underestimates TVL by rounding residual)
  → rewardAmount is overestimated → protocol takes excess fee
``` [8](#0-7)

    function _updateRsETHPrice() internal {
        address rsETHTokenAddress = lrtConfig.rsETH();
        uint256 rsethSupply = IRSETH(rsETHTokenAddress).totalSupply();

        if (rsethSupply == 0) {
            rsETHPrice = 1 ether;
            highestRsethPrice = 1 ether;
            return;
        }

        if (highestRsethPrice == 0) {
            highestRsethPrice = rsETHPrice;
        }

        uint256 previousPrice = rsETHPrice;

        // get total ETH in the protocol (normalized to 1e18)
        uint256 totalETHInProtocol = _getTotalEthInProtocol();

        // calculate previousTVL using rsethSupply multiplied by rsETHPrice
        uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

        IPausable lrtDepositPool = IPausable(lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL));
        IPausable withdrawalManager = IPausable(lrtConfig.getContract(LRTConstants.LRT_WITHDRAW_MANAGER));

        // determine if the protocol is active (not paused)
        bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;

        // only take fee if TVL increased and protocol is not paused
        uint256 protocolFeeInETH = 0;
        if (!protocolPaused && totalETHInProtocol > previousTVL) {
            uint256 rewardAmount = totalETHInProtocol - previousTVL;
            protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
        }

        // compute new rsETH price based on total ETH minus fee
        uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);

        if (newRsETHPrice > highestRsethPrice) {
            // check if the price is above the threshold
            uint256 priceDifference = newRsETHPrice - highestRsethPrice;
            // pricePercentageLimit is in 1e18 precision (100% = 1e18, 1% = 1e16)
            bool isPriceIncreaseOffLimit =
                pricePercentageLimit > 0 && priceDifference > pricePercentageLimit.mulWad(highestRsethPrice);

            // check if the price difference is above the threshold
            if (isPriceIncreaseOffLimit) {
                // if sender has a manager role, this doesn't revert.
                // if not, it reverts as price went above the threshold
                if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) {
                    revert PriceAboveDailyThreshold();
                }
            }
        }

        // downside protection — pause if price drops too far
        if (newRsETHPrice < highestRsethPrice) {
            uint256 diff = highestRsethPrice - newRsETHPrice;
            // normalizing to 1e18
            bool isPriceDecreaseOffLimit =
                pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);

            // if price decrease is off limit, pause the protocol (unless it's already paused)
            if (isPriceDecreaseOffLimit) {
                if (!lrtDepositPool.paused()) lrtDepositPool.pause();
                if (!withdrawalManager.paused()) withdrawalManager.pause();
                _pause();
                return;
            }

            // if price has decreased compared to the previous price, emit an event to reflect that
            if (previousPrice > newRsETHPrice) {
                emit RsETHPriceDecrease(newRsETHPrice, previousPrice);
            }

            // emit an event to notify that the price is currently below the peak (all time high) price
            emit RsETHPriceBelowPeak(highestRsethPrice, newRsETHPrice);
        }

        // update highest price if new price exceeds it
        if (newRsETHPrice > highestRsethPrice) {
            highestRsethPrice = newRsETHPrice;
        }

        // mint protocol fee as rsETH if there's a fee to take
        if (protocolFeeInETH > 0) {
            // Calculate rsETH amount to mint as protocol fee
            uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);

            _checkAndUpdateDailyFeeMintLimit(rsethAmountToMintAsProtocolFee);
            if (rsethAmountToMintAsProtocolFee > 0) {
                address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
                IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
                emit FeeMinted(treasury, rsethAmountToMintAsProtocolFee);
            }
        } else {
            _checkAndUpdateDailyFeeMintLimit(0);
        }

        rsETHPrice = newRsETHPrice;

        emit RsETHPriceUpdate(rsETHPrice, previousPrice);
    }
```
