### Title
Hardcoded `DEFAULT_GAS_LIMIT` in Messenger Contracts Prevents ETH Bridging if Bridge Protocol Requirements Change - (File: `contracts/bridges/BaseMessenger.sol`, `contracts/bridges/OptimismMessenger.sol`, `contracts/bridges/UnichainMessenger.sol`)

### Summary
`BaseMessenger`, `OptimismMessenger`, and `UnichainMessenger` each declare `DEFAULT_GAS_LIMIT = 200_000` as an immutable `constant`. This value is passed directly as the `_minGasLimit` argument to the respective L2 bridge's `bridgeETHTo` call. If the underlying bridge protocol raises its minimum accepted gas limit above 200,000, every call to `sendETHToL1ViaBridge` will revert, permanently blocking ETH from being bridged from L2 to L1 until the contracts are redeployed.

### Finding Description
All three messenger contracts define the gas limit as a compile-time constant:

```solidity
uint32 public constant DEFAULT_GAS_LIMIT = 200_000;
```

This value is then passed unconditionally to the bridge:

```solidity
IBaseMessenger(l2bridge).bridgeETHTo{ value: value }(target, DEFAULT_GAS_LIMIT, bytes(""));
// same pattern in OptimismMessenger and UnichainMessenger
```

Because the value is `constant`, there is no setter, no admin function, and no upgrade path to change it short of redeploying the entire contract. The OP-stack bridge (shared by Base, Optimism, and Unichain) validates the supplied `_minGasLimit` against protocol-level constraints. If those constraints change — for example, if the bridge enforces a new floor higher than 200,000 — every invocation of `sendETHToL1ViaBridge` will revert at the bridge layer.

This is structurally identical to the reported GMX analog: a configurable external parameter can be set to a value that conflicts with a hardcoded constant in the calling contract, causing all bridging operations to revert. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

### Impact Explanation
If the bridge protocol raises its minimum gas limit requirement above 200,000, `sendETHToL1ViaBridge` reverts on every call. ETH collected on L2 (Base, Optimism, Unichain) cannot be bridged back to L1. The funds are not lost — they remain in the calling contract — but they are inaccessible on L1 until the messenger contracts are redeployed with a corrected value. This constitutes **temporary freezing of funds** (Medium severity).

### Likelihood Explanation
The OP-stack bridge is an actively maintained external protocol. Its operators can and do adjust protocol parameters. The 200,000 gas figure is a reasonable current estimate, but it is not guaranteed to remain valid indefinitely. No on-chain mechanism exists to detect or react to a change in the bridge's requirements. The likelihood is low-to-medium but non-negligible given the long operational lifetime expected of these contracts.

### Recommendation
Replace the `constant` with a mutable state variable and add an admin-controlled setter, mirroring the pattern already used for `ccipGasLimit` in `L1VaultV2`:

```solidity
// Before (in BaseMessenger, OptimismMessenger, UnichainMessenger)
uint32 public constant DEFAULT_GAS_LIMIT = 200_000;

// After
uint32 public defaultGasLimit = 200_000;

function setDefaultGasLimit(uint32 _gasLimit) external onlyOwner {
    if (_gasLimit == 0) revert InvalidGasLimit();
    defaultGasLimit = _gasLimit;
    emit DefaultGasLimitSet(_gasLimit);
}
``` [7](#0-6) [8](#0-7) 

### Proof of Concept
1. Deploy `BaseMessenger` (or `OptimismMessenger` / `UnichainMessenger`) — `DEFAULT_GAS_LIMIT` is baked in at 200,000.
2. The Base/OP/Unichain bridge team upgrades the bridge contract to enforce `_minGasLimit >= 300_000`.
3. Any caller (pool contract, user, keeper) invokes `sendETHToL1ViaBridge(l2bridge, target, value)`.
4. The call reaches `IBaseMessenger(l2bridge).bridgeETHTo{value: value}(target, 200_000, "")`.
5. The bridge reverts because `200_000 < 300_000`.
6. ETH remains locked on L2 with no on-chain mechanism to update the gas limit or retry with a higher value. [9](#0-8)

    function sendETHToL1ViaBridge(address l2bridge, address target, uint256 value) external payable nonReentrant {
