### Title
Manager-callable `addPubkeys` can permanently block `stake32Eth` for any validator pubkey - (File: `contracts/PubkeyRegistry.sol`)

---

### Summary

`PubkeyRegistry.addPubkeys()` is callable by `onlyLRTManager`, while the normal staking path registers pubkeys exclusively through `addPubkey()` via `onlyLRTNodeDelegator`. If the manager accidentally pre-registers a pubkey that has not yet been staked, `NodeDelegator.stake32Eth()` will permanently revert for that pubkey with `PubkeyAlreadyRegistered`. There is no unregister path, making the block irreversible.

---

### Finding Description

`PubkeyRegistry` exposes two distinct registration paths:

**Path 1 — normal staking flow** (`onlyLRTNodeDelegator`): [1](#0-0) 

**Path 2 — manager bulk registration** (`onlyLRTManager`): [2](#0-1) 

`NodeDelegator.stake32Eth()` enforces a one-time-use guard before staking 32 ETH: [3](#0-2) 

The registry mapping is write-once (`true` only) with no deletion function: [4](#0-3) 

If the manager calls `addPubkeys([pubkey])` for a validator pubkey that has **not yet been staked**, the pubkey is permanently marked as used. Any subsequent call to `stake32Eth` with that pubkey hits the `hasPubkey` check and reverts with `PubkeyAlreadyRegistered`. Because there is no `removePubkey` or reset function anywhere in `PubkeyRegistry`, the block is irreversible.

This is structurally identical to the reported pattern: a second, broader-privileged entry point (`addPubkeys` / `onlyLRTManager`) can write the same one-time state that the canonical entry point (`addPubkey` / `onlyLRTNodeDelegator`) is supposed to own, and the canonical flow has no recovery path once that state is set.

---

### Impact Explanation

**Low — Contract fails to deliver promised returns, but doesn't lose value.**

The affected validator pubkey can never be used to stake 32 ETH through `stake32Eth`. The ETH already held in the `NodeDelegator` is not lost, but the intended validator activation is permanently blocked for that pubkey. The operator must source a different, unused validator key; the originally intended validator key is permanently poisoned in the registry. [5](#0-4) 

---

### Likelihood Explanation

Low. The manager must accidentally supply a pubkey that has not yet been staked — for example, during a bulk migration of pubkeys across NodeDelegators, or when pre-populating the registry with keys that are still pending activation. The `addPubkeys` function performs no staleness or staking-status check, making the mistake easy to commit silently. [2](#0-1) 

---

### Recommendation

Restrict `addPubkeys` to `onlyLRTNodeDelegator` so that only the staking contract can register pubkeys, mirroring the single-caller pattern of `addPubkey`. If a manager-accessible bulk path is genuinely needed (e.g., for migration), add an explicit check that the pubkey has not already been registered and document the invariant that only already-staked pubkeys may be passed:

```solidity
function addPubkeys(bytes[] calldata pubkeys) external onlyLRTManager {
    for (uint256 i = 0; i < pubkeys.length; i++) {
        bytes32 hashed = keccak256(pubkeys[i]);
        require(!pubkeyRegistry[hashed], "PubkeyRegistry: already registered");
        pubkeyRegistry[hashed] = true;
    }
}
```

Even with the duplicate check, the root fix is to ensure `addPubkeys` is only ever called for pubkeys whose 32 ETH has already been deposited on-chain.

---

### Proof of Concept

1. Manager holds `MANAGER` role and calls `PubkeyRegistry.addPubkeys([validatorPubkey])` for a validator whose 32 ETH has not yet been staked.
2. `pubkeyRegistry[keccak256(validatorPubkey)]` is set to `true`.
3. Operator later calls `NodeDelegator.stake32Eth(validatorPubkey, sig, depositDataRoot)`.
4. `pubkeyRegistry.hasPubkey(validatorPubkey)` returns `true`.
5. `stake32Eth` reverts with `PubkeyAlreadyRegistered`.
6. No unregister function exists; the validator pubkey is permanently blocked from the staking flow. [6](#0-5) [7](#0-6)

        bytes calldata pubkey,

    )
        public
        whenNotPaused
        onlyLRTOperator
    {
        IPubkeyRegistry pubkeyRegistry = IPubkeyRegistry(lrtConfig.pubkeyRegistry());
        if (pubkeyRegistry.hasPubkey(pubkey)) {
            revert PubkeyAlreadyRegistered();
        }
        pubkeyRegistry.addPubkey(pubkey);

        // tracks staked but unverified native ETH
        stakedButUnverifiedNativeETH += 32 ether;

        _getEigenPodManager().stake{ value: 32 ether }(pubkey, signature, depositDataRoot);

        if (address(eigenPod) == address(0)) {
            eigenPod = _getEigenPodManager().ownerToPod(address(this));
            emit EigenPodCreated(address(eigenPod), address(this));
        }
        emit ETHStaked(pubkey, 32 ether);
    }
