### Title
`whenNotPaused` on `completeWithdrawal()` Freezes In-Flight User rsETH With No Cancellation Path — (`File: contracts/LRTWithdrawalManager.sol`)

---

### Summary

`LRTWithdrawalManager` implements a two-step withdrawal process. In step 1, `initiateWithdrawal()` pulls rsETH from the user into the contract. In step 2, `completeWithdrawal()` releases the underlying asset back to the user. Both steps carry `whenNotPaused`. There is no cancellation or refund function. If the contract is paused after step 1 but before step 2, the user's rsETH is permanently held by the contract with no recovery path.

---

### Finding Description

`initiateWithdrawal()` irrevocably transfers the caller's rsETH into the contract at line 166: [1](#0-0) 

The only functions that can release value back to the user are `completeWithdrawal()` and `completeWithdrawalForUser()`, both gated by `whenNotPaused`: [2](#0-1) [3](#0-2) 

A grep across all production contracts confirms there is no `cancelWithdrawal`, `refund`, or equivalent function. Once rsETH is transferred in step 1, the only exit is through `completeWithdrawal`, which is blocked while paused.

The structural parallel to the reported bug is exact:

| | Original Report | LRT-rsETH |
|---|---|---|
| Step 1 (commits value) | `registerRedemption()` burns KPI tokens | `initiateWithdrawal()` transfers rsETH to contract |
| Step 2 (releases value) | `redeemCollateral()` | `completeWithdrawal()` |
| Blocking guard on step 2 | `if (_isExpired()) revert` | `whenNotPaused` |
| Cancellation path | None | None |

---

### Impact Explanation

Any user who has called `initiateWithdrawal()` and whose request has not yet been completed via `completeWithdrawal()` will have their rsETH frozen inside the contract for the entire duration of the pause. Because no cancellation function exists, the freeze persists until the contract is unpaused. If the contract is never unpaused (governance failure, lost admin keys, or a permanent deprecation pause), the freeze becomes permanent. This constitutes at minimum **temporary freezing of funds** (Medium) and at worst **permanent freezing of funds** (Critical).

---

### Likelihood Explanation

The `PAUSER_ROLE` can pause the contract at any time. Pauses are a routine operational response to security incidents. Given that the withdrawal delay is configured to 8 days by default (`withdrawalDelayBlocks = 8 days / 12 seconds`), there is a wide window during which a pause can occur between step 1 and step 2. Any user with a pending withdrawal request is exposed for the full delay period. [4](#0-3) 

---

### Recommendation

Remove `whenNotPaused` from `completeWithdrawal()` and `completeWithdrawalForUser()`. Completing an already-initiated withdrawal (where rsETH is already held by the contract) is a user-protective action that should not be blocked by a pause. The pause is correctly applied to `initiateWithdrawal()` to prevent new commitments during an incident, but it should not trap funds that are already committed. Alternatively, add a `cancelWithdrawal()` function that returns the held rsETH to the user and is callable even when paused.

---

### Proof of Concept

1. User calls `initiateWithdrawal(asset, amount, ref)`. rsETH is transferred from user to contract. Request is enqueued.
2. Operator calls `pause()` (e.g., in response to a security incident).
3. User calls `completeWithdrawal(asset, ref)` after the delay passes. Transaction reverts with `Pausable: paused`.
4. Operator calls `completeWithdrawalForUser(asset, user, ref)`. Also reverts with `Pausable: paused`.
5. User has no other function to call. rsETH remains locked in `LRTWithdrawalManager` indefinitely. [5](#0-4)

    function initiateWithdrawal(
        address asset,
        uint256 rsETHUnstaked,
        string calldata referralId
    )
        external
        override
        nonReentrant
        whenNotPaused
        onlySupportedAsset(asset)
        onlySupportedStrategy(asset)
    {
        if (rsETHUnstaked == 0 || rsETHUnstaked < minRsEthAmountToWithdraw[asset]) {
            revert InvalidAmountToWithdraw();
        }

        IERC20(lrtConfig.rsETH()).safeTransferFrom(msg.sender, address(this), rsETHUnstaked);

        uint256 expectedAssetAmount = getExpectedAssetAmount(asset, rsETHUnstaked);

        if (expectedAssetAmount > getAvailableAssetAmount(asset)) revert ExceedAmountToWithdraw();

        // preventing over-withdrawal.
        assetsCommitted[asset] += expectedAssetAmount;

        _addUserWithdrawalRequest(asset, rsETHUnstaked, expectedAssetAmount);

        emit ReferralIdEmitted(referralId);
    }

    /// @notice Completes a user's withdrawal process by transferring the ETH/LST amount corresponding to the rsETH
    /// unstaked.
    /// @param asset The asset address the user wishes to withdraw.
    function completeWithdrawal(address asset, string calldata referralId) external nonReentrant whenNotPaused {
        _processWithdrawalCompletion(asset, msg.sender, referralId);
    }
