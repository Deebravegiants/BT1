### Title
Zero-ETH Bridge Invocation via Missing Zero-Amount Guard in `bridgeAssetsViaNativeBridge` - (File: `contracts/bridges/UnichainMessenger.sol`, `contracts/pools/RSETHPool.sol`, `contracts/pools/RSETHPoolNoWrapper.sol`)

### Summary

The no-argument `bridgeAssetsViaNativeBridge()` functions in `RSETHPool`, `RSETHPoolNoWrapper`, and `RSETHPoolV3ExternalBridge` pass `getETHBalanceMinusFees()` directly to `UnichainMessenger.sendETHToL1ViaBridge` without checking whether the value is zero. `UnichainMessenger` only validates `msg.value == value`, which trivially passes when both are zero, and then forwards a zero-value call to the Unichain bridge — a no-op that consumes L2 gas without delivering any ETH to L1.

### Finding Description

**Call path:**

1. `BRIDGER_ROLE` calls `RSETHPool.bridgeAssetsViaNativeBridge()` (or `RSETHPoolNoWrapper` / `RSETHPoolV3ExternalBridge` equivalent).
2. `ethBalanceMinusFees = getETHBalanceMinusFees()` returns `address(this).balance - feeEarnedInETH`.
3. When `address(this).balance == feeEarnedInETH`, this is `0`.
4. The pool calls `IL2Messenger(messenger).sendETHToL1ViaBridge{ value: 0 }(l2Bridge, l1VaultETHForL2Chain, 0)`.
5. Inside `UnichainMessenger.sendETHToL1ViaBridge`: the guard `if (msg.value != value) revert MismatchedMsgValue()` evaluates `0 != 0` → **false**, so no revert.
6. `IUnichainMessenger(l2bridge).bridgeETHTo{ value: 0 }(target, DEFAULT_GAS_LIMIT, bytes(""))` is called — a zero-value bridge transaction.

**Root cause:** None of the no-argument `bridgeAssetsViaNativeBridge()` variants check `ethBalanceMinusFees > 0` before invoking the messenger. The `UnichainMessenger` guard only enforces `msg.value == value`, not `value > 0`.

Contrast with `RSETHPoolV2ExternalBridge.bridgeAssetsViaNativeBridge(uint256 amount)`, which explicitly guards: [1](#0-0) 

The no-argument variants in `RSETHPool`, `RSETHPoolNoWrapper`, and `RSETHPoolV3ExternalBridge` have no equivalent guard: [2](#0-1) [3](#0-2) [4](#0-3) 

`getETHBalanceMinusFees()` can legitimately return zero whenever the pool's total ETH balance equals accumulated fees: [5](#0-4) 

`UnichainMessenger.sendETHToL1ViaBridge` does not reject zero-value calls: [6](#0-5) 

### Impact Explanation

A zero-value bridge transaction is submitted to the Unichain bridge, consuming L2 gas (and potentially triggering L1 gas for the cross-chain message relay) without transferring any ETH to the L1 vault. The function emits `BridgedETHToL1ViaNativeBridge` with `amount = 0`, falsely signaling a successful bridge operation. No funds are lost, but the contract fails to deliver its promised behavior. This matches the **Low** scope: *Contract fails to deliver promised returns, but doesn't lose value*.

### Likelihood Explanation

The precondition `address(this).balance == feeEarnedInETH` is a realistic operational state — it occurs whenever the pool has accumulated fees but no unbridged user deposits (e.g., immediately after a prior bridge operation drained user ETH, or before any deposits arrive). The `BRIDGER_ROLE` is a trusted but non-admin operational role that is expected to call this function routinely; no compromise is required.

### Recommendation

Add a zero-amount guard at the start of each no-argument `bridgeAssetsViaNativeBridge()` function, mirroring the pattern already used in `RSETHPoolV2ExternalBridge`:

```solidity
uint256 ethBalanceMinusFees = getETHBalanceMinusFees();
if (ethBalanceMinusFees == 0) revert InvalidAmount();
```

Alternatively, add the guard inside `UnichainMessenger.sendETHToL1ViaBridge`:

```solidity
if (value == 0) revert InvalidAmount();
```

### Proof of Concept

```solidity
// Setup: pool.balance == pool.feeEarnedInETH (e.g., both == 1 ether)
// No user ETH available to bridge.

// BRIDGER_ROLE calls:
pool.bridgeAssetsViaNativeBridge();
// → getETHBalanceMinusFees() == 0
// → UnichainMessenger.sendETHToL1ViaBridge{value: 0}(l2Bridge, l1Vault, 0)
// → msg.value(0) != value(0) → false, no revert
// → l2bridge.bridgeETHTo{value: 0}(l1Vault, 200_000, "")
// → BridgedETHToL1ViaNativeBridge(l1Vault, 0) emitted
// Assert: L1 vault receives 0 ETH; gas consumed on L2 (and L1 relay).
```
