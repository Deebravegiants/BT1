### Title
`getAssetUnstaking` Uses Raw `scaledShares` Instead of Slashing-Adjusted Withdrawable Shares, Overstating TVL and Inflating `rsETH` Price - (File: contracts/NodeDelegator.sol)

### Summary

`NodeDelegator.getAssetUnstaking()` reads `scaledShares` from EigenLayer's withdrawal queue and passes them directly to `strategy.sharesToUnderlyingView()`. These `scaledShares` are an intermediate representation that must be multiplied by the operator's current `maxMagnitude * beaconChainScalingFactor` to obtain actual withdrawable shares. When slashing occurs during the withdrawal delay period, the actual withdrawable amount is less than what `scaledShares` implies, causing `getTotalAssetDeposits` to overstate TVL, inflating the `rsETH` price, and enabling existing holders to redeem at a price that exceeds the protocol's real backing.

### Finding Description

`NodeDelegator.getAssetUnstaking()` queries EigenLayer's `DelegationManager.getQueuedWithdrawals()` and uses the returned `withdrawalShares` directly:

```solidity
uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
amount += strategyAsset == LRTConstants.ETH_TOKEN
    ? sharesToUnstake
    : strategy.sharesToUnderlyingView(sharesToUnstake);
```

The `Withdrawal` struct in EigenLayer's `IDelegationManager` explicitly documents that the stored field is `scaledShares`, not actual withdrawable shares:

> *"Note that these scaledShares need to be multiplied by the operator's maxMagnitude and beaconChainScalingFactor at completion to include slashing occurring during the queue withdrawal delay."*

The relationship is:

```
scaledShares = depositShares / (maxMagnitude_at_queue * beaconChainScalingFactor_at_queue)
actual_withdrawable = scaledShares * maxMagnitude_at_completion * beaconChainScalingFactor_at_completion
```

If slashing occurs after a withdrawal is queued, `maxMagnitude_at_completion < maxMagnitude_at_queue`, so `actual_withdrawable < scaledShares`. The function returns `sharesToUnderlyingView(scaledShares)` — a value larger than the real amount — for every queued withdrawal where post-queue slashing has occurred.

This inflated value propagates through the entire TVL and pricing stack:

1. `getAssetUnstaking` → overstated `assetUnstakingFromEigenLayer`
2. `LRTDepositPool.getTotalAssetDeposits` → overstated total assets
3. `LRTOracle._getTotalEthInProtocol` → overstated ETH backing
4. `LRTOracle.rsETHPrice` → inflated `rsETH` price
5. `LRTWithdrawalManager.getAvailableAssetAmount` → overstated available liquidity

By contrast, the active-strategy path (`NodeDelegatorHelper.getAssetBalance`) correctly uses `DelegationManager.getWithdrawableShares()`, which returns slashing-adjusted shares. The queued-withdrawal path has no equivalent correction.

### Impact Explanation

**Impact: Critical — Protocol insolvency / direct theft of user funds.**

When slashing reduces the actual assets recoverable from the withdrawal queue, the `rsETH` price remains inflated. Any rsETH holder who redeems at this inflated price receives more underlying assets than the protocol will actually recover. Once the withdrawal completes and the real (lower) asset amount arrives, the protocol has paid out more than it received, leaving remaining holders with under-backed rsETH. With sufficient slashing magnitude or multiple concurrent queued withdrawals, the protocol can be rendered insolvent.

`LRTWithdrawalManager.getExpectedAssetAmount` locks in the payout at the time of the withdrawal request using the stored `rsETHPrice`. An attacker who calls `updateRSETHPrice()` (a public function) after slashing occurs but before the withdrawal queue is settled can lock in the inflated rate.

### Likelihood Explanation

**Likelihood: Medium.**

EigenLayer's slashing mechanism is live and operators can be slashed for AVS misbehavior. The withdrawal delay is at least `minWithdrawalDelayBlocks` (currently 7 days on mainnet), providing a window during which slashing can occur after a withdrawal is queued. The Kelp DAO protocol actively delegates to EigenLayer operators and regularly initiates unstaking. The public `updateRSETHPrice()` entry point means no privileged access is required to trigger the price update at the inflated value.

### Recommendation

Replace the raw `scaledShares` read in `getAssetUnstaking` with a slashing-adjusted calculation. The correct withdrawable shares for a queued withdrawal are:

```solidity
actual_shares = scaledShares * currentSlashingFactor
```

where `currentSlashingFactor = maxMagnitude_at_now * beaconChainScalingFactor_at_now`.

Alternatively, mirror the approach already used in `NodeDelegatorHelper.getAssetBalance`, which calls `DelegationManager.getWithdrawableShares()` — a function that already applies the current slashing factor. For queued withdrawals, EigenLayer exposes `getSlashableSharesInQueue` and the `depositScalingFactor` per staker/strategy, which can be used to compute the slashing-adjusted value.

### Proof of Concept

1. NDC is delegated to operator O and holds 1000 stETH in the EigenLayer strategy.
2. Operator calls `initiateUnstaking` — 1000 stETH worth of `depositShares` are queued. EigenLayer stores `scaledShares = depositShares / slashingFactor_at_queue` in the `Withdrawal` struct.
3. During the 7-day delay, operator O is slashed 10% — `maxMagnitude` drops from `1e18` to `0.9e18`.
4. `getAssetUnstaking` reads `scaledShares` and returns `sharesToUnderlyingView(scaledShares)` ≈ 1000 stETH (pre-slash value). The actual recoverable amount is ≈ 900 stETH.
5. Attacker calls `LRTOracle.updateRSETHPrice()` — price is computed using the overstated 1000 stETH, inflating `rsETH` price by ~1.1%.
6. Attacker (existing rsETH holder) calls `LRTWithdrawalManager.initiateWithdrawal`, locking in `expectedAssetAmount` at the inflated price.
7. Withdrawal completes — only 900 stETH arrives. The protocol pays out the locked-in inflated amount, leaving a 100 stETH deficit that is socialized across remaining rsETH holders.

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
