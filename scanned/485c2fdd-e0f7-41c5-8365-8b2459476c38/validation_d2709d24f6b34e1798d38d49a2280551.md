### Title
`swapAssetToPremintedRsETH` Cannot Be Paused — (File: `contracts/pools/RSETHPoolV3.sol`, `contracts/pools/RSETHPoolV3ExternalBridge.sol`, `contracts/pools/RSETHPoolV3WithNativeChainBridge.sol`)

---

### Summary

The `swapAssetToPremintedRsETH` function in all three production pool contracts does not check the `paused` flag, allowing the `OPERATOR_ROLE` to continue draining pool assets (ETH or supported tokens) even when the contract has been paused by the `PAUSER_ROLE`. This directly mirrors the FraxPoolV3 pattern: a fund-moving function bypasses the pause control, reducing the pauser's ability to fully contain an emergency.

---

### Finding Description

All three pool contracts implement a custom pause mechanism via a `bool public paused` storage variable and a `whenNotPaused` modifier:

- `RSETHPoolV3` — [1](#0-0) 
- `RSETHPoolV3ExternalBridge` — [2](#0-1) 
- `RSETHPoolV3WithNativeChainBridge` — [3](#0-2) 

The user-facing `deposit` functions in all three contracts correctly apply `whenNotPaused`:

- `RSETHPoolV3.deposit(string)` — [4](#0-3) 
- `RSETHPoolV3.deposit(address,uint256,string)` — [5](#0-4) 

However, `swapAssetToPremintedRsETH` — which transfers pool-held ETH or ERC-20 tokens out of the contract to the caller — carries **no** `whenNotPaused` check in any of the three contracts:

**RSETHPoolV3:**
```solidity
function swapAssetToPremintedRsETH(
    address rsETH,
    address token,
    uint256 rsETHAmount
)
    external
    nonReentrant
    onlySupportedTokenOrEth(token)
    onlyRole(OPERATOR_ROLE)   // ← no whenNotPaused
{
    ...
    tokenContract.safeTransfer(msg.sender, tokenAmount);  // pool assets leave
}
``` [6](#0-5) 

**RSETHPoolV3ExternalBridge:** [7](#0-6) 

**RSETHPoolV3WithNativeChainBridge:** [8](#0-7) 

The function computes `tokenAmount` using the live oracle rate and then transfers pool assets to the operator. It is the functional inverse of `deposit` and moves real value out of the pool.

---

### Impact Explanation

When the `PAUSER_ROLE` pauses the contract in response to an emergency (e.g., oracle rate anomaly, wrapper exploit, or pool accounting issue), the intent is to halt all fund movements. Because `swapAssetToPremintedRsETH` ignores `paused`, the `OPERATOR_ROLE` can continue to extract ETH or supported tokens from the pool at the current (potentially manipulated or stale) oracle rate. The pauser's emergency response is incomplete: deposits are frozen but reverse-swap outflows are not.

**Impact**: Medium — Temporary freezing of funds is not achievable for the `swapAssetToPremintedRsETH` path. The pauser cannot prevent the operator from draining pool collateral during an active emergency, increasing the potential loss window before the admin can take further action (e.g., revoking the operator role).

---

### Likelihood Explanation

The `OPERATOR_ROLE` must be active and executing swaps during the pause window. In a normal operational scenario the operator would respect the pause. However, if the operator is implemented as a smart contract with a bug (analogous to the FraxPoolV3 AMO contract scenario), or if the operator key is compromised, the pauser has no on-chain mechanism to stop the outflow. The likelihood is **Low** in isolation but the design gap is unconditional — the check is simply absent.

---

### Recommendation

**Short term**: Add the `whenNotPaused` modifier to `swapAssetToPremintedRsETH` in all three pool contracts:

```solidity
function swapAssetToPremintedRsETH(
    address rsETH,
    address token,
    uint256 rsETHAmount
)
    external
    nonReentrant
    whenNotPaused          // ← add this
    onlySupportedTokenOrEth(token)
    onlyRole(OPERATOR_ROLE)
```

**Long term**: Audit all fund-moving functions across every pool variant to confirm that `whenNotPaused` is applied consistently. The three pool contracts share near-identical logic but are maintained separately, making modifier omissions easy to miss during copy-paste upgrades.

---

### Proof of Concept

1. Pool is live; `PAUSER_ROLE` calls `pause()` due to an oracle anomaly or wrapper exploit. `paused` is set to `true`.
2. Any call to `deposit(...)` now reverts with `ContractPaused`.
3. `OPERATOR_ROLE` calls `swapAssetToPremintedRsETH(rsETH, token, rsETHAmount)`.
4. The function executes without reverting — `paused` is never read.
5. `tokenContract.safeTransfer(msg.sender, tokenAmount)` transfers pool tokens to the operator at the current oracle rate.
6. The pauser's emergency action did not prevent this outflow.

```text
    function swapAssetToPremintedRsETH(
        address rsETH,
        address token,
        uint256 rsETHAmount
    )
        external
        nonReentrant
        onlySupportedTokenOrEth(token)
        onlyRole(OPERATOR_ROLE)
    {
        UtilLib.checkNonZeroAddress(rsETH);

        IRsETHTokenWrapper wrapper = IRsETHTokenWrapper(address(wrsETH));
        IERC20 tokenContract = IERC20(token);

        if (!wrapper.allowedTokens(rsETH)) revert TokenNotAllowedInWrapper();
        if (rsETHAmount == 0) revert InvalidAmount();
        if (rsETHAmount > wrapper.maxAmountToDepositBridgerAsset(rsETH)) revert ExceedsMaxAmountToDepositInWrapper();

        // Get the amount of token to transfer to the user for the given amount of rsETH provided
        uint256 tokenAmount = viewSwapAssetToPremintedRsETH(token, rsETHAmount);

        // Transfer rsETH from sender to the wrapper
        IERC20(rsETH).safeTransferFrom(msg.sender, address(wrapper), rsETHAmount);

        // Transfer the token from the pool to the sender
        if (token == ETH_IDENTIFIER) {
            if (getETHBalanceMinusFees() < tokenAmount) revert InsufficientETHBalanceForReverseSwap();
            (bool success,) = payable(msg.sender).call{ value: tokenAmount }("");
            if (!success) revert TransferFailed();
        } else {
            if (getTokenBalanceMinusFees(token) < tokenAmount) revert InsufficientAssetBalanceForReverseSwap();
            tokenContract.safeTransfer(msg.sender, tokenAmount);
        }

        emit ReverseSwapOccurred(msg.sender, rsETH, token, rsETHAmount, tokenAmount);
    }
