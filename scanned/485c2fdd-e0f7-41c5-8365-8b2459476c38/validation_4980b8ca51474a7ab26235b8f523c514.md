### Title
Blacklisted Token Recipient Causes Permanent Freezing of Unclaimed Yield — (`contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

---

### Summary

`MerkleDistributor.claim()` unconditionally transfers yield to the `account` address encoded in the merkle leaf. If the yield token is a blacklist-capable ERC20 (e.g., USDC) and `account` is blacklisted after the merkle root is set, every future `claim()` call for that account will revert at the `safeTransfer` call. The contract contains no rescue, redirect, or recovery function, so the allocated yield is permanently frozen with no on-chain recovery path.

---

### Finding Description

The `claim()` function in `MerkleDistributor.sol` performs the following sequence:

1. Validates the merkle proof against the leaf `keccak256(abi.encodePacked(index, account, cumulativeAmount))`.
2. Updates `userClaims[account]` state.
3. Calls `IERC20(token).safeTransfer(account, amountToSend)`. [1](#0-0) 

The recipient is always `account` — the address baked into the merkle leaf. There is no `msg.sender == account` guard, but that is irrelevant: regardless of who calls `claim()`, the tokens are always sent to `account`. [2](#0-1) 

If `account` is blacklisted by the token contract (USDC, USDT, etc.), `safeTransfer` will revert on every invocation. Because the merkle leaf permanently encodes `account` as the recipient, there is no way to redirect the tokens to a different address. The contract exposes no `rescueTokens`, `emergencyWithdraw`, or alternative-recipient mechanism. [3](#0-2) 

The `setToken()` admin function only changes the token for future distributions; it does not unblock the stuck allocation. [4](#0-3) 

---

### Impact Explanation

The yield allocated to the blacklisted `account` is permanently locked inside the `MerkleDistributor` contract. No admin function can recover or redirect it. This matches the allowed impact: **Medium — Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The `MerkleDistributor` is a generic contract; its `token` field is configurable and can be set to any ERC20, including USDC or USDT.
- USDC/USDT blacklisting is an operational reality (OFAC sanctions, exchange compliance). Any user whose address is sanctioned after a merkle root is published triggers this path.
- The call sequence requires no privileged access by the attacker: `setMerkleRoot` (owner, normal operation) → `token.blacklist(account)` (token issuer, independent action) → `claim(...)` (anyone, public function).
- Likelihood is **low-to-medium** given that blacklisting is rare but not hypothetical.

---

### Recommendation

1. **Add an alternative-recipient parameter**: Allow the claimant (verified via `msg.sender == account`) to specify a different destination address, so a blacklisted account can redirect yield to a clean address.
2. **Add a token rescue function**: An `onlyOwner` `rescueTokens(address to, uint256 amount)` function would allow recovery of stuck tokens as a last resort.
3. **Document token compatibility**: If the distributor is intended only for non-blacklistable tokens, enforce this in `setToken()` or document it clearly.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

// Fork mainnet, deploy MerkleDistributor with USDC as token.
// 1. Owner calls setMerkleRoot(root) — root includes (index=1, account=VICTIM, cumulativeAmount=1000e6)
// 2. USDC blacklister calls USDC.blacklist(VICTIM)
// 3. Anyone calls claim(1, VICTIM, 1000e6, proof)
//    → safeTransfer(VICTIM, ...) reverts with "Blacklistable: account is blacklisted"
// 4. Assert: no rescueTokens() or recovery function exists on MerkleDistributor
//    → 1000e6 USDC permanently stuck in the contract

function testBlacklistedAccountFreezesYield() public {
    // Setup: fork mainnet, get USDC blacklister role
    address victim = address(0xBEEF);
    uint256 amount = 1000e6;

    // Build merkle tree with (index=1, victim, amount)
    bytes32 leaf = keccak256(abi.encodePacked(uint256(1), victim, amount));
    bytes32 root = leaf; // single-leaf tree
    bytes32[] memory proof = new bytes32[](0);

    // Owner sets root
    vm.prank(owner);
    distributor.setMerkleRoot(root);

    // USDC blacklists victim
    vm.prank(usdcBlacklister);
    IFiatToken(USDC).blacklist(victim);

    // Claim reverts
    vm.expectRevert("Blacklistable: account is blacklisted");
    distributor.claim(1, victim, amount, proof);

    // No recovery path: rescueTokens does not exist
    // Tokens remain locked in distributor forever
}
```

    /// @dev Set the merkle root for the given index.
    /// @dev only called by the owner.
    /// @param _merkleRootToSet The merkle root to set.
    function setMerkleRoot(bytes32 _merkleRootToSet) external onlyOwner {
        if (_merkleRootToSet == bytes32(0)) {
            revert ZeroValueProvided();
        }

        currentMerkleRoot = _merkleRootToSet;

        currentMerkleRootIndex++;
        currentIndex++;

        emit MerkleRootSet(currentMerkleRootIndex, currentMerkleRoot);
    }

    /// @dev Set the protocol treasury address.
