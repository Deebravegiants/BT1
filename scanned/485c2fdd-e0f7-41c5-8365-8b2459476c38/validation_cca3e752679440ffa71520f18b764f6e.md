### Title
Dust LST Token Donation Permanently Blocks NodeDelegator Removal from Queue - (`contracts/LRTDepositPool.sol`)

### Summary
An unprivileged attacker holding any supported LST token (stETH, ETHx, sfrxETH) can permanently block the admin's ability to remove a `NodeDelegator` from the queue by donating a small amount of a supported token directly to the `NodeDelegator` contract address. This prevents the protocol from decommissioning NodeDelegators, blocking protocol upgrades, emergency migrations, and the ability to add new NodeDelegators when the `maxNodeDelegatorLimit` is reached.

### Finding Description

`LRTDepositPool.removeNodeDelegatorContractFromQueue` (and its batch variant) calls `_removeNodeDelegatorContractFromQueue`, which internally calls `_checkResidueLSTBalance` before allowing removal: [1](#0-0) 

`_checkResidueLSTBalance` iterates over all supported assets and checks the raw ERC20 balance of the NodeDelegator: [2](#0-1) 

The critical line is:

```solidity
assetBalance = IERC20(supportedAssets[i]).balanceOf(nodeDelegatorAddress)
    + INodeDelegator(nodeDelegatorAddress).getAssetBalance(supportedAssets[i]);
```

`IERC20(supportedAssets[i]).balanceOf(nodeDelegatorAddress)` reads the raw on-chain ERC20 balance of the NodeDelegator. Because ERC20 tokens can be transferred to any address by anyone, an attacker can send even 1 wei of a supported LST directly to the NodeDelegator contract. This causes `assetBalance > maxNegligibleAmount` to revert with `NodeDelegatorHasAssetBalance`.

`maxNegligibleAmount` defaults to `0` (it is never set in `initialize`): [3](#0-2) 

Even if the admin raises `maxNegligibleAmount` via `setMaxNegligibleAmount`, the attacker simply donates `maxNegligibleAmount + 1` wei of the token to re-block removal. The NodeDelegator's `receive()` function also accepts raw ETH, making the same attack possible via `_checkResidueEthBalance`: [4](#0-3) [5](#0-4) 

### Impact Explanation

The admin (`onlyLRTAdmin`) cannot remove a NodeDelegator from the queue. Consequences:

1. **Protocol upgrade blocked**: If the `maxNodeDelegatorLimit` (default 10) is reached and existing NodeDelegators cannot be removed, no new NodeDelegators can be added, capping the protocol's restaking capacity.
2. **Emergency decommissioning blocked**: If a NodeDelegator is compromised or needs to be replaced urgently, the admin cannot remove it from the queue, leaving it active and potentially dangerous.
3. **Fund migration blocked**: Funds staked in EigenLayer through the targeted NodeDelegator cannot be migrated to a replacement NodeDelegator, constituting a temporary freeze of those funds from the protocol's operational perspective.

The admin's only workaround is to first drain the NodeDelegator via `transferBackToLRTDepositPool` (`onlyAssetTransferRole`) and then attempt removal — but the attacker can frontrun the removal step with another dust donation, making the block persistent without atomic drain-and-remove tooling.

### Likelihood Explanation

Any holder of a supported LST (stETH, ETHx, sfrxETH) or any ETH holder can execute this attack. The cost is negligible (1 wei of token + gas). No special permissions are required. The attack can be sustained indefinitely at minimal cost by re-donating dust after each admin drain attempt.

### Recommendation

Replace the raw `IERC20.balanceOf` check with a protocol-tracked internal accounting variable that only increments when tokens are received through sanctioned protocol paths (e.g., `depositAssetIntoStrategy`). Alternatively, add a permissioned `sweepDust` function on the NodeDelegator that allows the asset-transfer role to atomically drain and remove in a single call, or use a multicall pattern to make drain-and-remove atomic and frontrun-resistant.

### Proof of Concept

```
// Setup: NDC has been fully drained of protocol assets and is ready for removal
// maxNegligibleAmount = 0 (default)

// Attacker (any stETH holder) executes:
stETH.transfer(address(nodeDelegator), 1);  // donate 1 wei of stETH

// Admin attempts removal:
lrtDepositPool.removeNodeDelegatorContractFromQueue(address(nodeDelegator));
// Reverts with: NodeDelegatorHasAssetBalance(stETH, 1)

// Admin drains the dust:
// (onlyAssetTransferRole) nodeDelegator.transferBackToLRTDepositPool(stETH, 1)

// Attacker frontruns the next removal attempt:
stETH.transfer(address(nodeDelegator), 1);  // donate again

// Admin's removal reverts again — cycle repeats indefinitely
```

```text
    function _removeNodeDelegatorContractFromQueue(address nodeDelegatorAddress) internal {
        // 1. check if node delegator contract is in queue and find Index
        uint256 ndcIndex = _getNDCIndex(nodeDelegatorAddress);

        // 2. revert if node delegator contract has any asset balances.
        // 2.1 check if NDC has native ETH balance in eigen layer or/and in itself.
        _checkResidueEthBalance(nodeDelegatorAddress);
        // 2.2  check if NDC has LST balance
        _checkResidueLSTBalance(nodeDelegatorAddress);

        // 3. remove node delegator contract from queue
        // 3.1 remove from isNodeDelegator mapping
        isNodeDelegator[nodeDelegatorAddress] = 0;
        // 3.2 remove from nodeDelegatorQueue
        nodeDelegatorQueue[ndcIndex] = nodeDelegatorQueue[nodeDelegatorQueue.length - 1];
        nodeDelegatorQueue.pop();

        emit NodeDelegatorRemovedFromQueue(nodeDelegatorAddress);
    }
```

```text
    function _checkResidueLSTBalance(address nodeDelegatorAddress) internal view {
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetsLength = supportedAssets.length;

        uint256 assetBalance;
        for (uint256 i; i < supportedAssetsLength; ++i) {
            if (supportedAssets[i] == LRTConstants.ETH_TOKEN) {
                // this function only checks for residual LST balance
                continue;
            }

            assetBalance = IERC20(supportedAssets[i]).balanceOf(nodeDelegatorAddress)
                + INodeDelegator(nodeDelegatorAddress).getAssetBalance(supportedAssets[i]);
            assetBalance += INodeDelegator(nodeDelegatorAddress).getAssetUnstaking(supportedAssets[i]);

            if (assetBalance > maxNegligibleAmount) {
                revert NodeDelegatorHasAssetBalance(supportedAssets[i], assetBalance);
            }
        }
    }
```
