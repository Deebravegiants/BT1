### Title
Lack of Access Control on `LRTOracle::updateRSETHPrice()` Allows Any Caller to Trigger Protocol Pause and Fee Minting — (File: contracts/LRTOracle.sol)

---

### Summary

`updateRSETHPrice()` in `LRTOracle` is declared `public` with only a `whenNotPaused` guard and no role restriction. Any unprivileged external caller can invoke it at will. The internal `_updateRsETHPrice()` it delegates to (a) mints protocol-fee rsETH to the treasury (diluting rsETH holders), and (b) pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself when the computed price drop exceeds `pricePercentageLimit`. Both consequences are reachable by any EOA or contract with zero privileges.

---

### Finding Description

`updateRSETHPrice()` carries no role modifier:

```solidity
// contracts/LRTOracle.sol:87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

The privileged sibling `updateRSETHPriceAsManager()` is correctly gated with `onlyLRTManager`, making the intent clear that price updates should be restricted. The public variant was left unrestricted. [1](#0-0) 

`_updateRsETHPrice()` performs two security-critical side-effects reachable through the unguarded path:

**Side-effect 1 — Protocol-wide pause:**
When `newRsETHPrice` has fallen more than `pricePercentageLimit` below `highestRsethPrice`, the function calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` on the oracle itself, then returns early. [2](#0-1) 

**Side-effect 2 — Protocol-fee rsETH minting:**
When TVL has grown since the last update and the protocol is not paused, the function computes a fee in ETH, converts it to rsETH, and mints it to the treasury, diluting all existing rsETH holders. [3](#0-2) 

The role-checking infrastructure (`onlyLRTManager`, `onlyLRTOperator`, etc.) is fully implemented in `LRTConfigRoleChecker` and applied consistently across every other sensitive function in the protocol. Its absence here is an oversight, not a design choice. [4](#0-3) 

---

### Impact Explanation

**Primary — Temporary freezing of funds (Medium):**
An attacker monitors the rsETH price. Whenever the current oracle-derived price has fallen more than `pricePercentageLimit` below `highestRsethPrice` (e.g., after a slashing event or a sustained LST depeg), the attacker calls `updateRSETHPrice()`. The function pauses `LRTDepositPool` and `LRTWithdrawalManager`, immediately blocking all user deposits and withdrawals. Unpausing requires `onlyLRTAdmin`, so users are frozen until the admin acts. The attacker bears only gas cost.

**Secondary — Theft of unclaimed yield (High):**
Whenever staking rewards have accrued (TVL > previousTVL), any caller can invoke `updateRSETHPrice()` to trigger fee minting. The minted rsETH goes to the treasury, diluting every rsETH holder's share of the underlying ETH. An attacker can time calls to coincide with large reward accruals (e.g., immediately after EigenLayer reward distributions), maximising the fee extracted from holders in a single transaction. The daily cap (`maxFeeMintAmountPerDay`) limits per-day extraction but does not prevent the attacker from exhausting that cap on behalf of the treasury at a time of their choosing, depriving holders of yield they would otherwise retain until the next legitimate update.

---

### Likelihood Explanation

The function is `public` on a deployed, verified contract. No special setup, capital, or privileged access is required. Any EOA can call it in a single transaction. The conditions for the pause path (price below threshold) arise naturally during market stress; the conditions for the fee-minting path (TVL growth) are present in virtually every block after rewards accrue. Likelihood is **High**.

---

### Recommendation

Apply `onlyLRTOperator` (or `onlyLRTManager`, consistent with `updateRSETHPriceAsManager`) to `updateRSETHPrice()`:

```solidity
- function updateRSETHPrice() public whenNotPaused {
+ function updateRSETHPrice() external whenNotPaused onlyLRTOperator {
      _updateRsETHPrice();
  }
```

This mirrors the access pattern already enforced on every other state-mutating function in the protocol.

---

### Proof of Concept

1. Deploy or identify the live `LRTOracle` proxy.
2. Wait until either:
   - (Pause path) The current LST oracle prices yield a `newRsETHPrice` that is more than `pricePercentageLimit` below `highestRsethPrice`.
   - (Fee path) Staking rewards have accrued so that `_getTotalEthInProtocol() > rsethSupply * rsETHPrice`.
3. Call `LRTOracle.updateRSETHPrice()` from any EOA.
4. Observe:
   - (Pause path) `LRTDepositPool.paused() == true`, `LRTWithdrawalManager.paused() == true`; all user deposits and withdrawals revert.
   - (Fee path) `FeeMinted` event emitted; treasury rsETH balance increases; rsETH price is lower than it would have been without the fee extraction, diluting all holders. [5](#0-4) [6](#0-5) [7](#0-6)
