### Title
Fee Calculation Truncates to Zero for Small Deposits Due to Integer Division - (File: contracts/pools/RSETHPoolV3.sol)

### Summary
The `viewSwapRsETHAmountAndFee` function in all L2 pool contracts computes the protocol fee as `fee = amount * feeBps / 10_000`. When `amount * feeBps < 10_000`, Solidity integer division truncates the result to zero. The deposit still proceeds in full — rsETH is minted against the entire `amount` with no fee deducted — so the protocol earns zero fee revenue on those deposits. This is the direct structural analog of M-9: a division truncation silently produces zero, state is updated as if the operation succeeded, and one party (here, the protocol) loses the value it was owed.

### Finding Description
Every L2 pool variant shares the same fee computation pattern:

```solidity
// RSETHPoolV3.sol – ETH path
fee = amount * feeBps / 10_000;
uint256 amountAfterFee = amount - fee;
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
```

```solidity
// RSETHPoolV3.sol – token path
fee = amount * feeBps / 10_000;
uint256 amountAfterFee = amount - fee;
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
``` [1](#0-0) [2](#0-1) 

The same pattern is present in every pool variant: [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

After `viewSwapRsETHAmountAndFee` returns, the deposit functions unconditionally update state and mint:

```solidity
feeEarnedInToken[token] += fee;   // += 0, no-op
wrsETH.mint(msg.sender, rsETHAmount);
``` [7](#0-6) 

There is no guard that reverts when `fee == 0` but `feeBps > 0`. The deposit is accepted in full, the user receives rsETH for the entire `amount`, and `feeEarnedInToken` is incremented by zero.

**Concrete truncation threshold per `feeBps` value:**

| `feeBps` | Fee rate | Threshold below which fee = 0 |
|---|---|---|
| 1 | 0.01 % | `amount < 10 000 wei` |
| 5 | 0.05 % | `amount < 2 000 wei` |
| 10 | 0.10 % | `amount < 1 000 wei` |
| 50 | 0.50 % | `amount < 200 wei` |

For a 6-decimal token (e.g., USDC) with `feeBps = 5`, any deposit below 2 000 token-units (0.002 USDC) produces `fee = 0`. The user receives the full rsETH equivalent with no fee charged.

`RSETHPool.sol` additionally stores per-token fee rates in `tokenFeeBps`, which can be set independently and may be as low as 1 bps, widening the zero-fee window further. [8](#0-7) 

### Impact Explanation
The protocol fails to collect the fee revenue it is designed to earn on every deposit. Repeated small deposits — each individually below the truncation threshold — accumulate into a meaningful loss of protocol fee income over time. This maps to **Low** impact: the contract fails to deliver its promised economic behaviour (fee collection) without any direct loss of user principal.

### Likelihood Explanation
Any unprivileged depositor can trigger this by calling `deposit()` or `deposit(token, amount, referralId)` with an `amount` below the truncation threshold. No special role, front-running, or external dependency is required. The entry path is fully public and permissionless. [9](#0-8) [10](#0-9) 

### Recommendation
Add a guard in `viewSwapRsETHAmountAndFee` (or in the deposit functions) that reverts when `feeBps > 0` but the computed `fee` is zero:

```solidity
if (feeBps > 0 && fee == 0) revert InvalidAmount();
```

Alternatively, enforce a minimum deposit amount that guarantees `amount * feeBps >= 10_000` for any configured `feeBps`.

### Proof of Concept
1. Protocol deploys `RSETHPoolV3` with `feeBps = 5` (0.05 %).
2. Attacker calls `deposit{value: 1999}("")` (1 999 wei of ETH).
3. `viewSwapRsETHAmountAndFee(1999)` computes:
   - `fee = 1999 * 5 / 10_000 = 9995 / 10_000 = 0` (integer truncation)
   - `amountAfterFee = 1999 - 0 = 1999`
   - `rsETHAmount = 1999 * 1e18 / rsETHToETHrate` (full amount, no fee deducted)
4. `feeEarnedInETH += 0` — protocol earns nothing.
5. `wrsETH.mint(msg.sender, rsETHAmount)` — attacker receives rsETH for the full 1 999 wei.
6. Repeating this across many transactions accumulates fee-free rsETH minting while the protocol's `feeEarnedInETH` remains unchanged. [11](#0-10)

    function deposit(string memory referralId)
        external
        payable
        nonReentrant
        whenNotPaused
        limitDailyMint(msg.value, ETH_IDENTIFIER)
    {
        if (!isEthDepositEnabled) revert EthDepositDisabled();
        uint256 amount = msg.value;

        if (amount == 0) revert InvalidAmount();

        (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);

        feeEarnedInETH += fee;

        wrsETH.mint(msg.sender, rsETHAmount);

        emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
    }
