### Title
`MultiChainRateProvider.updateRate()` Payable Loop Sends Per-Iteration `estimatedFee` Without Validating `msg.value` Covers Total, Allowing Accumulated ETH to Be Drained - (File: contracts/cross-chain/MultiChainRateProvider.sol)

---

### Summary

`MultiChainRateProvider.updateRate()` is a `payable` function with no access control that loops over all registered `rateReceivers` and sends a separately-computed `estimatedFee` to LayerZero for each receiver. There is no check that `msg.value` equals or exceeds the sum of all per-iteration fees, and there is no refund of excess ETH. Any ETH that accumulates in the contract (from callers who overpay) can be consumed by a subsequent caller who sends 0 ETH, draining the contract's balance.

---

### Finding Description

`updateRate()` is declared `external payable nonReentrant` with no role restriction: [1](#0-0) 

Inside the function, a loop iterates over every registered receiver. For each receiver it independently calls `estimateFees` and immediately forwards that amount to the LayerZero endpoint: [2](#0-1) 

The total ETH consumed by the function is `Σ estimatedFee_i` across all receivers. There is no pre-check of the form `require(msg.value >= totalFee)`, and there is no post-call refund of `address(this).balance` back to `msg.sender`. Because the contract has no `receive()` function, ETH can only accumulate from callers who send more than the sum of fees in a prior `updateRate()` call. Once any ETH is stranded in the contract, any unprivileged caller can invoke `updateRate()` with `msg.value = 0` and the EVM will satisfy each `send{value: estimatedFee}` call from the contract's stored balance, draining it entirely.

The `CrossChainRateProvider` variant does not share this issue because it sends the full `msg.value` in a single call with no loop: [3](#0-2) 

---

### Impact Explanation

Any ETH stranded in `MultiChainRateProvider` (from overpayment by a legitimate caller) is permanently accessible to any external caller who triggers `updateRate()` with zero ETH. The ETH is forwarded to LayerZero as messaging fees rather than returned to the original payer, constituting a loss of those funds. This maps to **theft of unclaimed yield / loss of accumulated ETH** from the contract.

---

### Likelihood Explanation

The precondition is that ETH must first accumulate in the contract. This is realistic because:
1. `updateRate()` is `payable` and publicly callable.
2. Callers must estimate the total fee off-chain; any overestimate leaves a residue with no refund path.
3. Once even a small residue exists, the drain can be triggered by any external account at zero cost to the attacker.

---

### Recommendation

1. Compute the total required fee before the loop and enforce `require(msg.value >= totalFee, "insufficient fee")`.
2. After the loop, refund any excess: `if (address(this).balance > 0) payable(msg.sender).transfer(address(this).balance)`.
3. Alternatively, restrict `updateRate()` to a trusted role so only authorized callers can trigger cross-chain sends.

---

### Proof of Concept

```
// Setup: MultiChainRateProvider has 2 receivers, each with estimatedFee = 0.01 ETH

// Step 1: Alice calls updateRate() with 0.025 ETH (overestimates by 0.005 ETH)
//   → 0.01 ETH sent to receiver[0], 0.01 ETH sent to receiver[1]
//   → 0.005 ETH remains stranded in the contract (no refund)

// Step 2: Bob calls updateRate() with msg.value = 0
//   → estimatedFee for receiver[0] = 0.01 ETH  → paid from contract balance
//   → estimatedFee for receiver[1] = 0.01 ETH  → paid from contract balance
//   → Bob's updateRate() succeeds at zero cost; Alice's 0.005 ETH is consumed
//   → contract balance = 0

// Bob effectively stole Alice's overpayment and got a free cross-chain rate update
``` [4](#0-3)

    function updateRate() external payable nonReentrant {
        uint256 latestRate = getLatestRate();

        rate = latestRate;

        lastUpdated = block.timestamp;

        bytes memory _payload = abi.encode(latestRate);

        uint256 rateReceiversLength = rateReceivers.length;

        for (uint256 i; i < rateReceiversLength;) {
            uint16 dstChainId = uint16(rateReceivers[i]._chainId);

            bytes memory remoteAndLocalAddresses = abi.encodePacked(rateReceivers[i]._contract, address(this));

            (uint256 estimatedFee,) = ILayerZeroEndpoint(layerZeroEndpoint)
                .estimateFees(dstChainId, address(this), _payload, false, bytes(""));

            ILayerZeroEndpoint(layerZeroEndpoint).send{ value: estimatedFee }(
                dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
            );

            unchecked {
                ++i;
            }
        }

        emit RateUpdated(rate);
    }
