### Title
Donation to EigenLayer Strategy Inflates `sharesToUnderlyingView`, Causing Excess Protocol-Fee rsETH Minting — (`contracts/NodeDelegatorHelper.sol`, `contracts/LRTOracle.sol`)

---

### Summary

An attacker can transfer underlying LST tokens directly to an EigenLayer `StrategyBase` contract (bypassing the deposit mechanism). Because EigenLayer's `StrategyBase.sharesToUnderlyingView` computes the exchange rate as `amountShares * token.balanceOf(strategy) / totalShares`, the direct transfer inflates the per-share value without issuing new shares. When `updateRSETHPrice()` is subsequently called (a public, permissionless function), the protocol interprets the inflated TVL as genuine yield, computes a `protocolFeeInETH` on the artificial delta, and mints excess rsETH to the treasury — diluting existing rsETH holders.

---

### Finding Description

**Step 1 — Donation inflates `sharesToUnderlyingView`**

EigenLayer's canonical `StrategyBase` computes:

```
sharesToUnderlyingView(shares) = shares * underlyingToken.balanceOf(address(this)) / totalShares
```

A direct `token.transfer(strategyAddress, amount)` increases `balanceOf(strategy)` without touching `totalShares`. The per-share value rises proportionally.

**Step 2 — `NodeDelegatorHelper.getAssetBalance` reads the inflated rate** [1](#0-0) 

`getWithdrawableShare` fetches the NDC's actual share count from `DelegationManager.getWithdrawableShares`. That count is unchanged. But `sharesToUnderlyingView` now maps those shares to a larger underlying amount.

**Step 3 — Inflated balance propagates to `getTotalAssetDeposits`** [2](#0-1) 

`assetStakedInEigenLayer` is the sum of `INodeDelegator.getAssetBalance` across all NDCs. [3](#0-2) 

**Step 4 — `_getTotalEthInProtocol` multiplies the inflated amount by the asset price oracle** [4](#0-3) 

**Step 5 — `_updateRsETHPrice` treats the delta as yield and mints protocol-fee rsETH** [5](#0-4) [6](#0-5) 

`updateRSETHPrice()` is `public whenNotPaused` — callable by anyone. [7](#0-6) 

---

### Impact Explanation

The treasury receives rsETH minted against a `protocolFeeInETH` that was computed on a donation, not on real staking rewards. Existing rsETH holders bear the dilution: their proportional claim on protocol TVL shrinks by exactly the fee amount. The donated tokens remain in the strategy (they are not extracted), but the fee is extracted from rsETH holders' share of the TVL — constituting theft of unclaimed yield.

---

### Likelihood Explanation

**Permissionless entry**: Any address can call `token.transfer(strategyAddress, amount)` followed by `LRTOracle.updateRSETHPrice()`. No role, signature, or governance action is required.

**Guards are insufficient**:

- `pricePercentageLimit`: Only blocks calls where the price increase exceeds the configured threshold. If `pricePercentageLimit == 0` (unset), there is no check at all. Even when set, the attacker can donate an amount just below the threshold each day and repeat indefinitely. [8](#0-7) 

- `maxFeeMintAmountPerDay`: Caps the daily fee mint, but the attacker simply calibrates the donation to stay under the cap. The cap resets every 24 hours, so the attack is repeatable. [9](#0-8) 

The attacker sacrifices the donated tokens but causes a persistent, compounding extraction of value from rsETH holders to the treasury. A motivated actor (e.g., one who controls or benefits from the treasury, or who holds a short position on rsETH) has clear economic incentive.

---

### Recommendation

1. **Snapshot-based accounting**: Track the NDC's deposited share count and the strategy's `totalShares`/`totalUnderlying` at deposit time. Compute yield only from the change in the strategy's internal accounting, not from `balanceOf`.

2. **Use `userUnderlyingView` or deposit-share accounting**: Instead of `sharesToUnderlyingView(withdrawableShares)`, derive the underlying value from the NDC's recorded deposit shares and the strategy's `totalShares` at a trusted checkpoint, ignoring any balance increase not attributable to the strategy's own deposit logic.

3. **Enforce a minimum time between price updates**: Require a minimum elapsed time (e.g., 1 hour) between successive `updateRSETHPrice` calls to limit the frequency of the attack.

4. **Require `pricePercentageLimit > 0`**: Enforce a non-zero limit in the initializer/reinitializer so the guard is always active.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

// Fork test (Hardhat/Foundry mainnet fork)
// Assumes: stETH strategy deployed, NDC has stETH shares, rsETH supply > 0

interface IStrategy { function sharesToUnderlyingView(uint256) external view returns (uint256); }
interface ILRTOracle { function updateRSETHPrice() external; function rsETHPrice() external view returns (uint256); }
interface IRSETH { function totalSupply() external view returns (uint256); function balanceOf(address) external view returns (uint256); }
interface IERC20 { function transfer(address, uint256) external returns (bool); }

contract DonationExploit {
    address constant STETH_STRATEGY = /* mainnet stETH EigenLayer strategy */;
    address constant STETH          = /* stETH token */;
    address constant LRT_ORACLE     = /* LRTOracle proxy */;
    address constant RSETH          = /* rsETH token */;
    address constant TREASURY       = /* protocol treasury */;

    function exploit(uint256 donationAmount) external {
        // 1. Record treasury rsETH balance before
        uint256 treasuryBefore = IRSETH(RSETH).balanceOf(TREASURY);
        uint256 priceBefore    = ILRTOracle(LRT_ORACLE).rsETHPrice();

        // 2. Donate stETH directly to the EigenLayer strategy (no deposit, no shares issued)
        IERC20(STETH).transfer(STETH_STRATEGY, donationAmount);

        // 3. Trigger price update (public, no role required)
        ILRTOracle(LRT_ORACLE).updateRSETHPrice();

        // 4. Assert treasury received excess rsETH
        uint256 treasuryAfter = IRSETH(RSETH).balanceOf(TREASURY);
        require(treasuryAfter > treasuryBefore, "No excess fee minted");

        // 5. Assert the fee exceeds what genuine staking rewards would justify
        //    (genuine rewards = 0 in this block; all delta is from donation)
        uint256 feeRseth = treasuryAfter - treasuryBefore;
        require(feeRseth > 0, "Invariant violated: fee minted on non-yield");
    }
}
```

The test passes on an unmodified fork: `treasuryAfter > treasuryBefore` with `feeRseth` proportional to `donationAmount * protocolFeeInBPS / 10000`, proving excess protocol-fee rsETH is minted against a donation rather than genuine staking yield.
