### Title
Overly Tight `pricePercentageLimit` Causes Automatic Protocol-Wide Pause on Normal Oracle Fluctuations - (File: contracts/LRTOracle.sol)

### Summary
`LRTOracle.setPricePercentageLimit()` accepts any value with no lower bound. If set below the Chainlink deviation threshold (~1%), a normal oracle price movement can cause `updateRSETHPrice()` — a public function — to automatically pause the deposit pool, withdrawal manager, and oracle itself, freezing all user deposits and withdrawals until an admin manually unpauses.

### Finding Description

`setPricePercentageLimit()` stores `_pricePercentageLimit` without any minimum bound check: [1](#0-0) 

Inside `_updateRsETHPrice()`, when `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit`, the function immediately pauses the deposit pool, withdrawal manager, and oracle: [2](#0-1) 

The trigger function `updateRSETHPrice()` is public and callable by anyone: [3](#0-2) 

The rsETH price is computed from Chainlink-backed asset oracles (e.g., stETH/ETH, rETH/ETH) via `_getTotalEthInProtocol()`: [4](#0-3) 

Chainlink price feeds for ETH-denominated LST assets carry a **1% deviation threshold** and a **24-hour heartbeat**. This means the reported price can legitimately differ from the true price by up to ~1% and remain at that deviation for up to 24 hours before a new round is triggered.

The `highestRsethPrice` is a **peak** (all-time high) tracker: [5](#0-4) 

Once `highestRsethPrice` is set at a peak, any subsequent oracle price that is even slightly below that peak (due to normal Chainlink deviation) is compared against `pricePercentageLimit`. If `pricePercentageLimit` is set to, say, 0.5% (`5e15`), a routine 0.6% downward oracle fluctuation — well within Chainlink's normal 1% deviation band — will satisfy `isPriceDecreaseOffLimit = true` and trigger the automatic pause.

After the pause, `updateRSETHPrice()` is blocked by `whenNotPaused`: [6](#0-5) 

This means the price cannot self-recover. The protocol stays paused until an admin calls `unpause()` on the oracle, and separately unpauses the deposit pool and withdrawal manager.

### Impact Explanation

All user deposits (`LRTDepositPool.depositETH`, `depositAsset`) and all withdrawals (`LRTWithdrawalManager`) are gated by `whenNotPaused`. Once the automatic pause fires, every user's ability to deposit or withdraw is frozen. This constitutes **temporary freezing of funds** (Medium severity per the allowed impact scope). [7](#0-6) 

### Likelihood Explanation

- `pricePercentageLimit` is configurable by admin with no lower bound. A value below 1% (e.g., 0.5%) is plausible as a "conservative" safety setting.
- Chainlink LST/ETH feeds routinely move within their 1% deviation band during normal market activity.
- `updateRSETHPrice()` is public — any user, keeper bot, or MEV searcher can call it at any time.
- Once `highestRsethPrice` is set at a local peak, even a small retracement within the Chainlink deviation window satisfies the pause condition.
- The pause is irreversible without admin intervention, and the oracle's own `whenNotPaused` guard prevents automated recovery.

### Recommendation

Enforce a minimum lower bound on `pricePercentageLimit` that is strictly greater than the maximum expected Chainlink deviation for any supported asset oracle (at minimum 1%, i.e., `1e16`). Add a validation in `setPricePercentageLimit()`:

```solidity
uint256 public constant MIN_PRICE_PERCENTAGE_LIMIT = 1e16; // 1%

function setPricePercentageLimit(uint256 _pricePercentageLimit) external onlyLRTAdmin {
    if (_pricePercentageLimit > 0 && _pricePercentageLimit < MIN_PRICE_PERCENTAGE_LIMIT) {
        revert PricePercentageLimitTooLow();
    }
    pricePercentageLimit = _pricePercentageLimit;
    emit PricePercentageLimitUpdate(_pricePercentageLimit);
}
```

### Proof of Concept

1. Admin calls `setPricePercentageLimit(5e15)` (0.5%).
2. Protocol accumulates rewards; `updateRSETHPrice()` is called, `highestRsethPrice` is set to `1.050e18`.
3. Chainlink stETH/ETH feed moves down 0.6% within its normal deviation band (no new round triggered). `_getTotalEthInProtocol()` now returns a value yielding `newRsETHPrice = 1.0437e18`.
4. Any user calls `updateRSETHPrice()`.
5. Inside `_updateRsETHPrice()`: `diff = 1.050e18 - 1.0437e18 = 6.3e14`; `pricePercentageLimit.mulWad(highestRsethPrice) = 5e15 * 1.050e18 / 1e18 = 5.25e15`; `6.3e14 < 5.25e15` — wait, let me recalculate: `diff = 1.050e18 - 1.0437e18 = 6.3e15`; `pricePercentageLimit.mulWad(highestRsethPrice) = 5e15 * 1.050 = 5.25e15`; `6.3e15 > 5.25e15` → `isPriceDecreaseOffLimit = true`.
6. `lrtDepositPool.pause()`, `withdrawalManager.pause()`, `_pause()` are all called.
7. All deposits and withdrawals revert with `Pausable: paused` until admin manually unpauses all three contracts. [8](#0-7) [2](#0-1)
