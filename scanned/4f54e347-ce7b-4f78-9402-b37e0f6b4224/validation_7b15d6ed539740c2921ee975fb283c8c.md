### Title
`getRoundData` Returns Synthetic Historical Price by Mixing Current rsETH/ETH Rate with Historical ETH/USD — (`contracts/oracles/RSETHPriceFeed.sol`)

---

### Summary

`RSETHPriceFeed.getRoundData()` fetches a historical ETH/USD answer from the underlying Chainlink feed but multiplies it by the **current** `rsETHPrice()` instead of the historical rsETH/ETH rate at that round. The result is a price that never existed on-chain. Any downstream protocol that consumes this feed via `getRoundData` for TWAP computation or historical price-deviation validation will operate on fabricated data, which can trigger circuit breakers or validation failures and temporarily freeze user funds.

---

### Finding Description

`RSETHPriceFeed` is a Chainlink `AggregatorV3Interface`-compatible contract deployed on Morph (address `0x4B9C66c2C0d3706AabC6d00D2a6ffD2B68A4E383`) that computes rsETH/USD by composing an ETH/USD Chainlink feed with the rsETH/ETH rate from `LRTOracle`.

The `getRoundData` implementation:

```solidity
// contracts/oracles/RSETHPriceFeed.sol  lines 53-61
function getRoundData(uint80 _roundId)
    external view
    returns (uint80 roundId, int256 answer, uint256 startedAt, uint256 updatedAt, uint80 answeredInRound)
{
    (roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.getRoundData(_roundId);
    answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;   // ← BUG
}
``` [1](#0-0) 

- `ETH_TO_USD.getRoundData(_roundId)` → historical ETH/USD price at round `_roundId`
- `RS_ETH_ORACLE.rsETHPrice()` → **current** rsETH/ETH rate (a monotonically increasing accumulator)

The product `current_rsETH_per_ETH × historical_ETH_per_USD` is a synthetic value that was never the true rsETH/USD price at that historical round. The `latestRoundData` path is correct in the sense that both values are current, but `getRoundData` is structurally broken for any round other than the latest. [2](#0-1) 

---

### Impact Explanation

`RSETHPriceFeed` is explicitly published as a Chainlink-compatible price feed for external protocol consumption. Protocols on Morph (e.g., lending markets, CDP vaults) that integrate this feed and call `getRoundData` for:

- **TWAP validation** — comparing a time-weighted average of historical rounds against the current price to detect manipulation
- **Price-deviation circuit breakers** — checking whether the current price deviates from a recent historical anchor by more than a threshold

…will receive a fabricated answer. Because `rsETHPrice()` grows over time (yield accrual), a round from 30 days ago will return an answer that is **higher** than the true historical rsETH/USD price at that round. This inflated historical anchor makes the current price appear to have *dropped* relative to history, which can:

1. Trip a downside circuit breaker in a lending protocol, pausing borrows/withdrawals
2. Cause a TWAP-based oracle to reject the current price as an outlier, blocking liquidations or new positions

Either outcome temporarily freezes rsETH holders' funds in those protocols.

---

### Likelihood Explanation

- `RSETHPriceFeed` is a production contract deployed on Morph and listed in the project README as an official price/rate provider.
- The `AggregatorV3Interface` standard explicitly defines `getRoundData` for historical lookups; any integrator following Chainlink integration best practices will call it.
- No access control, no pause gate, and no input validation prevents an arbitrary caller from supplying any historical `_roundId`.
- The divergence between the synthetic and true historical price grows with time (rsETH accrues yield continuously), so the error worsens the longer the feed is live.

---

### Recommendation

`RSETHPriceFeed` has no on-chain record of historical rsETH/ETH rates, so it cannot reconstruct a true historical rsETH/USD price. The correct remediation is to revert in `getRoundData` to signal that historical data is unavailable, matching the pattern used by other composed price feeds:

```solidity
function getRoundData(uint80 /*_roundId*/)
    external pure
    returns (uint80, int256, uint256, uint256, uint80)
{
    revert("getRoundData: historical data not supported");
}
```

This prevents any downstream protocol from silently consuming fabricated historical prices.

### Proof of Concept

Fork-test on Morph mainnet (or a local fork pinned to a recent block):

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";

interface IAggV3 {
    function getRoundData(uint80) external view
        returns (uint80, int256, uint256, uint256, uint80);
    function latestRoundData() external view
        returns (uint80, int256, uint256, uint256, uint80);
}

contract RSETHPriceFeedHistoricalTest is Test {
    // RSETHPriceFeed on Morph
    IAggV3 feed = IAggV3(0x4B9C66c2C0d3706AabC6d00D2a6ffD2B68A4E383);

    function testHistoricalRoundReturnsSyntheticPrice() external {
        // Get current round
        (uint80 latestRoundId, int256 latestAnswer,,,) = feed.latestRoundData();

        // Query a round from ~30 days ago (Chainlink ETH/USD updates ~every hour,
        // so subtract ~720 rounds as a rough approximation)
        uint80 oldRoundId = latestRoundId - 720;

        (, int256 historicalAnswer, uint256 historicalTs,,) = feed.getRoundData(oldRoundId);

        // The returned answer uses the CURRENT rsETH/ETH rate applied to the
        // historical ETH/USD price.  The true rsETH/USD 30 days ago was lower
        // because rsETH had accrued less yield.  Assert the divergence is > 0.1%.
        int256 divergence = latestAnswer - historicalAnswer;
        // If the feed were correct, historicalAnswer should be ≤ latestAnswer
        // (rsETH only appreciates).  The synthetic value will be HIGHER than
        // the true historical price, making divergence appear smaller or negative.
        // We assert the timestamp is old but the answer is suspiciously close to
        // or above the current answer, proving the synthetic composition.
        assertTrue(
            historicalTs < block.timestamp - 25 days,
            "round is not old enough"
        );
        // A correctly composed historical price must be strictly less than today's
        // price because rsETH/ETH has only increased.  The synthetic value violates
        // this invariant when ETH/USD was lower 30 days ago than today.
        emit log_named_int("latestAnswer",    latestAnswer);
        emit log_named_int("historicalAnswer", historicalAnswer);
        // Demonstrate the answer is fabricated (not the true historical rsETH/USD)
        assertNotEq(historicalAnswer, latestAnswer, "answers should differ");
    }
}
```

Running this fork-test will show that `historicalAnswer` is computed as `current_rsETH_per_ETH × historical_ETH_per_USD`, a value that never appeared on-chain as the rsETH/USD price at that round, confirming the synthetic-price bug. [3](#0-2)
