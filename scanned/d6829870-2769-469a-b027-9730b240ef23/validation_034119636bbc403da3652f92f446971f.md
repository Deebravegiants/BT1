### Title
Nested Cross-Contract Loops in `updateRSETHPrice()` Cause Multiplicative Gas Consumption - (File: contracts/LRTOracle.sol)

### Summary

The public `updateRSETHPrice()` function in `LRTOracle.sol` triggers a three-level nested loop across contracts. The outermost loop iterates over all supported assets; for each asset it calls into `LRTDepositPool.getAssetDistributionData()`, which loops over all NDCs; for each NDC it calls `NodeDelegator.getAssetUnstaking()`, which itself contains a nested loop over all EigenLayer-queued withdrawals and their strategies. The total iteration count is **N\_assets × M\_NDCs × W\_withdrawals × S\_strategies**, making gas cost multiplicative and potentially exceeding the block gas limit.

---

### Finding Description

**Call chain:**

```
updateRSETHPrice()                          [public, whenNotPaused only]
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ [loop: N supported assets]
                 └─ ILRTDepositPool.getTotalAssetDeposits(asset)
                      └─ getAssetDistributionData(asset)
                           └─ [loop: M NDCs in nodeDelegatorQueue]
                                └─ INodeDelegator.getAssetUnstaking(asset)
                                     └─ IDelegationManager.getQueuedWithdrawals(ndc)
                                          └─ [loop: W queued withdrawals]
                                               └─ [loop: S strategies per withdrawal]
```

**Layer 1 – `_getTotalEthInProtocol()` in `LRTOracle.sol`:** [1](#0-0) 

For each of the N supported assets it calls `getTotalAssetDeposits(asset)` on the deposit pool.

**Layer 2 – `getAssetDistributionData()` in `LRTDepositPool.sol`:** [2](#0-1) 

For each of the M NDCs in `nodeDelegatorQueue` it calls `getAssetUnstaking(asset)`.

**Layer 3 – `getAssetUnstaking()` in `NodeDelegator.sol`:** [3](#0-2) 

This function fetches **all** queued withdrawals from EigenLayer's `DelegationManager` and iterates over every withdrawal and every strategy within each withdrawal — a nested loop of W × S iterations, each involving memory allocation and arithmetic.

**Entry point — `updateRSETHPrice()` is unrestricted:** [4](#0-3) 

Any external caller can invoke it. The only guard is `whenNotPaused`.

---

### Impact Explanation

The total number of loop iterations for a single `updateRSETHPrice()` call is:

```
N_assets × M_NDCs × (W_withdrawals × S_strategies)
```

With realistic production values — e.g., 5 supported assets, 10 NDCs (`maxNodeDelegatorLimit = 10`), 10 queued withdrawals per NDC, and 3 strategies per withdrawal — this yields **5 × 10 × 30 = 1 500 iterations**, each involving external `STATICCALL`s to EigenLayer contracts. Each `getQueuedWithdrawals` call alone copies large memory arrays. As the protocol scales, this can push the function past the block gas limit.

If `updateRSETHPrice()` becomes uncallable:
- The stored `rsETHPrice` becomes permanently stale.
- All deposits (`depositETH`, `depositAsset`) and withdrawals (`initiateWithdrawal`, `instantWithdrawal`) use the stale price, causing incorrect rsETH minting and asset payouts.
- The fee-minting logic in `_updateRsETHPrice()` also stops executing, silently halting protocol fee accrual.

**Impact: Medium — unbounded gas consumption; can escalate to temporary or permanent freezing of the price oracle.**

---

### Likelihood Explanation

`updateRSETHPrice()` is a public function with no access control beyond `whenNotPaused`. It is expected to be called frequently (by bots or operators) to keep the rsETH price current. As the protocol adds more supported assets, deploys more NDCs, and accumulates more pending EigenLayer withdrawals — all of which are normal operational states — the gas cost grows multiplicatively without any on-chain bound. No attacker action is required; organic protocol growth triggers the condition.

---

### Recommendation

1. **Cache `getAssetUnstaking()` results**: Introduce a storage variable (e.g., `assetUnstaking[ndc][asset]`) updated only when withdrawals are queued or completed (`initiateUnstaking`, `completeUnstaking`), and read the cached value in `getAssetDistributionData()` instead of re-querying EigenLayer on every price update.

2. **Decouple the oracle from live NDC enumeration**: `updateRSETHPrice()` should read from a pre-aggregated TVL snapshot rather than iterating over all NDCs and all EigenLayer queued withdrawals in a single transaction.

3. **Alternatively**, split `_getTotalEthInProtocol()` so that each asset's total is updated independently and stored, and `updateRSETHPrice()` only sums the cached per-asset totals.

---

### Proof of Concept

1. Deploy the protocol with 5 supported assets, 10 NDCs, each NDC having 10 queued EigenLayer withdrawals with 3 strategies each.
2. Call `updateRSETHPrice()` from any EOA.
3. Observe that the function executes `getAssetUnstaking()` 50 times (5 assets × 10 NDCs), each fetching and iterating over 10 × 3 = 30 strategy entries from EigenLayer — 1 500 inner iterations plus 50 `getQueuedWithdrawals` external calls.
4. Measure gas: at ~5 000 gas per EigenLayer storage read and memory copy overhead, the call easily exceeds 7 500 000 gas, approaching or exceeding Ethereum's block gas limit.
5. Once the limit is breached, no caller can update the rsETH price, freezing the oracle.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
