### Title
`LRTOracle._updateRsETHPrice()` Can Persist `rsETHPrice = 0`, Breaking All Deposit and Withdrawal Invariants — (`contracts/LRTOracle.sol`)

### Summary

`LRTOracle._updateRsETHPrice()` computes `newRsETHPrice` as a division of `totalETHInProtocol` by `rsethSupply`, but never validates that the result is non-zero before writing it to storage. The only guard — a downside-pause triggered by `pricePercentageLimit` — is disabled when `pricePercentageLimit == 0` (its default/unset value). If `rsETHPrice` is ever written as `0`, every downstream function that divides by `rsETHPrice` either reverts (freezing deposits) or silently returns `0` (zeroing withdrawal amounts).

### Finding Description

In `LRTOracle.sol`, `_updateRsETHPrice()` computes the new exchange rate:

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
```

If `totalETHInProtocol == 0` (and `protocolFeeInETH == 0`) while `rsethSupply > 0`, `newRsETHPrice` evaluates to `0`. The function then checks whether to trigger a downside pause:

```solidity
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
if (isPriceDecreaseOffLimit) { ... _pause(); return; }
```

Because `pricePercentageLimit` defaults to `0` and is only set by an explicit admin call to `setPricePercentageLimit()`, `isPriceDecreaseOffLimit` evaluates to `false` for any price drop — including a drop to zero. Execution falls through to:

```solidity
rsETHPrice = newRsETHPrice;   // rsETHPrice = 0
```

`updateRSETHPrice()` is `public whenNotPaused`, callable by any address.

### Impact Explanation

Once `rsETHPrice == 0`:

1. **`LRTDepositPool.getRsETHAmountToMint()`** (line 520) performs `/ lrtOracle.rsETHPrice()` → division-by-zero revert → all L1 deposits are permanently frozen until the price is corrected.
2. **`RSETHPool` / `RSETHPoolV3` / `RSETHPoolV2` `viewSwapRsETHAmountAndFee()`** performs `amountAfterFee * 1e18 / rsETHToETHrate` where `rsETHToETHrate == 0` → division-by-zero revert → all L2 pool deposits are frozen.
3. **`LRTWithdrawalManager.getExpectedAssetAmount()`** (line 593) returns `amount * 0 / assetPrice == 0` → any withdrawal processed at this price yields zero assets to the user.

Impact: **Temporary freezing of funds** (Medium) for deposits; **theft of unclaimed yield / zero-value withdrawal** (High) for withdrawals processed while `rsETHPrice == 0`.

### Likelihood Explanation

`totalETHInProtocol` is computed by `_getTotalEthInProtocol()`, which sums `totalAssetAmt.mulWad(assetER)` across all entries in `supportedAssetList`. It reaches `0` when either:

- All supported assets have zero deposits across the deposit pool, node delegators, EigenLayer, converter, and unstaking vault simultaneously (e.g., during a full protocol wind-down or migration), **or**
- The `supportedAssetList` is empty (all assets removed by admin).

Both conditions are edge-case but plausible during protocol lifecycle events. Critically, `pricePercentageLimit == 0` is the **default state** — the protection is opt-in, not opt-out. Any deployment that has not explicitly called `setPricePercentageLimit()` is fully exposed. The entry point (`updateRSETHPrice()`) is permissionless.

### Recommendation

Add an explicit post-computation invariant check in `_updateRsETHPrice()`:

```solidity
require(newRsETHPrice > 0, "rsETHPrice cannot be zero");
```

Additionally, consider requiring `pricePercentageLimit > 0` as part of the oracle initialization, so the downside-pause protection is always active.

### Proof of Concept

1. Deploy the protocol with `pricePercentageLimit == 0` (default — `setPricePercentageLimit` never called).
2. Reach a state where `_getTotalEthInProtocol()` returns `0` while `rsETH.totalSupply() > 0` (e.g., all assets migrated out during an upgrade, or `supportedAssetList` emptied).
3. Any address calls `LRTOracle.updateRSETHPrice()`.
4. `newRsETHPrice = 0.divWad(rsethSupply) = 0`. `isPriceDecreaseOffLimit = false` (because `pricePercentageLimit == 0`). Execution reaches `rsETHPrice = 0`.
5. Any subsequent call to `LRTDepositPool.depositETH()` or `RSETHPoolV3.deposit()` reverts with division-by-zero. Any call to `LRTWithdrawalManager.getExpectedAssetAmount()` returns `0`, and a withdrawal processed at this price transfers `0` assets to the user.
