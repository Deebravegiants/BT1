### Title
Unbounded Nested Iteration Over NDCs and EigenLayer Queued Withdrawals on Every Deposit - (File: contracts/LRTDepositPool.sol, contracts/NodeDelegator.sol)

### Summary
Every call to `depositETH()` or `depositAsset()` in `LRTDepositPool` triggers a nested loop: it iterates over all registered `NodeDelegator` contracts (NDCs), and for each NDC calls `getAssetUnstaking()`, which in turn makes an external call to EigenLayer's `DelegationManager.getQueuedWithdrawals()` and iterates over all queued withdrawals and their strategies. As the protocol scales (more NDCs, more pending EigenLayer withdrawals), the gas cost of every deposit grows proportionally, eventually causing all deposits to revert when the block gas limit is exceeded.

### Finding Description
The call chain triggered by every unprivileged deposit is:

1. `depositETH()` / `depositAsset()` → `_beforeDeposit()` → `_checkIfDepositAmountExceedesCurrentLimit()` → `getTotalAssetDeposits()`
2. `getTotalAssetDeposits()` → `getAssetDistributionData()` (or `getETHDistributionData()`)
3. Inside `getAssetDistributionData()` / `getETHDistributionData()`: a `for` loop over all `nodeDelegatorQueue` entries, calling `INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset)` on each.
4. Inside `NodeDelegator.getAssetUnstaking()`: an external call to `DelegationManager.getQueuedWithdrawals(address(this))` followed by a nested `for` loop over all returned `Withdrawal[]` and their `strategies[]`.

### Impact Explanation
The total gas consumed per deposit scales as **O(NDC_count × queued_withdrawals_per_NDC × strategies_per_withdrawal)**. As the protocol accumulates more NDCs (up to `maxNodeDelegatorLimit`, admin-settable) and each NDC accumulates pending EigenLayer withdrawals (up to `maxUncompletedWithdrawalCount`), the gas cost of every deposit grows. Once the aggregate gas cost exceeds the block gas limit, `depositETH()` and `depositAsset()` permanently revert, freezing all new deposits. This is a **Medium — Temporary freezing of funds** (deposits are blocked until operators complete pending EigenLayer withdrawals to reduce the queued withdrawal count).

### Likelihood Explanation
The protocol is designed to scale: multiple NDCs are expected, and EigenLayer withdrawals have a mandatory delay (days to weeks) before they can be completed. During periods of high unstaking activity, many withdrawals will be simultaneously queued across multiple NDCs. Any unprivileged depositor triggers this path on every call, making the gas exhaustion reachable under normal protocol operation at scale.

### Recommendation
1. **Cache `getAssetUnstaking` results off-chain or lazily**: Do not call `getAssetUnstaking()` (which makes an external call to EigenLayer and iterates over all queued withdrawals) inside the deposit hot path. Instead, maintain an on-chain accounting variable that is updated only when withdrawals are queued or completed.
2. **Decouple deposit limit checks from live EigenLayer state**: The deposit limit check in `_checkIfDepositAmountExceedesCurrentLimit` should use a cached/snapshotted TVL rather than recomputing it live across all NDCs and EigenLayer strategies on every deposit.
3. **Merge the two iteration pipelines**: `getAssetDistributionData` and `getETHDistributionData` both iterate over all NDCs independently. A single pass that accumulates all asset balances simultaneously would halve the NDC iteration count.

### Proof of Concept
```
// Attacker is any depositor. Protocol has N NDCs, each with M queued EigenLayer withdrawals.
// Gas cost per depositETH() call:
//   = O(N * M * S) where S = strategies per withdrawal
//
// With N=10 NDCs, M=50 queued withdrawals each, S=2 strategies:
//   = 1000 external calls to EigenLayer strategy contracts + loop overhead
//
// At ~5000 gas per external SLOAD/call, this is ~5M gas per deposit,
// approaching the 30M block gas limit with larger N or M values.

// Step 1: Protocol accumulates NDCs (admin action, normal operation)
// Step 2: Operators queue many EigenLayer withdrawals (normal operation, delayed by weeks)
// Step 3: Any user calls depositETH{value: 1 ether}(0, "") 
//         → reverts with out-of-gas
//         → all deposits are frozen until operators complete pending withdrawals
```

```text
    {
```

```text
            revert InvalidAmountToDeposit();

        if (_checkIfDepositAmountExceedesCurrentLimit(asset, depositAmount)) {
            revert MaximumDepositLimitReached();
        }

        rsethAmountToMint = getRsETHAmountToMint(asset, depositAmount);

        if (rsethAmountToMint < minRSETHAmountExpected) {
            revert MinimumAmountToReceiveNotMet();
```

```text
    function _checkIfDepositAmountExceedesCurrentLimit(address asset, uint256 amount) internal view returns (bool) {
        uint256 totalAssetDeposits = getTotalAssetDeposits(asset);
        if (asset == LRTConstants.ETH_TOKEN) {
            return (totalAssetDeposits > lrtConfig.depositLimitByAsset(asset));
        }
        return (totalAssetDeposits + amount > lrtConfig.depositLimitByAsset(asset));
    }
```

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
