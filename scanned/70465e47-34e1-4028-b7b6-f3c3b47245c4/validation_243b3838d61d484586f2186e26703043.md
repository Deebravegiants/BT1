### Title
`LRTOracle._getTotalEthInProtocol()` Reverts for All Assets If Any Supported Asset Lacks a Price Oracle — (`contracts/LRTOracle.sol`)

---

### Summary

`LRTOracle._getTotalEthInProtocol()` iterates over every asset in `lrtConfig.getSupportedAssetList()` and calls `getAssetPrice(asset)` on each. `getAssetPrice` is guarded by the `onlySupportedOracle` modifier, which reverts with `AssetOracleNotSupported` if `assetPriceOracle[asset] == address(0)`. If any supported asset has no price oracle registered in `LRTOracle`, the entire loop reverts, making `updateRSETHPrice()` permanently uncallable until the oracle is set.

---

### Finding Description

`LRTOracle._getTotalEthInProtocol()` loops over all supported assets and calls `getAssetPrice` for each:

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);   // reverts if oracle == address(0)
    ...
}
``` [1](#0-0) 

`getAssetPrice` carries the `onlySupportedOracle` modifier:

```solidity
modifier onlySupportedOracle(address asset) {
    if (assetPriceOracle[asset] == address(0)) {
        revert AssetOracleNotSupported();
    }
    _;
}
``` [2](#0-1) 

`LRTConfig.addNewSupportedAsset()` adds an asset to `supportedAssetList` with no requirement that a price oracle has been registered in `LRTOracle`: [3](#0-2) 

`updatePriceOracleFor` is a separate admin call that must be made independently: [4](#0-3) 

There is no atomicity or enforcement linking the two operations. During the window between asset addition and oracle registration — or if the oracle is ever cleared — `_getTotalEthInProtocol()` reverts for every asset, not just the one missing the oracle.

---

### Impact Explanation

`_getTotalEthInProtocol()` is called by `_updateRsETHPrice()`, which is called by the public `updateRSETHPrice()`: [5](#0-4) 

When `updateRSETHPrice()` is broken:
1. The rsETH price is frozen at its last value.
2. Protocol fee minting (yield to the treasury) is blocked — unclaimed yield is permanently frozen for the duration.
3. The price-drop auto-pause safety mechanism cannot trigger.
4. New depositors mint rsETH at the stale (lower) price, diluting existing holders' accrued yield.

**Impact: Medium — Permanent freezing of unclaimed yield** (protocol fee rsETH cannot be minted; existing holders' yield is diluted by stale-price deposits).

---

### Likelihood Explanation

`addNewSupportedAsset` is gated by `TIME_LOCK_ROLE` and `updatePriceOracleFor` by `onlyLRTAdmin`. These are separate roles and separate transactions. The protocol already supports multiple assets (stETH, ETHx, etc.), and adding a new asset without immediately registering its oracle in `LRTOracle` is a realistic operational gap — especially since there is no on-chain enforcement linking the two steps. The vulnerability window exists any time a new asset is added before its oracle is configured.

---

### Recommendation

Add a zero-address guard in `_getTotalEthInProtocol()` to skip assets without a registered oracle, analogous to the M-13 fix:

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
+   if (assetPriceOracle[asset] == address(0)) {
+       unchecked { ++assetIdx; }
+       continue;
+   }
    uint256 assetER = getAssetPrice(asset);
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    unchecked { ++assetIdx; }
}
```

Alternatively, enforce atomicity by requiring the oracle to be set in `LRTOracle` before `addNewSupportedAsset` completes.

---

### Proof of Concept

1. Admin calls `LRTConfig.addNewSupportedAsset(newToken, depositLimit)` — `newToken` is now in `supportedAssetList`.
2. Admin has not yet called `LRTOracle.updatePriceOracleFor(newToken, oracleAddr)` — `assetPriceOracle[newToken] == address(0)`.
3. Any caller (including a keeper bot or any user) calls `LRTOracle.updateRSETHPrice()`.
4. Execution reaches `_getTotalEthInProtocol()` → loop reaches `newToken` → `getAssetPrice(newToken)` → `onlySupportedOracle` modifier fires → `revert AssetOracleNotSupported()`.
5. `updateRSETHPrice()` reverts. The rsETH price is frozen. Protocol fee minting is blocked. All other assets with valid oracles are also unable to contribute to the price update. [6](#0-5) [7](#0-6)

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
```
