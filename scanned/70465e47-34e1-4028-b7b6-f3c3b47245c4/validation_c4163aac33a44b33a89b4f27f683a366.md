### Title
Missing Balance Check in `removeAllowedToken` Permanently Freezes Deposited altAgETH — (`contracts/agETH/AGETHTokenWrapper.sol`)

### Summary

`AGETHTokenWrapper.removeAllowedToken` sets `allowedTokens[_asset] = false` with no balance guard. Once removed, `_withdraw` reverts for all holders of that asset, and there is no `addAllowedToken` function in the contract to restore access. The freeze is permanent without a proxy upgrade.

### Finding Description

`removeAllowedToken` is callable by `DEFAULT_ADMIN_ROLE`: [1](#0-0) 

It performs no check on the contract's current balance of `_asset`. Both `withdraw` and `withdrawTo` route through `_withdraw`, which gates on `allowedTokens`: [2](#0-1) 

Once `allowedTokens[altAgETH]` is `false`, every call to `withdraw`/`withdrawTo` for that asset reverts with `TokenNotAllowed`. There is no `addAllowedToken` function in `AGETHTokenWrapper.sol` — the contract comment explicitly states tokens cannot be re-added: [3](#0-2) 

By contrast, `AGETHPoolV3.removeSupportedToken` enforces a balance-zero precondition before removal: [4](#0-3) 

This inconsistency within the same protocol confirms the missing check in `AGETHTokenWrapper` is an unintentional omission, not a design choice.

### Impact Explanation

All users who deposited `altAgETH` and hold wrapped `agETH` are unable to call `withdraw` or `withdrawTo` for that asset. Their collateral is locked in the contract with no on-chain recovery path. Because there is no `addAllowedToken` function, the freeze is **permanent** without a proxy upgrade — making the actual impact **Critical (permanent freezing of funds)**, not merely Medium/temporary.

### Likelihood Explanation

The admin does not need to be malicious. A routine token migration or deprecation attempt (e.g., replacing one altAgETH variant with another) would trigger this path. The absence of a balance guard makes accidental execution realistic.

### Recommendation

Add a balance check to `removeAllowedToken` mirroring `AGETHPoolV3`:

```solidity
function removeAllowedToken(address _asset) external onlyRole(DEFAULT_ADMIN_ROLE) {
    if (ERC20Upgradeable(_asset).balanceOf(address(this)) != 0) revert TokenBalanceNotZero();
    allowedTokens[_asset] = false;
    emit TokenRemoved(_asset);
}
```

Alternatively, add a corresponding `addAllowedToken` function so that removal is reversible, reducing the impact from permanent to temporary.
