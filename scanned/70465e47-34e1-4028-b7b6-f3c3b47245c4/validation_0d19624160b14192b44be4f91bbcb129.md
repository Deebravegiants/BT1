### Title
Silent Aave Deposit Failure Leaves ETH Idle and Not Earning Yield — (`contracts/LRTWithdrawalManager.sol`)

---

### Summary

When `unlockQueue` calls `depositToAaveExternal` via try/catch and the Aave deposit reverts (e.g., supply cap hit, pool paused), the ETH remains in the contract as idle balance. `totalETHDepositedToAave` is correctly **not** incremented (since no deposit occurred), but the ETH earns no yield until an operator manually calls `depositIdleETHToAave`. If the operator never calls it, the ETH sits idle indefinitely.

---

### Finding Description

In `unlockQueue`, after redeeming ETH from the unstaking vault, the contract attempts to deposit it to Aave via a try/catch self-call: [1](#0-0) 

If `depositToAaveExternal` reverts, the catch block emits `AaveDepositFailed` and silently continues: [2](#0-1) 

`_depositToAave` only increments `totalETHDepositedToAave` on success: [3](#0-2) 

So after a failed deposit, the ETH sits in `address(this).balance` earning nothing. The only recovery path is the operator calling `depositIdleETHToAave`: [4](#0-3) 

There is no automatic retry, no on-chain enforcement that the operator must act, and no time-bound guarantee.

---

### Impact Explanation

**Low — Contract fails to deliver promised returns, but doesn't lose value.**

The ETH is safe and available for user withdrawals (it sits in `address(this).balance` and `_processWithdrawalCompletion` checks `contractBalance` before touching Aave): [5](#0-4) 

No funds are lost. However, the protocol's Aave yield-generation promise is silently broken for an unbounded period if the operator is slow or unresponsive.

**Correction to the question's secondary claim:** The `InsufficientLiquidityForWithdrawal` path is **not** reachable via this mechanism. When the Aave deposit fails, the ETH remains in `address(this).balance`. `_processWithdrawalCompletion` reads `contractBalance = address(this).balance` first and only calls `_withdrawFromAave` if the contract balance is insufficient. Since the idle ETH covers the unlocked withdrawal requests (the same amount redeemed from the vault), the direct-balance path succeeds. `totalETHDepositedToAave` is also **not** stale — it correctly reflects only ETH that was successfully deposited to Aave, so `_withdrawFromAave`'s principal cap (`min(aaveBalance, totalETHDepositedToAave)`) remains accurate. [6](#0-5) 

---

### Likelihood Explanation

Aave supply cap exhaustion and pool pauses are documented, real-world events on mainnet. The `getAaveAvailableCapacity` view function exists precisely because the team anticipated this: [7](#0-6) 

The operator dependency is the only mitigation, and it is off-chain with no SLA enforced on-chain.

---

### Recommendation

1. Emit a more prominent alert (or revert with a specific error) when `AaveDepositFailed` fires, so monitoring systems can trigger an immediate `depositIdleETHToAave` call.
2. Consider adding a public `depositIdleETHToAave` path (with appropriate guards) or a keeper-compatible function so recovery is not gated solely on a privileged operator.
3. Optionally, check `getAaveAvailableCapacity()` before attempting the deposit and skip the try/catch if capacity is zero, making the failure mode explicit rather than silent.

    function getAaveAvailableCapacity() external view returns (uint256 availableCapacity) {
        if (address(aaveAWETH) == address(0) || address(aaveDataProvider) == address(0)) {
            revert InvalidAaveConfiguration();
        }

        // Get supply cap from Aave Data Provider
        (, uint256 supplyCap) = aaveDataProvider.getReserveCaps(WETH_ADDRESS);

        // If supply cap is 0, it means unlimited capacity
        if (supplyCap == 0) return type(uint256).max;

        // Get current total supply of aWETH
        uint256 totalSupply = aaveAWETH.totalSupply();

        // Convert supply cap to wei and calculate available capacity
        uint256 supplyCapWei = supplyCap * 1e18;
        if (supplyCapWei > totalSupply) {
            availableCapacity = supplyCapWei - totalSupply;
        }
        // else returns 0 (pool is at or over capacity)
    }

    function _withdrawFromAave(uint256 amount) internal returns (uint256 withdrawnAmount) {
        if (amount == 0) return 0;

        uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
        if (aaveBalance == 0) revert InsufficientAaveBalance();

        // Only withdraw up to the principal amount (don't use accrued interest for user withdrawals)
        uint256 withdrawablePrincipal = aaveBalance < totalETHDepositedToAave ? aaveBalance : totalETHDepositedToAave;

        withdrawnAmount = amount > withdrawablePrincipal ? withdrawablePrincipal : amount;
        if (withdrawnAmount == 0) return 0;
