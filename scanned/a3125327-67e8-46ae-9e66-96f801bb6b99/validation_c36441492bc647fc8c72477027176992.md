### Title
Owner Can Drain KERNEL Tokens During Active Vesting Period Without Condition Checks - (File: `contracts/KERNEL/KernelTop100MerkleDistributor.sol`)

---

### Summary

`KernelTop100MerkleDistributor.withdrawTokens` allows the contract owner to withdraw any amount of KERNEL tokens at any time — including during the active 30-day vesting period — with no time-based or participation-based conditions. This is the direct analog of the IDO premature finalization pattern: a privileged function that can be invoked to unilaterally terminate a distribution process before users have had the opportunity to claim their entitled yield.

---

### Finding Description

`KernelTop100MerkleDistributor` distributes KERNEL tokens to eligible users over a fixed 30-day vesting schedule (`VESTING_DURATION = 30 days`). Users call `claim()` or `claimAndStake()` to receive their linearly vested allocation, verified by a merkle proof against `merkleRoot`.

The admin function `withdrawTokens` at lines 461–472 is:

```solidity
function withdrawTokens(address _token, uint256 _amount, address _recipient) external onlyOwner {
    UtilLib.checkNonZeroAddress(_token);
    UtilLib.checkNonZeroAddress(_recipient);

    if (_amount == 0) {
        revert ZeroValueProvided();
    }

    IERC20(_token).safeTransfer(_recipient, _amount);

    emit TokensWithdrawn(_token, _amount, _recipient);
}
```

There are **no conditions** preventing the owner from calling this with `_token = address(kernel)` at any point during the vesting window. Specifically, there is no check that:

1. `block.timestamp >= vestingStartTimestamp + VESTING_DURATION` (vesting period has ended), or
2. All eligible users have already claimed their full allocation.

Contrast this with `setVestingStartTimestamp`, which correctly guards against modification after vesting has started:

```solidity
if (vestingStartTimestamp > 0 && block.timestamp >= vestingStartTimestamp) {
    revert VestingAlreadyStarted();
}
```

No equivalent guard exists on `withdrawTokens`. The owner can drain the entire KERNEL balance mid-vesting, leaving all unclaimed user allocations permanently unclaimable. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

**High — Theft / Permanent Freezing of Unclaimed Yield.**

If the owner calls `withdrawTokens(address(kernel), kernel.balanceOf(address(this)), recipient)` at any point after `vestingStartTimestamp` but before `vestingStartTimestamp + VESTING_DURATION`, all KERNEL tokens are removed from the contract. Every user whose vesting has not yet fully elapsed loses their remaining unclaimed allocation permanently. Subsequent calls to `claim()` or `claimAndStake()` will revert with an ERC-20 insufficient-balance error, as the contract holds no KERNEL to transfer. [4](#0-3) [5](#0-4) 

---

### Likelihood Explanation

The function is callable by the owner at any time with no on-chain friction. The vesting period is exactly 30 days. Any point during that window — including day 1 — is a valid attack moment. The owner does not need to compromise any external system; the function is a standard admin entry point with no preconditions. [1](#0-0) 

---

### Recommendation

Add a guard to `withdrawTokens` that prevents withdrawal of the KERNEL token before the vesting period has fully elapsed, or restricts withdrawal to only the surplus above the total committed-but-unclaimed allocation:

```solidity
function withdrawTokens(address _token, uint256 _amount, address _recipient) external onlyOwner {
    UtilLib.checkNonZeroAddress(_token);
    UtilLib.checkNonZeroAddress(_recipient);
    if (_amount == 0) revert ZeroValueProvided();

    // Prevent draining the vesting token before the vesting period ends
    if (_token == address(kernel)) {
        require(
            block.timestamp >= vestingStartTimestamp + VESTING_DURATION,
            "Vesting period not ended"
        );
    }

    IERC20(_token).safeTransfer(_recipient, _amount);
    emit TokensWithdrawn(_token, _amount, _recipient);
}
```

---

### Proof of Concept

1. `KernelTop100MerkleDistributor` is deployed and funded with 1,000,000 KERNEL tokens. `vestingStartTimestamp` is set to `T`.
2. At time `T`, vesting begins. Users are entitled to claim linearly over 30 days.
3. At time `T + 1 day`, the owner calls:
   ```solidity
   withdrawTokens(address(kernel), 1_000_000e18, ownerAddress);
   ```
4. All KERNEL tokens are transferred out of the contract.
5. Any user calling `claim(amount, merkleProof)` now reverts because `kernel.safeTransfer(user, amountToSend)` fails — the contract balance is zero.
6. Users permanently lose 29/30ths of their entitled KERNEL allocation with no recourse. [1](#0-0) [6](#0-5) [4](#0-3)

```text
    function _getUnclaimedVestedAmount(address user, uint256 userTotalClaimableAmount) internal view returns (uint256) {
        UserClaim storage userClaim = userClaims[user];

        // If user has claimed everything, return 0
        if (userClaim.amountClaimed >= userTotalClaimableAmount) {
            return 0;
        }

        // Calculate vesting end time
        uint256 vestingEndTime = vestingStartTimestamp + VESTING_DURATION;

        // Calculate start and end times for the period
        uint256 startTime = userClaim.lastClaimTimestamp > 0 ? userClaim.lastClaimTimestamp : vestingStartTimestamp;

        // Cap current time at vesting end time
        uint256 currentTime = block.timestamp;
        if (currentTime > vestingEndTime) {
            currentTime = vestingEndTime;
        }

        // If current time is before start time or vesting hasn't started yet, nothing to claim
        if (currentTime <= startTime || currentTime <= vestingStartTimestamp) {
            return 0;
        }

        // Calculate total vested amount based on time elapsed since vesting start
        uint256 totalElapsedTime = currentTime - vestingStartTimestamp;
        uint256 totalVestedAmount = (userTotalClaimableAmount * totalElapsedTime) / VESTING_DURATION;

        // Cap at total amount
        if (totalVestedAmount > userTotalClaimableAmount) {
            totalVestedAmount = userTotalClaimableAmount;
        }

        // Calculate unclaimed amount
        uint256 unclaimedAmount = totalVestedAmount - userClaim.amountClaimed;

        return unclaimedAmount;
```
